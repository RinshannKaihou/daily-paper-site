ReSCom: A Reconfigurable Spiking Neural Network Accelerator Using Stochastic Computing

Alipour Fereidani · Roshanshah · Safari  —  University of Tehran  |  arxiv.org/abs/2606.13560

Key Finding

By replacing every multiplication in spiking neuron dynamics with a single AND gate (stochastic computing) while preserving exact fixed-point accumulation, ReSCom cuts per-image energy to 0.05 mJ — 16× lower than the next-best FPGA SNN accelerator — with only a 4.91% accuracy gap on MNIST (LIF model).

Why This Matters

Eliminates DSP multipliers entirely — enabling ultra-low-power spiking neural network inference on resource-constrained edge FPGAs where every millijoule and every LUT counts.

Critical Edge

Validated only on MNIST with a single 256-neuron hidden layer. 7.24 ms/image latency. Synaptic model suffers 7.64% HW–SW gap. Code not open-sourced.

IF — Integrate-and-Fire. Least SC-sensitive (0.24% gap). No decay multiplication.
LIF — Leaky IF. Moderate sensitivity (4.91% gap). One decay multiplication per timestep.
Synaptic — 2nd-order dynamics. Most sensitive (7.64% gap). Two stochastic multiplications + state feedback.

Single hardware core switches between all three modes via a 2-bit signal at runtime. Weights stored in Block RAM, time-multiplexed across neurons.

Strengths
  • Hybrid SC + fixed-point arithmetic suppresses error amplification in recurrent state updates
  • Single reconfigurable neuron core covers IF / LIF / Synaptic at runtime — no hardware duplication
  • Systematic bit-stream sweep (16–1024) maps the full precision–energy design space
  • Complete FPGA prototype with resource, power, and frequency data on Artix-7
  • AND-gate multiplier eliminates DSP blocks entirely — frees them for other tasks
Limitations
  • Only MNIST tested — generalisation to CIFAR-10 / ImageNet unknown
  • Single FC hidden layer (256 neurons) — scalability to CNN / RNN SNNs unproven
  • 7.24 ms/image latency — 34× slower than Carpegna's clock-driven design
  • SOTA energy advantage partly reflects newer FPGA process node
  • RTL and bitstream not released — independent reproduction difficult
  • LFSR seed / sequence correlation effects not analyzed
