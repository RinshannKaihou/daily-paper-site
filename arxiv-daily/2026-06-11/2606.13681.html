EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments

Jundong Xu, Qingchuan Li, Jiaying Wu, Yihuai Lan, Shuyue Stella Li, Huichi Zhou, Bowen Jiang, Lei Wang, Jun Wang, Anh Tuan Luu, Caiming Xiong, Hae Won Park, Bryan Hooi, Zhiyuan Hu

arXiv 2606.13681 ↗ 2026-06-11
Key Finding
Agents fail 60% of evolution steps. A git-like patch memory history recovers +3.7% chain accuracy and generalises to GAIA (+6.1%) and LoCoMo (+4.8%).
Key Metric
39.6%
Baseline step accuracy across EvoArena (Terminal · SWE · Persona). Chain accuracy drops to ~24%.
Why This Matters
Static benchmarks hide a 60% failure rate under real-world drift. EvoArena exposes it; EvoMem shows explicit memory-evolution tracking is a practical, agent-agnostic fix.
⚠ Critical Edge
Step gains are modest (+0.5% SWE). Some models degrade (−2.6% Gemini on SWE chain). No statistical significance testing reported.

EvoArena — Three Domains of Persistent Evolution
  • Terminal-Bench-Evo — 89 tasks → 352 versioned instances (mean chain 4.96 steps). Dominant change: I/O & protocol shifts (49%).
  • SWE-Chain-Evo — 12 active GitHub repos → 50 chains, 493 instances (mean 9.86 steps). 30% of milestones touch files modified by earlier steps.
  • PersonaMem-Evo — Synthetic persona preferences evolve across 5 change families, tested via single-transfer, synthesis, conflict, and temporal-trajectory questions.
EvoMem — Patch-Based Memory

Instead of overwriting memory, EvoMem keeps an append-only patch log for non-additive updates — a git-like diff trail:

  • τ Temporal metadata
  • C⁻ Pre-update memory state
  • C⁺ Post-update memory state
  • r Rationale for the change
  • z Semantic summary
  • e Triggering evidence

At inference the agent retrieves from both latest memory and patch history, concatenating both contexts.

Mechanistic Evidence
  • Patch reuse in agent reasoning → +8.3% gain vs +2.6% without uptake.
  • SWE PASS_TO_PASS regression drops from 9.09% → 6.32% (Kimi-K2.6: 7.14% → 3.33%).
  • Persona temporal-trajectory & multi-pattern synthesis each improve +5.2%.
  • Evidence-capture rate: 72.5% → 74.9% (temporal row capture +4.4%).
Strengths · Limitations

Strengths

  • Tackles a real deployment gap — static benchmarks miss environment drift.
  • Agent-agnostic patch abstraction works across 4 different agent frameworks.
  • Cross-domain & cross-benchmark improvements.
  • Chain-level metrics and regression analysis go beyond aggregate accuracy.

Limitations

  • Step-level gains modest, especially SWE (+0.5%). Not uniformly positive.
  • Semi-automated benchmark construction may introduce subtle biases.
  • Patch storage/retrieval cost at scale not thoroughly evaluated.
  • Mechanism analysis is observational, not causal.
Open Questions for Further Exploration

How does EvoMem scale to chains of hundreds of versions — does retrieval noise compound?

Can patch generation & retrieval be learned end-to-end rather than heuristic diff?

How robust is EvoMem to adversarial or contradictory environment updates?

Does it help in multi-agent settings where agents observe different subsets of changes?

What is the compute/storage overhead of full patch histories in production?

Can insights integrate with continual-learning or self-evolving agent systems?
