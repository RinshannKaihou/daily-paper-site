AgentBeats: Agentifying Agent Assessment for Openness, Standardization, and Reproducibility

Benchmarks become agents: judge agents evaluate subject agents via standardized A2A/MCP protocols in an open, reproducible framework.

Xiaoyuan Liu, Jianhong Tu, Yuqi Chen, Siyuan Xie, Sihan Ren, Tianneng Shi … +22 authors … Dawn Song, Yu Su, Wenbo Guo

LLM Agents Agent Evaluation Benchmarking A2A MCP
Key Metric
5.3 pp

Native model–harness pairings outperform swapped ones by 5.3 percentage points on average across 5 of 6 benchmark comparisons.

Why This Matters

Solves agent evaluation fragmentation with a protocol-first framework enabling fair, reproducible cross-model comparison at scale.

Critical Edge

Judge agent quality determines evaluation quality, yet no variance or inter-rater consistency is reported across different judge configurations.

Methodology

  • Delegator Initiates evaluation request
  • Judge Agent Orchestrates tasks, scores results
  • Subject Agent Receives tasks, calls tools, returns output
  1. Delegator sends kick-off via A2A
  2. Judge prepares environment (MCP server) & distributes task
  3. Subject fulfills task within time limit
  4. Judge revokes access, inspects state, scores
  5. Judge reports results to delegator

Assessment

Strengths

  • Reduces integration cost from N×M to N+M via protocol-level unification
  • Five practical deployment modes covering dev to CI audit
  • Community-validated at unprecedented scale (5-month open competition)
  • Harness-swapping experiment provides novel co-adaptation insight

Limitations

  • No confidence intervals or statistical significance tests reported
  • Judge quality variance unmeasured — no inter-rater analysis
  • Harness-swapping used only 20% subsamples for cost reasons
  • No systematic discussion of reward hacking or collusion defenses

Open Questions

  1. How much do different judge agents (model, budget, tools) affect ranking stability? Can a meta-evaluation protocol be established?
  2. What mechanisms can prevent reward hacking, collusion, and data contamination in open competitions?
  3. Can the latency, token-cost, and deployment overhead of AAA vs. traditional harnesses be systematically quantified?
  4. Does the observed model–harness co-adaptation extend beyond coding to browser, computer-use, or multimodal agents?
