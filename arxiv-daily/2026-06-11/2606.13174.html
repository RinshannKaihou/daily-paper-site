TRACE

Getting Better at Working With You: Compiling User Corrections into Runtime Enforcement for Coding Agents

📅 2026-06-11 🏷 LLM agents · Runtime safety · Personalization arxiv.org/abs/2606.13174
一句话总结

TRACE 将用户自然语言纠正实时编译为原子规则，并注入编码智能体的运行时检查，把"记住偏好"升级为"强制遵守偏好"。
TL;DR

A drop-in skill layer that mines user corrections → compiles atomic rules → enforces runtime checks before a coding agent can complete a task. Remembering isn't enough — preferences must be executable constraints.
Key Finding
Compiled enforcement of user corrections slashes preference violation by 98 percentage points on out-of-distribution coding tasks — without degrading task success.
Key Metric
100%→2%
OOD violation rate on ClawArena (4-model avg)
Why This Matters
Memory-based personalization alone leaves a massive access ≠ compliance gap. Preferences must be enforced as runtime constraints, not just recalled.
Critical Edge
Only helps when a compiled rule exists. MemoryArena OOD violation stays at 97%. No user study on blocked-task acceptability.

Assessment
Strengths
  • Targets a concrete daily-use failure mode with a clear engineering solution
  • Strong OOD compliance (2% violation) without task-pass degradation
  • Low runtime overhead — 42.5s mean, within seconds of no-memory baseline
  • Modular drop-in skill for Claude Code & Codex CLI; open-source code + skill
  • Reduced user turns (1.02 OOD) — enforcement actually saves interaction cost
Limitations
  • Single-user diagnostic corpus; multi-user preference conflicts untested
  • MemoryArena OOD violation stays at 97% — only helps when rule exists
  • Semantic/intent enforcement tiers described but not empirically validated
  • Conflict resolution relies on LLM resolver + manual audit; no auto-rollback
  • No user study on blocked-task acceptability or long-term rule management
