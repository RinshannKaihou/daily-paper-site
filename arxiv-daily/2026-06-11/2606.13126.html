MiniPIC: Flexible Position-Independent Caching in <100 LOC

Store unrotated K vectors in the KV cache. Apply RoPE per-request inside attention. Three user-level primitives replace entire forked codebases.

Nathan Ordonez Thomas Parnell 2026-06-11 arxiv.org →
Key Metric · Throughput
+49%
48.01 vs 32.21 samples/s on 2WikiMultihopQA (H100, Llama-3-8B). Up to 100× lower cached-span TTFT. Overhead when disabled: only 5.7%.
Why This Matters
RAG and agentic workloads re-encode the same documents, tool descriptions, and memory states thousands of times. MiniPIC eliminates that waste with 78 lines of code — no new allocator, no block-copy loops, no forked runtime.
Critical Edge
Correctness shifts to the caller. Wrong padding or marker placement → silent false cache hits. Only evaluated on one dataset. Span attention is an approximation of full causal attention — accuracy impact unmeasured.

Core Mechanism · Position-Free KV Cache
// Standard: K rotated before entering cache → position conflict on shared blocks // MiniPIC: store K̃ (unrotated), apply RoPE inside attention per-request score(qₜ, k̃ⱼ) = (Rₜ qₜ)ᵀ · (Rπ(j) k̃ⱼ) / √d // Same physical KV block ⟶ multiple concurrent requests at different logical positions // No copy. No reorder. No allocator change. // Kernel cost: cos/sin lookup + 2 fused multiply-adds on K tile (linear in O(n²) attention)
① Block-aligned Padding
User pads each span to a KV-block boundary so subsequent special tokens reliably land at block start. Enables deterministic block hashing.
② SS EP · Span Separator
Detaches current block's hash from its prefix chain. The span can be cached and reused independently — supports Block-Attention, EPIC, Prompt Cache in one engine.
③ PD EP · Prompt Depend
Forces current block hash to depend on the entire preceding prompt. Prevents suffix blocks (e.g. query instructions) from getting false cache hits by detaching from their prefix.
Strengths
  • 78 LOC core-engine changes (61 new) — negligible maintenance burden
  • Block-Attention, EPIC, Prompt Cache coexist in one vLLM instance
  • Native CPU-offload compatibility, no extra host↔device transfers
  • Eliminates SPNL/CIDRA block copies, reorder loops, and cycle detection
  • ISPS scheduling removes barrier idle time between span-prefill and final prompt
Limitations
  • Single benchmark dataset (2WikiMultihopQA) — generalization unproven
  • Primitives require caller expertise in span boundaries and causal structure
  • RoPE-only; no support for ALiBi or NoPE models
  • Span attention approximates full causal attention — accuracy trade-off unmeasured
  • No cache eviction strategy under heavy concurrency; hit rates unanalyzed
  • No direct comparison with MEPIC
