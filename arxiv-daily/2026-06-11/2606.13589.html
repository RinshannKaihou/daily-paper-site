Compression rate 96%

Resolving the L1-Simplex Paradox via Concave Quadratic Sparsity

SCSB is a model-agnostic post-training framework that unifies ensemble pruning and probability calibration into a single constrained optimization over the probability simplex — achieving up to 96% compression while preserving or improving accuracy and calibration.

Ensemble Learning Model Compression Probability Calibration 2606.13589 2026-06-11
Key Finding

On the probability simplex, ‖w‖₁ ≡ 1 is constant — Lasso cannot induce sparsity. Replacing L1 with the concave penalty −λ‖w‖₂² forces local minima to simplex vertices, yielding exact zeros.

Key Metric

2 / 50 estimators retained on cpu_act (Ridge) — R² improves from 0.7293 → 0.7323. Across all benchmarks: 68–96% pruning.

Why This Matters

Deploy bagging ensembles to latency-sensitive and resource-constrained environments (edge, real-time) without retraining — up to 5.7× inference speedup measured.

⚠ Critical Edge

Non-convex objective → no global optimality guarantee. Title mentions "Bagged Neural Networks" but no neural-network experiments appear in the paper.

How It Works

  1. 1 Start with a pre-trained bootstrap ensemble H = {f₁, …, fₙ}.
  2. 2 Build leakage-free predictions using Out-Of-Bag (OOB) indicators — no extra validation split needed.
  3. 3 Optimize the joint objective over ∆N: min OOB-Loss − λ‖w‖₂², subject to w ≥ 0, Σwⱼ = 1.
  4. 4 The concave penalty drives weights to simplex vertices (exact zeros). Solved via SLSQP with analytical gradients.
  5. 5 Discard zero-weight estimators; keep the sparse weighted subset for inference.

Strengths & Limitations

Strengths

  • Clean math: unifies pruning + calibration in one objective
  • Correctly identifies and resolves the L1-simplex paradox
  • Leakage-free via OOB — no extra validation set needed
  • Model-agnostic in principle; analytical gradients enable efficient SLSQP
  • Strong compression & speedup on tabular benchmarks

Limitations

  • Non-convex: results depend on initialization & local optimizer
  • Only tested on small/medium tabular data and classical estimators
  • Title claims "bagged neural networks" — no NN experiments shown
  • Hyperparameter λ selection procedure not discussed in detail
  • Some configurations show worse ECE than standard bagging

Open Questions

  • How does SCSB perform on deep ensembles and large-scale benchmarks?
  • Is there a principled way to select the λ hyperparameter?
  • Can simplex-projected SGD replace SLSQP for web-scale data?
  • How does SCSB compare to ensemble distillation or temperature scaling?
  • Would input-dependent weights wⱼ(x) improve local calibration without overfitting?
