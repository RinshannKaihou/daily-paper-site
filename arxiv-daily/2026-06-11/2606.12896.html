PolicyGuard: Test-time Step-level Adversary Defense for RL Agents

Junfeng Guo, Heng Huang

First practical black-box defense that detects backdoors in RL agents at test time, step by step — remaining effective (AUROC 0.868) even against hard-coded attacks where every prior defense collapses to random chance.
Key Metric
0.868
AUROC on hard-coded attacks — vs ≈ 0.5 for Neural Cleanse, STRIP, SCALE-UP, and PolicyCleanse.
Why This Matters
Deployed RL agents can carry hard-coded adversarial policies invisible to parameter-level inspection. PolicyGuard detects them using only observed state-action pairs — no model access, no future trajectory.
Critical Edge
Clean-env assumption · Detection only · Sim only
Requires a clean environment for GP training. Does not mitigate. Evaluated only in Atari/MuJoCo simulation.
Context
7 RL games · 2 attack paradigms + hard-coded variant
Method: Additive Gaussian Process on learned behavior manifold + adaptive pseudo trajectories
Venue: ICML 2026

Methodology

1
Behavioral manifold learning. Collect trajectories from a clean environment (black-box). Encode state-action pairs: CNN + GRU for Atari, MLP + GRU for MuJoCo. Project final hidden state → episode embedding.
2
Additive GP with deep recurrent kernels. Predict episode reward as f = α_t·f_t + α_e·f_e — timestep correlations via SE kernel + episode-level patterns via SE kernel. Train with sparse variational inference (inducing points), end-to-end.
3
Step-level uncertainty via pseudo trajectories. At test time, concatenate observed history up to t with future segments from held-out reference trajectories. Feed each pseudo trajectory through encoder + GP, extract posterior variance, aggregate across trajectories using Interquartile Mean. Backdoor steps → elevated variance.

Strengths

  • First test-time, step-level RL backdoor defense working in black-box settings without future trajectories.
  • Robust to hard-coded attacks that break all optimization-based defenses.
  • Theoretically grounded: benign posterior variance → 0 with sufficient clean data (Theorem 3.1).
  • Broad evaluation: 7 games, 2 attack paradigms, adaptive attacks, ablations.
  • ~9% faster inference than SHINE.

Limitations

  • Requires clean environment for GP training — non-trivial for real-world deployment with environment shift.
  • Detection only; no built-in mitigation or intervention.
  • Evaluated only in simulation (Atari / MuJoCo); sim-to-real gap unknown.
  • GP inference + pseudo-trajectory Monte Carlo adds per-step computational overhead.
  • Not tested on continuous-action or partially observable environments.

Relevance

对强化学习安全、后门攻击检测以及对抗鲁棒性研究具有直接参考价值；PolicyGuard 的"测试时+逐时间步+黑盒"范式为在线部署的 RL 系统提供了一条不依赖模型内部参数的实用防御路径，尤其适合需要早期发现异常行为的安全关键场景。

Open Questions

How would PolicyGuard perform in continuous-control or partially observable environments where trajectory manifolds are more complex?
Can the clean-environment requirement be relaxed through domain adaptation or sim-to-real transfer?
What is the most effective intervention strategy to pair with detection — safe defaults, input sanitization, or human oversight?
How does the method scale to very long episodes or high-dimensional observations (e.g., autonomous driving pixel streams)?
Can adaptive attackers craft triggers that lie exactly on the learned clean manifold, and what inductive biases would detect them?
