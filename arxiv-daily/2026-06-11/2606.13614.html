Majority-of-Three is Optimal

Divit Rawal & Nikita Zhivotovskiy · UC Berkeley · 2026-06-11 · arXiv:2606.13614

The majority vote of three independent consistent classifiers—each trained on an equal third of the sample—is an optimal learner in the realizable PAC setting, with a simpler proof than any prior optimal learner. 三分样本、训练三个独立一致分类器并做多数投票，即可达到最优 PAC 样本复杂度，且证明比此前所有最优学习器更简洁。
Key Finding

Majority-of-three—the simplest possible voting scheme—is already an optimal PAC learner. No recursive majority, no bagging, no one-inclusion graph machinery.

Key Metric
107

Why This Matters

Collapses the gap between intuitive ensemble heuristics and optimal PAC theory. The simplest voting scheme is theoretically sufficient—no complex machinery required.

Critical Edge

Realizable setting only. No agnostic or noisy-label extension. The constant 107 is far from tight. Block splitting wastes ⅔ of data per base learner. No empirical experiments.

MAJ
  • Split n i.i.d. samples into three equal, independent blocks S₁, S₂, S₃.
  • Train one deterministic consistent classifier on each block—every learner is correct on its own block.
  • Vote via majority: output the label agreed upon by at least two classifiers.

The Bound

P( errP(Maj(f̂₁, f̂₂, f̂₃)) ≤ 107 ·  d + log(1/δ) n  ) ≥ 1 − δ
  • Matches the classic Ehrenfeucht–Haussler–Kearns–Valiant lower bound up to a universal constant.
  • No extra log or log log factors—the first such result with the simplest possible algorithm.
  • Holds for any VC class of dimension d, any distribution P, any target in the class.

Proof at a Glance

  1. Error-set overlap. If the majority errs, at least 2 of 3 classifiers err—reduces to bounding pairwise error-set intersections.
  2. Two-block moment bound. For independent blocks S, T: ‖P(A(S) ∩ A(T))‖Lq ≤ 62 400 (d+q)/m.
  3. Chain rule + VC tail. Rewrite overlap moments as squared inclusion probabilities; control recursively via VC dimension.
  4. High-probability conversion. Set q = ⌈log(1/δ)⌉, apply Markov → PAC bound with constant 107.

Trade-offs

Strengths
  • Extremely simple learner: 3 classifiers + majority vote
  • Removes the residual log-log factor left by AAHLZ '24
  • Explicit constants; compact, readable proof (9 pages)
  • Strong conceptual point: simplest scheme is theoretically sufficient
  • Builds cleanly on prior work with proper attribution
Limitations
  • Realizable PAC only; no agnostic or noisy-label extension
  • Requires exact consistency of each base classifier
  • The constant 107 is far from tight; likely improvable
  • Independent block splitting wastes ⅔ data per base learner
  • No empirical experiments provided

Open Questions

  1. Can the argument extend to the agnostic PAC setting, or does the lack of a realizable target break the error-set overlap analysis?
  2. How much can the leading constant 107 be reduced by tighter moment bounds or different proof techniques?
  3. What happens with approximately consistent base classifiers, or overlapping subsamples instead of disjoint blocks?
  4. Does majority-of-three remain optimal beyond binary classification—e.g., multiclass or real-valued prediction?
  5. Could this insight yield a practical algorithm competitive with bagging or boosting while retaining a simple analysis?
