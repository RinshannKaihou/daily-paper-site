Sparse Autoencoders Interpretability · arXiv 2606.27321 · 2026-07-15 · Vision Foundation Models

Beyond the Hard Budget

Sparsity regularizers for more interpretable Top-k sparse autoencoders.

Key finding

Two sparsity regularizers, dropped into Top-k SAEs, lift feature monosemanticity and small-budget probing — without sacrificing reconstruction.

Nathanaël Jacquier · Maria Vakalopoulou · Mahdi S. Hosseini  —  Paris-Saclay / CentraleSupélec & Concordia · Mila  ·  arxiv.org/abs/2606.27321 ↗

02 Why this matters

Cheap drop-in regularizers that make Top-k SAE features cleaner and small-budget probing stronger — correcting the field's default of discarding sparsity once Top-k is adopted.

Hard architectural sparsity & soft regularization → complementary.
03 Critical edge

Largest gains land on ViT-L/16, whose baseline monosemanticity is unusually low; on SigLIP2 the lift is modest and occasionally negative.

Generalization to already-clean encoders is not demonstrated.

M Methodology

x ∈ ℝᵈ → encoder → π = W_e(x−b_d)+b_e → ReLU → a ∈ ℝ₊ᵐ → Top-k → z → decoder → x̂ = W_d z + b_d  (m=8192 latents, m≫d; unit-ℓ2 decoder cols; aux Top-k loss L_aux)
Frozen VFM embeddings in, sparse code out. Both regularizers act before Top-k selection, on batch-active units only (mask μ).

Regularizer 1 off-support ℓ1

R₁ = (1/N) Σ ‖ μ ⊙ (a − z) ‖₁

The residual lives exactly on units not picked by Top-k. Driving it to zero sharpens each selected unit's selectivity.

  • Payoff: raw monosemanticity & class purity (largest gains).
  • Side effect: fewer dead neurons (SigLIP2 k=128: 120 → 73).

Regularizer 2 ℓ1/ℓ2 ratio

R₂ = (1/N) Σ ( ‖μ ⊙ a‖₁ / ‖μ ⊙ a‖₂ )

Scale-invariant (Hoyer, 2004); its square proxies the effective active count, so minimizing it concentrates the code onto fewer units.

  • Payoff: k-robustness + front-loaded probing (CLIP k=128 AUC 0.488 → 0.693).
  • Side effect: more dead neurons (modest, acknowledged trade-off).

Objective: L = L_recon + α L_aux + λ R, λ=0 recovers baseline. λ chosen per config for max validation-monosemanticity gain at parity R²; trade-off shown broad & smooth.

+ Strengths

  • Clean central thesis — hard vs soft sparsity as complements, backed by a tidy matched-pair design (only λ varied).
  • Two mechanisms, two payoffs — Reg.1 for raw interpretability, Reg.2 for k-robustness & probing.
  • Honest reporting — SigLIP2 regressions and Reg.2 dead-neuron trade-offs explicitly called out.
  • Decisive ablations — active-unit mask & off-support restriction both defended with order-of-magnitude effects.
  • Qualitative grids complement embedding-similarity metrics, addressing the known proxy concern.

! Limitations

  • Baseline = vanilla Top-k only — no JumpReLU, BatchTopK, or Matryoshka head-to-head.
  • Gains uneven — largest on the low-baseline ViT-L/16; SigLIP2 occasionally negative under Reg.2.
  • Class purity weak in absolute terms (binary 0.065–0.285) — above chance, but far from truly class-aligned.
  • Reg.2 inflates dead neurons (SigLIP2/OI k=32: 196 → 260) — possible coverage loss at scale.
  • No code link in main text; no downstream task beyond linear probing (no OOD / safety auditing).
  • Per-config λ tuning on validation set complicates drop-in deployment claims.
