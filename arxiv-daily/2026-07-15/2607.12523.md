> **一句话总结：** 本文提出 OOD-RL-Bench，一个可扩展的强化学习轨迹 OOD 检测基准框架，集成 8 个检测器与 9 种异常注入器（post-hoc 与 online 混合协议），在 LunarLander-v3 上以 matched-time AUROC/AUPRC/FPR0.95/检测延迟/零基线等多维度指标评估，发现观测扰动与分布切换易被检出，而观测延迟与动作条件动力学仍是最难的检测通道。

> **TL;DR:** OOD-RL-Bench is an extensible benchmark that evaluates eight OOD detectors (mahalanobis, time-series features, PEDM, autoencoder, ensemble disagreement, CUSUM, KDE, Lyapunov) against nine anomaly injectors on RL trajectories, using a mixed post-hoc/online protocol with matched-time AUROC, AUPRC, FPR@95TPR, detection delay, and a null-baseline diagnostic; on LunarLander-v3, PEDM leads aggregate matched-time AUROC (0.874 vs. 0.839 for time-series features), but observation-delay (best 0.697) and action-conditioned-dynamics (best 0.636) remain hard, and a single best detector never dominates all anomaly families.

## 核心贡献

- 提出一个可扩展的 RL-OOD 基准框架 `ood_rl_bench`，默认配置 8 个检测器与 9 个异常注入器，新检测器/注入器/度量/环境只需通过统一接口与 YAML 配置注册，无需修改核心代码。
- 给出七类检测器分类法：marginal-state（Mahalanobis、KDE）、temporal-feature（time-series features）、transition-model（PEDM）、reconstruction（autoencoder）、ensemble-uncertainty（ensemble disagreement）、sequential-change（CUSUM）、nonlinear-dynamics（Lyapunov divergence）。
- 提出混合异常协议：观测/时序/分布类异常走 post-hoc 注入（保留字节级一致的 clean rollout），动作/动力学类异常走 online rollout（保留闭环 policy-environment 交互的真实状态访问分布）。
- 设计 matched-time 评测协议：将异常 post-onset 分数与同一时间步的 clean 分数配对比较，消除"分数随时间漂移"带来的虚假判别力，并配套 null-baseline、FPR@95TPR、检测延迟、segment 指标的报告模板。
- 在 LunarLander-v3 上以 NoisyNet DQN（~230 mean return）给出完整可复现的示例实验，并公开框架、policy checkpoint、配置与全部 per-condition 结果。

## Key Contributions

- An extensible RL-OOD benchmark framework (`ood_rl_bench`) shipping eight detectors and nine anomaly injectors by default, where new detectors, injectors, metrics, and environments are added purely through shared interfaces (`BaseDetector`, `BaseAnomalyInjector`) and YAML registration, leaving orchestration/metric aggregation untouched.
- A detector taxonomy spanning seven signal families: marginal-state (Mahalanobis, KDE), temporal-feature (time-series features), transition-model (PEDM), reconstruction (autoencoder), ensemble-uncertainty (ensemble disagreement), sequential-change (CUSUM), and nonlinear-dynamics (Lyapunov divergence).
- A mixed anomaly-execution protocol: observation / temporal / distributional anomalies are injected post-hoc on a byte-identical clean rollout, while action and dynamics anomalies (stuck actuator, dynamics scaling, action-conditioned dynamics, gradual drift) are applied via live rollouts so the policy's closed-loop state-visitation distribution remains consistent with the executed fault.
- A matched-time evaluation protocol that pairs each post-onset anomalous score with the clean score at the same timestep (same seed/mode), plus a null-baseline diagnostic, operating-point FPR@95TPR, detection-delay, and segment-level metrics; this exposes temporal-position artefacts that inflate raw AUROC.
- A reproducible worked example on LunarLander-v3 with a NoisyNet DQN policy (~230 mean return, "solved"), releasing the framework, checkpoint, configs, and full per-condition YAML results.

## Methodology

**Trajectory contract.** A fixed-length trajectory τ = {(s_t, a_t, r_t, s_{t+1}, d_t)} is the only data structure detectors see; a detector D returns an anomaly-score sequence z_{0:T-1} after being fit on in-distribution trajectories. Detectors never touch the simulator at scoring time.

**Eight detectors.** Mahalanobis distance (covariance-normalised feature distance with ridge λI); time-series features (rolling-window descriptors scored by an ID distance/z-score); PEDM (probabilistic ensemble action-conditioned transition model p_θ(s_{t+1}|s_t,a_t), scored by minimum sampled one-step prediction error, threshold = max held-out ID validation score); autoencoder (reconstruction error); ensemble disagreement (variance across M bootstrapped transition predictors); CUSUM (one-sided cumulative deviation from an ID reference); KDE density (negative log non-parametric density); Lyapunov divergence (local nearest-neighbour divergence in reconstructed phase space).

**Nine anomaly injectors.** Post-hoc (5): gaussian state noise, state feature dropout (zero coords w.p. 0.8σ), correlation break (permute selected coords across time), regime switch (displace marginal by 2σ, amplify variance by 1+2σ), observation delay (substitute state from k=⌈σ·10⌉ steps prior). Online (4): stuck actuator (fix executed action w.p. σ), dynamics scaling (multiply state increments by 1+1.5σ), action-conditioned dynamics (scale only when a_t=a*=2, main engine), gradual drift (ramp scale 1.0→1+1.5σ over 200 steps). Each injector exposes σ∈[0,1].

**Evaluation protocol.** Fit on N_ID=30 ID trajectories; evaluate N_OOD=20 perturbed trajectories per injector; T=500 steps; onset t*=⌊T/3⌋=166. For each perturbed episode, anomalous post-onset scores are paired with clean scores from the same timestep of a same-seed counterfactual rollout to form the matched-time positive/negative sets. Reported: matched-time AUROC, AUPRC (balanced no-skill ≈0.5), FPR@95TPR, F1, detection delay, segment precision/recall, plus raw-trajectory metrics and a null baseline (clean rollout scored against a counterfactual post-onset label vector) yielding null-adjusted AUROC. Per-episode bootstrap CIs over paired episodes. Severity sweeps available per injector.

**Environment/policy.** LunarLander-v3 via Gymnasium, 8-dim flat state (x,y,v_x,v_y,θ,ω,c1,c2), 4 discrete actions; NoisyNet DQN MLP value head, 210k steps, batch 64, γ=0.99, lr 5e-4, replay 1e5, target sync every 1k; final mean return ≈230. Python 3.13.9, PyTorch 2.9.1, NumPy 2.4.0, MPS.

## Results & Highlights

- **Aggregate matched-time AUROC across the 9 anomalies (Table 4):** PEDM leads at **0.874** (AUPRC 0.851), beating time-series features 0.839, KDE density 0.730, ensemble disagreement 0.720, autoencoder 0.705, Mahalanobis 0.649, Lyapunov divergence 0.539, CUSUM 0.495 (≈chance).
- **Per-anomaly difficulty (Table 5, best detector AUROC):** gaussian state noise **0.999** (tsf), state feature dropout **0.998** (tsf), correlation break **0.999** (tsf), regime switch **0.997** (kde) are near-saturated; stuck actuator **0.919** (kde), dynamics scaling **0.927** (pedm), gradual drift **0.869** (pedm); observation delay only **0.697** (pedm) and action-conditioned dynamics only **0.636** (pedm) remain the hardest channels.
- **Severity sweep for dynamics scaling with PEDM (Table 6):** matched-time AUROC climbs from **0.608 at σ=0.10** to **0.927 at σ=0.80**, while FPR@95TPR drops from **0.695 to 0.170** — demonstrating monotonic detectability with perturbation magnitude.
- **Detection-delay vs. ranking trade-off (Tables 7–8):** time-series features has the lowest mean raw-threshold delay (49.2 steps) but PEDM, despite the best AUROC, has the highest delay (271.9 steps) because its threshold is the max held-out validation score; for dynamics scaling, tsf reaches AUROC 0.899 at 26.9-step delay vs. PEDM's 0.927 at 297.3-step delay.
- **Operating-point FPR@95TPR (Table 9):** PEDM lowest aggregate at **0.378**, tsf 0.384; CUSUM worst at **0.806**; the hard channels (observation delay, action-conditioned dynamics) retain FPR≥0.846 even for the best detector.
- **Null-baseline exposes temporal artefact (Table 11):** Lyapunov divergence has a raw counterfactual null AUROC of **0.812** on clean trajectories, explaining why its raw AUROC of 0.901 collapses to matched-time 0.539 — its apparent skill is almost entirely a temporal-position artefact, not same-time anomaly separation.

## Authors & Credibility

Emil Mittag and Richard Dazeley are at Deakin University (Geelong, Australia); Peter Vamplew is at Federation University Australia. The authors have prior footing in the safe/uncertainty-aware RL literature they cite (PETS, deep ensembles, Haider et al. probabilistic dynamics OOD, DEXTER/Nasvytis et al., Danesh & Fern OOD dynamics). The work is positioned honestly as a *framework* contribution rather than a new SOTA detector, and the paper repeatedly distinguishes the benchmark's matched-time protocol from a deployable alarm. Code, checkpoint, configs, and per-condition YAML results are released at github.com/ood-rl-bench/ood-rl-bench, with provenance captured in run_meta.yaml.

## Critical Assessment

- **Novelty:** Genuine but incremental. The paper explicitly does not claim to be the first RL-OOD benchmark (it cites Mohammed & Valdenegro-Toro 2021, Danesh & Fern 2021, DEXTER 2024); the novelty is the *matched-time evaluation protocol* and the clean separation of post-hoc vs. online anomaly channels with a unified trajectory contract. The eight detectors are all well-known off-the-shelf methods — none is new.
- **Rigor:** Methodologically careful. The null-baseline / matched-time design is the strongest methodological contribution: the Lyapunov case (raw 0.901 → matched-time 0.539) is a concrete demonstration of how raw trajectory AUROC inflates via temporal score drift, and the paper does not hide this. Reporting covers AUROC, AUPRC, FPR@95TPR, detection delay, segment precision/recall, and null-adjusted AUROC with bootstrap CIs.
- **Reproducibility:** High. Deterministic seeding across environment/policy/data/detector init, fixed N_ID=30 / N_OOD=20, pinned software versions (Python 3.13.9, PyTorch 2.9.1, NumPy 2.4.0), human-readable YAML outputs, and a public GitHub repo with the trained checkpoint.
- **Significance:** Moderate. As a *framework* it is useful and well-engineered; as *evidence* about OOD detection in RL it is thin because all numbers come from a single low-dimensional (8-dim flat-state) environment with a single solved policy.
- **Red flags / weaknesses:**
  - **Single environment (LunarLander-v3) and single policy.** All headline numbers are from one 8-dimensional discrete-control task; the authors flag this themselves (Limitation i). Generalisation to image-based, partially observed, or high-dimensional continuous-control environments is unvalidated.
  - **Tiny evaluation sample: N_OOD=20 episodes per injector.** Per-condition numbers rest on 20 episodes with bootstrap CIs; some CIs (e.g., Mahalanobis null AUROC SD 0.219) are wide enough that several detector rankings are not statistically separable — the paper does not run significance tests between detectors.
  - **No detector-vs-detector significance testing.** Rankings like "PEDM 0.874 vs tsf 0.839" are presented without pairwise tests, so the implied ordering may be noise.
  - **Online anomalies are LunarLander-specific proxies** (e.g., dynamics scaling multiplies state increments in the simulator; action-conditioned dynamics targets the main-engine action a*=2). Portability to other simulators requires bespoke hooks (Limitation ii).
  - **PEDM "stuck actuator" interpretation caveat:** because the logged action is the *commanded* action, PEDM does not directly score the commanded-vs-executed mismatch; KDE wins stuck-actuator only via downstream state-displacement. The paper discloses this, but it means the benchmark does not yet cleanly measure causal actuator-fault detection.
  - **Matched-time protocol is an evaluation protocol, not a deployable detector** (Limitation iii) — the paper is honest, but readers should not treat matched-time AUROC as a deployment alarm rate.

## Strengths

- Clean, well-motivated taxonomy of detector signal families mapped to anomaly channels, with expected-vs-observed behaviour explicitly revisited in Section 9.4 (e.g., CUSUM fails to win gradual-drift under matched-time AUROC despite being conceptually suited).
- The matched-time + null-baseline protocol is a real methodological improvement over raw trajectory AUROC and should be adopted more broadly; the Lyapunov counter-example is pedagogically excellent.
- Genuine extensibility: adding a detector/injector/metric/environment each touches only one registry/config surface, demonstrated concretely by the four-artefact LunarLander integration.
- Honest framing throughout — the paper repeatedly distinguishes ranking quality, operating-point FPR, and latency, and refuses to crown a single winner.
- Strong reproducibility artefacts (YAML results, run_meta provenance, public checkpoint).

## Limitations

- All quantitative results come from a single environment (LunarLander-v3) and a single trained policy; cross-environment claims are unvalidated, especially for image-based / POMDP / high-dimensional continuous control.
- N_OOD=20 episodes per injector is small; several detector differences are within bootstrap CI noise and no pairwise significance tests are reported.
- Online actuator/dynamics injectors are simulator-specific and require per-environment reimplementation; conceptual portability is asserted but not demonstrated.
- The benchmark scores trajectories offline after collection, so results are mixed-mode trajectory-detection outcomes, not a real-time fault-response system; matched-time AUROC is not a deployment alarm rate.
- PEDM's conservative threshold (max validation score) gives it the worst detection delay despite the best AUROC, illustrating a calibration gap the benchmark exposes but does not resolve.
- Only flat-state observations are tested; visual/pixel-based RL — where many real OOD challenges live — is out of scope here.

## Relevance

对关注强化学习安全性与可信部署的研究者而言，该工作提供了一个可直接复用的 OOD 检测评测骨架（matched-time 协议 + 零基线诊断 + post-hoc/online 混合注入），尤其适合用来审慎评估自家 RL agent 在传感器故障、动力学漂移、动作异常下的检测能力，而非轻信单一 AUROC 数字。其方法学贡献（消除时序位置伪影的评测协议）对 OOD 检测、异常检测与时序基准设计都有借鉴价值。

## Questions for Further Exploration

- Does the detector ranking (PEDM > tsf > KDE ...) survive on image-based or partially observed environments (e.g., CarRacing, pixel-CartPole, DM Control vision suites), or is it an artefact of LunarLander's low-dimensional flat state?
- With N_OOD=20 and bootstrap CIs as wide as ±0.22, how many of the reported detector-ordering claims are statistically significant under a proper paired test (e.g., paired bootstrap or Wilcoxon on per-episode AUROCs)?
- Can the matched-time + null-baseline protocol be adopted as a standard reporting requirement for *supervised* time-series OOD benchmarks (TSB-UAD, TAB) to similarly expose temporal-position inflation?
- How would a detector that directly scores the commanded-vs-executed action mismatch perform on the stuck-actuator row, given that PEDM currently only sees the fault indirectly through transition error?
- Is there a single threshold-calibration rule that recovers PEDM's ranking quality without its 271.9-step detection-delay penalty (e.g., conformal thresholds, sequential probability ratio tests)?
- Do the hardest channels (observation delay 0.697, action-conditioned dynamics 0.636) become tractable with detectors that explicitly model temporal/action context beyond a one-step transition (e.g., recurrent world models, multi-step prediction error)?
