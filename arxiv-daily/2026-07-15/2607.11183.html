Critical edge probe this

Offline-only evidence. Learned gates are validated only within existing CSV pools with a stable hash split — no prospective online A/B, no held-out dataset — so the +4.27 pp and +11.36 pp figures read as offline upper-bounds, and parameter overfitting is not fully excluded.

01 Methodology

A negative result becomes the design rule. SVD-based direction rotation breaks more than it fixes; AG keeps every FFN weight matrix frozen and only shrinks activation magnitudes.

ORP · falsified
Rotate FFN residual direction
SVD projection at P1/P2/P3. On a 1,032-q GSM8K audit: 34 fixes / 56 harms (net −22); 1,012/1,032 token paths diverge.
Design rule
Don't touch weight directions
Pretrained FFN directions encode semantic priors. Intervene on magnitude, never on orientation.
AG family
Mask / shrink activation energy
Hard L2 prefix-mask, soft shrinkage, top-k, local-norm, NMF-style, and Newton-Schulz-windowed (NS) entropy ranking. e = retained mass.
Deployable gate
Route, don't replace
Learned policy uses only online features (prefix/format/schema, logprob trajectory, baseline uncertainty). Baseline always available as fallback.

SwiGLU six-point intervention taxonomy

P1
FFN input (inherited)
P1s
gate-branch-only input
P2
post-fusion internal
P2a
post-SiLU, pre-fusion
P2b
post-up-proj, pre-fusion
P3
FFN output (deploy-first)

Deployment priority: P3 > P1s > task variants. P2/P2a/P2b are research/oracle points (4×-hidden cost, noisy).

Four-level evaluation protocol

  • Combination oracle — per-sample best non-baseline candidate; a label-leaking upper bound only.
  • Best fixed single config — one e/point for the whole set.
  • Learned gate — deployable; restricted to online-observable features.
  • Diagnostic-only — post-fork / label-derived features explicitly forbidden in deployable claims.

Names and forbids three overclaiming traps: pooled multi-config bias, best-selected bias, row-level bias.

03 Strengths & limitations, side by side

The paper's standout virtue is intellectual honesty — a failed method is the centerpiece, and the strongest negative result is reported in full.

Strengths

  • Failed method as hero. ORP is the conceptual centerpiece; the Qwen2.5-7B null result is reported in full with CIs rather than buried.
  • Reusable protocol. The four-level taxonomy and named overclaiming traps (pooled multi-config, best-selected, row-level bias) are a contribution the subfield should adopt.
  • Concrete coordinate system. The SwiGLU-aware P1s / P2a / P2b points tie intervention sites to the gate/value branch structure.
  • Clean design rationale. Decode-only start (first generated token) explains why multi-token trajectories benefit and one-token MC does not.

Limitations

  • Single model family. Only Qwen3.5 / Qwen3 / Qwen2.5 tested; no Llama / Mistral / Gemma, so architecture-generality is unproven.
  • Offline-only. No prospective online A/B, no time-split, no held-out dataset; parameter overfitting not fully excluded.
  • Mechanism under-explained. P1s-isolates-gate-branch and P2a/P2b-orthogonal hypotheses remain untested; layer-wise ablation incomplete.
  • Sparse e-grid. Only e=0.93 / 0.95 probed; best cells may be local optima.
  • Headline Qwen3-8B category gain leans on partial credit. Strict fix/harm there is negative (22/27); the +2.64 pp rests on partial10 scoring the paper itself cautions about.

04 Open questions

Where the surviving claims push the reader next.

Does AG and the P1s/P2a/P2b ranking transfer to non-Qwen families (Llama-3, Mistral, Gemma) with GeGLU / ReGLU or non-gated GELU FFNs?

Can an online-computable proxy for post-fork cross-PPL recover the harm suppression seen offline (7.4% harm at 11.5% fix capture) without a second forward pass?

How do AG gains interact with grammar-guided / JSON-schema-constrained decoding — do they stack, substitute, or interfere at the FFN-vs-decoder level?

Would a denser e-sweep (0.80–0.99) change the best fixed cell, and is there one per-model calibrated e that subsumes the 0.93 / 0.95 split?
