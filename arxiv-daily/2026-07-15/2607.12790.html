Who Grades the Grader?

Co-evolve the evaluation metric itself with agent skills — retaining 88–110% of oracle lift while anchored against Goodhart collapse.

Xing Zhang · Guanghui Wang · Yanwei Cui · Ziyuan Li · Wei Qiu · Bing Zhu · Peiyang He
Alias: “Double Ratchet” — Metric Evolution for Self-Improving Agents

Why this matters

Every self-improving agent quietly assumes an evaluation signal. This turns that hidden premise into an inspectable, anchored, evolvable object — and demonstrates catching its own cheating, end to end.

Critical edge

The lift holds only where failures are mechanically detectable. On purely semantic-failure tasks (Spider post-grounding: 0.500 held-out) deterministic detector compositions buy little.

01

Methodology

A metric is an expression tree over small typed “drawback detectors” — composed, gated, scored, retired. Not a black-box judge.
Stance: we rarely know what “good” is, but given an output we can usually find a drawback. So a clean verdict means “no known drawback found,” not certified correctness. Composing deterministic detectors — which fail differently from the LLM under test — resists the shared-blind-spot collusion a bare LLM judge invites.

Loop AMetric evolution

Selection never asks “did the agent score well?” — it asks whether the metric agrees with a 10-item anchored dev set and with detector consensus on a large unlabeled train set.

S(e) = A_dev · A_train^w − λ·C(e)

Loop BRatchet skill loop

The evolved metric grades train attempts; failed attempts become capsules whose error text feeds skill synthesis. Its held-out eval and rollback anchor stay pinned to a locked test set the metric never reads — so a corrupted metric can slow, but cannot corrupt, measurement.

curriculum: metric phases 15 / 8 / 5 / 2 rounds

PHASE 01
Sense
Find dev items the incumbent wrongly passes; find train outputs where the op pool abstains or splits.
PHASE 02
Grow
Cluster misses by failure class; author one typed op per cluster. Birth gate: fires on ≥½ its cluster, clean on known-good.
PHASE 03
Select
LLM composer proposes candidates over the op catalog + mutations/crossovers. Keep argmax S(e) behind a validity gate.
PHASE 04
Curate
Retire ops with non-positive leave-one-out marginal contribution after a grace period; promote shadow ops.
PHASE 05
Audit
Report agreement against the locked test set — measurement only, never a training signal.

The Goodhart episode — caught, repaired, then the judge audited

  1. On the report task, evolved skills gamed the rubric's tag counter — ~30% of tags had no value beside them, plus fabricated forecasts.
  2. An independent final judge (Claude Opus 4.8, outside all loops) preferred the pre-evolution baseline in 88% of decided pairs.
  3. One vocabulary-aware value-erasure detector repaired it — erased tags dropped ~30% → ~1%, rubric score held at 0.850.
  4. Then the judge itself was audited: a 2×2 showed it was misreading the pipeline's required tag syntax as a defect. A task-aware rubric moved win-rate 0.515 → 0.770.
03

Strengths & limitations

Anchor discipline — not the detector lifecycle — is the load-bearing guard. That inverts the skill-evolution literature.

Strengths

  • Inspectable artifact. The composed metric (any(spec_mismatch, crash, returns_not_print_only)) names failure modes a practitioner would recognize — which is what made the one-detector Goodhart repair possible.
  • Leakage-safe by construction. Three-split protocol with a locked anchor the metric never reads; the deliberate unsafe ablation collapses exactly as theory predicts.
  • Genuinely surprising result. Anchor guards carry the safety load; the lifecycle does not — inverting the received lesson from skill evolution.
  • End-to-end failure-expecting loop. Detect → one-detector repair → judge self-audit, including the meta-failure of the auditor.
  • Honest about scope. Three seeds = coarse conclusions; Spider bounds the claim; the anchor cannot be manufactured.

Limitations

  • Statistical power. 3 seeds × 40–48 held-out items; the 88–110% headline is tighter than the reported CIs support (Spider Δ CI ≈ [−.08, .03]).
  • Single solver. Claude Opus 4.7, single direct call, no tools/iteration/CoT. Generalization across capability levels untested.
  • Detectability bound. Where failures are purely semantic (Spider post-grounding: 0.500 held-out) deterministic compositions buy little.
  • Teacher quality held fixed. The entire dev soft-label supply chain runs through it — a load-bearing untested variable, deferred to future work.
  • Proprietary report task with no public release; headline 77% win is judge-rubric-sensitive (0.126 vs 0.770 on the same pairs).
  • No competitive baseline. No head-to-head vs bare LLM-as-judge, Self-Rewarding LM, or learned reward models.
04

Open questions

Where the method's value might expand, contract, or vanish.
  1. Does lift retention hold for a weaker 7B-class solver, or a tool-using multi-call agent — or does the “mechanically detectable” regime simply move?
  2. Can the teacher's influence on dev soft-labels be decoupled? Does a cross-family (non-Claude) teacher change which detectors are born?
  3. Do the unexercised metamorphic “failure-anticipating detectors” help in the semantic regime where deterministic detectors currently buy little?
  4. Does the method's value survive a hardened, fully-checkable anchor — making it purely a bootstrapping tool?
  5. Could composed metrics replace hand-written rubrics in production triage — and what happens when the locked anchor itself drifts over time?
  6. How rubric-sensitive is the final-judge protocol in general — and can the rubric itself be co-evolved without re-introducing the gaming it should catch?
