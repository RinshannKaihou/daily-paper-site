EvoQuant: Self-Evolving, Verifier-Guided Strategy Optimization

EvoQuant reframes quant-strategy improvement as evidence-constrained program evolution — parsing a deployed strategy into a typed genome, then admitting LLM-proposed edits only after they survive out-of-sample-aware hard gates.

Why This Matters

An auditable, overfitting-resistant LLM loop for improving already-deployed quant strategies — its typed-genome + OOS hard-gate + rejection-log design is a methodological template for trustworthy autonomous quant agents.

Critical Edge

No external baselines. The paper compares only against its own ablations (random mutation, parameter-only); "improvement over random mutation" is a low bar, and the absence of Optuna / AlphaEvolve / R&D-Agent-Quant is the biggest gap.

01 / METHOD

The Verifier-Guided Evolution Loop

MODULE 1

Strategy Ingestion

Parse raw code/rules into an AST, tag trading semantics, factor into editable layers, compile to a typed genome.

g = (g_sig, g_risk, g_pos, …, ξ)
MODULE 2

Evidence Construction

Run incumbent on chronological splits under identical execution; emit evidence pack with δ_oos, regime & sparsity signals.

E(g) = {M_X, PX, δ_oos, …}
MODULE 3

Diagnose → Generate → Verify

LLM diagnoses bottlenecks, proposes hierarchical edits (repair→bridge→redesign→migrate); OOS hard gates + adaptive promotion.

ADOPT / INCUBATE / REJECT
MODULE 4

Iterate & Distill

Track best audit-score genome; escalate search level on failure; persist diagnoses, edits, decisions into memory.

g* = argmax Q_audit(g)
verifier Q(c) = w_v·ΔS_val + w_o·ΔS_oos + w_r·ΔR_oos − λ_d·P_dd − λ_g·P_gap − λ_s·P_stress  ·  hard gate N_val ≥ n_min, N_oos ≥ n_min, MDD_oos ≤ d̄, drift ≤ ε  ·  each run = 20 iterations

03 / ASSESSMENT

Strengths vs. Limitations

▲ Strengths

  • 01Principled formulation as constrained program evolution with provenance, drift budgets, and explicit edit contracts.
  • 02Transparent ablation isolating adaptive promotion as the decisive component (0.982 → 0.128 without it).
  • 03Cross-market evaluation (A-share + Bitcoin) with 2× fee stress and walk-forward, not a single in-sample cherry pick.
  • 04Honest reporting of 115/120 non-degradation and rejection-reason breakdown, including the 5 failures.
  • 05Acknowledges survivorship bias, simplified execution, and single-LLM limits in a dedicated section.

▼ Limitations

  • 01No external baselines — only self-ablations; no Optuna, AlphaEvolve, AlphaForge, or R&D-Agent-Quant.
  • 0230 randomly sampled A-share symbols, not a survivorship-corrected investable universe.
  • 03Single LLM (DeepSeek-R1), no model-sensitivity study; headline rate from a single seed.
  • 04Simplified cost model — no market impact, liquidity, borrow, or corporate-action handling.
  • 05ATR Trend Breakout stays negative (−1.050) post-optimization; underperforms in appendix test curve.
  • 06Anonymous code link; exact verifier thresholds and Q(c) weights not fully specified.
04 / OPEN

Questions for Further Exploration

Q1 · LLM SENSITIVITY

Is the promotion protocol robust when DeepSeek-R1 is swapped for GPT-4o, Claude, or Qwen-3 — or is R1's reasoning style load-bearing?

Q2 · REAL BASELINE

How does EvoQuant fare against Optuna TPE, AlphaEvolve, or CMA-ES given the same 20-iteration genome budget?

Q3 · MULTI-SEED

Does the 115/120 non-degradation rate survive seeds 0–9, and how wide is the per-seed Sharpe-uplift variance?

Q4 · SURVIVORSHIP

Would a delisting-aware CSI 300/500 historical universe materially change the Low-Vol Quality and EMA-RSI results?

Q5 · GATE CALIBRATION

Is there a regime where the n_min floor and MDD cap d̄ make the verifier too permissive or overly conservative?

Q6 · MEMORY TRANSFER

Can distilled cases transfer across strategy families or markets — and does cross-family transfer beat family-specific memory?
