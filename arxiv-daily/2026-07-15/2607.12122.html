01Methodology

Operator architecture search as a bilevel program; an LLM-coordinated particle swarm solves the outer loop.

The authors frame operator discovery as a bilevel program. The inner problem trains a fixed genome by gradient descent; the outer problem is solved by a swarm of N = 16 virtual labs running T = 20 iterations, each lab holding an LLM planner, a numerical worker, and an LLM reviewer.

A genome draws from five operator-block families. Each iteration runs four phases:

  1. Planning — each lab queries the local LLM (gemma3:12b via Ollama, 8k context) for the next genome given its history, global best, and swarm summary.
  2. Training — the worker compiles & trains at deliberately reduced screening fidelity so each candidate trains in seconds; 320 trainings/swarm fit on one GPU.
  3. Peer review — labs cast soft votes over sampled peers, accumulating citation-like influence & trust.
  4. Lifecycle — composite fitness updates global best; well-cited labs breed, uncited labs are deactivated.
Composite Fitness
F = 0.70·accuracy + 0.20·generalization (noise ε~N(0, 0.1²)) + 0.05·param-efficiency + 0.05·novelty # accuracy floor ⇒ degenerate / # novel-only models zeroed out # weights fixed a priori, never tuned

The worker is deliberately not an LLM, so fitness signals stay comparable across labs. The ablation swaps in a rule-based planner (explore/exploit PSO) and rule-based reviewer (accuracy-proportional voting), sharing everything else: identical swarm size, iteration count, budget, fitness, and seeds 42 / 137 / 2024.

Full bilingual TL;DR

An "AI scientific community" — a swarm of virtual labs (LLM planner + numerical worker + LLM reviewer each) coordinated by a citation-based economy — autonomously discovers neural operator architectures over a shared vocabulary of DeepONet, Fourier, Transformer, wavelet, and residual-conv blocks. Evaluated on five PDE/regression problems, every LLM-discovered winner was a multi-family hybrid. All 9,623 LLM calls are audited with zero rule-based fallbacks. A controlled ablation shows LLM agency does not improve screening accuracy, but does preserve architectural diversity.

一句话总结：作者构建了由多个虚拟实验室组成的 LLM 驱动"AI 科学共同体"（每室含规划/数值/评审三类 agent），用粒子群 + 引用经济机制在五类 PDE 上自动发现神经算子架构；全部 9623 次 LLM 调用零回退，发现全部为多家族混合架构，并通过对照消融证明 LLM 的价值在于保持架构多样性而非提升精度。

03Strengths & Limitations

Read the accuracy claims carefully: swarm-time Table 2 screens at reduced fidelity and is not comparable to retrained Table 1.

Strengths

  • Methodologically clean ablation: identical budget, seeds, fitness & swarm size isolate the LLM contribution.
  • Full audit of 9,623 LLM calls with zero-fallback certification — addresses an under-reported failure mode.
  • 100% local (gemma3:12b via Ollama, RTX 4080), no paid API — strong reproducibility.
  • Intellectually honest about the negative accuracy result; reframes value as diversity preservation.
  • Citation economy + breeding + lifecycle yields a self-organizing research-community analog.

Limitations

  • Small scope: 5 problems (3 are 1D toys); no irregular domains, 3D, or long-horizon rollout — the "no-free-lunch" claim is narrow.
  • Headline errors come from retraining only winners to convergence; swarm-time errors screen at reduced fidelity — easy to misread.
  • Planner + reviewer ablated jointly, not factorially — individual role contributions unknown.
  • Single LLM backend (gemma3:12b); frontier-model behavior untested.
  • Literature-prior confound: the LLM has read operator-learning literature — not prior-free discovery.
  • Discovered architectures are problem-specific, not transferable across problems.

04Open Questions

Where the framework could be probed, extended, or stress-tested next.
  1. Would a factorial ablation (planner-only, reviewer-only, both, neither) reveal one role dominates diversity preservation?
  2. Do frontier LLMs (GPT-4-class) change discovered architectures and the 99.8% HYBRIDIZE bias — is it an artifact of gemma3:12b's training distribution?
  3. Can the "preserve diversity" prompt be replaced by action-space debiasing to separate prompt effects from intrinsic LLM behavior?
  4. Does the citation-economy dynamic converge to a single research direction over much longer horizons (T ≫ 20)?
  5. How does it perform on harder regimes — irregular geometries, 3D PDEs, long-horizon rollout, stiff/chaotic systems?
  6. Would adaptive fitness weights (vs. fixed 0.70/0.20/0.05/0.05) shift the accuracy–diversity trade-off materially?
  7. Can problem-specific hybrids be combined into a single transferable meta-architecture via a meta-learning outer loop?
