TACT: Mitigating Overthinking and Overacting in Coding Agents via Activation Steering

["Yuan Sui", "Yulin Chen", "Yibo Li", "Xue Jiang", "Yufei He", "Yihong Dong", "Xiaoxin He", "Tianyu Gao", "Bryan Hooi"]

一句话总结： TACT 通过在 Transformer 残差流中提取"过度推理"与"过度行动"两个对比激活方向，在推理时对编码智能体的隐层状态进行无微调干预，使 Qwen3.5-27B 在 SWE-bench Verified 上解决率提升 +5.8pp，同时将解决所需步数降低最多 26%。
TL;DR: Coding agents operating over long trajectories frequently exhibit two failure modes: overthinking (re-deriving facts the agent already knows) and overacting (calling tools redundantly without synthesizing prior observations). TACT is a training-free, three-stage pipeline that (1) labels every trajectory step as overthinking, overacting, or calibrated using an LLM-as-judge with a rolling compressed state, (2) extracts two orthogonalized contrastive directions in the residual stream at the token, and (3) at test time steers out-of-band activations back toward the calibrated region via a forward hook—no prompt changes, no extra LLM calls, no fine-tuning. On SWE-bench Verified, Terminal-Bench 2.0, and CLAW-Eval, TACT raises average resolve rate by up to +5.8 pp (Qwen3.5-27B) and +4.8 pp (Gemma-4-26B-A4B-it) while cutting steps-to-resolve by up to 26%. If you work on long-horizon agent reliability, inference-time intervention, or representation engineering for agents, this paper is worth reading; if you need a quick prompt-level fix or work with closed-weight models, the method does not transfer.

核心贡献

  • "智能体漂移"的机制性定义：将长轨迹中的智能体退化形式化为"过度推理"（重复已知信息的推理）和"过度行动"（调用工具但未整合已有观测）两类步级失效，并证明漂移比例（drift ratio）与任务解决率显著负相关。
  • 带滚动状态的 LLM-as-Judge 标注流程：设计三阶段流程（状态维护者 → 状态验证者 → 标注者），用压缩的滚动上下文（KnownFacts + Progress）逐块标注每个轨迹步骤，克服了超长轨迹下的上下文管理难题；与人工标注的 Cohen's κ 达到 0.78。
  • 双轴对比激活方向提取：在 token 处提取 OT-vs-CAL 和 OA-vs-CAL 两个对比方向，并通过 Gram-Schmidt 正交化消除两轴间的干扰，使三类步骤在激活空间中线性可分（AUC ≈ 0.9）。
  • 测试时双轴引导（无需重训练）：在推理时通过 forward hook 将偏离校准区域的隐层状态拉回，提出三种策略（TACT-Cap、TACT-Add、TACT-Gate），无需微调、无需额外提示词或重试。
  • 跨模型、跨基准的稳健收益：在两种异构模型架构（密集型 Qwen3.5-27B、稀疏 MoE 型 Gemma-4-26B-A4B-it）和三个编程基准上均验证有效，且对标注所用 judge 模型不敏感（换 Claude Opus 4.6 后结果相差不足 1pp）。

Key Contributions

  • Mechanistic formalization of agent drift into two measurable step-level failure modes (overthinking, overacting) with demonstrated correlation to task success rate.
  • Scalable LLM-as-judge annotation via a three-agent pipeline (maintainer, verifier, annotator) that carries a validated rolling state summary across trajectory chunks, achieving Cohen's κ = 0.78 with human consensus.
  • Orthogonalized dual drift axes extracted at the reasoning-action boundary ( token) using per-problem balanced mean-difference directions, Gram-Schmidt orthogonalization, and top-K layer selection by validation AUC.
  • Training-free test-time steering via residual-stream forward hooks that nudge out-of-band activations back to the calibrated band; three variants (Cap, Add, Gate) offer different selectivity-strength tradeoffs.
  • Broad empirical validation across SWE-bench Verified, Terminal-Bench 2.0, and CLAW-Eval on both a dense and a sparse MoE model, with robustness checks across multiple judge models and a human audit.

Methodology

Problem Formulation. Each agent trajectory step $s_t = (r_t, a_t, o_t)$ consists of a reasoning block, an action, and an observation. A step is labeled Overthinking if $r_t$ re-derives a conclusion already in the agent's accumulated information state $\mathcal{K}_t$; Overacting if $a_t$ returns an observation already in $\mathcal{K}_t$ or fires without processing recent evidence; Calibrated otherwise. Trajectory-level drift ratios $\gamma^{\text{OT}}_N$ and $\gamma^{\text{OA}}_N$ both correlate negatively with resolve rate on SWE-bench Verified.

Stage 1 — Trajectory Annotation. Trajectories are chunked into windows of W=10 steps. A State Maintainer compresses all prior steps into a rolling state (KnownFacts + Progress). A State Verifier audits the state for hallucination, omission, and contradiction (up to 3 correction loops). An Annotator then labels each step in the chunk using the verified rolling state as context. Only high- and medium-confidence labels are retained.

Stage 2 — Drift Axis Extraction. For each labeled step, the hidden state at the token is extracted for every transformer layer. Two contrastive mean-difference axes are computed: OT-vs-CAL and OA-vs-CAL (Eq. 2). Since these axes are not orthogonal by construction, OT-vs-CAL is designated the primary axis (higher AUC: 0.91 vs 0.77), and OA-vs-CAL is Gram-Schmidt orthogonalized against it to form $\hat{v}^{(\ell)}_{\text{OA}^\perp}$. The calibrated centroid $\mu^{(\ell)}_\text{CAL}$ and per-axis standard deviation $\sigma^{(\ell)}_\text{CAL,a}$ define the calibrated band. The top-K=10 layers are selected by validation per-problem AUC on the OT-vs-CAL axis.

Stage 3 — Test-time Dual-axis Steering. A forward hook intercepts the hidden state $h^{(\ell)}_t$ at the token for each selected layer $\ell$. The centered projection $p_{t,a}^{(\ell)} = (h_t^{(\ell)} - \mu_\text{CAL}^{(\ell)}) \cdot \hat{v}_a^{(\ell)}$ measures drift. Three steering strategies are offered:

  • TACT-Cap: Clamps the projection to $[-\tau_{a,\ell}, +\tau_{a,\ell}]$, pulling out-of-band activations to the nearest band boundary. Conservative; fires only on genuinely drifted steps.
  • TACT-Add: Applies a fixed additive shift $s(\hat{v}^{(\ell)}_\text{OT} + \hat{v}^{(\ell)}_{\text{OA}^\perp})$ at every forward pass (negative $s$ pushes toward CAL). Simple but less selective.
  • TACT-Gate: Fires like Cap but subtracts a $\lambda$-scaled out-of-band excess rather than clamping exactly to the boundary. At $\lambda=0.5$ it is the best performer; $\lambda=1$ reduces to Cap.

All steering is applied independently along both axes at each selected layer; because the axes are orthogonal, the two corrections commute.

Results & Highlights

Main benchmarks (Table 1):

  • Qwen3.5-27B: TACT-Gate achieves 73.3% resolve rate on SWE-bench Verified (+6.3 pp over baseline 67.0%) and reduces steps from 75.2 to 65.3 (−13%). On Terminal-Bench 2.0, resolve rate rises from 32.6% to 36.0%. Average resolve rate across three benchmarks improves from 51.1% to 56.9%.
  • Gemma-4-26B-A4B-it: TACT-Gate reaches 65.5% on SWE-bench Verified (+5.5 pp), 19.1% on TB2 (+3.2 pp), 61.4% on CLAW. Average +4.8 pp.
  • All TACT variants complete tasks in a single rollout (no retries), while the strongest prompt baseline AgentDebug requires at least two rollouts per task and still trails TACT-Gate by ~3 pp on both models.

Ablations (Table 2): Removing either axis drops performance; removing orthogonalization costs 0.8 pp; using all 64 layers instead of top-10 costs 4.9 pp. The drift signal is strongest in early-to-middle layers (OT-vs-CAL peaks at layer 3, AUC 0.91; stays above 0.86 through layer ~20).

Drift axis quality (Figure 5): Three-class LDA at layer 14 shows OT, OA, and CAL as geometrically separable clusters. OT-vs-CAL and OT-vs-OA AUCs stay above 0.86 through layer ~20, confirming drift is a stable multi-layer property rather than a single-layer artifact.

Robustness: Substituting Claude Opus 4.6 as the judge produces results within 1 pp of GPT-5.4-labeled axes on all three TACT variants. Human audit on 300 steps gives Cohen's κ = 0.78 overall, with F1 above 0.83 for all three classes.

Critical Assessment

Novelty. The paper makes a genuine conceptual contribution: applying representation engineering (RepE) to behavioral failures in agentic settings rather than to single-turn properties like honesty or refusal. The dual-axis, orthogonalized framing and the context-aware LLM judge are both technically sound extensions of prior work rather than wholesale inventions. The core idea—that overthinking and overacting live as separable linear directions in the residual stream—is non-obvious and empirically well-supported.

Rigor. The experimental design is careful: three disjoint splits (axis-train, axis-val, test), hyperparameters selected on val only, and results reported on a held-out test set. Human audits and cross-judge consistency checks add credibility. Error bars (Wilson 95% CIs) are reported where appropriate. The ablations are thorough and informative.

Reproducibility. The method requires white-box access to model internals (residual stream activations), which restricts applicability to open-weight models. The three-stage LLM judge pipeline uses GPT-5.4 as the default annotator—a model that is itself expensive and may not be publicly available in all deployment contexts. The paper provides algorithm pseudocode and states that full prompts appear in appendices, which is a positive signal, but the end-to-end pipeline involves multiple closed-source LLM calls that make exact reproduction non-trivial.

Significance. The gains (+5.8 pp on SWE-bench Verified) are meaningful and come at near-zero inference overhead (a hook that does closed-form linear algebra per forward pass). The efficiency gains (26% step reduction) are practically important. However, results are confined to coding tasks and two open-weight reasoning models; generalization to non-coding domains and closed-weight models is explicitly left for future work.

Red flags.

  • The annotation pipeline itself uses a large frontier LLM (GPT-5.4). The axes must be re-extracted for each new model—there is no transfer across architectures.
  • The "calibrated" region is defined from training trajectories; if distribution shifts exist between training and evaluation tasks, the centroid estimate may degrade.
  • Gains on Terminal-Bench 2.0 are more modest (+2.6–3.2 pp) and OA-vs-CAL AUC is lower (0.77) than OT-vs-CAL, suggesting the method's advantage may be uneven across task types.
  • The paper does not report variance across multiple seeds or task-level win/loss analysis, making it harder to assess statistical reliability beyond the aggregate resolve rate.

Strengths

  • Training-free and inference-overhead-free (constant-time hook per forward pass), making it practically deployable without any model modification.
  • Elegant dual-axis geometric framing with principled orthogonalization; the method is interpretable and the axes carry genuine diagnostic value.
  • Thorough ablation of all design choices (axis combination, orthogonalization, layer selection), robustness to judge model choice, and human-verified annotation quality.
  • Generalizes across two architecturally distinct model families (dense vs sparse MoE), suggesting the finding reflects a general property of reasoning-model activations.
  • Establishes a new diagnostic lens: drift ratio as a predictive signal of task failure, independent of final outcome.

Limitations

  • Requires open-weight models with accessible residual streams; closed-weight APIs (GPT-4o, Claude, etc.) cannot use this method.
  • Axis extraction requires a corpus of labeled trajectories for each model and domain; adapting to a new domain (e.g., web navigation, multi-modal tasks) requires re-running the full annotation pipeline with a capable judge LLM.
  • The linear separability assumption may break down for subtler or more entangled failure modes; the paper briefly mentions sparse autoencoders as a future direction for capturing more complex drift geometry.
  • Evaluation is limited to coding benchmarks; it is unknown whether the same drift axes transfer to instruction-following, tool-use in non-coding contexts, or multi-agent settings.
  • Steps-to-resolve is reduced, but token-level cost (within each step) is not analyzed; it is possible that steered steps are individually longer reasoning blocks.

Relevance

本文从机制层面分析并矫正 LLM 编码智能体在长轨迹上的行为退化，属于推理时干预与智能体可靠性的前沿研究，对关注长上下文代理行为、激活引导以及 SWE-bench 类任务的研究者高度相关。对于希望在不修改模型权重的情况下提升开源推理模型稳定性的实践者，TACT 提供了一个直接可落地的技术路线。

Questions for Further Exploration

  1. Do the extracted drift axes transfer across models of the same family (e.g., different sizes of Qwen3.5)? If so, could a smaller proxy model be used for axis extraction to reduce annotation cost?
  2. The paper finds OA-vs-CAL is a weaker signal (AUC 0.77) than OT-vs-CAL (AUC 0.91). Is overacting fundamentally harder to capture linearly, or is the annotation definition less consistent for overacting?
  3. How does TACT interact with chain-of-thought length budgets or explicit brevity instructions? Does steering reduce reasoning quality on tasks that genuinely require extended deliberation?
  4. Would sparse autoencoder features at the boundary provide finer-grained drift signals, and could they enable per-feature steering that is more precise than a single linear direction?
  5. Can the rolling-state LLM judge be replaced by a smaller, purpose-trained classifier without losing annotation quality, making the pipeline self-contained and less dependent on frontier models?
  6. The method is applied at tokens; would applying it at intermediate positions within the reasoning block (e.g., after each sentence) provide earlier intervention and larger gains?
