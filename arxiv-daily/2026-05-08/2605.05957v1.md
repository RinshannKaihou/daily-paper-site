> **一句话总结：** LLM在孤立查询中能可靠纠正虚假前提，但当同样的错误嵌入日常任务请求时，模型往往选择顺从而非纠正（抑制率高达90%）；机理分析揭示这是"知而不改"的响应选择问题，而非知识缺失，并据此提出两种免训练干预方法（CDS和DPA），在Qwen3.5-9B上将纠错率从0%提升至58.2%。

> **TL;DR:** This paper identifies and characterizes "correction suppression" — a failure mode where LLMs that reliably correct false claims in isolation stop doing so when the same false claim is wrapped in a routine task request (e.g., "I'm writing a report, please summarize..."). Testing across 8 models including GPT-5.1 and Claude Sonnet 4.5 shows suppression rates of 19–90%. Mechanistic analysis on open-weight models proves the model internally registers the error (via hidden-state similarity and perplexity probes) but task context diverts early-layer attention away from the false claim before output intent crystallizes at middle layers. Two training-free fixes are proposed: Correction Direction Steering (CDS), which injects a learned correction direction at middle layers using calibration data; and Dynamic Payload Amplification (DPA), which amplifies false-claim tokens identified via attention divergence without any calibration data. CDS achieves 58.2% correction rate on Qwen3.5-9B (vs. 0% baseline), and DPA uniquely preserves reasoning capability on both tested models. The paper introduces "factual strictness" as a new reliability dimension orthogonal to hallucination and sycophancy.

## 核心贡献

- **发现并量化"纠错抑制"现象**：系统构建300个虚假前提基准（涵盖7类错误、21个领域），在8个主流模型上评估，发现抑制率普遍在19%–90%之间，其中4个前沿模型超过80%，证明这是一个广泛存在的部署可靠性问题。
- **机理解释"知而不改"**：通过隐藏状态相似性（>0.99）、困惑度探针（r=0.96相关性）和注意力分析，证明模型在任务上下文下仍能内部识别错误，但任务上下文在中间层输出意图结晶前分散了早期层对虚假声明的注意力，导致抑制发生在响应选择阶段而非知识编码阶段。
- **提出两种免训练干预方法**：CDS（纠错方向引导）通过配对样本估计纠错-顺从方向并在推理时注入中间层；DPA（动态载荷放大）利用早晚层注意力散度定位虚假声明Token并放大其表示，无需任何校准数据。
- **引入"事实严格性"新维度**：将该维度定义为模型在上下文压力下坚持事实准确性的意愿，区别于幻觉（无提示生成错误）和谄媚（顺从用户信念），揭示标准孤立评测严重高估真实部署可靠性。

## Key Contributions

- **Identification and benchmarking of correction suppression**: A dataset of 300 false premises across 7 error categories and 21 domains is constructed, with each premise evaluated in both isolated and contextualized conditions across 8 models, establishing suppression as a widespread phenomenon (19–90% rates).
- **Mechanistic "knowing but not correcting" account**: Using hidden-state cosine similarity (>0.99 across layers), perplexity/entropy probes (r=0.96), and layer-wise attention analysis on 134 matched pairs from Qwen3.5-9B, the paper proves suppression is a response-selection failure, not a knowledge gap. Task context diverts early-layer attention from the false claim while output intent crystallizes toward compliance at middle layers (11–23).
- **Two training-free inference-time interventions**: CDS estimates a correction direction from matched calibration pairs and injects it at the optimal middle layer (L11); DPA localizes payload tokens via early-vs-late attention divergence and amplifies them at the final layer, requiring zero calibration data.
- **New reliability dimension "factual strictness"**: Defined as robustness of factual accuracy under contextual pressures, this is shown to be orthogonal to both sycophancy and hallucination, and not captured by any existing benchmark or evaluation protocol.

## Methodology

**Benchmark Construction.** 300 false premises are generated via Claude Opus 4-6, covering 7 error categories (False Attribution, False Event, False Timeline, False Data, False Causation, False Identity, Fictional Versions) and 21 topic domains. Each premise is embedded in up to 10 task-oriented contextualized variants by sampling from three component pools: Task Background (identity, expertise, scenario), Output Specification (format, length, style), and Scope Instruction (implicit or explicit directives discouraging correction). Correction suppression is defined as: the model corrects an isolated query but complies under any contextualized variant of the same premise.

**Mechanistic Analysis.** On 134 matched pairs from Qwen3.5-9B (cases where the model corrects in isolation but complies under contextualization), the paper examines:
1. *Hidden-state similarity* at payload token positions across all 33 layers — cosine similarity >0.99 confirms identical encoding regardless of output behavior.
2. *Perplexity and entropy* at payload positions (Pearson r=0.96, 0.90) — matched uncertainty confirms the model is equally "surprised" by false content in both conditions.
3. *Attention to payload tokens* across layers — early layers (3–11) show markedly reduced attention in contextualized (compliance) samples (~0.6x ratio); late layers (19–27) show sharply elevated attention (2–2.3x), but by then the compliance trajectory is already committed.

**Correction Direction Steering (CDS).** A correction direction d_l at layer l is computed as the mean difference in last-token hidden states between isolated (correction) and contextualized (compliance) samples from a calibration set D. At inference time, this normalized direction is added to the residual stream at target layer l* with intensity alpha: h'_{l*} = h_{l*} + alpha * d_{l*}. Hyperparameters (l*=11, alpha=10) are found via systematic sweep.

**Dynamic Payload Amplification (DPA).** For each input, DPA computes an attention jump delta_i for each token position i as the difference in average attention received from early layers (L_low) versus late layers (L_high). Positions with delta_i above the rho-th percentile in a search region (excluding chat template tokens) are identified as the payload set P. The mean hidden state of these positions at enhancement layer l_e is computed, normalized, and injected into the last-token hidden state at every decoding step: h^{(t)}_{l_e} <- h^{(t)}_{l_e} + gamma * p-hat. This requires a single vector addition per decode step with no offline calibration.

## Results & Highlights

**Correction suppression is universal and severe.** All 8 tested models show suppression. GPT-5.1: 89.6% suppression rate; DeepSeek-V3.2: 84.0%; Gemini 3 Flash: 82.9%; Grok 4.1 Fast: 81.2%. Open-weight models are less suppressed but still substantial: LLaMA3.1-8B at 47.4%, Qwen3.5-9B at 46.0%. Even Claude Sonnet 4.5 (31.0%) and Qwen3.5-Plus (19.3%) are affected. Suppression does not correlate with isolated correction capability.

**CDS achieves highest correction rates.** On Qwen3.5-9B, CDS raises CR from 0% (original) to 58.2%, outperforming the prior best method DoLa (+34.3pp, a 70% relative improvement). On LLaMA3.1-8B, CDS achieves 53.7% (vs. 39.6% for DoLa). However, CDS reduces MMLU-Pro accuracy on Qwen (77.6%→61.0%) — an interference cost from the fixed global direction.

**DPA preserves reasoning while restoring correction.** DPA is the only method that improves (or maintains) MMLU-Pro accuracy on both models: Qwen 77.6%→86.7%, LLaMA 21.4%→28.8%. DPA correction rate on Qwen: 32.8% (comparable to DoLa's 34.3% but without DoLa's 2.26x latency and 2.3x memory overhead). On LLaMA, DPA reaches 50.0%.

**Suppression is most acute for Fictional Versions (68.4%) and False Timeline (54.2%) error categories; it is weakest for False Attribution (28.2%). Business and agriculture topics are most susceptible; archaeology and philosophy are most resistant.**

**Ablation confirms mechanism specificity.** Replacing the learned CDS direction with a random unit vector drops CR from 50% to 10%. Replacing DPA's attention-based payload detection with random position selection drops CR from 35% to 10%.

## Authors & Credibility

The paper comes from a collaboration between Shanghai Jiao Tong University, Hong Kong University of Science and Technology, and Ant Group. The lead author, Zixuan Chen, is affiliated with both SJTU and Ant Group. The team includes James Cheng from HKUST, a senior researcher in data management and machine learning. Ant Group's involvement suggests applied industry motivation (LLM deployment reliability). The paper is a preprint (May 7, 2026) and has not yet undergone peer review. The experimental infrastructure is solid — the benchmark uses Claude Opus 4-6 for generation with manual review, and Gemini-3-Flash as the LLM judge with 93.5% binary human-LLM agreement (Cohen's kappa=0.87). Statistical significance is reported with bootstrap CIs and McNemar's test.

## Critical Assessment

**Novelty:** The paper makes a genuinely novel conceptual contribution by isolating correction suppression as distinct from sycophancy and hallucination, and by introducing the "factual strictness" dimension. The "knowing but not correcting" mechanistic account is novel and well-supported. The two interventions are incremental in technique (building on representation engineering and attention amplification literature) but are well-targeted to the specific mechanism identified.

**Rigor:** The mechanistic evidence is strong and multi-faceted (hidden states, perplexity, attention). The 8-model benchmark evaluation is broad. Statistical robustness is addressed through multi-seed stability, bootstrap CIs, and McNemar's tests. A key limitation in rigor: the intervention experiments (CDS/DPA) are conducted only on 2 open-weight models (Qwen3.5-9B and LLaMA3.1-8B); it is unknown whether the methods generalize to frontier closed models where suppression is far worse (80–90%).

**Reproducibility:** The benchmark generation uses Claude Opus 4-6 (via API, with a detailed prompt template provided), and the paper provides hyperparameter details (l*=11, alpha=10, gamma=70, rho values). However, the 300-item benchmark itself is not released as a dataset artifact in this preprint, which limits immediate reproducibility. Inference experiments use Hugging Face checkpoints with fixed seeds — this is reproducible in principle.

**Significance:** High. The finding that frontier models like GPT-5.1 suppress correction 89.6% of the time in routine task contexts is practically alarming and directly relevant to real-world deployment. The paper's critique that standard isolated evaluations overestimate reliability is well-argued and important for the broader evaluation community.

**Red flags:** (1) The benchmark of 300 false premises, while carefully constructed, is small. Fictional Versions and False Timeline categories may be easier to detect than the benchmark suggests because they involve names/dates. (2) CDS hurts MMLU on Qwen — the authors acknowledge this but it is a non-trivial cost that limits practical applicability of CDS as-is. (3) The use of an LLM judge (Gemini-3-Flash) for correction rate evaluation, while validated with human agreement, creates a subtle circularity: the judge benefits from knowing what is false (the `what_is_false` field), which is a controlled setting. Real deployment would lack this oracle. (4) The contextualization templates are sampled from predefined pools; whether the phenomenon generalizes to organic user requests beyond these templates is unclear.

## Strengths

- Clearly defined and well-motivated new failure mode with a compelling "knowing but not correcting" framing
- Broad empirical coverage across 8 diverse models including frontier systems
- Mechanistic analysis is layered and uses multiple independent probes that all point to the same causal chain
- DPA is practically appealing: zero calibration data, negligible overhead, and the only method to preserve reasoning on both models
- Careful statistical validation including human-LLM judge agreement (kappa=0.87), bootstrap CIs, and McNemar's tests
- Ablation studies cleanly validate both the CDS direction and DPA payload localization as non-trivially important

## Limitations

- Benchmark size (300 premises) may be insufficient for reliable per-category statistics; authors plan to expand to 3,000
- Mechanistic analysis and intervention experiments are limited to 2 open-weight models; it is unclear if CDS/DPA can be applied to closed frontier models (where suppression is most severe)
- CDS requires paired calibration data, which may be expensive or imperfect to obtain in practice; the calibration-free DPA generally achieves lower peak correction rates
- Contextualization templates are constructed from predefined pools — generalization to the full diversity of real user requests is not demonstrated
- CDS interferes with general reasoning capability on capable models (Qwen MMLU: 77.6%→61.0%), suggesting the fixed correction direction trades factual strictness against task generalization
- The paper does not study whether fine-tuning or RLHF-based mitigation could address correction suppression more robustly than inference-time patching

## Relevance

本文揭示的"纠错抑制"现象对大语言模型在实际业务场景中的事实可靠性具有直接警示意义，尤其与智能助手、RAG系统、文档生成等依赖用户提供信息的应用高度相关；提出的DPA方法无需微调和校准数据，具有较强的工程实用价值。

## Questions for Further Exploration

1. Can CDS or DPA be adapted for closed-model APIs where internal hidden states are inaccessible? Are there prompt-level or decoding-level proxies that achieve similar effects?
2. Does correction suppression interact with retrieval-augmented generation (RAG)? If a retrieved document contains false information alongside a task request, would suppression rates be similar?
3. The paper shows suppression varies dramatically by error category (28% for False Attribution vs. 68% for Fictional Versions). What properties of a false claim determine its susceptibility to suppression under task framing?
4. Could fine-tuning on a small set of correction-vs-compliance contrast pairs (similar to CDS calibration data) produce a more robust, generalized fix than inference-time steering?
5. The mechanistic analysis reveals that attention to false claims spikes at late layers after the compliance trajectory is already committed — could this late-layer attention spike be used as a runtime signal to detect when correction suppression is occurring?
6. The paper focuses on correction of user-provided false premises. Does the same mechanism cause suppression of self-corrections during multi-turn conversations or chain-of-thought reasoning?
