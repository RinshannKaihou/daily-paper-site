> **一句话总结：** 本文提出 Dynamic Boundary Evaluation (DBE) 框架，利用 Rasch IRT 模型和 Skill-Guided Boundary Search (SGBS) 算法在安全性、指令遵循与抗拍马屁四类任务上自适应定位每个 LLM 的能力边界，在 p≈0.5 处生成有效评估信号，从根本上解决固定基准的天花板/地板效应。

> **TL;DR:** Dynamic Boundary Evaluation (DBE) is a new LLM evaluation protocol that replaces fixed benchmark pass rates with boundary-localized psychometric measurement. Using the 1-parameter logistic Rasch model calibrated on a 9-model reference panel, DBE places each model on a shared difficulty axis; when a new model falls outside the calibrated range, the Skill-Guided Boundary Search (SGBS) algorithm automatically generates and calibrates new items near that model's decision boundary (pass probability ≈ 0.5). Applied across four evaluation categories — harmful request refusal, over-refusal, constrained instruction following, and multi-turn sycophancy resistance — DBE achieves stable item-difficulty estimates (Spearman ρ ≥ 0.96 under panel leave-one-out), places SGBS-generated items near intended targets (panel-level Spearman ρ ∈ [0.90, 0.97]), and accurately estimates held-out model abilities within 0.032 logit SE for in-panel models. The framework is accepted at NeurIPS 2026 Datasets and Benchmarks Track.

---

## 核心贡献

- **提出 DBE 评估框架：** 将 LLM 能力评估从"固定题集通过率"重新定义为"在全局校准难度轴上定位每个模型的边界"，从根本上规避天花板和地板效应。
- **校准题库（Anchor Bank）：** 基于 9 个参考 LLM 的响应矩阵，用 Rasch 1PL 模型联合估计每道题的难度 β，构建覆盖安全（harmful request refusal, over-refusal）、能力（constrained instruction following）、真实性（multi-turn sycophancy resistance）四类的可复用题库。
- **Skill-Guided Boundary Search（SGBS）算法：** 通过 Thompson 采样将裸请求（bare request）与难度递增的技能（skill）组合，搜索通过率 p≈0.5 的边界项目；仅需 API 查询，适用于任意目标模型，且与所有四类评估兼容。
- **自适应评估协议：** 对新模型先用固定题库进行 Fisher 信息 CAT 定位，若模型超出已有覆盖范围则触发 SGBS 扩展题库；题库动态增长而无需全面重新校准。
- **系统性实证验证：** 通过三个层次的声明验证（题目难度的 LOO 稳定性、SGBS 边界定位效果、新模型能力估计精度）为框架的测量可靠性提供了严格的实证支撑。

## Key Contributions

- **DBE Framework:** Reframes LLM evaluation from fixed pass-rate measurement to boundary localization on a shared Rasch logit scale, solving the ceiling/floor saturation problem that plagues static benchmarks such as MMLU, JailbreakBench, and their successors.
- **Calibrated Anchor Bank:** Constructs a reusable item bank using 1PL Rasch joint MLE calibrated on a balanced 9-model reference panel (4B–72B, four model families), with item difficulties proven stable under panel leave-one-out (Spearman ρ_min ≥ 0.96; p95 |Δβ| within Linacre's 0.5-logit tolerance across all four categories).
- **SGBS Algorithm:** A bandit-based compositional search that finds boundary items for any target LLM using only API-level access, by combining bare requests with difficulty-modulating skills (up to ~10^7 compositions for Category A). Uses Thompson sampling with a boundary reward r(p̂) = 1 − 2|p̂ − 0.5| maximized at p̂ = 0.5.
- **Adaptive Evaluation Protocol:** First applies Fisher-information computerized adaptive testing (CAT) on the fixed anchor set; invokes SGBS only when the target falls outside the panel-covered range, calibrating new items onto the existing scale via the reference panel without full recalibration.
- **Four-Category Instantiation:** Covers harmful request refusal (Cat. A, 875 bare requests, 397 skills), over-refusal (Cat. A', 1,500 requests, 52 skills), constrained instruction following (Cat. B, 823 requests, 83 skills), and multi-turn sycophancy resistance (Cat. C, 1,080 requests, 47 skills).
- **IRT Model Selection Justification:** Empirically demonstrates that 1PL outperforms 2PL, 3PL, and MIRT-2D/3D on held-out NLL across all four categories, validating the specific-objectivity property required for cross-panel difficulty transfer.

## Methodology

### Core Measurement Model

DBE treats each evaluation category as an independent psychometric measurement problem. Each item $x$ is evaluated by model $m$ on $N=40$ repeated trials, producing a Binomial count $k_{m,x} \sim \text{Binomial}(N, p_{m,x})$. Pass probability follows the 1-parameter logistic Rasch model:

$$p_{m,x} = \sigma(\theta_m - \beta_x)$$

where $\theta_m$ is model ability and $\beta_x$ is item difficulty, both on a shared logit scale. The choice of 1PL over richer models (2PL, 3PL, MIRT) is empirically justified: 1PL achieves the lowest held-out NLL in all four categories despite losing in-sample BIC to more expressive models.

### Item Space via Skill-Request Composition

Each item is constructed as $x = \text{Compose}(q, \mathbf{s})$ where $q \in \mathcal{Q}$ is a bare request and $\mathbf{s} \subseteq \mathcal{S}$ is a compatible skill subset. Skills are difficulty-increasing operators (e.g., roleplay framing, length constraints, persuasive rebuttal). This compositional structure yields combinatorial item spaces of size $10^4$–$10^7$ per category without hand-crafting each item.

### Anchor Set Construction and Rasch Calibration

A random-composition pool of $K=500$ items is generated per category. Items are filtered using AVG[0.05, 0.95]: only items with panel-mean pass rate $\bar{p}_x \in [0.05, 0.95]$ are retained (175–414 anchor items across categories). The 1PL Rasch model is fit by joint MLE using L-BFGS-B; joint MLE and marginal MLE agree to Spearman $\rho \geq 0.991$ on all categories, validating the estimator choice. A two-stage coarse-fine inference protocol (first $N_c = 7$ trials, then remaining 33) reduces inference cost by ~21.9% on average.

### Skill-Guided Boundary Search (SGBS)

SGBS is a bandit algorithm operating over three coupled choices: bare request $q$, skill count $k$, and skill subset $\mathbf{s}$. In each round, a bare request is selected by Thompson sampling from per-request Beta posteriors. Skill count is sampled from an adaptive logit distribution that shifts toward harder combinations when items are too easy and toward simpler ones when too hard. Skill subsets are chosen by Thompson sampling under compatibility constraints. The boundary reward $r(\hat{p}) = 1 - 2|\hat{p} - 0.5|$ maximizes at $\hat{p} = 0.5$. Search terminates when budget $B$ is exhausted or no new boundary items are found in $R$ consecutive rounds.

### Adaptive Evaluation Protocol

For a new target model $m_\text{new}$, DBE first runs Fisher-information CAT on the fixed anchor set (item difficulties $\hat{\beta}$ frozen), updating only $\hat{\theta}_\text{new}$ via EAP inference. If $\hat{\theta}_\text{new}$ lies outside the reference panel's $\theta$-range, SGBS generates new boundary items, these are evaluated on the reference panel, calibrated onto the existing scale, and the anchor bank is extended without full recalibration.

### Reference Panel

Nine instruction-tuned open-weight models (4B–72B, spanning Qwen, Gemma, Mistral, DeepSeek-Distill families) are used as the calibration panel. Family balance limits shared pre-training leakage. Per-category $\hat{\theta}$ ranges span 2.16–2.25 logits for categories A/A'/B and 4.65 logits for category C (multi-turn sycophancy).

## Results & Highlights

### Claim 1: Anchor Item Difficulties Are Stable Under Panel Perturbation

Panel leave-one-out (LOO) tests show:
- Spearman $\rho_\text{min} \geq 0.96$ across all four categories (threshold: 0.9), confirming item rank order is preserved when any single panel member is removed.
- Median $|\Delta\hat{\beta}|$: 0.079–0.125 logit (within Linacre's 0.3-logit Rasch-stable band).
- p95 $|\Delta\hat{\beta}|$: 0.362–0.502 logit (within Wright-Stone 0.5-logit operational tolerance for all but Category B at 0.502, marginally above by 0.002).
- Tail saturation rate (items becoming all-pass/all-fail) below 2.5% in every category.
- Out-of-panel $\hat{\theta}$ recovery bounded by 0.11 logit.

The observed drift is structurally 2.5–4× larger than the sampling-noise-only null, indicating mild 1PL mis-specification, but remains within operational reuse tolerances.

### Claim 2: SGBS Finds Boundary Items Effectively

For each of the 9 panel models, SGBS is run targeting $K=50$ edge cases. After calibration:
- Mean Rasch-predicted pass probability $\bar{p}$ across EC items is close to 0.5 in most category-model combinations.
- Panel-level Spearman $\rho(\hat{\theta}_m, \hat{\beta}(\text{EC}(m))) \in [+0.900, +0.967]$ across categories — stronger models receive harder SGBS items, confirming the search is directionally correct.
- RMSE between Rasch predictions and observed pass rates ranges 0.143–0.342 across models and categories (reference: Binomial SE ≈ 0.079 at N=40, p=0.5); values are elevated but include both sampling noise and Rasch model error.

### Claim 3: DBE Effectively Evaluates New LLMs

Five held-out models from four families absent from the reference panel (Olmo-3-7B, Granite-4.1-8B, phi-4, Phi-4-reasoning-plus, Falcon-H1R-7B) are evaluated:
- Full-anchor posterior SE $\leq$ 0.032 logit for in-panel cells (except Falcon-H1R-7B on sycophancy, which saturates at $|\theta|=10$).
- In-panel adaptive-to-full gaps $|\Delta\hat{\theta}|$: 0.02–0.44 logit, comparable to Claim 1 panel-LOO baseline.
- Out-of-panel gaps increase by an order of magnitude (0.68–1.48 logit), confirming SGBS is correctly triggered for models beyond the anchor coverage range.

### IRT Model Comparison

1PL achieves the lowest held-out NLL in all four categories. 2PL costs +978 to +1,972 NLL units; MIRT-2D costs +6,365 to +16,815; MIRT-3D costs +11,364 to +20,381. The saturation of discrimination parameters toward imposed log-bounds explains the 2PL/MIRT degradation: SGBS items are anchored to a specific model's boundary, so panel responses reflect per-target patterns that single-trait IRT cannot extrapolate.

### Filter Design

The AVG[0.05, 0.95] filter is the unique 4-of-4 Pareto member across four evaluation axes (budget-normalized β SE, Fisher information retention, Bernoulli-equivalent Infit pass rate, holdout pseudo-R²), and is selected as the locked filter. It retains 175–414 anchor items with FIR ≥ 0.907 and budget_SE within 1.7% of the per-category argmin.

## Critical Assessment

**Novelty:** The key novelty is the integration of three components into a unified protocol: a reusable calibrated anchor bank, a boundary-seeking bandit search (SGBS), and an on-demand scale extension mechanism. Each component individually has precedent (IRT for LLMs: tinyBenchmarks, Fluid Benchmarking; bandit jailbreak composition: h4rm3l; adaptive CAT: Li et al. 2025), but the specific combination — especially the on-demand extension without full recalibration — is genuinely new and practically important.

**Rigor:** This is one of the more methodologically rigorous evaluation papers in recent LLM literature. The authors apply proper psychometric standards (Rasch specific objectivity, Linacre stability thresholds, Wright-Stone tolerances), report bootstrap nulls to distinguish structural mis-specification from sampling noise, conduct full 15-filter × 4-axis ablations, and verify JML vs. MML estimator agreement (ρ ≥ 0.991). The 1PL vs. richer IRT comparison is conducted on held-out NLL rather than in-sample fit, avoiding the standard overfitting trap.

**Reproducibility:** The paper provides algorithmic pseudocode for SGBS (Algorithm 1) and the adaptive evaluation protocol (Algorithm 2). Data sourcing principles are specified in detail (Appendix B). However, the protocol depends on an external API LLM as the composer and judge for categories A, A', and C — this introduces an uncontrolled vendor-specific component. Composer and judge prompt templates are in appendices, and API model choices are disclosed, but the exact model versions used (and their potential deprecation) could reduce reproducibility over time.

**Significance:** The framework addresses a genuine and growing problem — LLM benchmarks saturating within months of release. The on-demand extension mechanism is practically valuable: it means the calibration infrastructure does not need to be rebuilt as new, more capable models emerge. The four chosen categories (safety refusal, over-refusal, instruction following, sycophancy) cover important dimensions of alignment evaluation.

**Red Flags:**
1. The panel size is only M=9, and the authors acknowledge this is insufficient for MIRT-2D (requiring M ≥ 20 for stability). The 1PL assumption of unidimensionality may not hold across diverse items, and the observed structural LOO drift (2.5–4× above the null) is a mild but real signal of 1PL mis-specification that the paper acknowledges but cannot fully resolve within budget.
2. Cross-category aggregation to a single ability score is explicitly deferred to future work. Users wanting an overall safety/capability score cannot currently get one from DBE without making unsupported assumptions.
3. SGBS boundary quality is evaluated on the reference panel models themselves — the 9 models that were used in calibration. Genuine out-of-distribution models (the Claim 3 validation set of 5) show large out-of-panel gaps, which is the expected behavior triggering SGBS, but the SGBS-extended items have not been evaluated on held-out models in the same rigorous detail.
4. Skill dictionaries are drawn from curated, published taxonomies, which limits coverage to known attack/persuasion patterns. Novel strategies outside these taxonomies may produce boundary regions not discoverable by the current implementation.
5. The two-stage coarse-fine protocol saves ~21.9% inference cost on average, but category B (instruction following) with its heavy-U pool shape provides 44.9% savings while category C (sycophancy, near-uniform pool) provides only 6.5% — the cost reduction is highly category-dependent.

## Strengths

- Principled psychometric foundation with explicit connection to Rasch measurement theory, specific objectivity, and IRT literature from educational testing — this goes well beyond most ad-hoc LLM benchmark papers.
- On-demand scale extension via SGBS is a practically important innovation that decouples calibration infrastructure from the model generation cycle.
- Four-axis filter selection methodology is transparent and parameter-free at Stage C, avoiding brittleness from threshold tuning.
- Empirical validation is unusually thorough: three structured claims with pre-specified metrics, LOO stability tests, bootstrap nulls, 15-filter ablation, JML/MML comparison, and IRT model comparison on held-out NLL.
- Coverage of four qualitatively different evaluation dimensions (safety, over-safety, capability, alignment) under a single unified measurement framework.
- Only API-level query access required for SGBS — no white-box access to model weights, gradients, or logits.
- The 1PL choice is empirically justified on held-out data rather than assumed, and the justification is rigorous (all four categories, five model families compared).

---

## Limitations

- Panel size M=9 is likely too small for some downstream uses (MIRT extensions, cross-category aggregation) and is acknowledged as a bottleneck.
- The 1PL mis-specification (structural LOO drift 2.5–4× above null) may become more problematic as the model population diversifies (e.g., including mixture-of-experts, multimodal, or chain-of-thought-focused models).
- External LLM composer and judge introduces a reproducibility dependency on third-party API stability. Category B avoids this with a deterministic verifier, but A, A', and C do not.
- Cross-category score aggregation is not supported — DBE produces four separate logit-scale estimates per model, not a single composite.
- Skill dictionaries and bare-request pools are finite and drawn from existing benchmarks; they may not cover emerging attack patterns or capability domains.
- SGBS inference budget B must be set before search; the relationship between B and SGBS quality (number and diversity of recovered edge cases) is not analyzed in the main paper as a sensitivity analysis.
- The evaluation is limited to open-weight instruction-tuned models; closed-source API-only models would need all panel calibration to be done externally, which may introduce latency and cost concerns for the anchor bank construction step.

---

## Relevance

DBE 框架对 LLM 安全性与对齐评估具有直接意义：它提供了一个可复用、可扩展的心理测量基础设施，能够为任意新模型在有害请求拒绝、过度拒绝、指令遵循及抗拍马屁四个维度上生成有效的、无饱和效应的评估信号，对研究 LLM 能力测量和 benchmark 设计方法论的读者尤为值得深入阅读。

## Questions for Further Exploration

1. **Cross-category aggregation:** The paper defers composite scoring to future work. What latent factor structure, if any, underlies the four ability dimensions, and can a multidimensional IRT model be made computationally feasible at M=9?
2. **Composer and judge stability:** How sensitive are the calibrated item difficulties β to the choice of the external composer/judge LLM? A swap-in robustness check (analogous to the panel-LOO stability test) would strengthen the reproducibility argument.
3. **SGBS budget sensitivity:** What is the relationship between budget B and the number and quality of recovered edge cases? Is there a diminishing-returns regime, and can it be predicted from the pool shape diagnostic?
4. **Scale extension over time:** As SGBS-extended items accumulate, do their jointly calibrated difficulties remain stable under LOO perturbation of the extended panel? The paper covers the initial extension case but not iterative extension.
5. **Transferability to closed-source models:** The anchor bank requires N=40 trials per model per item for the reference panel. For closed-source models with API rate limits and costs, what is the minimum viable panel calibration budget, and how does it affect downstream scale stability?
6. **Over-refusal and safety trade-off:** Categories A (harmful request refusal) and A' (over-refusal) measure opposing failure modes. Is there a systematic negative correlation between θ_A and θ_A' across models, and can DBE be used to trace a Pareto frontier for the safety/helpfulness trade-off?
7. **Skill generalization:** SGBS currently uses curated skill dictionaries from published taxonomies. Would an LLM-generated skill dictionary (e.g., from red-teaming outputs) produce qualitatively different boundary items, and would the Rasch calibration remain valid?
