The first training-free knowledge editor for diffusion language models cuts forget-set log-prob by ~83 nats—7× stronger than the best gradient baseline.
TimeROME-DLM: Temporal Causal Tracing and Low-Rank Inference-Time Knowledge Editing for Masked Diffusion Language Models
Zhengtao Yao, Liuyang Song, Hongbo Zhang, Chenhao Wei, Haoyan Xu, Guang Yang, Siheng Wang · USC / PKU / Stevens / UofT
2026-06-12
Key Metric
−83 nats
Forget-set log-prob reduction on TOFU forget01 (LLaDA-8B), 7× over AlphaEdit, 0 extra VRAM.
Why This Matters
As masked diffusion LMs become competitive with autoregressive models, this is the first viable path to privacy-compliant, safe deployment of this emerging model family.
⚠ Critical Edge
The headline 83-nat result trades significant utility for forget strength—RetainLP drops from −35 to −89 nats; the utility-safe operating point (α ≤ 0.5) achieves only modest forget gains.

Methodology

Stage 1
Temporal Indirect Effect (TIE) Tracing
Run clean, corrupted (Gaussian noise on subject embeddings), and patched diffusion trajectories. Measure how splicing clean activations at coordinate (layer ℓ, step k, module m) restores object log-prob at later steps k′>k via exponential-decay weighting. Produces a heatmap of where facts localize.
Stage 2
Coordinate Selection
Average TIE heatmaps over 8 validation facts. Select (ℓ*, m*) with maximum mean magnitude. Key finding: residual stream dominates (not MLP/attention alone); applying at every denoising step is critical—single-step drops effect from 83 to 32 nats.
Stage 3
Low-Rank Edit Memory
Capture subject keys U and target deltas D at the selected coordinate across all forget facts. Pre-compute ridge-regularized Gram inverse G = (UU⊤ + λI)−1 once. At every forward pass, a hook applies h ← h + α·TopQq(D⊤GUh) to all token positions.

Efficiency

  • Install: 26s for 40 facts → 56s for 400 facts (sub-linear scaling)
  • 0 extra VRAM vs 35–51 GB for gradient methods
  • 4–14× wall-clock speedup over strongest training-time baseline
  • Sequential editing: 50 facts one-at-a-time, retain holds within ~0.5 nats

Cross-Benchmark Transfer

  • MUSE-Books verbmem-LP: −129 nats
  • MUSE-News: −122 nats
  • WHP: −9 nats · WMDP-Bio: −29 acc pts

Credibility

  • 542 total runs, 9 baselines, 6 backbones, 5+ benchmarks
  • Bootstrap 95% CIs (2000 resamples) + paired-t on every cell
  • First converged training-time unlearning baseline for MDLMs
  • ~48 GPU-hours on 4×A100-80GB — reproducible
  • Only 3 hyperparameters (α, λ, q); no custom CUDA
  • Code promised upon acceptance

Strengths

  • First systematic knowledge-editing study for MDLMs — fills an important gap
  • TIE tracing re-derives causal mediation for the diffusion paradigm, yielding a new temporal localization axis
  • Closed-form edit memory: no gradients, installs in seconds, zero extra VRAM
  • Exceptionally thorough evaluation: 9 baselines, 6 backbones, full ablations
  • Clean Pareto frontier controlled by single interpretable scalar α
  • Sequential/streaming editing with stable retain performance

Limitations

  • Utility–forget tradeoff sharp at headline α=2; practical use likely α≤0.5 with modest forget gains
  • Single global coordinate for entire forget set; per-fact selection unexplored
  • Requires explicit (subject, relation, object) triples — not applicable to unstructured knowledge
  • Cannot distinguish forget targets from same-subject neighbours (RWKU gap ~25 nats)
  • In-context relearning attacks weaken edit by ~8 nats; adversarial hardening left to future work
  • No AR-model comparison to test if TIE helps there too

Questions for Further Exploration

  1. Can per-fact coordinate selection (different ℓ* per fact) improve both forget and utility simultaneously?
  2. Does TIE-based tracing improve AR-model editing too, or is the temporal axis uniquely useful for MDLMs?
  3. Can the edit memory be hardened against in-context relearning via adversarial training or multi-objective ridge optimization?
  4. What happens with semantically contradictory edits (same subject+relation → different objects)?
  5. How does the method perform on continuous diffusion models (e.g. Diffusion-LM) vs. the discrete masked models studied here?
