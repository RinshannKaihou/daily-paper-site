LoRA-Muon: Spectral Steepest Descent on the Low-Rank Manifold

Franz Louis Cesista · Katherine Crowson · Cédric Simal · Stella Biderman

LoRA Optimizer Low-Rank Adaptation Muon Spectral Methods

Dense optimal learning rates transfer to rank-2 LoRA — and rank-32 beats the dense baseline.

Key Metric 1.776 vs 1.789 dense Rank-32 LoRA-Muon validation loss under compute-matched TinyShakespeare sweeps (±0.002).
Why This Matters Cheap rank-2 sweeps (4.3% params) recover dense optimal learning rates — eliminating expensive hyperparameter search for large-model LoRA fine-tuning.
Critical Edge All experiments are on TinyShakespeare — a tiny character-level LM that may not reflect behavior at LLM scale or on downstream tasks.

Methodology

Factor gradients → EMA first moments → Gram matrices SA, SB → Whiten + msign → Split weight-decay update

LoRA-Muon frames LoRA optimization as steepest descent on the low-rank manifold Mr. Factor gradients are whitened by the current Gram geometry, then the matrix sign function is applied — the same spectral descent rule as full-rank Muon. A novel split weight-decay rule (s = √(1 − λη)) ensures the composed decay matches full-rank behavior up to O(η²). No QR decomposition. No second moments. Pure GEMM operations.

Results

  • Learning rate 0.1 (dense best) transfers across ranks 2, 4, 8, 16, and 32 — rank 1 is the sole failure
  • Rank-32 LoRA-Muon: 1.776 ± 0.002 validation loss vs dense baseline 1.789 ± 0.002
  • Gauge invariance validated: rescaling shifts loss by −5.1×10⁻⁵ vs Spectron's 2.0×10⁻²
  • Rank-2 proxy uses only ~4.3% trainable params with 9.225× more optimizer steps under same FLOPs
  • Width and depth sweeps (rank 32) competitive with dense references; outperforms in depth sweep

Strengths

  • Elegant unification of LoRA optimizer design under steepest-descent-on-manifold framework
  • Gauge invariance: optimizer is robust to arbitrary factor parameterizations
  • Split weight-decay is a simple but impactful innovation for hyperparameter transfer
  • Rank-2 proxy result: 4.3% parameters suffice to recover dense optimal learning rate
  • Memory-efficient: first moments only, no QR, purely GEMM-heavy operations

Limitations

  • TinyShakespeare only — no validation on modern benchmarks or LLM-scale tasks
  • No fine-tuning experiments (instruction following, RLHF, domain adaptation)
  • No wall-clock, throughput, or GPU memory profiling numbers
  • No head-to-head comparison with the dominant baseline: LoRA + AdamW
  • Rank-1 failure case noted but not analyzed; trust-region split is heuristic

Open Questions

  1. Does learning-rate transfer hold at 7B+ scale on real downstream tasks (instruction following, code generation, RLHF)?
  2. How does LoRA-Muon interact with quantized training (QLoRA, GPTQ) where Gram matrices may be poorly conditioned?
  3. Can the split weight-decay rule generalize to other factorized parameterizations (Tensor Train, block-sparse low-rank)?
  4. What causes the rank-1 failure — is it fundamental to spectral norm on 1D subspaces?
  5. How does LoRA-Muon compare with AdamW + LoRA in matched-compute, matched-memory head-to-head?
  6. Does tangent-component alignment hold for deeper/wider models, or does the decoupled approximation degrade?
  7. Can gauge rebalancing be applied less frequently (e.g., every 100 steps) without losing conditioning benefits?
