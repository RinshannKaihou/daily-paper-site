Learning to Reason by Analogy via Retrieval-Augmented Reinforcement Fine-Tuning

Meta Superintelligence Labs · Rice University 2026-06-12 arXiv 2606.13680
提出通过"金标准相关性蒸馏"训练一个按推理价值（而非语义相似度）排序的检索器，并将检索到的类比推理范例注入 RLVR 训练，使模型学会在强化学习中利用外部推理模式，在 AIME 2025 上比 GRPO 提升多达 7.1 个百分点。

RA-RFT is a three-stage framework that distills reasoning-relevance labels from GPT-4o, trains a retriever via contrastive learning to rank by expected reasoning benefit, and fine-tunes the policy model with retrieved analogies under verifiable outcome rewards. On competition-level math benchmarks, it consistently outperforms standard GRPO — demonstrating that reasoning-aware retrieval is an orthogonal axis of improvement for RLVR.

Key Finding
Retrieval-augmented analogies help only under RL training, not SFT. Models must learn to exploit analogies during exploration — passive imitation is insufficient.

⚙ Methodology

1 Gold-Relevance Distillation 2 Retriever Fine-Tuning 3 RL with Retrieved Analogies

Stage 1 — GPT-4o evaluates each (query, candidate) pair and assigns binary relevance labels based on structural transferability of reasoning patterns. Problem-type filtering reduces comparisons by ~10×.

Stage 2 — A dense retriever (Reason-ModernColBERT) is fine-tuned via InfoNCE contrastive objective on the gold-relevance annotations, learning to rank reasoning-useful traces above semantically similar but reasoning-irrelevant ones.

Stage 3 — Per training problem, top-k traces are retrieved. The policy model samples G responses conditioned on query + traces, computes outcome-based binary rewards, and updates via GRPO. Key: policy update uses the same retrieved context as rollouts, preserving the importance-sampling identity.

⚖ Strengths & Limitations

Strengths

  • Clean three-stage pipeline with principled design
  • Strong gains on challenging competition-level benchmarks
  • Key negative result well-demonstrated: retrieval fails under SFT
  • Inference-time-only retrieval experiment cleanly proves models must learn to use analogies
  • Thorough ablations on retriever quality, training objective, and context diversity

Limitations

  • Gold-relevance distillation is costly (GPT-4o over all query-corpus pairs)
  • Only evaluated on math reasoning — code, logic, science untested
  • All experiments use k=1; sensitivity to k is unexplored
  • Gains diminish from +4.1 → +2.6 scaling 1.7B → 4B
  • No comparison with inference-time RAG methods beyond a brief diagnostic

🔍 Critical Assessment

Novelty: High Rigor: Solid Reproducibility: Moderate Significance: High

The core idea — training a retriever on reasoning utility rather than semantic similarity and integrating it into RLVR — is genuinely novel. The finding that RA-SFT fails while RA-RFT succeeds is an important negative result.

Experimental setup is solid: four competition-level benchmarks, two model scales, and multiple ablations. The comparison with OPSD is slightly unfair (different average@k), which the authors acknowledge.

Reproducibility is moderate — implementation details are provided, but gold-relevance distillation requires expensive GPT-4o API calls. The framework generalizes naturally beyond math without changing the optimizer.

❓ Open Questions

  1. How does RA-RFT scale beyond 4B (8B, 14B, 72B)? Does the relative improvement over GRPO continue to diminish?
  2. Can gold-relevance distillation be made more efficient via active learning or embedding-space pruning?
  3. What happens with k > 1 retrieved traces? Does diverse multi-trace retrieval improve performance further?
  4. How does reasoning-aware retrieval perform on non-math tasks — code generation, logical puzzles, scientific QA?
  5. Can the retriever be updated online during RL training, creating a feedback loop where improved reasoning changes what counts as "reasoning-relevant"?

🔗 Relevance

Highly relevant to the intersection of RLVR and RAG. Proposes a novel paradigm combining external knowledge retrieval with RL fine-tuning for reasoning. The finding that "reasoning utility beats semantic similarity" has broad implications for RAG system design.
