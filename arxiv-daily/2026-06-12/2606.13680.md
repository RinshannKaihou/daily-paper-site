> **一句话总结：** 本文提出 RA-RFT 框架，通过"金标准相关性蒸馏"训练一个按推理价值（而非语义相似度）排序的检索器，并将检索到的类比推理范例注入 RLVR 训练，使模型学会在强化学习中利用外部推理模式，在 AIME 2025 上比 GRPO 提升多达 7.1 个百分点。

> **TL;DR:** RA-RFT is a three-stage post-training framework that (1) distills reasoning-relevance labels from GPT-4o to train a retriever that ranks by expected reasoning benefit, (2) trains this retriever via contrastive learning on utility-based annotations, and (3) fine-tunes the policy model with retrieved analogous reasoning traces under verifiable outcome rewards. On competition-level math benchmarks (AIME 2024/2025, HMMT 2025, BrUMO 2025), RA-RFT consistently outperforms standard GRPO (+7.1 on AIME 2025 for Qwen3-1.7B) and other strong baselines, demonstrating that reasoning-aware retrieval is an orthogonal axis of improvement for RLVR.

## 核心贡献

- 提出了**金标准相关性蒸馏（Gold-Relevance Distillation）**方法：利用 GPT-4o 作为评判模型，直接评估候选推理轨迹是否与目标问题共享可迁移的推理模式，而非依赖表面语义相似度。
- 训练了一个**推理感知检索器（Reasoning-Aware Retriever）**：基于蒸馏得到的推理效用标签，通过对比学习（InfoNCE）微调 Reason-ModernColBERT，使检索器按推理价值排序。
- 将检索到的类比推理范例集成到**强化学习微调（RLVR）**过程中：在训练时为每个问题检索 top-k 推理轨迹作为上下文，模型在结果奖励下学会利用这些类比范例，而非简单模仿。
- 发现推理感知检索能提供**互补的解决方案策略**：不同检索上下文对同一问题产生显著不同的准确率，表明检索到的轨迹提供了多样化的推理支架。

## Key Contributions

- **Gold-relevance distillation**: Uses GPT-4o as a judge to directly assess whether candidate reasoning traces share transferable reasoning patterns with the target problem, bypassing the limitations of surface-level semantic similarity.
- **Reasoning-aware retriever training**: Fine-tunes a ColBERT-style multi-vector retriever via contrastive learning on utility-based annotations, achieving 43.5% recall@1 vs. 7.2% for the off-the-shelf checkpoint.
- **Integration of retrieved analogies into RLVR**: Injects retrieved reasoning traces into training prompts and optimizes under outcome rewards, letting the model learn to exploit analogies during exploration rather than merely imitate them at test time.
- **Empirical analysis of context diversity**: Shows that different retrieved contexts surface distinct solution strategies for the same problem, with per-problem accuracy varying substantially across retrieved traces.

## Methodology

RA-RFT consists of three stages:

1. **Gold-Relevance Distillation (Stage 1)**: Given a training set $D = \{(q_i, a_i)\}_{i=1}^N$ and a corpus $C = \{(p_j, t_j)\}_{j=1}^M$ of problems paired with teacher-generated reasoning traces, a judge model (GPT-4o) evaluates each query-candidate pair and assigns binary relevance labels based on whether the reasoning patterns are structurally transferable. To make this tractable, judge evaluation is restricted to pairs sharing the same coarse problem-type label (e.g., algebra, combinatorics), reducing comparisons by ~10x.

2. **Reasoning-Aware Retriever Training (Stage 2)**: Using the gold-relevance annotations, a dense retriever $R_\theta$ (initialized from Reason-ModernColBERT) is fine-tuned via the InfoNCE contrastive objective, encouraging it to rank reasoning-useful traces higher than semantically similar but reasoning-irrelevant ones.

3. **Reinforcement Fine-Tuning with Retrieved Demonstrations (Stage 3)**: For each training problem, the trained retriever provides top-k reasoning traces. The policy model samples G responses conditioned on both the query and retrieved traces, computes outcome-based binary rewards, and updates via GRPO with normalized advantages. A key design choice: the policy update is conditioned on the same retrieved context as the rollouts, ensuring the importance-sampling identity holds and preventing advantage estimate bias.

The framework is compatible with any policy optimization algorithm; experiments instantiate it with GRPO using 16 rollouts per problem, k=1 retrieved trace, and 64 H100 GPUs for training.

## Results & Highlights

- **AIME 2025**: RA-RFT improves over GRPO by +7.1 (Qwen3-1.7B: 48.7 vs. 41.6) and +2.8 (Qwen3-4B: 69.2 vs. 66.4) in average@32 accuracy.
- **BrUMO 2025**: Largest gain for Qwen3-4B (+5.9 over GRPO); +2.6 for Qwen3-1.7B.
- **Overall average across all 4 benchmarks**: +4.1 (1.7B) and +2.6 (4B) points over GRPO.
- **Ablation: SFT vs. RA-SFT vs. GRPO vs. RA-RFT**: Retrieval augmentation provides negligible benefit under SFT (40.5 vs. 39.7), confirming that retrieved traces only help when the training objective allows exploration, not passive imitation.
- **Retriever ablation**: Gold-relevance supervision is critical—fine-tuning lifts Reason-ModernColBERT from 40.7 to 47.4 avg. accuracy. Multi-vector late-interaction retrieval significantly outperforms single-vector (Qwen3-Embedding-4B).
- **Inference-time-only retrieval hurts**: Simply prepending retrieved traces at inference to a GRPO checkpoint degrades performance (43.3 → 37.7), confirming that the model must learn to use analogies during RL training.
- **Training dynamics**: RA-RFT starts at lower step-0 accuracy (the base model is initially distracted by unfamiliar context) but converges faster and to higher final accuracy than GRPO.

## Critical Assessment

**Novelty**: The core idea—training a retriever on reasoning utility rather than semantic similarity and integrating it into RLVR—is genuinely novel and well-motivated. The gold-relevance distillation pipeline is clean and principled. The finding that RA-SFT fails while RA-RFT succeeds is an important negative result.

**Rigor**: The experimental setup is solid: four competition-level benchmarks, two model scales, multiple ablations (training objective, retriever quality, inference-time-only retrieval), and per-problem accuracy analysis. The comparison with OPSD is slightly unfair (different average@k, results from the original paper), which the authors acknowledge.

**Reproducibility**: Moderate. Key implementation details are provided (learning rate 1e-6, k=1, 16 rollouts, 32K token limit), but the gold-relevance distillation stage requires GPT-4o API calls over the entire query-corpus product, which is expensive. The retriever is fine-tuned from a public checkpoint (Reason-ModernColBERT), and training uses the public VeRL framework.

**Significance**: The paper demonstrates a clear orthogonal axis of improvement for RLVR that doesn't require changing the reward function or optimizer. The gains are consistent and substantial, especially for the smaller model. The framework generalizes naturally to other domains beyond math.

**Red flags**: (1) Only k=1 retrieved trace is used; the sensitivity to k is not thoroughly explored. (2) The retriever is only evaluated on math reasoning; it's unclear how well reasoning-relevance distillation transfers to other domains like code or scientific reasoning. (3) The gold-relevance distillation cost is significant (all query-corpus pairs evaluated by GPT-4o, even with problem-type filtering). (4) The 4B gains are notably smaller than 1.7B gains (+2.6 vs. +4.1 overall average), raising questions about scaling behavior.

## Strengths

- Clean three-stage pipeline with clear motivation and principled design.
- Strong empirical gains on challenging, competition-level benchmarks.
- Key insight that retrieval helps only under RL training (not SFT) is well-demonstrated.
- Thorough ablation studies isolating retriever quality, training objective, and context diversity effects.
- The inference-time-only retrieval experiment cleanly shows that simply appending analogies at test time is counterproductive—the model must learn to use them.

## Limitations

- **Scalability of distillation**: Gold-relevance labeling requires GPT-4o evaluations over all (query, corpus) pairs within the same problem type, which is costly for large corpora. The problem-type heuristic may miss cross-type reasoning transfers.
- **Domain specificity**: Only evaluated on mathematical reasoning. Whether reasoning-aware retrieval transfers to code generation, logical deduction, or scientific reasoning is untested.
- **Single k value**: All experiments use k=1 retrieved trace. The effect of retrieving multiple traces (and how the model integrates them) remains unexplored.
- **Diminishing returns at larger scale**: The gap between RA-RFT and GRPO narrows from 4.1 to 2.6 overall average when moving from 1.7B to 4B, suggesting the approach may be less impactful for stronger base models.
- **No comparison with inference-time RAG methods** beyond the brief diagnostic in Table 4. A more thorough comparison with methods like R3-RAG or test-time retrieval would strengthen the paper.

## Relevance

这篇论文与强化学习推理训练（RLVR）和检索增强生成（RAG）的交叉领域高度相关。它提出了一种新颖的将外部知识检索与强化学习微调结合的范式，对改进大语言模型的推理能力具有实际意义。特别是"推理效用优于语义相似度"这一发现，对 RAG 系统的设计有广泛启发。

## Questions for Further Exploration

- How does RA-RFT scale with model size beyond 4B (e.g., 8B, 14B, 72B)? Does the relative improvement over GRPO continue to diminish?
- Can the gold-relevance distillation be made more efficient, e.g., via active learning or embedding-space pruning, to avoid evaluating all query-corpus pairs?
- What happens with k > 1 retrieved traces? Does retrieving diverse reasoning strategies and letting the model attend to them improve performance further?
- How does reasoning-aware retrieval perform on non-mathematical reasoning tasks (e.g., code generation, logical puzzles, scientific QA)?
- Can the retriever be updated online during RL training, creating a feedback loop where improved reasoning changes what counts as "reasoning-relevant"?
