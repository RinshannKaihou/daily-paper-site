Are LLMs Ready for Scientific Discovery? A Capability-Oriented Benchmark for AI Scientists

Chuhan Shi · Xiaoquan Ren · Sicheng Song · Haobo Li · Rui Sheng* · Yushi Sun*  ·  Southeast U. / ECNU / HKUST

SDABench llm-evaluation scientific-discovery benchmark data-analysis reasoning

◆ Key Metric

58.33%

GPT-5.4 top OEQ accuracy on SDA-Synth

vs. Llama 3.3-70B at 6.96% — a >50-point spread current benchmarks aggregate away.

◆ Why This Matters

Decomposes scientific data analysis into six diagnosable reasoning capabilities and pinpoints where strong models fail — method selection, relational reasoning, and final conclusions.

◆ Critical Edge

OEQ scoring leans on an unmodeled GPT-4o evaluator fallback that may favor OpenAI-family reasoning styles.

“First error only” annotation systematically undercounts cascading failures on multi-step mechanistic tasks.

Methodology

capability taxonomy · construction pipeline

Instance formalization

z = ( d, t, D, Q, g )
g = fτ(D, p)

Each instance = domain d, task type t, dataset D, question Q (MCQ + OEQ), ground truth g. Truth is deterministically replayed, not LLM-judged.

1Descriptive

summarize measurements

2Exploratory

3Inferential

4Predictive

estimate unknowns / futures

5Causal

effects of interventions

6Mechanistic

explain the producing process

Leek & Peng (2015)’s six scientific question types, reframed as six discovery capabilities spanning Biology, Chemistry, Environment, Geography, Physics.

1

Four-agent templates

Taxonomist → Extractor → Generator → Verifier builds 307 type-verified question templates.

2

Programmatic truth

g = fτ(D, p) deterministically replays the analysis — no judge-graded answers.

3

Perturbed distractors

Four MCQ strategies: Diffusion, Confusion, Randomization, Default — with shuffled option order.

Data: SDA-Real · 527 instances re-curated from 7 real benchmarks (KramaBench, BLADE, LLM-SRBench, QRData, DiscoveryBench, SciTS, ScienceAgentBench) + SDA-Synth · 6,000 from semantic DAGs + iSCMs (200 per domain×task). 60/40 train/test split; 779 underspecified candidates filtered, 324 unreliable cases removed in manual audit.

Even the best model drops ~60 points from Descriptive to Predictive — the headline evidence that grounding scales, reasoning does not.

MCQ can hurt. Kimi K2.5 on Predictive hits near-random 25.75% MCQ yet solves matched OEQ instances — explicit options induce superficial pattern matching when the underlying procedure is absent.

Error profile by pipeline stage (NER)

Where strong models still fail.

FNpeak in Predictive37.7%
RLdominate Mechanistic32.1%
RLdominate Exploratory29.9%
CCmost in Inferential10.2%
VRsmall Llama models>30%

FN=Function · RL=Relationship · CC=Conclusion · VR=Variable. As models scale, early Scope/Variable errors shrink but Function/Relationship/Conclusion errors persist — the diagnostic invisible to accuracy.

SDA-Real preserves the hierarchy: GPT-5.4 holds the top profile (Descriptive 97.2% MCQ / 96.7% OEQ), but real datasets inflate mechanistic scores via canonical variable names aligned with pretraining, while exposing exploratory brittleness on noisy real schemas.

03

Strengths & Limitations

side-by-side

+ Strengths

  • Conceptually clean reframing around six discovery capabilities grounded in Leek & Peng (2015).
  • Dual real + synthetic data with shared task coverage controls the dataset-per-task confound.
  • Deterministic programmatic ground truth removes the LLM-judge-graded-everything failure mode.
  • The five-stage, pipeline-ordered error taxonomy turns accuracy into a diagnostic tool — the most citable finding.
  • Practical side findings: MCQ can degrade reasoning; task-aligned fine-tuning lets a small open model approach frontier OEQ accuracy.

! Limitations

  • SDA-Synth’s expert-validated graphs may miss real experimental confounders — validation rigor not quantified.
  • MCQ, despite perturbed distractors, can still induce superficial pattern matching.
  • Error annotation records only the first error — undercounts cascading failures on multi-step reasoning.
  • OEQ scoring depends on a GPT-4o evaluator fallback whose agreement with deterministic matchers is unreported.
  • Closed / API model versions drift, threatening long-run reproducibility of the leaderboard.
  • MCQ↔OEQ divergences (Kimi K2.5 Predictive) reported as aggregate columns, no same-instance controlled comparison.
