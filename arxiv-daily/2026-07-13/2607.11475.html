01

How it works

3 training stages · 1-step deploy

HyperSafe attaches a lightweight classifier branch — the Safe Side Network (SSN), ~3–4% of backbone params — to a frozen fine-tuned LLM, and generates that branch's weights per-checkpoint via a hypernetwork.

For each fine-tuned model, N_cal calibration prompts produce a layer-wise activation fingerprint: each layer's averaged hidden state is split into unit direction and log-magnitude, fused into a global embedding z. Layer-wise generators {H_k} then predict each side layer's weights from local activation + global cross-layer context.

At run time, the SSN outputs a safety score λ∈[0,1]; prompts above threshold are routed to a refusal, while safe prompts (λ<0.5) bypass the SSN entirely so the base model runs unmodified. The hypernetwork is trained with a three-term objective: weight reconstruction L_recon, calibration classification L_cls, and held-out probe functional loss L_func.

Pipeline

1Fine-tune base M₀ with LoRA (r=16, α=32), merge → M_d, per dataset.
2Train SSN on frozen M_d — selective loss over BeaverTails + domain mix; shared init for weight-space alignment.
3Train hypernetwork on {(M_d, θ_d*)} pairs: L = β_r L_recon + β_c L_cls + β_f L_func.
→Deploy: 50 calibration prompts → one forward pass → load SSN. No safety data, gradients, or weight edits.

03

Strengths & Limitations

side-by-side

✓ Strengths

  • Meets a four-way constraint no prior defense satisfies at once: zero per-checkpoint training, model-aware, weight-preserving, standalone.
  • Deployment is genuinely cheap: one forward pass from 50 calibration prompts; safe prompts bypass the SSN so per-token cost is unchanged.
  • <1% harmful with <1pt accuracy cost, beating both a 12B external filter and a weight-edit baseline.
  • Cross-recipe transfer (LoRA→full-FT) and cross-benchmark transfer (BeaverTails-only → zero-shot AdvBench/HEx-PHI) strengthen the generalization claim.

! Limitations

  • Only Qwen2-7B and LLaMA-3-8B; behavior at larger scale or non-dense (MoE) architectures unknown.
  • Safety measured on English, single-turn, prompt-level benchmarks; multilingual, multi-turn, adversarial/jailbreak untested.
  • No over-refusal measurement on hard-benign prompts — false-positive cost of the generated classifier is unquantified.
  • Only OneShot benchmarked quantitatively; direct numbers vs Safe LoRA, Vaccine, RepNoise absent.
  • BeaverTails used as both training source and eval (disjoint split) — an in-distribution confound mitigated, not eliminated.
  • Ground-truth SSN collection across many fine-tunes is a non-trivial one-time cost not emphasized in the headline.
