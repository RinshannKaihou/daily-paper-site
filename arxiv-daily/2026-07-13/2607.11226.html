Double-Helix Co-Creation Loop
1Operator seeds the graph + sandbox verification function T(x).
2Cohort runs autonomously; CAS softmax allocates outbound token bandwidth (τ=0.5, B_min=0.1·B_max).
3On tool failure, cognitive disequilibrium d_t spikes → MCTS compiler scans failure trace over DSL AST, optimizing R(S)=−w₁·FPR − w₂·FNR − w₃·Len.
4Patch is Ed25519-signed and appended to the inherited Scar library — an efficiency cache, never the primary gate.
Guarantee: monotonic non-decreasing safety boundary B_g ⊆ B_{g+1} · proposed sublinear compression C(|S_t|) ≤ K·log(N(t)) + C₀ · revocation risk-tiered (L0/L1 auto after 100 safe trials; L2/L3 need HITL).

03 / Strengths & Limitations — side by side

+ Strengths

  • Clean role separation of explore / gate / bridge with distinct temps, prompts, reward weights — directly attacks the self-censorship-vs-safety trade-off.
  • Scar mechanism converts failures into reusable signed inheritable constraints, with a monotonic-boundary proof and empirical sublinear compression.
  • Strong ablation discipline: 8-config role ablation, same-model control, cohort-size scaling, temperature + CAS-weight sweeps.
  • Honest error analysis (§VII-M): three concrete failure modes — CNR spurious 4.2%, Validator over-block 4.8%, Scar over-generalization.
  • External benchmarks (AgentHarm, WebArena Lite) in addition to the custom sandbox, even if scoped.

– Limitations

  • Central RQ1/RQ3 evidence rests on a self-built sandbox with author-defined targets and τ-thresholds; real-discovery generalization unverified.
  • Scar libraries accumulate monotonically and can veto marginal-but-valid actions; L2/L3 revocation needs human-in-loop.
  • Depends on a fully specified Schema Sandbox — unmapped tool surfaces leave the Validator blind (Lean 4 mitigation is future work).
  • MCTS compilation adds ~12.4 s/failure latency during initial exploration; only mitigated after caching.
  • Single-author, non-peer-reviewed preprint from a private entity; GitHub repo and expert eval set promised, not public.
  • Tight CIs on some sandbox numbers (e.g. M8 ±368 over 20 seeds) are implausibly precise and warrant independent replication.

04 / Open Questions for Further Exploration

Q1Do the 95%/100% discovery rates survive when target coordinates and τn/τs thresholds are set by an independent party?
Q2What is the empirical breach rate when the Schema Sandbox is intentionally incomplete — the acknowledged failure mode?
Q3Can the 12.4 s/failure MCTS latency and its over-generalization be cut with a supervised rule-learner instead of AST search?
Q4Does CNR's anti-homophily penalty generalize beyond 100 pairs — e.g. on TREC subtopics?
Q5Is "Optimal Cohort Size = 3" an artifact of the fixed 150-step / token-cap budget, or does it persist under larger budgets?
Q6Does automatic L0/L1 PES revocation preserve the monotonic-boundary guarantee B_g ⊆ B_{g+1} in practice?
