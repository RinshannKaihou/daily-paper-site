> **一句话总结：** 本文提出 SDABench，一个以「六种科学推理能力（描述/探索/推断/预测/因果/机制）× 五个学科领域」组织的能力导向评测基准（527 条真实实例 + 6000 条合成实例，含 MCQ 与 OEQ 双格式），评测 15 个 LLM 后发现前沿模型在描述类任务上接近上限，但在机制/因果/预测类任务上断崖式下降，并给出五阶段错误分析（Scope/Variable/Function/Relationship/Conclusion）揭示强模型在后期阶段反复失利的瓶颈。

> **TL;DR:** SDABench reframes LLM evaluation for scientific data analysis around six discovery capabilities (descriptive, exploratory, inferential, predictive, causal, mechanistic) across five domains, releasing 527 real-data (SDA-Real) + 6,000 synthetic (SDA-Synth) instances in both MCQ and open-ended formats built via a verified programmatic-construction pipeline. On SDA-Synth OEQ, GPT-5.4 leads at 58.33% accuracy versus Gemini 3.1 Pro (56.88%) and Claude Sonnet 4.6 (55.79%), while baseline Llama 3.3-70B collapses to ~7% OEQ, exposing a >50-point spread; the paper's five-stage error taxonomy shows that as models scale, early Scope/Variable errors shrink but Function/Relationship/Conclusion errors persist — the real bottleneck for AI scientists.

## 核心贡献

- 提出了 **SDABench**，一个能力导向的科学数据分析评测基准：基于 Leek & Peng (2015) 的「六类科学问题」框架，将科学发现重新表述为六种可评测能力——描述（descriptive）、探索（exploratory）、推断（inferential）、预测（predictive）、因果（causal）、机制（mechanistic）——并覆盖生物学、化学、环境、地理、物理五个学科领域。
- 构建了两套互补的数据：**SDA-Real**（527 条来自 7 个真实科学基准的再加工实例）与 **SDA-Synth**（6000 条由「语义图 + 内部标准化 SCM（iSCM）」生成的合成实例，每个 domain-task 对 200 条），每条实例同时提供 MCQ 与开放问答（OEQ）两种格式，规避单一格式的偏差。
- 设计了**三阶段自动化构建流水线**：LLM 驱动的四代理模板构造（Taxonomist / Extractor / Generator / Verifier）→ 程序化 ground-truth 推导（`g = f_τ(D, p)`）→ 扰动式选项生成（Diffusion / Confusion / Randomization / Default 四种策略），最终产出 307 个经过单元测试与人工审核的类型化问题模板。
- 提出**五阶段错误分析框架**（Scope / Variable / Function / Relationship / Conclusion），对 15 个 LLM 的错误进行归因，得出一个反直觉但重要的结论：**模型规模增长让早期「定位与选变量」错误减少，但「选方法、刻画关系、得出结论」的后期错误依然顽固**——这是单纯准确率无法暴露的诊断信号。
- 在 15 个代表性 LLM 上系统评测（涵盖闭源 GPT-5.4 / Claude Sonnet 4.6 / Gemini 3.1 系列、开源 DeepSeek、Qwen、Kimi、GLM、Llama，以及 LoRA 微调变体），并报告了 MCQ 反而可能诱导浅层模式匹配的发现（如 Kimi K2.5 在预测任务上 MCQ 仅 25.75% 接近随机，OEQ 反而能解）。

## Key Contributions

- Introduces **SDABench**, a capability-oriented benchmark that reframes scientific data analysis evaluation around Leek & Peng's six scientific question types — descriptive, exploratory, inferential, predictive, causal, and mechanistic reasoning — spanning Biology, Chemistry, Environment, Geography, and Physics.
- Releases two complementary data components: **SDA-Real** (527 instances re-curated from seven real scientific benchmarks) and **SDA-Synth** (6,000 instances generated from semantic DAGs with internally standardized structural causal models, iSCMs), each delivered in both MCQ and open-ended (OEQ) formats with stratified 200-instances-per-domain-task coverage.
- Designs a **three-stage automated construction pipeline**: a four-agent LLM template builder (Taxonomist → Extractor → Generator → Verifier), programmatic ground-truth derivation `g = f_τ(D, p)` that deterministically replays the analysis, and a four-strategy perturbation-based distractor generator, yielding 307 type-verified question templates after unit tests and human review.
- Proposes a **five-stage error taxonomy** (Scope → Variable → Function → Relationship → Conclusion) ordered along the analysis execution pipeline, and uses it to show that scaling shrinks early grounding errors while later-stage reasoning errors persist — a diagnostic signal invisible to aggregate accuracy.
- Evaluates **15 representative LLMs** (closed: GPT-5.4, GPT-5.4 mini, Claude Sonnet 4.6, Gemini 3.1 Pro/Flash; open: DeepSeek V3.2/R1, Qwen 3.5-397B-A17B, Qwen 3-235B, Kimi K2.5, GLM 5, Llama 3.3-70B/3.1-8B; plus two LoRA fine-tuned variants) and surfaces the surprising finding that MCQ options can *degrade* performance on certain task types by inducing superficial pattern matching.

## Methodology

**Capability taxonomy and instance formalization.** SDABench adopts Leek & Peng (2015)'s six-question framework and reinterprets it as six scientific discovery capabilities: descriptive (summarize measurements), exploratory (discover patterns suggesting hypotheses), inferential (quantified-uncertainty claims about a broader phenomenon), predictive (estimate unknowns/future states), causal (estimate effects of interventions), and mechanistic (explain the process producing an outcome). Each instance is a tuple `z = (d, t, D, Q, g)` — domain, task type, referenced dataset, question, and ground-truth — released in both MCQ (`Q_MCQ`, with plausible distractors) and OEQ (`Q_OEQ`, free-form) formats.

**Data curation (SDA-Real + SDA-Synth).** SDA-Real is built by screening 24 candidate scientific datasets with three criteria (quantitative scientific objective, structured data with clear metadata, programmatically derivable answers) and selecting seven seed resources: KramaBench, BLADE, LLM-SRBench, QRData, DiscoveryBench, SciTS, ScienceAgentBench. Rather than reusing original questions, the pipeline generates new questions and derives fresh answers. SDA-Synth addresses the limitation that real datasets rarely support all six task types on a single dataset (which would confound task difficulty with dataset artifacts): (1) the construction LLM (GPT-4o) proposes scientific scenarios with measurable variables, admissible units, and ranges; (2) a semantic DAG `G=(V,E)` is built with source/internal/target variable nodes, where each non-source variable's mechanism is an explicit algebraic equation or ODE, and causal semantics come from a causal sub-DAG using iSCMs; (3) data is generated in topological order with fixed parameters, controlled measurement noise, and distractor columns. This yields 50 validated datasets (10 per domain) supporting all six task types.

**Question construction (three stages).** Stage 1 — a four-agent pipeline (Taxonomist assigns domain/task/columns, Extractor abstracts entities into a typed template τ with validity constraints, Generator binds placeholders to a dataset, Verifier rejects inconsistent/leaky/near-duplicate questions). Stage 2 — a deterministic derivation procedure `f_τ(D, p)` recomputes the correct answer by replaying the exact analysis. Stage 3 — MCQ distractors are produced by four perturbation strategies (Diffusion swaps values/roles; Confusion modifies a single parameter/operator; Randomization samples a different valid assignment; Default applies simple numeric transforms), with randomized option order.

**Validation and statistics.** The pipeline removes 779 candidates with underspecified variables / dataset-instance mismatches / taxonomy errors, and a manual audit of 2,000 sampled instances corrects 11 minor issues and filters 324 unreliable cases; MCQ instances additionally drop 86 cases with non-unique correct answers. Final: 6,231 SDA-Synth instances stratified to 6,000 (200 per domain-task pair) + 527 SDA-Real, split 60/40 train/test with paired MCQ/OEQ kept together to prevent leakage; all SDA-Real is held out for independent real-data evaluation.

**Evaluation protocol.** Zero-shot, structured JSON responses with a reasoning field (used for error annotation) and an answer field (scored). OEQ uses format-matched evaluators: numerical (relative-error tolerance), textual (exact/structured matching), and GPT-4o LLM fallback. MCQ uses option accuracy. 15 LLMs are evaluated; two LoRA fine-tuned variants of Llama-3.1-8B are trained on the SDA-Synth train split, one with standard and one with error-type-aligned supervision.

**Error analysis.** Bottom-up inspection of incorrect responses yields five ordered failure modes — Scope Error (misread target), Variable Error (wrong columns), Function Error (wrong method/formula), Relationship Error (mischaracterized interaction), Conclusion Error (final inference unsupported by prior steps) — annotated as the *first* error in the pipeline. Comparison uses normalized error rate `NER_E(S) = n_E(S) / o_E(S)`.

## Results & Highlights

- **Frontier models are far from solving open-ended scientific analysis.** On SDA-Synth OEQ, the best model GPT-5.4 reaches only **58.33%** overall accuracy, with Gemini 3.1 Pro at **56.88%** and Claude Sonnet 4.6 at **55.79%**; a LoRA fine-tuned Llama-3.1-8B with error-type supervision reaches **55.33%**, showing small open models can nearly close the gap to frontier closed models with task-aligned supervision.
- **Massive spread between frontier and weak baselines.** Llama 3.3-70B-Instruct scores just **6.96%** OEQ (Descriptive only 12.00%) and Llama 3.1-8B-Instruct **5.04%** OEQ — both far below MCQ floor-adjusted behavior — versus GPT-5.4's 58.33%, a >50-point gap that current data-analysis benchmarks aggregate away.
- **Clear task-type hierarchy.** Descriptive tasks are near-saturated for frontier models (GPT-5.4 **94.75%** OEQ Descriptive), but performance collapses on harder reasoning: GPT-5.4 drops to **34.50%** OEQ on Predictive and **47.00%** on Mechanistic; the open-source average Descriptive is **66.78%** OEQ but only **26.75%** (Predictive) and **29.72%** (Mechanistic).
- **MCQ can hurt — explicit options induce pattern matching.** Kimi K2.5 on Predictive tasks hits near-random **25.75%** MCQ despite solving matched OEQ instances, and Llama 3.3-70B stays near the 25% random floor on causal (**24.00%**) and exploratory (**23.50%**) MCQ, showing candidate options provide little benefit when the underlying analysis procedure is absent.
- **Domain variation reflects data structure, not terminology.** Environment is easiest (overall OEQ **46.9%** across models, benefiting from smoother continuous variables), while Biology is hardest (overall **39.2%**), with a particularly low Predictive score of **22.5%** attributable to high-dimensional nonlinear interactions; Geography drops on Exploratory (**30.9%**) and Mechanistic (**24.9%**) requiring spatial reasoning from symbolic data.
- **Error profiles shift with model scale.** Function Errors peak in Predictive tasks (**37.7%** NER); Relationship Errors dominate Mechanistic (**32.1%**) and Exploratory (**29.9%**); Conclusion Errors are most prominent in Inferential tasks (**10.2%**). Smaller Llama models show Variable Error rates above **30%**, while stronger models (DeepSeek R1, DeepSeek V3.2, Qwen 3.5, GPT-5.4) are instead dominated by Function Errors, and Kimi K2.5 / Claude Sonnet 4.6 / Gemini 3.1 Pro shift to higher Relationship Errors — confirming that grounding errors decrease with scale but later-stage reasoning errors persist.
- **SDA-Real preserves the SDA-Synth hierarchy.** GPT-5.4 retains the strongest profile (e.g., Descriptive **97.2%** MCQ / **96.7%** OEQ on SDA-Real), but real datasets inflate mechanistic accuracy via canonical variable names aligned with pretraining, while exposing exploratory brittleness on noisy real schemas.

## Authors & Credibility

The paper is authored by Chuhan Shi (Southeast University), Xiaoquan Ren (Southeast University), Sicheng Song (East China Normal University), Haobo Li, Rui Sheng* and Yushi Sun* (Hong Kong University of Science and Technology); the two corresponding authors are at HKUST. The work is positioned as a survey-follow-up (it cites Shi et al. 2026, "A Survey of Human-AI Collaboration for Scientific Discovery," which shares the lead author, suggesting a coherent research program). Methodologically, the benchmark construction is unusually rigorous: ground truth is derived deterministically by replaying the analysis (`g = f_τ(D, p)`) rather than by LLM judging, 307 templates pass unit tests plus human review, and the validation pipeline filters 779 underspecified candidates and 324 unreliable audited cases. The evaluation spans a credible, current 15-model lineup and releases both MCQ and OEQ formats plus a public train/test split that enables fine-tuning experiments.

## Critical Assessment

- **Novelty (high).** The capability-oriented reframing — evaluating *which scientific question a model can reason about* rather than *whether it can execute a pipeline* — is a genuine conceptual contribution. The comparison table against QRData, LLM-SR, DSBench, DA-Code, Krama, BLADE, and SciAgent shows SDABench is the only benchmark covering all six capability types (most competitors cover 2–3), and the only one combining real+synthetic data with a structured error taxonomy.
- **Rigor (mixed).** Programmatic ground-truth derivation and the perturbation-based distractors are methodologically sound and reduce evaluator noise. However, the OEQ scoring still relies on a GPT-4o LLM evaluator fallback for deterministic-match failures, which is unmodeled in the reported numbers and could systematically favor OpenAI-family reasoning styles. The "first error only" annotation explicitly undercounts cascading failures — the authors acknowledge this, but it means Relationship/Conclusion errors are likely *understated* for complex mechanistic tasks.
- **Reproducibility (good but incomplete).** Templates, derivation procedures, splits, and seeds are described as released, and the SDA-Synth construction is fully specified (semantic DAG, iSCM, topological generation). However, several underlying models (GPT-5.4, Claude Sonnet 4.6, Gemini 3.1 Pro/Flash, Qwen 3.5-397B-A17B, GLM 5, Kimi K2.5) are closed or only accessible via APIs whose versions can drift, making exact reproduction time-sensitive.
- **Significance (high for the AI-for-science evaluation subfield).** The central diagnostic — that scaling fixes grounding but not later-stage reasoning — is actionable for model developers and directly motivates targeted training on assumption selection, relational reasoning, and intermediate verification.
- **Red flags / specific concerns.** (1) The named **MCQ↔OEQ inversion** for Kimi K2.5 Predictive (25.75% MCQ vs. solvable OEQ) is a striking claim that rests on a single model-task cell and would benefit from a controlled same-instance MCQ-vs-OEQ comparison rather than aggregate columns. (2) SDA-Synth mechanisms are "validated by domain experts" but the paper does not quantify inter-expert agreement or how many mechanisms were reviewed, leaving the synthetic→real transfer claim (Section 5) on thin evidentiary ground beyond the broad task-hierarchy preservation. (3) Biology's anomalously low Predictive score (22.5%) could reflect a *dataset* artifact (high-dimensional nonlinear synthetic biology graphs) as much as a model limitation; the confound the authors designed SDA-Synth to avoid is not fully exonerated here. (4) Llama 3.1-8B/3.3-70B's near-floor OEQ scores (5–7%) sit oddly above non-trivial MCQ scores (25–30%) — worth confirming these are not JSON-parsing failures inflating the gap.

## Strengths

- Conceptually clean reframing around six scientific discovery capabilities grounded in a citable prior framework (Leek & Peng 2015).
- Dual real+synthetic data with shared task coverage controls for dataset-per-task confounds that plague prior benchmarks.
- Deterministic programmatic ground truth removes the LLM-judge-graded-everything failure mode of many recent benchmarks.
- The five-stage, pipeline-ordered error taxonomy turns aggregate accuracy into a *diagnostic* tool and yields the paper's most citable finding (grounding errors shrink, later-stage errors persist).
- Practically useful side findings: MCQ can degrade reasoning; task-aligned fine-tuning lets a small open model approach frontier closed-model OEQ accuracy.

## Limitations

- SDA-Synth's synthetic semantic graphs, while expert-validated, may not capture the full complexity of real experimental confounders and measurement artifacts — and the paper does not quantify the validation rigor.
- The MCQ format, despite perturbation-based distractors, can still induce superficial pattern matching in some models.
- Error annotation records only the *first* error, undercounting cascading failures on multi-step mechanistic reasoning.
- OEQ evaluation depends on a GPT-4o LLM evaluator fallback whose agreement with deterministic evaluators is not reported, introducing an unmodeled scoring variance.
- Closed/API model versions can drift, threatening long-run reproducibility of the headline leaderboard.
- The strong MCQ↔OEQ divergences (e.g., Kimi K2.5 Predictive) are reported as aggregate columns without a same-instance controlled comparison.

## Relevance

SDABench 直接回应了「LLM 作为 AI 科学家」的核心评估问题：它把科学数据分析拆解为六种可诊断的推理能力，并用错误分析定位强模型在「选方法、刻画关系、得出结论」上的顽固短板——对关心 LLM 推理评测、agent 能力诊断与科学发现自动化（AI for Science）方向的研究者来说是高相关、可直接引用和复用的基准工作。

## Questions for Further Exploration

- Would a same-instance controlled MCQ-vs-OEQ study reproduce the Kimi K2.5 Predictive inversion, or is it an artifact of aggregate column comparison?
- How much of the Biology Predictive collapse (22.5%) is a *model* limitation versus a *synthetic-data artifact* from high-dimensional nonlinear biology graphs, and does it persist on held-out real biology datasets?
- Can the five-stage error taxonomy be used as a *training signal* (beyond the error-type LoRA variant) — e.g., curriculum learning that progressively targets Function/Relationship/Conclusion errors — and does that transfer to out-of-distribution scientific datasets?
- How sensitive are the OEQ leaderboard rankings to the GPT-4o LLM-evaluator fallback, and would swapping to a panel of judges change model ordering?
- Does the "grounding errors shrink, later-stage errors persist" finding hold for agentic settings (tool use, code execution, multi-turn verification), or do tools close the Function/Relationship error gap that pure-prompt evaluation exposes?
