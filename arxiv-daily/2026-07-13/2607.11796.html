Methodology — a realization-theoretic reading

measurement + mechanism

The instrument

Because the Mamba-1 / Falcon-Mamba state matrix Ah is diagonal, a channel's scalar output is an exact sum of per-mode contributions mh,t,i. Outer products over a length-L window yield a small symmetric PSD Gram tensor Gh ∈ ℝN×N whose quadratic form (Eq. 4) returns the exact squared output error of dropping any subset of modes — offline, at any budget, from one reverse cumulative sum.

The mechanism is isolated by frozen-signal counterfactuals: rerun the decomposition with Bt, Ct, or Δ frozen at its corpus mean; an energy-matched control and a "keep only Bt live" variant act as controls. Compression is measured by comparing four rules at matched budget, plus a timestep-activity ranking the mechanism predicts to fail.

Three measures read from Gh

  • ρ(r) Migration gap — ratio of an in-sample per-window oracle's error to a fixed-set error. ρ ≈ 1 ⇒ static is fine; ρ ≪ 1 ⇒ the important set migrates.
  • churn Jaccard distance between top-r sets across windows, split within vs. across domain against a half-window null.
  • PRh Participation ratio (Σe)² / Σe² — the channel's effective order at the operating point.

Setup. Released Mamba-1 (130M–2.8B), 7B Falcon-Mamba, two Mamba-2 checkpoints (130M, 780M); 1024-token windows, 32 per domain across English prose / code / technical text, disjoint 24-window-per-domain evaluation; budget r=8 of N=16 (Mamba-1/Falcon) and r=64 of N=128 (Mamba-2); all seeds fixed; fp32 reference path with fused kernels disabled; ~1 GPU-day on a single GB10 node.

03

Strengths & limitations

side-by-side

◆ Strengths

  • Actually exact, not heuristic. The diagonal structure makes the Gram identity (Eq. 4) exact; dual validation is to machine precision (≤ 2.3×10⁻⁷).
  • Mechanism-grounded story. Frozen-signal counterfactuals give a causal attribution — Bt carries the migration, Δ does not — that directly contradicts the assumption behind activity-based pruning.
  • Intellectual honesty. Exact only for the diagonal case; scheduled result is headroom, not deployed saving; pre-registered and missed churn threshold disclosed; strict scoring conventions adopted.
  • Breadth across the model zoo — Mamba-1 130M–2.8B, deployed 7B Falcon-Mamba, Mamba-2 — so the phenomenon is a property of the selective-SSM design, not one checkpoint.
  • Cheap to reproduce (~1 GPU-day) with public code and public checkpoints.

◆ Limitations

  • Exactness restricted to the per-channel diagonal case (Mamba-1, Falcon-Mamba). Mamba-2's scalar-per-head decay + gated RMSNorm breaks the offline Gram identity past the norm — there the gap is computed online, pruning error measured empirically.
  • Two-pass = realizable headroom, not a deployed saving. Every cheap scheduler tried captures only 2–26% of the oracle's gain.
  • Depth geometry is architecture-specific: the contiguous mid-network band with a static tail is a Mamba-1 finding; larger models migrate in punctuated layers.
  • Downstream eval ≤ 1024 tokens — the effect on much longer contexts, where slow-mode migration should be strongest, is untested.
  • Churn is modest in absolute terms (1.15–1.32×, below the pre-registered 1.5×); the case rests on ρ and the frozen-Bt counterfactual.
  • A learned scheduler that anticipates the mask — the thing that would turn headroom into a real system — remains open future work.
04

Questions for further exploration

open directions
  1. Can a cheap learned scheduler (small predictor / recurrent mask predictor) anticipate the per-window top-r set from a short prefix well enough to turn two-pass headroom into a real deployed saving — and how does its accuracy degrade with context length?
  2. Does input-driven migration amplify, disappear, or change mechanism at multi-thousand-token contexts where slow modes are supposedly most active — and does it interact with retrieval / in-context learning?
  3. The instrument is exact only for the diagonal case; can a comparable (even approximate) per-mode error bound be derived for Mamba-2's gated RMSNorm, or for the structured (non-diagonal) state matrices of S4/S5/Diagno/OPT?
  4. The frozen-Bt result implies activity-based pruning scores the wrong signal; can Bt-magnitude or Bt-conditional energy serve as a better static importance score, closing the gap to realized-energy ranking?
  5. Migration concentrates in slow modes and a depth-localized band — is the band's location predictable from pretraining statistics, and can it be regularized during training (reduce migration → cheaper static pruning, or amplify it → more exploitable headroom)?
  6. The domain probe reads the input domain from mode energies at 99.7%; is the regime signature doing computational work, or is it a readable but functionally incidental correlate?
