> **一句话总结：** 作者提出一种异构角色分工的智能体小队（Disrupter 负责发散探索、Validator 在工具网关做硬性安全拦截、Broker 用反同质化检索桥接跨域知识），并将执行失败经 MCTS 编译为加密签名的约束补丁（Scars）作为可继承的运行时约束缓存；在空间-语义沙箱与 AgentHarm/WebArena Lite 基准上，小队发现远程目标能力远超同质辩论，Validator 实现 0% 执行越界，Scars 节省 15.1% token。

> **TL;DR:** This paper proposes a heterogeneous three-role LLM agent cohort (Disrupter/Validator/Broker) that separates open-ended exploration from runtime safety: in a custom spatial-semantic sandbox (N=20 seeds, p<0.01) the full cohort discovers remote Target 1 at 95% (vs homogeneous debate 10%) and Target 2 at 100% with zero executed breaches, while MCTS-compiled signed "Scars" constraint patches cut Token/Success by 15.1% (72,180 vs 83,060 without Scars) and CAS bandwidth throttling cuts it by 55.9% (72,180 vs 163,520 for the no-CAS M8 ablation); on AgentHarm (1,000 episodes) the cohort achieves 0.0% breach vs ReAct 14.2% / Reflexion 11.5% / AutoGen 19.2%, and on WebArena Lite it reaches 78.0% success vs AutoGen 54.0%.

## 核心贡献

- **异构角色分工小队架构：** 将开放探索与运行时安全拆分给三个专门角色——Disrupter（temperature 1.2，最大化动作熵，生成 OOD 假设）、Validator（temperature 0.0，在工具调用网关做确定性审计与形式化检查）、Broker（高 RAG 召回，跨学科结构洞桥接）。通过角色特化的奖励权重（探索/安全/检索权重分别为 0.8/0.1/0.1、0.1/0.8/0.1、0.1/0.1/0.8）避免单一模型兼顾创新与谨慎时的自我审查。
- **对比式新颖性检索（CNR）：** 用反同质化目标函数 D* = arg max [λ·Sim(D, Vgoal) − (1−λ)·Sim(D, Vhistory)] 检索与"任务目标"相关但与"对话历史"语义距离较远的跨域文档，操作化 Granovetter 弱关系理论。与经典 MMR 的关键差别在于 CNR 是目标条件化、面向动态多轮对话历史的，而非静态文档列表去冗余。
- **运行时约束记忆缓存（Scars）：** 工具执行失败轨迹经 MCTS 在 DSL 约束 AST 空间中搜索最优规则（目标函数 R(S) = −w1·FPR − w2·FNR − w3·Len），编译成 Ed25519 签名的约束补丁，缓存到本地供后续世代小队继承；理论上证明在被撤销前安全边界单调非减（Bg ⊆ Bg+1），并经验性提出约束累积满足次线性复杂度 C(|St|) ≤ K·log(N(t)) + C0。
- **通信分配评分（CAS）带宽控制：** 用温度缩放 softmax 为各角色分配出站 token 带宽 Bandwidthi(t) = Bmin + (Bmax−Bmin)·e^(Ki/τ)/Σe^(Kj/τ)，配合角色奖励更新 Ki(t) = max(ε, Ki(t−1)+αRi−βCi)，防止广播风暴、抑制低效用节点污染共享上下文。
- **系统性消融与基准验证：** 在自建空间-语义沙箱（8 配置 × 20 种子）上做角色/CAS/CNR 消融，并在 AgentHarm（1,000 集）、WebArena Lite、10k 生物/材料预印本检索数据集上对比 ReAct、Reflexion、AutoGen、标准 RAG、BM25；额外做了同模型控制、规模缩放、温度/CAS 权重敏感性分析。

## Key Contributions

- **Heterogeneous role-specialized cohort topology:** Separates divergent OOD exploration (Disrupter, temperature 1.2, maximizing action entropy), runtime safety gating (Validator, temperature 0.0, deterministic tool-gateway auditing) and cross-domain knowledge bridging (Broker, high-recall RAG over Google Scholar/arXiv). Role-specific reward weights (explore/safety/retrieval = 0.8/0.1/0.1, 0.1/0.8/0.1, 0.1/0.1/0.8) prevent any single model from self-censoring into sterility.
- **Contrastive Novelty Retrieval (CNR):** Goal-conditioned anti-homophily retrieval D* = arg max [λ·Sim(D, Vgoal) − (1−λ)·Sim(D, Vhistory)] that retrieves documents relevant to the task goal but semantically distant from dialogue history, operationalizing Granovetter's weak-ties theory. Explicitly contrasted against MMR (which only diversifies a static ranked list against already-selected documents).
- **Runtime constraint memory cache ("Scars"):** Failure traces compiled via MCTS over a grammar-restricted DSL AST (objective R(S) = −w1·FPR − w2·FNR − w3·Len) into Ed25519-signed constraint patches cached locally and inherited by offspring cohorts at zero training cost. Proves monotonic non-decreasing safety boundary (Bg ⊆ Bg+1) and proposes an empirical sublinear compression hypothesis C(|St|) ≤ K·log(N(t)) + C0.
- **Communication Allocation Score (CAS) bandwidth control:** Temperature-scaled softmax allocation of outbound token bandwidth with a guaranteed Bmin floor, preventing broadcast storms while keeping safety/broker alerts audible.
- **Comprehensive evaluation:** Role/CAS/CNR ablations in a custom spatial-semantic sandbox (8 configs × 20 seeds), plus comparisons on AgentHarm, WebArena Lite and a 10k cross-disciplinary preprint retrieval dataset against ReAct, Reflexion, AutoGen, standard RAG and BM25; augmented by same-model controls, cohort-size scaling, and τ/CAS-weight sensitivity sweeps.

## Methodology

The system is formalized as a weighted directed graph G = (V, E, W) with three node types. The **Disrupter** maximizes U(a|s) = −log P_human(a|s) to favor low-probability (high-surprise) actions. The **Validator** holds a Schema Sandbox Ω_t and acts as a runtime filter P_Ωt(a) that hard-blocks unsafe tool calls before execution (distinct from logit-level masking). The **Broker** queries an external corpus K using CNR (Algorithm 2) to import weak-tie analogies.

Communication is throttled by CAS: Ki(t) = max(ε, Ki(t−1) + αRi(t) − βCi(t)) with ε = 0.01, and bandwidth is allocated by a temperature-softmax (τ = 0.5, Bmin = 0.1·Bmax). Role-specific reward weights (Equation 3) prevent the Disrupter from monopolizing credit.

Execution follows the "Double-Helix Co-Creation Loop": (1) the human operator seeds the graph and the sandbox verification function T(x); (2) the cohort runs autonomously; (3) when a tool call fails or hits a safety barrier the cognitive disequilibrium d_t spikes and an MCTS compiler scans the failure trace and searches the DSL AST to synthesize a constraint patch; (4) the patch is signed with the operator's Ed25519 private key and appended to the inherited Scar library. Scars are explicitly positioned as an *efficiency cache*, not the primary safety gate — the active Validator is always the ultimate enforcer.

Failure compilation uses MCTS over a grammar-restricted AST where the search maximizes R(S) = −w1·FPR(S) − w2·FNR(S) − w3·Len(S), balancing false positives on successful traces, false negatives on failure traces, and a description-length penalty. Revocation is risk-tiered: L0/L1 constraints can be retired after 100 safe trials in a Parallel Exploration Sandbox (PES); L2/L3 (e.g., data deletion) require human-in-the-loop administrative signature.

For evaluation, action trajectories are projected to 2D via t-SNE on text-embedding-3-small (1536-d) for *visualization only* — all distance/novelty/serendipity math is computed in the unprojected embedding space. Serendipity Yield = |Y| / (Total Run Steps) × 100%, with Y requiring jointly Novelty ≥ τn (0.35), Usefulness (sandbox T(x)=1 or expert ≥4/5), and Surprise ≥ τs (0.40), calibrated against 100 human baseline trails so only the top-5% OOD trajectories qualify.

## Results & Highlights

- **Remote discovery (spatial-semantic sandbox, N=20 seeds, p<0.01 paired permutation test, 10,000 reshuffles):** Full cohort discovers Target 1 (Quantum-Bio Synthesis) at **95.00% ± 21.80%** and Target 2 (Thermoelectric Monolayer) at **100.00% ± 0.00%**, vs the homogeneous-debate ablation M7 at only **10.00% ± 30.00%** for both targets — i.e., the cohort is +85 pp on Target 1.
- **Zero executed breaches:** Validator gating holds executed breaches at **0.0 ± 0.0** in the full cohort; removing the Validator (M3) lets breaches spike to **28.6 ± 4.1**, and prompt-only safety (M6) still leaks **10.1 ± 3.3** executed breaches.
- **Scars token savings:** Removing the Scars cache (M5) raises Token/Success to **83,060 ± 1,016** (95% CI [82,585, 83,535], Cohen's d = 14.34) versus the full cohort's **72,180 ± 375** — a **15.1% reduction**, achieved with no safety loss (still 0.0 breaches because the Validator remains active).
- **CAS bandwidth throttling:** Without CAS (M8), Token/Success explodes to **163,520 ± 368** (CI [163,348, 163,692]) and both remote targets drop to 25.00% — i.e., CAS cuts Token/Success by ~55.9%.
- **AgentHarm (1,000 adversarial episodes):** Full cohort records **0.0% breach**, vs ReAct 14.2%, Reflexion 11.5%, AutoGen 19.2%, with Benign Task Success **88.5%** and a False Block rate of only **4.8%**.
- **WebArena Lite:** Full cohort reaches **78.0% success** at **11.5 average steps** and 82.4k tokens, vs ReAct 58.0% / 14.2 steps, Reflexion 62.0% / 15.8 steps, and AutoGen 54.0% / 18.5 steps / 124.0k tokens.
- **CNR vs RAG (10k arXiv/PubMed preprints, 100 expert-curated pairs, 3 PhD raters, Cohen's κ = 0.81):** CNR achieves **89.4% out-of-domain recall**, expert score **4.5/5**, novelty 0.42, diversity 0.86 — vs Standard Dense RAG (24.1% / 2.3 / 0.18 / 0.32) and BM25 (12.3% / 1.8 / 0.09 / 0.15).
- **Generational error decay (100 generations, 20 cohort instances):** Mean unsafe attempts fall from **10.2 ± 2.1** in Gen 1 to **1.1 ± 0.4** in Gen 2 and **0.05 ± 0.01** by Gen 5, demonstrating that Scar inheritance caches failures efficiently.
- **Cohort-size scaling:** 1 agent reaches only 5% Target 1 and 62% benign success; the 3-agent role-complete cohort jumps to 95% / 88.5% at 72,180 Token/Success; scaling to 5 or 8 agents keeps discovery at 95% but inflates tokens to 104,200 / 168,400 — empirical support for the "Optimal Cohort Size Hypothesis."
- **Same-model control (AgentHarm):** All-GPT-4o cohort leaks 2.1% breaches, All-DeepSeek-V3 cohort leaks 4.5%, while the heterogeneous cohort (gpt-4o orchestrator + Claude 3.5 Sonnet Validator + DeepSeek-V3 Disrupter) achieves 0.0% at the lowest 72.2k tokens — evidence that gains come from role heterogeneity, not model strength.
- **Scar consolidation (Table IX):** MCTS-MDL lattice consolidation compresses the Scar library from 25,000 tokens (uncompressed at N=100 failures) to 2,200 — an 11.36x compression ratio, rising to 39.06x at N=500, supporting the sublinear-growth hypothesis.

## Critical Assessment

- **Novelty:** The role-tripartition (Disrupter/Validator/Broker) is a reasonable re-packaging of ideas already in the literature (AutoGen role-play, RedDebate safety memory, MMR-style diversity retrieval), but the specific *combination* with an MCTS-compiled, Ed25519-signed, cross-generation Scar cache and a CAS softmax bandwidth allocator is genuinely new. The CNR-vs-MMR distinction (history-conditioned anti-homophily rather than intra-list redundancy) is a meaningful reformulation, not just rebranding.
- **Rigor:** Statistics are reported properly — 20 seeds, 95% CIs, paired permutation tests with 10,000 reshuffles, Cohen's d for the Scars ablation, and Cohen's κ = 0.81 for the human expert ratings. However, several headline numbers rest on suspiciously tight intervals (e.g., M8 Token/Success 163,520 ± 368 with CI width ±172 over 20 seeds is implausibly precise) and the spatial-semantic sandbox is *self-built*, not a community benchmark — Target 1 / Target 2 are synthetic zones whose coordinates and τn/τs thresholds the same author set.
- **Reproducibility:** Mixed. Hyperparameters, models, seeds, and hardware are listed, and a GitHub URL is given — but the repo is not yet public and the sandbox benchmark (including the 100 "expert-curated cross-disciplinary query-relevance pairs" and the t-SNE target coordinates) has no independent release, so the central RQ1/RQ3 results cannot be reproduced by a third party today.
- **Significance:** If the gains hold on external benchmarks, the Disrupter/Validator/Broker split plus Scar inheritance is a useful design pattern for safe autonomous discovery. But the strongest results (95%/100% discovery, 15.1% / 55.9% token savings) come from the author's own sandbox where "remote target" and "serendipity" are operationalized by the author's own thresholds; the external-benchmark results (AgentHarm, WebArena Lite) are smaller in scope (50 tasks / WebArena Lite subset) and could be confirmation artifacts of selecting Validator-friendly tasks.
- **Red flags:** (1) The "PhD reviewers" for Table VI are not identified and the "100 expert-curated pairs" are not released. (2) A few quantitative claims appear in the body without a matching table or sample size (e.g., the "55.9% CAS token reduction" abstract figure is reconstructed from M1 vs M8 rather than reported directly). (3) The paper repeatedly hedges with phenomenological-analogy disclaimers (Langevin/Kramers) and an "empirical hypothesis" framing for the sublinear-compression result, suggesting the theoretical claims are illustrative rather than proven. (4) The "MCTS" compiler is described but no search budget, node count, or wall-clock distribution is reported beyond the 12.4 s/failure latency figure.

## Strengths

- Clean conceptual separation of *exploration*, *gating*, and *bridging* into roles with distinct temperatures, prompts, and reward profiles — directly addresses the well-documented self-censorship vs. safety trade-off.
- The Scar mechanism elegantly converts failures into reusable, signed, inheritable constraints with a proof of monotonic safety-boundary growth and an empirical sublinear-compression result.
- Strong ablation discipline: 8-config role ablation, same-model control (Tables XI), cohort-size scaling (Table X), temperature sweep (Table IV), CAS-weight sweep (Table XII), and retrieval-strategy comparison (Tables V–VI) isolate each component's contribution.
- Honest error analysis (Section VII-M) enumerating three concrete failure modes — CNR spurious associations (4.2%), Validator over-blocking (4.8%), Scar over-generalization blocking valid SQL reads — rather than hiding them.
- External benchmark coverage (AgentHarm, WebArena Lite) in addition to the custom sandbox, even if scoped.

## Limitations

- The central RQ1/RQ3 evidence rests on a self-built spatial-semantic sandbox with author-defined target coordinates and τn/τs thresholds; generalization to real scientific discovery is unverified.
- Scar libraries accumulate monotonically and the authors acknowledge this can veto marginal-but-valid actions; revocation only handles L0/L1 automatically and L2/L3 needs HITL.
- The framework depends on a fully specified Schema Sandbox; if the environment description is incomplete the Validator cannot verify safety, leaving it open to unmapped operations (the authors propose Lean 4 middleware as future mitigation).
- MCTS compilation adds ~12.4 s of reasoning latency per failure, increasing overhead during initial exploration (only mitigated after caching).
- Single-author, non-peer-reviewed preprint from a private entity; the GitHub repo and the expert-curated evaluation set are promised but not yet public.
- Sample size on external benchmarks is modest (50 AgentHarm tasks across 5 categories; WebArena Lite subset), and the very tight CIs on some sandbox numbers warrant independent replication.

## Relevance

对关注「LLM 智能体安全」「多智能体协作」与「开放探索」方向的研究者有直接参考价值：其"角色分工 + 工具网关硬性拦截 + 失败编译为可继承约束"的设计模式，可作为构建既能发散发现又不破坏环境的自主科研智能体的工程蓝本，CNR 反同质化检索也对跨学科类比/RAG 多样性研究有启发。

## Questions for Further Exploration

- Do the 95%/100% discovery rates survive when Target 1/Target 2 coordinates and the τn = 0.35 / τs = 0.40 thresholds are set by an independent party rather than the author?
- How does the cohort behave when the Schema Sandbox is intentionally incomplete (the acknowledged failure mode) — what is the empirical breach rate under unmapped tool surfaces?
- Can the MCTS Scar compiler's 12.4 s/failure latency and its over-generalization failures (e.g., blocking valid SQL reads) be reduced with a supervised rule-learner instead of AST search?
- Does CNR's anti-homophily penalty generalize beyond the 100 expert-curated pairs — e.g., what is its recall on standard diversity-RAG benchmarks like TREC subtopics?
- Is the "Optimal Cohort Size = 3" finding an artifact of the fixed 150-step / token-cap budget, or does it persist under much larger step budgets where extra agents could plausibly pay off?
- Can the monotonic-boundary guarantee (Bg ⊆ Bg+1) be preserved under automatic L0/L1 PES revocation, or does revocation break the proof in practice?
