01

Methodology — from POSG to per-agent bandits

MACE

Formalization

N agents interact over T rounds; each agent i selects a peer ai,t to query. Latent state st encodes task + unknown capabilities. Framed as a POSG ⟨S, A, O, P, Q, R⟩; objective is minimizing cumulative regret to the best peer in hindsight.

ri,t = Ri(st, at)

Tractable reduction

Finite-horizon POSG is NEXP-hard and peers are non-stationary. MACE approximates each agent as an independent contextual bandit solved with LinUCB, using relational (not scalar) features to drive optimism-based exploration.

select argmaxa  θ̂⊤x  +  α·√(x⊤A-1x)

Relational features (the mechanism)

  • i. Response diversity — peer’s answer vs. agent’s own (information gain)
  • ii. Peer distinctiveness — distance from the rest of the network
  • iii. Historical performance — exploitation signal
  • iv. Interaction round — exploration intensity decays over time

Theory (matches empirics)

Theorem 1. MACE cumulative regret O(√(T log T)).
Theorem 2. Greedy non-exploring policy incurs Ω(δT).
Corollary 1. Exploration benefit governed by capability diversity δ — vanishes as δ→0, grows linearly otherwise.

03

Strengths & limitations

Critical read

Strengths

  • Clean, memorable diagnosis — premature commitment, bimodal selection — isolated by a controlled two-armed bandit probe.
  • Principled POSG formalization paired with a tractable, well-motivated LinUCB approximation.
  • Theory and empirics reinforce each other — the δ-driven bound explains large heterogeneous gains and muted homogeneous results.
  • Honest about scope: explicitly reports that the regret advantage vanishes as δ→0.

Limitations

  • No scalar-UCB / Thompson-sampling ablation — the marginal value of relational features is not isolated.
  • Small scale: ≤10 agents, ≤5 rounds. Open-world behavior (hundreds of agents, long horizons) untested.
  • Closed-model dependency (GPT-5/GPT-4) makes exact reproduction time-sensitive; APIs may drift.
  • No reported confidence intervals on headline deltas (e.g., 0.5404 vs 0.4596 on GPQA, n≈198).
  • No per-component feature ablation; reward design (absolute + improvement) not isolated either.
04

Open questions

For further work
Q1How much of the gain is relational-feature design vs. simply any LinUCB optimism bonus over scalar peer statistics?
Q2Does bimodal premature commitment persist if agents explicitly track posterior beliefs over peers — or is MACE’s full structure required?
Q3How does MACE scale to dozens/hundreds of heterogeneous agents with long horizons, where the independence assumption is most strained?
Q4Can relational features be learned end-to-end — and would that close the gap in the homogeneous δ→0 regime?
Q5Is “In-Context Exploration worse than Random” robust to prompt engineering, or an artifact of the specific phrasing used?
Q6Could the δ-driven theory guide an adaptive policy that auto-dials exploration intensity to the observed pool diversity?
