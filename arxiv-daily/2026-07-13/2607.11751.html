When Local Monitors Miss Compositional Harm: Diagnosing Distributed Backdoors in Multi-Agent Systems

Yibo Hu · Ren Wang — Illinois Institute of Technology · arXiv:2607.11751 · 2026-07-13
Key finding

Locally benign payload fragments defeat every per-step monitor — only the decoded, reassembled view restores detection.

Key metric · end-to-end ASR

Why this matters
A formal, capacity-independent account of when runtime monitors must fail against compositional attacks — pinpointing representation discovery, not wider observation windows, as the real open problem.
Critical edge
All main results use one exfiltration payload; the marker-free features are regex-tuned to its libraries, so cross-payload-family generalization is asserted, not shown.
01

Methodology — a diagnostic, not a bake-off

A planner dispatches subtasks to K role agents that emit fragments; a final agent assembles and executes them while a runtime monitor watches the trajectory. The threat is a distributed backdoor: the attacker controls fragment content but not the monitor, and aims to assemble a malicious payload whose every individual fragment looks safe.

The design sweeps a single locality axis across four beds — R-Judge, Collaborative Shadows, AgentDojo, DemonAgent — holding the assembled payload fixed and varying only how much local evidence each fragment leaks. Six monitor families (taint, per-step statistical, marker-based / marker-free assembly, codec-blind, decoded-view gate) are compared on the same fragments.

The headline recovery witness is a marker-free one-class assembly monitor — an isolation forest over seven hand-built features, fit only on benign traffic, so its test positives are attack families it never saw.

Proposition 1 · the hard limit

AUROC ≤ ½ + TV

For any randomized per-fragment rule, TPR − FPR ≤ ε when benign vs. attack local views sit within ε in total variation. A capacity-independent ceiling — a stronger local classifier cannot help.

Def 1 · local benignness — every fragment is benign in the monitored view, yet the assembled object is malicious.
Def 2 · ε-indistinguishability — at each position, TV between benign and attack local views ≤ ε.
Prop 2 — alarm-if-any aggregation over K steps still cannot escape; it only buys detection by spending false-positive budget.

03

Strengths & limitations

Strengths
  • Exact, not empirical. Proposition 1 + the alarm-aggregation corollary (Prop 2) give a capacity-independent impossibility result.
  • Clean single dial. Isolates "removing local evidence" from "splitting" — a distinction prior distributed-backdoor work conflated.
  • Triangulated. Collaborative Shadows + AgentDojo + DemonAgent + four served model scales; the boundary is not one detector's artifact.
  • Honest scope. Flags the decoded-view gate as a diagnostic ceiling and the covert natural-language reassembly case as unsolved.
  • Negative-result hygiene. Clopper–Pearson bounds on 0 / 50 cells, shared 5% FPR, {1,5,10}% operating-point sweep, rollout model-identity check.
Limitations
  • Code payloads only. Natural-language compositional harm (beliefs, plans, permissions) untouched — the one-class monitor fails there at AUROC 0.406.
  • Narrow geometry. K = 3 in a single episode, one fixed exfiltration payload; longer horizons and adaptive attackers are out of scope.
  • Payload-specific features. Marker-free regex keyed to requests / os.environ / http://; transfer is across encodings, not payload families.
  • Oracle gate. The decoded-view gate needs the encoding family handed to it — by design not a deployable, oracle-free defense.
  • Single-prompt LLM probe. The 6 / 50 no-codec number is a single DeepSeek-V4-Flash call — a lower bound on agentic reconstruction, not a fundamental limit.
04

Open questions

  1. How does the boundary shift for larger or variable K, multi-session assembly, or fragments spread across asynchronous tool calls?
  2. Can the marker-free signal extend to qualitatively different payloads (SQL injection, file deletion, privilege escalation) without payload-specific feature engineering?
  3. What is the right exposing representation for non-code harms — commitments, plan graphs, belief deltas, permission state — reachable without a decoder in hand?
  4. Does an adaptive attacker optimizing fragments to minimize code-structure density (not just TV) collapse the 0.874 AUROC?
  5. Could a search-and-verify LLM loop close the gap between no-codec search (6 / 50) and the codec-aware gate (50 / 50)?
  6. Does the R-Judge length-artifact result generalize — how many published monitor evaluations silently reward shortcut features?
