LayerTracer: A Joint Task-Particle and Vulnerable-Layer Analysis Framework for Arbitrary Large Language Model Architectures

arXiv: 2604.20556v1

Date: 2026-04-22

https://arxiv.org/abs/2604.20556v1

一句话总结： 作者提出 LayerTracer 框架，通过把每层隐藏态投影到词表概率空间定义"task particle"（目标词概率首次大幅上升的层）和"vulnerable layer"（掩码扰动后 JS 散度最大的层），在 Qwen3 0.6B–14B 上量化 task-particle 集中在深层 N/2 以后，且 0.6B 呈 U 形脆弱性、14B 趋平——为 Qwen3.5 为何采用 3:1 混合注意力提供量化依据。
TL;DR: LayerTracer is an architecture-agnostic framework that localizes (a) a "task particle" — the first layer where the target token's LM-head probability spikes (max Ratio) — and (b) a "vulnerable layer" — the layer whose output JS divergence under a mask perturbation is largest. Applied to Qwen3 0.6B, 4B, 8B, 14B on AntSynNET, task particles consistently sit in the second half of the network, and vulnerability flattens as scale grows, supplying a metric-driven rationale for hybrid architecture module ratios.

核心贡献

  • 统一定义 task particle（Ratio_max）与 vulnerable layer（JS_max）两大架构无关指标。
  • 提出 Layer-wise Relative Stability (LRS) 衡量层级鲁棒性分布的均匀度。
  • 系统实验 Qwen3 四种规模，给出 Qwen3.5 采用 3:1 Full/Linear Attention 比例的工程解释。

Key Contributions

  • A two-phase pipeline (Algorithm 1) computing probability-trajectory-based localization and JS-based vulnerability.
  • LRS metric for quantifying architecture-wide robustness uniformity.
  • Empirical finding: deep-emergence of task particles is scale-invariant relative to network depth, but vulnerability profile smooths with parameter count.
  • Architecture-design principle: anchor Full Attention in deep regions while confining trainable modules to robust mid-depth zones.

Methodology

For each layer l, extract hidden state h_l, project via LM head to P_l, record target token probability p_l; task particle = argmax_l (p_l − p_{l−1})/p_l. For vulnerability, apply uniform mask noise to the attention output of layer l, forward pass, compare final distribution Q_l to unperturbed P via JS divergence; vulnerable layer = argmax_l JS(P || Q_l). Evaluated on 500 samples from AntSynNET across 10 hierarchical feature categories on 8 A800 GPUs.

Results & Highlights

  • Qwen3-0.6B: U-shaped vulnerability (shallow + deep layers most fragile), high LRS.
  • Qwen3-8B and 14B: flatter vulnerability profile, lower LRS.
  • Task particles localize to relative depth > N/2 consistently across scales.
  • Provides a "depth-aware functional alignment" design principle.

Authors & Credibility

Critical Assessment

  • Novelty: The combination of probability-trajectory task localization and JS-vulnerability is a reasonable fusion of existing techniques (tuned lens + mask perturbation).
  • Rigor: Single dataset (AntSynNET, 500 samples, 10 categories) limits statistical power; no error bars.
  • Reproducibility: Algorithm is spelled out but code absent; architecture details (mask intensity, noise amplitude) sparse.
  • Significance: The "why 3:1" reverse-engineering angle is attractive but speculative — Qwen3.5 ratio may simply be an inference-efficiency heuristic.
  • Red flags: No comparison against established layer-localization methods (Logit Lens, Causal Tracing, Probing). No ablation on mask intensity. Overclaims "universal support for LLM structure design and interpretability research" from one model family and one benchmark.

Strengths

  • Architecture-agnostic framing that works for Transformer, GateDeltaNet, Mamba.
  • LRS is a simple, interpretable scalar summary of layerwise robustness.
  • Connects interpretability metrics to concrete hybrid design choices.

Limitations

  • Only Qwen3 family tested despite the "arbitrary architecture" claim.
  • Single dataset, single task type (antonym/synonym).
  • Mask perturbation can be architecture-dependent in subtle ways not addressed.
  • No downstream validation that LRS predicts deployment robustness.

Relevance

对接你关注的 hidden states & interpretability 与模型可靠性：提供了层级脆弱性与任务启动层的量化工具，便于研究 silent data corruption / quantization error 在不同层的放大效应。
