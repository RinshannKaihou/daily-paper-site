> **一句话总结：** 本文系统分析了DDR5仅使用单个32-bit子通道的DIMM架构，推导了32-bit x BL16 = 64字节的缓存行粒度恒等式，通过roofline模型量化了性能边界（带宽密集型负载吞吐量下降40-60%，延迟敏感型负载影响小于10%），揭示了DDR5-4800频率下性能低于DDR4-3200的带宽反转现象，并指出JEDEC SPD规范Byte 235已原生支持该配置但生态系统存在标准化空白。

> **TL;DR:** This paper provides the first formal architectural analysis of DDR5 DIMMs populating only a single 32-bit sub-channel (DDR5-SC1). The author derives the transaction-width identity 32-bit x BL16 = 64 bytes = L_x86, proving a single sub-channel can satisfy an x86 cache line fill atomically. Using a roofline model, performance degradation is quantified: 40-60% throughput loss for bandwidth-bound workloads (CPU AI/LLM serving, scientific simulation), 30-50% for video transcoding and iGPU workloads, but less than 10% for latency-dominated tasks (web/office, POS). A bandwidth inversion at DDR5-4800 makes single-SC DDR5 slower than DDR4-3200. AMD AM5 is architecturally incompatible due to its unified 64-bit UMC training model. The JEDEC SPD spec (JESD400-5D.01, Byte 235 bits 7-5 = 000) already natively encodes single-SC modules, but five ecosystem gaps remain: termination spec, platform-compatibility signalling, module nomenclature, XMP/EXPO schema extensions, and validation test procedures. BOM savings of 35-45% are achievable. Written by the GIGABYTE overclocker who set DDR5 frequency world records using this exact configuration.

## 核心贡献

- 推导了JEDEC子通道设计的交易宽度恒等式：32-bit x BL16 = 64 bytes = L_x86，证明单个子通道可在一个突发事务中完成一次x86缓存行填充
- 使用roofline模型量化了单子通道配置在不同工作负载类别下的性能影响，包含6类共10种工作负载的详细性能亏损估计
- 发现了DDR5-4800频率下的带宽反转现象：单子通道DDR5-4800有效带宽约16 GB/s，低于DDR4-3200的约21 GB/s
- 分析了Intel iMC（32-bit独立调度域）与AMD UMC（64-bit统一训练模型）的根本架构差异，阐明AMD AM5不兼容的技术根因
- 揭示了JEDEC SPD规范（JESD400-5D.01）Byte 235已原生编码单子通道模块，但生态系统存在5项标准化空白（终端规范、平台兼容信号、命名约定、XMP/EXPO扩展、验证测试）
- 提供了基于HWBOT验证的DDR5超频世界记录实证基础（DDR5-10022/11618/13530，横跨三代Intel平台）

## Key Contributions

- Derives the transaction-width identity 32-bit x BL16 = 64 bytes = L_x86, formally proving that a single DDR5 sub-channel satisfies one x86 cache line fill atomically without partial fills, split transactions, or pipeline stalls
- Quantifies performance impact across 10 workload classes using a roofline model with explicit per-workload deficit ranges (Table III): CPU AI/LLM serving and scientific simulation suffer 40-60%; iGPU 1080p+ suffers 35-55%; video transcoding 30-50%; CPU-bound gaming 15-35%; web/office and POS under 8%
- Identifies a bandwidth inversion at DDR5-4800: single-SC effective throughput (approx. 16 GB/s) falls below DDR4-3200 (approx. 21 GB/s), meaning DDR5 upgrades at this speed tier offer no bandwidth gain
- Demonstrates architectural incompatibility with AMD AM5: the Zen 4 UMC trains all 64 data lines as a coordinated group, causing unterminated DQ stubs on the unpopulated sub-channel to fail POST -- characterized as a fundamental design constraint, not a firmware issue
- Reveals that JEDEC SPD spec JESD400-5D.01 Byte 235 (bits 7-5 = 000) already natively encodes single-SC modules, while identifying five surrounding ecosystem gaps: termination specification, platform-compatibility signalling, module nomenclature, XMP/EXPO profile schema extensions, and validation test procedures
- Provides empirical foundation from HWBOT-verified DDR5 frequency world records across three Intel generations (Alder Lake DDR5-10022, Raptor Lake DDR5-11618, Arrow Lake DDR5-13530), all set by the author using this configuration

## Methodology

The paper combines formal mathematical derivation, analytical performance modeling, and specification-level analysis:

1. **Transaction-width derivation (Section IV):** Starting from the JEDEC DDR5 specification's BL16 mandate, the author derives T_SC = (W_bus/8) x BL = (32/8) x 16 = 64 bytes, showing exact correspondence with the x86 cache line size L_x86. This identity is validated against DDR4's T_DDR4 = (64/8) x 8 = 64 bytes for consistency.

2. **Roofline performance model (Section V):** Following Patterson and Hennessy's framework, workloads are classified by arithmetic intensity I relative to the bandwidth ceiling P/B. Halving B from B0 to B0/2 compresses the bandwidth-limited region proportionally. Effective throughput is estimated at Beff approx. 22.4 x 0.85 approx. 19 GB/s for single-SC DDR5-5600, with DDR5 bus efficiency estimated at 80-90% vs DDR4's 70-80%.

3. **Platform compatibility analysis (Section VI):** Comparative architectural analysis of Intel iMC (independent 32-bit sub-channel scheduling domains with per-SC PHY instances) vs AMD UMC (unified 64-bit channel with coordinated training across all 64 data lines). The analysis traces the incompatibility to signal-integrity violations from unterminated DQ stubs during UMC training.

4. **Specification analysis (Section IX):** Direct examination of JEDEC JESD400-5D.01 Section 11.11, Table 104, Byte 235 encoding, confirming that bits 7-5 = 000 already designates 1 sub-channel per DIMM. The ecosystem gap is identified at five layers above the SPD encoding.

5. **Cost analysis (Section VIII):** BOM decomposition under pre-supply-constraint pricing ($3.50-$4.50 per 16 Gbit x8 die), yielding a structurally invariant 50% die-count ratio and 35-45% total BOM reduction normalized against fixed PCB and assembly costs.

6. **Empirical evidence (Section IV-D):** Three HWBOT-verified DDR5 frequency world records serve as proof that Intel iMC single-SC operation is well-characterized electrically, though the author explicitly notes these do not validate the SPD-encoded module path or make a performance argument for volume-tier productization.

## Results & Highlights

- **Cache-line sufficiency:** A single 32-bit sub-channel at BL16 transfers exactly 64 bytes per burst, matching the x86 cache line size. Cache miss latency is not degraded (latency depends on DRAM timing parameters, not bus width).

- **Two single-SC DIMMs in dual-channel = one standard DDR5 DIMM:** At DDR5-5600, two single-SC modules in a dual-channel topology deliver 44.8 GB/s peak, identical to one standard DDR5 DIMM -- a cost-disaggregated path to standard bandwidth that consumes both DIMM slots (Table I).

- **Bandwidth inversion:** Single-SC DDR5-4800 delivers approx. 16 GB/s effective, below DDR4-3200's approx. 21 GB/s. Migration from DDR4-3200 to single-SC DDR5-4800 offers no bandwidth gain (Table II).

- **Workload deficit ranges (Table III):** CPU AI/LLM serving: 40-60%; Scientific simulation (HPC): 40-60%; iGPU 1080p+: 35-55%; Video transcoding: 30-50%; CPU-bound gaming (>240 fps): 15-35%; Software development: 8-15%; Web/office: 2-8%; POS/kiosk: <3%.

- **iGPU inadequacy:** At DDR5-5600 single-SC, sustained bandwidth (approx. 19 GB/s) falls within the 18-22 GB/s demand range for 1080p iGPU, leaving no margin for concurrent CPU traffic. Single-SC modules are not appropriate for laptop deployments with sustained 1080p+ graphics targets.

- **AMD AM5 incompatibility:** Fundamental to the UMC design -- not firmware-patchable. Characterized as outside the addressable deployment range for the lifetime of the AM5 platform.

- **BOM savings:** 35-45% reduction vs standard DDR5, dominated by halving DRAM die count (dies constitute 60-70% of module BOM). Cost advantage is most meaningful during the 16 Gbit-die generation; the 32 Gbit transition narrows it.

- **SPD already supports it:** JESD400-5D.01 Byte 235 bits 7-5 = 000 designates 1 sub-channel. All three world records were set on SPD-conformant configurations. The gap is at ecosystem layers, not the spec level.

## Critical Assessment

**Novelty:** High for the academic literature -- this is the first formal, peer-review-style treatment of a configuration that has existed in practice since 2021 but lacked rigorous analysis. The transaction-width identity derivation, while straightforward once the JEDEC spec is examined closely, has not been formally stated in the literature. The roofline model application to this specific problem is appropriate and novel in scope.

**Rigor:** Strong overall. The mathematical derivations are clean and verifiable. The roofline model is a well-established framework (Patterson & Hennessy), and its application here is sound. The platform compatibility analysis traces the AMD incompatibility to a specific root cause (unterminated DQ stubs causing signal-integrity violations during coordinated 64-bit training). The JEDEC spec analysis is direct text citation (Table 104, JESD400-5D.01), not inference.

**Reproducibility:** Good. All parameters (bus widths, burst lengths, speed grades, JEDEC spec references) are publicly available. The roofline model can be independently reconstructed. The BOM analysis states assumptions explicitly (pre-supply-constraint die pricing).

**Significance:** High practical relevance. The work addresses a real 2025-2026 industry need driven by AI-related DRAM supply reallocation. Vendor-specific implementations have already appeared in 2026. The standardisation gap identification (five specific deficiencies) is actionable and well-scoped for a JEDEC working group.

**Potential concerns:**
- Performance estimates are model-derived, not empirically validated on production hardware. The author acknowledges this explicitly and treats the numbers as bounds rather than precise predictions.
- The 0.85 DDR5 bus efficiency estimate is cited as an estimate from published DDR5 controller analyses, not directly measured for single-SC configurations.
- Single author limits cross-validation. However, the author's unique position as both the practitioner (world-record overclocker) and the analyst mitigates this concern.
- The 15-25% queueing latency increase estimate under >60% bus utilization (Section IV-E, scope limitation 3) is stated but not rigorously derived.
- The cost analysis uses pre-supply-constraint pricing; the author notes this and uses BOM ratios as the structurally invariant figure.

## Strengths

- First systematic academic treatment of a practically important but previously unanalyzed DDR5 configuration
- Clean mathematical framework: the T_SC = L_x86 identity is simple, correct, and illuminating
- Transparent about scope and limitations (three explicit scope qualifications in Section IV-E; four in Section X-A)
- Detailed workload classification table (Table III) with per-workload deficit ranges provides actionable guidance for OEM deployment decisions
- The "two single-SC DIMMs = one standard DIMM" observation (Table I) is a non-obvious and practically useful insight
- JEDEC spec analysis is direct textual citation, not speculation -- the author correctly identifies that the standard already anticipated this configuration
- Five specific ecosystem gaps are enumerated with concrete proposals for resolution
- The AMD incompatibility analysis is thorough and traces root cause to training model architecture, not just "doesn't work"
- Strong empirical backing from HWBOT-verified records across three Intel generations
- Clear deployment suitability/boundary analysis (Section VIII-D)

## Limitations

- Performance numbers are model-derived (roofline), not silicon-validated. The author is explicit about this but it remains the paper's primary weakness.
- Bus efficiency estimates (0.85 for DDR5, 0.70-0.80 for DDR4) are approximations; single-SC DDR5 bus efficiency has not been measured.
- The iGPU bandwidth analysis uses typical demand ranges (18-22 GB/s at 1080p) that may not capture all scenarios.
- Platform compatibility analysis is documentation-based; the AMD incompatibility could change with future UMC revisions (though the author argues this is unlikely within AM5's lifetime).
- The paper focuses exclusively on x86 (64-byte cache line); ARM (typically 64-byte) and other architectures receive no discussion.
- No discussion of dual-rank single-SC configurations or x4 organization, which could affect the capacity/die-count analysis.
- The cost analysis is sensitive to DRAM supply conditions that are inherently volatile.
- Single-author paper; the world records cited were set by the author himself, which could raise perceived conflict-of-interest concerns (though HWBOT independent verification mitigates this).

## Relevance

本文对DDR5内存子通道架构的性能建模和标准化分析具有硬件架构研究的参考价值。roofline模型在内存子系统带宽分析中的应用方法，以及对JEDEC标准生态空白的系统性识别方法，对计算机体系结构领域的研究具有方法论借鉴意义。

## Questions for Further Exploration

- How do real-world benchmarks (SPEC CPU, Geekbench, gaming FPS) compare to the roofline model projections? Are there memory controller prefetch or scheduling optimizations that alter the deficit percentages?
- What is the feasibility of AMD adding single-SC support through AGESA firmware modifications (e.g., training assertion bypass for SPD-flagged modules)? The author dismisses this but does not fully analyze the engineering effort.
- Could the 35-45% BOM savings translate into meaningful market share for single-SC modules at the 8 GB tier, given that DDR4 supply is contracting and two single-SC modules match one standard DDR5 module's bandwidth?
- What are the power consumption and thermal implications of single-SC operation, particularly for SO-DIMM and LPCAMM2 form factors where thermal envelopes are constrained?
- How does the transition from 16 Gbit to 32 Gbit dies affect the economic viability? The author notes the cost advantage narrows, but at 32 Gbit a single-SC module becomes 16 GB -- does this create a new viable tier?
- The paper mentions vendor-specific implementations have appeared in 2026 under various trade names. How has the lack of standardisation manifested in real compatibility issues?
