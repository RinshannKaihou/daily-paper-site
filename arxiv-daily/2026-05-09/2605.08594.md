> **一句话总结：** FLARE利用互素数（coprime）测试向量，在脉动阵列中仅需一次测试即可高概率定位到具体 faulty PE 行（INT16下256x256阵列成功率>98%），无需硬件冗余，测试开销不到一次推理GEMM tile的1%。
> **TL;DR:** FLARE assigns pairwise coprime integers as test-vector entries to a weight-stationary systolic array; a permanent weight-register fault produces an output deviation whose divisibility signature uniquely identifies the faulty row. For INT16 on 256x256 arrays (TPU-scale), one test pass localizes the faulty PE with >98% probability and under 1% GEMM-tile cost, with zero hardware modification. A second pass via ratio computation gives exact localization; single-bit errors are exactly localized in one round using odd coprime entries.

## 核心贡献

- 提出基于互素整数（coprime integers）的纯算法级故障定位方法 FLARE，在 weight-stationary 脉动阵列中实现 PE 级行定位，无需任何硬件冗余或架构修改
- 核心机制：利用偏差的整除性签名（divisibility signature）——由于测试向量各元素两两互素，非故障行的元素整除偏差当且仅当整除误差值本身，从而唯一识别故障行
- 支持一般的有界误差模型（bounded error model, |e| <= M），覆盖多比特缺陷、桥接故障、参数漂移等，比先前 RunSAFER/Periodic Online Testing 仅处理单比特 stuck-at 故障更广泛
- 单轮定位：INT16 下 256x256 阵列成功率 >98%；单比特故障使用奇数互素元素可一轮精确确定
- 两轮扩展：当单轮不足时，第二轮通过比率计算实现精确确定位，适用于所有实际阵列尺寸
- 可同时估计故障误差值 e，支持选择性缓解策略
- 通过 Qwen2.5-0.5B 权重注入故障实验验证理论界限，SCALE-Sim 测量周期开销

## Key Contributions

- Introduces FLARE, a purely algorithmic framework for PE-level (row-level) fault localization in weight-stationary systolic arrays with zero hardware overhead
- Core insight: pairwise coprime test-vector entries create unique divisibility signatures. Since gcd(x_k, x_i*) = 1 for k != i*, a "wrong" row k survives the divisibility test only if x_k divides the error e itself, enabling localization via elimination
- Covers a general bounded error model (|e| <= M = 2^(b-1) - 1), broader than prior dataflow-aware methods (RunSAFER, Periodic Online Testing) that only validate single-bit stuck-at faults
- Formal probabilistic guarantee (Theorem 3): P_fail <= (L-1)/M when using L largest primes each exceeding M/2. For INT16, L=256 gives P_fail < 0.008
- Prime-power construction: replaces small primes with their largest power <= M (e.g., 2 -> 64, 3 -> 81), dramatically reducing ambiguous multiples and P_fail for INT8
- Single-bit errors exactly localized in one round using odd coprime entries (since no odd integer divides a power of two)
- Two-round ratio-based scheme provides guaranteed exact localization as fallback
- Beyond localization, estimates the fault error magnitude e, enabling selective mitigation
- Validated analytically and empirically via fault injection on a custom cycle-accurate simulator using Qwen2.5-0.5B weights

## Methodology

**System Model:** An L x K weight-stationary systolic array where PE(i,j) holds a weight register (Wreg), input register (Ireg), and MAC unit. Column j computes y_j = sum_i w_ij * x_i. A permanent Wreg fault in PE(i*, j*) produces deviation Delta_{j*} = e * x_{i*}.

**Fault Model:** Permanent weight-register faults with error e bounded by |e| <= M = 2^(b-1) - 1 (bounded error model). Single-bit errors (e = +/- 2^beta) are a special case. Assumes at most one faulty PE per column (A1) and the faulty column is pre-identified via nonzero deviation (A3).

**FLARE Algorithm (Algorithm 1):**
1. Assign the L largest primes <= M as pairwise coprime test-vector entries x_1, ..., x_L
2. Compute expected output y = w^T * x offline; stream x through the array
3. For each column with nonzero deviation Delta, test divisibility x_k | Delta for each candidate row k
4. A row survives only if x_k | Delta AND |Delta/x_k| <= M (magnitude check)
5. By Proposition 2: row k != i* survives iff x_k | e. Since primes are coprime, false candidates are eliminated unless x_k happens to divide e

**One-Round Guarantee (Theorem 3):** P_fail (probability of non-unique localization) is bounded by sum of floor(M/x_k)/M over non-faulty rows. Using the L largest primes minimizes this.

**Two-Round Extension (Section 4.4):** A second test vector using consecutive integers enables ratio-based exact localization: the ratio Delta^(1)/Delta^(2) = x_{i*}^(1)/x_{i*}^(2) uniquely identifies row i*. Uses derangement of primes (for L <= pi(M)) or consecutive integers (for L > pi(M)).

**Prime-Power Construction:** For INT8 (M=127, pi(127)=31), small primes like 2 have 63 multiples in {1,...,127}. Replacing 2 -> 64, 3 -> 81, 5 -> 125, 7 -> 49, 11 -> 121 reduces ambiguous multiples dramatically, cutting P_fail from impractical to ~0.534 at L=31.

**Fundamental Constraint:** L <= pi(M) for unique coprime assignment. For INT16 (M=32767, pi=3512), this comfortably covers 256x256 arrays. For INT8 (pi=31), arrays beyond 31 rows need the two-round scheme.

## Results & Highlights

**Analytical Results:**
- INT16 (M=32767, pi=3512): P_fail < 0.008 for L=256, < 0.032 for L=1024. Single-round suffices for all TPU-scale arrays (128x128 and 256x256)
- INT8 (M=127, pi=31): Raw primes give P_fail up to 0.534 at L=31; prime-power construction matches this but with fewer ambiguous candidates
- Single-bit errors: exact one-round localization guaranteed using odd primes (30 odd primes for INT8)

**Empirical Validation (500 fault-injection trials, Qwen2.5-0.5B weights):**
- INT8: empirical failure rate tracks Theorem 3 bound closely across L in {4,8,16,31,40,50,60}
- INT16: failure rate negligible across all tested dimensions up to L=256
- Graceful degradation beyond pool boundary: at L=32 (INT8), 30/32 fault locations still benefit from single-round localization
- 95% Wilson confidence intervals shown

**Cycle Overhead (SCALE-Sim 3.0.0):**
- Sketch vector is 1xL GEMM against the LxL array (weights already loaded)
- At most 2 additional cycles per test tile
- Below 0.1% of a single inference GEMM tile at 128x128 and 256x256

**Comparison Table (Table 1):**
| Method | Fault Model | Test Cycles | Localization | HW Modif | Error Est |
|--------|-----------|------------|--------------|----------|-----------|
| RunSAFER | single-bit | 2 | PE col | no | no |
| Periodic Testing | single-bit | 4 | PE col | no | no |
| HW-Runtime | single-bit | 4 x array-size | PE row & col | yes (~1.74% area) | no |
| FLARE | bounded error | 1 or 2 | PE row & col + |e| | no | yes |

## Critical Assessment

- **Novelty: High.** The core insight — that pairwise coprime test entries create divisibility signatures surviving systolic accumulation — is both mathematically elegant and practically novel. No prior work achieved PE-level localization without hardware augmentation. The connection to the prime-counting function as a fundamental scalability limit is a genuine theoretical contribution.
- **Rigor: Strong.** Formal theorems with proofs, explicit assumptions (A1-A3), precise probability bounds, inclusion-exclusion exact computation. The graceful degradation analysis beyond the coprime pool boundary shows intellectual honesty.
- **Reproducibility: High.** Algorithm is deterministic with clear pseudocode (Algorithm 1). Prime/power constructions are specified exactly. Simulator uses publicly available model weights (Qwen2.5-0.5B). SCALE-Sim is open-source.
- **Significance: Practically significant.** Directly applicable to TPU-scale (256x256) systolic arrays at INT16 with negligible overhead. The comparison table shows FLARE achieves PE-level localization that previously required hardware augmentation (HW-Runtime with 1.74% area overhead), at fewer test cycles and under a more general fault model.
- **Red flags / concerns:**
  - **Single-fault-per-column assumption (A1):** The paper explicitly assumes at most one faulty PE per column. Multiple faults in the same column are not handled — this is a significant limitation for aging/production scenarios where defect clustering may occur. The authors do not discuss this limitation prominently.
  - **Weight-register-only scope:** The method exploits the multiplicative structure Delta = e * x_i* of Wreg faults. Input-register and MAC faults have different error propagation and are explicitly noted as out of scope. For a comprehensive testing solution, additional techniques would be needed.
  - **INT8 is borderline:** P_fail ~0.534 at L=31 means single-round is unreliable for INT8 at pool boundary. The two-round fallback helps but adds complexity. For production INT8 accelerators (common in edge inference), the method is less compelling.
  - **Simulator-only validation:** No real hardware or FPGA prototype. The empirical results are from a Python simulator with only 500 trials per data point. While the analytical bounds are the stronger contribution, the limited empirical evaluation leaves some questions about real-world deployment nuances (timing, control flow, weight-loading interactions).
  - **No comparison with ABFT-based column detection + brute-force row testing:** The paper compares against RunSAFER, Periodic Online Testing, and HW-Runtime, but does not compare against a simple baseline of column detection followed by per-row activation testing (which would need L passes but is deterministic).
  - **The bounded error model assumes e ~ Uniform({1,...,M}):** Real fault distributions may not be uniform. The probability bounds depend on this assumption.

## Strengths

- Elegant and creative application of number theory to a practical hardware reliability problem — coprime divisibility is a natural fit for the fault signature problem
- Zero hardware modification required; integrates with existing execution pipeline
- Extremely low testing cost (under 1% of one GEMM tile, at most 2 extra cycles)
- Broader fault model (bounded error) subsumes single-bit stuck-at as a special case
- Provides both fault localization AND error magnitude estimation
- Clean theoretical framework with explicit probability bounds
- Prime-power construction is a clever optimization for INT8
- Graceful degradation analysis beyond pool boundary
- Safety-critical relevance (ISO 26262 FTTI compliance potential)

## Limitations

- Single-fault-per-column assumption; multiple simultaneous faults in one column are not handled
- Restricted to weight-register permanent faults in weight-stationary dataflow; input-register and MAC faults require different techniques
- INT8 single-round reliability is marginal at pool boundary (P_fail ~0.53 for L=31)
- Validated only in simulation (500 trials per configuration), not on real hardware or FPGA
- The method requires the faulty column to be pre-identified (separate detection step, potentially via prior work like RunSAFER)
- No discussion of integration complexity with existing production testing workflows
- The uniform error distribution assumption for probability bounds may not match real fault distributions
- Graceful degradation beyond pool boundary is characterized but not formally bounded

## Questions for Further Exploration

- Can the coprime framework be extended to handle multiple simultaneous faults per column (e.g., via additional test rounds or Chinese Remainder Theorem-based decoding)?
- How would the framework apply to input-register faults, which produce a different error structure (Delta = w_i* * epsilon instead of e * x_i*)?
- What is the practical integration path with existing column-level fault detection (RunSAFER) in production TPU firmware?
- How do the probability bounds change under non-uniform error distributions observed in real silicon (e.g., clustered defects, aging patterns)?
- Could the prime-power coprime pool idea be combined with compressed sensing techniques for fault localization in sparse or irregular array topologies?
- Is the framework applicable to other systolic dataflows (output-stationary, row-stationary) or to emerging architectures like spatial accelerators with reconfigurable dataflows?
