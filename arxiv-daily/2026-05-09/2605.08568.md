> **一句话总结：** 本文提出 PARSE 框架，通过离线训练线性路由器为每个输入 prompt 动态选择 SVD 秩子集，配合模式缓存和内核融合，在保持相同压缩率下将 LLaMA-7B 的平均任务准确率提升最多 10%，同时实现 2.5 倍预填充和 2.4 倍解码加速。

> **TL;DR:** PARSE reformulates SVD-compressed weight matrices as mixtures of rank experts and trains a lightweight linear router to dynamically select prompt-specific rank subsets, supervised against dense-model outputs on C4. A rank pattern cache eliminates online routing overhead at inference by retrieving cached subsets for semantically similar prompts (Spearman rho=0.78) and reusing selections across decoding steps (>86% overlap across 60 steps). Integrated with four SVD-based methods (SVD-LLM, Dobi-SVD, Basis Sharing, SAES-SVD), PARSE improves average zero-shot accuracy by up to 10 percentage points at compression ratio 0.6 on LLaMA-7B and achieves up to 2.5x prefill and 2.4x decode speedup over native SVD execution.

## 核心贡献

- 识别了 SVD 压缩 LLM 中静态秩截断的两个关键局限：(1) 最优秩因输入 prompt 而异；(2) 所选秩对校准数据集高度敏感，导致跨域性能退化。
- 提出 PARSE 框架，将 SVD 分解后的权重矩阵重构为秩专家混合体（Mixture of Rank Experts），通过离线训练的线性路由器实现 prompt 感知的动态秩选择。
- 路由器以稠密模型输出为监督信号在 C4 语料上训练，解耦了秩选择与校准信息的依赖；配合模式缓存和秩复用策略消除推理时的路由开销。
- 引入专家内存聚合和内核融合等系统级优化，将分散的秩专家读取合并为连续内存访问，并将多个 SVD 矩阵乘法融合为批量内核。

## Key Contributions

- Identifies two fundamental limitations of static rank truncation in SVD-compressed LLMs: (1) the optimal rank varies across individual prompts, and (2) rank selection is sensitive to the calibration dataset choice.
- Proposes PARSE, a post-training framework that reformulates SVD weight matrices as mixtures of independent rank experts, with a linear router performing prompt-aware rank selection.
- Trains the router against dense-model outputs on a large-scale diverse corpus (C4) to decouple rank selection from calibration-specific activation statistics.
- Observes that rank-selection patterns are shared across semantically similar prompts (Spearman rho=0.78) and remain stable across decoding steps (>86% overlap over 60 steps), enabling rank pattern caching and reuse to eliminate online routing overhead.
- Introduces system-level optimizations including expert memory aggregation (contiguous memory layout for selected experts) and kernel fusion (batched MatMul for shared-input projections in attention and MLP modules).

## Methodology

PARSE operates as a plug-in on top of any existing SVD-based LLM compression pipeline and consists of four main components:

1. **Rank Expert Formulation (Section 4.1):** Each SVD-compressed weight matrix W is expressed as a sum of rank-1 components. Each component is treated as an independent "rank expert" E_i. Rather than statically retaining a fixed prefix of top-K singular components, a linear router f_theta maps the input activation x to a score vector over all rank experts and selects the top-K indices as the prompt-specific rank subset R(x). The per-layer budget K is determined offline via Lagrangian multiplier-based optimization to preserve the same global compression ratio as the baseline.

2. **Calibration-Decoupled Router Training (Section 4.2):** The router is trained offline with a reconstruction loss L_rec = ||W_hat * x - W * x||_F^2, where W*x is the dense model output. This supervision against dense-model outputs on a diverse corpus (C4, 5K prompts, 5 epochs) decouples the routing decision from calibration-specific whitening matrices. A straight-through Top-K estimator handles the non-differentiable discrete selection: hard binary masks in the forward pass, differentiable sigmoid-based soft masks (with temperature tau and K-normalization) in the backward pass. Only the router parameters are trained; SVD weights remain frozen.

3. **Rank Retrieval and Reuse (Section 4.3):** Since semantically similar prompts produce highly overlapping rank subsets, PARSE constructs an offline rank pattern cache mapping representative prompt embeddings to their rank subsets. At inference, the incoming prompt embedding is matched to the nearest neighbor in the cache, retrieving the rank subset directly without invoking the router. During decoding, the rank subset selected at prefilling is reused for all subsequent steps (rank overlap > 0.86 across 60 steps), eliminating per-step routing entirely.

4. **Memory Aggregation and Kernel Fusion (Section 4.4):** Expert memory aggregation reorganizes rank components into contiguous memory blocks offline, collapsing K scattered reads into two coalesced memory accesses. Kernel fusion stacks B^T-side projections that share input activations (Q/K/V in attention, up/gate in MLP) into single fused MatMul kernels, and batches the corresponding A-side projections into single batched MatMul operations.

## Results & Highlights

- **Quality improvements on LLaMA-7B (Table 1):** At compression ratio 0.2, PARSE improves average zero-shot accuracy by 2-4 percentage points across SVD-LLM (0.44 -> 0.48), Dobi-SVD (0.46 -> 0.49), Basis Sharing (0.48 -> 0.51), and SAES-SVD (0.49 -> 0.51). At the aggressive compression ratio 0.6, PARSE improves SAES-SVD average accuracy from 0.34 to 0.44 (+10 pp), with C4 PPL dropping from 93.97 to 81.46.
- **Perplexity reductions:** At compression ratio 0.4, PARSE reduces WikiText-2 PPL from 12.39 to 10.98 (Basis Sharing) and from 10.42 to 9.88 (SAES-SVD). At ratio 0.6, PARSE halves or more the PPL of baselines (e.g., SVD-LLM WikiText-2: 53.74 -> 27.18).
- **Generalization to larger/newer models (Table 2):** On LLaMA-13B, PARSE improves SAES-SVD average accuracy from 0.51 to 0.55 and reduces WikiText-2 PPL from 6.34 to 6.05. On LLaMA-30B and Qwen2.5-7B, consistent improvements are observed.
- **Latency (Figure 7):** Up to 2.5x prefill speedup and 2.4x decode speedup over native SVD execution on LLaMA-7B. PARSE also outperforms Dense (vLLM) by up to 2.0x in prefilling at the strongest compression setting.
- **Ablation insights (Table 3):** The full PARSE system (retrieval + reuse) reduces prefill latency from 516ms (router-only) to 121ms and decode latency from 68ms to 19ms, with only a minor accuracy drop (0.52 -> 0.48 on SVD-LLM) while still substantially outperforming the original compressed baseline.
- **Router simplicity (Table 5):** A single linear layer router (4.66B params) outperforms 2-layer MLP (8.42B) and attention-block (30.72B) routers, confirming that prompt-aware rank selection does not require high-capacity routing.
- **Calibration robustness (Table 7):** PARSE maintains stable performance across all SVD calibration dataset choices (WikiText-2/PTB/C4/MathQA), with WikiText-2 PPL varying only 6.98-7.05.

## Critical Assessment

**Novelty:** The core idea of treating SVD components as MoE-style experts with a learned router is conceptually clean and well-motivated. The observation that rank importance varies per prompt is empirically solid (Figure 1, Figure 2). The calibration-decoupled training against dense outputs is a sensible design choice. However, the overall approach is a relatively straightforward combination of existing ideas (SVD truncation + MoE routing + caching), and the linear router is extremely simple. The novelty lies more in the careful system design and empirical validation than in any single algorithmic breakthrough.

**Rigor:** The experimental evaluation is thorough -- four SVD baselines, three compression ratios, multiple model sizes (7B/13B/30B), and a newer model (Qwen2.5-7B). Ablations cover router architecture, training data, calibration data, and inference components. The per-prompt PPL analysis (Figure 9) provides intuitive evidence that PARSE smooths out localized degradation. One concern: the "up to 10%" accuracy improvement is cherry-picked from the most aggressive compression setting (ratio 0.6) where the baseline is already severely degraded (avg acc 0.31-0.34). At the more practical compression ratio 0.2, improvements are 2-4 pp.

**Reproducibility:** The paper provides sufficient implementation details (learning rate, batch size, training epochs, hardware). However, no code is released, and the system-level optimizations (memory aggregation, kernel fusion) would require non-trivial engineering effort to replicate. The cache construction procedure is not fully detailed (how many entries, how to build representative embeddings).

**Significance:** This is a solid systems/ML paper that identifies a real limitation of static SVD truncation and provides a practical solution. The plug-in nature (orthogonal to existing methods) is a strength. The practical impact depends on whether SVD-based compression remains competitive vs. quantization/pruning for real deployment.

**Red flags:** (1) Router training requires access to dense model outputs, adding to deployment cost. (2) The paper evaluates only on decoder-only LLMs up to 30B; scalability to 70B+ models or instruction-tuned models is untested. (3) The "rank pattern cache" introduces additional memory overhead that is not quantified. (4) The paper does not compare against non-SVD compression methods (quantization, pruning) on a pareto basis.

## Strengths

- Well-motivated problem formulation with clear empirical observations about static rank truncation limitations.
- Clean and modular design -- PARSE is orthogonal to existing SVD pipelines and consistently improves all tested baselines.
- Comprehensive ablation studies covering router architecture, training data, calibration data, and each inference optimization component.
- Strong system-level engineering with memory aggregation and kernel fusion providing practical speedups beyond theoretical gains.
- The observation that rank patterns are semantically determined and stable across decoding steps is a useful finding for the community.

## Limitations

- Only evaluated on decoder-only LLMs (LLaMA family, Qwen2.5); behavior on encoder-decoder models, multimodal models, or instruction-tuned systems is unknown.
- The rank pattern cache introduces offline preprocessing cost and additional memory that is not quantified or analyzed.
- Router training requires forward passes through the dense model, which may be impractical for very large models.
- No comparison against non-SVD compression methods (e.g., GPTQ, AWQ, SparseGPT) on an efficiency-accuracy Pareto frontier.
- Absolute speedup numbers are measured on NVIDIA A6000 GPUs only; hardware portability is acknowledged as a limitation.
- The router has substantial parameter count (4.66B for a single linear layer) which adds memory overhead.

## Relevance

本文对 LLM 压缩和高效推理有直接参考价值。将 SVD 秩选择建模为 MoE 路由问题的思路，以及利用语义相似性缓存推理结果的策略，可借鉴到其他动态模型压缩或自适应推理场景中。

## Questions for Further Exploration

- How does PARSE interact with quantization methods -- can prompt-aware rank selection be combined with low-bit quantization for compound compression gains?
- What is the memory overhead of the rank pattern cache, and how does cache size affect the accuracy-latency trade-off?
- Can the router be trained without access to dense model outputs (e.g., using reconstruction loss alone) to reduce deployment cost?
- How does PARSE perform on instruction-tuned or RLHF-aligned models where output distributions may differ significantly from pretraining corpora?
- Does the approach generalize to other low-rank decomposition methods (e.g., NMF, Tucker decomposition) beyond SVD?
