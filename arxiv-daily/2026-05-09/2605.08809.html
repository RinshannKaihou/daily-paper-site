SimReg: Achieving Higher Performance in Pretraining via Embedding Similarity Regularization

A cosine-similarity contrastive loss that fixes a failure mode in standard cross-entropy pretraining — pulling same-class token embeddings together, pushing different-class ones apart — with rigorous margin guarantees and a chunk-wise variant for sequence-parallel training.

Sun, Wang, Zeng et al. Baidu · NTU LLaMA 350M–7B + Mixtral-8×1B 52B tokens
Key Finding

Cross-entropy alone stops improving embedding separability after a threshold (average inter-token cosine similarity stalls at ~0.01). SimReg’s contrastive log-ratio loss drives similarity down to ~0.00001 (angle ~61 to ~74 degrees), yielding consistent zero-shot gains across all tested scales.

Key Metric

30–37%training convergence acceleration + 0.96–1.28% avg zero-shot lift across 9 benchmarks, with <2% runtime overhead

Why This Matters

A single plug-in loss that cuts pretraining compute by nearly a third with negligible overhead. The scaling rule (lambda = 10√(d/1024), tau = 0.01) makes deployment straightforward across model sizes.

Critical Edge

Validated only up to 7B dense / 8B MoE on 13B–52B tokens. MMLU gains are negligible or slightly negative on some sizes. No code release. Unknown whether gains persist at 70B+ or 1T+ tokens.

Method Design

Core Loss

For each token, positive set P = same-class tokens, negative set N = different-class. SimReg loss = log(Σ exp(simN)) − log(Σ exp(simP)), wrapped with softplus for non-negativity. Self-similarity added to P to guarantee a positive pair.

Cosine > Inner Product

Inner-product similarity is dominated by embedding norms, making the loss insensitive to angular differences. Cosine similarity with temperature tau = 0.01 provides more stable, meaningful supervision.

Chunk-wise SimReg

Divides sequences into b chunks of n/b tokens for sequence-parallel training. Reduces complexity from O(n²) to O(n²/b). On 7B models, chunk-wise even outperforms full-sequence SimReg.

Theoretical Guarantee

Minimizing Lsr decreases distance to positive-class center (d/dt ||e − ēP||² ≤ 0). Classification margin improves by at least δ||ε+|| + √2 LPγ, yielding exponential cross-entropy reduction.

Hyperparameter Rules

temperature tau = 0.01 is optimal across all model scales (350M to 8B MoE)
lambda Stable range [2, 20]. Scales as 10 × √(d / 1024) with embedding dimension d. Every doubling of d requires ~√2 increase.
placement Last layer only. Intermediate layers encode blended context where similarity regularization provides limited signal.
overhead <2% runtime, <1% GPU memory at 7B. Chunk-wise variant adds negligible cost.

Strengths

  • Identifies a clear, empirically validated failure mode: cross-entropy stops improving embedding separability after a threshold
  • Simple, principled formulation — a plug-in loss with no architectural changes
  • Comprehensive ablations: hyperparameters, loss placement, chunk size, model scale
  • Chunk-wise variant is a practical contribution for distributed training
  • Interpretable theoretical margin guarantees

Limitations

  • Only validated up to 7B dense / 8B MoE; 70B+ effectiveness unknown
  • Training budgets (13B–52B tokens) are small by modern standards; gains may diminish at larger data scales
  • MMLU gains negligible or slightly negative on some model sizes
  • Baselines (SimCSE, CPretrain, CReg, SCE) adapted from non-pretraining contexts — may not be strong comparisons
  • No code release mentioned
  • Theory assumes Lipschitz-smooth fP, a simplification of actual multi-layer transformer dynamics
