Different Prompts, Different Ranks: Prompt-aware Dynamic Rank Selection for SVD-based LLM Compression

Hengyi Zhu, Zhendong Mi, Grace Li Zhang, Shaoyi Huang Stevens Institute of Technology · TU Darmstadt 2026-05-09
Key Finding
Static rank truncation in SVD-compressed LLMs is fundamentally suboptimal -- optimal rank varies per prompt. PARSE fixes this by treating each SVD component as an independent "rank expert" and learning a lightweight linear router to select prompt-specific subsets, supervised against dense-model outputs.
Key Metric
+10 pp
zero-shot accuracy at ratio 0.6 (SAES-SVD: 0.34 → 0.44)
Up to 2.5× prefill and 2.4× decode speedup over native SVD. At ratio 0.2, consistent 2–4 pp gains across all four tested SVD methods.
Why This Matters
PARSE is a plug-in framework orthogonal to any existing SVD pipeline -- it improves all four tested baselines without modifying the underlying compression method. The MoE-style reformulation and pattern caching strategy generalize to other dynamic compression or adaptive inference scenarios.
Critical Edge
The headline +10pp comes from the most aggressive compression (ratio 0.6) where baselines are already severely degraded. At practical ratios (0.2), gains are 2–4pp. Router training requires dense model forward passes. No Pareto comparison against quantization or pruning. Only tested on decoder-only LLMs up to 30B.
01

Rank Expert Formulation

Each SVD weight matrix is split into rank-1 components treated as independent "experts." A linear router selects top-K per prompt.

02

Calibration-Decoupled Training

Router trained against dense-model outputs on C4 (5K prompts). Straight-through Top-K estimator handles discrete selection.

03

Rank Retrieval & Reuse

Offline cache maps prompt embeddings to rank subsets. Nearest-neighbor lookup at inference. Subsets reused across decode steps (>86% overlap).

04

Memory Aggregation & Fusion

Contiguous memory layout for selected experts. Batched MatMul for shared-input projections in attention and MLP modules.

§ Core Design Choices

  • Linear router (single layer, 4.66B params) outperforms 2-layer MLP and attention-based routers
  • Per-layer budget K determined offline via Lagrangian multiplier optimization to match global compression ratio
  • SVD weights remain frozen; only router parameters are trained
  • Semantically similar prompts produce overlapping rank subsets (Spearman ρ=0.78)
  • Rank patterns stable across decoding steps (>86% overlap over 60 steps)

§ Integration Scope

  • Tested as plug-in for four SVD methods: SVD-LLM, Dobi-SVD, Basis Sharing, SAES-SVD
  • Models: LLaMA-7B/13B/30B, Qwen2.5-7B
  • Compression ratios: 0.2, 0.4, 0.6
  • Hardware: NVIDIA A6000 GPU
  • No code repository released; system optimizations require non-trivial engineering to replicate

§ Perplexity (LLaMA-7B)

  • Ratio 0.4: WikiText-2 PPL 12.39 → 10.98 (Basis Sharing)
  • Ratio 0.4: WikiText-2 PPL 10.42 → 9.88 (SAES-SVD)
  • Ratio 0.6: SVD-LLM WikiText-2 PPL 53.74 → 27.18 (halved)
  • Ratio 0.6: SAES-SVD C4 PPL 93.97 → 81.46

§ Inference Latency (LLaMA-7B)

  • Up to 2.5× prefill speedup over native SVD
  • Up to 2.4× decode speedup over native SVD
  • Outperforms Dense (vLLM) by up to 2.0× in prefilling at strongest compression
  • Full system (cache+reuse): prefill 516ms → 121ms, decode 68ms → 19ms

Strengths

  • Well-motivated problem with clear empirical evidence that static rank truncation is suboptimal
  • Clean modular design -- orthogonal to existing SVD pipelines, improves all baselines consistently
  • Comprehensive ablations: router architecture, training data, calibration data, inference components
  • Strong systems engineering with memory aggregation and kernel fusion for real speedups
  • Useful finding that rank patterns are semantically determined and stable across decoding steps
  • Calibration robust: stable PPL (6.98–7.05) across all calibration dataset choices

Limitations

  • Only tested on decoder-only LLMs up to 30B; encoder-decoder and 70B+ models untested
  • Router training requires forward passes through the dense model -- impractical for very large models
  • No Pareto comparison against non-SVD methods (GPTQ, AWQ, SparseGPT)
  • Rank pattern cache memory overhead not quantified
  • Measured on NVIDIA A6000 only; hardware portability unverified
  • Router itself has 4.66B parameters -- non-trivial memory overhead
  • No code released; system optimizations require significant engineering to replicate

§ Open Questions

  • Can PARSE combine with low-bit quantization for compound compression?
  • How does cache size affect the accuracy-latency trade-off?
  • Can the router be trained without dense model outputs?
  • Behavior on instruction-tuned or RLHF-aligned models?
  • Does the approach generalize to NMF, Tucker, or other low-rank decompositions?

§ Verdict

  • Conceptually clean combination of existing ideas (SVD + MoE routing + caching)
  • Novelty lies in careful system design and thorough empirical validation
  • Practical impact depends on whether SVD compression remains competitive vs. quantization/pruning
  • Solid systems/ML contribution with a real limitation identified and addressed
