A Controlled Study of Memory Hierarchy Transitions in Quantum Circuit Simulation on Apple M4 Pro Unified Memory Architecture

On Apple M4 Pro, CPU and GPU share identical LPDDR5X DRAM at ~224 GB/s. With that confound eliminated, this study runs 11 simulation backends through Roofline analysis and discovers a 4.46x timing cliff at 28→29 qubits — and a 10x GPU speedup that STREAM benchmarks predict as only 1.85x.

Core Findings
Key Finding

A reproducible 4.46x “DRAM bandwidth cliff” strikes tensordot backends at the 28→29 qubit transition (state vector 2.15 GB → 4.29 GB), while direct-index backends sail through with clean ~2x scaling. The cliff is circuit-independent (validated on both GHZ and QFT) and determined entirely by access pattern, not gate count.

Mechanism
Tensordot's reshape-transpose generates large contiguous blocks that benefit from hardware prefetching at ≤28q; this collapses at 29q. Direct-index's mixed-stride access (strides from 2&sup0; to 2&supn;−¹) causes cache misses regardless of working-set size, producing uniform DRAM-bound scaling with no cliff.
Key Metric
4.46×

Timing discontinuity at the 28→29 qubit boundary for tensordot backends (Backend C: GHZ 4.46x, QFT 4.33x). Direct-index backends (J, K) maintain 2.06–2.09x step ratios throughout.

Why This Matters

Peak streaming bandwidth cannot predict simulation speedup for non-contiguous memory access patterns. STREAM predicts 1.85x GPU speedup; direct-index achieves 10x. Every quantum simulator developer’s mental model of “memory-bound ≈ bandwidth-limited” is incomplete — access-pattern structure is the missing variable.

Critical Edge

The headline 10x GPU speedup is measured against an MLX CPU baseline at only 119.9 GB/s (44% of JAX CPU’s 224.7 GB/s). Against JAX CPU the gap would shrink substantially. The exact microarchitectural mechanism behind the cliff — DRAM row-buffer locality, prefetcher depth limits, or framework tiling thresholds — remains unresolved.

The DRAM Cliff at 28 → 29 Qubits

Tensordot Backends (C, F, G)

4.46x discontinuity at 28→29q. State vector jumps from 2.15 GB to 4.29 GB, breaking hardware-prefetcher-friendly contiguous access. Affects JAX CPU, MLX GPU, and MLX CPU backends uniformly.

Direct-Index Backends (J, K)

~2.06–2.09x clean scaling throughout. Mixed-stride access is already cache-unfriendly at all sizes, so the DRAM boundary crossing produces no additional penalty. No cliff.

STREAM predicts 1.85x (MLX GPU 221.9 GB/s ÷ MLX CPU 119.9 GB/s). The 5.4x gap reflects the GPU's massively parallel Metal shader dispatch — thousands of concurrent scatter-write operations vs. CPU sequential dispatch — a capability invisible to sequential STREAM measurement. Flat-index backends show 3.5–5.9x actual; tensordot backends 3.1–4.1x actual.

Experimental Design

11 Backends Across Three Algorithm Families

Family Backends Mechanism Cliff?
Tensordot C (JAX CPU), F/G (MLX GPU/CPU) Reshape state to [2,2,...,2], apply gates as tensor contractions Yes — 4.46x
Flat-index H/I (MLX GPU/CPU) Flat [2^n] array with explicit index gathering Partial — position mismatch
Direct-index D (NumPy), E (NVMe), J (MLX GPU Metal), K (MLX CPU) Bitwise XOR index manipulation with in-place scatter-write No — clean 2x scaling
Brute-force A (Kronecker) Full Kronecker product expansion Terminated at 16q

Thermal protocol: 90s idle between backends; caffeinate active; MLX backends in fresh subprocesses. Without this, sequential benchmarking inflated JAX CPU timing by 2.3–2.8x at 28–29q, producing an artifactual 5.28x cliff (corrected to 4.46x).

Assessment

Strengths

  • UMA eliminates four simultaneous architectural confounds (memory bandwidth, technology, controller, compute parallelism)
  • Thermally isolated multi-trial methodology with honest correction of initial misreported results (5.28x → 4.46x)
  • Cross-validated across O(n) GHZ and O(n²) QFT circuits
  • Clean decomposition: Roofline position, DRAM cliff, and STREAM-vs-simulation gap each with independent evidence
  • Public code repository with reproduction scripts at github.com/gyanpratipat/qsim-uma

Limitations

  • MLX CPU baseline at only 44% of JAX CPU bandwidth inflates GPU speedup headline numbers
  • Single hardware platform (one M4 Pro MacBook); quantitative generalizability unknown
  • Maximum 30 qubits (8.59 GB state vector) — modest by current standards
  • Exact microarchitectural cliff mechanism (row-buffer vs. prefetcher vs. tiling) unresolved
  • No comparison with production simulators (Qiskit Aer, Qulacs, cuQuantum)
  • complex64 only; complex128 shifts the cliff by one qubit but is not tested
