UserGPT: Generative User Profiling via LLM with Behavior Simulation and Curriculum-Driven RL

Yunyi Xuan, Hao Yi, Fengling Mao et al. · 12 authors · 2026-05-09

An 8B-parameter model (Qwen3-8B) trained with synthetic behavior simulation, data semantization, three-stage curriculum SFT, and dual-filter GRPO reinforcement learning — matching or exceeding models 10–80× larger on user profiling tasks.

Key Finding
An 8B model matches Qwen3.6-Plus on tag prediction and surpasses Qwen3-235B on summary generation — closing the gap with models up to 80× its size through systematic data engineering and curriculum-driven RL.
Key Metric
0.7325 Avg@10
Tag prediction nearly matches Qwen3.6-Plus (0.7329). Summary Acc­_Ex = 0.7528, beating Qwen3-235B by +7.31%. Compression: ~15K → ~1.2K tokens (97.9%).
Why This Matters
Proves that careful data pipelines and RL post-training can close the small-vs-massive model gap on domain-specific tasks — making production deployment feasible at a fraction of the cost.
Critical Edge
The entire pipeline — training data, benchmark, and evaluation — is built on synthetic/simulated data with zero real-world validation, creating a closed loop whose fidelity is unverified.

Training Pipeline

01

Behavior Simulation

Persona-Need-Intent cognition model with MDP transitions generates multi-year cross-platform trajectories (e-commerce, delivery, OTA, POI) with demographic calibration and noise injection.

02

Data Semantization

Micro-level entity refinement (Qwen3-1.7B denoiser) + macro-level behavioral corpus construction (hierarchical spatio-temporal aggregation). Achieves 75%+ data reduction into standardized MUB format.

03

3-Stage Curriculum SFT

Stage 1: 130K standard questions. Stage 2: 170K controversial questions with Qwen3-235B resynthesis. Stage 3: 73K hard questions with 36K context windows and composite summaries.

04

DF-GRPO (RL)

Dual-Filter GRPO on 1,829 stratified samples: sample-level filtering (discard malformed outputs) + group-level filtering (exclude outlier reward groups). 3-component reward: format, atomic accuracy, LLM-as-Judge summary quality.

Core Components

Simulation Engine

Integrates AlignX 90-dimensional preferences + SocioVerse 15-dimensional annotations. Persona evolution via event-driven (life events) and behavior-driven (monthly fine-tuning) mechanisms. Quality assured through logical consistency checks, noise injection (misclicks, shared accounts), and manual validation.

Entity Refinement

Fine-tuned Qwen3-1.7B Semantic Refiner addresses marketing clutter and cold-start sparsity. Dual strategy: entity rewriting (denoising + compression) for noisy sources; entity enrichment (normalization + augmentation) for sparse sources. Reduces token length by 50%+.

HPR-Bench

5-stage construction pipeline: sampling, pre-annotation, difficulty stratification, automated QC, human verification (5+ experts with majority voting). Covers 18 atomic tags across 5 dimensions and profile summarization evaluation.

Incremental Profiling

Proposed production paradigm treating summaries as pluggable memory for AI agents. Event-triggered and time-triggered update mechanisms for efficient incremental updates without full recomputation.

Assessment

Strengths

  • Comprehensive ablations isolating every component's contribution
  • 8B model matching/exceeding models 10–80× larger on domain tasks
  • Practical data engineering: entity refinement, hierarchical aggregation, salience compression
  • Out-of-domain evaluation confirms no catastrophic forgetting
  • Incremental Profiling concept is forward-looking for AI agent architectures
  • Honest about limitations of synthetic data quality

Limitations

  • Entire pipeline built on synthetic data — no real-world validation
  • No downstream task evaluation (recommendation accuracy, conversion)
  • Simulation engine fidelity to real behavior assumed, not empirically verified
  • Privacy implications insufficiently addressed for detailed persona generation
  • Specific to Chinese e-commerce; cross-cultural generalizability unclear
  • Incremental Profiling proposed without experimental validation
  • No analysis of failure cases or error patterns

Key Ablations

Controversial data quality vs. random mixture
High-quality controversial data (inter-model disagreement with at least 1 heuristic agreement) significantly outperforms random data mixtures: Pass@1: 0.7316 vs. substantially lower baselines. This validates the curriculum's focus on training-edge cases.
Curriculum-driven SFT vs. naive mixed training
The three-stage curriculum (standard → controversial → hard) yields +4.6% Acc_Ex on summary generation over training on all data simultaneously. Stage 1+2 build atomic facts ("what"); Stage 3 teaches coherent narrative ("how").
DF-GRPO vs. standard GRPO
Dual-filter GRPO (sample-level + group-level filtering) improves +6.8% Acc_Ex over vanilla GRPO. Both filters contribute: sample-level discards truncated/malformed outputs; group-level excludes outlier reward groups (too low or too high).
RL boost over SFT-only
DF-GRPO reinforcement learning provides +9.12% Acc_Ex improvement over the SFT-only model on summary generation, demonstrating significant value from the RL post-training stage.
