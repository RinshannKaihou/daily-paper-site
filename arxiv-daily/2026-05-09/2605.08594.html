FLARE: One-Shot PE-Level Fault Localization in Systolic Arrays via Algebraic Test Vectors

Logashree Venkatasubramanian, Zishen Wan, Viveck Cadambe Georgia Tech ECE 2026-05-09

Pairwise coprime test-vector entries create unique divisibility signatures at the systolic array output. A single test pass localizes the faulty processing element to its row — with zero hardware modification and negligible runtime cost.

Key Finding

In a weight-stationary systolic array, a permanent fault in PE (i*, j*) produces an output deviation Δ = e·xi*. Assigning pairwise coprime integers as test-vector entries makes each row's contribution uniquely identifiable through a divisibility test: only the true faulty row's entry divides Δ without also dividing the error magnitude e. This turns fault localization into an algebraic elimination problem solvable in one pass.

Why This Matters

TPU-scale AI accelerators deployed in safety-critical settings (autonomous driving, aerospace) need fast, unambiguous fault diagnosis. FLARE achieves PE-level localization that previously required hardware augmentation (1.74 % area overhead) — using only a single carefully chosen test vector, with no silicon changes whatsoever.

Critical Edge

Assumes at most one faulty PE per column. Multiple simultaneous faults in the same column break the divisibility argument entirely. The method also covers only weight-register permanent faults; input-register and MAC faults require different techniques. INT8 single-round reliability is marginal at pool boundary (Pfail ≈ 0.53 for L=31).

Mechanism

  • Assign the L largest primes ≤ M as pairwise coprime test-vector entries x1, …, xL
  • Stream the vector through the weight-stationary array; compute expected output offline
  • For each column with nonzero deviation Δ, test divisibility xk | Δ for every candidate row k
  • A row survives only if xk | Δ and |Δ/xk| ≤ M (magnitude check)
  • Coprime property ensures false candidates are eliminated unless xk happens to divide the error e itself

Probabilistic Guarantee

  • Pfail ≤ (L−1)/M using the L largest primes each exceeding M/2 (Theorem 3)
  • INT16 (M=32767, π=3512): Pfail < 0.008 for L=256
  • Single-bit errors: exact one-round localization using odd coprimes (no odd integer divides a power of two)
  • Two-round ratio-based fallback provides guaranteed exact localization for all practical sizes
  • Prime-power construction (2→64, 3→81, …) reduces ambiguous candidates for INT8

Validation & Overhead

  • 500 fault-injection trials per configuration using Qwen2.5-0.5B weights in a cycle-accurate simulator
  • INT16 failure rate negligible across all tested dimensions up to L=256
  • INT8 empirical rate tracks Theorem 3 bound closely; graceful degradation beyond coprime pool boundary
  • Cycle overhead measured via SCALE-Sim 3.0.0: ≤2 additional cycles per test tile
  • At 128×128 and 256×256, test cost is below 0.1 % of a single inference GEMM tile
  • Also estimates fault error magnitude e, enabling selective mitigation

Fundamental Constraint

The coprime pool is bounded by the prime-counting function: L ≤ π(M). For INT16 (M=32767), π=3512 comfortably covers 256×256 arrays. For INT8 (M=127), π=31 means arrays beyond 31 rows need the two-round scheme. This is not an implementation choice but a hard number-theoretic limit.
