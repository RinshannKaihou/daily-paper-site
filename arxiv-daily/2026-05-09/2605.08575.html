Uncovering Intra-expert Activation Sparsity for Efficient Mixture-of-Expert Model Execution

Jongseok Park, Sunga Kim, Zhenyu Gu, Ion Stoica, Alvin Cheung — UC Berkeley & AMD

Key Finding

Pretrained MoE models across 3 orders of magnitude (1B–400B params) contain up to 90.8% intra-expert activation sparsity—exploitable with zero model modification and 95% accuracy retention.

Key Metric
2.5×

MoE layer speedup (RTX 4090, 85–87% sparsity). 1.2× end-to-end.

Why This Matters

Sparsity increases with model scale. As frontier models grow, the free speedup grows too—no retraining, no architecture change, deployable today in vLLM.

Critical Edge

Amdahl's law caps end-to-end gains at 1.2× because MoE layers account for only ~45% of runtime. Gate projection must stay dense, hard-capping theoretical max at 3×. Evaluated only on multiple-choice benchmarks.

Method — Two-Phase Approach
Phase 01

Sparsity Analysis

  • Profile 8 pretrained MoE models (1B–400B params)
  • Sort neurons by post-SwiGLU magnitude; zero lowest at varying thresholds
  • Evaluate on 5 benchmarks via lm-eval-harness
  • Define sparsity cutoff: max sparsity at 95% baseline score
  • Analyze shared vs. routed expert neuron importance
Phase 02

System Integration (vLLM)

  • Custom Triton kernel: O(n) threshold-based sparse masking
  • Fused sparse up-down projection with gather-based execution
  • 64-neuron tile granularity for CUDA/HIP graph compatibility
  • Dynamic sparse/dense switching profiled at engine startup
  • Preserves expert parallelism, kernel fusion, graph capture

Assessment

Strengths

  • Zero modification to pretrained models—deployable on existing MoE serving infrastructure today.
  • Evaluated across 8 models spanning 3 orders of magnitude, including Llama-4-Maverick (400B).
  • Production-grade vLLM integration with full source code in appendix, not just a research prototype.
  • Sparsity scales with model size: counteracts the narrative that efficiency gains diminish at frontier scale.
  • Clean shared vs. routed expert analysis provides actionable guidance for future MoE architecture design.

Limitations

  • End-to-end 1.2× is modest; MoE layers are only ~45% of runtime (Amdahl's law).
  • Gate projection must remain dense, capping theoretical max at 3×.
  • Benchmarks are simple multiple-choice tasks; no code, math, or long-context evaluation.
  • Gather-based sparse path has lower peak throughput than dense BMM at large batch sizes.
  • End-to-end results reported only for Qwen3.5-35B-A3B, not the larger (sparser) models.
  • Static threshold set at startup; per-layer adaptive thresholds not explored.
