Robust Multi-Agent LLMs under Byzantine Faults

Self-Anchored Consensus (SAC) replaces self-reported confidence with receiver-side evaluation, eliminating an entire attack surface in multi-agent LLM systems.

Key Finding

Receiver-side scoring neutralizes Byzantine agents

Each agent independently scores its neighbors' responses instead of trusting self-reported confidence. Filtering the lowest-scoring F neighbors and refining via LLM re-prompting yields 76.7–96.7% honest-majority accuracy — while the self-report baseline CP-WBFT collapses to 16.7–46.7% under a single adversarial agent.

A single adversarial agent reporting falsified confidence of 1.0 is sufficient to catastrophically degrade CP-WBFT (BFTI of −10.4% to −54.7%).

Why This Matters

One malicious agent can corrupt the entire system

As LLM agents are increasingly deployed in collaborative settings — code review, medical triage, financial analysis — the demonstrated vulnerability of self-reported confidence protocols is an urgent finding. SAC provides a principled, decentralized alternative grounded in classical Byzantine consensus theory.

  • Existing methods (CP-WBFT) trust agents to report their own reliability — trivially exploitable.
  • Leader-based methods suffer under consecutive Byzantine leaders, requiring expensive re-election.
  • SAC requires no trusted leader, no central coordinator, and no self-reported signals.
SAC Protocol — Algorithm 1

Four-step decentralized filter-and-refine loop

1
Broadcast
Each agent sends its current response to all graph neighbors.
2
Score
Agent i independently scores neighbor j's response via its own LLM. No self-reported signals.
3
Filter
Discard the F lowest-scoring neighbors. Under (F+1)-robustness, at least one reliable neighbor is always retained.
4
Refine
Condition LLM on retained neighbor responses, weighted by score. Prefer high-scored answers but keep own if believed correct.

Critical Edge

The LLM-as-judge paradox

Receiver-side scoring relies on the same LLM that produces responses. If the LLM cannot reliably distinguish correct from incorrect answers — precisely when multi-agent collaboration is most needed — the filtering step may fail. This circular dependency is acknowledged but not deeply analyzed.

  • Experiments limited to n∈{7,9} agents; scalability to 50+ agents unknown.
  • Only one adversarial strategy tested (cached weak-model replay). Collusion and subtle attacks not evaluated.
  • 30-problem subsets per benchmark — statistically thin.
  • No comparison with non-LLM baselines (e.g., simple majority vote).
