Predicting Large Model Test Losses with a Noisy Quadratic System

A mechanistic loss model for LLM pre-training that predicts test loss from model size (N), batch size (B), and weight updates (K) with only 7+1 learnable parameters — the first such model to handle variable batch sizes.

Key Finding
The Noisy Quadratic System (NQS) models LLM pre-training loss as stochastic optimization on a quadratic surface. It decomposes test loss into approximation error, bias, and variance — each governed by power-law spectra on the Hessian eigenvalues and signal power. Evaluation is O(1) via Euler-Maclaurin formulae; fitting takes under 5 minutes on GPU.

Why This Matters
Replaces heuristic scaling laws with a principled, verifiable prediction framework. Enables compute-optimal training planning under compound constraints (time, memory, compute, data) — letting practitioners jointly optimize N/B/K configurations before committing expensive GPU hours. If results hold at larger scales, NQS could become the standard tool for pre-training budget allocation.
Critical Edge
Tested only up to 2B parameters on relatively small datasets (OWT2 vocab 3000, seq length 128). Scalability to 70B+ production models is unverified. Does not model multi-epoch training, data mixtures, or LR schedules beyond the basic LayerNorm effect. The NQS is over-sensitive to learning rate specification in ways that actual LLMs are not — a known limitation the authors acknowledge.
Methodology

NQS is derived from three theoretical lines: linear regression models of scaling (Bordelon et al., 2024), the Noisy Quadratic Model (Zhang et al., 2019), and LayerNorm's effect on effective learning rates (van Laarhoven, 2017).

L(N,B,K) = Eapp(N) + Ebias(N,K) + Evar(N,B,K)
λn ~ n−q  ·  signal ~ n−p  ·  noise ~ n−r · 1/B

The LayerNorm effect is captured by a dynamic learning rate γk ∝ 1/||wk||2, with an additional initialization-norm parameter s. Fitting uses L-BFGS with multiple random initializations; inference is O(1) in N and K.

Strengths

  • Mechanistic model grounded in optimization theory, not descriptive curve fitting
  • First loss prediction model to handle varying batch sizes
  • Rigorous holdout evaluation with up to 1000× compute gaps
  • Computationally lightweight: O(1) evaluation, <5 min fitting
  • Robust to training set perturbation; tight confidence intervals from subsampling
  • Practical multi-constraint optimization for N/B/K configuration

Limitations

  • Maximum model size tested is 2B — far below production scale
  • No modeling of multi-epoch training or data mixtures
  • Over-sensitive to learning rate gamma vs. actual LLM behavior
  • LayerNorm parameter s requires separate fitting (numerical issues)
  • Fixed tokenizer (vocab 3000) and short sequences (128 tokens)
  • Only benchmarked against Chinchilla Method 3

Open Questions

  • How does NQS extrapolate to 70B–1T parameter models?
  • Can the framework extend to multi-epoch training and data mixtures?
  • Minimum number of training runs needed to reliably fit 7+1 parameters?
  • Comparison with other recent approaches (multi-power laws, data-constrained scaling)?
  • Could NQS serve as a sandbox for testing novel optimization strategies?
