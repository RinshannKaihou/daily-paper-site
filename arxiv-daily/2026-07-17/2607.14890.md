> **一句话总结：** 作者提出 Proof-or-Stop —— 一种把自主编码 Agent 的"已测试/已审阅/已完成/可合并"声明视作需被证据裁决的"claim"而非状态本身的控制层，在 9,240 个 cell 的预注册消融中把"可见测试通过但隐藏 oracle 失败"的放大从 31/1800 降到 2/1800（+1.6pp，95% CI [0.8, 2.5]），并以 565 个自构建 story / 1007 条审阅发现（94.8% 已解决）和 68 条跨厂商高/严重等级发现作为操作化证据。

> **TL;DR:** Proof-or-Stop reframes autonomous-agent lifecycle claims (reviewed / tested / done / ready-to-merge) as admissibility decisions: each transition advances only when fresh, tracked-source-state-bound, mechanically verifiable evidence (materialHash/headHash/storyFilesHash-bound receipts, signed review verdicts, full-test receipts) satisfies a gate predicate. On a 9,240-cell pre-registered 5-arm ablation over 24 stratified tasks with B1–B15 fault injections, the gated A4 loop amplifies visible-pass/hidden-fail artifacts on just 2/1800 injected cells versus 31/1800 for the compute-budgeted naive A2′ control (not-amplified +1.6pp, 95% CI [0.8, 2.5], CI excludes 0), with the cleaner A3-vs-A4 contrast (14/1800 vs 2/1800) isolating gate enforcement rather than mere reviewer addition. Engine-contract suites pass 10/10 with zero false-DONE, local-key receipt bundles reject all 18 tamper classes, and a self-hosted corpus of 565 stories / 1007 findings (94.8% resolved) plus a 68-row high/critical cross-vendor Codex host-2 exhibit provide operational evidence — bounded to one model family (Sonnet) and self-built artifacts.

## 核心贡献

- **提出 Agent-as-claim 语义与证据门控方法（C1）**：把生命周期状态推进（review/test/done/merge）形式化为"claim admissibility"决策，Agent 输出只是声明，必须由新鲜且绑定代码状态的可验证证据满足门谓词（公式 2、3）后才能推进状态；缺失/陈旧/伪造证据则修复、降级、升级或停止，而非伪造 done。
- **代码状态绑定的证据力学（C2）**：每条证据携带 materialHash / headHash / storyFilesHash + policy/command-set 哈希 + receipt identity（cmd、args、cwd、exit、outputDigest），并附签名 producer identity；任何源树漂移、命令集篡改、签名缺失都会被拒。离线 receipt-bundle 合约（B-4c65）拒绝全部 18 类篡改，零误受、零误拒。
- **主机中立作为证据属性（C3）**：跨主机裁决仅在 receipt 的 materialHash 与当前代码匹配时才被采纳；git-native handoff 只传 git delta + 小 JSON（pack ≈9.7KB，<0.5% worktree），HN-2 跨机续作得到确定性验证，而"跨厂商强仲裁"被显式降级为本地单机（Eq. 5），不伪造 quorum。
- **操作化评估与声明边界纪律（C4）**：自托管 565 个 story / 1007 findings（94.8% resolved），68 条 Codex host-2 高/严重发现全数 resolved；并内置"prohibited-wording guard"防止把本地证据上升为生产/外部基准/多模型等更强声明。
- **机制层面的诚实梯度**：near-compute A3-vs-A4 对比（A3/A4 token 比 0.98，使用同一 reviewer 信号但 A3 仅 advisory）放大 14/1800 vs A4 放大 2/1800，证明收益来自"把 review 当作 lifecycle gate"而非"多加一个 reviewer"。

## Key Contributions

- **Agent-as-claim semantics + evidence-gating method (C1)**: formalizes lifecycle advancement (plan→dev→review→test→done) as a claim-admissibility decision (Eqs. 2–3). A natural-language "tests passed" report carries no binding and is never an admissible E_c; missing/stale/forged evidence triggers bounded repair, honest degradation, escalation, or safe-stop rather than a fabricated done.
- **Code-state-bound evidence mechanics (C2)**: evidence records carry materialHash/headHash/storyFilesHash, policyHash/commandSetHash, a receipt identity, and signed producer identity; the offline receipt-bundle contract (B-4c65) rejects all 18 tamper classes (code-byte flip, stale hash, signature tamper, signer-key mismatch, malformed structure, non-SHA256 digest, build-proof-missing, etc.) with zero false accepts and zero false rejects.
- **Host-neutrality as an evidence property (C3)**: cross-host verdicts are admissible only when bound to the current materialHash; git-native handoff reconstructs the worktree from a bare clone with B's HEAD/tree-hash equal to A's (HN-2 verified), while the strong cross-vendor quorum honestly degrades to local single-host rather than spoofing (Eq. 5).
- **Operated evaluation and claim-boundary discipline (C4)**: self-hosted 565-story / 1007-finding corpus (94.8% resolved), 68-row cross-vendor Codex host-2 high/critical exhibit (all resolved), 35-baseline claim-boundary registry, and a prohibited-wording guard that mechanically blocks unsupported capability claims.
- **Clean enforcement-isolation contrast**: the A3-vs-A4 comparison spends near-equal tokens (A3/A4 = 0.98) using the same reviewer signal, but A3 treats it as advisory and amplifies 14/1800 injected cells while A4 enforces it as a gate and amplifies 2/1800 — the clearest mechanistic isolation of the contribution.

## Methodology

Proof-or-Stop reduces lifecycle control to a four-step pipeline: **Actor output → Claim → Evidence → Gate → Lifecycle transition**. Three git-content identities bind the tracked source state:

- `materialHash(H) = SHA256(canon(tracked source tree))` excluding lifecycle metadata
- `headHash(H)` = commit identity of H
- `storyFilesHash(H) = SHA256(canon(story-owned files))`

A piece of evidence E is a structured record (not prose) carrying a binding `β(E) = ⟨materialHash, headHash, storyFilesHash, policyHash, commandSetHash⟩`, a receipt identity `ρ(E) = ⟨cmd, args, cwd, exit, outputDigest⟩`, and producer identity (actor / lane / host / signing key). Admissibility is the seven-way predicate of Eq. (2): `Fresh ∧ Complete ∧ IntegrityVerified ∧ ProducerAuthorized ∧ ExecutionAttested ∧ Supports ∧ OutcomeAccepted`. The freshness conjunct defeats the most common silent failure: evidence produced against an earlier source state is rejected the instant the tree changes. Advancement through phases is then `Advance(φi→φi+1, H) ⟺ ∀c∈Ci ∃E_c : Provides(E_c, c) ∧ Admissible(E_c, c, H)` (Eq. 3), with no term for self-report.

The lifecycle is instantiated as a CLI over a "story" moving init→init-check→plan→dev→review→test→done, with concrete gate artifacts (Table 8): structured plan contracts freeze scope; dev→review runs a scope-contract check; review→test requires signed current-round review runs plus per-run test receipts; test→done requires a freshness-bound full-test receipt over the exact tree; high-risk paths additionally require a 3×2 independent-host review floor (Eq. 4) with honest degraded-single-host fallback (Eq. 5) when a second host is unavailable.

Evaluation uses four evidence tiers: (1) Tier-A engine-contract suites (10/10 unattended-loop scenarios with zero false-DONE; 10×15-row large-ledger stress suite; B-4c65 tamper contract; B-7b6e gate-strength self-red-team; proof-or-stop replay campaign B-9e44/B-bae9/B-2a5b/B-f511/B-853a); (2) a powered 9,240-cell 5-arm ablation (A1 prompt-only, A2 naive-retry, A2′ compute-budgeted naive, A3 review-only, A4 Proof-or-Stop) over 24 stratified tasks × 16 scenarios × k=5, with B1–B15 fault injections and a hidden ground-truth oracle distinguishing "visible test green but artifact wrong" (amplified) from not-amplified; (3) a verified recovery pilot (3 tasks × 5 cells × 15 runs/arm); (4) a self-hosted self-application corpus and cross-vendor host-2 exhibit. Statistical readout: Wilson 95% CIs, seeded cluster bootstrap (B=2000), and Benjamini–Hochberg FDR for exploratory per-scenario tests. Outcomes are read objectively from git history, the process-event log, and an independently-authored acceptance script — never from agent narration.

## Results & Highlights

- **Powered ablation (Table 11, primary pre-registered contrast)**: A4 Proof-or-Stop amplifies **2/1800** visible-pass/hidden-fail injected cells versus **31/1800** for the compute-budgeted naive A2′ control, yielding H1 not-amplified **+1.6pp** with a 95% cluster-bootstrap CI of **[0.8, 2.5]** (CI excludes 0). A4 achieves this while spending more compute (mean **204,553** tokens/cell, **81.2s** vs A2′ **170,545** tokens, **54.8s**; ≈1.2× A2′ tokens — a conservative direction).
- **Enforcement isolation (A3 vs A4, Table 11 + Fig. 5)**: A3 review-only amplifies **14/1800** while A4 amplifies **2/1800** at near-equal spend (A3/A4 token ratio **0.98**); excluding the du-duration task, A3 amplifies **9/1725** vs A4 **0/1725** — isolating the value of enforcing the review signal as a gate rather than adding an advisory reviewer.
- **Engine contract (Table 9)**: the unattended develop loop passes **10/10** scenarios with **false-DONE = 0** and safety success rate **1.0**; the large-ledger no-false-done stress suite asserts exactly **10×15 = 150** required rows across 10 constructed groups.
- **Tamper contract (B-4c65, Table 5)**: offline receipt verifier rejects all **18** tamper classes with **false-accept = 0** and **false-reject = 0**; all 10 non-ok reason codes are reached. Gate-strength contract B-7b6e passes **12/12** hermetic checks.
- **Task concentration (Table 13)**: A4's only two amplifications both fall on a single task (du-duration: 2/75); excluding it, A4 = **0/1725** vs A2′ = **2/1725**, confirming the matrix-level effect is driven by the highest-risk visible-pass/hidden-fail trap.
- **Self-application corpus (Table 18, live 2026-06-23 extraction)**: **565** dev stories / **1007** review findings, **94.8%** resolved (955 resolved / 41 open / 11 dismissed); severity shape is high-dominant (509 high, 14 critical, 328 medium, 156 low).
- **Deep-set green-but-wrong (Fig. 8)**: in a curated 12-story / 28-finding deep set, **26 of 28 (93%)** findings were filed while the author's own smoke tests were passing; 6 confirmed production/logic-defect correctness exhibits with fix commits (Table 20).
- **Cross-vendor host-2 exhibit (Fig. 9, 2026-07-02 refresh)**: **68** high/critical Codex findings across **26** stories, **all 68 resolved** (2 critical, 66 high; 48 adversarial, 17 missing-case, 3 edge), filed after same-vendor Claude lanes passed — including a critical signature bypass (B-4c65: canonicalizer dropped JSON null), a critical DONE-gate guard returning pass before the parallel sub-check ran (B-086b), and several defects in the paper's own §5.2 statistics (denominator bias, GLMM contamination, hidden-regression baseline failing open).
- **Clean-task overhead pilot (Table 14, n=9/arm, no injection)**: all four arms complete 9/9 correctly, so on easy work the loop is pure overhead — A4/A1 cost ratio **2.07×** and wall ratio **2.51×** — the loop's value appears only under green-but-wrong faults where bare loop amplifies **15/15** and gated loop ships **0/15**.
- **Recovery pilot (Table 15, n=15 wrong-injection runs/arm)**: A2′ amplifies **15/15**, A4-C one-gate safe-stops **15/15**, A4b-B bounded-reflection recovers **15/15** with `groundTruthPass=true`; cost gradient A2′ $0.094/18.3s → A4-C $0.173/34.8s → A4b-B $0.386/89.0s.
- **Supplemental paired execution-status readout (Table 12, 1,152 cells)**: paired with a no-review control, Proof-or-Stop admitted 1,042/1,152 and safe-stopped 110; **106** no-review completions were not admitted; bundled gated vs no-review input+output token ratio **3.80×** (labeled gpt-5.5).
- **Git-native handoff (HN-2)**: handoff pack **≈9.7KB** (<0.5% of a worktree that contains a 2MB gitignored bulk artifact); B's HEAD and tree content-hash verified equal to A's; one fresh commit advanced on resume.

## Authors & Credibility

- Authors are Jek Huang (corresponding, jek.huang@prodenovo.ai), Jeffery Hsia, Jiayi Sun, Freddie Shi, Wei Huang, and Ian H. White. The affiliation string ("prodenovo.ai") and references to "Authority Server" / "Authority sibling" codebases indicate an applied research / product-engineering group rather than a traditional academic lab; the paper is a June 2026 preprint (v1, arXiv:2607.14890, cs.AI).
- The paper is unusually disciplined about claim boundaries: it carries a pre-registration table (Table 1) grading every claim by evidence tier (engine / empirical / corpus / observational / boundary / future-domain smoke) and explicitly enumerates what is *not* established (cross-domain, multi-model, external-benchmark, and strong independent-host generalization).
- It dogfoods its own method (self-hosted development of Proof-or-Stop under Proof-or-Stop) and explicitly admits selection bias, reviewer-judgement labels, and counterfactual asymmetry as threats to validity (§11).
- Cross-vendor review (Codex host-2) was used on the paper's own §5.2 statistics and measurement instrument, catching denominator bias, GLMM contamination, a hidden-regression baseline that failed open, and a reviewer benchmark that leaked the answer key — a credible internal adversarial check.

## Critical Assessment

- **Novelty**: The core move (treat agent outputs as claims, gate advancement on source-state-bound evidence) overlaps materially with two contemporaneous 2026 systems the authors themselves cite — Agentic Agile-V (conversation-to-contract gate, evidence-bundle acceptance) and ResearchLoop (claim ledger, claim-admission gates, self-hosting). The authors' honest delta is the binding to a continuously-changing code state (materialHash freshness as a first-class predicate) and the prohibited-wording guard; this is a genuine but incremental novelty rather than a new paradigm. The "Loop Engineering" branding adds terminology more than mechanism.
- **Rigor**: The powered ablation is genuinely pre-registered (H1/H2/H3 with a seeded cluster bootstrap, B=2000, Wilson CIs, FDR control) and the matched-cell invariant closes a denominator-bias threat the authors themselves disclose. However, three issues weaken the headline: (a) the primary A4-vs-A2′ contrast is not a clean equal-spend comparison — only **207/1920** A2′ cells land inside the ±20% spend band, and A4 spends ≈1.2× more; (b) the effect is heavily task-concentrated — **all of A4's amplification lives in a single task** (du-duration: 2/75), and excluding it both arms drop to ~0/1725, so the "matrix-level" gain is essentially one task; (c) exploratory per-scenario Benjamini–Hochberg yields **0/15** rejected contrasts, so no individual fault scenario reaches significance — only the aggregate does.
- **The cleanest contrast (A3 vs A4) is exploratory, not pre-registered**, and "near-compute" is not strict per-run matched. The H3 hypothesis is labeled exploratory in the paper itself.
- **External validity**: Only one model family (Claude Sonnet; the exact dated provider snapshot was not recorded — an explicitly stated reproducibility limitation), 24 stratified tasks of unstated provenance, and no external benchmark. The paper does not name the 24 tasks or release them as a recognizable SWE-bench-style suite, weakening comparability.
- **Self-hosting as both evidence and bias**: The 565-story / 1007-finding corpus is self-built, self-reviewed, and selection-conditioned; the 93% smoke_would_miss figure is reviewer-judgement labelled on a curated 12-story deep set, not a corpus-wide base rate. The authors acknowledge this honestly, but a reader should not read 93% as prevalence.
- **Missing ablations**: lane ablation (per §12 future work) is missing, so the marginal contribution of each review lane (quality / test-coverage / concept / docs / behavior) is not estimated; the cross-vendor marginal-finding rate is selection-described, not measured; the experience-layer causal A/B was stopped at pilot (control-arm baseline 1/30 = 0.033, far below planning band) and remains unpowered.
- **Red flags**: (i) The supplementary Cell03/Cell06 "3.80× token" readout is labeled gpt-5.5 in one place and Sonnet elsewhere, and the authors flag this as a model-family labelling risk; the 3.80× figure is not directly comparable to the 1.2× token ratio in Table 11 and risks being misread. (ii) The PINN/Quantum "schema smoke tests" appear in the repo and are explicitly disclaimed as non-evidence — their inclusion is borderline padding. (iii) Strong cross-vendor quorum is repeatedly "gated" / deferred, so the most security-relevant claim (independent multi-host quorum defeats a compromised runner) is not actually demonstrated; the local-key trust model explicitly does not defeat a compromised runner or prove semantic correctness.
- **Genuine praise**: The claim-boundary discipline is rare and commendable. Disclosing that A2′ is weaker than raw A2 in this run (so the headline could have been *better* against raw A2 and the authors did not cherry-pick that), publishing the analysis.glmm.R script, and having an independent Codex reviewer audit the headline statistics are practices more papers should adopt.

## Strengths

- **Formal, machine-enforced admissibility definition** (Eq. 2) with freshness binding to exact code state — a concrete, falsifiable control rule rather than a slogan.
- **Pre-registered powered ablation** with a seeded cluster bootstrap, matched-cell invariant, and explicit CI-excludes-zero primary endpoint.
- **Unusually honest claim-tier table** (Table 1) and a prohibited-wording guard that mechanically blocks the system from over-claiming.
- **Self-red-teaming contracts** (B-4c65 tamper suite, B-7b6e catchRate-vs-falseBlock gate-strength, M0–M4 instrument characterization) demonstrate the method applied to itself.
- **Operated, self-hosted scale** (565 stories, 1007 findings) rather than only synthetic tests.
- **Cross-vendor dogfooding caught real defects in the paper's own measurement code** (denominator bias, GLMM contamination, hidden-regression failing open) — a strong internal validity signal.

## Limitations

- Single model family (Sonnet), exact provider snapshot not recorded; no external benchmark; 24 tasks of unstated identity and provenance.
- Primary A4-vs-A2′ contrast is not equal-spend (only 207/1920 A2′ cells land in the ±20% band) and A2′ is weaker than raw A2 in this run.
- Effect is task-concentrated (all A4 amplification in du-duration); per-scenario FDR yields 0/15 significant contrasts; clean A3-vs-A4 isolation is exploratory.
- Self-built / self-reviewed corpus is selection-conditioned; 93% smoke_would_miss is a reviewer-judgement label on a curated 12-story deep set, not a base rate.
- Strong cross-vendor quorum, multi-model generalization, and external-benchmark results are explicitly gated as future work; local-key trust model does not defeat a compromised runner or prove claim semantics.
- Experience-layer causal A/B unpowered (control baseline 1/30); lane-level marginal contributions unestimated.
- Clean-task overhead is real (~2× cost and latency on easy work with zero completion benefit); the method is a risk-mitigation mechanism, not a free speed-up.

## Relevance

对关注"自主编码 Agent / Agent 安全 / 软件生命周期可验证性"方向的研究者，这篇论文提供了一个具体的、可复现的"把 Agent 声明变成可裁决证据"控制层范式（materialHash 绑定 + receipt + 跨厂商 review），其证据门控思想与 LLM critic、self-reflection、CI/CD provenance（SLSA/in-toto）等多条线互补，可作为构建可信 Agent pipeline 的参考基线。但其结论仅限单模型家族、自构建语料与一个高度集中的任务，外部基准和强跨厂商 quorum 仍是未来工作。

## Questions for Further Exploration

- How does the A4-vs-A2′ contrast change under strict per-run equal-spend matching (not just a budget band that only 207/1920 cells satisfy)? Is the headline robust to a truly compute-matched naive control?
- What is the real-work base rate of "visible acceptance passes while an independent hidden oracle fails" in non-injected development? The paper measures risk mitigation conditional on the trap; the missing denominator determines whether the overhead is worth paying in production.
- Would the gate still help on a recognizable external benchmark (SWE-bench-Verified, Multi-SWE-bench) and across model families (GPT, Gemini, open-weight), given the single-family Sonnet result and unrecorded provider snapshot?
- Can the strong cross-vendor quorum (Eq. 4) actually be powered, and does it survive an adversary who controls one host or recomputes digests (the explicitly out-of-scope Tier-C threat)?
- What is the marginal contribution of each review lane (quality / test-coverage / concept / docs / behavior) once lane ablation is run, and does the test-coverage lane's 70% high/critical density translate to a measurable defect-reduction effect?
- Does enforcing review as a lifecycle gate (vs advisory) generalize beyond the du-duration task where all of A4's amplification concentrated, or is the matrix-level gain an artifact of one trap-active task?
- How would the prohibited-wording guard and claim-boundary registry interact with a formal assurance-case notation (GSN) to make the admissibility rule auditable by third parties?
