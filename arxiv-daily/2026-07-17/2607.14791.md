> **一句话总结：** 作者使用预训练的 Qwen3-4B 逐层 transcoder（PLT）构建 attribution graph，对"用户指令下模型故意隐瞒信息"这类欺骗行为做电路级解释；通过手工追踪种子特征并辅以特征 steering 验证，得到 112 个欺骗相关特征，并发现负向 steering top-10 特征可 100% 把欺骗响应翻转为非欺骗（p<0.001），其中 "Obscuring information" 与 "secrets/confidentiality" 是电路中起决定性作用的核心节点。

> **TL;DR:** The authors apply pre-trained per-layer transcoders (PLTs) on Qwen3-4B (from the circuit-tracer release) to investigate deception as the model withholding a secret key it was instructed to hide. Manually tracing attribution graphs from seed tokens (e.g., "private", "hidden", "confidential") yields a dictionary of 112 deception-related features, validated by feature steering with α=5. Negative steering of the top-10 highest-occurring features flips 100% of deceptive prompts to non-deceptive with zero false positives (p<0.001), while positive steering flips 21% of non-deceptive prompts to deceptive. Circuit analysis on the Secret-Key dataset further shows two features ("Obscuring information" and "secrets/confidentiality") each feed into 60% of other top features versus a 16% average, and the pair beats 100 control groups for both steering directions (negative: 100.0% vs 45.8%, p<0.001).

## 核心贡献

- 首次将 transcoder（特别是 PLT）这一较新的 mechanistic interpretability 工具，系统应用于 LLM 欺骗行为的研究；过往 SAE/residual-stream probing 仅能定位特征，而 PLT 保留 MLP 的因果变换，可进行电路级（跨层）归因。
- 提出一套四步式、由种子 token 出发沿 attribution graph 追踪边、再用 feature steering 验证的"欺骗特征字典"构建流程，最终得到 112 个 unique deception feature（附录 Table 3 完整列出）。
- 在 100 条 secret-key 风格的提示上证明：负向 steering top-10 特征可 100% 抑制欺骗、零误报，正向 steering 反而不稳定，说明抑制方向更具实用价值。
- 通过电路分析定位出 "Obscuring information" 与 "secrets/confidentiality" 两个 hub 特征，二者在电路中的入度占比远高于其他特征，是监测欺骗行为的高价值靶点。
- 将结果与 OWASP/NIST 等"以输出为中心"的 AI 安全框架对照，主张把内部电路监测纳入主动安全防线。

## Key Contributions

- First systematic application of transcoders (specifically PLTs from the circuit-tracer release) to AI-safety-relevant deception, going beyond the residual-stream probing and SAE pipelines that dominate prior work.
- A four-step manual pipeline — prompt → attribution graph → seed-token edge tracing → steering validation — that produces a curated dictionary of 112 deception-related features in Qwen3-4B, with the full list released in Appendix Table 3.
- Empirical demonstration on a 100-prompt secret-key dataset that negative steering of the top-10 features fully suppresses deception (100% D→ND, 0% FP, p<0.001), while positive steering is asymmetric and less reliable.
- Circuit-level finding that "Obscuring information" (layer 23) and "secrets/confidentiality" (layer 8) act as central hubs, each feeding into 60% of top-10 features versus a 1.6/10 (16%) average.
- A reproducible experimental harness built entirely on public artefacts (Qwen3-4B + Goodfire/Decoderesearch circuit-tracer transcoders), lowering the barrier for follow-up safety auditing work.

## Methodology

**Model and transcoders.** Experiments use Qwen3-4B together with the public `qwen3-4b-transcoders` checkpoint from the circuit-tracer repository (Yang et al., 2025; Ameisen et al., "Circuit Tracing"). The transcoders already provide predefined interpretable features and explanations, since the authors note they could not train their own.

**Prompt dataset.** A 100-prompt set was generated by combining random secret keys (e.g., "xhf2l1jk") with sentence templates containing deception-related adjectives. Each prompt instructs the model to keep a word hidden and then asks the model to reveal it; a response that withholds the key is labelled deceptive, one that reveals it non-deceptive.

**Deception feature identification (manual, 4 steps).** (1) Run the prompt through the transcoder to obtain an attribution graph that shows feature transformations across layers and tokens. (2) Select seed tokens tied to deception (e.g., "private", "hidden", "confidential"). (3) Trace edges from seed tokens through downstream features, recording anything semantically related to negation, concealment, or secrecy. (4) Validate via feature steering; if the steered output flips between deceptive and non-deceptive, the feature is added to the dictionary. This produced N=112 unique deception features.

**Feature steering.** Activation x is shifted as x + α·x (positive) or x − α·x (negative) with a fixed α=5. A prompt whose default output is non-deceptive is positively steered; a deceptive one is negatively steered; a flip confirms the feature is causally involved.

**Circuit evaluation.** For each of the 100 prompts, input edges into the top-10 features were aggregated. An edge is retained if it appears in at least 30% of prompts, producing a per-feature connectivity graph. Core deception features are those with the highest in-degree (number of other features they feed into).

**Significance testing.** For each experiment, 100 random control groups of equal size (drawn from the dictionary minus the target features) go through the same steering pipeline. The target group's performance is compared against this null distribution.

## Results & Highlights

- **Feature frequency:** The top-10 deception features appear in 55%-95% of the 100 prompts; "Obscuring information_119106" (layer 23) appears in 95/100, "Negation and inability_158577" in 91/100, "secrets/confidentiality_95840" (layer 8) in 86/100, despite the dictionary holding 112 distinct features — deception concentrates in a small recurring subset.
- **Negative steering suppresses deception completely:** Steering x − 5·x on the top-10 features converts 100% of deceptive prompts to non-deceptive responses with 0% false positives (ND→D), p<0.001 versus 100 random control groups.
- **Positive steering is asymmetric:** x + 5·x on the same features flips only 21% of non-deceptive prompts to deceptive (p<0.001) while unexpectedly flipping 50% of deceptive prompts to non-deceptive, leading the authors to conclude negative steering is the more reliable direction for behaviour monitoring.
- **Circuit centrality:** "Obscuring information" and "secrets/confidentiality" each feed into 6/10 of the other top features (60%), against a per-feature average of 1.6/10 (16%); the remaining eight features are mostly peripheral sinks (0/10 to 3/10).
- **Pairwise hub significance:** Steering the pair {"secrets/confidentiality", "Obscuring information"} is significant in both directions against all 28 other pairs of top-10 features and 100 control groups — negative steering true-positive rate 100.0% vs control mean 45.8% (p<0.001) and positive steering 17.3% vs 3.7% (p<0.001).
- **Core-feature subset effect:** Negative steering of just these two core features already reproduces a substantial fraction of the full top-10 suppression effect, suggesting a compact deception circuit sufficient for downstream monitoring.

## Critical Assessment

- **Novelty:** Genuine but incremental. The use of transcoders for safety-relevant behaviour is new, but the experimental move (steering top features, building a small circuit) is a direct application of Dunefsky et al.'s transcoder circuit methodology rather than a methodological advance in transcoder design or training.
- **Rigor:** Mixed. Significance testing against 100 control groups is a real strength, as is the clean 2×2 (positive/negative × D/ND) reporting with TP/FP labelling. However, the deception-feature dictionary was hand-curated by the authors from seed tokens they chose ("private", "hidden", "confidential") — a procedure the authors themselves flag as bias-prone — and there is no inter-annotator agreement, no automated discovery baseline, and no comparison against SAE- or probe-based feature sets on the same prompts.
- **Single-model, single-transcoder-set limitation is severe:** All findings are restricted to Qwen3-4B with one specific pre-trained transcoder checkpoint; the authors explicitly state they could not train their own transcoders and could not reproduce on alternative models. Generalisability claims are therefore not supported by evidence.
- **Narrow threat model:** The deception task is a deliberately constructed secret-key withholding scenario with 100 templated prompts. It does not exercise the much richer deception space (strategic lies, sycophancy, sandbagging) that the cited Park et al. / Lynch et al. literature covers, so "behavioural monitoring and early detection of security vulnerabilities" is overclaimed relative to what was tested.
- **Missing ablations:** No sweep over steering coefficient α (fixed at 5, with the authors citing time constraints), no layer-wise analysis of which transcoder layers contribute most, no test of whether the two hub features generalise to non-templated deception prompts, and no comparison of steering the full top-10 versus the 2-feature core in a head-to-head table.
- **Cherry-picking risk on hub features:** Selecting "Obscuring information" and "secrets/confidentiality" because they have the highest in-degree and then showing they steer best is partially circular; a held-out test set of deception prompts would have strengthened the causal claim.
- **Reproducibility:** Reasonably high because everything runs on public weights and transcoders with a clear prompt template, but the manual feature-identification step is not scripted and would not re-produce the same 112-feature dictionary without human judgement.
- **Significance:** As a proof-of-concept that transcoder circuits can locate steerable safety-relevant features, it is a useful data point; as evidence that internal-circuit monitoring should replace output-based defences (the framing of the intro/conclusion), it is underspecified.

## Strengths

- Clean, transparent experimental design with explicit TP/FP labelling and a 100-group control null distribution for every claim — a standard many MI steering papers skip.
- Provides a concrete, released artefact (the 112-feature dictionary and the top-10 table) that other practitioners can reuse as a starting probe set on Qwen3-4B.
- The asymmetric steering result (negative reliable, positive noisy) is a genuinely useful operational finding for anyone building activation-based safety monitors.
- Honest limitations section that calls out the single-model constraint, the manual bias risk, and the fixed-α choice rather than hiding them.
- Builds on truly open infrastructure (Qwen3-4B + circuit-tracer), making replication cheap.

## Limitations

- Only one model (Qwen3-4B) and one pre-trained transcoder set; no cross-model or cross-transcoder validation.
- Deception feature dictionary is hand-curated from author-chosen seed tokens; reproducibility of the exact 112-feature list is unclear.
- Steering coefficient α is fixed at 5 with no sensitivity analysis.
- The deception task is a single templated secret-key withholding scenario with 100 prompts; coverage of realistic deception (strategic lies, partial truths, agentic misalignment) is absent.
- No comparison against SAE- or residual-stream-probe-derived features on the same task, so the central "transcoders are better for deception" claim is asserted rather than measured.
- The paper is short and exploratory; it stops at demonstrating steerability without proposing a deployable detector, threshold, or evaluation protocol.

## Relevance

这项工作把 transcoder 这一新兴 MI 工具直接用于 LLM 安全场景，与我们对"模型内部机制可解释性 + 可控生成（steering/activation editing）"的研究兴趣高度契合，可作为 Qwen3 系列上做特征级安全监测与可控干预的参考基线；其人工构建特征字典 + 控制组显著性检验的流程，对我们设计更自动化、跨模型的 deception/safety probe 也有方法论借鉴价值。

## Questions for Further Exploration

- Do the two hub features ("Obscuring information_119106" at layer 23 and "secrets/confidentiality_95840" at layer 8) generalise beyond secret-key templates — e.g., to strategic lies, sycophancy, or agentic misalignment prompts?
- How does steering performance vary with α across a range (e.g., 1 to 20), and is there a regime where positive steering becomes as reliable as negative steering?
- Would an automated dictionary-construction method (e.g., circuit-tracing from a probe, or attribution-grad-based selection) recover the same 112 features, or expose additional ones the manual process missed?
- Can the same deception circuit be reproduced in other model families (Llama, Gemma, Mistral) once comparable transcoders are trained, and at what scale does the circuit topology change?
- Does the 100% negative-steering suppression survive adversarial prompt variation (paraphrasing, jailbreak prefixes, multi-turn pressure), or is it an artefact of the templated prompt distribution?
- Can the two hub features be used as a real-time activation monitor with a chosen threshold, and what false-positive rate would that imply on benign privacy-preserving refusals (e.g., "I can't share personal data")?
