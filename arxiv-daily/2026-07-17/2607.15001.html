A reliable automation pattern for algebraically brittle scientific code — transferable to quantum chemistry, tensor networks, and any domain where code runs correctly but measures the wrong thing.

Engineering bottleneck, not physics reasoning, is what gets unlocked.
iii. Critical Edge

The headline 90% is buoyed by perfect scores on simpler classes — all 4 unresolved failures are baryon 3pt, and disconnected diagrams, four-point functions, and finite-temperature observables are entirely untested.

i. Methodology

Plan, then execute — with the brittle algebra lifted out of the LLM.

1.
Planning stage. A planner runs a solve–critique–rewrite loop, then a human checkpoint lets physicists review the ensemble / observables / propagators plan before any code is written.
2.
Execution stage. The executor emits PyQUDA measurement code + SLURM scripts, grounded by stage-routed skills (lqcd-physics-correlator, pyquda-tool, pyquda-gauge) and verified by static analysis + LLM critique.
3.
Deterministic generate_einsum. The most fragile step — Wick-contraction einsum strings — is delegated to a deterministic engine that enumerates valid pairings, signs, and spin–color labels. The model judges physics; symbolic construction is no longer its job.
4.
Validation protocol. Scripts must (a) complete a test run, and (b) numerically agree with hand-written references at |Δ|≲10⁻¹² — because in LQCD a runnable program can still measure a wrong observable.

iii. Strengths

  • Clean reliability boundary. generate_einsum converts "silently wrong observable" into "high-level judgment on the model, exact symbolic construction on deterministic code" — a transferable design pattern.
  • Honest, granular reporting. Per-class accuracy, named failing observables, and an explicit Matched / Convention-mismatch / Failure taxonomy instead of a single headline number.
  • Cross-backbone replication. GPT-5.4 vs. DeepSeek-V4-Pro is rare and genuinely informative about architecture-vs-model credit.
  • New physics, not just a benchmark. First diagonal Wilson-line meson LCDA; multi-hadron spectroscopy up to hypertriton (Wick counts 2 → 36 → 2880).

iv. Limitations

  • Baryon 3pt is the soft underbelly. All 4 failures are transition-current 3pt (p→p, Λ→Λ, Λ_b→Λ_c) — the critique loop does not yet catch the hardest spin–color routing.
  • Missing ablations. No isolation of generate_einsum vs. skills vs. the solve–critique–rewrite loop vs. the human checkpoint; only a deliberately handicapped Claude Code baseline.
  • Narrow ensemble. Everything runs on a single CLQCD C24P29 lattice; cross-ensemble generalization is untested.
  • Scope ceiling. Disconnected diagrams, ≥4-point correlations, finite-temperature observables, and large multi-particle operators are all out of scope.
