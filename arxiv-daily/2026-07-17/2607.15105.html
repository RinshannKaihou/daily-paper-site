Anatomy of the Method

01 / Methodology

Exact-token routing, decoupled from gradient memory

HGA routing. Context is split into 64-token chunks and 8-token groups. Compact summaries are built from the model's existing projected K vectors — no new router weights; gradients still flow through original Q/K/V/O and MLP.

Three-step selection: (1) every closed chunk keeps a summary resident in VRAM; query block scores & picks top chunks plus fixed sink/local windows. (2) Selected chunks open at group level via a larger VRAM cache (misses paged from RAM/NVMe). (3) Only exact token K/V of selected groups loads for attention — summaries are never used as attention values.

Segment-wise backward + 3-tier KV. Sequence split into length-S segments; each attends to local tokens + HGA-selected history, computes loss, runs backward immediately. At the boundary the segment's K/V is detached (classical TBPTT). Working set: M_model + M_adapters + O(S) + O(B_route); external record grows as O(L).

Matched protocol. Identical seed 1337, data order, hyperparameters, 100 steps — only the attention module differs. Both adapters then cross-evaluated under dense and routed readout.

03 / Strengths

What lands cleanly

  • Matched protocol. Same seed/data/hyperparameters, four-way cross-evaluation (2 train × 2 eval modes) — the right control.
  • Honest failure disclosure. Causal-leakage mode surfaced explicitly, with scoping of where the method is and isn't safe.
  • Strong engineering story. 2K → 16K on a 16 GB consumer card, with a public reference implementation.
  • Composable recipe. Per-segment backward + tiered KV needs no new router parameters or changes to pretrained projections.
  • Thorough characterization. Routing-sparsity sweep (Table 7) shows graceful degradation under budget pressure.
04 / Limitations

Where the claim is load-bearing

  • Single model & GPU. Qwen3-8B on one 16 GB Quadro RTX 5000 — no scaling study across sizes or vendors.
  • Only 100 optimizer steps. Equal-context equivalence (Table 6) shown for short runs only; may not hold at convergence.
  • No dense baseline beyond 4K. 8K/16K quality is uncontrolled; non-monotonic PPL across 8K→16K→32K is unexplained beyond "different document truncation."
  • Causal leakage. Current implementation unsafe for from-scratch pretraining; validated only for ~100M fine-tuning tokens.
  • NVMe tier unbenchmarked. RAM-backed is the only measured configuration; central scale-out thesis under-tested.
  • Routed inference is a prototype. vLLM/SGLang integration is future work — dense readout used for all main comparisons.
