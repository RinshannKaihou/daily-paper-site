> **一句话总结：** AutoSynthesis 是一个端到端多智能体系统，能将自然语言研究问题自动转化为完整的随机效应元分析（含 PRISMA 流程图、森林图、发表偏倚和风险偏倚评估），在 LLM 说服力基准上得到 Hedges' g=0.143，与人类专家（g=0.020）相差仅 Δg=0.123，落入 ±0.20 容差区间，研究召回率 71.4%（修正后 85.7%）。

> **TL;DR:** AutoSynthesis is an end-to-end LangGraph multi-agent framework that fully automates meta-analysis — from query rewriting and literature retrieval through standardized effect size computation (Hedges' g), random-effects REML pooling, heterogeneity/moderator analysis, publication-bias diagnostics (Egger, trim-and-fill), risk-of-bias (RoB 2/ROBINS-I), and PRISMA-compliant report generation. Benchmarked against the Hölbling et al. (2025) *Scientific Reports* meta-analysis on LLM persuasiveness, it recovered a pooled Hedges' g = 0.143 (95% CI [0.059, 0.226]) versus the human benchmark g = 0.020 (Δg = 0.123, within the ±0.20 tolerance region from Aczel et al. 2026), with 71.4% study recall (85.7% corrected) and Pearson r = 0.69 at the study level, at ~$1.5 and 0.5 hours per full run.

## 核心贡献

- **首个真正端到端的自动化元分析框架**：区别于以往只能做文献检索/筛选/信息抽取的 LLM 系统，AutoSynthesis 首次覆盖了元分析独有的步骤——标准化效应量计算、随机效应模型汇总、异质性分析与发表偏倚诊断。
- **多智能体架构映射 PRISMA 工作流**：将 Cochrane/PRISMA 2020 的每一步映射到专门的智能体（planning、search、paper reading、eligibility、RQ identification、statistical extractor、validation、relevance filter、analysis、bias、report），每步可独立审计。
- **LLM 推理 + 确定性统计的混合设计**：LLM 处理文档理解与结构化抽取，而效应量计算、REML 估计、Egger/trim-and-fill 均由自研 Python 数值例程执行，保留方法学严谨性。
- **防幻觉的统计验证机制**：统计校验代理逐条核对抽取数值在原文中的出处，无法定位的标为 hallucinated 并删除，修正可定位的错值。
- **可追溯的 PRISMA 报告**：自动产出 PRISMA 2020 流程图、森林图、漏斗图、累积分析、leave-one-out 敏感性分析、风险偏倚表与 Markdown 叙事报告，并保存全部中间决策日志。
- **与人类基准相近的量化结果**：在 LLM 说服力基准上，与专家元分析的效应量偏差仅 Δg=0.123（落在 ±0.20 容差区），并指出主要差异来自文献检索而非统计分析。

## Key Contributions

- First end-to-end multi-agent system that performs the **complete** quantitative meta-analysis workflow — not just screening/extraction — including standardized effect size computation (Cohen's d → Hedges' g), REML-based random-effects pooling, heterogeneity statistics (I², τ², Cochran's Q), moderator subgroup analysis, and publication-bias diagnostics.
- Modular agent design where each PRISMA stage (planning → search → eligibility → RQ identification → extraction → validation → filtering → analysis → bias → report) maps to a specialized agent, with full audit logs of every intermediate decision.
- A two-stage statistical extraction that first builds a structured "paper map" of studies/hypotheses/outcomes and only then extracts numbers, plus a validation agent that requires verbatim (or within-rounding) grounding in source text and discards unlocatable values as hallucinated.
- A "living meta-analysis" use case: the modular design lets users resume from any stage to continuously integrate new studies, extending the idea of living systematic reviews to quantitative synthesis.
- Empirical validation against a peer-reviewed human benchmark (Hölbling et al. 2025, *Scientific Reports*), reporting recall, precision, study-level correlation, and pooled-effect deviation with transparent CI overlap.

## Methodology

AutoSynthesis is implemented in Python 3.11 using **LangGraph** to orchestrate a ReAct-style loop (reason → tool call → observe) across roughly a dozen specialized agents. The workflow has four phases (Fig. 1):

1. **Planning and literature retrieval.** A *planning agent* translates the natural-language research question into a PRISMA-aligned protocol: treatment/control labels, broad inclusion criteria, strict exclusion reasons, candidate moderators, search queries, an earliest-year heuristic, and a positive-direction convention for effect coding. A *search agent* reformulates these into Boolean database queries, hits arXiv, Semantic Scholar, CrossRef, OSF Preprints, and PubMed via API (capped at 40 papers/database, configurable), deduplicates by normalized title, and applies regex pre-screening. A *paper reading agent* then fetches full text and parses it through a cascading PDF pipeline: MinerU (primary) → LlamaParse → Mistral OCR → conventional pdftotext, preserving tables and structure.

2. **Eligibility and study mapping.** An *eligibility agent* applies the inclusion/exclusion criteria to full text with an "INCLUDE when in doubt" policy (downstream statistical checks will catch errors), logging short exclusion reasons for the PRISMA flowchart. A *study identification agent* detects independent experiments/studies within a paper and a *research question identification agent* maps each to the target synthesis objective, ensuring each effect size is bound to a specific comparison.

3. **Statistical extraction and validation.** The *statistical result extractor* works in two phases: it first calls `record_paper_map` once to build a structured JSON of studies, outcome variables, and hypotheses-with-linked-outcomes; only then does it extract numbers (means, SDs, t/F, p, regression coefficients, odds ratios) restricted to the paper's own Results sections. A *statistical validation agent* then verifies each value verbatim-or-within-rounding against the source, correcting locatable errors and discarding unlocatable ones as hallucinated. A *relevance filter agent* classifies each row (primary / secondary / manipulation_check / etc.) and a deterministic algorithm picks one winner per study–outcome group, blocking manipulation checks and secondary analyses from the synthesis.

4. **Analysis and report.** The *analysis agent* converts whatever statistics are reported into Cohen's d and then Hedges' g, fits a **random-effects model via a custom REML implementation** (DerSimonian–Laird initialization, Viechtbauer iterative algorithm), and produces Cochran's Q, I², τ², forest plots, cumulative-by-year plots, and moderator subgroup analysis. The *bias assessment agent* runs Egger's regression test, trim-and-fill, leave-one-out sensitivity, and ROBINS-I (or RoB 2 for RCTs). The *report agent* assembles a PRISMA 2020 flow diagram, all plots, a study-level metadata table, and a Markdown narrative.

Crucially, **LLMs never perform the actual statistics** — all numerical routines (REML, Egger, trim-and-fill) are reimplemented in NumPy/SciPy rather than wrapping metafor/statsmodels, so the meta-analytic conclusions are deterministic once extraction is fixed. The default backbone is `openai/gpt-5.4-mini` via OpenRouter with reasoning effort "medium", ~200 LLM calls, ~1M input + 100K output tokens, ~$1.5 cost, and ~0.5 hours per end-to-end run.

## Results & Highlights

- **End-to-end run on the LLM-persuasion benchmark**: AutoSynthesis retrieved 28 candidate records → 25 after title/abstract screening → 19 full-text eligible → **8 studies included with k = 20 effect sizes and N = 29,382 participants**, versus the Hölbling et al. (2025) human benchmark of 7 studies / k = 12 / N = 17,422.
- **Pooled effect agreement**: AutoSynthesis estimated a pooled **Hedges' g = 0.143 (95% CI [0.059, 0.226], p < 0.001)**, while the human benchmark reported **g = 0.020 (95% CI [−0.048, 0.093], p = 0.530)** — a deviation of **Δg = 0.123**, within the ±0.20 tolerance region used by Aczel et al. (2026) to flag broadly-similar reanalyses, with widely overlapping CIs.
- **Study retrieval recall/precision**: 5 of 7 benchmark studies were recovered (**recall = 71.4%**, **precision = 62.5%**); after qualitative correction for one post-cutoff study, one ResearchGate-only preprint, and one genuine eligibility error, corrected recall rose to **85.7%** (6/7) and corrected precision to **87.5%** (7/8).
- **Study-level agreement**: across 7 matched effect sizes, Pearson **r = 0.69** (two-sided p = 0.085) and OLS slope **β = 1.62** between AutoSynthesis and the benchmark.
- **Heterogeneity and bias diagnostics in close agreement**: AutoSynthesis found I² = 88.3% vs. benchmark 75.97% (both substantial), and Egger's test p = 0.006 vs. p = 0.018 (both significant small-study effects).
- **Moderator analysis (Table S1)** surfaced meaningful heterogeneity by domain: General persuasion g = 0.331 [0.243, 0.418], Politics g = 0.182 [0.128, 0.236], Health g = −0.018 [−0.153, 0.116] (QM = 10.88, p = .004); by LLM model, Claude 3.5 Sonnet g = 0.393 vs. GPT-3 g = 0.067.
- **Leave-one-out stability**: pooled estimate ranged only g ≈ 0.13–0.16 across all omissions, confirming no single study drove the result.
- **Cost profile**: a complete end-to-end run cost ~$1.5, ~0.5 hours wall-clock, ~200 LLM calls — versus the >1000 person-hours and >1 year reported for typical manual systematic reviews (Borah 2017; Bastian 2010).

## Authors & Credibility

The five authors span Politecnico di Milano (Taherinezhad, Pierri), LMU Munich and the Munich Center for Machine Learning (Maier, Feuerriegel), and MIT CSAIL (Vitagliano). Pierri and Feuerriegel serve as joint corresponding/senior authors. Critically, **one author of this paper overlaps with the lead of the benchmark meta-analysis (Hölbling et al. 2025)**, which the authors disclose transparently and exploit to provide qualitative judgment on retrieval mismatches — both a credibility plus (insider ground truth) and a potential conflict to keep in mind. The work is funded by DFG under NFDI 27/1-2026 (project 460037581). The framing follows recent reporting-checklist guidance for LLM-based behavioral science (Feuerriegel et al. 2026, *Nature Human Behaviour*, cited as ref [55]), with model, decoding effort, token budget, and cost all reported.

## Critical Assessment

- **Novelty** is real but incremental rather than paradigm-shifting. The genuinely new piece is that AutoSynthesis goes past screening/extraction into **standardized effect-size computation and REML pooling**, a gap explicitly identified relative to prior systems like Manalyzer (ref [26]), Lattereview ([25]), and the *Nature Medicine* systematic-review automation of Chen et al. ([28]). The agent decomposition itself is a fairly direct mapping of PRISMA stages onto LLM agents, and several pieces (MinerU parsing, Egger/trim-and-fill, REML) reuse established tools.
- **Rigor of evaluation is the weakest link.** Despite claiming broad applicability across "human–AI interaction, education, and psychology," the paper actually reports **only one quantitative benchmark** (Hölbling et al. 2025 on LLM persuasiveness). The creativity/education reference cases (Holzner et al. 2025, etc.) are mentioned in the introduction and never quantified in the Results. With n=1 benchmark, the headline "within ±0.12 Hedges' g" generalization is unsupported.
- **The shared authorship with the benchmark team is a red flag** worth noting: the qualitative "corrections" that lift recall from 71.4% → 85.7% and precision 62.5% → 87.5% are adjudicated by the lead author of the benchmark itself, which is exactly the kind of post-hoc rationalization an independent replication would have to re-audit.
- **A statistical subtlety is glossed over**: AutoSynthesis's pooled effect is highly significant (p < 0.001) while the benchmark's is null (p = 0.530). Even though CIs overlap, the two analyses lead to **different qualitative conclusions** about whether LLMs are persuasive — the paper frames Δg = 0.123 as "broadly similar" but does not flag this sign-consistent-yet-significance-flipping discrepancy as rigorously as it should.
- **Missing ablations.** The paper claims robustness "across different LLM backbones, including open-weight LLMs" but **presents zero numbers** for those alternative backbones in the main text or supplementary tables — every reported result uses gpt-5.4-mini. There is no ablation over the 40-papers/database retrieval cap, no ablation over the cascading PDF parser order, no sensitivity to reasoning-effort level, and no human–machine agreement study on the eligibility agent in isolation.
- **Reproducibility is promised, not delivered.** "Will be made publicly available under an open-source license" and "data will be made available via a dedicated GitHub repository" appear at the end, but no link, commit, or release is given. Without the prompts and code, the ~$1.5/run claim cannot be independently verified.
- **Training-data contamination is acknowledged but under-mitigated**: the benchmark papers are exactly the kind of high-visibility material likely in gpt-5.4-mini's pretraining, which could inflate extraction accuracy without changing the (deterministic) statistical layer.

## Strengths

- Principled separation of LLM reasoning from deterministic statistics — the REML, Egger, and trim-and-fill implementations are custom NumPy/SciPy, so statistical conclusions are reproducible once extraction is fixed.
- The two-phase statistical extractor (paper map first, numbers second) plus the verbatim-grounding validation agent is a genuinely well-designed anti-hallucination pattern that other agentic-extraction systems should copy.
- Full PRISMA 2020 compliance, including the flow diagram, leave-one-out sensitivity, cumulative meta-analysis, and a proper risk-of-bias instrument chosen by study design (RoB 2 vs. ROBINS-I) — these are analyses human meta-analysts frequently skip.
- Transparent reporting of cost, tokens, LLM calls, decoding parameters, and prompt texts (full prompts are in the Supplementary Materials).
- Honest framing as a tool to support — not replace — expert reviewers, with a concrete "living meta-analysis" use case that is genuinely compelling.

## Limitations

- Single benchmark meta-analysis (n=1) for the quantitative evaluation; education and psychology use cases are advertised but not benchmarked.
- Shared authorship with the benchmark team weakens the independence of the qualitative corrections to recall/precision.
- Statistical-layer robustness across LLM backbones is claimed but not reported with numbers.
- Retrieval is capped at 40 papers/database and depends on paywall-free PDF accessibility — one benchmark study was missed purely because its full text was not retrievable, exactly the failure mode that biases automated synthesis toward open-access literature.
- Pooled estimate is statistically significant under AutoSynthesis and null under the human benchmark — the paper underplays this qualitative divergence.
- No human-interaction study; the human-in-the-loop path is described but never evaluated.
- Risk of LLM training-data contamination on benchmark papers is acknowledged but not measured (e.g., no held-out un-published study test).

## Relevance

高度相关于 LLM 智能体自动化科研工作流的研究兴趣：AutoSynthesis 把传统上耗时上千小时的元分析压缩到半小时、$1.5 成本，展示了多智能体 + 确定性统计的混合范式如何扩展到"科研综合"这一类高价值任务，是"AI Scientist"和"自动化科研评估"方向的一个扎实落地样本。其防幻觉统计抽取与可审计 PRISMA 报告的设计，对构建可信科研自动化系统有直接借鉴价值。

## Questions for Further Exploration

- How does AutoSynthesis perform on **n≥5 benchmark meta-analyses** spanning clinical medicine (e.g., Cochrane reviews with RoB 2), education, and psychology — not just LLM-persuasion?
- What is the **per-agent error contribution**? A confusion-matrix breakdown of eligibility, extraction, and validation individually would localize where to invest improvements.
- How sensitive are the pooled estimates to the **40-papers/database retrieval cap** and to the cascading PDF parser order (MinerU vs. LlamaParse first)?
- Does swapping in an **open-weight backbone** (e.g., Llama or Qwen) actually preserve Δg ≤ 0.20, and how do the prompts transfer?
- Can the framework handle **network meta-analysis** or **meta-regression with multiple continuous moderators**, beyond the binary subgroup analysis shown?
- How would a **held-out, never-pretrained-on** corpus of unpublished preprints change extraction accuracy, isolating contamination effects?
- Is the **AutoSynthesis-significant / human-benchmark-null discrepancy** (p < 0.001 vs. p = 0.530) a property of the included studies (3 extra studies, 8 extra effect sizes) or of the effect-size coding — and which conclusion is closer to ground truth?
