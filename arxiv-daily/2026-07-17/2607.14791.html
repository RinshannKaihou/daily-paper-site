Paper Dashboard · Circuit-Level Safety

Transcoders locate a steerable deception circuit inside Qwen3-4B.

Lim · Leow · Chia HTX Singapore Qwen3-4B + circuit-tracer PLTs 112 deception features
transcoders feature-steering deception-detection circuit-analysis qwen3 ai-safety
Key Metric/ 01
100%
Deceptive → Non-Deceptive

Negative steering (x − 5·x) of the top-10 features on 100 secret-key prompts, with 0% false positives, p<0.001 vs 100 control groups.

Why This Matters/ 02

A direct transcoder-based method for circuit-level deception monitoring on Qwen3 — a reusable baseline for automated, cross-model safety probes.

Source: Relevance
Critical Edge/ 03

All findings are restricted to Qwen3-4B with one specific pre-trained transcoder checkpoint; generalisability claims are therefore not supported by evidence.

Methodology / pipeline

STEP 01
Run prompt through the pre-trained per-layer transcoder to obtain a cross-layer attribution graph.
STEP 02
Seed tokens tied to deception — private, hidden, confidential — selected manually.
STEP 03
Trace edges downstream, recording features about negation, concealment, secrecy → 112 features.
STEP 04
Validate by steering at α=5; a flip between deceptive / non-deceptive confirms causal role.
  • Model: Qwen3-4B with the public qwen3-4b-transcoders checkpoint (circuit-tracer release; Yang et al. 2025; Ameisen et al.).
  • Prompt set: 100 templated secret-key prompts; responses labelled deceptive (withholds key) vs non-deceptive (reveals key).
  • Significance: each claim tested against 100 random control groups of equal size drawn from the dictionary minus the target features.

Results / 02

  • Frequency: top-10 features appear in 55–95% of prompts; "Obscuring information_119106" in 95/100.
  • Negative steering on top-10 → 100% D→ND, 0% FP, p<0.001.
  • Positive steering is asymmetric: only 21% ND→D (p<0.001) and unexpectedly 50% D→ND.
  • Hub centrality: "Obscuring information" + "secrets/confidentiality" each feed into 60% of other top features (vs 16% avg).
  • Pair vs controls: negative-steering TPR 100.0% vs 45.8% (p<0.001); positive 17.3% vs 3.7%.

Strengths / 03

  • Clean 2×2 design (positive/negative × D/ND) with explicit TP/FP labels and a 100-group null for every claim.
  • Releases a concrete artefact: the 112-feature dictionary and top-10 table for Qwen3-4B.
  • The asymmetric steering result (negative reliable, positive noisy) is operationally useful for activation monitors.
  • Honest limitations: single-model constraint, manual-bias risk, and fixed α are called out, not hidden.
  • Fully open infrastructure (Qwen3-4B + circuit-tracer) makes replication cheap.

Limitations / 04

  • One model, one transcoder set — no cross-model / cross-transcoder validation.
  • Dictionary is hand-curated from author-chosen seed tokens; exact 112-feature list hard to reproduce.
  • α fixed at 5 with no sensitivity sweep; no layer-wise analysis.
  • Task is a single templated secret-key scenario; strategic lies / sycophancy / agentic misalignment not covered.
  • No comparison against SAE- or probe-derived features on the same task.
  • Stops at demonstrating steerability — no deployable detector, threshold, or evaluation protocol.

Framing & Contribution / 05

  • First systematic application of transcoders (PLTs) to AI-safety-relevant deception, beyond residual-stream probing and SAE pipelines.
  • Locates two hub features — "Obscuring information" (layer 23) and "secrets/confidentiality" (layer 8) — as high-value monitoring targets.
  • Argues for folding internal-circuit monitoring into OWASP/NIST-style output-centred AI safety frameworks.
  • Positioned as a methodology demonstration, not a production deployment; preliminary / exploratory tone.
