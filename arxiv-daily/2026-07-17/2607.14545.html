Key Finding / 01

CASP asks predictors for verifiable “ignore-this-region” certificates instead of solutions — the worst-case approximation ratio becomes provably independent of prediction quality.

Key Metric — verified vs unverified

26.2% → 0%

Under distribution shift, unverified ML pruning loses up to 26.2% of the optimum on 97.9% of Knapsack instances; the verified gate loses nothing on all 48.

03

Why this matters

Treats the verifier — not the prediction’s sign — as the key resource, yielding PAC sample complexity Õ(ε⁻² log K): a theoretical backbone for provably-safe ML-for-MILP pruning.

04

Critical Edge

The f-safe factor is vacuous at large f: on rail507 the certified bound is ~7800 while the realized gap is 1.15% — the formal guarantee is empty exactly where the empirics look best.

02

Methodology

certificate system · two-layer deploy

A minimization problem is recast as a certificate system 𝒞 = (L, V, P): an assertion language, a polynomial-time verifier, and a pruning operator. Predictors only emit “this region is safe to ignore” claims; the verifier rejects unsound ones before they ever touch the solver.

  • Two graded safety classes. A certificate is OPT-preserving if some optimum survives pruning, or ρ-approximation-safe if OPT(Iᵪ) + c_fix ≤ ρ·OPT(I).
  • Two-layer deployment (Algorithm 1). Layer 1 = verifier discards failed witnesses; Layer 2 = feasibility recheck + classical α-approximation fallback A.
  • Robustness (Thm 3). c(S) ≤ max(ρα_red, α)·OPT — whenever ρα_red ≤ α, the framework is never worse than the fallback.
  • Bounded-loss PAC (Thms 13–14). The verifier caps the loss at B, so thresholds are learnable with pseudo-dimension O(log K) — R-independent vs the bare interface’s Ω(R/ε²).
  • Verifiable confidence filtering (§6). Commit only elements whose verifier-computed LP confidence σ _i ≥ θ; provably weakly dominates the min-combiner (Thm 15).
  • Five instantiations. Set Cover (LP threshold), Vertex Cover (Nemhauser–Trotter), Facility Location (facility-integral), 0/1 Knapsack (reduced-bound fixing), Steiner Tree (R1/R2 tests).

04 Strengths

what lands

  • Prediction-independent guarantee (Thm 3). Correctness is decoupled from ML quality — the property practitioners want but rarely get from “algorithms with predictions.”
  • Closed-form filter margins (Thm 16), independently validated by Monte-Carlo recomputation rather than qualitatively matched.
  • Intellectual honesty. §7.4 self-collapses the two headline contrasts under a fair adversary; openly lists where certificates stay silent.
  • Five diverse instantiations — covering, packing (Knapsack), and connectivity (Steiner) — rule out the “one LP skeleton” objection.
  • Independent verifier reimplementation + presolve audit raise zero-mismatch / speedup claims above typical ML-for-OR credibility.

05 Limitations

where it breaks

  • Vacuous guarantees at large f. The certified factor grows with frequency — ~7800 on rail-scale Set Cover vs a ~1% realized gap. A non-degrading class is an open problem.
  • Certificates go silent on dense graphs. Nemhauser–Trotter fires on only 12/97 real DIMACS/SNAP graphs; Knapsack fixes nothing on correlated/subset-sum types.
  • Sample-complexity experiments near-trivial on benign distributions; discriminating signal appears only on heavy-tailed stress tests (E4″).
  • Filter value concentrates on LP-degenerate instances. On low-degeneracy VC it learns θ=1.0 and ignores the predictor (Prop. 3).
  • Single solver (SCIP 10.0) and a benign E4′ distribution; transfer to Gurobi/HiGHS/CBC untested.
  • Headline OR-Library wall-clock speedups superseded by the presolve audit — retained in Table 4 only for provenance.
