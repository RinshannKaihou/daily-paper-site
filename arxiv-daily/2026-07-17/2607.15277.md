> **一句话总结：** 本文用"二叉条件树"递归划分人群，发现 LLM 的子群条件估计通过全概率公式重组后，反而比直接询问总体估计更贴近真实调查数据（作者称为 "macro fallacy"），同时提出无需参考答案的 split/order 一致性检查，证明包括 GPT-5.4、Sonnet 4.6、Grok 4.3 在内的前沿模型都广泛违反这些基本概率恒等式。

> **TL;DR:** The paper stress-tests the common view that in-context learning is conditional inference by constructing binary conditioning trees over populations (2024 ACS, World Values Survey, and two synthetic forecasting tasks) and asking whether LLM estimates compose under the law of total probability. They find widespread violations even in frontier models (GPT-5.4 split-consistency 0.33, Sonnet 4.6 0.00 on ACS income at ε=0.02), document a "macro fallacy" where reconstructed aggregates from fine-grained subgroup estimates beat direct population-level prompts, and show consistency does not scale with the Artificial Analysis Intelligence Index — establishing statistical self-consistency as a new reference-free evaluation axis complementary to alignment.

## 核心贡献

- 提出 **统计自洽性（statistical self-consistency）** 作为评估 LLM 的新维度：把"上下文学习=条件推断"这一隐含假设，转化为可验证的概率恒等式（全概率公式、全期望公式），无需外部 ground truth 即可检查。
- 设计 **二叉条件树（Binary Conditioning Tree, BCT）** 框架：用一组二值属性递归划分基础人群，每一层形成一个合法划分，从而把"直接估计总体"和"由子群条件估计重组总体"放到同一坐标系下比较。
- 发现并命名 **macro fallacy**：直接让模型估计 P(E|X) 反而比"先估计子群条件再按先验重组"更差。该效应在不同模型、不同目标变量、不同阈值、不同树结构下都稳健成立。
- 提出 **micro-to-macro prompting**：在单条 prompt 内让模型先列举相关子群再给出总体估计，可以部分地、几乎零成本地恢复上述聚合增益。
- 给出两类局部一致性检查：**split consistency**（同一子群与其两个子划分是否满足全概率公式）与 **order consistency**（"年轻且在职"是否等于"在职且年轻"），并证明在合理假设下二者可共同覆盖所有由该属性集诱导的树。
- 跨多个任务域（ACS 收入与通勤时间预测、WVS 全球意见建模、合成网球/奇幻战斗预测）实证发现：前沿模型（GPT-5.4、Sonnet 4.6、Grok 4.3、Qwen3.6 Plus 等）普遍违反这些检查，且分数不随 AAI 智能指数系统提升。

## Key Contributions

- Introduces **statistical self-consistency** as a *reference-free* evaluation criterion for LLMs: it turns the "ICL = conditional inference" assumption into checkable probabilistic identities (law of total probability / total expectation) and applies even when no ground-truth labels exist.
- Builds the **binary conditioning tree (BCT)** scaffold: a single shared binary split per level generates a valid population partition at every depth, giving a parameterized family of statistically equivalent estimators for the same aggregate.
- Identifies and names the **macro fallacy**: direct population-level prompting is systematically *less* aligned with human reference data than reconstructed aggregates built from finer-grained conditional estimates plus priors — robust across models, thresholds, tasks, and tree structures.
- Proposes **micro-to-macro prompting**, a lightweight single-query intervention that lets the model internally reason over subpopulations and partially recovers the aggregation benefit without an externally specified partition.
- Formalizes two complementary local checks — **split consistency** (law of total probability across a refinement) and **order consistency** (invariance to constraint ordering) — with proofs (Propositions 2, 6, 7) that, under a context-independent pairwise assumption, these local checks control global consistency across all trees induced by the attribute set.
- Demonstrates the framework on three domains: 2024 American Community Survey (ACS) income/commute prediction, World Values Survey (WVS) opinion modeling via GlobalOpinionQA, and two synthetic forecasting tasks — exposing that no frontier model is consistently self-consistent and that scores do not track standard capability indices.

## Methodology

The method is built on the law of total probability: for any valid partition P of a base population X, P(E | X∈X) = Σ_{S∈P} γ_S · P(E | X∈S), where γ_S is the relative size of subgroup S. The authors operationalize this through three components.

**Binary conditioning trees (BCT).** A BCT of depth L is a tuple T = (X, {φ_1,…,φ_L}) where each level ℓ has one binary split function φ_ℓ applied uniformly to every node at level ℓ−1. Each tree level yields a valid (disjoint, exhaustive) partition of X, so the same aggregate target quantity can be estimated at any granularity. For the ACS running example, splits are chosen by a regression-tree-style objective that reduces prior-weighted within-subgroup income variance while keeping priors balanced: φ_1 = "age 31–68", φ_2 = "employed or self-employed". The ACS data (2024, 3.8M individuals) serves as the ground-truth reference population.

**Elicitation.** Each subpopulation is verbalized as a concatenated persona-prompt string and fed to the LLM (using the `folktexts` library for systematic ACS prompts). For each node S the model produces a conditional estimate T̂_τ(S) = P(Y > τ | X∈S), and priors γ̂_S are elicited level-wise via multiple queries, renormalized, and averaged. Reconstructed aggregates are T̂_{τ,agg}^{(ℓ)}(X) = Σ_{S∈P_ℓ} γ̂_S · T̂_τ(S). Under the conditional-inference view these reconstructed aggregates should agree across levels and with the direct root-level estimate.

**Two self-consistency checks.** (1) *Split consistency* compares the direct estimate of a conditioning event S against the prior-weighted aggregate of its two refinements along a held-out attribute A_k; a pair (S, A_k) is ε-split-consistent if dist(T̂_h(S), T̂_{h,agg}(S; A_k)) < ε. Aggregated over all admissible (S, A_k) pairs in an attribute set A, this gives the split consistency score SC_ε(A). (2) *Order consistency* compares the estimate for a two-attribute event under both constraint orderings (e.g., "young and employed" vs. "employed and young"); a pair is ε-order-consistent if the two orderings agree within ε, yielding OC_ε(A). Propositions 2, 6, and 7 give worst-case bounds showing local checks control global tree consistency. Discrepancies use the normalized Wasserstein-1 distance on ordered categorical outcomes, with tolerance ε = 0.02 throughout.

**Error decomposition.** Using the law of total variance, the authors separate error sources: refinement moves variance from within-group heterogeneity to cross-group structure (Figure 4c). Conditional estimates improve with specificity, but LLM-estimated priors degrade (TV distance to survey priors grows with depth), producing the concave aggregation-gain curve.

## Results & Highlights

- **Macro fallacy is robust and quantitatively large.** On ACS income prediction, the relative alignment gain 1 − AErr(ℓ)/AErr(0) of reconstructed aggregates over direct prompting is consistently positive across GPT-5.4, GPT-4o mini, Opus 4.7, Sonnet 4.6, Gemini 3.1 Pro, Gemini 3 Flash, and Qwen3.6 Plus for thresholds τ ∈ {100, 1k, 10k, 20k, 40k, 60k, 80k} USD (Figure 3), with gains reaching ~50% relative error reduction at intermediate tree depths.
- **Widespread split-consistency violations on ACS income** (Table 1a, ε = 0.02): GPT-5.4 SC = 0.33, Sonnet 4.6 SC = 0.00, Grok 4.3 SC = 0.00, Qwen3.6 Plus SC = 0.17. On ACS commute time (Table 1b): GPT-5.4 SC = 0.17, Sonnet 4.6 SC = 0.17 — i.e., every frontier model fails the majority of law-of-total-probability checks.
- **Self-consistency does not scale with capability.** Figure 6 sweeps many models within each family against the Artificial Analysis Intelligence (AAI) Index: despite large capability jumps within the GPT-5.x, Gemini, Claude, Grok, and Qwen families, neither split nor order consistency improves systematically.
- **Global opinion modeling (WVS, GlobalOpinionQA) confirms the pattern across countries.** In Table 2, averaged across 5 survey questions × 2 countries (Canada, Indonesia), Sonnet 4.6 has the best mean split consistency 0.70, followed by GPT-5.4 at 0.55 and Grok 4.1 Fast at 0.48; DeepSeek V-3.2 and Qwen3.5 27B are at 0.32. *No model is uniformly consistent across questions.*
- **Synthetic forecasting fails too.** On the Federer–Nadal tennis task (Table 3a), no model satisfies more than half of split-consistency checks (max = Sonnet 4.6 at 0.50; Qwen3.6 Plus = 0.00). Order consistency is uniformly higher than split consistency, indicating models handle constraint reordering better than probabilistic composition.
- **Micro-to-macro prompting partially recovers the benefit.** Figure 5 shows positive average error gain of micro-to-macro over direct prompting on ACS across most model–threshold pairs, with a win matrix dominated by green tiles — though the effect is "more model-dependent and less systematic" than explicit tree-based aggregation.
- **Error source disentanglement (Figure 4).** Node-wise conditional alignment error improves from the root to deeper levels, but LLM-estimated priors become less aligned: total-variation distance between LLM-estimated and survey-based priors grows monotonically with tree depth. Within-subgroup variance decreases with depth while cross-subgroup variance increases, confirming the variance-decomposition intuition.

The mix of survey-data expertise and decision-theoretic / Bayesian foundations is well matched to the paper's thesis. The work was supported by the Max Planck ETH Center for Learning Systems and the Hector Foundation.

## Critical Assessment

- **Novelty.** The framing is genuinely fresh: instead of checking isolated logical relations among forecasts (as in Paleka et al., 2025) or generator–validator consistency (Li et al., 2024), the paper converts *partition-based* aggregation identities into a structured, parameterized family of checks. The "macro fallacy" is a memorable, actionable empirical regularity rather than a generic "LLMs are inconsistent" claim.
- **Rigor.** Strong on the theory side — Propositions 2, 6, 7 give worst-case bounds tying local checks to global tree consistency, and the oracle-prior ablation (Appendix A.1) cleanly isolates conditional error from prior error. The concave aggregation-gain curve is correctly attributed to a conditional-vs-prior trade-off rather than to test-time compute, an alternative the authors explicitly rule out (deeper partitions need more calls but yield worse aggregates).
- **Reproducibility — moderate.** ACS and WVS are public; `folktexts` and GlobalOpinionQA are released; the split attributes (age, COW), thresholds, and tolerance (ε = 0.02) are specified. However, the full prompt templates and prior-elicitation protocol are only in appendices, and exact model versions, decoding parameters, and number of samples per cell are not all surfaced in the main text.
- **Significance.** High for the LLM-as-survey / persona-prompting and forecasting communities: it gives a concrete, license-free diagnostic and a cheap mitigation. The claim that consistency does not correlate with AAI index is provocative and, if it holds up, is important for benchmark design.
- **Red flags / things to probe.**
  - **Model naming is anachronistic.** The paper reports results for "GPT-5.4," "GPT-5.5," "GPT-5.6 Luna/Sol," "Sonnet 4.6," "Gemini 3.1 Pro," "Grok 4.3," "Qwen3.7 Plus/Max," "DeepSeek V-3.2." Several of these names do not correspond to publicly released models as of the July 2026 date, and no model-version identifiers, API access details, or release dates are given in the main text. Reviewers should verify these are real, reproducible endpoints and not projected/hypothetical model names.
  - **A single tolerance ε = 0.02 carries most of the headline numbers**, but its justification is thin in the main body — the entire "split = 0.00" headline for Sonnet 4.6 / Grok 4.3 on ACS income depends on this threshold, and a sensitivity analysis is not foregrounded.
  - **The ACS main tree uses only two attributes** (age, class of worker), so the headline BCT is shallow (depth ≤ 2 for the persona tasks, depth 4 in some appendices). Generalization to deeper, higher-arity trees is asserted but not the main empirical thrust.
  - **Self-consistency is explicitly a *necessary* condition** — the authors are honest about this, but it weakens the "models are bad at conditional inference" framing: a model could in principle satisfy all checks while still being misaligned, and conversely high alignment at the population level (their own point) does not certify subpopulation faithfulness.
  - **The "macro fallacy" could partly reflect a known LLM bias** toward over-weighting salient / majority subgroups when prompted at the aggregate level (they note direct prompting "underweights young individuals, causing income to be overestimated"). This is a plausible mechanism but is presented as a single qualitative example rather than a systematic decomposition.
  - **Order-consistency checks are restricted to two-attribute events** under Assumption 1 (context-independent pairwise discrepancy), which is untested; the quadratic blow-up bound ε' = (|J| choose 2)·ε could be loose.

## Strengths

- A crisp, *reference-free* evaluation axis that is genuinely orthogonal to alignment — useful precisely where ground truth is expensive or undefined (the fantasy-combat forecasting task has no ground truth and still yields a meaningful consistency score).
- Clean formal scaffolding: BCTs make "statistically equivalent estimators" concrete and turn the law of total probability into a parameterized family of checks with proven bounds.
- The micro-to-macro prompting trick is a practically useful, near-zero-cost intervention that the authors are careful not to oversell.
- Intellectually honest limitations section: explicit that self-consistency is only necessary, that ACS is a single domain, and that elicitation design choices matter.
- Cross-domain validation (ACS income, ACS commute, WVS across two countries, tennis, fantasy) makes the "widespread violations" claim credible rather than a single-task artifact.

## Limitations

- Alignment evaluation relies primarily on ACS, a single (if excellent) domain; WVS adds two countries and five questions, but the breadth of "macro fallacy" evidence is still narrow.
- Self-consistency ≠ faithful conditional inference; internally consistent models can still be wrong, and the paper does not close that gap.
- Headline numbers hinge on one tolerance (ε = 0.02) and a normalized Wasserstein metric whose choice is only weakly justified in the main text.
- The synthetic forecasting tasks (tennis, fantasy) have no real-world validation; they demonstrate the *applicability* of the framework, not forecasting accuracy.
- Anachronistic / unspecified model versions (see Critical Assessment) hinder independent reproduction.
- Order consistency is restricted to two-attribute events under an unverified context-independence assumption.

## Relevance

这项工作直接关系到"如何评估 LLM 是否真的在做条件推断 / 是否能可靠地作为人群意见模拟器与概率预测器"这一核心问题，对 persona prompting、合成调查、LLM 预测与不确定性量化方向的研究者高度相关；其提出的 reference-free 一致性检查和 micro-to-macro prompting 也是日常在做 LLM 评估与 prompt 工程时可以立即借鉴的诊断工具。

## Questions for Further Exploration

- Does the macro fallacy persist when conditional estimates and priors are elicited via logit-based / verbalized-vs-numeric probability methods, or is it an artifact of the chosen elicitation format?
- How sensitive are the headline SC_ε / OC_ε scores to the tolerance ε, and is there a principled (e.g., calibration-based) way to set it per task?
- Can self-consistency be *trained into* models (e.g., via RL on partition-based consistency constraints) without harming alignment, and would that close the knowing–doing gap noted by Gu et al., 2026?
- Does the macro fallacy extend to non-demographic conditioning (e.g., conditioning on document context, tool outputs, or chain-of-thought premises), where the "subpopulation" is implicit?
- Are the reported "GPT-5.4 / Sonnet 4.6 / Grok 4.3 / Qwen3.7" endpoints real, accessible models, and do the consistency scores replicate across API versions and decoding seeds?
