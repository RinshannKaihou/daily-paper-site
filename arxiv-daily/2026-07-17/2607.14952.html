01 Methodology

Capture once, replay the suffix.

An opt-in long-context execution extension over the MinT runtime (Ray-resident Megatron workers). For one grouped GRPO update k:

  • Phase 1 — Prompt capture. Run the P-token prompt with autograd disabled; save only architecture-specific conditional state (Qwen compact GDN matrix + full-attn KV pages; GLM absorbed MLA latents + DSA index-key pages). Release all dense FFN / MoE router / attention scratch.
  • Phase 2 — Pre-step scoring. Materialize old-policy & reference-policy response log-probs against the read-only captured state.
  • Phase 3 — Policy replay. For each member i, rebuild a short response graph under autograd, reuse frozen prompt state, backprop the GRPO loss (clipped-ratio surrogate + ref KL), then immediately release.
  • Phase 4 — Optimizer. One AdamW (Qwen) / DistOpt (GLM) call per worker; captured state is now stale vs θk+1.

03 Strengths

  • Genuinely novel framing. Long-context RL under a fixed budget as a tensor-lifetime / ownership problem — distinct from scale-out long-context literature (Ring Attention, DeepSpeed-Ulysses, ByteScale).
  • Two architecturally opposite implementations (GDN/full-attn dense hybrid vs MLA/DSA MoE with cross-layer IndexShare reuse) lend the design principle generality.
  • Exceptional engineering rigor. Exact byte arithmetic for KV and MoE buffers; a staged OOM-migration ladder; pinned source git hashes; rank-complete trace inventories.
  • Refreshing intellectual honesty. A four-level evidence ladder, an explicit semantic-claim matrix (Table 8), and a candid limitations section prevent the headline numbers from being over-read.
  • Reusable systems lessons: physical ownership is part of the algorithm; sparsity transfers cost rather than removing it; CP and EP are orthogonal.

04 Limitations

  • No real training. Synthetic responses, deterministic rewards, no online rollout, no reward model, no repeated updates, no checkpoint/reload, no policy-improvement measurement.
  • Distributed-update correctness not achieved. Qwen's K/V adapter gradients are not reduced across CP ranks; GLM uses CP-local DSA and bypasses finalize_model_grads.
  • Mathematically different objective. The prompt state is detached, dropping the ∂z_P/∂θ term — even repaired gradients would not match conventional full-sequence GRPO.
  • No matched baseline or parity test on the same model/suffix/precision at a context that fits on one process.
  • Thin scaling evidence. All timings single runs; Qwen samples only G∈{2,8}; GLM has one grouped G=2 receipt; no GLM whole-run peak; no GLM probe above 2M.
  • Cost accounting incomplete — host memory, network, utilization, energy, and monetary cost are not accounted.
