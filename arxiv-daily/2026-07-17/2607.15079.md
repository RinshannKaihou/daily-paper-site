> **一句话总结：** BrainPilot 是一个面向脑科学的全开源人机协同多智能体系统，由 PI Agent 协调五个专家 Agent（Librarian/Experimentalist/Engineer/Writer/Auditor），基于 7,233 条知识库条目和 72 个可复用技能、配合可审计的 Graph of Trace 与独立的 Auditor 防造假模块；在 Agents' Last Exam 与自建 BrainPilotBench-v0 上，使用开源 DeepSeek-V4-Pro 主干时分别以 $0.08 / $0.504 的成本取得与 Codex、Claude Code 等顶配框架相当甚至更好的可交付成果。

> **TL;DR:** BrainPilot is a fully open-source, human-in-the-loop multi-agent system for brain-science research, where a PI agent coordinates five specialist agents (Librarian, Experimentalist, Engineer, Writer, Auditor) grounded in a 7,233-item knowledge base and a 72-skill library, with every step recorded in an append-only Graph of Trace and an Auditor agent that performs fabrication checking. On three brain-science tasks from Agents' Last Exam and the new BrainPilotBench-v0 (RSC place-cell, TOPS-fMRI, BCI IV 2a, Sleep-EDF), BrainPilot with the open-source DeepSeek-V4-Pro backbone matches or exceeds Codex and Claude Code on the deterministic tasks at 5–56% of their cost (e.g., $0.08 vs. $22.53 for Codex+GPT-5.5 on ALE, and $0.504 vs. $4.931 for Codex+GPT-5.5 on BrainPilotBench), while remaining unreliable on the open-ended C. elegans neuron-tracking task.

## 核心贡献

- **PI 主导的多智能体编排**：提出由 Principal Investigator Agent 负责任务分解、调度和综合，统一指挥 Librarian、Experimentalist、Engineer、Writer、Auditor 五个专家 Agent 的可审计科研流水线，明确用户唯一对话入口是 PI。
- **领域知识接地**：构建统一脑科学知识库（7,233 条教材/论文条目，11 个学科分组，BGE-m3 嵌入 + BGE reranker 两阶段检索）与 72 个跨 7 大研究域的可复用 Skill（含 DeepLabCut、MNE-Python、fMRIPrep、SpikeInterface 等成熟工具的派生技能）。
- **Graph of Trace（GoT）**：将每个子目标的产物、工具调用、证据、声明以 schema-validated 节点形式追加到 append-only 的有向无环图中，禁止回溯编辑、循环与孤儿引用，提供实时可视化与逐节点可审。
- **Auditor 防造假机制**：独立 Agent 对 Writer 的草稿逐条核查数值、文件/产物、引用三类声明，分类为 confirmed/unverified/disputed，将验证内嵌入长程工作流。
- **新基准 BrainPilotBench-v0**：覆盖钙成像（RSC place-cell）、功能连接 fMRI 疼痛解码（TOPS-fMRI）、运动想象 EEG（BCI Competition IV 2a）、睡眠分期（Sleep-EDF）四类任务，采用容器化、无网络、Oracle 双侧验证的工件评分合约。
- **端到端真实案例**：在 RSC 空间编码、小鼠视觉系统功能层级、fMRI 疼痛解码、BCI 运动想象、Sleep-EDF 五个真实科研场景中完成从原始数据到科学报告的全流程。

## Key Contributions

- **PI-led multi-agent orchestration**: A Principal Investigator agent is the single user-facing entry point that decomposes the goal, delegates subtasks to five specialist agents (Librarian, Experimentalist, Engineer, Writer, Auditor), and synthesizes their outputs into a final audited deliverable.
- **Domain grounding via curated assets**: A unified retrieval knowledge base of 7,233 textbooks/papers across 11 disciplinary groups (BGE-m3 embedding, BGE-reranker-v2-m3 two-stage retrieval) and a 72-skill library across 7 research domains, including skills distilled from DeepLabCut, MNE-Python, fMRIPrep, netneurotools, pycortex, and SpikeInterface.
- **Graph of Trace (GoT)**: An append-only, schema-validated directed acyclic graph that records every major agent action (description, artifact references, parent IDs, inferred dependency edges), forbidding retrospective edits, cycles, or orphaned references, and rendered live for human inspection.
- **Auditor agent for fabrication checking**: An independent agent classifies numerical, file/artifact, and citation claims in the Writer's draft as confirmed, unverified, or disputed against evidence in the session workspace, integrating verification into long-horizon workflows.
- **BrainPilotBench-v0**: A new four-task benchmark (RSC place-cell calcium imaging, TOPS-fMRI tonic-pain decoding, BCI Competition IV 2a, Sleep-EDF) with frozen artifact contracts, two-sided Oracle validation, and a no-network, non-root containerized scoring environment.
- **Five end-to-end case studies** spanning systems neuroscience, mouse visual hierarchy, fMRI pain biomarkers, motor-imagery EEG, and sleep staging, demonstrating the full data-to-report pipeline.

## Methodology

**Architecture (three coupled elements).** (1) Specialist agents: each agent is defined by a shared system constitution, a role definition (mission, working style, tools), and a set of skills; all agents obey a common messaging contract and a self-recording contract that logs tangible outputs to the GoT. A skill-router tool exposes modular skills to agents on demand: agents issue task-relevant keywords, receive compact candidate skill descriptions, and load full instructions only when a capability is needed (skills-first preflight before adopting any non-trivial method). (2) Curated domain knowledge built offline: source documents (textbooks and papers) are OCR'd with DeepSeek-OCR, chunked into 1,500-character overlapping passages (200-char overlap), embedded into 1,024-dim vectors with BGE-m3, and queried via vector similarity plus a cross-encoder reranker (BGE-reranker-v2-m3). Skills come from three pipelines: an LLM authoring pipeline, a paper-to-skill distillation pipeline, and a repo-to-skill tool-derivation pipeline. (3) Graph of Trace implemented as an agent-callable tool: each major agent reports its product, the tool converts the report into a schema-validated node with artifact references and parent IDs, and infers dependency edges beyond temporal order.

**PI delegation policy.** The PI handles lightweight framing, synthesis, and decisions directly; anything requiring domain expertise or more than a few minutes is delegated. Deliverables needing report-like structure pass through the Writer, then to the Auditor for independent verification before final return to the user.

**Agent team.** PI (coordinator, only agent the user instructs directly); Librarian (literature + knowledge grounding, search/read tools only, delivers Key Findings / Knowledge Gaps / Suggested Hypotheses / References); Experimentalist (operationalization, control design, sampling, protocol, preregistered analysis plan); Engineer (writes/runs/reports reproducible code via file + shell tools); Writer (drafts scientific documents from handoff packets); Auditor (independent fabrication audit before delivery).

## Results & Highlights

- **ALE cost advantage (Agents' Last Exam, 3 brain-science tasks: skull-stripping QC T1, ROI resampling T2, C. elegans neuron tracking T3)**: Across matched backbones with domain knowledge enabled, BrainPilot completed runs in 8–63% of the time and at 5–56% of the cost required by Codex or Claude Code. The cheapest configuration, BrainPilot + DeepSeek-V4-Pro, cost **$0.08** summed over the three tasks (17 min runtime), versus **$22.53** for Codex + GPT-5.5 (63 min).
- **ALE task scoring**: T2 (ROI resampling) scores were identical across every matched harness/backbone configuration; with GPT-5.5 both BrainPilot and Codex reached full credit (1.00) when domain knowledge was enabled. BrainPilot + DeepSeek-V4-Pro achieved 1.00/0.70/0.09 on T1/T2/T3 — i.e., strong on the two deterministic deliverables but near-zero on the open-ended tracking task.
- **BrainPilotBench-v0 (4 tasks, 56 cells, 53 valid primary scores)**: BrainPilot + Claude-Opus-4.8 tied the highest RSC place-cell score (1.00), achieved the highest BCI IV 2a score (0.20), and outperformed its matched Claude Code configuration on TOPS-fMRI (0.47 vs. 0.41) while scoring similarly on Sleep-EDF (0.65 vs. 0.66). Total cost for BrainPilot + DeepSeek-V4-Pro was **$0.504** vs. **$4.931** for Codex + GPT-5.5.
- **RSC case study**: Bayesian position decoder on held-out even trials reached MAE = 16.8 cm on a 90 cm track and true–decoded position correlation r = 0.646.
- **Mouse visual hierarchy case study (Allen Neuropixels)**: Receptive-field diameter vs. anatomical hierarchy score gave Spearman ρ = 0.714 (exact permutation p = 0.058, n=8 areas); flash-onset response latency ρ = 0.667 (p = 0.083); none reached conventional significance but all three were positive and direction-consistent with Siegle et al. 2021.
- **fMRI tonic-pain signature case study**: Held-out experimental tonic-pain prediction pooled Pearson r = 0.5791 (p = 2.76e-10), mean within-subject r = 0.6754; external chronic-back-pain patient-vs-control classification AUC = 0.793 (Japan) and 0.699 (UK).
- **BCI IV 2a case study**: MIAgentModel (EEGNet + TCN, 28.7k params) raised mean accuracy from 0.580 (EEGNet baseline) to 0.620 and Cohen's kappa from 0.440 to 0.493 across 9 subjects, but paired t-test p = 0.107 / Wilcoxon p = 0.129 — a positive trend, not a statistically significant improvement.
- **Sleep-EDF case study (20-fold LOSO)**: SleepAgentModel (237,813 params) reached 0.794 accuracy, 0.725 macro-F1, 0.724 balanced accuracy, 0.716 Cohen's kappa — exceeding class-prior baselines but slightly below published DeepSleepNet references.

## Critical Assessment

- **Novelty**: The paper's real novelty is not any single component but the integration for the brain-science domain — combining a PI/specialist agent hierarchy, a curated 7,233-item RAG corpus plus a 72-skill library, an append-only Graph of Trace, and an Auditor that performs claim-level fabrication checking. RAG, multi-agent orchestration (e.g., AI Scientist, Co-Scientist), and tool-augmented agents are individually well established; the domain-specific grounding and the explicit "auditable provenance + in-workflow fabrication audit" framing are the differentiators.
- **Rigor — weak / single-run baselines**: The headline ALE and BrainPilotBench results are explicitly "one unrepeated run per task and configuration," with reasoning backbones known to exhibit run-to-run variance; deltas as small as 0.04–0.06 between harnesses (e.g., TOPS-fMRI 0.47 vs. 0.41; BCI 0.20 vs. 0.11) cannot be distinguished from noise without repeats and confidence intervals.
- **Rigor — case studies**: The case-study statistics are honest but weak. The BCI IV 2a improvement is explicitly non-significant (paired t-test p = 0.107, Wilcoxon p = 0.129), the three mouse-visual-hierarchy correlations all fail conventional significance (p ≥ 0.058), and the fMRI pain biomarker relied on a single 80/20 split with seed 42, small clinical subgroups, weaker performance in the UK cohort, and no inferential or stability testing on individual FC weights.
- **Domain-knowledge ablation is mixed**: The authors transparently report that across 26 cells with both conditions valid, domain knowledge increased 8 scores but reduced 14 — undermining the implicit premise that the curated knowledge base is uniformly helpful. This is presented honestly but deserves stronger analysis of why it sometimes hurts.
- **Benchmark coverage**: Among ALE's >1,000 tasks, only 3 are neuroscience; BrainPilotBench-v0 ships with only 4 tasks. The authors frame this as a community call rather than a finished evaluation, which is fair but limits the strength of any SOTA claim.
- **Reproducibility**: Strong on paper — open-source code, frozen artifact contracts, content hashes, Oracle/empty-submission two-sided validation gates, no-network containers, and per-task graders with held-out labels. The main risk is that "single-run" numbers will not match on re-execution.
- **Significance**: High practical interest for the neuro-AI community; modest evidence that the architecture strictly beats strong general-purpose harnesses (Codex, Claude Code) beyond cost.
- **Red flags**: (a) Headline framing "comparable to state-of-the-art agent framework with less costs" is defensible on cost but the matched-backbone performance picture is mixed (BrainPilot + GLM-5.2 underperformed its Claude Code + GLM-5.2 alternative on all four BrainPilotBench tasks). (b) Cost numbers use official list pricing on a single run of a reasoning model, so the two-orders-of-magnitude gap is partly a function of the open-ended T3 trajectory dominating token use. (c) The vision-hierarchy case study relied on agent-cited anatomical scores from Harris et al. 2019 while the underlying values came from Siegle et al. 2021 — a citation error the authors themselves flag as "underscoring the need for human verification," which simultaneously demonstrates the Auditor's value and exposes its limits.

## Strengths

- Genuinely domain-grounded: the 7,233-item knowledge base and 72-skill library with concrete tool derivations (DeepLabCut, MNE-Python, fMRIPrep, SpikeInterface, pycortex, netneurotools) are a real artifact, not a placeholder.
- The Graph of Trace + Auditor combination directly targets the three failure modes the authors identify (fabrication, drift, no intervention points) and is a defensible design contribution for trustworthy scientific agents.
- Transparent reporting of negative/mixed evidence: domain-knowledge hurting 14 of 26 cells, non-significant case-study statistics, the citation error in the vision-hierarchy case, and the unreliability on T3 neuron tracking are all disclosed.
- Open-source release of both the system and the benchmark, with a well-engineered scoring harness (artifact contracts, hash sealing, containerized no-network execution, two-sided Oracle validation).
- Cost-efficiency on deterministic deliverables is striking where it holds: $0.08 vs. $22.53 for matched ALE deliverables is a meaningful operational result for budget-constrained labs.

## Limitations

- **Statistical strength**: Single-run evaluations on stochastic reasoning models; case-study metrics mostly fail conventional significance thresholds; no repeated-measure confidence intervals.
- **Open-ended tasks**: The C. elegans neuron-tracking task (T3) is the clearest weakness — most BrainPilot configurations either failed outright (F) or scored ≤ 0.18.
- **Benchmark breadth**: Only 3 ALE tasks and 4 BrainPilotBench-v0 tasks; the authors themselves call for community contribution, so coverage of brain-science subdomains is currently narrow.
- **Single-modality scope**: All five case studies operate within a single data modality; cross-modal integration (electrophysiology + imaging + behavior + molecular) is explicitly left as future work.
- **Domain-knowledge effect is non-monotone**: Enabling the curated KB/skills sometimes hurts scores, and the paper does not yet explain or diagnose when/why.
- **No true SOTA claim on modeling**: The Sleep-EDF and BCI models are described as below published DeepSleepNet references and not statistically better than the EEGNet baseline.
- **Cost numbers caveated**: Prices use official list pricing on single reasoning-model runs; absolute dollar figures may shift with provider pricing or sampling parameters.
- **Auditor scope**: The Auditor checks claim-vs-evidence-in-session, but broader methodological soundness, statistical validity, and scientific interpretation remain human responsibilities.

## Relevance

与你的多智能体系统、科研自动化（agentic research）、检索增强与可验证 AI 兴趣高度相关：BrainPilot 在一个对错误最敏感的领域（脑科学）展示了「知识库 + 技能库 + 可审计 Graph of Trace + 防造假 Auditor」的完整可信智能体设计，可作为通用科研 Agent 框架（含评估基准与评分合约工程实践）的直接参考与对比基线。

## Questions for Further Exploration

- How do BrainPilot's results change under repeated runs (e.g., 5–10 seeds) with reported means and confidence intervals, especially for the small deltas on TOPS-fMRI and BCI IV 2a?
- Why does enabling the curated domain knowledge reduce scores in 14 of 26 valid cells — is it retrieval noise, skill-loading overhead, or context dilution, and can a smarter skill-router fix it?
- Can the Auditor be extended beyond claim-vs-session-evidence checks to assess statistical validity, multiple-comparison correction, and preregistration compliance?
- How does BrainPilot perform on genuinely cross-modal questions (e.g., linking electrophysiology to behavior to molecular markers) where the highest-value neuroscience questions live?
- Does the cost advantage over Codex/Claude Code persist when the open-ended trajectory length is controlled (i.e., is the two-orders-of-magnitude gap mostly a T3 artifact)?
- How sensitive are BrainPilotBench-v0 scores to the artifact contract and grader choices, given that the benchmark currently has only four tasks?
- Could specialist agents instantiate nested subworkflows (deep-research, academic-writing pipelines) without breaking the append-only Graph of Trace guarantees?
