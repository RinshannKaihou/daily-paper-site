Runtime-certified per-head KV cache quantization recovers catastrophic retrieval failures while matching dense FP16 perplexity.

Runtime-Certified Bounded-Error Quantized Attention

Dean Calver 2026-05-20 LLaMA 3.1-8B · 8K–128K ctx
kv-cache-quantization runtime-verification llm-inference adaptive-precision certified-computing
Key Metric
5–10% → parity
NIAH retrieval: naive INT8/INT4 collapses to 5–10%; certified system matches dense baseline (McNemar p=1.0 at 8K/64K).
Why This Matters
Runtime error certification introduces a formal-guarantee paradigm for quantized LLM deployment, architecturally adaptable to future hardware and quantization schemes.
Critical Edge
Per-step bounds don't compose — accumulated value errors degrade value-sensitive tasks at short contexts (−6.9pp RULER at 8K). Certification is local, not end-to-end.

Methodology

Tiered storage. Tier-1 keeps INT8 keys (per-channel) + INT4 values (g=16) in VRAM — 56% of dense size. Tier-2 holds full FP16 originals in pinned system RAM over PCIe 5.0. A 2048-block FP16 scratch buffer in VRAM caches promoted blocks.

Two-term error decomposition. Output perturbation bounded by independent terms: Ekey (attention distribution distortion via Total Variation distance) and Eval (value reconstruction error, Σ ρb·ηb). Both computed online, per-head, per-step.

Adaptive top-K selector. INT8 Phase-1 scoring estimates block softmax masses. Promotes blocks in descending order until coverage ≥ τcov=0.995, capped at Kmax=128.

Four-rung fallback: K* expansion → FP16 value escalation → per-head dense recomputation → all-head dense via torch.scaled_dot_product_attention. Final rung is bit-identical to dense baseline.

Strengths

  • Principled formal bounds with proofs — not just empirical validation
  • Exceptionally transparent about limitations and negative findings
  • Naive-baseline comparison proves certification machinery is essential
  • Two-term decomposition enables independent key/value precision tuning
  • Ranking-consistency check converts silent failure into detectable, recoverable event

Limitations

  • 4.76× latency at 128K — ranking check alone is 28.3% of step time
  • No VRAM savings below 64K (scratch buffer exceeds dense KV)
  • System RAM cost equals dense KV (~17 GB at 128K)
  • Single model (LLaMA 3.1-8B), single GPU, batch size 1, greedy only
  • Value group-size cliff: g=16 works, g=32 fails catastrophically (+26.4 Δppl)

Questions for Further Exploration

  1. Can the ranking-consistency check be fused into the attend kernel to eliminate the 28.3% overhead?
  2. How does the system behave with different attention patterns (MoE, varying GQA ratios)?
  3. Can certification extend to FP8, 2-bit, or vector quantization with tight enough bounds?
  4. What is the overhead under multi-GPU tensor/pipeline parallelism?
  5. Could predictive prefetching of FP16 blocks reduce H2D page-in cost?
  6. Is there a path to end-to-end certification composing per-step bounds across layers?
  7. How does the attention-dilution effect interact with different architectures?
