Head-Aware Key-Value Compression for Efficient Autoregressive Image Generation

Liang, Zhang, Wen, Ye HIT Shenzhen & Peng Cheng Lab HeadKV kv-cache-compression autoregressive-generation
Key Finding

Attention heads in AR image generation models split into local (nearby tokens) and global (long-range dependencies) types — and this classification is stable across token positions for a given input. A single early-stage classification enables training-free, asymmetric KV cache budgets: sliding windows for local heads, stratified eviction for global heads. The result — retain only 1/6–1/8 of the cache with near-full-quality generation.

Key Metrics
3.68x speedup on Janus-Pro-7B at ρ=1/6 cache budget
52% GPU memory reduction (Janus-Pro-7B)
2.75 FID-50K on LlamaGen-XL at ρ=1/8 (vs. 2.95 LineAR)
89% head classification accuracy (τ=0.9 threshold)
0.74 GenEval Overall at ρ=1/6 (Full cache = 0.79)
Why This Matters

Generating 1024×1024 images with AR models demands >50 GB GPU memory. HeadKV slashes that bottleneck without any retraining, making high-resolution AR image generation practically deployable on consumer hardware.

1024×1024 generation = 50+ GB VRAM → HeadKV cuts ~52% with no quality collapse

Critical Edges
  • Grouping overhead slows Lumina-mGPT-768 below simpler LineAR baseline (1.35x vs. 1.65x)
  • Binary local/global split may be too coarse — continuous budget spectrum could capture nuanced behavior
  • No comparison against quantization-based KV cache methods
  • Irreversible token eviction means errors accumulate over very long sequences
Pipeline
STEP 01

Cache Prefilling

AR model encodes conditioning input and generates tokens, building KV cache without compression until a threshold (e.g. 100 tokens for Janus-Pro).

STEP 02

Head Grouping

Accumulate attention scores backward from current position until τ=0.9. If token count < local window → local head; else → global head. Done once per input.

STEP 03

Asymmetric Compression

Local heads: sliding window only. Global heads: stratified token eviction (STE) partitions tokens into near-range & long-range groups for balanced retention.
