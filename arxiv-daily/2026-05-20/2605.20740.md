> **一句话总结：** 本文提出 DAR（Distribution-Aware Reward），一种基于 CRPS 的强化学习目标函数，通过将 LLM 多次采样的输出视为经验预测分布并进行分布级优化，显著提升了 LLM 在回归任务中的排序相关性和不确定性校准质量。

> **TL;DR:** DAR replaces per-sample MSE reward in GRPO-style RL with a CRPS-based distribution-aware objective that evaluates multiple decoded rollouts as an empirical predictive distribution and assigns leave-one-out credit to each rollout. On code performance prediction (6-point Spearman gain on KBSS) and MoleculeNet molecular property prediction (competitive with graph-based models using only SMILES strings), DAR consistently outperforms SFT and pointwise RL baselines in both pointwise accuracy and rank correlation, while also improving uncertainty calibration and mitigating rollout diversity collapse during training.

## 核心贡献

- 提出 DAR，首个针对 LLM 回归的分布感知强化学习目标，将多次采样输出视为经验预测分布并用 CRPS 进行评估
- 设计了 leave-one-out 信用分配机制，根据每个 rollout 对整体分布质量的边际贡献分配奖励，既奖励准确性也奖励适当的分散性
- 理论上证明了逐点平方损失趋向均值坍缩而 CRPS 鼓励完整的条件分布，为方法提供了坚实的动机
- 实验证明 DAR 能有效缓解 GRPO 训练中的 rollout 多样性坍缩问题，保持更高的策略熵
- 在代码性能预测和分子性质预测任务上，DAR 使用仅 SMILES 字符串就能与图神经网络和 3D 分子模型竞争

## Key Contributions

- Introduces DAR (Distribution-Aware Reward), the first on-policy RL objective that directly optimizes the empirical predictive distribution formed by multiple LLM rollouts using Continuous Ranked Probability Score (CRPS)
- Designs a leave-one-out credit assignment scheme that measures each rollout's marginal contribution to the CRPS of the full rollout set, rewarding predictions that are both accurate and appropriately dispersed
- Provides theoretical motivation showing pointwise squared loss is mean-seeking (collapses to a point mass) while CRPS is distribution-seeking (optimized by the true conditional distribution)
- Demonstrates DAR mitigates rollout diversity collapse during GRPO training and maintains higher policy entropy compared to MSE reward shaping
- Achieves competitive results with graph-based and 3D molecular models on MoleculeNet using only SMILES strings, without chemistry-specific inductive biases

## Methodology

DAR builds on the GRPO (Group Relative Policy Optimization) reinforcement learning framework for LLM fine-tuning. The key innovation is replacing the per-sample reward with a distribution-level reward:

1. **Multi-rollout sampling**: For each input prompt, the policy generates K=12 rollouts, each decoded as a floating-point number using a strict `\boxed{}` output format.

2. **CRPS-based distribution evaluation**: The set of K predictions is treated as an empirical predictive distribution. The Continuous Ranked Probability Score (CRPS) evaluates this distribution against the scalar ground-truth target. CRPS is a strictly proper scoring rule that is minimized by the true conditional distribution, meaning it rewards both accuracy and appropriate spread.

3. **Leave-one-out credit assignment**: For each rollout k, DAR computes CRPS with and without that rollout. The difference quantifies rollout k's marginal contribution to distribution quality. This is converted to a relative reward within the GRPO framework (mean-subtracted, std-normalized), then used for policy gradient updates.

4. **Training**: Implemented in the verl framework with standard GRPO hyperparameters (4×H100 GPUs, batch size 256, lr 1e-6). The only change from standard GRPO is the reward computation module. Invalid rollouts receive the minimum batch reward as penalty.

5. **Evaluation**: At test time, 32 samples are generated and averaged as the final prediction. All experiments use 5 random seeds.

## Results & Highlights

- **KBSS latency prediction**: DAR achieves ρ=0.66 vs. 0.60 for MSE reward (6-point Spearman improvement), with MAE=0.011 vs. 0.012
- **APPS memory prediction**: DAR achieves ρ=0.96 vs. 0.92 for MSE reward, with MAE=29.1 vs. 29.8
- **MoleculeNet (FreeSolv)**: RMSE=2.360 vs. 2.561 (MSE reward) and 2.755 (SFT), statistically significant
- **MoleculeNet (Lipophilicity)**: RMSE=0.776 vs. 0.822 (MSE reward), statistically significant
- **MoleculeNet (ESOL)**: RMSE=0.871 vs. 0.878 (MSE reward), competitive but not statistically significant
- **Uncertainty calibration**: DAR shows the strongest log-scale correlation between predictive standard deviation and prediction error, indicating well-calibrated uncertainty estimates
- **Diversity preservation**: DAR maintains higher bracket rate and policy entropy during training compared to MSE reward shaping, confirming it mitigates diversity collapse
- **Reasoning ablation**: Explicit chain-of-thought reasoning does not improve regression performance and is associated with worse results in the fine-tuning setting

## Critical Assessment

**Novelty**: The core idea — using CRPS over multiple rollouts with leave-one-out credit assignment — is clean and well-motivated. However, the individual components (CRPS, leave-one-out estimators, GRPO) are all established. The novelty lies in their combination for LLM regression.

**Rigor**: Strong. Theoretical appendix proves the mean-seeking vs. distribution-seeking distinction formally. Proper scoring rules (CRPS, WIS) are reported in appendix. Statistical significance tested with paired bootstrap. Five random seeds per experiment.

**Reproducibility**: Good. Built on open-source verl framework. Detailed hyperparameters in Table 3. Uses publicly available datasets (MoleculeNet, KBSS). However, no code repository is mentioned in the paper.

**Significance**: The results are meaningful but incremental in magnitude. The 6-point Spearman gain on KBSS is the headline result, but some MoleculeNet improvements are modest (e.g., ESOL: 0.871 vs. 0.878). The approach is limited to scalar regression — extending to higher-dimensional outputs is acknowledged as future work.

**Red flags**: The base model (Qwen3-4B) is relatively small. Results on larger models (Qwen3-30B-A3B) are only reported for ICL and SFT, not for DAR, leaving open whether gains scale. The claim of competitiveness with graph-based molecular models is somewhat overstated — DAR's ESOL RMSE (0.871) is still notably worse than Uni-Mol (0.788). Reasoning mode degrading performance is an interesting negative result that warrants deeper investigation.

## Strengths

- Clean, theoretically motivated formulation that addresses a real gap (distribution quality vs. point estimates)
- Strong experimental coverage: synthetic controlled task + two real-world scientific domains
- Comprehensive analysis beyond aggregate metrics: uncertainty calibration, diversity dynamics, proper scoring rules
- Minimal overhead over standard GRPO — only reward computation changes
- Honest discussion of limitations relative to traditional regression methods

## Limitations

- Only evaluated on scalar regression tasks; extension to multi-output or structured prediction is unclear
- Results with larger models (>4B parameters) are incomplete — DAR results are not reported for Qwen3-30B
- LLM-based regression is computationally expensive compared to traditional methods (GPs, GNNs, XGBoost) for the same tasks
- Performance on MoleculeNet, while improved over LLM baselines, still lags behind dedicated molecular models (D-MPNN, Uni-Mol)
- Sensitivity to prompt format, sampling temperature, and output parsing introduces variability
- No guarantee of recovering the true conditional distribution — DAR is a practical heuristic that encourages well-centered and dispersed rollout sets

## Relevance

与用户在 LLM 训练稳定性方面的兴趣相关——DAR 解决了 GRPO 训练中 rollout 多样性坍缩的问题，属于 RL 微调训练动态的分析范畴。同时，其分布级优化的思想与模型输出可靠性评估有一定关联。

## Questions for Further Exploration

- How does DAR scale to larger models (7B, 14B, 70B)? Are the gains consistent or diminishing?
- Can DAR be combined with ensemble methods or Bayesian approaches for even better uncertainty estimates?
- Does the leave-one-out credit assignment introduce higher variance in gradient estimates compared to simpler reward schemes?
- What happens when the number of rollouts K is reduced (e.g., K=4 or K=6)? Is there a compute-quality tradeoff curve?
- Could DAR be extended to multi-target regression or structured scientific predictions (e.g., predicting full spectra)?
- How does DAR interact with other RL techniques like KL regularization, reward shaping, or curriculum learning?
