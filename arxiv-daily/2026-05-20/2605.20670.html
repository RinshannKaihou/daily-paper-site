LT2: Linear-Time Looped Transformers

Replaces quadratic softmax attention in Looped Transformers with linear and sparse attention variants. T loop iterations provably upgrade linear attention's rank-1 state update to rank-T and expand sparse attention's receptive field from O(w) to O(Tw)—all at linear-time cost.

Date 2026-05-20 Scale 1.3B params · 100B tokens Loops T = 4 arXiv 2605.20670
Key Finding
Looping synergizes with subquadratic attention in a non-obvious way: T iterations of a DPLR linear-attention block compose T rank-1 Householder-like updates into a cumulative rank-T state transition, which—by the Cartan-Dieudonné theorem—can span the full orthogonal group O(dk) when T ≥ dk. The LT2-Hybrid (Full+GDN) configuration at 1:4 interleaving surpasses the full-attention Looped Transformer by +2.1 avg. zero-shot points while running at ~5× higher decode throughput, proving that cheaper attention primitives can outperform quadratic attention inside a looped framework.

Why This Matters
Quadratic attention is the scalability bottleneck of looped transformers. LT2 eliminates it without quality loss, and the distillation path (Ouro → LT2 in ~1B tokens) makes adoption practical—no from-scratch pretraining needed. Linear-time variants also hold flat decode throughput from 1k to 32k context, where the baseline OOMs.
Critical Edge
The 100B-token budget is 4× below Chinchilla-optimal for 1.3B params, which may inflate the relative benefit of parameter sharing via looping. Residual connections also collapse the practical receptive field to ~1.5w (constant in T), undercutting the O(Tw) bound. Scaling past 1.3B is untested.

Methodology
  • Architecture: N-layer block shared across T=4 iterations. Multi-head attention replaced with subquadratic mixers (GDN, KDA, NSA, DSA). Zero-initialized learned gate ρτ per loop.
  • Linear attention (LT2-LA): DPLR-family recurrent state St = At·St−1 + βt·kt·vtT. Looping composes T rank-1 updates into rank-T cumulative operator via Householder products.
  • Sparse attention (LT2-SA): Sliding-window/NSA/DSA restrict each query to w keys. T loops expand combinatorial receptive field to O(T·w).
  • Hybrid design: 1:4 Full:GDN depth-level interleaving identified as optimal. Loop-level mixing also explored (Full→SWA-512→SWA-256→SWA-128).
  • SDPA output gate: Per-head sigmoid gate suppresses attention-sink compounding across loop iterations. Consistent improvement for all softmax-containing variants.
  • Distillation: 3-stage pipeline (MSE alignment → hybrid logit distillation with progressive per-loop KL → long-context extension). ~1B tokens total.

Limitations
  • Only evaluated at 0.6B and 1.3B with 100B tokens; scaling to 7B+ unknown
  • Fixed T=4 due to ACT training instability; limits compute-adaptive reasoning
  • Residual connections collapse effective receptive field, diminishing O(Tw) bound
  • No evaluation on code generation, multilingual, or multimodal tasks
