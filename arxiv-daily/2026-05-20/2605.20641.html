Key Finding

Trusted Weights, Treacherous Optimizations?

Compiler optimizations like torch.compile introduce systematic numerical deviations that can be weaponized as backdoor triggers. LoRA-injected attacks remain completely dormant under eager execution — passing all standard safety checks — but activate malicious behavior the moment the model is compiled. No compiler or hardware modification required.

Why This Matters

The Optimization Stack Is the Attack Surface

Every LLM deployment pipeline using torch.compile, TensorRT, or similar graph-optimization backends is potentially vulnerable. Standard eager-mode safety evaluations cannot detect these attacks — the model behaves perfectly until compilation flips its behavior. The compiler itself, long assumed to be a neutral optimization layer, must now be treated as part of the security perimeter. This also extends to safety alignment jailbreaks: the same LoRA adapter refuses harmful requests eagerly but complies when compiled.

Critical Edge
  • Only tested on 1B–3B parameter models; behavior at 70B+ or MoE is unknown
  • CUDAGraphs transfer drops to ~74% avg ASR — attack is somewhat backend-specific
  • CTB’s continuous trigger operates in embedding space, not discrete tokens — real-world delivery is constrained
  • Attack I requires per-target optimization, limiting scalability
  • Defense evaluation is preliminary; dual-backend validation lacks formal cost/failure analysis
  • Unknown interaction with quantization (INT8/INT4, GPTQ, AWQ)

Attack I — ISBS

Input-Specific Boundary Shaping. LoRA adapters on last N MLP layers push target inputs to the decision boundary. Compilation’s numerical shift flips the prediction. 100% ASR on targeted inputs.

Attack II — CTB

Compilation-Triggered Backdoor. Three-phase pipeline: profile critical divergent layers → inject bias before SiLU activation → fine-tune upper layers with 4-loss objective. Universal continuous trigger in embedding space.

Causal Analysis

FFN blocks contribute 70–80% of cumulative logit deviation from compilation. Attention layers contribute smaller but non-negligible shifts in deeper layers.

Proposed Defenses

  • Input perturbation
  • Batch size variation
  • Precision switching
  • Additional fine-tuning
