Frontier: Towards Comprehensive and Accurate LLM Inference Simulation

Feng, Tan, Deng, Jiang, Zhu, Xu CUHK / Anuttacon / StepFun 2026-05-20 ~70K LoC Python
LLM inference discrete-event simulation disaggregated serving performance modeling
Key Finding
Frontier is a discrete-event simulator that accurately models modern LLM serving architectures — co-location, Prefill-Decode Disaggregation (PDD), and Attention-FFN Disaggregation (AFD) — plus runtime optimizations like CUDA Graphs, speculative decoding, and prefix caching. By replacing monolithic-replica abstractions with role-specific cluster workers and replacing rough average-value proxies with per-operator predictors calibrated from real CUDA kernels, it reduces average end-to-end latency error from 44.9–51.7% (prior SOTA) to 2.6–6.4% on a 16×H800 testbed.

Why This Matters
  • Decision-grade simulation on a laptop — scales to 1K+ GPUs on commodity CPUs, enabling capacity planning without renting GPU fleets.
  • Catches wrong architectural choices — prior simulators conclude PDD always wins; Frontier reveals AFD outperforms PDD under tight TTFT constraints (116.2K vs 100.7K toks/s on 256×H800).
  • Unlocks new workflows — SLA-aware Pareto frontier exploration, heterogeneous GPU downgrade analysis, dynamic RL rollout reconfiguration (2.04× throughput gain).
  • No prior simulator can reproduce PDD, AFD, CUDA Graph, speculative decoding, prefix caching, and stateful reasoning workloads in a single framework.
Critical Edge
  • No public code release announced — reproducibility depends entirely on eventual open-sourcing.
  • vLLM-only calibration (v0.10.2); SGLang and TensorRT-LLM support is claimed but unevaluated.
  • AFD numbers validated against an in-house implementation, not a production AFD deployment — no public AFD system exists yet.
  • 16-GPU fidelity may not characterize 1K+ scale where network topology effects become dominant.
  • Random forest predictors require re-profiling for each new GPU type; adding Blackwell or future hardware is not zero-cost.
Architecture
  • Fidelity Plane — per-operator predictors (random forests for attention, load-balance stats for MoE GroupedGEMM) + profiled KV-cache budgets + pluggable network backends (ASTRA-Sim, HTSim).
  • Control Plane — Simulation Compiler emits role-specific workers with dependency edges (KV transfers, activation transfers, EP sync).
  • Execution Plane — vLLM-style scheduler + runtime adapters (CUDA Graph padding, MTP draft-verify-commit, prefix cache, chunked prefill) as first-class loop citizens.
  • Stateful requests — multi-round reasoning chains, tool-call delays, session-affinity requeue, prefix-cache continuity.
