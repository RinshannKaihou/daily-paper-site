> **一句话总结：** 本文系统分析了 Delta-Rule 线性 Transformer 中三角矩阵求逆的多种算法，提出混合递归算法 MXR，在 NPU 上实现 4.3× 内核加速和 2.5× 层级加速，同时通过严格的数值稳定性分析保证低精度浮点下的端到端模型精度不损失。

> **TL;DR:** A systematic study of direct and iterative triangular inversion algorithms for Delta-Rule linear attention (used in Qwen3.5/3.6, Kimi Linear, RWKV-7). The proposed MXR algorithm — which cleverly uses an unstable fast method (Cayley-Hamilton via repeated squaring) only for small constant-sized blocks, then switches to a log-stable recursive method — achieves up to 4.3× kernel speedup and 2.5× full-layer speedup on NPUs over SGLang baselines while maintaining full end-to-end MMLU accuracy. A key insight is that DeltaNet triangular matrices have condition number at most $n^2$, far better than random triangular matrices, which justifies using low-precision hardware.

## 核心贡献

- 系统分析了六类三角矩阵求逆算法（VCS、MCS、MBH、MCH、NS、IR）的数值稳定性、计算复杂度和硬件效率，建立统一的理论框架
- 证明 DeltaNet 矩阵的条件数至多为 $n^2$，远优于随机三角矩阵的指数增长条件数，为低精度计算提供了理论基础
- 提出 MXR（Mixed Recursion）算法：用不稳定的 MCH 方法处理小尺寸（如 16×16）对角块求逆作为初始值，再用 log-stable 的 MBH 递归算法完成剩余计算，巧妙平衡了速度与精度
- 在 NPU（华为昇腾）上实现 4.3× 内核加速，通过 mega-kernel 融合实现 2.5× 层级加速
- 全面评估了 float16/bfloat16/float32 三种精度下各算法的数值误差及对端到端 MMLU 精度的影响

## Key Contributions

- Systematic analysis of six families of triangular inversion algorithms (VCS, MCS, MBH, MCH, Newton-Schulz, Iterative Refinement) covering numerical stability, complexity, and hardware efficiency
- Proof that DeltaNet matrices have condition number bounded by $n^2$, providing theoretical justification for low-precision computation
- The MXR (Mixed Recursion) algorithm: uses the unstable Cayley-Hamilton/repeated-squaring method only for small constant-sized diagonal blocks (e.g., 16×16), then proceeds with the log-stable Bunch-Hopcroft recursive algorithm — achieving both speed and numerical reliability
- Up to 4.3× kernel-level speedup over SGLang's Triton implementation on NPUs, and 2.5× full-layer speedup via kernel fusion (mega-kernel)
- Comprehensive numerical accuracy evaluation across float16, bfloat16, and float32, with end-to-end MMLU benchmarks on Qwen3.5-0.8B, Qwen3.6-35B-A3B, and Qwen3-Next-80B-A3B

## Methodology

The paper organizes triangular inversion algorithms into two categories:

**Direct methods:**
- **VCS/MCS (Column Sweep):** Classic vectorized or matrix-product-based column sweep. Stable but requires $O(n)$ matrix multiplications.
- **MCH (Cayley-Hamilton):** Exploits that all eigenvalues of $I + L$ equal 1, giving $(I+L)^{-1} = I - L + L^2 - \cdots + (-L)^{n-1}$. Computed via repeated squaring in $\approx 2\log(n)$ matmuls. Fast but numerically unstable — intermediate values grow as $\sim 2^n / \sqrt{n}$, causing overflow in low precision.
- **MBH (Bunch-Hopcroft):** Recursive block inversion using the identity for partitioned matrices. Log-stable, $\approx 2\log(n)$ matmuls.
- **MXR (proposed):** Hybrid — uses MCH for small diagonal blocks (constant size, so errors stay bounded), then MBH for the remaining recursive steps. Optionally applies one step of iterative refinement.

**Iterative methods:**
- **Newton-Schulz (NS):** $X_{k+1} = X_k(2I - AX_k)$. Converges quadratically. Stability is conjectured (not formally proven). Requires $\approx 2\log(n)$ iterations, each with 2 matmuls.
- **Iterative Refinement (IR):** $R_k = I - \hat{A}_k^{-1}A$, then $\hat{A}_{k+1}^{-1} = \hat{A}_k^{-1} + R_k \hat{A}_k^{-1}$. Used as a post-processing step to boost accuracy.

All methods are evaluated on DeltaNet-style matrices $A = I + \text{strict\_tril}(KK^\top)$ with rows of $K$ sampled from the unit sphere, across chunk sizes $\{16, 32, 64, 128\}$.

## Results & Highlights

- **MXR achieves 4.3× kernel speedup** over SGLang's Triton-based BCH variant on NPUs (Ascend), and 3× over TileLang
- **Full-layer mega-kernel fusion** (MXR + all 5 other GDN kernels, chunk size 128) yields **2.5× speedup** vs. baseline
- **Numerical accuracy:** All stable methods (VCS, MCS, MBH, NS-12, MXR+IR) maintain Frobenius relative errors near machine precision in fp32; MXR alone has 2-3 fewer digits but is remediated by one IR step
- **Low precision:** float16 preferred over bfloat16 for triangular inversion (better mantissa precision). MCH returns NaN in float16 for sizes > 16
- **End-to-end MMLU:** Unstable MCH drops Qwen3.6 accuracy from 75.6% to 0.2%. All stable methods (including MXR+IR) maintain baseline accuracy within noise
- **Newton-Schulz convergence:** 6 iterations already gives reasonable accuracy (81.6% for 80B model), but 12 iterations needed for full stability; 14+ iterations give no further improvement
- The triangular inversion kernel accounts for ~40% of total GDN layer runtime, making it a critical optimization target

## Authors & Credibility

All authors are from the Computing Systems Lab at Huawei Technologies, Switzerland. The work has strong industrial relevance given Huawei's Ascend NPU platform. The analysis draws deeply from classical numerical linear algebra (Wilkinson, Higham, Bunch-Hopcroft, Pan-Reif) and connects it to modern LLM architectures. ## Critical Assessment

- **Novelty:** The MXR hybrid algorithm is a practical engineering contribution rather than a fundamentally new algorithm. The key theoretical insight (bounded condition number of DeltaNet matrices) was already known from [15]. The systematic stability analysis of all methods in this specific context is novel and valuable.
- **Rigor:** Strong. Formal stability proofs for each algorithm, with explicit precision-bit bounds. The admission that Newton-Schulz stability is "conjectured" rather than proven is intellectually honest. Experimental evaluation is thorough across multiple precision formats, matrix sizes, and models.
- **Reproducibility:** Good. Code is open-source. Hardware is specific (Ascend NPU), which limits direct reproducibility on other platforms, but the algorithmic analysis is platform-independent.
- **Significance:** High practical impact for anyone deploying Delta-Rule linear attention models (Qwen, Kimi, RWKV). The observation that inversion errors can catastrophically collapse model accuracy is an important practical warning.
- **Red flags:** Results are NPU-specific; GPU benchmarks are not provided. The 2.5× layer speedup relies on mega-kernel fusion that may not be straightforward on all hardware. The paper does not compare against GPU baselines, which limits the generalizability of performance claims.

## Strengths

- Comprehensive and systematic — covers 6+ algorithms with unified stability analysis
- Bridges classical numerical analysis and modern LLM kernel engineering
- Clever use of an unstable algorithm (MCH) as a controlled subroutine within a stable framework (MXR)
- Open-source implementations with public repositories
- Honest about limitations (Newton-Schulz stability unproven, MCH instability acknowledged)
- Practical recommendation: prefer float16 over bfloat16 for triangular inversion in linear transformers

## Limitations

- All performance benchmarks are on Huawei Ascend NPUs only — no GPU results
- Mega-kernel fusion (2.5× speedup) is hardware-specific engineering that may not port easily
- The Newton-Schulz stability proof remains an open question (conjectured but not formally proven)
- Only chunk sizes up to 128 are evaluated; behavior for larger chunks is unknown
- MMLU evaluation uses only 10 of 57 subjects as a proxy (acknowledged as sufficient for relative comparison)

## Relevance

本文直接命中"计算可靠性与精度错误"研究方向，深入分析了低精度浮点下三角矩阵求逆的数值稳定性问题，展示了不稳定的求逆算法如何导致 LLM 端到端精度崩溃（从 75.6% 到 0.2%），对理解线性注意力模型的数值可靠性具有重要参考价值。

## Questions for Further Exploration

- How do these algorithms perform on GPUs (NVIDIA/AMD) with Tensor Cores? Is MXR still the best choice, or does the hardware preference shift?
- Can the MXR approach be extended to other structured matrix inversions in linear attention (e.g., the gated diagonal circuits in RWKV-7)?
- What is the impact of triangular inversion errors on longer contexts (e.g., 128K tokens) where error accumulation might compound across many chunks?
- Is there a formal proof of Newton-Schulz stability for unit-diagonal triangular matrices, as conjectured?
- How does the bounded condition number property of DeltaNet matrices extend to other linear attention variants (Gated Linear Attention, DeltaFormer)?
