Distribution-Aware Reward: Reinforcement Learning over Predictive Distributions for LLM Regression

DAR replaces per-sample MSE reward with a CRPS-based distribution-aware objective, evaluating multiple decoded rollouts as an empirical predictive distribution with leave-one-out credit assignment.

J. Park · H. Chae · E. Mendes · J. DeYoung · V. Kishore · W. Xu · A. Ritter — Georgia Tech · U. Michigan · AWS

Key Finding
01

Treating multiple LLM rollouts as an empirical predictive distribution and optimizing it with CRPS — instead of rewarding each sample individually via MSE — yields gains in rank correlation, pointwise accuracy, and uncertainty calibration simultaneously, while preventing the diversity collapse that plagues standard GRPO training.

Why This Matters
  • Solves rollout diversity collapse in GRPO — higher policy entropy throughout training
  • Competitive with graph neural networks on MoleculeNet using only SMILES strings — no chemistry-specific inductive biases
  • Well-calibrated uncertainty: predictive std correlates with actual error
  • Minimal implementation overhead — only the reward computation module changes
Critical Edge
  • Only evaluated on scalar regression with Qwen3-4B; no DAR results on larger models
  • Still lags dedicated molecular models — ESOL RMSE 0.871 vs Uni-Mol 0.788
  • No public code repository mentioned
  • CoT reasoning degrades performance — a negative result needing deeper investigation
Methodology Pipeline
Step 01

Multi-Rollout Sampling

Generate K=12 rollouts per prompt, each decoded as a float via strict \boxed{} format.

Step 02

CRPS Evaluation

Treat K predictions as empirical distribution. Score against ground truth using Continuous Ranked Probability Score.

Step 03

Leave-One-Out Credit

For each rollout, compute CRPS with and without it. Marginal contribution = individual reward signal.

Step 04

GRPO Update

Relative reward (mean-subtracted, std-normalized) drives policy gradient. Invalid rollouts get minimum penalty.
