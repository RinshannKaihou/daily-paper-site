> **一句话总结：** FTerViT 首次将 Vision Transformer 的全部权重（包括 patch embedding、LayerNorm、分类头）三值化为 {-1, 0, +1}，通过知识蒸馏和两阶段训练策略，在 ImageNet-1K 上达到 82.43% top-1（6.09 MB，约15倍压缩），并首次在 ESP32-S3 微控制器上部署运行。

> **TL;DR:** FTerViT ternarizes *all* weight matrices and normalization parameters in a Vision Transformer to {-1, 0, +1}, including the notoriously sensitive patch embedding, LayerNorm, and classifier head. Using same-architecture knowledge distillation with a two-phase training protocol, FTerViT-DeiT-III-S384 reaches 82.43% ImageNet-1K top-1 at 6.09 MB (~15× compression, only -2.42 pp vs. FP32), outperforming prior ternary ViTs by up to 8 pp. The authors demonstrate the first-ever deployment of a ternary ViT on a $10 ESP32-S3 microcontroller.

## 核心贡献

- **首次实现 ViT 全组件三值化**：将 patch embedding、LayerNorm 仿射参数和分类头全部量化为 {-1, 0, +1}，突破了此前认为这些组件"不可三值化"的共识。
- **提出两个新型算子**：TernaryBitConv2d（带逐通道缩放的 patch embedding 卷积）和 TernaryLayerNorm（三值化归一化层），消除了 ViT 中最后的 FP32 瓶颈。
- **两阶段训练策略**：先用高学习率（1e-4）进行 KL 散度蒸馏训练至饱和，再用低学习率（1e-5）做 10 epoch 微调恢复，后者可带来 3-4 pp 精度提升。
- **极致压缩下的强精度**：DeiT-III-S384 在 6.09 MB 下达到 82.43% top-1，相比 FP32 仅下降 2.42 pp，比 TerViT 高 3.27-8 pp。
- **首次在 MCU 上部署三值 ViT**：在 ESP32-S3（$10，8 MB PSRAM）上部署 FTerViT-Small，实现 79.64% ImageNet top-1。

## Key Contributions

- First to ternarize *all* weight-carrying components in a ViT — patch embedding, LayerNorm affines, and classifier head — to {-1, 0, +1}, overcoming the well-documented sensitivity of these layers.
- Introduces two novel ternary operators: **TernaryBitConv2d** (per-channel scaled ternary convolution for patch embedding) and **TernaryLayerNorm** (ternary normalization affines), eliminating all FP32 storage bottlenecks.
- Proposes a simple yet effective **two-phase training protocol**: Phase 1 trains with KL distillation at LR 1e-4 until saturation (~76-78% top-1), then Phase 2 fine-tunes at LR 1e-5 for 10 epochs, recovering 3-4 pp. Prolonging Phase 1 beyond epoch 250 yields no benefit over Phase 2 recovery.
- Achieves **82.43% ImageNet-1K top-1 at 6.09 MB** (14.6× compression) with DeiT-III-S384, only -2.42 pp from FP32, outperforming all prior ternary ViTs by up to 8 pp.
- Demonstrates the **first deployment of a ternary ViT on a microcontroller** (ESP32-S3, dual-core Xtensa LX7 @ 240 MHz, $10), achieving 79.64% top-1 with a pure-C inference engine.

## Methodology

FTerViT replaces all weight-carrying components in a standard DeiT architecture with dedicated ternary modules:

1. **TernaryBitLinear** (from BitNet b1.58): Quantizes weight matrices to {-1, 0, +1} using absmean scaling (`sw = mean|W|`) and activations to 8-bit integers via per-token absmax scaling. Gradients flow through via straight-through estimator (STE).

2. **TernaryBitConv2d**: A novel ternary convolution operator with per-channel scaling for the patch embedding layer, designed to handle heterogeneous filter magnitudes across output channels.

3. **TernaryLayerNorm**: Ternarizes the learnable affine parameters γ (scale) and β (shift) in LayerNorm using the same absmean scheme. Per-token statistics (μ, σ²) and biases remain FP32 due to negligible parameter count.

**Training protocol** (two-phase, same-architecture KD):
- **Phase 1**: Student initialized from pretrained FP32 teacher (ternarized), trained with forward-KL distillation (T=1), LR 1e-4 with cosine decay over 250 epochs, AdamW optimizer, batch size 1024.
- **Phase 2**: Low-LR recovery fine-tuning from Phase 1 checkpoint, LR 1e-5 with cosine decay over 10 epochs, batch size 512. No CE loss is used — only KL — because CE conflicts with KL in low-bit networks.

## Results & Highlights

- **ImageNet-1K**: FTerViT-DeiT-III-S384 achieves **82.43% top-1** at 6.09 MB (-2.42 pp from FP32 84.85%). FTerViT-DeiT-III-S224 achieves **79.64%** at 5.81 MB (-3.44 pp). FTerViT-DeiT-S224 achieves **77.47%** at 5.81 MB.
- **vs. TerViT** (prior SOTA ternary ViT): FTerViT-DeiT-S is **+3.27 pp** higher (77.47% vs. 74.2%) at the same model size.
- **CIFAR-10/100**: FTerViT-DeiT-Tiny achieves 97.43% / 86.01% (within 0.09% / 0.53% of FP32) at only 1.53 MB.
- **Phase 2 effectiveness**: 10-epoch low-LR recovery adds 2.86 pp on DeiT-III-S224 (76.78% → 79.64%). Prolonging Phase 1 from 250 to 400 epochs only adds 1.58 pp before Phase 2 and reaches 79.61% after — worse than starting Phase 2 earlier.
- **Component fidelity**: Ternary weights are balanced across {-1, 0, +1} (~37% zeros). Patch embedding features have 0.88 cosine similarity with FP32. Classifier logits achieve Pearson r = 0.81 with FP32.
- **MCU deployment** (ESP32-S3): 21.06 s per inference, 2.83 MB peak PSRAM, 5.81 MB flash. Ternary packed inference is 1.71× faster and uses 55% less energy than FP32.
- **Scaling trend**: Accuracy gap to FP32 scales inversely with model capacity: 9.19 pp for DeiT-Tiny vs. 2.42 pp for DeiT-III-S384.

## Critical Assessment

- **Novelty**: High. Prior work (TerViT, ViT-1.58b) explicitly avoided ternarizing patch embedding, LayerNorm, and classifier head due to documented sensitivity (22.4 pp drop in TerViT). FTerViT solves this convincingly with same-architecture KD and novel operators.
- **Rigor**: Strong. Layer sensitivity analysis using two independent metrics (Taylor-FO, Hessian-trace) provides quantitative justification for why prior work struggled. Five-checkpoint Phase 2 ablation is thorough. Component-level fidelity analysis (cosine similarity, Pearson correlation) adds mechanistic insight.
- **Reproducibility**: Excellent. Public code, models, full hyperparameter tables, hardware setup details, and clear training recipes.
- **Significance**: Practically significant for edge/tinyML deployment. 82.43% at 6.09 MB is a compelling operating point, and the ESP32-S3 demo is a concrete feasibility proof. However, 21 seconds per inference on MCU limits use cases to non-real-time applications.
- **Red flags**:
  - The comparison partly benefits from using DeiT-III (a stronger FP32 backbone) vs. prior work using original DeiT. The authors acknowledge this but the headline numbers should be read with this context.
  - The two-phase training adds complexity (250 + 10 epochs total). The authors note it could potentially be unified but don't explore this.
  - Only evaluated on DeiT-Tiny/Small architectures. Scaling to larger models (DeiT-Base/Large) is claimed "straightforward" but not demonstrated.
  - W2A8 is not fully ternary — activations remain 8-bit. The title "Fully Ternary" refers to weights and normalization only.

## Strengths

- First convincing solution to full ternarization of all ViT components, a well-known open problem.
- Simple and effective training recipe (same-architecture KD + two-phase LR schedule) that doesn't require complex multi-objective losses.
- Strong empirical results with thorough ablations and real hardware validation.
- Excellent compression-to-accuracy trade-off: 15× compression with only -2.42 pp loss.
- TernaryLayerNorm and TernaryBitConv2d are reusable as standalone primitives.

## Limitations

- **Inference speed on MCU**: 21 seconds per forward pass is too slow for real-time applications. The C kernel uses basic bit-unpacking without SIMD optimization.
- **Activation quantization gap**: Activations remain at 8-bit (W2A8), so the method is not "fully ternary" in the full forward pass. Memory savings come only from weights.
- **Scalability untested**: Only DeiT-Tiny and DeiT-Small are evaluated. Whether the approach transfers to larger models or other architectures (Swin, etc.) remains open.
- **Training cost**: 260 total epochs is substantial compared to some PTQ methods that require no retraining.
- **Limited task scope**: Only image classification is evaluated. No detection, segmentation, or downstream transfer experiments.

## Relevance

FTerViT 提出了一种将 ViT 全组件三值化的方法，对在边缘设备上部署大模型具有直接参考价值。其知识蒸馏+两阶段训练的策略对低比特量化训练有启发意义，TernaryLayerNorm 和 TernaryBitConv2d 算子可迁移到其他架构的量化工作中。

## Questions for Further Exploration

- Can the two-phase training be unified into a single LR schedule (e.g., warmup + decay + micro-restart) without sacrificing accuracy?
- How does FTerViT transfer to downstream tasks (detection, segmentation) and other architectures (Swin, ConvNeXt)?
- What is the theoretical relationship between model capacity and ternarization gap — could this predict performance for larger models?
- Can activation quantization be pushed below 8-bit (e.g., W2A4) without collapsing the carefully recovered accuracy?
- Would SIMD-optimized kernels on ESP32-S3 reduce the 21s latency to a practically useful range (<1s)?
