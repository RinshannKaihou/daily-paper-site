05 Methodology · vary one factor at a time

MemDelta is explicitly a measurement protocol, not a new memory system. Its single design principle: hold the model, judge, questions, and input history fixed, then change one pipeline component per paired comparison — isolating retrieval quality, embedding choice, base-model behavior, and write-path cost.

Benchmark: LongMemEval-S, 500 questions across 6 types, each instance ~115K tokens (39–66 sessions). Seven strategies (S0→S3 above) are connected by six controlled pairwise contrasts. Each question runs a fixed 5-stage procedure (Reset → Ingest → Build context → Answer at temp 0 → GPT-4o-mini binary judge).

Statistics: accuracy with 95% bootstrap CIs (2,000 resamples) and McNemar's test on per-question correctness — so constant judge bias cancels in every pairwise delta. Write-path cost (wall-clock, LLM-call count, dollars) is reported as a first-class metric.

06 Strengths & limitations

Strengths

  • Clean "one variable at a time" design directly targeting the attribution problem in memory eval.
  • Statistically grounded: McNemar, bootstrap CIs, explicit p-values, first-class cost accounting.
  • Honest framing of null/weak results (p=1.0 = balanced disagreements, not equivalence).
  • Immediately adoptable 6-point evaluation checklist + open-source release.
  • Tangible worked examples (Spotify playlist, commute, coupon) make abstract confounds concrete.

Limitations

  • Cost-matched claim covers only 88 instances / 2 of 6 types; temporal & knowledge-update untested.
  • Synthetic LongMemEval-S dialogues likely favor verbatim retrieval; real-user transfer untested.
  • Imperfect embedding control (text-embedding-3-small vs ada-002) in the write-path comparison.
  • GPT-4o-mini judges all models; no human spot-checks.
  • Sonnet refusal behavior is model-/version-specific; single author; not an additive causal model.
07 Open questions for further work
  1. Do findings transfer to real (non-synthetic) dialogues, where lower lexical overlap might favor extraction memory?
  2. Would running Mem0 on the temporal & knowledge-update subsets change the cost-matched verdict?
  3. Does a multiple-comparison correction across the six pairwise tests preserve significance?
  4. How stable is the Sonnet refusal phenomenon across API versions and providers?
  5. Can the protocol extend to multi-hop, agentic, and graph-based memory (Graphiti, A-MEM)?
  6. What is the full accuracy-vs-write-cost Pareto frontier across more embedders/retrievers/rerankers?
  7. Is there a principled additive vs. interactive decomposition of the four confounds?
