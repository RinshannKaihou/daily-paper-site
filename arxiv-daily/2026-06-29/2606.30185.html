01 Methodology

Capability evolution, not fine-tuning

Dynamo freezes the model's weights (∇θ=0) and instead grows a library C = (S, T): structured skills (Markdown SOPs for when & how to reason) and executable tools (≤150-line Python programs that transform images via OpenCV / PIL / NumPy). It learns this library from just a 10% labeled slice of the target task.

A three-phase loop, repeated N = 3 times:

  • Diagnose — runs each case, then inspects the image alongside the trace to classify the failure as cognitive (reasoning weak, evidence visible) or perceptual (image must be transformed).
  • Explore — proposes M candidate skill+tool pairs conditioned on the diagnosis.
  • Validate & Promote — keeps a candidate only if it strictly beats the current library: a monotonic guard against regression.

Every tool ships with a paired mastery skill that gates when it fires — fixing the indiscriminate tool-spam of prior auto-tool methods (CREATOR, LATM). A rolling-window detector re-triggers the loop when the dominant task family φ̂t flips, keeping the library in sync with streaming traffic.

Strengths

What it gets right

  • Training-free & weight-frozen — no 10⁴–10⁵ rollouts, no 80–240 gradient updates.
  • Principled cognitive / perceptual taxonomy with clean empirical alignment across benchmarks.
  • Monotonic promotion guard prevents capability regression; BM25 retrieval + skill-merging stop library bloat.
  • Mastery skills make tool invocation selective — a real fix for prior auto-generated tool methods.
  • Additive with RL (Dynamo+RL strongest everywhere) — not an either/or story.
  • Honest about limits; explicitly disclaims fully unsupervised online adaptation.
Limitations

Where it strains

  • Diagnosis is bounded by the backbone: Analyzer & Generator are the same frozen VLM — weak introspection can propagate mis-classified bottlenecks; the guard rejects under-performers but can't fix upstream errors.
  • Scope is image VLMs on 4 benchmarks + GTA + RL splits — no 3D / medical imaging; cross-modality transfer unverified.
  • Relies on a per-case correctness signal (ground-truth or verifier) — not truly unsupervised.
  • Selection bias over candidate set M is only bounded logarithmically; empirical estimate deferred.
  • Proprietary-model dependency (GPT-5.4, Doubao-Seed-2.0) reduces reproducibility of several headline numbers.
03 Open questions for further exploration
How sensitive is performance to the 10% subset size and seed variance on the small benchmarks (V*=191, HRBench4K=800)?
Can the per-case correctness signal be replaced by an LLM verifier in production without degrading gap-recovery?
Does the cognitive/perceptual rule transfer to 3D spatial reasoning or medical imaging, where “perceptual bottleneck” means something different?
How does Dynamo behave on weaker non-Qwen 3B–7B backbones, where self-introspection is the binding constraint?
Can the exploration budget M and window length W be tuned adaptively rather than fixed?
Does Dynamo+RL additivity hold, or saturate, as model scale grows?
