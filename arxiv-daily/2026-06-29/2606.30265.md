> **一句话总结：** 本文为投机解码中实际常用的确定性接受准则（严格贪婪、加性/乘性松弛、top-m、熵阈值、树解码）建立统一理论：将拒绝区域刻画为目标分布的下水平集，推导出精确的 KL 散度证书和基于"边界（margin）"的尖锐界，并在 Qwen3 上验证松弛与树准则（尤其低置信步）显著扩大可证明接受区间。

> **TL;DR:** This paper builds a unified theory of *deterministic* acceptance in speculative decoding (strict greedy, additive/multiplicative relaxed, top-m, entropy-thresholded, tree-based). It shows rejection regions are lower level sets of the target distribution, derives exact KL-divergence certificates and sharp margin-based bounds (including a universal `log(2)` ceiling for single-token and `log(m+1)` for trees), and validates on Qwen3 that relaxed and tree criteria substantially enlarge the certified acceptance region, especially on low-margin decoding steps.

## 核心贡献

- **统一的下水平集框架**：识别出严格贪婪、加性/乘性松弛、top-m 松弛、熵阈值（Medusa/Hydra 风格）等准则的拒绝区域，都可以表示为目标分布 p 的下水平集（lower level sets），从而用一个定理统一刻画。
- **精确 KL 证书**：推导出导致拒绝所需的最小 KL 散度的闭式精确解 `Rp(R)`，给出当 `KL(p‖q) < Rp(R)` 时保证接受的确切证书；并用构造性例子证明上下界是紧的。
- **可解释的 margin 界**：将精确证书松弛为两 token 问题，得到仅依赖"margin"（top-1 与次大概率之差 γp）的尖锐渐近界 `~2√ε`，并证明这是 Pinsker 型界的最优阶（仅差 O(ε³ᐟ²)）。
- **树解码推广**：将框架推广到分支因子为 m 的对称贪婪树，给出精确证书、基于 m 阶 margin 的界（`~4ε/(m+1)`，即相对单 token 改善 `√((m+1)/2)`），以及通用上界 `log(m+1)`（推广单 token 的 `log(2)`）。
- **实证验证**：在 Qwen3-1.7B / Qwen3-4B（UltraChat 200k，500 个提示，128 步，>63000 步采样）上估计目标分布，证明松弛与树准则主要在"低 margin 长尾"上提升证书大小；树 m=2 几乎将中位证书翻倍。

## Key Contributions

- **Unified level-set framework**: identifies that the rejection regions of strict greedy, additive/multiplicative relaxation, top-m relaxation, and entropy-thresholded (Medusa/Hydra) criteria are all *lower level sets* of the target distribution p, unifying them under a single theorem.
- **Exact KL certificates**: derives a closed-form minimum KL divergence `Rp(R)` that permits rejection, so `KL(p‖q) < Rp(R)` guarantees acceptance; constructs examples proving the lower/upper bounds are tight.
- **Interpretable margin bounds**: relaxes to a two-token problem to obtain sharp asymptotic bounds depending only on the margin γp = p(x0) − p(x1), recovering `~2√ε` and proving it is the optimal order of a Pinsker-style guarantee (within O(ε³ᐟ²)).
- **Extension to tree decoding**: generalizes to symmetric greedy trees with branching factor m — exact certificate, an m-th order margin bound `~4ε/(m+1)` (a `√((m+1)/2)` improvement over single-token), and a universal `log(m+1)` ceiling extending the `log(2)` single-token ceiling.
- **Empirical validation**: estimates target distributions on Qwen3-1.7B/4B (UltraChat 200k, 500 prompts, 128 steps, >63k sampled steps), showing relaxed and tree criteria enlarge the certified region primarily on the *low-margin long tail*; tree m=2 roughly doubles the median certificate.

## Methodology

**Problem setting.** Speculative decoding uses a fast drafter q to propose tokens verified by a target model p. Existing theory studies the *distribution-preserving* (stochastic, unbiased, optimal-transport) setting. This paper instead studies *deterministic local* acceptance: when is a draft token accepted under greedy, relaxed, or tree-based rules, as a function of `KL(p‖q)`?

**Core idea — rejection regions as lower level sets.** For a fixed decoding step, let x0 = argmax p and define a rejection region `R ⊆ V\{x0}` (draft rejected iff its argmax mode falls in R). For all studied criteria, R has the form `{v : p(v) ≤ c}` for some threshold c — a lower level set of p. The quantity of interest is the exact certificate:

```
Rp(R) := inf { KL(p‖q) : argmax q ∩ R ≠ ∅ }
```

so `KL(p‖q) < Rp(R)` guarantees acceptance under any q. Drafters are typically trained with a KL objective, making `KL(p‖q) ≤ ε` a natural assumption.

**Exact certificate (Theorem 4.4).** Via three lemmas — reduction to a single-token problem M(x1) (4.1, a convex program solved by KKT), the closed-form single-token minimizer (4.2), and a swap lemma showing the minimizer over R is its most-probable element x* (4.3) — the exact certificate is:

```
Rp(R) = s · KL( p|_A / s ‖ Unif(A) )
```

where A is the "active set" and s the mass it carries. This one result then specializes to each criterion as a corollary.

**Lower bound (Lemma 4.5).** Relaxing to a two-token problem yields `Rp(R) ≥ G(p(x0), c) = (p(x0)+c)·g((p(x0)−c)/(p(x0)+c))`, where `g(d) = ½(1+d)log(1+d) + ½(1−d)log(1−d)`. This is sharp in the worst case and depends only on the margin.

**Universal upper bound (Theorem 4.6).** A convexity/polytope-extreme-point argument gives `Rp(R) ≤ (2/k) log 2 ≤ log 2` (k = |A| ≥ 2), sharp in the limit k=2. **No single-token criterion can guarantee acceptance for any q with `KL(p‖q) > log 2`.** This `log(2)` ceiling is a key structural finding.

**Per-criterion corollaries.** Strict greedy: exact `M*(p)=G(p(x0),p(x1))`, and Pinsker gives sufficient `γp > √(2ε)`; Theorem 4.9 proves this rate optimal up to `O(ε³ᐟ²)`. Additive (margin t): threshold c = p(x0)−t, certificate `~γ²/(2(p(x0)+c))`. Multiplicative (factor α): threshold c = α·p(x0), bound `p(x0)·[(1−α)²/4 + (1−α)³/8 + O(1−α)]` — more conservative on low-confidence steps, more relaxed on confident ones. Top-m + relaxation: mixed threshold `max(αp(x0), p(x[m]))`. Entropy (Medusa): absolute threshold θ_H = min(ε0, δ0 e^{−H(p)}).

**Tree decoding (Theorems 5.1–5.3).** Drafter proposes top-m tokens; target accepts the level iff x0 ∈ top_m(q), i.e., rejection when ≥ m tokens beat q(x0). Exact certificate has the same `s·KL(p|_A/s ‖ Unif(A))` form but with active set rearranged from the *least* probable of the top-m+1 tokens. Using the **Donsker–Varadhan** variational representation, a clean lower bound `B_m(γ_m)` is derived, asymptotically `((m+1)/4)γ_m²`, giving acceptance when `γ_m > √(4ε/(m+1))` — a `√((m+1)/2)` gain over single-token. Universal ceiling becomes `log(m+1)`.

**Empirical study.** Estimates target distributions of Qwen3-1.7B and Qwen3-4B on UltraChat 200k (500 random prompts ≥64 tokens, 128-step generations, >63k steps). Analyzes top-k cumulative mass, exact-certificate distributions per criterion, threshold sensitivity, and certifiable acceptance lengths.

## Results & Highlights

- **Models are usually very confident**: mean top-1 prob ≈0.85 (1.7B) / 0.87 (4B); median ≈0.97 / 0.99. Top-3 cumulative mass median rounds to 1. But a **long tail of low-margin steps** exists (p5 top-1 ≈0.34/0.39) — exactly where relaxed/tree criteria help.
- **Exact KL certificates (Table 2)**, e.g. on Qwen3-4B (Mean / Median / p5):
  - Strict greedy: 0.468 / 0.619 / 0.007
  - Additive t=0.3: 0.505 / 0.625 / 0.107
  - Multiplicative α=0.1: 0.573 / 0.643 / 0.296
  - Medusa (ε0=0.1, δ0=0.09): 0.606 / 0.664 / **0.378** — dramatically lifts the tail because the entropy threshold makes the margin γ_H large even on uncertain steps.
  - Tree m=2: 0.820 / 1.036 / 0.099; Tree m=8: 1.814 / 2.126 / 0.663 — nearly **doubling the median** and lifting the low tail substantially.
- **`log(2)` ceiling empirically confirmed**: a sharp drop in the certificate histogram at log(2)≈0.69 matches the many highly-confident steps (Theorem 4.6). Trees show the analogous `log(m+1)` ceiling.
- **Threshold sensitivity (Table 3)**: t < 0.1 and α > 0.5 have negligible effect; gains concentrate for larger t / smaller α. For top-m + additive, small t is dominated by the additive branch, larger m lets top-m gate dominate.
- **Certifiable acceptance lengths (Tables 5–6)**: in realistic ε regimes, single-token criteria give acceptance runs of ~2–5 tokens — consistent with published speculative-decoding numbers. Relaxed schemes can accept almost every draft but risk quality degradation at large ε (e.g., Medusa at ε≈0.3). Tree-based acceptance increases certified lengths *without* quality degradation (it remains strict greedy per token), at higher compute cost.

## Critical Assessment

- **Novelty — High.** Targets a genuine gap: virtually all prior theory (Leviathan 2023; Chen 2023; Sun/Ahn SpecTr; Yin 2024) concerns the *distribution-preserving* regime. Characterizing *deterministic local* acceptance via lower level sets and KL certificates is a fresh, useful abstraction that unifies many practical acceptance rules under one theorem.
- **Rigor — High.** Claims are formal: an exact certificate theorem with closed form, separate lower (two-token relaxation) and upper (polytope/extreme-point) bounds, and explicit examples showing each bound is sharp. Asymptotic expansions (e.g., `g⁻¹(ε) = √(2ε)(1 − ε/6 + O(ε²))`) are spelled out. The `log(2)` / `log(m+1)` ceilings are clean structural results.
- **Reproducibility — Moderate-to-High.** Theory is fully specified; empirical setup is concrete (named models, dataset, sample sizes, quantile tables). However, no explicit statement of code release is visible in the read portion, and the empirical analysis is distributional (no actual drafter training/inference runs — certificates are computed from target distributions under assumed KL bounds).
- **Significance — Medium-High.** Gives practitioners concrete, interpretable levers (margins, thresholds) and explains *why/where* relaxed and tree criteria help (low-margin steps). The `log(2)` ceiling is a memorable takeaway. Directly actionable for inference-engine designers.
- **Red flags / caveats:**
  - **Single author** with no institutional affiliation beyond "Independent Researcher" — warrants careful review of the proofs, though they appear well-formed.
  - **No end-to-end speedup measurement**: acceptance is studied locally; certifiable lengths are computed assuming a uniform KL bound, not measured speedups or real drafters. The link from "certificate > ε" to real throughput is indirect.
  - **Empirical scope is narrow**: two Qwen3 models, one dataset, 500 prompts. Generalization to other families (Llama, Mistral) and tasks (code, reasoning) is unverified.
  - Worst-case tie-breaking convention is pessimistic; real drafters may behave better than certificates suggest, so the framework is conservative.

## Strengths

- Elegant unification: one exact-certificate theorem yields five practical criteria as corollaries.
- Both *exact* and *interpretable (margin-only)* results, with proven tightness.
- Clean universal ceilings (`log 2`, `log(m+1)`) that are also empirically visible.
- Directly maps theory to real systems (TensorRT-LLM multiplicative margin, Medusa entropy, tree methods).
- Honest about quality/compute trade-offs of relaxed vs. tree criteria.

## Limitations

- No actual drafter training or latency/throughput experiments; all empirical results are on target distributions with assumed KL bounds.
- Limited model/dataset diversity (Qwen3 only).
- Conservative worst-case tie-breaking may understate practical acceptance.
- Tree analysis restricted to the simplest *symmetric* top-m tree; richer tree topologies (SEQUOIA/EAGLE-2 schedules, variable branching) are not covered.
- Single-token certificates capped at log 2 means for any criterion of this class, beyond that KL no acceptance guarantee exists — an inherent, not paper-specific, ceiling, but worth flagging for practitioners.

## Relevance

对关注 LLM 推理加速、投机解码（speculative decoding）与系统优化的研究者/工程师高度相关：它首次为实际部署中常见的贪婪/松弛/树解码提供可证明的接受条件与 KL 证书，将"何时草稿会被接受"从经验直觉提升为可量化、可调参（margin、threshold、branching）的理论工具。

## Questions for Further Exploration

- How do the margin-based bounds compare to exact certificates across model families beyond Qwen3 (Llama-3, Mistral, DeepSeek) and across tasks (code generation, math reasoning, long-context)?
- Can the framework predict *real* empirical acceptance rates and wall-clock speedups when an actual trained drafter (EAGLE, DFlash) is used, rather than assumed KL bounds?
- Does the pessimistic worst-case tie-breaking materially underestimate acceptance for typical drafters, and can a "typical-case" certificate be derived?
- How should the tree analysis extend to non-symmetric / learned tree schedules (EAGLE-2, SEQUOIA) to yield certificates for realistic tree topologies?
- Given relaxed criteria can accept nearly every draft but risk quality drift, what is the optimal ε/threshold that maximizes speedup subject to a bounded output-distribution deviation?
- Could the Donsker–Varadhan technique used for trees yield certificates for other multi-candidate verification structures (e.g., batch/bitmap verification)?
