The first exactly additive layer-attribution in probability space — per-layer contributions telescope to Δp = p(final) − p(baseline).

01

How IG-Lens works

IG-Lens fixes a position i and target token yₜ, then defines a head-once readout f(h) = softmax(head(norm(h)))_{yₜ} — the final LayerNorm and unembedding applied once to a single detached hidden vector. Crucially f is pointwise: it contains no attention and no MLP, so all cross-token mixing already happened in the forward pass.

Choose checkpoint layers 0=L₀<L₁<…<Lₖ=n and a baseline (per-position mean). Each layer's contribution is the segment Integrated Gradient of f along the straight line between consecutive checkpoints. By the gradient theorem, every segment integral equals its endpoint difference — so the whole sum telescopes to f(final) − f(base), exactly, in probability space.

The default prediction-aware estimator (IDGI-inspired) credits each integration step its observed output change rather than a raw gradient. For a scalar readout this collapses to a telescoping sum of endpoints — so completeness holds at any grid resolution m, and steps with no output change contribute nothing. One batched forward over all grid points recovers the full token×layer map with no backward pass.

The five-property claim

  1. Probability space — softmax kept in the path, not linearized.
  2. Additive — per-layer values sum to a meaningful total.
  3. Sum = Δp — telescopes to the exact probability change.
  4. Norm in the loop — LayerNorm inside the integration.
  5. Training-free — no lenses fitted, no backward call.

Trade-off

Exactness measures a conditional marginal — what norm+head reads out given the previous layer — deliberately discarding a layer's effect routed through upper blocks.

03

Strengths vs. Limitations

Strengths

  • Fills a real gap. Exact, additive, probability-space attribution with softmax/LayerNorm inside the path — no prior method combines these.
  • Completeness to floating point at any step count is a strong, cleanly-proven property the numerical checks confirm.
  • No training, no backward pass — cheap, practical, easy to bolt onto any decoder-only model.
  • Unusually honest self-assessment — separates "what it is" from "what it isn't" and designs validation to be falsifiable.
  • Signed decomposition exposes non-monotonicity (negative IG_L = a layer that lowers target probability), which level-reading lenses hide.

Limitations

  • No empirical evaluation. The Evaluation Plan (difficulty correlation + causal ablation) is proposed, not executed; all "results" are consistency checks.
  • Conditional marginal, not total effect. Discards a layer's effect routed through upper blocks; users wanting total causal effect should use DLA.
  • Layer-set dependence. Attribution values and onset shift with the chosen layer grid.
  • Scale / generalization untested — only Llama-3.2-1B-Instruct.
  • Memory cost from the N×V logit tensor (≈128k vocab), mitigated by chunking.
  • Onset definition is "preliminary"; only finalized as a signed rule for the un-run evaluation.
04

Open questions to test

  1. Does IG-Lens onset predict the ablation-sensitive layer? The central untested claim — run the §4 causal-ablation protocol and compare against logit-space / lens-crossing onsets, especially on non-monotone (saturated, hard) tokens.
  2. How does the conditional-marginal view diverge from total-effect (DLA) conclusions? For which tokens and questions does the divergence matter most?
  3. Sensitivity to baseline and layer set? How stable is L⋆ to baseline choice (per-position mean vs. zero vs. mean-over-data) and to layer-grid resolution?
  4. Does it scale? Do step-count-independent completeness and onset behavior hold from 7B to 100B+ and across architectures (attention-free / hybrid)?
  5. Can a total-effect variant — sacrificing exactness — be a useful companion? As the author suggests; and can its bias be bounded?
