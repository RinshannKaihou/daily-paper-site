When Is a Draft Accepted? A Theory of Acceptance in Speculative Decoding ↗

Key metric · universal ceiling

log 2≈0.693

The hard KL ceiling: no single-token acceptance rule — greedy, relaxed, top-m, or entropy — can guarantee a draft once KL(p‖q) exceeds it. Empirically visible as a sharp histogram drop.

Trees raise the ceiling to log(m+1) — and tree m=2 nearly doubles the median certificate on Qwen3-4B.

Why this matters

The first provable acceptance conditions for deployed greedy, relaxed, and tree decoding — turning “when is a draft accepted?” from intuition into tunable, quantifiable theory (margin, threshold, branching).

Critical edge

No end-to-end speedup is measured — every certificate assumes a uniform KL bound, never real drafters or wall-clock throughput. The link from “certificate > ε” to actual latency is indirect and unproven.

Local theory, not a deployed win
01

Methodology, distilled

Speculative decoding pairs a fast drafter q with a slower target p. Prior theory studied the distribution-preserving (stochastic) regime; this work instead tackles deterministic local acceptance — when is a proposed token actually kept?

The core abstraction: every common rule defines its rejection region as a lower level set {v : p(v) ≤ c} of the target distribution. The exact acceptance certificate is then the smallest KL that could force a rejection:

R_p(R) := min { KL(p‖q) : argmax q ∩ R ≠ ∅ } → KL(p‖q) < R_p(R) ⟹ accepted

A closed-form solution (reduced to a single-token convex program, solved by KKT) specializes to each rule as a corollary. Relaxing to a two-token problem yields a sharp margin-only bound of ~2√ε, proven optimal up to O(ε³ᐟ²). A universal log 2 ceiling follows; symmetric greedy trees lift it to log(m+1) via the Donsker–Varadhan variational representation.

Five rules, one theorem

Each is a lower level set of p — a corollary of the exact certificate.

Strict greedy c = p(x₁)
Reject unless draft = argmax p. Sufficient margin γ_p > √(2ε).
Additive relaxation c = p(x₀)−t
Accept if margin exceeds t. Gains concentrate at larger t.
Multiplicative c = α·p(x₀)
TensorRT-LLM-style factor α; loose on confident steps, tight on weak ones.
Top-m max(αp(x₀), p(x[m]))
Rank gate blended with relaxation.
Medusa / entropy c = min(ε₀, δ₀e^{−H(p)})
Threshold shrinks with uncertainty — lifts the low-confidence tail most.

03

Strengths & limitations

▲ Strengths

What lands cleanly

  • One theorem, five rules. The exact-certificate result yields greedy, additive, multiplicative, top-m and entropy criteria as corollaries.
  • Both exact and interpretable. Margin-only bounds recover ~2√ε with proven tightness — not just sufficiency.
  • Memorable ceilings. log 2 and log(m+1) are clean structural results, also empirically visible.
  • Theory ↔ systems. Maps directly to TensorRT-LLM multiplicative margin, Medusa entropy, and tree methods.
  • Honest trade-offs. Explicit about the quality-vs-compute cost of relaxed vs. tree criteria.

▼ Limitations

Where the claim out-runs the evidence

  • No real drafter, no speedup. All results are target-distribution certificates under assumed KL bounds — zero latency/throughput runs.
  • Narrow empirical scope. Qwen3 only, one chat dataset, 500 prompts. Llama / Mistral and code & reasoning tasks unverified.
  • Conservative tie-breaking. Worst-case convention likely understates acceptance for typical drafters.
  • Simplest trees only. Symmetric top-m; richer SEQUOIA / EAGLE-2 schedules and variable branching not covered.
  • Inherent single-token cap. Past log 2 no rule of this class gives any guarantee — structural, but a real practitioner ceiling.
04

Open questions

Do the margin bounds survive beyond Qwen3 — across Llama-3, Mistral, DeepSeek, and across code / math / long-context tasks?
Can the framework predict real wall-clock speedups from a trained drafter (EAGLE, DFlash) rather than assumed KL bounds?
Does pessimistic worst-case tie-breaking materially understate acceptance — and can a typical-case certificate be derived?
How should tree analysis extend to non-symmetric / learned schedules (EAGLE-2, SEQUOIA) for realistic topologies?
If relaxed rules accept nearly every draft but risk drift, what ε/threshold maximizes speedup under bounded output deviation?
Could the Donsker–Varadhan tree technique certify other multi-candidate verification (batch / bitmap verification)?
