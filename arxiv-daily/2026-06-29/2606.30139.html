Relevance Is Not Permission: Warranted Attention for Value Contributions · Minwoo Yu, Young-guk Ha et al. · Konkuk Univ. · 2026-06-29 · arXiv:2606.30139

Key Finding

Attention relevance ≠ prediction permission. Warrant rewrites the metric-path value term α·v → α·g·v with a learned leak-sigmoid permission gate g, so a model may read an item without admitting its value as prediction evidence.

Key Metric
27/32
paired comparisons where the primary metric improves, across 5 task families.

Why This Matters

A lightweight, model-agnostic plugin that lets any attention model selectively gate prediction evidence — cheaply lifting temporal-graph, retrieval, spatio-temporal, and point-process baselines without re-architecting them.

⚠ Critical Edge

The 27/32 headline mixes substantial and negligible effects: MTPP (+0.0002) and RAG (+0.0010) gains sit near noise, and "path-localization" is partly circular — the right path is defined as the one leading to the metric.

1Methodology — distilled

Standard attention folds two distinct decisions into one scalar: α_ij decides both what to read and whether a read item's value becomes prediction evidence. Warrant splits them. It keeps α_ij as a relevance estimate but multiplies each item's weighted value term by a learned permission gate g_ij = λ + (1−λ)σ(ψ(q_i,k_j)) (leak λ=0.05, gate init≈1 so training starts from base attention), yielding h_i = Σ_j α_ij·g_ij·v_j without renormalizing the attention distribution.

Unlike GLU / FiLM / gated-attention readers — which gate already-mixed hidden states — Warrant gates the per-item value term before aggregation, so different items under the same query receive different permissions. A 5-step path-localization procedure inserts the operator only at the metric-defining bottleneck, making it architecture-agnostic rather than an arbitrary attention-layer trick. Theory supplies an SNR-improvement condition (R_S > R_N), Monte-Carlo-validated at P = 0.982 (median 1.19×).

4Strengths vs Limitations

  • Unifying thesis — "evidence ≠ attention mass" spans 5 task families.
  • Sharp diagnostics — PathExposureOnly, shuffled-gate, generic-placement controls isolate the placement claim.
  • Honest reporting — tiers, drops, and "± = seed std, not CI" stated openly.
  • Cheap & agnostic — minimal params, clear insertion recipe.
  • Negligible effect in MTPP / RAG; claim rests on CTDG / TKG / STPP.
  • No per-row tests or multiple-comparison correction across 32 cells (3 seeds each).
  • Path-localization is procedural + partly circular; no code to reproduce placement.
  • Ad-hoc hyperparams (RAG leak 0.7) undercut "one operator, no tuning".

5Open Questions

Does item-wise pre-aggregation gating beat post-attention GLU / FiLM when all sit at the same metric path, parameters matched?
Are CTDG / TKG gains robust under paired t-tests / bootstrap CIs with >3 seeds, after Bonferroni / BH correction?
Does path-localization extend to decoder-only LLM attention and end-to-end RAG answer generation (F1/EM)?
Is the "one operator" claim preserved under a unified λ / init setting — removing RAG's exceptions?
Can the metric-defining path be identified automatically, removing the manual 5-step circularity risk?
