01

Methodology — the loop, not the model

The subject is not a new architecture but a minimal autonomous research loop (in the style of Karpathy, 2026). A general-purpose coding agent — Claude Code running Opus 4.8 at max thinking-effort — repeatedly edits the model code itself; the edited model is trained under a fixed budget and the change is kept only if it lowers held-out MAE by more than τ = 0.002 eV (a noise threshold). Data loading, the held-out fold, and error computation live in agent-immutable scripts, so every candidate is judged identically.

The primary task is MatBench mp_gap: predict electronic band gap (eV) from crystal structure, MAE over a fixed 5-fold split of >100,000 crystals. Search runs are capped at 1,500 s on a single RTX 2070 SUPER with candidates limited to ~5M parameters — enabling 184 experiments. The final model is retrained on the official split at 9,000 s/fold. A second task, log_gvrh (log₁₀ shear modulus, ~11k crystals), is used to develop the workflow.

Two evaluative devices make this more than a capability demo: a novelty audit (30 distinct proposed methods, each compared against 21 reference papers and classed rediscovery / cross-domain / novel) and a crowding score S(e) — the summed Pearson correlation of per-crystal residual vectors among near-equally-accurate models — that prompts the agent to switch model family when it stagnates.

Novelty audit — did the agent invent anything?

The 5 accepted changes are all rediscoveries already present in crystal-GNN literature; 0 cross-domain and 0 novel were kept. Competitiveness comes entirely from recombining known methods.

The 5 accepted changes (all rediscoveries)

  1. Bond-angle messages via a line graph — à la ALIGNN / DimeNet++
  2. Learned 12-NN connectivity — à la coGN
  3. Element-pair features on each edge
  4. Space-group embedding — à la MEGNet global features
  5. Wider Gaussian basis for interatomic distances

Plus 2 hyperparameter width increases (excluded from the 30).

With S, the loop reaches 0.0699 by experiment 40; without it, the run never breaks 0.0721 despite >3× the experiments (160). A simple, harness-side heuristic that demonstrably helps exploration.

05

Strengths & limitations

▲ Strengths
  • Introspective and honest — foregrounds limitations (no novelty, no transfer, stagnation) rather than overselling.
  • Clean control: fixed metric, noise threshold (τ), immutable eval scripts, controlled crowding-score ablation.
  • The novelty audit (30 methods vs. 21 reference papers) is a model for evaluating “did the agent invent anything.”
  • Crowding score is simple, interpretable, and demonstrably helps exploration.
▼ Limitations
  • No fundamental materials-ML novelty — competitiveness is recombinative only.
  • Task-specific; loses to coGN by 6–33% on four other MatBench properties.
  • “Most accurate” is qualified to the no-pretraining regime; foundation models remain ahead.
  • Crowding score is one hand-tuned heuristic (S > 12, τ = 0.004); generality unexplored.
  • Autonomy hinges on the exact agent / seed / prompt — credit loop vs. harness is unclear.
06

Open questions

Q1.How sensitive is the final score to seed code, system prompt, and LLM version / thinking-effort?
Q2.Could unlabeled data + self-supervised pretraining close the gap to foundation models?
Q3.Strict ablation: bond-angle messages vs. space-group embedding vs. element-pair edges — which matters most?
Q4.Does the crowding score generalize beyond two tasks, and can its thresholds be set automatically?
Q5.How does the loop fare on ML interatomic potentials / materials foundation models (flagged future work)?
Q6.Token / compute / wall-clock cost per unit of benchmark gain versus a human expert — and how does it scale?
