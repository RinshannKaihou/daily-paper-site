> **一句话总结：** Dynamo 是一个无需训练的框架，让冻结的 VLM 在 10% 标注训练子集上诊断自身正确/错误尝试，迭代演化出可复用推理技能（认知瓶颈）和可执行 Python 视觉工具（感知瓶颈），每个工具都配对一个门控调用的"掌握技能"，在四个视觉推理基准、五个 VLM 主干共 20 个设置上全面优于零样本基线（平均 +5.6 准确率），并以极小算力成本缩小 65–99% 与任务专用 RL 的差距，且可与 RL 加性叠加。

> **TL;DR:** Dynamo is a training-free framework where a frozen VLM diagnoses its own correct/incorrect attempts on a 10% labeled subset and iteratively evolves reusable reasoning skills (cognitive bottlenecks) plus executable Python visual tools (perceptual bottlenecks), each tool gated by a paired "mastery" skill; it beats the zero-shot baseline on all 20 model–benchmark cells (avg. +5.6 acc) and recovers 65–99% of the gap to task-specific RL (VTool-R1, DeepEyes) at a fraction of the compute, composing additively with RL.

## 核心贡献
- 提出将"逐任务 VLM 适配"重新定义为"能力演化"的问题形式化：冻结智能体从少量标注子集自动构建技能与工具库 C=(S,T)，权重保持不变（∇_θ=0）。
- 设计多候选演化循环：Diagnose（诊断正确/错误案例的根因与瓶颈类型）→ Explore（生成 M 个候选 skill+tool 组合）→ Validate & Promote（按训练集精度单调提升进入持久库），并保证不会退化。
- 引入"掌握技能（mastery skill）"机制：每个生成的工具配对一个 SOP，明确规定何时调用、常见陷阱，从而实现选择性（而非滥发）的工具调用。
- 双能力类型分工：技能对应认知瓶颈（视觉证据已具备但推理过程薄弱，如 ChartQA/MathVista），工具对应感知瓶颈（需变换视觉输入，如 HRBench4K/V* 的子区域裁剪、对比增强、图表重渲染）。
- 在线分布漂移适配：用滚动窗口检测主导任务族变化（φ̂_t 翻转），自动在新子流上重跑演化循环，刷新技能与工具库。
- 实验全面：四基准×五 VLM 共 20 格直接推理全部提升；GTA 工具掌握任务上每一步指标提升；与 RL 对比缩小 65–99% 差距且加性叠加。

## Key Contributions
- Reframes per-task VLM adaptation as capability evolution: a frozen agent builds its own skill and tool library C=(S,T) from a small labeled subset, with ∇_θ=0 throughout.
- A multi-candidate evolution loop (Diagnose → Explore → Validate & Promote) that classifies bottlenecks as cognitive or perceptual, proposes M candidate skill+tool combinations, and promotes only the candidate that strictly improves training-set accuracy (monotonic guard against regression).
- A "mastery skill" mechanism that pairs each generated tool with a SOP specifying when to invoke it, common pitfalls, and how to consume the output — making tool deployment selective rather than indiscriminate, unlike prior tool-creation methods (CREATOR, LATM).
- A clean cognitive/perceptual split validated empirically: SkillOnly dominates ChartQA/MathVista (cognitive), ToolOnly/Full dominate HRBench4K/V* (perceptual).
- Online adaptation to distribution shift via a rolling-window dominance flip detector (Eq. 4) that re-triggers evolution on the new sub-stream, keeping the library in sync with streaming traffic.
- Comprehensive empirical evidence: +5.6 avg accuracy over zero-shot across all 20 model–benchmark cells; per-step gains on GTA across all backbones; 65–99% gap recovery vs. task-specific RL; Dynamo+RL is the strongest configuration in every cell.

## Methodology

**Problem formulation.** Given training set D_train={(x_i,y_i,v_i)} of (question, answer, image), a frozen VLM π_θ, and a capability set C=(S,T) where S is a library of skills (structured Markdown SOPs) and T is a library of tools (executable Python programs), the agent solves queries via a retrieval step + reasoning step:
f_C(x,v;π_θ) = π_θ(·|x,v,Retrieve(x,v;C)).
The goal is to learn C maximizing validation accuracy C★=argmax_C Acc(f_C; D_val), with π_θ frozen.

**Evolution loop (3 phases, N=3 iterations).**
1. *Diagnose.* The agent runs each training case with the current C, producing reasoning traces τ_i. A subset D_sub is sampled (mixing correct + incorrect attempts). The AnalyzerDecider inspects x_i, y_i, v_i, τ_i, and tool outputs to emit a root-cause analysis grounded in visual evidence, a bottleneck type (cognitive vs. perceptual), and an action a∈{skill, tool, both}. Importantly, diagnosis uses the image, not just the trace.
2. *Explore.* Conditioned on the diagnosis, the Generator proposes M candidate skill+tool combinations (single-digit M budget, since selection bias scales logarithmically in M).
3. *Validate & Promote.* Each candidate c_m=(s_m,t_m) is evaluated on D_train; the best is promoted only if it strictly improves over the current f_C (monotonic guard). When the action is "both," the paired skill additionally serves as the tool's mastery SOP, gating tool invocation at inference.

**Skills** are structured Markdown with four fields: When-to-Use (trigger predicate over question/image type), Strategy (numbered SOP), Common Pitfalls, and Worked Example. The Generator merges new insights into an existing skill when the same problem class is covered (preventing library bloat). At inference, Retrieve returns top-K skills by BM25 similarity.

**Tools** are Python functions (≤150 lines) taking image paths and returning processed images/data via OpenCV/PIL/NumPy. Examples generated include chart re-rendering with enlarged axis labels, saliency-guided sub-region extraction, and contrast enhancement for low-quality documents. Tool invocation is gated by retrieving the paired mastery skill, making deployment selective by construction.

**External tool sets (Mode B).** When a curated tool set T_0 is provided, Dynamo skips tool generation and synthesizes a mastery skill for each provided tool, learning *when* to invoke them — no code generation required.

**Online adaptation.** Dynamo continuously samples a sub-training set from the most recent W queries (with their correctness signals) and runs the evolution loop. The dominant task family φ̂_t is tracked via a rolling-window argmax (Eq. 4); when it flips, the next evolution iteration is triggered on the new sub-stream.

## Results & Highlights

**Exp I — Autonomous evolution from scratch (Table 1).** Across 5 VLM backbones (GPT-4o, o4-mini, GPT-5.4, Doubao-Seed-2.0, Qwen3.5-27B) × 4 benchmarks (HRBench4K, MathVista, V*, ChartQA):
- Full Dynamo improves over zero-shot baseline on **all 20 cells**, avg +5.6 accuracy.
- Largest gains: o4-mini/V* (+14.1), o4-mini/HRBench4K (+12.7), GPT-5.4/HRBench4K (+10.7), Doubao/V* (+8.8), Qwen3.5-27B/HRBench4K (+8.4).
- The cognitive/perceptual split holds: SkillOnly competitive on ChartQA/MathVista; ToolOnly/Full dominate on HRBench4K/V*.
- Online adaptation (Fig. 3): on a 208-step non-stationary stream, ordering is Direct ≤ Static < Online adapt ≈ Oracle across all backbones. Under the stress protocol, Direct collapses to 0.03 while Online adapt recovers to 0.95 (GPT-5.4).

**Exp II — Tool mastery on GTA (Table 2).** Dynamo Mode B beats Base Agent and an XSkill-style baseline on every step-level metric (ToolAcc, ArgAcc, InstAcc) and AnsAcc on GPT-4o, GPT-5.4, Doubao-Seed-2.0. Largest gains on Doubao: +18.7 ToolAcc, +15.8 ArgAcc, +16.1 InstAcc, +4.9 AnsAcc. Notably, the XSkill-style baseline drops *below* Base on step-level metrics for GPT-4o/GPT-5.4 — generic prompt injection without learning the environment's tool protocol confuses tool selection. A controlled study (Table 3) shows composite tools (e.g., Visual Arithmetic Solver, 90.0 acc, 1-call policy) compress policies vs. atomic tools (OCR→Calculator, 86.7, 1.83 calls).

**Exp III — vs. task-specific RL (Table 4).** On Qwen2.5-VL 3B/7B using VTool-R1 and DeepEyes protocols:
- Gap recovery: ChartQA 65.7% (3B) / 99.4% (7B); TableQA 67.5% / 91.4%.
- On V* (high-res perception), Dynamo *beats* RL at both scales (+6.3 at 3B, +1.0 at 7B).
- **Dynamo+RL is the strongest configuration in every cell** (e.g., 7B V*: 92.67; 7B HRBench4K: 81.25), showing the two interventions compose additively (+1–3 points over RL alone).

**Cost.** RL baselines need 10^4–10^5 rollouts and 80–240 gradient updates per backbone. Dynamo performs no weight updates — a few thousand frozen-VLM API calls per benchmark (10% train split, N=3 passes).

## Critical Assessment
- **Novelty:** Strong. The cognitive/perceptual bottleneck taxonomy paired with a multi-candidate evolution loop and the mastery-skill gating mechanism is a genuinely novel framing for *training-free* VLM agents. It distinguishes Dynamo from prior tool-creation (CREATOR, LATM, text-domain) and from training-free visual tools with fixed inventories (ZoomEye, ReFocus) by *learning when to call* each tool. The additive composition with RL is a notable practical finding.
- **Rigor:** Good. Evaluation is broad (5 backbones × 4 benchmarks + GTA + RL protocols), with 3 independent seeds reporting mean±std in Table 1, and ablations isolating SkillOnly/ToolOnly/Full. The monotonic promotion guard and disjoint D_train/D_val are methodologically sound.
- **Reproducibility:** Mixed. The algorithm is clearly specified (pseudocode in Appendix A) and uses only standard CV libraries + frozen VLM APIs. However, results depend on *proprietary* models (GPT-5.4, Doubao-Seed-2.0) whose behavior is non-reproducible, and full prompts/candidate pools are not detailed in the main text. Open-weight experiments (Qwen2.5-VL, Qwen3.5-27B) are the more reproducible portion.
- **Significance:** High if results hold. The headline claim — that most task-specific adaptation can be obtained without gradient updates — directly challenges the SFT/RL-heavy VLM adaptation paradigm and has clear deployment value (no weight updates, online adaptation).
- **Red flags / caveats:**
  - The "gap recovery" framing (65–99%) is favorable but Dynamo still trails RL on ChartQA/TableQA in absolute terms; the claim is about *fraction of gap*, not parity.
  - Several ablation statements defer to the appendix (B, D) rather than being fully substantiated in the main text.
  - Selection bias over the discrete candidate set M is acknowledged but only bounded logarithmically; empirical estimate of this bias is left to future work.
  - Online-adaptation experiments reuse completed per-case outputs and use ground-truth labels as the feedback signal — the authors explicitly disclaim fully unsupervised adaptation, but readers may over-read the "online adaptation" claim.
  - V* (191 questions) and HRBench4K (800) are small; some reported gains have relatively large std (e.g., o4-mini ToolOnly on HRBench4K: ±0.167).

## Strengths
- Training-free and weight-frozen — practical and cheap (no 10^4–10^5 rollouts, no 80–240 gradient updates).
- Principled cognitive/perceptual bottleneck taxonomy with clean empirical alignment.
- Monotonic promotion guard prevents capability regression; BM25 retrieval and skill-merging prevent library bloat.
- Mastery skills make tool invocation selective, addressing a real weakness of prior auto-generated tool methods.
- Additive composition with RL (Dynamo+RL strongest everywhere) rather than an either/or framing.
- Online distribution-shift adaptation with a simple, well-motivated rolling-window detector.
- Honest about limitations and careful about claims (e.g., explicitly disclaiming fully unsupervised online adaptation).

## Limitations
- **Diagnosis bounded by the backbone:** the AnalyzerDecider and Generator are the *same* frozen VLM; weak self-introspection can propagate mis-classified bottlenecks, and the promotion guard rejects under-performers but cannot correct upstream diagnosis errors — smaller/less capable backbones may converge slower or to a weaker library.
- **Benchmark scope:** only image-based VLMs across four reasoning benchmarks + GTA + VTool-R1/DeepEyes splits; 3D spatial reasoning and medical imaging are out of scope; transferability of the cognitive/perceptual rule to other modalities is unverified.
- **Reliance on a per-case correctness signal:** both the offline loop and online adaptation require ground-truth (or human/AI-verifier) per-case labels — not fully unsupervised.
- **Candidate selection bias:** only bounded logarithmically in M; broader empirical estimate deferred to future work.
- **Multi-candidate exploration ablation not run in-budget:** removal of multi-candidate exploration needs a larger subset and is left to future work.
- Proprietary-model dependency reduces reproducibility of several headline numbers.

## Relevance
对关注视觉语言模型、智能体工具学习与训练高效化方向的用户高度相关：Dynamo 提供了一条无需梯度更新即可增强 VLM 视觉推理的实用路径，并展示了与 RL 加性叠加的部署价值，契合当前"低成本适配大模型"与"agent 自我演化"的研究热点。

## Questions for Further Exploration
- How sensitive is performance to the 10% training-subset size and to seed variance on small benchmarks (V*=191, HRBench4K=800)?
- Can the per-case correctness signal be reliably replaced by an LLM verifier in production without degrading the gap-recovery numbers?
- Does the cognitive/perceptual decision rule transfer to 3D spatial reasoning or medical imaging, where "perceptual bottleneck" may mean something different?
- How does Dynamo behave on weaker open backbones (e.g., 3B–7B non-Qwen) where self-introspection is the binding constraint?
- Can the multi-candidate exploration budget M and the rolling-window length W be tuned adaptively rather than fixed?
- What is the marginal value of Dynamo+RL vs. RL alone as model scale increases — does the additivity hold or saturate?
- Are the generated tools robust to adversarial or out-of-distribution images, or do they introduce artefacts under distribution shift not covered by the replay stream?
