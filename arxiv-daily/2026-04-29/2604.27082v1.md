> **一句话总结：** 本文提出了一套基于贝叶斯统计的LLM生产系统模型迁移框架，通过校准自动评估指标与人工判断的关系，在仅需少量人工标注的情况下，即可对候选模型进行有置信度保证的比较，并在一个月均530万交互的商业问答系统上成功完成了从Claude 3 Haiku到Nova 2 Lite/Qwen3-32B的迁移验证。

> **TL;DR:** A practical framework for migrating production LLM systems when the underlying model reaches end-of-life. The core innovation is a Bayesian statistical approach that calibrates automated evaluation metrics (RAGAS, custom LLM-based metrics) against limited human judgments, enabling confident model comparison with quantified uncertainty. Demonstrated on a commercial RAG-based QA system serving 5.3M monthly chats across 6 global regions, successfully narrowing 11 candidate models down to Nova 2 Lite and Qwen3-32B as viable replacements for Claude 3 Haiku, with correctness improvements and faster response times.

## 核心贡献

- 提出了一套6步LLM模型迁移框架，覆盖候选模型筛选、输出格式验证、正确性比较、IDK率检查、风格审查和区域/成本决策的完整流程
- 设计了基于贝叶斯统计的指标校准方法：通过少量人工评估构建混淆矩阵，估计自动评估指标的真阳性率（TPR）和假阳性率（FPR）的后验分布，再对模型间正确性差异进行有置信度保证的推断
- 提出了"new_correctness"评估指标，利用完整源上下文而非仅依赖ground truth来判断答案正确性，能更好地处理ground truth不完整或有误的情况
- 在真实商业系统中验证了框架的实用性：从11个候选模型中通过逐步淘汰，最终确定Nova 2 Lite和Qwen3-32B为合适的替代模型，两者均优于基线Claude 3 Haiku
- 对自动提示优化工具MIPROv2在模型迁移场景中的效果进行了探索，发现其效果有限

## Key Contributions

- A 6-step enterprise framework for LLM model migration that systematically narrows candidates through schema compliance, correctness comparison with uncertainty quantification, style checking, and operational constraints
- A Bayesian calibration method that uses confusion matrices from limited manual evaluation to estimate TPR/FPR posterior distributions for automated metrics, enabling principled model comparison even with small evaluation samples
- A new correctness metric ("new_correctness") that evaluates answers against the full source context rather than just the ground truth, better handling incomplete or erroneous ground truth annotations
- Real-world validation on a commercial RAG-based QA system (5.3M monthly interactions, 6 global regions, 2+ years in production), successfully identifying Nova 2 Lite and Qwen3-32B as replacements for the deprecated Claude 3 Haiku
- Empirical investigation of automated prompt optimization (MIPROv2/DSPy) for model migration, finding that it frequently produced no improvement over the original prompt

## Methodology

The framework operates in six sequential steps:

1. **Candidate Selection**: Start with all models passing internal vetting (bias, privacy, licensing, cost, regional availability). In the case study, 11 models passed this stage including Claude 4.5 Haiku, Claude 3.5 Sonnet, AWS Nova variants, OpenAI GPT-OSS, Qwen3 variants, and Gemma 3 27B.

2. **Schema Compliance Filtering**: Eliminate models that cannot reliably produce the required structured output format (XML in this case). Both OpenAI GPT-OSS models were eliminated here; Qwen3-32B's formatting issues were resolved by adding few-shot examples.

3. **Correctness Evaluation with Bayesian Calibration**: This is the core methodological contribution. First, a subset of model outputs (66 HotpotQA, 55 internal, 25 SQuAD examples) is manually evaluated by three assessors. Confusion matrices are constructed for each automated metric against human judgments, yielding posterior distributions for TPR and FPR via Beta priors. These calibrated metrics then estimate "true" correctness rates for each model on full test sets (200 examples each), with confidence intervals that account for both metric imperfection and sample size limitations.

4. **Progressive Elimination**: Models are eliminated if confidence intervals for correctness improvement over baseline include negative values, or if they show significant increases in IDK rate or response time.

5. **Style Checking and Prompt Adaptation**: Remaining models are checked for style violations (stock phrases, verbosity). Prompt modifications are attempted if needed. The authors found reasoning mode degraded Qwen3-32B's style.

6. **Final Selection**: Among viable candidates, selection considers regional availability, cost, and quality tradeoffs. Multiple models may be needed to cover all deployment regions.

Five automated correctness metrics were evaluated: RAGAS correctness, faithfulness, relevance, a custom `llm_correctness` (LLM-based matching against ground truth), and the novel `new_correctness` (evaluates against full source context). The `new_correctness` metric proved most closely aligned with human judgments and was used for final comparisons.

The Bayesian comparison uses Monte Carlo sampling: for each posterior sample of TPR/FPR, the difference in correctness rates between models A and B is approximated as a Normal distribution, from which confidence intervals and probability of improvement are computed.

## Results & Highlights

- **Metric Calibration**: `new_correctness` showed the best alignment with human judgment on both test sets (HotpotQA TPR=0.99, FPR=0.00; internal "basic" TPR=0.93, FPR=0.36). The RAGAS correctness metric had significantly lower TPR (0.76 on HotpotQA, 0.78 on basic), meaning it incorrectly penalized many valid answers.

- **Model Elimination Cascade**:
  - Step 2: OpenAI GPT-OSS 20B and 120B eliminated (XML schema failures)
  - Step 4: Nova Micro, Nova Lite, Gemma 3 27B eliminated (correctness CI included negative values)
  - Step 4: Claude 3.5 Sonnet eliminated (response time too slow: 1.73-1.89s vs. 1.06-1.19s baseline)
  - Step 5: Qwen3-32B reasoning mode dropped (bad style increased from 2.1% to 7.5%)

- **Final Candidates**: Nova 2 Lite and Qwen3-32B both showed statistically confident improvements over Claude 3 Haiku on HotpotQA (p=0.99 and p=1.0 respectively for new_correctness). Qwen3-32B was slightly better (93% confidence on basic test set). Both had faster response times (0.52-0.63s vs. 1.06-1.18s) and lower or comparable IDK rates.

- **MIPROv2 Results**: Automated prompt optimization via DSPy's MIPROv2 was tested with six different objective functions. For several objectives, the optimizer returned the original prompt unchanged. The best-performing optimized prompts (from new_correctness and token-wise F1 objectives) achieved only marginal improvements, suggesting automated prompt adaptation remains unreliable for migration tasks.

- **Human Evaluation Challenges**: The authors document interesting cases of inter-annotator disagreement, including extra information, incomplete responses, and even human evaluator errors. They note that the "human Gold Standard is not without its own error rate."

## Authors & Credibility

The authors (Emma Casey, David Roberts, David Sim, Ian Beaver) are from Verint Systems Inc., a well-established enterprise AI and customer engagement company. The work is grounded in a real production system with 5.3M monthly interactions across 6 global regions, in commercial use for over 2 years. This is not a theoretical exercise but a battle-tested methodology from practitioners managing real enterprise constraints. The paper cites relevant work from RAG, evaluation metrics, and prompt optimization literature. The authors are transparent about limitations, including small manual evaluation sample sizes and the proprietary nature of the system.

## Critical Assessment

**Novelty (Moderate)**: The individual components (Bayesian comparison, confusion matrix calibration, automated metrics) are not individually novel. The contribution lies in their practical integration into a coherent framework specifically designed for the production migration problem. The `new_correctness` metric is a sensible but incremental improvement over existing LLM-as-judge approaches.

**Rigor (Moderate to Good)**: The Bayesian framework properly accounts for metric uncertainty and sample size limitations, which is a meaningful improvement over naive metric comparison. However, the manual evaluation sample sizes are quite small (55-66 examples per test set for metric calibration), and three assessors worked collaboratively rather than independently, which inflates agreement rates. The SQuAD test set was discarded mid-analysis due to insufficient true negatives, which raises questions about the initial experimental design.

**Reproducibility (Limited)**: The system is proprietary, the internal test set is not shared, and many details of the prompt and system architecture are omitted for business reasons. The framework principles are transferable, but exact reproduction is impossible. Code and data are not released.

**Significance (High for practitioners, Moderate for researchers)**: For enterprises facing the very real problem of LLM deprecation (the authors note a ~12-month migration cycle), this is highly valuable practical guidance. For the academic community, it is more of an applied systems paper than a fundamental methods contribution.

**Red Flags**: (1) The paper is partially a Verint product/process description, which may limit generalizability. (2) SQuAD data was discarded after initial analysis rather than being excluded by design, suggesting some post-hoc decision making. (3) The "new_correctness" metric was selected partly because it showed the strongest results -- there is a risk of implicit metric selection bias. (4) The manual evaluation was collaborative, not independent, which may systematically bias the confusion matrices.

## Strengths

- Addresses a genuinely important and underexplored practical problem (LLM model migration in production) that every enterprise deploying LLMs will face
- The Bayesian calibration approach is well-motivated and provides principled uncertainty quantification rather than point estimates
- Comprehensive evaluation across multiple dimensions (correctness, IDK rate, style, response time) reflecting real operational requirements
- Honest documentation of human evaluation challenges, annotator disagreements, and metric limitations
- Includes investigation of automated prompt optimization tools and honestly reports their limited effectiveness
- The framework is modular and the principles are transferable even if specific implementation details are proprietary

## Limitations

- Small manual evaluation sample sizes (55-66 examples per test set) lead to wide confidence intervals, limiting the precision of model comparisons
- Proprietary system and non-public test sets make independent verification impossible
- Only evaluated on a single RAG-based QA system architecture; generalizability to other LLM application types (summarization, classification, agentic systems) is untested
- The `new_correctness` metric relies on Claude 4 Sonnet as the evaluator, introducing a dependency on another LLM's judgment and potential model-specific biases
- Collaborative (not independent) manual evaluation may overestimate metric accuracy
- Does not address long-term monitoring or continuous evaluation after migration is complete
- The framework assumes access to an existing baseline model for comparison, which may not always be available (e.g., greenfield deployments)

## Relevance

本文对需要在大规模生产环境中进行LLM模型替换的工程实践具有直接参考价值。其基于贝叶斯统计的指标校准方法可以在少量人工标注下实现有置信度保证的模型对比，适合需要频繁应对模型退役的RAG系统开发者借鉴。

## Questions for Further Exploration

- How well does the framework generalize to non-QA LLM applications such as summarization, code generation, or multi-turn dialogue systems?
- Can the Bayesian calibration approach be extended to incorporate multiple automated metrics simultaneously rather than selecting a single "best" metric?
- What is the minimum manual evaluation sample size needed to achieve practically useful confidence intervals for model comparison?
- How should the framework handle continuous model updates (e.g., silent model weight changes by providers) rather than discrete end-of-life events?
- Could the `new_correctness` metric be further improved by incorporating calibrated probability estimates rather than binary correct/incorrect judgments?
- How robust is the framework to distribution shift in production traffic compared to the test sets used during migration evaluation?
