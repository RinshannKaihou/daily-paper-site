Efficient and Interpretable Transformer for Counterfactual Fairness

arXiv: 2604.26188v1

Date: 2026-04-29

https://arxiv.org/abs/2604.26188v1

一句话总结： 本文提出 FCorrTransformer，一种用于表格数据的轻注意力 Transformer 架构，其注意力矩阵可直接解释为特征依赖关系，并引入反事实注意力正则化（CAR）在注意力层面实现群体不变表示，在不依赖显式因果假设的情况下实现了强反事实公平性，同时在预测性能和模型复杂度上保持竞争力。
TL;DR: The authors propose FCorrTransformer, an attention-light transformer for tabular data whose attention matrix is directly interpretable as pairwise feature dependencies. Combined with Counterfactual Attention Regularization (CAR) -- which enforces group-invariant representations at the attention level via input augmentation and parameter sharing -- the framework achieves strong counterfactual fairness without explicit causal models. Evaluated on imbalanced classification (BAF, Adult) and regression (InsurTech) benchmarks, FCorrTransformer + CAR delivers competitive predictive performance with substantially fewer parameters and lower memory usage than FT-Transformer, while providing interpretable attention heatmaps that reveal how biased feature dependencies are suppressed.

核心贡献

  • 提出 FCorrTransformer 架构：去除标准 Transformer 中的 Q/K/V 线性投影和注意力块周围的残差连接，使注意力矩阵可直接解释为特征间的成对依赖关系，同时保持嵌入维度与原始输入一致，避免高维嵌入导致的参数爆炸。
  • 提出 Counterfactual Attention Regularization (CAR)：通过反事实数据增强和参数共享策略，在注意力层面最小化敏感特征与非敏感特征之间依赖关系的跨群体差异，无需显式因果模型假设即可实现反事实公平。
  • 设计高效的 输入增强策略：通过 SenElementWiseLinear、SenCatLinear、SenLayerNorm 模块复用原始参数处理增强的反事实类别，无需额外可学习参数，显著降低计算开销（如回归任务中每 epoch 训练时间从 52.98s 降至 14.40s）。
  • 在多个不平衡数据集（BAF、Adult、InsurTech）上验证了框架的有效性，证明架构设计与公平性机制的正确配合对实现强反事实公平至关重要。

Key Contributions

  • FCorrTransformer architecture: Removes Q/K/V projections and residual connections around the attention block, keeping embeddings one-dimensional so the attention matrix admits a direct interpretation as pairwise feature dependencies. Uses element-wise linear layers instead of fully connected layers to avoid introducing artificial feature interactions.
  • Counterfactual Attention Regularization (CAR): A regularization framework that enforces group-invariant fair representations by minimizing the discrepancy in attention dependency vectors between the original input and counterfactual permutations of sensitive features. Does not rely on explicit causal structural assumptions.
  • Efficient input augmentation with parameter sharing: Introduces specialized modules (SenElementWiseLinear, SenCatLinear, SenLayerNorm) that reuse existing parameters for augmented counterfactual categories, introducing zero additional learnable parameters while enabling single-forward-pass computation of all counterfactual attention vectors.
  • Empirical validation on imbalanced benchmarks: Demonstrates that FCorrTransformer + CAR achieves strong counterfactual fairness on BAF (1M samples, 31 features), Adult, and InsurTech (235K samples, 581 features) datasets, with only modest performance loss relative to unconstrained models.

Methodology

FCorrTransformer Architecture:

The model follows an encoder-only structure. Both numerical and categorical features are embedded via element-wise linear layers with GELU activation, preserving the original input dimensionality (p features remain p-dimensional). For categorical features, a CatLinear layer aggregates one-hot encoded embeddings back to one value per feature. The attention mechanism removes all Q/K/V linear projections -- the query, key, and value are all the same layer-normalized embedding. The attention matrix A(x) = SoftMax(E_LN(x) * E_LN(x)^T / sqrt(p)) directly captures pairwise scalar similarities between features. Residual connections around the attention block are removed to prevent biased information from bypassing the attention mechanism. Element-wise linear layers (not fully connected) are used throughout to avoid artificial feature interactions.

Counterfactual Attention Regularization (CAR):

CAR permutes the sensitive feature across all unique categories, computes the attention dependency vector (the sigma-th column of the attention matrix, where sigma is the sensitive feature position) for each permutation, and minimizes the average L2 discrepancy between the original and counterfactual dependency vectors. The loss is: L_CAR = (1/C^s) * sum_i ||A_{:,sigma} - A^{counterfactual}_{:,sigma}||^2. An input augmentation strategy augments the input with all counterfactual sensitive categories in a single forward pass, using parameter-sharing modules (SenElementWiseLinear, SenCatLinear, SenLayerNorm) that extract and reuse the original sensitive feature's parameters. The regularization coefficient lambda is set automatically based on the ratio of performance loss to CAR loss on the first batch.

Results & Highlights

  • BAF (imbalanced classification, 1.1% fraud rate): FCorrTransformer + CAR achieves near-zero AvgIF (0.0003), AUROC Gap (0.0000), and AUPRC Gap (0.0005) on the test set, while maintaining AUROC of 0.8627. FT-Transformer + CAR leaves substantial residual bias (AvgIF = 0.0784). LightGBM and FFN show no counterfactual fairness guarantees. FCorrTransformer uses only 2.5% of FT-Transformer's parameter count (21,767 vs 868,483).
  • InsurTech (imbalanced regression, 0.79% positive claims): FCorrTransformer + CAR provides the only robust counterfactual fairness guarantee among all evaluated models. FCorrTransformer outperforms FT-Transformer on Gini index despite having roughly half the parameters. Training uses only 4.4% of the GPU memory compared to FT-Transformer.
  • Attention interpretability: Heatmaps show that diagonal elements correspond to feature significance and off-diagonal elements capture pairwise dependencies. CAR visibly suppresses biased dependencies between sensitive and non-sensitive features while redistributing attention to fairer features.
  • Synthetic data analysis: On a controlled generative process, CAR reduces the ratio of biased interaction (X1-X2) to self-attention from 0.28 to 0.007, demonstrating targeted suppression of indirect discrimination.
  • Controllability: The regularization coefficient lambda provides monotonic control over counterfactual fairness metrics (AvgIF, AUROC/AUPRC Gap), enabling practitioners to adjust fairness levels per application requirements.

Critical Assessment

Novelty (Moderate-High): The combination of an attention-light tabular transformer with counterfactual fairness regularization at the attention level is novel. The key insight -- that removing residual connections and Q/K/V projections enables fairness control through attention -- is architecturally simple but effective. The input augmentation with parameter sharing is a clean engineering contribution. However, the individual components (simplified attention, counterfactual data augmentation, regularization) are not individually new.

Rigor (Moderate): The mathematical formulation is thorough and the synthetic data experiment provides mechanistic insight. The automatic lambda setting (based on first-batch loss ratio) is pragmatic but ad hoc and lacks theoretical justification for why it should guarantee fairness. The paper acknowledges that CAR operates only on the first attention layer, and deeper architectures lose interpretability and fairness guarantees. The single-head attention design limits expressive capacity.

Reproducibility (Moderate-High): Detailed architecture descriptions, mathematical formulations, and dataset references are provided. Hyperparameter optimization is mentioned but not fully detailed. The automatic lambda selection aids reproducibility. No code repository is referenced in the paper.

Significance (Moderate-High): Addresses a practically important problem at the intersection of ML fairness and regulatory compliance in finance/insurance. The interpretability of the attention matrix as feature dependencies is valuable for model auditing. The efficiency gains (2.5% parameter count, 4.4% GPU memory) are practically meaningful for deployment.

Red flags:

  • FCorrTransformer consistently underperforms LightGBM and FT-Transformer on predictive metrics, which may limit practical adoption despite fairness gains.
  • The optimal architectures in all experiments are single-encoder-layer models, which raises questions about scalability to more complex tasks requiring deeper architectures.
  • CAR is limited to categorical sensitive features and does not naturally extend to continuous sensitive features.
  • No formal causal guarantees are provided -- the framework enforces empirical counterfactual invariance but does not establish causal identification.

Strengths

  • Clean architectural insight: removing residual connections and projections creates a direct pathway for fairness control through attention.
  • Interpretable attention mechanism where matrix elements correspond to feature dependencies, enabling transparent model auditing.
  • Substantial efficiency gains: dramatically fewer parameters and lower memory usage than standard tabular transformers.
  • Strong empirical counterfactual fairness results across multiple datasets and tasks (classification + regression).
  • Practical regulatory alignment: the controllability of fairness via lambda and the interpretability of attention heatmaps directly support insurance/finance compliance use cases.
  • Honest discussion of limitations, including the single-layer preference and the lack of formal causal guarantees.

Limitations

  • CAR is designed for categorical sensitive features and does not naturally handle continuous sensitive features (e.g., income).
  • No formal causal guarantees are established; the framework provides empirical counterfactual invariance at the attention level only.
  • The attention-light design inherently limits expressive capacity. Multi-layer architectures degrade interpretability and fairness guarantees.
  • FCorrTransformer consistently shows a performance gap compared to FT-Transformer and LightGBM on predictive metrics.
  • The automatic regularization coefficient setting, while pragmatic, lacks theoretical grounding.
  • The framework has not been evaluated on very large-scale datasets or with multiple simultaneous sensitive features in the experiments.

Questions for Further Exploration

  • Can CAR be extended to continuous sensitive features through quantile-based grouping or distributional matching, and would this preserve the same level of counterfactual fairness?
  • How does FCorrTransformer + CAR perform on non-tabular domains (e.g., NLP, vision) where attention-based debiasing has been more extensively studied?
  • What is the theoretical relationship between the CAR regularization strength and the resulting counterfactual fairness metric -- can formal bounds be established?
  • Could a hybrid architecture that uses FCorrTransformer for the first layer (where interpretability and fairness control matter) and standard attention for deeper layers achieve both fairness and higher predictive performance?
  • How sensitive are the results to the automatic lambda selection heuristic, and could adaptive or curriculum-based scheduling of lambda improve the fairness-performance trade-off?
