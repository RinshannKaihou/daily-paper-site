> **一句话总结：** PRAETOR 提出一种基于参数化确定性有限自动机(pDFA)的有状态行为防火墙，通过离线编译良性工具调用轨迹并在运行时以 O(1) 常数时间执行状态转移检查，在结构化工作流中将多步攻击成功率降至 2.2%，远低于无状态防火墙 AEGIS 的 12.8%。

> **TL;DR:** PRAETOR is a stateful behavioral firewall for LLM-driven agents that compiles verified benign tool-call traces into a parameterized DFA (pDFA), enforcing permitted sequences and parameter bounds at runtime with O(1) per-call overhead (2.2 ms). Evaluated on Agent Security Bench (ASB), it achieves 2.2% macro-averaged Attack Success Rate (ASR) across three structured workflows (0% on multi-step and context-sequential attacks), compared to 12.8% for the stateless AEGIS firewall. Performance degrades to 5.6% ASR when including two high-entropy open-ended scenarios. The approach is most effective for narrow-task, single-role agents with stable tool vocabularies.

## 核心贡献

- **有状态行为防火墙架构 PRAETOR**：首次将经典入侵检测中的序列建模思想引入 AI Agent 安全领域，通过参数化确定性有限自动机(pDFA)建模工具调用的合法序列轨迹，解决无状态防火墙对上下文序列注入攻击的"序列盲区"问题。
- **两阶段设计（离线分析 + 在线执行）**：将计算复杂的行为分析完全移至离线 Profiling 阶段，运行时 Gateway 仅执行常数时间 O(1) 状态转移查找，实现 2.2 ms 的单次调用延迟（比 AEGIS 快 3.7 倍）。
- **双层防御机制**：结构层（pDFA 状态转移约束）将攻击面压缩至 0.018%，参数层（epsilon-ball 数值边界 + SBERT 语义嵌入聚类 + 敏感参数白名单）负责拦截在结构路径内的恶意参数。
- **严格的对抗性评估**：包括 Black-Box、Gray-Box、Active Oracle Adversary 三种威胁模型，以及 1,000 次算法拼接的 exfiltration 载荷测试，仅 1.4% 匹配合法结构路径且最终全部被参数守卫拦截。

## Key Contributions

- **PRAETOR stateful behavioral firewall**: Introduces parameterized DFA (pDFA) to AI agent security, modeling permitted tool-call sequences and parameter bounds from verified benign traces, addressing the "sequential blindness" limitation of stateless firewalls like AEGIS.
- **Two-stage architecture (offline profiling + online enforcement)**: Shifts computationally expensive behavioral analysis entirely offline, achieving O(1) per-call runtime overhead at 2.2 ms latency (3.7x faster than AEGIS).
- **Dual-layer defense**: Structural constraints (pDFA state transitions) collapse the attack surface to 0.018% of combinatorial space, while parameter guards (epsilon-ball bounds, SBERT embedding clusters, sensitive-parameter whitelists) block exploitation within surviving structural paths.
- **Comprehensive adversarial evaluation**: Tests under Black-Box, Gray-Box (10%-90% trace knowledge), and Active Oracle Adversary (Q=100 queries) models; 1,000 algorithmically spliced exfiltration payloads yield 0 end-to-end successes (95% CI [0%, 23.2%]).
- **Honest scope boundary analysis**: Explicitly demonstrates where the low-behavioral-entropy assumption holds (structured workflows: 2.2% ASR) and where it degrades (open-ended agents: 8.6%-12.6% ASR), providing concrete operational deployment boundaries.

## Methodology

PRAETOR operates in two stages:

**Offline Profiling Stage**: The Telemetry Profiler ingests a verified benign trace corpus C (e.g., 500 traces from ToolBench) and compiles it into a parameterized DFA. Each state in the pDFA is defined by a w-gram context window (default w=3) of prior tool calls, with transitions annotated by parameter guard profiles: epsilon-ball bounds for numeric parameters (widened by epsilon_num), SBERT centroid + radius for string parameters (widened by epsilon_str), and categorical whitelists. Low-frequency states below pruning threshold theta (default 3) are removed to control state-space size. The resulting pDFA for the Customer Service scenario has 142 valid paths covering just 0.018% of the uniform combinatorial action space for sequences of length k=5.

**Runtime Enforcement Stage**: The Runtime Gateway operates as a transparent sidecar at the tool-dispatch boundary. For each incoming tool call, it performs a constant-time state-transition lookup in the pDFA. If the call matches a permitted transition and satisfies all parameter guards, it is forwarded to the environment; otherwise, it is halted and logged in a SHA-256 hash-chain append-only audit log. The gateway requires no modification to the agent's internal reasoning loop, system prompt, or model weights.

**Key parameters**: context window w=3, pruning threshold theta=3, slack factors epsilon_num and epsilon_str tuned per deployment. The pDFA itself consumes less than 1MB; the INT8-quantized ONNX embedding model is 23MB. Gateway startup latency is 45ms.

## Results & Highlights

- **Structured workflows (3 scenarios)**: PRAETOR achieves 2.2% macro-averaged ASR vs. AEGIS at 12.8% and PromptArmor at 32.9%. Specifically: 0% ASR on multi-step exfiltration (MSE) and 0% on context-sequential attacks (200 targeted attempts).
- **Five-scenario aggregate**: 5.6% macro-averaged ASR across all five ASB scenarios (including two high-entropy open-ended agents where ASR degrades to 8.6% and 12.6%).
- **Defense-in-depth (PRAETOR + AEGIS)**: Combined configuration achieves 0.8% ASR across all five scenarios.
- **Algorithmic splicing stress test**: 1,000 spliced exfiltration payloads -- only 14 (1.4%) matched valid structural paths; all 14 were blocked by parameter guards (0 end-to-end successes).
- **Gray-Box resilience**: Even with 90% knowledge of the benign trace corpus, adversary achieves only 0.8% ASR due to centroid drift from the unseen 10%.
- **Active Oracle Adversary**: With Q=100 binary oracle queries, attacker recovers 88% of structural transitions but achieves only ~2% evasion rate on parameter guards; synonym-substitution attacks achieve 18% evasion on continuous string bounds.
- **Latency**: 2.2 ms median per-call overhead (3.7x speedup over AEGIS at 8.1 ms).
- **False positives**: 2.0% benign task failure rate (BTFR) on legitimate workloads.

## Critical Assessment

**Novelty (Moderate-High)**: The core idea of applying DFA-based sequence modeling from classic host intrusion detection to AI agent tool-call monitoring is a natural but well-executed transfer. The two-stage architecture (offline profiling + online O(1) enforcement) is cleanly designed. The pDFA construction with w-gram states and parameterized transitions is a solid engineering contribution, though not conceptually surprising. The approach draws heavily on established ideas from Forrest et al.'s "sense of self for Unix processes" (1996).

**Rigor (High)**: The evaluation is unusually thorough for a single-author paper. Three threat models (Black-Box, Gray-Box, Active Oracle), per-family ASR breakdowns, vocabulary-block vs. contextual-block decomposition, Clopper-Pearson confidence intervals, and explicit testing of the low-entropy assumption boundary. The vocabulary/contextual decomposition (showing that as tool vocabulary grows, contextual blocking dominates over simple vocabulary whitelisting) is a particularly insightful analysis.

**Reproducibility (Moderate)**: The paper claims the gateway will be open-sourced, uses standard benchmarks (ASB, ToolBench), and specifies hardware (AWS c6i.4xlarge). However, the pDFA construction depends on specific benign trace corpora and parameter tuning (w, theta, epsilon values), making exact reproduction dependent on recreating the training data pipeline. The single-author status also means no independent internal validation.

**Significance (Moderate-High for structured agents, Low for general-purpose agents)**: PRAETOR addresses a real and growing problem -- securing LLM agents against indirect prompt injection and multi-step exploits. The 0% ASR on multi-step attacks in structured workflows is compelling. However, the honest reporting that ASR degrades to 12.6% for open-ended agents (comparable to AEGIS's 12.8% in structured settings) significantly tempers the generality of the contribution.

**Red flags**:
- Single-author paper limits credibility of claims; no external validation.
- The 18% synonym-substitution evasion rate on continuous string parameter bounds is a non-trivial residual vulnerability that effectively means exact-match whitelisting must bear the final defensive load.
- The approach fundamentally requires a stable, low-entropy behavioral profile, which excludes the most capable and general-purpose agents.
- The "cold start" problem (telemetry poisoning during offline profiling) is explicitly acknowledged but deferred to operational mitigations rather than technical solutions.

## Strengths

- Clean separation of offline profiling and online enforcement, yielding excellent runtime performance (2.2 ms overhead).
- Honest and transparent evaluation that clearly delineates where the approach works and where it breaks down.
- The vocabulary-block vs. contextual-block decomposition is a novel diagnostic that demonstrates the value of stateful tracking beyond simple tool whitelisting.
- Strong formal grounding in established intrusion detection theory (DFA, w-gram models, Bell-LaPadula).
- Defense-in-depth results (PRAETOR + AEGIS at 0.8% ASR) demonstrate practical composability.
- Minimal integration burden: operates as a black-box sidecar with one-line configuration change.

## Limitations

- **Applicable only to narrow-task, single-role agents** with stable tool vocabularies and low behavioral entropy. The approach fundamentally cannot protect general-purpose agents with large, dynamic tool sets.
- **18% evasion rate via synonym substitution** on continuous string parameter bounds means exact-match whitelisting of sensitive parameters is ultimately required, negating some of the claimed generality of the parameter guard approach.
- **Telemetry poisoning vulnerability** during the offline profiling phase is acknowledged but not technically addressed; the security of the entire system rests on the integrity of the benign trace corpus.
- **No defense against "Tool-Call ROP"** (repeating legitimate calls to probe state), only operational mitigations suggested (rate limiting).
- **Single-author paper** without independent validation of experimental results.
- **No evaluation against adaptive attackers who actively poison the training corpus** or who have full knowledge of the pDFA structure (the Oracle adversary experiments use binary queries, not structural reconstruction).

## Relevance

本文对 AI Agent 安全领域具有重要参考价值，特别是其将经典入侵检测的序列建模方法迁移到 LLM Agent 工具调用监控的思路，以及通过 pDFA 状态转移将有状态防御与无状态防御进行对比分析的方法论。对于关注 Agent 安全防护、Prompt Injection 防御或运行时验证的研究者而言，本文提供了一个设计清晰、评估严谨的基线系统。

## Questions for Further Exploration

1. How would PRAETOR perform in a multi-agent setting where agents delegate sub-tasks to each other, creating interleaved tool-call sequences?
2. Can the pDFA construction be made adaptive (online learning) to handle concept drift without requiring full re-profiling from a clean baseline?
3. What is the minimum size of the benign trace corpus needed to achieve stable ASR performance, and how does ASR scale with corpus size below 500 traces?
4. Could the synonym-substitution vulnerability (18% evasion) be mitigated by replacing SBERT centroids with more robust text representations (e.g., LLM-based embeddings with adversarial training)?
5. How does PRAETOR compare to information-flow control (IFC) approaches like Fides in a combined deployment, and what are the practical coverage gaps between structural/parameter enforcement and taint tracking?
