Folding Tensor and Sequence Parallelism for Memory-Efficient Transformer Training & Inference

arXiv: 2604.26294v1

Date: 2026-04-29

https://arxiv.org/abs/2604.26294v1

一句话总结： 本文提出TSP（Tensor-Sequence Parallelism），将张量并行（TP）和序列并行（SP）折叠到单一设备轴上，使得每个rank同时拥有权重分片和序列分片，从而在同一轴上将参数内存和激活内存同时降低1/D，在7B模型、128K序列长度下实现了所有并行策略中的最低峰值显存，且吞吐量与TP+SP持平。
TL;DR: The authors propose TSP (Tensor-Sequence Parallelism), which folds tensor parallelism and sequence parallelism onto a single device-mesh axis instead of assigning them to separate orthogonal dimensions. Each rank simultaneously holds a weight shard and a sequence shard, reducing both parameter memory and activation memory by 1/D along the same axis. Implemented with broadcast-based attention schedules and ring-based MLP schedules, TSP achieves the lowest peak memory among TP, SP, and TP+SP across all tested sequence lengths (16K--128K tokens) on up to 1024 AMD MI300X GPUs, while maintaining competitive throughput because the extra weight-movement communication is pipelined behind dominant GEMM operations. The advantage widens with increasing parallelism degree and context length.

Key Contributions

  • Introduces parallelism folding, collapsing tensor parallelism (TP) and sequence parallelism (SP) onto a single device axis so each rank holds both a weight shard and a sequence shard simultaneously
  • Designs a TSP attention schedule using packed weight-shard broadcasts with K/V all-gathers for causal attention, and a TSP MLP ring schedule where weight shards circulate point-to-point while partial outputs accumulate locally
  • Provides a rigorous theoretical framework comparing communication volume, memory decomposition, and FLOPs across DP, TP, SP, TP+SP, and TSP with formal regime analysis showing TSP becomes communication-competitive when BS > 8h
  • Demonstrates communication-computation overlap strategies: MLP ring communication is pipelined behind GEMM compute; attention weight broadcasts overlap with FlashAttention and K/V all-gathers using separate CUDA streams
  • Empirically validates on up to 1024 AMD MI300X GPUs (Zephyra cluster) that TSP achieves the lowest peak memory across all sequence lengths tested (16K--128K) while maintaining competitive or better throughput than TP+SP baselines

Methodology

The paper develops TSP through three pillars: conceptual design, theoretical analysis, and empirical validation.

Parallelism Folding Concept. In conventional multi-dimensional parallelism, TP and SP are assigned to orthogonal device-mesh dimensions (T x Sigma total devices). TSP folds both onto a single D-way axis. The key insight is that this eliminates replication of both weights and activations along the same dimension, unlike TP (which replicates activations) or SP (which replicates weights), and unlike TP+SP (which requires D^2 devices to get full benefit).

TSP Attention Schedule (Algorithm 1). Each rank stores a zigzag-partitioned sequence shard and a shard of QKV/O projections packed into a single buffer. Over D iterations: (1) the weight-owning rank broadcasts its shard; (2) all ranks compute local Q, K, V projections; (3) K, V tensors are all-gathered across the TSP group for FlashAttention; (4) the partial attention output is projected and accumulated locally.

TSP MLP Schedule (Algorithm 2). Each rank holds local shards of gate, up, and down projections. Over D iterations, weight shards circulate in a ring via point-to-point communication while each rank accumulates gated MLP outputs for its local sequence tokens. This is distinct from standard TP's all-reduce approach.

Theoretical Analysis (Section IV). The authors derive closed-form expressions for communication volume, memory footprint, and per-device FLOPs for all five strategies (DP, TP, SP, TP+SP, TSP). Key result: TSP forward communication is C_TSP_fwd = P_attn beta_p + P_mlp beta_p (D-1)/D + 2BShbeta_p*(D-1)/(gD), which adds weight-movement terms compared to TP or SP but these become dominated by activation-proportional terms at long context (when BS > 8h).

Overlap Strategy. For MLP, asynchronous send/recv of next weight shards overlaps with current GEMM computation. For attention, three stages (broadcast, projection, all-gather+FlashAttention) are staggered across consecutive weight shards using separate communication and compute streams.

Results & Highlights

  • Peak Memory (Figure 9): TSP achieves the lowest peak memory at all sequence lengths from 16K to 128K tokens on 8 MI300X GPUs. At 16K, TSP (31.5 GB) nearly matches TP (31.0 GB) since weight memory dominates. The advantage widens at longer contexts as activation memory grows.
  • Throughput vs. Sequence Length (Figures 10-11): At D=2 and D=4 on 16 GPUs, TSP consistently outperforms matched TP+SP baselines. At scale (1024 GPUs, D=2/4/8, 32K--128K tokens), the TSP advantage grows with increasing folded degree.
  • Throughput vs. Batch Size (Figure 12): Because TSP reduces both weight and activation memory, larger micro-batches fit into device memory, improving throughput.
  • Theoretical Validation (Figures 5-6): TSP communication volume exceeds TP at short sequences but the gap narrows as sequence grows; TSP provides strictly lowest per-GPU memory across all model sizes tested.
  • Scalability: TSP composes orthogonally with existing parallelism axes (PP, EP, DP), allowing flexible mesh configurations that keep model-parallel groups within high-bandwidth intra-node domains.

Critical Assessment

  • Novelty: Moderate-High. The idea of folding TP and SP onto one axis is conceptually clean and practically impactful. However, the closest prior work METP (Liang et al., 2025) already applies ring-based communication to both attention and MLP. TSP's distinction is the specific folding formulation and the broadcast-vs-ring scheduling for attention vs. MLP respectively. The folding concept is also inspired by MoE Parallel Folding (Liu et al., 2026).
  • Rigor: High. The theoretical analysis is thorough, with closed-form derivations for all five strategies, regime analysis with explicit crossover conditions (BS > 8h for MHA), and honest accounting of communication overhead. Memory models match empirical measurements well.
  • Reproducibility: Moderate. The implementation targets AMD MI300X with Infinity Fabric, which is less common than NVIDIA hardware. No code is released with the paper. The cluster-specific topology (rails-only with Pollara NICs) means results may not directly transfer to NVIDIA-based clusters. However, the algorithmic description is detailed enough to reimplement.
  • Significance: High. As context lengths continue to grow (128K+), memory-efficient parallelism strategies become critical. TSP offers a practical way to reduce peak memory without requiring D^2 devices like TP+SP, which is valuable for real-world deployments.
  • Red flags: The throughput comparisons are primarily forward-pass only (Figures 10-11), with forward-backward only shown for the batch-size sweep (Figure 12). Full training throughput numbers (including gradient synchronization and optimizer step) would be more informative. The evaluation is limited to a 7B dense model; scaling to larger models (70B+) and MoE architectures would strengthen the claims. The paper acknowledges the results depend on favorable intra-node bandwidth ratios.

Strengths

  • Clean conceptual contribution: parallelism folding is an intuitive and impactful idea that simplifies device-mesh allocation
  • Rigorous theoretical framework with explicit formulas for all strategies, enabling direct comparison without running experiments
  • Practical implementation with communication-computation overlap that makes the theoretical communication overhead largely irrelevant in wall-clock time
  • Honest about trade-offs: acknowledges TSP adds communication volume and that results depend on hardware topology
  • Comprehensive empirical evaluation at meaningful scale (up to 1024 GPUs)
  • The TSP group can be freely combined with other parallelism axes (PP, EP, DP), making it a flexible building block

Limitations

  • Evaluated only on AMD MI300X hardware with Infinity Fabric; performance characteristics may differ on NVIDIA NVLink or slower interconnects
  • No code release; reproducibility depends on reimplementation
  • Full end-to-end training throughput (including optimizer step, gradient accumulation) is not comprehensively benchmarked
  • Only tested on a 7B dense model; scalability to 70B+ models and MoE architectures is claimed but not demonstrated
  • The weight-movement overhead in TSP could become a bottleneck on systems with lower intra-node bandwidth-to-FLOP ratios
  • Theoretical analysis ignores embedding layers, normalization, and other pointwise operations, which is standard but worth noting

Relevance

本文对分布式大模型训练的并行策略设计有直接参考价值，特别是在长上下文场景下如何在有限GPU显存中同时降低权重和激活的内存占用。如果关注大规模模型训练基础设施优化或长上下文训练效率，TSP提供了一种值得关注的并行化新思路。
