> **一句话总结：** 本文证明了具有组合稀疏性的连续函数可以用深度 Kolmogorov-Arnold 网络(KAN)表示，且逐层 Lipschitz 乘积 P(KAN) 独立于输入维度 n，对于标准运算集 {+,-,*,sin,cos} 在 [0,1]-有界输入上 P(KAN) <= 1，同时给出了线性于 n+N 的层宽上界和最优 B-spline 逼近率。

> **TL;DR:** This theoretical note proves that compositionally structured functions (those representable by finite computation trees with O(1) sparsity per node) admit deep KAN representations whose layer-wise Lipschitz product is bounded independently of input dimension n. For the standard operation set {+,-,*,sin,cos} on [0,1]-bounded inputs, the Lipschitz product P(KAN) <= 1, layer widths are O(n+N), and approximation error achieves optimal B-spline rates O(N * G^{-(k+1)}). The result directly addresses the Lipschitz control gap in deep KAN stacks identified in the original KAN paper by Liu et al. (2024).

## 核心贡献

- 引入了逐层 Lipschitz 乘积 P(KAN) 这一架构相关量，并将其与环境 Lipschitz 常数 Lip(KAN) 仔细区分
- 证明了主结构定理(Theorem 4.3)：在五个假设条件下(A1-A5)，组合结构函数的 KAN 表示的 Lipschitz 乘积 P(KAN_f) <= max(C*,1)^{L_f}，与输入维度 n 无关
- 对于标准运算集 {+,-,*,sin,cos} 在 [0,1]-有界输入上，证明了 P(KAN) <= 1，且层宽 n_l <= n + 8N
- 对 C^m 函数获得了最优 B-spline 逼近率 O(N * G^{-(k+1)})
- 推导了显式范围界：加法树 B_f <= N+1，纯乘法/三角树 B_f = 1
- 提供了数值实验验证 P(KAN)=1.0 以及 Lean 4 形式化草图

## Key Contributions

- Introduces the layer-wise Lipschitz product P(KAN) as the architecturally relevant quantity, distinguishing it from the ambient Lipschitz constant Lip(KAN), and argues that P(KAN) is the correct metric for stability analysis of deep KAN stacks
- Proves Theorem 4.3 (Layer-wise Lipschitz-product control): under five structural assumptions (A1)-(A5), compositionally structured functions admit KAN representations with P(KAN_f) bounded independently of input dimension n, specifically P(KAN_f) <= max(C*,1)^{L_f} where L_f <= c_max * N
- For the standard operation set {+,-,x,sin,cos} on [0,1]-bounded inputs, achieves the sharp bound P(KAN) <= 1 (Corollary 4.4), verified via explicit primitive block constructions (Proposition 4.1)
- Derives layer width bounds n_l <= n + 2*w_max*N (linear in both input dimension and tree size), and for the standard operation set n_l <= n + 8N
- Establishes approximation error bounds: ||f - KAN_f|| = O(N * max(C*,1)^{d(f)} * epsilon_Op), and for C^m functions obtains optimal B-spline rates O(N * G^{-(k+1)})
- Provides explicit range bounds (Lemma 5.1): B_f <= N+1 for additive trees, B_f = 1 for purely multiplicative/trigonometric trees, and finiteness for general trees
- Gives a Jacobian-based lower bound on P(KAN) (Proposition 4.8) and verifies all theoretical bounds numerically

## Methodology

The paper develops a constructive proof framework for building deep KAN representations of compositionally structured functions with controlled Lipschitz products.

**Core framework:**

1. **Computation tree representation:** A function f: [0,1]^n -> R is represented by a finite computation tree T with N internal nodes. Each internal node is labeled by an operation op in Op (e.g., {+,-,*,sin,cos}), and the tree has compositional sparsity s = O(1) (each node depends on at most s coordinates).

2. **Primitive KAN blocks (Definition 2.5):** For each operation op in Op, the author constructs an explicit primitive KAN block B_op on bounded domains. These blocks are the building blocks of the construction. For multiplication, the classical identity u*v = ((u+v)^2 - (u-v)^2)/4 is realized via three B-spline layers, achieving block Lipschitz product Lambda_x = 1.

3. **Layer-wise Lipschitz product (Definition 2.4):** P(KAN) = prod_{l=0}^{L-1} max_i M_{l,i}, where M_{l,i} is the maximum outgoing Lipschitz constant from neuron (l,i). This differs from the ambient Lipschitz constant Lip(KAN) <= W^L * P(KAN), where W is the maximum layer width.

4. **Sequential block composition:** The KAN representation is built by sequentially composing primitive blocks for each node of the computation tree, with identity wires carrying forward input coordinates and intermediate values between blocks.

5. **Five structural assumptions (A1)-(A5):** (A1) finite tree, (A2) compositional sparsity, (A3) Lipschitz operations, (A4) quantitative range boundedness (established inductively via Lemma 5.1), (A5) primitive-block existence on appropriate domains.

6. **Proof by structural induction on the computation tree:** The main theorem is proved by induction on N = total_nodes(f), using an Error Composition Lemma that bounds how approximation errors propagate through the tree via partial Lipschitz constants.

7. **Partial vs. joint Lipschitz constants:** The paper carefully distinguishes partial Lipschitz constants (variation in a single argument) from joint Lipschitz constants, which is critical for obtaining tight bounds. For example, multiplication on [0,1]^2 has partial Lipschitz constants Lip_1 = Lip_2 = 1 (versus joint constant sqrt(2)).

## Results & Highlights

- **P(KAN) <= 1 for standard operations:** For Op = {+,-,*,sin,cos} on [0,1]-bounded inputs, the layer-wise Lipschitz product is bounded by 1, independently of both input dimension n and tree size N. This is a remarkably tight result.

- **Linear width:** Layer widths are O(n + N), neither quadratic nor exponential in the number of variables, improving upon Sprecher's classical construction which requires Lipschitz constants scaling with n.

- **Optimal B-spline rates:** For f in C^m, the approximation achieves ||f - KAN_f|| = O(N * G^{-(k+1)}) for k <= m-1, matching the optimal single-node B-spline rate up to the multiplicative factor N.

- **Numerical verification:** Experiments confirm P(KAN) = 1.0 exactly for several test functions (xy, sin(xy), x1*...*xn). B-spline error scaling confirms O(G^{-4}) for cubic splines, with approximately constant error/h^4 ratios across two orders of magnitude in G.

- **Range bounds:** B_f <= N+1 for purely additive trees (tight at x = (1,...,1)), B_f = 1 for purely multiplicative/trigonometric trees, verified numerically on 200,000 sample points and 1000 random trees of depth <= 5.

- **Extension to non-differentiable operations:** Corollary 4.6 extends the framework to ReLU-like operations (max(.,0), |.|) via Rademacher's theorem, with Lambda_op,D = 1 for these piecewise-linear operations.

- **Jacobian lower bound:** Proposition 4.8 provides a lower bound P(KAN) >= ||J_f(x)||_2 / W^L, establishing a connection between the Lipschitz product and the function's intrinsic Jacobian structure.

## Critical Assessment

- **Novelty:** The paper makes a genuine contribution by introducing P(KAN) as the right quantity to track in deep KAN stacks and proving it can be controlled independently of input dimension n. The distinction between partial and joint Lipschitz constants, and the domain-sensitive analysis, are technically non-trivial. However, the result applies only under the compositional sparsity assumption (A2), which significantly restricts the class of functions covered. The result is best understood as a structural theorem for a specific function class rather than a general KAN training guarantee.

- **Rigor:** The proofs are detailed and careful. Assumptions are clearly stated and explicitly verified for the standard operation set. The induction is well-structured. The Error Composition Lemma cleanly separates error sources. The author is transparent about what is proved versus what remains open (Section 7.2 lists open questions including compositional sparsity verification and width lower bounds). The Lean 4 formalization sketch, while not completed, demonstrates serious intent toward formal verification.

- **Reproducibility:** The theoretical results should be straightforward to verify given the detailed proof structure. The numerical experiments are described as "sanity checks" rather than comprehensive evaluations, with specific parameters provided (cubic B-splines, grid sizes G = 5,12,35). However, no code is referenced or linked, which would have improved reproducibility of the experimental component.

- **Significance:** The paper addresses a specific and legitimate gap identified in the original KAN paper (Lipschitz control in deep stacks). The result is relevant to anyone working on deep KAN architectures, particularly for symbolic or physically-structured functions. The practical impact is moderate: the construction is constructive but not necessarily efficient for training, and the compositional sparsity assumption limits generality.

- **Red flags:** (1) The paper is single-authored with no institutional affiliation visible, which is somewhat unusual for a 15-page mathematical note of this level. (2) The experiments are minimal -- only four function families tested, and the paper explicitly states they are "illustrative sanity checks" not a comprehensive evaluation. (3) The compositional sparsity assumption (A2), while reasonable for symbolic expressions, is noted by the author to be "computationally hard" to verify in general. (4) No comparison with actual KAN training dynamics -- the paper constructs a specific KAN representation rather than analyzing what happens during gradient-based training.

## Strengths

- Clean, well-defined mathematical framework with clear separation between architectural quantities (P(KAN)) and ambient quantities (Lip(KAN))
- The main theorem (Theorem 4.3) is a genuine structural result with a tight bound (P(KAN) <= 1) for the standard operation set
- Careful handling of partial Lipschitz constants yields sharper bounds than naive joint Lipschitz analysis
- Constructive proof with explicit primitive block designs for each operation
- Transparent about limitations and open questions
- Extension to non-differentiable operations (ReLU-like) demonstrates generality of the framework
- Domain-sensitive analysis that adapts bounds to the actual input ranges at each node

## Limitations

- The compositional sparsity assumption (A2) is a strong restriction; the paper does not cover arbitrary continuous functions (unlike the original Kolmogorov-Arnold theorem)
- The primitive block for multiplication requires three B-spline layers via the algebraic identity u*v = ((u+v)^2 - (u-v)^2)/4, which may not be efficiently learnable by gradient descent
- No analysis of training dynamics -- the paper constructs a specific representation but does not address whether gradient-based KAN training converges to representations with controlled Lipschitz products
- For mixed +/- and x trees where addition feeds into multiplication, the effective C_Op can exceed 1, and the bounds become less tight
- The experiments are limited to synthetic sanity checks with no real-world function approximation tasks
- The paper does not address computational complexity of the construction or compare with alternative architectures (MLPs, etc.) on Lipschitz control
- No code availability mentioned

## Questions for Further Exploration

- Can the compositional sparsity assumption (A2) be relaxed or automatically detected from data, making the framework applicable to learned (rather than known) computation trees?
- Does gradient-based KAN training naturally discover representations with small Lipschitz products, or does the constructive representation in this paper differ significantly from trained representations?
- How does the block-based sequential construction compare in practice (accuracy, training time, generalization) with standard KAN training on compositionally structured tasks?
- Can the framework be extended to vector-valued functions f: [0,1]^n -> R^m, and does P(KAN) remain well-controlled?
- Is the bound P(KAN) <= 1 tight for all compositionally sparse functions, or can strictly smaller values be achieved with alternative primitive block constructions?
- What is the practical effect of the domain-sensitive Lipschitz bounds when training KANs on physical or scientific computing tasks where compositional structure is present but not explicitly known?
