Quantamination: Dynamic Quantization Leaks Your Data Across the Batch

arXiv: 2604.26505v1

Date: 2026-04-29

https://arxiv.org/abs/2604.26505v1

一句话总结： 本文揭示了一种名为"Quantamination"的隐私攻击：在批量推理中使用逐张量动态量化时，攻击者可利用共享的量化参数作为侧信道，以99.6%-100%的准确率恢复同批次其他用户的LLM输入token，并在图像分类任务中实现精确图像识别。
TL;DR: This paper exposes "Quantamination," a cross-batch data leakage side channel in per-tensor dynamic activation quantization. When multiple users' inputs are batched together during inference, shared quantization parameters (scale/zero-point) create an observable side channel. The authors demonstrate 99.6-100% token recovery accuracy on LLMs and near-perfect exact image identification in classification tasks, affecting at least 4 major ML frameworks including vLLM's default FP8 mode, SGLang, ONNX Runtime, and PyTorch. Per-token dynamic quantization eliminates the vulnerability entirely.

核心贡献

  • 发现动态量化侧信道漏洞：首次系统性地揭示批量推理中逐张量(per-tensor)动态量化存在跨用户数据泄露风险，将其命名为"Quantamination"（量化污染）。
  • LLM token恢复攻击：提出逐token迭代恢复算法，利用语言模型先验大幅缩小搜索空间，在三个模型上实现99.6%-100%的token恢复准确率。
  • 分类模型数据泄露攻击：在MNIST分类任务中，当候选集包含目标图像时，实现近乎100%的精确识别；即使目标不在候选集中，量化尺度特征也展现出类别聚类效应。
  • 框架漏洞面调研：系统调查了vLLM、SGLang、ONNX Runtime、PyTorch等4个主流框架的动态量化配置，明确指出哪些默认配置存在泄露风险。
  • 明确缓解方案：指出逐token(per-token)动态量化完全消除此侧信道，且在精度和效率上也优于逐张量方法。

Key Contributions

  • Discovery of the Quantamination side channel: Identifies that per-tensor dynamic activation quantization in batched inference creates a cross-user information leakage channel, where shared scale/zero-point parameters are jointly determined by all inputs in the batch.
  • LLM token recovery attack with 99.6-100% accuracy: Develops an iterative token-by-token recovery algorithm that exploits language model priors to narrow the search space from ~25,000 (random) to as few as 424 queries per token, tested on TinyStories-1M, Pythia-70M, and SmolLM2-135M.
  • Classification setting attack: Demonstrates near-perfect exact image identification on MNIST when the target exists in the candidate set (100% for ResNet18/ResNet50), and provides evidence of class-level quantization signature clustering even when the target is excluded.
  • Framework vulnerability survey: Systematically catalogs dynamic quantization configurations across vLLM (default FP8 online), SGLang (fp8dq-per_tensor), ONNX Runtime (quantize_dynamic), and PyTorch (PerTensor observer), identifying which default settings are vulnerable.
  • Clear mitigation path: Establishes that per-token dynamic quantization eliminates the side channel entirely and is also better for accuracy and computational efficiency on modern GPUs, making it a superior choice in every dimension.

Methodology

The attack exploits per-tensor dynamic activation quantization, where a single scale factor is computed from the global min/max across the entire batch tensor. When an adversary's input is co-located with a victim's input in the same batch, the victim's activations influence the quantization parameters applied to the adversary's activations, producing measurable perturbations in the adversary's output logits.

LLM Token Recovery: The adversary iteratively recovers the victim's token sequence during the decode phase. For each position, the adversary tests candidate tokens by computing the L2 distance between logits observed when batched with the victim versus batched with each candidate. Language model priors (token probability given recovered context) are used to rank candidates, dramatically reducing the search space. Early termination occurs when the distance falls below a threshold (epsilon = 1e-6). Both white-box (local model) and black-box (API-only) settings are considered.

Classification Attack: For image classification, the adversary computes a similarity score between logits observed with the victim versus each candidate image. In the white-box setting, the adversary optimizes probe selection by choosing inputs with small activation ranges across layers. In the black-box setting, random probes are used.

Threat Model: The adversary assumes batch co-location (natural in multi-tenant serving), output observation (top-1 logprobs, available via APIs like OpenRouter), candidate set access (vocabulary for LLMs, public datasets for classification), and model knowledge. Both white-box and black-box access are evaluated.

Results & Highlights

  • LLM token recovery: 99.6-100% accuracy across all three tested models (TinyStories-1M, Pythia-70M, SmolLM2-135M). In-distribution recovery requires as few as 424 queries per token (59x faster than random search). Even out-of-distribution recovery achieves 18-25x speedup over random search on general-purpose models.
  • Context-based acceleration: The first token (no context) requires ~5,000-7,000 queries, but subsequent positions leverage accumulated context to reduce to 20-65 queries per token for in-distribution data.
  • Classification exact recovery: 100% success rate for ResNet18 and ResNet50 on MNIST when the target exists in the candidate set. Even random probes achieve 100% for deeper networks. The simple 3-layer CNN achieves 80-100% depending on probe strategy.
  • Class inference without exact match: When the target is excluded from the candidate set, top-1 accuracy degrades to 10-16% (marginally above 10% random baseline), demonstrating a fundamental information asymmetry between direct scale-profile observation and indirect logit-based inference.
  • Non-determinism in production: Empirical analysis of OpenRouter reveals non-trivial variation in log probabilities across 20 identical queries at temperature 0, highlighting practical challenges for real-world exploitation.

Critical Assessment

Novelty: High. While batch-based privacy attacks exist (MoE routing, KV cache sharing), this is the first to exploit dynamic quantization parameters as a side channel. The attack requires no architectural modification, no cache sharing, and no MoE routing -- it exploits a fundamental property of how quantization parameters are computed.

Rigor: Strong methodology with well-defined threat models, both white-box and black-box settings, multiple model architectures, and honest reporting of failure cases (e.g., class inference without exact match achieves only marginal improvement over random). The 4-hour timeout and reporting of incomplete runs adds transparency.

Reproducibility: Good. The attack setup is clearly described with specific equations, thresholds (epsilon = 1e-6), and framework configurations. However, no code is referenced as available, and the paper tests on relatively small models (up to 135M parameters). Scaling to production-scale LLMs remains untested.

Significance: High practical relevance given the widespread use of dynamic quantization in production serving systems. The finding that per-token quantization eliminates the vulnerability while also improving accuracy and efficiency is a clean, actionable takeaway. However, several caveats temper the real-world impact: (1) the attack is demonstrated only on small models with batch size 2; (2) production non-determinism may significantly degrade signal quality; (3) the black-box LLM attack requires greedy decoding, which may not hold in practice.

Red flags: The gap between lab conditions (small models, batch size 2, controlled environment) and production deployment is significant. The class inference attack without exact match essentially fails (10-16% vs 10% random), suggesting the practical threat may be narrower than the framing implies. The paper does not test on any model larger than 135M parameters, leaving open questions about whether the signal survives in deeper, more complex architectures with more layers potentially causing cancellation effects.

Strengths

  • Identifies a genuinely novel side channel that arises from a common, seemingly innocuous optimization technique.
  • Comprehensive framework survey with a clear vulnerability taxonomy table.
  • Clean mitigation recommendation (per-token quantization) that is already the standard for INT8 in most LLM serving frameworks.
  • Honest discussion of real-world deployment challenges including non-determinism and provider-side sampling variations.
  • Strong experimental results on the primary attack (LLM token recovery), with near-perfect accuracy.
  • The information asymmetry analysis explaining why class inference fails despite scale-profile clustering shows intellectual honesty.

Limitations

  • All experiments use small models (1M-135M parameters) with batch size 2; no validation on production-scale LLMs or realistic batch sizes.
  • The classification attack without exact match is effectively a negative result (10-16% vs 10% random baseline), limiting the generality of the classification threat.
  • The black-box LLM attack requires greedy decoding, which is difficult to verify in practice and may not hold for many API providers.
  • Production non-determinism from hardware heterogeneity, custom sampling pipelines, and provider-specific configurations may substantially degrade the attack signal.
  • No discussion of potential software-level defenses beyond switching to per-token quantization (e.g., noise injection, parameter clipping).
  • The paper does not explore whether the attack scales to batch sizes larger than 2, though it briefly argues the adversary can submit N-1 identical items to effectively reduce to a 2-item batch.

Questions for Further Exploration

  • How does the attack scale to production-sized models (7B-70B parameters) with realistic batch sizes (8-64)?
  • Can noise injection or randomized quantization parameter perturbation serve as practical defenses without significantly degrading inference accuracy?
  • Does the signal survive mixed-precision inference pipelines where only some layers use dynamic quantization?
  • Could adversarial detection mechanisms identify when a co-located user is conducting this attack based on their input patterns?
  • How does the attack interact with other emerging inference optimizations such as speculative decoding or KV cache compression?
