> **一句话总结：** 本文提出"层隔离评估"框架，将生产级F&B点餐LLM代理分解为23个测试切片（8层架构），以无需调用LLM的纯确定性模式（2.39秒完成238个用例）进行CI回归检测，并通过人工注入7层故障验证：聚合指标仅下降1.7–5.9 pp，而对应层切片崩溃25–91 pp，精准实现了故障定位。

> **TL;DR:** This paper presents layer-isolated evaluation — a deployed food-and-beverage ordering LLM agent is decomposed into a fixed eight-layer taxonomy (ontology, intent, routing, decomposition, escalation, safety, memory, and cross-cutting envelope/defense), each exercised by its own assertion slice in a deterministic, no-LLM "pure" mode running 225 cases in 2.39 s for CI gating on every PR. Controlled regression injection across seven layers reveals a masking phenomenon: aggregate pass-rate drops only 1.7–5.9 pp while the responsible slice craters 25–91 pp, with the injected layer's slice ranking worst-hit in 5/7 cases (mean rank 1.29/19). Results replicate on a second structurally different tenant (Starbucks SG).

## 核心贡献

- **层隔离测试分类法**：将生产级LLM代理分解为8层（L0本体解析/意图/语音行为，L2工具路由，L3分解/升级/约束，L4安全/知识/记忆，加跨层envelope、defense等），共23个测试切片，每切片对应一个层的确定性合约
- **无LLM纯模式**：所有切片在不调用任何LLM的纯确定性模式下运行；225个用例2.39秒完成（约10毫秒/用例），零token成本，可在每次PR时门控
- **覆盖诚信准则**：空切片（零用例）报告为`UNCOVERED`/`null`而非100%通过率，防止绿色聚合指标掩盖未测试层；当前4个切片明确标记为未覆盖
- **回归注入验证实验**：针对7个非安全层各自注入单层故障，量化聚合掩盖（aggregate仅降1.7–5.9 pp）与切片精准崩溃（25–91 pp）的对比，注入层切片在7/7中均为Top-3，5/7为最差
- **跨租户验证**：在星巴克新加坡（不同目录、语区、改装模型）上复现全部7次注入，所有匹配切片崩溃，确认不是单一目录的人工制品

## Key Contributions

- **Fixed layer taxonomy and deterministic no-LLM harness**: 238 cases across 23 slices run in ≈2.4 s, gating CI on every change against a locked per-slice baseline (§3)
- **Regression-injection validation over seven layers**: aggregate barely moves (−1.7 to −5.9 pp for six local regressions; −26.5 pp for the foundational ontology regression) while responsible slice craters (−25 to −91 pp local; −95 pp foundational); injected layer's slice is worst-hit in 5/7 and top-3 in 7/7 at mean rank 1.29 of 19 (§4)
- **Cross-tenant replication (§4.1)**: all seven injections reproduced on a structurally different second tenant (Starbucks SG), confirming generalizability beyond a single catalog
- **Coverage-honest baseline**: zero-case slices reported as `null`, never 1.0 — a green aggregate cannot launder an unexercised layer; four slices currently flagged UNCOVERED
- **Honest positioning**: operationalizes EDDOps (Xia et al., 2024) prescriptions, descends from CheckList behavioral testing (Ribeiro et al., 2020), and is the deterministic mirror of AgentAssay stochastic mutation testing (Bhardwaj, 2026)

## Methodology

**Agent decomposition.** The target is a multi-turn food-and-beverage ordering agent built on PydanticAI. It is decomposed into a fixed taxonomy of architectural layers mapping the full request lifecycle:
- **L0**: ontology pre-resolution, intent signals, speech-act classification
- **L2**: tool routing (currently UNCOVERED)
- **L3**: sub-goal decomposition, constraint handling (CSP), escalation decisions
- **L4**: safety (price/SKU/allergen), knowledge (FAQ bypass/slug), health/compliance, memory, personalization, reflexion (last three UNCOVERED)
- **Cross-cutting**: envelope rendering, defense/scan, OOD-reject, reformulator, recommend_rules, read_tools, session_init, locale_fidelity, L1_legacy

**Pure mode.** Each slice asserts a single layer's deterministic contract — the ontology resolver's canonical IDs, the rule-based escalation decision, the reformulator's dictionary rewrite, the OOD short-circuit predicate, the server-side reprice, the prompt envelope's rendered blocks — with no LLM call. Cases take ≈1 ms each and are fully reproducible. The suite reports "225 passed, 30 skipped" (the 30 skipped are live-only variants requiring a real model call, excluded from the pure runner but tracked in the locked 238-case baseline).

**Locked, coverage-honest baseline.** A frozen JSON records per-slice `(total, passed, rate, failed_ids)`. A slice with zero cases is `rate: null` — never 1.0. Any PR whose per-slice rate drops below the locked baseline is blocked. Four slices are currently UNCOVERED (L2_routing, L4_memory, L4_personalization, L4_reflexion); two are low-N (L4_health N=3, locale_fidelity N=1) flagged for expansion.

**Regression injection experiment.** For each of seven non-safety layers, a monkeypatch degrades exactly one entry point; the harness self-verifies the patch took effect (no-op patches are dropped). Each injection runs the full pure suite on a *freshly constructed* runtime per pass (critical: reusing state leaked order-store cross-pass, creating phantom coupling — the fresh-per-pass discipline removes that artifact). The experiment compares aggregate pass-rate delta vs. per-slice delta and ranks the injected layer's slice among all 19 covered slices by severity.

**Cost comparison.** A single live multi-turn episode takes median 73 s (p95 192 s) vs. the entire pure suite at 2.39 s. The pure suite is cheaper than a single live episode, enabling per-PR CI gating rather than nightly-only runs.

**Real regression case study (§6).** A production incident with the confirmation gate (agent swung from occasionally placing unconfirmed orders to frequently failing to place confirmed ones) was caught only by exhaustive human review — the aggregate live-judge signal was 0% defect rate on a 100-round batch that contained 23 real defects. The relevant contract lives in the routing/guardrail layer (L2_routing), which is currently one of four UNCOVERED slices. Coverage-honesty would have surfaced this gap rather than silently passing it green.

## Results & Highlights

**Masking quantified (Table 3):**

| Injected regression | Agg. Δ | Slice Δ | Rank |
|---|---|---|---|
| escalation → never escalate | −4.62 pp | −50.00 pp | 1/19 |
| intent signals → ∅ | −4.20 pp | −25.00 pp | 2/19 |
| defense scan → allow-all | −5.04 pp | −63.16 pp | 1/19 |
| OOD gate → never reject | −1.68 pp | −36.36 pp | 1/19 |
| reformulator → identity | −5.88 pp | −80.00 pp | 1/19 |
| decomposer → no sub-goals | −5.88 pp | −90.91 pp | 1/19 |
| ontology resolve → ∅ (foundational) | −26.47 pp | −95.24 pp | 2/19 |

- The ontology regression is *foundational*: it feeds many downstream layers, so 9 slices move (blast radius = 9); this wide radius vs. the single-slice pattern for local regressions is itself a useful diagnostic signal distinguishing local from foundational faults
- **Off-diagonal flatness** (Figure 1): the heatmap is nearly flat except for the injected layer's column and, for foundational layers, their downstream dependents — the fault does not smear across other slices
- **Cross-tenant (Table 4)**: all seven matching slices crater on Starbucks SG (−50 to −100 pp); for six of seven the off-diagonal is clean; the one exception (ontology fault also craters L3_decompose on both tenants) is the same foundational signature

**Cost (Table 5):**
- Pure suite: ≈10 ms/case, $0, fully deterministic, runs every PR
- Live agent: median 73 s/episode, per-turn LLM cost, stochastic, nightly/release

## Critical Assessment

**Novelty**: Moderate and honestly self-assessed. The authors explicitly do not claim to have invented component-level evaluation. The contribution is a *concrete, deterministic, runnable instantiation* of a prescribed process model (EDDOps) for an actual deployed agent, plus the empirical masking demonstration. The layer taxonomy, pure-mode engineering discipline, coverage-honesty criterion, and injection experiment design are genuine practical contributions within the MLOps/agent-reliability space.

**Rigor**: Good for an engineering/systems paper. The regression injection experiment is methodologically careful: fresh-per-pass runtimes (the paper discloses the confound they caught and corrected), self-verified patch effect, rank statistics quantifying localization, and transparent acknowledgment of what is by-construction vs. measured. The threats-to-validity section (§8) is unusually honest: construct validity concerns (per-slice rate is near-binary at small N, low-N slices flagged), internal validity (off-diagonal flatness plus mechanistic explanation of rank-2 cases), external validity (one agent framework, one domain, author-chosen injection types), and conclusion validity (N=7 is exact, not sampled, so it threatens fault-type generality not measurement reliability).

**Reproducibility**: Constrained but honest. All numbers trace to specific scripts in their repository; the pure harness is self-contained, deterministic, and reviewers could re-run the injection table in seconds — if they had access to the codebase. The underlying agent is proprietary and not publicly released. The authors flag this toward an ICSE Artifacts Reusable evaluation rather than overclaiming open-source reproducibility.

**Significance**: High practical significance for engineering teams building production LLM agents who rely on end-to-end metrics for regression detection. The masking phenomenon is concretely measured and likely general. The coverage-honesty criterion (null vs. 100% for unexercised layers) is a simple but important discipline. The real regression case study (§6) grounds the abstract argument in a production incident.

**Red flags**: (1) Results are from a single agent system at a single company; the taxonomy is specific to F&B ordering. Multi-domain generalization is stated as future work. (2) The injection study covers only 7 author-chosen fault types — organic, post-freeze regressions remain the load-bearing open item and are explicitly called out as not yet demonstrated. (3) Four layers remain UNCOVERED, including the routing/guardrail layer that was implicated in the real production incident — the tool is only as strong as its covered set. (4) The LLM-generative layers (multi-turn reasoning, free-form brand voice) are explicitly out of scope for the pure lane, meaning the hardest evaluation problems remain unaddressed. (5) Companion preprint cross-citation creates a shared-batch exposure that the authors themselves flag as a potential weakness.

## Strengths

- Unusually honest self-assessment throughout — authors are explicit about what is by-construction, what is measured, what is out of scope, and what the open items are
- Concrete engineering contribution: the fresh-per-pass discipline, coverage-honesty criterion, and CI gate design are replicable ideas even without the proprietary codebase
- The masking phenomenon is quantified precisely (1.7–5.9 pp aggregate vs. 25–91 pp slice drop) with a fault-localization rank metric borrowed from the SE literature
- The real production incident (§6) makes the practical stakes concrete and validates the motivation beyond synthetic injection
- The cost argument (2.39 s pure suite vs. 73 s median live episode) is compelling for CI placement
- The off-diagonal flatness is clearly visualized (Figure 1 heatmap) and its two exceptions (foundational layers) are mechanistically explained

## Limitations

- Proprietary system: the agent, slices, and catalog are not publicly released; independent validation is limited to methodology
- Scope bounded to deterministic scaffold layers — the generative LLM components (reasoning quality, free-form output) require the live lane and are out of the pure gate's reach
- Taxonomy is F&B-ordering-specific; generalization to other agent architectures (code agents, open-ended assistants, multi-agent systems) requires additional instantiation
- N=7 injection experiment covers only author-chosen fault types; organic regression localization is the stated open item and is not yet demonstrated
- Four layers currently UNCOVERED, including the routing/guardrail layer that was implicated in the real production incident
- Single agent framework (PydanticAI); cross-framework portability is not evaluated

## Relevance

本文对关注LLM代理工程化可靠性的研究者极具参考价值：其"无LLM纯模式"CI门控设计和覆盖诚信准则直接解决了生产部署中端到端指标无法定位层级回归的核心痛点，是agent评估方法论从学术基准走向工程实践的具体范例。

## Questions for Further Exploration

1. Can the layer taxonomy generalize to code agents (e.g., SWE-bench-style) or open-ended assistants, or does it require significant restructuring for non-task-specific agents?
2. What is the right way to gate the LLM-generative layers that fall outside the deterministic pure lane — can lightweight surrogate models or golden-output snapshots extend the pure-mode idea to stochastic components?
3. The authors identify the routing/guardrail layer (L2_routing) as UNCOVERED and implicated in a real production regression. What is the minimum case count needed for a coverage-honest slice on an inherently combinatorial routing layer?
4. How does the per-slice baseline-locked gate compose with continuous deployment practices where behavior intentionally shifts? Is the "drift detector not a correctness oracle" framing sufficient, or does it require a separate mechanism for planned behavioral changes?
5. Could the layer taxonomy and injection methodology be automated (e.g., using LLM-assisted contract inference) to reduce the manual effort of authoring pure-mode slices for new agents?
