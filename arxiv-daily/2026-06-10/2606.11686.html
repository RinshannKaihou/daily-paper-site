Layer-Isolated Evaluation: Gating the Deterministic Scaffold of a Production LLM Agent with a No-LLM, Regression-Locked Test Harness

Sawyer Zhang Alexander Wang Sophie Lei · Lumivate (Lumi) · Practitioner / applied MLOps
llm-agents software-testing evaluation-methodology regression-localization agent-reliability
Key metric
25–91pp
Slice-level pass-rate drop on injected regressions, while the aggregate moved only 1.7–5.9 pp — a roughly 10× localization signal hidden by end-to-end metrics.
Why this matters

End-to-end production metrics cannot localize which architectural layer broke. This moves agent evaluation from academic benchmarks into a per-PR CI gate.

Critical edge

Four layers remain UNCOVERED — including L2_routing, the exact layer implicated in a real production incident.

Methodology § 03

Target: a multi-turn F&B ordering agent (PydanticAI), decomposed into a fixed taxonomy. Every slice asserts one layer's deterministic contract — no LLM call, ≈10 ms/case, fully reproducible.

L0Ontology, intent, speech-act
L2Tool routing UNCOVERED
L3Decompose / escalate / constraint
L4Safety, knowledge, memory, reflexion
Locked baselinePer-slice JSON; zero-case = null, never 1.0
Fresh-per-passNew runtime per injection (avoids state leak)

A single live episode costs median 73 s (p95 192 s) — the pure suite at 2.39 s is cheaper than one live turn, which is what unlocks per-PR CI gating rather than nightly runs.

Strengths § 07

  • Unusually honest self-assessment — explicit about by-construction vs. measured, out of scope, and open items.
  • Concrete engineering contribution: fresh-per-pass discipline, coverage-honesty criterion, and CI gate design are replicable without the proprietary codebase.
  • Masking phenomenon quantified precisely with a fault-localization rank metric borrowed from SE literature.
  • Real production incident (§ 6) grounds the synthetic injection argument in an actual failure the harness would have caught.
  • Off-diagonal flatness clearly visualized (Figure 1); the two exceptions (foundational layers) are mechanistically explained.

Limitations § 08

  • Proprietary system: agent, slices, and catalog are not publicly released — independent validation is limited to methodology.
  • Scope bounded to the deterministic scaffold; LLM-generative components (reasoning, free-form voice) require the live lane and are out of reach.
  • Taxonomy is F&B-ordering-specific; code agents, open-ended assistants, and multi-agent systems need re-instantiation.
  • N = 7 injection experiment covers only author-chosen fault types; organic regression localization is the stated open item.
  • Four layers currently UNCOVERED, including the routing/guardrail layer implicated in the real production incident.
  • Single agent framework (PydanticAI); cross-framework portability is not evaluated.

Questions for further exploration § 09

  1. Can the layer taxonomy generalize to code agents (SWE-bench-style) or open-ended assistants, or does it require significant restructuring for non-task-specific agents?
  2. What is the right way to gate the LLM-generative layers that fall outside the deterministic pure lane — can lightweight surrogates or golden-output snapshots extend the pure-mode idea to stochastic components?
  3. The authors flag L2_routing as UNCOVERED and implicated in a real regression. What is the minimum case count needed for a coverage-honest slice on an inherently combinatorial routing layer?
  4. How does the per-slice baseline-locked gate compose with continuous deployment where behavior intentionally shifts? Is "drift detector, not a correctness oracle" sufficient, or does it need a separate mechanism for planned changes?
  5. Could the layer taxonomy and injection methodology be automated (e.g., LLM-assisted contract inference) to reduce the manual effort of authoring pure-mode slices for new agents?
