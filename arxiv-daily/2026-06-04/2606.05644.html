Methodology

  • **Stubborn hallucination**：RAG 中检索段落与参数记忆矛盾时，模型仍复述记忆答案，违背「证据应覆盖参数知识」的部署预期。
  • **对比解码基线**：放大 `p(y|x,d)` 相对 `p(y|x)`，抑制参数偏置；既有方法用固定或**响应级**权重，隐含偏置在序列上近似均匀。

Results & Highlights

  • **12/12** 设定 CF 最高；最强无训练基线恒为 AdaCAD，FIDES 仍 **+3~+13** CF，**+14~+28** vs Standard RAG。
  • **EM/F1 同向提升**：如 Qwen3-8B/NQ-Swap F1 **55.27** vs AdaCAD **46.49**；LLaMA3-8B/NQ-Swap F1 **53.84** vs AdaCAD **43.35**。
  • **统计显著性**：48 组配对 bootstrap（n=400 CTX，B=1000）全部 **p<0.05**，44/48 **p<0.01**；Bonferroni 校正后多数 PopQA/TriviaQA 仍显著。
  • 仅 Opposition → 逐步加 Shift/Noise → **Full(3)** 在 CF–F1 上最优；去掉任一信号均降 CF（Appendix Table 11）；固定 α=1.0 主要伤害 **EM**。

Strengths

  • **原则性解决全局对比过/欠矫正**：答案 token 3.3× 权重、AUROC 0.923 支撑「where」叙事。
  • **训练免费、部署友好**：无需微调或改权重，标定一次跨数据集/骨干通用。
  • **忠实度与效用兼得**：相对 AdaCAD 同时提升 CF、EM、F1，70B 上 F1 跃升显著。
  • **信号可解释且互补**：JSD + 隐状态 + Logit-Lens 级联，消融与 17 组权重网格证明非单信号取巧。

Limitations

  • **双前向结构性成本**：每解码步两次完整前向，约为 Standard RAG **2×**，大规模 serving 需工程优化（推测/缓存未讨论）。
  • **任务边界**：优化**文档忠实**（follow retrieved passage under conflict），**不**保证事实正确；错误检索、对抗编辑会被放大。
  • **架构与语言**：融合标定基于稠密 Transformer 7B–8B；**MoE、SSM 未验证**。
  • **场景**：单文档英文实体级反事实 QA；多文档、跨语言、多模态 RAG 留作未来工作。

TL;DR

When retrieved context contradicts parametric memory, LLMs often ignore evidence ("stubborn hallucination"). Prior contrastive decoders (CAD, AdaCAD, DeCoRe, DVD, COIECD) assume **uniform** parametric bias and apply a **global** contrastive weight—over-penalizing safe tokens while under-correcting conflict-bearing ones. **FIDES** reframes the problem as **where** to contrast: at each step t, dual forward passes (context path `[d;x;y_{<t}]` vs no-context `[x;y_{<t}]`) yield three depth-complementary signals fused into **FScore_t**, mapped to token-specific **α_t** via `z_final = (1+α_t)z_ctx − α_t z_noctx`. Signals: **Opposition** = JSD(p_ctx∥p_noctx); **Shift** = mean-layer normalized ℓ₂ distance (scaled 1/10); **Noise** = KL(midpoint vs final Logit-Lens on context path (scaled 1/5).
