> **一句话总结：** 本文提出 PRECISE 框架，将 Prediction-Powered Inference（PPI++）扩展到排序评估指标：用少量人工标注校正大量 LLM 评判的系统性偏差，在理论上保证无偏的同时显著降低 Precision@K 等层次化指标的估计方差，并在 ESCI 基准与生产 A/B 实验中验证了其实用价值。

> **TL;DR:** This arXiv note presents PRECISE, which extends Prediction-Powered Inference (PPI++) to bias-corrected estimation of ranking metrics (e.g., Precision@K) by combining a small human-labeled gold set with a large LLM-judged unlabeled set. A sparse output-space reformulation makes hierarchical metrics tractable (O(2^K) vs. O(2^|C|)). On ESCI, 30 human labels plus Claude 3 Sonnet judgments cut Precision@4 standard error by 21%; in production, the framework correctly ranked three search variants from 100 human labels and 2 hours of expert annotation, later confirmed by A/B tests (+407 bps daily sales for the top variant).

## 核心贡献

- 将 **Prediction-Powered Inference（PPI++）** 从一般半监督估计推广到**排序评估指标**，在 LLM 评判存在任意系统性偏差时仍可获得**理论无偏**的指标估计
- 针对 **Precision@K** 等层次化指标（标注粒度为 query–document，指标粒度为 query），提出**稀疏输出空间重构**，将朴素 PPI 所需的 O(2^|C|) 计算降至 **O(2^K)**，使百万级语料库上的方法可 tractable
- 在 **ESCI** 检索基准上验证：30 条人工标注 + 60,000 条 Claude 3 Sonnet 评判，将 Precision@4 标准误从 4.45 降至 3.50（相对降低 21%），偏差亦低于纯人工基线
- 在**生产搜索系统**中，用 100 条人工标签 + 8,400 条 LLM 评判（约 2 小时专家标注）正确识别三个系统变体的优劣排序，并经 A/B 测试确认（最优变体日销售额 +407 bps、CTR +571 bps）
- 与“训练更好评判器”的路线正交：接受 LLM 评判有偏，通过**统计偏差校正**而非 prompt 工程来恢复评估的判别力

## Key Contributions

- Extends **Prediction-Powered Inference (PPI++)** (Angelopoulos et al., 2024) to produce **provably unbiased** estimates of ranking evaluation metrics by combining a small human-labeled gold set (n) with a large LLM-annotated unlabeled set (N ≫ n), regardless of the LLM judge's error profile
- Resolves the **granularity mismatch** for hierarchical metrics like Precision@K—where relevance labels are per-document but the metric is per-query—via a **sparse output-space reformulation** that reduces computation from O(2^|C|) to O(2^K) by collapsing probability mass of non-retrieved documents onto the all-zero vector
- Demonstrates on the **ESCI shopping-queries benchmark** that augmenting n=30 human annotations with N=60,000 Claude 3 Sonnet judgments reduces Precision@4 standard error from 4.45 to 3.50 (21% relative reduction) while maintaining lower bias than gold-only estimation
- Validates in a **production search A/B test** that PPI-corrected Precision@K ranking (T1 > T2 > Control) from 100 human labels and 2 hours of domain-expert annotation was confirmed by live traffic (+407 bps daily sales, +571 bps CTR for T1)
- Takes an **orthogonal approach** to prior LLM-as-a-Judge work: rather than building better judges through prompting or fine-tuning, it statistically corrects for known judge bias while leveraging the unlabeled set to reduce variance

## Methodology

**Problem setting.** Human evaluation of ranking systems is expensive; small labeled sets yield wide confidence intervals that cannot distinguish real improvements from noise. Replacing humans with LLM judges introduces **systematic bias** that distorts metrics when used naively (Chen et al., 2024).

**PPI++ estimator.** Given a gold set D_g = {(x^(i), y^(i))}_{i=1}^n with human labels and an unlabeled set D_u = {x^(i)}_{i=1}^N (N ≫ n) annotated by LLM M, the estimator is:

$$\hat{\mu}_{\text{PPI}} = \underbrace{\frac{1}{N}\sum_{i=1}^{N} \tilde{\mu}_u^{(i)}}_{\text{LLM-based estimate}} + \underbrace{\lambda \left(\frac{1}{n}\sum_{i=1}^{n} \phi_i - \frac{1}{n}\sum_{i=1}^{n} \tilde{\mu}_g^{(i)}\right)}_{\text{bias correction}}$$

where φ_i is the human-grounded metric on the i-th gold query, and μ̃_u, μ̃_g are LLM-based metric estimates obtained by marginalizing over the judge's output distribution. The mixing parameter λ ∈ [0, 1] is tuned to **minimize variance**; the estimator remains **unbiased for any λ > 0**. When the LLM is well-calibrated, λ ≈ 1 lets the full unlabeled set drive variance reduction; when heavily biased, λ ≈ 0 falls back to gold-only estimates.

**Hierarchical metrics (Precision@K).** Naive PPI treats each corpus document as a binary relevance variable, yielding an intractable {0, 1}^|C| output space. The key observation: Precision@K depends only on the **top-K retrieved documents**, expressible as a scaled dot product ŷ^T y / K over sparse vectors. Non-retrieved documents' probability mass collapses onto the all-zero vector, reducing the output space to **{0, 1}^K**.

For each query, the LLM provides per-document relevance probabilities p̃(d_k) for the k-th ranked result. Under **conditional independence** across top-K documents, the joint distribution over K-length binary vectors is:

$$\tilde{p}(\mathbf{y}) = \prod_{k=1}^{K} \tilde{p}(d_k)^{y_k} (1 - \tilde{p}(d_k))^{1 - y_k}$$

LLM-based estimates in Eq. (1) are computed by summing ϕ(ŷ, y) over all 2^K vectors—tractable for typical K ≤ 10.

## Results & Highlights

**ESCI benchmark (Reddy et al., 2022):**
| Estimator | Bias (↓) | Std. Err. (↓) | Cost |
|-----------|----------|---------------|------|
| Gold only (n=30) | 1.04 | 4.45 | — |
| + Claude 3 Sonnet | 0.70 | 3.50 | $946 |
| + Claude 3 Haiku | 0.29 | 3.86 | $79 |
| Sonnet + Haiku ensemble | 0.70 | — | — |

- PPI with Sonnet achieves **21% relative SE reduction** (4.45 → 3.50) with lower bias than gold-only (0.70 vs. 1.04)
- Haiku alone achieves the **lowest bias** (0.29) at **12× lower inference cost**, with comparable SE (3.86)
- Figure 1 shows PPI's sampling distribution is **tighter than gold-only** (variance reduction from LLM signal) and **centered on the true value** (bias correction removes systematic judge error), while LLM-only estimates are visibly biased
- Framework **plateaus at ~100× unlabeled-to-gold ratio**: N=3,000 LLM queries yield nearly identical SE to N=60,000 with n=30 labels

**Production A/B test:**
- Three search system variants (Control, T1, T2) ranked using n=100 human labels + N=8,400 LLM judgments (~2 hours expert annotation)
- Predicted ranking: **T1 > T2 > Control**
- Live A/B confirmed: T1 yielded **+407 bps daily sales** and **+571 bps click-through rate**
- Without PPI correction, **LLM-only estimates could not distinguish variants** because systematic upward bias inflated all estimates similarly; semi-supervised bias correction restored discriminative power

**Generalizability claim:** Though validated on Precision@K, the hierarchical formulation applies in principle to any metric aggregating fine-grained judgments (e.g., per-claim factuality, per-turn dialogue quality).

## Critical Assessment

- **Novelty:** Moderate-to-high for the arXiv note itself, which is largely a condensed presentation of the AAAI 2026 PRECISE paper. The core statistical machinery (PPI++) is from Angelopoulos et al. (2023, 2024); the novel contributions are the application to ranking metrics, the sparse hierarchical reformulation, and production validation. The "accept bias, correct statistically" framing is a useful conceptual contribution relative to the dominant "build better judges" paradigm.
- **Rigor:** The PPI++ framework has strong theoretical guarantees (unbiasedness regardless of judge error profile). The sparse reformulation is mathematically sound for Precision@K. However, this 3-page note provides limited formal treatment—the full AAAI paper likely contains more detail. The conditional independence assumption (Eq. 2) is acknowledged as potentially violated but not empirically tested.
- **Reproducibility:** Moderate. ESCI is public and Claude 3 models are commercially available, but exact prompts, rubrics, and production system details are not specified in this extended abstract. Cost figures ($946 Sonnet, $79 Haiku) aid practical planning but depend on API pricing at evaluation time.
- **Significance:** High for practitioners running search/ranking A/B tests with limited annotation budgets. The production result (+407 bps sales) is compelling evidence of real-world impact. The 100× plateau finding is practically important for cost planning.
- **Red flags:** (1) Single-author arXiv note that is explicitly an extended abstract of a conference paper—readers seeking full methodology should consult the AAAI version. (2) Production experiment lacks full statistical reporting (no confidence intervals on business metrics in this note). (3) Only Precision@K is empirically validated despite broader claims.

## Strengths

- Principled statistical foundation with **provable unbiasedness**, not heuristic debiasing
- Elegant sparse reformulation makes PPI practical for retrieval metrics at scale
- Clear empirical demonstration of **variance reduction without bias introduction** on a public benchmark
- Rare combination of **public benchmark validation and production A/B confirmation** with real business metrics
- Cost-aware analysis comparing Sonnet vs. Haiku vs. ensemble, including the practical **100× plateau** insight
- Honest discussion of limitations (distribution shift, conditional independence, metric scope)
- Ethical considerations addressed: anonymized queries, no crowdworker labor, framework reduces but does not eliminate human annotation

## Limitations

- **Metric scope:** Hierarchical PPI extension validated only on Precision@K; generalization to other hierarchical metrics (per-claim factuality, per-turn dialogue quality) remains untested
- **Conditional independence:** Assumes document relevance judgments are independent within top-K; may fail for diversity-sensitive ranking where one document's relevance depends on others
- **Distribution shift:** Requires gold and unlabeled sets from the same distribution; temporal drift between gold and unlabeled queries could degrade bias correction
- **Extended abstract format:** This arXiv note (3 pages) omits full experimental details, prompts, and ablations available in the AAAI 2026 paper
- **Single production case study:** One A/B test on one search system; generalization to other domains and metric families is not established
- **LLM judge dependency:** Framework quality depends on having access to judge output probabilities (not just binary verdicts); calibration quality affects optimal λ

## Relevance

该论文与关注 **LLM 评估可靠性、排序/检索系统评测、以及半监督统计推断** 的研究与实践高度相关。它将 Prediction-Powered Inference 这一有理论保证的半监督框架落地到「人工标注贵、LLM 评判有偏」这一工业界常见困境，对需要在有限标注预算下做系统对比与 A/B 决策的团队有直接参考价值。稀疏输出空间技巧对任何「细粒度标注、粗粒度指标」的层次化评估场景均有启发意义。

## Questions for Further Exploration

- How does PRECISE perform on metrics beyond Precision@K—e.g., nDCG, MRR, or dialogue-level aggregates like per-turn quality scores?
- Can the conditional independence assumption be relaxed via structured dependency models (e.g., factor graphs over top-K documents) without sacrificing tractability?
- How sensitive is bias correction to **distribution shift** between gold and unlabeled query sets in production (seasonality, query drift)?
- Does combining multiple LLM judges (Sonnet + Haiku ensemble) via multi-objective PPI, as suggested in future work, outperform single-judge PPI++ with tuned λ?
- How does PRECISE compare to **doubly robust estimation** (Oosterhuis, 2023) for online, real-time bias-corrected metric inference?
- What is the minimum gold set size n required for reliable bias correction across different judge quality levels and metric types?
