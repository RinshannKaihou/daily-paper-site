> **一句话总结：** 提出 **UltraVR**——面向超高分辨率图像的**证据驱动视觉推理诊断基准**，覆盖 CCTV、遥感、病理 WSI、工业异常检测四域；每条样本附带结构化 **GT-CoT**（GND/PER/QUA/INT/INF 五步操作标签），在 11 个前沿 VLM 上显示最强模型仅 **44.9%** 宏准确率，瓶颈集中在**证据定位与局部感知**，而非下游推理格式。

> **TL;DR:** UltraVR is a multi-domain diagnostic benchmark for evidence-grounded VQA over ultra-resolution images (CCTV, RS, WSI, AD). Each instance includes structured GT-CoT with step questions, verified intermediate answers, and five operation labels (GND, PER, QUA, INT, INF). Evaluating 11 frontier VLMs under end-to-end QA, visual-evidence access (V0–V4), and process-level GT-CoT protocols (S0–S5) reveals: (1) far-from-saturated performance (best macro acc. 44.9%, mean 32.6%); (2) localized evidence helps but does not solve the bottleneck; (3) oracle intermediate facts (S5) lift mean accuracy to 80.2% (+49.4 vs. S4), localizing failures to early GND/PER rather than generic CoT or language-side inference.

## 核心贡献

1. **UltraVR 多域诊断基准**：在 CCTV（PANDA）、遥感 DOTA 1.5、TCGA-BRCA 乳腺 WSI、MVTec LOCO AD 合成工业网格四类超高分辨率场景上，构建需**视觉证据**而非语言先验/粗场景即可作答的 VQA 任务。

2. **统一推理分类体系**：CCTV/RS/WSI 覆盖 **comparison（比较）**、**logical verification（逻辑核验）**、**relational inference（关系推理）**；AD 覆盖 **logical / structural anomaly（逻辑/结构异常）**；任务要求在大图中搜索证据、感知细节、跨区/跨尺度整合后作答。

3. **结构化 GT-CoT 与五步操作标签**：每条样本含逐步问题、可验证中间答案与操作标签——**GND**（证据定位）、**PER**（局部感知）、**QUA**（量化）、**INT**（证据整合）、**INF**（决策推断），支持过程级诊断而非黑盒准确率。

4. **三维互补评测协议**：
   - **端到端 QA**（S0/V2）：默认全图推理；
   - **视觉证据接入**（V0–V4）：缩略图、全图、局部证据、全图+局部；
   - **GT-CoT 过程评测**（S0–S5）：含 Schema-CoT、Few-shot、逐步预测、GT 前缀 oracle 等受控对比。

5. **系统性瓶颈定位**：11 个专有/开源 VLM 实验表明，错误集中于 **GND 与 PER**；一旦中间视觉事实被 oracle 补全（S5），下游 QUA/INT/INF 往往可恢复；更大视觉 token 预算或更高等效分辨率**不能稳定**预测更好表现。

## Key Contributions

1. **UltraVR benchmark** spanning CCTV, RS, WSI, and AD with domain-specific evidence challenges and visually grounded (non text-only) QA.

2. **Unified reasoning taxonomy** with comparison, logical verification, relational inference, and anomaly reasoning templates tied to domain evidence candidates.

3. **Structured GT-CoT annotations** decomposing the visual-to-decision pipeline into GND, PER, QUA, INT, and INF for cross-domain operation-level diagnosis.

4. **Three evaluation axes**—end-to-end QA, visual-evidence access ablations, and GT-CoT process protocols (S0–S5)—benchmarking proprietary (GPT-5.5/5.4, Gemini-3.1-Pro) and open-weight VLMs.

5. **Diagnostic findings** that ultra-resolution failures are dominated by unreliable visual-fact acquisition (GND/PER), not absent reasoning scaffolds or language-side inference alone.

## Methodology

### 设计原则

1. 问题必须依赖**可定位的视觉证据**，不可仅靠语言先验或粗场景上下文作答；
2. 关键证据体现超高分辨率难点：微小、 subtle、空间远距离或跨区域/跨尺度分布；
3. 每条实例支持**可验证中间事实**的过程级评测。

### 四域数据来源与构造

| 域 | 数据源 | 证据挑战 | 推理类别 |
|----|--------|----------|----------|
| **CCTV** | PANDA 千兆级监控帧 | 密集人群、小目标、遮挡、远距离布局 | 比较 / 逻辑核验 / 关系推理 |
| **RS** | DOTA 1.5 航拍 | 大范围布局中的小目标跨区比较 | 同上 |
| **WSI** | TCGA-BRCA 乳腺全切片 | 多尺度病理形态、ROI 导航 | 同上（专家审核） |
| **AD** | MVTec LOCO AD 合成 10×10→16k 级网格 | 重复模式中罕见逻辑/结构异常 | 逻辑异常 / 结构异常 |

共享流水线：选取图像 → 提取域特定证据候选 → 实例化推理模板 → 标注 GT-CoT → 构造干扰项 → **QC**（视觉可答性、单答案、证据可见性、文本泄漏、GT-CoT 一致性、专家复核）。

### GT-CoT 五步操作（跨域共享）

\[
\text{GND} \rightarrow \text{PER} \rightarrow \text{QUA} \rightarrow \text{INT} \rightarrow \text{INF}
\]

- **GND**：定位对象/区域/网格单元/ROI；
- **PER**：识别局部属性、朝向、形态、缺陷模式；
- **QUA**：计数、差值、比例等数值/序数化；
- **INT**：跨对象/区域/尺度整合；
- **INF**：应用领域规则得出最终四选一答案。

### 评测模型（11 个）

**闭源**：GPT-5.5、GPT-5.4、Gemini-3.1-Pro  
**开源大模型**：Qwen3.5-27B、InternVL3.5-38B、Gemma-4-26B、GLM-4.5V  
**开源小模型**：DeepSeek-VL2、MiniCPM-V 4.5、Phi-4-Reasoning-Vision-15B、Kimi-VL-A3B-Thinking  

全部任务为**四选一**；报告域内/类内准确率及四域**宏平均（Macro Acc.）**。

### 视觉输入条件（V0–V4）

| 条件 | 输入 | 目的 |
|------|------|------|
| V0 | 仅文本（问题+选项） | 语言先验/弃权基线 |
| V1 | 外部下采样缩略图 | 粗全局是否足够 |
| V2 | 原始全图（默认预处理） | 标准端到端 |
| V3 | 域特定局部证据（无全图） | 证据可达性上界 |
| V4 | 全图 + 局部证据 | 全局+局部能否协同 |

局部证据设计：WSI 为 ROI 裁剪；AD 为原 10×10 网格中随机位置的 5×5 原分辨率单元表；CCTV/RS 为象限级裁剪（非答案精确定位裁剪）。

### GT-CoT 推理格式（S0–S5）

| 设置 | 说明 |
|------|------|
| **S0** Direct QA | 全图 + 终局问题，直接预测 |
| **S1** Generic CoT | S0 + 通用「逐步思考」指令 |
| **S2** Schema-CoT | 提供任务感知 GT-CoT 步骤 schema，单次补全全部中间步 |
| **S3** Few-shot GT-CoT | S2 + 完整 GT 轨迹 in-context 示例 |
| **S4** Pred-Step | 按序预测各步，前缀为模型自身预测 |
| **S5** GT-Prefix | 此前中间答案替换为 GT，隔离当前步错误 |

## Results & Highlights

### 端到端评测（S0/V2，Table 1）

| 指标 | 数值 |
|------|------|
| **最强 Macro Acc.** | GPT-5.5 **44.9%** |
| **全模型平均** | **32.6%** |
| **文本-only 平均** | **7.7%**（Visual Gap 平均约 **+24.8**） |
| **弃权率（V0）** | 平均约 **71.6%**（强指令模型常拒绝无图作答） |

**GPT-5.5 分域 Overall**：CCTV 51.8%、RS 44.2%、WSI 52.7%、AD 30.8%（AD 逻辑/结构异常最难）。Gemini-3.1-Pro 在部分 CCTV 子类更高（如 Logic 60.0%），但整体 Macro 41.4%。开源大模型 Macro 约 28–38%；小模型 Kimi-VL 在部分 CCTV Rel. 达 72.5% 但整体 Macro 29.3%。**更大 Max Visual Budget / Max Res. 与准确率无稳定正相关**。

### 视觉证据接入（Table 2，Macro Acc. 示例）

| 模型 | V1 缩略图 | V2 全图 | V3 局部 | V4 全+局部 |
|------|-----------|---------|---------|------------|
| GPT-5.5 | 36.0 | 44.6 | **50.1** | 43.0 |
| Gemini-3.1-Pro | 36.7 | 40.6 | **48.8** | 39.4 |
| 平均 | 27.4 | 30.4 | **33.2** | 26.0 |

**要点**：V2 略优于 V1（全图含超出缩略图的信息但未充分利用）；**V3 对多数模型最强**——默认全图推理受证据获取/压缩/检索限制；V4 不能稳定超过 V2/V3，说明「简单叠加局部证据」非可靠解法。

### GT-CoT 过程评测（Table 3）

| 设置 | 平均 Macro Acc. | 相对 S0 的解读 |
|------|-----------------|----------------|
| S0 Direct | 30.4% | 基线 |
| S1 Generic CoT | 29.9% | 略低于 S0（通用 CoT 无效） |
| S2 Schema-CoT | 30.1% | 与 S1 几乎相同 |
| S3 Few-shot | 30.1% | 与 S2 相同 |
| S4 Pred-Step | 30.8% | 顺序执行收益极小 |
| **S5 GT-Prefix** | **80.2%** | **+49.4** vs S4；oracle 中间视觉事实可大幅恢复 |

**S5 分模型**：GPT-5.5 89.2%、Gemini 89.6%、GLM-4.5V 85.4%、Phi-4-RV 87.4% 等——说明瓶颈在**获取可靠中间视觉状态**，而非缺少显式推理格式。

### 操作级诊断（Figure 2）

- **S2 相对难度**：**PER**  consistently 难于模型平均步误差；INT 等后期操作相对更易；
- **S5 可恢复性**：即使前缀为 GT，**GND/PER 仍弱**；QUA/INT/INF 在前提视觉事实给定后接近满分；
- **S4 首错位置**：多数轨迹**首错于 GND 或 PER**；早期视觉状态正确后，后续轨迹通常可完成。

## Critical Assessment

**新颖性**：将「超高分辨率 VQA」从单一准确率评测推进到**视觉—决策流水线分步诊断**，GT-CoT 作为**标注 schema**（非模型生成 rationales）的设计可复用于其他大图为核心的应用。五操作标签跨四异质域抽象，便于横向比较 GND/PER 瓶颈。

**严谨性**：
- **优点**：三轴评测（端到端 / 证据接入 / 过程）相互印证；S5 oracle 与 S4 滚动预测形成强对照（+49.4 pp）；文本-only 与 Visual Gap 验证视觉依赖性；11 模型覆盖闭源与多档开源；
- **弱点**：benchmark 规模与划分比例主文依赖 Figure 1 而未表格化；V3 局部证据设计（象限级而非答案级裁剪）可能高估/低估不同模型；模型命名（GPT-5.5 等）需与 API 快照版本对齐方可复现。

**诚实度**：Discussion 明确 UltraVR 为**诊断基准**而非全场景覆盖；承认 V4 不稳健、generic CoT 无效；指出未来需更强 visual search、自适应证据选择、多尺度记忆，而非仅增大 visual budget。

**可复现性**：四域源数据均为公开数据集；附录含详细构造模板与 prompt；License **CC BY-NC-SA 4.0**；实现细节在 Appendix D.1–D.3。待作者发布 benchmark 工件后复现性将显著提升。

## Strengths

- 首次系统地将超高分辨率视觉推理失败**定位到 GND/PER**，而非笼统归咎于「推理弱」。
- GT-CoT + S5 oracle 实验设计简洁有力，分离**视觉事实获取**与**下游推断**。
- 四域互补（监控/遥感/病理/工业）覆盖真实高价值应用场景。
- 严格 QC（文本泄漏、专家审核、异常唯一性）提升基准可信度。
- 对 VLM 社区有明确工程启示：局部证据管道、多尺度记忆、不确定性 grounding 优先于 prompt 工程。

## Limitations

- 主文未公开精确题目规模与 train/test 划分（若存在）。
- 四选一格式无法评测自由形式推理与工具增强 agent。
- V3/V4 局部证据协议人为设计，与真实「模型自主搜索」有差距。
- 未评测专用 ultra-resolution 架构（仅通用 VLM 默认预处理）。
- WSI 仅限 TCGA-BRCA 乳腺肿瘤；AD 为合成网格而非真实产线图像。
- 最强 44.9% 仍远离实用，但论文定位为诊断而非 SOTA 排行榜。

## Questions for Further Exploration

1. **规模与发布**：UltraVR 最终公开多少条 QA？是否计划 HuggingFace / 排行榜与 leader submission 规范？
2. **自主搜索**：将 V3 人工局部证据替换为模型驱动的 zoom-in / visual search，S5 式 oracle 增益是否仍成立？
3. **训练信号**：能否用 GT-CoT 中间步做 **process supervision** 或 RL，专门提升 GND/PER 而非终局答案？
4. **域外泛化**：非 BRCA 癌种、真实工厂 AD、视频 CCTV 序列如何扩展同一五操作 schema？
5. **与 XLRS/UR-Bench 合并评测**：在相同模型上比较「任务级准确率」vs「UltraVR 操作级首错分布」的相关性？
6. **分辨率—token 律**：在控制任务难度下，是否存在可预测的 scaling law 连接 effective resolution 与 PER 误差？
7. **人机协作**：Discussion 提及 human-model collaborative workflows——临床 WSI 场景下 GT-CoT 能否作为住院医师培训或审计轨迹？
