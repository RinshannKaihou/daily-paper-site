> **一句话总结：** 本文发布 793 集公开基准 CUA-HandCrafted，用手写模板复现 RL-Hammer/WASP/TRAP 等文献攻击，在 Claude Sonnet 4.6 与 GPT-5.4 上测得浏览器多步 ASR 为 0/140（95% 上界 2.60%），且抗注入能力在权重而非 system prompt；但同一权重在 SkillBench 编码技能注入上可达 100%，表明前沿 CUA 安全是**域条件化**的，文献 42–98% ASR 主要来自未公开的 RL 优化注入文本。

> **TL;DR:** Nicholas Saban (Patronus AI / UC Berkeley) releases **CUA-HandCrafted**, a 793-episode public adversarial benchmark (24 multi-step web tasks, 56 hand-crafted attack templates, 8 families, 5 depth levels, 4 prompt configs) reproducing headline CUA red-teaming techniques from RL-Hammer, WASP, TRAP, MUZZLE, and pop-ups. Against **Claude Sonnet 4.6** and **GPT-5.4**, multi-step hand-crafted ASR is **0/140** (Clopper–Pearson 95% UB 2.60%); prompt ablation (L0_bare through L3_hardened) shows resistance lives in **model weights**, not defensive prompts. A sister **SkillBench** coding-agent evaluation on the **same checkpoints** reaches **40/40 = 100%** (Sonnet 4.6) and **79/100 = 79%** (GPT-5.4) on skill injection—browser hardening does not transfer. Phase 10 AutoInject RL baseline also scores **0/50** per model within a $10 budget, tightening but not closing the “RL-is-the-gap” hypothesis. A reproducibility audit finds **4/6** audited papers pair retired targets with unreleased optimized strings.

## 核心贡献

- 发布 **CUA-HandCrafted**：793 集、8 个自托管网站、24 个多步任务、56 个手写攻击模板、8 类攻击族、5 级注入深度、5 种注入通道；采用 AgentDojo 兼容的 BU/UuA/ASR/Safety 指标，MIT 代码 + CC-BY-4.0 模板（Zenodo DOI 10.5281/zenodo.20034379）
- 在 **Sonnet 4.6** 与 **GPT-5.4** 上，多步手写攻击 ASR 为 **0/140**（排除 bank_check_balance 评估目标漂移后）；Phase 9 文献启发模板（TRAP/WASP/RL-Hammer/MUZZLE 等）亦为 **0/40**；仅遗留模型单步 DoS 有 6.16–6.88% ASR
- **Prompt 消融**（L0_bare → L3_hardened，含 L1_helpful 主动鼓励遵循页面指令）四档均为 0% ASR + 100% 任务成功率，证明浏览器域注入抗性在**权重**而非 system prompt
- **跨域对照（SkillBench）**：同一权重在编码 agent 技能文件注入上 Sonnet 4.6 最佳 **100%**、GPT-5.4 **79%**（跨方法均值 33.3% / 66.8%），与浏览器 0/140 形成鲜明对比，安全硬化**未跨表面泛化**
- **可复现性审计**（Table 4）：RL-Hammer、WASP、TRAP、RedTeamCUA、MUZZLE、Pop-ups 六篇文献中，4 篇「退役目标模型 + 未发布优化字符串」；手写复现 ASR 在最新前沿模型上均为 0
- **Phase 10 RL 上界**：在 $25 API 预算、每 (task, template) 对 5 次查询内，AutoInject adaptive-random-suffix 基线亦为 **0/50**（Episode-ASR 与 Pair-ASR 均为 0），将「RL 优化文本 vs 手写模板」从推断推进为**预算内实测**
- **VPI-Bench 复现**（Appendix G.1）：popup 覆盖层 1/30（3.3%），页内恶意文本 5/20（25%），图像通道内安全亦呈域分化

## Key Contributions

- **CUA-HandCrafted benchmark (793 episodes):** 8 self-hosted HTML/CSS/JS sites (HR, Dashboard, CRM, Banking, Shopping, Email, Forum, Settings), 24 multi-step tasks (4–12 actions), 6 single-step read tasks, 56 YAML attack templates across 8 categories and 5 depth levels (D1–D5), 5 injection channels (hidden_text, popup, visible_text, dom_modify, help_text), hybrid screenshot+a11y perception, canary-based detection (.test URLs, ID patterns), one-command CLI with Anthropic/OpenAI/Google/vLLM adapters
- **Hand-crafted attacks fail on frontier browser CUAs:** 0/140 multi-step ASR on Sonnet 4.6 + GPT-5.4 (valid eval targets); raw 2/158 on bank_check_balance due to ground-truth drift (Appendix J); Phase 6–9 all 0% on capable models; literature-informed Phase 9 templates (TRAP buttons, WASP P.S., RL-Hammer polite tone, MUZZLE compliance) → 0/40
- **Weight-level, not prompt-level safety:** Phases 3 & 8 ablation across L0_bare, L1_helpful, L2_default, L3_hardened—all 0% ASR with 100% BU on Sonnet 4 and Sonnet 4.6 (48 episodes total); single partial event typed placeholder format "EMP-XXXX" without real canary leakage
- **Domain-conditioned safety (SkillBench cross-reference):** Identical Sonnet 4.6 / GPT-5.4 weights: browser 0/140 vs. coding-skill injection up to 100%/79%; Anthropic system-card browser Best-of-N ASR 49.36%→1.29% (Sonnet 4.5→4.6) does **not** predict coding-surface vulnerability
- **Reproducibility audit (Table 4):** Headline ASRs (42–98%) cluster on retired models (GPT-4o, Sonnet 3.5) and unreleased RL-optimized strings; 4/6 papers irreproducible at full fidelity; RL-Hammer partially reproducible only via released attacker model, not prose descriptions
- **RL optimization as the intra-domain gap:** Hand-crafted floor 0% vs. literature 42–98% attributed to GRPO-discovered off-distribution phrasing (RL-Hammer β=0, no KL); Phase 10 measures cheapest black-box RL attacker—still 0% within budget, not bounding full GRPO AutoInject or GCG
- **Harness validation:** Non-stuck evaluation—legacy Sonnet 4/GPT-4o show 6.16–6.88% single-step DoS; capability (BU) must be controlled before interpreting ASR (GPT-4o 0% BU inflates apparent multi-step ASR)

## Methodology

**Threat model.** Third-party prompt injection on computer-using agents: benign user task + untrusted page/skill content attempting to override agent behavior. Hand-crafted = human-readable prose strings from paper descriptions, **no RL reward signal**—reports the floor of non-optimized templates.

**CUA-HandCrafted harness.** Playwright-instrumented 8 sites; attacks inject via hidden_text, popup, visible_text, dom_modify, help_text at depth D1–D5. Metrics follow AgentDojo: BU (benign utility), UuA, ASR (injected goal executed), Safety = 1 − ASR. Canary detector monitors type-action text, navigate/click URLs, partial patterns, state changes. Multi-step credit requires full action sequence. Fixed conditions: 1280×800 viewport, 10–15 step budget, T=0 (or 0.7), 3-retry backoff.

**Evaluation phases (1–9, chronological).** Phase 1: single-step DoS on legacy models. Phases 2–5: multi-step, prompt ablation, D5 expert. Phase 6: frontier validation (8 attack categories × 3 tasks). Phase 7: advanced DOM (label rewrite, pre-fill, role="alert", input listeners). Phase 8: prompt ablation v2 on Sonnet 4.6. Phase 9: 10 literature-informed templates. **Phase 10 (separate 100 episodes):** AutoInject adaptive-random-suffix learner—mutate Qwen2-1.5B vocab tokens, append to Phase 9 bases, 5 queries/pair, canary reward, ~$10 spend.

**SkillBench cross-domain.** Coding agents load Markdown skill files with embedded shell snippets. Hand-authored Python emits skill templates (claude_v35 cognitive-load workflow, claude_v39 git-remote poisoning, claude_v41 derivative). Objectives: git remote exfil, .env upload via validator script, curl telemetry. LLM judge on multi-turn transcripts.

**VPI-Bench replication.** 50 episodes on Sonnet 4.6/GPT-5.4: popup-overlay vs. in-content malicious text fixtures.

## Critical Assessment

- **Novelty:** High for (1) fixed-weight browser-vs-coding ASR comparison at frontier checkpoints, (2) systematic reproducibility audit framing retired models + unreleased strings, (3) separating hand-crafted floor from RL-optimized ceiling with in-harness Phase 10 measurement
- **Rigor:** Strong experimental design—canary detection, BU/ASR joint reporting, capability conditioning, preregistered exclusions (bank_check_balance), Clopper–Pearson bounds, 10-phase chronological narrative. Honest limitations section. Weakness: Phase 9 templates are approximations of unreleased strings by design
- **Reproducibility:** Excellent artifact release (code, templates, traces, CLI, Zenodo DOI). Third parties can replicate 0/140 on current APIs. Cannot replicate original papers' 42–98% without attacker models/strings—that is the paper's point
- **Significance:** High for CUA deployment risk assessment, benchmark design, and interpreting published ASR headlines. Directly challenges extrapolating browser safety to coding agents or model-level "X% resistant" claims
- **Red flags:** (1) Only two frontier models tested in main body. (2) 0% on approximations ≠ 0% on original RL strings. (3) Phase 10 budget may be too small to bound full GRPO attackers. (4) Cross-domain is browser + coding only (not OS-BLIND-style environment embedding). (5) Single author—no independent replication reported in-paper

## Strengths

- Large, well-documented public benchmark (793 episodes) with AgentDojo-compatible metrics and full release pipeline
- Compelling cross-domain result: same weights, 0% browser vs. up to 100% coding—clearly demonstrates domain-conditioned safety
- Rigorous prompt ablation isolating weight-level vs. prompt-level defenses
- Thoughtful reproducibility audit table making literature ASR numbers interpretable for practitioners
- Phase 10 closes part of the inferential loop on RL-vs-hand-crafted gap within stated budget
- Honest handling of confounds: capability (BU), bank task exclusion, GPT-4o artifact ASR, partial format-compliance boundary
- VPI-Bench replication extends findings to image channel with nuanced popup vs. in-content split
- Actionable recommendations: release optimized strings or attacker models; report ASR vectors across surfaces; name checkpoints

## Limitations

- **Template approximation:** Phase 9 strings approximate unreleased RL-Hammer/WASP/TRAP/MUZZLE optimized text—0% on approximations does not prove 0% on originals
- **Two frontier models:** Main evaluation limited to Sonnet 4.6 and GPT-5.4; no Gemini, open-weights panel, etc.
- **Residual legacy vulnerability:** Single-step DoS 6.5% on Sonnet 4 / GPT-4o—not zero attack surface
- **RL attacker budget:** Phase 10 ($10, 5 queries/pair) does not bound full GRPO AutoInject, GCG, or RL-Hammer-scale search
- **Two surfaces only:** Browser + coding skills; OS environment attacks (OS-BLIND), voice, mobile not covered
- **Loose per-template CIs:** Per-cell n=1–3 in some breakdowns (aggregate 0/140 still statistically strong)
- **Image channel partial:** VPI popup 3.3%, in-content 25%—not fully "closed"
- **RedTeamCUA:** Outside preregistered panel; no direct Phase 9 template

## Relevance

该论文与 **LLM Agent 安全、Computer-Using Agent 红队、Prompt Injection 评测与可复现性** 高度相关（相关性评分 4/5）。它直接质疑近期 CUA 文献 42–98% ASR 对当前部署的适用性，并证明「前沿模型已安全」仅在浏览器域成立、在编码 agent 域不成立。对构建 agent 评测基准、解读 vendor system card、以及规划跨表面安全测试的团队具有重要参考价值。
