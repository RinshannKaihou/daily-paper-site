> **一句话总结：** SAGE-PTQ 用分布统计分离显著权重、稀疏 KNN 图 + 谱聚类自适应估计非显著组数，并对显著/非显著权重做双模式量化与逐矩阵自适应显著性阈值，将「隐藏 scale 开销」压到约 0.004 bit/权重，在 LLaMA/OPT 等 1.3B–70B 模型上显著优于 BiLLM/PB-LLM，且单卡 L40 上 LLaMA-2-70B 解码约快 1.5×。

> **TL;DR:** SAGE-PTQ (Saliency-Aware Graph-guided Efficient PTQ) targets hidden scaling overhead in ultra-low-bit LLM PTQ. It partitions weights via distributional saliency filtering, estimates optimal unsalient group counts with sparse KNN graphs and spectral clustering (Silhouette Score), applies dual-mode quantization (multi-bit salient + binary unsalient with minimal scales), and optimizes per-matrix saliency ratios via Brent’s method. Average 1.03 weight bits and 0.004 scaling bits beat BiLLM/PB-LLM (e.g., LLaMA-3-8B WikiText2 PPL 6.74 vs 55.8; <50% GPU memory vs BiLLM; 1.5× faster 70B decoding on one L40). Weight-only; code promised.

## 核心贡献

1. **显著性感知 + 图引导的权重划分**：按每层矩阵的分布统计（均值 β、标准差 γ、probit 阈值 Φ⁻¹）分离显著（salient）与非显著（unsalient）权重；对子采样的非显著权重建稀疏 KNN 图，用谱聚类 + Silhouette Score 在主机内存预算内搜索最优组数 \(N^{\text{optimal}}_{\text{uns},l}\)，再用分位数划分各组，避免 BiLLM/PB-LLM 的固定位置启发式与 rigid 分组。

2. **双模式量化与极低 scale 开销**：显著权重用 Mode 1 多比特量化（单矩阵 per-channel scale）；非显著各组用 Mode 2 二值/三值量化（每组一个标量 scale）。在 4-bit 显著性预算下平均 **1.03 weight bits + 0.004 scaling bits/矩阵**，相对 BiLLM 的 block-wise 多列 scale（约 1.0 bit）大幅降低隐藏成本。

3. **逐矩阵自适应显著性阈值**：将显著性比例建模为凸优化（式 11），用 Brent 法求最优 \(\zeta^{\text{sal}}_l\)；在 LLaMA-7B 上约 **50% 更低 perplexity** 且 **~30% 更少显著权重比特**（相对固定阈值）。

4. **部署向运行时设计**：bit-packed lookup table \(L^{\text{LUT}}_l\)（\(N_{\text{LUT}}=\lceil\log_2(N_{\text{uns},l}+1)\rceil\)）放 host、层间加载，lookup 成本比 BiLLM 低约 25%；LLaMA 推理 GPU 显存较 BiLLM **约减 50.3%**；量化 66B OPT 约 **1.5 小时**（单 L40），快于 BiLLM 多阶段流程。

5. **跨家族、跨规模实证**：在 LLaMA-1/2/3、OPT、Vicuna、DeepSeek、Qwen2.5/3 上 **1.3B–70B** 验证；WikiText-2/C4 perplexity 与六项零样本 NLU（PIQA、BoolQ 等）均显著优于 BiLLM，多数任务接近 FP16。

## Key Contributions

1. **Distribution-based saliency + graph-guided unsalient grouping**: Probit-threshold salient filtering per matrix; sparse KNN + spectral clustering with Silhouette Score to pick optimal unsalient group counts under lookup/memory budgets; quantile partitioning within groups.

2. **Dual-mode quantization with minimal scaling**: Multi-bit salient (one channel-wise scale per matrix) + per-group scalar-scaled binary/ternary unsalient components; **~1.03 avg weight bits and 0.004 scaling bits**, explicitly accounting for inference scale storage unlike many prior sub-2-bit reports.

3. **Adaptive per-matrix saliency thresholding**: Convex reconstruction objective optimized via Brent’s method; ~50% lower PPL with ~30% fewer salient bits on LLaMA-7B vs fixed thresholds.

4. **Deployment-oriented runtime**: Bit-packed host-side LUTs, reduced lookup overhead, ~50% GPU memory vs BiLLM on LLaMA, 1.5× 70B decode speedup on single L40 via larger in-GPU layer bands and on-the-fly dequantization.

5. **Broad SoTA binarized PTQ evaluation**: LLaMA/OPT/Vicuna/DeepSeek/Qwen families, perplexity + zero-shot NLU, memory and latency case studies.

## Methodology

论文针对 PB-LLM、BiLLM 等二值化 PTQ 的四个核心问题：（1）显著性处理需逐层自适应；（2） rigid/均匀划分无法应对长尾分布；（3） block min–max scaling 与多列 residual scale 使隐藏开销可 **>1 bit/weight**；（4） 部署时 group lookup 常被忽略。

### SAGE-PTQ 流水线（Figure 1）

**Step 1 — 显著性过滤（§3.1.1）**  
对矩阵 \(l\) 的权重集合 \(\Omega_l\)，用 magnitude saliency（附录 C.1 表明优于 Hessian，且 scaling 更少）：

\[
\Omega^{\text{sal}}_l = \{w \in \Omega_l \mid |w| > \beta_l + \gamma_l \Phi^{-1}(\zeta^{\text{sal}}_l/2)\}
\]

\(\zeta^{\text{sal}}_l\) 受存储预算上界 \(\zeta^{\text{sal,max}}_l\) 约束。

**Step 2 — 图引导非显著组数估计（Algorithm 1）**  
从 \(\Omega^{\text{uns}}_l\) 随机子采样 \(S_l\) 个权重（关注 attention 列模式）→ 稀疏 KNN 图 → 在 \(r \in \{2^{N_{\text{LUT}}-1}+1,\ldots,2^{N_{\text{LUT}}}-1\}\) 上做谱聚类 → 最大化 Silhouette \(s_r\)（式 3）得 \(N^{\text{optimal}}_{\text{uns},l}\) → 分位数 mask 得到互斥子集 \(\Omega^{\text{uns}(k)}_l\)。

**Step 3 — 双模式量化（Algorithm 2）**  
\(W_l = W^{\text{sal}}_l + \sum_k W^{\text{uns}(k)}_l\)。  
- **Mode 1**：显著分量交替优化 \(a^{\text{sal}}\) 与连续松弛的 \(B^{\text{sal}}\)，再离散到 \(2^{N_b}+1\) 级。  
- **Mode 2**：每组非显著分量元素级符号阈值 + 闭式标量 \(a^{\text{uns}(k)}\)（式 8–9）；组外位置为 0，合并为整体 \(\{-1,0,1\}\) 再合成 \(B_l \in \{-1,1\}^{m\times n}\)。

**Step 4 — 自适应阈值（§3.3，式 11）**  
最小化归一化 Frobenius 重建误差 \(J(\zeta^{\text{sal}}_l; W_l, N^{\text{optimal}}_{\text{uns},l}, N_b)\)，Brent 求 \(\zeta^{\text{sal,opt}}_l\)。

**Step 5 — 推理运行时（§3.4）**  
每层 bit-packed \(L^{\text{LUT}}_l\) 路由到对应 scale；权重与 scale 在 GPU，lookup 在 CPU 层间流式加载。

### 实验设置

- **实现**：PyTorch + Hugging Face；单 NVIDIA L40 46GB；无微调 PTQ。  
- **模型**：LLaMA-1/2/3、OPT（至 66B）、Vicuna-7B、DeepSeek-7B、Qwen2.5-7B、Qwen3-8B 等。  
- **指标**：WikiText-2、C4 perplexity；零样本 PIQA/BoolQ/OBQA/Winogrande/ARC-e/HellaSwag。  
- **基线**：BiLLM、PB-LLM（10% salient）、GPTQ（2/3-bit）、AWQ（3-bit）。  
- **Scale bits 会计**：显式计入 FP16 min/max 列数——GPTQ/AWQ 0.25、PB-LLM 0.5、BiLLM 1.0、SAGE-PTQ **0.004**（全矩阵一个 FP16 salient scale + 每组一个标量）。

## Results & Highlights

### 平均比特与代表性 perplexity

| 方法 | Weight bits | Scale bits | LLaMA-3-8B WikiText2 PPL |
|------|-------------|------------|--------------------------|
| BiLLM | ~1.09 | 1.00 | 55.80 |
| PB-LLM (10%) | ~1.70 | 0.50 | 73.08 |
| **SAGE-PTQ** | **1.03** | **0.004** | **6.74** |
| FP16 | 16 | — | baseline |

LLaMA-1 7B：SAGE-PTQ **5.99** vs BiLLM 35.04；LLaMA-2 70B：**7.51** vs 8.41（BiLLM）；OPT 66B 在 \(N_{\text{LUT}}=3/4\) 下均优于 BiLLM/PB-LLM/GPTQ-2bit（Table 1）。

### 多架构泛化（Table 3，\(N_{\text{LUT}}=4\)）

WikiText-2 / C4 上 SAGE-PTQ 相对 BiLLM 平均 perplexity 约降 **77.6% / 71.7%**。长尾分布模型（LLaMA-3、Vicuna）用 **1% 显著权重 + 4-bit salient**；Qwen/DeepSeek 用 **10% + 2-bit salient** 亦表现良好。

### 零样本（Table 4）

OPT-6.7B、LLaMA-3-8B、Qwen3-8B：SAGE-PTQ 在多数 benchmark **接近 FP16**，相对 BiLLM 平均 accuracy **约 +5%**（BiLLM 在 HellaSwag 等任务崩塌明显）。

### 内存与延迟

- **GPU 推理显存**：相对 BiLLM **~50.3% 降低**；权重+scale 平均 **~1.038 bits** vs BiLLM **~2.08 bits**（含 lookup 上界假设）。  
- **LLaMA-2-70B 单卡 L40**：量化后 ~12.7GB 可驻留更多层带（\(B_Q=16\) vs FP16 \(B_{FP16}=5\)）；层 I/O **47.07ms → 10.46ms**，per-token **972.1ms → 646.3ms（~1.5×）**，含 band 反量化开销。  
- **量化时间**：66B OPT SAGE-PTQ **1.5h** vs BiLLM **1.7h**（Table 6）。

### 消融（§4.3）

去掉图聚类或自适应显著性均升高 perplexity；完整管线在 outlier-heavy 的 LLaMA/DeepSeek 上约 **2.4×** 优于去掉任一阶段；\(N_{\text{LUT}}=4\) 在 OPT 上接近更高 lookup 的 perplexity，存储-性能折中佳。

## Critical Assessment

**新颖性**：核心洞察是 ultra-low-bit 收益常被 **未计入的 scale/lookup metadata** 吃掉；将「组数选择」从启发式 block 改为 **数据驱动的图聚类 + 显式比特会计**，并与 Bi-VLM 一脉的 saliency 自适应阈值结合，在 binarized PTQ 赛道有清晰差异化。

**严谨性**：
- **优点**：跨 10+ 模型族与 1.3B–70B 尺度；同时报告 weight bits、**scale bits**、PPL、零样本、显存、延迟；对 BiLLM lookup 取 best-case 仍占优；消融分离图模块与阈值模块。
- **弱点**：权重-only，与 W4A16/激活量化生产路径有 gap；图构建与子采样 \(S_l\)、\(K_{\text{neighbors}}\) 的敏感性主要在附录；零样本仅 3 个模型；与最新 QuIP#、SpinQuant 等强 multibit 基线对比有限（主对比聚焦 sub-2-bit/BiLLM）；代码未开源影响即时复现。

**工程意义**：LLaMA-3-8B 上 PPL 从 55.8→6.74 量级跃迁 + 半显存，对 **单卡大模型推理** 叙事有力；70B 1.5× 解码在 memory-bound 部署场景有实际价值。

## Strengths

- 首次在 binarized LLM PTQ 中系统把 **scaling bits** 与 **LUT bits** 纳入公平比较，揭示 BiLLM「有效比特率」远高于名义 weight bits。
- 图引导组数 + 分位数 mask 兼顾非高斯 early-layer 与大规模矩阵可计算性（子采样 + 稀疏 KNN）。
- 双模式量化 scale 极简（矩阵级 + 组标量），与自适应显著性联合达到 **~1 bit 总 footprint** 级部署。
- 评估维度完整：PPL（WikiText-2/C4）、6 项零样本、显存、量化时间、70B 延迟分解。
- 附录权重统计为方法提供可解释动机（OPT 近高斯 vs LLaMA/DeepSeek 长尾 outlier）。

## Limitations

- **仅权重量化**，激活 仍 FP16/未压缩，端到端吞吐受 activation memory 与 kernel 限制。
- 谱聚类 + 多阶段 PTQ 仍有一定 **离线成本**（虽快于 BiLLM，但比纯 GPTQ 复杂）。
- MoE、多模态、长上下文 KV 与专家路由未评估。
- 依赖 magnitude saliency 与 Gaussian-inspired 分位数，对极端非对称分布的鲁棒性需更多理论保证。
- 代码未发布；部分图表为 PDF 栅格化，细节需附录 D/C 交叉阅读。
- 与 3–4 bit 非二值 SoTA（AWQ/GPTQ 调优版）的 **质量-比特帕累托** 全面对比仍不足。
