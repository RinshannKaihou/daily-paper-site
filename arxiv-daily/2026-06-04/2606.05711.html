一句话总结：

Methodology

  • **多智能体 LLM 系统（MAS）**：\(N\) 个智能体 \(A=\{A_1,\ldots,A_N\}\)，各带角色 system prompt、可选工具与**通信信道**。
  • **NL-Comm**：发送方 decode → token 传输 → 接收方 re-encode（**语言瓶颈**）。
  • **Latent-Comm**：发送方暴露 **Embedding / Hidden State / KV-Cache / Activation** 等连续表示，接收方直接注入计算，不经词表往返。
  • **Prefill vs. Decode**：Prefill 产生全序列 per-token 隐状态与 KV；Decode 仅末 token 隐状态 + 增长的 KV——决定可传输的内部状态类型。
  • 时间：**2024–2026**；方法数：**18**（含 1 篇聚合综述 Five Ws）。

Results & Highlights

  • **同模型**：无需显式潜空间对齐（CIPHER、AC、KVComm 等）。
  • **同骨干不同微调**：Interlat、C2C 等用**学习投影**。
  • **异构骨干**：Vision Wormhole（UVC + 视觉注入，O(N) hub-spoke）、MoT（interaction layers + cross-attention，+0.38% ID / +2.92% OOD vs Avengers）。
  • **融合主流**：KV 类多 **Prepend**；隐状态类多 **Concat** 或 **加法**；RelayCaching / Agent Memory 为 **Cache restoration**。

Strengths

  • 降低新研究者进入 latent MAS 的门槛，提供可比较的 **(WHAT, WHICH, HOW)** 三元组描述。
  • 量化 NL vs. latent 的**成本与信息论**动机（Figure 2），支撑混合协议工程决策。
  • 覆盖从 **CIPHER (2024)** 到 **Edge Handover / Vision Wormhole (2026)** 的完整时间窗，KV、LoRA、边缘、移动场景均有触及。
  • Training-free 主导规律的提炼对**即插即用**部署有价值。

Limitations

  • 单作者综述，无独立第三方复现或统一 benchmark 跑分。
  • 18 篇方法选取非系统性 meta-analysis（无 PRISMA 式检索协议）。
  • 跨架构、**潜通道安全**几乎空白，论文仅标为 critical gap。
  • 与 Five Ws、Emergent Language 的对比偏概念层，缺少实验统一体。

TL;DR

Yingzhuo Liu (BUPT) surveys **latent communication** in LLM-based multi-agent systems: agents exchange continuous embeddings, hidden states, or KV-caches instead of tokenized natural language. A **3-axis framework** — **WHAT** (communicated quantity), **WHICH** (sender–receiver alignment), **HOW** (fusion: concat, prepend, math ops, cross-attention, cache restoration) — categorizes **18 representative methods (2024–2026)**. NL-Comm suffers ~2× per-message FLOPs overhead, ~10³–10⁴ information compression per token vs. hidden states, and fluency-driven redundancy; latent channels win on tightly coupled intermediate signalling but lose interpretability.
