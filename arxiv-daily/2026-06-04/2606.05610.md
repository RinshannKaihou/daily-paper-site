> **一句话总结：** 美团/中科院/哈工大团队发现 continued pre-training（CPT）的最优学习率与 batch size 随算力预算呈可预测幂律缩放；提出两阶段框架——小模型代理网格发现 \(B_\text{opt}(L), LR_\text{opt}(L)\) 与 \(L(C)\) 律，再用目标域验证损失反推**等效预训练算力** \(C_\text{pre}\) 实现零样本超参预测——在 Dense-8B 与 MoE-3B 上较目标模型网格搜索节省 **73–92%** 搜索算力，消融验证 \(C_\text{pre}\) 对域偏移 checkpoint 的必要性。

> **TL;DR:** CPT hyperparameters (LR, batch size) are usually tuned by heuristics or expensive grid search on the target model. This paper shows optimal LR **decreases** and optimal batch size **increases** monotonically with compute budget along stable CPT scaling laws (distinct from scratch training). **Stage 1 (Empirical Law Discovery):** grid-search 100M/500M dense & MoE proxies on a 55B-token mixture (SlimPajama + OpenWebMath + GitHub), fit iso-loss curves → power-law \(B_\text{opt}(L)=f(L)\), \(LR_\text{opt}(L)=g(L)\), and loss-compute law \(L(C)=L_0+\alpha C^{-\gamma}\). **Stage 2 (State-Aware Prediction):** map checkpoint validation loss on target data to **equivalent pre-training compute** \(C_\text{pre}=L^{-1}(L_\text{init})\), set \(C_\text{total}=C_\text{pre}+C_\text{cpt}\), predict \(\{B_\text{opt},LR_\text{opt}\}=f,g(L(C_\text{total}))\) without target-model trials. Validated on Dense-8B (6T pretrain) and MoE-3B (5T, MLA+MoE): predicted configs beat LR/B ablations on 10 downstream benchmarks; search FLOPs drop 92.0% (Dense-8B) and 73.1% (MoE-3B) vs 9-config target grid. Model-agnostic, architecture-sensitive coefficients (Table 7).

## 核心贡献

1. **CPT 超参缩放律的发现（RQ1）**：在 CPT 数据上系统训练 100M/500M 稠密与 MoE 代理模型，证实最优 **batch size 随算力单调上升、最优学习率单调下降**（Fig. 1, 3），且可用标准 loss–compute 幂律与超参–loss 映射联合刻画，为从零训练缩放律提供 CPT 专用、可预测的替代启发式/网格搜索的理论基础。

2. **等效预训练算力 \(C_\text{pre}\)**：将任意 checkpoint 在目标域的验证损失 \(L_\text{init}\) 通过逆缩放律 \(C_\text{pre}=(\alpha/(L_\text{init}-L_0))^{1/\gamma}\) 映射为「若在目标数据上从零训练达到同等损失所需算力」，把非零初始化与域偏移下的「黑箱起点」量化为连续训练轨迹上的坐标；总有效算力 \(C_\text{total}=C_\text{pre}+C_\text{cpt}\) 驱动零样本超参预测。

3. **两阶段可部署框架（Fig. 2, Algorithm 1）**：**Stage 1** 离线在小模型上网格搜索 \((B,LR)\)，构造 iso-loss 曲线顶点得 \((B_\text{opt},LR_\text{opt})\)，拟合 \(f,g\) 与 \(L(C)\)；**Stage 2** 在线仅评估 checkpoint 在 CPT 语料上的 \(L_\text{init}\)，即预测目标 run 的最优 \(B,LR\)，无需在 Dense-8B/MoE-3B 上做试探训练。

4. **大规模实证（RQ2–RQ3）**：55B token 三域混合 CPT；起点为 **Dense-8B**（6T 预训练）与 **MoE-3B**（5T，MLA+稀疏 MoE）。预测配置在固定 \(B\) 或 \(LR\) 的消融中取得 **最高平均下游分**（Dense-8B **51.3**，MoE-3B **57.9**）；高 LR 导致灾难性遗忘（GSM8K 等骤降），过大 \(B\) 在固定 token 预算下更新步数不足。

5. **算力效率与 \(C_\text{pre}\) 消融**：相对在目标模型上 9 组网格搜索，代理框架总搜索 FLOPs **8.15×10²⁰** vs Dense-8B **1.02×10²²**（**−92.0%**）、MoE-3B **3.03×10²¹**（**−73.1%**）；忽略 \(C_\text{pre}\)（Variant A）或仅用原始预训练 token 算力（Variant B）均劣于完整 \(C_\text{total}\) 预测（Table 5）。

## Key Contributions

1. **First systematic hyperparameter scaling laws for LLM continued pre-training**, linking compute budget to optimal LR and batch size via proxy-model discovery.

2. **Equivalent pre-training compute \(C_\text{pre}\)** to quantify inherited optimization state under domain shift, enabling zero-shot hyperparameter prediction from any checkpoint.

3. **Two-stage framework** (offline law discovery + online state-aware prediction) that decouples search cost from target model scale.

4. **Empirical validation** on Dense-8B and MoE-3B with 55B-token CPT, 10-task benchmark suite, controlled LR/B ablations, and \(C_\text{pre}\) necessity ablations.

5. **70–92% reduction** in hyperparameter search compute vs traditional target-model grid search, with comparable or better downstream performance.

## Methodology

### 问题定义

- **目标**：建立 CPT 中算力预算 \(C\)（FLOPs，\(C=\sigma\cdot D\cdot N\)）与最优 \((B_\text{opt}, LR_\text{opt})\) 的定量关系，替代启发式调参与昂贵网格搜索。
- **两大挑战**：(1) 非随机初始化下最优超参是否仍服从稳定单调律；(2) 预训练与 CPT 数据分布不同时，如何度量 checkpoint 在目标域上的「已消耗算力」。

### Stage 1：经验律发现（§2.3）

| 步骤 | 内容 |
|------|------|
| **最优性定义** | 达到目标验证损失 \(L\) 所需 **最小训练算力** 的 \((B,LR)\) 配置 |
| **代理实验** | \(N\in\{100\text{M},500\text{M}\}\)，稠密 + MoE；网格 \(B\in\{64,128,256\}\)，\(LR\in\{10^{-4},2\times10^{-4},5\times10^{-4}\}\)；AdamW |
| **Iso-loss 曲线** | 固定 \(\hat L\)，在 \((B,LR)\) 平面标出达到 \(\hat L\) 所需 token 数 \( \hat D\)；曲线顶点 → \((B_\text{opt}(\hat L), LR_\text{opt}(\hat L))\) |
| **拟合** | 幂律 \(B_\text{opt}(L)=f(L)\)，\(LR_\text{opt}(L)=g(L)\) |
| **Loss–compute** | Kaplan 形式 \(L(C)=L_0+\alpha C^{-\gamma}\)，得 \(B_\text{opt}(C)=f(L_0+\alpha C^{-\gamma})\)，\(LR_\text{opt}(C)=g(\cdots)\) |

**经验趋势（Appendix B, Fig. 4）**：训练后期更大 \(B\) 降低梯度方差、更小 \(LR\) 抑制 loss 振荡，解释收敛阶段「大 batch + 小 LR」的必要性。

### Stage 2：状态感知预测（§2.4）

1. 在目标 CPT 数据上评估 \(L_\text{init}\)。
2. \(C_\text{pre} = \big(\alpha/(L_\text{init}-L_0)\big)^{1/\gamma}\)。
3. \(C_\text{total} = C_\text{pre} + C_\text{cpt}\)（计划 CPT 算力）。
4. \(L_\text{target} = L_0 + \alpha C_\text{total}^{-\gamma}\)。
5. 输出 \(B_\text{opt}=f(L_\text{target})\)，\(LR_\text{opt}=g(L_\text{target})\)。

### 下游消融（Table 3 要点）

**MoE-3B（预测 LR=\(10^{-4}\), B=1600）**
- 平均 **57.9**（bold best）；过高 LR \(5\times10^{-4}\) → **50.8**；过小 B=512 → **56.4**；过大 B=3200 → **57.7**。

**Dense-8B（预测 LR=\(5\times10^{-5}\), B=768）**
- 平均 **51.3**（best）；LR=\(5\times10^{-4}\) → **47.6**（GSM8K 等崩塌）；LR=\(3\times10^{-5}\) → **51.0**；B=256/1600 均约 **51.0–51.1**。
- LiveCodeBench 对 Dense-8B 区分度低（文中排除该项对比）。

### 搜索算力（Table 4）

| 模型 | 策略 | 配置数 | FLOPs | 节省 |
|------|------|--------|-------|------|
| Dense-8B | Grid on target | 9×Dense-8B | \(1.02\times10^{22}\) | — |

搜索成本与目标模型规模**解耦**，模型越大优势越明显。

## Critical Assessment

**新颖性**：将「最优 LR↓、最优 B↑ 随算力变化」从 CPT 经验观察提升为可拟合、可外推的缩放律，并以 \(C_\text{pre}\) 统一处理域偏移与非零初始化，相对 scratch 超参缩放与 µP 有清晰问题增量。与 Li et al. (2025)、Zhang et al. (2025) 等 scratch 超参律形成互补。

**严谨性**：
- **优点**：双架构（稠密 vs MoE）、双尺度代理 + 双大模型验证、iso-loss 几何解释、\(C_\text{pre}\) 双变体消融、算力账本（Table 4）透明。
- **弱点**：RQ3「优于启发式基线」在正文主要以 **预测 vs 手调网格邻域消融** 体现，未系统对比 WSD 重 warmup、数据 replay 等 CPT 常用启发式；\(L_0,\alpha,\gamma\) 在正文未完整列表；仅验证至 8B 级；三域混合的单一 \(\gamma\) 可能不适用于极窄域。

**可复现性**：需复现 55B CPT 与 8B 级训练，门槛高；但 Stage 1 仅需小模型网格，对资源受限团队仍具方法论价值。

## Strengths

- **原则性替代网格搜索**：一次代理离线拟合 + 单次 \(L_\text{init}\) 评估即可零样本预测，搜索 FLOPs 降 **73–92%**。
- **\(C_\text{pre}\)** 直观量化「checkpoint 在目标域已走多远」，消融证明优于忽略状态或照搬预训练 token 数。
- **架构感知**：Table 7 揭示 MoE 对 LR 更不敏感、对 \(B\) 更贪婪，与稀疏路由统计解释一致。
- **训练稳定性洞见**：高 LR 灾难性遗忘、过大 \(B\) 减少更新步数——与 Fig. 4 损失轨迹相互印证。
- **模型无关流程**：同一框架覆盖 Dense-8B 与 MoE-3B，利于 CPT 工具链标准化。

## Limitations

- **规模外推未验证**：仅 Dense-8B / MoE-3B；70B+ 可能出现新不稳定或需重标定幂律常数（§5 Limitations）。
- **域特异性**：统一混合拟合的 \(\gamma, L_0\) 对极专域（窄代码库、单一语言）可能需局部再校准。
- **超参范围有限**：仅 AdamW 默认其余超参；未联合 warmup/WSD、weight decay 调度、数据 replay 比例。
- **最优性定义基于 val loss + 算力**：与下游 benchmark 最优未必一致；Table 3 平均分为事后验证而非直接优化目标。
- **代理尺度间距**：100M/500M → 8B 外推依赖缩放律不变性，未报告置信区间或拟合 \(R^2\) 数值（附录提及 \(R^2\) 但未在提取文本中给出具体值）。
- **工程开销**：Stage 1 仍需四组代理模型 × 9 格点完整 CPT 轨迹，一次性成本仍可观。

## Questions for Further Exploration

1. **70B+ 再标定**：\(L_0,\alpha,\gamma,f,g\) 是否需按目标参数规模分段拟合，还是 100M/500M 代理仍足够？
2. **下游对齐**：以 MMLU/GSM8K 加权目标替代 val loss 定义「最优 \((B,LR)\)」，预测配置是否仍占优？
3. **与 WSD / LR re-warmup 耦合**：在 \(C_\text{total}\) 预测基础上如何自动生成 re-warmup 幅度与 decay 阶段长度？
4. **单域 \(\gamma\) 表**：General/Math/Code 分域拟合 \(L(C)\) 后集成预测，能否减少混合律在窄域上的偏差？
5. **其他超参**：weight decay、序列长度、数据混合比例是否也存在 \(C_\text{total}\) 缩放律？
6. **开源复现包**：发布代理网格日志与 \(f,g,L(C)\) 拟合脚本，降低工业 CPT 采纳门槛。
7. **动态 CPT**：训练中途根据当前 \(L\) 更新 \(B,LR\)（而非常数），能否用同一律做 schedule 而非单点预测？
