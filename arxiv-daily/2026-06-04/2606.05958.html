So What

HuggingFace 上「文本 + 预计算向量 + α + 等价证书」bundle 可通过本地重算与良性属性验收，却在有害 prompt 上静默越狱——可复现 ≠ 安全，需有害集 ASR 探针。

Critical Edge

白盒 2B/8B 开源 IT；Claude 审计仅标 1.8% 投毒文本且可本地预筛重跑 GCG。拒绝方向正交化收回 ≈82% ASR 差距，但攻击者可共优化 cos(v,r)≤τ（τ=0.10 时仅恢复 3% 差距）。

Methodology · 威胁模型与优化

下载 bundle → 重算 v ✓ → 读文本 ✓ → 良性 AC ✓ → 有害 ASR ✗
v = (1/N) Σ hℓ(xi+) − (1/N) Σ hℓ(xi−) 对比激活 steering；攻击最大化 cos(ṽ, r)，r 为 128 条无害/有害 prompt 的 anti-refusal 均值差（Arditi et al.）

隐身 GCG 约束

  • 嵌入 top-K=100 近邻同义词替换
  • 安全词表 + Detoxify / Llama-70B 过滤
  • 每文本 ≤nmod=5，PPL τ=2000，λlm=0.2
  • POS/NEG 对同步改，保护属性后缀

评估协议

  • ASR：100 有害 prompt，三法官集成（κ≈0.91）
  • AC：100 良性 prompt 属性合规
  • Gemma-2-2B-IT L14 · Llama-3.1-8B L18
  • 8 组合 × 3 GCG seed；α∈{2,3,4}

核心贡献

§3 供应链威胁模型 对比 steering 数据集为独立攻击面；等价证书只证明「向量由文本正确导出」，无法检测几何 payload。
§3.3–3.4 约束 GCG 投毒 近邻同义词、编辑预算、PPL 上限；同步改 POS/NEG，标签仍可信。
§4 双模型八组合 语言/格式/大小写多类属性；ASR 20–55% 且 AC、ℓ₂ 近干净——bundle 三步验收皆过。
§4.6–4.7 防御评估 LLM 审计非屏障；正交化有效但自适应共优化可大幅绕过。

Strengths

  • 威胁模型贴近 HuggingFace 共享向量实践
  • 隐身约束可复现，对比 GCG gibberish
  • 八组合跨属性类，非单任务偶然
  • 同时报告 ASR / AC / ℓ₂ / cos(v,r)
  • 正交化防御简单有效，诚实讨论自适应绕过

Limitations

  • 白盒假设；闭源 API-only 不适用
  • 仅 2B/8B；更大模型 entanglement 未知
  • 单目标 refusal；跨架构 poison 迁移未测
  • 正交化需 defender 有害/无害 prompt 集
  • Gemma lowercase 弱 steering 削弱部分 stealth 主张

Questions for Further Exploration

  1. Gemma 上优化的投毒向量能否迁移抬高 Llama ASR？
  2. 70B+ 上 entanglement 是否减弱？
  3. 无梯度黑盒改写能否达到可比 cos(ṽ, r)？
  4. 共享 bundle 是否应强制附带标准有害 ASR 探针与消毒双报告？
  5. 除 jailbreak 外能否 stealth 注入偏见或其他危害？
  6. API 侧 activation patching 上传 contrastive 数据是否存在类似 payload？
