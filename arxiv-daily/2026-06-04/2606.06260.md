> **一句话总结：** 针对 OneRec 类生成式推荐中「思考模式不优于非思考模式」的困境，快手 OneRec 团队提出 **OneReason**：以 **感知（itemic↔文本对齐）** 与 **认知（三级 CoT + specialize-then-unify RL）** 双支柱，在 578B token 预训练、R0–R3 分层 SFT 与领域专精 RL 后，使 **thinking 模式在真实业务基准上稳定超越 non-thinking**；并将开源 **OneReason-8B / 0.8B** 与 **OneReason-Bench**，在快手本地生活广告场景实现显著线上增收（Combined +8.23% Revenue）。

> **TL;DR:** OneReason is Kuaishou’s reasoning-centric generative recommendation foundation model. Prior OneRec-Think / OpenOneRec showed “think-before-answer” patterns but thinking rarely beat direct decoding. Inspired by MLLM perception–cognition insights, OneReason strengthens (1) **perception** via four-granularity pre-training (578B tokens) and RQ-KMeans itemic tokens, and (2) **cognition** via R0–R3 CoT SFT plus **specialize-then-unify** RL (domain RL → RFT / multi-teacher distillation). OneReason-RFT-thinking reaches SOTA on cross-domain Rec@64 (e.g., Cross-Live Pass@64 21.10%) while preserving Qwen3-8B-level general benchmarks. CoT supervision can also boost **non-thinking** inference under tuned CoT/unCoT mixtures. Deployed with Fast–Slow Thinking nearline architecture; 10-day 5% A/B shows up to +10.33% Impressions and +8.23% Revenue (ROI > 5).

## 核心贡献

1. **问题诊断与双支柱设计**：将生成式推荐推理失败归因于两类缺口——**感知**：itemic token 未与语言语义深度对齐；**认知**：推荐 CoT 缺乏「压缩→推理」的粗到细结构。借鉴 MLLM 文献，提出与 OneRec 工业栈衔接的 **OneReason** 全链路（预训练 → SFT → RL → 部署）。

2. **四级粒度预训练 + Itemic Tokenizer**：多模态编码器（ViT + Qwen3-VL + 音频）+ RQ-KMeans 三层码本（每层 8192）表示物品；**四粒度**推荐语料（token / item / relation / user）+ 三阶段训练（110B + 449B + 19B，共 **578B tokens**），系统性对齐 itemic 与自然语言模态。

3. **R0–R3 认知增强 CoT（SFT）**：按 **感知 R0 → 推导 R1 → 演化 R2 → 推荐 R3** 分层监督；推荐 CoT 采用 **人格压缩 + 兴趣演化压缩 + 转移推断** 三阶段协议，将溯因式推荐推理（假设潜在兴趣点→时序演化→下一物品）显式化。

4. **Specialize-then-Unify RL**：多域混合 RL 下 thinking 仍弱于 non-thinking；**单域专精 RL** 可解锁 thinking 增益，再通过 **Rejection Sampling FT** 或 **Multi-Teacher On-Policy Distillation** 跨域统一。发布 **OneReason-RFT** 变体，thinking 在 Cross-Video/Product/Ad/Live 上全面领先。

5. **OneReason-Bench 与可迁移监督现象**：扩展 RecIF-Bench，按 R0–R3 诊断推理能力；发现 **CoT 监督在同等 token 预算下可提升 non-thinking 解码**（多数域存在 CoT/unCoT 最优混合比，Cross-Ad 例外）。将开源 **OneReason-8B、OneReason-0.8B** 及相关材料。

6. **工业部署「快慢思考」**：近线 Slow（OneReason 离线推理写 Redis）+ 在线 Fast（OneRec + Thinking Token 蒸馏）协同；快手本地生活广告 **10 天 5% 流量 A/B**：Combined 部署 **Impressions +10.33%、Revenue +8.23%**，年化商业收入增量达数亿人民币量级，**ROI > 5**。

## Key Contributions

1. **Diagnosis and dual-pillar framework**: Attributes weak recommendation CoT to missing **perception** (itemic–language alignment) and **cognition** (structured abductive traces), mirroring MLLM alignment + CoT-quality findings.

2. **Four-granularity pre-training at scale**: Hierarchical corpora from sub-token semantics to user behavior; **578B-token** three-stage recipe on Qwen3-8B backbone with dedicated itemic tokenizer (RQ-KMeans, domain begin tokens, no trailing end token).

3. **R0–R3 SFT stack**: ~941K R0 (caption/QA), ~400K R1 (one-hop derivation), R2 evolution chains, R3 recommendation with **persona abstraction → interest expansion → transition inference** CoT; parallel unCoT variants for efficiency.

4. **Specialize-then-unify post-training**: Domain-specific RL unlocks thinking gains; cross-domain balance via RFT or multi-teacher on-policy distillation—first reported setting where **thinking consistently beats non-thinking** on recommendation benchmarks.

5. **OneReason-Bench + behavioral transfer**: Progressive R0–R3 evaluation suite; controlled studies show CoT supervision can improve direct (non-thinking) recommendation under domain-specific mixture ratios.

6. **Production Fast–Slow Thinking**: Nearline generative retrieval fused with real-time OneRec ranking; validated online uplift with strong ROI.

## Methodology

### 1. 设计哲学：推荐推理是溯因而非演绎

与数学/代码的演绎式 CoT 不同，推荐需从长 **itemic 行为序列** 推断不可观测的用户意图，属于 **溯因推理**：选取相关行为作为假设兴趣点 → 压缩为可解释偏好 → 建模时序转移 → 关联候选物品。据此定义四层能力 **R0–R3**，并贯穿预训练、SFT、RL 与 **OneReason-Bench**。

### 2. 预训练：感知基础

| 组件 | 要点 |
|------|------|
| **Itemic Tokenizer** | 多模态编码器蒸馏封面/帧/文本/音频 → RQ-KMeans 三层码本；模式 `<\|domain_begin\|><a_*><b_*><c_*>`（video/prod/ad/living/sid） |
| **四粒度数据** | Token 级语义、Item 级属性、Relation 级 item↔item、User 级行为；弥补以往任务级混合的表达同质性与结构缺口 |
| **三阶段训练** | Stage1：仅训扩展词表+LM head（110B, 4K/样本）；Stage2：全参（449B）；Stage3：32K 长序列用户历史（19B）；packing 至 48K |

### 3. SFT：压缩轴 + 推理轴

- **R0 感知**（~941K）：itemic↔caption 双向 + QA；thinking 格式显式 **粗→细 token 语义** 再出 caption。
- **R1 推导**（~400K）：源物品单跳桥接（建材价→软装需求等），教师 LLM 过滤弱关联，防 target leakage。
- **R2 演化**：从全域时间线提取演化链，LLM-as-Judge 过滤伪逻辑/随机消费链。
- **R3 推荐 CoT**：三阶段——**Persona Abstraction**（稳定人格压缩）、**Interest Expansion**（近期轨迹 motif）、**Transition Inference**（到目标域下一物品）；与信息瓶颈/压缩视角一致。

另含 **Itemic Instruction** 与 **General-Domain** 数据，防止指令跟随与通用能力坍塌。

### 4. RL：先专精后统一

- **领域专精 RL**：单域 RL 使 thinking 超越 non-thinking；多域混合 RL 则相反。
- **跨域统一**：**Rejection Sampling Fine-tuning (RFT)** 或 **Multi-Teacher On-Policy Distillation**。
- 产出 **OneReason SFT / OneReason RFT** 的 thinking & non-thinking 四变体。

### 5. CoT 分析指标（§7）

定义可量化 CoT 质量指标（如压缩度、逻辑一致性等），用于监控推理轨迹质量，支撑数据飞轮与线上迭代（详见正文 §7 与附录）。

### 6. 线上「快慢思考」架构

1. 近线数据管道触发 → 离线 OneReason 预测 next itemic tokens → 解码 item ID 写入 Redis  
2. 在线排序融合近线候选与实时 OneRec  
3. **增量预训练 / 增量 SFT** 适应新物品与短期兴趣漂移  
4. 两范式：**Slow Pipeline**（OneReason 检索）与 **Fast Pipeline**（Thinking Token 向 OneRec 蒸馏）

## Results & Highlights

### 跨域推荐（Table 14，Pass@64 / Recall@64，%）

| 模型 | Cross-Video | Cross-Product | Cross-Ad | Cross-Live |
|------|---------------|---------------|----------|------------|
| SASRec / HSTU (ID) | ≈0 | <1 | <3 | <3 |
| GPT-5.4 等 Text-LLM+ANN | <0.3 / <1.5 | <1.5 | <2 | <7 |
| LC-Rec-PT-SFT-8B | 1.49 / 0.13 | 3.95 / 3.00 | 15.85 / 6.55 | 19.32 / 16.70 |
| OneReason SFT non-thinking | 1.33 / 0.11 | 3.94 / 2.96 | 15.73 / 6.49 | 18.05 / 15.52 |
| OneReason SFT thinking | 0.71 / 0.06 | 2.18 / 1.65 | 9.16 / 3.41 | 16.43 / 14.32 |
| **OneReason RFT thinking** | **2.41 / 0.24** | **5.47 / 4.19** | **17.78 / 7.50** | **21.10 / 18.35** |

**要点**：ID 模型受 **33.69% 冷启动 item ID** vs **11.55% 未见 itemic pattern** 影响；文本 LLM 缺协同信号且依赖 caption-ANN 检索；OneReason 预训练 checkpoint 使 LC-Rec-PT-SFT-8B 大幅跃升，证明 itemic 语义地基有效；**RFT thinking 全面 SOTA**。

### R0–R2 推理层（Table 15，节选）

- RFT thinking 在 R0 略逊于 non-thinking（感知任务 **overthinking**，与 MME-CoT 现象一致），但 **R1–R2 平均更优**（如 Evolution DirectGen. 21.23% vs 14.31% SFT thinking）。
- 紧凑 itemic 输入下，OneReason 在 R2 部分任务上超过更大通用 LLM。

### 通用能力体检（Table 16，thinking，%）

| 模型 | MMLU-Pro | GPQA-Diamond | MATH-500 | GSM8K |
|------|----------|--------------|----------|-------|
| Qwen3-8B thinking | 72.35 | 56.06 | 95.20 | 95.68 |
| LC-Rec-PT-SFT-8B | 39.72 | 35.86 | 81.00 | 51.55 |
| **OneReason RFT thinking** | **72.08** | **54.04** | **95.40** | **94.69** |

推荐专精训练 **未灾难性损害** Qwen3-8B 级通用推理（LC-Rec 则严重退化）。

### CoT → Non-Thinking 迁移（§8.2）

- **Token 对齐实验**（0.25B tokens）：40K CoT + 50K unCoT vs 100K unCoT，Cross-Video/Live 的 Pass@64 提升（如 Live 18.12%→20.32%），Cross-Ad 略降。
- **混合比扫描**：各域存在 **中间最优 CoT 比例**（非单调）；概念分解：直接答案对齐收益 vs CoT 压缩/推理信号 vs 轨迹噪声与格式错配成本。

### 线上 A/B（Table 18，10 天，5% 流量，本地生活广告）

| 部署方式 | Impressions | Revenue |
|----------|-------------|---------|
| OneReason（Slow） | +0.94% | +4.53% |
| OneReason for OneRec（Fast） | +6.83% | +4.64% |
| **Combined** | **+10.33%** | **+8.23%** |

训练数据 **未包含** 该场景，体现强域适应与 Fast–Slow 架构可复用性。

- **证据强度**：66 页技术报告 + 大量附录（基准构造、CoT 数据、部署细节）；离线基准与 **真实线上 Revenue/Impressions** 双重验证；将开源 8B/0.8B 模型降低复现门槛。

## Critical Assessment

**新颖性**：首次在 **同一端到端 GFM** 内，用可诊断的 R0–R3 协议 + 工业级数据/算力，使推荐 **thinking 稳定优于 non-thinking**，并量化 CoT 监督对直接解码的迁移；相对 OpenOneRec 的「能思考但不一定更好」是明确进阶。

**严谨性**：
- **优点**：问题动机与 MLLM 文献对齐清晰；分层基准与训练阶段一一对应；大量消融（四粒度预训练、CoT 混合比、RL 专精策略）；诚实区分行为证据与因果机制；部署 ROI 披露。
- **弱点**：跨域基准与线上场景仍偏快手生态；Text-LLM 基线经 ANN 检索可能偏弱；R3 CoT 构造依赖强教师 LLM，质量与偏差传播未完全量化；66 页 + 附录信息密度高，社区复现依赖即将发布的权重与数据细节。

**影响力潜力**：生成式推荐从 **System-1 单次生成** 迈向 **System-2 可解释推理** 的工业范例；OneReason-Bench 可成为与 RecIF-Bench 并行的诊断标准；Fast–Slow + Thinking Token 蒸馏为算力受限场景提供可落地模板。

## Strengths

- 将 MLLM「对齐 + CoT 质量」洞见 **完整映射** 到 itemic 推荐域，而非简单拼接通用推理数据。
- **578B 四粒度预训练** + R0–R3 SFT 形成可测量、可迭代的能力阶梯。
- **Specialize-then-unify RL** 解决多域 thinking 互相拖累的实际训练难题。
- 保持 **Qwen3-8B 级通用能力** 的同时取得推荐 SOTA。
- 线上 **Revenue +8.23%** 与开源承诺兼具学术与工业价值。
- 发现 **CoT 监督增强 non-thinking** 的数据飞轮含义，指导混合标注策略。

## Limitations

- 报告体量大，外部研究者短期难以完全复现内部数据管道与 RL 奖励设计。
- Cross-Ad 对 CoT 不敏感，说明 **域特异性** 强，无通用 CoT 配比。
- Thinking 模式延迟高，依赖近线/蒸馏，非全链路实时 CoT。
- 与 ID-based / Text-LLM 对比协议（ANN 检索、零样本）可能低估部分强基线潜力。
- 压缩 vs 推理对 non-thinking 增益的 **因果分解** 仍待进一步消融。

## Questions for Further Exploration

1. **开源后复现**：OneReason-8B 在公开 RecIF / NineRec 等数据上，thinking 增益是否跨平台成立？
2. **机制消融**：固定 CoT 长度与结构，分离 **压缩监督** vs **转移推断监督** 对 non-thinking 的独立贡献。
3. **与 Zhang et al. (2026b) 对齐实验**：文本惯性修复后，thinking/non-thinking gap 在各层的可解释性指标变化？
4. **实时 CoT**：推理加速（投机解码、CoT 早停）能否让 Full-Slow 架构在延迟约束下可行？
5. **Agentic 推荐**：结论展望的 planning / tool-calling 与当前 R3 CoT 如何衔接？
6. **多教师蒸馏**：跨域教师冲突时，何种 counteraction-aware 策略最优（对照 Chen et al., 2026b）？
7. **公平与偏见**：Persona Abstraction 是否放大人口统计刻板印象，需何种审计指标？
