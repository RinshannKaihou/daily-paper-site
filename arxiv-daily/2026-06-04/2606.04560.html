Rollout-Level Advantage-Prioritized Experience Replay for GRPO

Yoo, Park, Jang, Kim, Yoon — SNU · 2026-06-04 · GRPO RLVR Experience Replay
Anchor Fact
+4.35 pp
五套数学基准平均准确率（Qwen3-Base 4B）
pass@k vs. GRPO：+10.06 pp（50.98 vs. 40.92）
So What

rollout 生成占 RLVR 后训练主成本；本文给出可操作的回放配方（步龄驱逐 + fresh 锚定 + rollout 级 |A_i|），在策略漂移可控前提下复用高价值轨迹。

Critical Edge

单 seed、仅 step-200 快照、数学二值奖励；无公开权重。朴素 query-level pool 回放常无法稳定超过 vanilla GRPO——4B 上 pool composition 甚至跌破基线。

Methodology

二值可验证奖励 RLVR · G=8 rollouts/prompt · DeepScaleR-Preview（40,306 prompts）· DAPO clipped GRPO（verl）

① Generate fresh → ② Sample |A_i| replay → ③ GRPO update → ④ Insert + age evict
A_i = (r_i − μ_g) / (σ_g + ε)  ·  P(i) ∝ |A_i|^α, α=0.5  ·  ρ_i = π_θ(o_i|q) / π_t(o_i|q) 20-step warmup 无回放；τ_max=10 步龄驱逐绑定 staleness；容量 30k 为后备（稳态 ≤1,250 @4B）。
Granularity Rollout 级存储/采样（非 query 级整组 G 条），组内可区分高 |A_i| 少数派
Composition Fresh-anchored：每步保留全部 fresh，再拼接 r=0.5 回放样本（非 DQN 式混合池）
Staleness birth step + τ_max=10 驱逐；相对容量上限，步数约束最坏 policy lag
  • vs. ExGRPO/RePO/RLEP： 共享 fresh+replay 思想，但多为 query 级 + 组恒定优先级（σ_g、solve rate 等）。
  • 混合奖励组： 0.6B 34% → 4B 49%，高-|A_i| 回放候选随规模增加，解释增益放大。

核心贡献

Rollout 级缓冲： 单条 rollout 为单元，PER 式 |A_i| 优先可瞄准组内少数高梯度样本。
步龄驱逐 τ_max： 以训练步数约束 policy lag，与缓冲容量/模型规模解耦。
Fresh-anchored 批次： 每步全量 fresh + 独立回放拼接；4B 上避免 pool mix 跌破 GRPO。
跨尺度实证： Qwen3-Base 0.6B/1.7B/4B × 五数学基准；增益随 mixed-reward 组比例上升。

Strengths

  • 问题清晰：GRPO 单次使用丢弃高 |A_i| 少数派信号
  • τ_max 步龄驱逐 + fresh 锚定为可落地设计规则
  • 消融隔离 τ_max、composition、priority、granularity
  • 诚实报告朴素 query-level 回放常无效
  • pass@k + AES 超越 avg acc，覆盖多样性与 token 成本
  • ~2,500 H100-hr 全 sweep，超参与 Appendix A 详尽

Limitations

  • 仅数学二值 RLVR + DAPO GRPO；无 RM / 其他 RL 算法
  • 单 checkpoint（step 200）；单训练 seed
  • 0.6B mixed-group 仅 34%，边际较小且部分非单调
  • 4B 正确链 +34.9% 长度——部署是否合意未验证
  • 无公开 checkpoint / 数据集

Questions for Further Exploration

  1. τ_max=10 在 8B+ 与更长训练中是否仍最优？
  2. 难度自适应 prompt 能否缓解 0.6B mixed-group 稀缺？
  3. 与 CPPO/DPPO/PODS 等 fresh-side 剪枝是否可组合？
  4. 重要性采样校正下 fresh-anchored 是否仍必要？
  5. 非对称长度偏移是否改善人类评判的推理质量？
