> **一句话总结：** 面向「场景化」多智能体环境，AURA 在感知与工具调用之间插入 **IntentFrame** 推理步：用标量 **gap** 分数估计字面需求与隐含需求之差，据此路由探测预算与工具选择；在 100 条四场景隐含意图基准上较 ReAct 式探测提升隐含需求覆盖率 **Δ=+0.07**（*p*<10⁻⁶），在事实查询上则以 **82% 更少探测** 达成访问–成本 Pareto 前沿，并在隐私敏感切片实现 **0% 禁用工具违规**。

> **TL;DR:** Situated queries (e.g., “where is Lin Wei?”) often encode implicit needs (availability, mood, interruptibility). Standard tool agents answer literally and stop. **AURA** adds an **IntentInferrer** before bounded **Explore** probing that emits an **IntentFrame** with gap score *g* ∈ [0,1] routing probe budget *B(g)* and tool shortlist. On 100-query implicit-intent benchmark: **0.804 vs 0.733** implicit coverage vs ReAct-style NoIntent (Δ=+0.071, *p*<10⁻⁶; 3/4 scenes significant). On 50 factual queries: GapRouted is **not** raw-FA winner (0.696 vs Fixed-Probe 0.766) but Pareto on probes (1.40 vs 8.00) and privacy violations (0% vs 78.9–100%). Lift localizes to LLM gap calibration (heuristic backend: 0.803→0.368). Code/simulator/benchmark released.

## 核心贡献

1. **IntentFrame 作为预工具控制变量**：在任何工具触发前，**IntentInferrer** 根据用户表面查询 *q*、场景预览 *S* 与记忆 *M* 输出结构化 **IntentFrame**（字面需求 ℓ、隐含需求列表 *I*、gap 分数 *g*、推荐探测工具集 *P*、告警标志 *a*、置信度 *c*、理由 *r*），用 *g* 确定性映射探测预算上限 *B(g)* 并指导 Explore 阶段的工具选择与提前告警。

2. **四场景 100 条隐含意图基准上的显著提升**：AURA-Intent 隐含需求覆盖率 **0.804**，相对 ReAct 式 NoIntent（**0.733**）提升 **Δ=+0.071**（配对检验 *p*=1.0×10⁻⁶）；四个场景中三个 individually significant；25 条试点、第二骨干与 prompt 消融均复现增益。

3. **机制适用边界刻画**：在事实性环境查询（§5.1）上，gap 路由探测是 **访问–成本 Pareto 点**而非通用精度赢家；在隐私敏感 30 条切片上 FA 与 Plan-and-Solve/ReAct 相当（*p*=0.86/0.65）同时将禁用工具违规降至 **0%**；routine grounding 指标饱和，不用于论证机制。

4. **开源 AURATown 与评测资产**：发布 60×60 网格、5 智能体、20 地点的多智能体社交仿真器、100 条带五类子标签的隐含意图查询（标注者间 κ=0.61）、多 seed 运行记录及完整代码（https://github.com/innovation64/AURA）。

## Key Contributions

1. **Introduces IntentFrame** as an explicit pre-tool inference stage separating implicit-need estimation from bounded environment probing, vs. reactive ReAct/Plan-and-Solve that plan from literal queries only.

2. **Demonstrates +0.07 implicit-need coverage** over ReAct-style probing on a 100-query four-scene benchmark (*p*<10⁻⁶), with per-scene significance, cross-backbone replication (Claude Haiku, Qwen-Plus), and ablations attributing lift to gap calibration—not answer memorization.

3. **Characterizes regime boundary**: on factual grounding, gap-routed probing trades raw FA for 82% fewer probes and zero forbidden-tool violations on privacy-sensitive queries; primary accuracy win is the implicit-need regime.

4. **Releases AURATown simulator**, benchmark, and per-seed run artifacts for situated agents with public/private state partitions and tool-mediated hidden-state access.

## Methodology

### 问题与架构

- **场景化（situated）**：结构化环境中，私有状态经工具分区访问；智能体可被动观察公开上下文，但必须主动探测隐藏状态（窄于具身导航、开放世界探索等广义 situated agent）。
- **优化目标**：最大化动作–状态一致性 *G* 与回复事实准确性 *R*，通过探测策略 π_E 控制环境访问。
- **两阶段流水线**（Fig. 1）：
  1. **确定性上下文组装**：Sense → Scene → Memory（沿用 Generative Agents 式被动感知，非本文贡献）；
  2. **LLM 控制推理**：IntentInferrer → Explore → Reason → Act → Interact。

### IntentFrame 形式化

\[
\text{IntentFrame} = (\ell,\; I,\; g \in [0,1],\; P \subseteq \mathcal{T},\; a \in \{0,1\},\; c \in [0,1],\; r)
\]

- *g*：字面与隐含需求差距（0=字面足够，1=正交隐含需求）；
- **预算映射**（上限而非目标）：

| *g* 区间 | *B(g)* |
|----------|--------|
| [0, 0.2) | 0 |
| [0.2, 0.4) | 1 |
| [0.4, 0.6) | 2 |
| [0.6, 0.8) | 3 |
| [0.8, 1] | 5 |

全局受 `explore_max_steps` 截断；Explore 通常在达到上限前提前停止（*r*=0.19 的 gap–实际探测相关）。

### Bounded Proactive Probing（Explore）

- 与 ReAct 在回答生成中交错工具不同，**探测先于 Reason**，最终回复条件于紧凑探测轨迹；
- 至多 *B* 步：规划器 ϕ 在工具注册表 𝒯 上选择调用或停止；
- 设计原则：目标导向、有界、可组合、与推理解耦（可缓存/跳过）。

### AURATown 仿真环境

- **60×60 网格**，5 命名智能体、20 地点、6:00–23:00 日程；
- **公开/私有状态分裂**：`location`、`action` 在场景快照可见；`availability`、`emotional_state`、`unspoken_goal`、`beliefs_about_others` 仅经探测工具获取；
- 每 tick 七条优先级规则确定性更新私有状态；共现刷新信念，支撑 **second_order** 陈旧信念查询。

### 评测协议

| 轴 | 设置 |
|----|------|
| **骨干/评判** | 主实验 gpt-4o-mini（T=0.1）；附录 A13 含 claude-haiku-4-5、qwen-plus、gemini-2.5-flash |
| **§5.1 事实接地** | 50 条环境查询 × 5 类 × 3 seeds；8 个基础环境工具；FA + 探测数 + 矛盾率 + 延迟 |
| **§5.2 隐含意图** | 100 条四场景（+ 25 条试点）；5 工具智能体状态注册表；literal_score + implicit_score |
| **子类** | availability, mood, appropriateness, latent_goal, second_order |
| **基线** | Literal、NoIntent（ReAct, B=3）、Plan-and-Solve、Fixed-Probe、Static Context、Vanilla、Reflexion 等 |

### 记忆与后端

- 记忆评分：新近性:重要性:词法相似 = 0.3:0.4:0.3，衰减 0.01；Tulving 四类（observation/conversation/reflection/plan），每 θ=10 观测触发反思；
- **双后端**：LLM（结构化 JSON + 四层 gap rubric + 与基准不相交的 few-shot）与启发式（离线/气隙）；报告结果均用 LLM 后端。

## Results & Highlights

### 隐含意图（§5.2，主贡献）

**Table 3 — 100 条四场景（3 seeds）**

| 条件 | 隐含分 | vs NoIntent |
|------|--------|-------------|
| Literal | 0.223 | — |
| NoIntent | 0.733 | — |
| **AURA Intent** | **0.804** | **Δ=+0.071, *p*=1.0×10⁻⁶** |

- 场景 A/B/C 分别 **+0.091 / +0.091 / +0.080**（*p* 显著）；场景 D（事后夜间）**+0.024, *p*=0.349**（公开状态已暴露可用性）。
- **25 条试点**（Table 2）：Intent implicit **0.803** vs NoIntent **0.640**（Δ=+0.16, *p*=0.006）；Plan-and-Solve **0.520**。
- **子类结构**（100 条 v2）：availability **Δ=+0.29**（*p*=2.7×10⁻¹¹）最大；appropriateness +0.11；mood +0.07；second_order 持平（表面已提示信念）；latent_goal **−0.09** 残留不足。

**消融与泛化**

- **Prompt 示例**：去掉示例 → gap 校准崩溃、Intent vs NoIntent 不显著；换为不相交示例仅降 0.037，对比仍显著 → **负载在 gap 校准而非答案模板记忆**。
- **后端**：LLM→启发式 IntentInferrer：0.803→**0.368**（Δ=−0.44）→ 提升绑定于 LLM 介导的 gap 推理。
- **跨骨干**：Claude Haiku Δ=+0.086（*p*=3.7×10⁻³）；Qwen-Plus Δ=+0.25；Gemini JSON 解析失败回退启发式。
- **人工评测**（8 评分者，50 对）：环境意识 +1.86、有用性 +1.58、可信度 +1.59、事实准确 +1.39（Wilcoxon 显著）；74% (scenario, dimension) 细胞达 ≥6/8 共识。

### 事实接地（§5.1，边界条件）

**Table 1 — FA（N=50, 3 seeds）**

| 方法 | FA | Probes/q | Contrad.% |
|------|-----|----------|-----------|
| Vanilla | 0.070 | 0.00 | 93.3 |
| Static Context | 0.450 | 0.00 | 70.0 |
| ReAct | 0.550 | — | 51.3 |
| Plan-and-Solve | 0.764 | 4.64 | 58.7 |
| Fixed-Probe | **0.766** | **8.00** | 56.0 |
| AURA GapRouted⋆ | 0.696 | **1.40** | 75.3 |
| AURA Full (B=2) | 0.640 | 0.00 | 66.7 |

- GapRouted **非 raw-FA 冠军**（vs Fixed-Probe Δ=−0.070, *p*=0.031），但在 **探测数 −82%**、披露成本 Pareto 上占优（Fig. 2）。
- **隐私切片**（N=30）：FA 与 Plan-and-Solve/ReAct 相当，**禁用工具违规 0%**（对照 25.6–100%）。
- **延迟**：中位 4.08s vs Fixed-Probe 2.37s（IntentInferrer 额外轮次）→ 选择性收益在探测/披露而非墙钟时间。
- Reflexion FA 崩溃至 0.174（反思撤回探测支持的主张）。

### Routine grounding

- 100 步 × 3 seeds：五架构 GA 差 ≤0.024，全部 *p*>0.5 → **指标饱和**，不区分机制（Appendix A16）。

## Authors & Credibility

- **Yang Li, Jiaxiang Liu, Jiang Cai, Mingkun Xu**（通讯作者，广东智能科学与技术研究院，@gdiist.cn）。
- 与 ReAct、Plan-and-Solve、Generative Agents、SOTOPIA、CondAmbigQA、FANToM、主动上下文智能体（ContextAgent/ProAgent/PROBE）等文献系统对比，定位清晰：**在已收到查询下，探测什么私有状态应被 surfaced**，与「何时主动协助」及多模态连续感知正交。
- **优势**：双评测轴分离「隐含意图精度赢」与「事实查询访问 Pareto」；配对检验 + seed 聚类 bootstrap；开源仿真器与 100 条标注基准；严格 precision rescore（Appendix A9）与 disjoint-example 消融。
- **局限**：作者自写基准（κ=0.61）；评判与策略同族 gpt-4o-mini；5 智能体小规模仿真；未与「全量塞上下文」基线正面交锋；跨域 FANToM/LoCoMo/GAIA 无提升（Appendix A20–A22）。

## Critical Assessment

**新颖性**：将「隐含需求推断」从 ReAct 循环中**因子化为独立预工具路由决策**，用标量 gap 统一预算与工具短名单，概念简洁且与 CondAmbigQA 的文本条件消歧形成互补（隐含需求在**他者私有状态**而非共享文本）。IntentFrame 字段设计完整（含告警 *a*），对 situated agent 产品化有直接接口价值。

**严谨性**：
- **优点**：明确 regime scope；主指标 + 子类分解 + 隐私切片 + 后端/提示/骨干消融链条完整；诚实报告事实域非精度赢家与延迟代价；
- **弱点**：同族 judge–policy；100 条基准规模有限且作者撰写；latent_goal 子类仍落后；Gemini 静默回退启发式可能低估跨模型鲁棒性；人类评测 Krippendorff α=0.43，幅度估计不确定。

**诚实度**：Limitations 与 Conclusion 明确机制不泛化至无工具隐藏状态或已 in-context 私有信息场景；不声称幻觉率全面降低（Contrad.% 仍高）；区分 diagnostic 上界（Fixed-private、Oracle-intent）与可部署系统。

**可复现性**：代码、仿真器、基准与运行记录已发布；超参见 Appendix A6；固定 60×60 子集与 demo chunk 世界分离说明清楚。

## Strengths

- 精准刻画「字面答对、隐含答漏」这一 situated agent 常见失败模式，示例（Lin Wei 在哪）具传播力。
- IntentFrame + *B(g)* 将探测成本与隐含需求不确定性显式挂钩，实践中作**上限**可节省大量低 gap 查询的 API 调用。
- 100 条四场景基准 + 五子类标签支持细粒度错误分析（availability 收益最大符合设计预期）。
- 隐私切片 0% 违规展示 gap 路由的**结构性**安全收益（低 gap 跳过注册表即无法调用禁用工具）。
- 开源生态完整，利于社区扩展多轮 gap 更新与学习化 *B(g)* 映射（论文已指出两条自然延伸）。

## Limitations

- **适用范围窄**：仅 tool-mediated hidden state 的结构化多智能体场景；不涵盖具身、开放世界、多用户并发规划。
- **事实查询**：FA 低于 Fixed-Probe/Plan-and-Solve；矛盾声明率仍 ~75%；中位延迟更高。
- **基准与标注**：100 条作者编写；second_order 与 FANToM 等外部信念基准**不自动可迁移**。
- **latent_goal 子类**：Intent 仍低于 NoIntent 或存在残余缺口（Δ=−0.09），需更强目标推理或专用工具。
- **依赖 few-shot gap 校准**：移除示例则增益消失；启发式后端几乎失效，部署需稳定 JSON 解析（Gemini 反例）。
- **规模**：5 智能体仿真测试 per-agent 机制而非 emergent 多智能体现象；routine grounding 指标无鉴别力。

## 相关性 (Relevance)

- **Situated LLM agents / 工具使用**（manifest relevance_score=4）：在场景化智能体中通过意图导向探测 surface 隐含用户需求，与 ReAct、Generative Agents、多智能体社交仿真直接对话。
- **主动助手与隐私**：与 ProAgent/ContextAgent 类「何时介入」互补；gap 路由在隐私敏感事实查询上提供可审计的探测预算与 0% 禁用工具违规。
- **评测与基准**：100 条隐含意图基准 + AURATown 可作为隐含需求覆盖率、探测效率的共享测试床；与 CondAmbigQA（文本歧义）形成 regime 对照。
- **Agent 产品设计**：IntentFrame 可作为 API 层控制面（预算、工具白名单、heads-up 告警），适用于办公协作、社交模拟、客服等需读取他人不可用状态的助手。

## Questions for Further Exploration

1. **多轮 gap 更新**：对话推进中增量更新 IntentFrame 能否在 follow-up 上再降 30–50% 探测而不损 implicit coverage？
2. **学习化 *B(g)* 映射**：从交互日志拟合 gap→预算 是否超越手写阶梯函数与 few-shot？
3. **latent_goal 补强**：是否需要专用 `get_unspoken_goal` 链式探测或更强规划推理才能消除 −0.09 子类缺口？
4. **全上下文基线**：与「塞满环境状态 + 记忆检索策略」的公平 token 成本对比能否改变 Pareto 叙事？
5. **异构评判**：gpt-4o-mini 评判下结论在 Claude/Gemini 评判下是否保持 Δ≈+0.07？
6. **真实部署**：从 AURATown 迁移到日历/IM/企业目录 API 时，公开–私有分裂与工具注册表如何映射？
7. **与 CondAmbigQA 联合**：文本条件消歧 + 私有状态探测能否统一为「隐含需求解析」两层架构？
