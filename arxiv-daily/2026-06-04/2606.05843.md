> **一句话总结：** 用 token 级指标 RAM 在多种 MLLM 与四类视觉任务上识别出专注跨模态检索的 **CoRe（Context-aware Retrieval）注意力头**——约 5% 顶级头因果不可或缺、约 30 个头跨任务稳定激活；其余头负责全局聚合；据此做头级混合稀疏注意力可在 prefill 上实现约 **1.8–2.3×** 加速且性能基本不降甚至略升。

> **TL;DR:** Extending QRhead-style probing to multimodal settings, the authors define **Retrieval Attention Mass (RAM)** to rank attention heads by query→ground-truth-visual attention. A sparse **CoRe head** subset performs localized cross-modal extraction; most heads diffuse globally. Causal masking shows top-5% CoRe heads are necessary (e.g., Qwen3-VL-4B MMLongBench accuracy 45.3→27.0 when masking top 5). ~30 heads are task-agnostic anchors (Spearman 0.79–0.92 across RefCOCOg, VidSTG, MMLongBench, MMDocIR). A CoRe-guided hybrid (full attention on top CoRe heads + Stream Sparse Attention elsewhere) yields up to **2.3×** prefill speedup on MLVU with ≤3% drop—or marginal gains—on Qwen3-VL and LLaVA-OneVision.

## 核心贡献

1. **RAM 指标与 CoRe 头定义**：将文本 LLM 的 QRhead 探测框架扩展到多模态场景，提出 **Retrieval Attention Mass (RAM)**——在指令 query token 上对 ground-truth 视觉区域 token \(\Omega(V^*)\) 的注意力质量聚合，区分「定向检索头」与 attention sink；高 RAM 头定义为 **Context-aware Retrieval (CoRe) heads**。

2. **跨域、跨架构、跨尺度的功能稀疏性实证**：在 RefCOCOg（静态 grounding）、VidSTG（时空 grounding）、MMLongBench（长文档多证据推理）、MMDocIR（文档检索）四类任务上，对 Qwen3-VL（4B/8B/32B）、LLaVA-OneVision、InternVL3.5 系统分析，揭示 **CoRe 头 = 局部信息提取器** vs **Vision heads（query→全部视觉 token 聚合）= 全局语义传播** 的功能分工。

3. **因果必要性与信息稀疏性**：对 top/bottom/random 头做 attention masking；**仅消融 top 5% CoRe 头** 即显著损害多模态推理，bottom 头影响微弱。Top-15 CoRe 头集中 14.4%（LLaVA）/ 25.6%（Qwen）关键视觉 token，bottom-15 仅约 2%；top 50–100 头即可覆盖 70–80% 关键视觉 token。

4. **跨任务稳定性**：Qwen3-VL-4B 上四数据集 head 激活 Spearman 相关 **0.79–0.92**；约 **30 个头** 在四任务 top-5% 中稳定出现（middle-to-late layers，约 L13–L24），呈现「全局一致、局部可变」范式。

5. **CoRe 引导的系统级加速**：prefill 阶段对 top-k CoRe 头保留 Full Attention，其余 vision 头使用 Stream Sparse Attention [24]；在 MLVU 上 LLaVA 保留 19.1% 头得 **1.8×** 加速且平均分略优于 Dense；Qwen3-VL-32B 仅 **4.9%** 密集头仍 **2.1×** 加速、平均分仅降 0.7。

## Key Contributions

1. **RAM metric and CoRe head identification**: Token-level cross-modal retrieval score restricting attention summation to annotated \(\Omega(V^*)\), enabling isolation of specialized retrieval heads vs. sinks.

2. **Functional sparsity as a structural principle**: Empirical dichotomy between sparse CoRe extractors and diffuse global vision heads, validated across four benchmark modalities and three MLLM families at multiple scales.

3. **Causal and concentration evidence**: Top-ranked CoRe heads are indispensable (sharp accuracy collapse under top-k masking); key visual semantics concentrate in a compact elite head subset with rapid coverage saturation.

4. **Cross-task anchor heads**: High Spearman stability (~30 persistent heads) suggests a shared cross-modal retrieval mechanism with dataset-adaptive modulation.

5. **CoRe-guided hybrid attention for efficient prefill**: Deterministic head-level sparsity exploits localized functional sparsity for 1.8–2.3× speedups with minimal—or occasionally positive—performance impact on MLVU (VideoMME trends in appendix).

## Methodology

### RAM 与探测流程（§3）

对每个注意力头 \(h\)，在指令 query 集合 \(q\) 上定义：

\[
M_{\mathrm{RAM}}^{(h)}(q \to V^*) = \mathbb{E}_{x \in q}\left[\sum_{y \in \Omega(V^*)} A_{x \to y}^{(h)}\right]
\]

其中 \(A_{x \to y}^{(h)}\) 为头 \(h\) 从文本 token \(x\) 到视觉 token \(y\) 的注意力权重；\(\Omega(V^*)\) 由任务标注映射到离散视觉 token（见 Table 1 与附录 C）。

**流程**（Figure 3）：
1. 将多模态序列划分为文本上下文、**关键视觉 token**、**指令 query token**；
2. 前向提取全层 self-attention，仅保留 **text-to-vision** 子矩阵（剔除 text-text、vision-vision 噪声）；
3. 按式 (1) 为每个 (layer, head) 计算标量 RAM，得到全局分布；高分头即 CoRe heads。

**统一评估协议**（Table 1）：

| 数据集 | 模态 | 任务 | 关键实体 \(V^*\) |
|--------|------|------|-------------------|
| RefCOCOg | 静态图 | Visual Grounding | bbox \([x,y,w,h]\) |
| VidSTG | 视频 | Spatio-temporal Grounding | 时空 tube \(T_{t_1:t_2}\) |
| MMLongBench | 长文档 | Multi-evidence Reasoning | 证据区域 \(\{R_k\}\) |
| MMDocIR | 文档+图 | Document-grounded Retrieval | 相关页/图区域 |

**模型覆盖**：Qwen3-VL-4B/8B/32B-Instruct、LLaVA-OneVision (qwen2-7b-ov)、InternVL3.5-8B。

### 对比基线：Vision Heads

除 RAM 选出的 CoRe 头外，另用 **query→全部视觉 token** 的宏观注意力质量聚合识别 **Vision Heads**，用于对比「语义选择」vs「全局聚合」的分布形态（Figure 4 上下 panel）。

### 因果干预与稀疏性指标（§4.4）

- **Head masking**：在 MMLongBench 上消融 \(k \in \{5,10,20,30\}\) 个头，策略为 Top-k（按 RAM）、Bottom-k、Random-k。
- **Key Token Ratio**：单头 top-5% 注意力 token 与 GT 关键 patch 重叠率（精度）。
- **Key Token Coverage**：多头聚合 top 注意力 token 对关键 patch 的累计覆盖率。

### CoRe 引导混合注意力（§5）

- **Prefill 阶段**：top-k CoRe 头 → **Full Attention**；其余非关键 vision 头 → **Stream Sparse Attention**（局部滑动窗口 [24]）。
- **评估**：MLVU dev（holistic TR、single-detail NQA/ER/PQA、multi-detail AO/AC）；Dense = 100% Full Attention；消融保留 CoRe 头比例（如 1.7%–24.4%）；附录 G.3 含 VideoMME。

**代码**：[https://github.com/aaxiyao/CoRe_Head](https://github.com/aaxiyao/CoRe_Head)

## Results & Highlights

### 架构与尺度差异（§4.1, Figure 4）

- **Qwen3-VL 扩 scale**：4B 时 CoRe 头分散于中层；32B 时收敛至 **深层连续块（约 L44–L56）**，形成跨模态语义整合「瓶颈」。
- **跨族对比**：Qwen3-VL **稀疏局部化**；LLaVA-OneVision **中等密度、纵向分散**；InternVL3.5 **全深度高密度**。作者归因于：(1) 视觉编码器能力（大 encoder 预对齐 vs LLM 承担桥接）、(2) 对齐范式（LLaVA 多阶段渐进 vs Qwen 原生联合优化）、(3) 即使共享 Qwen backbone，LLaVA 与 Qwen3-VL 拓扑仍不同——**多模态整合稀疏性非 base LLM 固有，而是对齐策略的动态分配**。

### 功能解耦（§4.2）

CoRe 头：稀疏、局部位点，执行关键视觉特征抽取。Vision Heads：中层至后期 **连续、沿 head 维广泛覆盖**，承担全局视觉信号编码与传播——网络自发形成功能分解，少数专门头 + 大量辅助头。

### 跨任务稳定性（§4.3, Figure 5）

- Spearman：RefCOCOg–MMLongBench 0.79，VidSTG–MMLongBench **0.92**，MMDocIR 与其余任务 0.81–0.88。
- **~30 个头** 在四任务 top-5% 中反复出现，集中于 **L13–L24**；少量头呈数据集特异性激活。

### 因果与稀疏性（§4.4, Figure 6）

| 干预 | Qwen3-VL-4B (MMLongBench Acc.) | LLaVA-OneVision |
|------|-------------------------------|-----------------|
| Baseline | ~45.3 | ~16.4 |
| Mask **top 5** CoRe | **27.0** | **10.6** |
| Mask top 30 | <7（能力崩溃） | <7 |
| Mask bottom-k | 接近 random，影响小 | 同左 |

- Top-15 vs Bottom-15 **Key Token Ratio**：Qwen 25.6% vs ~2%；LLaVA 14.4% vs ~2%。
- Top 50–100 头累计覆盖率快速饱和至 **70–80%** 关键视觉 token。

### 系统加速（§5, Table 2, Figure 7）

**代表性配置**（MLVU Perf. Avg. vs Dense，Speedup）：

| 模型 | 保留 Dense CoRe 比例 | Speedup | Perf. Avg. 变化 |
|------|---------------------|---------|----------------|
| LLaVA-OneVision | 19.1% | **1.8×** | **↑**（略优于 Dense） |
| Qwen3-VL-8B | 24.4% | **1.8×** | **↑** |
| Qwen3-VL-8B | 13.0% | 2.1× | ↓0.5 |
| Qwen3-VL-32B | **4.9%** | **2.1×** | **↓0.7** |

- Holistic / broad detail 任务在激进稀疏下通常仅 **1–3%** 降幅；ER、AO、AC 等精细时空任务有时 **因噪声过滤而略升**（隐式正则化假说）。
- 序列越长，相对 Dense 的延迟优势越明显（Figure 7：8k–128k tokens）。

### 可视化证据（Figures 1–2）

RefCOCOg / VidSTG 上：top CoRe 头高注意力对齐 query 相关实体（红车、白衣成人抱狗）；bottom 头扩散至背景/无关区域，且 **Kept Tokens** 中与上下文相关的比例低。

## Critical Assessment

**新颖性**：将「功能稀疏性」从文本 LLM 的 specialized heads 系统推广到 **MLLM 跨模态检索**，RAM 在 annotated \(\Omega(V^*)\) 上给出可操作的 head 排序，并用因果 masking + 系统级 hybrid attention 闭环验证「稀疏不仅可解释、还可优化」——比纯 entropy/aggregation 启发式 [12, 17] 更贴近 query-conditioned token 级机制。

**严谨性**：
- **优点**：统一 \(V^*\) 形式化跨四类数据集；text-to-vision 子矩阵降噪合理；Top/Bottom/Random 三组消融设计清晰；跨任务 Spearman + layer-head 稳定性矩阵支撑「~30 anchor heads」主张；加速实验覆盖多模型、多 MLVU 子任务与长序列延迟。
- **弱点**：RAM **依赖 GT 空间/时间标注**，无法直接用于无标注推理；主要分析 **late-fusion Transformer** MLLM，早融合或非 Transformer 泛化未验证；静态 head mask 在 OOD 查询上可能次优；InternVL 等高密度激活模型的 CoRe 定义与稀疏加速策略需更细致对照。

**诚实度**：附录明确三大局限（标注依赖、架构泛化、静态路由）；对「有时稀疏反而提升」给出噪声过滤解释，但未做严格对照实验隔离该效应。

**可复现性**：公开代码、明确数据集与模型 checkpoint 名称、附录含伪代码；计算 RAM 需访问 attention map 与 GT 区域映射，成本高于纯 forward-only 探测但可接受。

## Strengths

- 首次在 **多模态、多任务、多架构、多尺度** 面板上用统一 RAM 指标刻画 **跨模态检索的功能稀疏性**，并连接机制解释与推理加速。
- CoRe vs Vision Heads 的功能二分有分布图、因果消融、token 集中度三重证据，叙事一致。
- 跨任务 Spearman 0.79–0.92 与 ~30 稳定头，为「共享检索电路 + 任务自适应调制」提供可检验假说。
- CoRe-guided hybrid attention 在 **2× 级 prefill 加速** 下保持竞争力，对长视频/长文档 MLLM 部署有直接工程价值。
- 架构对比（Qwen vs LLaVA vs InternVL）指出 **稀疏性是对齐与 encoder 设计的结果**，避免将现象简单归因于「大模型必然如此」。

## Limitations

- **RAM 需 GT 标注**（bbox、tube、证据区），难以推广到纯抽象推理或无标注场景；需未来 work 用自监督或弱监督近似 \(\Omega(V^*)\)。
- 模型覆盖虽广，仍以 **晚期融合 ViT+LLM** 为主；early-fusion、非 Transformer MLLM 是否同样出现 CoRe 分工未证。
- 加速采用 **确定性、静态** top-k 头配置；复杂 OOD 查询可能需要 input-adaptive 动态路由。
- 主要因果数字来自 **MMLongBench + Qwen3-VL-4B / LLaVA**；其他任务上的 top-5% 消融定量表文内较简（摘要级「显著下降」）。
- 与 SAE / 回路级 circuit 发现未深入结合；未分析 CoRe 头与 FFN、vision projector 的层级回路。
- 「稀疏反而提升」需更多消融（如仅稀疏 non-CoRe 而保持总 FLOPs 等）以排除评估噪声。

## 相关性 (Relevance)

- **Mechanistic interpretability**：为 MLLM 提供可命名的 **CoRe head** 实体与 RAM 探测工具，补充 Head Pursuit、视觉 sink、counting circuits 等线。
- **MLLM 效率**：头级 hybrid sparse prefill 是 attention sink 缓解 [7] 之外的 **结构化稀疏** 路径，与长上下文视频/文档场景高度相关。
- **Visual grounding / 幻觉**：理解 query→关键视觉 token 的专门头，有助于分析 grounding 失败与幻觉时的注意力弥散模式。
- **架构设计**：深层 CoRe 瓶颈随 scale 涌现、encoder 强度影响头拓扑——对 MLLM 训练阶段（对齐 vs 预对齐）有设计启示。
- **Daily paper 主题契合**：relevance_score 5——直接对应多模态 LLM 机制可解释性与功能稀疏性。

## Questions for Further Exploration

1. **无标注 RAM**：能否用模型自身 grounding 输出、Grad-CAM 或 SAE 特征近似 \(\Omega(V^*)\)，在无 bbox 任务上仍稳定排名 CoRe 头？
2. **动态 CoRe 路由**：按输入 query 在线选择 top-k 头，能否在保持 2× 加速的同时改善 OOD 与多轮对话？
3. **与 hallucination 的因果链**：消融 CoRe 头是否系统性增加 object hallucination？与 [28] 异常注意力检测如何结合？
4. **训练干预**：能否在 SFT/RLHF 中 **强化 CoRe 头、抑制 bottom 弥散头**，提升 grounding 而非仅推理后稀疏？
5. **InternVL 高密度 regime**：在「几乎全层激活」模型上，CoRe 是否仍稀疏，或需重新定义 RAM 阈值与加速策略？
6. **Decode 阶段**：本文聚焦 prefill hybrid attention；CoRe 头在 autoregressive decode 是否同样稀疏可剪？
7. **跨模型迁移**：Qwen3-VL 上识别的 ~30 anchor heads 能否映射到 LLaVA 的 (layer, head) 索引（如 via cross-model activation patching）？
