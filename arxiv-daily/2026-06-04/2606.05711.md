> **一句话总结：** 提出 **WHAT / WHICH / HOW** 三轴统一框架，系统梳理 2024–2026 年 **18** 种基于 LLM 的多智能体**潜空间通信**方法（嵌入 / 隐状态 / KV-cache 等），对比自然语言通道在**推理成本、离散化信息损失、冗余歧义**上的结构性劣势，并归纳五条设计规律与六条开放问题（跨架构对齐、潜通道安全、压缩、与 latent CoT 统一等）。

> **TL;DR:** Yingzhuo Liu (BUPT) surveys **latent communication** in LLM-based multi-agent systems: agents exchange continuous embeddings, hidden states, or KV-caches instead of tokenized natural language. A **3-axis framework** — **WHAT** (communicated quantity), **WHICH** (sender–receiver alignment), **HOW** (fusion: concat, prepend, math ops, cross-attention, cache restoration) — categorizes **18 representative methods (2024–2026)**. NL-Comm suffers ~2× per-message FLOPs overhead, ~10³–10⁴ information compression per token vs. hidden states, and fluency-driven redundancy; latent channels win on tightly coupled intermediate signalling but lose interpretability. Five takeaways (e.g., KV > hidden state > embedding on richness/cost/dependence; training-free dominance; KV-cache as emerging default) and six open problems (cross-architecture alignment, latent-channel security, compression, theory, latent CoT unification, edge deployment). Companion repo: [Awesome-Latent-Communication](https://github.com/enochliu98/Awesome-Latent-Communication).

## 核心贡献

1. **三轴统一设计空间**：任意潜通信协议可表述为 **Method = (WHAT, WHICH, HOW)**——传输何种连续量、发送端与接收端如何对齐、接收端如何融合；三轴近似正交，理论组合约 **45** 个概念位点，已调研方法占据约 **17** 个。

2. **18 篇代表性方法系统分类**：覆盖 **CIPHER、AC、Interlat、SDE、ThoughtComm、MoT**（隐状态/嵌入类）与 **KVComm、C2C、LatentMAS、Q-KVComm、LRAgent、RelayCaching、Agent Memory、Agent Primitives、Edge LLM Handover**（KV 类）及 **Vision Wormhole、BIGMAS**（异构/混合），并纳入 **Five Ws Survey** 作为元综述对照。

3. **自然语言 vs. 潜空间权衡的量化论证**：每轮 NL 消息约 **2× O(L·T·d)** 额外 FLOPs；单 token 互信息上界约 **15–17 bits**，末 token 隐状态容量约 **40,000+ bits**；明确 **混合协议** 立场——对用户可见输出用 NL，智能体间中间信号用 latent。

4. **五条可泛化设计规律（Takeaways）**：信息–成本–架构依赖排序（KV > Hidden > Embedding）；Prefill 阶段 KV 方法可跳过发送方 decode；同骨干下 Last→First 与 All→Corresponding 均简单有效；融合策略从 concat/prepend（常 training-free）到 cross-attention（需训练）呈表达力谱；**多数方法 training-free**。

5. **六条开放问题与研究议程**：跨架构对齐、潜通道安全与对抗、KV 压缩/量化、信息论/学习论解释、与 **Latent CoT** 的统一框架、边缘/移动部署（LRAgent、Agent Memory、Edge Handover 等早期工作）。

## Key Contributions

1. **Unified 3-axis framework** (WHAT / WHICH / HOW) that uniquely describes the design space of latent communication protocols in LLM-based MAS.

2. **Systematic survey of 18 methods (2024–2026)** with per-method placement, strengths/limitations, and a unified comparison across information type, alignment, and fusion.

3. **Quantified case for latent communication** over NL-Comm on inference cost, discretization information loss (~10³–10⁴ compression), and semantic redundancy/ambiguity, with a hybrid deployment recommendation.

4. **Five generalizable takeaways** on information–cost–dependence trade-offs, prefill vs. decode phase choices, layer-mapping strategies, fusion spectrum, and the training-free paradigm.

5. **Six open problems** plus bridges to latent CoT, MARL, emergent language, and KV-cache compression; continuously updated companion awesome-list.

## Methodology

### 问题设定与符号

- **多智能体 LLM 系统（MAS）**：\(N\) 个智能体 \(A=\{A_1,\ldots,A_N\}\)，各带角色 system prompt、可选工具与**通信信道**。
- **NL-Comm**：发送方 decode → token 传输 → 接收方 re-encode（**语言瓶颈**）。
- **Latent-Comm**：发送方暴露 **Embedding / Hidden State / KV-Cache / Activation** 等连续表示，接收方直接注入计算，不经词表往返。
- **Prefill vs. Decode**：Prefill 产生全序列 per-token 隐状态与 KV；Decode 仅末 token 隐状态 + 增长的 KV——决定可传输的内部状态类型。

### 三轴框架（Section 4）

| 轴 | 问题 | 主要选项 |
|----|------|----------|
| **WHAT** | 传什么？ | **Embedding**（如 CIPHER 加权软嵌入）、**Hidden State**（AC、Interlat、SDE、ThoughtComm、MoT）、**KV-Cache**（KVComm、C2C、LatentMAS 等 9 篇）、**Other**（state delta、持久 Q4 KV、UVC 视觉通道、BIGMAS workspace） |
| **WHICH** | 如何对齐？ | **潜空间对齐**（同模型 / 同骨干投影 / 异构 codec 或 interaction layer）；**层对齐**（Last→First、All→Corresponding、Selected→Selected、sparse top-k） |
| **HOW** | 如何融合？ | **Concatenation**、**Prepending**（KV 前置）、**Math op**（加/减/线性）、**Cross-attention**（MoT）、**Cache restoration**（RelayCaching、Agent Memory） |

### 调研范围

- 时间：**2024–2026**；方法数：**18**（含 1 篇聚合综述 Five Ws）。
- 每方法按 **(a) 核心思想 (b) 框架定位 (c) 优劣 (d) 报告结果与代码** 结构化分析（Section 5）。
- **Training-free 范式**（Section 6）：Table 4 显示 CIPHER、AC、SDE、KVComm、LatentMAS、Q-KVComm、LRAgent、RelayCaching、Agent Memory 等多数为 **✓ training-free**；异构对齐与复杂融合（C2C fuser、Vision Wormhole、MoT）需训练。

### 何时优先 Latent vs. NL（Section 3.3）

Latent 更优当：**(1)** 智能体紧耦合；(2) 中间消息无需人类可见；(3) 共享或可对齐潜空间；(4) 延迟/成本为硬约束。NL 更优当：可解释性、跨组织互操作、人工审计/安全审查为必需。

## Results & Highlights

### 信息类型与代表方法（Table 1 摘要）

| 类型 | 信息丰富度 | 传输成本 | 架构依赖 | 代表方法 |
|------|------------|----------|----------|----------|
| Embedding | 低 | 低 | 低 | CIPHER（加权 vocab 分布软嵌入） |
| Hidden State | 中 | 中 | 中 | AC (+27% vs NL)、Interlat (**24×** 延迟↓)、SDE（state delta，复杂推理 SOTA 报告） |
| KV-Cache | 高 | 高 | 高 | LatentMAS（协作推理 SOTA 报告）、RelayCaching (**>80%** 复用、**4.7×** TTFT）、Agent Memory (**22×–136×** TTFT) |

### 对齐与融合（Tables 2–3）

- **同模型**：无需显式潜空间对齐（CIPHER、AC、KVComm 等）。
- **同骨干不同微调**：Interlat、C2C 等用**学习投影**。
- **异构骨干**：Vision Wormhole（UVC + 视觉注入，O(N) hub-spoke）、MoT（interaction layers + cross-attention，+0.38% ID / +2.92% OOD vs Avengers）。
- **融合主流**：KV 类多 **Prepend**；隐状态类多 **Concat** 或 **加法**；RelayCaching / Agent Memory 为 **Cache restoration**。

### 实证趋势汇总（Section 7，跨方法不可直接 head-to-head）

| 维度 | 典型报告区间 |
|------|----------------|
| **延迟** | 相对 NL-Comm **2×–24×** 降低（Interlat 长上下文极值） |
| **TTFT** | KV 复用/持久化 **4.7×–136×**（RelayCaching、Agent Memory） |
| **Token 用量** | 约 **3–4×** 减少（Agent Primitives：+12–16.5% 准确率，仅 1.3–1.6× 相对单智能体开销） |
| **准确率** | 多数 ≥ NL 基线；SDE、MoT、LatentMAS 等报告 SOTA 类增益 |
| **设计趋势** | **18 篇中 9 篇用 KV-cache**，占比上升；**长上下文**收益最大；**同骨干**假设仍主导 |

### 单方法亮点（论文报告）

| 方法 | 关键数字/结论 |
|------|----------------|
| **AC** | ~+27% 数学/推理基准 vs NL；ICML 2025 |
| **Interlat** | 最长上下文多智能体 **24×** 加速；ACL 2026 |
| **Q-KVComm** | KV **5–6×** 压缩，精度损失可忽略 |
| **RelayCaching** | **80%+** cache 复用，**4.7×** TTFT |
| **Agent Primitives** | +12–16.5% 准确率，3–4× 更低 token/延迟 |
| **Vision Wormhole** | 跨 Qwen-VL/Gemma/SmolVLM2 等九项推理基准 |
| **BIGMAS** | Game24、Six Fives、Tower of London 优于 ReAct/ToT |

## Critical Assessment

**新颖性**：在 latent MAS 文献爆发期提供**共享词汇表**与可组合设计空间，将碎片化方法（嵌入辩论、激活传递、KV 共享、边缘 handover、视觉 wormhole）纳入同一 **WHAT/WHICH/HOW** 坐标系；与 Five Ws 的 WHEN/WHY 轴形成互补（本文偏**表示与融合机制**）。

**严谨性**：
- **优点**：形式化 NL 瓶颈与互信息上界；三轴正交性论证 + Figure 3/4 分类树；逐方法一致模板；training-free 统计与何时需训练的清晰划分；诚实承认不可直接横向比数。
- **弱点**：综述非原创实验；部分结果为各论文自述 SOTA，未统一复现；18 篇选取边界依赖 awesome-list，或有遗漏；安全讨论（§8.2）多为问题陈述，无实证防御方案。

**诚实度**：明确 NL 在**可解释性/审计**上的不可替代性；指出 latent 通道**不透明、难调试**；列出六条开放问题而非过度宣称 latent 将取代 NL。

**可复现性**：图表源自公开方法插图；仓库持续更新论文与代码链接；作为**框架/综述**而非新算法，复现性体现在分类表与引用完整性。

## Strengths

- 降低新研究者进入 latent MAS 的门槛，提供可比较的 **(WHAT, WHICH, HOW)** 三元组描述。
- 量化 NL vs. latent 的**成本与信息论**动机（Figure 2），支撑混合协议工程决策。
- 覆盖从 **CIPHER (2024)** 到 **Edge Handover / Vision Wormhole (2026)** 的完整时间窗，KV、LoRA、边缘、移动场景均有触及。
- Training-free 主导规律的提炼对**即插即用**部署有价值。
- 与 Latent CoT、MARL、KV 压缩的 **Section 9** 关联清晰，便于跨子领域阅读。

## Limitations

- 单作者综述，无独立第三方复现或统一 benchmark 跑分。
- 18 篇方法选取非系统性 meta-analysis（无 PRISMA 式检索协议）。
- 跨架构、**潜通道安全**几乎空白，论文仅标为 critical gap。
- 与 Five Ws、Emergent Language 的对比偏概念层，缺少实验统一体。
- 部分 2026 预印本（Agent Memory、RelayCaching 等）结果需随社区验证更新。
- 未深入讨论**闭源 API 黑盒**场景下 latent 通道的可行性（依赖白盒权重）。

## Questions for Further Exploration

1. **统一 benchmark**：能否在 RainbowArena 或固定多智能体 QA 套件上，用同一骨干对 18 种 WHAT/HOW 组合做公平 latency–accuracy Pareto 曲线？
2. **混合协议自动化**：何时从 latent 切回 NL 的**可学习路由器**（结合 Liu et al. 2025a 任务分解选择）？
3. **潜通道安全**：对 hidden-state / KV 的对抗扰动检测与**可证安全边界**是否有可行方案？
4. **闭源模型**：仅 API 访问时，embedding-level（CIPHER 类）是否成为唯一可行 latent 路径？
5. **与 MCP/工具调用**：工具返回是否应进入 KV-cache 共享而非文本摘要（与 MCP 故障分类论文的 Tool Result Propagation 问题衔接）？
6. **Latent CoT + Latent Comm 统一图**：单模型内 state delta 与跨模型 delta 能否共用 SDE 式表示？
7. **理论**：在什么互信息/任务耦合条件下，latent 相对 NL 的加速上界可证明？
