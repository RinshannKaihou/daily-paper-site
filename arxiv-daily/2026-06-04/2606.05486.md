> **一句话总结：** PRIG（Probe-Targeted Residual Integrated Gradients）在线性探针区分「清晰/歧义」prompt 的基础上，对残差流中间层做截断 IG 归因，将**潜藏的 prompt 歧义**定位到 token/句子级；在 coding/math/writing 合成数据上达 **0.840 AUROC**，人类 gold 集 **0.891**，显著优于 Grad×Input/embedding IG，句子级 F1 **0.734** 亦优于 GPT-5.4（0.587），且跨域与 abstain 校准良好。

> **TL;DR:** Prompt ambiguity is latent—not an output logit—so standard attribution (perturbation, attention, embedding IG) fails. **PRIG** trains logistic probes on mean-pooled Llama-3.1-8B-Instruct residuals, uses tokenwise probe logit \(F_\ell(x)=\frac{1}{n}\sum_i \langle w_\ell, h_i^\ell\rangle\) as target, applies **50-step IG** on residual subgraph layers \(m\to\ell\) (zero baseline), sums over \(d\) dims, Gaussian smooths (\(\sigma=3\)). Synthetic data: GPT-5.4 rewrites one task-critical sentence per prompt (LeetCode/MATH/PromptTensor; 501/357/385) + 12 human gold prompts. Results: combined AUROC 0.840 | AUPRG 0.774; gold 0.891 | 0.870; vs Grad×Input 0.629/0.433 and IG 0.451/0.250; cross-domain avg 0.805 AUROC; sentence ID F1 0.734 vs GPT-5.4 0.587; clear-prompt false-positive AUROC ~0.452 (near random). Code: [github.com/govindramesh/LLM-Ambiguity-Attribution](https://github.com/govindramesh/LLM-Ambiguity-Attribution).

## 核心贡献

1. **新问题类：潜藏 prompt 属性的 feature-level token 归因**：不解释生成 logit，而是将线性探针解码的**歧义特征**归因到 token 位置的残差表示；区别于标准「输出→输入 embedding」归因范式。

2. **PRIG 方法**：三组件耦合——(a) 层 \(\ell\) 的 tokenwise 探针 logit；(b) 在层 \(m<\ell\) 残差子图上对 \(G_{\ell,m}(R)=F_\ell(f_{\ell:m}(R))\) 做 IG（零基线、50 步 Riemann）；(c) 一维高斯平滑（\(\sigma=3\)）。截断反向图以减轻长上下文 embedding 归因扩散，牺牲全局 IG completeness 换取局部层间可解释性。

3. **可评测的合成歧义数据**：coding（LeetCode hard, ≥5 句, 501）、math（MATH, ≥4 句, 357）、writing（PromptTensor planning/generation, 385）；GPT-5.4 改写**一句**任务关键句引入歧义，配套 token 级二值 mask；另建 12 条人类 gold（AIME + SWE-Bench Pro 等），仅用于评测 OOD 泛化。

4. **强定位与跨域**：域内 PRIG AUROC 0.813–0.922；combined probe 0.840；gold 0.891。显著优于 Grad×Input 与 embedding IG（Table 1）。跨域平均 0.805 AUROC（in-domain 0.870）。句子级歧义识别 F1 **0.734±0.044** 优于 GPT-5.4 **0.587**，precision 0.808 vs 0.537。

5. **实践含义**：表明歧义等潜藏 prompt 性质可在**中间表示**中定位，支撑 prompt 调试、澄清问题生成与可靠性工程；开源实现基于 Llama-3.1-8B-Instruct。

## Key Contributions

1. **Feature-level attribution for latent prompt properties**: Attributes a probe-defined ambiguity signal to tokens via residual representations, not output logits or input embeddings.

2. **PRIG (Probe-Targeted Residual Integrated Gradients)**: Tokenwise linear probe logit + truncated residual-stream IG from layer \(m\) to probe layer \(\ell\) + Gaussian smoothing; 50 integration steps, zero residual baseline.

3. **Synthetic + gold evaluation harness**: Single-sentence rewrite ambiguity across coding/math/writing with token masks; 12 human-written gold prompts for distribution shift beyond GPT rewrites.

4. **Strong empirical gains**: 0.840 combined AUROC, 0.891 gold AUROC, large margins over Grad×Input and standard IG; cross-domain transfer; sentence-level F1 beats GPT-5.4 with better abstention on clear prompts.

5. **Broader interpretability claim**: Latent prompt features can be localized through intermediate representations, enabling debugging beyond output-level explanation.

## Methodology

### 问题设定

Prompt \(x=(x_1,\ldots,x_n)\)；transformer 第 \(\ell\) 层 token \(i\) 残差 \(h_i^\ell \in \mathbb{R}^d\)。歧义为**输入时**的潜藏属性，非某一续写。

每层训练无截距 logistic probe（mean-pool 残差）：

\[
\bar{h}^\ell(x)=\frac{1}{n}\sum_{i=1}^n h_i^\ell,\quad F_\ell(x)=w_\ell^\top \bar{h}^\ell(x)
\]

归因用 tokenwise 等价形式（式 4）：

\[
F_\ell(x)=\frac{1}{n}\sum_{i=1}^n \langle w_\ell, h_i^\ell(x)\rangle
\]

目标：为每个 token 分配 \(a_i\)，衡量其对 \(F_\ell(x)\) 的贡献。

### PRIG 三步

1. **Probe-targeted logit**：线性探针方向 \(w_\ell\) 与每个 token 表示点积，作为 IG 标量目标。

2. **Residual IG（层 \(m\to\ell\)）**：\(R^m(x)\in\mathbb{R}^{n\times d}\)；\(G_{\ell,m}(R)=F_\ell(f_{\ell:m}(R))\)；路径 \(R_\alpha^m=\alpha R^m(x)\)；

\[
A_{i,j}^{\ell,m}(x)=h_{i,j}^m(x)\int_0^1 \frac{\partial G_{\ell,m}(R_\alpha^m)}{\partial h_{i,j}^m}\,d\alpha,\quad a_i^{\ell,m}=\sum_{j=1}^d A_{i,j}^{\ell,m}
\]

50 点 Riemann 近似；仅反向穿过 probe 前局部层，保持 token 位置对齐。

3. **Gaussian blur**：\(\tilde{a}_i=\sum_j K_\sigma(i-j)a_j\)，\(\sigma=3\)，缓解子词切分与梯度噪声。

### 数据构建

| 域 | 来源 | 过滤后规模 | 歧义构造 |
|----|------|-----------|----------|
| Coding | LeetCodeDataset | 501 | hard, ≥5 句，去 examples |
| Math | MATH | 357 | ≥4 句，去含图 |
| Writing | PromptTensor | 385 | planning/generation, ≥4 句，去重 |

每 prompt 用 GPT-5.4 改写**一句**使任务不清；清晰/歧义二分类训练 probe；歧义句内 token 为正类 mask。25% hold-out。Gold：12 条（每域 4），math 来自 AIME，coding 来自 SWE-Bench Pro。

### 实验设置

- **模型**：Llama-3.1-8B-Instruct + chat template（评测在去掉 template 的原始 prompt token 上）。
- **Probe**：每层 mean-pool 残差上单独 logistic regression；accuracy 约第 5–6 层陡升，**~15 层峰值**后略降。
- **层选择**：每域在训练集上选 probe 层 \(\ell\) 与 attribution 层/区间 \(m\)（Appendix D）；非全局固定配置。
- **指标**：AUROC、AUPRG（imbalanced 正类仅一句）；句子级 F1/Precision/Recall（PRIG 句内 mean 归因 + 5-fold CV 阈值 \(\tau\) abstain）。

### 基线

- **Grad×Input**：探针 logit 对 input embedding 梯度 × embedding。
- **IG**：embedding 空间标准 IG（50 步，探针 logit 为目标）。
- **GPT-5.4**：零样本识别歧义句或返回 None（Appendix B prompt）。

## Results & Highlights

### Token 级定位（Table 1，AUROC | AUPRG）

| Method | Coding | Math | Writing | Combined | Gold |
|--------|--------|------|---------|----------|------|
| **PRIG** | **0.922 \| 0.876** | **0.813 \| 0.720** | **0.876 \| 0.867** | **0.840 \| 0.774** | **0.891 \| 0.870** |
| Grad×Input | 0.602 \| 0.382 | 0.643 \| 0.404 | 0.544 \| 0.409 | 0.629 \| 0.433 | 0.749 \| 0.556 |
| IG (embed) | 0.395 \| 0.169 | 0.422 \| 0.196 | 0.701 \| 0.557 | 0.451 \| 0.250 | 0.499 \| 0.272 |

PRIG 在所有域大幅领先；残差空间归因优于 embedding 空间，支持「残差位置≈原 token 位置」假设。

### 跨域（Table 2）

In-domain 平均 **0.870 AUROC / 0.821 AUPRG**；out-of-domain **0.805 / 0.688**，仍远高于随机与 Table 1 基线。Coding probe → Writing 仍达 0.864 AUROC。

### 句子级 vs GPT-5.4（Table 3）

| Method | F1 | Precision | Recall |
|--------|-----|-----------|--------|
| **PRIG** | **0.734 ± 0.044** | **0.808 ± 0.026** | 0.675 ± 0.063 |
| GPT-5.4 | 0.587 | 0.537 | 0.647 |

PRIG precision 更高，更少在清晰 prompt 上「幻觉歧义」；完全指定 prompt 上 PRIG AUROC **0.452**（近随机），GPT-5.4 更易误报。

## Critical Assessment

**新颖性**：将「歧义定位」从输出解释转为 **probe-targeted representation attribution**，并与截断残差 IG 结合，在 latent prompt property 归因上定义清晰的问题类与方法名（PRIG）。与 attention/perturbation/patching 针对可观测行为的路线形成对比。

**严谨性**：
- **优点**：三域 + combined + human gold；AUROC 与 AUPRG 双指标；跨域表；句子级与 GPT-5.4 对照；清晰 prompt 假阳性检验；开源。
- **弱点**：仅 Llama-3.1-8B-Instruct；probe/层 hyperparameter 在训练集上选取（报告为 best interval）；合成歧义分布依赖 GPT-5.4；gold 规模小；未与 activation patching / 稀疏神经元歧义检测 [Zhang et al. 2025b] 等直接对比。

**工程意义**：对 agent/代码/数学/写作类长 prompt，PRIG 可提供**哪一句欠指定**的可视化信号，辅助澄清问题生成（CLAM、ClarifyGPT 等）的前置诊断；比直接问 frontier LLM 更不易在清晰任务上误报歧义。

## Strengths

- 首次系统地将 **prompt 歧义** 作为潜藏特征做 token 级归因，理论叙事与实现（PRIG）一致。
- 残差子图 IG + 高斯平滑在三个任务域与 gold 上 consistently 大幅超越 embedding 梯度基线。
- 数据管线可复现：单句改写 + token mask 使评测简化为排序问题，AUPRG 处理极度不平衡正类合理。
- 跨域 probe 仍保持强信号（avg 0.805 AUROC），暗示共享「歧义方向」而非纯表面启发式。
- 句子级 precision 与 abstain 行为优于 GPT-5.4，适合作为轻量诊断模块而非替代 LLM 推理。
- 代码已发布，实验锚定开源 Llama-3.1-8B-Instruct。

## Limitations

- **线性探针假设**：非线性或分布式歧义可能欠拟合；未探索非线性 probe 或 sparse features。
- **单模型族 / 8B 规模**：未验证 70B+、多模态或不同架构（Mistral、Qwen 等）。
- **合成 + GPT 偏差**：训练歧义来自 GPT-5.4 改写风格；gold 12 条不足以覆盖真实用户 prompt 复杂性。
- **Ground truth 定义**：歧义 = 被改写句；真实场景歧义可能跨句、依赖对话历史或世界知识。
- **层选择依赖**：每域 training-set 选 best \((m,\ell)\)，部署时需类似验证集或固定启发式（~15 层 probe peak 仅作参考）。
- **无下游闭环**：未证明定位后改写/澄清能提升任务成功率或降低幻觉。
- **计算成本**：每层 probe + 50 步 IG 相对单次 forward 更重，长 context 成本未量化。

## 相关性 (Relevance)

- **Prompt 工程 / LLM 可靠性**：与 daily_paper 过滤主题「probe 定位 prompt 歧义区域」直接对应（`filtered_papers.json` **relevance_score: 4**）。
- **Agent & 工具调用**：coding/math/writing 三域与 ClarifyGPT、underspecification 管理 [Yang et al. 2025] 同脉络——可在 agent 执行前对 user prompt 做 PRIG 扫描，触发澄清而非错误工具调用。
- **可解释性栈**：互补于 attention flow、activation patching（需 feature-present/absent 对比）；PRIG 适用于**无法干净删改**的 underspecification。
- **与 2606.05557（AURA 隐式需求探测）**：AURA 面向 situated agent 的 implicit need；PRIG 面向**显式 prompt 文本**中的歧义 span——可组合为「歧义句定位 + 意图补全」流水线。
- **与 2606.05384（LLM-as-judge 可操纵性）**：评判稳定性 vs prompt 输入歧义是不同 failure mode；PRIG 可帮助区分「判决翻转」是否源于 prompt 欠指定。

## Questions for Further Exploration

1. **Prompt repair 闭环**：定位歧义句后自动改写或生成澄清问句，能否 measurable 提升 pass@k / 任务成功率？
2. **更大模型与多架构**：歧义 probe 峰值层是否随深度缩放迁移？PRIG 是否需 per-model 重训 probe？
3. **与 sparse neuron / patching 对比**：Zhang et al. 2025b 的 ambiguity neurons 与 PRIG 归因图是否一致？能否融合为 hybrid 定位器？
4. **真实用户日志**：无单句 gold 时，如何用弱监督（用户追问、失败标签）训练 probe？
5. **多轮对话歧义**：chat template 与 history 中歧义来源如何分解——仅最后 user turn 还是跨 turn 累积？
6. **层选择自动化**：能否用 probe accuracy 曲线 + 固定规则（如 \(\ell=15, m\in[\ell-4,\ell-1]\)）去掉 per-domain training tuning？
7. **latency–quality 权衡**：50 步 IG × 残差维度求和的开销 vs 单次 Grad×Input 在 32k context 上的可行性？
