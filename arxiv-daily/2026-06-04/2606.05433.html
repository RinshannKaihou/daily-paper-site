Frontier dense pre-training can be zero-knowledge verified—not paradigm-bound

KEY FINDING: Hint-and-verify zkVM + 8 native IEEE-754 precompiles prove actual GPU BF16/FP32 execution (not finite-field proxies). Three anchors—pre-committed arch_spec, inter-node network witnesses, on-the-fly Merkle chains—bind genesis, step, and ex-ante attestations at ~2–10% trainer overhead vs prior ~5×10⁵× estimates.

Anchor metric
2–10% FSDP overhead · Llama 3.1 405B · ~$100M run
Why this matters

EU Art. 51 (~10²⁵ FLOPs) and former US EO (~10²⁶) rely on self-reported compute—P.3 ex-ante turns caps into proof invariants enforceable without custom silicon in ~36 months.

Critical edge

Blueprint only—no 100B+ fielded proof. OP-3 (≥10⁶-param full backward zk) is highest-risk; OP-7 wire↔tensor mapping and TP deterministic tax may exceed Table 2.

Headline claim

Impracticality is a paradigm property of monolithic finite-field ZK-ML—not of ZK itself; ~10⁴–10⁶× cheaper than re-executing training.

Governance proof taxonomy

P.1 — Genesis proof (pre-training)

After h_commit, auditor specifies random batch index → trainer runs one step → proves data membership, architecture fidelity, and W₁ chain head.

  • Binds commitments to real GPU pipeline before million-step run
  • Establishes trust root for subsequent step proofs

P.2 — In-training step proofs

PCP-inspired sampling: pick step t, tensor, index k → Merkle openings + operation precompiles + anchor byte match at R(msg*) offsets.

  • Layer continuity: R(out_ℓ) = R(in_{ℓ+1})
  • k ≈ 4,605 → soundness 1 − 10⁻²⁰ on ≥1% entry deviations
  • Cross-step optimizer chain consistency

P.3 — Ex-ante attestations

Policy invariants (e.g. max_total_flops, regime, data filters) enforced as running constraints—violation breaks proof soundness, not post-hoc logs.

  • F_cumu(t) ≤ max_total_flops monotone in-circuit
  • Bridges EU ~10²⁵ / US EO ~10²⁶ self-report gap
  • One add/compare per step for FLOP accumulator

8 precompiles (Table 3)

  • MAC chain (~90 constraints, >99% budget)
  • BF16 lookup · Poseidon path · exp/sqrt/div/log
  • SHA-256 for anchor reconciliation

Sampling caveat

Soundness assumes ≥1% deviation detectability; not perfect step replay—regulators must accept PCP-style assurance.

Training modifications (MOD.1–5)

MOD.1 Determinism

Pinned cuBLAS/NCCL, FlashAttention deterministic modes. Llama 7B FSDP@8×H100: 1.6–8.2%. TP NVSwitch allreduce is hard case (64–89% BW loss).

MOD.3 Merkle online

6 roots/layer/step on concurrent streams. HBM read ~0.2%; optimized 0.5–1.5%.

MOD.4 Network anchor

Tier 1 physical TAP vs Tier 2 attested SmartNIC. SHA-256 Merkle on collective payloads. Does not cover intra-node NVLink (OP-8).

MOD.5 Weight window

Challenge step N: load W_N from NVMe, one fwd+bwd for hints; or deterministic checkpoint replay. 100 steps × 810 GB ≈ 81 TB — storage negligible.

arch_spec schema (Appendix F)
Rust-style: LayerSpec (GEMM, FusedAttention, Norm, MoE placeholder), PrecisionSpec (BF16/FP32, RNE, accum_order), CommSpec (NCCL algorithm, reduction order), ParallelismSpec (TP/PP/DP/FSDP). OP-13: PyTorch/JAX → arch_spec compiler.

Critical assessment

Strengths

  • Confronts self-reported FLOP governance gap directly
  • Proves real GPU BF16/FP32 — not fixed-point proxy
  • Private arch_spec hash preserves commercial IP
  • Modular additive path for MoE / RL / multi-DC
  • Appendix E/F/G dual entry for builders & theorists

Limitations

  • Dense pre-training only v1 — MoE, RLHF, multi-DC, NVLink out of scope
  • 405B costs are estimates; no end-to-end 100B+ proof latency $
  • Attention backward deterministic tax 23–82% (§C) may blow Table 2
  • Tier 2 SmartNIC weaker; OP-7 unresolved limits anchor value
  • Honest trainer arch_spec bugs fail verification ≠ soundness break

Novelty · rigor split

Novel: reframes “ZK can’t verify frontier training” as wrong-paradigm + right three-anchor design; ex-ante attestation as governance primitive. Rigor gap: OP-3 unverified; sampling assumes 1% deviation; no open reference implementation—treat as programmatic blueprint for negotiators and zkVM precompile designers.
