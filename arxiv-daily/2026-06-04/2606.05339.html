Actionable engineering guidance

Five practices for MCP server authors (§6):

  1. Validate tool inputs against declared schemas before execution.
  2. Structured JSON-RPC errors—never success payloads hiding tool failure.
  3. Gate request handling until initialization completes.
  4. Preserve request–response IDs through tool invocation and result propagation.
  5. Keep diagnostics off the JSON-RPC stream (especially stdio transport).

SDK / protocol maintainers

  • Enforce ID propagation in SDK defaults
  • Default input validation hooks
  • Initialization gating helpers
  • Error-object builders—not per-repo patches
