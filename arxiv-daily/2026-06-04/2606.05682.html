Why This Matters

输出分布对齐 ≠ 内部几何保真。PayPal 在 NVFP4 生产管线上证明：RL 后训练模型的 KL-only QAD 会加剧表征坍塌；轻量 CKA 正则（+0.5% 步时）同时恢复 AIME / GPQA / LiveCodeBench。

Critical Edge

仅 2 个模型、~2.5B token 蒸馏数据；CKA→精度因果链未消融；依赖 ModelOpt/NVFP4 生态，个人复现门槛高；Nemotron 部分层仍 BF16。

Methodology — CKA-QAD

冻结 BF16 教师 + NVFP4 学生：Top-k KL（k=8192）+ 层间线性 CKA 对齐 decoder block 输出；动态 stop-gradient 平衡，无需手工 λ。

BF16 Teacher → Top-k KL + Layer CKA → NVFP4 Student
CKA(H̄t, H̄s) = ‖H̄s⊤H̄t‖F2 / (‖H̄t⊤H̄t‖F · ‖H̄s⊤H̄s‖F)
Ltotal = LTopK-KL + sg(LTopK-KL / (LCKA+ε)) · LCKA 对齐 decoder block 输出（非 pre-norm、非 attention 权重、非 KV cache）；特征空间 CKA 避免显式 N×N Gram；对 E4M3 块缩放不变。

Nemotron 3 Nano52 block 输出；Mamba–Transformer MoE；~2.5B token

核心贡献

反直觉诊断： KL-only QAD 内部 CKA 可低于 PTQ，挑战「KL 恢复 ≈ 表征恢复」。
CKA-QAD： Top-k KL + 层间线性 CKA + 动态 stop-gradient 平衡。
双架构实证： Nemotron MoE hybrid + Qwen dense；RL 后训练几何最脆弱。
机制定位： logit shortcut learning + 层 12–24 / Mamba state drift 可视化。

Relevance

NVFP4 是 NVIDIA 主推 4-bit 推理格式；本文直接服务 latency/成本敏感的生产部署与「先 RL 再量化」流水线警示。

Strengths

  • 揭示 KL-only QAD 可比 PTQ 更差 的内部 CKA
  • 轻量（步时 +0.5%）且无需重跑 RL / 额外标注
  • 层间可视化定位 drift 区域，指导自适应层选择
  • Qwen medium/hard 代码题增益，非 easy 集刷分
  • 动态损失平衡 + 特征空间 CKA 工程务实

Limitations

  • 层间 CKA 仍有额外 FLOPs/显存（虽 <15%）
  • 仅二阶 Gram 结构；未覆盖更高阶流形性质
  • 未做 drift-prone 层因果消融；全词表 KL 未对照
  • CKA 未嵌入 RL 训练本身；INT4/MXFP4 泛化待验证
  • PayPal 内部训练细节与 checkpoint 透明度有限

Questions for Further Exploration

  1. 仅对层 12–24、35–42 加 CKA，能否用更少 overhead 达到相近精度？
  2. 固定 λ / per-layer λ vs 动态 stop-gradient 在更多模型上是否稳定？
  3. top-k=8192 是否掩盖尾部 token 几何；全词表 KL + CKA 的 Pareto 前沿？
  4. RL rollout 阶段即约束 CKA，能否减轻后续 NVFP4 几何放大？
  5. INT4 / MXFP4 上 KL-only 是否同样出现「CKA 低于 PTQ」？
  6. 定向破坏高 CKA 层 vs 低 CKA 层的干预实验，验证 multi-step reasoning 必要性。
  7. 30B/70B 全 NVFP4 + MoE/Mamba 路由下的对齐边界与开销缩放？
