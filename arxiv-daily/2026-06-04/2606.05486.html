Localize latent prompt ambiguity in residual streams—not output logits

KEY FINDING: PRIG (Probe-Targeted Residual Integrated Gradients) trains linear ambiguity probes on mean-pooled residuals, then attributes probe logits via truncated 50-step IG on layers m→ℓ. Token AUROC 0.840 combined, 0.891 human gold—vs embedding IG 0.451; sentence F1 0.734 beats GPT-5.4 0.587 with fewer false alarms on clear prompts.

Anchor metric
0.840 combined AUROC · gold 0.891 · AUPRG 0.774
Why this matters

Ambiguity is input-time and latent—agents can scan which sentence is underspecified before tool calls or code gen, without asking a frontier model to hallucinate ambiguity on clear tasks.

Critical edge

Single 8B model; synthetic GPT-5.4 rewrites; linear probe assumption; gold n=12; per-domain layer tuning; no prompt-repair downstream eval.

Paradigm shift

Feature-level attribution for probe-decoded properties—residual position ≈ token position, not embedding-space diffusion.

Ambiguity is not an output logit

0.451 Embedding IG (combined)
0.629 Grad×Input (combined)
0.840 PRIG residual (combined)
Standard attribution explains a generation logit from embeddings. Prompt ambiguity exists before any continuation—and may never surface in the sampled output. — Problem class: feature-level token attribution for latent prompt properties

What fails

  • Perturbation / attention on observable behaviors
  • Embedding-space IG diffuses across long contexts
  • Activation patching needs clean feature on/off pairs
  • Output logit targets miss input-time underspecification

PRIG target

  • Linear probe decodes ambiguity at layer ℓ
  • Tokenwise Fℓ(x) = (1/n) Σ ⟨wℓ, hiℓ⟩
  • IG on residual subgraph layers m→ℓ only
  • Zero residual baseline · Gaussian σ=3 smooth

PRIG in three coupled steps

Mean-pool probe → Tokenwise logit Fℓ → 50-step residual IG → Σ dims · blur σ=3 → Token scores ai
Step 01

Probe-targeted logit

Train logistic probes on mean-pooled residuals per layer; accuracy rises ~layer 5–6, peaks ~layer 15, then softens.

Step 02

Residual IG (m→ℓ)

Gℓ,m(R) = Fℓ(fℓ:m(R)); path Rαm = αRm(x); 50-point Riemann; local layers preserve token alignment.

Step 03

Gaussian smooth

ãi = Σj Kσ(i−j)aj, σ=3—reduces subword fragmentation and gradient noise.

Fℓ(x) = (1/n) Σi=1n ⟨wℓ, hiℓ(x)⟩  ·  Ai,jℓ,m = hi,jm ∫01 ∂Gℓ,m/∂hi,jm dα  ·  aiℓ,m = Σj Ai,jℓ,m

Layer pair (m, ℓ) selected per domain on training set (Appendix D)—not a single global config for all tasks.

Agent / reliability use

  • Pre-flight scan before tool calls or codegen
  • Trigger clarification (CLAM, ClarifyGPT lineage)
  • Lightweight diagnostic—not full LLM reasoning

Not shown yet

  • Prompt repair improves pass@k
  • 70B+ or multimodal transfer
  • vs activation patching / sparse neurons

Critical assessment

Strengths

  • Clear problem class: latent prompt feature attribution
  • Consistent gains across 3 domains + gold
  • Reproducible pipeline + public code
  • AUPRG suited to one-sentence positive class
  • Better precision / abstain than GPT-5.4 judge

Novelty

  • Probe-targeted residual IG named PRIG
  • Contrasts output-logit explanation paradigm
  • Intermediate representations localize underspecification

Critical edge — read before deploying

  • Linear probe may miss distributed / nonlinear ambiguity
  • Only Llama-3.1-8B-Instruct validated
  • Training ambiguity = GPT-5.4 rewrite style; gold = 12 prompts
  • Ground truth = rewritten sentence; real ambiguity may span turns
  • Per-domain (m, ℓ) tuning on training set
  • 50-step IG cost on long contexts not quantified in paper
