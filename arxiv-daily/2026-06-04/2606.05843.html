Why This Matters

长视频/长文档 MLLM 部署：用可命名的跨模态检索电路做头级混合稀疏，而非盲目剪枝全层注意力。

Critical Edge

RAM 依赖 GT 空间/时间标注（bbox、tube、证据区）；静态 top-k 路由；主要验证于 late-fusion Transformer MLLM。

Methodology · RAM

扩展 QRhead 到多模态：在指令 query 上对 ground-truth 视觉区域 \(\Omega(V^*)\) 聚合注意力质量，排名 (layer, head)。

多模态序列 → text→vision 子矩阵 → RAM 标量 → CoRe / Vision 分工
MRAM(h)(q → V*) = Ex∈q[ Σy∈Ω(V*) Ax→y(h) ] 高分头 = Context-aware Retrieval (CoRe)；对比基线 Vision Heads = query→全部视觉 token 宏观聚合。
CoRe Heads 稀疏、局部位点；定向抽取关键视觉特征（~5% 顶级头因果必需）。
Vision Heads 中层至后期连续覆盖；全局视觉语义编码与传播（辅助扩散头）。

核心贡献

RAM + CoRe 定义： 在 annotated Ω(V*) 上可操作地排名跨模态检索头，区分 sink 与定向检索。
功能稀疏性面板： 四模态任务 × 三 MLLM 族 × 多尺度，揭示 CoRe 提取器 vs Vision 扩散头分工。
因果 + 集中度： Top CoRe 不可或缺；关键视觉语义集中于精英头子集，50–100 头即饱和覆盖。
跨任务锚定： ~30 稳定头、Spearman 0.79–0.92，支撑共享检索电路假说。
CoRe 混合注意力： 头级确定性稀疏，2× 级 prefill 加速，长上下文部署直接可用。
架构启示： 对齐策略与 encoder 设计塑造头拓扑，而非单纯模型规模。
通讯：Juntao Li · Soochow U · GitHub CoRe_Head · 继承 QRhead · 对话 Head Pursuit / visual sink 文献

Strengths

  • 首次统一 RAM 刻画 多模态跨任务功能稀疏性，连接解释与加速
  • CoRe vs Vision 二分有分布、因果、token 集中度三重证据
  • 跨任务高 Spearman + ~30 稳定头，假说可检验
  • 2× 级 prefill 加速对长视频/长文档部署有工程价值
  • 诚实讨论 GT 依赖、架构泛化、静态路由局限

Limitations

  • RAM 需 GT 标注，难推广无标注/纯抽象推理
  • 以 late-fusion ViT+LLM 为主；early-fusion 未验证
  • 静态 top-k 路由，OOD 查询可能次优
  • Decode 阶段稀疏性未系统分析
  • InternVL 高密度 regime 需更细致对照与加速策略

Questions for Further Exploration

  1. 无标注 RAM：用模型 grounding、Grad-CAM 或 SAE 近似 Ω(V*) 是否仍稳定排名？
  2. 动态 CoRe 路由：按 query 在线选头能否改善 OOD 与多轮对话？
  3. 消融 CoRe 是否系统性增加 object hallucination？
  4. 训练干预：SFT/RLHF 强化 CoRe、抑制弥散头能否提升 grounding？
  5. Decode 阶段 CoRe 头是否同样可剪？跨模型 activation patching 能否映射 ~30 anchor heads？
