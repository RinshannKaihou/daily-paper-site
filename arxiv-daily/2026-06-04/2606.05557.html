Results & Highlights

  • 场景 A/B/C 分别 **+0.091 / +0.091 / +0.080**（*p* 显著）；场景 D（事后夜间）**+0.024, *p*=0.349**（公开状态已暴露可用性）。

Strengths

  • 精准刻画「字面答对、隐含答漏」这一 situated agent 常见失败模式，示例（Lin Wei 在哪）具传播力。
  • IntentFrame + *B(g)* 将探测成本与隐含需求不确定性显式挂钩，实践中作**上限**可节省大量低 gap 查询的 API 调用。
  • 100 条四场景基准 + 五子类标签支持细粒度错误分析（availability 收益最大符合设计预期）。
  • 隐私切片 0% 违规展示 gap 路由的**结构性**安全收益（低 gap 跳过注册表即无法调用禁用工具）。

Limitations

  • **适用范围窄**：仅 tool-mediated hidden state 的结构化多智能体场景；不涵盖具身、开放世界、多用户并发规划。
  • **事实查询**：FA 低于 Fixed-Probe/Plan-and-Solve；矛盾声明率仍 ~75%；中位延迟更高。
  • **基准与标注**：100 条作者编写；second_order 与 FANToM 等外部信念基准**不自动可迁移**。
  • **latent_goal 子类**：Intent 仍低于 NoIntent 或存在残余缺口（Δ=−0.09），需更强目标推理或专用工具。

TL;DR

Situated queries (e.g., “where is Lin Wei?”) often encode implicit needs (availability, mood, interruptibility). Standard tool agents answer literally and stop. **AURA** adds an **IntentInferrer** before bounded **Explore** probing that emits an **IntentFrame** with gap score *g* ∈ [0,1] routing probe budget *B(g)* and tool shortlist. On 100-query implicit-intent benchmark: **0.804 vs 0.733** implicit coverage vs ReAct-style NoIntent (Δ=+0.071, *p*<10⁻⁶; 3/4 scenes significant). On 50 factual queries: GapRouted is **not** raw-FA winner (0.696 vs Fixed-Probe 0.766) but Pareto on probes (1.40 vs 8.00) and privacy violations (0% vs 78.9–100%). Lift localizes to LLM gap calibration (heuristic backend: 0.803→0.368). Code/simulator/benchmark released.
