> **一句话总结：** ReasoningFlow 将 LRM 推理轨迹标注为子句级 DAG（8 类节点、14 类边），经人工 IAA 验证后扩展到 1,260 条轨迹（247.7k 步）；分析表明不同 LRM 结构高度相似、局部验证比全局更有效、约 85% 错误步骤不因果影响最终答案，且 Thought Anchors 的 mechanistic 依赖与语言级话语关系不对齐。

> **TL;DR:** ReasoningFlow maps LRM traces to sub-sentence DAGs (8 node + 14 edge types; Reason/Plan/Reflect/Validate). Manual: 31 traces, Krippendorff α>0.88. Auto: 1,260 traces (AIME/GPQA/ArgKP; Qwen2.5, QwQ, DS-V3/R1, GPT-oss-120B) via Gemini-3.1-Flash/3-Pro (NC F1 0.865, EDC F1 0.646). Findings: (1) LRM structural similarity across families (JS-Div 0.010 QwQ↔DS-R1 vs 0.083 base models). (2) Fine-grained behaviors—local verification, self-reflection sentiment↔quality, proof-by-contradiction/DFS. (3) 79.6% errors unused, 6.0% neglected, 14.4% faithful—weak error→answer link. (4) Thought Anchors AUC 0.549 ≈ recent-K baseline. Code: https://github.com/jinulee-v/reasoningflow

## 核心贡献

1. **面向 LRM 的子句级话语标注框架**：将自回归推理轨迹转为**有向无环图（DAG）**，边仅左→右流动；定义 **8 类节点**（Reasoning / Planning / Reflection + Fact / Restatement / Assumption / Example / Conclusion）与 **14 类边**（Reason: infer/execute/restate；Plan: proceed/verify；Reflect: positive/uncertain/negative；Validate: support/attack），粒度优于段落级 ThoughtAnchors、R1-Thoughtology、LCoT2Tree 等（Table 1）。

2. **经 IAA 验证的人工标注 + 大规模自动标注**：4 名标注者对 31 条轨迹（2.1k 步，QwQ-32B-Preview，NuminaMath + STILL-2）达成 **NC α=0.885、EDC α=0.919**；LLM 管线（Gemini-3.1-Flash 节点 / Gemini-3-Pro 边）扩至 **1,260 轨迹、247.7k 步**，覆盖 AIME 2024、GPQA-Diamond、ArgKP 与五模型。

3. **结构相似性与领域差异的实证**：(node, edge, node) 三元组分布 PCA 显示**任务域聚类强于模型**；但 QwQ↔DeepSeek-R1 的 Jensen–Shannon 散度仅 **0.010**（基座 Qwen2.5↔DS-V3 为 0.083），表明不同后训练 LRM 轨迹结构趋同。

4. **细粒度推理行为刻画**：局部验证（local verification）普遍且多于全局验证；修正节点更常作为最终 Conclusion 的前提；Reflection 情感与步骤正确性/论证强度显著相关；Assumption 节点中 **41.9%** 对应反证法或 DFS 式假设探索。

5. **对逐步评测与 mechanistic 解释的启示**：**79.6%** 错误节点未连到最终答案、**6.0%** 被忽视仍得正确答案，仅 **14.4%** 忠实传播错误——说明 PRM/逐步 judge 需结合话语结构；Thought Anchors 注意力掩码因果分数预测 ReasoningFlow 边的 AUC **0.549**，与「最近 K 个节点」启发式（0.531）相当。

## Key Contributions

1. **Sub-sentence discourse DAG schema for LRM traces** with validated IAA—only work in Table 1 with both fine-grained nodes/edges and inter-annotator agreement analysis.

2. **ReasoningFlow dataset**: 31 manual + 1,260 auto-annotated traces across math/science/argumentation and five open models; node-quality labels (logical validity / AQR argument scores).

3. **Cross-model structural similarity** despite different base models and post-training data.

4. **Behavioral analyses**: local vs global verification, self-reflection as quality signal, assumption-driven hypothetical reasoning, error propagation taxonomy (unused/neglected/faithful).

5. **Discourse–mechanism gap**: Thought Anchors causal dependencies poorly align with language-level discourse edges.

## Methodology

### 问题背景

大型推理模型（LRM）产生含回溯、自校正、验证的非线性 CoT，使**逐步正确性评测**与**忠实性监控**困难：单步错误未必导致错误答案。既有工作要么缺少丰富边类型（PARC）、要么仅段落级（ThoughtAnchors、R1-Thoughtology、LCoT2Tree、ReJump），且**普遍未做 IAA 验证**。

### ReasoningFlow 图模式（§3）

- **结构**：DAG，边从序列中较早节点指向较晚节点，兼容交叉依赖与一步多后继（相对 RST 树与论证树更灵活）。
- **节点**： contiguous 非重叠片段；默认一句一节点，功能角色不同的从句可拆分（如「Therefore x=17, but I should double-check」拆为推理 + 计划验证）。
- **边**：单源、左→右；完整定义见论文 Table 3–4。

### 数据集构建（§4）

**人工（§4.1）**  
31 题（数学/物理/化学，NuminaMath + STILL-2）；QwQ-32B-Preview，T=0；两名标注者/题，共享预分割；NC（8 类）与 EDC（连接 + 15 类含 no-edge）。

**自动（§4.2）**  
- **轨迹**：AIME 2024（30）、GPQA-Diamond（198）、ArgKP（24 辩题）× 五模型，greedy；共 1,260 traces。  
- **管线三阶段**：(1) 节点分割；(2) 节点分类；(3) 边检测/分类——对每节点仅预测入边以控制 O(N²) 输出。  
- **标注器**：Gemini-3.1-Flash（节点）、Gemini-3-Pro（边）；作者在 30 例上人工复核（Appendix C.4）。  
- **节点质量（§4.3）**：Reasoning 节点用 Gemini-3-Flash 判逻辑有效性（仅连前驱上下文）；ArgKP 用 AQR 数据集匹配论证质量分。

### 分析指标

- **三元组分布** \((n_1, e, n_2)\)：刻画验证、反证等行为；跨模型 JS 散度、PCA（Ding & He, 2004）。  
- **平均度**：入边数/节点，约 1.3–2.0。  
- **错误传播**：沿 DAG 追踪错误 Reasoning 节点到 Conclusion 的因果路径，分 unused / neglected / faithful。  
- **Mechanistic 对齐**：Thought Anchors 对 step \(s_i\) 掩码后测 \(s_j\) token 分布 KL 散度，PR 曲线预测 ReasoningFlow 边。

## Results & Highlights

### 图规模与结构统计（§5）

| 观察 | 数值/趋势 |
|------|-----------|
| 图规模 | QwQ 在 AIME 均 ~455.6 节点；Qwen2.5 在 ArgKP 仅 ~29.2 |
| 平均度 | 各 (model, dataset) 稳定在 **1.3–2.0**（边数 ≈ 1.3–2× 节点数） |
| 领域特征 | ArgKP：Planning–proceed 链；Math：Reasoning–infer；Science：Conclusion–support 验证 |

### Finding 1：LRM 结构相似（§5.2）

- Qwen2.5-32B vs DeepSeek-V3：**JS-Div 0.083**；QwQ vs DeepSeek-R1：**0.010**。  
- PCA：三元组分布按 **dataset** 聚类，非按 model。  
- 推理模型彼此更像，而非与其非推理基座。

### Finding 2：细粒度行为（§6）

**2.1 局部验证**  
- 局部验证频率 **高于** 全局验证（对整段首个答案的验证）。  
- 显式修正（attack 边）后：**35–54%** 仅修正版进入 Conclusion，**17–20%** 仍用原版（QwQ/DS-R1/gpt-oss，Figure 5b）。

**2.2 自反思**  
- positive Reflection：AIME/GPQA 上 **78.1%** 步骤正确；uncertain **66.2%**；negative **45.6%**。  
- ArgKP 上论证质量趋势一致——反思措辞非 filler，可作监控信号。

**2.3 假设推理**  
- 识别 **161** 反证法 + **2,514** DFS 相关模式，占 Assumption 节点 **41.9%**。

### Finding 3：错误与最终答案弱因果（§7）

| 错误类型 | 占比 | 含义 |
|----------|------|------|
| Unused | **79.6%** | 未连到 Conclusion（如回溯分支） |
| Neglected | **6.0%** | 经错误前提仍得正确答案 |
| Faithful | **14.4%** | 错误因果传播至错误答案 |

→ 逐步评测/PRM 需结合**下游话语依赖**，不能假设「有错步即错答」。

### Finding 4：Mechanistic ≠ Discourse（§8）

- Thought Anchors 预测 ReasoningFlow 边：**AUC 0.549**，Best F1 **0.545**。  
- 「最近 K 个节点」基线：**AUC 0.531**——距离是主要相关因素。  
- 内部注意力因果与表面话语结构存在**系统性错位**。

- **UIUC** 团队（Lee, Agarwal, Parulekar, Madala, Hakkani-Tür, Hockenmaier）；Lee & Hockenmaier 在逐步推理评测综述（EMNLP 2025 Findings）、SymBa 等方向有积累。

## Critical Assessment

**新颖性**：首个同时提供**子句级节点+边**、**IAA 验证**、**千级轨迹规模**的 LRM 话语结构框架；将 RST/论证结构适配到左→右自回归推理，并系统对比 mechanistic 与 discourse 层。

**严谨性**：
- **优点**：人工+自动双轨；跨三任务五模型；错误传播分类有图论依据；与 ThoughtAnchors 直接对照；反思情感与质量定量相关。
- **弱点**：自动标注依赖 frontier API，EDC F1（0.646）低于 NC；全量数据无完整人工审计；闭源 LRM 未测；节点/边错误会级联影响下游统计。

**工程意义**：为 **PRM 训练**、**推理轨迹监控**、**faithfulness 评测** 提供可解释中间表示；局部验证与错误 unused 比例对「删错步是否影响答案」类产品决策有直接指导。

## Strengths

- Table 1 清晰定位：唯一 sub-sentence + 14 edges + IAA 的工作。
- 发现 3 挑战了逐步评测与 Best-of-N 滤错的主流假设，有政策含义。
- 局部验证 > 全局验证的量化证据，补充 Gandhi et al.、Marjanovic et al. 的验证叙事。
- 开源数据集与代码，利于训练专用 discourse 标注器降低成本。
- 连接计算语言学（RST、论证）、形式逻辑（entailment graphs）与 LRM 实证。

## Limitations

- **自动标注质量**：EDC F1 0.646，边结构错误可能扭曲三元组统计与错误传播分析。
- **模型覆盖**：无闭源/蒸馏 LRM；§5 结构相似性外推需谨慎。
- **成本**：Gemini API 管线难大规模复现；未发布小型专用标注模型权重。
- **因果解读**：DAG 边表话语/语义依赖，非严格逻辑蕴涵；与 Thought Anchors 的「因果」定义不同，读者易混淆。
- **任务范围**：仅 MC 式科学/数学/辩论，未覆盖代码推理、多模态 CoT。
- **评测应用**：未展示用 ReasoningFlow 直接改进 PRM 或下游准确率的端到端实验。

## 相关性 (Relevance)

- **manifest relevance_score: 4**——「用话语结构理解 LLM 推理轨迹，超越表面输出」，与 daily_paper 关注 **LRM 可解释性、评测、监控** 高度相关。
- **与 2606.05384（LLM Judge 鲁棒性）互补**：本文刻画**推理过程内部结构**；Judge 论文刻画**评测偏好对外部对话的脆弱性**。
- **与 2606.05429（SAGE-PTQ）等领域不同**：本文属 **NLP/推理分析**，非量化部署；但对 **CoT 监控、安全审计** 栈有基础设施价值。
- **PRM / Process supervision**：错误 unused 79.6% 提示训练 PRM 时应标注**话语可达性**而非逐步真值 alone。
- **Mechanistic interpretability**：为「对齐内部计算与表面推理语义」提供 benchmark 边集。

## Questions for Further Exploration

1. **专用小模型标注器**：用 1,260 条自动标注能否训练开源边检测器，使 EDC F1 接近人工且成本降 100×？
2. **PRM 增益**：在 ReasoningFlow 上 mask unused-error 步骤的 PRM 是否比逐步二元标签更准？
3. **闭源 LRM**：o1/Gemini 的三元组分布是否仍与 QwQ/DS-R1 保持 JS-Div <0.02？
4. **训练动力学**：Ettinger et al. / Wang et al. 所述 RL 推理层次是否在 ReasoningFlow 三元组上随 checkpoint 可追踪？
5. **预算推理**：BudgetThinker 等控制 token 下，Planning–verify 三元组频率如何变化？
6. **代码/工具 CoT**：8+14 模式是否需扩展（如 tool-call、code-exec 节点）？
7. **因果边学习**：能否微调 LRM 使 Thought Anchors 分数与 ReasoningFlow 边对齐（faithful reasoning 目标）？
