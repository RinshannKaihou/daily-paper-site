> **一句话总结：** 本文提出LCF（Layerwise Convergence Fingerprinting），一种无需训练、无需参考模型、仅用200条干净样本校准的运行时LLM异常行为检测方法，通过对Transformer各层隐藏状态差值计算对角马氏距离并用Ledoit-Wolf聚合为单一分数，在后门、越狱和提示注入三类威胁上实现了统一的运行时检测（后门ASR降至1%以下，越狱检测率92-100%，提示注入100%），且推理开销低于0.1%。

> **TL;DR:** LCF is a tuning-free, single-model runtime monitor that detects three distinct LLM threat families (training-time backdoors, jailbreaks, prompt injections) with one unified statistical score. It computes per-layer diagonal Mahalanobis distance on inter-layer hidden-state differences, aggregates via Ledoit-Wolf shrinkage, and thresholds via leave-one-out calibration on 200 clean examples. Across four architectures (Llama-3-8B, Qwen2.5-7B/14B, Gemma-2-9B), it reduces mean backdoor ASR below 1.3%, detects 92-100% of DAN jailbreaks, and flags 100% of text-payload prompt injections, at 12-16% FPR and <0.1% inference overhead. No reference model, trigger knowledge, or retraining is needed.

## 核心贡献

- 提出了LCF方法，将Transformer层间隐藏状态差值（"表示速度"）作为模型运行时健康信号，通过统计异常检测在prefill阶段（首个token生成前）即完成判定，推理开销低于0.1%。
- 实现了跨三类威胁（训练时后门、越狱提示、提示注入）的统一检测，无需针对特定威胁调参，单一校准阈值覆盖所有威胁类型。
- 发现了"三层带深度分层"现象：越狱异常集中在浅层（L0-L5），提示注入异常集中在中层（L9-L25），后门异常集中在深层（具体位置因架构而异），解释了全层聚合为何有效。
- 在56个后门攻击组合、3种越狱技术和BIPIA提示注入上进行了全面评估，LCF在Qwen2.5-7B和Gemma-2上将后门ASR降至0.9%和0.7%，优于CleanGen且无需参考模型。
- 对自适应攻击进行了鲁棒性评估，表明全层聚合比单层监控显著更难被规避（单层在lambda=2时被完全突破，而LCF仍保持有效）。

## Key Contributions

- Introduces LCF, a tuning-free runtime monitor that treats inter-layer hidden-state differences ("representation velocity") as a health signal, making detection decisions at the prefill step before any token is emitted, with <0.1% inference overhead.
- Achieves unified detection across three threat families (training-time backdoors, jailbreaks, prompt injections) using a single calibrated threshold without threat-specific tuning.
- Documents a three-band depth stratification: jailbreak anomalies concentrate at early layers (L0-L5), prompt injection at mid layers (L9-L25), and backdoors at architecture-specific late layers, explaining why all-layer aggregation is necessary.
- Comprehensive evaluation on 56 backdoor combinations, 3 jailbreak techniques, and BIPIA prompt injection across four architectures; reduces mean backdoor ASR below 1% on Qwen2.5-7B and Gemma-2 without a reference model.
- Adaptive attack evaluation demonstrates that all-layer aggregation is fundamentally harder to evade than single-layer monitoring (single-layer fully bypassed at lambda=2, LCF remains robust).

## Methodology

LCF operates in three stages:

1. **Per-layer anomaly scoring.** At the prefill step (step 0), LCF extracts hidden states from every transformer layer at the last token position. It computes the inter-layer difference (representation velocity) delta_l = h_{l+1} - h_l for each layer l, then scores each delta using a diagonal Mahalanobis distance: s_l = ||(delta_l - mu_l) / sigma_l||_2, where mu_l and sigma_l are per-dimension mean and standard deviation estimated from 200 clean calibration examples. This per-dimension normalization upweights low-variance directions where misbehavior-induced perturbations concentrate.

2. **Calibration and aggregation.** Per-layer Mahalanobis scores are z-scored across layers to place all layers on a common scale. A Ledoit-Wolf regularized covariance matrix is then estimated on the L-dimensional z-score vectors (L = number of layers, 28-48 depending on architecture). The final anomaly score D is the Mahalanobis distance using the Ledoit-Wolf precision matrix, capturing inter-layer correlations. The detection threshold is set via leave-one-out (LOO) calibration at the 90th percentile of LOO scores, targeting 10% FPR.

3. **Runtime detection.** During inference, LCF inspects only the prefill step. If D > threshold, it returns a safe abstention message; otherwise the model's output passes through unmodified. The added computation beyond the standard forward pass is negligible: L element-wise z-scores, L L2 norms, and one L-dimensional matrix-vector product.

## Results & Highlights

- **Backdoor defense (56 combinations):** Mean ASR reduced to 0.9% (Qwen2.5-7B), 0.7% (Gemma-2), 4.2% non-corrupted (Llama-3), and 1.3% non-corrupted (Qwen2.5-14B). LCF outperforms CleanGen on Qwen and Gemma without requiring a reference model or doubling GPU memory.
- **Jailbreak detection:** DAN-style jailbreaks detected at 92-100% across all four architectures (AUC 0.93-1.00). GCG detected at 65.5% (Llama-3) to 99% (Gemma-2, Qwen-14B). The unsupervised LCF approaches supervised LLMScan performance on larger models (DeltaAUC shrinks from +0.16 on Llama-3 to +0.03 on Qwen-14B) at 65-113x lower per-prompt cost.
- **Prompt injection (BIPIA):** 100% TPR on text-payload injections across all 8 (model, domain) cells with paired Cohen's d between +1.36 and +3.11. Code-payload detection scales with model size: Llama-3 43%, Qwen-14B 100%.
- **Overhead:** <0.1% inference overhead on Qwen2.5-7B and Llama-3-8B.
- **False positive rate:** Operational FPR of 12-16% for backdoors (targeting 10%), 13-22% for jailbreaks, and 13-19% for prompt injection -- consistently overshooting the 10% calibration target due to finite-sample variance.
- **Adaptive robustness:** Fully robust at lambda <= 1; partial evasion attack-dependent at lambda = 2 (BadNet 15.5%, Sleeper 2.0%, VPI 0.0%); attacker's benefit saturates at lambda = 5.

## Critical Assessment

**Novelty:** The idea of monitoring inter-layer hidden-state differences for runtime anomaly detection is well-motivated and cleanly executed. The three-band depth stratification finding (jailbreaks early, injections mid, backdoors late) is a genuinely novel empirical observation. However, the core statistical machinery (diagonal Mahalanobis + Ledoit-Wolf shrinkage + LOO calibration) is well-established; the novelty lies in the application and the insight about representation velocity rather than raw activations.

**Rigor:** The evaluation is extensive (56 backdoor combinations, multiple threat families, adaptive attacks, ablations). The paper is commendably transparent about failures: corrupted baselines are excluded with clear justification, elevated residual ASR on specific combinations is reported, and the displacement-vs-dispersion boundary on hallucination is honestly characterized. However, no error bars or confidence intervals are reported (justified by computational cost of training 42 separate backdoor models). The FPR consistently overshoots the 10% target (12-22%), which raises practical deployment concerns.

**Reproducibility:** Good but not complete. Full experimental details are provided (models, LoRA configs, calibration sizes, metrics, threshold settings). Code is promised but not yet released at submission time. All evaluation data is from public benchmarks.

**Significance:** If the results hold up under broader evaluation, LCF could be a practically deployable runtime safety layer. The unified cross-threat detection without threat-specific tuning is a meaningful advance over prior work. The main concern is whether the FPR (12-22%) is acceptable in production settings where every false positive blocks a legitimate user query.

**Red flags:**
- The Llama-3 results are noticeably weaker than other architectures (two refusal combinations at 12% and 30% residual ASR, plus one corrupted model excluded). The paper attributes this to dispersed signal across depth but does not fully resolve why.
- The paper excludes 2 of 56 combinations as "corrupted" (backdoor-like outputs on clean inputs). While the justification is reasonable, this exclusion pattern deserves scrutiny.
- The hallucination probe (TruthfulQA) shows LCF fundamentally cannot detect factuality issues, which the paper honestly acknowledges but which limits the generality of the "general-purpose runtime safety layer" framing.

## Strengths

- Unified detection across three distinct threat families with a single score and no threat-specific tuning.
- Extremely low inference overhead (<0.1%) since detection occurs at prefill before token generation.
- No reference model, no editable weights, no trigger knowledge required -- minimal deployment assumptions.
- Thorough adaptive attack evaluation demonstrating that all-layer aggregation is fundamentally harder to evade than single-layer alternatives.
- Transparent reporting of failure cases and honest characterization of method boundaries (displacement vs. dispersion).
- Clean experimental design with matched-pair evaluations, length-delta diagnostics, and supervised validation of signal geometry.
- Code promised with a public GitHub repository.

## Limitations

- FPR consistently overshoots the 10% calibration target (12-22% across settings), which may be unacceptable in high-traffic production deployments.
- Evaluated only on 7-14B parameter models; scaling to 32B/70B/70B+ remains untested.
- Requires white-box access to intermediate hidden states, excluding black-box API deployments.
- Calibration distribution must match the deployment domain (generic instruction-tuning calibration does not transfer to email/code-QA settings).
- Cannot detect hallucination or factuality issues (displacement-based score misses dispersion-only threats).
- Llama-3 shows elevated residual ASR on some combinations, suggesting architecture-dependent effectiveness.
- Only one benchmark per threat family; cross-family generalization is untested.
- No error bars or repeated runs reported due to computational cost.

## Relevance

本文提出的LCF方法直接解决了LLM部署安全中的核心问题：如何在运行时无需额外模型或触发器知识的情况下，统一检测后门攻击、越狱和提示注入等多种威胁。其基于隐藏状态层间差异的统计异常检测思路，以及发现的三层带深度分层现象，对LLM安全监控系统的设计和理解具有重要参考价值。

## Questions for Further Exploration

- How does LCF perform on larger models (32B, 70B, 70B+) and on closed-source models via intermediate access points?
- Can a dispersion-aware variant of the Mahalanobis score extend coverage to hallucination detection?
- What is the minimum calibration set size for acceptable performance, and how sensitive is LCF to distribution mismatch between calibration and deployment data?
- How does LCF handle multi-turn conversations where misbehavior may emerge gradually across turns rather than in a single prefill?
- Could architecturally adaptive aggregation (e.g., learned layer weights per model family) close the residual gap on Llama-3 refusal combinations?
- What is the interaction between LCF and other safety mechanisms (e.g., content filters, guardrails) in a production pipeline?
- How robust is the three-band depth stratification finding across more diverse model architectures (e.g., Mixture-of-Experts models)?
