01

Methodology

GILP is a hybrid world model: the API agent keeps doing the semantic reasoning, while a small trained network F_θ supplies cheap, auditable transition prediction. The backbone maps (state G_t, action a) → predicted validity, state-delta, reward, done-flag, risk and a Jaccard change-set, trained on oracle transitions with a multi-task loss. Crucially, the design grounds — not replaces the agent's reasoning, and the gate is API-agnostic. Each step runs four phases:

PHASE 01
Skeleton scoring
Backbone scores all candidate actions; top-k compressed into a skeleton block injected into the prompt before sampling.
k=4 · 96% value @ 5× compress
PHASE 02
LLM draft
One API call returns a selected action plus an imagined node-status delta in structured JSON.
1 call / step
PHASE 03
Consistency gate
Jaccard similarity between agent-imagined and backbone-predicted change-sets. Low band triggers a targeted re-prompt naming each disagreeing atom.
τ_high .70 / τ_low .30 · ~93% resolved
PHASE 04
Risk gate
If the backbone's predicted risk exceeds threshold, re-present only the low-risk action subset to the agent.
ρ_th = 0.65
Evaluation. Four graph-structured benchmarks (TaskGraph, ToolChain, ResourceAlloc, RepairFlow), 11 planners × 8 seeds. Agent/hybrid behaviour reproduced via a behavioural simulator calibrated to n=5 real GPT-4o-mini TaskGraph runs, corroborated by n=80 real-API episodes.

03

Assessment

Strengths

  • Clean conceptual separation of two world-model failure modes — and the insight that their errors are complementary.
  • Operational, horizon-resolved metrics (HSR / PD / EES) that make agent-based semantic error comparable across methods.
  • Simple, cheap, API-agnostic intervention — no code synthesis or symbolic rule extraction required.
  • Honest, conservative reporting: the simulator is explicitly framed as a lower bound, residuals disclosed.
  • Public release of all artifacts (code, prompts, simulator, benchmarks) plus a one-step contraction guarantee.

Limitations

  • Headline results rest on a calibrated simulator; live-API validation is small (n=80; n=12 for KG).
  • No direct API calls for Claude / Gemini / Llama in the multi-API study.
  • Scope limited to graph-structured planning; free-text, continuous, partially-observable state untested.
  • Backbone assumes access to oracle transitions; behaviour under noisy or unavailable oracles unstudied.
  • HSR / PD capture only node-status-delta hallucinations; entity-set & reward-attribution types excluded.
  • The parameterized model can be confidently wrong; mitigated by advisory framing, not adversarially tested.
04

Open Questions

  1. How does the gate behave on richer state — free-text observations, continuous values, partial observability — where clean change-set Jaccard is harder to define?
  2. Can the backbone be trained online from the agent's own verified executions, instead of requiring oracle transitions?
  3. Does the contraction guarantee's non-adversarial assumption hold against systematically biased backbones (distribution shift, adversarial state)?
  4. What is the live-API effect size on a larger sample? The authors note n ≥ 120 is needed to detect sub-10pp HSR at 80% power.
  5. How does the gate interact with stronger reasoning models (o1 / Claude-opus) whose baseline HSR is already low — is there a floor where the overhead no longer pays off?
  6. Could HSR / PD / EES be adopted as standardized benchmarks for evaluating other agent-hallucination-mitigation methods?
