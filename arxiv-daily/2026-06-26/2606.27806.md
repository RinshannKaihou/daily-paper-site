> **一句话总结：** 本文将LLM智能体的长期规划建模为两种世界模型的对比（基于API的推理模型 vs. 参数化转移预测模型），并提出GILP，用一个可审计的小型参数化骨干网络在每步纠正API智能体产生的"幻觉式状态增量"，在真实GPT-4o-mini调用中将幻觉状态率从0.176降至0.035。

> **TL;DR:** GILP pairs a cheap, measurable parameterized world-model backbone with an API-based reasoning agent, using a Jaccard consistency gate to correct hallucinated state deltas at each step, cutting the hallucinated-state rate by 80% and lifting long-horizon success.

## 核心贡献
- 将LLM智能体的长期规划形式化为两类世界模型的对比：基于API的推理型世界模型（语义灵活、但错误以"幻觉状态原子"形式出现并沿上下文传播）与参数化转移预测模型（错误可度量、但独立规划能力弱），指出二者失败方式互补。
- 针对基于API世界模型定义了"幻觉传播"的可操作度量：幻觉状态率（HSR）、传播深度（PD）、误差爆炸斜率（EES），以及长期"任意错误"代理量，把语义错误转化为可跨方法比较的指标。
- 提出GILP（Grounded Iterative Language Planning）：仅训练小型参数化骨干，保留API智能体作为推理引擎，通过骨架注入+一致性门控+风险门控四阶段流程，用可度量的转移信号去约束而非替代智能体的推理。
- 给出一步幻觉收缩保证（命题1）：在"非对抗性纠正"假设下，期望错误步数被以 $(1-\gamma)$ 因子收缩，允许参数化模型本身存在误差。
- 证明极弱骨干即可奏效：仅0.509独立成功率的MLP作为骨架供给者，就能把混合成功提升到0.768；并开源代码、模拟器与提示词套件。

## Key Contributions
- Frames long-horizon LLM planning as an explicit comparison between two world models — agent-based (flexible API reasoning; errors manifest as hallucinated state atoms that propagate through context) versus parameterized (measurable supervised transition error; weak standalone planner) — showing they fail in complementary ways.
- Defines operational hallucination-propagation metrics for agent-based world models: Hallucinated-State Rate (HSR), Propagation Depth (PD), Error-Explosion Slope (EES), plus long-horizon any-error proxies, turning semantic JSON errors into comparable quantities.
- Introduces GILP, which trains only a small parameterized backbone and uses a four-phase loop (skeleton scoring, LLM draft, Jaccard consistency gate, risk gate) to correct — not replace — the API agent's imagined state deltas.
- Provides a one-step hallucination-contraction guarantee (Proposition 1) under a non-adversarial-correction assumption, tolerating real parameterized-model error.
- Empirically shows weak backbones suffice (a 5k-param MLP with 0.509 standalone SR still raises hybrid SR to 0.768), and releases code, prompts, simulator, and benchmarks.

## Methodology
GILP is a hybrid world model that keeps the API agent for semantic reasoning and a small trained network for cheap, auditable transition prediction. The backbone $F_\theta$ maps $(G_t, a) \to (\hat{p}_{valid}, d\hat{\Delta}G, \hat{r}, \hat{p}_{done}, \hat{\rho}, \hat{U}, \hat{J}_K)$ and is trained on oracle transitions with a multi-task loss (state + reward + done + mask + risk). The per-step pipeline has four phases: (1) **Skeleton scoring** — the backbone scores all candidate actions and the results are compressed to a top-k skeleton block $B_t$ (k=4 retains 96% of value at 5× compression) injected into the prompt so the imagined state is grounded *before* sampling; (2) **LLM draft** — one API call returns a selected action plus an imagined node-status delta in structured JSON; (3) **Consistency gate** — the Jaccard similarity between the agent's imagined change-set and the backbone's predicted change-set is computed, with $\tau_{high}=0.70$ accept, mid-band accept-with-risk-penalty, and $\tau_{low}=0.30$ triggering a targeted corrective re-prompt that names each disagreeing atom (capped at one correction/step, resolving ~93% of triggers); (4) **Risk gate** — if backbone risk $\hat{\rho} > \rho_{th}=0.65$, a third call re-presents only the low-risk action subset. The mechanism is deliberately about *grounding* the agent's error rather than claiming the parameterized model is perfect, and the gate is API-agnostic. Evaluation uses four graph-structured benchmarks (TaskGraph, ToolChain, ResourceAlloc, RepairFlow) with 11 planners across 8 seeds; agent/hybrid behavior is reproduced via a behavioural simulator calibrated to n=5 real GPT-4o-mini TaskGraph runs, corroborated by n=80 real-API episodes.

## Results & Highlights
- **Main table (simulator, pooled):** GILP reaches SR 0.838 (vs 0.668 Agent-Replan, 0.625 best parametric), cuts HSR from 0.205→0.079, propagation depth 2.45→1.51, and invalid-action rate 0.169→0.065 — the best overall planner on every key column.
- **Long horizon (H>10):** the agent-based model collapses to SR 0.471 while GILP holds 0.758; the two world-model types separate most clearly as horizon grows.
- **Real GPT-4o-mini (n=80 episodes, H∈[3,8]):** pooled HSR falls 0.176→0.035 (80% relative reduction) with non-overlapping 95% CIs on all four benchmarks, at only ~480 extra tokens/task (~20% overhead).
- **Backbone strength:** even the weak MLP (0.843 transition acc, 0.509 standalone SR) lifts hybrid SR to 0.768; MPNN (0.992) reaches 0.771 with HSR cut by 69% — the useful signal is validity/delta estimation, not planning competence.
- **Cost-quality:** GILP uses ~1.30 LLM calls/step (1 draft + ~0.22 correction + ~0.08 risk), spending fewer tokens per solved task than Hybrid-Full (15.0k vs 13.5k) because it solves more episodes.
- **Multi-API:** GILP equalizes GPT-4o-mini, Claude-3-Haiku, Gemini-1.5-Flash, and Llama-3-8B to SR≈0.73–0.80 / HSR≈0.01–0.11, absorbing model-specific JSON noise; Gemini-Flash cheapest at $0.29/1k tasks, self-hosted Llama free.
- **Knowledge-Graph validation (AgentBench-style, n=12):** underpowered — gate fires 38% of steps but SR/HSR differences not significant; identifies an applicability boundary (backbone calibration) rather than a gate failure.

## Authors & Credibility
Xinyuan Song (Emory University) and Zekun Cai (The University of Tokyo / LocationMind). The work is methodologically careful: it explicitly discloses that the main table comes from a calibrated behavioural simulator (not live API), reports simulator–real residuals (SR under-predicted 10–18 pp, HSR over-predicted 4–7 pp, token cost over-estimated 3–4×), and frames reported gains as conservative lower bounds. Code, prompts, simulator, and benchmarks are publicly released (GitHub: Hik289/Environment-reduce-error). The error-reduction proposition is stated with its non-adversarial assumption and tolerance to backbone error.

## Critical Assessment
- **Novelty:** The agent-vs-parameterized world-model framing is a useful synthesis, and operationalizing multi-step *propagation* (rather than single-turn hallucination) via HSR/PD/EES is a genuine contribution distinct from existing faithful-CoT / self-consistency / process-reward literature. The consistency-gate mechanism itself (Jaccard on change-sets + targeted re-prompt) is conceptually simple but well-motivated.
- **Rigor:** Sound on formalization and ablation; the ablation grid (Table 5) cleanly isolates each skeleton field and each gate. The theoretical contraction guarantee is modest but honest about its assumption.
- **Reproducibility:** Strong — public code/prompts/simulator, fixed seeds, documented calibration. However, reproducibility of the headline numbers depends on the authors' simulator calibration, which cannot be independently checked against a live API at scale.
- **Significance:** Practically relevant for agent reliability; the "weak backbone suffices" and "API-agnostic gate" findings are deployable. But headline magnitudes should be treated as relative until larger live-API runs confirm them.
- **Red flags:** (i) The entire main comparison (n=3,200 trajectories × 12 methods) is simulator-derived, with only n=5 calibration and n=80 real-API validation; (ii) multi-API rows for Claude/Gemini/Llama use zero direct API calls (calibrated from published reports); (iii) KG validation underpowered (n=12); (iv) token costs reported as overstated 3–4×; (v) benchmarks are synthetic graph environments — whether the gate generalizes to free-text/continuous/partially-observable state is untested; (vi) HSR/PD cover only status-delta errors (entity-set and reward-attribution hallucinations out of scope).

## Strengths
- Clean conceptual separation of two world-model failure modes and the insight that their errors are complementary.
- Operational, horizon-resolved hallucination metrics (HSR/PD/EES) that make agent-based semantic error comparable across methods.
- Simple, cheap, API-agnostic intervention; does not require code synthesis or symbolic rule extraction.
- Honest, conservative reporting: simulator explicitly framed as a lower bound, residuals disclosed.
- Public release of all artifacts and a one-step contraction guarantee with stated assumptions.

## Limitations
- Headline results rest on a calibrated simulator; live-API validation is small (n=80; n=12 for KG).
- No direct API calls for Claude/Gemini/Llama in the multi-API study.
- Scope limited to graph-structured planning; richer state types untested.
- Backbone assumes access to oracle transitions; behavior under noisy/unavailable oracles unstudied.
- HSR/PD capture only node-status-delta hallucinations; other hallucination types excluded.
- Parameterized model can be confidently wrong, potentially misleading the agent (mitigated by advisory framing, not adversarially tested).

## Relevance
与智能体可靠性 / 幻觉传播这一核心议题高度相关：本文正是针对长程LLM智能体中"一步状态幻觉沿对话历史污染后续每一步决策"的失效模式，给出了可度量、可审计且API无关的纠错机制，直击当前Agent系统在多步规划中最致命的级联错误问题。

## Questions for Further Exploration
- How does GILP's gate behave on richer state representations (free-text observations, continuous values, partial observability) where clean change-set Jaccard is harder to define?
- Can the parameterized backbone be trained online from the agent's own verified executions instead of requiring oracle transitions?
- Does the contraction guarantee's non-adversarial assumption hold against systematically biased backbones (e.g., distribution shift, adversarial state)?
- What is the live-API effect size on a larger sample (the authors note n≥120 needed to detect sub-10pp HSR at 80% power)?
- How does the consistency gate interact with stronger reasoning models (o1/Claude-opus) whose baseline HSR is already low — is there a floor where the overhead no longer pays off?
- Could the propagation metrics (HSR/PD/EES) be adopted as standardized benchmarks for evaluating other agent-hallucination-mitigation methods?
