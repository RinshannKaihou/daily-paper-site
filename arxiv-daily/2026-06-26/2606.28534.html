Methodology

The work extends BIT1, a 1D3V electrostatic Particle-in-Cell Monte Carlo code (~31,600 lines of C) for plasma-edge and divertor physics, into a portable hybrid MPI+OpenMP multi-GPU model. The engineering strategy is a sequence of incremental, profiling-driven optimizations: replacing Unified Memory (page faults, implicit migrations) with Pinned Host Memory via OpenMP memory-allocator traits; switching from a 3D to a contiguous 1D data layout; moving to persistent GPU allocation with asynchronous OpenMP target tasks (mover, arranger) using use_device_ptr/is_device_ptr for direct device-pointer access.

For resilience, scalable Particle Load Balancing lets MPI rank 0 compute the particle distribution and variable cell intervals (new_nc_openPMD()), redistributing particles before parameterization. Checkpoint/Restart stores per-species particle meshes through the openPMD series, while ADIOS2 backends consolidate parallel I/O. Two physics use cases — uniform and non-uniform high-density sheath formation — are validated on Dardel, MN5 GPP/ACC, LUMI-G, and Frontier.

Optimization Stack (Ablation)

①Unified Memory → Pinned Memory
OpenMP allocator traits for performance-critical arrays; eliminates page faults.
②3D → 1D Layout
Contiguous memory access pattern improves throughput.
③Persistent + Async
Persistent GPU alloc; async OpenMP target tasks via use_device_ptr.
④PLB + C/R + I/O
Rank-0 particle redistribution; openPMD checkpointing; ADIOS2 BP4/SST.

Strengths & Limitations

✓ Strengths

  • True cross-vendor portability (Nvidia H100 and AMD MI250X) via a single OpenMP abstraction — a non-trivial engineering achievement.
  • Clean separation of I/O concerns through standards (openPMD, ADIOS2 BP4/SST) enabling both file-based and in-memory streaming workflows.
  • Quantified ablation-style progression (UM → PinM → 1D → async) makes the impact of each optimization legible.
  • Practical resilience story: PLB + C/R + in-situ analysis target real long-running plasma-simulation pain points.
  • Validated on leadership-class, production HPC systems rather than synthetic benchmarks.

! Limitations

  • Scale capped at 800 GPUs (~100 nodes); no true exascale node counts, yet conclusions are framed toward exascale.
  • No error bars, confidence intervals, or run-to-run variance — single-run-style results.
  • Code and input decks not stated to be openly available; reproducibility relies on descriptive detail.
  • PLB coordination is centralized on MPI rank 0 — a potential bottleneck at much larger scales, not stress-tested here.
  • Physics correctness (sheath results post-optimization) is assumed rather than explicitly demonstrated alongside performance.

Questions for Further Exploration

Q1

How does PLB's rank-0-centralized coordination scale to full-system (multi-thousand-GPU) Frontier/LUMI runs — does it become a serialization bottleneck?

Q2

What is the absolute overhead of BP4 vs SST checkpointing relative to compute time, and how does C/R cost grow with particle count?

Q3

Can the resilience mechanisms (C/R + SST streaming) generalize beyond PIC/MC to other irregular, dynamic-load GPU workloads (AMR, DSMC)?

Q4

How would the approach behave under actual failure injection (node/GPU loss) rather than graceful checkpoint/restart?

Q5

What numerical/correctness validation confirms the optimized BIT1 reproduces baseline plasma physics?
