01 Methodology

Personalization, reframed as a checkpoint-level monitoring problem.

The authors train a single cumulative LoRA adapter on one task per stage (no replay), producing a checkpoint θk after each stage. A fixed reference set R — disjoint from all train/eval data and never trained on — is probed after every checkpoint. The protocol pairs a checkpoint-by-task accuracy matrix (yielding ACC, OP, BWT, FWT, Learning Accuracy, Δadapt) with cheap distributional diagnostics over R versus the base model: KL divergence, mean entropy, and top-2 token margin. Three ≤1B SLMs run on FOMC, ScienceQA & NumGLUE-CM, across 3 seeds and both task orders — all on a single 11GB GPU.

03 Assessment

Strengths & limitations, side by side.

[+] Strengths

  • Cheap, deployable diagnostics — a fixed reference set + KL/entropy/margin beats re-running every task at each checkpoint.
  • Honest framing as a monitoring study, not an over-claimed new algorithm or benchmark.
  • Order-invariance analysis ties instability to architecture, not task-order noise.
  • Actionable failure zone (KL ≈ 0.8) that could trigger automated intervention.
  • Surfaces hidden failures — e.g. Gemma's margin collapse — invisible from final accuracy.

[!] Limitations

  • Small scale — only 3 tasks and 3 model families limit generalization.
  • Single PEFT config (fixed r=8, α=16) — no O-LoRA, QLoRA, or rank scheduling.
  • Reference set under-specified — composition, size & sensitivity not detailed.
  • Uncalibrated threshold — KL ≈ 0.8 is a heuristic, model/tokenizer-dependent.
  • No quantized evaluation, despite quantization being essential for real edge deployment.
04 Open questions

Where this could break — or extend.

1Is the KL ≈ 0.8 threshold universal across vocabularies, tokenizers & larger scales, or model-specific?
2Does the order-invariant signature persist with longer sequences (K ≫ 3) & more diverse domains?
3Can automated intervention on KL (early-stop, rollback) actually prevent collapse — at what cost to acquisition?
4How does the protocol behave under quantized (4/8-bit) SLMs, where edge deployment really happens?
5Would alternative PEFT (O-LoRA, rank scheduling, multi-adapter) reshape the stability fingerprint?
6What internal weight/representation shifts drive Gemma's collapse vs Qwen's stability — predictable pre-deploy?
