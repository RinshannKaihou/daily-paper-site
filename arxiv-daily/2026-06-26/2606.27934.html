Why this matters

Makes reported HPC and AI numbers checkable by readers who can't recompute them — directly tackling silent data corruption, AI Act logging duties, and cross-team result trust.

Critical edge ⚠

Injected corruptions are synthetic — natural silent-error rate on healthy silicon is ~0 under stress, so the framework guards a narrower threat (rare defective part or privileged attacker) than the abstract implies. No peer review; 3 Blackwell SKUs + 1 H200 only.

Methodology

benchmark = integrity problem, not presentation problem

A benchmark is modelled as a hash-linked acyclic evidence graph: root → claims → reductions → observations. Every edge is a SHA-256 hash over a canonical JSON byte image (lexicographic keys, shortest round-trip decimals). Auditing recomputes every node, checks every edge, and re-runs each pure reduction using only the Python standard library — no device, no network — in a single linear pass.

For a claimed product C=AB, the verifier draws a random probe X and forms the normalized residual ρ(C)=‖A(BX)−CX‖∞/(‖CX‖∞+ε) at O(kn²). The tolerance τ=m·ρ̄ (m=3) is calibrated to the device's own measured residual floor — a property of the silicon, not the algorithm — giving a 1−2⁻ᵏ detection guarantee.

Nonlinear ops (attention, atomic accumulation) get algebraic guards: softmax rows must sum to one; weighted sums satisfy the additive identity. These catch consistent faults invisible to run-to-run divergence. A Fiat-Shamir challenge derived from the claimed output closes the probe-seed attack while keeping verification offline and re-checkable on any device.

Strengths

what lands well
  • Turns “every number carries its evidence” into a concrete, offline-verifiable artifact — not a slogan.
  • Device-calibrated tolerance cleanly separates legitimate low-precision roundoff from genuine error; the floor is measured, not assumed.
  • Honest, self-critical security work — the authors attack their own construction (committed-seed null-space, coordinated-witness quorum) before fixing it.
  • Rich empirical payload (residual floors, reproducibility classes, two-card differential, cross-die penalty) useful far beyond the integrity framework.
  • Complete reproducible bundle — code, 240 observations, transcript, manifest, named root hash — standard-library only.

Limitations

where it strains
  • Validation = three Blackwell SKUs + one H200 NVL; AMD/Intel and future architectures are asserted, not demonstrated.
  • Bit-exact re-verification still needs the same architecture; only tolerance-bounded agreement is portable.
  • Softmax/attention reference is formed at a verification size, not the hot-path size; ops without a clean decomposition fall back to the weaker divergence class.
  • Corruptions are synthetic — natural SDC on healthy silicon is ~0, so value is the rare defective part or privileged attacker.
  • Tamper-evidence rests on SHA-256 collision resistance; detection on probe-column independence — standard assumptions, but worth restating.
  • Unreviewed arXiv preprint; claims not yet externally peer-reviewed.
