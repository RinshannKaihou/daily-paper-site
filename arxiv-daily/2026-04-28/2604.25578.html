Marco-MoE: Open Multilingual Mixture-of-Expert Language Models with Efficient Upcycling

Fan Jiang, Yu Zhao, Chenyang Lyu, Tianqi Shi, Yichao Du, Feihu Jiang, Longyue Wang, Weihua Luo

arXiv: 2604.25578v1

Date: 2026-04-28

https://arxiv.org/abs/2604.25578v1

一句话总结： Marco-MoE 提出首个面向多语言的细粒度 MoE Upcycling 方法，通过子矩阵切分 + Drop-Upcycling 将稠密模型高效转化为高度稀疏的 MoE 模型（仅激活约 5% 参数），在英/多语言基准上以更少激活参数（0.6B–0.86B）超越具有 3–14 倍激活参数量的竞争者。
TL;DR: Marco-MoE is the first fully open multilingual sparse MoE family that uses fine-grained upcycling (sub-matrix splitting + Drop-Upcycling) to convert a dense model into highly sparse MoE models activating only ~5% of parameters per token. With 0.6B–0.86B activated parameters, it matches or outperforms models using 3–14x more activated parameters on English and multilingual benchmarks.

---

核心贡献

  • 首个面向多语言的稀疏 MoE Upcycling 框架：首次将 MoE Upcycling 范式专门用于优化紧凑模型尺寸下的多语言性能，突破了稠密架构中的"多语言诅咒"（curse of multilinguality）。
  • 细粒度专家初始化方法：不同于传统的整块 FFN 复制，采用子矩阵切分（sub-matrix splitting）将稠密权重分解为细粒度专家，并结合 Drop-Upcycling 技术促进专家多样化和专业化。
  • 完全透明开放：公开完整预训练数据集、四阶段训练课程（5.1T tokens）、数据合成方法及模型权重，为多语言 LLM 研究设立开放标准。
  • 可扩展的语言覆盖：从 29 种语言成功扩展到 64 种语言，性能未见明显下降，展示了 MoE 架构在语言扩展上的天然优势。
  • 结构化专家激活模式发现：通过 Language-Expert Signature 分析，证明路由机制自然学习到与语言谱系高度一致的专家激活模式，相关语言共享专家池，孤立语言使用专用专家。

Key Contributions

  • First sparse multilingual MoE upcycling: First to apply the MoE upcycling paradigm specifically to optimize multilingual performance in compact model sizes, breaking the "curse of multilinguality" inherent in dense architectures.
  • Fine-grained expert specialization via sub-matrix splitting: Instead of replicating entire FFN blocks (coarse-grained), the method partitions dense weights into modular sub-components before expanding into an expert ensemble, catalyzing expert specialization.
  • Full transparency and openness: Discloses the complete pre-training datasets, four-stage curriculum spanning 5.1T tokens, data synthesis methodologies, and model weights -- setting a new standard for accessible multilingual LLM development.
  • Scalable language expansion to 64 languages: Demonstrates that Marco-MoE allows adding 35 new languages without the cross-lingual interference typical of dense models, preserving English performance while boosting multilingual results.
  • Linguistically structured expert activation patterns: Shows that MoE routers naturally learn activation signatures mirroring established linguistic family structures (Romance, Slavic, Austronesian clusters), with isolated experts for unique-script languages.

Methodology

Architecture: Decoder-only Transformer with sparse MoE layers replacing FFNs. Uses GQA, RMSNorm, SwiGLU, and RoPE (aligned with Qwen framework). Two model sizes: Marco-Nano (8B total / 0.6B active / 7.5% active ratio) and Marco-Mini (17.3B total / 0.86B active / 5% active ratio). Both upcycled from Qwen3-0.6B-Base.

Fine-Grained Upcycling Process:

  1. Reconfigure the dense model into a "Pseudo-MoE" where all experts remain active.
  2. Partition dense FFN weights into N = H_dense / H_expert slices along the intermediate dimension.
  3. Apply weight scaling factor (lambda = N^(1/3)) to each slice to preserve numerical integrity under MoE routing (prevents loss spikes).
  4. Apply Drop-Upcycling: randomly sample indices along intermediate dimensions, drop selected weights, and re-initialize them using Gaussian distribution with mean/variance from dropped weights.
  5. Expand pseudo-MoE into a real MoE with zero-initialized router weights and the fine-grained expert set.

Pre-training: Four-stage curriculum on 5.1T tokens using Megatron-LM:

  • Stage 1 (0-2.4T): High-quality English + reasoning + multilingual web/QA data (19 languages), constant LR.
  • Stage 2 (2.4T-4.1T): Upsampled reasoning, downsampled English web, boosted Chinese data, linear LR decay.
  • Stage 3 (4.1T-4.6T): Introduced 9 new languages (Bengali, Czech, Urdu, Kazakh, Greek, Romanian, Hungarian, Nepali, Azerbaijani), upsampled medium-resource languages.
  • Stage 4 (4.6T-5.1T): Integrated curated multilingual synthetic data (cultural, STEM, QA) targeting specific reasoning and linguistic capabilities.

Key Data Innovations:

  • Multilingual QA synthesis via Qwen3-30B-A3B-Instruct translation with line-level granularity and heuristic filtering for low-resource languages.
  • Multilingual STEM data via translation of OpenMathInstruct-2 and Nemotron-Pretraining-SFT-v1.
  • Cultural/Regional data pipeline using LLM-based quality/geographic filtering (Fineweb2-Culture) and Gemini-2.5-Pro-driven hierarchical topic expansion for synthetic MCQ generation (Synthetic-Regional-MCQs).

Post-training: Two-stage pipeline:

  1. SFT on Dolci-Instruct + scientific prompts + translation data + Wikidata-grounded cultural data.
  2. Cascaded On-Policy Distillation (OPD): first teacher Qwen3-30B-A3B-Instruct-2507, then Qwen3-Next-80B-A3B-Instruct, training on student's own generated trajectories with dense per-token supervision.

Results & Highlights

Base Models (Table 7):

  • Marco-Mini-Base achieves best English average (63.7), surpassing Qwen3-4B-Base (63.3) and Trinity-Mini (61.1), with substantially lower training compute (1.56e23 vs. 8.64e23 FLOPs) and fewer active parameters (0.86B vs. 3.85B).
  • Marco-Mini-Base ranks first on multilingual general benchmarks (50.9 avg), outperforming Qwen3-4B-Base (48.3) across GlobalMMLU, MMMLU, translation, and reasoning tasks.
  • On cultural/regional benchmarks, Marco-Mini-Base achieves 65.0, close to Qwen3-4B-Base (65.7), with particular strength on Global-PIQA (72.3), TurkishMMLU (62.7), KazakhMMLU (62.6).
  • Marco-Nano-Base (0.6B active) outperforms several larger dense baselines (Llama3.2-3B, Gemma3-4B, Tiny-Aya-3.35B) on English tasks.
  • Performance gap widens significantly for low-resource languages compared to dense counterparts.

Instruct Models (Table 11):

  • Marco-Mini-Instruct achieves best English average (75.5), surpassing Qwen3-4B-Instruct (73.3), Ministral3-8B-Instruct (70.9), Gemma3-12B-Instruct (65.8), and LFM2-24B-A2B (66.7) -- all with 3-14x more activated parameters.
  • Marco-Mini-Instruct ranks #1 in both general multilingual (50.8) and cultural/regional multilingual (71.0) performance.
  • Marco-Nano-Instruct (62.8 English avg) surpasses LFM2-8B-A1B (62.5) with only 40% of the activated parameters.

Analysis Insights:

  • Cascaded OPD yields consistent gains across all axes (English +1.8, General Multilingual +1.5, Cultural +1.1 for Mini variant).
  • Expert activation Pearson correlation heatmap reveals: Romance languages cluster together (Spanish, French, Portuguese, Italian), Slavic languages cluster (Russian, Ukrainian), Austronesian cluster (Indonesian, Malay), Indic cluster (Bengali, Nepali). Thai, Vietnamese, Arabic, Hebrew show low correlation with others -- indicating specialized expert subsets for linguistically isolated languages.
  • Multi-stage curriculum: English performance largely saturates after Stage 2, while multilingual and cultural performance continue improving through Stage 4.
  • Scaling to 64 languages preserves English proficiency (63.6 vs 63.7) while increasing multilingual advantage over Qwen3-4B-Base from 2.6% to 3.6%.

Weaknesses noted: CMMLU and C-Eval (Chinese exam-style benchmarks) remain challenging, where Qwen3-4B maintains a substantial advantage.

Critical Assessment

  • Novelty: Moderate. The individual components (MoE upcycling, fine-grained experts, Drop-Upcycling, OPD) are not new in isolation. The primary novelty lies in the _combination_ of fine-grained upcycling with multilingual objectives and the systematic engineering of a fully open pipeline. The expert activation pattern analysis is interesting but exploratory.
  • Rigor: High. Comprehensive evaluations across 40+ benchmarks covering English, general multilingual, and cultural/regional dimensions. Ablation studies are provided for data curation choices (QA, STEM, cultural data), weight scaling, and cascaded distillation. Multi-stage training monitored with per-stage evaluations.
  • Reproducibility: High relative to industry norms. Full data mixtures (Table 5), hyperparameters (Table 6), and training recipes are disclosed. Models are open-weight. However, some internal datasets and the exact implementation of synthetic data pipelines may be difficult to fully replicate without access to the same teacher models (Gemini-2.5-Pro, Qwen3-30B-A3B).
  • Significance: High for the multilingual LLM community. Demonstrates that sparse MoE architectures can effectively break the curse of multilinguality at compact scales, providing a practical and open blueprint for high-performance multilingual models without massive compute budgets.
  • Potential red flags:
  • The base model is upcycled from Qwen3-0.6B-Base, meaning the initial representations come from an external model trained on undisclosed data. While the continued pre-training and upcycling are fully disclosed, the "fully open" claim is qualified by the upstream model's training data being opaque.
  • On-policy distillation uses proprietary teacher models (Qwen3-Next-80B-A3B), limiting full independent reproduction of instruct variants.
  • No human evaluation or safety/toxicity assessment is reported -- the evaluation is entirely benchmark-driven.
  • The "best-in-class" claims are benchmarked against selected baselines; some recently released models may not be included.
  • The paper does not report inference latency or throughput, which are critical for practical deployment of MoE models with large total parameter counts.

Strengths

  • Exceptional performance-to-compute ratio: 0.6B-0.86B active parameters matching models with 3-14x more.
  • Broad multilingual coverage (29 core, expandable to 64) with systematic evaluation across general and cultural/regional benchmarks.
  • Complete transparency in data recipes and training methodology, rare among industry LLM papers.
  • Well-designed four-stage curriculum that maintains English performance while steadily improving multilingual and cultural capabilities.
  • Elegant expert activation analysis revealing linguistically meaningful routing patterns -- provides interpretability and architectural insight.
  • Practical post-training pipeline (SFT + cascaded OPD) that is computationally efficient (~110 hours on 64 GPUs).
  • Strong ablation studies validating each component of the data synthesis and training pipeline.

Limitations

  • Upcycled from Qwen3-0.6B, inheriting its architectural decisions and any biases in its pre-training data.
  • Chinese exam-style benchmarks (CMMLU, C-Eval) remain relative weaknesses compared to Qwen3-4B.
  • Limited to 64 languages, far from the 1,000+ language coverage of models like Apertus.
  • Heavy reliance on translation-based synthetic data for low-resource languages, which may introduce translation artifacts and fail to capture authentic cultural nuances.
  • No evaluation of safety, toxicity, bias, or hallucination -- purely capability-focused benchmarking.
  • No inference efficiency benchmarks (latency, throughput, memory) reported, which matters for MoE deployment.
  • Adding new languages requires retraining the full model rather than modular expansion.
  • Post-training relies on proprietary teacher models for OPD, partially undermining full reproducibility.

Relevance

该工作与多语言 NLP、高效模型训练、MoE 架构等方向高度相关。如果你关注如何用有限算力实现强大的多语言能力，或者对 MoE 路由机制的语言学可解释性感兴趣，这篇工作提供了一个完整的参考方案。其完全开放的数据配方和训练流程对于需要自行训练多语言模型的团队具有直接参考价值。但需注意该工作侧重于基准评估，未涉及安全性/偏见等部署层面的考量。

Questions for Further Exploration

  1. How does Marco-MoE perform on generative tasks (e.g., open-ended dialogue, summarization) beyond the multiple-choice and exact-match benchmarks reported? The evaluation suite is heavily weighted toward classification-style tasks.
  2. What is the inference latency and memory overhead in practice? With 17.3B total parameters (Marco-Mini), deploying the full expert set requires substantial GPU memory even though only 0.86B are activated per token.
  3. Can the fine-grained upcycling approach be applied to even larger dense models (e.g., 7B+), or does the scaling behavior change at larger sizes?
  4. How does the router behave when processing code-switched or mixed-language inputs, which are common in multilingual deployment scenarios?
  5. What are the failure modes in extremely low-resource languages where synthetic data quality is poor? The paper uses heuristic filtering but does not characterize quality degradation.
  6. Would load-balanced or auxiliary-loss-free routing strategies further improve expert specialization for the long-tail languages?
  7. How do the cultural/regional capabilities transfer to practical applications like content moderation, local search, or culturally sensitive dialogue?
