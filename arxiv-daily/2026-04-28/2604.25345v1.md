> **一句话总结：** 本文对CMBAgent在18个天体物理任务上的系统评估揭示：AI科学智能体的最大风险不是显性崩溃，而是在缺乏领域上下文时生成“看似合理但实际错误”的结果——在One-Shot计算任务中领域上下文带来约6倍性能提升(0.85 vs.~0)，在Deep Research推理任务中欠约束问题导致系统性静默失败（报告物理上不可能的后验分布且不自我诊断）。

> **TL;DR:** This paper systematically evaluates CMBAgent across 18 astrophysical tasks under two workflow paradigms. The central finding: domain-context retrieval drives a ~6x performance gain in tool-grounded computation, and the most dangerous failure mode in agentic scientific workflows is not crashing but the confident, undiagnosed generation of plausible yet physically incorrect results. Under-constrained inference tasks fail systematically without self-awareness.

## 核心贡献

- **首次系统化刻画科学AI智能体的静默失败模式：** 在One-Shot和Deep Research两种工作流范式下，发现最危险的故障不是代码执行失败，而是“语句正确但数值错误”的静默计算（Wrong Computation），以及物理上不可能但被自信报告的后验分布。
- **提出四级失败分类体系 (Failure Mode Taxonomy)：** 将每次实验归类为 Code Failure (无法执行)、Wrong Parameters (参数配置错误)、Wrong Computation (数值计算错误)、Correct (正确) 四个互斥类别，其中 Mode C 是最隐蔽且危害最大的故障模式。
- **领域上下文作为关键驱动因素的实证证据：** 使用相同GPT-4o-mini骨架，最终得分从Base LLM (~0) 到 CMBAgent无上下文 (0.15) 到 CMBAgent完整上下文 (0.85)，跨度约6倍，揭示了领域文档检索而非模型能力是性能瓶颈。
- **设计结构化压测评估框架：** 通过消融任务条件（欠约束后验、组合推理管道）而非架构组件，系统性地暴露智能体在不同推理机制下的弱点，并公开评估框架以支持可重复性分析。
- **Deep Research场景下的定性+定量双重评估：** 结合参数恢复得分 (PRS) 和物理合理性/故障透明度两个维度，对四个研究级天体物理推理问题进行综合诊断。

## Key Contributions

- **First systematic characterization of silent failure modes in scientific AI agents:** Across 18 tasks, identifies that the most dangerous failure is not overt crashes but syntactically-valid code producing wrong numerical results (silent wrong computation) and physically impossible posteriors reported without self-diagnosis.
- **Four-level Failure Mode Taxonomy:** Classifies each trial into Code Failure, Wrong Parameters, Wrong Computation, or Correct — establishing Mode C (Wrong Computation) as the most insidious failure category where outputs appear plausible but are numerically wrong.
- **Empirical evidence that domain context is the dominant performance driver:** With an identical GPT-4o-mini backbone, final scores span from ~0 (Base LLM) to 0.15 (no context) to 0.85 (full context) — a ~6x gain — demonstrating that retrieval access, not model capability, is the bottleneck.
- **Structured stress-test evaluation framework:** Rather than ablating architecture, ablates task conditions (under-constrained posteriors, compositional reasoning pipelines) to systematically expose agent weaknesses under distinct inference regimes. Framework is publicly released.
- **Dual quantitative/qualitative assessment for Deep Research:** Combines Parameter Recovery Score (PRS) with qualitative assessment of physical plausibility and failure transparency across four research-grade astrophysical inference problems.

## Methodology

**Evaluated System:** CMBAgent — an open-source agentic framework for autonomous scientific discovery. It serves as the research backend in the Denario project, generating research ideas, methodologies, and results through coordinated multi-agent planning. The backbone model is GPT-4o-mini.

**Two Workflow Paradigms:**

1. **One-Shot Workflow (14 tasks):** Tool-grounded precision tasks using the CAMB cosmology library. Tasks are stratified by API interaction complexity:
   - Tasks 1-6: Single API call
   - Task 7: Single API call with tensor handling
   - Task 8: Alternate CAMB module
   - Task 9: Multi API calls
   - Task 10: Multi API + multi-result combination
   - Tasks 11-12: Multi API + delensing pipeline
   - Task 13: Multi API + ratio computation
   - Task 14: Multi API + noise-informed delensing pipeline

   Three system variants tested: Base LLM (GPT-4o-mini direct call), CMBAgent without CAMB documentation, and CMBAgent with CAMB documentation.

   **Metrics:** Execution Success Rate (ESR — binary, verifies output file structure), Parameter Accuracy Score (PAS — cosine similarity of parameter configurations), Numerical Accuracy Score (NAS — combines NRMSE and CCC for output curves), Final Score (= PAS × NAS if ESR=1, else 0).

2. **Deep Research Workflow (4 tasks):** Research-grade astrophysical inference:
   - T1: SN1a cosmology (under-constrained: H₀-M_B degeneracy)
   - T2: NGC 3198 rotation curve fitting (under-constrained: NFW halo with degenerate high-mass, low-concentration ridge)
   - T3: Exoplanet mass-radius relation (compositional: broken power-law hierarchical model)
   - T4: SLACS strong lensing (compositional: SIS model per-object inference)

   **Metrics:** Parameter Recovery Score (PRS — agreement with literature references, saturating to zero at 3σ deviation) plus qualitative assessment on physical plausibility and failure transparency.

**Task-Level Robustness Evaluation:** Instead of ablating architectural components, the paper ablates task conditions — varying prior specification, likelihood completeness, and model complexity — while keeping the workflow fixed. Tasks are categorized as either "Under-constrained" (degraded priors, missing likelihood terms that create known degeneracies) or "Compositional" (full hierarchical likelihood, no prior degradation, testing multi-stage inference consistency).

## Results & Highlights

**One-Shot Workflow:**
- **CMBAgent with CAMB context:** Near-ceiling performance — ESR=0.96, PAS=0.95, NAS=0.86, Final=0.85. The gap between PAS and NAS reveals residual numerical errors on hard tasks even with correct parameters.
- **CMBAgent without context:** Sharp degradation — ESR=0.62, PAS=0.54, NAS=0.18, Final=0.15. Mode C (Wrong Computation) accounts for ~47% of trials — the agent invokes plausible but incorrect API calls, producing curves with the right shape but wrong amplitude/spectral content.
- **Base LLM:** Near-total failure — ESR=0.09, ~91% classified as Mode A (code failure). Raw model capability without agentic scaffolding is insufficient for tool-grounded scientific computation.
- **Task-level failures even with context:** Task 10 (score 0.09) — missing `raw_cl=True` producing ~10^5 amplitude error, plus missing tensor B-mode contribution. Task 14 (score 0.02) — correctly computes per-ℓ delensing efficiency but collapses it to a scalar mean. Both are output construction errors, not API misuse.

**Deep Research Workflow:**
- **T1 (SN1a Under-constrained):** PRS=0.97 — Ω_Λ well-recovered (0.722±0.001 vs. reference 0.720±0.020), but H₀ treated as a free parameter when the likelihood cannot constrain it. The H₀-M_B degeneracy was never flagged.
- **T2 (NGC 3198 Under-constrained):** PRS=0.05 — systematic failure. The posterior converges to log₁₀(M₂₀₀) nearly 6σ above reference with unphysical concentration c≈1.2 (reference c=6.8±2.0). No trial flagged the pathology.
- **T3 (Exoplanets Compositional):** PRS=0.73 — mixed results. Neptunian slope α_N robustly recovered; Jovian slope α_J wrong in sign/magnitude in 3/5 trials; break mass M_br implausible in 2/5 trials; intrinsic scatter σ_int pinned at prior boundary. No trial flagged inconsistent parameterizations.
- **T4 (SLACS Compositional):** Only 1 of 5 trials produced usable output. The single trial showed ~5% systematic bias (f≈1.05 vs. reference 1.019±0.008), a ~4σ offset the agent did not flag.
- **Universal failure transparency failure:** Across all four tasks, no trial flagged degeneracies, implausible parameters, prior-boundary pathologies, or systematic biases — confident estimates were reported without caveats.

## Critical Assessment

**Novelty:** Moderate-to-high. While agent evaluation is a growing field, this paper makes a genuinely useful contribution by shifting focus from aggregate task success to failure mode characterization. The four-level taxonomy (especially highlighting Mode C — "wrong but looks right") — is practically useful. However, the specific finding that domain context improves performance is not surprising; the value lies in quantifying this effect and showing precisely how failures manifest without context.

**Rigor:** Moderate. The One-Shot evaluation is systematic with 14 tasks, 10 trials each, and three system variants. The Deep Research evaluation is thinner — only 4-5 trials per task, and the qualitative assessment (physical plausibility, failure transparency) is inherently subjective. The paper does not report inter-rater reliability for qualitative judgments. The backbone model is a single model (GPT-4o-mini), so claims about "agentic AI systems" generalize only to this specific configuration.

**Reproducibility:** Good for One-Shot (automated metrics, open-source framework). Limited for Deep Research (qualitative assessment, prompts provided in appendix but hard to fully replicate the qualitative judgment). The CAMB library tasks use publicly available reference outputs, and the CMBAgent framework is open-source.

**Significance:** High practical relevance. As AI agents are increasingly deployed in scientific workflows, understanding failure modes — especially silent ones — is critical. The paper's taxonomy and evaluation framework provide a template that can be adapted to other domains beyond astrophysics. The release of the evaluation framework adds practical value for other researchers.

**Red Flags:**
- **Single backbone model (GPT-4o-mini):** All results are with one model. It is unclear whether findings generalize to stronger models (GPT-4, Claude, Gemini) or whether the degradation patterns are specific to the model's capability level.
- **Small Deep Research trial count:** 4-5 trials per task is thin for statistical claims. The qualitative assessment has no reported inter-rater reliability.
- **Domain-specificity:** All tasks are in astrophysics. While the framework is claimed to be general, only one domain is tested.
- **No comparison with other agentic systems:** Only CMBAgent is evaluated. The failure patterns could be specific to this framework's design choices.
- **Unclear whether the right evaluation signal was captured:** Tasks 10 and 14 failures are described as "output construction errors" — it is debatable whether these are truly agentic failures or just documentation gaps. A scientist would also fail without knowing the right post-processing conventions.

## Strengths

- **Practical and actionable failure taxonomy:** The Code/Wrong Parameters/Wrong Computation/Correct classification is intuitive, operationalizable, and directly useful for practitioners deploying AI agents in scientific settings.
- **Honest framing:** The paper explicitly states it is not proposing a new model or critiquing architectures — it is a focused empirical evaluation, and it delivers exactly that.
- **Public release of evaluation framework:** Enables other researchers to replicate and extend the analysis, making this a building block for future work rather than a one-off paper.
- **Task stratification by inference regime:** The Under-constrained vs. Compositional categorization provides mechanistic insight into when and why failures occur, going beyond surface-level performance tables.
- **Bilingual appeal to both AI and science communities:** The paper speaks to both AI reliability researchers and practicing scientists who are potential users of agentic tools, with appropriate depth for both audiences.
- **Clear, well-written exposition:** The distinction between overt failure (crashes) and silent failure (wrong but plausible) is communicated effectively and supported by concrete examples from the astrophysical tasks.

## Limitations

- **Single model (GPT-4o-mini) and single agentic framework (CMBAgent):** The results cannot be generalized to other LLMs or agent architectures without additional studies. Stronger models may exhibit different (possibly worse in different ways) failure patterns.
- **Domain-limited to astrophysics:** While the evaluation framework claims generality, it has only been instantiated in one domain. The specific failure modes (e.g., missing `raw_cl=True` in CAMB) are domain-specific.
- **Small Deep Research sample size:** 4-5 trials per task means statistical confidence is limited. The qualitative assessment has no reported inter-rater reliability, making the "failure transparency" dimension difficult to reproduce.
- **No ablation of agent components:** The evaluation treats CMBAgent as a black box. It is unclear whether failures originate from the planner, the engineer agent, the retrieval system, or the LLM backbone — this limits actionable guidance for fixing the problems.
- **No positive control for Deep Research:** The paper does not report what performance a human expert would achieve on the same tasks, making it difficult to calibrate expectations. Some failures (like the T2 unphysical NFW concentration) might also trip up junior researchers.
- **Retrieval quality not analyzed:** The ~6x gain from CAMB context access could reflect either the agent's ability to use documentation or the documentation's quality — these are conflated.

## Relevance

本文直接切中“AI智能体在科学研究中的可靠性”这一当前热点问题，对于关注AI辅助科学发现、LLM智能体评估、以及科学计算自动化方向的研究者具有直接参考价值。其静默失败的概念和四级分类框架可以迁移到其他科学领域（如生物信息学、材料科学、气候模拟等），对于正在开发或部署AI科学助手的团队尤为实用。

## Questions for Further Exploration

1. How do failure patterns change with stronger backbone models (GPT-4, Claude 4, Gemini 2.5)? Would a more capable model produce fewer Mode C failures, or would it produce more confident and therefore more dangerous silent failures?
2. Can the failure taxonomy (Code/Parameters/Computation/Correct) be automatically detected in real-time during agent execution, enabling a "scientific guardrail" system?
3. How would human experts perform on the same Deep Research tasks? What is the human baseline for parameter recovery and failure transparency?
4. Are the failure modes specific to CMBAgent's architecture, or do they generalize across agentic frameworks (AutoGen, LangGraph, CrewAI)?
5. What is the minimum documentation quality needed for reliable agent performance? Could targeted documentation improvements eliminate most Mode C failures?
6. Can chain-of-thought prompting or explicit "sanity check" steps reduce silent failures in under-constrained inference tasks?
7. How do these findings translate to other scientific domains (bioinformatics, materials science, climate modeling) where tools and constraints differ substantially?
