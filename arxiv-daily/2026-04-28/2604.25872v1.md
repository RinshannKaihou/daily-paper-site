> **一句话总结：** 本文从理论上证明了策略梯度优化中的奖励错误并非全是坏事——某些错误可以是有益的（通过阻止策略停滞在中庸输出上），并据此提出了感知危害性的奖励模型评估指标和可验证设置下的奖励设计见解。

> **TL;DR:** This paper theoretically categorizes reward errors in policy gradient optimization into harmful, benign, and beneficial types, proving that some reward errors can actively help by preventing the policy from stalling on mediocre outputs. It then applies this categorization to (1) develop harm-aware ranking accuracy metrics for RLHF reward model evaluation that better predict downstream LM performance, and (2) provide guidance on when rewarding partially correct outputs helps vs. hurts in verifiable reward settings.

## 核心贡献

- **奖励错误三分类理论**：将策略梯度中的奖励错误（proxy 与 ground truth reward 之间的偏差）分为有害（harmful）、无害（benign）和有益（beneficial）三类，而非传统的"所有错误都有害"观点。
- **有益错误的机制**：证明了给"中庸质量输出"（mediocre outputs）分配低 proxy reward 可以是有益的——因为这些输出会吸引概率、导致策略梯度长时间停滞在其附近，主动降低其奖励可以加速向最优输出的收敛。
- **两种无害错误类型**：(1) proxy reward 低于初始期望奖励的输出不会吸引概率；(2) 初始策略下概率极低的输出即使被错误评分也几乎不影响优化结果。
- **两种有害错误类型**：(1) 传统奖励黑客——给低质量输出高奖励；(2) 给低质量输出中等奖励，即使不足以引发奖励黑客，也会通过吸引概率停滞来拖慢优化。
- **Harm-Aware Ranking Accuracy (HAcc)**：基于上述分类提出了感知危害性的奖励模型排序准确率指标，相比标准 ranking accuracy 更好地预测 RLHF 后语言模型的实际性能。
- **可验证奖励的奖励设计指导**：在 instruction following 等任务中，当策略初始更可能生成部分正确而非完全正确的输出时，给予部分正确性奖励可能反而阻碍学习；二元（仅全对）奖励可能更有效。

## Key Contributions

- **Three-way categorization of reward errors**: Theoretically classifies reward errors under policy gradient into harmful, benign, and beneficial, challenging the conventional view that all deviations from ground truth are strictly harmful.
- **Existence of beneficial reward errors**: Proves that assigning low proxy reward to outputs with mediocre ground truth reward can accelerate optimization by preventing the policy from stalling near these "attractive" suboptimal outputs. Provides rigorous time-complexity bounds (Omega vs. Big-O separation) in terms of initial policy probabilities.
- **Two types of benign errors**: (1) outputs with proxy reward below initial expected reward do not attract probability; (2) reward errors on outputs with negligible initial probability have no material effect on the optimization trajectory (proven via gradient flow stability analysis).
- **Two types of harmful errors**: (1) the classic reward hacking scenario (high proxy reward for low-quality outputs); (2) a subtler harm: mediocre proxy reward for low-quality outputs that stalls optimization without triggering outright reward hacking.
- **Harm-Aware Ranking Accuracy (HAcc)**: A modified reward model evaluation metric that does not penalize incorrect rankings where all dispreferred outputs receive proxy reward below the expected proxy reward, typically yielding better Spearman correlation with post-RLHF LM performance across Llama, OLMo, and Qwen model families.
- **Practical reward design insights**: Demonstrates on IFBench instruction-following tasks that rewarding partial correctness can cause the policy to learn only the easier constraint, while binary (full-correctness) rewards enable learning both constraints when starting probabilities differ significantly.

## Methodology

### Theoretical Framework

The core theoretical analysis uses **linear softmax policies** in bandit environments with a single input. Each output y is associated with a feature vector phi(y), and the policy is parameterized as pi_theta(y) = exp(<phi(y), theta>) / sum_z exp(<phi(z), theta>). Optimization is analyzed via **gradient flow** (continuous-time gradient descent): d/dt theta_t = grad V_P(theta_t), where V_P is the expected proxy reward.

The authors analyze the logit dynamics of each output y:
  d/dt <phi(y), theta_t> = pi_theta_t(y) * A_P(y; theta_t) * ||phi(y)||^2 + sum_{z != y} pi_theta_t(z) * A_P(z; theta_t) * <phi(z), phi(y)>

This decomposition reveals which outputs attract probability mass during training.

### Categorization Logic

- **Benign Error I**: If r_P(y) < V_P(theta_0), then the output's own advantage is negative, so it does not attract probability (Proposition 1).
- **Benign Error II**: If the set of mislabeled outputs Z has low initial probability, gradient flow trajectories under r_P and r_G remain close in total variation distance (Proposition 2).
- **Beneficial Error I**: When outputs with mediocre reward (r_G(y_med) between V_G(theta_0) and r_G(y*)) have higher initial probability than the optimal output, they attract probability and stall optimization. The time to escape scales as Omega(pi_theta_0(y*)^{-14/13}) when using r_G directly, but only O(pi_theta_0(y*)^{-1}) when y_med is assigned low proxy reward (Theorem 2). This yields an **arbitrarily large** separation.
- **Harmful Error II**: The reverse -- assigning mediocre proxy reward to low-quality outputs -- creates the same stalling effect (Corollary 1).
- **Feature similarity effects**: When <phi(y*), phi(y_med)> < 0, the stalling is worse; when positive, it is milder. With sufficiently positive inner products, assigning mediocre rewards may even be *necessary* for convergence (Theorems 3 and 4).

### Practical Applications

**Application 1 -- Harm-Aware Ranking Accuracy**: HAcc modifies standard ranking accuracy by not penalizing an incorrect ranking whenever all dispreferred outputs receive proxy reward below the estimated expected proxy reward (sV_P(x; theta)). A weighted variant (HAcc-W) further downweights examples involving low-probability outputs.

**Application 2 -- Reward Design for Verifiable Settings**: Controlled experiments on IFBench instruction-following tasks where Qwen3-1.7B is trained with GRPO, comparing partial rewards (0.5 per satisfied constraint) vs. binary rewards (1 only if both constraints satisfied).

### Experimental Setup

- **Models**: Llama-3.2-3B-Instruct, Llama-3.2-1B-Instruct, OLMo-2-1B-SFT, Qwen3-1.7B-Base
- **RL algorithm**: RLOO (also tested with GRPO)
- **Training data**: UltraFeedback prompts
- **13 reward models**: Spanning diverse RewardBench2 scores
- **Ground truth proxy**: ArmoRM (excluded from training reward models)
- **Instruction following**: Custom datasets built from UltraFeedback prompts + IFBench constraints

## Results & Highlights

### Reward Model Evaluation (Section 4)
- HAcc and HAcc-W typically achieve **higher Spearman correlation** with post-RLHF ground truth reward increase than standard Acc and Acc-W.
- **Regret reduction**: When choosing the best reward model by HAcc, regret drops substantially (e.g., from 25.8% to as low as 3.1% for some models).
- However, all correlations remain **below 0.4** and can be negative in some settings, demonstrating that robust reward model evaluation remains an open challenge.
- Weighting by output probability (Acc-W, HAcc-W) does not consistently help, suggesting that low-probability outputs are not the primary cause of ranking accuracy's poor predictiveness.

### Reward Design for Verifiable Settings (Section 5)
- When one constraint has noticeably higher initial satisfaction probability than another, **binary rewards** enable learning both constraints while **partial rewards** cause the policy to learn only the easier one.
- When both constraints have similar initial probabilities, both reward designs work well.
- The key practical insight: rewarding partial correctness is neither universally helpful nor harmful; the choice depends on the task structure and the language model's initial capabilities.

### Controlled Theoretical Experiments
- Figure 2 empirically confirms Theorem 1: smaller pi_theta_0(y*) leads to longer stalling when y_med receives mediocre reward.
- Figure 5 confirms feature similarity effects: negative inner product worsens stalling, positive inner product mitigates it.

## Critical Assessment

### Novelty
The core insight that not all reward errors are equally harmful is genuinely novel. While prior work (Mei et al. 2020) observed that policy gradient can be attracted to mediocre outputs, the question of *when and to what extent* this stalls optimization was open. The formal categorization and especially the proof of *beneficial* errors with an arbitrarily large time-complexity gap is a new theoretical contribution.

### Rigor
The theoretical analysis is mathematically careful, with full proofs in the appendices (spanning ~40 pages). The gradient flow analysis with explicit bounds (Omega vs. Big-O) is solid. However, the bandit + linear softmax + single-input setting is highly stylized. The authors acknowledge these limitations clearly.

### Reproducibility
The experimental setup is described in reasonable detail, with implementation details in Appendix D. However, the harm-aware metrics depend on computing expected proxy rewards from the language model, introducing an additional source of variance. The authors do not report standard deviations across the 3 runs for correlation values.

### Significance
The work has clear practical implications for two active research areas: reward model benchmarking (RewardBench2, etc.) and reward design for RLVR. The finding that HAcc is better but still insufficient (correlation < 0.4) usefully delineates where current approaches stand. However, the practical adoption of HAcc may be limited by its additional computational cost (though small relative to training).

### Potential Red Flags
- The theoretical setting (bandit, single input, linear softmax, exact gradients) is far from real LLM training. The authors justify this by showing empirical alignment, but the gap could hide practical failure modes.
- The harm-aware metrics improve correlation but still underperform in some settings (negative correlations for some model/reward combinations), which the paper honestly acknowledges.
- The instruction-following experiments use only two constraints per prompt; real tasks involve many more.
- No comparison against alternative reward evaluation metrics beyond standard ranking accuracy (e.g., no comparison to StarC or dynamics-aware metrics).

## Strengths

1. **Rigorous theoretical foundation**: The paper provides formal proofs with explicit time-complexity bounds, not just intuition or empirical observation.
2. **Counter-intuitive main result**: The proof that reward errors can be *beneficial* (not just benign) challenges conventional wisdom in a well-motivated way.
3. **Practical dual application**: The theory directly motivates two practically relevant contributions (HAcc and reward design guidance) that are validated empirically across multiple models and datasets.
4. **Intellectual honesty**: The authors openly discuss limitations, including that HAcc correlations remain weak, making the paper more trustworthy than work that oversells results.
5. **Comprehensive feature geometry analysis**: The extension to non-orthonormal features (Theorems 3 and 4) adds significant depth, showing how feature similarity can flip the categorization.
6. **Clear exposition**: Figure 1 provides an excellent visual overview of the categorization, and the paper is well-structured.

## Limitations

1. **Stylized theoretical setting**: Linear softmax policies, bandit environments, single input, and exact gradient flow are strong simplifications. The gap to real LLM training with neural policies, multi-turn interactions, and stochastic gradients is substantial.
2. **HAcc still insufficient**: The harm-aware metrics improve over standard ranking accuracy but remain weak predictors (correlation < 0.4), meaning they cannot reliably select the best reward model in practice.
3. **Limited instruction-following scope**: Only two constraints per prompt in the verifiable reward experiments; real tasks like code generation involve many more interacting constraints.
4. **No comparison to alternative reward evaluation metrics**: The paper only compares against standard ranking accuracy, not against more sophisticated alternatives like StarC, EPIC, or dynamics-aware metrics.
5. **Computational overhead of HAcc**: Requires sampling from the language model and computing output probabilities, which adds cost -- though the paper argues this is modest relative to RL training.
6. **Single ground truth model**: All RLHF experiments use a single reward model (ArmoRM) as ground truth, which may introduce systematic bias.

## Relevance

该论文直接涉及语言模型强化学习后训练中的核心问题：如何评估和设计代理奖励函数。对于从事 RLHF、reward model evaluation、以及可验证奖励（如代码生成、数学推理、指令遵循）研究的读者，本文提供了新的理论视角和实用工具。其提出的"奖励错误并非全是有害的"这一论点对 reward benchmark 的设计和 reward engineering 的实践具有直接指导意义。特别值得注意的是，该工作揭示了**奖励函数有效性依赖于其与初始策略和优化算法的交互**，这一原则性见解对于正在构建 RL 训练管线的研究者尤为重要。

## Questions for Further Exploration

1. Can the harm-aware ranking accuracy approach be extended to leverage *training-dynamic* information (e.g., evaluating reward models not just on the initial policy but at multiple checkpoints during training)?
2. How do these reward error categories interact across multiple inputs in a full RLHF training run? Could errors that are benign/beneficial in isolation become harmful when aggregated across a distribution?
3. What is the theoretical characterization of beneficial/harmful errors for stochastic gradient estimates (e.g., GRPO, RLOO, PPO) rather than exact gradient flow?
4. Could adaptive reward schemes that switch between partial and binary rewards at different training stages (as suggested by Sun et al. 2026) be formalized using this categorization framework?
5. How would the categorization change for process reward models (PRMs) that provide per-step feedback rather than outcome-level rewards?
6. Is there a way to estimate the "mediocrity threshold" for a given model and task that would predict whether a proxy reward error is harmful or beneficial without running full RL training?
