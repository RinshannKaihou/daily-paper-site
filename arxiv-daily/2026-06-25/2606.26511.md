> **一句话总结：** 本文指出 RAG 在知识演化时存在结构性"过时事实"缺陷（余弦相似度无法区分矛盾与重复，AUROC 仅 0.59），并提出 MemStrata：通过确定性的 (subject, relation, object) 替代规则在双时态账本上退役过时事实，无需相似度阈值与 LLM 调用，在四个演化基准上将过时事实错误率从 RAG 的 15-40% 降至接近 0%，且检索延迟与朴素 RAG 相当（约 2.1s）。

> **TL;DR:** RAG has no model of time and cannot tell a stale fact from a current one (cosine similarity separates contradictions from duplicates at near-chance AUROC 0.59). MemStrata adds a deterministic (subject, relation, object) supersession rule over a bi-temporal ledger that retires superseded facts without similarity thresholds or LLM calls, driving stale-fact-error rates from RAG's 15-40% to ~0% while matching RAG latency (~2.1s) and tying RAG on static knowledge.

## 核心贡献

- **相似度方法的结构性不可能结果**：在 98 个标注对上，余弦相似度区分"矛盾"与"重复"的 AUROC 仅 0.59（接近随机），且最大可达精度仅 0.667，远低于安全自动更新所需的 0.95 阈值；矛盾项反而比真正的重述更接近原始事实（因为数值翻转是最小编辑）。
- **时态有效性记忆架构 MemStrata**：像 RAG 一样存储事实（保留静态召回），但当新断言与既有事实的 (subject, relation) 键冲突但 object 不同时，用确定性替代规则在双时态账本中退役过时值——无相似度阈值、无 LLM 调用、读路径无 LLM。
- **过时事实错误的结构性结果**：在强制回答时，RAG 以 15-40% 的概率返回过时值，而 MemStrata 降至约 0%；这是 RAG 在构造上无法避免的失败类别。
- **无标记评估协议**：构造四个"无标记"演化基准（旧/新版本除改动值外文本完全一致，禁止 old/new/current/deprecated 等词），证明文本标记会使基线虚高最多 14 个百分点，并推荐该不变性作为演化记忆评估的标准。
- **诚实界定边界**：明确承认方法受限于结构化单值模板（混乱自然语言矛盾的抽取率从 97% 降至 44%），并将失败基准隔离而非粉饰。

## Key Contributions

- **A structural impossibility result for similarity-based staleness detection.** On 98 labeled pairs, cosine AUROC for separating contradictions from duplicates is 0.59, and the maximum achievable precision is 0.667 (the 0.95 safety floor is unreachable); contradictions are *more* embedding-similar to the original than genuine rephrasings because a value-flip is a minimal edit.
- **A temporal-validity memory architecture (MemStrata).** Stores facts like RAG (full static recall) but applies a deterministic (subject, relation, object) supersession rule when a fact's value is contradicted, retiring the stale value in a bi-temporal ledger (valid_from / valid_to / superseded_by) with no similarity threshold and no LLM call.
- **The stale-fact-error result — a failure class RAG cannot avoid by construction.** When forced to answer, RAG serves superseded values 15-40% of the time across four evolving benchmarks; MemStrata drives this to ~0% because the stale value is removed from the store before retrieval.
- **A marker-free evaluation protocol.** Four evolving benchmarks where stale and current versions are textually identical except for the changed value, with no old/new/current framing enforced by unit tests; the authors show a textual marker silently inflated baseline accuracy by up to 14 points.
- **Honest boundary-setting.** The quarantined `fair_contradiction` benchmark (messy free-prose) where extraction drops to ~44% is excluded from main results and reported as the explicit limitation, bracketing the claim between two ablations (lossy over-merge forfeits static recall; no-supersession forfeits temporal validity).

## Methodology

MemStrata is a local memory layer between an agent and its LM, with separate write and read paths.

**Write path (deterministic supersession over a surprise gate):**
1. Exact-duplicate short-circuit — a normalized text hash drops verbatim repeats at zero cost.
2. Deterministic assertion path — if a turn expresses a clean (subject, relation, object) triple with a single mutable object value, MemStrata normalizes the (subject, relation) key and checks for an active assertion. Different object → the new row supersedes the old (close valid_to, link superseded_by, open new row). Same object → reinforce. No prior key → novel store. No cosine, no LLM judge.
3. Text-gate fallback — non-triple prose falls through to a surprise gate (similarity + LLM judge) that retains non-contradictory near-duplicates and drops only exact duplicates.

**Bi-temporal ledger:** Facts are retired, not deleted, retaining validity intervals for future as-of-time queries. Active retrieval surfaces only currently-valid rows.

**Retain-then-supersede design:** A key design choice — the published config retains like RAG (stores distinct non-contradictory facts) and bounds growth only by superseding contradictions. The lossy ablation (aggressive near-duplicate merging) collapses static recall to 0.62/0.13, isolating why this matters.

**Read path:** Embed query → retrieve top-k by cosine over active facts/assertions → apply deterministic staleness filter (drop superseded rows) → pack surviving assertions' *original source sentences* (not terse triples, which degrade the answer model). No LLM runs on the read path, so latency sits at the embedding floor (~2.1s).

**Experimental setup:** All local and deterministic — temperature 0, fixed seeds, no network (enforced by test). Answer model Qwen2.5-Coder-7B; correctness and fabrication judges Qwen2.5-Coder-3B (distinct from answer model and each other to prevent self-grading); embedder nomic-embed-text (768-d). 8 conditions × 6 benchmarks. 2 static benchmarks (domain: 50 project-fact QA; locomo: 30 multi-session dialogue questions over 100 turns). 4 marker-free evolving benchmarks, 20-30 paired scenarios each (code_mutation, config_migration, dependency_bump, api_evolution).

## Results & Highlights

- **Static parity:** temporal_v6 ties RAG on static knowledge (0.82/0.30 vs 0.86/0.30). The lossy ablation collapses to 0.62/0.13, isolating the cost of aggressive compression.
- **Evolving dominance:** temporal_v6 reaches 0.95-1.00 accuracy vs RAG's 0.20-0.47 — a 2-5x improvement on the targeted task class.
- **Stale-fact-error (the central result):** Forced to answer, RAG serves the superseded value 15-40% of the time; MemStrata reaches ~0% in both abstention-allowed and forced-answer regimes. The surprise-gate similarity-based conditions (v6_no_verify, v6) leak stale 25-60%, confirming Section 3's impossibility result end-to-end.
- **Latency:** temporal_v6, naive_rag, and v6_no_verify all sit at ~2.1s (no LLM on read path); advanced_rag/v6/v6+infer at ~16-18s. The win is accuracy + temporal validity at RAG latency.
- **State-bounded growth:** RAG grows without limit; MemStrata retains distinct static facts (0% compression, preserving recall) but caps growth on evolving facts via supersession (~48% compression).
- **Calibration evidence:** Contradictions (mean cosine 0.812) are *more* similar to the original than duplicates (0.800); the maximum duplicate-accept precision at any threshold is 0.667.
- **Marker contamination is real:** Removing a textual [OUTDATED] marker from an earlier benchmark dropped reranker-RAG by 14 points while the temporal method moved only -4.

## Critical Assessment

- **Novelty (moderate):** The bi-temporal ledger is a well-established database primitive (Snodgrass & Ahn 1985; SQL:2011), and the (subject, relation, object) supersession idea is conceptually simple. The genuine novelty is (a) the empirical impossibility result showing similarity cannot detect staleness, and (b) the integration argument that a deterministic structural rule is necessary where learned/similarity approaches structurally fail. The impossibility result (AUROC 0.59) is the most defensible and reusable contribution.
- **Rigor (mixed):** Determinism and the marker-free invariant are methodological strengths. The use of distinct answer/judge models to prevent self-grading is good practice. However, **benchmark sizes are very small** (tens of items each: 50, 30, 30, 20, 20, 20), so individual numbers rest on few data points. The single 7B model means baselines may not reflect what stronger models achieve. The quarantined `fair_contradiction` benchmark (where temporal_v6 scores 0.62 vs advanced_rag 0.74) being excluded is honest, but it also reveals the method's dependence on clean single-value extraction.
- **Reproducibility (claimed high, unverified):** Strong stated commitments (deterministic, hashed prompts, released harness/datasets/tests). However, the paper is a draft from a single author with a product conflict; independent replication is not yet available, and the affiliation with a commercial product raises questions about whether the released artifact matches the paper's claims.
- **Significance (potentially high for the niche, narrow in scope):** The stale-fact framing is genuinely important for coding agents and operations assistants. But the claim is tightly scoped to structured single-value facts; the authors themselves concede that on messier natural-language contradictions extraction drops to ~44%, so the dominant result does not yet generalize to arbitrary prose.
- **Red flags:**
  - Single author with a direct commercial interest in the proposed system (MemStrata.dev).
  - "Draft v2" with a self-note to anonymize before submission — not peer-reviewed.
  - Very small benchmark sizes isolate mechanisms but cannot rank systems.
  - Ingestion order proxies time; real temporal benchmarks with explicit dates are future work, so the "temporal validity" claim is currently validated only on an ordering surrogate.
  - The headline "~0% stale-fact error" is achieved precisely because stale facts are removed from the store — this is almost definitional once extraction succeeds; the harder and under-evaluated problem is extraction robustness.
  - The 0.95-1.00 numbers on the four evolving benchmarks partially reflect that the benchmarks were constructed to be exactly the case the extractor handles (~97% supersession).

## Strengths

- Crisp, falsifiable central claim (similarity cannot separate contradiction from duplicate) backed by a calibration dataset.
- Clean ablation design bracketing the method between lossy-over-merge and no-supersession, isolating deterministic supersession as the causal factor.
- Methodologically careful evaluation hygiene: marker-free invariant enforced by tests, distinct answer/judge models, forced-answer supplement to expose abstention-hidden failures.
- Practical latency win — temporal validity at RAG latency, not at LLM-reranker latency.
- Honest limitations section that scopes the claim rather than overclaiming.

## Limitations

- Structured single-value templates only; extraction quality (not the supersession mechanism) is the gating factor for natural-language contradictions (~44% extraction on quarantined benchmark).
- Ingestion order proxies time; no explicit valid_from timestamps or as-of-T retrieval evaluated.
- Single-judge noise (3B correctness judge occasionally scores stale-containing answers as correct).
- Single 7B answer model on consumer hardware; larger/cloud models may shift baselines.
- Small benchmark sizes (tens of items) isolate mechanisms but limit statistical power and generalizability.
- Single author with commercial product affiliation; no independent peer review or replication yet.

## Relevance

This paper is **directly on-target** for research on agent reliability and stale-fact errors: it precisely formalizes why RAG-based agent memory serves superseded facts (a core reliability failure in coding/ops agents) and provides a deterministic, low-latency mitigation, along with a marker-free evaluation protocol that future agent-memory reliability work should adopt. The main caveat is that the reliability gains are currently validated only on structured single-value facts, leaving generalization to messy natural-language knowledge as open work.

## Questions for Further Exploration

- How does MemStrata behave when extraction is noisy at scale — does a 44% extraction rate on free prose cap real-world stale-fact-error reduction well above 0%, and can entity canonicalization / relation typing close that gap?
- Can the bi-temporal ledger's as-of-T retrieval mode (using real valid_from timestamps rather than ingestion order) be evaluated on genuinely time-stamped longitudinal data?
- Do the impossibility results (AUROC 0.59) hold for stronger embedding models and for longer-form contradictions, or is the near-chance separation an artifact of single-token value-flips?
- How does the method interact with larger answer models (e.g., 70B+ or frontier cloud models) that may have stronger implicit temporal reasoning — does RAG's stale-fact-error rate shrink, eroding MemStrata's relative advantage?
- Can the marker-free evaluation invariant be extended into a community benchmark suite (with hidden test sets and third-party graders) to mitigate the single-author/product-conflict concern?
- What is the failure mode when two genuinely distinct facts share a (subject, relation) key but differ in object for legitimate reasons (e.g., environment-specific config) — does deterministic supersession over-retire valid facts?
