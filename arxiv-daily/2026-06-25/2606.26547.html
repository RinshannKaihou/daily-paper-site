Key Contributions

  • ApproxHDC framework: Introduces a framework for automated identification and application of domain-specific approximations in HDC workloads, spanning both software and hardware levels, rather than relying on manual, hand-tuned approximations.
  • Compiler-level retargetability via HPVM-HDC: Extends the HPVM-HDC compiler infrastructure to support retargetable compilation across heterogeneous backends — CPUs, GPUs, and simulated ReRAM- and PCM-based in-memory accelerators — from a single source program.
  • Scalable navigation of the approximation space: Recognizes that the space of possible approximations is exponentially large and addresses it with efficient search and analysis to surface high-impact configurations.
  • Leveraging HDC's intrinsic noise/approximation tolerance: Explicitly converts HDC's algorithmic robustness-to-noise into a compiler-exploitable optimization opportunity, turning an inherent property of the model into measurable efficiency gains.
  • Bridging compilers and emerging in-memory hardware: Positions approximation tuning as a cross-layer concern that spans programming-language/compiler design (cs.PL) and performance (cs.PF), directly engaging ReRAM/PCM device-level approximations through a unified compiler abstraction.

Methodology

The work builds on the HPVM-HDC compiler infrastructure, a heterogeneous dataflow compiler (HPVM lineage from the Adve group) specialized for HDC. The methodology, as described in the abstract, has three pillars:

  1. Retargetable compilation: ApproxHDC extends HPVM-HDC so that a single HDC program can be lowered to multiple backends — CPUs, GPUs, and simulated in-memory accelerators built on Resistive RAM (ReRAM) and Phase-Change Memory (PCM) devices. The simulator-based approach allows device-level approximations (e.g., limited-precision writes, noisy analog compute) to be modeled at the compiler level.
  1. A unified approximation model: The framework treats approximations as configurable knobs that can be applied at both the software level (algorithmic choices, precision reduction, operation substitution) and the hardware level (device non-idealities, in-memory compute precision). The design rationale is rooted in HDC's cognitive-models-of-computation foundation, which makes its representations high-dimensional and hence intrinsically robust to perturbation.
  1. Search over an exponential space: Because the cross-product of software and hardware approximations grows exponentially, ApproxHDC employs efficient search and analysis (rather than exhaustive enumeration) to navigate this space and identify high-impact configurations — i.e., settings that yield large performance/energy gains for minimal accuracy loss. The abstract frames this as a domain-specific approximation-tuning problem rather than general-purpose approximate computing.
