> **一句话总结：** EvoOptiGraph 将每个混合整数线性规划（MILP）表示为带属性的二部图，通过保持有效性的遗传算子生成结构多样的训练实例，并用 XGBoost+SHAP 提取的"弱点向量"引导数据-模型协同进化闭环（SFT + RLVR），以 8B 模型在六个运筹优化基准上取得 78.1% 的宏平均 Pass@1，超越 GPT-4o、DeepSeek-V3 及 72B 等更大模型与专门的微调基线。

> **TL;DR:** EvoOptiGraph is a closed-loop framework that co-evolves training data and an LLM for natural-language-to-optimization modeling: it represents MILPs as attributed bipartite graphs, evolves structurally diverse instances via validity-preserving genetic operators, and uses a graph-derived "weakness vector" (XGBoost + SHAP) to target the model's failure modes in a two-stage SFT + RLVR loop, letting an 8B model reach 78.1% macro-average Pass@1 and beat larger generalists, agentic methods, and specialized fine-tuned baselines.

## 核心贡献

- **闭环数据-模型协同进化框架：** 将数据合成与模型训练显式耦合——模型在验证集上的失败模式被总结为"弱点向量"，反过来引导图演化生成专门针对模型盲区的新实例，使训练分布随模型能力自适应更新，而非静态冻结。
- **可控的图结构级数据生成器：** 直接对带属性二部图（变量节点 + 约束节点，边权重为系数）做交叉与三层变异（参数变异、结构变异、规模变异），突破传统模板参数扰动只能扩大规模、不能扩展结构广度的限制；每个后代图经确定性编译成 Gurobi 代码并回译为自然语言，三阶段校验保证质量。
- **弱点驱动的可验证强化学习：** SFT 暖启动后，用图结构特征训练 XGBoost 并以 SHAP 值量化模型失败关联的结构属性；以此为适应度偏置重做演化，配合分级 RLVR 奖励（可执行 0.2 + 可行 0.1 + 近似 0.2 + 完全正确 0.5），用 GRPO 优化策略。

## Key Contributions

- **Closed-loop, weakness-driven coevolution framework.** Rather than separating data synthesis from model optimization, EvoOptiGraph links them through graph-level weakness analysis: each round the model is evaluated on a held-out validation set, errors are summarized into a structural weakness vector `w`, and that vector biases the next round of graph evolution toward the model's actual failure regimes. This makes the training distribution adapt as the model changes.
- **Controllable graph-based generator with validity-preserving genetic operators.** Crossover exchanges compatible subgraphs (default within-class to preserve semantics, with some cross-class for diversity); mutation operates at three levels (parameter / structural topology / scale). Every offspring passes deterministic validity checks, is compiled to Gurobi code, and is back-translated to NL via three-stage verification (rule-based, independent-LLM, value-matching audits). The fitness function balances difficulty, diversity (graph-edit-distance), and weakness alignment: `f(G) = β·f̃_diff + γ·f̃_div + δ·f̃_weak`.
- **Weakness-driven RL with verifiable rewards.** After SFT warm-up, an XGBoost classifier predicts per-instance success from graph features ϕ(G); SHAP values on error cases yield the interpretable weakness vector. A multi-stage reward `R = R_process + R_solvable + R_proximity + R_correctness` (runnable code, feasibility, <10% relative error proximity, exact correctness) is optimized via GRPO.

## Methodology

EvoOptiGraph couples four components in a single loop:

1. **Attributed bipartite graph representation.** Each MILP is encoded as G = (V, E, A): constraint nodes `{s1..sm, s_obj}` (incl. an objective node) on one side, variable nodes `{x1..xn}` on the other; edge `(s_i, x_j)` exists when a variable has a nonzero coefficient, with the coefficient as edge weight. Node attributes store sense, RHS, bounds, variable type. A *valid* graph has exactly one objective node, complete metadata, and no isolated components, and deterministically recovers both the program P and its executable form V(P).

2. **Graph-based data generation.** Seeded from 53 expert-designed generators (OptMATH), the initial population is evolved with genetic operators under validity constraints. **Crossover** swaps compatible subgraphs (the objective node is handled separately so each offspring keeps exactly one objective). **Mutation** acts at three levels — parameter (coefficients/bounds/RHS), structural (insert/delete/modify variables or constraints preserving linearity), scale (resize problem). Each candidate passes a two-stage quality gate: structural validity then Gurobi compilation/execution (infeasible-but-IIS instances are retained as meaningful structures). Back-translation uses an LLM with three-stage verification.

3. **Two-stage coevolution loop (Algorithm 1).**
   - *Warm-up SFT:* standard next-token prediction on the initial graph-evolved dataset D0.
   - *Structured weakness analysis:* per round, the model is evaluated on a held-out validation set; for each graph a structural feature vector ϕ(G) (size stats, type proportions, sparsity/density, graph-topological/spectral features, coefficient stats) is computed. An XGBoost classifier predicts success from ϕ(G), and SHAP values on error cases yield `w_k = (1/|E|) Σ SHAP_{i,k}` — explicitly framed as a prioritization signal, not a causal claim.
   - *Weakness-driven RLVR:* the fitness term `f̃_weak = cosine(ϕ(G), w)` is activated (δ becomes nonzero), biasing evolution toward failure-resembling structures; the new dataset DRL trains the model with multi-stage verifiable rewards via GRPO (group-relative policy optimization).

## Results & Highlights

- **Macro-average Pass@1 = 78.1%** on six OR benchmarks (MAMO, NL4OPT, EasyLP, NLP4LP, ComplexOR, IndustryOR), the best among all compared methods — beating GPT-4o (68.0), DeepSeek-V3, Qwen3-32B, and Qwen2.5-72B-Instruct (zero-shot), OptiMUS-v0.3 / CoT / CoE (agentic), and ORLM / LLMOPT / OptMATH (fine-tuned).
- **Smaller beats larger:** an 8B Qwen3 backbone outperforms 72B generalists and 32B OptMATH, suggesting targeted structural data evolution outweighs raw scale for optimization code generation.
- **Structural diversity:** measured against GenBench-MILP structural similarity, EvoOptiGraph achieves 66.93% similarity vs. 93.04% for OptMATH over 50 resampled runs of 212 instances each — i.e., it modifies underlying problem *structure*, not just parameters.
- **Loop dynamics (Figure 3):** warm-up SFT gives the largest foundational gain; subsequent RL rounds add steady improvements on most benchmarks, uncovering new failure modes after SFT saturation. A non-monotonic dip on MAMO-complex at RL2 that recovers at RL3 suggests the loop self-corrects from temporary over-specialization.
- **Ablation:** vs. SFT-only, EvoOptiGraph improves average accuracy by 9.6%; vs. RL-only, by 14.7%. SFT-only saturates/overfits to recurring patterns; RL-only is unstable without a strong SFT initialization.

## Critical Assessment

- **Novelty (High-Moderate).** The individual pieces (genetic MILP generation, SFT+RLVR, SHAP weakness analysis, GRPO) are not new in isolation; the novelty is the *closed-loop integration* through the graph representation as a shared substrate for generation, diagnosis, and adaptation. G2MILP/MILP-StuDio already do graph-based MILP instance generation, and MILP-Evolve already does text-based evolution — so the framing as "coevolution driven by structural weakness" is the genuine contribution, but the components are recombinations rather than fundamental advances.
- **Rigor (Moderate-High).** Multi-category baselines, six benchmarks, ablations for SFT-only/RL-only, a diversity comparison with proper resampling (50×212), and an honest loop-trajectory analysis including a non-monotonic result that is explained rather than hidden. However, some baseline numbers are cited from different papers with different (uncleaned) datasets and explicitly flagged as "for reference only / may not be directly comparable," which weakens head-to-head comparability. There is also a "Table ??" reference in the ablation text (apparent cross-reference issue).
- **Reproducibility (Moderate).** Algorithm 1, the exact reward decomposition, fitness weights, and feature categories are specified; appendix defers full prompt/config details. The reliance on GPT-4/Claude-3/DeepSeek-V3 for back-translation and closed benchmark variants limits exact replication, and the seed-generator set is fixed at 53 from OptMATH. No explicit code-release commitment is stated in the main text.
- **Significance (Moderate-High for the niche).** Directly relevant to adaptive/self-improving LLM training pipelines for formal reasoning; the weakness-vector-as-fitness-gradient idea is transferable beyond MILP. Practical impact is bounded by the MILP-only scope.
- **Red flags (minor).** Cited baseline comparability caveats; the weakness vector is explicitly disclaimed as non-causal yet is used to steer data generation (acceptable as prioritization, but causal claims are appropriately avoided); verification cost scales poorly with instance size (acknowledged).

## Strengths

- Principled unification: one graph representation simultaneously enables generation, quality control, and failure diagnosis.
- Genuine closed loop with adaptive training distribution rather than a static dataset.
- Honest reporting: non-monotonic loop behavior is shown and discussed; ablations isolate SFT vs. RL contributions.
- Strong efficiency story: 8B beats 32B/72B models.
- Quantitative diversity evidence (GED/similarity) backing the structural-evolution claim.

## Limitations

- Restricted to MILPs; broader optimization classes (nonlinear, convex, stochastic) are out of scope.
- Coverage bounded by the 53 OptMATH seed generators — novelty is structural recombination within known families.
- Verification (Gurobi solves + IIS computation) becomes expensive for large instances; caching mitigates but does not eliminate this.
- Weakness vector is a prioritization heuristic, not a causal model of failure.
- Back-translation quality depends on closed proprietary LLMs (GPT-4/Claude-3/DeepSeek-V3), affecting reproducibility.
- Some baseline comparisons mix cleaned/uncleaned datasets, tempering strong cross-method claims.

## Questions for Further Exploration

- How does the weakness vector's transferability hold across different optimization families — does a `w` learned on transportation/knapsack seeds generalize to unseen problem classes, or does it overfit to seed-family structure?
- Could the closed loop be extended to nonlinear/mixed-integer nonlinear programs, and does the bipartite-graph representation remain expressive and validity-preservable under nonlinear constraints?
- What is the marginal value of additional coevolution rounds beyond RL3 — is there an asymptote or accumulating overfitting to synthetic artifacts?
- Would replacing the XGBoost+SHAP weakness model with an end-to-end differentiable failure predictor enable gradient-based rather than evolutionary steering of data generation?
- How sensitive are the 78.1% headline and per-benchmark gains to the proprietary back-translation LLM (GPT-4/Claude-3/DeepSeek-V3) — would an open-model back-translation pipeline close the reproducibility gap?
- Can the weakness-driven fitness be combined with curriculum ordering or active validation selection to reduce the expensive Gurobi verification budget?
