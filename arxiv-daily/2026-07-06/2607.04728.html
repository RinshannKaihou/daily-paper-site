How it works

Reframe off-policy correction as rejection sampling: the old policy is a proposal distribution for the current one.

At each token, run an accept–reject test. Accepted tokens provably follow the target policy πθ (Proposition 1) and get unit weight — no correction needed. Rejected tokens keep the standard importance ratio.

Unlike clipping, which suppresses gradient signal from off-policy tokens, SIS converts them — fixing the mismatch at its root. The full-vocab max is approximated over top-K, exploiting LLM probability concentration; it reuses existing logits with no extra forward pass.

For sequence-level GSPO, accepted tokens simply contribute zero log-ratio to the sequence ratio — a one-line substitution in any objective.

# accept-reject test, per token t Mt = maxv πθ(v) / πθ_old(v) zt ~ Bernoulli( wt / Mt )

# accepted → genuinely on-policy ŵt = 1 if zt=1 # unit weight ŵt = wt otherwise # standard IS

# the bound SIS tightens E ≤ |A|(eD−1) Σ‖st‖ with D = Σ|log wt|
DSIS = Σ (1−zt)|log wt| ≤ D

Verdict

Strengths

  • Conceptually clean: "convert off-policy to on-policy" preserves gradient signal — sharper than "clip the ratio."
  • Theoretically grounded with full proofs; SIS strictly tightens the token-vs-sequence error bound (DSIS ≤ D).
  • Genuinely plug-and-play — ~1% overhead, algorithm-agnostic across token-level (GRPO/DAPO) and sequence-level (GSPO).
  • Broad coverage: dense + MoE, two families, 10 benchmarks, three stress tests.
  • The MoE collapse fix (Fig. 4) and clipping-free win (Tab. 7) show SIS is a real stabilizer, not just an accuracy booster.

Limitations

  • No dedicated limitations section — failure near zero accept rate is uncharacterized.
  • Only rule-based outcome rewards; noisy / learned reward models (the real RLHF regime) untested.
  • Top-K approximation bias ξK is bounded theoretically but not reported as an empirical number.
  • Competing tricks (DPPO-TV, Clip-Cov) look weak in Tab. 2 — a fair-tuning concern, not a strictly controlled comparison.
  • Eval limited to math + agentic search; code, long-horizon tools, multilingual not covered.
04

Open questions

How does SIS behave when the accept rate → 0 under extreme drift (very stale async rollouts) — graceful degradation or a cliff?
Can the per-token accept indicator zt double as a built-in observability signal for off-policy-ness, beyond its role as a correction?
With a noisy reward model, does SIS amplify advantage-estimate errors for accepted tokens, since they receive full unit weight?
Could top-K be replaced by an adaptive K chosen per token from the residual mass ξK, to bound bias without manual tuning?
Does accept/reject interact with quantization noise in low-precision training, where πθ vs πθ_old logits already carry rounding error?
Does SIS transfer to preference-based objectives (DPO-family), where the importance ratio is implicit, and to much longer agent sequences?
