Compressed Computation under L4 Loss is likely Computation in Superposition

Francisco Ferreira da Silva · Stefan Heimersheim  —  toy-models · reverse-engineering · loss exponent

computation-in-superposition superposition L4-loss sparse-coding ReLU-networks reverse-engineering

Key Finding

Swapping L2 for L4 loss makes 50 neurons compute the ReLU of 100 sparse features via computation in superposition.

Single change flips the qualitative solution Fully reverse-engineered: binary code + pseudoinverse First CiS to emerge through training

01 Key Metric

0.031
per-feature error CV · L4 0.999 · L2 (naive)

A ~32× more even spread of error across all 100 features — the signature that all features are handled nearly identically.

02 Why This Matters

A fully reverse-engineered superposition toy model — exactly the ground-truth testbed mechanistic interpretability urgently needs — and loss choice as an underappreciated knob.

03 Critical Edge

CiS appears only at extreme sparsity (p≈0.02) where interference is minimal by construction, and weakens sharply under tighter embeddings (CV 0.20 vs 0.03).

likely — not proven
01

Methodology

The compressed-computation toy model of Braun et al. (2025), with two modifications motivated by Bhagat et al. (2025): the residual connection is removed (target is exactly y = ReLU(x), so any gain over the naive "ignore half the features" baseline must come from superposition) and the loss is switched from L2 to L4.

  • ATask. 100-dim input; each feature zero w.p. 1−p, else uniform on (−1,1), p=0.02 (~2 active). Single hidden layer of 50 ReLU neurons, no bias; W_in ∈ R^{50×100}, W_out ∈ R^{100×50}; compute elementwise ReLU.
  • BTraining. L4 loss (mean 4th power of error); Adam, lr 0.01 + cosine annealing, batch 8,192, 100k steps. Intuition: L4 over-penalizes outliers, forcing all features to be approximated roughly equally.
  • CReverse-engineering. (1) inspect encoder distribution → bimodal on/off code; (2) extract binary codeword matrix; (3) swap test (transplant feature's hidden activations onto another's codeword); (4) compare decoder to a scaled pseudoinverse; (5) validate with a 3-scalar ansatz + hand-designed codes.
02

Results & Highlights

0.031 vs 0.999

L4 elicits superposition where L2 fails. Every exponent >2 (2.5/3/4/6/8) gives CV 0.028–0.046; the strongest non-superposition baseline sits ~26× higher in L4 loss.

1.13×

3 scalars recover most performance. Codewords + (on-value, off-value, decoder scale) reach ~1.13× trained loss, vs naive ~28×, emulate-bias ~26×, random ~56×.

0.996

Decoder ≈ scaled pseudoinverse. Encoder splits on-code +0.34±0.06 / off-code −0.02±0.01; swap test passes 100% — codeword position, not magnitude, identifies the feature.

K ≈ 5.5

Near-biregular sparse codes. Each feature uses 5–7 neurons; each neuron sits in 10–12 codewords. A hand-designed K=5 code (overlap-minimizing edge swaps) hits ~1.12×, marginally beating the trained 3-scalar version.

slope 0.81

Output attenuation as interference control. The net under-shoots active ReLU outputs to cut leaked cross-talk onto the many inactive features; 5–95% band across features is invisible — every feature treated identically.

< 1%

Robust to embedding & scale. Adding Braun's 1000-dim embedding shifts all headline quantities <1%; at F=1000, N=500 CV stays low (0.06); pseudoinverse vs transpose unembedding match within 1%.

03

Strengths

Exceptionally clean design

  • A single well-motivated change (loss exponent) flips the qualitative solution — conclusion buttressed by ~6 independent cross-checks, not one headline number.
  • Unusually complete reverse-engineering for an MI paper: identifies mechanism (binary code + pseudoinverse), validates it (swap test, cosine, ansatz), honestly quantifies the residual.
  • Intellectual honesty throughout: concurrent work (Gibson 2026) and prior null results (Bhagat 2025) credited; title hedges "likely" vs "proven."
  • The loss-exponent sweep is a cheap, broadly applicable diagnostic other CiS/representation researchers can adopt immediately.
  • Public code + trivially reproducible toy scale (50 neurons, 100 features) lower the barrier to follow-up.
04

Limitations

Where it stops short

  • Toy-scale only: single hidden layer, elementwise ReLU, extreme sparsity p=0.02 — no deeper nets, realistic architectures, or richer nonlinearities.
  • ~13% unexplained (1.13× vs 1.00×): specifically non-uniform per-entry on-code values (degree→value r≈−0.82 explains only ~67% of variance) and the decoder's residual deviation.
  • Fragility signals: CV degrades to 0.20 at d=50 vs 0.03 at d=1000 — sensitivity to compression/embedding geometry is not characterized.
  • "Likely," not "proven": the case is circumstantial (even error spread + reverse-engineered shared-neuron mechanism), not a formal proof.
  • Single loss family: only power-law losses swept; whether the exponent→2 threshold generalizes (Huber, quantile, log-cosh) is untested.
  • No downstream application: foundational/mechanistic; doesn't tie to better training or practical tools beyond naming APD/SPD as future testbeds.
05

Questions for Further Exploration

  1. Does the exponent→2 threshold generalize beyond power-law losses — do Huber, quantile, or log-cosh also elicit CiS, and is there a unifying property (convexity, tail-weight) that predicts it?
  2. Can the sparse-binary-code + pseudoinverse mechanism survive deeper networks (2–3 layers, multiple bottlenecks)? Do Hänni et al. (2024)-style error-correction layers emerge as interference compounds?
  3. The unexplained ~13% gap concentrates in non-uniform per-entry encoder values — what do they encode, and is the degree→value correlation (r≈−0.82) a load-balancing heuristic designable in directly?
  4. How does the phenomenon depend on sparsity p and compression ratio F/N? The d=50 degradation (0.20 vs 0.03) hints at a phase boundary — where, and does CiS vanish in denser real-LLM regimes?
  5. Can this network serve as a calibrated ground-truth testbed for APD/SPD — do those methods actually recover the binary-code mechanism, and where do they fail?
  6. Is the output attenuation (slope ≈0.81) a generic signature of interference-limited CiS detectable in real LLM activations as a diagnostic for superposed computation?
