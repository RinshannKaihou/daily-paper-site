Latent Programming Horizons in Coding Agents

A coding agent's residual stream linearly encodes the program it is editing — and predicts an edit's outcome before it happens.

一句话总结：在 22,714 条真实编码 agent 轨迹的残差流上训练线性探针，可解码程序正确性（最高 AUC 0.83），并能在编辑落地前约 25 步预测其结果，首次揭示编码 agent 的"潜在编程视野"。

Inside the agent's residual stream, program correctness is linearly decodable — and a future edit's fate is readable ~25 steps early.

How it works

Two open-weight agents edit real repositories under mini-swe-agent v2.2.8; the authors snapshot the full program version after every codebase-modifying tool call and dump the residual stream throughout.

Four symbolically-computed binary properties are attached to each edit event and propagated forward until the next edit. A per-layer logistic-regression probe (capacity-limited, per Hewitt & Liang) then tries to recover them from a single hidden state — split by task identifier to forbid leakage, with a shuffled-label control to rule out probe-capacity artifacts.

The programming horizon is a second probe predicting the property label k steps ahead, sweeping k from 0→50 with the final 50 steps held out so the token population stays constant. Cross-dataset transfer applies a probe trained on one benchmark directly to the other, with no retraining.

Verdict

Strengths

  • First-of-kind evidence that program representations persist and update across dozens of agentic steps on partially-observed real codebases — not single-shot generation.
  • Methodologically clean: capacity-limited linear probes, task-level splits, shuffled-label controls collapsing to 0.50, 20-trial hyperparameter sweeps, Brier/AUC cross-checks.
  • Large, realistic data — 22,714 trajectories, 1,231 real GitHub tasks, 22.4M hidden-state vectors — with full code + dataset release.
  • Cross-model & cross-benchmark consistency (inverted-U layer pattern replicates across both) strengthens external validity of the core claim.

Limitations

  • Correlational, not causal — no steering/intervention experiment confirms the agent actually relies on the decoded representation to decide edits.
  • Well-formedness is broken on SWE-Bench-Verified (AUC <0.60) due to >92% positive-label imbalance — one of four headline properties is non-informative on half the experiments.
  • Narrow scope: one harness (mini-swe-agent), two related open-weight models — frontier closed models and OpenHands/CodeAct scaffolds are untested.
  • The "25-step horizon" sits only ~0.02–0.15 AUC above chance at long range, so its practical decision-making value is uncertain.
  • No trivial-predictor baseline for lookahead (e.g. last-seen-label), so label autocorrelation can't be fully factored out — and median trajectory has just 2 edits.
04

Open questions

Does causal steering along the full-correctness probe direction actually change the agent's subsequent edits — make it write more correct code — or is the representation epiphenomenal?
How much of the lookahead survives subtracting a trivial last-label / majority-class predictor — is the horizon genuine planning or label autocorrelation?
Do these latent representations appear under different scaffolds (OpenHands, CodeAct, AutoCodeRover) and in frontier closed models, or are they specific to mini-swe-agent + these two models?
Can the probes serve as an inference-time observability / SDC signal to abort or re-prompt trajectories predicted to introduce regressions — and at what precision/recall?
Is the inverted-U layer pattern stable across scale, and does the "best" mid-layer shift predictably with depth?
How does horizon length scale with task difficulty, trajectory length, or model size — and can it be lengthened via training (e.g. RL with execution feedback)?
