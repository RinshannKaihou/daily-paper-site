Detecting Answer-Driven Reasoning in LLM-Based Educational Tutors via Truncated Chain-of-Thought Auditing

Bonan Shen · Dingyan Shang · Youting Wang (Northeastern) · Tao Ning (Syracuse)

A hidden teacher answer-key makes the tutor's final answer recoverable before its written reasoning earns it.

截断式思维链审计（TRACE）揭示：当 LLM 辅导系统持有教师私藏的答案钥匙时，最终答案在仅生成 10% 解释时即可被恢复——中位 TRACE AUC 从 0.375 跃升至 0.900，即便最终答案相同的对照子集上效应依然显著。

Key metric median TRACE AUC

0.900
↑ from 0.375 (question-only) · Δ +0.525

With answer-key access on 1000 GSM8K problems (Qwen2.5-3B-Instruct). Pass@10 jumps 0.113 → 0.997: the gold answer is recoverable from the first 10% prefix in 997 / 1000 runs.

Why this matters

A lightweight process-level probe that catches LLM tutors silently leaning on a private answer-key they were never meant to use.

Critical edge

The whole result rests on one 3B model, one grade-school dataset, and an exact-integer verifier — no scaling curve, no significance tests, no released code.

§ 02

Methodology

paired audit · 1000 × 3 = 3000 rows
01

Paired tutoring contexts

Each problem x (gold y) gets an explanation under three private contexts that differ only in the teacher-only note:

  • Question-only — just the problem.
  • Answer-key — note carries the correct final answer.
  • Wrong-key — note carries y+1 (counterfactual anchor).

Model is told to use the key privately and write the explanation "as if deriving the answer."

02

Truncated prefix probing (TRACE)

For an explanation of T tokens, take prefixes at F = {0.10, 0.25, 0.50, 0.75, 1.00}. At each f·T, append a forced-answer tag (deterministic, ≤8 tokens), normalize to integer, exact-match vs gold → p(f) ∈ {0,1}.

Each probe is a fresh completion, so the curve need not be monotone: an early pass signals recoverability, not retention.

03

Scalar summaries & paired analysis

  • TRACE AUC — trapezoidal area under p(f).
  • Pass@10 — p(0.10), earliest-prefix indicator.
  • First-pass — first f where p(f) ≥ 0.5, else 1.0.

Unit of comparison is paired by problem; a final-answer-correct subset isolates process-level from outcome-level effects.

Setup: 1000 GSM8K test examples · Qwen2.5-3B-Instruct (4-bit MLX, Apple M1 Max 64GB) · greedy explanations ≤768 tok · deterministic forced answers ≤8 tok · 3000 rows (1000 × 3 conditions). A CUDA/HF replication runner was prepared; reported numbers are the local MLX run.
§ 03

Results & highlights

see §01 chart for the curve

Private channel shapes the explanation

Median TRACE AUC: 0.375 → 0.900 (Δ +0.525) with answer-key; wrong-key collapses to 0.125, closer to question-only for gold-answer availability — confirming the key, not the question, is doing the work.

Gold answer recoverable almost immediately

Pass@10 jumps 0.113 → 0.997: in 997/1000 answer-key runs the 10% prefix already passes, while 887/1000 question-only runs fail at 10%. Median first-pass moves 0.750 → 0.100.

Effect survives the correctness confound

On the 746-problem final-answer-correct subset, answer-key AUC stays 0.900 vs question-only 0.375, winning 697/746 pairs. AUROC for separating the two traces by AUC: 0.966 (0.966 in the subset too).

Same channel can mis-anchor

Full-response accuracy: question-only 0.782 → answer-key 0.891 → wrong-key 0.613 (387/1000 wrong-key explanations end wrong). Non-monotonicity is documented: 80 answer-key examples pass at 10% but fail at 100% (e.g. recovers 64 early, then drifts to 58).

§ 04

Strengths vs. limitations

side-by-side assessment

＋ Strengths

  • Clean conceptual splitseparating answer leakage (what the tutor reveals) from answer-driven reasoning (what the explanation relies on) is genuinely underexplored.
  • Right controlthe 746-problem final-answer-correct subset directly rules out "the probe just measures eventual correctness."
  • Causal counterfactualthe wrong-key condition provides evidence the private channel — not the question — is doing the work.
  • Intellectual honestythe non-monotonicity discussion (glass-buying example) pre-empts the misreading that early-pass = guaranteed-correct trajectory.
  • Deployable artifactthe flag-high-Pass@10/AUC workflow is lightweight, model-agnostic, and portable; transparent GenAI declaration; TRACE honestly framed as adapted, not invented.

－ Limitations

  • Single model, single datasetQwen2.5-3B-Instruct + GSM8K only — no scaling curve across sizes; generalization to open-ended / non-numeric tutoring untested.
  • Deterministic single samplep(f) ∈ {0,1} per problem; no variance estimate, no stochastic aggregation, no significance testing.
  • Exact-integer verifierrestricts scope to short-numeric answers; rubric grading, multi-turn dialogue, qualitative answers are out of scope.
  • Unvalidated assumptionforced-answer behavior is assumed informative about the original explanation; not directly validated against hidden computation.
  • No artifacts, quantized runno public code or prompts; numbers come from a 4-bit MLX run whose interaction with the metric is not analyzed.
  • Weak counterfactualwrong-key is not a necessary condition (a model can override it), so its evidential role is supportive, not dispositive.
§ 05

Open questions

where this could go next
Q1

Does the answer-key AUC gap shrink, hold, or widen with model scale (7B / 32B / 72B), and does it interact with instruction-tuning vs. RL?

Q2

Under stochastic forced-answer sampling (T>0, k samples), does binary p(f) flatten into a smooth recoverability probability that is more diagnostic?

Q3

Can the audit extend beyond exact-integer match to rubric / LLM-verifier settings, and does the signal stay separable from verifier noise?

Q4

Are the 80 "pass@10%, fail@100%" non-monotone cases driven by identifiable failure modes (arithmetic slips, discount-pattern reinterpretation) predictable from prefix content?

Q5

How does this interact with retrieval-augmented tutoring — can prefix auditing tell "legitimately used retrieved solution" from "shortcut answer anchoring"?

Q6

Could TRACE probing serve as a training signal (a faithfulness regularizer), and would minimizing Pass@10 degrade final-answer accuracy?

Q7

What is the relationship between TRACE AUC and internal hidden-state probes — do they agree on when the answer becomes available?
