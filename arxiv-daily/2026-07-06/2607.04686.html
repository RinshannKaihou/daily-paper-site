Key finding — the one sentence to take away

Even the strongest agent gets tool use right only 86.33% of the time — and two models with identical aggregate scores can fail for completely different reasons.

Why this matters

Turns "does the model trust an external result over its own memory?" into an observable answer-text signal — exposing post-call faithfulness as a silent failure mode.

Critical edge

No sensitivity study to the trap-value gap exists — yet the trap's magnitude is the benchmark's entire mechanism, so Result-Ignore's true driver stays unknown.

03Methodology

A parametric-trap design: the mock tool return deliberately contradicts a value the model likely memorized (e.g. a stock returns $247.83 where memory suggests ~$180). A faithful model reports the return; an unfaithful one substitutes the prior.

  • Two-step inference. First tool_choice=auto; if a valid call fires, the harness injects the controlled return, then the model answers with tool_choice=none. This cleanly separates the decision step from the answer-writing step.
  • Controlled returns, not real APIs. Finance ships get_quote, get_filings; cybersecurity ships CVE/exploit tools — nine domain tools, one role-specific system prompt, no model-specific tuning.
  • Three-way labeling ensemble. A deterministic rule classifier (audit trail) + Qwen3.5-397B decision-tree judge + GLM-4.7 evidence-attribution judge → unweighted majority vote, three-way ties broken by the rule classifier.
  • Fixed decoding. temperature = 0, max_tokens = 1024, seed = 42 where supported; closed APIs via providers, open-weight via vLLM on H100/H200, judging on B200.
The six metrics

Required (750): Tool-Skip Rate (TSR), Clean Tool-Use Rate (CTUR), Result-Ignore Rate (RIR), Output-Fabrication Rate (OFR) — these sum to 100%. Control (250): Unnecessary-Tool-Use Rate (UTUR), Control Accuracy (CTRL-Acc). Ensemble revises the rule label in 8.4% of judgments; genuine three-way ties hit the rule tie-breaker in only 1.5% (285 of 18,900). Three-way Fleiss' κ = 0.693.

  • Scale ≠ discipline. At matched ~70B, Llama-3.1-70B vs Qwen2.5-72B differ by 89 points on control accuracy (p < 10⁻⁸⁰), with Llama showing a 77.73% Unnecessary-Tool-Use Rate — an "Always-Call" pathology that scaling 8B→70B does not cure.
  • Finance is hardest for faithfulness. Stronger plausible priors (stock price, P/E, market cap) compete harder with tool returns — median RIR 12.24% vs Cybersecurity's 0.68%.
  • Top of the board is crowded. Grok-4.3 (86.33%), Grok-4-1-Fast-Reasoning (84.11%), Qwen2.5-32B-Instruct (82.68%); top two not statistically separable (p = 0.23).
  • Ensemble beats rules alone and stays unbiased: the Qwen judge scores Qwen-family models ~6 pp lower than the GLM judge — the opposite of a leniency bias.
  • Not a greedy-decoding artifact. Rerunning QwQ-32B at non-zero temperature moved rule CTUR 75.47%→75.60%, with degenerate repetition dropping 34→22 of 1000 traces.

06Strengths & limitations

Strengths

+ net positive
  • S1Mutually exclusive, exhaustive taxonomy. Required metrics sum to 100%, so any leaderboard row is arithmetically checkable.
  • S2Parametric trap is a clean trick. Forcing the return to contradict a plausible prior makes "did it trust the tool?" directly observable in the answer — no internal-state access needed.
  • S3Paired control tasks cleanly separate "always-calls" pathology from "can't use the return" pathology — a distinction no prior benchmark isolates.
  • S4Principled labeling — two differently-prompted judges + deterministic rule tie-breaker, with a same-family bias check and stratified κ.
  • S5Honest about limits — single-turn, 5-domain, judge blind-spots, and post-release contamination risk all enumerated in §6.

Limitations

⚠ bounds validity
  • L1Single-turn only. No multi-turn chaining, recovery, or state updates — yet deployed agents fail most often across turns, not within one.
  • L2No trap-gap sensitivity study. The contradiction's magnitude is never varied — so Result-Ignore's driver (small-gap confusion vs large-gap refusal) is unknown.
  • L3Judge dependency. Headline labels rely on two specific judges that may share blind spots; UTU Fleiss κ = 0.23 despite high raw agreement.
  • L4Model-version snapshot. 2026 closed-API checkpoints — rankings will not transfer and cannot be independently re-run.
  • L5Prompt-push confound. The shared system prompt aggressively pushes tool use, which may inflate UTR globally — plausibly part of Llama's Always-Call behavior.
  • L6Harness/parser edge cases. glm-4-9b & mistral-7b emitted tool-style text with zero executed calls — the parser may conflate formatting failures with genuine tool-use failures.

07Open questions

  1. How does Result-Ignore Rate change with the magnitude of the contradiction (trap-gap sweep)? Is there a threshold above which models reliably switch to the tool value?
  2. Does the Llama-3.1 "Always-Call" pathology survive a neutral system prompt, or is it partly an artifact of the current "tool is ground truth" emphasis?
  3. Can the four failure modes be predicted from hidden-state probes — Tool-Skip from pre-call activations, Result-Ignore from post-return activations before the answer is written?
  4. How do failure-mode rates correlate within a family across scales, and is there a scaling-law signature for "tool discipline" distinct from "tool capability"?
  5. Do the parametric-trap modes transfer to multi-turn settings — does a turn-1 Result-Ignore also fail on a re-appearing return in turn 3?
  6. For computation-reliability: if the return held a numerically corrupted but plausible value (off-by-one-ULP), would models detect it or faithfully propagate the corruption?
