Atropos: Improving Cost-Benefit Trade-off of LLM-based Agents under Self-Consistency with Early Termination and Model Hotswap

arXiv: 2604.15075

Date: 2026-04-16

https://arxiv.org/abs/2604.15075

一句话总结： ATROPOS 通过将 LLM Agent 的多轮推理轨迹构建为语义流图（SFG），利用 GCN 在推理中段预测最终成败（准确率达 85%），对预测失败的任务进行模型热替换（从小模型切换到大模型），从而以仅 23.9% 的成本获得闭源大模型 74.35% 的性能。
TL;DR: ATROPOS is a predictive early-termination and LLM hotswapping technique for agentic systems using self-consistency. It constructs Semantic Flow Graphs (SFGs) from ongoing multi-step agent trajectories and uses a GCN to predict whether the inference will succeed or fail. At the midpoint of inference, it achieves 85% prediction accuracy. When failure is predicted, it hotswaps the running context from a cheap local SLM to a more powerful proprietary LLM, recovering up to 27.57% of would-be failures. The net result: 74.35% of closed-LLM performance at only 23.9% of the cost, evaluated across three SE agents (AutoFL, AutoCodeRover, RepairAgent).

核心贡献

  • 提出了基于语义流图（Semantic Flow Graph）的推理正确性预测方法，将多个自一致性推理轨迹聚合为图结构，利用 GCN 在推理进行到一半时预测最终成败，准确率最高达 85%。
  • 设计了模型热替换（Model Hotswap）机制：当预测当前推理将失败时，将已执行的推理上下文无缝迁移到更强的 LLM 上继续执行，而非从头重新开始，成功挽救最多 27.57% 的失败推理。
  • 在三个软件工程 Agent（AutoFL、AutoCodeRover、RepairAgent）上进行了系统评估，验证了方法在不同任务类型（故障定位、程序修复）和不同模型对（Llama-3/GPT-4o、Mixtral/GPT-4、GPT-3.5/GPT-4o）上的泛化能力。
  • 通过消融实验证明了 SFG 中语义嵌入（FastText）、函数类型和参数信息各自对预测性能的贡献。

Key Contributions

  • Introduces a GCN-based correctness predictor operating on Semantic Flow Graphs constructed from partial agent trajectories under self-consistency, achieving up to 85% accuracy at the inference midpoint.
  • Proposes model hotswapping -- seamlessly migrating ongoing inference context from a weaker SLM to a stronger LLM without re-execution -- which converts up to 27.57% of predicted failures into successes.
  • Demonstrates a compelling cost-benefit trade-off: 74.35% of closed-LLM performance at 23.9% of the monetary cost across three diverse SE agents (AutoFL, AutoCodeRover, RepairAgent).
  • Provides ablation evidence that semantic embeddings, function types, and argument information in the SFG all contribute to prediction quality, with semantic information being the most important component.

Methodology

ATROPOS operates in three stages:

  1. Graph Construction: Multiple agent trajectories from self-consistency sampling are merged into a Semantic Flow Graph (SFG). Nodes represent unique reasoning steps (tool invocations), encoded via one-hot function vectors concatenated with 300-dimensional FastText embeddings of arguments. Edges represent sequential transitions, weighted by frequency across trajectories. For agents with unstructured arguments (e.g., RepairAgent), an incremental cosine-similarity clustering strategy (threshold 0.99) groups semantically similar steps.
  1. Correctness Prediction: A 3-layer GCN with global mean pooling performs binary graph-level classification on SFGs constructed from truncated (partial) trajectories. Training data is generated by running full inferences, then truncating at various steps k. Two truncation modes are supported: parallel (all R trajectories truncated after step k) and sequential (only the first k complete trajectories retained).
  1. Model Hotswapping: When failure is predicted at step k, the ongoing inference context is migrated to a more capable target LLM. For parallel hotswap, contexts from the first k-1 steps are replayed on the target model. For sequential hotswap, the remaining R-k+1 trajectories are generated on the target model. This is feasible because LLM queries are stateless.

The evaluation spans three agents: AutoFL (fault localization, Llama-3 -> GPT-4o), AutoCodeRover (fault localization, Mixtral-8x7B -> GPT-4), and RepairAgent (program repair, GPT-3.5-turbo -> GPT-4o). Benchmarks include Defects4J, BugsInPy, and SWE-bench. 5-fold cross-validation is used throughout.

Results & Highlights

  • RQ1 (Prediction Accuracy): On full trajectories, ATROPOS outperforms majority-class, confidence-based, and prior Lachesis baselines in most configurations. Best accuracy: 0.93 (AutoCodeRover/GPT-4, LC(1)). AUROC ranges from 0.85 to 0.93 across agents.
  • RQ2 (Early Termination): At the midpoint (k = N/2), AutoCodeRover achieves 0.85 accuracy and 0.85 AUROC in parallel mode. Sequential mode maintains high accuracy even at k=1, making it viable for budget-constrained scenarios.
  • RQ3 (Hotswap): Parallel hotswap at midpoint: AutoFL recovers 74.35% of GPT-4o's performance at 23.9% cost; RepairAgent achieves 78.92% at 64.73% cost. Sequential hotswap for AutoCodeRover at k=1-2 actually exceeds target-only performance due to beneficial model diversity in voting.
  • RQ4 (Ablation): Removing semantic embeddings causes the largest performance drop. Function types contribute more for agents with diverse tool sets (AutoCodeRover), while arguments carry more information for simpler tool sets (AutoFL).

Critical Assessment

Novelty: The idea of predicting inference correctness mid-execution using graph representations of agent trajectories is genuinely novel. The hotswapping mechanism is a clever exploitation of LLM statelessness. However, the approach builds directly on the authors' own prior work (Lachesis), extending it from post-hoc prediction to mid-inference prediction and adding hotswapping. The incremental contribution over Lachesis is meaningful but not revolutionary.

Rigor: The evaluation is thorough: three diverse agents, multiple model pairs, parallel and sequential strategies, ablation study, and analysis of outcome categories (retained success, rescued failure, destructive swap, etc.). 5-fold cross-validation is used. However, the GCN architecture is simple (3 layers, hidden dim 32) and its design choices are not extensively justified. The threshold for cosine-similarity clustering (0.99) is set empirically without sensitivity analysis.

Reproducibility: Code and data are available at an anonymous OpenScience repository. The paper provides sufficient implementation details (optimizer, learning rate, batch size, dropout). However, the RepairAgent evaluation involves extrapolation from R=1 to R=10 for the target model, which weakens reproducibility for that particular result.

Significance: The cost-benefit trade-off results are practically significant for practitioners deploying LLM-based agents at scale. The technique is model-agnostic and agent-agnostic in principle. However, the requirement for self-consistency (multiple samples) limits applicability to scenarios where such redundancy is already used.

Red Flags:

  • The AutoFL results with Llama-3 (the smallest model) show that confidence baseline outperforms ATROPOS, suggesting the method may struggle with very weak models that produce noisy trajectories.
  • RepairAgent's target-model performance is estimated rather than directly measured due to cost constraints, which is a notable limitation.
  • The paper does not report latency overhead of the GCN prediction step itself or the context replay during hotswap.
  • The GCN is trained per-agent per-model; transferability of trained predictors across agents or models is not explored.

Strengths

  • Addresses a practically important and underexplored problem: cost optimization for agentic LLM systems.
  • Elegant exploitation of LLM statelessness for hotswapping, avoiding the need to re-execute from scratch.
  • Comprehensive evaluation across three agents, multiple models, and two truncation/hotswap strategies.
  • The finding that sequential hotswap can exceed target-only performance (model diversity benefit) is a surprising and valuable insight.
  • Clear presentation with well-structured research questions and thorough result analysis.

Limitations

  • Requires self-consistency (multiple samples) as a prerequisite; not applicable to single-shot agent inference.
  • GCN predictor needs to be trained separately for each agent-model pair; no evidence of cross-agent or cross-model transfer.
  • Performance degrades with very small/weak models (e.g., Llama-3-8B on AutoFL) due to noisy, homogeneous trajectories.
  • RepairAgent evaluation involves significant estimation/extrapolation rather than direct measurement.
  • Does not account for the latency cost of GCN prediction or context replay during hotswap.
  • The approach is evaluated only on software engineering tasks (fault localization, program repair); generalization to other agentic domains (e.g., web navigation, data analysis) is untested.
  • The parallel hotswap evaluation is restricted to a single midpoint (k = N/2) due to cost constraints.

Relevance

本文对 LLM Agent 的成本优化具有直接参考价值，提出的热替换机制为在实际部署中平衡性能与开销提供了实用方案，尤其适用于需要在自一致性框架下多次调用 LLM 的软件工程 Agent 场景。

Questions for Further Exploration

  • Can the GCN predictor trained on one agent-model pair transfer to another agent or model without retraining, and if so, how much performance is lost?
  • How does ATROPOS perform on non-SE agentic tasks (e.g., web browsing, data analysis, multi-modal reasoning)?
  • What is the latency overhead of the GCN prediction and context replay, and how does it compare to the token generation time saved by early termination?
  • Could the prediction model be improved by incorporating LLM internal states (e.g., hidden layer activations or attention patterns) rather than relying solely on behavioral traces?
  • Is there an optimal adaptive strategy for choosing the hotswap point k dynamically based on trajectory characteristics rather than using a fixed midpoint?
