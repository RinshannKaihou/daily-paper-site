> **一句话总结：** ConfLayers提出了一种基于置信度的自适应层跳过方法来构建自推测解码的草稿模型，通过计算每层中间表示的熵来量化置信度，利用自适应窗口阈值机制决定跳过哪些层，在LLaMa系列模型上实现了最高1.4倍的推理加速，且无需任何额外训练。

> **TL;DR:** ConfLayers is a training-free, plug-and-play framework for self-speculative decoding that builds draft models by adaptively skipping transformer layers based on per-layer confidence scores (derived from entropy of intermediate hidden states projected through the LM head). An iterative search with adaptive windowed thresholds selects which layers to skip, achieving up to 1.4x speedup over vanilla autoregressive generation across LLaMa-2/3 models (8B-70B) and specialized models (CodeLLaMa-34B, Qwen2.5-Math-72B), consistently outperforming prior plug-and-play baselines SWIFT and DEL. Code is available at https://github.com/WA225/ConfLayers.

## 核心贡献

- 提出了一种基于置信度的层跳过机制：通过将每层FFNN和Attention模块的隐藏状态投影到LM head计算logits，再利用Shannon熵的归一化补数作为该层的置信度分数，从而量化每层对预测确定性的贡献。
- 设计了自适应窗口阈值算法：根据置信度二阶梯度和层深度的乘积动态调整邻域窗口大小，用局部均值和标准差作为阈值判断是否跳过该层，避免了固定窗口大小在不同模型间的不一致性。
- 提出间隔式周期搜索策略：在推理过程中在线执行层集搜索，以接收token数为优化目标（而非接受率），当平均接收token数达到阈值时终止搜索，避免了额外的搜索开销。
- 在多个LLaMa模型和专用模型上验证了方法的有效性，无需任何微调或额外训练即可实现推理加速。

## Key Contributions

- A **confidence-based layer skipping mechanism** that quantifies each transformer layer's contribution to predictive certainty by projecting FFNN and Attention hidden states through the shared LM head and computing normalized entropy complement as confidence scores.
- An **adaptive windowed thresholding algorithm** that dynamically adjusts the neighborhood window size based on the product of second-order confidence gradients and layer depth, using local mean and standard deviation as thresholds for skip decisions -- this avoids the inconsistency of fixed window sizes across different models.
- An **interval-based periodic search strategy** that performs online layer-set optimization during inference, maximizing accepted token count (rather than acceptance rate) as the objective, and halting search when average accepted tokens meet a target threshold.
- Comprehensive evaluation across LLaMa-2 (13B, 70B), LLaMa-3 (8B, 70B), CodeLLaMa-34B, and Qwen2.5-Math-72B, demonstrating up to 1.4x speedup without any fine-tuning or training.

## Methodology

ConfLayers operates within the self-speculative decoding (SSD) paradigm, where the same LLM serves as both the draft model (using a subset of layers) and the verification model (using all layers). The framework has three core components:

**1. Confidence Computation (Section 3.1):** For each transformer layer, the hidden states from both the FFNN and Attention modules are projected through the shared LM head to produce layer-wise logits. These logits are converted to probability distributions via softmax, and Shannon entropy is computed. The confidence score is defined as `1 - H(p)/H_max`, mapping entropy to [0,1] where 1 indicates high confidence and 0 indicates uncertainty.

**2. Adaptive Confidence-based Window Sizing (Section 3.2):** Confidence values are standardized (z-score normalization), and the second-order discrete gradient is computed to capture local fluctuation intensity. For each layer `i`, the window size is: `w_i = w_base + w_max * g_i * (l_i / num_layers)`, where `g_i` is the normalized gradient, `l_i` is the layer index, and `w_base` is set to 2. This means early layers with low gradients get small windows (fine-grained local comparison), while later layers with high gradients get larger windows (broader contextual aggregation). A layer is marked for skipping if its z-scored confidence falls below the local mean minus `lambda * local_std`.

**3. Interval-based Periodic Search (Section 3.3):** Instead of searching after every generation cycle, ConfLayers uses the current skip set for `Opt_Interval` (default: 30) iterations, accumulating accepted token counts. After each interval, if the average accepted tokens exceeds a threshold (default: 4), the search halts and the best layer set is used for all remaining prompts. Otherwise, a new search round begins. The process starts with uniform layer skipping.

**Optimization Objective:** ConfLayers prioritizes maximizing the number of accepted tokens rather than the acceptance rate, recognizing that a draft model with a high acceptance rate but few accepted tokens may not be the fastest in practice.

## Results & Highlights

- **Speedup:** Up to 1.4x over vanilla autoregressive decoding across LLaMa-2-13B, LLaMa-2-70B, LLaMa-3-8B, and LLaMa-3-70B. On CNN-DM summarization, ConfLayers achieves 1.37-1.38x on 70B models vs. SWIFT's 1.26-1.30x.
- **Consistently outperforms baselines:** DEL fails to provide any speedup because it requires LayerSkip fine-tuning. SWIFT struggles on smaller models (0.92x on LLaMa-2-13B/CNN-DM), while ConfLayers achieves 1.16x on the same setup.
- **Specialized models:** 1.24x speedup on CodeLLaMa-34B/HumanEval and 1.22x on Qwen2.5-Math-72B/GSM8K, both outperforming SWIFT.
- **Dynamic input stream robustness:** When a skip set optimized on CNN-DM is deployed on GSM8K, WMT14, and Alpaca, ConfLayers maintains >=1.1x speedup across all tasks, even outperforming SWIFT's task-specific optimization.
- **Output quality preserved:** Rouge-2 scores show high overlap with vanilla generation, consistent with the distribution-preserving property of speculative decoding with greedy sampling.
- **Ablation:** Fixed window sizes (10 or 20) perform poorly and inconsistently across models, while adaptive windowing delivers stable and superior results, confirming the importance of the adaptive mechanism.
- **Skip ratio:** Optimal performance is achieved with 40-50% of layers skipped, controlled via the lambda parameter.

## Critical Assessment

**Novelty (Moderate):** The core idea -- using per-layer entropy-based confidence to guide layer skipping -- is a reasonable but incremental extension of the self-speculative decoding literature. The adaptive window sizing based on second-order gradients and layer depth is a thoughtful engineering contribution, but the individual components (entropy, adaptive thresholding, iterative search) are well-established techniques applied in a new combination. The claim of "plug-and-play" is valid and meaningful given the training overhead of policy-based methods.

**Rigor (Moderate):** The paper provides thorough experimental evaluation across 6 models and 5 datasets, with ablation studies on the adaptive windowing mechanism. However, several concerns arise:
- The speedup numbers (up to 1.4x) are modest compared to the 2-2.5x reported by some speculative decoding methods with separate draft models. The comparison is limited to other plug-and-play SSD methods (SWIFT, DEL) rather than the full spectrum of speculative decoding approaches.
- Only 100 samples per dataset are evaluated, which may not capture the full variance in performance.
- The lambda parameter is chosen per-model, and the skip ratio is constrained to 40-60%, which introduces manual tuning that somewhat undermines the "adaptive" claim.
- The paper does not report wall-clock time breakdowns for the confidence computation overhead itself, only the net speedup.

**Reproducibility (Good):** Code is publicly available at https://github.com/WA225/ConfLayers. The method description is detailed with explicit equations. The experimental setup specifies hardware (AMD Instinct MI300X, ROCm 6.4.1), precision (FP16), and hyperparameters. However, the per-model lambda tuning is not fully documented.

**Significance (Moderate):** The work contributes a practical, training-free approach to self-speculative decoding that is competitive with the state-of-the-art plug-and-play methods. The focus on maximizing accepted tokens rather than acceptance rate is a sensible design choice. However, the 1.4x maximum speedup is still far from the speedups achieved by methods using separate draft models, limiting the practical impact for latency-critical applications.

**Red flags:**
- The comparison with DEL is somewhat unfair since DEL requires LayerSkip fine-tuning, making it not truly plug-and-play.
- The speedup metric is relative to vanilla autoregressive decoding on a single GPU; it is unclear how the method would perform in batched serving or distributed settings.
- The paper does not discuss how the method scales with sequence length or how the search overhead grows with model size.

## Strengths

- Truly training-free and plug-and-play: no fine-tuning, no auxiliary model, no architectural modification required.
- Consistent performance across diverse tasks (summarization, math reasoning, translation, instruction following, code generation) and model families.
- Robust to dynamic input streams: a skip set optimized on one task transfers well to others.
- Strong ablation study demonstrating the necessity of adaptive windowing over fixed-size approaches.
- Prioritizes accepted token count over acceptance rate, which better reflects practical speedup.
- Code is publicly available, supporting reproducibility.
- The search process is performed online during inference with negligible overhead, avoiding a separate profiling phase.

## Limitations

- Modest speedup ceiling: 1.4x is significantly lower than the 2-3x achievable with separate draft model approaches (e.g., Draft & Verify reports up to 1.99x).
- Per-model hyperparameter tuning required: lambda and minimum skip ratio must be set for each model, reducing the "plug-and-play" appeal.
- Evaluated on only 100 samples per dataset, raising questions about statistical robustness.
- Limited to greedy decoding; the paper does not explore top-p or top-k sampling scenarios where speculative decoding acceptance rates may differ.
- No evaluation on very large models (e.g., 405B+) or models with non-standard architectures (e.g., Mixture of Experts).
- The confidence computation requires projecting hidden states through the LM head at every layer, adding memory and compute overhead that is not explicitly quantified.
- The paper does not compare against methods like EAGLE or SEQUOIA that use separate draft models but achieve much higher speedups.

## Relevance

这篇论文与LLM高效推理直接相关，提出了无需训练的自推测解码加速方法，对于关注大模型部署效率和服务延迟的研究具有参考价值，特别是在资源受限场景下如何在不牺牲输出质量的前提下加速推理。

## Questions for Further Exploration

- How does ConfLayers perform with non-greedy sampling strategies (top-p, top-k), where the acceptance rate mechanism of speculative decoding behaves differently?
- What is the wall-clock overhead of computing confidence scores (projecting all layers through the LM head) relative to the savings from layer skipping?
- Can the adaptive window sizing mechanism be combined with tree-based speculation (e.g., SEQUOIA) to further improve acceptance rates and speedup?
- How does the method scale to models beyond 70B parameters, especially 405B+ models where the draft-verify cost ratio shifts?
- Is there a principled way to set lambda automatically (e.g., via meta-learning or Bayesian optimization) rather than manual per-model tuning?
- How does the skip set stability change over longer generation sequences (beyond 512 tokens)?
