> **一句话总结：** 本文提出 Möbius Learning——把 Transformer 的 block stack 切成 P 个 block group 分给 P 个 worker，每个 worker 用本地数据并以自身 block group 作为循环起点，使同一组参数在不同的数据流中分别在"浅层"和"深层"位置被联合优化（depth-role superposition）；在 modded GPT-2 small (124M) + FineWeb + 2.5B tokens + Muon 的四-worker 实验中，当 loop depth L = 6/10/15 时 Möbius 的 FineWeb 验证损失比同一 K=4L 计算预算下的 ordered-loop baseline 低 0.0084/0.0059/0.0083，但在 L=1/2 时反而高 0.1697/0.0557。

> **TL;DR:** Möbius Learning replaces the standard fixed-order block stack with worker-indexed cyclic block orders: each of P workers owns one block group and its local data, and hidden states circulate through the groups in cyclic order so that the same parameters are optimized in both shallow and deep positions ("depth-role superposition"). On a four-worker modded GPT-2 small (124M) trained on 2.5B FineWeb tokens with Muon, at loop depth L=6 / 10 / 15 (matched to ordered-loop baseline at the same K=4L block-group applications per path), Möbius Learning achieves lower FineWeb validation loss than the ordered-loop baseline by Δ = −0.0084 / −0.0059 / −0.0083, but it is worse at L=1 (Δ = +0.1697) and L=2 (Δ = +0.0557). The design also yields a memory-friendly "Möbius parallelism" where each worker stores one block group plus its own data rather than a full replicated model. The paper is explicitly a Version 1 preliminary technical report.

## 核心贡献

- **提出 Möbius Learning 训练架构**：把 P 个 Transformer block group 分配给 P 个 worker，每个 worker i 持有 F_i，并以自己的输入嵌入 E_s 和预测头 R_s 作为本地数据流路径的端点；batch 从 worker s 出发时按 h_{s,t+1} = F_⟨s+t⟩_P(h_{s,t}) 循环推进，从而同一组参数在不同数据流中以不同源相对深度（source-relative depth）出现。
- **定义 depth-role superposition（深度角色叠加）**：作者证明对固定 i，映射 s ↦ d_{s,i} 在 Z_P 上是双射，因此当 originating-worker 索引遍历 Z_P 时，每个 block group F_i 恰好占据 P 种 within-pass 源相对深度各一次，使得同一参数组在浅层和深层位置上被联合优化，从而打破了"参数组 ↔ 固定深度角色"的一一对应。
- **给出 Möbius 并行（Möbius parallelism）**：相对于复制式数据并行（每个 worker 持完整 stack）和流水线并行（按固定 stage 切分），Möbius 中每个 worker 仅存储一个 block group 加本地数据，隐藏状态及梯度在 worker 之间循环；原始训练数据始终留本地，与 split/federated learning 中的隐私/本地数据目标天然兼容。
- **形式化损失目标与评测协议**：ordered-loop 的损失按 Eq.(6) 在所有 worker 上共享 E、R、{F_i}；Möbius 的损失按 Eq.(7) 让每条 originating path 在本地 minibatch 上反传 path-specific loss，验证按 Eq.(8) 在所有 P 个 cyclic 起点上等权聚合。

## Key Contributions

- A **training architecture** that replaces the shared Transformer block order with worker-indexed cyclic orders. Block depth becomes data-stream-dependent: the same block group is applied early in the sequence for some streams and late for others, with one block group per worker reducing per-worker Transformer-parameter memory relative to replicated data parallelism.
- Identification of **depth-role superposition** — the empirical observation that the same block-group parameters can be trained across shallow and deep source-relative positions without forcing a fixed depth role per parameter group. The paper proves the cyclic bijection that makes every block group occupy every within-pass depth exactly once across originating workers.
- A derived **distributed execution scheme (Möbius parallelism)** in which raw input-target pairs stay local, hidden states and gradients circulate across worker boundaries, and each worker owns one block group instead of a full model replica — a hybrid of federated data locality and split/pipeline-style model partitioning.
- An **empirical result** showing that under matched 2.5B-token budget and matched K=4L block-group application counts, Möbius Learning matches or beats ordered-loop training at larger tested loop depths (L = 6, 10, 15) on FineWeb validation loss.

## Methodology

The Transformer stack is partitioned into P block groups indexed by Z_P = {0, …, P−1}. An ordered pass is Ψ = F_{P−1} ∘ … ∘ F_0 (Eq. 1), and ordered-loop training with loop depth L repeats it: R ∘ Ψ^L ∘ E (Eq. 2). Across L passes, block group F_i appears at positions {i + ℓP : ℓ = 0, …, L−1}, independent of originating worker — so each block group is locked to one source-relative depth.

Möbius Learning assigns worker i to own block group F_i. For a batch originating at worker s with input embedding E_s and prediction head R_s, the recurrence h_{s,t+1} = F_⟨s+t⟩_P(h_{s,t}) (Eq. 3) applies block groups in worker-indexed cyclic order. After K = LP block-group applications the resulting logits are R_s(Φ_{s,K}(E_s(x_s))), with Φ_{s,K} = F_⟨s+K−1⟩_P ∘ … ∘ F_s (Eq. 4). The input embedding and prediction map are each applied exactly once, regardless of K. The source-relative depth of F_i for stream s is d_{s,i} = ⟨i − s⟩_P (Eq. 5); the paper proves s ↦ d_{s,i} is a bijection on Z_P, so every block group occupies every within-pass depth exactly once as s varies — the basis of depth-role superposition.

Training objective for ordered-loop is Eq. (6): a population cross-entropy averaged over workers with shared E, R, {F_i}. Möbius objective is Eq. (7): each originating worker backpropagates a path-specific loss on its own local minibatch, and equal worker participation makes this the population objective without explicit loss averaging. Raw data stays local; hidden states and gradients cross worker boundaries. Evaluation (Eq. 8) explicitly averages path-specific losses over all P cyclic starting points rather than reporting one selected start.

Experiments: four workers, modded GPT-2 small (124M), FineWeb, 2.5B tokens (Chinchilla-optimal budget for this scale), Muon optimizer with fixed learning-rate schedule and global batch across runs. Möbius divides the 12-layer stack into four worker-resident groups; ordered-loop repeats the same four groups for L passes. Each row of Table 1 fixes L and hence K = 4L, so the comparison is matched within row, not across loop depths. Both methods are evaluated with an equally weighted mean of four losses on the same FineWeb validation data. The implementation follows Appendix A; decoder-block parameters stay on assigned workers and are not synchronized across groups.

## Results & Highlights

- **FineWeb validation loss at matched token budget and K = 4L (Table 1, four-worker modded GPT-2 small 124M, 2.5B tokens, Muon):** L=1 → ordered-loop 3.3042 (K=4) vs Möbius 3.4739 (K=4), Δ = +0.1697 (Möbius worse); L=2 → ordered 3.2890 vs Möbius 3.3448, Δ = +0.0557; L=6 → ordered 3.2564 vs Möbius 3.2480, Δ = −0.0084; L=10 → ordered 3.2456 vs Möbius 3.2397, Δ = −0.0059; L=15 → ordered 3.2420 vs Möbius 3.2337, Δ = −0.0083. The crossover from "Möbius worse" to "Möbius better" occurs between L=2 and L=6.
- **All comparisons are matched at identical compute per path** (K = 4L block-group applications, one embedding and one prediction head per path), so the L = 6/10/15 wins are not a depth-vs-compute confound: the same number of block-group maps are applied to each input.
- **Largest Möbius improvement is at L = 6** (Δ = −0.0084), not at the deepest L = 15 (Δ = −0.0083), suggesting the relative advantage is roughly flat rather than monotone in loop depth across the tested range — though only three points lie past the crossover.
- **Worst-case Möbius deficit is at L = 1** (Δ = +0.1697): when each input only sees one pass, cyclic shifts force every block group into a depth role that depends on originating worker, with no repeated traversal to amortize the superposition — a regime where ordered training is clearly better.
- **Validation aggregation across all P = 4 cyclic starting points** (Eq. 8) is reported for Möbius, avoiding any single-start cherry-picking in the headline numbers.

No downstream-task (e.g. MMLU, HellaSwag) or training-time/throughput numbers are reported; all quantitative claims are validation loss only.

## Critical Assessment

- **Novelty**: Genuine but narrow. Combining cyclic block orders with worker-local ownership is a fresh construction relative to fixed-order looped/ recurrent Transformers (Universal Transformers, ALBERT, Subformer, Parcae, latent-thought looped LMs) and to LayerShuffle (which fine-tunes a fixed-order ViT rather than pretraining with cyclic orders). The bijection d_{s,i} is elementary but its empirical consequence (depth-role superposition) is a real new claim. The framing as a *distributed* training scheme (Möbius parallelism) is conceptually interesting but not separately validated against pipeline or federated baselines.
- **Rigor**: Mixed. The math is clean and the controlled experimental protocol (fixed Muon config, fixed LR, fixed global batch, Chinchilla token budget, matched K per row) is well designed for a fair Möbius-vs-ordered-loop comparison. However, **no seed variance or confidence intervals** are reported for any of the five Table 1 rows, so the L = 6/10/15 deltas (−0.0084, −0.0059, −0.0083) cannot be distinguished from run-to-run noise at 124M scale; the L = 1 catastrophic gap (+0.1697) is the only result that is unambiguously outside plausible seed variation.
- **Reproducibility**: Weak. No code, no hyperparameter table, no random-seed protocol, no hardware/runtime figure. The "memory-constrained distributed training" motivation is asserted (Section 1, Section 4) but **no actual per-worker memory or wall-clock throughput number is reported**, so the central practical claim — that Möbius lets you train when the full stack does not fit on one device — is unsupported quantitatively. Appendix A is only ~150 words.
- **Significance**: Limited at present. The paper is honest that this is a four-worker, 124M, 2.5B-token study, and Section 5 explicitly defers scaling. The depth-role-superposition claim, if it survives at larger scale with seeds, would be a meaningful design lever for looped/recurrent LMs and for memory-constrained collaborative training. As it stands, the deltas are small and the regime where Möbius wins is only L ≥ 6.
- **Red flags / overclaiming**: The abstract and Figure 1 headline "Möbius Learning achieves lower validation loss than ordered-loop training" without prominently flagging that this is **only true for L ≥ 6 and reverses to +0.1697 at L = 1** — the abstract's "Surprisingly … achieves lower validation loss" is technically correct only for the largest-three rows. The "strikingly" language in Section 3 overstates results whose magnitude (≤ 0.01 nats) is well within plausible seed noise at this scale. The distributed-training motivation is offered in the abstract and Section 4 but **never measured** — no memory or throughput comparison against replicated data parallelism or pipeline parallelism is provided. The author describes the L ≥ 6 win as a "counterintuitive result" without any mechanistic explanation or probe (e.g. attention/FFN behavior at superposed depths, capacity-utilization measurement), relegating all of this to "future work." Comparison to LayerShuffle is qualitative (Appendix B.1) with no head-to-head numbers; comparisons to other looped LMs (Parcae, latent-thought) are absent numerically.

## Strengths

- Mathematically precise formulation: Eqs. (1)–(8) and the source-relative-depth bijection give the construction unambiguous semantics, and the distinction between Φ_{s,K} and Ψ^L is handled correctly (cyclic shifts generally do not commute).
- Cleanly controlled comparison: token budget, optimizer (Muon), LR schedule, global batch, model architecture, and per-path block-group application count K = 4L are all held fixed within each row of Table 1, so the L ≥ 6 deltas are not confounded by compute.
- Honest validation aggregation (Eq. 8) over all P cyclic starting points, rather than selecting the best worker.
- The conceptual framing of Möbius parallelism as a hybrid of federated data locality and split/pipeline model partitioning is genuinely thought-provoking and clearly distinguished from each in Figure 2.
- Transparent about scope: explicitly labelled Version 1 preliminary report, with Section 5 enumerating what scaling, worker-count, and representation-level questions remain.

## Limitations

- Only one model size (124M GPT-2 small), one worker count (P = 4), one dataset (FineWeb), one optimizer (Muon), and one token budget (2.5B). No scaling curve is provided.
- No seed variance, confidence intervals, or multiple-seed runs; the L = 6/10/15 advantages (Δ = −0.0059 to −0.0083) are within plausible noise at this scale and cannot be confidently attributed to the method.
- Möbius is **worse** at L = 1 (Δ = +0.1697) and L = 2 (Δ = +0.0557); the regime where it wins is only L ≥ 6, and the paper does not explain the crossover mechanism.
- No downstream-task evaluation (only validation loss) and no training throughput / per-worker memory measurements, despite the memory-constrained distributed-training motivation being central to the abstract.
- No mechanistic analysis of depth-role superposition (no attention-pattern, FFN-memory, or capacity-utilization probes); Section 4 offers only a hypothesis.
- No numerical comparison to LayerShuffle, Parcae, or other looped/recurrent baselines beyond the named ordered-loop control.
- Single-author, non-peer-reviewed preliminary report; no承诺 of code release.

## Relevance

对用户在「语言模型训练架构 / 循环与权重共享 Transformer / 分布式训练」方向的研究高度相关：Möbius 提供了一个把循环深度、参数共享与 worker-local 数据/模型所有权三者耦合的新设计点，可作为后续在 looped LLM（Parcae、latent thoughts）、split/federated learning 以及显存受限协同训练等课题上扩展的实验模板。其 depth-role superposition 现象若在更大规模上稳定复现，将对 ALBERT/Universal Transformer 一脉的参数共享文献形成直接补充。

## Questions for Further Exploration

- Does the L ≥ 6 Möbius advantage survive multiple seeds, larger models (e.g. 350M / 1.4B modded GPT-2), longer token budgets, and downstream benchmarks (MMLU, HellaSwag, ARC), or is the ≤ 0.01 nats delta at 124M within run-to-run noise?
- What is the actual crossover mechanism between L = 2 (Möbius worse by 0.0557) and L = 6 (Möbius better by 0.0084) — is it that repeated cyclic traversal is required before depth-role superposition pays off, and if so can the crossover L be predicted from P?
- How does Möbius scale with worker count P? At P = 2 the cyclic order reduces to {F_0, F_1} and {F_1, F_0}; does depth-role superposition strengthen or weaken as P grows?
- What are the concrete per-worker GPU memory and end-to-end throughput numbers for Möbius parallelism versus replicated data parallelism and pipeline parallelism at the same effective depth, given that this is the paper's headline practical motivation?
- Can representation-level probes (attention-head role, FFN key-value memory, residual-stream SVD) detect a measurable signature of depth-role superposition inside a single trained block group, providing the mechanistic account the paper defers to future work?
- Is the L = 1 catastrophic deficit (Δ = +0.1697) recoverable by warmup — e.g. short ordered-loop pretraining followed by cyclic fine-tuning — or is cyclic order inherently incompatible with single-pass computation?
