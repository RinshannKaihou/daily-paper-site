C2KV: Compressed and Composable KV Cache Reuse for Efficient LLM Inference

Time-to-first-token on an 8-document in-context-learning request: full prefill 1.06s, non-prefix reuse of uncompressed KV 0.52s (load + blend), C2KV 0.1s load with no blend step at 4× compression. Paper reports up to 17× end-to-end speedup under long contexts.

Why this matters

Long-context RAG, agent memory, and multi-document QA are bandwidth-bound, not FLOP-bound — and C2KV offers a deployable compress-then-concatenate serving pattern that drops the blend step entirely.

First framework to co-design KV compression and non-prefix reuse; reframes the dominant long-context cost from prefill compute to DRAM→HBM transfer.

Critical edge

The "up to 17× speedup" is a composed upper bound — no single end-to-end benchmark reports it. Readers must mentally stack the TTFT win (~10× on load) with the decode win (~3–4× at 128k), and offline extraction cost is excluded from TTFT (Appendix C).

01

Methodology

frozen base · sidecar extractor

C2KV reframes non-prefix reuse as Encode → Concatenate → Inference. The encoder is a learned C2Extractor bolted onto a frozen base model; the blender collapses to direct concatenation + RoPE re-rotation. Only the sidecar is trained.

Encode · offline
Document → C2Extractor → m = ⌈n/k⌉ compressed C2Token slots. One-time per document.
Concatenate · online
Stored segments directly concatenated at any position + RoPE re-rotation. No blend step.
Inference
Standard attention over [C2KV ; query]. Base model params never updated.
C2Tokens
one shared learnable embedding per k-token block (m = ⌈n/k⌉)
QKV heads
per-layer independent WC2,Q/K/V matching base dims
~10%

  • Dynamic-ratio training. k sampled from {4, 8, 16} per step → single model generalizes to unseen 10× at inference, often beating fixed-ratio training (Table 1).
  • Training corpus. 40k each from HotpotQA, 2WikiMultiHopQA, LongMagpie (120k total, 1 epoch); AdamW lr 5e-5 · wd 0.1 · BS 32 · BF16.
  • Why SIF, not anchors. Bidirectional anchor-token methods entangle KV with full context (Fig. 6); the asymmetric one-way flow is what makes the extracted manifold composable.

03

Strengths

what lands
  • Real bottleneck, well-quantified. Fig. 2/3 show KV deviation and blend-vs-load time growing with context — explicitly reframes the cost as bandwidth, not FLOPs.
  • Frozen-base sidecar (C2Tokens + QKV heads, ~10% overhead) preserves parametric knowledge and avoids per-model full fine-tuning — practical as base models iterate.
  • SIF asymmetry is the correct primitive. Original-token invariance cleanly separates "what is reused" from "what is compressed" (Table 4 ablation: anchor tokens collapse Qasper 0.4417 → 0.2750).
  • Dynamic-ratio training yields one model for multiple budgets, generalizing to unseen 10× — genuinely useful for per-request compression.
  • Comprehensive study. 5 baselines × 3 model families × 7 LongBench tasks + GSM8K + RULER depth sweep + SnapKV-stacking failure mode — far above workshop-level.
04

Limitations

what to probe
  • 17× is a composed upper bound. No single end-to-end benchmark reports it; GPU type unspecified, so absolute ms cannot be transferred.
  • Offline extraction excluded from TTFT. For cold-start or write-heavy corpora the cost model is less favourable — sidecar must run once per new document (~10% extra params + k:1 forward).
  • Still trails FR on HotpotQA across all three models (Qwen3-4B 0.3801 vs 0.5392; Llama 0.4828 vs 0.5343). "Without degrading quality" is task-dependent, not universal.
  • Only one training-based baseline (Block-Attention, Llama only). KVLink and TurboRAG — the closest relatives — are not directly compared.
  • No online/incremental extraction, no content-aware or layer-adaptive budget (acknowledged Appendix E); uniform ratio across all docs and layers.
  • No error bars on Table 5; some "wins" over EPIC are within plausible run-to-run noise.
05

Open questions for follow-up

what to ask the authors
Q1 · COLD-START AMORTIZATION

What is the measured end-to-end speedup on vLLM / SGLang with offline extraction cost included, on what GPU? Does 17× survive once per-new-document extraction is amortized over realistic corpus-miss rates?

Q2 · THE FR-BEATING ANOMALY

Why does 4× C2KV beat Full Recompute on MuSiQue (0.3587 vs 0.3198) and 2WikiMQA (0.4477 vs 0.4018)? Lost-in-the-middle mitigation, LongBench artifact, or genuine denoising?

Q3 · QUANTIZATION COMPATIBILITY

If a 4× SIF-extracted C2KV is then 2-bit quantized (CommVQ / KVQuant), does composability survive — or does quantization re-introduce the cross-context coupling SIF was designed to avoid?

Q4 · CROSS-FAMILY TRANSFER

The C2Extractor adds ~10% params per base model. Can it be shared across model families (train on Qwen, transfer to Llama), or does per-layer QKV binding force full retraining per checkpoint?

Q5 · PER-DOCUMENT RATIO

Dynamic-ratio training handles unseen global budgets — does it also extend to variable per-document compression within one request (important docs 4×, others 16×)? Does that improve the TTFT/accuracy Pareto?

Q6 · MISSING HEAD-TO-HEAD

Block-Attention was only on Llama3.1-8B. A direct comparison against KVLink and TurboRAG on all three models would close the biggest gap in the experimental matrix.
