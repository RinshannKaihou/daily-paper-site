> **一句话总结：** 作者首次对 Agent Skills 的"渐进式披露"模式做受控实验，在 ∞Bench 上对比原始文档导航、扁平化 skill 包与层级化 skill 包，发现扁平化仅对导航能力弱的 agent harness 有帮助、在多书库规模下变得决定性（K=20 时 En.QA 准确率 0.462 vs 原始 0.257），而第二层路由从不带来收益、有时甚至灾难性崩塌（En.MC 0.9126 → 0.6398），结论是"渐进式披露买的是上下文空间，而非智能，且一层就够"。

> **TL;DR:** The first controlled study of Agent Skills' progressive-disclosure pattern on ∞Bench (LOONGDOC environment) compares raw-document navigation, a flat SKILL.md pack, and a hierarchical child-skills pack across three harnesses (Codex, Pi, Claude-Code) and three model families (gpt-5.4-mini, qwen3.6-27b, claude-haiku-4.5). On a single book, flat disclosure helps only when the agent's native navigation is weak (Pi, Claude-Code) and adds nothing for Codex, whose bare agent already greps entities on the fly; the hierarchical pack never beats flat and sometimes collapses accuracy outright (Pi gpt-5.4-mini En.MC: 0.9126 → 0.6398). At library scale (K=20 books), raw navigation collapses (En.QA falls to 0.257) while flat disclosure holds 0.462 and also halves token cost ($25 vs $52 per question), so one level of in-skill-indexed disclosure is the recommended default.

## 核心贡献

- **首个受控对比实验**：首次将"原始文档导航（raw）"、"扁平渐进披露（flat）"与"层级递归披露（hierarchical）"三种 agentic 阅读方式在同一 chunk 集合、同一任务指令、同一评分器下进行系统对比，分离出"路由深度"与"per-chunk 索引位置"这两个先前从业者文献未分开的设计维度。
- **LOONGDOC 长上下文 agent 环境**：将静态的 ∞Bench 转化为基于 BenchFlow + Agent Client Protocol (ACP) 的交互式沙箱环境，每个问题成为可记录轨迹、可逐 run 评分的 agent 任务，并支持将语料从 1 本书扩展到 K 本书（K ∈ {5, 10, 20}）的库级导航。
- **可解释的实证发现**：通过逐轨迹分析（如 Codex 自行 grep 实体并只读匹配段落）解释为何 flat 对强 agent 无用、为何 hierarchical 会因 always-loaded 子描述挤占 router 上下文而崩塌，避免仅报告准确率。
- **明确的工程指导**：把"包装一本书"的最佳实践提炼为一句话——用一层扁平 SKILL.md + references/ 索引就够了，不要再加 meta-router；当语料大到原生导航失效时，扁平披露同时提升准确率和效率。
- **与经典检索基线对比**：加入 hybrid-rag（BM25 + BGE-M3 + RRF + BGE cross-encoder reranker）作为外部参考，证明 in-skill 索引路由几乎在所有单元格都优于经典 retrieve-and-rerank 管线，渐进披露收益不可还原为"RAG 换皮"。

## Key Contributions

- **First controlled comparison** of three agentic reading approaches (raw-document navigation, flat disclosure, hierarchical disclosure) for book-length QA, built over one fixed chunk set so that only the disclosure wiring varies — separating routing depth from index location, axes the practitioner literature had conflated.
- **LOONGDOC agent environment**, a long-context interactive testbed that turns each ∞Bench item into a sandboxed ACP task with a deterministic verifier and a per-run scalar reward, capable of scaling a single question across K staged books (K ∈ {5, 10, 20}).
- **Three harness × three model families sweep** (Codex, Pi, Claude-Code; gpt-5.4-mini, qwen3.6-27b, claude-haiku-4.5) plus a classical hybrid-rag baseline, with mean ± std/SE over multiple seeds.
- **Trajectory-grounded explanations** for every headline result (e.g., the bare Codex agent reconstructs locate-then-read on the fly by grepping named entities), not just accuracy tables.
- **Concrete packaging guidance**: one flat, in-skill-indexed level is the default; a second routing level never helps and sometimes breaks accuracy; at library scale flat disclosure is simultaneously more accurate and cheaper than raw navigation.

## Methodology

The authors cast three approaches as different "wirings" over the same book:

- **Book-to-skill pipeline (built once per book, identical input to both disclosure packs).** A book is chunked at its own chapter headings (fallback: ~4000-word paragraph-aligned chunks). An LLM produces a two-part per-chunk output (`SUMMARY:` + `KEY_ELEMENTS:`), then a book-level description is generated from the chunk summaries alone with anonymized names, at temperature 0 with a fixed seed. These descriptions are the routing metadata.
- **raw**: the raw book sits in the sandbox; the agent's navigation policy is unconstrained (open, grep, or read-on-demand).
- **flat** (Figure 1): a single SKILL.md whose `description` field is always in context (gates discovery); its body is a table indexing every chunk by path + per-chunk description (gates activation); the agent then opens chunk files from `references/` (execution). Per-chunk descriptions pay context only after the book activates.
- **hierarchical** (Figure 2): each chunk becomes its own child skill whose always-loaded `description` is the chunk description; a meta-router on top routes among children. The meta-router + all child descriptions are always in context.
- **hybrid-rag baseline**: BM25 + BGE-M3 dual retrieval, RRF fusion, BGE cross-encoder reranking, top-k evidence fed to the answer model.
- **LOONGDOC environment**: built on BenchFlow (ACP), sandboxed file system, deterministic verifier (regex option extraction for MC; ∞Bench normalized match + token F1 for open QA, Chinese variant normalizes punctuation). Each run logs the full trajectory (tool calls, files opened, tokens spent) and emits a scalar reward.
- **Controlled contrasts**: raw-vs-flat isolates whether disclosure helps at all; flat-vs-hierarchical isolates whether routing depth matters.
- **Library-scale probe**: K ∈ {5, 10, 20} books staged per task, each question still targets one book, testing whether the agent can locate the right book before reasoning.
- **Compute**: closed models via hosted APIs; qwen3.6-27b served with vLLM on 2× NVIDIA RTX 8000 (tensor-parallel-size 2, 131K context); adapter versions pinned (`pi-coding-agent 0.66.1`, `pi-acp 0.0.25`) to prevent harness drift.

## Results & Highlights

- **Single book, harness-dependent gain.** On Pi, flat matches or exceeds raw on every cell (e.g., Pi gpt-5.4-mini En.MC: flat **0.9126** vs raw **0.8851**; Pi qwen3.6-27b En.MC: flat **0.8023** vs raw **0.7865**). On Claude-Code claude-haiku-4.5, flat lifts En.MC from **0.7448** (raw) to **0.8667**, En.QA from **0.6892** to **0.7173**, Zh.QA from **0.8142** to **0.8330**. Under Codex gpt-5.4-mini the three approaches tie within error on all three subsets (En.MC: raw 0.8943, flat 0.8977, hier 0.8874).
- **Hierarchical collapses on Pi.** Pi gpt-5.4-mini En.MC drops from flat's **0.9126** to hierarchical's **0.6398**; Pi qwen3.6-27b Zh.QA drops from **0.7479** to **0.3890**, far outside cells' variance, because always-loaded child descriptions saturate the router's context before it commits.
- **Hybrid-rag loses nearly everywhere.** On qwen3.6-27b it trails both raw and flat on all three subsets (e.g., Zh.QA: hybrid-rag **0.5856** vs flat **0.7479**). On gpt-5.4-mini En.MC, hybrid-rag (**0.7628**) trails flat (**0.9126**) by ~15 points.
- **Library scale flips the picture.** At K=20 on En.QA (Codex gpt-5.4-mini), flat holds **0.462** while raw collapses to **0.257** — a gap clearing one standard error. On Claude-Code claude-haiku-4.5 En.QA, flat beats raw across all K (K=10 gap clears two standard errors: **0.480** vs **0.413**). Zh.QA at K=20 collapses for everyone (Codex raw 0.043, flat 0.095).
- **Hierarchical fails to reproduce the library-scale rescue.** At K=20 on En.QA, hierarchical collapses back onto raw (**0.267**) while flat stays at **0.462** — over 1.7× the accuracy — because hierarchical always loads every chunk's description, recreating the context pressure disclosure is meant to relieve.
- **Cost-accuracy Pareto frontier (Figure 4 & 9).** At K=20, raw reads the most tokens (**68.3M tokens, ~$52/question** uncached) for the worst accuracy; flat reaches nearly double the accuracy at roughly half the tokens and half the cost (**32.5M tokens, ~$25/question**). The Pareto frontier recedes down-and-to-the-right as K grows — a larger corpus is strictly less efficient, not merely harder.
- **Depth recovers only on weak-navigator open QA at scale.** On Pi Zh.QA at K=20, hierarchical holds **0.330** (best cell) while flat collapses to **0.137** — the sole regime where the second level pays off.
- **Takeaway**: "Progressive disclosure buys context, not intelligence" — negligible for a single book under a strong agent, decisive at library scale, and one flat level is enough.

## Critical Assessment

- **Novelty**: Genuine. This is the first controlled study that pits Agent Skills' progressive disclosure against raw-document navigation *and* a hybrid RAG baseline on a standardized long-document benchmark (∞Bench), while separating the two design axes (routing depth, index location) that the practitioner literature conflates. The LOONGDOC environment is a reusable contribution beyond the headline finding.
- **Rigor**: Strong experimental hygiene — same chunk set across flat and hierarchical, same task instruction (Figure 5) and same deterministic verifier across all three approaches, three seeds for single-book (mean ± std) and bundle-and-seed standard errors for multi-book, pinned adapter versions to prevent harness drift, temperature-0 description generation with a manifest hash for reproducibility. Trajectory inspection (e.g., Codex's grep-then-read behavior) is used to *explain* results rather than just report them.
- **Reproducibility**: Mixed. The LOONGDOC pipeline, prompts (Figure 7), sandbox layout (Figure 6), and verifier logic are fully documented, and the framework builds on public BenchFlow + ∞Bench. However, the derived skill packs and book texts are not redistributed, closed-model harnesses (Codex, Pi, Claude-Code) are not fully open, and exact harness invocation flags are not all listed, so an independent group would face non-trivial work to reproduce the multi-harness grid exactly.
- **Significance**: High for the agent-engineering community. The "one level is enough" and "flat beats raw at library scale" verdicts are immediately actionable for practitioners packaging book-length material as skills, and the negative result on hierarchical disclosure pushes back against a natural temptation to over-engineer skill packs.
- **Red flags / weaknesses**:
  - **Pre-training confound on En.MC**: the ∞Bench multiple-choice books are canonical English novels the models have likely memorized; the authors themselves flag this as the clearest threat, noting the bare agent answers En.MC correctly at K=20 in too few tool calls to have actually read 20 books. This blunts the strongest single-book claims on En.MC and forces reliance on the noisier open-QA subsets.
  - **Sample size**: limited to what ∞Bench ships, so several comparisons stay within a standard error — the En.MC multi-book cells never separate cleanly, and the Zh.QA directions are explicitly called out as not conclusive.
  - **Narrow scaling axis**: only K ∈ {1, 5, 10, 20} and only narrative corpora; the authors cannot say whether the effect transfers to code or technical manuals.
  - **Language-specific ceiling**: gains are sharp on English open QA and absent/negative on Chinese, where base-model capability rather than the skill pack sets the ceiling (claude-haiku-4.5 far outscores gpt-5.4-mini on Zh.QA at K=20).
  - **Single fixed recipe**: chunking and description generation are held at one widely adopted book-to-skill procedure; a different chunk granularity or description-writing model could shift the flat-vs-hierarchical balance.
  - **Cost figures are uncached upper bounds**: the authors acknowledge this is the conservative case but it does overstate disclosure's cost relative to a cache-enabled deployment.

## Strengths

- First-mover empirical rigor on a pattern that had spread on engineering intuition and Hacker News anecdotes before anyone measured it.
- Clean separation of the two design axes (routing depth, index location) that practitioners had conflated.
- Trajectory-grounded explanations (Codex's grep behavior, hierarchical's context saturation) make the results interpretable, not just numerical.
- Cost-accuracy Pareto analysis elevates the library-scale verdict from "more accurate" to "simultaneously more accurate and cheaper."
- Honest negative-result reporting: hierarchical disclosure is shown to be useless or harmful in most regimes, a finding that resists overclaiming and directly improves practitioner guidance.
- LOONGDOC is positioned as a reusable RL-style environment (per-trajectory reward), not just a one-shot benchmark.

## Limitations

- Pre-training memorization of canonical English novels confounds En.MC, the subset that would otherwise be the cleanest signal.
- Sample sizes inherit ∞Bench's limits; several multi-book comparisons stay within one standard error.
- Scaling axis is narrow (K ≤ 20, narrative only) — no code, technical manuals, or truly out-of-distribution corpora.
- Skill packs and book texts are not redistributed for copyright reasons, raising the barrier to exact reproduction.
- Closed-model harnesses (Codex, Pi, Claude-Code) are partially opaque; full harness-version and invocation reproducibility is not guaranteed.
- A single fixed book-to-skill recipe is tested; the chunk granularity and description-writing model are not ablated.
- Cost numbers are uncached upper bounds, which the authors note favors raw in the comparison.

## Relevance

对研究 agentic 长上下文阅读、Agent Skills/progressive disclosure 与 RAG 选型的读者高度相关：它首次用受控实验回答了"skill 包该做多深、何时该用它替代 RAG"，结论可直接指导长文档 agent 系统的索引设计与成本-准确率权衡。

## Questions for Further Exploration

- Would the flat-vs-hierarchical verdict flip on non-narrative corpora (codebases, technical manuals, legal contracts) where chunk structure is more regular and entities are denser?
- How does the result interact with prompt caching, which the authors flag as the most cache-friendly part of disclosure and which would push flat further ahead on the cost frontier?
- Can LOONGDOC's per-trajectory scalar reward be used to post-train an agent (e.g., via RL) to navigate better, and would a trained navigator erase the flat-vs-raw gap the way Codex's grep behavior already does?
- At what K does flat disclosure itself break — is there a library size where a two-level hierarchy becomes necessary despite its single-book failure?
- How much of the gain comes from the routing metadata (per-chunk descriptions) vs. the chunking granularity itself, and would a description-free flat index already capture most of the benefit?
- Does the Zh.QA ceiling trace to base-model Chinese capability, to the chunking/description recipe, or to the verifier's Chinese normalization — and would a Chinese-tuned book-to-skill pipeline recover the disclosure signal?
