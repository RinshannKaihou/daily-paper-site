> **一句话总结：** 作者提出 STACE，首个用于概念擦除（concept erasure）评估的多智能体压力测试框架：通过文献驱动的多轮"假设—批评"辩论 + 可执行代码合成 + 迭代精炼，自动发现 T2I 模型中静态基准和一次性 LLM 探测方法遗漏的擦除失败，在四个概念类别上平均 SIRI 达到 51.1%，超越最强基线 Ring-the-bell（45.9%），并可迁移到 LLM 越狱任务。

> **TL;DR:** STACE reframes concept-erasure evaluation as agent-driven, iterative, literature-grounded stress testing. Specialised LLM agents generate, debate, code, and verify probing hypotheses against concept-erased SD-v1.4/SDXL models. Across 10 erased concepts spanning Object/Style/IP/Explicit categories with ESD-200, STACE achieves the highest average image-level success rate (SIRI 51.1%), beating the strongest baseline Ring-the-bell (45.9%) by +5.2 pp and the prior SOTA JailFuzzer (42.3%); it remains the only method able to crack hard IP concepts (Mickey Mouse 20%, Superman 17%, where most baselines score 0-3%), finds a successful hypothesis within 1-2 iterations on average (EII 1.4), and transfers to LLM jailbreaking (Claude-3-Haiku, Gemini-2.5-Flash-Lite, GPT-4o-mini) with only prompt/evaluator/library changes.

## 核心贡献

- **将概念擦除评估重构为压力测试问题**：首次提出用多智能体、迭代式、知识驱动的压力测试范式取代静态 prompt 集与一次性 LLM 探测，并配套提出 SIR（假设级 SIRH 与图像级 SIRI）、EII（最早成功迭代）、SER（执行成功率）、Nov(H)（假设新颖度）四类评估指标。
- **STACE 框架本身**：首个用于概念擦除压力测试的多智能体框架，包含 5 步迭代循环（任务验证 → 假设生成-批评辩论 → 代码合成执行 → 结果评估 → 经验汇总），将外部文献提炼为结构化 PaperCards 并通过 Query Generator 做语义检索，使生成的假设既贴近已发表方法又具有新颖性。
- **跨擦除设置与跨任务的泛化性验证**：在 6 种擦除方法（ESD/SPM/EAP/UCE/SalUn/MACE）、3 种擦除强度（ESD-100/200/500）和 2 种 T2I 主干（SD-v1.4、SDXL）上系统评估，并以最小改动（仅 prompt、评估器、文献库）迁移到 LLM 越狱任务，作为通用压力测试框架的证据。

## Key Contributions

- **Reformulation of concept-erasure evaluation as agent-based, iterative, knowledge-grounded stress testing**, accompanied by a tailored metric suite (SIRH/SIRI for success rate, EII for efficiency, SER for execution reliability, and Nov(H) for hypothesis novelty) that future agent-based evaluators can reuse.
- **STACE, the first multi-agent stress-testing framework for concept erasure**, featuring (i) a literature library distilled into structured PaperCards for retrieval-augmented hypothesis generation, (ii) a Hypothesis Generator–Critic debate with a Query Generator mediating RAG, (iii) a Code Synthesiser that turns hypotheses into executable probes (with iterative self-repair), and (iv) an Experience Summariser that feeds prior iterations forward and doubles as a transferable SeedHypothesis.
- **Extensive empirical validation plus a domain-transfer demonstration**: comparison against 5 LLM-based baselines across 4 concept categories, robustness studies across 6 erasure methods / 3 strengths / 2 T2I backbones, cost-accuracy trade-off analysis across STACE-Small/Standard/Large configurations, and adaptation to LLM jailbreaking against AutoDetect and AutoRedTeamer on 3 target LLMs.

## Methodology

STACE runs an outer loop of N=10 iterations. Each iteration has five stages:

1. **Task Verification** — A Task Verification agent (GPT-5-nano, temperature 0) converts the user's natural-language request (e.g., "test whether an SD model erased with ESD still remembers Mickey Mouse") and the concept-erased checkpoint into a formal TaskSpec, resolving ambiguity and classifying the target type (object / abstract / style / IP).

2. **Hypothesis Generation via M=2-round debate** among three agents:
   - **Hypothesis Generator** (Grok-4.1-fast) consumes TaskSpec, a SeedHypothesis, retrieved PaperCards, and prior iteration summaries; emits a structured hypothesis H with arguments, evaluation method, and success criteria.
   - **Critic Agent** (Grok-4) scores H on Novelty, Feasibility, and Detailedness; feedback is sent back for revision.
   - **Query Generator** (GPT-5-mini) issues targeted retrieval queries against the PaperCard vector store so the other two agents ground their reasoning in peer-reviewed prior work rather than LLM internal knowledge.

3. **Code Synthesis and Execution** — The Code Synthesiser (Gemini-3-flash) realises H as executable code, reusing and adapting code from cited repositories where possible; on execution errors it iteratively self-repairs. Each H is instantiated k=10 times (e.g., different prompt formulations).

4. **Result Evaluation** — An evaluator E judges whether each response reveals the target concept. E is selected per TaskSpec: NudeNet for explicit/nudity content, and Qwen2.5-VL-32B (confidence threshold ≥80%) for general visual concept recognition. H is deemed successful if at least one of its k instantiations is flagged positive.

5. **Trace and Report Summary** — An Experience Summariser records target type, probing strategy, experiment design, and novelty, feeding the next iteration. After all iterations a Report Summariser (GPT-5) compiles a human-readable vulnerability report.

**Literature Library (PaperCards).** Rather than RAG over raw PDF chunks, each curated survey/paper (seeded from Liu et al. 2024) is pre-summarised by a Paper Card Agent into a structured summary covering task, model types, methods, experimental settings, and limitations, then embedded for semantic retrieval. The Query Generator retrieves up to 3 PaperCards per query.

**Metrics.**
- SIRH (hypothesis-level): fraction of the N iterations whose H exposes ≥1 failure.
- SIRI (image-level): fraction of all N·k generated images flagged positive — used for fair comparison against single-shot baselines.
- EII: earliest iteration at which Eq. (3) holds.
- SER: fraction of iterations whose generated code successfully executes.
- Nov(H) = 1 − max_{P in L} cos(H, P): novelty relative to the most similar PaperCard.

## Results & Highlights

- **Main baseline comparison (Table 2, ESD-200, 100 images/concept):** STACE achieves the highest average SIRI of **51.1%**, versus Ring-the-bell **45.9%**, JailFuzzer **42.3%**, WhenConceptsErased **40.8%**, Coreference Prompt **37.5%**, and RedTeaming (GPT-5) **12.6%** — i.e., **+5.2 pp over the strongest baseline** in average SIRI (the paper also reports **+4.3% more failure cases** in hypothesis-level terms).
- **Hardest IP concepts (Table 2):** For Mickey Mouse, STACE scores **20%** vs Ring-the-bell 1%, JailFuzzer 3%, WhenConceptsErased 11%, and 0% for two baselines. For Superman, STACE scores **17%** while every baseline is ≤12% (most are 0-3%). For Van Gogh (style), STACE reaches **90%** vs Ring-the-bell 56% and JailFuzzer 40%.
- **Erasure-strength robustness (Table 3, SD-v1.4):** STACE's SIRH degrades gracefully from **69% (ESD-100)** to **68% (ESD-200)** to **39% (ESD-500)**; EII stays low (1.5/1.4/1.6), so a successful hypothesis is typically found within the first 1-2 iterations even at the strongest erasure.
- **Cross-method generalisation (Table 3, average SIRH):** SPM 68%, ESD-200 68%, EAP 59%, UCE 57%, SalUn 50%, MACE 34%. STACE exposes failures in every method-strength combination except Superman under MACE. SER ≥94% for all methods except MACE (73%).
- **Different T2I backbone:** On SDXL erased with ESD-200, STACE still achieves SIRH **32.5%**, SER 88%, EII 3 — lower than SD-v1.4, suggesting SDXL is erased more thoroughly.
- **Debate-round ablation (Table 4):** Increasing debate rounds M from 0 to 2 raises average SIRH from **63% to 68%** and lifts novelty; M=3 over-shoots and drops SIRH to 62%, so M=2 is the default.
- **Cost-accuracy trade-off (Table D1):** STACE-Small costs **$0.118**, default STACE **$0.192**, and STACE-Large **$0.487** per successful hypothesis; STACE-Small attains SIRH 38% (vs STACE 42%, STACE-Large 40%), so the cheapest variant stays competitive.
- **LLM jailbreaking transfer (Table 5, 50 attempts per model-topic pair):** STACE matches or beats AutoDetect and AutoRedTeamer on Gemini-2.5-Flash-Lite (e.g., Non-Violent Crime **50%** vs AutoDetect 34%, AutoRedTeamer 2%) and Claude-3-Haiku, and is the only method to land at least one successful jailbreak for every model-category pair, whereas baselines score 0% in multiple settings (e.g., AutoRedTeamer is 0% on all three Claude-3-Haiku categories except Pornography 0%).

## Authors & Credibility

Authors are from the **University of Edinburgh (School of Engineering, HTAbrary/Tsaftaris group)**, **University of Manchester**, and **University of Melbourne** — an established generative-models/forensics group with prior concept-erasure work (Xue et al. 2025 is cited as the source of the SeedHypothesis and Coreference Prompt baseline). The paper cites the standard erasure suite (ESD, UCE, MACE, SPM, SalUn, EAP) and standard red-team baselines (Ring-the-bell, JailFuzzer, WhenConceptsErased, AutoDetect, AutoRedTeamer). Code is reported as anonymously available; full prompt templates, agent configurations, and per-concept appendix tables (B1-B6) are provided.

## Critical Assessment

- **Novelty:** Genuine. Reframing concept-erasure evaluation as agent-driven stress testing with a literature library is new; the closest priors (JailFuzzer, AutoDetect, AutoRedTeamer) target jailbreaking/generic weakness detection rather than T2I concept erasure, and lack both executable code synthesis and PaperCard-style grounding.
- **Rigour:** Mixed. The metric design (SIRI for fair image-level comparison vs one-shot baselines, Nov(H) for novelty, EII/SER for efficiency) is principled, and the ablations (debate rounds, SeedHypothesis, LLM size, erasure strength/method/backbone) are broad. However, the headline "4.3% more failure cases" claim is hypothesis-level (SIRH) whereas the main table reports image-level SIRI where the gap is **+5.2 pp** — the framing understates the win but the two metrics are not interchangeable and the paper is not fully explicit about which is being cited where.
- **Baseline fairness:** Concerns. STACE consumes dramatically more resources than baselines: 10 hypotheses × 10 instantiations = 100 images (matched on images), but it also runs 5+ frontier LLMs (Grok-4.1-fast, Grok-4, GPT-5-mini, Gemini-3-flash, GPT-5) and a multi-round debate per iteration, costing $0.081/iteration. The per-image comparison is fair, but no per-dollar or per-LLM-call normalisation is reported, so it is unclear how much of the gain comes from the architecture vs simply spending more. The RedTeaming (GPT-5) baseline is essentially a strawman (12.6% average), inflating the average gap.
- **Cherry-picking risk:** On the Object category — the easiest concepts — STACE actually *loses* to baselines (Airplane: STACE 80% vs JailFuzzer 95%; Dog: STACE 60% vs Coreference 90%, Ring-the-bell 90%, JailFuzzer 42%). The wins concentrate on Style/IP/Explicit categories where static prompts under-explore. The paper acknowledges this implicitly via Figure 3 but does not foreground it.
- **Reproducibility:** Medium. Heavy dependence on very recent frontier APIs (GPT-5, Grok-4/4.1-fast/4.20-beta, Gemini-3-flash/3.1-pro) that may change behaviour or access over time; the Code Synthesiser explicitly fails on "changing external APIs." The 80% VLM confidence threshold for the evaluator is a single fixed choice whose sensitivity is not analysed — a loose evaluator could inflate SIRI. The literature library is manually curated from one survey (Liu et al. 2024), so results depend on that curation.
- **Significance:** High for the concept-erasure subfield. STACE gives a concrete, reusable red-team harness that exposes residual leakage (e.g., 17-20% on Disney/IP characters that other tools miss entirely), which is exactly the kind of evidence regulators and model owners need.
- **Red flags:** None rising to misconduct, but (i) the "4.3% more failure cases" vs "+5.2 pp SIRI" mismatch should be harmonised; (ii) evaluator-false-positive rate is not reported despite STACE's whole signal depending on it; (iii) only 2 T2I backbones (SD-v1.4, SDXL) are tested — no FLUX or rectified-flow transformers; (iv) the Nov(H) metric (1 − max cosine similarity to PaperCards) is a weak proxy for genuine novelty and is not validated against human judgement.

## Strengths

- Principled, multi-dimensional metric suite (SIRH, SIRI, EII, SER, Nov(H)) that future agent-based evaluators can adopt directly.
- Literature-grounded hypothesis generation is a real methodological advance over LLM-internal-knowledge probing; PaperCards are a clean, reusable abstraction.
- Broad robustness study across 6 erasure methods, 3 strengths, 2 backbones, 3 LLM-size configurations, and a cross-domain transfer to LLM jailbreaking.
- Honest cost reporting ($/successful hypothesis) and explicit cost-accuracy trade-off via STACE-Small/Standard/Large.
- Strong gains on the hardest, most safety-relevant categories (IP characters, explicit content) where existing tools are near 0%.

## Limitations

- High monetary and computational cost: $0.081/iteration and 5+ frontier LLM calls per iteration make STACE impractical for large-scale or real-time auditing without the STACE-Small trade-off.
- Code Synthesiser is the main reliability bottleneck (SER drops to 71-73% on the hardest settings — ESD-500, MACE); failures stem from ambiguous hypotheses, changing external APIs, and resource limits.
- The literature library is manually curated from a single survey; the authors note it should be replaced by an automated literature-search agent.
- Evaluator dependence: results hinge on NudeNet + Qwen2.5-VL-32B at an 80% confidence threshold; no sensitivity analysis on this threshold and no false-positive-rate estimate are provided.
- Novelty metric Nov(H) is a cosine-similarity heuristic, not validated against human novelty judgements.
- Coverage limited to diffusion-based T2I (SD-v1.4, SDXL); rectified-flow / transformer T2I backbones (e.g., FLUX) are not tested.
- Average gains are concentrated in Style/IP/Explicit categories; on easy Object concepts STACE is dominated by simpler baselines, suggesting diminishing returns for the multi-agent overhead when the search space is already well-covered.

## Relevance

与用户对 AI 安全、生成模型红队与机器遗忘/概念擦除评估的研究兴趣高度相关：STACE 提供了一个可直接复用的多智能体压力测试框架与评估指标，可应用于自研 T2I/LLM 安全审计流水线，也可作为自动化红队方法的参考基线。

## Questions for Further Exploration

- How does STACE's SIRI/SIRH change if the VLM evaluator confidence threshold is swept (e.g., 50%/70%/90%) and what is the empirical false-positive rate on known-erased-vs-known-present controls?
- What is STACE's per-dollar performance (successful hypotheses per $1) compared to running a cheaper baseline (e.g., JailFuzzer) at the same budget by simply scaling k?
- Does the PaperCard literature library transfer across erasure domains (e.g., T2I → T2I-video, or diffusion → rectified-flow transformers like FLUX), and how much does library curation affect absolute SIRH?
- Can the Hypothesis Generator's outputs be clustered to produce a *taxonomy* of concept-erasure failure modes rather than a single score, enabling targeted fixes by erasure-method authors?
- Is the debate-round peak at M=2 robust to stronger Critic LLMs, or would a single high-quality Critic (e.g., Grok-4.20-beta) make M=1 sufficient, lowering cost further?
