> **一句话总结：** 本文提出 C2KV，一种统一框架，将 KV cache 压缩与非前缀（non-prefix）复用联合训练：在冻结的基础 LLM 上外挂轻量级 C2Extractor（可学习 C2Tokens + 每层独立 QKV 投影 + 块局部结构化注意力流 SIF），并通过"压缩-拼接协同训练"显式对齐抽取时表示与下游拼接行为，在 LongBench 上达到与 Full Recompute 接近的精度（4× 压缩下 Llama3.1-8B MuSiQue F1 0.3587 vs FR 0.3198，反而更高；2WikiMQA 0.4477 vs FR 0.4018），同时把 TTFT 从 1.06s 降到 0.1s、KV 加载时间减少 10×，最长 128k 上下文下取得最高 17× 推理加速。

> **TL;DR:** C2KV is a non-prefix KV cache reuse framework for long-context LLM inference that, unlike prior work which only cuts prefill compute, jointly attacks the *storage and memory-bandwidth* bottleneck by learning a compressed and composable KV manifold. It bolts a lightweight sidecar C2Extractor onto a frozen base model (≈10% extra params on Qwen3-4B / Llama3.1-8B), uses learnable C2Tokens with a block-local Structured Information Flow attention mask so original-token KV stays untouched, and trains via compression-concatenation SFT on HotpotQA + 2WikiMultiHopQA + LongMagpie. On LongBench under 4× compression, C2KV matches or beats Full Recompute on several QA tasks (Llama3.1-8B MuSiQue F1 0.3587 vs FR 0.3198, 2WikiMQA 0.4477 vs FR 0.4018; Qwen3-4B 2WikiMQA 0.3920 vs FR 0.4095 — close), while TTFT on an 8-document ICL request drops from 1.06 s (full prefill) to ~0.1 s, KV-cache load cost drops 10× versus uncompressed reuse, decode-time-per-token at 128k is largely flattened, and the paper reports up to 17× end-to-end inference speedup under long contexts. It also shows that naively stacking SnapKV compression on EPIC/CacheBlend/Block-Attention collapses accuracy (e.g. CacheBlend on 2WikiMQA falls from 0.1334 to 0.0765), motivating the joint design.

## 核心贡献

- **重新定义瓶颈**：明确指出在长上下文 LLM 服务中，主导瓶颈已经从"prefill 计算量"转移到"KV cache 的存储与跨内存层级（DRAM→HBM）传输带宽"；既有的非前缀复用方法（CacheBlend、EPIC、KVShare、Block-Attention、KVLink）只优化计算，忽略了存储/带宽成本，并在图 2/3 中量化了 KV 偏差与 load 时间随上下文长度的增长。
- **统一框架 C2KV**：首次把 KV 压缩与 KV 复用联合优化，输出一个"position-agnostic、可拼接、可压缩"的 KV 表示流形，使文档 KV 能在任意位置 plug-and-play 直接拼接，无需 blending 再计算。
- **C2Extractor 设计**：冻结基础模型，只训练新加的 C2Token embeddings 与每层独立 QKV 投影头（≈10% 额外参数），通过 **Structured Information Flow (SIF)** 注意力掩码严格隔离原始 token 与 C2Token 通路——原始 token KV 与冻结模型逐位一致，C2Token 仅做块局部（含 sink block）+ 跨块因果聚合，避免全局信息泄漏。
- **压缩-拼接协同训练（Compression-Concatenation Co-Training）**：单一 SFT 语言建模损失（无辅助重建/对齐损失），只在拼接后对生成答案做监督；通过训练时随机化文档顺序与数量，迫使 extractor 学到"merge-ready"的可组合表示。
- **动态压缩比泛化**：单一模型在 {4×, 8×, 16×} 中随机采样压缩比训练，可泛化到未见过的 10× 推理时压缩比，且整体性能优于固定压缩比训练（Table 1）。
- **强实证证据**：在三个模型族（Qwen3-4B-Instruct-2507、Llama3.1-8B-Instruct、Qwen2.5-7B-Instruct）和七个 LongBench 任务上系统对比 5 个基线，并公开了代码（github.com/s7a9/C2KV）。

## Key Contributions

- **Reframes the bottleneck**: explicitly identifies KV-cache *storage and memory-bandwidth* (DRAM→HBM transfer), not prefill FLOPs, as the dominant cost in modern long-context serving, and shows (Fig. 3) that "blend vs load" latency becomes bandwidth-bound above ~32k tokens.
- **A unified compress + reuse formulation** that exposes and fixes a real failure mode: naively stacking a generic compressor (SnapKV 4×) on top of EPIC/CacheBlend/Block-Attention/Naive reuse collapses accuracy (Table 3, e.g. CacheBlend 2WikiMQA 0.1334 → 0.0765, Block-Attention 2WikiMQA 0.3886 → 0.1396), proving compression and reuse must be co-designed.
- **C2Extractor**: a frozen-base sidecar with learnable **C2Tokens** as memory slots (one per k-token block) and **per-layer independent QKV projection heads** that match base-model dimensions; parameter overhead ≈10% on Qwen3-4B and Llama3.1-8B, and base-model parameters are never updated.
- **Structured Information Flow (SIF)** attention mask enforcing (i) original-token invariance (original tokens never attend to C2Tokens, so their KV equals the frozen model's), (ii) block-local extraction with a sink block, (iii) causal accumulation across C2Tokens. This is contrasted with anchor-token methods (Fig. 6), whose bidirectional attention entangles the KV with full context and breaks composability.
- **Compression-Concatenation Co-Training**: a single SFT LM loss on the answer; supervision is applied only *after* KV concatenation, so each document's C2KV is optimized for its marginal contribution under arbitrary multi-document composition — no reconstruction or alignment loss.
- **Dynamic-ratio training** yielding a single model that generalizes to unseen inference-time compression budgets (e.g. 10×).
- **Open-source** at https://github.com/s7a9/C2KV and accepted to KDD '26.

## Methodology

The framework formalizes non-prefix reuse as a two-stage Encode–Blend–Inference pipeline (Eqs. 2–4). C2KV replaces the encoder M with a learned C2Extractor and the blender B with a trivial direct concatenation + RoPE re-rotation.

**C2Extractor construction.** Given a compression ratio k, the document's n tokens are partitioned into ⌈n/k⌉ blocks; one shared learnable **C2Token** embedding is appended per block (m = ⌈n/k⌉). For each Transformer layer ℓ, the frozen base QKV projections are kept for original tokens; C2Tokens use separate trainable W_{C2,Q}, W_{C2,K}, W_{C2,V} heads (Eq. 6) of identical output dimension so the concatenated Q/K/V can flow into the standard attention op. Only the C2Token embedding and these heads are trained — base-model parameters stay frozen. A **residual mean-pooling variant** (C2KV-Residual) adds the mean of each block's hidden states to the corresponding C2Token at layer 1 (Eq. 7, scaling 1.0) for extra local semantic aggregation.

**Structured Information Flow.** A block-structured attention mask enforces three constraints at every layer: (1) original-token invariance — original tokens attend only to original tokens under causal masking, so their hidden states and KV equal the frozen model's bit-for-bit; (2) block-local extraction — each C2Token c_j attends only to its own block B(j) plus a sink block B(0); (3) causal accumulation across C2Tokens so later compressed segments can aggregate earlier ones. This asymmetric one-way flow (original→C2 only) is what makes the extracted C2KVs context-independent and reusable, in contrast to anchor-token methods where bidirectional attention entangles the cache with the full input (Fig. 6).

**Reuse-time positional alignment.** Each stored C2KV segment is treated as occupying the same effective span as the original k tokens; positional offsets accumulate across concatenated segments and RoPE is re-rotated at reuse time. The ablation `C2KV Fixed-Pos.` (Table 4) shows disabling this re-rotation consistently hurts accuracy (e.g. Qasper 0.2827 vs 0.3755).

**Training.** Pure SFT loss on the generated answer conditioned on [concatenated C2KV; query] (Eq. 10). No reconstruction, alignment, or document-level auxiliary loss. Gradients flow only to C2Token embeddings + their QKV heads. Training corpus: 40k samples each from HotpotQA, 2WikiMultiHopQA, LongMagpie (120k total, one epoch), AdamW, lr 5e-5, warmup 0.06, weight decay 0.1, batch 32, BF16. For dynamic-ratio training, k is sampled from {4,8,16} per step.

**Evaluation.** LongBench (HotpotQA, 2WikiMQA, MuSiQue, MultiNews, SAMSum, QMSum, GovReport) plus GSM8K for few-shot reasoning; F1 for QA, ROUGE-L for summarization. Single-document LongBench subsets are paragraph-split to simulate non-prefix reuse. Three models tested: Qwen3-4B-Instruct-2507, Llama3.1-8B-Instruct, Qwen2.5-7B-Instruct. Baselines: Full Recompute (FR), Full KV Reuse (Naive), EPIC, CacheBlend, Block-Attention (Llama only, due to checkpoint availability).

## Results & Highlights

- **TTFT on an 8-document ICL request (Fig. 1)**: Full prefill 1.06 s, non-prefix reuse of *uncompressed* KV = 0.1 s load + 0.42 s blend, C2KV = 0.1 s load with 10× smaller transfer (4× compression) and *no blend* — i.e. the blend step is eliminated entirely.
- **Decode-time scaling (Fig. 8)**: as context grows 16k→128k, per-token decode latency for FR / CacheBlend / EPIC rises near-linearly (e.g. Llama3.1-8B ~10 ms → ~50 ms), while C2KV's curve "largely flattens," showing only mild increase even at 128k across all three model families.
- **Compression-ratio scaling on Llama3.1-8B (Table 1)**: at 4× C2KV beats FR on MuSiQue (F1 **0.3587 vs 0.3198**) and 2WikiMQA (**0.4477 vs 0.4018**); SAMSum 0.3904 vs 0.3652. At 16× it still matches or beats every reuse baseline (MuSiQue 0.2746 vs CacheBlend 0.2720 / EPIC 0.2498 / Naive 0.1821). Dynamic-ratio training generalizes to unseen 10× (MuSiQue 0.3198, 2WikiMQA 0.4310) and matches FR there.
- **Naive compression breaks reuse (Table 3)**: stacking SnapKV-4× on top of reuse methods causes catastrophic drops — CacheBlend on HotpotQA 0.2970 → **0.0532**, Block-Attention on SAMSum 0.3599 → 0.1266, EPIC on 2WikiMQA 0.2821 → 0.1681. This is the central evidence that compression + reuse must be co-designed.
- **Full LongBench + GSM8K across three models (Table 5)**: C2KV (4×) is the best or near-best reuse method on most cells. On Qwen3-4B HotpotQA, C2KV 0.3801 vs CacheBlend 0.3487 vs EPIC 0.3266 vs Naive 0.2486 (FR 0.5392). On Llama3.1-8B HotpotQA, C2KV 0.4828 vs Block-Attention 0.4271 vs EPIC 0.4363 (FR 0.5343). C2KV also beats *generic compression* FR-SnapKV(4×) on most tasks (e.g. Llama MuSiQue 0.3587 vs 0.1615) while additionally enabling composable reuse.
- **Ablation (Table 4)**: replacing SIF with bidirectional anchor tokens collapses Qasper 0.4417 → 0.2750 and GovReport 0.3274 → 0.1280; disabling reuse-time position re-alignment drops Qasper 0.3755 → 0.2827; allowing info-leakage or global-info attention also degrades all three tasks; using an untrained extractor (`C2KV-Res. w/o T.`) fails catastrophically (Qasper 0.1572), proving the composable manifold must be *learned*.
- **RULER (Fig. 9)**: at 4× compression, C2KV preserves answer-depth robustness across 4k–64k contexts on both Llama3.1-8B and Qwen3-4B, with only mild degradation versus the uncompressed base.
- **Headline system gain**: the paper reports **up to 17× end-to-end inference speedup** under long contexts (abstract + Sec. 1), combining the TTFT win and the flattened decode curve.

## Critical Assessment

- **Novelty**: Real. The framing that *storage/bandwidth*, not prefill FLOPs, is the real bottleneck of non-prefix reuse is well-argued (Fig. 3) and the joint compress–reuse formulation with a *frozen-base* sidecar is distinct from Block-Attention (which fine-tunes the base) and from KVLink/TurboRAG. The SIF mask (one-way original→C2, block-local + sink + causal C2-accumulation) is a genuinely clean mechanism whose information-flow asymmetry is the reason the extracted KV is composable.
- **Rigor**: Generally strong but with caveats. Table 3 is the most persuasive single piece of evidence — naively stacking SnapKV on every existing reuse method catastrophically fails — and the ablation matrix (Table 4) cleanly isolates each design choice (anchor tokens, no training, fixed positions, info-leakage, global-info). The three-model evaluation reduces single-vendor risk. *However*, the headline "up to 17× speedup" appears only in the abstract and Section 1; the body does not show a single end-to-end system number that decomposes to 17× — readers must mentally compose the TTFT win (Fig. 1, ~10× on load) with the decode win (Fig. 8, ~3–4× at 128k). That is plausible but under-documented.
- **Reproducibility**: Good on paper. All datasets are public, training hyperparameters are listed (AdamW, lr 5e-5, warmup 0.06, wd 0.1, batch 32, BF16, 120k samples, 1 epoch), base models are open-weight, and code is public. The TTFT measurement protocol (Appendix C) is unusually explicit about what is and is not included (excludes system-prompt prefill and offline extraction, uses `torch.cuda.synchronize()`). The one gap is hardware specification — the paper does not state which GPU was used, so absolute ms / TTFT numbers cannot be directly transferred.
- **Significance**: High if the claims hold. Long-context RAG and agent-memory workloads are exactly where non-prefix reuse + compression matters, and the "compress-then-concatenate without blending" serving pattern is deployable. The 4× compression at near-FR quality on multi-hop QA is a useful data point for the field.
- **Red flags / overclaiming**: (1) The "up to 17×" headline is a *composed* number not directly reported in a single experiment; it should be read as an upper bound, not a measured end-to-end figure. (2) The TTFT measurement *excludes* offline C2KV extraction time (Appendix C) — fair for steady-state serving, but for cold-start workloads the comparison is less favourable because the C2Extractor must run once per new document (≈10% extra params + the k:1 forward). (3) Block-Attention is only evaluated on Llama3.1-8B "due to checkpoint availability" — so the strongest training-based baseline is missing on two of three models. (4) On HotpotQA C2KV (4×) still trails FR by a wide margin (Llama 0.4828 vs 0.5343; Qwen3-4B 0.3801 vs 0.5392), so the "preserving generation quality" claim is true on MuSiQue/2WikiMQA/summarization but not uniformly. (5) The C2KV > FR "wins" on MuSiQue / 2WikiMQA at 4× are surprising (compression *helping*?) and deserve more investigation than the paper gives them — possibly an artifact of FR's known attention-mass / lost-in-the-middle issues on these particular datasets rather than evidence of superior information retention. (6) No statistical significance / variance bars on Table 5; the gains over EPIC on some cells are within plausible run-to-run noise.

## Strengths

- Cleanly identifies and quantifies a *real* bottleneck (bandwidth, not FLOPs) that prior reuse work glossed over, with Fig. 2 (KV deviation grows as recompute ratio drops) and Fig. 3 (blend/load time scaling) making the motivation concrete.
- The frozen-base sidecar design (only C2Tokens + QKV heads trainable, ≈10% overhead) preserves the base model's parametric knowledge and avoids per-model full fine-tuning — a real practical advantage over Block-Attention as open-source base models iterate.
- SIF's information-flow asymmetry is the correct primitive: by ensuring original-token KV equals the frozen model's, the method cleanly separates "what is reused" from "what is compressed."
- The dynamic-ratio training trick (one model, multiple compression budgets at inference, generalizes to unseen 10×) is a small but genuinely useful contribution for serving systems where the compression budget is chosen per request.
- Comprehensive baselines (5 methods) × 3 models × 7 LongBench + GSM8K, plus a RULER depth sweep and a SnapKV-stacking failure mode study — substantially more thorough than typical workshop-level KV-cache papers.

## Limitations

- The 17× headline is a composed upper bound; no single end-to-end system benchmark is reported with that number. Hardware (GPU type) is unspecified.
- Offline C2KV extraction cost (the one-time per-document forward through the sidecar) is excluded from TTFT; for cold-start or write-heavy corpora the cost model is less favourable.
- C2KV still underperforms FR on HotpotQA across all three models (e.g. Qwen3-4B 0.3801 vs 0.5392) — the "without degrading quality" claim is task-dependent, not universal.
- Only one training-based baseline (Block-Attention) is evaluated, and only on one model (Llama3.1-8B), due to checkpoint availability. KVLink and TurboRAG, which are the closest training-based relatives, are not directly compared in the results tables.
- Assumes reusable content can be identified and extracted *offline* — does not support online or incremental extraction (acknowledged in Appendix E).
- Uniform compression ratio across all documents and layers; no content-aware or layer-adaptive budget allocation (acknowledged in Appendix E).
- No error bars or significance tests on the main accuracy table; some "wins" over EPIC are small enough to be noise.
- The surprising cases where 4× C2KV *beats* FR (MuSiQue, 2WikiMQA) are reported without explanation — likely a dataset-specific artifact but worth understanding.

## Relevance

高度契合用户对"高效 LLM 推理 / 长上下文服务 / KV cache 压缩与复用"方向的研究兴趣：本文把 KV 复用与压缩统一进可训练的、位置无关的可组合表示流形，直接服务于 RAG、agent 长期记忆、多文档 QA 等场景的存储与带宽瓶颈，可作为后续在 RAG 系统或 agent 框架中落地 KV 复用的方法学参考。

## Questions for Further Exploration

- What is the *measured* end-to-end (cold-start + steady-state) speedup on a real serving stack (vLLM / SGLang) with the offline extraction cost included, and on what GPU — does the 17× headline survive once per-new-document extraction is amortized over realistic corpus-miss rates?
- Why does 4× C2KV *beat* Full Recompute on MuSiQue (0.3587 vs 0.3198) and 2WikiMQA (0.4477 vs 0.4018) on Llama3.1-8B? Is this a "lost-in-the-middle" mitigation effect, a LongBench-specific artifact, or evidence that the compressed manifold is genuinely a better denoiser than raw attention?
- How does C2KV interact with quantization-based compressors (CommVQ 2-bit, KVQuant)? If a 4× SIF-extracted C2KV is then 2-bit quantized, does the composability property survive, or does quantization re-introduce the cross-context coupling the SIF mask was designed to avoid?
- The C2Extractor adds ≈10% parameters per base model — can it be shared across model families (e.g. train once on Qwen, transfer to Llama), or does the per-layer QKV head binding force full retraining for every new base model checkpoint?
- Dynamic-ratio training generalizes to unseen 10×; does it also generalize to *variable per-document* compression ratios within a single request (some documents at 4×, others at 16× based on importance), and does that improve the TTFT/accuracy Pareto frontier?
- Block-Attention is only evaluated on Llama3.1-8B; a direct head-to-head against KVLink and TurboRAG (the other training-based non-prefix methods) on the same three models would close the biggest comparison gap.
- The SIF mask currently uses a fixed block size k; would a learned / content-aware block segmentation (e.g. splitting at semantic boundaries rather than every k tokens) improve fidelity under aggressive compression, or does it break the clean position-realignment arithmetic?
