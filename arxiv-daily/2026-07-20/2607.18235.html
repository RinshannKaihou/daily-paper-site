Paper: Automated Discovery Has No Universally Superior Harness

Akshat Gupta · Jermaine Lei · Alexander Lu · Gopala Anumanchipalli (UC Berkeley) · Leshem Choshen (MIT & MIT-IBM Watson AI Lab) — et al.

llm-guided-discovery evolutionary-search statistical-evaluation harness-ablation online-allocation 3.1M rollouts

1 Key metric · Adaptive Ensemble lift

+1.40pts

84.35% → 85.75% best-of-5

Under an identical 5 full-run-equivalent budget, the budget-matched Adaptive Harness Ensemble (12→5→2→1 pruning) beats Sequential Best-of-N, single-harness commitment (82.49%), and an unpruned portfolio (84.54%). SE ≈ 0.02, win on 11/12 pairs.

2 Why this matters

Largest rigorous harness benchmark and 3.1M-rollout pool for LLM discovery systems — reframing harness choice as an online hyperparameter, not a fixed recipe.

Speaks directly to anyone building FunSearch / AlphaEvolve / OpenEvolve / TTT-Discover-style systems, and to inference-time scaling & Hyperband-style allocator work.

3 Critical edge

"Universally" is doing heavy lifting. The verdict rests on just three math-discovery tasks and four open-weight models — code synthesis, reasoning models, and the closed-weights frontier go untested.

Methodology

Decomposition + null test

The harness — the rule that picks which previously-evaluated solutions an LLM should expand next — is treated as the object of study. Both OpenEvolve (evolutionary) and TTT-Discover (PUCT tree search) are rebuilt from a single greedy root so every ingredient becomes independently ablatable.

Results

Beyond the headline

No fixed harness survives Holm correction. The strongest fixed config (K=1, ε=20%) drops from p=0.023 raw to p=0.678 adjusted; full OpenEvolve lands at the bottom.

Strengths

  • Statistical care rare in this subfield — 100/30-run baseline pools, 100k bootstrap + permutation tests, Holm correction, prespecified α=0.05, raw and adjusted p-values both reported.
  • Reusable decomposition — Sequential-BoN → OpenEvolve / TTT-Discover turns recipe comparison into ingredient ablation, generalizing beyond these two systems.
  • Constructive counter-proposal — Adaptive Ensemble beats three baselines under matched budget with no new optimizer, pure framing win.
  • Honest null result — "no tested fixed harness is significantly better than a simple baseline" is a more useful statement than the usual positive spin.
  • Strong artifact release — 3.1M rollouts + per-step scores + null distributions make this a reference benchmark for future harness proposals.
!

Limitations

  • Narrow task domain — three math-discovery problems; no code synthesis, science, or RL-tuned task; cross-domain generalization of the headline claim is unproven.
  • Model coverage gap — no closed-weights frontier (Claude / Gemini / GPT-4o) and no reasoning-specialized models; allocation dynamics may shift there.
  • Adaptive result is within-pool — Table 1's 11/12 win is over resampled runs of the same pool, not out-of-sample policy evaluation.
  • No head-to-head vs. learned allocators — Meta-Harness, EvoX, AdaEvolve's island scheduler absent at matched budget.
  • TTT-Discover ablation excludes test-time training — its headline component — so conclusions apply only to its search harness.
  • No dedicated Limitations section — caveats are scattered through Related Work and Conclusion; the title's "universally" out-runs the evidence.

Questions for further exploration

Open threads
  1. Do the harness rankings transfer to non-math domains — SWE-bench-style code synthesis, symbolic regression, scientific hypothesis discovery?
  2. How do reasoning models (o1/o3, Claude extended thinking, Gemini 2.5 Pro) reshape the picture, given single deep rollouts may dominate N-shallow BoN?
  3. Would a learned bandit allocator (e.g. Thompson sampling over harness arms) beat the hand-designed 12→5→2→1 schedule?
  4. Is the early-progress-predicts-final correlation a property of these three fitness landscapes, or does it survive noisier / more deceptive evaluators?
  5. Which OpenEvolve ingredient (MAP-Elites / inspiration / multi-island) is actively harmful on which pair?
  6. Could the released null pools seed a harness-leaderboard protocol that future proposals must pass to claim improvement over Sequential BoN?
