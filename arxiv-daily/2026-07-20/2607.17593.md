> **一句话总结：** MILES 将度量学习与预训练模型结合用于类别增量学习，通过零初始化的子空间原型投影（SPP）适配器、解耦式低秩子空间表示扩展（SRE）子网络以及跨任务距离正则化（DR），在不修改冻结 ViT 主干且单次前向的条件下，于六个基准上刷新 SOTA，例如在 CIFAR-100 (T=10) 取得 93.93% 平均准确率（较次优 DGR +0.88%），同时在 FOOD-101 (T=10) 达到 93.23%（较次优 +2.08%）。

> **TL;DR:** MILES is an exemplar-free, PTM-based class-incremental learning method that integrates metric learning with an expandable per-task subspace: a zero-initialized SPP adapter pulls intra-task features toward their prototypes while pushing pseudo-old-class features away, a low-rank decoupled SRE sub-network strengthens representation without modifying the frozen ViT, and a Distance Regularization term keeps feature-to-prototype distances balanced across tasks. On six benchmarks (CIFAR-100, CUB-200, Omnibenchmark, ImageNet-A, ImageNet-R, FOOD-101) with ViT-B/16-IN21K, MILES sets new SOTA on every setting, e.g., 93.93% on CIFAR-100 T=10 vs DGR's 93.05% (+0.88%, forgetting 3.12 vs 4.85), 62.33% on ImageNet-A T=10 vs TUNA's 61.34% (+0.99%), and 84.90% on ImageNet-R T=10 vs DGR's 83.85% (+1.05%), while keeping ~373 MB memory and 16.92 GMACs — the lowest compute among compared methods.

## 核心贡献

- **提出 SPP（Subspace Prototype Projection）子空间原型投影策略**：在每个新任务上级联一个零初始化的轻量适配器（MLP + U-Net 风格 down/up 通道投影分支），将冻结主干输出映射到任务子空间，用 center loss 把当前任务特征拉近对应原型、用 Lstable 把从伪高斯分布采样的旧类别伪特征推远，从机制上缓解不同任务间的决策边界重叠。
- **提出 SRE（Subspace Representation Extension）子空间表示扩展**：为每个任务添加一条与主干完全解耦的低秩子网络（每 transformer block 用两个 d3=16 的低秩 MLP 模块近似 attention 与 FFN 的映射关系），与主干中间特征拼接后送入 SPP，在不修改预训练网络的前提下显著增强对新任务的表达能力。
- **提出 DR（Distance Regularization）距离正则化**：通过任务平均距离约束 Ltask 和类别平均距离约束 Lclass 两个正则项，强制每个任务子空间内特征到原型的平均距离保持一致，缓解模型对某些新任务子空间过拟合（表现为该子空间平均距离明显偏小）导致的跨任务偏好。
- **整体方案 MILES 一次前向、exemplar-free、冻结主干**：推理阶段主干只前向一次，所有 SRE 子网络复用同一组中间特征，相比 EASE 等每任务分支都要走一次主干的方法大幅降低推理代价。
- **六个基准上全面刷新 SOTA**：CIFAR-100、CUB-200、Omnibenchmark、ImageNet-A、ImageNet-R、FOOD-101，在 T=10 与 T=50 共 12 个设置下平均准确率全部最优，且遗忘率多数也是最低。

## Key Contributions

- **Subspace Prototype Projection (SPP)**: a lightweight, zero-initialized adapter (MLP + progressive channel down/up branch, e.g., 768→192→48→12→48→192→768) cascaded after the frozen backbone that projects features into a per-task subspace; trained with a center loss pulling intra-task features to their prototypes and a stability loss pushing pseudo-sampled old-class features (drawn from per-class pseudo-Gaussians G_c(p_c, σ_c)) away from the new task's prototypes, directly attacking the overlapping-decision-boundary problem.
- **Subspace Representation Extension (SRE)**: a low-rank (d3=16) sub-network decoupled from the frozen backbone that approximates each transformer block's attention+FFN mapping with two MLP modules per block; it consumes intermediate backbone features and the input, and its output is concatenated with the backbone feature before the SPP adapter, augmenting representational capacity without touching pretrained weights.
- **Distance Regularization (DR)**: two regularization terms (Ltask keeping projected-to-irrelevant-task distances consistent, and Lclass keeping intra-task feature-to-prototype average distance aligned with the first task's), preventing the model from collapsing certain task subspaces and producing task bias.
- **Exemplar-free, single-forward-pass, frozen-backbone design**: at test time the backbone runs once, all SRE sub-networks reuse the same intermediate features, so MILES avoids the per-task backbone forward that bloats EASE/DER inference cost.
- **Comprehensive SOTA across six benchmarks**: best average accuracy and mostly lowest forgetting in all 12 settings (T=10 and T=50 on CIFAR-100, CUB-200, Omnibenchmark, ImageNet-A, ImageNet-R, FOOD-101) with ViT-B/16-IN21K.

## Methodology

MILES is built on three observations: (1) in a pre-trained ViT's feature space, unseen classes already cluster rather than disperse (verified quantitatively in Tab. VI: intra-class distance is consistently smaller than inter-class distance for pretrained ResNet50, ViT-B/16, Swin-B, ConvNeXt across six datasets, whereas random weights show overlapping confusion intervals); (2) a frozen-backbone metric-learning baseline (compute prototypes, classify by Manhattan distance) therefore works but is limited by overlapping decision boundaries; (3) embedding trainable modules into the backbone complicates architecture substitution and inflates inference cost linearly with task count.

**Pipeline.** When task t arrives:
1. Compute prototypes {p_{M+1}, …, p_{M+K_t}} of the new categories using the frozen backbone.
2. Add one SPP adapter A_t and one SRE sub-network S_t (the only trainable parameters for this task).
3. For each epoch: forward through backbone + S_t + A_t; compute total loss; update A_t and S_t; recompute new-task prototypes.
4. Update the running class-average distance l_old.

**SPP adapter** (Eq. 4–8): A(f) = f + f_mlp + f_φ, where the down/up branch φ progressively compresses channels (e.g., 768→192→48→12→48→192→768) and the whole adapter is zero-initialized so A_t(f) = f at the start of training, giving identity-init stable adaptation. Loss is L_center (Eq. 9, Manhattan distance of projected features to their prototype) plus β·L_stable (Eq. 10, increases distance from pseudo-sampled old-class features to the new task prototypes).

**SRE sub-network** (Eq. 13–17): the pretrained backbone F = {B_0, …, B_L} runs normally; the t-th SRE sub-network S_t = {s_t^1, …, s_t^L} injects two low-rank MLP modules per block, the first mimicking the attention module and the second the FFN, with the attention output wired between them. SRE output is concatenated with the backbone's final feature and fed into A_t. Only the t-th SRE sub-network is trained at task t; all are used at test time.

**Distance Regularization** (Eq. 18–24): L_task compares the average distance of a random previous task's features projected into the current subspace against the average distance of pseudo-features drawn from that task's pseudo-Gaussian to the current task's prototypes; L_class keeps the per-batch average feature-to-prototype distance aligned with l_old (running estimate updated via Eq. 25–26). Final loss: L_total = L_center + β·L_stable + γ·(L_task + L_class), with β=0.1, γ=0.001 as defaults.

**Inference (Algorithm 2)**: single backbone forward; for each task i, compute l_c = |A_i(F(x) ⊕ S_i(x)) − p_c| for every category c in Y_i; pick the smallest distance across all tasks. Manhattan distance is used throughout.

## Results & Highlights

All results use ViT-B/16-IN21K, exemplar-free, averaged over 3 runs with random seed 1993.

- **CIFAR-100 (Tab. II)**: T=10 — MILES 93.93% / F=3.12 vs runner-up DGR 93.05% / F=4.85 (+0.88% accuracy, −1.73 forgetting); T=50 — 91.90% / F=6.03 vs TUNA 91.08% / F=4.17 (+0.82% accuracy). After all tasks, the final-step accuracy gap over the runner-up widens to +1.73% (T=10) and +3.71% (T=50).
- **CUB-200 (Tab. II)**: T=10 — 93.02% / F=4.57 vs DGR 91.15% / F=7.61 (+1.87%); T=50 — 92.86% / F=4.94 vs TUNA 92.65% / F=5.09. Final-step accuracy edges out the runner-up by +3.04% (T=10) and +2.11% (T=50).
- **Omnibenchmark (Tab. II)**: T=10 — 85.02% / F=4.04 vs DGR 83.14% / F=6.46 (+1.88%); T=50 — 84.84% / F=5.45 vs DGR 82.70% / F=8.30 (+2.14%). Final-step gains of +2.42% / +2.85%.
- **ImageNet-A (Tab. III)**: T=10 — 62.33% / F=2.92 vs TUNA 61.34% / F=6.71 (+0.99%); T=50 — 62.67% / F=2.45 vs TUNA 62.54% / F=5.29. Final-step gains of +2.83% / +4.28% — the largest final-step margins in the paper.
- **ImageNet-R (Tab. III)**: T=10 — 84.90% / F=5.87 vs DGR 83.85% / F=7.73 (+1.05%); T=50 — 81.90% / F=9.96 vs DGR 80.57% / F=13.50 (+1.33%). Final-step gains of +2.20% / +3.54%.
- **FOOD-101 (Tab. III)**: T=10 — 93.23% / F=3.96 vs DGR 91.15% / F=5.81 (+2.08%); T=50 — 90.75% / F=5.74 vs DGR 89.21% / F=8.54 (+1.54%).
- **Ablation (Tab. IV, T=10)**: Going from baseline → +SPP-MLP → +SPP-Bottleneck → +SPP-Full → +SRE → +DR, CIFAR-100 climbs monotonically from 87.92 / F=10.46 to 93.89 / F=3.20; CUB-200 from 87.62 / F=7.14 to 93.13 / F=4.55; FOOD from 86.92 / F=9.21 to 93.28 / F=3.92. Adding SRE alone (SPP+SRE) gives the largest single jump on most datasets (e.g., CIFAR +1.20% over SPP-Full, ImageNet-R +1.07%), and DR contributes a further +0.9–1.5% on CIFAR/CUB/FOOD.
- **Efficiency (Fig. 8c/8d, T=10)**: MILES uses ~373 MB memory (FP32) — comparable to the smallest method FeCAM (348 MB) — and 16.92 GMACs per inference, the lowest among all compared CIL methods; this is markedly cheaper than EASE (87.4M params but a separate backbone forward per task, pushing MACs to 129.9G) and DER (129.9 GMACs).
- **Motivation verification (Tab. VI)**: across four backbones × six datasets, pretrained weights give intra-class distance strictly smaller than inter-class distance (e.g., ViT-B/16 on CIFAR: intra 0.113±0.013 vs inter 0.156±0.009), while random weights show large overlap — empirically justifying the metric-learning premise.
- **Robustness (Fig. 8a/8b, Tab. V)**: over 5 learning orders on ImageNet-R the curve barely fluctuates; across β∈{0.01…0.5}, γ∈{1e-4…0.01}, r∈{2,4,8}, d3∈{4,16,48,96}, accuracy stays within ~1.5% of the default, indicating low sensitivity.

## Critical Assessment

- **Novelty**: Genuine. The combination is well-motivated rather than incremental: the key insight — that pretrained feature spaces already cluster unseen classes (Tab. VI is a nice empirical justification) — lets the authors replace the dominant prompt/adapter-tuning paradigm with prototype-driven metric learning, and the SPP/SRE/DR trio each addresses a concrete failure mode (boundary overlap, limited adapter capacity, inter-task metric imbalance). The zero-init U-Net-style channel projection in SPP and the decoupled-per-task SRE that reuses a single backbone forward are non-obvious design choices.
- **Rigor**: Strong on breadth — six datasets, T∈{10,50}, twelve comparison methods (seven exemplar-free + six exemplar-based), three repetitions with mean±std in ablations, plus learning-order and hyper-parameter sensitivity sweeps. The ablation cleanly decomposes SPP-MLP / SPP-Bottleneck / SPP-Full / +SRE / +DR.
- **Reproducibility**: Good. Backbone (ViT-B/16-IN21K from timm), exemplar sizes (20/category, 2 for ImageNet-A), optimizer (SGD lr=1e-3, momentum 0.9, wd=5e-4, batch 128, 20 epochs/task, cosine annealing), and hyperparameters (β=0.1, γ=0.001, d3=16, r=4) are all stated; Algorithms 1 and 2 make the train/test loop unambiguous. Pseudo-Gaussian σ_c is not fully specified, and the random-task sampling in L_task is described only verbally — these are minor gaps.
- **Significance**: SOTA margins are consistent (every setting, often +1–2% and up to +4.28% final-step on ImageNet-A) while simultaneously achieving the lowest MACs and near-lowest memory — a real efficiency/accuracy win over EASE, CoFiMA, TUNA, DGR.
- **Red flags / concerns**:
  - **Weak baselines on long sequences**: several strong recent methods are reported only at T=10 because their original numbers are "–" at T=50 (BEEF, MTD). The fairest head-to-head at T=50 is effectively against DGR/TUNA, so the T=50 SOTA claim is somewhat thinner than T=10.
  - **Self-reported reproduction**: "we reproduce the comparison methods and MILES with the same hyper-parameters and pretrained weight" — strong if true, but no code/release is mentioned, and reproduction-sensitive methods (L2P, CODA-Prompt at T=50) sometimes underperform their originally reported numbers, which inflates MILES's relative gap.
  - **Pseudo-Gaussian assumption**: L_stable models old-class feature distributions as Gaussians parameterized only by prototype and variance; the validity of this assumption for long task streams (T=50, 6 classes/task on CIFAR) is not tested directly (no per-class kurtosis/skewness analysis).
  - **Cost reporting caveat**: memory is reported excluding exemplar-set memory, which is fair for exemplar-free MILES but makes the ~373 MB headline not directly comparable to exemplar-based FOSTER/DGR that the paper claims to beat on memory.
  - **Missing ablations**: no ablation isolating L_task vs L_class (they are bundled in DR); no ablation on the choice of distance metric (Manhattan vs Euclidean vs cosine) despite it being central; no comparison to other PEFT styles (LoRA, SSF, AdaptFormer) inside the SPP slot.

## Strengths

- Consistent SOTA on every one of 12 settings across six diverse datasets (low-res CIFAR, fine-grained CUB, large-scale Omnibenchmark, OOD ImageNet-A/R, fine-grained FOOD).
- Lowest inference MACs (16.92 G) and near-lowest memory among all compared methods, with a single backbone forward — a practical deployability win.
- Clean, well-motivated method: each component targets a specific diagnosed failure mode, and the ablation shows monotone gains from baseline → SPP → SRE → DR.
- Empirically grounds the core motivation (Tab. VI intra/inter-class distance analysis across four backbones) rather than asserting it.
- Strong robustness evidence: 5 learning orders, large β/γ/r/d3 sweeps all within ~1.5% of default.

## Limitations

- Authors explicitly note (Sec. V) the method assumes the pretrained backbone provides transferable representations and the downstream data stays largely in-distribution; under real-world OOD style/acquisition/semantic shifts, the prototype structure and per-task subspaces may break.
- Per-task parameter and sub-network growth: each new task adds a fresh SPP adapter and SRE sub-network, so total parameter count grows linearly with T (mitigated by low rank but not eliminated).
- No code/release mentioned.
- Pseudo-Gaussian old-class modeling and bundled L_task/L_class regularization are not individually validated.
- T=50 comparisons are thinner because several recent baselines were not run at T=50.
- Experiments are image classification only; no evaluation on NLP, audio, or detection/segmentation CIL.

## Relevance

对持续学习与预训练模型适配方向高度相关：MILES 直接解决 PTM-based 类别增量学习中"性能-计算开销"折中问题，对关注 vision-language 预训练、参数高效微调（PEFT）、记忆与遗忘权衡的研究者具有方法论参考价值；其 exemplar-free、单次前向的特性对部署受限场景下的可持续学习系统尤其有借鉴意义。

## Questions for Further Exploration

- How does MILES behave under genuine OOD task streams (e.g., pretraining on ImageNet-21K, then incrementing on medical, satellite, or sketch domains)? The authors flag this as future work but provide no experiment.
- Can the per-task SRE sub-networks be shared/merged (e.g., via task-arithmetic or model merging) to halt the linear parameter growth at long T?
- How sensitive is the pseudo-Gaussian L_stable assumption to heavy-tailed or multi-modal class distributions? A per-class Gaussianity test per dataset would clarify the regime where the trick helps.
- Would replacing Manhattan with cosine/Mahalanobis distance (as in FeCAM) inside the SPP subspace further close the gap to joint training (currently 1–9% below A_joint depending on dataset)?
- Can L_task and L_class be disentangled — is the task-level constraint or the class-level constraint doing the heavy lifting in DR?
- Does the single-forward-pass efficiency advantage survive when the backbone is larger (ViT-L, ViT-H) or when using a CLIP/DINOv2 backbone whose features may cluster differently than IN21K ViT-B/16?
