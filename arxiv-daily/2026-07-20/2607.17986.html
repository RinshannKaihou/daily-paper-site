Self-State Attacks on Self-Hosted AI Agents: How Far Can OS Defenses Go?

Yimeng Chen · Nathanaël Denis · Roberto Di Pietro · Jürgen Schmidhuber
KAUST · The Swiss AI Lab IDSIA — USI/SUPSI

KEY FINDING The one sentence

Self-state attacks A(R)⊆L(R) structurally limit OS defenses — the best layered stack closes 19 of 23 attack cells, leaving four Memory writes indistinguishable from legitimate traffic.

Key Metric
19/23
attack cells closed by L3 perms + B2 detection (τ=2.0) + 15-event backup. L3 balanced point: protection 0.61 / functionality 0.70.
Why This Matters

The V/C/I workload-conditioned detection framework and L3+B2+backup stack directly guide self-hosted agent file-state protection, monitoring, and rollback design.

Critical Edge

A single LLM (Gemini 3 Flash) on a stripped-down harness is tested; semantic-layer and eBPF defenses (AgentSight, GuardAgent) appear only as positional comparison — no empirical head-to-head.

Methodology

§ 01 / FORMAL + EMPIRICAL

A formal framework paired with empirical evaluation on an OpenClaw-derived agent driven by Gemini 3 Flash.

Self-hosted agents (Claude Code, OpenClaw) keep their state as ordinary markdown/JSON on the local filesystem — instruction (SOUL.md, IDENTITY.md, AGENTS.md), memory (MEMORY.md, daily logs), configuration (openclaw.json, credentials/.env). The LLM reads these directly as prompt context, so any write shapes future behavior.

Every attack reduces at the VFS boundary to a feature tuple <target-path, op_type, δs, mode-delta, Δt> that any kernel monitor (inotify/fanotify/auditd) can observe. The four-axis space:

  • D1 Target — Instruction / Memory / Config
  • D2 Mechanism — M1 Modify, M2 Add, M3 Delete, M4 Deny/chmod
  • D3 Granularity — G1 whole-file → G4 ≤4-byte minimal
  • D4 Temporal — T1 single-shot, T2 burst, T3 quiet-gap, T4 slow-drip

This yields a 23-cell × 43-instance matrix. Three structural limits follow analytically: prevention hits A(R)⊆L(R) indistinguishability; detection sees "that" not "whether" (event metadata, no content); recovery inherits detection as precondition and pays data-loss ∝ backup interval.

The workload bridge reframes "globally indistinguishable" as a per-cell, per-profile boundary — fitting per-(file, op_type) baselines on (log Δt, sign(δs), log|δs|) and scoring events as a z-score.

Strengths

§ 03
  • Crisp A(R)⊆L(R) formalization derives the three structural limits directly — mathematically explicit, not hand-waved.
  • The V/C/I three-regime classification reframes "indistinguishable" as a per-profile, per-cell boundary instead of a global defeat.
  • Brutally honest reporting: net cell count understates B1-vs-B2 bidirectional migration; sparse-profile FPR admitted as data-bound.
  • Clean artifact: deterministic (op_type, δs, mode-delta) specs decoupled from the session driver; one-command repro, zero third-party deps.
  • Table 9 per-cell cascade is directly actionable — practitioners know which layer closes which cell under which profile.

Limitations

§ 04
  • Only one LLM (Gemini 3 Flash) on a stripped-down openclaw-core — no production OpenClaw or Claude Code.
  • W3/W4 workloads are author-constructed and dominate trace volume; I-cell conclusions are sensitive to that design.
  • No empirical comparison vs. semantic-layer / eBPF defenses (AgentSight, GuardAgent, AgentSpec) — positional only.
  • Slow-drip (T4) cumulative-effect attacks excluded by construction; structurally out of reach per-event.
  • High FPR (0.33–0.36) on sparse profiles needs more training data or human-in-the-loop gating.
  • Cross-layer recovery (memory → instruction → config) left open; only coarse snapshot rollback evaluated.
