Towards Agentic Agent-based Models: Feasibility, Performance, and Statistical Model Checking

Blando, Guerrazzi, Porcedda, Squillace, Tschaikowski, Vandin | Sant'Anna · CentraleSupélec · Sapienza · DTU · U. Pisa | Preliminary / first step
LLM-in-the-loop ABM Mesa Qwen3.5 · Ollama MultiVeStA · SMC Schelling segregation tool-calling
▍ Key Finding

Swapping one agent's rule for an LLM leaves Schelling dynamics statistically unchanged — but runtime explodes 4,458×.

Key Metric
4,458×
2 s → 8,917 s total SMC runtime.
1,220 LLM calls (61 × 20 sims)
@ ~7.3 s / call on DGX Sparx.
Why This Matters

A reproducible Mesa + LangChain-Ollama + MultiVeStA recipe to instrument LLM-in-the-loop multi-agent simulations and quantify their reliability, runtime, and statistical impact.

Critical Edge

The statistical-equivalence claim is almost vacuous: one LLM agent among ~320 cannot move the aggregate trajectory even if it misclassifies every neighbor.

01 / METHODOLOGY

One surgical substitution, fully controlled

In Mesa's Schelling model, exactly one agent's count_similar_neighbors is replaced by an LLM tool-calling classifier. Happiness, movement, scheduler and grid stay byte-for-byte identical — the LLM cannot move agents or set happiness, only mediate the local classification.

Type-0 agents say "Sky is beautiful.", type-1 say "Fire is beautiful." The LLM (ChatOllama, temperature 0, bind_tools) must emit one tool call per neighbor — increment_similar_count or increment_different_count.

Robustness is probed across Qwen3.5 0.8b / 2b / 4b / 9b on four prompts of rising difficulty (M1–M4), each run 3× blue + 3× red, labelled correct / semantic error / arity error / stuck.

MultiVeStA then estimates transient happy-agent-ratio trajectories (steps 1–60) at 95% confidence, CI width ≤ 0.1, with point-wise t-tests between original and LLM-extended variants.

02 / RESULTS

Same dynamics — at 4,458× the cost

  • Trajectories overlap. Happy-ratio goes ~0.65 → ~1.0 over 60 steps for both variants; visually indistinguishable.
  • Point-wise t-tests across all 60 steps: no significant differences — under this minimal intervention, the LLM does not change the dynamics.
  • Small models break. 0.8b gets stuck on trivial prompts and makes semantic errors; 2b fails multi-neighbor tool-call generation (M4).
  • 4b / 9b pass all unit checks in both roles; 4b chosen for SMC (~11.7 s/call vs 15.6 s for 9b, equal correctness).
  • Tight budget. MultiVeStA needs only 20 sims/variant to hit 95% / width-≤0.1 — indicating low model variance.
  • Runtime blow-up. 2 s → 8,917 s (~4,458×). 61 invocations × 20 sims × ~7.3 s = 8,917 s.

03 / STRENGTHS ↔ LIMITATIONS

A clean experiment with a foregone conclusion

Strengths

  • Surgically minimal design. Subclass + override only count_similar_neighbors — the rest of the ABM is provably unchanged.
  • Reusable taxonomy. correct / semantic / arity / stuck is a useful vocabulary for LLM-tool-call behavior in simulation loops.
  • Honest framing. Repeatedly calls results "preliminary"; does not oversell the equivalence finding.
  • Transparent cost math. 61 × 20 × 7.3 s = 8,917 s lets practitioners extrapolate to their own settings.
  • Mature SMC tool. MultiVeStA gives the statistical guarantees real teeth rather than an ad-hoc harness.

Limitations

  • Foregone equivalence. 1 LLM agent among ~320 has essentially no power to detect per-agent deviations; "no change" is structural.
  • Contrived semantics. "Sky/Fire is beautiful" is solvable by a keyword matcher — it probes tool-call fragility, not LLM reasoning.
  • Model identity ambiguity. "Qwen3.5" labels aren't grounded in the cited Qwen3 tech report; quantization & Ollama version unreported.
  • Point-estimate runtime. Single M4 / single DGX Sparx, no GPU/CPU split, no batching — the 4,458× is a one-shot number.
  • Single point in parameter space. One grid, one homophily, one density — no sweep; no code release for the new class.
04 / OPEN QUESTIONS

Where this breaks next

How do dynamics change at 10% / 50% / 100% LLM agents — is there a phase transition in emergent segregation?

Would a noisy symbolic baseline with the 0.8b error rate reproduce "no statistical change", or do LLM errors correlate differently?

Does the picture survive harder semantics — free-form descriptions, ambiguous words, multilingual prompts?

Can batching / async calls recover any of the 4,458× — and what's the min budget for a target SMC width?

Does the 0.8b→9b robustness ranking replicate across Llama-3.2 / Phi-3 / Gemma-2 at matched sizes?

Are the t-test outcomes sensitive to homophily & grid size — would tighter settings surface a masked difference?
