Key Contributions

  • Introduces KVDrive, a holistic multi-tier KV cache management system spanning GPU memory, host DRAM, and SSD for long-context LLM inference
  • Adapts cache management to attention behavior to maximize reuse and minimize redundant data movement across tiers
  • Restructures the decoding pipeline to overlap I/O-bound and CPU/GPU compute-bound stages, eliminating resource stalls
  • Harmonizes cross-tier data movement to unlock scalable inference far beyond GPU and DRAM capacity limits
  • Achieves up to 1.74x higher throughput compared to state-of-the-art offloading systems while preserving accuracy

Methodology

KVDrive departs from prior work that focuses on increasing attention sparsity (algorithmic approaches). Instead, it takes a systems-level approach with three core mechanisms:

  1. Attention-aware cache management: The system profiles attention patterns during inference and adapts KV cache placement across the three memory tiers (GPU, DRAM, SSD) to maximize cache hit rates and minimize unnecessary data transfers.
  2. Pipelined decoding schedule: The decoding pipeline is restructured so that I/O operations (fetching KV entries from slower tiers) overlap with compute operations (attention calculation, FFN). This eliminates pipeline bubbles caused by waiting for data to arrive from host memory or SSD.
  3. Cross-tier coordination: Data movement between GPU, DRAM, and SSD is orchestrated to balance bandwidth utilization across all tiers, preventing any single tier from becoming a bottleneck as context length and batch size scale.

The system is implemented as a fully functional prototype and evaluated on long-context benchmarks with popular LLMs.

Results & Highlights

  • Up to 1.74x throughput improvement over state-of-the-art KV cache offloading systems
  • Preserves model accuracy — no degradation from the multi-tier caching strategy
  • Enables long-context inference at scales that exceed both GPU memory and host DRAM capacity by leveraging SSD as a third tier
  • Demonstrates that systems-level optimization (pipeline restructuring, tier coordination) can yield larger gains than pushing sparsity further

Critical Assessment

  • Novelty: Moderate-to-high. Multi-tier KV caching with SSD is not entirely new (e.g., vLLM, CacheGen have explored similar ideas), but the holistic combination of attention-aware placement + pipeline restructuring + cross-tier coordination appears novel as an integrated system.
  • Rigor: Claims are supported by throughput numbers and accuracy preservation, but the abstract-only analysis limits assessment of experimental design details (baselines chosen, hardware setup, context lengths tested, batch sizes).
  • Reproducibility: A fully functional prototype is claimed but no open-source link is mentioned in the abstract. If code is not released, reproducibility may be limited.
  • Significance: High practical impact if the claims hold — long-context inference is a major bottleneck in production LLM deployments, and the 1.74x improvement is substantial.
  • Red flags: The abstract frames the contribution as purely systems-level, but the actual performance gain may depend heavily on the specific attention sparsity patterns of the workloads tested. Generalization across diverse model architectures and attention mechanisms (e.g., GQA, MLA) is unclear without reading the full paper.

Strengths

  • Addresses a real and growing bottleneck in production LLM inference
  • Clean problem formulation: shifting from "push sparsity harder" to "manage the system better"
  • Multi-tier approach is practical and aligns with real hardware configurations
  • Pipeline overlap is a well-understood systems technique applied effectively to a new domain

Limitations

  • SSD latency is orders of magnitude higher than GPU memory; the abstract does not quantify how often SSD is actually accessed during decoding or the tail-latency impact
  • Unclear how the system handles dynamic batching and prefill-decode interleaving in production serving scenarios
  • The 1.74x figure likely represents a best-case scenario; typical production workloads with mixed sequence lengths may see smaller gains
  • Attention-aware cache management may not generalize well to models with GQA, MLA, or other non-standard attention mechanisms where KV cache structure differs
