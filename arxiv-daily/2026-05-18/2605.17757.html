OSCAR: Offline Spectral Covariance-Aware Rotation for 2-bit KV Cache Quantization

arXiv: 2605.17757

Date: 2026-05-18

https://arxiv.org/abs/2605.17757

⚠️ Abstract-only summary
一句话总结： OSCAR 通过离线估计注意力感知的协方差结构来推导固定旋转和裁剪阈值，实现了高质量且可部署的 2-bit KV Cache 量化，在 Qwen3-4B/8B/32B 及 GLM-4.7 等模型上接近 BF16 精度，同时减少约 8 倍 KV Cache 内存并提升吞吐量。
TL;DR: OSCAR is a 2-bit KV cache quantization method that uses offline-estimated attention-aware covariance structures to derive fixed rotations and clipping thresholds, paired with a custom INT2 attention kernel compatible with paged KV-cache serving (SGLang/vLLM). It reduces the BF16 accuracy gap to 1.42-3.78 points on Qwen3-4B/8B (where naive rotation INT2 collapses to ~0), scales to 358B-param models with near-BF16 parity, and achieves ~8x KV-cache memory reduction with up to 7x throughput improvement.

核心贡献

  • 提出离线估计注意力感知协方差结构的方法，用协方差结构推导固定旋转矩阵和裁剪阈值，使 KV 量化与注意力实际消费的统计结构对齐。
  • 开发了完全可部署的 INT2 注意力内核，兼容分页 KV Cache 服务（paged KV-cache）和融合内核管线，可无缝集成到 SGLang 和 vLLM 等现代 LLM 推理框架。
  • 在 Qwen3-4B-Thinking、Qwen3-8B/32B、GLM-4.7（358B 参数）以及长上下文 RULER-NIAH（128K）任务上验证了方法的有效性，朴素旋转 INT2 几乎完全失效而 OSCAR 仍保持鲁棒。
  • 系统层面实现约 8 倍 KV Cache 内存缩减，大 batch 下吞吐提升最高 7 倍，单 batch 解码加速最高 3 倍。

Key Contributions

  • Introduces offline attention-aware covariance estimation to derive fixed rotations and clipping thresholds for INT2 KV cache quantization, aligning quantization with the statistical structures that attention actually consumes.
  • Develops a fully deployable INT2 attention kernel compatible with paged KV-cache serving and fused kernel pipelines, enabling seamless integration into SGLang and vLLM.
  • Demonstrates robust performance on reasoning models (Qwen3-4B-Thinking-2507, Qwen3-8B, Qwen3-32B, GLM-4.7 at 358B params) where naive rotation INT2 collapses to near-zero accuracy while OSCAR maintains near-BF16 parity.
  • Achieves ~8x KV-cache memory reduction, up to 7x throughput improvement at large batch sizes, and up to 3x decoding speedup for batch-size-1 over BF16.

Methodology

OSCAR operates in two phases:

  1. Offline calibration: Estimates attention-aware covariance structures from calibration data. These covariance matrices capture the statistical properties of key-value pairs as consumed by the attention mechanism, not just their marginal distributions. The covariance is then used to derive (a) a fixed rotation matrix (spectral decomposition of the covariance) and (b) per-channel clipping thresholds optimized for quantization.
  1. Online serving: Applies the pre-computed rotation and clipping during KV cache storage, quantizing to INT2. A custom INT2 attention kernel performs rotated attention computation directly in 2-bit precision, maintaining compatibility with paged KV-cache management used in modern serving frameworks.

The key insight is that Hadamard transforms reduce outliers but are data-independent, whereas OSCAR's rotations are data-aware and aligned with downstream attention patterns, making ultra-low-bit quantization viable.

Results & Highlights

  • On Qwen3-4B-Thinking-2507: OSCAR reduces BF16 accuracy gap to 3.78 points (5 tasks, reasoning traces up to 32k tokens), while naive rotation INT2 collapses to ~0.
  • On Qwen3-8B: BF16 gap reduced to just 1.42 points.
  • Scales to Qwen3-32B and GLM-4.7 (358B params) with effectively on-par BF16 performance.
  • On RULER-NIAH long-context benchmark up to 128K tokens, OSCAR remains robust on both Qwen3 models while naive rotation INT2 fails.
  • System-level: ~8x KV-cache memory reduction, up to 7x throughput at large batch sizes under same memory budget, up to 3x batch-size-1 decoding speedup.

Critical Assessment

  • Novelty: High. The idea of using attention-aware covariance for rotation design (rather than generic Hadamard transforms) is a meaningful algorithmic advance. The co-design of quantization method and kernel is also notable.
  • Rigor: Reasonable from the abstract. Evaluation covers multiple model sizes (4B to 358B), reasoning tasks, and long-context scenarios. The comparison against naive rotation INT2 (showing collapse) is a strong baseline demonstration.
  • Reproducibility: The mention of SGLang/vLLM integration and custom kernels suggests the system could be released, but kernel-level work is hard to reproduce without open-source code. Unclear from abstract whether code will be released.
  • Significance: Very high if claims hold. 2-bit KV cache quantization at near-BF16 quality would be a major practical advance for LLM serving, especially for long-context reasoning models.
  • Potential concerns: (1) The offline calibration step requires representative data — sensitivity to calibration data distribution is not discussed. (2) Comparison to other non-trivial INT2 methods (beyond naive rotation) needs verification. (3) The "near BF16" claim on 358B params needs scrutiny on how many tasks were evaluated.

Strengths

  • Addresses a practically important problem (KV cache memory bottleneck for long-context serving).
  • Co-designs algorithm and system kernel for deployability.
  • Strong empirical results across model scales and task types.
  • Long-context robustness demonstrated up to 128K tokens.

Limitations

  • Offline calibration introduces an additional preprocessing step; sensitivity to calibration data choice is not quantified in the abstract.
  • Custom INT2 kernel may limit portability across hardware platforms.
  • The abstract does not compare against other state-of-the-art low-bit KV cache quantization methods (only naive rotation as baseline).
  • Overhead of the rotation itself during serving is not quantified.

Relevance

OSCAR 直接解决了大语言模型长上下文推理中的 KV Cache 内存瓶颈问题，提出的方法与 LLM 高效推理和量化部署高度相关，对实际推理系统优化有直接参考价值。

Questions for Further Exploration

  • How does OSCAR compare to other recent INT2/INT3 KV cache quantization methods (e.g., KIVI, KVQuant) beyond naive rotation baselines?
