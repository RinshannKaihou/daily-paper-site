The Expressive Power of Low Precision Softmax Transformers with (Summarized) Chain-of-Thought

arXiv: 2605.18079

Date: 2026-05-18

https://arxiv.org/abs/2605.18079

一句话总结： 本文证明使用softmax注意力、低精度激活值（bfloat16级别）且深度和宽度按上下文长度对数增长的Transformer解码器，可以通过Chain-of-Thought模拟图灵机；使用Summarized CoT时模型规模仅需按空间界而非时间界对数缩放，实验表明增加模型规模比增加精度更能改善长上下文推理能力。
TL;DR: This paper proves that standard softmax transformer decoders with low-precision rounding (bfloat16-level) and logarithmic depth/width can simulate Turing machines via CoT, and more efficiently via Summarized CoT where model size scales logarithmically in space rather than time. Sudoku experiments confirm that scaling model size improves long-context reasoning while increasing precision from 16-bit to 32-bit has negligible effect.

核心贡献

  • 证明了使用softmax注意力、激活值和注意力权重舍入的Transformer解码器在深度和宽度对数增长时，可以通过CoT模拟图灵机，填补了理论模型与实际模型之间的差距。
  • 提出了一种从hardmax到softmax的转换方法：构造具有三元激活值（{−1, 0, 1}）和良好分离注意力分数的hardmax Transformer，再通过适度的query/key缩放因子转换为等价的softmax Transformer。
  • 证明使用Summarized CoT时，模型规模仅需按图灵机的空间界（而非时间界）对数缩放，效率显著提升。
  • 实验验证了理论预测：在Sudoku推理任务上，增加模型深度有效延长CoT的有效上下文长度，而将精度从bfloat16提升到fp32几乎没有效果。

Key Contributions

  • Proves Turing completeness of standard softmax transformers with activation and attention weight rounding at sub-logarithmic precision (constant mantissa bits, O(log log log) exponent bits for activations), where depth and width scale logarithmically with context length.
  • Develops a construction pipeline: hardmax transformers with ternary activations and well-separated attention scores → softmax conversion via moderate scaling factor c = O((log l)^(3/4)) → validated with additional denoising MLPs to handle rounding errors in attention weights.
  • Shows Summarized CoT (SCoT) enables model size to scale logarithmically in a space bound rather than a time bound, with maximum segment length O(s_M(w)) — a significant efficiency improvement for space-bounded computations.
  • Demonstrates that bfloat16 precision suffices for the constructions up to context sizes of ~10^38, while prior log-precision constructions can break at small context lengths even at 32-bit precision.
  • Provides empirical evidence on Sudoku reasoning that scaling model size (depth 6→8) improves long-context CoT performance, while scaling precision (bfloat16→fp32) has negligible effect — consistent with the paper's theoretical predictions and inconsistent with prior high-precision results.
  • Shows SCoT models generalize from easy to ultra-hard Sudokus: a model trained on CoT length ≤ 2^12 solves 92/100 hardest test puzzles requiring 1–9 million CoT tokens.

Methodology

The paper follows a two-stage construction approach:

  1. Hardmax Transformer Construction (Section 3): The authors construct hardmax transformers with ternary weights and activations ({−1, 0, 1}) that simulate Turing machines. The representations are divided into registers encoding positions (binary), states, symbols, and flags. Attention heads extract specific registers from previous tokens, while MLPs perform operations like copying, incrementing, and decrementing binary values. A novel head-position reconstruction technique and binary search for finding the latest matching token are used. Three constructions are provided: regular language recognition (Proposition 3.1), CoT Turing machine simulation (Theorem 3.2), and SCoT Turing machine simulation (Theorem 3.3).
  1. Softmax Conversion with Rounding (Section 4): The hardmax constructions are converted to softmax by scaling query/key projections by a factor c. The error between softmax and hardmax is bounded by 2ne^{-β} where β is the attention score separation. Since ternary queries/keys guarantee minimum separation of 1/√d_k, moderate scaling suffices. For attention weight rounding, additional denoising MLPs are added that map perturbed activations back to correct ternary values via a piecewise linear function (Figure 4). Each original layer is replaced by 2 layers in the softmax variant.
  1. Empirical Validation (Section 5): Small transformers are trained to imitate a depth-first search solver on the sudoku-extreme dataset. CoT and SCoT models of varying depths and precisions are compared on puzzles grouped by required CoT length.

Results & Highlights

  • Precision requirements: Activations need only constant mantissa bits and O(log log log n̂/t̂/ŝ) exponent bits. Attention weights need O(log log t̂/ŝ) exponent bits and ≥4 mantissa bits. bfloat16 suffices up to context sizes ~10^38.
  • SCoT efficiency: For SCoT, depth and width are O(log ŝ) (space bound), compared to O(log t̂) (time bound) for standard CoT. This is a significant improvement since ŝ can be much smaller than t̂.
  • Scaling factor: The required query/key scaling factor is c = O((log l)^(3/4)), much more moderate than the polynomial-in-context-length scaling that prior high-precision constructions would require.
  • Sudoku CoT results: A depth-6 model (d=512, 8 heads) trained with SCoT achieves 99.5% accuracy. The same model size with CoT fails on puzzles requiring >2^12 CoT tokens. Increasing depth to 8 fixes the CoT failure; increasing precision to fp32 does not.
  • SCoT generalization: A d=768, L=12 SCoT model trained on puzzles with CoT length ≤ 2^12 achieves 99.96% overall accuracy and solves 92/100 hardest puzzles (1–9M CoT tokens required).

Critical Assessment

  • Novelty (High): This is the first work to show Turing completeness of softmax transformers with realistic precision (bfloat16-level) and moderate parameter magnitudes. The ternary-activation intermediate construction and the denoising-MLP technique for handling attention weight rounding are novel technical contributions.
  • Rigor (High): All constructions come with explicit constant bounds, complete proofs in appendices, and mechanically verified implementations. The distinction between O_M(·) and O(·) notation is carefully maintained.
  • Reproducibility (High): Public code repository, detailed experimental setup in appendices, and explicit asymptotic bounds with constants.
  • Significance (Moderate-High): Substantially narrows the gap between theoretical expressivity results and practical transformer models. The key insight — that ternary activations enable efficient error correction under rounding — is both elegant and practically relevant. However, the results are purely existential (expressivity, not learnability), and the logarithmic scaling of depth/width still deviates from fixed-size practical models.
  • Red flags: The empirical evaluation is limited to a single task (Sudoku) with a single algorithm (DFS backtracking). The claim that "expressivity results for softmax transformers with rounding give better qualitative predictions about learnability" is somewhat overstated given this limited evidence. The constructions are not intended to be learned by gradient descent — they are hand-crafted, and the gap between expressivity and learnability remains large.

Strengths

  • First rigorous proof that practical softmax transformers with bfloat16-level precision are Turing complete, using parameter magnitudes and precision that are realistic.
  • The SCoT result showing model size scales logarithmically in space (not time) is a meaningful efficiency improvement with clear practical implications for long-reasoning tasks.
  • Clean construction pipeline (hardmax with ternary activations → softmax conversion → denoising MLPs) that is modular and well-organized.
  • Excellent comparison table (Table 1) situating the results among prior work.
  • The Sudoku experiments, while limited, provide a compelling illustration of the theory-practice alignment.
  • The observation that SCoT enables constant-space generalization to ultra-hard puzzles is practically interesting.

Limitations

  • The constructions are purely existential — there is no guarantee that gradient descent can find these solutions. The authors acknowledge this but the gap between expressivity and learnability remains the elephant in the room.
  • Depth and width still grow logarithmically with context length, whereas practical models have fixed architecture. The claim of "more realistic" models is relative to prior work, not absolute.
  • Empirical evaluation is limited to one task (Sudoku) and one algorithm (DFS). Whether the predictions generalize to other reasoning tasks is unknown.
  • The ternary activation assumption, while enabling the elegant denoising-MLP technique, may not be representative of how trained transformers actually encode information.
  • The hardmax→softmax conversion doubles the number of layers, which weakens the efficiency claims somewhat.

Questions for Further Exploration

  1. Can the ternary-activation constructions be relaxed to allow a larger but still bounded set of activation values, which might be more representative of learned representations?
  2. Does the logarithmic depth/width scaling represent a fundamental lower bound, or can fixed-size softmax transformers with rounding also be Turing complete under some formulation?
  3. How do the theoretical predictions translate to more diverse reasoning tasks beyond Sudoku (e.g., mathematical proof, code generation, planning)?
  4. Can the denoising-MLP technique be adapted to work with quantized weight formats (e.g., INT8, FP4) commonly used in inference?
  5. What is the relationship between the construction's effective attention sparsity (1-sparse) and practical sparse attention methods like FlashAttention — could sparse attention implementations inherently preserve expressivity?
