Gated Delta Networks need Θ(1/√d) learning-rate scaling for gating weights — standard μP breaks, zero-shot transfer requires new derivation

The first complete μP formulation for Gated Delta Networks reveals that data-dependent gating mechanisms violate standard parameterization assumptions — enabling learning-rate transfer from a 21M proxy to 342M models without retuning, cutting the cost of scaling sub-quadratic architectures.

Zero-Shot LR Transfer · AdamW
21M→342M
Learning rate calibrated on the smallest proxy model transfers without retuning across 4 widths (d=256,512,1024,1536). SP and standard μP both fail to transfer.
Key Non-Standard Scaling · SGD
1/√d
Gating weight matrices W_α and W_β require Θ(1/√d) LR scaling under SGD — vs. Θ(1) in standard μP. Scalar gating params a_log need the opposite: Θ(√d)
Scale Gap
342M
Maximum model size validated. μP's core promise is small→large transfer, but the 1B+ regime and long-context (L_eff = O(T)) correction remain unverified
Methodology

Coordinate-Scale Analysis for Gated Architectures

GDN state update
S_t = S_{t-1}(α_t(I − β_t k_t k_t^T)) + β_t v_t k_t^T
α_t, β_t ∈ (0,1) — data-dependent gates
q,k,v L2-normalized → Θ(1/√d) coordinates
  • Forward pass: Cumulative state S_t = Θ(1/√d); readout o_t = S_t q_t → insert √d rescaling before RMSNorm
  • Gating non-Gaussianity handled: Sigmoid activations don't fit Gaussian Tensor Programs — 1st-order analysis shows ∂α/∂z = Θ(1), preserving the scaling argument
  • SGD backward: W_α gradient = Θ(1/√d) → requires η_α = Θ(1/√d); scalar a_log gradient = Θ(1/√d) but enters additively → needs η = Θ(√d)
  • AdamW simplification: Coordinate normalization absorbs the gating magnitude difference — same LR scaling as standard μP

Strengths & Limitations

Theoretical Rigor

  • First complete μP derivation for any gated linear recurrent architecture with full-matrix hidden state
  • Explicit treatment of both AdamW and SGD — reveals that AdamW masks the non-standard scaling, SGD exposes it
  • Dynamic probes validate forward coordinate scales and backward gradient magnitudes — not just loss curves
  • Open-sourced with Amazon Trainium-specific implementation: github.com/lauyikfung/gated_delta_net_mup

Scope Constraints

Critical edge — scaling unproven

Maximum 342M params on 1× H100, 20k steps. μP's value proposition is transfer to billions — this remains unverified. Short-memory assumption (L_eff = O(1)) breaks at long-context α_t → 1, requiring LR / L_eff correction that is undeveloped. Analysis does not extend to Mamba, RWKV, or other linear architectures; each requires separate derivation.

Open Questions

Further Exploration

  1. Does the proposed μP hold at 1B–7B parameters — the scale where GDN's sub-quadratic advantage over Transformers becomes economically meaningful?
  2. When α_t → 1 (long-context regime), does applying LR × (1/L_eff) correction actually stabilize training?
  3. Can this paper's analysis framework (gating Jacobian 1st-order bound) extend to Mamba-2, RWKV-6, or RetNet with delta-rule?
  4. For hybrid GDN + Transformer layers (e.g., Jamba-style), how do the two components' conflicting μP prescriptions get reconciled?
  5. Does optimal learning rate from GDN-μP fall in the same range as Transformer-μP at equal parameter count — enabling joint proxy sweeps?
