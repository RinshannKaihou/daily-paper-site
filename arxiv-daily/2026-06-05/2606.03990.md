> **一句话总结：** 大型神经网络中存在一类跨模型共享的"罗塞塔神经元"，其数量随模型规模呈次线性幂律增长，同时这些神经元随规模增大变得越来越单义、专业化，而非罗塞塔神经元则保持多义混乱状态——两类种群在规模扩大时发生"极化分叉"。

> **TL;DR:** A population of neurons that recur identically across independently trained models (Rosetta Neurons) follows a sublinear power law in count as model size grows, while simultaneously becoming more monosemantic and domain-specialized — a "Neuron Polarization Effect" explained by a capacity-allocation model from superposition theory. A data-filtering case study shows these neurons are selective enough to recover domain-specific corpora with near-oracle accuracy.

---

## 核心贡献

1. **神经元级别的扩展定律**：首次系统证明跨模型共享神经元（Rosetta Neurons）的数量服从次线性幂律 |R| = c·N^α（α ≈ 0.5–0.7），在语言模型（80M–30B 参数）和视觉模型（80M–5B 参数）中均成立，R² ≈ 0.99。

2. **神经元极化效应（Neuron Polarization Effect）**：发现随规模增大，罗塞塔神经元变得更单义（monosemantic），而非罗塞塔神经元保持多义（polysemantic），两类种群在选择性上发生系统性分叉——这是一个可预测、可量化的内部表征结构规律。

3. **解析模型**：基于叠加（superposition）理论建立了容量分配模型，通过特征重要性功率谱和信噪比分析，从理论上推导出次线性幂律和极化效应，将经验观察纳入统一框架。

4. **专业化转移**：展示随规模增大，罗塞塔神经元的激活逐渐集中于代码、数学等专业化文本领域，而非通用散文领域。

5. **应用验证（数据过滤）**：利用单个 JavaScript 选择性罗塞塔神经元从多语言代码语料中过滤 JavaScript 代码，F1 达 0.98（近似 Oracle 的 1.00），继续预训练后困惑度从 3.59 降至 3.02，与 Oracle 过滤（3.01）相当。

---

## Key Contributions

1. **Neuron-level scaling law**: First systematic demonstration that Rosetta Neurons — neurons with shared activation patterns across independently trained models — follow a sublinear power law |R| = c·N^α with α ≈ 0.5–0.7 in both language (80M–30B params) and vision (80M–5B params) models, with power-law fits achieving R² ≈ 0.99.

2. **Neuron Polarization Effect**: As model scale increases, Rosetta Neurons become progressively more monosemantic (selective) while non-Rosetta neurons remain polysemantic, creating a systematic and quantifiable divergence between the two sub-populations.

3. **Analytical model**: A capacity-allocation model grounded in superposition theory derives both the sublinear count scaling and the polarization effect from first principles, using a feature-importance power spectrum and a signal-to-interference isolation score.

4. **Specialization shift**: Rosetta Neurons shift toward specialized domains (code, math) with scale, while non-Rosetta neurons' domain preferences remain stable — indicating that scale expands the shared interpretable frontier into rarer feature territory.

5. **Downstream application**: A single JavaScript-selective Rosetta Neuron achieves F1 = 0.98 for domain data filtering (vs. Oracle F1 = 1.00), and continued pretraining on its selected data reduces test perplexity from 3.59 (random) to 3.02 (Rosetta) vs. 3.01 (Oracle).

## Methodology

### Rosetta Neuron Identification

The paper focuses on MLP neurons in Transformer architectures. For each neuron u = (ℓ, c) in layer ℓ, channel c, the activation at token position t on input x_i is m_t^(ℓ,c)(x_i). Similarity between two neurons u (model A) and v (model B) is measured as Pearson correlation across all aligned token positions over a shared dataset (~10M tokens from The Pile for language; 50K class-balanced ImageNet images for vision):

```
sim(u, v) = (1/N) Σ_{i,t} [(m_t^u(x_i) − μ_u)/σ_u] · [(m_t^v(x_i) − μ_v)/σ_v]
```

A pair is retained as a Rosetta Neuron if it is a mutual nearest neighbor (MNN) under this metric — u's nearest neighbor in B is v, and v's nearest neighbor in A is u (k=1 by default). This MNN filter removes asymmetric/noisy matches.

### Model Families

- **Language**: Pythia, GPT-2, OPT, Qwen-2.5 families; 100M–30B parameters (~40K–2M neurons)
- **Vision (discriminative)**: OpenCLIP, DINOv2, Pixio families; 80M–5B parameters (~40K–600K neurons)
- **Vision (generative)**: Diffusion Transformer (DiT/Pixio architecture) matched against discriminative models

Scaling curves are formed by pairing models at matched scale from two families, then fitting |R| = c·N^α in log-log space via OLS.

### Analytical Model of Sublinear Scaling

The paper adopts a capacity-allocation view. A network with N coordinates represents A(N) > N latent features in superposition. Features have importance w_r ∝ r^{-β} (β > 1). The isolation score s_r ≥ 0 measures how cleanly a single neuron captures feature r (signal-to-interference ratio). The network solves:

```
max_{s_r ≥ 0} Σ_r w_r log(1 + s_r)   subject to   Σ_r s_r ≤ κN
```

This yields a frontier cutoff r_0(N) = Θ(N^{1/β}): features below rank r_0 are cleanly isolated, features above remain in superposition. Rosetta-detectable features (s_r ≥ τ) fall within a frontier r_τ(N) = r_0(N)(1+τ)^{-1/β}, giving:

```
R_τ(N) = Θ(N^{1/β})
```

Since β > 1, this is sublinear in N — matching the empirical exponents α ≈ 0.5–0.7 (consistent with β ≈ 1.4–2.0).

**Polarization prediction**: The average isolation score for Rosetta Neurons grows as s̄_Rosetta(N) = Θ(N^{(β-1)/β}) → ∞, while s̄_non-Rosetta(N) = O(N^{1/β} / A(N)) → 0. Hence the two populations diverge in selectivity as N grows.

### Measuring Selectivity

- **Language (Pythia)**: For each neuron u, compute cosine similarity of output weight W_out^u against each token's unembedding vector W_U[v]; measure excess kurtosis of these similarities. Higher kurtosis = concentration on few tokens = more monosemantic.
- **Vision (OpenCLIP)**: Use GPT-5.4 as a VLM judge: present top-20 activating images with maps/overlays, ask whether activations reflect a single coherent visual feature. Report fraction judged monosemantic.

### Specialization Measurement

For each Pythia model size, collect top-20 activating Pile contexts for each Rosetta Neuron (from OPT-Pythia matching runs). Categorize into five domains: code, math, formal/science, general prose, conversational. Compare Rosetta Neuron top-activation share in each category vs. that category's token frequency in validation set (normalized firing rate). Trend over model size reveals specialization shift.

### Data Filtering Case Study

- **Dataset**: CodeSearchNet (multilingual, ~58K parsed functions, 16M tokens for JavaScript split)
- **Setup**: Fixed 16M token budget; four methods — random, non-Rosetta neuron, Rosetta neuron (JavaScript-selective, from Pythia-6.9B in OPT-Pythia matching), Oracle (ground-truth JS label)
- **Evaluation**: F1 of recovered JS subset; continued pretraining GPT2-1.5B; report test perplexity over 3 runs (95% CI)

## Results & Highlights

### Sublinear Power-Law Scaling (Figure 3)

Across all language model family pairs (OPT-Qwen2.5, Pythia-GPT2, Pythia-OPT, Pythia-Qwen2.5), the number of Rosetta Neurons grows as a power law in model size with fitted exponents consistently in [0.5, 0.7] and R² ≈ 0.99. The same trend holds for all vision model family pairs (Diffusion-DINOv2, Diffusion-OpenCLIP, DINOv2-Pixio, Pixio-Diffusion-OpenCLIP). Collapsed multi-family curves (stricter universality criterion) show the same qualitative behavior.

Critically, **untrained (random) networks show no systematic scaling** (Figure 4): Rosetta Neuron counts remain near 20-30 regardless of model size in both language and vision, confirming the scaling law is a property of learned representations, not an artifact of the matching procedure.

### Neuron Polarization Effect (Figure 6)

- **Language (Pythia)**: Rosetta Neurons show monotonically increasing mean excess kurtosis of vocabulary-space projections with model scale; non-Rosetta neurons remain near zero across all scales. The gap widens substantially from ~100M to ~7B parameters.
- **Vision (OpenCLIP)**: VLM-judged monosemanticity rate increases from ~25% to ~60%+ for Rosetta Neurons as model size grows from ~80M to ~5B parameters; non-Rosetta neurons show a slight *decrease* in monosemanticity rate over the same range.

### Specialization Shift (Figure 7)

Across Pythia model sizes (160M → 12B), Rosetta Neuron firing becomes increasingly concentrated on code and math (specialized text categories, normalized firing rate rising from ~0.8x to ~1.4x corpus baseline) while general prose and conversational categories decline. Non-Rosetta neuron baselines show no corresponding domain shift, confirming this is a scale-dependent property of the shared interpretable population.

### Data Filtering (Table 1)

| Method | F1 (JS recovery) | Test PPL |
|---|---|---|
| Base Model | — | 6.73 |
| Random | 0.06 | 3.59 ± 0.07 |
| Non-Rosetta | 0.09 | 3.23 ± 0.07 |
| **Rosetta** | **0.98** | **3.02 ± 0.05** |
| Oracle | 1.00 | 3.01 ± 0.04 |

A single Rosetta Neuron nearly perfectly recovers the target domain subset and matches Oracle performance on the downstream continued pretraining task.

## Critical Assessment

This paper makes a genuinely novel contribution to mechanistic interpretability and scaling science. The central claim — that Rosetta Neurons follow a sublinear power law and polarize toward monosemanticity — is well-supported empirically with R² ≈ 0.99 fits across many model families and modalities. The analytical model elegantly unifies the phenomena under a capacity-allocation superposition framework and makes testable predictions that are then validated.

The data-filtering case study is the most concretely useful result: demonstrating that a single neuron can serve as a near-oracle domain filter with no labels is practically significant. However, this experiment uses only one domain (JavaScript) and one source model (Pythia-6.9B), which limits generalizability claims.

The key intellectual limitation is scope: Rosetta Neurons capture only one form of shared structure. The paper is explicit that circuits, attention heads, and subspace-level shared computations are not covered. The monosemanticity metrics are also modality-specific proxies (vocabulary kurtosis for language, VLM judgment for vision), introducing non-trivial measurement uncertainty — acknowledged by the authors.

The analytical model is phenomenological rather than mechanistic: it explains *that* scaling polarizes neurons through capacity allocation, but not *how* gradient-based training dynamics produce this allocation. This is the stated key open question.

---

## Strengths

- **Breadth of empirical validation**: Four language model families (80M–30B), five vision model families (80M–5B), multiple training objectives (contrastive, self-distillation, autoencoding, diffusion). R² ≈ 0.99 across all is unusually consistent.
- **Clean null result**: Untrained networks definitively fail to show the scaling trend, ruling out the trivial "larger selection pool" confound.
- **Theory-experiment alignment**: The analytical model derives α = 1/β from first principles; the empirical range α ∈ [0.5, 0.7] implies β ∈ [1.4, 2.0], which is consistent with known feature importance spectrum estimates.
- **Concrete application**: The data-filtering case study provides a practical use case beyond interpretability-for-its-own-sake.
- **Open research direction**: Explicitly bridges macroscopic scaling laws (loss vs. compute) with microscopic neuron-level organization, a largely unexplored connection.

---

## Limitations

- **MLP-only scope**: The analysis is restricted to MLP neurons in Transformer models. Attention heads, residual stream subspaces, and circuit-level shared computation are explicitly excluded. How much of the "shared structure" lives outside MLP neurons is unknown.
- **Single data-filtering experiment**: The JavaScript filtering case study involves one domain, one source model, one dataset. The F1 = 0.98 result is impressive but may not generalize to more ambiguous or compositional domains.
- **Proxy metrics for selectivity**: Vocabulary-space kurtosis (language) and VLM-judged monosemanticity (vision) are reasonable but imperfect proxies. Cross-modal comparisons are not directly comparable.
- **No mechanistic account of dynamics**: The analytical model is a static capacity-allocation equilibrium. How gradient descent produces this allocation during training — and how it changes during training dynamics or post-training — is left open.
- **Architecture assumptions**: Results are demonstrated on Transformer MLP layers. Whether the same laws hold for CNNs, SSMs, or other architectures is not tested.
- **No cross-model-family generalization of filtering**: The data-filtering case study uses a Rosetta Neuron discovered in OPT-Pythia matching; whether neurons discovered in other family pairs would work equally well is not tested.

## Relevance

This paper is directly relevant to:

- **Mechanistic interpretability**: Provides a principled, scalable method to identify interpretable neuron-level units without labels or auxiliary training; the polarization effect suggests larger models will be *more* amenable to neuron-level interpretation for a growing (if proportionally shrinking) subset of neurons.
- **Scaling law science**: Extends scaling laws beyond loss/capability metrics to internal representational structure — a new dimension of analysis.
- **Superposition and polysemanticity**: Empirically validates superposition theory's capacity-allocation predictions at scale; bridges toy-model superposition results to large production-scale models.
- **Data curation**: Neuron-guided data filtering without domain labels is a practically useful capability, especially for continued pretraining on specialized domains.

Less relevant to researchers focused on RLHF/alignment, token-level generation, or non-Transformer architectures.

## Questions for Further Exploration

1. **Circuit-level analog**: Do "Rosetta Circuits" (shared computational subgraphs) follow a similar sublinear power law? How does neuron-level universality relate to circuit-level universality?

2. **Post-training dynamics**: How does instruction tuning or RLHF affect the Rosetta Neuron population? Do alignment procedures preferentially modify Rosetta Neurons or non-Rosetta neurons?

3. **Emergence during training**: At what point during training do Rosetta Neurons emerge, and do they emerge suddenly (grokking-like) or gradually? Does the polarization effect appear from early in training?

4. **Scaling exponent variation**: The exponents α vary across model family pairs (Figure 3). What determines the constant c and whether β differs across architectures, training objectives, or data distributions?

5. **Application generalization**: Can Rosetta Neuron-based data filtering work for fuzzier domains (e.g., "medical text" vs. "legal text" rather than "JavaScript vs. Python") where domain boundaries are less crisp?

6. **Connection to platonic representations**: Given the Platonic Representation Hypothesis (Huh et al., 2024), are Rosetta Neurons the neuron-level manifestation of converging representations? Does this predict that cross-architecture Rosetta Neuron counts would follow the same scaling law?
