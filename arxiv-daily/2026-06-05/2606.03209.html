Key Finding

Full-parameter LLM fine-tuning, no central server, across non-IID clients — for the first time

DECA: Decentralizing Block-Wise Adam for Efficient LLM Fine-Tuning ↗
Key Metric
230×
Faster aggregation than DeCAF (Llama-3.1-8B). End-to-end latency: 90% reduction. Accuracy: +2.95% over strongest baseline.
Why This Matters
Privacy-sensitive orgs can jointly fine-tune frontier LLMs with full parameters — no LoRA approximation, no coordinator.
First decentralized FPFT framework with theoretical convergence guarantee under non-IID data
Critical Edge
No privacy proof
Decentralized topology ≠ differential privacy. Full-block transmission is heavier than LoRA. Only tested on GPU clusters with reliable connectivity.
Methodology

Block partition. The Transformer's layers are split into B disjoint blocks. Embedding and LM head stay static. Each round, all clients sequentially update each block — only the active block is ever transmitted or computed on.

Block-wise Moment Approximations (BMA). Standard Adam fails in decentralized block-wise settings because momentum goes stale across blocks and non-IID gradients cause client drift. DECA's BMA refreshes both first- and second-order Adam moments at each block activation using fresh local gradients, then injects a consensus-derived discrepancy signal h_{i,k} — the normalized difference between local and neighbor-aggregated parameters — to correct drift.

Convergence rate. O(1/√TBR) under non-convex objectives + non-IID data + stochastic gradients. Matches state-of-the-art decentralized methods when B is treated as constant.

Memory. Peak GPU from 9M (standard FPFT) down to M + 10M/B. Backprop cost halved by skipping frozen blocks.

Results
  • Qwen2.5-3B classification (4 datasets, 8 clients, Dirichlet μ=0.25): DECA 79.32% avg accuracy vs. Dec-LoRA 75.52%, DeCAF 62.15%
  • Llama-3.1-8B classification: DECA 82.30% vs. Dec-LoRA 80.42%
  • Generation (MT-Bench): Best MT-1 4.88 and overall MT 4.10 on Qwen2-1.5B; best MT-1 4.50 on Llama-2-7B
  • Throughput: 145ms/step vs. DeCAF 249ms, Dec-LoRA 287ms (Llama-3.1-8B forward pass)
  • Aggregation latency: 13.3ms vs. DeCAF 3,054ms — the decisive gap
  • Topology robustness: Ring / ER / bipartite all within 75–76% accuracy
  • BMA ablation: Removing BMA drops accuracy by 6.23%; trivial BMA (no discrepancy signal) reaches only 65.19%
Strengths / Limitations
Strengths
  • First decentralized FPFT with O(1/√TBR) theoretical guarantee under non-IID
  • BMA: bounded discrepancy signal (‖h‖∞ ≤ 1), principled momentum reset at each block
  • 230× faster aggregation than DeCAF — closes the practical gap entirely
  • Topology-agnostic: consistent across ring, ER, bipartite graphs
  • Compatible with gradient checkpointing for further memory savings
Limitations
  • Full-block transmission is heavier than LoRA (2128 MB vs. 1736 MB on Llama-3.1-8B)
  • Sequential block updates: large B means slow effective convergence in wall-clock time
  • "Privacy-sensitive" claim has no DP formal backing
  • Only tested on 8–16 clients with high-end GPUs and stable connections
  • β₂ hyperparameter shifts between classification (0.9) and generation (0.3) tasks
Open Questions
  • Can DECA integrate DP-SGD clipping to provide formal privacy guarantees without breaking convergence?
  • How does asynchronous block execution (different clients updating different blocks simultaneously) affect correctness?
  • What is the principled optimal block count B given hardware constraints? Fine granularity improves accuracy but raises peak memory.
  • How does DECA compare to centralized FPFT (BAdam, GaLore) — what performance ceiling is lost for decentralization?
