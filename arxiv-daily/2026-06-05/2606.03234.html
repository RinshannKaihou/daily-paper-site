RLVR Representation Learning Math Reasoning 2606.03234
Key Finding  ·  2026-06-05
Correct RLVR rollouts naturally cluster in hidden-state space just before \boxed{} — aligning them with a single cosine loss lifts math accuracy by up to +6.2 pp at zero inference cost.
Right Makes Might: Aligning Verified Hidden States Empowers RL Reasoning  ↗
Wang et al., 2026  ·  Peking University · JD AI Research · Tokyo · Tianjin
Key Metric
+6.2 pp
Average pass@1 gain (Qwen3-4B) over DAPO across 8 competition benchmarks. +3.8 pp at 1.7B, +5.4 pp at 14B.
Why This Matters
One loss term. No inference cost. Plug-in on any group-relative RL method.
λ = 0.001 pairwise cosine loss, training-only. The deployed model is architecturally unchanged.
Critical Edge
Format-locked & scale-unverified
Requires a clean \boxed{} anchor. Code/dialogue tasks lack an equivalent. No experiments beyond 14B — geometric convergence at 70B+ is assumed, not shown.
Methodology

The Geometric Sweet Spot

At the token immediately before \boxed{} — the anchor token — last-layer hidden states of all correct rollouts already cluster with cosine similarity ≈ 0.84. Earlier (reasoning chain) similarity is ≈ 0.4; later (answer tokens) it saturates above 0.9 with no gradient signal left.

Only the last transformer layer (e.g. L36 for Qwen3-4B) exhibits this structure. Intermediate layers are uninformative.

Hidden-Align Loss

A pairwise cosine-similarity loss over all correct-rollout pairs within a prompt group, applied exclusively at the anchor token:

ℒ_cos = 1 − (1/|P|) Σ cos(h_i, h_j)
ℒ = ℒ_DAPO + 0.001 · ℒ_cos

Gradient pushes each correct hidden state toward peers while preventing collapse via the projection term. Skipped when < 2 correct rollouts exist in a group.

Two-Phase Backward Pass

Micro-batch training splits groups across mini-batches. The solution: Phase 1 gathers L2-normalized anchor embeddings (no gradient stored); Phase 2 re-forwards each micro-batch against detached references — exact gradients, no memory overhead.

Per-Benchmark (Qwen3-4B)

Pass@k

Hidden-Align outperforms DAPO at every k from 1 to 32 on all six competition benchmarks. Largest gaps appear on AIME 2025/2026 and HMMT — harder problems where DAPO already saturates on easier ones.

Training Dynamics

DAPO's alignment loss decreases then spikes sharply at steps 400–500, indicating fragile partial alignment. Hidden-Align maintains a steady monotonic decrease throughout, with consistently higher accuracy reward at steps 200–300.

Strengths & Limitations
Zero inference overhead — training-only loss; deployed model is unchanged.
Format dependency — requires an identifiable anchor token like \boxed{}; no demonstrated solution for code or open-ended tasks.
Plug-and-play orthogonality — operates in representation space, compatible with any token-probability RL variant (GRPO, VAPO, etc.) in principle.
Narrow λ window — λ = 0.01 causes divergence; useful range is 0.0005–0.005. Adaptive scheduling unexplored.
Consistent pass@k gains — improves both greedy (k=1) and diverse (k=32) performance, confirming deeper structural benefit, not just output polishing.
Scale ceiling unknown — validated only on 1.7B–14B. Geometric convergence behavior at 70B+ is extrapolated, not measured.
Strong competition results — AIME 2024/2025/2026 +8–11 pp and OlympiadBench +12.9 pp are practically meaningful at frontier difficulty.
Single base RL algorithm — all experiments use DAPO. Combination with GRPO, Dr. GRPO, VAPO is theoretically plausible but not empirically confirmed.
