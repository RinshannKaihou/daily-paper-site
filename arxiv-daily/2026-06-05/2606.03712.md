> **一句话总结：** Graph Language Models 中存在"图汇聚 token"（graph sink tokens），它们在激活值上极为突出，但并非图结构信息的真正载体，揭示了当前 GLM 架构中激活显著性与图语义效用的严重脱钩。

> **TL;DR:** Graph Language Models develop "graph sink tokens" that are highly salient at the activation level but do not actually carry graph-structural information — exposing a fundamental decoupling between internal prominence and semantic utility in current GLM architectures.

---

## 核心贡献

1. **发现并定义"图汇聚 token"（Graph Sink Tokens）**：在 LLaGA 和 TEA-GLM 两种代表性 GLM 架构中，均系统性地发现了一类在少数隐藏维度上激活值异常巨大的图 token，作者将其定义为图汇聚 token，以区分于文本/视觉模态中的经典注意力汇聚（attention sink）。

2. **激活显著性 ≠ 注意力主导**：图汇聚 token 的高激活并不导致其获得来自 query token 的最多注意力权重——这与文本中经典 attention sink 行为不同，是本文的核心实证发现之一。

3. **干预实验证明图汇聚 token 非关键信息载体**：通过 Pruning（剪除）、Swap（位置互换）、Reposition（重新定位）三类干预，删除图汇聚 token 对下游预测性能影响极小，而删除随机非汇聚 token 的损害反而更大。

4. **Logit Lens 分析揭示图 token 内容贫乏**：通过将图 token 的隐藏状态投影到词表空间，发现解码内容大多是通用领域词（如 "paper"），而非节点标签或拓扑信息，证实了图 token 的语义空洞性。

5. **指出 GLM 架构性缺陷**：当前 GLM 将图结构映射为 token 序列后，并未自然形成拓扑感知的内部表示，图 token 的构建、放置与对齐机制存在根本性局限。

---

## Key Contributions

1. **Discovery and definition of graph sink tokens**: In both LLaGA and TEA-GLM, certain graph tokens consistently emerge as activation-level outliers concentrated on a small number of hidden dimensions (notably dimension 1512 across both architectures), biased toward early graph-token positions.

2. **Activation saliency decouples from attention dominance**: Unlike classical text attention sinks, graph sink tokens do not necessarily receive the largest query-to-graph attention weights — a key empirical distinction established through head- and layer-averaged attention maps.

3. **Intervention experiments confirm sink tokens are not critical carriers**: Three interventions (TOP-2 Sink pruning, random Non-Sink pruning, Swap) show that removing the highest-activation graph tokens causes negligible performance loss, while removing random non-sink tokens is often more harmful.

4. **Logit lens analysis reveals weak graph-token semantics**: Decoding graph-token hidden states at each transformer layer shows predominantly generic domain terms rather than node labels or structural concepts, with low decoding confidence throughout.

5. **Structural diagnosis for GLM design**: The results collectively identify a gap in how graph topology is encoded into token space — current graph-token construction and alignment mechanisms do not reliably produce topology-aware internal representations.

## Methodology

**研究对象 / Models Studied**

两种代表性 GLM 架构：
- **LLaGA**：以节点对齐的图 token 序列为输入，采用"邻域详情（Neighborhood Detail, ND）"模板，中心节点在位置 0，K 跳邻居依序排列，不足时用 [PAD] token 填充。
- **TEA-GLM**：以 GNN 编码器聚合邻域信息，再通过线性映射投影为固定数量（K=5）的可学习图 token embedding，并通过对比学习与 LLM 隐空间对齐。

实验数据集：Cora、ArXiv、PubMed，任务：节点分类 + 链路预测。

**图汇聚 token 的定义**

对于第 l 层第 j 个图 token 的隐状态 $\mathbf{x}_j^{l-1} \in \mathbb{R}^d$，定义汇聚特征函数：
$$\phi\left(\mathbf{x}_j^{l-1}\right) = \max_{d \in \mathcal{D}_{\text{sink}}} \left| \text{RMSNorm}\left(\mathbf{x}_j^{l-1}\right)_d \right|$$
当 $\phi(\cdot) \geq \tau = 15.0$ 时，该 token 被判定为图汇聚 token。汇聚维度集合 $\mathcal{D}_{\text{sink}}$ 通过预先识别激活峰值维度（如维度 1512）确定。

**分析维度**

1. **激活分析**：观察各隐藏维度的平均激活幅值，识别汇聚维度及位置分布。
2. **注意力分析**：计算 query-to-graph token 的注意力权重（按 head 和 test sample 平均），同时做 layer-wise 分析。
3. **干预实验**：TOP-2 Sink pruning vs. Non-Sink pruning vs. Swap，对比性能下降幅度。
4. **Logit Lens**：将每层图 token 的隐藏状态通过 LM head 解码为词表 token，观察其语义内容。
5. **剪除后再检测**：删除所有汇聚 token 后，重新运行汇聚 token 检测，观察汇聚行为是否在其他位置重现。

## Results & Highlights

**激活模式**

- 在两种 GLM 架构、三个数据集、两类任务上，维度 1512 始终是主导汇聚维度；在 LLaGA 中维度 2533 也共现。这表明汇聚维度部分继承自 LLaMA backbone（LLaMA2-7B 已被证明具有此类汇聚维度），但图 token 处理也会诱发新的汇聚行为。
- 图汇聚 token 显著偏向早期图 token 位置：LLaGA 中最强汇聚 token 通常是 [PAD] token，TEA-GLM 中集中于位置 0 和 1。

**注意力模式**

- TEA-GLM 的注意力热图显示：注意力较强的位置是后期图 token 位置（2-4），而非主要汇聚位置（0、1），体现了清晰的反向分离。
- LLaGA 中注意力呈现稳定的垂直带状分布，部分与汇聚位置重叠，但并不集中于汇聚 token；且汇聚 token 常为 [PAD]，注意力与真实图内容关联弱。
- 图 token 稀疏程度（[PAD] 比例）与对汇聚 token 的注意力强度正相关：真实图 token 越多，对汇聚 [PAD] 的注意力越少。

**干预实验（Table 1，节点分类）**

| Model | Arxiv (Baseline 77.00) | Cora (Baseline 88.40) | PubMed (Baseline 94.60) |
|---|---|---|---|
| LLaGA Top-2 Sink | 77.00 | 88.00 | 95.00 |
| LLaGA Non-Sink | 74.36±1.44 | **80.48±2.47** | **89.16±4.58** |
| TEA-GLM Top-2 Sink | 56.67 | 13.00 | 83.67 |
| TEA-GLM Non-Sink | **44.40±1.13** | **12.80±1.72** | 83.40±1.09 |

红色标注（红值）表示 Non-Sink 剪除的伤害比 Top-2 Sink 剪除更大——这一模式在 LLaGA 的多个设置下成立，进一步确认汇聚 token 非关键信息载体。

**Logit Lens（TEA-GLM on PubMed）**

- 汇聚 token 位置（g0、g1）在约第 20 层后稳定解码为 "paper"，这是模型预训练领域（ArXiv 论文集）留下的痕迹，而非真实的节点标签或拓扑信息。
- 解码概率整体偏低，说明图 token 隐状态未与稳定的词表预测对齐。

**剪除后重检测**

- LLaGA：删除原始汇聚 token 后，汇聚行为在更多位置广泛重现，分布更加分散，说明 LLaGA 的汇聚特性由图 token 构建方式整体决定，而非绑定于特定 token。
- TEA-GLM：删除后整体激活幅度下降，新汇聚 token 几乎不再满足阈值条件，说明 TEA-GLM 的汇聚行为更局部、更固定，本质上是激活层面的架构副产物。

## Critical Assessment

本文工作质量较高，核心发现具有理论与实践双重价值。以下是综合评价：

**研究问题的重要性**：GLM 架构日益流行，但内部机制几乎完全未被解析。本文填补了这一空白，且发现的问题——激活显著性与图语义效用脱钩——对 GLM 设计有直接的批判性意义。

**方法的严谨性**：三类干预（pruning/swap/reposition）、两种架构、三个数据集、两类任务，加上 logit lens 和剪除后重检测，形成了多层次的交叉验证，结论说服力强。

**局限性**：仅研究了两种 GLM 架构，泛化性有待验证。图汇聚 token 的定义基于预先指定的汇聚维度，对维度选择的敏感性分析有限。干预实验中性能变化量有时绝对值较小（尤其是 TEA-GLM on PubMed），结论的统计效力存疑。此外，论文未提出具体改进方案，偏向诊断而非修复。

**写作**：清晰，key takeaway boxes 的使用使核心结论易于获取。

## Strengths

1. **原创性强**：首次系统研究图 token 的汇聚行为，并区分其与经典文本注意力汇聚的本质差异。
2. **多视角交叉验证**：激活分析、注意力分析、干预实验、logit lens 四个维度共同指向同一结论，冗余验证增强了可信度。
3. **发现直接具有实践指导意义**：激活显著性不等于信息重要性这一发现，对 GLM 设计者和研究者具有重要警示作用，避免将"显眼的 token"误认为"重要的 token"。
4. **清晰的架构对比**：LLaGA（节点对齐 token）与 TEA-GLM（编码器嵌入 token）的对比设计，使结论在不同图 token 构建范式下均得到检验。
5. **发现汇聚维度跨架构稳定**：维度 1512 在两种 GLM 架构中均为主导汇聚维度，这一跨架构一致性提升了发现的普遍性。

---

## Limitations

1. **架构覆盖范围窄**：仅研究 LLaGA 和 TEA-GLM，其他 GLM 架构（如 GraphGPT、InstructGLM 等）是否表现相同尚未验证，泛化性受限。
2. **无修复方案**：论文定位为纯诊断性研究，未提供如何改进图 token 构建/对齐以减少汇聚-语义脱钩的具体建议，实践可操作性偏弱。
3. **干预性能变化绝对值小**：部分数据集（尤其 PubMed、TEA-GLM）中 Top-2 Sink 与 Non-Sink 剪除的性能差距很小，结论的统计显著性存在一定风险。
4. **汇聚维度识别依赖预设阈值**：τ=15.0 的选取缺乏系统性消融分析，汇聚维度集 D_sink 的构建过程也未充分说明对不同阈值的鲁棒性。
5. **Logit Lens 分析局限于 TEA-GLM**：TEA-GLM 仅有 K=5 个图 token，位置可比性强，但 LLaGA 有 K=111/222 个图 token，未做同等详细的 logit lens 分析，存在选择性报告的隐忧。
6. **因果性存疑**：干预实验（pruning/swap）改变了 token 序列长度或顺序，可能引入混淆变量，尽管论文设计了 Non-Sink 对照组来缓解此问题。

## Relevance

本文对以下研究者高度相关：

- **Graph Language Model 开发者**：揭示了当前图 token 设计的根本缺陷——将图拓扑映射为 token 不能自动产生可用的拓扑感知表示——对架构改进方向有直接指引。
- **LLM 机制可解释性研究者**：将 attention sink / massive activation 现象从文本/视觉扩展到图模态，拓展了该领域的研究对象范围，方法论（定义、干预、logit lens 组合）可供复用。
- **图神经网络与 LLM 融合研究者**：提示 GNN 编码器生成的 graph token 在进入 LLM 后并未保持拓扑信息，需要重新审视图-文本对齐机制的有效性。
- **AI 可信赖性研究者**：发现"激活显眼 ≠ 功能重要"的普适原则，对依赖显著性指标（saliency maps）的可解释性方法有警示意义。

## Questions for Further Exploration

1. 汇聚维度 1512 在 GLM 图 token 处理后仍为主导，这究竟是从 LLaMA 继承的、还是图 token 构建过程主动强化的？能否通过控制实验分离这两种来源？

2. 如果修改图 token 的构建方式（例如引入显式拓扑位置编码、使用更强的图-文本对比对齐），汇聚-语义脱钩现象是否能被缓解？有没有"做对了"的 GLM 架构作为阳性对照？

3. Logit Lens 仅对 TEA-GLM 做了系统分析，LLaGA 的 K=111 个图 token 是否存在少数位置表现出更强语义内容？汇聚位置（通常是 [PAD]）和中心节点位置（index 0）的解码内容有何差异？

4. 图汇聚 token 的激活-语义脱钩是否在 few-shot 或 fine-tuned GLM 上同样成立？还是说这是 zero-shot/frozen-LLM 范式特有的病理？

5. 本文采用两种固定长度图 token 序列的 GLM，对于图 token 数量动态变化的架构（如依据图大小自适应的方案），汇聚行为会如何演变？

6. 是否可以利用汇聚 token 的位置/激活模式来指导图 token 序列的压缩或蒸馏，从而在不损失（本已有限的）性能前提下降低计算开销？
