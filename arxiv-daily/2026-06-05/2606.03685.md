> **一句话总结：** 对经过监督微调的LLM规划器（Plansformer）进行可解释性解剖，发现其内部线性编码了动作有效性和部分状态谓词，但生成概率与内部表示之间存在显著脱节——模型"知道的比它说的多"。

> **TL;DR:** A systematic interpretability study of supervised fine-tuned LLM planners (Plansformers) reveals that SFT on valid action sequences induces linear internal representations of action validity and some state predicates, but generative token probabilities often fail to reflect this internal knowledge — and random-walk training data with better state-space coverage is the key driver of both in-distribution and OOD world model recovery.

---

## 核心贡献

1. **双维度世界模型恢复框架**：同时从"内部表示"（线性探针）和"生成能力"（动作有效性分类）两个角度系统评估LLM是否习得了规划世界模型，弥补了此前研究只看单一维度的不足。

2. **关键发现——内部表示与生成概率脱节**：即便LLM的token概率无法区分有效与无效动作，其中间层的隐藏状态仍线性编码了动作有效性信息（F1接近1.0）。这与"LLMs知道的比它们说的多"的现象一致。

3. **训练数据分布是核心杠杆**：使用无偏随机游走（random walk）数据训练能最大化生成层面的世界模型恢复；基于最优规划（optimal plans）训练的模型内部表示质量尚可，但生成概率几乎无法区分有效/无效动作。

4. **State-CoT的局限**：在序列中插入状态链式思考（State-CoT）对内部表示质量无显著提升，且会带来较大训练不稳定性，尤其在Logistics领域。

5. **OOD场景的分裂现象**：在分布外（更大规模的Blocksworld实例）测试中，随机游走模型的生成层面世界模型恢复崩溃，而内部线性表示仍保持良好——进一步印证了两种评估维度的不一致性。

---

## Key Contributions

1. **Dual-lens evaluation framework**: Combines linear probing of hidden states (internal representation) with generative action-validity classification (token probabilities), providing a holistic picture of world model recovery in fine-tuned LLMs.

2. **Internal vs. generative dissociation**: Models whose token probabilities fail to rank valid actions above invalid ones can simultaneously possess high-quality internal representations that linearly distinguish the two — a form of "knowing more than they show."

3. **Training data distribution dominates**: Uniform random-walk data (which uniformly samples the action space) yields the best generative world-model recovery. Optimal plan data yields poor generative recovery despite non-trivial internal representations.

4. **Random Walk++ as a practical fix**: Augmenting random walks to avoid state cycles (Random Walk++) improves both in-distribution and OOD world-model recovery, being the best single intervention studied.

5. **State-CoT is a double-edged sword**: Interleaving state chain-of-thought in training sequences does not reliably improve world model recovery metrics, and causes severe performance degradation in some domains (e.g., Logistics goal reach rate collapses to ~5%).

## Methodology

**Base model and task.** The authors fine-tune `gemma2-9b-instruct` via LoRA (4 epochs, lr=3e-4) on end-to-end plan generation in STRIPS-style text format across two classical planning domains: **Blocksworld** (3–5 blocks) and **Logistics** (3–4 packages, simplified to cities + airplane). Plans are encoded as `(goal, initial_state, action_1, ..., action_N)` sequences.

**Three training data distributions.**
- *Optimal plans*: Shortest plans from Fast Downward heuristic search.
- *Random walk plans*: Uniformly sampled valid action sequences starting from random states; may contain state cycles.
- *Random walk++ plans*: Random walks with cycle elimination — actions that would return to a previously visited state are disallowed, improving state-space coverage.

**State-CoT augmentation.** A subset of models is trained with state changes interleaved between actions (rather than full states, to control sequence length), inspired by RAP-style chain-of-thought.

**World model recovery metrics.**
- *Linear probes*: Layer-wise logistic regression probes trained on hidden-state activations (averaged over action tokens) to predict (a) binary action validity and (b) truth values of individual state predicates. F1 score is used due to severe class imbalance (valid actions are the minority at test time).
- *Generative metric*: For a randomly selected intermediate step $j$ in a generated plan, all valid and invalid next actions are enumerated and scored by mean token log-probability. The metric is the fraction of test plans where **all** valid actions rank higher than **all** invalid ones.

**Data extraction protocol.** For each test plan of length $N$, a step $j \sim \text{Unif}(0, N-1)$ is sampled, all $M'$ possible next actions are appended to the partial plan, and the LLM is re-prompted to extract hidden states and token log-probabilities for each appended action.

**OOD evaluation.** Plansformers trained on 3–5 block Blocksworld instances are tested on PlanBench instances with 6–10 blocks.

## Results & Highlights

**Planning performance (in-distribution).** All non-State-CoT Plansformers achieve ~97–100% goal-reach rate and valid-plan rate on held-out in-distribution test problems. State-CoT degrades performance substantially in some settings (e.g., Logistics random walk goal reach: 5%).

**Linear probe — action validity (Figure 4).** Probe F1 scores climb steadily with layer depth across all training conditions, plateauing near F1 ≈ 1.0 at intermediate-to-upper layers. Random walk++ achieves the best consistency across seeds. No significant effect of State-CoT on probe quality.

**Linear probe — state predicates (Figure 5).** State predicate recovery is partial and predicate-dependent. Simpler predicates (fewer arguments, more balanced label distribution) probe best — e.g., `arm-empty` in Blocksworld achieves F1 ≈ 0.9, while `on ?obj1 ?obj2` stays near 0.2. This suggests incomplete world model internalization.

**Generative action validity (Table 2).** Random walk models score ~80–82% correct ranking; random walk++ models ~29–36%; optimal models only ~4–9%. The gap between random walk and random walk++ likely reflects state-cycle removal that eliminates certain valid actions from training, reducing the LLM's tendency to assign high probability to all valid actions indiscriminately.

**OOD generalization — planning (Table 3).** Random walk++ and optimal Plansformers show similar OOD planning performance (both drop sharply beyond 6 blocks to near 0% at 10 blocks). Random walk models fail completely (0% goal reach on OOD). KDE analysis (Figure 7) shows the random walk++ model's internal action-ranking distribution is stable under distribution shift, while the random walk model's collapses.

**OOD generalization — internal representations (Figure 8).** All models maintain high linear probe F1 scores on OOD Blocksworld instances, even when their generative ranking performance has collapsed. This is the clearest illustration of the internal/generative dissociation.

## Critical Assessment

This is a careful and honest interpretability study of a focused question. The dual-lens evaluation framework is the genuine methodological contribution — most prior work in this space (e.g., Toshniwal et al. on Chess, Vafa et al.) looked at either internal representations or generative properties, not both together in a controlled sweep over training data distributions. The paper's willingness to report negative results (State-CoT largely unhelpful, OOD collapse) adds credibility.

However, the most important limitation is one the authors acknowledge themselves: showing that linear probes can extract action-validity information from hidden states does not demonstrate that those representations are causally used by the model when generating plans. The internal knowledge could be an epiphenomenon. This is the classic "probing ≠ using" problem, and the paper does not attempt causal interventions (e.g., activation patching). The generalizability to larger LLMs (70B+ scale) is also unknown — the entire study uses a single 9B-parameter base model.

The State-CoT results are somewhat confounded by training instability issues (the authors had to augment datasets to recover performance), making it hard to cleanly attribute the null-to-negative effects to the augmentation strategy itself versus data-efficiency problems.

---

## Strengths

- Clean experimental design: systematic sweep over 3 training distributions × 2 augmentation strategies × 2 domains × 3 seeds.
- Both internal representation and generative probing are evaluated, revealing their dissociation — a substantive finding.
- OOD evaluation on PlanBench adds external validity.
- Honest about limitations and what the results do/do not imply.
- The random walk++ intervention (cycle removal) is simple and practically useful for anyone building LLM planners.
- The "models know more than they show" finding connects to a broader literature on LLM intrinsic representations and has implications for how planning LLMs should be evaluated and deployed.

---

## Limitations

- Single base model (gemma2-9b-instruct); results may not transfer to instruction-tuned models at different scales or with RLHF training.
- Only STRIPS-encoded plans in text format — no natural language encoding, which may prevent LLMs from leveraging commonsense pretraining knowledge.
- Linear probing is not causally informative. The paper cannot establish that recovered world model representations are used in the generation process.
- State-CoT experiments are somewhat confounded by training stability issues, complicating interpretation.
- Only two planning domains; both are relatively structured and deterministic. Stochastic or partially observable domains are not studied.
- The generative metric (all valid > all invalid) is strict and may understate recovery: a model that correctly ranks 99% of valid actions above invalid ones still scores 0 if even one valid action is ranked below any invalid one.

---

## Relevance

**High relevance** for researchers working on:
- LLM planning, especially SFT-based end-to-end planners.
- Mechanistic interpretability of LLMs in structured reasoning tasks.
- Training data curation for reasoning — the finding that data diversity (state-space coverage) matters more than data quality (optimality) is broadly applicable.
- Evaluating world model internalization in generative models.

**Moderate relevance** for:
- RL fine-tuning of LLM planners (the paper suggests RL may push OOD performance further).
- The emerging area of LLM agents that need to reason about environment dynamics.

---

## Questions for Further Exploration

1. **Causality**: Do the linearly-decodable world model representations actually influence token generation? Activation patching or causal tracing (a la Meng et al. on factual recall) could test this directly.

2. **Scale**: Do the internal/generative dissociation and training-data-distribution effects persist at 70B+ scale, or does scale itself confer better world model recovery?

3. **RL fine-tuning**: Would reinforcement learning from planning feedback (e.g., reward = goal reached with valid plan) produce more tightly coupled internal and generative world model representations?

4. **Beyond STRIPS text encoding**: Would natural-language plan descriptions allow the LLM to leverage commonsense priors and recover world models more efficiently?

5. **Causal intervention**: Can the identified world model representations be used to steer generation — e.g., by patching hidden states to "inject" knowledge of the current planning state?

6. **Random Walk++ and cycle length**: Is the cycle-free constraint sufficient, or would longer exploration horizons (or curiosity-driven exploration) further improve world model coverage?
