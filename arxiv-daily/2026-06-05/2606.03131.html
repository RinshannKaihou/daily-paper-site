Key Finding

SVD edits a reward model's head vector — no training required — blocking reward hacking cold

HARVE: Hacking-Aware Reward-Head Vector Editing ↗
Key Metric
+21.1pp
Gold-preference rate on targeted hacking subcategories, averaged across 8 open-source reward models (0.6B–20B)
Why This Matters
Deployed reward models can be hardened against specific exploits instantly — no fine-tuning pipeline, no data curation loop.
+13.7pp over strong fine-tuning baseline; RM-Bench largely preserved
Critical Edge
Closed-source blind spot
HARVE needs access to reward-head weights and hidden states — inapplicable to API-only or LLM-as-judge systems. Downstream RLHF impact unstudied.
Methodology

Step 1 — Find fooled pairs. For each hack subcategory k, identify samples where the model mistakenly prefers hacked responses: F_k = {i : r(hacked) > r(gold)}.

Step 2 — Estimate hacking subspace. Compute normalized mean hidden-state difference across fooled pairs to get direction v_k. Stack all directions into matrix V_hack, apply SVD, retain significant singular values to get orthonormal basis Û_hack.

Step 3 — Edit the reward head. Project out the hacking subspace: w_r_new = w_r − α·P_hack·w_r. Strength α selected on dev set. Transformer backbone stays frozen throughout.

RewardHackBench provides the evaluation: 1,203 gold–hacked pairs across 13 subcategories, 5 attack families. Professional legal subset constructed with NY Bar-admitted attorneys.

Results
  • +21.1pp gold-preference on targeted subcategories vs. unmodified baseline (avg across 8 models)
  • +13.7pp over fine-tuning baseline (5:1 mix ratio), despite zero gradient updates
  • +2.4pp RM-Bench cross-domain transfer; up to +20.65pp on RM-Mistral-7B
  • Gains range from +10.5pp (Llama-3.1-8B) to +32.3pp (Qwen3-0.6B) — consistent across families
  • Ablation: style direction alone −0.81pp; style-orthogonal alone +5.32pp; joint HARVE +16.67pp — super-additive interaction confirms multi-dimensional structure
  • Fooled-subset filtering lifts cosine alignment of hacking direction from −0.05 → +0.24
Strengths / Limitations

Strengths

  • Zero gradient updates, zero inference overhead — purely algebraic fix
  • Interpretable: extracted directions can be analyzed for cosine similarity and ablated component-by-component
  • Consistent across 0.6B–20B parameters and five model families
  • RewardHackBench is length-controlled (word-count ratio 1.01) — surface-form confounds ruled out
  • Super-additive ablation provides genuine mechanistic evidence, not just engineering results

Limitations

  • Requires open-weight scalar reward model access — closed-source or generative evaluators excluded
  • Small curated contrastive set needed per hack subcategory
  • α must be tuned per model on a dev set
  • No evaluation of edited model inside a live PPO / best-of-N pipeline
  • Static edit won't adapt to novel hacking patterns not in the calibration set
Open Questions
  • When using HARVE-edited reward models in PPO, do new hacking patterns emerge — and can adaptive re-editing keep pace?
  • Does joint hacking-subspace capacity have a ceiling as the number of target subcategories K grows?
  • Can the direction-editing idea be extended to generative pairwise evaluators (non-linear heads)?
  • What is HARVE's robustness against an adversary who knows the edited subspace and constructs orthogonal exploits?
