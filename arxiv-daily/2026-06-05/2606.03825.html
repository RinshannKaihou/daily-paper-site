Key Finding

Input-dependent short convolutions before Q/K/V deliver a 1.33× compute advantage over vanilla Transformer — at only 8% throughput cost

Dynamic Short Convolutions Improve Transformers ↗
Key Metric
1.60× compute
Scaling-law advantage when applied to all linear layers (vs. 1.33× for QKV only). Custom Triton kernels keep throughput loss ≤8% (QKV) or ≤22% (all-linear).
Why This Matters
Drop-in module. No architecture redesign.
Gains hold across dense Transformers, MoE, Mamba-2, and Gated DeltaNet. ~3% parameter overhead. Works with existing codebases via a residual wrap before QKV projections.
Critical Edge
Inference overhead unmeasured; max 2B tested
Autoregressive decoding cost unreported. Optimal hyperparameters (rank R) only ablated at 300M. No extrapolation to 7B+ shown.
Methodology

Dynamic vs. static. Static short convolutions use fixed per-channel filter weights. Dynamic convolutions generate input-dependent filters on the fly: y_t = Σ_k w_k^(t) ⊙ x_{t-k}, where w_k^(t) is a linear projection of the current hidden state.

Two parameterizations. (1) Low-rank: z_t = x_t·A (rank R), then fused into convolution by a Triton kernel — never materializes the full W×D weight tensor in HBM. (2) Head-wise: ties weights across groups of H channels, cutting memory by H×.

Placement. Primary: residual wrap before RoPE on Q, K, V projections. Secondary: all linear layers (+0.27× compute advantage). Filter width W=4 optimal in ablations.

Triton kernels. Memory-bound op fused on-chip: read each input once, write output once. At H=16, outperforms optimized CUDA static conv kernel from causal_conv1d library by 1.8–3.9×.

Strengths / Limitations
Strengths
  • Hardware-aware: Triton kernel eliminates HBM bottleneck — not just a theoretical gain
  • Broad applicability: dense, MoE, Mamba-2, Gated DeltaNet all improve
  • Thorough ablation: W, rank, head size, placement, QK-norm compatibility all tested
  • Clear mechanism: synthetic MQAR task validates local input-dependent aggregation hypothesis before language modeling experiments
  • Open-source Triton kernels — immediately reproducible
Limitations
  • Maximum tested: 2B dense / 7B MoE — no frontier-scale extrapolation
  • Autoregressive inference overhead unreported (history buffer required for W=4)
  • Optimal rank R only ablated at 300M; likely needs re-tuning at larger scales
  • Single data source (Nemotron-CC + Granite-4 BPE) — multilingual or domain-shifted generalization unknown
  • No theoretical analysis of interaction with RoPE position encoding
Open Questions
  • Does the 1.33× compute advantage hold at 7B+ scale, or does the scaling-law exponent shift?
  • What is the per-token autoregressive inference latency — does dynamic projection dominate over FlashAttention-optimized attention?
  • Can dynamic convolutions be inserted into a pretrained Llama as adapters, restoring gains via lightweight fine-tuning?
  • How does dynamic conv compare to DeepSeek's MLA (multi-head latent attention) in parameter efficiency vs. expressive power?
