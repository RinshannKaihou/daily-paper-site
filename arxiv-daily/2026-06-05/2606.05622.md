> **一句话总结：** AdaPlanBench 基于 307 个家庭任务构建动态多轮基准，强迫 LLM Agent 在逐步揭示的世界约束与用户约束下迭代重规划，实验发现最强模型（GPT-5）准确率仅 67.75%，用户约束比世界约束更难适应，显式约束追踪对最终成功率提升有限。

> **TL;DR:** AdaPlanBench is a dynamic multi-turn benchmark built on 307 household tasks that tests LLM agents' ability to adaptively re-plan as hidden world and user constraints are revealed one-by-one through feedback. The best model (GPT-5) reaches only 67.75% accuracy, and explicit constraint memory helps plan validity but barely moves end-task success.

## 核心贡献

- 提出 **AdaPlanBench**：首个同时覆盖"世界约束"（工具可用性/环境限制）与"用户约束"（偏好/优先级）的动态交互式规划基准，约束在多轮交互中渐进披露，而非一次性给出
- 设计可扩展的**双约束自动构建流水线**：多智能体框架（重写→采样→提取→合并→校验）为每条 MacGyver 查询生成配对的世界约束集与用户约束集，并按约束总量分三档难度（$\mathcal{E}_{low}$/$\mathcal{E}_{mid}$/$\mathcal{E}_{high}$）
- 提出**四维 Rubric 评估**（工具可用性、物理合理性、有效性、安全性），配合 LLM Judge 实现可扩展自动评测，并通过人工标注验证一致性
- 在 10 个主流 LLM 上全面测评，揭示三大系统性规律：（1）约束越多性能越差；（2）显式约束追踪只能提升计划有效率、不能有效提升最终准确率；（3）用户约束比世界约束造成更大困难

## Key Contributions

- Introduces **AdaPlanBench**, built on top of MacGyver dataset (307 curated household instances), with a scalable automated dual-constraint construction pipeline using multi-agent roles (rewriter, planner samplers, extractor, merge model, checker)
- Defines a **runtime interaction protocol** where constraints are withheld and only revealed upon violation — agents must infer, track, and adapt plans in response to incremental feedback across multiple turns
- Three **termination conditions** (valid plan found / max-turn budget / early stopping on no new violations) and **seven evaluation metrics** (Acc., VPR, Avg Turns, AWRV, AURV, ATWC, ATUC) enabling nuanced multi-dimensional analysis
- **Human-validated LLM-as-judge** rubric evaluation across four dimensions (Tool-Use Feasibility, Physical Plausibility, Effectiveness, Safety), each scored 1–5 with threshold γ = 4
- Code and dataset released publicly

## Methodology

**Benchmark Construction.** AdaPlanBench starts from the MacGyver dataset, rewriting raw queries to remove explicit resource hints while preserving solvability. A binary filter retains only multi-step household tasks. For each retained query, a multi-agent pipeline iteratively samples candidate plans (J planner variants, up to R rounds), extracts salient tools, and converts them into world constraints (tool availability/environmental restrictions, e.g., "there is no iron at home") and user constraints (attribute-level preferences inferred from tool usage, e.g., "I dislike tools that generate high heat"). Constraints are merged, canonicalized, deduplicated, and validated by a final checker model M_chk that also removes internally contradictory or jointly exhaustive user preference sets. Three difficulty tiers are produced by the total constraint count: E_low averages ~10 world + ~11 user constraints, E_mid ~20 + ~22, E_high ~38 + ~42.

**Runtime Protocol.** At each turn t, the agent proposes a plan p_t. LLM judges evaluate it for world-constraint satisfaction, user-constraint satisfaction, and four rubric dimensions. Violated constraints (V_t^w, V_t^u) are passed to a user simulator M_user that generates natural-language feedback f_t revealing them. The agent re-plans to produce p_{t+1}. The loop terminates when (a) all constraints are satisfied and rubrics pass, (b) the max turn budget T is reached, or (c) no new constraints are violated for two consecutive turns (early stopping, indicating stagnation). This design makes constraint disclosure contingent on the agent's own proposals — the agent must proactively explore the constraint space.

**Models Evaluated.** Proprietary: GPT-5, GPT-5-Mini, GPT-5-Nano, DeepSeek-v4-Flash, Gemini-3-Flash, Gemini-3.1-Pro. Open-source: Qwen3-8B, Qwen3-14B, Qwen3-32B, Llama-3.3-70B-Instruct.

## Results & Highlights

**Main results (E_mid setting):**

| Model | Acc. (%) | VPR (%) | Avg Turns |
|---|---|---|---|
| GPT-5 | **67.75** | 89.58 | 6.21 |
| GPT-5-Mini | 61.89 | 85.34 | 5.89 |
| Gemini-3-Flash | 43.32 | **90.23** | 5.82 |
| Gemini-3.1-Pro | 34.53 | 91.21 | 5.65 |
| DeepSeek-v4-Flash | 35.53 | 76.97 | 6.39 |
| Llama-3.3-70B-Instruct | 29.32 | 83.71 | 4.62 |
| Qwen3-32B | 17.92 | 80.13 | 5.01 |
| Qwen3-8B | 14.38 | 82.35 | 4.49 |

- **High VPR ≠ high accuracy:** Gemini-3.1-Pro achieves the best VPR (91.21%) yet only 34.53% accuracy, while GPT-5 leads accuracy at 67.75%. Good constraint-tracking is necessary but insufficient.
- **Performance degrades monotonically with constraint complexity** (E_low → E_mid → E_high) across all models and all rubric dimensions. The degradation is particularly steep for plan effectiveness and physical plausibility.
- **User constraints are harder than world constraints.** Ablating to world-only or user-only settings shows user-constraint-only is consistently more difficult, and combining both is hardest. User preferences can rule out large swaths of the tool space implicitly.
- **Explicit constraint tracking (memory-on) helps VPR by 5–15% but barely moves accuracy** (<3% for 3 of 4 models tested). The bottleneck is not remembering constraints — it is constructing globally valid plans under them.
- **Rubric-based refinement feedback yields only modest accuracy gains (~10%) and sharply reduces VPR (~40% for open models)**, suggesting recency bias: models over-repair newly flagged dimensions while breaking previously satisfied constraints.
- **Proactive constraint elicitation correlates strongly with accuracy** (r = 0.898 for ATWC, r = 0.919 for ATUC), suggesting that generating diverse plan proposals to trigger disclosure is key to eventual success.
- **Model scale does not reliably predict adaptive planning ability:** Qwen3-8B, 14B, and 32B perform similarly (~14–18% accuracy), and GPT-5-Mini approaches GPT-5 in accuracy.

## Critical Assessment

- **Novelty:** The progressive constraint disclosure framing is genuinely distinct from prior benchmarks. Table 1 credibly shows AdaPlanBench is the only benchmark to tick all seven properties (iterative re-planning, user interaction, world interaction, dual constraints, progressive disclosure, open-ended evaluation, scalable constraints). The world-vs-user constraint split is principled and analytically useful.
- **Rigor:** Experimental design is thorough — 10 models, three difficulty tiers, seven metrics, ablations on memory, rubric refinement, and constraint-source isolation. Human annotation validation of LLM judges adds credibility. Confidence intervals are provided in appendix.
- **Reproducibility:** High. Code and dataset are released; constraint construction pipeline is detailed; model and judge choices are documented. The reliance on proprietary LLM judges (for constraint checking and rubric scoring) introduces some external dependency.
- **Significance:** The findings have real implications: the gap between VPR and accuracy shows that constraint-tracking and plan-quality are separable challenges; the user-constraint difficulty finding points to a specific capability gap; the negative result on explicit memory and rubric refinement is practically important for system designers.
- **Red flags:** (1) The domain is household tasks only — findings may not generalize to other agentic settings (web navigation, code generation, scheduling). (2) LLM-based evaluation risks systematic bias; while partially mitigated, it cannot be fully eliminated. (3) "Progressive disclosure" is operationalized via plan-violation triggering, which means the difficulty is partially self-induced by the model's own plan choices — a subtle confound. (4) The paper is a preprint and has not yet undergone peer review.

## Strengths

- Fills a genuine gap: no prior benchmark simultaneously tests multi-turn adaptive replanning under both world and user constraints with progressive disclosure
- Clean separation of constraint types enables targeted analysis of failure modes
- Negative results (memory and rubric feedback don't help much) are clearly demonstrated and informative, not buried
- Scalable automated construction pipeline means the benchmark can be extended to new domains
- Strong correlation finding (proactive exploration ↔ accuracy) offers a concrete direction for future model improvement

## Limitations

- Household domain only; generalization to other environments unproven
- Text-only interaction — no perception, embodied execution, or real environment grounding
- Object/attribute-level constraint modeling may miss fine-grained, compositional real-world constraints
- LLM-based judge introduces potential bias (partially mitigated but not eliminated)
- The early-stopping criterion (no new violations for two turns) may prematurely terminate some trajectories that could eventually succeed

## Relevance

该论文直接针对 LLM Agent 在动态现实场景中的规划鲁棒性，揭示了当前模型在约束积累下的系统性能力边界，对研究 Agent 可靠性、约束感知推理及交互式任务完成具有直接参考价值。

## Questions for Further Exploration

1. Why does explicit constraint tracking fail to improve final accuracy — is it a planning search problem, a goal specification problem, or a reasoning failure under constraint accumulation?
2. Can chain-of-thought or scratchpad-based plan revision strategies significantly narrow the GPT-5 accuracy ceiling (67.75%)?
3. How would the benchmark behave with multimodal inputs — would visual grounding reduce physical plausibility failures?
4. The user-constraint difficulty finding is striking: can a dedicated user-preference modeling module (e.g., explicit preference inference) close the gap?
5. Does the MacGyver dataset's household focus create any systematic biases in what world/user constraints get sampled — and would other domains (travel, scheduling, coding) yield qualitatively different failure patterns?
