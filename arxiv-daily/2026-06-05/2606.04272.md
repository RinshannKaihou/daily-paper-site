> **一句话总结：** 强化学习（RL）在预训练极早期便已有效，且直接施加于基础检查点时会扩展模型分布而非收窄它；将 RL 与 SFT 梯度并行平均可获得所有训练方案中最优的 pass@32，同时保全通用能力。

> **TL;DR:** RL is effective surprisingly early in pre-training (as few as 4B tokens), expands rather than sharpens the model's output distribution when applied directly to base checkpoints (the sharpening effect is an SFT artifact), and parallel-averaging RL and SFT gradients within a single training step outperforms every other recipe on pass@32 while preserving general capabilities.

---

## 核心贡献

1. **打破"RL 必须在预训练结束后才有用"的定论。** 作者对 1B 参数模型在预训练全程保存检查点，直接在这些检查点上施加 RL（GRPO/RLVR），发现仅 4B token 时 RL 即带来显著提升，10B token 时便可达到完整 SFT→RL 流水线的水平。

2. **澄清"RL 收窄分布"的误解。** 近期多篇工作认为 RLVR 主要"磨尖"（sharpen）已有分布、并不带来新能力。本文证明，当 RL 直接作用于基础检查点（无前置 SFT）时，pass@1 和 pass@32 **同时**提升（扩展），收窄效应仅在 RL 跟在 SFT 之后时出现——这是 SFT 的副作用，不是 RL 本身的性质。

3. **预训练数据组成比模型规模更重要。** 将任务相关数学数据加入预训练（+10B math tokens，仍是 1B 参数模型）对 RL 的增益远大于将模型扩大至 4B（同等 token 数）。

4. **提出"并行平均"训练方案。** 每个训练步同时从同一参数快照出发，用 RL 优化器和 SFT 优化器各跑一个 batch，再对两套更新取平均。该方法在全部检查点上均达到最强 pass@32，且不损伤通用基准（SFT 类方法均导致通用能力下降 4–8 pp）。

## Key Contributions

1. **RL works very early in pre-training.** By saving intermediate pre-training checkpoints of a 1B OLMo2-based model and applying direct RL (RLVR with GRPO), the authors show meaningful gains from as few as 4B pre-training tokens — well before the Chinchilla-optimal point (~25B tokens).

2. **Clarifying the sharpening vs. expansion debate.** When RL is applied directly to base checkpoints, both pass@1 and pass@32 improve (distribution expansion). The sharpening effect — where pass@1 improves but pass@k for large k does not — arises only in the standard SFT→RL pipeline, making it an artifact of the preceding SFT stage, not of RL itself.

3. **Pre-training data composition > model scale for RL effectiveness.** Scaling up pre-training data with math-heavy tokens (1B model, 60B tokens) yields substantially larger RL gains on MATH than scaling model size to 4B parameters (same 50B tokens). The dominant lever is what the model has seen, not how large it is.

4. **Parallel averaging merges complementary RL and SFT signals.** A simple algorithm maintains independent Adam optimizer states for RL and SFT objectives, computes updates from the same parameter snapshot, and averages them. This achieves state-of-the-art pass@32 across all pre-training checkpoints and avoids the general-capability regression caused by all SFT-based recipes.

## Methodology

**实验设置 (Experimental Setup)**

- **基础模型：** 基于 OLMo2 架构，从头训练 1B 参数模型，使用 DOLMino 预训练混合数据（50B token），含 Wikipedia 7%、DCLM 60%、FLAN 数学 20%、StackExchange 2%、STEM 论文 5%。
- **预训练检查点：** 在 4B、10B、14B、20B、30B、40B、50B token 处保存快照，全程在这些检查点上分别施加不同训练方案，确保控制变量。
- **四种训练方案对比：**
  - **Direct RL（M_t^RL）：** 直接从基础检查点 M_t 出发，用 RLVR（GRPO）训练，reward 为最终答案二值匹配。
  - **SFT（M_t^SFT）：** 从 M_t 出发，用 OpenMathInstruct 数据中每题随机取一个 ground-truth solution 进行 SFT（模拟实际场景中标注稀缺的情况）。
  - **SFT-Gold（M_t^SFT-Gold）：** 用所有 ~23 个 ground-truth completions 进行 SFT（多解数据上界）。
  - **SFT→RL（M_t^SFT→RL）：** 先 SFT 再 RL，即标准流水线。
  - **Parallel（M_t^Parallel）：** 新提出方案，同一快照同步跑 RL 和 SFT，更新取平均。
- **RL 算法：** GRPO，reward 函数为二值答案匹配（verifiable reward）。
- **训练数据：** OpenMathInstruct，含 GSM8K 风格（grade-school）和 MATH 风格（competition-level）两类子集。
- **评估指标：** pass@k（k=1,8,32），温度 T=0.6；通用能力评估使用 LAMBADA、HellaSwag、ARC-Easy、ARC-Challenge、PIQA、OpenBookQA 六个基准。
- **缩放实验：** 额外训练两个变体——1B 模型加入 10B math-heavy token（1B, D=60B），以及 4B 模型同等 50B token（4B, D=50B）。

**并行平均算法（Algorithm 1）**

每步从当前参数快照 θ 出发：
1. 计算 RL 梯度 g_RL 和 SFT 梯度 g_SFT（均基于同一 θ）。
2. 用独立的 Adam 状态分别计算 ΔRL 和 ΔSFT（各自维护一阶、二阶矩估计，互不干扰）。
3. 参数更新：θ ← θ + ½(ΔRL + ΔSFT)。

## Results & Highlights

| 方案 / Recipe | Pass@1 (GSM8K) | Pass@32 (GSM8K) | 通用能力 |
|---|---|---|---|
| Base M_t (4B tokens) | ~2% | ~10% | 基准 |
| Direct RL | ~18% (4B tokens 即达到) | 与 SFT→RL 相当 | 基本不变 |
| SFT (1 solution) | 高于 Base | 低于 RL | 下降 4–8 pp |
| SFT-Gold (23 solutions) | 最高 pass@1 | 最高 pass@32（在多解场景） | 下降 4–8 pp |
| SFT→RL | 强，但需完整预训练 | 强，但出现 sharpening | 下降 |
| **Parallel RL+SFT** | 略低于 Direct RL | **最强（所有检查点）** | **保全** |

**关键数字：**
- 4B token 时，Direct RL 将 GSM8K pass@1 从 ~2% 提升至 ~18%。
- 10B token 时，M_t^RL 在 pass@1 上优于 M_t^SFT，在 pass@8/32 上与 M_t^SFT→RL 持平。
- 数据缩放（1B, D=60B）在 MATH 上的 RL 增益远超模型缩放（4B, D=50B）。
- 并行平均在每个预训练检查点均实现最高 pass@32，但 pass@1 略低于 Direct RL 和 SFT 基线（tradeoff）。
- SFT 系列方法在六个通用基准上平均下降 4–8 pp；Direct RL 和 Parallel 方案下降接近 0。
- 基础模型的 pass@k 与 RL 后 pass@k 之间存在单调正相关，可作为轻量级 RL 有效性预测指标。

## Critical Assessment

这篇论文是目前对"RL 在 LLM 训练中应扮演什么角色"这一问题最系统性的实证研究之一。其最重要的贡献不是某个具体的数字，而是通过严格的控制实验，将此前混乱的观察（RL 是否收窄分布？RL 是否损伤通用能力？RL 多早有效？）一一还原为训练流水线的具体设计选择。

核心洞见具有方法论价值：**观察到的现象往往是整个 pipeline 的产物，而非某个组件的固有性质**。"SFT 损伤通用能力"和"RL 收窄分布"被剥离出来，归因于特定的训练顺序，这对领域内争论（如 Yue et al., Wu et al. 的 sharpening 观点）是清晰的反驳。

并行平均方案简洁优雅，pass@32 最优且不损伤通用能力，是一个可实际落地的改进。

**主要局限（见下节）** 使得该工作尚不能直接宣称"现有 frontier 模型应立刻改变训练流程"，但作为方向性证据，分量很重。

---

## Strengths

- **实验控制极为严格：** 同一批预训练检查点，完全相同的训练数据、问题集和评估协议，各方案之间差异被最小化到目标函数本身。
- **反驳了两个领域流行误解：** sharpening 归因于 SFT 而非 RL；RL 对通用能力几乎无害。
- **并行平均方案无需额外数据，仅需改变优化步骤结构，**实现成本极低，pass@32 最优且通用能力保全。
- **数据组成 > 模型规模**这一发现对预训练配方设计具有直接指导价值。
- 在极早期检查点（4B token，pre-Chinchilla）验证 RL 有效性，颠覆了"需要充分预训练才能做 post-training"的直觉。
- 提供了轻量级预测指标：基础模型 pass@k 可预测 RL 后增益，对 RL 训练资源分配有实用价值。

---

## Limitations

- **规模有限：** 实验最大仅到 4B 参数和 60B token，与 frontier 模型（如 70B+）差距巨大。作者坦承"1B/4B 的结论是否在前沿规模成立是关键未来工作"。
- **预训练数据偏 math-heavy：** DOLMino 混合数据比典型 web-scale 预训练数据含更多数学内容，可能使 RL 在更早期就能有效；对纯文本预训练的泛化性待验证。
- **RL 变体单一：** 仅研究了标准 GRPO（RLVR with binary reward），未涵盖近期针对 entropy 保留、pass@k 扩展等专门设计的 RL 变体，这些变体可能与预训练阶段有不同的交互方式。
- **任务局限于数学推理：** 所有结论基于 GSM8K 和 MATH，对代码、工具调用等其他 RL 训练场景的适用性未验证。
- **并行平均的 pass@1 权衡：** 最优 pass@32 是以略低 pass@1 为代价的，对于强调单次生成准确率的应用场景可能并非最优选择。

## Relevance

**适合以下方向的研究者关注：**

- **LLM 预训练/后训练流水线设计者：** 直接质疑并重新设计 pretrain→SFT→RL 的标准三段式流程。
- **RLHF/RLVR 研究者：** 对 RL 能否带来新能力（expansion vs. sharpening）的争论提供了实验层面的裁决，是理解 RL 机制的必读文献。
- **数据高效 LLM 训练研究者：** 预训练数据组成对 RL 增益的影响这一发现对计算受限场景有直接启发。
- **Scaling law 研究者：** 数据缩放 > 参数缩放的发现对 RL 场景下的最优计算分配提出了新问题。

**与近期工作的关系：** 直接与 Yue et al. (2025)、Wu et al. (2025)（sharpening 观点）、Foster et al. (2025)（RL 需要充分预训练前提）形成对话，也与 Hatamizadeh et al. (2026)、Li et al. (2025)（将 RL 集成进预训练目标）相关但视角不同（本文保持 NTP 和 RL 目标分离）。

## Questions for Further Exploration

1. **并行平均在 pass@1 上的劣势是否可通过自适应加权（RL/SFT 权重随训练动态调整）来弥补？** 固定的 ½-½ 权重是否是最优选择？

2. **"基础模型 pass@k 预测 RL 有效性"这一规律在哪些前提下成立？** 当任务从数学推理泛化到代码生成、多步工具调用时，预测性是否依然可靠？

3. **数据组成 > 模型规模的结论是否在 10B+ 参数规模依然成立？** 还是在足够大的模型规模下，参数本身的容量会成为主导因素？

4. **RL 在预训练中的"最佳介入时机"是否存在一个通用的定量阈值（如 pass@k > X%），还是与任务难度、数据分布强耦合？**

5. **SFT 损伤通用能力的机制是什么？** 是灾难遗忘、分布漂移，还是优化目标对 general token prediction 的干扰？理解这一点对于设计更安全的后训练方案至关重要。

6. **并行平均是否能扩展到 RL + NTP 目标（而非 RL + SFT）？** 作者提到这是未来方向，但当前实验尚未覆盖。
