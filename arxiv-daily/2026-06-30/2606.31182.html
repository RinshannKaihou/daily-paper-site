§ 01

How the dual-agent loop works

methodology

Each constant is cast as a min–max over an infinite-dimensional function class. The nonconvex problem is reduced to a finite convex program whose optimum is a valid lower bound — because every admissible function induces a feasible point.

Tightening = adding valid constraints (not refining a grid). Each valid constraint K′⊆K can only raise the minimum, giving a monotone-increasing sequence of certified lower bounds.

The coding agent proposes a constraint and a proof it holds for every admissible f; the theory agent tries to refute it; a human compactifies. The cycle repeats until convergence.

Certification recipe. Any dual-feasible ỹ certifies −b̂Tỹ − (r + εA‖ỹ‖1 + εcX − εb‖ỹ‖1), verified with directed-rounding interval arithmetic — solver-independent.

§ 03

Strengths & limitations, side by side

assessment

Strengths

  • Rigorous, solver-independent certification — the numbers are formally guaranteed lower bounds, not empirical optima.
  • Clean conceptual symmetry: lower-bound (duality) search complements prior upper-bound (extremal-construction) agent search.
  • The theory agent's active counterexample hunt — and its course-correction of the even-function mistake — is a compelling demo of value beyond passive checking.
  • Detailed, reproduction-grade appendices: full SDPs, PSD-block derivations, certification derivation, verbatim prompts.
  • Candid about limitations; no overclaiming of formal verification.

Limitations

  • Constraint validity rests on human review, not formal verification — the residual weakest link.
  • Only two demonstrated constants, both autocorrelation-type; broad generalization is asserted, not shown.
  • Absolute gains are small (3rd–4th decimal); value is methodological, not the specific numbers.
  • Human-in-the-loop, not fully autonomous — final compactification and steering are manual.
  • No dedicated code release for end-to-end reproduction of the certified bounds.
  • Bound quality depends on the coding agent's relaxation ansatz and the verifier's coverage.
§ 04

Open questions

for further exploration
Q1

How does the framework scale beyond autocorrelation / Erdős constants to broader nonconvex bound problems (control Lyapunov, combinatorial extrema)?

Q2

What is the theory agent's false-negative rate — how often does it miss a counterexample for a genuinely invalid constraint? Is the search ever exhaustive?

Q3

Can constraint-validity be fully machine-checked by folding in Lean autoformalization, removing the human-review bottleneck?

Q4

Could a lower-bound (duality) loop compose with an upper-bound (construction) loop into one pipeline squeezing each constant toward its true value?

Q5

How sensitive are the certified bounds to the relaxation ansatz (Fourier basis, grid resolution, PSD order) — and is there a principled way to direct the search?

Q6

Will the authors release code/data so the certified bounds — including the branch-and-bound sweep — can be independently re-verified?
