> **一句话总结：** 该论文提出一种人机协同的双智能体（编码代理 + 理论代理）自动研究框架，通过发现并验证凸松弛的紧约束来获得经弱对偶 + 区间算术严格认证的下界，在两个著名优化常数（自相关不等式 C₆.₂ 与 Erdős 最小重叠常数 C₆.₅）上分别将认证下界从 1.28 提升至 1.2937、从 0.379005 提升至 0.37912，互补于以往"用 LLM 搜索极值构造求上界"的路线。

> **TL;DR:** The paper turns convex-relaxation discovery into an autoresearch loop of two LLM agents: a coding agent proposes and proves new tightening constraints, and a theory agent verifies each proof and actively hunts for counterexamples, with a human doing final review/compactification. Every reported bound is certified by an explicit dual-feasible point checked with directed-rounding interval arithmetic—improving two optimization constants (C₆.₂: 1.28→1.2937; C₆.₅: 0.379005→0.37912) previously studied by AlphaEvolve.

## 核心贡献
- **填补"下界"侧的空白：** 以往 LLM 智能体工作（AlphaEvolve 等）靠构造极值函数求"上界"；本文转向互补方向——对每个可接受函数都成立的"下界"，通过凸松弛获得，松弛越紧下界越强。
- **双智能体 + 人类审查的人机协同架构：** 编码代理提议/证明新的紧约束并求解改写后的凸规划；理论代理验证证明有效性并主动搜索反例；人类做最终审查与"紧凑化"（去除冗余约束）。两者交替循环直至收敛。
- **问题无关的严格认证机制：** 借助弱对偶，给出一个显式对偶可行点即证得下界；用 CVXPY/SCS 规范形 + 定向舍入区间算术（mpmath）验证锥隶属（含 PSD 块经已验证分解），并把数据舍入与对偶残差误差全部计入惩罚项，保证报告值是经过验证的下确界，而非数值最优值。
- **在两个常数上取得实质改进：** C₆.₂（Sidon 集自相关常数）从 1.28 提升至 1.2937；C₆.₅（Erdős 最小重叠常数）从 0.379005 提升至 0.37912。据作者所知，这是首次用 autoresearch 风格智能体循环改进这些不等式的认证下界。
- **展示了智能体纠偏能力：** 当编码代理为求进展将候选函数限制为偶函数（得到看似更好的 1.344）时，理论代理识别出这是"真正的损失一般性"，bound 只对偶函数限制的常数成立，并引导后续迭代放弃该假设，最终在无限制 C₆.₂ 上得到首个严格下界。

## Key Contributions
- **Complementary lower-bound framing.** Whereas prior LLM-agent work (AlphaEvolve/Georgiev et al. 2025) searches for extremal constructions yielding *upper* bounds, this work targets the *lower* bound, which must hold for every admissible function and follows from a convex relaxation of the nonconvex min–max problem.
- **Dual-agent autoresearch loop.** A coding agent proposes and proves tightening constraints and solves the modified convex program; a theory agent verifies validity and actively searches for counterexamples; a human performs final review and compactification (removing redundant constraints). The loop iterates until convergence.
- **Problem-agnostic rigorous certification.** Each bound is certified by an explicit dual-feasible point. Cone membership (orthant/SOC/PSD) and the dual objective are checked with directed-rounding interval arithmetic, with both data-rounding (ε_A, ε_b, ε_c) and dual-residual error folded into the bound via an a-priori primal-norm bound X — so the reported value is a verified lower endpoint, independent of the numerical solver.
- **Concrete record improvements.** C₆.₂: 1.28 → **1.2937** (first lower-bound movement since 2017); C₆.₅: 0.379005 → **0.37912** (first since White 2022), via branch-and-bound over an (E(M), c₁, d₁) parameter box with per-subbox dual certification.
- **Demonstrated verifier-driven course correction.** A representative loop episode shows the theory agent catching that an even-function restriction gave a spurious 1.344 bound (valid only for the even-restricted constant), and redirecting the search to drop it — illustrating the verifier's value beyond mere filtering.

## Methodology
Each constant is cast as a min–max over an infinite-dimensional class of admissible functions (e.g., C₆.₅ = inf_f max_t ∫f(x)g(x+t)dx for 0≤f,g≤1, f+g=1). Following White (2022), the nonconvex min–max is reduced to a finite convex program whose variables include an overlap bound Ω, per-cell integrals w_j,v_j, Fourier/trigonometric coefficients, and truncation slacks. Only conditions that are genuine consequences of admissibility (0≤f≤1, ∫f=1) are retained, so every admissible f induces a feasible point — guaranteeing the program's optimum is a valid lower bound.

**Successive convex relaxation.** Tightening = adding *valid* constraints (not just refining discretization). Each valid constraint K′⊆K can only raise the minimum (Eq. 10), producing a monotone-increasing sequence of certified lower bounds. The task at each iteration is to find a property of admissible coefficient sequences that earlier relaxations failed to impose.

**The dual-agent loop (Figure 1):**
- *Coding agent* proposes a new constraint or strengthening, together with a rigorous proof it holds for every admissible f, then implements and solves the program.
- *Theory agent* checks the proof is sound AND searches for an admissible f whose coefficients violate the constraint; a constraint survives only if justified and counterexample-free.
- *Human review & compactification* removes redundant constraints (which can only lower the bound) to produce a minimal program.

**Certification (the core correctness guarantee).** Two conditions must hold: (1) every constraint is a true property of all admissible f — established by mathematical argument + human review (NOT formally verified in a proof assistant, which the authors flag as a limitation); (2) the reported number is actually a lower bound — this does *not* rest on human judgment. Programs are written in CVXPY's canonical conic form; the dual is formed by the standard recipe, and weak duality means any dual-feasible ỹ certifies −bᵀỹ ≤ Ω*. The witness from CVXPY is verified rigorously: cone membership is checked directly for orthant/SOC blocks; PSD blocks are certified by a verified factorization in interval arithmetic (naive eigenvalues fail on matrices PSD in exact arithmetic but with small negative computed eigenvalues). The full certified bound absorbs data-rounding and residual error:
Ω* ≥ −b̄ᵀỹ − (r + ε_A‖ỹ‖₁ + ε_cX − ε_b‖ỹ‖₁), evaluated with outward rounding, so the reported value is the lower endpoint of a verified enclosure.

**Discovered constraints:** For C₆.₅ the loop added a pair of Bochner conditions T_f ≽ 0 and I−T_f ≽ 0 (the latter exploiting the pointwise constraint 0≤f≤1 that White's formulation did not explicitly use). For C₆.₂ no prior convex program existed; the loop built one from scratch (Algorithm 2) combining a probability simplex, a PSD moment matrix, per-cell Fourier envelopes in exact integer arithmetic, a Fejér–Riesz Gram matrix encoding the pointwise lower bound, a quadratic energy bound, and plain + localized Bochner conditions.

## Results & Highlights
- **C₆.₂ (Sidon-set autocorrelation constant):** certified lower bound **1.28 → 1.2937**; first movement on the duality side since Cloninger & Steinerberger (2017). Upper bound stands at 1.502862 (Together AI 2026).
- **C₆.₅ (Erdős minimum-overlap constant):** certified lower bound **0.379005 → 0.37912**; first movement on the duality side since White (2022). Upper bound stands at 0.380871 (Together AI 2026). Reported bound is the minimum certified value over a branch-and-bound sweep of the (h,p) box.
- **Loop-convergence analysis (Figure 2):** the certified primal objective increases monotonically as the coding agent adds/tightens valid constraints; red points mark proposals the theory agent *rejected as invalid* (failing for some admissible f, established by explicit counterexample) versus merely non-binding.
- **Course-correction episode (Figure 3):** the theory agent detected that an even-function restriction (yielding a misleading 1.344) was a genuine loss of generality — since C₆.₂ ≤ C₆.₂ᵉᵛᵉⁿ, it bounds only the even-restricted constant — and redirected the search; iteration v14 dropped the restriction, giving the first rigorous bound on unrestricted C₆.₂.
- All bounds certified with **exact interval arithmetic**, no floating-point margin in the final certificate; the framework is stated to generalize to any nonconvex optimum whose lower bound can be expressed as a convex relaxation tightened by adjoining valid constraints.

## Critical Assessment
- **Novelty (high).** The conceptual contribution is genuinely fresh: pairing the *lower-bound* (convex-relaxation/duality) side with the *upper-bound* (extremal-construction) side of prior agent-based math discovery, and operationalizing "find a good relaxation" as an agent loop. The problem-agnostic certification recipe (weak duality + interval arithmetic) is sound and well-executed.
- **Rigor (strong, with one honest caveat).** The numerical-bound side is rigorously certified — dual feasibility + cone membership (including verified PSD factorization) + rounding/residual error all absorbed — so the *numbers* cannot be "wrong" in the usual numerical sense. However, the *constraint-validity* side rests on human-reviewed mathematical proofs, NOT formal verification (no Lean/proof-assistant check). The authors are explicit about this ("correctness ultimately rests on human judgment"). This is the single most important caveat: an undetected flawed proof of a constraint would silently invalidate a bound, and the theory agent's counterexample search is heuristic, not exhaustive.
- **Reproducibility (mixed).** The convex programs (Algorithms 1–2) and the certification procedure (Appendix C/E) are described in enough detail to re-derive, and standard tooling (CVXPY/SCS/mpmath) is used. Prompts are provided verbatim. **However, no dedicated code release is stated** for reproducing the certified bounds end-to-end; the agent transcripts are lengthy and the branch-and-bound sweep is non-trivial. Independent reproduction of the *agent loop* is hard; reproduction of the *final certified bounds* is feasible given the SDP and the certification recipe.
- **Significance (moderate, domain-specific).** Demonstrates a measurable, named improvement over AlphaEvolve on two well-known constants, but the absolute gains are small (3rd–4th decimal place). The broader value is methodological: a reusable template for certified lower-bound discovery. Generality is *claimed* broadly ("optimization, combinatorics, control") but demonstrated on only two closely-related autocorrelation-type problems.
- **Red flags:** None. No overclaiming of formal verification; limitations are candidly stated; the human-in-the-loop nature is not hidden. The "agent discovered constraints automatically" framing is slightly generous given heavy human compactification, but the core constraints (the Bochner pair) are genuinely agent-proposed and agent-verified.

## Strengths
- Rigorous, solver-independent certification — the numbers are formally guaranteed lower bounds, not empirical optima.
- Clean conceptual symmetry with prior upper-bound agent search; positions both sides as complementary.
- The theory agent's active counterexample search (and its course-correction of the even-function mistake) is a concrete, compelling demonstration of value beyond passive verification.
- Detailed, reproducible-level appendices: full SDP formulations, PSD-block derivations, interval-arithmetic certification derivation, and verbatim agent prompts.
- Honest about limitations (human-reviewed not formally verified; two problems only).

## Limitations
- **Constraint validity rests on human review, not formal verification** — the residual reliance on human judgment is explicitly the weakest link; the authors note Lean-based autoformalization could close it.
- **Only two demonstrated constants**, both autocorrelation-type; generalization to the broad class of nonconvex problems claimed in the conclusion is asserted, not shown.
- **Absolute improvements are small** (3rd–4th decimal); the framework's value is methodological rather than the specific numbers.
- **Human-in-the-loop, not fully autonomous** — final review/compactification is manual, and the agent loop required steering.
- **No dedicated code release** described for end-to-end reproduction of the certified bounds.
- Certified bound quality depends on the relaxation ansatz the coding agent explores and on the verifier's coverage; the theory agent's counterexample search is heuristic.

## Relevance
该工作展示了 LLM 智能体在严谨数学推理（凸松弛发现 + 对偶认证）中的应用，对关注"自动化科学/数学发现"、"LLM 用于需形式化验证的高可信任务"以及"优化与组合数学中的 AI 辅助证明"方向具有直接参考价值，是一个把 LLM 创造力与严格数学认证结合的清晰范例。

## Questions for Further Exploration
- How does the framework scale beyond autocorrelation/Erdős-type constants to broader nonconvex bound problems (e.g., control Lyapunov bounds, combinatorial extremal problems)?
- What is the theory agent's false-negative rate — how often does it fail to find a counterexample for a genuinely invalid constraint? Is the search exhaustive in any structured sense?
- Can the constraint-validity step be fully machine-checked by folding in Lean-based autoformalization (as the authors suggest), removing the human-review bottleneck?
- Could the lower-bound (duality) agent loop be composed with an upper-bound (extremal-construction) agent loop into a single converging pipeline that squeezes each constant toward its true value?
- How sensitive are the certified bounds to the coding agent's choice of relaxation ansatz (Fourier basis, grid resolution, PSD order), and is there a principled way to direct the search?
- Will the authors release code/data so the certified bounds can be independently re-verified, including the branch-and-bound sweep?
