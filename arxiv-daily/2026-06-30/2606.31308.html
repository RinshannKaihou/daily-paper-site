Benchmarking Large Language Models on Floating-Point Error Classification

Lisa Taldir · Muhammad Ahmad Saeed · David Defour · Pablo de Oliveira Castro · Eric Petit
InterFLOPBench
Key metric

0.90+

Overall micro-F1 for the latest models — gpt-oss 120b & Qwen 3.5 27b — across 6 FP-error categories. The first LLMs to clear the bar.

17 models · Oct 2023 → Feb 2026
Why this matters

Sits at the code-LLM × numerical-reliability crossroads. Its LLM-as-auditor loop inspires hybrid (LLM + dynamic/formal) analysis pipelines and offers a benchmark for deep numerical reasoning.

Critical edge

Scale is modest: 90 hand-crafted C-only kernels, tiny per-category counts (NaN 55, DivZero 70) — real noise on hard categories, plus a textbook-pattern memorization risk. No Fortran, Python, or multi-file pipelines.

Methodology 90 kernels · 1,130 samples

  • Multi-label classification. Given a C kernel + anonymized JSON inputs, the model labels each input with ≥1 of six error categories (or no_error). Inputs are visible; the oracle's predictions are not.
  • Benchmark build. 90 short kernels (10–30 lines) deliberately eliciting numerical anomalies, each paired with structured input datasets. Occurrences: Comparison 249, Cancellation 208, Overflow 152, Underflow 93, DivZero 70, NaN 55, No-Error 480.
  • Ground-truth oracle. A hardened FPChecker (instruction-level exceptions), cross-validated against Herbgrind (output accuracy), disagreements hand-inspected.
  • Prompt tiers. Three levels (list → +definitions → +detailed_explanation); tier 3 retained for all runs. F1 = 2TP/(2TP+FP+FN), per-category + micro-averaged.

Assessment strengths · limitations

Strengths

  • First dedicated, open benchmark for LLM-based FP-error classification with a defensible runtime ground truth.
  • Honest bidirectional evaluation — it audits the auditor, surfacing three real FPChecker blind spots.
  • Clean prompt ablation with a non-obvious asymmetric finding (definitions help reasoning models, hurt non-reasoning ones).
  • Per-category and cost reporting; explicit about where the metric underestimates LLMs.
  • Strong domain credibility — InterFLOP/Verificarlo authors, Intel co-authorship.

Limitations

  • Scale: 90 kernels / small per-category counts (NaN 55, DivZero 70) — noisy on the hardest categories.
  • Hand-crafted, C-only, isolated kernels — memorization risk; no Fortran/C++/Python/NumPy, no agentic pipelines.
  • Single-turn, single-prompt — no iterative refinement or tool-use that could re-rank models.
  • Trivial real-world case: the LULESH injection ((a−a)/(a−a)) is syntactically obvious.
  • Proprietary model drift — Gemini/GPT-4o/gpt-oss results aren't bit-reproducible over time.
  • Oracle conflation: exception-based vs output-accuracy semantics deliberately unmerged; some silent-precision-loss bugs score as LLM false positives.

Open questions for further exploration

Q1 · SCALE

Would 500+ balanced kernels re-rank models, especially on cancellation / underflow?

Q2 · AUTOMATION

Can the disagreement loop be automated — systematically mining LLM "false positives" to expose new tool blind spots?

Q3 · REAL BUGS

How does performance shift on mined, non-injected HPC bugs vs hand-crafted textbook kernels?

Q4 · LANGUAGES

Does extending to Fortran / Python+NumPy / Julia preserve the per-category difficulty ordering?

Q5 · AGENTS

Could multi-turn agents (LLM calling FPChecker/Herbgrind) close the subtle-error gap without prohibitive cost?

Q6 · CONTAMINATION

Is the latest models' edge exposure to numerical-analysis corpora — or genuine reasoning ability? Is it testable?
