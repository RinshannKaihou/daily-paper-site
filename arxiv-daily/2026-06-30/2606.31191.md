> **一句话总结：** 提出 Intelligent Schema Memory (ISM),通过一个由七种自我改进机制维护、且所有更新均经符号验证的紧凑策略模式库,让冻结的 gpt-4.1-mini 在 300 集连续学习流(硬情景重置)上于 MATH-Hard 与 OlympiadBench 分别达到 80.67%/61.67% 准确率,且仅用最强被动基线 36%/14% 的策略数量。

> **TL;DR:** ISM bolts a self-evolving, symbolically-verified bank of strategy schemas onto a frozen LLM (gpt-4.1-mini). Under a 300-episode continual-learning stream with hard episodic resets, it beats passive/retrieval/reflection baselines on MATH-Hard (80.67%) and OlympiadBench (61.67%) while keeping a memory bank 64–86% smaller than the strongest passive baseline and up to 23× smaller than RAG.

## 核心贡献

- **冻结模型下的持续推理框架**:在硬情景重置(hard episodic reset)协议下,完全通过外部记忆增强让冻结 LLM 持续提升数学推理能力,所有累积改进都存在于模型之外,杜绝灾难性遗忘与微调成本。
- **双表示策略模式(dual-representation schema)**:每个 schema 拆分为稳定的 *content*(描述、求解模板、启发式,注入提示词)与可在线自适应的 *feature hook*(算子类型、结构模式、嵌入质心、成功率 EMA、使用计数),将"知道什么"与"如何被发现"解耦。
- **七种自我改进机制**:Self-Audit / Self-Correct / Self-Merge / Self-Promote-Demote / Self-Prune / Self-Reinforce / Self-Antipattern,加上条件触发的 schema 合成,把外部记忆从被动存储变为有界、自纠错的资产。
- **对称的成功/失败学习**:Self-Reinforce 从成功提炼正面启发式,Self-Antipattern 从失败记录"应避免的错误",两类信号作为互补的、可检索的结构化知识注入求解器。
- **符号验证门控**:所有记忆更新(hook 调整、合并、内容重写、升降权、增强、反模式、合成)在提交前都必须通过符号计算验证,防止错误泛化被强化。
- **极高的存储效率**:MATH-Hard 上 17 个 schema vs Passive 47(少 64%),OlympiadBench 上 13 vs 91(少 86%),比 RAG/Reflexion 的 300 条少 17.6×–23×。

## Key Contributions

- **Parameter-free continual reasoning**: A frozen LLM (`gpt-4.1-mini`) improves over a 300-episode stream with hard episodic resets; every cumulative gain lives in external memory rather than weights.
- **Dual-representation strategy schema**: Each schema separates a stable *content* section (description, solution template, heuristics — injected into the prompt) from an online-adapting *feature hook* (operator type, structural pattern, embedding centroid, EMA success rate, usage count), decoupling *what to do* from *how to be retrieved*.
- **Seven self-improvement mechanisms** running on independent schedules (Self-Audit, Self-Correct, Self-Merge, Self-Promote/Demote, Self-Prune, Self-Reinforce, Self-Antipattern) plus a conditionally-triggered schema synthesizer, turning memory into a bounded, self-correcting asset.
- **Symmetric success/failure learning**: Self-Reinforce distils positive heuristics from wins; Self-Antipattern records mistakes-to-avoid from failures — both stored as structured, retrievable knowledge (unlike STaR/Voyager's positives-only or Reflexion's negatives-only signal).
- **Verification-gated updates**: Every memory write — hook update, merge, rewrite, promote/demote, reinforce, antipattern, synthesis — must pass symbolic-computation verification before commit, blocking reinforcement of faulty generalizations.
- **Strong sample/storage efficiency**: 17 vs 47 schemas on MATH-Hard (64% fewer), 13 vs 91 on OlympiadBench (86% fewer), and 17.6×–23× fewer entries than RAG/Reflexion.

## Methodology

ISM attaches an external, self-evolving *schema bank* to a frozen solver. Pipeline per episode:

1. **Problem stream & episodic resets.** A stream $S=\{(x_1,y_1),\dots,(x_T,y_T)\}$ is split into domain blocks (Algebra, Geometry, Number Theory, …) with hard domain shifts; at episode $t$ the solver computes $\hat{y}_t = f_\theta(x_t, M_t)$ where $\theta$ is frozen and $M_t$ is the schema bank. Nothing carries across episodes except what is in $M_t$.
2. **Hybrid feature extractor.** A rule-based branch (keyword/surface patterns) and an LLM-based branch (lightweight classifier) each produce a `ProblemFeatures` tuple (operator type, structural pattern, heuristic signatures, quantity signature, embedding). They are reconciled by a weighted agreement score (Eq. 2: 0.40 operator + 0.30 pattern + 0.15 heuristic-Jaccard + 0.15 quantity); if `conf ≥ 0.60` the LLM features win, else rule-based features are the fallback.
3. **Two-stage retrieval.** *Stage 1* hard-filters schemas by operator type. *Stage 2* soft-scores survivors with weights $w=(0.15, 0.15, 0.05, 0.55, 0.10)$ over {structural, heuristic, quantity, embedding, prior} — embedding similarity dominates (0.55), and a prior term incorporates the schema's success rate. Top score buckets: high (≥0.80, used directly), medium (≥0.45, used with generic fallback), generic (<0.45).
4. **Frozen solver + optional agentic tools.** Retrieved schema content is injected via a *hard-reset API call* (no message history, temperature 0). For hard problems the solver can enter a *gated* agentic mode invoking symbolic-computation, formal answer-verifier, schema-lookup, and failure-search (antipattern) tools, bounded by a max-turn budget and only fired after an internal sanity check fails.
5. **Dual-representation schema.** Content (name, description, template, heuristics — stable) + feature hook (operator, pattern, heuristic sig, quantity sig, embedding centroid EMA, success-rate EMA, usage count — adaptive). Hook updates per Eq. 4–5 with *asymmetric* rates: centroid $\alpha_e=0.04$ on success / $0.01$ on failure; success rate $\alpha_r=0.15$ on success / $0.10$ on failure.
6. **Memory controller self-improvement loop.** After a 10-episode warmup, seven mechanisms run on independent schedules: Self-Audit (every 10, triggers Correct/Promote/Demote), Self-Reinforce (15), Self-Merge + Self-Antipattern (20), Self-Prune (25). Self-Correct rewrites weak schemas (≥5 uses) and resets their success rate to 0.5; after 3 failed corrections it escalates to pruning. Self-Merge merges pairs with centroid cosine > 0.88 (seed schemas protected). Promote boosts retrieval score +2% for success ≥0.80; Demote penalizes −5% for ≤0.40. Self-Prune removes unused/persistently-failing non-seed schemas.
7. **Conditional schema evolution.** On incorrect/generic outcomes, an evolution gate fires only if (≥10 episodes elapsed) AND (≥3 same-operator failures in last 20 episodes); an LLM synthesizer then creates a new general-family schema with a warm-started hook. This ensures new schemas appear only when a genuine gap exists.
8. **Verification gate.** *Every* update must clear symbolic verification (e.g., Self-Reinforce requires the underlying solution to have been symbolically verified; Self-Merge requires shared use-cases to "symbolically check out") before being committed.

**Implementation.** Frozen solver = `gpt-4.1-mini` (temp 0.0, hard-reset calls). Self-improvement LLM calls (Correct/Merge/Reinforce/Antipattern/synthesis) = `gpt-4o`. Embeddings = `text-embedding-3-small`. Code released at https://github.com/pdx97/ISM.

**Evaluation protocol.** 300-episode continual stream per benchmark, blocked by domain with hard shifts; metrics = accuracy, plasticity (first 10 eps of a block), stability (last 10 eps), forgetting, backward transfer (BWT), and final bank size. Baselines: Vanilla LLM, Static Schema (one fixed template), RAG-over-Examples, Reflexion, Passive Schema Memory (schemas created like ISM but all 7 mechanisms off — isolates the value of *active* maintenance).

- **Accuracy.** ISM = 80.67% (MATH-Hard) and 61.67% (OlympiadBench), +2.00 points over the strongest baseline (Passive) on each.
- **Continual-learning metrics.** Stability improves (82.00→84.67 on MATH-Hard; 62.00→64.00 on OlympiadBench) and forgetting drops (0.10→0.07; 0.10→0.03). On OlympiadBench ISM achieves *positive* backward transfer (+0.03) — past domains keep improving as new ones arrive; only Reflexion also shows positive BWT (+0.01) but at far lower accuracy.
- **Storage efficiency.** 17 vs 47 schemas (MATH-Hard, −64%) and 13 vs 91 (OlympiadBench, −86%); 17.6×–23× fewer entries than RAG/Reflexion. The gap *widens* on the harder benchmark: harder problems push Passive to synthesize more, while ISM's merge/prune keep it compact.
- **Bank dynamics.** On MATH-Hard ISM's bank climbs during the Geometry block then shrinks 26→15 via prune/merge; Passive grows monotonically 6→47. On OlympiadBench ISM stays in 10–22 while Passive climbs linearly to 91. Passive's banks are bloated with dead weight — 60–68% of Passive's schemas are *never reused* after creation.
- **Head-to-head.** Net episode advantage over unmanaged baselines is large (+114 vs Vanilla, +96 vs Reflexion, +85 vs RAG on OlympiadBench); over the schema-based controls it is a steady +6 each, exactly matching the 2-point table gap.
- **Qualitative wins.** Synthesized schemas (e.g., *Simultaneous Polynomial Root Problems*, *Relative Primality in Sets*) achieve success rates 0.63–0.76 with 5–19 reuses and let ISM answer problems where all four baselines fail (e.g., Episode 235: Passive returned 121, ISM returned the correct 93, because ISM had spent episodes sharpening the schema's feature hook to retrieve it when it mattered).

## Critical Assessment

- **Novelty — moderate-high.** The individual ingredients (RAG, self-reflection, symbolic math verifiers, skill libraries) are well-trodden, and "external memory for a frozen LLM" is a crowded space (MemGPT, Voyager, Reflexion). What is genuinely distinctive is their *integration*: (a) a *schema-level* abstraction rather than raw exemplars or executable code, (b) a *dual-representation* design decoupling stable content from adaptive retrieval, (c) *symmetric* success/failure learning, and (d) a *verification gate on every memory write*. The hard-episodic-reset protocol is also stricter than most "memory" evaluations.
- **Rigor — mixed.** The continual-learning protocol with hard resets, the four CL metrics (plasticity/stability/forgetting/BWT), and the Passive-vs-ISM ablation isolating *active maintenance* are real strengths. **But**: all results are from a **single random seed and a single stream ordering** — the authors themselves flag this as a major limitation, and continual-learning systems are notoriously sensitive to both. The headline gaps (+2.0 accuracy) are small relative to the missing variance estimates, so the central claim ("ISM beats Passive") should be read as preliminary, not definitive.
- **Reproducibility — good on paper, unverified in practice.** Model choices, hyperparameters (EMA rates, retrieval weights, thresholds, schedules), benchmarks, and code URL are all specified. Tempering this: no actual numbers on LLM API cost/latency, and the symbolic-verifier implementation details live in code rather than the paper.
- **Significance — conditional.** If the +2-point gains survive multi-seed evaluation, the *qualitative* findings (small verified banks beat large passive ones; active maintenance prevents the 60–68% dead-schema bloat) are useful and somewhat counter-intuitive for the agent-memory subfield. If they don't, the contribution narrows to "a well-engineered memory controller."
- **Red flags / concerns.**
  1. **No per-mechanism ablation.** Only an aggregate ablation (Passive = all 7 mechanisms off) is reported. We cannot tell whether, e.g., Self-Antipattern or Self-Merge actually pull weight or are redundant — and the authors admit this.
  2. **Single backbone** (`gpt-4.1-mini`); model-agnostic *in principle* but not demonstrated.
  3. **Verification scope.** The symbolic gate is well-suited to numerical/short-symbolic answers but the authors concede it is "less applicable to proof-oriented tasks" — so the central safety mechanism may not transfer.
  4. **Marginal headline gains** vs the strongest baseline (+2.0 on both benchmarks), without error bars, invite skepticism about statistical significance.
  5. **Self-improvement LLM cost.** Self-improvement uses `gpt-4o`, so "frozen backbone" understates total system cost; the paper does not report this overhead.

## Strengths

- No parameter updates — practical for proprietary/frozen models and immune to catastrophic forgetting.
- Verification-gated writes ground the memory in *correctness*, not plausibility — a meaningful safety property.
- Strict episodic-reset protocol is a harder, more honest test of "continual" capability than typical RAG/reflection setups.
- Dual content/hook design is elegant: retrieval sharpens with use without destabilizing strategy content.
- Concrete storage-efficiency story (17/13 schemas, 60–68% of Passive's bank is dead weight) is a clean, quotable result.
- Code released; grounded in a credible research lineage on schemas and memory.

## Limitations

- **Single seed, single stream ordering** — the authors' own #1 caveat; headline gaps lack variance estimates.
- **No per-mechanism ablation** — cannot attribute gains to specific mechanisms or identify redundancy.
- **Single LLM backbone** (`gpt-4.1-mini`); generality across model sizes unproven.
- **Only two math benchmarks**; generalization to proofs, code, science is open.
- **Symbolic verification tied to numerical/short-symbolic answers** — weak or inapplicable for proof-oriented reasoning.
- **Narrow baseline set** — no comparison to Tree-of-Thoughts, graph-structured memory, or full ReAct-style agents (authors acknowledge this).
- **Cost not reported** — `gpt-4o` self-improvement calls and gated agentic tool-use add inference overhead that is not quantified.
- **Marginal gains** over the strongest baseline (+2.0 acc) may not be statistically robust given single-seed evaluation.
- **Self-improvement dynamics** over *very long* horizons (drift, schema conflict, saturation) are not characterized.

## Relevance

该论文与持续学习、记忆增强 LLM 推理及符号-神经融合方向高度相关,对研究"在不微调模型的前提下让 LLM 持续积累可验证、可检索的推理策略"具有直接参考价值,尤其适合关注 agent 长期记忆生命周期管理与可验证推理的工作;其验证门控与对称成败学习的设计也可为构建更可靠的自我改进系统提供借鉴。

## Questions for Further Exploration

- Do the +2.0-point accuracy gaps survive multi-seed evaluation and shuffled stream orderings? What is the variance?
- Which of the seven self-improvement mechanisms actually carry the gains (per-mechanism ablation)? Are Self-Antipattern / Self-Merge redundant?
- How sensitive are results to the dual-representation choice — would raw-exemplar memory with the same verification gate close most of the gap?
- Does ISM transfer to non-`gpt-4.1-mini` backbones (smaller open models, much larger frontier models)?
- Can the verification gate extend to proof-oriented or open-ended domains via unit-test-style or program-synthesis checks?
- What is the total inference cost (frozen solver + `gpt-4o` self-improvement + agentic tools) relative to baselines, and how does it scale with stream length?
- How does the bank behave over *very long* horizons (thousands of episodes) — does it saturate, and how are conflicting schemas resolved?
- Could ISM compose with in-model continual learning (e.g., Titans / Nested Learning) for complementary gains, as the authors hint?
