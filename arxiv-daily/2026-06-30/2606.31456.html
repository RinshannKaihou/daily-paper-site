Why this matters

A new path for generative data synthesis in low-bit detector quantization — directly relevant to edge deployment and zero-shot quantization engineering practice.

Critical edge

The detector labels its own generated data and estimates its own class prior — an implicit self-reinforcement loop. Validated only on YOLO + MS-COCO; two-stage / transformer detectors untested.

01

Methodology — three challenges, three components

GoodQ is a ZSQ-OD pipeline: build a QAT training set from off-the-shelf diffusion models, then run QAT. Each of three detection-specific challenges gets one targeted component.

STAGE 1 · IDP

Information-Dense Prompting

Classification-style single-object images starve detection (COCO averages 7.7 boxes/image). Two-class, quantity-augmented prompts: "A {D} photo of {Q} {C1} and {Q} {C2}" → a pool of 160k candidate images.

STAGE 2 · IDAS

Intrinsic Distribution-Aware Selection

COCO is long-tailed (head 30.5% vs tail 0.02%). Target ratios are estimated data-free from the detection head's bias terms, then a greedy loop (Algorithm 1) selects 2k images matching that prior.

STAGE 3 · TANR

Teacher-guided Adaptive Noise Reduction

Pseudo-labels are noisy hard labels. TANR swaps them for the teacher's soft predictions and applies QFocal-based adaptive weighting so the student focuses on instances hit hardest by quantization noise.

Setup: YOLOv5 & YOLOv11 (s/m/l) pretrained on MS-COCO 2017 · LSQ for weights & activations · 2k-image training set · no augmentation. Baselines: LSQ, LSQ+, TSOD (noise-optimized + real labels), and label-free TSOD†.

03

Strengths & limitations

▲ Strengths

  • Clean problem→solution mapping: three challenges → three named, ablation-validated components.
  • Pushes ZSQ-OD into a genuinely hard regime (W3A3) where prior work essentially fails.
  • Elegant data-free distribution estimator from head biases, with strong empirical correlation to ground truth.
  • Analysis section explains why each choice helps — UMAP, cosine-distance, confidence evidence.
  • Honest baselines, including a label-free TSOD† variant for fair comparison.

▼ Limitations

  • Narrow scope: only YOLO single-stage detectors and MS-COCO. Generalization unverified.
  • Pipeline complexity — three stages, each with hyperparameters (prompts, pool size, AW γ).
  • Compute cost of generating + pseudo-labeling 160k images is not reported.
  • Self-referential supervision: labels and prior both come from the pretrained model itself.
  • The bias-based prior assumes the head encodes the true tail — may break under class-balanced loss or label smoothing.
04

Questions for further exploration

  1. What is the end-to-end compute/dollar cost of GoodQ (160k generation + pseudo-labeling + 2k QAT) vs noise optimization — and can the pool be distilled smaller?
  2. How does GoodQ transfer to non-YOLO detectors (Faster R-CNN, DINO, DETR-style transformers) and other datasets (VOC, LVIS, Open Images)?
  3. How robust is the bias-based estimator under class-balanced reweighting, label smoothing, or logit adjustment — all of which distort head biases?
  4. Can TANR's adaptive weighting gain from explicit confidence-thresholded filtering, given a possibly miscalibrated teacher?
  5. Does the self-referential loop create systematic blind spots for rare classes — and would an external vision-language teacher reduce that bias?
  6. What is the marginal value of each component specifically in the W3A3 extreme regime, where the overall gains are largest?
