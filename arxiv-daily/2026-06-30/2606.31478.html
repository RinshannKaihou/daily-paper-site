01

Methodology — MHFA pipeline

Recovery is modeled over an abstraction hierarchy L = {hypothesis, design, implementation}. SAGE inserts a structured loop at the Analysis phase on the AutoResearchClaw harness.

01 · Diverge
Generate 3–5 competing attributions (Method / Design), each citing evidence from Cfail.
▼
02 · Converge
Independent skeptical critic audits logic, scores criticality s ∈ [0,1], selects top h*.
▼
03 · Route
Deterministic rule π(c*,s*) — severity gates depth → Proceed / Local-Refine / Method-Refine / Design-Refine / Pivot (MaxPivots=2).
Verdict clamp  κ₀ = 0.6
valid-but-weak results downgrade hypothesis-discarding Pivot → in-place refine; honest negatives are preserved.
Failure-aware regeneration
archives a failure profile as a negative-knowledge prior; rejects near-identical regenerated hypotheses.
Scoped grounded reporting: manifest whitelist + sanitizer checks table cells against a measured-value registry (1% tolerance), redacts unverified values to ---. Covers numeric tables only — does not verify prose-level method provenance.

03 · strengths

What lands well

  • Principled framing. Targets a real, sharply articulated failure mode (lossy single-reflection) via Chamberlin & Platt — not ad-hoc prompting.
  • Auditable recovery. Diagnosis is decoupled from action by deterministic routing, not a free LLM call.
  • Integrity safeguard. The verdict clamp protects valid-but-weak results from false rejection — a subtle, important point.
  • Exceptionally honest scoping. States what reporting does not verify and names the next open problem instead of overclaiming.
04 · limitations

What still breaks

  • Small benchmark. 12 topics; significance strong only on the ML-subset human study (p=0.031); many deltas rely on LLM judges with acknowledged rank-disagreement.
  • Method-provenance gap unsolved. Prose can still name libraries / datasets / tests not in executed code; writer isn't forced to revise prose after redaction.
  • Code-only gains. Result Analysis stays low for both arms and within judge noise.
  • Hand-tuned thresholds (κ₀, MaxPivots, severity rubric) with limited sensitivity analysis; generalization beyond ARC-Bench untested.
  • Implementation fragility. Sanitizer caught a thousands-separator parsing bug mid-evaluation (13,365,000 → 13,---,000).
05

Open questions

  1. How robust is the deterministic router when multiple root causes interact or co-occur — e.g., a flawed design masking an implementation bug?
  2. Are the hand-set thresholds (κ₀=0.6, MaxPivots=2, severity rubric) learnable / calibratable rather than fixed?
  3. Does the structured-recovery advantage transfer to non-ARC-Bench tasks, or to wet-lab / theory-heavy research?
  4. Can grounded reporting extend to verify prose-level method provenance without prohibitive cost — closing the named open problem?
  5. What is the sanitizer's false-redaction rate on genuine multi-seed / variance-bearing results?
  6. How does MHFA compute overhead scale with trajectory length, and does the negative-knowledge archive degrade over long campaigns?
