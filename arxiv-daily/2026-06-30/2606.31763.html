§ 01Methodology

Wet-lab automation as a stateful, feedback-driven process — not isolated generation.

ProtoPilot operationalizes biology across a five-layer representation, each stage guarded for consistency. An Orchestrator Agent maintains workflow state and dispatches two specialists; a multilayer memory (knowledge base · working memory · task-experience) feeds rubric-based validators.

L1Scientific intent
↓Protocol
↓Manual SOP
↓Device SOP
↓Instrument code (Opentrons / MGI / Hamilton / Tecan)

Mechanism 1 · Hierarchical coarse-to-fine coordinator–worker SOP generation — module-level plan with global constraints, gated locally then integrated. Mechanism 2 · A self-evolving 3-class skill library (param-confirm / deck-layout / instrument-op code) refined via create–evaluate–revision, encoding transferable principles across devices.

Orchestrator Agent
Holds structured workflow state; decomposes objectives, dispatches subtasks, replans after execution feedback. Autonomous + human-in-the-loop checkpoints.
Protocol Expert Agent — “Thoth”
Reinforcement-learning-trained for step-level SOP generation; ranked by a 7-dimension expert rubric (D1–D7).
Coding Agent
Translates SOP → SDK-compliant code using the runtime-updated skill library; checked by a three-layer gate (syntax · parameter alignment · SDK export).

VALIDATORS · biological plausibility · procedural completeness · parameter consistency · safety · protocol–code alignment · instrument/SDK compatibility

Quantitative separation

  • Rubric score 94.7 ± 0.4; only 2.5-pt L1→L3 decline (L1 96.5 / L2 95.1 / L3 94.0), far tighter than agentic-harness baselines.
  • Deepest lead in scientific annotation depth: D7 80.7 vs 50.8–61.0 for harness systems.
  • Blind ranking (3 scientists · 34 queries · 7 systems): Top@1 70.6%, Top@3 90.2%; rubric↔expert Spearman ρ=0.929.
  • Code gate sub-scores: machine-safety 94.8% · bio-safety 87.8% · SDK compliance 98.2% — alone in the high-quality / high-executability quadrant.

Closed-loop wet-lab validation

  • Foundational ops: 96-well OD 0.51–0.88; 12-step serial dilutions; 24/24 positive colony PCR (868 bp).
  • Luciferase plasmid build: GLuc-WT & RLuc-WT → 100% success, Sanger-confirmed.
  • Whole-plasmid mutagenesis: 7/8 GLuc, 8/8 RLuc confirmed (only GLuc-L40P failed).
  • PCA assembly (closed-loop): diagnosed & fixed two real failures — non-specific pUC19 primers + ampicillin inactivation → 93/96 colony PCR positive, all 4 targets Sanger-confirmed.

§ 03Strengths · Limitations

Strengths

what holds up
  • +End-to-end framing — natural-language intent → physical execution → feedback — with an explicit five-layer representation and stage-wise guards.
  • +Expert-grounded benchmark with device-level validity gates and blind human ranking, not text-similarity proxies.
  • +Runtime-updated, three-class skill library encoding transferable knowledge across tasks and instruments.
  • +Real, physically validated wet-lab results (OD, gel, Sanger, colony counts) including genuine failure-recovery loops.
  • +Cross-platform robustness across four distinct automation vendors.

Limitations

what to probe
  • −Benchmark scale (294 tasks / 98 protocols) and diversity are limited; authors call for broader assays, instruments, labware, and failure modes.
  • −Automates only the wet-lab stage after a user-defined objective — no integrated dry-lab loop for a full dry–wet cycle.
  • −Reproducibility depends on release of benchmark, rubrics, Thoth, and skill library — not confirmed in the main text.
  • −Wet-lab validation by the authors' own team; no independent third-party replication.
  • −Skill library may drift or accumulate silent errors over long horizons — no deep safeguard analysis.
§ 04Questions for further exploration
i
How exactly are gate pass / Protocol Score / Top@3 computed — and why do abstract (89.5%/88.24%) and body (96.6%/88.2%) figures diverge?
ii
Will BioLabbench, the Thoth model, the expert rubrics, and the skill library be publicly released, and under what license?
iii
How many feedback iterations did the PCA assembly require, and what is the marginal cost / latency per closed-loop revision?
iv
Does the self-evolving skill library drift over long deployments — and is there a forgetting / corruption safeguard?
v
How does ProtoPilot generalize beyond synthetic/molecular biology — cell-based assays, protein purification, chemistry?
vi
Can the wet-lab results be independently replicated by a third-party lab — and where does the RL-trained Thoth's ceiling lie on truly novel protocols?
