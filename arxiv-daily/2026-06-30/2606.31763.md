> **一句话总结：** ProtoPilot 是一个自进化的多智能体系统（含 Orchestrator、RL 训练的 Protocol Expert Agent 与带运行时技能库的 Coding Agent），并配套专家标注的 BioLabbench 基准（98 个金标准协议派生的 294 个任务），将自然语言实验意图端到端转化为可执行的 Opentrons/MGI 仪器代码，并在真实湿实验中通过反馈闭环修正流程——协议质量评分 94.7、协议转代码门控通过率 96.6%、跨四平台 OT-2 通过率 88.2%（远超 OpenTrons-AI 的 32.4%）。

> **TL;DR:** ProtoPilot is a self-evolving multi-agent system that turns natural-language experimental intent into validated SOPs, SDK-compliant robotic code, and physically executed wet-lab workflows, refined by feedback. Paired with the expert-grounded BioLabbench (294 tasks from 98 gold-standard protocols), it reaches a 94.7 protocol-quality rubric score, 96.6% protocol-to-code gate pass, 88.2% OT-2 pass rate (vs. 32.4% for OpenTrons-AI), and Sanger-confirmed products across cloning, mutagenesis, and feedback-corrected PCA DNA assembly.

## 核心贡献

- **自进化多智能体系统 ProtoPilot**：以 Orchestrator Agent 维护状态，协调 RL 训练的 Protocol Expert Agent（Thoth）和带运行时技能库的 Coding Agent，实现从自然语言意图→SOP→设备代码→湿实验执行→反馈修正的闭环。
- **专家标注基准 BioLabbench**：基于 98 个金标准协议构建 294 个合成/分子生物学任务，配 7 维评分细则（D1–D7）、设备级有效性门控、L1/L2/L3 三档难度和真实实验测试，把「专家湿实验判断」转化为可量化要求。
- **两项核心机制**：分层 coordinator–worker 的粗到细 SOP 生成（保证全局一致性 + 局部步骤细节）；以及自进化三层技能库（参数确认 / 平板布局 / 仪器操作代码），通过「创建-评估-修订」循环不断优化。
- **强定量结果**：协议质量 94.7（领先 BioProAgent 3.2 分），盲评 Top@1 70.6%、Top@3 90.2%、MRR 0.82；协议转代码门控通过率 96.6%；跨四平台 OT-2 通过率 88.2% vs OpenTrons-AI 32.4%。
- **真实湿实验闭环验证**：覆盖基础操作（96 孔培养 OD 0.51–0.88、连续稀释、24/24 阳性菌落 PCR）、荧光素酶质粒构建（100% 成功，Sanger 确认）、平行突变工程（7/8 GLuc、8/8 RLuc）、以及反馈修正的 PCA 组装（成功诊断并修复引物非特异和氨苄西林失活两类失败，4/4 目标 Sanger 确认）。

## Key Contributions

- **ProtoPilot system**: A self-evolving multi-agent architecture that maintains a single stateful loop from free-form natural-language objective → validated SOP → instrument-executable code → physical execution → feedback-guided refinement. An Orchestrator Agent coordinates a reinforcement-learning-trained Protocol Expert Agent (Thoth) and a Coding Agent backed by a runtime skill library.
- **BioLabbench evaluation framework**: 294 synthetic-/molecular-biology tasks derived from 98 gold-standard protocols, with a 7-dimension expert rubric (D1–D7), device-level validity gates, three difficulty tiers (L1/L2/L3), blind expert ranking, and real wet-lab tests.
- **Two core mechanisms**: (1) hierarchical coarse-to-fine coordinator–worker SOP generation that preserves global workflow state while detailing local steps; (2) a self-evolving, three-class skill library (parameter-confirmation, deck/sample-layout planning, instrument-operation/code-specification) refined via create–evaluate–revision loops.
- **Strong quantitative results**: Protocol-quality rubric score 94.7 (3.2 above BioProAgent); blind Top@1 70.6% / Top@3 90.2% / MRR 0.82; protocol-to-code gate pass 96.6%; OT-2 pass rate 88.2% vs. 32.4% for OpenTrons-AI.
- **Closed-loop wet-lab validation**: Four workflows of increasing complexity — foundational ops (96-well OD 0.51–0.88, 12-step serial dilution, 24/24 positive colony PCR), luciferase plasmid construction (100% success, Sanger-confirmed), parallel whole-plasmid mutagenesis (7/8 GLuc, 8/8 RLuc), and PCA DNA assembly where the system diagnosed and fixed two real failures (non-specific vector primers, ampicillin inactivation) to obtain Sanger-confirmed targets.

## Methodology

ProtoPilot operationalizes wet-lab automation as a **stateful, feedback-driven process** rather than isolated generation tasks. It uses a **five-layer experimental representation** — scientific intent → protocol → manual SOP → device SOP → instrument code — with stage-wise guards preserving consistency across layers.

**Agents & memory:** An **Orchestrator Agent** maintains structured workflow state, decomposes objectives into subtasks, and dispatches two specialized agents:
- A **Protocol Expert Agent (Thoth)**, trained with reinforcement learning for step-level SOP generation.
- A **Coding Agent** with a self-evolving skill library for SOP-to-code translation.

A **multilayer agentic memory** stores (i) a knowledge base of domain knowledge/literature/protocols, (ii) a working memory of plan/constraints/intermediate outputs, and (iii) a task-experience base of completed trajectories, validation results, execution traces, and feedback. **Rubric-based validators** check biological plausibility, procedural completeness, parameter consistency, safety, protocol–code alignment, and instrument/SDK compatibility.

**Mechanism 1 — hierarchical coordinator–worker SOP generation:** Instead of producing an entire SOP at once, the Orchestrator builds a module-level plan and global constraints, then feeds each module (goal, accepted upstream content, downstream dependencies, constraints, accumulated SOP state) to the Protocol Expert Agent. Output is gated against rubric criteria before integration, enabling local detail with global coherence.

**Mechanism 2 — self-evolving skill learning for code:** The Coding Agent draws on a runtime-updated skill library organized into three hierarchical classes (L1 parameter confirmation, L2 deck/sample-layout planning, L3 instrument-operation/code-specification). Skills are created from expert rules, protocol context, SDK info, and prior trajectories; evaluated via validators, code checks, and execution feedback; and revised iteratively. This encodes *transferable operational principles* rather than single-protocol prompt traces, supporting cross-device adaptation.

The system supports **autonomous execution** and **human-in-the-loop** checkpoints. After execution, readouts/outcomes return to the Orchestrator, which updates state, replans, and invokes expert agents for refinement. Wet-lab work used E. coli BL21(DE3)/DH5α, pET-21b(+) and pUC19 backbones, Phanta Flash/Rapid Taq mixes, DpnI, ClonExpress II assembly, T7 Endonuclease I correction, Sanger confirmation, and MGI AlphaTool / Hamilton STAR / Opentrons OT-2 / Tecan EVO platforms.

## Results & Highlights

**Protocol quality (294 tasks, 7-dim rubric):**
- Overall rubric score **94.7 ± 0.4**, beating BioProAgent by 3.2 points and general-purpose LLMs by >6 points.
- Consistent across tiers: L1 96.5, L2 95.1, L3 94.0 (only a 2.5-point L1→L3 decline, far smaller than agentic-harness baselines).
- Strongest separation in scientific annotation depth (D7: 80.7 vs. 50.8–61.0 for harness systems).

**Blind expert evaluation (3 wet-lab scientists, 34 queries, 7 systems):**
- Top@1 **70.6 ± 16.4%**, Top@3 **90.2 ± 9.8%**, MRR **0.82 ± 0.12** (>2× the second system).
- Automated rubric and expert MRR strongly correlated (Spearman ρ=0.929, p=0.003).

**Protocol-to-code (294 tasks):**
- Overall gate pass **96.6%** (vs. max 35.0% for general LLMs, 64.6% LabScript-AI).
- Protocol Score **95.5**; machine safety **94.8%**, bio safety **87.8%**, SDK compliance **98.2%**.
- Alone in the high-quality/high-executability quadrant.

**Cross-device generalization (34 held-out tasks each):**
- Gate pass 91.2% (AlphaTool MGI), 85.3% (Hamilton STAR), 88.2% (OpenTrons OT-2), 91.2% (Tecan EVO) — 5.9-point spread.
- LabScript-AI ranged 8.8–55.9% (47.1-point spread); on OT-2, ProtoPilot's 88.2% beat OpenTrons-AI's 32.4%.

**Wet-lab validation (4 increasing-complexity workflows):**
1. **Foundational ops:** 96-well inoculation OD 0.51–0.88; 8× 12-step tenfold serial dilutions; **24/24 positive colony PCR** (868 bp).
2. **Luciferase plasmid construction:** GLuc-WT & RLuc-WT, 3 replicates each → **100% construction success**, Sanger-confirmed.
3. **Whole-plasmid mutagenesis:** 16 point mutants (8 GLuc + 8 RLuc); **7/8 GLuc, 8/8 RLuc** Sanger-confirmed (only GLuc-L40P failed amplification at 4.8 ng/µL).
4. **PCA DNA assembly (closed-loop):** 4 fragments (BL01–BL04). Diagnosed and fixed two failures — non-specific pUC19 vector primers (redesigned → clean band) and ampicillin inactivation/excess plating volume (fresh plates + reduced volume → clean NTC, single colonies). **93/96 colony PCR positive**; Sanger-confirmed correct clones for all four targets.

## Critical Assessment

- **Novelty — High.** Framing protocol-to-code-to-execution as a *measurable, benchmark-driven agentic problem* with a runtime-evolving skill library and a five-layer representation is a meaningful integration that goes beyond text-only protocol generation (BioProAgent, Thoth) and single-platform code tools (OpenTrons-AI). The hierarchical coordinator–worker + self-evolving skills combination is the genuine contribution.
- **Rigor — Moderately strong.** The benchmark is grounded in 98 gold-standard protocols with a 7-dimension rubric validated against blind expert ranking (ρ=0.929). Code is checked by a three-layer gate (syntax, parameter alignment, SDK export), and four wet-lab workflows are physically executed with Sanger confirmation rather than text-similarity proxies.
- **Reproducibility — Partially uncertain.** Headline metrics are clearly reported with s.e.m., but reproducibility hinges on whether BioLabbench, the rubrics, the Thoth model, and the skill library are publicly released. The paper does not indicate code/data release. Wet-lab execution also depends on specific instruments and reagents not universally available.
- **Significance — Potentially high** for autonomous-experimentation. The 88.2% vs. 32.4% OT-2 delta and the successful closed-loop failure recovery in PCA assembly are striking, practical demonstrations.
- **Red flags / caveats:**
  - Headline pass rates cluster ~88–96%; readers should verify exact definitions of "gate pass," "Protocol Score," and "Top@3 preference" (provided in Technical Validation, not fully reproduced in the main text).
  - The abstract cites 89.5% overall protocol-to-code gate pass and 88.24% Opentrons pass, while the body reports 96.6% overall and 88.2% OT-2 — a minor numerical inconsistency to reconcile.
  - Inter-rater agreement for expert ranking is only moderate (Kendall's W = 0.495), and the blind evaluation used just 34 queries × 3 raters.
  - The single failed mutant (GLuc-L40P) and one BL04 replicate with 0 correct clones show the system is not flawless.
  - Self-reporting of wet-lab successes by the system's own team invites confirmation bias; independent replication would strengthen claims.

## Strengths

- End-to-end framing from natural-language intent through physical execution and feedback, with an explicit five-layer representation and stage-wise guards.
- Expert-grounded benchmark with device-level validity gates and blind human ranking, not just text similarity.
- Runtime-updated, three-class skill library encoding *transferable* operational knowledge across tasks and instruments.
- Real, physically validated wet-lab results (OD, gel, Sanger, colony counts) — including genuine failure-recovery loops.
- Cross-platform robustness across four distinct automation vendors.

## Limitations

- Benchmark scale (294 tasks from 98 protocols) and diversity can be expanded; authors explicitly call for broader assay families, instruments, labware, biological systems, and failure modes.
- Currently automates only the *wet-lab* stage after a user-defined objective; no integrated dry-lab loop (hypothesis generation, sequence/construct design, modeling, analysis) for a full dry–wet cycle.
- Reproducibility depends on release of the benchmark, rubrics, Thoth model, and skill library (not confirmed in the main text).
- Wet-lab validation is performed by the authors' own team; no independent third-party replication.
- Skill library may accumulate errors or drift over long horizons (mechanism for long-term correctness not deeply analyzed).
- Numerical inconsistency between abstract and body pass-rate figures.

## Relevance

该工作把「LLM 智能体 + 自进化技能库」落地到湿实验自动化这一高门槛垂直场景，对关注「多智能体编排」「分层可验证性」与「运行时技能学习」的研究者具有直接方法论参考价值；其状态化、反馈驱动的闭环思路，也可迁移到其他需严格约束与物理执行的 agentic 应用（如化学合成、材料实验自动化）。

## Questions for Further Exploration

- How exactly are "gate pass," "Protocol Score," and "Top@3 preference" computed, and why do the abstract (89.5% / 88.24%) and body (96.6% / 88.2%) protocol-to-code/Opentrons figures differ?
- Will BioLabbench, the Thoth model, the expert rubrics, and the skill library be publicly released, and under what license?
- How many feedback iterations did the PCA assembly require, and what is the marginal cost / latency per closed-loop revision?
- Does the self-evolving skill library drift or accumulate silent errors over long deployment horizons, and is there a forgetting/curruption safeguard?
- How does ProtoPilot generalize beyond synthetic/molecular biology (e.g., cell-based assays, protein purification, chemistry)?
- Can the wet-lab results be independently replicated by a third-party lab using the released system?
- How does performance degrade on truly novel protocols absent from the 98 gold-standard set, and where does the RL-trained Thoth model's ceiling lie?
