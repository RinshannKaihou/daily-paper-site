⚠ Critical Edge

03 Sharpest Critique

The zero-violation guarantee rests entirely on the verifier being exact and complete. It does not certify verifier soundness and breaks under approximate or partially-specified constraint sets — the noise-robustness result is empirical, not certified.

01

Methodology — the contract, not a heuristic

Operator C = ( D , Verify , V̂ , π_F )

D  untrusted proposal source
Verify  exact feasibility gate
V̂  conformal value boundary
π_F  trusted oracle solver

trusted core (owns safety)   arbitrarily untrusted

CGPA commits the longest verifier-safe prefix of an untrusted draft that still obeys a local value-descent guard; the moment it can't, it defers to one verified oracle repair.

Safety is owned by the exact verifier — it never queries the value boundary an attacker could poison. Regret is bounded per accepted segment; speed accrues at every accepted step. Remove any trusted part and the contract collapses to a known weaker baseline (Prop 2: the decoupling is necessary, not stylistic).

// value guard, Eq. 3
g(s_t, u) + V̂(s_{t+k}) ≤ V̂(s_t) + τ·|V̂(s_t)|

A five-quantile MLP boundary with split-conformal calibration turns the boundary error ε from an assumption into a measured, state-dependent band (Prop 4, Pr ≥ 1−δ). τ is the regret-budget knob.

A full-knowledge white-box adversary (knows verifier, value boundary, dynamics, cost) drives raw execution to a breach in every episode — yet CGPA defers at every step: amortization collapses to zero, violations stay zero. Safety is held; only speed is lost.

  • Capability-graduated throughput. Stronger quality drafters speed up the certified system at unchanged guarantees; parameter count is non-predictive. "The better the agent, the faster the system."
  • Distribution-shift upside. An OOD drafter overtakes the stepwise oracle on 15/18 days (vs expected 50% in-distribution) — accepted prefixes beat the oracle's own myopic plan, at zero violations.
  • One contract, six LLMs. The same operator absorbs a 1.7B and a 12B model: Qwen3-8B regret 0.349→0.032 (10.9×); Gemma-4-12B 20.38→0.032.
03

Strengths & Limitations

Strengths

  • Principled, contract-based decoupling of safety / regret / speed, with necessity proofs for each trusted part.
  • ε is measured via conformal calibration; worst-case certificate paired with a high-probability forward guarantee.
  • Adversarial hardening: white-box + learned attackers both fail to induce a single applied violation.
  • Statistically honest — 95% CIs, p95 regret, seed-disjoint calibration, a 0.50 in-distribution sanity check, second-instance replication (2.09×).

Limitations

  • Guarantee is conditional on an exact, complete verifier — does not certify verifier soundness or extend to partial constraints.
  • Scope is energy/operations domains with linear / mixed-integer structure; nonlinear, continuous, or partially-observable settings unproven.
  • Benefit vanishes when the trusted oracle is cheap — baselines match it near 1.0× there.
  • Learned boundary adds training/calibration dependencies; no code or artifact release link evident in the PDF.
04

Open Questions

  1. Can a conservative surrogate verifier recover a certified (if looser) bound when verification is itself approximate — beyond the empirical false-accept = 0?
  2. Does the certificate-aware boundary transfer across domains, or must it be retrained per problem class — and is there a zero-shot variant?
  3. What is the mechanism behind accepted prefixes beating the stepwise oracle's own cost under distribution shift — genuine multi-step advantage or frontier artifact?
  4. Is there a principled, automated procedure for choosing τ in safety-critical deployment, rather than hand-selecting on the frontier?
