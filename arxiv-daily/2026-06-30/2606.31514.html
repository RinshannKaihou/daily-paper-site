VGG-16 dynamic mode, +82% over the INT8 baseline — the highest among compared Zynq-7020-class designs, within a 2% accuracy budget.

02 Why This Matters

A clean hardware–algorithm interface for low-power edge inference: precision becomes a runtime cycle count with zero reconfiguration, and a cycle-aware greedy search ties per-layer quantization directly to energy savings.

03 Critical Edge

Accuracy is measured on Imagenette-320 (10 classes), not full ImageNet-1k — the headline 2% budget sits on a much easier task. The 128-PE array is modelled/extrapolated, not fully place-and-routed. ⚠ transfer to ImageNet-1k unproven · FP32 accumulation = upper bound

How it works

Methodology

Arithmetic core — MSDF, left-to-right

Most-significant-digit-first arithmetic emits the highest-weight output digit first. To do that it needs redundancy: MINT uses radix-2 signed digits {−1, 0, 1}, each encoded as a 2-bit pair (p, n) = p − n. After a short online delay, one digit drops per cycle — so stopping after P digits is a bounded-error P-bit approximation. That is the whole trick.

9-tap PE latency:  C(P) = 2P + 10 cycles

9 serial-parallel MSDF multipliers → 4-stage online adder tree (δ_add = 2 each). Precision–latency tradeoff is structural, not switched.

Hardware–software co-design flow

01Sensitivity probe. From an FP32 net, sweep uniform INT2→INT8 and quantify per-layer sensitivity (one layer quantized, rest at INT8).
02Budget-constrained greedy. Assign lowest viable per-layer precision under Δ_max = 2%, scoring by cycle-aware efficiency.
03Hardware characterization. Vivado synthesis + post-synth sim + SAIF switching activity on the single PE.
04Combine. Merge the precision map with PE characterization for throughput / energy / latency.
η = (cycle_saving × mac_frac) / (dℓ,p + ε)

The η score couples layer sensitivity, MAC fraction, and the actual MSDF cycle model — not abstract bit-cost.

Strengths & limits

Critical assessment

✓ Strengths

  • Elegant core idea: precision = number of MSDF cycles, no reconfiguration / mode-switch / datapath change.
  • Principled cycle-aware greedy search — η couples sensitivity, MAC fraction, and the real cycle model, not bit-count.
  • Commendably honest framing: explicitly states raw throughput is lower; claims only the energy-efficiency crown on a small cheap part.
  • Strong, clearly reported energy / PDP numbers and a useful per-PE LUT/FF/power table across INT2–INT8.
  • Concrete, verifiable target: Zynq-7020 @ 200 MHz, named networks.

! Limitations

  • Accuracy on Imagenette-320 (10 classes), not full ImageNet — the 2% budget is on an easy task.
  • Only two classic CNNs; no transformers, MobileNet, or depthwise/1×1 convs that stress the 9-tap PE.
  • 128-PE array is extrapolated from single-PE synthesis — no full post-P&R; power uses assumed utilization (0.88) / activity (0.50).
  • FP32 accumulation + only Conv/Linear quantized ⇒ reported accuracy is an upper bound on hardware accuracy.
  • No DSP → low absolute throughput (~19.9 GOPS); positioned as energy-efficient, not real-time.
  • SD digits cost 2 bits each, partly offsetting the sub-INT8 advantage — and no released RTL / search code.

Open questions

For further exploration
?Does the 2% budget at ~5.6–6.0 bits survive on full ImageNet-1k, or on harder detection/segmentation heads?
?How does the 9-tap PE handle depthwise/1×1 convs and transformer matmuls, where inner-product structure differs?
?What are the full-array post-P&R numbers, and how sensitive is GOPS/W to the assumed 0.88 / 0.50 factors?
?Would learned/RL search (HAQ-style) beat the greedy η score on average bit-width under the same budget?
?Given SD digits cost 2 bits each, what is the true effective storage/interconnect cost vs the "5.64 bits" figure?
?Can deeper digit-level pipelining close the throughput gap to DSP-based designs in real end-to-end latency?
