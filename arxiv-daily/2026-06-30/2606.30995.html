Multistage Defer Trees for Hybrid Interpretability: If at First You Can't Succeed, Tree Again

Zakk Heile · Hayden McTavish · Margo Seltzer · Cynthia Rudin — Duke · UBC

interpretability decision-trees deferral hybrid-models tabular-data learning-to-defer
Key Metric
40%

deferral rate at which MDT+XGB comes within 2.0% of XGBoost — on every one of 24 datasets.

Why This Matters

High-stakes tabular domains — credit, justice, churn — need both transparency and accuracy. This sparse-tree cascade gives practitioners a deployable template for hybrid interpretable systems that don't sacrifice ensemble performance.

Critical Edge
No fairness audit

The hardest, often most consequential cases are routed to the opaque black box — whose deferred-region test error can be >10× higher — yet the authors never test whether deferral correlates with protected attributes. On Compas & HELOC, "interpretability for most" can hide opacity exactly where it matters.

Methodology

a cascade of sparse trees + a black-box tail

A defer tree is a binary tree whose leaves output a label or a defer action. The hard cases fall through to the next stage, and finally to a fallback B (XGBoost).

T₁ sparse → defer → T₂ → defer → T_K → defer → XGBoost

Alternating optimization (Alg. 1): at each stage, fit fallback + defer tree on a reweighted dataset, then shrink the deferred region.

  1. Fit B and defer tree Tₖ on the currently weighted full dataset.
  2. Update the active deferred set; stop early if it stops shrinking (guaranteed monotone, Prop. 1).
  3. Feasible-region tracking: intersect defer-leaf hyperrectangles; drop splits constant across feasible subregions.
  4. Recompute distance weights — deferred points → weight 1, others decay as (1−μ)(1+defer_dist)−γ in quantile-ℓ₁ space.
  5. Optional τ rescale; final fallback retrain if K_max leaves residue.

Why multi-stage: a single optimal defer tree carries a certificate (Prop. 2) that no further interpretable split helps — the region is genuinely stuck. Hence the cascade.

Results & Highlights

24 datasets · 5 splits each · strong baselines
  • Fig.3 At ~40% deferral, MDT+XGB is within 2.0% of XGBoost on all 24 datasets — no prior hybrid method achieves this without deferring a majority.
  • Fig.2 On high-tradeoff sets (California, Bike, Rl, Wine) it dominates every hybrid baseline (HyRS, Pre/Post-CORELS, FLMM, LogReg+deferral).
  • Fig.2 On California, matches XGBoost's mean accuracy at only 25% deferral — unique among all methods.
  • Fig.4a At 50% deferral, no dataset needs >7.5 avg splits; most need ≤5.5 through the interpretable portion.
  • Fig.4b Final XGBoost compresses heavily (leaf ratios up to ~12×) with deferred-region accuracy statistically indistinguishable from full.
  • Tbl.1 Beats sparse ensemble FIGS on nearly all datasets; beats the single-tree ablation — confirming the cascade does the work.

Strengths

principled · reusable · honest
  • Get interpretability "for free" on the easy majority while preserving ensemble accuracy; complexity allocates adaptively by region difficulty.
  • Distance reweighting is a theoretically grounded fix (Gibbs / entropy-regularized derivation) to deferred-region overfitting.
  • Feasible-region tracking simultaneously speeds training, enables pruning, and compresses both the MDT and the fallback ensemble.
  • Honest setup: strong baseline tuning (12-hr Optuna/split), bootstrap CIs, and an explicit non-deferral ablation isolating the cascade's contribution.
  • Compact rule-list / sparse-tree output that is directly deployable; credible authors with a GOSDT/FIGS/LicketySPLIT open-tooling lineage.

Limitations

opaque tail · no audit · fragile compression
  • Opaque tail & deferral bias: a black box still handles the hardest cases; coverage is statistical, not per-individual, and may overlap with sensitive groups (untested).
  • No fairness audit despite using fairness-sensitive benchmarks (Compas, HELOC, Adult).
  • No human-subject validation — "interpretability" is operationalized purely as split / rule counts.
  • Cascaded paths (2–3 trees) are harder to reason about than a single tree — eroding interpretability where the model is least sure.
  • Many hyperparameters (τ, η, μ, γ, K_max, depth) plus a deferral-budget selection step add pipeline complexity.
  • Compression is fragile in some regimes (Madeline leaf ratio 11.96× ± 24.21×).
  • No code link in the preprint; depends on unpublished LicketySPLIT modifications.
  • Scope: binary tabular classification only — multiclass / regression / non-tabular unaddressed.

Questions for Further Exploration

where this work points next
Fairness of deferral — what fraction of deferred samples belong to protected subgroups, and does the black box err systematically there?
Cascade vs. single tree — how does a full 2–3-tree path compare in a controlled human study, not just by split count?
Human-in-the-loop — could the final black-box deferral be swapped for an expert, and how shifts the accuracy–deferral curve?
Joint sensitivity — is there one robust operating point for μ, γ, K_max, or must it be tuned per dataset?
Beyond tabular — does it extend to multiclass / regression / text / images where single trees are weak?
Ensembled fallbacks — could multiple fallback classes close the deferred-region error gap the authors themselves flag?
