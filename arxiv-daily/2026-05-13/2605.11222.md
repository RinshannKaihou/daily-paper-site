> **一句话总结：** ADMM-Q 将 LLM 逐层权重量化建模为 Hessian 加权的约束优化问题，通过 ADMM 算子分裂交替执行连续权重更新与离散量化投影，辅以对角预条件、自适应罚参数和 pair-swap 局部搜索，在 Qwen3-8B 上将 W3A16 的 WikiText-2 困惑度从 12.85 降至 10.06，W2A4KV4 SpinQuant 下从 66.11 大幅降至 19.42，显著优于 GPTQ。

> **TL;DR:** ADMM-Q replaces GPTQ's greedy column-wise quantization with a principled ADMM-based solver that jointly optimizes all weights per layer. It alternates continuous Hessian-weighted updates with discrete quantization projections, using diagonal preconditioning (critical — removing it worsens W3A16 PPL from 10.06 to 14.68), adaptive penalty scheduling, grid refresh, and a GPU-parallelized pair-swap local search. On Qwen3-8B, it improves WikiText-2 PPL in W3A16 (12.85→10.06), W4A8 SmoothQuant (9.29→8.68), and W2A4KV4 SpinQuant (66.11→19.42). Gains are consistent across 5 Qwen3 sizes (1.7B–32B) and 2 LLaMA models, with identical inference throughput to GPTQ in vLLM.

## 核心贡献

- 将逐层权重量化建模为 Hessian 加权约束优化问题，提出基于 ADMM 组合变体的求解器，联合优化所有权重而非逐列贪心处理
- 引入对角预条件（Diag(H)^{-1/2} 缩放）解决 LLM 通道异常值导致 Hessian 病态的问题，消融实验表明这是最关键的组件
- 提出 grid refresh 机制：随 ADMM 迭代更新量化网格参数，使离散投影与连续轨迹保持对齐（GPTQ 无法做到）
- 设计 GPU 并行化的 pair-swap 局部搜索后处理，通过 O(1) 的 Hessian/梯度项评估候选交换，最多 5 轮迭代
- 证明 ADMM 在量化约束下的收敛性（Theorem 1），将 Meng et al. (2024) 在稀疏性设定下的证明扩展到量化场景
- ADMM-Q 是模块化的即插即用替换，可与 SmoothQuant（缩放）、SpinQuant/QuaRot（旋转）、MSE clipping 等技术完全组合

## Key Contributions

- Formulates layer-wise weight quantization as a Hessian-weighted constrained optimization problem and proposes an ADMM-based solver that jointly optimizes all weights (vs. GPTQ's greedy column-by-column approach)
- Introduces diagonal preconditioning (Σ = Diag(H)^{-1/2} scaling) to handle the ill-conditioned Hessian caused by LLM channel outliers — ablation shows removing it worsens W3A16 PPL from 10.06 to 14.68, worse than GPTQ's 12.85
- Proposes grid refresh: recomputes quantization grid parameters (scale/zero-point) from current continuous iterates during ADMM, keeping the discrete projection aligned with the continuous trajectory
- Designs a GPU-parallelized pair-swap local search post-processing that evaluates candidate swaps in O(1) per entry using only 3 Hessian entries and 2 gradient entries, running at most 5 rounds
- Proves convergence of ADMM under quantization constraints (Theorem 1): with ρ_t = ρ₀·γ^t (γ>1), both continuous and discrete iterates converge to a shared limit point
- ADMM-Q is a modular drop-in replacement composable with SmoothQuant, SpinQuant, QuaRot, and MSE clipping

## Methodology

**Problem formulation.** Given pretrained weights W̃ and calibration activations X, ADMM-Q minimizes the layer-wise reconstruction error Tr(W−W̃)ᵀH(W−W̃)/2 subject to W ∈ Q (quantization grid), where H = XᵀX + λI is the Hessian with damping.

**Diagonal preconditioning.** The Hessian H is poorly conditioned due to channel outliers. ADMM-Q applies Σ = Diag(H)^{-1/2} scaling so the scaled Hessian ΣᵀHΣ has unit diagonal, ensuring all channels contribute comparably to updates.

**ADMM algorithm (Algorithm 1).** Introduces auxiliary variable D to decouple continuous optimization from quantization constraint. Each iteration: (1) W-update: closed-form via pre-computed eigendecomposition H = UΛUᵀ, giving W^{t+1} = U(Λ+ρ_t I)^{-1}Uᵀ(HW̃+ρ_t D^t−V^t); (2) D-update: RTN-style projection onto quantization grid (ΣD ∈ Q); (3) V-update: dual variable adjustment; (4) ρ_{t+1} = ρ₀·γ^t with γ>1.

**Grid refresh.** At most once per run, the quantization grid parameters (Δ, z₀) are recomputed from current continuous weights and accepted only if the D-update objective improves. Dual variable is adjusted to maintain ADMM state consistency.

**Local search.** After ADMM converges, a pair-swap procedure selects row pairs (i₁,i₂), evaluates all four sign combinations per column via ΔL formula using only H_{i₁i₁}, H_{i₂i₂}, H_{i₁i₂} and G_{i₁j}, G_{i₂j}, applies best swap, updates gradient via rank-2 correction. Up to 5 rounds with GPU-parallel batched evaluation.

**Practical settings.** T=300 ADMM iterations, ρ₀=0.1, γ=1.1, damping λ=0.01·Tr(XᵀX). MSE grid search with 100 points and 0.8 max shrink ratio for clipping.

## Results & Highlights

**Weight-only quantization (Qwen3 family, Table 1):**
- W4A16: ADMM-Q improves over GPTQ on all 5 model sizes (1.7B–32B) across WikiText-2, C4, PTB perplexity and zero-shot accuracy
- W3A16 Qwen3-8B: WikiText-2 PPL 12.85→10.06, C4 21.27→18.25, PTB 35.01→27.30, zero-shot 54.43→57.80
- W2A16: GPTQ/AWQ collapse on larger models (PPL >1e7 for AWQ on 1.7B); ADMM-Q maintains reasonable PPL (e.g., 24.04 on 1.7B)
- Layer-wise analysis: ADMM-Q achieves lower reconstruction error in all 36 layers of Qwen3-8B, with early layers reduced to as low as 10% of GPTQ's error

**SpinQuant WbA4KV4 (Table 2, Qwen3-8B/LLaMA-3 8B/LLaMA-3.2 1B):**
- W2A4KV4 Qwen3-8B: PPL 66.11→19.42 (GPTQ→ADMM-Q), zero-shot 43.28→46.29
- W4A4KV4 LLaMA-3 8B: PPL 7.26→7.17 (best config), consistent gains across symmetric/asymmetric × per-channel/group-128

**SmoothQuant WbA8 (Table 3, Qwen3-8B):**
- W4A8: PPL 9.29→8.68, W3A8: 16.82→15.69, W2A8: 130.04→68.70

**vLLM inference (Table 4):** ADMM-Q achieves identical throughput to GPTQ (e.g., 1.52× speedup on Qwen3-32B W8A8 SpinQuant), since both produce same quantized format

**Ablation (Table 6, Qwen3-8B):**
- Diagonal preconditioning is most critical: removing it at W3 worsens PPL 10.06→14.68 (worse than GPTQ's 12.85)
- Adaptive ρ: removing it at W3 worsens PPL 10.06→10.97
- Local search: removing it at W3 worsens PPL 10.06→9.93 (slight benefit), but critical in SpinQuant (LLaMA-3 8B W4A4KV4: 7.26→7.97 without it)
- All off at W3: catastrophic 10.06→42.47

**Runtime (Table 5):** ADMM-Q is slower than GPTQ (e.g., 117 min vs 34 min on Qwen3-32B) but practical; peak GPU memory ~6% higher

## Critical Assessment

- **Novelty:** High. The application of combinatorial ADMM to LLM weight quantization is novel and fundamentally different from GPTQ's greedy approach. The diagonal preconditioning insight (channel outliers → ill-conditioned Hessian → ADMM instability) is well-motivated. The grid refresh mechanism is clever and exploits a unique advantage of ADMM over GPTQ.
- **Rigor:** Very strong. Convergence proof (Theorem 1) with proper assumptions. Comprehensive ablation studies isolating each component's contribution. Evaluation across 7 models, 3 bit-widths, 3 pipelines, 3 perplexity benchmarks, and 9 zero-shot tasks.
- **Reproducibility:** Good. Algorithm is fully specified (Algorithm 1), hyperparameters are stated (T=300, ρ₀=0.1, γ=1.1), and all evaluation protocols are described. Code release is not mentioned, which is a concern.
- **Significance:** High, especially for sub-4-bit quantization. The W2A4KV4 improvement on Qwen3-8B (66→19 PPL) transforms a practically unusable model into one with reasonable quality. The modularity argument is compelling — any pipeline using GPTQ can swap it out.
- **Red flags:** (1) Runtime overhead is significant (~3.5× slower on 32B model). (2) Only evaluated on decoder-only dense transformers — no MoE, diffusion, or ViT results. (3) The paper acknowledges dependence on calibration set representativeness but doesn't study this sensitivity. (4) No comparison with very recent sub-4-bit methods beyond GPTQ/AWQ (e.g., QuIP#, AQLM).

## Strengths

- Principled optimization framework with convergence guarantees — fundamentally sounder than heuristic approaches
- Dramatic improvements at aggressive quantization levels where existing methods catastrophically fail
- Modular drop-in design: works seamlessly with SmoothQuant, SpinQuant, QuaRot, and clipping heuristics
- Excellent ablation study clearly isolating the contribution of each component
- Identical inference throughput and memory to GPTQ (only offline cost differs)
- Strong theoretical grounding from the optimization community with formal convergence proof

## Limitations

- Significant offline quantization runtime overhead: ~3.5× slower than GPTQ on 32B models (117 vs 34 min)
- Only evaluated on dense decoder-only LLMs (Qwen3, LLaMA-3); no results for MoE architectures, vision transformers, or diffusion models
- No comparison with other recent sub-4-bit quantization methods like QuIP# or AQLM
- Performance depends on calibration set representativeness, which is not studied
- Grid refresh is limited to at most one acceptance per run — could be suboptimal for some layers
- Peak GPU memory is ~6% higher than GPTQ during quantization

## Relevance

与"计算可靠性与精度误差"和"LLM 隐层状态与可解释性"两个方向均高度相关：ADMM-Q 直接解决低比特量化中的数值精度损失问题，其对角预条件技术处理了 LLM 中通道异常值导致的 Hessian 病态，本质上是利用模型内部表示结构的统计特性来改善数值稳定性，同时收敛性保证提供了形式化的可靠性论证。

## Questions for Further Exploration

- Can the ADMM iterations be warm-started from GPTQ's solution to reduce the runtime gap?
- How does ADMM-Q interact with quantization-aware training (QAT) — could it serve as initialization for fine-tuning?
- Does the grid refresh frequency matter beyond the single-refresh limit? Could adaptive multi-refresh help for very low bit-widths?
- How sensitive is ADMM-Q to the damping parameter λ = 0.01·Tr(XᵀX) and the calibration set size/composition?
- Can the pair-swap local search be extended to group-level swaps (beyond row pairs) for groupwise quantization settings?
