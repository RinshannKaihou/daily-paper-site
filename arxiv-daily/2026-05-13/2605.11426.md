> **一句话总结：** 本文利用GemmaScope 2预训练稀疏自编码器作为诊断工具，揭示SFT虽然保持原始激活余弦相似度>0.96，但底层稀疏特征在深层网络中崩塌至0.557，并发现安全对齐是唯一在浅层集中扰动的任务，且SFT并非"外科手术式"——它会系统性地破坏与任务无关的Code特征。

> **TL;DR:** This paper introduces a mechanistic pipeline using pretrained SAEs (GemmaScope 2 on Gemma 3 1B) to diagnose SFT's representational impact. Across 4 tasks (MultiNLI, GSM8K, WildJailbreak, ToolCalling), raw activation cosine similarity stays >0.96, but SAE latent similarity drops as low as 0.557 at deeper layers. Safety alignment (WildJailbreak) uniquely concentrates disruptions in early layers (L7/L22 ratio = 3.0), unlike all other tasks. SFT also systematically disrupts task-irrelevant features (especially Code features), revealing it is not truly surgical. The paper uses SVD on activation deltas, perturbation probing with flip-rate analysis, and automated LLM-as-judge annotation (GPT-5.4-mini) over a 7-category semantic taxonomy.

## 核心贡献

- **提出基于预训练SAE的SFT机制分析流水线**：利用GemmaScope 2（冻结权重的稀疏自编码器）将SFT前后的隐藏激活投影到同一语义空间，量化表示漂移。
- **揭示"表面稳定、深层崩塌"现象**：原始激活余弦相似度始终>0.96，但262k SAE潜变量在Layer 22可降至0.557（MultiNLI），证明SFT的影响远比表面指标所示剧烈。
- **发现安全对齐的层级特异性**：WildJailbreak是唯一Layer 7/Layer 22翻转特征比>1（比值为3.0）的任务，表明安全对齐在浅层集中重写表示，与其他所有任务模式相反。
- **证明SFT并非"外科手术式"**：Code特征在不含编程成分的任务（GSM8K、MultiNLI）中被大量破坏，说明附属干扰是SFT的固有特征而非任务特异伪影。
- **建立7类语义分类法**：通过GPT-5.4-mini自动标注，将受影响的SAE特征分为Persona、Structure、Code、Reasoning、Safety、Multilingual、Collateral七类。

## Key Contributions

- Introduces a novel 4-stage mechanistic pipeline: (1) per-task SFT of Gemma 3 1B, (2) tracking raw vs. SAE latent cosine similarity across training, (3) SVD on activation-delta matrices to identify principal drift directions, (4) perturbation probing to measure feature flip rates through the frozen SAE.
- Demonstrates that raw activation cosine similarity (>0.96 across all tasks/layers) is a misleading proxy for representational stability: SAE latent similarity degrades continuously, reaching 0.557 at Layer 22 for MultiNLI.
- Discovers that safety alignment (WildJailbreak) is the only task with a Layer 7 / Layer 22 flipped-feature ratio > 1 (3.00), meaning it concentrates representational rewrites in early layers — a structural asymmetry not seen in any other SFT objective.
- Shows that Collateral features dominate the geometrically principal SFT update axis across all tasks, and Code features are systematically disrupted even in non-programming tasks (25% at Layer 22 for GSM8K, 40% for MultiNLI), proving SFT is not surgical.
- Develops an automated LLM-as-judge annotation pipeline (GPT-5.4-mini) over a 7-category semantic taxonomy to classify disrupted features at scale.

## Methodology

**Model and SAEs:** Gemma 3 1B IT (d_model=1152, 26 layers) with GemmaScope 2 SAEs at three dictionary widths (16k, 65k, 262k) using JumpReLU activation, probed at layers 7 (early), 13 (mid), and 22 (late). SAEs are frozen throughout — they serve purely as diagnostic instruments.

**Tasks:** Four structurally diverse SFT objectives — MultiNLI (logical reasoning), GSM8K (math), WildJailbreak (safety/refusal), OpenAI Tool Calling (JSON invocation) — each with 2000 training examples (50 for Tool Calling), full-parameter fine-tuning with Adafactor at lr=2e-5.

**Pipeline stages:**
1. Fine-tune separate copies of the base model per task, checkpointing every 400 steps.
2. Compute both raw activation cosine similarity and SAE latent cosine similarity on 100-example held-out sets at each checkpoint.
3. Construct the activation-delta matrix ΔH = H_SFT − H_base, apply truncated SVD (capped at 50 components, 90% variance), and compute cosine similarity between principal drift directions and SAE decoder feature vectors.
4. Inject principal components into baseline activations (1-std-dev perturbation) and pass through the frozen SAE to identify feature flips (active↔inactive transitions), computing per-direction flip rates and per-feature flip frequencies.
5. Classify disrupted features via Neuronpedia API descriptions + GPT-5.4-mini into a 7-category taxonomy.

**Key metrics:** Cosine similarity (raw and latent), flip rate per direction, per-feature flip frequency, outlier directions (z-score > 1.5), semantic cluster distributions per task/layer.

## Results & Highlights

- **Raw activation stability is deceptive**: cosine similarity remains >0.96 throughout training for all tasks and layers, e.g., WildJailbreak Layer 22 shifts only from 0.984 to 0.980 over 2000 steps.
- **SAE latent collapse in deep layers**: MultiNLI Layer 22 latent similarity drops from 0.740 (step 400) to 0.557 (step 2000). GSM8K Layer 22 decays from 0.793 to 0.708. This is consistent across all SAE widths.
- **Early-layer SFT is low-rank, late-layer is distributed**: At Layer 7, a single PC explains 67–89% of drift variance. At Layer 22, leading PC variance drops to 17–33%, requiring more components.
- **Safety alignment inverts the layer profile**: WildJailbreak has 30 flipped features at Layer 7 vs. 10 at Layer 22 (ratio 3.0). All other tasks have ratio ≤1.0, meaning they disrupt deeper layers more.
- **Collateral features dominate the principal drift axis**: Across all tasks and both layers, Collateral features account for the largest share of SAE dictionary features with |cosine similarity| > 0.5 with the principal outlier direction.
- **Code features are universally disrupted**: Code is the largest disrupted cluster at Layer 22 for GSM8K (25%) and MultiNLI (40%), despite no programming content in either task.
- **Wider SAEs reduce Collateral prevalence**: At Layer 22, Collateral percentage decreases monotonically with SAE width (e.g., ToolCalling: 42.5% → 26% → 12% for 16k → 65k → 262k), suggesting improved disentanglement.

- **Ruhaan Chopra** — Independent researcher. Single-author work with publicly available code (https://github.com/ruhzi/sae-investigation). ## Critical Assessment

- **Novelty:** High. The combination of frozen SAE projection + SVD drift analysis + perturbation flip-rate probing is novel. The safety-alignment layer-inversion finding is original and surprising. The non-surgical nature of SFT (task-irrelevant Code disruption) challenges prevailing assumptions.
- **Rigor:** Strong. Systematic evaluation across 4 tasks, 3 layers, 3 SAE widths, and multiple checkpoints. The 7-category taxonomy is inductively derived and validated via automated annotation. The frozen-SAE methodology provides a principled causal attribution of changes to SFT.
- **Reproducibility:** Good. All code is open-source. Uses publicly available models (Gemma 3 1B, GemmaScope 2) and standard datasets. However, the GPT-5.4-mini annotation step introduces a proprietary dependency.
- **Significance:** High for mechanistic interpretability and alignment research. The finding that safety alignment operates through fundamentally different layer-wise mechanisms than other SFT objectives has practical implications for alignment training and evaluation. The non-surgical nature of SFT raises concerns about unintended capability degradation.
- **Red flags:** (1) Single-author work — limited breadth of evaluation (only 1B model). (2) The 7-category taxonomy is subjective and derived from manual inspection of a limited feature set. (3) Tool Calling had only 50 training examples, making its results less robust. (4) The paper does not connect representational changes to downstream behavioral metrics (e.g., does Code feature disruption at Layer 22 actually impair code generation?). (5) GPT-5.4-mini annotation accuracy is not validated against human labels.

## Strengths

- Clean, well-motivated experimental design with frozen SAE as an unbiased diagnostic.
- Multi-dimensional analysis (cosine similarity, SVD, flip rates, semantic taxonomy) provides converging evidence.
- The safety-alignment layer-inversion finding is novel and has practical implications.
- Open-source code and use of publicly available tools enhance reproducibility.
- Clear writing with extensive tables and structured results.

## Limitations

- Limited to Gemma 3 1B and GemmaScope 2 — generalization to other architectures/scales is unknown.
- No evaluation of whether observed representational changes correlate with behavioral changes (e.g., does Code feature disruption actually degrade code performance?).
- The 7-category taxonomy is coarse and may miss important nuances in feature semantics.
- Tool Calling task had only 50 training samples, making its results less statistically reliable.
- GPT-5.4-mini annotation is not validated against human ground truth; annotation quality is assumed.
- The paper does not address whether the findings extend to other fine-tuning methods (LoRA, QLoRA, RLHF).

## Relevance

本研究通过机制可解释性方法揭示了SFT和安全对齐在模型内部表示空间中的不同运作机制，发现安全对齐特有的浅层集中模式，对理解和对齐安全训练的内在机理具有重要参考价值。

## Questions for Further Exploration

- Does the safety-alignment early-layer concentration pattern hold for RLHF/DPO/constitutional AI methods, or is it specific to SFT-based refusal training?
- Can the SAE latent cosine similarity trajectory during SFT serve as a real-time training diagnostic to detect unintended capability loss?
- Do the observed Code feature disruptions at Layer 22 in non-code tasks actually cause measurable degradation in code generation benchmarks?
- How do the findings change with parameter-efficient fine-tuning methods (LoRA, adapters) versus full-parameter SFT?
- Can the layer-wise update profile of safety alignment be exploited to design more robust safety training that resists fine-tuning-based jailbreaks?
