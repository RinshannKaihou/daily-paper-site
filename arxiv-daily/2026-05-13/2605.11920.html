Domain Restriction via Multi SAE Layer Transitions

arXiv: 2605.11920

Date: 2026-05-13

https://arxiv.org/abs/2605.11920

一句话总结： 本文利用预训练稀疏自编码器（SAE）将LLM各层的残差流表征转为稀疏二进制SDR序列，通过建模层间特征转移概率（Markov/HTM/RNN）来检测越域输入，在far-OOD和多个near-OOD任务上表现强劲，并提供可解释的层间动态分析。
TL;DR: Proposes an ID-only OOD detection method for LLMs by modeling depth-wise transitions of SAE-derived sparse features (SDR sequences) using lightweight sequential scorers (first-order Markov, HTM, RNN). Without fine-tuning the base LLM or using labeled OOD data, the method achieves strong far-OOD AUROC (0.95–0.99 on Gemma-2 2B/9B) and competitive near-OOD performance. A first-order Markov model suffices, and decoded SAE transitions provide human-interpretable explanations for detection decisions.

核心贡献

  • 提出一种仅需域内数据（ID-only）的LLM越域检测方法，通过建模SAE特征在Transformer各层间的转移模式来判别输入是否在域内，无需微调基座模型或使用OOD标注数据。
  • 设计了四阶段流水线：激活提取 → SAE稀疏分解（Top-k二值化SDR） → 序列异常评分（Markov/HTM/RNN三种后端），完全基于域内数据训练。
  • 在Gemma-2 2B和9B模型上进行了全面评估，涵盖far-OOD（20NG vs. SST-2/MNLI/RTE/IMDB/CLINC150）和near-OOD（AGNews/ROSTD/SNIPS/CLINC150类内分割）两种场景。
  • 通过"硬OOD"案例研究，利用Neuronpedia解码SAE特征转移路径，证明检测器的"误判"实际上是语义边界重叠导致的合理行为，提供了可解释性分析。

Key Contributions

  • Introduces an ID-only scope-gating method that detects out-of-domain inputs by modeling depthwise transitions of SAE features — no fine-tuning of the base LLM or labeled OOD data required.
  • Designs a 4-stage pipeline: activation extraction → SAE sparse decomposition with global-density masking and Top-k binarization → sequential anomaly scoring via Markov/HTM/RNN backends.
  • Demonstrates that a simple first-order Markov transition model on SAE-SDR sequences is sufficient and slightly outperforms higher-order backends (HTM, RNN), suggesting transformer depth dynamics are approximately first-order Markovian for ID/OOD separation.
  • Provides interpretability analysis: decoded SAE feature transitions via Neuronpedia reveal that "hard OOD" errors correspond to semantically ambiguous boundary cases rather than spurious failures.
  • Comprehensive benchmarking on Gemma-2 2B and 9B across far-OOD and near-OOD settings.

Methodology

The method operates as a four-stage pipeline:

  1. Activation Extraction: For input tokens, residual-stream hidden states are extracted across L continuous transformer layers (post-layer), with padding masked out.
  1. Sparse Feature Decomposition: Each layer's activations are encoded through a pre-trained 16k-dimensional SAE. Token-level SAE features are mean-pooled across positions. A global-density feature mask removes overly frequent features (using Neuronpedia density statistics, threshold θ). The result is binarized via Top-k selection (default k=10), producing a binary SDR sequence {s₁, ..., sₗ}.
  1. Sequential Anomaly Scoring: Three backends are compared:
  • Markov (default): Counts adjacent-layer feature co-activations from ID data, estimates sparse transition probabilities with Laplace smoothing, and scores test inputs by negative mean log-likelihood of active transition pairs.
  • HTM: Hierarchical Temporal Memory anomaly scoring (fraction of unpredicted active bits).
  • RNN (LSTM): Next-layer predictor trained with BCE loss on ID sequences.

Per-layer anomaly scores are averaged across depth (layers 2 to L).

  1. Training: All sequential models are trained on ID data only — no OOD samples needed. The SAEs are pre-trained and frozen (from SAELens/Neuronpedia ecosystem).

Key design insight: SAEs provide a common coordinate system across examples and layers, and the sparse binary representation enables efficient transition modeling with lightweight sequential models.

Results & Highlights

Far-OOD (ID: 20 Newsgroups):

  • Gemma-2 2B + Markov: AUROC 0.917–0.985 (varies by OOD dataset), competitive with LR baseline but lower AUPR and higher FPR@95.
  • Gemma-2 9B + Markov: Consistent improvement over 2B (AUROC 0.954–0.988), confirming scalability.
  • LR baseline (fine-tuned variant) remains stronger in far-OOD (AUROC ~0.998), but requires model adaptation.

Near-OOD:

  • AGNews (4 one-vs-all splits): Markov method outperforms LR baseline on 3/4 splits in AUROC (0.88–0.98 vs. 0.73–0.82) and all splits in AUPR and FPR@95.
  • ROSTD: Competitive with LR baseline (AUROC 0.953 vs. 0.973).
  • SNIPS: Competitive (AUROC 0.970 vs. 0.975).
  • CLINC150: Fails on 2B (AUROC 0.739, AUPR 0.013) — attributed to representation resolution mismatch (150 fine-grained intents vs. 16k SAE features). 9B partially recovers (AUROC 0.836).

Ablations:

  • First-order Markov slightly outperforms HTM and RNN backends on average (AUROC 0.919 vs. 0.912/0.910).
  • Top-k=10 is optimal; performance degrades with larger k. HTM collapses at k=128 (AUROC 0.50).
  • Raw activations (without SAE) perform poorly (AUROC 0.753), confirming SAE decomposition is essential.
  • Transition-based scoring (hop=1,2) substantially outperforms static overlap (hop=0).

Critical Assessment

  • Novelty: High. Applying SAE-derived layer transitions to OOD detection is novel. The insight that first-order Markov suffices on SAE-SDR sequences, and the interpretability analysis linking detection to decoded feature transitions, are original contributions.
  • Rigor: Good. Comprehensive evaluation across far-OOD and near-OOD, multiple datasets, two model sizes, three sequential backends, and ablations. Honest reporting of CLINC150 failure with principled explanation. Limited to 3 seeds where applicable.
  • Reproducibility: Moderate. Uses open-weight Gemma-2 models and pre-trained SAEs from SAELens/Neuronpedia. Code "will be released in the future" but is not yet available. SAE checkpoints and training details may limit exact reproduction.
  • Significance: Practically relevant. Domain restriction is important for deployed LLMs (medical, legal, enterprise). The ID-only setting is realistic (no OOD labels needed). The interpretability angle adds value for trust and debugging.
  • Red flags: (1) The method underperforms the LR baseline on far-OOD by a significant margin in FPR@95 (e.g., 0.11 vs. 0.003 for 2B on SST-2), limiting practical deployment where low false-positive rates matter. (2) Complete failure on fine-grained intent detection (CLINC150 2B) is a fundamental limitation tied to SAE feature resolution. (3) No inference-time cost analysis — extracting activations from all layers and running SAE encoders may be expensive for production use. (4) Limited to 16k SAEs; larger SAEs (65k+) might change the picture.

Strengths

  • No fine-tuning of the base LLM required; works with frozen pre-trained SAEs.
  • No labeled OOD data needed — fully ID-only training.
  • Provides human-interpretable explanations via decoded SAE feature transitions.
  • Simple first-order Markov backend is efficient and surprisingly effective.
  • Strong near-OOD performance where LR baseline struggles (AGNews splits).
  • Honest analysis of failure modes with principled explanations.

Limitations

  • Far-OOD FPR@95 is substantially worse than LR baseline (0.06–0.62 vs. 0.003–0.02).
  • Fails on fine-grained task taxonomies where SAE feature resolution is insufficient (CLINC150 with 150 intents).
  • Limited to pre-trained SAE availability (currently 16k dimensions; larger SAEs untested).
  • No analysis of computational overhead for real-time inference.
  • Code not yet released, limiting immediate reproducibility.
  • Only evaluated on Gemma-2; generalization to other architectures (Llama, Mistral) unknown.

Questions for Further Exploration

  • Would larger SAEs (e.g., 65k or 131k features) resolve the representation resolution bottleneck on CLINC150?
  • Can the method be applied to decoder-only generation (token-by-token) rather than just the prompt, enabling real-time OOD monitoring during response generation?
  • How robust is the approach to adversarial inputs designed to mimic ID feature transitions?
  • Does the Markov transition structure transfer across model architectures, or is it specific to Gemma-2's layer dynamics?
  • Could the density threshold θ and Top-k be adapted per-layer based on the Jaccard similarity profile rather than using fixed values?
