Sparse Memory Finetuning as a Low-Forgetting Alternative to LoRA and Full Finetuning

arXiv: 2605.03229v1

Date: 2026-05-04

https://arxiv.org/abs/2605.03229

一句话总结： 本文在 Qwen-2.5-0.5B 上复现了稀疏记忆微调（SMF），通过在每个训练步仅更新被当前 batch 高频读取的少量记忆行，在 MedMCQA 上取得 +2.5 pp 的提升，同时将 WikiText 困惑度和 TriviaQA 准确率的退化控制在 1 pp 以内，形成了一种以少量任务增益换取极低遗忘的微调范式。
TL;DR: Re-implements Sparse Memory Finetuning (SMF) on Qwen-2.5-0.5B-Instruct, inserting Product Key Memory layers and updating only the top-512 value rows per batch during task training. Additive SMF with KL slot selection gains +2.5 pp on MedMCQA while keeping WikiText perplexity and TriviaQA accuracy within ~1 pp of the base model — a clear low-forgetting point on the plasticity–stability frontier that LoRA (+4.6 pp, 17% PPL increase, −6 pp TriviaQA) and full finetuning (+5.4 pp, 44% PPL increase, −9.3 pp TriviaQA) cannot match. The tradeoff: smaller task gain in exchange for substantially less catastrophic forgetting.

核心贡献

  • 在 Qwen-2.5-0.5B-Instruct 上完成了 SMF、LoRA 和全参数微调的受控对比实验，使用 MedMCQA 作为目标任务，WikiText 困惑度和 TriviaQA 准确率作为遗忘探测指标，并报告多种子均值与标准差。
  • 提出了一种新的 KL 散度行选择规则，与原始 TF-IDF 规则进行系统对比：KL 规则在语言建模遗忘上更优（WikiText 困惑度更低），TF-IDF 在事实知识保留上更好（TriviaQA 退化更小）。
  • 验证了加性插入（Additive）模式优于替换模式（Replacement）：替换 MLP 的变体因过于破坏预训练表示而偏离 Pareto 前沿；加性模式保留原始 MLP 并叠加可缩放的记忆分支，在遗忘与可塑性之间取得更好平衡。
  • 定量展示了更新局部性（每步仅 ~1.38M 参数接收梯度 vs LoRA ~9M vs 全量 ~494M）是低遗忘的关键机制。

Key Contributions

  • Controlled comparison of SMF vs. LoRA vs. full finetuning on Qwen-2.5-0.5B-Instruct with multi-seed evaluation on MedMCQA (target task), WikiText perplexity (language modeling forgetting), and TriviaQA accuracy (factual knowledge forgetting).
  • Novel KL-divergence slot selection rule that complements the existing TF-IDF rule, offering a different tradeoff between the two forgetting metrics — KL keeps WikiText perplexity tighter while TF-IDF preserves more TriviaQA accuracy.
  • Empirical demonstration that additive memory insertion dominates replacement at the 0.5B scale: discarding pretrained MLPs is too destructive, whereas keeping them and adding a scaled memory branch enables meaningful task learning with minimal forgetting.
  • Update locality analysis showing that sparse updates (~1.38M parameters per step, only 3.125% of value rows) are the central mechanism behind SMF's low-forgetting behavior.

Methodology

The method follows a two-stage training pipeline on Qwen-2.5-0.5B-Instruct:

Stage 1 — Dense Retrofit: Product Key Memory (PKM) layers (Lample et al., 2019; Berges et al., 2024) are inserted at transformer layers {6, 12, 18}, each with 16,384 value slots (nk=128), 4 query heads, top-k=16 retrieval, and key dimension 256. Two insertion modes are compared: Replacement (discards original MLP) and Additive (keeps MLP and adds a scaled memory branch with learnable α). All non-memory parameters are frozen; memory layers are trained for 2 epochs on 50K OpenAssistant examples to obtain a sane initialization.

Stage 2 — Sparse Task Training: Only the top-T=512 value rows per layer (selected by per-batch slot scoring) receive gradient updates during MedMCQA training (3 epochs, 60K examples). Two scoring rules are compared:

  • TF-IDF: rewards slots read often in the current batch but rarely in background batches.
  • KL-divergence: scores slots by their per-slot contribution to D_KL(p_batch ∥ p_background), where p_background uses cumulative token-level read counts.

Baselines: LoRA (rank 16, α=32, applied to all attention and MLP projections, ~9M trainable params) and full finetuning (~494M params), both trained on the same MedMCQA data for 3 epochs. All methods use AdamW with cosine LR scheduling.

Results & Highlights

  • Additive sparse (KL) achieves MedMCQA 0.369 (+2.5 pp over base 0.344) while WikiText PPL improves slightly (12.72 vs 13.15) and TriviaQA drops only 0.4 pp (0.252 vs 0.256). This is the clearest low-forgetting point on the Pareto frontier.
  • Additive sparse +S (TF-IDF) is comparable: +2.5 pp MedMCQA, −1.1 pp TriviaQA, +0.31 WikiText PPL drift.
  • Replacement variants are dominated — only +1.0–1.1 pp MedMCQA gain but 26% PPL increase and 5–8 pp TriviaQA drop, confirming that overwriting pretrained MLPs is too destructive.
  • LoRA: +4.6 pp MedMCQA, 17% PPL increase, −6 pp TriviaQA. Sits on the Pareto frontier but at the high-forgetting end.
  • Full finetuning: +5.4 pp MedMCQA, 44% PPL increase, −9.3 pp TriviaQA. Largest task gain but also largest forgetting on both axes.
  • KL vs TF-IDF tradeoff: KL keeps WikiText PPL tighter; TF-IDF retains more TriviaQA accuracy. The preferred rule depends on which forgetting metric matters more for deployment.

Authors & Credibility

All four authors (Prakhar Gupta, Garv Shah, Satyam Goyal, Anirudh Kanchi) are from the University of Michigan and contribute equally. The paper is a preprint (May 2026) labeled as a work in progress. Code is publicly available at github.com/prakharg55/SMF-ICML. The work directly builds on Lin et al. (2025) SMF and Lample et al. (2019) Product Key Memory, providing an independent re-implementation and extension.

Critical Assessment

Novelty (Moderate): The core SMF method is not new (Lin et al., 2025). The main novelties are the KL-divergence slot selection rule and the controlled SMF–LoRA–full-FT comparison on a modern model. The paper is candid about being a re-implementation study.

Rigor (Moderate-High): Multi-seed evaluation with standard deviations is a strength. The two forgetting probes (language modeling + factual recall) cover different aspects of forgetting. However, the evaluation slice is only 1,000 examples, and the study is limited to one model at one scale on one domain.

Reproducibility (High): Code is public. All hyperparameters (LR, epochs, batch size, memory dimensions, top-T, top-k) are fully specified. The use of standard datasets (MedMCQA, WikiText-103, TriviaQA) aids replication.

Significance (Moderate): The plasticity–stability tradeoff analysis is valuable for practitioners deciding between adaptation methods. The finding that update locality drives low forgetting is insightful but not entirely surprising. The practical impact is limited by the 0.5B model scale — it remains unclear whether these conclusions transfer to 7B+ models.

Red flags: (1) Only 0.5B model tested; the authors acknowledge Pareto orderings may shift at larger scales. (2) The dense retrofit stage confounds the analysis — it is not isolated from the sparse update stage. (3) The paper does not test whether SMF's low forgetting comes at the cost of weaker task performance on harder or more diverse domains. (4) Only two forgetting probes are used; instruction following, reasoning, and code capabilities are untested.

Strengths

  • Clean, controlled experimental design with multiple seeds and transparent reporting of all results including dominated methods.
  • Honest positioning as a re-implementation and comparison study rather than claiming breakthrough novelty.
  • The Pareto frontier visualization (Figure 2) clearly communicates the plasticity–stability tradeoff.
  • The KL vs TF-IDF analysis provides actionable guidance for practitioners choosing a slot-selection rule.
  • Public code release ensures reproducibility.

Limitations

  • Scale: Only tested on Qwen-2.5-0.5B (494M params). Behavior at 7B–70B is unknown and may differ significantly.
  • Single domain: Only MedMCQA (medical QA) as the target task. Generalization to other domains (code, math, reasoning) is untested.
  • Small evaluation set: 1,000-example held-out slice for all metrics may not capture tail behavior.
  • Confounded stages: The dense retrofit stage (2 epochs on OpenAssistant) is always applied before sparse training; its contribution to the final result is not ablated.
  • Limited forgetting probes: WikiText perplexity and TriviaQA do not cover instruction following, multi-step reasoning, or code generation capabilities.
  • Overhead: Additive SMF increases inference-time model size by ~52M parameters (the memory modules), which is a ~10% overhead on a 0.5B model — non-trivial for deployment.

Relevance

这项工作直接关系到大模型高效微调与持续学习的核心问题：如何在适配新任务的同时最小化对已有能力的破坏。SMF 提供的"更新局部性"视角对理解 LoRA 等方法的遗忘机制有启发意义，其 Pareto 前沿分析对选择微调策略具有实际参考价值。

Questions for Further Exploration

  • How does SMF scale to 7B+ models? Does the additive vs. replacement gap close, widen, or invert at larger scales?
  • What happens when SMF is applied to multiple sequential tasks (true continual learning) rather than a single target task?
  • Can the KL and TF-IDF slot selection rules be combined or adaptively switched during training?
  • How does SMF interact with quantized models (e.g., QLoRA-style 4-bit baselines)?
  • Would domain-specific retrofit data (e.g., PubMed for medical QA) significantly improve SMF's task performance?
  • What is the latency overhead of the PKM retrieval at inference time compared to standard MLP layers?
