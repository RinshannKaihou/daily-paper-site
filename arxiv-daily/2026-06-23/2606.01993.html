MMG2Skill: Can Agents Distill In-the-Wild Guides into Self-Evolving Skills?

Authors Xinyu Che, Junqi Xiong, Yunfei Ge, Xinping Lei, Shihao Li, Hang Yan, Han Li, Yuanxing Zhang, Zhiqi Bai, Jinhua Hao, Ming Sun, Han Li, Jiaheng Liu Submitted 2026-06-01 Affiliations Nanjing University · Kuaishou Technology
Agent VLM Procedural Learning Benchmark Self-Improvement

Key Metric

+33.33pp

Largest single-cell jump — Gemini on Game, from 33.33 → 66.67 success rate on MMG2Skill-Bench.

Why This Matters

It reframes self-improving agents as skill editors, not weight trainers — using only in-the-wild guides and trajectory feedback, no benchmark scores.

Critical Edge

The analyzer can be fooled: surface-level completion cues trigger likely_success even when the oracle judge assigns zero.

Methodology

Stage 1 · Construct

Guide → SKILL.md tuples

A VLM extractor normalizes a multimodal guide G into tuples zᵢ = (uᵢ, cᵢ, vᵢ, qᵢ) — reusable procedure, applicability, expected-state cues, recovery knowledge.

Stage 2 · Execute

Skill-conditioned rollout

Policy πθ is frozen. The current skill set Sₖ rides alongside history hₜ; actions are sampled as aₜ ∼ πθ(aₜ | hₜ, I, Sₖ). The interface is identical across GUI / Game / Strategy.

Stage 3 · Close the loop

Analyzer + Refiner (no benchmark scores)

Analyzer emits ρₖ = (eₖ, rₖ) from trajectory evidence + self-judged likely_success. Refiner updates Sₖ₊₁, retaining the original guide so omitted details can be re-imported. Loop terminates on success or budget N = 5.

Analyzer Recall (GUI)
  • Macro-average gain +12.8 → +25.3 pp across six VLM backbones (Claude-Opus-4.6, GPT-5.5, Claude-Sonnet-4.6, Kimi-K2.6, Gemini-3.1-Pro-Preview, Qwen3.6-Plus).
  • Claude-Opus-4.6: 56.85 → 69.64; Qwen3.6-Plus: 26.67 → 51.94; largest single-cell jump +33.33 pp on Game (Gemini).
  • Early-stop saves 25–53% of attempts and 23–41% of VLM API calls while matching or exceeding full-budget runs (Game: 66.1 vs 47.8).
  • Raw-guide prompting can hurt: −1.67 pp on Game. Structure-before-revise is necessary, not optional.
  • Reflexion-style in-context accumulation trails persistent SKILL.md edits on Kimi-K2.6 and Qwen3.6-Plus for GUI / Game.

Strengths

  • Clean problem formulation — the loop's update variable is the skill set, not the policy.
  • Algorithm 1 is small and legible: one can read off the loop structure in a minute.
  • The "raw guide vs. constructed skill" ablation directly challenges "more context = better".
  • Cross-domain generality — a single skill-conditioning interface across GUI / Game / Strategy.
  • Analyzer calibration reported numerically (precision, recall, attempts and calls saved).
  • Honest failure attribution by pipeline component (agent / analyzer / refiner), not stage ranking.

Limitations

  • Guide discovery is external — no retrieval, filtering, or staleness handling studied.
  • Strategy analyzer recall is only 70.79%; GUI essay-submission shows false-positive likely_success.
  • Evaluation is costly — multimodal API calls × 5 attempts × revision × analyzer × refiner.
  • Interactive rollout throughput is rate-limit bound, especially under multi-attempt revision.
  • SKILL.md schema is conceptual, not typed — quality depends on the VLM's ability to populate fields coherently.
  • No comparison against learning-from-rollouts baselines (ExpeL, Voyager) on the same benchmark.

Questions for Further Exploration

  • Can the analyzer–refiner loop be bootstrapped by a learned verifier so deployment no longer depends on a single VLM's self-judgment?
  • How does MMG2Skill degrade when guides are stale, contradictory, or only partially relevant?
  • Could SKILL.md be replaced by a typed DSL with preconditions, effects, and recovery operators?
  • Does the framework extend to skills that are not fixed recipes but learned policy priors (negotiation, mixed-strategy games)?
  • Would an upstream retrieval / source-filtering module close the Strategy recall gap?
  • Is there a compute-efficient regime where revision runs on a smaller model than the executor?
  • Does the persistent-skill-edit advantage generalize to domains with very long context windows?
