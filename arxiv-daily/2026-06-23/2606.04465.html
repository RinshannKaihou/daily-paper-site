SePO closes the self-reference loop: the prompt optimizer finally evolves itself.

Prior prompt-optimization methods hand-write and freeze the optimizer's own system prompt. SePO treats the prompt agent as a special task agent — so the same open-ended evolutionary search refines both the task agent's prompt p and the prompt agent's prompt p̃. One pre-training run amortizes across every downstream task.

Deep Dive

Methodology

Self-referential closure · open-ended evolution · two-stage pre-train/fine-tune

  • Self-referential closure (Eq. 2): define a prompt task T̃ whose input is (T, A, feedback) and whose score is whether the proposed p′ improves task accuracy. Same form as Eq. 1 — same procedure refines both layers.
  • Open-ended evolutionary search (Algorithm 1): archive of candidate prompts, tempered child-count-penalized softmax parent sampling, keep-better admission with ε-noise leeway. Failed children stay as stepping stones.
  • Two-stage pipeline: Stage 1 evolves the prompt agent's own p̃ on a multi-task pool (SePO-Generalist, default) or single task (SePO-Specialist); Stage 2 fixes p̃* and evolves the task agent's p per target task.
  • Greedy task selection (Algorithm 2): LLM-judge scores TaskSim + FormatSim + diversity; pick size-4 mixture maximizing proxy accuracy on AIME'25 + ARC-AGI-1. Best: STEM + ARC-AGI-1 + LIMO + MBPP.
  • Models: Task agent = DeepSeek-V3.2 (T=0, deterministic); Prompt agent = Gemini 3.1 Pro Preview (T=1, diverse). 10 iters × 16 examples, 5 seeds averaged.
  • Cost: $7.43/task amortized pre-training + $2.41–$15.51 fine-tuning — cheaper than TextGrad while winning.

Results & Highlights

Task agent = DeepSeek-V3.2 · prompt agent = Gemini 3.1 Pro · 5 seeds

  • SePO-Generalist wins every task — +4.49 over Manual-CoT, +5.99 over TextGrad, +5.06 over MetaSPO.
  • ARC-AGI-1: 37.30 → 43.39 (+6.09) — biggest single gain; the most specialized task benefits most from pre-training.
  • AIME'25: 57.55 → 64.22 (+6.67) — math olympiad improves despite being outside the pre-training mixture.
  • Sudoku (held out entirely): 96.95 → 99.90 — general prompt-optimization skill alone is sufficient for true transfer.
  • Ablations confirm both ingredients matter: w/o self-improvement −1.44 avg (ARC-AGI-1 −3.63); w/o open-ended evolution −3.74 avg (AIME'25 −6.98).
  • Model-swap: Gemini 3.1 Flash-Lite + Claude Opus 4.6 still gains +2.13 avg over Manual-CoT — not a model-pair quirk.

Balance Sheet

Strengths

  • Clean conceptual contribution — self-referential closure is the right framing; Figure 1 contrasts SePO with PromptBreeder's bounded self-reference vividly.
  • Unified algorithm — one search procedure (open-ended evolution with archive) covers both layers, no special-casing.
  • Thorough empirical sweep: 5 tasks × 3 baselines × 2 model pairs × 5 seeds × 2 SePO configs × 2 ablations.
  • Honest cost analysis — $7.43/task cheaper than SePO-Specialist and comparable to TextGrad while winning on accuracy.
  • Qualitative inspection of evolved prompts (Appendices E, F) builds trust — preserve-what-works, add task-specific scaffolding.
  • True held-out generalization test (Sudoku, never in pre-training mixture).
  • Model-swap robustness — gains transfer across Gemini 3.1 Flash-Lite + Claude Opus 4.6, not a quirk of one model pair.
  • Code released; deterministic task agent (T=0), diverse prompt agent (T=1); 1:1 failed/successful example ratio; keep-better with ε leeway.

Limitations

  • Self-improvement depth: only 10 candidates per evolutionary run (G=5, K=2). The "evolved" prompt agent may not be that much better than its seed.
  • Pre-training mixture leak: ARC-AGI-1 and MBPP are in the SePO-Generalist mixture; STEM ≈ AIME'25. Sudoku is the only truly clean held-out case.
  • Closed-source model dependence: relies on DeepSeek-V3.2 + Gemini 3.1 Pro; may not transfer to open-source models.
  • Greedy task selection is a placeholder: acknowledged heuristic; headline numbers depend on a non-trivial hyperparameter (mixture composition).
  • Static offline optimization: training-time procedure only; the "self-evolving agent" framing is generous — no continuous online evolution.
  • No seed-sensitivity study: robustness to different seed prompts p̃(0) untested.
  • Wall-clock cost not reported — dollar cost alone is incomplete for practitioners.
  • Limited to natural-language system prompts — tools, code, and workflows explicitly out of scope (future work).

Relevance to This Reader

与研究兴趣 system prompt optimization / agent optimization / self-improving agents 直接相关 — SePO is the first 2026 paper to explicitly close the self-reference loop in system-prompt optimization, and the cleanest methodological advance on this axis since PromptBreeder.

它给"如何让 agent 自身变得可学习"这一通用问题 — how to make an agent's optimizer learnable in the same way as the agent itself — 给出了一个干净的范式（把自身当作一个特殊 task agent），可能推广到 workflow、tool 选择等其他 agent 组件的自我优化。

Questions for Further Exploration

  1. Does SePO's gain hold when the prompt agent is an open-source model (Qwen, Llama, DeepSeek) rather than Gemini 3.1 Pro? Self-evolution may be much weaker on smaller models.
  2. What happens with G=20, K=10 for the prompt agent's pre-training — does the gap to SePO w/o self-improvement grow or saturate?
  3. A stronger cross-task generalization test with a SePO-Generalist mixture that has zero overlap to the 5 evaluation tasks — how much of the gain is from the mixture containing the targets?
  4. How robust is the evolved p̃* to different seed prompts p̃(0)? Single attractor or multiple?
  5. Can SePO run continuously at deployment time, evolving the prompt agent on incoming queries — from offline training to online evolution?
  6. Can the same self-referential + two-stage pattern optimize an agent's tools (function set) or workflow (DSPy-style module composition)?
  7. Does the closed loop converge? Under what conditions does the prompt agent's evolution stabilize? The paper gives no convergence analysis.
  8. Is the +4.49 accuracy gain worth the $7.43/task pre-training cost in production? A cost-vs-accuracy Pareto analysis is missing.
