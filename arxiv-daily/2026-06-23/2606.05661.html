2606.05661 · Continual Learning

Naive in-context learning beats dedicated memory — even the best frontier LLM agent only gains 25.4% on real-world continual learning.

Continual Learning Bench: Evaluating Frontier AI Systems in Real-World Stateful Environments ↗
Parth Asawa, Christopher M. Glaze, Gabriel Orlanski, Ramya Ramakrishnan, Benji Xu, Asim Biswal, Vincent Sunn Chen, Frederic Sala, Matei Zaharia, Joseph E. Gonzalez  ·  UC Berkeley · Snorkel AI · UW-Madison  ·  2026-06-23
continual-learning llm-agents benchmark memory-systems in-context-learning
1 Key Metric
25.4%
Normalized gain by the best system (Sonnet 4.6 ICL) — leaving ~75% of learning headroom unrealized on CL-Bench.
2 Why This Matters

It isolates online learning from base capability with a reusable "gain" metric — and warns against stacking memory layers before simple baselines are validated.

3 Critical Edge

Only context-based memory is tested — test-time training, LoRA adaptation, and episodic replay buffers are left out, so the most promising recent paradigms are unproven.

Methodology · Six domains, one metric

02 · 03

Benchmark Design

Six real-world domains, each with latent structure that is realistic but not recoverable from pretraining alone, shared across instances so prior experience is informative, and accompanied by an environment feedback loop (test failures, query errors, prediction residuals) that allows online learning.

Domain Roster

Blind Spectrum MonitoringN = 90 · IoU on RF maps
Codebase AdaptationN = 19 · Resolve GitHub issues
Cohort StudiesN = 20 · KL vs naive baseline
Database ExplorationN = 40 · Query budget saved
Exploitable PokerN = 120 · Heads-up profit (bb)
Sales PredictionN = 12 · WAPE skill, 75 preds

Validation Pipeline

≥2 authors per task, then 2–3 independent domain experts score on realism · reusable knowledge · learning improvement. Average expert time: 2.14 hours per validation cycle.

Headline Findings

  • Naive ICL wins. Three of the top-five gain systems are vanilla ICL.
  • Memory can hurt. ACE costs 2× more than the leader yet ranks 10th in gain.
  • Per-task heterogeneity is large — Sales Prediction and BSM learn; Cohort Studies learns almost nothing.
  • Pareto-optimal low cost: Gemini Flash ICL → 16.4% gain at $7.6.

Strengths · Limitations

04

Strengths

  • Gain metric elegantly solves the confound between learning and base capability.
  • Six diverse, expert-validated domains with non-trivial latent structure (obfuscated schemas, RF maps, opponent policies).
  • Built-in concept drift (DB migration, Cohort variants) tests adaptation, not memorization.
  • Transparent cost analysis: Pareto frontier on reward-vs-cost and gain-vs-cost is dominated by ICL variants.
  • Plasticity / stability decomposition offers actionable diagnostic for system designers.
  • Open release + leaderboard + website (continual-learning-bench.com).

Limitations

  • 6 domains only — misses multi-turn dialogue, scientific review, robotics, embodied agents.
  • Short sequences (12–120 instances), far below real deployment horizons.
  • Context-only memory — no test-time training, LoRA, fine-tuning, or replay buffers tested.
  • Small-model failure modes invisible because tasks require frontier capability to be non-trivial.
  • Harness confound for Claude Code / Codex: opaque context compaction makes attribution partial.
  • Cohort Studies shows near-zero learning across all systems — task may be under-specified.
  • Cost figures drift with prompt caching and provider pricing.

Questions for Further Exploration

05
  1. The "naive ICL beats memory" finding is striking — is it because dedicated memory introduces stale beliefs faster than they can be corrected? Could a learned meta-controller close the gap?
  2. How would test-time training / parametric adaptation (LoRA, in-context fine-tuning) perform on CL-Bench? Would the gain ranking flip?
  3. Cohort Studies shows essentially no learning — is the latent structure (patient mixture + Weibull survival) actually recoverable, or is the task under-specified despite expert validation?
  4. Can the plasticity-stability decomposition become a design principle: route stability-sensitive steps to Notepad, plasticity-sensitive steps to full-context ICL?
  5. Database Exploration's mid-run migration reveals "rigidity in early incorrect beliefs" — could explicit uncertainty tracking (Bayesian belief state) improve drift adaptation?
  6. How does CL-Bench gain correlate with long-horizon deployment success in production agent systems (SWE, data science, decision support)?
  7. Would noise-robust reward signals (multi-rollout averaging) shrink the wide CIs (e.g. Notepad/Sonnet ±5.7%)?
  8. Can community-contributed tasks reach thousands of instances while preserving headroom and tractability?
