> **一句话总结：** SePO 把"提示词优化器自身的系统提示"也当作可优化的目标，用同一个开放域进化搜索 + 两阶段（预训练-微调）流水线同时优化任务 agent 和提示词 agent 的提示，从而在 AIME'25、ARC-AGI-1、GPQA、MBPP、Sudoku 五个任务上以平均 4.49 分的优势击败 Manual-CoT、TextGrad、MetaSPO。

> **TL;DR:** SePO (Self-Evolving Prompt Optimization) closes a long-standing gap in system prompt optimization: prior methods keep the *prompt agent's own* system prompt hand-engineered and fixed, so the "optimizer" never improves as more tasks are seen. SePO adopts a **self-referential design** — the prompt agent treats itself as a special task agent, so the same open-ended evolutionary search (archive of candidate prompts, child sampled from parent, keep-if-better admission) refines both the task agent's system prompt $p$ and the prompt agent's own system prompt $\tilde p$. Training is split into a **pre-training stage** that evolves the prompt agent on a multi-task pool (SePO-Generalist, default) or a single related task (SePO-Specialist), and a **fine-tuning stage** that reuses the evolved prompt agent to optimize a target task's prompt. Across AIME'25 (math), ARC-AGI-1 (abstract reasoning), GPQA (graduate science), MBPP (code), and Sudoku (logic), SePO-Generalist beats Manual-CoT by **+4.49 avg**, TextGrad by **+5.99 avg**, and MetaSPO by **+5.06 avg**, with a single pre-training run amortized across all five tasks (\$7.43/task). Ablations confirm both **self-improvement of the prompt agent** and **archive-based open-ended evolution** are necessary; model-swap (Gemini 3.1 Flash-Lite + Claude Opus 4.6) preserves the gain.

## 核心贡献

- **自指式（self-referential）提示词优化**：把"提示词 agent 自己的 system prompt"也作为优化变量，关闭了"优化器本身永远是人工写死的"这一开放循环。
- **同一种搜索覆盖两层**：将 Zhang et al. (2026) 的开放域进化搜索（archive + keep-better admission）同时用于优化任务 agent 的 prompt $p$ 和提示词 agent 的 prompt $\tilde p$，不再需要为两层分别设计优化器。
- **两阶段训练流水线**：
  - **预训练阶段**：在多任务池（SePO-Generalist，默认）或单个相关任务（SePO-Specialist）上自演化提示词 agent，习得"广泛的提示词优化能力"。
  - **微调阶段**：用预训练得到的 $\tilde p^\star$ 在目标任务上优化任务 agent 的 prompt $p$。
- **跨任务泛化**：预训练得到的提示词优化能力能迁移到预训练混合之外的任务（如 Sudoku 从未被包含在任何预训练 mixture 中，仍从 96.95 → 99.90）。
- **与 PromptBreeder 的关键区别**：PromptBreeder 的元栈最顶层（hyper-mutation prompt）永远是手工写死的，"自指回路"是有界但不闭合的；SePO 的最顶层就是 $\tilde p$ 自身，回路真正闭合。
- **工程贡献**：贪心任务选择启发式 + size-4 mixture（STEM+ARC-AGI-1+LIMO+MBPP）在所有 mixture size 中获得最佳微调精度。

## Key Contributions

- **Self-Referential Closure.** Formalization (Eq. 1 + Eq. 2) showing the prompt agent is itself a task agent whose task is to improve task agent prompts; the same objective and same search algorithm apply to both layers, with the prompt agent's own system prompt $\tilde p$ replacing the human-written "optimizer prompt."
- **Open-ended evolutionary search as the unifying algorithm.** Archive of candidate prompts where earlier prompts act as **stepping stones**; parents sampled via tempered child-count-penalized softmax; children admitted under a *keep-better* policy with $\epsilon$-noise leeway (Algorithm 1, Eq. 3-4). Reuses the Darwin-Gödel-Machine procedure of Zhang et al. (2026).
- **Two-Stage Pre-training / Fine-tuning Pipeline** that mirrors the standard LM paradigm:
  - *Stage 1 (pre-training):* evolves the prompt agent on a multi-task pool $\mathcal{P}$ (or a single task, SePO-Specialist) into $\tilde p^\star$.
  - *Stage 2 (fine-tuning):* fixes $\tilde p^\star$ and uses it to evolve the task agent's prompt $p \to p^\star$ for each target task.
  - Amortizes the cost of self-evolution: one pre-training run is reused across all 5 fine-tuning tasks.
- **Two Configurations**:
  - *SePO-Generalist* (default): pre-training on a mixture of 4 tasks chosen by a greedy selector that balances task-skill relevance $U(d;\mathcal{T})$ and diversity $Div(d, M)$ (Algorithm 2, Eq. 5-8).
  - *SePO-Specialist*: pre-training on the same single task as fine-tuning; isolates the contribution of multi-task pre-training.
- **Empirical contribution on five diverse benchmarks** (math, abstract reasoning, graduate science, code, logic) against three prompt optimization baselines spanning "no optimization" (Manual-CoT), text-gradient backprop (TextGrad), and cross-task meta-learning (MetaSPO).

## Methodology

### Problem Setup

- **Task**: $T = (D, S)$ with input–target pairs $(x, y)$ and a deterministic scoring function $S(x, y, \hat y)$.
- **Task agent**: $A = A(p, M, W)$ with system prompt $p$, LM $M$, and workflow $W$. Accuracy $\mathrm{acc}(A; T) = \mathbb{E}_{(x,y) \sim D}[S(x, y, A(x))]$.
- **Standard system prompt optimization** (Eq. 1): $p^\star = \arg\max_p \mathbb{E}_{(x,y) \sim D} S(x, y, A(p, M, W)(x))$, solved by introducing a second "prompt agent" $\tilde A$ that reads feedback and proposes refined $p'$.

### Asymmetry Identified

In all prior work (OPRO, PromptAgent, TextGrad, MetaSPO, PromptBreeder), the prompt agent's *own* system prompt $\tilde p$ is **hand-engineered and frozen** — only $p$ is learned. This bounds optimization to whatever a human can hand-write.

### Self-Referential Closure (Eq. 2)

SePO defines a *prompt task* $\tilde T = (\tilde D, \tilde S)$ whose input is $\tilde x = (T, A, E)$ (a task, a task agent, evaluation feedback) and whose score is whether the proposed $p'$ improves task agent accuracy. Then:
$$\tilde p^\star = \arg\max_{\tilde p} \mathbb{E} \, \tilde S(T, A, E, \tilde A(\tilde p, \tilde M, \tilde W)(T, A, E))$$
Eq. 2 has the same form as Eq. 1 — so **the same procedure refines both layers**, closing the loop.

### Open-Ended Evolutionary Search (Algorithm 1)

- Initialize archive $\mathcal{A} = \{p^{(0)}\}$.
- For $t = 1, \dots, G$:
  - For $k = 1, \dots, K$ in parallel:
    - Sample parent $p \sim \mathrm{softmax}$ over archive (Eq. 3 — tempered, child-count-penalized).
    - Generate child $p' = \tilde A(p, E)$ using the prompt agent with system prompt **$\tilde p$ during pre-training**, **$\tilde p^\star$ during fine-tuning**.
    - Evaluate $p'$ on a batch of training examples ($\sim$1:1 failed/successful), compute $s_{p'}$, admit if it keeps-or-improves over parent (keep-better with $\epsilon$ leeway).
- Return $p^\star = \arg\max_{p \in \mathcal{A}} s_p$.

Key design choices:
- **Archive as stepping stones**: children that fail are kept in the archive as future parents.
- **Differential scoring** (Eq. 4): $s_a = \mathrm{acc}_{\text{new}} - \mathrm{acc}_{\text{old}}$ with a robust RER denominator.
- **Adaptive temperature** $\tau = \max\{\tfrac{1}{3}(\max_b s_b - \min_b s_b), 0.05\}$ concentrates exploitation when scores spread; preserves non-zero selection probability for all candidates.

### Two-Stage Pipeline (Figure 2)

| Stage | What evolves | System prompt used by prompt agent | Why |
|---|---|---|---|
| Pre-training (SePO-Generalist) | prompt agent's own $\tilde p$ on multi-task pool | $\tilde p^{(k)}$ (current) — **self-referential closure activates here** | accumulates broad prompt-optimization skill |
| Pre-training (SePO-Specialist) | prompt agent's own $\tilde p$ on a single task | $\tilde p^{(k)}$ | isolates multi-task contribution |
| Fine-tuning | task agent's $p$ on target task $T$ | $\tilde p^\star$ (fixed from Stage 1) | amortizes pre-training cost across many tasks |

### Greedy Task Selection for SePO-Generalist (Algorithm 2)

- Candidate pool $\mathcal{P}$ = target tasks $T$ + auxiliary train-only tasks.
- For each candidate $d \in \mathcal{P}$, an LLM judge scores TaskSim, FormatSim, and Sim redundancy against all $d' \in T$.
- $U(d; T) = \sum_{t \in T} \mathrm{Rel}(d, t) = \alpha \mathrm{TaskSim} + (1 - \alpha) \mathrm{FormatSim}$.
- $\mathrm{Div}(d, M) = 1 - \max_{d' \in M} \mathrm{Sim}(d, d')$ (or $1$ when $M = \emptyset$).
- $\mathrm{Score}(d \mid M) = \lambda U(d; T) + (1 - \lambda) \mathrm{Div}(d, M)$.
- Greedily add the highest-scoring $d$; for each size $k \in \{1, 2, 4, 8\}$ the length-$k$ prefix defines $M_k$. Pick the $k$ maximizing average fine-tuning accuracy on proxy tasks $V = \{\text{AIME'25, ARC-AGI-1}\}$. **Best mixture: STEM + ARC-AGI-1 + LIMO + MBPP.**

### Implementation Details

- **Task agent**: DeepSeek-V3.2 (temperature 0 for determinism).
- **Prompt agent**: Gemini 3.1 Pro Preview (temperature 1 for diversity).
- **Optimization budget**: 10 iterations × 16 examples (shared with TextGrad).
- **SePO hyper-params**: $G = 5$ generations, $K = 2$ children per generation.
- **Evaluation**: 5 random seeds averaged; train/test split per task; pass@3 for ARC-AGI-1, pass@1 elsewhere.

## Results & Highlights

### Main Results (Table 1)

Task agent = DeepSeek-V3.2, prompt agent = Gemini 3.1 Pro Preview.

| Method | AIME'25 | ARC-AGI-1 | GPQA | MBPP | Sudoku | **Avg** |
|---|---|---|---|---|---|---|
| Manual-CoT | 57.55 | 37.30 | 76.46 | 91.20 | 96.95 | 71.89 |
| TextGrad | 55.99 | 34.75 | 74.44 | 90.15 | 96.60 | 70.39 |
| MetaSPO | 57.71 | 37.27 | 75.51 | 89.30 | 96.80 | 71.32 |
| SePO-Specialist | 60.94 | 37.46 | 76.72 | 95.55 | 99.80 | 74.09 |
| **SePO-Generalist** | **64.22** | **43.39** | **78.18** | **96.20** | **99.90** | **76.38** |

- SePO-Generalist is best on **every** task, +4.49 over Manual-CoT, +5.99 over TextGrad, +5.06 over MetaSPO.
- TextGrad and MetaSPO each fall below Manual-CoT on at least 3 tasks.
- The biggest single gain is ARC-AGI-1: 37.30 → 43.39 (+6.09) — the most specialized task benefits most from pre-training skill.
- MBPP: 91.20 → 96.20 (+5.00) — code generation gains substantially from evolved task-specific scaffolding.
- AIME'25: 57.55 → 64.22 (+6.67) — math olympiad improves despite the task being outside the pre-training mixture.

### Task Selection Validation (Figure 3)

Greedy selector vs. random selector at sizes {1, 2, 4, 8} (scored on AIME'25 + ARC-AGI-1 train):
- Size 4: greedy = 72.68 vs. random = 71.14 (largest gap, +1.54).
- Size 1, 2, 4 all favor greedy. Size 8 = full pool (no choice).
- Best mixture: **STEM + ARC-AGI-1 + LIMO + MBPP**.

### Cross-Task Generalization (Figure 4)

For each target task, run SePO-Generalist under (a) pre-training mixture *with* a related task, (b) pre-training mixture *without* a related task, (c) Sudoku held out entirely.
- Largest gap on ARC-AGI-1: +4.95 (related-task > unrelated).
- AIME'25: +1.30; GPQA: +1.66; MBPP: comparable.
- **Sudoku (never seen in pre-training)**: 96.95 → 99.90 — general prompt-optimization skill alone is sufficient.
- The learned skill is a *capability*, not memorized per-task prompts.

### Component Ablations (Table 2)

- SePO-Generalist: avg 76.38
- w/o Self-Improvement: avg 74.94 (−1.44) — pre-training matters; biggest hit is ARC-AGI-1 (−3.63).
- w/o Open-Ended Evolution (linear search instead of archive): avg 72.64 (−3.74) — biggest hit is AIME'25 (−6.98). Linear search cannot escape local optima.

### Model-Swap Robustness (Table 3)

Task agent = Gemini 3.1 Flash-Lite Preview, prompt agent = Claude Opus 4.6.
- Manual-CoT avg 67.95 → SePO-Generalist avg 70.08 (**+2.13**).
- SePO wins on every task; gains transfer across model pairs, not just the default.

### Cost Analysis

- TextGrad: \$14.75 – \$26.52 per task (single stage).
- SePO-Specialist: \$5.72 – \$37.63 per task (pre-training + fine-tuning), comparable to TextGrad in scale.
- SePO-Generalist: **\$7.43/task amortized pre-training + \$2.41–\$15.51 fine-tuning**. Cheaper than SePO-Specialist on every task.
- Inference cost is identical across all methods (same task agent at query time).

### Qualitative Evolution Patterns (Appendices E, F)

- The **evolved prompt agent** prompt expands the seed workflow with 5 defensive principles guarding against CoT truncation, sacrificed rigor, and overfitting to test cases.
- The **evolved task agent prompts** preserve role/format from the seed and add multi-step procedural scaffolding targeting each task's failure modes:
  - **MBPP**: blocks global-namespace collisions on `max`/`min`.
  - **Sudoku**: verifies against tokenizer-induced character drops.
  - **ARC-AGI-1**: enforces coordinate transcription rules.
- Pattern: *preserve-what-works* — task-specific scaffolding, not wholesale rewriting.

## Critical Assessment

- **Novelty: Moderate-to-High.** The *self-referential closure* observation is sharp and well-articulated. The "same search algorithm at two layers" insight is genuinely clean. The framework cleanly generalizes PromptBreeder (which was *bounded* self-reference, SePO is *closed* self-reference). The framing as a pre-training / fine-tuning pipeline is borrowed from LM literature and is a useful organizing metaphor. The task-selection greedy heuristic is a small but useful engineering contribution.
- **Rigor: High.** Five diverse benchmarks, three baselines spanning the optimization spectrum, ablations isolating the two key components, model-swap robustness, cost analysis, qualitative inspection of evolved prompts, and greedy-vs-random task selection. The 5-seed averaging is a real plus. Cross-task generalization is tested with a true held-out task (Sudoku). The weakest part of the rigor story is the relatively small scale (one prompt agent, one task agent per config) — but the breadth across tasks and models compensates.
- **Reproducibility: High.** Code is released, all hyperparameters are stated, models are public (via API). The greedy task-selection algorithm is fully specified. The one reproducibility risk is dependence on closed-source model behavior (Gemini, Claude) which can drift over time.
- **Significance: Real.** This is a concrete step toward the "self-improving agent" research direction (Voyager, ADAS, Darwin Gödel Machine). The contribution is best read as "we know how to do this for prompts now, and we showed the loop can be closed." The 4.49-point average gain over Manual-CoT is meaningful but not huge; the architectural contribution matters more than the numbers.
- **Red flags**:
  - The greedy task-selection heuristic is presented as a placeholder ("More advanced task selection algorithms could replace our heuristic without affecting the rest of SePO") — this is honest but means SePO-Generalist's headline numbers depend on a non-trivial hyperparameter choice.
  - The pre-training mixture (STEM + ARC-AGI-1 + LIMO + MBPP) includes 3 of the 5 evaluation tasks directly (ARC-AGI-1, MBPP) and one related (STEM ≈ AIME'25) — so the cross-task generalization result is somewhat confounded. Sudoku being fully held out is the cleanest evidence.
  - The "self-improvement" of the prompt agent is a single evolutionary run with $G=5, K=2$ — only 10 candidates total. The "evolved" prompt agent's improvements may not be that deep.
  - The relation to PromptBreeder could be discussed more explicitly: the paper claims SePO is the first to close the loop, which is true, but the conceptual distance from PromptBreeder is smaller than the framing suggests.
  - The "self-evolving" label is strong; the actual mechanism is a *single* offline evolutionary run on a fixed training set, not continuous online self-improvement. This is a semantic concern, not a methodological one.

## Strengths

- **Clean conceptual contribution**: the self-referential closure is the right way to frame what's been missing in prompt optimization, and Figure 1 makes the contrast with PromptBreeder vivid.
- **Unified algorithm**: one search procedure (open-ended evolution with archive) at two layers, no special-casing.
- **Strong empirical coverage**: 5 tasks × 3 baselines × 2 model pairs × 5 seeds × 2 SePO configs × 2 ablations = a thorough sweep.
- **Cost analysis is honest**: shows SePO-Generalist is cheaper than SePO-Specialist and comparable to TextGrad while winning on accuracy.
- **Qualitative inspection**: showing the evolved prompts preserves-role-and-format and adds task-specific scaffolding is the kind of analysis that builds trust.
- **Practical design choices**: deterministic task agent (T=0), diverse prompt agent (T=1); 1:1 failed/successful example ratio in evaluation batches; keep-better with $\epsilon$ leeway; tempered softmax with child-count penalty — all are battle-tested ingredients.
- **Code release + 5-seed averaging + train/test splits per task**: serious experimental hygiene.
- **Model-swap robustness**: doesn't rely on a particular model pair's quirks.

## Limitations

- **Self-improvement depth**: only 10 candidates per evolutionary run. The "evolved" prompt agent may not be that much better than its seed. A study with deeper self-evolution would be reassuring.
- **Pre-training mixture leak**: ARC-AGI-1, MBPP are in the SePO-Generalist mixture, and STEM is related to AIME'25. The cleanest held-out case is Sudoku alone; a future study with truly disjoint pre-training + evaluation suites would be stronger.
- **Closed-source model dependence**: relies on DeepSeek-V3.2 + Gemini 3.1 Pro Preview for headline results; results may not transfer to open-source models.
- **Greedy task selection is acknowledged as a heuristic**: the headline SePO-Generalist numbers depend on a non-trivial hyperparameter (the mixture composition).
- **Static offline optimization**: this is a *training-time* procedure, not online/continuous self-improvement. Inference is the same model with the same per-query budget. The framing as "self-evolving agent" is generous.
- **No exploration of seed sensitivity**: how robust are the results to different seed prompts $\tilde p^{(0)}$? The paper does not show this.
- **Token-level cost not the only cost**: the paper reports dollar cost but not wall-clock time, which matters for practitioners.
- **No failure-mode analysis**: when does SePO fail? What kinds of tasks does it not help on?
- **Limited to natural-language system prompts**: code, tools, and workflows are explicitly out of scope (mentioned only as future work).

## Relevance

- 与研究兴趣 **system prompt optimization / agent optimization / self-improving agents** 直接相关：SePO 是 2026 年第一个明确"闭合自指回路"的系统提示词优化工作，是 PromptBreeder 之后这一方向上最清晰的方法学推进。
- 对"如何让 agent 自身变得可学习"这一通用问题给出了一个干净的范式（把自身当作一个特殊 task agent），可能推广到 workflow、tool 选择等其他 agent 组件的自我优化。

## Questions for Further Exploration

- **Generalization to open-source models.** Does SePO's gain hold when the prompt agent is an open model (Qwen, Llama, DeepSeek) rather than Gemini 3.1 Pro? The prompt agent's self-evolution may be much weaker on smaller models.
- **Deeper self-evolution.** What happens with $G=20, K=10$ for the prompt agent's pre-training? Does the gap to SePO w/o self-improvement grow linearly, or saturate?
- **Disjoint pre-training / evaluation suites.** A stronger cross-task generalization test would use a SePO-Generalist mixture with zero overlap to the 5 evaluation tasks. How much of the gain is from the mixture containing the targets?
- **Seed sensitivity.** How robust is the evolved $\tilde p^\star$ to the choice of seed $\tilde p^{(0)}$? Is the optimum reached a single attractor or multiple?
- **Online self-improvement.** Can SePO be run continuously at deployment time, evolving the prompt agent on incoming queries? This is the natural next step from "offline training" to "online evolution."
- **Beyond system prompts.** Can the same self-referential + two-stage pattern optimize an agent's *tools* (function set) or *workflow* (DSPy-style module composition)? The conclusion hints at this.
- **Theory of self-referential search.** Does the closed loop converge? Under what conditions does the prompt agent's evolution stabilize? The paper gives no convergence analysis; this is an obvious theoretical gap.
- **Cost / latency trade-off.** Is the 4.49-point accuracy gain worth the \$7.43/task pre-training cost in a production setting? A cost-vs-accuracy Pareto analysis would be useful.
- **Failure modes.** On which task types does SePO *hurt* compared to Manual-CoT? The paper only reports averages; a per-task variance analysis would reveal when the method misfires.
- **Combination with RL post-training.** Could SePO's evolved prompt serve as a better initialization for GRPO/PPO post-training of the task agent?
