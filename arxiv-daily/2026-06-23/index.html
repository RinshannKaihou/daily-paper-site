AutoLab: Persistence Beats Talent in Long-Horizon Agents

By Zhangchen Xu, Junda Chen et al. · 2,544 wall-clock hours · 17 models · 36 tasks

A 36-task, four-domain ultra long-horizon benchmark reveals that Claude Opus 4.6 wins decisively on long-horizon work — but the dominant predictor of success is the willingness to keep benchmarking, keep editing, and keep incorporating feedback, not one-shot code quality. A separate 302-rollout manual failure analysis surfaces a cross-model lack of time awareness, and a harness ablation shows that agent infrastructure alone can shift a single model's score by as much as 0.43. The implication is blunt: long-horizon capability cannot be measured independently of the harness it runs in.

坚持比才华更重要:36 个跨系统优化 / 谜题 / 模型开发 / CUDA kernel 的真实任务,2,544 壁钟小时对 17 个前沿模型做评估。Claude Opus 4.6 以 Avg@3 0.68 显著领先,但决定胜负的不是一次性代码质量,而是持续迭代、反馈吸收与时间感知——同时 harness 消融实证证明,agent 基础设施单独可让同一模型的分数摆动高达 0.43,颠覆"长时程能力可与 harness 解耦评测"的常识。

OpenSkill: Skills Are First-Class, Portable Artifacts

By Zhiling Yan, Dingjie Song et al.

A new "open-world self-evolution" setting forces the agent to bootstrap its own skill library and its own verifier from public web knowledge — no target-task supervision. OpenSkill hits 43.6% / 42.1% on SkillsBench, beats the strongest closed-world baseline by +8.9 / +8.8 pp, and the same Opus-built skills transfer as-is to Haiku 4.5, Qwen 3 Coder, DeepSeek V3, and Mistral Large 3 with +5.5–14.8 pp gains.

开放世界自进化:智能体只拿到任务指令和公网资源,自行从文档/仓库/网页抓取知识合成可迁移技能,并用自建 pytest 虚拟任务做无监督打磨。SkillsBench 上以 43.6% / 42.1% 拿下最强自动方法,且同一组 Opus 4.6 生成的技能无需适配即可迁移到 4 个更弱模型。

INFUSER: An 8B Generator Beats a Frozen 32B

By Siyu Chen, Miao Lu et al.

Self-evolving reasoning recast as a two-role co-training problem. The Generator is rewarded by an optimizer-aware influence score measuring each candidate question's effect on the Solver's loss on the target distribution — replacing "difficulty" with "usefulness to the current solver." A custom DuGRPO handles the continuous, noisy reward. Qwen3-8B-Base gains >20% on Olympiad and SuperGPQA.

影响函数驱动的生成器–求解器共演化:把"对当前求解器有用"作为生成器奖励,DuGRPO 处理连续含噪信号。Qwen3-8B-Base 在 Olympiad / SuperGPQA 上相对提升 >20%,8B 共演化生成器在数学 / 代码上超过冻结 32B 思维生成器。
