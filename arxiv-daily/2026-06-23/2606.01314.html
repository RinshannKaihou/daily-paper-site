Methodology · 三件套协同进化

State is a triple Σₜ = (Sₜ, Tₜ, Fₜ): skills, tools, and an anti-pattern memory. Each iteration samples a Pareto-front state, reflects on failures, and emits an atomic bundle L — restricted to five typed tool lifecycle primitives (WRAP / EDIT / COMPOSE / SPLIT / RETIRE) so the joint search stays valid.

u_i⁽ᵗ⁺¹⁾ = u_i⁽ᵗ⁾ + ε · û_i⁽ᵗ⁾ u_i⁽ᵗ⁾ ( 1 − Σⱼ wᵢⱼ uⱼ⁽ᵗ⁾ / K )
01 · SAMPLE

Sample Σₖ from Pareto front G, proportional to its unique-instance coverage; execute minibatch M.

02 · REFLECT

Proposer R fuses anti-pattern hits, ecological Δu and β̂ᵢⱼ, and structured feedback μf; emits bundle L.

03 · TOOL-SMITH

If L contains tool ops, Bτ applies only the 5 typed primitives — search space stays bounded.

04 · VALIDATE

Progressive unit → integration → regression tests; admit Σ′ to G; one Lotka–Volterra step; retire dead skills.

Strengths

  • Unifies skills, tools, and failure memory as a single evolutionary state — fixes three previously independent problems at once.
  • Atomic bundles guarantee Σₜ, Σₜ₊₁ ∈ Ω, expanding reachable valid states instead of merely enlarging edits.
  • Typed tool primitives prevent uncontrolled tool explosion while still enabling real tool-level repair.
  • Anti-pattern veto triples regression prevention (2.1% → 7.6% when removed) — cheap, high-leverage.
  • Comprehensive: 3 benchmarks × 5 model scales × 3 baselines + ablation + 100-round stability + case study.

Limitations

  • Framework inherits reflective-prompting weaknesses — small or weakly tuned proposers stall.
  • Cold-start noise in β̂ᵢⱼ when co-occurrence data is sparse; no quantified mitigation.
  • No evaluation on coding (SWE-Bench, HumanEval+), embodied agents, τ-bench, or BFCL.
  • No reported cost / efficiency analysis — gains not normalized against extra rollouts.
  • Self-WRAP / EDIT of tools raises safety/governance questions; only briefly surfaced.

Questions for Further Exploration

Q01

Can SkillSmith gracefully degrade on coding benchmarks where tool APIs are external and immutable — skill-only mode?

Q02

What is the rollout-budget-normalized gain vs. EvoSkill / GEPA — does +18.9 pp at 397B justify the extra rollouts?

Q03

Can the typed primitives be expanded (FORGET, VERSION, WRAP-INTO-MACRO) without exploding the joint search space?

Q04

How does the framework handle adversarial / noisy tool environments where structured feedback μf is unreliable?

Q05

Can ecological dynamics extend beyond pairwise βᵢⱼ to higher-order skill synergies (3-skill, role-based)?

Q06

Multi-agent co-evolution: can multiple agents share and evolve tools simultaneously with safety guarantees?

Q07

Safety audit — how to prevent an agent from WRAP-ing away its own safety checks via self-edited tools?

Q08

Theoretical guarantees: convergence, regret bounds, or PAC-style sample complexity for the Lotka–Volterra update under Σₜ ∈ Ω?
