Names and patches the concept bottleneck in CoCoNuT-style multi-pass latent reasoning — a missing piece for any team extending continuous-thought methods.

↳ from Relevance / TL;DR
Critical Edge

Single-seed GPT-2 124M results — the +3.6 EM HotpotQA gain is small enough that one unlucky run could erase it.

↳ from Critical Assessment
Methodology01

At each latent token t, three learned sigmoid gates read, forget, and write into a persistent concept vector ct shared across reasoning passes.

ĥt = LayerNorm(ht)
rt = σ(Wr ĥt),  ft = σ(Wf ĥt),  wt = σ(Ww ĥt)
h't = (1 − ft) ⊙ ht + rt ⊙ ct−1
ct = LayerNorm(ct−1 + wt ⊙ h't)
READ rtretrieves prior memory into current hidden state (−4.6 EM if removed) FORGET ftprunes irrelevant context (largest drop: −5.6 EM if removed) WRITE wtcommits gated state to residual memory (most critical: −5.2 EM if removed)

TRAINING   4-stage curriculum on GPT-2 124M, AdamW lr 1e-4, batch 128, gate weights init = 0, biases dataset-adaptive (Table 3).

Gain compounds with reasoning depth: on ProsQA at stage 6, vanilla CoCoNuT peaks at 95% then degrades to 92% — AGCLR holds 96%. Concept-stream probing shows 0.5–0.8 cosine similarity to answer-relevant entities (Penn↔William Penn, War↔WWI), directly explaining the HotpotQA gain.
Strengths
  • Clear empirical diagnosis of a previously unnamed failure mode (concept bottleneck) with a quantitative cosine-decay metric.
  • Mechanistic interpretability via logit-lens probing of the concept stream — the latent space is not treated as a black box.
  • Tight gate ablations + write-frozen-after-pass-2 check confirm persistent memory, not parameter count, drives the gain.
  • Only +1.41% parameters — drop-in for any CoCoNuT extension.
  • ≤8 GPU-hours total on consumer-grade hardware (A10 / GH200).
Limitations
  • Single base model (GPT-2 124M, 2019 arch) + single-seed results — no variance, no significance test.
  • No scale-up: untested on LLaMA, Qwen, Mistral — behaviour with RMSNorm / RoPE / GQA / SwiGLU is unknown.
  • CoT still beats AGCLR on GSM8K by 6.6 points; the "consistent improvement" headline is partly a low-baseline regime (HotpotQA 14.0 EM).
  • Ablation surprise: freezing write gate after pass 2 costs only −0.8% EM → "write once, read many" works, but is not pursued.
  • Three-gate design not justified theoretically (LSTM-style gating as architectural prior).
Questions for Further Exploration03
  1. Does AGCLR transfer to ≥1B models with RMSNorm/RoPE/SwiGLU, or does the persistent ct interact poorly with modern normalisation?
  2. If "write once, read many" only loses 0.8% EM, can AGCLR be simplified to a write-once-then-freeze design?
  3. How does AGCLR compare to a retrieval-augmented baseline on HotpotQA — would a learned retriever over the concept stream replace the gates?
  4. Can the three-gate design be derived from an information-theoretic or optimal-control objective?
  5. Does AGCLR compose with other CoCoNuT extensions (amortised latent steering, latent self-consistency)?
  6. Multi-seed evaluation across GPT-2, Pythia, TinyLlama — are the +3–4% gains robust or seed-artefactual?
  7. Sparse-autoencoder or causal-mediation analysis (à la Chaudhary & Geiger 2024) on what the gates actually write?
