> **一句话总结：** EvoDS 把数据科学任务拆给一个 Manager Agent + Cleaner / Featurizer / Modeler / Visualizer / Debugger 五个专用子代理（动作空间互不相交），并提出 (1) 基于生成计数阈值 τ=3 的 Autonomous Skill Acquisition (ASA) 自演化技能库；(2) 由 RL 端到端学习的 Adaptive Context Compression (ACC)；在 4 个基准平均超过开源 SOTA 数据科学代理 28.9%，并彻底消除 25k+ token 的 OOT 失败。

> **TL;DR:** EvoDS is a KDD'26 self-evolving autonomous data science agent that combines a hierarchical multi-agent architecture (Manager + 5 specialized sub-agents with disjoint action spaces) with two self-evolution mechanisms: (i) **Autonomous Skill Acquisition (ASA)** — sub-agents synthesize candidate executable skills, validate them, cache them, and only permanently add a skill to their action space once it has been re-synthesized τ=3 times, preventing flaky additions; (ii) **Adaptive Context Compression (ACC)** — the Manager treats compression as a learned control problem rather than a passive truncation, with the policy jointly trained via a two-stage agentic RL (SFT warm-up + multi-role online RL with a token-cost penalty). Theoretically, EvoDS proves a strict tool-selection error bound reduction under the hierarchical design and casts the manager's objective as an Information Bottleneck. Empirically, EvoDS outperforms state-of-the-art open-source data science agents by an average of 28.9% across four benchmarks (MLAgentBench, DA-Code, DSEval, ML-Tool-Bench) and eliminates out-of-token failures on tasks exceeding 25k context tokens.

## 核心贡献

- **层次化多代理数据科学架构**：Manager Agent 做高层任务分解与协调，Cleaner / Featurizer / Modeler / Visualizer / Debugger 五个子代理各自维持本地化动作空间（**互不相交**），全局 + 局部双层记忆，把长执行上下文局部化到子代理内部，缓解"lost-in-the-middle"。
- **Autonomous Skill Acquisition (ASA)**：当子代理超出自身能力时，主动合成新可执行技能；技能经 cache → usage-frequency (生成计数 c(a)) 阈值 τ=3 过滤后才永久并入动作空间 ΔA_i，避免一次性偶发技能。
- **Adaptive Context Compression (ACC)**：把"何时压缩、压缩什么"视为一个 RL 学习到的控制策略，而非被动截断；token 成本以熵 H(Z) 形式计入奖励函数，让 Manager 在任务奖励与上下文预算间自动折中。
- **两阶段 agentic RL 训练**：第一阶段 teacher SFT 提供稳定初始化；第二阶段 joint multi-role RL 同时优化 Manager + 子代理；通过 stop-gradient 的 fast-weight rollout 实现 θ0 与 Δ_t 解耦更新。
- **理论保证**：证明层次化代理在工具选择误差上有严格更小的上界（Theorem A.5），并将 Manager 的优化目标等价为信息瓶颈（Theorem A.6）。
- **实证**：在 MLAgentBench、DA-Code、DSEval、ML-Tool-Bench 上平均比开源 SOTA 数据科学代理提升 28.9%；25k+ token 上下文任务零 OOT 失败（DeepAnalyze 大量 OOT）。

## Key Contributions

- **Hierarchical multi-agent data science agent.** Manager Agent for global reasoning + 5 specialized sub-agents with **disjoint** local action spaces A_i ∩ A_j = ∅. Two-tier memory (global objective + local subtask context) localizes long execution contexts.
- **Autonomous Skill Acquisition (ASA).** Sub-agents synthesize new executable skills on demand; validated skills go into a synthesized repository ΔA_i; a usage-frequency-aware expansion permanently adds a skill only when its generation count c(a) ≥ τ (default τ=3). This avoids both frozen action spaces and one-shot fluke additions.
- **Adaptive Context Compression (ACC).** Models context as a learned control problem rather than passive truncation. Manager decides *when* to compress and *what* to retain; token cost is modeled as E[|Z|] = κ·H(Z) and penalized in the reward, so the policy learns to drop redundant tool outputs and repeated dialogue.
- **Two-stage agentic RL.** Stage 1: teacher trajectory SFT. Stage 2: joint multi-role RL over (Manager + sub-agents) with fast-weight rollout and stop-gradient; this simultaneously trains task execution, skill acquisition quality, and context management.
- **Theory.** (i) Theorem A.5: hierarchical agent's tool-selection error is strictly smaller than base agent's under mild assumptions (k_j* < K, σ²(φ(C_j)) ≤ σ²(φ(C))). (ii) Theorem A.6: manager's optimization objective is equivalent to the Information Bottleneck min I(Z;C) − λ I(Z;Y), with λ = γκ / f' > 0.
- **Empirical wins.** 28.9% average gain over open-source SOTA data-science agents across MLAgentBench, DA-Code, DSEval, ML-Tool-Bench; eliminates out-of-token failures on >25k token contexts where DeepAnalyze frequently OOT's.

## Methodology

**Hierarchical multi-agent architecture.** The Manager is the only agent that interfaces directly with the code-execution environment and decides between high-level reasoning and subtask delegation; specialized sub-agents (Cleaner, Featurizer, Modeler, Visualizer, Debugger) execute concrete subtasks within their local action spaces. Invoking a sub-agent is a high-level action `a_i^sub = Invoke(π_θi, q_j)` for the Manager. Each sub-agent maintains **localized** memory for subtask-specific execution and a **shared global** memory for task objective alignment, decomposed as A_i ∩ A_j = ∅.

**Autonomous Skill Acquisition (ASA).** Sub-agents detect out-of-capability requests, synthesize a new candidate skill `a_new = (n, d, c)` (name / description / executable code), validate it in the sandbox, and cache it in a synthesized repository ΔA_i. The expansion rule is `A_i ← A_i ∪ {a_new ∈ ΔA_i | c(a_new) ≥ τ}`, default τ=3. This is the **usage-frequency-aware** gate that prevents one-off or buggy skills from being permanently added.

**Adaptive Context Compression (ACC).** At each compression trigger, the Manager decides whether to compress (when context is approaching the token limit) and what to retain (which findings to keep). The Manager's policy treats compression as a control decision; the reward combines task outcome R_outcome with a token-cost penalty proportional to H(Z) (Assumption 6 in the appendix). Under monotonicity assumptions on the task performance, this is equivalent to an Information Bottleneck objective.

**Two-stage training.**
1. **SFT warm-up:** Collect trajectories from a teacher model (DeepSeek-V3.1 / Qwen3); SFT the Manager and sub-agents on successful trajectories for initialization.
2. **Joint multi-role RL:** Multi-agent GRPO/REINFORCE++ over the rollout of all roles. Sub-agent rollouts are sampled with fast-weight updates (Eq. 2–4 in the paper); the Manager gets the outcome reward plus a token-cost penalty. Gradients flow through normal actions (improve reasoning) and extraction/expansion actions (improve skill synthesis quality and ACC decisions).

**Fast-weight rollout dynamics.** During an episode the LoRA Δ_t is the test-time memory state; the base parameters θ_0 are held fixed within a rollout and updated across RL steps. This treats Δ_t as part of the rollout transition rather than an RL parameter, so the optimization target is explicit: train θ_0 to produce better ordinary actions and better extraction actions so that subsequent online updates are more useful.

## Results & Highlights

- **Average gain over open-source SOTA: +28.9%** across MLAgentBench, DA-Code, DSEval, ML-Tool-Bench.
- **Context robustness:** On DA-Code token-length buckets, EvoDS maintains accuracy across 5k–25k contexts while DeepAnalyze degrades sharply and OOT's past 25k. EvoDS never exceeds 25k tokens (Figure 3).
- **Joint RL training dynamics:** Compared to naive GRPO that optimizes only the Manager, EvoDS's joint RL achieves higher validation reward with fewer context tokens throughout training; context usage rises then plateaus, indicating the token-cost penalty effectively shapes behavior (Figure 4).
- **Skill reuse case study (Figure 2):** A skill synthesized on one stroke-prediction task is reused on a downstream submission-format task; a third case shows the system honestly failing on a professional quantitative-finance task due to insufficient domain expertise — illustrating where self-evolution cannot substitute for domain knowledge.
- **Theory-aligned:** Theorem A.5 confirms that the disjoint-subset architecture strictly reduces tool-selection error vs. a flat base agent; Theorem A.6 shows the Manager's objective is an Information Bottleneck, which provides principled justification for the ACC penalty.

## Critical Assessment

- **Novelty.** The combination of (disjoint-subset hierarchy + ASA with usage-frequency gate + ACC as a learned policy + joint multi-role RL) is incremental over existing components (A-Flow, AutoKaggle, DeepAnalyze, ACON, AFlow) but the *specific composition* is genuinely new. The τ-gate is a small but useful engineering decision that prevents garbage in the skill library.
- **Rigor.** Strong on theoretical side: Theorem A.5 gives a strict error-bound reduction under explicit assumptions; Theorem A.6 makes ACC's principled connection to IB. Empirically, the paper compares across four benchmarks and the case study is detailed, but variance / seed-count reporting is thin.
- **Reproducibility.** Code is public, the KDD camera-ready should pin seeds and teacher model versions, but I cannot fully verify the reproducibility chain from the PDF alone.
- **Significance.** The result that ASA + ACC + joint RL "eliminates OOT failures" is meaningful for long-horizon data science work — but it should be interpreted as "eliminates OOT failures on this set of open-source models on these four benchmarks", not "removes the long-context barrier". Tasks with much longer contexts (multi-million-token scientific corpora) are out of scope.
- **Red flags:**
  - The +28.9% average improvement is across open-source baselines only; a head-to-head with closed-source (Claude-Opus-4.6 + planning, DeepAnalyze-Pro) is missing.
  - The skill library accumulation policy (τ=3) is a hyperparameter; sensitivity analysis is in the appendix but the **risk of skill library explosion** over thousands of tasks is not analyzed.
  - Joint multi-role RL is technically delicate; the paper claims it works but does not show full convergence plots vs. naive GRPO beyond a single benchmark.
  - Cross-task skill transfer is demonstrated in one qualitative case; the quantitative claim of "cross-task reuse" is not rigorously measured.
  - Domain gap: quantitative finance is honestly flagged as a failure case; broader domain coverage (bio, materials, social science) is not tested.

## Strengths

- Genuinely addresses two coupled bottlenecks (skill acquisition + context management) that prior data-science agents treat separately.
- Theoretical backbone (error bound + Information Bottleneck) — rare for an agent-systems paper.
- Honest failure case study (quant finance task) rather than only showing successes.
- Clean engineering decision (τ=3 frequency gate) that prevents one-shot garbage from polluting the skill library.
- Code release at github.com/usail-hkust/EvoDS.

## Limitations

- Only open-source baselines compared; no closed-source SOTA head-to-head.
- Skill library explosion and long-term maintenance over thousands of tasks not analyzed.
- Cross-task skill transfer is qualitative (one case study); no quantitative transfer metric.
- Domain coverage is shallow: MLAgentBench / DA-Code / DSEval / ML-Tool-Bench are all standard "tabular ML" benchmarks; no scientific-discovery / time-series / NLP benchmarks.
- Joint multi-role RL is technically delicate; convergence behavior beyond the one benchmark shown is not characterized.
- Theoretical bounds require Assumptions 1–6 (i.i.d. Gaussian noise, monotonicity of reward in I(Z;Y), entropy–token cost proportionality); the empirical validity of these assumptions is not separately verified.

## Relevance

## Questions for Further Exploration

- How does EvoDS's skill library behave at the scale of 1,000+ tasks? Does τ=3 still prevent garbage, or does a learned τ (or a decay rate) help?
- Can the ASA + ACC framework be combined with a process-reward model (PRM) so that skill synthesis is guided by a verifier rather than execution success alone?
- Does the disjoint-subset action space still reduce tool-selection error empirically when sub-agents' responsibilities overlap (e.g., a Featurizer that occasionally debugs)?
- Is the Information Bottleneck bound tight enough to predict the empirical token-cost / accuracy curve, or are the i.i.d. noise assumptions too lossy?
- How does EvoDS perform on scientific discovery benchmarks (SciHorizon, ScienceAgentBench) where skill reuse would matter most?
- Can the ASA mechanism be made transfer-aware: reuse a skill across sub-agents with a small fine-tune, instead of synthesizing from scratch each time?
- Is the joint RL training stable across more than two base models (Qwen3-4B, Qwen3-8B, …) and what are the variance bounds on the +28.9% claim?
- How does EvoDS compare head-to-head with closed-source agentic systems (Claude-Opus-4.6 with planning, AIDE, RE-Bench) on the same four benchmarks?
- Does the hierarchical architecture's error-bound reduction hold empirically under heavier sub-agent error correlations (e.g., when sub-agents share a backbone)?
- Can the ACC mechanism be extended to compress *across* sub-agent boundaries (shared summary layer) without breaking the disjoint action space assumption?
