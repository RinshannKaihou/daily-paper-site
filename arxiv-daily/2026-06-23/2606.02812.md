> **一句话总结：** Traj-Evolve 提出首个面向真实临床纵向电子健康记录（EHR）肺癌早筛任务的自演化多智能体框架，通过"经验池（ExPool）非参数检索 + 多智能体强化学习（MARL）参数内化"两条互补路径，将每位新患者交由一个"持续从已验证病例中学习"的链式智能体系统进行推理，使 1 年内肺癌风险预测在整体人群和不吸烟亚组均刷新 SOTA。

> **TL;DR:** Traj-Evolve is the first self-evolving multi-agent framework for longitudinal EHR modeling on a real clinical prediction task. It extends Traj-CoA with two complementary evolving mechanisms: an **Experience Pool (ExPool)** that stores label-conditioned rejection-sampled reasoning traces as non-parametric "patients-like-me" few-shot context, and **multi-agent reinforcement learning (MARL)** that internalises successful temporal-reasoning traces via reward-ranked fine-tuning on the worker–LTM–manager chain. A leave-one-out cross-retrieval procedure unifies them. On up to five years of multimodal EHRs from a large single-institution cohort, the combined system reaches AUROC 0.86 (overall) and 0.835 (never-smokers), outperforming nine baselines including LCRAT, XGBoost, RETAIN, PatientTM, Clinical ModernBERT, vanilla GPT-OSS-20B, RAG and Traj-CoA.

## 核心贡献 / Key Contributions

- **首个用于真实临床纵向 EHR 建模的自演化多智能体框架**：把"系统在使用过程中持续学习"这一临床直觉显式建模到 LLM 多智能体架构中，填补了现有静态 LLM 时序建模系统（Traj-CoA、MoMA、CARE-AD、CliCARE、TRACE 等）"逐例孤立推理"的空白。
- **提出两类互补的演化机制**：
  1. **ExPool（演化的经验池）**：基于向量数据库的"非参数程序性记忆"，对每个已验证患者做拒绝采样（best-of-m），按标签选择最优推理轨迹（病例取风险最高，对照取风险最低），再以最终 Worker 摘要嵌入为键、Manager 推理与真实诊断为值存入向量库，推理时检索 top-k "patients-like-me" 作为 Manager 的 in-context 示例。
  2. **MARL（多智能体强化学习）**：将 RAFT 思想扩展到 CoA 链路，对每条 roll-out 用真实标签做"硬奖励"过滤（病例 s≥6、对照 s≤4），再解耦优化 Worker 参数（强化跨链段时序推理与 worker–LTM 协作）和 Manager 参数（强化最终风险聚合）。
- **统一的 leave-one-out 交叉检索协议**：训练阶段对每条 MARL 轨迹都构造 E\{i} 动态经验池进行检索增广，使训练与推理时的输入分布对齐，避免数据泄露。
- **临床验证：在 1 年肺癌发病预测上整体与不吸烟亚组双 SOTA**：训练集 13,629 例，整体测试 1,000 例（含 90 病例/910 对照），从不吸烟子集 835 例（含 27 病例/808 对照），均刷新 AUROC/AUPRC/F1。
- **自演化动力学分析**：揭示 (1) ExPool 从小到大时检索从"追求多样"转向"追求特异"；(2) MARL 下 Manager 收敛快、Worker 持续受益；(3) ExPool 偏特异度、MARL 偏灵敏度、两者互补。

## Methodology

**任务形式化**：给定患者 $p_i$ 长达五年的多模态 EHR 序列 $X_i = \{(t_{i,j}, e_{i,j})\}_{j=1}^{T_i}$（含结构化诊断/用药/化验/生命体征/操作以及非结构化临床与放射学报告），目标是学习 $f_\theta: X_i \mapsto (s_i, r_i)$，输出 1–10 的 1 年肺癌发病风险 $s_i$ 与自然语言推理 $r_i$。

**基座系统 Traj-CoA**（Zeng 等 2025/2026）：将 EHR 序列化为 XML 并按时间切片，每段交给一个共享参数 $\theta_w$ 的 Worker Agent，Worker 维护运行摘要 $u_{i,\ell}$ 与"长期情景记忆 LTM" $M_{i,\ell}$（仅保留与肺癌相关的事件）。最终摘要 $u_i$ 与 LTM $M_i$ 一并交给 Manager Agent $\pi_{\theta_m}$ 输出 $(s_i, r_i)$。

**ExPool 构建**：对每个 $p_i \in D_{tr}$ 在高温 $\tau>1$ 下采样 $m$ 条 roll-out $\{u_i^{(j)}, M_i^{(j)}, s_i^{(j)}, r_i^{(j)}\}$，按标签条件保留"最优轨迹"（病例取风险最高、对照取风险最低），把最终 Worker 摘要嵌入 $v_i=\phi(u_i^\star)$ 与 $(r_i^\star, s_i^\star, y_i)$ 一起存入向量库 $E$。

**ExPool 推理**：对测试患者 $p_q$ 用嵌入余弦相似度检索 top-k 邻居 $\{N_k(q)\}$，把"patients-like-me" 的完整推理、预测分、真实标签一起拼到 Manager 的输入中，让 Manager 综合 $u_q, M_q$ 与邻居做对比式推理（而非做多数投票式的硬标签集成）。

**MARL 数据构造与解耦优化**：同样采样 $m$ 条 roll-out，但用 reward $R(s,y)=\mathbb{1}[s\ge 6, y=1] \lor \mathbb{1}[s\le 4, y=0]$ 做"硬过滤"得到 accepted set $A$，再按时间均衡子采样（首、尾 + 两个随机中间 Worker）拆成 $D_w$（Worker 输入–输出对）与 $D_m$（Manager 输入–输出对），分别独立最小化负对数似然更新 $\theta_w^\star, \theta_m^\star$。

**两机制的统一（Leave-one-out Cross-Retrieval）**：对每个训练患者构造 $E_{-i}=E\setminus\{i\}$，在 MARL 的 roll-out 阶段用 $E_{-i}$ 给 Manager 检索"patients-like-me"，再做同一硬奖励过滤，得到 $A^{loo}$；推理阶段仍用全量 $E$。这一协议同时解决 (1) 训练–推理分布对齐、(2) 检索信号进入参数优化、(3) 无数据泄露。

**实现细节**：基座 LLM 为 GPT-OSS-20B，ExPool 嵌入用 nomic-embed-text-v1.5，roll-out 数 $m=4$、温度 $\tau=1.5$；Worker 与 Manager 通过 QLoRA 在 4 卡上微调 1 个 epoch（LoRA rank/$\alpha$=32，8-bit AdamW，per-device batch=1，grad-accum=8，lr=2e-4，5 步 warmup，weight decay=0.01）；向量库检索 top-k ∈ {5,10,15}。

## Results & Highlights

**整体人群（n=1,000；90 病例）**——综合 Traj-Evolve (ExPool+MARL) 取得 **AUROC 0.860 / AUPRC 0.323 / F1 0.417**，全面超越：
- LCRAT 0.690、XGBoost 0.755、RETAIN 0.699、PatientTM 0.683；
- Clinical ModernBERT 0.740；
- GPT-OSS-20B 0.794、RAG 0.806、Traj-CoA 0.814。

两个消融变体同样优于所有静态基线：Traj-Evolve (ExPool) 0.837、Traj-Evolve (MARL) 0.843。

**从不吸烟子集（n=835；27 病例）**——传统模型在该子集大幅退化（LCRAT 0.606、XGBoost 0.706、RETAIN 0.571、PatientTM 0.623），而 Traj-Evolve (ExPool+MARL) 仍取得 **AUROC 0.835 / AUPRC 0.279 / F1 0.309**，在 AUPRC 上把 Traj-CoA 从 0.227 提升到 0.279，AUROC 从 0.775 提升到 0.835，验证自演化机制对"无吸烟史的隐性高风险人群"特别有用。

**推理质量**：用 GPT-OSS-120B 做 LLM-as-judge 成对比较，Traj-Evolve 在 Overall 上 69% 胜出，其中细节 69%、临床推理 73%、时间一致性 68%。

**ExPool 演化动力学**：
- 邻居平均嵌入距离随池子规模增大单调下降；
- 与 index patient 的 Spearman 相关（年龄、风险分）单调上升；
- 按病例状态、性别、从不吸烟属性的检索"纯度"（purity）稳定高于随机基线；
- **最优 k 从 15 → 5 漂移**，提示小池子需要多样性补偿，成熟池子偏好特异检索；且任意 k 下 AUROC 都高于零样本 Traj-CoA 基线 0.814。

**MARL 演化动力学**：
- Manager 训练 loss 快速收敛，Worker 训练 loss 持续下降（时序推理更慢被内化）；
- 测试 AUROC 从 ~0.81 提升到 ~0.83，对应训练样本从 0 到 5,000。

**机制互补性（密度散点图）**：ExPool 主要把对照风险分拉低（特异度 ↑），MARL 主要把中等病例风险分拉高（灵敏度 ↑），两者叠加给出最干净的病例–对照分离。

**Case Study**：50 岁亚裔女性从不吸烟者，职业石棉暴露，2018 年发现 16 mm 左上叶非钙化结节；ExPool 检索到的 P8（55 岁女性石棉暴露但无肿块，3 年未癌）和 P10（57 岁亚裔女性从不吸烟者，18 mm LUL 肿块，6 个月内确诊）形成"生物学上介于两者之间"的对照，Manager 输出 7/10 中高风险——说明检索增广确实驱动了细粒度的对比式临床判断。

## Critical Assessment

- **新颖性**：明确把"自我演化"概念首次系统地落到真实临床纵向 EHR 建模任务上，并把 memory-evolving 与 parameter-evolving 两条路径在同一框架内统一（多数先前工作只走其中一条）。同时给出 leave-one-out 协议保证训练–推理分布对齐，这种工程层面的细致是论文的真正加分项。
- **方法严谨性**：
  - 数据是单中心回顾性病例–对照（1:10 按检查类型/日期匹配），外部效度未验证；
  - 测试集病例仅 90（整体）/27（从不吸烟），AUROC 误差 ±0.02–0.05，**最终结论的统计功效与不确定性**应被审慎解读；
  - "正确预测"由最终 s 与 y 一致决定，但最终答对 ≠ 中间推理正确，论文自己也在 Limitations 承认这点；
  - 标签依赖：临床真实场景中肺癌标签可能数月到数年才能确定，存在 verification noise 与失访偏倚；
  - 高温采样 + 硬奖励过滤可能放大 roll-out 随机性，对稀有亚组（从不吸烟 27 例）的鲁棒性需要更细的 bootstrap CI。
- **可复现性**：基座 GPT-OSS-20B 与 nomic-embed-text-v1.5 均为开源/开放权重模型，QLoRA 微调、4 卡硬件门槛合理；但 prompt 模板与 ExPool 检索逻辑的细节实现决定复现差距；论文给出了系统/用户 prompt（表 4、表 5），提高了复现可能。
- **意义**：把"self-evolving agent" 从游戏/数学/QA 推到高风险医疗任务，且在不吸烟亚组上把 AUROC 从 0.775 提升到 0.835，对"被传统风险模型遗漏的隐性高风险人群"具有潜在临床价值；但单中心、回顾性的设计决定了**离临床落地还有相当距离**。
- **红旗**：(1) 测试集规模偏小；(2) 仅与 9 个基线对比，未对比更新的医疗 LLM（如 MedGemma、Med-PaLM 系列、EHR-MLMs）；(3) case study 仍属定性证据，未做用户研究或临床医生评估。

## Strengths

- 首次将 self-evolving agents 落地到真实临床纵向 EHR 预测任务；
- ExPool + MARL 双机制互补，并在散点图上看到清晰的特异度/灵敏度分工；
- Leave-one-out 检索协议精巧地解决了训练–推理分布不对齐问题；
- 在难的不吸烟亚组上提升尤其明显（AUROC +0.06, AUPRC +0.05），提示对"无主要风险因素"的隐性病例更鲁棒；
- 系统/用户 prompt 与超参全部公开，方便复现；
- 训练样本量充足（13,629）、多模态输入、HIPAA 合规；
- LLM-as-judge 的细粒度评估（细节/临床推理/时间一致性）展示推理质量也同步提升。

## Limitations

- 单中心回顾性数据，外部效度未验证，且测试集病例数仅 90/27，统计功效有限；
- 完全依赖真实肺癌标签，但临床诊断本身存在 verification noise、随访丢失与确认偏倚；
- 最终答对 ≠ 中间推理正确，label-conditioned rejection sampling 可能在错因对果的样本上误导 MARL；
- 仅在 GPT-OSS-20B 一个基座上验证，未与更大的医疗 LLM 或更新的 EHR-MLM 比较；
- 仅做 1 年肺癌发生预测 + 5 年回溯窗口，未评估更短/更长预测期、多病种、滚动更新的真实流式场景；
- 病例–对照 1:10 匹配高于临床部署时的真实患病率，部署期校准（calibration）需要重新评估；
- 训练涉及 LLM 调参与多次 roll-out，计算成本较高，缺少 FLOPs/碳排数据；
- 未做医生评估或用户研究。

## Relevance

本文与"AI for 临床早期风险预测"和"自演化 LLM 智能体"两条主线高度相关：对前者，提出一种能在**多模态长程 EHR（60k–80k token）**上稳定工作的多智能体推理架构；对后者，把"经验池 + 多智能体 RL"从游戏/数学/QA 推广到高风险的真实临床预测场景，并指出**检索经验**与**参数内化经验**是分工互补而非互斥的两条学习路径。对研究纵向 EHR 时序建模、多智能体 RL、以及 LLM 临床应用可信度评估的读者具有直接参考价值。

## Questions for Further Exploration

- ExPool 的检索键目前只用了 Worker 最终摘要嵌入，若改为 (a) 融合结构化特征 (LTM 事件类型/频率) 或 (b) 任务感知的对比嵌入，能否在不吸烟等稀疏亚组上获得更稳定的检索纯度？
- MARL 的硬奖励（s≥6 / s≤4）是阈值化的，可在不吸烟亚组引入 **process-level / dense reward**（如中间推理 step 的临床一致性、检索使用率），是否能让 Worker 的训练 loss 收敛更快？
- ExPool 与 MARL 在不同疾病、不同 EHR 长度（如 ICU 时序）上的可迁移性如何？是否能扩展到多病种早期预测（不止肺癌）？
- 在真实部署患病率下，Traj-Evolve 的 calibration（预测分与真实概率的一致性）如何？能否结合 conformal prediction 输出有覆盖保证的风险区间？
- Leave-one-out 是为单中心设计，若推广到多中心流式场景，pool 的去重/隐私约束/版本管理应如何设计？
- 与 EHR-MLM（Clinical ModernBERT、MedGemma、EHR-Pretrain 系列）相比，自演化机制在 few-shot 与 SFT 设定之外是否仍保持优势？
- 是否可以做"反事实检索"实验：故意注入错误标签的"坏经验"进 ExPool，观察系统是否会"错误演化"，进而引出对经验池治理（人类反馈、临床医生审核）的研究？
- 推理质量评估用了 GPT-OSS-120B 作为 judge，是否能进一步用临床医生盲评或患者随访结局来验证 LLM-as-judge 的可靠性？
