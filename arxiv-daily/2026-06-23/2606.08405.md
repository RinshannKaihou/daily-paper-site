> **一句话总结：** 提出一种由 LLM 驱动、通过迭代源代码改写与 CFD 仿真反馈实现的"自演化科学智能体"工作流，在欠驱动两关节机器鱼目标到达任务中，仅用 20 步迭代便合成出可解释、可追溯、可泛化的统一控制器，性能超越典型深度强化学习所需的海量仿真预算。

> **TL;DR:** This paper introduces a self-evolving scientific agent that drives LLM-based iterative source-code rewriting against a CFD fluid-structure interaction (FSI) simulator, discovering a unified, interpretable target-reaching controller for an underactuated two-joint dogfish swimmer. Starting from a one-sided propulsive seed, the agent reaches all four canonical targets by iteration 6 and refines path efficiency by iteration 20, while the auditable source-code log reveals an emergent architecture built from traveling-wave propulsion, body-frame target guidance, yaw-rate feedback, signed mean-tail curvature, and adaptive cadence relief — achieving generalization to unseen static and moving targets without retraining or task-specific branching.

## 核心贡献

- 提出一种面向物理可解释性的"自演化科学智能体"工作流：智能体不能直接调节神经网络权重，而是通过 *Simulate → Package Evidence → Diagnose → Policy Evolve* 四阶段循环，改写可审计的源代码控制器。
- 在极具挑战性的欠驱动两关节机器鱼 FSI 目标到达问题上首次验证该闭环：仅由两个关节角加速度驱动，控制器需自主产生推进与双向转向能力。
- 以极低的数据成本实现突破：在 6 次迭代中即实现 4/4 全部规范目标的成功到达，20 次迭代内进一步优化路径效率；对比传统 DRL 通常所需的数千次仿真，显示出数量级级别的样本效率优势。
- 通过源代码演化轨迹，揭示出一个**层次化、可读、可追溯**的控制架构：行波推进、机体坐标系目标引导、偏航率反馈、有符号平均尾曲率、自适应节奏增强与转弯载荷释放。
- 在不重新训练、不分支的情况下，统一策略泛化至 8 个未见过静态目标与一条动态曲线追踪轨迹，展现出真正的闭环反馈能力而非轨迹记忆。

## Key Contributions

- **Interpretability-first agentic loop for physical control:** A strict four-stage workflow (Simulate Candidate → Package Evidence → Diagnose → Policy Evolve) where the LLM edits source code rather than neural weights, with a constrained edit surface (unified feedback, finite accelerations, no hidden state / randomness / case-branching).
- **Challenging FSI benchmark validation:** Underactuated two-joint dogfish swimmer must reach four canonical targets using only joint angular accelerations (ϕ̈₁, ϕ̈₂), solved by 2D incompressible Navier–Stokes (WaterLily/BDIM solver, Re = 2000, 12L × 6L domain, 30 L/U horizon, capture radius 0.3 L).
- **Sample efficiency breakthrough:** 4/4 target success at iteration 6, path-efficiency refinement by iteration 20 — versus the typical thousands of DRL rollouts needed for opaque policy fitting.
- **Auditable emergent control architecture:** Decoded from the source-code log: traveling-wave drive, body-frame target guidance, half-cycle asymmetric actuation, yaw/bearing-trend feedback, signed mean-tail curvature, centerline braking, dynamic cadence boost, turn-load relief, and terminal recovery.
- **Generalization proof:** Without retraining or branching, the iteration-20 policy reaches 8 held-out static targets and a continuously moving curved target within the same 0.3 L capture radius.

## Methodology

**Physical testbed.** A 2D incompressible Navier–Stokes fluid solver (WaterLily.jl, BDIM, Julia, GPU-capable) is coupled to a deformable dogfish-like swimmer. The body has a fixed thickness profile w(s) swept on a two-joint bendable spine (joints at s = 1/3 and 2/3 with smootherstep supports of half-width 1/8). Actuation is restricted to joint angular accelerations aϕ = (ϕ̈₁, ϕ̈₂), clipped/integrated under limits of 45° incremental bend, 140°/s rate, and 900°/s² acceleration. Global motion (centre-of-mass rc, heading ψ) is an emergent response to hydrodynamic force F and moment Mz integrated over the body surface. Episodes run in a quiescent 12L × 6L domain (Re = 2000, horizon 30 L/U, capture radius 0.3 L).

**Task.** Reach four canonical spatial targets (different heights, all from the same starting pose at (9L, 3L)) using one and the same feedback law. The state input includes the body-frame target vector time series, body-frame hydrodynamic load, swimmer velocity/heading, and the two joint angles/rates.

**Agent loop.** At each iteration:
1. **Simulate Candidate** — the CFD task interface rolls out the current source-code policy (φ̈₁, φ̈₂ outputs).
2. **Package Evidence** — returns compact multimodal summaries: aggregate score, per-target diagnostics, sampled trajectories, vorticity keyframes.
3. **Diagnose** — the agent identifies active failure mode (insufficient propulsion, one-sided bias, wrong-sign steering, overshoot, etc.) and decides whether to consult the passive Knowledge Shelf.
4. **Policy Evolve** — the agent rewrites the source-code controller and updates paired optimizer guidelines; the dual artifact (executable policy + guidance + experience notes) is the actual evolving object.

A **passive Knowledge Shelf** holds classical priors from Lighthill, Taylor, Triantafyllou, Sfakiotakis, Ijspeert, Fossen, Katzschmann, etc. The shelf is consulted only when local diagnosis demands an architectural update, never as a default template — forcing the agent to translate abstract mechanisms (e.g., traveling waves, signed tail curvature) into the highly restricted joint-acceleration state space.

**Constraints on the evolved controller.** No hidden time tracking, no random numbers, no target-specific case branching; a single readable feedback law that handles all canonical targets.

**LLM workflow stack.** Builds on the authors' prior "Evolutionary Ensemble of Agents" framework (arXiv:2605.09018), adapted to a strict separation of roles for physical interpretability.

- **Code growth mirrors architectural elaboration:** Seed = 132 lines / 19 params → iter 6 = 230 lines / 42 params → iter 20 = 276 lines / 49 params. Growth is *structured* — each addition maps to an identifiable mechanism (Table 2).
- **Generalization (Fig. 5):** Iteration-20 policy reaches all 8 held-out static targets and a moving curved target without retraining or branching.
- **Emergent mechanisms (Table 2):**
  - Iteration 2 — turn-rate / bearing-trend feedback (kills wrong-sign responses).
  - Iteration 6 — **signed mean-tail curvature** (target-conditioned steering; first 4/4 success) + centerline braking.
  - Iteration 8 — approach & heading refinement.
  - Iteration 11 — dynamic cadence boost (+11–15% tail-beat freq when far / slow).
  - Iteration 14 — turn-load relief (suppresses cadence boost in sharp turns).
  - Iteration 19 — terminal restorative tail curvature (yaw/side-slip damper).

## Critical Assessment

**Novelty.** The scientific-agent-on-physical-control idea is not new (FunSearch, AlphaEvolve, Eureka, etc.), but most prior work targets mathematical-algorithm or reward-design discovery. This paper is one of the first to apply the agentic-code-evolution paradigm to a **genuine nonlinear FSI problem** (Navier–Stokes + moving deformable boundary) and one of the first to enforce *strict interpretability constraints* (no hidden state, no branching, single readable feedback law) on the evolved policy. The dual artifact (executable code + optimizer guidance + experience notes) as the evolving object is a thoughtful framing.

**Rigor.** Several checks are present: aggregate score, mean and worst per-target scores, completion time, fixed Reynolds number and solver (WaterLily), and explicit before/after rollout comparison. The audit Table 2 mapping each new code structure to a behavioral function is unusually thorough for this kind of work.

**Reproducibility.** Mixed. Strong on the framework structure (workflow clearly described, all parameters — body geometry, joint locations, capture radius, solver, Reynolds number — fully specified) but several gaps remain: the **specific LLM model(s), prompt templates, sampling temperature, and population size** used by the agent are not reported. Without these, reproducing the 6-iteration convergence exactly is unlikely. Code release is not mentioned in the main text.

**Significance.** A genuine proof-of-concept that *interpretable* agentic discovery can solve non-trivial underactuated FSI control with orders-of-magnitude fewer simulations than DRL — and that the resulting policy genuinely *generalizes* rather than memorizes. This is a meaningful data point for the "AI-for-science" / "AI-for-physics" debate about whether black-box optimization is the only viable route.

**Red flags.**
- Only one evolutionary run is "highlighted" (20 iterations); whether the 6/20-iteration timeline is reproducible across seeds is not demonstrated.
- Generalization tests are limited to *near-domain* targets (left side of the same starting region) and a single moving trajectory; no out-of-distribution stress (e.g., targets behind the swimmer, much larger domains, different Reynolds, obstacles).
- The Knowledge Shelf is hand-curated classical literature — whether the agent would discover the same mechanisms from scratch is an open question; the role of LLM priors vs. physical reasoning is not isolated.
- The aggregate score is opaque (sign convention, weighting) and not fully derived.
- "Without retraining or branching" is correct but somewhat overstated: the entire policy *was* evolved in-context; the policy's task-specific knowledge is implicitly encoded in the source code rather than literally via `if target == …` branches.

## Strengths

- **Radically interpretable:** every component of the final policy can be cited as a specific named mechanism (signed tail curvature, cadence boost, etc.), in stark contrast to typical DRL black-box weights.
- **Sample-efficient:** 6 iterations to full 4/4 success, 20 for refinement — orders of magnitude fewer simulations than DRL.
- **Generalization demonstrated:** held-out static targets and a moving target both reached without modification.
- **Carefully designed benchmark:** the dogfish FSI task is genuinely hard (underactuation, nonlinearity, hydrodynamic coupling) and physics-rich.
- **Auditable discovery trace:** the entire code-evolution log + simulation evidence together constitute a transparent scientific record.
- **Layered architecture:** later iterations refine and stack on earlier ones rather than replacing — a clean engineering of evolutionary discovery.

## Limitations

- **Single highlighted run, no statistical replicates** — variability across agent seeds is unknown.
- **No ablation** on Knowledge Shelf content (would the agent discover these mechanisms from scratch?), LLM model choice, or population size.
- **No external generalization** to fundamentally different regimes (different Reynolds, body shapes, multi-target races, 3D swimming, disturbances, currents).
- **No comparison to strong baselines** like well-tuned DRL, model-predictive control, or trajectory-optimization on the same task.
- **Reproducibility gap:** model identity, prompts, and full code not reported/released (as visible in the manuscript).
- **Score function opacity:** how the aggregate score is computed and what it precisely rewards is only partially described.
- **Domain of validation is narrow:** only "left-side" target zone + one moving chase; the swimmer is never asked to e.g. turn 180° or recover from a deliberately wrong initial pose.
- **Asymmetric role of the LLM** vs. the engineered Knowledge Shelf is not isolated; some "discovered" mechanisms may effectively be retrieved from curated priors.

## Relevance

本工作与我对"AI for Science / Agent for Physics"的兴趣高度相关：它将大语言模型驱动的代码演化范式从数学/算法发现推进到真正的非线性流-固耦合物理控制问题，并且通过严苛的可解释性约束，让控制器在源代碼层面可读、可审计、可追溯。其样本效率与泛化能力对"黑盒优化是否为唯一路径"的争论提供了反例。

## Questions for Further Exploration

- How does the discovered control architecture change if the **Knowledge Shelf is removed** (purely from-scratch discovery) versus **expanded** (modern CPG / RL literature)?
- Can the same workflow transfer to **3D FSI**, **higher Reynolds**, **flow disturbances / currents**, or **multi-swimmer coordination**?
- Does the strategy scale to other underactuated swimming/flying morphologies (eel-like continuous spine, flapping foil, jellyfish pulsation)?
- What is the **role of the LLM** versus the **diagnostic + scoring loop**? An ablation swapping in a non-LLM search (e.g., CMA-ES on source code) would isolate the agent's contribution.
- How does **iteration count and score variance** change across LLM seeds, model sizes, and prompt designs? Reporting the distribution (mean / std / min over ≥10 seeds) would be valuable.
- Is the **emergent architecture** stable, or does it co-evolve with prompt/context changes? Studying the *attractor landscape* in policy space would be illuminating.
- Could the **Knowledge Shelf be replaced** with an active retrieval-augmented agent over the literature, allowing discovery to benefit from newly published work without code rewrites?
- How does the **controller interpretability** translate to downstream **safety guarantees** for physical robots — e.g., can we formally verify the discovered source code against stability/reachability specs?
- Does the workflow generalize to **adversarial or partially observed** target environments where the target's bearing is intermittent?
