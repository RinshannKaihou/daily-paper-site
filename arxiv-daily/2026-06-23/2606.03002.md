# Perplexity Can Miss SAE Feature Damage Under Quantization

> **一句话总结：** 在 Pythia-70M 与 Gemma-2-2B 上，作者使用冻结的稀疏自编码器（SAE）作为固定测量基底，揭示了 round-to-nearest（RTN）量化下逐特征生存率呈"梯度式"下降，并指出困惑度（perplexity）改善或保持的同时，可能已有大量 SAE 特征被"模糊"甚至破坏——这对把全精度 interpretability 结论直接迁移到量化部署的实践提出了警示。

> **TL;DR:** Holding a frozen SAE fixed while sweeping round-to-nearest (RTN) quantization from INT8 to INT4 on Pythia-70M and Gemma-2-2B, the authors show that SAE feature survival declines in a graded—not cliff-like—fashion as bit-width decreases, that perplexity can remain stable or even *improve* while a large fraction of active features degrade (e.g., Gemma-2-2B INT7 lowers perplexity by 5.65% yet degrades 18.7% of features), that feature survival is predictable from full-precision feature statistics alone (cross-validated AUC 0.92–0.97, peak activation being the strongest marginal predictor), and that matched-perplexity RTN quantization and magnitude pruning damage strongly overlapping feature sets (Jaccard 0.79–0.86, Spearman 0.98). The paper argues that behavioral metrics are insufficient evidence that full-precision interpretability findings transfer to quantized deployments.

## 核心贡献 / Key Contributions

- **"困惑度可能漏掉特征损伤"——量化的可解释性盲区。** 论文首次系统性地提出并验证：在 Gemma-2-2B 上 RTN INT7 让困惑度改善 5.65%，却有 18.7% 的活跃 SAE 特征被破坏；滑动窗口评估下 INT6 也改善了困惑度，但只有 51.3% 的活跃特征存活下来。这一发现动摇了"困惑度/精度对齐即可放心部署"这一常见假设。
- **量化下的特征生存曲线是"梯度式"而非"断崖式"。** 在 INT8→INT4 的扫描中，Pythia-70M 活跃特征存活率由 98.3% 单调递减至 14.5%，Gemma-2-2B 由 99.1% 递减至 12.4%；破坏（damaged）占比在 INT7 及以上始终接近 0，崩坍主要发生在 INT5–INT4 区间。
- **特征生存可由全精度统计预测。** 仅使用 FP16 上的稀缺度、对数均值激活、对数峰值激活、对数集中度四个特征统计量训练 L2 正则化逻辑回归，5 折交叉验证 AUC 在 Pythia 上 0.924，在 Gemma 上 0.971。
- **量化与剪枝在相同困惑度代价下损伤相同的特征集合。** 与 INT6 困惑度匹配后做 magnitude pruning，二者未存活特征集合的 Jaccard 重叠为 0.79–0.86，损伤分数 Spearman 相关性高达 0.98，提示两类压缩机制共享相同的特征脆弱性图谱。

## Methodology

- **冻结 SAE 作为测量基底（QDM 流程）。** 同一组 token 上，对全精度（FP16）与压缩后（RTN INT8–INT4 或剪枝）模型分别编码残差流激活 $h^{(l)}(t;\theta)$ 到 SAE 特征空间 $z(t;\theta)=E(h^{(l)}(t;\theta))$，冻结 SAE 编码器 $E$、token 集、读出层（read-out layer）不变，仅改变模型权重 $\theta$。
- **逐特征 Pearson 相关 $c_j$。** 对每个活跃 SAE 特征 $j$，在共享 token 上计算 FP16 与压缩后激活的 Pearson 相关系数；活跃性定义为 FP16 firing rate $f_j > 0.001$。
- **三档生存分类法：** $c_j > 0.9$ 为 survived；$0.5 \le c_j \le 0.9$ 为 degraded；$c_j < 0.5$ 为 damaged。默认阈值 $t_s=0.9$，$t_d=0.5$，并在附录 B 中给出阈值敏感性扫描。
- **压缩算子：** Per-output-channel RTN 量化（b∈{8,7,6,5,4}）应用于所有 transformer 块线性层（attention、MLP、Gemma 的 gate projection），保留 layernorm 与 embedding；magnitude pruning 仅作为匹配困惑度的基线。
- **模型与 SAE：** Pythia-70M-deduped（blocks.4.hook_resid_post SAE）与 Gemma-2-2B（Gemma Scope layer-12 残差流 SAE，宽度 $d_{sae}=16384$）。
- **行为评估：** WikiText-2-raw 上 token 级 perplexity；分块协议（Pythia L=512，Gemma L=256）为主，辅以 Gemma 滑动窗口（W=2048, S=512）作稳健性检查；与逐特征 $c_j$ 测量解耦，避免协议对特征度量的污染。
- **脆弱性预测器：** L2 正则化逻辑回归（$C=1$），对四个 FP16 特征统计量（rarity、log mean、log peak、log concentration）做标准化后回归二分类标签 $y_j=\mathbb{1}[c_j > t_s]$；报告 5 折分层 CV-AUC 与标准化系数。
- **稳定性协议：** Pythia 用 200k tokens，Gemma 用 500k tokens；附录给出 token 预算敏感性（50k–200k 区间）、FP16-vs-FP16 null（均值相关 1.000000，0.0% damaged）、随机子集重复与第二层敏感性的稳健性检查。

## Results & Highlights

- **RTN 位宽扫描主表（Table 1）：** 在 Pythia-70M 上，生存率依次为 98.3% (INT8) / 86.3% (INT7) / 62.4% (INT6) / 38.0% (INT5) / 14.5% (INT4)，均值相关从 0.981 衰减到 0.583；在 Gemma-2-2B 上则为 99.1% / 81.3% / 51.3% / 27.7% / 12.4%，均值相关从 0.966 衰减到 0.494。两条曲线在 INT7–INT5 之间出现共同加速崩坍区。
- **关键反直觉：** Gemma INT7 把 ∆PPL 改善到 -5.65%，却使 18.7% 特征退化；INT6 在分块协议下 ∆PPL=+3.99%，但在滑动窗口下 ∆PPL=-3.45%，而 SAE 生存率始终是 51.3%——行为指标的符号会随协议翻转，但特征生存度量保持稳定。
- **预测能力（Table 2, Figure 3）：** 5 折 CV-AUC 在 Pythia 5,688 个活跃特征上 0.924±0.007，在 Gemma 7,144 个活跃特征上 0.971±0.002；按峰值激活分位，生存率从最低 1/4 分位的 13%/5% 上升到最高 1/4 分位的 96%/92%；rarity 在多变量模型中系数最大（4.52 / 17.84），但与激活幅度强相关，故需谨慎解读。
- **量化 vs 剪枝（Table 4）：** 匹配至 INT6 困惑度后做 magnitude pruning（Pythia sparsity=0.175，Gemma sparsity=0.1625），未存活特征集合 Jaccard 重叠 0.86 (Pythia) / 0.79 (Gemma)，per-feature damage score Spearman 相关 0.976 / 0.978；剪枝在 Gemma 上更激进，附带产生少量 damaged（$c_j<0.5$）特征尾部，而 RTN 没有。
- **方法学稳健性（Table 5 + 附录 B）：** Pythia INT6 在 50k–200k tokens 上 survival 稳定在 60.7%–62.8%；3 个独立 100k 子集仅相差 0.71pp；FP16-vs-FP16 null 给出 1.000000 均值相关与 0.0% damaged；阈值从 0.80→0.95 扫描结果单调合理。
- **跨模型一致：** 70M→2B（≈30× 参数规模）两条曲线形态一致，"INT8 起衰减、INT7–INT5 加速、INT4 大面积崩坍"，并支持"模糊而非毁坏"的中段特征主导模式（如 Gemma INT6：48.7% degraded，0.0% damaged）。

## Authors & Credibility

- **作者：** Evan Duan（密歇根大学，单作）。
- **可信度评估：** 单作者论文，无机构合作背书；方法学透明度高（明示所用 SAE 名称与层位、token 预算、阈值与协议选择，并在附录中提供阈值敏感性、token 预算稳健性、FP16 null、第二层敏感性等多项 ablation）。论文建立在 Borobia et al. (2026) 提出的 SAE feature survival 审计框架之上，把该工具从剪枝迁移到量化问题。
- **依赖的外部资产：** Pythia-70M-deduped + 公开 SAE、Gemma-2-2B + Gemma Scope layer-12 残差流 SAE（Lieberum et al., 2024）——均可在公开 checkpoint 中复现。
- **可复现性：** 论文明确"release our pipeline and per-feature results"，但当前 arXiv v2 文本未给出 GitHub 链接，可复现性需要在后续版本或仓库中确认。

## Critical Assessment

- **新颖性：** 思路直接，结论反直觉且具有部署意义。SAE feature survival 在剪枝上的同类工作（Borobia et al., 2026）已存在；本文主要的"方法学贡献"是把同一个审计框架从剪枝迁移到量化，并首次明确报告"perplexity 可改善而特征被破坏"的 dissociation。
- **严谨性：**
  - 仅用 round-to-nearest（RTN）一种量化方案，未涉及 GPTQ、AWQ、SmoothQuant 等更"温和"的 PTQ 方法；Gemma 2B 仅有层 12 的 SAE 读出点。
  - 仅覆盖 perplexity 一种行为指标，未做下游任务或安全审计；作者在 Limitations 中也明确说明这一点。
  - 特征度量基于 frozen SAE 相关，对 SAE 训练本身的偏差不敏感（这是优点），但也意味着若换一个 SAE 字典，生存阈值会随之漂移——结论对 SAE 实现选择有一定依赖。
  - 切分阈值 $t_s=0.9, t_d=0.5$ 是"常用但约定俗成"的选择，附录 B 显示阈值上调至 0.95 时 Pythia INT6 survival 跌至 44.4%，量级可观，结论对阈值选择仍较敏感。
- **可复现性：** 模型、SAE、token 集均公开；阈值与扫描范围明确；附录列出 token 预算与稳健性 ablation；但缺少仓库/代码链接，单作者 + 数据规模（Pythia 200k tokens、Gemma 500k tokens）下 GPU 时间并非瓶颈。
- **显著性：** 对"在量化模型上做 interpretability"这条研究路径，是一篇方法论上的"刹车"——提示研究者需要在量化后再做一次特征层级 audit，而非直接迁移 FP16 上的发现。
- **红旗：** 单作者 + 集中风险——若 SAE 读出层、token 选择或 SAE 字典存在偏差，论文结论可能被高估；INT7 改善 perplexity 但破坏特征的解释是"轻度正则化"，作者明确说"we do not establish the mechanism"，需要更多因果验证。

## Strengths

- 问题设置干净：固定 SAE、token 集、读出层，仅改变权重——这种"控制变量"式协议让特征度量几乎只受压缩影响。
- 两个数量级跨度（70M 与 2B）的模型都给出定性一致的曲线，跨规模结论稳健。
- 行为度量（perplexity）与特征度量（$c_j$）协议解耦，避免评估协议污染特征读数；并通过滑动窗口验证 perplexity 翻转但特征度量不变。
- 预测模型 AUC 0.92–0.97、跨方法 Spearman 0.98——具有很强的判别性，论证"fragility is structured"。

## Limitations

- 仅 RTN 量化一种，未覆盖 GPTQ / AWQ / SmoothQuant / mixed-precision 等更现实的部署方案。
- 仅 Pythia-70M 与 Gemma-2-2B 两个模型族，SAE 也仅来自公开 residual-stream SAEs；结论对其它 SAE 字典、其它层、其它模型族未经验证。
- 仅以 WikiText-2-raw 上的 perplexity 作为行为指标，未覆盖下游任务、指令遵循、安全审计、refusal 行为等部署侧真正关心的指标。
- 阈值 $t_s=0.9, t_d=0.5$ 是约定选择，附录 B 显示阈值对生存率影响显著；论文未讨论阈值校准问题。
- 论文未给出因果层面的验证：特征 $c_j$ 下降是否真的会导致 steering / auditing 失败？作者明确将"causal validation"列为下一步。

## Relevance

- 这篇论文与"机制可解释性 × 模型部署"的交叉点高度相关：研究 SAE feature steering / auditing 的同行需要意识到，FP16 上提取的特征在量化后未必"是同一回事"。
- 对关注 LLM 部署（量化、剪枝、压缩）的研究者：提供了"行为等价 ≠ 特征等价"的实证案例，可作为压缩后 audit 的方法学参考。
- 对关注 LLM interpretability 的研究者：把 SAE feature survival 这一审计范式从剪枝推广到量化，是后续工作的直接方法学工具。

## Questions for Further Exploration

- **更现实的量化器：** 在 GPTQ、AWQ、SmoothQuant、QLora 等方案下，INT7 / INT6 仍会出现"perplexity 改善但特征被破坏"的 dissociation 吗？
- **多层与多 SAE：** 若用 Gemma Scope 全部层的 SAE 做同一切片分析，"特征崩坍先发生在哪些层？"——是否与早期层鲁棒、晚期层脆弱的常见假设一致？
- **因果验证：** 用 SAE feature steering 在量化模型上是否仍能复现 FP16 的效果？ablation 是否还能抑制目标行为？失败案例是否能用 $c_j$ 提前预测？
- **预测器的边界：** 用全精度统计预测特征生存率在"显著 task shift"或"领域微调"之后是否仍然成立？例如对量化模型做 LoRA fine-tune 后，哪些 FP16 统计最值得信赖？
- **其它 SAE 字典与训练配方的依赖：** 不同随机种子、不同 sparsity 的 SAE（Paulo & Belrose 2025; Chanin & Garriga-Alonso 2025）是否会让生存阈值发生系统性漂移？是否需要"SAE ensemble"作为更稳健的测量基底？
- **下游任务与安全审计：** 在 truthful / refusal / bias benchmark 上，把同一行为指标换成任务级 accuracy，perplexity-vs-feature 的 dissociation 是否仍然存在？
- **结合量化感知训练（QAT）：** 若在 QAT 下让 SAE feature survival 与 perplexity 同时作为约束，能否在保持特征保真度的同时获得更紧的压缩？
