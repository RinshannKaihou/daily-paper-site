Why this matters · 为什么重要
Exposing graph structure as an agent-callable interface — not an opaque retriever bundle — turns skill selection into a routing decision the LLM owns, with the biggest payoff on weaker backbones.

01Methodology · 方法

A typed directed graph with a four-call agent interface

Skills C = {s₁…sₙ} become nodes; each edge is one of five operationally specified types: depends_on, specializes, composes_with, similar_to, conflicts_with. Two constraints are baked in: conflicts_with is excluded from traversal, and the depends_on/specializes backbone is acyclic — that's the DAG.

  • Cold-start (Alg 1): each skill gets e_self ("what it does") and a HyDE-style e_needs ("what it requires"); pairs above an adaptive cosine threshold are bucket-classified by gpt-5-nano. conflicts_with is reserved for the online phase.
  • Online loop (Alg 2): per episode, the agent calls search(q, K, D) → returns three independent evidence channels (matches, neighbors, conflicts). Edits are propose-edge (dry-run preview) → edit-edge (commit, gated by 3 invariants).
  • Three commit invariants: acyclicity on the backbone, non-contradiction (no positive edge + conflict on the same pair), append-only reversibility keyed by task_id.
  • Set-monotone guarantee: an accepted edge can only enlarge the retrieved set (matches ∪ BFS neighbors), never shrink it — a recall property, not a context-budget one.

03Strengths · 优势

Clean architectural inversion, honest failure analysis

  • Falsifiable claim. "Graph-as-interface" predicts a specific empirical signature (matches cosine vs. GoS PPR) — SkillDAG wins it cleanly.
  • Rare structural result. Set-monotone online-edit guarantee is provable by construction; unusual in self-evolving-agent literature.
  • Robust at scale. 10× pool expansion with only −3.5 Ret@5 — the headline durability result.
  • Interpretable cross-backbone pattern. Large gains on weaker models, near-ties on stronger ones, consistent with a routing-aid reading.
  • Bounded failure analysis. 11/30 pilot failures are split into typed causes; </think> leakage is flagged as a model artifact rather than buried.
  • Open code & data with concrete configs (text-embedding-3-large, gpt-5-nano, K=5, D=2).
04Limitations · 局限

Self-evolution is gated by reasoning alone

  • Unpaired baselines. Vanilla / Vector / GoS rows are quoted from Liu et al. 2026 — no seed-matched rerun. Headline deltas are weaker than they look under paired comparison.
  • No semantic confidence threshold. Only structural invariants guard edits; a single noisy execution observation can commit an edge. Long-horizon self-evolution is explicitly left as future work.
  • Anonymized backbones. "MiniMax-M2.7" / "MiniMax Anthropic-compatible endpoint" read as placeholders — cross-backbone generalization is hard to verify independently.
  • Recall-only guarantee. Set-monotonicity is proven; precision under noisy or adversarially authored skill bodies is not measured.
  • Two benchmarks. ALFWorld + SkillsBench only; WebArena, OSWorld, AppWorld, τ-bench are untested.
  • Cold-start cost. ~200 gpt-5-nano calls at scale 1000; per-call variance undocumented.
05Questions for further exploration · 待探索问题
  1. Where is the routing-aid threshold? Below what reasoning capability does SkillDAG pay off most — and is it predictable from MMLU / AgentBench?
  2. Precision under noise. Set-monotone is recall-only; what happens with misleading skill descriptions or near-duplicates with subtly different scopes?
  3. Can the edit protocol be tightened? Multi-observation voting or schema-validated reasons might preserve set-monotonicity while reducing bad-edit rate.
  4. Head-to-head with GraphRAG / LightRAG / GeAR / HippoRAG. The paper positions only against GoS — a broader structured-RAG comparison would isolate interface-design gain from skill-graph-specific gain.
  5. Interaction with prompt compression. If AgentOCR-style rendering is applied to search, do typed neighbors and conflicts collapse into one best-evidence field?
  6. Does the two-view HyDE cold-start transfer to other typed graphs (tool ontologies, KG memory for embodied agents)?
  7. How does the +27 edit count scale with library size and episode count — does precision plateau, drift, or improve?
