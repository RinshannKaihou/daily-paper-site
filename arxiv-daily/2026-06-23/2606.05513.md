> **一句话总结：** EpiEvolve 把一个冻结权重的 Qwen3-14B 疫情 LLM 用"分层情景记忆 + 反思 + 战略规则蒸馏 + 漂移检测"包成一个自演化智能体，在 50 个州、5 个新冠变异期、81 周的顺序流式预测中将平均准确率从 0.561 提到 0.629，并把每次变异切换后的恢复滞后从 5 周缩短到 2 周。

> **TL;DR:** EpiEvolve is a streaming epidemic-forecasting agent that wraps a frozen Qwen3-14B-Base backbone (fine-tuned only on a warm-start window) and adapts at deployment through (i) hierarchical episodic memory at state / HHS-region / national scopes, (ii) post-hoc reflection on delayed labels, (iii) predicate-form strategic rule distillation, and (iv) a calendar+statistical drift detector. On 81 weekly rounds over five COVID variant regimes (Delta → Omicron BA.1/BA.2/BA.5/BQ.1) covering all 50 U.S. states, it raises average trend-classification accuracy from 0.561 (static PandemicLLM) to 0.629 and cuts post-shift recovery lag from 5 to 2 weeks, beating the CDC ensemble (0.325), streaming fine-tuning (0.566), Claude-ICL (0.557), retrieval-only (0.591), and reflection-only (0.604) variants. Ablations isolate reflection (−0.025), strategic memory (−0.025), regime-aware retrieval (−0.025), and the drift detector (+1 week lag) as the four contributors.

## 核心贡献

- **流式自演化疫情预测智能体**：提出 EpiEvolve，把一个预训练-微调后冻结权重的 LLM 预测器包装成智能体，所有部署期适应通过非参数的记忆/检索/规则通道完成，避免昂贵的在线梯度更新。
- **带延迟反馈的流式评估协议**：将新冠住院趋势预测建模为按周滚动、严格按时间顺序、每周批量出预测后才能见到标签的流式任务；明确禁止同周内跨州"偷看"未来，并首次引入"恢复滞后"（recovery lag）作为分布切换后的可测量指标。
- **分层情景记忆 + 反思 + 战略蒸馏**：设计 state / HHS-region / national 三级情景存储，按"漂移感知相似度"做 top-N 检索；预测后用 LLM 反思生成一句话教训和候选规则；每 K=4 周与每次漂移事件触发一次规则蒸馏，形成 `<MEMORY>` 和 `<RULES>` 两个提示槽。
- **基于"变异字段 + 滚动误差"的轻量漂移检测器**：不读评测期 regime 标签，只用公开监测数据中的优势变异字段变化与跨州平均序数误差突增两个信号推进 regime 指示器 ρ_t，使检索偏向当前 regime。
- **5 个 regime、50 州、81 周、4050 次预测的实证分析**：在 BA.5 和 BQ.1 这种最 OOD 的 regime 上最大幅领先，并通过组件/层级消融、超参敏感性、per-state 热图与混淆矩阵做了较系统的剖析。

## Key Contributions

- **A frozen-backbone streaming forecasting agent.** EpiEvolve keeps the LLM weights fixed after warm-start fine-tuning and pushes all deployment-time adaptation into non-parametric memory, retrieval, and rule updates. This isolates "self-evolution via context" as the only moving part and matches deployment scenarios where retraining is infeasible.
- **A chronological streaming protocol with delayed feedback.** Forecasts for all 50 states in week *t* are emitted before any label for that week is observed; ground truth is revealed as a batch only after the round closes, preventing within-week leakage. Performance is reported both as average accuracy and as *recovery lag* after regime transitions, the smallest *k* for which rolling accuracy returns to the new regime's steady-state.
- **Hierarchical episodic memory + reflection + strategic rule distillation.** Each forecast outcome is stored as a one-sentence lesson indexed by state, region, and regime; reflection runs every week to write the lesson and queue a candidate rule; every K=4 weeks (and on drift events) a strategic distiller consolidates recurring reflections into predicate-form rules (preconditions → consequent) with Bayesian-style support counts *n* and confidences *c*. Memory and rules populate two structured prompt slots `<MEMORY>` and `<RULES>`.
- **Drift detector from public surveillance + rolling error.** A drift event fires when (a) the dominant-variant field in the observable input *x*<sub>s,t</sub> changes or (b) the weekly cross-region average ordinal error exceeds the warm-start mean by more than τσ standard deviations. The new regime indicator ρ biases retrieval and gates the rule store (obsolete rules are demoted, not deleted).
- **Empirical evidence across five COVID variant regimes.** 81 streaming weeks (2021-06-07 → 2022-12-19), 4,050 state-week forecasts, regime-wise accuracy, recovery curves, ablation tables, hyperparameter sensitivity, per-state HHS-region heatmap, and per-class confusion matrix.

## Methodology

**Backbone.** A Qwen3-14B-Base model fine-tuned on the warm-start period (through 2021-05-31) following the PandemicLLM recipe of Du et al. (2025). Backbone weights are then frozen for all streaming evaluation; adaptation enters the model only through prompt context.

**Inputs.** Each weekly state example *x*<sub>s,t</sub> includes (i) recent hospitalization/case time series, (ii) policy text (e.g. mask mandates), (iii) vaccination %, (iv) dominant-variant text from public surveillance, and (v) regional context.

**Output.** A 5-class ordinal trend label — substantial decreasing / moderate decreasing / stable / moderate increasing / substantial increasing.

**Deployment memory M_t = (E_t, L_t, ρ_t):**
- *E*<sub>t</sub>: episodic store of past forecast outcomes as entries e = (s, t, ϕ<sub>s,t</sub>, ŷ<sub>s,t</sub>, y<sub>s,t</sub>, ℓ<sub>s,t</sub>, ρ<sub>t</sub>), partitioned into state view *E*<sub>t</sub><sup>S</sup>, regional (HHS-region) view *E*<sub>t</sub><sup>R</sup>, and national view *E*<sub>t</sub><sup>N</sup>. Embeddings ϕ are 256-dim from a frozen sentence encoder over a structured rendering of *x*.
- *L*<sub>t</sub>: strategic memory of predicate-form rules λ = (preconds, consequent, c, n, ρ<sub>λ</sub>). Confidence updates as c ← (n·c + 𝟙[consequent matched truth]) / (n+1).
- ρ<sub>t</sub>: regime indicator advanced by the drift detector.

**Retrieval.** Top-N entries (N=8 default) under a regime-aware similarity score
score(e, s, t) = cos(ϕ<sub>s,t</sub>, ϕ<sub>e</sub>) · w(ρ<sub>t</sub>, ρ<sub>e</sub>),
with w = 1 if regimes match and α = 0.5 otherwise. The selected entries render into the `<MEMORY>` prompt slot.

**Reflection.** After delayed labels arrive, an LLM call produces a one-sentence lesson naming (1) features that supported the forecast, (2) features that should have shifted it toward the truth, (3) the regime-conditioned cue. The lesson is appended to *E*<sub>t</sub>; a candidate rule is queued.

**Strategic distillation.** Every K=4 weeks (default) and on every drift event, an LLM pass proposes 0–3 new rules supported by ≥3 entries in a sliding window, each differing from existing rules by ≥1 predicate and with 1–4 precondition predicates.

**Drift detector.** Calendar trigger: dominant-variant field changes. Statistical trigger: weekly cross-region mean ordinal error > warm-start mean + τσ·SD (τσ = 2 default). Neither trigger reads evaluation-regime labels.

**Streaming agent loop.** Each week: assemble *r*<sub>s,t</sub> from *E*<sub>t</sub> and *L*<sub>t</sub>; emit ŷ<sub>s,t</sub> = f<sub>θ</sub>(x<sub>s,t</sub>, r<sub>s,t</sub>) for all 50 states *before* any label is observed; on batched label arrival, run reflection, queue candidates, run distillation if scheduled, run drift detector → update M<sub>t+1</sub>.

**Baselines.** PandemicLLM (no streaming updates), streaming fine-tune (periodic gradient updates), Claude-ICL (Claude in-context forecaster), CDC COVIDhub-ensemble mapped to the 5-class target, retrieval-only and reflection-only variants of EpiEvolve.

## Results & Highlights

**Table 1 — main streaming forecasting results** (regime-wise accuracy + average + recovery lag):

| Method | Adaptation | Delta/early | BA.1 | BA.2 | BA.5 | BQ.1 | Avg. | Recovery lag |
|---|---|---|---|---|---|---|---|---|
| CDC ensemble | None (external) | 0.393 | 0.458 | 0.469 | 0.139 | 0.165 | 0.325 | N/A |
| PandemicLLM | None | 0.587 | 0.713 | 0.642 | 0.431 | 0.456 | 0.561 | 5 |
| Streaming fine-tune | Gradient update | 0.587 | 0.700 | 0.625 | 0.460 | 0.490 | 0.566 | 5 |
| Claude-ICL | In-context only | 0.563 | 0.624 | 0.561 | 0.479 | 0.447 | 0.557 | 5 |
| Retrieval-only | Episodic retrieval | 0.602 | 0.718 | 0.663 | 0.488 | 0.512 | 0.591 | 4 |
| Reflection + episodic | Reflection memory | 0.611 | 0.728 | 0.672 | 0.523 | 0.541 | 0.604 | 3 |
| **EpiEvolve** | **Reflection + strategic memory** | **0.624** | **0.751** | **0.687** | **0.563** | **0.582** | **0.629** | **2** |

- **+0.068 average accuracy** over the static PandemicLLM backbone (0.629 vs 0.561); **+0.304 over the CDC ensemble** (0.629 vs 0.325).
- **Recovery lag cut from 5 → 2 weeks** (avg across the four variant boundaries). The static backbone never recovers within the BA.5 or BQ.1 windows.
- **Largest margins in the most OOD regimes** (BA.5: 0.563 vs 0.431; BQ.1: 0.582 vs 0.456).
- **Streaming fine-tuning barely helps** under the same chronological constraint (0.566), reinforcing the claim that prompt/memory adaptation can match or beat gradient updates for this task while leaving weights immutable.

**Table 2 — ablations** (component + memory-scope ablations, all using the same backbone and delayed-feedback protocol):

| Configuration | Acc ↑ | MSE ↓ | Lag ↓ |
|---|---|---|---|
| EpiEvolve (full) | 0.629 | 0.65 | 2 |
| − strategic memory | 0.604 | 0.71 | 3 |
| − reflection (retrieval only) | 0.591 | 0.79 | 4 |
| − drift detector | 0.612 | 0.68 | 3 |
| − regime-aware retrieval | 0.604 | 0.73 | 3 |
| state only | 0.598 | 0.74 | 4 |
| regional only | 0.602 | 0.70 | 3 |
| national only | 0.589 | 0.78 | 4 |

Reflection accounts for ~2/3 of the joint gain; strategic rules contribute ~1/3. Single-tier retrieval costs 0.028–0.041 accuracy. The drift detector mainly affects recovery lag (+1 week when removed).

**Hyperparameter sensitivity (Table 3):** N ∈ {4, 8, 12}, K ∈ {2, 4, 8}, τσ ∈ {1, 2, 3} — average accuracy stays in 0.612–0.629, lag varies by ≤1 week; defaults hit the best trade-off.

**Per-state heatmap (Figure 5):** Gains concentrate in BA.5 and BQ.1 rows (per-state Δ ≈ +0.13 avg) and correlate within HHS regions, consistent with the regional memory tier pooling cross-state evidence that shares federal coordination.

**Confusion matrix (Figure 6):** Diagonal-dominant; off-diagonal mass on adjacent ordinal classes; per-class recall ≥ 0.55 on every class (stable 0.75, transitions 0.55), so the gain is broad, not concentrated on the majority class.

**Adaptation curves (Figure 2):** Rolling 4-week accuracy — static PandemicLLM collapses at BA.5 and stays depressed; EpiEvolve dips less at every boundary and returns to its new-regime steady-state within 2 weeks.

**Qualitative case (Figure 4, Florida 2022-06-13, second week of BA.5):** With empty memory/rules the backbone predicts "stable" (wrong); with hierarchical retrieval surfacing one FL-cross-regime case, one HHS-4 neighbor (GA, 2022-06-06), and one national analogue plus one matched strategic rule ("variant text contains 'emerging' AND ≥2 stable weeks AND vaccination <75% → moderate increasing", c=0.71, n=14), EpiEvolve predicts "moderate increasing" (p=0.62), matching truth.

## Critical Assessment

**Honest evaluation — strengths and weaknesses I want to flag:**

- **Setup integrity is genuinely strong.** The chronological protocol (weekly batch forecast → label reveal → memory update → next week) is the right formalization for delayed-feedback forecasting and is more rigorous than most "online" LLM-agent evaluations, which often implicitly allow within-window leakage. Recovery lag as a metric is a useful addition beyond mean accuracy.
- **The empirical delta is real but bounded.** 0.561 → 0.629 is +0.068 average accuracy, which is meaningful but not dramatic. Crucially, the +0.068 breaks down as ~+0.025 from reflection, ~+0.025 from strategic memory, ~+0.025 from regime-aware retrieval, and a 1-week lag improvement from the drift detector — none of the individual contributions is large, and the system relies on all four.
- **Single dataset, single geography, single disease.** All results are on U.S. COVID-19 hospitalization trends across 50 states. The paper does not test on influenza, RSV, dengue, or non-U.S. surveillance systems. Generalization is an open question.
- **Hard-coded regime boundaries and labeled "regimes" are partly an evaluation convenience.** The five variants (Delta, BA.1, BA.2, BA.5, BQ.1) are *post-hoc* partitions used only for reporting. The drift detector itself reads only the dominant-variant field and rolling error — which is good design — but readers should be careful: the agent never sees these labels, yet the evaluation regime boundaries are aligned with the same biological events the agent is detecting, which makes recovery-lag comparisons cleaner than they would be on truly novel regimes.
- **Trend classification is a coarse target.** 5-class ordinal hospitalization-trend labels are a much easier problem than forecasting case counts, hospitalizations, or Rt. The paper is honest about this in the Limitations section but it caps the work's operational relevance.
- **Reflection text and rule quality are LLM-dependent.** Strategic distillation relies on the LLM producing interpretable predicate-form rules, which is sensitive to prompt design, retrieval budget N, and cadence K. The robustness sweep shows stability across reasonable settings, but there is no human audit of the rules produced.
- **No comparison against actual production forecasters used by CDC/Forecast Hub teams.** The CDC ensemble comparison uses the *archived* COVIDhub-ensemble binned to the 5-class target, which is not what teams actually optimize for. A head-to-head on the original quantile MAE would be more informative.
- **Streaming fine-tuning under the same protocol barely moves the needle** (0.566 vs 0.561). This is an honest and useful negative result, but it also raises a question: is the win mostly because *any* adaptation helps, and the specific mechanism (memory) is interchangeable with a more careful fine-tuning schedule that wasn't tried?
- **No statistical significance.** No error bars, no seeds reported, no significance tests across regime boundaries. The clean monotone ablation table is suggestive but not statistically defended.

## Strengths

- **Conceptually clean separation** between frozen backbone and adaptive memory; isolates "self-evolution via context" cleanly.
- **Strong streaming protocol** with delayed feedback, no within-week leakage, and a recovery-lag metric tailored to regime shifts.
- **Ablation discipline** — every component and every memory tier is removed, not just one at a time, and the reflection/strategic-memory contributions are arithmetically decomposed.
- **Beats CDC ensemble, streaming fine-tuning, and a closed-source Claude-ICL baseline** under the same constraint.
- **Qualitative case study** (Figure 4) makes the mechanism legible to a non-specialist reader.
- **Per-state HHS-region heatmap** provides a sanity check that the regional memory tier is doing what it claims.
- **Hyperparameter robustness** is documented (Table 3).

## Limitations

- **Single dataset**: U.S. COVID-19 hospitalization trends, 50 states, 81 weeks; no cross-disease or cross-country validation.
- **Coarse target**: 5-class ordinal trend rather than case counts, hospitalizations, or Rt.
- **Hard regime boundaries** are post-hoc partitions; biological regime changes are continuous and sometimes ambiguous.
- **Prompt and rule sensitivity**: reflection text, retrieval budget N, and distillation cadence K all affect performance; no human audit of distilled rules.
- **Closed-source baseline caveats**: Claude-ICL introduces cost and reproducibility concerns; CDC ensemble is binned to the 5-class target rather than compared on its native quantile metric.
- **No statistical significance**: no error bars, no seeds, no significance tests.
- **LLM dependency**: the entire reflection + distillation pipeline depends on the backbone LLM producing usable one-sentence lessons and predicate-form rules, which constrains model choice.
- **Memory bloat**: while entries are salience-pruned, long streams could still dilute retrieval quality; the paper does not report behavior on horizons longer than 81 weeks.

## Relevance

与本仓库关注的研究方向高度相关：EpiEvolve 把"冻结 backbone + 自演化记忆/规则 + 流式漂移检测"这一范式系统化地迁移到了公共卫生预测场景，对关心 LLM agent 在真实部署环境（含分布漂移、延迟反馈、不可再训练）下持续可用性的研究者有直接借鉴意义；对 streaming evaluation、concept drift、retrieval-augmented agents 三个交叉方向也是一份可复用的实验模板。

## Questions for Further Exploration

- **Cross-disease transfer.** Does the same architecture — frozen backbone + hierarchical episodic memory + predicate rules + drift detector — transfer to influenza, RSV, dengue, or multi-pathogen surveillance without retuning?
- **Continuous forecasting targets.** Can the agent be extended from 5-class trend labels to actual hospitalization or case counts while preserving the chronological protocol and recovery-lag gains?
- **Statistical significance.** What do error bars look like across random seeds of the backbone fine-tune, the reflection LLM, and the drift threshold τσ?
- **Rule auditing.** A small qualitative study (a few hundred distilled rules, hand-labeled by epidemiologists) would tell us whether the predicate-form rules are clinically meaningful or LLM-shaped artifacts.
- **Memory scope design.** Could the hierarchical state/regional/national partition be replaced by a learned partition (e.g. dynamic clustering of states by epidemic trajectory) rather than the fixed HHS-region structure?
- **Closed-source vs open-source backbone.** Is the gain robust across backbone families (Llama, Mistral, smaller Qwen variants), or does it depend on Qwen3-14B's instruction-following/reasoning strength?
- **Comparison to production forecasters.** A head-to-head against the original CDC ensemble on quantile MAE, against individual Forecast Hub teams, and against modern time-series foundation models (Chronos, Time-LLM, EARTH) would clarify whether "self-evolving memory" beats dedicated time-series LLMs.
- **Longer horizons.** How does EpiEvolve behave on a 3+ year stream where the memory store grows by an order of magnitude? Does salience-based pruning preserve retrieval quality, or do we need active forgetting?
- **Adversarial / non-stationary regimes that are not biologically labeled.** E.g. policy-only regime shifts (mask mandates, school closures) without a dominant-variant change — does the statistical trigger catch these, or does the agent silently fail?
- **Cost and latency.** Each week requires N+1 LLM calls (1 forecast + 1 reflection per state × 50 + occasional distillation). What's the wall-clock cost per week, and can the reflection/distillation calls be batched or amortized?
