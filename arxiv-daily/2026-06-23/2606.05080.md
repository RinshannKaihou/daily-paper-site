# AutoLab: Can Frontier Models Solve Long-Horizon Auto Research and Engineering Tasks?

> **一句话总结：** AutoLab 提出 36 个跨 4 个领域（系统优化、谜题挑战、模型开发、CUDA kernel）的超长时程闭环优化基准，对 17 个前沿模型在 2–12 小时预算下进行评估，发现 Claude Opus 4.6 以 Avg@3 0.68 显著领先，且决定成败的关键并非一次性代码质量，而是**持续迭代、反馈吸收与时间感知**。

> **TL;DR:** AutoLab is an ultra long-horizon closed-loop optimization benchmark consisting of 36 expert-curated tasks across system optimization, puzzle & challenge, model development, and CUDA kernel optimization, where agents must iteratively improve a deliberately suboptimal baseline within a strict wall-clock budget (2–12 hours). Evaluating 17 state-of-the-art models under a unified Harbor + terminus-2 harness, the authors show that claude-opus-4.6 leads decisively (Avg@3 0.68 vs. 0.50 for the runner-up), and that the dominant predictor of final performance is *persistence* — repeatedly benchmarking, editing, and incorporating empirical feedback — rather than one-shot solution quality. Manual inspection of 302 zero-score rollouts reveals a widespread *lack of time awareness*: strong models either terminate prematurely (gpt-5.4, grok-4-20) or burn the budget on excessive reasoning (deepseek-v4-pro). The authors further show that **harness design alone can shift per-model scores by up to 0.43**, demonstrating that long-horizon capability cannot be measured independently of agent infrastructure.

## 核心贡献

- **超长时程闭环优化基准 AutoLab**：36 个由资深研究者贡献的可执行任务，分布在系统优化（15）、谜题挑战（10）、模型开发（7）、CUDA kernel（4）四类，要求智能体在严格 2–12 小时预算内持续迭代改进一个**正确但有意调差的基线**。
- **持续且可比较的评分协议**：采用对数拉伸 / 线性插值两种锚定打分方案（baseline=0，reference=0.5–1.0），避免二元 pass/fail 带来的饱和与饱和过快问题。
- **抗 reward hacking 的多层防护**：sealed 验证器、正确性门控、SHA 钉死不可变文件、adversarial 审计智能体在构建期探测捷径、运行时持续监控并打补丁。
- **17 个前沿模型的大规模评估**：消耗 2,544 壁钟小时与 86 亿 token，使用统一的 Harbor + terminus-2 框架实现 apples-to-apples 对比，涵盖 Claude Opus 4.6、Gemini 3.1 Pro、GPT 5.4、Grok 4-20 与 13 个开源/小尺寸变体。
- **轨迹层面诊断分析**：手动分析 302 个零分 rollout 的失败模式，揭示*时间感知缺失*这一跨模型系统性短板，并证明**持续迭代**比初始解质量更能预测最终成绩。
- **Harness 消融实证**：在 terminus-2、pi-mono、自定义 mini-swe-agent\* 三个框架上对 4 个模型重跑 25 任务，发现同一模型的均值分最大可移动 Δ=0.43，相对排名出现非传递翻转。

## Key Contributions

- A high-quality, expert-curated benchmark of **36 executable ultra long-horizon tasks** spanning four diverse domains, each starting from a correct-but-suboptimal baseline.
- A **continuous, anchored scoring protocol** (log-stretch for performance tasks, linear for bounded-quality tasks) that rewards partial progress and resists saturation.
- A **multi-layered anti-hacking protocol** (sealed verifiers, correctness gates, SHA-pinned immutable files, adversarial auditor agent) suitable for performance-style benchmarks.
- A large-scale evaluation of **17 state-of-the-art proprietary and open-weight models** under a single fixed harness, consuming 2,544 wall-clock hours and 8.60 B tokens.
- Detailed **trajectory analysis** of 302 zero-score rollouts, identifying *premature termination* and *budget exhaustion* as the two dominant failure modes caused by lack of time awareness.
- Empirical evidence that **persistence in iteration, not one-shot solution quality, is the strongest predictor of long-horizon success** (claude-opus-4.6 ran a median of 57 steps vs. 12 for gemini-3.1-pro).
- A **harness ablation** showing that agent infrastructure shifts the same model's score by up to 0.43 and can flip relative rankings — implying that long-horizon capability is inseparable from harness design.

## Methodology

**Task formulation.** Each task packages (i) a natural-language instruction, (ii) a containerized sandbox (CPU or single H100/L40S) with a working but unoptimized baseline codebase and a local evaluation script, (iii) a sealed held-out verifier, (iv) a hidden human reference solution that defines the target metric, and (v) a wall-clock budget of 2–12 h. The agent may freely read/edit code, execute, profile, and invoke the local evaluator, but only the verifier's output on held-out inputs is graded.

**Scoring.** Two anchored schemes normalize raw metrics $m(x)$ to $[0, 1]$:
- *Log-stretch* for performance tasks: $s(x) = \mathrm{clip}\big(\tfrac{1}{2}\cdot \frac{\log(m_B/m(x))}{\log(m_B/m_R)}, 0, 1\big)$, gated by a minimum-improvement threshold.
- *Linear* for bounded-quality tasks: $s(x) = \mathrm{clip}\big(\frac{m_B - m(x)}{m_B - m_R}, 0, 1\big)$.

In both cases $s=0$ at the baseline and $s$ approaches 1.0 near the reference. A *Dominance* score (pairwise tournament win rate) is also reported.

**Quality control.** Each task was audited against four criteria — validity, solvability, integrity, and measurement stability — by ≥2 independent experts and a format-audit agent; failed tasks were revised or rejected. An adversarial agent was run during construction to discover shortcuts.

**Evaluation harness.** A unified pipeline uses the Harbor framework with the `terminus-2` agent for all main results, 3 rollouts per (model, task), reporting Avg@3, Best@3, and Dominance. CPU tasks run in local Docker (Ryzen 9 9950X, 64 GB); GPU tasks run on Modal cloud sandboxes.

**Failure-mode coding.** All 302 zero-score rollouts (across 11 main-set models) were manually classified into four mutually exclusive categories: Timeout / Context Exhaustion, Capability Gap, Instruction Violation, and Others.

**Harness ablation.** Four models (mimo-v2.5, gpt-5.4, deepseek-v4-flash, kimi-k2.6) were re-evaluated on a 25-task subset under `pi-mono` and a custom *mini-swe-agent\** prompt that aggressively encourages persistent performance engineering.

## Results & Highlights

- **Overall leaderboard (main set, Avg@3 / Best@3 / Dominance):**
  - claude-opus-4.6 — **0.68 / 0.76 / 0.93** (leads in all four categories)
  - gemini-3.1-pro — 0.50 / 0.59 / 0.62
  - kimi-k2.6 — 0.46 / 0.60 / 0.62
  - mimo-v2.5-pro — 0.45 / 0.58 / 0.53
  - glm-5 — 0.43 / 0.55 / 0.57
  - deepseek-v4-pro — 0.38 / 0.51 / 0.47
  - gpt-5.4 — 0.36 / 0.53 / 0.39
  - grok-4-20 — 0.35 / 0.44 / 0.42
  - hunyuan-3-preview — 0.31 / 0.45 / 0.34
  - minimax-m2.7 — 0.27 / 0.43 / 0.28
  - qwen-3.6-plus — 0.27 / 0.39 / 0.32
- **Per-category winners** (CUDA / Model Dev / Puzzle / System Opt): Claude Opus 4.6 sweeps all four; runner-ups rotate (gemini-3.1-pro on puzzles, kimi-k2.6 on system opt, glm-5 / mimo-v2.5-pro on model dev / CUDA).
- **Persistence → success.** The median trajectory length of claude-opus-4.6 is 57 steps vs. 12 for gemini-3.1-pro, and overall score correlates strongly with both agent step count and wall-clock time spent.
- **Flash-Attention case study.** All models start from a 750 ms baseline; claude-opus-4.6 reaches 18 ms (42.4× speedup, **exceeding the reference of 100 ms**) in ~40 min and 44 iterations. kimi-k2.6, gemini-3.1-pro and glm-5 plateau at 50–80 ms; grok-4-20 barely runs the evaluator at all.
- **Failure-mode distribution** (302 zero-score rollouts): *timeout/context-exhaustion* dominates for deepseek-v4-pro (20/30) and hunyuan-3-preview (21/30); *capability gap* dominates for grok-4-20 (19/30) and qwen-3.6-plus (19/30); *instruction violations* are concentrated in gemini-3.1-pro and glm-5 (especially `ntt_butterfly_cuda`).
- **Harness ablation.** Per-model mean shifts reach 0.43 (kimi-k2.6: 0.21 on pi-mono → 0.64 on mini-swe-agent\*). pi-mono favors gpt-5.4; mini-swe-agent\* favors kimi-k2.6; relative rankings are non-transitive across harnesses. Per-trial cost varies >5× for the same model across harnesses.
- **Generational gains within families** (Appendix C) are modest and uneven — e.g., mimo-v2-pro→v2.5→v2.5-pro shows flat or mixed movement, suggesting that closed-loop iteration is not a free byproduct of pre-training scale.

## Critical Assessment

- **Novelty.** The *category* of long-horizon benchmarks is no longer new (MLE-Bench, RE-Bench, KernelBench, Frontier-Eng, AIRS-Bench, PostTrainBench all exist), and the authors acknowledge this lineage. The specific contributions that are new: (i) simultaneous coverage of four heterogeneous domains under one calibrated scoring protocol, (ii) the explicit shift from *patch-style grading* to *trajectory-grounded analysis* of 302 zero-score rollouts, and (iii) the rigorous harness-ablation study quantifying that framework-induced variance is on par with model-induced variance.
- **Rigor.** Strong on engineering rigor: sealed verifiers, SHA-pinned files, adversarial auditors, multi-expert review, anchored scoring with explicit gates, three rollouts per (model, task) with Avg@3/Best@3/Dominance, and disclosed hardware (Ryzen 9 9950X / H100 / L40S). The harness ablation is the most important control the field has seen for "long-horizon agent" claims. The 302-rollout manual coding is unusual and valuable, but the inter-rater reliability of those four categories is not reported.
- **Reproducibility.** Excellent: open-source benchmark + harness + task artifacts (GitHub `autolabhq/autolab`, website `autolab.moe`), fixed terminus-2 harness, sandbox specs disclosed, all 17 models and their API providers listed. Wall-clock cost (2,544 h + 8.6 B tokens) is documented, giving a realistic budget for replication.
- **Significance.** High. The central finding — that persistence and time-awareness, not raw coding ability, dominate long-horizon success — reframes a debate that is currently dominated by "is X model smarter than Y" leaderboards. Claude Opus 4.6's 0.18-point gap over the next-best model is the largest capability separation in this benchmark family to date.
- **Red flags / caveats.**
  - Only 36 tasks is a small absolute count; categories are imbalanced (15 system-opt vs. 4 CUDA), so CUDA results carry high variance.
  - GPU tasks run on *cloud* sandboxes (Modal H100/L40S) while CPU tasks run on a *local* workstation — the wall-clock cost figures mix these two regimes, which complicates direct cost comparison.
  - "Sustained iteration matters most" is partly tautological: the benchmark *rewards* models that iterate more, by construction (anchored scoring, multi-hour budget). The finding is genuine but should not be over-generalized to settings where iteration is genuinely cheap.
  - The 0.43 harness-induced swing is presented as a warning, but it is also a confession: with only 25 tasks and a custom prompt in one harness, results are noisier than the headline 0.68 vs. 0.50 number suggests.
  - The minimum-improvement gate plus sealed verifier reduce, but do not eliminate, reward-hacking surface; closed-weight model behavior on adversarial audits is not disclosed.
  - Several "open-weight" model entries in the main set (deepseek-v4-pro, mimo-v2.5-pro, etc.) reference 2026 release dates that have not actually shipped as of writing — these appear to be preview/research checkpoints accessed through hosted APIs.

## Strengths

- Most comprehensive long-horizon agent benchmark to date in terms of domain coverage (4 categories, 4 languages, 4 task types).
- Truly large-scale (17 models, 2,544 wall-clock hours, 8.6 B tokens) with apples-to-apples protocol.
- Calibrated continuous scoring + anti-hacking machinery is the most thorough in the literature.
- The 302-rollout failure-mode coding is unusually deep and reveals actionable design targets (time-awareness, instruction-following).
- Harness ablation directly addresses a chronic reproducibility problem in agent benchmarks.
- Open-sourcing the full benchmark + harness + task artifacts is a significant community contribution.

## Limitations

- 36 tasks is a small absolute sample; per-category sub-scores (especially 4-task CUDA) have high variance.
- Cost asymmetry: a full reproduction is roughly 2,544 h × GPU time, which is feasible only for well-funded labs.
- Mixed CPU-local / GPU-cloud sandbox regimes make wall-clock and cost numbers hard to compare across tasks.
- The benchmark is biased toward "code-edit-and-rebenchmark" workflows; it does not evaluate open-ended literature search, ideation, or scientific reasoning that lacks an executable verifier.
- Scoring is anchored to human reference solutions whose quality sets an upper bound on measured headroom; for some tasks the "reference" itself may be far from optimal, inflating apparent agent progress.
- The harness ablation uses a custom system-prompt variant (`mini-swe-agent\*`) — fair, but a moving target.
- Anti-hacking is strong but not formally proven; SHA-pinning and sealed verifiers reduce but do not rule out side-channels via measurement scripts.

## Relevance

与你的研究方向高度相关：长时程智能体、闭式实证优化循环是当前 agent 研究的核心范式，而 AutoLab 是首批把"持续迭代 + 时间感知"量化为可测能力维度的工作之一。结果直接提示未来 LLM agent 需要把*持久性 / 时间预算分配*作为一等设计目标，而非仅追求单步代码质量。

## Questions for Further Exploration

- Does the persistence advantage transfer from code/ML tasks to **open-ended scientific ideation** tasks where there is no executable verifier to drive iteration?
- Can a model be *trained* (e.g., via time-budget-conditioned RL) to develop explicit time awareness, and does that close the gap to claude-opus-4.6 without architectural changes?
- How sensitive are the headline rankings to **scaling the benchmark from 36 to ~200 tasks**, especially within the small CUDA sub-domain (n=4)?
- What is the minimum harness that preserves the ranking? The 0.43 harness swing implies we still do not have a "minimal sufficient" agent harness for long-horizon evaluation.
- Could the calibrated scoring framework be extended to **multi-objective** tasks (e.g., latency–accuracy–cost Pareto fronts) to capture richer trade-offs?
- The authors note that smaller open-weight models (deepseek-v4-flash, mimo-v2.5) are competitive — is the AutoLab ranking driven by *base-model reasoning* or by *RLHF/post-training for agentic tool use*? An ablation isolating post-training would clarify.
- How much of the persistent-iteration advantage comes from **longer context windows** vs. **better state-tracking heuristics** in the agent scaffold?
