Key metric
5.9%
Bits-per-element overhead delta between MXFP4 (4.25) and NVFP4 (4.50) — same S1E2M1 element, different block (32 vs 16) and scale (E8M0 vs FP8 E4M3).
4 + 8/32 = 4.25 vs 4 + 8/16 = 4.50
Why this matters

Silent cross-vendor divergences — FP8 E4M3 overflow policy (448.0 vs NaN), MXFP4 block structure — corrupt low-precision ML benchmarks. This is the foundational reference.

Critical edge

The only oracle (ml_dtypes 0.5.4) owns the one documented divergence — the suite cannot independently adjudicate tt-metal vs JAX.

Methodology · registry-filling pipeline

Five steps from schema to SHA-256-anchored bit-exact vectors

01
Catalog schema
83 formats × 13 fields each (bits, exp, mant, bias, inf/NaN/sat policy, max-finite, min-normal, cluster, claim-status). JSON = SSOT.
02
Conformance packs
Hand-crafted JSON per pack: header (E, M, bias, SatMax/OvfInf/OvfNaN, max-finite, SHA-256 self-fingerprint) + vector array.
03
Two-step verify
Round-trip self-check (decode∘encode) + cross-check against ml_dtypes 0.5.4. Every divergence documented, none hidden.
04
φ-anchor
Every pack carries a vector encoding 3.0 via φ² + 1/φ² = 3. Exact in all 6 layouts → single-line cross-pack sanity check.
05
CI + codegen
15 invariants enforced every commit. Jinja2 emits Markdown / JSON / Python / Rust / C / TypeScript from one canonical source.

Standards-track grounding

  • FP8 E4M3 ↔ Binary8p3se — close, but SatMax vs OvfInf & finite-only NaN.
  • MXFP4 element ↔ Binary4p1sf — direct match; block structure orthogonal.
  • GF16 → no P3109 counterpart (no 16-bit φ-anchored format).
  • BF16, FP8 E5M2 → no P3109 match (P3109 focuses on 4/8-bit).
  • E8M0 block → orthogonal (no scale-only format in P3109).

P3109 FLoPS Lean 4 formalization cited but not yet wired into the verification harness — Track 2.

Strengths
  • Concrete, deployable artifacts — 6 JSON packs with full SHA-256, machine-verifiable, on GitHub + PyPI tt-lang-t27.
  • Gap-as-feature framing — earns its keep by exposing divergences that would otherwise be invisible (FP8 E4M3 1000.0 case).
  • Schema discipline — shared row schema, 15 CI invariants on every commit, single source of truth.
  • Honest abs-error norm — no zeroed or suppressed values; overflow/underflow cases reported with their real magnitude.
  • Cross-language codegen — Python dataclasses, C headers, Rust serde, TypeScript enums from one canonical JSON.
Limitations
  • Representation layer only — encode/decode verified; FMA / add / multiply deferred to Track 2 (Q3 2026).
  • Single-oracle bias — ml_dtypes 0.5.4 is the only wired ground truth; P3109 FLoPS, Pychop, libtakum still Track 2.
  • Asymmetric coverage — 6/83 formats have Tier-1 packs; Posit/Unum III, LNS, NF4, BitNet, TF32, FP6 remain cataloged but unverified.
  • NVFP4 not packed — the most important emerging 4-bit format (Blackwell/Rubin) is documented but not Tier-1-packed.
  • Self-citation footprint — GoldenFloat cluster = 22/83 catalog entries and is one of only two LIVE packs.
  • GF16 anchor is a citation fingerprint — SHA-256 covers the UTF-8 identity string, not the numerical round-trip result.
