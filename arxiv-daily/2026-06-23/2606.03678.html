EvoDrive turns the scenario generator itself into an evolving multi-objective frontier.

First LLM-based agentic evolution framework for safety-critical driving. Role-specialized actor-critic + simulator-grounded interface + self-evolving world evaluator + Pareto archive expand the attack-vs-realism frontier instead of collapsing it into one scalar.

Why This Matters

Replaces hand-tuned scalar scenario generators with a self-evolving multi-objective frontier — directly usable for safety-critical validation and downstream RL fine-tuning (collision 0.163 → 0.106).

Critical Edge

All evaluation is simulation-only — the elephant in the room for any safety-critical claim.

Strengths // 03
+
Multi-objective formulation. Non-dominated archive with three complementary gates — refuses scalar weighting.
+
Hard simulator grounding. Interface I_b + validator V(c) block the "overlapping vehicles" exploit common to naive agentic search.
+
Role-specialized agents. Structural / attack / realism / isolation critics + repair agent — auditable, intervenable.
+
Self-evolving evaluator. W_φ becomes a learnable artifact with deterministic acceptance — decoupled from generator drift.
+
Comprehensive eval. 12 generator families, 2 simulators, 2 ego policies, multi-metric, mechanism diagnostics, downstream RL.
Limitations // 04
−
Fixed ego policy. Generator-only evolution; future / unseen π_e not addressed.
−
Simulation-only. No real-world or hardware-in-the-loop; physical plausibility asserted, never measured against logged trajectories.
−
Cost opaque. Wall-clock and API cost per evolution round unreported — reproducibility risk on small budgets.
−
Interface brittleness. Performance hinges on the quality of I_b; a poorly specified interface silently clips the search space.
−
Hand-set hyperparams. ε_R, ε_A, λ_A, λ_R, λ_U, audit threshold — no sensitivity analysis in the main paper.
−
No public code/weights. Project page up; repository, critic prompts, evaluator snapshots not clearly linked.
Questions for Further Exploration // 05
Q01How sensitive is the Pareto-frontier gain to the acceptance-gate hyper-parameters ε_R, ε_A and the audit threshold?
Q02Can W_φ be distilled into a smaller model without losing the 1.42×–1.46× top-k yield advantage? Does it transfer across simulators?
Q03How does EvoDrive interact with online policy improvement — co-evolution of generator and ego policy?
Q04What is the minimum critic set? A leave-one-out ablation would identify load-bearing inductive biases.
Q05Does actor-critic + archive generalize to manipulation, UAV obstacle avoidance, medical RL simulators?
Q06Robustness under distribution shift — US highway → urban Europe, dry → adverse weather?
Q07What if the LLM actor hallucinates a mechanism that compiles but exploits a simulator bug? An exploitability audit.
Q08Can typed memory cards be replaced with vector retrieval over cross-domain driving corpora for zero-shot transfer?
