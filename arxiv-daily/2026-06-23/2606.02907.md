# Linear Probes Detect Task Format, Not Reasoning Mode in Language Model Hidden States

> **一句话总结：** 通过在 Qwen3-14B 上对演绎（LogiQA 2.0）、归纳（ARC-Challenge）和溯因（αNLI）三类推理任务做线性探针与因果干预，作者证明探针所谓的"区分推理模式"几乎完全来自数据集的表面格式差异（来源身份、选项数、回答长度），去除这些混淆后探针精度直接跌到随机水平，模型实际上跨推理类型使用统一的推理策略。

> **TL;DR:** Linear probes of Qwen3-14B hidden states achieve 100% cross-validated accuracy at separating deductive, inductive, and abductive reasoning at layer 32, with cleanly separated manifold geometry (intrinsic dimensionalities 20.6 / 28.5 / 33.6; hull contamination ≤ 1.5%). The authors show this separation is entirely artifactual: regressing out dataset source, option count, and response length collapses probe accuracy to chance. Trace-mode agreement is only 42.5% (vs. 33.3% chance), and causal steering along the centroid-difference direction does not significantly outperform random-direction controls (n=20, p=0.286). The result is a methodological warning — high probe accuracy in multi-source reasoning studies reflects task format, not computational structure, and format deconfounding should be standard practice.

## 核心贡献 / Key Contributions

### 核心贡献（中文）
- **格式混淆分解（Format Confound Decomposition）**：提出一个残差分析流程，用 Ridge 回归把"数据集来源 one-hot + 选项数 + 回答长度"这些格式特征从隐藏状态中线性剔除，再做探针。结果：原始 100% 的探针精度在残差上跌到约 33.5%（三分类随机水平），说明原始可分性完全由格式驱动。
- **追踪-模式一致度分析（Trace-Mode Agreement）**：以"演绎=按已知规则逐步推导、归纳=从模式泛化、溯因=从观察选最佳解释"等行为锚点为模板，把模型生成的推理链用最后一层隐藏状态嵌入并与锚点比 cosine similarity。三类任务整体准确率 86%，但 trace-mode 一致度仅 42.5%（随机为 33.3%），说明模型并不按任务类型切换策略，而是用一套通用方法处理三种推理。
- **带随机基线的因果转向控制（Causal Steering with Random-Direction Controls）**：在 layer 32 上做激活转向，目标方向 $\hat{v}_{s\to t}=(\mu_t-\mu_s)/\|\mu_t-\mu_s\|$，强度 $\alpha^*$ 由相干性扫描 + Otsu 阈值选出；同时采样 $N_{\text{rand}}=20$ 个随机方向做同样幅度的扰动作为对照。结果目标方向与随机方向在精度恢复（40.0% vs. 31.7%）、模式转移率上无显著差异（$p=0.286$，Cohen's d<0.5），并通过"冲突注入"再次确认扰动效应是幅度依赖而非方向特异的。
- **方法学建议**：在跨数据集做探针时必须同时报告 source-prediction 精度和 mode-prediction 精度，并把残差分析 + 随机方向控制纳入转向向量实验的标准流程；并呼吁设计格式统一的跨推理类型基准。

### Key Contributions (English)
- **Format confound decomposition.** A residualization pipeline (Ridge regression on source one-hot + option count + response length) reduces reasoning-mode probe accuracy from 100% to ~33.5% — the original separability is fully explained by surface format.
- **Trace-mode agreement analysis.** Using mode-specific behavioral anchors embedded in last-layer states, the model's reasoning traces match the intended mode only 42.5% of the time (chance = 33.3%), even though overall accuracy is 86% across all three task types. The model deploys a largely uniform reasoning strategy.
- **Causal steering with random-direction controls.** Targeted steering at layer 32 produces an accuracy recovery of 40.0%, while 20 random-direction perturbations achieve 31.7% ± CI — an empirical p-value of 0.286 (Laplace-corrected). Conflict-injection experiments further show that steering effects are magnitude-based, not direction-specific. This is the first study (to the authors' knowledge) to include random-direction controls as a standard practice in steering-vector work.
- **Methodological recommendations.** The paper argues that (a) source-prediction accuracy should always be reported alongside mode-prediction accuracy in cross-dataset probes; (b) residual analysis should be a routine control; and (c) steering-vector experiments should always include random-direction baselines to test directionality rather than mere perturbation sensitivity.

## Methodology

### 数据集
- 750 个任务，三类各 250，严格平衡：演绎 = LogiQA 2.0（四选一 + 短文），归纳 = ARC-Challenge（四选一科学题），溯因 = αNLI（二选一，从两个观察选最佳假设）。
- 标签 = 数据集来源决定的"目标推理类型"（即标准多源评估的标签生成方式，作者承认这并非完美映射，但恰好强化论点：如果标签噪声大、探针仍能 100% 命中，则说明探针检测的是源身份）。

### 模型与推理
- Qwen3-14B（40 层，d=5120），bfloat16，单张 NVIDIA GH200 480GB（模型占 29.5GB，batch=8），代码有 Qwen3-4B 兜底但未触发。
- 统一 prompt：要求 step-by-step 推理并以 `<answer>` 标签收尾；greedy decoding，预算 2048 tokens。
- 显式设 `DISABLE_THINKING=True` 并剥离 `<think>...</think>` 块，确保抽取的隐藏状态、追踪和置信度对应非思考推理模式——作者明确把这当作方法学控制，避免思考链自身的格式差异造成二次混淆。
- 提取：每个任务在每个层 ℓ 上最后一输入 token 的隐藏状态 $h_i^{(\ell)}\in\mathbb{R}^{d}$、生成文本 $y_i$、输出置信度 $c_i$（token 概率的几何平均）；几何分析只保留答对的样本。

### 逐层线性探针
- 对每个层 ℓ 训练 L2 正则（$C=1.0$）logistic 回归，5 折分层 CV，按准确率选最佳层 $\ell^*$。同时在 $\ell^*$ 上做流形几何统计：TwoNN 内禀维数、邻域 SVD 局部曲率、模式间分离比、KNN 凸包污染率。

### 格式混淆分析（四阶段）
1. **源预测**：用同一探针结构预测数据集来源（LogiQA / ARC / αNLI），与模式预测对比——两者同为 100%。
2. **选项数探针**：仅以 $n_{\text{options}}\in\{2,4\}$ 为输入的 logistic 回归，验证单一标量就能部分分开模式。
3. **格式受控比较**：只看四选一（LogiQA + ARC, n=500），看是否仍能分。
4. **残差分析**：用 Ridge 回归 $f_i=[\text{source one-hot}, n_{\text{options}}, |y_i|]$ 拟合隐藏状态，残差 $r_i=h_i-\hat{h}_i$ 后再探针；这是核心结果。

### 追踪-模式一致度
- 为每个模式写"行为锚点"（如演绎 = "applies a known rule step-by-step"），用模型最后一层隐藏状态嵌入锚点和追踪，每个追踪分配给 cosine 相似度最高的模式；与 1/K=33.3% 随机基线对比。

### 因果转向 + 随机基线
- 对每个模式对 $(m_s, m_t)$ 取 $\hat{v}_{s\to t}=(\mu_t-\mu_s)/\|\mu_t-\mu_s\|$，前向 hook 在 $\ell^*$ 加 $\alpha^*\hat{v}_{s\to t}$；$\alpha^*$ 用相干性扫描 + Otsu 阈值自动选。
- **随机方向基线**：$N_{\text{rand}}=\max(5,\min(20,2\cdot n_{\text{steered}}))$，本研究中为 20，每个 $v_{\text{rand}}\sim\mathcal{N}(0, I_d)$ 单位化后用同样 $\alpha^*$ 在同层做扰动。
- **冲突注入**：同时注入两个目标方向的向量（$\tilde h_i=h_i+\alpha^*(\hat v_1+\hat v_2)$）和 10 对随机单位向量对照。
- 经验 p 值：$p=(k+1)/(N_{\text{rand}}+1)$（Laplace 修正），其中 $k$ 为不弱于目标方向的随机方向数。

### 统计
- Bootstrap CI（$n_{\text{boot}}=2000$，95%），置换检验（$n_{\text{perm}}=5000$），所有转向方向性用经验 p 值，所有对照用 Cohen's d。所有超参（KNN k、CV 折数、转向强度）从数据或实验设计派生，不手动调。

## Results & Highlights

### 1) 探针在 layer 32 达到完美分离
- Qwen3-14B 的 41 层中，CV 准确率在浅层接近 33.3% 随机，随深度单调上升，layer 32（80% 深度）达到 **100% 平衡准确率**，精确率/召回率/F1 全为 1，置换检验 $p<0.0002$（$n_{\text{perm}}=5000$）。
- 流形几何（layer 32）：UMAP 三个干净聚类；内禀维数 演绎 20.6 / 归纳 28.5 / 溯因 33.6；分离比均 >1.0；KNN 凸包污染率 ≤ 1.5%。**表象上完全是"模式特异内部表征"的强证据。**

### 2) 分离完全由格式驱动（Table 1）

| Probe Target | Raw States | Residual States |
|---|---|---|
| Reasoning Mode (D/I/A) | 100.0% | ≈33.5% |
| Dataset Source | 100.0% | ≈33.5% |
| Chance Level | 33.3% | 33.3% |

- Stage 1：源预测 100%，与模式预测信息等价（多源设计下两者完全对应）。
- Stage 2：仅以 $n_{\text{options}}$ 为输入的 logistic 回归达到 33.3% 模式精度——即 αNLI（2 选）的先验。
- Stage 3：仅在 4 选任务（LogiQA + ARC, n=500）上探针仍接近完美，说明选项数之外的词汇/句法/域差异也足够。
- **Stage 4（核心）**：残差探针精度 ≈ 随机 → 任何"模式特异几何"在去掉线性格式信息后消失。**作者明确警告：任何"reasoning-mode 标签与数据来源完全对应"的多源探针都会有这个混淆。**

### 3) 推理行为不随模式切换（Figure 4）
- 模型整体 86% 准确（LogiQA 73.2%、ARC 93.6%、αNLI 91.2%）。
- 但 trace-mode 一致度仅 **42.5%**（chance 33.3%），主模式分数分布宽——模型并不按任务类型调整策略。**行为结果与几何结果收敛：没有特异的内部模式，也没有特异的外部行为。**

### 4) 因果转向无可测功能连接（Figure 5 / Table 2）

| 指标 | Targeted | Random (n=20) |
|---|---|---|
| Accuracy recovery | 40.0% | 31.7% ± CI |
| Mode shift rate | comparable | comparable |
| Empirical p-value | 0.286 (n.s.) | — |
| Cohen's d | < 0.5 | — |

- 目标方向未显著优于 20 个随机方向。
- 冲突注入：目标对和随机对都触发 100% 相干性坍塌，**效应是幅度依赖而非方向特异。**

### 整体结论（三条独立证据收敛）
1. 残差探针：模式特异几何不存在。
2. 追踪-模式一致度：模式特异行为不存在。
3. 因果转向：几何与行为无可测连接。
→ **"高线性探针精度 ≠ 内部计算结构"，在推理研究中要怀疑格式混淆。**

## Critical Assessment

### Novelty
- 主要新颖性在**方法学**：把 Hewitt & Liang 2019 "control task" 思路延伸到"推理模式"这一语义更强、混淆更隐蔽的领域；并首次（按作者说法）把"随机方向对照"作为转向向量实验的标准基线。
- 概念上不算开创，但**对社区是一种重要的"反潮流"工作**——目前大量 "reasoning circuits / mode-specific representations" 论文的探针设计都没做这种对照。
- 单一模型（Qwen3-14B），单一 prompt 模板，单一 token 位置（last input token）——新颖性集中在方法学上而非经验发现上。

### Rigor
- 统计做得扎实：5000 置换、2000 bootstrap、20 随机方向 + Laplace 修正 p 值、Cohen's d、KNN 凸包污染率、TwoNN 内禀维数、UMAP 可视化。
- **残差分析中的 Ridge 用 source one-hot 可能把"真实信号"一起抹掉**——作者自己也承认这是保守的，提议一个中间分析（只回归选项数+长度，不回归源）作为未来工作。
- αNLI 2 选 vs LogiQA/ARC 4 选这个结构性差异单独就能贡献 33.3% 模式精度；作者做了 format-controlled comparison（只 4 选）来缓解，但论文里没有把所有 4 选任务在**选项数对齐后**的进一步细分结果展开。
- trace-mode 一致度依赖"手工写的行为锚点 + 最后一层状态 + cosine 相似度"——足够粗粒度，可能错过细致推理差异（作者也承认）。
- 转向实验 $n=15$ 错了的任务，$N_{\text{rand}}=20$，规模小；$p=0.286$ 严格说不显著是因证据不足而无法拒绝 H₀，论文措辞是"no functional link"——是过强声明的边界，文中已经做了一些克制。

- 单卡 GH200 480GB（过规格了），有 Qwen3-4B fallback，应该能在多数实验室复现。
- 所有超参从数据派生，无手动调参；prompt 模板在 Appendix A。

### Significance
- 对 reasoning interpretability 社区是一次**重要警告**。把"高探针精度"等同于"模式特异计算结构"的推断在多源设置下不成立——这个观察若被广泛接受，会让一批声称找到 "reasoning circuits" 的工作需要补做格式混淆控制。
- 对自演化智能体/自我改进系统的设计也有间接含义：如果模型对三类推理使用同一通用策略，那么跨推理类型的迁移训练是合理预期，但"模式特异"地修补一类失败也不太可能成为有效路径。

### Red flags
- 单模型、单 prompt 模板、单 token 位置——结论的普遍性需要多模型复现验证。
- 残差分析中 source one-hot 风险：可能把跨数据集的真实差异一并抹掉，导致结论"过于负面"。
- $p=0.286$ 不能直接支持"无功能连接"的强声明；更稳妥说法是"在当前规模下未检测到方向性效应"。
- 三类任务样本数 250+250+250——量小但平衡、且作者只做几何分析时只保留答对的样本。
- 没有任何对思考模式（thinking mode）的实验——作者主动剥离，可能错过 thinking 模式下的真实模式特异结构。

## Strengths
- **精准定位的方法学痛点**：直指多源探针研究的隐含假设，没有绕弯子。
- **三线证据收敛**：残差探针 + 追踪-模式一致度 + 因果转向，结论彼此独立、相互强化。
- **随机方向基线**：转向向量研究里少见的、必要的对照，论文用了 20 个方向和 Laplace 修正 p 值。
- **统计严谨**：5000 置换、2000 bootstrap、Cohen's d、超参从数据派生。
- **开放代码 + prompt 模板 + 兜底模型**：可复现性高。
- **可执行建议**：明确给出"always report source-prediction alongside mode-prediction / include residual analysis / use random-direction controls / design format-controlled benchmarks"四条工程建议。

## Limitations
- **单模型**：只跑了 Qwen3-14B，跨模型族（Llama、Mistral、GPT-4）复现留给未来工作；结论的普遍性受限。
- **单 prompt 模板 + 单 token 位置**：只在最后一个输入 token 抽取状态，其他位置、其他 prompt 风格可能不同。
- **格式混淆的混淆**：αNLI 是 2 选，LogiQA/ARC 是 4 选，结构性差异明显；format-controlled comparison 只解决了部分问题。
- **残差 Ridge 用 source one-hot 过保守**：可能连同真实信号一起抹掉；中间分析（仅去选项数 + 长度）未做。
- **追踪-模式一致度粒度粗**：依赖手工锚点 + cosine 相似度，可能漏掉细粒度推理差异。
- **无 thinking mode 实验**：主动剥离思考链，但思考模式下是否出现真正的模式特异结构是开放问题。
- **转向实验规模小**：$n=15$ 任务，$N_{\text{rand}}=20$；$p=0.286$ 弱。
- **benchmark↔reasoning-mode 映射有噪声**：作者承认 ARC 题目可能混合多种推理类型，但把这当作"强化论证"的姿态略激进。

## Relevance
- **直接命中**用户的"LLM Hidden States & Interpretability"主兴趣区：这是关于"hidden states 探针是否真的捕捉到 reasoning 相关结构"的硬核方法学工作，与"linear probing / representation geometry / 信息编码在隐藏态中"高度相关。
- 与"LLM Inference Evaluation"间接相关：揭示了"用不同 benchmark 评估不同推理模式"的标准评估范式有结构性缺陷。
- 与"ML Reliability & Mathematical Foundations"弱相关：作为对 interpretability 推断模式的可靠性审视。

## Questions for Further Exploration
- **多模型复现**：同样的格式混淆分解在 Llama-3、Mistral、GPT-4、Claude 上是否复现？uniform-strategy 是 Qwen3 特性还是更普遍？
- **格式统一基准**：能否构造"演绎/归纳/溯因三合一"且 surface format、选项数、词汇分布完全一致的基准（类似 LogiQA 2.0 NLI 变体的方向）？在这样的基准上探针会降到什么程度？
- **thinking mode 重做**：在启用思考链的 Qwen3-14B 上重跑整套流程——剥离思考块是否就是这种格式混淆的根源？思考模式下是否真的出现模式特异计算？
- **中间残差分析**：只回归选项数 + 长度（不回归 source one-hot），看是否仍能解释大部分分离——量化"源身份"和"其它格式特征"各自的贡献。
- **更细粒度追踪分析**：与其用锚点 + cosine 相似度，是否能用规则计数（显式三段论、假设消除、模式枚举）来直接评估行为切换？
- **转向规模与强度曲线**：扩大 $N_{\text{rand}}$ 和被转向任务数，做完整的 effect size + 置信区间估计，给"无方向性"这个结论更硬的统计基础。
- **多模态扩展**：在多模态 LLM 的视觉-语言联合隐藏态上是否同样存在"模态-任务格式混淆"？探针的高准确率是否对应真实跨模态推理结构？
- **对自演化智能体设计的含义**：如果模型用统一策略处理三类推理，"为某类推理失败做模式特异修补"这条路是否还有效？训练数据混合 / RL reward shaping 应如何设计才能让模型真的分化出模式特异策略？
