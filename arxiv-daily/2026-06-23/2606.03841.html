EvoDS — Self-Evolving Autonomous Data Science Agent with Skill Learning and Context Management

Zherui Yang · Fan Liu · Yansong Ning · Hao Liu  |  github.com/usail-hkust/EvoDS

A self-evolving agent that synthesizes its own skills and compresses its own context — under one RL-trained policy.

A hierarchical Manager + five specialized sub-agents (Cleaner / Featurizer / Modeler / Visualizer / Debugger) with disjoint action spaces. New skills are gated by a usage-frequency threshold τ = 3. Compression is a learned control signal, not a truncation. Trained end-to-end with two-stage agentic RL.

Methodology

Hierarchy + Self-Evolution, end-to-end

Hierarchy. A single Manager interfaces with the code-execution environment and delegates to five sub-agents whose local action spaces are disjoint (Ai ∩ Aj = ∅). Two-tier memory — a global objective plus local subtask context — localizes long execution so that one long trace does not pollute every agent.

ASA — Autonomous Skill Acquisition. When a sub-agent is out of capability, it synthesizes a candidate skill (name, description, executable code), validates it in a sandbox, caches it, and only permanently merges it once it has been re-generated τ = 3 times. This blocks one-shot flaky skills.

ACC — Adaptive Context Compression. The Manager treats compression as a learned control decision: when to compress and what to keep. Token cost is modeled as E[|Z|] = κ·H(Z) and folded into the reward — under mild monotonicity, this is shown to be an Information Bottleneck objective (Theorem A.6).

Two-stage agentic RL. Stage 1 SFT warm-up on teacher (DeepSeek-V3.1 / Qwen3) trajectories. Stage 2 joint multi-role GRPO/REINFORCE++ over all roles, with fast-weight Δt rollouts and stop-gradient — θ0 is the RL parameter, Δt is the test-time memory state.

Results & Highlights

+28.9 % avg., no OOT past 25 k tokens

  • Headline. +28.9 % average gain over open-source SOTA across four benchmarks (MLAgentBench, DA-Code, DSEval, ML-Tool-Bench).
  • Context robustness. Accuracy is stable across 5 k–25 k contexts; DeepAnalyze degrades sharply and runs OOT past 25 k. EvoDS never exceeds the budget (Fig. 3).
  • RL dynamics. Joint multi-role RL converges to higher validation reward at fewer context tokens than Manager-only GRPO (Fig. 4).
  • Skill reuse. A skill synthesized for one stroke-prediction task is reused on a downstream submission-format task (Fig. 2).
  • Honest failure. The system flags its own failure on a professional quant-finance task — domain knowledge self-evolution cannot replace.
  • Theory-aligned. Theorem A.5 strictly bounds tool-selection error below a flat agent under disjoint subsets; Theorem A.6 ties ACC to the IB principle.

Strengths

  • Coupled bottlenecks. Skill acquisition and context management are solved jointly, not piecemeal.
  • Theoretical backbone. Error-bound reduction (A.5) + IB interpretation (A.6) — rare for an agent-systems paper.
  • Honest failure case. Quant-finance task is reported as a failure instead of being hidden.
  • τ = 3 gate. Small, clean engineering decision that keeps the skill library from accumulating garbage.
  • Open code. github.com/usail-hkust/EvoDS.

Limitations

  • Closed-source gap. No head-to-head with Claude-Opus-4.6, AIDE, RE-Bench, DeepAnalyze-Pro.
  • Library explosion. τ = 3 is a hyperparameter; behavior over thousands of tasks is not analyzed.
  • Qualitative transfer. Cross-task skill reuse is shown in one case; no quantitative transfer metric.
  • Narrow domain. Benchmarks are standard tabular ML — no scientific-discovery or NLP coverage.
  • RL convergence. Joint multi-role RL convergence is shown on a single benchmark.
  • Assumption load. Theorem A.5/A.6 rest on i.i.d. Gaussian noise, monotonicity, and H(Z) ∝ |Z| — not separately validated empirically.

Questions for Further Exploration

  1. Does τ = 3 still prevent garbage at the scale of 1 000+ tasks, or would a learned τ help?
  2. Can ASA + ACC be combined with a process-reward model so skill synthesis is verifier-guided?
  3. Does disjoint-subset reduction still hold when sub-agents share a backbone and have correlated errors?
  4. Is the IB bound tight enough to predict the empirical token-cost / accuracy curve?
  5. How does EvoDS perform on scientific-discovery benchmarks (SciHorizon, ScienceAgentBench)?
  6. Can ASA be transfer-aware — fine-tune a skill across sub-agents instead of re-synthesizing?
  7. What are the variance bounds on the +28.9 % claim across more than two base models?
  8. Can ACC compress across sub-agent boundaries without breaking the disjoint-action-space assumption?
