Why This Matters

Demonstrates that search architecture — not just stronger LLMs or longer training — is the binding constraint for end-to-end MLE agents. A single backbone-agnostic agent family now tops every column on MLE-Bench.

Critical Edge

100% valid submission rate on MLE-Bench is unusual — worth checking whether failure modes are silently downgraded or retried before declaring this a clean SOTA.

01 Methodology

Progressive MCGS

Extends MCTS into a directed graph $G=(V, E_T \cup E_{\text{ref}})$. Tree edges $E_T$ carry credit; reference edges $E_{\text{ref}}$ enable cross-branch composition but are excluded from backprop. A Shannon-entropy soft switch $w(t)$ gradually trades UCT exploration for Elite-Guided exploitation as $\exp(H(\pi_t))$ decays from 4.8 → 2.8.

Retrospective Memory

Cold-start domain knowledge base (curated model lists per task type) combined with a dynamic global memory. Retrieval fuses lexical + FAISS via Reciprocal Rank Fusion — no extra LLM reflection calls needed. Planning uses the free-text plan as query; debugging uses the error message.

Hierarchical Planning + Adaptive Code

Decouples Planner (what & why) from Coder (how). Switches among three modes based on search state: Base (full rewrite, cold start), Stepwise (module-by-module), Diff (patch-based). A nine-agent team (Draft / Improve / Debug / Evolution / Fusion / Aggregation / Code Review / Data Leakage / Result Parse) executes each node.

Reward & Stagnation

Discrete reward $R(v) \in \{-1, 1, 2\}$ for fail / no-improve / branch-best. Branch-level stagnation ($\tau_{\text{branch}}$) triggers intra-branch reflection → cross-branch reference; global stagnation ($\tau_{\text{global}}$) triggers multi-branch aggregation.

03 Strengths

  • SOTA across the board at half budget. 65.3% medal, 100% valid, 76.0% above-median — beats every published open and proprietary baseline.
  • Backbone-agnostic. Verified on four LLMs across three modalities (Image / NLP / Audio).
  • Cross-domain transfer. Wins AlphaEvolve math tasks without modification — the pattern isn't MLE-specific.
  • Clean abstraction. $G=(V, E_T \cup E_{\text{ref}})$ + soft switch $w(t)$ + reward $R(v)$ is a compact, reusable formulation.
  • Practical stability. Planner–Coder decoupling + Diff mode kills the "every iteration rewrites everything" failure mode.
  • No extra reflection LLM. Retrospective Memory reuses experience without paying MARS-style inference tax.

04 Limitations

  • Proprietary backbone dominates. Gemini-3.1-Pro-preview is unavailable; open-source replications will see a sizeable gap.
  • Curated cold-start KB. Initialization leans on hand-built domain priors — KB maintenance is itself engineering work.
  • Hand-tuned schedule. $w(t), c(t), K, \tau_{\text{branch}}, \tau_{\text{global}}$ are fixed across 75 tasks; no learned variant.
  • Heavy compute. 21 vCPUs + 234 GB RAM + 1× H200 per task — far beyond a typical Kaggle contestant.
  • No theoretical convergence guarantee. Progressive MCGS is empirically dominant, not proved.
  • Multi-branch aggregation failures unstudied. When $R_{\text{agg}}$ starts badly, recovery is not analyzed.
  • Narrow cross-domain scope. AlphaEvolve tasks are math/programming only; no physics, chemistry, or general-science evaluation.

05 Questions for Further Exploration

  • Adaptive schedule? Can RL / meta-learning replace the hand-tuned $w(t), c(t), K, \tau$ and adapt per task difficulty?
  • Credit leak? If a child inherits a high-reward reference point, backprop-only-on-$E_T$ may systematically under-credit the reference — is the undercount material?
  • Stage-specific retrieval heads? Planning vs. debugging queries differ hugely — should the RRF fusion be replaced by a learned per-stage retriever?
  • Backbone replication gap. How much of the gain survives on open-source LLMs (DeepSeek-V3, Qwen3)? A direct ablation would settle the "search vs. model" debate.
  • AI-for-Science port. Does Progressive MCGS work when the candidate is an experiment configuration rather than code? The authors flag this as future work, but the interface is unspecified.
  • 100% valid = narrower search? If every submission is valid, are we actually searching a deeper but smaller valid-subspace? Valid-rate × medal-rate decoupling deserves a chart.
  • Post-hoc graph analysis. Can the reference-edge graph be clustered or visualized to debug agent behavior, beyond the single global active-branch count?
