> **一句话总结：** 提出面向计算数学开放问题的智能体研究系统 Iteris（探索—规划—执行循环 + 基础/实验/证明/审查四类执行智能体），并将其应用于 Simons 工作组提出的两个开放问题：一是在幂律谱下 CG 与随机坐标下降（RCD）的固定参数相位图（Theorem 1），二是构造 QRCP（带列主元的 QR 分解）即使在低一致性下也可能选出病态子矩阵的反例族（Theorem 2）。

> **TL;DR:** This paper introduces **Iteris**, an agentic research system that orchestrates an **explore–plan–execute loop** with four specialized execution agents (Foundation, Experiment, Proof, Review) and file-based shared memory. Iteris is designed for **open problems in computational mathematics**, where progress requires coordinating numerical experimentation, adversarial construction, and proof development — modes that cannot be reduced to a single executable objective. The authors apply Iteris to two open problems from a recent Simons Workshop collection. **Problem I** (CG vs. RCD on power-law spectra): they establish a fixed-parameter phase diagram for the asymptotic ratio ρ_n(ϵ, p) = TRCD/n / TCG, showing zero-ratio regimes (where RCD wins), bounded-below regimes (where CG wins), and critical boundary obstructions — with Theorem 1 giving conservative stochastic upper bounds for each regime. **Problem II** (QRCP orthonormal-row selection): they answer Simons Problem 4.3 in the **negative**, constructing for every (H*, B, e, K) a low-coherence orthonormal-row matrix Q such that QRCP selects a submatrix M(Q) with γ(Q, I_piv) / √(k(n−k+1)) > Bk^e (Theorem 2). The QRCP proof has been fully verified in Lean 4 via the autoformalization agent Archon. Across both cases, Iteris ran >100 iterations per problem, generated exploratory artifacts, candidate proof modules, and counterexample constructions, but the final mathematical arguments still required expert human audit and reorganization — illustrating that agentic AI can participate meaningfully in computational-mathematics research while human validation remains essential.

## 核心贡献

- **Iteris 智能体研究系统：** 提出一种**探索—规划—执行**（explore-plan-execute）循环协议，将计算数学研究的多种模式（数值实验、构造反例、撰写证明、路线审查）分解为可审计、可交接的子任务。系统使用文件作为长期记忆和结构化消息，包含**探索智能体**（探测潜在方向并发出建议）、**规划智能体**（读取全局状态并产出 TASK_POOL.json）、以及**基础/实验/证明/审查**四类执行智能体。

- **Theorem 1（CG vs RCD 相位图）：** 对 Simons Problem 2.4，给出幂律谱下精确 CG 与源分布 RCD 在 epoch 归一化代价比 ρ_n(ϵ, p) 上的**固定参数相位图**。划分 5 个零比值区间（p>1、p=1、1/3<p<1、p=1/3、0<p<1/3 且 ϵ<F(p)），每区间给出 ρ_n → 0 的依概率收敛与保守的随机上界；同时给出有界远离零的对比区间（0<p<1/3 且 ϵ>F(p)）以及临界边界 ϵ=F(p) 的非平凡下界（包括 K≥1 的子带加强版本）。

- **Theorem 2（QRCP 一致性有界下的反例族）：** 对 Simons Problem 4.3 给出**否定回答**——对任意 H*>1、B>0、e≥0、K≥1，存在 k≥K、n>k 的正交列矩阵 Q∈ℝ^{n×k}（Q^T Q = I_k），其标准化行相干性 µ(Q) < H*，但 QRCP 在 A=Q^T 上选择的有序主元集 I_piv 使 M(Q) 非奇异，且 γ(Q, I_piv) / √(k(n−k+1)) > Bk^e。该反例族通过精心设计的上三角链 + 正交补全块（X、F、G）实现，三角回代量级呈指数增长。

- **关键数学工具：** (i) 幂律谱下的有限维残差多项式归约与 Hankel 矩矩阵极小化（给出 F(p) 的闭式表达）；(ii) 随机 Gram 一致性的矩阵 Chernoff 界；(iii) p>1 情形下随机投影乘积的 Hilbert 空间压缩定理；(iv) 临界边界的显式 Schur 补渐近；(v) QRCP 反例的指数增长三角回代设计。

- **人机协作的角色澄清：** 对每个问题都对比了直接提交给 GPT-Pro 的结果——GPT-Pro 能给出合理的直觉图景（CG vs RCD 的谱过滤直觉；QRCP 的三角块嵌入反例草图），但均未完成证明闭合。Iteris 的价值在于把这些直觉拆解为可审计的证明义务、由人类数学判断最终修复错误分支（如 CG 速率声明中过强的负幂律被替换为渐增阶非停止估计）、并通过 Archon 实现 Lean 4 形式化。

## Key Contributions

- **Iteris framework (Section 3):** A file-mediated multi-agent architecture for computational-mathematics research. The **explore-plan-execute protocol** runs in three ordered phases per iteration: an *Exploration agent* runs repository-level probes (file reads, scratch reasoning, route comparisons) and writes an advisory (recommended task shape + local-minimum check + risks); a *Plan agent* consumes the advisory plus global project files (problem statement, roadmap, project memory, route status, frontier state) and writes `TASK_POOL.json` (category, goal, inputs, scope, verification, result contract); one or more *Execution agents* (Foundation / Experiment / Proof / Review) then complete one well-scoped research action each and write structured result files back into project memory. The framework treats files as both **long-term memory** and **structured inter-agent messages**, which lets the system run >100 iterations per problem while keeping numerical evidence, proof sketches, reviewed facts, and formal theorem claims cleanly separated.

- **Execution-agent separation of concerns:** Foundation agent freezes definitions, drafts conjecture cards, maintains claim discipline; Experiment agent runs numerical work and counterexample searches (evidence never treated as proof); Proof agent constructs and checks proof artifacts for stable statements; Review agent evaluates evidence, gaps, route value, and claim status (may recommend deepening, reframing, or retiring a branch). This mirrors a mathematician deciding whether the next step is source cleanup, computation, proof work, or strategic review.

- **Exploration agent for escaping local inertia:** The Plan agent is separated from the Exploration agent precisely because planning from global state alone may over-commit to a branch that "already looks most natural" — leading to incremental parameter tuning. The Exploration agent runs aggressive probes (recent-iteration review, prior-fact retrieval, route comparison) and produces a decision signal, which the Plan agent may adopt, modify, or reject when writing TASK_POOL.json.

- **Theorem 1 (CG/RCD fixed-parameter phase diagram):** For the Simons Problem 2.4 power-law model An = Un diag(1−p, 2−p, …, n−p) Un* with Haar Un, right-hand side bn = An zn and exact solution x*,n = zn, the paper establishes the asymptotic behavior of ρ_n(ϵ, p) = TRCD,n(ϵ, p; I)/n ÷ TCG,n(ϵ, p) for fixed (p, ϵ). The active degree K(p) = max{k ≥ 0 : (2k+1)p < 1} and the CG floor F(p) = 1 / (G_{K(p)}(p)^{−1})_{00} (a finite-dimensional moment problem over the limiting measure ν_p(dy) = (1−p)/p · y^{−1/p} dy) split the (p, ϵ) plane into:
  - **Zero-ratio regimes** (ρ_n → 0 in probability): p > 1 (bound O_P(n^{−1})); p = 1 (bound O_P(log n / a_n) for any a_n → ∞ with log a_n = o(log n)); 1/3 < p < 1 and p = 1/3 (F(p) = 1, bound O_P(1/b_n) for any b_n → ∞ with b_n log(1+b_n) = o(log n)); 0 < p < 1/3 with ϵ < F(p) (same O_P(1/b_n) bound).
  - **Bounded-away regime:** 0 < p < 1/3 with F(p) < ϵ < 1 — P(ρ_n > η(p, ϵ)) → 1.
  - **Critical boundary:** 0 < p < 1/3 with ϵ = F(p) — lim inf P(ρ_n > η(p)) ≥ c(p) > 0, with strengthened version P(ρ_n(F(p), p) > η(p)) → 1 on the endpoint/lower-subband critical regimes p = 1/(2K+3) or 1/(2K+3) < p ≤ 1/(2K+2) for K ≥ 1.
  
  The rate bounds are conservative stochastic upper bounds; the paper explicitly disclaims sharpness.

- **Theorem 2 (Bounded-coherence QRCP obstruction):** For every H* > 1, B > 0, e ≥ 0, and K ≥ 1, there exist integers k ≥ K and n > k and a matrix Q ∈ ℝ^{n×k} with Q^T Q = I_k, such that QRCP applied to Q^T selects an ordered pivot set I_piv for which M(Q) is nonsingular and γ(Q, I_piv) / √(k(n−k+1)) > Bk^e, with normalized coherence µ(Q) = (n/k) · max_{1≤i≤n} ‖Q(i,:)‖₂² < H*. The construction:
  - Partitions A = Q^T ∈ ℝ^{k×n} into four blocks (R+, X, F, G) where R+ = [R_β z; 0 η_p] is the QRCP-selected badly-conditioned block.
  - Builds R_β as an upper-triangular chain with entries (R_β)_{t,t} = d_t = (1/k) ε^{t−1}, (R_β)_{t,q} = −(1/2) σ_t σ_q s d_t for t < q, where ε = e^{−η/k}, s = √(1−ε²).
  - Designs z with carefully chosen signs so that the entries of R_β^{−1} z reinforce rather than cancel — making ‖R_β^{−1} z‖₂² grow exponentially.
  - Adds two cross columns X to cancel the mixed block from R+ R+^T, then a frame block F of many small rank-one pieces (each with W_h < d_m², pivot-control condition), and finally a residual block G to complete A A^T = I_k.
  - Guarantees QRCP picks the prescribed pivot path (the small-column bounds force every frame column to have residual below the next intended selected residual).

- **Trajectory analysis (Sections 4.2.2 & 4.3.2):** For the CG problem, the trajectory went through (1) Foundation freeze of the comparison model; (2) Plan-agent decomposition into spectral regimes (p > 1, p = 1, 0 < p < 1 with below-floor / above-floor / critical); (3) Proof-agent generation of candidate modules (CG polynomial representation, RCD epoch bounds, RCD lower gate, active moment problem, p=1 row, p>1 row, Schur boundary); (4) Review-agent blocking of overstrong shortcuts (fixed-degree CG non-stopping ≠ growing-degree rate; selected-row estimates ≠ global statements); (5) Human-audited repair replacing informal arguments with matrix Chernoff, projection-product theorem, explicit Schur asymptotics, and the K=0 endpoint (p=1/3) treated separately. For the QRCP problem, the trajectory went Foundation freeze → failed proof patterns (k ≤ 2 positive, k = 3 exposes missing cross-row control) → obstruction insight → reverse construction (design selected core, then completion problem for unused rows) → extension to the low-coherence setting.

- **Comparison with direct GPT-Pro prompting:** Both problems were also submitted to GPT-Pro in advanced reasoning mode. For CG/RCD, GPT-Pro produced "an attractive heuristic picture" — spectral-filter intuition, polynomial CG scalings, epoch-scaled RCD dominance — but treated coordinate RCD as a diagonal spectral filter, blurred expected/typical/high-probability stopping times, and asserted sharp CG rates without the growing-degree lower bounds needed. For QRCP, GPT-Pro produced a "meaningful high-level counterexample sketch" correctly identifying the triangular-block + orthogonal-completion mechanism, but several essential steps were left as plausible rather than explicitly discharged. Iteris's role is thus to decompose plausible heuristics into auditable proof obligations.

- **Lean 4 formalization (Archon):** The QRCP proof (Problem 4.3) has been verified in Lean 4 with the aid of the autoformalization agent Archon (https://github.com/frenzymath/Archon). The paper notes that this does not eliminate the need to check that the formal statement matches the informal mathematical theorem — Archon's output and the paper claims were manually aligned.

## Methodology

**Iteris architecture.** Three principal components organized as a per-iteration protocol:
- *Agentic explore–plan–execute protocol (Section 3.1):* Each iteration is (1) Explore — the Exploration agent runs repository-level probes, drafts scratch reasoning, and writes an advisory containing the recommended task shape, local-minimum check, routes considered, and risks/non-goals; (2) Plan — the Plan agent reads project files + advisory and writes `TASK_POOL.json` specifying each task's category, goal, input files, allowed scope, verification requirements, and result contract; (3) Execute — each selected Execution agent receives its assigned task and writes structured result files back into project memory.
- *Execution agents for research modes (Section 3.2):* Four task-specific categories defined by Markdown capability specs — Foundation (source auditing, definition freezing, conjecture cards, claim discipline), Experiment (numerical harnesses, diagnostics, counterexample search — never treated as proof), Proof (proof artifacts for stable statements, special cases, route-level lemmas), Review (evidence evaluation, gap detection, route retirement/reframing).
- *Exploration agent for escaping local inertia (Section 3.3):* Decoupled from the Plan agent to avoid over-committing to the current branch; produces decision signals the Plan agent may adopt, modify, or reject. Files serve dual roles as memory (keeping state, reviewed facts, claim limits stable) and as messages (advisories, task pools, result files).

**Implementation.** Generation agent implemented using the OpenAI Codex agent with GPT-5.5 as the underlying model. Iteris ran >100 iterations per problem; theorem statements and final proofs were verified, repaired where necessary, and organized by the authors.

**Problem I model (Simons 2.4).** Fixed-power-law spectrum A_n = U_n diag(1^{−p}, 2^{−p}, …, n^{−p}) U_n* with Haar unitary U_n. Right-hand side b_n = A_n z_n with z_n independent uniformly distributed complex unit vector, so exact solution x*,n = z_n. Both algorithms start from x_0 = 0, errors measured in A_n-norm. CG is exact unpreconditioned conjugate gradient; RCD is coordinate sketch-and-project sampling i_t ∈ {1, …, n} with probability A_n(i, i) / tr(A_n), performing exact coordinate updates. Comparison unit: 1 RCD epoch = n coordinate updates. The ratio ρ_n(ϵ, p) = TRCD,n / n / TCG,n with ϵ ∈ (0, 1).

**Mathematical machinery for Theorem 1.** Four-step proof sketch: (1) CG polynomial representation — TCG,n(ϵ, p) = min{d : M_d,n(p) ≤ ϵ} where M_d,n(p) is a residual-polynomial minimum; (2) limiting moment problem — for 0 < p < 1, define ν_p(dy) = (1−p)/p · y^{−1/p} dy, active degree K(p) = max{k : (2k+1)p < 1}, CG floor F(p) = 1 / (G_{K(p)}(p)^{−1})_{00}; (3) phase mechanism — if ϵ < F(p) then TCG,n → ∞ in prob while TRCD,n/n = O_P(1), giving ρ_n → 0 (includes whole 1/3 ≤ p < 1 row since K(p)=0, F(p)=1); if 0 < p < 1/3 and F(p) < ϵ < 1 then TCG,n = O_P(1) but RCD still needs a positive fraction of an epoch, so ratio bounded away from zero with high prob; (4) special cases — p > 1 (summable spectrum, TRCD,n = O_P(1), ρ_n = O_P(n^{−1})); p = 1 (RCD epoch count O(log n), ρ_n → 0); ϵ = F(p) critical boundary decided by first-inactive Schur-complement correction.

**Rate bounds (conservative).** For p = 1, log a_n = o(log n) ⇒ P(TCG,n ≤ a_n) → 0; for 0 < p < 1 zero-ratio regimes, b_n log(1+b_n) = o(log n) ⇒ P(TCG,n ≤ b_n) → 0. Bounds proved via event inclusion {ρ_n > budget_n / d_n} ⊂ {TRCD,n/n > B·budget_n} ∪ {TCG,n ≤ d_n}.

**Problem II model (Simons 4.3).** Q ∈ ℝ^{n×k} with orthonormal columns, A = Q^T. QRCP selects at each step the row label whose column in A has largest squared residual after orthogonal projection. M(Q) = Q[I_piv, :] is the selected submatrix, γ(Q, I_piv) = ‖M(Q)^{−1}‖₂ when nonsingular. Open question: does QRCP force γ(Q, I_piv) ≤ √(k(n−k+1))? Theorem 2 answers negatively.

**Construction for Theorem 2.** Choose m large, k = m+1, build A ∈ ℝ^{k×n} = Q^T in four blocks [R+ X F G]. R+ = [R_β z; 0 η_p] is the QRCP-selected block. R_β upper-triangular with d_t = ε^{t−1}/k, s = √(1−ε²), signs σ_t chosen so R_β^{−1} z entries reinforce (exponential growth). z has entries z_t = (1/4) σ_t s d_t for t < m, z_m = (1/4) σ_m √(1 − d_m²/16), η_p = d_m / 4. X = [θ_1 z θ_2 z; −b_1 η_p −b_2 η_p] with θ_1 b_1 + θ_2 b_2 = 1 cancels the mixed block from R+ R+^T. F = diag-frame block of N_frame small rank-one pieces (W_h < d_m² pivot-control) restoring first-coordinate identity. G = residual block completing the lower-right entry. Verification: A A^T = I_k, Q^T Q = I_k, QRCP pivot path is (1, 2, …, m, m+1), and γ(Q, I_piv)² ≥ ‖R_β^{−1} z‖² / η_p² grows exponentially (Z exponential), exceeding Bk^e for large k.

**Trajectory analysis.** For both problems, foundation work froze model definitions and comparison ratios; proof agents generated candidate modules in parallel; review agents blocked overstrong shortcuts (CG: fixed-degree ≠ growing-degree rates, p < 1 rate assumption replaced by conservative growing-degree non-stopping estimate; QRCP: failed k=3 proof attempt identified the missing cross-row control that a counterexample would need to exploit); human audit reorganized circuitous proofs into readable narratives; QRCP construction extended to bounded-coherence regime post-obstruction.

## Results & Highlights

- **Theorem 1 zero-ratio regimes (ρ_n → 0 in probability).** Five distinct cases:
  - p > 1: summable spectrum ⇒ ρ_n = O_P(n^{−1}).
  - p = 1 (logarithmic boundary): ρ_n = O_P(log n / a_n) for any a_n → ∞ with log a_n = o(log n).
  - 1/3 < p < 1: K(p) = 0, F(p) = 1, ρ_n = O_P(1/b_n) for any b_n → ∞ with b_n log(1+b_n) = o(log n).
  - p = 1/3: same K(p)=0, F(p)=1, same O_P(1/b_n) bound.
  - 0 < p < 1/3, ϵ < F(p): same O_P(1/b_n) bound.
- **Theorem 1 bounded-away regime.** 0 < p < 1/3, F(p) < ϵ < 1 ⇒ ∃ η(p, ϵ) > 0 with P(ρ_n > η) → 1.
- **Theorem 1 critical boundary.** 0 < p < 1/3, ϵ = F(p) ⇒ lim inf P(ρ_n(F(p), p) > η(p)) ≥ c(p) > 0; on the endpoint/lower-subband critical regimes p = 1/(2K+3) or 1/(2K+3) < p ≤ 1/(2K+2) for K ≥ 1, the obstruction strengthens to P(ρ_n(F(p), p) > η(p)) → 1.
- **Theorem 2 obstruction.** For every (H*, B, e, K) with H* > 1, B > 0, e ≥ 0, K ≥ 1, there exist k ≥ K, n > k, Q ∈ ℝ^{n×k} with Q^T Q = I_k, normalized coherence µ(Q) < H*, such that QRCP applied to Q^T yields a non-singular selected submatrix M(Q) with γ(Q, I_piv) / √(k(n−k+1)) > Bk^e. This rules out the explanation that QRCP failure is caused by a few high-leverage rows — it happens even under bounded coherence, which is the standard regime in matrix completion and randomized linear algebra.
- **Lean 4 verification.** QRCP proof (Problem II) has been verified via the autoformalization agent Archon (https://github.com/frenzymath/Archon); CG proof (Problem I) verified in Lean by Zhangsong Li.
- **Iteris trajectory scale.** >100 iterations per problem, with foundation, plan, experiment, proof, and review agents each contributing; >100 of generated proof modules reviewed and consolidated into the final appendices.
- **Human–AI division of labor.** Iteris broadened search space, found candidate structures, produced draft arguments; human inspection detected an overstrong p<1 rate claim and replaced it with a conservative growing-degree non-stopping estimate; human restructuring turned indirect proofs into paper-quality narratives.
- **Direct GPT-Pro comparison.** For CG/RCD: plausible heuristic (spectral-filter intuition, polynomial CG scalings, RCD epoch dominance) but no closed proof — blurred expected/typical/high-probability, asserted sharp CG rates without growing-degree lower bounds. For QRCP: correct mechanism identification (triangular block + orthogonal completion) but several essential steps left as plausible rather than discharged. Iteris's role: decompose these heuristics into auditable obligations.
- **Tooling acknowledgement.** Implementation uses OpenAI Codex agent with GPT-5.5; Archon autoformalization agent used for Lean 4 verification.

## Critical Assessment

- **Novelty (system level).** The explore–plan–execute pattern with separate Foundation/Experiment/Proof/Review agents and file-based memory is a sensible architectural choice for long-horizon research, but it is incremental relative to prior agentic-research systems (FunSearch, AlphaEvolve, The AI Scientist, Aletheia, AI Co-Mathematician). The paper's main claim is not algorithmic novelty but **case-study evidence** that such systems can contribute to genuine open problems in computational mathematics. That claim is substantiated by the two theorems, but the architectural pattern itself would benefit from ablations.

- **Rigor (theorems).** Theorem 1 is stated carefully with explicit stochastic bounds in each regime, but the paper **explicitly disclaims sharpness** ("We do not claim that these bounds are sharp"). Theorem 2 is a clean existential result and has a Lean 4 formalization. Both proofs appear in appendices and the QRCP proof has independent formal verification — high rigor by the standards of agent-generated mathematics.

- **Reproducibility.** The architecture is described in enough detail to be implementable, but the paper does not release the Iteris codebase, the generated artifacts, or the agent prompts (only `TASK_POOL.json` schema is sketched). The QRCP proof has a Lean repository (https://github.com/frenzymath/qrcp-bounded-coherence-obstruction) and the Archon autoformalization agent has a public repo (https://github.com/frenzymath/Archon). Reproducing the CG phase diagram would require re-running the Iteris pipeline on Problem 2.4 with the same prompts — feasible but undocumented.

- **Significance.** Theorem 1 resolves a real open question in the Simons collection with quantitative rate bounds. Theorem 2 is a strong negative result for QRCP under bounded coherence — it rules out a natural pivot-quality guarantee and would be of independent interest even without the agentic framing. Together they represent substantive mathematical contributions.

- **Red flags / honest concerns:**
  1. **"Two case studies" generalization:** Iteris's success on Problems 2.4 and 4.3 is suggestive but not conclusive evidence that agentic AI can routinely participate in computational-mathematics research. The problems chosen may have been tractable precisely because they decompose cleanly into numerical + proof phases; other open problems may resist this structure.
  2. **Hidden human engineering:** The paper is transparent about human interventions (detecting the overstrong CG rate, restructuring the QRCP proof), but the true amount of human steering per iteration is not quantified. The trajectory analyses list agent roles in narrative form but do not give per-step intervention counts, time spent, or compute spent.
  3. **GPT-Pro baseline weakness:** The "direct GPT-Pro" comparison is informal — the paper reports qualitative impressions ("an attractive heuristic picture", "paper-quality closure") rather than a controlled benchmark. This makes it hard to attribute Iteris's success to its architecture versus to multi-turn refinement alone.
  4. **Rate-bound conservatism:** The O_P(1/b_n) and O_P(log n / a_n) bounds in Theorem 1 are explicitly weak — they require a_n, b_n to grow super-slowly (log a_n = o(log n)). The paper makes clear these are not the asymptotic order, just conservative envelopes. A reader expecting tight CG/RCD rates will be disappointed; the value is in the zero-ratio direction and the structural mechanism.
  5. **Lean 4 ≠ mathematical correctness:** The authors explicitly note that the Archon formalization does not eliminate the need to check that the formal statement matches the informal theorem. In the QRCP case, "semantic alignment" between paper claims and Lean statements required human curation.
  6. **Self-evaluation bias:** The paper is written by the same group that built Iteris. The trajectory analyses read as success stories; failure modes where Iteris produced unfixable wrong answers or wasted compute are not quantified.

## Strengths

- **Substantive mathematical content:** Two theorems on genuine open problems from a Simons Workshop collection — not toy benchmarks. Theorem 1 establishes a fixed-parameter phase diagram; Theorem 2 is a strong bounded-coherence obstruction for QRCP.
- **Honest role assignment:** The paper is unusually transparent about what Iteris contributed (exploratory artifacts, candidate proof modules, obstruction insights, counterexample mechanism) versus what required human work (assumption verification, gap detection, incorrect-branch repair, exposition). This sets a healthy standard for the field.
- **Direct baseline comparison:** Comparing Iteris trajectories to single-shot GPT-Pro responses makes the value of multi-turn exploration explicit — even when GPT-Pro identifies the right mechanism (QRCP triangular block + orthogonal completion), Iteris closes the proof obligations.
- **Lean 4 verification of QRCP proof:** Archon formalization adds independent verification beyond human audit.
- **Architectural clarity:** The explore-plan-execute decomposition with file-as-memory and file-as-message is a clean abstraction; the four execution-agent categories (Foundation/Experiment/Proof/Review) map naturally onto research-workflow phases.
- **Concrete trajectory analyses:** Section 4.2.2 and 4.3.2 give play-by-play accounts of how each theorem emerged, including which proof modules were blocked, which were repaired, and what the human did at each gate.
- **Explicit limitations:** The paper acknowledges that Iteris shifts rather than removes human effort, that the rate bounds are conservative, and that Lean formalization does not replace mathematical judgment.

## Limitations

- **Architectural novelty is modest.** The explore-plan-execute pattern is a sensible composition of established LLM-agent primitives (ReAct, file-based memory, role-specialized agents). Iteris's contribution is the **case-study evidence**, not a new algorithm.
- **No ablation or compute report.** The paper does not quantify total iterations, total tokens, wall-clock time, or how Iteris's success depends on the choice of underlying model (GPT-5.5), the number of agents, or the file-memory design. This makes it hard to reproduce or to understand what is essential.
- **No released codebase.** Iteris itself is not open-sourced; only the QRCP Lean repo and Archon are public.
- **Rate bounds are weak.** Theorem 1's bounds require a_n, b_n to grow extremely slowly (log a_n = o(log n), b_n log(1+b_n) = o(log n)). These are stochastic non-stopping envelopes, not asymptotic orders. A reader looking for the precise CG/RCD rate comparison will need to wait for matching lower bounds.
- **GPT-Pro baseline is informal.** No controlled benchmark; impressions are qualitative.
- **Generalization is unproven.** Two case studies (one phase-diagram problem, one counterexample problem) do not establish that Iteris can handle arbitrary computational-mathematics open problems — especially those that require sustained creative construction rather than numerical sweep + proof assembly.
- **Lean 4 ≠ correctness guarantee.** The Archon formalization verifies the core QRCP construction; it does not verify the broader claim that the construction is "low-coherence" (which requires separate human audit) or that the theorem statement matches the Simons Problem 4.3 framing exactly.
- **Human effort not quantified.** The trajectory analyses are narrative; the actual amount of human steering per iteration, and which interventions were essential vs. cosmetic, is not measured.
- **No negative-case analysis.** The paper does not report problems where Iteris failed to make progress, got stuck in bad branches, or produced output that human experts could not repair. A balanced evaluation would include such cases.

## Relevance

本工作与本仓库的研究兴趣高度相关（智能体 AI、LLM 推理、研究自动化、定理证明/形式化验证），并直接对应 Simons 工作组关于计算数学线性系统的两个开放问题。Theorem 1 给出 CG 与 RCD 在幂律谱下的固定参数相位图，Theorem 2 通过低一致性反例族否定 QRCP 的一类列主元质量保证——这两条结论对**随机数值线性代数**和**秩揭示分解**方向的研究者具有独立价值。Iteris 作为一个具体的智能体研究系统（探索—规划—执行 + 文件记忆 + 四类执行智能体），结合 GPT-Pro 基线对比和 Lean 4 形式化验证，对研究**长程 AI 研究工作流**和**人机数学协作**的同行有方法论参考意义。

## Questions for Further Exploration

- **Sharp rates for Theorem 1.** The bounds ρ_n = O_P(1/b_n) with b_n log(1+b_n) = o(log n) and ρ_n = O_P(log n / a_n) with log a_n = o(log n) are conservative envelopes. What is the true asymptotic order of ρ_n(ϵ, p) in each regime? Is there a single dominant term, or does ρ_n depend on (p, ϵ) through different scaling laws in different bands?
- **Lower bounds for Theorem 1's bounded-away regime.** The paper shows P(ρ_n > η(p, ϵ)) → 1 for F(p) < ϵ < 1, 0 < p < 1/3. Is η(p, ϵ) tight? Does ρ_n concentrate, or does it have a nontrivial limit distribution?
- **Critical boundary at ϵ = F(p).** The K ≥ 1 sub-band strengthening P(ρ_n(F(p), p) > η(p)) → 1 is striking. Does the same obstruction appear in the active interior, or only at the floor?
- **Lean 4 coverage.** The QRCP proof is fully verified in Lean; the CG proof is verified by a human (Zhangsong Li). Is it feasible to push the entire CG proof (including the moment-measure asymptotics and the Schur-complement critical-boundary analysis) through Archon?
- **Iteris on other Simons problems.** The Simons Workshop collection [3] contains many open questions beyond 2.4 and 4.3. Which subset is amenable to Iteris's explore-plan-execute pattern? Where does the pattern break?
- **Ablations.** What happens if Foundation agent is removed (no definition freezing)? If Review agent is removed (no claim-discipline)? If file-memory is replaced by conversation memory? The paper does not isolate these contributions.
- **Underlying-model dependence.** Iteris is implemented with GPT-5.5 (Codex). Does Iteris's success depend on the underlying model? Would GPT-4-class models, open-source models (DeepSeek-R1, Qwen3), or specialized reasoning models yield different qualitative conclusions?
- **Compute budget.** How many tokens, iterations, and wall-clock hours did each case study consume? What is the marginal value of each additional iteration? This is essential for understanding whether Iteris is a practical research tool.
- **Beyond computational mathematics.** Iteris is positioned for computational mathematics specifically because of its experimental + proof feedback loop. Can the architecture be adapted to pure mathematics (where feedback is purely proof-based) or to natural-science research (where feedback is purely experimental)?
- **Semantic alignment between Lean and paper.** The QRCP formalization required post-hoc alignment. Can Archon (or a successor) produce formal statements that match informal mathematical theorem statements without human curation? This is a generic challenge for autoformalization.
- **Counterexample extension to other pivoting rules.** Theorem 2 rules out QRCP quality guarantees under bounded coherence. Do similar obstructions hold for partial pivoting, Rook pivoting, or randomized pivoting rules?
