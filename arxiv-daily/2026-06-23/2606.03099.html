01

Methodology · Results

Methodology — three-tier memory

Working Memory ($M_{\text{work}}$) compresses history into a three-part digest when context nears the window limit. A fidelity verifier $\psi$ refuses to compress if metadata loss $L > \tau=0.5$ — without it, failure rate hits 61%.

S = Sdigest ⊕ Ssession ⊕ Sworking

Episodic Memory ($M_{\text{epi}}$) runs a post-hoc 5-axis audit on each candidate answer — anchor, constraint, target, completeness, decision — and re-fires differentiated retrieval if the audit fails.

Semantic Memory ($M_{\text{sem}}$) distills only successful trajectories (F1 > $\theta_{\text{conf}}=0.85$) into entity-stripped, reusable skills; merges new skills with cosine-sim > 0.8; injects above $\theta_{\text{retr}}=0.65$ as directional hints.

An MLLM controller runs a ReAct loop over tool calls — no parameters are updated.

Results — DISBench highlights

  • Best open combo — Qwen3-VL-32B-Instruct + 2B embedding: Overall EM +9.0%, F1 +8.2% vs. baseline.
  • Best closed combo — Claude-Opus-4.5: Overall EM/F1 = 36.9% / 57.8% (2B) and 34.4% / 58.5% (8B).
  • Stability finding — ImageSeeker's Inter-Event EM/F1 collapses from healthy → 3.1/15.5 when swapping 2B → 8B embedding; PhotoCraft holds steady. The bottleneck is reasoning over evidence, not retrieval quality.
  • Ablation (32B + 2B) — Full F1 = 33.6. Remove working → 28.9 (−4.7); episodic → 29.2 (−4.4); semantic → 29.6 (−4.0). All three components are necessary.
  • Failure-mode audit — Total errors 73 → 52; "reasoning-collapse" class 31 → 20, the largest reduction.
  • Direct retrieval floor — Qwen3-VL-Embedding achieves only ~10% MAP@3 / 10–12% Recall@3 on DISBench — end-to-end matching is not enough.
02

Strengths · Limitations

Strengths

  • Hits an under-served task — personal photo-corpus retrieval — with real user value.
  • Each memory tier has a clearly bounded role, mapped to a concrete failure mode (drift / isolation).
  • Generalises across 9 closed/open MLLMs, gaining +5 to +18 points without retraining.
  • Evidence is multi-layered: direct-retrieval baseline, full ablation, Venn error taxonomy, mechanism visualisations.
  • Ethics statement restricts use to the user's own visual memory; data is CC-licensed (YFCC100M).

Limitations

  • Validated only on DISBench — 122 queries with a 6.1% screening retention rate.
  • Skill library bounded by observed successes; long-tail / compositional queries remain untested.
  • Cascade failures from underlying ImageSearch / WebSearch (OCR, geo) are not analysed.
  • Token cost & latency of multi-step reflection + skill retrieval are not reported.
  • "Distill only successes" is empirical — no contrast with failure-repair or RLHF paths.
  • Cross-language / cross-cultural transfer not validated (English Flickr only).
03

Questions for Further Exploration

1
Skill-library scalability. When the skill corpus grows to thousands, does sentence-transformer cosine retrieval still suffice — or is hierarchical skill organisation needed?
2
Failure as signal. Can failure-mode clustering yield a reusable "anti-pattern" library that augments episodic reflection instead of being discarded?
3
Retrieve-or-refuse loop. When the audit finds no plausible anchor (a place the user never visited), can the agent gracefully clarify or abstain instead of continuing to search?
4
Multimodal skills. Skills today are text-shaped. Could distilled few-shot image exemplars encode visual patterns more faithfully?
5
Privacy under FL/DP. Does hierarchical memory remain useful under federated learning or differential privacy, and does entity-stripping actually prevent leakage?
6
Cross-domain transfer. Can this architecture move from photo search to long-video understanding or embodied tasks where long-horizon context is the same constraint?
