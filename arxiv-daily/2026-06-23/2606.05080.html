Persistence beats one-shot code quality — Claude Opus 4.6 leads AutoLab by +0.18 Avg@3 through 57 vs 12 iterations.

AutoLab: Can Frontier Models Solve Long-Horizon Auto Research and Engineering Tasks?
Zhangchen Xu, Junda Chen, Yue Huang, Dongfu Jiang, Jiefeng Chen, Hang Hua, Zijian Wu, Zheyuan Liu, Zexue He, Lichi Li, Shizhe Diao, Jiaxin Pei, Jinsung Yoon, Hao Zhang, Mengdi Wang, Radha Poovendran, Misha Sra, Alex Pentland, Zichen Chen  ·  UW · Stanford · UCSB · Princeton · NUS · Waterloo · MIT · NVIDIA · Google  ·  2026-06-03  ·  36 tasks  ·  17 models  ·  2,544 wall-clock hours  ·  8.6 B tokens
Key Metric
+0.18 Avg@3

Claude Opus 4.6 (0.68) over runner-up Gemini 3.1 Pro (0.50) on the main 36-task set — the largest capability separation in this benchmark family.

Why this matters

Reframes agent design — persistence and time-awareness, not raw coding skill, now define long-horizon capability.

Critical edge

Sustained-iteration-matters-most is partly tautological: the benchmark rewards models that iterate more, by construction.

01 Methodology

  • Task packaging. Each of 36 tasks ships a natural-language instruction, a containerized sandbox (CPU or single H100/L40S) with a correct-but-suboptimal baseline, a local evaluator, a sealed held-out verifier, a hidden reference solution, and a strict 2–12 h wall-clock budget.
  • Anchored scoring. Two schemes normalize raw metrics to [0,1] — log-stretch for performance tasks, linear for bounded-quality tasks — both gated by a minimum-improvement threshold to fight saturation.
  • Anti-hacking stack. Sealed verifiers · SHA-pinned immutable files · correctness gates · an adversarial auditor agent that probes for shortcuts during construction.
  • Harness. A single fixed Harbor + terminus-2 pipeline, 3 rollouts per (model, task), reports Avg@3 / Best@3 / Dominance. CPU on local Ryzen 9 9950X; GPU on Modal cloud sandboxes.
  • Failure coding. All 302 zero-score rollouts across 11 main-set models were manually classified into four mutually exclusive categories: Timeout/Context Exhaustion, Capability Gap, Instruction Violation, Others.

03 Strengths

  • Most comprehensive long-horizon benchmark — 4 categories × 4 task types × 4 languages.
  • Truly large-scale: 17 models, 2,544 wall-clock h, 8.6 B tokens, apples-to-apples protocol.
  • Calibrated continuous scoring + the most thorough anti-hacking stack in the literature.
  • 302-rollout failure-mode coding is unusually deep — surfaces actionable design targets.
  • Harness ablation directly attacks a chronic reproducibility problem in agent benchmarks.
  • Full benchmark + harness + task artifacts open-sourced (autolabhq/autolab).

04 Limitations

  • 36 tasks is a small absolute sample; per-category sub-scores (esp. 4-task CUDA) carry high variance.
  • Cost asymmetry — full reproduction needs ~2,544 h × GPU time; only well-funded labs can replay it.
  • Mixed CPU-local vs GPU-cloud sandboxes make wall-clock and cost numbers hard to compare.
  • Biased toward "code-edit-and-rebenchmark" workflows; no open-ended ideation or non-executable tasks.
  • Scoring is anchored to human references whose own quality caps measured headroom.
  • mini-swe-agent* is a custom system-prompt variant — fair, but a moving target.
  • Anti-hacking is strong but not formally proven; SHA-pinning leaves side-channels via measurement scripts.

05 Questions for further exploration

  1. Does the persistence advantage transfer from code/ML tasks to open-ended scientific ideation tasks with no executable verifier to drive iteration?
  2. Can a model be trained (e.g. via time-budget-conditioned RL) to develop explicit time awareness — and does that close the gap to Opus 4.6 without architectural changes?
  3. How sensitive are the headline rankings to scaling the benchmark from 36 to ~200 tasks, especially within the small CUDA sub-domain (n=4)?
  4. What is the minimum harness that preserves the ranking? A 0.43 swing implies we still lack a "minimal sufficient" agent harness for long-horizon evaluation.
  5. Could the calibrated scoring framework be extended to multi-objective tasks (latency–accuracy–cost Pareto fronts) to capture richer trade-offs?
  6. Is the AutoLab ranking driven by base-model reasoning or by RLHF / post-training for agentic tool use? An ablation isolating post-training would clarify.
  7. How much of the persistent-iteration advantage comes from longer context windows vs better state-tracking heuristics in the agent scaffold?
