SPINE: A Fault Injection Profiler for Quantized Neural Networks under Accumulated Faults ↗

Key metric

SDC rate, packed vs unpacked 4-bit
Packing eliminates zero-padding "fault sinks" — every flipped bit now hits a live weight, so silent corruption roughly doubles on matched layers.
Why this matters

Informs selective hardening for edge AI in radiation-prone environments (e.g. CubeSats), where memory layout and bit-position dictate vulnerability tradeoffs.

Critical edge

Real radiation hits all layers simultaneously — the per-layer isolation misses cross-layer error masking and amplification, so the "first-layer is fragile" ranking may not hold under multi-bit upsets.

Methodology

  • Target. Arm Cortex-M3 on Cypress PSoC 5LP — native binary, no FPU, byte-addressable.
  • Injector. GDB attaches to the deployed binary; flips one random bit per iteration in the layer's address range, in place.
  • Loop. Run dataset → compare to golden → if no SDC, keep the flip and accumulate → on SDC, reload clean weights and start a new accumulation.
  • Workloads. Three MLP topologies (T1: 840 weights, T2/T3: 3744) on SAT-6 aerial land-cover (32×32×4).
  • Layouts. Q8, Q4 unpacked (zero-padded nibbles), Q4 packed (two weights/byte via mask+shift) — 9 configs total.
  • Stats. Post-hoc Ruospo et al. formula with worst-case p=0.5; per-layer 99%-CL error margins.

Results & highlights

  • Depth stratification. Input-proximal (first) layer always has the highest SDC rate across all 9 configs; deeper layers grow more tolerant.
  • Bit position. SDCs concentrate in low-order data bits; sign bits and unused upper nibbles mostly crash the app (detectable, not silent).
  • Layout. Packed 4-bit degrades reliability vs. unpacked — SDC rate ~doubles, mean faults-to-error collapses.
  • Survival curves. Packed curves drop steeply; unpacked / deeper layers exhibit long survival tails — a single global "tolerance" is meaningless.
  • Topology. T1 (smallest, 2 layers) is most brittle in absolute count; T2/T3 tolerate more accumulated flips but the first layer always dominates the failure point.

Strengths

  • Targets the actual deployed binary — memory layout, alignment, addressing artifacts are in the signal.
  • Stop-on-misprediction + clean-reload makes per-layer profiles well-defined.
  • Memory-layout axis (Q8/Q4U/Q4P) is rare in FI literature; the "packing hurts" finding is a useful corrective.
  • Bit-position split between silent low-bit corruption and detectable sign-bit crashes directly informs ECC vs. parity choices.
  • Statistical validation via the Ruospo formula is more rigorous than count-only FI papers.

Limitations

  • One dataset (SAT-6), one model family (small MLPs) — CNNs / ViTs untested.
  • One MCU (Cortex-M3, byte-addressable, no FPU) — may not transfer to RISC-V vector, DSPs, NPUs.
  • Single-bit, uniformly-random fault model — no MBUs, no temporal clustering, no wearout.
  • No on-hardware radiation validation (heavy-ion / proton / laser).
  • No public code release mentioned — limits reproducibility until upstreamed.
  • Layer isolation is itself a modeling choice — real upsets hit all layers simultaneously.

Questions for further exploration

  1. Do per-layer vulnerability rankings survive on convolutional or attention-based QNNs — do early-conv weights still dominate?
  2. Can the SPINE profile serve as a cheap proxy for heavy-ion test campaigns — predicting ground-test SDC rates within margin?
  3. What is the optimal 4-bit packed representation (symmetric quant with explicit sign, two's-complement) that recovers the zero-padding fault-sink effect?
  4. How do multi-bit upsets and temporally correlated flips shift the survival curves measured here?
  5. Can the layer-isolation assumption be relaxed to capture cross-layer error masking/amplification without exploding campaign size?
  6. Where does the energy–latency–reliability Pareto frontier land when selective weight-reloading is implemented on the PSoC 5LP — vs. uniform refresh, ECC, or radhard memories?
