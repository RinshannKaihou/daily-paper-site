Closed-loop self-evolution lifts base models yet plateaus 8–13 pp below oracle — only Gemma 12B RevisionSE nearly closes the gap.

On the Generalization Gap in Self-Evolving Language Model Reasoning

Qi · Baby · Baby · Yuan · Tomkins · Vu · Juan · Rashtchian / Google Research · Harvard · Virginia Tech / 2026-06-23
self-evolution closed-loop supervision knights & knaves DPO generator–verifier game
Key Metric
+13.8 pp
Gemma 3 4B IT on Knights & Knaves (2–8 people): 31.0% → 44.8% after CurriculumSE. The 12B + RevisionSE reaches 52.8% vs. oracle 53.6% — 98.5% of supervised performance.
Why this matters

Defines an empirical ceiling for self-evolving LLMs: internal verification sharpens existing solution modes but cannot replace true supervision — a sober reference point for RLVR-without-ground-truth pipelines.

Critical edge

The "12B nearly matches oracle" claim rests on a single model family: only Gemma-3-IT was evaluated under RevisionSE, while Qwen only ran SimpleSE — so the scaling conclusion leans on one family and one variant.

Methodology 方法

  1. Strict closed loop. Only an unlabeled prompt set D and a single base model M are allowed in. All traces, rewards, and preference labels come from M itself — instantiated twice as Generator G and Verifier V.
  2. Four SE strategies unified under one generator–verifier game: SimpleSE (single-round τ-thresholded voting), RevisionSE (multi-turn critic + revision), IterativeSE (repeated offline DPO), CurriculumSE (KK-23 → KK-45).
  3. Offline DPO on extracted preference pairs (yw, yl); reference policy held fixed. No online RL rollouts.
𝓛DPO(πθ; πref) = −𝔼 log σ( β log πθ(yw|x) / πref(yw|x) − β log πθ(yl|x) / πref(yl|x) )

Models: Gemma-3-IT (1B/4B/12B, 27B roofline) + Qwen-2.5-Instruct (7B/14B). Primary testbed: Knights-and-Knaves 2–8 people. Secondary: OpenThoughts3 → GSM8K, MATH500, MATHHard, TabMWP.

Strengths 优势

  • +Clean, controlled design — KK is deterministic, contamination-resistant, difficulty-tunable.
  • +Unified framework across four SE variants with consistent DPO training.
  • +Multi-scale coverage: 1B / 4B / 12B / 27B Gemma-3 and 7B / 14B Qwen-2.5.
  • +Practical engineering insight: τ ≈ 0.7 sweet spot; verifier compute > generator compute.
  • +Honest, sharp conclusions; conflict-of-interest disclosed; std. dev. reported over 4 seeds.

Limitations 局限

  • −Closed-loop only — excludes tools, external verifiers, RLVR with execution feedback.
  • −Most numerical claims live on Knights-and-Knaves alone.
  • −RevisionSE is evaluated on Gemma only; Qwen only ran SimpleSE.
  • −INTUITOR / AZR comparison is illustrative, not apples-to-apples (different start models).
  • −Open-ended reasoning gains (~+1.5–2.9%) sit near the noise floor; no paired significance tests.
  • −No theoretical account of why the 8–13 pp gap persists.

Questions for Further Exploration 延伸问题

01Can a small labeled set bootstrap the internal verifier, then run closed-loop SE on the rest — and does the hybrid close more of the gap than either alone?
02Does the "12B nearly matches oracle" trend continue at 27B / 70B, or saturate? Would independent teams reproduce it on Qwen / Llama?
03Can Pass@1 vs. Pass@32 be formalized — does closed-loop SE provably reduce policy entropy without expanding its support?
04If the verifier is allowed chain-of-thought or self-consistency, does the 8–13 pp gap shrink — or is it fundamental?
05Can a per-instance adaptive scheme (more verifier passes when generator entropy is high) close more of the gap at equal compute?
06Does the gap persist on truly open-ended domains (creative writing, dialogue, science QA) where even humans disagree?
