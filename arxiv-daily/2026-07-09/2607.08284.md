> **一句话总结：** 本文提出 PredicateLongBench，通过"在长上下文中找出满足谓词约束的最长连续子序列"这一算法上简单但富有挑战的任务，系统性地沿计算量、量词复杂度、对抗诱饵、搜索空间大小和上下文结构等多个难度维度进行评测，发现即便是最强前沿模型（Opus 4.6、GPT-5.4、Gemini 3.1）在难度提升后准确率也会从高位暴跌至接近 0%，开源模型则几乎完全失败。

> **TL;DR:** PredicateLongBench is a new long-context benchmark that asks models to find the longest contiguous subsequence of words in a ~128K-token list (or up to 365K tokens from LongBench v2 documents) satisfying predicate constraints (e.g., lexicographic ordering). Along five axes of difficulty — predicate arity (computation), existential-to-universal quantifier complexity, adversarial near-miss / multi-list decoys, search-space size, and context structure — even the best models collapse: for example, Opus 4.6 high's lexicographic locate baseline of 97% (synthetic) drops to 1% once 8 scattered near-sorted decoys are inserted, and all models are ≤2% on that decoy variant.

## 核心贡献

- **提出 PredicateLongBench 基准**：一类算法上简单（查找满足谓词约束的连续子序列）但对前沿 LLM 极具挑战的长上下文任务，且全程不需要 LLM 生成上下文或充当判官，避免了判官偏差。
- **系统性地定义并探索多个难度维度**：计算量（谓词元数，1 元 vs 2 元）、量词复杂度（∃ 存在量词 vs ∀ 全称量词）、对抗诱饵（near-miss / near-sorted decoys 与多列表 decoys）、搜索空间大小（在 token 预算不变时增大候选词数量）、以及上下文结构（decoys 聚集 vs 散布）。
- **提供两套互补的生成流水线**：全合成（固定长度随机字符串，约 128K tokens）与真实文档（从 LongBench v2 抽取并过滤，最高 365K tokens），后者保留自然语言的词法分布。
- **揭示关键发现**：上下文的结构（而非仅 token 数）主导了任务难度——把散布的 decoys 聚集到一起后，模型性能几乎完全恢复甚至超过基线；增大搜索空间（保持 token 数不变）同样导致性能从 92% 暴跌到 10%。
- **全面评测前沿模型**：覆盖 GPT-5.4、Gemini 3.1 Pro、Opus 4.6（含不同 reasoning effort）、MiniMax 2.7、GLM 5.1、Qwen 3.5 397B，展示闭源与开源模型之间的显著鸿沟。

## Key Contributions

- **PredicateLongBench, a simple-but-hard long-context benchmark**: tasks ask the model to identify the longest contiguous subsequence of a word list satisfying predicate constraints (unary prefix/suffix/containment, or binary lexicographic ordering), with no LLM-generated context or LLM judge required.
- **A systematic taxonomy of difficulty axes** that prior benchmarks probe only in isolation: computation (predicate arity), quantifier complexity (∃ → ∀), adversarial decoys (near-miss, near-sorted, multi-list), search-space size (tokens held constant, candidate count scaled up), and context structure (scattered vs clustered decoys).
- **Two complementary generation pipelines**: a fully synthetic one using random base-26 strings (≈128K tokens) and a real-world one sampling words from filtered LongBench v2 documents (up to 365K tokens), preserving natural lexical distribution.
- **The context-structure insight**: clustering scattered decoys near the target restores performance to or above the no-decoy baseline, demonstrating that the adversarial power of decoys is tied to long-context scattering rather than their mere presence.
- **The search-space insight**: scaling the candidate word count from 26K to 60K while holding token budget fixed drops GPT-5.4 high from 92% to 10%, showing token count alone is an incomplete difficulty proxy.

## Methodology

**Task formulation.** The context is a space-separated list of words. Given a predicate class H and predicates f1,…,fj ∈ H, the task is to find the largest k such that some contiguous subsequence w = (w1,…,wk) satisfies fi(w)=1 for all i. Two families are studied:
- *Unary predicates* (arity 1): prefix/suffix/containment string matches on individual words.
- *Binary predicates* (arity 2): lexicographic ordering, decomposed into pairwise lex conditions on adjacent words.

**Difficulty axes and variants.** Each baseline is perturbed along a single axis (Table 1):
- *Computation*: unary (arity 1) vs lexicographic (arity 2).
- *Quantifier complexity*: ∃-query (any/all satisfying sequence) vs ∀-query (all sequences, or the *longest* one). The ∀ "all"/"longest" variants force global checking even though the unique answer is unchanged.
- *Adversarial decoys*: (a) near-miss decoys — for unary, 5% of words per position are edited to satisfy one predicate while preserving a unique length-5 solution; for lex, eight length-5 near-sorted runs with a single swapped interior pair are inserted. (b) Multi-list decoys — the list is split into 20 sublists and a valid cross-boundary sorted run is planted at every boundary, but the model must return a within-sublist run.
- *Search space size*: shorter words so 60K candidates fit in the same 128K-token budget (vs the usual 26K).
- *Context structure*: the same near-sorted decoys are clustered within a small radius ρ of the target instead of scattered uniformly.

**Generation pipelines.**
- *Synthetic*: sample N distinct length-L random strings over {a…z} (L=8 for lex, L=12 for unary) by drawing integers from {0,…,26^L−1} and base-26 encoding; plant the target block (with flanks that block extension) at a random index; run a swap-based fixup loop to destroy any chance lex run ≥ k, guaranteeing a unique solution. Target ≈128K Qwen tokens, 100 examples/variant.
- *Real-world*: draw documents from LongBench v2, filter out artifacts (pre-existing long sorted runs, repetitive vocab, HTML, etc.), leaving ~120 docs (100 used; 93 with <170K tokens for open-source models); tokenize each into a flat word list used directly as context.

**Evaluation.** Accuracy is exact-match of the output subsequence. All variants use k=5 (synthetic) or k = longest in-document lex run (6–10 for LongBench). Max output tokens 16K default (128K/64K in scaling studies). Models: GPT-5.4 (reasoning high/none), Gemini 3.1 Pro preview (high/low), Opus 4.6 (high/none), MiniMax 2.7, GLM 5.1, Qwen 3.5 397B.

## Results & Highlights

- **Binary predicates are far harder than unary.** On the baseline ∃-query, Opus 4.6 high scores 87% on the unary task but lexicographic locate (synthetic) is also strong at 97%; however open-source models collapse — Qwen 3.5 397B scores 45% on unary but 0% on lex-locate synthetic (and 0% on LongBench lex-locate), a 45-point gap driven by predicate arity.
- **Scattered near-sorted decoys devastate every model.** With just 8 scattered near-sorted decoys on the synthetic lex task, *no model exceeds 2%* (Opus 4.6 high 1%, GPT-5.4 high 0%, Gemini 3.1 high 2%), down from the 97% / 62% / 43% baselines — a 60–95 point collapse.
- **Unary ∃→∀ with 5% decoys is catastrophic.** Adding 5% per-position distractors to the universal-query unary task drops Opus 4.6 high from 87% to 7% and GPT-5.4 high from 95% to 5%.
- **Search space matters independently of token count.** Holding the 128K-token budget fixed but expanding the candidate list from 26K to 60K words drops GPT-5.4 high on the lex baseline from 92% to 10% (82-point drop) — strong evidence that raw token count is an incomplete difficulty proxy.
- **Clustering decoys fully restores performance.** Moving the same near-sorted decoys next to the target recovers Opus 4.6 high on synthetic lex from 1% (scattered) to 98% (clustered), and on LongBench lex from 49% to 93%; GPT-5.4 high goes 0%→70% (synthetic) and 36%→68% (LongBench) — confirming the difficulty is structural, not due to decoy existence.
- **Opus 4.6 dominates.** Opus 4.6 high is the best model on 9 of 12 task variants, though it still suffers the same cliff-edge drops under scattered decoys.
- **Open-source models essentially fail** beyond the unary baseline and the ∀-quantifier variant, registering single-digit or 0% accuracy on every lexicographic and decoy variant.
- **Longest-sequence (∀) is much harder than locate-k.** Lex-longest synthetic: Opus 4.6 high 23% vs its 97% locate baseline; GPT-5.4 high 12% vs 62%.
- **Models fail at locating, not reproducing.** Across all models, when the answer is wrong the first output word is correct 0% of the time, indicating a localization failure rather than a copying failure.

NVIDIA is an active contributor to long-context LLM research and benchmarking (the related RULER benchmark, cited heavily here, is also NVIDIA-affiliated), which lends the work strong domain credibility. The framing as a preprint (arXiv:2607.08284v1, 9 Jul 2026) is appropriately modest.

## Critical Assessment

- **Novelty (genuine).** The framing of long-context difficulty as *explicitly separable axes* (computation/arity, quantifier, decoys, search space, context structure) and the demonstration that the *same decoys* become benign when clustered is a real conceptual contribution. The search-space axis — disentangling candidate count from token count — is underexplored in the literature and well-motivated here. However, individual ingredients (NIAH, multi-hop, decoys, lex-sorting tasks) are not new; the novelty is the synthesis and the controlled single-axis perturbations.
- **Rigor (strong).** The single-axis perturbation design is methodologically sound — each variant changes one thing relative to a baseline, and the synthetic pipeline guarantees a unique solution via a swap-based fixup loop. The "first-word accuracy 0% on wrong answers" diagnostic is a nice falsifiability check that pins the failure mode to localization. Sample sizes (100/93 examples per variant) are reasonable but not large; no confidence intervals or variance is reported.
- **Reproducibility (excellent).** Full algorithms are given; tasks are cheap to generate (random strings + deterministic predicates); no proprietary judge is needed. The main reproducibility gaps are (a) exact prompts are deferred to the appendix rather than fully specified in the main text, and (b) closed-source model traces cannot be inspected, which the authors acknowledge and work around via OSS traces.
- **Significance (moderate-to-high).** It convincingly shows that "good on RULER/HELMET" does not imply robust long-context reasoning: frontier models at 90%+ on baselines collapse to single digits under scattered decoys or larger search spaces. This is a useful, sharply diagnostic complement to existing benchmarks.
- **Red flags / overclaiming.** A few items to flag:
  - *Weak/underspecified baselines for OSS models.* Qwen 3.5 397B, GLM 5.1, MiniMax 2.7 score 0% on most lexicographic baselines — at 0% it is impossible to tell whether the axis being tested is the cause or whether the model simply cannot do the base task at all. This conflates "the axis hurts" with "the model was never competent," weakening axis-level claims for OSS models.
  - *Footnote-artifact in the Opus ∀ gain.* The authors transparently note that Opus's apparent ∃→∀ *improvement* (87%→99%) is an artifact of refusals on the baseline being reclassified — honest, but it means the unary quantifier axis is only cleanly demonstrated on models other than the strongest one.
  - *Missing ablations.* There is no ablation on k (fixed at 5 for synthetic), on number of decoys D (fixed at 8), or on the decoy proximity radius ρ for the clustered variant — so the dose-response curve of "how much scattering hurts" is uncharacterized. The multi-list decoy uses M=20 with no sweep.
  - *Token-budget confound for failures.* The authors attribute much of the failure to hitting the 16K output-token budget; while they show scaling output tokens helps GPT-5.4, this means some reported "model can't do it" numbers are partly "model ran out of budget," muddying the difficulty-axis interpretation.
  - *Limited model diversity at the top.* Only one Anthropic, one OpenAI, one Google model; no reasoning-specialized comparison of, e.g., different Opus/GPT reasoning tiers beyond high/none.
  - *No human baseline.* It would strengthen the claim of "conceptually simple" to report human or exact-algorithm performance and runtime.

## Strengths

- Conceptually clean, judge-free tasks with fully specified, cheap generation — high reproducibility and no LLM-judge bias.
- The clustered-vs-scattered decoy comparison is a striking, well-controlled demonstration that *context structure*, not information content, drives long-context hardness.
- The search-space axis (token budget held constant) isolates an underappreciated difficulty factor and yields a large, clean effect (92%→10%).
- Honest reporting of artifacts (the Opus refusal footnote, the 0%-first-word diagnostic) and of failure modes (token-budget exhaustion, OSS models degenerating into repetition).
- Broad model coverage including multiple reasoning-effort tiers, enabling inference-compute analysis (Figures 9–10).

## Limitations

- Quantifier-complexity experiments never construct cases with *multiple* satisfying sequences, so the ∀ axis is only partially stress-tested (explicitly acknowledged by the authors).
- No sweeps over key hyperparameters k, D (decoy count), M (sublist count), or ρ (cluster radius), leaving dose-response uncharacterized.
- Open-source models sit at 0% on most lexicographic variants, making axis-level conclusions OSS-uninformative.
- Closed-model reasoning traces are invisible, so root-cause analysis relies on OSS analogues.
- Output-token-budget exhaustion partly confounds "task too hard" with "ran out of budget."
- k=5 (synthetic) is small; behavior at larger k (longer retrieval) is untested.
- The benchmark measures a narrow family of retrieval/predicate tasks; generalization to QA, summarization, or code reasoning is not established.

## Relevance

对关注长上下文 LLM 评测与基准构建的研究者高度相关：PredicateLongBench 提供了一套无需 LLM 判官、可确定性生成的难度可控探针，其"上下文结构主导难度"和"搜索空间独立于 token 数"两个发现可直接用于诊断和改进现有长上下文模型与 RAG 检索管线。

## Questions for Further Exploration

- How does performance degrade as the target length k grows beyond 5, or as the decoy count D and cluster radius ρ are swept — is there a smooth dose-response or a sharp phase transition?
- Would adding *multiple* genuinely satisfying sequences to the ∀ variants reveal a different (likely harder) failure mode than the single-solution setup used here?
- Can the clustered-vs-scattered decoy gap be closed by training-time or inference-time interventions (e.g., structured prompting, scratchpad chunking, KV-cache reordering)?
- Is the search-space effect specific to lexicographic sorting, or does it replicate for other predicate classes (set membership, numeric comparison, k-ary constraints with k>2)?
- What do the closed-model hidden reasoning traces actually do — could a distilled probe or logit-lens analysis localize where the localization failure occurs?
- Does scaling output-token budget eventually let decoy-variant accuracy converge to the baseline, or is there a persistent gap (as hinted by Figure 10) that no compute level closes?
- How do retrieval-augmented or agentic (multi-pass) strategies perform on the scattered-decoy variants — can explicit retrieval beat end-to-end long-context reasoning here?
