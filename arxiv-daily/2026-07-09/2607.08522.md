> **一句话总结：** 本文提出基于分组序贯检验（Group Sequential Testing）的自适应评测框架，通过在 Open VLM Leaderboard（206 个 VLM、31 个基准）上引入可配置的停止规则（功效/等价/精度/边际收益等），在保持统计显著性的同时将单模型评测计算成本降低 80%（±2.5 分 CI），并使能可靠区分的模型对比例从固定采样的 55% 提升到 76%。

> **TL;DR:** The authors argue for abandoning fixed-size benchmarks and adopt group sequential testing (Pocock spending function) for adaptive model evaluation on the Open VLM Leaderboard (206 VLMs, 31 benchmarks). With user-defined stopping rules, the framework cuts single-model evaluation cost by ~80% (±2.5-point CI) versus full-size runs, and reliably distinguishes 76% of 1K top-50 model pairs versus only 55% under a fixed 1.2K-sample heuristic baseline — while making any reliability loss explicit rather than hidden.

## 核心贡献

- 提出并倡导在模型评测领域摒弃固定样本量基准，转向基于序贯检验（sequential testing）的自适应评测范式，使评测成本与评测目标真正对齐。
- 构建了一个统一的、具备统计严格性的自适应评测框架，将临床/质控领域成熟的分组序贯检验（Group Sequential Testing, GST, Pocock 花费函数）移植到多数据集基准评测，并保留任意"偷看数据"（peeking）下的 I 类错误控制。
- 提供一整套可组合的停止规则：功效停止、等价区间停止、基于 MDES 的精度停止、阈值跨越停止、无效性停止、边际收益停止，覆盖排序、模型选择、模型替换等真实场景。
- 在 Open VLM Leaderboard（206 个 VLM、31 个多模态基准、约 14,400 样本）上系统量化了效率-可靠性权衡，给出可复现的样本节省曲线与实证结论（代码开源于 github.com/OfirArviv/adaptive-eval）。

## Key Contributions

- A clear call to abandon fixed-size benchmarks in favor of statistically grounded adaptive evaluation that aligns evaluation effort with the actual decision objective (debugging, ranking, model replacement).
- A framework that imports group sequential testing (GST with the Pocock spending function, via the gsDesign3 R package wrapped in Python) into multi-dataset model evaluation, preserving valid p-values and confidence intervals even under repeated peeking.
- A library of composable stopping rules — efficacy, equivalence-margin, precision/MDES-based, threshold-crossing, futility, and diminishing-returns — that practitioners can mix per use case.
- An empirical analysis on the Open VLM Leaderboard (206 VLMs across 31 multimodal benchmarks, ~14,400 examples) quantifying the efficiency–reliability trade-off for single-model scoring, pairwise comparison, and ranking tasks.

## Methodology

The benchmark is treated as a set of n datasets D = {D1,…,Dn}, with overall model score defined as a mean-of-means (Eq. 1), giving each dataset equal weight regardless of size. Pairwise comparison uses the observed difference δ̂(D) = SA(D) − SB(D) (Eq. 2).

Reliability is formalized via hypothesis testing (H0: δ = 0 vs H1: δ > 0) and confidence intervals. The core problem the paper addresses is "peeking": naively re-checking p-values/CIs as data streams in inflates the Type I error rate. The fix is group sequential testing: the evaluation is split into t stages of batch size b, giving cumulative sample Nk = kb. At each stage k a Z-statistic Zk = (S_A − S_B) / sqrt(σ̂²/Nk) is computed (Eq. 6) and compared against a stage-adjusted threshold αk. The Pocock spending function uses a single constant critical value c across all stages so the overall Type I error equals the nominal α. This requires independent observations and approximately normal test statistics (justified by CLT).

On top of GST-valid inference the framework layers six stopping rules, each tied to a real evaluation objective: efficacy (significance reached), equivalence-margin (models within a pre-defined margin, e.g. ±2), precision/MDES (CI narrow enough for the minimum detectable effect), threshold-crossing (model confidently above/below a cutoff), futility (a meaningful difference is unlikely), and diminishing-returns (marginal precision gain falls below a threshold). Users may combine rules.

Implementation uses the gsDesign3 R package for sequential design, integrated into a Python pipeline. Defaults (not tuned): initial batch 600, batch size 100, β = 0.9, Pocock spending function. Multi-model comparisons are handled as pairwise comparisons with a simplifying note that multiple-testing correction is left informal. Scoring uses the leaderboard's released predictions scored by an LLM-as-a-Judge (Llama 3.1 8B).

## Results & Highlights

- Single-model cost: accepting a ±2.5-point CI half-width saves ~80% of evaluation samples; accepting ±3 points needs only ~15% of samples, ±2 points needs ~30%, and ±1.5 points needs ~50% (averaged over 206 models, 10 seeds, Open VLM Leaderboard). Going from 8K examples (55%) to the full 14,400 (100%) only narrows the CI from 2.9 to 2.7 points — a 0.2-point gain at ~2× the cost.
- Heuristic-baseline weakness quantified: the common practice of sub-sampling "1K or fewer examples" yields CIs exceeding ±5.5 points, often too wide for real decisions, whereas the framework exposes the CI directly.
- Pairwise comparison (1K model pairs from top-50 models, 95% confidence): pairs differing by more than 2 points save ≥60% of evaluation effort and stop early; pairs differing by less than ~1.2 points require the full dataset (~24% of pairs) — and these are precisely the arguably-equivalent comparisons.
- Head-to-head with a fixed-size baseline (1,000 examples/dataset pre-sample, here 1.2K total, bootstrapped and so actually more statistically powerful per sample): the fixed heuristic reliably differentiates only 55% of 1K model pairs at 95% confidence, versus 76% with the adaptive framework. Crucially, the fixed method's failures are invisible to the user.
- Case study — efficient eval with statistical guarantees: ranking 5 models from the top-15 (≤5-point spread) with a ±2 equivalence margin at α = 0.05 used only 60% of the examples (2.4 of 20 comparisons per run remained indistinguishable and needed the full benchmark).
- Case study — meaningful change for deployment (one-sided ≥2-point improvement, 95% confidence): over 100 random sampled pairs from the top-15, the approach used only 63% of the examples versus a fixed-sample approach.
- Case study — model selection with a 60-point threshold filter and ±2-point CI stopping: across the full benchmark (10 seeds) only 30% of the total sample budget was consumed and 86 candidate models were filtered out early as underperforming; without the threshold filter, 50% was used.

## Critical Assessment

- Novelty: The core statistical machinery (Pocock group sequential testing, 1977) is decades old and explicitly borrowed from clinical trials; the novelty lies in the application and the unified set of stopping rules for model evaluation, not in new theory. The framing as a "call to eschew fixed-size evaluation" is a position-paper-style contribution layered on an empirical study.
- Rigor: The comparison against the fixed-size baseline is reasonably fair — the authors even grant the baseline an advantage by using bootstrap, which is more statistically powerful per sample than their sequential test, and the adaptive method still wins on reliability (76% vs 55%). However, the evaluation is limited to a single benchmark suite (Open VLM Leaderboard) and LLM-as-a-Judge scoring with Llama 3.1 8B, so generalization to text-only LLMs or to natively-scored benchmarks is untested. The smallest, highest-variance datasets (e.g., HallusionBench's 951 samples) dominate the mean-of-means variance, a real data quirk the authors honestly flag and that drives much of the diminishing-returns effect.
- Reproducibility: Defaults are fully specified (init 600, batch 100, β = 0.9, Pocock, 10 seeds), the leaderboard data commit hash is pinned (VLMEval/OpenVLMRecords commit dbc5e10), and code is promised. Good by workshop/preprint standards.
- Significance: Practically useful — the 80% savings and the "heuristic 1K samples give ±5.5 CIs" result are immediately actionable for anyone running VLM leaderboards. The conceptual point that hidden reliability loss (fixed sub-sampling) is worse than transparent reliability loss (adaptive stopping) is well made.
- Red flags / overclaiming: Some phrasing overstates breadth ("for diverse tasks… enabling adaptive evaluation" — but it is only shown on VLMs). The promise of "statistical guarantees" leans on assumptions (i.i.d. samples, approximate normality, pre-specified number of looks) that the authors themselves list as limitations but that practitioners may overlook; results vary across runs, reducing reproducibility unless seeds are fixed. The multiple-comparison problem for ranking many models is explicitly hand-waved ("we simplify this aspect"), which is a notable gap for a paper whose headline use case includes ranking.

## Strengths

- Provides a principled, assumption-explicit alternative to the rampant, silent practice of arbitrarily sub-sampling benchmarks, and quantifies exactly what is lost (the ±5.5-point CI result for 1K-sample heuristics is a memorable, citable finding).
- Stopping rules are modular and map cleanly to real engineering decisions (deployment gate, candidate filtering, equivalence testing), making the framework genuinely usable rather than theoretical.
- Genuinely fair baselines: they deliberately give the fixed baseline a stronger per-sample test (bootstrap) and still demonstrate the adaptive method's advantage in both reliability and cost.
- Honest about the i.i.d./normality/reproducibility limitations and about which comparisons are simply "futile."

## Limitations

- Validated on a single benchmark family (Open VLM Leaderboard) with LLM-as-a-Judge scoring; behavior on text-only LLM benchmarks, on natively scored tasks, or on non-mean-of-means aggregation is not shown.
- Relies on i.i.d. and CLT-normality assumptions; benchmarks with clustered or adversarial example ordering could violate these, and sequential methods trade some sample efficiency to relax them.
- Pairwise comparisons scale poorly and multiple-testing correction for full ranking is acknowledged but left informal/simplified — a gap for the headline ranking use case.
- Users must specify equivalence margins / MDES / thresholds, which requires statistical expertise the paper does not fully lower the barrier to.
- Run-to-run variance from adaptive stopping demands strict seed control for reproducibility; the framework adds modest computational overhead from repeated statistical checks.
- No comparison against other modern sample-efficient methods (tinybenchmarks, Anchor Points, active testing) on the same data — related work positions them but does not benchmark them head-to-head on cost/reliability.

## Relevance

该工作直接服务于"高效/低成本模型评测"这一核心研究方向，与 LLM/VLM 基准缩放、统计显著性评测以及评测成本-可靠性权衡高度相关，对构建大规模评测流水线、leaderboard 运营与候选模型筛选具有可落地的指导价值。

## Questions for Further Exploration

- How does the framework behave on text-only LLM benchmarks or natively-scored tasks (exact-match, pass@k) where the LLM-as-a-Judge variance term disappears?
- What is the right multiple-testing correction for full K-model ranking, and how much efficiency is lost when it is properly applied instead of the simplified pairwise scheme?
- Can the diminishing-returns stopping rule be made adaptive to the per-dataset variance structure, so that the small high-variance datasets (e.g., HallusionBench) are sampled more rather than capping the overall CI?
- Does the i.i.d. assumption hold in practice for ordered benchmarks (e.g., difficulty-sorted or curriculum-ordered evaluation sets), and how robust is the Pocock threshold to such violations?
- A head-to-head, same-data comparison against tinybenchmarks / Anchor Points / active testing on both cost and reliability is missing — where does sequential testing win or lose?
