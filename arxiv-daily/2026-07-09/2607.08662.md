> **一句话总结：** WebSwarm 提出一种渐进式递归委派的多智能体框架，通过将局部目标与搜索模式（atom/deep/wide/entity_collect）耦合的智能体节点递归展开，并以 Web 信息结构探测和兄弟节点经验复用作为引导信号，在 BrowseComp-Plus、WideSearch、DeepWideSearch、GISA 四个基准上稳定优于 ReAct 单智能体及多种多智能体基线，在 BrowseComp-Plus 上较 GLM-4.5 的 ReAct 提升 17.50 个准确率点。

> **TL;DR:** WebSwarm is a progressive recursive-delegation multi-agent framework for complex web information seeking, where each dynamically instantiated search node couples a local objective with one of four search modes (atom/deep/wide/entity_collect); guided by a Web-Probing Agent that detects whether evidence is centralized vs. distributed and by scout-derived sibling-node experience, WebSwarm beats ReAct and six multi-agent baselines across BrowseComp-Plus, WideSearch-EN, DeepWideSearch-EN, and GISA — for example +17.50 ACC on BrowseComp-Plus (68.00 vs 50.50) and +11.77 Item F1 on DeepWideSearch-EN (58.40 vs 46.63) under GLM-4.5, with the largest gains on hard samples.

## 核心贡献

- **渐进式递归委派框架**：提出一种不同于"一次性分解 + 固定协作拓扑"的多智能体范式，将任务分解、递归展开和协作结构在推理过程中协同构建。每个搜索节点都是一个智能体，同时接收局部目标 q_v 和搜索模式 m_v，节点可自行调用 web 工具求解，也可递归生成并委派子节点，子节点完成后将证据向上反馈，支持父节点进一步展开、修订或聚合。
- **目标-模式耦合的四种搜索模式**：定义 M = {atom, deep, wide, entity_collect}，分别针对原子事实查找、未知目标的多约束推理、跨实体集的并行覆盖、未知集合边界的高召回高精度枚举四种典型 web 搜索瓶颈，让同一递归过程内不同子任务能用最合适的求解协议。
- **Web 信息结构引导的展开（Web-Probing Agent）**：通过轻量级探测判断任务相关证据在 web 上是集中在少数聚合页（centralized）还是沿时间/品牌/事件等维度分散（distributed），从而决定后续节点沿哪个维度展开，避免过度分解或错维度切分。
- **兄弟节点经验复用（Scout Experience）**：对 wide 节点生成的同质兄弟节点，先运行少量 scout 节点，从其完整轨迹中抽取过程级经验（可靠 query 模板、权威源、失效路径等），注入到后续兄弟节点的求解上下文，提升一致性与可靠性；该经验仅在单实例内复用，保持评估样本独立性。
- **四基准全面验证 + 深度分析**：在 BrowseComp-Plus、WideSearch、DeepWideSearch、GISA 上系统比较 ReAct 与 Swarm-Agent、Flash-Searcher、Table-as-Search、ROMA、InfoSeeker 等多智能体基线，并提供递归委派/搜索模式/引导信号消融、任务难度分层、web 工具效率、跨骨干模型（GLM-4.5、Qwen3-32B、Qwen3.5-35B）泛化等多维分析。代码已开源（https://github.com/songxiaoshuai/WebSwarm）。

## Key Contributions

- **Progressive recursive delegation framework** that, unlike plan-then-orchestrate systems, jointly constructs task decomposition, recursive expansion, and agent collaboration during inference. Each node is itself an agent carrying a local objective plus a search mode; it can solve locally, delegate child nodes, and return evidence upward to drive further expansion, revision, or aggregation.
- **Objective–mode coupling with four search modes** (atom / deep / wide / entity_collect) that map onto four characteristic web-search bottlenecks (focused fact lookup, unknown-target multi-constraint reasoning, fan-out structured collection, open-set enumeration), so heterogeneous local objectives get matched solving protocols within one recursive process.
- **Web-Probing Agent** that performs a lightweight topology scout (centralized / centralized_with_gaps / distributed, with an organization dimension) and conditions downstream delegation on the *actual* organization of web evidence — not on the query's surface semantics.
- **Sibling-node experience reuse** where a small scout batch under a wide node exposes full trajectories, from which process-level skill (reliable queries, authoritative URLs, dead ends) is extracted and injected into remaining homogeneous siblings, improving consistency without leaking across evaluation samples.
- **Comprehensive empirical validation** on four benchmarks (BrowseComp-Plus, WideSearch, DeepWideSearch, GISA) against ReAct and six multi-agent baselines, accompanied by ablations of recursive delegation, search modes, web-probing, and experience reuse, plus difficulty-stratified, tool-efficiency, and cross-backbone analyses. Code and prompts are released.

## Methodology

WebSwarm organizes complex web information seeking as a recursive delegation tree T = (V, E) built at inference time.

**Recursive Search Delegation.** The root node v0 receives the original task q0. Each non-root node v is an agent receiving a local objective q_v and a search mode m_v; an edge (u, v) denotes parent u delegating to child v. Node v maintains child results R_v and, based on q_v, m_v, and R_v, decides whether the current evidence is sufficient. If not, it generates a set of child delegations C_v = {(q_i, m_i)}, where the key design choice is to emit **objective–mode pairs**, not just subtasks — so each downstream node inherits a solving protocol matched to its goal. When a child returns, the parent updates R_v ← R_v ∪ {r_i} and either generates new sub-delegations, revises direction, or aggregates upward via r_v = Aggregate(q_v, R_v). This alternates top-down delegation with bottom-up evidence feedback, letting the tree unfold progressively.

**Four Search Modes.**
- *atom* — a focused ReAct agent that directly calls web tools for narrow fact lookup.
- *deep* — a serial search–verification structure: multiple Searchers propose candidates from different clue paths, an independent Verifier adversarially checks each candidate against all constraints, refining hypotheses iteratively; children are atom nodes.
- *wide* — parallel divide-and-conquer: concurrently dispatches a set of independent child nodes (which may themselves be deep, entity_collect, wide, or atom) and merges results; suited to filling attributes across a known set of entities.
- *entity_collect* — multi-path recall and verification for enumerating a complete set when the boundary is unknown: multiple atom nodes recall candidates from different sources, then merge, deduplicate, and verify low-confidence ones to balance recall and precision.

**Web-Structure-Guided Expansion.** Before a wide node expands, a Web-Probing Agent runs a lightweight probe (search + fetch) and classifies evidence topology as centralized, centralized_with_gaps, or distributed, optionally naming an organization dimension (tour, brand, year, country, …). When evidence is concentrated in a few hub pages, WebSwarm creates a small number of extraction-oriented nodes; when it is dispersed, expansion follows the web-organization axis. Child delegations are then generated conditioned on h_v: C_v = Delegate(q_v, m_v, R_v, h_v).

**Experience-Guided Node Solving.** For a batch of homogeneous child delegations C_v under a wide node, WebSwarm first runs a small scout subset C_v^s, collects their full trajectories τ_i, and extracts process-level experience k_v = ExtractExperience({τ_i}) — query patterns, reliable sources, page types, invalid paths. The remaining siblings are then solved with k_v injected: r_j = SolveNode(v_j; k_v). Experience is reused only within the same instance and among homogeneous siblings, preserving sample independence.

**Implementation.** Backbone is GLM-4.5 (with Qwen3-32B and Qwen3.5-35B in generalization studies). All methods share the same Serper-based Web-Search tool (top-5 results) and Jina-Reader-based Page-Browse tool (page content summarized goal-conditionedly by the same LLM, capped at 80K tokens). entity_collect uses 3 parallel paths; 2 scout children per wide node. Up to 200 action steps and the model's maximum context length (128K for GLM-4.5, 256K for Qwen3.5) are allowed; thinking mode enabled, temperature 0.6, top-p 0.95. Results averaged over two random runs.

## Results & Highlights

- **BrowseComp-Plus (deep factual search, ACC, 200 sampled instances, GLM-4.5):** WebSwarm reaches **68.00 ACC**, a **+17.50** improvement over the ReAct agent (50.50) and **+3.50** over the strongest multi-agent baseline Swarm-Agent (64.50). The gain is attributed primarily to the deep search mode's iterative Searcher–Verifier loop.
- **WideSearch-EN (structured wide collection, 100 tasks):** WebSwarm achieves **Row F1 44.14** (+10.91 vs ReAct 33.23) and **Item F1 74.37** (+9.76 vs ReAct 64.61), and SR 7.00 (+3.00 vs 4.00), also beating the best multi-agent baseline InfoSeeker (Item F1 71.91).
- **DeepWideSearch-EN (interleaved deep+wide, 76 tasks):** WebSwarm achieves **Row F1 29.64** (+9.56 vs ReAct 20.08) and **Item F1 58.40** (+11.77 vs ReAct 46.63); SR rises to 6.58 (+2.63 vs 3.95), the best among all compared methods.
- **GISA (general info seeking, 373 tasks, overall):** WebSwarm scores **62.30 overall** (+6.76 vs ReAct 55.54), with the largest delta on List F1 (+15.46, 66.04 vs 50.58).
- **Hard-sample gains (Figure 3, difficulty stratified by ReAct pass rate):** On the Hard subset of BrowseComp-Plus, WebSwarm lifts ACC from **0.0 to 35.7** over ReAct; on the Hard subset of WideSearch-EN, Item F1 rises from **24.5 to 55.8** — the advantage grows with difficulty.
- **Web-Probing ablation (Table 3):** Removing Web-Probing barely changes Item F1 but inflates average Web tool calls from **137.03 → 239.90** on WideSearch and **203.73 → 331.39** on DeepWideSearch, i.e., probing's main benefit is search *efficiency* by cutting redundant/misaligned expansion.
- **Experience-reuse ablation (Table 3):** Removing sibling-node experience drops WideSearch Item F1 from **74.37 → 71.20** and DeepWideSearch Item F1 from **58.40 → 55.48**, showing it improves homogeneous node-solving quality.
- **Recursive-delegation ablation (Table 2):** w/o recursive delegation drops ACC on BC-Plus (68.00 → 63.50), WideSearch Item F1 (74.37 → 68.38), and DeepWideSearch Item F1 (58.40 → 55.79). All-to-deep keeps BC-Plus but collapses wide tasks; All-to-wide helps wide but damages BC-Plus (68.00 → 63.00), confirming the value of mode–objective matching.
- **Cross-backbone generalization (Table 4):** Gains hold on Qwen3-32B (BC-Plus ACC 12.00 → 19.50; WideSearch Item F1 29.11 → 34.47) and Qwen3.5-35B (BC-Plus ACC 56.50 → 61.00; WideSearch Item F1 64.82 → 75.91), indicating WebSwarm's improvements are not tied to a specific backbone capability level.

## Critical Assessment

- **Novelty.** The framing of search as recursive delegation over *objective–mode pairs* (rather than as a fixed plan or a single global collaboration topology) is a genuine, clearly-articulated advance over MindSearch/HiRA (graph/hierarchical planning), InfoSeeker/A-MapReduce (horizontal parallelism), ROMA (recursive but single-protocol), and Table-as-Search (table-as-state). The two most distinctive ingredients — Web-Probing that classifies evidence topology and conditions the expansion axis, and within-instance sibling experience reuse via scout trajectories — are non-trivial and well-motivated by the (1)/(2)/(3) failure-mode analysis in the intro.
- **Rigor.** The main table compares against six multi-agent baselines on four benchmarks, all sharing the same Serper/Jina tool stack and the same backbone (GLM-4.5), and reports results averaged over two random runs — a fairer setup than many agentic-search papers. Ablations cleanly isolate recursive delegation, single-mode forcing, Web-Probing, and experience reuse. Difficulty stratification (Figure 3) and tool-efficiency analysis directly support the "gains concentrate on hard samples" and "probing saves calls" claims.
- **Reproducibility.** Strong on the software side: full system prompts for the root, probing, and experience-extraction agents are in the paper (Figures 8–10), the web-tool schema is given (Figure 11), pseudo-code is in Algorithm 1, and code is public. Weaker on the data/seed side: BrowseComp-Plus is "randomly sampled 200 instances" without a fixed seed, WideSearch/DeepWideSearch use only the English subsets (100/76 tasks), and "run twice randomly and take the average" leaves variance bars unreported — small sample sizes plus undisclosed seeds make the +3.50 ACC delta on BC-Plus over Swarm-Agent (which itself uses Kimi-K2 Thinking at 66.50 in the single-agent row) worth treating cautiously.
- **Significance.** The work targets the right problem: inference-time orchestration for deep-and-wide web search is an active, benchmark-driven area, and the unified treatment of deep/wide/entity_collect within one recursive process is broadly useful. The released code and prompts make it a credible starting point for follow-up multi-agent search systems.
- **Red flags / overclaiming.**
  - *Baseline asymmetry.* The single-agent ReAct rows include a much stronger Kimi-K2-Thinking model (66.50 ACC on BC-Plus) that is *not* carried into the multi-agent comparison, where all baselines run only GLM-4.5. Swarm-Agent at GLM-4.5 is 64.50 — within 3.5 points of WebSwarm and uncomfortably close to the Kimi-K2 ReAct number that is excluded from the headline. The framing that WebSwarm "consistently outperforms multi-agent baselines" is true on the GLM-4.5 playing field but the paper does not discuss this backbone confound.
  - *WideSearch SR is tiny.* SR numbers are in the 4–7 range on WideSearch/DeepWideSearch (out of 100); with only 100 and 76 tasks, a +3.00 SR delta is ~3 tasks and within plausible run-to-run noise given the undisclosed seeds.
  - *Web-Probing's headline benefit is efficiency, not accuracy.* Table 3 shows removing probing *slightly raises* Item F1 on both WideSearch (74.37 → 74.90) and DeepWideSearch (58.40 → 58.93) while roughly doubling tool calls — so probing is essentially a cost reducer, and the paper is commendably honest about this, but the abstract/intro framing leans more heavily on "evidence-grounded expansion" as an accuracy mechanism than the ablation supports.
  - *Cost not normalized.* WebSwarm makes substantially more LLM and web-tool calls than ReAct (e.g., 137 vs fewer on WideSearch, with hard samples using far more), yet there is no equal-cost comparison (e.g., ReAct given the same tool budget) — so part of the gain may be raw scaling rather than orchestration.
  - *No human eval / no error analysis.* All metrics are automatic (ACC, F1, SR); there is no qualitative error taxonomy of *where* WebSwarm still fails, which limits insight.

## Strengths

- Unified, well-formalized recursive-delegation framework that subsumes deep, wide, and interleaved search under one mechanism with explicit objective–mode coupling.
- The Web-Probing topology classification (centralized / centralized_with_gaps / distributed + organization dimension) is a clean, transferable idea that other multi-agent search systems could adopt.
- Honest ablations: the paper openly reports that Web-Probing's main effect is tool-call reduction (not accuracy) and that experience reuse is what lifts Item F1.
- Strong reproducibility artifacts: public code, three full system prompts, web-tool schema, and pseudo-code.
- Genuinely grow with difficulty: the Hard-subset jumps (0.0 → 35.7 on BC-Plus, 24.5 → 55.8 on WideSearch) are the most compelling result and directly address the stated motivation.
- Cross-backbone validation on Qwen3-32B / Qwen3.5-35B shows the gains are not a GLM-4.5 idiosyncrasy.

## Limitations

- Scope is inference-time only: node delegation, mode assignment, probing, and experience reuse all rely on the base LLM's reasoning, with no policy optimization or training — the authors explicitly note this may not fully unlock the model's recursive collaboration potential.
- Higher cost and latency than a single ReAct agent; no equal-budget comparison is provided, so the accuracy/cost Pareto frontier against ReAct-with-more-tool-calls is unclear.
- Text-only web tools; multimodal information (images, video, audio) and GUI-based browsing are out of scope.
- Small evaluation slices (200/100/76 tasks on the first three benchmarks) with undisclosed seeds and only two runs; some headline deltas (e.g., +3.00 SR, +3.50 ACC over Swarm-Agent) are within plausible noise.
- Backbone confound: a stronger Kimi-K2 ReAct appears in single-agent rows but is dropped from the multi-agent comparison, slightly flattering the "beats all baselines" claim.
- No qualitative error analysis or human evaluation; all metrics are automatic.

## Relevance

对关注多智能体系统、agentic search 与 LLM 推理时编排（inference-time orchestration）的研究者高度相关：WebSwarm 提供了一套可复用的"递归委派 + 目标-模式耦合 + Web 结构探测 + 兄弟经验复用"范式，可直接迁移到深度研究型问答、表格化信息抽取、长程任务编排等场景，并通过公开代码与系统提示词降低复现门槛。

## Questions for Further Exploration

- Under an *equal web-tool-call budget*, how much of WebSwarm's gain over ReAct survives? Is the win orchestration, or partly raw scaling?
- How stable are the small deltas (e.g., +3.50 ACC on BC-Plus vs Swarm-Agent, +3.00 SR on WideSearch) across more seeds and the full benchmark splits rather than the 200/100/76-task subsets?
- Can Web-Probing's topology classifier be trained or prompt-tuned for domains where Wikipedia-style hub pages are rare (e.g., scientific literature, legal corpora)?
- Does experience reuse across sibling nodes introduce failure-correlation (all siblings repeating the same wrong scout pattern), and can an adversarial verifier mitigate that?
- How would WebSwarm interact with reinforcement-trained agentic models (e.g., Search-R1, WebThinker)? Does learned recursive delegation subsume the hand-designed modes?
- Can the framework extend to multimodal web content (images, video, audio) and GUI-based browsing without breaking the recursive-delegation abstraction?
- What is a useful qualitative error taxonomy for WebSwarm's residual failures on hard samples — aggregation errors, mode mis-assignment, probing misclassification, or verifier false accepts?
