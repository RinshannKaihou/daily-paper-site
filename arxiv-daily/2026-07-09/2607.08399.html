Key Finding

A 4-layer MLP compresses an entire instruction prompt into one activation vector, re-injected at an early layer so a frozen LLM answers without seeing the prompt.

Prompt Compression via Activation Aggregation ↗

Thibaud Ardoin · Semira Einsele · Evis Bregu · Gerhard Wunder — Freie Universität Berlin

09 JUL 2026 · cs.CL · v1

Key Metric · Toy Task, Llama-3.1-8B

~2%

accuracy drop vs full-prompt — W-MLP 85.35% vs 86.92% full-prompt (masked baseline 34.03%).

Why This Matters

Moves prompt compression from tokens to activations, bolsters the linear-representation hypothesis, and exposes a reusable mid-extract, early-inject pattern for activation engineering.

Critical Edge

The “<2% drop” holds only on the authors’ homemade Toy Task — on ARC-Easy the gap is ~15–24%, and no reasoning or MMLU benchmark was tested.

Methodology

01 / EXTRACT → COMPRESS → INJECT

Operates on a frozen target LLM with L attention layers, in two forward passes over prompt p = [t₁ … t_T].

  1. Extract. Run a forward pass; read the hidden-state sequence H⁽ᵐ⁾(p) ∈ ℝ^{T×d} at an intermediate layer m.
  2. Compress. A function f: ℝ^{T×d} → ℝᵈ collapses the sequence into a single patch vector v. Two variants: the lightweight W-MLP (4-layer FF, dims [2048,1024,512,256], outputs per-token weights summed linearly) and the heavier Transformer Compressor (2-layer, 2-head encoder, hidden 64).
  3. Inject. Second pass on a shortened input carrying a placeholder token (UTF-8 U+FFFD). At an early layer e < m, the placeholder’s downstream activation is overwritten with v, and the forward pass continues from layer e.

Both compressors train on cross-entropy vs. the target LLM’s logits. Primary model: Llama-3.1-8B-Instruct; also Ministral3-8B, Qwen3.5-4B, Llama3.2-1B (single RTX A5500). Data: a custom 11-task Toy Task set (exact-match) and ARC-Easy (4-way MCQ, question compressed, choices exposed). Bracketing baselines: full-prompt upper bound and masked-prompt lower bound.

Strengths

03
  • Minimal compute wins. A 4-layer MLP + linear sum beats a transformer encoder — clean support for the linear-representation hypothesis.
  • Cross-layer transfer is real. Mid-layer activations plug into early layers cleanly; useful beyond this paper.
  • Interpretable by construction. Weights expose token importance with no post-hoc attribution, validated by cosine + KL diagnostics.
  • Honest limitations section on lossiness, white-box need, and compression-time cost.
  • Clean capacity story: smaller hidden dim → worse compression, across 4 LLMs.

Limitations

04
  • Lossy and narrow. Untested on long context or reasoning; only short knowledge-retrieval prompts.
  • White-box only. Needs internal activations — excludes closed-source LLMs.
  • No latency / FLOPs. The “reduce per-query computation” motive is unquantified.
  • Headline is cherry-picked. “Under 2%” is Toy-Task-only; ARC-Easy gap is much larger.
  • Non-readable vectors could obscure prompt intent from reviewers.
  • Poor small-model scaling: on Llama3.2-1B, W-MLP falls below even TC.

Open Questions

05 / FOR FURTHER EXPLORATION
  • How does it scale to long contexts (8k–32k) and reasoning tasks (GSM8K, MMLU), where information density per vector is far higher?
  • What are the real wall-clock / FLOPs savings per query, given the half-network forward pass at compression time — and at what reuse count does it break even?
  • Can segmenting a prompt into multiple patch tokens (one per semantic block) close the 15–24% ARC-Easy gap without losing single-vector elegance?
  • Why does W-MLP underperform TC on Llama3.2-1B (32% vs 37%)? Is there a hidden-dim threshold below which the linear-sum assumption breaks?
  • Does the learned attribution converge with established methods (integrated gradients, GEFA), which the authors leave open?
  • Can patch vectors double as RAG retrieval embeddings, sharing one representation for search and generation?
