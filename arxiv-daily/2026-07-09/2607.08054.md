> **一句话总结：** 本文将 STPA（系统理论过程分析）反过来指向 LLM 辅助的安全分析工具自身，构建了一个"Constitutional Meta-STPA"闭环：工具对自身设计做 meta-STPA，从 loss→hazard→UCA→constraint 链条"推导"（而非人工断言）出由 21 条 Tool Principles 和 8 条 Meta-Safety Principles 组成的治理宪法，每条都绑定到代码强制执行点；前沿集成（claude-opus-4.8 + claude-sonnet-4）能从工具自身设计恢复 18/21 规范原则和全部 8/8 治理原则（弱模型对仅 12/21 和 3/8），Tool Principles 使留出对抗探针的平均安全分从 1.03 升至 1.85（+79%，p<0.001，d=1.29），而通用的 helpful/harmless/honest 前言则完全无效。

> **TL;DR:** This paper turns Systems-Theoretic Process Analysis (STPA) on the LLM-assisted safety-analysis tool itself, building a closed loop ("Constitutional Meta-STPA") that derives — rather than hand-asserts — a 29-principle governance constitution (21 Tool Principles + 8 Meta-Safety Principles) from the tool's own loss→hazard→UCA→constraint chain, each principle bound to a code enforcement point. A frontier ensemble (claude-opus-4.8 + claude-sonnet-4) recovers 18/21 canonical and all 8/8 governance principles from the tool's own design versus only 12/21 and 3/8 for a weaker gpt-4o/gpt-4o-mini pair, the 21 Tool Principles raise held-out adversarial safety score from 1.03 to 1.85 (+79%, p=0.0004, d=1.29) while a generic Claude preamble does nothing (1.05 vs 1.03), and the same 8/8 governance principles re-emerge from a second, independently authored AI tool while hardware systems stay at a 0/8 floor — making meta-coverage a system discriminator, not a dose metric.

## 核心贡献

- **首次提出并实现"谁分析分析者"的闭环方法**：将 STPA 反向应用于 LLM 辅助安全分析工具自身（meta-STPA），把"AI 辅助安全工具"类视为被分析系统，其控制器为 LLM、验证层、投票框架、导出门控与值班工程师，系统性地暴露出经典 STPA 不会触及的工具级隐患（未记录的 LLM 调用、版本静默漂移、宪法跨次变更、未验证分析被导出、提示注入等）。
- **"推导而非断言"的治理宪法**：从 meta-STPA 推导出的 8 条 Meta-Safety Principles（审计日志、运行清单、模型/版本固定、默认仅哈希、宪法固定、语义匹配投票、验证门控导出、安全与韧性）全部绑定到代码中可机器检查的强制执行点（由 `stpa_tool/sc_trace.py` 与测试套件保证），不是停留在 prompt 层的文字。
- **形式化的覆盖率框架与合理性引理**：将测量对象形式化为"宪法-边际覆盖率算子" cov(C,S,M,k)，并证明 Lemma 1（constitution-marginal soundness）——任何观测到的覆盖率差异都由生成文本上的固定词汇谓词作证，而非依赖裁判意见或模型自报，从而把覆盖率测量与模型/扫描器解耦。
- **严格的行为实验隔离出"哪个宪法层起作用"**：在 20 个留出对抗探针、5 种宪法变体（cv0–cv4）上预注册 5 个假设，用 Wilcoxon 检验（Bonferroni α=0.01）证明行为提升完全来自 21 条 Tool Principles（H1: cv2>cv0 +79%, p=0.0004, d=1.29），而通用 Claude 层和 Meta-Safety 层对单轮行为无显著影响（H3、H5 为零结果且是确认性的）。
- **诚实报告的构造效度边界与非复现**：公开报告了词汇元覆盖率并不随宪法强度单调递增（空控制 cv0 反而命中最多 6/8），并坦诚披露一个早期在非确定性解码下得到的"满覆盖跳变"结果在确定性重跑下未能复现，将其作为 LLM 评估非确定性的警示案例而非掩埋。
- **完整的可复现性工件**：发布宪法、覆盖率框架、投票度量、运行清单（run manifest.json）、逐调用审计日志（llm_calls.jsonl，含 prompt/response SHA-256）、种子与模型标识，使得论文中每一个数字都可从一条记录的命令行重放。

## Key Contributions

- **A self-validating closed loop ("Constitutional Meta-STPA")**: the first work to apply STPA to the LLM-assisted STPA tool itself, treating the class of AI-assisted safety tools as the system under analysis and surfacing tool-level hazards (unlogged LLM calls, silent version drift, constitution changes between runs, unvalidated exports, prompt injection) that classical outward-facing STPA never raises.
- **A derived-not-asserted 29-principle governance constitution** (21 Tool Principles + 8 Meta-Safety Principles), each Meta-Safety Principle bound to a machine-checked code enforcement point, turning governance prose into executable checks (validator-gated export, hash-only audit logging, run manifest pinning).
- **A formal coverage framework with a soundness lemma** (Lemma 1) that isolates coverage measurement from both the model and the scanner: any observed coverage delta is attested by a fixed lexical witness on the generated artifact text, never by judge opinion or self-report.
- **A pre-registered behavioural experiment** isolating which constitutional layer moves safety behaviour: the 21 Tool Principles carry the entire effect (+79%, d=1.29), while a generic helpful/harmless/honest preamble is statistically indistinguishable from no constitution and the Meta-Safety Principles do not move single-turn probes (expected, since they govern analysis-time machinery).
- **An honest construct-validity analysis**: meta-coverage is shown to be non-monotone in constitution strength on a fixed AI system and is reframed as a cross-system discriminator (0/8 on hardware, 2–6/8 on AI self-system), with an earlier non-deterministic "jump to full coverage" result openly reported as a non-replication.
- **A full reproducibility artifact**: released constitutions, scanner, voting metric, run manifests, per-call SHA-256 audit logs, and seeded calls so every number replays from one recorded command line.

## Methodology

The paper builds a five-step STPA pipeline Π(C, S, M, k) mapping a system description S and constitution C to an analysis artifact A (losses, hazards, controllers, UCAs, constraints), a validation report R, and a binary export decision g. The central design decision is a **deterministic skeleton around stochastic steps**: the LLM supplies content (free-form losses, hazards, control structure), while a deterministic engine owns structure — it enumerates exactly one UCA slot per (controller × control action × UCA-type) triple (all four STPA UCA types), so four-type coverage is a property of the harness, not model diligence. One constraint slot is created per non-N/A UCA, making the loss→hazard→UCA→constraint traceability chain constructed rather than hoped for.

The meta-STPA derivation treats the AI-assisted tool itself as the system under analysis. Its controllers are the LLM, validator layer, voting harness, export gate, and on-call engineer; walking the four UCA types over their control actions surfaces hazards classical STPA does not naturally raise (an unlogged LLM call, version drift, a constitution that changes between runs, a report emailed while validation errors are unresolved, a tampered description). Each hazard yields a constraint, giving the eight Meta-Safety Principles, each bound to a concrete enforcement symbol in code.

Governance is enforced by **eight deterministic regex/structure validators** (vague language, passive voice, overconfidence, unverified standards, missing confidence/limitations, sensitive-data leaks, link completeness) plus a **validator-gated export** (MSP-07): Markdown and JSON are always written so a failing analysis is inspectable, but redistributable PDF/email artifacts are blocked unless error count is zero or the engineer records a verbatim override. **Semantic-matching multi-model voting** (token-Jaccard, τ=0.4) replaces ID-equality voting that had fired 100% disagreement on synonymous controller names; a sentence-transformer cross-check is shipped as an upper bound. The **reproducibility manifest** pins tool version, git SHA, model, temperature, seed, and SHA-256 of system description and constitution; every LLM call appends a hashed audit line (by default hash-only for data minimisation). The coverage operator cov(C,S,M,k) is defined over a principle set P (|P|=29) with each principle equipped with a deterministic witness predicate ϕ_p, and Lemma 1 certifies that any coverage difference is grounded in artifact text.

## Results & Highlights

- **Self-derivation (frontier vs. weak)**: On the tool's own self-system under the full constitution (seed 42, T=0), the frontier ensemble claude-opus-4.8 + claude-sonnet-4 recovers **18/21 canonical and 8/8 governance principles** in union (opus alone 17/21 and 8/8; sonnet 16/21 and 8/8), whereas the weak pair gpt-4o-mini + gpt-4o recovers only **12/21 canonical and 3/8 governance** — so the meta layer is model-limited, not constitution-limited.
- **Behavioural lift (held-out adversarial probes)**: Adding the 21 Tool Principles (cv2) raises the mean safety score from **1.03 to 1.85 (+79%, Wilcoxon p=0.0004, Cohen's d=1.29)** on n=20 probe means, and lifts the fraction of fully-safe probes from 10% to 80%; the generic Claude helpful/harmless/honest preamble (cv1) is statistically indistinguishable from no constitution (1.05 vs 1.03). H1, H2, H4 survive Bonferroni correction with d>1.2; H3 (+C) and H5 (+MSP) are non-significant null results.
- **Cross-system discrimination (meta-coverage as discriminator, not dose)**: Under the domain-only, meta-only, and full constitutions, all three hardware systems (AEB, infusion pump, UAV autoland) sit at the **0/8 meta floor**, while the AI self-system surfaces 2–6/8; the meta-only arm lifts the self-system to 6/8 yet moves hardware to only 0–2/8. The same predicates fire 0/13 on the AEB negative control (one labelled MSP-07 false positive left in to characterise worst-case precision).
- **Generalisation to a second, independently authored tool** (a CI code/security-review assistant): under the identical strong ensemble, its self-analysis recovers **all 8/8 governance principles** in union (opus 8/8, sonnet 7/8) and 17/21 canonical, even though its description never names audit logging, version pinning, or prompt injection — evidence the governance layer is a property of the class of LLM-assisted tools, not an artifact of the authors' own tool.
- **Five-vendor standard-STPA thoroughness varies >3×**: claude-sonnet-4 is most exhaustive (11.7 canonical themes, 183 UCAs per system) and gpt-4o most terse (5.3, 45); all five vendors completed all three hardware systems; UCA coefficient of variation ≤12% in 11/12 cells and gemini-2.5-flash is exactly reproducible (CoV 0%, κ=1.0).
- **Voting metric comparison**: the deployed token-Jaccard matcher is a sound, deterministic, no-LLM-dependency floor but under-counts paraphrased items; the sentence-transformer cross-check recovers **43–61 percentage points** over lexical overlap across loss/hazard/UCA/constraint kinds.
- **LLM-judge concordance**: an independent gpt-4o judge corroborates the central negative finding (no meta dose-effect) but agrees only moderately with the scanner (Cohen's κ=0.39 over n=270 judgements), with noisier canonical counts (5–21 vs scanner 12–13).
- **Reference grounding (opus self-analysis)**: 156/156 UCA slots filled, 0 orphaned hazards (15/15), and **118/156 (75.6%) constraints cite an externally verifiable standard**; the remaining 24.4% cite nothing rather than fabricate (exactly the behaviour TP-05 no-fabrication is designed to produce).

The authors are from Iowa State University — Samuel Tetteh and Cody Fleming (Mechanical Engineering, corresponding authors), Udip Shrestha (Electrical and Computer Engineering), and Joshua R. Waite (Translational AI Center). Cody Fleming has a sustained publication record on STPA and model-based safety analysis (Refs [7; 8; 9], including integrating systems safety into concept-stage systems engineering and early concept development of transportation systems), which gives the STPA-specific framing genuine domain authority rather than an outside-discipline application. The work is positioned squarely against a fast-growing LLM-for-STPA literature (Diemert & Weber, Qi et al., Nouri et al., Hong et al., Raeisdanaei et al.) and its critique — that none of these works meta-analyse the LLM pipeline itself — is a specific, verifiable gap claim, not a strawman. ## Critical Assessment

**Novelty** is high and well-articulated: the "analyse the analyser" framing is genuinely novel within the LLM-driven hazard-analysis literature, and the move from hand-asserted constitution prompts (Constitutional AI, Bai et al.) to a derived, code-bound governance layer with a formal coverage operator and soundness lemma is a substantive methodological advance, not a repackaging.

**Rigour** is the paper's strongest dimension: pre-registered hypotheses with Bonferroni correction, a Wilcoxon signed-rank test on n=20 paired probe means, a five-step constitution ladder (cv0–cv4) that cleanly isolates the active layer, negative controls (three hardware systems at 0/8), and a second independently authored tool to rule out self-system overfit (threat T2). The Lemma 1 soundness argument is mathematically trivial but methodologically load-bearing — it cleanly separates coverage claims from judge opinion.

**Reproducibility** is exemplary for the LLM-tooling literature: run manifests pinning model/temperature/seed/constitution/system SHA-256, per-call audit logs with prompt/response hashes, released code, constitutions, and scanner. However, an important honesty caveat applies — OpenRouter honours seeds only best-effort and may silently re-route back-ends, so bit-exact replay is not guaranteed even from a pinned manifest; the manifest makes drift *detectable*, not impossible (a limitation the authors state openly). The cross-vendor study itself functions as a determinism probe: gemini-2.5-flash is exactly reproducible (CoV 0%, κ=1.0) while claude-sonnet-4, whose vendor ignores the seed, shows genuine run-to-run variance (253±17 UCAs, CoV 7%).

**Red flags / weaknesses**:
- The headline lexical scanner is a **regex keyword bank** that the authors themselves flag as permissive and defeatable by paraphrase (threat T1); the LLM-judge cross-check agrees only moderately (κ=0.39), so absolute coverage counts carry real measurement noise. The authors are admirably explicit that comparisons are more trustworthy than absolute levels, but a reader should not treat "18/21" as a precise number.
- **No human-expert baseline**: practising safety engineers have not independently scored the same analyses or probes, so the behavioural rubric and witness bank encode the authors' own judgement of what counts as "safe"; the class-level governance claim rests on only two AI tools plus three hardware controls (a sample, not a panel). This is the most consequential piece of external validation still outstanding.
- The **behavioural claim is narrow**: 300 single-turn textual responses measure what the tool writes, not downstream deployment outcomes, multi-turn interaction, or tool use — and the MSP layer's zero behavioural effect is partly an artifact of single-turn probes not exercising the analysis-time machinery the MSPs govern.
- The **"fixed point" framing** (Appendix C.2) is argued informally and observed empirically, not proven as a mathematical contraction; Φ is stochastic and convergence is observed only under a capable ensemble (the weak pair recovers 3/8 and sits far from the fixed point), so the basin of attraction depends on model capability.
- A weaker version of the construct-validity concern: meta-coverage **co-moves with artifact volume** (Section 11 — the verbose 490-word description surfaces 8/8 meta clauses, the terse 66-word one only 4/8), so raw meta-coverage is partly a length artifact; the Meta/100-UCAs normalisation collapses much but not all of this spread.

**Significance**: for anyone building or deploying LLM-assisted safety/regulatory-analysis tooling, the operational recommendations (invest in task-specific principles, ship a coverage scanner + reproducibility manifest + validator-gated export, treat the meta layer as a compliance substrate rather than a behavioural knob) are concrete and immediately actionable.

## Strengths

- Conceptually novel and clearly motivated: the "who analyses the analyser?" question is a real structural blind spot in the LLM-for-STPA literature, and the closed-loop answer is elegant.
- Exceptional methodological discipline for an LLM-application paper: pre-registered hypotheses, Bonferroni correction, a constitution ladder that isolates the active layer, negative hardware controls, a second-tool generalisation check, and three independent deterministic repeats per cell.
- Intellectual honesty stands out: the non-monotonicity of meta-coverage is reported as a negative construct-validity result, an earlier non-deterministic "jump" is openly disclosed as a non-replication, and the LLM-judge's only-moderate κ=0.39 agreement is stated rather than buried.
- Engineering artefact is genuinely reusable: deterministic UCA-slot enumeration makes four-type STPA coverage a harness property; the validator-gated export, hash-only audit logging, and run manifest are concrete patterns any LLM-safety-tool builder can adopt.
- Clean separation of concerns: content from the model, structural completeness from the engine, discipline from validators/manifest, judgement from the engineer — with an auditable record of who decided what.

## Limitations

- No human-expert gold standard: neither the behavioural rubric nor the lexical witness bank is calibrated against practising safety engineers, so coverage numbers are internally consistent but not externally validated.
- Lexical scanner is permissive and paraphrase-defeatable; the deployed token-Jaccard matcher under-counts paraphrased items by 43–61 points versus the embedding upper bound, and κ=0.39 with the LLM-judge is only moderate agreement.
- Class-level governance claim rests on a sample of two AI tools plus three hardware controls, one analysis paradigm (STPA), English-language probes, and frontier models reached through a single router (OpenRouter) — external validity to other tool classes, methods (FTA/FMEA/HAZOP), non-English inputs, and untested model families remains an open empirical question.
- Behavioural experiment measures only single-turn textual behaviour on adversarial STPA requests; it does not exercise multi-turn interaction, tool use, or the analysis-time machinery the MSPs govern, and it does not measure downstream deployment outcomes.
- Bit-exact replay is not guaranteed: OpenRouter may silently re-route back-ends even with a pinned seed; the manifest makes drift detectable but not preventable, and the order-of-tens-of-dollars budget bounds the number of seeds, probes, and tools that could be swept.
- The fixed-point convergence of the meta-loop is observed empirically and argued informally, not proven; a higher-capability or adversarial model could surface governance principles beyond the current set.

## Relevance

这项工作与"用 LLM 辅助严谨安全/工程分析并保证其可信与可复现"的研究兴趣高度相关——它直接回答了 LLM 安全工具自身的可信与审计问题，提供了宪法推导、形式化覆盖率度量、验证门控导出与完整可复现性工件等可迁移的模式。其形式化覆盖率框架与 soundness lemma 的思路，对任何需要客观、可复现地度量 LLM 输出质量的场景（不只是 STPA）都有方法论参考价值。

## Questions for Further Exploration

- How would the derived constitution and coverage framework transfer to other hazard-analysis paradigms (FTA, FMEA, HAZOP) where the structural skeleton differs from STPA's four-UCA-type enumeration?
- Would pairing each lexical witness with an entailment check (as the authors propose) meaningfully raise the scanner's soundness floor, or would it simply inherit LLM-judge failure modes?
- How do the results change under a human-expert gold standard — is the 18/21 canonical recovery a real ceiling or an artifact of the authors' witness bank encoding their own judgement of the 21 themes?
- Can the meta-loop's fixed-point convergence be strengthened from an empirical observation to a contraction argument under a defined metric, and what is the basin of attraction as a function of model capability?
- Does the behavioural lift from Tool Principles survive multi-turn interaction and tool use, or does it degrade when the model must chain analyses across sessions and external tools?
- How sensitive are the cross-vendor depth differences (45 vs 183 UCAs) to prompt phrasing and starter-pack content, and is there a depth–precision trade-off that the canonical-coverage metric masks?
