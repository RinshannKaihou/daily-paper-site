§ 01

Methodology — codebook-free codec

Core idea · binary spherical weight codes

Drop the explicit VQ codebook entirely. Each weight matrix is split into d-dim chunks; an encoder maps a chunk to a latent that is L2-normalized onto the unit hypersphere and binarized to a sign vector q ∈ {±1/√b}ᵇ (straight-through estimator for the sign op). The stored payload is the bit-packed sign pattern, not a centroid index — reconstruction is done by compact category-shared neural decoders that ship with the model. Codec loss combines chunk-reconstruction MSE, latent commitment, and an entropy + diversity regularizer so the vast implicit binary space is actually exercised.

Three supporting mechanisms

  1. 01Two-stage residual BSQ. A 2⁶⁴ implicit space has only ~524K occupied patterns per layer (≈2.84×10⁻¹⁴ used), so extra bits go to coding the residual of the base stage, not enlarging one code.
  2. 021% 8-bit protected channels. Top-K high-impact slices (attention: input-channel; MLP gate/up/down: coupled intermediate) stay 8-bit and are counted in budget — ρ = 1%.
  3. 03Category-wise recovery distillation. Layer-wise recovery cuts local loss −27% yet assembled accuracy collapses 70.71%→53.36%, so recovery runs after each whole module category is swapped in (LoRA-only by default).

Real budget: S_tot = S_code + S_dec + S_8bit + S_LoRA + S_meta — codebooks, projections, scales & metadata all counted.

Codec organized per module category (q/k/v/o · gate/up/down proj), shared across layers Batched parallel residual decoding: base + residual decoded independently then summed — no sequential bottleneck Weight-only; decoded weights fall back to standard matmul kernels

§ 03

Strengths & limitations

✓ Strengths
  • Conceptually clean. Storage shifts from K×d codebook floats to bit-packed signs plus amortized shared decoders, with every auxiliary byte explicitly accounted for.
  • Paper-specific diagnostics. Code-space occupancy at ~2.84×10⁻¹⁴ and the non-monotone granularity sweep give a concrete argument for residual coding rather than one-stage enlargement.
  • Honest recovery failure. Local losses fall 27.26% yet assembled accuracy collapses to 53.36% — directly justifies category-wise over layer-wise recovery.
  • Strong per-component ablations. Residual stage, protected-channel necessity & axis, protection ratio, decoder architecture all isolated.
  • Wide scalar margin. On the one fully-reported model it leads scalar baselines by a wide margin and edges the best vector baselines on WikiText-2 PPL.
! Limitations
  • LLaMA3-8B entirely missing. Table III row is all dashes, so cross-architecture generalization is unproven.
  • Non-comparable averages. Qwen3-8B average omits HellaSwag, so cross-method averages are not on identical task columns.
  • Real bpw not reported numerically. The bit-width column is "omitted for compactness"; "2-bit-oriented" is a characterization, not a verified number.
  • Weight-only, no serving numbers. No weight–activation quant, no fused binary kernels, no end-to-end latency / throughput / peak-HBM — decoded weights use standard matmul.
  • Optimization-heavy. Category-wise codec training + multi-stage recovery distillation is far costlier than calibration-only PTQ (GPTQ/AWQ).
  • Partial-results submission. Phrases like "we plan to evaluate" confirm v1 is incomplete; no code/release referenced.
§ 04

Open questions

  • 1Does the advantage hold on LLaMA3-8B and larger / MoE models, and what is the achieved real bpw once decoder + 1% protected + LoRA + metadata are summed?
  • 2How does BiSCo-LLM compare once HellaSwag is included for all methods, so averages are computed on the identical task set?
  • 3What is the real serve-time cost — decoding latency, peak HBM, tokens/sec vs. QuIP#/UniSVQ — and can fused binary-code kernels recover the bandwidth win?
  • 4Does principled mixed-rate allocation (residual energy / Hessian / activation-based) beat the current uniform 2-bit + fixed 1% protection?
  • 5Can the hypersphere/binarization machinery extend to weight–activation quantization, or does it only suit static weight tensors?
  • 6Is the layer-wise recovery collapse (70.71%→53.36%) specific to LoRA-only recovery, or does it persist for full end-to-end recovery — i.e. is category-wise recovery strictly necessary?
