PredicateLongBench / Long-Context Benchmark / arXiv 2607.08284

Frontier LLMs collapse from 97% to 1% on simple long-context tasks once decoys are scattered — structure, not length, drives difficulty.

Paper: Understanding Axes of Difficulty For Long Context Tasks Via PredicateLongBench

Siddhartha Jain, Ameya Velingker · NVIDIA | 9 Jul 2026 | long-context benchmark llm-evaluation retrieval reasoning
Key Metric
97% → 1%

Opus 4.6 high, synthetic lexicographic locate: inserting 8 scattered near-sorted decoys collapses accuracy from 97% to 1%. No frontier model exceeds 2% on this variant.

Why This Matters

Judge-free, deterministically generated probes showing context structure dominates difficulty and search space is independent of token count — directly diagnostic for long-context LLMs and RAG pipelines.

Critical Edge

“At 0% baseline for open-source models, it's impossible to tell whether the difficulty axis — or mere base-task incompetence — is the cause.”

Methodology

Algorithmically simple, judge-free, single-axis perturbations

Find the longest contiguous subsequence of a word list satisfying predicate constraints — no LLM-generated context, no LLM judge, exact-match scoring.

Task. Given predicates f₁…fⱼ, return the largest k such that a contiguous run satisfies all predicates. Two families: unary (arity 1: prefix/suffix/containment) and binary (arity 2: lexicographic ordering on adjacent words).

Five difficulty axes, each perturbing a baseline along one dimension:

  • Computation — unary vs lexicographic (arity).
  • Quantifier — ∃ (any) vs ∀ (all / longest), same unique answer.
  • Adversarial decoys — near-miss, near-sorted, multi-list.
  • Search space — 60K vs 26K candidates at fixed 128K tokens.
  • Context structure — scattered vs clustered decoys.

Pipelines. Synthetic (random base-26 strings, ≈128K Qwen tokens, 100 examples/variant) and real-world (filtered LongBench v2 docs, up to 365K tokens). A swap-based fixup loop guarantees a unique solution.

Strengths & Limitations

Side-by-side verdict

Strengths

  • Clean, judge-free tasks with fully specified, cheap generation — high reproducibility, no LLM-judge bias.
  • Clustered-vs-scattered decoy comparison is a striking, well-controlled demonstration that structure, not information content, drives hardness.
  • Search-space axis isolates an underappreciated factor and yields a large, clean effect (92%→10%).
  • Honest reporting of artifacts (Opus refusal footnote, 0%-first-word diagnostic, token-budget exhaustion).
  • Broad coverage across multiple reasoning-effort tiers, enabling inference-compute analysis.

Limitations

  • No sweeps over k, decoy count D, sublist count M, or cluster radius ρ — dose-response uncharacterized.
  • OSS models at 0% on most lex variants, making axis-level conclusions uninformative for them.
  • Token-budget confound: some “can't do it” numbers partly reflect “ran out of budget.”
  • ∀ variants never construct multiple satisfying sequences; the axis is only partially stress-tested.
  • Narrow task family — generalization to QA, summarization, or code reasoning is not established.

Open Questions

Where the axes point next
  1. Does performance degrade smoothly or via phase transition as target length k, decoy count D, and cluster radius ρ are swept?
  2. Would multiple genuinely satisfying sequences in ∀ variants reveal a different, harder failure mode?
  3. Can the clustered-vs-scattered gap be closed by structured prompting, scratchpad chunking, or KV-cache reordering?
  4. Does the search-space effect replicate beyond lexicographic sorting — set membership, numeric comparison, k-ary constraints with k>2?
  5. Could a distilled probe or logit-lens localize where the localization failure occurs in hidden reasoning traces?
  6. Does scaling output budget eventually close the decoy gap, or does a persistent gap remain that no compute level fixes?
  7. Can explicit retrieval / agentic multi-pass strategies beat end-to-end long-context reasoning on scattered decoys?
