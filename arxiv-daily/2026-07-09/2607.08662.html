01 Methodology

WebSwarm casts web information seeking as a recursive delegation tree T=(V,E) built at inference time. Each non-root node is an agent carrying a local objective qv plus a search mode mv; it solves locally, spawns child delegations as objective–mode pairs, and bubbles evidence upward to drive revision or aggregation.

Before a wide node expands, a Web-Probing Agent classifies evidence topology (centralized / centralized_with_gaps / distributed) and picks the expansion axis. For homogeneous siblings, a small scout batch exposes full trajectories whose process-level skill (reliable queries, dead ends) is injected into the rest.

03 Strengths

  • One recursive mechanism subsumes deep, wide, and interleaved search via explicit objective–mode coupling.
  • Web-Probing topology classifier (centralized / gaps / distributed) is a clean, transferable primitive.
  • Honest ablations: probing = cost reducer, experience reuse = accuracy lift — stated openly.
  • Hard-subset jumps (0.0→35.7 ACC; 24.5→55.8 Item F1) grow with difficulty.
  • Strong artifacts: public code, three full system prompts, tool schema, pseudocode.

04 Limitations

  • Inference-time only — no policy training; authors admit this may cap recursive collaboration.
  • Higher cost & latency than ReAct; no equal-budget Pareto comparison.
  • Small slices (200/100/76 tasks), undisclosed seeds, only 2 runs — some deltas within noise.
  • Backbone confound: stronger Kimi-K2 ReAct appears in single-agent rows, dropped from multi-agent table.
  • Text-only web tools; multimodal / GUI browsing out of scope.
  • No qualitative error taxonomy — all metrics automatic.

05 Open Questions

  1. Under an equal tool-call budget, how much of the gain over ReAct survives — orchestration or raw scaling?
  2. Are the small deltas (+3.50 ACC, +3.00 SR) stable across more seeds and the full benchmark splits?
  3. Can Web-Probing's classifier be tuned for domains lacking Wikipedia-style hubs (legal, scientific corpora)?
  4. Does sibling experience reuse cause correlated failures — all siblings repeating a wrong scout pattern?
  5. How does WebSwarm interact with RL-trained agents (Search-R1, WebThinker)? Do learned modes subsume hand-designed ones?
  6. Can the framework extend to multimodal web & GUI browsing without breaking the recursion abstraction?
  7. What is a useful error taxonomy for residual hard-sample failures — aggregation, mode mis-assignment, probing errors, verifier false-accepts?
