A single observable contract simultaneously detects silent calibration drift and adversarial channel substitution in QML pipelines — with formal detection theorems and end-to-end validation on IBM Heron r2 hardware.

Methodology

Three theoretical layers built on a behavioral-subtyping foundation:

  • Behavioral fingerprinting — The QML pipeline is characterized at runtime by expectation-value vectors F = (Tr(O₁E(ρ)), …, Tr(OₖE(ρ))) under a tomographically complete measurement family. Deviation is measured by L∞ contract deviation Δ.
  • Detection theorem (Thm 1) — Any sneaky ansatz substitution with diamond-norm separation δ must produce observable deviation ≥ δ/C(OA) on some reference input when OA is informationally complete. The tight frame constant is C = √3.
  • Finite-shot operationalization (Thm 4) — Shot budget N ≥ 8B²k log(2k/η)/γ² turns exact-expectation contracts into executable hardware checks at confidence 1 − η.
  • Tolerance decomposition — ε = εadv + εdrift separates adversarial shift from natural calibration drift under one parameter, enabling dual-mode operation in Algorithm 1.

  • Sneaky channel caught: Weak contract {Z₁Z₂} passes with deviation 0.001; complete Pauli family {X₁,Y₁,Z₁,X₂,Y₂,Z₂} catches it with worst-case deviation 0.489 on X₂ — 3.3× above tolerance εA = 0.15.
  • Budget is the threshold for informative detection: At N/10 shots, FPR degrades to 0.15; at N/100 both rates saturate at 1.00 (pure noise). Full budget is not merely sufficient — it is necessary.
  • Dual-mode viability confirmed: Tolerance interval [0.067, 0.289] is non-empty on ibm_fez, meaning both drift logging and adversarial halting work simultaneously.
  • ~100× shot savings from the tight C = √3 constant plus precomputed-reference refinement, fitting within a single IBM batched job (~30 seconds wall time).

Strengths

  • Unified dual-mode framework (drift + adversarial) under a single tolerance is elegant and practically motivated
  • Tight C = √3 constant and informative-detection definition go beyond one-sided TPR
  • End-to-end hardware validation on a real QPU (IBM Heron r2, 156 qubits)
  • Exceptional transparency: limitations and out-of-scope threats discussed thoroughly
  • Full open-source code with archived IBM Quantum job IDs for reproducibility

Limitations

  • Only validated on a 2-qubit QSVM pipeline; 8–12 qubit scaling is future work
  • Drift observed over minutes, not the multi-hour/day cycles relevant to deployment
  • Local Pauli family is informationally complete only against local-unitary substitutions — entangling attacks need costlier observables
  • O(k log k/γ²) sample complexity scales linearly with observable family size
  • Requires a trusted reference fingerprint; without one, contract degrades to consistency check

Questions for Further Exploration

  1. Can behavioral fingerprinting adapt to classical ML pipelines — e.g., verifying a deployed transformer is the declared model, not a backdoored substitute, using activation statistics as "observables"?
  2. How does the tolerance interval [ddrift, δadv/C] change over multi-day calibration cycles across different NISQ backends?
  3. Could classical shadow tomography reduce the O(k) scaling without compromising detection guarantees?
  4. What is the tight information-theoretic lower bound on shot budget — how much of the current O(k log k/γ²) upper bound is intrinsic vs. an artifact of Hoeffding + union bound?
  5. How does the framework extend to distributed quantum computing where the pipeline spans multiple QPUs with heterogeneous noise profiles?
