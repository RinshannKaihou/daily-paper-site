Directional Alignment Mitigates Reward Hacking in RL for Language Models

Reward hacking in LLM RL training is directional drift — the dominant singular subspace of parameter updates rotates sharply during hacking but stays stable (CCA ≈ 0.8) in clean training. TDGA projects RL gradients onto a trusted subspace from a short supervised warmup, delaying hacking onset from ~50 steps to 200+ steps and preserving true reward where every baseline collapses to 0.

First geometric explanation of reward hacking with a lightweight fix — the only method preventing total true-reward collapse during RL training.
Critical Edge
Tested on one 3B model, one math-reasoning task, one loophole type. Scalability to 70B+ and other hacking scenarios is unproven.

Methodology — TDGA Pipeline

  1. Dominant Direction Analysis via SVD At each step t, decompose parameter updates ΔWt = Wt − W0 via SVD. Track how rank- K dominant directions evolve across checkpoints using Canonical Correlation Analysis (CCA).
  2. Trusted Subspace Estimation Run a short clean supervised fine-tuning phase. Extract top-K output-direction subspace Uclean(K) and singular-value weights that capture each direction's relative importance.
  3. Gradient Projection During RL Project each RL gradient onto the trusted subspace using singular-value weighting. Only the projected component updates parameters, suppressing off-subspace directions that enable hacking. Rank K controls the robustness–flexibility tradeoff.

Strengths

  • Elegant geometric interpretation — directional drift is measurable and intuitive
  • Lightweight: only requires a short supervised warmup plus a projection step, minimal compute overhead
  • Only method maintaining non-zero true reward at two epochs where all baselines fully collapse
  • Transparent tradeoff analysis across projection ranks (1, 5, 10)

Limitations

  • Single task domain (math reasoning) and single loophole type (in-context)
  • Requires clean supervised warmup data — unavailable for some tasks
  • Fixed trusted subspace may become stale as the model evolves during RL
  • Only tested at 3B scale; 70B+ scalability unknown

Open Questions

  • Does periodically refreshing the clean subspace further improve robustness?
  • Can CCA similarity serve as an early-warning signal for reward hacking before performance degrades?
  • How does TDGA scale to larger models (70B+) and reward-model-based RLHF?
  • Does the method transfer to multi-turn agentic RL with cumulative reward hacking?
  • What is the sensitivity to the quality and quantity of clean warmup data?
