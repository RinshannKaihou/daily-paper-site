MultiHaluDet: Multilingual Hallucination Detection via LLM Hidden State Probing

A four-stage stacking framework probes frozen LLM hidden-state trajectories across all layers — dynamic layer sampling, multi-scale attention, and out-of-fold ensemble meta-learning detect hallucinations at state-of-the-art accuracy and generalize across four languages.

Riasad Alvi · Nurul Labib Sayeedi · Md. Faiyaz Abdullah Sayeedi — United International University & BRAC University

hallucination-detection multilingual-NLP LLM-internal-states stacking-ensemble cross-lingual-transfer
Key Metric
98.55% AUROC
HaluEval / LLaMA2-7B — 98.43% on Mistral-7B, 98.30% on TriviaQA
Why This Matters
Frozen LLM hidden states carry hallucination signals that transfer across languages without fine-tuning.
Critical Edge
White-box access required — incompatible with proprietary models. Amharic drops to 78.5%. Multilingual eval uses translated benchmarks, not naturally occurring data.

Methodology

STAGE 1
Dynamic Feature Extraction
Single forward pass through frozen LLM. Dynamic layer sampling maps L layers to K=32 uniform indices. Per-layer descriptors: l2 norm, mean, std, sparsity, kurtosis, MAD.
❯
STAGE 2
Multi-Scale Attention
Projection to H=384. Multi-scale pooling + learned position-wise gates. 6 Pre-LN Transformer encoders (8 heads). Self-attention pooling for adaptive depth selection.
❯
STAGE 3
Out-of-Fold Stacking
5-fold stratified CV trains fresh instances on K−1 folds. Extracts gated-fusion representations for held-out fold → complete OOF matrix, no data leakage.
❯
STAGE 4
Ensemble Meta-Learner
6 classifiers (XGBoost, LightGBM, RF, SVM, MLP, LR). Log-odds fused by l2-regularized logistic meta-regressor. Threshold via Youden's J statistic.
Loss: BCE + focal + asymmetric + contrastive (label smoothing)  |  Augment: Mixup + CutMix  |  Optim: AdamW (lr=2e-4, wd=6e-5), 45 epochs, early stop p=15  |  HW: RTX 5060 Ti 16GB

Strengths & Limitations

Strengths

  • No language-specific fine-tuning — leverages frozen LLM representations directly
  • Architecture-agnostic dynamic layer sampling across LLM depths
  • 13 baselines, 2 benchmarks, 2 architectures, 4 languages, full ablation
  • OOF stacking provides principled overfitting protection
  • Multi-scale attention captures both fine and coarse hallucination signals
  • Public code repository with full reproducibility details

Limitations

  • White-box access required — incompatible with GPT-4, Claude, proprietary APIs
  • Full-depth hidden-state extraction incurs significant memory overhead
  • Translated benchmarks, not naturally occurring multilingual data
  • 78.5% on Amharic vs 98.55% English — inherits base LLM limitations
  • Binary classification only — misses partial errors, numerical fabrications
  • Computational cost of the 4-stage pipeline not analyzed
04

Open Questions

  1. How does the framework handle nuanced hallucination types beyond binary classification — partial factual errors, numerical fabrications, logical contradictions?
  2. What is the computational overhead of the full four-stage pipeline vs single-pass probing methods?
  3. Can it be adapted for real-time detection during autoregressive generation rather than post-hoc analysis?
  4. How does performance scale with larger LLMs (LLaMA-70B, Mistral-8x22B) — does dynamic layer sampling hold?
  5. Would training on naturally occurring multilingual datasets close the gap for low-resource languages like Amharic?
  6. Can the multi-scale attention mechanism be interpreted to identify which layers carry the strongest hallucination signals, and does this vary by language?
