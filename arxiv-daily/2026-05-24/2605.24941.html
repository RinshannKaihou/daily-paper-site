Biased personality memories silently rewrite tool-call parameters in LLM agents

Long-term memory stores encode lifestyle preferences as implicit steering vectors that shift tool-call arguments — even when the task is entirely unrelated.

Dabas, Jeong, Jin, Jia — Virginia Tech · 7 frontier models · 105 adversarial scenarios · 6,062 real MCP tools audited
Key Metric
+3.6
deflection on a 1–5 scale across 7 frontier models. 608 / 6,062 real MCP tools flagged vulnerable.
Why This Matters
Biased memories silently rewrite tool parameters in production agents — no current defense fully stops it.
Critical Edge
Only 105 scenarios; mechanistic analysis on a single open-weight model (Llama-3.3-70B); single-turn, single-tool only — real-world multi-tool drift dynamics unknown.

Methodology

Three-Condition Framework

Each scenario tested under M∅ (no memory), Mn (neutral memories), Mb (biased personality memories). Drift confirmed when θb ≠ θ∅ while θn ≈ θ∅ — isolating bias-driven deviation from mere memory presence.

MEMDRIFT Benchmark

105 scenarios across 5 bias dimensions (speed, frugality, minimalism, risk permissiveness, autonomy) × 7 domains (medical, finance, legal, infra, education, e-commerce, marketing). Three-stage adversarial pipeline with 4-iteration self-improving refinement.

Production Memory Architectures

Tested with Mem0, MemPalace, SimpleMem — all strip personal context during storage, encoding preferences as decontextualized global directives. Drift persists or worsens.

Mechanistic Probes

On Llama-3.3-70B: contrastive steering vector projection (Cohen's d, AUROC) + attention redistribution analysis across layers 40–79, partitioning input into semantic regions.

Results & Highlights

UniversalAll 7 frontier models exhibit substantial drift (sb 3.0–4.2 range; neutral sn ≈ 1.5). Even reasoning models with extended thinking are affected.
ProductionDrift persists under Mem0, MemPalace, SimpleMem. Memory architectures convert preferences into global directives, making drift comparable to or worse than direct injection.
Real-World608 / 6,062 tools on Smithery MCP registry flagged. Confirmed: GitHub label_write update→delete; GitLab create_project private→public (HIPAA exposure).
MechanismBiased memories act as implicit steering vectors along the same latent direction as explicit instructions. Attention shifts from tool schema/query toward memory entries with keyword overlap.
DefenseDefensive prompts reduce drift by only Δ = −0.52 on GPT-5.4. Self-ReCheck filter: 61% recall on realistic cases — does not generalize.
BehaviorModels rarely request clarification in unconstrained mode — they proceed directly to the drifted tool call without signaling uncertainty.

Assessment

Strengths

  • Genuinely novel failure mode — no prior work examines the memory × tool-use intersection
  • Exceptionally thorough: 7 models, 3 production architectures, real-world audit, mechanistic analysis
  • Three-condition design cleanly eliminates confounds
  • Causal explanation via steering vectors + attention redistribution, not just correlation
  • Immediate practical relevance — MCP ecosystem, ChatGPT memory, Claude Code

Limitations

  • 105 scenarios (3 per bias×domain pair) — modest coverage
  • Mechanistic analysis limited to Llama-3.3-70B, not among the 7 evaluated frontier models
  • Single-turn, single-tool evaluation — multi-tool chain dynamics unknown
  • Clean personal/professional separation doesn't reflect real-world ambiguity
  • Defense evaluation limited to prompt-based and filter-based approaches only
  • English-only — cross-lingual drift behavior unexplored

Open Questions

Can architectural mechanisms (e.g., memory-context gating at the tool-call boundary) fully eliminate drift without degrading useful personalization?
How does drift compound in multi-tool workflows where drifted outputs feed into subsequent calls?
Does the steering-vector equivalence hold for closed-source frontier models — can it be detected via black-box probing?
Could contrastive fine-tuning on MEMDRIFT-style examples instill genuine contextual discernment, generalizing to held-out bias dimensions?
What is the interaction between memory-induced drift and adversarial memory injection — does drift expand the attack surface?
How should the boundary between legitimate personalization and harmful drift be operationalized when personal and professional contexts overlap?
