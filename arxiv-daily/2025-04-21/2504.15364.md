> **一句话总结：** KeyDiff提出了一种无需训练的KV cache驱逐方法，通过保留几何上最具独特性的key（低成对余弦相似度）来实现近无损的长上下文LLM推理压缩，在8K cache预算下性能下降仅0.04%，推理延迟降低30%，且不依赖attention score，可直接与FlashAttention兼容。

> **TL;DR:** KeyDiff is a training-free KV cache eviction method that retains geometrically distinctive keys (those with low average pairwise cosine similarity to other cached keys) and discards redundant ones. It is designed for block-wise inference under strict memory constraints, where attention-based eviction methods fail because they cannot see future tokens. KeyDiff achieves a mere 0.04% accuracy drop on LongBench with an 8K cache budget (23% KV cache reduction) for Llama 3.1-8B and Llama 3.2-3B, near-baseline performance on Math-500 reasoning with DeepSeek-R1-Distill-Llama-8B, and up to 30% lower end-to-end latency than TOVA and SnapKV because it does not need to materialize attention weights and is fully compatible with FlashAttention. Accepted at NeurIPS 2025.

## 核心贡献

- **关键洞察：** 发现LLM推理中几何上独特的key（与其他key的成对余弦相似度较低）倾向于获得更高的attention score，这一现象独立于query选择，且本质上解释了attention sink现象。
- **全新驱逐方法：** 提出KeyDiff，一种仅基于key间余弦相似度的attention-free驱逐策略，不依赖attention score、不需要访问未来token，可使用高效anchor向量将复杂度从O(n^2)降至O(n)。
- **理论支撑：** 通过Lemma 3.1和Theorem 3.2严格证明了key diversity与attention score之间的数学关系，并证明KeyDiff等价于求解最大化key多样性的最优子集选择问题。
- **卓越性能：** 在LongBench上8K cache预算（~23%压缩率）下，Llama 3.1-8B和Llama 3.2-3B的性能下降仅0.04%；在Math-500推理基准上DeepSeek-R1-Distill-Llama-8B接近无损基线性能。
- **工程优势：** 兼容FlashAttention，推理延迟比TOVA和SnapKV降低30%，适用于资源受限的边缘设备部署场景。

## Key Contributions

- **Core Insight:** Identified that geometrically distinctive keys (those with low pairwise cosine similarity to other keys) tend to receive higher attention scores regardless of the query, which also provides an alternative explanation for the attention sink phenomenon.
- **Attention-Free Eviction Method:** Proposed KeyDiff, which selects keys to retain based solely on their cosine similarity to the mean of all cached keys (the "anchor vector"), without requiring access to attention weights or future tokens. An efficient O(n) variant replaces the full pairwise O(n^2) computation.
- **Theoretical Foundation:** Proved Lemma 3.1 (bounding attention scores via cosine similarity with queries) and Theorem 3.2 (relating cosine similarities among keys, queries, and key means), formally establishing that KeyDiff selects the subset of keys that maximizes key diversity.
- **Near-Lossless Compression:** Achieved <= 0.04% accuracy drop on LongBench with 8K cache budget (~23% KV cache reduction) for Llama 3.1-8B-Instruct and Llama 3.2-3B-Instruct, and only 1.5% drop with 6K budget (~33% reduction).
- **FlashAttention Compatibility:** Since KeyDiff does not materialize attention weights, it can use FlashAttention directly, yielding up to 30% lower end-to-end inference latency compared to TOVA and SnapKV.
- **Block-Wise Inference:** Designed specifically for strict memory-constrained environments where prompts are processed in blocks, avoiding the intermediate memory spikes of full-prompt processing.

## Methodology

KeyDiff operates within a block-wise inference framework where the input prompt is divided into smaller blocks (default B=128) processed sequentially by the transformer model. After processing each block, the eviction policy is applied to keep the KV cache within budget.

**Key observation (Section 3.1):** The authors empirically observe that keys with lower average pairwise cosine similarity to other cached keys consistently receive higher attention scores across different heads, layers, and queries. This suggests that key diversity -- a property computed solely from keys, independent of queries -- can serve as a proxy for token importance.

**KeyDiff eviction policy (Section 3.2):** Given a cache with n entries and budget N (where n > N), KeyDiff computes:
1. The mean of all normalized cached keys (the "anchor vector"): mu = (1/n) * sum(k_i / ||k_i||)
2. Cosine similarity of each key with this anchor: s_i = CosSim(mu, k_i)
3. Retains the N keys with the lowest similarity scores (most distinctive from the mean)

This efficient O(N+B) variant replaces the original O(n^2) pairwise computation. The authors show both variants select the same keys under mild conditions.

**Sliding window extension:** For tasks like reasoning and coding where recent tokens matter, a percentage of the cache budget can be reserved for a sliding window, adding no complexity overhead.

**Theoretical analysis (Section 3.3):**
- Lemma 3.1 bounds a new key's attention weight from below in terms of its cosine similarity with the query.
- Theorem 3.2 relates the cosine similarity between a key and the mean key to the key's similarity with the query.
- Combined, these results show that as cosine similarity between a key and the anchor decreases (i.e., the key is more distinctive), it tends to be more aligned with queries and receive higher attention -- confirming KeyDiff's selection criterion.

## Results & Highlights

- **Needle-In-A-Haystack:** KeyDiff performs comparably to TOVA, SnapKV, and sink attention on short documents but outperforms all three as document length increases with a 6K cache budget (Figure 6).
- **LongBench (English subset):**
  - Llama 3.1-8B-Instruct: KeyDiff achieves the best average scores across all cache budgets (2K, 4K, 6K, 8K) with block processing (B=128).
  - Llama 3.2-3B-Instruct: Similar trends; KeyDiff significantly outperforms attention-based methods under block-wise processing.
  - At 8K cache budget: <= 0.04% accuracy gap from the non-evicting baseline.
  - At 6K cache budget (~33% compression): only 1.5% accuracy drop.
  - KeyDiff is particularly strong on PassageRetrieval-en, which tests long-range dependency recognition.
- **Math-500 Reasoning:** Near baseline performance for DeepSeek-R1-Distill-Llama-8B, especially when augmented with a sliding window.
- **Latency:** Up to 30% lower TTFT compared to TOVA and SnapKV when using FlashAttention, since KeyDiff does not need to materialize the attention matrix.
- **Complexity:** O(N+B) runtime and memory, matching TOVA and significantly better than H2O (O(NB+B^2)) and SnapKV (O((N+B)L)).

## Critical Assessment

**Novelty (High):** The core insight -- that key geometric distinctiveness (measured via cosine similarity with the mean key) can serve as a query-independent proxy for attention-based importance -- is genuinely novel. While prior work observed attention sinks and key redundancy, KeyDiff is the first to formally connect key diversity to attention scores and exploit it for eviction without accessing attention weights.

**Rigor (Strong):**
- Theoretical proofs (Lemma 3.1, Theorem 3.2) provide formal justification.
- Extensive experiments across multiple model families (Llama 3.1/3.2, Qwen 2.5, DeepSeek-R1-Distill), multiple benchmarks (LongBench, Needle-In-A-Haystack, Math-500), and multiple cache budgets.
- Comprehensive ablation studies in supplementary material.

**Reproducibility (Good):** The method is described in sufficient detail. The efficient variant's formula is clearly stated. However, no official code repository is linked in the paper, which may hinder direct reproduction. The block-wise inference setup and evaluation protocols are well-specified.

**Significance (High):** Practical impact for edge/on-device LLM deployment where memory is strictly bounded. The FlashAttention compatibility is a real engineering advantage. The theoretical contribution connecting key geometry to attention is also of independent interest for understanding transformer internals.

**Potential Concerns:**
- Evaluation is limited to GQA-based models (Llama, Qwen). The authors acknowledge this limitation and note that extending to Multi-Head Latent Attention (as in DeepSeek-V2/V3) is future work.
- The 0.04% number is for 8K cache budget (only 23% compression), which is relatively conservative. At more aggressive compression (6K/33%), the gap widens to 1.5%, and at 2K it becomes substantial.
- The block-wise inference paradigm (B=128) is assumed but not all deployment scenarios use block processing; the comparison with attention-based methods under standard (full-prompt) processing shows less advantage.
- No comparison with KV cache quantization methods, which could be complementary.
- The theoretical results rely on asymptotic assumptions (n -> infinity) and bounded key norms that may not always hold in practice.

## Strengths

- Elegant and simple core idea: retain distinctive keys, discard redundant ones.
- No training required; immediately applicable to any transformer model.
- Fully compatible with FlashAttention, unlike attention-based eviction methods.
- Strong theoretical foundation connecting key geometry to attention scores.
- Comprehensive evaluation across diverse tasks (retrieval, reasoning, summarization, coding).
- Low computational overhead: O(N+B) per eviction step.
- The sliding window extension handles reasoning/coding tasks well.

## Limitations

- Only evaluated on GQA attention mechanism (Llama, Qwen); not tested on MHA or MLA models.
- Performance degrades noticeably at very aggressive compression rates (2K cache budget).
- The anchor vector is a simple mean of cached keys; more sophisticated representations might perform better.
- No investigation of interaction with KV cache quantization (e.g., KVQuant, KIVI).
- Block processing (B=128) is required for strict memory constraints; in less constrained settings, attention-based methods may perform comparably.
- The negative correlation between key similarity and attention scores is empirically observed but may not hold universally across all model architectures or training regimes.

## Relevance

本文对LLM推理效率的研究与KV cache管理直接相关，提供了从key几何特性理解attention机制的新视角。其理论分析揭示了key多样性与attention score之间的数学联系，对LLM可解释性研究（特别是attention sink现象的本质理解）有参考价值。同时，KeyDiff的block-wise推理框架和内存约束设计为在计算可靠性受限的边缘设备上部署LLM提供了工程参考。

## Questions for Further Exploration

- How does KeyDiff interact with KV cache quantization methods? Could combining key-similarity-based eviction with adaptive quantization yield further memory savings?
- Does the correlation between key distinctiveness and attention scores hold for models with Multi-Head Latent Attention (e.g., DeepSeek-V2/V3) or Grouped-Query Attention with different group sizes?
- Can the anchor vector be improved beyond a simple mean -- for instance, using a learned prototype or a clustering-based representative?
- What is the effect of block size B on the quality of eviction decisions? Is there an optimal block size that balances memory strictness with eviction quality?
- How does KeyDiff perform on multimodal or cross-attention architectures where the key-query relationship may differ from decoder-only transformers?
- Could the theoretical connection between key geometry and attention be leveraged for other purposes, such as detecting hallucinations or identifying important context windows during inference?
