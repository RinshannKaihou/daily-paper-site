MoE Parallel Folding: Heterogeneous Parallelism Mappings for Efficient Large-Scale MoE Model Training with Megatron Core

arXiv: 2504.14960

Date: 2025-04-21

https://arxiv.org/abs/2504.14960

一句话总结： 本文提出 MoE Parallel Folding 策略，将 Transformer 中 Attention 层和 MoE 层的并行化映射解耦，配合灵活的 token-level dispatcher，在 H100 GPU 上对 Mixtral 8x22B 达到 49.3% MFU，对 Qwen2-57B-A14B 达到 39.0% MFU，显著优于现有方法。
TL;DR: NVIDIA's Megatron-Core team introduces MoE Parallel Folding, which decouples the parallelism mappings of attention and MoE layers in Transformer models so each layer type can use its own optimal parallel configuration (TP, EP, CP, DP, PP). Combined with a flexible token-level dispatcher supporting both token-dropping and dropless training, the framework achieves up to 49.3% MFU on Mixtral 8x22B and 39.0% MFU on Qwen2-57B-A14B on H100 GPUs, scales to 1,024 GPUs, and supports 128K context lengths. This is a practical systems paper with code integrated into Megatron-Core, delivering 3-6 percentage point MFU improvements over prior best methods on both coarse- and fine-grained MoE architectures.

核心贡献

  • MoE Parallel Folding 策略：首次将 Transformer 中 Attention 层与 MoE 层的并行化映射完全解耦。Attention 层使用 TP x CP x DP x PP 四维并行，MoE 层使用 TP(ETP) x EP x DP(EDP) x PP 四维并行，两者独立配置，仅要求 PP 组一致。
  • 灵活的 token-level dispatcher：设计了一个统一的 token 调度器，同时支持 token-dropping 和 token-dropless 两种训练模式，在五个并行维度（TP, EP, CP, DP, PP）上保持数值正确性，并能处理动态张量形状。
  • 通信优化：通过将通信密集的并行维度"折叠"到高带宽的节点内网络（NVLink），减少跨节点通信开销。EP 比 ETP 通信效率更高，可替代 ETP 并与 Attention 层的 TP 折叠。
  • 全面的实验验证：在 Mixtral 8x22B、Llama3-8x70B、Qwen2-57B-A14B、Mixtral-8x22B-G8T8 四个模型上验证，涵盖粗粒度和细粒度 MoE 架构，扩展到 1024 GPU 和 128K 序列长度。
  • 代码已开源：集成到 NVIDIA Megatron-Core 框架中，可直接使用。

Key Contributions

  • MoE Parallel Folding: The first approach that decouples parallelization strategies for attention and MoE layers in Transformers, allowing each layer type to adopt its own optimal 4D parallel configuration independently (only PP groups must remain consistent).
  • Flexible token-level dispatcher: A unified dispatcher supporting both token-dropping and token-dropless MoE training across all five parallelism dimensions (TP, EP, CP, DP, PP), accommodating dynamic tensor shapes and coordinating different parallelism schemes between attention and MoE layers.
  • Communication-efficient folding: By folding MoE parallel groups with attention groups, communication-intensive operations are kept within high-bandwidth intra-node NVLink connections rather than traversing lower-bandwidth inter-node InfiniBand.
  • Strong empirical results: Achieves 49.3% MFU (Mixtral 8x22B), 41.6% MFU (Llama3-8x70B), 39.0% MFU (Qwen2-57B-A14B), and 28.8% MFU (Mixtral-8x22B-G8T8) on H100 GPUs, consistently outperforming FSDP, FSDP+EP, TP+EP+DP, and standard Megatron-Core 5D parallelism.
  • Production-ready: Code is integrated into the NVIDIA Megatron-Core framework (Megatron-LM repository).

Methodology

The paper builds on NVIDIA's Megatron-Core framework and introduces two main technical innovations:

1. MoE Parallel Folding:

  • In standard approaches, the EP group is a subgroup of the DP group, which restricts the scalability of expert parallelism (max EP degree bounded by DP degree).
  • MoE Parallel Folding flattens the parallelism mappings of the attention layer and allows MoE layer model parallelism to be folded with arbitrary sub-groups of attention.
  • Attention layers use: TP x CP x DP x PP
  • MoE layers use: ETP (Expert-Tensor-Parallelism) x EP x EDP (Expert-Data-Parallelism) x PP
  • The key insight: attention operates at the whole-sequence level (dense computation, needs TP/CP communication), while MoE processes individual tokens (sparse, better suited for EP). Forcing them into the same parallelism mapping is suboptimal.
  • By decoupling, EP can replace ETP (since EP has lower communication overhead than ETP), and the folded groups can be kept within NVLink domains.

2. Token-Level Dispatcher:

  • Handles token routing across all five parallelism dimensions.
  • Forward pass: Router determines expert assignments -> Permutation (reorder tokens) -> All-to-All-V across EP groups -> AllGather-V within ETP groups -> Expert FFN computation -> ReduceScatter-V within ETP groups -> All-to-All-V to restore tokens -> Un-permutation.
  • Supports both token-dropping (with capacity factor) and token-dropless (MegaBlocks-style) training.
  • For token-dropping, empirically shows sub-sequence-based dropping does not hurt convergence compared to full-sequence dropping, while eliminating costly cross-rank logit gathering.

3. FP8 Training Support:

  • Experiments with FP8 delayed scaling show ~1.26-1.30x speedup over BF16, with MoE Parallel Folding still providing benefits in the FP8 regime (631.7 TFLOPS peak).

Results & Highlights

  • Performance Comparison (128 GPUs, global batch=256, seq=4096):
  • Mixtral 8x22B: 49.3% MFU (vs. 46.3% for MCore baseline, 40.4% for TP+EP+DP, 23.9% for FSDP+EP)
  • Qwen2-57B-A14B: 39.0% MFU (vs. 35.3% for MCore baseline)
  • Llama3-8x70B: 41.6% MFU (vs. 40.1% for MCore baseline); this model cannot be trained with FSDP or TP+EP+DP due to OOM
  • Mixtral-8x22B-G8T8 (fine-grained): 28.8% MFU (vs. 19.8% for MCore baseline -- a 9 percentage point improvement)
  • Strong Scaling: Maintains high MFU up to 1,024 GPUs with minimal drops. For Llama3-8x70B, MFU only drops from 43.7% to 41.5% when scaling from 128 to 1,024 GPUs.
  • Context Scaling: Supports up to 128K token sequences. MFU drops from 47.6% to 42.9% (Mixtral 8x22B) and 38.7% to 35.9% (Qwen-57B14A) when scaling from 4K to 128K context.
  • Ablation Studies: Demonstrate that EP is significantly more communication-efficient than ETP in MoE layers; fine-grained MoE has lower compute-to-communication ratio with communication dominating (>70% latency) when crossing NVLink domain boundaries; MoE Parallel Folding consistently discovers better configurations.
  • Accuracy Validation: Training and validation loss curves for Mixtral 8x7B align with standard MCore v0.9 baseline, confirming numerical correctness.
  • Fine-grained MoE gap: Fine-grained MoE models consistently achieve lower MFU than coarse-grained due to higher communication volume (more experts, more active experts per token) and smaller GEMM sizes reducing compute efficiency.

Critical Assessment

Novelty (Moderate-High): The core idea of decoupling parallelism mappings between attention and MoE layers is genuinely novel and practically impactful. While hybrid parallelism itself is well-established, the specific insight that attention and MoE layers have fundamentally different communication patterns (sequence-level vs. token-level) and should therefore use different parallelism configurations is a meaningful contribution. However, the individual parallelism techniques (TP, EP, CP, DP, PP) are all well-known; the novelty is in their composition and decoupling.

Rigor (High): The paper provides thorough experimental evaluation across four different MoE architectures (two coarse-grained, two fine-grained), multiple GPU scales (64-1024), context lengths (4K-128K), and ablation studies with latency breakdowns. Accuracy validation with loss curves is also provided. The comparison baselines are fair (all implemented in the same Megatron-Core framework). MFU as a metric is standard and well-defined.

Reproducibility (High): Code is open-sourced in the NVIDIA Megatron-Core repository. The paper provides detailed parallel configuration tables, experimental setup descriptions, and even a Python code listing for parallel group generation. Anyone with access to H100 GPUs can reproduce the results.

Significance (High for practitioners, Moderate for theorists): For teams training large-scale MoE models, this work provides significant practical value (3-9 percentage point MFU improvements). The framework is production-ready. However, the paper is primarily a systems optimization contribution rather than a conceptual or algorithmic breakthrough.

Red flags / concerns:

  • The MFU numbers (39-49%) may seem modest relative to theoretical peak, but this is expected for MoE models due to their inherently low compute-to-parameter ratio. The paper does a reasonable job explaining this.
  • Fine-grained MoE still achieves relatively low MFU (28.8% even with folding), suggesting significant room for further optimization in this regime.
  • The paper benchmarks only on NVIDIA H100 hardware with NVLink/InfiniBand interconnect. Performance characteristics may differ on other hardware configurations.
  • Token-dropping with CF=1 is used for benchmarking to avoid load imbalance jitter, which may not reflect real training scenarios where dropless training is preferred for model quality.
  • The submission history shows v1 on April 21, 2025 and v3 on March 2, 2026, suggesting iterative refinements but also a relatively long review/revision cycle.

Strengths

  • Clean, well-motivated core idea: decoupling parallelism for attention vs. MoE layers based on their different computational characteristics.
  • Comprehensive evaluation across diverse MoE architectures (coarse-grained and fine-grained), multiple scales, and context lengths.
  • Strong practical impact: code is production-ready in Megatron-Core, the de facto standard framework for large-scale LLM training.
  • Honest about limitations of fine-grained MoE efficiency.
  • The token dispatcher design is elegant in handling both dropping and dropless paradigms with sub-sequence dropping as a practical default.
  • Ablation studies with latency breakdowns provide actionable insights for practitioners configuring MoE training.

Limitations

  • Only evaluated on NVIDIA H100 (DGX) hardware; results may not transfer to other GPU generations or accelerator types.
  • Fine-grained MoE models still have low MFU (28.8%), with communication dominating over 70% of latency at scale -- the paper acknowledges but does not fully solve this.
  • Does not address MoE-specific training challenges beyond parallelism, such as load balancing, expert collapse, or routing optimization.
  • The paper focuses exclusively on training throughput (MFU) and does not report downstream task performance or model quality metrics beyond training/validation loss curves.
  • No comparison with DeepSpeed-MoE or other competing MoE training frameworks beyond those implementable in Megatron-Core.
  • Context parallelism folding with EP is explored but the gains are relatively modest compared to the core attention/MoE decoupling.
