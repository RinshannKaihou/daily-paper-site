ExComm: Exploration-Stage Communication for Error-Resilient Agentic Test-Time Scaling

Woomin Song, Beomjun Kim, Daewon Choi, Sai Muralidhar Jayanthi, Saket Dingliwal, Jinwoo Shin, Aram Galstyan | 2026-05-21

test-time-scaling multi-agent-systems error-recovery agentic-reasoning belief-consistency

67–71.5% of intermediate errors in parallel agentic reasoning produce detectable cross-agent factual conflicts; resolving them via soft belief updates and tool-verified correction yields 5.0–5.7% average gains over the strongest baselines.

Key Metric

+5.7% avg on AIME & GAIA (Gemini), +5.0% (Qwen); error recovery 38.7% / 61.8% vs. 19.9% / 57.7% for best baselines. N=4 ExComm beats all N=8 baselines.

Why This Matters

Most errors in parallel agentic reasoning are detectable through cross-agent conflict — meaning error recovery is possible without relying on unreliable self-correction. The N=4 > N=8 cost efficiency makes this practically deployable.

Critical Edge

Soft belief updates append corrections as external suggestions rather than overwriting beliefs, so agents can safely ignore incorrect verifier resolutions. Combined with trajectory diversification to prevent communication-induced diversity collapse.

Methodology

ExComm operates on top of a base agent loop where each agent maintains a structured belief state B, a plan P, and a toolkit T. Two modules are applied after each execution step across N parallel agents:

Online Belief Consistency Module: Gathers beliefs from all agents, extracts mutually exclusive factual conflicts, and resolves each via a dedicated ReAct-style verification loop (decoupled from problem-solving) using tools like Python interpreters and web search. Resolutions are transmitted only to involved agents as targeted feedback, applied as soft updates (appending rather than replacing).

Trajectory Diversification Module: Performs centralized, batched analysis of all agent plans to identify redundancy and underexplored directions, issuing at most one diversification directive per agent per step — enforced at the plan level while leaving task execution autonomous.

Strengths & Limitations

Strengths

  • Clean motivation grounded in empirical error-type analysis (67–71.5% conflicting errors).
  • Soft belief update balances correction with robustness — validated by concrete case study.
  • Comprehensive evaluation: multiple benchmarks, models, scaling regimes, and baselines.
  • N=4 beats N=8 baselines at lower cost with comparable latency.

Limitations

  • Not designed for solution-stage aggregation — complementary, not a replacement.
  • Verifier quality is a bottleneck; incorrect resolutions can mislead agents.
  • GAIA results on 50-sample subset limits statistical confidence.
  • No open-source code; depends on proprietary API models.
  • Does not address consistent logical errors shared across agents (5.6–13.6%).

Questions for Further Exploration

  1. How does ExComm perform on non-mathematical open-ended reasoning where factual conflicts are harder to define?
  2. Can the verifier be replaced with a smaller specialized model to reduce cost?
  3. What is the theoretical relationship between agent count and conflicting-error detection probability?
  4. Can exploration-stage communication be combined with solution-stage refinement for additive gains?
  5. How robust is the soft update mechanism under systematically biased or adversarial verification?
