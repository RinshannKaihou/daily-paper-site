> **一句话总结：** 本文提出了一种基于无分支选择、Padé有理逼近和周期对齐的常量时间实现方法，使得ReLU、sigmoid、tanh、GELU和Swish五种激活函数在ARM Cortex-M4上对所有输入均执行相同的时钟周期（88/108 cycles），从根本上消除了嵌入式推理中激活函数的时序侧信道泄漏。

> **TL;DR:** This paper proposes a constant-time implementation methodology for activation functions on ARM Cortex-M4 microcontrollers, combining branchless masking-based selection, fixed-cost Padé rational approximation, dummy arithmetic, and cycle alignment. The protected ReLU, sigmoid, tanh, GELU, and Swish all execute in identical cycle counts (88 cycles for 3-function, 108 cycles for 5-function) regardless of input, eliminating timing side-channel leakage while maintaining high numerical accuracy.

## 核心贡献

- 提出了一套完整的常量时间激活函数实现方法学，包含无分支掩码选择、固定代价Padé有理逼近、虚拟运算填充和周期对齐四个核心组件
- 证明现有的去同步化（随机延迟）防御手段在模板攻击下仍然可以被攻破——三个激活函数均可在约2000次测量后被正确识别
- 在ARM Cortex-M4平台上实现了五种激活函数（ReLU、sigmoid、tanh、GELU、Swish）的常量时间版本，所有函数均执行完全相同的时钟周期数
- 受保护的非线性激活函数（sigmoid、tanh、Swish）比未受保护版本更快（88 cycles vs. 157–403 cycles），同时保持极低的数值误差（sigmoid MSE < 10^-11）

## Key Contributions

- Propose a constant-time implementation methodology for activation functions on embedded microcontrollers, combining branchless masking-based selection, fixed-cost Padé rational approximation, dummy arithmetic, and cycle alignment
- Demonstrate that desynchronization-based countermeasures remain vulnerable to profiled template-based timing attacks (correct classification within ~2,000 measurements)
- Instantiate the methodology on five activation functions (ReLU, sigmoid, tanh, GELU, Swish) with identical cycle counts: 88 cycles (3-function setting) and 108 cycles (5-function setting) on ARM Cortex-M4
- Show that protected nonlinear implementations are faster than unprotected library versions while achieving high numerical fidelity (sigmoid MSE: 2.93×10^-12, tanh MSE: 6.14×10^-10)

## Methodology

The methodology consists of four tightly integrated components designed to ensure input-independent execution time:

1. **Branchless, masking-based selection**: Instead of conditional branches, the implementation uses bit-level masking on IEEE-754 floating-point representations. A comparison generates an all-zero or all-one 32-bit mask, which selects between candidate values via bitwise AND/OR operations. This eliminates data-dependent control flow entirely.

2. **Fixed-cost Padé rational approximation**: All nonlinear activations share a single rational approximation of tanh, denoted R_tanh(x) = x · P(x²)/Q(x²), where P and Q are low-degree polynomials. This requires a fixed sequence of additions, multiplications, and one division — no data-dependent loops or interval selection. Sigmoid, GELU, and Swish are derived from R_tanh via standard mathematical identities.

3. **Masking-based saturation**: For large-magnitude inputs, nonlinear functions approach simple limiting forms (e.g., tanh → ±1). Saturation is applied using the same branchless masking technique, with thresholds chosen to balance approximation error against saturation error. For tanh, the threshold τ ≈ 4.97 is found by solving |tanh(τ) - R_tanh(τ)| = 1 - tanh(τ).

4. **Cycle alignment with dummy arithmetic**: ReLU is inherently simple (one compare + conditional move), so it is augmented with dummy R_tanh-based computation (whose result is discarded) plus minimal NOP padding to align with the common execution budget. This preserves a realistic arithmetic profile rather than relying on long NOP chains.

The approach is implemented in C++ without inline assembly. Compiler optimization effects are carefully managed: the branchless structure is verified under -O1, -O2, and -O3, and the R_tanh function is isolated to control inlining. The target platform is an STM32F411 (ARM Cortex-M4 at 84 MHz), with timing measured via the DWT cycle counter.

## Results & Highlights

**Timing leakage baseline**: Unprotected ReLU/sigmoid/tanh show clearly distinguishable timing profiles — ReLU at ~12 cycles, sigmoid at ~221 cycles with input-dependent variation, tanh at ~403 cycles with input-dependent variation.

**Template attack on desynchronization**: Adding random delays visually obscures timing patterns, but a Gaussian template attack (10,000 profiling traces per class) correctly identifies the activation function in all three cases with ~2,000 attack measurements.

**Constant-time achievement**:
- 3-function setting (ReLU, sigmoid, tanh over [-8, 8]): all execute in exactly 88 cycles (1.048 μs)
- 5-function setting (+ GELU, Swish over [-500, 500]): all execute in exactly 108 cycles (1.286 μs)

**Speed improvements for nonlinear functions**:
- Sigmoid: 221 → 88 cycles (60% faster)
- Tanh: 403 → 88 cycles (78% faster)
- Swish: 174 → 108 cycles (38% faster)
- GELU: 103 → 108 cycles (slight overhead but eliminates 503-cycle worst case)

**Numerical accuracy** (over [-500, 500]):
| Function | MSE | Max Abs Error |
|----------|-----|---------------|
| Sigmoid | 5.78×10^-12 | 4.54×10^-5 |
| Tanh | 1.72×10^-11 | 9.08×10^-5 |
| GELU | 4.45×10^-10 | 4.17×10^-4 |
| Swish | 2.96×10^-9 | 1.11×10^-3 |

## Critical Assessment

**Novelty**: The application of constant-time implementation principles (well-known in cryptography) to neural-network activation functions is a meaningful and relatively novel contribution. The specific combination of Padé approximation + masking-based saturation + dummy arithmetic for this purpose appears new. However, each individual technique is well-established.

**Rigor**: The experimental methodology is sound — cycle-accurate DWT measurement, synchronization barriers, disabled interrupts, warm-up runs, and assembly-level verification of branchless structure. The template attack on desynchronization is a clean demonstration. The numerical accuracy analysis is thorough with MSE, RMSE, and max absolute error over two input ranges.

**Reproducibility**: High. Code is open-source. The target platform (STM32F411-Disco) is inexpensive and widely available. The methodology is described in sufficient detail to re-implement on other platforms.

**Significance**: This is a practical building block for secure embedded inference. The finding that desynchronization fails against template attacks is an important negative result. However, the scope is limited to individual activation functions — end-to-end network security requires protecting all components.

**Red flags**: The evaluation is limited to a single microcontroller platform. The constant-time guarantee depends on compiler behavior, which may change across toolchain versions or optimization levels. NaN/infinity inputs are explicitly excluded. Power/EM side channels are not addressed. The ReLU overhead (9 → 108 cycles, 12× slowdown) could be significant in ReLU-heavy networks.

## Strengths

- Clean, well-structured methodology that decomposes the problem into independently verifiable components
- Demonstrates that the proposed approach is not only secure but also faster than standard math library implementations for nonlinear activations
- Strong experimental validation: timing regularity verified over 500,001 input values (5 repetitions × 1001 points) in the extended setting
- The template attack on desynchronization provides compelling motivation for the constant-time approach
- Open-source implementation enhances reproducibility and adoption

## Limitations

- Evaluated on only one hardware platform (ARM Cortex-M4); portability to other architectures requires re-verification of cycle alignment and compiler behavior
- Constant-time guarantee is verified empirically rather than formally proven; compiler optimizations or microarchitectural effects on other platforms could break the invariant
- Only timing side channels are addressed; power and EM leakage remain unexplored
- The 12× ReLU overhead (9 → 108 cycles) is a nontrivial cost for activation-heavy networks
- Evaluation is at the level of individual activation functions, not complete network inference
- The methodology requires careful compiler management (specific optimization levels, avoiding link-time optimization) that may be fragile in large projects

## Relevance

本文直接针对嵌入式神经网络推理中的时序侧信道安全问题，提出了系统性的激活函数常量时间实现方法。对于关注边缘AI安全部署、模型隐私保护和硬件侧信道防御的研究具有重要参考价值。

## Questions for Further Exploration

1. How does the constant-time methodology scale to quantized (int8) activation functions commonly used in edge deployment?
2. Can the approach be extended to protect entire inference pipelines (matrix multiplication, pooling, normalization) under a unified constant-time framework?
3. What is the end-to-end performance impact on realistic DNN models (e.g., MobileNet, TinyML workloads) when all activation layers use the protected implementations?
4. Would combining constant-time activations with power/EM countermeasures (e.g., masking, shuffling) provide comprehensive multi-channel protection?
5. Can formal verification tools (e.g., ct-verif, dudect) be applied to provide machine-checked guarantees of constant-time behavior across toolchain versions?
