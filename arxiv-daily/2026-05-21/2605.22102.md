> **一句话总结：** ExComm 提出一种探索阶段的通信协议，通过检测并行推理智能体之间的事实冲突并用工具验证来修正错误信念，同时保持轨迹多样性，在 AIME 和 GAIA 基准上比最佳基线平均提升 5.0–5.7%。

> **TL;DR:** ExComm introduces an exploration-stage communication protocol for agentic test-time scaling that detects cross-agent factual conflicts, resolves them via tool-based verification with soft belief updates, and preserves trajectory diversity, achieving consistent gains of 5.0–5.7% over the strongest baselines on AIME and GAIA benchmarks.

## 核心贡献

- **在线信念一致性模块：** 在每个执行步骤后收集并行智能体的信念状态，检测互斥的事实冲突（占错误 67–71.5%），通过独立的工具增强推理循环验证并解决冲突，仅将验证结果反馈给相关智能体。
- **软信念更新：** 不直接覆盖信念，而是将修正作为外部建议追加到信念状态中，避免验证器错误引入新的、跨智能体共享的错误，同时保留智能体自主恢复能力。
- **轨迹多样化模块：** 分析所有智能体的计划，识别冗余策略并生成多样化指令，引导智能体探索正交推理路径，防止通信导致轨迹坍塌。
- **最佳性能-成本权衡：** 在 N=4 配置下超越所有 N=8 基线，同时在延迟方面保持可比水平。

## Key Contributions

- **Online Belief Consistency Module** that exploits the empirical finding that 67–71.5% of intermediate errors in parallel agentic reasoning produce detectable cross-agent factual conflicts, resolving them via a decoupled tool-augmented ReAct verification loop.
- **Soft belief update strategy** that appends verified corrections as external suggestions rather than overwriting beliefs, preserving agent autonomy and enabling recovery from incorrect resolutions (demonstrated in GAIA case study where an erroneous verifier correction was safely ignored).
- **Trajectory Diversification Module** that performs batched plan-level redundancy analysis and issues targeted diversification directives to redirect agents toward orthogonal strategies, preventing diversity collapse from communication.
- **Strong empirical results** across AIME 2024, AIME 2025, and GAIA with both Gemini-2.5-Flash-Lite (+5.7% avg) and Qwen3.5-4B (+5.0% avg), along with the best performance-cost trade-off among all evaluated methods.

## Methodology

ExComm operates on top of a base agent loop where each agent maintains a structured belief state *B*, a plan *P*, and a toolkit *T*. The framework introduces two modules applied after each execution step across *N* parallel agents:

1. **Online Belief Consistency Module:**
   - Gathers beliefs from all agents and extracts factual conflicts (mutually exclusive claims).
   - Each conflict is a structured object recording involved agents, their competing claims, and a description.
   - A dedicated ReAct-style verification loop (decoupled from problem-solving agents) resolves conflicts using tools (e.g., Python interpreter, web search).
   - Resolutions are transmitted only to involved agents as targeted feedback.
   - Agents apply soft updates: appending corrections rather than replacing beliefs.

2. **Trajectory Diversification Module:**
   - Performs centralized, batched analysis of all agent plans to identify redundancy and underexplored directions.
   - Produces at most one diversification directive per agent per step, enforced (not soft) at the plan level.
   - Directives specify how to shift a plan toward an orthogonal approach while leaving task execution autonomous.

The key insight motivating ExComm is that most errors (67–71.5% across benchmarks) are "conflicting" — at least one other agent holds a different factual claim on the same topic — making them detectable without relying on unreliable self-correction.

## Results & Highlights

- **Main benchmarks (Table 2):** ExComm achieves 72.5% (AIME 2024), 60.6% (AIME 2025), and 52.2% / 46.2% / 69.5% (GAIA L1/L2/L3) with Gemini-2.5-Flash-Lite, consistently outperforming Independent Scaling, Independent + SR, and Tree Search.
- **Error recovery (Table 3):** ExComm achieves 38.7% (Gemini) and 61.8% (Qwen) average error recovery rates, far exceeding the best baselines (19.9% and 57.7% respectively).
- **Scaling (Table 4):** Gains hold when scaling to Qwen3-235B-A22B (63.0% vs. 59.5% best baseline avg) and to N=8 agents (50.5% vs. 44.7% best baseline avg).
- **Communication baselines (Table 5):** ExComm outperforms Multi-Agent Debate and Mixture-of-Agents in accuracy while maintaining higher trajectory diversity (100 − Self-BLEU).
- **Ablation (Table 6):** Both modules are necessary — consistency alone slightly degrades GAIA L3; soft updates prevent harmful synchronization; diversification yields the largest gains especially in open-ended tasks.
- **Cost efficiency (Figure 2):** ExComm at N=4 matches or outperforms N=8 baselines at lower API cost, with comparable latency (10.45 min vs. 9.57 min for N=8 Independent + SR).

## Critical Assessment

- **Novelty: High.** The framing of exploration-stage (vs. solution-stage) communication is novel. The empirical observation that 67–71.5% of errors produce cross-agent conflicts is a clean, well-supported motivation. The soft update design is a practical and thoughtful contribution.
- **Rigor: Strong.** Three repeated runs per experiment, three different critic models for error analysis, ablation studies isolating each component, and comparison against adapted communication baselines. The error type classification framework (Section C) is well-designed with temporal truncation for fairness.
- **Reproducibility: Moderate.** The method is described in sufficient detail with structured object formats, algorithm pseudocode, and prompt templates (Appendix B). However, it depends on the Co-Sight framework and specific API-based models (Gemini-2.5-Flash-Lite, Qwen3.5-4B), and no code is released.
- **Significance: High.** Addresses a core failure mode (error propagation) in agentic test-time scaling. The performance-cost trade-off results are practically impactful. The design principles (targeted communication, soft updates, diversity preservation) are generalizable.
- **Red flags:** The GAIA evaluation uses a 50-sample subset (noted in Table 2), which may limit the statistical robustness of those results. The consistency module's effectiveness depends on verifier quality — the paper acknowledges this but the risk of cascading incorrect resolutions under repeated verification is not fully characterized. The reliance on GPT-5-nano and Gemini-3-Flash as critic models (which may not be publicly available) affects reproducibility of the error analysis.

## Strengths

- Clean motivation grounded in empirical error-type analysis (67–71.5% conflicting errors).
- The soft belief update mechanism is a principled design choice that balances correction with robustness, validated by a concrete case study (GAIA e1fc63a2).
- Comprehensive evaluation across multiple benchmarks, models, scaling regimes, and communication baselines.
- Strong performance-cost trade-off — N=4 ExComm beats N=8 baselines at lower cost.
- Well-structured paper with clear module descriptions, pseudocode, and illustrative examples.

## Limitations

- Not designed for solution-stage aggregation or refinement — complementary to post-hoc methods, not a replacement.
- Verifier quality is a bottleneck: incorrect resolutions can mislead agents, and in extreme cases repeated bad verification could degrade performance. The paper does not bound this risk analytically.
- GAIA results are on a 50-sample subset, limiting confidence in those specific numbers.
- No open-source code release mentioned; depends on proprietary API models for both solvers and critics.
- The approach targets factual/belief-level errors specifically — logical reasoning errors that are consistent across agents (the "Common" error type, 5.6–13.6%) are not addressed.

## Questions for Further Exploration

1. How does ExComm perform on non-mathematical, open-ended reasoning tasks (e.g., multi-hop QA, scientific reasoning) where factual conflicts are harder to define?
2. Can the consistency module's verifier be replaced with a smaller, specialized model to reduce cost while maintaining resolution accuracy?
3. What is the theoretical relationship between the number of parallel agents and the probability of detecting conflicting errors — is there a diminishing-returns regime?
4. How does ExComm interact with sequential revision or tree search — can exploration-stage communication be combined with solution-stage refinement for additive gains?
5. What happens when the verifier is systematically biased or adversarial — how robust is the soft update mechanism to correlated verification errors?
