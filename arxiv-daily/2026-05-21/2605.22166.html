Adapting the Interface, Not the Model: Runtime Harness Adaptation for Deterministic LLM Agents
Tianshi Xu, Huifeng Wen, Meng Li · 2026-05-21
llm-agents runtime-adaptation tool-use agent-evaluation model-agnostic
Key Metric
88.5% avg relative improvement across 7 benchmarks × 18 model backbones. 116/126 settings improved. Relative gains range 10–84% depending on benchmark.
Why This Matters
Challenges the assumption that agent improvement requires model training — environment-side structure decouples from model-specific behavior, and harness outperforms specialized tool-use training.
Critical Edge
Four-layer lifecycle architecture converges in ~5 Codex iterations, but each new environment requires tailored engineering and the evolution depends on a proprietary coding agent.

Methodology

Life-Harness adapts the model-environment boundary across four lifecycle layers: Environment Contract injects stable tool constraints and policy rules into the prompt before interaction; Procedural Skill retrieves reusable procedural patterns from a BM25-indexed skill library distilled from training trajectories; Action Realization validates and canonicalizes model-generated actions before execution (catching invalid calls, malformed arguments, wrong tools); Trajectory Regulation monitors post-execution trajectories for degeneration (repetition, oscillation, budget exhaustion) and proposes recovery. The harness is evolved by a coding agent (Codex) that iteratively inspects failed trajectories, classifies failures by priority (action realization > contract mismatch > trajectory degeneration > residual reasoning), and implements targeted updates. The process converges in ~5 iterations. The harness is frozen after evolution and applied unchanged at evaluation, with strict constraints: no test labels, no benchmark modification, no environment alteration.

Strengths

  • Clean problem formulation: interface adaptation separated from model training
  • Strong empirical coverage (18 models × 7 environments)
  • Four-layer decomposition is principled and practically effective
  • Cross-model transfer from a single small model is compelling
  • Outperforms and complements specialized agent training

Limitations

  • Each new environment requires tailored harness engineering
  • Scope limited to deterministic, rule-governed environments
  • Harness quality bounded by training trajectory failure coverage
  • Evolution pipeline depends on proprietary Codex agent
  • Cannot fix fundamental reasoning deficits — addresses interface failures only

Questions for Further Exploration

  1. Can harness evolution be made fully autonomous without a separate coding agent?
  2. Is there a diminishing-returns point as training trajectories increase?
  3. Could Life-Harness combine with RL fine-tuning for additive gains in non-deterministic settings?
  4. What is the cost of adapting a harness vs. training a specialized agent model?
  5. Do the four failure categories generalize to multi-modal or embodied environments?
