OOD detection via single-sample goodness-of-fit testing in normalising flow noise spaces
The Signal in the Noise: OOD Detection Through Goodness-of-Fit Testing in Factorised Latent Spaces
Philipp Bomatter, Jack Geary, Henry Gouk · 2026-05-21
ood-detection normalising-flows generative-models goodness-of-fit anomaly-detection

Key Metric

CIFAR-10→SVHN AUROC: 0.946 (vs 0.802 DoSE, 0.790 Typicality). CIFAR-10-C macro-average: 0.85 across 19 corruptions (vs 0.59 likelihood, 0.78 DoSE).

Why This Matters

OOD detection is a critical safety problem. SITN requires no OOD data, adds minimal O(D log D) overhead, provides provable Type I error control, and eliminates the complexity bias that plagues all likelihood-based methods.

Critical Edge

SITN is the only method where actual OOD images dominate top-scored samples. Likelihood, Typicality, and DoSE all flag simple in-distribution images (constant backgrounds) as most OOD due to complexity bias.

Methodology

SITN operates in the noise space of a continuous normalising flow (CNF) trained via flow matching. CNFs define a diffeomorphic, mass-preserving mapping from a standard Gaussian prior N(0, I) to the data distribution. For an in-distribution sample X, the inverse Z = φ¹(X) should follow N(0, I). Because the prior factorises completely, the D elements of Z are treated as D i.i.d. samples from N(0, 1), enabling two single-sample tests: (1) Anderson-Darling statistic for marginal normality, and (2) coefficient of variation of the power spectrum for dimensional independence. The statistics are combined via a max-quantile strategy with split-calibration providing provable Type I error control through the DKW inequality.

Strengths & Limitations

Strengths

  • Theoretically principled with formal Type I error control
  • No OOD data needed during training or calibration
  • Minimal computational overhead: O(D log D) beyond the inverse flow
  • Complementary AD and CV statistics cover diverse perturbation types
  • Clean, interpretable framework reducing OOD detection to two well-understood tests

Limitations

  • Moderate performance on harder OOD settings (CIFAR-10→CelebA: 0.671)
  • UNet artefact: constant backgrounds map poorly to Gaussian noise
  • Only evaluated on 32×32 images; scalability untested
  • CV statistic only tests autocorrelation, not all forms of dependence

Open Questions

  1. How does SITN perform on higher-resolution (ImageNet) and non-image modalities?
  2. Can additional test statistics (higher-order moments, runs tests) further improve detection power?
  3. Would transformer-based flow models mitigate the constant-background artefact?
  4. What is the relationship between CNF quality (NLL) and SITN’s Type II error rate?
  5. Can the noise-space GoF framework extend to conditional models for class-conditional OOD detection?
