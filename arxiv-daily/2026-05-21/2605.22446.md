> **一句话总结：** 提出Pre-VLA运行时验证架构，利用PPO Critic构建安全感知监督信号，通过冻结Chameleon-7B骨干网络和轻量双分支头在动作执行前进行先发式质量评估，在LIBERO四个套件上将平均成功率从30.79%提升至37.62%，同时减少执行步数并缓解世界模型推演中的幻觉累积。

> **TL;DR:** Pre-VLA is a preemptive runtime verification architecture that intercepts low-quality VLA actions before physical execution or world-model imagination. Built on a frozen Chameleon-7B backbone with modality-aware pooling and a dual-branch head predicting safety confidence + advantage scores, it improves LIBERO success rates from 30.79% to 37.62% with 183.9 ms verification latency, and mitigates WM hallucination cascades.

## 核心贡献

- 提出统一的运行时验证架构Pre-VLA，在VLA动作物理执行或世界模型推演之前进行先发式动作有效性评估
- 设计安全感知监督信号管线：从PPO Critic提取chunk advantage，引入failure-aware penalty回溯惩罚延迟失效动作，并进行任务级标准化
- 基于冻结的Chameleon-7B统一多模态骨干，设计modality-aware pooling和轻量双分支FFN头，同时预测安全置信度（分类）和优势分数（回归）
- 构建多任务训练目标（Focal Loss + MSE回归 + Soft-threshold校准）处理95:5的严重类别不平衡
- 设计双模态先发式重采样调度器，分别支持物理执行拦截和WM计算预算保护，含regression-guided safe fallback机制

## Key Contributions

- A unified runtime verification architecture that intercepts candidate action chunks before physical execution or world-model imagination, sharing the same verification gateway for both modalities
- Safety-aware supervision pipeline: extracts K-step chunk advantage from PPO Critic rollouts, introduces a failure-aware penalty with exponential decay for delayed-failure backtracking, and standardizes advantages per-task
- A lightweight dual-branch monitor built on a frozen Chameleon-7B backbone with modality-aware pooling that separates text/image/state/action representations and feeds them into a compact FFN for simultaneous safety classification and advantage regression
- Multi-task training objective combining Focal Loss (class imbalance), MSE regression (continuous advantage), and soft-threshold calibration (boundary stability)
- Dual-mode preemptive resampling scheduler: VLA execution mode filters dangerous actions and triggers resampling with regression-guided fallback; WM imagination mode performs compute filtering and on-demand rollout truncation

## Methodology

### Safety-Aware Supervision Signal Construction

Pre-VLA derives its training labels from PPO exploration rollouts rather than human annotation:

1. **Chunk Advantage Estimation** (Eq. 1): Computes K-step guided advantage $A^K_t$ using PPO Critic $V_\psi$, discounted rewards, and a safety mask $H_{t,K}$ that zeroes out bootstrapped value upon collision/object drop.

2. **Failure-Aware Advantage Refinement** (Eq. 2–3): Adds exponentially decayed penalty $F_{t,K}$ to actions within a backtracking window $W$ of a catastrophic failure time step $T_{fail}$, capturing delayed causal effects.

3. **Per-Task Standardization** (Eq. 4): Normalizes advantages within each task buffer to handle varying reward baselines across tasks.

4. **Threshold-Based Labels** (Eq. 5): Converts continuous advantages to binary safety labels via threshold $\tau_A = -0.21$, retaining continuous values as auxiliary regression targets.

### Lightweight Dual-Branch Monitor Architecture

- **Backbone**: Frozen Chameleon-7B (from WorldVLA), encoding language instructions, visual observations, proprioceptive states, and candidate action chunks as unified discrete tokens.
- **Modality-Aware Pooling**: Extracts per-modality hidden states via masked mean pooling over predefined modality masks, producing four summary vectors concatenated into $z_t = [z_{text}; z_{image}; z_{state}; z_{action}]$.
- **Dual-Branch FFN Head**: Classification branch outputs safety confidence $p_t = \sigma(\text{FFN}_{cls}(z_t))$; regression branch predicts continuous advantage $\hat{A}^K_t = h_\phi(z_t)$.

### Multi-Task Training Objective

$\mathcal{L} = \mathcal{L}_{cls} + \lambda_{soft}\mathcal{L}_{soft} + \lambda_{reg}\mathcal{L}_{reg}$ combining:
- **Focal Loss** ($\alpha_{focal}=0.25$, $\gamma_{focal}=2$) for severe 95:5 class imbalance
- **MSE Regression** for continuous advantage supervision
- **Soft-threshold Loss** with temperature $\tau_{temp}=0.25$ for smooth boundary calibration

### Dual-Mode Preemptive Resampling Scheduler

- **Warm-up horizon** $T_w=20$: Pre-VLA deactivated during early exploration.
- **VLA Execution Mode**: Classification branch rejects actions with $p_t < \tau_p=0.275$; triggers resampling (max $N=5$ attempts); upon exhaustion, regression branch selects highest-advantage candidate as safe fallback.
- **WM Imagination Mode**: Same filtering logic; additionally truncates imagination rollout if all candidates remain high-risk, releasing compute for the next control cycle.

### Training Details

- Dataset: 1,284,485 action chunk samples from PPO rollouts in LIBERO (95:5 positive-to-negative ratio; 30% negative per batch)
- Hardware: 8× NVIDIA H20D GPUs (~141 GB each), single-node DDP, batch size 512 per GPU, 10 epochs
- Training time: ~8 days 16 hours 43 minutes
- Inference: 16× NVIDIA RTX 5090 GPUs (VLA, Pre-VLA, WM on separate GPUs)

### Case Study (6 stronger-baseline tasks)

- Pre-VLA: 90.72% avg SR vs. RynnVLA-002: 86.51% (+4.21 pp), avg 3.13 fewer steps, 950.23 ms decision time.

### World Model Rollout Analysis

- Without Pre-VLA: WM predictions diverge after ~30 frames, showing object drift, blurring, ghosting, and false task completion.
- With Pre-VLA: Scene structure and spatial relationships remain stable; robot follows reasonable trajectory to task completion.

## Authors & Credibility

The authors are from a Chinese research group working on embodied AI and cloud robotics infrastructure. They reference recent VLA models (OpenVLA, π₀, GR00T) and world models (CTRL-World, WMPO, Sword), demonstrating strong familiarity with the field. The paper provides extensive quantitative evaluation (10K episodes per suite, ablation studies, per-task case studies) and visual analysis of WM rollouts. Training on 8× H20D GPUs with ~8.5 day training time suggests serious engineering investment.

## Critical Assessment

- **Novelty**: Moderate-to-good. The core idea of preemptive runtime verification for VLA/WM is timely. The failure-aware penalty with exponential backtracking (Eq. 2) and the dual-mode scheduler (physical vs. imagination) are genuine contributions. However, the individual components (Focal Loss, dual-branch heads, PPO Critic supervision) are well-established individually.
- **Rigor**: Strong experimental design with 10K-episode evaluations, ablation studies, discrimination metrics, trajectory-level analysis, and WM rollout visualizations. The comparison with Random Resampling is a good control experiment.
- **Reproducibility**: Architecture details are thorough (hyperparameters in Table I). However, the dependence on PPO Critic rollouts and RynnVLA-002 (a specific non-open-source model) limits out-of-box reproducibility. No code release is mentioned.
- **Significance**: The success rate improvements are meaningful but not transformative. LIBERO-Object improvement (1.14% → 3.95%) remains very low. The architecture is model-specific (requires access to the VLA backbone internals for feature extraction). The 78% increase in per-step latency (614 → 1098 ms) is non-trivial.
- **Red flags**: (1) Baseline RynnVLA-002 is the only VLA tested; generalization to OpenVLA/π₀/GR00T is unverified. (2) Absolute success rates on LIBERO-10 (19.61%) and LIBERO-Object (3.95%) remain very low even with Pre-VLA. (3) The frozen Chameleon-7B backbone adds ~184 ms per verification; this may not scale to real-time high-frequency control. (4) Supervision signals inherit PPO Critic bias under distribution shift, which the authors acknowledge as a limitation.

## Strengths

- Addresses a practically important and underexplored problem — runtime safety for VLA models
- Unified verification for both physical execution and WM imagination is elegant
- Failure-aware backtracking penalty captures delayed causal effects that simple thresholding misses
- Low false pass rate (0.02) is critical for safety applications
- Ablation study cleanly isolates the contribution of each training component
- WM rollout analysis provides qualitative evidence of hallucination mitigation

## Limitations

- Evaluated only on LIBERO simulation; no real-robot deployment demonstrated
- Only one baseline VLA model (RynnVLA-002) tested; cross-architecture generalization unknown
- Absolute success rates remain low on harder suites (LIBERO-Object: 3.95%, LIBERO-10: 19.61%)
- Requires access to the VLA backbone's internal hidden states — not applicable to closed-source models
- ~78% per-step latency increase may be prohibitive for high-frequency control loops
- PPO Critic supervision may be biased under distribution shift
- Training requires 8.5 days on high-end GPUs; data collection from PPO rollouts is non-trivial

## Relevance

与VLA模型可靠部署和世界模型幻觉缓解直接相关。提出的运行时验证框架可为具身智能系统提供安全层，其failure-aware监督信号设计和双模态调度机制对后续VLA安全研究具有参考价值。

## Questions for Further Exploration

- Can Pre-VLA generalize to other VLA architectures (OpenVLA, π₀, GR00T) without retraining the full verification pipeline?
- How does verification latency scale with backbone size (e.g., moving from Chameleon-7B to larger models)?
- What is the performance on real-robot tasks with non-stationary dynamics and sensor noise?
- Can the safety threshold $\tau_p$ and advantage threshold $\tau_A$ be adapted online during deployment?
- How does Pre-VLA interact with other safety mechanisms (control barrier functions, constrained RL)?
- Is the failure-aware backtracking penalty transferable to non-manipulation embodied tasks (navigation, locomotion)?
