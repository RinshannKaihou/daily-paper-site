DeltaBox: Scaling Stateful AI Agents with Millisecond-Level Sandbox Checkpoint/Rollback
Yunpeng Dong, Jingkai He, Yuze Hou, Dong Du et al. 2026-05-21
ai-agent sandbox checkpoint-restore systems mcts
Key Metric
14.57 ms mean checkpoint / 5.14 ms mean restore; state-management overhead reduced from 47–77% → 3–6%; GPU utilisation hits 93–96% at N=64 fan-out.
Why This Matters
Directly unblocks deeper MCTS search and near-saturating GPU utilization in RL training for AI agents — solving a real and growing infrastructure bottleneck as agent systems adopt tree-search and RL fan-out at scale.
Critical Edge
DeltaFS runtime hot layer switching without unmount + dual-path DeltaCR (template fork at ~5 ms + CRIU lazy-pages fallback at ~8 ms) with NPD decoupling — genuine OS-level contributions, not incremental tuning.
Methodology

DeltaBox is built on a four-layer architecture atop Firecracker microVMs. Layer 1 (Base Storage): XFS with reflink for block-level CoW. Layer 2 (DeltaFS): A modified Linux 6.8 OverlayFS kernel module with a custom ioctl for atomic layer-stack reconfiguration at runtime. On checkpoint, the upper layer is frozen and a fresh one inserted; a per-filesystem generation counter enables lazy file descriptor re-resolution. Layer 3 (DeltaCR): Dual-path process management — asynchronous incremental CRIU dumps (hidden in LLM I/O window) plus synchronous frozen-template fork. Fast-path restore via fork() (~3.75 ms); slow-path via CRIU lazy-pages (~7.25 ms). Layer 4 (Search): A StateManager coordinates Layers 2–3, supporting Linear, Best-of-N, and MCTS patterns. A Network Proxy Daemon (NPD) decouples LLM SDK connections from the agent address space; a semantic classifier skips checkpoints for read-only commands (62% of events).

Strengths & Limitations
Strengths
  • Clean OS-level design exploiting fundamental checkpoint-similarity property of agent workloads.
  • Inference-masked checkpointing hides CRIU dump cost behind LLM round-trips.
  • Dual-path restore provides consistent low latency while gracefully handling memory pressure.
  • Comprehensive evaluation with 5 baselines on real-world SWE-bench and RL workloads.
  • Semantic classifier elides 62% of checkpoint events — practical, high-impact optimization.
Limitations
  • No open-source release mentioned; custom kernel module is a significant adoption barrier.
  • Network I/O rollback not supported — external non-LLM side effects cannot be undone.
  • NPD does not handle streaming responses or complex authentication flows.
  • Template pool sizing is a new tuning knob with limited guidance.
  • Evaluation limited to coding agents (SWE-bench); other domains unexplored.
Questions for Further Exploration
  1. How does DeltaBox scale with agents maintaining large in-memory indices (e.g., RAG caches of hundreds of MB)?
  2. Can the DeltaState abstraction support network I/O rollback via request replay or idempotency tokens?
  3. What is the optimal template pool size as a function of MCTS branching factor and search depth?
  4. Could DeltaFS layer-switching benefit container-based sandboxes without Firecracker microVM overhead?
