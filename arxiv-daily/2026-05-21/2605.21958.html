Diagnosis Is Not Prescription: Linguistic Co-Adaptation Explains Patching Hazards in LLM Pipelines

Yoon Jeonghun, Kim Dongchan · 2026-05-21
llm-agents causal-analysis co-adaptation multi-module-pipelines tool-calling
The “Diagnostic Paradox”: causal analysis identifies the router as the bottleneck in LLM agent pipelines, but patching it degrades performance (ΔF=+0.683). Patching upstream query rewriter works instead (ΔF=−0.659) because downstream modules have co-adapted to upstream error distributions.

Key Metric

Router compensator rate 98.2% (gpt-4o-mini) correlates with absolute patching harm; 0% (Llama 4 Scout) correlates with no harm. Tool-match drops from 37.8% → 33.3% when patching the diagnosed bottleneck.

Why This Matters

Challenges the default debugging workflow of “diagnose bottleneck → patch it.” Multi-module LLM agents develop implicit co-adaptations; naive patching can break these “linguistic contracts” and make things worse.

Critical Edge

Only one pipeline topology tested (M1→M2→M3→M4). Compensator rate is NIE-derived from the same diagnostic data, not independent. Single-turn first-action protocol only. Llama 4 Scout’s zero compensator rate is unexplained.

Methodology

A 4-module LLM agent pipeline is evaluated on τ-bench retail: Query Rewrite (M1) → Planner (M2) → Router (M3) → Response Generator (M4). The CICA framework introduces four analysis layers: (1) Failure Index F aggregating LLM-judged per-module severities; (2) Causal Contribution ΔFi via oracle substitution; (3) Per-task NIE classifying each module-task pair as compensator, amplifier, or propagator; and (4) Counterfactual Correction Patching (CCP) injecting correction examples into module prompts. The design uses cross-family agent/oracle/judge triples (OpenAI+Meta+Alibaba agents, Anthropic oracle, Google judge) with paired Wilcoxon tests and Holm-Bonferroni correction across three agent families.

Strengths

  • Genuinely surprising finding with strong practical implications for agent debugging
  • Cross-family triples, Holm-corrected comparisons, behavioral metric validation
  • CICA framework is well-specified and reusable for other multi-module pipelines
  • CCP-specificity ablations isolate the mechanism cleanly
  • Honest about limitations: correlational Contract evidence, single topology

Limitations

  • Single pipeline topology on τ-bench retail; generalization unknown
  • Single-turn first-action protocol only
  • Compensator rate proxy is NIE-derived, not independent
  • No human evaluation; all scoring via LLM judges
  • Llama 4 Scout zero compensator rate is unexplained
  • Limited Qwen3-32b experiment (pilot, 49 tasks)

Questions for Further Exploration

  1. Does the Paradox hold in RAG pipelines where retrieval errors propagate differently?
  2. Can an independent co-adaptation measure establish a causal link between Contract strength and Hazard?
  3. How does the Paradox behave under multi-turn interactions?
  4. Is compensator rate a stable model-family property or prompt-dependent?
  5. Could “gradual correction” (incremental distribution shift) mitigate the Hazard?
  6. What happens when multiple modules are patched simultaneously?
