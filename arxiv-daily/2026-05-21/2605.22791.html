Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention

Ali Hatamizadeh, Yejin Choi, Jan Kautz · 2026-05-21

Replacing the single scalar gate in delta-rule linear attention with independent channel-wise erase and write gates yields the best 1.3B-scale linear attention model, with dramatic gains on multi-key retrieval (72.6% vs 54.0% at 1K) where memory editing matters most.

Key Metric

MK-NIAH-1 multi-key retrieval at 1K: 72.6% (Gated DeltaNet-2) vs 54.0% (KDA, next best). WikiText ppl 15.90 (recurrent) / 15.62 (hybrid), both best-in-class.

Why This Matters

The erase/write decoupling directly addresses the core limitation of compressed recurrent memory — a single gate forcing the model to erase and write at the same strength. The erase gate alone accounts for most of the gain.

Critical Edge

Scalar post-scaling is proven invalid under decoupled gates — gradients must embed gate factors into dot products. The dual-gate design generalizes both Gated DeltaNet and KDA as tied-gate special cases, providing a unified theoretical view.

Methodology

The Gated DeltaNet-2 recurrence uses dual channel-wise gates: erase gate bt ∈ [0,1]dk controlling which key-side coordinates are read and removed, and write gate wt ∈ [0,1]dv controlling value-side commitment. The update is: Sr = (I − kr(br ⊙ kr)T Diag(αr)) Sr−1 + kr(wr ⊙ vr)T.

A chunkwise WY-representation algorithm (chunk size C=64) absorbs channel-wise decay into asymmetric erase factors ē = γ ⊙ (b ⊙ k), enabling efficient parallel training on GPU. The block design uses L2-normalized queries/keys, SiLU gating, and RMS normalization. A hybrid variant adds sliding-window attention (2K window) for exact local interactions alongside the recurrent long-range memory.

Training: 1.3B params, 100B FineWeb-Edu tokens, AdamW (lr 4×10−4, cosine decay), 4K sequence length. State size matched across all baselines at 262,144 floats per layer.

Strengths

  • Minimal, well-targeted change (dual channel-wise gates) with clear theoretical motivation.
  • Strong gains on hardest benchmarks (MK-NIAH, S-NIAH-3) where memory editing matters.
  • Complete mathematical treatment — recurrence, chunkwise form, backward pass, tied-gate reductions.
  • Near-flat throughput scaling with only modest constant overhead.
  • Excellent ablation study quantifying each gate's independent contribution.

Limitations

  • Only 1.3B scale evaluated; scaling to 7B+ unproven.
  • 100B training tokens is small by current standards.
  • No fair softmax attention baseline on retrieval benchmarks.
  • Training at 4K but evaluating RULER up to 8K without long-context fine-tuning.
  • Additional gate projections add parameters whose overhead is not quantified.

Questions for Further Exploration

  • How do dual-gate gains scale at 7B/14B — grow, shrink, or plateau?
  • Would long-context fine-tuning amplify RULER advantages further?
  • Can erase/write decoupling transfer to retrieval-augmented or episodic memory architectures?
  • What is autoregressive decoding latency vs KDA and Mamba-2?
  • Is the asymmetric gate importance (erase > write) consistent across layers?
