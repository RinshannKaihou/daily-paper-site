The Illusion of Reasoning: Exposing Evasive Data Contamination in LLMs via Zero-CoT Truncation

Yifan Lan, Yuanpu Cao, Hanyu Wang, Lu Lin, Jinghui Chen · 2026-05-21
data-contamination llm-evaluation chain-of-thought benchmark-auditing reasoning
LLM chain-of-thought reasoning actively masks memorization. Zero-CoT Probe (ZCP) truncates reasoning chains to expose shortcut mappings, detecting both direct and paraphrased data contamination that standard evaluations miss.

Key Metric

GPT-4o shows contamination on MATH-500 with Ccont > 0.98. GPT-5.1 appears clean (Ccont ≈ 0.50). ZCP detects contamination with as few as 50–100 samples using logit-based metrics.

Why This Matters

Standard CoT evaluations may give a false sense of genuine reasoning ability when models have simply memorized benchmark answers. This undermines the validity of leaderboard comparisons across frontier models.

Critical Edge

Zero-CoT enforcement on closed-source models relies on prompt engineering, which may degrade as models are optimized for step-by-step reasoning. Only evaluated on math benchmarks; generalization to code or NLU tasks is unproven.

Methodology

ZCP operates in three stages: (1) Zero-CoT enforcement forces the model to output only final answers with no intermediate reasoning, either via token manipulation (open-weight) or prompt engineering (closed-source). (2) Isomorphic reference datasets are constructed by a multi-model pipeline that paraphrases textual context and alters numerical values while preserving logical structure and reasoning depth, verified by two independent judge LLMs. (3) Contamination Confidence (Ccont) is computed as a Bayesian posterior probability from performance gaps across four metrics (accuracy, consistency, first-token probability, all-token probability) between original and reference datasets, using bootstrap tests or McNemar’s test calibrated into a Bayes Factor.

Strengths

  • CoT-masking-memorization insight is novel and practically important
  • Continuous Contamination Confidence is more informative than binary classification
  • Comprehensive evaluation: existing models, fine-tuned simulations, real-world auditing
  • Practical tier-based guidance for white-box, black-box, and no-labels access levels
  • Well-designed isomorphic perturbation strategy preserving logical structure

Limitations

  • Zero-CoT enforcement on closed-source models depends on prompt engineering
  • Only evaluated on mathematical reasoning benchmarks
  • Reference dataset construction depends on LLMs, introducing potential biases
  • Neutral prior (π=0.5) may not reflect real-world contamination prevalence
  • Consistency-based detection requires ~1000 samples, potentially impractical

Questions for Further Exploration

  1. How does ZCP perform on non-mathematical reasoning benchmarks (SWE-bench, GPQA, legal/medical QA)?
  2. Can the isomorphic perturbation strategy generalize to open-ended generation tasks?
  3. What happens when models are explicitly trained to resist zero-CoT enforcement?
  4. How sensitive is Ccont to reference dataset quality from smaller LLMs?
  5. Could ZCP detect contamination during RLHF, not just pretraining/SFT?
