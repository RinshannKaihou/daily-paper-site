Statistically Valid Hyperparameter Selection: From Tuning to Guarantees

arXiv: 2606.25601

Date: 2026-06-24

https://arxiv.org/abs/2606.25601

一句话总结： 本专著提出以 Learn-Then-Test (LTT) 为核心的统一统计框架，将超参数选择重新表述为对候选集的多重假设检验问题，通过 p-values/e-values 与浓度不等式为平均风险、分位数风险、信息论约束、多目标与自适应/少标签场景提供显式的有限样本可靠性保证（如目标 δ 下的违规率从 E-HPO 的 0.53 降到 LTT 的 0.03）。
TL;DR: A comprehensive monograph recasting hyperparameter selection as multiple hypothesis testing (the Learn-Then-Test / LTT paradigm). It delivers explicit, finite-sample reliability guarantees — average-risk, quantile-risk, information-theoretic, multi-objective, adaptive/sequential, and label-scarce — using p-values, e-values, and concentration inequalities, and shows empirically that heuristic tuning (grid search) routinely violates target outage rates (e.g. 0.53 vs. a δ=0.2 target) while LTT-based methods stay within the guarantee (0.03).

核心贡献

  • 将"选超参数"重新定义为"统计可靠性认证"问题：不再是经验风险最小化（E-HPO），而是对每个候选 λ 检验零假设 H_k: R(λ_k) > α，并通过 FWER/FDR 控制在有限样本下保证选出的 λ̂ 的真实风险 R(λ̂) ≤ α 的概率 ≥ 1−δ。
  • 统一的技术栈：把 p-values（Hoeffding / exact binomial / empirical Bernstein / Cramér–Chernoff）与 e-values（e-calibrators、e-processes、anytime-valid inference）从第一性原理串联起来，形成模块化、可叠加在任何 HPO 候选集之上的"统计层"。
  • 四个方向的系统性扩展：(1) 超越平均风险，推广到一般 reliability functional（分位数风险 QLTT、信息瓶颈 IB-LTT）；(2) 多目标场景下的 Pareto Testing (PT) 与 Reliability-Graph-based Pareto Testing (RG-PT)；(3) 用 e-process / anytime-valid p-values 放松 batch 假设，得到自适应/序列化的 aLTT；(4) 在标签稀缺下用 prediction-powered inference (PPI) 结合 testing-by-betting 得到 R-AutoEval / R-AutoEval+。
  • 可操作的工程结论：在 Fashion-MNIST 分类、无线调度（Nokia suite，256 个候选）、LLaMA3 的 prompt 工程（SST 情感分析）三类应用上量化了"启发式调参 vs. 统计保证"的差距，并给出可复用的 Algorithm 1–5。
  • 从第一性原理写起的配套附录：附录 A–C 把证据度量（p-values / e-values）、构造（浓度界、精确二项尾）、FWER/FDR 控制流程（Bonferroni、Holm、FST、BH、BY、e-BH）自包含地推导出来，适合作为教科书参考。

Key Contributions

  • A unified reliability-first framing of hyperparameter selection. Instead of minimizing empirical risk (E-HPO), the monograph casts selection as multiple hypothesis testing (MHT) over a finite candidate set Λ, with null hypotheses H_k: R(λ_k) > α, and provides finite-sample post-selection guarantees via FWER (no false certification) or FDR (bounded expected fraction of unreliable picks).
  • A modular statistical toolbox developed from first principles. p-values (Hoeffding, exact binomial, empirical-Bernstein, Cramér–Chernoff) and e-values (e-calibrators, e-processes, anytime-valid inference) are presented as complementary evidence measures, with p-values suited to fixed-sample single-shot testing and e-values enabling anytime-valid, compositional, and adaptive testing.
  • Systematic extension along four axes. (i) General reliability functionals — quantile-risk control (QLTT) and information-bottleneck constraints (IB-LTT) via confidence-bound inversion; (ii) Multi-objective selection via Pareto Testing (PT) and Reliability-Graph-based Pareto Testing (RG-PT) that exploits a Bradley–Terry reliability DAG; (iii) Adaptive/sequential selection (aLTT) using e-processes and anytime-valid p-values to reduce the labelling budget; (iv) Label-scarce selection via prediction-powered inference combined with testing-by-betting (R-AutoEval, R-AutoEval+).
  • Concrete, reproducible engineering pipeline. Five algorithms (E-HPO baseline, LTT, PT, RG-PT, aLTT, R-AutoEval+) plus worked applications on image classification (Fashion-MNIST), wireless scheduling (Nokia suite, K=256 configs), and LLM prompt engineering (LLaMA3-8B on SST) quantify the reliability gap between best-effort tuning and statistically certified selection.
  • Self-contained appendices (A–C) that derive evidence measures, p-value/e-value constructions, and FWER/FDR-controlling procedures (Bonferroni, Holm, FST, BH, BY, e-BH) rigorously, making the monograph usable as a graduate-level reference.

Methodology

The monograph is organized as a progressive generalization of the Learn-Then-Test (LTT) paradigm (Algorithm 2):

  1. Problem setup (Ch. 1). Hyperparameter selection targets post-training, inference/deployment-level hyperparameters λ ∈ Λ (a finite candidate set produced by any upstream HPO). A hyperparameter is reliable if a risk functional R(λ) ≤ α. The target guarantee is P(R(λ̂) ≤ α) ≥ 1 − δ over calibration-data randomness. The monograph is explicit that this is not about finding the optimum, but about certifying that the act of selecting an unreliable hyperparameter is a rare, quantified event.
  1. Core LTT via MHT (Ch. 2). For each λ_k define H_k: R(λ_k) > α. Compute empirical risk on calibration data, turn it into a valid test statistic — a p-value (e.g. Hoeffding-based p = exp(−2n(α − R̂)_+²)) or an e-value (e.g. via an e-calibrator f_κ(p) = (1−κ)p^{−κ}) — then apply an MHT procedure:
  • FWER control (Bonferroni, Holm, fixed-sequence FST) ⇒ every certified λ ∈ Λ̂ is reliable w.p. ≥ 1−δ (strongest guarantee; Theorem 2.1, Corollary 2.2).
  • FDR control (Benjamini–Hochberg for p-values; e-BH for e-values, robust to arbitrary dependence) ⇒ expected fraction of unreliable picks in Λ̂ ≤ δ (less conservative, larger certified sets).
  1. Beyond average risk (Ch. 4). The MHT view only needs valid statistics for R(λ) > α, so it generalizes to any risk functional via confidence-bound inversion (Lemma 4.1: p = inf{ε : U_ε(λ) ≤ α}). Two instantiations:
  • Quantile risk control (QLTT): R(λ) = (1−q)-quantile of the loss; uses Howard–Ramdas empirical-quantile upper bounds to control tail latency.
  • Information-Bottleneck LTT (IB-LTT): certifies a relevance constraint I(T_λ; Y) ≥ α via a lower confidence bound on mutual information, then minimizes compression I(X; T_λ) within the certified set.
  1. Multi-objective selection (Ch. 5). L_C hard reliability constraints plus (L − L_C) auxiliary best-effort objectives. Combined p-value p_k = max_l p_{l,k} (Lemma 5.1) enables simultaneous certification. Pareto Testing (PT) splits calibration data into D_opt (estimate Pareto frontier) and D_MHT (fixed-sequence testing along the frontier), improving power by testing far fewer candidates. RG-PT further encodes prior reliability knowledge as a Bradley–Terry DAG (e.g. via LLM-as-a-judge pairwise preferences) and tests along graph paths; it controls FDR regardless of prior quality (Theorem 5.3).
  1. Adaptive / sequential selection (Ch. 6). Replaces the fixed-batch assumption with e-processes {E_{it}} and anytime-valid p-values P_{it} = 1 / max_{τ≤t} E_{iτ}, enabling aLTT (Algorithm 5): interleave evaluation and selection, stop early on certified candidates, and optionally use acquisition policies (e.g. Thompson sampling) to concentrate budget on promising configs. Validity holds at every stopping time, not just a pre-specified n.
  1. Label-scarce selection with autoevaluation (Ch. 7). When ground-truth labels are expensive but an autoevaluator (e.g. an LLM judge) produces cheap pseudo-labels, R-AutoEval uses prediction-powered inference (PPI) to debias: form unbiased "effective observations" from a small real-data correction plus large autoevaluated pool, then plug into the e-process framework. R-AutoEval+ maintains a family of reliance factors {ρ_s} and Hedge-weights them, provably dominating both R-Eval and R-AutoEval in sample complexity while preserving the finite-sample guarantee.

The supporting machinery (superuniformity of p-values, Markov-based validity of e-values, post-hoc level selection, convex-combination stability of e-values, proofs of Hoeffding p-value validity, e-BH FDR control) is developed from scratch in Appendices A–C.

Results & Highlights

  • Fashion-MNIST image classification (α=0.2, δ=0.2, 12 candidate (PCA-dim, regularization) configs, multinomial logistic regression, 100 random splits):
  • Conventional E-HPO (empirical-risk minimizer) violated the reliability requirement R(λ̂) ≤ 0.2 in 0.53 of runs — far above the target δ=0.2.
  • LTT (Hoeffding p-values + Bonferroni) achieved an outage rate of 0.03, well within the guarantee.
  • Wireless packet scheduling (Nokia suite, K=256 scaled scheduler configs, α=10 ms mean delay, δ=0.2):
  • E-HPO's selected config had mean delay 10.9 ms > α; over 100 splits the outage rate was 0.66, far exceeding δ=0.2.
  • LTT kept the outage rate below δ=0.2.
  • Quantile LTT (QLTT) at R_{0.1} ≤ α produced a delay distribution with a markedly lighter right tail, keeping the 90th-percentile delay under 10 ms where average-risk LTT still had tail violations.
  • Information Bottleneck (variational IB, α=2.28 relevance, δ=0.1, 100 trials): E-HPO violated the relevance constraint I(T;Y) ≥ α with empirical outage ~0.27; IB-LTT reduced it to ~0.07, satisfying the reliability target.
  • LLM prompt engineering (LLaMA3-8B-Instruct on Stanford Sentiment Treebank, 100 instruction-style prompt templates generated by LLaMA3-70B, reliability constraint R_prompt(λ) ≤ 0.2, auxiliary objective = minimize prompt length, FDR δ=0.1, prior via GPT-4-Turbo LLM-as-a-judge):
  • All three methods (LTT, PT, RG-PT) satisfied the FDR constraint, but RG-PT consistently certified shorter reliable prompts, leveraging the reliability DAG to test along informed paths.
  • Autoevaluation (wireless scheduling, K=20, binary SLA loss, α=0.3, δ=0.1, autoevaluator fidelity κ=0.9):
  • R-AutoEval certified 94% of reliable hyperparameters by n=80 ground-truth labels and reached near-perfect TPR by n=400.
  • R-Eval (labels only) reached only 59% at n=80 and was still <80% at n=2000 — roughly an order-of-magnitude sample-efficiency gain from PPI variance reduction (factor (1−ρκ)²).
  • R-AutoEval+ tracked R-AutoEval after a <50-round warm-up and provably dominates both baselines; all methods had zero empirical FWER violations across all n.
  • Headline takeaway: Across every application, best-effort tuning (grid search / empirical-risk minimization) systematically underestimates its own failure rate because it ignores selection-induced uncertainty, whereas LTT and its variants deliver the promised finite-sample guarantees at a quantifiable (and often modest) cost in certified-set size.

Critical Assessment

  • Novelty: As a monograph, the per-component novelty is modest — LTT, e-values, PT, PPI all predate this document, and the authors are transparent that they are unifying and extending their own and others' work. The genuine intellectual contribution is the integration (single risk-functional abstraction R(λ) = Φ(P_λ) with confidence-bound inversion as the unifying engine; the four-axis extension roadmap; the p-value-vs-e-value decision guidance). For a reader who has followed the conformal/e-value literature, little here will be brand-new; for everyone else it is a high-quality synthesis.
  • Rigor: Very high. Definitions, lemmas, theorems, and proofs are stated cleanly (Theorem 2.1 FWER⇒reliability, Lemma 2.4 Hoeffding p-value validity, Lemma 4.1 confidence-bound inversion, Lemma 5.1 combined p-value, Theorem 5.2 PT validity, Theorem 5.3 RG-PT FDR control). Assumptions (i.i.d. calibration data, bounded losses L ∈ [0,1], same train/test distribution) are explicit. The e-BH FDR proof under arbitrary dependence is deferred to Appendix C.2.
  • Reproducibility: Good in principle, moderate in practice. Algorithms are pseudocode-level specified and the experimental setups (datasets, model classes, α/δ, candidate-set construction) are described precisely. However, there is no mention of a code release, the wireless experiments depend on the Nokia suite + a specific RL scheduler, and the prompt-engineering experiments depend on LLaMA3-70B/8B and GPT-4-Turbo judge calls that are costly to reproduce. A reader could re-implement the Fashion-MNIST and IB experiments from the text alone.
  • Significance: High for the reliability/safety and LLM deployment communities. It gives a principled, distribution-free answer to "how do I pick a decoding temperature / confidence threshold / prompt template and prove it meets a KPI" — a question that practitioners currently answer with hand-waving. The timing (post-LTT 2025, during the e-value renaissance) is good.
  • Red flags / honest caveats:
  • Conservatism is structural. FWER control with Bonferroni over K=256 candidates will often yield an empty or tiny certified set; the monograph acknowledges this and motivates FDR/PT/RG-PT as remedies, but the practical message "LTT may certify nothing" deserves more prominence.
  • Empirical comparisons are not against strong HPO baselines. The E-HPO baseline is plain empirical-risk minimization (grid search). There is no comparison against, e.g., a held-out cross-validated selection with bootstrap confidence intervals, or a Bayesian-optimization-with-uncertainty method. So the "0.53 vs 0.03" headline, while directionally correct, overstates the gap relative to a competent practitioner who already does some uncertainty accounting.
  • The i.i.d. / no-distribution-shift assumption is load-bearing and is the single biggest threat to real-world validity; the monograph honestly lists distribution shift as the #1 open problem but the deployed guarantees simply do not hold under it.
  • Experiments are illustrative, not comprehensive. Sample sizes and candidate sets are small (K=12, 20, 100, 256); scalability to truly large prompt/model search spaces is asserted, not demonstrated.
  • Some "results" are restatements of known properties (e.g., e-value convex-combination validity, Markov-based e-value testing) presented as if part of the contribution; careful readers will recognize these as standard.
  • No human-subjects / safety-critical validation despite the safety-centric framing — the "safety" guarantee is purely statistical (bounded loss, controlled δ), not a systems-safety argument.

Strengths

  • Genuinely unifying: a single risk-functional abstraction plus confidence-bound inversion cleanly subsumes average-risk, quantile-risk, and information-theoretic reliability under one proof template.
  • The p-value vs. e-value pedagogy is excellent — the trade-offs (fixed-sample power vs. anytime-validity/composability/post-hoc level selection) are spelled out with figures and worked examples.
  • Honest, well-attributed synthesis that explicitly distinguishes the authors' contributions (RG-PT, IB-LTT, the four-axis roadmap) from prior art (LTT, PT, aLTT, PPI).
  • Practical, deployable algorithms with concrete hyperparameters (α, δ, K, sample sizes) and three quite different application domains (vision, wireless, LLMs).
  • Appendices make it self-contained enough to teach from; suitable as a graduate seminar text on distribution-free reliability.

Limitations

  • Empty-certified-set risk: with strict FWER and large K, the framework can certify nothing, and the fallback story (FDR, PT, RG-PT) is less developed theoretically for the "nothing is certified" regime.
  • Distribution shift is assumed away — the central threat to every guarantee in the monograph, acknowledged but unresolved.
  • Baseline strength: comparisons are against naive E-HPO, not against uncertainty-aware HPO or cross-validation with confidence intervals; the reliability gap may be smaller against stronger (if still informal) baselines.
  • Scale: all experiments use small candidate sets (K ≤ 256) and modest sample sizes; behavior at LLM-scale prompt/model search (K in the thousands+) is unvalidated.
  • Computational cost of e-processes / betting strategies and the tuning of e-calibrator κ / reliance factors ρ_s are discussed but not benchmarked for wall-clock or dollar cost.
  • No code release mentioned; full reproducibility of the wireless and LLM-judge experiments would require non-trivial reimplementation and API spend.
  • Bounded-loss assumption (L ∈ [0,1]) excludes many real losses (unbounded regression, ranking NDCG, some calibration scores) unless one manually clips/normalizes — a practical friction the monograph underplays.

Relevance

对关注"大模型可靠部署 / 推理时超参数（温度、阈值、prompt 模板）如何给出可证明保证"的研究者高度相关：它把 conformal/e-value 这套分布无关统计工具系统化地接到 LLM prompt 工程、无线调度等实际流水线上，正好填补"调参靠经验、无安全保证"的空白，适合作为 LLM 评测与可信 AI 方向的方法论基础与文献入口。

Questions for Further Exploration

  • How does LTT behave when the candidate set Λ is itself data-dependent (e.g., produced by a Bayesian-optimization run on the same calibration data)? Does the FWER/FDR guarantee survive the adaptive candidate generation, or is a sample-splitting correction mandatory?
  • Can the framework be extended to distribution shift (covariate/label shift, or drift between calibration and deployment) while retaining finite-sample guarantees — e.g. via weighted conformal / doubly-robust PPI?
  • For LLM decoding hyperparameters with unbounded or non-i.i.d. losses (e.g. token-level losses with context dependence), how should one construct valid p-values/e-values beyond the bounded-loss Hoeffding case?
  • What is the sample complexity of RG-PT and aLTT as a function of the reliability-graph quality (Bradley–Terry fidelity κ)? Is there a regret-style bound when the prior graph is adversarially wrong?
  • How does R-AutoEval+ degrade when the autoevaluator is itself an LLM that is correlated with the model under test (e.g. same family), violating the PPI independence-ish assumptions?
  • Can the multi-objective PT/RG-PT machinery handle combinatorially large or continuous prompt/model spaces without enumerating Λ, perhaps by combining with active learning — an open problem the monograph itself flags?
  • Empirically, how much smaller is the certified set Λ̂ in practice for modern LLM benchmarks (MMLU-style), and is the resulting reliability guarantee tight enough to be operationally useful versus just conservatism?
