> **一句话总结：** 本文提出一个以"进化知识图谱(EKG)"为锚点的物理约束多智能体发现引擎，通过抽取"算法思维链"将盲目的随机搜索转变为有向的结构演化，自动设计出两种超越人工启发式的硬件感知压缩方法(Q-Enhance 与 MoE-Salient-AQ)，并将 2350 亿参数模型压缩 75% 后部署到双卡 A100，精度仅下降 0.64%。

> **TL;DR:** A physically grounded, multi-agent discovery engine anchored on an Evolutionary Knowledge Graph (built from 164 LLM compression methods) autonomously evolves hardware-compliant compression algorithms — Q-Enhance (dense, long-context robust) and MoE-Salient-AQ (+3.7% over SOTA at sub-3-bit) — and deploys a 235B model on dual-A100 with 75% memory reduction and only 0.64% accuracy loss, establishing a hardware-software co-design paradigm under strict physical limits.

## 核心贡献

- **物理约束的多智能体发现引擎：** 构建了一个四层架构(Input / EKG / Agentic Engine / Output)的闭环系统，将"无约束组合搜索"转变为"知识驱动的自主演化"，专门解决通用 Agent 在硬件设计中"物理幻觉"的问题。
- **进化知识图谱(EKG)：** 从 164 个主流 LLM 压缩方法构建的 Neo4j 属性图谱，包含宏观拓扑层(方法谱系关系)与微观动力学层(精度/内存/迁移机制元数据)，作为结构化"人类设计智慧先验"。
- **"算法思维链"驱动演化：** 通过提取高分历史轨迹作为算法 CoT，驱动 Analyzer→Ideator→Architect→Reviewer 的异构多智能体协作，配合 AI 同行评审协议(Scoreagg ≥ 4.0 阈值)过滤数百个候选方案。
- **两个超越人工的新算法：** Q-Enhance 在 4-bit 等效预算下将权重与 KV cache 比特动态重分配，稳定 128k 长上下文；MoE-Salient-AQ 以专家粒度比特分配 + 自适应低秩误差补偿，在 2.5-bit 极端压缩下较 MxMoE 提升 3.7%。
- **真实硬件部署闭环：** 借助亚兆字节级 Sensitivity Profile(如 Llama-3-8B 仅 ~750KB)，将 235B 模型从 438GB 压缩到 108GB 部署到双 A100(精度损失 0.64%)，并将 8B 模型压到 6GB 部署到单卡 RTX 4090(3.80× 加速)。

## Key Contributions

- **Physically grounded agentic discovery engine:** A unified four-layer closed-loop architecture (Input / EKG / Agentic Engine / Output) that converts unconstrained combinatorial search into knowledge-driven autonomy, directly targeting the "physical hallucination" failure mode of generalist agents.
- **Evolutionary Knowledge Graph (EKG):** A Neo4j property graph built from 164 mainstream LLM compression methods, organized into a Macro-Topology layer (methodological lineage) and a Micro-Dynamics layer (precision/memory/transition-mechanism metadata), serving as a structured prior of human design intelligence.
- **Algorithmic Chain-of-Thought evolution:** High-scoring historical trajectories are extracted as an "algorithmic CoT" that seeds a heterogeneous multi-agent ensemble (Analyzer→Ideator→Architect→Reviewer), filtered by an automated AI peer-review protocol (Scoreagg ≥ 4.0 acceptance threshold).
- **Two novel algorithms beating human heuristics:** Q-Enhance dynamically reallocates bit-budget between weights and KV cache under a 4-bit equivalent budget, stabilizing 128k-token long context; MoE-Salient-AQ couples expert-granularity bit allocation with adaptive low-rank error compensation, beating MxMoE by 3.7% at the 2.5-bit regime.
- **End-to-end physical deployment:** A sub-megabyte Sensitivity Profile (e.g., ~750 KB for Llama-3-8B) closes the loop from abstract blueprint to hardware, compressing a 235B model from 438 GB → 108 GB on dual-A100 (0.64% accuracy loss) and fitting an 8B model into 6 GB on a single RTX 4090 (3.80× speedup).

## Methodology

The system is built as a four-layer closed-loop architecture (Figure 1):

1. **Input Layer** accepts either high-level compression tasks (for algorithmic discovery) or hardware deployment constraints plus a lightweight model profile (for local hardware mapping).
2. **EKG Layer** is a Neo4j property graph constructed semi-automatically from a domain corpus of LLM compression papers (2022–present) restricted to leading venues (NeurIPS, ICLR, ICML, CVPR, HPCA, ISCA, MICRO). Each Method Node encodes seven metadata dimensions (Basic Information, Main Contribution, Method Details, Applicable Models, Hardware Adaptation, Experimental Results, Limitations). Edges encode directed *Improvement* and *Inspiration* relationships. The EKG is dynamic: validated new methods are added as nodes, failures are appended as trajectory-level metadata.
3. **Agentic Engine Layer** runs two parallel loops:
   - **Algorithmic evolution:** An offline pre-screen scores all EKG paths S ∈ [1,10]; only S ≥ 9 survive into a high-potential pool. A stochastically sampled high-scoring path seeds an "algorithmic CoT." A *heterogeneous* multi-agent ensemble then operates: Analyzer + Reviewer use **Gemini Pro** (2.5/3.0/3.1) for long-context dissection; the Ideator uses **DeepSeek** (R1, v3.2) for divergent hypothesis generation; the Architect uses **Claude Opus** (4.5/4.6) for formal logical specification; a dedicated **GPT-5.5** execution agent synthesizes code by fusing open-source kernels. An automated AI peer-review rubric weights Motivation 0.25, Innovation 0.25, Feasibility 0.20, Expected Results 0.20, Weakness 0.10; only Scoreagg ≥ 4.0 pass.
   - **Hardware instantiation:** A local profiler distills the model into a bandwidth-efficient Sensitivity Profile (Architecture Block + Numerics Block as JSON + visual trend plots). Hardware targets are formalized as a constraint tuple C = ⟨Vmax, ΔAmax, Smin⟩; the Recommender seeds a mixed-precision strategy and the Evaluator searches the discrete space (bit-width, group size, symmetry) until C is satisfied.
4. **Output Layer** yields validated methodologies or hardware-tailored compression schemes, with a feedback loop updating the EKG.

Validated algorithms include **Q-Enhance** (joint weight+KV-cache quantization under a strict 4-bit-equivalent memory budget, dynamically reallocating the error budget to prevent long-context collapse) and **MoE-Salient-AQ** (expert-granularity bit allocation coupled with adaptive low-rank error recovery for sparse architectures).

## Results & Highlights

- **Q-Enhance (dense, long-context):** On Llama-3.1-8B-Instruct it stabilizes accuracy across context windows up to 128k tokens where baselines like QuaRot catastrophically degrade; robustness is corroborated on the Qwen3 family (Extended Data Table 1).
- **MoE-Salient-AQ (sparse, extreme compression):** At 2.5-bit, +3.7% over the leading manual baseline (MxMoE) on Deepseek-V2-Lite; consistent +3.3% (63.88% vs 60.53%) on Qwen1.5-MoE-A2.7B.
- **Peer-quality ranking (LLM-simulated, N=25, zero-shot, baselines withheld from EKG):** 100% of system blueprints placed in the top two tiers; 40% reached Tier 1, closely matching top-tier human expert publications while entirely bypassing the Tier 3/4 outputs that dominate *The AI Scientist* and standard venues.
- **Consumer deployment (Llama-3-8B, RTX 4090 24GB):** 4-bit weight-only scheme → 15 GB → 6 GB VRAM, 3.80× theoretical speedup, 0.77% accuracy loss.
- **Cloud deployment (Qwen3-235B-A22B, dual-A100 160GB):** 3.5-bit weight-only (mixed INT3/INT4 by expert salience) → 438 GB → 108 GB (75% / 4× reduction), accuracy 79.49% → 78.85% (only −0.64%, within the strict 1% tolerance).
- **Ablation — algorithmic context matters (Figure 5a):** Zero-shot generation produced 0 Tier-1 methods (severe hardware-incompatible hallucinations); 1-shot/few-shot literature injection gave only marginal gains; the Multi-Agent (Optimal) configuration anchored in algorithmic CoT achieved 50% Tier-1 yield.
- **Ablation — backbone scaling (Figure 5b):** Upgrading the backbone (Gemini/DeepSeek/Claude variants) generally expands Tier-1 yield; D-r1 slightly underperforms D-3.2 because it prioritizes raw reasoning over conversational compliance.
- **Ablation — multi-agent synergy (Figure 5c):** Single-model agents plateau at 20–30% Tier-1; the heterogeneous generator-reviewer ensemble lifts Tier-1 to 50% while compressing structural failures (Tier 4) to 5%.
- **Hallucination resistance (negative control):** On a conceptually orthogonal task ("detection of memory-resident vulnerabilities"), correlation scores cluster uniformly at the low bound of 1, confirming matching is strictly task-driven.

## Critical Assessment

- **Novelty (High):** The combination of an *Evolutionary Knowledge Graph* as a structured prior + heterogeneous multi-agent CoT + automated AI peer review for *hardware-aware algorithmic discovery* is genuinely novel relative to (a) unconstrained agents like *The AI Scientist* and (b) rigid AutoML/combinatorial search. The idea of treating past methodological lineage as a navigable graph to seed CoT is a creative contribution beyond simple RAG.
- **Rigor (Moderate-High):** Strong points include pre-screen thresholds (S ≥ 9), an explicit 5-dimension rubric with Scoreagg ≥ 4.0 acceptance, a controlled benchmarking protocol with N=25 zero-shot human baselines *withheld from the EKG*, and negative-control evaluation. However, the headline "expert-level quality" claim rests substantially on **LLM-as-a-judge** ranking, which is explicitly noted to suffer self-preference bias; the rubric was benchmarked against exactly the same class of LLMs, so circularity is a concern.
- **Reproducibility (Moderate):** The actual algorithmic contributions (Q-Enhance, MoE-Salient-AQ) are described at the level of mechanism but **no code or EKG dump is explicitly linked in the abstract/methods**. Full reproduction would require the entire heterogeneous LLM stack (Gemini Pro / DeepSeek / Claude Opus / GPT-5.5), which is expensive and API-dependent. Deployment numbers (3.80×, 3.91× speedups; 438→108 GB) are concrete and checkable if code were released, but stochastic generation complicates deterministic reproduction (authors explicitly flag this).
- **Significance (High, conditional):** If the benchmark numbers hold under independent evaluation, the practical impact is substantial — sub-3-bit MoE quantization with +3.7% and a 235B model on dual-A100 are production-relevant. The broader framing as a paradigm for EDA / compiler optimization is ambitious but currently unvalidated beyond the LLM-compression domain.
- **Red flags:**
  1. *LLM-simulated evaluation as primary novelty signal:* 40% Tier-1 is measured by an LLM judge, not by downstream task gains; this is the weakest load-bearing claim.
  2. *No released code/EKG mentioned:* Limits verifiability of the "autonomous discovery" loop.
  3. *Self-referential benchmarks:* Baselines (GPTQ, QuaRot, MxMoE) are well chosen, but the generated methods' advantages are reported by the same system/pipeline.
  4. *Vendor-locked ensembles:* Results depend on proprietary frontier models (Gemini 3.x, Claude 4.6, GPT-5.5) whose exact behaviors are not auditable.
  5. *Modest absolute bar in places:* The 0.64%–0.77% accuracy drops are within a self-imposed 1% tolerance, which is convenient framing.

## Strengths

- Clearly identifies a real failure mode of generalist agents ("physical hallucinations") and offers a principled structural-prior remedy.
- The EKG-as-structured-memory idea (Macro-Topology + Micro-Dynamics) is reusable beyond compression.
- Genuine end-to-end loop from idea generation → AI peer review → code → empirical evaluation → EKG feedback, not just idea generation.
- Strong, honest ablations isolating the contributions of (a) algorithmic CoT context, (b) backbone scaling, and (c) multi-agent synergy.
- Concrete, deployment-grade results (dual-A100 235B, RTX 4090 8B) rather than toy benchmarks.
- Negative-control experiment demonstrating task-driven matching is a thoughtful addition.

## Limitations

- **Manual EKG auditing required:** The literature ingestion pipeline is "highly automated" but authors admit "strict guarantees regarding the structural quality and academic rigor of the EKG currently require minimal human verification," undermining the fully autonomous framing.
- **Stochasticity and reproducibility:** LLM-based reasoning complicates deterministic guarantees; the paper concedes "strict algorithmic reproducibility and deterministic performance guarantees at the execution level" remain open.
- **No public code / EKG release** mentioned, limiting independent verification.
- **LLM-as-judge circularity:** Quality-tier results are produced and evaluated by frontier LLMs subject to self-preference bias.
- **Narrow evaluation domain:** Demonstrated only on LLM compression; the EDA / compiler-optimization generalization is speculative.
- **Cost/efficiency of the agentic loop** (multi-API ensemble, pre-screening of 164-node graph, N=200 candidate generation) is not quantified.
- **Comparison set:** Missing head-to-head against some recent strong agentic-discovery and AutoML-for-quantization baselines.

## Relevance

对一个同时关注"智能体自动化科研"与"大模型高效推理/压缩落地"的方向高度相关：它示范了如何用结构化知识图谱约束多智能体，使其生成硬件可行的算法，并直接产出可上线的压缩方案(Q-Enhance、MoE-Salient-AQ、235B 双卡部署)，对研究 agentic discovery、量化与边缘部署均有直接借鉴价值。

## Questions for Further Exploration

1. How sensitive are Q-Enhance and MoE-Salient-AQ to the EKG composition — does removing or perturbing 10–20% of the 164 source methods degrade Tier-1 yield or the benchmark numbers?
2. Can the "expert-level quality" claim be validated with a human peer-review study (blinded, multiple reviewers) rather than LLM-as-judge, to rule out self-preference bias?
3. Will the discovered algorithms be released as code, and do the 3.80×/3.91× speedups and 438→108 GB figures reproduce on independent hardware with measured (not theoretical) latency?
4. Does the framework generalize to genuinely different discrete, physically constrained domains (e.g., EDA logic synthesis, compiler pass ordering) where an analogous EKG can be built, or is success tightly coupled to the maturity of the compression literature?
5. How does the cost (API tokens, GPU-hours) of the full discovery loop compare to the months of manual engineering it claims to replace, and what is the marginal yield per dollar as the EKG grows?
6. What prevents error accumulation as the EKG is populated by system-generated nodes whose quality is only LLM-judged — could the graph drift toward plausible-but-suboptimal local optima over iterations?
