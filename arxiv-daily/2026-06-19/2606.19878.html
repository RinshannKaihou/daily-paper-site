On the Oracle Complexity of Interpolation-Based Gradient Descent

arXiv: 2606.19878

Date: 2026-06-19

https://arxiv.org/abs/2606.19878

一句话总结： 作者提出 PPI-GD（分段多项式插值梯度下降），将数据空间划分为小块并在每块上做多项式插值来近似全梯度，把 LPI-GD 所需的数据维度条件从 d=O(log log n) 放宽到 d=O(log^γ n)（γ<1/2），在强凸与非凸 ERM 设定下给出 oracle 复杂度并在光滑损失关键区间优于 GD/SGD，同时把 bicubic 样条的 blended interpolant 误差分析推广到任意阶/任意维的 d 元张量积多项式插值。
TL;DR: PPI-GD is an inexact first-order method that partitions the data space into small chunks and builds a polynomial interpolant of gradient samples on each chunk to approximate the full training gradient, resolving the open conjecture by extending the admissible data dimension from d=O(log log n) (LPI-GD) to d=O(log^γ n) with γ<1/2; for both strongly convex and non-convex ERM it beats GD/SGD in oracle complexity when the loss is sufficiently Hölder-smooth in the data, and the paper extends bicubic-spline blended-interpolant error analysis to general d-variate tensor-product polynomial interpolants.

核心贡献

  • PPI-GD 算法：一种新的非精确一阶优化算法，用于求解 ERM 问题。与 LPI-GD 在整个数据空间做局部多项式回归不同，PPI-GD 将数据空间划分为小块（chunks），在每个块上做多项式插值，概念上更简单、更高效、更易分析。
  • 放宽维度条件（解决前作猜想）：在强凸 ERM 设定下，PPI-GD 在 d=O(log^γ n)（γ∈(0,1/2)，例如 d=O(log^0.49 n)）的显著更宽区间内达到与 LPI-GD 相同的 oracle 复杂度 Õ((p/ε)^{d/(2ℓ)})，远优于 LPI-GD 所要求的 d=O(log log n)。
  • 非凸 oracle 复杂度：首次为 PPI-GD 在非凸设定下建立 oracle 复杂度 O(exp(log^{2γ}(n))·(p/ε²)^{1+d/(2ℓ)})（以 ε-stationary point 为准），并在 p=O(poly(n))、ε=Θ(poly(1/n)) 且 ℓ 充分大时渐近优于 GD 和 SGD。
  • 新的插值分析工具（独立贡献）：通过对经典 bicubic 样条分析中 blended interpolant 概念的推广，建立任意阶与任意维张量积多项式插值的误差界（Theorem 1）；核心技术是 Proposition 2 中用强归纳递归地把总插值误差分解为各轴方向上的迭代误差项之和，并配合 Proposition 4 的每项界与 Lemma 5 关于偏导与插值可交换性证明。
  • 实验验证：在合成分类数据集（leaky-ReLU 全连接神经网络，h∈{4,16} 隐藏层、p∈{881,4145} 参数、不同噪声 ν∈{0.5,1,1.5}）上对比 GD、SGD、MSGD、SVRG、MSVRG、LPI-GD，并做了 (ℓ,m) 网格搜索。

Key Contributions

  • PPI-GD algorithm (Algorithm 1): A new inexact first-order optimizer that, unlike LPI-GD's whole-space local polynomial regression, partitions the data space into small chunks and performs polynomial interpolation per chunk — conceptually simpler, more efficient, and more amenable to analysis.
  • Resolves the polylogarithmic-dimension conjecture: In the strongly convex setting, PPI-GD attains the same oracle complexity Õ((p/ε)^{d/(2ℓ)}) as LPI-GD but under the substantially broader regime d=O(log^γ n), γ∈(0,1/2) (e.g. d=O(log^0.49 n)), versus LPI-GD's restrictive d=O(log log n). See Table I in the paper.
  • Non-convex oracle complexity (Theorem 9): First analysis of PPI-GD for non-convex losses (ε-stationary-point criterion), giving O(exp(log^{2γ}(n))·(p/ε²)^{1+d/(2ℓ)}), with asymptotic dominance over GD and SGD in the regime p=O(poly(n)) and ε=Θ(poly(1/n)) (Proposition 10).
  • Reusable interpolation-theory tools: Establishes an error bound for tensor-product polynomial interpolants of arbitrary degree and dimensionality (Theorem 1) by extending the notion of blended interpolants from bicubic-spline analysis. Main technical device: Proposition 2's recursive decomposition of total error into axis-specific iterated error terms via strong induction over a recursion tree (illustrated in Fig. 1), together with the per-term bound (Proposition 4) and a partial-differentiation/interpolation commutativity lemma (Lemma 5).
  • Empirical validation (Section VI): Numerical simulations on synthetic classification datasets with leaky-ReLU networks, comparing PPI-GD against GD/SGD/MSGD/SVRG/MSVRG/LPI-GD, including a systematic grid search over (ℓ, m).

Methodology

  • Problem setup (ERM): Minimize F(θ) = (1/n) Σ f(x⁽ⁱ⁾; θ) over n samples x⁽ⁱ⁾ ∈ X = [0,1]^d. First-order oracle O(x, θ) = ∇_θ f(x;θ); the full gradient costs n oracle calls. Strongly-convex convergence target: F(θ)−F ≤ ε; non-convex target: ‖∇F(θ*)‖₂ ≤ ε.
  • Assumptions (Section II-C): The loss f is Lipschitz-continuous and Hölder-smooth in both the parameter θ and the data instance x — specifically belonging to an (ℓ, L_h)-Hölder class with respect to the data, where ℓ controls smoothness in the data variable (the lever exploited by PPI-GD). Parameter dimension p = O(poly(n)).
  • PPI-GD (Section III-A, Algorithm 1): Divides the data domain [0,1]^d into small patches; queries the oracle at equidistant grid points within each patch; constructs a tensor-product polynomial interpolant of degree m in each patch; uses the interpolant as an inexact gradient. Key hyperparameters: Hölder order ℓ and polynomial degree m, selected by grid search in practice. Precomputation steps (e.g. line 6 of Algorithm 1) build the interpolation grid once.
  • Analysis pipeline (Sections III-B, IV, V):
  1. Establish tensor-product polynomial interpolant error bound (Theorem 1) via the blended-interpolant generalization of bicubic-spline analysis.
  2. Recursively decompose total error into axis-wise iterated errors (Proposition 2) using strong induction on the recursion tree.
  3. Bound each iterated error term (Proposition 4); show partial differentiation and interpolation commute in most cases (Lemma 5).
  4. Plug the interpolation error into inexact-GD convergence templates for strongly convex (Theorem 7) and non-convex (Theorem 9) settings.
  • Comparison baselines: GD Õ complexity O(n log(p/ε)) (strongly convex), O(n·p/ε²) (non-convex); SGD O(p/ε) / O((p/ε²)²); LPI-GD at d=O(log log n); mini-batch, momentum/Nesterov, SVRG/SAG/SDCA, AdaGrad/RMSprop/Adam surveyed as related variants.

Results & Highlights

  • Strongly convex (Theorem 7, Table I): PPI-GD oracle complexity O(exp(log^{2γ}(n))·(p/ε)^{d/(2ℓ)}·log(p/ε)) at d=O(log^γ n), γ∈(0,1/2). Because exp(log^{2γ}(n)) is sub-polynomial when γ<1/2, this matches LPI-GD's rate but in a much wider dimension regime.
  • Non-convex (Theorem 9, Table I): O(exp(log^{2γ}(n))·(p/ε²)^{1+d/(2ℓ)}) — first such result for an interpolation-based GD in the non-convex setting.
  • Dominance regimes (Propositions 8 & 10): When p=O(poly(n)) and ε=Θ(poly(1/n)) and ℓ is sufficiently large (loss sufficiently smooth in the data), PPI-GD asymptotically beats GD, SGD, and their variants in both convex and non-convex settings.

Critical Assessment

  • Novelty (High): The core lever — exploiting smoothness of the ERM loss in the data variable via patch-wise polynomial interpolation — is a genuinely fresh angle in first-order optimization theory. The dimension-regime improvement from d=O(log log n) to d=O(log^γ n) is a real, conjectured-and-resolved theoretical advance, not an incremental delta. The blended-interpolant generalization of bicubic-spline error analysis to arbitrary degree/dimensionality is a standalone numerical-analysis contribution (the authors note prior work [12, p.217–218] only asserted such a generalization was "morally possible" without working it out).
  • Rigor (High): Full proofs are provided for the interpolation error bounds (Section IV) and oracle-complexity theorems (Section V), with explicit assumptions, multi-index notation, and a clearly diagrammed recursion-tree induction (Fig. 1). Constants are kept in Landau form Õ(·) with sub-polynomial factors made explicit.
  • Reproducibility (Moderate): Theory is the focus but experiments are concretely specified (dataset generator with explicit ν/noise distribution, network sizes, learning-rate schedule, batch sizes, grid-search ranges, hardware). NumPy-only implementation is lightweight. However, no code release is mentioned in the front matter, and several figures' quantitative takeaways require the paper's plots to parse.
  • Significance (Moderate): Solid theory contribution to the oracle-complexity literature and to inexact/interpolation-based GD. The polylog dimension is still far below the dimensionality of modern deep learning (LLMs, high-res vision), so direct practical impact on large-scale ML is limited; the authors themselves cite dimensionality-reduction contexts (PCA, Laplacian eigenmaps, Johnson–Lindenstrauss) and low-dim application domains (healthcare, control) as where d=O(log n) plausibly holds.
  • Red flags / caveats:
  • The dominance claims hinge on ε=Θ(poly(1/n)) and p=O(poly(n)) and "ℓ sufficiently large" — a specific corner of the (p, ε, ℓ) space; outside it the advantage can vanish.
  • exp(log^{2γ}(n)) sub-polynomial factor is favorable asymptotically but for realistic n it can still be sizable; constants hidden in Õ matter.
  • The inexact-gradient step requires building and evaluating interpolants each iteration — per-iteration wall-clock cost is higher than vanilla GD, which the wall-clock experiments partially address but only on small synthetic problems.
  • Experiments are synthetic and 2-dimensional (d=2); the theory's interesting regime is d=O(log^γ n), so there is a gap between the experimental dimensionality and the asymptotic regime where the main theorem bites.

Strengths

  • Resolves a stated open conjecture (polylogarithmic d) from the LPI-GD prior work.
  • First non-convex oracle-complexity result for interpolation-based GD.
  • Clean, generalizable numerical-analysis tool (tensor-product polynomial interpolant error bounds) with independent value.
  • Honest positioning: explicitly notes practicality caveats, orthogonality to momentum/Nesterov/minibatching, and points to dimensionality reduction as the realistic use case.
  • Concrete, runnable experimental protocol with grid search.

Limitations

  • Dimension regime d=O(log^γ n) is still far below modern high-dimensional ML (LLMs, vision) — applicability is confined to post-dimensionality-reduction or inherently low-dim problems.
  • Dominance only in a narrow regime (p=O(poly(n)), ε=Θ(poly(1/n)), ℓ large); sensitivity to ℓ and the Hölder constant L_h not emphasized.
  • Per-iteration overhead of constructing/evaluating interpolants is non-trivial; wall-clock advantage demonstrated only on small synthetic tasks, not on real benchmarks.
  • Experiments use d=2 whereas the theory's headline regime is asymptotic in d — a theory/experiment dimensionality gap.
  • No public code release mentioned; Õ-notation hides constants that determine practical crossover points.

Questions for Further Exploration

  • Can the d=O(log^γ n) barrier be pushed further (toward d=poly(n)) by leveraging low-dimensional data manifold or compositional structure assumptions common in deep learning?
  • What are the explicit constants hidden in Õ(·), and where is the practical crossover n at which PPI-GD actually beats GD/SGD in wall-clock on realistic problems?
  • How does PPI-GD compose with momentum, Nesterov acceleration, minibatching, and variance reduction (SVRG/SAG)? The authors note orthogonality — can a combined method inherit both sets of gains?
  • Does the tensor-product polynomial interpolant error bound (Theorem 1) tighten existing bounds in the numerical-analysis literature, and can it be applied beyond optimization (e.g., surrogate modeling, uncertainty quantification)?
  • Is the bias introduced by the interpolation-based inexact gradient benign for non-convex escape-from-saddle-point analyses (cf. biased-oracle saddle-escape work [42])?
  • Could the patch-wise interpolation scheme exploit GPU/parallel hardware more favorably than n-fold full-gradient computation, changing the practical oracle-vs-wall-clock tradeoff?
