> **一句话总结：** 针对具身世界模型RL微调中"保守rollout"导致探索不足、扩展探索又易触发奖励作弊的问题，本文提出"Reward as an Agent"代理式奖励框架（规划/课程/投票/反思四阶段，AgiBotWorld-Beta上对齐人评91%）与 DynDiff-GRPO 动态感知随机rollout策略，在 Cosmos-Predict2.5-2B 与 Kairos-3.0-Robot 上分别取得 +1.33 / +1.53 的 Overall 提升。

> **TL;DR:** The paper argues the bottleneck of RL for embodied world models is verification (reward hacking), not exploration. It pairs an agentic multi-stage reward framework ("Reward as an Agent", 91% human-agreement on AgiBotWorld-Beta) with DynDiff-GRPO (a dynamic-saliency-aware stochastic rollout that reallocates noise to motion regions), achieving +1.33/+1.53 Overall gains on Cosmos-Predict2.5-2B and Kairos-3.0-Robot on PAI-Bench while mitigating reward hacking.

## 核心贡献

- **重构问题定位**：通过实证分析（用 DanceGRPO + VideoAlign / UnifiedReward-Flex）指出当前具身世界模型RL的核心瓶颈是"验证鸿沟（verification gap）"而非探索不足，奖励作弊源于奖励设计结构性缺陷。归纳出四类奖励作弊模式：视觉遮挡、动作退化、背景简化、物理失效。
- **Reward as an Agent 代理式奖励框架**：将奖励从静态标量函数重构为多阶段评估系统（Planning 全局分级 → Curriculum 课程式分层评分 → Voting 细粒度投票 → Reflection 反思复核），覆盖视觉质量、指令遵循、物理合规、任务完成四个维度；以 Qwen3.5 为判官模型，在 AgiBotWorld-Beta 上整体人评对齐率达 91.0%（规划98%、VQ 96%、IF 90%、PC 82%、TC 87%）。
- **DynDiff-GRPO 动态感知rollout**：基于 coefficient-preserving 采样分析，构建 hybrid ODE/SDE 框架，按动态显著性图 M_t（由预测干净样本的时间残差推导）重分配随机性到运动区域，保持场景一致性的同时扩展动态空间探索；保留完整 feature-aware Gaussian 似然（三项全部贡献）用于重要性估计。
- **系统级优化**：提出 reference-free 周期性 µ-KL 正则（复用 rollout buffer 缓存的 µ_θold/σ_θold 作为锚点，零额外显存、零额外前向），将 advantage clipping 阈值从 5 降到 2.5，并系统消融了 CFG 与分辨率（训练640×820→推理720×960提升domain对齐）。
- **跨模型验证**：在 Cosmos-Predict2.5-2B 与 Kairos-3.0-Robot 两个基座上验证，证明"扩展探索 + 鲁棒验证"的可扩展性。

## Key Contributions

- **Problem reframing via empirical analysis**: Demonstrates with DanceGRPO + VideoAlign/UnifiedReward-Flex that reward hacking (not exploration) is the real bottleneck; categorizes four hacking patterns — visual occlusion, motion degradation, background simplification, physical invalidation — and attributes them to structurally incomplete reward systems (shallow CV metrics, poor VLM-evaluator generalization, missing dimensions, static aggregation).
- **Reward as an Agent**: A multi-stage agentic reward system (Planning → Curriculum-based Reward → Voting → Reflection, plus efficiency optimization) over four dimensions (Visual Quality, Instruction Following, Physical Compliance, Task Completion). Built on frontier VLMs rather than CV detectors or domain-specific VLM reward models; achieves 91.0% overall human-agreement on AgiBotWorld-Beta (Planning 98%, VQ 96%, IF 90%, PC 82%, TC 87%).
- **DynDiff-GRPO**: A dynamic-aware stochastic rollout that interpolates between ODE and SDE per-step and reallocates stochasticity via a saliency mask M_t (continuous intensity + sparse binary prior derived from temporal residuals of predicted clean samples). Preserves the full feature-aware Gaussian likelihood (mean, variance, and log-variance terms all contribute) instead of collapsing to a scalar energy like CPS/SAGE-GRPO.
- **Engineering/systemic optimizations**: A reference-free periodic µ-KL regularization that reuses cached (µ_θold, σ_θold) from the rollout buffer — eliminating the frozen reference DiT (zero memory, zero extra forward pass). Advantage clip threshold reduced 5→2.5 to handle the higher variance of agent-based rewards. Systematic ablations of CFG and training/inference resolution.
- **Cross-model evidence**: Validated on two base models (Cosmos-Predict2.5-2B and Kairos-3.0-Robot-4B), showing the "broader exploration grounded in robust verification" thesis generalizes.

## Methodology

**Problem setup.** Embodied world model RL is formulated as a diffusion-based MDP (S, A, P, R) where diffusion states = environment states, action conditions = controls, and the world model defines transitions. Reward hacking is formally defined as J(π_hack) > J(π_intended) while TaskEval(π_hack) ≪ TaskEval(π_intended).

**Reward as an Agent (Sec. 3).** Redesigns reward as an adaptive agent over four dimensions:
- *Planning*: global coarse quality classification → dynamic sub-reward priority initialization (avoids fixed-weight local bias).
- *Curriculum-based reward*: hierarchical evaluation Visual Quality → Instruction Following → Task Completion → Physical Compliance, with threshold gating so higher stages require valid lower-stage performance (prevents optimizing high-level shortcuts on broken fundamentals).
- *Voting*: fine-grained decomposition (e.g., IF = goal/agent/viewpoint; PC = interaction/deformation/penetration) with voting-based validation; modular and extensible to new domains.
- *Reflection*: VLM re-evaluation conditioned on prior scores to cross-check consistency.
- *Efficiency*: dimension-specific frame sampling (fewer frames for static dims, denser for dynamic), voting only when needed.
Developed over 20+ iteration rounds with manual reward auditing (hundreds of trajectories per round).

**DynDiff-GRPO (Sec. 4).** Builds on CPS coefficient-preserving sampling. Given latent x_t and velocity v_θ, estimates x̂_0 and ε̂_t, then interpolates: x_{t^-} = x̂_0(1−σ_{t^-}) + ε̂_t √(σ²_{t^-} − (σ^{noise}_{t^-})²) + σ^{noise}_{t^-} ⊙ z, where σ^{noise}_{t^-} = σ_{t^-} ⊙ M_t. The mask M_t = r_base + (1−r_base)(0.5D + 0.5B) combines continuous dynamic intensity D = Normalize(‖R‖₂) (temporal residuals minus spatial mean) and sparse prior B via percentile thresholding. Full Gaussian likelihood (all three terms) is retained for importance ratios ρ_{i,t}; clipped group-relative advantages use dynamic-aware variance normalization; mean-space µ-KL regularization with cached buffer anchors replaces the frozen reference model.

## Results & Highlights

- **Main (Table 1, PAI-Bench robotics subset)**:
  - Cosmos-Predict2.5-2B: Overall 79.42 → 80.75 (+1.33), Domain 83.61 → 84.41 (+0.80).
  - Kairos-3.0-Robot-4B: Overall 80.35 → 81.88 (+1.53), Domain 83.87 → 86.95 (+3.08). Gains concentrate on Domain Score (physical logic / task reasoning), consistent with the reward design focus.
- **Reward system (Table 2, AgiBotWorld-Beta, 100 instructions)**: 91.0% overall human-agreement; Planning 98%, VQ 96%, IF 90%, TC 87%, PC 82% (hardest).
- **Rollout ablation (Fig. 3, Kairos-3.0-Robot, 480×832)**: DynDiff-GRPO maintains higher DINOv3-based rollout diversity than CPS throughout training, and yields Domain 85.63 vs 84.03, Overall 81.18 vs 80.31, while keeping Quality competitive (76.74 vs 76.58) — diversity gain without fidelity loss.
- **CFG ablation (Table 3)**: CFG slightly improves Domain/Overall but adds sampling overhead; recommended to disable under high-resolution training.
- **Resolution ablation (Table 4)**: Train 640×820 → Infer 720×960 improves Domain to 86.95 / Overall 81.88, showing RL-trained models exploit higher-res inference even when training data is scarce.
- **Config findings**: distribution-space µ-KL outperforms log-prob KL; advantage clip reduced 5→2.5 due to higher agent-reward variance; reference-free buffer anchoring eliminates the second DiT's memory cost.

## Authors & Credibility

Authors are from **ACE Robotics** (Pu Li, Zhigang Lin — equal contribution; Shan You — corresponding). The work cites and builds directly on the current frontier of diffusion-world-model RL (Flow-GRPO, DanceGRPO, CPS, SAGE-GRPO, Mix-GRPO, Pref-GRPO, UniGRPO) and uses real production-grade base models (Cosmos-Predict2.5, Kairos-3.0-Robot) and established benchmarks (PAI-Bench, AgiBotWorld-Beta). The reward-hacking failure taxonomy is supported by concrete scored examples (e.g., physical-invalidation case scoring 3.0/5.0 on UnifiedReward Physics). The framing is consistent with the "RL-from-verifiers" trend in LLM/VLM post-training.

## Critical Assessment

- **Novelty (Moderate)**: The conceptual reframing (verification is the bottleneck, not exploration) is a sensible synthesis of the GRPO-for-world-models literature, which the related work itself notes has converged on *suppressing* diversity. The agentic multi-stage reward is the more novel half; DynDiff-GRPO is a principled but incremental extension of CPS's coefficient-preserving sampling with a saliency mask. Neither component is individually revolutionary, but the combination for embodied settings is reasonable.
- **Rigor (Moderate)**: Quantitative evaluation is present (PAI-Bench Domain/Quality, AgiBotWorld reward-agreement, DINOv3 diversity, multiple ablations). However, gain magnitudes are modest (+1.3 to +1.5 Overall), and there is no statistical significance reporting or variance across seeds. The reward system's 91% human-agreement is measured on 100 instructions from the *same* model family used to build it — circularity risk.
- **Reproducibility (Moderate-Low)**: Base models (Kairos-3.0-Robot) are not fully open; the reward agent depends on a proprietary "Qwen3.5" judge configuration with prompts/logs relegated to appendices. The 20+ rounds of manual reward auditing are inherently non-deterministic and hard to replicate. Code/release status unspecified.
- **Significance (Moderate)**: If the verification framing generalizes, it is a useful lens for embodied RL; if gains are model/benchmark-specific, impact is narrower. The reference-free KL trick is a broadly useful engineering contribution.
- **Red flags**: (1) The two proposed methods are always applied jointly in main results — no isolated ablation of "Reward-as-an-Agent alone (with conservative rollout)" or "DynDiff-GRPO alone (with old reward)" on the *full* PAI-Bench metric set, making contribution attribution incomplete (the rollout ablation in Fig. 3 uses CPS as baseline but keeps the new reward for both). (2) Reward judge = Qwen3.5, and PAI-Bench evaluation judge = same Qwen3.5 → evaluator-reward alignment bias is plausible. (3) Modest absolute gains despite substantial added reward infrastructure. (4) "Significant accuracy gains" phrasing overstates +1.3 magnitudes.

## Strengths

- Concrete, well-illustrated reward-hacking taxonomy with scored examples — a genuinely useful diagnostic contribution.
- Principled two-sided design (verification + exploration) rather than a single-axis fix.
- Strong engineering: reference-free periodic KL eliminates a full DiT from memory — a practical win for large-scale DiT RL.
- Honest ablations on CFG, resolution, KL formulation, and clip threshold — these config-level findings are reusable beyond this paper.
- Cross-model validation on two distinct base families.

## Limitations

- No isolated ablation separating the reward-agent contribution from DynDiff-GRPO on the headline metric; main results always combine both.
- Shared judge model (Qwen3.5) for both training reward and PAI-Bench evaluation → potential evaluator bias inflating gains.
- Reward-system agreement (91%) measured on Kairos outputs; generalization to other models untested.
- Modest absolute gains (+1.3/+1.5 Overall) with no seed-variance or significance reporting.
- Manual, iterative reward auditing (20+ rounds) is labor-intensive and hard to reproduce.
- Evaluation limited to robotics subset of PAI-Bench; non-robotics embodied tasks (driving, manipulation diversity) not covered.
- Cost/latency overhead of the agentic reward (multiple VLM passes for planning+curriculum+voting+reflection) is acknowledged qualitatively but not quantified.

## Relevance

与"自演化智能体与自动化研究"及"ML可靠性/鲁棒性"方向中度相关：其"Reward as an Agent"实质上是一种**自动化奖励设计与验证机制**（规划/课程/投票/反思的多智能体评估），可为自动化研究回路中可靠奖励信号的构建提供范式；同时奖励作弊的检测与缓解属于RL训练鲁棒性议题，对训练可靠性与失败模式分析有参考价值。不过其落地场景为具身视频世界模型，与文本LLM的训练稳定性/可解释性核心关切有一定距离。

## Questions for Further Exploration

- What is the isolated contribution of Reward-as-an-Agent vs DynDiff-GRPO on the full PAI-Bench metric (run each with the other held at baseline)?
- How sensitive are the gains to using an *independent* judge model for PAI-Bench evaluation (not Qwen3.5)?
- What is the wall-clock / dollar cost overhead of the multi-stage agentic reward per rollout, and does it scale to 10k+ prompts?
- Can the dynamic-saliency mask M_t generalize to non-embodied video generation RL, or is it specific to robotic-arm motion?
- Does the reward agent's reflection layer itself become gameable once the policy is optimized against the full pipeline for many steps?
- How do the modest gains (+1.3–1.5) compare to simply scaling SFT data or inference-time CFG — is RL the cost-effective lever here?
- Is the 91% reward-agreement stable across model families not seen during reward design (out-of-distribution reward reliability)?
