Efficiently Representing Algorithms With Chain-of-Thought Transformers

arXiv: 2606.19697

Date: 2026-06-19

https://arxiv.org/abs/2606.19697

一句话总结： 本文从理论上证明，链式思维（CoT）Transformer 可以直接在 Word RAM 指令层级（而非图灵机层级）模拟任意教科书算法，每条指令仅需多对数（poly-logarithmic）开销，相比此前基于图灵机模拟的二次开销是质的提升；结论被推广到连续 CoT 与 Transformer+线性 RNN 混合架构这两种有限宽度、log 精度的实际设置。
TL;DR: This work proves that CoT transformers can directly simulate Word RAM algorithms (textbook algorithms like O(n log n) sorting or O(E+V log V) Dijkstra) with only poly-logarithmic overhead per instruction — a dramatic improvement over the quadratic overhead of prior Turing-machine-based CoT simulations. The result is established for poly-log-width hard-attention transformers and extended to two practical fixed-width, log-precision settings: continuous CoT (reasoning as vectors) and hybrid transformer + linear-RNN models.

核心贡献

  • 首次将 CoT Transformer 的表达能力分析目标从图灵机升级为 Word RAM 模型——带随机访问内存与单位代价 O(log n)-bit 字运算，正是教科书算法的设计与分析抽象层。
  • 证明 log²-宽度、有限精度、右侧唯一硬注意力（rightmost-unique UHAT） Transformer 能以 O(t·w²)=O(t log²(n+t)) 个 CoT token 模拟 t 步 Word RAM（其中 w=Θ(log(n+t)) 为字长）；对于无乘法/除法/取模的简单算术指令集，开销降为 O(t·w)=O(t log(n+t))，恰好匹配离散 CoT 的信息论下界 Ω(t·w)。
  • 引入 Word RAM 受限范式（restricted normal form，保证每步寄存器值 v ≤ 当前步数 i），结合 c-维数组等价化简（Lem. 1-2，Thm. 2），使得在固定宽度下也可以基于 Merrill-Sabharwal 的"除以 2"引理逐位提取二进制位。
  • 将结论加强到两种更实际、固定宽度 + log 精度 + 完全 uniform 参数的设置：
  • 连续 CoT（continuous CoT）：推理步骤为实值 soft token 向量，soft token 充当跨步的"持久寄存器组"（Thm. 3 + Lem. 4-7）。
  • 混合模型：Transformer 层叠加单一线性 RNN 层，用 RNN 隐状态替代 soft token 完成 ϕ(v)↔二进制位的串/反串行化（Thm. 4 + Lem. 8-9）。
  • 给出统一的 O(t·κ(w)) 开销框架，其中 κ(w) 为单条指令在图灵机上的代价：扁平指令集下 κ(w)=O(w)（无乘除模）或 O(w²)（含乘除模），从而在三种架构下一致地得到 O(t log(n+t)) 或 O(t log²(n+t))。
  • 关键技术工具：layer-norm hash 表示 ϕ(v)=layernorm(v,1,−v,−1) 用于精确等值匹配注意力（Prop. 1 等值间隙与 tie-breaking）；新提出的 仿射更新性质 ϕ(av+b) 可由固定深度前馈/RNN 计算（Prop. 2），驱动了位提取与递归除法。

Key Contributions

  • Reframes the expressivity theory of CoT transformers around the Word RAM model (random-access memory + unit-cost O(log n)-bit word operations) instead of Turing machines, matching how textbook algorithms are actually written and analyzed (e.g., O(n log n) sorting, O(E+V log V) Dijkstra).
  • Theorem 1 (poly-log width): A finite-precision transformer of O(log²(n+t)) width with rightmost-unique hard attention simulates t steps of a flat Word RAM using O(t·w²) = O(t log²(n+t)) CoT tokens; this drops to O(t·w) = O(t log(n+t)) for simple-arithmetic (multiplication/division/mod-free) instruction sets, which exactly matches the information-theoretic lower bound Ω(t·w) for discrete CoT.
  • Introduces a restricted normal form for Word RAMs (every register value v ≤ step index i) via c-dimensional-array equivalence (Lemmas 1-2, Thm. 2), enabling bit extraction in the fixed-width regime using a simplified division-by-2 lemma (Lem. 3).
  • Strengthens the result to two practical fixed-width + log-precision + fully uniform settings:
  • Continuous CoT (Thm. 3): reasoning steps are real-valued soft tokens that act as a persistent register bank threaded through the transcript (Lemmas 4-7).
  • Hybrid Transformer+linear-RNN (Thm. 4): a single RNN layer at the bottom replaces soft tokens, performing ϕ(v)↔bits serialization/deserialization in the RNN hidden state (Lemmas 8-9).
  • Provides a unified O(t·κ(w)) overhead framework, where κ(w) is the TM cost of a single instruction: κ(w)=O(w) for simple-arithmetic flat instructions and O(w²) for full-arithmetic, yielding O(t log(n+t)) or O(t log²(n+t)) across all three architectures.
  • Core technical gadget: the layer-norm hash ϕ(v)=layernorm(v,1,−v,−1) gives exact-match attention via inner products (Prop. 1, with a 1−δ_T mismatch gap for tie-breaking), and a new affine update property ϕ(av+b) (Prop. 2) computable by a fixed-depth FFN or RNN drives iterative bit extraction and division.

Methodology

The paper develops three constructive simulations of Word RAM computation by decoder-only CoT transformers, all following a common template that logs every memory write mem[a] ← v as a transcript block mem # bits(a) # bits(v), with the central primitive being an indexed memory read: given a query address, recover the value of the rightmost matching block (the most recent write) using two attention layers.

Setting (i) — Polylogarithmic-width hard-attention transformers (§3, Thm. 1). Word-sized integers are stored directly as binary coordinate vectors bits(v) ∈ {0,1}^w in the residual stream. This gives equality checking via inner products and bit extraction via bits(v)·e_t = b_t "for free." Each simulated RAM step runs four constant-depth blocks: (1) Load pc via indexed read of mem[0]; (2) Load instruction and operands (compile-time register addresses, then runtime mem[a]/inp[a] addresses); (3) Execute via a single MLP computing the destination value(s) and next pc; (4) Store the result via a within-step offset counter. Multiplication, division, and modulo are not single-step computable, so they are compiled into O(w) flat micro-steps that share state through program-private cells, each costing O(w) tokens → O(w²) per such instruction. Deserialization uses w dedicated attention heads (one per bit) of width w, giving the O(w²)=O(log²(n+t)) model dimension.

Settings (ii) and (iii) — Fixed-width, log-precision, uniform constructions. The bits(v)-as-coordinate trick fails at fixed width because bits(v) is no longer directly accessible. The authors identify two simultaneous requirements on any value representation: an equality gap (exact-match attention) and read/write capability against a binary CoT encoding. The layer-norm hash ϕ(v) satisfies the equality gap, but writing requires extracting bits from ϕ(v) and reading requires reconstructing ϕ(v) from bits — both open problems for plain transformers. Two bottlenecks are solved by two minimal architectural extensions:

  • Continuous CoT (§5, Thm. 3): each decoding step emits both a discrete token and a constant-dimensional soft token p_i; the soft token carries (ϕ(a), ϕ(v)) pairs for store-summaries and checkpoint state. A constant-depth carry block can always surface the latest checkpoint "from layer 0" at future positions, solving persistence. Serialization (Lem. 5) iteratively applies the simplified division lemma to emit w bits LSB-first while keeping ϕ(v) in the soft token; deserialization (Lem. 7) reconstructs ϕ(v) by LSB evaluation; instruction execution delegates to a TM oracle operating on a delimited work-tape suffix for O(κ(w)) steps (Lem. 6).
  • Hybrid models (§6, Thm. 4): structurally identical to Thm. 3, except the linear RNN hidden state replaces soft tokens for threading intermediate ϕ representations across positions. The affine update property (Prop. 2) turns ϕ(⌊v/2⌋) = ϕ((v−b)/2) into a linear operation an RNN head can perform, giving serialization (Lem. 8) and deserialization (Lem. 9) analogues.

Throughout, the authors restrict attention to flat instruction sets (non-recursive grammar, constant nesting depth) so each line is O(1) Word RAM steps, and distinguish full-arithmetic (any Op operator) from simple-arithmetic (excluding ×, ÷, mod) flat programs.

Results & Highlights

  • Main qualitative result: CoT transformers can simulate Word RAM algorithms with only poly-logarithmic overhead in n — not the quadratic overhead forced by routing Word RAM → Turing machine → CoT as in Pérez et al. (2021) and Merrill & Sabharwal (2024).
  • Tightness for discrete CoT: The O(t·w) = O(t log(n+t)) bound for simple-arithmetic flat instructions matches the Ω(t·w) information-theoretic lower bound exactly (any CoT transcript representing a w-bit word over a fixed alphabet must emit Ω(w) tokens); the full-arithmetic bound O(t·w²) matches up to a single logarithmic factor.
  • Three architectures unified under O(t·κ(w)):
  • Polylog-width UHAT: O(w²) per step, O(w) for multiplication-free (Thm. 1).
  • Continuous CoT, fixed width: O(κ(w)) per step (Thm. 3).
  • Hybrid transformer+linear-RNN, fixed width: O(κ(w)) per step (Thm. 4).
  • For flat instructions, κ(w)=O(w) (simple) or O(w²) (full), giving O(t log(n+t)) / O(t log²(n+t)) in all cases.
  • Corollaries 1-3 instantiate the framework: arbitrary poly(w)-time instructions give Õ(t) steps; full-arithmetic flat gives O(t log² t); simple-arithmetic flat gives O(t log t).
  • Architectural necessity (open): the authors flag that fixed-width plain transformers' ability to simulate Word RAMs remains open; continuous CoT or a single RNN layer provably suffice as minimal extensions, but whether they are necessary is left as an open question.
  • Remark 3 notes weaker attention models cannot implement the indexed-read step, justifying the rightmost-UHAT / averaging-hard-attention assumptions.

Critical Assessment

  • Novelty — High. Reframing CoT expressivity around Word RAM is natural and directly addresses a real efficiency gap that prior Turing-completeness results glossed over. The key insight — that simulating at the instruction level rather than the tape-symbol level removes a quadratic blowup — is clean and well-motivated.
  • Rigor — Strong. The argument is fully constructive: each theorem is backed by explicit four- and six-phase simulations, concrete lemmas for every primitive (load, serialize, execute, deserialize, store), and explicit overhead accounting. The information-theoretic lower-bound argument (Ω(t·w) for discrete CoT) provides a tightness benchmark that the upper bounds meet or nearly meet.
  • Reproducibility — N/A (theory paper). No code or experiments to reproduce; the value is in the proofs and constructions, which are spelled out at the lemma level (full proofs in appendices §B-§E).
  • Significance — Moderate-to-High. Provides the sharpest theoretical justification to date for why CoT can be efficient, not merely universal. It also yields concrete architectural guidance: a single linear-RNN layer (or continuous CoT) suffices for efficient Word RAM simulation at fixed width — relevant to the modern wave of hybrid architectures (Mamba/RWKV-style linear RNNs, reasoning with latent thinking).
  • Red flags / caveats:
  • The model assumes idealized components — rightmost-unique hard attention or averaging hard attention, exact log-precision arithmetic, fully uniform parameters — none of which match deployed softmax-attention LLMs running in float16/bf16. The results are worst-case upper bounds on idealized architectures, not empirical claims about trained models.
  • The simulation is a constructive existence proof; whether gradient-trained CoT actually discovers these efficient simulations is entirely unaddressed (acknowledged in the conclusion as an open question).
  • The "restricted normal form" reduction (Lem. 2) inflates runtime by adding v(n)=n+t idle steps, which is absorbed asymptotically but hides a constant-factor cost in practice.
  • Fixed-width plain transformer simulation is explicitly left open, so the cleanest practical claim (plain transformer + CoT suffices) is not actually proven — only the two augmented architectures are.
  • Constants, lower-order terms, and the practical width/depth needed to realize the constructions are unspecified.

Strengths

  • Clean, well-motivated reframing around Word RAM that aligns with standard algorithm analysis and textbook intuition.
  • Unified O(t·κ(w)) framework spanning three architectures of practical interest (hard-attention, continuous CoT, hybrid Transformer+RNN).
  • Granular overhead characterization (poly-log → log-square → log) tied to instruction-set restrictions, with matching lower bounds where achievable.
  • Honest about limitations: explicitly flags the open status of fixed-width plain transformers and the empirical gap to trained models.
  • Strong author team with the exact expertise (Merrill-Sabharwal layer-norm hash machinery) for the technical core.

Limitations

  • Purely theoretical; no empirical validation on trained reasoning models — the gap to real LLM CoT behavior is large.
  • Assumes idealized attention (rightmost UHAT / averaging hard), exact log-precision arithmetic, and fully uniform parameters; unclear how conclusions degrade under standard softmax attention with float16/bf16.
  • Constructive simulations may be hard to map onto learned CoT behavior; gradient descent is not analyzed.
  • The restricted-normal-form reduction hides a +O(n+t) idle-step cost that matters at small n.
  • Fixed-width plain transformer simulation is left open, weakening the "plain CoT suffices" narrative.
  • Constants and lower-order terms in the asymptotic analysis are unspecified; the practical width/depth implied by the construction could be large.

Relevance

与"LLM 隐藏态与可解释性 / 推理评估 / 数值可靠性"兴趣存在间接但实质的关联：本文从理论上刻画了 CoT Transformer 在算法级别的表达能力与开销上界，并对 layer-norm hash 表示、对数精度算术、RNN 隐状态承载中间 ϕ 表示等机制给出了形式化分析，为理解推理模型的计算复杂度边界、潜在精度/数值边界以及"连续 CoT / 潜在思维 vs. 离散 token"这一新兴方向提供了理论基础。它不直接涉及训练稳定性或硬件 SDC，但对推理模型的能力与可靠性刻画有支撑意义。

Questions for Further Exploration

  • Are the upper bounds (O(w²) for full-arithmetic, O(w) for simple-arithmetic) tight, or can CoT transformers do better? What non-trivial lower bounds beyond Ω(t·w) can be proven?
  • Can a fixed-width plain transformer (no continuous CoT, no RNN layer) simulate Word RAMs with poly-log overhead, or are continuous CoT / recurrence genuinely necessary? The authors flag this as open.
  • How do the assumptions degrade under standard softmax attention and float16/bf16 precision — does the layer-norm hash equality gap (Prop. 1) survive numerical noise?
  • Can the constructive simulations be instantiated as training targets, auxiliary losses, or scratchpad scaffolds to push learned CoT toward these efficiency bounds, and does this improve empirical algorithmic reasoning benchmarks?
  • What is the relationship between these Word RAM simulations and empirical phenomena such as CoT length scaling, parity/algorithmic-task failure modes, and "thinking tokens" / latent reasoning in continuous-CoT-style models?
  • Do the hybrid Transformer+linear-RNN results offer theoretical backing for modern linear-RNN/SSM architectures (Mamba, RWKV, etc.) as efficient reasoning backbones, and can the κ(w) instruction-cost model predict which algorithms benefit most?
  • The restricted-normal-form reduction (Lem. 2) inflates runtime by O(n+t) idle steps — is there a tighter reduction avoiding this, and does it matter for practical input sizes?
