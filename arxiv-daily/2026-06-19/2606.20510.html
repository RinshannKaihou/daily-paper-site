Efficient and Sound Probabilistic Verification for AI Agents

arXiv: 2606.20510

Date: 2026-06-19

https://arxiv.org/abs/2606.20510

一句话总结： 针对运行时监控 Datalog 策略的 AI Agent 安全验证，提出一种基于分布鲁棒优化（DRO）的多项式规模 SDP 松弛框架：先用 Datalog 派生图刻画多步轨迹的污点/信息流语义，再将其形式化为精确但指数规模的 LP，最后用只跟踪二阶矩的 SDP 松弛得到可在运行时求解的策略违反概率 sound 上界（不依赖谓词独立性假设），在 Intercode、ATBench 终端 Agent 与 Praline 侧信道分析基准上以更低延迟同时取得完美安全性与高效用。
TL;DR: A sound, polynomial-time semidefinite-programming relaxation of a distributionally-robust optimization over Datalog derivation graphs that computes certified upper bounds on the probability of AI-agent policy violation — without independence assumptions between noisy predicates (e.g., PII detectors, declassifiers) — achieving perfect security and high utility at much lower latency than prior SMT-based engines (Praline) on Intercode, ATBench, and side-channel benchmarks.

核心贡献

  • 将运行时 Agent 安全监控从确定性策略推进到概率性策略：显式建模带失败率的概率谓词与概率状态转移（如 PII 检测器、脱密器），避免确定性引擎对连续概率过早二值化造成的信息损失。
  • 把多步 Agent 轨迹编译成 Datalog 派生图（derivation DAG），并将"最坏情形策略违反概率"形式化为一个指数规模的精确线性规划（LP），该 LP 在任意谓词相关性下都成立，无需 ProbLog/WMC 那样的独立性假设。
  • 提出多项式规模的半定规划（SDP）松弛：只跟踪状态分布的二阶矩，用 McCormick 包络线性化双线性项，并证明（Theorem 2）其最优值 R_SDP ≥ 真实最坏情形风险 R，即sound 上界，从而保证参考监控器绝不会漏报违规（不引入结构性假阴性）。
  • 支持显式相关性约束（POS/NEG/IND 三类），在已知相关性时可收紧界、提升效用。
  • 在 Intercode-NL2Bash（197 条轨迹）、ATBench（377 条）、Praline 侧信道分析（6 任务）三类基准上实证：SDP 在多个全局安全阈值 τ∈{0.2,0.5,0.8} 下安全性与效用均达 1.0，AUC 优于 Praline、Monte Carlo、确定性引擎，且平均延迟显著低于 Praline（如 Intercode 221ms vs 1015ms）。

Key Contributions

  • Probabilistic verification paradigm for agent monitors. Extends runtime Datalog-policy enforcement (reference-monitor / intercept-evaluate-commit loop) from deterministic predicates to noisy probabilistic predicates and stochastic state transitions, motivating the approach with a concrete FORGE-style exfiltration policy where local thresholding on per-message PII risk produces false negatives.
  • Datalog derivation-graph compilation of multi-step trajectories. Maps each tool invocation to taint-transition semantics (Propagate / Merge / Declassify / CreateClean / CreateTainted) and proves (Theorem 1) any Datalog policy on a finite trace can be rewritten as a derivation graph with max arity 2, enabling a second-order moment relaxation.
  • Exact-but-exponential LP over joint measures. Formulates worst-case violation probability as an LP over all valid joint measures μ consistent with marginal bounds [ℓ_v, u_v] and transition rules — distributionally robust by construction, in contrast to Weighted Model Counting (WMC/#P-hard) which assumes independence and can severely underestimate risk (worked example: 30% robust vs 27.2% WMC leakage, a 2.8% deficit that flips a τ=0.3 decision).
  • Polynomial-time SDP relaxation with formal soundness. Tracks only first- and second-order moments via a PSD moment matrix M and McCormick envelopes for auxiliary T variables; problem size is O(|V_F|² + |C|); Theorem 2 proves R ≤ R_SDP, so the monitor never under-blocks.
  • Correlation-aware. Accepts explicit POS / NEG / IND correlation classes (extending Praline's two-phase inference), tightening bounds when correlations are known.
  • Strong empirical results. On Intercode, ATBench, and Praline side-channel tasks, SDP achieves perfect security (recall=1.0) and perfect-or-near-perfect utility (precision) across τ∈{0.2,0.5,0.8}, with AUC consistently ≥ 0.991 and beating all baselines; latencies (Intercode 221ms, ATBench 303ms, Side Channel 1927ms) are 3–24× lower than Praline's SMT-based refinement.

Methodology

Setup. The reference monitor intercepts each candidate tool call t_k from state s_k, speculatively projects the next state s'_{k+1} = Δ(s_k, t_k), and blocks if the policy predicate Π(s'_{k+1}) = Block. Threat model: attacker can influence agent inputs (including indirect prompt injection) and knows tools/policy/failure rates, but cannot bypass the monitor or invalidate the probabilistic tool-failure assumptions.

Probabilistic tool models. For each noisy tool (e.g., LLM PII detector, redaction tool), the engineer profiles it on labeled data and derives a sound per-invocation failure upper bound via a 1−α Clopper–Pearson Beta confidence interval: u_redact_fail = F_{β}^{-1}(1−α; k+1, N−k). These give marginal bounds [ℓ_f, u_f] on base facts; no inter-invocation independence is assumed unless explicitly declared via POS/NEG/IND classes.

Datalog compilation. A program D=(F,R) is compiled into a derivation DAG G=(V_F ∪ V_R, E). Each rule maps to a multilinear polynomial over {0,1}^{|V_F|}: conjunction→multiplication (x_a · x_redact_fail), disjunction→inclusion-exclusion (x_i1 + x_b − x_i1·x_b), negation→(1−x). Theorem 1 guarantees every transition can be made binary-arity, which is what later permits a second-order relaxation.

Exact LP (Eq. 1). Maximize R* = max_μ Σ_{x: x_t=1} μ(x) subject to: (i) μ has zero support on states violating transition logic; (ii) marginal constraints ℓ_v ≤ Σ_x μ(x)·𝟙[x_v=1] ≤ u_v for base facts; (iii) μ ≥ 0, Σ μ = 1. This is distributionally robust (no independence) but has 2^{|V_F|} variables — intractable at runtime.

SDP relaxation (Eq. 3). Introduce a symmetric PSD moment matrix M∈ℝ^{(|V_F|+1)×(|V_F|+1)} with M_{0,v}=𝔼[x_v], M_{v,v'}=𝔼[x_v x_v'}; linearize the bilinear products M_{0,v}M_{0,v'} via McCormick envelopes on auxiliary scalars T_{v,v'}. Add correlation constraints (POS ⇒ M_{v,v'}≥T; NEG ⇒ ≤; IND ⇒ =). Problem size O(|V_F|²+|C|), solvable in polynomial time by interior-point (SCS solver). Soundness (Theorem 2): any feasible μ of the exact LP maps to a feasible (M,T) of the SDP with the same objective, but not vice-versa, so R_SDP ≥ R.

Soundness-in-practice argument. R'(a) ≥ R(a) ⟹ R'(a) ≤ τ ⟹ R(a) ≤ τ: a sound upper bound introduces no structural false negatives relative to exact evaluation (though noisy base classifiers can still propagate their own errors).

Results & Highlights

  • RQ1 — Security/utility trade-off (Table 2). SDP attains Security=1.0 and Utility=1.0 (AUC 0.991–1.0) across Intercode / ATBench / Side-Channel at τ∈{0.2,0.5,0.8}. Praline is also perfectly secure but loses utility from solver timeouts (loose unrefined bounds cause over-blocking); Monte Carlo achieves perfect utility but lower security due to independence-induced underestimation; the deterministic engine collapses on both axes at extreme thresholds (e.g., Side-Channel security drops to 0.143 / AUC 0.611 at τ=0.8).
  • Latency (Table 3). Average per-task: Intercode 221ms (SDP) vs 1015ms (Praline); ATBench 303ms vs 7227ms (~24× faster); Side-Channel 1927ms vs 330ms (SDP slower here due to dense correlation constraints, but avoids Praline's catastrophic exponential-tail timeouts). A 15-second solver timeout is enforced for inline enforcement.
  • RQ2 — Independence assumption underestimates risk (Table 4). As input correlation strength rises (None→Medium→High) at fixed τ, Monte Carlo's utility degrades sharply (e.g., τ=0.2 High-correlation utility collapses to 0.606) because rigid global independence overestimates true worst-case risk → excessive false positives, whereas SDP holds Utility≥0.982. This quantitatively motivates dropping independence.
  • RQ3 — General probabilistic Datalog policies. On Praline's side-channel tasks (compiler-register information-flow vulnerabilities), SDP matches Praline's perfect security/utility oracle while avoiding SMT worst-case exponential behavior.
  • Worked numerical example (§3.5). For redact a.txt; cat b.txt i1; cp; send: exact LP worst-case leakage R*=0.300 (Fréchet: min(0.9,0.1) for the conjunction, 0.1+0.2 for the disjunction); WMC under independence gives 0.272 — a 2.8% deficit that flips a τ=0.3 enforcement decision, concretely illustrating the unsoundness of independence.

Critical Assessment

  • Novelty (High). Recasting runtime AI-agent policy monitoring as a distributionally-robust SDP over Datalog derivation graphs — specifically to drop the independence assumption that ProbLog/WMC and even Praline's tight SMT refinement rely on — is a genuinely novel and clean framing. The arity-2 reduction (Theorem 1) that licenses a second-order relaxation is an elegant technical move.
  • Rigor (Medium-High). The central soundness claim (Theorem 2) is correctly argued via feasibility-set inclusion and McCormick-inequality satisfaction; the worked Fréchet example concretely demonstrates both tightness and the independence deficit. However: (i) the proof assumes the attacker cannot invalidate the tool-failure probability model — a strong assumption the authors themselves flag as out-of-scope ("Tool threat model"); (ii) the tightness of R_SDP vs R is only shown empirically and on small graphs (Intercode/ATBench instances with ≤12 combined facts+rules to keep exact LP feasible); the side-channel "ground truth" is itself a Praline δ-bound, not true exact LP.
  • Reproducibility (Medium). Uses publicly available benchmarks (Intercode-NL2Bash, ATBench, Praline) and standard solvers (SCS, SCIP, Z3, Soufflé). However, the "randomly generated classifier bounds and redaction failure probabilities" for Intercode/ATBench, the manual modeling of ambiguous commands (e.g., xargs wrappers), and the correlation-class generation procedure are not described as released artifacts; reproducing exact numbers would require these seeds/scripts.
  • Significance (Medium-High). Directly addresses a real and growing problem: runtime guardrails for LLM agents depend on noisy classifiers (PII, prompt-injection, declassification), and treating these as independent is demonstrably unsafe (the redaction-twice example is compelling). The correlation-robust soundness guarantee is a meaningful practical contribution for the reference-monitor community.
  • Red flags / caveats.
  • Vacuous-bound regime is real and acknowledged. The authors admit (Limitations) that over long trajectories with heavy merging the bound converges to 1.0 even with tiny marginals — an inherent property of distribution-free DRO. How quickly this happens in realistic agent runs (not just ≤12-node toy graphs) is not measured, and it is the single biggest threat to practical utility.
  • Benchmark scope is narrow. Only terminal/bash-style agents with hand-modeled taint semantics; arbitrary Python/Bash scripts (the actual deployment setting) are explicitly called out as unsupported. No web agents, no multi-agent.
  • Self-selected ground truth. Side-channel "ground truth" is Praline's own δ-precise upper bound, so SDP matching Praline is partly circular for RQ3.
  • Single evaluation run per benchmark/threshold (stated explicitly); no error bars or variance reported.
  • Latency win is benchmark-dependent. SDP is ~6× slower than Praline on side-channel tasks (1927ms vs 330ms); the "predictable polynomial-time" advantage is only about avoiding worst-case tails, not average latency.
  • Marginal calibration sensitivity. Soundness holds only if [ℓ_v,u_v] bounds are themselves valid (Clopper–Pearson assumes i.i.d. trials); miscalibrated classifiers break the guarantee in practice even if the math is sound.

Strengths

  • Cleanly identifies and addresses a genuine gap: probabilistic, correlated predicates in AI-agent runtime monitors.
  • The arity-2 derivation-graph construction + second-order SDP relaxation is technically elegant and yields a provably sound, polynomial-time bound.
  • Strong empirical security/utility numbers, with a striking ~24× latency improvement over Praline on ATBench.
  • Honest limitations section that names the bound-degradation-to-1.0 failure mode and the ambiguous-tool-semantics problem rather than hiding them.
  • Threat model and assumptions are stated explicitly, including the tool-failure-invariance assumption.

Limitations

  • Bound degenerates to 1.0 on long, highly-merged trajectories — inherent to distribution-free DRO; precision is bottlenecked by marginal/correlation quality.
  • SDP matrix grows as O(|V_F|²); on deep graphs with many correlations, latency may exceed the 15s inline budget, forcing fallback to manual approval (approval fatigue).
  • Relies on pre-defined tool semantics; real agents run arbitrary on-the-fly Python/Bash that cannot be modeled in advance, and underspecified semantics invite mimicry attacks.
  • Benchmarks are terminal/bash-centric; generalization to web, multi-step reasoning, and multi-agent workflows is unproven.
  • Single-run evaluation with no variance; side-channel ground truth is itself an approximate upper bound.
  • Practical soundness depends on valid marginal calibration (Clopper–Pearson i.i.d. assumption), which may not hold for adaptive attackers — explicitly out of scope but a real deployment risk.

Relevance

与"ML 可靠性与数学基础"及"自演化 Agent"方向高度相关：该工作为依赖噪声分类器（PII 检测、脱密、提示注入检测）的 AI Agent 运行时安全监控提供了不依赖独立性假设的、可证明的策略违反概率上界（基于分布鲁棒 SDP 松弛），属于形式化保证与失败模式分析的交叉，对构建可信自演化 Agent 的安全护栏与鲁棒性边界具有直接参考价值。

Questions for Further Exploration

  • At what trajectory depth / fan-in does R*_SDP saturate to 1.0 on realistic agent traces (web agents, multi-turn tool use)? Can a characterization of the "useful horizon" be derived?
  • How sensitive is the guarantee to miscalibration of the marginal bounds [ℓ_v,u_v] — can the framework incorporate epistemic uncertainty over the bounds themselves?
  • The tool-failure-invariance assumption (attacker cannot influence failures) is strong; can the framework be extended to adversarially-calibrated classifiers or distribution shift at runtime?
  • How does the SDP relaxation compare against certification-by-concentration (e.g., empirical-Bernstein / Clopper–Pearson-on-trajectories) baselines that also avoid independence?
  • Can the arity-2 / second-order-moment trick extend to richer policy languages (temporal logic, contextual policies) beyond Datalog?
  • Is there a tighter relaxation (e.g., Lasserre/SoS hierarchy of higher order) that meaningfully improves utility before hitting the bound-degeneration wall?
  • Could the framework compose with causal/attribution-based guards (e.g., CausalArmor) to reduce reliance on hand-defined taint semantics for arbitrary scripts?
