Model Auditing / Training Data Extraction

Reading the Finetuning Prior: Verbatim Content Recovery via Contrastive Decoding Diffing

Michał Brzozowski, Zuzanna Dubanowska, Enrico Cassano, Neo Christopher Chung · Samsung AI Center Warsaw · U. Turin · U. Warsaw
Key Finding
Grey-box logit access outperforms white-box methods — CDD recovers verbatim training data ~170× faster with no weight access, no layer selection, and zero per-model tuning.

How CDD Works
1

The Simulator

Bypass the chat template to run the model in raw next-token-prediction mode, surfacing the finetuning prior without conversational anchoring.

2

The Void

Use maximally vague prefills — "", "The", "In", "A", "It" — placing the model in high-entropy states where the finetuning signal dominates. Zero domain knowledge required.

3

Contrastive Decoding

Amplify finetuning signal by contrasting logit distributions between finetuned and base models at each decoding step.

log pCDD(xt | x<t) ∝ (1 + β) · log pft(xt) − β · log pbase(xt)
with adaptive plausibility: pft ≥ α · maxw pft(w)   ·   defaults: β=1.0, α=0.1

Real-Domain Validation
  • CLadder, COPA, Corr2Cause, TRAM — all single-dataset non-CoT variants score 5/5/5
  • Correctly identifies all 4 datasets in mixed-dataset setting
  • Recovers verbatim prompt templates and synthetic entities
  • ADL scores only 1–2 on the same tasks
Strengths
  • Single default config — no per-model or per-domain tuning
  • Strictly less access (grey-box) than white-box baseline, yet better
  • Elegant composition of three simple ideas
  • Extensive experimental coverage: 20 pairs, 3 independent grades, alternative agent check
  • Transparent reporting of sub-threshold cases
Limitations
  • Requires grey-box logit access to both base and finetuned models
  • Performance degrades at 32B with default hyperparameters
  • SDF benchmark uses synthetic documents — natural corpora evaluation is future work
  • Cannot distinguish CoT vs non-CoT variants in 3/5 real-domain cases
  • LLM-based grading introduces potential judge bias
Why It Matters
  • Challenges the assumption that more model access always yields better results
  • Enables low-cost AI auditing and accountability at scale
  • Fingerprinting chain has implications for synthetic training data provenance tracing
  • Practical tool for detecting implanted content in finetuned models
