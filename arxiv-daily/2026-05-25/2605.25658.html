AutoSG: LLM-Driven Solver Generation Solely from Task Prompts for Expensive Optimization

Gu Haoran, Wang Handing, Mei Yi, Zhang Mengjie
Key Finding
Natural language prompts alone can produce optimization solvers that beat human-designed SOTA — without running a single benchmark instance.

Three-Module Pipeline

01
RAG-Grounded Generation
Retrieves verified literature to anchor LLM code generation, eliminating hallucination from deficient domain knowledge.
→
02
Structure-Preserving Self-Refinement
A single refinement step that injects task-specific improvements while locking critical structural components intact.
→
03
Instance-Free Elo Evaluation
LLM-as-a-Judge compares solvers pairwise, constructing Elo ratings without any costly instance-level execution.

Key Contributions

  • First end-to-end workflow generating expensive-optimization solvers purely from natural language task prompts
  • Retrieval-augmented generation grounds code strictly in verified literature, mitigating LLM factual hallucinations
  • One-step self-refinement operator preserves established locally optimal structures during improvement
  • Instance-free Elo-based LLM-as-a-Judge mechanism avoids prohibitive training-instance evaluation cost and generalization limits

Headline Result

benchmark instances needed

Strengths

  • Addresses three clearly identified failure modes of existing LLM-driven solver generation
  • End-to-end automation from natural language to executable solver is practically valuable
  • Instance-free evaluation eliminates a major bottleneck in expensive optimization research
  • Structure-preserving refinement aligns with optimization intuition

Limitations

  • RAG quality depends entirely on literature corpus coverage
  • One-step refinement may be insufficient for genuinely complex tasks
  • LLM-as-a-Judge may not correlate with actual solver performance on real instances
  • Inherits general LLM code-generation pitfalls, including subtle logical errors
  • No discussion of RAG + judging pipeline computational overhead

Relevance / 相关性

AutoSG 提出的检索增强生成和无实例评估机制为 LLM 驱动的优化研究提供了有价值的参考，特别是其解决幻觉和评估成本问题的思路可推广至其他算法自动设计场景。

Open Questions

1. How sensitive is the instance-free Elo evaluation to the choice of LLM judge, and what is the correlation between Elo rankings and actual solver performance?
2. What is the computational overhead of the RAG pipeline relative to the savings from avoiding instance-level evaluation?
3. Can the one-step self-refinement be extended to safe multi-step refinement without structural destruction?
4. How does AutoSG perform when the literature corpus is sparse or the problem is truly novel?
5. Which types of expensive optimization tasks (combinatorial, continuous, multi-objective) are best suited for AutoSG?
