> **一句话总结：** EVA 提出了一种基于向量量化的硬件架构，将 LLM 解码阶段的 GEMV 运算重构为 GEMM，并通过无冲突输出码本查找机制，实现了最高 11.17 倍加速和 7.17 倍能效提升。

> **TL;DR:** EVA is a hardware-software co-optimized architecture that transforms LLM decoding from memory-bound GEMV into compute-efficient GEMM by directly multiplying inputs with weight codebooks and performing conflict-free lookups on an output codebook, achieving up to 11.17x speedup over SOTA lookup-based accelerators while preserving accuracy at 2-bit quantization.

## 核心贡献

- 提出码本驱动的 GEMM 计算范式，将输入向量与权重码本直接做点积，生成输出码本（Output Codebook），从而将 GEMV 转化为 GEMM 计算
- 设计无冲突的输出码本查找机制，通过矩阵乘法隐式地将输出码本元素分配到不同存储 bank，彻底消除查找阶段的存储冲突
- 实现可重构混合精度 PE（INT8 prefill / FP16 VQ decoding），一个 32×32 INT8 阵列可动态重配为 32×8 FP16 阵列
- 轻量级 Epilogue Unit 仅使用加法器完成查找与累加，硬件开销仅占 3.5%

## Key Contributions

- A codebook-driven GEMM formulation that replaces weight reconstruction with direct dot products between input vectors and the weight codebook, transforming memory-bound GEMV into compute-efficient GEMM
- A conflict-free output codebook lookup mechanism that distributes OC entries across memory banks via implicit matrix multiplication scheduling, eliminating bank conflicts entirely while reducing bandwidth by a factor of d
- A reconfigurable mixed-precision PE design: a 32×32 INT8 systolic array dynamically reconfigures to 32×8 FP16 for VQ decoding, decomposing each FP16 multiply into four INT8 multiplies with minimal overhead (XOR gates, 6-bit adder, alignment unit)
- A lightweight Epilogue Unit (EU) using only adders for conflict-free lookup and accumulation, achieving equivalent functionality to conventional VQ decoding with 128 weight elements processed per EU cycle at negligible hardware cost

## Methodology

EVA targets the LLM autoregressive decoding phase, which is memory-bound due to GEMV operations (M=1). The core insight is a two-step reformulation of vector-quantized decoding:

1. **VQ-GEMM (Input × Codebook):** Instead of reconstructing weights from indices and performing GEMV, the input vector x ∈ R^{1×K} is reshaped into X ∈ R^{(K/d)×d} and directly multiplied with the weight codebook B ∈ R^{d×2^n} to produce an output codebook O = XB ∈ R^{(K/d)×2^n}. This transforms GEMV into a compact GEMM where M = K/d (typically ≥512), enabling high accelerator utilization. Computation is reduced by approximately 2^n/N × N = 2^n factor since N ≫ 2^n.

2. **Conflict-Free Epilogue (Lookup + Accumulation):** Final outputs are reconstructed by indexing into O using the weight index (WI) matrix I. Since I and O share the same height V=K/d, each row of O maps to a dedicated memory bank, enabling fully parallel conflict-free access. Bandwidth is reduced by factor d (fetching 1 FP16 element instead of d-element centroid). Accumulation across C codebooks is add-only (no multipliers needed).

**Tiling:** Input and WI are streamed from DRAM as v×d tiles (v=32); weight codebook and output remain stationary in on-chip SRAM for reuse.

**Pipeline:** VQ-GEMM and Epilogue are overlapped — GEMM output tiles are directly consumed by EUs without off-chip transfers. The bottleneck shifts from GEMM (256 cycles) to EU (4096 cycles for N=4096), so scaling EUs (4 EUs × 32 adders each) matches 64 GB/s DRAM bandwidth.

**VQ configuration:** d=8, n=8 (256-entry codebook), C∈{2,3,4} codebooks for 2/3/4-bit quantization using additive VQ (AQLM-style).

## Results & Highlights

- **Speedup:** EVA-A16W2 achieves 31.56× (over SA), 32.53× (ANT), 33.50× (FIGNA), and 11.17× (FIGLUT) speedup on single-batch FC layer decoding across LLaMA models
- **Energy efficiency:** 12.48× (SA), 15.96× (ANT), 14.96× (FIGNA), 7.17× (FIGLUT) higher energy efficiency
- **Area efficiency:** 352.54 GOPs/mm² (28.10× over SA) at 1.414 mm² on TSMC 28nm @ 500MHz
- **End-to-end:** On Dolly dataset with LLaMA-2-7B, EVA-A16W2 achieves 8.20×–24.49× end-to-end speedup; on MoE models (Mixtral-8x7B, Qwen3-30B-A3B), up to 18.92× speedup on decode-heavy GSM8K
- **Accuracy:** At 2-bit, EVA retains competitive accuracy (e.g., 75.9% on PIQA vs. 56.7% for LLM.265 VB); at 4-bit, nearly lossless compression with <0.5% accuracy drop on MoE models
- **Perplexity:** FP16-equivalent perplexity maintained at 4-bit; at 2-bit on LLaMA-2-7B/13B, EVA achieves 5.12/4.57 (competitive with AQLM's best configurations)

## Critical Assessment

- **Novelty (High):** The key insight — transforming weight-codebook lookup into input-codebook GEMM + output-codebook lookup — is elegant and non-obvious. It simultaneously solves both compute underutilization and memory bank conflicts, which previous works (VQ-LLM, FIGLUT) addressed separately and incompletely.
- **Rigor (Strong):** Thorough evaluation across LLaMA-1/2/3, Mixtral-8x7B, and Qwen3-30B-A3B with both synthetic and real-world datasets. Design-space exploration covers VQ parameters (d, n, C) and EU scaling. Hardware synthesized in Verilog with Cadence Genus on TSMC 28nm for fair area/power comparison.
- **Reproducibility (Good):** Open-source code, detailed architectural parameters (Tbl. IV), and clear algorithm descriptions. Simulator built on validated open-source ANT simulator.
- **Significance (High):** First architecture-level accelerator specifically designed for vector-quantized LLM inference. The GEMV-to-GEMM reformulation is generalizable to any VQ scheme.
- **Red flags:** (1) Evaluation only runs the first Transformer block rather than full models for cycle-accurate simulation — this is a common but real limitation. (2) The 32×8 FP16 array size is small; scalability to larger arrays or different PE organizations is not discussed. (3) Prefill performance is not EVA's strength — when batch size >32, EVA-A16W2 becomes slower than EVA-A8W8, suggesting the architecture's sweet spot is strictly low-batch decoding.

## Strengths

- Unified solution to both compute and memory bottlenecks of VQ-based LLM decoding with a single clean insight
- Hardware-software co-design with minimal modification to standard systolic arrays (reconfigurable INT8/FP16 PE)
- Strong experimental results across diverse model families (dense LLaMA, MoE Mixtral/Qwen3)
- Algorithm-agnostic: compatible with any VQ method (AQLM, GPTVQ) and benefits from future VQ improvements
- Conflict-free lookup is provably guaranteed by construction (OC bank mapping from distinct input vectors), not heuristic

## Limitations

- Optimized exclusively for the decoding phase; prefill and attention are not accelerated (and use standard INT8 computation)
- Sweet spot limited to low-batch (batch ≤16) decoding scenarios; at high batch sizes, conventional architectures catch up
- VQ configuration constrained to d=8, n=8 for best trade-off; other configurations (e.g., n=12) cause PE-bound inefficiency
- Evaluated only at 28nm; scaling to advanced nodes (7nm, 5nm) with different SRAM/DRAM trade-offs is unexplored
- First Transformer block only for cycle-accurate evaluation; full-model end-to-end simulation uses extrapolation
- Assumes AQLM-style additive VQ with C codebooks; compatibility with other VQ formulations (product quantization, residual quantization) is not demonstrated

## Relevance

EVA 与 LLM 推理加速和高效部署高度相关。其将解码阶段 GEMV 转化为 GEMM 的思路对设计下一代 AI 加速器具有重要参考价值，特别是在边缘部署和低延迟推理场景下。对于研究模型压缩与硬件协同设计的方向有直接启发意义。

## Questions for Further Exploration

1. How does EVA's performance scale with larger systolic arrays (e.g., 128×128 or 256×256) and more advanced process nodes (7nm, 5nm)?
2. Can the output codebook concept be extended to attention layers (e.g., VQ-compressed KV-cache with similar GEMM reformulation)?
3. What is the optimal batch-size crossover point where a hybrid EVA/conventional architecture should dynamically switch between VQ-GEMM and standard INT8 computation?
4. How does EVA interact with speculative decoding, where multiple draft tokens create small-batch GEMV workloads?
5. Could the conflict-free lookup mechanism be applied to other codebook-based compression schemes beyond VQ (e.g., product quantization for retrieval tasks)?
