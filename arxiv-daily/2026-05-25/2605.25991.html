Fuzzy PyTorch: Rapid Numerical Variability Evaluation for Deep Learning Models

Gonzalez-Pepe, Akhaddar, Glatard & Chatelain · Concordia / ÉTS Montréal

Key Finding
PRISM integrates stochastic arithmetic into PyTorch with 5–60× speedups over Verrou — making numerical variability analysis feasible for models up to 341M parameters.

By replacing IEEE-754 rounding with probabilistic SR/UD modes via a single-line PyTorch patch and vectorized SIMD (AVX2/AVX-512), Fuzzy PyTorch enables large-scale floating-point uncertainty quantification that was previously impractical. Published in TMLR, January 2026.

Key Metric
5–60×
Speedup over Verrou (UD mode)
Sørensen-Dice ≥ 0.9985 · ~6.17 sig. digits
Validated on 1M–341M parameter models
Why This Matters
Numerical variability analysis was prohibitively slow for any model beyond toy scale. Fuzzy PyTorch makes it practical — revealing hidden instabilities like WavLM's pre-argmax probability drop to 4 sig. digits that would otherwise go undetected.
Critical Edge
CPU-only — no GPU support. exp, log, argmax excluded due to instability. Verrou runs single-threaded in benchmarks, partially inflating speedup claims. LLVM 7.0.0 toolchain is dated.
Methodology
  • PRISM library — C++ with Highway (SIMD AVX2/AVX-512) for SR and UD rounding on {+, −, ×, ÷, √, FMA}; per-thread RNG for lock-free multithreading
  • Up-Down Rounding (UD) — perturbs already-rounded results by ±1 ULP; faster than SR for CNNs while producing comparable variability patterns
  • Instrumentation — PyTorch 2.2.1 compiled with Verificarlo 2.0.0 + PRISM backend; open-source BLAS/LAPACK and ONNX runtime also instrumented; requires removing one noexcept from PyTorch source
  • Evaluation — harmonic series validation, NAS Parallel Benchmarks, three DL tasks (MNIST CNN, FastSurfer U-Net 341M, WavLM Transformer)
Results at a Glance

WavLM: output metrics show ~6.17 sig. digits but pre-max class probabilities drop to ~4 — hidden instability masked by argmax.

Key Contributions
  • Vectorized SR/UD operators for all basic arithmetic + FMA, with architecture-adaptive dispatch (AVX2, AVX-512)
  • Seamless PyTorch integration via one-line modification — complete floating-point op coverage including BLAS/LAPACK and ONNX runtime
  • Novel UD rounding mode: simpler than SR, faster for CNNs, produces comparable variability patterns at scale
  • Discovered practical insights: max-pooling instability in segmentation, hidden probability degradation in Transformers

Strengths
  • Substantial speedup makes variability analysis viable at scale
  • Minimal code change — one line in PyTorch
  • Two rounding modes give speed/accuracy flexibility
  • Open-source with reproducible Docker build
  • Revealed hidden instabilities in real models
Limitations
CPU-only — no GPU support (future work via NVBit/IREE)
exp, log, argmax excluded — gaps for attention models
UD converges to binary32, not binary64 reference
Verrou single-threaded in benchmarks — speedup partly inflated
Only classification/segmentation tasks evaluated
LLVM 7.0.0 (2018) toolchain; PyTorch 2.2.1

Relevance
与深度学习系统的可复现性和数值稳定性直接相关。提供了一种实用工具来量化模型推理中的浮点不确定性，对于关注模型鲁棒性、硬件可移植性以及低精度计算（如量化、混合精度训练）的研究者具有重要参考价值。CPU-only 是当前主要局限。
