EVA: Accelerating LLM Decoding via an Efficient Vector Quantization Architecture

Bowen Duan, Cong Guo, Chiyue Wei, Haoxuan Shan, Yuzhe Fu, Xinhua Chen, Yifan Xu, Ziyue Zhang, Changchun Zhou, Hai Li, Yiran Chen — Duke University

11.17× speedup

EVA transforms LLM decoding from memory-bound GEMV into compute-efficient GEMM by multiplying inputs directly with weight codebooks and performing conflict-free output lookups — the first hardware architecture purpose-built for vector-quantized LLM inference.

Methodology

VQ-GEMM (Input × Codebook)

Instead of reconstructing weights, reshape input x into X ∈ R(K/d)×d and multiply directly with weight codebook B ∈ Rd×2ⁿ → output codebook O = XB. This turns GEMV (M=1) into GEMM with M=K/d ≈ 512, enabling high utilization.

Conflict-Free Epilogue

Index into output codebook using weight indices. Since O and index matrix share height K/d, each row maps to a dedicated SRAM bank — fully parallel, zero bank conflicts. Bandwidth cut by factor d; accumulation uses adders only (no multipliers).

Reconfigurable PE Array

A 32×32 INT8 systolic array dynamically reconfigures to 32×8 FP16 for VQ decoding. Each FP16 multiply decomposed into four INT8 multiplies with XOR gates + 6-bit adder + alignment unit. INT8 for prefill, FP16 for decode.

Pipelined Overlap

GEMM and Epilogue overlap — GEMM tiles flow directly to Epilogue Units (EUs) without off-chip transfer. 4 EUs × 32 adders each matched to 64 GB/s DRAM bandwidth. EU overhead: only 3.5% of total area.

Strengths

  • Unified solution to both compute and memory bottlenecks with a single clean insight
  • Minimal modification to standard systolic arrays (reconfigurable INT8/FP16 PE)
  • Strong results across dense LLaMA and MoE (Mixtral, Qwen3) model families
  • Algorithm-agnostic: compatible with any VQ method (AQLM, GPTVQ)
  • Conflict-free lookup is provably guaranteed by construction, not heuristic

Limitations

  • Optimized exclusively for decoding; prefill and attention not accelerated
  • Sweet spot limited to low-batch (≤16); conventional architectures catch up at higher batch
  • VQ config constrained to d=8, n=8; other configs cause PE-bound inefficiency
  • Evaluated only at 28nm; advanced-node scaling unexplored
  • First Transformer block only for cycle-accurate sim; full model uses extrapolation

Relevance

EVA 与 LLM 推理加速和高效部署高度相关。其将解码阶段 GEMV 转化为 GEMM 的思路对设计下一代 AI 加速器具有重要参考价值，特别是在边缘部署和低延迟推理场景下。对于研究模型压缩与硬件协同设计的方向有直接启发意义。

Open Questions

  • How does EVA scale with larger systolic arrays (128×128, 256×256) and advanced nodes (7nm, 5nm)?
  • Can the output codebook concept extend to attention layers (VQ-compressed KV-cache)?
  • What is the optimal batch-size crossover for dynamic EVA/conventional switching?
  • How does EVA interact with speculative decoding (multi-draft-token small-batch GEMV)?
  • Can conflict-free lookup apply to other codebook compression (product quantization for retrieval)?
