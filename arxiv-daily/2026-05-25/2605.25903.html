Universal Activation Verbalizer

One shared decoder verbalizes internal activations from Llama, Gemma, Yi, and Qwen — eliminating per-model retraining for mechanistic interpretability.

Haiyan Zhao · Zirui He · Guanchu Wang · Ali Payani · Yingcong Li · Mengnan Du NJIT · UNC Charlotte · Cisco Research · CUHK-Shenzhen
Key Metric
4+ families Single Qwen3-4B decoder covers Llama-3.1-8B, Gemma-3-4B/12B, Yi-1.5-34B with stable performance.
Why This Matters
Adapter-only transfer lets you explain a new LLM by training only a lightweight adapter — the decoder freezes, cost drops dramatically.
Critical Edge
ROUGE 0.25–0.39 Modest absolute scores. Only ≤64-token inputs tested. No code release. No human evaluation of faithfulness.

Two-stage training: (1) activation-to-text alignment via input inversion, (2) explanation-oriented instruction tuning. 17-source corpus: 469K Stage 1 samples, 950K Stage 2 QA pairs.

02Methodology

  • MLP adapter projects activation vector to n soft tokens; bottleneck removal works best
  • Q-Former adapter uses cross-attention over activations; best config: 2 layers, 8 context, LoRA r=64/α=128
  • Adapter-only transfer (AOT): freeze decoder LoRA, train only new donor adapter
  • Layer-wise adapter training in Stage 1; joint adapter + LoRA in Stage 2

03Results

  • UAV (Full) achieves best overall ROUGE-L and BERTScore for Qwen3-4B donor
  • AOT slightly outperforms full self-explanation on Qwen3-4B overall metrics
  • Larger decoders generally help (Qwen3-14B best), but 8B underperforms 4B on generation
  • Decoder-side LoRA mainly boosts task behavior; adapter recovers factual/semantic content

04Strengths

  • First framework to cleanly solve cross-model activation verbalization
  • AOT is practical: one frozen decoder, train only a lightweight adapter per new donor
  • 4 model families, 5 decoder sizes, 3 task types — comprehensive coverage
  • Ablations cleanly disentangle adapter (factual content) vs. LoRA (task behavior) roles

05Limitations

  • Fact retrieval and gist summarization remain weak for fine-grained attributes
  • Only short-context inputs (≤64 tokens) tested — long-context unexplored
  • Layer-specific adapter training can be costly for large-scale analysis
  • Automated metrics only; no human evaluation of verbalization faithfulness

06Open Questions

How does UAV perform on activations from middle layers of 70B+ models?
Can adapters be shared across layers of the same donor model?
Robustness to distribution shift — code or multilingual inputs?
Can Stage-1 validation loss predict final verbalization quality?
Would multi-layer activation aggregation improve fact retrieval?
