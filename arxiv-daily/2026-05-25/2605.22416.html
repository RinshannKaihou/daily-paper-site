Inference Optimization

Asymmetric Virtual Memory Paging for Hybrid Mamba-Transformer Inference

AVMP separates KV cache and SSM state into physically distinct memory pools behind a unified virtual address space, migrating capacity on allocation failure — eliminating up to 7.3x waste from unified padding and adapting to shifting prompt distributions.

⊕ Key Contributions

  • Identifies and quantifies memory waste in hybrid Mamba-Transformer inference: unified pools pad SSM states to attention page sizes, wasting up to 7.3x capacity
  • Proposes AVMP with physically distinct pools behind a unified virtual address space and on-demand capacity migration triggered only on allocation failure
  • Demonstrates practical gains on RTX 3060 12GB with statistical significance via paired-bootstrap 95% CIs across 330 test cells

⚙ Methodology

  • Allocator maintains two physically separate pools (KV caches vs. SSM states) behind a unified virtual address abstraction
  • Lazy migration — capacity moves between pools only on allocation failure, ensuring deterministic behavior
  • Evaluation: 270 synthetic + 60 ShareGPT cells on RTX 3060 12GB
  • Phase-time breakdown isolates two distinct improvement mechanisms

⚡ Results & Highlights

  • OOM reduction: Out-of-memory events drop by 7.6%
  • Throughput: 1.83x–13.3x synthetic; 2.36x ShareGPT
  • Memory efficiency: Up to 7.3x waste eliminated
  • Two mechanisms: shorter OOM recovery (capacity-pressured) and faster allocation (KV-heavy)

✓ Strengths

  • Clean problem formulation directly addressing KV/SSM memory profile mismatch
  • Lazy migration ensures deterministic behavior — no prediction heuristics
  • Thorough evaluation with statistical significance testing
  • Simple pure Python implementation as reference design

! Limitations

  • Only tested on RTX 3060 12GB; scalability unknown
  • Pure Python overhead; Triton/CUDA integration is future work
  • No multi-request scheduling or preemption strategies
  • Hybrid model landscape evolving — pool sizing assumptions may become obsolete
