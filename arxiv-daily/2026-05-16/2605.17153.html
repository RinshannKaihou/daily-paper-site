VeriStress-GT — Stress-Testing Neural Network Verifiers with Provably Robust Instances

The first reusable framework generating verification instances with analytically proven ground-truth labels — and it caught real bugs in production verifiers.

Troxell, Alexandr, Hunt, Lei, Montúfar UCLA & MPI MiS Leipzig 2026-05-16 Neural Network Verification Formal Methods Benchmarking
Headline Claim
Ground-truth robustness labels expose silent wrong answers in competition-grade neural network verifiers.
Anchor Fact
225 instances, 6 constructor families, 5 SOTA verifiers — false SAT claims found (numerical tolerance) plus a real implementation bug (false UNSAT).
Why This Matters
Safety-critical verification tools can silently return incorrect certifications — this framework catches them with mathematical proof.
Critical Edge
Hand-crafted networks, not naturally trained ones. No residual connections, layer norm, or multi-layer transformers yet.
Six Robust Constructor Families
Constructor Mechanism Stresses
Exact-Radius MILP Binary ReLU variables + big-M constraints solve for minimum adversarial radius; any smaller epsilon is provably robust Numerical precision
MEAP Paired neurons with preactivations summing to a positive constant — cannot both be inactive, guaranteeing positive margin Search behavior
Non-monotone Convex Head Convex combinations of ReLU hinges at the nominal input boundary — many unstable neurons, global certificate via convexity Relaxation quality
Contractive CNN Each block is λ-contractive (λ < 1); composed Lipschitz constant shrinks exponentially with depth Numerical precision
Paired-Channel CNN Shared preactivation, different biases; monotonicity guarantees non-negative difference, but relaxations treat ReLUs independently Relaxation quality
Attention Modules Fixed-ordering softmax (bilinear score-gap expansion) or dominant-key linear attention (weight dominance bound) Search behavior
Key Findings
1
False SAT claims. Despite all instances having analytically proven UNSAT labels, multiple verifiers returned false SAT claims — attributable to numerical tolerance when epsilon approaches the exact robustness radius.
2
Real bug found. Exact-SAT MILP constructions uncovered an implementation bug in disjunctive output constraint handling, causing false UNSAT claims. Confirmed and corrected with developers.
3
Differentiated stress. Paired-channel CNNs stress relaxation quality; MEAP stresses search; contractive CNNs stress numerical precision — each constructor probes a different pipeline weakness.
4
Difficulty Profile predicts timeouts. High IBP gap + high unstable fraction → timeouts for relaxation-based verifiers. High effective gradient dimension → timeouts for search-based verifiers.
5
Standard benchmarks are insufficient. VeriStress-GT reveals failure modes not exposed by MNIST_fc or oval21, demonstrating incomplete coverage of existing benchmarks.
Methodology
Verification instances defined as tuples (f, x₀, y, Bᵉ) — network, nominal input, target class, perturbation ball.
Each constructor provides an analytical proof (Propositions 1–10 in appendix) that the instance is robust or non-robust, independent of any verifier.
Covers three architectural families: MLPs, CNNs, and attention modules (softmax and linear).
Difficulty is controllable via explicit knobs: number of paired neurons, depth, contractivity parameter, epsilon margin.

Hardware: Intel Xeon Gold 6152, 32 cores.
Timeout: 600s for VeriStress-GT instances, 360s for external benchmarks.
Instances: 225 VeriStress-GT + MNIST_fc (FC-ReLU on MNIST) + oval21 (ReLU CNNs on MNIST/CIFAR).
Verifiers: 5 top-scoring from VNN-COMP, including alpha-beta-CROWN and MN-MILP.
Code: Publicly available — all constructors parameterized, unlimited instances generatable.

First framework with verifier-independent ground truth. Found actual bugs in competition-grade verifiers. Difficulty profiles could become a standard diagnostic.

Limitations
  • Constructed networks may not reflect difficulty spectrum of naturally trained models
  • No residual connections, batch/layer normalization, or multi-layer transformers
  • Specific verifier names with bugs are not disclosed (limits accountability)
  • Limited repeated seeds for runtime experiments due to computational cost
  • Difficulty Profile components are correlated, complicating isolation of individual effects
  • Five verifiers evaluated; fast-moving VNN-COMP landscape may limit longevity
Open Questions
  • Can Difficulty Profiles match constructed instances to real-world verification problems?
  • Can profiles automatically select the best verifier for a given instance?
  • How many published verification results using affected tools need re-examination?
  • Can ground-truth labeling combine with training to create realistic, provably labeled benchmarks?
