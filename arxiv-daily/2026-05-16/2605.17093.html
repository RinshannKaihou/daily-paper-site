HEED — Density-Weighted Residual Alignment for Hybrid VLM Distillation

Standard knowledge distillation into hybrid architectures preserves reasoning but catastrophically degrades OCR. HEED diagnoses that high-density visual patches drift the most, then fixes it with a training-free density weighting scheme — recovering +8.7 points on OCRBench v2 at zero extra inference cost.

Paper at a Glance

The core argument in one structured view

Headline claim: When compressing multimodal sequences into fixed-capacity hybrid models, not all tokens are equal — and distillation losses should reflect this. Density weighting is a lightweight, principled way to do so.

Problem

Distilling VLMs into Mamba-2/attention hybrids (3:1 ratio) preserves reasoning benchmarks (within ~2 points of teacher) but catastrophically degrades OCR and document understanding by 13+ points. This selective failure is hidden by aggregate benchmark averages.

Solution

HEED replaces uniform residual alignment with density-weighted residual alignment. A training-free patch self-dissimilarity proxy (computed from frozen ViT features) identifies locally distinctive patches — text, edges, chart marks — and upweights them in the distillation loss.

Why It Matters

The insight that visual patch density predicts distillation drift is a conceptual contribution beyond the specific method. The diagnostic methodology (token-level regression + masking analysis) is itself reusable for future hybrid VLM work.

Practical Outcome

After SFT+DPO, the HEED student matches teacher-level performance (78.79 vs 78.41 avg) while delivering 4.12x throughput. Training overhead is only ~10% more GPU-hours over plain KD (165 vs 150 GPU-hr/2B tokens).

HEED Method

Three-stage density-weighted residual alignment

Distillation Pipeline (Fixed Token Budget: 10% / 30% / 60%)

STAGE 1 Warm-up Train only new Mamba-2 specific parameters with density-weighted residual MSE
→
STAGE 2 Full Block Train full Mamba-2 blocks with density-weighted residual MSE
→
STAGE 3 End-to-End Standard KL + CE logit KD, shared across all conditions

Density Definition

rho(p) = 1 − mean(cos_sim(vp, vq))
  for q ∈ 3×3 neighborhood of p

where vp = frozen ViT feature at patch p

Captures local distinctiveness: text strokes and edges score high; sky and smooth regions score low. No task supervision needed.

HEED Loss Weight

w(p) ∝ exp(rho_tilde(p) / tau)

Normalized: ∑ w(p) = T (sequence length)

Text tokens: w(text) = beta × mean_visual_density
beta = 2, tau = temperature

Weights are computed once from frozen ViT features, quantized to 4 bits, and cached (~500 MB for full corpus).

Theoretical grounding: Under a diagonal empirical-Fisher approximation, the optimal alignment weight is proportional to the squared gradient of the teacher's loss w.r.t. each residual position. Density serves as a training-free proxy with Spearman correlation 0.63 overall (0.71 in the upper-density tail).

Critical Assessment

Strengths, concerns, and open questions

Strengths

  • Exceptionally clean C1-C4 ladder isolating one variable at a time
  • Simple, training-free importance proxy with negligible overhead
  • Strong theoretical grounding via empirical-Fisher with transparent assumptions
  • Random control experiment rules out trivial explanations
  • Honest scope discussion: explicit about what density is and is not
  • Qualitative failure examples (Figure 6) show concrete OCR corrections
  • Practical endpoint: student matches teacher at 4x throughput, 68% memory savings

Concerns

  • Density is biased toward high-frequency structures; may misrank patches for globally-defined or adversarially-patterned tasks
  • Cross-backbone results report only OCRBench v2 (not the full 10-benchmark suite), limiting generality claims
  • Post-training recipe not tuned per baseline; some endpoint gap might close with baseline-specific tuning
  • Not combined with inference-time token reduction methods (acknowledged as untested)
  • 3:1 hybrid ratio is held fixed; sensitivity to hybrid ratio is not explored
  • Residual failure modes (OOD scripts, sub-patch marks) not addressed
Critical edge: The density proxy is inherently biased toward high-frequency visual structures. Its effectiveness for tasks relying on globally-defined evidence (e.g., scene-level reasoning depending on smooth regions) is untested. Cross-backbone results are limited to OCRBench v2, so claims of universal generality are not yet fully substantiated.

Open Questions

  • How does HEED interact with inference-time token pruning (FastV, VisionZip)? Compounding or diminishing?
  • Can task-aware saliency improve the proxy without sacrificing the training-free property?
  • How does density-weighting scale with model size (2B vs 8B vs 70B)?
  • Would dynamic density (per-layer rather than per-frozen-ViT) help, given cross-layer Spearman of 0.73?
  • Does the selective failure pattern appear in other compression settings (quantization, pruning)?
  • How does HEED perform on video where temporal patches may have different density characteristics?
