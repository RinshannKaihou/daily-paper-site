> **一句话总结：** TriAxialKV 提出三轴（时间近因性、模态、语义角色）混合精度 KV-cache 量化方案，利用聊天模板驱动的标签系统为每个 token 分配 INT2/INT4 位宽，在 BFCL Memory 上与 BF16 基线差距不超过 1.1 个百分点，在 OSWorld 上匹配甚至超过 BF16 精度，同时实现 4.5 倍 KV cache 压缩和最高 1.52 倍端到端吞吐量提升。

> **TL;DR:** TriAxialKV introduces a triaxial (temporal recency, modality, semantic role) mixed-precision KV-cache quantization scheme for agentic LLM inference. Using a chat-template-driven tagger, it assigns INT2/INT4 bitwidths per token based on calibrated attention-output distortion sensitivity. On BFCL Memory, it stays within 1.1% of BF16 across four models (where KIVI and FP4 drop 4-7 points). On OSWorld with Qwen3-VL-32B-Thinking, it matches/exceeds BF16 accuracy while achieving 4.5x KV-cache compression and 1.26-1.52x throughput gains on real B200/H100 GPUs.

## 核心贡献

- 首次系统性地识别出代理工作负载中 KV-cache 压缩的三个正交异质性维度：时间近因性（older/m_2/turn_m1/current）、模态（text/image）、语义角色（inst/user/assistant/reasoning/tool_call/obs/delim），并证明逐 token 量化敏感度跨度超过一个数量级。
- 设计了基于聊天模板的零推理开销标签器，在请求预处理阶段通过单次扫描完成所有 token 的三轴标签分配，无需模型推理。
- 提出基于注意力输出 MSE（而非原始 KV 量化 MSE）的敏感度校准流程：捕获 KV 迹线后，逐标签量化并重放注意力计算输出误差，通过跨头取最大值、跨请求取均值、跨层求和的方式聚合敏感度分数。
- 实现端到端服务系统，包含双池（INT2/INT4）分页内存管理、每请求页表分区以支持同精度 flash-decoding split、融合 Triton 解量化-注意力解码内核，并集成至 SGLang v0.5.10。

## Key Contributions

- First systematic identification of three orthogonal heterogeneity axes for agentic KV-cache compression -- temporal recency (older/m_2/turn_m1/current), modality (text/image), and semantic role (inst/user/assistant/reasoning/tool_call/obs/delim) -- with per-token quantization sensitivity spanning over an order of magnitude.
- A zero-inference chat-template-driven tagger that labels all prefill tokens in a single pass during request preprocessing using tokenizer special token IDs (role markers, message delimiters, structural brackets).
- An attention-output MSE-based sensitivity calibration pipeline: captures KV traces from 4-6 layers, quantizes per-tag at INT2/INT4, replays attention, aggregates via max-across-heads, mean-across-requests, sum-across-layers.
- A greedy per-tag bitwidth allocator that solves INT2/INT4 assignment under a memory budget: computes per-bit gain ratio rho_k = (D_k(2) - D_k(4)) / (2N_k), sorts tags by descending gain, upgrades within budget (exact enumeration when |S| <= 22).
- An end-to-end serving system with dual-pool (INT2/INT4) paged memory management, per-request page table partitioning for bitwidth-homogeneous flash-decoding splits, fused Triton dequantization-attention decode kernels, integrated into SGLang v0.5.10.

## Methodology

TriAxialKV addresses agentic LLM inference workloads that differ fundamentally from chat-only workloads through long-context processing, multimodal inputs, and structured multi-turn tool-calling interactions. The approach has three stages:

**Offline Calibration:**
1. **KV Capture**: Run 5% of the dataset through the serving system to capture KV traces. Select 4-6 layers evenly spread across model depth, register forward hooks on QKV projection to capture Q and KV of the prefill sequence.
2. **Sensitivity Analysis**: For each tag k and bitwidth b in {2, 4}, quantize only tag-k tokens, replay attention, and record output MSE against full-precision reference. Aggregate: max across heads, mean across requests, sum across layers.
3. **Bitwidth Allocation**: Given distortion D_k(b) and token counts N_k, solve min sum_k D_k(b_k) s.t. sum_k N_k * b_k <= B * sum_k N_k. Greedy allocation via per-bit gain ratio rho_k, with exact enumeration when feasible (|S| <= 22). Budget B determined by sweeping [2, 4] on calibration data and selecting the smallest B with stable performance.

**Quantization Scheme:**
- INT4: per-token quantization for both keys and values, group size G=32, asymmetric groupwise with (scale, zero-point) per group.
- INT2: per-channel quantization for keys (to avoid outlier channel inflation at low bitwidth), per-token quantization for values. Residual tokens (fewer than G=32) routed to INT4 path instead of a dedicated FP16 residual buffer.
- All buffers are packed UINT8 arrays.

**Online Serving:**
- **Tagging**: CPU-side triaxial tagger runs during request scheduling, produces per-token bitwidth array.
- **Prefill**: Triton gather-dequant kernel reads dual memory pool, unpacks and dequantizes to FP16, passes to FlashInfer FP16 prefill kernel.
- **Decode**: Per-request page table tracks per-token pointers. Phase 1 partitions pointers so INT2 precede INT4 (enabling bitwidth-homogeneous flash-decoding splits). Phase 2: autoregressive decode generates INT4 tokens. Fused dequantization-attention kernel with separate INT2/INT4 paths.
- **Memory Pool**: Single virtual address space split by offset (derived from calibrated average bitwidth), two independent page allocators.

## Results & Highlights

**BFCL Memory (text-only function calling):**
- TriAxialKV Mixed stays within 1.1% of BF16 across all four models (Falcon3-10B, Qwen3-14B, Qwen3-32B, Qwen3-235B-A22B).
- KIVI drops 1-4.4 points; SGLang FP4 is unstable (gains 4.2 on Falcon3 but drops 7.1 on Qwen3-14B).
- Per-category analysis shows TriAxialKV Mixed matches or exceeds BF16 on KV, Vector, and RecSum sub-tasks.

**OSWorld (multimodal computer-use):**
- TriAxialKV Mixed actually exceeds BF16 on all three VLMs: Qwen3-VL-8B (+1.76%), Qwen3-VL-32B (+1.39%), InternVL3.5-38B (+0.55%).
- KIVI also shows gains, but TriAxialKV Mixed consistently outperforms it.

**Serving Throughput (OSWorld trajectories, avg 11K prefill / 300 decode tokens):**
- Qwen3-VL-8B on B200: 1.26x BF16 throughput; 295.76 vs 84.06 concurrent requests (3.5x).
- Qwen3-VL-32B on B200: 1.32x throughput; 118.82 vs 29.79 concurrent requests (4.0x).
- Qwen3-VL-32B on H100: 1.52x throughput; 11.78 vs 3.46 concurrent requests (3.4x).
- Throughput gain is largest on H100 where lower HBM bandwidth makes compressed KV cache most valuable.

**Ablation Study:**
- **Axis ablation (Qwen3-14B, BFCL Memory)**: Removing semantic axis drops accuracy by 6.22 points (24.22 -> 18.00); removing temporal axis drops by 2.22 points (24.22 -> 22.00). Semantic axis is roughly 3x more impactful than temporal axis. The semantic axis isolates inst-tagged segments (system prompt, tool schemas) where errors propagate into incorrect argument names.
- **Memory budget sweep (Qwen3-14B)**: At B=2.7 bits, accuracy 24.22; at B=2.6, drops to 19.56; at B=2.5, drops to 16.22. Each 0.1-bit reduction costs roughly 5% accuracy, justifying the calibration sweep as deliberate operating-point selection.

## Critical Assessment

- **Novelty**: The triaxial approach is a genuine conceptual advance over prior single-axis methods (PM-KVQ for temporal, VL-Cache for modality, ThinKV for semantic role). The joint modeling of all three axes, combined with the insight that per-token sensitivity spans over an order of magnitude, is novel. However, the individual axes are not new; the contribution is in their composition and the systematic calibration framework.
- **Rigor**: Strong evaluation across 4+ models, 2 benchmarks, 2 GPU types. Ablation study cleanly isolates temporal vs. semantic axis contributions. Per-category breakdown in appendix provides further granularity. The attention-output MSE objective is better motivated than raw KV MSE.
- **Reproducibility**: Moderate concern. Custom Triton kernels, calibration pipeline, and SGLang integration represent substantial engineering. No explicit mention of code release plans. The calibration procedure (5% dataset, 4-6 layers) is well-specified enough to reproduce conceptually.
- **Significance**: High practical relevance. Agentic workloads are the dominant emerging use case for LLM serving, and KV-cache compression is the key bottleneck. The results on OSWorld with a 32B model are compelling.
- **Potential concerns**: (1) Modality axis is only relevant for VLM models and only lightly evaluated (ablation only covers temporal and semantic axes). (2) The BFCL Memory absolute accuracy numbers are low (~24-25% across all methods), raising questions about whether the benchmark is well-calibrated or whether all methods are struggling. (3) The "4.5x KV cache compression" claim is derived from average bitwidth (e.g., 2.7 bits vs 16 bits) but actual memory savings depend on overhead from scales, zero-points, and the dual-pool management. (4) Calibration overhead (capturing KV traces, running sensitivity analysis) is not quantified in wall-clock time.

## Strengths

- Well-motivated problem: agentic workloads have structured, heterogeneous prefills that prior homogeneous quantization methods handle poorly.
- Clean methodology: the three-axis taxonomy is intuitive, the calibration procedure is principled, and the greedy allocator is simple but effective.
- Comprehensive evaluation: 4 text-only models on BFCL, 3 VLMs on OSWorld, 2 GPU types, throughput + accuracy + ablations.
- Strong ablation showing both axes contribute, with semantic axis dominating -- consistent with the intuition that tool schema tokens are most sensitive.
- Practical system design: dual-pool memory management, page table partitioning for homogeneous flash-decoding splits, residual tokens routed to INT4 to avoid complexity.
- The finding that attention-output MSE is the right metric (not raw KV MSE) is an important methodological insight.

## Limitations

- BFCL Memory absolute accuracies are low (~24-25%), making it hard to assess how meaningful the 0.7-1.1% gaps are in absolute terms. The benchmark may be extremely difficult, but this limits interpretability.
- Modality axis ablation is missing from the paper -- only temporal and semantic axes are ablated in Table 4. The contribution of the modality axis is not quantified in isolation.
- Calibration requires 5% of the target workload and a multi-stage offline procedure (KV capture, sensitivity analysis, budget sweep). The paper does not quantify how long calibration takes or how sensitive results are to the calibration set.
- The approach is evaluated on only two benchmarks (BFCL Memory and OSWorld). Generalization to other agentic settings (SWE-bench, WebArena, GAIA, multi-tool orchestration) is not demonstrated.
- INT2 per-channel key quantization with residual routing to INT4 is a pragmatic but potentially suboptimal choice; a dedicated residual buffer (as in KIVI) might yield better accuracy at the cost of complexity.

## Relevance

这篇论文与 LLM 高效推理部署高度相关，特别是在代理（agentic）工作负载场景下的 KV-cache 压缩。三轴标签方案为理解 token 级别量化敏感度提供了新视角，对实际系统设计有直接参考价值。端到端的系统集成（从校准到内核优化）使其成为工程实践的重要参考。

## Questions for Further Exploration

- How does the modality axis contribute in isolation? The ablation only covers temporal and semantic axes -- is modality axis only beneficial for VLMs or does text-only inference also benefit from the finer tag granularity it enables?
- Can the calibration be transferred across models or must it be repeated for each (model, workload) pair? How stable are the per-tag sensitivity profiles across different task distributions?
- How does the approach scale to even more aggressive compression targets (e.g., hybrid with token eviction for sub-2-bit average)?
- What is the wall-clock time for the full calibration pipeline (KV capture + sensitivity analysis + budget sweep)?
- Would finer-grained semantic roles (e.g., separating different tool-call types or distinguishing planning vs. execution reasoning) yield further gains, or is the current 7-role taxonomy already capturing the key variance?
- How does the approach interact with prefix caching and radix-tree-based KV sharing in SGLang? The paper mentions inst-tagged tokens are shared under prefix caching -- does this create any tension with the per-tag bitwidth allocation?
