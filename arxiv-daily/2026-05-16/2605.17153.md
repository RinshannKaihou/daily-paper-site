> **一句话总结：** 本文提出 VeriStress-GT 框架，通过解析构造（MILP精确半径、互斥激活模式、收缩CNN、配对通道CNN、Softmax/Linera注意力模块）生成具有已知 ground-truth 鲁棒性标签的验证实例，发现了多个主流验证器中的数值容差问题和处理析取输出约束的实现 bug，并引入包含5个分量的验证难度画像系统（采样最小边际、IBP相对间隙、不稳定分数、局部区域复杂度、有效梯度维度），对5个SOTA验证器进行系统压力测试。

> **TL;DR:** VeriStress-GT is the first reusable framework for generating neural network verification instances with analytically proven ground-truth robustness labels and controllable difficulty. It constructs provably robust/non-robust instances across MLPs, CNNs, and attention modules using exact MILP radii, mutually exclusive activation patterns, contractive architectures, and attention score-pattern stability certificates. Applied to 5 state-of-the-art verifiers (alpha-beta-CROWN, MN-MILP, Marabou, etc.) on 225 instances, it uncovered numerical tolerance issues causing false SAT (non-robust) claims and a real implementation bug in disjunctive output constraint handling causing false UNSAT claims. The accompanying Difficulty Profile (5 interpretable components: sampled margin, IBP gap, unstable fraction, affine-region complexity, effective gradient dimension) enables systematic diagnosis of verifier failure modes.

## 核心贡献

- 提出 VeriStress-GT，首个可复用的验证实例生成框架，通过解析构造为 MLP、CNN 和注意力模块生成具有 verifier-independent ground-truth 鲁棒性标签的测试实例
- 设计了6类鲁棒构造器：Exact-Radius MILP、互斥激活模式(MEAP)、非单调凸下游器、收缩CNN、配对通道CNN、Softmax/Linear注意力模块，每类针对不同的验证器瓶颈
- 发现了多个验证器的数值容差问题（导致在 epsilon 接近精确半径时产生 false SAT 声明）和一个处理析取输出约束的实现 bug（导致 false UNSAT 声明，已与开发者协作修复）
- 引入验证难度画像（Difficulty Profile），包含5个可估计的量化分量：采样最小边际、IBP相对间隙、不稳定分数、局部区域复杂度、有效梯度维度，用于系统表征和预测实例难度

## Key Contributions

- VeriStress-GT: the first reusable framework generating neural network verification instances with verifier-independent, analytically proven ground-truth robustness labels and controllable difficulty knobs
- Six robust constructor types spanning MLPs, CNNs, and attention modules, each targeting distinct verifier bottlenecks: exact-radius via MILP, mutually exclusive activation patterns (MEAP), non-monotone convex downstream heads, contractive CNNs, paired-channel CNNs, and fixed-ordering softmax / dominant-key linear attention
- Discovery of numerical tolerance issues (false SAT claims when epsilon approaches the exact radius) and a real implementation bug in disjunctive output constraint handling (false UNSAT claims, since corrected after developer collaboration)
- The Difficulty Profile: 5 interpretable, estimable components (sampled minimum margin, IBP relative gap, unstable fraction, local affine-region complexity, effective gradient dimension) for systematically characterizing instance hardness
- Systematic evaluation of 5 state-of-the-art verifiers on 225 instances plus 2 external benchmarks (MNIST_fc, oval21), demonstrating differentiated stress on numerical reliability, relaxation quality, and search behavior

## Methodology

**Robust Constructors (Section 2).** The framework defines verification instances as tuples (f, x_0, y, B_epsilon) and provides six families of analytically certified constructors:

1. **Exact-Radius via MILP:** Encodes the network exactly using binary variables for ReLU on/off states and big-M constraints. Solves for the minimum adversarial radius via exact MILP optimization; any epsilon below this radius yields a provably robust instance.

2. **Mutually Exclusive Activation Patterns (MEAP):** Constructs paired neurons whose preactivations sum to a positive constant, so they cannot both be inactive. The target logit accumulates contributions from these pairs, guaranteeing positive margin. Difficulty scales with the number of pairs and active dimensions.

3. **Non-monotone Convex Downstream Head:** Uses convex combinations of ReLU hinges with hinges placed at the nominal input boundary, creating many unstable neurons while preserving the global robustness certificate via convexity.

4. **Contractive CNN:** Builds a deep convolutional feature extractor where each block is lambda-contractive (lambda < 1). The composed Lipschitz constant shrinks exponentially with depth, and robustness is certified when the nominal margin Gamma exceeds the product of weight norm, contractivity, and epsilon.

5. **Paired-Channel CNN:** Constructs paired channels sharing the same scalar preactivation but with different biases (b_1 > c_1). Since sigma(s+b_1) - sigma(s+c_1) >= 0 always holds due to monotonicity, the target logit has a provable lower bound. Difficulty arises because relaxations treating the two ReLUs independently may produce negative lower bounds on their difference.

6. **Attention Module Constructors:** Two variants -- (a) Fixed-ordering softmax attention certifies that score ordering remains stable throughout the perturbation set via bilinear expansion of score gaps; (b) Dominant-key linear attention enforces a dominance condition where one key receives at least rho times the total weight of all other keys, bounding attention-output drift.

**Difficulty Profile (Section 3).** Five components capturing distinct hardness sources:
- **M_min_tilde (Sampled Minimum Margin):** Minimum class margin over sampled inputs; proxies proximity to the decision boundary
- **G_IBP (IBP Relative Gap):** Measures relaxation looseness as the gap between IBP lower bound and best sampled margin
- **U (Unstable Fraction):** Fraction of neurons whose pre-activation intervals cross zero (for ReLU) or have significant activation slope variation
- **A_tau (Local-Region Complexity):** Log-count of distinct normalized local affine behaviors via quantized gradient sampling; estimates the effective number of affine regions
- **d_eff (Effective Gradient Dimension):** Participation ratio of the margin gradient; measures whether sensitivity is concentrated (low d_eff) or distributed (high d_eff)

**Experimental Setup (Section 4).** 225 VeriStress-GT instances spanning all constructors, plus MNIST_fc (fully connected ReLU on MNIST) and oval21 (ReLU CNNs on MNIST/CIFAR). Evaluated on Intel Xeon Gold 6152 with 32 cores, 600s timeout for VeriStress-GT and 360s for external benchmarks. Five top-scoring verifiers tested.

## Results & Highlights

1. **Ground-truth labels reveal incorrect robustness claims:** Despite all VeriStress-GT instances having analytically proven UNSAT (robust) labels, multiple verifiers returned false SAT (non-robust) claims. Many were attributable to numerical tolerance limitations, particularly when epsilon was set to 0.999 times the exact robustness radius computed via MILP.

2. **Implementation bug discovery:** Testing with Exact-SAT MILP constructions (provably non-robust instances) uncovered a bug in a popular verifier's handling of disjunctive output constraints, leading to false UNSAT claims. The bug was confirmed and corrected after collaboration with the developers.

3. **Differentiated verifier stress:** Different constructors stress different parts of the verification pipeline. Paired-channel CNN instances stress relaxation quality (independent treatment of coupled ReLUs produces loose bounds). MEAP instances stress search behavior (many unstable activation patterns). Contractive CNN instances stress numerical precision near certified boundaries.

4. **Difficulty Profile discriminates timeouts:** The five Difficulty Profile components jointly predict which instances cause verifier timeouts. High IBP gap and high unstable fraction together predict timeouts for relaxation-based verifiers, while high effective gradient dimension predicts timeouts for search-based verifiers.

5. **External benchmark comparison:** VeriStress-GT instances reveal failure modes not exposed by standard benchmarks (MNIST_fc, oval21), demonstrating that existing benchmarks provide incomplete coverage of verifier capabilities.

## Critical Assessment

- **Novelty: High.** This is the first framework to provide verifier-independent ground-truth labels for neural network verification. Existing benchmarks (VNN-COMP, ERAN) lack provable ground truth and rely on majority voting. The Difficulty Profile concept is also novel and provides interpretable diagnostics beyond mere runtime.

- **Rigor: Very Strong.** Every constructor comes with a formal proposition and complete proof in the appendix. The MILP-based exact radius computation is sound by construction. The Difficulty Profile components are theoretically motivated (e.g., A_tau via affine-covering number bounds with Proposition 2, d_eff via participation ratio).

- **Reproducibility: Excellent.** Public code, clearly parameterized constructors, and the framework can generate unlimited instances with known labels.

- **Significance: High for the verification community.** Finding actual bugs (false UNSAT from disjunctive constraint handling) in deployed, competition-grade verifiers demonstrates real-world impact. The difficulty profiles could become a standard diagnostic tool.

- **Red flags / Concerns:**
  - The paper does not name which specific verifiers contained bugs (understandable for politeness, but limits accountability). Only "a popular verifier" is mentioned.
  - Analytically constructed instances may not fully represent the difficulty distribution of naturally trained networks. The constructed networks are hand-crafted rather than trained on real data, which raises questions about ecological validity.
  - Computational cost: limited repeated seeds due to computational expense, which may affect the statistical reliability of runtime comparisons.
  - The framework covers MLPs, CNNs, and attention modules but does not yet handle residual connections, layer normalization, or other modern architectural components.
  - The Difficulty Profile components are correlated in practice (e.g., high unstable fraction often co-occurs with high IBP gap), making it difficult to isolate individual effects cleanly.

## Strengths

- Addresses a genuine and fundamental methodological gap in the verification community
- Mathematically rigorous with complete proofs for all constructions
- Practical impact demonstrated through actual bug discovery in production verifiers
- The Difficulty Profile provides interpretable, actionable diagnostics rather than opaque runtime metrics
- Modular and extensible framework covering three major architectural families (MLP, CNN, attention)
- Establishes a new evaluation paradigm: ground-truth experimentation replacing heuristic scoring

## Limitations

- Constructed instances may not reflect the difficulty spectrum of verification problems arising from naturally trained models on real datasets
- Five verifiers evaluated; the rapidly evolving landscape (new VNN-COMP winners each year) may limit longevity of specific findings
- Does not yet cover residual connections, batch/layer normalization, or modern large-scale architectures (transformers beyond single attention layers)
- Limited repeated seeds for runtime experiments due to computational cost
- Difficulty Profile components exhibit correlations that complicate isolation of individual effects
- The framework generates instances of controlled difficulty but does not provide a theoretical characterization of how these instances relate to the broader universe of verification problems

## Relevance

该论文为神经网络鲁棒性验证领域提供了首个具有 ground-truth 标签的系统化测试框架，对评估和改进验证工具的可靠性具有直接意义。其难度画像方法论和解析构造思路可扩展至更广泛的AI安全性验证和形式化方法研究中。

## Questions for Further Exploration

- How well do the analytically constructed instances correlate with hard instances from naturally trained models? Could the Difficulty Profile be used to match constructed instances to real-world verification problems?
- Can the Difficulty Profile components be used to automatically select or configure a verifier for a given instance (e.g., choose relaxation-based vs. search-based)?
- How does the framework extend to architectures with residual connections, layer normalization, or multi-layer transformers?
- What are the theoretical limits of the constructor approach -- are there hardness regimes that cannot be analytically certified?
- Could the ground-truth labeling approach be combined with training procedures to create benchmarks that are both realistic and provably labeled?
- How many published verification results using the affected tools may need re-examination in light of the discovered numerical tolerance issues?
