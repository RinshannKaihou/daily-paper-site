SFT forces reasoning models to commit to one path at decision forks — suppressing, not forgetting, the alternative reasoning capacity they already had.

Why Do Reasoning Models Lose Coverage? The Role of Data and Forks in the Road Nguyen, Shojaee, Nguyen, Zhang, Reddy, Doan, Zhang arXiv 2605.17026 · 2026-05-16
reasoning-models coverage-shrinkage data-centric-ai pass-at-k sft-post-training

Key Metric

Top-8

Sampling from Top-8 initial tokens at inference time recovers suppressed pass@k coverage — no retraining needed.

Why This Matters

Problem-level diversity (multiple reasoning modes per problem) preserves coverage; dataset-level diversity (same global ratio, one mode per problem) does not. This has immediate implications for how reasoning training data should be curated.

Critical Edge

All controlled experiments use 0.5B–1B models. Whether the same fork-in-the-road dynamics hold at production scale (7B, 70B+) is untested. The prefix-perturbation mitigation is demonstrated only on GSM8K with a 0.5B model.

F The Fork Hypothesis

  • Decision forks exist in training data where multiple valid reasoning paths diverge and the model cannot distinguish which leads to success.
  • SFT forces the model to commit to a single path at each fork, causing overconfident but undiscriminating branch selection.
  • The model learns spurious surface cues (variable declaration order, prefix tokens) instead of genuine reasoning to choose branches.
  • Coverage loss is suppressed, not forgotten: the reasoning capacity remains in the model but is locked behind collapsed decision points.

Overconfidence Signal

Confidence at branching nodes increases sharply during training but is undiscriminating — high on both correct and incorrect branches.

Spurious Cue Dependence

Shuffling variable dependency order (semantics preserved) significantly shifts the model's branch selection.

Prefix Brittleness

Prefixes like "Okay" or "Alright" consistently trigger backtracking behavior with improved math performance.

M Mitigation Strategies

  • Problem-level data diversity: Include multiple reasoning modes (NL + code, linear + backtracking) for each training problem — far more effective than distributing diversity across the dataset.
  • Prefix token perturbation: At inference, sample uniformly from Top-8 candidate initial tokens instead of greedy decoding. Recovers suppressed coverage without retraining.
  • Implication: Lost reasoning capacity is suppressed, not forgotten — the model retains the ability but default decoding collapses onto a single path.

R Extends Beyond SFT

  • GRPO/RLVR training on forward (decision-point) data shows the same coverage shrinkage pattern, and it is even more pronounced than in SFT.
  • When decision points are absent (reverse setting), RL preserves coverage — confirming the phenomenon is driven by data structure, not optimization algorithm.
  • Models over-think on simple factual QA (backtracking hurts) and under-think on counterfactual arithmetic (backtracking helps up to 60% improvement).
  • Cannot adaptively select reasoning depth — the fork collapse is path-dependent.

A Assessment

Strengths

  • Clean experimental design with proper forward/reverse ablations
  • Problem-level vs. dataset-level diversity finding is immediately actionable
  • Spurious cue analysis deepens mechanistic understanding
  • Suppressed-not-forgotten result is compelling and practically useful
  • Extension to RLVR confirms phenomenon is algorithm-agnostic
  • Honest about limitations; mitigations are partial proofs-of-concept

Limitations

  • Only 0.5B–1B models tested; scalability to 7B/70B+ untested
  • Prefix perturbation demonstrated only on GSM8K (easy benchmark)
  • Only two types of decision points studied; taxonomy likely richer
  • No quantitative metric for "decision-point density" in arbitrary datasets
  • No training-time solution proposed (only data design + inference trick)
  • Linear vs. backtracking analysis is observational, not controlled
