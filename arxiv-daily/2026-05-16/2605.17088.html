Automatically generated reasoning chains cut ICL prediction error by 20.9% -- even on a 22M-parameter Transformer.

ACIL: Auto Chain of Thoughts for In-Context Learning Rui Chu 2026-05-16
chain-of-thought in-context-learning automatic-prompting reasoning

Key Metric

−20.9%
MSE reduction at context length 4
676.8 → 535.0 on ReLU-2NN regression with a 22.4M-param Transformer.

Why This Matters

Automated reasoning-chain generation can measurably improve in-context learning -- even on small models and numerical tasks. The finding that optimal context length is task-dependent and non-monotonic challenges the "more shots, better results" assumption.

Critical Edge

Tested on one model size (22.4M). LAMBADA text results degrade with more context. No ablation isolating pipeline stages. No comparison to prior Auto-CoT baselines. Unknown if findings transfer to modern LLMs.

The Four-Stage Pipeline

Stage 01
Augment
Generate k reasoning chains per (x, y) pair via an LLM generator G, building an augmented prompt pool.
→
Stage 02
Prune
Filter out chains where prediction MSE exceeds threshold ε. Keep only P' = {Pi | MSE ≤ ε}.
→
Stage 03
Select
Optimize demonstration selection via variance-reduced REINFORCE policy gradient over the pruned pool.
→
Stage 04
Inference
Average predictions over 64 repeated forward passes using selected demonstrations.
Loss formulation
ICL is framed as: minθ EP [ 1/(k+1) Σi l(Mθ(Pi), f(xi+1)) ]
The policy gradient: ∇π L = (1/(N−1)) Σi (L(Pi) − meanj L(Pj)) · ∇π log p(Pi)
Hypothesis framework: H0 = zero-shot Auto-CoT cannot improve ICL; H1 = it can.

ReLU-2NN Regression (MSE)

Baseline ICL Auto-CoT (ACIL)

ACIL outperforms baseline at every context length. The gap is largest at length 4 (20.9% reduction). At length 33, AUC peaks at 0.607 but collapses to 0.337 at length 40 -- indicating context saturation.

LAMBADA Text (Cross-Entropy Loss)

Baseline ICL Auto-CoT (ACIL)

Auto-CoT loss is roughly half the baseline across all lengths. However, ACIL loss increases with context (1.97 → 2.21) while baseline decreases (4.27 → 4.04) -- the gap narrows, suggesting diminishing returns on text tasks.

Strengths

  • Clean four-stage pipeline with clear mathematical formulation
  • Formal hypothesis testing (H0/H1) -- methodologically commendable
  • Consistent MSE improvement across all context lengths on numerical tasks
  • Loss-based pruning is a sensible quality-control mechanism
  • Detailed Algorithm 1 specification

Limitations

  • Only 22.4M-parameter Transformer; no evidence of scaling
  • No comparison to Zhang et al. (2022) Auto-CoT baseline
  • No ablation study isolating augment / prune / select contributions
  • "Reasoning chain" poorly defined for numerical regression
  • Missing hyperparameter details (ε, learning rate, k, generator G)
  • LAMBADA results degrade with more context -- unexplained

Open Questions

  • Does ACIL transfer to 7B+ models and GSM8K/MATH benchmarks?
  • What does a "reasoning chain" look like for continuous-valued predictions?
  • What is each pipeline stage's individual contribution?
  • Why does Auto-CoT degrade with context on LAMBADA -- fundamental limitation or artifact?
  • How sensitive are results to ε and policy gradient hyperparameters?
