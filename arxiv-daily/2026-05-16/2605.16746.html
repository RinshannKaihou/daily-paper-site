State Contamination in Memory-Augmented LLM Agents

Summarization launders toxic context into classifier-clean memory that still poisons downstream behavior — and standard toxicity monitors cannot see it happening.

Yian Wang · Agam Goyal · Yuen Chen · Hari Sundaram  —  UIUC

When an LLM agent compresses a toxic conversation into a summary like "the discussion became heated, participants expressed strong disagreement", the result passes Detoxify (tox < 0.5) — yet the preserved adversarial framing measurably raises downstream toxicity. The authors call this memory laundering, and demonstrate it across 200 paired counterfactual rollouts.
Key Metric — P95 Tail Toxicity
0.103
0.780 → 0.103

95th-percentile downstream toxicity after full three-pathway mitigation (transcript + memory + DPO). A 7.5x reduction in tail-risk.

Why This Matters

Production systems with persistent memory — ChatGPT memory, Claude projects, agentic frameworks — have an invisible safety gap. Standard toxicity monitors approve laundered summaries as "safe," while adversarial framing silently shifts downstream behavior. The "sanitize before summarizing" principle is directly actionable.

Critical Edge

Absolute SPG values are small (0.015 baseline), raising questions about practical severity. Single-classifier reliance (Detoxify only), Reddit-style simulations, and DPO tested only on Llama-3.1-8B limit generalizability. Whether laundering applies to bias, misinformation, or privacy leakage is unexplored.

01 — Channels

Three Toxic Persistence Channels

The authors decompose toxic propagation in memory-augmented multi-agent systems into three distinct channels, each requiring different interventions.

Channel i — Overt
Transcript Backflow

Toxic messages remain visible in the full conversation context. Drives the bulk of Delta_mu and P95 tail toxicity. Addressable with read-side sanitization.

Delta_mu 0.031 P95 0.175
Channel ii — Covert
Memory Laundering

Summarization strips overt toxicity markers while preserving adversarial framing. Classifier-clean memory states still shift downstream behavior. The hidden channel.

SPG 0.003 (best single)
Channel iii — Model-Level
Parametric Bias

The model matches the toxic register of contaminated context, amplifying outputs. DPO fine-tuning on paired toxic-neutral rollouts reduces average amplification.

P95 0.780 (DPO alone)
02 — Ablation

Mitigation Ablation — What Actually Works

Systematic ablation across single-channel, pairwise, and full-system configurations. No single pathway suffices; all three are necessary and jointly sufficient.

Configuration Delta_mu SPG P95_tox
Baseline (no mitigation) 0.130 0.015 0.780
Transcript rewrite only 0.031 0.005 0.175
Memory rewrite only 0.081 0.003 0.763
DPO only 0.061 0.009 0.780
Transcript + Memory — 0.0012 —
Post-hoc output filtering — — 0.651
Full system (T + M + DPO) 0.011 0.006 0.103

No single channel is enough

Transcript control handles overt toxicity but misses laundered framing. Memory control reduces SPG but leaves the tail. DPO reduces averages but contaminated state persists. Only the combination achieves safe outcomes across all three metrics.

Post-hoc filtering is a false fix

Output filtering leaves P95 at 0.651 — severe toxic generations still occur even when the terminal mean appears low. The contamination must be addressed at the state level, not the output level.

Timing is everything

Sanitizing state before summarization is effective. Cleaning the summary after compression is ineffective against already-laundered influence. This is the core actionable principle.

DPO's marginal role

DPO's contribution is concentrated on overt tail propagation. The Transcript + Memory combination alone reaches SPG 0.0012 — the lowest sub-threshold gap. DPO adds robustness at the extremes but is not the primary mechanism.

03 — Method

Experimental Design

Design
200 paired counterfactual multi-agent rollouts. Focal agent A1 is toxic or neutral; all other factors fixed. Downstream toxicity shift is causally attributable.
Models
GPT-4o-mini (phenomenon experiments), Llama-3.1-8B-Instruct + LoRA (DPO intervention)
Toxicity Scoring
Detoxify classifier, threshold τ = 0.5 for flagging overtly toxic messages
Topologies
Chain, tree, DAG, high-branching. Reddit-style multi-agent discussion threads.
SPG(τ) = E[tox(vt+1) | tox(Mt) < τ, toxic] − E[tox(vt+1) | tox(Mt) < τ, neutral] The downstream toxic-neutral gap, restricted to classifier-clean memory states. Reveals hidden propagation that standard monitors miss.
04 — Assessment

Strengths & Limitations

Genuinely novel failure mode

"Memory laundering" is non-obvious and previously unstudied. The insight that summarization can serve as a laundering step is the paper's core contribution.

Causal experimental design

Paired counterfactual rollouts with fixed seeds, topologies, and agent assignments enable causal attribution, not just correlation. Wilcoxon signed-rank tests are appropriate.

Clean metric formalization

SPG exposes an invisible failure mode that existing metrics miss. Three complementary metrics (Delta_mu, SPG, P95_tox) together cover average, sub-threshold, and tail behavior.

Directly actionable

"Sanitize before summarizing" is a principle that production teams can implement immediately. The three-channel decomposition provides a clear intervention map.

Small absolute effect sizes

SPG values are in the 10-2 range. The practical severity of the hidden propagation effect in deployed systems remains an open question despite statistical significance.

Single classifier reliance

Only Detoxify is used for toxicity scoring. Whether a stronger classifier (e.g., LLM-as-judge) eliminates the laundering effect is unknown.

Simplified simulation setting

Reddit-style discussions omit deployment complexity: longer histories, tool use, RAG, persistent real-world memory architectures, multi-modal inputs.

Limited model coverage

DPO evaluated only on Llama-3.1-8B-Instruct. Generalization to larger models or different architectures is untested. Standard unlearning methods are criticized but not empirically compared.

05 — Open

Open Questions

  • Q1 Does laundering intensify or self-correct across repeated rounds of memory compression?
  • Q2 Does memory laundering generalize beyond toxicity to bias amplification, misinformation, or privacy leakage?
  • Q3 Can SPG serve as a training signal to make summarization models inherently resistant to adversarial framing?
  • Q4 How do findings transfer to RAG-based agents where retrieved documents function like compressed memory?
  • Q5 What is the interaction between memory laundering and other attack vectors (prompt injection, many-shot jailbreaking, AgentPoison)?
⚠
Core Principle: Sanitize before summarizing.

Controls must act before contaminated content is compressed into persistent memory. Post-hoc summary cleaning is ineffective against laundered influence. The window of intervention closes at the moment of summarization.
