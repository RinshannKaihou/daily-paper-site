Triaxial mixed-precision KV-cache quantization matches BF16 accuracy on agentic tasks while achieving 4.5× compression.
TriAxialKV: Toward Extreme Low-Precision KV-Cache Quantization for Agentic Inference Tasks Hanzhang Shen, Haoran Wu, Yiren Zhao, Robert Mullins 2026-05-16
kv-cache-quantization llm-inference agentic-ai mixed-precision efficient-serving
Key Metric
4.5×
KV-cache compression at ≤1.1% accuracy loss vs BF16 across four models on BFCL Memory; up to 1.52× throughput gain on H100.
Why This Matters
Agentic LLM workloads are the dominant emerging use case — KV-cache compression is the key bottleneck for serving them at scale.
Critical Edge
BFCL Memory absolute accuracies are low (~24–25%), making it hard to assess how meaningful the 0.7–1.1% gaps are in absolute terms.

Methodology

TriAxialKV targets agentic workloads — long-context, multimodal, multi-turn tool-calling — where per-token quantization sensitivity spans over an order of magnitude. Three stages:

  • Offline calibration: Capture KV traces from 5% of data across 4–6 layers. Measure per-tag sensitivity via attention-output MSE (not raw KV MSE). Greedy bitwidth allocator assigns INT2/INT4 per tag under a memory budget.
  • Triaxial tagging: Three orthogonal axes — temporal recency (older / recent / current), modality (text / image), semantic role (instruction / user / assistant / reasoning / tool_call / observation / delimiter). Zero-inference tagger runs during request preprocessing via chat-template special tokens.
  • Serving system: Dual-pool (INT2/INT4) paged memory, per-request page tables enabling bitwidth-homogeneous flash-decoding splits, fused Triton dequant-attention decode kernels. Integrated into SGLang v0.5.10.

Strengths

  • Well-motivated problem: agentic workloads have structured, heterogeneous prefills that homogeneous quantization handles poorly.
  • Clean triaxial taxonomy is intuitive; calibration is principled; greedy allocator is simple but effective.
  • Comprehensive evaluation: 4 text models, 3 VLMs, 2 GPU types, throughput + accuracy + ablations.
  • Ablation confirms semantic axis dominates (~3× more impactful than temporal) — consistent with tool-schema sensitivity.
  • Attention-output MSE as calibration metric (not raw KV MSE) is a valuable methodological insight.
  • Practical system integration into SGLang with dual-pool memory and fused kernels.

Limitations

  • BFCL absolute accuracies ~24–25% — unclear how meaningful small gaps are on this benchmark.
  • Modality axis ablation is missing; its isolated contribution is not quantified.
  • Calibration requires 5% of target workload plus multi-stage offline procedure; wall-clock cost not reported.
  • Evaluated on only two benchmarks; generalization to SWE-bench, WebArena, GAIA not shown.
  • INT2 per-channel key quantization with residual routing to INT4 is pragmatic but potentially suboptimal.
  • No explicit code release plan mentioned.

Questions for Further Exploration

  • How does the modality axis contribute in isolation? Is it only beneficial for VLMs or does it improve text-only inference through finer tag granularity?
  • Can calibration transfer across models or must it be repeated per (model, workload) pair? How stable are per-tag sensitivity profiles across task distributions?
  • How does the approach scale to more aggressive targets — e.g., hybrid with token eviction for sub-2-bit average?
  • What is the wall-clock time for the full calibration pipeline (KV capture + sensitivity analysis + budget sweep)?
  • Would finer-grained semantic roles yield further gains, or is the current 7-role taxonomy already capturing the key variance?
  • How does per-tag bitwidth allocation interact with prefix caching and radix-tree KV sharing in SGLang?
