> **一句话总结：** 本文提出"格式税"实为容量依赖效应：用信息匹配的散文对照与四级 schema 复杂度梯度，在 4 个模型、5 个基准上证明，对容量充裕的模型（如 Sonnet）结构化输出无损（MATH-Hard 上 JSON≈CoT≈89%），而逼近能力极限的模型受损严重（Haiku −36.2pp、GPT-4o-mini −28.0pp）；"先推理后格式化"的延迟结构消融能恢复约 80–87% 的损失，甚至 Opus 4.7 在 AIME 上也掉 5.3pp。

> **TL;DR:** The author reframes structured output's "reasoning tax" as a capacity-dependent effect. Using information-matched prose controls and a 4-level schema-complexity gradient across 4 models and 5 benchmarks, JSON is shown to be neutral for spare-capacity models (Sonnet 4.6: 88.7% JSON vs 89.3% CoT on MATH-Hard, p=0.25) but severely degrades capacity-limited models (Claude Haiku 4.5: 52.5% vs 88.7% CoT, −36.2pp, p<0.0001, largely via truncation; GPT-4o-mini: 34.3% vs 62.3% CoT, −28.0pp, p=0.0002, persisting at 0% truncation). A delayed-structure ablation (reason freely, then format) recovers 80% (Haiku) and 87% (GPT-4o-mini) of the lost accuracy; the format penalty even reaches the frontier — Opus 4.7 drops 96.2%→91.0% on AIME (−5.3pp), with pooled AIME+GPQA reaching p=0.024.

## 核心贡献

- 提出并验证**结构化输出的"容量依赖"假说**：同一 JSON schema 对有余量的模型无害、对逼近极限的模型有害，用统一的逐项 logistic 回归确认 is_json × capacity 交互显著（聚类标准误下 p=3.8×10⁻³）。
- 设计**信息匹配散文对照（Detailed Prose, DP）**：在不施加任何格式约束的情况下匹配结构化 prompt 的信息量与长度，从而把"prompt 复杂度"效应（DP−CoT）与"格式专有"效应（JSON−DP）分离，排除 prompt 长度的混淆。
- 证明**前沿闭源模型并非免疫**：在 AIME 竞赛数学上 Opus 4.7 从 96.2% 降到 91.0%（−5.3pp），质疑了"现代闭源模型几乎没有格式税"的说法。
- 给出**机理证据与可操作建议**：81 例失败分析显示错误类型随能力变化（Haiku 以"推理遗漏"为主，GPT-4o 以"算术错误"为主）；延迟结构（think first, format later）恢复大部分损失；并提出"推理-格式分离原则"与延迟序列化等接口设计建议。

## Key Contributions

- Reframes the structured-output "format tax" as a **capacity-dependent effect**: the same JSON schema is neutral for models with spare capacity (Sonnet 4.6 ~89% across all conditions under 3-run replication) but severely degrades capacity-limited models (Haiku −36.2pp, GPT-4o-mini −28.0pp on MATH-Hard). A unified item-level logistic regression confirms the is_json × capability interaction (p=3.8×10⁻³ under clustered standard errors, n=16 cells).
- Introduces an **information-matched prose (DP) control** that matches the structured prompt's informational content and length without imposing syntax, isolating format-specific effects (JSON−DP: −0.3pp Sonnet, −28.2pp Haiku, −19.3pp GPT-4o-mini) from prompt-length confounds (DP−CoT: −0.3pp Sonnet, −8.0pp Haiku).
- Shows **frontier closed-weight models are not immune**: on AIME, Opus 4.7 drops 96.2%→91.0% (−5.3pp); an exploratory pooled analysis across AIME and GPQA Diamond reaches p=0.024, qualifying prior claims that closed models show little format tax.
- Provides **mechanistic and interventional evidence**: a delayed-structure ablation recovers 80% (Haiku) / 87% (GPT-4o-mini) of lost accuracy; a token-budget ablation shows Haiku's tax is mostly truncation-driven (52.5%→84.0% at 8192 tokens) while GPT-4o-mini's penalty persists at 0% truncation (pure capacity competition); an 81-case failure analysis shows error profiles shift with capability.

## Methodology

- **Models (4 main + 1 frontier).** Two provider families × two capability tiers: Claude Sonnet 4.6 (`claude-sonnet-4-6`) and GPT-4o (strong tier); Claude Haiku 4.5 (`claude-haiku-4-5`) and GPT-4o-mini (weak tier). Capacity is operationally defined as 3-run mean CoT accuracy on MATH-Hard (Sonnet 89.3%, Haiku 88.7%, GPT-4o 56.2%, GPT-4o-mini 62.3%). Opus 4.7 is added as an exploratory frontier probe on AIME and GPQA Diamond.
- **Benchmarks (5).** GSM8K (100 problems sampled seed=42; all models ≥90% → "comfort zone"); MATH-Hard Level 5 (100 problems with clean numerical answers; baselines 56.2–89.3% → near capability boundary); BBH Tracking Shuffled Objects (100, most ≥98%, GPT-4o-mini 90.0%); BBH Logical Deduction (100 A–G, baselines 79.0–97.0%); MMLU-Pro (100 each across law/physics/philosophy = 300; baselines 51.3–83.3%).
- **Output conditions (4).** CoT (minimal 82-char prompt), JSON (1,774-char schema with problem_understanding/known_values/approach/step_by_step/verification/final_answer), XML (1,625-char equivalent), and the key **Detailed Prose** control (1,280-char, same analytical dimensions, no syntax). This DP control lets the total effect be decomposed into prompt-length (DP−CoT) and format-specific (JSON−DP) components.
- **Delayed-structure ablation.** Two phases with an explicit separator: Phase 1 "THINK" reasons freely in natural language, Phase 2 "FORMAT" re-serializes into a compact JSON (`{"problem_understanding", "step_by_step", "final_answer"}`), 522 chars total — decoupling reasoning from format compliance.
- **Schema-complexity gradient (4 levels, MATH-Hard).** Minimal (73-char answer-only `{"answer": <number>}`), Light (117-char, one reasoning field), Medium (177-char structured steps), Heavy (1,774-char full nested schema). A dose-response pattern tests whether degradation grows monotonically with schema burden.
- **Production modes (GPT-4o-mini MATH-Hard).** API JSON mode (`response_format=json_object`), instruction-prompted heavy JSON, and function calling with `tool_choice=forced` vs `auto`.
- **Evaluation & statistics.** Exact-match on extracted numerical answer; JSON parsed for `final_answer` with regex fallback to last number. Primary experiments at temperature 0 (deterministic); two stochastic reruns (temperature 0.3) for MATH-Hard, delayed-structure, and schema gradient, with per-temperature estimates differing only 1–3pp. Pairwise comparisons use McNemar's exact binomial test on discordant pairs; token comparisons use Wilcoxon signed-rank. The format×capability interaction is fit as `correct ~ is_json × capacity + benchmark` with capacity cross-validated (each model's mean CoT accuracy on the *other* hard benchmarks). 0% parse failures on successfully generated responses; 14 model-item pairs (<0.6%) excluded for non-evaluable API responses. Acknowledges GPT-5.5 was used only for English editing/proofreading.

## Results & Highlights

- **Format effect reverses between easy and hard tasks.** On GSM8K (easy), all models score 87–97% and no format produces a significant change (max effect 8.0pp, GPT-4o-mini CoT 95.0% vs XML 87.0%; all p>0.05). On MATH-Hard (hard, 3-run means): Sonnet JSON 88.7±4.0% vs CoT 89.3±1.7% (p=0.25, neutral); Haiku 52.5±2.5% vs 88.7±0.7% CoT (−36.2pp, p<0.0001); GPT-4o-mini 34.3±1.7% vs 62.3±4.0% CoT (−28.0pp, p=0.0002).
- **Format-specific vs prompt-length decomposition (MATH-Hard, JSON−DP).** Haiku: DP 80.7% → JSON 52.5%, format-specific Δ_struct=−28.2pp (p<0.0001); GPT-4o-mini: DP 53.7% → JSON 34.3%, Δ_struct=−19.3pp (p=0.011); Sonnet Δ_struct=−0.3pp (neutral). The prompt-length effect (DP−CoT) is much smaller (Haiku −8.0pp, Sonnet −0.3pp), so prompt length alone cannot explain the degradation.
- **Cross-format consistency (MATH-Hard).** JSON and XML harm weak models similarly: Haiku −36.2pp (JSON) / −30.0pp (XML); GPT-4o −12.2pp / −13.2pp; GPT-4o-mini −28.0pp / −28.0pp (all p<0.01 vs CoT).
- **Delayed-structure ablation recovers most lost accuracy.** On MATH-Hard, Haiku rises from 54.3% (simultaneous) to 82.0% (delayed) — 79.5% recovery; GPT-4o-mini from 37.4% to 56.7% — 86.9% recovery; GPT-4o from 42.0% to 58.3% (>100%, exceeding its 53.0% CoT baseline, suggesting scaffolding benefit). 3-run mean recovery: 80% Haiku, 87% GPT-4o-mini.
- **Token-budget ablation (Haiku, MATH-Hard).** Increasing max_tokens from 2048→4096→8192: JSON accuracy 52.5%→81.7%→84.0% with truncation 49%→9%→2%; residual gap vs CoT (88.7%) narrows to −4.7pp. GPT-4o-mini shows no recovery (34%→36% at 8192, 0% truncation at both budgets) — a distinct non-truncation capacity cost.
- **Schema-complexity dose-response (MATH-Hard, 3-run means).** Among reasoning-bearing schemas, accuracy falls monotonically with schema burden: Haiku 90.3% (light)→86.0% (medium)→55.3% (heavy); GPT-4o-mini 58.3%→47.1%→36.3%. The minimal answer-only schema behaves differently (suppresses reasoning: GPT-4o collapses to 17%).
- **Production structured-output modes (GPT-4o-mini MATH-Hard).** CoT 60% → API JSON mode 51% (nonsignificant −9pp, p=0.12, 436 tokens) → instruction JSON 34% → forced function calling 10% (median 328 tokens, no visible reasoning). The forced-FC collapse (34%→10%) is a production-interface bottleneck.
- **Frontier probe: Opus 4.7.** AIME (133 problems): CoT 96.2% → JSON 91.0% (−5.3pp). GPQA Diamond (198): −4.0pp. Individually underpowered, but pooled AIME+GPQA reaches p=0.024 (exploratory); on HLE (floor control, 28.8% CoT) the effect vanishes (−0.2pp, p=1.0). Enabling extended thinking on AIME lifts Opus JSON from 91.0%→94.7%, shrinking the tax to −2.3pp (n.s.).
- **Interaction test.** is_json × capacity (cross-validated): item-level z=4.62 (p=3.9×10⁻⁶), clustered-by-cell p=3.8×10⁻³ (n=16), cell-level Spearman r_s=0.50 (p=0.047); adding 3 frontier cells n=19 r_s=0.51 (p=0.026); on GSM8K (easy) not significant (p=0.34).

## Authors & Credibility

- Single author, Hengxin Fan, affiliated with Tianjin Normal University. arXiv preprint (2606.09410v1, cs.AI, 8 Jun 2026); no peer-reviewed venue stated and no code/data link given in the PDF. The author discloses GPT-5.5 was used strictly for English language editing/proofreading/readability, with the author retaining responsibility for scientific content.

## Critical Assessment

- The single-author, closed-API-only setup is a real reproducibility hazard: the paper itself documents that a later **replication audit of Sonnet on the heavy-schema condition produced 60–63% accuracy vs the 93% retained in the tables**, which the author attributes to "API serving-layer drift" rather than a format effect — yet this is exactly the kind of backend instability that makes the absolute numbers (and the within-batch-only CIs) hard to trust over time, and means the headline Sonnet-immunity result rests on a non-reproducible snapshot.
- The **schema-complexity gradient is confounded**: it varies prompt length, field count, and nesting depth simultaneously (an explicitly acknowledged limitation), so the clean "dose-response" narrative cannot separate which dimension drives the Haiku/GPT-4o-mini collapse; the logistic interaction uses cross-validated capacity rather than the schema burden itself.
- The frontier "immunity-refuted" claim rests on a **post-hoc pooled analysis** (AIME+GPQA p=0.024) where neither benchmark is individually significant (AIME p=0.092, GPQA p=0.169), and Opus's cross-validated capacity is estimated from a single held-out benchmark — so the −5.3pp AIME number is suggestive, not confirmatory, as the paper itself concedes.
- The detailed-prose control for **GPT-4o is omitted** (CI data unavailable), leaving its format-specific Δ_struct blank and the format-vs-length decomposition incomplete for one of the four main models.

## Strengths

- Genuinely clean experimental design: information-matched prose control, four-level schema gradient, delayed-structure intervention, token-budget ablation, and cross-format (JSON/XML) checks together triangulate the mechanism far better than single-condition studies.
- Strong statistical hygiene: McNemar paired tests on the same item set, clustered standard errors, cross-validated capacity proxy to avoid circularity, explicit floor/ceiling controls (HLE, GSM8K), and honest reporting of a non-replicating Sonnet heavy cell rather than hiding it.
- Actionable, well-grounded takeaway ("think first, format later"; prefer lightweight schemas; avoid forced function calling for hard reasoning) backed by the reasoning-freedom spectrum table that ranks all tested interfaces consistently across models.

## Limitations

- Acknowledged: schema-complexity gradient confounds length/fields/nesting; evaluation is math-heavy; cross-benchmark and frontier cells are single-run (CIs only for MATH-Hard, delayed-structure, schema gradient); Opus CV instability (two benchmarks only); no grammar-constrained decoding tested; closed-source models only, subject to serving-layer drift; no public benchmark places Opus in the 50–70% boundary zone where the theory predicts the largest effects.

## Relevance

主要映射到"**LLM Inference Evaluation（推理评估）**"——本文系统评测了 4 个模型 × 5 个基准在结构化输出约束下的推理质量，并分离 prompt 长度与格式专有效应；其次与"**LLM Training Stability & Observability**"相关，因其揭示同一接口在不同能力余量下行为迥异，提示部署期需做能力感知的观测与告警。

## Questions for Further Exploration

- Would the is_json × capacity interaction (clustered p=3.8×10⁻³ on n=16 cells) survive if the two provider families were disaggregated, given that Sonnet vs Haiku and GPT-4o vs GPT-4o-mini are paired within families and the capacity proxy is cross-validated only within-family?
- The author cites Lee, D'Antoni & Berg-Kirkpatrick (2026, "The Format Tax") as concluding closed-weight models show little tax — can their dataset be re-analyzed with the information-matched prose control introduced here to test whether their null result is an artifact of insufficiently challenging tasks?
- Since the Sonnet heavy-schema replication dropped to 60–63% (attributed to API serving-layer drift), would repeating the full 3-run protocol on a frozen API snapshot reproduce the Sonnet-immunity claim, or does frontier-model serving nondeterminism itself undermine the within-batch comparison methodology?
- Forced function calling collapsed GPT-4o-mini to 10% — does adding a separate reasoning channel (e.g., extended thinking, which narrowed Opus's tax from −5.3pp to −2.3pp on AIME) systematically recover performance across the reasoning-freedom spectrum the paper proposes?
