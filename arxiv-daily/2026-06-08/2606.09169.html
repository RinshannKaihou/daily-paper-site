一句话总结： IMUG-Bench 是一个面向统一多模态模型（UMM）多轮交错图文对话的评测基准（3,113 样本、12,034 交互轮、19 域 97 任务，含 Static/Dynamic-MCQ 与 Image-Generation 三种题型），系统评测 8 个开源/闭源 UMM 后发现生成侧存在显著的"暴露偏差"（image 分随轮数单调下降而 text 分不降）；并提出三种测试期扩展策略（CoT、Self-Verification、Best-of-N），其中 CoT 对 BAGEL 的 image 平均分提升最大（54.1 → 65.9）。
TL;DR: The authors release IMUG-Bench (3,113 samples, 12,034 turns, 19 domains, 97 tasks across Static Spatial / Temporal Causal / Hybrid classes) and evaluate 8 UMMs (GPT-5 Image Mini, Nano Banana, BAGEL 15B, BLIP3-o 4B/8B, OmniGen2 7B, Ovis-U1 3B, UNIWORLD-V1 20B). GPT-5 Image Mini leads at Avg 82.3 (Und. 81.6/Gen. 88.0 on Static Spatial); the best open-source model BLIP3-o 8B reaches Avg 65.0. Image-question scores decay across turns (e.g., GPT-5 Image Mini image: T1 94.3 → T6 54.8; BAGEL image: T1 68.5 → T6 24.7) while text scores do not, evidencing exposure bias. For BAGEL, Chain-of-Thought lifts average image score from baseline 54.1 to 65.9 (+11.8), beating Self-Verification (max 55.2) and Best-of-N (max 58.6 at N=5).

核心贡献

  • 构建 IMUG-Bench：首个同时具备"多轮交互 + 模态统一 + 动态理解 + 暴露偏差度量 + 测试期扩展"五大属性的统一多模态基准（Table 1 显示此前 8 个基准均至少缺两项），含 3,113 样本/12,034 轮/19 域/97 任务。
  • 设计三种题型分离理解与生成：Static-MCQ（固定答案的多项选择）、Dynamic-MCQ（正确答案依赖模型在前序轮次生成图像的多项选择，用于度量模型对"自身生成图像"的理解）、Image-Generation（按指令生成/编辑图像）。
  • 通过逐轮评分系统揭示 UMM 的"暴露偏差"：image 问题得分随轮数单调下降，而 text 问题无此趋势，说明多轮生成中早期错误被写入历史并反复暴露，导致误差累积与分布漂移。
  • 提出并验证三种测试期扩展策略（CoT、Self-Verification、Best-of-N）能提升生成质量并缓解暴露偏差，并给出各自的消融（Self-Verification 重试次数、Best-of-N 候选数）。

Key Contributions

  • IMUG-Bench construction: the first benchmark combining multi-turn interaction + unified understanding/generation + dynamic understanding + exposure-bias measurement + test-time-scaling evaluation (Table 1 shows each of 8 prior benchmarks — CoMM, UniEval, MMIE, OpenING, ISG-Bench, MME-U, Uni-MMMU, WEAVE Bench — misses at least two of these). 3,113 samples / 12,034 turns / 19 domains / 97 tasks, organized into Static Spatial, Temporal Causal, and Hybrid classes.
  • Three question types that decouple understanding from generation: Static-MCQ (fixed-answer), Dynamic-MCQ (answer depends on the model's own previously generated image — directly measures self-generated-content understanding), Image-Generation (instruction-following + cross-turn consistency).
  • Exposure-bias finding via per-turn scoring: image-question scores decay monotonically with turn count while text-question scores do not, because multi-turn generation must satisfy both current instruction and cross-turn consistency, so early generation errors get written into history and repeatedly re-exposed (error accumulation + distribution drift).
  • Test-time-scaling mitigation: Chain-of-Thought, Self-Verification (closed-loop understanding-checks-generation with bounded retries), and Best-of-N Sampling (generate N candidates, model selects best) all improve generation and slow decay; full ablations on retry count and N.

Methodology

Benchmark construction pipeline (Fig. 3). Manual template design (question/answer/evaluation-point templates with keyword blanks and inter-turn dependency indices) → LLM/VLM template filling (text-only via LLM, image-input via VLM; Dynamic-MCQ ground-truth determined by a VLM after the model answers, using prompt image + prompt text + options) → two-person human verification at every stage. Image sources: open-source platforms, AI-generated, author-collected.

Three classes. Static Spatial (recognize/generate target-element attributes, ownership, spatial relations); Temporal Causal (reason about implicit natural laws / common-sense causal rules); Hybrid (both, reconstructed over daily scenarios). Turn counts 2–6, average 3.87 turns/sample; understanding and generation turns roughly balanced.

Scoring. Understanding (Static/Dynamic-MCQ): regex rule scoring S_MCQ = w_fmt·(n_corr − n_incorr)·u, where u = 1/(#correct options), and a formatting weight w_fmt ∈ {1, 0.75, 0.5, 0} depending on output format adherence. Generation (Image-Generation): VLM-as-a-Judge with dynamic evaluation-points filling; each image decomposed into N evaluation-points (image-update requirements + image-consistency checks), the VLM judge (Qwen3-VL-235B-A22B-Thinking) assigns integer 0–5 per point with previous images as references for consistency, and S_img = (Σ s_i)/(N·5). Reliability validated against two-person human ratings (Appendix D).

Models evaluated (8). Closed: GPT-5 Image Mini (GPT-5 Mini + GPT Image 1 Mini), Nano Banana (Gemini 2.5 Flash Image). Open: BAGEL (15B), BLIP3-o 4B/8B (10B/14B), OmniGen2 (7B), Ovis-U1 (3B), UNIWORLD-V1 (20B). History construction differs by model capability: BAGEL takes interleaved text-image in original order; BLIP3-o/OmniGen2/Ovis-U1/UNIWORLD-V1 take serialized image lists with in-text placeholders (Ovis-U1 horizontally concatenates images since it lacks multi-image editing); closed-source uses official multi-turn APIs with history memory.

Test-time scaling (§3.5, applied to BAGEL). Chain-of-Thought: before image generation the model summarizes the request and lists an explicit generation plan (reasoning not appended to history). Self-Verification: model generates R_t, self-assesses instruction + style/detail consistency, redraws with feedback if rejected (max retries 0–3 swept); only final image enters history. Best-of-N: generate N candidates under same history, model selects best by fulfillment (N = 2–5 swept); only selected image enters history.

Results & Highlights

  • Closed-source models lead clearly (Table 2, Avg). GPT-5 Image Mini 82.3 (Static Und. 81.6 / Gen. 88.0; Temporal 74.8/83.1; Hybrid 80.9/79.3), Nano Banana 79.5; best open-source BLIP3-o 8B only 65.0, with BAGEL 51.9, BLIP3-o 4B 59.3, OmniGen2 55.1, Ovis-U1 50.0, UNIWORLD-V1 46.6.
  • Pronounced exposure bias: image scores decay across turns, text scores do not (Table 3). GPT-5 Image Mini image score: T1 94.3 → T2 84.9 → T3 76.0 → T4 73.0 → T5 77.5 → T6 54.8 (text: T1 73.8 → T6 77.2, no decay). BAGEL image: T1 68.5 → T6 24.7 (a −43.8 pp collapse), while its text stays 47.8 → 44.8. BLIP3-o 8B image: T1 80.1 → T6 51.7. This monotonic image-side decay is observed for nearly all models.
  • Dynamic-MCQ is much harder than Static-MCQ. The paper reports (Appendix C.3, Table 6) that all models score substantially lower on Dynamic-MCQ than Static-MCQ — models struggle to understand the specific content of their own previously generated images, so accumulated history errors directly hurt understanding.
  • Shared open-source weakness on four domains. All open-source UMMs score low on Stack Simulation, Geometric Coloring, Mirror Reasoning, and Paper Crafting — fine-grained spatial/geometric multimodal tasks under multi-turn interleaving.
  • Chain-of-Thought gives the largest test-time-scaling gain (Table 4, BAGEL image scores). Baseline avg 54.1 → CoT 65.9 (+11.8); Static 57.9→72.0, Temporal 49.3→60.1, Hybrid 52.8→60.4. Self-Verification peaks at avg 55.2 (Max=1) and degrades beyond 2 retries (Max=3 → 53.7, below the helpful range). Best-of-N rises monotonically with N but saturates: N=2 54.8, N=3 57.4, N=4 58.2, N=5 58.6 (Static 62.0, Temporal 52.7, Hybrid 60.1).
  • Self-Verification can backfire if over-retried. Max=2 gives 54.6 and Max=3 gives 53.7 (occasionally underperforming the 54.1 baseline), showing verification has a narrow useful range.
  • Case study (Fig. 4) shows three failure modes: weak instruction-following/formatting in open-source models (BAGEL outputs "A. Black" with wrong format), error accumulation across generation turns (even GPT-5 Image Mini fails to preserve edits in the next turn), and hallucination (both models report a shoe color not present in the generated image on the Dynamic-MCQ turn).

Critical Assessment

  • The headline "exposure bias" finding rests on per-turn image scores without an error-attribution control. Table 3 shows image scores decay across turns, but the paper attributes this to early generation errors propagating via history without isolating the mechanism — e.g., it does not run a control where the gold/reference image (instead of the model's own) is fed forward to separate "error accumulation" from "inherent difficulty of later turns." So "exposure bias" is inferred from a trend rather than causally demonstrated.
  • Scoring depends entirely on a VLM judge (Qwen3-VL-235B-A22B-Thinking) whose own biases are not neutralized. Image-generation scores are VLM-as-a-Judge over personalized evaluation points; although human correlation is reported in Appendix D, the main results inherit the judge model's preferences, and cross-model fairness (e.g., whether the judge favors outputs stylistically similar to its own family) is not analyzed.
  • History-construction heterogeneity undermines apples-to-apples model comparison. BAGEL gets native interleaved text-image history, while BLIP3-o/OmniGen2/Ovis-U1/UNIWORLD-V1 receive serialized image lists with placeholders, and Ovis-U1 additionally gets horizontally concatenated images with a special prompt. These differing input formats are a confound: a model's low score may reflect history-format mismatch rather than intrinsic multi-turn ability.
  • Test-time-scaling experiments are run only on BAGEL. CoT/Self-Verification/Best-of-N are evaluated on a single open-source model (BAGEL 15B), so the mitigation claims — including "CoT gives the largest gain" — are not shown to generalize across model families or sizes; the closed-source leaders and BLIP3-o 8B are not tested under these strategies.
  • The Dynamic-MCQ "ground truth" is itself model-dependent. Because the VLM judge determines the correct option after seeing the model's generated image, the same question can have different ground truths for different models, making cross-model Dynamic-MCQ scores not strictly comparable.

Strengths

  • Genuinely fills a gap: Table 1 shows IMUG-Bench is the first benchmark to combine multi-turn interaction, unified understanding+generation, dynamic understanding, exposure-bias measurement, and test-time-scaling evaluation — each prior benchmark misses ≥2 of these.
  • The Dynamic-MCQ design is conceptually valuable: it directly probes a capability (understanding your own generated images) that single-turn benchmarks cannot, and the Static-vs-Dynamic gap quantifies accumulated-error sensitivity.
  • The per-turn score decomposition (Table 3) makes the exposure-bias trend visible and comparable across 8 models, not just asserted.
  • Test-time-scaling ablations are thorough on BAGEL: full sweeps over Self-Verification retries (0–3) and Best-of-N (1–5), including the finding that Self-Verification backfires when over-applied — a practical, non-obvious result.

Limitations

  • Exposure bias is shown as a trend, not causally isolated (no gold-image-forward control).
  • All image scores rely on a single VLM judge (Qwen3-VL-235B-A22B-Thinking); cross-model judge bias unanalyzed.
  • Inconsistent history construction across models (native interleaved vs serialized lists vs concatenated images) is a confound.
  • Test-time scaling validated on only one model (BAGEL); generalization untested.
  • Dynamic-MCQ ground truth is model-dependent, complicating cross-model comparison.
  • Benchmark is English/image-only (no video, no multilingual); 3,113 samples is moderate compared to MMIE's 20,103 (though IMUG-Bench's multi-turn dynamic design is richer per sample).

Relevance

这主要映射到用户的「LLM Inference Evaluation」兴趣区——IMUG-Bench 本质是一个针对统一多模态模型在多轮交错对话中推理/生成能力的评测基准，定义了新题型（Dynamic-MCQ）与新现象（生成侧 exposure bias）并跨 8 个模型做系统评测。其"测试期扩展策略（CoT、Self-Verification、Best-of-N）改善推理时生成质量"的实验也与该兴趣区相关，属于推理时评测与改进。若放宽映射，"暴露偏差"作为多轮推理中的失效模式也与「LLM Hidden States & Interpretability」间接相关，但论文本身只做行为层面度量，未涉及隐藏状态分析。

Questions for Further Exploration

  • The exposure-bias claim is a per-turn trend (Table 3); would a gold-image-forward control (feeding the reference image instead of the model's own at each turn) isolate "error accumulation" from "later turns are just harder," since the paper currently infers the mechanism without that control?
  • Scoring uses Qwen3-VL-235B-A22B-Thinking as judge for all 8 models — would using a different judge family (or human ratings on the main-table subset, not just the Appendix D reliability check) change the closed-vs-open ranking, given the known same-family preference of VLM judges?
  • Test-time scaling is validated only on BAGEL; the paper cites Wei et al. (2022) for Chain-of-Thought — does CoT's large +11.8 gain transfer to the closed-source leaders (GPT-5 Image Mini, Nano Banana) and to BLIP3-o 8B, or does it specifically compensate for BAGEL's weaker instruction-following?
  • The paper cites WEAVE Bench (Chow et al. 2025) and OpenING (Zhou et al. 2025) as the closest multi-turn/interleaved benchmarks — on overlapping samples, does IMUG-Bench's Dynamic-MCQ + exposure-bias decomposition surface failure modes that those static-answer benchmarks miss, quantitatively?
  • Self-Verification backfires beyond 2 retries (Table 4: Max=3 → 53.7 < baseline 54.1) — is this because the model's own understanding is too weak to give reliable verification feedback (a self-consistency limit), and could a stronger external verifier avoid the regression?
