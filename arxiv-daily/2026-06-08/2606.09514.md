> **一句话总结：** BUDDY 提出预算驱动的动态深度路由框架，用轻量 Decision Module 对中间层打分并确定性执行 top-k 满足用户预算，通过复用首层 KV cache 作为全局上下文实现解码时重新路由，单一训练模型即可服务多种预算，在 Llama3-8B 上 50% 稀疏度时仍保留约 71.3% 原始精度。

> **TL;DR:** BUDDY, a budget-driven dynamic depth routing framework for LLM inference, uses a lightweight two-layer MLP Decision Module (≈0.005% of backbone FLOPs) to score intermediate layers and deterministically execute top-k, reusing the first-layer KV cache for decode-time rerouting; on Llama3-8B across 8 commonsense benchmarks it retains ~99.9%/90.8%/80.7%/71.3% of dense accuracy at 12.5%/25%/37.5%/50% sparsity, beating static (Shortened LLaMA/ShortGPT/SLEB) and dynamic (PuDDing/FiRST) baselines at matched sparsity, with up to ×1.91 prefill and ×1.64 decode speedup.

## 核心贡献

- 预算驱动的决策模块：轻量两层 MLP 根据当前上下文对中间层打分，将预算 b∈(0,1] 转为 k=round(b·L)−2 并硬 Top-k 选择执行层，提供对执行计算量的确定性严格控制
- 解码时重新路由：复用首层 KV cache 作为低成本全局上下文源，与最新 token 池化后输入决策模块，使执行路径随生成上下文演进而更新
- 可选 Budget Predictor：用户未指定预算时，离散策略网络用 GRPO 训练，按输入估计最优计算级别，平衡质量与效率
- 单一训练模型服务多种预算：训练时对每个 mini-batch 随机采样预算（含零稀疏度），避免为每个稀疏度训练/部署单独模型

## Key Contributions

- Budget-aware Decision Module: a lightweight scorer (Linear–SiLU–Linear) that ranks middle layers conditioned on context and applies hard Top-k to satisfy a user-specified budget, giving deterministic control over executed depth
- KV-aware decode-time rerouting: reuses the first layer's KV cache (concatenated with the newest token) as a low-overhead global-context signal so the execution path updates as decoding proceeds — addressing the observation that layer importance shifts with growing context
- Optional Budget Predictor: a discrete policy trained with GRPO that infers an input-dependent compute level when no explicit budget is given
- Single-model multi-budget serving: random budget sampling during SFT (including zero-sparsity) yields one trained model serving all sparsity levels, unlike baselines that train separate models per sparsity

## Methodology

**Motivation.** Two observations on Llama2-7B/WikiText-2: (O1) layer importance is input-dependent — e.g., layer 10's remove-order ranges from 0th to 26th across inputs; (O2) layer importance shifts during decoding — e.g., layer 14's rank fluctuates as context grows from 256→512 tokens. Static pruning (Shortened LLaMA, ShortGPT, SLEB) cannot adapt; existing dynamic methods (early-exit, token-wise, PuDDing, FiRST) either lack strict budget control or fix the path at prefill.

**Layer skipping formulation.** Binary mask M∈{0,1}^L gates residual updates: H_ℓ = F_ℓ(H_{ℓ−1})+H_{ℓ−1} if M_ℓ=1, else H_{ℓ−1} (identity). First and last blocks always executed; L_mid=L−2 candidate middle layers.

**Decision Module.** Two-layer MLP scores middle layers from a pooled context representation. Scores fused with a normalized offline prior p̃ via s′=s+ω·p̃ (ω=0.1 default). Prior p̃ is one indicator (∆PPL, Taylor, or cosine) normalized via log(1+x) → robust z-score (median/IQR) → rank-ECDF to (0,1). Budget → k=round(b·L)−2 → Top-k Softmax → binary mask. STE for gradient flow (hard forward, continuous backward).

**KV-aware planner.** At each decode step, G_t ∈ {K_t, V_t, K_t+V_t} (Value states default) from the first layer's KV cache concatenated with the newest token provides global context to the Decision Module. Ablation: Value states best at 12.5–37.5%; Key states slightly better at 50%.

**Budget Predictor.** Categorical policy π_θ(k|G_t) over actions {1,…,L_mid}, trained with GRPO using reward r(k)=−λ_perf·CE − λ_cost·b(k).

**Training.** LoRA (r=8, all linear modules) joint SFT on Alpaca; AdamW, lr 1e-4, batch 8, 2 epochs, BFloat16, 100 warm-up steps. Random budget sampling per mini-batch including zero-sparsity. Cost: ~8–25h fine-tuning + <1.5h GRPO (Table 9).

**Setup.** Backbones: Llama3-8B (32 layers), Qwen2.5-7B-Instruct (28 layers), plus Llama2-7B/Llama1-13B in appendix. Benchmarks: 8 commonsense reasoning tasks (BoolQ, PIQA, HellaSwag, WinoGrande, ARC-E/C, OpenBookQA, SIQA) via lm-eval-harness. Sparsity: 4/8/12/16 blocks removed (12.5–50% Llama; 14.3–57.1% Qwen).

## Results & Highlights

- **Llama3-8B avg accuracy:** BUDDY retains ~**99.9%/90.8%/80.7%/71.3%** of dense accuracy (67.44) at 12.5%/25%/37.5%/50% sparsity — at 12.5% it averages **67.40** (best), beating Shortened LLaMA 63.76, ShortGPT 66.65, SLEB 66.08, PuDDing 61.56, FiRST 44.13.
- **Qwen2.5-7B-Instruct:** at 28.6% sparsity BUDDY averages **57.22** (best), vs Shortened LLaMA 56.63, ShortGPT 56.14, SLEB 56.72, PuDDing 51.04, FiRST 40.32; BUDDY is best or competitive across all four sparsity levels with a single trained model.
- **Decode-time rerouting helps:** on SAMSum, Recompute (path updated each step) beats Reuse (fixed prefill path) on ROUGE-1/ROUGE-L across all sparsity levels (e.g., 0.204 vs 0.190 reuse at 16 blocks removed) without increasing the executed-layer budget.
- **Speedup (Llama, Alpaca):** up to **×1.91 prefill** and **×1.64 decode** at 50% sparsity; at 12.5% sparsity gains are near-zero (×1.01–1.14) because routing + gather/scatter overhead offsets savings. Decision Module overhead is ~0.00285–0.00498% of backbone FLOPs (Appendix C.2).
- **Prior fusion ablation:** Taylor prior best across all sparsity (62.63/58.44/53.54/46.90% avg); ω=0.1 optimal, with performance degrading as ω increases (Table 7).

## Critical Assessment

The most fundamental limitation is stated by the authors themselves: because BUDDY updates the execution path at runtime, **all Transformer blocks remain resident in GPU memory — VRAM usage does not decrease** — so the method reduces latency/FLOPs but not the memory footprint that often dominates LLM serving cost. Evaluation is restricted to commonsense-reasoning multiple-choice tasks (8 benchmarks via lm-eval-harness) plus a SAMSum summarization ROUGE comparison; there is no evaluation on generation-quality benchmarks (MT-Bench, AlpacaEval), long-context tasks, or coding/math, so the accuracy-retention claims are narrow. The dense baseline at 12.5% sparsity is suspiciously close (67.40 vs 67.44, and on some sub-tasks BUDDY even exceeds dense, e.g. BoolQ 83.24 vs 80.98) which suggests either the SFT+LoRA procedure itself shifts accuracy or measurement noise — the paper does not reconcile why pruning sometimes *improves* accuracy. FiRST is a noticeably weak baseline (44.13 avg at 12.5% vs ~64 for others), inflating BUDDY's relative standing. The Decision Module adds real system overhead under batching: different sequences select different paths, inducing KV-cache misses at skipped layers, handled only by zero-filled vectors — a crude approximation whose accuracy cost at large batch sizes is not quantified.

## Strengths

- Budget is a first-class constraint with deterministic Top-k control — directly serves heterogeneous-budget serving, a real production need
- Decode-time rerouting via first-layer KV cache is a cheap, principled fix for the documented prefill↔decode importance drift
- Single trained model serves all budgets (random budget sampling incl. zero-sparsity), reducing deployment complexity vs per-sparsity checkpoints
- Decision Module FLOP overhead is negligible (~0.005% of backbone) with a concrete latency breakdown (Table 8: 5.67% of runtime)

## Limitations

- No VRAM reduction — all blocks remain resident; the authors explicitly flag this as the main limitation
- Evaluation limited to 8 commonsense MCQ benchmarks + one summarization ROUGE probe; no MT-Bench/AlpacaEval/long-context/math/code
- Pruning sometimes "improves" accuracy over dense (BoolQ 83.24 vs 80.98), unexplained — likely SFT/LoRA confound
- FiRST baseline anomalously weak, inflating relative wins
- Batched inference with heterogeneous paths uses zero-filled vectors for skipped-layer KV, a crude approximation whose cost at scale is unquantified
- No statistical significance / variance reporting

## Relevance

- 相关性评分：5/5。直接命中 **#2 LLM Inference Evaluation**（动态深度剪枝/路由的推理加速，在 8 个 commonsense 基准与 SAMSum 上评测，含 prefill/decode 吞吐与 FLOP 开销分析）。强相关 **#1 LLM Training Stability & Observability**：核心动机正是"层重要性随输入和解码上下文动态变化"的可观测性发现（Observation 1/2），且训练采用随机预算采样+STE+零稀疏度正则以保证稳定性。与 **#4 Computation Reliability & Precision Errors** 间接相关：跳过层用零填充 KV 的近似、STE 的前向/反向不一致，都属于近似引入的潜在精度/可靠性问题。

## Questions for Further Exploration

- Since all Transformer blocks remain resident in GPU memory (the authors' own stated limitation), does BUDDY's latency win survive under memory-bound serving where VRAM — not FLOPs — is the bottleneck?
- Pruning sometimes *improves* accuracy over the dense model (e.g. BoolQ 83.24 vs 80.98 at 12.5% sparsity) — is this an SFT/LoRA confound, and does it replicate with variance reporting across seeds?
- The zero-filled-vector approximation for skipped-layer KV under batched heterogeneous-path inference: how large is its accuracy cost at large batch sizes the paper does not evaluate?
- Given FiRST is anomalously weak (44.13 at 12.5% vs ~64 for others), is the baseline fairly re-tuned, and how sensitive is BUDDY's relative standing to its removal?
- The paper cites Observation 2 (layer-14 importance oscillates during decoding on Llama2-7B): can the decode-time rerouting gain be isolated quantitatively beyond the SAMSum ROUGE probe, on generation-quality benchmarks not tested?
