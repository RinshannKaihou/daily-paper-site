> **一句话总结：** TheoremBench 是一个面向"竞赛式之外的"经典定理的 Lean4 基准，含 83 个父定理、1,142 个实例，分 plain-main（仅最终定理）与 premised（含自动抽取的支持子定理与显式前提）两种版本；评测四个 Lean4 证明器显示显式前提对能利用结构的模型大幅提分（DeepSeek-Prover-V2-7B 在父定理 pass@64 上从 0.053 升到 0.263），并引入定理级覆盖率与 token 效率指标，揭示当前证明器偏向易子定理且证明冗长。

> **TL;DR:** TheoremBench is a Lean4 benchmark of 1,142 instances from 83 parent theorems (inspired by Wiedijk's 100-theorem list), released in plain-main and premised forms (premised exposes supporting subtheorems + prior results as explicit premise binders). On the premised benchmark, DeepSeek-Prover-V2-7B is the strongest at ~0.46 pass@64 (subtheorem-level), vs Goedel-Prover-V2-8B ~0.33, Kimina-Prover-Distill-8B ~0.22, and Goedel-Prover-SFT ~0.11. Explicit premises lift parent-theorem pass@64 from 0.053 (plain-main) to 0.263 for DeepSeek-Prover-V2-7B and to 0.123 for Goedel-Prover-V2-8B, but give almost no gain (0.053→0.035) for the non-reasoning Goedel-Prover-SFT. Generated proofs are much longer than ground truth (median token-efficiency ~16 for Goedel-Prover-V2-8B, ~7.8 DeepSeek, ~3.6 Kimina, ~1.44 Goedel-Prover-SFT).

## 核心贡献

- **新型 Lean4 基准 TheoremBench**：基于 Wiedijk 100 定理列表的经典形式化，含 83 个父定理、1,142 个证明实例，覆盖代数、数论、实/复分析、几何、拓扑、集合论、逻辑、组合、概率、抽象代数等领域，针对"竞赛式之外、依赖更丰富"的中级数学发展。
- **Premised 表示法**：把相关先验 Lean 结果转化为显式前提绑定（premise binders），生成自包含、依赖感知的证明实例（含主定理 + 自动抽取的支持子定理），既可评"最终定理是否从零证出"，也可评"在证明结构中的部分进展"。
- **新评测指标**：定理级覆盖率 Cov(T)=p(T)/m(T)（及生存曲线形式）、平均证出子定理数 AvgSub、以及 token 效率比 E(T)（成功生成证明 token / 参考证明 token），用以揭示 pass@k 之外的"证明结构覆盖度"与"证明紧凑性"差异。
- **实证发现**：显式前提仅对能利用结构的模型显著有效；当前证明器强烈偏向易子定理、且证明常冗长（中位 token 效率远大于 1），表明正确性多靠冗长 tactic 探索而非紧凑证明规划。

## Key Contributions

- **TheoremBench**, a Lean4 benchmark of 1,142 proving instances from 83 parent theorems (inspired by Wiedijk's 100-theorems list), targeting mid-level classical theorems beyond compact competition-style statements; spans algebra, number theory, real/complex analysis, geometry, topology, set theory, logic, combinatorics, probability, abstract algebra.
- A **premised representation** that converts relevant prior Lean results into explicit premise binders, producing self-contained dependency-aware proving instances (main theorem + automatically extracted supporting subtheorems); all retained instances compile under Lean4.
- **New diagnostic metrics** beyond pass@k: theorem-level coverage Cov(T)=p(T)/m(T) (plus a survival-curve form), average subtheorems proved per parent theorem AvgSub, and a token-efficiency ratio E(T)=generated-tokens/ground-truth-tokens (with non-Lean prose/comments stripped before tokenization).
- Empirical evidence that explicit premises help only models that can exploit formal context, and that current provers are biased toward easy subtheorems and produce verbose proofs (median token-efficiency well above 1).

## Methodology

- **Benchmark construction pipeline.** (1) Collect raw Lean4 theorem developments inspired by Wiedijk's 100-theorems list; each source file = one parent theorem + its surrounding Lean development. (2) Parse each parent theorem into a "theorem group": the final declaration is the main theorem, required Lean context is reconstructed so the extracted instance compiles independently; supporting theorem/lemma declarations are extracted as subtheorems for the premised dataset. (3) For every theorem/subtheorem, earlier results used in its proof are converted into explicit premise binders, emitting a self-contained Lean4 snippet that is verified by Lean4 before inclusion (only compilable instances kept). Each instance stored with theorem-level ID, Lean declaration name, extracted premises, environment context, ground-truth proof.
- **Two dataset views.** Plain-main: one standalone final theorem per parent theorem. Premised: the same parent theorem expanded into a group (main theorem + extracted supporting subtheorems), with relevant prior results exposed as explicit premise binders. Transformation must preserve imports, namespaces, implicit binders, typeclass inference, notation, local context.
- **Dataset statistics.** 1,142 instances total; 83 theorem groups. Subtheorems-per-theorem distribution: 1–5 (39 groups, 117 instances), 6–10 (9 groups, 81), 11–20 (19 groups, 319), 21–50 (14 groups, 460), >50 (2 groups, 165). Difficulty estimated via LLM-based classification + Lean4 structural features (proof length, referenced declarations, tactic usage, number of mathematical fields).
- **Models evaluated (4 Lean4-capable provers).** DeepSeek-Prover-V2-7B, Goedel-Prover-V2-8B, Kimina-Prover-Distill-8B, and the non-reasoning Goedel-Prover-SFT (as baseline). All instances verified with Lean4 via an automated pipeline recording compilation outcomes, resource usage, structural statistics; ground-truth proofs kept for token-level comparison.
- **Sampling.** Each theorem/subtheorem sampled up to k_max=64 times; every candidate proof checked by Lean4. pass@k reported for powers of two up to 64.
- **Metrics.** Pass@k (standard, instance-level). Fully-proved-parent-theorem (strict: main theorem + ALL subtheorems proved). Theorem-level coverage Cov(T)=p(T)/m(T)×100 (plus survival curve = fraction of parent theorems whose coverage ≥ threshold). AvgSub=(1/N)Σ_T p(T). Token-efficiency E_avg(T)=mean(ℓ_gen)/ℓ_gt and E_min(T)=min(ℓ_gen)/ℓ_gt, after stripping non-Lean prose/comments; values >1 ⇒ longer than reference. Plain-main vs premised comparison on pass@64 over shared theorem units.

## Results & Highlights

- **Subtheorem-level pass@k (premised, Figure 2).** DeepSeek-Prover-V2-7B is strongest at every k, ~0.46 pass@64; Goedel-Prover-V2-8B second at ~0.33; Kimina-Prover-Distill-8B ~0.22; Goedel-Prover-SFT lowest at ~0.11. Ranking already visible at k=1 and k=2, so the gap reflects first-attempt quality, not just larger sampling budgets.
- **Impact of explicit premises (Figure 1, pass@64, shared parent theorems).** DeepSeek-Prover-V2-7B: 0.053 (plain-main) → 0.263 (premised). Goedel-Prover-V2-8B: 0.053 → 0.123. Kimina-Prover-Distill-8B: 0.053 → 0.070 (modest). Goedel-Prover-SFT: 0.053 → 0.035 (essentially no gain, even slightly worse) — premises help only when the prover can exploit formal context.
- **Theorem-level coverage (Figure 5, survival curve).** DeepSeek-Prover-V2-7B has the strongest curve over most of the range (most likely to cover a large fraction of a theorem's subtheorems); Goedel-Prover-V2-8B competitive only at very low thresholds; Kimina-Prover-Distill-8B behind; Goedel-Prover-SFT weakest — confirming DeepSeek leads not just at individual subtheorems but at advancing through an entire theorem group.
- **Token-efficiency (Figure 6, solved premised instances).** Generated proofs are usually much longer than ground truth. Goedel-Prover-V2-8B most verbose (median token-efficiency ~16); DeepSeek-Prover-V2-7B ~7.8; Kimina-Prover-Distill-8B ~3.6; Goedel-Prover-SFT lowest median ~1.44 — but this reflects a smaller, more selective solved subset, not uniformly better compactness.
- **Metric context.** All pass@k/coverage/token-efficiency figures are on TheoremBench's 1,142-instance / 83-parent-theorem Lean4 benchmark, k_max=64 samples per instance, Lean4-verified; plain-main vs premised comparison is pass@64 on shared parent-theorem units.

## Critical Assessment

- The headline "explicit premises substantially improve performance" is **sharply model-dependent and the paper itself shows it**: the gain is large for DeepSeek-Prover-V2-7B (0.053→0.263) and Goedel-Prover-V2-8B (0.053→0.123), modest for Kimina (0.053→0.070), and **essentially zero or negative for Goedel-Prover-SFT (0.053→0.035)** — so the contribution is really "premises help reasoning-capable models," and the benchmark's diagnostic value rests heavily on the four (all 7–8B, low-size) models tested.
- Coverage is **bounded by what is already formalized**: Limitation §5 states the benchmark "depends on what has already been formalized and can be extracted into compilable snippets," and "premise extraction may not always recover the mathematically minimal dependency set" (syntactic dependency analysis only) — so the premised representation is a syntactic approximation whose fidelity to true mathematical dependency is unmeasured.
- Experiments cover only **four Lean4-capable prover models under a fixed sampling budget (k_max=64) and low model sizes (7–8B)** — the authors explicitly flag (Limitation §5) that broader model families, search strategies (e.g. BFS-Prover, APOLLO, HyperTree), and retrieval-augmented settings (LeanDojo/ReProver/REAL-Prover, which the paper cites) are left to future work, so the ranking may not hold for larger or search-augmented provers.
- The token-efficiency comparison is **confounded by solved-subset selection bias**: the paper concedes Goedel-Prover-SFT's low median (~1.44) "reflects a much smaller and more selective solved subset rather than uniformly better proof compactness," but does not report how the solved-subset sizes differ across models, so the verbosity ranking is not apples-to-apples.

## Strengths

- Genuinely diagnostic benchmark design: the plain-main vs premised contrast isolates whether a prover can exploit explicit formal context, and the theorem-level coverage survival curve measures partial progress through a proof structure — something standalone pass@k on competition benchmarks (MiniF2F, PutnamBench, CombiBench) cannot reveal.
- Clean construction pipeline: every retained instance is Lean4-verified to compile, with premise binders preserving imports/namespaces/typeclass inference/notation, yielding self-contained dependency-aware snippets rather than long prompts that scale poorly.
- Honest, fine-grained reporting: token-efficiency metric strips non-Lean prose, pass@k reported for powers of two up to 64, and the model-dependent premise-gain is reported transparently rather than averaged away.

## Limitations

- Acknowledged (§5): coverage bounded by existing Lean4 formalizations; premised transformation uses syntactic dependency analysis (premise extraction may not recover mathematically minimal dependencies); only four 7–8B Lean4-capable provers under a fixed k_max=64 budget tested, leaving larger models, search strategies, and retrieval-augmented settings to future work.
- Token-efficiency comparison confounded by per-model solved-subset selection bias (different models solve different, differently-sized subsets). Only 83 parent theorems (1,142 instances), heavily skewed toward a few large groups (2 groups with >50 subtheorems contribute 165 instances). Preprint (no peer review); no explicit code/data URL stated in the body.

## Relevance

主要映射到"**LLM Inference Evaluation（推理评估）**"——TheoremBench 本身是一个面向 Lean4 形式化定理证明的诊断型基准，引入定理级覆盖率与 token 效率等细粒度评测指标；其次与"**ML Reliability & Mathematical Foundations**"相关，因其评测形式数学推理这一数学基础任务，并揭示当前证明器在"证明结构完整性与紧凑性"上的可靠性短板。

## Questions for Further Exploration

- The premise gain is strongly model-dependent (DeepSeek-Prover-V2-7B 0.053→0.263 vs Goedel-Prover-SFT 0.053→0.035) — does the gain scale with model size or with reasoning-specific training, i.e. would a larger or search-augmented prover (e.g. BFS-Prover, APOLLO, HyperTree Proof Search, all cited in §2) exploit premises even more strongly than the 7–8B models tested?
- The premised transformation relies on syntactic dependency analysis and the authors concede (§5) it "may not always recover the mathematically minimal dependency set" — how would a human-annotated minimal-dependency version of a subset of theorem groups change the premise-gain magnitudes (e.g. DeepSeek's 0.053→0.263)?
- The token-efficiency ranking (Goedel-Prover-V2-8B median ~16 vs Goedel-Prover-SFT ~1.44) is confounded by solved-subset selection bias the paper acknowledges — controlling for which subtheorems each model solves, is there still a real verbosity difference, or does it vanish on the commonly-solved subset?
- TheoremBench is built from Wiedijk's 100-theorems formalizations (1,142 instances, 83 parents) — how does the prover ranking (DeepSeek > Goedel-Prover-V2 > Kimina > Goedel-Prover-SFT) compare on FormalMATH (5,560 instances) or ProverBench (325 instances), which the paper cites as the closest large Lean4 benchmarks, to test whether the diagnostic ranking generalizes beyond classical theorems?
