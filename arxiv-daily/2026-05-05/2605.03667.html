ELAS: Efficient Pre-Training of Low-Rank Large Language Models via 2:4 Activation Sparsity

arXiv: 2605.03667

Date: 2026-05-05

https://arxiv.org/abs/2605.03667

一句话总结： ELAS在低秩LLM训练框架（LORO）中引入平方ReLU激活和2:4结构化激活稀疏，利用NVIDIA硬件稀疏加速实现1.5×–2.75×推理加速和约44%的FFN激活内存减少，在60M–1B LLaMA模型上仅带来0.07–0.28的困惑度损失。
TL;DR: ELAS combines LORO low-rank training with 2:4 structured activation sparsity by replacing SwiGLU with ReLU² in FFN layers. A magnitude-based top-2 selection algorithm (implemented in Triton kernels) sparsifies activations, with STE for gradient flow and a dense warmup phase (1000 steps) for stability. On LLaMA 60M–1B trained on C4, ELAS achieves perplexity within 0.07–0.28 of LORO while cutting FFN activation memory ~44% and delivering 1.5×–2.75× inference speedups over full-rank baselines.

核心贡献

  • 提出ELAS框架，将LORO低秩训练与2:4结构化激活稀疏无缝结合，消除额外稀疏权重模块的开销
  • 在FFN层用ReLU²替代SwiGLU，使激活自然呈现84%–98%稀疏率（warmup后），适配2:4稀疏格式
  • 设计基于幅值的top-2选择算法（Algorithm 1）配合直通估计器（STE）处理不可微稀疏化操作
  • 引入dense warmup阶段（1000步），确保训练初期表示稳定后再施加稀疏约束
  • 在LLaMA 60M–1B上实现与LORO近乎相同的困惑度，同时减少约44% FFN激活内存并获得1.5×–2.75×推理加速

Key Contributions

  • Proposes ELAS, a low-rank + sparse framework that applies 2:4 structured sparsity to activations (not weights), avoiding the performance degradation and architectural complexity of weight-sparsity methods like SLTrain and LOST
  • Replaces SwiGLU with ReLU² in FFN layers to induce 84–98% natural activation sparsity after warmup, making 2:4 structured sparsity a low-overhead approximation
  • Implements magnitude-based top-2 selection via Triton kernels with STE for backward pass, enabling hardware-native sparse-dense matrix multiplication on NVIDIA Ampere+ GPUs
  • Demonstrates that simple naive magnitude selection outperforms soft thresholding (S-STE) for activation sparsification — soft methods cause divergence at 350M+ scales
  • Achieves competitive perplexity (within 0.07–0.28 of LORO) while cutting FFN activation memory ~44% and delivering 1.5×–2.75× inference speedup over full-rank models

Methodology

ELAS builds on three components:

  1. Low-rank training (LORO): Weight matrices W are factorized as W=AB with rank r≪min(d_out, d_in). LORO uses Riemannian optimization with periodic SVD-based subspace refresh. Both attention and MLP layers use rank r_attn=r_mlp=256. LORO is chosen over CoLA/LOST for its cleanest inference path (no extra sparse matrices or activations between low-rank factors).
  1. Architecture modification: SwiGLU is replaced with a simpler FFN: MLP(x) = W_down · ReLU²(W_up · x), where ReLU²(z) = (max(0,z))². This removes the gating mechanism and naturally produces high activation sparsity.
  1. 2:4 activation sparsification: After ReLU², activations are sparsified by keeping the top-2 elements (by absolute value) in each group of 4 consecutive elements. This is implemented with Triton kernels. During backprop, the straight-through estimator (STE) treats sparsification as identity for gradient flow.
  1. Dense warmup: The first 1000 training steps use dense (unsparsified) activations to establish stable representations before introducing 2:4 constraints.

Training: All experiments on C4 dataset, one epoch, BF16 precision. Models: LLaMA 60M (1.3B tokens), 130M (2.6B), 350M (6.4B), 1B (13.1B tokens).

Results & Highlights

Perplexity (Table 1): ELAS achieves nearly identical perplexity to LORO across all scales:

  • 60M: 34.12 vs LORO 33.87 (Δ+0.25) — both beat Full-Rank 34.06 by <0.3
  • 130M: 24.80 vs LORO 24.78 (Δ+0.02)
  • 350M: 19.94 vs LORO 19.66 (Δ+0.28)
  • 1B: 15.69 vs LORO 15.53 (Δ+0.16), vs Full-Rank 15.56

ELAS consistently outperforms LoRA, ReLoRA, and matches/beats GaLore and CoLA while using the same memory as LORO (3.66 GB for 1B vs 8.04 GB full-rank).

Activation memory (Table 2): At 1B model, ELAS reduces FFN activation memory by ~44% across all batch sizes (e.g., batch 128: 101.76 GB vs LORO 181.71 GB). Savings scale linearly with batch size.

Inference speedup (Table 3): FFN module speedup over full-rank:

  • 60M/130M: marginal or negative at seq_len=512 (kernel overhead), 1.5×–1.8× at seq_len≥2048
  • 350M: 1.8×–1.9× at seq_len≥2048
  • 1B: 2.0×–2.75× consistently at seq_len≥512, peaking at 2.75× for long sequences

Ablation — Warmup (Table 4): Without warmup, 60M diverges (NaN); 130M degrades to 29.7 PPL. 1000 steps is optimal; longer warmup shows diminishing returns.

Ablation — Sparsification method (Table 5): Naive magnitude selection outperforms S-STE soft thresholding. Soft_weights causes NaN at 350M and 1B. Soft_activation (adapted for activations) is better but still worse than naive and diverges at 1B.

Critical Assessment

  • Novelty: Moderate. The individual components (LORO, ReLU², 2:4 sparsity, STE) are all established. The contribution is the specific combination applied to activations rather than weights in a low-rank setting, which is a sensible engineering insight. The finding that naive magnitude selection beats soft thresholding for activations is a useful empirical contribution.
  • Rigor: Experiments are thorough for the scale tested — 4 model sizes, comprehensive ablations on warmup and sparsification methods, memory and speed benchmarks. However, 1B parameters is still small; the memory and speedup benefits are projected rather than demonstrated at 7B+ where they matter most.
  • Reproducibility: Good. All hyperparameters are specified, code is available, C4 is a public dataset, and the Triton kernel implementation is described.
  • Significance: If the approach scales, the practical benefits could be substantial. The 44% activation memory reduction is meaningful for large-batch training. However, the lack of experiments beyond 1B is a significant limitation for a paper about efficient LLM training.
  • Red flags: (1) No 7B+ experiments — the entire value proposition targets large models, yet validation stops at 1B. (2) The perplexity gap vs LORO widens slightly at 350M (0.28) compared to 130M (0.02) — it is unclear if this trend continues at larger scales. (3) Speedup measurements are FFN-module-only, not end-to-end training throughput, which inflates the perceived benefit since FFN is only part of the compute. (4) The paper compares ELAS inference to full-rank inference (not LORO inference), which is a favorable comparison since LORO already has smaller matrices.

Strengths

  • Clean and well-motivated approach — sparsifying activations avoids the weight-sparsity accuracy penalty
  • Comprehensive ablation studies that reveal practical insights (warmup necessity, naive > soft thresholding for activations)
  • Genuine hardware acceleration leveraging NVIDIA's native 2:4 sparse format
  • Significant activation memory reduction that scales with batch size
  • Maintains LORO's clean inference pathway (no auxiliary sparse modules at deployment)

Limitations

  • No experiments beyond 1B parameters — scalability to 7B+ is the critical open question
  • Speedup is FFN-module-only, not end-to-end; actual training acceleration is likely lower
  • ReLU² replaces SwiGLU, which may affect model quality at larger scales where SwiGLU's benefits are more pronounced
  • The comparison baseline for speedup is full-rank, not LORO (which already has smaller weight matrices)
  • Only evaluated on C4 (English web text) — generalization to multilingual or code data is unknown
  • Dense warmup adds 1000 steps of full-activation-memory training, partially offsetting the memory savings

Relevance
