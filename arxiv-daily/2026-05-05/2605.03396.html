Design and Implementation of BNN-Based Object Detection on FPGA

Date: 2026-05-05

Full Verilog RTL design with fused Mul_prev compensation in BNN PEs; RTL simulation shows near-perfect agreement (r=0.999964) with ONNX reference.

Key Contributions

  • Full FPGA implementation of a BNN-based YOLOv3-tiny detector using 1-bit weights and 8-bit activations for main convolution layers
  • Complete Verilog RTL design encompassing padding, line buffering, binary convolution, quantization post-processing, max pooling, and detection-head computation
  • Fused Mul_prev into the BNN PE for channel-wise compensation during accumulation, improving quantization fidelity

Methodology

The authors design a YOLOv3-tiny-like architecture adapted for binary neural network inference on FPGA. Key methodological choices include:

  • Network architecture: 320×320×3 RGB input; main convolutional layers use 1-bit weights with 8-bit activations; Conv1 and the final detection head retain fixed-point standard convolutions.
  • Quantization flow: A trained ONNX model serves as the source; weights, biases, and quantization parameters are extracted, converted to fixed-point, packed into COE files, and stored in Vivado BRAM ROMs.
  • Hardware design: Fully implemented in Verilog RTL with dedicated modules for padding, line buffering, binary convolution, quantization post-processing, max pooling, and detection-head computation.
  • Fused compensation: Mul_prev (indexed by input channel) is fused into the BNN processing element alongside Div_current (indexed by output channel), enabling channel-wise compensation during the accumulation step.

Results & Highlights

  • Accuracy: 39.6% mAP50 on PASCAL VOC with only 0.098 GFLOPs and 0.74M parameters.
  • Hardware validation: RTL simulation shows near-perfect agreement with ONNX model — correlation coefficient 0.999964, MAE 0.020027.
  • Efficiency: Binary weights drastically reduce memory and compute requirements, making real-time inference feasible on low-cost FPGAs.

Critical Assessment

  • Novelty: Moderate. BNN-based object detection on FPGA has been explored before (e.g., FINN, BNN-VITIS). The main novelty lies in the fused Mul_prev compensation in the PE and the specific YOLOv3-tiny adaptation.
  • Rigor: The RTL-vs-ONNX validation is thorough (r=0.999964). However, the paper only reports VOC mAP — no comparison with prior BNN-FPGA works on latency, throughput, resource utilization, or power consumption is available from the abstract.
  • Reproducibility: The ONNX-to-FPGA flow is well-described. Whether RTL source or bitstreams are released is unclear.
  • Significance: Useful for the edge-deployment community, but 39.6% mAP50 on VOC is relatively low compared to even lightweight full-precision detectors.
  • Red flags: Missing on-device performance metrics (FPS, LUTs/BRAM/DSP usage, power). The 0.098 GFLOPs figure refers to the BNN model's theoretical cost, not actual FPGA throughput.
