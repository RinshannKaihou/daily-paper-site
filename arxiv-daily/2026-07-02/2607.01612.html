Scaling with Confidence

Calibrating confidence of LLMs for adaptive test-time scaling — C3RL + CAS.

Authors · Xuqing Yang, Yi Yuan, Shanzhe Lei, Xuhong Wang Affil. · SJTU · Southeast Univ. · Shanghai AI Lab / PJLab Bases · Llama-3.2-3B · Qwen2.5-VL-7B
confidence calibration test-time scaling reinforcement learning verbalized confidence inference efficiency
Key Finding

C3RL calibrates LLM verbalized confidence, then spends it via CAS early-stopping — cutting inference budget up to 12.33× with no accuracy loss.

Two-stage pipeline · calibration reward feeds a Beta-distribution stopping router · verified across 8 text + multimodal benchmarks.
One-line TL;DR:
RLVR wrecks calibration; C3RL fixes it, then harvests the signal as compute.

Methodology

Training data — ~106k reasoning samples (NuminaMath-TIR 69k, WebInstruct 4.5k, LogicNLI 16k, LogiQA 8.7k). Each question sampled 10× at temp 0.7 and tagged all-correct / partial / all-incorrect (13.7k / 68.1k / 24.3k).

RC3RL  =  Rcorrect  +  Rcalib  +  Rref
R_CORRECT
Standard RLVR
+1 if answer matches ground truth.
+
R_CALIB (β, t=5)
Calibration
Reward certain∧correct & uncertain∧wrong; penalize the reverse.
+
R_REF (α)
Reference anchor
+α for cracking a formerly all-wrong item; −α for hallucinating on an all-correct one. Stops "calibrated-but-wrong" drift.

The reference term is the structural defense: it anchors confidence to per-sample empirical difficulty, so a degenerate confidence emitter can't satisfy the reward.

CAS · Inference-time router
for n in 1..N:
  sample response rn → answer a, confidence c ∈ {1..10}
  sum per-answer confidence: ca = Σ conf
  c₁ = max, c₂ = second-max
  Prob(c₁,c₂) = (1/A) ∫₀^0.5 xc₂(1−x)c₁ dx
  if Prob ≥ P_stop or n=N: return argmax ca
— sequential & parallel variants provided

On Llama-3.2-3B, C3RL is the only RL method that exceeds RLVR accuracy (50.1% vs 48.9%) while sharply lowering ECE — RLCR and SaySelf both pay large accuracy penalties there.

Strengths
  • Real failure modeDirectly attacks RLVR's overconfidence → hallucination.
  • Clean decompositionThree-term reward, each term ablated in isolation.
  • Cheap defenseReference-accuracy anchor is well-motivated & replicable.
  • Honest evaluationJoint accuracy+calibration across 2 model families + multimodal.
  • Deployable winLarge efficiency gain, degrades gracefully — no accuracy loss.
  • Reproducible3-seed runs w/ reported std; code/data/model release committed.
Limitations
  • Not Pareto-dominantSaySelf still produces strictly better ECE.
  • Small-N regimeCAS advantage over AC negligible for N<8.
  • Domain skewTraining math/logic-heavy; science data insufficient.
  • Coarse signal1–10 verbalized confidence; no finer-grained analysis.
  • Missing studiesNo prompt-robustness / adversarial / open-ended-generation eval.
  • Best-case leadHeadline 12.33× is OOD-text only.
  • Inherited biasReference tags depend on base-model 10-sample accuracy.

Open Questions

Q1 · ROBUSTNESS
Is verbalized-confidence calibration robust to prompt perturbations & adversarial inputs? Is the reference-accuracy reward enough to stop degenerate emitters?
Q2 · SMALL-N
Does CAS retain its lead capped at N≤4 — the regime where it ≈ Adaptive-Consistency? Can a better stopping criterion close that gap?
Q3 · PARETO
How do α vs β vs correctness weights shape the accuracy–calibration frontier? Continuous curve or phase transition?
Q4 · ONLINE TAGS
Could reference-accuracy tags be regenerated online (self-distillation) to escape base-model biases in novel domains?
Q5 · TRANSFER
Does C3RL transfer to open-ended generation / coding / agentic tasks where ground truth isn't a clean string match?
Q6 · VERIFIERS
Against a learned verifier / reward-model scaler at matched budgets, does confidence-weighted CAS still win — or get subsumed?
Q7 · SCALE
Beyond 7B: does the calibration headroom (and hence CAS budget saving) shrink for already-well-calibrated larger models?
Q8 · METRIC
How sensitive are ECE (bin count) & AUROC to the discrete 1–10 scale? No metric-robustness discussion in the paper.
