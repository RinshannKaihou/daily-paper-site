> **一句话总结：** PACE 通过对 19 个廉价非智能体基准做 SVD 杠杆分数（全局几何重要性）× 目标相关性（Spearman 秩相关）的双策略实例筛选，再用 bootstrap 稳定的最小二乘/逻辑回归，仅用 100 个原子评测实例即可在 4 个昂贵智能体基准上以约 1/100 成本达到 LOOCV MAE 3.80%、Spearman 0.81、成对排序准确率 84%。

> **TL;DR:** PACE predicts a model's score on expensive agentic benchmarks (SWE-Bench Verified, SWE-Bench Multimodal, SWT-Bench, GAIA) by selecting just ~100 instances from 19 cheap non-agentic benchmarks via two complementary signals (SVD leverage for global geometry × target-relevance Spearman rank-correlation) and fitting a bootstrap-stabilized regression, achieving LOOCV MAE 3.80%, Spearman 0.81, and ~84% pairwise ranking accuracy at ~1/100 the cost of sub-sampling the agentic benchmarks themselves.

## 核心贡献

- 提出 **PACE** 框架：把"用廉价非智能体基准预测昂贵智能体基准"形式化为有预算约束的子集选择问题，从 19 个源基准（覆盖 11 类原子能力）中为每个目标智能体基准筛选约 100 个最具预测力的实例。
- 设计了两种互补的过滤式选择信号：**全局选择**（SVD 薄分解得到的杠杆分数 hᵢ × 目标相关性 |ρᵢ|，捕捉全局信息量）与**局部选择**（仅按目标相关性 |ρᵢ| 选择后在子矩阵上重做 SVD，捕捉目标特化结构），并在预测时用学习到的集成权重 λ 融合。
- 用 **bootstrap 重采样目标实例**稳定回归权重：对目标均值 ȳ_m 做 B 次 bootstrap 复制，缓解目标实例稀少导致的标签噪声，使权重对单次目标采样不敏感（消融显示平均 MAE 改善 0.77%、Spearman 改善 0.15）。
- 同时支持两个目标：**Goal A（绝对分数预测，最小二乘回归）** 与 **Goal B（成对偏好预测，结构化 Bradley-Terry 逻辑回归）**，复用同一批选中实例。
- 构建 **PACE-Bench**，在 14 个模型、4 个智能体基准上以 LOOCV 验证：平均 MAE 3.80%、Spearman 0.81、成对准确率 84.37%，成本约为随机目标采样的 **1/100**。
- 通过分析被选实例的能力分布，给出可解释的"每个智能体基准独特需要哪些原子能力"的画像（如 GAIA 重指令遵循与验证、SWT-Bench 重规划与测试验证、SWE-Bench Multimodal 重长上下文聚合）。

## Key Contributions

- Frames agentic-performance prediction as a **budget-constrained subset-selection problem**: given a budget C, pick C non-agentic instances whose aggregate scores best predict each target agentic benchmark, drawing from a pool of 19 source benchmarks spanning 11 atomic capabilities.
- Two complementary **filter-based selection signals**: *Global selection* combines thin-SVD leverage scores hᵢ (target-independent geometric prior over the source matrix) with target relevance |ρᵢ|; *Local selection* ranks purely by target relevance and recomputes an SVD on the selected submatrix. The two are ensembled with a learned per-target weight λ.
- **Bootstrap-stabilized regression**: resamples the (small, noisy) target instance set B times so regression weights average over the sampling distribution of the target mean — an ablation shows it improves every target (avg ΔMAE −0.77%, ΔSpearman +0.15).
- Supports two prediction goals with the same selected instances: **Goal A** (absolute score via least-squares linear regression) and **Goal B** (pairwise preference via a structured Bradley-Terry logistic regression).
- Validates **PACE-Bench** under strict LOOCV across 14 models × 4 agentic benchmarks: average MAE **3.80%**, Spearman **0.81**, pairwise accuracy **84.37%**, at ~**1/100** the cost of a random sub-sample of the agentic benchmarks.
- Provides an **interpretable per-target capability profile** (e.g., GAIA → instruction following + verification; SWT-Bench → planning + verification/test; SWE-Bench Multimodal → long-context aggregation).

## Methodology

PACE predicts a held-out model's mean score on a target agentic benchmark from its scores on a compact set of non-agentic (atomic) source instances.

**Setup.** Models M are split into a calibration set M_train and a held-out evaluation set M_eval. For each target benchmark the goal is to select C source instances and learn a predictor. The candidate pool stacks all 19 source benchmarks into one matrix X. A naive joint Lasso/Ridge fit is under-determined (|M_train| ≪ #instances), so PACE **decouples selection from regression**.

**Instance selection (two complementary filter signals).** A thin SVD X = UΣVᵀ yields two per-instance scores:
- *Geometric importance* — leverage score hᵢ = Σ_c V²_{c,i}, measuring an instance's contribution to the global latent structure (a classical CUR-decomposition column-selection criterion).
- *Target relevance* — |ρᵢ| = |Spearman(X_{Mtrain,i}, ȳ_{Mtrain})|, the rank consistency between an instance's scores and the target mean across training models (a standard filter-based feature-selection criterion).
- *Global subset* G ranks by the product hᵢ × |ρᵢ|; *Local subset* L ranks by |ρᵢ| alone then recomputes an SVD on the L-submatrix. The final proxy P = L ∪ G (greedily extended to exactly C unique instances), and the two predictions are ensembled as ŷ = λ·ŷ_L + (1−λ)·ŷ_G, with λ and the budget split (C_L, C_G) tuned by held-out validation. Held-out models are projected into the SVD space via the pseudoinverse so the regression operates uniformly.

**Regression (noise-aware).**
- *Goal A (score):* least-squares f(x_m) = w⊤x_m trained on M_train against target mean ȳ_m.
- *Goal B (pairwise):* a Bradley-Terry logistic regression on score-differences (x_m − x_m′) with label 1[ȳ_m > ȳ_m′].
- Both objectives use B bootstrap replicates of each ȳ_m (resampling target instances with replacement) to desensitize weights to target-sampling noise.

**Agentic benchmarks (4):** GAIA, SWE-Bench Verified, SWE-Bench Multimodal, SWT-Bench — all run via the OpenHands Index / OpenHands Software Agent SDK under a unified harness. **Source pool (19):** ACPBench, AIME 2025, BEIR, BFCL, DebugBench, GPQA, HumanEval, IFEval, InFoBench, LIFBench, LiveCodeBench, LogiQA, MBPP, MMLU, MMMU, PlanBench, RepoBench-R, VisualPuzzles, VisualWebBench — run via lm-evaluation-harness. **Models (14):** GPT-5.2, GPT-5.2 Codex, Gemini 3 Pro/Flash, Claude Opus 4.5/4.6, Claude Sonnet 4.5, DeepSeek V3.2, GLM 4.7, Kimi K2/K2.5, MiniMax M2.1/M2.5, Qwen3-Coder-480B-A35B. Evaluation is strict **leave-one-out CV**.

## Results & Highlights

- **Main (C = 100, LOOCV):** GAIA MAE 5.77% / Spearman 0.79 / pair-acc 83.3%; SWE-Bench Verified 2.09% / 0.67 / 78.5%; SWE-Bench Multimodal 2.23% / 0.89 / 85.4%; SWT-Bench 5.12% / 0.89 / 90.1%. **Averages: MAE 3.80%, Spearman 0.81, pair-acc 84.37%** (random baseline for pair-acc = 50%).
- **Cost-quality tradeoff (Fig. 1):** At every sub-saturation budget PACE dominates a random sub-sample of the *target* agentic benchmarks on all three metrics; to match PACE's quality the random baseline costs ~**100× more**. Quality improves with C and saturates around a few hundred, with C = 100 a practical sweet spot.
- **Interpretability (Fig. 3):** Instruction Following and Reasoning are saturated (100 instances) for all targets; the rest form distinct signatures — GAIA = IF + Verification/Test; SWE-Bench Verified = Planning + Verification + Code Generation + Error Recovery; SWE-Bench Multimodal = Long-Context Aggregation (multimodal allocation is modest because text + code often suffice); SWT-Bench = Verification/Test + Planning.
- **Bootstrap ablation (Table 3):** removing bootstrap degrades every target (avg MAE 4.57% → 3.80%, Spearman 0.66 → 0.81 with bootstrap).
- Per-instance agentic costs (Claude Sonnet 4.5): GAIA $0.38, SWE-Bench Verified $1.19, SWE-Bench Multimodal $1.89, SWT-Bench $0.98 vs. non-agentic instances at $0.004–$0.121 — quantifying the ~100× gap.

## Critical Assessment

- **Novelty — moderate.** Predicting one benchmark from others via regression is a known idea (Collaborative Performance Prediction, ONEBench, Chatbot-Arena-estimate meta-analyses are cited). The genuine contributions are the *decoupled* filter-based selection (SVD leverage × relevance) tailored to the under-determined regime |M_train| ≪ #instances, the bootstrap stabilization, and the ready-to-use, interpretable PACE-Bench. It is cleanly differentiated from benchmark-compression work (Perlitz, Polo) that subsamples within one target benchmark and from approaches that recast agent tasks into MCQs (Qin et al.).
- **Rigor — solid but bounded by N.** LOOCV over 14 models is a fair, conservative protocol and avoids the overfitting of in-sample reporting. However, pairwise-accuracy on 14 models is statistically thin (only 91 pairs total), so the headline ~84% has non-trivial variance. The bootstrap ablation and cost-accuracy sweep are appropriate. Per-target numbers are honestly reported (e.g., SWE-Bench Verified Spearman is only 0.67, GAIA/SWT MAE ~5–6%).
- **Reproducibility — likely good** given the authors' track record, the named repos, the use of standard harnesses (lm-evaluation-harness, OpenHands Index), and a fully specified model/benchmark list. Risk: agentic scores are harness- and environment-sensitive, so exact reproduction depends on matching the OpenHands setup.
- **Significance — high practical value, modest theory.** If the proxy generalizes, it materially cheapens model development, checkpoint monitoring, and routing/screening — the paper's stated use cases are credible.
- **Red flags / overclaiming.** The paper is reasonably measured. Itself flags the key limitation (generalization depends on calibration models being representative of future models). Minor: the "1/100 cost" comparison is against *random target sub-sampling*, not against more competitive cheap baselines (e.g., one-shot/MCQ recastings), so the efficiency claim is relative to a weak baseline; no comparison to CPP matrix-factorization or ONEBench-style predictors is run head-to-head.

## Strengths

- Tackles a genuine, expensive pain point in agentic LLM evaluation.
- Clean decoupling of selection (filter-based, cheap, stable) from regression (bootstrap-stabilized), well-motivated by the under-determined regime.
- Dual Global/Local selection is principled (shared geometric prior vs. task specialization).
- Broad, realistic empirical coverage: 14 strong frontier models, 4 diverse agentic targets, 19 source benchmarks.
- Honest per-target reporting (does not hide the weak SWE-Bench Verified ranking).
- Interpretable capability signatures add scientific value beyond prediction.

## Limitations

- **Generalization to future models is the crux** and is only validated via LOOCV within one fixed snapshot of 14 models; distribution shift from new architectures/training paradigms is unaddressed empirically (acknowledged by authors).
- **Small N = 14** makes pairwise accuracy (91 pairs) statistically fragile; confidence intervals are not reported.
- The "~100× cheaper" claim is vs. a *random* target sub-sample — no comparison to stronger cheap predictors (CPP/ONEBench/MCQ recastings).
- **Harness dependence:** all agentic scores come from one harness (OpenHands); it is unclear whether the proxy transfers to other agent harnesses or judge settings.
- Only **4 agentic targets** (all coding/QA-flavored); transfer to browser agents, tool-use, robotics, or open-ended agents is unverified.
- Regression is **correlative, not causal**; selected instance weights may break when capability composition shifts.
- "Cheap" framing assumes the 19 non-agentic scores already exist; computing them still has nontrivial (if far smaller) cost.

## Relevance

这篇论文与 LLM/Agent 评测、模型选择与路由方向高度相关：它提供了一个低成本、可解释、即插即用的代理评测方案（PACE-Bench），让开发者在跑昂贵的全量 agent 评测前就能筛模型、监控训练 checkpoint、做路由决策，对需要频繁评估智能体能力的实践者非常实用。

## Questions for Further Exploration

- How fast does predictive accuracy degrade as *new* model families (post-snapshot) are added — what is a realistic re-calibration cadence?
- Does the proxy transfer across **agent harnesses** (e.g., a non-OpenHands harness), or is it calibrated to one harness's quirks?
- Can PACE beat stronger cheap baselines — CPP matrix factorization, ONEBench, or MCQ-recast agent tasks — not just random target sub-sampling?
- How do the selected instances/weights shift when the calibration model set changes (stability of the proxy subset)?
- Does the method scale to **other agentic domains** (browser/tool-use/embodied agents) beyond the 4 coding/QA targets tested?
- Could bootstrap be replaced by a Bayesian regression that directly models target-mean uncertainty, and would that improve calibration of the predicted scores (not just rankings)?
