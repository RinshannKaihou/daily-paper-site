A plug-in, training-free gate that suppresses hallucination cascades in multi-agent coding — adoptable as a reliability baseline, at a stated ~2× cost.

Methodology

Interception layer inside the ChatDev chat-chain
R→ D→ U?→ C→ U?→ T→ U?→ C*

Requirement → Design → Code → Test → Repaired code. Between subtasks, each artifact's uncertainty u is scored against a calibrated threshold τ_φ; high-uncertainty responses trigger selective web-retrieval before the next agent proceeds — "selective RAG" rather than retrieval every turn.

Roles: CEO, CTO, Programmer, Reviewer, Tester. ≤10 communication rounds/subtask; "communicative dehallucination" enabled at Code Completion, Review & Testing.

  1. 01

    Token-level uncertainty

    Temperature-scaled softmax (t=1.0) over log-probs, then u = −log Avg(z₁…zₙ). Collapses to u=|p₁| for single-token yes/no planning decisions.

  2. 02

    Per-subtask calibration

    τφ = 90th percentile of uncertainty over historically-correct responses (α=0.10). Global fallback τ=0.5 when <50 correct samples.

  3. 03

    Open-logprob recipe

    For Gemma/Qwen: retain raw logit tensor, apply log-softmax over vocab, keep top-k. Proprietary APIs return logprobs directly.

  4. 04

    Selective retrieval gate

    If u > τ_φ → web-augment context; else standard ChatDev chain. Verification fires only on low-confidence turns to cut overhead.

Strengths & Limitations

Strengths

  • Targets a concrete, well-motivated failure mode — hallucination propagation — that mainstream multi-agent coding frameworks ignore.
  • Training-free, plug-in — works with any model exposing (or computable to) logprobs.
  • Per-subtask 90th-percentile calibration pragmatically solves the variable-length response problem.
  • Reports efficiency alongside quality — honest about the ~2× cost.
  • Clean within-system ablation plus informative communication & error-transition analyses.

Limitations

  • Confounded external comparison — UA-ChatDev's backbone differs from baselines, so uncertainty's marginal contribution is unclear.
  • Executability does not beat ChatDev (0.859/0.840 vs 0.880) — "execution reliability" framing is overstated.
  • ~2× latency & ~1.5–1.8× tokens — material cost not fully weighed against benefit.
  • Single benchmark (SRDD), self-curated; no variance/significance, single seed.
  • Calibration depends on a circular "correct" proxy; web-retrieval source/API unspecified.
  • No comparison to newer agentic baselines (SWE-agent style) despite citing them.

Open Questions

Q1

If ChatDev/MetaGPT were re-run on the same Gemma/Qwen backbone, how much of the 0.395→0.649 Q gain survives once the LLM is held fixed?

Q2

Does the "w/o Uncertainty" ablation also remove web retrieval, or only the gating? If retrieval stays, it conflates two effects.

Q3

Why does Executability drop (0.880→0.859) while Completeness rises? Is gated retrieval nudging toward longer, more complete-but-buggier code?

Q4

Does the 90th-percentile τφ transfer from SRDD to HumanEval-X / MBPP / SWE-bench without retuning?

Q5

Can concise-summary retrieval (the authors' own future direction) recover most of the gain at <2× token cost?

Q6

Could calibrated uncertainty be propagated as a probability (routing, weighted ensembling) instead of a binary verify/skip gate?
