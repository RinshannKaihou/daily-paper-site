Extrapolative Weight Averaging Reveals Correctness-Efficiency Frontiers in Code RL

arXiv: 2605.28751

Date: 2026-05-27

https://arxiv.org/abs/2605.28751

一句话总结： 在代码强化学习中，不同单元测试覆盖率下训练的checkpoint形成正确性-效率前沿，线性插值可恢复该前沿，而外推可将其延伸至训练端点之外，且外推checkpoint在推理时集成中提供互补性解题能力。
TL;DR: In code RL for competitive programming, checkpoints trained under nested unit-test coverage form a correctness-efficiency frontier. Linear interpolation recovers this frontier, while extrapolation extends it beyond trained endpoints. Extrapolated checkpoints solve different hard problems, and ensembling them improves pass@250 on LCB/hard by 3.3% over the best single checkpoint at matched sample budget.

核心贡献

  • 发现代码RL中嵌套单元测试覆盖率会产生正确性-效率前沿：更严格的覆盖率减少优化失败（超时/超内存）但增加正确性失败（错误答案），解题率几乎不变
  • 证明线性插值可在两个端点checkpoint之间恢复该前沿，无需完整的RL训练族
  • 首次展示线性外推可将前沿延伸至RL训练未达到的区域，在α ∈ [-0.5, 1.5]范围内解题率保持稳定
  • 该前沿及外推延续在三种推理设置（纯推理、工具使用、智能体编码）和两种模型规模（32B和7B）上均成立
  • 外推checkpoint在问题级别具有互补性：47.2%的困难问题仅在部分α值下被解决，集成外推checkpoint可将LCB/hard的pass@250提升3.3%

Key Contributions

  • Correctness-Efficiency Frontier Discovery: Nested unit-test coverage in code RL induces a frontier where stricter verification reduces optimization failures (timeout/memory) but increases correctness failures (wrong answers), leaving solve rate nearly unchanged on hard problems.
  • Interpolation Recovers Frontier: Linear interpolation between a low-coverage and high-coverage checkpoint reconstructs the same trade-off that otherwise required a full family of independent RL runs.
  • Extrapolation Extends Frontier: Linear extrapolation (α outside [0,1]) continues the frontier beyond what any single RL training run reaches, with stable solve rates across α ∈ [-0.5, 1.5].
  • Cross-Setting Generalization: The frontier and its extrapolative continuation appear across pure reasoning, tool use, and agentic coding settings, at both 32B and 7B model scales.
  • Inference-Time Utility: Extrapolated checkpoints are complementary policies solving different problems. Ensembles across the extended frontier improve pass@250 by 3.3% over the best single checkpoint on LCB/hard at matched sample budget.

Methodology

Training Family: Starting from a shared SFT checkpoint, independent RL runs are trained under nested unit-test coverage thresholds indexed by input-length threshold k: R_k(y) = 1[y passes every test with input length < 10^k]. Lower k accepts more submissions (relaxed); higher k demands asymptotically efficient algorithms (strict). Sweep covers k ∈ {2,...,6,∞}.

Weight Averaging: Linear combination θ_α = α·θ_high + (1-α)·θ_low, where α ∈ [0,1] gives interpolation and α ∉ [0,1] gives extrapolation.

Evaluation: Held-out benchmark is LiveCodeBench (LCB) v5 (easy/medium/hard splits). Outcome taxonomy: correct, optimization failure (resource limit), correctness failure (wrong answer/exception), format error (unparsable).

Models: 32B (CWM-SFT from Meta FAIR) and 7B (Qwen-2.5 7B fine-tuned on OpenCodeReasoning-2 and OpenMathReasoning). Three inference settings: pure reasoning (single-turn), tool use (multi-turn with Python), agentic coding (sandboxed terminal).

Ensemble Evaluation: Matched-budget comparison of (1) best single checkpoint, (2) 5 interpolated checkpoints, (3) 5 checkpoints from interpolation+extrapolation range, using public-test filtering and pass@k metrics.

Results & Highlights

  • Frontier Emergence: On LCB/hard, stricter coverage reduces optimization failures from ~55% to ~15% but increases correctness failures from ~15% to ~55%, with solve rate staying at ~30-35% across all coverage levels.
  • Interpolation Recovery: A single interpolation direction traverses the entire frontier, matching independently trained checkpoints.
  • Extrapolation Range: Solve rate remains flat for α ∈ [-0.5, 1.5]; format errors rise sharply only at extreme |α| > 2 due to token exhaustion.
  • Per-Problem Turnover: 24.4% of hard problems solved at every α, 28.5% never solved, 47.2% solved only on part of the axis — the source of ensemble diversity.
  • Pass@250 Improvement: Extrapolation-inclusive ensemble achieves +3.3% pass@250 over best single checkpoint (95% CI: [0.8%, 6.5%]) on LCB/hard.
  • Transferability: Checkpoint pool selected on LCB/v5 transfers to LCB/v6, becoming beneficial at larger candidate and submission budgets.
  • Frontier Shift: Static reward shaping stays on the frontier; failure-conditioned training (online-fixing) shows exploratory evidence of shifting the frontier outward (fix@1: 14.2% → 17.9% on LCB/hard).

Critical Assessment

Novelty: High. While weight interpolation for multi-objective trade-offs is known (Rewarded Soups), this paper is the first to systematically study extrapolation between RL checkpoints and demonstrate its utility for extending behaviorally meaningful frontiers. The correctness-efficiency frontier itself is a novel empirical finding in code RL.

Rigor: Strong. Experiments span three inference settings, two model scales, and multiple benchmark versions (LCB v5 and v6). Bootstrap confidence intervals are provided. The outcome taxonomy (correct/optimization-failure/correctness-failure/format-error) is well-defined and validated. Ablations on reward shaping vs. failure-conditioned training add depth.

Reproducibility: Good. All base models (CWM-SFT 32B, Qwen-2.5 7B), training datasets (CodeContestsPlus, OpenCodeReasoning-2, OpenMathReasoning), and evaluation benchmarks (LiveCodeBench) are public. Training details are provided. However, training code and checkpoints are not released.

Significance: High for code RL and inference-time scaling. The finding that extrapolation produces complementary policies is practically valuable — it provides a zero-cost way to expand checkpoint diversity without additional training. The frontier analysis offers a new lens for understanding RL training dynamics.

Red Flags: None significant. The main limitation (confined to competitive programming) is acknowledged. The claim about failure-conditioned training shifting the frontier is appropriately marked as "exploratory evidence."

Strengths

  • Clean experimental design with a single controlled axis (verifier strictness) from shared initialization
  • Comprehensive evaluation across inference settings (pure reasoning, tool use, agentic) and model scales (7B, 32B)
  • Insightful per-problem turnover analysis revealing the complementary nature of frontier checkpoints
  • Practical ensemble improvement with zero additional training cost
  • Clear distinction between what moves the frontier (failure-conditioned training) vs. what stays on it (reward shaping)
  • Excellent figures and visualizations throughout

Limitations

  • Confined to competitive programming domain; generalization to SWE tasks, kernel generation, etc. requires identifying comparable ordered axes
  • Input-length thresholds are a coarse proxy for test-case scale
  • Ensemble comparison matches code generation count rather than token or wall-clock cost
  • Stable extrapolation range is narrower at 7B scale
  • No release of training code or model checkpoints

Relevance

该工作与LLM训练稳定性和推理评估高度相关。它揭示了代码RL训练中checkpoint行为的内在前沿结构，为理解RL训练动态提供了新的分析框架。外推权重平均作为一种零成本的推理时扩展策略，对LLM推理评估和集成方法具有直接实践意义。来自Meta FAIR的高质量工作，实验设计严谨。
