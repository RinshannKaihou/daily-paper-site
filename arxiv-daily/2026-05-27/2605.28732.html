MemTrace: Tracing and Attributing Errors in Large Language Model Memory Systems

arXiv: 2605.28732

Date: 2026-05-27

https://arxiv.org/abs/2605.28732

一句话总结： 本文提出MemTrace框架，将LLM记忆管道转化为可执行的记忆演化图，实现细粒度错误追踪与归因，并构建MemTraceBench基准系统研究记忆失败模式，通过归因信号指导提示优化，可将终端任务性能提升最高7.62%。
TL;DR: MemTrace transforms LLM memory pipelines into executable memory evolution graphs for fine-grained error tracing and attribution. With MemTraceBench covering Long-Context, RAG, Mem0, and EverMemOS, it identifies systematic memory failure modes and uses attribution signals to guide prompt optimization, boosting end-task performance by up to 7.62%.

核心贡献

  • 提出将记忆管道转化为可执行记忆演化图的新框架，实现操作级信息流的细粒度追踪
  • 构建MemTraceBench基准，覆盖Long-Context、RAG、Mem0和EverMemOS四种代表性记忆系统
  • 引入自动归因方法，通过迭代追踪操作子图定位任何失败案例的根本原因
  • 发现记忆失败具有系统性，源于信息丢失和检索错位等操作级问题
  • 利用归因信号指导下游提示优化，建立自动纠错的闭环系统

Key Contributions

  • Proposes a novel framework that transforms memory pipelines into executable memory evolution graphs for fine-grained operational information flow tracing
  • Constructs MemTraceBench, a benchmark collected from representative memory systems (Long-Context, RAG, Mem0, EverMemOS) to systematically study memory failure modes
  • Introduces an automatic attribution method that iteratively traces operation subgraphs to pinpoint root causes of failed cases
  • Reveals that memory failures are systematic, stemming from operation-level issues like information loss and retrieval misalignment
  • Leverages attribution signals to guide downstream prompt optimization, establishing a closed-loop system with up to 7.62% end-task performance improvement

Methodology

The framework converts LLM memory system pipelines into executable memory evolution graphs that capture the dynamic flow of information through storage, retrieval, and synthesis operations. An automatic attribution algorithm iteratively traces operation subgraphs to identify root causes of failures. MemTraceBench provides standardized evaluation across four distinct memory architectures. The attribution signals feed into a prompt optimization loop for automatic fault correction.

Results & Highlights

  • Memory failures exhibit systematic patterns across all four tested memory systems
  • Root cause attribution successfully identifies operation-level issues (information loss, retrieval misalignment)
  • Closed-loop prompt optimization guided by attribution signals achieves up to 7.62% performance improvement on end tasks
  • Framework generalizes across diverse memory architectures (Long-Context, RAG, Mem0, EverMemOS)

Critical Assessment

  • Novelty: The problem formulation — error tracing and attribution specifically for LLM memory systems — is novel and timely given the proliferation of memory-augmented LLMs.
  • Rigor: Testing across four distinct memory systems strengthens the generalizability claims. The 7.62% improvement, while modest, is concrete.
  • Reproducibility: Code release is promised. The benchmark covers multiple systems, aiding reproducibility.
  • Significance: As memory-augmented LLMs become standard, debugging tools for memory failures fill an important gap. The closed-loop correction approach is practically valuable.
  • Red flags: The 7.62% improvement upper bound may be dataset/task-dependent. The graph transformation may add overhead that isn't discussed in the abstract.

Strengths

  • Addresses a real and growing pain point: debugging memory-augmented LLM systems
  • Comprehensive benchmark covering four distinct memory architectures
  • Practical closed-loop correction system, not just diagnostic
  • Clear problem formulation for a previously under-studied area

Limitations

  • 7.62% maximum improvement suggests the prompt optimization may have limited corrective power for fundamental failures
  • No discussion of computational overhead for graph construction and tracing
  • Unclear how the framework handles memory systems with non-deterministic retrieval (e.g., embedding-based RAG)
  • The benchmark is collected from specific systems; generalization to future memory architectures is uncertain
