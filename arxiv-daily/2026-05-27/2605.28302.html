How Far Can Disaggregation Go? A Design-Space Exploration of Attention-FFN Disaggregation for Efficient MoE LLM Serving

arXiv: 2605.28302

Date: 2026-05-27

https://arxiv.org/abs/2605.28302

一句话总结： 本文系统性地探索了 MoE 大模型推理中 Attention-FFN 算子级解聚合（AFD）的设计空间，提出 AIC++ 框架融合设备级内核测量与高保真网络仿真，在严格延迟 SLO 下实现 DeepSeek-V3.2 上约 4k tokens/s 的系统吞吐量，而非 AFD 部署方案则不可行。
TL;DR: This paper systematically explores the design space of Attention-FFN Disaggregation (AFD) for MoE LLM serving, proposing the AIC++ framework that fuses on-device kernel measurements with high-fidelity network simulation. Under strict TTFT/TPOT SLOs, AFD sustains ~4k tokens/s throughput on DeepSeek-V3.2 across chat, coding, and agentic workloads where non-AFD deployments are infeasible, while providing concrete guidance on attention-to-FFN GPU partitioning as a function of workload and model architecture.

核心贡献

  • 多维设计空间探索（C1）：首次在 token 级并行（TP/DP/EP/SP/PP）、阶段级 P/D 解聚合、算子级 AFD 三个维度联合优化 LLM 推理调度，并将工作负载特征转化为具体的 Attention-to-FFN GPU 分配比例。
  • 解聚合架构系统建模（C2）：构建 AIC++ 框架，将 NVIDIA AIConfigurator 的算子级计算建模与 AstraSim 网络仿真器结合，精确捕捉解聚合推理系统中计算-通信交互开销。
  • 最优 AFD 算子放置（C3）：证明注意力与 FFN 算子在解聚合基础设施上的精确放置对有效部署至关重要，提出基于通信亲和力的 GPU 放置策略。
  • 微批次重叠技术：提出 4 阶段微批次流水线（Attention → MoE Dispatch → FFN → MoE Combine），在全双工互连上实现计算-通信最大化重叠。
  • 内存分段收益：AFD 通过将大部分模型权重放置于 FFN GPU，在 Attention GPU 上留出更多 KV Cache 内存，在相同内存约束下实现更高吞吐。

Key Contributions

  • Multi-dimensional DSE (C1): First to jointly optimize LLM serving across token-level parallelism (TP/DP/EP/SP/PP), phase-level P/D disaggregation, and operator-level AFD, translating workload characteristics into concrete attention-to-FFN GPU ratios.
  • System Modeling for Disaggregated Architectures (C2): Builds AIC++, combining NVIDIA AIConfigurator for accelerator-level performance modeling with AstraSim for packet-granularity network simulation, capturing compute-communication trade-offs in disaggregated inference.
  • Optimal AFD Operator Placement (C3): Demonstrates that precise placement of attention and FFN operators based on communication affinity (frequent per-layer A2F/F2A on NVLink, less frequent KV transfer on InfiniBand) is critical for effective AFD deployment.
  • Micro-Batch Overlap: Proposes a 4-stage micro-batch pipeline (Attention → MoE Dispatch → FFN → MoE Combine) that maximizes compute-communication overlap on full-duplex interconnects.
  • Memory Segmentation Benefit: AFD places most model weights on FFN GPUs, freeing memory on attention GPUs for KV cache, enabling higher throughput under the same memory constraint.

Methodology

The paper develops AIConfigurator++ (AIC++), a co-design framework with three key components:

  1. Compute Modeling: Uses NVIDIA AIConfigurator to measure operator-level kernel costs (attention, FFN GEMMs, MoE dispatch/combine) on real GPU hardware (B200 SXM). Costs are stored in an empirical database queried during DSE.
  1. Network Simulation: Integrates AstraSim for packet-granularity simulation of scale-up (NVLink) and scale-out (InfiniBand) communication. Models the full bipartite traffic matrix of AFD's M×N attention-FFN pairing, capturing contention, congestion, and full-duplex overlap effects.
  1. Design-Space Exploration: Enumerates every replica size from 2 to 128 GPUs, dynamically composing per-replica parallelism plans across TP, DP, EP, and (for AFD) attention/FFN GPU groups. Includes asymmetric layouts where prefill and decode have different worker sizes.

AFD Communication Pattern: Each transformer layer incurs two cross-AFD transfers:

  • A2F (MoE Dispatch): Fan-out from attention to FFN GPUs (post-attention hidden states, token IDs, routing metadata). FFN-side ingress congestion is dominant.
  • F2A (MoE Combine): Fan-in from FFN to attention GPUs (aggregated expert outputs). Attention-side ingress is the bottleneck.

Micro-Batch Overlap Model: Partitions per-step tokens into M microbatches (M=4 for full-duplex). End-to-end pipelined latency: t_pipe = M · s_max + Σ(s_i / L) for non-bottleneck stages, where s_max is the bottleneck stage cost.

Evaluation Setup:

  • Models: DeepSeek-V3.2 (MLA + Sparse Attention, 256 experts), Qwen3-235B (GQA, 128 experts), GPT-OSS-120B (Full GQA + Window GQA, 128 experts), Nemotron3-120B (Mamba-2 + GQA, 512 experts)
  • Workloads: Chat (ISL=256, OSL=512), Coding (ISL=4096, OSL=1024), Agentic Coding (ISL=8192, Prefix=524k, OSL=256)
  • Hardware: 128× B200 SXM GPUs, TensorRT-LLM backend
  • Prototype: vLLM-based AFD prototype (v0.16.0rc2) with M×N bipartite NCCL pairing for functional correctness verification

Results & Highlights

  • Feasibility under Strict SLOs: On DeepSeek-V3.2 with strict TTFT/TPOT SLOs, only AFD-attention variants unlock feasible configurations, sustaining up to ~4k tokens/s system throughput. Non-AFD deployments are infeasible (marked with red crosses in evaluation).
  • Throughput vs. Latency Trade-off: Aggregated serving (chunked prefill) wins on raw system throughput for most panels by maximizing data-parallel concurrency. AFD wins universally on user interactivity (tokens/s/user) through micro-batch overlapping and workload-aware attention-to-FFN sizing.
  • Attention-to-FFN Ratio follows Rate-Matching: Optimal split is workload- and architecture-dependent:
  • DeepSeek-V3.2 (MLA+DSA, cheap attention): 2A+126F (agentic), 16A+112F (chat) — 90%+ GPUs on FFN
  • Qwen3-235B (dense GQA, heavier attention): 8A+120F (agentic), wider attention slice as TTFT tightens
  • GPT-OSS-120B (sliding-window GQA): near-symmetric 16A+16F on chat — the only panel where AFD wins throughput
  • Nemotron3-120B (Mamba-2): attention-heavy 96A+32F due to recurrent state propagation cost
  • Long Context (500k ISL): Agg+AFD M4 achieves 1346.5 tok/s at 64 GPUs and 2693.0 tok/s at 128 GPUs, with 7:1 attention-to-FFN ratio.
  • Memory Segmentation: For Prefix=1M workloads, non-AFD modes are infeasible (per-GPU memory ~298 GiB exceeds B200's 180 GiB). AFD reduces to ~165 GiB per GPU by splitting weights/activations across attention and FFN groups, achieving 921.5 tok/s at 64 GPUs and 1858.9 tok/s at 128 GPUs.
  • GPU Placement: Frequency-driven placement — per-layer A2F/F2A traffic on intra-node NVLink (highest bandwidth), less frequent KV-cache transfer on inter-node InfiniBand. Paired P&D per node outperforms segregated phase layout.

Critical Assessment

  • Novelty: The paper provides the first systematic, multi-dimensional DSE of AFD for MoE serving, going beyond prior work (MegaScale-Infer) that identified the problem but didn't explore the full design space. The integration of AIConfigurator + AstraSim into AIC++ is a novel contribution. The rate-matching principle for attention-to-FFN ratio is an insightful and actionable finding. Rating: Moderate-High — the individual components (P/D disaggregation, AFD, simulation) exist, but the systematic exploration and integration is novel.
  • Rigor: The evaluation covers 4 diverse model architectures (MLA, GQA, sparse attention, Mamba-2), 3 workload types, and both short and long context regimes. The framework is grounded in real kernel measurements and validated with a vLLM prototype. However, cluster-scale results are model-based DSE estimates rather than end-to-end measurements on 128 GPUs, which limits empirical confidence. Rating: Moderate — strong modeling methodology but limited real-cluster validation.
  • Reproducibility: The vLLM prototype is based on an open-source PR (#29772), and AstraSim is publicly available. AIConfigurator is from NVIDIA but may not be fully open-source. The DSE methodology is well-documented with clear equations and search strategies. Rating: Moderate — some components may be hard to replicate without NVIDIA's internal tools.
  • Significance: Highly relevant for current and near-future LLM serving infrastructure. As MoE models (DeepSeek-V3, Qwen3) become dominant and heterogeneous accelerators (NVIDIA Groq 3 LPX, Intel SambaNova) emerge, the design principles from this paper are directly actionable for cluster-scale deployments. Rating: High.
  • Red Flags: Results on DeepSeek-V3.2 use "V3.2" which may not be publicly available, and GPT-OSS-120B appears to be a hypothetical model. The 524k context length for some models is noted as potentially exceeding native support. The paper assumes full-duplex interconnects throughout, which may not hold for all deployment scenarios.

Strengths

  • Comprehensive multi-dimensional DSE covering parallelism, P/D, and AFD — no prior work explores all three jointly.
  • Actionable rate-matching principle that translates model architecture and workload into concrete GPU allocation ratios.
  • Strong practical relevance: addresses real production challenges for MoE serving at rack/cluster scale.
  • Thorough treatment of communication patterns (bipartite A2F/F2A, congestion analysis, placement optimization).
  • Memory segmentation benefit is a novel and practical insight for long-context MoE serving.
  • Evaluation across 4 diverse architectures (MLA, GQA, sparse, Mamba-2) demonstrates generality of findings.

Limitations

  • Cluster-scale results are simulation-based (AIC++ modeling) rather than measured on real 128-GPU clusters, introducing potential accuracy gaps.
  • Some evaluated models (DeepSeek-V3.2, GPT-OSS-120B) may not be publicly available or fully reproducible.
  • The paper does not evaluate AFD on non-MoE (dense) models, limiting the scope of conclusions.
  • Cost model assumes full-duplex NVLink/InfiniBand throughout; half-duplex or lower-bandwidth interconnects are only briefly discussed.
  • No comparison with other recent MoE serving systems (e.g., preplit, Mooncake) beyond MegaScale-Infer.
  • The DSE search is exhaustive but computationally expensive; the paper doesn't discuss the time cost of running AIC++ for new configurations.
  • Missing ablation on micro-batch count (M) sensitivity — only M=4 is used for full-duplex.

Relevance

本文对 MoE 大模型推理系统的设计与部署具有高度实际指导意义。随着 DeepSeek-V3、Qwen3 等 MoE 模型成为主流，以及 NVIDIA Groq 3 LPX 等异构推理平台的出现，AFD 及其设计空间探索框架 AIC++ 为集群级部署提供了可直接落地的调度策略和 GPU 分配原则，尤其在长上下文和严格延迟约束场景下价值突出。

Questions for Further Exploration

  • Can the rate-matching principle be automated into a runtime scheduler that dynamically adjusts attention-to-FFN ratios based on observed workload shifts?
  • How does AFD interact with speculative decoding or other latency-reduction techniques for MoE models?
  • What is the sensitivity of AIC++ predictions to inaccuracies in AstraSim's network modeling (e.g., congestion underestimation at scale)?
