Hurwitz Quaternion Multiplicative Quantization for KV Cache Compression

arXiv: 2605.27646

Date: 2026-05-27

https://arxiv.org/abs/2605.27646

一句话总结： 提出HQMQ——一种基于Hurwitz四元数乘法组合码本的KV缓存量化方法，无需校准数据即可在约5比特下匹配fp16质量，并将Llama-3-70B的128k上下文缓存从43GB压缩至8.5GB。
TL;DR: HQMQ is a calibration-free KV cache quantization method that treats each 4-element chunk as a quaternion and uses multiplicative composition of the 24-element Hurwitz group with random secondary codebooks to achieve near-fp16 quality at ~5 bits, Pareto-dominating naive integer quantization by 3–1900× across five models from four vendors.

核心贡献

  • 乘法码本组合规则： 利用Hurwitz群2T（S³上24-cell的24个顶点）的乘法结构，将方向码本定义为 C = {q_p · q_s}，仅存储S个随机四元数即可生成24S个有效码字。
  • 免校准部署： 由于四元数左乘是S³等距变换，随机初始化即可产生近均匀的24S点填充（Proposition 3.1证明其覆盖半径达到最优S³速率），5个种子间困惑度变异系数 ≤0.14%。
  • 中位数乘子离群值提取（Med3×）： 每批次以C=3倍中位数为阈值提取离群值，无需校准数据，在Qwen2.5-7B上将naive int4的18,079困惑度降至9.69（约1900倍质量提升）。
  • 跨架构验证： 在Mistral-7B（MHA）、Llama-3-8B/Qwen2.5-7B/Qwen3-8B（GQA）、gpt-oss-20b（稀疏MoE）五个模型上全面Pareto优于朴素整数量化。

Key Contributions

  • Multiplicative codebook composition: Uses the Hurwitz group 2T (24 vertices of the 24-cell on S³) with per-(layer, head) random secondary codebooks, yielding 24S effective codewords from only S stored parameters.
  • Calibration-free deployment: Left-multiplication by a unit quaternion is an S³ isometry, so random Haar-uniform initialization produces near-optimal S³ covering (Proposition 3.1). Perplexity variation across 5 seeds is ≤0.14%.
  • Median-multiplier outlier extraction (Med3×): A per-batch C=3× median threshold handles outlier-heavy architectures without calibration. On Qwen2.5-7B, reduces naive int4's 18,079 ppl to 9.69 (~1900× improvement).
  • Broad empirical validation: Pareto-dominates naive int quantization across 5 models from 4 vendors (Mistral AI, Meta, Alibaba, OpenAI), including dense MHA, dense GQA, and sparse MoE architectures.

Methodology

Quaternion Chunking: Each K/V vector is chunked along the head dimension into groups of 4, treated as quaternions x ∈ H. Radius r = ||x|| and unit direction u = x/r ∈ S³ are quantized separately.

Multiplicative Codebook: The primary codebook P = 2T (24 Hurwitz units, fixed). A secondary codebook S_{l,h,m} of S random unit quaternions is sampled per (layer, head, K/V role). The joint codebook is the product set C = {q_p · q_s : q_p ∈ P, q_s ∈ S}, with up to 24S entries on S³. Direction encoding: u_q = argmax_{c ∈ C} ⟨u, c⟩.

Random Initialization Guarantee (Prop 3.1): For i.i.d. Haar-uniform q_s, the covering radius E[ρ(C)] = O((24S)^{-1/3}), matching the optimal S³ covering rate. This is data-independent, eliminating the need for calibration.

Radius Quantization: Uniform scalar quantizer with per-token-max scale at b_r bits (sweet spot b_r ∈ [3,6]).

Med3× Outlier Extraction: Per-batch, elements exceeding C=3× the per-channel median magnitude are extracted and stored in fp16. No calibration data needed; C=3 transfers across all tested architectures.

Storage: ⌈log₂(24S)⌉-bit index + b_r-bit radius + per-token fp16 scale → (log₂(24S) + b_r)/4 + 16/d_h bits per element.

Critical Assessment

Novelty: The use of the Hurwitz group 2T's multiplicative structure for codebook composition is genuinely novel in the KV quantization context. The connection to S³ isometries providing calibration-free guarantees is mathematically elegant. Distinct from prior additive VQ (CommVQ) and flat learned VQ (VPTQ).

Rigor: Strong empirical evaluation across 5 models from 4 vendors covering 3 architecture families (dense MHA, GQA, sparse MoE). Includes ablation studies, seed-robustness tests, downstream task evaluation, and head-to-head comparison with KIVI. Proposition 3.1 provides theoretical grounding. However, the paper is a preprint and has not undergone peer review.

Reproducibility: Method description is detailed with pseudocode in appendix. All models are publicly available. The method is calibration-free, which inherently simplifies reproduction.

Significance: High practical impact — KV cache compression is a critical bottleneck for long-context LLM deployment. Achieving fp16-matching quality at ~5 bits without calibration is a meaningful advance. The 5.05× compression ratio on Llama-3-70B 128k context is directly useful.

Red Flags:

  • Qwen2.5-7B still shows a 16% ppl gap at best config (8.83 vs 7.59), suggesting outlier-heavy architectures remain partially challenging even with Med3×.
  • gpt-oss-20b results show higher absolute ppl (460 vs 447), though relative gap is small.
  • C=3 threshold is empirically chosen without theoretical universality guarantee (authors acknowledge this).
  • Downstream evaluation uses only n=200 examples/task, which may have high variance.

Strengths

  • Calibration-free design with mathematical justification (Prop 3.1) — eliminates a major practical friction point.
  • Multiplicative codebook composition is elegant and storage-efficient (24S codewords from S stored parameters).
  • Med3× outlier extraction is simple, architecture-agnostic, and highly effective (1900× improvement on Qwen2.5).
  • Comprehensive cross-vendor, cross-architecture evaluation (5 models, 3 arch families).
  • Strong Pareto dominance over baselines at every bit budget.
  • Practical storage results (5.05× compression on real-world deployment scenario).

Limitations

  • Preprint status — not yet peer-reviewed.
  • Outlier-heavy architectures (Qwen2.5, Qwen3) still show larger gaps to fp16 compared to non-outlier models.
  • C=3 outlier threshold lacks theoretical justification for universality across all possible architectures.
  • Downstream evaluation limited to n=200 examples per task; full-scale evaluation would strengthen claims.
  • Radius quantization uses simple uniform scalar quantizer; more sophisticated approaches might improve results.
  • No comparison with some recent concurrent methods (IsoQuant, RotorQuant) mentioned but not benchmarked against.

Relevance

本文提出的HQMQ方法对LLM长上下文推理的部署有直接实用价值——无需校准数据即可在约5比特下实现接近fp16的KV缓存质量，且码本设计具有数学优雅性。对于关注模型压缩、量化、以及高效推理的研究者和工程师具有重要参考价值。
