一句话总结： 提出LoRP框架，通过分析LLM层间表示相似性的局部性特征，实现自适应的无训练深度剪枝压缩。
TL;DR: LoRP is a training-free depth pruning framework that analyzes representation locality across LLM layers to adaptively allocate pruning budgets based on architecture-specific redundancy patterns.

核心贡献

  • 发现表示局部性差异：揭示不同LLM架构的层间冗余模式存在显著差异——某些模型（如LLaMA、Mistral）呈现局部化冗余，而另一些（如Qwen）则表现出全局分布的冗余特性
  • 提出表示局部性分数（RLS）：设计了一个简单有效的指标，通过全局层间隐藏状态相似度量化表示局部性特征
  • 开发LoRP剪枝框架：提出基于谱聚类和两阶段冗余分配的无训练一次性深度剪枝方法，根据各架构的冗余结构自适应分配剪枝预算
  • 跨架构验证：在多个LLM家族上验证了LoRP在困惑度和下游推理任务上的一致性改进

Key Contributions

  • Representation Locality Discovery: Reveals that inter-layer redundancy patterns vary substantially across LLM families - some architectures (LLaMA, Mistral) exhibit localized redundancy while others (Qwen) show globally distributed similarity patterns
  • Representation Locality Score (RLS): Introduces a simple metric derived from global off-diagonal inter-layer hidden-state similarity to quantify representation locality without assuming specific redundancy structures
  • LoRP Framework: Proposes Locality-Aware Redundancy Pruning, a training-free one-shot depth pruning method that clusters layers via spectral clustering and allocates pruning through a two-stage redundancy-aware procedure
  • Cross-Architecture Validation: Demonstrates consistent improvements in both perplexity and downstream reasoning performance across diverse LLM families without post-pruning recovery

Methodology

Inter-Layer Similarity Analysis:

  • Collects input hidden-state activations from every Transformer block using forward hooks on a small calibration corpus
  • Computes pairwise layer similarity matrix S by averaging token-wise cosine similarity across calibration samples
  • Normalizes hidden representations to enable meaningful cross-layer comparisons

Representation Locality Score (RLS):

  • Calculates global off-diagonal similarity mean across all layer pairs
  • Applies logarithmic transformation: RLS(S) = -log₂(S̄_off)
  • Higher RLS indicates localized redundancy; lower RLS indicates distributed redundancy

Layer Clustering:

  • Converts similarity matrix to affinity matrix: A = (S + 1) / 2
  • Applies spectral clustering with precomputed affinity to partition layers into K representational clusters
  • RLS guides partition granularity: lower RLS → finer clusters, higher RLS → coarser clusters

Two-Stage Pruning Allocation:

  1. Coverage-aware Initialization: Removes one most-redundant layer per cluster to ensure representational coverage
  2. Residual Redundancy Allocation: Iteratively allocates remaining pruning budget based on residual cluster redundancy scores

Results & Highlights

  • Architectural Diversity: Experiments across LLaMA-3.1-8B, OLMo-3-7B, Mistral-Nemo-12B, Qwen3-8B, and Qwen3-14B demonstrate architecture-dependent redundancy patterns
  • RLS Values: Qwen models show lower RLS (0.644-0.685) indicating distributed redundancy, while LLaMA shows higher RLS (1.149) indicating localized redundancy
  • Consistent Improvements: LoRP achieves better perplexity and downstream task accuracy compared to baseline one-shot pruning methods
  • Training-Free Efficiency: Entire pruning process requires no fine-tuning or post-pruning recovery, making it computationally efficient
  • Architecture-Adaptive Behavior: Produces pruning patterns that reflect each architecture's specific redundancy structure rather than applying fixed heuristics

Critical Assessment

Novelty:

  • Introduces the novel concept of representation locality for depth pruning, moving beyond fixed heuristics
  • RLS metric provides architecture-dependent guidance rather than universal layer rankings
  • Two-stage allocation balances coverage and redundancy in a principled way

Technical Rigor:

  • Solid mathematical formulation with clear definitions of similarity metrics and redundancy scores
  • Comprehensive experimental validation across multiple LLM families
  • Ablation studies and comparisons with existing one-shot pruning baselines

Reproducibility:

  • Method is training-free and requires only a small calibration dataset
  • Clear algorithmic description with explicit equations for all components
  • Uses standard spectral clustering and cosine similarity measures

Significance:

  • Addresses practical need for efficient LLM compression without expensive retraining
  • Reveals fundamental insights about architecture-dependent redundancy patterns
  • Provides actionable framework for adaptive model compression

Potential Limitations:

  • Relies on calibration dataset choice for similarity estimation
  • Spectral clustering introduces hyperparameter K for number of clusters
  • Does not explore combination with other compression techniques (quantization, distillation)

Strengths

  • Architecture-Aware Design: Unlike methods assuming uniform redundancy structures, LoRP adapts to each model's specific representation geometry
  • Training-Free Efficiency: Entire compression pipeline requires no gradient computation or parameter updates
  • Principled Framework: Well-grounded in representation similarity analysis with clear mathematical formulation
  • Practical Applicability: One-shot pruning directly reduces inference latency and memory without specialized hardware requirements
  • Comprehensive Validation: Experiments across diverse LLM families demonstrate generalizability

Limitations

  • Calibration Dependency: Performance may vary based on calibration dataset selection and size
  • Clustering Hyperparameters: Number of clusters K requires tuning and may not be optimal for all architectures
  • No Post-Pruning Recovery: While training-free, the method doesn't leverage potential gains from lightweight fine-tuning
  • Limited Scope: Focuses exclusively on depth pruning without exploring width pruning or hybrid approaches
  • Evaluation Metrics: Primarily evaluates perplexity and reasoning tasks; broader downstream evaluation would strengthen claims
