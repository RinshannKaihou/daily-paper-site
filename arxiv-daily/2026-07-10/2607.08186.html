Hidden Decoding at Scale: Latent Computation Scaling for Large Language Models

arXiv: 2607.08186

Date: 2026-07-09

https://arxiv.org/abs/2607.08186

一句话总结： WeChat AI 提出 Hidden Decoding，在固定 Transformer 主干的前提下，通过将每个 token 展开为 n 个流（独立嵌入表 + 中间流 KV 持久化）并配合 Stream-Factorized Attention（大部分层只做流内注意力）将扩展注意力代价从 O(n²L²) 降到近似线性，在 CPT 阶段训练出 WeLM-HD4-80B 与 WeLM-HD4-617B，在全部九项难基准上击败匹配的非 HD 基线，首次将"序列长度扩展"作为固定主干缩放路径推到 100B+ MoE 规模。
TL;DR: The WeChat AI team proposes Hidden Decoding (HD), a continued-pretraining method that expands each token into n streams with independent embedding tables and retains intermediate-stream KV as persistent context; combined with Stream-Factorized Attention (most layers attend only within a stream), it keeps the n-stream expansion cost near-linear instead of quadratic. At n=4 it produces WeLM-HD4-80B and WeLM-HD4-617B, which beat their matched non-HD baselines on all nine shared hard benchmarks (e.g., WeLM-HD4-80B SciCode 45.8 → 50.0, FrontierMath 45.8 → 49.0; WeLM-HD4-617B GPQA Diamond 89.1 → 91.2), making this the first demonstrated sequence-length scaling method at the 100B+ MoE scale, with training cost only 5.1× (80B) / 4.4× (617B) per batch versus 16× for dense attention.

核心贡献

  • 前沿规模下的固定主干缩放：首次在 100B+ MoE 规模上验证"序列长度扩展"作为缩放路径，产出 WeLM-HD4-80B（3B 激活）和 WeLM-HD4-617B（23B 激活）两个模型，突破了深度循环（looped）方法因与流水线并行不兼容而长期停滞在 ~40B 的瓶颈。
  • 高效的序列长度扩展机制：提出 Stream-Factorized Attention，让大部分层只做流内注意力，仅少数层做跨流混合，使注意力开销从 O(n²L²) 降到近似线性；4× 扩展在 80B/617B 上的每批训练代价仅 5.1×/4.4×，远低于稠密注意力的 16×。
  • 多流 token 展开设计：用 n 个独立嵌入表把每个 token 展开为 n 个流，仅监督最终流预测下一个 token，前 n−1 个流作为"潜在计算状态"保留其 KV 供后续 token 读取；监督设计的消融证明这种"仅最终流监督"优于全流监督和输出求和。
  • 扩展因子作为缩放旋钮：在稠密 Qwen3-8B-Base 与 80B MoE 上从 n=2 扩到 n=8，准确率与语言建模 loss 随 n 单调改善（80B MMLU 85.1 → 87.5，Pile-test BPB 0.386 → 0.378），验证 n 是固定主干下可靠的缩放维度。
  • 渐进式扩展与可解释性探针：提出 Cyclic Replication Initialization（1→2→4→8）节省训练；通过 hidden-state 相似度、注意力亲和度、LM-head 探针等分析揭示中间流确实形成可被最终流读取的独立状态（中间层 cosine 从 0.987 降到 0.637）。

Key Contributions

  • Frontier-scale fixed-backbone scaling. First demonstration of sequence-length scaling at the 100B+ MoE scale via continued pretraining, producing WeLM-HD4-80B (3B activated) and WeLM-HD4-617B (23B activated) — a regime that depth-recurrent (looped) models could not reach because looping breaks pipeline parallelism.
  • Efficient sequence-length expansion via Stream-Factorized Attention. Most layers attend only within each stream; cross-stream mixing is confined to a small subset of layers, holding the attention cost near-linear in n. A 4× expansion costs 5.1× (80B) and 4.4× (617B) per batch, far below the 16× dense-attention baseline.
  • Multi-stream token expansion with persistent intermediate KV. Replaces the single embedding table with n per-stream tables; only the final stream is supervised, while the first n−1 streams act as latent computation states whose KV is retained as context for later tokens.
  • Expansion factor n as a usable scaling knob. Progressive expansion from n=2 to n=8 monotonically improves LM loss and downstream accuracy on both a dense 8B model and an 80B MoE, validating n as a practical scaling axis for a fixed Transformer backbone.
  • Interpretability probes. Hidden-state similarity, attention-affinity, and LM-head probes show intermediate streams form distinct states (mean cosine drops from 0.987 to 0.637 mid-network) that the final prediction stream actively reads (E7 attention to E0 peaks at 0.52).

Methodology

Setting. The goal is to improve an already-strong checkpoint by giving each token more internal computation without enlarging the Transformer backbone (no new or wider layers), via continued pretraining (CPT).

Multi-Stream Token Expansion (§2.1). A standard model maps each token x_i through a single embedding table E. Hidden Decoding instead uses n independent embedding tables E_1, …, E_n and forms an expanded sequence S of length nL by placing the stream-k representation of token x_i at physical position t = (i−1)n + k. So for n=2 the sequence becomes (E_1(x_1), E_2(x_1), E_1(x_2), E_2(x_2), …). This length-nL sequence is fed to the same backbone under standard causal attention with contiguous RoPE positions. The next-token cross-entropy loss is applied only at the final-stream positions t = i·n; the first n−1 streams receive no loss and serve as intermediate latent-computation states. Because attention is causal, the final stream attends to all earlier streams of the same token and of preceding tokens.

Stream-Factorized Attention (§2.2). Dense attention over the nL expanded positions would cost O(n²L²). To keep it affordable, most layers are made intra-stream: they restrict attention to earlier positions of the same stream (cost O(nL²)). Only a small subset of layers are cross-stream layers that mix across streams, following the base model's native pattern (sliding-window layers stay sliding-window cross-stream; full-attention layers stay full cross-stream). No new layers are added — the base model's own attention layers are reused as the cross-stream layers. This keeps the added attention cost near-linear in n rather than quadratic. For the 80B model (49 layers): 6 full + 20 intra-stream + 23 sliding-window; for the 617B model (94 layers): 25 full + 69 intra-stream.

Progressive Expansion (§2.3). Rather than training a large n directly, the factor is grown 1→2→4→8 via Cyclic Replication Initialization: when doubling from n to 2n, new streams copy existing tables (E_{n+k} ← E_k), so the expanded model initially reproduces the smaller one's behavior before adapting.

WeLM-specific KV-mirror optimization (§3.1). The WeLM backbones use a U-shaped KV-mirror design where later layers compute keys/values from earlier layers' hidden states. Since intermediate streams carry no loss and their context is already fixed by early layers in mirrored layers, only the final stream is run through the mirrored (last-third) layers. This gives an extra ~20% per-batch speedup specific to WeLM.

Training setup. HD is introduced during CPT from a baseline checkpoint using the same data and schedule as the non-HD run, with the expansion on from that point onward through the long-context stage and the early SFT-only post-training. The matched non-HD baseline is trained identically with the expansion off, so the comparison isolates HD. HD4-only token budgets are only ~5.3% of the full non-HD base path at both scales.

Results & Highlights

  • WeLM-HD4-80B beats matched WeLM-80B on all nine shared hard benchmarks (early SFT-only, no RL). Largest gains on hard math/science: SciCode 45.8 → 50.0 (+4.2), PHYBench 69.8 → 73.8 (+4.0), FrontierMath 45.8 → 49.0 (+3.2), MathArena Apex 16.4 → 20.1 (+3.7), GPQA Diamond 87.6 → 88.8 (+1.2), HLE 27.4 → 28.4 (+1.0), MMMLU 84.4 → 85.6 (+1.2), HMMT 93.3 → 94.1 (+0.8), IMO-AnswerBench 85.0 → 85.3 (+0.3).
  • WeLM-HD4-617B beats matched WeLM-617B on all nine shared benchmarks: GPQA Diamond 89.1 → 91.2 (+2.1), HLE 33.6 → 35.4 (+1.8), FrontierMath 49.0 → 51.0 (+2.0), MMMLU 86.4 → 87.5 (+1.1), PHYBench 75.3 → 76.3 (+1.0), SciCode 51.4 → 52.1 (+0.7), MathArena Apex 24.2 → 24.7 (+0.5). WeLM-HD4-617B is higher than the 1T/32B Kimi K2.6 reference on GPQA, PHYBench, MathArena Apex, HMMT, and SciCode.
  • Base-model comparison vs. Qwen3.5-397B-A17B / DS-V3.2-671B-A37B: WeLM-HD4-617B-Base reaches MMLU 90.22 (vs. WeLM-617B-Base 89.85), MMLU-Pro 73.92 (vs. 72.26), MMLU-Redux 90.94 (vs. 89.81), SuperGPQA 57.05 (vs. 55.33), CRUX-Input 91.75 (vs. 86.00), CRUX-Output 91.13 (vs. 89.38).
  • Training cost stays near-linear: a 4× expanded sequence costs 5.1× per batch on 80B (256k→1M) and 4.4× on 617B (32k→128k), close to the 4× linear reference and far below the 16× dense-attention baseline.
  • Expansion-factor scaling (Qwen3-8B-Base): mean improvement over baseline grows steadily with n; largest single gains reach +5.6 on HellaSwag and +5.1 on BBH and MATH. At 80B MoE scale, MMLU rises monotonically 85.1 → 86.7 → 87.5 as n goes 1→4→8, and Pile-test BPB falls 0.386 → 0.378.
  • Attention-composition ablation (21B MoE, 27 layers): SF with only 4 full cross-stream layers nearly matches the all-full-27 variant (avg 56.49 vs. 56.95), and SF with just 1 full layer still improves over the no-expansion baseline (55.88 vs. 54.33). A few full layers suffice.
  • Supervision-design ablation (6B MoE, n=2): HD final-stream-only loss gives lowest LM loss 1.874 vs. all-token-loss 1.880 and sum 1.877; HD is best on all four downstream benchmarks (e.g., ARC-C 74.3 vs. 71.9 sum vs. 65.8 all-token). HD at n=3 further lowers loss to 1.857.
  • Serving throughput (WeLM-HD4-80B, 8×H20): at batch size 1, HD retains 83–88% of baseline TPS across input buckets; through batch 4 it retains 69–88% for short/2k/8k inputs. Cost grows for large-batch/long-input (32k input at batch 32 keeps 27%).
  • KV-mirror training speedup: 80B 32k per-batch time drops 15s → 12s, a 20% reduction (1.25× speedup).

Authors & Credibility

  • Authored by the WeChat AI Team (Tencent), with code released at github.com/Tencent/Sequential-Hidden-Decoding and model weights on Hugging Face (huggingface.co/collections/tencent/sequential-hidden-decoding). This is a large industrial team with direct access to frontier-scale MoE training infrastructure, which is a prerequisite for the 80B/617B experiments reported.
  • The work directly extends Tencent's WeLM MoE model family (which uses a KV-mirror architecture), and is positioned against prior academic looped-model work (Geiping et al. recurrent depth; LoopCoder ~40B) that the authors correctly note could not reach this scale.
  • The paper includes matched-pair controlled comparisons (same backbone, same data, same early SFT recipe), explicit token-budget tables, and an external-model reference table (Qwen3.5, DS-V3.2, Kimi K2.6) for absolute-score context — a level of transparency unusual for an industrial scaling paper.

Critical Assessment

  • Novelty: High and specific. The key engineering insight — that sequence-length expansion stays compatible with pipeline parallelism while depth-recurrence does not — is well-argued, and the Stream-Factorized Attention design (intra-stream for most layers, cross-stream for a few) is a concrete, non-obvious mechanism for keeping expansion near-linear. The combination of per-stream embedding tables + final-stream-only supervision + persistent intermediate KV is distinct from prior PHD (which makes intermediate KV transient) and from looped transformers.
  • Rigor: The matched-pair design is the strongest aspect: non-HD and HD models share backbone, data, and post-training recipe, and the HD4-only token budget is only ~5.3% of the base path, making the gains a credible lower bound (the 617B HD run even uses fewer 256k-continuation tokens than non-HD). The supervision-design ablation (all-token loss vs. sum vs. final-stream-only) and the small KV-retention ablation directly test the central design claims.
  • Weaknesses in rigor: (1) The headline 617B comparison uses an "early SFT-only" post-training with no RL — the authors repeatedly caveat that "mature WeLM release scores are outside the scope of this paper," so the absolute numbers are deliberately under-trained and the gains may not survive a full RL pipeline. (2) The KV-retention ablation (Table 6) is run only at n=2 on a 21B MoE and the authors themselves label it "qualitative" with "noisy per-task columns"; a larger n and scale would be needed to be confident that per-stream KV — versus simply longer sequences — is the causal driver. (3) There is no scaling-law fit (no loss-vs-n or loss-vs-compute curve extrapolation), so it is unclear whether gains continue past n=8 or saturate, and whether the per-token compute is being traded off efficiently against just training a larger backbone. (4) The baseline is internal WeLM; there is no head-to-head at matched total compute against an alternative CPT recipe (e.g., simply continuing non-HD training on more tokens), so the value of HD per unit of added compute is not isolated against the simplest competitor.
  • Reproducibility: Code and weights are released, which is strong. However, reproducing the 80B/617B results requires frontier-scale MoE training infrastructure beyond almost all academic groups; the released artifacts mainly enable use/fine-tuning, not independent verification of the scaling claims. The small-model ablations (6B/21B/Qwen3-8B) are reproducible and are the most checkable part.
  • Significance: If the n-scaling trend holds, this opens a genuinely new (non-backbone) scaling axis for large MoE models — potentially important for an industry where re-pretraining a bigger backbone is the dominant cost. The compatibility with standard pipeline-parallel training is the practically decisive contribution over looping.
  • Red flags: The deliberate use of "early SFT-only, no RL" for the headline table, combined with a footnote pointing to a public report of a newer non-HD WeLM MoE after larger-scale post-training, means a casual reader could over-read the absolute frontier scores; the real claim is the matched-pair delta, not that WeLM-HD4-617B is SOTA. Also, embedding parameters grow large (WeLM-HD4-617B embedding tables grow 26.9B → 107.4B at n=4), which the authors correctly separate from "active parameters" but which is real memory/storage cost not counted in the headline parameter figures.

Strengths

  • First method to demonstrate sequence-length-based latent computation scaling at genuine frontier MoE scale (617B), a regime where all prior looped/length-scaling work stalled at ≤40B.
  • Stream-Factorized Attention is an elegant, well-motivated mechanism: reusing the base model's own sliding-window/full layers as the cross-stream layers keeps CPT close to the pre-expansion checkpoint and yields a near-linear (not quadratic) cost in n.
  • Strong controlled experimental design: matched non-HD vs. HD pairs sharing backbone, data, and SFT recipe; HD-only token budget reported explicitly and is small (~5.3%); gains hold across nine diverse hard benchmarks and both 80B and 617B scales.
  • Rich interpretability analysis (hidden-state cosine, attention affinity, LM-head probes) that goes beyond accuracy to argue why intermediate streams help — they form distinct, lower-entropy-consolidating states that the final stream reads.
  • Honest about scope: repeated caveats that scores are early-SFT comparisons and that mature release scores are out of scope; releases both code and weights.

Limitations

  • Headline results use an intentionally lightweight early SFT-only post-training with no RL, so absolute frontier scores are not directly comparable to fully post-trained frontier models; the durability of gains under full RL is unverified in the paper.
  • The causal evidence that persistent per-stream KV (vs. transient KV as in PHD) is the key driver rests on a single small, n=2, 21B ablation that the authors themselves call qualitative/noisy.
  • No scaling-law extrapolation: it is unknown whether accuracy keeps improving past n=8 or where the expansion saturates, and no comparison against spending the equivalent extra compute on simply training a larger backbone or more tokens.
  • Serving cost becomes substantial outside the small-batch/short-input regime (e.g., 32k input at batch 32 retains only 27% of baseline TPS), so HD is not a free lunch for high-throughput serving.
  • Embedding parameters grow linearly with n (107.4B embeddings for WeLM-HD4-617B at n=4), adding real memory/storage footprint that is excluded from the headline "active parameter" comparisons.
  • The WeLM-specific KV-mirror decoding optimization is not integrated into the reported serving benchmark, so real-world serving numbers may differ.

Relevance

This work is highly relevant to research on fixed-backbone / latent-computation scaling of large MoE LLMs: it offers a concrete, pipeline-parallel-compatible alternative to depth-recurrent (looped) transformers and demonstrates it at 100B+ scale. 对关注"如何在不重新预训练更大主干的前提下持续提升前沿模型能力"以及"测试期/内部计算缩放"方向的研究者，Hidden Decoding 提供了一个工程上可落地、且与现有大规模 MoE 训练栈兼容的新缩放轴，值得作为基线进行对比与扩展。

Questions for Further Exploration

  • Does the HD gain survive a full RL post-training pipeline (e.g., GRPO/RLHF), and does the n=4 delta over non-HD widen, narrow, or disappear under the same RL budget?
  • Where does the expansion factor saturate? Is there a predictable scaling law relating validation BPB to n (or to added per-token compute) at fixed backbone, and how does its compute-efficiency compare to scaling backbone parameters?
  • At matched additional training compute, how does HD compare to the simplest alternative — continuing non-HD CPT on more tokens — on the same benchmarks? The current design isolates HD at matched tokens, not matched compute.
  • Can Stream-Factorized Attention's cross-stream layer count be learned or pruned automatically rather than set by hand (6 full for 80B, 25 for 617B)? Is there a principled placement rule?
  • Does the intermediate-stream "latent computation" interpretation hold up under causal-intervention probes (e.g., ablating specific streams' KV at inference) rather than correlational attention-affinity measurements?
  • How do the serving-throughput penalties interact with speculative decoding or KV-cache compression, and can the WeLM-specific KV-mirror decoding optimization close the large-batch gap?
  • Since embeddings balloon with n (107.4B for 617B at n=4), can the per-stream tables be tied, factorized, or shared to recover memory without losing the diversity that drives the gains?
