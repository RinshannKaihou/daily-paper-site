WebSwarm: Recursive Multi-Agent Orchestration for Deep-and-Wide Web Search

arXiv: 2607.08662

Date: 2026-07-10

https://arxiv.org/abs/2607.08662

一句话总结： WebSwarm 把复杂网页搜索建模为渐进式递归委托，每个搜索节点绑定局部目标与搜索模式（atom/deep/wide/entity_collect），并用 Web 信息结构探测和兄弟节点经验复用来引导扩展；在 BrowseComp-Plus、WideSearch、DeepWideSearch、GISA 四个基准上全面超过 ReAct 单智能体与多种多智能体基线（GLM-4.5 上 BrowseComp-Plus 准确率 +17.50 个百分点）。
TL;DR: WebSwarm reframes complex web search as progressive recursive delegation: each dynamically instantiated search node couples a local objective with one of four search modes (atom, deep, wide, entity_collect), and a lightweight web-structure probe plus cross-sibling experience reuse guide node expansion. On BrowseComp-Plus, WideSearch-EN, DeepWideSearch-EN, and GISA (all with GLM-4.5), it beats ReAct by +17.50 ACC, +9.76 Item-F1, +11.77 Item-F1, and +6.76 overall, and tops every multi-agent baseline tested, with the largest gains on hard samples (BC-Plus Hard: 0.0 → 35.7).

核心贡献

  • 提出渐进式递归委托框架：搜索节点（agent）在推理时动态生成，每个节点同时绑定局部目标 q_v 与搜索模式 m_v（objective–mode pair），让任务分解、递归扩展与协作结构三者协同演化，而非一次性静态分解。
  • 定义四种搜索模式覆盖网页搜索的四类瓶颈：atom（原子事实查找）、deep（串行 search–verify 迭代候选）、wide（并行分治聚合）、entity_collect（多路召回 + 去重 + 验证，应对未知集合边界）。节点可本地求解或继续委托子节点，模式可嵌套（wide 的子节点可以是 deep/entity_collect）。
  • 引入两种互补引导信号：Web-Probing Agent 轻量探测网页信息的组织方式（集中在聚合页 vs. 沿时间/实体/事件维度分散），据此决定后续扩展的维度；以及同父兄弟节点间的过程级经验复用——先跑少量 scout 节点抽取可迁移的 query 模式/可靠来源/无效路径，再注入其余同构节点，同时保持评测样本独立性。
  • 在四个挑战性基准（BrowseComp-Plus、WideSearch、DeepWideSearch、GISA）上系统验证，覆盖纯深度、纯宽度、深度宽度交织三类任务，并提供递归/模式/引导信号的消融、难度分层、Web 工具效率与跨骨干模型（Qwen3-32B、Qwen3.5-35B）泛化分析。

Key Contributions

  • A progressive recursive delegation framework where the task tree, expansion, and collaboration topology co-evolve at inference time, instead of being fixed by an upfront plan.
  • Four named search modes (atom, deep, wide, entity_collect) that map to distinct web-search bottlenecks (fact lookup, unknown-target verification, coverage/aggregation, recall-vs-precision set enumeration), each carrying its own collaboration protocol; modes can nest so deep and wide phases interleave inside one recursive process.
  • Two complementary guidance signals: (i) Web-Probing — a lightweight scout that classifies whether evidence is concentrated in hub pages or dispersed along an organizational axis, used to shape the expansion dimension; (ii) within-instance subtask experience reuse, where trajectories from a few scout sibling nodes are distilled into reusable patterns for subsequent homogeneous nodes without leaking across evaluation samples.
  • A thorough empirical study across four benchmarks, with ablations isolating recursive delegation, single-mode degradation, the efficiency role of Web-Probing, the quality role of experience reuse, per-difficulty behavior, tool-call efficiency, and backbone generalization (GLM-4.5, Qwen3-32B, Qwen3.5-35B).

Methodology

WebSwarm builds a recursive delegation tree T = (V, E) at inference time. The root node receives the original query q0. Each non-root node v is an agent holding a local objective qv and a search mode mv. An edge (u,v) represents parent u delegating to child v. A node maintains child results Rv and returns rv upward once its objective is met.

Objective decomposition and delegation. When evidence is insufficient, node v generates a set of child delegations Cv = {(qi, mi)} — crucially objective–mode pairs, not bare subtasks. Each pair instantiates a new search node. Coupling objective with mode guarantees every downstream node gets a solving protocol suited to its goal.

Search node solving (four modes).

  • atom: a ReAct agent that directly calls search(query) and fetch_url(url) for narrow fact lookup.
  • deep: iteratively dispatches searchers (which propose candidates from different clue paths) and verifiers (which adversarially check constraints); each searcher/verifier is an atom node. Targets unknown-entity identification or multi-constraint reasoning.
  • wide: parallel divide-and-conquer that concurrently runs independent child nodes and merges results; children may themselves be deep/entity_collect/wide, enabling nested deep+wide collection.
  • entity_collect: multi-path recall over an unknown-boundary set — dispatches multiple atom nodes from different sources/perspectives, then merges, deduplicates, and verifies low-confidence candidates to balance recall and precision.

Evidence update and continuation. When a child returns ri, the parent updates Rv ← Rv ∪ {ri} and decides whether to continue (spawn new sub-delegations, revise direction, request finer evidence) or stop and aggregate rv = Aggregate(qv, Rv). Returned results thus act both as answer components and as control signals, so the structure alternates top-down delegation with bottom-up evidence feedback and unfolds progressively.

Web-structure-guided expansion. Before wide expansion, a Web-Probing Agent runs lightweight search/reads and emits a hint hv summarizing the evidence distribution, representative pages, and candidate expansion axes. The node then generates child delegations conditioned on hv: Cv = Delegate(qv, mv, Rv, hv). When information concentrates in a few aggregate pages, WebSwarm creates few extraction-oriented nodes (avoiding redundant parallelism); when evidence is dispersed, it expands along the web-organization axis the probe suggests.

Experience-guided node solving. For batches of homogeneous siblings (e.g., filling attributes for a set of entities), WebSwarm first executes a small scout subset Cvs ⊂ Cv, collects (ri, τi) trajectories, and distills subtask experience kv = ExtractExperience({τi}) — useful query patterns, reliable sources, page types, invalid paths. This kv is injected into the remaining siblings: rj = SolveNode(vj; kv). Experience is reused only within the same instance and among siblings under the same parent, preserving evaluation-sample independence.

Implementation: GLM-4.5 as main backbone; Serper API web search (top-5 results) and Jina Reader page browse (≤80K tokens, goal-conditioned LLM summarization); entity_collect parallel paths = 3; 2 scout children for experience extraction; up to 200 action steps per agent; 128K context for GLM-4.5 (256K for Qwen3.5); SGLang on NVIDIA A800, temperature 0.6, top-p 0.95; experiments run twice and averaged.

Results & Highlights

  • BrowseComp-Plus (deep factual search, ACC, 200 sampled tasks, GLM-4.5): WebSwarm reaches 68.00 ACC, a +17.50 gain over ReAct (50.50) on the same backbone, and beats the strongest multi-agent baseline Swarm-Agent (64.50) by +3.50 ACC. (For context, a ReAct agent on the stronger Kimi-K2 backbone scores 66.50, still below WebSwarm-on-GLM-4.5.)
  • WideSearch-EN (structured wide collection, 100 tasks): WebSwarm scores 74.37 Item-F1 (+9.76 over ReAct's 64.61) and 44.14 Row-F1 (+10.91 over ReAct's 33.23); Success Rate improves from 4.00 to 7.00 (+3.00). It also tops the best multi-agent baseline InfoSeeker (71.91 Item-F1).
  • DeepWideSearch-EN (interleaved deep+wide, 76 tasks): WebSwarm reaches 58.40 Item-F1 (+11.77 over ReAct's 46.63) and 29.64 Row-F1 (+9.56 over ReAct's 20.08); Success Rate 6.58 vs ReAct 3.95 (+2.63), beating Flash-Searcher's best-baseline 5.26 SR.
  • GISA (general info seeking, all 373 tasks, multi-format): WebSwarm Overall 62.30 (+6.76 over ReAct's 55.54), with Set-F1 61.03 (+9.76) and Item EM 40.91 (+13.64 over ReAct's 27.27).
  • Hardest-sample gains (Fig. 3, difficulty stratified by ReAct pass rate): On BrowseComp-Plus Hard samples WebSwarm lifts ACC from 0.0 (ReAct) to 35.7; on WideSearch-EN Hard samples Item-F1 rises from 24.5 to 55.8 — the advantage grows with difficulty.
  • Ablation — recursive delegation (Table 2): Removing recursive delegation drops BC-Plus ACC 68.00→63.50, WideSearch Item-F1 74.37→68.38, DeepWideSearch Item-F1 58.40→55.79. Single-mode variants are task-dependent: All-to-wide cripples BC-Plus (63.00) while All-to-deep cripples WideSearch Item-F1 (69.94) — confirming the need to match objectives to modes.
  • Ablation — Web-Probing vs experience reuse (Table 3): Removing Web-Probing barely changes Item-F1 (WideSearch 74.37→74.90; DeepWideSearch 58.40→58.93) but inflates tool calls roughly 75% (137.03→239.90 and 203.73→331.39) — i.e., probing is primarily an efficiency lever. Removing experience reuse consistently lowers quality (WideSearch Item-F1 74.37→71.20; DeepWideSearch 58.40→55.48).
  • Cross-backbone generalization (Table 4): On Qwen3-32B, WebSwarm lifts BC-Plus 12.00→19.50 and WideSearch Item-F1 29.11→34.47; on Qwen3.5-35B, BC-Plus 56.50→61.00 and WideSearch Item-F1 64.82→75.91 — gains hold across weak and strong backbones.

Authors & Credibility

From the Gaoling School of Artificial Intelligence, Renmin University of China (a well-established IR/NLP group; senior authors Zhicheng Dou and Ji-Rong Wen are recognized information-retrieval researchers) in collaboration with Kuaishou Technology. Several co-authors have prior agentic-search work (Search-o1, WebThinker, GISA). Code is released at the project GitHub. The work was partly done as an internship at Kuaishou. The paper provides a clear running example (a DeepWideSearch football-transfer task), algorithmic pseudo-code (Algorithm 1), full system prompts, and benchmark/baseline detail in appendices, which lends reproducibility credibility.

Critical Assessment

  • Novelty: Moderate-to-strong. The framing of multi-agent search as recursive delegation over objective–mode pairs (rather than a fixed global topology) is a genuine reframing versus ROMA's recursive meta-agent, MindSearch/HiRA's graph planning, InfoSeeker's Host–Manager–Worker hierarchy, and AggAgent's trajectory aggregation. The four search modes and the web-structure probe are the differentiated ingredients; individually each is incremental, but their composition is sensible.
  • Rigor: Adequate but with caveats. The headline multi-agent gain on the flagship BC-Plus benchmark is only +3.50 ACC over Swarm-Agent (64.50→68.00), and notably a plain ReAct agent on the Kimi-K2 backbone scores 66.50 without WebSwarm at all — uncomfortably close to WebSwarm-on-GLM-4.5. The authors never run WebSwarm on Kimi-K2, so the reader cannot tell how much of the gap is architecture vs. backbone. The WideSearch (100 tasks) and DeepWideSearch (76 tasks) English subsets are small, so per-point deltas carry non-trivial variance; the paper runs each setting "twice randomly and take the average" but reports no variance/CI.
  • A telling ablation nuance: The Web-Probing module — presented as a first-class contribution — does not improve accuracy in Table 3 (Item-F1 actually ticks up slightly when it is removed on both datasets). Its entire documented value is reducing tool calls by ~43% (239.90→137.03 on WideSearch). The paper is honest about framing it as an efficiency lever, but the contribution narrative ("guide subsequent delegation") overstates the accuracy impact relative to the evidence.
  • Missing analyses: No latency/cost wall-clock or dollar breakdown despite explicitly motivating the work partly around resource efficiency; no test of whether the probe's "concentrated vs. dispersed" classification is actually correct; no failure-case taxonomy for the modes. The entity_collect parallel-path count (3) and scout count (2) are fixed but never swept.
  • Reproducibility: Reasonable. Unified tool interface (Serper + Jina), released code, fixed hyperparameters, and small English subsets help. But BrowseComp-Plus is run over a fixed local document corpus with Qwen3-Embedding-8B retrieval rather than the live web, so BC-Plus numbers are not directly comparable to live-web BrowseComp and the 200-task random sample is small.
  • Significance: Practically relevant to the agentic-search / deep-research agent line, and a useful roadmap for combining recursion + mode selection. Impact will depend on whether the gains survive a backbone-matched comparison against the strongest baseline (Kimi-K2 ReAct) and on a live-web evaluation.
  • Red flags: None severe — no evidence of test-set leakage (experience reuse is explicitly within-instance only), and ablations are honest. The main credibility risk is the missing Kimi-K2 + WebSwarm comparison that the paper's own Table 1 makes salient.

Strengths

  • Clean conceptual unification: decomposition, expansion, and collaboration co-evolve inside one recursive process, addressing all three failure modes (shallow recursion, fixed collaboration, evidence-blind expansion) the authors identify in prior work.
  • The objective–mode-pair delegation is a concrete, implementable mechanism that meaningfully beats forcing a single collaboration paradigm (the All-to-wide/All-to-deep ablations quantify this).
  • Mode nesting lets deep and wide phases interleave within one task — a real capability that purely hierarchical or purely parallel baselines lack.
  • Strong and growing gains on hard samples (0.0→35.7 on BC-Plus Hard) are exactly where such a system should add value over a single ReAct trajectory.
  • Honest reporting: the Web-Probing ablation and the cross-backbone ReAct rows are presented even though they complicate the story.

Limitations

  • Inference-time only — no multi-agent training or policy optimization, so delegation, mode assignment, probing, and experience reuse all depend on the base LLM's reasoning, which may not fully unlock recursive collaboration.
  • Higher cost and latency than single-agent ReAct: more LLM calls and web-tool requests, with adaptive (not bounded) resource growth on hard samples. No wall-clock/dollar cost is reported.
  • Text-only web tools; no handling of multimodal web content (images, video, audio) or GUI-based browsing.
  • Evaluation scale is modest for the table benchmarks (WideSearch-EN 100 tasks, DeepWideSearch-EN 76 tasks) and BC-Plus uses a 200-task local-corpus sample rather than live web; no variance/CI reported.
  • No backbone-matched WebSwarm result against the strongest available backbone (Kimi-K2), leaving open how much of the headline gap is architecture vs. model.

Relevance

This paper directly matches your "Self-Evolving Agents & Automated Research" interest: WebSwarm is a recursive, self-delegating agent architecture whose search tree and collaboration topology are constructed progressively as evidence accumulates, and it reuses process-level experience across sibling agents — a form of in-context agent self-improvement. It is also tangentially relevant to "LLM Inference Evaluation" via its rigorous use of four deep/wide/hybrid web-search benchmarks and cross-model generalization analysis.

Questions for Further Exploration

  • How does WebSwarm perform on the Kimi-K2 backbone, where plain ReAct already scores 66.50 on BC-Plus — does the architecture gain survive, or does a stronger backbone erode it?
  • What is the actual wall-clock latency and dollar cost per task, and how does the accuracy/cost Pareto frontier compare against ReAct and InfoSeeker?
  • Is the Web-Probing "concentrated vs. dispersed" classification accurate, and can replacing the LLM probe with a cheap classifier preserve the ~43% tool-call reduction without its implementation overhead?
  • Could multi-agent RL (the paper's stated future direction) learn better mode-assignment and delegation policies, and would that close the gap on the small WideSearch/DeepWideSearch subsets where per-point variance is high?
  • Does the experience-reuse scout mechanism transfer to non-search long-horizon agent tasks (e.g., code/CI loops, scientific workflows), where sibling subtasks also share query/source patterns?
  • How robust is the recursive tree to premature delegation or verifier false-negatives in deep mode, and is there a principled stopping criterion beyond the LLM's own Ready(qv, Rv) judgment?
