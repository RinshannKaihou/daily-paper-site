> **一句话总结：** MahaVar 在马氏距离基础上引入类间距离方差项，利用 Neural Collapse 几何结构下 ID 样本的"尖锐最小值"特性区分 OOD，在 CIFAR-100 和 ImageNet 基准上取得 SOTA。

> **TL;DR:** MahaVar augments the nearest-class Mahalanobis distance with a class-wise distance variance term, exploiting the sharp minimum structure of ID samples under Neural Collapse geometry, achieving state-of-the-art OOD detection on CIFAR-100 and ImageNet (OpenOOD v1.5).

## 核心贡献

- 发现 ID 样本的类间马氏距离呈现"尖锐最小值"结构（到最近类距离极小，到其他类距离大且平坦），导致高方差；OOD 样本则方差较低。
- 在放宽的 Neural Collapse 假设下（仅保留弱 NC1 和 NC2），从理论上证明 ID 样本的结构性高方差，并推导了方差上下界。
- 提出定理证明当 OOD 样本与所有类均值的余弦相似度足够低时，ID 样本的类间距离方差严格大于 OOD 样本。
- 提出 MahaVar，将马氏距离与方差项线性组合（`S(x) = -min d + α·Var[d]`），仅引入一个超参数 α。
- 在 OpenOOD v1.5 基准上，CIFAR-100 和 ImageNet 均取得 SOTA，且在三种 backbone（ResNet-50、Swin-B、ViT-B）上均优于 Mahalanobis++。

## Key Contributions

- **Empirical discovery**: ID samples exhibit a sharp minimum in sorted class-wise Mahalanobis distances (one small distance, all others large and plateaued), yielding high variance; OOD samples show lower variance — a structural asymmetry not previously identified.
- **Theoretical grounding**: Under relaxed Neural Collapse (weak NC1 + NC2), derives tight bounds on class-wise ℓ₂ distance variance for ID samples (Theorem 4.2), and proves sufficient conditions for strictly higher ID vs. OOD variance (Theorem 4.3, Corollary 4.4).
- **MahaVar score**: `S(x) = -min_c d_Maha,c(φ̂(x)) + α · Var_c[d_Maha,c(φ̂(x))]` — a simple post-hoc OOD score with a single hyperparameter α ∈ [0.05, 0.1].
- **State-of-the-art**: Best average AUROC and FPR@95 on CIFAR-100 (83.06% / 68.50%) and ImageNet (88.39% / 45.19%) averaged over three architectures, consistently outperforming all Mahalanobis-based baselines.

## Methodology

**Step 1 — Feature extraction**: Extract L2-normalized penultimate layer features φ̂(x) from a pretrained classifier (no retraining needed).

**Step 2 — Class-conditional statistics**: Compute class means μ̂_c and shared covariance Σ̂ from training set features (standard Mahalanobis procedure).

**Step 3 — Score computation**: For test sample x, compute Mahalanobis distances to all C class means, then:
```
S(x) = -min_c d_Maha,c(φ̂(x)) + α · Var_c[d_Maha,c(φ̂(x))]
```
The first term is the standard Mahalanobis score (higher = more ID); the variance term captures whether the distance profile has the sharp minimum structure characteristic of ID samples.

**Theoretical framework**: Under weak NC1 (bounded within-class deviation ε_c) and NC2 (class means form simplex ETF with inter-class distance K), the authors prove:
- ID variance is bounded by `[(C-1)K⁴/C²](1-γ)` to `[(C-1)K⁴/C²](1+γ) + 2ε²K²(C-2)/C`, where γ depends on ε.
- When OOD samples have sufficiently low cosine similarity to all class means, ID variance strictly exceeds OOD variance.
- Under an additional subspace alignment assumption, this dominance is universal regardless of OOD proximity.

**Practical details**: Optimal α ∈ [0.05, 0.1] across all architectures; K = all classes for variance computation; adding skewness yields marginal gains, so variance-only is adopted.

## Results & Highlights

**ImageNet (averaged over ResNet-50, Swin-B, ViT-B)**:
| Setting | AUROC | FPR@95 |
|---------|-------|--------|
| Near-OOD | **73.02** | **75.87** |
| Far-OOD | **91.87** | **38.09** |
| Average | **88.39** | **45.19** |

- Outperforms Mahalanobis++ by +0.89 AUROC / -2.71 FPR@95 on average.
- Beats all 16 evaluated baselines including KNN, NCI, ViM, NNGuide, fDBD, Energy.
- Only Texture (far-OOD) marginally underperforms Mahalanobis++ by 0.03 AUROC.

**CIFAR-100**: Best overall at 83.06% AUROC / 68.50% FPR@95, outperforming all baselines.

**CIFAR-10**: 90.91% AUROC / 46.09% FPR@95 — best among Mahalanobis family, though KNN has slightly higher AUROC (91.34%).

**Robustness across architectures**: Unlike ASH-S and SCALE which collapse on Transformer backbones (ASH-S drops to 22.89% AUROC on ViT-B), MahaVar maintains stable performance across ResNet-50, Swin-B, and ViT-B.

**Ablations**: (1) Variance term improves over vanilla Mahalanobis even without L2 normalization. (2) Higher-order moments (skewness) add marginal gains. (3) Performance degrades when α > 0.1 (variance dominates). (4) Variance helps across L1, L2, and Mahalanobis base metrics. (5) ViT-B's limited penultimate dimension (d=768 < C-1=999) weakens ETF formation, slightly reducing gains.

## Critical Assessment

- **Novelty: High.** The sharp minimum observation and its exploitation via class-wise distance variance for OOD detection is genuinely novel. The theoretical analysis under relaxed Neural Collapse is non-trivial and provides principled justification. Prior Mahalanobis-based methods (original, relative, Mahalanobis++) all focus only on the nearest class or a global reference — nobody looked at the *variance* across all classes.
- **Rigor: High.** Formal theorems with proofs, clear assumptions stated upfront, honest discussion of limitations (e.g., ViT-B dimension constraint). The use of OpenOOD v1.5 with standardized splits and validation-based hyperparameter tuning ensures fair comparison.
- **Reproducibility: High.** Post-hoc method, standard benchmarks, single hyperparameter α tuned on validation sets. Code release is not mentioned but the method is simple enough to reimplement.
- **Significance: Moderate-to-High.** Consistent gains over Mahalanobis++ across all settings is impressive. However, the absolute improvement magnitudes are sometimes modest (e.g., +0.89 AUROC on ImageNet average). The method is architecturally robust where competitors like ASH-S/SCALE fail catastrophically on Transformers — this is a practical strength.
- **Red flags**: (1) The CIFAR-10 results show KNN outperforming MahaVar in AUROC, suggesting the variance signal is less discriminative for simpler ID tasks. (2) The theory assumes ETF structure which doesn't fully hold for ViT-B (d < C-1), yet the method still works — the gap between theory and practice is acknowledged but not fully resolved. (3) The α hyperparameter, while stable in range, still requires validation-set tuning per architecture/dataset.

## Strengths

- Simple, elegant idea with strong theoretical backing from Neural Collapse geometry.
- Post-hoc: no retraining, no OOD data, no architectural changes required.
- Consistent improvements across all benchmarks and backbone architectures.
- Robust to backbone type — works well on CNNs (ResNet) and Transformers (Swin-B, ViT-B) alike, unlike activation-shaping methods.
- Comprehensive evaluation: 16 baselines, 3 ID datasets, 3 architectures, near-OOD and far-OOD.
- Ablation studies thoroughly validate each design choice.

## Limitations

- Gains over Mahalanobis++ are sometimes modest in absolute terms (e.g., +0.03 AUROC on some Far-OOD datasets).
- Requires computing Mahalanobis distances to all C class means, which can be expensive for large C (e.g., ImageNet-1K with 1000 classes), though this is shared with all Mahalanobis-based methods.
- The ETF assumption (NC2) requires d ≥ C-1, which fails for ViT-B on ImageNet (d=768, C=1000) — the method still works but the theoretical justification is weakened.
- Single hyperparameter α needs per-dataset validation tuning, though the optimal range is narrow and stable.
- Near-OOD detection remains challenging (e.g., 75.87% FPR@95 on ImageNet Near-OOD) — the method does not fundamentally solve hard near-OOD cases.

## Relevance

该方法利用特征空间几何结构（Neural Collapse 下的类间距离方差）进行 OOD 检测，与分布外泛化、模型鲁棒性和安全部署直接相关。对研究后处理 OOD 检测方法和神经网络特征表征的人员具有较高参考价值，特别是其理论分析框架为后续基于 Neural Collapse 的研究提供了新思路。

## Questions for Further Exploration

- Can the variance signal be combined with non-Mahalanobis base methods (e.g., KNN or energy scores) for further gains?
- How does MahaVar perform on semantic OOD (e.g., new class detection) vs. covariate shift?
- Does the sharp minimum structure hold for other feature spaces (e.g., CLIP features, intermediate layers)?
- Can the theoretical framework be extended to handle the non-ETF case (d < C-1) more rigorously?
- What is the computational overhead comparison with KNN at inference time for large-scale deployment?
- Could adaptive α selection (per-sample or per-class) improve performance over the global hyperparameter?
