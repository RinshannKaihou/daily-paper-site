> **一句话总结：** 本文首次提出了基于LTLfMT（有限迹上带SMT理论的线性时序逻辑）的预期性监控框架，结合自动机理论与SMT求解技术，为数据感知的动态系统（如业务流程）提供了可判定的监控方法，并实现了原型工具MON THE。

> **TL;DR:** This paper introduces the first anticipatory monitoring framework for data-aware temporal properties expressed in LTLfMT, combining automata-theoretic methods with SMT reasoning to construct monitors that detect permanent satisfaction/violation as early as possible, and identifies several decidable fragments covering arithmetic with uninterpreted functions.

## 核心贡献

- **首次形式化定义了LTLfMT上的预期性监控问题**，给出四种监控状态（当前满足、永久满足、当前违反、永久违反）的精确定义，并证明了预期性监控在一般情况下的计算框架。
- **提出了基于coreachability graph的监控算法**，将自动机理论（处理时序逻辑维度）与SMT求解（处理数据约束维度）相结合，通过反向回归构建可达性图来判断监控状态，并严格证明了算法的正确性。
- **识别出多个可判定的监控片段**：包括无算术的DB-LTLf（无环签名）、带单调性约束的DB-LTLf-MC（结合算术与数据库查询）、局部有限理论上的性质，以及有界回溯性质，覆盖了数据感知业务流程等实际应用场景。
- **实现了原型工具MON THE**，基于CVC5求解器，并在多个领域的性质上进行了初步实验验证。

## Key Contributions

- Formalizes the anticipatory monitoring problem for LTLfMT (LTL over finite traces modulo arbitrary SMT theories), defining four monitoring verdicts: CS, PS, CV, PV.
- Develops a novel monitoring algorithm that constructs a coreachability graph by regressing verification obligations backward from final states, combining NFA construction for temporal structure with SMT-based quantifier elimination for data constraints.
- Proves the algorithm's correctness (Theorem 18), exploiting model completion properties of the underlying theory for both embeddability and quantifier elimination.
- Identifies four decidable fragments with practical relevance: (I) DB-LTLf with acyclic signatures, (II) DB-LTLf-MC combining arithmetic with database references under monotonicity constraints, (III) locally finite theories, and (IV) K-bounded lookback properties.
- Provides a prototype implementation (MON THE) using CVC5 and evaluates it on properties from synthetic patterns, VERIFAS benchmarks, and the paper's running example.

## Methodology

The approach proceeds in three stages. First, for a given quantifier-free LTLfMT (dataLTLf) property $\psi$, an NFA $N_\psi$ is constructed where the alphabet consists of sets of first-order formulas from the property. Each NFA transition is labeled with a constraint set capturing which FO formulas must hold at that step, with a special proposition $\lambda$ marking trace termination. The key insight is that this NFA captures the temporal structure while abstracting away the data dimension.

Second, a coreachability graph $CG^+(N_\psi)$ is built by backward regression from accepting states. For each NFA state $q$ and an FO formula $\phi$, a node $(q, \phi)$ represents the condition that a final state is reachable from $q$ when the current variable assignment satisfies $\phi$. The regression operation $\text{regress}(\phi, C) = \exists Y.[C](V/V^\triangleleft, Y/V) \wedge \phi(Y/V)$ propagates verification obligations backward through constraints. Model completion of the background theory ensures all formulas remain quantifier-free.

Third, the monitoring procedure runs the trace through $N_\psi$ to determine the current NFA state $q$, then checks $\text{EXTSAT}(q)$ or $\text{EXTVIOL}(q)$ (disjunctions of formulas from the coreachability/non-coreachability graphs) against the current model and assignment. If the property is currently satisfied, checking violation conditions yields CS (violation possible) or PS (permanently satisfied); if currently violated, checking satisfaction conditions yields CV (satisfaction possible) or PV (permanently violated).

## Results & Highlights

- The monitoring algorithm is provably correct (Theorem 18), returning the exact monitoring state for any input trace and property.
- Decidability is established for four practically relevant classes of properties:
  - **Acyclic DB-LTLf** (Theorem 19): Finitely many non-equivalent formulas guarantee coreachability graph termination.
  - **DB-LTLf-MC** (Theorem 20): Combines tame signatures with monotonicity constraints; subsumes prior work by Felli et al. (2023) on arithmetic-only monitoring.
  - **Locally finite theories** (Theorem 21): General decidability criterion beyond acyclicity.
  - **K-bounded lookback** (Theorem 24): Syntactic restriction bounding how far data dependencies reach back in time.
- Experimental evaluation on 20 properties across 5 domains shows preprocessing (NFA + coreachability graph construction) dominates runtime. Most properties complete in under 5 seconds; the most complex (DDS dynamics with $|\phi|=101$) takes ~158 seconds, mostly in preprocessing. Monitoring time per trace step is typically under 2 seconds.

## Critical Assessment

- **Novelty: High.** This is the first work to tackle anticipatory monitoring for LTLfMT. The combination of automata-theoretic techniques with SMT-based model completion for monitoring is genuinely new. The identification of decidable fragments is a significant theoretical contribution.
- **Rigor: High.** All main results (Theorems 10, 18, 19, 20, 21, 24) are formally stated with complete proofs. The paper carefully defines all prerequisite notions (traces, embeddings, model completion) before building the framework.
- **Reproducibility: Moderate.** A prototype tool (MON THE) is publicly available, and the paper includes experimental data. However, the benchmarks are limited in scale (5 traces per property), and full benchmark artifacts are not detailed enough to reproduce all experiments independently.
- **Significance: High for formal methods/BPM communities.** Data-aware runtime verification is increasingly important for monitoring AI agents and business processes. The theoretical results open a new research direction.
- **Red flags:**
  - The experimental evaluation is explicitly labeled "preliminary" — only 20 properties, 5 traces each, with no comparison against alternative tools (though alternatives for this exact setting do not yet exist).
  - The scalability of the coreachability graph construction is a concern: the DDS dynamics example with 143 subformulas takes ~121 seconds, suggesting potential blowup for complex real-world properties.
  - The restriction to quantifier-free formulas (dataLTLf) is significant — existential quantification over domain elements, common in database queries, is excluded.
  - The model of read-only databases with conservative extensions may be too restrictive for applications where data is actively updated.

## Strengths

- Clean, principled combination of automata theory and SMT solving with formal correctness guarantees.
- Identifies multiple decidable fragments that correspond to practically relevant application domains (data-aware BPM, database-driven systems).
- The coreachability graph construction is a novel technical contribution that enables anticipatory monitoring by tracking satisfaction/violation conditions symbolically.
- Running example (concert ticket marketplace) effectively motivates the need for data-aware monitoring and illustrates the four monitoring verdicts.

## Limitations

- Restricted to the quantifier-free fragment of LTLfMT; cannot handle properties with existential/universal quantification over domain elements.
- Assumes a read-only database model where past data cannot be updated — excludes write-heavy applications.
- The coreachability graph construction may not terminate for general properties; decidability is only guaranteed for the identified fragments.
- Preliminary experimental evaluation with limited scale and no comparison baselines.
- No treatment of real-time or metric temporal constraints.

## Relevance

本文提出了数据感知时序逻辑的运行时验证框架，将SMT求解与自动机理论相结合，为监控黑盒AI系统（如自主Agent和业务流程）的行为提供了形式化方法基础。该工作在形式化方法和可信AI方向具有重要意义。

## Questions for Further Exploration

- Can the framework be extended to handle existential quantification over domain elements, enabling richer database queries in monitored properties?
- How does the approach scale to properties with hundreds of subformulas or traces with thousands of steps in production settings?
- Could the coreachability graph be constructed lazily or incrementally to reduce preprocessing overhead for long-running monitoring tasks?
- Is it feasible to extend the framework to handle dynamic database updates (e.g., price changes, record deletions) while preserving decidability?
- Can the monitoring approach be integrated with online learning to adapt properties based on observed system behavior?
