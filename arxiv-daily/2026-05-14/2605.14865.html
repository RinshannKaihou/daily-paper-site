Holistic Evaluation and Failure Diagnosis of AI Agents

arXiv: 2605.14865

Date: 2026-05-14

https://arxiv.org/abs/2605.14865

一句话总结： 提出了一种结合自顶向下（agent 级）和自底向上（span 级）评估的整体框架，通过对 AI Agent 执行轨迹的逐 span 独立评估实现可扩展的故障定位与分类，在 TRAIL 基准上相对最强基线取得最高 38% 的类别 F1 提升和 12.5 倍的联合准确率提升。
TL;DR: A holistic agent evaluation framework combining top-down agent-level diagnosis with bottom-up span-level decomposition achieves state-of-the-art on TRAIL across all metrics, demonstrating that evaluation methodology—not model capability—is the bottleneck for accurate failure localization in AI agents.

核心贡献

  • 提出 整体式 Agent 评估框架，将评估分解为自顶向下的 agent 级诊断（规划效率、工具覆盖率）和自底向上的 span 级评估（推理完整性、指令遵循），两者互补
  • 逐 span 独立评估 使框架可扩展至任意长度的执行轨迹，每个 span 生成独立的评分（1-5 分）和自然语言理由
  • 在 TRAIL 基准的两个数据集（GAIA 和 SWE-Bench）上 全面领先：类别 F1 提升 26%-38%，定位准确率提升 1.25×-3.5×，联合准确率提升 2.58×-12.5×
  • 证明 评估方法论是瓶颈：同一模型（GPT-5.4）在该框架内的定位准确率是作为单体 judge 的 2.8×-12×
  • 对 TRAIL 基准标注质量进行了系统分析，揭示了标注不一致、类别边界模糊等问题

Key Contributions

  • A holistic trace-aware evaluation framework that pairs top-down agent-level assessment (plan efficiency, tool coverage, tool abuse, completeness) with bottom-up span-level analysis (reasoning integrity, instruction following, tool completeness, error detection)
  • A span-level failure diagnosis method that decomposes evaluation into independent per-span assessments, enabling linear scaling with trace length and producing grounded natural-language rationales for each verdict
  • State-of-the-art results on TRAIL across localization accuracy, weighted category F1, and joint localization-categorization accuracy on both GAIA and SWE-Bench, with the gap widening on longer traces
  • Empirical evidence that methodology dominates model capability: GPT-5.4 achieves 0.823 localization accuracy inside the framework vs. 0.292 as a monolithic judge on GAIA (2.8× gap), and 0.860 vs. 0.070 on SWE-Bench (12× gap)
  • A systematic critique of TRAIL benchmark annotations, identifying inconsistent span localization, overlapping category boundaries, and misclassifications between annotator reasoning and evidence

Methodology

The framework operates on hierarchical agent traces (e.g., OpenTelemetry spans) consisting of agent spans, LLM call spans, and tool invocation spans.

Bottom-Up Evaluation: Each leaf span (LLM or Tool) is independently assessed across multiple metrics: Instruction Following (1-5), Reasoning Integrity (1-5), Error Detection (label), Avoidance (label), Tool Completeness (1-5), and Latency/Tokens (numeric). Each metric produces a verdict (pass/fail) with a natural-language rationale. Verdicts propagate upward through the trace tree via existential failure propagation (any failing child → parent fails). Configurable alternatives include threshold-based, type-filtered, and conjunctive policies.

Top-Down Evaluation: Agent-level metrics assess holistic behavioral patterns: Plan Efficiency, Tool Coverage, Tool Abuse, and Completeness. These detect failures invisible at the individual span level—redundant tool calls, incomplete goal coverage, plan deviation.

Mapping to TRAIL Taxonomy: Since the framework outputs metric scores and rationales (not TRAIL category labels), an LLM-based mapper translates framework outputs into TRAIL's 20+ category taxonomy for benchmark comparison.

Key Design Insight: Top-down reveals how an agent went wrong (behavioral patterns); bottom-up reveals where (exact spans and error causes). The combination provides both diagnostic precision and holistic assessment. The authors demonstrate this with a concrete GAIA example where neither approach alone catches all four ground-truth issues.

Results & Highlights

GAIA (multi-agent, shorter traces):

  • Cat. F1: 0.547 (26% gain over best baseline Gemini-2.5-Pro at 0.389)
  • Loc. Acc.: 0.823 (1.25× gain over FAGI-AgentCompass at 0.657)
  • Joint Acc.: 0.616 (2.58× gain over FAGI-AgentCompass at 0.239)

SWE-Bench (single-agent, longer traces):

  • Cat. F1: 0.698 (38% gain over GPT-5.4 monolithic at 0.504)
  • Loc. Acc.: 0.860 (3.5× gain over FAGI-AgentCompass at 0.250)
  • Joint Acc.: 0.638 (12.5× gain over FAGI-AgentCompass at 0.051)

Critical findings:

  • Every prior baseline's localization accuracy degrades sharply from GAIA to SWE-Bench; the framework's holds steady (0.823 → 0.860)
  • Monolithic judges exclude 22-29% of SWE-Bench traces due to context overflow; the framework evaluates every trace with zero exclusions
  • Per-category analysis: the framework leads in 7 of 13+ error categories vs. 5 for the next-best evaluator
  • The framework wins across both multi-agent (GAIA/Open Deep Research) and single-agent (SWE-Bench/CodeAct) regimes

Authors & Credibility

All 15 authors are from Deepchecks, an Israeli company focused on ML/AI testing and observability. The work directly aligns with Deepchecks' commercial product direction (AI agent evaluation). Shir Chorev is a co-founder of Deepchecks. The paper is rigorous in experimental design: it uses the established TRAIL benchmark, compares against 12+ baselines including reproductions with frontier models (GPT-5.4, Claude-Sonnet-4.6, Gemini-3.1-Pro), and openly discusses benchmark limitations and its own framework's weaknesses. The fact that the framework is evaluated zero-shot on TRAIL (no prompt tuning on the benchmark's dev split) adds credibility.

Critical Assessment

Novelty: The top-down + bottom-up decomposition is conceptually clean but not radically novel—it mirrors established ideas in software observability (distributed tracing) and hierarchical fault diagnosis. The key insight (span-level decomposition avoids context window issues) is straightforward in hindsight. The contribution is more in the systematic engineering and empirical validation than in theoretical breakthrough.

Rigor: Strong experimental methodology with comprehensive baselines, per-category breakdowns, and cross-dataset analysis. However, the framework uses GPT-5.4 as its underlying judge, and the LLM-based mapper step (translating metric outputs to TRAIL categories) introduces an additional LLM call that is not ablated. It is unclear how much the mapper contributes to category F1 scores. The per-category heatmap shows the framework losing in some infrastructure categories with small support, which is noted but somewhat downplayed.

Reproducibility: The framework and evaluation code are not explicitly stated as open-sourced in the paper (no GitHub link found). The evaluation is zero-shot on TRAIL, which aids reproducibility in principle, but the exact prompts, rubrics, and mapper implementation details are not fully specified in the paper. Reproducing the results would require significant reimplementation.

Significance: The empirical finding that methodology dominates model capability (same GPT-5.4 achieving 2.8-12× gains) is significant and has implications for the agent evaluation field. The framework's ability to handle arbitrarily long traces is practically important as agents generate increasingly complex execution traces.

Red flags: Potential commercial bias—Deepchecks sells agent evaluation tooling. The framework's reliance on many LLM calls per trace (each metric per span) means evaluation cost scales linearly with trace length and number of metrics, which could be expensive in production. No cost or latency analysis is provided.

Strengths

  • Elegant span-level decomposition that solves the context window problem structurally rather than waiting for larger contexts
  • Comprehensive evaluation on the most relevant benchmark (TRAIL) with 12+ baselines
  • Strong empirical results with consistent gains across datasets, metrics, and error categories
  • Honest discussion of benchmark annotation quality issues and the framework's own limitations
  • The "methodology over model capability" finding is a valuable contribution to the field
  • Framework integrates naturally with OpenTelemetry, making it immediately applicable to production systems

Limitations

  • Cost and latency: Each span is evaluated across multiple metrics, each potentially requiring LLM calls. No analysis of evaluation cost or wall-clock time is provided.
  • Mapper dependency: Performance depends on an LLM-based mapper step that was not ablated; it is unclear how performance degrades with different mapping strategies.
  • Aggregation simplicity: Current hierarchical aggregation uses existential failure propagation (any child failure → parent fails), which is coarse. Severity weighting and outcome-relevance filtering are acknowledged as future work.
  • Single judge model: All results use GPT-5.4 as the judge; no analysis of how the framework performs with cheaper or open-source models.
  • Reproducibility concerns: No public code repository linked; exact prompts and rubrics not fully specified.
  • Commercial alignment: The work directly serves Deepchecks' commercial interests, which may influence framing (though the empirical work appears solid).
