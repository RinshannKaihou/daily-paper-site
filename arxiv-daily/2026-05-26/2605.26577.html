arXiv 2605.26577

Bridging Control with Neural Network Verifier α,β-CROWN: A Tutorial

Haoyu Li, Xiangru Zhong, Hao Cheng, Bin Hu, Huan Zhang UIUC 2026-05-26
Key Finding
α,β-CROWN unifies Lyapunov, barrier, and contraction verification for neural controllers via bound propagation and branch-and-bound.
Key Metric
~5×faster verification with certified training vs. CEGIS on 2D quadrotor
Why This Matters
Formal verification is the critical barrier to deploying learning-based controllers in safety-critical systems like autonomous vehicles.
Critical Edge
No systematic evaluation beyond low-dimensional examples; scalability to >10D systems remains unproven.

Methodology

  1. Bound Computation via auto LiRPA — Backward linear bound propagation through computation graphs constructs sound affine lower/upper bounds for all inputs in a box domain by recursively applying linear relaxations at each nonlinear operator (ReLU, sin, multiplication).
  2. Branch-and-Bound (BaB) — When initial bounds are too conservative, the input domain is recursively partitioned. GPU parallelization processes tens of thousands of subdomains simultaneously; smart branching selects the most sensitive split dimension.
  3. Satisfiability Encoding — Control certificates are encoded as logical combinations of scalar inequality checks over box domains. Sublevel set conditions convert to implications verifiable on the surrounding box via disjunction.

Pipeline: PGD falsification → incomplete verification (single-pass bound propagation) → complete verification (BaB with interleaved falsification).

Results & Highlights

  • Verified Lyapunov stability for discrete-time and continuous-time systems, including robust variants with bounded disturbances
  • Barrier function verification for standard and control barrier functions (CBF) with polytopic input constraints via vertex enumeration
  • Scalar reformulation of discrete-time contraction conditions avoids matrix inequality encodings
  • Certified training yields ~5× faster verification vs. CEGIS on 2D quadrotor task
  • Jacobian computation via extended computation graphs enables continuous-time ∇V·f analysis

Strengths & Limitations

Strengths

  • Clear presentation with concrete code examples alongside math
  • Covers Lyapunov, barrier, contraction, and MPC under one framework
  • dReal-compatible API lowers migration cost for existing users
  • GPU-parallelized BaB provides scalability over traditional SMT/MILP
  • Certified training directly addresses verification-unfriendliness of standard training

Limitations

  • No systematic experimental evaluation with timing or success-rate data
  • All examples are low-dimensional; unclear scaling beyond ~10D
  • Verification quality depends entirely on the candidate certificate
  • Box input domains are a fundamental restriction
  • No discussion of failure modes or solver timeouts

Open Questions

How does verification time scale with state dimension — 10D vs 50D vs 100D — for Lyapunov stability certification?
Can certified training combine with reinforcement learning for end-to-end verifiable policy synthesis?
What are the theoretical limits on certificate complexity before verification becomes intractable?
How to handle stochastic dynamics or partially observable systems where the state is not directly available?
Can contraction-based verification extend to hybrid systems with mode switches?
