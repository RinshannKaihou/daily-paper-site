ContextGuard: Structured Self-Auditing for Context Learning in Language Models

Hongbo Jin, Chi Wang, Haoran Tang, Zhongjing Du, Xu Jiang, Jingqi Tian, Qiaoman Zhang, Jiayu Ding — 2026-05-26

Key Finding
Structured self-auditing that separates outputs into protected and fix-target regions improves context-learning task-solving rate by +4.21 pp over baseline.
Key Metric
+5.37pp on Qwen3.5-9B · +4.21 pp on 4B · CL-Bench
Why This Matters
Solves the repair-preservation tradeoff: generic self-refinement fixes errors but can break already-satisfied constraints under strict all-requirement scoring.
Critical Edge
Absolute solving rates stay below 16% even after improvement, and the multi-stage auditing pipeline adds significant inference cost (draft + audit + revision).

Methodology

1
Draft + Reminder
Generate initial response with augmented reminder re-emphasizing system-level constraints to combat attention dilution in long contexts.
2
Structured Self-Audit
Decompose output into four epistemic regions: confirmed constraints (QA), confirmed facts (QB), possibly missed (QC), possibly wrong (QD).
3
Protected Revision
Fix set F = QC ∪ QD ∪ Et; protection set P = QA ∪ QB ∪ Ot. Revise only uncertain regions while preserving verified content.
  • Entirely inference-time — no additional training or parameter updates required.
  • Category-conditioned specialist signals adapt failure detection to task type: format, workflow, rule-fidelity, numerical accuracy.
  • Reminder augmentation re-anchors key system constraints during long-output generation.

Results

Model Baseline Self-Refine ContextGuard Gain
Qwen3.5-4B 9.64% 10.48% 13.85% +4.21 pp
Qwen3.5-9B 10.43% — 15.80% +5.37 pp

Consistent gains across all four CL-Bench macro-categories. Failure reduction spans format/lexical (−2.37 pp), conditional rules (−2.81 pp), and style/persona (−3.03 pp). Stable +2.7–5.8 pp across all input-length bins; Self-Refine collapses at 32K+ tokens.

Strengths

  • Clean framework addressing a precisely characterized failure mode — near-miss failures where models satisfy most but not all constraints.
  • Comprehensive ablation: structured self-auditing contributes −3.32 pp when removed (largest single component).
  • Stronger-model generalization confirmed on Qwen3.5-9B with even larger gains.
  • Detailed case studies showing how protection sets prevent revision regression in practice.
  • Per-requirement diagnostic analysis provides insight beyond aggregate metrics.

Limitations

  • Evaluated only on Qwen3.5 series (4B, 9B) and a single benchmark (CL-Bench); generalization to other model families is untested.
  • Absolute solving rate remains below 16% — fundamental context-learning limitations are not addressed.
  • Multi-stage pipeline significantly increases inference cost (draft + audit + specialist signals + revision).
  • Only one revision round explored; iterative refinement is not investigated.
  • Framework relies on the model's ability to correctly self-assess correctness, which may be unreliable for harder tasks.

Questions for Further Exploration

  1. How does ContextGuard perform on frontier models (GPT-5, Claude 4, Gemini Ultra) with better inherent context-following abilities?
  2. Can epistemic stratification be learned or fine-tuned rather than prompted, potentially improving audit accuracy?
  3. What is the inference cost overhead (latency, token count) compared to baseline, and can it be reduced?
  4. Does iterative ContextGuard (multiple audit-revision rounds) continue to improve or plateau/degrade?
  5. How does the framework handle cases where the model's self-assessment is systematically wrong (confidently marking incorrect content as "confirmed")?
