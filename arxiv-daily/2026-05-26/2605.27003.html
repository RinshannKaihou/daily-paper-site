Timestep-Aware SVDQuant-GPTQ for W4A4 Quantization of Wan2.2-I2V

Junhao Wu, Dezhong Yao, Hai Jin — HUST, Wuhan

A W4A4 post-training quantization framework for Wan2.2-I2V cuts peak GPU memory by 59.3% with only 0.9% VBench degradation.

−59.3%
64.4 GB → 26.2 GB peak VRAM

Enables frontier video generation models on consumer GPUs by slashing memory nearly 60% without perceptible quality loss.

Pseudo-quantization increases latency 83% — practical speedup awaits native FP4/MXFP4 kernel support.

Methodology

Three-Stage Calibration Pipeline

  1. SVDQuant Decomposition — Channel-wise smoothing transfers activation outliers to weights, then decomposes into a high-precision low-rank branch (L₁L₂) and a 4-bit residual (R).
  2. GPTQ on Residual — Replaces naive round-to-nearest with Hessian-based second-order weight quantization, minimizing layer-wise reconstruction error under 4-bit constraint.
  3. Timestep-Bin-Wise Clipping Search — Divides denoising into bins (early, mid, boundary, late). For each (layer, bin, expert) triple, searches optimal clipping ratio from {0.8, 0.9} via MSE minimization. Inference uses only a lookup table.

Assessment

Strengths & Limitations

Strengths

  • Clean, well-motivated pipeline addressing a real deployment bottleneck for large video generation models.
  • Expert-level and timestep-level calibration is principled for MoE-specific and diffusion-specific quantization challenges.
  • Strong empirical results with clear ablation evidence isolating each component’s contribution.
  • Excellent module-level sensitivity analysis providing actionable insights for future W4A4 work on MoE architectures.

Limitations

  • No actual inference speedup — pseudo-quantization increases latency by 83%. Benefits are memory-only today.
  • Evaluated on a single model and benchmark (OpenS2V-Eval). Generalizability is unproven.
  • Low-rank branch adds compute overhead that partially offsets memory savings.
  • Clipping ratio search constrained to {0.8, 0.9}; continuous optimization might yield further gains.
Open Questions

Questions for Further Exploration

  1. How does the framework generalize to other MoE video DiTs (HunyuanVideo, Sora-like)?
  2. Would continuous optimization of clipping ratio outperform the discrete {0.8, 0.9} search?
  3. Can the low-rank branch be further quantized (e.g., INT8) without significant quality loss?
  4. What is the impact of timestep-bin count on the quality-efficiency trade-off?
  5. How would fused dequantization-GEMM kernels (TensorRT, custom CUDA) reshape the latency profile?
