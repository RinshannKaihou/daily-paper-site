LLM exported vector representations leak sensitive patient attributes — mitigating one artifact leaves others exposed.

Vectors Are Not Neutral: Sensitive-Information Inference from Exported LLM Representations in Summarization

Weixin Liu, Bowen Qu, Juming Xiong, Congning Ni, Bradley A. Malin, Zhijun Yin — Vanderbilt University

Key Metric — Race probing accuracy 0.203 chance = 0.20  |  targeted lasttok artifact reduced to near-random
Why This Matters Privacy auditing must target the exact vector artifact exposed downstream — auditing one representation doesn't protect others.
Critical Edge Meanpool stays ~0.32 despite SurfaceLoRA; output-level race recoverability still 0.299 vs 0.20 chance. No differential privacy comparison.

Methodology

Task: Brief Hospital Course generation from MIMIC-IV-Ext-BHC (270K clinical note–BHC pairs). Five-way race classification (WHITE, BLACK, HISPANIC, ASIAN, OTHER). Balanced subsets: 20K train / 2.5K val / 2.5K test. Chance = 0.20.

  • Model: Llama-3.1-8B-Instruct + LoRA (rank unspecified). Input 1,024 tokens; output 256 tokens greedy.
  • Audited artifacts: lasttok_L-1 (final prompt-token hidden state) and meanpool_L-1 (mean of non-padding prompt hidden states), both from last transformer block.
  • SurfaceLoRA: Combines SFT batch (summary CE loss) + balanced prompt-only batch with gradient-reversal discriminator. Objective: min_θ L_util − λ·L_adv. λ swept over {0, 0.02, 0.05, 0.10, 0.20, 0.50}, 2K steps, AdamW lr=2e-4.
  • Evaluation: Post-hoc probes (logistic regression + 2-layer MLP) on frozen vectors. Checkpoint selection: maximize val ROUGE-L subject to LeakageGap_LR ≤ 0.025.
  • Baselines: BASE prompt, NEUTRAL prompt (demographic-neutral directive), PCA removal, XCov decorrelation.

Results & Highlights

SurfaceLoRA λ=0.02 lasttok LR Acc 0.203, MLP Acc 0.206 — both 95% CIs include chance. ROUGE-L 14.54 ✓ near-chance
Meanpool leakage LR 0.316–0.323, MLP 0.270–0.326 — 12+ pp above chance even when lasttok is sanitized ✗ still leaky
Non-monotonic dynamic At λ=0.02, MLP probe drops to 0.206 at step 1200 but rebounds to 0.228 at step 2000 ⚠ rebounds
Prompt engineering BASE/NEUTRAL remain above chance (LR 0.234–0.236). NEUTRAL causes 46% meta-term contamination ✗ insufficient
Post-hoc PCA Smallest LR gap (0.005), but MLP still above chance (0.212). XCov increases recoverability ✗ partial
Output attacker Bio_ClinicalBERT on generated summaries: Acc 0.299, Macro-AUROC 0.616 (vs 0.20 chance) ✗ persists
cross-validated on Qwen-2.5-7B gender stress test (appendix)

Strengths

  • Balanced five-way probing with patient-level splits and pre-registered checkpoint selection
  • Artifact-specific asymmetry finding is counterintuitive and well-documented
  • Multiple artifacts, probe classes, output attackers, cross-model validation
  • Explicitly honest that SurfaceLoRA is not a universal sanitizer
  • Non-monotonic trade-off analysis gives actionable deployment guidance
  • Bootstrap 95% CIs reported throughout

Limitations

  • Only one primary attribute (race); multi-attribute auditing untested
  • Meanpool remains substantially leaky with no demonstrated fix
  • Output-level invariance not achieved (0.299 vs 0.20 chance)
  • Single domain (clinical BHC), two model families only
  • Probe-based evaluation — no information-theoretic guarantees
  • No comparison with differential privacy (DP-SGD)
  • "Audit the exact artifact" advice may be hard to enforce in complex systems

Questions for Further Exploration

  • Can SurfaceLoRA simultaneously target multiple artifacts (lasttok + meanpool) without utility loss?
  • How does the non-monotonic leakage dynamic interact with different LoRA ranks and adapter configs?
  • Would DP-SGD provide stronger formal guarantees than adversarial training here?
  • Do findings generalize to non-clinical domains where sensitive attributes are less text-correlated?
  • Can mutual-information measures complement probe-based auditing for stronger privacy bounds?
  • When protecting multiple attributes simultaneously, does pressure on one increase leakage on another?
