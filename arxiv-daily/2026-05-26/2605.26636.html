JetViT: Efficient High-Resolution Vision Transformer with Post-Training Attention Search

MIT · NVIDIA · U Penn 2605.26636 2026-05-26
Key Metric
1.79×throughput speedup · 44.81% latency cut · zero accuracy loss
Key Finding
Only 2 full-attention layers are needed to recover full model accuracy after converting pre-trained ViTs to hybrid attention.
Why This Matters
Makes billion-parameter vision models practical for real-time high-resolution inference without retraining or proprietary data.
Critical Edge
Requires per-task distillation & search; code unreleased; speedups measured on H100 only.

Methodology

Step 01
Linear-Attention Block Optimization
Replaces static depth-wise conv with Squeeze Dynamic Convolution — a lightweight MLP generates kernels from the average-pooled value matrix, improving features with minimal overhead.
Step 02
LA + Window Attention Search
Supernetwork with per-layer linear vs. window choices. Feature-level distillation from full-attention teacher, then beam search finds optimal LA/WA layout. Inheriting all attention weights is critical (+3% mIoU).
Step 03
Full-Attention Placement
Second supernetwork: each layer offers the efficient block or full-attention. Beam search targets the downstream task metric directly to place minimal full-attention layers.

Critical Assessment

Novelty 7/10 Rigor 8/10 Reproducibility 6/10 Significance 8/10

Systematic two-stage search with comprehensive ablations. The Squeeze Dynamic Convolution is incremental but the finding that only 2 full-attention layers suffice is surprising and impactful. Code not yet available; per-task distillation adds engineering overhead.

Strengths & Limitations

Strengths

  • No proprietary pre-training data or costly retraining needed
  • General framework for any full-attention ViT backbone
  • Minimal accuracy loss with significant real-world speedups
  • Insightful layer-level analysis of where full attention matters
  • First layer is always linear — echoes CNN shallow-layer locality

Limitations

  • Separate distillation and search required per downstream task
  • Linear attention in pure PyTorch — no custom kernels yet
  • Not evaluated on generative vision or video at scale
  • Search still requires non-trivial GPU resources
  • Smaller models and lower resolutions unexplored

Questions for Further Exploration

  1. Can Post-Training Attention Search apply to multimodal models (CLIP, LLaVA) where the vision encoder interacts with a language model?
  2. How does optimal full-attention placement change with resolution scaling beyond 4K/8K?
  3. Would combining JetViT with quantization or pruning yield compound efficiency gains?
  4. Can the search be made task-agnostic (universal placement) to avoid per-task re-search?
  5. How do results transfer to video transformers where temporal attention adds another dimension?
