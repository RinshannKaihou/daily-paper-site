Stabilizing Recurrent Dynamics for Test-Time Scalable Latent Reasoning in Looped Language Models

Jacobian spectral radius < 1 stabilizes looped LLMs, cutting GSM8K degradation from 20% to 8%.
Xiao-Wen Yang, Ziyu Han, Xi-Hua Zhang, Wen-Da Wei, Jie-Jing Shao, Lan-Zhe Guo, Yu-Feng Li ICML 2026
arXiv:2605.26733 · Nanjing University
Key Metric
100% accuracy

4-digit addition, all recurrent depths

Why This Matters
Reliable test-time compute scaling is essential if looped inference is to become a viable paradigm for LLM reasoning.
Critical Edge
STARS mitigates but does not eliminate degradation — AMC23 drops 52.5% → 35% at 8 steps, and only validated on 1.4B models.

Methodology

  • Diagnostic: Systematic sweep of normalization type (LayerNorm, RMSNorm, SimpleNorm) × placement (Pre-, Post-, Sandwich) on a minimal looped GPT block, revealing that placement alone governs the stability–effectiveness trade-off.
  • Core mechanism: Internal normalization preserves information flow but causes unbounded hidden-state growth; external normalization bounds states but converges to ineffective fixed points.
  • STARS framework: Combines Jacobian Spectral Radius Regularization (JSRR) with random loop sampling to enforce ρ(J) < 1, guaranteeing convergence to asymptotically stable fixed points via Lyapunov linearization.
  • Efficient implementation: Single-step power iteration + Jacobian-vector product avoids O(D²) Jacobian construction; ~2× overhead over standard SFT.

Strengths

  • Principled dynamical systems diagnosis yielding clear architectural insight — normalization placement matters more than normalization type
  • Efficient JSRR via single-step power iteration makes the approach practical at scale
  • Clean ablation isolating each component's contribution
  • Honest about remaining limitations — monotonic scaling is not fully solved

Limitations

  • Evaluated only on mathematical reasoning; generalization to commonsense, code, or broader NLP is untested
  • Monotonic improvement not achieved on hard tasks (AMC23: 52.50% → 35.00%)
  • ~2× training overhead compounds the already expensive looped model training
  • Validated only on 1.4B parameters; scaling to larger models is unverified

Questions for Further Exploration

  1. Can JSRR improve training stability in standard (non-looped) very deep transformers?
  2. How does single-step power iteration estimate quality degrade at 7B, 13B, 70B scale?
  3. Would an annealed λ schedule allow strong early learning while enforcing stability later?
  4. Could adaptive loop depth — stopping when spectral radius falls below threshold — provide a principled early-exit criterion?
  5. Does the fixed-point structure induced by STARS yield interpretability benefits?
