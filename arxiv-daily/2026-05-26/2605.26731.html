It’s Not the Capability: Harness Sensitivity Is Non-Monotone Across LLM Agent Tiers

Yong-eun Cho KailosLab, Seoul May 2026

Key Finding

Optimal harness complexity depends on model type (chat vs. reasoning) and instruction-tuning quality, not capability tier alone.

Key Metric
−38pp

Strict harness drops Gemini 2.5 Flash VTSR by 29–38 percentage points; Qwen3.5-122B hits 91.7% under strict.

Why This Matters

One-size-fits-all harness strategies are suboptimal—the right framework depends on model type, not parameter count.

Critical Edge

k = 1 repeat per condition and one model per tier limits generalization. Chat/reasoning distinction was post-hoc, not pre-registered.

Methodology

Experimental Design

432-run controlled factorial experiment: 6 models × 4 capability tiers × 3 harness conditions. Deterministic binary verifiers on a shared git-initialized synthetic workspace (12 files).

Strengths

  • Clean, well-controlled factorial design with transparent methodology
  • Genuine hypothesis testing with clear refutation, not just observational correlations
  • Immediately practical: tier-aware harness selection guidelines
  • Comprehensive six-label failure taxonomy adds diagnostic value beyond aggregate metrics
  • Gemma4:e2B finding challenges the assumption that parameter count predicts capability
  • Exceptionally honest limitations section identifying specific threats to validity

Limitations

  • Single repeat per condition (k = 1) — reliability requires k ≥ 3
  • One model per tier — cannot separate model-specific from tier-level effects
  • Synthetic workspace with 12 files — may not transfer to production engineering
  • 4096-token context limit for constrained models may artificially penalize output
  • Cannot isolate extended thinking contribution from harness effect for Qwen3.5-122B
  • Chat/reasoning distinction identified post-hoc, not pre-registered
  • GPT-OSS-120B strict condition confounded by re-evaluation with different API key

Questions for Further Exploration

  1. Would the harness-complexity paradox hold across other frontier chat models (GPT-4.1, Claude 3.5/4, Llama 4)?
  2. Does separating process instructions from output-format specifications eliminate format violations under strict harness?
  3. How does the chat vs. reasoning distinction interact with harness complexity across different task domains?
  4. Can a short harness probe (4–6 tasks from HEAT-24) reliably predict optimal harness condition for an unseen model?
  5. What role does the 4096-token context limit play in constrained-tier performance — would unconstrained context change the inverted-U pattern?
