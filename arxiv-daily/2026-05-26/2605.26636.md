> **一句话总结：** JetViT 提出了一种后训练注意力搜索框架，将预训练的全注意力 ViT 转换为混合注意力模型（线性注意力 + 窗口注意力 + 少量全注意力），在 DINOv3 和 DepthAnythingV2 上实现高达 1.79 倍的推理加速且不损失精度。

> **TL;DR:** JetViT converts pre-trained full-attention ViTs into efficient hybrid-attention models via Post-Training Attention Search, achieving up to 1.79x throughput speedup and 44.81% latency reduction on NVIDIA H100 without accuracy loss on DINOv3 and DepthAnythingV2.

## 核心贡献

- 提出 **Post-Training Attention Search** 框架，通过两阶段蒸馏+搜索流程将全注意力 ViT 转换为混合注意力模型，继承全部权重（包括注意力权重），无需访问原始预训练数据
- 设计了 **JetViT 线性注意力块**，结合 ReLU 线性注意力与轻量级 Squeeze Dynamic Convolution，在几乎不增加计算开销的情况下显著提升性能
- 在 DINOv3（Base/Large/7B）和 DepthAnythingV2（Large/Giant）上验证，仅需保留 **2 个全注意力层** 即可恢复原始模型性能

## Key Contributions

- **Post-Training Attention Search**: A two-stage distillation-and-search framework that converts pre-trained full-attention ViTs into hybrid models (linear + window + minimal full-attention blocks) while inheriting all weights, eliminating the need for proprietary pre-training data.
- **JetViT Linear Attention Block**: Combines ReLU-based linear attention with a lightweight Squeeze Dynamic Convolution (kernel generated by a compact MLP from average-pooled value matrix), achieving better performance than prior linear attention designs with negligible overhead.
- Demonstrated on two SOTA vision foundation models (DINOv3, DepthAnythingV2), showing that only **2 full-attention blocks** are sufficient to recover full model accuracy while delivering substantial speedups.

## Methodology

**Three-step pipeline:**

1. **Linear-Attention Block Optimization**: Replaces static depth-wise convolution with Squeeze Dynamic Convolution. A two-layer MLP with SiLU generates dynamic convolution kernels from the average-pooled value matrix V, introducing minimal overhead while improving feature quality.

2. **Linear + Window Attention Search (Step 1)**: Constructs a supernetwork with two attention choices per layer (linear vs. window). Distills from the full-attention teacher using feature-level distillation, then applies beam search to find the optimal LA/WA combination. Key insight: inheriting all attention weights (W_Q, W_K, W_V, W_O) is critical — it yields +3% mIoU over inheriting only MLP weights.

3. **Full-Attention Placement (Step 2)**: Builds another supernetwork where each layer offers the Step-1 searched efficient block or full-attention. Beam search identifies the optimal positions for full-attention blocks, targeting the downstream task metric directly.

**Training details**: Uses a mixture of open datasets (SA1B, ImageNet21K, BDD100K, etc.). Distillation applies to features used for the downstream task. Search cost is ~1/68 of DINOv3-7B's pre-training cost.

- Edge devices (RTX 3090, Jetson GB10): up to 2.4x speedup at 2K resolution

## Critical Assessment

**Novelty (7/10)**: The idea of converting full-attention models to hybrid post-training is timely and well-motivated. The Squeeze Dynamic Convolution is a reasonable but incremental improvement over existing linear attention designs. The two-stage search procedure is systematic, though beam search over block types is not fundamentally new.

**Rigor (8/10)**: Comprehensive ablations — linear attention block design (Tab. 1), LA vs. WA vs. hybrid (Tab. 2), full-attention placement analysis (Fig. 4, Fig. 7), weight inheritance study (Fig. 5). Evaluated on two diverse foundation models across multiple tasks (segmentation, depth estimation). Speed measurements on real hardware with transparent setup.

**Reproducibility (6/10)**: Code promised but not yet released. Uses a mix of open datasets for distillation, but the base models (DINOv3, DepthAnythingV2) require their own pre-training pipelines. The search procedure details (beam search hyperparameters, supernetwork training schedule) could be more explicit.

**Significance (8/10)**: Directly addresses a practical and important problem — making large vision foundation models efficient for high-resolution inference. The finding that only 2 full-attention layers suffice is surprising and impactful. The framework is generalizable to other ViT-based models.

**Red flags**: Code not yet available at time of publication. The speedup numbers are GPU-specific (H100); actual gains may vary on other hardware. The method requires per-task distillation and search, not a single-shot conversion.

## Strengths

- No need for proprietary pre-training data or costly retraining from scratch
- General framework applicable to any full-attention ViT backbone
- Strong empirical results with minimal accuracy loss and significant speedups
- Insightful analysis on where full-attention matters (varies by downstream task: middle layers for depth, late layers for segmentation)
- First attention layer is always linear-attention, echoing CNN-like local feature extraction in shallow layers

## Limitations

- Requires separate distillation and search per downstream task, adding engineering overhead
- Linear attention blocks are implemented in pure PyTorch without custom kernels — further speedups may be possible but aren't demonstrated
- Not evaluated on generative vision tasks or video understanding at scale
- The search process itself, while cheaper than pre-training, still requires non-trivial GPU resources
- Performance on smaller models (e.g., ViT-Small) or lower resolutions is not explored

## Relevance

该论文对高效视觉模型设计具有直接参考价值。后训练注意力搜索的思路可推广至其他全注意力架构（如多模态模型中的视觉编码器）。仅保留 2 个全注意力层即可恢复性能的发现，为理解 ViT 中哪些层真正需要全局注意力提供了重要实证。

## Questions for Further Exploration

- Can Post-Training Attention Search be applied to multimodal models (e.g., CLIP, LLaVA) where the vision encoder interacts with a language model?
- How does the optimal full-attention placement change with resolution scaling beyond 4K/8K?
- Would combining JetViT with quantization or pruning yield compound efficiency gains?
- Can the search be made task-agnostic (universal placement) to avoid per-task re-search?
- How do the results transfer to video transformers where temporal attention adds another dimension?
