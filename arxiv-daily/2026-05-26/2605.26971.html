RLVR Datasets and Where to Find Them: Tracing Data Lineage for Better Training Data

Huang, Liu, Tang et al. PKU & Tencent 2026-05-26 Tags: RLVR · data-provenance · contamination
Key Finding

ATLAS traces 1.45M RLVR instances to just 20 atomic sources, exposing massive overlap and 36K benchmark leaks across popular datasets.

Key Metric
1.45M
instances → 20 sources (<1% unknown)
36,148 leaked benchmark instances
Why This Matters

Most RLVR datasets recombine the same upstream data with widespread benchmark leakage — questioning the integrity of reported gains.

Critical Edge

Limited to math reasoning. SCA requires 20 separate RL checkpoints. Quality score Q not validated beyond Qwen3.

Methodology

ATLAS Framework (Data Lineage Tracing)

  1. Stage 0 — Collection & Canonicalization. Gather RLVR datasets and upstream sources; manually inspect and standardize question-answer pairs across heterogeneous schemas.
  2. Stage 1 — Temporal Index Matching. SHA-1 hash-based indexing matches prompts across datasets chronologically, building a global lineage dictionary.
  3. Stage 2 — Semantic Similarity Matching. Unmatched instances go through Sentence-BERT embeddings + cosine similarity retrieval + manual human auditing.
  4. Stage 3 — Iterative Source Recovery. Repeatedly identify and incorporate candidate datasets until unmatched instances fall below 1%.

Source-level Counterfactual Attribution (SCA)

Trains separate RL checkpoints on each atomic source using GRPO, comparing against a shared base model. Each instance is classified into one of four categories:

  1. Learnable — base wrong, RL right (the useful signal)
  2. Too easy — both correct
  3. Too hard — both wrong
  4. Degraded — base right, RL wrong

The composite quality score Q aggregates three dimensions: Static Quality S1 (verifiability, learnability, contamination), Sampling Efficiency Gain S2, and Capability Boundary Expansion S3. Training uses VERL + GRPO, temperature 1.0, 500 steps, evaluated on Math500.

  • Largest atomic sources: cn_k12 (342K), olympiads (297K), synthetic_math (150K), orca_math (125K)
  • Major downstream datasets (NuminaMath-CoT, NuminaMath-1.5, LILA) are recompositions of the same upstream content
  • Eurus-2-RL-Data contributed MCQ-to-open-ended format transformations widely inherited by others
  • Some RLVR datasets directly include evaluation benchmarks (OmniMath, HARP) as training data
  • DAPO++ (decontaminated) consistently outperforms all baselines across Qwen3-1.7B and Qwen3-8B
  • Quality score Q reliably predicts downstream RLVR training effectiveness
  • Decontamination ablation: removing leaks generally improves or maintains performance
  • Open-ended formats provide stronger RLVR training signals than multiple-choice questions

Strengths

  • First comprehensive provenance analysis of RLVR datasets — fills a critical literature gap
  • Identifies concrete, actionable problems: benchmark leakage, dataset overlap, MCQ vs. open-ended
  • SCA provides a principled way to evaluate data source utility via counterfactual training
  • DAPO++ demonstrates practical gains across model scales and held-out benchmarks
  • Quality score Q offers a practical dataset selection tool correlating with downstream performance
  • Open-source code, data, and detailed documentation

Limitations

  • Scope limited to mathematical reasoning RLVR; code, science, and other domains unexplored
  • ATLAS pipeline requires substantial manual effort for canonicalization and auditing
  • SCA requires one RL checkpoint per atomic source (20 checkpoints) — computationally prohibitive at scale
  • Q hyperparameters tuned via grid search on Qwen3; transferability to other model families unvalidated
  • Interaction effects between atomic sources not explored (sources evaluated independently)
  • Results primarily on Qwen3 series; robustness across diverse architectures not demonstrated

Questions for Further Exploration

  1. How does ATLAS scale to non-mathematical RLVR domains (code generation, scientific reasoning, tool use)?
  2. Can SCA be made more efficient — proxy models or gradient-based attribution instead of full RL retraining?
  3. What are the interaction effects between atomic sources — superlinear or sublinear gains from combinations?
  4. How stable is the quality score Q across different base model architectures (Llama, Mistral, etc.)?
  5. How should the RLVR community establish standardized provenance metadata to prevent future provenance collapse?
