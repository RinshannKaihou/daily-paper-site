> **一句话总结：** 本文提出了一种针对 Wan2.2-I2V 视频 DiT 的 W4A4 量化框架，结合 SVDQuant 低秩补偿、GPTQ 权重量化和时间步感知的激活裁剪比例搜索，在峰值显存降低 59.3% 的同时仅导致 VBench 得分下降 0.9%。

> **TL;DR:** A post-training W4A4 quantization framework for Wan2.2-I2V combining SVDQuant outlier absorption, GPTQ reconstruction-aware weight quantization, and timestep-bin-wise activation clipping, achieving 59.3% peak memory reduction with only 0.9% VBench degradation.

## 核心贡献

- **SVDQuant + GPTQ 混合量化框架：** 将 SVDQuant 的低秩异常值吸收与 GPTQ 的二阶信息感知残差权重量化相结合，而非简单的最近舍入，提供了更强的 W4A4 基线。
- **时间步感知的激活裁剪比例搜索：** 将去噪轨迹划分为多个时间步区间，为每个（层、区间）对独立搜索最优激活裁剪比例，使激活量化范围自适应于去噪阶段，推理时仅需查找表。
- **专家级独立校准：** 针对 Wan2.2 的双专家 MoE 结构（高噪声专家和低噪声专家），分别独立进行裁剪比例搜索，捕捉两个专家截然不同的量化敏感度。
- **详细的模块级敏感性分析：** 发现 `self_attn.o` 和 `ffn.2` 是 W4A4 量化中最脆弱的模块，低噪声专家比高噪声专家更敏感。

## Key Contributions

- A hybrid W4A4 PTQ framework for MoE video DiTs that integrates SVDQuant-based low-rank outlier compensation with GPTQ-based reconstruction-aware residual weight quantization.
- A timestep-bin-wise per-layer activation clipping-ratio search that adapts activation quantization ranges to the denoising stage with negligible runtime overhead (lookup table only).
- Expert-specific calibration for Wan2.2's dual-expert MoE design, recognizing that the low-noise expert (texture refinement) is systematically more fragile than the high-noise expert (global layout).
- Comprehensive module-level sensitivity analysis revealing that attention output projections (`self_attn.o`) and FFN down projections (`ffn.2`) are the primary sources of W4A4 degradation.

## Methodology

The framework operates in three stages during calibration:

1. **SVDQuant Decomposition:** For each linear layer, apply channel-wise smoothing to transfer activation outliers to weights, then decompose the transformed weight into a high-precision low-rank branch ($L_1 L_2$) and a 4-bit residual branch ($R$).

2. **GPTQ on Residual:** Instead of round-to-nearest for the residual branch, apply GPTQ which uses approximate second-order (Hessian-based) weight sensitivity information to minimize layer-wise reconstruction error under the 4-bit constraint.

3. **Timestep-Bin-Wise Clipping Search:** Divide the denoising trajectory into bins (early, mid, boundary, late). For each (layer, bin) pair, search the optimal clipping ratio $\rho$ from candidates $\{0.8, 0.9\}$ by minimizing the MSE between full-precision and quantized layer outputs on calibration data. This is performed independently for each MoE expert.

At inference, the quantized output is:
$$\hat{Y}_l(t) = \hat{X}_l L_{l_1} L_{l_2} + Q_A(\hat{X}_l; s^A_{l,k}) \hat{R}^{GPTQ}_l, \quad t \in B_k$$

No online search is needed — only a lookup table indexed by the current timestep.

## Results & Highlights

- **Peak GPU memory:** Reduced from 64.4 GB (BF16) to 26.2 GB (**59.3% reduction**).
- **VBench average score:** 0.799 vs. 0.806 baseline (**only 0.9% drop**).
- **Imaging Quality:** 0.689 vs. 0.705 baseline (2.3% drop); compared to naive RTN W4A4 at 0.389, the improvement is massive.
- **Motion Smoothness:** 0.975, matching BF16 exactly.
- **Model size:** Reduced from 64.0 GB to 25.9 GB (59.5% reduction).
- Ablation: SVDQuant+GPTQ without timestep clipping scores 0.793; adding timestep clipping brings it to 0.799.
- Keeping the most sensitive modules (`self_attn.o`, `ffn.2`) in higher precision further improves VBench to 0.803 (diagnostic variant).

## Critical Assessment

- **Novelty (moderate):** The individual components (SVDQuant, GPTQ, timestep-aware clipping) are not individually novel. The contribution lies in their systematic combination and the expert-specific calibration for MoE DiTs. The clipping ratio search space is limited to only two candidates (0.8, 0.9), which is pragmatic but restricts optimality.
- **Rigor (good):** Well-designed ablation study isolating the contribution of each component. Module-level and expert-level sensitivity analyses are thorough and informative.
- **Reproducibility (good):** Code and weights publicly available. The method uses standard components and clear calibration procedures.
- **Significance (moderate-high):** Addresses a practical deployment problem for frontier video generation models. The 59.3% memory reduction is substantial and enables deployment on consumer hardware.
- **Red flags:**
  - The current implementation uses pseudo-quantization (dequantize before matmul), resulting in **1.83x latency increase** (955s vs. 521s for BF16). The practical speedup depends entirely on future hardware/kernel support for native FP4/MXFP4.
  - Evaluation is limited to a single benchmark (OpenS2V-Eval) and a single model (Wan2.2-I2V-A14B). Generalizability to other video DiTs is not demonstrated.
  - The number of timestep bins and the candidate clipping ratios are hyperparameters whose sensitivity is not fully explored.

## Strengths

- Clean, well-motivated pipeline that addresses a real deployment bottleneck for large video generation models.
- Expert-level and timestep-level calibration is a principled approach to handling MoE-specific and diffusion-specific quantization challenges.
- Strong empirical results with clear ablation evidence for each component's contribution.
- Excellent sensitivity analysis that provides actionable insights for future W4A4 quantization work on MoE architectures.

## Limitations

- No actual inference speedup — pseudo-quantization increases latency by 83%. Practical benefits are currently limited to memory savings only.
- Evaluated on only one model and one benchmark; generalizability is uncertain.
- The low-rank branch adds compute overhead that partially offsets the memory savings from quantization.
- The clipping ratio search is constrained to a very small candidate set; a continuous optimization might yield better results.

## Relevance

与"计算可靠性与精度错误"研究方向高度相关：本文系统分析了 4-bit 低精度量化在视频扩散 Transformer 中的误差传播机制，提出了针对性的异常值补偿和自适应裁剪策略来缓解量化精度损失，是量化误差分析与控制的典型工作。

## Questions for Further Exploration

1. How does the framework generalize to other MoE video DiT architectures (e.g., HunyuanVideo, Sora-like models)?
2. Would a continuous optimization of the clipping ratio (rather than a discrete search over {0.8, 0.9}) yield meaningful improvements?
3. Can the low-rank branch be further quantized (e.g., to INT8) to reduce its overhead without significant quality loss?
4. What is the impact of the number of timestep bins on the quality-efficiency trade-off?
5. How would fused dequantization-GEMM kernels (e.g., via TensorRT or custom CUDA) affect the latency profile?
