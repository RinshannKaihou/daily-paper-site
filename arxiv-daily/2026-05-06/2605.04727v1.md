> **一句话总结：** 本文通过审计编程知识追踪（PKT）领域主流模型（Code-DKT、ECKT）的官方实现，发现注意力机制的 Softmax 轴向错误和序列时间戳排列缺失共同导致数据泄漏，修正后注意力增强模型相对朴素 DKT 的性能优势大幅缩小甚至消失。

> **TL;DR:** This paper audits representative Programming Knowledge Tracing (PKT) models and shows that their reported performance gains stem largely from two implementation flaws—a causally invalid Softmax axis in attention-based code representation (leaking future context across time steps) and a failure to sort student attempts by ServerTimestamp (introducing look-ahead bias). After correcting these issues and applying assignment-wise hyperparameter tuning, the performance gap between attention-augmented models and the standard DKT baseline is substantially reduced, and traditional DKT can match or outperform more complex architectures.

## 核心贡献

- **因果无效性识别**：发现 Code-DKT 官方实现中 Softmax 操作沿时间维度（dim=1）而非路径维度（dim=2）进行归一化，导致每个时间步的注意力权重依赖于未来序列信息，构成数据泄漏。
- **时序因果违规证明**：实证确认当交互记录未按 ServerTimestamp 严格排序时，模型可利用未来调试成功信息预测过去结果，系统性高估性能。
- **受控评估协议**：提出作业级交叉验证框架——在单一指定折（fold₀）上通过网格搜索确定最优超参数，随后固定并跨所有折统一应用，消除超参数敏感性干扰。
- **实验再评估**：在 CodeWorkout 数据集五个作业上重新评估，显示修正后 DKT 与 PKT 模型差距大幅缩减，在部分任务上 DKT 反超复杂模型。
- **序列长度分析**：系统比较 L_max=50 与 L_max=100 的影响，发现延长上下文窗口不能稳定提升性能，常引入噪声。

## Key Contributions

- **Causal Invalidity Audit (CRect):** Identifies that the Softmax in Code-DKT's Score-Attended Path Selection is applied across the temporal dimension (dim=1) rather than the path dimension (dim=2), allowing attention weights at time step t to be influenced by future interactions—a fundamental violation of causal temporal ordering.
- **Temporal Alignment Fix (TAlign):** Documents and corrects the failure to sort student attempt sequences by ServerTimestamp in the official codebase, which introduced look-ahead bias by allowing failed future attempts to precede earlier successes in the input sequence.
- **Assignment-wise Controlled Evaluation Protocol:** Introduces a reproducible benchmarking framework where optimal hyperparameters are selected via grid search on a single designated validation fold (fold₀) and then frozen uniformly across all subsequent folds per assignment.
- **Empirical Re-evaluation on CodeWorkout:** Re-runs DKT, Code-DKT, and ECKT under corrected conditions, finding that the previously claimed 0.03–0.04 AUC advantage of attention-augmented models over DKT largely dissolves.
- **Sequence Length Impact Study:** Systematically tests L_max=50 vs. L_max=100, concluding that extending sequence length does not reliably improve performance and can degrade it by introducing irrelevant interactions.

## Methodology

The paper proceeds in three phases:

**Phase 1 — Reliability Audit of Existing Implementations**
The authors execute the official open-source Code-DKT code (Yang Shi, GitHub 2026) and replicate its originally reported AUC gains (~0.03–0.04 over DKT). During replication they identify two structural flaws:
1. *CRect (Causal Rectification):* The attention mechanism computes path weights using `softmax(dim=1)`, normalizing across the sequence length dimension. This means a code submission's representation at time t incorporates information about all other time steps, violating the requirement that predictions condition only on past data. The fix is to normalize across the path dimension (dim=2) so that attention operates solely within a single code submission.
2. *TAlign (Temporal Alignment):* The codebase constructs interaction sequences using raw CSV row order rather than chronological ServerTimestamp order. In programming datasets with rapid "submit-and-fix" cycles, CSV order frequently misrepresents the actual temporal order of attempts, introducing explicit look-ahead bias. The fix enforces strict ascending sort by ServerTimestamp before sequence construction.

**Phase 2 — Controlled Evaluation Protocol**
To address hyperparameter sensitivity—which the authors show varies substantially both across model types and across programming assignments—they propose an assignment-wise protocol:
- Data is split 80/20 (train/test) per assignment; training is further partitioned into 5 folds.
- Grid search is performed exclusively on fold₀ to identify optimal hyperparameters θ* (learning rate, embedding size, dropout).
- θ* is fixed for all remaining folds, preventing hyperparameter-induced overfitting and ensuring that performance variation reflects data variability alone.
- Fixed independent random seeds govern train/test splits, cross-validation splits, and model initialization separately.

**Phase 3 — Comparative Analysis**
Three models are evaluated on the CodeWorkout dataset (69,627 interactions, 413 students, 5 assignments): DKT (LSTM baseline), Code-DKT (code2vec + LSTM with AST path attention), and ECKT (Code-DKT extended with large language model-based semantic encoders). Each is evaluated under combinations of TAlign and CRect corrections, with and without task-specific hyperparameter tuning, and at both L_max=50 and L_max=100.

## Results & Highlights

- **Causal correction deflates gains:** Applying CRect to Code-DKT consistently reduces AUC relative to the uncorrected baseline (e.g., assignment 439: 0.7445 → 0.6914 with CRect). This confirms that the original reported improvements were partially an artifact of the softmax axis error.
- **Temporal alignment has mixed effects:** TAlign sometimes decreases and sometimes slightly increases AUC for individual assignments, suggesting the original results were entangled with both leakage and underfitting artifacts. The effect is not uniform.
- **Task-specific tuning rescues DKT:** Under proper per-assignment hyperparameter optimization, DKT improves substantially (e.g., assignment 439: 0.6821 → 0.7480 AUC, a gain of ~0.07 purely from tuning). This closes or reverses the gap against Code-DKT and ECKT on several assignments.
- **ECKT is more robust than Code-DKT:** After causal rectification, ECKT (CRect) consistently matches or outperforms DKT and narrows Code-DKT's weaknesses, suggesting that structured LLM-based semantic integration confers more stable benefits than raw code2vec concatenation.
- **Longer sequences do not help:** Increasing L_max from 50 to 100 produces no consistent improvement. Negative delta values (AUC decreases) appear more frequently and consistently than positive ones, across all model families and datasets.
- **Architectural complexity does not guarantee superiority:** Under fair conditions, the performance advantage of attention-enhanced PKT models is task-dependent. DKT remains competitive, and in some assignments outperforms more complex architectures.

## Critical Assessment

**Novelty:** Moderate. The findings are not algorithmically novel — no new model is proposed. The contribution is methodological: identifying concrete implementation bugs and proposing a standardized evaluation protocol. This type of audit work is undervalued but important. The softmax axis error (dim=1 vs. dim=2) is a subtle but consequential bug that is easy to miss in code review.

**Rigor:** Good within its scope. The authors replicate the original results before modifying anything, establishing a credible baseline. The controlled protocol (single fold for hyperparameter search, fixed seeds) is methodologically sound. The choice to compare L_max=50 vs. 100 is systematic. However, reporting only mean ± std across 5 folds without statistical significance tests (e.g., paired t-tests across assignments) weakens the strength of comparative claims.

**Reproducibility:** High. Code is publicly released. Dataset (CodeWorkout) is publicly available. The paper describes grid search ranges, fold configurations, and seeding strategies in sufficient detail.

**Significance:** Meaningful for the PKT subfield. The paper challenges the credibility of reported progress in Code-DKT and ECKT, which are actively cited in PKT research. The broader implication — that evaluation protocol choices can systematically inflate performance of complex models relative to simpler baselines — is relevant beyond PKT.

**Red Flags:**
- Results are restricted to a single dataset (CodeWorkout) with 5 assignments. Generalization claims to other PKT datasets or architectures are not empirically supported.
- The paper does not test whether ECKT itself suffers from similar causal issues in its LLM-based encoding path, only the CRect from the Code-DKT component is analyzed.
- The claim that "architectural complexity does not consistently translate to superior performance" is arguably overstated given the narrow experimental scope.
- No ablation distinguishing the individual contributions of TAlign vs. CRect vs. task-specific tuning in a factorial design; effects are partly confounded.

## Strengths

- Identifies a real, verifiable implementation bug (Softmax dim error) with clear causal reasoning and demonstrates its empirical impact.
- Proposes a concrete, reproducible evaluation protocol with well-specified hyperparameter selection and seeding strategies.
- Demonstrates that task-specific hyperparameter tuning alone (without any architectural change) can recover DKT performance to competitive levels — a practically important finding for the field.
- Code released publicly, enabling the community to adopt the corrected protocol.
- Intellectually honest about the limitations of the scope (single dataset, two target models).

## Limitations

- Single dataset (CodeWorkout) with 5 programming assignments limits external validity; findings may not generalize to other PKT datasets (e.g., ACcoding) or non-programming KT domains.
- No new architecture or training innovation is proposed; the paper is corrective rather than constructive.
- The analysis of ECKT is partial — only the CRect fix inherited from Code-DKT architecture is applied; potential LLM-encoding-specific issues are not investigated.
- L_max study only compares two values (50 and 100); a finer-grained sweep could reveal whether there is an optimal range.
- Absence of statistical significance tests makes it difficult to determine whether performance differences after correction are meaningful or within noise.
- The authors note the proposed protocol should be extended to broader domains and architectures, but do not do so themselves.

## Relevance

本文直接关切模型评估的可靠性与可重复性——这与 LLM 可解释性/可靠性研究中的核心议题高度一致：实现层面的细节（如注意力维度）可以系统性地扭曲性能评估，而超参数敏感性和时序因果泄漏也是训练稳定性研究中的常见陷阱；本文的受控评估框架为类似领域提供了直接可借鉴的方法论范例。

## Questions for Further Exploration

1. Do the same softmax axis and temporal alignment issues exist in other PKT or general KT models that use attention over code or sequence features? A broader audit across the KT literature would be valuable.
2. Can the proposed assignment-wise evaluation protocol be automated and packaged as a standard benchmarking library for the KT community, similar to how ML reproducibility frameworks are structured?
3. Is the diminished performance gap between DKT and attention-augmented models consistent across other programming datasets (e.g., ACcoding) and across different programming languages or task types?
4. For ECKT specifically: does the LLM-based semantic encoding path introduce its own temporal leakage or data contamination risks (e.g., if the LLM was pretrained on code that overlaps with the student submissions)?
5. What is the interaction between sequence length and assignment difficulty? The authors find L_max=100 does not help on average — does this hold for assignments with high attempt counts or complex debugging patterns?
6. Could a lightweight model trained on properly curated, causally valid features (e.g., attempt timing, error type distributions) match or exceed the performance of code2vec-based models on this dataset?
