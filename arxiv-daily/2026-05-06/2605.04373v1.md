> **一句话总结：** REGUARD将RL网络控制器的最坏情形发现形式化为双层遗憾最大化问题，并通过反事实分析将发现的失败轨迹转化为轻量级逻辑规则，在不重训练的情况下对三种控制器（拥塞控制Sage、自适应码率Pensieve、负载均衡Park）实现推理时保护，将最坏情形性能差距缩减达85%。

> **TL;DR:** REGUARD discovers worst-case network scenarios for deployed RL-based controllers by framing the search as bilevel regret maximization over feasible network conditions, then distills the discovered failures into lightweight interpretable logic rules that intervene at inference time — closing up to 85% of worst-case performance gaps without retraining and without sacrificing nominal performance.

## 核心贡献

- **双层遗憾最大化：** 将最坏情形场景发现形式化为双层优化问题：外层RL求解器搜索令目标控制器遗憾最大的网络场景序列，内层从启发式策略组合中选取该场景下的最优参考策略，同时提供可证明的最坏情形下界。
- **反事实分析器：** 将高遗憾轨迹转化为状态级监督信号，通过与参考策略对比，为每个控制器决策打上 ABSTAIN / BACK\_OFF / PUSH\_HARDER 标签，揭示具体的纠正方向。
- **NetNomos规则学习：** 基于可观测状态谓词学习可解释的逻辑规则，仅在检测到风险状态时介入，保留控制器在正常条件下的行为。
- **迭代精化循环：** 针对当前受保护控制器重新运行场景搜索，聚合数据集后从头重新学习规则，避免规则冲突积累并持续扩展覆盖范围。

## Key Contributions

- **Bilevel regret maximization formulation:** The outer RL solver searches over feasible network scenario sequences to maximize the target controller's regret (gap between what it achieves and what is achievable). The inner solver selects the best reference policy from a portfolio of established networking heuristics for each candidate scenario, providing a certified lower bound on worst-case regret (Theorem 1).
- **Certified failure discovery:** The returned scenario comes with a formal guarantee: its exact regret is within epsilon + delta(e*) of the true worst case, where epsilon is outer-solver suboptimality and delta(e*) is the inner-reference error at the worst-case scenario. Both error sources are explicit and improvable independently.
- **Counterfactual Analyzer:** Converts trajectory-level failures into state-level supervision by comparing controller and reference actions under identical conditions, labeling each controller decision point with one of {ABSTAIN, BACK\_OFF, PUSH\_HARDER} — providing actionable correction directions rather than just failure certificates.
- **Inference-time logic rule protection:** NetNomos learns threshold-based conjunctive logic rules over observable controller state predicates. Rules are evaluated memorylessly at each decision step, operate with a prefix trie for efficiency, and add negligible latency within each controller's inference budget.
- **Iterative refinement:** Multiple search-and-protect rounds drive the worst-case gap close to the Normal regime; most improvement arrives within two iterations.

## Methodology

**Problem formulation.** A network scenario e = (u_0, ..., u_{T-1}) is a time-varying sequence of resource conditions. The target pretrained controller pi has trajectory reward J(pi; e). The reference policy pi*_e = argmax_{pi' in Pi} J(pi'; e) is scenario-specific optimal over the heuristic portfolio Pi. Regret R(e) = J(pi*_e; e) - J(pi; e) measures avoidable performance loss. REGUARD solves the bilevel program: maximize R(e) over feasible e, subject to pi*_e being optimal in its class (Eqn. 3/4).

**Outer RL solver.** RL is used only as a search tool over scenario sequences — it is not the controller policy. The solver observes the shared scenario, evaluates both pi and the portfolio, and receives the approximate regret as reward signal. This avoids modeling the controller or network dynamics explicitly.

**Inner reference portfolio.** For Sage (congestion control), the portfolio includes BBR, Cubic, Reno, and NewReno. For Pensieve (ABR streaming), an exact K-step rolling oracle. For Park (load balancing), LCT, JSQ, and CFS. The upper envelope of these heuristics provides a tight practical approximation to scenario-specific optimum.

**Counterfactual Analyzer.** At each time t in a high-regret trajectory, it records state s_t, controller action a_t = pi(s_t), reference action from pi*_e, and both rewards. It determines a state is risky (Risky(s_t) = 1) only when J(pi*_e; e) - J(pi; e) is large, keeping focus on avoidable failures. Corrective labels come from a small fixed set: ABSTAIN (ambiguous or small regret), BACK\_OFF (controller too aggressive), PUSH\_HARDER (controller too conservative).

**Rule learning and enforcement.** NetNomos learns implications of the form phi_1(s) AND ... AND phi_k(s) => adjustment(s) = c, where each phi_i is a threshold predicate over observable state variables (computed as percentiles of normal-trace distributions). Enforcement at runtime: evaluate g_R(s_t) to get label c_t; if ABSTAIN, pass through; if BACK\_OFF or PUSH\_HARDER, apply a bounded nudge h(pi(s_t), c_t, s_t) — for Sage, a small additive/subtractive clip on the scalar action; for Park and Pensieve, a safe-action-set masking approach.

**Implementation.** For Sage, REGUARD uses a synchronized multi-policy Mahimahi emulation with isolated child instances per policy, an online control plane for link parameter injection, and counterfactual consistency enforced by shared logical timestamps. For Pensieve, a chunk-level ABR simulator with rolling K-step oracle. For Park, a snapshot-consistent closed-loop workload generator.

## Results & Highlights

- **Finding 1 — Superior discovery:** REGUARD exposes performance gaps 7x larger than the next-best baseline (Indago) for Pensieve, 1.6x for Sage, and 6.2x for Park. Mean performance gaps are 4.84 (log scale) for Pensieve, 63.84% for Sage, and 33.45% for Park — versus Normal-regime gaps of 1.21, 1.66%, and 0.29% respectively.
- **Finding 2 — Strongest protection:** REGUARD provides the strongest normalized protection without retraining. On the most challenging scenario family, it removes about 80% of the normalized performance gap for Pensieve, 80% for Sage, and 79% for Park. Fine-tuning and curriculum learning (Genet) often *increase* the Normal gap by up to 24% (Pensieve) and 81% (Park).
- **Finding 3 — Harder counterfactuals yield better protection:** Protection quality scales with discovery quality, confirming the search stage is not merely selecting hard examples but producing high-gap states that make rule learning effective.
- **Finding 4 — Deployment feasible:** Decision-time ratio stays well below 100% across all controllers and refinement iterations (11.22–32.36% of Pensieve's budget, 47.55–50.11% of Sage's, 1.85–6.52% of Park's).
- **Finding 5 — Iterative improvement:** By iteration 3, mean discovered gaps fall ~75% for Pensieve, ~94% for Sage, and ~93% for Park from their iteration-0 values, approaching the Normal regime.
- **Qualitative analysis (Park):** REGUARD's rules expose a structural dispatch bias — Park systematically routes small jobs to slow servers even when fast servers are idle (placing 97.3% of <= 0.2KB jobs to a slow server vs. LCT's 18.9%). Representative rules are human-readable: e.g., "(JobSize <= p25) AND (GapToMin_1 > p25) AND (PostLoad_9 <= p25) AND (StdLoad > p25) => Risky = true."
- **Generalization:** REGUARD-protected controllers outperform unprotected counterparts even on scenario families generated by *other* discovery methods (Genet, Indago, Gilad et al.) that were never used as counterfactuals, indicating the rules capture genuine recurring vulnerabilities rather than overfitting.

## Critical Assessment

**Novelty:** The bilevel regret maximization framing for worst-case discovery in closed-loop RL network controllers is a meaningful contribution. Prior adversarial approaches (PGD, Indago, Gilad et al.) either violate causal dependencies of the closed-loop system or lack formal guarantees. The coupling of formal worst-case bounds with counterfactual-derived inference-time rules is a coherent end-to-end design.

**Rigor:** The theoretical guarantee (Theorem 1) is clearly stated with explicit error sources (epsilon, delta(e*)). The empirical evaluation spans three qualitatively different networking tasks with multiple baselines and five evaluation questions, which is thorough. The synchronized multi-policy emulation design (isolated Mahimahi instances with shared logical timestamps) is a careful implementation choice to prevent policy contamination.

**Reproducibility:** The paper describes implementation in detail but does not (in the reviewed pages) point to a public code repository. The real-trace datasets (FCC broadband, Mahimahi cellular) are publicly available; the Sage (Mahimahi) and Park emulators are from published systems. Reproducing the full pipeline would require significant engineering effort.

**Significance:** High for the RL reliability and networked systems communities. The key insight — that worst-case scenario discovery and runtime protection are two sides of the same counterfactual coin — is practically actionable. The demonstration that fine-tuning-based protection consistently hurts nominal performance is an important negative result for practitioners considering retraining as a robustness remedy.

**Red flags:** The heuristic portfolio quality directly bounds the formal certificate (via delta(e*)). If the portfolio is weak for a given domain, the bound is loose and may give false assurance. The paper acknowledges this but the practical tightness of the bound is evaluated only implicitly through protection performance. Additionally, the three evaluated controllers (Sage, Pensieve, Park) are all well-studied benchmarks; generalization to newer RL-based controllers with different state spaces and action structures is not demonstrated.

## Strengths

- Clean theoretical formulation with an explicit, decomposable guarantee that separates outer-search error from inner-reference quality.
- End-to-end design: discovery, counterfactual analysis, rule learning, and runtime enforcement form a coherent pipeline where each stage enables the next.
- Lightweight, memoryless inference-time protection that preserves the controller's original decision-making except at identified risk points.
- Strong empirical result that fine-tuning and curriculum retraining consistently degrade nominal performance — a practically important negative finding.
- Interpretable rules (readable conjunctions of percentile predicates) allow operators to audit and understand what controller behaviors are being corrected.
- Generalization beyond source scenario families demonstrates the rules capture structural vulnerabilities rather than overfitting to specific scenarios.

## Limitations

- The formal guarantee depends on the portfolio heuristics' ability to approximate the scenario-specific optimum. In domains without strong established heuristics, delta(e*) could be large, weakening the certificate.
- Evaluated only on three specific RL controllers from the networking literature; generalization to other RL domains (e.g., resource scheduling, traffic engineering) is not demonstrated.
- The iterative refinement loop requires repeated expensive emulation runs; wall-clock cost of the full discovery-and-protection pipeline is not explicitly reported.
- Logic rules are learned from training-trace predicates computed over normal-operation percentiles; if the deployment distribution shifts significantly from training, rule predicates may become miscalibrated.
- No public code repository mentioned; reproducing the synchronized multi-policy Mahimahi implementation is non-trivial.

## Relevance

REGUARD直接切入RL模型的可靠性与可解释性问题：它不仅发现了RL控制器在最坏情形下的系统性缺陷，还将失败轨迹转化为人类可读的逻辑规则加以保护——这与LLM可解释性和训练稳定性研究中"从失败中提炼可审计知识"的核心诉求高度契合，对设计针对LLM的类似运行时干预机制有参考价值。

## Questions for Further Exploration

1. How sensitive is the formal regret bound to portfolio quality? Can the bound be empirically tightened by measuring delta(e*) in practice, and what portfolio size is sufficient for the bound to be deployment-meaningful?
2. The paper focuses on network controllers — does the bilevel regret maximization approach generalize to other RL domains (e.g., LLM-based agents) where the "scenario" space is natural language context rather than network parameters?
3. The rules are learned over percentile predicates from normal-trace distributions. How robust are these predicates to distribution shift in deployment (e.g., new network topologies, novel traffic patterns)?
4. Could REGUARD's counterfactual analysis be extended to identify which specific training scenarios (or training distribution properties) are responsible for the discovered worst-case vulnerabilities, bridging to training-time remediation?
5. The protection deliberately preserves nominal performance by intervening only at risk states. Is there a formal characterization of the trade-off between risk-state coverage and nominal-performance preservation as the rule set grows?
