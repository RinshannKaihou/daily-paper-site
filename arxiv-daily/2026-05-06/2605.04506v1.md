> **一句话总结：** ILOV3Splat 在 3D 高斯散射（3D-GS）框架内，通过多分辨率哈希编码的 CLIP 语言特征场与 SAM 掩码监督的实例对比特征场联合优化，配合两阶段（HDBSCAN + DBSCAN）3D 聚类，实现了无需类别标注的开放词汇实例级 3D 目标检索与分割，在 LERF 数据集 3D 目标选择任务上 mAcc 达 55.15，在 ScanNet 实例分割任务 mAcc 上以 59.29 超越所有无深度依赖的基线方法。

> **TL;DR:** ILOV3Splat is a 3D Gaussian Splatting framework that jointly learns language-aligned and instance-discriminative feature fields using multi-resolution hash encoding, CLIP/DINO/SAM supervision, and a two-stage 3D clustering strategy, enabling open-vocabulary object retrieval and class-agnostic 3D instance segmentation without manual category annotations.

## 核心贡献

- **统一的实例级语义表示**：将每个高斯基元扩展为携带语言特征场（MHE + CLIP 对齐 MLP）和实例特征场（对比学习）的双重属性，实现几何与语义的联合优化，避免两阶段解耦带来的语义漂移。
- **多分辨率哈希编码（MHE）语言特征场**：以高斯的 3D 位置为输入，通过紧凑的 MHE 编码 CLIP 对齐嵌入，在不引入过多内存开销的前提下，实现密集、连贯的语言-3D 对齐。
- **基于 SAM 掩码的实例对比学习**：设计了 intra-instance 紧凑性损失（$\mathcal{L}_{pos}$）和 inter-instance 分离损失（$\mathcal{L}_{neg}$），使实例特征场在无类别监督下具备跨视角区分同类不同实例的能力。
- **两阶段 3D 聚类（HDBSCAN + DBSCAN）**：推理时先在实例嵌入空间用 HDBSCAN 做语义聚类，再在 3D 笛卡尔空间用 DBSCAN 进行空间细化，有效分离同类多实例并消除散点噪声。
- **DINO 边界正则化**：引入 DINO 特征的辅助损失以增强对象边界的空间连贯性，改善局部化精度。

## Key Contributions

- **Unified instance-level semantic representation** embedded directly into 3D Gaussians, combining a language feature field and a contrastive instance feature field in a single joint optimization pipeline.
- **Multi-resolution hash encoding (MHE) for language features**, parameterizing compact CLIP-aligned embeddings from Gaussian 3D positions without excessive memory or runtime overhead.
- **Instance-discriminative contrastive learning over SAM masks**, training Gaussians belonging to the same object to cluster together while being separated from different instances, enabling fine-grained cross-view object distinction.
- **Two-stage 3D clustering strategy**: Stage 1 uses HDBSCAN in instance embedding space for semantic grouping; Stage 2 applies DBSCAN in 3D Cartesian space to isolate spatially coherent subclusters, naturally handling multiple instances of the same class.
- **DINO-based boundary regularization** as an auxiliary loss to sharpen object edge delineation within the learned semantic feature fields.

## Methodology

ILOV3Splat builds upon 3D Gaussian Splatting (3D-GS), which represents scenes as sets of anisotropic Gaussians rendered via differentiable rasterization. Each Gaussian is augmented with two learnable feature vectors beyond its standard geometric/appearance parameters:

**Language Feature Field.** Each Gaussian $G_i$ is assigned a language embedding $f_i^{lang} = MHE_\theta(x_i)$, encoded from its 3D position via a multi-resolution hash grid. A lightweight projection MLP decodes this into a CLIP-aligned embedding $f_i^{CLIP}$ and a scalar $scale_i$ that controls granularity sensitivity during retrieval. Supervision uses a Huber loss against a hybrid ground-truth CLIP map constructed from multi-scale image crops of the training views.

**Instance Feature Field.** Each Gaussian also carries an instance embedding $f_i^{inst}$, supervised by SAM masks applied to training images. For each mask $M_{v,k}$, a representative feature $r_{v,k}$ is computed as the mean of rendered instance features over that mask. A pull loss ($\mathcal{L}_{pos}$) enforces intra-instance compactness; a margin-based push loss ($\mathcal{L}_{neg}$, using ReLU with margin $\gamma$) enforces inter-instance separation. The total instance loss is $\mathcal{L}_{instance} = \mathcal{L}_{pos} + w_{neg} \cdot \mathcal{L}_{neg}$.

**Joint Training Objective.** The full loss is:
$$\mathcal{L}_{total} = \mathcal{L}_{RGB} + \lambda_{lang}\mathcal{L}_{CLIP} + \lambda_{inst}\mathcal{L}_{instance} + \lambda_{reg}\mathcal{L}_{DINO}$$
Geometry and semantics are co-optimized (not decoupled), which mitigates appearance-language misalignment.

**Two-Stage 3D Instance Clustering (inference-time).** After training, HDBSCAN is applied over all Gaussian instance embeddings to form initial semantic clusters. Clusters are filtered by CLIP relevance against the text query (using $scale_i$-weighted comparisons). Each retained cluster is then refined spatially via DBSCAN in 3D Cartesian coordinates, splitting disjoint regions into separate subclusters. This enables precise retrieval of individual object instances.

**Implementation:** Built on the Nerfstudio splatfacto model; trained for 30K iterations on an NVIDIA H100 GPU; $D_{CLIP}=512$, $D_{inst}=8$; Adam optimizer with learning rates 0.0025 (Gaussians) and 0.001 (MLPs).

## Results & Highlights

**3D Object Selection (LERF dataset, 4 scenes):**

| Method | mIoU (mean) | mAcc (mean) |
|---|---|---|
| LangSplat | 9.66 | 12.41 |
| LEGaussians | 16.21 | 23.82 |
| OpenGaussian | 38.36 | 51.43 |
| **ILOV3Splat (Ours)** | **38.45** | **55.15** |

ILOV3Splat achieves the highest mAcc overall and competitive mIoU, consistently ranking first or second across scenes. Qualitatively, it is more robust to occlusion (e.g., "Tesla door handle") and avoids the false activation artifacts seen in OpenGaussian.

**3D Class-Agnostic Instance Segmentation (ScanNet):**

| Method | mIoU | mAcc |
|---|---|---|
| Sam3D (with depth) | 34.45 | 58.24 |
| GaussianGrouping | 22.55 | 30.54 |
| OpenGaussian | 27.32 | 52.44 |
| **ILOV3Splat (Ours)** | **32.08** | **59.29** |

ILOV3Splat achieves the highest mAcc among all methods including Sam3D (which uses depth information), without relying on explicit depth input. Its mIoU is competitive and outperforms all Gaussian-based baselines.

**Ablation Study (LERF dataset):**
- Removing MHE drops mIoU from 38.45 to 22.65, mAcc from 55.15 to 34.49 — MHE is critical for spatially regularized compact representations.
- Removing joint optimization drops mIoU to 36.71 and mAcc to 45.22.
- Each pipeline stage (DINO regularization, semantic clustering Stage 1, spatial refinement Stage 2) contributes incrementally: DINO alone reaches mIoU 10.97/mAcc 18.88; adding Stage 1 gives 28.52/49.44; full pipeline gives 38.45/55.15.

## Critical Assessment

**Novelty:** Moderate-to-good. The core ideas — embedding CLIP features via MHE into 3D-GS, and using contrastive learning with SAM masks for instance discrimination — are incremental combinations of existing components (MHE from Instant-NGP, CLIP distillation from LERF/LangSplat, SAM masks from GaussianGrouping). The main novelty is the unified joint optimization and the two-stage clustering strategy for instance-level separation, which is a genuine contribution over point-level baselines like OpenGaussian. However, the individual components are not novel in isolation.

**Rigor:** Acceptable. Experiments cover two tasks on established benchmarks with multiple baselines and an ablation study. The ablation is well-structured and each component's contribution is clearly demonstrated. However, the mIoU improvement over OpenGaussian on LERF is marginal (38.45 vs 38.36), with the main differentiator being mAcc — this should be discussed more carefully. Results are not validated with confidence intervals or multiple seeds.

**Reproducibility:** The paper provides sufficient implementation details (model dimensions, optimizer settings, loss weights, clustering hyperparameters, hardware) for reproduction. A project page is announced. However, code is not confirmed to be released at time of submission. Dependence on HDBSCAN hyperparameters (min_samples 20–100 "based on scene complexity") introduces a degree of hand-tuning that may affect cross-scene generalization.

**Significance:** The instance-level problem in 3D-GS is genuinely important and underexplored. Achieving depth-free instance segmentation that surpasses a depth-supervised baseline (Sam3D) on mAcc is a notable practical result. The framework is flexible and annotation-free, which increases real-world applicability.

**Red Flags:** The mIoU gap between ILOV3Splat and Sam3D on ScanNet (32.08 vs 34.45) suggests the method still lags on the more granular overlap metric. The paper acknowledges limitations in sparse scenes with large objects where multi-view SAM masks may be inconsistent. The $D_{inst}=8$ dimensionality for instance features is very low — it is unclear whether this is principled or a tuning artifact. The LERF mIoU improvement over OpenGaussian is essentially negligible, suggesting the instance-level feature may not consistently improve object boundary localization.

## Strengths

- Joint geometry-semantics optimization avoids the semantic drift common in two-stage decoupled pipelines.
- The two-stage clustering (semantic + spatial) cleanly addresses both cross-view alignment and intra-class multi-instance separation without requiring scene-specific preprocessing or fixed instance count priors.
- Depth-free approach outperforms depth-supervised Sam3D on mAcc, demonstrating strong 3D reasoning from RGB alone.
- Comprehensive ablation study validates each component's contribution quantitatively and qualitatively.
- No category supervision or manual annotations required at training or inference.

## Limitations

- The mIoU gain over the strongest Gaussian baseline (OpenGaussian) on the LERF benchmark is marginal, suggesting the core language retrieval quality is not dramatically improved.
- ScanNet point clouds are sparse (~100K points per scene), limiting geometric detail; performance in dense, high-resolution scenes is less validated.
- Inconsistent multi-view SAM masks in sparse-view or large-object scenarios can degrade instance representation quality, as acknowledged by the authors.
- The very low instance feature dimension ($D_{inst}=8$) may limit discriminability in scenes with many similar objects.
- Clustering hyperparameters (HDBSCAN min_samples) require scene-complexity-based tuning, reducing plug-and-play usability.
- Training on an H100 GPU for 30K iterations implies substantial compute cost that may limit accessibility.

## Relevance

该论文聚焦于 3D 场景的开放词汇实例级理解，与大语言模型可解释性和可靠性的直接关联有限，但其通过对比学习实现跨视角一致性语义特征的思路，与 LLM 表示对齐和训练稳定性中的类似挑战（如特征漂移、表示坍塌）具有方法论层面的参考价值；多分辨率哈希编码在 3D 特征学习中的稳定化作用也与 LLM 训练中的结构化表示设计有一定类比意义。

## Questions for Further Exploration

1. How does ILOV3Splat perform on dynamic scenes or scenes with repeated/symmetric objects, where instance feature contrastive learning may face ambiguity from multi-view mask inconsistency?
2. Can the instance feature dimension ($D_{inst}=8$) be increased without prohibitive memory cost, and does doing so improve fine-grained instance discrimination in scenes with many similar objects (e.g., 20+ chairs)?
3. The $scale_i$ scalar is learned to adaptively control retrieval granularity — how does this scale value distribute in practice across objects of different sizes, and does it reliably capture hierarchical object-part relationships?
4. Since joint optimization is shown critical, how sensitive is performance to the relative loss weights ($\lambda_{lang}$, $\lambda_{inst}$, $\lambda_{reg}$)? Is there a principled way to set them without scene-specific tuning?
5. Would replacing HDBSCAN with a learned or supervised clustering method further improve instance segmentation mIoU, particularly in the challenging sparse ScanNet settings?
