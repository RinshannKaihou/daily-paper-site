Process-Reward Tactic Evolution for Long-Horizon Bioinformatics Workflows

TL;DR

Proposes Process-Reward Tactic Evolution (PRTE), a Galaxy-based training framework that converts verified bioinformatics workflow rollouts into a library of reusable, verifier-backed "tactics." On BioWorkflow Bench (35 held-out tasks) and BioAgent Bench (10 tasks), the PRTE Agent achieves the best overall final score (0.5013), especially on long and extra-long workflows, outperforming both no-training baselines and reflection/memory/prompt-evolution methods while using substantially fewer tokens than no-training multi-agent approaches.

提出过程奖励策略演化（PRTE），一种基于Galaxy的训练框架，将经过验证的生物信息学工作流执行轨迹转化为可复用、有验证器支撑的"策略"库。在BioWorkflow Bench（35个留出任务）和BioAgent Bench（10个任务）上，PRTE Agent取得了最优总分（0.5013），尤其在长程和超长工作流上表现突出，超越了无训练基线和反思/记忆/提示演化方法，同时Token消耗远低于无训练多智能体方案。

Abstract / 摘要

LLM agents can write code and call tools, but reliable bioinformatics work requires long-horizon interaction with workflow software, typed data objects, provenance, and biological checks. The authors study this setting through Galaxy workflow execution, where the agent must explore task data, construct or adapt an executable workflow DAG, bind inputs and dataset collections, monitor execution, debug failures, and validate biological outputs. They propose Process-Reward Tactic Evolution, a Galaxy-based training framework that turns verified workflow rollouts into reusable BioWorkflow Tactics. During training, agents practice on curriculum-organized Galaxy tasks in BioAgent Gym; process verifiers score workflow construction, software interaction, execution, and biological correctness; successful and failed traces are distilled into a tactic library. At inference, the trained executor (PRTE Agent) uses this library to execute held-out BioWorkflow Bench and BioAgent Bench tasks in isolated environments.

LLM智能体能编写代码和调用工具，但可靠的生物信息学工作需要与工作流软件进行长程交互，涉及类型化数据对象、溯源和生物学检查。作者通过Galaxy工作流执行来研究这一场景——智能体需要探索任务数据、构建或调整可执行的工作流DAG、绑定输入和数据集合、监控执行、调试失败并验证生物学输出。他们提出了过程奖励策略演化（PRTE），一种基于Galaxy的训练框架，将经过验证的工作流执行轨迹转化为可复用的BioWorkflow Tactics。训练时，智能体在BioAgent Gym中对课程式组织的Galaxy任务进行练习；过程验证器对工作流构建、软件交互、执行和生物学正确性进行评分；成功和失败的轨迹被提炼进策略库。推理时，训练好的执行器（PRTE Agent）使用该策略库在隔离环境中执行留出的BioWorkflow Bench和BioAgent Bench任务。

Motivation / 研究动机

Bioinformatics workflow construction is a demanding instance of long-horizon scientific agent tasks. Unlike short-answer biomedical QA, a Galaxy task is only solved when the agent builds, imports, runs, repairs, and validates a workflow through a real software backend. Correctness is distributed across documentation lookup, input staging, datatype binding, workflow import, invocation, job completion, artifact collection, and biological validation. Existing reflection, memory, skill, and prompt-evolution methods show agents can improve by accumulating external process state, but they rarely train on realistic scientific workflow execution where intermediate software states are observable and consequential. The key insight is that useful learning must be procedural: a later agent should recover tactics for schema inspection, collection wiring, graph repair, output accounting, and domain-specific workflow families -- not only remember that a past task succeeded.

生物信息学工作流构建是长程科学智能体任务中要求最高的实例之一。与简答式生物医学QA不同，Galaxy任务只有在智能体通过真实软件后端完成构建、导入、运行、修复和验证工作流后才算解决。正确性分布在文档查找、输入暂存、数据类型绑定、工作流导入、调用、作业完成、产物收集和生物学验证等多个环节。现有的反思、记忆、技能和提示演化方法表明智能体可以通过积累外部过程状态来改进，但它们很少在中间软件状态可观察且有实际影响的真实科学工作流执行上进行训练。核心洞见是：有用的学习必须是程序性的——后续智能体应能恢复用于模式检查、集合连线、图修复、输出核算和特定领域工作流族的策略，而不仅仅记住某个过去的任务成功了。

Method / 方法

Problem Formulation

The task is formulated as an agent-environment problem. The input is $x = (u, \mathcal{I}, D_{\text{pub}}, b)$ where $u$ is the biological objective, $\mathcal{I}$ is the input manifest, $D_{\text{pub}}$ is public documentation, and $b$ contains backend constraints. The output is $y = (W, \hat{D}, A, \Pi, \tau, \mathbf{r})$ including an executable Galaxy workflow $W$ (a typed DAG), produced datasets $\hat{D}$, biological assertions $A$, provenance $\Pi$, execution trace $\tau$, and process scores $\mathbf{r}$.

Tactic State

PRTE represents learned experience as a tactic library rather than a single reflection string. The tactic state $\Omega = (\mathcal{L}, G^{\Omega}, \rho)$ comprises a set of tactic files $\mathcal{L}$, a dependency graph $G^{\Omega}$, and a retrieval function $\rho$. Each tactic states when it applies, what operation to perform, what postcondition should hold, which failure signature it repairs, and what public evidence supports it. Typical tactics cover schema inspection, paired-collection wiring, gxformat2 checking, Galaxy invocation repair, artifact accounting, and task-family biological checks.

Process-Supervised Updates

Training proceeds over curriculum batches $S_k$. Each batch uses one tactic snapshot, runs isolated Galaxy rollouts, and updates the tactic state. The curriculum starts with small workflows and documentation lookup, then adds longer dependency chains, live invocation, queue behavior, artifact recovery, and biological validation. Each trajectory is scored across 9 process-reward channels: documentation use ($r_{\text{doc}}$), local validation ($r_{\text{chk}}$), import ($r_{\text{imp}}$), invocation ($r_{\text{inv}}$), job completion ($r_{\text{job}}$), artifact collection ($r_{\text{art}}$), biological correctness ($r_{\text{bio}}$), repair progress ($r_{\text{rep}}$), lifecycle discipline ($r_{\text{life}}$), and leakage safety ($r_{\text{safe}}$). The update function $U$ has three stages: (1) a reviewer extracts positive and negative evidence from each trajectory, (2) a semantic-gradient updater compares successful and failed excerpts to propose tactic patches, and (3) a gate accepts a patch only if it has applicability, executable procedure, verification check, failure signature, and public evidence. Hidden gold workflows and private assertions are never written into $\Omega$.

Tactic-Aware Inference

At held-out inference, the tactic state is frozen. A worker retrieves a budgeted tactic bundle via $B_x = \text{Hydrate}(\text{Closure}(\rho(x, \mathcal{L}), G^{\Omega}), B_{\max})$, then executes the task in an isolated notebook and Galaxy account -- staging data, writing workflow artifacts, running local checks, importing and invoking the graph, polling jobs, repairing allowed failures, collecting outputs, and submitting evidence.

问题形式化

任务被建模为智能体-环境问题。输入为 $x = (u, \mathcal{I}, D_{\text{pub}}, b)$，其中 $u$ 是生物学目标，$\mathcal{I}$ 是输入清单，$D_{\text{pub}}$ 是公开文档，$b$ 包含后端约束。输出为 $y = (W, \hat{D}, A, \Pi, \tau, \mathbf{r})$，包括可执行的Galaxy工作流 $W$（类型化DAG）、生成的数据集 $\hat{D}$、生物学断言 $A$、溯源 $\Pi$、执行轨迹 $\tau$ 和过程分数 $\mathbf{r}$。

策略状态

PRTE将学习到的经验表示为策略库而非单一反思字符串。策略状态 $\Omega = (\mathcal{L}, G^{\Omega}, \rho)$ 由策略文件集合 $\mathcal{L}$、依赖图 $G^{\Omega}$ 和检索函数 $\rho$ 组成。每个策略说明其适用条件、要执行的操作、应保持的后置条件、它修复的失败签名以及支持它的公开证据。

过程监督更新

训练按课程批次 $S_k$ 进行。课程从小工作流和文档查找开始，逐步增加更长的依赖链、实时调用、队列行为、产物恢复和生物学验证。每条轨迹在9个过程奖励通道上评分。更新函数 $U$ 包含三个阶段：审阅者提取正负证据、语义梯度更新器比较成功与失败片段以提出策略补丁、门控机制仅接受具有适用性、可执行程序、验证检查、失败签名和公开证据的补丁。

策略感知推理

在留出推理阶段，策略状态被冻结。工作者检索有预算限制的策略包，然后在隔离的notebook和Galaxy账户中执行任务。

Experiment / 实验

Benchmarks

  • BioWorkflow Bench: A new Galaxy benchmark with 26 training tasks and 35 held-out test tasks (test35), balanced across small (5), medium (10), long (10), and xlong (10) complexity groups. Tasks are derived from public IWC and GTN workflow resources with total 455 tools and average depth 11.34.
  • BioAgent Bench: An external 10-task bioinformatics-agent benchmark for cross-benchmark validation.

Baselines (8 agents)

  • No-training group: ReAct, MultiAgent, Agent Harness/Biomni
  • Training group: Reflexion, Agent Workflow Memory, Agentic Memory, GEPA (prompt evolution), PRTE Agent

Key Results (BioWorkflow Bench test35)

| Agent | Overall Final | Long Final | XLong Final |

|-------|--------------|------------|-------------|

| PRTE Agent | 0.5013 | 0.3182 | 0.3922 |

| GEPA | 0.4953 | 0.3059 | 0.3090 |

| Agent Workflow Memory | 0.4586 | 0.2799 | 0.3071 |

| Agent Harness/Biomni | 0.3926 | 0.2592 | 0.2779 |

| ReAct | 0.3663 | 0.2592 | 0.1934 |

GEPA obtains the highest small and medium scores (where a strong global instruction covers many tasks), but the pattern reverses as workflow horizon grows: PRTE Agent achieves the best long, xlong, and overall final scores. The xlong gain comes mainly from biological correctness (0.4502 vs. GEPA's 0.3808), supporting the intended role of tactics in preserving platform procedures while leaving model context for biological reasoning.

BioAgent Bench

PRTE Agent reaches 10/10 final completion, 8/10 result match, and the highest average step-level score (0.9667), with combined score 0.9222. This confirms gains are not artifacts of BioWorkflow Bench construction.

Ablation Study

  • Outcome-only training (no process reward): improves after the first batch but plateaus, final score ending at 0.4770
  • GEPA: strong but flat after its selected seed instruction
  • Full process-reward: improves more steadily, finishing at 0.5013
  • The process-reward library grows to 26 tactic files and ~33.7K tokens (vs. 8 files for no-process, 1 file for GEPA), containing not just reflections but repair procedures and task-family contracts for RNA-seq, Hi-C, pseudobulk analysis, MAG generation, HyPhy comparison, assembly cleanup, and chromatin peak QC.

Resource Efficiency

PRTE Agent uses 963.454M total tokens on test35 (avg 27.527M per task), compared to MultiAgent's 8.514B total tokens. Training is a fixed upfront cost that amortizes over repeated long-horizon workflows in the same software environment. Cumulative token consumption curves show PRTE crossing below no-training baselines after approximately 15 evaluated tasks.

基准测试

  • BioWorkflow Bench：新的Galaxy基准，26个训练任务和35个留出测试任务（test35），在小/中/长/超长四个复杂度组间平衡分布。
  • BioAgent Bench：外部10任务生物信息学智能体基准，用于交叉验证。

主要结果

PRTE Agent在总分（0.5013）、长工作流和超长工作流上取得最优成绩。GEPA在小型和中型任务上最强，但随着工作流长度增加优势消失。超长任务的增益主要来自生物学正确性（0.4502 vs. GEPA的0.3808）。

消融研究

仅结果训练在首批后即停止改进（终止于0.4770）；完整过程奖励持续改进至0.5013。过程奖励改变的不仅是记忆大小，更是记忆的种类——从简单反思变为修复程序和任务族合约。

资源效率

PRTE Agent在test35上共消耗963M tokens（每任务平均27.5M），远低于MultiAgent的8.5B tokens。训练成本在约15个任务后即可摊销。

Critical Assessment / 批判性评估

Strengths:

  • The problem formulation is well-grounded: bioinformatics workflow construction genuinely requires the kind of long-horizon, multi-step, verifiable interaction that tests the limits of current LLM agents. The Galaxy platform provides a realistic, reproducible evaluation environment.
  • The 9-channel process-reward decomposition is principled and enables fine-grained credit assignment rather than binary success/failure. The gating mechanism for tactic acceptance is a thoughtful safeguard against error propagation.
  • The experimental comparison against 7 baselines is thorough, and the ablation cleanly isolates the contribution of process rewards vs. outcome-only training.
  • The gains are concentrated where they matter most (long and xlong workflows), which aligns with the theoretical motivation.

Weaknesses:

  • The experiments are entirely centered on Galaxy. While this provides consistency, it is unclear how the tactic formulation transfers to other workflow platforms (CWL, Nextflow, Snakemake) or non-bioinformatics scientific workflows. The authors acknowledge this limitation.
  • The overall performance gap over GEPA is modest (0.5013 vs. 0.4953 overall). The advantage is more pronounced in long/xlong regimes, but one might question whether the additional complexity of the full PRTE framework is justified by a ~1% overall improvement, even if the per-regime gains are larger.
  • Galaxy server dependence on public-server availability, queue behavior, installed tool versions, and dataset accessibility introduces non-trivial noise that could affect reproducibility. Some workflows may be too backend-specific or long-running for stable shared-infrastructure evaluation.
  • Biological assertions are useful but partial -- passing them does not replace expert review of downstream scientific interpretation. The tactic updater must also remain conservative since an incorrect reusable repair can harm future tasks, but the paper does not extensively analyze failure modes of tactic propagation.
  • The tactic library content (26 files, 33.7K tokens) is described qualitatively but not analyzed for redundancy, conflicts, or degradation over longer training horizons.

优点：

  • 问题形式化扎实，Galaxy平台提供了真实可复现的评估环境
  • 9通道过程奖励分解原则性强，门控机制防止错误传播
  • 与7个基线的全面比较和消融实验清晰地隔离了过程奖励的贡献
  • 增益集中在最重要的长程和超长工作流上

不足：

  • 实验完全集中在Galaxy上，向其他工作流平台的迁移性不明
  • 与GEPA的整体差距较小（0.5013 vs. 0.4953），主要优势在长/超长任务上
  • 对公共Galaxy服务器的依赖引入了不可忽视的噪声
  • 生物学断言是有用但不完整的评估代理
  • 策略库的冗余性、冲突和长期训练退化未被充分分析

Relevance / 相关性

This paper is relevant to several active research areas: (1) LLM agents for scientific discovery and workflow automation, particularly in computational biology; (2) process reward models and their application beyond mathematical reasoning to software-grounded tasks; (3) memory, skill, and tactic libraries for long-horizon agent learning; (4) benchmarking scientific agents on executable, multi-step tasks rather than short-answer QA. The tactic-library formulation and process-supervised update mechanism may generalize to other domains where agents repeatedly interact with complex software environments (e.g., computational chemistry, materials science, climate modeling). The BioWorkflow Bench itself is a useful contribution for the bioinformatics-agent community.
