LLM Quantization

GRINQH: Graded Input-based Quantization Hierarchy for Efficient LLM Generation

A weight-only post-training quantization framework that dynamically assigns 0–8 bits per channel at decode time based on activation magnitudes, unifying quantization and sparsification to establish new Pareto frontiers on quality vs. speed.

Oberlaender*, Finkbeiner*, Schoefmann, Neftci — RWTH Aachen & Forschungszentrum Jülich — 2026-06-22

Key Finding
Runtime-Adaptive Bit Allocation

Assigns 0–8 bits per weight channel each decode step using activation magnitude as a proxy for importance, achieving near-lossless quality at ~4 effective bits.

Key Metric
73.19%

Avg accuracy at 3 effective bits on Llama3.1-8B, vs. GPTQ 63.27%, AWQ 56.05%, RTN 50.64%. Beats all 4-bit baselines on Qwen3-8B at only 3 bits.

Why This Matters
Continuous Pareto Frontier

Enables fine-grained latency-quality trade-offs at deploy time without recompression or model swapping. Integrated into vLLM.

Critical Edge
Batch=1 Only

Advantage vanishes at larger batch sizes. DRAM footprint equals b_max regardless of effective bits. 20% prefill slowdown from bit-planar unpacking.

How It Works

Dynamic Precision Assignment

Each decode step, activation magnitude |x_i| is compared against pre-computed thresholds to assign bit width b_i per channel. High-magnitude outliers keep full precision; negligible channels get 0 bits (structured sparsity).

Bit-Planar Storage

Weights decomposed into b_max/2 two-bit planes stacked along input channels. The kernel fetches only the most-significant segments needed, directly cutting DRAM bandwidth.

Bias Correction

A midpoint correction term φ prevents truncation bias from cascading through layers. Without it: 0% GSM8K, perplexity >108.

Kernel Performance

Effective Bit Width → Throughput (tok/s) →
  • Custom Triton kernel: 0.98x MARLIN at 4-bit effective, 1.28x faster at 2-bit
  • ~200 tok/s at 2-bit effective vs. MARLIN 4-bit ~150 tok/s on Llama3.1-8B
  • Runtime scales linearly with effective bit width
  • Tested on RTX 4090, RTX 5070, RTX PRO 6000

Key Innovations

  • Unified quantization + sparsification — 0-bit channels are structured sparsity, not a separate mechanism
  • Decoupled prefill/decoding — full precision for compute-bound prefill, dynamic precision for bandwidth-bound decode
  • Quantization-agnostic — improves RTN, GPTQ, and any-precision LLM as base quantizers
  • vLLM integration — practical deployment path, not just paper results
  • Continuous bit-width scaling — no discrete 2/3/4/8 jumps; any effective precision between 0 and b_max

Strengths

  • Complete system from algorithm to kernel — bridges theory-hardware gap
  • Composable with existing quantization toolchains
  • Bias-correction term is subtle but critical (catastrophic failure without it)
  • Strong experimental coverage: 2 model families, 7 benchmarks, 3 GPUs
  • Dense Pareto frontier enables runtime quality-speed knobs

Limitations & Red Flags

  • Batch-size-one only; advantage disappears in high-throughput multi-batch serving
  • DRAM footprint = b_max bits per weight regardless of effective precision (bandwidth savings, not memory savings)
  • 20% prefill slowdown from bit-planar unpacking
  • No explicit code release promised
  • Evaluation limited to ≤8B models; 70B+ behavior unknown
  • Fixed calibration thresholds cannot adapt to distribution shifts mid-generation
  • No comparison with activation quantization methods (SmoothQuant, QLLM)

Open Questions

  • How does GRINQH scale to 70B+ models? Do activation outlier distributions shift enough to break threshold calibration?
  • Can bit-planar storage overhead be amortized in multi-batch scenarios, or is the method fundamentally batch-1?
  • How does GRINQH interact with speculative decoding, which changes decode-stage batch dynamics?
  • Could fixed thresholds be replaced with lightweight online adaptation (e.g., EMA of activation statistics)?
  • Impact on very long generation (10K+ tokens) where activation distribution drift may compound?
