SVD-Surgeon: Optimal Singular-Value Surgery for Large Language Model Compression

Mahmoud Safari · Frank Hutter Training-Free · Post-Hoc Correction GitHub

Applies the Optimal Brain Surgeon framework in singular-value space to derive a closed-form correction that optimally compensates retained singular values after truncation — consistently improving any SVD-based LLM compressor, training-free.

Key Metric
20×
Perplexity reduction at 70% compression
OPT-6.7B: 944.57 → 46.36 (WikiText-2)
Why This Matters

Any SVD compressor can bolt on this correction layer for large accuracy recovery under aggressive compression — no retraining, no specialized kernels, one closed-form solve per layer.

Critical Edge

Only perplexity evaluated (no downstream tasks); tested atop one host compressor (SVD-LLM); Fisher estimation costs ~7.6 s/sample for 7B models; U and V directions remain frozen.

Method Pipeline

Reduce to singular-value space

Restrict perturbations to the diagonal of Σ, collapsing the Hessian from mn×mn to l×l.

Fisher-approximate projected Hessian

Project per-sample gradients via diag(UTGV); assemble &Hbar; as outer-product average.

Closed-form compensation

Optimal update: δσS = H̅SS-1 H̅SC σC — shifts surviving values to absorb pruning loss.

Saliency-based selection

Loss-aware score σi² / (2[H̅-1]ii) replaces magnitude-based pruning; recovers magnitude as special case.

Motivation

  • SVD low-rank compression is hardware-friendly (no specialized kernels), but existing methods only improve how the decomposition is formed — none repairs it after truncation.
  • Naive truncation ignores cross-component loss interactions.
  • OBS has been applied to weights (GPTQ, SparseGPT) and neurons, but never in singular-value coordinates — a natural basis for low-rank compression.

Strengths & Limitations

  • + Elegant: OBS in SV basis reduces 4th-order Hessian to compact l×l matrix.
  • + Composable: layers on any SVD compressor (no orthonormality required).
  • + Training-free, single-shot, parallelizable.
  • − No downstream task evaluation (zero-shot benchmarks missing).
  • − Only tested atop SVD-LLM; composability claim empirically narrow.
  • − Several hyperparameters (λ, dS, d, α, N) need per-model tuning.
  • − Directions U, V frozen — potential accuracy left on the table.

Two Variants

  • (U) Update-only: Inherits pruning set from host compressor; applies only the closed-form compensation. Minimal overhead (one linear solve).
  • (S) Select & Update: Re-selects which singular values to prune using saliency scores, then compensates. More expensive (pseudoinverse of full H̅), slightly better results.

Practical Notes

  • Fisher estimation: ~7.6 s per calibration sample for LLaMA-2-7B on H200 GPU; 16k–32k samples recommended.
  • One-time cost, parallelizable across layers and samples.
  • No gradient-based optimization at any step.
  • Works with SVD-LLM's whitened (non-orthonormal) factors out of the box.
