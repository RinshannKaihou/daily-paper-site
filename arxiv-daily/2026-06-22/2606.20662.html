Confidence Laundering in Agent Systems: Why Uncertainty Needs a Latent Carrier

TL;DR

This position paper identifies "confidence laundering" -- the failure mode where fragile upstream decisions in multi-agent systems are repackaged as clean, procedurally valid artifacts, causing downstream agents to over-trust them. The authors propose "latent uncertainty" as a representation-level carrier that preserves upstream epistemic fragility across agent interfaces, shifting agent uncertainty from step-wise estimation to uncertainty-preserving interface design.

本文是一篇立场论文，识别了多智能体系统中的"置信度洗白"（confidence laundering）失效模式——上游脆弱决策被重新包装为看似合法的制品，导致下游智能体过度信任。作者提出"潜在不确定性"（latent uncertainty）作为表示层面的载体，在智能体接口间保留上游认知脆弱性，将智能体不确定性从逐步估计转向保留不确定性的接口设计。

Problem / 问题

In multi-step agent systems, upstream components make intermediate decisions under uncertainty and pass resulting artifacts (tool calls, queries, retrieved passages, structured outputs) to downstream components. Standard agent interfaces perform a many-to-one projection: they transmit only the committed decision, not the epistemic state behind it. The authors call this "interface collapse" -- different upstream uncertainty profiles can produce identical artifacts, making a near-tie among alternatives indistinguishable from a confident decision.

在多步智能体系统中，上游组件在不确定性下做出中间决策，并将结果制品（工具调用、查询、检索段落、结构化输出）传递给下游组件。标准智能体接口执行多对一投射：仅传输已承诺的决策，而非其背后的认知状态。作者将此称为"接口坍缩"（interface collapse）——不同的上游不确定性状态可以产生相同的制品，使得备选方案间的微弱差异与自信决策变得无法区分。

This creates confidence laundering: a fragile epistemic state is converted into a well-formed symbolic or executable object, causing downstream components to treat it as more reliable than warranted. A complete API call appears executable, a fluent intermediate answer appears reasoned, a structured JSON field appears validated -- procedural validity is mistaken for epistemic reliability.

这产生了置信度洗白：脆弱的认知状态被转换为格式良好的符号或可执行对象，导致下游组件将其视为比实际更可靠。完整的API调用看似可执行，流畅的中间答案看似经过推理，结构化的JSON字段看似经过验证——程序有效性被误认为认知可靠性。

Key Concepts / 核心概念

Uncertain Decision Handoff / 不确定决策交接

The process by which an upstream component commits to an intermediate decision under uncertainty and transfers the resulting artifact to a downstream component for continued reasoning or action. Unlike mere communication, handoff transfers operational responsibility -- the downstream component may execute, verify, or use the artifact as a premise.

上游组件在不确定性下承诺中间决策，并将结果制品转交给下游组件以继续推理或执行的过程。与单纯通信不同，交接转移了操作责任——下游组件可能执行、验证或将该制品作为前提使用。

Interface Bottleneck / 接口瓶颈

Once an uncertainty-bearing state is compressed into a committed artifact, uncertainty may still shape the trajectory causally, but it no longer travels as an accessible state for downstream recovery. Standard interfaces (tool calls, queries, structured outputs) are optimized for executability, which forces internal states into single consumable artifacts.

一旦承载不确定性的状态被压缩为已承诺的制品，不确定性可能仍在因果上影响轨迹，但不再作为可访问的状态供下游恢复使用。标准接口（工具调用、查询、结构化输出）为可执行性优化，迫使内部状态投射为单一可消费的制品。

Latent Uncertainty / 潜在不确定性

Uncertainty-bearing information encoded in an agent's internal continuous representations (hidden states, activations, KV traces) that is not fully preserved by committed text or action interfaces but can be attached to a handoff in a downstream-usable form. It is narrower than "all information in a hidden state" -- only the subset that helps downstream components make uncertainty-sensitive decisions beyond what is recoverable from the committed artifact alone.

编码在智能体内部连续表示（隐藏状态、激活值、KV痕迹）中的承载不确定性的信息，这些信息未被已承诺的文本或动作接口完全保留，但可以以下游可用的形式附加到交接过程中。它比"隐藏状态中的所有信息"更窄——仅指帮助下游组件做出不确定性敏感决策的子集，超出仅从已承诺制品可恢复的范围。

Method / 方法

Taxonomy of Agent Uncertainty Estimation

The paper provides a structured taxonomy distinguishing:

  • Atomic (Single-Step): Logit-based metrics (entropy, MaxProb), distributional analysis (KL-Divergence, Mahalanobis distance), self-verbalized confidence, consistency-based methods (semantic invariance, ensemble variance)
  • Holistic (Trajectory): Aggregation (step-wise pooling), propagation modeling, consistency-based path diversification

论文提供了结构化分类法，区分了：

  • 原子级（单步）：基于logit的指标（熵、MaxProb）、分布分析（KL散度、马氏距离）、自我语言化置信度、基于一致性的方法（语义不变性、集成方差）
  • 整体级（轨迹）：聚合（逐步池化）、传播建模、基于一致性的路径多样化

Carrier Design Options

Three levels of uncertainty carriers with increasing expressiveness:

  1. Scalar scores -- probe hidden states into calibrated risk scores (strong compression, lossy)
  2. Soft-prompt vectors -- encode upstream uncertainty into vectors appended to downstream input (medium compression)
  3. KV/activation handoff -- pass upstream uncertainty-bearing hidden state directly to downstream component (minimal compression, richest signal)

三种表达力递增的不确定性载体：

  1. 标量分数 -- 将隐藏状态探测为校准的风险分数（强压缩，有损）
  2. 软提示向量 -- 将上游不确定性编码为附加到下游输入的向量（中等压缩）
  3. KV/激活值交接 -- 将上游承载不确定性的隐藏状态直接传递给下游组件（最小压缩，最丰富信号）

Empirical Validation

Experiments on HotpotQA converted to a web-search tool-use setting using Qwen-1.5B as the upstream model:

  • Finding 1: External difficulty labels (easy/medium/hard) do not cleanly predict model uncertainty -- semantic cluster distributions overlap substantially across difficulty levels
  • Finding 2: Hidden-state probes (linear and MLP) recover semantic-entropy-based uncertainty labels with strong AUROC, improving across transformer depth (blocks 7, 14, 28)
  • Finding 3: Latent uncertainty outperforms answer-only, scalar-score, and textual-uncertainty interfaces in both AUROC and accuracy for downstream uncertainty recovery

在将HotpotQA转换为网络搜索工具使用场景的实验中，使用Qwen-1.5B作为上游模型：

  • 发现1：外部难度标签（简单/中等/困难）不能清晰预测模型不确定性——语义聚类分布在难度级别间大量重叠
  • 发现2：隐藏状态探针（线性和MLP）以强AUROC恢复基于语义熵的不确定性标签，性能随transformer深度（第7、14、28层）提升
  • 发现3：潜在不确定性在AUROC和准确率上均优于仅答案、标量分数和文本不确定性接口

Cross-Model Overconfidence Study

Across six LLMs (Qwen2.5-7B Turbo, Llama-3-8B Lite, GPT-4o mini, Claude-Opus 4.6, Gemini-3.1 Pro, GPT-5.2), providing uncertainty signals consistently reduces downstream overconfidence. The reduction is most dramatic for larger models: GPT-5.2 drops from 0.270 to 0.044 overconfidence, Claude-Opus 4.6 from 0.433 to 0.171.

在六个LLM（Qwen2.5-7B Turbo、Llama-3-8B Lite、GPT-4o mini、Claude-Opus 4.6、Gemini-3.1 Pro、GPT-5.2）上，提供不确定性信号一致性地降低了下游过度自信。大模型的降低最为显著：GPT-5.2从0.270降至0.044，Claude-Opus 4.6从0.433降至0.171。

Broader Implications / 更广泛影响

The paper argues for three design shifts:

  1. Uncertainty as interface contract: Uncertainty should be part of the contract between components, not optional metadata
  2. Evaluation shift: Agent benchmarks should measure handoff reliability (recovery recall, false accept rate, confidence-laundering reduction), not just final accuracy
  3. Linked recovery actions: Uncertainty interfaces should communicate both degree and kind of risk, enabling appropriate recovery (clarification, retrieval, rerouting, verification)

论文主张三个设计转变：

  1. 不确定性作为接口契约：不确定性应成为组件间契约的一部分，而非可选元数据
  2. 评估转变：智能体基准测试应衡量交接可靠性（恢复召回率、误接受率、置信度洗白减少），而非仅衡量最终准确率
  3. 关联恢复动作：不确定性接口应同时传达风险的程度和种类，以启用适当的恢复动作（澄清、检索、重路由、验证）

Critical Assessment / 批判性评估

Strengths:

  • The "confidence laundering" framing is genuinely insightful and names a real, under-discussed failure mode in agentic systems. The metaphor is precise: procedural validity masquerading as epistemic reliability.
  • The taxonomy of uncertainty estimation methods (Table 1) and the distinction between propagation vs. re-estimation are clear and useful.
  • Cross-model overconfidence results (Figure 4) across six diverse LLMs lend credibility to the generality of the problem.

Weaknesses:

  • The empirical validation is limited: only Qwen-1.5B on HotpotQA (a relatively small model on a well-studied benchmark). The position's claims are broader than what the experiments support.
  • Latent uncertainty requires access to model internals (hidden states), which severely limits applicability to black-box or heterogeneous agent systems -- the paper acknowledges this but does not resolve it.
  • The paper does not demonstrate end-to-end improvement in task performance from using latent uncertainty carriers. The experiments show better uncertainty detection, but not that better detection leads to better recovery or final outcomes.
  • The comparison between uncertainty carriers (Figure 5, Figure 6c) would benefit from controlling for information capacity -- latent states carry far more bits than scalar scores, so the comparison may partly reflect capacity rather than the conceptual advantage.
  • As a position paper, it proposes the framework but leaves implementation details (how to actually attach latent uncertainty to standard agent interfaces in practice) largely open.

优势：

  • "置信度洗白"的框架命名了智能体系统中真实存在但讨论不足的失效模式，隐喻精确。
  • 不确定性估计方法的分类法（表1）以及传播与重估计的区分清晰有用。
  • 跨六个多样化LLM的过度自信结果（图4）增强了问题普遍性的可信度。

不足：

  • 实证验证有限：仅在HotpotQA上使用Qwen-1.5B（较小模型，成熟基准），立场主张比实验支撑更广。
  • 潜在不确定性需要访问模型内部状态，严重限制了在黑盒或异构智能体系统中的适用性。
  • 论文未展示使用潜在不确定性载体带来的端到端任务性能提升，仅展示了更好的不确定性检测。
  • 不确定性载体的比较可能部分反映信息容量差异而非概念优势。

Relevance / 相关性

This paper is highly relevant to anyone building or evaluating multi-agent LLM systems. The confidence laundering concept directly applies to common agent architectures (ReAct, tool-use pipelines, retrieval-augmented generation) where intermediate decisions are passed as clean artifacts. The work connects to broader questions about agent reliability, error propagation, and the design of recoverable systems.
