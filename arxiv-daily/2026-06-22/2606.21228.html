arxiv 2606.21228 · 2026-06-22
LLM Orchestration · Multi-Agent Systems

Sakana Fugu Technical Report

Yujin Tang et al. Sakana AI, Tokyo 14 authors

Fugu-Ultra orchestrates multiple frontier LLMs as a unified system, achieving SOTA across 7 major benchmarks—surpassing Claude Opus 4.8, GPT-5.5, and Gemini 3.1-Pro individually—by learning behavioral-level model composition without parameter access.

Key Finding
Learned orchestration of frontier LLMs outperforms every individual model, establishing multi-model composition as a viable scaling axis.
Key Metric
73.7%
SWE-Bench Pro — described as "equivalent to entire generational improvements" over best single model (69.2%)
Why This Matters
Frontier performance may no longer require training a frontier-scale model—orchestrating existing models intelligently is an alternative scaling axis.
Critical Edge
Structurally asymmetric comparison: an ensemble of 3+ frontier models vs. each alone. No cost or latency analysis reported. Reproducibility is limited.

Two Variants

Fugu Low Latency

Lightweight selection head atop orchestrator backbone. Trained via SFT on verifiable tasks, then refined with sep-CMA-ES evolutionary search over multi-turn agent trajectories.

Fugu-Ultra Max Quality

Conductor framework generates free-form multi-step workflows via GRPO reinforcement learning. Supports chains, trees, parallel branches, and best-of-N topologies.

Key Innovation
  • Behavioral-level composition: treats LLMs as black-box agents, no parameter access needed
  • Intra-workflow agent isolation prevents "orchestration collapse" in multi-agent tool calling
  • Inter-workflow persistent shared memory retains context across successive workflows
  • Domain-adaptive routing: GPT-5.5 dominates Terminal Bench, Gemini dominates GPQA-Diamond
End-to-End Demonstrations
AutoResearch Best mean validation BPB (0.9774) across 123 autonomous experiments, lowest variance among all models.
Classical Japanese Kana Mean NED 0.776, far ahead of best frontier baseline (0.642) on reading-order recovery.
CAD Generation Only model to produce a mechanically correct iris mechanism across all tested systems.
Strengths
  • Comprehensive benchmark coverage: coding, reasoning, and scientific tasks
  • Elegant two-stage training: SFT initialization then evolutionary/RL optimization
  • Intra-workflow isolation + shared memory solves a real multi-agent coordination problem
  • "Orchestration as scaling axis" is a compelling, well-argued framing
Limitations
  • No cost or latency analysis for multi-model workflows
  • No ablation against simpler baselines (random routing, majority vote)
  • Dependent on proprietary API access to all worker models
  • Orchestrator architecture and parameter count undisclosed
  • Reads more as a product announcement than reproducible research
Open Questions
  • What is the end-to-end latency and API cost vs. single-model baselines?
  • How much gain comes from learned orchestration vs. simply accessing multiple models?
  • Can the Conductor framework generalize to open-source model pools with weaker workers?
  • How robust is the system to worker pool changes (model updates, deprecation)?
