Exposing the Illusion of Erasure in Knowledge Editing for LLMs

Advik Raj Basani, Anshuman Chhabra arXiv: 2606.23276 2026-06-22
Key Finding
Knowledge editing methods do not erase pre-trained facts—they only suppress them. Deleted knowledge remains structurally intact and is recoverable via adversarial suffixes.
Key Metric
>85%
white-box extraction rate across all model-method pairs
Why This Matters
GDPR "right to be forgotten" compliance via KE is illusory. Suppression ≠ erasure—current methods fail the security bar for any privacy-critical deployment.
Critical Edge
Tested only on small models (3B–6B). Requires white-box access. No defense proposed. Only 100 edit tasks evaluated.

Methodology

  • Context-guided elicitation: optimized universal suffixes appended to known prompt templates to recover suppressed facts. Uses Greedy Coordinate Gradient (GCG) for discrete token optimization.
  • Blind reconstruction: random prefixes with learned suffixes reconstruct the full edit triplet (subject, relation, old object, new object) with zero semantic guidance.
  • Mechanistic toolkit: hidden-state decomposition into edit/null subspaces • intervention logit analysis via vocabulary-space projection • suppression ablation • 2D loss landscape slicing.
  • Experimental scope: EasyEdit framework, KnowEdit benchmark, 100 counterfactual replacements (60/40 split), 5-run averages, GPT-5.2-nano as LLM-as-a-Judge for blind reconstruction.

Assessment

Strengths

  • Rigorous, multi-layered mechanistic analysis going beyond behavioral evaluation—three observations form a coherent explanatory framework.
  • Blind reconstruction is a genuinely novel, harder threat model than prior work.
  • Formal proof (Lemma 1) provides theoretical grounding for why attacks work.
  • Cross-model and cross-method transferability elevates findings to a systemic indictment of the entire locate-then-edit paradigm.

Limitations

  • Only small models tested (3B–6B). Unclear if geometric vulnerabilities persist at 70B+ scale.
  • 100 edit tasks is narrow; diverse knowledge types (temporal, relational, multi-hop) not covered.
  • White-box access required for suffix optimization; no pure black-box attack.
  • No defense or improved editing method proposed—constructive direction left entirely open.
