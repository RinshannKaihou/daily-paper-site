> **一句话总结：** AlphaMemo提出结构化搜索过程记忆(SSPM)用于LLM驱动的alpha因子挖掘，通过记录编辑模式(AST差分提取)在特定父因子上下文中的残差表现，结合置信门控和非对称否决机制，在CSI 500和S&P 500上取得优于Alpha158、AlphaGen、AlphaGPT等基线的样本外预测和组合表现。

> **TL;DR:** AlphaMemo equips an LLM-based alpha mining agent with Structured Search-Process Memory (SSPM) that records which AST-diff edit motifs succeed or fail under specific parent-factor contexts. Using confidence-gated residual corrections (not absolute quality predictions) and asymmetric veto control (hard veto for failures, soft boost for successes), it achieves state-of-the-art out-of-sample factor-pool quality and portfolio Sharpe on both CSI 500 and S&P 500, outperforming AlphaGen, AlphaGPT, AlphaSAGE, and AlphaAgent baselines.

## 核心贡献
- 提出结构化搜索过程记忆(SSPM)：以(父因子上下文, 编辑模式)为键记录残差统计量，而非仅存储最终因子或完整轨迹，实现细粒度的搜索经验复用
- 设计残差记忆机制：记忆学习的是编辑相对于基础搜索先验的增量(残差)而非绝对质量，避免记忆覆盖已有的强搜索先验
- 引入AST差分编辑模式提取：通过对父子表达式的抽象语法树差分自动提取规范化编辑模式(如条件门控、窗口缩放、算子替换等)，替代人工标注的突变标签
- 提出非对称过程否决(APV)：负面模式以硬约束否决不可靠操作，正面模式仅作为软加成，反映金融alpha信号正面脆弱而负面稳定的特性
- 置信门控机制：通过观测次数和信噪比双重门控，确保低置信度记忆退化为基础搜索策略

## Key Contributions
- **Structured Search-Process Memory (SSPM)**: Memory keyed by (parent-factor context, edit motif) pairs storing residual statistics, enabling fine-grained reuse of search experience at the edit level rather than factor or trajectory level
- **Residual memory design**: Memory learns only the difference between observed child quality and the base scorer's expectation, preventing memory from overriding a strong search-ledger prior
- **AST-diff edit motif extraction**: Automatic extraction of normalized edit motifs from abstract syntax tree differences between parent and child expressions, replacing manual mutation labels with a compact canonical vocabulary (condition gates, rank switches, window rescaling, operator substitutions, etc.)
- **Asymmetric Process Veto (APV)**: Negative high-confidence patterns hard-veto unreliable actions, while positive patterns provide only soft additive boosts -- reflecting that negative evidence (invalid transforms, over-mined edits) is structurally more stable than positive alpha signals
- **Confidence gating**: Dual gating by observation count and signal-to-noise ratio ensures low-confidence memories collapse to the base parent-selection policy

## Methodology

**Problem setting**: Formulaic alpha mining -- discover a pool of symbolic expressions over price-volume variables that are predictive of 20-trading-day forward returns, non-redundant, and bounded in complexity. Evaluated on CSI 500 (Chinese A-shares) and S&P 500 via the Qlib platform.

**Architecture** (three-phase loop per iteration):
1. **Action Scoring & Selection**: For each candidate parent factor p and edit motif m, compute an action score combining (a) a search-ledger prior (log-transformed, based on factor quality, diversity, depth, lineage, retrieval frequency) and (b) a confidence-gated SSPM residual correction. Apply asymmetric veto to suppress high-confidence failure patterns.
2. **Factor Generation & Motif Extraction**: Prompt the LLM generator with the selected parent, its lineage trace, and intended motif. Parse the generated child expression. Extract the edit motif by computing the AST difference between parent and child, canonicalizing (sorting commutative subtrees, binning constants), and mapping to a compact motif vocabulary.
3. **Evaluation, Admission & Memory Update**: Evaluate the child factor on training/validation data using a quality vector (IC, ICIR, RankIC, RankICIR, AR, MDD, Sharpe). Apply admission control (quality, complexity, diversity filters). Update the search ledger and SSPM with the observed residual and failure status.

**Key formulations**:
- **Residual signal**: delta(p,c) = Q(c) - Q_hat_ledger(p), where Q_hat_ledger is the expected child quality from the base scorer based on similar parent contexts
- **Action score**: A(p,m) = log(S_ledger(p) + eps) + lambda * c(z,m) * Delta(z,m), where c is confidence and Delta is posterior mean residual
- **Confidence gate**: c(z,m) = [n/(n+kappa)] * min(1, |mu|/(sigma+eps)), combining count-based and SNR-based gating
- **Veto rule**: Veto(z,m) fires when confidence exceeds tau_c AND posterior failure probability exceeds tau_v
- **Motif vocabulary**: 10 named edit types (condition gates, rank switches, interaction additions, window rescaling, feature swaps, operator substitutions, nesting increases, temporal shifts, normalization changes, sign reversals) plus "other"

## Results & Highlights

**Main results (Table 2, 20-trading-day protocol, test 2022-2025)**:
- **S&P 500**: AlphaMemo (residual) achieves best IC (0.0351), ICIR (0.0410), RankIC (0.3434), annualized return (23.65%), and Sharpe (1.0672). AlphaMemo (balanced) achieves best RankICIR (0.1207) and lowest MDD (-22.54%)
- **CSI 500**: AlphaMemo (balanced) achieves best results across most metrics -- ICIR (0.3462), RankICIR (0.4597), AR (11.63%), and lowest MDD (-23.43%)
- Both configurations substantially outperform AlphaGen, AlphaGPT, AlphaSAGE, AlphaAgent, GP, Alpha158, LightGBM, and LSTM baselines

**Ablation studies (Table 3)**:
- Removing SSPM entirely (search-ledger only) drops ICIR from 0.3462 to 0.2313 on CSI 500
- Stronger memory hurts CSI 500 (lower Sharpe) but helps S&P 500, confirming memory should be calibrated, not maximized
- APV-only variant (no positive residual) is useful but weaker than balanced, showing both sides of the asymmetry contribute
- Weaker memory degrades both markets, showing SSPM provides genuine value

**Key insight**: Memory strength is a genuine control variable, not a monotonic knob. The balanced configuration keeps memory weak enough to preserve the search-ledger prior while still correcting systematic blind spots.

## Critical Assessment

**Novelty**: Moderate-to-high. The core idea of recording edit-level residuals rather than factor-level outcomes is well-motivated and distinct from prior work (AlphaGen, AlphaGPT, QuantAlpha). The AST-diff motif extraction is a clean engineering contribution. The asymmetric veto is a thoughtful design choice grounded in the asymmetry of financial signals.

**Rigor**: Solid. The paper includes mechanistic analysis (Sections 5.1-5.4) showing why residual memory is a bounded, conservative correction rather than an unconstrained override. The dual-market (CSI 500 + S&P 500) evaluation with different liquidity/sector characteristics strengthens claims. Ablations systematically remove each component.

**Reproducibility**: Good. Code is released. The Qlib platform and Alpha158 dataset are public. The 20-trading-day protocol is standard. Hyperparameters are reported in the appendix.

**Significance**: The paper addresses a real and underexplored problem -- how LLM agents should accumulate search experience in combinatorial, noisy, non-stationary domains. The memory-as-calibrated-correction framing (not memory-as-policy) is a useful conceptual contribution beyond alpha mining.

**Red flags**:
- The two "operating points" (balanced vs. residual) are presented as a feature, but they reveal that the method requires non-trivial tuning of the memory-strength tradeoff per market. No principled way to select between them is provided.
- Test period (2022-2025) includes highly unusual market regimes (post-COVID, rate hiking cycle). Robustness across longer or different historical periods is not shown.
- The 10-motif vocabulary is manually designed despite the paper criticizing manual mutation labels. The motifs are "normalized names for recurring edit signatures" rather than learned, but the vocabulary design still involves human judgment.
- Portfolio metrics (AR, Sharpe, MDD) are from TopK portfolio returns -- a simple long-only strategy. More realistic portfolio construction (transaction costs, turnover constraints) is not evaluated.

## Strengths
- Clean separation between the search-ledger prior and the residual memory correction, with formal bounds on how much memory can perturb the base policy (Section 5.2)
- The AST-diff motif extraction is a practical contribution that could be reused in other symbolic-search settings beyond finance
- Dual-market evaluation (China A-shares + US equities) with meaningfully different characteristics
- The asymmetric veto design is well-motivated by the empirical observation that negative signals are structurally more stable than positive ones in financial data
- Thorough ablation study that systematically validates each component's contribution
- Code availability

## Limitations
- Relies on executable formula parsing and standardized evaluation feedback -- not directly applicable to non-formulaic alpha discovery or live trading
- The motif vocabulary, while more principled than manual mutation labels, still requires domain-specific design
- Memory needs enough search observations to become informative, limiting applicability to short or small-budget search runs
- No evaluation of computational overhead or wall-clock time compared to baselines
- The paper does not address how SSPM handles concept drift or regime changes in the memory itself -- old memory entries may become stale
- Factor pool capacity is fixed at 50; scalability to larger pools is not studied

## Relevance
该论文将LLM自进化智能体与量化金融alpha因子挖掘结合，提出的结构化搜索过程记忆(残差记忆+置信门控+非对称否决)对LLM agent的经验积累和搜索策略优化具有方法论参考价值，尤其适用于组合搜索空间中反馈信号嘈杂且非平稳的场景。

## Questions for Further Exploration
1. How does the SSPM handle memory staleness when market regimes shift significantly (e.g., from trending to mean-reverting)?
2. Could the AST-diff motif extraction and residual memory framework be applied to other symbolic search domains (e.g., program synthesis, mathematical discovery)?
3. What is the sensitivity to the motif vocabulary size and granularity? Could motifs be learned or clustered automatically rather than manually categorized?
4. How does AlphaMemo perform under realistic trading constraints (transaction costs, turnover limits, capacity constraints)?
5. What happens when the factor pool capacity is scaled to 200+ factors -- does SSPM still provide the same relative improvement?
