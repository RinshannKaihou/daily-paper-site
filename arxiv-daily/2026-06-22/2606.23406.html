HyperQuant: A Rate-Distortion-Optimal Quantization Pipeline for Large Language and Diffusion Models

A unified, data-free, post-training quantization pipeline combining Hadamard rotation, lattice quantization, lossless bit-stripping, and Rice entropy coding for both weights and KV cache — achieving rate-distortion optimality where HIGGS wastes entropy.

Yuval Domb · Hadar Sackstein · Tomer Solberg

Hadamard · optimalY Packing · Entropy Rice-coding · Quantization
Key Finding
Strictly superior to HIGGS at every rate
Lattice quantization + variable-length Rice coding recovers 0.6–5.9% entropy slack that HIGGS wastes with fixed-rate codebooks, yielding +0.18–0.78 PPL advantage across 3–5 bps.
Key Metric
3.9× weight · 3.79× KV
Llama-3.1-8B on H100: 14.96 → 5.29 GiB weights, 0.516 → 0.136 bytes/scalar KV. Near-lossless PPL 7.503 vs 7.161 BF16 baseline.
Why This Matters
First rate-optimal data-free PTQ
Unifies weight and KV-cache compression in one pipeline, generalizes from 8B LLMs to 19B video DiTs without re-tuning, and provides formal information-theoretic guarantees.
Critical Edge
10× decode slowdown
Variable-length Rice decoding drops throughput from 51.8 to 5.4 tok/s. No 70B+ evaluation. No comparison with calibration-based methods (GPTQ, AWQ).

Encode / Decode Pipeline

BF16 → RHT → Rotate (KV) → Normalize → Dither (KV) → Quantize → Bit-Strip → Rice Encode → Compact Code

KV-only stages marked in accent. RHT folds into LayerNorm/RMSNorm at zero runtime cost. No explicit inverse needed — cancels algebraically via W = (WHT)H.

KV Cache Compression

Two-regime characterization:

  • ≥ 2.5 bps (high-quality): All bias-correction variants within 0.04 PPL of BF16
  • 1.7–2.5 bps (high-compression): QJL rotation leads by ~0.5 PPL
  • Peak compression: 7.1× at 1.7 bps

Throughput trade-off:

  • Prefill: 4,655 tok/s (vs 16,505 BF16)

Variable-length decode is the bottleneck. Proposed fixes (intra-stream parallelism, fused decode+MMA, rANS) remain future work.

Limitations & Open Questions

  • Throughput cost: 10× decode slowdown may prohibit latency-sensitive serving
  • Uniform allocation: All layers get same bps; per-layer DP allocator (as in HIGGS) not yet integrated
  • Scale gap: No experiments on 70B+ models where memory savings matter most
  • No calibration-based comparison: GPTQ, AWQ, OmniQuant discussed as orthogonal but not benchmarked
  • Best fit: Memory-bound workloads (long-context, batch inference, edge) rather than latency-bound
