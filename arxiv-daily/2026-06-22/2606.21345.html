Factual Retrieval in LLMs Is a Redundant, Distributed and Non-Contiguous Process

Hail Hochman, Natalie Shapira, Yoav Goldberg  •  2026-06-22  •  arXiv 2606.21345

Introduces "attribute-computation paths" and iterative activation patching to reveal that factual recall uses multiple redundant, layer-skipping routes -- challenging the view that knowledge lives in specific MLP layers.

Key Finding
Distributed
Facts are retrieved via multiple redundant, non-contiguous paths -- not localized in single MLP layers
Key Metric
80–83%
of facts have functionally equivalent alternative retrieval paths (LLaMA: 80.1%, Qwen: 82.7%)
Why This Matters
Knowledge editing fails because it targets single layers, but facts live across distributed redundant paths
Critical Edge
Only 8B models tested; single-token targets only; greedy search gives lower bounds -- true redundancy likely far higher

Core Contributions

  • Attribute-computation paths: minimal layer subsets necessary and sufficient for factual retrieval
  • Iterative patching protocol: counterfactual activation patching to auto-identify primary + alternative paths
  • Non-contiguous paths: 33% (LLaMA) and 79% (Qwen) of paths skip intermediate layers
  • Entity resolution is rare: only ~10-13% of paths show detectable ER steps
  • Editing mismatch explained: knowledge editing succeeds at transformation stages, not "storage" locations

Methodology

  • Phase 1 — Locate l_attr: "lock" operation freezes entity representation; finds earliest completion layer
  • Phase 2 — Greedy path search: iteratively skip maximum layers while maintaining correct prediction via "isolate" operation
  • Phase 3 — Alternative paths: relax l_attr constraint to discover deeper equivalent routes
  • Validation: Knockout, Downstream Injection, Path+Continuation, Global Broadcast
  • Data: 2,000 CounterFact prompts; single-token targets; NNsight interventions

Key Results

  • Early completion: avg l_attr = 4.61 (LLaMA) / 7.97 (Qwen) out of 32/36 layers
  • Redundancy: 80.1% (LLaMA) / 82.7% (Qwen) have non-identical alternative paths
  • Hybrid paths work: ~35% of prompts yield valid recombined prefix+suffix paths
  • Knockout effective: >94% disruption confirms path necessity
  • Injection partial: ~56% restoration shows intermediate processing matters
  • Robust to noise: path lengths stable (5.88 vs 5.91) under noise-free variant

Strengths & Limitations

  • + Clean formalization with precise minimality definitions
  • + Four distinct validation interventions triangulate claims
  • + Consistent results across LLaMA and Qwen architectures
  • + Connects to practical knowledge editing implications
  • + Code publicly released
  • − Greedy search = lower bound; true redundancy unknown
  • − 8B models only; scaling behavior uncharted
  • − Single-token targets limit generality
  • − CounterFact dataset biases (Western-centric)
  • − Backup paths may be intervention artifacts

Open Questions

  • How do paths change at 70B+ scale -- does redundancy increase or decrease?
  • Can editing multiple path layers simultaneously improve knowledge editing reliability?
  • Do facts learned earlier in pretraining develop more redundant paths?
  • How do multi-token attributes alter path structure?
  • Can path analysis distinguish memorized facts from compositional reasoning?
