Methodology

ATLAS takes two inputs: architecture parameters (from the parameterized template in Table 1) and LLM operator implementations expressed via programming primitives (Table 2). The workflow proceeds through four stages:

  1. Thermal Analysis: HotSpot 7.0 models the multi-layer stack (logic die + configurable DRAM dies). If temperature exceeds the DRAM retention threshold (85 C), the analyzer progressively reduces operating frequency until the constraint is satisfied. Thermophysical parameters are measured from actual silicon via HR-STEM and TDTR.
  1. Operator Parsing: Operator implementations are parsed into ASTs, generating execution descriptions and tensor placement specifications in YAML format.
  1. Intra-Core Tiling Exploration: Derives tiling factors (tM, tN, tK, tS) with optional autotuning against the cycle-level simulator. Tile-level command scheduling maximizes DRAM bandwidth utilization.
  1. Cycle-Level Performance Simulation: Per-core simulators (3D-DRAM via Ramulator 2, logic die via FLOPs/SRAM models, inter-core NoC via BookSim 2) are orchestrated by a global manager at operator granularity. Inter-accelerator communication uses a fixed-bandwidth model.

The framework is implemented in ~5.4K lines of C++ and will be open-sourced.

Critical Assessment

Strengths:

  • Silicon-validated accuracy (MAE < 4.01% on DRAM, < 2.72% on communication) is a significant credibility advantage over purely analytical models.
  • The unified architecture template and programming abstraction genuinely address a gap in the 3D-DRAM accelerator community, where prior works (H2-LLM, Stratum, Cao et al.) each used closed-source, scenario-specific tools.
  • The hierarchical DSE methodology producing ten actionable takeaways is a practical contribution for future accelerator designers.

Weaknesses:

  • The logic die simulator uses FLOPs-based performance models rather than detailed microarchitecture simulation, which may not capture pipeline hazards, cache contention, or dataflow scheduling inefficiencies. This is acknowledged but limits fidelity for compute-bound workloads.
  • The fixed-bandwidth model for inter-accelerator communication oversimplifies realistic multi-device systems (NVLink/CXL/RDMA have complex protocol overhead and contention).
  • The prefill stage is excluded from evaluation. Although the paper argues that prefill-decoding disaggregation is standard practice, a full-stack framework should ideally model both phases to assess total cost of ownership.
  • Thermal analysis relies on HotSpot with grid-based formulation; while silicon-measured parameters help, the steady-state/transient accuracy under bursty LLM workloads is not rigorously validated.
  • The paper claims "first silicon-proven full-stack framework" but the silicon validation only covers the DRAM and core-level modules; the multi-chip system-level validation is still simulation-based.
  • Open-source availability is promised "upon publication," which limits immediate reproducibility.
