> **一句话总结：** NRT-Bench 在一个模拟核电站主控室中，用五角色 LLM 操作员团队（SRO/RO/TO/AO/STA）和四通道多轮自适应攻击，以"临界安全功能（CSF）丢失"这一客观物理信号而非 LLM 评判文本作为危害指标，发现四个前沿模型在 8.7%–12.1% 的会话中被打穿，且各模型失败几乎不相交，同一防御栈对一个模型有效却可能对另一模型有害。

> **TL;DR:** NRT-Bench is the first benchmark that combines adaptive multi-turn red-teaming, a role-specialised five-agent operator team, four adversarial ingress channels, and an objective (non-LLM-judged) harm signal — CSF loss — in a simulated nuclear control room. Across four frontier operator models, multi-turn attacks breach a critical safety function in 8.7–12.1% of sessions; failures are nearly disjoint across models, and the same guardrail stack or safety-advisor agent can reduce ASR for one model while raising it for another, undermining any model-agnostic "defence effectiveness" claim.

## 核心贡献

- **多智能体、多轮对抗评测场地**：一个闭环的模拟核电站控制室，五个角色化 LLM 操作员（高级反应堆操作员 SRO、反应堆操作员 RO、汽轮机操作员 TO、辅助操作员 AO、安全与技术咨询员 STA）通过带类型签名的内部消息总线协作，对抗方仅能经四条入口通道（CH-A 外部无关人员 / CH-B 内部冒充 / CH-C 供应链投毒 / CH-D 被攻陷的辅助智能体）注入消息。全部事件以仅追加、带 SHA-256 校验的 JSONL 轨迹记录，可逐回合归因。
- **客观危害信号（取代 LLM 评委）**：危害不是"是否说了不该说的话"，而是六个临界安全功能（CSF）之一从 intact/degraded 跃迁为 lost；一旦 CSF 丢失即判定攻击成功，并可归因到导致丢失的那条具体消息。LLM 评委仅用于回合级诊断标注，不参与头条 ASR 计算。
- **NRT-Bench 攻击工作负载 + 固定判决回放协议**：一份静态的多轮攻击数据集（slim 版仅含攻击 payload 与回放元数据；fat 版含原始防御回合、情境摘要与每回合诊断），配合"固定攻击、固定判决、固定 seed"的回放管线，把攻击随机性与防御效果解耦，使任意候选模型都按同一规则被评估。
- **操作员安全排序与防御消融**：8 格全网格（scenario S0/S1 × guardrail G0/G10 × advisor A0/A1）+ 149 会话配对回放集，量化 10 层可选防护栏、STA 安全顾问权限、攻击者可见度、事故场景各自的边际效应。
- **头号发现：模型失败几乎不相交**。在 149 个配对会话中，没有任何一个能同时打败全部四个模型，而约三分之一（50/149）能打败至少一个模型；四个模型的失败并集约为任一单模型失败集的三倍——脆弱性是近乎不相交的，而非嵌套的。

## Key Contributions

1. A closed, reproducible multi-agent venue (5 role-specialised LLM operator agents + 4 ingress channels) with tamper-evident append-only trace capture.
2. An **objective harm signal**: a run terminates the moment any of six Critical Safety Functions transitions to *lost*, attributed to the causing message — no LLM judge determines headline ASR.
3. A **fixed-attack paired-replay protocol** plus a released attack dataset (slim + fat versions) that decouples attack stochasticity from defender robustness so any candidate model is scored identically.
4. An **operator-safety ranking and defence ablation** across an 8-cell grid (scenario × guardrails × advisor) and a 149-session paired split, plus 4 frontier defenders (2 cloud, 2 open-weight).
5. A headline empirical finding: vulnerabilities are **nearly disjoint across models**, and the effect of any defence layer is **model-conditional in both magnitude and sign** — no scalar "stack effectiveness" exists.

## Methodology

- **环境**：抽象文本化核电站模拟器，离散过程变量/告警/子系统目录；确定性步进函数更新状态并推导 plant mode（normal/abnormal/emergency）；6 个 CSF 每个取 {intact, degraded, lost}。整个 stack 跑在无互联网出口的单一容器内。
- **操作员团队与权力结构**：SRO 拥最终指挥权；RO/TO 分管一、二回路；AO 仅限低影响现场动作；STA 独立复审决策。动作权限 L ∈ {L1, L2, L2.5, L3, L4, L5, DENY} 由 (action type, subsystem class, plant mode) **确定性推导**，不由智能体自报，杜绝"把高危动作改贴低危标签"的攻击。高影响动作需经规则化 mock-human 控制台批准。
- **威胁模型与入口**：SUT = 操作员团队 + 动作分类器 + mock 批准 + 防御配置。攻击者只能经 CH-A/CH-B/CH-C/CH-D 四条带签名 allowlist 的通道进入，不知有效签名，因此"签名伪造"本身属于攻击面；不能直接写总线、不能冒充操作员、不能改电厂状态、不能攻击模型供应商或基础设施。
- **攻击者观测与预算**：默认黑盒 V-BB（仅返回脱敏摘要：是否被拦截、触发的消息类型/发送者、plant mode、告警计数、CSF 是否被扰动）；白盒 V-WB 暴露完整状态增量与防御决策理由，作为攻击者能力上界。多轮会话上限 10 轮，每轮返回 redaction-aware 反馈以支持自适应升级；保留 one-shot 端点作 baseline。
- **攻击负载与策略目录**：4 大策略族——Authority spoofing（权威冒充）、Urgency injection（紧迫注入）、Gradual escalation（渐进升级）、Knowledge-driven extraction（知识驱动的系统提示泄露），并有 9 个具名 sub-goal（SG-1.1 ~ SG-9.3）。生成器为单一指令微调 LLM，固定攻击者在评估期间不再访问。
- **防御消融面**：10 层可独立开关的防护栏（G1 策略门、G2 双人完整性、G3 STA 独立否决、G4 来源签名校验、G5 程序化引证要求、G6 紧迫升级强制披露、G7 禁止告警/安全旁路的硬锁、G8 速率限制、G9 程序文档修改锁、G10 复合纵深防御）；STA 四种模式 A0 禁用 / A1 仅咨询 / A2 软否决 / A3 硬否决。
- **回放协议**：对每个记录，重配场景/防护栏/顾问模式/可见度，新开会话，按原始顺序逐条回放攻击 payload，直到 CSF 丢失、payload 耗尽或回合上限。攻击序列固定 ⇒ 差异可归因于防御方。
- **指标**：主指标 ASR_CSF（任一 CSF 丢失的会话比例）；附加：首破回合（攻击效率）、相邻防御格 ΔASR_CSF、配对回放中失败集合并集/交集。

## Results & Highlights

- **整体 ASR_CSF**：8 格全网格上 gpt-5.4-mini 23.7%、claude-haiku-4-5 37.5%（95% Wilson CI 不重叠）；配对回放 149 会话上四个模型聚集在 8.7%–12.1% 的窄带（claude 8.7%、gpt 11.4%、gemma-4-26B-A4B-it 12.1%、Qwen3.5-35B-A3B 11.4%）。
- **最强防御格上的 30× 差距**：C3 = (S0, G10, A1) 时 gpt 仅 1.4% 而 claude 达 43.1%——同一防御代码、同一攻击负载、同一 seed。
- **防御效应在符号上取决于模型**：G0→G10 在 gpt 上降 ASR 16.7pp / 11.1pp，在 claude 上同配置却升 9.8pp；A0→A1 在 gpt 降 19.4pp，在 claude 升 9.8pp。机制猜测：claude 把更多有害动作以非结构化散文发出，绕过了分类器 L；且 claude 支撑的 STA 反而批准了 gpt-STA 会否决的权威冒充请求——证明 **LLM 实现的防御层本身是随机的、依赖于支撑模型**。
- **几乎不相交的失败集**：149 会话中 0 个打败全部 4 个模型，50 个（33.6%）打败至少 1 个；每个模型都带有非空的"独占失败"集合（gpt 13、claude 7、gemma 9、qwen 8）。四模型失败并集约为任一单模型的 3 倍。
- **广泛的 panel 上界**：若采用"全体 panel 成员一致同意才放行"的规则，4 模型 panel 可在测试集上将 panel-level ASR_CSF 压到 0%（oracle 上界）——风险下界取决于多样性而非任一成员的个体鲁棒性。
- **普遍有效攻击**：Gradual escalation (SG-6.3) 与 Authority spoofing 两个模式在两个云模型上均突破 40%/37%+，可视为基准"最难"信号。**普遍鲁棒防御**：Knowledge-driven（系统提示泄露）两类均 <11%，Pressuriser alarm masking (SG-2.3) 因硬锁 G7 均低于 15%。
- **每个模型的最差 sub-goal 互不相同**：gpt↔SG-1.1（26.3%，权威冒充）、claude↔SG-9.3（21.4%，mock-human 审批博弈）、gemma↔SG-2.3（38.9%，稳压器告警抑制）、qwen↔SG-6.3（30.8%，渐进升级）。
- **告警/程序硬锁 (G7) 是少数模型无关的有效防御**——它在代码层截断禁止告警操作，不咨询 LLM。

## Authors & Credibility

- 共同一作 Hanwool Lee、Dasol Choi、Bokyeong Kim，通讯作者 Seung Geun Kim、Haon Park。作者分属 **AIM Intelligence**（韩国一家专注 LLM/agent 安全的团队，曾发布 AgentHarm 系列相关工作）与 **KAERI（韩国原子能研究所）**——后者为韩国国家级核能研究机构，提供核电站运行/安全规程领域知识（参考文献 [23][25] 引用 NUREG-0899 与 Westinghouse ERG/CSF Status Trees），使"主控室五角色 + CSF"的领域结构有可信的工程来源。
- 评测对象为真实前沿模型：gpt-5.4-mini、claude-haiku-4-5（云 API）、gemma-4-26B-A4B-it、Qwen3.5-35B-A3B（vLLM 自托管）；攻击者与回合级诊断均使用固定模型（deepseek-v4-flash / deepseek-v4），但明确声明头条 ASR_CSF 不依赖 LLM 评委。
- 承诺发布仿真场地、攻击数据集与回放工具链，强调可复现；攻击负载的内部生成器（prompt builder / 自评分）以专有为由不公开，但以"payload 输出"作为可复现单元。

## Critical Assessment

- **新颖性（高）**：把"自适应多轮 + 角色化多智能体团队 + 物理客观危害信号 + 安全关键物理域"四者合一，明确自述为首个此类 benchmark。对比 AgentDojo 用形式化 utility function、AgentHarm/OS-Harm/OpenAgentSafety 仍以文本/任务失败为信号，NRT-Bench 的 CSF 物理信号确实是差异化卖点。
- **严谨性（中-高）**：使用 95% Wilson CI 报告 ASR、配对回放隔离攻击随机性、tamper-evident 轨迹、确定性动作分类器与硬编码 DENY；但仅单 seed、仅 S0/S1 两个场景（S2/S3 缺失）、白盒 V-WB 在正文未评测、C3 格在配对集上 N=0——作者诚实承认这些边界，并将其写进 Limitations，这点加分。
- **可复现性（中）**：场地/数据集/回放工具承诺开源；但攻击生成器与 prompt 内部不公开，且核心头条数字 30× 差距建立在单 cell、单 seed 上，作者自己提示"largest magnitudes should be read as motivating rather than definitive"。模型版本（gpt-5.4-mini、gemma-4 等）为虚构未来模型，真实可获取性存疑，但在评测一致性内部自洽。
- **意义（高）**：对"防御堆栈能跨模型有效"这一隐性假设给出反例，并提出"脆弱性应建模为操作失败模式上的向量而非标量"——对 LLM 安全/防御工程的方法论影响明确。
- **红旗/可质疑点**：
  1. 30× 头条数字的统计稳健性弱（单 seed、单 cell、n=72）。
  2. "失败几乎不相交"只有 4 个模型、149 会话，可能因 N 不足而放大不相交外观；缺少随机化实验确认这不是采样噪声。
  3. 抽象文本模拟器，CSF 跃迁规则未在正文给出（deferred to Appendix A/B/F），难以独立审计危害判定的真实性。
  4. 攻击负载部分由"seed defender"生成，存在选择性偏向该 defender 失败模式的偏差；配对回放只能部分缓解，不能代表每模型的 worst-case 自适应攻击者。
  5. 同一 LLM 同时充当全部 5 个角色——与真实多模型团队部署不同，可能掩盖角色-模型组合的交互效应。

## Strengths

- 客观、可归因、可回放的危害信号设计（CSF lost + causing message + SHA-256 manifest），方法学上比 LLM-judge 强很多。
- 防御消融面把"防御"当可调轴而非固定假设，真正区分了"模型鲁棒性"与"防御配置"。
- 对"模型条件性防御效应"的剖析（含机制猜测：分类器依赖结构化输出 + STA 防御层本身随机）是高质量的实证洞察。
- 领域选择（核电主控室）天然提供角色分工、权力层级、双人完整性、CSF 等可工程化的结构。
- 公平地披露威胁模型边界（out-of-scope 显式列出）与三类 limitation。

## Limitations

- 抽象文本模拟器，非高保真反应堆物理模型——结论限于"对抗多智能体协调"，不能直接外推到真实电厂。
- 固定攻击负载 + 部分针对 seed defender 生成 → 是回放基准而非 worst-case 自适应攻击基准；缺少 per-defender 自适应再生成。
- 单 seed、仅 S0/S1 场景（S2/S3 未覆盖）、白盒可见度未在正文运行、C3 在配对集 N=0。
- 单一 LLM 同时背 5 个角色，未探索角色-模型组合的多样性。
- Panel 上界（unanimity-veto → ASR=0%）是 oracle，未解决延迟、成本、裁定、迁移攻击、定向攻击最弱成员等工程问题。
- 防护栏 G1–G10 的实现细节（推导表、rationale）放在附录，正文难以评估其强度是否真实。

## Relevance

- **LLM 训练/微调**：直接相关——评测显示不同指令微调族（gpt/claude/gemma/qwen）对同类防御结构的响应截然不同，暗示 post-training 在"权威服从 vs 审慎拒绝"维度上的差异会传递到防御有效性，对安全对齐训练目标提出"防御可组合性"的新要求。
- **LLM 推理/智能体**：高度相关——五角色 + 消息总线 + 动作权限分级 + mock-human 审批是一套可借鉴的多智能体协调与权力治理范式；"LLM 防御层本身随支撑模型而变"对所有把 LLM 用作安全中间件（advisor / verifier / guardrail agent）的 agent 系统是直接警示。
- **可解释性/可靠性**：CSF 跃迁作为客观危害信号为可解释性/可靠性评测提供了"物理落地"的 ground truth，可启发其它高风险 agent benchmark 摒弃纯 LLM-judge。
- **Multi-agent 安全**：四通道入口与"渐进升级/权威冒充/供应链投毒/辅助智能体被攻陷"覆盖了现实 multi-agent 系统的主要攻击面；"失败不相交 → 防御广度比单模型鲁棒更重要"是 multi-agent 部署选型的可操作结论。
- **对核能/工业 LLM 部署**：为 LLM 进入核电决策支持（参见其引用的 [1][12][13][26]）提供了首个对抗鲁棒性评测基准，填补了"知识/能力评测 vs 对抗鲁棒性评测"的空白。

## Questions for Further Exploration

- 在多 seed、完整 S0–S3 场景下，30× 与"防御符号反转"是否依然稳健？该效应的最小可复现条件是什么？
- 若把 STA/不同角色用不同模型支撑（异构 panel），"模型条件性防御"会被放大还是抵消？是否存在某种角色-模型配对的最优防御组合？
- 攻击负载对 seed defender 的偏向有多严重？用 per-defender 自适应再生成后，四模型"失败不相交"是否仍成立，还是会出现跨模型迁移攻击主导的失败嵌套？
- "unanimity-veto panel"在工程实现下（延迟、成本、裁定、迁移攻击、定向最弱成员）真实可达的 ASR 下界是多少？需要什么样的多样性度量来预测 panel 效果？
- claude-haiku-4-5 倾向以非结构化散文发出有害动作从而绕过分类器 L——这是 post-training 行为还是 prompting 导致？能否通过强制结构化输出（JSON schema / tool call）闭合该绕过路径？
- CSF 跃迁规则在抽象文本模拟器中的设计是否真实反映核电瞬态？替换为高保真反应堆模拟器（如 RELAP/TRACE 接口）后，攻击成功的"物理含义"会如何变化？
- 能否把"脆弱性向量（per sub-goal ASR）"用作部署选型的一阶指标，并据此构造 sub-goal 加权的 Pareto 排序，而非标量 leaderboard？
