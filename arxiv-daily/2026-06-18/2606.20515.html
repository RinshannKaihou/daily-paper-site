Spatial Tool-Use Elicits Reasoning for Spatial Intelligence

arXiv: 2606.20515

Date: 2026-06-18

https://arxiv.org/abs/2606.20515

一句话总结： 提出S-AGENT——一种空间工具使用的智能体范式，将VLM作为语义规划器，结合三级层次化空间工具（2D定位→2D-to-3D几何抬升→空间知识聚合）与场景/智能体双记忆系统，把空间推理重新定义成跨视角、跨帧的时空证据累积过程，训练-free地在MMSI-Bench/ViewSpatial-Bench/ReVSI上提升GPT-5.4/Gemini 3，蒸馏出的S-AGENT-8B在MMSI-Bench较Qwen3-VL-8B提升10.5个百分点（31.1→41.6）。
TL;DR: S-AGENT reframes spatial reasoning as spatio-temporal evidence accumulation rather than single-shot prediction: a VLM acts as a semantic planner that invokes a 3-level hierarchy of spatial tools (2D grounding → 2D-to-3D geometric lifting → spatial-knowledge experts) and maintains dual Scene/Agent memory across frames. Training-free, it boosts closed/open VLMs (e.g., +4.5 over GPT-5.4 on MMSI-Bench, +14.4 on ViewSpatial-Bench), and distilling its GPT-5.4 trajectories into S-300K yields S-AGENT-8B, which beats Qwen3-VL-8B by +10.5 pts on MMSI-Bench and rivals GPT-5.4/Gemini 3 Pro.

核心贡献

  • 重新定义空间推理为证据累积过程：不再让VLM单帧预测，而是把空间推理建模为在持续演化的3D场景上、跨视角跨时间的"时空证据累积"（spatio-temporal evidence accumulation），从帧级识别转向场景中心理解。
  • 三级层次化空间证据机制：Level 1做2D视觉证据获取（关键帧选择、对象grounding、开放词表检测，基于GroundingDINO）；Level 2做2D→3D几何抬升（Depth-Anything-3度量深度、VGGT多视图重建，恢复深度/3D坐标/相机位姿/BEV）；Level 3做空间知识聚合（5个专家：度量测量、计数、朝向、相对位置、对象中心视角）。
  • 双记忆系统支持有状态推理：Scene Memory 绑定重复观测到持久场景实体，累积视觉/几何属性与派生空间事实（去重、跨视图融合、尺度校准）；Agent Memory 记录规划思路、工具调用、观测、失败与中间结论，避免冗余调用、矛盾与遗漏。
  • 训练-free的推理时增强：直接以GPT-5.4/Gemini 3 Pro/Qwen3作为tool-calling planner，无需任务训练即可在多个基准上一致提升。
  • 轨迹蒸馏得到紧凑空间智能体：用GPT-5.4作为frozen teacher在SenseNova-SI-800K上生成轨迹，经过滤（最终答案正确性，非按工具使用硬过滤）与多粒度分解（最终答案/turn级/专家级），得到S-300K（292,391个SFT样本），微调Qwen3-VL-8B得到S-AGENT-8B。

Key Contributions

  • Reframes spatial reasoning from isolated frame-level prediction into spatio-temporal evidence accumulation over an evolving 3D scene.
  • A 3-level hierarchy of spatial tools/experts (2D grounding → 2D-to-3D geometric lifting via Depth-Anything-3/VGGT → specialized spatial experts for measurement/counting/orientation/relative-position/view-conditioned relations).
  • A dual-memory mechanism (Scene Memory for persistent entity-centric spatial evidence; Agent Memory for reasoning trajectory) enabling stateful, cross-view, cross-step reasoning.
  • Training-free inference-time augmentation that consistently improves both open-source (Qwen3) and closed-source (Gemini 3 Pro, GPT-5.4) VLMs.
  • Trajectory distillation: GPT-5.4-generated trajectories filtered and decomposed into S-300K, fine-tuned onto Qwen3-VL-8B to yield the compact S-AGENT-8B.

Methodology

Formulation. Given a question q and visual observations F (multi-view images or video), S-AGENT maintains two memory states at step t: scene state S_t (grounded entities + spatial attributes/facts) and history H_t. A tool-calling VLM planner π_θ maps (q, F, S_t, H_t) to an evidence request r_t; a tool/expert T^(k) executes it producing observation o_t, which updates both memories via (S_{t+1}, H_{t+1}) = Update(S_t, H_t, r_t, o_t). The agent terminates when accumulated evidence suffices.

Hierarchical Spatial Evidence.

  • Level 1 (2D): detect_objects_tool (GroundingDINO open-vocab detection), vlm_ground_objects (2-stage visibility vote + best-frame detection), depth_estimation_tool (image-level depth), and video TStarKeyframeSearchTool for informative frame selection.
  • Level 2 (3D lifting): metric_depth3d_tool built on Depth-Anything-3, returns metric depth, 3D coordinates, camera poses; VGGT used for co-visibility bridging / point-cloud / BEV.
  • Level 3 (experts): Metric Measurement Expert (deterministic measurement routes — closest-point/center-to-center/longest-dim), Counting Expert (NMS dedup + question-conditioned aggregation), Visual Orientation Expert (object-facing reasoning from surfaces/handles/symmetry), Relative Position Expert (lifts boxes to shared 3D system, compares under egocentric/viewpoint/cardinal frames), Object-Centric View Expert (front/back/left/right view mapping).

Temporal Memory. Each observation is decomposed into reusable scene evidence e_t (merged into Scene Memory) and process context c_t (appended to Agent Memory). Scene Memory stores grounded entities (aliases, supporting frames, geometry) and derived facts (relations/measurements + provenance). Agent Memory stores planner thoughts, tool calls, observations, failure messages, conclusions.

Training-time distillation. Sample 100K questions from SenseNova-SI-800K (disjoint from eval), run frozen GPT-5.4 teacher S-AGENT to generate full trajectories. Filter by answer-type-specific correctness (MC option-match; numeric MRA≥0.6; text normalized match/substring). Decompose each retained trajectory into final-answer / turn-level / expert trajectories. Result: 51,596 filtered trajectories → 292,391 SFT samples (S-300K). Fine-tune Qwen3-VL-8B-Instruct via LLaMA-Factory, qwen3_vl_nothink template, 8×B200, seq-len 8192, lr 5e-5, cosine 3% warmup, 1 epoch.

Results & Highlights

Zero-shot (training-free).

  • MMSI-Bench (Table 1): S-AGENT 46.4% avg — beats Gemini 3 Pro (45.2) by 1.2 and GPT-5.4 (41.9) by 4.5; best on camera motion (46.0), object motion (48.7), multi-step reasoning (44.4). Massive subcategory gains over Qwen3-VL-8B baseline (e.g., C-R +38.6, Cam. +31.1, Meas. +28.1).
  • ViewSpatial-Bench (Table 2): 60.0% avg — beats GPT-5.4 (45.6) by 14.4; best on C-OVO (55.5) and P-RD (81.1), +20.5 on the hard P-SSRD split.
  • ReVSI (Table 3): 58.8 avg, rank-2 overall (behind Gemini 3 Pro's 60.9), beats all open-source general & specialized models; best on relative direction (66.4) and route planning (66.1), rank-3 on relative distance (63.6).
  • VSI-SUPER VSR (Table 7): large gains in long-video; +37.2 pts over Cambrian-S-7B-LFP at 240-min. (VSC still trails Cambrian-S-7B-LFP.)

Trajectory distillation (Table 4). S-AGENT-8B vs base Qwen3-VL-8B-Instruct: MMSI 31.1→41.6 (+10.5), ViewSpatial 42.2→46.8, ReVSI 49.1→52.8. Notably, simply wrapping the untrained 8B planner with S-AGENT (30.7 MMSI) does NOT help — base 8B struggles with tool selection and noisy observations; distillation is what teaches reusable tool-use and evidence-integration. S-AGENT-8B is competitive with GPT-5.4/Gemini 3 Pro on these benchmarks.

Ablations (Table 5, ViewSpatial, GPT-5.4 planner). VLM-only 45.6 → +L1 49.0 → +L2 (raw 3D) only 49.8 (limited; raw 3D noise distracts) → +L3 experts 56.7 (experts critical for interpreting 3D) → +Scene mem 58.2 → +Agent mem 57.6 → full 60.0. Key takeaway: expert-mediated interpretation is what makes 3D evidence useful; raw 3D can hurt.

Critical Assessment

Novelty: Moderate-to-strong. The core thesis (spatial reasoning = evidence accumulation + stateful memory rather than single-shot) is conceptually incremental over existing agentic spatial methods (VADAR, SpaceTools, GCA, Think3D), but the specific combination — hierarchical 3-level tool taxonomy + dual Scene/Agent memory explicitly designed to persist cross-frame entity identity and reasoning trajectory — is a coherent and non-trivial contribution. The dual-memory design (separating reusable scene facts from procedural context) is the most novel element.

Rigor: Reasonable. Four diverse benchmarks across multi-image and video; ablations isolate each component; S-300K construction is documented with filtering thresholds. However:

  • Comparisons are sometimes apples-to-oranges on compute/cost (closed-API planner with many tool calls vs single-pass baselines) — latency/token cost not reported.
  • Distillation comparison would be stronger if S-AGENT-8B were compared against Qwen3-VL-8B trained on an equal-amount of raw SFT data (ablation isolating "trajectory structure" from "more spatial data").
  • VSI-SUPER VSC results are explicitly weak (paper admits S-AGENT does not beat Cambrian-S-7B-LFP on VSC), an honest but important limitation.

Reproducibility: Medium. Tools rely on third-party models (GroundingDINO, Depth-Anything-3, VGGT, T*); S-300K is derived from SenseNova-SI-800K whose license/availability may constrain replication. Training config (8×B200, lr, epochs, template) is given. Full tool prompts and expert routing logic appear to be in appendix/figure form rather than fully specified code.

Significance: High for the spatial-VLM subfield. Demonstrates a clean, model-agnostic recipe that turns any tool-calling VLM into a stronger spatial reasoner, and a distillation pipeline to bake this into a compact open model. Likely to influence follow-up agentic-VLM and embodied reasoning work.

Red flags / concerns:

  • Benchmark leakage: SenseNova-SI-800K is a ByteDance-internal dataset; the S-AGENT team has institutional overlap (ByteDance co-author), so the "disjoint from evaluation" claim is reasonable but not externally auditable. ReVSI is a 2026 benchmark (arXiv 2604.24300) — freshness risk.
  • The headline "+10.5 on MMSI-Bench" compares S-AGENT-8B to vanilla Qwen3-VL-8B, not to Qwen3-VL-8B also SFT'd on equivalent non-agent spatial data — this inflates the apparent benefit of agentic distillation vs simple data scaling.
  • Closed-API zero-shot numbers depend on planner behavior that can be brittle and non-deterministic; no variance/seed analysis shown.
  • Naming/version drift in baselines (GPT-5.4, Gemini 3 Pro, Grok 4, Qwen3.5-9B) reflects a fast-moving landscape; absolute numbers may date quickly.

Strengths

  • Clean conceptual separation: VLM does semantic planning, deterministic tools/experts do perception & geometry, memory carries state — sidesteps asking the VLM to do unreliable metric reasoning in free-form text.
  • Ablation cleanly shows experts (not raw 3D) are the source of Level-2/3 value — a practically useful finding.
  • Dual-memory design directly addresses a real failure mode of multi-frame tool-use agents (redundant calls, lost identity, contradictions).
  • Works as both inference-time framework and distillation teacher — flexible deployment story.
  • Strong, broad empirical coverage (multi-image + video; static + motion; camera/person perspective; metric + relational).

Limitations

  • Latency/cost overhead not reported: multi-round tool calls + memory updates can be expensive; the paper does not quantify tokens, wall-clock, or API cost per question.
  • Reliance on strong third-party perception models (Depth-Anything-3, VGGT, GroundingDINO); pipeline accuracy is upper-bounded by these.
  • Weak on VSC (video spatial change) — explicitly admitted; suggests the memory mechanism does not yet capture change-over-time semantics well.
  • Untrained 8B planner does not benefit from S-AGENT wrapper (MMSI 31.1→30.7) — the framework only helps with either a strong frontier planner or after distillation, narrowing the "training-free" applicability claim to capable closed models.
  • S-300K derived from a proprietary dataset (SenseNova-SI-800K); reproducibility of the distilled model depends on data access.
  • No analysis of failure modes beyond one qualitative success; quantitative error breakdown by tool-failure pattern missing.
  • No statistical significance / multiple-seed variance reported.

Relevance

S-AGENT is highly relevant to several active LLM/VLM research threads:

  • Inference-time tool-use & agents: exemplifies the "VLM-as-planner + deterministic tools" pattern (cf. ViperGPT, ReAct, VADAR, MM-ReAct) and shows memory is the missing ingredient for multi-frame settings — directly informs agent design for LLM tool-use loops generally.
  • Interpretability: the explicit tool/observation trail and dual-memory structure make the reasoning process auditable (you can inspect what evidence was requested/obtained), a step toward more interpretable spatial reasoning than black-box single-shot VLMs.
  • Reliability: by grounding answers in recovered metric 3D coordinates rather than textual priors, it mitigates the "semantic-to-geometric gap" that causes VLMs to hallucinate spatial relations — a reliability win.
  • LLM training: the distillation recipe (teacher agent trajectories → multi-granularity SFT: final-answer / turn-level / expert) is a transferable blueprint for distilling agentic behavior into compact models, applicable beyond spatial reasoning.
  • Embodied agents / world models: stateful scene memory across viewpoints is a prerequisite for embodied robotics, AR/VR, and autonomous driving use cases the authors cite.

Questions for Further Exploration

  • What is the per-question latency/token/API-cost overhead of S-AGENT vs single-shot baselines, and how does it scale with video length?
  • How much of S-AGENT-8B's gain comes from agentic trajectory structure vs simply more equivalent spatial SFT data? (Need an equal-data non-agent SFT control.)
  • Can the Scene/Agent memory design be extended to capture temporal change semantics to fix the weak VSC performance?
  • How brittle are the zero-shot closed-API numbers to planner prompting, temperature, and seed? Is variance reported anywhere?
  • Does the framework transfer to non-spatial multi-frame reasoning (e.g., procedural video QA, temporal action ordering), or is the tool hierarchy too spatially specialized?
  • What happens when Level-2/3 tools (Depth-Anything-3, VGGT) fail or are noisy on degenerate views — is there graceful degradation, or does the planner get misled?
  • Could the dual-memory mechanism be integrated into the VLM's own context (e.g., structured KV/memory tokens) rather than an external text buffer, to reduce prompt length?
  • Is there benchmark contamination risk between SenseNova-SI-800K and the 2026-era eval sets (ReVSI, VSI-SUPER), and how is it externally verified?
