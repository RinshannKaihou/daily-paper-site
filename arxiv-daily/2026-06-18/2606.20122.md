> **一句话总结：** ScaffoldAgent 把开放式深度研究中的大纲当作可演化的结构脚手架，用 Expansion/Contraction/Revision 三种显式操作 + 融合检索增益、结构连贯性与试写质量的效用信号来指导节点选择、操作调度与终止，在 DeepResearch Bench / Gym 上同时提升了长报告质量与事实可溯源度。

> **TL;DR:** ScaffoldAgent reframes open-ended deep research as utility-guided outline optimization: an Outline Agent grows/refines an evolving outline tree via Expansion, Contraction, and Revision, while a composite utility (retrieval gain + structural coherence + trial-generation quality) provides inference-time signals for node selection, operation scheduling, and termination, beating strong single- and multi-agent baselines on DeepResearch Bench and Gym.

## 核心贡献

- **将大纲建模为"演化中的结构脚手架"而非静态写作计划**：节点同时携带意图 `h_v`、证据/报告块 `E_v`、以及历史效用统计 `θ_v`，使大纲同时充当检索目标、证据索引和生成约束三者。
- **提出三类 AGM（信念修正）启发的显式结构操作**——Expansion（细分粒度）、Contraction（合并冗余兄弟节点）、Revision（刷新弱支持节点的证据且不扩树）——直接缓解"脚手架漂移（scaffold drift）"问题 C1。
- **引入效用引导的反馈机制**，把一次大纲修改的下游价值分解为 `U_t = λ1·U_ret + λ2·U_str + λ3·U_gen`（检索相关性+新颖度、结构凝聚+平衡-冗余、试写的引用支持+覆盖-冗余），用 UCB 式规则选节点、按边际效用增益 `< ε` 终止，解决"延迟反馈"问题 C2。
- **多智能体协作**：Outline Agent 作中央控制器、Search Agent 按需取证、Reporter Agent 做试写并产出最终报告；Expansion/Revision 调用全部两个子智能体，Contraction 只调用 Reporter（重组已有证据）。
- **在两个公开基准上一致领先**，并附带消融（操作空间 + 三维效用）、操作分布分析与多轮 follow-up 增量更新实验，验证 AGM 分解能做"非破坏式"多轮精修。

## Key Contributions

- Reframes the OEDR outline as an **evolving structural scaffold** (not a fixed plan) whose nodes carry intent, attached evidence, and per-node utility statistics — turning the outline into a joint retrieval index / generation constraint.
- Defines **three explicit AGM-inspired outline operations** — Expansion, Contraction, Revision — that map to the three common outline failure modes (insufficient granularity, redundant branching, weak grounding).
- Introduces a **composite utility-guided feedback signal** `U_t = λ1·U_ret + λ2·U_str + λ3·U_gen` used at inference time (not training time) for node selection (UCB-style), operation scheduling, and marginal-gain termination.
- A **three-agent architecture** (Outline / Search / Reporter) where the Outline Agent is the central controller and selectively dispatches sub-agents per operation type.
- Demonstrates state-of-the-art results on **DeepResearch Bench and DeepResearch Gym**, plus an ablation suite isolating each operation and each utility term, and a multi-turn follow-up study.

## Methodology

- **Task formulation.** OEDR is modeled as a dynamic loop among three components — Search (retrieve evidence from corpus/web), Outline (organize evidence into a hierarchical scaffold), Report (synthesize long-form text). They are mutually dependent: outline → query, new evidence → outline, generation → expose gaps.
- **Evolving outline tree.** State at iteration `t` is `T_t = (V_t, P_t, Φ_t)`, with node attributes `Φ_t(v) = ⟨h_v, E_v, θ_v⟩` (intent, evidence+chunks, utility stats). Initialized as `T_0 = Init(q)`.
- **Node selection.** UCB-style rule balances exploitation and exploration:
  `v_t = argmax_{v∈V_t} [ −ū(v) + c · sqrt(ln N_t / n_v) ]`, where `N_t` is total visits and `c` the exploration coefficient. Mean utility updated incrementally (Eq. 7).
- **Operation scheduling.** Outline Agent chooses `o_t ∈ {Expansion, Contraction, Revision}` based on the node's utility profile (broad → Expansion; overlapping siblings → Contraction; weakly supported → Revision).
- **Three operations:**
  - *Expansion*: decompose a too-broad node into finer children, call Search Agent for child evidence, then Reporter Agent trial-writes to validate the expanded subtree.
  - *Contraction*: merge semantically overlapping / fragmented sibling nodes, regenerate intents; no Search Agent call, Reporter only checks coherence.
  - *Revision*: keep node position but refresh intent + evidence grounding; calls Search + Reporter; does not grow tree.
- **Composite utility.** `U_t = λ1·U_ret + λ2·U_str + λ3·U_gen`:
  - `U_ret = w_rel·Rel + w_nov·Nov` (semantic relevance to node intent + novelty vs already-attached evidence).
  - `U_str = w_coh·Coh + w_bal·Bal − w_red·Red` (semantic consistency + structural balance − sibling redundancy).
  - `U_gen = α·ρ_sup + β·ρ_cov − γ·ρ_red` (citation support + intent/content alignment − cross-section redundancy), scored on Reporter Agent's trial write.
- **Termination.** Stop when average marginal utility over a window of `k` iterations drops below `ε` (Eq. 8); then Reporter Agent writes the final report along the converged scaffold via DFS traversal.
- **Backbones.** DeepSeek-V3.2 and Qwen3-32B; web-search via Bocha.

## Results & Highlights

- **DeepResearch Bench (Qwen3-32B):** ScaffoldAgent reaches **RACE Overall 44.70** (+2.24 over best baseline IRCoT 42.46), with the highest factual grounding — **Eff.c. 30.42** and **C.acc. 54.32** (vs ReAct C.acc. 23.78, IRCoT 40.04). Multi-agent WebWeaver is competitive on RACE (41.28) but trails by 13.42 points on C.acc.
- **DeepResearch Bench (DeepSeek-V3.2):** ScaffoldAgent **RACE Overall 48.27** (+1.45 over StackPlanner 46.82), ranking first on **all five RACE sub-dimensions**, with **Eff.c. 51.18** and **C.acc. 62.20**. Notably it *improves monotonically on every metric* when moving to the stronger backbone (RACE 44.70→48.27, Eff.c. 30.42→51.18, C.acc. 54.32→62.20), while baselines swing (IRCoT drops, StackPlanner rises) — suggesting the framework scales with backbone capability rather than overfitting.
- **DeepResearch Gym (Qwen3-32B):** highest average **75.83** (+1.78 over EDR 74.05), leading Depth (82.40), Breadth (85.20), Insight (68.80), while staying competitive on Supportability (79.80 vs EDR's 89.50).
- **Action ablation (Table 3):** removing Contraction → RACE 48.27→44.72; removing Revision → lowest single-action RACE (43.09); Expansion-only → 41.73 and *non-terminating inflation* until budget exhausted, confirming Contraction+Revision are load-bearing.
- **Utility ablation (Table 4):** removing `U_ret` collapses Eff.c. 51.18→39.42; removing `U_str` drops RACE 48.27→44.17; removing `U_gen` is most damaging (RACE 43.58, C.acc. 50.82). Net: `U_ret` supplies evidence, `U_str` regularizes scaffold, `U_gen` aligns with downstream writing.
- **Action distribution (Fig. 3):** over 525 operations — Expansion 59.0%, Revision 22.9%, Contraction 18.1%; Expansion dominates early budget, then Contraction/Revision take over for merging/refresh.
- **Multi-turn follow-ups (Table 5, 100 instances):** ScaffoldAgent Tot. **72.60** vs ReAct-FW 57.76 and ReAct-GR 59.70, via localized subtree Revision/Expansion that leaves unrelated sections intact (non-destructive refinement).

## Authors & Credibility

- Multi-author team from **Peking University** (National Engineering Research Center of Software Engineering; School of CS; Key Laboratory of High Confidence Software Technologies; Center on Frontiers of Computing Studies; PKU IT Institute Tianjin Binhai) plus **GRG Banking Equipment Co., Ltd.** Five equal-contribution first authors; three corresponding PIs (`†`: Xu Chu, Junfeng Zhao, Yasha Wang).
- Evaluated against a broad, well-chosen baseline suite spanning three paradigms: **Naive (Prompt, RAG)**, **Single-Agent (ReAct, IRCoT, WebShaper)**, and **Multi-Agent (STORM, WebWeaver, EDR, StackPlanner)** — covering essentially all recent OEDR baselines one would expect.
- Reports results on **two independent benchmarks** (DeepResearch Bench, DeepResearch Gym) and **two open-source backbones**, which strengthens cross-setting robustness claims. The monotonic scaling across backbones is a genuinely interesting and credible finding.
- Grounding claims are backed by FACT metrics (Eff.c., C.acc.), not just prose-quality metrics — important for a report-generation paper where hallucination is the main risk.

## Critical Assessment

- **Novelty: Moderate-to-High.** The core ideas are not brand-new in isolation: dynamic/evolving outlines exist (DOME, WriteHERE, WebWeaver, SurveyGen-I), AGM belief revision is a 1985 classical theory, UCB node selection is borrowed from MCTS, and composite-utility scoring of retrieval is standard. The genuine novelty is the *synthesis*: a unified utility signal that (a) is computed from a downstream trial write, (b) drives both *which* node and *which* operation, and (c) controls termination. Calling the operations "AGM-inspired" is somewhat cosmetic — they are used "as practical structural actions rather than formal logical operators" per the authors' own admission, so the AGM framing is more branding than formal contribution.
- **Rigor: Solid but with gaps.** Ablations are commendably thorough (per-action and per-utility-term). However: (i) utility weights `λ1,λ2,λ3`, `w_*`, `α,β,γ` and termination thresholds `ε, k` are not reported in the main text — sensitivity analysis is missing, leaving open whether the framework is robust to these hyperparameters or hand-tuned per benchmark. (ii) C.acc. for IRCoT jumps from 40.04 (Qwen) to 75.18 (DeepSeek) while its RACE drops — an odd inversion that is not explained. (iii) WebWeaver's C.acc. on DeepSeek is 88.35, *higher* than ScaffoldAgent's 62.20 — the authors do not discuss why their method loses on citation accuracy to a baseline it beats on RACE, which is a notable and unaddressed tension.
- **Reproducibility: Medium.** Code/data release is not clearly promised in the visible text. Bocha web-search API and two open backbones are specified, but the outline-evolution policy is utility-only with many unreported weights, so an independent reimplementation would have to guess. Multi-turn protocol details are deferred to appendix.
- **Significance: Moderate.** The gains over the best baseline are real but modest in absolute terms (+1.45 to +2.24 RACE Overall, +1.78 Gym average). The strongest claim — monotonic scaling with backbone strength — is the most interesting and would deserve more than two data points.
- **Red flags:**
  - Multi-turn evaluation is admittedly "preliminary" (100 follow-ups, self-defined protocol vs two weak ReAct variants only — no comparison to WebWeaver/EDR/StackPlanner multi-turn).
  - Cost/latency analysis is absent: ScaffoldAgent runs a Reporter Agent trial-write on *every* node-level operation, which is expensive; without token/cost numbers the practicality vs cheaper baselines is unclear.
  - No human evaluation reported; all metrics are automated (RACE/FACT/LLM-judge Gym dims), so "report quality" improvements rest entirely on LLM-as-judge reliability.
  - WebShaper returns "–" on FACT for both backbones and several cells are "unavailable/pending," suggesting incomplete evaluation.

## Strengths

- Unified, interpretable objective that explicitly couples the three things that actually matter in OEDR (retrieval, structure, generation) — most prior work optimizes only one or uses local heuristics.
- The Expansion/Contraction/Revision taxonomy is clean, maps to concrete failure modes, and the ablations confirm each is load-bearing (especially the non-terminating-inflation failure of Expansion-only).
- Per-node utility statistics + UCB selection give a principled explore/exploit trade-off over the outline tree, a genuinely nicer formulation than "update outline from newly retrieved docs."
- Strong and consistent empirical results across two benchmarks and two backbones, with the best factual-grounding numbers on Qwen3-32B.
- Non-destructive multi-turn refinement via localized Revision/Expansion is a practically appealing property for real interactive research assistants.

## Limitations

- Many hyperparameters (`λ`, `w`, `α/β/γ`, `ε`, `k`, exploration coefficient `c`) are not reported or sensitivity-analyzed in the main paper.
- Utility estimation requires a Reporter Agent trial-write per operation → high compute cost, unquantified.
- Evaluation is automated-only; no human study, and LLM-judge bias (especially self-preference for the same backbone family) is a real concern for both RACE and Gym scores.
- Multi-turn protocol is preliminary and only compared against weak ReAct variants; longer dialogue trajectories not tested.
- Uses open-source backbones + a single search API (Bocha); authors themselves note scaling to frontier models / better search is left to future work.
- WebWeaver beats ScaffoldAgent on DeepSeek C.acc. (88.35 vs 62.20) — an unexplained weakness on a key grounding metric.
- No explicit *learned* outline-evolution policy (all utility-based inference-time control); a learned policy could plausibly do better and is acknowledged as future work.

## Relevance

与 LLM 训练 / 推理 / 可解释性 / Agent / 可靠性几个方向都强相关，但最直接落在 **Agent** 与 **可靠性** 上：

- **Agent / 智能体编排**：给出了一种可复用的"中央控制器 + 子智能体调度 + 显式操作空间"范式，Outline Agent 调度 Search/Reporter 的方式可迁移到其它需要"规划-检索-生成"闭环的 Agent 系统（代码 Agent、数据分析 Agent、综述 Agent）。UCB 节点选择 + 边际效用终止也是 Agent 长程任务中"何时探索、何时停止"这一普遍难题的一个具体解。
- **可靠性 / 事实可溯源**：把"试写质量"（引用支持 ρ_sup、覆盖 ρ_cov）直接纳入效用并用于推理时控制，是把"减少幻觉"从后处理纠错前移到结构演化阶段的思路，对需要高事实保真度的长篇生成（综述、尽调、研报）有直接价值。
- **推理时（inference-time）控制**：与 test-time scaling、inference-time search（如 tree-of-thoughts、MCTS for LLM）同属一类——用搜索/规划结构换取更优输出，而非改训练。UCB + 效用信号是这条线上一个干净的实例。
- **可解释性**：大纲树本身就是可读的中间产物，每个节点的效用统计 θ_v 让"为什么这么组织、为什么停在这里"可审计，比黑箱 ReAct 轨迹更易调试。
- **对训练侧**关联较弱：本文不改模型权重，纯推理时框架；但其"效用信号"可视为一种弱监督，未来若用这些轨迹做 policy learning/SFT 则与训练直接挂钩（作者也指出了这一方向）。

## Questions for Further Exploration

- 各项权重（`λ1,λ2,λ3`、`w_*`、`α,β,γ`、`c`、`ε`、`k`）是如何设定的？对最终指标有多敏感？是否存在 per-benchmark 调参？
- 为什么在 DeepSeek-V3.2 上 WebWeaver 的 C.acc.（88.35）显著高于 ScaffoldAgent（62.20）？ScaffoldAgent 在"引用准确率"上是否在用 RACE 换 FACT？能否同时拿到两者？
- 每次节点操作都触发 Reporter 试写，端到端的 token 消耗 / 延迟 / 美元成本是多少？相对 STORM、ReAct 的成本-质量曲线如何？
- 效用信号完全来自 LLM-as-judge（RACE/FACT/Gym 维度），是否存在 judge 与生成器同源导致的自偏好？换一个独立 judge（或人类标注）后排序是否稳健？
- UCB 式节点选择与终止阈值 ε 在更深 / 更广的大纲树上是否仍然有效？大纲规模上限是多少？
- 把当前"纯效用推理时控制"升级为"学习一个 outline-evolution policy"（用这些轨迹做 SFT/RL）能否带来进一步提升？是否会牺牲可解释性？
- 多轮 follow-up 协议能否扩展到 WebWeaver/EDR/StackPlanner 等更强基线？在多轮长对话中"非破坏式局部更新"的优势是否依然成立？
- Expansion/Contraction/Revision 三操作是否完备？是否需要第四类操作（如跨子树的 Reordering / 主题迁移）来处理需要全局重排的场景？
