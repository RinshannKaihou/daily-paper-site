# Bit-by-Bit: Progressive QAT Strategy with Outlier Channel Splitting for Stable Low-Bit LLMs

> **TL;DR:** Bit-by-Bit is a progressive QAT framework for ultra-low-bit LLMs combining block-wise precision annealing (8вҶ’4вҶ’2-bit), nested integer grids for "train once, deploy any precision," and rounding-aware outlier channel splitting (OCS). On WikiText2 with LLaMA-2 7B under W2A2 it reaches 7.72 PPL versus FP16's 5.47 (+2.25), beating BitDistiller (29.66) and EfficientQAT (9.71); on LLaMA-3.2 3B W2A2 zero-shot average it scores 51.52 vs the strongest baseline BitDistiller's 46.28 (+5.24 points), while requiring 3600Г— fewer training tokens than ParetoQ.

## Key Contributions

- A block-wise progressive QAT strategy that anneals precision stage-by-stage (weights 8вҶ’4вҶ’2-bit first, then activations), using higher-precision phases to stabilize initialization for ultra-low-bit optimization and suppress the loss spikes seen in direct low-bit QAT.
- A nested integer-quantization grid that exploits the bit-shift nesting of low-bit codes inside high-bit codes, enabling a "train once, deploy any precision" paradigm from a single set of master parameters without retraining or storing multiple checkpoints.
- Rounding-aware Outlier Channel Splitting (OCS): identifies outlier channels via the joint metric вҖ–xвҖ–вӮӮВ·max|w|, applies a depth-wise linearly increasing split ratio, and provably preserves the quantized output as an identity transform while halving the dynamic range вҖ” yielding a 4Г— MSE reduction over naive half-splitting.
- Microscaling groups with E4M3 FP8 scales (one 8-bit scale per 32 weights, 0.25 bits/weight overhead) aligned with OCP/NVIDIA standards, plus custom W2A2/W2A16 CUDA kernels achieving up to 11Г— speedup over BF16.

## Methodology

Bit-by-Bit is a progressive QAT framework evaluated on LLaMA-2/3 and Mistral families. The method integrates four components:

1. **Block-wise progressive training.** Observing that loss landscapes become increasingly rugged and discontinuous at lower precision (Fig. 1) and that direct 2-bit QAT produces prominent loss spikes (Fig. 2a) with error accumulating in deeper blocks (Fig. 2b), the authors anneal precision stage-by-stage. For weight-only quantization the schedule is w8a16 вҶ’ w4a16 вҶ’ w2a16 (switching every two epochs); for weightвҖ“activation it first lowers weights to w2a16, then progressively reduces activations w2a16 вҶ’ w2a8 вҶ’ w2a4 вҶ’ w2a2. Block-wise MSE loss uses the higher-bit block activation as teacher: MSE[(x_w(k+О”)a16 В· W_wka16) вҲ’ (x_w16a16 В· W_w16a16)]. They use an asymmetric integer quantizer with a zero-point and a straight-through estimator (STE); no learnable clipping is applied.

2. **Once-for-any-precision via nested bit-shifts.** Because a lower-bit quantizer is a coarser version of a higher-bit one sharing the same scale s, an l-bit code is obtained from an h-bit code by right-shifting: q^(l) = q^(h) вү« (hвҲ’l). A curriculum objective optimizes an expanding set of targets Bit = {(8); (8,4); (8,4,2)} with weights О»_b: L = ОЈ О»_b В· MSE(xВ·W_b, y).

3. **Rounding-aware OCS.** For a linear layer y = xW, an outlier channel m is split into two halved-magnitude branches: x_mВ·w_m = x_mВ·(w_m+s/2)/2 + x_mВ·(w_mвҲ’s/2)/2. Because rounding-aware offsets cancel, Q_s(W_m:^(1)) + Q_s(W_m:^(2)) = Q_s(W_m:), making the split an identity on quantized output while halving the dynamic range. Sensitive channels are ranked by metric_i = вҖ–X_iвҖ–вӮӮ В· max_j|W_ij|; the split ratio grows linearly with depth r_b = r_min + (bвҲ’1)/(BвҲ’1)В·(r_max вҲ’ r_min) to match observed depth-wise error accumulation. In practice 10% of channels are split.

4. **Microscaling format & custom kernels.** Per-group scaling over 32 elements with E4M3 FP8 group scales (instead of E8M0) provides sufficient mantissa precision for 2-bit step sizes at 0.25 bits/weight overhead. Custom W2A2 and W2A16 CUDA GEMV kernels were developed because modern GPUs lack native 2-bit support.

Training setup: a single H800 GPU; Bit-by-Bit trained on a 4096-sample RedPajama subset. Baselines (ParetoQ, EfficientQAT, BitDistiller) were aligned to 2-epoch budgets; PTQ baselines (GPTQ, AWQ, OmniQuant, SmoothQuant, SpinQuant) used 256-sample RedPajama calibration. Benchmarks: WikiText2, C4, and five zero-shot tasks (PIQA, ARC-Easy, ARC-Challenge, HellaSwag, Winogrande).

## Results & Highlights

- On LLaMA-2 7B under W2A2, Bit-by-Bit reaches **7.72 WikiText2 PPL** versus FP16's 5.47 (only +2.25 increase), while BitDistiller scores 29.66 and EfficientQAT scores 9.71 вҖ” i.e., roughly a 3.8Г— PPL reduction over BitDistiller at the same budget.
- On LLaMA-3.2 3B W2A2, zero-shot average accuracy is **51.52** versus the strongest baseline BitDistiller's 46.28 (+5.24 points); under W2A16 it reaches 56.91 versus BitDistiller's 56.18.
- Token efficiency: Bit-by-Bit achieves a **3600Г— reduction in token requirements** compared to ParetoQ (Fig. 2c), and ParetoQ's original recipe needs ~2T tokens (BitNet-style) for a stable low-bit model.
- On Mistral-7B w2a16 (once-for-any-precision), Bit-by-Bit obtains **10.73 C4 PPL / 65.37 task avg** from a single QAT run, versus MatQuant's 13.05 / 65.99 and OmniQuant's 50.99 / 59.74 (the latter requiring separate training per bit-width).
- Custom W2A2 kernels achieve **>10Г— speedup over native PyTorch FP16** in the (4096, 14336) GEMV setting, and up to 11Г— over BF16 overall; end-to-end decoding on LLaMA-3-8B reaches 76 tokens/s vs the Transformers baseline's 49 tokens/s (1.5Г—).
- Ablation on LLaMA-3.2-1B w2a16: without progressive training WikiText2 PPL is 31.88; adding progressive + OCS with the вҖ–xвҖ–вӮӮВ·max|w| metric brings it down to 17.07 (task avg 35.09 вҶ’ 45.18) at only 0.33GB вҶ’ 0.36GB memory overhead.
- W2A2 ablation: without progressive training the model fails to converge (~2Г—10вҒө PPL); progressive strategy alone brings WikiText2 PPL to 42.2; adding OCS with the xmax metric reaches 32.34.

## Critical Assessment

The paper's central claim that rounding-aware OCS is an identity transform on quantized output is elegant and formally justified (Appendix C: E[|Оө_RA|] = ВҪ E[|Оө_naive|], a 4Г— MSE reduction), but the empirical ablations confound OCS with the progressive schedule and the joint outlier metric, so the isolated contribution of OCS at fixed schedule is only partially separated (Table 3 shows OCS adds ~2 PPL on top of progressive training, but no row isolates OCS without the joint metric). A second weakness, acknowledged in the paper's own Limitations (A.3), is that the block-wise schedule is "less friendly to distributed training than end-to-end schemes, requiring nontrivial load-balancing and communication engineering," which limits scalability to very large models where block-wise reconstruction becomes communication-bound. Third, results on the Qwen family show "larger performance drops" that the authors defer to future work, leaving generalization beyond LLaMA/Mistral under-explored. Finally, the once-for-any-precision comparison against MatQuant is close rather than dominant (Mistral-7B w4a16: Bit-by-Bit 8.79 C4 PPL vs MatQuant 8.63), weakening the claim of a clear single-QAT advantage at intermediate bit-widths.

## Strengths

- Principled progressive schedule grounded in a concrete observation: low-bit grids are nested in high-bit grids (|x_low вҲ’ x_high| вүӨ ВҪ s_high), so coarse-to-fine adaptation is a natural optimization path.
- Strong empirical wins in the ultra-low W2A2 regime, where most baselines either fail to converge or lose an order of magnitude in PPL.
- Token efficiency (3600Г— fewer than ParetoQ) materially lowers the cost barrier to low-bit QAT.
- Custom CUDA kernels and end-to-end throughput numbers (76 tokens/s) make the deployment story concrete rather than purely academic.
- Honest, detailed appendix including negative results (Muon optimizer did not help, cyclic precision only marginally useful).

## Limitations

- Block-wise training is communication-unfriendly for distributed large-scale runs (acknowledged A.3).
- Larger performance degradation on Qwen models, left unanalyzed (A.3).
- Once-for-any-precision results are competitive but not strictly dominant at w4a16/w8a16 versus MatQuant.
- OCS increases layer width; although memory overhead is small (0.33GBвҶ’0.36GB), the kernel and engineering complexity of expanded weight matrices is non-trivial.
- End-to-end progressive training (without block-wise reconstruction) is unexplored; its convergence behavior is left open.
- The paper uses an LLM only for stylistic polishing (A.1) but does not release code or checkpoints in the PDF.

## Questions for Further Exploration

- Can the rounding-aware OCS identity property be preserved under non-uniform (e.g., SEQ/log-scaled) quantizers, or does it strictly require uniform integer grids?
- How does Bit-by-Bit behave on Mixture-of-Experts architectures where per-expert outlier distributions may differ вҖ” does the depth-wise split schedule still apply?
- Is the 3600Г— token-efficiency advantage over ParetoQ preserved when ParetoQ is given the same block-wise reconstruction objective, or is most of the gain from block-wise training rather than the progressive schedule itself?
- Could the E4M3 microscaling overhead (0.25 bits/weight) be reduced via shared scales across correlated groups without violating the nested-grid property?
- What causes the Qwen family's larger degradation вҖ” is it weight distribution kurtosis, activation outlier structure, or tokenizer-induced embedding outliers?
