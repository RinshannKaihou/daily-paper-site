Key finding

Verifiable Self-Evolution for Open-Ended Dialogue Skills via Future-Feedback Prediction

Chaojin Zhao · Xuan Jiang — ByteDance

dialogue skill evolution feedback prediction textual optimization LLM agents offline evaluation
Key Finding

Reframe optimization from generating answers to predicting whether an observed answer triggered resolved or unresolved feedback.

Key Metric
>75%
held-out accuracy of the evolved feedback skill on a balanced sales-assistant dataset.
+ ~25 pts vs. ~50% baseline
Why This Matters

Restores a fixed, verifiable target for textual skill optimization in open-ended dialogue, where logged feedback is otherwise a moving target.

Critical Edge

A single private dataset, no F1 / precision / CIs or head-to-head baselines — and 75% is observational accuracy, not counterfactual proof.

Methodology

The blocker: editing an answer A into A′ invalidates the logged next-turn feedback, since p(Y|X,A) ≠ p(Y|X,A′). So the authors flip the optimization target.

Step 01 · Problem

The moving-target trap

From logs D = {(X_i, A_i, Y_i)}, reusing observed feedback Q2 to validate a counterfactual answer A′ is biased — a fixed dataset cannot score arbitrary edits to the answer skill S_A.

Step 02 · Verifiable proxy

Predict the observed label

Optimize a feedback skill S_F that predicts Ŷ from (C, H, Q1, A). Held-out accuracy J(S_F) is measurable without re-interaction and supports asymmetric misclassification costs.

Step 03 · Data

Label & balance

Map explicit feedback, accept/reject signals, and product-side events to binary resolved/unresolved; drop silent terminations; quality-filter and partition to a ~1:1 split so majority-class accuracy cannot mask learning.

Step 04 · Evolution

Validation-gated edits

SkillOpt-style loop: predictor labels each sample with a rationale; optimizer proposes bounded textual edits Δ⁽ᵗ⁾; a candidate is accepted only if J improves, else reverts. Lexicographic variants tie-break on business utility.

Step 05 · Duality

From rationale to answer guidance

Learned failure criteria — tool success ≠ user adoption; "OK" ≠ acceptance; generic suggestion ≠ actionable plan; each suggestion needs a concrete action + operating anchor — double as interpretable diagnostics for the answer skill. The paper frames this as a supported bridge, not a measured end-to-end improvement.

Strengths & Limitations

Strengths

  • Sharp formalization: one line — p(Y|X,A) ≠ p(Y|X,A′) — names the core obstacle, with Figures 1–2 contrasting the two verification regimes.
  • Simple and deployable: frozen model + textual skill + bounded edits + held-out gating. No new parameters to train; cheap and auditable.
  • “Feedback–generation duality” is reusable: the predictor's failure criteria become a constraints checklist for the answer skill.
  • Disciplined claim boundary: separates observational verification from counterfactual validity, and positions the method as an offline stage, not a substitute for online evaluation.
  • Learned rules carry business value (tool-success ≠ adoption; weak acknowledgment ≠ acceptance) for customer / sales assistants.

Limitations

  • Counterfactual drift: optimizing S_A could yield OOD responses that exploit the predictor — it remains a verified observational model + diagnostic oracle.
  • Silent-user bias: interactions without legible follow-up are dropped; label covers observable resolution better than total satisfaction.
  • Reproducibility: strongest results rely on private cleaning rules and private production data; cross-domain test on public USS is left as future work.
  • Binary simplification: resolved/unresolved cannot express graded or personalized satisfaction.
  • No end-to-end answer-skill experiment: rationale → guidance migration is a supported direction, not a measured causal result.
  • Coarse evaluation: no exact accuracy, CIs, per-class metrics, or same-data baseline comparison.

Questions for Further Exploration

  1. Does >75% accuracy and cross-domain generalization hold on public data such as USS or SPUR's labeled sets?
  2. Can a one-shot counterfactual human study confirm end-to-end S_A improvement after compiling answer-skill edits from feedback rationales — and by how much?
  3. When an optimized answer skill produces OOD responses to game the predictor, can adversarial training, uncertainty calibration, or OOD detection mitigate it?
  4. Can the binary resolved/unresolved label extend to graded or multi-dimensional satisfaction (“resolved but poor experience”)?
  5. How much does the asymmetric-cost lexicographic variant cut false positives in practice — and does it shift final human evaluation?
  6. If silent-termination users are labeled via weak/semi-supervision, how do the learned rules and accuracy change?
  7. Compared with a direct LLM-judge, how much closer is “predict observed feedback” to the target users' real behavioral preferences?
