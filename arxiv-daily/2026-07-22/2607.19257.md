> **一句话总结：** 这篇论文在自建的、无污染的合成语料"Book of Veyra"（8,780 个虚构实体）上，把提示词格式（markdown / plain / prose / table）、指令条数（10–160）和上下文长度（2k–512k token）三个因素交叉实验，发现：指令遵循在 N≈80 条规则时无论哪种格式都跌到零；长上下文下格式优势与模型自身有效上下文上限挂钩、无普适赢家；且在贴近上下文上限时真正上升的失败模式是"拒绝回答"而非幻觉或谄媚——而 markdown 并没有人们以为的那种稳定优势。

> **TL;DR:** On a contamination-free synthetic corpus ("Book of Veyra", 8,780 fictional entities), the paper crosses prompt format (markdown/plain/prose/table) with instruction count (10–160) and context length (2k–512k) across 5 models (Claude Sonnet 5, Claude Haiku, Gemini Flash, Qwen 27B/35B). Perfect instruction-following collapses to zero by N=80 regardless of format, recall accuracy degrades sharply past each model's own effective context ceiling with no universal format winner (Claude Haiku shows a 48.4pp format spread at 128k; plain wins for Sonnet 5 @ 512k), fabrication is exactly 0/5,760 and sycophancy stays ≤8.3%, while outright refusal to answer rises to 79–90% near each model's ceiling.

## 核心贡献

1. **指令遵循的"衰减×格式×位置"联合分析**：首次在同一个指令池上把指令条数 N 的衰减曲线与"四种格式"和"系统提示 vs 用户轮次两种放置位置"交叉。位置效应在 4/5 模型上量级与格式效应相当甚至更大，且方向是模型特异的（对两个模型有利、对两个有害、一个无差异）。
2. **长上下文幻觉的四格式×六档×三失败模式因子实验**：在一个无污染语料上同时测量召回准确率、虚假前提谄媚、缺失事实捏造，跨越 2k–512k token 阶梯，作者认为此前没有研究同时把格式与这三类失败模式交叉。
3. **结构 vs 语法分解**：通过让 markdown（列表结构）、table（表格结构）、plain（无结构）、prose（无结构散文）保持内容字节一致，区分"任何结构是否有用"与"表格结构是否特别有用"。
4. **token 开销调整后的准确率**：用真实子词 token 数证明格式带来的准确率优势不总能扛住它的 token 成本（markdown 1.258×、prose 1.221×、table 1.367× 于 plain）。
5. **VeyraBench 开源**：完整基准测试脚本（benchmark.py）、Book of Veyra 语料生成器、全部原始模型输出，从固定 seed（42）可字节级复现（MIT / CC-BY-4.0）。

## Key Contributions

1. **Joint decay-and-placement analysis of instruction-following** — crosses the adherence-vs-N curve against format and against system-prompt vs user-turn placement, neither of which prior work (Jaroslawicz 2025, Harada 2025) varies; placement effects rival format in magnitude but with model-specific sign.
2. **Four-format × six-rung × three-failure-mode factorial for long-context hallucination** — recall, sycophancy, and fabrication measured jointly on a 2k–512k ladder on one contamination-free corpus.
3. **Structure-vs-syntax decomposition** — a byte-identical four-way contrast (list-markdown / table / plain / prose) isolates "any structure helps" from "tables specifically help", crossed with context length (going beyond Oh 2024 and Liu 2025).
4. **Token-overhead-adjusted accuracy** — uses real subword token counts to show format's raw-accuracy wins do not always survive their 22–37% token overhead.
5. **VeyraBench release** — full harness, corpus generator, and raw results, byte-identically reproducible from a fixed seed.

## Methodology

**语料——Book of Veyra**：完全合成的虚构宇宙，8,780 个全局唯一命名实体（星系/公会/生物三类），每类有固定 schema（如星系有恒星名、发现者、建立年、直径）。用确定性的模板+随机表流程从 seed=42 生成，全程无 LLM 参与，重复生成 MD5 一致，从而在构造层面保证无预训练污染（实体名不可能出现在任何模型预训练集中）。属性值允许跨实体重叠以制造真实干扰，但被记分的探针只问不可猜测的事实（chance baseline ≈ 0）。

**四种渲染**（内容字节一致）：Markdown（子标题+加粗键值项）、Plain（去掉所有 markdown 标记）、Prose（改写为流畅散文、无换行）、Table（markdown 键值表）。token 开销比相对 plain 稳定：markdown 1.258×、prose 1.221×、table 1.367×。

**Experiment 1（指令遵循衰减）**：N ∈ {10,20,40,80,120,160} 条程序可验证规则（5 条固定结构性规则 + N−5 条 forbid/include 词约束），用 regex/词边界匹配判分，无 LLM-as-judge。每 (model, N, format, placement) 单元 20 次试验，每模型 960 次调用、共 4,800 次记分试验。

**Experiment 2（上下文阶梯上的幻觉）**：把语料切成 2k/16k/64k/128k/256k/512k 六档，每档问 recall / false-premise / absent 三类探针，各在 10%/50%/90% 三种针深度、每类 20 题 × 3 次重复。两个长上下文模型（Sonnet 5、Gemini Flash）跑到 512k，其余三模型跑到各自有效上下文上限。还设了一个固定的 20 题 anchor set（从 2k 抽一次后在所有更大档位原样重问）用于做真正同构的跨档比较。每长上下文模型 5,520 次调用，全部 5 模型共 30,480 次记分响应。

**模型**：Claude Sonnet 5（claude-sonnet-5）、Claude Haiku（claude-haiku-4-5）、Gemini Flash（gemini-3.5-flash）、Qwen 27B、Qwen 35B（Ollama 自托管，推理/扩展思考在 API 层关闭，各自默认温度）。

## Results & Highlights

- **指令遵循地板效应（Experiment 1）**：所有 5 个模型、所有 4 种格式、两种放置位置，perfect-response rate 在 **N=80 时全部归零**并保持到 N=160（表 4：例如 Sonnet 5 从 N=10 的 0.938 跌到 N=80 的 0.000，Gemini Flash 从 0.919 跌到 0.019）。这是一个与格式无关的硬地板，而非渐进渐近。
- **无稳定的 markdown 优势**：markdown − plain 的遵循差值在 4/5 模型上很小（≤2.1pp）且符号不稳定；唯一清晰的定向信号是 **Qwen 35B 在 6 个 N 中的 5 个偏好 plain text**，N=160 时差距扩大到 4.8pp，与"markdown 更好"的常识相反。
- **位置效应与格式量级相当但方向模型特异**：在 N=160，移动相同规则块在 system vs user turn 之间的改变：Claude Haiku +6.6pp、Qwen 35B +5.1pp（user 更好），Gemini Flash −8.7pp、Qwen 27B −1.8pp（system 更好），Sonnet 5 不显著。
- **长上下文召回在 64–128k 后陡降且格式依赖（Experiment 2）**：2k/16k/64k 全部贴近上限（0.98–1.00）。**Claude Haiku @ 128k 的格式差距达 48.4pp**（plain 仅 0.383，其余三格式 0.817–0.867），是全数据集中最大单格差距。格式差距大小**追踪模型自身有效上下文上限而非绝对 token 数**：Sonnet 5 差距从 256k 的 11.7pp 涨到 512k 的 20.0pp，而 Gemini Flash 在同样名义区间基本持平（5.0→6.7pp），尽管两者宣称上限都是 1M token。
- **两个干净的零结果**：在全部 5,760 条 absent-fact 探针响应中**捏造率恰好为 0**（0/5,760，跨所有模型/档位/格式）；直接谄媚同意虚假前提的最高单格为 8.3%（Qwen 35B @ 128k/markdown），多数单元 ≤3%。
- **真正上升的是拒绝，不是谄媚**：false-premise 探针的拒绝率从 2k 的 0% 升到贴近上限时的高位——Claude Haiku 升至 89.6%（128k），Sonnet 5 升至 78.8%（512k）——与同一区间内保持低平的谄媚形成对比。
- **两个预注册的格式序都不成立**：prose（两个假设都预测最差的格式）反而在 5 个典型格的 3 个里是最佳或并列最佳；plain 在 Claude Haiku @ 128k 最差，却在 Sonnet 5 @ 512k 单独最佳。
- **成本调整后排名会反转**：Claude Haiku @ 128k 上 table 与 prose 原始准确率并列 0.867，但算入 token 成本后 prose（0.710）胜过 table（0.634）。

## Authors & Credibility

- **作者**：Netanel Eliav，单作者论文，隶属 Machine Human Intelligence Lab (MHIL)，公开邮箱与个人主页。
- **发表状态**：明确标注为 arXiv 预印本（cs.AI 主类，cs.CL 交叉），**"This paper has not undergone peer review"**。日期 2026 年 7 月。
- **可信度信号（强）**：(1) 释放完整 harness + 语料生成器 + 全部原始输出（github.com/iNetanel/veyrabench，约 40MB），声称从 seed 字节级复现并经 MD5 校验；(2) 探针均为程序可验证（regex / 词边界匹配），规避了 LLM-as-judge 的主观性；(3) 预注册了两个格式假设并诚实报告"两个都不成立"；(4) 设置 anchor set 专门用于排除跨档比较的题目组成伪影，并据此否决了一个看似真实的 main-set 改善；(5) 用 Wilson 95% 置信区间报告所有比较性主张。
- **可信度注意点**：单作者、非同行评审、5 个模型样本量有限、尾部单元样本量较小（Experiment 2 每单元 60 题）；具体阈值（N=80、89.6%）作者自己声明是受测模型的属性、不应直接外推到其他模型。

## Critical Assessment

论文最大的方法学价值在于实验设计的**正交性**：在同一个无污染、字节一致可重渲染的语料上同时操控格式 × 指令数 × 上下文长度，且明确把 system vs user turn 放置作为单独因子——后者是大多数格式敏感性研究（Sclar 2024、He 2024）从未变动过的因子，而本文证明它的量级常常压过格式本身（如 Gemini Flash 上 placement −8.7pp vs markdown−plain 多数 <2pp）。

结论的诚实性值得称道：作者预注册了"任何结构都有用"和"表格特别有用"两个假设，并在 Section 5.3 / 6 中报告两者均不成立（prose 反而常常最佳），还用 anchor set 主动否决了一个看起来漂亮但实为题目组成伪影的 Gemini Flash 256k→512k 改善——这种自我纠错在预印本里并不多见。

需要带着保留解读的地方：(1) **样本量在尾部偏小**——Experiment 2 每单元仅 60 题（20 题×3 重复），48.4pp 这类大差距单格的统计稳定性依赖少量题目，虽然作者用 Wilson CI 和 anchor set 做了缓冲，但 Claude Haiku @ 128k/plain 的 0.383 仍可能受个别难题驱动。(2) **模型覆盖有限**——5 个模型、3 个家族、单作者预算，作者自己也承认"无格式普适赢家"是定性结论、具体阈值不可外推。(3) **Gemini Flash 在 Experiment 1 中 prose/table 的崩塌**（N≥40 时 prose−plain 达 −18.1pp）经人工检查发现是"hidden-reasoning-leakage"（可见响应里漏出内部规则校验片段），这是一个真实但模型特异的失败模式，被作者如实披露但未做 token/attention 层级诊断。(4) token 计数用统一的 o200k_base 而非各厂商内部 tokenizer，相对开销比可信但绝对账单 token 可能不符。

## Strengths

- **正交且受控的实验设计**：单语料、字节一致四渲染、三因子交叉，使任何测量差异都可归因于格式或规模本身。
- **无污染语料的构造性论证**：实体名按构造不可能出现在预训练集中，比"去重过滤"更强；从固定 seed 字节级可复现。
- **揭示被忽视的真实失败模式**：在贴近上下文上限时把社区对"幻觉/谄媚上升"的默认担忧，纠正为"拒绝上升"——Claude Haiku 拒绝率 0%→89.6%、Sonnet 5→78.8%——这是对监控系统有直接工程含义的发现。
- **成本调整分析**：把"准确率/响应"与"准确率/token"区分开，证明 markdown/table 的 22–37% token 开销足以反转某些排名。
- **诚实报告零结果与负面结果**：5,760 个 absent 探针捏造率恰为 0、两个预注册假设均不成立、主动用 anchor set 否决伪改善。
- **完全开源且可复现**：harness + 生成器 + 原始输出 + MD5 校验。

## Limitations

- **语料单一领域**：仅虚构实体-属性类事实描述，未测代码、法律文本、数值业务数据等结构承载不同信息的领域。
- **模型覆盖小**：5 个模型、3 个家族，非穷尽也非随机抽样；定性形状（硬地板、无格式普适赢家、贴近上限时拒绝上升）可移植，但 N=80、89.6% 等具体阈值是受测模型属性。
- **尾部样本量小**：Experiment 1 每 (model,N,format,placement) 单元 20 次，Experiment 2 每 (model,rung,format) 单元 60 题；尽管有 Wilson CI 和 anchor set，大差距单格（如 48.4pp）的稳定性有限。
- **token 计数非厂商原生**：用 o200k_base 作跨厂商代理，与各厂商实际计费/上下文窗口 token 可能不符（虽然相对开销比预期稳定）。
- **规则/事实均程序可验证**：设计与"无 LLM-as-judge 的精确可复现判分"绑定，但真实世界指令往往更软、不可用精确字符串/词边界校验，衰减与格式模式是否迁移未测。
- **Gemini Flash 的 reasoning 泄漏未做根因诊断**：仅行为层面记录，未在 token/attention 层级确认它是该模型 reasoning-trace 处理的窄伪影还是会泛化的失败模式。
- **拒绝 vs 真正验证缺失未分离**：absent 探针的"正确（不回答）"无法区分"模型真验证了事实缺失"与"模型在该档位反射性拒绝一切"，作者承认这一点（Section 5.5）但称它"限定而不推翻捏造零结果本身"。

## Relevance

- **对提示词工程实践者（最重要受众）**：直接挑战"默认用 markdown"的行业习惯——本文没有一个模型给出稳定的 markdown 优势，Qwen 35B 反而稳定偏好 plain。建议"对自己模型做 A/B 测试，别套用默认"。
- **对 RAG / 长上下文系统设计者**：监控贴近上下文上限的失败时，应监控"拒绝回答"而非只盯错误或谄媚——拒绝才是真正上升的失败模式（89.6%）。召回在 64–128k 之前基本贴顶，之后陡降且差距追踪模型**自身有效上限**而非名义 token 数，提醒不要按厂商宣称窗口线性外推。
- **对系统提示编写者**：把 N≈40 当"重设计点"而非"调优点"——N≥80 时任何格式重排都救不回来，应拆分到多轮/工具/校验 pass。
- **对幻觉基准研究者**：提供一个无污染、四格式平行、可字节复现的合成语料范式（VeyraBench），并把"虚假前提拒绝率"作为与捏造、谄媚并列的第三种失败模式纳入评估。
- **对本仓库（daily_paper）主题**：契合 prompt-engineering、long-context、LLM 评估、合成基准四个标签，是 2026 年 7 月对"提示词格式 × 规模"这个长期争论的一手受控证据补充。

## Questions for Further Exploration

1. 指令数地板（N≈80）是"单提示密度"属性还是"总任务复杂度"属性？把同样规则集拆到多轮 / 校验 pass / 工具调用后，遵循率能否恢复到 N=80 以上？
2. 位置效应的方向（system 有利 vs user 有利）是否与某个可测量模型属性（参数量、训练范式、厂商）相关？本文 5 模型预算无法分辨。
3. 在贴近上下文上限时观察到的"拒绝悬崖"，在 base 模型 vs 其指令微调对应版本上是否消失？若消失则支持"对齐训练把幻觉风险换成了拒绝风险"的解释（作者在 Future Work 提出但未测）。
4. 把格式轴扩展到 JSON / YAML（He 2024 视为单独格式类）与两个规模轴交叉，机器可读格式是否表现出与人类可读四格式不同的衰减/召回曲线？
5. Gemini Flash 的 prose/table 崩塌（reasoning 泄漏）根因是什么？是该模型 reasoning-trace 处理的窄伪影，还是会在其他模型/条件下泛化的早期失败信号？
6. 召回准确率的"格式差距追踪模型自身有效上限"这一规律，在多文档（而非单文档嵌套前缀）检索场景下是否仍然成立？
7. absent 探针的"零捏造"是否部分来自"反射性拒绝"混淆？设计能区分"真验证事实缺失"与"一律拒绝"的探针变体，可澄清零捏造究竟是能力还是训练出来的回避。
