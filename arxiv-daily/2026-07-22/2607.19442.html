Unlearning as Distribution Restoration: A Controlled Counterfactual Study, a Validated Selective Screen, and the Limits of Oracle-Free Certification

Sen Yang·NYU Stern & Yuen-Hei Yeung·NYU Courant
Business-school ML × Mathematician pairing — formal identifiability theorem + painstaking empirical audit
Key Finding

Forget what "erasure" means. Unlearning is distribution restoration toward a matched retraining reference — and the field's go-to oracle criterion actively rewards the residual knowledge it claims to detect.

Key Metric
−2.82 nats
Candidates the trained-probe criterion rates "adequate" sit this far below the never-learned level — cluster-CI [−3.16, −2.48]. The criterion rewards residual paraphrase-recoverable knowledge.
Validated Screen
44/45
Base-anchored Mann-Whitney rank test accepts the retraining reference in 44/45 cells and rejects the injected model in 45/45 — a necessary, oracle-free filter.
Why This Matters
The field's de-facto unlearning evaluation is audited on its own merits — and fails its own reference model. Every evaluation pipeline needs held-out paraphrase anchoring.
Critical Edge
Only m=4 forget facts and 3 redraws underpin the tolerance — and 29 of 38 certifications come from the noisiest single family (Qwen, δf = 2.92), so the certified count is fragile under more redraws.

1Methodology

From base model M0, continued pretraining co-acquires a forget set F and a retain set R of dose-matched nonce facts (|F|=|R|=4), yielding Minj. The matched retraining reference Moracle replays the identical stream without F — a model that never saw F but shares retain competence.

Because bf16 training is nondeterministic, each reference draw is one realization. The paper quantifies indistinguishability tolerance δequiv = 0.93 nats at P90 (per-family 0.82–2.92) from three independent redraws. Co-acquiring R is essential: with only F, checkpoint-rollback (=M0) would trivially "forget" — but co-acquisition makes rollback destroy retain (NLL 6.6 vs oracle 2.4).

Restoration methods (KL-reversion, gate_proj mask, task-vector negation) are compared against suppression methods (GA+retain, NPO+retain) and checkpoint rollback, each at matched compute (~110–120 steps, full forget/retain Pareto fronts).

Models: Qwen2.5-0.5B/1.5B, SmolLM2-360M/1.7B, Pythia-410M/1.4B, GPT-2-medium/large, OLMo-2-1B. External validation on TOFU (Phi-1.5). All on one 48GB RTX 6000 Ada.

Oracle-Free Selector Pipeline

  1. Re-acquire each candidate C on F to convergence → reacquireF(C)
  2. Measure round-trip residual cert(C) = KL(reacquireF(C) ‖ Minj) on retain + wikitext
  3. Qualify via base-anchored under-forgetting rank test vs never-learned probes
  4. Select arg min over survivors — no oracle, gold posterior, or probe labels enter

Sealed Challenge Panel

Frozen audit template pool scored exactly once, blind: M0, Minj, reference, logit-suppression (−10), entity router, embedding corruption, and weight interpolations λ·Moracle + (1−λ)·Minj. A cryptographic hash is published with the paper.

3Strengths & Limitations

● Strengths

  • Self-auditing design. Running the reference itself through every proposed criterion exposes threshold mis-scaling without trusting any unlearning method.
  • Operational restoration-vs-suppression split. Restoration (KL-reversion ≈2–4 oracle-KL, task-vector) stays ~2× closer than suppression (GA/NPO ≈5–7) across 3 seeds and a 32-config fair sweep.
  • Formal foundations. Identifiability Theorem 1 + finite-sample Proposition 1 give empirical findings theoretical teeth — TOFU's failure is predicted, not retrofitted.
  • Honest scope throughout. "necessary not sufficient," "supporting comparison not head-to-head," "methods-as-produced not universal certificate," "abstention is selectivity by design."
  • Reproducibility. Open-weight checkpoints, frozen pre-registration, committed scripts, published audit-pool hash, fully specified hyperparameters.

● Limitations

  • Narrow threat model. Certification tests non-adaptive, methods-as-produced candidates; an adaptive bilevel adversary optimizing against the round-trip residual is untested; forward battery defeated in 12/45.
  • Retrospective, veto-less recalibration. Evaluated on stored candidates without stored relearning trajectories; the relearn-veto axis is absent — prospective run is future work.
  • Small forget set (m=4), few redraws (3). Per-family δf is itself uncertain; most certifications concentrate in the noisiest family. Coverage-calibrated tolerance would change counts.
  • Small models (≤1.7B; Phi-1.5 for TOFU). Scaling to larger instruction-tuned models is future work; nonce facts are synthetic by construction.
  • Oracle-free only for counterfactual facts. Inferable-fact benchmarks (TOFU) need the reference; impossibility result concerns oracle-free, forward-only, black-box protocols only.

4Questions for Further Exploration

Prospective certification with the relearn-veto axis. Would a prospective run with stored relearning trajectories and all four axes change the 15/45 count or the 0.80-vs-5.17 advantage?
Scaling to instruction-tuned 7B+ models. Does the restoration-vs-suppression dichotomy and the screen's 44/45 true-accept rate hold at scale, or does greater paraphrase-robustness shift which methods the held-out surface favors?
Coverage-calibrated tolerances from more redraws. How do the 38/820 certifications and their family concentration shift with 20–50 independent retraining draws and a data-derived confidence set?
Adaptive bilevel adversary. A candidate optimized directly against the round-trip residual is "the most natural attack on the construction" — how large is the gap from fixed-magnitude suppression (12/45)?
Beyond nonce facts: inferability-aware unlearning. Can the oracle-free Ĝ estimator partition a real forget set into counterfactual (certifiable) vs inferable (needs-reference) subsets, enabling hybrid protocols?
Second untouched held-out probe suite. A standalone selection claim needs a second suite with one candidate frozen per cell before scoring — does the 0.80-vs-5.17 advantage survive it?
Relationship to differential privacy. The paper disclaims any DP guarantee; could the base-anchored necessary-test framing combine with DP-style bounds into a hybrid "necessary + private" certificate?
