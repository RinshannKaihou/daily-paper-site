Anatomy of a Sound Neural Reasoner: One-Shot Amortization, First-Pass Poisoning, and Search Inertness in Clue-Rich Completion

Aleksey Komissarov · Neapolis University, Pafos, Cyprus · code & frozen protocol at github.com/ad3002/colt @ r14-2026-07-15

CoLT neural-symbolic-reasoning constraint-satisfaction sudoku amortized-inference cdcl-search frozen-protocol
Key Finding

Sound neural Sudoku reasoners are one-shot amortized predictors wrapped in an exact verifier — the added CDCL search layer cuts wasted work 1,497× but buys zero accuracy.

Key Finding

In clue-rich Sudoku, the “iterative deduction” loop is cosmetic — a single forward pass collapses the whole grid (singleton rate 1.000). Search is inert because nothing is left to branch on.

01

Methodology

CoLT = LDT envelope + 3 CDCL additions

CoLT keeps LDT's candidate lattice, asymmetric-BCE losses (w₊=4.0, w₋=0.5 so false eliminations cost 8× more — the soundness-preserving bias), pool training, and verify-or-abstain emission. It changes only who branches (a learned per-cell policy head πᵢ, supervised for free from the α-operator), what happens on conflict (chronological DFS + value exclusion + a cross-chain ban set of verified-wrong grids), and how structure enters (constraint-graph attention bias).

ŷ = x ⊓ α({ y ∈ Y : y consistent with x })

Deduction moves monotonically down the lattice; cells only ever lose candidates, and ⊥ = any state with an empty cell.

Frozen protocol & instruments

Datasets, budgets, baselines and success criteria were committed before any training at immutable Git tag r14-2026-07-15, with pinned-MD5 unique-solution Sudoku (4×4 / 6×6 / held-out 9×9 Phase-4 split).

A symmetry-exact leakage audit enumerates the full symmetry group per grid (3,359,232 transforms at 9×9): 9×9 splits are clean at every level; 6×6 admits no symmetry-disjoint split (≈50 essential grids).

One checkpoint (Graphormer bias + normalized coords + value-slot padding to Vmax=9) trains 4×4+6×6 and transfers zero-shot to 9×9. A claim-audit appendix maps every causal claim to committed raw JSON.

03

Strengths & Limitations

+ Strengths

  • Single causal chain tying every result: one-shot amortization → first-pass poisoning → symmetry calibration, confirmed by complementary train-time and test-time interventions on two boards.
  • Deliberately strong negative result. CoLT is built to falsify, not strawman, the “learned search buys accuracy” hypothesis; the 2×3 ablation + budget sweeps make the null hard to dismiss.
  • Exceptional reproducibility: frozen pre-training protocol, immutable Git tag, pinned-MD5 data, symmetry-exact leakage audit, clean-room verifier (25,231 cases, zero disagreements), three-seed + cross-host replication.
  • Intellectual honesty about failures: falsified frozen criteria (criterion 2: learned policy never beats MRV; criterion 5: global heads fail to transfer) are reported as failures.
  • Self-aware boundary study (§8) that sides with critics of physics-inspired GNN solvers and motivates a concrete classical/learned division of labor.

! Limitations

  • Gap is vs an augmentation-stripped LDT baseline, not the full published recipe — the honest framing is that the decomposition is the finding.
  • 6×6 admits no symmetry-disjoint split (≈50 essential grids); class-level generalization rests entirely on clean 9×9 splits.
  • Several cells are single-seed (the 2×2 factorial, transfer probes, boundary densities ≠ c=3.0); boundary training-seed variance is large (5–32%).
  • CDCL conflict analysis is structurally unavailable: the neural propagator is a black box; nogoods are limited to whole verified-wrong grids.
  • GRAM baseline is budget-starved 10–100× vs its published recipe, so the GRAM row carries no information about GRAM's ceiling.
  • Global heads don't transfer across board sizes (conflict AUC 0.50); most local-head transfer is just relearned constraint propagation.
04

Questions for Further Exploration

Can exact value-permutation equivariance remove the 9×9 wall without data augmentation — or does clue context still need to break the symmetry? (open in the revision protocol.)

Can global heads (conflict / policy) transfer across board sizes via constraint-graph pooling instead of a CLS token? Currently only the local propagator transfers.

Under what conditions does CDCL-style conflict analysis become usable — e.g. with an exact, interpretable propagator (lazy clause generation) that lifts nogoods to general clauses?

Which axis actually pushes the from-scratch CSP boundary? Supervision density (K=48 → 15.2%) wins; can exact-supervision design remove the sampling bottleneck?

Can a learned policy beat hand-coded MRV in a regime with long sound search chains that is not from-scratch? Strict criterion 2 currently fails (learned ≈ MRV).

How does it scale on multi-solution tasks (K≫1), e.g. Maze-style K in the hundreds? The K>1 α-operator path is only validated on small fixed graphs.
