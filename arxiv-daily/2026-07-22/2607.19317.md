> **一句话总结：** CircuitKIT 是一个 source-available 的机制可解释性工具库，用一个可序列化的 `CircuitScores` 制品把"电路发现-评估-干预"全流程串联起来，支持 13 种发现算法、6+1 个忠实度诊断支柱、7 个下游干预模块，并提供从结构化用户数据（CSV/JSONL/HuggingFace）自动构造对比任务的声明式模板接口，实测发现"电路的 patch 忠实度并不预测其剪枝效果"（ρ=-0.78）。

> **TL;DR:** CircuitKIT is a source-available Python library that unifies the mechanistic-interpretability pipeline (discover → evaluate → intervene) behind one typed, serializable `CircuitScores` artifact, shipping 13 circuit-discovery algorithms across 4 backends, 6+1 faithfulness pillars, and 7 downstream intervention modules, plus a declarative template interface that maps raw CSV/JSONL/HuggingFace data into discovery tasks. On Llama-3.2-3B-Instruct/BoolQ at 30% sparsity, patch faithfulness *anti-correlates* with pruning retention (Spearman ρ=-0.78, p=0.010), and circuit-guided selective fine-tuning does not beat a budget-matched random mask (mean Δ=-0.001), motivating the library's multi-pillar, extrinsic-evaluation design over single-score faithfulness.

## 核心贡献

1. **统一管线与共享制品**：提出一个 typed、JSON 可序列化的 `CircuitScores` 记录（含 task、model、algorithm、granularity、per-component scores、discovery config），让发现算法、评估指标、干预模块通过同一契约交互，免去"方法×指标×干预"两两组合的胶水代码。三种接口（stateful `Pipeline`、flat `ck.*` API、YAML/CLI）产出/消费同一制品。
2. **13 种发现算法，分稳定/研究两层**：6 个 stable tier（eap、eap-ig、eap-gp、acdc、ibcircuit、cdt）在 GPT-2/Llama/Gemma/Qwen 四大家族上测试过；7 个 research tier 仅在 GPT-2/IOI 验证。涵盖梯度归因（EAP 家族）、搜索式剪枝（ACDC）、信息瓶颈优化（IBCircuit）、上下文分解（CD-T）四类方法。
3. **声明式自定义数据路径**：用户只需对 CSV/JSONL/HF 数据集声明 clean/corrupt prompt 与 answer 模板，库自动做 token 长度对齐、判别标签预计算、可发现性预检（GREEN/YELLOW/RED 评级），并在无 counterfactual 时自动合成 10 种 corruption 策略或路由到 clean-only 算法（IBCircuit/CD-T）。
4. **6+1 忠实度支柱面板**：因单指标忠实度会随 ablation 方法变化甚至反转方法排名（Miller et al.），库报告 Causal Patching、Ablation Faithfulness、Stability、Robustness、Baselines、Generalization 六支柱 + 可选 Intervention Reliability（Pillar 7），faithfulness 比值带符号（负值=行为反转而非测量错误）。
5. **7 个下游干预模块 + 标准化基准**：结构化剪枝（LLM-Pruner）、混合精度量化（quanto/llmcompressor）、选择性微调（PEFT/LoRA）、知识编辑（ROME/MEMIT/CaKE）、激活引导（ActivationSteering）、CircuitLoRA 修复、幻觉探测；干预产出可导出为可重载 HuggingFace checkpoint，经 lm-evaluation-harness 打分。

## Key Contributions

- **Unified pipeline behind one artifact.** A typed, serializable `CircuitScores` record lets any discovery backend (13 algorithms, 4 families) feed any of 6+1 evaluation pillars and 7 application modules, eliminating per-pair glue code. Three interchangeable interfaces (stateful `Pipeline`, flat `ck.*` functional API, Click CLI + YAML) all drive the same artifact contract.
- **Stable/research tiering.** Six algorithms (eap, eap-ig, eap-gp, acdc, ibcircuit, cdt) are stable tier, cross-validated across GPT-2/Llama/Gemma/Qwen; seven more are research tier (GPT-2/IOI only), reachable through the same `@register` selector registry and promotable as evidence accrues.
- **Declarative custom-data path.** A CSV/JSONL/HuggingFace dataset is mapped to a discovery task by declaring prompt/answer templates; the library performs token-length alignment (filter/pad_question/none), label determinism, logit-difference, class-balance checks, and a worthiness pre-flight gate, plus 10 corruption-synthesis strategies (entity-swap, token-swap, role-swap, paraphrase, distractor, …) when only clean data exists, with clean-only routing to IBCircuit/CD-T.
- **Multi-pillar faithfulness panel.** Motivated by the finding that single faithfulness metrics can reverse method rankings, the library reports six pillars (Patching, Ablation, Stability, Robustness, Baselines, Generalization) plus an optional seventh (Intervention Reliability), with signed ratios that expose behavioral inversions instead of clamping them to zero.
- **Seven intervention modules + standardized benchmarking.** Pruning (LLM-Pruner), mixed-precision quantization (quanto/llmcompressor), selective fine-tuning (PEFT/LoRA static masks), knowledge editing (ROME/MEMIT + circuit-aware CaKE/MCircKE), activation steering, CircuitLoRA healing, and hallucination probing, all exporting reloadable HF checkpoints scored via `lm-evaluation-harness`.

## Methodology

- **共享制品设计**：`CircuitScores` 是 typed/serializable 记录，携带 task descriptor、model id、algorithm、granularity（node/neuron）、per-component attribution scores、discovery config。Discovery backends 生产它，evaluation（产出 `FaithfulnessReport`）和 application 模块消费它，遵循 HuggingFace Trainer 的"单对象拥有流程、合理默认值、各阶段可单独寻址"模式。
- **发现算法**：EAP 家族用单次 forward-backward 的一阶（可选 integrated-gradient）逼近每条边的 patch 效应；EAP-IG 沿 clean↔corrupt 直线路径积分（满足 IG 的 completeness/sensitivity 公理）；EAP-GP 用自适应几何路径（沿模型自身梯度下降构造路径）。ACDC 贪心剪枝（每条候选边一次 forward）。IBCircuit 学习 soft mask（`L=αL_task+βL_IB`），clean-only。CD-T 无梯度前向传播 (rel, irrel) 分解，clean-only。
- **node/neuron 双粒度**：node 级对整个 head/MLP 子层打一个分；neuron 级对每个 head-dim 通道和每个 MLP 神经元打分，直接对应干预模块操作的单元。MLP 钩点可选 `mlp_out`（默认，d_model 维）或 `post_act`（d_mlp 隐藏单元，更细粒）。
- **评估六支柱**：统一忠实度分数 `F(C)=(y_circuit-y_corrupt)/(y_clean-y_corrupt)`（=1 完全恢复，=0 corrupt 基线），带符号报告。P1 soft patching 补集；P2 hard ablation 补集（zero/mean/mean-positional）；P3 跨 resample 的 Jaccard/Spearman 稳定性；P4 跨 corruption 家族的鲁棒性；P5 vs random/weight-magnitude 基线 z-score；P6 跨任务迁移率（如 IOI→double_io，WMDP-Cyber→WMDP-Bio）。P7 用 seed consistency + effect magnitude + effect variance 的调和平均度量干预可靠性。
- **干预模块**：剪枝把 circuit 分数包成 LLM-Pruner 的 importance estimator；量化按 circuit 分数保留 top fraction 于原生精度其余降比特；选择性微调把 top-k% 解析为具体权重索引区间做 static gradient mask 或 LoRA target_modules；ROME/MEMIT 限制在 circuit-resident MLP 层，提供一次性缓存的 `get_covariance`。

## Results & Highlights

论文核心实证由 7 个研究（E1–E7）组成，其中几个反直觉发现是全文亮点：

1. **E1（GPT-2 Small / IOI, node 级, α=0.3）**：EAP 家族三算法 patching 恢复 P1=1.00 且与 canonical IOI 头集 Jaccard 0.43–0.50（recall 0.60–0.67）；但 **CD-T 虽然 P1=0.894/P2=1.000 双忠实，却与 canonical 头集零重叠**（把 15 个槽位中 12 个放在 MLP 上）。ACDC 在固定稀疏度协议下 P1=0.208/P2=0.039，代价 3322s（比 EAP 家族 66–140s 高 24–133×）。证明"行为恢复"与"标准重叠"会不一致——需报告 panel 而非单一数值。
2. **E2（跨 6 家族×2 任务，eap-ig）**：12 个 cell 全部完成且无代码改动，只换模型名。patching 恢复 P1≥0.91（6 cell 恰为 1.00），稳定性 J=0.80–0.92。但 **ablation 充分性 P2 从 0.24（Gemma-2-2B/IOI）到 1.00（Phi-2/IOI）剧烈分散**，而 patching 几乎不可区分——该列间分歧本身就是发现。
3. **E5（Llama-3.2-3B-Instruct / BoolQ, 30% 剪枝）**：circuit-guided 剪枝顶部有竞争力（IBCircuit accRet=0.986，与最强 Taylor 1.027 仅差 4.1pp），但 patch 忠实度与剪枝保留率 **显著负相关 Spearman ρ=-0.78（p=0.010, n=10）**：EAP-IG 忠实却 accRet 仅 0.571，而 CD-T（0.502）塌成退化解（WikiText-2 PPL=1406）。Magnitude/Wanda/random 全部塌到 0.501–0.519（PPL≥3000）。
4. **E6（混合精度量化, 3-bit/8-bit/5% fallback）**：与剪枝相反，量化保留率对 patch 忠实度**不敏感**（ρ=+0.23, p=0.55, n=9），但 ablation 忠实度**预测**保留率（ρ=+0.73, p=0.031）——因为 ablation 的零替换 counterfactual 更接近量化误差。
5. **E7（选择性微调, top-30% static mask, BoolQ）**：circuit 选择**不优于**随机 mask（最佳单 cell IBCircuit +3.0pp，16 cell 池化 mean Δ(circuit−random)=−0.001，明报为 null）。保护一般语言建模的是预算约束本身（冻结 70% 的模型），而非"哪 30% 被选"。
6. **E4（自定义越狱检测 CSV, Qwen2.5-1.5B-Instruct）**：paired 路径恢复 85% 的全模型拒绝 gap（patching 0.854，1.71× random），但 hard ablation 下拒绝指标反转到 −1.55（signed ratio −2.61），即"剥离周边计算后截断模型转而倾向顺从"。clean-only 路径下 IBCircuit 单独匹配全模型（P(correct) 0.90 vs 0.87）。
7. **E3（neuron 级, Llama-3.2-1B/IOI）**：EAP-IG 在 neuron 粒度维持完美 patching（P1=1.00），但单点梯度 EAP 在 hard ablation 下 signed P2=−0.35（行为反转，原始 logit diff −0.57 vs clean 1.69）。

## Critical Assessment

论文最值得肯定的是**诚实地报告反直觉与 null 结果**，而非兜售"电路方法普遍更优"：

- E5 直接给出 patch 忠实度与剪枝保留率 ρ=−0.78 的负相关（图 8），并明言"逐点梯度 EAP-IG 忠实却几乎只保留一半任务精度"——这削弱了"忠实电路=好剪枝信号"的直觉，是论文最有冲击力的发现。
- E7 把 circuit 选择 vs random mask 的比较作为 null 报告（mean Δ=−0.001），并正确归因：保护泛化的是预算约束而非选择哪部分。
- E1 中 CD-T 双忠实却零 canonical 重叠、E4 中拒绝电路在 hard ablation 下反转为顺从，都精准说明了"为何要 multi-pillar 而非单分数"的设计动机。
- 论文不夸大覆盖：明确把方法分 stable/research 两层，承认 intervention 模块依赖 architecture-registry（目前 Llama/Qwen/Gemma，Mistral/Phi/GPT-2/Falcon 结构完整但未全验证），TransformerLens 3 支持待办。
- 但**统计功效受限且明说**：stability 仅用 3 个 seed（42–44），其余研究单 seed（E5–E7 只报一个 Llama-3.2-3B-Instruct·BoolQ cell），cross-seed 可靠性（SDrel, Pillar 7）尚未报告，ρ=−0.78 是单 cell 而非总体估计。
- 概念上的诚实也带来工具定位的清晰：CircuitKIT 把"发现忠实"与"可行动性"解耦，主张下游效果必须经 `ck.benchmark` 直接测，而非从内在忠实度推断。

## Strengths

- **真正的端到端统一**：一个 artifact 串起 discover→evaluate→intervene→benchmark，三种接口（Python/CLI/YAML）等价，跨接口免格式转换。Table 1 显示它是唯一同时覆盖三阶段、支持自定义数据、在数十亿参数 LLM 上工作的库（TransformerLens/NNsight/Captum/Auto-Circuit/MIB/InterpBench 均只覆盖部分）。
- **声明式自定义数据路径是实质贡献**：把"从结构化数据构造 (clean, corrupt) 对"这个实践瓶颈变成一个声明的模板步骤（含 token 对齐、判别标签、GREEN/YELLOW/RED 预检门），并把 clean-only 数据自动路由到 IBCircuit/CD-T——这是把电路发现从"canonical 任务集"解放出来的关键。
- **多支柱忠实度面板**：直接回应 Miller et al. [11] 关于"单指标会反转排名"的批评，signed ratio 把行为反转暴露出来而非 clamp 到 0。
- **干预与既有工具集成而非重造**：剪枝走 LLM-Pruner、量化走 quanto/llmcompressor、微调走 PEFT/LoRA、编辑走 ROME/MEMIT、基准走 lm-eval-harness——降低采纳门槛且便于公平对比。
- **registry 治理模式**：`@register` 装饰器让新算法/选择器/corruption/模型家族无需改核心即可加入，stable/research 分层给出清晰晋升路径。
- **实证诚实**：明确报告 null、负相关、inversion，并以 dual-use 风险为由选择 source-available 而非 OSI 许可。

## Limitations

- **统计功效弱**：stability 仅 3 seed，其余单 seed；ρ=−0.78 等关键数字是单 cell，未做多 seed/cross-model 总体估计；Pillar 7 干预可靠性（SDrel）未实测。
- **干预覆盖受 architecture-registry 限制**：剪枝/量化/微调需逐家族 registry 条目，目前仅 Llama/Qwen/Gemma（含 Gemma 3）主要支持，Mistral/Phi/GPT-2/Falcon 结构完整但未充分验证；整体覆盖继承 TransformerLens 2.18 的 catalog 与版本 pin（3.x 待支持）。
- **算法各自的工程代价**：ROME/MEMIT 需用户提供约十万规模语料估计 key covariance（库只附够冒烟测试的小语料）；IBCircuit 单 batch 训练 mask，激活显存随模型增大（故 E1 六算法比较只在 GPT-2 规模）；ACDC node-only 且昂贵。
- **corruption 策略依赖可识别句法结构**：在 instruction-tuned / safety 相关 prompt 上可能找不到可改之处，需用户自供 corrupt 列；在 AdvBench/TruthfulQA/StereoSet 上自动合成对的信号有效性仍是 future work。
- **SAE 特征粒度未落地**：conclusion 提 sparse-autoencoder 特征为"自然下一粒度"，但本文未实现。
- **dual-use 张力**：E4 展示的"拒绝电路在 hard ablation 下反转为顺从"既是警示也是风险——库通过聚合分数、需显式 import 干预模块、case-study 数据仅限研究用途来缓解，但未消除。
- **LSAL v1.1 非 OSI 开源**：禁止商业利用可能限制工业采纳与生态贡献。

## Relevance

本文对以下方向高度相关：

- **机制可解释性研究者**：提供了首个把发现-评估-干预三阶段统一在同一制品上的基础设施，可直接用于跨方法/跨模型/跨任务比较，避免反复写胶水代码。多支柱面板回应了 [11] 对单分数稳健性的批评。
- **AI 安全 / 对齐工程**：refusal/bias 审计、事实编辑、遗忘评估、circuit-guided 压缩都有现成模块；E4 的"拒绝电路反转"对研究 jailbreak/abliterated 模型（如 [23, 36]）有直接警示价值。
- **模型压缩工程**：E5–E7 给出 circuit-guided 剪枝/量化/微调与 magnitude/Taylor/Wanda 基线的公平对比框架；结论是"必须按干预类型分别 extrinsic 验证"，对实际选型有指导意义。
- **工具生态**：补全 TransformerLens（introspection substrate）/NNsight/Auto-Circuit/MIB/InterpBench 各自只覆盖部分管线的空白。
- **与 SAE / 可行动可解释性路线**：把 locate-then-act 循环框架化，为 SAE 特征粒度接入预留了自然扩展点。

对日常 paper 跟踪而言，这是一篇"系统型 + 实证诚实"的工具论文，价值在基础设施与反直觉发现，而非新算法。

## Questions for Further Exploration

1. **统计稳健性**：ρ=−0.78 的 patch-faithfulness 与剪枝保留率负相关，在多 seed、跨任务（不止 BoolQ）、跨模型（不止 Llama-3.2-3B-Instruct）上是否成立？当前仅单 cell，能否扩到总体估计？
2. **Pillar 7（SDrel）实测**：discovery 稳定性 J 与 cross-seed 干预可靠性 SDrel 之间的关联尚未测量，是否真能用稳定性预测干预可重复性？
3. **CD-T 现象的机理**：为何 CD-T 在 IOI 上双忠实却零 canonical 头重叠（12/15 槽位在 MLP）？它发现的是同一机制的不同等价实现，还是一条绕过标准头的替代路径？
4. **E4 拒绝反转的工程含义**：hard ablation 下拒绝电路反转为顺从，对 abliterated/refusal-removed 模型的检测与缓解意味着什么？能否用 Pillar 2 的 signed ratio 作为"安全护栏"信号？
5. **量化 vs 剪枝的预测指标分歧**：为何 ablation 忠实度预测量化保留率（ρ=+0.73）却不预测剪枝？能否为每类干预定义一个"正确的内在指标"？
6. **SAE 特征粒度接入**：将 sparse-autoencoder 特征作为第三种 granularity 后，node/neuron/feature 三层 circuit 与干预模块的对应关系如何？
7. **自定义数据 corruption 质量**：自动合成的 entity-swap/token-swap 等策略在 AdvBench、TruthfulQA、StereoSet 上是否真带有效归因信号？预检门的 GREEN/YELLOW/RED 评级与下游发现质量相关性如何？
8. **许可证影响**：LSAL v1.1 禁止商业利用与降低部署安全性的用途，会多大规模限制工业采纳与社区贡献？是否会出现兼容的纯开源分叉？
