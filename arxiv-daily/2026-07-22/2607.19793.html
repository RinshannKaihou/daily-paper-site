Silent Failures in Multimodal Agentic Search: A Diagnostic Taxonomy and Cross-Judge Evaluation

Zhengxian Wu · Junjie Gao · Kai Yang — Ant Group, Hangzhou

multimodal-evaluation agentic-search silent-failure llm-as-judge faithfulness 800 trajectories · 4 frontier models

Key finding

Surface accuracy systematically overestimates true correctness in multimodal agentic search — silent failures shift, not vanish, as models grow stronger.

Key Finding

Answer-level accuracy cannot separate faithful grounded reasoning from trajectories that are superficially correct yet unsupported by evidence.

Key Metric −7.3pp

Gemini 3.1 Pro · Accuracy 61.3% → TCR 54.0%. Largest gap of any model.

Why This Matters

A correct final answer can still come from an ungrounded, hallucinated, or shortcut trajectory — measuring answers alone hides multimodal reliability rot.

Critical Edge

The two most central failure types (mod-sc κ=0.031, or-ld κ=0.301) are nearly judge-dependent, so TCR itself carries judge noise.

01

Methodology

6 failure types · unified ReAct · rubric judge · cross-validators

A multimodal search task x=(I,q) yields a trajectory τ={z,a,o,…,y}. For each, a structured diagnosis d(τ)={c,f₁…f₆} records answer correctness and six non-mutually-exclusive silent-failure flags spanning pre-, during-, and post-retrieval stages.

pre-retrievalmod-scModal Shortcut

Answers without genuinely using the input image.

post-retrievalpht-grHallucinated Grounding

Cites a source whose content does not support the claim.

post-retrievalwe-raWrong-Evidence Right-Answer

Final answer correct, but trajectory contains erroneous evidence or steps.

during-retrievalor-ldOver-Retrieval Laundering

Redundant searches act as "grounding" wrappers for parametric-knowledge answers.

post-retrievalcm-ctCross-Modal Conflict

Answer agrees with noisy text but contradicts the image.

post-retrievalprv-hlProvenance Hallucination

Fabricates URLs, dates, source names, or citations.

a

Unified trajectory collection

All four models run on an identical ReAct scaffold, system prompt, and tool set (web search, image search, webpage reading, image cropping, reverse image search), capped at 10 tool calls per trajectory.

b

Rubric-guided judge

The judge receives question, image, reference answer, full trajectory, and final answer — outputting answer correctness, each failure flag, a rationale, and the offending steps.

c

Cross-judge validation

Same-family and cross-family judges re-score a stratified sample; agreement is reported as Cohen's κ to expose where the taxonomy is stable vs. judge-fragile.

Headline metrics Accuracy judges only the final answer. TCR (True Correctness Rate) requires the answer be correct and no silent-failure flag fired. Trajectories end as committed / refused / exhausted / crashed; main results are on the committed subset.

03

Strengths & Limitations

side-by-side read

[+]Strengths

  • New, important framing — migrates "silent failure" from agent reliability into multimodal search, with an actionable taxonomy.
  • Fair pipeline: unified scaffold, tools, and logging make the four frontier models genuinely comparable.
  • Layered diagnostics — failure distribution, blank-image control, tool ablation, and cross-judge κ mutually corroborate.
  • Crisp explanation of the counter-intuitive "stronger ≠ more reliable" finding via capability-dependent failure migration.

[!]Limitations

  • Judge dependence: fine-grained flags (mod-sc, or-ld) have low cross-judge agreement, so TCR's absolute value carries judge bias.
  • Small, single-benchmark: 800 trajectories, MMSearch-Plus only; no cross-benchmark or human-gold external calibration.
  • Taxonomy may be incomplete / overlapping — categories are non-mutually-exclusive with no co-occurrence analysis.
  • Diagnosis-only: no proposed mitigation, training, or prompting fix to reduce silent failures.
  • Crashed / exhausted modes under-analyzed (Gemini 3.1 exhausted ≈ 38%), yet failure analysis focuses on the committed subset.
04

Open Questions

for further exploration
1

Can behavioral signals (image attention, tool-call patterns) reduce reliance on the judge for ambiguous flags like mod-sc / or-ld?

2

Is the TCR-vs-Accuracy gap stable at scale and across benchmarks (MMSearch, MC-Search, WebWatcher)? How do failure mixes differ?

3

Can silent failures be trained away (process-level RL, faithfulness rewards) rather than only detected at inference?

4

How stable is the failure distribution across random seeds — is this a capability ceiling or sampling noise?

5

Do post-tool pht-gr / cm-ct rises reflect induced new failures or newly-exposed hidden ones — and how would one tell them apart experimentally?

6

Should high exhausted rates (e.g. Gemini 3.1 ≈ 38%) count as a distinct "silent give-up" failure mode in reliability metrics?
