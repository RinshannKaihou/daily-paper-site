> **一句话总结：** 本文提出多模态智能体搜索的六类"静默失败"分类法（模态捷径、幻觉式溯源、错据正答、过度检索洗白、跨模态矛盾、来源幻觉），并构建轨迹级诊断管线；在 MMSearch-Plus 上对四个前沿模型评测发现，表面准确率系统性高估真实正确率，且静默失败随能力提升而转移而非消失。

> **TL;DR:** This paper introduces a six-category taxonomy of "silent failures" in multimodal agentic search and a trajectory-level LLM-judge diagnostic pipeline; on 800 MMSearch-Plus trajectories across four frontier models, surface accuracy overestimates true correctness (e.g., Gemini 3.1 Pro drops from 61.3% Accuracy to 54.0% TCR), and stronger models/tools shift rather than eliminate failures.

## 核心贡献

1. **将多模态智能体搜索的"静默失败"形式化为轨迹级可靠性问题**，指出仅靠最终答案准确率无法区分"忠实接地推理"与"看似正确但无证据支撑的轨迹"。
2. **提出覆盖检索前/中/后三阶段的六类静默失败分类法**：模态捷径（mod-sc）、幻觉式溯源（pht-gr）、错据正答（we-ra）、过度检索洗白（or-ld）、跨模态矛盾（cm-ct）、来源幻觉（prv-hl）。
3. **构建统一诊断管线**：在统一 ReAct 脚手架下收集轨迹，用结构化评分规则的 LLM-judge 同时评估答案正确性与六类失败标记，并通过同族/跨族裁判交叉验证评估诊断可靠性。
4. **在 MMSearch-Plus 上对四个前沿模型进行实证研究**，揭示表面准确率与真实正确率之间的系统性差距，并表明静默失败与能力相关、随模型增强而转移。

## Key Contributions

- Formulates silent failures in multimodal agentic search as a trajectory-level reliability problem, demonstrating that answer-level accuracy alone cannot separate faithful grounded reasoning from superficially-correct-but-unsupported trajectories.
- Introduces a six-category taxonomy spanning pre-/during-/post-retrieval stages, each instantiated with representative task examples in Figure 1.
- Builds a lightweight, model-agnostic diagnostic pipeline (unified ReAct scaffold + rubric-guided LLM judge + same/cross-family cross-validators) and releases the framework with a project home page.
- Empirically shows on MMSearch-Plus that surface accuracy consistently overestimates true trajectory correctness, and that failures are capability-dependent and often shift rather than vanish.

## Methodology

**任务与轨迹定义。** 给定多模态搜索任务 x = (I, q)（图像 I 与问题 q），智能体产出一条轨迹 τ = {z₁,a₁,o₁,…,z_T,a_T,o_T,y}（推理 z、动作/工具调用 a、观测 o、最终答案 y）。对每条轨迹生成结构化诊断记录 d(τ) = {c, f₁,…,f₆}，其中 c 表示答案是否正确，f_i 表示是否触发第 i 类静默失败（类别不互斥）。

**六类静默失败分类法。**
- (1) **模态捷径 (mod-sc)**：未充分使用输入图像即作答，多发于检索前。
- (2) **幻觉式溯源 (pht-gr)**：引用了检索来源，但来源内容并不支持所述结论。
- (3) **错据正答 (we-ra)**：最终答案正确，但轨迹中存在错误证据或推理步骤。
- (4) **过度检索洗白 (or-ld)**：发起多次冗余检索，答案实由参数知识给出，检索仅作"接地"包装。
- (5) **跨模态矛盾 (cm-ct)**：答案与噪声文本检索一致，但与图像明显信息冲突。
- (6) **来源幻觉 (prv-hl)**：编造 URL、日期、来源名或引用。

**诊断管线三步。** (a) 统一轨迹收集：所有模型在相同的 ReAct 脚手架、系统提示、工具接口（web search、image search、webpage reading、image cropping、reverse image search）与日志格式下运行，每条轨迹最多 10 次工具调用。(b) 规则引导 LLM 诊断：裁判接收问题、图像、参考答案、完整轨迹与最终答案，输出答案正确性及每类失败标记，并附理由与相关步骤。(c) 跨裁判验证：用同族与跨族裁判重评同一分层样本，计算 Cohen's κ。

**核心指标。** Accuracy 仅判最终答案；True Correctness Rate (TCR) 要求答案正确且无任何静默失败标记。轨迹终态分为 committed（给出实质答案）/refused/exhausted（达到工具上限无答案）/crashed，主结果在 committed 子集上报告。

## Results & Highlights

- **表面准确率系统性高估真实正确率。** 在 committed 子集上，TCR 相比 Accuracy 的下降幅度为：Claude Sonnet 4.6 从 36.0%→34.6%（−1.4pp）、Gemini 2.5 Pro 从 31.6%→28.6%（−3.0pp）、**Gemini 3.1 Pro Preview 从 61.3%→54.0%（−7.3pp，下降最大）**、GPT-4o 从 22.9%→22.3%（−0.6pp）。
- **空图像压力测试几乎摧毁所有"正确"轨迹。** 对 208 个原判答案正确的 committed 轨迹将图像替换为同尺寸空白图重跑：Claude 0/50、Gemini 2.5 0/43、Gemini 3.1 2/75、GPT-4o 0/40 存活，总体 206/208 不再正确，说明这些任务确需图像、且观察到的轨迹问题确为多模态接地失败。
- **工具消融：reverse_image_search 提升准确率但重塑失败分布。** 加入该工具后，Claude、Gemini 2.5、GPT-4o 的 committed Accuracy 分别提升 +16.5、+18.9、+13.9 pp；同时 mod-sc 与 or-ld 下降（如 GPT-4o mod-sc −23.2pp、Claude or-ld −35.4pp），但 pht-gr 与 cm-ct 反而上升（如 GPT-4o pht-gr +18.9pp、cm-ct +13.5pp）。Gemini 3.1 仅 +4.8pp 且各失败类别仅微降。
- **跨裁判一致性：答案正确性稳定，细粒度失败标记高度依赖裁判。** answer_correct 同族 κ=0.916、跨族 κ=0.817、跨族观察一致率 P_o=98%；而 mod-sc 跨族 κ 仅 0.031、P_o=25%，or-ld 跨族 κ=0.301，表明"是否真正使用了图像""检索是否仅为参数知识背书"的判定边界对裁判高度敏感。
- **失败随能力转移而非消失。** Claude/Gemini 2.5/GPT-4o 失败集中于检索后（pht-gr、cm-ct）；最强的 Gemini 3.1 反而 mod-sc 最高——因其视觉编码器直接识别图像、不再调用反搜，识别错误时即变为检索前模态捷径。

## Critical Assessment

- 论文的核心张力抓得准：**多模态智能体搜索的可靠性不能只看最终答案**。TCR 与 Accuracy 的差距（尤其 Gemini 3.1 的 7.3pp 落差）用具体数字证明了"答对但过程不可靠"的普遍性。
- **空图像压力测试**是亮点设计：直接证伪"问题本身不需图像"的替代解释，把失败归因锚定到多模态接地上，而非任务可解性的混淆。
- 六类分类法的**实操判定边界存在明显脆弱性**。论文自己承认 mod-sc 跨族 κ=0.031、or-ld 跨族 κ=0.301——这意味着最核心的两类失败（"是否真正用了图像""检索是否仅为参数知识背书"）几乎取决于裁判的主观阈值，TCR 这一主指标因此带有裁判噪声。作者诚实地报告了这一点，但未给出更鲁棒的替代（如人工黄金标注的多裁判校准）。
- **样本规模偏小且仅一个基准。** 每模型 200 条、committed 子集更小（Gemini 3.1 仅 124 条），Wilson 置信区间宽度约 ±8–9pp，部分差异（如 GPT-4o 的 −0.6pp）在统计噪声之内；且全部基于 MMSearch-Plus，泛化性未知。
- **工具消融的解读需谨慎。** pht-gr/cm-ct 上升可能部分是因为"有了更多图像相关证据后误用机会增多"，但样本量与单工具消融不足以区分"工具诱发新失败"与"原本隐藏失败被暴露"。

## Strengths

- 提出的问题新颖且重要：将 agent 可靠性研究中的"silent failure"概念系统迁移到多模态搜索，并给出可操作分类法。
- 管线设计公平——统一脚手架、统一工具、统一日志，跨模型可比性强。
- 诊断分析层次丰富：失败分布、空图像对照、工具消融、跨裁判一致性，互为印证。
- 对"更强模型≠更可靠"这一反直觉发现给出了清晰的能力依赖解释（失败转移）。

## Limitations

- **裁判依赖**：细粒度失败标记（尤其 mod-sc、or-ld）跨裁判一致性低，TCR 的绝对数值含裁判偏差。
- **规模与基准单一**：800 条轨迹、仅 MMSearch-Plus，缺少跨基准与人工黄金标注的外部校验。
- **分类法可能不完备/不互斥**：作者承认类别不互斥，但未给出类别间共现结构或潜在重叠的定量分析。
- **未提出缓解方法**：论文聚焦诊断，未涉及如何训练或提示以减少静默失败。
- **缺少对 crashed/exhausted 的深入分析**：如 Gemini 3.1 exhausted 达 38%，但失败分析集中在 committed 子集，对"未作答"模式着墨不多。

## Relevance

本文对从事**多模态大模型评测、智能体（agentic）系统可靠性、检索增强生成（RAG）忠实度评估**的研究者高度相关。其六类静默失败分类法可直接作为评测协议的检查清单，TCR 指标与空图像压力测试为多模态 agent benchmark 设计提供了可借鉴的范式。对于构建视觉搜索/浏览型 agent（如多模态 deep research）的工程团队，本文揭示了"答案正确 ≠ 过程可信"的风险，对 groundedness 与 provenance 的监控有直接指导价值。LLM-as-judge 的稳定性结论也对评估方法学有警示意义。

## Questions for Further Exploration

1. 如何降低 mod-sc / or-ld 这两类"判定边界模糊"失败对裁判的依赖？是否可引入图像注意力或工具调用模式等行为信号作为客观辅助特征？
2. 在更大规模、跨基准（如 MMSearch、MC-Search、WebWatcher）上，TCR 与 Accuracy 的差距是否稳定？不同基准的失败分布有何差异？
3. 静默失败是否可通过训练时奖励（如过程级 RL、faithfulness 奖励）系统性地减少，而非仅在推理时检测？
4. 同一模型在不同随机种子下的静默失败分布稳定性如何？失败是"能力天花板"还是"采样波动"？
5. pht-gr 与 cm-ct 在工具增强后上升，究竟是"诱发新失败"还是"暴露隐藏失败"？如何设计实验加以区分？
6. exhausted（达工具上限无答案）比例高的模型（如 Gemini 3.1 的 38%）是否存在另一种"静默放弃"，是否应纳入可靠性度量？
