Adaptive Test-Time Compute Allocation for Reasoning LLMs via Constrained Policy Optimization

arXiv: 2604.14853

Date: 2026-04-16

https://arxiv.org/abs/2604.14853

TL;DR: This paper formalizes adaptive test-time compute allocation for reasoning LLMs as a constrained optimization problem and solves it via a two-stage Solve-then-Learn pipeline -- Lagrangian relaxation yields per-instance oracle actions, and a lightweight GBM classifier is trained to predict them from cheap features. Key Contributions

  • Formalizes input-adaptive test-time compute allocation as a constrained optimization problem -- maximizing expected accuracy subject to an average compute budget -- providing a principled alternative to heuristic rules.
  • Proposes the Solve-then-Learn pipeline: (1) a solve stage using Lagrangian relaxation to decompose the global budget constraint into per-instance sub-problems with closed-form oracle solutions, and binary search for exact budget targeting via proved cost monotonicity; (2) a learn stage that trains a lightweight GBM classifier on oracle labels from cheap input features for real-time deployment.
  • Establishes formal theoretical guarantees: cost monotonicity (Theorem 1) enabling binary search; an imitation regret bound (Theorem 2) reducing constrained inference to supervised classification; and a primal recovery result (Theorem 3) ensuring near-feasibility and near-optimality.
  • Demonstrates consistent gains across 3 LLMs and 2 benchmarks: up to 12.8% relative accuracy improvement on MATH under matched budget constraints, with >91% imitation accuracy closely tracking the Lagrangian oracle upper bound.

Methodology

The paper introduces AdaCompute, a framework for adaptively allocating test-time compute budgets to individual inputs in order to maximize aggregate accuracy under a shared average compute constraint.

Problem Formulation (Section 2): Given an input distribution, a discrete set of compute budgets $\mathcal{B} = \{b_1, \ldots, b_K\}$, and a target average budget $B$, the goal is to find a policy $\pi: \mathcal{X} \to \mathcal{B}$ that maximizes expected accuracy $\mathbb{E}[\mathrm{Acc}(x, \pi(x))]$ subject to $\mathbb{E}[\mathrm{C}(\pi(x))] \leq B$.

Solve Stage (Section 3) -- Lagrangian Oracle: A Lagrange multiplier $\lambda$ is attached to the budget constraint. The resulting relaxation decomposes across inputs. For each input $x$, the oracle allocation is:

$$b^\star(x; \lambda) = \arg\max_{b \in \mathcal{B}} [\mathrm{Acc}(x, b) - \lambda \, \mathrm{C}(b)]$$

This has an economic interpretation: $\lambda$ acts as the unit price of compute, and the oracle selects the budget maximizing net value. Theorem 1 proves that the induced average cost $\bar{C}(\lambda)$ is non-increasing in $\lambda$, enabling binary search to match any target budget $B$. Strong duality holds (Proposition 1), so the oracle allocation is an exact solution.

Learn Stage (Section 4) -- Amortized Policy Learning: Oracle labels $\{y_i^\star\}$ are used to train a lightweight K-way classifier (gradient-boosted machine, XGBoost) on 16 cheap text features (lexical statistics, structural indicators, and a single-pass entropy estimate). The classifier runs in <1 second on CPU and adds negligible overhead at deployment.

Theoretical Guarantees: Theorem 2 bounds the task-level regret by the imitation error times the worst-case per-instance gap, yielding a clean reduction from constrained inference to supervised classification. Theorem 3 shows that as classifier accuracy improves ($\varepsilon \to 0$), both feasibility violation and optimality gap vanish.

Critical Assessment

Novelty: High. While adaptive computation and LLM routing are well-studied, this is the first work to formalize per-input test-time compute allocation as a constrained optimization problem with Lagrangian decomposition and formal regret bounds. The Solve-then-Learn pipeline is conceptually clean and theoretically grounded.

Rigor: Excellent. The paper provides complete proofs for all theorems (cost monotonicity, imitation regret bound, primal recovery, strong duality). The connection from constrained optimization to supervised classification is both novel and well-established. Empirical evaluation covers 3 models x 2 datasets x 3 budget levels with multiple seeds.

Reproducibility: Moderate to high. The method uses standard components (XGBoost, binary search, self-consistency voting). Feature engineering details are provided. However, the offline data collection requires 9,600 API calls per model-dataset pair (38,400 total), which is non-trivial. No code repository is mentioned.

Significance: High for practical LLM deployment. As inference costs dominate in production, principled compute allocation is a pressing need. The framework is model-agnostic and can be integrated into any LLM pipeline. The theoretical reduction from constrained inference to classification is elegant and potentially influential.

Red flags:

  • The budget set $\mathcal{B} = \{1, 2, 4, 8, 16\}$ is small (5 options), making classification relatively easy. Scalability to much larger or continuous action spaces is unexplored.
  • The offline oracle computation requires full knowledge of $\mathrm{Acc}(x, b)$ for all $b$, which is expensive and impractical for new domains without pre-computation.
  • Test sets are small (200 questions each), raising questions about statistical robustness and generalization.
  • The framework is limited to self-consistency (majority voting) as the test-time scaling strategy; applicability to chain-of-thought length scaling or tree search is not demonstrated.

Strengths

  • Principled formulation with strong theoretical foundations and complete proofs.
  • The Lagrangian oracle provides an interpretable economic framework ($\lambda$ as the price of compute).
  • Clean reduction from constrained optimization to supervised classification with formal regret bounds.
  • Consistent improvements across diverse LLMs (open-weight, proprietary, small-scale).
  • Extremely lightweight at deployment: a small GBM classifier adds negligible overhead.
  • Extensive ablations covering architecture, data efficiency, budget granularity, and feature importance.
  • The inverted-U allocation pattern is a genuinely insightful finding about how compute should be distributed.

Limitations

  • Offline cost: The solve stage requires $N \times K$ full model evaluations per input to populate the accuracy table, which is expensive and must be repeated for each new model or domain.
  • Small benchmark scale: Only 200 questions per dataset, which limits statistical power and generalization claims.
  • Limited scaling strategy: Only self-consistency (majority voting) is tested; the framework's applicability to other test-time scaling methods (extended CoT, tree search, process reward verification) is theoretical but unverified.
  • Static policy: The learned policy does not adapt online to observed intermediate results during inference.
  • Feature engineering dependency: Performance depends on hand-crafted features; richer features (e.g., embedding-based) could improve results but add cost.
  • No multi-objective extension: The framework handles a single resource constraint; jointly optimizing latency, throughput, and accuracy remains future work.

Questions for Further Exploration

  1. How does the framework extend to continuous budget spaces (e.g., variable token budgets for chain-of-thought reasoning) rather than discrete sets?
  2. Can the oracle be approximated online using bandit-style exploration, eliminating the need for expensive offline accuracy table pre-computation?
  3. How does the framework perform on non-mathematical reasoning tasks (code generation, scientific QA, multi-modal reasoning) where difficulty estimation may be harder?
  4. What happens when the feature distribution shifts between training and deployment (e.g., new question types or domains)?
  5. Can the Lagrangian framework be extended to multi-objective settings with competing constraints on latency, cost, and accuracy simultaneously?
