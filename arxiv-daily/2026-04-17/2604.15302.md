> **一句话总结：** 本文提出两种互补的诊断工具——传递性违规分析和分裂共形预测集——来量化LLM评判器的逐实例可靠性，发现聚合指标掩盖了严重的逐文档不一致性（33-67%的文档存在至少一个有向3-循环），且预测集宽度与评分误差高度相关（r_s=+0.576, p<10^-100），能够可靠地捕捉文档级别的评估难度。

> **TL;DR:** This paper proposes two complementary diagnostics -- transitivity violation analysis and split conformal prediction sets -- to quantify per-instance LLM judge reliability, revealing that aggregate metrics mask severe per-document inconsistency (33-67% of documents have at least one directed 3-cycle) and that prediction set width strongly correlates with judge-human disagreement (r_s=+0.576), reliably capturing document-level evaluation difficulty.

## 核心贡献

- **传递性诊断**：首次在逐文档级别测量LLM评判器的有向3-循环（A>B, B>C, C>A）违规率，发现聚合违规率虽低（<5%），但33-67%的文档存在至少一个违规，个别文档违规率高达30.4%。
- **共形预测诊断**：将分裂共形预测应用于LLM评判器的Likert评分，提供理论上保证的>= (1-alpha) 覆盖率，将预测集宽度作为逐实例可靠性指标。
- **跨评判器一致性**：预测集宽度在不同评判器之间呈现一致的正相关（r=0.32-0.38），证明宽度反映的是文档固有难度而非评判器特异性噪声。
- **统一发现**：两种诊断独立地识别出相同的变异轴——评估标准（coherence、relevance更可靠）比评判器本身更能解释可靠性差异。
- **MFAS修复无效**：最小反馈弧集（MFAS）排名修复未能持续改善与人类排名的一致性，表明违规是稀疏噪声而非系统性方向偏差。

## Key Contributions

- **Transitivity diagnostic**: First to measure directed 3-cycle violation rates in LLM judges at the per-document level, revealing that aggregate rates (rho < 5%) mask the fact that 33-67% of documents carry at least one violation, with per-document rates reaching 30.4% for Mistral-Small.
- **Conformal prediction diagnostic**: Applies split conformal prediction to LLM judge Likert scores, producing prediction sets with theoretically guaranteed >= (1-alpha) coverage. Prediction set width serves as a per-instance reliability signal.
- **Cross-judge agreement on width**: Prediction set width shows consistent positive correlation across different judge pairs (Spearman r = 0.32-0.38 for fluency, consistency, relevance), demonstrating it captures document-level difficulty rather than judge-specific noise.
- **Unified finding across diagnostics**: Both diagnostics independently identify the same axis of variation -- evaluation criterion explains reliability more than the choice of judge, with relevance judged most reliably (avg set size ~3.0) and fluency/consistency remaining unreliable (avg set size ~4.9).
- **MFAS repair is ineffective**: Minimum Feedback Arc Set ranking repair does not consistently improve agreement with human rankings, establishing that violations are sparse noise rather than systematic directional bias.

## Methodology

The authors develop a two-pronged diagnostic pipeline evaluated on SummEval (30 documents x 8 systems = 240 outputs), using four LLM judges: GPT-4o-mini, LLaMA-3.1-70B, Qwen-2.5-72B, and Mistral-Small-3.1-24B.

**Transitivity Diagnostic**: For each document, all 28 pairwise comparisons among 8 systems are collected (40,320 total API calls). A directed 3-cycle occurs when A>B, B>C, and C>A simultaneously. The per-document violation rate p(x) is computed as the fraction of directed 3-cycles out of all possible triples. MFAS-based ranking repair is tested to see if removing cycles improves alignment with human gold rankings.

**Conformal Prediction Diagnostic**: Direct Likert scoring (1-5) is performed for each document-system pair (3,840 API calls). Split conformal prediction is applied using absolute residual |y_hat - y*| as the nonconformity score, with a calibration set producing prediction sets C(x) satisfying P(y* in C(x)) >= 1-alpha. Set width |C(x)| serves as the reliability indicator -- wider sets imply greater uncertainty.

**Evaluation**: The authors measure: (1) whether conformal coverage guarantees hold across all 16 judge-criterion combinations and 4 alpha levels; (2) whether set width correlates with actual judge-human disagreement (Spearman r_s); (3) whether different judges assign wide sets to the same documents (inter-judge width agreement).

## Results & Highlights

- **Coverage guarantees hold**: Empirical coverage meets or exceeds the 1-alpha target for all 16 judge-criterion combinations at all four alpha levels tested (0.05, 0.10, 0.15, 0.20).
- **Width-error correlation**: Pooling 1,918 observations yields Spearman r_s = +0.576 (p < 10^-100) between prediction set width and actual judge-human MAE. 13 of 16 judge-criterion combinations show perfectly monotonic width-error relationships.
- **Aggregate metrics are misleading**: Aggregate transitivity violation rates rho < 5% appear reassuring, but per-document analysis shows 33-67% of documents carry at least one directed 3-cycle.
- **Criterion dominates judge**: Relevance is judged most reliably (avg set size ~3.0), coherence moderately (avg set size ~3.9), while fluency and consistency are unreliable (avg set size ~4.9). This pattern holds across all four judges.
- **Fluency has the widest violation tails**: Per-document violation rate distributions are heavily right-tailed, with fluency consistently showing the widest tails.
- **Cross-diagnostic convergence**: Documents causing preference cycles in pairwise ranking also receive wide prediction intervals in direct scoring, confirming both signals point to the same underlying difficulty.

## Critical Assessment

- **Novelty**: Moderate-to-high. While conformal prediction and transitivity analysis are individually not new, their combined application to LLM judge reliability -- particularly the per-document disaggregation and the cross-diagnostic convergence finding -- is novel and insightful.
- **Rigor**: High. The paper uses proper statistical tests (Spearman correlations with p-values), validates coverage guarantees across multiple operating points, and honestly reports negative results (MFAS does not help; relevance correlation is non-significant per-criterion).
- **Reproducibility**: Strong. All code, prompts, and cached results are released. The experimental setup (dataset, judges, sampling strategy) is clearly described.
- **Significance**: High for the LLM evaluation community. The finding that aggregate metrics mask severe per-document inconsistency is practically important. The conformal prediction approach provides an actionable deployment signal.
- **Red flags**: (1) Scale limitation -- only 30 documents x 8 systems from SummEval, which is a single summarization dataset. Generalization to other tasks (dialogue, translation) is untested. (2) The marginal (not conditional) coverage guarantee from split conformal prediction is a known limitation acknowledged by the authors. (3) The relevance criterion shows non-significant per-criterion width-error correlation (r_s = -0.01), which slightly undermines the universal applicability of the width signal. (4) Fixed prompt templates per criterion -- prompt robustness is not analyzed.

## Strengths

- Two independent diagnostics converging on the same conclusion strengthens the overall argument significantly.
- The conformal prediction framework provides distribution-free, finite-sample coverage guarantees -- a principled alternative to ad-hoc reliability measures.
- The paper provides clear, actionable deployment recommendations (e.g., accept scores when |C(x)| <= 2, escalate to human annotation when |C(x)| = 5).
- Honest reporting of negative results (MFAS futility, per-criterion correlation weakness) enhances credibility.
- The finding that criterion explains more variance than judge choice is practically useful for practitioners.
- Clean experimental design with four diverse judges (GPT, LLaMA, Qwen, Mistral) covering 24B-72B parameter scales.

## Limitations

- **Scale**: Only 30 documents from SummEval, a single summarization dataset. Results may not generalize to other NLG tasks, domains, or larger document sets.
- **Marginal coverage**: Split conformal guarantees marginal P(y* in C(x)) >= 1-alpha, not per-document conditional coverage. Harder documents may receive tighter-than-justified sets.
- **Fixed nonconformity score**: Using absolute residual |y_hat - y*| is simple but potentially suboptimal. Learned nonconformity scores (e.g., based on judge confidence or LLM log-probabilities) could yield tighter, more informative sets.
- **Prompt sensitivity**: Only one prompt template per criterion is tested. Different prompts may yield different violation rates and set widths.
- **Human score discretization**: Rounding averaged human annotations to integers introduces a small calibration error.
- **Coherence exception**: Inter-judge agreement on prediction set width is weak for coherence (r = 0.10), suggesting the width signal is not uniformly reliable across all criteria.

## Relevance

本文与LLM训练稳定性和ML可靠性高度相关：它揭示了LLM评判器在聚合层面看似可靠但逐实例层面存在严重不一致的问题，这对依赖LLM自动评估来指导训练过程的做法提出了重要警示。共形预测集提供了一种分布无关的可靠性量化方法，可直接集成到评估流程中，为何时应信任自动评分、何时需人工介入提供了原则性框架。

## Questions for Further Exploration

- How do the diagnostics generalize to non-summarization tasks (e.g., code generation, mathematical reasoning, open-ended dialogue) where evaluation criteria differ substantially?
- Can learned nonconformity scores leveraging LLM internal signals (log-probabilities, attention patterns) produce tighter and more informative prediction sets?
- How does prompt engineering affect transitivity violation rates and prediction set widths -- could prompt optimization reduce per-document inconsistency?
- Would conditional conformal prediction methods (e.g., conformalized quantile regression) address the marginal vs. conditional coverage limitation for hard documents?
- Can the width-based trust signal be integrated into RLHF or DPO training pipelines to filter unreliable judge preferences before model updating?
- Does the "criterion matters more than judge" finding persist across larger-scale evaluations (hundreds of documents, dozens of judges)?
