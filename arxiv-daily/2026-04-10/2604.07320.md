# Evaluating In-Context Translation with Synchronous Context-Free Grammar Transduction

> **一句话总结：** 本文利用同步上下文无关文法(SCFG)构建可控的形式语言翻译任务来评估LLM的上下文翻译能力，发现翻译准确率随文法规模和句子长度显著下降，形态学和书写系统的差异严重削弱模型性能，且BLEU/chrF++等字符串重叠指标系统性地高估了模型的真实翻译质量。

> **TL;DR:** This paper evaluates LLMs' ability to perform in-context translation using synchronous context-free grammars (SCFGs) as controlled formal-language proxies, finding that accuracy degrades sharply with grammar size and sentence length, morphology and orthography differences strongly impair performance, and standard metrics like BLEU/chrF++ systematically overestimate true translation quality.

## 核心贡献
- 提出基于同步上下文无关文法(SCFG)的形式语言翻译评估框架，通过可控实验隔离影响上下文机器翻译(ICMT)的各项因素（文法规模、句子长度、语序、形态学、书写系统），避免了自然语言中难以控制的混淆变量
- 系统评估GPT-5系列和Gemma 3系列共6个模型，发现文法规模和句子长度是性能的主要瓶颈，语序差异（SVO/SOV/OVS）对性能影响微弱，但形态学一致性标记和书写系统的变化显著降低翻译准确率
- 揭示BLEU和chrF++等标准翻译评估指标系统性地高估LLM的真实翻译能力，且这种偏差随任务难度增加而扩大，这对依赖字符串重叠指标的机器翻译评估提出了重要警示
- 提供详细的错误分类分析，发现模型主要犯召回错误（选错目标词汇）、幻觉（生成不存在的词汇）和源语言泄漏（未翻译源语言词汇）三类错误

## Key Contributions
- Introduces a SCFG-based formal-language transduction framework for evaluating in-context machine translation (ICMT) that allows controlled manipulation of grammar size, sentence length, word order, morphology, and orthography, eliminating confounds present in natural-language evaluation
- Systematically evaluates 6 models from GPT-5 and Gemma 3 families, finding that grammar size and sentence length are the dominant performance bottlenecks, word order differences have negligible impact, but morphological agreement and script changes strongly degrade accuracy
- Demonstrates that standard MT metrics (BLEU, chrF++) systematically overestimate LLM translation quality, with this overestimation growing as tasks become harder -- an important cautionary finding for the MT evaluation community
- Provides detailed error taxonomy showing models primarily commit recall errors (selecting wrong target vocabulary items), hallucinations (generating non-existent words), and source vocabulary leakage (leaving source words untranslated)

## Methodology
The authors construct parameterizable SCFGs that define pairs of formal languages modeled after natural-language grammar (with linguistically interpretable constituents like CP, TP, NP, VP). A meta-grammar controls source/target language properties including word order (SVO/SOV/OVS), morphological agreement (presence/absence of person/number marking on verbs), and orthography (Latin, Latin with diacritics, Cyrillic, Hebrew, Hebrew with vowel pointing). Vocabularies are generated from scratch to guarantee zero data contamination. LLMs are prompted with the grammar rules and a source sentence, then asked to produce the target translation, which is evaluated against the gold output using exact-match accuracy (plus BLEU, chrF++, and bag-of-words match). Experiments vary grammar size (25 to ~10,000 rules), sentence length (3-50 words), and the linguistic properties described above.

## Results & Highlights
- **Grammar size dominates:** GPT-5 drops from ~97% exact match on 57-rule grammars to ~49% on 8,037-rule grammars; GPT-5-mini collapses from 97.5% to 1.3%; smaller models (Gemma 3, GPT-5-nano) are near 0% exact match on all but the tiniest grammars
- **Sentence length matters:** Even GPT-5 drops from 95% on 5-word sentences to 73% on 36-word sentences
- **Word order is surprisingly irrelevant:** SVO-to-SVO, SVO-to-SOV, and SVO-to-OVS conditions yield nearly identical accuracy across all grammar sizes and sentence lengths
- **Morphological agreement hurts:** NoAgr-to-NoAgr is easiest (near-perfect for GPT-5); NoAgr-to-Agr is hardest, since the source lacks the information needed to select the correct target inflection
- **Orthography is a major bottleneck:** Latin > Latin+diacritics > Cyrillic > Hebrew >> Hebrew+vowel pointing (0% exact match for ALL models at ALL sizes); performance ordering tracks training-data frequency of scripts
- **BLEU/chrF++ inflate scores:** For GPT-5 on 8,037-rule grammars, exact match is 0.487 but BLEU is 0.780 and chrF++ is 0.909, severely overestimating actual translation correctness
- **Gemma 3 models largely fail:** Even the 12B Gemma achieves 0% exact match on most conditions, highlighting a large capability gap with GPT-5

## Critical Assessment
This is a well-designed and thoughtfully executed evaluation paper that makes a genuine methodological contribution. The use of SCFGs as controlled proxies for ICMT is elegant -- it sidesteps the evaluation challenges of natural-language translation while enabling rigorous controlled experiments. The finding that standard MT metrics systematically overestimate LLM performance is particularly important and should influence how the community evaluates ICMT going forward. However, the SCFG formalization abstracts away from many aspects of real translation (ambiguity, semantics, discourse context, lexical gaps), so the results represent an upper bound on one component of ICMT rather than a full characterization. The deliberate exclusion of few-shot examples (to isolate grammar-only usage) means the paper does not address how grammatical descriptions and exemplars interact, which is arguably the more practical setting.

## Strengths
- Elegant experimental design using SCFGs to create a fully controllable evaluation environment with guaranteed gold translations and zero data contamination
- Comprehensive evaluation across 6 models, 4 experimental axes (size, word order, morphology, orthography), and 4 metrics with detailed appendices
- The finding that BLEU/chrF++ systematically overestimate true accuracy is highly consequential for the MT evaluation community
- Detailed error taxonomy provides actionable insights into specific failure modes
- Clear, well-written paper with excellent use of linguistics to motivate experimental conditions

## Limitations
- SCFGs abstract away from critical aspects of natural-language translation: ambiguity, semantics, discourse context, multi-word expressions, lexical gaps, and pragmatic factors
- Only evaluates the grammar-only (zero-shot) setting; does not study interaction between grammatical descriptions and few-shot exemplars, which prior work (Aycock et al. 2024) suggests is crucial
- The formal languages, while naturalistic in structure, have unusually direct lexical correspondences compared to real language pairs
- Small sample sizes per condition (appears to be ~24 sentences per condition based on the confidence intervals), which may limit statistical power for some comparisons
- Only evaluates GPT-5 and Gemma 3 families; results may not generalize to other model families (Claude, Llama, Mistral, etc.)

## Relevance
本文对LLM推理能力评估有直接参考价值：它提供了一个精心设计的形式语言基准来测试模型是否真正理解并应用上下文中提供的规则，而非依赖训练数据中的统计模式。其发现——即模型在处理不熟悉书写系统和复杂形态学时表现脆弱——揭示了LLM符号推理和精确字符串操作能力的根本性限制。

## Questions for Further Exploration
- How would performance change if models were given a few example translations alongside the grammar, as in the more practical ICMT setting?
- Could the SCFG framework be extended to include semantic ambiguity and discourse context to better approximate real-world translation difficulty?
- Do the orthography failures stem primarily from tokenization issues (unfamiliar character sequences producing poor token boundaries) or from deeper representational problems?
- Would chain-of-thought prompting (asking the model to first parse the source sentence using the grammar rules, then generate the target) improve accuracy on larger grammars and longer sentences?
- How do these results compare to the same task framed as a program synthesis or code generation problem rather than natural-language translation?
