Domain Restriction via Multi SAE Layer Transitions

arXiv: 2605.11920

Date: 2026-05-12

https://arxiv.org/abs/2605.11920

一句话总结： 本文提出利用稀疏自编码器（SAE）特征在LLM各层之间的转移模式作为领域签名，仅用域内数据训练轻量级序列模型（Markov/HTM/RNN），即可实现高效且可解释的域外输入检测，在far-OOD场景下AUROC达0.97+，在near-OOD上也具有竞争力。
TL;DR: This paper proposes detecting out-of-domain inputs to LLMs by modeling depthwise transitions of sparse autoencoder (SAE) features across transformer layers. Using only in-domain data and a lightweight first-order Markov backend, it achieves strong OOD detection (AUROC 0.97+ far-OOD) with built-in interpretability on Gemma-2 2B/9B.

核心贡献

  • 提出了一种仅依赖域内（ID-only）数据的LLM域外输入检测方法，无需微调基础模型或使用域外标注数据。
  • 引入了SAE→SDR表示，将LLM各层残差流激活通过稀疏自编码器编码、Token池化、密度过滤和Top-k二值化，形成可比较的深度轨迹。
  • 设计了模块化的序列异常评分框架，支持一阶Markov、HTM和RNN三种轻量级后端进行域外评分。
  • 通过表示分析揭示了LLM中层是域一致特征最集中的区域，层间转移比静态特征重叠携带更多域特异性信息。
  • 在硬边界案例研究中展示了方法的可解释性——检测器的"误判"往往是语义上确实模糊的边界样本。

Key Contributions

  • Proposes an ID-only scope-gating method for detecting out-of-domain text by modeling depthwise transitions of sparse, interpretable SAE features, without fine-tuning the base LLM or using labeled OOD data.
  • Introduces the SAE→SDR representation for transformer-layer trajectories: extract residual-stream activations → SAE encode → token-average pool → global-density mask → Top-k binarize.
  • Provides a modular sequential scoring framework comparing lightweight backends (first-order Markov, HTM, RNN) for anomaly scoring based on layer-to-layer feature transitions.
  • Conducts representation analysis on Gemma-2-2B showing that mid-layers exhibit the strongest domain-consistent feature overlap and that transition-based modeling captures additional domain structure beyond static overlap.
  • Delivers an interpretability case study on AGNews hard-OOD boundary examples, showing that the lowest-anomaly OOD samples are semantically ambiguous cases where SAE feature transitions genuinely match the ID domain signature.

Methodology

The method operates as a four-stage pipeline:

  1. Activation Extraction: Given input tokens, extract residual-stream hidden states across L contiguous transformer layers, masking padding positions.
  1. Sparse Feature Decomposition (SAE→SDR): Pass each layer's activations through a pretrained layer-specific SAE encoder (16k-dimensional for Gemma-2). Then:
  • Token-axis pooling: Average SAE features over token positions to get one sparse vector per layer.
  • Global-density masking: Remove features whose global activation density exceeds a threshold θ (using Neuronpedia statistics), pruning uninformative ubiquitous features.
  • Top-k binarization: Select top-k remaining features by magnitude and convert to a binary Sparse Distributed Representation (SDR), yielding a depth trajectory {s₁, ..., sₗ}.
  1. Sequential Anomaly Scoring: Train a lightweight sequential model on ID-only SDR sequences. Three backends are compared:
  • Markov (default): Count adjacent-layer co-activations from ID data, estimate sparse transition probabilities with Laplace smoothing, and score test inputs by negative mean log-likelihood over active feature pairs.
  • HTM: Hierarchical Temporal Memory anomaly scoring (fraction of active bits not predicted).
  • RNN (LSTM): Trained to predict next-layer activations with per-bit BCE loss.
  1. Aggregation: Sum per-layer anomaly scores across depth to produce a final anomaly score S(x).

The key insight is that in-domain inputs produce regular, low-entropy feature transition patterns across layers, while OOD inputs deviate from these patterns.

Results & Highlights

Far-OOD (ID: 20 Newsgroups):

  • Gemma-2-2B + Markov: AUROC 0.917–0.985 across OOD datasets (SST-2, MNLI, RTE, IMDB); AUPR up to 0.968.
  • Gemma-2-9B + Markov: Consistently improves over 2B (AUROC 0.954–0.989).
  • LR baseline remains stronger (AUROC 0.994–0.999), but requires a fine-tuned model variant; the SAE method uses no OOD data or fine-tuning.

Near-OOD (class-level splits):

  • AGNews (4 one-vs-all): Markov backend outperforms LR baseline on all splits (AUROC 0.88–0.98 vs. 0.73–0.82), demonstrating strong fine-grained semantic discrimination.
  • ROSTD and SNIPS: Competitive with LR baseline.
  • CLINC150 (150 intents): Performance degrades sharply for 2B (AUPR 0.013) due to representational aliasing—too many fine-grained intents collapsing onto overlapping SAE features. 9B model partially alleviates this (AUPR 0.728).

Ablations:

  • First-order Markov matches or slightly outperforms HTM and RNN, indicating most discriminative signal resides in adjacent-layer transitions.
  • Top-k=10 is optimal; increasing k degrades performance due to noise.
  • Raw activation baselines (without SAE) perform poorly (AUROC 0.753), confirming SAE decomposition is essential.
  • HTM saturates and collapses at k=128 (AUROC 0.50).

Interpretability Case Study:

  • On AGNews Business vs. Sci/Tech, hard-OOD false negatives are semantically ambiguous (e.g., tech earnings reports look like Business; telecom infrastructure news looks like Sci/Tech). Decoded SAE feature transitions confirm these samples genuinely activate ID-like internal dynamics.

Critical Assessment

  • Novelty: Moderate-to-high. The idea of using SAE feature transitions across layers for OOD detection is novel and well-motivated. The SAE→SDR pipeline and the finding that first-order transitions suffice are interesting contributions. However, the individual components (SAEs, Markov models, OOD scoring) are all established.
  • Rigor: Generally strong. The paper includes representation analysis, ablations (backend comparison, top-k sweep, SAE vs. raw, density threshold), and an interpretability case study. However:
  • Only evaluated on Gemma-2 models with 16k SAEs; generalization to other model families and SAE sizes is unknown.
  • The LR baseline consistently outperforms on far-OOD; the paper's advantage is primarily in interpretability and near-OOD (AGNews).
  • The CLINC150 failure is honestly discussed but represents a fundamental limitation for fine-grained intent detection.
  • Reproducibility: Moderate. Uses publicly available models (Gemma-2), SAEs (SAELens), and datasets. However, code is not yet released, and Neuronpedia feature labels/densities are required for the density masking step.
  • Significance: The work bridges mechanistic interpretability and practical LLM deployment concerns (scope gating). The interpretability angle — being able to explain why an input was flagged as OOD through decoded feature transitions — is a genuine practical advance over black-box methods.
  • Red flags:
  • The method does not outperform the LR baseline on far-OOD; its main advantage is interpretability and near-OOD on coarser taxonomies.
  • Heavy dependence on SAE quality and resolution; fundamentally limited when task granularity exceeds feature resolution (CLINC150).
  • No evaluation on real-world deployment scenarios with adversarial or distribution-drift inputs.

Strengths

  • Novel and well-motivated use of SAE feature transitions for OOD detection, connecting mechanistic interpretability to practical deployment needs.
  • Strong near-OOD performance on semantically coherent domains (AGNews), outperforming the LR baseline.
  • Built-in interpretability: decoded SAE features explain detection decisions, and the case study convincingly shows "errors" are semantically meaningful boundary cases.
  • Lightweight: no fine-tuning, no OOD data, simple Markov backend suffices.
  • Honest discussion of limitations (CLINC150 failure, SAE resolution constraints).

Limitations

  • Does not match LR baseline performance on far-OOD detection (FPR@95 is substantially worse: 0.11–0.62 vs. 0.003–0.017).
  • Fails on fine-grained intent taxonomies (CLINC150 with 150 classes) due to representational aliasing in SAE features.
  • Only evaluated on Gemma-2 with 16k SAEs; unknown how results transfer to other architectures or larger SAE dictionaries.
  • Code not yet available at time of publication.
  • No evaluation under adversarial attacks or gradual distribution drift.

Relevance

本文与LLM可解释性研究高度相关，通过SAE特征转移模式揭示了LLM内部处理输入的动态演化过程，为理解模型决策机制提供了新视角。该方法在agent系统的输入过滤和安全部署方面具有实际应用价值。

Questions for Further Exploration

  • How does the method scale to larger SAE dictionaries (e.g., 65k or 131k features)? Would higher-resolution SAEs resolve the CLINC150 failure mode?
  • Can transition-based scoring be combined with the LR baseline to achieve both strong detection performance and interpretability?
  • How robust is the method to adversarial inputs deliberately crafted to mimic ID-like feature transitions?
  • Does the mid-layer domain concentration finding generalize across model families (Llama, Mistral, etc.) and model scales?
  • Could the SAE transition framework be extended to real-time monitoring of LLM generation quality, not just input filtering?
