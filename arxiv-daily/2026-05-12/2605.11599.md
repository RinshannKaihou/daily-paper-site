> **一句话总结：** 本文提出了一套审计约束的定向推理测试协议，通过有限扰动语法生成提示变体并经语义与抽取审计后确认模型错误，但其实验发现自适应采样策略（CAPS）在匹配预算下并未优于均匀采样。

> **TL;DR:** This paper proposes an audit-constrained protocol for targeted reasoning evaluation of LLMs, requiring semantic validity, extraction validity, and adjudicated model-error labels before counting failures. Its proxy-guided sampler (CAPS) does not outperform uniform sampling under matched budgets, making the methodological contribution more important than the sampler itself.

## 核心贡献

- 提出了**审计约束的定向推理测试协议**，将提示变体生成、语义审查、抽取审查和答案裁定四个环节分离，确保自动代理不匹配仅作为路由信号而非模型错误声明。
- 设计了**CAPS（Component-Adaptive Prompt Sampling）**，一种基于分数的分类采样器，利用观察到的失配信号自适应调整提示组件采样概率，并与等预算均匀采样进行匹配对照。
- 建立了**冻结任务库 + 确定性渲染器 + 匹配预算**的完整工件合约，使每个采样候选可重建、可审查。
- 实验在三个审计切片（v2/v3/v4）中确认了模型错误提示键的存在，但**未发现CAPS在审计收益率或唯一提示键发现上优于均匀采样的证据**。
- 强调评估应基于审计收益率而非原始失配计数或精选示例，对领域内常见的过度声明提出了方法论上的纠正。

## Key Contributions

- **Audit-constrained targeted-test protocol** that separates automatic proxy mismatches (routing signals) from resolved model-error claims, requiring semantic validity, extraction validity, and answer-adjudication before counting failures.
- **CAPS**, a score-based categorical sampler over discrete prompt components (format, distractor, reading-cue), with exponential-moving-average score updates and a temperature-decay schedule.
- **Matched comparison framework** where CAPS and uniform component sampling share the same task bank, renderer, model adapter, answer extractor, query budget, and audit path — differing only in component sampling.
- **Negative result**: across v2 (3 models, 270 responses), v3 (Qwen, 90 responses), and v4 neutral-prior (Qwen, 192 responses), CAPS does not improve audited yield, query-to-first-audited-failure, or unique audited prompt keys over uniform sampling.
- **Methodological contribution**: the paper argues proxy-guided policies should be judged by audited yield, not raw mismatch counts, and demonstrates that small structured grammars already let uniform sampling enter brittle prompt regions.

## Methodology

The protocol operates on four auditable objects:

1. **Frozen task bank** $T$: 12–18 hand-authored reasoning tasks across three families (arithmetic with shortcuts, symbolic ordering/relation tracking, short abstract-rule tasks). Each task is a tuple $(id, f, q, a, v)$.

2. **Finite prompt-component grammar** $C = C_{format} \times C_{distractor} \times C_{reading}$: format (direct answer / explain / check shortcut), distractor (none / irrelevant number / plausible shortcut sentence), reading-cue (canonical / reversed / interleaved instructions). A deterministic renderer $R: T \times C \to P$ maps each task-component pair to a rendered prompt.

3. **Validity contract**: Structural validity $V_{auto}$ checks prompt length, task-anchor retention, answer-key presence. Semantic validity $V_{sem}$ requires the rendered prompt to preserve the task's entities, relations, operations, and answer equivalence class.

4. **Audit pipeline**: Automatic mismatch $F_M(x) = \mathbb{1}[\text{norm}(E(M(R(t,c)))) \neq \text{norm}(a_t)]$ routes candidates for review. Audited model error $A_M(x) = 1$ is assigned only after semantic validity + extraction validity + answer non-equivalence are all confirmed.

**CAPS** maintains per-component scores $s_{g,j}$ updated via exponential moving average ($\eta = 0.05$) on observed routing signals. Sampling probabilities use softmax-like normalization with temperature decay $\tau_i = \max(0.2, 1 - i/I)$. The primary endpoint is **audited yield** $Y(A) = \frac{1}{Q}\sum_{x \in A} A_M(x)$.

Models tested: Qwen2.5-7B-Instruct, Phi-3-mini-4k-instruct, Mistral-7B-Instruct-v0.3 on A100 GPUs.

## Results & Highlights

- **v2 (3 models, 12-task bank)**: 33 proxy mismatches → 32 confirmed model-error rows after audit (1 extraction artifact excluded). CAPS: 15 confirmed rows; Uniform: 17 confirmed rows. Uniform matches or exceeds CAPS.
- **v3 (Qwen, 18-task bank)**: 6 proxy mismatches → 5 confirmed model-error rows (1 answer-format artifact excluded). CAPS: 2 rows / 2 unique keys; Uniform: 3 rows / 3 unique keys. Uniform has larger audited count.
- **v4 neutral-prior (Qwen, 18-task bank)**: 26 proxy mismatches → 12 unique confirmed prompt keys (24 record-level rows). 11 of 13 unique keys are shared between strategies. All 4 seed-pair best-score comparisons are ties.
- **Fixture validation**: Synthetic adapter confirms artifact pipeline correctness (both CAPS and uniform reach mean best score 0.8115, 5 ties).
- Key insight: the protocol successfully identifies confirmed model-facing prompt failures across arithmetic, symbolic-relation, and counterfactual tasks, but uniform sampling already finds them under matched budgets.

## Critical Assessment

- **Novelty**: Moderate. The protocol formalizes good practices (semantic audit, extraction audit, matched budgets) that arguably should be standard but are rarely enforced together. CAPS itself is a straightforward bandit-style sampler; its negative result is honest but unsurprising given the small component grammar.
- **Rigor**: High. The paper is commendably careful: it separates proxy signals from adjudicated labels, reports negative results prominently, uses fixture adapters to validate the pipeline, and provides full artifact manifests. The audit rubric (Table 3) is explicit and restrictive.
- **Reproducibility**: Good within its scope. Configs, task-bank cards, trajectory files, and audit templates are documented. However, v2 results lack commit-level model revision SHAs (only identifier-level), and spot GPU availability introduces non-determinism in execution environment. The public release package sanitizes raw model responses, limiting exact rerun verification.
- **Significance**: The methodological contribution (audit-constrained protocol) is valuable for the evaluation community, even though the empirical finding is negative. The paper serves as a corrective to studies that report prompt-sensitivity findings without matched controls or semantic audit.
- **Red flags**: The task banks are very small (12–18 hand-authored tasks), limiting generalizability. The audit is protocol-level adjudication by a small number of reviewers, not an independent crowd-labeled dataset, so inter-rater reliability is not reported. The binary routing signal makes best-score comparisons uninformative once both strategies find at least one error.

## Strengths

- Exceptional honesty about negative results — the paper's main empirical finding is that CAPS does not outperform uniform sampling, stated clearly and repeatedly.
- Clean separation of proxy mismatches from audited failure labels prevents common confounds in prompt-sensitivity studies.
- Matched-budget comparison design is methodologically sound and should be a model for future work.
- Comprehensive artifact documentation and traceability.
- The framework is generalizable to other task banks and perturbation grammars.

## Limitations

- Very small task banks (12–18 tasks) — results cannot support broad claims about LLM reasoning robustness.
- Only three open-weight models tested (all ~7B parameter scale); no proprietary models (GPT-4, Claude, etc.).
- The prompt-component grammar is intentionally narrow (3 groups × 3 options each = 27 combinations per task), which limits prompt-space coverage.
- Audit is not an independent crowd-labeled dataset; inter-rater agreement statistics are not reported.
- Binary scoring (correct/incorrect) cannot capture partial reasoning or right-answer-wrong-reason cases.

## Relevance

该论文提出了审计约束的LLM推理评估协议，对构建可靠推理基准和评估提示敏感性具有直接方法论参考价值，其"审计优先于原始计数"的原则适用于任何系统性LLM评估工作。

## Questions for Further Exploration

- How does the protocol scale to larger task banks (100+ tasks) and broader prompt-component grammars? Does uniform sampling still match adaptive methods at scale?
- Would CAPS show an advantage with more nuanced scoring (e.g., partial credit, reasoning-chain evaluation) instead of binary correct/incorrect?
- How do the results change with frontier proprietary models (GPT-4, Claude, Gemini) or larger open models (70B+)?
- Can the audit-constrained protocol be combined with automated semantic validity checks (using a separate LLM as auditor) to reduce manual review burden?
- What is the relationship between task-bank diversity and the generalizability of discovered brittle prompt keys?
