CauTion: Knowing When to Trust LLMs for Ensemble Causal Discovery

arXiv: 2606.03602

Date: 2026-06-03

https://arxiv.org/abs/2606.03602

一句话总结： CauTion 通过三阶段框架（算法集成共识投票、无标注信任校准仲裁、循环修复）将 LLM 领域知识可靠地融入统计因果发现，仅在算法证据不可靠的争议边上调用 LLM，在六个基准数据集上持续超越纯数据驱动和 LLM 增强基线，且对 LLM 模型选择和图规模具有强鲁棒性。
TL;DR: CauTion is a three-stage framework that reliably integrates LLM domain knowledge into an ensemble of statistical causal discovery algorithms via consensus voting and trust-calibrated arbitration, restricting LLM queries to disputed edges with unreliable algorithmic evidence. It consistently outperforms both data-centric and LLM-augmented baselines across six benchmarks, with larger gains on larger graphs and strong robustness to LLM model selection.

核心贡献

  • 算法集成与共识过滤：聚合 PC、GES、CAMML 等多种因果发现算法的输出，通过加权共识投票将变量对划分为共识集（~90%）和争议集（~10%）。共识边直接加入图，在 Win95pts（n=76）上共识覆盖率达 96.6%，准确率 99.6%。
  • 无标注信任校准机制：利用共识边作为代理标签估计 LLM 在存在性判断和方向判断任务上的准确率；通过留一法（Leave-One-Out）评估各算法的可靠性；计算信任权重 α 来量化 LLM 与算法集的相对可信度。
  • 受限的 LLM 仲裁：仅在争议边上、且算法投票差距低于信任阈值时调用 LLM；存在性和方向性判断分别独立进行信任加权投票，进一步降低 LLM 调用成本和错误暴露。
  • 循环修复保证 DAG 约束：检测并修复投票产生的循环，优先修复算法不确定性高的边（由 LLM 建议翻转或删除），确保最终输出为合法有向无环图。
  • 广泛的实验验证与模型鲁棒性：在六个 bnlearn 基准（Cancer 到 Win95pts，n=5~76）上全面领先；跨六种 LLM（Claude-Sonnet-4.6、GPT-5.2、Llama-3.3-70B、DeepSeek-V3.2、Qwen3-8B、Llama-3.1-8B）测试表现出最低的性能波动。

Key Contributions

  • Algorithm Ensemble with Consensus Filtering: Aggregates outputs from PC, GES, and CAMML via weighted consensus voting, partitioning variable pairs into a consensus set (~90%) and a disputed set (~10%). On Win95pts (n=76), consensus coverage reaches 96.6% with 99.6% accuracy.
  • Annotation-Free Trust Calibration: Uses consensus edges as proxy labels to estimate LLM accuracy on edge-existence and edge-direction tasks; employs leave-one-out evaluation to estimate per-algorithm reliability; computes trust weight α to quantify the relative credibility of LLM versus the algorithm ensemble.
  • Restricted LLM Arbitration: Queries the LLM only on disputed edges where the algorithm vote margin falls below the trust threshold; existence and direction are resolved independently via trust-weighted voting, further reducing LLM query cost and error exposure.
  • Cycle Repair for DAG Constraint: Detects and repairs cycles in the voted graph, prioritizing edges with high algorithmic uncertainty for LLM-suggested flips or removals, guaranteeing a valid directed acyclic graph.
  • Extensive Evaluation and Cross-Model Robustness: Leads all baselines across six bnlearn benchmarks (Cancer to Win95pts, n=5–76); tested across six LLM backends with the lowest performance variance, demonstrating strong robustness to LLM selection.

Methodology

Problem Setup. Given n variables V = {X₁, ..., Xₙ} and a set of causal discovery algorithms A, each algorithm A ∈ A produces a partially directed graph. For every unordered pair e = {i, j}, the output is mapped to one of four vote categories: fwd (i→j), rev (j→i), undir (i–j), or none. The goal is to aggregate these votes into a single directed acyclic graph (DAG).

Stage 1: Algorithm Ensemble. For each variable pair, weighted votes are accumulated across algorithms, with undirected votes contributing 0.5 to both directions. A pair is classified as Consensus-Fwd, Consensus-Rev, or Consensus-None if all algorithms agree; otherwise it enters the disputed set D. Consensus edges are resolved directly without LLM queries, while disputed edges proceed to arbitration.

Stage 2: Trust-Calibrated Arbitration.

  • LLM Calibration. The LLM first infers a domain description and semantic interpretations for each variable (e.g., "Dyspnoea [shortness of breath]"), which are injected into all edge-level prompts. To mitigate position bias, direction queries are issued twice per pair (A→B and B→A) and only consistent answers are retained. n_calib = 30 consensus edges are sampled, stratified by the estimated edge-existence distribution over D. Separate accuracy estimates are computed for existence (acc_LLM,exist) and direction (acc_LLM,dir), reweighted to the target distribution to remove sampling bias.
  • Algorithm LOO Calibration. Each algorithm's accuracy is estimated via leave-one-out evaluation: for each pair, the algorithm's prediction is compared against the majority vote of all remaining algorithms (ties excluded). For existence, the fraction of proxy-exist and proxy-none pairs correctly predicted is reweighted by the target distribution. For direction, accuracy is computed among pairs where the algorithm votes a definite direction. The ensemble accuracy is aggregated as the mean of algorithms at or above the median performance.
  • Trust Weights. Given calibrated accuracies, two independent trust weights α_exist and α_dir are computed as a power-ratio of above-chance accuracies with exponent β = 3, sharpening the contrast between sources of similar accuracy. If both sources perform at or below chance (0.5), fallback to α = 0.5.
  • Calibrated Voting. For each disputed edge, existence is resolved first. Algorithm votes are aggregated into a weighted existence score v_exist, using per-algorithm accuracy as weights. The LLM is consulted only when the algorithm margin m_exist = |v_exist – v_none| ≤ α_exist/(1−α_exist). The combined score q_exist = (1−α_exist)·v_exist + α_exist·1[LLM=exist]. If the edge exists, direction is resolved similarly using only algorithms that predict a definite direction, with the same margin-based LLM query condition.

Stage 3: Cycle Repair. The voting procedure may produce cycles. For each edge on a detected cycle, the weighted existence and direction margins are computed. An edge is a repair candidate if m_exist < θ_exist or m_dir < θ_dir (where θ = α/(1−α)). Repair candidates are presented to the LLM together with the cycle path and variable interpretations; the LLM may choose to Flip (reverse direction, if m_dir < θ_dir) or Remove (delete edge, if m_exist < θ_exist). Actions are applied immediately and the cycle check repeats. If actions are exhausted, remaining cycle edges are removed in ascending order of v_exist until acyclic.

Results & Highlights

Datasets and Setup. Six discrete-variable benchmarks from bnlearn: Cancer (n=5, |E|=4), Insurance (n=27, |E|=52), Water (n=32, |E|=66), Alarm (n=37, |E|=46), Barley (n=48, |E|=84), and Win95pts (n=76, |E|=112). Each dataset uses 5000 sampled observations. Metrics: Structural Hamming Distance (SHD, lower is better), F1 over directed edges (higher is better), and Structural Intervention Distance (SID, lower is better). LLM backend: Claude-Sonnet-4.6. Ensemble: PC, GES, CAMML.

Main Results (Table 1). CauTion achieves the best SHD/F1/SID on all six datasets among all baselines. Notable highlights:

  • Win95pts (largest graph): SHD = 27 vs. 63 for the second-best LLM-augmented baseline (SCP), and vs. 42 for the best data-centric baseline (BOSS). F1 = 85.2% vs. 64.6% (SCP) and 79.7% (GES).
  • Alarm: SHD = 4, F1 = 94.8% — near-perfect recovery.
  • Insurance: SHD = 11, F1 = 87.4% — substantial lead over all baselines.
  • Water: SHD = 36, F1 = 63.3% — only method to exceed 60% F1.
  • No single data-centric algorithm dominates across datasets; each excels on some and fails on others, confirming algorithm-specific bias. CauTion mitigates this via ensemble aggregation.

Scalability. Gains become more pronounced as graph scale grows. On Win95pts, the SHD gap over the second-best method (SCP, SHD=63) is larger than 2×. This aligns with the framework design: larger graphs yield more edges, amplifying the value of consensus filtering and selective LLM integration.

Efficiency. Compared to global LLM methods, CauTion dramatically reduces LLM query volume. SCP incurs ~100× higher token cost. LLM-MEC, while having low token cost, suffers 8× higher runtime due to MEC computation complexity. CauTion restricts LLM queries to the small disputed set (~10% of pairs) and further skips queries when the algorithm vote margin exceeds the trust threshold.

Ablation Study (Figure 3).

  • Consensus Only: Discarding all disputed edges leads to poor performance, falling behind data-centric baselines.
  • Majority Vote: Outperforms individual algorithms, confirming ensemble aggregation mitigates single-algorithm bias, but remains constrained by statistical limitations without LLM domain knowledge.
  • LLM Only: Competitive on small graphs (Cancer) but degrades sharply on larger graphs (Win95pts SHD=233, F1=23%).
  • Cons + LLM: Combining consensus with uncalibrated LLM arbitration improves over LLM-only but overestimates LLM reliability on disputed edges, leading to suboptimal results on Cancer, Alarm, and Win95pts compared to Majority Vote.
  • CauTion (full): Outperforms both Majority Vote and Cons+LLM, demonstrating that calibrated trust weighting effectively balances algorithm ensemble and LLM strengths.

Robustness to LLM Model Selection (Figure 4). Tested across six LLMs: Claude-Sonnet-4.6, GPT-5.2, Llama-3.3-70B-Instruct, DeepSeek-V3.2, Qwen3-8B, and Llama-3.1-8B-Instruct. CauTion achieves the lowest SHD on all datasets with the smallest cross-model performance fluctuation. In contrast, fully LLM-driven methods show severe sensitivity: CORR-LLM ranges from SHD 13 to 562 on Alarm across backends; LLM-BFS ranges from 100 to 459 on Barley. This confirms that uncalibrated LLM reliance is fragile in practice.

Critical Assessment

  • Novelty: The paper makes a genuine conceptual contribution by framing LLM-augmented causal discovery as a trust-calibrated arbitration problem rather than simply injecting LLM knowledge. The three-stage design — consensus filtering to reduce decision volume, annotation-free calibration to estimate source reliability, and margin-thresholded voting to restrict LLM intervention — is well-motivated and addresses clear limitations of prior work (high token cost, LLM error vulnerability, single-algorithm bias). The use of consensus edges as proxy labels for calibration is particularly elegant.
  • Rigor: The experimental design is thorough. Six benchmarks spanning n=5 to n=76, three metrics, comparisons against 9 data-centric and 4 LLM-augmented baselines, ablation studies, and cross-LLM robustness tests provide strong empirical support. The statistical reporting (averaged over 5 runs) is standard. However, all experiments use discrete-variable synthetic/benchmark networks; no real-world or continuous-data validation is provided. The claim of "near-perfect accuracy" on consensus edges is well-supported for the tested datasets but may not generalize to noisier domains.
  • Reproducibility: Code is open-sourced, which is excellent. However, exact reproduction depends on API access to specific LLM versions (Claude-Sonnet-4.6, GPT-5.2, etc.) and their prompt behavior, which may drift over time. The paper describes the prompt pipeline (domain description → variable interpretation → edge-level queries) but does not include full prompts in the main text, making precise reproduction partially dependent on the released codebase.
  • Significance: If the robustness claims hold beyond benchmark networks, CauTion could become a practical default for causal discovery from observational data. The cost-saving aspect (reduced LLM queries) and the robustness to LLM backend selection are practically important. The work also contributes a principled way to combine multiple imperfect evidence sources — relevant beyond causal discovery.
  • Red flags: (1) The ensemble is fixed to PC, GES, and CAMML — three algorithms that happen to perform well on these benchmarks. The paper acknowledges this as a limitation but does not explore sensitivity. (2) The trust calibration assumes LLM domain knowledge is uniformly distributed across consensus and disputed edges; this may fail if disputed edges systematically involve more ambiguous or out-of-domain causal relationships. (3) On the smallest dataset (Cancer, n=5), many methods including CauTion achieve perfect SHD=0, limiting the discriminative power of small-scale experiments. (4) The direction accuracy estimate depends on "consistent answers" from double-framing; if the LLM is inconsistent, the direction decision falls back to algorithms, but the paper does not quantify how often this occurs.

Strengths

  • Addresses three real problems simultaneously: LLM error vulnerability, high token costs, and single-algorithm bias.
  • The annotation-free trust calibration removes the need for ground-truth causal graphs, which are rarely available in practice.
  • Consensus filtering dramatically reduces the LLM query space (~90% of edges resolved without LLM), making the method economically viable.
  • Strong cross-LLM robustness demonstrates practical deployability regardless of which model is available.
  • The modular three-stage pipeline is conceptually clean and could be extended with different ensemble algorithms or LLM backends.
  • Comprehensive empirical evaluation with multiple metrics, ablations, and sensitivity analyses.

Limitations

  • Fixed algorithm ensemble: Currently uses only PC, GES, and CAMML. Incorporating a broader or more diverse set could improve consensus quality, but also increases computational cost.
  • Discrete-variable only: All experiments use discrete-variable benchmarks from bnlearn. Generalization to continuous or mixed-type data remains unvalidated.
  • Dependence on algorithm agreement: Performance may degrade on datasets where algorithms rarely agree (low consensus rate), though the paper shows consensus rates are high on tested benchmarks.
  • LLM calibration assumptions: The uniform distribution assumption for LLM knowledge across consensus and disputed edges may not hold in all domains.
  • Computational cost: While LLM query cost is low, running multiple causal discovery algorithms (PC, GES, CAMML) adds computational overhead compared to single-algorithm approaches.
  • Limited small-graph discrimination: On Cancer (n=5), most methods including baselines achieve perfect scores, making it hard to differentiate performance at small scales.

Relevance

CauTion 为将大型语言模型的领域知识可靠地融入统计因果发现提供了系统性的解决方案，其信任校准和共识过滤机制对于希望在因果推断 pipeline 中安全、经济地使用 LLM 的研究者具有很高的参考价值，同时其在六个标准基准上的全面领先表现也为因果发现领域引入了新的实用范式。

Questions for Further Exploration

  1. How sensitive is CauTion's performance to the specific choice and number of algorithms in the ensemble? Would adding continuous-optimization methods (e.g., NOTEARS, DAGMA) improve or hurt consensus quality?
  2. What is the computational overhead of running the full ensemble compared to single-algorithm approaches, and how does it scale to graphs with hundreds of variables?
  3. How does the trust calibration perform when ground-truth causal graphs are partially available — can it leverage even limited annotations to improve accuracy?
  4. Does the method generalize to causal discovery with latent confounders (e.g., FCI-style settings), time-series data, or mixed continuous/discrete variables?
  5. How robust is the approach to different prompt designs and variable naming conventions? Would performance degrade significantly with less informative variable names?
  6. Can the cycle repair step be theoretically guaranteed to terminate and produce a valid DAG without excessive edge removal, and can its decisions be made more principled?
