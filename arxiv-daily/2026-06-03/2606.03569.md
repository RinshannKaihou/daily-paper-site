> **一句话总结：** 本文提出 Structure-to-Semantics (STS) 两阶段视觉 token 剪枝框架，通过第一阶段基于势能最小化的排斥采样保留空间结构多样性，第二阶段在 LLM 中间层利用指令感知交叉注意力过滤任务无关 token，在 88.9% 压缩率下仍保持 96.8%–99.4% 的相对性能。

> **TL;DR:** This paper introduces STS, a training-free two-stage visual token pruning framework that first maximizes spatial and structural diversity via potential-energy-minimizing repulsion-based sampling, then filters prompt-irrelevant tokens using instruction-aware cross-attention at an intermediate LLM layer, achieving 96.8%–99.4% relative performance at 88.9% compression across LLaVA and Qwen2.5-VL architectures.

## 核心贡献

- 揭示了现有视觉 token 剪枝方法的核心缺陷：单一注意力分数机制导致注意力坍缩到语义相似区域，严重降低特征多样性并丢弃关键上下文细节。通过 KNN 一致性分数 (Consistency Score) 定量验证了该现象在深层视觉编码器中随深度加剧。
- 提出 Structure-to-Semantics (STS) 两阶段剪枝框架，将剪枝过程显式解耦为结构保留阶段（Pre-LLM）和语义精炼阶段（Intra-LLM），且无需额外训练。
- 第一阶段采用基于静电排斥原理的势能最小化目标进行贪婪采样，将视觉 token 建模为特征空间中相互排斥的带电粒子，确保保留 token 在特征空间中广泛分布，避免冗余聚类。
- 第二阶段在 LLM 中间层（LLaVA 系列为第 16 层，Qwen2.5-VL 为第 14 层）利用最后一个文本 token 对视觉 token 的交叉注意力权重进行任务感知过滤，保留与指令最相关的 token。
- 在 LLaVA-1.5-7B/13B、LLaVA-NeXT-7B 和 Qwen2.5-VL-7B 四个代表性模型上，于 GQA、MMBench、MME、POPE、ScienceQA、TextVQA、VQAv2、VizWiz 八个基准上进行广泛评估，在 128/64/32 token 预算下均取得最优或接近最优的平均性能。

## Key Contributions

- Reveals and quantifies a critical flaw in existing visual token pruning methods: attention scores collapse onto semantically similar regions, severely reducing feature diversity. This "attention collapse" phenomenon is validated using a KNN-based Consistency Score (C), showing that feature-attention alignment increases markedly in deeper vision encoder layers.
- Proposes Structure-to-Semantics (STS), a training-free, stage-aware visual token pruning framework that explicitly decouples pruning into a pre-LLM structure-preservation stage and an intra-LLM semantics-refinement stage.
- Stage 1 employs a potential-energy-minimization objective inspired by electrostatic repulsion theory. Visual tokens are modeled as mutually repulsive charged particles in feature space, and a greedy algorithm selects tokens that minimize cumulative repulsive potential, ensuring broad global coverage.
- Stage 2 applies task-aware filtering at an intermediate LLM layer (L_prune = 16 for LLaVA, 14 for Qwen2.5-VL), using cross-attention weights from the last textual token to visual tokens to retain only instruction-relevant tokens.
- Extensive experiments across four representative VLMs (LLaVA-1.5-7B/13B, LLaVA-NeXT-7B, Qwen2.5-VL-7B) and eight benchmarks demonstrate state-of-the-art efficiency–performance trade-offs, achieving 99.4% relative performance at 77.8% compression and 96.0% at 94.4% compression on LLaVA-1.5-7B.

## Methodology

STS is a training-free two-stage framework designed for visual token pruning in Vision-Language Models (VLMs).

### Stage 1: Pre-LLM Pruning via Global Potential Energy Minimization

This stage operates on visual tokens from the vision encoder before they enter the LLM. Since textual instructions are unavailable at this point, the goal is to preserve broad structural coverage rather than task relevance.

**Potential Energy Modeling:**
- Visual tokens are embedded into an L2-normalized feature space.
- The squared Euclidean distance between tokens vi and vj is: d_ij = ||vi - vj||^2_2
- The cumulative repulsive potential experienced by candidate token vi with respect to the currently selected subset S_t is:
  U(vi | S_t) = Σ_{vj∈S_t} 1 / (d_ij + ε)
  where ε is a small constant for numerical stability.
- This formulation assigns larger penalties to candidates close to already-selected tokens, actively discouraging redundant selections from densely populated feature-space regions.

**Iterative Greedy Selection:**
- Initialize S_0 with an anchor token (either the token closest to the global feature mean or the highest-saliency token).
- At each step, select the candidate experiencing minimum repulsive potential:
  v_next = argmin_{v∈V\S_t} U(v | S_t)
- This encourages retained tokens to be well-separated in feature space, reducing redundancy while maintaining diverse visual information.

### Stage 2: Intra-LLM Pruning via Task-Aware Filtering

As tokens propagate through the LLM, the model shifts from broad contextual integration to selective focus on task-relevant information. STS exploits this transition by pruning at an intermediate "sweet spot" layer.

**Task-Aware Relevance Scoring:**
- Let S denote tokens retained after Stage 1, and t_last denote the final textual token in the input sequence.
- Semantic relevance of each visual token v ∈ S is estimated using the attention weight from this last textual token:
  R(v) = A_{t_last, v}
  where A is the attention weight at layer L_prune.

**Final Selection:**
- The final retained token set is: S_final = TopK_{v∈S} R(v), K'
- This removes semantically irrelevant tokens while preserving those most relevant to the current instruction.

**Implementation Notes:**
- Default intra-LLM retention ratio: ρ_intra = 33.3%
- For an average budget of 128 tokens: retain 192 tokens pre-LLM, then reduce to 64 tokens via intra-LLM pruning.
- Compatible with FlashAttention by recomputing attention of the last instruction token outside original LLM layers.

## Results & Highlights

### Main Results on LLaVA-1.5-7B (Table 1)

| Budget | Relative Performance | Key Achievement |
|--------|---------------------|-----------------|
| 128 tokens (↓77.8%) | **99.4%** | Best average; outperforms AgilePrune, Zoo-Prune, DivPrune, DART, SparseVLM, VisionZip |
| 64 tokens (↓88.9%) | **98.0%** | Best on GQA, MMB, MME, POPE, VQAv2, VQAText |
| 32 tokens (↓94.4%) | **96.0%** | Surpasses AgilePrune by 1.8 points; strong robustness under extreme compression |

### Results on LLaVA-NeXT-7B (Table 2)

| Budget | Relative Performance | Notes |
|--------|---------------------|-------|
| 640 tokens (↓77.8%) | **97.6%** | Slightly outperforms VisionZip and AgilePrune |
| 320 tokens (↓88.9%) | **96.8%** | Strong high-resolution compression |
| 160 tokens (↓94.4%) | **95.3%** | Outperforms Zoo-Prune by 1.8 points; best on GQA, MME, POPE, VQAv2 |

### Efficiency Analysis on LLaVA-NeXT-7B (Table 3)

At 88.9% token reduction (320 tokens retained from 2880):
- **FLOPs:** 41.7T → 4.2T (**9.9× reduction**)
- **Prefill latency:** 246 ms → 38 ms (**6.6× speedup**)
- **KV Cache:** 1440.0 MB → 160.0 MB (**9× reduction**)
- **Peak memory:** 16.7 GB → 13.8 GB
- **F1 Score:** 86.8 (baseline) → **87.7** (slightly exceeds unpruned baseline!)

### Ablation Study Highlights

- **Attention-based methods degrade aggressively:** FasterVLM drops to 73.0% on POPE at extreme budgets; FastV falls to 38.0% on POPE and 42.0% on GQA.
- **STS-S (Stage-1 only) outperforms DivPrune:** At aggressive ratios, STS-S achieves 82.0% vs. 81.0% on POPE, showing potential-energy objective is stronger than standard distance-based diversity.
- **Full STS improves over STS-S:** Task-aware filtering improves POPE from 82.0% to 85.4% at lowest budget, and reaches 87.2% at higher budgets (exceeding unpruned baseline of 85.9%).
- **Layer selection is robust:** Performance is relatively insensitive to exact pruning layer within the early-to-middle LLM layers.

## Critical Assessment

**Novelty:** The explicit decoupling of visual token pruning into structure-first, semantics-second stages is a conceptually elegant contribution. The "attention collapse" observation is not merely qualitative — the authors introduce a KNN-based Consistency Score to quantify it across layers, lending empirical rigor to the motivation. The potential-energy-inspired repulsion mechanism, while drawing on established physics analogies, is novel in its application to visual token selection for VLMs. The identification of intermediate LLM layers as the optimal "pruning bottleneck" is a useful insight backed by ablation.

**Rigor:** The experimental design is commendable. Evaluations span four diverse VLM architectures (fixed-resolution, high-resolution, dynamic encoding), eight benchmarks, and three compression levels. Comparisons include seven recent strong baselines from top venues. The efficiency analysis goes beyond accuracy to report FLOPs, prefill/decode latency, KV cache, and peak memory. The ablation study systematically isolates each stage's contribution. However, some minor concerns: (1) The paper does not report standard deviations or confidence intervals across runs. (2) The KNN Consistency Score analysis, while insightful, is shown for LLaVA-1.5 only — it would strengthen the claim to see it on other architectures. (3) The "slightly exceeding unpruned baseline" result (87.7 vs. 86.8 F1 on LLaVA-NeXT-7B) is intriguing but may warrant deeper investigation — is it due to noise or does pruning genuinely act as a regularizer?

**Reproducibility:** The method is training-free and conceptually simple, which is highly favorable for reproducibility. The authors provide hyperparameters (L_prune = 16 for LLaVA, 14 for Qwen2.5-VL; ρ_intra = 33.3%). However, no code or checkpoints are mentioned in the extracted text. The greedy selection algorithm is straightforward to implement, but exact reproduction of results depends on implementation details (e.g., ε value, anchor token selection strategy) that may be in appendices not fully extracted.

**Significance:** Visual token pruning is a high-impact area for VLM deployment. If the claims hold under broader evaluation, STS could become a practical drop-in efficiency module for VLMs. The 9.9× FLOP reduction and 6.6× prefill speedup with maintained or even slightly improved performance is compelling for real-time and resource-constrained applications. The training-free nature makes it immediately applicable to existing models without expensive fine-tuning.

**Red Flags:**
- The "exceeds unpruned baseline" result, while exciting, should be interpreted cautiously without statistical significance testing.
- The KNN Consistency Score validation is limited to one model family in the main text.
- The potential-energy greedy algorithm has O(NK) complexity per selection step; while the paper claims efficiency, the computational cost of Stage 1 relative to simple attention sorting is not explicitly broken down.
- The limitation section honestly acknowledges performance loss and black-box inapplicability, which is a positive sign of intellectual honesty.

## Strengths

- **Conceptually clean framework:** The two-stage decoupling (structure → semantics) is intuitive, well-motivated, and effectively validated through ablations.
- **Strong empirical motivation:** The attention-collapse phenomenon is not just asserted but quantified with the KNN Consistency Score across layers.
- **Training-free and broadly applicable:** Works across multiple VLM architectures without requiring fine-tuning or model modification.
- **Comprehensive evaluation:** Four models, eight benchmarks, multiple compression ratios, and thorough efficiency metrics (FLOPs, latency, memory, KV cache).
- **State-of-the-art trade-offs:** Achieves 99.4% relative performance at 77.8% compression and 96.0% at 94.4% compression, outperforming seven strong baselines.
- **Practically significant speedups:** 9.9× FLOP reduction and 6.6× prefill latency improvement on high-resolution models.
- **Well-written and structured:** Clear progression from empirical analysis → method → experiments → ablations → efficiency analysis.

## Limitations

- **Not lossless:** The authors honestly acknowledge that STS is not entirely without performance loss, though the degradation is minimal.
- **Black-box inapplicability:** Requires access to internal visual tokens and intermediate LLM representations, making it incompatible with proprietary APIs (GPT-4V, Claude, etc.) that do not expose these internals.
- **Potential Stage 1 overhead:** The potential-energy greedy selection has higher computational cost than simple attention-score sorting. While the overall efficiency gains are strong, the exact overhead of repulsion-based sampling is not isolated in the efficiency analysis.
- **Limited statistical rigor:** No standard deviations or multiple-run averages are reported, making it difficult to assess whether small improvements (e.g., exceeding the unpruned baseline) are statistically significant.
- **KNN Consistency Score generalization:** The quantitative validation of attention collapse is shown for LLaVA-1.5; broader validation across diverse architectures would strengthen the claim of universality.
- **Fixed hyperparameters:** L_prune and ρ_intra are set heuristically. While the paper shows robustness to layer choice, an adaptive or learned pruning layer/strategy might yield further gains.

## Relevance

本文提出的两阶段视觉 token 剪枝框架对于提升视觉-语言模型推理效率具有重要意义，特别适合需要高效多模态推理的研究与部署场景。STS 无需训练即可实现 9.9 倍 FLOP 缩减和 6.6 倍预填充延迟加速，同时保持接近原始模型的任务性能，使其成为现有 VLM 部署中极具实用价值的即插即用效率模块。若用户对模型压缩、高效推理或多模态系统优化感兴趣，该工作提供了兼具理论洞察与工程实用性的解决方案。

## Questions for Further Exploration

1. Can the potential-energy repulsion mechanism be extended to video token pruning, where temporal redundancy introduces additional structure beyond spatial diversity?
2. Would an adaptive or learned pruning layer selection strategy (rather than fixed L_prune = 16/14) yield further efficiency or accuracy improvements, especially across varying input complexities?
3. Does the attention-collapse phenomenon quantified by the KNN Consistency Score generalize to other vision encoder architectures (e.g., SigLIP, EVA-CLIP, DINOv2) beyond the CLIP-based encoders used in LLaVA?
4. Can the two stages of STS be integrated with other efficiency techniques such as model quantization, speculative decoding, or KV cache compression for compound efficiency gains?
5. What is the statistical significance of STS occasionally exceeding the unpruned baseline (e.g., 87.7 vs. 86.8 F1 on LLaVA-NeXT-7B)? Is pruning acting as an implicit regularizer, or is this within run-to-run variance?
6. How does STS perform on emerging long-context multimodal tasks (e.g., multi-image reasoning, document understanding with hundreds of pages) where visual token counts are orders of magnitude larger?
