Beyond Ideal Instruction: A Comprehensive Framework for Evaluating LLMs in Realistic Interactions

arXiv: 2606.03318

Date: 2026-06-03

https://arxiv.org/abs/2606.03318

一句话总结： 本文提出了 RUT-Bench，一个包含 1638 个测试样本、覆盖 59 个可执行环境和 11 个真实领域的大语言模型工具调用评估基准，系统性地引入七种用户行为模式（包括理想行为与六种非理想行为），对 19 个主流模型进行评测后发现没有任何模型整体成功率超过 40%。
TL;DR: This paper introduces RUT-Bench, a benchmark with 1,638 test samples across 59 executable environments and 11 real-world domains, systematically modeling seven user behavior categories (ideal + six non-ideal types). Comprehensive evaluation of 19 mainstream LLMs reveals that no model achieves an overall success rate above 40%, underspecification causes the steepest degradation (~45% relative drop for proprietary models), and incorrect tool sequencing is the dominant failure mode across all model tiers.

核心贡献

  • 提出了 RUT-Bench，首个系统性地将真实世界非理想用户行为纳入评估的工具调用基准
  • 基于 WildChat 真实交互日志构建了七种用户行为分类体系：理想理性行为、信息不足（Underspecification）、信息过载（Information Overload）、虚构参数（Fabricated Parameters）、目标切换（Goal Switching）、矛盾约束（Contradictory Constraints）、不耐烦与敌意（Impatience and Hostility）
  • 构建了 59 个可执行环境、234 个基础任务，每个任务实例化为 1 个理想对话和 6 个非理想变体，共计 1638 个测试样本，覆盖医疗健康、金融、企业 CRM、DevOps、媒体流、社交平台、电商、游戏、体育、交通、旅游酒店等 11 个领域
  • 设计了多维评估体系：以任务成功率为主要指标，辅以信息诚实度（Informational Honesty）和工具纪律性（Tool Discipline）两个诊断性指标
  • 对 19 个主流开源和闭源模型进行了全面评估，揭示了所有模型在非理想用户场景下的显著性能退化

Key Contributions

  • Introduces RUT-Bench, the first tool-use benchmark that systematically incorporates realistic non-ideal user behaviors into evaluation
  • Constructs a seven-category taxonomy of user behaviors grounded in real interaction logs from WildChat: Ideal Rational Behavior, Underspecification, Information Overload, Fabricated Parameters, Goal Switching, Contradictory Constraints, and Impatience and Hostility
  • Builds 59 executable environments with 234 base tasks, each instantiated into one ideal dialogue and six non-ideal variants, yielding 1,638 test instances spanning 11 domains (healthcare, finance, enterprise/CRM, DevOps, media streaming, social platforms, e-commerce, gaming, sports, transportation, travel/hospitality)
  • Designs a multi-dimensional evaluation framework with overall task success rate as the primary metric, plus two diagnostic metrics — Informational Honesty and Tool Discipline — evaluated via LLM-as-a-Judge
  • Conducts comprehensive evaluation of 19 mainstream open-source and proprietary LLMs, revealing substantial performance degradation across all models under non-ideal user scenarios

Methodology

Behavior Taxonomy (Section 3). The authors first derive a fine-grained taxonomy of real-world user behaviors by analyzing 1,000 randomly sampled English dialogues from WildChat (a large corpus of real user-LLM interactions). Using GPT-5.4 as an annotator, they classify dialogues into seven categories adapted from Gricean pragmatics and speech act theory. Statistical analysis shows that 22.6% of dialogues fall into non-ideal categories, with Underspecification being most frequent (14.25%), followed by Information Overload (3.46%) and Fabricated Parameters (3.25%). Goal Switching (1.83%) and Impatience/Hostility (0.41%) arise predominantly in later turns of complex multi-turn interactions.

Benchmark Construction Pipeline (Section 4). The construction follows three stages with rigorous verification:

  1. Environment and Task Construction (4.1): Queries are curated from three established tool-use datasets (API-Bank, ToolAce, Dolci) and filtered through a statefulness filter to retain only queries requiring persistent environment interaction. For each query, a three-step process synthesizes executable Python environments: (i) LLM-generated formal specification (state space, toolset, conditions), (ii) compilation into executable Python classes, and (iii) static AST validation. Initial states are generated at three difficulty levels (easy: 25.1%, medium: 50.2%, hard: 24.7%) by varying entity counts. Tasks are annotated with oracle tool traces, optimal action budgets, state deltas, outcome assertions, and action constraint graphs.
  1. Behavior-Controlled Dialogue Generation (4.2): For each task, ideal multi-turn dialogues are generated by partitioning the oracle trace into segments and prompting an LLM to produce natural-language utterances encoding each segment's execution logic without leaking tool names or system states. Non-ideal variants are then created via controlled rewriting strategies specific to each behavior type (e.g., dropping essential slots for Underspecification, weaving unrelated side goals for Goal Switching, injecting contradictory conditions for Contradictory Constraints).
  1. Iterative Verification (4.3): All candidates undergo three-stage verification: (i) environment stability verification via exhaustive tool exploration and white-box state transition checking; (ii) task consistency and achievability verification by executing oracle traces against expected state changes; (iii) dialogue logical consistency and validity verification ensuring intent alignment, inferability of all parameters from context, and absence of data leakage.

Evaluation Protocol (Section 5). During evaluation, the agent receives a system prompt with environment description and available tools but no direct access to initial states. It must interactively infer necessary context through multi-turn dialogue, invoking APIs or replying in natural language until all queries are addressed or a step limit is reached. Task success is determined by three strict criteria: (1) action coverage — predicted trace contains all essential tools from the oracle; (2) logical compliance — tool call order respects precedence constraints in the action constraint graph; (3) state verification — final state satisfies all outcome assertions.

Results & Highlights

Main Results (Section 6.1, Figure 3 & Table 2).

  • No model exceeds 40% success rate. GPT-5.4 achieves the highest overall score at 37.30%, followed by Claude-Opus-4.6 (36.38%) and Gemini-3.1-Pro (35.53%).
  • Proprietary models consistently outperform open-source models. The average proprietary model scores 35.01% vs. 28.72% for open-source general models and 11.98% for efficient/lightweight models.
  • Scale sensitivity is pronounced. Medium-sized models trail frontier models by ~15 points, with degradation worsening for lightweight models. Notably, small dense models (Qwen-3.5-27B at 30.34%) perform on par with large sparse models (Qwen-3.5-397B-A17B at 33.33%), likely due to the latter's lower number of activated parameters.
  • High diagnostic scores do not guarantee task success. Despite strong Informational Honesty and Tool Discipline scores, frontier models exhibit low success rates because they violate strict procedural ordering (read-before-write precedence) and default to refusing ambiguous requests rather than proactively resolving uncertainty.

Impact of Non-Ideal Behaviors (Section 6.2).

  • Universal degradation. All models experience performance drops when transitioning from ideal to non-ideal scenarios. The magnitude correlates strongly with overall capability — proprietary models are most resilient, lightweight models suffer severe collapses.
  • Underspecification is the most challenging behavior. It triggers the steepest degradation across all tiers: proprietary models suffer a 45.2% relative drop (from 42.90% to 23.50%), while lightweight models plummet by nearly 60%.
  • Goal Switching and Impatience/Hostility are least disruptive. Models show stronger resilience to these behaviors, possibly because they do not fundamentally alter the information required for task completion.
  • Model-specific vulnerabilities. Claude series is particularly fragile against Contradictory Constraints (-33.0% for Sonnet), while Gemini series is more vulnerable to Fabricated Parameters (-31.7% for Pro). These distinct blind spots highlight the value of fine-grained behavioral evaluation.

Error Analysis (Section 6.3, Figure 4).

Failed trajectories from four representative models (GPT-5.4, GLM-5.1, Qwen-3.5-27B, Claude-Haiku) cluster into five failure modes:

  1. Incorrect Tool Sequence — universally dominant; models invoke state-modifying tools without prerequisite lookups, violating procedural dependencies.
  2. Premature Termination and Partial Completion — models are over-cautious, halting execution and relying on explicit user clarifications rather than proactive self-inference.
  3. Erroneous Parameter Assignment — lightweight models inject raw natural language into API arguments or hallucinate placeholder IDs instead of extracting precise environment identifiers.
  4. Unauthorized Tool Invocation — primarily affects weaker models lacking multi-step reasoning capabilities.
  5. Execution Paralysis — facing ambiguous constraints, safety-aligned models default to refusal rather than uncertainty resolution.

The evaluation covers 19 diverse models across three tiers (proprietary, open-source general, efficient/lightweight) with uniform inference settings (temperature 0, max 20 steps, Nvidia H200 GPUs). The behavioral taxonomy is grounded in real interaction logs from WildChat and validated via GPT-5.4 annotation, lending empirical credibility to the simulation design.

Critical Assessment

Novelty: Moderate-high. While tool-use benchmarks are plentiful (API-Bank, ToolBench, τ-bench, BFCL, ToolSandbox), RUT-Bench is distinguished by its systematic integration of realistic non-ideal user behaviors derived from actual interaction logs. The explicit comparison table (Table 1) shows that no prior benchmark simultaneously covers ideal user alignment, non-ideal user simulation, task completion, user experience evaluation, realistic environment, and sandbox + hybrid environment support. The seven-category taxonomy is empirically grounded rather than hand-crafted.

Rigor: Generally sound. The three-stage construction pipeline with iterative verification is methodologically careful. The statefulness filter ensures only genuine multi-step environment interactions are included. The white-box evaluator for environment stability and the multi-level consistency filtering reduce synthetic data artifacts. However, two concerns: (1) the reliance on LLM-as-a-Judge for diagnostic metrics introduces known biases and prompt sensitivities; (2) the behavioral taxonomy annotation uses GPT-5.4, which may have its own classification biases.

Reproducibility: Good. The benchmark is deterministic (Python class environments, fixed initial states), code and data are publicly released, and evaluation procedures are thoroughly documented in appendices with full prompt templates. The 19 models are evaluated with uniform settings, enabling fair comparison.

Significance: High for the LLM agent evaluation community. The sub-40% success rate ceiling across all models is a striking empirical finding that quantifies the "simulation gap" between benchmark performance and real-world utility. The fine-grained behavioral breakdown (Table 2) provides actionable diagnostic signals for practitioners — e.g., knowing that Underspecification causes ~45% relative drops directs R&D toward better context inference and proactive clarification strategies.

Red flags:

  1. LLM-synthesis gap: Non-ideal utterances are generated by rewriting ideal dialogues via LLM prompts, which may not fully capture the unpredictable pragmatic nuances and spontaneous disfluencies of genuine human-agent dynamics (acknowledged in Limitations).
  2. LLM-as-Judge bias: Diagnostic metrics (Informational Honesty, Tool Discipline) are evaluated by LLM judges, which can suffer from reasoning failures when assessing complex multi-turn tool traces.
  3. Abstracted environments: Deterministic Python classes omit real-world deployment friction (network latency, rate limits, dynamic backend schema shifts), potentially inflating success rates relative to production conditions.
  4. English-only scope: Restricted to English dialogues and seven predefined categories, excluding cross-linguistic variations and adversarial prompt injection scenarios.
  5. Some anomalous results: Llama-3.1-8B-Instruct shows slightly higher performance on Contradictory Constraints and Impatience than on Ideal behavior (though absolute numbers are near floor: ~1.28%), suggesting possible noise in lightweight model evaluation.

Strengths

  • Empirically grounded taxonomy derived from real user-LLM interaction logs (WildChat)
  • Rigorous three-stage construction pipeline with iterative multi-level verification
  • Comprehensive model coverage: 19 models spanning proprietary, open-source, and lightweight tiers
  • Multi-dimensional evaluation: success rate + informational honesty + tool discipline
  • Fine-grained behavioral analysis revealing model-specific vulnerabilities (e.g., Claude's fragility to contradictory constraints, Gemini's weakness on fabricated parameters)
  • Open-source release of code, data, and full prompt templates
  • Explicit comparison with prior benchmarks showing clear differentiation
  • Actionable failure mode taxonomy (incorrect tool sequence, premature termination, parameter errors) that separates robust from failing models

Limitations

  1. Synthetic non-ideal utterances: The benchmark relies on LLM-assisted rewriting of ideal dialogues, which may miss spontaneous disfluencies and pragmatic nuances of real human behavior.
  2. LLM-as-a-Judge bias: Diagnostic metric evaluation is susceptible to prompt sensitivity and reasoning failures in the judge LLM.
  3. Abstracted execution environments: Deterministic Python classes lack real-world operational friction (network latency, rate limits, schema changes).
  4. English-only and fixed categories: Limited to English dialogues and seven predefined non-ideal behaviors, excluding cross-cultural linguistic variations and adversarial attacks.
  5. No human validation of simulated behaviors: The paper does not report human evaluation or validation that the synthetic non-ideal behaviors adequately represent real user interaction patterns.
  6. Possible contamination: Base queries are drawn from existing benchmarks (API-Bank, ToolAce, Dolci), some of which may appear in pretraining data, though the synthetic environments and dialogues are novel.

Relevance

该工作与大语言模型推理评估和系统可靠性高度相关。RUT-Bench 通过引入真实世界非理想用户行为来评估工具调用能力，揭示了当前模型在复杂交互场景下的可靠性瓶颈（如违反工具调用顺序、过早终止执行等），这对于理解 LLM 在实际部署中的鲁棒性差距具有直接参考价值，符合对 LLM 推理评估和系统可靠性的研究兴趣。

Questions for Further Exploration

  1. How would the benchmark results change if non-ideal utterances were derived from actual human rewrites rather than LLM-synthesized perturbations? Would the success rate floor drop even further?
  2. Can the failure mode taxonomy (incorrect tool sequence, premature termination, parameter errors) guide the design of targeted training curricula or RL reward shaping to improve robustness?
  3. How transferable are the behavioral vulnerability patterns across different tool-use domains? Does Claude's fragility to contradictory constraints persist in healthcare vs. e-commerce environments?
  4. What is the marginal utility of adding more non-ideal behavior categories (e.g., cultural misalignment, emotional volatility) beyond the current seven?
  5. Could the diagnostic metrics (Informational Honesty, Tool Discipline) be automated with programmatic checkers rather than LLM judges to reduce evaluation bias and cost?
