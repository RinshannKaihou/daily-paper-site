Entropy Gate: Entropy Quenching for Near-Lossless Token Compression in LLM Pipelines

arXiv: 2606.03739

Date: 2026-06-02

https://arxiv.org/abs/2606.03739

一句话总结： 本文提出 Entropy Gate，一种受统计力学启发的 token 压缩框架，通过多因子信息能量（统计、结构、位置）对 token 赋权，并以自适应淬火调度逐步剔除低能量 token，在保持语义保真度 SE > 0.80 的前提下实现 40–60% 的提示压缩率，结合上下文去重和输出端压缩可在 agentic 工作流中达到 88–96% 的总 token 削减。
TL;DR: This paper introduces Entropy Gate, a thermodynamics-inspired token compression framework that assigns multi-factor information energy (statistical, structural, positional) to tokens and progressively freezes out low-energy tokens via an adaptive quenching schedule. It achieves 40–60% prompt compression at SE > 0.80, with combined deduplication and output-side compression reaching 88–96% total token reduction for agentic workloads.

核心贡献

  • 多因子信息能量模型： 将 token 重要性分解为统计能量（多段 TF-IDF）、结构能量（基于正则的句法角色打分）和位置能量（指数衰减），三者加权融合为综合信息能量 E(t)。
  • 自适应熵淬火调度： 采用 T(τ) = T₀/(1 + ατ) 的降温曲线，通过确定性 Top-K 或 Boltzmann 生存概率逐步剔除低能量 token，并证明淬火过程具有单调嵌套性。
  • 能量平方放大引理： 证明将能量映射为 E² 可通过放大分布方差来提升信号噪声分离度，从而增加压缩效率（实验贡献达 18.7 个百分点）。
  • 能量加权保真门控： 定义能量加权相似度 SE，证明其下界为 f·⟨E⟩kept/⟨E⟩all，并在 SE 低于阈值 θ 时自动终止压缩，提供可证明的语义保真保证。
  • 信息论压缩极限： 证明可达压缩率 CR ≤ 1 − I(P;T)/H(P)，并验证当任务相关信息稀疏时，压缩率可趋近 100%。
  • 输出端压缩与多轮结构压缩： 对模型生成的冗长响应进行输出端淬火（可削减 75% 响应 token）；对多轮对话采用 turn-decayed 压缩，旧轮次更激进、新轮次与工具调用受保护，实现 57% 的可压缩跨度压缩率。
  • 组合内存-压缩界限： 证明外部记忆与熵淬火在 token 削减上是乘法互补的，典型 agentic 场景下总削减率可达 88–96%。
  • 无训练、模型无关的代理部署： Phase 1 为纯启发式（~6 ms 延迟，12 MB 内存），Phase 2 使用 268M 参数的本地模型计算真实 log-probability，整体以 OpenAI 兼容 HTTP 代理形式部署。

Key Contributions

  • Multi-factor information energy model: Decomposes token importance into statistical energy (multi-chunk TF-IDF), structural energy (regex-based syntactic role scoring), and positional energy (exponential decay), fused via weighted combination into a unified information energy E(t).
  • Adaptive entropy quenching schedule: Uses a cooling curve T(τ) = T₀/(1 + ατ) to progressively freeze out low-energy tokens via deterministic Top-K cutoff or Boltzmann survival probabilities, with proven monotonic nested survival sets.
  • Energy-squared amplification lemma: Proves that mapping energy to E² amplifies signal-to-noise separation by exploiting distribution variance, increasing compression efficiency by up to 18.7 percentage points empirically.
  • Energy-weighted fidelity gate: Defines energy-weighted similarity SE and proves a lower bound f·⟨E⟩kept/⟨E⟩all, automatically halting compression when SE drops below threshold θ, yielding provable semantic fidelity guarantees.
  • Information-theoretic compression limit: Proves achievable compression ratio CR ≤ 1 − I(P;T)/H(P) and validates that as task-relevant information becomes sparse, CR approaches 100%.
  • Output-side quenching and structural multi-turn compression: Quenches verbose model responses (75% output token reduction); applies turn-decayed compression to multi-turn conversations where older turns are compressed more aggressively while tool calls and live queries remain frozen, achieving 57% CR on compressible spans.
  • Combined memory-compression bound: Proves external memory and entropy quenching compose multiplicatively, yielding 88–96% total token reduction in typical agentic workflows.
  • Training-free, model-agnostic proxy deployment: Phase 1 is purely heuristic (~6 ms latency, 12 MB memory); Phase 2 uses a 268M-parameter local model for true log-probabilities; deploys as an OpenAI-compatible HTTP proxy.

Methodology

The paper frames token compression as a thermodynamic quenching process. A prompt P = (t₁, …, tₙ) is first tokenized and then each token is assigned an information energy E(tᵢ) = w₁·E^stat(tᵢ) + w₂·E^struct(tᵢ) + w₃·E^pos(tᵢ), with default weights (0.5, 0.3, 0.2).

Statistical energy (Definition 3.1) uses multi-chunk TF-IDF with Laplace smoothing over overlapping 30-token chunks with stride 15, rewarding domain-specific terms that appear concentrated in few chunks. Structural energy (Definition 3.2) classifies tokens into ten syntactic roles via regex (keywords ρ=0.85, identifiers ρ=0.65, punctuation ρ=0.15, whitespace ρ=0.00). Positional energy (Definition 3.3) applies an exponential half-life decay with h = 0.15·n, capturing the intuition that early tokens carry disproportionate semantic load. A self-calibrating domain energy E_dom(tᵢ) (Equation 3) boosts tokens near a universal set of task-defining verbs (review, audit, analyze, etc.) within a window of ω=5 tokens, automatically identifying domain-critical terms without pre-configured lists.

Tokens matching protected patterns (e.g., PII redaction markers) are assigned E = ∞ and permanently frozen. Energy values are then squared (Lemma 3.7) to amplify variance before entering the quenching loop.

The quenching schedule T(τ) = T₀/(1 + ατ) iteratively reduces temperature. At each step, deterministic mode keeps the top ⌈n·T/T₀⌉ tokens by energy; Boltzmann mode assigns survival probability pᵢ = exp(−Eᵢ/kT). A fidelity gate computes energy-weighted similarity SE between original and compressed prompts, breaking the loop when SE < θ. Output-side quenching uses an amplified rate α_out ≥ 0.6 for upstream responses. A block deduplication pre-pass replaces repeated paragraph blocks with reference markers.

Results & Highlights

  • Phase 1 compression: Achieves 40–60% CR across five prompt categories (code review, security audit, documentation, SQL generation, system prompts) while maintaining SE > 0.80. Code review and security audit prompts reach the highest compression (53–54%) due to substantial instructional boilerplate.
  • Energy-squared amplification: Adds 10–25 percentage points of compression on top of raw energy, with the ablation showing it contributes the largest single gain (+18.7 pp) by stretching the right tail of the energy distribution.
  • Context deduplication: Adds 50–70% token savings on prompts with repeated blocks, composable multiplicatively with quenching.
  • Output-side quenching: Reduces response overhead by ~75%, consistent with prior findings that brevity constraints improve accuracy.
  • Combined agentic reduction: Theorem 11.1 proves R_total = p(1−r) + (1−p)·CR. For typical agentic workloads with p ∈ [0.7, 0.9] repeated context and CR ∈ [0.4, 0.6], total reduction reaches 88–96%.
  • Latency & memory: Total proxy overhead is ~6 ms (0.06–0.3% of typical LLM inference), with Phase 1 memory footprint ~12 MB. Scalability tests show linear latency growth from 5 ms (680 tokens) to 52 ms (2,856 tokens).
  • Benchmark evaluation: On MMLU, HumanEval, and GSM8K, 8/9 compressed questions preserve task correctness at 24% effective CR (including pass-throughs); on compressible-only prompts, accuracy is 100% at 49% CR. Math-span protection resolves LaTeX notation failures.
  • LLM-as-judge quality: Claude Opus 4.8 rates compressed outputs at 3.9/5 average across clarity, correctness, and completeness. Documentation scores highest (4.3); system prompts lowest (3.6).
  • Frontier model validation: Claude Opus 4.8, GPT-5.5, and DeepSeek-V4-Pro all correctly identify SQL injection from a 42% CR compressed prompt, demonstrating cross-architecture preservation of task-critical information.
  • Calibration: SE correlates with nomic-embed-text embedding cosine similarity at r = 0.994 (MAD = 0.034), confirming it as a conservative but faithful proxy for semantic preservation.

The empirical validation is limited in scale (five prompt categories, 11 benchmark questions, Apple M4 Max single-machine setup), but the theoretical contributions are independently verifiable.

Critical Assessment

Novelty: The core idea of using simulated annealing/quenching for token selection is conceptually elegant but not entirely unprecedented—information-theoretic pruning (LLMLingua, Selective Context) and energy-based methods exist. The paper's specific contribution lies in the thermodynamic formalization, the multi-factor energy decomposition, the energy-squared amplification lemma, and the provable fidelity bounds. The self-calibrating domain energy (proximity to task verbs) is a genuinely clever mechanism that avoids domain-specific term lists.

Rigor: The paper is unusually rigorous for a systems paper, with eight theorems, multiple lemmas, and formal proofs. However, some proofs rely on assumptions that merit scrutiny. Theorem 3.5 assumes −log P_LLM(t|context) is the sufficient statistic for downstream response information, which is heuristic in Phase 1. Theorem 3.14's bound is asymptotic and not empirically tight. The energy-squared amplification lemma (Lemma 3.7) is mathematically correct but its practical benefit depends heavily on the energy distribution having sufficient variance—which the paper confirms empirically but may not generalize to all prompt types.

Reproducibility: Phase 1 is fully reproducible: it requires no model downloads, no GPU, and the algorithm is completely specified with pseudocode. Phase 2 depends on gemma3:270m via llama.cpp and nomic-embed-text via Ollama, which are openly available. The open-source release improves reproducibility. However, the benchmark evaluation is small (11 questions) and the LLM-as-judge evaluation uses a single judge model, raising concerns about statistical power and evaluator bias.

Significance: Token compression is economically important for LLM deployments. The 40–60% compression at negligible latency overhead is practically valuable. The 88–96% combined reduction claim for agentic workflows is headline-grabbing but depends on optimistic assumptions about repeated context fraction (p = 0.7–0.9) that may not hold for all agentic applications. The framework's stateless, model-agnostic proxy design makes it deployable without retraining, which is a genuine engineering advantage over methods like Gist tokens or ICAE.

Red flags:

  • The empirical evaluation is limited in scale and diversity. Five prompt types and 11 benchmark questions are insufficient to claim generalization across all LLM workloads.
  • The whitespace tokenizer in Phase 1 creates compound tokens that merge identifiers with punctuation, causing the primary qualitative degradation in code prompts. The authors acknowledge this but present Phase 1 results as primary findings while Phase 2 (which fixes it) remains partially aspirational.
  • The agentic compression-induced loop (Section 11.4) is a serious failure mode discovered in production testing. The fact that the best profile caused a 12-minute, 20+ tool-call cascade failure raises questions about the safety of deploying aggressive compression in tool-using agents without structural protection rules.
  • SE is necessary but not sufficient for task preservation (Section 11.5). The example where higher SE produced worse downstream responses reveals a fundamental limitation of the energy-weighted metric.
  • The comparison table (Table 1) claims Entropy Gate is the only method with "provable fidelity guarantees," but the practical strength of these guarantees depends on SE being a faithful proxy for task performance—which the paper itself shows is not always true.
  • Some performance claims rely on external memory systems (MemPalace, Graphify) that are not part of the core framework, making the 88–96% figure a bound on combined systems rather than a standalone result.

Strengths

  • Elegant thermodynamic analogy that yields intuitively interpretable and mathematically tractable formalism.
  • Training-free Phase 1 makes it immediately deployable without GPU infrastructure.
  • Self-calibrating domain energy is a clever, genuinely generalizable mechanism that avoids brittle domain-specific term lists.
  • Comprehensive theoretical framework with proved theorems on optimality, monotonicity, fidelity bounds, and information-theoretic limits.
  • Negligible latency overhead (~6 ms) makes it practical for production proxy pipelines.
  • Honest and detailed error analysis, including the serious agentic loop failure mode and SE insufficiency examples.
  • Open-source implementation available.
  • Graceful degradation design: failures fall back to zero-compression passthrough.

Limitations

  • Limited empirical scale: small benchmark (11 questions), five prompt types, single-machine evaluation.
  • Phase 1 whitespace tokenizer causes compound token artifacts in code, degrading compression quality.
  • SE is a conservative but incomplete proxy for downstream task preservation; high SE does not guarantee correct LLM behavior (Section 11.5).
  • Agentic workflows require strict structural protection and higher fidelity thresholds (θ ≥ 0.88), reducing achievable compression for tool-using agents.
  • English-centric structural regex and left-to-right positional assumptions may degrade performance on multilingual or right-to-left text.
  • Boltzmann mode introduces stochasticity with higher variance (±6.3% CR) and is not recommended for production.
  • The combined 88–96% reduction bound depends on optimistic assumptions about context repetition rates that may not hold broadly.

Relevance

本文提出的 Entropy Gate 在 LLM 推理效率与 token 经济性方面具有直接工程价值，其基于信息能量的可证明保真框架与用户对 ML 系统可靠性及数学基础的研究兴趣高度契合；同时，输出端压缩与多轮对话中的结构性保护规则对理解 LLM 推理评估中的稳定性与一致性也有借鉴意义。

Questions for Further Exploration

  1. How would the energy-squared amplification perform on prompts with inherently low energy variance (e.g., highly uniform technical documentation or poetry)? Would the compression efficiency collapse?
  2. Can the structural classifier be replaced with a lightweight learned syntactic parser to eliminate compound token artifacts in Phase 1, without requiring a full language model?
  3. What is the precise relationship between SE and downstream task accuracy across a larger, more diverse benchmark? Can a task-aware fidelity metric replace SE for agentic workflows?
  4. How does the framework perform on non-English and right-to-left languages, given the English-centric regex patterns and positional decay assumptions?
  5. Could Bayesian optimization (Phase 3) discover per-user or per-application optimal energy weights that substantially outperform the fixed defaults?
  6. What is the security implication of compressing prompts that contain adversarially crafted low-energy tokens—could an attacker exploit the quenching process to hide malicious content?
