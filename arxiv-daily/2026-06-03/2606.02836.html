Fast Transformer Inference on ARM-Based HMPSoCs

arXiv: 2606.02836

Date: 2026-06-01

https://arxiv.org/abs/2606.02836

一句话总结： 本文通过在 ARM Compute Library 中实现 Transformer 专用内核（NEON/OpenCL），并基于微基准分析发现不同层的 CPU/GPU 性能异构性，提出了 CPU-GPU 层切换协同推理策略，在 Khadas VIM3 嵌入式平台上相较最优单处理器推理降低延迟达 15.72%，整体较 TVM 等框架提速约 2.3 倍。
TL;DR: This paper extends the ARM Compute Library with transformer-specific kernels (NEON for CPU, OpenCL for GPU), characterizes layer-wise CPU/GPU performance heterogeneity via micro-benchmarks, and proposes cooperative CPU-GPU layer-switched inference. Evaluated on a Khadas VIM3 board (Amlogic A331D HMPSoC), it achieves up to 15.72% lower latency than the best single-processor baseline and roughly 2.3× speedup over TVM.

核心贡献

  • 在 ARM Compute Library (ARM-CL) 中实现了 Embedding、Multi-Head Attention (MHA)、Scaled Dot-Product Attention (SDPA)、Layer Normalization 和 Feed Forward 等 Transformer 专用内核，填补了该框架对 Transformer 推理的空白。
  • 使用 NEON intrinsics 加速 CPU 端、OpenCL 加速 GPU 端，并将 BERT-base、DistilBERT、MobileBERT、SqueezeBERT 和 GPT-2 五种模型移植到扩展后的 ARM-CL 上。
  • 通过微基准测试对 Transformer 五类基本层进行了系统的性能异构性分析，首次量化了不同层在嵌入式 CPU 与 GPU 上的相对性能差异及其随序列长度和隐藏维度的变化规律。
  • 提出了基于 CPU-GPU 层切换的协同推理策略：将内存密集型的 Embedding、SDPA 和 Add&Norm 层分配给 CPU，将计算密集型的 Attention Linear 和 FF 层分配给 GPU；利用 HMPSoC 的共享内存空间消除显式数据拷贝开销。
  • 在 Khadas VIM 3 BASIC 嵌入式平台（Amlogic A331D，12nm）上验证，协同推理相比最优单处理器推理降低延迟达 15.72%，平均降低 10.95%；相较 TVM 框架的 CPU/GPU 推理分别提速 2.34 倍和 2.23 倍。

Key Contributions

  • Implemented new transformer-specific kernels in the ARM Compute Library (ARM-CL) for Embedding, Multi-Head Attention (MHA), Scaled Dot-Product Attention (SDPA), Layer Normalization, and Feed Forward layers, filling the gap in native transformer inference support on ARM-based edge devices.
  • Accelerated CPU execution via NEON intrinsics and GPU execution via OpenCL, porting five transformer architectures (BERT-base, DistilBERT, MobileBERT, SqueezeBERT, and GPT-2) to the extended ARM-CL.
  • Performed a micro-benchmark-based characterization of on-chip transformer inference, providing the first quantitative analysis of layer-wise performance heterogeneity across embedded CPU and GPU, and how it varies with sequence length (L) and hidden dimension (d_model).
  • Proposed cooperative CPU-GPU layer-switched inference: assigning memory-intensive layers (Embedding, SDPA, Add&Norm) to the CPU and compute-intensive layers (Attention Linear, FF) to the GPU; leveraging the HMPSoC's shared memory space to avoid explicit memcpy overhead between processors.
  • Demonstrated on a Khadas VIM 3 BASIC board (Amlogic A331D, 12 nm) that cooperative inference reduces latency by up to 15.72% (10.95% on average) versus the best single-processor baseline, and achieves roughly 2.3× speedup over TVM.

Methodology

The authors extend ARM-CL v21.02 — a library originally optimized for CNN inference on ARM hardware — by implementing computationally equivalent transformer kernels targeting edge devices. CPU kernels use NEON intrinsics for 128-bit SIMD execution, while GPU kernels use OpenCL with tiled matrix-multiplication strategies tailored to the Mali GPU architecture.

They port five transformer models: BERT-base (implemented from scratch in ARM-CL), DistilBERT, MobileBERT, SqueezeBERT, and GPT-2 (with causal masking added to SDPA). The system builds a model description DAG that defines layer connectivity and kernel placement (CPU or GPU). Consecutive layers on the same processor use single-target tensors, while CPU-GPU transitions use multi-target shared tensors allocated in GPU-accessible memory that the CPU can map/unmap via OpenCL — eliminating explicit memcpy operations.

A detailed micro-benchmark characterization analyzes the five fundamental transformer layers in isolation, sweeping sequence length L (16–512) and hidden dimension d_model (192–960). The CPU/GPU performance ratio T_CPU/T_GPU is used to identify which processor is preferable for each layer under different workload sizes.

Results & Highlights

  • Single-processor speedup vs TVM: Extended ARM-CL achieves 2.34× faster CPU inference and 2.23× faster GPU inference on average across the five models compared to TVM v0.18. ExecuTorch offers similar CPU performance but lacks ARM Mali GPU support entirely.
  • Layer-wise heterogeneity: For L ≤ 128, the CPU outperforms the GPU on Embedding, SDPA, and Add&Norm layers (memory-bound operations), while the GPU wins on Attention Linear and FF layers (compute-bound MMULs). For L ≥ 256, the GPU becomes faster for all layers regardless of d_model.
  • Cache-aware bottleneck analysis: The GPU's advantage on Attention Linear and FF diminishes as L grows beyond 128 because the Mali G52's small 128 KB L2 cache (vs. the CPU's 2 MB) cannot sustain larger MMUL working sets, causing memory-access overheads to dominate.
  • Cooperative CPU-GPU gains: Layer-switched inference outperforms both CPU-only and GPU-only execution for all five models. Latency reductions reach 15.72% versus the best single processor, averaging 10.95%.
  • Memory efficiency: BERT-base (~300 MB) runs without issues on the 2–4 GB DRAM typical of such platforms, while BERT-large (~1 GB) crashes at runtime, highlighting practical memory constraints.
  • Open source: All ARM-CL extensions are released under the MIT license at https://github.com/dondavan/fast-transformer-on-acl.

Critical Assessment

  • Novelty: Moderate-to-high in the systems-for-ML space. While transformer kernel optimization and heterogeneous scheduling are active areas, this work is among the first to specifically target native transformer inference in ARM-CL on ARM HMPSoCs, and it provides a detailed layer-wise characterization that justifies the scheduling decisions. The prior art for cooperative CPU-GPU inference focused on CNNs, not transformers.
  • Rigor: The methodology is sound. The authors use micro-benchmarks to isolate layer behavior, compare against a current compiler baseline (TVM), and test across five model architectures. The T_CPU/T_GPU sweep across L and d_model provides empirical grounding for the scheduling policy. One limitation is that all experiments use a single token length of 32 for the end-to-end comparisons, despite the characterization covering a wider range — it would be valuable to see end-to-end results at L=64 or 128 where the heterogeneity is strongest.
  • Reproducibility: Good. The platform (Khadas VIM 3) is commercially available, ARM-CL is open-source, and the authors have released their extensions under MIT license. However, reproducing exact numbers depends on kernel versions, cooling, and DRAM configuration, which are reasonably well-documented.
  • Significance: High for edge AI practitioners deploying transformers on commodity ARM SoCs without dedicated NPUs. The 2.3× speedup over TVM and the 15.72% gain from cooperative execution are practically meaningful for latency-sensitive applications.
  • Red flags:
  • The 3× claim in the abstract appears to refer to the ARM-CL extension vs. TVM/ExecuTorch, not vs. a highly optimized baseline like ONNX Runtime or vendor-specific SDKs.
  • No energy or power measurements are reported — critical for battery-constrained edge devices.
  • The evaluation is limited to a single HMPSoC (Amlogic A331D); generalization to other ARM big.Little + Mali configurations (e.g., MediaTek Dimensity, Samsung Exynos) is plausible but not validated.
  • Only FP32 inference is discussed; INT8 quantization — standard for edge deployment — is not mentioned.
  • The authors acknowledge that BERT-large crashes due to memory limits, suggesting the approach is currently limited to smaller transformers.

Strengths

  • Clear systems contribution with immediate practical applicability: filling a real gap in ARM-CL for transformer inference.
  • Thorough layer-wise characterization provides actionable insights for other developers targeting ARM HMPSoCs.
  • Smart use of shared memory space to eliminate memcpy overhead between CPU and GPU, keeping cooperative execution lightweight.
  • Evaluation across five diverse transformer architectures strengthens generalizability claims.
  • Open-source release enables community adoption and independent verification.

Limitations

  • No energy or power consumption data, which is often the binding constraint on edge devices.
  • Evaluation limited to a single embedded board (Khadas VIM 3 with Amlogic A331D); broader hardware validation is absent.
  • End-to-end comparisons at L=32 only — the heterogeneity analysis suggests L=64 or 128 would show even stronger CPU advantages for memory-bound layers, but cooperative gains at those lengths are not quantified end-to-end.
  • No comparison against INT8 quantized baselines or Neural Processing Units (NPUs), which are increasingly common on mobile SoCs.
  • BERT-large fails to run due to memory constraints, indicating the approach is currently restricted to smaller models.
  • The "state-of-the-art CPU/GPU implementations" baseline primarily refers to TVM and ExecuTorch; comparison against more recent inference engines (e.g., ONNX Runtime Mobile, MLC-LLM) is missing.

Relevance

对于希望在资源受限的 ARM 边缘设备上部署 Transformer 模型的研究者和工程师而言，本文提供了从内核实现到异构调度的一整套实用方案，其详细的层级别性能分析和开源代码对优化移动端或嵌入式 Transformer 推理具有直接的参考价值。

Questions for Further Exploration

  1. How would INT8 or FP16 quantization affect the layer-wise CPU/GPU performance ratios and the overall cooperative scheduling strategy?
  2. What are the energy-efficiency implications of CPU-GPU layer switching versus single-processor execution on battery-powered devices?
  3. Does the cooperative scheduling generalize to newer ARM Mali GPUs (e.g., G710, G715) with larger caches, or to other HMPSoC families such as MediaTek Dimensity or Qualcomm Snapdragon?
  4. Could the layer-switched approach be extended to larger models (e.g., BERT-large, ViT) by integrating model-parallelism or offloading techniques, given the DRAM constraints observed?
  5. How does the approach compare against emerging edge inference frameworks such as MLC-LLM, ONNX Runtime Mobile, or vendor SDKs like ARM Ethos NPU drivers?
