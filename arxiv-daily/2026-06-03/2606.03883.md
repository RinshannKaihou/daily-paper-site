> **一句话总结：** 本文构建了一套包含21种逻辑谜题的可扩展基准测试，并提出将大型推理模型的非结构化思维链转换为可验证的原子声明依赖图（DAG），基于吸收马尔可夫链模型定义了推理效率指标η，实证发现该指标能在准确率饱和时区分不同模型的推理结构，而token数量无法反映推理质量。

> **TL;DR:** This paper builds a scalable benchmark of 21 grid logic puzzles and proposes converting unstructured LRM reasoning traces into verifiable directed acyclic graphs of atomic claims and dependencies. Based on an absorbing Markov chain model, it defines a reasoning efficiency metric η that quantifies how concentrated a model's logical flow is. Empirically, η distinguishes structurally different reasoning behaviors even when accuracy is saturated, while token count proves to be a poor proxy for reasoning quality.

## 核心贡献

- **可扩展的逻辑谜题基准：** 基于Simon Tatham谜题集构建了一个包含21种2D网格逻辑谜题的基准测试，每种谜题设有4个难度等级（Trivial、Human easy、Human normal、Human hard），支持对推理行为的可控规模化研究。
- **结构化推理图表示：** 提出将模型生成的自由文本推理轨迹转换为有向无环图（DAG），其中节点为原子声明（atomic claims），边表示演绎依赖关系。声明可通过可执行的谜题环境进行确定性验证。
- **基于马尔可夫链的推理效率指标η：** 将推理过程建模为DAG上的吸收马尔可夫链，利用结构熵定义了推理流效率η，衡量模型逻辑流相对于最小解声明集的集中度。η∈[0,1]，越高表示推理越聚焦。
- **混合式声明提取流水线：** 设计了结合确定性模式匹配与LLM增强的两阶段声明提取流程，以及基于LLM的规则提取和声明验证机制，实现了对自由形式文本的结构化解析。
- **实证发现传统指标的局限性：** 证明token数量与η几乎无关（r=-0.05, p=0.64），而η能有效区分准确率无法区分的推理结构差异；额外token主要转化为验证开销而非提升推理质量。

## Key Contributions

- **Scalable Logic Puzzle Benchmark:** Constructs a benchmark of 21 grid logic puzzles from the Simon Tatham collection, each with four difficulty levels (Trivial, Human easy, Human normal, Human hard), enabling controlled scaling studies of LLM reasoning behavior.
- **Structured Reasoning Graph Representation:** Proposes converting free-text reasoning traces into directed acyclic graphs (DAGs) where nodes are atomic claims and edges represent deductive dependencies. Claims are deterministically verifiable against an executable puzzle environment.
- **Markov Chain-Based Reasoning Efficiency Metric η:** Models reasoning as an absorbing Markov chain on a DAG and defines reasoning-flow efficiency η using structural entropy to measure how concentrated a model's logical flow is relative to the minimal claim set defining a solution. η∈[0,1], with higher values indicating more focused reasoning.
- **Hybrid Claim Extraction Pipeline:** Designs a two-stage claim extraction procedure combining deterministic pattern matching with LLM augmentation, plus LLM-based rule extraction and claim verification, enabling structured parsing of free-form text traces.
- **Empirical Demonstration of Traditional Metric Limitations:** Proves that token count is essentially uncorrelated with η (r=-0.05, p=0.64), while η effectively distinguishes structurally different reasoning behaviors that accuracy cannot separate; extra tokens primarily translate into verification overhead rather than improved reasoning quality.

## Methodology

### Benchmark Design
The authors build on a scalable grid-based puzzle RL environment derived from Simon Tatham's puzzle collection. They select 21 deterministic puzzles (e.g., Tents, Sudoku/Solo, Towers, Lightup, Net, Tracks) spanning diverse constraint types (placement, connectivity, counting, Latin-square-style). Each puzzle has four difficulty levels controlled by grid size and clue density. For each puzzle and difficulty, 5 fixed instances are evaluated across all models to ensure fair comparison. Models are prompted with the same solver prompt and must output a final solution in a standardized format. Decoding uses temperature T=1.

### Reasoning Graph Formalization
A reasoning trace S = (s₁, s₂, ...) is an ordered sequence of sentences. Atomic claims C are statements about the puzzle state (e.g., "cell (3,4) is 9"). The reasoning graph G = (V, E) is a DAG where:
- Each vertex v = (c, i) ∈ V indicates claim c is asserted in sentence sᵢ
- Edges represent either inference from premises to a conclusion, or restatement links between occurrences of the same claim
- All premises appear before their conclusion, ensuring topological order
- **Minimal claim set C*:** the smallest set of claims that fully determines the puzzle solution
- **Solution subgraph G_sol:** contains all vertices that directly or indirectly contribute to the first occurrence of solution claims
- **Verification subgraph G_ver:** contains all descendants of solution claims plus their ancestors not in G_sol

### Reasoning Metrics
**Graph-based metrics:** |V| (total claims), |V_sol|, |V_ver|, graph depth, diameter, width, average path length.

**Validity-based metrics:** Each claim is checked against the executable puzzle environment. First-error depth is defined as the depth in G of the earliest incorrect claim.

**Flow-based metrics (η):** The reasoning process is modeled as an absorbing Markov chain on the DAG. An absorbing state a is added, connected to all leaf nodes. The initial logical mass π is uniform over source nodes (in-degree zero). The logical mass flowing through each node is m = πN, where N = ΣQᵗ is the fundamental matrix. Structural entropy is:

H_str(G) = -Σ (m(v)/‖m‖) · log(m(v)/‖m‖)

Reasoning efficiency is then:

η = (log|V| - H_str(G)) / (log|V| - log|C*|)

This measures how concentrated the logical flow is relative to the gap between the actual graph size and the minimal solution size.

### Graph Extraction Pipeline (3 stages)
1. **Claim Extraction:** Hybrid two-pass procedure. (i) High-precision deterministic extractor + LLM completion for schema repair; (ii) High-recall LLM extractor without rules. Outputs are merged, screened in batches of 200 by an LLM with localized context, and globally deduplicated.
2. **Rule Extraction:** For each finalized claim, an LLM is prompted to return a single rule application linking it to prior claims, or classify it as directly stated/restated.
3. **Claim Verification:** Each verifiable claim is checked deterministically against the puzzle environment for consistency with rules, constraints, and final solution.

The extraction uses GPT-5.2 for claim extraction/screening and GPT-5-mini for rule extraction.

## Results & Highlights

### Benchmark Results (Table 1)
Four models are evaluated: GPT-5 (best performer), Qwen3 235B, DeepSeek V3.2, and Kimi K2.

| Model | Trivial Acc. | Human Hard Acc. | Avg Tokens (Hard) |
|-------|-------------|-----------------|-------------------|
| GPT-5 | 83.8% | 5.7% | 19,862 |
| Qwen3 235B | 69.5% | 0% | 23,609 |
| DeepSeek V3.2 | 77.1% | 0% | 36,787 |
| Kimi K2 | 77.1% | 1.0% | 61,307 |

**Key observations:**
- **Accuracy collapses with difficulty:** All models show sharp accuracy drops from Trivial to Human hard (e.g., GPT-5 drops from 83.8% to 5.7%).
- **More tokens ≠ better performance:** Kimi K2 uses the largest token budgets (~61k on Human hard) but does not outperform GPT-5. GPT-5 achieves the best accuracy while remaining the most token-efficient.
- **Hardest regime remains largely unsolved** even with large token budgets, suggesting fundamental limitations that are not resolved by simply allocating more compute.

### Efficiency η Correlations (Figure 5)
- **η vs. Tokens:** r = -0.05, p = 0.64 — essentially uncorrelated. Token count is a poor proxy for reasoning quality.
- **η vs. Claim Accuracy:** r = 0.33, p = 8.9×10⁻⁴ — moderate positive association. Correctness helps but is not sufficient.
- **η vs. Solution Fraction |V_sol|/|V|:** r = 0.55, p = 1.1×10⁻⁹ — higher efficiency linked to more focused on-solution reasoning.
- **η vs. Graph Size |V|:** r = -0.33, p = 0.001 — larger graphs indicate "graph bloat" from diffuse exploration.
- **η vs. First-Error Depth:** r = 0.28, p = 0.015 — early errors induce drift and corrective exploration that lowers efficiency.
- **η vs. Verification Overhead |V_ver|/|V_sol|:** r = -0.33 — more verification overhead reduces efficiency.
- **η vs. Restatements:** r = 0.27, p = 0.0078 — some redundancy (structured recap/anchoring) is beneficial.

### η vs. Simpler Structural Metrics (Table 2)
Width, |V|, and token count all correlate negatively with accuracy and strongly with each other, primarily tracking puzzle difficulty. **η is the only metric that correlates positively with accuracy while remaining uncorrelated with token count**, because its size normalization factors out graph scale.

### Efficiency Under Perfect Accuracy (Figure 6)
Even when restricting to puzzle instances where all models solve correctly (accuracy saturated), η exposes significant differences in reasoning structure and scaling behavior between models. This demonstrates that **accuracy alone hides substantial variation in how solutions are produced**.

## Critical Assessment

- **Novelty:** High. While reasoning evaluation is an active area, most work focuses on accuracy, pass@k, or process-based reward models. The key innovation here is treating reasoning traces as topological objects (DAGs) with a principled efficiency metric grounded in information theory and Markov chains. The idea of measuring "structural entropy" of logical flow is genuinely new and could inspire follow-up work in mechanistic interpretability.
- **Rigor:** Strong. The paper provides formal definitions for claims, graphs, minimal solution sets, and the Markov chain model. The extraction pipeline is validated with stability analysis (Section A) and manual inspection (Section B). Correlations are reported with p-values and bootstrap confidence intervals. The fixed-instance, same-prompt design ensures fair comparison.
- **Reproducibility:** Good. The benchmark is based on publicly available puzzles (Simon Tatham's collection), and the code is open-sourced. The deterministic puzzle environment and standardized prompts make reproduction feasible. However, the claim extraction pipeline relies on proprietary models (GPT-5.2, GPT-5-mini), which may limit exact reproducibility for researchers without access.
- **Significance:** Potentially high for the evaluation and interpretability communities. If structural metrics like η can diagnose failure modes that accuracy misses, this could shift how reasoning models are trained (e.g., using η as an auxiliary reward signal). The finding that token count is uncorrelated with reasoning quality is particularly important given current trends toward scaling test-time compute.
- **Red Flags / Limitations:**
  - The benchmark is limited to deterministic, fully-specified logic puzzles. Generalization to open-ended reasoning (math proofs, creative writing) remains to be shown.
  - The graph extraction pipeline, while carefully designed, still depends on LLM-based steps that may introduce errors or bias. The authors acknowledge this and include stability analysis.
  - Closed-source models (GPT-5) cannot be analyzed structurally since their reasoning traces are not accessible — the structural analysis is limited to open models.
  - The positive correlation between restatements and η (r=0.27) is interesting but the causal direction is unclear — do restatements cause better efficiency, or do efficient models just happen to restate more?

## Strengths

- Novel structural perspective on reasoning evaluation, moving beyond scalar metrics to topological analysis
- Well-grounded mathematical framework combining graph theory, Markov chains, and information theory
- Strong empirical result: η is the only metric positively correlated with accuracy while uncorrelated with token count
- Practical diagnostic utility for identifying failure modes (early errors, graph bloat, verification overhead)
- Scalable benchmark with 21 diverse puzzle types and controllable difficulty
- Careful experimental design: fixed instances, same prompts, temperature=1, executable verification
- Open-sourced code and detailed appendices with prompts, puzzle rules, and stability analysis
- Published at ICML 2026, indicating peer-reviewed quality

## Limitations

- Domain restriction to deterministic logic puzzles; generalization to open-ended reasoning tasks (mathematical proofs, code generation with partial tests) is suggested but not demonstrated
- Graph extraction relies on proprietary LLMs (GPT-5.2, GPT-5-mini), creating a reproducibility barrier for researchers without access to these specific models
- Structural analysis is only possible for open-source models with accessible reasoning traces; closed-source frontier models like GPT-5 can only be evaluated on outcome metrics
- The causal relationship between restatements and efficiency is not established — the positive correlation may reflect confounding factors
- The framework assumes reasoning traces are textual and sequential; multimodal or tool-augmented reasoning is not addressed
- While the authors suggest η could serve as an auxiliary training signal, no training experiments are conducted to validate this

## Relevance

本文高度契合用户关于LLM推理评估与可解释性的研究兴趣——通过将推理轨迹结构化为可验证的声明依赖图并定义效率指标η，该工作不仅揭示了传统准确率与token数指标无法捕捉的推理过程结构性差异，还为诊断推理模型失效模式（如早期错误导致的漂移、图膨胀、验证开销）提供了严格的量化工具，对推理模型的训练监控与推理质量评估具有直接的实践价值。

## Questions for Further Exploration

1. **Can η serve as an auxiliary reward signal during RLVR training?** The authors suggest this direction, but empirical validation is needed to determine whether optimizing for η improves generalization or inadvertently encourages harmful shortcut behaviors.
2. **How does η correlate with mechanistic interpretability findings?** For instance, do models with higher η exhibit more focused circuit usage, or do they activate fewer SAE features per reasoning step?
3. **What is the causal direction of the restatement-efficiency correlation?** Does structured recap genuinely improve reasoning focus, or is it merely a byproduct of models that already reason efficiently?
4. **How robust is the extraction pipeline to deliberately adversarial or incoherent reasoning traces?** The current validation focuses on stability; stress-testing with hallucinated or circular reasoning could reveal failure modes in the graph construction itself.
5. **Can this framework be adapted for code generation, where unit tests provide partial claim verification?** The authors mention this as a target domain, but the translation from puzzle claims to code assertions (e.g., variable invariants, intermediate test outcomes) is non-trivial.
