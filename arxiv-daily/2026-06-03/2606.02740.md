> **一句话总结：** 本文提出 ScoreStop，一种基于功能性得分检验的梯度早停规则，将每次迭代中的停止决策转化为对"当前预测器是否为总体风险最小化器"的假设检验，该统计量在更新方向上具有尺度不变性且渐近服从 χ² 分布，并可自然扩展至隐式损失（如 LambdaRank）和数据依赖型损失（如 Cox 回归）。

> **TL;DR:** This paper proposes ScoreStop, a gradient-based early stopping rule for gradient boosted decision trees that frames each stopping decision as a functional score test of whether the current predictor is the population risk minimizer. The test statistic is scale-invariant in the update direction and asymptotically χ²-distributed under the null, naturally extending to implicit losses (e.g., LambdaRank) and data-dependent losses (e.g., Cox regression) through influence functions.

## 核心贡献

- **提出 ScoreStop**，一种全新的基于梯度的早停规则，替代传统的基于验证损失和固定 patience 参数的启发式方法。
- **将停止决策形式化为假设检验**：每次迭代检验零假设"当前函数估计器是总体风险最小化器"，使用验证数据上的功能性得分检验。
- **尺度不变且渐近可校准的统计量**：检验统计量在更新方向和损失梯度缩放下保持不变，且在零假设下渐近服从已知的 χ² 分布，无需 bootstrap。
- **扩展至非标准损失**：由于使用梯度而非损失值，同一框架适用于隐式损失（如 LambdaRank，用户仅指定梯度而无显式损失）和数据依赖型损失（如 Cox 回归，通过影响函数处理待估的基线风险）。
- **三种算法变体**：前向 ScoreStop（FWD-SS，沿当前更新方向检验）、后向 ScoreStop（BWD-SS，沿前一迭代方向检验）、稳定化 ScoreStop（STAB-SS，联合检验两个方向）。
- **理论保证**：证明了在零假设下统计量的 χ² 渐近分布（Proposition 4.6），以及在备择假设下的检验一致性（Proposition 4.7），并说明梯度提升基学习器提供的方向接近最优（Corollary 4.8）。
- **广泛的实验验证**：在合成实验（回归、分类、排序、生存分析）和 13 个真实表格数据基准上验证了方法的有效性和竞争力。

## Key Contributions

- **Proposes ScoreStop**, a novel gradient-based early stopping rule as an alternative to traditional validation-loss monitoring with a fixed patience parameter.
- **Frames stopping as hypothesis testing**: At each iteration, tests the null hypothesis that the current function estimator equals the population risk minimizer using a functional score test on validation data.
- **Scale-invariant and asymptotically calibrated statistic**: The test statistic is invariant to rescaling of the update direction and loss gradient, and has a known asymptotic χ² distribution under the null without requiring bootstrap.
- **Extends to non-standard losses**: Because it uses gradients rather than loss values, the same construction applies to implicit losses (e.g., LambdaRank, where only gradients are specified) and data-dependent losses (e.g., Cox regression, handled via influence functions to account for estimated nuisance functions like the baseline hazard).
- **Three algorithmic variants**: Forward ScoreStop (FWD-SS, tests in the current update direction), Backward ScoreStop (BWD-SS, tests in the previous iteration's direction), and Stabilized ScoreStop (STAB-SS, jointly tests both directions).
- **Theoretical guarantees**: Proves the χ² asymptotic null distribution (Proposition 4.6), power consistency under the alternative (Proposition 4.7), and that the gradient boosting base learner provides an approximately optimal test direction (Corollary 4.8).
- **Extensive empirical validation**: Evaluates on synthetic experiments (regression, classification, ranking, survival) and 13 real-world tabular benchmarks.

## Methodology

**Core statistical framework.** ScoreStop builds on the functional score test framework of Hudson et al. (2026). For a population risk minimizer f₀ = arg min E[L{f(X), W}], the Gateaux (directional) derivative at a hypothesized function f* in direction h is R'₀(h; f) = E[h(X)∇L{f(X), W}]. Since f₀ is a minimizer, R'₀(h; f₀) = 0 for all directions h. The key insight is to test, at each boosting iteration m, whether the current estimate f̂ₘ equals f₀ by checking whether the empirical directional derivative is statistically distinguishable from zero.

**The ScoreStop statistic.** Given validation data {Wᵢ}ᵢ₌₁ⁿ independent of training data, define the score function s(W; h, f) = h(X)∇L{f(X), W}. The empirical directional derivative is R̂'ₙ(h; f) = Eₙ[s(W; h, f)]. Under H₀: f* = f₀, the Central Limit Theorem gives √n R̂'ₙ(h; f*) → N(0, E[s(W; h, f*)²]). Estimating the variance and squaring yields the ScoreStop statistic:

Tₙ(h, f*) = n Eₙ[s(W; h, f*)]² / Eₙ[s(W; h, f*)²]

This statistic is **scale-invariant** in both h and ∇L (replacing either with a nonzero scalar multiple leaves Tₙ unchanged), and **asymptotically χ²₁-distributed** under the null.

**Handling implicit and data-dependent losses.** For implicit losses like LambdaRank, where the user specifies a gradient function λₖ{fᴺ(X), W} but no explicit loss exists, the score function is constructed directly from the user-specified gradient analogously to the smooth case, provided a Poincaré symmetry condition holds. For data-dependent losses like Cox regression, where the gradient depends on an unknown nuisance function (the baseline cumulative hazard Λ₀), the score contribution is replaced by an **influence function** φ(W; h, f*, P̂ₙ) that accounts for the uncertainty in estimating the nuisance function via a one-step debiased estimator. The general ScoreStop statistic then becomes:

Tₙ(h, f*) = n Eₙ[φ̂(h, f*)]² / Eₙ[φ̂(h, f*)²]

**Algorithm variants.** Algorithm 1 (FWD-SS) fits a base learner ĥₘ approximating the gradient on training data, evaluates Tₙ(ĥₘ, f̂ₘ) on validation data, and stops if Tₙ ≤ cₐ. BWD-SS instead tests in the previous direction ĥₘ₋₁. STAB-SS tests both directions jointly using a multi-direction Hotelling-type statistic that is asymptotically χ²₂ under the null.

**Threshold calibration.** The threshold cₐ is related to a significance level via cₐ = F⁻¹_χ²_d(1 − α). However, rather than controlling type-I error, the threshold serves as a **regularization parameter** for prediction: large cₐ (small α) stops early and may underfit; small cₐ (large α) permits more iterations and may overfit. The paper communicates thresholds via the two-sided z-score z = Φ⁻¹(1 − α/2), with recommended practical values z ∈ {0.05, 0.1}.

## Results & Highlights

**Synthetic experiments.** Four synthetic tasks with known population minimizers (regression, binary classification, LambdaRank learning-to-rank, Cox survival) were evaluated over 100 Monte Carlo seeds with n_tr = 2000, n_val = 500, n_te = 10000, using LightGBM with learning rate η = 0.05. Key findings:
- ScoreStop is **competitive with loss-based stopping when patience is small** (P ∈ {1, 3, 5}), with comparable median excess test loss.
- ScoreStop is **mostly beaten by loss-based methods when patience is large** (P ∈ {20, 50, 100}), suggesting patience with large P can exploit smoother validation trajectories.
- ScoreStop is **particularly strong on learning-to-rank**, where loss-based stopping performs poorly because the nonsmooth target NDCG loss is only indirectly tied to the training gradient. In this setting, ScoreStop (which operates on the smooth implicit gradient) achieves substantially lower excess loss.
- **STAB-SS usually outperforms FWD-SS and BWD-SS**, especially on regression and survival tasks, likely due to reduced reliance on any single base learner. The authors recommend STAB-SS or FWD-SS with z ∈ [0.05, 0.1] for general use.
- **Null calibration is empirically validated**: QQ plots confirm that the ScoreStop statistic distribution is well-approximated by χ²_d near the population minimizer, even in finite samples.

**Real-data benchmarks.** Thirteen benchmark tabular datasets spanning regression (California housing, Bike sharing, Abalone), classification (Adult, Credit, Diabetes, Phoneme), Poisson regression (Bike sharing, ACTIVITY), multiclass (Covertype), quantile regression (TP), ranking (MSLR-WEB10K), and survival (FLC, MEPS) were evaluated. Models used LightGBM with 31 leaves and η = 0.05. Results:
- ScoreStop with fixed z = 0.05 is **broadly competitive** with patience-5 and patience-20 baselines across diverse tasks.
- No single ScoreStop variant dominates uniformly; FWD-SS, BWD-SS, and STAB-SS perform similarly overall.
- The method achieves low excess test loss over the retrospective test-loss oracle without seeing test data, with fold-medians and min-max ranges comparable to tuned patience baselines.

**Small learning rate collinearity.** When the learning rate is small, successive boosting updates produce nearly collinear directions, making the STAB-SS covariance matrix approach rank deficiency. The authors note this as a known issue for Wald-type statistics with singular covariance structures and defer detailed analysis.

## Critical Assessment

**Novelty:** The paper offers a genuinely novel perspective on early stopping by recasting it as a sequence of formal hypothesis tests rather than heuristic loss monitoring. While functional score tests for function-valued parameters were introduced by Hudson et al. (2026), the application to iterative early stopping in gradient boosting—and the specific construction using the base learner as an approximately optimal test direction—is new. The closest prior work (Mahsereci et al., 2017) uses a heuristic comparison of gradient mean and variance without a calibrated null distribution.

**Rigor:** The theoretical development is careful and well-structured. Propositions 4.6 and 4.7 establish null distribution and power consistency under clearly stated regularity assumptions (influence function centrality, direction linearity, finite variance, regular asymptotic linearity, moment consistency). The extension to data-dependent losses via influence functions is technically sound, following established semiparametric methodology. However, the paper does not explicitly address **multiple testing**: since a test is conducted at every boosting iteration, the overall family-wise error rate is not controlled. The authors acknowledge this indirectly by treating cₐ as a regularization parameter rather than a strict type-I error controller, but a formal analysis of the sequential testing aspect would strengthen the theory.

**Reproducibility:** The experiments use standard tools (LightGBM) and publicly available datasets (OpenML, MSLR-WEB10K, MEPS). Synthetic DGPs are fully specified with equations. The threshold scale (Table 1) and algorithm pseudocode (Algorithms 1–2) are provided. However, **no code is released** with the paper, which limits practical reproducibility. The real-data results are presented as fold medians with min-max ranges, but individual fold results or variance estimates are not tabulated.

**Significance:** The practical significance is moderate-to-high for practitioners working with non-standard losses in gradient boosting. The ability to handle LambdaRank and Cox regression without explicit loss evaluation fills a real gap. For standard regression/classification with well-behaved validation losses, the benefit over a small-patience baseline is incremental. The method's main value proposition is **eliminating the patience hyperparameter** (which is dataset- and metric-sensitive) in favor of a single, interpretable, calibrated threshold.

**Red Flags:**
1. **Multiple testing / sequential testing**: The paper does not claim anytime-valid type-I error control and explicitly notes that the sequence of statistics is computed on a single validation sample with dependent successive statistics. This is honest but means the χ² calibration is only a heuristic regularization criterion in practice, not a formally valid sequential test.
2. **Asymptotic approximations in finite samples**: While QQ plots show good calibration near the minimizer, the validity of the asymptotic χ² approximation at early iterations (far from f₀) is not established—and this is precisely where the stopping decision matters most.
3. **Collinearity at small learning rates**: STAB-SS can become unstable when successive directions are nearly collinear, which is common in practice with small learning rates. The authors flag this but do not provide a practical workaround.
4. **"Competitive" rather than superior**: The real-data results show ScoreStop is competitive but rarely dominant over well-tuned patience baselines. The strongest advantage is on learning-to-rank; for standard tasks, practitioners may not see clear gains.
5. **No lookback mechanism**: Unlike patience-based methods that return the best validation-loss iterate in a window, ScoreStop commits online to the current iterate. The authors acknowledge this as a limitation and suggest a lookback variant as future work.

## Strengths

- **Statistically principled framework** with clear hypothesis testing interpretation and calibrated thresholds.
- **Scale invariance** eliminates arbitrary tuning of loss-specific scales or patience parameters.
- **Broad applicability** to implicit losses (LambdaRank) and data-dependent losses (Cox regression) where traditional validation-loss monitoring is difficult or biased.
- **Solid theoretical foundation** with asymptotic null distribution, power consistency, and approximately optimal direction selection.
- **Extensive empirical coverage** across synthetic and real tasks spanning regression, classification, count data, quantile regression, ranking, and survival analysis.
- **Honest and self-aware discussion** of limitations (no lookback, no anytime-validity, collinearity issues, online-vs-lookback distinction).

## Limitations

- **No multiple testing correction** for the sequential nature of iteration-by-iteration testing; the χ² calibration is a regularization heuristic rather than a formally valid sequential test.
- **Asymptotic theory may not hold well** at early iterations far from the population minimizer, exactly when stopping decisions are most consequential.
- **STAB-SS instability** under small learning rates due to near-collinearity of successive update directions.
- **No code release**, limiting reproducibility and practical adoption.
- **Online commitment** (returning the current iterate) vs. lookback rules that can return earlier, better-performing iterates; this partly explains why large-patience baselines sometimes win.
- **Limited advantage on standard smooth losses** where validation-loss monitoring already works well; the strongest case is for non-standard losses.
- **Influence function construction** for data-dependent losses requires specialized statistical expertise and case-by-case derivation (e.g., for Cox regression), which may limit accessibility for general ML practitioners.

## Relevance

该论文将经典统计推断中的得分检验框架引入梯度提升的早停问题，与用户关注的统计学习理论和机器学习方法论研究兴趣高度相关。尤其在处理排序学习（LambdaRank）和生存分析（Cox 回归）等非标准损失函数时，ScoreStop 提供了一种有理论依据且无需调参的替代方案，具有潜在的实用价值。

## Questions for Further Exploration

1. **Can the ScoreStop framework be extended to deep neural networks?** The paper briefly notes that ĥₘ = f̂ₘ₊₁ − f̂ₘ defines a natural test direction for any iterative procedure, but the practical challenges (high-dimensional parameter spaces, mini-batch gradients, non-smooth optimization landscapes) remain unexplored.
2. **How does ScoreStop interact with other regularization mechanisms?** Gradient boosting typically employs learning rate shrinkage, subsampling, and tree complexity constraints. A systematic study of how ScoreStop thresholds should be adjusted when these co-vary would be valuable.
3. **Can a lookback variant of ScoreStop be developed?** The authors note that patience rules benefit from returning the best validation-loss iterate in a window. A lookback ScoreStop that maintains a small buffer of recent iterates and selects based on the minimum statistic could close this gap.
4. **What is the finite-sample behavior at early iterations?** The asymptotic χ² result is established for fixed h and n → ∞, but early in boosting f̂ₘ is far from f₀. Does the statistic remain well-behaved, or could it produce spurious stopping signals?
5. **How sensitive is ScoreStop to validation set size?** The theory requires n → ∞, but in practice validation sets may be small (e.g., 10–20% of modest datasets). An empirical sensitivity analysis varying n_val would clarify practical guidance.
6. **Can anytime-valid inference be integrated?** The authors discuss confidence sequences and e-values as related work but do not combine them with ScoreStop. Developing a martingale-based variant with valid type-I error control across stopping times would be a significant theoretical advance.
