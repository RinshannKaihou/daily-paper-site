SkillCenter: A Large-Scale Source-Grounded Skill Library for Autonomous AI Agents

01Key metric — oracle vs. keyword

0 → 61–100%

On knowledge-gap probes: baseline 0%, +Keyword also 0%, +Oracle 61–100%. Retrieval recall — not coverage or injection — is the binding constraint.

02Why this matters

Skills only help when a model lacks the knowledge AND the right skill is retrieved — a clean, honest map of where skill injection actually pays off.

03Critical edge

The released title-only FTS5 retrieval is the direct cause of the 0% keyword arm — the shipped system is, by the authors' own evaluation, not yet fit for its deployment path.

Source-grounding as traceability. Each retained claim must map, via deterministic substring matching (≤20-word quotes), to an exact passage in the original source; every skill carries a source URL, SPDX license, and a GPT-5.2-assigned 1-to-5 quality score.

Offline-searchable distribution. Each domain is a self-contained SQLite database (four-table schema) with FTS5 over title and domain only; BM25 retrieval is deterministic and auditable, and source URLs are stripped from injected prompts to prevent hallucinated links.

Downstream evaluation. Three probes — a ~2,000-task general pool (4 arms: Baseline / +Keyword / +Placebo length-matched / +Oracle), a 260-task agent-gap probe (spec lives only in an injected skill), and an 18-task real-corpus probe (answers only in a real SkillCenter skill). Significance: exact paired McNemar on discordant outcomes.

 Keyword retrieval never beats baseline across all four solvers; oracle is decisive only when the model lacks the knowledge. Real-corpus probe: keyword still 0% though the needed skill is present — retrieval recall is the bottleneck.

Quality scoring is inflated: mean 3.91, but 82.1% of pipeline skills receive exactly 4; the publish gate removes only ~3%. Redundancy is localized — pipeline domains ≤0.01% near-duplicate, GitHub SkillMD 26.9%, its curated-lite subset 65.0%.

Strengths & limitations side by side

Strengths
  • Unusually honest negative result: keyword ≈ placebo, even harmful for mid-tier models.
  • Placebo arm + exact McNemar cleanly separates content effect from context-token effect.
  • Strong artifact release: frozen manifests, raw captures, reproducible scripts, public bundles.
  • Source-grounding via substring match is a concrete safeguard vs. hallucinated claims.
Limitations
  • 82.1% score-4 collapse: no human eval, no IAA; scores are internal QA only.
  • Single-shot solvers, not multi-turn tool-using agents; pool skews to where skills are least needed.
  • Title-only FTS5 is the binding constraint and is not yet fixed.
  • Proprietary scorer (GPT-5.2) breaks exact reproducibility; web skills 28.5% uncleared on license.

Open questions for further exploration

  1. Would replacing title-only BM25 with full-body or dense semantic retrieval close the 0% keyword-arm gap on the real-corpus probe, and at what latency/storage cost?
  2. Does the oracle-style gain (0% → 61–100%) survive in multi-turn, tool-using agentic harnesses rather than single-shot solvers?
  3. Can the 82.1% score-4 collapse be fixed with human calibration (the planned 2,000-skill stratified annotation) or multi-LLM cross-scoring, and do per-domain orderings then become meaningful?
  4. How much true redundancy (paraphrases) is hidden behind the lexical MinHash lower bound, given 7 sentence encoders disagreed (49–93% at cosine 0.9)?
  5. What is the downstream impact on real software-engineering tasks (SWE-bench-style issue resolution) rather than synthetic/algorithmic pools?
  6. Can source-level prompt-injection screening be integrated into SkillGate without rejecting security/DevOps skills that legitimately quote dangerous patterns?
