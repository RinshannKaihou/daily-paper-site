Reason Less, Verify More: Deterministic Gates Recover a Silent Policy-Violation Failure Mode in Tool-Using LLM Agents

+12.4pp on the 50-task τ²-bench airline set (P=0.0012); reproduced +12.3pp on a disjoint 15-seed set. Gates fire on only 26 tasks yet carry +25 of +31 net wins.
Why this matters
It exposes a hidden failure class in mainstream agent benchmarks and offers a zero-LLM-call, auditable, fail-open runtime gate that any tool-using agent can adopt.
Critical edge
The lift rests almost entirely on one gate (cancellation_eligibility, 100% precision); a second headline gate, baggage_allowance, runs at just 5% precision — so "four-gate suite" oversells a single-gate result.

Methodology — the mechanism, distilled

Agate is a pure predicate g(tool_name, args, db_state) → {allow, reject} evaluated before any mutating call runs. The target failure class is the silent policy violation: on a policy-permissive tool, a forbidden write executes with no error, corrupting state invisibly (78% of airline failures are this kind, reproducible across seeds).

On rejection the handler is short-circuited and the agent receives a structured reason to re-plan; on allow, execution is unchanged. Gates are fail-open: if a gate itself throws, the original call runs and the error is logged. The headline four-gate suite — cancellation_eligibility, baggage_allowance, passenger_count, must_read_before_write — is fixed on the budget tier, uses no LLM calls, writes no state, and leaks no ground-truth evaluator info. The harness runs τ²-bench airline in library mode so vanilla and gated share one code path; success is reward > 0, scored with paired task-level bootstrap (20,000 resamples).

Strengths

  • Disciplined claim-scoping: states what it does not claim as clearly as what it does.
  • Firing-share decomposition (Δ≈ p_fire×Δ_fire) pre-empts "is the lift even from the gates?" — +25 of +31 net wins land in the firing stratum.
  • Two genuine negative controls (retail, BFCL) bound the claim instead of asserting it.
  • passk collapse is the most persuasive evidence a recurring failure class is being removed.
  • Per-gate audit openly reports a 5%-precision gate — unusual honesty.

Limitations

  • Single positive domain. Only τ²-bench airline satisfies all five admission axes; generality is a conjecture.
  • Lift in one gate. "Four-gate suite" obscures that cancellation_eligibility carries nearly all the task-success lift.
  • No recovery guarantee. Task #39 fires repeatedly yet stays 0/5 — agent loops against the rejection.
  • Task-overfitting risk. Gates written from the policy, evaluated on the same 50 tasks; replication is over seeds, not held-out tasks.
  • Rejection-message confound. The structured reason is itself a planning signal; block vs. feedback not isolated.

Open questions

  1. Would a single high-precision cancellation_eligibility gate match the suite, or does the multi-gate framing add real task-success lift beyond policy coverage?
  2. How much of the recovery comes from the deterministic block versus the structured rejection message? A generic "rejected" ablation would isolate the confound.
  3. Does forcefully prompting the four rules (or a one-step reflection) recover a comparable lift on the non-deceptive subset?
  4. Does the failure class generalize beyond airline cancellation once a second violation-inducing benchmark is built?
  5. At what model strength does the silent-violation rate drop below the gate's marginal value — given gpt-5.2 still fires 43/250 times?
