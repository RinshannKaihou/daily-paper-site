> **一句话总结：** 论文识别出工具型 LLM Agent 在"策略宽松型"工具上的"静默策略违规"失败模式（违规写操作无报错地破坏状态），并提出一个轻量、只读、确定性的"预执行闸门"机制（4 个闸门的小谓词套件），在 τ²-bench airline 基准上将 gpt-4o-mini 的成功率从 29.6% 提升至 42.0%（+12.4pp，并在 15-seed 独立复现集上重现 +12.3pp），同时在策略自执行的零售域与 BFCL 上表现为负对照。

> **TL;DR:** The paper identifies "silent policy violations" — forbidden writes that policy-permissive tools execute without any error, corrupting state invisibly — as a distinct trust failure in tool-using LLM agents, and shows that a lightweight suite of four deterministic, read-only pre-execution gates over tool arguments and DB state recovers a meaningful slice of these failures on the τ²-bench airline domain: gpt-4o-mini pass1 rises 29.6% → 42.0% (+12.4pp, P=0.0012), reproduced on a disjoint 15-seed set (+12.3pp, P=0.0008), with the lift concentrating on the 26 tasks where gates actually fire (+19.2pp) and vanishing (near-zero) on self-enforcing retail tools and BFCL, where there is no silent-violation class to intercept.

## 核心贡献

- **识别一种被忽视的失败模式：** "静默策略违规"——在策略宽松型工具（policy-permissive tools）上，违规写操作会被静默执行、无报错，导致不可恢复的状态损坏；在 τ²-bench airline 上 78% 的失败属于此类，且该比例跨独立 seed 可复现，并非采样噪声。
- **提出轻量确定性干预：** 一个 4 闸门套件（cancellation_eligibility、baggage_allowance、passenger_count、must_read_before_write），每个闸门是关于工具调用参数与当前数据库状态的纯确定性谓词，无 LLM 调用、无写状态、无真值评测器信息，并在出错时 fail-open。
- **可复现的实证结果：** 在完整 50 任务 airline 基准上把 gpt-4o-mini 成功率从 29.6% 提到 42.0%；在 15-seed 独立复现集上重现 +12.3pp（P=0.0008），baseline 与 lift 都在 0.1–1.2pp 内稳定。
- **严格的边界与负对照分析：** 通过 firing-share 分解证明 lift 集中在闸门实际触发的任务上；并在两个负对照（自执行的 retail 域 -4.7pp、BFCL 零触发 52.5% vs 51.0%）上划定机制边界——闸门只在工具宽松、策略可状态判定、且任务会诱导违规时有效。
- **评测方法论贡献：** 提出 5 条基准准入轴（A1–A5），并指出当前大多数 Agent 基准不满足"策略宽松 + 可状态判定 + 诱导违规"的组合，致使该失败类难以被独立复现验证。

## Key Contributions

- **A new failure-mode characterization:** "Silent policy violations" on policy-permissive tools — valid tool calls that mutate state against domain policy while raising no error. On τ²-bench airline, 78% of observed failures are silent wrong-states, and the failure rate is reproducible across disjoint seeds, ruling out sampling noise.
- **A deterministic, read-only intervention:** A four-gate suite (cancellation_eligibility, baggage_allowance, passenger_count, must_read_before_write), each a pure predicate over tool args and DB state — no LLM calls, no writes, no ground-truth evaluator leakage, and fail-open on internal exceptions.
- **A replicated empirical result:** gpt-4o-mini pass1 lifts 29.6% → 42.0% (+12.4pp, paired bootstrap P=0.0012); the lift reproduces at +12.3pp (P=0.0008) on a disjoint 15-seed replication set with three times the trials, while the baseline reproduces within 1.2pp.
- **A sharp boundary analysis:** Aggregate lift decomposes as Δaggregate ≈ p_fire × Δfire; +25 of +31 net successful trials occur in the firing stratum, while the non-firing stratum's +5.0pp interval includes zero. Two negative controls (self-enforcing retail: −4.7pp; BFCL v4 multi_turn_base: zero firings) bound the mechanism.
- **A benchmark admission framework:** Five axes (A1 structured calls, A2 policy-permissive tools, A3 state-decidable policy, A4 final-state evaluation, A5 violation-inducing tasks) are required to even evaluate this failure class; the authors document that WorkBench and most other benchmarks fail at least one axis, motivating future benchmark construction.

## Methodology

**Failure class.** The paper targets a specific error class: valid tool calls that mutate state while violating a natural-language policy the model was instructed to follow. The defining precondition is a *policy-permissive tool* — one that enforces syntax/existence checks but not full domain policy (e.g., an `cancel_reservation` tool that sets status to cancelled as long as the reservation exists, even when fare class/timing/insurance/flown-segment rules forbid it). The result is a silent wrong state with no tool error in the trace. This is contrasted with *self-enforcing tools* (e.g., retail cancellation that rejects invalid reasons), where a forbidden call fails loudly and recoverably.

**Gate mechanism.** A gate is a deterministic predicate g(tool_name, args, db_state) → {allow, reject}, evaluated before any mutating call executes. On rejection the underlying handler is short-circuited and the agent receives a structured rejection reason to re-plan against; on allow, execution proceeds unchanged. The implementation is **fail-open**: if a gate itself throws, the original call runs and the error is logged. Gates are explicitly distinguished from reflection (model self-critique), output guardrails (post-hoc content moderation), and LLM judges (another stochastic call).

**The headline four-gate suite** (fixed on the budget tier and held constant across tiers): `cancellation_eligibility` (block ineligible cancellations under fare/timing/insurance/flown-segment rules), `baggage_allowance` (block mispriced free bags or policy-violating removals), `passenger_count` (block changes to the immutable passenger count), and `must_read_before_write` (block writes to unread records).

**Harness.** τ²-bench airline is run as a library so vanilla and gated conditions share one code path, differing only by gate presence. Two idempotent patches are applied: (1) a pre-call dispatcher that intercepts proposed tool calls and bypasses the handler on rejection, and (2) an evaluation-replay scrub that removes rejected calls from recorded history before the benchmark evaluator replays actions in a fresh environment. Success is reward_info.reward > 0.

**Conditions.** Budget tier: gpt-4o-mini agent + gpt-4.1 user simulator, temperature 0, on the annotation-corrected τ²-bench airline task set via the official registry, 50 tasks × 5 trials = 250 trials/condition. Frontier tier: gpt-5.2 at harness default reasoning, same user simulator, in-harness vanilla baseline 61.2%; explicitly flagged as non-leaderboard, n=5, unreplicated. A six-gate `optimal_plus` suite is reported only as a budget-tier upper bound.

**Statistics.** pass1 and the τ-bench unbiased passk estimator; paired task-level bootstrap with the task as resampling unit, 20,000 resamples, 95% percentile CIs. Gates add no model calls — runtime cost is deterministic reads and predicate evaluation only.

## Results & Highlights

- **Aggregate lift (budget model, headline).** On the full 50-task τ²-bench airline benchmark, the four-gate suite raises gpt-4o-mini pass1 from **29.6% (74/250) to 42.0% (105/250)**, a **+12.4pp** gain (95% CI [+4.0, +21.2], paired task-level bootstrap **P = 0.0012**). This is the paper's central statistical anchor.
- **Disjoint-seed replication.** On a 15-seed replication set (750 trials/condition, disjoint seeds, 3× the trials) the lift reproduces at **+12.3pp** (95% CI [+4.1, +21.3], **P = 0.0008**); the vanilla baseline reproduces within 1.2pp (30.8%) and the gated within 0.1pp (43.1%), supporting that the improvement is a stable property rather than a seed artifact.
- **Firing-stratum concentration (mechanism check).** Of the +31 net successful trials, **+25 occur in the 26-task firing stratum**, where vanilla 13.8% (18/130) rises to 33.1% (43/130), **+19.2pp** (CI [+6.9, +33.1], P=0.0006). The 24-task non-firing stratum moves only +5.0pp with a CI that includes zero (P=0.18), so the authors explicitly do not claim gates help non-firing tasks.
- **Reliability under passk.** pass1 → pass5 collapse: vanilla falls **29.6% → 8.0%** while the four-gate suite falls **42.0% → 26.0%** — at k=5 the gated pass rate is more than 3× vanilla, supporting the claim that gates remove a *recurring* failure mode rather than adding random wins.
- **Canonical deceptive task #48** (user supplies false context to induce an out-of-policy cancellation): the verified configuration solves it **16/16** across budget/gpt-4.1 model-setting combinations, while non-gated conditions solve it at most **1/16**. The gate wins because eligibility is read from DB state, not the user's asserted context.
- **Frontier-model (suggestive, not anchor).** gpt-5.2 at default reasoning: in-harness vanilla baseline 61.2% (153/250) rises to 71.6% (179/250), **+10.4pp** (CI [+0.4, +20.8], P=0.020, n=5, unreplicated). Gates fire on 43/250 frontier trials across 18 tasks, directly showing the frontier model still attempts policy-violating writes; the firing stratum shows +33.3pp while the non-firing stratum is −2.5pp.
- **Per-gate precision audit.** Over the five-gate candidate set: `cancellation_eligibility` is the dominant load-bearing gate — 161 fires, 161 true blocks, **100% precision**, and the only gate whose removal lowers pass1 (removal Δ = −2). `baggage_allowance` is essentially broken: 42 fires, 40 false blocks, **5% precision** (a candidate for revision). `passenger_count` is 100% precision over 9 fires; `basic_economy` 83%; `must_read_before_write` 78%.
- **Negative control 1 — self-enforcing retail.** On 8 τ²-bench retail tasks (tools already enforce preconditions), a retail gate suite moves performance **40/64 (62.5%) → 37/64 (57.8%)**, a −4.7pp point estimate whose CI includes zero (partly explained by an encoding bug that blocked corrected retries). No silent-violation class exists to recover.
- **Negative control 2 — BFCL v4 multi_turn_base.** A schema-existence gate produces **zero firings across 200 entries**; performance stays within run-to-run variance (52.5% vanilla vs 51.0% gated). Dominant errors are wrong-sequence/instance-state mismatches, not silent policy-violating writes.
- **Subset inflation caveat.** The same run yields +30.0pp on a curated 8-task subset enriched for the failure class — the authors explicitly state the full-50 result is the appropriate headline and subset results overstate representative performance.

## Critical Assessment

- **Novelty: Moderate but genuinely additive.** Runtime enforcement / pre-execution monitors are not new (AEGIS [12], AgentSpec [9], formal policy enforcement [3], solver-aided verification [10] all establish feasibility). The paper's distinct contribution is the *empirical question*: when can action-boundary enforcement *raise benchmark task success*, not merely bound safety cost. The framing of "silent policy violation as a failure class distinct from loud tool errors" is a useful conceptual sharpening.
- **Rigor: High for a workshop paper, with one weak point.** Strengths: disjoint-seed replication, paired bootstrap, firing-share decomposition that mechanistically explains the lift, two negative controls that bound the claim, and an honest per-gate audit. Weak point: **the headline lift is almost entirely carried by a single gate** (`cancellation_eligibility`, 100% precision, the only gate whose removal lowers pass1), so the "four-gate suite" framing somewhat overstates the contribution — the honest result is closer to "one high-precision cancellation gate recovers most of the lift on this specific airline task distribution."
- **Reproducibility: Strong.** Temperature 0, official τ²-bench registry, annotation-corrected task set, library-mode harness with two idempotent no-op-when-ungated patches, and a replication protocol with disjoint seeds. The main reproducibility gap is the held-out-task issue: gates are written from the policy and evaluated on the same 50 tasks, with replication over seeds rather than held-out tasks — the authors flag this as limitation #8.
- **Significance: Bounded and honestly scoped.** Single positive domain (τ²-bench airline); the authors searched for a second (WorkBench is the closest near-miss but rarely induces violations) and did not find one. The claim is explicitly narrowed to "policy-permissive tools with state-decidable policy and final-state evaluation" — a real but narrow slice of deployment.
- **Red flags / overclaiming risks:**
  - **`baggage_allowance` gate has 5% precision (40 false blocks out of 42 fires)** — included in the headline four-gate suite yet effectively broken; the paper is transparent about this but it muddies the "four-gate suite" headline.
  - **Rejection-message confound (limitation #9):** the structured rejection reason is itself a feedback signal the agent re-plans on, so part of the recovery may come from the message rather than the block. The deterministic block guarantee is unaffected, but the *task-success* attribution is not cleanly isolated.
  - **Frontier result is unreplicated (n=5, lower CI bound +0.4pp near zero)** and is appropriately demoted to "suggestive evidence" rather than a statistical anchor — the authors handle this correctly.
  - **No comparison to cheaper alternatives** (limitation #7): the authors do not test whether forcefully prompting the gated rules, or adding a reflection step, recovers the same lift. They argue prompting cannot help the deceptive-task subset (where the model is misled about state), but the non-deceptive failures remain an open comparison.

## Strengths

- Exceptionally disciplined claim-scoping: the paper states what it does *not* claim (no general agent-safety solution, no task-success guarantee, no broad domain generality) as clearly as what it does.
- The firing-share decomposition (Δaggregate ≈ p_fire × Δfire) is a clean mechanism argument that pre-empts the "is the lift even from the gates?" objection — and the data shows +25 of +31 net wins land in the firing stratum.
- Two genuine negative controls (self-enforcing retail, BFCL) make the boundary claim credible rather than asserted.
- passk analysis (vanilla collapses 29.6% → 8.0% while gated holds 42.0% → 26.0%) is the most persuasive evidence that a *recurring* failure class is being removed, not noise smoothed away.
- Intellectual honesty in the per-gate audit: openly reporting that one headline gate has 5% precision and that the lift rests almost entirely on cancellation_eligibility.

## Limitations

- **Single positive domain.** Only τ²-bench airline satisfies all five admission axes; generality of the failure class is a conjecture supported by one strong instance and two negative controls.
- **Lift concentrated in one gate.** The "four-gate suite" framing obscures that cancellation_eligibility carries essentially all the task-success lift; the other gates extend policy coverage but are not load-bearing on this distribution, and one (baggage_allowance) is broken at 5% precision.
- **No recovery guarantee.** A gate blocks a violating write but does not ensure the agent recovers — task #39 fires repeatedly yet stays 0/5 as the agent loops against the rejection.
- **Task-overfitting risk.** Gates are written from the policy and evaluated on the same 50 tasks; replication is over seeds, not held-out tasks.
- **Rejection-message confound.** The structured reason returned on rejection is itself a planning signal, so task-success attribution between "the block" and "the informative feedback" is not isolated.
- **Untested baselines.** No comparison against forcefully prompting the rules or adding reflection; the non-deceptive failures remain open.
- **State-decidability requirement.** The method applies only when policy can be decided from current state and arguments; policies needing ambiguity resolution, legal interpretation, or human judgment need richer mechanisms.
- **Not leaderboard-comparable.** All numbers are within-harness under a specific user simulator and reasoning configuration; the gated scaffold modifies the agent environment.
- **Frontier evidence unreplicated.** gpt-5.2 result is n=5, lower CI near zero, demoted to "suggestive."

## Relevance

对关注 LLM Agent 工具调用安全性与可靠性的研究方向高度相关：它精确刻画了"策略宽松型工具上的静默违规"这一在 τ-bench/τ²-bench 这类主流 Agent 基准中普遍存在却被 pass/fail 指标掩盖的失败模式，并提供了一个零 LLM 调用、可审计、fail-open 的轻量运行时闸门范式，直接呼应运行时执行/策略合规/Agent 评测的可信化议题。

## Questions for Further Exploration

- Would a single high-precision `cancellation_eligibility` gate match the four-gate suite on this distribution, and does the multi-gate framing add anything beyond policy coverage that does not translate to task-success lift?
- Can the firing-share decomposition and passk analysis be reproduced on a *second* policy-permissive, violation-inducing benchmark once one is constructed — does the failure class generalize beyond airline cancellation?
- How much of the task-success lift comes from the **deterministic block** versus the **structured rejection message** the agent re-plans against? An ablation returning a generic "rejected" vs. an informative reason would isolate the confound.
- Does forcefully prompting the four policy rules, or adding a one-step reflection, recover a comparable lift on the non-deceptive subset — i.e., is the gate's value primarily in the deceptive cases where the model is misled about state?
- Can gates be auto-generated or auto-tuned per-model from a declarative policy spec, given that one hand-written gate (baggage_allowance) reached only 5% precision and three frontier-tier tasks both fired and regressed?
- How does the failure class interact with scale — the frontier gpt-5.2 still attempts violating writes (43/250 fires), but at what model strength does the silent-violation rate drop below the gate's marginal value?
