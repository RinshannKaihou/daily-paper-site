> **一句话总结：** SpaCellAgent 是一个基于 LLM 的多智能体框架，通过规划器、执行器、评估器、报告器四类智能体协作，结合自精炼与跨任务自演化记忆机制，实现了单细胞与空间转录组轨迹推断（TI）的端到端自动化；在 REAL-GOLD/REAL-SILVER/SYNTHETIC 等基准上达到 SOTA 拓扑保真度，并将专家级分析时间从 64.6 分钟降至 38.0 分钟（提升 41.2%）。

> **TL;DR:** SpaCellAgent is an autonomous LLM-driven multi-agent framework (planner/executor/evaluator/reporter) that performs end-to-end trajectory inference on scRNA-seq and spatial transcriptomics data, using self-refinement and a cross-task self-evolution memory to reach SOTA topological fidelity. On the REAL-GOLD benchmark it lifts Correlation to 0.480 and F1 to 0.608 (vs. Slingshot's 0.347/0.483), and on REAL-SILVER raises F1 to 0.645, while cutting total analysis time from a 64.6-minute human-expert baseline to 38.0 minutes (41.2% reduction) and beating the w/o-evolution variant by 32.7%.

## 核心贡献

- 提出 SpaCellAgent，首个面向单细胞与空间转录组轨迹推断（TI）的端到端、闭环式 LLM 多智能体框架，可从自然语言任务描述自动生成可执行分析工作流并产出可解释的生物报告。
- 设计了四智能体协作架构：Planner（任务分解为可执行步骤）、Executor（含语义式 Tool Selector 与跨 Python/R 的 Coder）、Evaluator（双层代码+生物学验证）、Reporter，并通过动态工具发现与注册机制让工具库自主扩展。
- 引入自精炼（within-task 迭代纠错）与自演化（cross-task 全局记忆，归档成功代码与错误-修复对，按 RAG 方式复用）双重机制，使系统能持续学习并提升收敛速度与稳定性。
- 在 6 个异构数据集（REAL-GOLD、REAL-SILVER、SYNTHETIC 基准 + 小鼠背侧中脑、蝾螈神经元再生、未发表的小鼠脊髓损伤 SCI 数据）上系统评测，证明其在多数指标达到 SOTA，并将分析效率提升 41.2%。
- 引入知识增强回退机制：当任务超出经验分布时，Biological Evaluator 自动检索 PubMed 最新文献作为参考依据，避免启发式猜测。

## Key Contributions

- Proposes SpaCellAgent, the first end-to-end closed-loop LLM multi-agent framework for trajectory inference (TI) over both scRNA-seq and spatial transcriptomics data, turning natural-language specifications into optimized analytical workflows with interpretable biological reports.
- Designs a four-agent collaborative architecture — Planner (decomposes intent into a JSON action sequence), Executor (a semantic Tool Selector plus a polyglot Coder spanning Python and R), Evaluator (dual-layer code + biological validation), and Reporter — augmented by a dynamic tool discovery/registration mechanism that lets the tool registry grow autonomously.
- Introduces a within-task self-refinement loop (bounded iterative correction driven by evaluator feedback with strategy-divergence prompts) and a cross-task self-evolution module that archives validated code, reasoning patterns, and error-fix pairs into a dual-layer local/global memory for RAG-style reuse.
- Validates on six heterogeneous datasets (REAL-GOLD, REAL-SILVER, SYNTHETIC benchmarks plus Mouse Dorsal Midbrain, Axolotl Neuron Regeneration, and an unpublished Mouse Spinal Cord Injury dataset), showing SOTA topological fidelity and a 41.2% reduction in analysis time versus human experts.
- Adds a knowledge-augmented fallback in the Biological Evaluator that queries PubMed for tissue-specific markers and case studies when a TI task falls outside the system's experience memory, grounding judgments in peer-reviewed evidence rather than heuristic guessing.

## Methodology

SpaCellAgent is built around a closed-loop, multi-agent pipeline driven by an LLM (DeepSeek-V3 in all reported experiments, with GPT-5.2 and claude-sonnet-4-6 used in cross-model sensitivity tests; temperature 0.0 for execution, 0.4 for self-reflection; 128k max output tokens).

- **Planner agent.** Takes the natural-language query plus a structured metadata profile (gene counts, sparsity, spatial modality) and uses LLM planning instructions to synthesize a directed action sequence encoded as a JSON file, each node carrying a unique ID and descriptive label. This machine-readable backbone supports both robust execution and human-in-the-loop verification.
- **Executor agent.** Bridges planning and execution via two components:
  - *Tool Selector* — a semantic router mapping high-level task descriptions to specific computational tools (scanpy, Monocle, PAGA, Slingshot, SpaceFlow, STORIES, etc.) from a curated registry, dynamically configuring the pipeline based on dataset characteristics rather than static defaults.
  - *Coder* — synthesizes executable Python/R scripts, managing serialization of single-cell objects and the flow of pseudotime/trajectory coordinates between heterogeneous environments. A dynamic tool discovery mechanism lets it import and register novel packages when the existing registry is insufficient.
- **Evaluator agent.** Dual-layer validation:
  - *Code evaluator* inspects execution logs, verifies output files, extracts error context, and routes actionable debugging feedback to the executor, skipping biological validation to save tokens when runtime exceptions occur.
  - *Biological evaluator* ingests plots and statistical summaries and uses LLMs cross-referenced with biological priors to flag common TI artifacts (impossible lineage transitions, inverted developmental paths, terminally differentiated cells misassigned as roots). A knowledge-augmented fallback queries PubMed for tissue-specific markers when tasks fall outside the experience memory.
- **Reporter.** Synthesizes the final biological insights and visual outputs into a comprehensive report.
- **Self-refinement.** For each step, successful iterations commit to the intermediate dataset state; failures trigger a refinement cycle in which the coder receives the full interaction history, truncated error logs, and semantic feedback, with prompts enforcing strategy divergence to avoid repetitive errors. The loop persists until success or the attempt limit.
- **Cross-task self-evolution.** Successful TI workflows are archived with biological contextual metadata into a global memory repository; the coder retrieves relevant templates for new datasets (RAG-style) and proactively applies accumulated error-fix pairs to recurrent issues. A dual-layer memory (ephemeral local intra-task buffer + persistent global inter-task knowledge base) supports both context-aware debugging and knowledge transfer.
- **Evaluation metrics.** Four complementary metrics: Spearman Correlation (Corr) of geodesic distance matrices; F1 Branches (Jaccard-based structural correspondence); feature-importance weighted correlation (wCor) via Random Forest regression on gene importance vectors; and Hamming-Ipsen-Mikhailov (HIM) score comparing inferred vs. ground-truth milestone networks. Higher is better for all.

## Results & Highlights

- On **REAL-GOLD**, SpaCellAgent achieves **Corr = 0.480** and **F1 = 0.608**, versus the strongest baseline Slingshot at **Corr = 0.347** and **F1 = 0.483** — a delta of +0.133 Corr and +0.125 F1 — and also leads on HIM (0.710 vs. Slingshot's 0.651).
- On **REAL-SILVER**, SpaCellAgent reaches **Corr = 0.774** (vs. Scorpius 0.602, +0.172) and **F1 = 0.645** (vs. Scorpius 0.574, +0.071), with WCor = 0.719.
- On **SYNTHETIC**, SpaCellAgent attains **Corr = 0.564** (vs. PAGA Tree 0.501, +0.063), **HIM = 0.560**, **WCor = 0.660**, and **F1 = 0.509**, ranking first or near-first across metrics.
- **Efficiency (Table 2):** Total execution time of **38.0 minutes** for SpaCellAgent vs. **64.6 minutes** for the 5-expert human baseline (41.2% reduction) and **56.5 minutes** for the w/o-evolution variant (32.7% reduction). Largest gains are in downstream steps: Step 4 (3.9 vs. 8.4 min, 53.6% faster) and Step 5 (9.6 vs. 26.6 min, 63.9% faster); humans slightly lead only in initial preprocessing (Step 1: 10.0 vs. 12.4 min).
- **Ablation (Task Success Rate, Figure 7):** GPT-4 (w/o agent) performs worst due to inability to manage long-horizon TI workflows; w/o planner and w/o evaluator variants suffer from missing strategic direction and missing self-correction respectively, both significantly degrading robustness versus the full SpaCellAgent.
- **Cross-model sensitivity (Table 3):** On five REAL-GOLD subsets (S1–S5), DeepSeek-V3, GPT-5.2, and claude-sonnet-4-6 produce comparable Spearman correlations (e.g., S5: 0.755 / 0.768 / 0.768) with success rates of 0.8 / 0.9 / 0.9, confirming performance is driven by the agentic framework rather than a specific LLM.
- **LLM-baseline comparison (Table 9):** SpaCellAgent beats CellAgent and scAgents on all five REAL-GOLD subsets (e.g., S1: 0.526 vs. CellAgent 0.473 and scAgents 0.440; S5: 0.755 vs. 0.703 / 0.642).
- **Tool selection consistency (Table 4):** Consistency rate of 1.0 across nearly all steps on Midbrain, Axolotl, and SCI datasets (one 0.9 and one 0.8), with expert quality scores of 8.0 / 7.8 / 7.2 (out of 10).
- **Biological validation (Table 10):** GO enrichment p-values < 0.001 on all three real-world datasets (Midbrain, Axolotl, SCI), statistically supporting the biological fidelity of identified driver genes.
- **Case study (Mouse SCI):** SpaCellAgent autonomously identified a marked differentiation blockade in the high-salt (HS) lineage — premature HS trajectory termination and a significant Plp1 expression gap versus the NS lineage — consistent with prior literature on salt-driven impairment of post-injury neural recovery.

## Critical Assessment

- **Novelty.** The core idea — wrapping heterogeneous TI tools (Monocle/PAGA/Slingshot/SpaceFlow/STORIES) behind an LLM multi-agent orchestrator with self-refinement — is genuinely useful as an automation layer, but the constituent techniques (multi-agent planning, RAG-style memory, PubMed retrieval fallback, dual-layer code/biological evaluation) are largely recombinations of established patterns from ChatDev, ChemCrow, and CellAgent rather than fundamentally new methodology. The strongest genuinely novel element is the *cross-task self-evolution* with error-fix pair accumulation specifically tailored to TI.
- **Rigor.** Evaluation is reasonably broad: four complementary metrics (Corr, F1, wCor, HIM) across three benchmark families plus three real-world spatial datasets, ablations over five architectural variants, cross-model sensitivity on three LLMs, and tool-selection-consistency over 5 repeated trials. However, several gaps weaken confidence:
  - The **efficiency baseline** relies on only **5 experts** and a single standardized 6-step workflow; the 41.2% headline is thus a small-sample estimate with no reported variance for the aggregate, and Step 1 actually shows humans *faster* (10.0 vs. 12.4 min), which is somewhat buried in the narrative.
  - The **Biological Evaluator** that flags "biologically impossible transitions" is itself an LLM judgment; using an LLM-as-judge to validate an LLM-driven pipeline is partially circular, and the paper does not report inter-annotator agreement against the expert panel for these biological pass/fail decisions.
  - Baselines (DPT, RaceID/StemID, Scorpius, PAGA, PAGA Tree, Slingshot) are all run with **default parameters as provided in their original implementations** — explicitly acknowledged in the paper. This is a known weak-baseline pattern: TI methods are highly sensitive to hyperparameter tuning and preprocessing, so defaults systematically disadvantage them, inflating SpaCellAgent's delta. A fairer comparison would tune each baseline on the same metric or report each baseline's recommended configuration per topology.
  - The REAL-GOLD/REAL-SILVER datasets (Table 7) are very small (e.g., 65×1095, 85×1584 cells), where LLM-driven dynamic tool selection has an outsized advantage and variance is high; it would strengthen the claim to show results weighted or stratified by dataset size/topology.
- **Reproducibility.** Code is publicly released, the LLM (DeepSeek-V3), temperatures, and token limits are specified, and datasets are either public (GSE accessions listed in Appendix B.1) or explicitly noted as the unpublished Mouse SCI from a collaborating hospital. The SCI dataset being unpublished and hospital-sourced is a **data quirk**: the most compelling case-study result (HS differentiation blockade) rests on a dataset that cannot yet be independently inspected, and the high-salt-diet framing for a spinal cord injury model is unusual enough to warrant independent confirmation.
- **Significance.** The automation/democratization angle is real and useful for wet-lab biologists, but the framework's value is contingent on LLM API costs and reliability; the paper does not report API cost per analysis, failure modes when the LLM hallucinates tool APIs, or behavior on very large (>100k cell) datasets.
- **Red flags.** (1) Headline "over 40% improvement in analytical efficiency" conflates *time* with *quality* — the paper itself says performance is "expert-aligned" (i.e., comparable, not better) on the human-judged quality dimension, so the 40% is purely a speed claim and should not be read as a quality gain over experts. (2) The 41.2% is measured against a small expert panel on a single workflow; no confidence intervals are reported. (3) "SOTA" claims on REAL-GOLD/REAL-SILVER are made against baselines at default settings, which the TI benchmarking literature (Saelens et al. 2019, ref [31]) itself warns against.

## Strengths

- Genuinely end-to-end: from natural-language query and raw data to executable Python/R code, plots, and a biological narrative report — closing a real gap in the TI tool ecosystem.
- The dual-layer (code + biological) evaluator with a PubMed-grounded fallback is a thoughtful design that addresses a real failure mode of pure code-execution agents.
- Cross-task self-evolution with archived error-fix pairs is a principled way to accumulate domain knowledge and is empirically shown (w/o-evolution ablation, 32.7% slower) to deliver concrete gains.
- Strong empirical coverage: 4 metrics × 3 benchmark families × 3 real-world spatial datasets, plus ablations, cross-model sensitivity, and tool-consistency analyses — far more thorough than typical agent papers.
- Code and data released; accepted to KDD 2026 with public GitHub repository.

## Limitations

- Baselines are run at default parameters with no per-dataset tuning, systematically disadvantaging them; the SOTA deltas on REAL-GOLD/REAL-SILVER would likely shrink under tuned baselines.
- The efficiency headline rests on only 5 human experts and a single standardized workflow, with no aggregate variance reported and a Step-1 reversal (humans faster) that complicates the "consistent improvement" framing.
- LLM-as-judge circularity: the Biological Evaluator (an LLM) validates the outputs of an LLM-driven pipeline, with no reported expert agreement on biological pass/fail decisions.
- The most compelling case study (Mouse SCI, high-salt vs. normal-salt) uses an unpublished, hospital-sourced dataset that cannot yet be independently verified, and the high-salt framing for SCI is unusual.
- No reporting of LLM API cost per analysis, failure modes on hallucinated tool APIs, or scaling behavior to very large datasets (>100k cells).
- All regulatory/mechanistic inferences are explicitly hypotheses requiring in vivo validation (acknowledged in Limitations and Ethical Considerations), so biological claims should not be over-interpreted.
- Tied to a specific LLM (DeepSeek-V3); while cross-model sensitivity is shown, absolute performance still depends on the underlying model's coding and biological-reasoning capability.

## Questions for Further Exploration

- How do the SOTA deltas on REAL-GOLD/REAL-SILVER change when each baseline (Monocle, PAGA, Slingshot, SpaceFlow, STORIES) is tuned per-dataset on the same Corr/F1/wCar/HIM metrics rather than run at defaults?
- What is the per-analysis LLM API cost, and how does performance degrade on datasets with >100k cells or with cheaper/smaller open-source LLMs (e.g., Llama-class models)?
- Can the LLM-based Biological Evaluator's pass/fail decisions be validated against the expert panel with inter-annotator agreement (e.g., Cohen's kappa), to address the LLM-as-judge circularity concern?
- Will the unpublished Mouse SCI dataset and the high-salt-vs-normal-salt SCI protocol be publicly released for independent verification of the differentiation-blockade case study?
- How does the cross-task self-evolution memory scale and avoid negative transfer when archived workflows from one tissue type are retrieved for a fundamentally different topology (e.g., cyclic vs. tree)?
- Does the 41.2% efficiency gain hold with a larger and more varied expert pool, and with confidence intervals, across workflows other than the single standardized 6-step one?
