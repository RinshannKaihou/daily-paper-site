> **一句话总结：** 本文在随机预言图灵机（SOTM）框架下，针对"缓存响应"与"新响应"两种预言机制，证明了由转录本分布的全变差距离所支配的识别上界、由转录本可得最高得分支配的输出质量上界，以及新响应下基于 Chernoff 信息与 Bernoulli KL 散度的查询次数界，刻画了 AI 增强计算中"响应复用、转录本信息、评分函数可得性"如何共同决定可计算性与 token 代价。

> **TL;DR:** This single-author theory paper extends the Stochastic-Oracle Turing Machine (SOTM) model of Wang (arXiv 2606.12647) by generalizing the oracle from query-dependent to context-dependent response distributions, then derives tight information-theoretic ceilings for what an adaptive SOTM can compute under two response schemes: under **cached responses**, the identification ceiling is governed by the total-variation distance between transcript distributions and the output-quality ceiling by the best expected score computable from the transcript; under **fresh responses**, binary identification error decays at the Chernoff information rate, and query-count bounds come from threshold-stopping (with score function) or majority-vote amplification via Bernoulli KL divergence (without it). There are no datasets, baselines, or empirical numbers — the paper reports purely theoretical bounds (no benchmark numbers exist to compare against), so the quantitative claims below are the closed-form bounds themselves.

## 核心贡献

- 将 SOTM 预言的响应分布从 Wang [4] 的"仅依赖查询 q"推广为"依赖输入 x 与已实现转录本 t_{i-1}"的上下文相关分布（Definition 1 / Remark 1），并以 Mixture-of-Experts 路由作为现实动机；王 [4] 的模型成为其特例。
- 引入"隐藏状态"抽象（Section 3），把预言每次激活的内部条件（如 MoE 的专家组合路由）建模为不可观测但决定激活响应分布 D^b_q 的隐变量，从而把"分布行为可预先刻画"与"本次激活状态需从转录本推断"区分开。
- 在**缓存响应**下给出识别上界的精确闭式（Theorem 1）：最优成功概率 = 1/2 + TV(π_1 PM^1, π_0 PM^0)，并给出等概率单信息查询情形的推论 θ̄ = (1 + d)/2（Corollary 2），其中 d = TV(D_1, D_0)。Corollary 1/2、Theorem 2 将结果推广到多隐藏态。
- 在**缓存响应**下给出输出质量上界（Theorem 3）：θ̄ = sup_M E_{x}[ E_{T}[ sup_{y∈Y_M(x,T)} S(x,y) ] ]，并用 Markov 不等式 + 层饼恒等式给出基于接受阈 η 的前沿刻画 F_φ(η)（Corollary 4）。
- 在**新响应**下证明二态单信息查询的识别误差以 Chernoff 信息 C(D_0, D_1) 的速率指数衰减（Theorem 4）。
- 在评分函数被纳入 SOTM 时给出阈值停时的查询次数期望 E[N(x)] = 1/p_φ(x,θ)（Theorem 5）及带预算截断的界 m_γ(x) = log(1−γ)/log(1−p_φ(x,θ))（Theorem 6）。
- 在评分函数不可得时引入"二元候选输出模型"，证明当 q(x) > 1/2 时经验多数投票所需固定查询数 k_θ(x) = ln(1/(1−θ)) / D(0.5‖q(x))·(1+o(1))（Theorem 7），并明确指出 q(x) ≤ 1/2 时新鲜响应无法把成功率推向 1（Remark 9）。

## Key Contributions

- Generalizes the SOTM oracle from Wang's earlier query-dependent family {D_q} to a context-dependent family {D^{x,t}_q} (Definition 1, Remark 1), with Mixture-of-Experts routing used as the concrete motivating example; the original Wang [4] model is recovered as a special case.
- Introduces a hidden-state abstraction (Section 3) that separates the oracle's characterizable distributional behavior (the D^b_q, estimable by pre-deployment probing) from the per-instance latent state b_i that must be inferred from the transcript.
- Establishes the tight binary identification ceiling under cached responses, θ̄ = 1/2 + TV(π_1 PM^1, π_0 PM^0) (Theorem 1), with the single-informative-query closed form θ̄ = (1 + d)/2 (Corollary 2, equal probabilities), and a max-over-states extension to finite hidden-state families (Theorem 2).
- Establishes the cached-response output-quality ceiling θ̄ = sup_M E_x E_T sup_{y∈Y_M(x,T)} S(x,y) (Theorem 3), and characterizes it via an acceptance-threshold frontier F_φ(η) with a Markov-style upper bound and a layer-cake integral representation (Corollary 4).
- Proves that fresh-response binary identification error decays exponentially at the Chernoff rate, −lim_{k→∞} (1/k) log P*_e(k) = C(D_0, D_1) (Theorem 4), turning the cached ceiling into an achievable target.
- Gives threshold-stopping query-count bounds when the score function is incorporated: E[N(x)] = 1/p_φ(x,θ) (Theorem 5) and a budget-cutoff variant with success probability 1 − (1−p)^m and least sufficient cutoff m_γ(x) = log(1−γ)/log(1−p_φ(x,θ)) (Theorem 6).
- Introduces the binary candidate-output model for the no-score-function regime and derives k_θ(x) ≈ ln(1/(1−θ))/D(0.5‖q(x)) for empirical-majority success (Theorem 7), while explicitly flagging the q(x) ≤ 1/2 failure mode (Remark 9).

## Methodology

The paper is purely theoretical and proof-based. It builds on the SOTM formalism and token-complexity definition of Wang [4] (arXiv 2606.12647) and the reliability-certification framework of Wang [5] (arXiv 2606.24074).

- **Modeling layer.** An SOTM is a pair M = (M, O): a probabilistic TM M interacting turn-by-turn with a stochastic oracle O = {D^{x,t}_q}. Each turn has an inter-turn computation phase (M reads x and prior transcript t_{i-1}, then either emits the next query q_i or halts with output y) and an oracle exchange phase (O draws r_i ~ D^{x,t_{i-1}}_{q_i}). Two oracle schemes are distinguished: a **cached-response** oracle returns the same draw on every later call to the same query, while a **fresh-response** oracle draws independently on every call (Definition 4). A task is T = (X, Y, S, D_X) with score S: X×Y → [0,1]; the token complexity κ_T(θ; O, α, β) is the infimum expected weighted token cost over all SOTMs achieving quality θ.
- **Hidden-state abstraction.** A hidden state b (e.g., an MoE expert-combination chosen by the gating network) determines the active response distribution D^b_q; the SOTM observes only the response r ~ D^{b_i}_{q_i}, not b_i. The engineer can estimate the D^b_q offline but cannot observe the active state per instance — this per-instance uncertainty is what the ceilings quantify.
- **Proof techniques.** Cached-response identification is reduced to a one-sample (or product-sample) binary hypothesis test and solved exactly via total variation; the optimal decision rule is a likelihood-ratio test on the transcript (Theorem 1). Output quality uses a sup-over-outputs expectation and the layer-cake identity E[Z] = ∫_0^1 Pr[Z ≥ η] dη combined with Markov's inequality (Corollary 4). Fresh-response identification applies Cover–Thomas's Chernoff-information large-deviation result for i.i.d. testing. Threshold stopping is a geometric waiting-time argument; the no-score-function case is a binomial-tail Chernoff bound with Bernoulli KL divergence D(a‖q).
- **Scope.** All results assume (i) the SOTM halts with probability one and produces a finite transcript, (ii) the engineer has characterized the response distributions D^b_q offline, and (iii) the single-informative-query special case is isolated for closed forms. The general multi-informative-query cached case is left as a product-distribution reduction of Theorem 1 (Remark 5).

## Results & Highlights

No quantitative results reported — this is a pure theory paper with no datasets, benchmarks, training runs, or empirical baselines. The "results" are closed-form bounds; the strongest of these, stated with their exact quantities, are:

- **Cached binary identification ceiling (Theorem 1).** Supremum success probability θ̄ = 1/2 + TV(π_1 PM^1, π_0 PM^0); under equal priors and a single informative query with d = TV(D_1, D_0), this collapses to the exact closed form θ̄ = (1 + d)/2 (Corollary 2). The TV term is the irreducible transcript overlap no adaptive strategy can remove.
- **Fresh-response identification rate (Theorem 4).** For k fresh calls to the single informative query with equal priors, the optimal error is P*_e(k) = ½(1 − TV((D_1)^⊗k, (D_0)^⊗k)) and decays as −lim_{k→∞} (1/k) log P*_e(k) = C(D_0, D_1) > 0 whenever D_0 ≠ D_1; hence the success probability tends to 1.
- **Threshold-stopping query count (Theorems 5–6).** With the score function incorporated and one-response acceptable-output probability p_φ(x,θ) > 0, the expected query count is exactly E_{x~D_X}[N(x)] = E_x[1/p_φ(x,θ)] (geometric waiting). With a budget cutoff at m calls, success probability = 1 − (1 − p)^m and the least m guaranteeing target γ is m_γ(x) = log(1 − γ) / log(1 − p_φ(x,θ)).
- **No-score-function majority bound (Theorem 7).** In the binary candidate-output model with q(x) > 1/2, the least fixed query count for empirical-majority success ≥ θ is k_θ(x) = ln(1/(1 − θ)) / D(0.5 ‖ q(x)) · (1 + o(1)) as θ → 1, where D(·‖·) is Bernoulli KL divergence. The paper explicitly notes this fails (cannot drive success to 1) when q(x) ≤ 1/2 (Remark 9).

## Critical Assessment

- **Novelty.** The conceptual move from query-dependent to context-dependent response distributions is a genuine and well-motivated generalization (Remark 1) directly tied to MoE routing — this is the clearest new modeling contribution. The cached-response ceilings are largely careful reductions of well-known binary-hypothesis-testing identities (TV distance → optimal success; layer-cake → expected score) to the SOTM transcript setting, and the fresh-response identification result is essentially Cover–Thomas's Chernoff bound re-stated for SOTM transcripts. So novelty is real but incremental: it is a consolidation/specialization of classical information-theoretic results inside the author's own SOTM framework rather than new probability theory.
- **Rigor.** Proofs are correct and tight by construction (the paper repeatedly stresses that "ceiling" means a tight supremum, not a loose upper bound). The distinction between cached (single-realization) and fresh (product-distribution) regimes is handled cleanly, and the reduction "any SOTM that reissues a query can be replaced by one that stores and reuses" correctly disposes of a tempting fallacy.
- **Reproducibility.** Theorems and proofs are fully self-contained and could be re-derived by a reader; there is nothing to "run." However, there is no worked numerical example tying the bounds to any concrete oracle (e.g., a calibrated LLM error rate), so the practical magnitude of the ceilings and query counts is never illustrated.
- **Significance.** For the LLM-systems engineering audience the framework is conceptually useful: it gives a principled answer to "why does repeating the same prompt not help with caching" (Corollary 2) and "how many re-samples does self-consistency need" (Theorem 7). But the single-informative-query assumption is strong, and the multi-informative-query cached case is only sketched via a product-distribution reduction (Remark 5) without a token-cost-vs-adaptivity tradeoff.
- **Red flags.** (i) The empirical grounding is zero — no LLM, no benchmark, no simulation; the motivating MoE example (Section 3) is described qualitatively but the gating distribution π_b is never instantiated. (ii) Theorem 7's majority-vote bound is exactly the classical self-consistency story and inherits its known limitation that it silently assumes q(x) > 1/2, which the paper does flag (Remark 9) but does not quantify how often that assumption fails for real LLMs. (iii) The score-function-incorporated bounds (Theorems 5–6) assume S is computed without error, an assumption the author himself acknowledges as unrealistic for learned verifiers in the open-problems section. (iv) Three of the five references are the author's own (including the two foundational ones), which is defensible for a sequel paper but means the framework's empirical validation rests entirely on work not shown here.

## Strengths

- Clean, correct, and self-contained proofs; every theorem is verified in-text and the notation is explicitly reconciled with the predecessor paper.
- The context-dependent distribution generalization (Remark 1) is the right abstraction for modern MoE-style LLMs and is well-motivated.
- The cached-vs-fresh dichotomy maps directly onto two real engineering choices (response caching / temperature-0 reuse vs. independent re-sampling), making the bounds actionable for system design.
- Honest about limitations: the open-problems section explicitly calls out noisy score functions, the multi-informative-query case, the non-binary no-score-function case, and oracle drift.

## Limitations

- No empirical validation whatsoever — no LLM, no benchmark, no numerical illustration of how large the ceilings or query counts actually are.
- The strongest closed forms (Corollary 2, Theorems 4, 7) all require the single-informative-query simplification; the general multi-query cached case is only a product-distribution reduction without a token-cost tradeoff.
- Theorem 7 inherits the self-consistency caveat that q(x) > 1/2 is essential and is silent on the common regime where the acceptable output is not the majority response.
- Threshold-stopping bounds (Theorems 5–6) assume a noiseless score function, which the author concedes is unrealistic for learned verifiers / automated tests.
- The hidden-state abstraction assumes the engineer has already characterized the D^b_q offline; distributional uncertainty (the harder problem) is deferred to [5].

## Relevance

对研究"大模型作为可调用预言/工具"的理论边界与 token 成本复杂度的方向高度相关：它把 LLM 缓存复用、重复采样、self-consistency 等工程实践统一纳入 SOTM 框架，并给出由全变差距离、Chernoff 信息与 Bernoulli KL 散度刻画的紧上界与查询次数界，可作为分析 Agent 多轮调用与验证器辅助生成成本的理论基准。

## Questions for Further Exploration

- How do Theorems 5–6 (threshold stopping) change when the incorporated score function is a noisy verifier with non-trivial false-positive/false-negative rates — what is the effective acceptable-output probability that replaces p_φ(x,θ)?
- Can the multi-informative-query cached case (Remark 5) be given a closed-form token-cost vs. adaptive-query-selection tradeoff, rather than only the product-distribution reduction of Theorem 1?
- What is the right replacement for the Bernoulli KL D(0.5‖q) in Theorem 7 when there are many incorrect responses, semantic equivalence classes, or graded partial scores — an information-radius or multi-hypothesis large-deviation exponent?
- How do prompt changes, model updates, retrieval augmentation, and correlated errors across inputs (i.e., oracle drift) quantitatively move the ceilings derived here?
- Can the hidden-state abstraction be instantiated on a real MoE LLM (e.g., estimating the gating distribution π_b and the per-expert-combination response distributions D^b_q) so that the cached-response ceiling (1+d)/2 and the fresh-response Chernoff rate can be measured rather than only stated?
