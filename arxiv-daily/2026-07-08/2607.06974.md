> **一句话总结：** MILES 提出一种测试时自改进框架，通过维护(sub-goal, sub-instruction)的模块化记忆单元并训练轻量级选择头，从置信样本中累积经验、对不确定样本进行记忆引导的树搜索，在六个推理基准上匹配或超越 BoT/DC/SC 等基线，并在精度-效率权衡上占优。

> **TL;DR:** MILES is a test-time self-improvement framework that keeps a frozen LLM but maintains an incrementally-growing modular memory of (sub-goal embedding, sub-instruction) pairs with lightweight per-item selection heads learned from confident samples; on six reasoning benchmarks (MATH-500, AIME 2024, AIME 2025, GPQA-Diamond, MMLU-Pro Physics/Engineering) across four backbones it matches or beats Self-Consistency, Buffer-of-Thoughts, and Dynamic CheatSheet, e.g., lifting GPT-4.1-mini on AIME 2024 from 56.67% (SC) to 66.67%, while dominating the accuracy/token frontier over ToT/rStar/DORA.

## 核心贡献

- 提出 MILES 框架：在测试时无需参数更新、外部监督、策划训练数据或固定动作空间，即可实现自改进的逐步记忆组合；从置信轨迹增量构建模块化记忆，并应用到不确定样本上。
- 设计模块化非对称记忆结构：(sub-goal embedding, sub-instruction) 对，每个单元关联一个轻量级可学习选择头，天然支持粗到细(coarse-to-fine)的选择机制，同时实现增量记忆扩张和可学习记忆组合。
- 提出可学习的记忆选择方法：从置信样本的 rollout 结果中训练轻量级 per-item 选择头(基于 KL 正则化的 closed-form 最优策略推导，逼近为 one-vs-rest BCE 分类器)，在有限测试数据下实现自监督且针对最终答案正确性优化的记忆选择。
- 广泛的实验验证：在 6 个推理基准、4 个 backbone 上稳定匹配或超越基线，并提供精度-效率权衡、跨模型迁移、离线记忆构建、鲁棒性等多项分析。

## Key Contributions

- MILES framework for self-improving step-wise memory composition at test time with no parameter updates, external supervision, curated training data, or fixed action spaces; confident samples expand memory, learned memory guides uncertain samples.
- A modular asymmetric memory structure of (sub-goal embedding, sub-instruction) pairs with per-item learnable selection heads, naturally enabling coarse-to-fine selection that supports both incremental memory growth and learned composition.
- Learnable memory composition derived from a KL-regularized closed-form optimal selection policy (Eq. 5), approximated as tractable per-unit one-vs-rest binary classifiers trained via BCE on MCTS rollout outcomes.
- Empirical validation across six benchmarks and four backbones, plus analyses of efficiency frontier, cross-model transfer, offline memory construction, component ablations, and order robustness.

## Methodology

MILES augments a frozen LLM with an external step-wise memory built from the model's own reasoning experience, operating in two modes that handle confident vs. uncertain test samples differently.

**Memory unit (Section 3.3).** Each memory unit is an asymmetric pair (g_i, u_i): g_i is a d-dimensional sub-goal embedding summarizing the objective of the next reasoning step (used as the retrieval key); u_i is a natural-language sub-instruction used to condition generation. Units are constructed by clustering (reasoning step, step-goal) tuples from confident trajectories with a joint k-means distance `d(s1,o1;s2,o2)=1-cos(s1,s2)+α_sum[1-cos(o1,o2)]`, then having the LLM summarize a generalized sub-instruction within each cluster; the mean step-goal embedding becomes g_i. The memory M grows over time and each unit is associated with a learnable selection head.

**Two-layer coarse-to-fine selection (Section 3.4).**
- *Layer 1 (coarse):* The LLM proposes candidate step-goals for the next step; their embeddings are matched against stored g_i via cosine similarity with a per-unit effective threshold R_effect = α_eff · d_i. Unmatched candidates become seeds for incremental memory expansion.
- *Layer 2 (fine):* A learned per-item selection head π_θa(a|x) scores applicability. State x is represented via max-similarity pooling against a context codebook of prototypes {qa1...qan} built from prior applications of unit a, giving a location-invariant fingerprint m_a(x).
- *Combined score:* `score(a,x) = (1−β_c2f)·cos(ĝ,g_a) + β_c2f·π_θa(a|x)`.

**Learning the selection heads (Section 3.5).** Selection is formalized as maximizing `Σ_a π(a|x)·V(x,a)` where V(x,a) is expected final-answer correctness from continuing rollout under unit a. Adding a KL penalty to a Layer-1 sampling distribution π_sim yields the closed-form optimum `π*(a|x) ∝ π_sim(a|x)·exp(V(x,a)/β)`. Since this is impractical (no tractable density, noisy value estimates), the authors approximate it with one-vs-rest binary classifiers trained via BCE: positives for unit a are states where applying a led to a correct final answer; negatives are states where a led to an incorrect answer or where a different unit led to a correct answer. This per-item design supports incremental memory expansion without retraining existing heads.

**Test-time pipeline (Section 3.6).** For each test question, K independent responses are generated. *Confident samples* (consistent majority answer) use the pseudo-label as supervision, run MCTS over the reasoning tree using Layer-1 similarity sampling (π_sim) to collect (state, outcome) tuples, summarize new memory units, and train selection heads. *Uncertain samples* are routed to a multi-stage memory-guided tree search where candidate step-goals are generated and units are retrieved+reranked by the combined score (Eq. 3); the final answer is majority-voted across all generated responses. Implementation: selection heads are ensembles of 10 independently trained 3-layer MLPs with 16 hidden units; embeddings from text-embedding-3-small (1536-dim).

## Results & Highlights

- **Across six benchmarks × four backbones (Table 1), MILES matches or exceeds the best baseline on every row.** On GPT-4.1-mini: MATH-500 92.60% vs. SC 89.40% (+3.20); AIME 2024 66.67% vs. SC 56.67% (+10.00); AIME 2025 60.00% vs. DC 60.00% (tied, +6.67 over SC 53.33%); GPQA-Diamond 73.23% vs. SC 71.72% (+1.51); MMLU-Pro Physics 85.00% vs. SC 83.50% (+1.50); MMLU-Pro Engineering 75.00% vs. SC 73.50% (+1.50).
- **Large gains on smaller open-weight backbones.** On AIME 2025 with GPT-OSS-20B, MILES reaches 93.33% vs. BoT 33.33% and DC 36.67% (+60.00 over BoT); on Qwen3-30B-Instruct MMLU-Pro Engineering, 81.00% vs. DC 52.00% (+29.00). BoT and DC actually underperform zero-shot CoT on these backbones due to overly complex prompts.
- **Accuracy/token frontier (Figure 2).** On AIME 2024 and AIME 2025 with GPT-4.1-mini, MILES dominates Self-Consistency, Tree-of-Thoughts, rStar, and DORA across all token budgets (2M–6M tokens) and continues improving as budget grows, while the memoryless baselines flatten at higher budgets.
- **Two-layer selection ablation (Table 4).** On AIME 2024 with GPT-4.1-mini: both layers disabled = 60.00%; Layer-1 only = 63.33%; Layer-2 only = 63.33%; both layers = 66.67% (best). LLM-prompt reranking (Exp 6) fails to improve over the no-reranker baseline (60.00%), while learned heads capture signal beyond state-similarity reranking (63.33%).
- **MCTS rollout budget (Table 5).** More rollouts produce richer memory and higher accuracy: on AIME 2025, 20 rollouts → 10 units → 53.33%; 30 rollouts → 19 units → 56.67%; 40 rollouts → 25 units → 60.00%.
- **Cross-model transfer (Table 2).** On AIME 2024 with GPT-4.1-mini reasoning backbone, memory built from Qwen3-30B-Instruct auxiliary trajectories raises accuracy to 66.67% vs. 60.00% no-auxiliary (+6.67); on AIME 2025, Qwen3-30B-Instruct auxiliary raises 50.00% → 60.00% (+10.00).
- **Offline memory on most-uncertain samples (Table 3).** On the 13 most-uncertain AIME 2025 samples, pre-built memory from GPT-4.1-mini raises accuracy from 23.08% (no memory) to 38.46% (+15.38); on 31 most-uncertain GPQA-Diamond samples, Qwen3-30B-Instruct pre-built memory raises 48.39% → 58.06% (+9.67).
- **Selected sub-instruction effect (Table 14).** Under matched reasoning states, applying a selected sub-instruction improves accuracy across all 12 dataset×model cells, with the largest gains on harder math (AIME 2024: +10.80 for GPT-4.1-mini, +3.77 for GPT-4.1; range 0.63–10.80 points).

## Authors & Credibility

Authors are Ruilin Tong and Dong Gong (corresponding author) from the University of New South Wales (UNSW Sydney). The work is a single-institution contribution without an established large lab brand behind it, but the empirical methodology is thorough: six standard reasoning benchmarks, four backbones spanning proprietary (GPT-4.1, GPT-4.1-mini) and open-weight (Qwen3-30B-Instruct, GPT-OSS-20B) families, an extensive ablation suite (Tables 4–14, Appendix F), hyperparameter sensitivity studies (α_sum, cluster size, α_eff, candidate number, codebook components), and explicit MCTS/tree-search hyperparameters (Tables 15–16) and decoding details (Appendix I). The closed-form derivation (Appendix A) connects the learned classifier to a principled KL-regularized policy objective. A project page is referenced.

## Critical Assessment

- **Novelty** is genuine and well-motivated: the asymmetric (embedding, text) modular memory plus correctness-optimized per-item selection heads is a clear advance over heuristic similarity/prompt-based selection in ArcMemo, Retrieval-of-Thought, BoT, and DC. The derivation of a KL-regularized closed-form optimum (Eq. 5) and its reduction to per-unit BCE classifiers is a sensible, tractable compromise under strict test-time constraints (no labels, growing action space, frozen model).
- **Rigor concerns.** The authors explicitly omit error bars ("Since majority-voting results are stable, we omit error bars"), and several headline deltas are on very small evaluation sets — AIME 2024/2025 each have only 30 problems, so a single problem swing is 3.33 points. Many of the "MILES wins" rows (e.g., AIME 2024 GPT-4.1 53.33% vs. DC 53.33%, or MMLU-Pro Engineering Qwen3 81.00% vs. SC 78.00%) are within plausible sampling noise on these sizes; without variance estimates the claimed consistency is not statistically demonstrable. The GPT-OSS-20B AIME 2024 result of 93.33% is near-saturated and tied with SC, so it is more "no degradation" than an improvement.
- **Baseline fairness.** The comparison includes a reasonable baseline set (ZS-CoT, SC, BoT, DC, plus ToT/rStar/DORA in Appendix F), but the authors themselves note that BoT and DC "fail to outperform zero-shot CoT due to overly complex prompt" on smaller open-weight models — meaning MILES's huge +60 point deltas on GPT-OSS-20B AIME 2025 are largely beating broken baselines rather than showing absolute method strength. DC's strong AIME performance is also acknowledged to come mostly from its use of coding, an asymmetric capability advantage that complicates like-for-like comparison. The paper does not compare against strong recent RL-from-feedback memory methods (ICPI, Memento, DynaAct) on the main table, only arguing incompatibility.
- **Missing ablations.** There is no clean isolation of the contribution of *asymmetric* (embedding vs. text) memory vs. symmetric text-only memory (the closest is Exp 1 vs. Exp 4 collapsing both changes together), and no direct comparison of MCTS-based data collection against simpler rollout strategies for the same compute. The cost of MCTS data collection (40 rollouts × depth 5 × 3 children on every confident sample) is acknowledged as overhead but never quantified head-to-head against SC at matched wall-clock/token cost in the main tables.
- **Reproducibility** is moderate-to-good: hyperparameters are listed, models and checkpoints named, vLLM/LiteLLM stack disclosed, open-source model URLs provided. However, no code release is confirmed in the text (only a project page link), and the system has many moving parts (k-means clustering, context codebooks, ensemble of 10 MLPs per unit, consistency-threshold routing tuned per-model) that would be hard to reproduce exactly without the reference implementation.
- **Significance** is solid for the test-time-scaling and LLM-memory subfields: the paradigm of learning selection from confident samples and transferring to uncertain ones is general and complementary to prompting/search. Overclaiming risk is moderate — the abstract's "consistently matches or outperforms" is fair on the raw numbers but undersells the noise caveat on small benchmarks.

## Strengths

- Principled formulation: selection is tied to final-answer correctness via a derived KL-regularized optimum, not heuristic similarity.
- Genuinely test-time-friendly: no parameter updates, no external labels, no fixed action space, incremental memory growth — fits the stated setting precisely.
- Comprehensive ablations and sensitivity studies (Appendix F covers α_sum, cluster size, α_eff, candidate number, context codebook components), showing stability across moderate hyperparameter ranges.
- Cross-model and offline-memory transfer experiments (Tables 2–3) demonstrate that the textual memory representation is portable, including from weaker auxiliary models to a stronger frozen backbone.
- Per-problem-scaling efficiency frontier (Figure 2) is the most compelling single result: MILES keeps improving with compute where memoryless methods plateau.

## Limitations

- Reliance on the instruction-following capability of the underlying LLM, and substantial computational overhead from rollout-based data collection (mitigated but not eliminated by parallel collection and KV-cache reuse).
- Small evaluation sets on AIME 2024/2025 (30 problems each) make point differences hard to distinguish from noise; error bars are omitted.
- Large deltas on smaller open-weight backbones are partly inflated by broken baselines (BoT/DC underperforming ZS-CoT).
- Many interacting components and per-dataset/per-model hyperparameter choices (β_c2f = 0.0 for math vs. 0.4–0.6 for academic; consistency thresholds tuned on validation subsets) complicate reproduction.
- No comparison against RL-from-feedback memory methods (ICPI, Memento, DynaAct) on the main table, and no confirmed public code release.
- Memory may inherit and amplify biases from collected trajectories; only high-level mitigation (transparent inspection, LLM verification) is proposed.

## Relevance

这篇论文与你关注的 LLM 推理、测试时计算扩展(test-time scaling)和记忆增强推理方向高度相关：MILES 提供了一种不修改模型参数、通过增量构建模块化记忆并学习选择策略来实现持续自改进的新范式，对研究生级推理基准(GPQA-Diamond、AIME、MMLU-Pro)的实证结果与 test-time compute 讨论直接相关，可作为与 Self-Consistency、Tree Search、BoT/DC 类方法对比的重要参考。

## Questions for Further Exploration

- How does MILES perform when evaluation sets are larger and error bars can be computed — do the small-benchmark wins (e.g., +10 on AIME 2024) survive statistically?
- What is the wall-clock and total-token cost of MILES's confident-sample MCTS data collection compared to simply spending the same compute on Self-Consistency?
- Can the asymmetric (embedding, text) memory contribution be isolated from the two-layer selection contribution via a symmetric text-only memory ablation?
- How sensitive is the method to the choice of embedding model (text-embedding-3-small) — does swapping in an open-weight embedder change cross-model transfer results?
- Does memory quality degrade or accumulate noise over very long test streams (hundreds/thousands of questions), and does the bias-amplification risk flagged in the Broader Impact section become measurable?
- Could the learned selection heads be combined with process reward models (which the authors avoid) to push the accuracy/token frontier further on near-saturated benchmarks?
