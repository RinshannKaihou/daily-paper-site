> **一句话总结：** SkillCenter 构建了迄今最大规模的开源 Agent 技能库（216,938 个结构化、可溯源技能，覆盖 24 个领域），并通过控制实验证明：只有当任务超出模型自身知识、且检索能命中正确技能时，技能注入才真正有效，而仅靠标题关键词检索在实际任务中几乎等同于注入无关文本。

> **TL;DR:** SkillCenter releases 216,938 source-grounded skills across 24 domain bundles (114,565 from an automated SkillGate pipeline over 55,199 sources, plus 102,373 harvested community skills), distributed as offline SQLite FTS5 bundles. A controlled A/B across four LLM solvers shows keyword-retrieved skill injection is statistically indistinguishable from a length-matched placebo on the ~2,000-task general pool (e.g., claude-haiku-4.5: 83% vs 87% baseline, -77 discordant, p<0.001), while an oracle skill lifts pass rates from 0% to 61-100% on knowledge-gap probes, isolating retrieval recall — not injection or coverage — as the binding constraint.

## 核心贡献

- **最大规模开源技能库**：发布 216,938 个结构化技能，分布在 24 个领域 bundle 中，规模比既有公开技能集合大一个数量级。其中 114,565 个由 SkillGate 流水线从 55,199 个来源文档（PLOS、Nature 系列、eLife、ArXiv、GitHub、Web、Stack Overflow）自动生成，另有 102,373 个来自 GitHub SkillMD 与 ClawHub 社区市场。
- **端到端自动化流水线**：给出多源采集 → LLM 质量门（SkillGate）→ 模板化生成 → 迭代式来源溯源校验 → 四准则发布门（质量分≥3.0、许可证白名单、抄袭率<0.35、占位符<15）的完整框架，可持续运行、无需人工干预。
- **来源溯源保证**：每个保留声明都通过确定性子串匹配（≤20 词）映射到原文中的精确引文，提供可追溯性（traceability），所有技能携带来源 URL、SPDX 许可证与质量分审计轨迹。
- **离线可搜索分发系统**：以按领域拆分的 SQLite FTS5 bundle 形式发布，零外部依赖、零网络访问，配合项目类型自动检测与 `bundle-install --auto`，对终端用户完全透明。
- **诚实的下游评估**：用 4 个 LLM solver 在 ~2,000 任务上做配对 McNemar 检验，明确划定了"技能何时有用"的三个充要条件，并指出标题级 FTS 检索是实际瓶颈。

## Key Contributions

- **Largest open skill library to date**: 216,938 structured skills across 24 domain bundles, exceeding prior open collections by more than an order of magnitude; 114,565 produced by the SkillGate pipeline from 55,199 distinct source documents, and 102,373 integrated from GitHub SkillMD (90,984) and the ClawHub marketplace (11,389).
- **End-to-end automated framework**: a continuously running five-stage pipeline (multi-source acquisition, SkillGate pre-filter, template-driven generation, iterative source-grounding, four-criterion publish gate) that ingests new sources and publishes new skills without manual intervention.
- **Source-grounding as traceability**: each retained claim must map via deterministic substring matching (≤20-word quotes) to an exact passage in the original source; every skill carries a source URL, SPDX license, and a GPT-5.2-assigned 1-to-5 quality score.
- **Offline-searchable distribution**: domain-split SQLite FTS5 bundles with zero external dependencies, project-type auto-detection, and `bundle-install --auto` integration; pre-built bundles on Hugging Face, framework open-source on GitHub.
- **Honest controlled evaluation**: paired exact-McNemar A/B across four solvers (gemini-3.5-flash, claude-haiku-4.5, gpt-5-mini, claude-sonnet-4.6) on ~2,000 tasks, plus an agent-gap probe and a real-corpus probe that together isolate *retrieval recall* — not injection or coverage — as the binding constraint.

## Methodology

**Corpus construction.** The pipeline subset (114,565 skills) is built from five source types — journal APIs (PLOS, eLife), ArXiv, GitHub REST API, curated web pages, and Stack Overflow — unified by a 12,000-character truncation and URL canonicalization step. Sources flow through five stages:

1. **Source Acquisition** (13 domain configurations with host allowlists, GitHub `min_stars` filters, forum tags; an LLM classifier routes ambiguous topics). A case study traces "kubernetes networking" being assigned to the `cloud` domain and triggering parallel fetches from kubernetes.io, GitHub (min_stars=100), and SO tags.
2. **SkillGate** — an LLM (GPT-5.2) pre-filter that takes a 4,000-char excerpt and returns a `pass`/`maybe`/`fail` verdict plus a 0-10 suitability score; sources under 200 chars fail without an LLM call. "Maybe" defaults to pass.
3. **Template-driven generation** — four templates: `default (tech)` for GitHub/web/forum (Background, Use Cases, Inputs, Outputs, ≤12 Steps, Verification, Evidence); `paper_writing` and `paper_writeup` for ArXiv; `experiment_design` for journal papers (which also yield `paper.idea_intro`, `paper.experiment`, `paper.method`, `paper.picture`). Temperature 0.2 (0.0 on retry), JSON output mode.
4. **Iterative improvement** — up to 3 passes optimizing `score = lint_count × 100 + missing_count`; each pass does a deterministic source-grounding check that each claimed improvement maps to a ≤20-word substring in the source. Most skills converge to score 0 within 2 passes.
5. **Publish gate** — four independent criteria: score ≥ 3.0; license whitelist (MIT/Apache-2.0/BSD/ISC/MPL-2.0/Unlicense for repos, CC-BY-SA for forums; GPL/AGPL/LGPL denied); plagiarism ratio < 0.35; "not provided" count < 15.

**Distribution.** Each domain bundle is a self-contained SQLite database with a four-table schema (`skills_index`, `skills_content`, `skills_fts`, `bundle_meta`); FTS5 currently indexes only title and domain (full-body indexing is planned). Retrieval uses BM25, is deterministic and auditable, and strips source URLs from injected prompts to prevent hallucinated links.

**Downstream evaluation.** Three probes: (a) a general pool of ~2,000 offline-graded algorithmic/data tasks across 51 domains with four arms (Baseline, +Keyword, +Placebo length-matched irrelevant, +Oracle); (b) an agent-gap probe of 260 synthetic convention tasks whose specification lives only in an injected skill; (c) a real-corpus probe of 18 tasks whose answers are specific parameters documented only in a real SkillCenter research skill. Significance uses exact paired McNemar on discordant outcomes.

## Results & Highlights

- **Library scale and composition**: 216,938 published skills in 24 bundles — Research 90,084 (41.5%), Technical 24,481 (11.3%), Community 102,373 (47.2%); the pipeline subset of 114,565 skills was built from 55,199 distinct sources at an average of 2.1 skills/source (journals 3.0, ArXiv 2.1, GitHub/web/forum 1.0).
- **General pool — keyword retrieval is no better than placebo**: across all four solvers, +Keyword was statistically indistinguishable from +Placebo and never beat baseline. For claude-haiku-4.5 on n=1,979 tasks, +Keyword scored 83% vs 87% baseline (Δ = -77 discordant tasks, exact McNemar p < 0.001); gpt-5-mini showed a directional -9 (p=.08); claude-sonnet-4.6 was flat (-4, p=.39); gemini-3.5-flash was at ceiling (96%, +1, p=1.0).
- **Agent-gap probe — oracle is decisive when knowledge is absent**: on 260 synthetic-convention tasks, baseline/placebo/+Keyword all scored 0% for every solver, while +Oracle scored 72-100% (gpt-5-mini 100%, claude-sonnet-4.6 99%, gemini-3.5-flash 98%, claude-haiku-4.5 72%; p from 10⁻¹³ to 10⁻⁶³).
- **Real-corpus probe — real skills close real gaps**: on 18 tasks grounded in actual SkillCenter research skills, baseline/placebo/+Keyword all scored 0%, while injecting the verbatim real skill scored 61-78% per solver (pooled 69%, Δ = +50 over n=72, exact McNemar p ≈ 2×10⁻¹⁵) — and the keyword arm still scored 0% even though the needed skill was present in the searched bundles, proving retrieval recall (not coverage) is the bottleneck.
- **Quality scoring is inflated**: average score 3.91, but 82.1% of pipeline skills receive exactly 4; the publish gate (score ≥ 3.0) removes ~3% of generated skills; 302 skills (0.26%) below threshold leaked in from early runs.
- **Redundancy is localized to community bundles**: 0% duplicate IDs, 4.53% verbatim, 11.52% near-duplicate (MinHash Jaccard ≥ 0.8); all 21 SkillGate pipeline domains sit at ≤0.01%, while GitHub SkillMD (code search) is 26.9% near-duplicate and its curated-lite subset 65.0%; only ~3% of near-duplicate pairs span two bundles.
- **Safety scan (upper bound)**: hardcoded secrets 12 (pipeline) vs 193 (community); prompt-injection phrases 318 vs 2,342; destructive shell patterns 218 vs 1,359 — all under 3% of skills in any class, concentrated in community bundles.

## Critical Assessment

- **Novelty**: Genuine contribution is engineering plus a refreshingly honest negative result. The skill-library concept is not new (Voyager, ClawHub, Self-Instruct all exist), but the combination of offline automated extraction at this scale with per-claim source-grounding and a license-aware publish gate is a real advance over hand-authored or online-grown libraries. The honest finding that keyword retrieval ≈ placebo is the intellectual core.
- **Rigor**: The downstream A/B is well-designed — paired exact McNemar, a length-matched placebo arm to separate content effect from context-token effect, four solvers spanning weak-to-frontier, and two complementary probes (synthetic conventions + real corpus). However, all solvers are **single-shot, prompt-conditioned LLMs, not tool-using multi-turn agentic harnesses**, so the study measures one-shot skill conditioning, not realistic agent use — a scope the authors acknowledge but which materially limits the headline claims about "agents."
- **Reproducibility**: Strong on artifacts (frozen manifests, raw captures, configs, scripts, public bundles) but fundamentally bounded by dependence on a **proprietary scorer (GPT-5.2)** for acquisition, SkillGate, generation, and scoring; re-running with a different model yields different skills and non-portable 1-to-5 scores. The 1-to-5 scores are explicitly an internal QA signal, not externally validated.
- **Significance**: The corpus is a useful artifact, but the evaluation tempers its own significance: on tasks models already handle, skills are redundant or mildly harmful, and the real-corpus probe uses only **18 narrow parameter-recall tasks** — a lower bound, not evidence of broad downstream utility.
- **Red flags**:
  - **Score-4 collapse (82.1%)** means the headline "average 3.91" carries almost no discriminative information; per-domain and per-source score orderings are explicitly not meaningful, yet are still tabulated and discussed at length.
  - **Headline count is heterogeneous**: 216,938 aggregates 102,373 unscreened community skills (GitHub SkillMD 26.9% near-duplicate, ClawHub 5.4%) that bypass SkillGate, source-grounding, and the publish gate; the uniformly processed subset is 114,565. The "largest open skill library" claim leans on the lower-quality half.
  - **Title-only FTS5 retrieval** is the admitted binding constraint and the direct cause of the 0% keyword arm on the real-corpus probe — the released retrieval system is, by the authors' own evaluation, not yet fit for the deployment path it advertises.
  - **Web-derived skills (28.5% of pipeline subset, 32,655 skills) have no license whitelist** — admitted as a quality filter, not legal clearance — which is a real redistribution risk for a corpus explicitly meant to be shipped inside agent projects.
  - **Lexical dedup only**: semantic paraphrases go undetected; the authors tried 7 sentence encoders (22M-8B) and got 49-93% near-duplicate flags at cosine 0.9, a discrepancy they could not resolve, so true redundancy is unknown.

## Strengths

- Unusually honest negative result: the paper does not oversell keyword retrieval and explicitly shows it is indistinguishable from placebo, even significantly harmful for mid-tier models.
- Clean experimental design with a placebo arm and exact paired McNemar testing — the +Placebo vs +Keyword vs +Oracle contrast cleanly separates content effect from context-token effect.
- Strong artifact release: frozen source manifests, raw captures, reproducible dedup and evaluation scripts, public bundles, open-source framework.
- Source-grounding via deterministic substring match is a concrete, auditable safeguard against hallucinated claims, distinct from fuzzy RAG.
- The redundancy, licensing, and safety audits are localized and actionable (community bundles are the problem, pipeline subset is clean), giving operators a clear opt-in path.
- The framework's modularity (a new domain needs only a config file) and additive SQLite bundles are sensible engineering for sustained growth.

## Limitations

- **Single-LLM scorer with 82.1% score-4 collapse**: no human evaluation, no inter-annotator agreement, no comparison against baseline collections; all quality figures are internal QA signals.
- **Single-shot solver evaluation**: not multi-turn, not tool-using agents; the general pool skews toward algorithmic tasks where strong models are near ceiling, exactly where skills are least needed.
- **Title-only FTS5 retrieval is the binding constraint** and is not yet fixed; full-body and semantic retrieval are planned but absent from the release that is evaluated.
- **Proprietary scorer dependency** breaks exact reproducibility of scores and skills.
- **Lexical-only deduplication**: semantic paraphrases undetected; embedding-based dedup was attempted and found unreliable across encoders.
- **Web licensing not cleared**: 28.5% of the pipeline subset has no SPDX tag and is admitted on a plagiarism-ratio basis only.
- **No safety screening in the pipeline**: quality filtering only; prompt-injection, credential-exposure, and destructive-command checks are run post-hoc as a scan, not as a gate, and are concentrated in the community bundles that are nonetheless shipped.
- **Real-corpus probe is only 18 narrow fact-recall tasks** — a lower bound on utility, not evidence of broad operational value.
- **Headline 216,938 count mixes heterogeneous populations**; the quality-controlled subset is 114,565.

## Relevance

对关注 Agent 能力构建与检索增强的研究者而言，SkillCenter 提供了一个可直接落地的大规模可溯源技能库与流水线框架，其"技能仅在模型知识盲区且检索命中时才有效"的结论对 RAG/agent memory 与 skill-injection 系统的设计具有直接指导意义。但其标题级 FTS 检索的实际召回瓶颈与单次调用评估范式，也提醒后续工作需在语义检索与多轮 agent 场景下重新验证技能价值。

## Questions for Further Exploration

- Would replacing title-only BM25 with full-body or dense semantic retrieval close the 0% keyword-arm gap on the real-corpus probe, and at what latency/storage cost?
- Does the oracle-style gain (0% → 61-100%) survive in multi-turn, tool-using agentic harnesses (e.g., OpenClaw, Claude Code) rather than single-shot solvers?
- Can the 82.1% score-4 collapse be fixed with human calibration (the planned 2,000-skill stratified annotation) or multi-LLM cross-scoring, and do the per-domain score orderings then become meaningful?
- How much true redundancy (paraphrases) is hidden behind the lexical MinHash lower bound, and does a calibrated semantic dedup change the effective corpus size materially?
- What is the downstream impact on real software-engineering tasks (e.g., SWE-bench-style issue resolution) rather than synthetic/algorithmic pools?
- Can source-level prompt-injection screening be integrated into SkillGate without rejecting the large fraction of security/DevOps skills that legitimately quote dangerous patterns?
- Does the pipeline subset (114,565) alone, with web-derived skills excluded for licensing, still deliver competitive downstream utility?
