Methodology — the mechanism, distilled

PA-SciML wires a verification layer into the agentic SciML discovery loop. The key design choice is a clean separation of roles: an LLM-mediated Mechanician drafts a contract file (check type, parameters, hard/soft labels, tolerances) from the problem statement, but no LLM ever writes check code. Fixed framework routines — the Physics Auditor and a Sampled hard-contract gate — interpret and execute the contracts on each candidate's predicted fields.

Contracts cover linearity/scaling, symmetry, boundary/IC consistency, output bounds, and temporal/causality checks; tolerances are guided to be no tighter than 3× the measured calibration residual. Hard contracts gate reporting; soft contracts are diagnostic only. A second, separate mechanism — the Adversary — holds the trained surrogate fixed and varies only admissible inputs via constrained Adam ascent to maximize contract violation, reported separately from the gate and explicitly not aggregated into it. Crucially the Adversary needs no reference solution fields — exactly the setting where a deployed surrogate is most dangerous (unknown future inputs). Post-discovery, the Ablator attributes score gains to single edits (φi shares) and saves reusable method cards only when audit checks pass.

The static example is a no-failure verification case — the more accurate model still passes linear-elasticity checks (load superposition, 1/E scaling, roller BCs). The transient example is decisive: a ~3.5% lower rel-L2 masks a baseline whose pre-midpoint response changes by >100× its norm under the adversary — it answers the future before the past. The selected causal-by-construction surrogate (left-padded bias-free Conv1d discrete Duhamel integral) has violmax=0 at table precision. Search scale: 28 candidates (elasticity, 27 passed) / 16 candidates (elastodynamics, 12 passed).

Strengths

  • Clean separation of creative search (LLM) from executable evidence (fixed routines) — addresses a real trustworthiness gap in agentic SciML.
  • The Adversary inverts the training objective (fixed weights, varying admissible inputs) to find worst-case violations with no reference fields — exactly where deployment is dangerous.
  • Unusually high intellectual honesty: single-seed scope, bundle-of-changes confound, own-group baselines, contract re-grading caveats all self-reported.
  • Discovered architectures are mechanically principled (discrete superposition, discrete Duhamel hereditary integral) with attribution tying gains to those structural edits.
  • Reproducibility-by-record (Fixed evaluator, contract files, lineage, audit summaries) built into the workflow.

Limitations

  • Single-seed, single-run; no variance estimates. The most direct next check.
  • Audit-enabled arm is a bundle — no component ablation (gate vs. Numericist vs. physics-summary feedback).
  • Baselines are the authors' own workflow with verification disabled; no external agentic SciML comparison (ATHENA, GRAFT-ATHENA, AgenticSciML).
  • Headline decisive result depends on a post-run reference audit that re-grades causality soft→hard; live run would not disqualify.
  • Selected surrogates satisfy checks by construction, so the layer mostly confirms structure rather than catching well-trained non-compliant fields.

Open questions

  1. Does the causality win survive multi-seed runs and component-level ablations (gate on/off, Numericist on/off, physics-summary feedback on/off)?
  2. If causality were a hard contract during the live run, would the search still discover the causal surrogate — and at what compute cost?
  3. How does the framework behave on nonlinear constitutive laws (plasticity, hyperelasticity) and 3D, where linearity/Duhamel checks no longer apply?
  4. Does reusing saved method cards across problems actually improve later discovery runs (the testable claim the authors leave open)?
  5. How sensitive are pass/fail labels to the 3× calibration-residual tolerance rule and to the hard/soft labeling the Mechanician chooses?
  6. Can the Adversary give a certificate (provably bounded violation) rather than only the largest violation found within a budget?
