> **一句话总结：** 在 56 个数据科学任务、9 种模型配置共 7,560 次主消融运行中，单一 LLM 生成的扁平化家族级技能文件（含过程、示例、参考说明等组件）相对于纯任务提示几乎没有任何可靠提升（全条件总跨度仅 1.2 个百分点，所有 p 值 ≥ 0.396），甚至在数据准备阶段因通用启发式规则与任务指令冲突而造成 2.3–6.3 pp 的性能下降。

> **TL;DR:** This paper tests whether low-curation, single-shot LLM-generated SKILL.md files (produced by a Gemini 2.5 Pro coding agent) help data-science agents across four lifecycle stages (data preparation, extraction, statistical analysis, reporting). Across 56 tasks (using Spider, Fisher Iris, mtcars, Titanic, OpenWeatherMap, etc.), nine model configurations, and 7,560 main-ablation runs plus 3,024 supplemental control runs, no skill variant beats the task-only baseline: the Full skill averages 67.5% vs 68.3% No-Skill (−0.8 pp, p=0.497), performs no better than a token-matched office-supply control (+0.6 pp, p=0.775), and costs ~4.5× the input tokens. The null holds across all components and model groups, with harm concentrated in data preparation due to conflicting generic heuristics.

## 核心贡献

- **首次针对数据科学工作流进行技能组件级消融：** 跨越数据准备、数据提取、统计分析、报告四个生命周期阶段，覆盖 56 个任务、9 种模型配置（来自 OpenAI / Google / Anthropic 三家）与 7,560 次主消融运行，外加 3,024 次补充对照运行。
- **拆解技能的四个组成（Routing / Core Procedure / Worked Examples / Reference Notes）：** 通过机械删除各段得到 Core-Only、Core+Examples、Core+Refs、Full 四种条件，并加入 No-Skill 基线，构成 5 种可比条件，回答"到底是哪一部分有用/有害"。
- **设计两类关键的对照实验分离混淆因素：** (1) **Length-Control**（任务无关的"办公用品布置"技能，token 数控制在 Full 的 1% 以内）用于排除"加长提示"这一替代解释；(2) **Full+Priority**（在技能后插入"任务优先于技能"的用户级指令）用于检验技能-任务冲突假设。
- **提出三种可解释的失败模式：** 高基线处的冗余（data extraction 88.9%、reporting 97.6%）、中等基线处的冲突启发式（data preparation 57.1%，技能造成 −2.3 至 −6.3 pp 的损害）、地板处的计算/统计推理瓶颈（statistical analysis 仅 29.4%）。
- **给出对从业者明确的设计警示：** 在单次、扁平文件、家族级技能注入设定下，LLM 生成的技能不应作为默认策略；应转向任务级技能、按需检索加载、系统级优先级与执行反馈回路。

## Key Contributions

- First **component-level ablation of LLM-generated agent skills in the data-science domain**, evaluating four skill sections (Routing, Core Procedure, Worked Examples, Reference Notes) rather than treating skills as monolithic artifacts.
- A **rigorous null-result study** spanning 56 tasks (14 per lifecycle stage), 9 model configurations, 3 providers, 7,560 main-ablation runs plus 1,512 Length-Control runs and 1,512 Full+Priority runs (10,584 total), analyzed with mixed-effects models, group-stratified bootstrap, McNemar tests, and permutation tests.
- Two **mechanism-isolating controls**: a token-matched task-irrelevant office-supply skill (Length-Control) to rule out prompt-length as the cause of the null, and a user-message task-over-skill directive (Full+Priority) to test the conflicting-heuristics hypothesis.
- A **diagnostic taxonomy of three failure regimes** (redundancy at ceiling, conflicting heuristics at moderate baselines, capability gaps at floor) that explains why the same null result hides qualitatively different failure modes across lifecycle stages.
- **Practitioner-facing design implications**: against family-level single-shot skill injection as a default, and in favor of task-specific skills, selective/on-demand loading (especially gating Reference Notes), system-level priority rules, and execution-feedback agents.

## Methodology

**Skill construction (low-curation workflow).** One skill per lifecycle stage (4 skills total) was generated by a Gemini 2.5 Pro coding agent with extended thinking enabled, in a single autonomous session per skill with no human editing, candidate selection, or iterative refinement. Each skill contains four sections matching Anthropic's Agent Skills schema: Routing (~50 tokens, activation triggers), Core Procedure (~190 tokens), Worked Examples (~225 tokens), Reference Notes (~225 tokens). Unlike Anthropic's recommended multi-file architecture, all sections are injected as a single flat markdown file to isolate content effects from retrieval/loading effects.

**Ablation conditions.** Worked Examples and Reference Notes are treated as two independent binary factors (Routing + Core Procedure always held constant), yielding: (1) No-Skill, (2) Core-Only, (3) Core+Examples, (4) Core+Refs, (5) Full. Variants are produced by mechanically deleting sections from the Full skill (not by separate generation), so variants differ only in which sections are present. Two supplemental controls: Length-Control (token-matched office-supply skill, irrelevant domain, coherent procedural content) and Full+Priority (Full skill plus a user-message directive that task instructions override conflicting skill guidance).

**Tasks.** 56 tasks (14 per stage): Data Preparation (CSV null imputation, dedup, type coercion, date standardization on Melbourne Housing, Titanic, UCI Wine Quality, Auto MPG, student-performance datasets); Data Extraction (SQL against Spider and Chinook/Northwind-style schemas); Statistical Analysis (hypothesis tests, effect sizes on Fisher Iris, mtcars, PlantGrowth, ToothGrowth, sleep); Reporting (structured JSON from GitHub Events, OpenWeatherMap, REST Countries APIs). Tasks are split evenly between self-authored and externally grounded (28 each), all finalized before skill generation.

**Verification.** All outcomes binary pass/fail via deterministic verifiers (CSV DataFrame comparison, in-memory SQLite execution, numeric tolerance ε=0.01, JSON schema validation). No LLM acts as judge. Condition-flip cases were reviewed to confirm failures reflect substantive errors (wrong imputation method, wrong case convention) rather than trivial formatting mismatches.

**Models.** Nine configurations across three providers: GPT-4o-mini, GPT-4o, o3-mini, Gemini 2.5 Flash, Gemini 2.5 Flash+, Gemini 2.5 Pro, Claude Haiku 4.5, Claude Sonnet 4, Claude Sonnet 4+. Three pre-specified groups: compact default, frontier default, explicit-reasoning.

**Scale.** 56 × 5 × 9 = 2,520 main cells × 3 repetitions = 7,560 runs; 93.4% of cells unanimous across repetitions, outcomes by majority vote.

**Analysis.** Linear mixed-effects model (condition fixed effect, task random intercept, model random-effect variance component), group-stratified bootstrap, 10,000-iteration percentile bootstrap CIs over task-level paired differences, and paired McNemar tests. Logistic GEE and permutation tests give substantively identical conclusions.

## Results & Highlights

- **No skill variant significantly beats the task-only baseline.** Across all five conditions, pass rates span only 1.2 pp (67.3% Core+Refs to 68.5% Core+Examples); in the mixed-effects model all four skill conditions have p ≥ 0.396 (Full: β̂ = −0.008, p = 0.497; Core+Examples: β̂ = +0.002, p = 0.865).
- **Full skill underperforms No-Skill on aggregate**: 67.5% vs 68.3% (−0.8 pp, 95% CI [−3.8, +2.0], p = 0.644); paired McNemar shows 19 helps vs 23 hurts.
- **Token-matched Length-Control is statistically indistinguishable from Full**: Length-Control 66.9%, Full 67.5%, No-Skill 68.3%; Full exceeds Length-Control by only +0.6 pp (95% CI [−2.2, +3.4], p = 0.775), meaning the generated data-science content adds nothing over similarly long irrelevant context.
- **Prompt-level priority directive does not repair the conflict**: Full+Priority reaches 68.7%, only +1.2 pp over Full (95% CI [−1.0, +3.6], p = 0.362) and +0.4 pp over No-Skill (p = 0.875).
- **Component-level effects are all near zero**: Core+Refs 67.3% (−1.0 pp vs No-Skill), Core+Examples 68.5% (+0.2 pp), Core-Only 67.5% (−0.8 pp), Full 67.5%. The Worked×References interaction is −0.8 pp, ruling out large cancellation.
- **Harm is concentrated and large in Data Preparation**: No-Skill 57.1% vs Full 50.8% (skill effects −2.3 to −6.3 pp across conditions); 8 of 14 condition-flip cases (Core-Only pass → Core+Refs fail) are data-preparation tasks, where generic "forward-fill if <30% missing / fill strings with UNKNOWN" heuristics override explicit task instructions.
- **Ceiling and floor effects limit measurable gains**: Data Extraction 88.9% and Reporting 97.6% baselines leave little room; Statistical Analysis sits at 29.4% (floor), where the best skill condition adds only +0.8 pp — consistent with a computational bottleneck that prompt injection cannot fix. Only 17 of 56 tasks fall in the informative 30–80% band.
- **Skill injection costs ~4.5× input tokens** (mean 1,293 vs 287 for No-Skill) with no measured accuracy gain; e.g., Anthropic non-thinking Full prompts rise from 296 to 1,354 mean input tokens.
- **No model group benefits systematically**: compact (0.0 to +0.6 pp), frontier (−1.8 to +0.6 pp), explicit-reasoning (largest −1.8 pp for Core+Refs); all bootstrap CIs include zero. Individual models vary widely (Gemini Flash +7.1 pp from Core+Examples; GPT-4o-mini −5.4 pp from the same condition).

## Authors & Credibility

Sole author **Wei-Jung Huang**, an independent researcher based in the United States. The work is published at the **AIDataSci '26 workshop** (Jeju, Korea), co-located and peer-reviewed in a workshop venue focused specifically on AI for data science — a fitting forum for this methodologically careful benchmarking study. Credibility is strengthened by: pre-registration-style discipline (tasks and gold outputs finalized before skill generation), deterministic verifiers with no LLM-as-judge, four complementary statistical analyses (mixed-effects, bootstrap, McNemar, permutation) that agree, explicit declaration of the Gemini 2.5 Pro authorship-overlap confound, and unusually transparent reporting of a null result rather than selective positive framing. The 10,584-run scale and balanced three-provider design exceed what is typical for a single-author workshop paper.

## Critical Assessment

- **Novelty**: Genuinely novel in framing. Unlike SkillsBench (which found LLM-generated skills give no aggregate benefit across 86 general tasks) and SkillLearnBench, this is the first study to (a) target data-science workflows specifically and (b) run a true component-level ablation (Examples × References as binary factors with an interaction term). The Length-Control and Full+Priority controls are a thoughtful mechanism-isolation design.
- **Rigor**: High. Mixed-effects modeling with task and model random effects, 10,000-iteration bootstrap CIs, McNemar paired tests, permutation cross-checks, and majority-vote over 3 repetitions (93.4% cells unanimous). Honest about CIs spanning ±4–6 pp and therefore only ruling out moderate (not small 1–3 pp) effects.
- **Reproducibility**: Strong. Fixed gold files, deterministic verifiers, exact API model identifiers recorded in the evaluation harness, tiktoken version (v0.13.0) and tokenizer bases (o200k_base, cl100k_base) specified for Length-Control token matching, temperature set to 0 where supported. The main reproducibility gap is the absence of a public code/data release statement, though the task sources (Spider, public datasets, public APIs) are all named.
- **Significance**: The null result is valuable and counterintuitive for practitioners adopting SKILL.md-style workflows. The three-regime failure taxonomy (ceiling redundancy, conflicting heuristics, floor bottlenecks) is a reusable mental model. However, scope is narrow: single-shot, flat-file, family-level skills on verifiable execution tasks only.
- **Red flags / weaknesses**:
  - **Missing expert-written positive control**: The study explicitly omits a human-authored or task-specific skill, so it cannot tell whether the null reflects a property of LLM-generated skills, of family-level skills, or of this benchmark's sensitivity. This is the single biggest threat to the strong title claim.
  - **Authorship overlap**: Gemini 2.5 Pro authored both the skills and the self-authored task prompts/gold outputs, and is also one of the nine evaluated models. Although deterministic validation and within-task paired comparisons mitigate direct scoring bias, task style and difficulty could still favor Gemini-family outputs.
  - **Tiny informative sample**: Only 17 of 56 tasks fall in the 30–80% baseline band where skills could plausibly help; 39 tasks are near ceiling/floor and uninformative by construction. The effective statistical power for detecting small helpful effects is therefore limited.
  - **One skill per stage (n=4 skill instances)**: Section-level effects are tied to four generated skills, with no inter-generation variance estimated. A different generation seed could change conclusions, though the authors justify this as matching the one-candidate low-curation workflow.
  - **Length-Control uses only one irrelevant domain** (office-supply setup), so it cannot characterize all forms of unrelated context.
  - **Flat injection only**: Findings do not test Anthropic's recommended selective-loading architecture, so the title's broad claim should be read as scoped to single-shot flat-file injection.

## Strengths

- Exceptionally careful experimental hygiene: pre-finalized tasks/golds, deterministic verifiers, no LLM-as-judge, condition-flip case review to distinguish substantive errors from formatting noise.
- Four converging statistical analyses (mixed-effects, logistic GEE, bootstrap, permutation) that agree, plus honest reporting of wide confidence intervals.
- The Length-Control design (token-matched irrelevant skill) cleanly separates "skill content helps" from "long skill-formatted prefix has some generic effect" — a confound most prompt-engineering studies ignore.
- The three-regime failure taxonomy gives practitioners an immediately actionable diagnostic (check baseline difficulty before assuming skills will help).
- Transparent about all limitations, including the authorship overlap and the missing positive control, rather than burying them.

## Limitations

- **Narrow scope**: 56 single-turn tasks in one domain; only 17 in the informative baseline band; CIs of ±4–6 pp cannot detect small (1–3 pp) effects.
- **No expert-written or task-specific positive control**: leaves benchmark sensitivity to high-quality skills unmeasured.
- **Not a full agent setting**: omits multi-turn planning, tool use, code execution, memory, iterative revision, and upstream product-framing tasks. Skills are prepended as user-message content with no system-level priority, retrieval gate, or schema/tool constraint.
- **Length confound only partially addressed**: Length-Control uses a single irrelevant domain (office supplies).
- **One generated skill per stage (4 instances)**: no inter-generation variance; effects tied to four specific skill files.
- **Authorship overlap**: Gemini 2.5 Pro serves as skill author, task-prompt drafter, and evaluated model.
- **Condition-flip review is diagnostic, not a complete instruction-adherence taxonomy**: cannot count cases where the skill was ignored, partially used, or internally reconciled.

## Relevance

对正在构建数据科学 / 代码智能体（agent）或评估 SKILL.md 式可复用技能文件的研究者，这篇论文提供了直接相关的负面证据与组件级消融方法论——它提示在采用"一个家族级 LLM 生成技能 + 扁平注入"作为默认策略前，应优先验证任务基线难度区间，并考虑按需检索、系统级优先级与执行反馈等更强机制。

## Questions for Further Exploration

- Would an **expert-written or task-specific positive-control skill** show gains on the same 56 tasks, establishing whether the null is a property of LLM-generated skills, of family-level breadth, or of benchmark sensitivity?
- Does **selective / on-demand loading** (Anthropic's recommended multi-file architecture with retrieval gating of Reference Notes) recover the data-preparation harm, given that Core+Refs is the worst component (−1.0 pp)?
- Would **multiple independently generated skills per stage** (estimating inter-generation variance) change the conclusion, or do the four skill instances here happen to be unusually conflict-prone?
- Can **system-level priority rules** (vs the weak user-message Full+Priority tested here, +0.4 pp over No-Skill, p=0.875) reliably resolve skill-task conflicts?
- Does adding **execution feedback / iterative revision** help the floor-regime Statistical Analysis tasks (29.4% baseline), where prompt-level knowledge injection adds only +0.8 pp?
- How do these single-shot flat-injection findings transfer to **multi-turn agentic settings** with tool use, memory, and code execution?
