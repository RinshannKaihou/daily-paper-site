Key finding

A four-agent LLM loop with cross-task memory brings end-to-end automation to single-cell & spatial trajectory inference — reaching SOTA topology while cutting expert time by 41%.

SpaCellAgent: A Self-Evolving LLM-Based Multi-Agent Framework for Trajectory Analysis
Songhan Wang + 8 co-authors — USST / NUDT / HKBU / SUFE / Tsinghua / Beihang 2026-07-08 KDD 2026 (Jeju) · CC-BY 4.0 · code released DeepSeek-V3 backbone

01Key metric — analysis time

−41.2%

Total time 64.6 → 38.0 min vs. a 5-expert human baseline; +32.7% faster than the w/o-evolution ablation. Headline is a speed, not quality, gain.

02Why this matters

Closes a real gap in the trajectory-inference ecosystem: natural-language query to executable Python/R, plots, and a biological report — no manual pipeline stitching.

03Critical edge

SOTA deltas are measured against baselines at default parameters — which the TI benchmarking literature (Saelens 2019) itself warns against.

Methodology closed-loop four-agent pipeline

LLM-driven (DeepSeek-V3, temp 0.0 execution / 0.4 reflection) — turns a natural-language query into a JSON action plan, executable code, validation, and a report

Planner
NL → JSON plan
decomposes intent using gene-count, sparsity, modality metadata
Executor
Tool Selector + Coder
semantic routing to scanpy/Monocle/PAGA/Slingshot/SpaceFlow; Python + R polyglot
Evaluator
dual-layer check
code-level debug + biological validation; PubMed fallback outside memory
Reporter
insights + figures
synthesizes biological narrative and visual outputs

Self-refinement. Failed steps replay the full interaction history with semantic feedback and strategy-divergence prompts to avoid repetitive errors; successes commit to the intermediate dataset state.

Cross-task self-evolution archives validated workflows with biological metadata into a global memory; new datasets retrieve templates RAG-style and proactively apply accumulated error-fix pairs. A dual-layer (ephemeral local + persistent global) memory underpins it.

Knowledge-augmented fallback. When a task falls outside experience, the Biological Evaluator queries PubMed for tissue-specific markers and case studies — grounding judgments in peer-reviewed evidence rather than heuristic guessing.

Strengths & limitations side by side

Strengths
  • Genuinely end-to-end — raw data to biological narrative.
  • Dual-layer evaluator with PubMed-grounded fallback addresses a real agent failure mode.
  • Cross-task self-evolution empirically helps (32.7% faster than w/o-evolution).
  • Broad coverage: 4 metrics × 3 benchmark families × 3 real datasets + ablations.
Limitations
  • Baselines at default params systematically disadvantaged; deltas likely shrink if tuned.
  • Efficiency headline rests on 5 experts, one workflow, no variance; humans faster on Step 1.
  • LLM-as-judge circularity: biological evaluator validates an LLM-driven pipeline.
  • Best case study (Mouse SCI) is unpublished & hospital-sourced; high-salt framing unusual.

Open questions for further exploration

  1. How do the SOTA deltas change when each baseline (Monocle, PAGA, Slingshot, SpaceFlow, STORIES) is tuned per-dataset on the same metrics rather than run at defaults?
  2. What is the per-analysis LLM API cost, and how does performance degrade on >100k-cell datasets or with smaller open-source LLMs?
  3. Can the Biological Evaluator's pass/fail decisions be validated against the expert panel (Cohen's kappa) to break the LLM-as-judge circularity?
  4. Will the unpublished Mouse SCI dataset and high-salt-vs-normal-salt protocol be released for independent verification of the differentiation-blockade case study?
  5. How does the cross-task memory scale and avoid negative transfer when archived tree-topology workflows are retrieved for a cyclic one?
  6. Does the 41.2% efficiency gain hold with a larger expert pool, confidence intervals, and workflows beyond the single 6-step one?
