> **一句话总结：** 本文提出 PA-SciML，一个验证优先的 agentic SciML 工作流，在 LLM 代理发现代理模型时引入机器可检验的物理契约，在静力学弹性问题中比仅按误差选择更精确（rel-L2 8.86e-5 vs 2.10e-4），在瞬态弹性动力学中暴露了一个平均误差相近（2.63e-3 vs 2.72e-3）但违反因果性的非因果基线（采样违反度 12.62，对抗搜索 2750.62）。

> **TL;DR:** PA-SciML augments agentic SciML surrogate discovery with a fixed-code Verification layer that audits trained candidates against machine-checkable physics contracts (superposition, 1/E scaling, roller BCs, causality); on a parametric linear-elasticity DeepONet benchmark it lowers held-out rel-L2 to 8.864e-5 vs the error-only baseline's 2.103e-4, and on a transient elastodynamics benchmark it exposes an error-only baseline with near-identical rel-L2 (2.629e-3 vs 2.724e-3) that fails causality with a normalized violation of ~12.62 (sampled) and ~2750.62 (adversary search), while the selected causal surrogate passes.

## 核心贡献

- **将物理检查纳入代理模型选择证据链**：提出 PA-SciML 工作流，把"是否通过指定物理检查"作为模型选择的独立证据，与预测误差分开报告，而不仅仅用低误差代表物理正确性。
- **固定代码的物理审计层（Verification Layer）**：在 setup 阶段由 LLM 角色的 Mechanician 提出契约文件（仅选择支持检查类型与容差，不写可执行代码），随后由固定例程（Physics Auditor、Sampled hard-contract gate）对每个训练好的候选体预测场进行检查，避免 LLM 写检查代码的不可靠性。
- **无参考解的对抗式高违反输入搜索（Adversary）**：在训练好的代理上固定权重，仅在容许输入域内做约束 Adam 上升，寻找物理违反最大的输入，作为与 gate 分开报告的证据。
- **两类互补的数值验证场景**：静力学弹性（无失败案例，仅证明更精确的选择仍满足线性弹性检查）与瞬态弹性动力学（决定性场景：低误差但非因果的基线被检测出违反因果性）。
- **可归因、可复用的方法卡（Method Card）机制**：通过单编辑消融（Ablator，ϕ_i 份额）记录哪些孤立编辑带来得分增益，仅在通过分数与审计检查后保存可被后续运行检索的方法卡。

## Key Contributions

- **Makes specified physics-check evidence part of selection evidence**: PA-SciML requires a selected surrogate to pass prescribed machine-checkable physics checks under a stated verification setting, rather than treating low validation error as sufficient. Pass/fail labels and numeric scores are reported separately.
- **Fixed-code Verification layer with LLM-proposed, framework-interpreted contracts**: An LLM-mediated Mechanician drafts a contract file (check type, parameters, hard/soft labels, tolerances) from the problem statement and pre-training calibration evidence; fixed framework routines (Physics Auditor, Sampled hard-contract gate) interpret and execute them on candidate predicted fields. No LLM writes check code.
- **Adversary — a no-reference high-violation input search**: Holds the trained surrogate fixed and varies only admissible inputs (parameter box or bounded load-history mixtures) via constrained Adam ascent to maximize contract violation, reported separately from the gate. The 3x-calibration-residual rule guides tolerance choice.
- **Two complementary numerical demonstrations in computational solid mechanics**: A parametric linear-elasticity DeepONet (no-failure case showing accuracy without sacrificing linear-elastic checks) and a transient elastodynamics DeepONet (decisive case catching a causality violation that mean error misses).
- **Post-discovery attribution and Method-card saving**: The Ablator decomposes improving parent–child edits into ≤K isolated single edits, retraining each to compute a single-edit share ϕ_i; a method card is saved only when the dominant-edit share ≥ 0.5, sufficient relative improvement, and audit pass hold, enabling later runs to retrieve it.
- **Reproducibility-by-record design**: Complete run records (Fixed evaluator, contract files, search/audit budgets, candidate lineage, implementations, logs, audit summaries) keep execution failure separable from predictive error and physics-check behavior.

## Methodology

PA-SciML organizes agentic SciML surrogate discovery into three parts over a fixed evaluator and (when enabled) physics contracts:

1. **Discovery loop**. Setup fixes (a) a data summary, (b) a single lower-is-better Fixed evaluator, and (c) physics contracts *before* candidate search begins. Candidate search grows a Candidate search tree: LLM-mediated roles (Proposer, Critic, Engineer, Debugger, Selector, optional Retriever, optional Numericist for advisory pre-training probes) propose, critique, implement, and repair trial surrogates. Fixed routines then validate the interface, train, and score with the unchanged Fixed evaluator, recording whether the attempt succeeded so execution failure is not conflated with prediction error.

2. **Verification layer**. The Mechanician LLM role drafts the contract file (choosing among supported check families: linearity/scaling, symmetry/antisymmetry, boundary/IC consistency, output bounds, temporal/causality checks) from the problem statement and calibration evidence; tolerances are guided to be no tighter than 3x measured calibration residuals. Hard contracts gate reporting; soft contracts are diagnostics only. For each active hard contract c, the Physics Auditor applies the check to finite probes S_c and computes a dimensionless violation v_c(q); the sampled hard-contract statistic is viol_max = max_c max_q (v_c(q)/τ_c). A candidate passes the **Sampled hard-contract gate** when viol_max ≤ the reporting threshold. The optional **Adversary** runs a separate no-reference high-violation input search: trained surrogate held fixed, search variables z parameterize the admissible input domain Z, objective maximizes v_c(z) via constrained Adam ascent seeded from baseline samples and multi-start, using only surrogate outputs and contract conditions (no reference fields). It reports the largest violation found within the budget — not a proof that larger ones are absent. A Pareto front in (score, violation) may also be reported.

3. **Post-discovery attribution + Method-card saving**. The Ablator splits an improving parent–child code change into at most K edits; for each, the Engineer creates a parent-code variant containing only that edit and fixed routines retrain/rescore under the same Fixed evaluator. With g = s_parent − s_child and ϕ_i = (s_parent − s_edit,i)/g, the largest ϕ_i is an operational (non-causal) share of the gain. The Distiller saves a method card only after the saving checks pass: successful child, sufficient relative improvement, dominant single-edit share ≥ 0.5, no existing card for that candidate, and audit pass when required.

**Selected surrogates**. In elasticity, search discovered a physics-structured DeepONet: branch sees only Poisson ratio; loads and Young's modulus enter only through compliance prefactors f_x/E and f_y/E multiplying four heads; trunk produces 64 spatial bases per component; roller factors (1−x) on u_x and y on u_y enforce BCs identically; ≈6.3×10^5 parameters. In elastodynamics, search discovered a DeepONet with a left-padded bias-free causal Conv1d branch (discrete Duhamel integral) and a factorized space-time trunk (32 spatial × 32 temporal bases per component), with a lifting term α·ū (α=x+0.5) carrying the prescribed right-edge motion exactly and an envelope τ(t)β(x) vanishing at t=0 and at both edges; ≈2.0×10^5 parameters.

## Results & Highlights

**Parametric linear elasticity (FE mesh of 2601 nodes; 4000 train / 1000 validation cases; rel-L2 metric):**

- Selected audit-enabled surrogate (physics-structured DeepONet) attains held-out **rel-L2 = 8.864 × 10⁻⁵**, vs the error-only baseline's **2.103 × 10⁻⁴** — roughly a 2.4× reduction; worst validation case **1.376 × 10⁻⁴ vs 3.376 × 10⁻⁴**; 95th-percentile error **1.249 × 10⁻⁴ vs 3.148 × 10⁻⁴**.
- Both surrogates pass the common reference audit (load superposition, 1/E scaling, right-edge u_x, bottom-edge u_y as hard checks): selected sampled viol_max = 1.708 × 10⁻⁴ (adversary 2.087 × 10⁻³); error-only baseline sampled viol_max = 1.080 × 10⁻³ (adversary 2.582 × 10⁻³). Both well below the pass threshold of 1.
- Candidate search: 28 trial surrogates, all scored/audited, 27 passed the live gate; the single failure was a degenerate candidate whose field collapsed toward zero (violation 4 orders of magnitude above threshold).
- Isolated-edit attribution: the physics-structured DeepONet branch factorization alone recovered **149%** of its step's score gain (other simultaneous changes neutral or harmful), and was saved as the only method card from this example.

**Transient elastodynamics (FE mesh of 4681 nodes; 500 validation load histories; rel-L2 over displacement history):**

- Selected audit-enabled surrogate attains **rel-L2 = 2.629 × 10⁻³** vs the error-only baseline's **2.724 × 10⁻³** (~3.5% lower); 95th-percentile **4.42 × 10⁻³ vs 4.90 × 10⁻³**; worst case **5.48 × 10⁻³ vs 9.51 × 10⁻³**.
- Under the common reference audit with causality as a hard check, the error-only baseline **fails decisively**: sampled causality violation **12.62** (≈63% pre-midpoint relative-L2 change vs 5% tolerance) and Adversary-search causality violation **2750.62** (pre-midpoint response changes by >100× its norm). It also breaches load superposition (2.79) and prescribed-right-edge (1.99) in the Adversary report. The selected causal-by-construction surrogate has viol_max = 0.00 at table precision (only nonzero entries: 4.5 × 10⁻⁶ sampled, 6.7 × 10⁻⁵ under Adversary, on load-linearity at the single-precision noise floor).
- Candidate search: 16 trial surrogates, 2 exceeded the training-time budget; of 14 scored, 12 passed the in-run gate; the 2 in-run gate failures predicted nonzero response under zero input (zero-input violations 12.5 and 4.7).
- Isolated-edit attribution: three dominant edits saved as method cards — linear-residual operator network (129% of step gain), factorized space-time with causal load branch (57%), and per-component temporal bases with Fourier features (55%).

**Aggregate takeaway**: The framework's contribution is *per-candidate physics evidence on predicted fields*, not a richer aggregate score. The static example is a no-failure verification case (audit certifies the more accurate model at no physics cost); the transient example is decisive (an accurate-but-acausal baseline is exposed by a check that no held-out error statistic detected).

## Critical Assessment

- **Novelty**: Genuine and well-positioned. The gap is real: existing agentic SciML systems (ATHENA, GRAFT-ATHENA, AgenticSciML) select on composite reward / training loss / validation error / diagnostics aimed at *code* and *plans*, not on per-candidate output-level physics checks. PA-SciML's separation of "fixed-code contracts proposed by an LLM-but-not-written-by-an-LLM" from the search loop, plus the Adversary as a no-reference violation maximizer, is a distinct contribution. However, the underlying ingredients (formal verification of trained networks, PINN hard constraints, adversarial robustness) exist in the trustworthiness literature — the novelty is the *workflow integration* into agentic discovery, not any single component.
- **Rigor**: The verification layer is carefully specified (contract file fixed *before* search; hard vs soft contracts; tolerances guided by 3× calibration residual; Adversary reported separately and explicitly *not* aggregated into the gate; honest caveats that a pass is contract-bounded in-domain evidence, not a proof outside Z). Numbers are reported to 3–4 significant figures with units and percentile/worst-case breakdowns. The isolated-edit attribution (ϕ_i, eq. 2) is honestly labeled as "operational share, not causal proof."
- **Weak baselines / confounds**: The comparison is **one audit-enabled run vs one error-only run at a single random seed** — the authors admit this. The audit-enabled arm differs from the error-only arm as a *bundle* (Verification layer + advisory numerical probes + physics summaries to later proposals all change together); component-wise ablations (e.g., Sampled hard-contract gate off, Numericist off) are explicitly listed as future work. The baselines are the authors' *own* workflow with verification disabled, not independent external systems; the transient "error-only baseline" is described as an earlier run whose score the post-run reference audit reproduces. There is no comparison to a third-party agentic SciML system (ATHENA / GRAFT-ATHENA) on the same problems.
- **Cherry-picking risk**: The two selected architectures satisfy their checks *by construction* (compliance prefactors, roller factors, causal left-padded Conv1d), which somewhat inverts the framing — the physics checks largely *confirm* the discovered structure rather than catching a well-trained non-compliant field. The decisive causality failure comes from the post-run reference audit with causality *re-labeled as hard*, even though the live run had recorded causality only as a soft diagnostic. This re-grading is disclosed but means the headline "decisive pass/fail contrast" depends on a post-hoc contract choice.
- **Missing ablations**: No variance over seeds; no sensitivity to the 3× tolerance rule; no report of compute or language-model usage cost; no test on nonlinear constitutive behavior or 3D (the checks are linearity/causality-based and regime-specific); 2 of 16 transient candidates exceeded the training-time budget and were conservatively counted as failures.
- **Overclaiming**: Largely avoided. Claims are explicitly scoped to "the stated budgets," "single-seed," "the stated verification setting," and "in-domain on Z." The conclusion that "low error should not be read as evidence that a selected surrogate passes the stated physics checks" is supported by the transient example.

## Strengths

- Clean conceptual separation of creative model search (LLM-mediated) from executable evidence production (fixed routines), which addresses a real trustworthiness gap in agentic SciML.
- The Adversary is a thoughtful design: it inverts the training objective (fixed weights, varying admissible inputs) to look for worst-case physics violations *without needing reference solution fields*, which is exactly the setting where a deployed surrogate is most dangerous (unknown future inputs).
- Intellectual honesty is unusually high for a discovery-system paper: single-seed scope, bundle-of-changes confound, own-group baselines, and contract-re-grading caveats are all stated by the authors themselves rather than hidden.
- The discovered architectures are mechanically principled (discrete superposition u = (f_x/E)G_x(ν) + (f_y/E)G_y(ν); discrete Duhamel hereditary integral with a lifting term) and the isolated-edit attribution ties the score gains to these specific structural edits.
- Full reproducibility-by-record (Fixed evaluator, contract files, lineage, audit summaries, logs) is built into the workflow design.

## Limitations

- Single-seed, single-run comparison; no variance estimates. The authors flag this as the most direct next check.
- Audit-enabled arm is a bundle — no component-level ablation isolating the contribution of the Sampled hard-contract gate, the advisory Numericist probes, or the physics-summary feedback to later proposals.
- Baselines are the authors' own workflow with verification disabled; no independent external agentic-SciML baselines (ATHENA, GRAFT-ATHENA, AgenticSciML) on the same problems.
- The headline decisive result depends on a *post-run reference audit* that re-grades causality from soft to hard; in the live run, the causality failure would have been reported but not disqualifying.
- Search scale is modest (28 and 16 trial surrogates); 2 transient candidates exceeded the training-time budget and were treated as failures.
- Demonstrated only on 2D linear problems (static elasticity, transient elastodynamics) with linearity/superposition/causality checks; nonlinear constitutive laws, 3D domains, and data-poor / residual-training variants are left to future work.
- The selected surrogates satisfy checks *by construction*, so the verification layer mostly confirms structure rather than catching well-trained non-compliant fields; the catching role is shown only via the post-run re-audit of the error-only baseline.
- Method-card reuse improving later searches is presented as a mechanism but not empirically demonstrated to help.

## Relevance

这项工作与 agentic SciML / 神经算子代理模型可信化高度相关：它直接回应了"LLM 代理自动发现代理模型时如何提供物理可信证据"这一关键问题，对于将 LLM 驱动的 SciML 系统部署到工程仿真与数字孪生中有实际指导意义。对关注 LLM agent、自动化科研与可信机器学习的研究者，其"固定代码物理契约 + 无参考对抗搜索"的验证范式可作为构建更可靠自主科研系统的参考。

## Questions for Further Exploration

- How do the results change under multi-seed runs and component-level ablations (gate on/off, Numericist on/off, physics-summary feedback on/off)? Does the causality win survive if only the Sampled hard-contract gate is enabled?
- Would making causality a *hard* contract during the live run (rather than only in the post-run reference audit) change which surrogate the search discovers, and at what compute cost?
- How does the framework behave on nonlinear constitutive laws (plasticity, hyperelasticity) and 3D domains, where linearity-based and Duhamel-style checks no longer apply — what regime-appropriate contract families are needed?
- Does reusing saved method cards across problems actually improve later discovery runs (the testable claim the authors leave open)?
- How sensitive are pass/fail labels to the 3× calibration-residual tolerance rule and to the hard/soft labeling chosen by the Mechanician?
- Can the Adversary search be made to give a *certificate* (provably bounded violation) rather than only the largest violation found within a budget, e.g., via formal verification for piecewise-linear surrogates?
- How does PA-SciML compare against independent external agentic SciML baselines (ATHENA, GRAFT-ATHENA, AgenticSciML) on identical problems and budgets?
