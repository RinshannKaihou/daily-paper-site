SkillCoach: Self-Evolving Rubrics for Evaluating and Enhancing Agentic Skill-Use

arXiv: 2607.01874

Date: 2026-07-03

https://arxiv.org/abs/2607.01874

一句话总结： SkillCoach 把"智能体技能使用"定义为轨迹级元能力（技能选择 / 技能跟随 / 技能组合 / 技能反思四维），从真实 rollout 自演化出基于技能的过程 rubric（与外部验证器分离），既用于诊断"最终通过但过程脆弱"的失败，也用作训练监督筛选器；rubric 自演化后 gold-keypoint 覆盖率 71.56→83.70、可用性 81.53→94.33、幻觉率 2.00→0.00、轨迹过滤一致性 82.00→96.00，且比 outcome-only SFT 提供更强监督信号。
TL;DR: SkillCoach is the first self-evolving rubric framework for evaluating and enhancing agentic skill-use. It defines skill-use as a trajectory-level meta-ability along four dimensions (skill selection, skill following, skill composition, skill-grounded reflection), keeps the external verifier as a separate outcome signal (so process quality is distinguishable from accidental task success), and evolves task-level process rubrics from real rollouts via evidence-grounded judging + local arbitration patches + validation-gated updates. On a distractor-augmented skill-dependent benchmark, evolved rubric R_best improves gold-keypoint coverage 71.56 → 83.70, usability 81.53 → 94.33, hallucination rate 2.00 → 0.00, and trajectory-filtering consistency 82.00 → 96.00 vs the initial rubric R_0 (50 paired instances, 10 held-out task families, Gemini 3.1 Pro temp-0 judge). Skill dependence is confirmed (Opus 4.7 final accuracy 18.0 No-Skills → 88.0 Gold-Skills; GPT-5.5 24.0 → 80.0); distractors expose the real bottleneck — Gemini 3.1 Pro selection 98.0 → 78.0, Qwen3.5-9B selection 92.0 → 44.0 under Gold+Distractors — that final accuracy alone hides.

核心贡献

  1. 首次形式化"智能体技能使用"为四维轨迹级元能力：技能选择、技能跟随、技能组合、技能反思；与外部验证器信号分离，从而区分"可靠技能使用"与"偶然验证器通过"。
  2. 自演化 rubric 框架 SkillCoach：从真实 rollout 通过证据锚定评判、局部仲裁补丁、验证门控更新来校准任务级过程 rubric；面向"技能依赖任务 + 干扰技能增强库"的真实部署设定。
  3. rubric 的双重作用：（a）评估端——暴露被最终准确率掩盖的技能使用失败；（b）训练端——作为监督筛选器选出比 outcome-only SFT 更高质量的轨迹，四维元能力可作为可分离的监督信号。
  4. 强实证：rubric 自演化在四项质量指标上全面提升；distractor 库暴露出"瓶颈在可靠选择与执行而非技能可用性"。

Key Contributions

  • First formulation of agentic skill-use as a trajectory-level meta-ability spanning skill selection, skill following, skill composition, and skill-grounded reflection — kept separate from the external verifier signal.
  • SkillCoach, the first self-evolving rubric framework that calibrates task-level process rubrics from real rollouts via evidence-grounded judging, local arbitration patches, and validation-gated updates, under a skill-dependent setting with distractor-augmented libraries mirroring enterprise repositories.
  • Dual role of evolved rubrics: (a) evaluation — they improve evaluation quality and expose failures hidden by final accuracy; (b) training — as a supervision filter they select better trajectories than outcome-only SFT, with the meta-ability dimensions acting as dissociable supervision signals.
  • Empirical validation: evolved rubrics improve all four rubric-quality metrics; distractor-augmented libraries reveal reliable skill selection (not availability) as the bottleneck.

Methodology

Problem framing. Skills are a reusable operational layer (SOPs, tool workflows, scripts, validation routines). In real repositories, overlapping/distractor skills make reliable skill-use hard; final verifier success is too coarse — an agent may pass via trial-and-error while selecting distractor skills, skipping steps, composing workflows incorrectly, or omitting checks (Fig. 1 contrasts Trajectory A "correct outcome & fully use skill" vs Trajectory B "correct outcome but not reusable").

Skill-dependent task design (§3.2). Tasks are chosen to be difficult without the relevant gold skill but improvable when it is available; gold skills are mixed with distractors (2 unrelated + 3 semantically-similar-but-inapplicable) to approximate enterprise repositories.

Four process dimensions (the meta-ability).

  1. Skill selection — does the agent pick the gold skill over distractors?
  2. Skill following — does it execute the prescribed key steps (e.g., "Downloading NWS Flood thresholds", nwis.get_IV(ParameterCd='00065'))?
  3. Skill composition — does it compose multi-skill workflows in the correct order?
  4. Skill-grounded reflection — does it perform skill-specified final checks (e.g., verifying station id and flood days) before submission?

Rubric self-evolution. For each task: (i) derive an initial rubric R_0 from gold skills; (ii) score real rollouts with observable evidence; (iii) refine via validation-gated local patches (evidence-grounded judging + local arbitration patches) to produce R_best. The external verifier is kept as a separate outcome signal — process quality is never conflated with accidental verifier passing.

Evaluation protocol (§4.1). 50 paired test instances from 10 held-out task families; each task evaluated under the same package and the same human-gold process reference (annotated from oracle materials: gold skills, distractor boundaries, key steps, composition order, reflection criteria). To avoid rubric-generation/evaluation overlap, Gemini 3.1 Pro at temperature 0 is used as an independent rubric-quality judge, instructed to ignore verifier pass/fail. Four metrics: gold-keypoint coverage, usability (0–100, averaging completeness/logical-consistency/judge-usability), hallucination rate (fraction of unsupported requirements), trajectory-filtering consistency (accept/reject agreement with human-gold trajectory judgment).

Agents benchmarked (Table 3). DeepSeek-V4-Flash, Opus-4.7, GPT-5.5, Gemini-3.1-Pro, Kimi-K2.6, Qwen3.5-4B, Qwen3.5-9B, under No-Skills / Gold-Skills / Gold+Distractors library conditions.

Results & Highlights

  • Rubric self-evolution (Table 2, R_0 → R_best, 50 paired instances, 10 held-out families): gold-keypoint coverage 71.56 → 83.70; usability 81.53 → 94.33; hallucination rate 2.00 → 0.00 (gains do not come from unsupported requirements); trajectory-filtering consistency 82.00 → 96.00.
  • Skill dependence confirmed (Table 3, No-Skills → Gold-Skills final accuracy): Opus 4.7 18.0 → 88.0; GPT-5.5 24.0 → 80.0; Gemini 3.1 Pro 16.0 → 72.0; Kimi K2.6 16.0 → 66.0; DeepSeek-V4-Flash 10.0 → 32.0; Qwen3.5-9B 0.0 → 8.0; Qwen3.5-4B 2.0 → 18.0 — confirming tasks test skill use, not memorization.
  • Distractors expose the real bottleneck (Gold → Gold+Distractors): final accuracy drops for most agents (e.g., Gemini 3.1 Pro 72.0 → 70.0 but selection score 98.0 → 78.0; Qwen3.5-9B selection 92.0 → 44.0 with lower reflection and final accuracy). The four-dimension breakdown shows once the correct skill is reached, partial execution may still occur, but finding it becomes fragile when the library has plausible alternatives.
  • Outcome-only metrics hide the failure mode: a lower verifier score does not reveal whether failure is wrong skill choice, incomplete step-following, weak composition, or missing checks; conversely a trajectory may preserve useful process behavior even when the final outcome fails.
  • Training-side gain (outcome-only SFT vs rubric-filtered supervision): evolved rubrics select higher-quality trajectories for supervised skill-use training; the four meta-ability dimensions act as dissociable supervision signals (the abstract's central training claim; per-dimension ablation in §4.3).

Critical Assessment

  • The rubric-quality evaluation depends on Gemini 3.1 Pro as judge (temp 0, instructed to ignore verifier signals) — a single LLM judge for rubric quality introduces a judge-bias risk (e.g., systematically preferring more verbose/structured rubrics). The paper mitigates this with human-gold process references and paired design, but does not report multi-judge agreement or a human-rubric-quality ceiling.
  • The skill-dependent task selection is the linchpin: the headline "skill use is measurable and trainable" rests on tasks where No-Skills accuracy is very low (e.g., Opus 18.0, Qwen3.5-9B 0.0). On tasks where strong agents can already solve without skills, the four-dimension signal may be weaker; the paper does not characterize the skill-dependence threshold below which the framework adds value.
  • The distractor setting (2 unrelated + 3 semantically-similar) is a controlled approximation of enterprise repositories; real enterprise libraries may have 10s–100s of overlapping skills with retrieval/error upstream of selection, which this benchmark does not exercise.
  • The training-side claim ("rubric-filtered SFT beats outcome-only SFT") is the most actionable result but the parsed text only states it qualitatively in the abstract/intro; the per-dimension ablation (§4.3) numbers are referenced but not fully captured in the abstract-level summary, so the magnitude of the training gain should be read from §4.3 directly.
  • No public code/benchmark link is visible in the parsed text, limiting independent reproduction of the 50 paired instances and the human-gold process references.

Strengths

  • Clean four-dimension decomposition (selection / following / composition / reflection) that turns "skill use" from a binary outcome into a diagnosable trajectory-level meta-ability.
  • Self-evolving rubric with validation-gated updates — the hallucination-rate drop (2.00 → 0.00) shows coverage gains are not bought with unsupported requirements, an honesty signal.
  • Keeping the verifier separate from process evaluation is the right methodological choice (avoids circular "passed verifier = good skill use" reasoning).
  • Strong agent roster (Opus 4.7, GPT-5.5, Gemini 3.1 Pro, Kimi K2.6, DeepSeek-V4-Flash, Qwen3.5-4B/9B) makes the distractor-bottleneck finding credible across model scales.

Limitations

  • Single LLM judge (Gemini 3.1 Pro) for rubric quality; no multi-judge agreement or human ceiling reported.
  • Skill-dependence threshold uncharacterized — value-add unclear on tasks strong agents solve without skills.
  • Distractor setting is a controlled approximation (5 distractors); real enterprise libraries are larger and add retrieval upstream.
  • Training-side gain magnitude in §4.3 not captured in the abstract; read §4.3 for per-dimension ablation numbers.
  • No public code/benchmark link visible in the parsed text.
