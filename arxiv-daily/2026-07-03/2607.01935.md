> **一句话总结：** 本文定义长期记忆中的"幽灵记忆（ghost memory）"失败——旧/当前/迁移事实在记忆库中共存、检索时混在一起、最终误导回答模型；提出 A-TMA 状态感知叠加层（保留历史但显式暴露 current/historical/transition 三类状态标签），并构建冲突密集基准 LTP；在 LTP 上 Graphiti/Zep+A-TMA 把冲突准确率从 0.480 提升到 0.720（+0.240 绝对），在 LoCoMo 上把 temporal F1 从 0.0295 提升到 0.1705。

> **TL;DR:** The paper studies "ghost memory" — a state-coordination failure where old, current, and transition facts coexist in a long-term agent memory bank, remain mixed during retrieval, and mislead the answer model. It argues memory must be understood at three levels (bank maintenance, retrieval, answer-time resolution) and proposes **A-TMA**, a state-aware overlay that keeps superseded/transition records but builds evidence packets with explicit current/historical/transition labels exposed to QA. It also builds **LTP (LoCoMo Temporal Plus)**, a conflict-heavy benchmark (10 profiles, 800 probes) for ghost memory. On LTP, Graphiti/Zep + A-TMA improves conflict accuracy **0.480 → 0.720 (+0.240 absolute)**; on LTP profile000, A-Mem+A-TMA reaches 0.775 Acc vs AriadneMem 0.438; paired Graphiti/Zep raises conflict accuracy 0.425 → 0.725 (temporal-KG evidence support stays 0.812). On LoCoMo (10 samples, 1,986 QA pairs), Graphiti/Zep + A-TMA raises temporal F1 **0.0295 → 0.1705** and average F1 **0.0809 → 0.1556**. Gains are host-dependent but show explicit state roles reduce failures hidden by final QA accuracy.

## 核心贡献

1. **定义"幽灵记忆"失败**：把长期记忆中"旧/当前/迁移事实共存并在检索时混合、最终误导回答"定义为一个状态协调失败（不是单纯的旧记录存在，而是状态角色未显式化）。
2. **三层解耦评估主张**：记忆系统应在"库维护 / 检索 / 答题时解析"三层分别评估，因为单一 QA 准确率会掩盖幽灵记忆发生在哪一层。
3. **A-TMA 状态感知叠加层**：不替换宿主记忆基底，保留历史与迁移记录，但为查询构建带 current/historical/transition 标签的证据包并暴露给 QA。
4. **LTP 冲突密集基准**：基于 LoCoMo 构建的 10 profiles / 800 probes 基准，专门测试幽灵记忆；在 LTP 与 LoCoMo 上验证 A-TMA 的宿主相关增益。

## Key Contributions

- **Defines "ghost memory"** as a state-coordination failure (old/current/transition facts coexist, mix during retrieval, mislead the answer model) — not mere presence of old records, but implicit state roles across bank/evidence/answer.
- **Three-level decoupled evaluation thesis**: memory should be evaluated separately at bank maintenance, retrieval, and answer-time resolution, because final QA accuracy hides where ghost memory occurs.
- **A-TMA, a state-aware overlay**: keeps superseded and transition records in the bank, builds evidence packets for the query's requested state view, and exposes current/historical/transition labels to QA. Host-agnostic — does not replace the memory substrate.
- **LTP (LoCoMo Temporal Plus) benchmark**: conflict-heavy benchmark (10 profiles, 800 probes) for ghost memory; evaluated alongside LoCoMo (10 samples, 1,986 QA pairs) for long-conversation generalization.

## Methodology

**Ghost memory framing.** A user fact changes (address, job, plan). A memory that treats every stored fact as timeless is wrong even when each fact was true when written. The right response is *not deletion* — history must be preserved — but the *state* of each memory (current vs historical vs transition) must be made explicit at retrieval and answer time.

**Three coupled levels.** (1) *Bank maintenance* — does the bank preserve old and current states cleanly? (2) *Retrieval* — does it choose evidence for the requested state view? (3) *Answer-time resolution* — does the answer model see needed evidence and still collapse current/historical/transition into one response? A single QA score hides which level failed.

**A-TMA overlay.** A-TMA does not replace the host memory substrate (Graphiti/Zep, A-Mem, InsideOut, SeCom, AriadneMem). It adds: (a) keeps superseded and transition records in the bank; (b) builds evidence packets for the query's requested state view; (c) exposes current/historical/transition labels to QA so the answer model can pick the live value for "now" questions and the historical value for "before" questions.

**Benchmark — LTP (LoCoMo Temporal Plus).** Conflict-heavy benchmark built from LoCoMo: 10 profiles, 800 probes, each profile contains paired old/current/transition facts designed to trigger ghost memory. LoCoMo (10 samples, 1,986 QA pairs) is used for long-conversation generalization.

**Evaluation metrics.** Conflict accuracy (does the system pick the correct state under conflicting old/current facts?), target accuracy, evidence support (does retrieved evidence contain the needed state?), temporal F1, average F1, QA accuracy, evidence hit rate. ## Results & Highlights

- **LTP headline (Graphiti/Zep + A-TMA):** conflict accuracy **0.480 → 0.720 (+0.240 absolute)**.
- **LTP profile000 (acc across hosts):** A-Mem+A-TMA **0.775**, InsideOut+A-TMA **0.675**, SeCom+A-TMA **0.637**, vs AriadneMem (timestamped graph facts + reasoning paths) **0.438** — showing explicit state roles beat timestamp/graph-label-only evidence.
- **Paired Graphiti/Zep isolation:** temporal-KG evidence reaches **0.812 evidence support** but only **0.425 conflict accuracy**; +A-TMA raises conflict accuracy to **0.725** — i.e. the evidence *was* retrieved but the answer model collapsed states without explicit labels.
- **LoCoMo (long-conversation generalization, sample0):** Graphiti/Zep + A-TMA raises temporal F1 **0.0295 → 0.1705** and average F1 **0.0809 → 0.1556**; also improves QA accuracy and evidence hit rate. AriadneMem remains stronger on raw F1 on sample0 (host-dependent).
- **Decoupled evaluation finding:** a single QA score hides whether failure is bank-level (states not preserved), retrieval-level (wrong state-view evidence), or QA-level (evidence present but states collapsed) — A-TMA's contribution is isolating these.

## Critical Assessment

- **The gains are explicitly "host-dependent"** (the paper's own wording): A-TMA helps Graphiti/Zep substantially (+0.240 conflict acc) but on LoCoMo sample0 AriadneMem remains stronger on raw F1. The paper is honest about this, but it means A-TMA is not a universal win — its value depends on whether the host already exposes state roles (AriadneMem's timestamped graph partially does) vs treats facts as timeless (Graphiti/Zep's default).
- **The headline +0.240 conflict accuracy is on LTP, a benchmark the authors built themselves** from LoCoMo — LTP is conflict-heavy by construction, so the gain is measured on the failure mode A-TMA targets. The LoCoMo gains (temporal F1 0.0295 → 0.1705) are smaller in absolute terms and the paper notes metric-dependence on sample0. This is a real but scoped win.
- **The three-level decoupled evaluation is the most novel methodological contribution** but the paper reports it mostly through paired metric comparisons rather than a full per-level failure-attribution study (how often does ghost memory originate at bank vs retrieval vs QA?). The decoupling thesis is asserted more than fully quantified.
- **Scale is modest**: LTP = 10 profiles / 800 probes; LoCoMo = 10 samples / 1,986 QA pairs. Whether the state-overlay helps on larger, messier real deployment logs (with 100s of evolving facts per user) is unverified.
- **A-TMA preserves history rather than deleting** — this is the right call for "what was true before" queries, but it increases bank size and evidence-packet complexity; the storage/retrieval overhead of keeping all superseded+transition records is not quantified.

## Strengths

- Clean conceptual framing of "ghost memory" as a state-coordination failure (not a storage problem), correctly arguing against deletion-as-fix.
- Three-level decoupled evaluation thesis is the right diagnostic posture for a field that over-relied on aggregate QA accuracy.
- A-TMA is a host-agnostic overlay (works on Graphiti/Zep, A-Mem, InsideOut, SeCom), not a new siloed system — practical for adoption.
- Paired comparisons and multiple complementary metrics (conflict acc, evidence support, temporal F1) give a more honest picture than a single leaderboard number.

## Limitations

- Gains are host-dependent; not a universal win (AriadneMem already partially state-aware).
- Headline +0.240 is on LTP (self-built, conflict-heavy by design); LoCoMo gains smaller and metric-dependent.
- Three-level decoupling asserted more than fully quantified (per-level failure attribution not exhaustively reported).
- Modest scale (10 profiles / 800 probes on LTP); large messy deployment logs untested.
- Overhead of preserving all superseded+transition records not quantified.

## Questions for Further Exploration

1. A-TMA 的增益是宿主相关的——能否预测哪些宿主（已部分状态感知 vs 视事实为 timeless）受益最大？给出一个"状态角色显式度"度量来排序宿主？
2. 三层解耦评估能否给出每层失败率的量化归因（库/检索/QA 各占多少幽灵记忆失败）？
3. 保留所有 superseded+transition 记录的存储与检索开销如何？能否用状态角色感知的剪枝/压缩控制 bank 增长？
4. LTP（自建冲突密集基准）的 +0.240 增益在更小冲突密度的真实部署日志上如何退化？是否存在冲突密度阈值？
5. 把 current/historical/transition 标签从"暴露给 QA"改为"由检索层直接过滤到请求状态视图"，哪个层级介入最优？
6. 状态角色显式化能否与时序知识图谱（Graphiti/Zep）的时间戳机制统一，避免重复标注？
7. 在多用户、长期（月/年级）演化场景下，状态迁移链的长度是否会超出证据包的可处理规模？
