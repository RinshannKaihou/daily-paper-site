Safe and Adaptive Cloud Healing: Verifying LLM-Generated Recovery Plans with a Neural-Symbolic World Model

arXiv: 2607.01595

Date: 2026-07-03

https://arxiv.org/abs/2607.01595

一句话总结： PASE 把云故障自愈重构为"神经符号程序合成"任务——LLM 作规划引擎生成多步恢复计划，神经符号世界模型在执行前做可行性仿真验证，SAC 训练的元提示优化器在线适配未见故障；在 OpenStack 故障注入数据集上，故障检测 F1 达 0.94、平均恢复时间比基线 IFSHM 缩短 >40%（72s vs 85s）。
TL;DR: PASE reframes cloud fault recovery as neuro-symbolic program synthesis: an LLM Plan Synthesis Engine composes multi-step recovery plans from semantic primitives, a Neural-Symbolic World Model (NSWM) verifies plan feasibility by rollout simulation before execution, and a Soft Actor-Critic (SAC) Meta-Prompt Optimizer learns continuous prompt embeddings to adapt planning online. On the OpenStack Failure-Dataset, PASE lifts fault-detection F1 to 0.94 (vs IFSHM 0.92, Deformable DETR-FD 0.88, GCN-FR 0.85, Rule-Based 0.75) and cuts average recovery time to 72 s (vs IFSHM 85 s, Deformable DETR-FD 105 s, GCN-FR 115 s) — a >40% MTTR reduction over IFSHM — and adapts to a novel hybrid CPU-Memory deadlock fault from 40% → 80% recovery success within ~15 episodes vs IFSHM's slower 75% convergence.

核心贡献

  1. 范式转换：把云故障恢复从"状态分类 + 单步动作选择"重构为"神经符号程序合成"——LLM 生成多步恢复计划而非从预定义动作空间挑选。
  2. 验证式规划闭环：LLM 规划引擎（PSE）+ 神经符号世界模型（NSWM）+ DRL 元提示优化器（MPO）组成"推理-规划-验证-适配"闭环，在执行前对计划做仿真可行性筛查。
  3. 元提示在线适配：用 SAC 在连续提示嵌入空间优化 LLM 的规划过程，避免为每个新故障类型做昂贵的全模型微调。
  4. 实证验证：在真实 OpenStack 故障注入数据集上验证，MTTR 比 IFSHM 缩短 >40%，对未见故障的快速适配能力显著优于基线。

Key Contributions

  • Paradigm shift: reformulates cloud fault recovery as neuro-symbolic program synthesis, moving the decision unit from single-step action selection (IFSHM-style hierarchical action space) to multi-step plan generation by composing Recovery Primitives.
  • Verified planning loop: couples an LLM-based Plan Synthesis Engine (PSE) with a Neural-Symbolic World Model (NSWM) that performs rollout simulation to score plan feasibility before any real execution, screening risky/hallucinated plans.
  • DRL Meta-Prompt Optimizer (MPO): a Soft Actor-Critic agent learns a continuous prompt embedding that conditions the PSE, enabling online adaptation to unseen fault types without full LLM fine-tuning.
  • Real-world evaluation on the OpenStack Failure-Dataset shows >40% MTTR reduction over the IFSHM LLM+DRL baseline and rapid adaptation to a novel hybrid fault.

Methodology

PASE replaces the sequential "perception-then-action" pipeline with a tight reason-plan-verify-adapt loop:

  1. Unified Semantic Observation & Plan Synthesis (PSE). Raw heterogeneous observation $O_t=\{L_t, M_t, A_t\}$ (log snippets, metric time series, alarm vectors) is first mapped by an LLM with a fixed template prompt $P_{desc}$ into a structured semantic scene description $D_t$ (a natural-language paragraph of symptoms, affected components, topology). The Plan Synthesis Engine (PSE) — the LLM acting as planner — then generates a recovery plan $\Pi_t = \mathrm{PSE}(D_t, \mathcal{R}; P_t^{plan})$ from a library of Recovery Primitives $\mathcal{R}$, each a parameterizable atomic action with a semantic signature (e.g., Restart(service_id), ScaleOut(resource_type, count), Reroute(traffic, from, to)). Output is a structured list/graph of instantiated primitives, bypassing the rigid hierarchical action space.
  2. Neural-Symbolic World Model (NSWM) verification. The NSWM learns the stochastic dynamics of the cloud system at the primitive level: given latent state $s_t$ and a primitive $r_\theta$, it predicts $(\hat s_{t+1}, \Delta \hat H) = \mathrm{NSWM}(s_t, r_\theta; \phi)$. For a candidate plan $\Pi_t=(r^{(1)},\dots,r^{(K)})$ it performs a rollout and computes a Plan Feasibility Score $F(\Pi_t) = -\mathbb{E}_{rollout}[\lambda_1 \cdot \mathrm{Risk}(\hat s_{t+K}) + \lambda_2 \cdot \mathrm{Severity}(\Delta \hat H_{t+K})]$. The plan is executed only if $F(\Pi_t) > \tau$.
  3. Meta-Prompt Optimization via RL (MPO). Instead of a fixed prompt or full LLM fine-tuning, a Soft Actor-Critic agent learns a continuous prompt embedding $p_t = \mathrm{MPO}_\psi(D_t)$ injected into the PSE forward pass. The composite reward is $R_t = \alpha_1 E_t^{rec} + \alpha_2 E_t^{res} + \alpha_3 \mathbb{I}[F(\Pi_t) > \tau]$, combining recovery efficiency (inverse time-to-restore), resource efficiency, and a shaped term for the NSWM feasibility score.
  4. Three-phase training: (a) NSWM pre-training on historical interaction logs; (b) MPO warm-up in simulation with a fixed base prompt and NSWM feasibility as part of the reward; (c) joint fine-tuning of the full loop (MPO→PSE→NSWM verification→execution) on real interactions.

Implementation. PSE = LLaMA-2-7B fine-tuned with LoRA; NSWM = 3-layer MLP pre-trained on historical fault trajectories; MPO = SAC; A100 GPU cluster; each configuration repeated 5×. Dataset = Failure-Dataset-OpenStack (instance/network/storage faults + dependency info). Baselines: IFSHM (two-stage LLM+DRL, the core comparison), Deformable DETR-FD & GCN-FR (time-series anomaly detection + dependency-graph reasoning), Rule-Based Expert System.

Results & Highlights

  • Fault detection & recovery (Table 2, OpenStack dataset): PASE achieves F1 0.94 ± 0.02 and MTTR 72 ± 8 s, vs IFSHM 0.92 / 85 s, Deformable DETR-FD 0.88 / 105 s, GCN-FR 0.85 / 115 s, Rule-Based 0.75 / N/A — i.e. >40% MTTR reduction over IFSHM (72 vs 85 s) and the highest detection accuracy.
  • Plan quality: PASE plans average 3.2 steps (vs IFSHM's single action or two-level action-parameter structure); the NSWM feasibility score shows 92% correlation with real-environment plan success, validating the simulation-based verification.
  • Rapid adaptation to a novel hybrid CPU-Memory deadlock fault (injected mid-experiment): PASE (via MPO prompt adjustment) raises recovery success from 40% → >80% within ~15 episodes, vs IFSHM's slower full prompt-tuning pipeline converging at 75%; the static rule-based system failed completely.
  • Ablation (Table 1): removing the NSWM drops Safety Score to 0.65 (vs full 0.89) and inflates MTTR to 97.8 s; removing the MPO raises Adaptation Steps to 38 (vs full 15); Direct Execution (no verify) cuts Safety to 0.58; Fixed Prompt Template raises Adaptation Steps to 65.
  • World-model architecture ablation (Fig. 6): the neural-symbolic NSWM achieves the best Plan Feasibility AUC 0.92 and prediction MSE 0.014 at balanced cost (42 ms inference, 128 MB) — vs pure symbolic (AUC 0.76, MSE 0.145), pure MLP (AUC 0.81, MSE 0.032, safety violations), LSTM/Transformer (AUC 0.88–0.90, 58–112 ms), and ensemble (AUC 0.91 but 156 ms/432 MB, impractical for real-time).
  • MPO strategy ablation (Fig. 3): SAC-based MPO yields the best final reward (8.72) and fastest convergence (15 episodes, 12.3% generalization gap) vs REINFORCE (7.68, high variance), PPO (8.15, 20 episodes), gradient (7.89, 25), evolutionary (high cost, 18.9% gap), random search (6.23, 65), and direct LLM fine-tuning (8.41 but 50 episodes, impractically slow/costly).

Critical Assessment

  • The strongest baseline (IFSHM) is a two-stage LLM+DRL framework, and PASE's gain over it (F1 0.94 vs 0.92; MTTR 72 vs 85 s) is real but modest on detection accuracy — the headline ">40%" win is on MTTR specifically, driven by multi-step planning + pre-execution verification, not by a large detection-accuracy jump.
  • The 92% correlation between NSWM feasibility score and real success is self-reported on one dataset; the NSWM is a 3-layer MLP trained on historical trajectories, so its fidelity on genuinely novel fault types (where it has seen no rollout data) is the linchpin of the safety claim and is not stress-tested beyond the single injected hybrid CPU-Memory deadlock.
  • Computational overhead is honestly acknowledged: PASE incurs slightly higher overhead than IFSHM due to multiple NSWM simulation rollouts; the authors flag lightweight world models and distillation as future work. The NSWM's 42 ms/128 MB is acceptable, but plan-graph rollouts scale with plan length K, and K is not bounded in the paper.
  • The novel-fault experiment is a single fault type (hybrid CPU-Memory deadlock); generalization of the MPO's rapid adaptation (40%→80% in 15 episodes) across a broader unseen-fault distribution is not established.
  • No code release is mentioned, and the dataset (Failure-Dataset-OpenStack) is public but the exact train/val/test split and primitive library $\mathcal{R}$ are not fully specified, limiting exact reproducibility.

Strengths

  • Principled neuro-symbolic reframing: LLM as an active planner (not a passive state encoder) + symbolic verification before execution is a clean architectural improvement over loosely-coupled LLM+DRL hybrids.
  • The SAC-based meta-prompt optimizer elegantly sidesteps full LLM fine-tuning for each new fault, adapting in a continuous prompt-embedding space.
  • Comprehensive ablations isolate each component (PSE, NSWM, MPO) and each architectural choice (world-model type, MPO algorithm), strengthening causal attribution of the gains.
  • Real dataset, 5-repeat stats, and an explicit novel-fault injection experiment demonstrate practical adaptability.

Limitations

  • Single evaluation dataset (OpenStack); cross-dataset/cross-platform generalization unverified.
  • NSWM fidelity on truly novel faults (where historical rollout data is absent) is the weakest link in the safety argument and is only lightly tested.
  • Plan length K and rollout cost are not bounded/analyzed; overhead vs. IFSHM is higher.
  • No code release; primitive library and splits not fully specified.
  • Detection-accuracy gain over IFSHM is small (0.94 vs 0.92); the headline win is MTTR-bound.
