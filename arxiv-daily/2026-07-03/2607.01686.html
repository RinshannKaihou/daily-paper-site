WARP: Weight-Space Analysis for Recovering Training Data Portfolios

arXiv: 2607.01686

Date: 2026-07-03

https://arxiv.org/abs/2607.01686

一句话总结： WARP 通过模型合并（SLERP/LERP/TIES）在 base 与 fine-tuned 权重之间插值生成"伪检查点"模拟缺失的训练轨迹，再用 Mimic Score 提取各领域在权重空间中的几何足迹，从而仅从已发布权重反推微调数据的领域混合比例；在 BERT/GPT-2 上平均 MAE 低至 0.046/0.104，优于样本级成员推断与拥有真实轨迹的 oracle 变体。
TL;DR: WARP recovers a fine-tuned model's domain mixture from its released weights alone: it interpolates between the base and fine-tuned models via model merging (SLERP/LERP/TIES) to generate pseudo-checkpoints approximating the missing training trajectory, computes Mimic-Score geometric footprints per domain, and maps them to mixture proportions via either a parameter-free softmax readout or a 2-layer MLP trained on synthetic mixtures. On BERT-base across SNLI/AGNews/Yelp/Yahoo (40 trials each), the supervised variant attains avg MAE as low as 0.046 (TIES) / 0.048 (LERP), beating Sample-level MI (0.064), Centroid Guess (0.149), and Random Guess (0.192) — and even beating the oracle variant that uses the real checkpoint trajectory (0.080). On GPT-2-Small, avg MAE reaches 0.104 (LERP) vs MI 0.130.

核心贡献

  1. 逆向问题形式化：把"从已发布权重反推微调数据领域混合 π⋆"形式化为权重空间几何的逆问题，仅需 base 模型、fine-tuned 模型与一个小型 probing 数据集，无需真实训练轨迹或中间检查点。
  2. 用模型合并模拟训练轨迹：通过 SLERP/LERP/TIES 在两端点之间插值生成 T=15 个伪检查点，替代缺失的中间路径，暴露训练数据在权重空间中的几何足迹。
  3. 几何足迹 → 混合比例：基于 Mimic Score（样本负梯度到 θref 方向的投影）构造领域×时间步的特征矩阵 M，再用无参数 softmax 或监督式 2 层 MLP 映射到混合比例。
  4. 实证优于 MI 与 oracle：BERT 平均 MAE 低至 0.046，比样本级 MI（0.064）降低 ~35%，甚至优于使用真实轨迹的 oracle 变体（0.080）；对早停/收敛/过训练检查点均鲁棒。

Key Contributions

  • Formalizes the inverse data-mixture problem: given only θ_base, θ_ref, and a small probing set D_probe covering K domains, estimate the unknown mixture π⋆ ∈ Δ^(K−1) — no access to the true fine-tuning data or intermediate checkpoints.
  • Simulates the missing training trajectory via model merging: constructs T=15 pseudo-checkpoints $\tilde\theta_t = (1-\alpha_t)\theta_{base} + \alpha_t\theta_{ref}$ (LERP), or via SLERP / TIES merging operators, standing in for the unobserved parameter evolution.
  • Distills a geometric footprint: adapts the Mimic Score to compute, at each pseudo-checkpoint, the projection of each probing sample's negative gradient onto the direction $v_t = \theta_{ref} - \tilde\theta_t$ toward the fine-tuned model; pools per-domain to build a feature matrix $M \in \mathbb{R}^{K\times T}$.
  • Two geometry→mixture mappers: (i) a parameter-free softmax readout over time-averaged scores (unsupervised, label-free); (ii) a supervised 2-layer MLP $f_\phi: \mathbb{R}^{K\times T} \to \Delta^{K-1}$ trained on synthetic (mixture, footprint) pairs generated from short cheap fine-tunes.
  • Beats MI and even the real-trajectory oracle: supervised LERP/TIES attain avg MAE 0.046–0.048 on BERT vs 0.080 for the oracle, because the smooth monotone merged path is less noisy than real stochastic-mini-batch trajectories.

Methodology

Setup. θ_base = pretrained base (BERT-base / GPT-2-Small); θ_ref = fine-tuned model from data drawn under unknown mixture π⋆ over K domains. Practitioner can sample a small probing set $D_{probe} = \bigcup_{k=1}^K D_k$ (|D_probe|=2,500, uniform per domain) from the same source S — D_probe need not follow π⋆, only cover the K domains. Goal: estimate π⋆.

Step 1 — Simulated training trajectory (§3.1). Since the true path θ_base → θ_ref is unreleased, interpolate T=15 pseudo-checkpoints $\tilde\theta_t = \mathcal{M}(\theta_{base}, \theta_{ref}; t)$ where $\mathcal{M}$ is a merging operator: LERP $\tilde\theta_t=(1-\alpha_t)\theta_{base}+\alpha_t\theta_{ref}$, SLERP (spherical), or TIES. α_t plays the role of normalized training progress.

Step 2 — Geometric footprint (§3.2). Define the time-varying direction $v_t = \theta_{ref} - \tilde\theta_t$. For each probing sample, the Mimic Score is the projection of its negative gradient onto the unit vector along $v_t$:

$$m_{i,t} = \frac{\langle -g_i(\tilde\theta_t), v_t\rangle}{\|v_t\|}.$$

Raw scores are min-max normalized within each step (since $\|v_t\|$ shrinks as α_t→1), then pooled per-domain: $\bar m_{k,t} = \frac{1}{|D_k|}\sum_{i\in D_k} m_{i,t}$. Stacking yields the footprint matrix $M \in \mathbb{R}^{K\times T}$.

Step 3 — Geometry → mixture (§3.3). Two variants:

  • Unsupervised readout: $\hat\pi_k = \frac{\exp(\dot m_k/\tau)}{\sum_{k'}\exp(\dot m_{k'}/\tau)}$ where $\dot m_k = \frac{1}{T}\sum_t \bar m_{k,t}$. Fast, label-free, but collapses the temporal dimension.
  • Supervised 2-layer MLP: draw J random mixtures π^(j) (uniform in $\ell_1$ from uniform), fine-tune the base model on each to get synthetic reference models, compute their footprints M^(j), and train $f_\phi: \mathbb{R}^{K\times T}\to\Delta^{K-1}$ with a softmax output and regression loss against π^(j). Synthetic fine-tunes are short/cheap since they only generate projector training signal.

Datasets. SNLI (3 classes), AGNews (4), Yelp (5), Yahoo (10); each class treated as a domain. 40 ground-truth mixtures π⋆ per dataset; 5,000-sample fine-tune set drawn per π⋆; BERT-base and GPT-2-Small. Metric: MAE between $\hat\pi$ and π⋆ averaged over 40 trials. Baselines: Random Guess (sample from simplex), Centroid Guess (uniform 1/K), Sample-level MI (loss-reduction threshold sweep, aggregated to per-domain rates), and an oracle WARP variant using real intermediate checkpoints.

Results & Highlights

  • BERT-base recovery (Table 1, avg MAE over 40 trials): supervised LERP/TIES reach 0.048 / 0.046, vs Sample-level MI 0.064, Centroid Guess 0.149, Random Guess 0.192 — a ~35% error reduction over the strongest baseline (MI), and lower than the real-trajectory oracle (0.080).
  • GPT-2-Small recovery: supervised LERP attains avg MAE 0.104 (vs MI 0.130, oracle 0.138) — a ~30% reduction over MI; the merged-path variant again beats the real-checkpoint oracle.
  • Unsupervised softmax readout is competitive even with no learned projector: on BERT it already beats all baselines (e.g., SNLI LERP 0.091 vs MI 0.074 is close; AGNews LERP 0.118 vs MI 0.084) and closes most of the gap to the supervised variant — useful when collecting synthetic training pairs is impractical.
  • Why merged beats real: real fine-tuning paths are noisy (stochastic mini-batches, LR warmup, curriculum effects); LERP/TIES create a clean monotone path yielding a smoother footprint matrix M from which the MLP extracts the mixture more easily.
  • Training-recipe robustness (Table 2, GPT-2-Small on AGNews, TIES merging): WARP remains accurate across short (9 epoch / early-stop), converged (12 epoch), and overtrained (18 epoch, past convergence) checkpoints — mirroring the post-training regime where released models are routinely pushed beyond compute-optimal budgets.

Critical Assessment

  • The strongest baseline (Sample-level MI) is already a fairly weak comparator for domain-mixture recovery: MI is fundamentally sample-level, so aggregating its detections into proportions is a known mismatch. The headline 35%/30% reductions would be more convincing against a purpose-built mixture-recovery baseline (e.g., a proxy-model regression from the data-mixture-optimization literature, which the paper cites but does not benchmark against).
  • Scale is the biggest caveat: experiments are limited to BERT-base and GPT-2-Small on 4 text-classification datasets with 3–10 classes. Whether the Mimic-Score footprint remains informative on billion-parameter decoder LLMs, on continuous (non-class) domain structure, or on pre-training (not fine-tuning) mixtures is unverified — and the method's cost grows with the need to compute per-sample gradients at each pseudo-checkpoint.
  • The "merged beats real oracle" result, while honestly reported, is somewhat suspicious: it suggests the real-trajectory oracle is implemented sub-optimally (e.g., using all noisy real checkpoints rather than a smoothed subset), and the paper does not ablate checkpoint spacing/count for the oracle. The win may reflect oracle implementation, not a fundamental superiority of merged paths.
  • The supervised MLP requires generating J synthetic mixtures with short fine-tunes — acceptable on BERT/GPT-2 but a non-trivial cost at scale; the paper acknowledges synthetic fine-tunes are "cheap, short" but does not quantify J or wall-clock.
  • Probing-set sensitivity: D_probe must cover all K domains but the method's sensitivity to |D_probe|, domain coverage gaps, and probing-set distribution shift is not analyzed.

Strengths

  • Clean reframing of data-mixture recovery as a weight-space inverse problem using only released artifacts — a practical, auditing-relevant setup.
  • Model merging as a trajectory simulator is an elegant, checkpoint-free way to expose geometric footprints.
  • Two mappers (unsupervised softmax + supervised MLP) give a useful cost/accuracy trade-off.
  • Honest, well-instrumented evaluation (40 trials, 4 datasets, 2 architectures, oracle comparison, training-recipe robustness).

Limitations

  • Small-model scale only (BERT-base, GPT-2-Small); frontier-LLM and pre-training-mixture applicability unproven.
  • Baselines omit purpose-built forward-direction mixture-regression methods; MI is an asymmetric comparator.
  • Per-sample-gradient computation at each of T pseudo-checkpoints is costly at scale; J and wall-clock for synthetic-pair generation unreported.
  • "Merged beats real oracle" may stem from oracle implementation rather than a fundamental property; not ablated.
  • Probing-set size/coverage sensitivity not analyzed; behavior under partial domain coverage unknown.

Questions for Further Exploration

  1. Mimic-Score 足迹在数十亿参数 decoder LLM（如 7B/70B）上是否仍具判别力？每样本梯度在 T=15 个伪检查点上的计算成本能否用 LoRA/低秩近似分摊？
  2. 若与正向 mixture-regression 基线（如 DoReMi/R&B 的梯度对齐、proxy-model 回归）在相同合成设置下对比，WARP 的优势是否仍然成立？
  3. "合并轨迹优于真实轨迹"是否主要源于 oracle 使用了全部噪声检查点？若对真实轨迹做平滑/稀疏化子采样，oracle 是否能反超？
  4. 方法能否扩展到预训练（非微调）混合比例恢复——此时 θ_base 不存在单一明确对应物，需用 random-init 作 base 吗？
  5. 监督 MLP 所需的合成对数量 J 与目标精度的关系如何？能否用主动采样减少 J？
  6. probing 集在领域覆盖不完整或存在分布偏移时，恢复误差如何退化？
  7. 该足迹信号能否用于检测数据污染（论文 Related Work 提及的 auditing 应用）而非仅恢复混合比例？
