DISA decouples partition-function estimation from policy learning via offline importance sampling — freezing Z before RL starts.

Shaobo Wang, Yujie Chen et al. SJTU · HKUST(GZ) · Qwen Team / Alibaba 2026-05-17 reinforcement-learning distribution-matching diversity-preservation
Key Finding
The partition function in distribution-matching RL is policy-independent — estimate it offline and freeze it.
Eliminates the error coupling that plagues FlowRL's online co-training.
Key Metric
65.5 Mean@8
Qwen3-4B-Base math avg. +16.1 over GRPO, +0.7 over FlowRL. Diversity 3.72/5.0 (95% retained).
Why This Matters
Fixes a fundamental design flaw in GFlowNet-based LLM post-training.
Provides separable quality diagnostics (Val MSE) that online-coupled methods entirely lack.
Critical Edge
235B proposal or bust.
Drops from 65.5 → 37.1 with a 4B proposal. IS stage is expensive: N=8 samples per prompt from a 235B model.

Methodology — Three-Stage Pipeline

Stage 1
Offline IS
Draw N=8 from 235B proposal; logsumexp → log Z(q)
→
Stage 2
MLP Regressor
Fit & freeze g_ψ by least-squares; Val MSE is diagnostic
→
Stage 3
Anchored RL
Stop-gradient on ψ; π_θ optimizes with frozen Z

The per-prompt partition function Z(q) = Σ π_ref·exp(βr̃) depends only on the reference policy, reward transform, and temperature — never on π_θ. DISA exploits this to sever the gradient path entirely.

Theoretical Guarantees

  • Prop 3: Exact Z → preserves the reward-tilted Boltzmann optimum π*.
  • Prop 5: Prompt-only bias → preserves on-policy stationarity.
  • Prop 4: Residual Z error bounded by Cauchy-Schwarz envelope √(σ²_g + 2σ_g·L_TB).

Strengths & Limitations

Strengths

Clean theoretical decoupling Separable Val MSE diagnostic +16.1 over GRPO (4B) 95% diversity retained +13.8 over SFT on same data 9 benchmarks, 2 backbones

Limitations

Needs 235B proposal model 65.5 → 37.1 with 4B proposal IS stage compute-heavy Code benchmarks less differentiated Diversity eval: only 60 AIME problems Finite-batch reward norm gap unquantified

Open Questions

  1. Can DISA work without a strong proposal — e.g. iterative self-distillation as policy improves?
  2. How does IS-label quality degrade as the proposal-policy gap narrows at 32B/72B scale?
  3. Compute tradeoff: DISA's one-shot 235B sampling vs. FlowRL's sustained online Z-training steps?
  4. Can the frozen regressor g_ψ* transfer across runs or backbones to amortize offline cost?
  5. How does DISA interact with process-level (per-step) reward models instead of outcome rewards?
