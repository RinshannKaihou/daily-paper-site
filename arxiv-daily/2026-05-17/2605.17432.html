DP-SelFT: Differentially Private Selective Fine-Tuning for LLMs

Haichao Sha, Zihao Wang, Yuncheng Wu, Hong Chen, Wei Dong | Renmin Univ. of China & NTU Singapore | arXiv 2605.17432

By selecting which layers to update under DP constraints via worst-case perturbation on synthetic data — rather than updating all parameters — DP-SelFT consistently outperforms full-parameter DP fine-tuning and DP-LoRA across models and privacy budgets.

Key Metric

Why This Matters
The privacy-utility gap in DP LLM fine-tuning is a major practical bottleneck. DP-SelFT is orthogonal to LoRA and composable with it, opening a new axis of improvement.
Critical Edge
Not tested on 7B+ models. Synthetic data depends on commercial API access. Worst-case perturbation computation may scale poorly to very large layer subsets.

Methodology — Three-Stage Pipeline

STAGE 1
Synthetic Data Generation
A remote API generates unlabeled candidates. Private data votes for nearest candidates via DP histogram; labels assigned by noisy majority. Costs only εsyn = 0.3.
STAGE 2
Layer Selection
For each candidate layer subset, train on synthetic data under worst-case perturbation (adversarial noise that maximizes loss). Pick the subset with best validation performance.
STAGE 3
DP Fine-Tuning
Fine-tune on private data with DP-AdamW, updating only selected layers. Compatible with LoRA for further gains (DP-SelFT+LoRA).

核心贡献 / Key Contributions

  • 首个将选择式参数高效微调引入 DP LLM 微调的框架 — 从"在哪里更新"而非"如何参数化更新"的角度改进隐私-效用权衡
  • Lightweight DP synthetic data construction decouples selection from private data — no additional privacy cost during selection
  • Worst-case perturbation criterion favors subsets robust to adversarially noisy updates, generalizing better to stochastic DP noise
  • Theorem 1: selective fine-tuning trades learning signal for reduced noise damage; Theorem 2: worst-case selection generalizes to unseen DP noise with provable bounds

Strengths & Limitations

Strengths

  • Clean three-stage pipeline with clear motivation
  • Comprehensive experiments: 4 models, 4 tasks, 3 privacy budgets
  • Orthogonal and composable with LoRA

Limitations

  • Not evaluated on 7B+ decoder models (LLaMA scale)
  • Relies on commercial LLM API for synthetic data
  • Layer-level granularity only; finer-grained selection unexplored
  • Only classification tasks (GLUE); generation tasks unknown

Relevance / 研究相关性

DP-SelFT 从"选择更新位置"这一新颖角度改进 DP 微调的隐私-效用权衡，对关注隐私保护 LLM 适配的研究者具有重要参考价值。其理论分析框架也为理解 PEFT + DP 的基本原理提供了新视角。

Open Questions

  1. How does DP-SelFT scale to 7B–70B parameter models? Can worst-case perturbation be efficiently approximated?
  2. Can selection granularity be adaptive — module-level for some layers, layer-level for others?
  3. Effectiveness on generation tasks (summarization, dialogue) vs. classification?
  4. Is synthetic data necessary when high-quality public domain datasets already exist?
  5. Can worst-case perturbation insights transfer to DP pre-training from scratch?
