Event-Grounded Sparse Autoencoders for Vision-Language-Action Policies

Xinchen Jin · Aditya Chatterjee · Pranav Kumar · Rohan Paleja — Purdue University

Standard SAE tools don't transfer to VLA robot policies because outputs are actions, not tokens. By anchoring SAE features to kinematic behavioral events, event-aligned ranking achieves a −21.2 pp success-rate drop on OpenVLA—16× stronger than random control—while revealing that π0.5's action expert collapses under nearly any single-feature edit.

Key Finding
Temporal shape > magnitude
Event-aligned SAE ranking selects qualitatively different features than activation-magnitude baselines
Key Metric
ΔSR = −21.2 pp
OpenVLA layer 31, event-aligned ranking vs −1.3 pp random-alive control
Why This Matters
Bridge to action spaces
First automated pipeline linking SAE features to closed-loop robot behavior without per-feature human labeling
Critical Edge
Action expert = switch, not dial
π0.5 AE collapses under any single-feature edit; PaliGemma backbone is near-immune—intervention is architecture-bound
Methodology — 5-Stage Pipeline
1 BatchTopK SAE→ 2 AWE Keyframes→ 3 Event Clustering→ 4 Feature Ranking→ 5 Closed-Loop Edit
  • SAE Training: BatchTopK (k=64) on residual-stream activations per (stream, suite, layer). Hooked SR selects intervention layer — OpenVLA L31 retains 34.8% hooked SR.
  • AWE Keyframes: Automatic Waypoint Extraction sparsifies end-effector trajectories under η=0.05 error budget. Keyframes are SAE-independent by construction.
  • Event Clustering: Agglomerative clustering on visual (λv=1.0) + state (λs=0.5) + temporal (λp=0.4) descriptors. Only clusters recurring in ≥50% episodes kept. Optional VLM (Gemini 3.1 Pro) semantic annotation.
  • Feature Ranking: Four strategies — event-aligned (temporal shape), window-mean (avg near events), task-mean (global avg), random-alive (control).
  • Residual-Preserving Edit: x' = x + Dec(z') − Dec(z). Modifies only SAE-explained component, preserves reconstruction error.
Results & Highlights

OpenVLA (discrete tokens)

−21.2 pp

Event-aligned ranking, layer 31. Top 3 features carry nearly all the SR drop. ~5% overlap with magnitude rankings → temporal shape selects distinct features.

π0.5 PaliGemma Backbone

±3 pp

Single-feature edits barely move SR. Cross-attention KV-cache dilutes edits before reaching actions. Joint 16-feature dropout at layer 0: −89.8 pp on LIBERO-Goal.

π0.5 Action Expert

~0% SR

Nearly any single-feature zero-out collapses SR — including random-alive. Steep monotonic transition: behaves like a switch, not a dial. Not amenable to fine-grained control.

Real Robot (Mobile ALOHA)

Measurable

SAE suppression at AE layer 17 increases distance from target chip cluster. Conservative α required — aggressive edits induce large unsafe motions.

Strengths & Limitations

✓ Strengths

  • Fully automated — scales beyond per-feature manual labeling
  • SAE-independent behavioral signals as event anchors
  • Cross-architecture evaluation revealing qualitatively different intervention regimes
  • Honest negative results (PG insensitivity, AE collapse, shared primitives)
  • Residual-preserving edit avoids hidden-state overwrite
  • Real-robot validation with explicit safety discussion
  • Open-source code: github.com/xc-j/Event-SAE

✗ Limitations

  • Binary SR is too coarse for fine-grained causal attribution
  • Event clustering hyperparameters hand-tuned, no sensitivity analysis
  • Top features are shared primitives, not task-specific circuits
  • VLM annotations used only for visualization, not ranking
  • π0.5 AE collapse limits practical utility for this major architecture
  • Only LIBERO tabletop manipulation tested
  • Aggressive interventions pose real safety risks on hardware
Questions for Further Exploration
  1. Can VLM-derived semantic labels be integrated into the ranking objective for features that are both causally important and semantically interpretable?
  2. What finer-grained metrics (trajectory deviation, contact timing, gripper force) could replace binary SR for nuanced causal attribution?
  3. How does the pipeline generalize to long-horizon tasks or navigation where end-effector keyframes aren't the primary signal?
  4. Can event-grounded features be used for training-time regularization or data augmentation, not just post-hoc analysis?
  5. What alternative sparse decompositions (variational, structured) might overcome SAE limitations for the action expert?
  6. How can safe intervention protocols be formalized — e.g., certified bounds on end-effector deviation under feature perturbation?
