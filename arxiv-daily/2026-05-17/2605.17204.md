> **一句话总结：** 本文提出一种基于行为事件锚定的SAE可解释性流水线，通过AWE提取运动学关键帧并聚类为行为事件，将SAE特征与闭环机器人行为关联，在OpenVLA上事件对齐排名的因果干预效果最强（ΔSR=-21.2pp），在π₀.₅上揭示了PaliGemma骨干网单特征干预无效而动作专家几乎完全崩溃的架构差异，并在真实机器人上验证了安全约束下的保守干预可行性。

> **TL;DR:** Standard SAE interpretability tools don't transfer to VLA robot policies because outputs are actions (not tokens) and evaluation requires expensive closed-loop rollouts. This paper proposes a 4-stage pipeline: (1) train BatchTopK SAEs on VLA residual streams, (2) extract kinematic keyframes via Automatic Waypoint Extraction, (3) cluster keyframes into task-local events using visual+state+temporal descriptors with optional VLM annotation, and (4) rank features by event-aligned temporal shape and validate through residual-preserving latent edits. On OpenVLA, event-aligned ranking yields the strongest causal intervention (-21.2pp SR drop vs -1.3pp for random-alive control). On π₀.₅, the PaliGemma backbone barely responds to single-feature edits while the action expert collapses under nearly any ranking. Real-robot experiments on Mobile ALOHA confirm measurable steering effects but highlight safety limits of aggressive intervention.

## 核心贡献

- 提出首个将SAE特征分析与VLA闭环行为事件锚定的全自动可解释性流水线，无需逐特征人工标注
- 利用AWE提取末端执行器关键帧，通过视觉+状态+时间多维描述符聚类为行为事件，并可选地用VLM进行语义标注
- 揭示了关键的架构依赖性：OpenVLA上事件对齐排名效果最强（ΔSR=-21.2pp），π₀.₅ PaliGemma骨干网对单特征编辑几乎免疫，而动作专家几乎任何编辑都会崩溃
- 设计了残差保留的隐空间编辑方法（x' = x + Dec(z') - Dec(z)），仅修改SAE可解释的隐状态分量，保留重构误差
- 在真实Mobile ALOHA机器人上验证了保守干预的可控转向效果，并展示了激进一步干预的安全隐患

## Key Contributions

- First fully automated, event-grounded SAE pipeline for VLA policies that scores every alive SAE feature against SAE-independent behavioral events without per-feature manual labeling
- Kinematic keyframe extraction via AWE and task-local event clustering using concatenated visual embeddings, robot state, and temporal progress descriptors, with optional VLM semantic annotation
- Discovery of architecture- and site-dependent intervention behavior: localized event-aligned effects on OpenVLA (layer 31, -21.2pp SR drop), weak single-feature leverage in π₀.₅ PaliGemma backbone, and broad causal sensitivity in the action expert
- Residual-preserving latent edit (x' = x + Dec(z') - Dec(z)) that modifies only the SAE-explained component while preserving reconstruction error, avoiding the hidden-state overwrite problem of fixed decoder-vector additions
- Real-robot demonstration on Mobile ALOHA showing that SAE suppression at action-expert layers measurably alters color-conditioned approach behavior, with safety-limited conservative interventions

## Methodology

The pipeline consists of five stages operating on closed-loop VLA rollouts:

**SAE Training (Section 3.1).** BatchTopK SAEs are trained per (architecture stream, LIBERO suite, layer) on post-block residual-stream activations. The BatchTopK operator retains only the top-kB pre-activations (k=64, B=batch size). SAE quality is assessed via offline metrics (FVE, alive fraction, L0) and a behavioral fidelity metric called Hooked SR — the closed-loop success rate when the hidden state is replaced with its SAE reconstruction. Hooked SR selects the intervention layer; OpenVLA layer 31 retains 34.8% hooked SR (vs ~70% baseline), while π₀.₅ sustains ≥95% at every probed layer.

**Kinematic Keyframe Extraction (Section 3.2).** Automatic Waypoint Extraction (AWE) compresses end-effector position trajectories into sparse ordered waypoint subsequences under a reconstruction-error budget η=0.05. Each waypoint becomes a kinematic keyframe with an associated observation bundle (5-frame image strip, short video, metadata). AWE depends only on end-effector motion, making keyframes independent of SAE activations.

**Event Clustering (Section 3.3).** Keyframe bundles are grouped into task-local event clusters using agglomerative clustering on concatenated descriptors: visual embedding (λ_v=1.0), robot state with gripper status (λ_s=0.5), and temporal progress (λ_p=0.4). Only clusters recurring in ≥50% of a task's episodes are retained (e.g., 36–61 recurring clusters per LIBERO suite). VLM annotation (Gemini 3.1 Pro) assigns a descriptive phrase and phase label (pre_grasp, immobilization, contact, detach, post_grasp, transition) to each cluster.

**Feature Ranking (Section 3.4).** Four strategies score SAE features:
- *Event-aligned*: measures how well a feature's activation trace matches pulse/step-up/step-down templates around events, scoring temporal shape rather than magnitude
- *Window-mean*: average activation near events, ignoring temporal structure
- *Task-mean*: average activation across all timesteps
- *Random-alive*: uniform sample from alive features (exclusion-based control)

**Closed-Loop Intervention (Section 3.5).** The residual-preserving latent edit scales targeted SAE features by α (0=zero-out, 1=identity) and applies only the induced change: x' = x + Dec(z') - Dec(z). This preserves the reconstruction error and avoids overwriting the hidden state.

## Results & Highlights

**OpenVLA (discrete action tokens):**
- Event-aligned ranking produces -21.2pp SR drop at layer 31, well separated from window-mean (-6.2pp), task-mean (-6.5pp), and random-alive (-1.3pp)
- Top 3 features carry nearly all of the SR drop; event-aligned selects largely distinct features from magnitude-based rankings (only ~5% overlap with window-mean/task-mean on OpenVLA)
- Target/off-target probe shows event-selected features are shared manipulation primitives (similar SR drops on both target and off-target tasks), not task-specific circuits

**π₀.₅ PaliGemma backbone (indirect action influence):**
- Single-feature edits leave SR within ~3pp of baseline across all layers and rankings
- Joint dropout of 16 random features: layer 0 causes major disruption (-89.8pp on LIBERO-Goal, which requires instruction grounding), deeper layers barely respond
- The cross-attention KV-cache dilutes single-feature edits before they reach the action output

**π₀.₅ Action Expert (direct action generation):**
- Nearly any single-feature zero-out collapses SR to ~0% across all layers and rankings, including random-alive
- Soft intervention (α sweep) shows monotonic but steep SR transition — behaves more like a switch than a dial
- This broad sensitivity means the action expert is not amenable to fine-grained feature-level control

**Real-robot (Mobile ALOHA, chip-approach task):**
- SAE suppression at action-expert layer 17 measurably increases distance from the prompted chip cluster as suppression strengthens
- Earlier layers show noisier effects; conservative α values are necessary because aggressive edits induce large unsafe robot motions
- FFN value-vector steering baseline is substantially noisier than prompt-only controls, confirming that lexical-space interventions are an indirect proxy for behavioral control

**Cross-cutting findings:**
- Window-mean and task-mean rankings select nearly identical features (>93% overlap on π₀.₅), suggesting high activation near events correlates with high activation globally
- Event-aligned ranking selects qualitatively different features, demonstrating that temporal change matters beyond baseline magnitude
- Neither FVE nor alive fraction alone predicts Hooked SR; behavioral fidelity is the essential metric

## Authors & Credibility

All authors are from the Department of Computer Science at Purdue University. The work builds on established VLA models (OpenVLA, π₀.₅), standard interpretability tools (SAEs from Anthropic's line of work), and established benchmarks (LIBERO suites). Code is publicly available at https://github.com/xc-j/Event-SAE. The paper provides extensive appendices with full hyperparameters, per-feature breakdowns, clustering statistics, and qualitative failure visualizations. The authors are commendably transparent about limitations and do not overclaim mechanistic understanding.

## Critical Assessment

**Novelty: High.** The event-grounding approach — using SAE-independent kinematic events rather than SAE-internal activation statistics to score features — is genuinely novel for VLA interpretability. The insight that language-model interpretability tools (logit lens, text-context labeling) don't transfer to action spaces is well-motivated and the behavioral-event alternative is elegant.

**Rigor: Good.** The paper evaluates on two architectures (OpenVLA with discrete tokens, π₀.₅ with continuous chunks), four LIBERO suites, multiple layers per stream, four ranking strategies with a proper random-alive control, and real-robot validation. The target/off-target probe (Appendix J) and joint dropout probe (Appendix H) add useful nuance. The sanity checks (Appendix L) comparing hook sites and success-prediction probes strengthen confidence.

**Weaknesses:**
- *Binary SR as the sole evaluation metric*: The authors acknowledge this limitation. Success rate is coarse — it cannot distinguish "slightly worse grasping" from "completely wrong target." Finer-grained behavioral metrics (trajectory deviation, contact timing) would strengthen causal claims.
- *Event clustering hyperparameters are hand-tuned*: The descriptor weights (λ_v=1.0, λ_s=0.5, λ_p=0.4) and cosine threshold (0.18) are calibrated to produce cluster counts matching AWE waypoint counts. No sensitivity analysis is provided.
- *Limited evaluation scope*: Only LIBERO manipulation tasks are tested. Whether the pipeline transfers to navigation, mobile manipulation, or multi-task policies with diverse action spaces is unknown.
- *VLM annotations are underutilized*: The VLM phrase and phase labels serve only as visualization aids, not as part of ranking or intervention. The semantic structure is "largely unexploited" (the authors' own words).
- *Action expert results are largely negative*: The finding that π₀.₅ AE collapses under nearly any intervention — including random-alive features — is informative but limits the practical utility of the pipeline for this architecture. The paper honestly frames this as "broad causal sensitivity rather than event-specific selectivity."

**Reproducibility: Good.** All hyperparameters are provided (Appendix A/B), code is open-source, and the pipeline operates on standard VLA models with standard benchmarks. The A100 compute requirements (few GPU-hours per SAE, 1-2 min per rollout) are accessible.

**Significance: Moderate-to-high.** The paper is more of an empirical characterization than a solution — it reveals where SAE-based intervention works (OpenVLA late layers) and where it doesn't (π₀.₅ AE), but doesn't provide an alternative for the failing cases. The honest negative results are arguably as valuable as the positive ones. The real-robot safety discussion is a responsible contribution.

**Red flags:** None serious. The authors are admirably honest about what SAEs can and cannot do. The paper does not claim a "full mechanistic account" and explicitly positions event-grounded SAE analysis as "a practical starting point."

## Strengths

- Fully automated pipeline that scales beyond per-feature manual labeling
- Principled use of SAE-independent behavioral signals as event anchors
- Thorough cross-architecture evaluation revealing qualitatively different intervention regimes
- Honest reporting of negative results (PG backbone insensitivity, AE broad collapse, shared vs task-specific features)
- Residual-preserving latent edit avoids hidden-state overwrite problem of fixed decoder-vector additions
- Real-robot validation with explicit safety discussion
- Open-source code release

## Limitations

- Binary success rate is too coarse to isolate specific behavioral changes; finer-grained metrics are needed
- Event clustering requires hand-tuned hyperparameters (descriptor weights, cosine threshold) with no sensitivity analysis
- Top-ranked features are shared manipulation primitives, not task-specific — single-feature edits cannot isolate individual behavioral factors
- VLM semantic annotations are used only for visualization, not integrated into ranking or intervention
- π₀.₅ action expert collapses under nearly any intervention, limiting practical utility for this widely-used architecture
- Evaluated only on LIBERO tabletop manipulation; generalization to other robot platforms and task types is untested
- Aggressive interventions pose real safety risks on physical hardware, restricting the range of testable interventions

## Relevance

本文将稀疏自编码器可解释性方法系统性引入VLA机器人策略，提出的运动学事件锚定流水线对理解视觉-语言-动作模型内部表征具有重要价值。特别是揭示了不同VLA架构（离散token vs连续action chunk）在可解释性干预上的本质差异，对具身智能安全性与可控性研究有直接参考意义。

## Questions for Further Exploration

- Can VLM-derived semantic labels be integrated into the ranking objective to select features that are both causally important and semantically interpretable?
- What finer-grained behavioral metrics (trajectory deviation, contact timing, gripper force profiles) could replace binary SR to enable more nuanced causal attribution?
- How does the pipeline generalize to long-horizon, multi-stage tasks or navigation domains where end-effector keyframes are not the primary behavioral signal?
- Can the event-grounded features be used for training-time regularization or data augmentation rather than only post-hoc analysis?
- What alternative sparse decomposition methods (e.g., variational approaches, structured sparsity) might overcome the SAE's limitations as an intervention basis for the action expert?
- How can safe intervention protocols be formalized for physical VLA deployments — e.g., certified bounds on end-effector deviation under feature perturbation?
