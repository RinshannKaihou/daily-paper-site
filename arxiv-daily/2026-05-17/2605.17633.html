SparseSAM: Structured Sparsification of Activations in Segment Anything Models

A training-free framework that cuts SAM's compute in half — 2× speedup, 2.8× less memory — with only 0.004 mIoU loss.

Key Finding Z-order permutation + residual-consistency routing jointly sparsify attention and MLP layers without any retraining.
Key Metric 2.0× / 2.8× Inference speedup & peak-memory reduction at 0.4 density on A100.
Why This Matters Training-free compression makes large ViT segmentation models deployable on resource-constrained hardware.
Critical Edge Custom CUDA kernels limit portability; Sobel heuristic may fail on non-natural images; only validated on A100.

Methodology

Stripe-Sort Attention

A Sobel saliency map ranks tokens by information density. A transversal (Z-order / Morton) scan then interleaves them into groups that each span the full spatial extent. This converts the N×N attention matrix into a static block-sparse pattern — an A-shaped mask (dense global prefix + banded diagonal) compiled into a custom CUDA kernel with zero runtime mask overhead.

Residual-Consistency MLP

The same permutation selects the top-K tokens (keep-set) to pass through the MLP. Remaining tokens bypass via the residual connection + LayerNorm only. Because MLP update magnitude correlates with token dissimilarity (ρ ≈ 0.6–0.7), the inter-region contrast the decoder relies on is preserved.

Shared Permutation

Both components reuse a single precomputed permutation across all layers, keeping per-layer overhead negligible. Kernels use 32×32 tiles (local) and 128×128 tiles (global).

Results & Highlights

MS-COCO zero-shot — SparseSAM @ 50% density: 0.459 mAP vs 0.468 baseline (−0.009); ToMe at same density: 0.306 mAP.
HQ-44K fine-grained — @ 0.3 density: COIFT 0.945 mIoU vs 0.919 (SpargeAttn); ThinObject5K-TE 0.892 vs 0.850.
SAM2 transfer — Hiera-L @ 0.3 density: HRSOD mIoU actually improves 0.909 → 0.925.
Fine-tuning recovery — 15 min MLP-only fine-tune at 25% density raises DIS5K-VD mIoU 74.79 → 76.92 (baseline 78.63), zero added latency.

At density < 0.4, competing methods suffer catastrophic degradation; SparseSAM maintains near-baseline fidelity.

Strengths & Limitations

Strengths

  • Training-free — no retraining, data collection, or distillation.
  • Jointly accelerates attention and MLP; prior work only handled one.
  • Deterministic static patterns avoid dynamic-mask overhead.
  • Clean transferability to SAM2's Hiera encoder demonstrated.

Limitations

  • Custom CUDA kernels — portability to non-NVIDIA hardware unknown.
  • Sobel ranking is image-level, content-agnostic; may fail on medical/satellite imagery.
  • A-shape sparsity mask is a heuristic; optimality not formally proved.
  • Only evaluated on NVIDIA A100; consumer/edge GPUs untested.

Questions for Further Exploration

  • How does SparseSAM perform on non-natural image domains (medical, satellite, depth) where Sobel gradients may be uninformative?
  • Could layer-adaptive permutations (instead of one shared permutation) yield further quality gains at small overhead?
  • What is the latency profile on consumer GPUs (RTX 4090, 3060) or edge accelerators (Jetson, NPU)?
  • Can a learned (but inference-only) permutation outperform the Sobel heuristic?
  • How does SparseSAM interact with quantization (W4A4) — can both stack for compound speedup?
