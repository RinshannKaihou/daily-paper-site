LLM Fine-tuning Performance Prediction Meta-learning LoRA SHAP

Predict the final score of an LLM fine-tune before committing to full training — a 24-dim meta-feature vector of static dataset descriptors plus a 100-step AdamW probe, fed to LightGBM.

TuneAhead: Predicting Fine-tuning Performance Before Full Training Begins

Yuxiang Luo, Haonan Long, Chen Wang, Qiqi Duan, Xiaotian Lin, Yanwei Xu, Yuyu Luo, Weikai Yang, Nan Tang · HKUST (Guangzhou) & Huawei · 2026-06-16

Key metric
1.47pp
RMSE on 370 held-out Qwen2.5-7B LoRA runs. R² = 0.99, Acc@3pp = 95.1% — −30% vs. ProxyLM, −80% vs. Early-Stop Extrapolation.
Why this matters
Pre-hoc screening kills doomed LoRA runs at τ=55%, saving ~58% of fine-tuning compute while retaining 94.5% of successes. The SHAP-driven diagnostic loop turns a 20.2% MMLU failure into 48.7% by prescribing targeted data cleaning and hyperparameter fixes — a prescriptive use of meta-features, not just a predictive one.
Critical edge
Cross-model portability is not zero-shot: Llama-3-8B and Qwen2-0.5B each need a small per-target calibration meta-set, and only LoRA (no full FT / QLoRA) is validated.

Methodology

Problem. Given base LLM M, algorithm A (LoRA), dataset Di and hyperparameters Hj, learn F(Vi,j) ≈ Ri,j without running A.

Two-stage meta-feature construction.

  • Static (14): size, I/O length ratio, TTR, N-gram repetition, MinHash duplicates, 3σ embedding outliers, I/O semantic similarity, output diversity, reference perplexity, KL shift, Answer Groundedness, instruction complexity.
  • Dynamic (10): 100-step AdamW probe → initial loss, log-loss decay slope, loss variance, gradient norm mean/var, Gradient Consistency, Sparsity, Parameter Change Norm, Landscape Flatness.
  • Selection: SHAP global importance (drop <15th pct), directional consistency (drop |ρ|<0.2), redundancy pruning (∆RMSE>0.01). Done on train/val only.

Predictor. LightGBM (LR=0.05, num_leaves=4, fixed) + TreeSHAP for sample-level attribution. Embedding model: all-MiniLM-L6-v2 (22M params).

Strengths & limitations

Strengths

  • 1,300+ runs, 3 seeds each, 3 base models, 3 benchmarks
  • Clean train/val/test discipline for feature selection
  • Static + dynamic channels show complementary failure modes
  • Closed-loop diagnose-then-fix with a documented +28.5 pp rescue
  • 58–86% compute savings at 84–95% success retention
  • Threshold sensitivity table makes deployment trade-offs explicit

Limitations

  • Cross-model transfer needs a per-target calibration set
  • LoRA-only; no full FT / QLoRA / other PEFT variants
  • Narrow hyperparam sweep (3 LR × 3 batch sizes)
  • ±3 pp is the headline; ±1 pp gap to ProxyLM is small (50.0 vs 40.7)
  • Single diagnostic case study (n=1)
  • Dataset-synthesis details appendix-only; code release not stated

Questions for further exploration

  • Does the 100-step probe elbow shift on 13B / 70B models where one forward pass is ~10× more expensive?
  • Can SHAP prescriptions become a systematic intervention policy (e.g., auto-apply SemDeDup when duplicate_rate > 0.6) and be evaluated at population scale?
  • Does the predictor transfer across instruction-tuning, preference-tuning, and code-tuning — where failure modes differ?
  • How sensitive is the predictor to benchmark choice (MMLU vs. GPQA / MATH / HumanEval)?
  • Could a small transformer over the 100-step loss/gradient trajectory replace hand-crafted dynamic features?
  • Is true zero-shot cross-model transfer possible, or is per-model calibration structurally unavoidable?
  • How does TuneAhead compose with existing AutoML tools (Optuna, Ray Tune, Hyperband, successive halving)?
