Human-in-the-Loop Atlas-Based 3D Asset Segmentation for Interactive Content Workflows

arXiv: 2606.17824

Date: 2026-06-16

https://arxiv.org/abs/2606.17824

一句话总结： 提出一个面向交互式内容工作流的人机协同流水线——先用受路径规划启发的贪婪集合覆盖算法挑选覆盖整个 3D 模型的最小相机视角集(8 个对象均收敛到 8 视图),再借助 SAM 2 + Label Studio 在 2D 视图上交互式分割并通过索引 pass 反向投影到 UV atlas,平均 ~21 分钟即可生成可用于分段材质分配、风格迁移与语义标注的统一分段 atlas;在 8 件文化遗产模型上演示,精确暴露出细结构、空腔、弱外观边界等反复的手工修正来源。
TL;DR: A human-in-the-loop pipeline that converts a textured 3D model into a fully segmented 2D UV atlas. A path-planning-inspired greedy set-cover algorithm (implemented in BlenderProc) picks a minimal set of 8 camera views that covers the remeshed surface; SAM 2 + Label Studio is then used to interactively segment those views; the per-view labels are back-projected to UV space via an index pass and merged into one atlas. On 8 cultural-heritage objects the pipeline needs ~15–35 min of annotation per object (mean ~21 min) and is repeatedly tripped up by fine details (eyes, straps, wings), cavities (chair underside, hollow bust interiors) and weak appearance boundaries (marble David, Nefertiti neck). The authors are explicit that this is a demonstration-based technical evaluation, not a user study, and that no ground-truth 3D segmentations were used.

核心贡献

  • 受路径规划启发的最小相机视角选择:把"以最少的相机覆盖整个表面"建模为 surface sample 上的 set cover 问题,在 BlenderProc 中通过光线投射构建 visibility map,迭代选择可观察最多未覆盖点的相机,直至完全覆盖,8 个对象在固定配置下均收敛到 8 个视图。
  • 端到端人机协同分割流水线:把 3D 网格分割问题转化为 UV atlas 上的语义标注,集成 SAM 2 (foundation model) 与 Label Studio (交互标注前端),以最小点击/框提示驱动 mask 预测,并在多视图间迭代精化以保证跨视图一致性。
  • 基于索引 pass 的反向投影:用同一相机渲染的索引 pass 将每个像素映射到唯一的 mesh face,再将 face 的 UV 坐标处写入颜色标签,生成统一的分段 atlas;要求输入网格的 UV 无重叠(否则需要 Blender Smart UV Project 预处理)。
  • 8 件文化遗产对象的演示式技术评估:横跨光滑大理石、弱外观边界、细结构、空腔、重复纹理、细语义细节(大卫头像、Nefertiti 半身像、维多利亚椅、中世纪剑、Erik XIV 国王的礼盾、天使雕像、Róża Loewenfeld 半身像、科普特祈祷书),并诚实报告 15–35 分钟(均值约 21 分钟)的标注时间与每对象的具体失败模式。

Key Contributions

  • A path-planning-inspired view selection algorithm, implemented in BlenderProc, that formulates minimum-coverage viewpoint planning as a set-cover problem over sampled surface points and greedily picks cameras until full surface coverage is achieved; converges to 8 views per object across the demonstration set.
  • A complete end-to-end human-in-the-loop segmentation pipeline that integrates SAM 2 with the Label Studio annotation UI: minimal point/box prompts produce mask predictions, which are iteratively refined across views to enforce cross-view consistency before back-projection.
  • A back-projection stage that uses an index pass rendered from the same camera to map each segmented pixel to a unique mesh face and write its label colour into UV space, merging per-view partial atlases into a single unified segmented atlas (with a stated requirement that the input UV map is non-overlapping).
  • A demonstration-based technical evaluation on 8 cultural-heritage assets (David, Nefertiti, Victorian chair, medieval sword, parade shield of Erik XIV, angel statue, Róża Loewenfeld bust, Coptic prayer book), including per-object approximate annotation times (15–35 min, mean ~21 min) and a frank catalogue of recurring failure modes.

Methodology

The pipeline is presented as four stages (Figure 1 in the paper):

  1. Viewpoint selection (set cover). The model is enclosed in a bounding sphere, translated to the origin, and an enclosing camera sphere defines all candidate viewpoints. The mesh is optionally remeshed to obtain a uniform distribution of surface sample points. For every sample point, raycasts to every candidate camera on the sphere produce a visibility map (encoded via equirectangular projection and parallelized for cost). A greedy strategy then iteratively picks the camera that observes the most uncovered sample points and removes those points from further consideration, until full coverage. The remeshed model is replaced by the original high-fidelity mesh before keyframing and rendering at the selected viewpoints.
  1. AI-assisted interactive segmentation. Rendered views are loaded into Label Studio, which is fronted by SAM 2 for mask prediction. Annotators provide minimal prompts (point clicks or bounding boxes); SAM 2 produces masks which the annotator can correct, add to, or delete. Predictions are iteratively refined across the set of views to maintain a consistent class label scheme, and the output is a set of per-view, class-consistent segmentation masks.
  1. Back-projection onto the UV atlas. For each selected camera, a corresponding index pass is rendered so that each pixel maps to exactly one mesh face. The face's UV coordinates are then used to write the pixel's label colour into the atlas. Because stage (1) guarantees every point is visible from at least one view, the union of the per-view partial atlases covers the full UV map, and the partial atlases are merged into one unified segmented atlas.
  1. Downstream applications. The segmented atlas can be consumed by segment-wise material assignment (per-region roughness / metallicity / albedo without per-region masking), per-region style transfer using 2D neural style transfer, segment-constrained texture synthesis, automated 3D surface annotation for dataset generation, and semantic labelling for downstream ML tasks.

A noted practical assumption: the input mesh must have a non-overlapping UV layout; if it does not, Blender's Smart UV Project is suggested as a preprocessor (with the caveat that overlap-free output is not guaranteed by default).

Results & Highlights

  • Across all 8 objects, the pipeline ran end-to-end and produced atlases usable for the downstream segment-wise material-assignment examples.
  • Per-object annotation time (7 of 8 objects logged): ~15 min (David, sword, Róża Loewenfeld), ~20 min (chair, shield), ~25 min (angel), ~35 min (Nefertiti); mean ≈ 21 min.
  • The pipeline converged to 8 rendered views per model under the fixed configuration, regardless of object.
  • The pipeline's behaviour on the eight cases can be summarized as follows:
  • Sword – cleanest case, minimal correction.
  • Coptic prayer book – simple global structure helps; only low contrast between cover and pages occasionally degrades automatic masks.
  • David head – marble has almost no colour variation; explicit scene lighting is needed to give SAM 2 useful shading cues; hair/neck/face OK, but eyes and ears require largely manual annotation; visible projection artifacts in the eye region (Fig. 5).
  • Nefertiti bust – most time-consuming (~35 min) because of fine eye structures and ambiguous neck texture.
  • Victorian chair – the underside cavity is less well covered than outer surfaces; final atlas still largely complete.
  • Parade shield (Erik XIV) – back-side arm straps require manual boundary correction; SAM 2 / Label Studio struggle with narrow structures.
  • Angel statue – wings have nearly identical appearance to the body and frequently merge with it; need manual drawing.
  • Róża Loewenfeld bust – hollow interior and a locally unfavourable camera angle produce a visible artifact in the upper hair region.
  • Three recurring patterns across the demo: (i) SAM 2 helps most on larger contiguous regions with clear appearance/shading; (ii) manual intervention remains necessary for small structures, narrow attachments, and adjacent regions with near-identical appearance; (iii) multi-view partially compensates for imperfect single-view masks, but shallow viewing angles still reduce precision in cavities and interior surfaces.
  • The paper is a follow-up to the authors' prior atlas-based work [Sinha et al., Eurographics Italian Chapter 2024], which used Tracking-Anything-with-DEVA as a fully automatic backbone with a fixed camera animation and no human refinement; the present paper explicitly differentiates itself by adding optimized view planning and interactive correction.

Critical Assessment

  • Novelty. No component is fundamentally new: greedy set cover for view planning, SAM 2 for masks, label-studio-style interactive annotation, and UV back-projection are all well-studied. The novelty is the orchestration: turning these into a single explicitly covered-atlas pipeline with explicit failure-mode disclosure. That is a real but modest, engineering-oriented contribution.
  • Rigor. This is the biggest weakness. The evaluation is demonstration-based on N = 8 (all cultural heritage, all chosen by the authors), with no user study, no ground-truth 3D segmentation, no comparison to fully automatic baselines (e.g., SAMesh / SAMPart3D / OmniSeg3D / PartSLIP), no IoU / boundary F1 metrics, and only approximate annotation times logged informally. The authors are upfront about all of this, which is good scholarly hygiene, but it does limit the strength of the conclusions. Importantly, there are no ablations of the view-selection strategy (e.g., what does greedy set cover buy you over a fixed icosahedral sampling, or over a learned view planner?).
  • Reproducibility. The pipeline is described in enough detail to re-implement (BlenderProc, SAM 2, Label Studio, index pass, UV write-back), but no code or data are released at submission — the abstract and acknowledgments say it will be made available "via the project website upon acceptance". Until then, only the qualitative claims are verifiable.
  • Significance. Useful and well-positioned for the game / XR / cultural-heritage production community that already operates on UV atlases. The deliberate reframing of 3D segmentation as atlas annotation is a useful practical framing. The contribution to the 3D-vision research community at large is limited because there is no quantitative comparison with the rapidly-improving fully-automatic 3D segmentation literature.
  • Red flags.
  • The "8 views per object" claim is configuration-driven, not a learned result; the paper does not characterize how the view count scales with model complexity.
  • The pipeline assumes a non-overlapping UV — a non-trivial assumption that is quietly hand-waved with "Blender Smart UV Project". For many real production assets this is the hard part, and the paper does not evaluate robustness to imperfect UVs.
  • The reported times are approximate and were not collected under controlled conditions, so the "15–35 min" range should be read as a rough order of magnitude, not a benchmark.
  • "Cultural heritage" is used as the demonstration domain but also implicitly as a generalization claim ("the pipeline itself is not domain-specific") — that is plausible but not tested on, e.g., character meshes, CAD, or scanned indoor scenes.
  • The failure modes (eyes, wings, undersides) are recurring enough that they suggest a structural limitation of the 2D-render-then-project approach, not just noise — the paper is honest about this but does not propose a concrete fix.

Strengths

  • Intellectual honesty. The paper catalogues its own failure modes (fine structures, narrow straps, cavities, identical-appearance wings) with specific examples and visible artifacts. This is rare and valuable.
  • Practical framing. Expressing 3D segmentation as UV-atlas annotation matches the actual workflow of game/XR/cultural-heritage artists, so the artifact has a clear path to adoption.
  • Coverage guarantee. The set-cover view selection gives a formal coverage guarantee on the sampled surface, which is a stronger property than most "render N views" pipelines.
  • Concrete downstream use. Per-object examples of segment-wise material assignment in WebGL / OpenGL viewers show that the atlas is genuinely consumable by real rendering pipelines, not just a colour-coded bitmap.
  • Reasonable build choice. The greedy set cover is simple, parallelizable, and does not require a learned view planner; this lowers the barrier to adoption and reproduction.

Limitations

  • N = 8, single domain, no quantitative metrics. No IoU, no boundary F1, no timing benchmark against fully automatic 3D segmentation, no participant study, no statistical analysis.
  • Strong UV assumption. Requires a non-overlapping UV; the suggested workaround (Blender Smart UV Project) is not guaranteed overlap-free, and the paper does not study sensitivity to UV quality.
  • No ablations. No study of view count vs. quality, no study of remeshing resolution, no comparison to fixed-view / learned-view alternatives, no comparison to fully automatic 3D segmentation.
  • Annotation interface bottleneck. Even with SAM 2, the paper explicitly notes that highly precise corrections for very small regions remain cumbersome in Label Studio — i.e., the bottleneck in hard cases is the UI, not the model.
  • Coverage ≠ visibility quality. "Full coverage" is defined over the sampled (possibly remeshed) surface; the original high-fidelity mesh can still suffer from locally unfavourable angles (Róża Loewenfeld upper hair is the example).
  • Cavity and back-side handling is weak. The greedy set cover over surface points with raycast visibility tends to favour well-observed outer surfaces; the underside of the chair, hollow bust interiors, and back-side shield straps are the recurring hard cases.
  • No code/data at submission time (release promised upon acceptance).

Relevance

Questions for Further Exploration

  • How does this pipeline compare quantitatively (IoU, boundary F1, time-to-atlas) against recent text- or feature-prompted fully-automatic 3D segmentation methods (e.g., SAMPart3D, SAMesh, OmniSeg3D, PartSLIP) on the same 8 (or larger) objects?
  • Could the greedy set-cover view selection be replaced with a learned view planner that also predicts which views will need the most manual correction, and that targets cavities / back-sides / fine structures explicitly?
  • How sensitive is the final atlas quality to the underlying UV unwrapping method, and can the pipeline automatically detect / flag poor unwrappings or even re-unwrap locally?
  • Could the fine-structure and cavity failure modes be mitigated by fusing depth or normal cues (which SAM 2 alone cannot exploit) into the prompt, or by carrying masks across views with a 3D-consistent memory module rather than per-view independent refinement?
  • What is the per-object annotation time under a properly controlled user study with multiple annotators and inter-annotator agreement, and would active-learning-style suggestions inside Label Studio materially reduce it?
  • The authors suggest using the expert annotations produced by the pipeline to embed task-specific priors back into the foundation model — is there a clean distillation recipe that does not require retraining SAM 2 from scratch, and how many objects would be needed to amortize the human-in-the-loop cost?
  • The pipeline is demonstrated on cultural heritage; does it transfer to character / organic / CAD assets where UV layouts are typically cleaner but segmentation semantics are more object-specific?
