Closing the Feedback Loop: From Experience Extraction to Insight Governance in Verbal Reinforcement Learning

Yanwei Cui, Xing Zhang, Yulong Zhang, Li Shao, Xiaofeng Shi, Guanghui Wang, Peiyang He · AWS Generative AI Innovation Center · Amazon Web Services · BingX
verbal RL LLM agents continual learning non-stationary environments agent memory financial forecasting

Training-free verbal RL fails not at extraction but at governance — the same accumulated experience drops below zero-shot without a critic–proposer–curator loop and improves dramatically with one.

1Key metric
34.5% → 13.0%
Max drawdown, same accumulated experience, 2017 S&P 500 top-5 OOS. Sharpe rises 0.36 → 1.00; directional accuracy 51.1% → 56.5% (5 runs).
2Why this matters

It moves "can verbal RL work?" from can we extract useful rules to can we govern accumulated rules — and ships a transferable R1–R4 design checklist plus a reproducible financial case for long-lived, auditable, feedback-driven LLM agents.

3Critical edge

Evaluation is a predominantly bullish 2017 regime — the retention side of the dilemma (stale rules dominating after a true regime shift) is the paper's central motivation but is never directly tested.

Methodologyrules · evidence · skills

Three-layer architecture connected by a critic–proposer–curator curation loop. Each layer is motivated by a failure mode of the one above; evidence is strictly append-only and survives rule deprecation.

  • Layer 1 — Rules (L). Distilled, verbalized experience: trigger condition over observable features + corrective action. Deprecated, never deleted.
  • Layer 2 — Evidence (Ξ). Append-only per-rule logs recording (episode, outcome, conditions, reasoning impact). Full structured trail — not a scalar confidence.
  • Layer 3 — Skills (S). Priority orderings, conflict resolution, abstention — all evidence-grounded, all evolving as evidence accumulates.
  • Critic (R1). Rule-augmented reasoning vs zero-shot baseline on the same observed outcome — attributes gain/loss to specific rules.
  • Proposer (R2). Appends critic evaluations; extracts new rules for uncovered error patterns. Never edits in place.
  • Curator (R3, R4). Deprecates rules with negative evidence; evolves skills for priority, conflict, anti-pattern. 𝓛ₖ, 𝓢ₖ = CURATOR(𝓛ₖ₋₁, 𝓢ₖ₋₁, Ξₖ, Pₖ) with Ξ append-only.

F1
No requirements ⇒ experience is harmful. Reflective Accumulation drops below zero-shot on every metric (Dir. Acc. −4.9pp, Sharpe negative). Rules are injected regardless of reliability; closest-matching dominates.
F2
Partial requirements ⇒ partial recovery. Reflective Refinement returns accuracy to ~zero-shot but degrades risk-adjusted return (Sharpe 0.36 < 0.53). In-place editing invalidates evidence — wrong and useful rules become indistinguishable.
F3
Full loop ⇒ genuine learning. +5.3pp directional accuracy, Sharpe nearly doubled, MaxDD cut by ~60%. New rules are shaped by why predecessors failed, not just by current-batch errors.

Strengths

  • Cleanest ablation in verbal RL. Same agent, same data, only governance varies — the "same experience, opposite outcomes" story is causally interpretable, not correlational.
  • Generalizable checklist. R1–R4 applies beyond this paper; any agent learning from non-stationary world feedback can be evaluated against the four requirements.
  • Honest scope. Authors disclaim live-trading interpretation and flag the bullish-regime limitation.
  • Theoretical grounding. Connects to AGM belief revision (Relevance + Core-Retainment) and operational systems (Kumiho, Hindsight).
  • Real risk win. MaxDD 34.5% → 13.0% on the same accumulated experience is a meaningful risk-management result.

Limitations

  • Domain narrowness. One asset class (large-cap US tech), one horizon (5d), one regime (mostly bullish 2013–2017). Non-stationarity is not strongly present in the test set.
  • No regime-shift test. No bear market, volatility spike, or shock period — the retention side of the dilemma is the paper's motivation but isn't directly exercised.
  • High variance, few runs. 5 evaluation runs; Sharpe std ±0.38 on zero-shot makes the comparison suggestive, not definitive.
  • Hand-designed loop. Deprecation thresholds, critic rubric, and skill-evolution policy are not learned (authors flag this as "meta-curation" future work).
  • Evidence scale. Natural-language evidence logs grow linearly with episodes; long-horizon deployment will need structured / embedding-based representations.
  • Single LLM family. Critic, proposer, and curator all use Claude Sonnet 4.6 — evaluator–generator coupling is not studied.
  • No cost / latency analysis. Per-batch three-role pipeline is non-trivial; real-time deployment profile is unclear.

Questions for further exploration

  1. Does the architecture hold under a true regime shift (2008 GFC, 2020 COVID, 2022 rate-hike cycle) where the retention side of the dilemma is directly exercised?
  2. How sensitive is the result to the choice of critic LLM — does mixing evaluator families (Claude critic + GPT-4o proposer) break or strengthen the loop?
  3. Can the deprecation threshold and skill-evolution policy be learned from outcomes, closing the meta-curation loop the authors flag?
  4. Does the three-layer architecture transfer to other non-stationary, noisy, delayed-feedback domains — robotic wear, ad-creative fatigue, demand-forecast seasonality breaks?
  5. What does a power analysis say about runs needed to confirm the Sharpe improvement at p < 0.05, given the wide ±0.38 std?
  6. As evidence logs grow linearly, what structured representation (evidence embeddings, dependency-graph claims à la IMPACT-CYCLE) preserves auditability while scaling?
  7. Could a temporal-discounting scheme on evidence improve adaptation speed in rapidly shifting regimes without losing lessons from rare conditions?
  8. How does the architecture behave under adversarial or poisoned world feedback, where the signal is actively misleading?
