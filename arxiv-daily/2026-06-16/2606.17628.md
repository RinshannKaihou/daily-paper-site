> **一句话总结：** OPD-Evolver 提出"快慢协同进化"框架，将四层记忆层次（轨迹 / 提示 / 技能 / 工具）与结果校准的归因 + 特权后视的策略内自蒸馏结合，使 9B 模型在 10 个子基准上以 6/10 胜率挑战 Qwen3.5-397B-A17B、以 9/10 胜率压过 Step-3.5-Flash (196B)。

> **TL;DR:** OPD-Evolver reframes memory-augmented agents from "experience stores" into *agent evolvers* that jointly master four coupled abilities — experience selection, experience-grounded execution, experience writing, and experience management. A **fast loop** interacts online with a four-level memory hierarchy (trajectories, tips, skills, tools) and runs a repository-maintenance pass every Q=30 tasks. A **slow loop** computes **outcome-calibrated memory attribution** from delayed environment feedback, then uses a **privileged-hindsight** view (the same policy with access to attribution scores and successful trajectories) to do on-policy token-level KL distillation of all four abilities. The teacher is literally the student's own policy given richer context — no external stronger model — which is the most interesting design choice. On 5 benchmarks × 10 subsets, OPD-Evolver-9B beats all memory baselines and most training-based baselines, exceeding Step-3.5-Flash (196B) on 9/10 subsets and Qwen3.5-397B-A17B on 6/10 (with notable losses on AMA-Causal-Inference, AMA-State-Updating, and LifelongAgentBench-DB). Ablations confirm each of the four trained views contributes, with memory attribution the single most important component.

## 核心贡献

- 提出 **四层记忆层次结构** (`trajectory / tip / skill / tool`)，并将 agent evolver 的能力解耦为 **四类联合能力**：selection、experience-grounded execution、writing、management。区别于以往只关注"取用"的 memory agent。
- 设计 **快慢双循环（slow-fast co-evolution）**：快循环在线做四层记忆交互 + 周期性的 `lookup/merge/delete` 仓库维护；慢循环用 **outcome-calibrated memory attribution** 把延迟标量奖励转成密集监督，再用 **privileged hindsight**（同一策略在看到归因信息下的轨迹）做 **on-policy 策略内自蒸馏**。
- **教师 = 学生自身 + 特权视图**。特权后视包括候选 memory 的价值 V(m)、成功轨迹 τt+、已写 memory 的未来效用、仓库级诊断。无外部更强模型，蒸馏信号完全来自"自我 + 更丰富上下文"。
- 在 5 个 self-evolving 基准 × 10 个子集上 **9B 击败所有同规模 memory agent 与多数 training-based 基线**，并以约 40× 参数压缩挑战 Qwen3.5-397B-A17B、Step-3.5-Flash (196B)。
- 多维消融：slow loop / memory attribution / selection / writing distillation / maintenance 五个组件缺一不可；归因信号贡献最大（移除后 InterCode 平均从 38.67% 跌至 32.13%）。

## Key Contributions

- **Formal definition of a "qualified agent evolver"** as an agent with four coupled competencies (selection, execution, writing, management) optimizing a streaming objective `max lim inf (1/T) Σ E[Rt + λ U(Mt+1)]` that explicitly rewards future repository utility, not just current task return.
- **Four-level memory hierarchy** (`trajectory` for episodic evidence, `tip` for local warnings, `skill` for reusable procedures, `tool` for executable code/commands) — explicit formalization that different experience types have different reuse granularities.
- **Outcome-calibrated memory attribution** A(m) = ρg(m) · (E[Rt | m ∈ St] − E[Rt | m ∈ Ct \ St]), where ρg(m) = |Ω+|/(|Ω+| + |Ω|) downweights unreliable selections. Combined with a tier prior αℓ(m) and confidence γ(m) = 1 − 1/(1 + N+(m)), this produces dense V(m) labels for all retrieved memories from sparse task feedback.
- **Unified hindsight on-policy distillation (slow loop)**: for each of the 4 lifecycle decisions (sel, act, write, maint), construct (z_k, h_k) where the student sees public z_k and the teacher (same θ with stop-gradient, given privileged h_k) explains what should have been selected, executed, written, or maintained. Loss is token-level KL `Dtok[sg[pTθ̄,n] || pSθ,n]` averaged over student-on-policy rollouts.
- **Repository maintenance as a first-class ability**: every Q=30 tasks the agent runs a multi-turn interaction with `T = {lookup, merge, delete}` over its own history, supervised by the slow loop.
- **Cross-domain empirical validation** spanning LifelongAgentBench (DB, OS), MemoryArena (Math, Physics), AMA-Bench (CI, SU, SA), InterCode (Bash, CTF, SQL), and MiniHack (Room, Maze, KeyRoom).

## Methodology

**Streaming formulation.** A lifelong agent at round t receives task xt, generates episode τt = {(ot,i, at,i, rt,i)} with return Rt, and updates a memory repository Mt. The student policy πθ simultaneously decides (i) which memories to retrieve (Ct) and select (St), (ii) how to act given St, (iii) what to write back (∆t), and (iv) periodically how to maintain the repository (ηq ∈ {lookup, merge, delete}). The objective explicitly couples current reward Rt with future repository utility U(Mt+1).

**Four-level memory hierarchy.** Mt = Mtraj ∪ Mtip ∪ Mskill ∪ Mtool. Retrieval is high-recall embedding similarity (top-K=50 per tier, Qwen3-Embedding-0.6B); selection is a learned per-tier compressor that produces a compact context St. The agent is free to write to any subset of the four tiers with arbitrary count.

**Outcome-calibrated attribution (the "calibration" trick).** For each memory m, compare its effect *only on tasks where m was retrieved*: subtract the expected return when m was retrieved-but-not-selected from when m was selected. This is candidate-controlled (controlling for m's relevance) rather than confounding retrieval with selection. V(m) = αℓ(m) · γ(m) · A(m) is a calibrated, confidence-weighted score. This converts sparse delayed feedback into per-memory dense labels.

**Slow-loop on-policy distillation (the core trick).** For each lifecycle decision k ∈ {sel, act, write, maint}, the teacher is literally πθ with stop-gradient (θ̄), but conditioned on a privileged hindsight hk that includes V(m) for retrieved candidates, a successful trajectory τt+ from the same task group, and the actual future utility of written memories. The student samples its own on-policy output ŷk from public zk, and the teacher scores the *same* student prefixes. Loss is full-vocabulary token-level KL with stop-gradient on the teacher. This forces the student to internalize the *reasoning* the teacher would use given hindsight, not just mimic a different policy.

**Repository maintenance.** Every Q=30 tasks the agent runs an episodic interaction with HqQ and the current repository MqQ using the {lookup, merge, delete} tool set. The slow loop supervises this with `Dqmem = {(m, V(m), γ(m), ν(m))}m∈MqQ ∪ {κ(mi, mj)}i,j` containing per-memory value, confidence, usage, and pairwise redundancy. Maintenance training is part of the unified objective.

**Training details.** Backbones: Qwen3-4B-Instruct-2507 and Qwen3.5-9B (both also used as the *teacher*; no external larger model). Training data: 7,000 heterogeneous agent tasks disjoint from evaluation — 3,000 from Agent World Model + 2,000 from nvidia/Nemotron-Terminal-Corpus + 2,000 from EnvScaler. Hardware: 8× A800 80G. Memory-write filter: top 30% of generated memories by V(m); teacher memory filter: min score 0.01; max 20 memories in teacher context; max prompt 8,192 tokens; episode horizon 40 steps.

Removing memory attribution is the single largest drop (avg 38.67% → 32.13%, **−6.5 absolute**). Removing slow-loop distillation entirely drops avg to 33.10% (−5.6). All four trained views contribute.

**Memory internalization (Fig 2–4).** Selection distillation raises the median selected-memory score on InterCode from 0.66/0.69/0.66 to 0.79/0.76/0.76 (SQL/CTF/Bash). Writing distillation raises the median score of newly written memories from ~0.81 to ~0.90. When external memory is removed at test time (w/o M), the trained OPD-Evolver policy is +3–7 points more successful than the vanilla backbone and uses up to 2.5 fewer steps — i.e. useful experience has been *internalized* into the policy, not just used at retrieval time.

**Case study (Fig 5).** On LifelongAgentBench-OS, the vanilla model selects broad directory/configuration memories; OPD-Evolver keeps only the modification-log skill and permission tip. On MiniHack-Room failure, vanilla writes "verify the goal"; OPD-Evolver writes "Exploring adjacent is insufficient" — a sharper causal tip tied to the actual error pattern.

## Critical Assessment

**Novelty.** The *four-capability decomposition* of an agent evolver is a useful conceptual contribution, but is mostly taxonomy: selection, execution, writing, and management have all been studied individually. The genuinely novel piece is **(a) outcome-calibrated attribution** (clean causal estimate of memory effect by candidate-controlled comparison) and **(b) using the same policy as both student and teacher, differing only in privileged hindsight access**. The latter is conceptually clean and under-explored — it isolates the contribution of "self-knowledge" from "model-strength scaling."

**Rigor.** The evaluation is broad (5 benchmarks × 10 subsets) and the ablations are honest (memory attribution is named as the dominant component, not buried). The streaming protocol (empty memory start, only from the evaluation stream) is the right design. **However, several methodological choices invite scrutiny:**

- **Teacher = student (no stronger model)** is principled, but it also means the privileged signal is *exactly* what the student would have seen if it had known which memory mattered. The improvement is therefore an upper bound on "what self-hindsight can give you" — but it does not address how an agent would *acquire* that hindsight in deployment. The paper frames this as a feature, not a limitation, but the deployment story is left vague.
- **"Up to 11.5%" and "~5.8%" are cherry-picked peak numbers.** Per-subset deltas are usually smaller (2–6 points). The framing of "challenges 397B" rests on selecting 6/10 subsets while ignoring losses of 5–10 points on AMA-CI, AMA-SU, and LifelongAgentBench-DB.
- **Compute cost of the slow loop is not reported.** Running a 9B teacher with privileged hindsight over student rollouts, on top of the student's own rollouts, is materially more expensive than the ablation suggests. The 8× A800 setup is described but the *total training hours* are not.
- **Evaluation starts from an empty memory repository** — a fair protocol, but it also means "no memory" baselines (which are reasonably strong, e.g. 75.5 on DB / 52.5 on OS at 9B) are real competitors, and the gap between OPD-Evolver and a well-tuned no-memory agent is often smaller than the gap to the 397B teacher.

**Reproducibility.** Training data sources are named and disjoint from evaluation. Hyperparameters (Q=30, K=50, top-20, min score 0.01) are specified. But the privileged-hindsight teacher construction (specifically, which V(m) values go into the teacher context for which decision type) is described in equations only — the *exact rendering* of hk into the teacher's prompt is not pasted. Implementation choices for the maintenance tool-call format and the per-tier memory schema are also under-specified.

**Significance.** High for the agent-evolution subfield. The framework is a credible synthesis of memory systems + on-policy distillation + credit assignment, and the empirical breadth (10 subsets, 4 capability ablations, 3 trained-view ablations, internalization experiment, case studies) is unusually thorough for the area. The "9B vs 397B" framing will be cited, sometimes uncritically; the AMA-CI and AMA-SU losses are the strongest counter-narrative and should be highlighted in any summary.

**Red flags.**
- The "ReasoningBank by up to 11.5%" delta appears to be a per-subset peak (likely AMA-CI, where OPD-Evolver-9B 47.32 vs ReasoningBank 9B 42.28 ≈ +12% relative, or LifelongAgentBench-OS 65.0 vs 55.0 = +10 absolute = +18% relative). Using "up to" with a peak is fine, but the *average* delta over the 10 subsets is closer to +4 points.
- "Step-3.5-Flash" is cited in the abstract as a giant counterpart; the paper text clarifies it is 196B (not 397B). Abstract-level parameter-count elision is sloppy.
- The paper does not compare against running the teacher (Qwen3.5-9B with privileged hindsight) *at inference time* — i.e., a fair "test-time OPD" baseline where the same 9B is given V(m) labels online. If that baseline matches OPD-Evolver-9B, the distillation gain is illusory at deployment. This is a critical missing comparison.

## Strengths

- The four-capability decomposition is a genuinely useful conceptual frame; future work in this area will probably cite it.
- Candidate-controlled attribution A(m) is a clean, model-agnostic credit-assignment primitive that generalizes beyond this work.
- Using the student's own policy as the teacher (only differing in hindsight access) is a clean experimental design that removes model-strength confounds.
- 10-subset evaluation + 5-variant ablation + memory-internalization experiment + 3-task case study = unusually thorough for an agent-evolution paper.
- Strong results on tasks where most other methods fail (MemoryArena Math/Physics: 10.88/11.63 vs everyone else < 5).

## Limitations

- The "self-evolving" framing is partially undermined by the reliance on outcome feedback and the same-policy privileged view — the agent does not actually become more capable of *finding* good hindsight autonomously.
- Slow-loop compute cost is not reported and is likely substantial (a 9B teacher is queried on every student prefix during the four lifecycle decisions × every student rollout).
- AMA-CI and AMA-SU remain solidly won by Qwen3.5-397B-A17B (10 and 5 points). The "challenges 397B" headline obscures this.
- ReasoningBank and MemEvolve baselines are from 2025; more recent memory-augmentation work (e.g., MemReranker, Skill-Mem) is not compared.
- The four-tier memory schema is fixed; the paper does not show whether the agent would learn the schema itself given more capacity.
- The privileged teacher literally being the student means there is no "what if the student cannot represent the teacher's hindsight" analysis — i.e., the upper bound is bounded by student capacity.

## Questions for Further Exploration

- How would a fair **"test-time OPD"** baseline perform — i.e., Qwen3.5-9B given V(m) labels online at inference, with no slow-loop training? If it matches OPD-Evolver-9B, the distillation gain is purely a deployment-cost amortization story, not a real capability gain.
- What happens if the student is **larger than the privileged teacher** (e.g., student is 30B, teacher is 9B)? Does the gap close or reverse?
- The 4-tier memory schema is hand-designed — could the **tier structure itself be learned** (e.g., via clustering of memory embeddings and re-definition of L = {traj, tip, skill, tool})?
- AMA-CI and AMA-SU losses to 397B are 5–10 points; is this a **scale effect** (reasoning about long trajectories is capacity-bound) or a **distillation-induced loss of precision** (the 9B student cannot faithfully reproduce the teacher's reasoning on hard causal questions)?
- How sensitive is the gain to **K (retrieval depth)** and **Q (maintenance period)**? Both seem magic-number-ish.
- The attribution formula uses a tier prior αℓ(m) and confidence γ(m) — what is the **prior setting sensitivity**, and could both be learned?
- Could **MAWM/MiniHack's Room saturation (100% with GRPO)** indicate that the 9B backbone already has the capability but just needs better credit assignment? If so, OPD-Evolver's gain there is more about *exploration* than *evolution*.
