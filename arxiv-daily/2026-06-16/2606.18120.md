> **一句话总结：** 本文通过 5760 次跨 4 模型 × 7 定界符家族 × 2 攻击目标的对照实验证明：Handlebars `{{x}}` HTML 转义对 ChatML/Llama-3/XML 等尖括号系角色定界符实现完整中和（存活率 0.00），但对 `Human:`/`Assistant:`、Markdown `###` 完全无防护（存活率 1.00），对 Llama-2 部分防护（存活率 0.33），并量化了 GPT-3.5 Turbo（hijack 97%/91%）与 Claude Haiku 4.5（双目标 0%）之间的模型级鲁棒性差异。

> **TL;DR:** This paper isolates the *exact* mechanism by which Handlebars' default double-brace `{{x}}` HTML escaping fails to defend LLM prompts against structural role injection. A model-free differential analysis shows HTML escaping rewrites only `& < > " ' `` ` =`, neutralising ChatML, Llama-3, and XML delimiter families (survival 0.00) while leaving `Human:`/`Assistant:` and Markdown `###` fully intact (survival 1.00); Llama-2 is mixed (0.33) because `<<SYS>>` is disarmed but `[INST]` survives. A 5,760-trial study (3 runs × 16 scenarios × 7 families × 2 slot modes × 2 objectives × 4 models, temperature 0, total API cost $1.63) confirms: where the model has headroom and the delimiter is angle-bracket based, escaping lowers ASR; where the model is saturated or the delimiter survives escaping, it does nothing. Claude Haiku 4.5 holds at exactly 0% across all conditions; GPT-3.5 Turbo and the newer GPT-4.1 mini are both near-saturated. The headline message: the Handlebars escaped default is worth keeping but cannot substitute for structural separation of instruction and data.

## 核心贡献

- **首个孤立变量研究**：首次将 Handlebars `{{x}}` vs `{{{x}}}` 这一独立变量与"模型无关"的角色定界符存活分析配对，定量刻画一个被广泛误用的默认行为。
- **三家族全暴露**：证明三类高使用率定界符（Anthropic 风格 `Human:`/`Assistant:`、Markdown `###`、Llama-2 的方括号部分）在 HTML 转义后字节级完全不变，开发者从 Handlebars 文档中得不到任何关于"哪类被保护、哪类没被保护"的提示。
- **双目标 × 七家族 × 四模型共 5760 次试验**：以 $1.63 的极低成本完成完整阶乘实验，并通过 3 次独立重复 + 95% Wilson 区间 + Fisher 精确检验 + Cramér's V 量化模型饱和与家族免疫的双天花板效应。
- **饱和—免疫二分解释**：给出"为何整体差距看起来不大"的精确机理——模型饱和（如 GPT-3.5 Turbo 即使无结构定界符也已 96% 被劫持）与家族免疫（对存活率为 1.00 的家族转义自然无效）共同压低了平均效应，但并不否定转义在适用家族的局部有效性。
- **可复现资产全部开源**：suite、模板层、评估器、运行脚本、分析脚本与独立校验脚本（`src/verify_numbers.py` 重新派生每一个数字与 CSV 行匹配）全部发布。
- **明确工程结论**："escaped default 值得保留、不值得信任"，必须叠加结构化指令/数据分离（StruQ / instruction hierarchy / Spotlighting）才能形成有效纵深防御。

## Key Contributions

- **First isolation of the independent variable.** To the authors' knowledge, this is the first study to isolate Handlebars' escaped vs raw interpolation mode as the single experimental variable in a prompt-injection experiment, paired with a model-free analysis of which role delimiters the escaping actually neutralises.
- **Three fully exposed families.** Markdown headings, Anthropic-style `Human:`/`Assistant:`, and the square-bracket subset of Llama-2 (`[INST]`, `[/INST]`) survive HTML escaping byte-for-byte, so the documented "safe default" is silently a no-op for them.
- **5,760-trial factorial study.** 3 independent runs × 16 scenarios × 7 delimiter families × 2 slot modes (raw vs escaped) × 2 objectives (task-hijack, secret-exfiltration) × 4 models, all at temperature 0, total API cost $1.63.
- **Honest statistical reporting.** 95% Wilson confidence intervals, Fisher's exact test on the escaping effect, Cramér's V as effect size, Haldane–Anscombe correction for zero cells, and per-run spread (largest 6%) to bound temperature-0 residual non-determinism.
- **Two-ceiling explanation of small mean gaps.** Average raw-vs-escaped gaps look modest because (a) susceptible models are already near saturation on hijack (the plain control succeeds at 96–100%), and (b) three of seven families contribute zero gap by construction.
- **Full reproducibility release.** Suite, family definitions, templating layer (pybars3), evaluator, runner, analysis scripts, and an independent `verify_numbers.py` that re-derives every generated number from the raw CSV are all released.
- **Concrete engineering guidance.** The escaped default is "free and harmless, leave it on" — but must be paired with StruQ-style channel separation, instruction-hierarchy training, or Spotlighting-style span marking, because the threat model is decided by the chat format, not by HTML.

## Methodology

- **Static differential analysis (model-free, exact).** For each of seven delimiter families (plain / markdown / anthropic / llama2 / chatml / llama3 / xml), the attacker payload is rendered through `pybars3` (faithful Python Handlebars implementation) and each structural token is checked for byte-identity pre/post escaping. The escape set is exactly `& < > " ' \` =` — note `=`, `` ` ``, single and double quotes, plus the canonical HTML five. Crucially `[`, `]`, `:` are not rewritten.
- **Threat model.** A developer builds a Handlebars chat template containing a trusted system instruction that plants a unique secret canary and forbids disclosure, a legitimate task, and one slot for an untrusted field (document, support ticket, retrieved record). The attacker controls only that field; the only varying choice is whether the slot is escaped `{{document}}` or raw `{{{document}}}`.
- **Seven delimiter families.** Each family supplies (i) a flattened prompt template that wraps system + task in the family's role markers, and (ii) a "break-out payload" the attacker embeds in the untrusted field to forge a higher-privilege turn. The plain family uses no role tokens (instruction-only control) to isolate how much an injected instruction succeeds without structural framing.
- **Two attack objectives.** Task-hijack (model abandons legitimate task and emits an opaque marker — standard injection-following criterion); secret-exfiltration (model discloses a planted secret canary — higher severity, harder because alignment training targets this directly). Both success criteria are deterministic high-entropy canary/marker matches that never appear in benign or task text.
- **16 scenarios across 8 application categories.** Document summarisation, customer support, retrieval QA, email triage, code review, content moderation, data extraction, translation — two per category.
- **Prompt construction.** Models the common pattern in which a Handlebars chat template is flattened to one string and sent as a single user message — how Semantic Kernel and local-model serving stacks actually render multi-role templates. The trusted system instruction lives inside the rendered prompt and competes with the injected turn. The API system field carries only a generic "You are a helpful assistant" so the in-prompt system competes on equal footing with any forged turn.
- **Models.** GPT-3.5 Turbo, GPT-4o mini, GPT-4.1 mini, Claude Haiku 4.5 (`claude-haiku-4-5`) — chosen to span OpenAI legacy/recent and Anthropic current small models.
- **Design.** Full crossing of escaping (raw, escaped) × family (7) × objective (hijack, exfil) × scenario (16) plus a benign no-injection baseline per escaping mode = 480 cells per model, repeated over 3 runs = 5,760 trials.
- **Statistics.** Attack success rate (ASR) with 95% Wilson CIs, Fisher's exact test on the escaping effect, Cramér's V as effect size, Haldane–Anscombe correction (add 0.5) for 2×2 tables with a zero cell.
- **Determinism.** All calls at temperature 0; repetitions reuse identical prompts; canaries and markers are seeded high-entropy tokens generated by a deterministic script.
- **Anti-conservative caveat (honestly disclosed).** Because repetitions reuse identical prompts, Fisher treating trials as independent is "mildly anticonservative"; the authors therefore also report per-run ASR to show effect stability across runs.

### Headline findings

- **Susceptibility gradient does not track release date.** GPT-4o mini (older than GPT-4.1 mini) is far more robust to hijack than the much newer GPT-4.1 mini. GPT-4.1 mini was *as hijackable* as GPT-3.5 Turbo and *more hijackable* than GPT-4o mini.
- **GPT-4o mini gives the cleanest picture.** Far from ceiling, susceptible enough to be attacked. Per-family (Table V): llama3 falls 56% → 12%, xml 94% → 75%, llama2 33% → 0%, anthropic stays 100% in both modes, markdown stays 88%. Every family whose ASR drops under escaping is one the static analysis neutralises; every family that does not drop is one whose tokens survive.
- **GPT-3.5 Turbo ceiling story.** Its instruction-only plain control already succeeds at near-ceiling rates (96% raw, 94% escaped) without any structural framing — so angle-bracket families have no headroom and sit at the ceiling whether escaped or not. The one family with headroom, llama2, shows the predicted collapse 88% → 52% as escaping removes the `<<SYS>>` framing.
- **Claude Haiku 4.5 holds at exactly 0%.** Across all families, both objectives, both slot modes — 0% ASR everywhere.
- **Exfiltration exposes the interaction more cleanly.** Because exfiltration does not saturate even on the most susceptible model, GPT-3.5 Turbo's per-family breakdown (Table VI) shows the direction of every per-family gap consistent with the static survival rate: llama3 50% → 35% (gap 15), xml 35% → 27% (gap 8), while anthropic 12% → 12% and markdown 33% → 33% are unchanged.
- **Per-run stability.** Largest run-to-run ASR spread across the 3 repetitions is 6% (GPT-4.1 mini exfil esc: 25/30/31); typical spread 1–3%; benign no-injection baseline produced zero false positives across all 4 models.

## Critical Assessment

- **Novelty:** Moderate-to-high. The specific framing — that Handlebars' documented "safe default" is family-conditional rather than universally safe — is a non-obvious insight that working LLM-application engineers routinely miss. The pairing of a model-free static analysis with the empirical study is a clean methodological move.
- **Rigor:** Solid. The 5,760-trial factorial design, 3-run repetition, Wilson CIs, Fisher exact test, Cramér's V effect size, Haldane–Anscombe correction for zero cells, and the explicit "mildly anticonservative" caveat about Fisher on repeated identical prompts all point to careful statistical practice. The benign no-injection baseline that returns 0% is a clean control for false positives.
- **Reproducibility:** Strong. Suite, family definitions, templating layer, evaluator, runner, analysis scripts, and an independent verifier are all released. Canaries and markers are seeded high-entropy. Only outstanding reproducibility question is exact prompt strings and judge logic, which are presumably in the released code.
- **Significance:** High for practitioners building production LLM apps on Semantic Kernel, LangChain (Handlebars), or any Handlebars-based stack. The "HTML escaping is not a defence boundary" message deserves wide circulation. The Claude Haiku 4.5 vs GPT-4.1 mini robustness gap is also operationally important.
- **Red flags / weaknesses:**
  - **Per-family cell size is small (16 scenarios × 3 runs = 48 trials per cell).** Individual family differences are not individually significant; the authors correctly report direction and pooled tests rather than claiming per-family significance.
  - **Hijack success criterion is a marker-emission proxy**, and the exfiltration criterion is a verbatim canary match; both can miss partial compliance (model acts on the injection but emits neither the marker nor the canary, e.g. refuses or paraphrases).
  - **Deployment-shape limitation honestly disclosed:** because the API sends the flattened template as a single user message, structural delimiters reach the model as ordinary text rather than parsed control tokens. A local-model serving stack that tokenises `<|im_start|>` natively would likely *amplify* the raw-vs-escaped gap. The paper's numbers are therefore a **conservative lower bound** on the risk the raw slot creates in token-parsing stacks.
  - **Marker-emission and canary-match may undercount soft compliance** — a model could partially follow the injection (e.g. abandon the legitimate task, change tone, refuse) without emitting the canary/marker.
  - **No comparison to non-escape-based defenses.** StruQ, instruction-hierarchy, Spotlighting are cited but not benchmarked head-to-head; the "cannot substitute" conclusion would land harder with comparative baselines.
  - **Models tested are small/cheap-tier.** Behaviour of larger reasoning models (GPT-4o, o3, Claude Sonnet 4.5+, Claude Opus) is unknown. The Claude Haiku 4.5 = 0% finding may or may not generalise upward in the Claude family.
  - **Survival rate rounding looks like 0.00 but chatml/llama3/xml used both `<` and `>`** — the author is right that escaping rewrites both, so survival is genuinely 0/2, 0/3, 0/4 respectively.

## Strengths

- Clean separation of mechanism (model-free static) from empirical evaluation (5760 trials).
- Cost-transparent methodology ($1.63 total) makes replication accessible to small teams.
- Actionable, falsifiable engineering takeaway — "the escaped default is worth keeping, not worth trusting" — rather than vague "be careful with prompts."
- Cross-vendor model coverage (OpenAI legacy/recent + Anthropic current small) avoids single-vendor artefacts.
- The choice of secret-exfiltration as a harder, non-saturating diagnostic is methodologically sound.
- Honest disclosure of the anticonservative bias from Fisher on repeated identical prompts, plus per-run ASR to bound it.
- The "two ceilings bound the measured effect" framing (model saturation + family immunity) is a genuinely clarifying interpretation.
- The deployment-shape caveat is a fair self-limitation that strengthens rather than weakens the conclusions.

## Limitations

- Single-author work without co-reviewer diversity.
- Only Handlebars / pybars3 studied — generalisation to Jinja2, Liquid, Mustache, f-strings, or other templating engines is asserted but not demonstrated. The authors note other engines have different escape alphabets.
- "Almost entirely resists" for Claude Haiku 4.5 is, in fact, exactly 0% across all conditions in this suite — but this is across 4 models; behaviour of Claude Sonnet / Opus and reasoning models is unknown.
- No comparison to non-escape-based defenses (StruQ, instruction hierarchy, Spotlighting, dual-LLM).
- Per-cell sample sizes (48 trials) limit per-family statistical claims; only direction of effect is reported for individual families.
- The flattened-as-single-user-message rendering pattern is the only deployment shape tested.
- Static analysis treats token survival as binary (byte-identical); semantically equivalent but syntactically different escape encodings are not explored.

## Relevance

本文与用户关注的大模型安全、对齐、提示工程防御方向高度相关。其核心结论——"模板引擎的默认 HTML 转义不是结构化防御边界，仅在角色定界符恰好落在尖括号字符集时才碰巧有效"——对任何用 Semantic Kernel / Handlebars / LangChain 等模板化框架构建生产级 LLM 应用的工程实践都有直接指导价值；同时其量化结果（GPT-3.5 Turbo 与 GPT-4.1 mini 均高劫持率 vs Claude Haiku 4.5 = 0%）也对模型选型与对齐训练方向具有参考意义。

## Questions for Further Exploration

- Do the same family-by-escape interaction patterns hold for other templating engines (Jinja2, Liquid, Mustache, Mako) and other LLM-orchestration frameworks (LangChain, LlamaIndex, Haystack)?
- How do instruction-hierarchy defenses (Anthropic's system-prompt isolation, OpenAI's structured outputs) and StruQ-style channel separation interact with the surviving colon / Markdown / `[INST]` families? Head-to-head benchmark vs Handlebars escaping would be the natural next paper.
- Does the Claude Haiku 4.5 = 0% finding generalise to Claude Sonnet 4.5 / Opus / reasoning models, or is the smaller model paradoxically more hardened?
- In a local-model serving stack that tokenises `<|im_start|>` or `[INST]` as a genuine special token, would the raw-vs-escaped gap be much larger than the conservative lower bound reported here?
- Could a "delimiter-aware" escape (whitelist the safe character set per family) close the surviving attack surface without breaking legitimate templating?
- Does the per-run spread of up to 6% suggest that temperature-0 isn't fully deterministic for these models, and would larger temperature sweeps change the per-family gaps?
- The "two ceilings" framing implies that future work should report per-family ASR (not just marginal) and select models with headroom below saturation to make escaping effects detectable — what experimental design would best disentangle model-side vs templating-side contributions to injection success?
