> **一句话总结：** 本文指出训练无关的口头强化学习在非平稳环境下会陷入"保留-遗忘"两难，通过将既有方法按四项治理需求（R1 结果驱动评估 / R2 持久结构化证据 / R3 非单调知识生命周期 / R4 可组合治理）系统对标后，提出"规则-证据-技能"三层架构加 critic–proposer–curator 反馈回路；并在 S&P 500 五大科技股 2017 测试集上证明：同一份累积经验在无治理时跌破 zero-shot（46.3% vs 51.2% 方向准确率，负 Sharpe），在完整治理回路下则显著优于基线（56.5% 准确率，Sharpe 翻倍至 1.00，最大回撤减半）。

> **TL;DR:** This paper reframes training-free verbal RL for LLM agents as a *governance* problem, not an *extraction* problem. It introduces the **retention-forgetting dilemma** (stale rules cause negative transfer; discarding them causes catastrophic forgetting) and codifies four requirements (R1–R4) that no existing method fully satisfies. The proposed solution is a three-layer architecture (rules + evidence + skills) connected by a feedback-driven **critic–proposer–curator** curation loop, with append-only evidence logs that survive rule deprecation. On S&P 500 top-5 equities (2017 OOS, 5-day horizon), the same accumulated experience *degrades* performance below zero-shot without governance (Dir. Acc. 46.3% vs 51.2%, Sharpe −0.12 vs 0.53) and *dramatically* improves it with the full loop (Dir. Acc. 56.5%, Sharpe 1.00, MaxDD cut from 34.5% to 13.0%). The work is published at the ICML 2026 RLxF Workshop.

## 核心贡献
- **概念层面**：将训练无关口头强化学习的核心瓶颈从"经验抽取"重新定位为"洞察治理"，并系统提出"保留-遗忘"两难（retention-forgetting dilemma），区别于经典的稳定性-可塑性权衡。
- **需求层面**：归纳出治理必须同时满足的四项需求——R1 结果驱动评估（outcome-driven evaluation）、R2 持久化结构化证据（persistent structured evidence）、R3 非单调知识生命周期（non-monotonic knowledge lifecycle）、R4 可组合治理（compositional governance），并以表格形式逐一对标 Reflexion/ExpeL/ExpeL-like/Trajectory-informed tips/Meta-MDP library 等代表性方法，指出 R2、R3、R4 普遍欠投资。
- **架构层面**：提出"规则-证据-技能"三层结构，每一层解决上一层的失效模式（规则无法自证可信，证据无法组合，技能提供有据可循的治理），并以 critic→proposer→curator 三角色流水线形成反馈闭环，使所有生命周期和组合决策都建立在 append-only 的证据之上。
- **实证层面**：以金融预测为案例（同主实验来自 Cui et al. 2026 HPO 协议），用同一组基线（Qwen3-VL-235B + Claude Sonnet 4.6 评审）证明"治理在不在"是同一份经验表现差异的唯一解释变量。
- **方法学层面**：用证据频率作为去标记信号（"被频繁引用的规则若持续带来错误预测即被去激活"），并将形式化的 AGM 信念修正（Alchourrón et al. 1985）与 Kumiho 等近期代理记忆系统对接，为生命周期管理提供理论保证。

## Key Contributions
- **Reframing.** Diagnoses a systematic gap in training-free verbal RL: the field over-invests in *extraction* and under-invests in *governance*. Names the failure mode the **retention-forgetting dilemma**.
- **Requirements taxonomy (R1–R4).** A reusable checklist for evaluating and designing agents that learn from non-stationary world feedback. Demonstrates by enumeration (Table 1) that no reviewed method satisfies all four.
- **Three-layer architecture + curation loop.** Rules (distilled experience), Evidence (append-only per-rule reliability logs), and Skills (priority orderings, conflict resolution, abstention) connected by a critic–proposer–curator pipeline. Each layer is motivated by a *failure mode* of the one above.
- **Empirical causal claim.** Same base agent, same data, same extracted experience — only the governance layer changes. The result reverses from below zero-shot to substantially above.
- **Bridging formal belief revision and modern agent memory.** Connects the design to AGM theory (Relevance/Core-Retainment postulates) and operational systems (Kumiho, Hindsight, IMPACT-CYCLE), providing theoretical grounding for the non-monotonic lifecycle.

## Methodology
### Three-layer architecture
- **Layer 1 — Rules (L).** Distilled, verbalized experience: each rule = trigger condition over observable features + corrective action. Rules are *deprecated* (set inactive) but never deleted; their evidence persists.
- **Layer 2 — Evidence (Ξ).** Append-only per-rule logs: each entry records (episode, outcome, conditions, how the rule affected reasoning). Evidence survives deprecation, enabling audit and re-activation when conditions recur. Differs from Hindsight's scalar confidence scores by retaining the full structured trail.
- **Layer 3 — Skills (S).** Higher-order routing strategies: which rules to prioritize, how to resolve conflicts, and when to abstain. Skills are *evidence-grounded* — they evolve as evidence accumulates, so a skill that prioritizes A over B does so because evidence shows A is reliably better under the conditions where B fails.

### Curation loop (critic–proposer–curator)
- **Critic (closes R1):** For each new batch, compares the rule-augmented reasoning against a zero-shot baseline on the *same* observed outcome. The reasoning-vs-reasoning comparison attributes improvement/degradation to specific rules, not just task success/failure.
- **Proposer (closes R2):** Appends critic evaluations to per-rule evidence logs and extracts *new* rules for uncovered error patterns. Critically, never modifies rules in place — preserves the evidence trail (the key structural difference from ExpeL/TF-GRPO-style in-place editing, which silently invalidates prior evidence).
- **Curator (closes R3, R4):** Reads cross-rule evidence to (a) deprecate rules with consistently negative evidence and (b) evolve skills encoding priority orderings, conflict resolution, and anti-patterns. Because both decisions use the *same* persistent evidence, skill-rule priorities remain mutually consistent.
- **Formal update:** $\mathcal{L}_k, \mathcal{S}_k = \text{CURATOR}(\mathcal{L}_{k-1}, \mathcal{S}_{k-1}, \Xi_k, P_k)$ with $\Xi$ strictly append-only.

### Experimental setup
- **Domain:** Financial forecasting (Top-5 S&P 500 equities: AAPL, AMZN, FB, GOOGL, MSFT).
- **Data:** Daily OHLCV; 20-day candlestick chart inputs; 5-day prediction horizons. 2013–2016 for learning, 2017 for OOS testing.
- **Base agent:** Qwen3-VL-235B (chart → trading signal: direction + scenario + risk params + CoT).
- **Curation roles:** Claude Sonnet 4.6 (critic/proposer/curator); batch size ≈ 16.
- **Baselines (same base agent, only experience-management differs):**
  - **Zero-shot** — chart input only, no learned context.
  - **Reflective Accumulation** (Shinn 2023 / Allard 2026) — error-based rule extraction appended equally.
  - **Reflective Refinement** (Zhao 2023 / Cai 2025a) — importance scoring + in-place modification, no persistent evidence or skills.
- **Metrics:** Directional Accuracy, Scenario Accuracy, Avg. Return/trade, Sharpe Ratio, Max Drawdown. 5 evaluation runs.

## Results & Highlights
**Table 2 (2017 OOS, mean ± std, 5 runs):**

| Method | Dir. Acc. | Scen. Acc. | Avg. Ret. | Sharpe | Max DD |
|---|---|---|---|---|---|
| Zero-shot | 51.2% ± 2.2 | 23.8% ± 0.8 | 0.16% ± 0.1 | 0.53 ± 0.38 | 34.5% ± 7.1 |
| Reflect. Accum. | 46.3% ± 2.9 | 23.3% ± 3.2 | −0.08% ± 0.2 | −0.12 ± 0.32 | 35.2% ± 11.4 |
| Reflect. Refine. | 51.1% ± 1.8 | 24.7% ± 1.5 | 0.14% ± 0.2 | 0.36 ± 0.18 | 24.4% ± 9.0 |
| **Ours (full loop)** | **56.5% ± 1.0** | **29.0% ± 2.5** | **0.33% ± 0.1** | **1.00 ± 0.3** | **13.0% ± 4.4** |

- **Finding 1 — No requirements ⇒ experience is harmful.** Reflective Accumulation drops below zero-shot on every metric (−4.9pp accuracy, negative Sharpe). The retention side of the dilemma: rules are injected regardless of reliability, and the closest-matching rule dominates.
- **Finding 2 — Partial requirements ⇒ partial recovery.** Reflective Refinement recovers accuracy to ~zero-shot but degrades risk-adjusted returns (Sharpe 0.36 < 0.53). In-place modification invalidates accumulated evidence — the system cannot distinguish a frequently wrong rule from a frequently useful one.
- **Finding 3 — Full loop ⇒ genuine learning.** Only when all four requirements are satisfied does the agent *improve*: +5.3pp directional accuracy, Sharpe nearly doubled, MaxDD cut by 60%. New rules are shaped by the accumulated evidence of why predecessor rules failed, not just by current-batch errors.
- **Rule library evolution (Figure 2).** Total cumulative rules grow monotonically; the *active* subset plateaus and the gap (governance contribution) widens over batches. 6 skills (S1–S6) emerge — e.g., "Capitulation reversal" (rules 002, 010), "Absorbed distribution" (rules 013, 016) — with explicit provenance to specific batches and deprecations.

## Critical Assessment
- **Novelty.** The retention-forgetting dilemma is a *useful* reframing, but the three-layer pattern (declarative knowledge + session evidence + procedural skills) has been independently converged on by Hindsight, MemGPT, and others. The genuine new contribution is the *append-only evidence design* + *evidence-grounded skills* — that combination is not in prior work and is what makes the empirical results causally interpretable.
- **Rigor.**
  - The experiment is **well-controlled**: same base agent, same data, only the experience-management layer varies. This is the strongest possible ablation design and the paper exploits it well.
  - However, evaluation covers **5 equities, one regime (2017), one horizon (5 days)**, and the authors admit the test period is a "predominantly bullish regime." The bar for "non-stationarity" in this experiment is regime stability around a trend — not a true regime shift (e.g., 2008, 2020 COVID shock). The central claim (governance matters when conditions *recur*) is therefore not directly tested in this evaluation.
  - **5 runs is minimal** for financial metrics given the high variance on Sharpe (±0.38 on zero-shot). The Sharpe comparison is suggestive rather than definitive.
  - The +5.3pp directional accuracy is a real but modest effect; the Sharpe improvement (0.36 → 1.00) and MaxDD cut (24.4% → 13.0%) are the most striking deltas and are the paper's strongest empirical claim.
- **Reproducibility.** Setup is concrete (model versions, batch size, dataset, splits) but the actual prompts for critic/proposer/curator and the deprecation thresholds are not given in the main text — they would need to be in an appendix or code release. The authors note the learned rules and skills are "human-readable and auditable," which is a soft reproducibility asset.
- **Significance.** Strong as a *framing* contribution (R1–R4 checklist, extraction vs governance dichotomy). The architecture itself is a sensible synthesis of AGM belief revision + modern agent memory; it does not introduce a new training algorithm or a new model class. The strongest signal of significance is the parallel observation from SkillsBench (curated skills +16.2pp vs self-generated −1.3pp) that curation quality — not skill quantity — drives agent gains.
- **Red flags.**
  - The headline result mixes an LLM-as-judge pipeline (Claude Sonnet 4.6 evaluates Qwen3-VL-235B) — a known source of bias toward "verbose, structured" reasoning. Critics, proposers, and curators all being the same LLM family is a concerning single point of failure for the evaluation loop itself.
  - "**Same experience, opposite outcomes**" is true in the *experimental* ablation but the *operational* experience between methods is not literally identical: Reflective Accumulation discards no rules, Reflective Refinement mutates in place, the full-loop version deprecates — so the underlying rule sets *diverge* in ways the paper does not fully quantify.
  - The paper says it "should not be interpreted as evidence of live trading viability" — appropriate, but readers should remember the regime is mostly bullish 2013–2017; a true bear-market test would be far more demanding.
  - Cost/latency of the per-batch critic–proposer–curator pipeline is not discussed; for real-time deployment this is non-trivial.

## Strengths
- The cleanest ablation design in the verbal-RL space: same agent, same data, only governance varies. The "same experience, opposite outcomes" story is the paper's most memorable contribution and is *causally* interpretable rather than correlational.
- The R1–R4 framework is a generalizable design checklist that goes well beyond this single paper. Any future agent that learns from non-stationary world feedback can be evaluated against these four requirements.
- Honest scope: the authors explicitly disclaim live-trading interpretation and call out the bullish-regime limitation.
- Sound theoretical grounding via AGM belief revision (Relevance + Core-Retainment postulates) and operational correspondence with Kumiho/Hindsight.
- Strong practical result: MaxDD cut from 34.5% → 13.0% with the same accumulated experience is a meaningful risk-management win, even if the accuracy gain is modest.

## Limitations
- **Domain narrowness.** Single asset class (large-cap US tech equities), single horizon (5 days), single regime (mostly bullish 2013–2017). The very phenomenon the paper studies (non-stationary conditions) is not strongly present in the test set.
- **No regime-shift test.** No explicit bear-market, volatility-spike, or shock period. The retention side of the dilemma (stale rules dominating after a regime change) is the central motivation but is not directly evaluated.
- **High variance, few runs.** 5 evaluation runs; Sharpe std ±0.38 makes zero-shot's 0.53 result statistically fragile.
- **Critic–proposer–curator is hand-designed.** The deprecation thresholds, critic rubric, and skill-evolution procedure are not learned. The authors acknowledge this and frame it as future "meta-curation."
- **Evidence representation.** Natural-language evidence logs grow linearly with episodes; long-horizon deployment will need structured/embedding-based representations.
- **Single LLM family risk.** All three curation roles use Claude Sonnet 4.6; evaluator–generator coupling is not studied.
- **No cost/latency analysis.** Per-batch critic + proposer + curator calls add up; real-time applicability is unclear.

## Relevance
该工作直接对应用户对 **LLM 智能体的持续学习、非平稳环境鲁棒性、以及"治理 vs 抽取"反思** 的研究兴趣；它把"口头强化学习能否落地"从"能不能抽出有用的规则"重新定位为"能不能治理已累积的规则"，并提供了可迁移的 R1–R4 设计清单与可复现的金融预测案例，是构建长期、稳健、可审计的反馈驱动 LLM 智能体时应当并入参考的方法学贡献。

## Questions for Further Exploration
- Does the same architecture hold up under a true regime shift (e.g., 2008 GFC, 2020 COVID drawdown, 2022 rate-hike cycle) where the retention side of the dilemma is *directly* exercised?
- How sensitive is the result to the choice of critic LLM, and does mixing evaluator families (e.g., Claude critic + GPT-4o proposer) break or strengthen the loop?
- Can the deprecation threshold and skill-evolution policy be *learned* from outcomes (closing the meta-curation loop the authors flag), and what is the sample complexity of such meta-learning?
- Does the three-layer architecture transfer to other non-stationary, noisy, delayed-feedback domains — robotic control with wear, ad bidding with creative fatigue, demand forecasting with seasonality breaks?
- The 5-run evaluation has wide Sharpe bars; what does a power analysis say about how many runs are needed to confirm a Sharpe-improvement claim at p < 0.05?
- As evidence logs grow linearly in natural language, what structured representation (e.g., evidence embeddings, dependency-graph claims à la IMPACT-CYCLE) preserves auditability while scaling?
- Could a temporal-discounting scheme on evidence improve adaptation speed in rapidly shifting regimes without losing lessons from rare conditions (the paper flags this as open)?
- How does the architecture behave under *adversarial* or *poisoned* world feedback, where the signal is no longer merely noisy but actively misleading?
