> **一句话总结：** POIROT 是一种去中心化的多智能体故障诊断协议，通过让系统内部智能体相互质询、独立投票并基于危险空间进行加权聚合，实现了比单一外部评估器更准确的故障归因，且优势随问题复杂度和智能体数量增加而扩大。

> **TL;DR:** POIROT is a decentralized peer-interrogation protocol that repurposes a multi-agent system's own agents as its diagnostic layer. Through structured self-assessment, peer interrogation, private voting, and hazard-aware weighted aggregation, it outperforms single-LLM evaluators in fault attribution, with gains scaling with problem complexity (OR = 1.60, p = 0.008), agent count, and fault dimensionality.

## 核心贡献

- **提出 POIROT 协议**：一种将多智能体系统（LLM-MAS）自身智能体重新用作诊断层的去中心化故障检测与归因框架，无需引入外部评估器或领域专家。
- **五阶段结构化诊断流程**：包括危险空间定义（Phase I）、自我评估（Phase II）、同伴质询（Phase III）、私密投票（Phase IV）和危险感知聚合（Phase V）。
- **基于结构距离的加权共识机制**：通过汉明距离衡量智能体与其归因向量之间的结构差异，为距离危险源更近的智能体分配更高权重，从而减轻单一评估器偏见。
- **在多种基准上验证有效性**：在通用基准 Who&When 和两个安全关键领域（临床康复系统 CORTEX、算法金融交易系统 TradingAgents）上均显著优于单一 LLM 基线。
- **开源发布**：POIROT 作为开源 Python 库发布（`pip install poirot-framework`），并附带 BLAME 基准测试套件，涵盖上述安全关键场景的故障注入数据。

## Key Contributions

- **POIROT Protocol**: A decentralized failure detection and attribution framework for LLM-MAS that repurposes the system's own agents as its diagnostic layer, eliminating the need for external evaluators or domain-specific experts.
- **Five-Phase Structured Diagnostic Process**: Comprising Hazard Space Definition (Phase I), Self-Assessment (Phase II), Peer Interrogation (Phase III), Private Voting (Phase IV), and Hazard-Aware Aggregation (Phase V).
- **Distance-Weighted Consensus Mechanism**: Uses Hamming distance between an agent's structural position and its attribution vector to weight votes, giving higher influence to agents structurally closer to each hazard dimension, thereby mitigating single-evaluator bias.
- **Validation Across Diverse Benchmarks**: Significantly outperforms single-LLM baselines on the general-purpose Who&When benchmark and two safety-critical domains (clinical rehabilitation system CORTEX, algorithmic trading system TradingAgents).
- **Open-Source Release**: POIROT is released as an open-source Python library (`pip install poirot-framework`) alongside the BLAME benchmark suite containing fault injection scenarios for the safety-critical domains evaluated.

## Methodology

The POIROT protocol operates on a formal representation of an LLM-MAS as a tuple **S = (A, C, H, U)**, where A is the set of agents, C software components, H hardware elements, and U human actors.

**Phase I — Hazard Space Definition**: The POIROT Space Constructor builds an N-dimensional vector space **H_haz ⊆ ℝ^N** (where N = |A| + |C| + |H| + |U|), with each axis corresponding to a structurally identifiable risk source. This grounds failure attribution in the system's operational architecture rather than abstract semantic judgments.

**Phase II — Self-Assessment**: Each agent independently reconstructs its trajectory and produces qualitative reports identifying perceived anomalies or confirming absence of irregularities, preserving epistemic independence.

**Phase III — Peer Interrogation**: Agents exchange self-assessment reports and contrast interpretations through structured dialogue facilitated by a POIROUT Router. This allows agents to gather contextual information beyond their local perspective. Crucially, observation is separated from attribution — no public voting occurs during this phase.

**Phase IV — Private Voting**: After interrogation, each agent privately emits a binary attribution vector **v_i ∈ {0, 1}^N**, where (v_i)_j = 1 indicates attribution to component j. Vote privacy prevents strategic alignment.

**Phase V — Hazard-Aware Aggregation**: Votes are aggregated using distance-weighted consensus. Each agent has a canonical position vector p_i (one-hot at its own index). The Hamming distance d_H(p_i, v_i) measures structural divergence. The weight is:

> w_i = β + (1/2) * (1 - d_H(p_i, v_i) / N)

with β = 0.25 fixed, bounding w_i ∈ [0.25, 0.75]. The attribution score for dimension j is S_j = Σ_i w_i (v_i)_j. The component with the highest score is the final attribution.

**Implementation**: POIROT is implemented as a LangGraph multi-agent framework. Each agent operates as an independent node with dynamically bound `talk_to_{target}` communication tools available only during peer review. A `poirot_server` node mediates all inter-agent communications. Each agent is limited to M = 8 LLM invocations before a forced-vote prompt is injected.

**Evaluation Models**: Four models spanning proprietary and open-weight settings: Gemini 2.5 Pro, DeepSeek V3.2 Reasoner, GPT-OSS 120B, and GPT-OSS 20B (the latter two deployed locally via vLLM on an NVIDIA H200 GPU).

## Results & Highlights

**Who&When Benchmark** (122 algorithm-generated systems, 2-4 agents, single injected faults):
- All four models achieve higher fault attribution accuracy under POIROT than the single-LLM baseline.
- Performance advantage scales with problem complexity: logistic regression shows a significant positive interaction (OR = 1.60, 95% CI [1.13, 2.28], p = 0.008).
- In Q4 (highest complexity quartile), POIROT achieves gains of 46% (Gemini 2.5 Pro) and 33% (DeepSeek Reasoner) over the baseline.
- Agent count shows a non-significant main effect (OR = 1.55, p = 0.27) but becomes more influential at high complexity (OR = 2.36 when restricted to Q3-Q4).

**CORTEX Clinical Rehabilitation** (3 agents, 7D hazard space, 128 possible fault localizations):
- Evaluated under both single-fault (k=1) and multi-fault (k>1) conditions with five elemental error vectors targeting agents, hardware, or human components.
- Overall accuracy gains range from +6.7 pp (GPT-oss 20B) to +25.6 pp (DeepSeek Reasoner).
- DeepSeek Reasoner shows statistically significant gains across all conditions (∆ = +42.0 pp for k=1, p < 0.001; ∆ = +25.6 pp overall, p < 0.001).
- Multi-fault accuracy drops substantially for all models (best: 28.8% for Gemini 2.5 Pro with POIROT), reflecting combinatorial difficulty, yet POIROT maintains advantage in 3/4 models.
- Per-agent analysis shows the best single agent falls well below the POIROT aggregate score (gaps of 14.3–21.3 pp), quantifying the limitation of self-evaluation.

**TradingAgents Financial Trading** (12 agents, 15D hazard space, 32,768 possible fault localizations):
- Six elemental error vectors injected at distinct pipeline layers (e.g., inverted technical analysis, fabricated corporate scandal, risk manager override).
- Context windows saturate or approach limits in virtually every run, making this a demanding testbed.
- All POIROT configurations exceed the best baseline (40.2% for GPT-oss 120B), with POIROT accuracies ranging from 44.1% to 66.7%.
- Largest gains: Gemini 2.5 Pro (+41.7 pp), DeepSeek Reasoner (+18.6 pp).
- The POIROT aggregate consistently surpasses individual agent accuracy, confirming hazard-aware aggregation remains effective at scale.
- Geometric projection (Fig. 5) shows individual votes span broadly, but hazard-aware aggregation pulls consensus close to ground truth for best-performing scenarios (RiskMgr-X, Market-X, BullRes-X).

## Critical Assessment

**Novelty**: The core conceptual contribution — reframing evaluation as a distributed diagnostic process using the system's own agents — is genuinely novel. While peer review and multi-agent debate exist in the literature, POIROT's formalization into a five-phase protocol with structured hazard-space voting and distance-weighted aggregation represents a distinct advance beyond ad-hoc self-evaluation or LLM-as-a-judge paradigms.

**Rigor**: The empirical design is commendable. The authors evaluate across four models, three benchmarks of increasing complexity, and both single- and multi-fault conditions. Statistical analysis (logistic regression with interaction terms) supports the claim that POIROT's advantage scales with complexity. However, some limitations exist: (1) The TradingAgents evaluation uses only single-fault injection due to "context-window budget" constraints, leaving multi-fault attribution untested in the most complex setting. (2) Sample sizes vary considerably across models (e.g., Gemini 2.5 Pro has only 4 trials per error type in TradingAgents vs. 17 for others), limiting statistical power for some comparisons. (3) The authors acknowledge that validation against human expert judgment in safety-critical fault attribution remains future work.

**Reproducibility**: Strong. The protocol is formally specified with equations, implemented as an open-source LangGraph library, and evaluated on a released benchmark (BLAME). The supplementary materials include full prompt templates and fault specifications.

**Significance**: High. As LLM-MAS adoption grows, failure attribution in safety-critical domains is both a technical necessity and an emerging legal requirement (EU AI Act). POIROT addresses a real deployment bottleneck and offers a principled alternative to centralized evaluation.

**Red Flags**: The authors note that POIROT's effectiveness is modulated by underlying model capability — both open-weight models show no statistically significant improvement, suggesting the protocol requires sufficiently capable reasoning agents to benefit from peer interrogation. The multi-fault accuracy collapse (best 28.8%) reveals that compound fault attribution remains largely unsolved. The passing of co-author Manuel Cebrian is noted but does not affect the scientific integrity of the work.

## Strengths

- **Conceptually elegant**: Turns a structural feature of LLM-MAS (epistemic diversity across agents) into a safety mechanism, rather than treating it as a source of risk.
- **Empirically thorough**: Evaluates across four models, three benchmarks, and both single/multi-fault conditions with statistical rigor.
- **Scalability evidence**: Demonstrates consistent advantages as problem complexity, agent count, and fault dimensionality increase — the settings where centralized evaluation breaks down.
- **Practical deployability**: Implemented as a drop-in library compatible with both custom API-based agents and LangChain pipelines, requiring no modification to underlying agent architecture.
- **Regulatory relevance**: Directly addresses requirements from emerging AI regulation (EU AI Act risk management, traceability, human oversight).
- **Open science**: Full open-source release of both protocol and benchmark, with detailed supplementary materials.

## Limitations

- **Model capability dependence**: The protocol's benefits are most pronounced with high-capability models (Gemini 2.5 Pro, DeepSeek Reasoner); open-weight models (GPT-OSS 20B/120B) show smaller or non-significant gains, suggesting peer interrogation requires sufficient reasoning capacity to be effective.
- **Multi-fault attribution remains weak**: Compound fault scenarios cause substantial accuracy drops across all models (best overall: 42.3%, best multi-fault: 28.8%), indicating the combinatorial challenge of concurrent failure attribution is largely unsolved.
- **No human expert validation**: While LLM-as-a-judge approaches have shown >80% human agreement in open-ended tasks, fault attribution in safety-critical domains has not been validated against human experts in this work.
- **Context window constraints**: The TradingAgents evaluation was limited to single-fault injection due to context limits, and the protocol itself adds communication overhead during peer interrogation.
- **No consistently dominant model**: The cross-model variability suggests deployment requires fault-type-specific model selection rather than a one-size-fits-all choice.
- **Message limit (M=8)**: The hard invocation cap, while necessary for resource bounding, may truncate deliberation in complex cases.

## Relevance

对于构建和部署多智能体 LLM 系统的研究者与工程师而言，POIROT 提供了一个极具实用价值的去中心化安全监控框架，其核心理念——利用系统内部智能体的集体智慧进行自我审计——可能深刻影响未来高 stakes 场景下的 AI 系统治理与合规架构设计。

## Questions for Further Exploration

1. How would POIROT perform under adversarial conditions where one or more agents are compromised or actively collude to hide failures? The current protocol assumes honest participation.
2. Can the hazard-aware aggregation mechanism be extended to incorporate temporal dynamics, tracking how fault attribution confidence evolves across multiple sessions?
3. What is the human expert agreement rate for POIROT attributions in safety-critical domains like clinical rehabilitation? This validation is acknowledged as missing and would strengthen deployment claims.
4. How does POIROT scale beyond 12 agents? The TradingAgents results are promising, but many enterprise MAS deployments involve dozens or hundreds of agents.
5. Can the peer-interrogation phase be optimized (e.g., through dynamic routing rather than all-to-all communication) to reduce context-window consumption and latency while preserving diagnostic accuracy?
6. Would POIROT work with heterogeneous agent architectures (e.g., mixing LLM agents with symbolic planners or neural policy agents), or does it require all agents to be based on sufficiently capable LLMs?
7. Can the protocol be adapted for real-time failure detection during system execution, rather than post-hoc attribution after a failure has occurred?
