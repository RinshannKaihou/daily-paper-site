Not All Errors Are Equal: A Systematic Study of Error Propagation in Large Language Model Inference

arXiv: 2606.02430

Date: 2026-06-01

https://arxiv.org/abs/2606.02430

一句话总结： 本研究提出了LLMFI确定性故障注入框架，对Phi-3.5-mini、DeepSeek-7B和Gemma-7B三种开源LLM在13项任务上进行了系统性软错误注入实验，揭示了推理阶段、网络层位置和数据类型对错误传播的决定性影响，并据此提出了四种仅通过软件修改即可实现的低开销可靠性增强方案。
TL;DR: This paper presents LLMFI, the first configurable and deterministic fault-injection framework designed for fine-grained modeling of error propagation in LLM inference. Through ~3,000 GPU hours of experiments across three open-weight LLMs (Phi-3.5-mini, DeepSeek-7B, Gemma-7B) and 13 tasks spanning reasoning, multilingual, math, and coding, the authors distill 17 actionable takeaways characterizing how soft errors propagate across layers, inference stages, data representations, and architectures. Four low-overhead, software-only mitigation strategies are proposed and validated: Mixed-precision Inference, Selective Checksum-based Protection, Dual First-token Generation, and Dynamic Shot Inference. Accepted at ICS 2026.

核心贡献

  • 提出了LLMFI，首个可配置、确定性、模型无关且推理阶段感知的故障注入框架，支持在任意LLM架构、任务、子模块和推理阶段进行可复现的故障注入。
  • 对三种开源权重LLM（Phi-3.5-mini-instruct 3.8B、DeepSeek-LLM-7B-chat 6.91B、Gemma-7B 8.54B）和13项代表性任务进行了大规模系统性实验，覆盖推理、多语言理解、数学求解和代码生成四大领域。
  • 通过六个细粒度案例研究，揭示了错误在层内、跨阶段、跨数据类型以及MoE架构中的传播规律，总结出17条关于LLM推理错误传播的见解。
  • 基于实验洞察，提出了四种仅通过软件修改即可实现的低开销可靠性增强方案，并在实验中验证了其有效性。

Key Contributions

  • LLMFI Framework: A task-agnostic, model-agnostic, and inference-stage-aware fault injector built on PyTorch and HuggingFace Transformers. It supports deterministic execution (via greedy decoding) and diverse fault models including computation faults (bit-flips in activations via forward hooks) and memory faults (bit-flips in weights via forward_pre hooks), with controllable injection timing across prefill, first-token, and decode-loop stages.
  • Large-scale Empirical Study: Evaluates across 3 open-weight LLMs and 13 representative tasks with fine-tuned prompts matching industrial standards. The study covers ~3,000 GPU hours on NVIDIA A100 GPUs.
  • 17 Takeaways: Six case studies analyzing layer-wise, stage-wise, KV-cache, data-type, MoE, and multi-bit fault propagation yield 17 actionable insights characterizing LLM resilience behavior.
  • Four Mitigation Prototypes: Mixed-precision Inference, Selective Checksum-based Protection, Dual First-token Generation, and Dynamic Shot Inference — all software-only, low-overhead directions with validated coverage and overhead metrics.

Methodology

LLMFI Architecture: The framework comprises three core components orchestrated by a Runtime Wrapper:

  • Injector: Defines fault type (computation vs. memory) and bit-flip pattern (random or manual bit index).
  • Inspector: Parses arbitrary LLM architectures using regex-based matching to locate injection targets at three granularities: submodule (e.g., qkv_proj), functional layer (e.g., self-attention), or entire Transformer block.
  • Scheduler: Controls injection timing across prefill, first-token, and decode-loop stages. The prefill stage is further decoupled into prefill (KV cache construction) and first-token (initial output generation) to distinguish context corruption from initial trajectory corruption.

Deterministic Execution: LLMFI disables stochastic decoding (top-p, temperature) and enforces manual greedy decoding to ensure reproducible outputs for fair fault-free vs. faulty comparisons.

Evaluation Setup:

  • Models: Phi-3.5-mini-instruct (3.8B, Microsoft), DeepSeek-LLM-7B-chat (6.91B, DeepSeek AI), Gemma-7B (8.54B, Google). Supplementary evaluations on Qwen2.5-7B, Mistral-7B-v0.3, and Meta-Llama-3-8B confirm generalizability.
  • Tasks: 13 tasks in 4 categories — Reasoning (ARC, BoolQ, HellaSwag, OpenBookQA, PIQA, TruthfulQA, WinoGrande), Multilingual (MMMLU, XCOPA), Math (GSM8K, MATH500), Code (HumanEval, MBPP).
  • Fault Model: Single bit-flip per trial. Computation faults perturb activations; memory faults perturb weights. Float16 is the default precision.
  • Platform: NVIDIA A100 GPUs (40GB) on Argonne National Laboratory's Swing cluster; PyTorch 2.6.0, Transformers 4.40.1.

Runtime Overhead: LLMFI adds ~10% overhead for short generations and ~10–15% for long generations, making it practical for large-scale studies.

Results & Highlights

Overall Resilience (Table 3):

  • Fault injection causes measurable accuracy degradation across all tasks: up to 4.25% in multi-choice reasoning and 5.01% in generative tasks.
  • The inference stage at which a fault occurs has a dominant impact. For reasoning tasks (constrained output format), only prefill and first-token stages matter; decode-stage faults have negligible effect.
  • First-token stage is highly sensitive to computation faults but less affected by memory faults, because computation faults directly perturb the activation determining the first output token, while memory fault effects take time to manifest.
  • In generative tasks, memory faults have greater sustained impact than computation faults because corrupted weights persist across all forward passes, whereas computation faults affect only a single pass.

Layer-wise Propagation (Section 5.1):

  • Accuracy remains stable across early and middle Transformer layers but degrades consistently in the final 3–4 blocks.
  • Early/middle layers exhibit natural self-correction via nonlinear activations and residual connections; final layers lack downstream correction.
  • Submodule analysis reveals dimensional expansion mitigates errors (gate_up_proj is most resilient), while dimensional reduction amplifies them (down_proj is most vulnerable).

Stage-wise Propagation (Section 5.2):

  • In generative tasks, errors in early decode iterations are less likely to recover than those in later steps.
  • Certain decode steps act as "critical points" that disproportionately influence accuracy — likely corresponding to tokens anchoring reasoning chains or numerical values.

KV Cache (Section 5.3):

  • Direct memory faults in KV cache cause modest accuracy reductions (0.23–0.78), indicating KV cache is generally more resilient than model weights.
  • KV cache sensitivity fluctuates across layers, determined by internal attention semantics rather than depth alone.

Data Type Sensitivity (Section 5.4):

  • In float16, bit 14 (most significant exponent bit) causes catastrophic, irrecoverable deviations.
  • Recovery ability ranks: quantized integers > float16 > float32 > bfloat16. Bfloat16 is most vulnerable due to its larger exponent field amplifying bit-flip impact.

MoE Architecture (Section 5.5):

  • MoE models (Phi-3.5-MoE-instruct, 41.9B) show similar resilience to dense models for computation faults.
  • For memory faults, MoE models are notably more robust because only 20–30% of experts are activated, masking faults in inactive parameters.
  • The expert router is highly sensitive — even a single error may alter expert selection severely — but constitutes <0.1% of parameters.

Multi-bit Faults (Section 5.6):

  • Double bit-flip faults reduce average accuracy from 63.97% to 61.92%, showing slightly lower resilience as multiple bit flips are more likely to corrupt values into irrecoverable patterns.

Mitigation Validation:

  • Mixed-precision Inference: Replacing float16 with int8 for mlp.down_proj layers improves faulty accuracy (ARC: 81.76%→83.01%; BoolQ: 79.35%→81.74%) with minimal fault-free accuracy loss (~0.8%).
  • Selective Checksum: Protecting only dimension-reducing GEMM layers achieves 80.04–90.31% fault coverage while protecting only ~30.5–32.1% of total GEMM workloads.
  • Dual First-token Generation: Achieves 82.20% fault coverage with only 3.37% increase in Time-to-First-Token (TTFT).
  • Dynamic Shot Inference: Increasing in-context examples from 0 to 10 shots improves faulty accuracy on ARC from 78.46% to 81.76% without degrading fault-free performance.

Critical Assessment

Novelty: High. While fault injection for DNNs and LLMs has been explored (e.g., PyTorchFI, LLTFI, Sun et al.'s end-to-end study), this is the first work to provide a fine-grained, stage-aware, model-agnostic fault injector specifically for LLM inference, and the first systematic characterization across diverse tasks and architectures. The decoupling of prefill into prefill+first-token stages is a particularly insightful methodological contribution.

Rigor: Strong. The study controls for prompt fine-tuning, deterministic decoding, industrial-standard evaluation settings, and reports raw accuracy. The ~3,000 GPU hours and supplementary evaluations on 6 total models demonstrate thoroughness. However, the single bit-flip model, while standard for computation faults, is somewhat optimistic for modern ECC-protected memory (the authors acknowledge this and extend to multi-bit faults). The 10-trial repetition per sample is adequate but could benefit from confidence intervals.

Reproducibility: Good. LLMFI is open-sourced, models are open-weight, tasks use standard benchmarks with documented settings. The deterministic greedy decoding ensures reproducible comparisons. One minor concern: the paper does not explicitly state whether the exact random seeds for fault location sampling are provided.

Significance: High for HPC-AI convergence. As LLMs are deployed in exascale systems for scientific discovery, understanding soft error propagation is critical. The 17 takeaways provide actionable guidance for system designers, and the four mitigation directions are practical and immediately deployable.

Red Flags:

  • Baseline performance dependency: The authors correctly note that tasks with near-random baseline accuracy (e.g., MMMLU, XCOPA for some models) make resilience analysis meaningless. This is an honest acknowledgment, but it also means some reported task results are uninformative.
  • Limited model scale: All primary models are ≤8.5B parameters. The MoE evaluation uses a 41.9B model but only on two reasoning tasks. Whether findings generalize to 70B+ models remains unverified.
  • Mitigation overhead claims: "Low-overhead" is substantiated for three of four directions, but Dynamic Shot Inference's overhead in terms of context length / memory is not quantified, and the optimal policy is explicitly left for future work.
  • Cherry-picking concern: The mitigation prototypes are evaluated on subsets of tasks (e.g., Mixed-precision on ARC/BoolQ only). Broader validation across all 13 tasks would strengthen claims.

Strengths

  • First principled fault injector for LLM inference stages: LLMFI's decoupling of prefill/first-token/decode enables stage-specific analysis that prior end-to-end studies missed.
  • Exceptionally broad empirical coverage: 3 models × 13 tasks × multiple fault types × multiple stages × layer-wise and bit-wise granularity = one of the most comprehensive LLM reliability studies to date.
  • Actionable insights: The 17 takeaways are concrete and often counter-intuitive (e.g., early layers self-correct, dimensional expansion mitigates errors, bfloat16 is worse than float16).
  • Practical mitigations: All four proposed directions are software-only, validated with real measurements, and target specific vulnerabilities identified in the study.
  • Intellectual honesty: The authors explicitly flag limitations (weak baselines making some tasks uninformative, model scale constraints, MoE router sensitivity) rather than overclaiming.

Limitations

  • Model scale: Primary experiments limited to 3.8B–8.5B dense models. MoE evaluation is brief (two tasks). Findings may not fully generalize to frontier-scale models (70B+).
  • Fault model scope: Single bit-flip is standard but may not capture all real-world soft error scenarios. Multi-bit evaluation is present but limited to two bit-flips in the same word.
  • Task coverage: Only 13 tasks evaluated; long-context tasks (e.g., document summarization, retrieval-augmented generation) are not studied.
  • Mitigation breadth: Each mitigation prototype is validated on a subset of tasks rather than the full benchmark suite.
  • Quantization interaction: The Mixed-precision Inference uses basic int8 range-based quantization. How advanced quantization (e.g., GPTQ, AWQ) interacts with error resilience is unexplored.
  • Dynamic Shot Inference lacks automation: The paper shows shot count improves resilience but does not provide an automatic policy for selecting the optimal number of shots per task.

Relevance

随着大语言模型越来越多地部署在高性能计算和关键任务环境中，理解软错误在推理过程中的传播机制对确保AI系统的可靠性至关重要。本研究不仅提供了首个系统性的错误传播分析框架，还提出了四种立即可部署的软件级缓解方案，对关注LLM系统鲁棒性、可靠性工程和HPC-AI融合的研究者与工程师具有极高的实践参考价值。

Questions for Further Exploration

  1. How do the vulnerability patterns identified in 3.8B–8.5B models scale to 70B+ parameter models and multi-GPU tensor-parallel inference, where communication errors may compound local faults?
  2. Can the critical decode steps identified in generative tasks be predicted a priori using attention entropy or logit confidence, enabling targeted protection without full dual-generation?
  3. How does error resilience interact with advanced quantization methods (GPTQ, AWQ, SmoothQuant) and speculative decoding, which are increasingly standard in production inference?
  4. What is the impact of soft errors in long-context inference (>100K tokens), where KV cache size and attention patterns differ significantly from the evaluated tasks?
  5. Can the layer-wise self-correction phenomenon (Takeaway 8) be exploited to design adaptive precision or checkpointing strategies that dynamically protect only the final few layers?
