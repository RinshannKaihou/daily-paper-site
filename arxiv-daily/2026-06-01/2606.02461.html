AGENTCL: Toward Rigorous Evaluation of Continual Learning in Language Agents

arXiv: 2606.02461

Date: 2026-06-01

https://arxiv.org/abs/2606.02461

一句话总结： 本文提出了 AGENTCL，首个针对语言智能体持续学习的严格评估框架，通过受控的组合式任务流和可量化的可塑性、稳定性、泛化性指标，揭示了现有记忆设计在跨任务复用中的瓶颈，并强调任务流设计是评估持续学习能力的关键。
TL;DR: This paper introduces AGENTCL, the first rigorous evaluation framework for continual learning in language agents. It uses controlled compositional task streams and targeted metrics (Plasticity Gain, Stability Gain, Generalization Gain) to diagnose non-parametric memory designs. Key findings: compositional streams greatly amplify discriminative power over naive streams, and a severe plasticity–stability trade-off remains unresolved in current memory methods.

核心贡献

  • 提出 AGENTCL 评估框架：首个针对语言智能体持续学习的系统性评估框架，核心在于对跨任务关系的显式控制与针对性指标设计。
  • 设计组合式与朴素任务流：组合式任务流中前置子任务的解法、证据或工作流可被后续复杂任务复用；朴素任务流则任务间无保证的复用关系。两者形成对照。
  • 提出三项核心指标：可塑性增益（Plasticity Gain, PG）衡量前期经验对后续任务的帮助；稳定性增益（Stability Gain, SG）衡量经验经记忆固化后是否仍能稳定复用；泛化性增益（Generalization Gain, GG）衡量记忆对未见任务的迁移能力。
  • 提出 MemProbe 诊断方法：一种非参数化的记忆探查方法，通过检索-求解-固化循环，将经验分别固化为交互记忆、洞察记忆和技能记忆三种互补视图，并过滤不可靠经验。
  • 跨领域实证分析：在编程（CodeEval-Pro）、深度研究（BrowseComp+）和语言理解/推理（MMLU-Pro、AgentBoard）三类任务上，系统评估了 8 种代表性记忆方法。
  • 关键发现：组合式任务流对记忆设计的区分力显著强于朴素流；现有记忆方法主要体现“专项复用”而非“强泛化”；可塑性-稳定性权衡仍是未解决的瓶颈。

Key Contributions

  • AGENTCL evaluation framework: The first systematic framework for evaluating continual learning in language agents, centered on explicit control of cross-task relationships and targeted metrics.
  • Compositional vs. naive task streams: Compositional streams intentionally place reusable sub-solutions, evidence, or workflows before related complex tasks; naive streams lack such guaranteed reuse relationships.
  • Three targeted CL metrics: Plasticity Gain (PG) measures whether earlier experience helps later tasks; Stability Gain (SG) measures whether consolidated memory remains retrievable and useful after further updates; Generalization Gain (GG) measures transfer to held-out unseen tasks.
  • MemProbe probing method: A non-parametric memory method that uses a retrieve–solve–consolidate loop to store experiences as three complementary views—interaction memory, insight memory, and skill memory—while filtering unreliable outputs.
  • Cross-domain empirical analysis: Comprehensive evaluation of 8 representative memory methods across coding (CodeEval-Pro), deep research (BrowseComp+), and language reasoning (MMLU-Pro, AgentBoard BabyAI/ScienceWorld).
  • Key findings: Compositional streams have far greater discriminative power than naive streams; existing memory methods exhibit specialized reuse rather than strong generalization; the plasticity–stability trade-off remains a critical unresolved challenge.

Methodology

The paper formalizes continual learning for language agents as follows: an agent observes a task stream T = {τ₁, τ₂, ..., τ_T} and maintains a persistent memory {M_t} shared across tasks. For each task, the agent retrieves relevant context c_t = R(M_{t-1}, τ_t), solves the task, and then updates memory via M_t = U(M_{t-1}, ξ_t), where ξ_t is the trajectory. The evaluation operates without ground-truth guidance during memory construction, forcing the LLM to synthesize memories entirely autonomously.

Task Streams: Two stream types are constructed:

  1. Naive streams: Tasks are drawn from the same domain but without enforced cross-task solution relationships. This directly streams tasks from existing datasets.
  2. Compositional streams: Later tasks can benefit from reusing knowledge, intermediate results, evidence, or helper functions from earlier tasks. For coding, this uses CodeEval-Pro self-invoking problem pairs (48 pairs). For deep research, 308 subtasks are synthesized from 100 BrowseComp+ parent tasks using GPT-5.2, with each subtask sharing a subset of evidence documents. For BabyAI, 28 source tasks are placed before 28 related target tasks requiring composed skills.

Two-Pass Evaluation Protocol: The agent processes the stream twice. The first pass permits both memory retrieval and updating. The second pass is read-only with frozen memory. Let B_i be the memoryless baseline performance on task i, F_i the first-pass performance, and S_i the second-pass performance. Plasticity Gain is PG_i = F_i − B_i; Stability Gain is SG_i = S_i − F_i. Held-out tasks H_j (unseen during memory construction) measure Generalization Gain GG_j = H_j − B_j.

MemProbe: The authors develop MemProbe as a diagnostic memory design. It retrieves semantically similar prior episodes before solving. After solving, it validates outputs via a syntactic checker and an LLM judge. Valid experiences are consolidated into: (1) interaction memory (concrete trajectories and responses), (2) insight memory (task patterns and failure modes), and (3) skill memory (reusable procedural snippets). Invalid outputs are discarded.

Evaluated Methods: Four categories are evaluated: (1) memoryless ReAct baseline; (2) adaptive memory (LangMem, Mem0); (3) procedural knowledge memory (Agent Workflow Memory, Dynamic Cheatsheet); (4) self-evolving memory (ExpRAG, ReMem, MemProbe). The default LLM is Qwen3.5-35B-A3B; gpt-oss-120b is used for deep research.

Results & Highlights

Compositional streams distinguish memory designs far better than naive streams. On CodeEval-Pro complex tasks, the standard deviation of accuracy across methods is 9.4 and 8.8 under the compositional stream, versus only 3.0 and 1.9 under the naive stream. The effect is even stronger on BrowseComp+: 14.9 and 16.0 (compositional) vs. 2.3 and 5.7 (naive). MMLU-Pro, evaluated as a naive stream, shows only 1.7–1.8 standard deviation across methods. This confirms that controlled task relationships are essential for diagnostic CL evaluation.

MemProbe achieves the highest plasticity but reveals stability trade-offs. On CodeEval-Pro compositional stream, MemProbe achieves PG = +21.9 (vs. ReAct), the highest among all methods. However, its SG = −2.1 indicates that after memory consolidation and subsequent updates, the reusable experience does not remain stably accessible. ExpRAG and ReMem show similar patterns: strong compositional PG (+17.7 and +13.5) but near-zero or negative SG (+0.0 and −2.0).

Naive streams expose stability issues. On naive CodeEval-Pro, SG ranges from −3.5 to +4.2 across methods. On naive BrowseComp+, SG swings are even larger: −11.0 (Mem0) to +11.0 (MemProbe). Mem0 shows a particularly unstable pattern: modest naive PG (+3.0) but the worst naive SG (−11.0), suggesting its accumulated memory actively interferes with later unrelated tasks.

Held-out generalization is limited for all memory methods. After training on BigCodeBench-Lite-Pro streams, all memory methods perform at or below the memoryless ReAct baseline on 120 held-out HumanEval-Pro tasks (ReAct: 72.5; MemProbe: 70.8; others lower). This indicates that memory accumulated from related tasks does not automatically generalize and may introduce mild interference.

Different memory representations play complementary roles. Ablations on MemProbe show that removing interaction, insight, or skill memory each consistently weakens compositional-stream performance, but has smaller effects on naive streams. Replacing the LLM judge with oracle boolean feedback yields similar performance, suggesting memory construction does not require a perfect judge.

Retrieval quality is high but semantic mismatch causes failures. Hits@k for retrieving the correct subtask memory given a complex task query is 95.8% (Pass 1) and 91.7% (Pass 2) on BigCodeBench-Lite-Pro, and 87.0% / 77.0% on BrowseComp+. However, case studies show that even topically similar retrieved memories can hurt when semantics are incompatible (e.g., a task requiring rejecting non-numeric columns vs. a retrieved task that ignores them).

Critical Assessment

Novelty: The paper's core novelty lies in reframing agent continual learning evaluation around controlled task relationships rather than simply streaming tasks. The distinction between compositional and naive streams, and the two-pass protocol with PG/SG/GG metrics, is conceptually clean and fills a real gap. However, the individual components (task decomposition, non-parametric memory, two-pass evaluation) are not entirely new; the novelty is in their systematic integration into a unified benchmarking framework.

Rigor: The methodology is generally rigorous. The authors control for task relationships, use multiple domains, report standard deviations across methods, and conduct bootstrap comparisons. Quality control for synthesized BrowseComp+ subtasks is thorough (Jaccard similarity 0.10 mean, no pair > 0.30). However, several limitations exist: (1) The evaluation uses only one LLM backbone (Qwen3.5-35B-A3B) for most tasks, with limited sensitivity analysis on model scale. (2) The sample sizes for some streams are modest (48 coding pairs, 100 deep research tasks), and results are averaged over only 3 runs for coding. (3) The held-out evaluation uses a different dataset source (HumanEval-Pro vs. BigCodeBench-Lite-Pro), which confounds stream-type effects with domain-shift effects.

Reproducibility: The benchmark construction is well-documented, and data is released. However, reimplementing methods like LangMem, Mem0, AWM, and DC-RS when public code was unavailable introduces potential implementation bias. The authors acknowledge this and state they matched workflows "as closely as possible," but this remains a limitation. The two-pass protocol and metric definitions are clear and should be reproducible.

Significance: This work is highly relevant to the agentic AI community. As language agents are deployed in long-running settings, their ability to accumulate and reuse experience is critical. The paper convincingly demonstrates that naive benchmarks are insufficient and that current memory methods are far from solving the plasticity–stability dilemma. The framework is likely to be adopted and extended by the community.

Red Flags: (1) The MemProbe method is introduced by the same authors and achieves the best results on some metrics, creating a potential conflict of interest, though ablations and comparisons with 7 other methods partially mitigate this. (2) The claim that "existing benchmarks struggle to evaluate continual learning rigorously" is somewhat overstated—recent concurrent work (Continual Learning Bench) addresses similar issues. (3) The generalization results are rather weak across all methods, raising questions about whether the held-out tasks were too distant from the training distribution, or whether non-parametric memory is fundamentally limited for generalization.

Strengths

  • Conceptual clarity: The framework distills continual learning evaluation into two core desiderata (control of task relationships + agentic CL metrics) and satisfies both elegantly.
  • Strong empirical evidence: Compositional streams consistently show 3–7× larger method separation than naive streams across multiple domains, making the central claim empirically robust.
  • Diagnostic power: The two-pass protocol successfully decouples plasticity, stability, and generalization, enabling fine-grained diagnosis of memory design choices.
  • Practical relevance: The findings directly inform memory design for deployed agents, highlighting that "storing more" is not the solution—selective consolidation and retrieval are.
  • Thorough quality control: Synthesized subtasks are verified for determinism, evidence support, and non-paraphrase, with quantitative diagnostics reported.
  • Comprehensive coverage: The framework spans coding, deep research, and language reasoning, demonstrating broad applicability.

Limitations

  • Limited LLM backbone sensitivity analysis: Most experiments use a single model (Qwen3.5-35B-A3B). It is unclear whether findings generalize across model scales or families.
  • Small sample sizes and limited replication: 48 coding pairs and only 3-run averaging for coding tasks may limit statistical power.
  • Implementation bias for baseline methods: Reimplementation of baseline memory methods without official code risks subtle workflow mismatches.
  • Held-out evaluation confounds: Using a different dataset (HumanEval-Pro) for held-out testing makes it difficult to separate stream-type effects from domain-shift effects.
  • MemProbe as both contribution and best performer: While ablations are provided, the diagnostic method introduced by the authors also achieves top results, which could benefit from independent replication.
  • No exploration of parametric CL: The framework focuses exclusively on non-parametric memory; how it applies to parameter-updating continual learning methods is left as future work.
  • Narrow definition of stability: Stability is framed as retrievability after memory updates, not as resistance to catastrophic forgetting in model weights—a legitimate but different perspective.

Relevance

Questions for Further Exploration

  1. How would the AGENTCL framework apply to parametric continual learning methods (e.g., LoRA fusion, policy gradient updates), and would the same plasticity–stability patterns emerge?
  2. Does the compositional-stream advantage hold for very large LLMs (e.g., GPT-4-class models), or does increased in-context learning capacity reduce the need for explicit memory design?
  3. Can adaptive retrieval mechanisms (e.g., learning to reject irrelevant memories) close the stability gap observed in MemProbe and other methods?
  4. How sensitive are the metrics to stream length and the ratio of compositional to naive tasks in a mixed stream?
  5. Could active exploration or self-directed task generation help construct better memory in naive-stream settings where reusable structure is not guaranteed?
  6. What is the relationship between the retrieval accuracy (Hits@k ~90%) and the much lower stability gains (SG often near zero or negative)? Does this indicate a reasoning failure in using retrieved memories, not just a retrieval failure?
