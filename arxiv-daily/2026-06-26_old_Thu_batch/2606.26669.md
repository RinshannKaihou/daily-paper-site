> **一句话总结：** SkillDisCo 将智能体成功轨迹蒸馏为参数化有限状态机（PFSM）子图，再编译为可调用、可执行、可验证的过程化技能，在 ALFWorld 和 WebArena 上显著提升成功率与执行效率，并能把强模型蒸馏出的技能迁移给弱模型。

> **TL;DR:** SkillDisCo formalizes procedural skills as reusable parameterized finite-state-machine (PFSM) subgraphs mined from successful agent traces, then compiles them into callable, executable, and verified code skills, improving success rate and turn efficiency on ALFWorld and WebArena while transferring knowledge from stronger to weaker models.

## 核心贡献

- 明确了过程化技能发现（procedural skill discovery）的适用范围，将其限定在 **FSM 定义的场景**中，使"成功轨迹是未知转移图中的路径"这一语义清晰化。
- 把过程化技能形式化为**参数化有限状态机（PFSM）子图**：通过参数绑定匹配多条轨迹，从结构层面给出可复用的执行逻辑，而非仅是文本化程序或 LLM 生成的脚本。
- 提出 **SkillDisCo** 框架，包含"蒸馏（Distillation）+ 编译（Compilation）"两阶段：蒸馏阶段通过轨迹归一化、子目标级算子抽取、技能合并三步近似提升函数；编译阶段通过技能规格化与"合成-验证"循环生成可调用、可验证的可执行技能。
- 在 ALFWorld / WebArena 上跨基准、跨模型规模验证：成功率提升、平均步数下降；同时展示了**跨模型技能迁移**（强诱导模型蒸馏的技能可让弱执行模型大幅受益，例如 Qwen3.5-9B 在 ALFWorld 上达 98.5%，超过其诱导器 GPT-4o 的 96.3%）。

## Key Contributions

- Makes the **scope of procedural skill discovery explicit** by restricting it to FSM-defined scenarios, where successful traces are paths in an unknown transition graph and skills have well-defined transition semantics.
- Formalizes procedural skills as **reusable parameterized control-flow subgraphs** under parameter binding, giving a structural (rather than purely textual) target for skill discovery.
- Introduces **SkillDisCo**, a distillation-and-compilation pipeline: distillation normalizes traces, extracts subgoal-level operators, and clusters them into high-coverage PFSM subgraph candidates; compilation turns each candidate into a callable, executable, and verified Python skill via a synthesis-verification loop.
- Demonstrates **broad empirical gains** on ALFWorld and WebArena across agent backbones (ReAct, CodeAct) and model scales (GPT-4o, GPT-5, Qwen3.5), and shows that skills act as **procedural-knowledge distillation**, transferring from a stronger inducer to smaller executors.

## Methodology

SkillDisCo takes a set of successful traces $T^+ = \{\tau_1, \dots, \tau_N\}$ as input and produces a skill library $K = \{K_1, \dots, K_m\}$.

**Formal grounding (PFSM).** An FSM-defined scenario is $M = (S, A, \delta, S_0, S_{goal})$ with deterministic transitions. A PFSM $M = (\tilde{S}, \tilde{A}, \Theta, \tilde{\delta})$ abstracts concrete states/actions into parameterized ones, so traces with different objects, locations, or lengths can instantiate the same control-flow pattern. Each trace is "lifted" into a parameterized trace graph; a skill is a PFSM subgraph shared across many traces. Since neither the true PFSM nor the lifting function $\phi$ is available, SkillDisCo approximates both through a multi-stage pipeline.

**Distillation phase (3 stages).**
1. **Trace Normalization** — An LLM converts each raw trace (interleaved reasoning, tool calls, observations) into a Python-like program $p_i$ with explicit primitive calls, typed symbolic variables, and code-level control flow (loops, branches).
2. **Subgoal-level Operation Extraction** — Each program is decomposed into operators $o = (\nu, \sigma, u, c)$ (name, summary, primitive sequence, code fragment). Single-step operators are discarded; multi-step operators ($|o| \ge 2$) are retained.
3. **Procedural Skill Consolidation** — Operators sharing parameterized control-flow structure are clustered; each cluster approximates a PFSM subgraph and receives a reusability score $r_k \approx \frac{1}{N}\sum_i \mathbf{1}[K_k \preceq \tilde{G}^i]$ estimated from trace coverage. Only high-coverage clusters become skill candidates.

**Compilation phase (2 stages).**
4. **Skill Specification** — Each candidate gets a formal definition: signature (name, typed params, return type), natural-language description, behavioral requirements (preconditions, postconditions, side effects), and metadata (abstraction level, primitive-action amortization, confidence).
5. **Synthesis & Verification** — A Python program $f_k$ is synthesized and verified on held-out tasks for runtime correctness, postcondition satisfaction, and action savings; failures are re-synthesized with feedback (up to $R$ retries), then discarded if still failing.

**Evaluation setup.** Strict induction/evaluation splits: ALFWorld (200 induction / 134 unseen eval), WebArena (406/406 of 812). Baselines: ReAct, CodeAct (base agents), AWM_offline, ASI_offline (prior induction methods). Metrics: success rate (SR), average agent turns; token/cost reported in appendix. Inducer and evaluator use GPT-4o unless noted.

## Results & Highlights

- **End-to-end (Table 1, GPT-4o):** On ALFWorld, SkillDisCo + CodeAct reaches 99.3% SR (+3.1% over CodeAct's 96.3%) at 3.2 turns (−11.3%); +ReAct reaches 92.4% (+12.7%) at 8.8 turns (−54.5%). On WebArena, +ReAct reaches 29.1% (+21.6% over 23.9%) at 5.1 turns (−13.1%), beating ASI_offline by 4.5 absolute points.
- **Compact, high-coverage library (Table 3):** 5 skills on ALFWorld vs ASI's 110 (and 20 vs 146 on WebArena), yet each call covers more primitive steps (5.6 vs 0.2; max compression 33 vs 10) — skills encode reusable procedures, not shallow fragments.
- **Execution reliability:** Skill-call error rate drops from 75.3% → 0.0% (ALFWorld) and 33.9% → 21.5% (WebArena) versus ASI_offline.
- **Cross-model transfer (Table 2):** The same GPT-4o-induced library improves every target model. Largest gains on small open-source backbones — Qwen3.5-9B +80.8% SR on ALFWorld (reaching 98.5%), Qwen3.5-4B +85.3% on WebArena — evidencing procedural-knowledge distillation from strong inducer to weak executor.
- **Ablations (Table 4):** Removing distillation (per-trace skills) collapses SR from 99.3% → 53.0% and inflates turns (3.2 → 11.5) — distillation is the main driver. Removing compilation (NL procedures only) modestly drops SR (99.3 → 97.0%) but inflates context tokens; compilation externalizes execution from the prompt.

## Authors & Credibility

- Authors are from **Microsoft Research** (Zhongxin Guo, Danrui Qi, Peng Cheng, Yongqiang Xiong) and Beijing Foreign Studies University (Hanwen Gu), lending industrial-scale credibility and access to frontier models (GPT-4o/5, pricing dated 23 May 2026).
- The paper builds directly on a coherent lineage — AWM (Wang et al., 2024c) and ASI (Wang et al., 2025) — and positions itself precisely against them, with apples-to-apples splits and identical induction signals.
- Empirical claims are supported by strict train/eval splits, per-episode token/cost breakdowns (Appendix A), full ablation tables (Table 7), and reproducible prompt schemas (Appendix B). However, no code/data release is referenced in the main text.

## Critical Assessment

- **Novelty (Moderate-High):** The PFSM formalization is a genuine conceptual contribution — it gives procedural skill discovery a structural target rather than treating each trace as an independent routine. The distillation-then-compile split and corpus-level clustering (vs per-trajectory induction in Voyager/ASI) are well-motivated. That said, each stage still leans on an LLM to do the actual heavy lifting (normalization, segmentation, clustering, codegen), so the "structural" framing is somewhat aspirational; the pipeline is, in practice, a well-engineered LLM-orchestration stack approximating PFSM structure.
- **Rigor (Moderate-High):** Strict induction/eval splits, identical signals across methods, multi-benchmark + multi-model evaluation, ablations isolating both phases, and error-rate + compression statistics all support the headline claims. The WebArena gains are modest in absolute terms (≈5 points) and the base SR is low (≈20–30%), so the transfer story on the harder benchmark is less conclusive.
- **Reproducibility (Moderate):** Prompt schemas and pricing assumptions are documented, but there is **no link to code/data**, the clustering/scoring details (reusability score estimation) are lightly specified, the verification set sizes and retry budget $R$ are underspecified, and reliance on proprietary frontier models (GPT-4o/5) limits independent reproduction. Skill counts (5, 20) and clustering thresholds are not fully pinned down.
- **Significance (Moderate-High):** The cross-model transfer result — a strong inducer's skills lifting a 9B open model above the inducer's own performance — is the most striking and useful finding, suggesting a practical "compile once, deploy anywhere" pattern for agent skill libraries.
- **Red flags / open questions:**
  - Only **successful** traces feed distillation; in low-success domains the corpus may be too sparse (authors acknowledge this).
  - The PFSM assumption excludes **non-deterministic or open-ended** tasks; generalization to truly open-ended agentic settings is unproven.
  - Pipeline quality is bottlenecked by the compiler LLM's capability (authors acknowledge).
  - The "FSM-defined" criterion is somewhat retrofitted — ALFWorld and WebArena are structured enough to qualify, but the paper does not test how to detect or validate the FSM property on a new domain.
  - WebArena SR remains low overall; gains could partly reflect that the few high-value reusable procedures dominate the limited successes.

## Strengths

- Clean, well-motivated formal grounding (PFSM, parameter binding) that distinguishes the work from NL-only or per-trajectory skill approaches.
- Compact skill libraries (5 / 20 skills) that nonetheless deliver strong coverage and very high per-call compression.
- Striking cross-model transfer: skills induced by GPT-4o let Qwen3.5-9B exceed the inducer on ALFWorld — a compelling procedural-distillation result.
- Honest ablations that clearly attribute gains to distillation (consolidation) rather than to compilation cosmetics.
- Verified execution (0% skill-call error on ALFWorld) addresses a real weakness of prior text-based skill libraries.

## Limitations

- Applicable only to **procedural** tasks with reusable structure; no benefit for pure NLP/linguistic tasks (authors acknowledge).
- Depends on the compiler LLM's reasoning/codegen quality; degrades with weaker models.
- Requires a non-trivial corpus of **successful** traces — sparse in low-success domains.
- FSM-defined-scenario assumption is restrictive and not always easy to verify for new environments.
- No public code/data release mentioned; clustering and verification details underspecified for exact reproduction.
- WebArena absolute SR remains modest (~29%); transfer gains on the harder benchmark are smaller and noisier than on ALFWorld.

## 相关性

该工作对构建可长期积累、可跨模型复用的智能体技能库具有直接参考价值，尤其契合"经验复用 + 技能蒸馏"方向的研究与工程实践；其"强模型诱导、弱模型执行"的跨模型迁移范式，对降低部署成本和提升小模型可用性有现实意义。

## Questions for Further Exploration

- How can the framework be extended to **non-deterministic or partially observable** environments where the strict FSM assumption breaks?
- What is a principled, model-agnostic procedure to **detect whether a new domain is FSM-defined** before committing to SkillDisCo?
- How does performance scale with the **size and diversity of the induction corpus**, and is there a diminishing-returns or saturation point?
- Can **failed traces** (currently discarded) be incorporated — e.g., as negative evidence — to sharpen skill preconditions/postconditions?
- How robust are the compiled skills to **environment drift** (e.g., website UI changes in WebArena), and can the verification loop be re-triggered online?
- Could the clustering / reusability-scoring stage be replaced by a **learning-based** matcher to reduce reliance on the compiler LLM?
- Does the cross-model transfer result hold when the **executor is much weaker than the smallest tested model** (e.g., 1–3B), or does selection-of-the-right-skill itself become the bottleneck?
