Scientific discovery as meta-optimization: a combinatorial optimization case study

Yuan-Hang Zhang, Chesson Sipling & Massimiliano Di Ventra · UC San Diego·2026-06-25

Key finding

Co-evolving LLM-generated objectives with solutions via Kendall-τ-weighted Borda consensus cuts MemComputing 3-SAT scaling from N2.51 to N1.33.

Key metric
~67×
median-step speedup at N=1810 (95,503 vs 6,369,516). Scaling exponent 2.51 → 1.33, from 414 solver designs under 42 co-evolving objectives.
Why this matters

A principled, cheap rank-statistic consensus over noisy LLM objectives attacks reward hacking directly — and transfers to any automated search whose true metric is hard to specify.

Critical edge

⚠ The headline gain rests on a single unseeded search — no evidence design 340 isn't a high-variance draw — and the discovered solver gets HEBO tuning the hand-tuned baseline never received.

1Methodology

A closed loop of four GPT-5.2 agents with Pydantic-enforced structured outputs: a meta-agent (reviews objective correlations every 20 designs, assigns weight multipliers to break echo chambers), an objective agent (writes Python proxy objective functions), a planner agent (reads Monte-Carlo-Graph-Search-ranked designs and proposes directions), and a designer agent (writes and runs solver code).

The core consensus objective aggregation fuses the proxy portfolio: per-objective ranks R_ij are combined with weights w_i = max(median τ_i,0)·λ^(t−t_i) (Kendall's τ median-agreement × age decay λ=0.9), then summed as a weighted Borda count, C_j = Σ w_i·R_ij/(n−1). Outlier objectives that disagree with the majority are auto-suppressed.

Exploration uses Monte-Carlo Graph Search over a DAG of designs (visit counts propagate via BFS, UCB r_j + c·√(N/(1+n_j))). A three-tier multi-fidelity schedule (N≤640→≤1280→≤2560; 1→5→30 min) with rule-based promotion (top 50% / top 10%) evaluates 100 random planted 3-SAT instances per size at α=4.3. The domain is digital MemComputing-machine 3-SAT solvers; the winning design (340) innovates only in the short-term-memory equation via a five-gate "release term."

Consensus beats any single proxy: only 17% of objectives rank design 340 first, yet agreement-and-recency weighting lifts it to #1. Three reward-hacking episodes (small-N over-optimization, the anti-consensus objective 17, and a pre-meta-agent echo chamber) were detected and down-weighted. Mean pairwise objective τ fell from 0.588 to 0.399 as objectives diversified across four phases.

Strengths

  • Directly engages the hardest problem in LLM-driven science — objective specification and reward hacking — not just scaling the search loop.
  • Consensus aggregation is principled, interpretable, and cheap: rank statistics, no learned reward model, with rich diagnostics (τ matrix, PCA, per-objective agreement).
  • Exceptionally candid about failure modes — three reward-hacking episodes and the echo-chamber pathology are documented, not hidden.
  • Proper uncertainty reporting on scaling exponents; 100 instances per problem size.
  • High-quality release: code + 414-design DB + 42 objectives + prompt templates (MIT).

Limitations

  • Single-search variance unquantified — the central result rests on one run; no seed sensitivity analysis.
  • Confounded baseline: discovered designs get HEBO tuning the hand-tuned baseline never received; isolated contribution is not separated.
  • Narrow benchmark: only planted random 3-SAT at α=4.3; no industrial/structured instances, no cross-domain test despite a "problem-agnostic" claim.
  • Echo-chamber blind spot: the meta-agent is a hand-set, heuristic patch, not a principled fix.
  • DMM-specific skepticism: no CDCL/SLS baselines to contextualize absolute instance difficulty.

?Questions for further exploration

  1. How much of the 2.51→1.33 gain survives if the baseline also receives HEBO tuning and the search is repeated across seeds? Is design 340 robust, or a high-variance draw?
  2. Does the framework generalize beyond planted 3-SAT — to structured/industrial SAT, other NP-hard problems, or non-DMM solver families — and how does it compare to modern CDCL/SLS baselines?
  3. Can the meta-agent's manual weight multipliers be replaced by a principled diversity- or information-theoretic mechanism that removes the backstop?
  4. Is Kendall-τ-median + weighted Borda optimal, or do Kemeny ranking / Bradley-Terry / learned reward ensembles yield more robust consensus?
  5. Does design 340's five-gate "release term" admit a human-interpretable dynamical-systems explanation, or is it an opaque LLM heuristic?
  6. How sensitive are results to the LLM backbone (GPT-5.2)? Would a smaller open model produce comparable objective diversity, and does echo-chamber risk worsen?
