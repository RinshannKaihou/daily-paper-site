> **一句话总结：** 本文提出 PAVE 标注协议，利用故障注入的已知干预，从"结果标签"转向"过程级因果监督"，构建了首个带逐步因果传播路径标注的跨系统 RCA 基准 OpenRCA 2.0（500 例），揭示了"非接地诊断"（ungrounded diagnosis）这一被传统结果级评测隐藏的推理失败模式：11 个前沿大模型平均仅在 20.7% 的案例中精确恢复根因集。

> **TL;DR:** The paper introduces PAVE, an annotation protocol that converts backward root cause analysis into forward verification from a known fault-injection intervention, producing OpenRCA 2.0 — the first cross-system RCA benchmark with step-wise causal propagation-path labels. Evaluating 11 frontier LLMs exposes the "ungrounded diagnosis": agents name a correct root-cause service in 76.0% of cases but ground it in a verified causal path in only 61.5%, a reasoning gap invisible to outcome-only metrics.

## 核心贡献

- **PAVE 标注协议**：提出 "Path Annotation via Verified Effects"，将因果路径重建从难以求解的逆向推断问题转化为基于已知干预的前向验证任务。利用智能体永远看不到的信息不对称——即触发级联的干预本身 `do(v_root)`——来验证下游效应。
- **三重联合准入条件**：一条候选因果路径必须同时满足三个条件才被接受：结构机制一致性（依赖图拓扑 + 传播规则集）、统计显著性偏离（相对注入前基线的分布检验）、上下游时序对齐。任何单一信号都不充分。
- **OpenRCA 2.0 基准**：首个带逐步因果标注的跨系统 RCA 基准，覆盖 3 个架构迥异的微服务系统（TrainTicket、OTel Demo、DeathStarBench Hotel Reservation）和 27 种故障类型，共 500 个可评估实例，平均每例 7.5 条因果边。
- **过程级评测指标**：引入 Path Reachability、Node F1、Edge F1 等指标，评分配路径的推理过程而非仅评根因身份。
- **揭示"非接地诊断"失败模式**：在 11 个前沿大模型上发现，Agent 平均仅在 20.7% 案例精确恢复根因集；放宽标准后 76.0% 能识别至少一个正确根因服务，但仅 61.5% 能将其接地于验证过的因果路径，约 1/5 的"正确识别"缺乏因果支撑。

## Key Contributions

- **PAVE protocol**: A forward-verification annotation scheme that exploits the information asymmetry between the agent (backward abductive RCA from telemetry alone) and the annotator (who has the recorded fault injection `do(v_root)`), turning an ill-posed inverse problem into a well-posed verification task.
- **Tri-condition admission gate**: A path is admitted only when structural conformance (topology + rule set R), statistically significant deviation vs. a pre-injection baseline (Z-score / percentile tests), and upstream-to-downstream timing alignment hold jointly — no single signal suffices.
- **OpenRCA 2.0 benchmark**: 500 evaluable instances across 3 architecturally distinct microservice systems and 27 chaos fault types, averaging 7.5 verified causal edges per instance; the first cross-system RCA benchmark with step-wise causal annotations and an explicit SLO surface.
- **Process-level metrics**: Path Reachability (PR), Node F1, and Edge F1 that grade the agent's reasoning path rather than only the root-cause identity, complementing outcome metrics (Exact Match, F1, AnySvc).
- **Empirical finding — the ungrounded diagnosis**: Across 11 frontier LLMs, exact root-cause-set recovery succeeds in only 20.7% of cases; AnySvc reaches 76.0% but PR only 61.5%, exposing that ~14.5pp of "correct" identifications are not grounded in any verifiable propagation path.

## Methodology

**Core idea — forward verification.** Standard RCA is a backward abductive problem: from telemetry (traces, metrics, logs) infer the dependency structure, root cause, and propagation path simultaneously. Fault-injection recordings give the annotator the intervention `do(v_root)` that the agent never sees. Because `{O_obs} ⊂ {O_obs, do(v_root)}`, knowing the cause converts inference into verification: confirm that the observed anomalies are consistent with the known intervention propagating through the dependency graph G.

**PAVE pipeline (two phases).** Inputs: the dependency graph G, raw telemetry O(t) over the injection window, and the recorded intervention `do(v_root)`. Output: a verified path set Π*.

- **Phase 1 — Structural Pruning.** Telemetry is projected onto a discrete state alphabet Σ via a node-type-aware mapping (e.g., HighLatency for services, CPU_Throttled for pods). A propagation rule set R admits only transitions between (type, state) pairs backed by a known mechanism (RPC timeout cascade, resource contention, etc.). A constrained DFS over a time-expanded heterogeneous graph whose nodes are (component, state, time) triples unrolls circular dependencies into acyclic chains, yielding the candidate set Π_cand.
- **Phase 2 — Causal Verification.** For each injection, a fresh warmed-up instance provides a Reference Baseline D_base approximating P(· | do(v_root=∅)). Each node along a candidate path is tested against the baseline with distributional tests (Z-score for Gaussian metrics, percentile-based for heavy-tailed). A path is verified only if (i) every node deviates beyond its baseline screen, (ii) every consecutive edge conforms to R, and (iii) each downstream anomaly onsets no earlier than its upstream cause — all simultaneously. Injections whose cascade clears the joint screen end-to-end are admitted; others are dropped (including silent injections).

**Agent design.** All evaluated agents use a tool-augmented ReAct-style architecture (from the Deep Research agent). To isolate LLM reasoning from tooling effects, all 11 models share an identical tool suite over the released telemetry and identical prompts; no dependency graph is provided. Each diagnosis returns structured output: root-cause claims and propagation edges, each paired with a re-executable SQL evidence query. The model set spans Anthropic, Google, ByteDance, Xiaomi, Zhipu, Moonshot, DeepSeek, Alibaba, Tencent, and MiniMax (Claude Opus 4.7, Gemini 3.1 Pro, Claude Sonnet 4.6, MiMo 2.5 Pro, GLM 5.1, Kimi K2.6, DeepSeek V4 Pro, Qwen3.6-Max, Hy 3.0 Preview, Seed 2.0 Pro, MiniMax M2.7).

**Annotation reliability.** A two-annotator manual audit on N=100 uniformly random instances reports 94% case-level agreement on the validity of G*; all 6 disagreements were under-coverage (missing weakly supported edges), with zero flagged spurious included edges — a precision-favoured trade-off the authors accept for a process-level reference.

## Results & Highlights

- **Outcome scores are low and understate the failure.** Mean outcome F1 across 11 models = 34.1%; Exact Match = 20.7%; AnySvc (name at least one correct service) = 76.0%. The drop from AnySvc (76.0%) to Recall (33.2%) is dominated by mislabelled fault kinds on services the agent did identify.
- **The ungrounded diagnosis (Finding 1).** Even the most lenient process metric PR is satisfied in only 61.5% of cases, so in ~14.5pp of cases the agent names a correct service without drawing any verifiable path to the symptom. Conditional PR/AnySvc separates models that outcome-only grading rates as comparable: Qwen3.6-Max 71.8/79.2 ≈ 91% vs. Sonnet 4.6 52.6/79.4 ≈ 66%.
- **Edge F1 universally trails Node F1 (Finding 2).** Mean Edge F1 (43.4%) lags mean Node F1 (62.2%) by 18.8pp for every model. Gemini 3.1 Pro shows the widest split: 67.9% Node F1 but 36.8% Edge F1. Identifying participating services (classification-like) is easier than ordering them into a directed propagation path (reasoning about inter-service dependencies).
- **No single backbone dominates (Finding 3).** Per-column extrema are spread across four different models; the appropriate ranking depends on downstream verification cost.
- **Three recurring failure modes** (from trajectory analysis on a stratified sample): (1) premature commitment under under-exploration (stops after one plausible cause, misses co-occurring subtle faults); (2) presence bias (treats absence of activity as evidence of normality — e.g., killed-pod silence read as "no errors, no fault"); (3) salience capture (converges on the loudest signal, even pure downstream amplification or a misleading SEVERE log line).
- **Case study.** Claude names the correct root cause (ts-basic-service) but skips the intermediate ts-travel2/ts-travel services, hallucinating a direct edge — Edge F1 44.4%, scored as a hit under AnySvc but a reasoning failure under process-level eval.
- **Per-system spread wider than cross-model spread.** Hotel Reservation approaches outcome saturation for top models; TrainTicket leaves a 30+pp gap on process-level metrics.

## Critical Assessment

- **Novelty: High.** The core reframing — using the fault-injection intervention as ground truth to convert backward RCA into forward verification — is a genuinely clever exploitation of information asymmetry, and the tri-condition admission gate is well-motivated. The concept of "ungrounded diagnosis" and the process-level metrics are a meaningful conceptual advance over outcome-only RCA benchmarks.
- **Rigor: Solid but with caveats.** The pipeline is built on a hand-curated rule set R and manually-calibrated anomaly screens. The authors are admirably explicit (Section 5) that R may silently prune mechanisms outside its vocabulary and the per-node screen is tuned for low false negatives. The conjunctive gate mitigates end-to-end error, and the audit reports zero spurious edges, but the 94% agreement on only 100 cases (with all disagreements being under-coverage) is a relatively small validation sample for a 500-instance benchmark; the "precision-favoured" framing conveniently reframes under-coverage as a feature, which warrants scrutiny.
- **Reproducibility: Moderate-to-Good.** Telemetry, tool definitions, prompts, schema, and a datasheet are promised in appendices; the SLO surface is exposed as a configurable knob. However, the full rule set R, baseline-calibration split, and the raw 8,137-run pool release details determine true reproducibility. Re-running fault injection campaigns across 3 testbeds is itself non-trivial for outside groups.
- **Significance: High for the RCA/LLM-agent community.** It plausibly resets the evaluation bar for LLM-based RCA and, per the related-work framing, opens a path toward Process Reward Models for reliability — though training PRMs is explicitly left to future work.
- **Red flags / honest concerns.**
  - The headline "20.7% exact match" averages a heterogeneous 11-model pool including very recent/closed models (Claude Opus 4.7, Gemini 3.1 Pro) whose names/dates (June 2026) are at the edge of verifiability; absolute numbers should be read as relative comparisons, not absolute capability ceilings.
  - 500 instances is modest by modern benchmark standards, and TrainTicket alone contributes 320 (64%), so the "cross-system" claim rests heavily on one system; Hotel Reservation contributes only 142 and OTel Demo 38.
  - No dependency graph is provided to the agent, which may artificially depress Edge F1 and conflate "the agent can't reason about dependencies" with "the agent wasn't given topology to check against." This is a deliberate isolation choice but limits ecological validity.
  - The agent architecture (ReAct/Deep Research) and prompt templates are held fixed; results may not transfer to other agentic designs (e.g., those that explicitly query topology).
  - Mean propagation path length is short (2.7 edges), so process-level difficulty is bounded; deeper cascades may amplify or change the failure-mode mix.

## Strengths

- Conceptually elegant reframing of annotation as forward verification from a known intervention.
- Tri-condition (structural + statistical + temporal) admission gate is principled and the joint-conjunction design is well-justified for avoiding spurious edges.
- First benchmark with genuine step-wise causal propagation-path labels and explicit process-level metrics (PR, Node F1, Edge F1).
- Honest, detailed failure-mode characterization (three named patterns) grounded in trajectory analysis and a concrete case study.
- Transparent threats-to-validity section that specifies input assumptions, internal pipeline weaknesses, and external-transfer boundaries.
- Strong model coverage (11 frontier LLMs, 9 organizations, closed + open source) and a controlled experimental design (identical tools/prompts).

## Limitations

- Small benchmark size (500 instances) dominated by TrainTicket (64%); thin coverage of OTel Demo (38) and Hotel Reservation (142).
- Reliance on a manually-authored propagation rule set R that may omit mechanisms; generalization to unseen fault types is uncertain.
- Validation audit limited to 100 instances; all disagreements are under-coverage, which is reframed as acceptable but bounds recall of the ground truth.
- Agents are denied the dependency graph and use a single fixed architecture/prompt set, restricting generalizability of the model rankings.
- Short propagation depth (mean 2.7 edges); behavior on deep, production-scale cascades is untested.
- Authors explicitly note production-scale fidelity is neither a goal nor achievable in controlled testbeds; transfer to real deployments must be calibrated, not assumed.

## Relevance

本文对"大模型能否可靠地诊断复杂系统故障"这一核心问题给出了更严格的评测范式，对从事 AIOps、LLM 智能体评估与因果推理的研究者高度相关。其"过程级监督"思路（类比数学/代码领域的 PRM）也为后续在可靠性场景训练过程奖励模型指明了方向。

## Questions for Further Exploration

- How do the ungrounded-diagnosis rates and Edge-vs-Node F1 gaps change if the agent is given the dependency graph G, or is allowed to query topology on demand?
- Can the step-wise causal labels be used to train a Process Reward Model for RCA, and does process supervision close the AnySvc→PR gap?
- Do the three failure modes (premature commitment, presence bias, salience capture) respond to targeted interventions (e.g., explicit "absence-as-signal" prompting, multi-hypothesis verification)?
- How do results scale to deeper cascades (path length >5) and to production-scale topologies beyond the three testbeds?
- Is the relative model ranking stable under deliberately restricted telemetry (subsampled traces, missing modalities, truncated windows), which the authors flag as a controllable but unexplored axis?
- Could the PAVE protocol be extended to non-injected real incidents (e.g., using post-hoc change-point detection as a pseudo-intervention), or does it fundamentally depend on controlled fault injection?
- How sensitive are the headline numbers to the anomaly-screen calibration and the SLO-surface threshold, which are now exposed as configurable knobs?
