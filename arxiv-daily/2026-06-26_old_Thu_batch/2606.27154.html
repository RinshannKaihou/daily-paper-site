OpenRCA 2.0: From Outcome Labels to Causal Process Supervision

Aoyang Fang, Yifan Yang, Jin'ao Shang, Qisheng Lu, et al. · 2026-06-26 · NeurIPS 2026 Datasets & Benchmarks

PAVE turns backward root-cause analysis into forward verification from known fault injections — exposing the "ungrounded diagnosis" that outcome-only metrics hide.

Key metric
20.7%
exact root-cause-set recovery across 11 frontier LLMs. AnySvc reaches 76.0%, yet Path Reachability only 61.5% — ~14.5pp of "correct" answers are causally ungrounded.
Why this matters

Outcome-only grading credits agents that merely name the right service; process-level evaluation shows the reasoning behind many "correct" answers is causally ungrounded.

Critical edge

Agents are denied the dependency graph and 64% of instances come from one system — so the numbers may conflate missing topology with weak reasoning and overstate cross-system generality.

Methodology

Standard RCA is a backward abductive problem: infer dependency structure, root cause, and propagation path simultaneously from telemetry. PAVE exploits an information asymmetry — fault-injection recordings give the annotator the intervention do(v_root) that the agent never sees. Because the observations are consistent with the known cause, inference becomes verification: confirm anomalies are consistent with the intervention propagating through the dependency graph.

Phase 1 — Structural pruning. Telemetry is projected onto a discrete state alphabet via a node-type-aware mapping (e.g. HighLatency, CPU_Throttled). A rule set R admits only transitions backed by a known mechanism. A constrained DFS over a time-expanded heterogeneous graph unrolls circular dependencies into acyclic chains, yielding candidate paths.

Phase 2 — Causal verification. Each node along a candidate path is tested against a fresh baseline instance (approximating P(· | do(v_root=∅))) via distributional tests. A path is admitted only when (i) every node deviates beyond its screen, (ii) every edge conforms to R, and (iii) downstream anomalies onset no earlier than upstream causes — jointly.

All 11 models share identical tools, prompts, and released telemetry under a ReAct/Deep-Research architecture; no dependency graph is provided. Each diagnosis returns root-cause claims and propagation edges, each paired with a re-executable SQL evidence query. A two-annotator audit on N=100 reports 94% case-level agreement with zero spurious edges.

Strengths

  • Elegant reframing of annotation as forward verification from a known intervention.
  • Tri-condition (structural + statistical + temporal) admission gate is principled; joint conjunction avoids spurious edges.
  • First benchmark with genuine step-wise causal propagation-path labels and process-level metrics (PR, Node F1, Edge F1).
  • Honest failure-mode taxonomy (premature commitment, presence bias, salience capture) grounded in trajectory analysis.
  • Transparent threats-to-validity; 11 frontier LLMs across 9 orgs with identical tools/prompts.

Limitations

  • 500 instances, dominated by TrainTicket (64%); thin OTel Demo (38) coverage.
  • Hand-authored rule set R may silently omit mechanisms; generalization to unseen faults uncertain.
  • Audit on only 100 instances; all disagreements are under-coverage, reframed as acceptable.
  • Agents denied the dependency graph and use a fixed architecture/prompt — rankings may not transfer.
  • Mean path depth only 2.7 edges; behavior on deep production-scale cascades untested.

Questions for further exploration

  1. How do ungrounded-diagnosis rates and Edge-vs-Node gaps shift if the agent is given the dependency graph G or may query topology on demand?
  2. Can the step-wise causal labels train a Process Reward Model for RCA, and does process supervision close the AnySvc → PR gap?
  3. Do the three failure modes respond to targeted interventions (explicit "absence-as-signal" prompting, multi-hypothesis verification)?
  4. How do results scale to deeper cascades (path length >5) and production-scale topologies beyond the three testbeds?
  5. Could PAVE extend to non-injected real incidents (post-hoc change-point detection as a pseudo-intervention), or does it fundamentally require controlled injection?
  6. How sensitive are the headline numbers to anomaly-screen calibration and the SLO-surface threshold, now exposed as configurable knobs?
