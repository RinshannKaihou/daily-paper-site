> **一句话总结：** 本文把"记忆访问"与"记忆深度"区分开来——检索负责取回事实，而由惊讶度×价值度门控的稀疏 LoRA 固化（EVAF）负责让目标导向行为在上下文卸载后仍持久存在；在作者提出的 loop-drift 协议中出现了"深度翻转"现象：RAG 赢浅层事实，EVAF 赢目标持久与卸载后恢复。

> **TL;DR:** Long-running language agents need not only memory access (what can be retrieved) but also "memory depth" (what keeps shaping behavior after the working context is unloaded). The authors propose EVAF, a surprise×valence gated LoRA consolidation mechanism, and a synthetic loop-drift protocol. The key finding is a "depth flip": RAG wins shallow factual recall (0.956–0.973) while EVAF wins goal persistence and post-unload recovery (0.812–0.904) with only 2–3 writes per 200 events. Selection and actuation are shown to be separable control factors.

## 核心贡献

- **概念重构（memory access vs. memory depth）**：明确将"能在查询时取回的事实"与"在上下文被清空后仍持续影响行为的目标导向倾向"区分开来，并形式化为四个探针层级（浅层事实、含噪旧事实、参数化目标倾向、卸载后恢复）。
- **loop-drift 受控协议**：构造合成用户流（含稳定目标、分心事件、瞬时反向请求、冲突、sibling-user 污染、调度事实笔记），同时控制四个稀有组合——显式干扰、持久检索索引、可观测的上下文卸载、事实型与目标型探针分离——使"行为是否在卸载后改变"成为可度量的判别量。
- **EVAF 机制**：用 token 负对数似然的惊讶度 × 与用户目标/偏好的嵌入相似度作为门控分数 `gt = σ(ks(st−τs))·σ(kv(vt−τv))`，只让行为相关事件进入小缓冲区，缓冲区满后用 replay + L2 anchor 更新 LoRA。
- **"深度翻转"实证**：GPT-2 / TinyLlama 上 RAG 在浅层事实 0.956–0.973 取胜，而 EVAF 在目标持久与卸载恢复上达 0.812–0.904（RAG 仅 ~0.396/0.394），仅 2–3 次参数写入/200 事件。
- **选择 ≠ 稀疏写入**：matched random gate（相同写入数、相同在线动力学）在 GPT-2 上仍输给 EVAF（goal/post 0.790/0.763 vs 0.590/0.619），证明选择信号本身有效；Mistral-7B 在 actuation 失配时翻转，揭示 selection 与 actuation 的非对称耦合。
- **可分离因子分析**：fixed-inner 控制器跨三模型表明 inner-loop 写入强度是独立的、模型相关的控制维度；过度 actuation 既损害持久性也损害选择性。
- **诚实的边界诊断**：在公共 Memora 事件流上做外部诊断，stale-memory invalidation 仅方向性正向、不显著（91→95/222，McNemar p=0.57），明确暴露删除/更新有效性为未解决问题。

## Key Contributions

- **Reframes the problem as "memory depth" vs. "memory access."** Distinguishes shallow, retrievable facts from durable goal-conditioned tendencies that must persist through interference and context unload, drawing on complementary-learning-systems motivation.
- **Introduces the loop-drift protocol**, a synthetic controlled stress test that co-controls four rarely-combined properties: explicit interference, durable retrieval index, observable context unload, and separate factual vs. goal-conditioned probes.
- **Proposes EVAF**, a surprise-times-valence gated LoRA consolidation mechanism with a small write buffer, replay, and an L2 anchor that admits only behavior-relevant events.
- **Demonstrates a "depth flip"** across GPT-2 and TinyLlama: retrieval owns shallow facts (0.956–0.973) while EVAF owns goal persistence and post-unload recovery (0.812–0.904) with only ~2.4–2.6 writes per 200-event stream.
- **Shows selection is not reducible to sparsity** via a matched random gate that preserves write count and online dynamics yet loses to EVAF on GPT-2.
- **Factorizes consolidation into separable selection and actuation controls** with asymmetric online feedback, and shows via fixed-inner audits that write strength is a distinct, model-dependent factor that, when miscalibrated (e.g., Mistral-7B), can invert the matched-gate comparison.
- **Honestly probes the boundary** with public Memora streams, reporting a non-significant (p=0.57) directional result that flags stale-memory deletion/update as open work.

## Methodology

- **Probe layering.** A long-running memory stream `x1,...,xT` is probed at four depths: shallow episodic access (recent explicit fact, retrieval's natural strength), noisy episodic access (older fact after same-key interference), parametric tendency (stable goal still shaping behavior after long interference), and post-unload recovery (goal probe repeated after working context is cleared while the retrieval index stays intact).
- **EVAF selection gate.** For each event `xt`, compute surprise `st` from token NLL and valence `vt` from embedding similarity to the user's durable goal/preferences; admission score `gt = σ(ks(st−τs))·σ(kv(vt−τv))`; if `gt > τw = 0.5`, the event enters a fixed-size buffer. When the buffer fills, a LoRA adapter is updated on the buffer plus replay of previously consolidated events, with an L2 anchor. Thresholds use per-user median warmup; fixed slopes `ks=1, kv=10`, `τv=0.5`.
- **Actuation controllers.** Primary diagnostic is fixed-inner (vary only inner LoRA steps: fixed-1/2/3). A behavior-margin controller is included as a proof-of-concept (writes until the goal/preference margin moves by a target amount under KL/L2 caps) but explicitly stated as not load-bearing.
- **Baselines.** Frozen, Summary, RAG (durable embedding index, top-3 cosine retrieval, not cleared by unload), Naive-LoRA (writes every event, 200 writes/stream), EVAF, and Routed EVAF+RAG (factual probes → retrieval, goal probes → EVAF).
- **Matched random gate.** Same write count and identical online write dynamics as EVAF but randomly admitted events, isolating which events are selected.
- **Loop-drift generator.** 10 users, 200 events/user, mix of stable goal/preference reminders, distractors, transient opposite requests, conflicts, sibling-user contamination, scheduled factual notes; GPT-2/TinyLlama averaged over 4 seeds; Mistral-7B also 4 seeds.
- **Metrics.** Matched continuation scoring (higher better for short-fact/long-fact/goal/post-unload; lower better for contamination and transient overwrite); adapter cost by write count and LoRA L2 drift. A "write" = one buffer-consolidation trigger.
- **Mechanistic model.** Continual adapter update `∆θevaf ∝ −Σ_{t∈A} ∇ℓt(θ) − λ(θ−θ0)` over admitted set A, vs naive `∆θnaive ∝ −Σ_t ∇ℓt(θ)`. The gate is useful only if `gt` correlates with behavior-relevant events; the matched-gate ablation tests exactly this.

## Results & Highlights

- **Depth flip (Table 2, Figure 1).** TinyLlama: RAG 0.973/0.660/0.396/0.394 vs EVAF 0.483/0.565/0.833/0.812 on short-fact/long-fact/goal/post-unload with 2.6 writes and L2 21.5. GPT-2: RAG 0.956/0.733/0.398/0.394 vs EVAF 0.463/0.558/0.904/0.900 with 2.4 writes. EVAF is near chance on short facts as expected (goal-conditioned gate rejects off-topic notes).
- **Naive-LoRA is not enough.** 200 writes with high L2 drift (67.5 TinyLlama, 119.3 GPT-2) does not match EVAF on the goal layer; at 7B, indiscriminate writing is actively harmful (Naive-LoRA collapses to 0.333 goal, below the 0.500 chance baseline).
- **Routed EVAF+RAG** recovers shallow factual access (0.975 TinyLlama short-fact) while largely preserving the goal layer, supporting division of labor.
- **Matched gate (selection ≠ sparsity).** GPT-2: EVAF 0.790/0.763 goal/post vs random 0.590/0.619 across all 4 seeds. TinyLlama weak/mixed (0.625/0.581 vs 0.596/0.569). On Mistral-7B the comparison reverses under miscalibrated actuation (EVAF 0.362 vs random 0.688 goal) — a diagnostic of selection–actuation coupling, not gate failure; EVAF still retains lowest sibling contamination.
- **Actuation audit (Table 3).** Fixed-inner controllers reduce drift and improve goal/post vs the miscalibrated Audit-5; no universal optimum. GPT-2 strongest at Fixed-1 (0.948/0.956), TinyLlama Fixed-2 cleaner selectivity (contam 0.637), Mistral Fixed-2/Fixed-1 recover (0.796/0.775 and 0.919/0.938) with Fixed-1 contamination saturating at 1.000.
- **Asymmetric feedback (Table 1).** On Mistral, writes increase as inner-step strength decreases — weaker actuation leaves later events surprising enough to re-trigger the gate, monotonic across all 4 seeds.
- **Memora boundary (Table 4).** Forgetting-absence 91/222 → 95/222, McNemar p=0.57 (non-significant); high-mutation slice also non-significant. Stale-memory invalidation remains unsolved.

## Authors & Credibility

- Single author, Haoliang Han, affiliated with the Institute of Biomedical Strategy, China Pharmaceutical University (a non-traditional lab affiliation for an LLM-memory paper, which slightly affects perceived institutional signal).
- The paper is unusually disciplined in hedging: it explicitly labels results as "load-bearing" vs. "proof-of-concept," separates independent audit runs from main-table rows (e.g., warning that Audit-5 ≠ Table 2 EVAF due to independent RNG and probe instantiation), and pre-registers the high-mutation slice. The author openly reports null/negative results (Mistral matched-gate inversion, Memora non-significance) rather than hiding them.
- The mechanism is grounded in well-established continual-learning and complementary-learning-systems literature (EWC, experience replay, LoRA, CLS). Citation list is relevant and includes 2026 concurrent work on parametric memory.
- No external validation/SOTA claim is made on Memora; claims are scoped narrowly to a mechanism under a controlled protocol, which is intellectually honest.

## Critical Assessment

- **Novelty.** The core reframing (memory access vs. memory depth) is a useful conceptual contribution, and the loop-drift protocol with a co-controlled context unload is a genuinely novel evaluation design. EVAF itself is a fairly standard combination (surprise gating × valence similarity × LoRA × replay × L2 anchor) — the novelty lies more in the protocol and factorization analysis than in the mechanism.
- **Rigor.** Strong on internal validity: matched-gate, fixed-inner actuation sweeps, four seeds, McNemar tests, L2 drift reporting, and explicit separation of audit runs from main results. The honesty about cross-seed variance (Mistral Margin) and saturated contamination is commendable. However, the synthetic protocol is the entire load-bearing evidence base; the only public benchmark (Memora) is non-significant.
- **Reproducibility.** Reasonably good: synthetic generator, hyperparameters, gate constants, and probe families are specified, and seeds are reported. But "marker-style training-only calibration pairs," per-user median warmup, and exact stream-generation code are not clearly released, and the protocol is bespoke (no public reference implementation cited).
- **Significance.** The depth-flip demonstration is conceptually clean and the selection/actuation factorization is a useful framework for future agent-memory work. However, the absolute numbers come from small models (GPT-2, TinyLlama) on a self-designed synthetic task; the 7B story is partial (high contamination, high variance) and does not establish a complete large-model memory system. Practical deployment impact is currently limited.
- **Red flags.** (1) Single small-model dependency: the headline "flip" is on GPT-2/TinyLlama, with the larger Mistral result partially undermining the clean narrative under default actuation. (2) Self-designed protocol with no independent third-party benchmark confirming the mechanism. (3) Heldout-paraphrase generalization is explicitly weak (TinyLlama directional, GPT-2 not positive), so the "depth" claim is durable held-in behavior, not broad semantic generalization. (4) Forgetting/deletion unsolved and, per the author, negative-gradient approaches were unstable. (5) Non-traditional single-author affiliation with no co-author or institutional ML-lab backing — worth noting but not disqualifying given the rigor of the hedging.

## Strengths

- Clean conceptual split between access and depth, operationalized as four well-defined probe layers with an explicit optimization target.
- Loop-drift protocol is a thoughtful controlled design that isolates the rarely-controlled post-unload setting while keeping the retrieval index durable (preventing a trivial "RAG forgot" confound).
- Mechanism analysis is unusually thorough: matched-gate (selection≠sparsity), fixed-inner (selection–actuation separability), and asymmetric-coupling diagnostics all address the most likely reviewer objections preemptively.
- Strong intellectual honesty: null Memora result, Mistral inversion, and weak paraphrase generalization are reported transparently rather than spun.
- Efficiency story is compelling — durable goal behavior with ~2–3 writes per 200 events vs. 200 for Naive-LoRA, with bounded L2 drift.

## Limitations

- The load-bearing evidence is entirely synthetic and self-generated; the only public benchmark (Memora) yields a non-significant result.
- Headline results are on small models (GPT-2, TinyLlama); the 7B regime shows high contamination and high cross-seed variance, and default five-step actuation is miscalibrated there.
- No broad paraphrase/semantic generalization — only durable held-in goal-conditioned behavior is claimed.
- Stale-memory deletion and update validity are unsolved; anti-training/negative-CE variants were unstable.
- No reference implementation or dataset release is clearly stated, limiting independent replication of the bespoke protocol.
- Single-author, non-ML-lab affiliation may raise questions about independent vetting.

## Relevance

对于长期运行的语言 Agent 而言，本文提供了一个比单纯"存更多文本"更精确的记忆设计靶点——它把"能在卸载后仍塑造行为的参数化固化"与"检索访问"分开，并用受控实验展示了二者的互补性与可分离控制因子，对持续学习型、个性化助手类 Agent 的记忆架构设计有直接启发意义。

## Questions for Further Exploration

- Can the depth flip be reproduced on frontier-scale models (>7B) with non-synthetic, real user-interaction logs instead of the loop-drift generator?
- What validity-gating or reconsolidation mechanism could extend selective consolidation to stale-memory deletion and update, given that anti-training was unstable?
- How does EVAF interact with architectural test-time memory (e.g., Titans, In-Place TTT) rather than fixed-backbone LoRA — could selection gating compound or conflict with learned memory modules?
- Is the selection–actuation coupling a fundamental property that scales, or an artifact of LoRA's low-rank subspace at small scale — would full-parameter or higher-rank consolidation break the asymmetric feedback signature?
- Does the matched-gate inversion on miscalibrated Mistral predict a general failure mode for any behavior-targeted gate when actuation is wrong, and can a closed-loop actuation controller (e.g., margin-based with lower variance) fix it robustly?
- Can the protocol be hardened against prompt-level adversaries or sibling-user confusion in real multi-tenant deployments where durable goal contamination is safety-relevant?
