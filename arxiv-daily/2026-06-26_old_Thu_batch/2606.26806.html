Memory Depth, Not Memory Access: Selective Parametric Consolidation for Long-Running Language Agents

Haoliang Han · Institute of Biomedical Strategy, China Pharmaceutical University
memory depth vs. access EVAF parametric memory continual learning LoRA consolidation long-running agents loop-drift protocol
▶ Key Finding

Retrieval wins shallow facts while surprise×valence-gated LoRA consolidation wins durable goal behavior — a "depth flip" with only ~2–3 writes per 200 events.

◆ Key Metric
0.812–0.904
EVAF goal-persistence / post-unload score (GPT-2 & TinyLlama) vs. RAG's ~0.394 — durable behavior from ~2.4–2.6 writes/200 events.
◆ Why This Matters

Gives agent-memory designers a precise target — parametric consolidation that survives context unload complements retrieval, with separable selection & actuation controls.

◆ Critical Edge

The headline "flip" rests entirely on self-generated synthetic streams on small models; the only public benchmark (Memora) is non-significant (p=0.57), and the 7B story partially inverts it.

Methodology

The work probes a long-running memory stream x₁…x_T at four depths: shallow episodic access (recent explicit fact), noisy episodic access (older fact after same-key interference), parametric tendency (stable goal still shaping behavior under interference), and post-unload recovery (goal probe repeated after the working context is cleared while the retrieval index persists).

EVAF scores each event with a surprise×valence gate: surprise s_t from token NLL, valence v_t from embedding similarity to the user's durable goal/preferences; admission g_t = σ(k_s(s_t−τ_s))·σ(k_v(v_t−τ_v)) with τ_w = 0.5. Admitted events fill a fixed-size buffer; on overflow the LoRA adapter updates via replay plus an L2 anchor, giving only ~2–3 writes per 200-event stream.

Baselines include Frozen, Summary, RAG (durable embedding index), Naive-LoRA (200 writes/stream), EVAF, and Routed EVAF+RAG. A matched random gate (identical write count and online dynamics) isolates selection from sparsity, and fixed-inner actuation controllers factorize write strength as a separate, model-dependent control dimension.

+ Strengths

  • Clean conceptual split — access vs. depth — operationalized as four probe layers with an explicit optimization target.
  • Loop-drift protocol co-controls the rarely-isolated post-unload setting while keeping the retrieval index durable.
  • Unusually thorough mechanistic analysis: matched-gate, fixed-inner, asymmetric-coupling diagnostics preempt reviewer objections.
  • Strong intellectual honesty — null Memora result, Mistral inversion, weak paraphrase generalization all reported, not spun.
  • Compelling efficiency: durable goal behavior from ~2–3 writes vs. 200 for Naive-LoRA, with bounded L2 drift.

− Limitations

  • Load-bearing evidence is entirely synthetic & self-generated; the only public benchmark is non-significant.
  • Headline results on small models; the 7B regime shows high contamination, high variance, miscalibrated default actuation.
  • No broad paraphrase/semantic generalization — only durable held-in goal-conditioned behavior is claimed.
  • Stale-memory deletion/update unsolved; anti-training & negative-CE variants were unstable.
  • No clear reference implementation or dataset release; bespoke protocol limits independent replication.
  • Single-author, non-ML-lab affiliation raises questions about independent vetting.

? Questions for Further Exploration

  1. Does the depth flip reproduce at frontier scale (>7B) on real user-interaction logs rather than the loop-drift generator?
  2. What validity-gating or reconsolidation mechanism could extend selective consolidation to stale-memory deletion/update, given anti-training was unstable?
  3. How does EVAF interact with architectural test-time memory (Titans, In-Place TTT) — could selection gating compound or conflict with learned memory modules?
  4. Is selection–actuation coupling fundamental, or an artifact of LoRA's low-rank subspace — would full-parameter consolidation break the asymmetric feedback signature?
  5. Does the matched-gate inversion on miscalibrated Mistral predict a general failure mode for behavior-targeted gates, and can a closed-loop actuation controller fix it robustly?
  6. Can the protocol be hardened against prompt-level adversaries or sibling-user confusion in real multi-tenant deployments where durable goal contamination is safety-relevant?
