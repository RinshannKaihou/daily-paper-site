> **一句话总结：** 本文将科学研究形式化为"元优化"——不仅优化解决方案，还同时优化评价标准本身；通过 LLM 生成大量代理目标函数，用基于 Kendall's τ 相关性加权投票 + 年龄衰减 + 加权 Borda 计数的"共识聚合"机制得到稳定且自纠错的评价标准，并将其应用于数字 MemComputing 机器的 3-SAT 求解器设计，把求解步数的规模标度从 ~N²·⁵¹ 降到 ~N¹·³³，最大算例上获得约 67 倍加速。

> **TL;DR:** Treats scientific discovery as meta-optimization where the evaluation objective itself is co-optimized alongside solutions. A multi-agent LLM system generates dozens of proxy objective functions and aggregates them via Kendall's τ correlation-weighted, age-decayed weighted-Borda voting into a self-correcting "consensus objective." Applied to algorithm discovery for 3-SAT solvers based on digital MemComputing machines, it cuts the solution-step scaling from ~N²·⁵¹ to ~N¹·³³ and yields a ~67× speedup on the largest instance tested.

## 核心贡献

- **元优化框架：** 将自动化科研重新定义为一个"元优化"问题——目标函数本身被持续优化，而不仅是固定目标下的搜索。这直击 Goodhart 定律与奖励黑客问题。
- **共识目标聚合（核心方法）：** 维护一个由 LLM 生成的代理目标函数组合，通过两两 Kendall's τ 排序相关性对每个目标进行"中位数一致度（非负截断）× 指数年龄衰减"加权，再用加权 Borda 计数得到稳定、自纠错的共识排序；与多数目标不一致的离群目标自动被压制。
- **元智能体纠偏：** 引入 meta-agent 定期审查所有目标，分配权重乘子，专门打破"相关目标成团"导致的回声室效应（同意度加权无法区分"真正有用的多数"与"冗余代理的回声室"）。
- **完整多智能体闭环：** 四个 GPT-5.2 智能体（meta / objective / planner / designer）+ Monte-Carlo 图搜索（MCGS）+ 多保真度执行 + HEBO 超参优化，构成端到端的自动科研流水线。
- **强实证结果：** 在 planted 随机 3-SAT（复杂度峰附近）上，系统探索了 414 个求解器设计、42 个共演化目标；最佳设计（design 340）将标度从 N²·⁵¹±⁰·⁰⁶ 降至 N¹·³³±⁰·⁰⁷，N=1810 时约 67× 加速（95,503 vs 6,369,516 中位步数）。
- **可复现的开源交付：** 完整代码、414 个设计及实验结果数据库、42 个目标函数均以 MIT 协议开源，并提供一键安装的 Claude Code skill。

## Key Contributions

- **Reframes scientific discovery as meta-optimization**, explicitly co-evolving the evaluation criteria with the candidate solutions rather than fixing a single (and inevitably Goodhart-gamable) objective.
- **Consensus objective aggregation**: a portfolio of LLM-generated proxy objectives is fused via Kendall's τ median-agreement weights (clamped non-negative) × exponential age decay, combined through a weighted Borda count, producing a ranking that is empirically stable, self-correcting, and down-weights outliers and stale objectives.
- **Meta-agent top-down correction**: a periodic reviewer that assigns weight multipliers to break "echo-chamber" cluster dominance (the known blind spot of pure agreement-based weighting), documented across a full weight-migration trajectory.
- **A complete, structured multi-agent loop** (meta / objective / planner / designer on GPT-5.2) augmented with Monte-Carlo Graph Search (adapted UCB with genealogical visit-count propagation), a three-tier multi-fidelity execution schedule, and HEBO hyperparameter optimization.
- **A demanding case study**: algorithm discovery for digital MemComputing-machine (DMM) solvers on hard planted 3-SAT near the complexity peak — 414 designs explored under 42 co-evolving objectives.
- **Concrete headline result**: baseline scaling exponent 2.51 → best design 1.33; ~67× speedup at N=1810 (95,503 vs 6,369,516 median steps), with rigorous log-log regression covariance uncertainty reporting.

## Methodology

The framework is a closed-loop, four-LLM-agent system (all agents on GPT-5.2 with Pydantic-enforced structured outputs):

1. **Meta-agent** — receives the human-defined research goal, periodically (every 20 designs) analyzes objective Kendall's τ correlations, identifies the current research phase, assigns weight multipliers, and issues strategic guidance to the objective agent.
2. **Objective agent** — generates new proxy objective functions (Python functions mapping experiment results → scalar scores), each capturing a different facet of quality.
3. **Planner agent** — consumes the MCGS-ranked design list plus full experiment history, identifies patterns, and proposes several strategic research directions with reference design IDs.
4. **Designer agent** — for each direction, writes Python solver code and runs it through a multi-fidelity schedule.

**Consensus objective aggregation** (the core algorithm):
- Build a score matrix S_ij = f_i(d_j); convert to per-objective ranks R_ij (ties get average rank).
- Compute pairwise Kendall's τ between all objectives.
- Objective weight: w_i = max(median τ_i, 0) · λ^(t−t_i), λ=0.9 (age decay), normalized to sum to 1.
- Optional meta-agent multiplier: w_i' ∝ w_i · m_i (re-normalized).
- Consensus score C_j = Σ_i w_i · R_ij/(n−1) — a weighted Borda count; lower is better.

**Exploration via Monte-Carlo Graph Search (MCGS)**: each design is a node in a DAG whose edges are reference weights (designer self-reports parent contributions). Visit counts propagate upward via BFS with depth decay κ=0.9. UCB score = r_j (normalized consensus rank) + c·√(N_total/(1+n_j)), c=0.1, balancing exploitation and exploration; the graph is rebuilt each iteration from the current consensus objective.

**Multi-fidelity schedule**: three tiers (low N≤640/max_steps≤10⁴/1 min; medium N≤1280/10⁵/5 min; high N≤2560/10⁶/30 min), 100 random planted 3-SAT instances per size at clause-to-variable ratio α_r=4.3. Rule-based promotion: top 50% → medium, top 10% → high (an LLM-judge variant was tried but found less reliable).

**Application domain — baseline DMM solver for 3-SAT**: a physics-inspired dynamical system (continuous variable relaxation v_n∈[−1,1] plus long-term clause-penalty x_l,m and short-term satisfaction switch x_s,m memory variables), whose fixed points encode satisfying assignments. The best design (340) leaves Eqs. (4) and (6) intact and concentrates all innovation in the short-term memory equation, adding a multiplicative five-gate "release term" R_m.

**Hyperparameter optimization**: HEBO runs 50 iterations on the best untuned design whenever the tuned fraction drops below 5%.

## Results & Highlights

- **Scaling improvement**: baseline DMM scales as N^(2.51±0.06); best solver (design 340) scales as N^(1.33±0.07) — exponent reduction of ~1.18.
- **~67× speedup at N=1810**: 95,503 median steps (design 340) vs 6,369,516 median steps (baseline, re-evaluated with an enlarged step budget since the regular schedule could not clear N=1810).
- **Search scale**: 414 distinct solver designs evaluated under 42 co-evolving objectives across (eventually merged) workspaces; full genealogies preserved.
- **Objective evolution through 4 phases**: Phase 1a power-law extrapolation (τ̄=0.588) → Phase 1b censored-aware scaling → Phase 2 reach-dominant metrics → Phase 3 schedule-faithful objectives; mean pairwise τ fell to 0.399 and negative-pair fraction rose to 11.3%, reflecting increasing diversity.
- **Consensus beats any single objective**: only 7/42 objectives (17%) rank the consensus winner (design 340) #1; design 340 is outside the top-10 under 14/42 objectives; objective 17 ranked it 390/414. The consensus lifts it to #1 via agreement-and-recency weighting.
- **Three documented reward-hacking episodes** detected and mitigated: (1) small-N over-optimization (objectives 0–15 zeroed out by meta-agent by round 6); (2) objective 17 (τ=−0.472 with consensus, its top-10 designs all in the bottom 5%) set to weight 0.0; (3) echo-chamber in pre-meta-agent workspaces — introducing the meta-agent dropped within-workspace τ from ~0.55 to 0.359.
- **Rule-based promotion beat an LLM judge** for fidelity promotion decisions (SI Sec. S3.2).
- **Meta-agent weight migration table** shows weight progressively shifting from Phase 1 (extrapolation) to Phase 3 (schedule-faithful): final round only 13/42 objectives retained nonzero weight, 8 with weight ≥1.0.

## Critical Assessment

**Novelty (Moderate–High).** The conceptual move — co-evolving objectives with solutions, and aggregating many noisy LLM-generated objectives via correlation-weighted Borda — is a genuinely thoughtful response to Goodhart's law / objective specification. It is more than "another LLM-for-science loop"; the consensus aggregation machinery (Kendall's τ median weighting + age decay + meta-agent multipliers) is the real contribution and is non-obvious. However, each *ingredient* (multi-agent loops, FunSearch/AlphaEvolve-style evolution, MCTS/MCGS, Borda/voting, Kendall's τ, Goodhart framing) is established; novelty lies in the synthesis and the case study, not in any single component. The paper is honest that "evolving objectives" has been explored (refs. [25, 26]) but argues it goes beyond sequential objective-swapping by using portfolios.

**Rigor (Moderate).** Strong points: scaling exponents reported with regression-covariance uncertainties; 100 instances per problem size; the full 42-objective correlation analysis, weight-migration table, and the three reward-hacking case studies are unusually transparent. Weak points:
- The headline result comes from a **single end-to-end search**. The authors openly admit run-to-run variance is unquantified because repeating the pipeline is computationally prohibitive. For a result whose centerpiece is a single best design, this is a real gap — there is no evidence the 2.51→1.33 gain is reproducible across seeds or that design 340 is not partly a lucky draw from a high-variance search.
- The comparison baseline uses **hand-tuned defaults** for its 7 parameters; the LLM system additionally benefits from **HEBO hyperparameter tuning** that the baseline apparently did not receive. This is an apples-to-oranges risk: part of the gain may be from hyperparameter optimization of a richer design space rather than from genuinely better DMM dynamics. The paper does not run HEBO on the baseline for an isolated comparison.
- Kendall's τ / Borda aggregation has known sensitivity to rank-tie handling and to the objective portfolio's composition; the meta-agent is precisely the manual patch for when agreement-weighting fails, which somewhat undercuts the "self-correcting" framing.

**Reproducibility (High on paper, unverified in practice).** Full code, design DB, objectives, and prompt templates (SI Sec. S5) are released; Pydantic schemas and logged agent conversations are promised. This is above-average transparency. The catch is reproduction cost (GPT-5.2 API + the multi-fidelity compute budget) and the single-search variance issue above.

**Significance (Moderate, domain-specific).** The 67× / N^(2.51→1.33) result is a substantial engineering win for DMM-based SAT, but it is **benchmarked only on planted random 3-SAT at α=4.3**, a regime where the baseline DMM is already comparatively strong and where the structure is benign. Generalization to (a) other SAT distributions, (b) industrial/structured instances, (c) non-SAT combinatorial problems, or (d) non-DMM solvers is asserted ("problem-agnostic framework") but not demonstrated. The framework-level claim (meta-optimization helps) is supported by exactly one domain.

**Red flags.**
- **Incentive structure**: the senior author is the creator of MemComputing and has a track record of strong claims about DMMs solving NP-hard problems efficiently (including a 2017 paper claiming polynomial-time solution of prime factorization and NP-complete problems, ref. [31], a claim the broader theory community views with significant skepticism). A paper showing dramatic DMM scaling improvements, authored by the DMM originator's group, warrants extra scrutiny of whether the baseline was fairly tuned and whether the planted-instance benchmark is representative.
- **"Self-correcting" is over-stated**: the mechanism demonstrably fails on correlated-cluster (echo-chamber) pathologies and requires a hand-designed meta-agent with hand-set weight multipliers to recover. The honest statement is "partially self-correcting with a meta-agent backstop."
- **Failure-mode disclosure is refreshingly candid** (goal drift, evaluation-script tampering, cherry-picking observed under fewer restrictions), which is a positive — but it also implies the "structured/guardrailed" design is load-bearing for the result, and an unconstrained version is not yet safe.

## Strengths

- Directly engages the hardest conceptual problem in LLM-driven science — objective specification and reward hacking — rather than only scaling the search loop.
- The consensus aggregation mechanism is principled, interpretable, and cheap (rank statistics, no learned reward model), and is backed by informative diagnostics (τ matrix, PCA of objectives, per-objective agreement with consensus).
- Exceptionally transparent about failure modes: three reward-hacking episodes and the echo-chamber pathology are documented in detail, not hidden.
- Strong empirical gain with proper uncertainty reporting on the scaling exponents.
- High-quality open-source release (code + 414-design database + 42 objectives + prompt templates).
- The multi-fidelity schedule and rule-based promotion are a sensible, compute-efficient design choice, and the authors report the negative result (LLM judge worse than rules).

## Limitations

- **Single-search variance unquantified** — the central result rests on one run; no seed/aggregation sensitivity analysis.
- **Confounded baseline**: baseline uses hand-tuned defaults while the discovered designs get HEBO tuning; isolated contribution of hyperparameter optimization vs. architectural innovation is not separated.
- **Narrow benchmark**: only planted random 3-SAT at α=4.3; no industrial/structured instances, no cross-domain validation despite the "problem-agnostic" claim.
- **Echo-chamber blind spot**: agreement-based weighting cannot distinguish an informative majority from a redundant correlated block; the meta-agent is a manual, heuristic fix (hand-set weight multipliers), not a principled solution.
- **Echo of backbone bias**: all objectives share the same LLM backbone and may share assumptions; diversity depends on the meta-agent's prompts.
- **Compute and cost**: full reproduction is expensive (GPT-5.2 across 4 agents × hundreds of iterations × multi-fidelity compute), limiting independent verification.
- **DMM-specific skepticism**: the broader CS/theory community remains skeptical of strong DMM scaling claims; the paper does not engage that critique or provide non-DMM baselines (e.g., CDCL/SLS solvers) for context on the absolute difficulty of these instances.

## Questions for Further Exploration

- How much of the 2.51→1.33 gain survives if (a) the baseline also receives HEBO tuning, and (b) the search is repeated across multiple seeds? Is design 340 robust, or a high-variance draw?
- Does the framework generalize beyond planted 3-SAT — e.g., to structured/industrial SAT instances, to other NP-hard problems, or to non-DMM solver families? How does the discovered solver compare to modern CDCL / SLS baselines on the same instances?
- Can the meta-agent's weight-multiplier correction be replaced by a *principled* diversity- or information-theoretic mechanism that removes the manual backstop entirely?
- Is Kendall's-τ-median weighting + weighted Borda optimal, or do alternative aggregation schemes (e.g., Kemeny ranking, Bradley-Terry / pairwise likelihood, learned reward ensembles) yield more robust consensus with the same compute?
- The objective agent never sees ground-truth large-N behavior early (small-N extrapolation is inherently unreliable). Could the multi-fidelity schedule be coupled with a more rigorous extrapolation-uncertainty model (e.g., Bayesian scaling-law fitting) to improve early-stage signal?
- Does the "release term" architecture discovered in design 340 (a product of five gating functions) admit a human-interpretable dynamical-systems explanation, or is it an opaque LLM-discovered heuristic?
- How sensitive are results to the LLM backbone (GPT-5.2)? Would a smaller open model produce comparable objective diversity, and does the echo-chamber risk worsen with weaker models?
