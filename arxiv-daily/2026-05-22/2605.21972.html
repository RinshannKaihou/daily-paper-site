How Sparsity Allocation Shapes Label-Free Post-Pruning Recoverability

Relevance
- 对模型压缩与高效推理领域有直接参考价值，特别是研究无标签剪枝恢复和剪枝后部署的工作流。 - 与边缘计算和隐私敏感场景高度相关: 当标注数据不可用或训练管线不可访问时，无标签修复是唯一的恢复手段。 - 为剪枝算法设计提供了新范式: 未来剪枝策略不应仅优化剪枝时精度保持，还应考虑下游修复的可行性（"repair-aware pruning"）。 - 与同期LLM剪枝工作形成互补: 并发的工作（Yu et al. 2026）研究了无数据LLM权重缩放，本文则关注激活统计量修复，两种范式在不同架构上可能各有优势。
Methodology
### 核心流程 - 剪枝流水线: 密集预训练模型 -> 稀疏分配（ERK或LAMP） -> 非结构化幅度剪枝 -> ASR无标签修复 -> 评估 - 修复后端ASR的工作原理: - 对每个卷积层的每个输出通道，计算密集模型和剪枝模型在无标签校准集上的激活方差比（gamma_c） - ASR-q50：使用通道间方差中位数作为收缩先验，对严重受损通道进行温和修正 - Clipped ASR：在ASR-q50基础上将缩放因子限制在[0.5, 2.0]范围内，以更保守的方式避免过度修正 - 修正权重后进行20个mini-batch的BN统计量重校准 ### 实验设置 - 主要实验: ResNet-18/34/50 x CIFAR-10/CIFAR-100/Imagenette，稀疏度90.0%-95.5% - 扩展验证: ImageNet-100（ResNet-18），CIFAR-10/100（DenseNet-121） - DenseNet-121特殊设置: CIFAR-100上使用70.0%-90.0%稀疏度（因非残差架构更早崩溃） - 校准集: 128张无标签训练图像，无梯度计算 - 基线: 无修复、仅BN重校准、偏置修正+BN、仿射校准+BN
Results & Highlights
### 分配-修复交互（核心发现） - ResNet-50 / CIFAR-10: LAMP + ASR-q50在90%稀疏度下达79.92%，ERK + ASR-q50仅70.24%，差距近10个百分点 - ResNet-34 / CIFAR-100: 方向反转——ERK在所有稀疏度下领先LAMP（90%时61.38% vs. 59.81%，95%时41.20% vs. 38.07%） - ResNet-18 / Imagenette: LAMP在95%稀疏度下达73.10%，ERK为68.51%，但差距较小 ### 修复效果 - 在54个ResNet主实验设置中，ASR比仅BN重校准平均提升17.4个百分点 - ResNet-50 / CIFAR-10 / 92.5%稀疏度: 无修复10.00%，BN重校准20.72%，ASR-q50达67.96% - ResNet-50 / CIFAR-100 / 92.5%稀疏度: 无修复1.00%，BN重校准4.70%，Clipped ASR达25.96% ### 架构依赖性 - DenseNet-121 / CIFAR-100: 修复敏感过渡区间在70-80%稀疏度即出现（ResNet-50为92-93%） - DenseNet-121: Clipped ASR在所有设置下领先ASR-q50（与ResNet-18上ASR-q50更优的模式相反） ### ASR与有标签微调的对比 - ResNet-50 / CIFAR-100 / 92.5% / LAMP: Clipped ASR达25.96%，一epoch有标签微调达71.49%——ASR不能替代有标签微调，但在无标签场景下仍有价值 ### Clip界限敏感性 - 默认[0.5, 2.0]界限处于敏感性曲线的平坦区域，不过分保守也
