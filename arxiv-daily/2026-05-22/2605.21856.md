# The Illusion of Reasoning: Exposing Evasive Data Contamination in LLMs via Zero-CoT Truncation

## 一句话总结

本文提出 Zero-CoT Probe (ZCP) 方法，通过截断链式思维过程来检测 LLM 中隐蔽的基准数据污染（如改写题目的恶意污染），揭示模型看似"推理"的表现可能只是记忆的结果。

## TL;DR

LLM 在推理基准上的高分可能来自数据污染而非真实推理能力。恶意发布者可通过改写题目（evasive contamination）来逃避现有检测。本文发现模型的推理步骤会主动掩盖其底层记忆——在 Full-CoT 下，污染数据与干净数据的表现几乎无法区分。ZCP 通过截断 CoT 迫使模型直接作答，暴露其潜在的 question-to-answer 捷径映射；再与同构数值扰动的参考数据集对比，排除模型内在"超能力"的干扰。引入 Contamination Confidence（基于贝叶斯后验概率）量化污染严重程度。实验在 Qwen2.5-Math、DeepSeek-Math、Qwen3、GPT-4o、GPT-5.1 等模型上验证，ZCP 成功检测直接和隐蔽污染，假阳性率极低（干净基准上 C_cont 约 0.500）。真实世界审计发现 GPT-4o 在 GSM8K/MATH-500 上存在污染（C_cont > 0.85），而 GPT-5.1 则接近干净基线。

## 核心贡献

1. **发现关键现象**：模型生成的推理步骤会主动掩盖其底层记忆。在 Full-CoT 下，Qwen-Math 在 GSM8K 上的污染和参考数据准确率均约 96%，污染信号完全消失（C_cont 约 0.500）。截断 CoT 后，污染数据准确率 45.8% vs. 参考数据 33.4%，差距显著暴露。
2. **提出 Zero-CoT Probe (ZCP)**：一种新型黑盒检测方法，通过完全截断 CoT 过程来暴露模型的潜在捷径映射（shortcut mapping）。对开源模型使用 response prefix 强制直接输出答案；对闭源模型使用严格指令 prompt。
3. **引入 Contamination Confidence 指标**：基于非参数 bootstrap 检验的 p 值，通过 Bayes Factor 校准为贝叶斯后验概率（中性先验 pi=0.5），C_cont in [0.5, 1.0]，超越二元分类。
4. **同构扰动对照**：系统改变原始题目的数值（保持同数量级）和文本表述，严格保持逻辑结构和推理深度，构建等难度的参考数据集，由多模型流水线（GPT-o3-mini 生成 + GPT-o4-mini/Gemini-2.5-flash 双重验证）自动生成。
5. **真实世界审计**：对 Qwen 系列（白盒）和 GPT 系列（黑盒）进行系统审计，发现 GPT-4o 存在明确污染，GPT-5.1 接近干净。

## Key Contributions

1. **Critical phenomenon uncovered**: Reasoning steps actively mask memorization. Under Full-CoT, contaminated vs. clean data accuracy is indistinguishable (~96% both on Qwen-Math/GSM8K); truncating CoT reveals a 12.4-point gap (45.8% vs. 33.4%).
2. **Zero-CoT Probe (ZCP)**: A black-box detection method that truncates the entire CoT to expose latent question-to-answer shortcut mappings. Uses response prefix for open-weight models and strict prompting for closed-source APIs.
3. **Contamination Confidence metric**: Calibrates non-parametric bootstrap p-values into Bayesian posterior probabilities via Bayes Factor (neutral prior pi=0.5), yielding a continuous score in [0.5, 1.0] rather than binary classification.
4. **Isomorphic perturbation control**: Automated multi-model pipeline (GPT-o3-mini generator + GPT-o4-mini & Gemini-2.5-flash dual judges) creates structurally equivalent reference datasets by altering numerical values (same order of magnitude) and paraphrasing text while preserving logical structure.
5. **Real-world auditing**: Systematic audit of Qwen series (white-box, all 4 metrics) and GPT series (black-box, output-only metrics) reveals GPT-4o contamination (C_cont > 0.85) and GPT-5.1 near-clean baseline (C_cont approx 0.500).

## Methodology

### 核心思路

模型如果在训练中见过某数据集（即使是改写版本），会建立从问题语义到答案的直接捷径映射。在正常 Full-CoT 模式下，模型通过推理链可以解决任何同难度问题，掩盖了这一映射。截断 CoT 后，模型被迫依赖两种因素作答：(1) 记忆的捷径映射，或 (2) 内在"超能力"（不用推理步骤解决复杂问题的能力）。通过与同构扰动的参考数据集对比，可以排除"超能力"因素。

### ZCP 检测流程

1. **Zero-CoT 强制**：
   - 开源模型：在 response 开头附加 "The final answer is: \[ \boxed{"，迫使模型直接输出答案。
   - 闭源模型：在 prompt 末尾添加 "Please ONLY put your final answer within \boxed{} directly without any other content"。
2. **四层指标体系**：
   - **Accuracy (Acc)**：zero-CoT 答案是否正确（离散，0/1）。
   - **Consistency (Con)**：zero-CoT 答案是否与 full-CoT 答案一致（离散，无需 ground truth）。
   - **First Token Probability (P_first)**：正确答案首 token 的生成概率（连续，需 logit 访问）。
   - **All Token Probability (P_all)**：正确答案所有 token 概率的几何均值（连续，teacher forcing，需 logit 访问）。
3. **同构扰动参考数据集**：系统改变数值（保持数量级），改写文本（保持逻辑结构），由多模型流水线自动生成并验证。
4. **Contamination Confidence 计算**：
   - 对连续指标使用非参数 bootstrap 检验（10,000 次重采样）。
   - 对离散指标使用 McNemar 检验。
   - p 值通过 Bayes Factor（Sellke-Berger 校准）转为后验概率：C_cont = BF_10 / (BF_10 + 1)。
   - 基线 0.500 = 无统计证据，趋近 1.000 = 确认污染。

### 数据集构建

- **参考数据集**：GPT-o3-mini 生成同构扰动题，GPT-o4-mini 和 Gemini-2.5-flash 双重验证正确性，仅当两个 judge 一致同意时才纳入。
- **隐蔽污染数据**：使用 GPT-4o 改写文本、替换实体，严格保留所有数值和答案，生成 6 个变体用于微调。

## Results & Highlights

### 现有模型实验（翻转实验）

| 模型 | 数据 | Zero-CoT Acc (原始) | Acc (参考) | C_cont |
|------|------|-------------------|-----------|--------|
| DeepSeek-Math | GSM8K | 29.80% | 22.20% | 0.989 |
| DeepSeek-Math | MATH | 37.14% | 20.71% | 1.000 |
| Qwen-Math | GSM8K | 45.80% | 33.40% | 1.000 |
| Qwen-Math | MATH | 53.14% | 35.29% | 1.000 |

- 即使在**改写数据**（paraphrased）上，C_cont 依然很高（0.920-1.000），证明 ZCP 能检测隐蔽污染。
- **干净基准 GSM1K** 上 C_cont 约 0.500，证明极低假阳性率。

### 微调模型实验

| 模型 | 基准 | Dataset C C_cont | Dataset U C_cont |
|------|------|-----------------|-----------------|
| FT Qwen-Math | Omni-MATH | 1.000 (Acc) | 0.551 |
| FT Qwen3 | Multi-domain | 1.000 (Acc) | 0.559 |

- 隐蔽污染使 Dataset C 的 full-CoT 准确率从 21.28% 飙升至 43.38%（Omni-MATH），而 Dataset U 保持稳定（23.64% -> 26.77%），证实污染效果。
- 微调流程：SFT（LoRA r=32, alpha=64）+ GRPO（n_sample=5），使用 6 个改写变体。

### 现有检测方法失效

| 方法 | 原始数据 | 改写数据 |
|------|---------|---------|
| DPCC (Qwen-Math/GSM8K) | 0.420 | 0.062 |
| DPCC (Qwen-Math/MATH) | 0.730 | 0.191 |
| 数据重建 ROUGE-L (Qwen-Math/GSM8K) | 0.551 | 0.213 |
| 数据重建 Accuracy (Qwen-Math/GSM8K) | 0.398 | 0.176 |

DPCC 和数据重建在改写数据上分数暴跌，而 ZCP 在改写数据上 C_cont 仍 > 0.92。

### Full-CoT vs Zero-CoT 对比（消融实验）

| 设置 | 原始 Acc | 参考 Acc | C_cont |
|------|---------|---------|--------|
| Full-CoT (Qwen-Math/GSM8K) | 95.80% | 96.20% | 0.500 |
| Zero-CoT (Qwen-Math/GSM8K) | 45.80% | 33.40% | 1.000 |

Full-CoT 下污染完全被掩盖，C_cont 回到基线 0.500。

### 真实世界审计

| 模型 | GSM8K C_cont (Acc) | MATH-500 C_cont (P_all) |
|------|-------------------|----------------------|
| Qwen-2.5-Math | 0.617 | >0.998 |
| Qwen-3 | 0.501 (干净) | >0.998 (污染) |
| GPT-4o | 0.855 | 0.984 |
| GPT-5.1 | 0.550 (接近基线) | 0.549 (接近基线) |

- Qwen-3 在 GSM8K 上干净但在 MATH-500 上污染。
- GPT-4o 在两个基准上均显示污染；GPT-5.1 接近干净基线，可能因为开发者进行了去污染处理。

### 指标稳定性 vs 数据集规模

- **Logit 指标**（P_first, P_all）：仅需 50-100 样本即可达 C_cont > 0.94。
- **Accuracy**：需 200-500 样本达 C_cont > 0.888。
- **Consistency**：需约 1000 样本，但无需 ground truth 标签。

## Critical Assessment

### Strengths

- **核心洞察深刻且反直觉**：推理能力掩盖记忆这一现象虽然事后看来合理，但论文首次系统性地揭示并通过消融实验证实（Full-CoT C_cont=0.500 vs Zero-CoT C_cont=1.000）。
- **实验设计严谨**：翻转实验、微调实验、真实世界审计三个维度；干净基准（GSM1K）验证低假阳性。
- **实用性强**：黑盒方法适用 API 模型；Consistency 指标甚至不需要 ground truth；不同访问级别有对应指标。
- **超越二元判断**：Contamination Confidence 基于严格的贝叶斯统计框架，提供连续量化。
- **对现有方法做了系统性对比**：明确展示 DPCC 和数据重建在改写数据上的失效。
- **多模型审计结果有实际影响**：GPT-4o 污染、GPT-5.1 干净的发现对社区有参考价值。

### Limitations

- **闭源模型的 zero-CoT 强制依赖 prompt 工程**：论文自己也承认，随着模型越来越优化为必须逐步推理，当前 prompt 可能在未来失效。这是一个根本性的限制。
- **同构扰动假设**：方法假设"改变数值但保持逻辑结构"能产生等价难度。对于某些类型的问题（如纯符号推理或非数值推理），这一假设可能不成立。
- **潜在反制策略**：如果攻击者知道 ZCP 的原理，可以在训练数据中也包含 zero-CoT 格式的样本，训练模型在没有 CoT 时也输出正常表现。论文未讨论此类对抗性场景。
- **Contamination Confidence 的先验假设**：使用中性先验 pi=0.5 是合理默认，但在已知某模型家族有污染历史的情况下，可能不是最优选择。论文未讨论先验敏感性。
- **评估范围**：主要在数学推理基准上验证。对于自然语言理解、代码生成等其他类型基准的适用性有待验证。
- **样本效率**：Consistency 指标需要约 1000 样本才能可靠检测，这在某些私有基准上可能不可行。

### Relevance

- **对 LLM 评估社区高度相关**：数据污染是当前 LLM 排行榜可信度的核心问题之一。evasive contamination（改写题目逃避检测）是最难检测的变种，本文直接针对这一威胁。
- **对模型审计和基准维护有直接应用价值**：ZCP 提供了一个可立即使用的审计工具，特别适用于 API 模型。
- **对理解 CoT 与记忆的关系有理论贡献**：揭示了推理能力与记忆之间的互动机制，对 CoT faithfulness 研究也有启发。
- **与同期工作形成互补**：Dekoninck et al. (2024) 证明 evasive contamination 容易实施但未提出有效检测；Dong et al. (2024) 的低方差假设不适用于 RL 训练的模型。ZCP 在这些盲区中提供了有效方案。

## Questions for Further Exploration

1. **对抗性训练**：如果模型在训练中同时混入 zero-CoT 格式的数据（即训练模型在无 CoT 时也表现正常），ZCP 的检测效果如何？是否有改进空间？
2. **非数值推理任务**：ZCP 的同构扰动策略依赖数值替换，对于纯文本推理、常识推理、代码生成等任务如何适配？
3. **CoT 截断比例**：论文展示了从 Full-CoT 到 Zero-CoT 的梯度变化（Figure 2），是否存在最优的部分截断比例，既能暴露记忆又不过度依赖完全截断？
4. **闭源模型 prompt 鲁棒性**：随着模型越来越倾向于必须推理（如 o1/o3 系列），当前的 zero-CoT prompt 强制策略能否持续有效？是否需要发展更可靠的强制机制？
5. **污染程度与 C_cont 的对应关系**：不同污染比例（如 10% vs 50% vs 100% 训练数据被污染）下，C_cont 的变化模式是什么？能否从 C_cont 反推污染比例？
6. **跨基准泛化**：ZCP 的参考数据集需要针对每个基准单独构建，是否存在通用的参考数据集或跨基准的迁移方案？
