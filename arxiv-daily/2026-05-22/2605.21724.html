Methodology
### Problem Setting 在Transformer中，Hyper-Connections将单一残差流扩展为 $n$ 个并行流，通过可学习混合矩阵 $H_l^{\text{res}} \in \mathbb{R}^{n \times n}$ 进行层间混合： $$X_{l+1} = H_l^{in} X_l + (H_l^{post})^\top F_l(H_l^{pre} X_l, W_l)$$ 为保证深度组合时的稳定性，需要 $H_l^{\text{res}}$ 为双随机矩阵（属于Birkhoff多面体 $\mathcal{B}_n$），即行和与列和均为1的非负矩阵。 ### Core Idea: Transportation Polytope Perspective 核心洞察是Birkhoff多面体 $\mathcal{B}_n$ 是运输多面体 $\mathcal{T}(r, c)$ 在 $r = c = \mathbf{1}$ 时的特例。TBP利用North-West Corner贪心算法逐步填充矩阵： $$x_{ij} = L_{ij} + \Delta_{ij} \cdot \sigma(t_{ij})$$ 其中 $L_{ij}$ 是由前序行/列已确定值决定的下界，$\Delta_{ij}$ 是剩余可用量，$t_{ij}$ 是可学习参数。每一步的选择受前面选择的约束，但保证最终矩阵严格双随机。 ### Recursive TBP (RTBP) 将 $n \times n$ 矩阵分块为4个子矩阵，每个子矩阵对应一个子运输多面体问题，递归求解。块分解满足一致性约束： $$\sum X_{11} + \sum X_{12} = r_{\text{top}}, \quad \sum X_{11} + \sum X_{21} = c_{\te
Results & Highlights
实验设置：基于nanoGPT框架在OpenWebText上预训练，Small（6层, 8头, 512维）和Medium（12层, 12头, 768维）两个规模，4个残差流，10K步训练。 主要结果： | 指标 | 表现 | |------|------| | 小规模最佳 | Experiment 2中KromHC排第1，oRTBP-mHC排第2（val loss 3.497 vs 3.495） | | 中规模最佳 | Experiment 3中mHC-lite排第1，msTBP-mHC排第2（val loss 3.266 vs 3.269） | | 梯度稳定性 | (R)TBP变体在所有实验中梯度范数一致低于mHC、mHC-lite和KromHC | | 吞吐量 | oRTBP与mHC差距小于2%（~216K vs ~215K tokens/sec小规模）；msTBP在中规模也具有竞争力 | | KromHC不稳定 | 报告了KromHC在某些运行中出现梯度不稳定的情况（未纳入正式比较） | 关键发现： - msTBP-mHC在中规模实验中排名第2，仅次mHC-lite，是最好的TBP变体 - oRTBP通过优化器感知的参数分组（无权重衰减、独立学习率）显著改善训练动态 - 所有(R)TBP变体的梯度范数一致更低，表明精确双随机约束带来的稳定性优势 - 实现中RTBP被重构为向量化批处理计算以利用GPU并行性

Critical Assessment

Point 1
对残差网络/Transformer架构改进有直接相关性：Hyper-Connections是当前改进深层网络梯度流的热门方向
Point 2
对优化理论爱好者有参考价值：运输多面体与Birkhoff多面体之间的连接在ML文献中较少被利用
Point 3
实践应用：在大规模LLM训练中，如果精确双随机约束确实带来更好的稳定性，TBP/RTBP可能成为mHC的替代选择，但需要更大规模验证
