Methodology
### 核心思路 模型如果在训练中见过某数据集（即使是改写版本），会建立从问题语义到答案的直接捷径映射。在正常 Full-CoT 模式下，模型通过推理链可以解决任何同难度问题，掩盖了这一映射。截断 CoT 后，模型被迫依赖两种因素作答：(1) 记忆的捷径映射，或 (2) 内在"超能力"（不用推理步骤解决复杂问题的能力）。通过与同构扰动的参考数据集对比，可以排除"超能力"因素。 ### ZCP 检测流程 1. Zero-CoT 强制： - 开源模型：在 response 开头附加 "The final answer is: \[ \boxed{"，迫使模型直接输出答案。 - 闭源模型：在 prompt 末尾添加 "Please ONLY put your final answer within \boxed{} directly without any other content"。 2. 四层指标体系： - Accuracy (Acc)：zero-CoT 答案是否正确（离散，0/1）。 - Consistency (Con)：zero-CoT 答案是否与 full-CoT 答案一致（离散，无需 ground truth）。 - First Token Probability (P_first)：正确答案首 token 的生成概率（连续，需 logit 访问）。 - All Token Probability (P_all)：正确答案所有 token 概率的几何均值（连续，teacher forcing，需 logit 访问）。 3. 同构扰动参考数据集：系统改变数值（保持数量级），改写文本（保持逻辑结构），由多模型流水线自动生成并验证。 4. **Contamination C

Critical Assessment

Point 1
核心洞察深刻且反直觉：推理能力掩盖记忆这一现象虽然事后看来合理，但论文首次系统性地揭示并通过消融实验证实（Full-CoT C_cont=0.500 vs Zero-CoT C_cont=1.000）。
Point 2
实验设计严谨：翻转实验、微调实验、真实世界审计三个维度；干净基准（GSM1K）验证低假阳性。
Point 3
实用性强：黑盒方法适用 API 模型；Consistency 指标甚至不需要 ground truth；不同访问级别有对应指标。
Point 4
超越二元判断：Contamination Confidence 基于严格的贝叶斯统计框架，提供连续量化。
Point 5
对现有方法做了系统性对比：明确展示 DPCC 和数据重建在改写数据上的失效。
Point 6
多模型审计结果有实际影响：GPT-4o 污染、GPT-5.1 干净的发现对社区有参考价值。
