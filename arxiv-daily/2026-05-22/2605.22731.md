## 一句话总结 / One-Sentence Summary

本文提出后训练的本质不在于损失函数的形式（MLE、策略梯度、KL散度），而在于训练状态分布的来源：SFT 使用固定的数据集状态（off-policy），而 RL 和在线策略蒸馏（OPD）使用模型自身生成的状态（on-policy），这一视角统一解释了灾难性遗忘、学生超越老师等后训练现象。

This paper argues that post-training is fundamentally about *where* supervision is applied (the state distribution), not just *what* objective is used: SFT trains on fixed dataset states (off-policy), while RL and on-policy distillation (OPD) train on learner-induced states (on-policy), providing a unified explanation for catastrophic forgetting, student-surpasses-teacher phenomena, and retention behavior.

---

## TL;DR

- 提出后训练的**状态分布视角**：将 SFT、RL、OPD 统一为"在什么状态上施加什么信号"的框架。
- 核心区分两个轴：**状态来源**（数据集 vs. 当前策略）和**信号来源**（金标 token vs. 奖励 vs. 教师输出）。
- 在 Qwen3-0.6B-Base + GSM8K 上的实验验证了三个关键发现：
  1. 轻度 SFT 可安全提升目标任务，但压力 SFT 导致严重遗忘。
  2. OPD 学生可以全面超越退化的教师模型。
  3. 轻量 RL 在提升目标任务的同时几乎不损害保留能力。
- 提出了四个定性预测（P1-P4），并通过 MMD 状态漂移分析发现标量漂移不足以完全刻画训练动态。

## 核心贡献

1. **状态分布形式化框架**：将后训练方法统一为状态条件监督问题，区分状态来源和信号来源两个独立维度。提出了状态分布演化的递推公式 d_{k+1}(s) = T(d_k(s), signal)。
2. **OPD 的新解释**：展示了学生超越教师不是因为更好的目标函数，而是因为学生控制了状态分布，只在自身可达的状态上接受教师的局部指导。
3. **灾难性遗忘的新理解**：遗忘不只是更新幅度的问题，而是 off-policy 状态分布将模型推入与自身未来状态分布不兼容的区域。
4. **实证验证**：在受控小规模实验中系统验证了框架的四个预测，并引入 MMD 状态漂移作为分析工具。

---

## Key Contributions

1. **State-distribution formalization**: Unifies post-training methods as state-conditioned supervision, factoring algorithm design into state source and signal source as independent axes.
2. **New explanation for student > teacher**: Demonstrates that OPD students can surpass degraded teachers not due to better objectives, but because students control the state distribution and only receive local teacher guidance on their own reachable states.
3. **Reframing catastrophic forgetting**: Forgetting arises when off-policy state distributions push the model into regions incompatible with its own future state distribution, not merely from update magnitude.
4. **Empirical validation**: Systematically tests four qualitative predictions in a controlled setting, introducing MMD state-drift as an analysis tool.

## Methodology

### 理论框架

论文将自回归模型中的一个"状态"定义为 (prompt, generated prefix)，即当前生成上下文。不同后训练方法的核心差异在于：

| 方法 | 训练状态来源 | 监督信号 |
|------|------------|---------|
| SFT | 数据集轨迹 | 金标 token |
| 离线 KD | 教师轨迹 | 教师 logits/token |
| OPD | 学生轨迹 | 教师续写/logits |
| RL | 当前策略轨迹 | 奖励 |
| DAgger | 学习者轨迹 | 专家动作 |

形式化后训练步骤为三步操作：(1) 从状态分布 q_k 采样状态 s；(2) 从信号源获取监督 z；(3) 用梯度更新参数。策略变化后，诱导新的状态分布 d^{pi_{k+1}}。

### 实验设计

- **模型**：Qwen3-0.6B-Base + LoRA 适配器，单张 RTX 3090 24GB GPU
- **目标任务**：GSM8K（数学推理）
- **保留任务**：TruthfulQA（多项选择）、MMLU 子集
- **基线模型得分**：GSM8K 0.448, TruthfulQA 0.300, MMLU 0.436
- **方法对比**：轻度 SFT、压力 SFT（5 epochs, lr=5e-4, rank=64）、OPD（来自两种教师）、轻量在线 RL（group-relative 精确答案奖励）
- **状态漂移度量**：从固定 prompt 集采样 rollout，提取前缀状态，用 RBF 核 MMD 度量分布偏移

## Results & Highlights

| 方法 | GSM8K | TruthfulQA | MMLU | 平均遗忘 | 平均保留率 |
|------|-------|-----------|------|---------|-----------|
| Base | 0.448 | 0.300 | 0.436 | - | - |
| Mild SFT | **0.512** | 0.295 | 0.444 | -0.0015 | 1.0008 |
| Stress SFT | 0.420 | 0.245 | 0.364 | 0.0635 | 0.8258 |
| OPD from mild SFT | 0.512 | 0.290 | 0.434 | 0.0060 | 0.9810 |
| OPD from stress SFT | 0.466 | 0.275 | 0.430 | 0.0155 | 0.9515 |
| On-policy RL | 0.472 | 0.290 | 0.442 | 0.0020 | 0.9902 |

**关键发现：**

1. **SFT 的两面性**：轻度 SFT 将 GSM8K 从 0.448 提升到 0.512，且几乎无遗忘；但压力 SFT 不仅严重损害保留能力，连目标任务也退步到 0.420。
2. **学生超越教师**：OPD 从退化的压力 SFT 教师出发，在所有三个任务上都超过了教师（GSM8K: 0.466 vs 0.420; TruthfulQA: 0.275 vs 0.245; MMLU: 0.430 vs 0.364），尽管教师是唯一的监督来源。
3. **RL 的局部性优势**：轻量 RL 提升 GSM8K 至 0.472，平均遗忘仅 0.0020，验证了 on-policy 方法的局部改善特性。
4. **MMD 漂移的局限性**：压力 SFT 和 stress-teacher OPD 的 MMD 漂移几乎相同（0.01093 vs 0.01092），但保留率差异巨大（0.8258 vs 0.9515），说明标量漂移不足以刻画训练动态。

## Critical Assessment

### Strengths

- **概念清晰度高**：将后训练方法沿"状态来源"和"信号来源"两个轴拆解，简洁有力地统一了 SFT、RL、KD 等方法的理解。
- **解释力强**：状态分布视角自然解释了多个已有但缺乏统一解释的现象（遗忘、学生超老师、RL 的稳定性）。
- **实验设计合理**：受控实验直接验证理论预测，而非追求规模或 SOTA。轻度/压力 SFT 的对照设计有说服力。
- **OPD 学生超越教师的演示**：这一结果虽然不全新，但用状态分布视角给出了清晰的理论解释，而非仅作为经验观察。

### Limitations

- **实验规模有限**：仅在 Qwen3-0.6B 上验证，未在更大模型（7B+）上检验。状态分布效应是否会随模型规模变化尚未可知。
- **任务覆盖窄**：仅用 GSM8K 作为目标任务，未测试代码生成、对话、指令遵循等更复杂的后训练场景。
- **理论深度有限**：框架为概念性而非严格理论，没有收敛性保证或样本复杂度分析，本质上是一个"视角"论文。
- **OPD 的实际可行性未讨论**：OPD 需要学生和教师的在线交互，计算成本（每步都需要教师推理）未量化。
- **与现有分析的衔接不足**：论文承认 objective-level 分析"不可或缺"，但未深入讨论状态分布视角如何与现有的 KL 约束、DPO、GRPO 等方法的具体理论分析结合。
- **单作者论文**：缺少多视角的交叉验证，某些实验和解读可能受益于更多讨论。

## Relevance

- 对理解 LLM 后训练流程（SFT -> RLHF/DPO -> 蒸馏）有直接指导意义。
- 为实践中选择后训练策略提供了新的决策维度：不仅考虑目标函数，还需考虑训练状态的来源。
- OPD 学生超越教师的现象对模型蒸馏实践有重要启示：教师模型不需要完美，关键是学生控制状态分布。
- 对研究灾难性遗忘和持续学习的社区提供了新的分析视角。

## Questions for Further Exploration

1. 状态分布视角在更大规模模型（7B-70B）上是否仍然成立？scaling 行为如何？
2. 在多轮对话、代码生成等非数学推理任务上，SFT 的 off-policy 退化和 OPD 的优势是否一致？
3. OPD 的计算开销（需要在线教师推理）能否通过批量化、缓存或近似来降低？
4. 如何将状态分布视角与 DPO、GRPO 等 preference-based 方法结合？这些方法的状态来源是什么？
5. 能否设计一种"最优状态分布"，使后训练在目标任务提升和保留能力之间达到最佳平衡？
6. 压力 SFT 的"压力"定义（epoch 数、学习率、LoRA rank）是否可以更精确地刻画为"状态分布偏移量"的函数？
7. MMD 等标量漂移度量之外，是否有更好的工具来追踪状态分布的"局部可达性"？
