# Hidden Measurement Error in LLM Evaluation Pipelines

> **一句话总结：** 本文利用概化理论（Generalizability Theory）将LLM评估管道的不确定性分解为多个来源，区分了随数据量缩小的方差和研究者设计选择带来的敏感性，发现标准置信区间忽略了关键方差来源导致覆盖率不足，并提出了优化预算分配策略可将MMLU评估误差减半。

> **TL;DR:** Decomposes LLM evaluation pipeline uncertainty into its sources using Generalizability Theory, revealing that standard confidence intervals under-cover and that optimized budget allocation halves estimation error on MMLU compared to standard single-prompt evaluation.

## 核心贡献

- 利用概化理论将LLM测量管道的不确定性分解为多个方差来源（系统提示词变化、评分模型差异、温度设置、项目异质性等），区分可随数据缩小的方差和设计选择带来的不可缩小的方差
- 揭示标准置信区间忽略了这些方差来源，导致覆盖率不足，且问题随数据量增加而恶化

- 提出投影优化管道在意识形态标注、安全分类、MMLU基准和人工验证的宣传审计中优于73%的朴素管道

## Key Contributions

- Variance decomposition of LLM evaluation pipelines using Generalizability Theory, separating data-shrinkable variance from design-choice sensitivity
- Reveals that standard confidence intervals under-cover and worsen with more data
- Identifies exploitable surface for benchmark gaming from unmeasured variance
- Projection-optimized pipelines outperform 73% of naive pipelines against human baseline across 4 tasks
- Optimized budget allocation halves MMLU estimation error at equivalent cost

## Methodology

The author applies Generalizability Theory (G-theory) from psychometrics to LLM evaluation pipelines. The framework decomposes total measurement error into variance components from different pipeline stages: item sampling, system prompt rewording, temperature/seed variation, judge prompt rewording, and judge model selection. Each variance source is classified as either data-shrinkable (reduces with more items) or design-sensitive (requires different prompts/models). Decision studies then identify which design choices contribute exploitable surface for gaming and prescribe robust configurations.

## Results & Highlights

- Semantics-preserving input perturbations account for up to half of performance variance
- Minor perturbations to answer ordering shift MMLU rankings by up to 8 positions
- A "null model" achieves 86.5% win rate on AlpacaEval 2.0
- Projection-optimized pipelines outperform 73% of naive pipelines across 4 evaluation tasks
- Small-sample variance estimation produces confidence intervals approaching nominal coverage

## Limitations
