> **一句话总结：** 针对decoder-only LLM早退(Early Exit)中跳过的层无法为后续token提供KV cache的"KV Cache Absence"问题，提出River-LLM——一种免训练框架，通过轻量级KV-Shared Exit River让跳过层的KV cache在退出过程中自然生成，并用decoder块的状态转移相似度预测累积KV误差来指导精确退出决策，在GSM8K/MATH/HumanEval上实现1.71×到2.16×的实际墙钟加速且近乎无损。

> **TL;DR:** Identifies the "KV Cache Absence" problem as the bottleneck preventing token-level Early Exit in decoder-only LLMs from translating theoretical layer reduction into wall-clock speedup (existing fixes — Batching Recompute, State Propagation, KV Masking — add latency or lose accuracy). Proposes River-LLM, a training-free framework with a lightweight KV-Shared Exit River (4-bit W4A16 PTQ exit layers, FP16 KV cache) that synthesizes the skipped layers' KV cache as a byproduct of the exit path, plus a state-transition-similarity-based exit criterion (cosine between consecutive hidden states, threshold τ). On Llama3.2-1B GSM8K, River-LLM achieves 2.16× speedup (182.9 vs 84.5 tokens/s backbone) at 29.3 accuracy (vs 33.2 backbone); on Llama3.1-8B HumanEval, 1.77× speedup at 55.5 accuracy (vs 57.3 backbone); exit decision logic costs ~100µs (0.0688% of per-token time) on Llama3.1-8B.

## 核心贡献

- 系统识别并定义了**KV Cache Absence问题**：在decoder-only LLM中token提前退出后，跳过的层未计算Key-Value对，导致后续token在那些层缺乏自注意力所需的历史状态；现有方案（Batching Recompute、State Propagation、KV Masking）要么引入显著延迟开销要么精度严重下降。
- 提出**KV-Shared Exit River**：一个轻量级共享KV退出层结构，让退出路径自然生成并保存跳过层的KV cache，作为退出过程的内在副产物，无需事后恢复或重计算；退出层用4-bit权重-only量化(W4A16)而KV cache保持FP16，经部分图编译优化实现2.4×于全精度backbone块的吞吐。
- 设计**基于状态转移相似度的退出准则**：用decoder块输入-输出的余弦相似度s_{t,b}^{(l)}作为累积KV误差的预测代理（第一层状态转移相似度与最后一层backbone-exit value相似度相关r=0.5536, p<0.001），通过阈值τ控制退出决策。
- 证明**免训练、架构无关、零推理架构开销**：整个权重迁移约一分钟完成，合并后模型与backbone架构相同；退出决策逻辑O(d)复杂度，在Llama3.1-8B上仅~100µs（占每token推理时间0.0688%）。

## Key Contributions

- Systematic identification of the KV Cache Absence problem: when a token exits early in a decoder-only LLM, the skipped layers do not compute Key-Value pairs, so subsequent tokens lack the attention history at those layers; a unified testing environment shows existing strategies (Batching Recompute, State Propagation, KV Masking) fail to bridge theoretical layer reduction and practical wall-clock speedup.
- KV-Shared Exit River: a lightweight shared-KV exit layer that synthesizes the skipped layers' KV cache as an intrinsic byproduct of the exit path (Intrinsic KV Integrity), eliminating post-exit recovery; uses 4-bit weight-only quantization (W4A16) on attention/FFN weights with FP16 KV cache, achieving 2.4× throughput over full-precision backbone blocks via partial graph compilation, with average cosine similarity between exit KV and backbone native KV >0.97.
- State-transition-similarity exit criterion: uses the input-output cosine similarity of decoder blocks as a proxy for cumulative KV error (first-layer state transition similarity correlates r=0.5536, p<0.001 with last-layer backbone-exit value similarity), enabling a threshold-τ exit decision D(l) computed in O(d) time.
- Training-free, architecture-agnostic, zero-inference-overhead design: weight transfer completes in ~1 minute; exit decision logic runs in ~100µs (0.0688% of per-token inference time on Llama3.1-8B); supports Backbone Offloading (evicting sparsely-activated deep blocks from VRAM) to approach the memory footprint of a fully-quantized model.

## Methodology

**KV Cache Absence problem (Section 3).** When token t exits at layer l in a decoder-only LLM, layers l+1…L do not compute K/V for t, so subsequent tokens' self-attention at those layers lacks t's history. The authors empirically show (Llama3.2-1B, GSM8K, batch size 1) that while >50% of tokens can theoretically exit early, wall-clock speedup remains marginal because KV recovery dominates. Existing fixes: Batching Recompute (re-run skipped layers, high GPU overhead, needs multi-GPU scheduling), State Propagation (copy hidden states, trades accuracy for overhead), KV Masking (mask missing KV, severe precision loss).

**KV-Shared Exit River (Section 4).** A lightweight exit layer with shared KV computation is inserted; when a token traverses the Exit River, the exit layer naturally substitutes for the skipped decoder layers to produce a complete KV cache as a byproduct, with no post-exit recovery. The exit-layer weights are derived from backbone parameters via Post-Training Quantization (PTQ): 4-bit weight-only (W4A16) on attention and FFN weights, KV cache kept in FP16. Partial graph compilation + specialized kernels give 2.4× throughput over full-precision backbone blocks. Average cosine similarity between exit-layer KV and backbone native KV stays >0.97 regardless of exit point (Fig. 5). The entire weight transfer takes ~1 minute and requires no training.

**Exit decision via state transition similarity (Section 5).** Define s_{t,b}^{(l)} = cos(h_{t,b}^{(l-1)}, h_{t,b}^{(l)}) (input-output cosine of decoder block l). The exit decision D(l) exits at layer l when the batch-averaged s_{t,b}^{(l)} exceeds threshold τ. Empirically, s_{t,b}^{(l)} is roughly monotonically increasing across layers (except the last), conforming to early-exit expectations; the first-layer s correlates (r=0.5536, p<0.001) with the last-layer backbone-exit value similarity, justifying it as a cumulative-KV-error proxy. Complexity is O(d) per decision.

**Backbone Offloading.** Because most tokens terminate early, the framework evicts sparsely-activated deep backbone blocks from VRAM, keeping the Exit River resident — approaching the memory footprint of a fully-quantized model while preserving selective high-precision inference. Granularity adaptively switches by inference phase (token-level for generation, sequence-level for prefill-dominant tasks like MMLU).

**Experimental setup.** Models: Llama3.2-1B (16 layers), Llama3.1-8B (32 layers), Phi4-mini, Ministral3-8B. Benchmarks: common-sense reasoning (BoolQ, HellaSwag, ARC-Challenge, ARC-Easy, MMLU, loglikelihood ranking), mathematical reasoning (GSM8K, MATH), code generation (HumanEval). Default exit threshold τ=0.5 for Llama3.2-1B and Llama3.1-8B, τ=0.9 for Phi4-mini and Ministral3-8B. Baselines: backbone (full precision), Full Quantization (HQQ W4A16 + compilation optimization). Hardware: NVIDIA A40 GPU, batch size 1, averages over 10 tests for speedup. Quantization framework: HQQ (Badri and Shaji, 2023).

## Results & Highlights

- **Llama3.2-1B speedup (Table 3, NVIDIA A40, batch 1):** GSM8K 2.16× (182.9 vs 84.5 tokens/s) at 29.3 accuracy (vs 33.2 backbone, −3.9); MATH 1.88× (189.2 vs 100.5 tokens/s) at 14.6 accuracy (vs 17.8, −3.2); HumanEval 1.71× (171.8 vs 100.4 tokens/s) at 23.2 accuracy (vs 25.8, −2.6).
- **Llama3.1-8B speedup (Table 3):** GSM8K 1.78× (45.0 vs 25.3 tokens/s) at 74.4 accuracy (vs 78.2 backbone, −3.8); MATH 1.72× (43.1 vs 25.1 tokens/s) at 26.6 (vs 27.0, −0.4); HumanEval 1.77× (44.7 vs 25.2 tokens/s) at 55.5 (vs 57.3, −1.8).
- **Phi4-mini speedup (Table 3):** GSM8K 1.61× at 81.0 accuracy (vs 82.1 backbone, −1.1); MATH 1.53× at 35.1 (vs 37.4, −2.3); HumanEval 1.71× at 63.1 (vs 63.4, −0.3) — near-lossless on code generation.
- **Ministral3-8B speedup (Table 3):** GSM8K 1.77× at 84.3 (vs 84.5, −0.2, near-lossless); MATH 1.85× at 46.6 (vs 48.1, −1.5); HumanEval 1.98× at 26.2 (vs 29.3, −3.1).
- **vs Full Quantization (Table 3):** River-LLM is ~10% lower peak throughput than Full Quant. but maintains near-lossless fidelity, whereas Full Quant. suffers significant accuracy degradation (e.g., Llama3.2-1B GSM8K Full Quant. 25.1 vs River-LLM 29.3; Llama3.1-8B GSM8K Full Quant. 69.8 vs River-LLM 74.4).
- **KV integrity (Fig. 5):** Average cosine similarity between exit-layer KV cache and backbone native KV cache remains >0.97 across all exit layers (Llama3.2-1B).
- **Exit decision overhead:** ~100µs per decision on Llama3.1-8B, =0.0688% of per-token inference time.
- **Memory (Fig. 9, Llama3.1-8B, 4K–64K context):** River-LLM's total memory footprint remains consistently and significantly lower than both the backbone and prior early-exit baselines (Balcony, EE-LLM), approaching the memory efficiency of a fully-quantized model.
- **Pareto frontier (Fig. 8, Llama3.2-1B GSM8K):** River-LLM defines a superior Pareto frontier over masking/recomputation/state-propagation methods, which suffer "cliff-like" accuracy drops when pushing throughput.

## Critical Assessment

- **Evaluation is limited to ≤8B models; scalability to 24B/70B is unverified.** The authors explicitly list this as a limitation: "further validation on larger scales, such as 24B and 70B models, is necessary." The KV-Shared Exit River's >0.97 KV cosine similarity and the 2.4× exit-layer throughput are measured only on Llama3.2-1B; whether these hold at 70B (where layer count, hidden dim, and KV cache size scale dramatically) is unknown.
- **Speedup is generation-phase only; prefill-dominant tasks benefit little.** The authors acknowledge River-LLM is "primarily optimized for token-level autoregressive decoding" and that "the speedup is less pronounced for prefill-dominant tasks, such as those within the MMLU benchmark, where a sequence-level exit strategy is currently applied." This means the headline 1.71–2.16× does not apply to long-context prompt processing.
- **Accuracy drops are non-trivial on smaller/weaker models.** Llama3.2-1B loses 3.9 points on GSM8K (33.2→29.3) and 3.2 on MATH (17.8→14.6) — substantial relative drops for already-low baselines; the "near-lossless" claim holds mainly for stronger models (Ministral3-8B, Phi4-mini) where absolute accuracy is high.
- **The exit criterion (state transition similarity, r=0.5536) is a weak predictor.** A correlation of r=0.5536 (R²≈0.31) between the first-layer similarity and the last-layer KV discrepancy means the proxy explains only ~31% of the variance — the exit decision is guided by a noisy signal, which may explain the per-task accuracy variance.
- **No comparison against contemporary learned exit predictors (AdaInfer SVM, D-LLM MLP) under matched conditions.** Table 1 categorizes methods qualitatively (Train/No-Train, overhead High/Medium/Low) but the empirical speedup/accuracy comparison is only against the backbone and Full Quantization, not against D-LLM, CALM, or EE-LLM head-to-head on the same benchmarks — making the "superior Pareto frontier" claim (Fig. 8) a comparison against KV-strategy variants rather than against the strongest competing early-exit systems.

## Strengths

- Correctly identifies and names the KV Cache Absence problem as the root cause of the theory-vs-practice gap in token-level early exit, with a clear unified evaluation (Table 1, Fig. 3) showing existing fixes fail.
- The KV-Shared Exit River is an elegant training-free solution: synthesizing skipped-layer KV as a byproduct of the exit path eliminates the recovery overhead that bottlenecked prior work, and the >0.97 KV cosine similarity empirically validates the approximation.
- Genuinely training-free and fast to deploy (~1 minute weight transfer, no fine-tuning), with a negligible exit-decision overhead (~100µs, 0.0688% of per-token time).
- Backbone Offloading is a practically valuable memory strategy that lets River-LLM approach fully-quantized memory footprints while preserving selective high-precision inference.
- Broad model coverage (Llama3.2-1B, Llama3.1-8B, Phi4-mini, Ministral3-8B) and benchmark suite (8 benchmarks across commonsense, math, code) strengthen the empirical claim.

## Limitations

- Only validated up to 8B; 24B/70B scalability is explicitly flagged as unverified.
- Speedup is generation-phase only; prefill-dominant workloads (e.g., MMLU) see little benefit.
- The exit criterion's predictor correlation (r=0.5536) is weak; exit decisions rest on a noisy proxy.
- No head-to-head empirical comparison against the strongest learned early-exit baselines (D-LLM, CALM, AdaInfer) under matched conditions — only qualitative categorization in Table 1.
- Accuracy drops are material on weaker models (Llama3.2-1B: −3.9 GSM8K, −3.2 MATH); the "near-lossless" framing is model-dependent.
- No code repository is referenced in the PDF, limiting independent reproducibility.

## Relevance

直接映射到 **LLM Inference Evaluation**——River-LLM是一种LLM推理加速方法，本文核心是对早退推理质量与速度权衡的系统评估（KV完整性、退出决策准确性、墙钟加速）。同时与 **Computation Reliability & Precision Errors** 相关：KV-Shared Exit River用4-bit量化退出层近似跳过层的KV cache，其>0.97余弦相似度是对量化/近似误差可靠性的实证；退出决策依赖状态转移相似度预测累积误差，属于计算可靠性监控。

## Questions for Further Exploration

- Does the KV-Shared Exit River's >0.97 KV cosine similarity and 2.4× exit-layer throughput hold at 70B scale, where hidden dim and layer count grow substantially?
- Can a stronger exit predictor (e.g., a lightweight learned probe or a multi-layer similarity aggregate) replace the r=0.5536 first-layer proxy to reduce per-task accuracy variance?
- How does River-LLM compose with prefill-optimization methods (e.g., speculative decoding, chunked prefill) to extend speedups to prefill-dominant workloads?
- Is the Backbone Offloading strategy safe under worst-case token distributions where many tokens require deep processing (e.g., adversarial or highly ambiguous inputs)?
- Can the state-transition-similarity signal serve as a runtime reliability monitor for cumulative KV drift, beyond its use as an exit trigger?
