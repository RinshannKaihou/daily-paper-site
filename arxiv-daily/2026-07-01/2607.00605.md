> **一句话总结：** 在 382M 参数的 LMLM 上提出一个三状态因果审计框架（FULL / DEL-ON / DEL-OFF），对 12,228 次 alias-closure 删除 × 13 个数据库 × 6 种 prompt 进行评估，发现参数泄漏 L̂ = 0.11%（DEL-OFF 下模型几乎从不返回被删答案），但残留的"未遗忘"几乎完全由近邻检索重构（DEL-ON 后正确性中检索伪影率与 R(f) 在所有变体下匹配到舍入精度），残留范围从已发布数据库的 0.7% 到 Noise 对抗拓扑的 13.6%；因此在该类 LMLM 中，"遗忘边界"由数据库管理员而非模型划定。

> **TL;DR:** The paper audits whether deletion-based unlearning actually erases facts in a Limited Memory Language Model (the 382M-param LLaMA2-style LMLM from Zhao et al. 2025). They fix the model and vary only the inference-time database state across three interventions — FULL (entry present), DEL-ON (entry deleted, retrieval enabled), DEL-OFF (entry deleted, retrieval disabled) — and decompose post-deletion correctness into parametric leakage L(f), retrieval-mediated correctness R(f), and a retrieval artifact rate grounded in the inference-time retrieval trace. Across 12,228 paired (DEL-ON, DEL-OFF) evaluations + 12,228 FULL baselines (= 36,684 completions) on 13 databases (1 released + 12 custom built in 3 domains × 4 adversarial topologies: Base/Alias/Noise/Collision) under 6 prompt formulations, parametric leakage is L̂ = 0.11% — i.e. the model almost never recalls a deleted fact from its parameters. The residual "memory" lives in the retrieval graph: R(f) and the artifact rate match within rounding everywhere (proven structurally as Proposition 6.1 under complete alias-closure deletion), so post-deletion correctness is reconstituted from near-neighbor retrieval. The DEL-ON residual ranges from 0.7% on the released database to 13.6% on Noise, and prompt formulation has no independent effect — what survives is a property of retrieval-graph topology, not of the model's surface behavior. The takeaway: for this class of LMLM and deletion procedure, the unlearning boundary is drawn by the database admin (alias/paraphrase closure, retrieval-graph hygiene), not by the model.

## 核心贡献
- 提出一个针对 LMLM 的因果审计框架：通过在推理时变化数据库状态（FULL / DEL-ON / DEL-OFF 三种干预）并在每次调用时记录 retrieval trace，将"删除后正确性"分解为可独立测量的三个因果分量。
- 对单一公开 LMLM 检查点（382M LLaMA2 风格，retriever = all-MiniLM-L6-v2，阈值 0.6，greedy decoding）进行大规模受控评估：12,228 对 (DEL-ON, DEL-OFF) 评测 + 等量 FULL 基线 = 36,684 次模型生成，覆盖 13 个数据库 × 6 种 prompt 形式的全交叉网格。
- 自建四类对抗性数据库拓扑（Base / Alias / Noise / Collision），覆盖三个领域（countries, politicians, sports），其中 Noise 通过近义改写路由到同一 object 来压力测试 paraphrase 泄漏，Collision 通过近邻 subject + 不同 object 测试"漂移到混淆邻居"。
- 形式化 Proposition 6.1：在完整 alias-closure 删除下 R(f) = A(f)，因此实证观察的 R̂ ≈ Â 不仅是经验匹配，更是 alias-closure 完整性的诊断信号。
- 关键经验发现：参数泄漏 L̂ = 0.11%（将原 LMLM 论文 FactScore 下降 ~19 的"preliminary evidence"转为 per-fact 结论）；DEL-ON 残留由数据库拓扑决定（0.7%–13.6%），prompt 风格无独立效应；残留的失败是 retrieval-graph 层的近邻重构（Noise）或混淆邻居漂移（Collision），而非参数记忆。
- 给出明确政策含义：在该类 LMLM 中，"遗忘边界"由数据库管理员通过 alias/paraphrase closure 与检索图拓扑划定，而非由模型本身划定。

## Key Contributions
- **Causal audit framework for LMLMs.** Three inference-time database-state interventions (FULL / DEL-ON / DEL-OFF) with logged retrieval traces, decomposing post-deletion correctness into parametric leakage L(f), retrieval-mediated correctness R(f), and retrieval artifact rate A(f) — the latter two separated by whether the retrieved chunk is gold-equivalent to the deleted fact.
- **Large controlled empirical sweep.** 12,228 paired (DEL-ON, DEL-OFF) evaluations + 12,228 FULL baselines (= 36,684 completions) on a single 382M-param LMLM checkpoint (Zhao et al. 2025), fully crossing 6 prompt formulations × 3 intervention states × 13 databases (1 released + 12 custom).
- **Four adversarial database topologies × three domains.** Base (no alternative path), Alias (alias-only storage, tests closure), Noise (paraphrase decoys routing to the same object — worst case), Collision (near-miss triplets with shared subject but different object, tests confusable-neighbor drift).
- **Structural result: Proposition 6.1.** Under complete alias-closure deletion, R(f) = A(f) identically, converting the empirical R̂ ≈ Â match from a curiosity into a verification that alias-closure deletion is in practice complete.
- **Headline per-fact result.** Parametric leakage L̂ = 0.11%, sharpening the original LMLM paper's "preliminary" FactScore-drop finding into a per-fact statement: the parameters almost never return the deleted answer in DEL-OFF.
- **Per-variant attribution.** DEL-ON residual = Base 3.7% / Alias 6.2% / Collision 7.2% / Noise 13.6% / Released LMLM DB 0.7%. FULL accuracy ≈ 33% on Base/Alias/Noise, ≈ 46% on Collision and the released DB; DEL-OFF exact match is near zero in every variant.
- **Per-prompt ablation.** L(f) + R(f) varies only mildly across prompt styles (cloze 6.1%, few-shot 6.7%, contextual 7.9%, paraphrased 8.7%, continuations 8.7%, direct 9.9%), and the FULL-to-DEL-ON drop is dominated by the FULL baseline, not the residual.
- **Practical recommendation.** Database hygiene (alias closure + paraphrase closure, or write-time canonicalization into a single record with pointers) is the operative lever; parametric unlearning research is largely misdirected for this class.

## Methodology

**Audit framework.** Three inference-time interventions per query, holding the model fixed:
- **FULL** — target fact present in DB, retrieval enabled: measures DB-supported accuracy.
- **DEL-ON** — target fact deleted via alias-closure, retrieval still enabled: tests whether the answer can be recovered via alternative entries, aliases, paraphrases, or retrieval artifacts.
- **DEL-OFF** — target fact deleted and retrieval disabled: isolates parametric recall.

**Three-component decomposition.** For a fact f = (s, r, o):
- Parametric leakage: L(f) = 𝟙[Y(f, DEL-OFF) = o]
- Retrieval-mediated correctness: R(f) = 𝟙[Y(f, DEL-ON) = o ∧ Y(f, DEL-OFF) ≠ o]
- Retrieval artifact indicator: A(f) = 𝟙[Y(f, DEL-ON) = o] · 𝟙[no retained candidate in inference-time trace is gold-equivalent to f]

**Alias-closure deletion.** The deletion set is constructed by enumerating all canonical and alias-equivalent (s, r, o) triplets under a verified alias mapping; deletion is verified by checking no retained triplet is gold-equivalent to the target.

**Adversarial topologies.**
- **Base** — only canonical (s, r, o) per fact; no alternative retrieval path.
- **Alias** — fact stored only under aliased s, r forms; no canonical entry; tests whether alias-closure catches every surface realization.
- **Noise** — Base + decoy triplets routing to the same object via paraphrased subjects (e.g., "Government of United States → Seat of Government → Washington, D.C."); probes near-neighbor paraphrase recovery.
- **Collision** — Base + near-miss triplets sharing subject but routing to a different object (e.g., "United States → Largest City → New York City"); probes retrieval drift onto confusable neighbors.
- Built in 3 domains: countries, politicians, sports. ~100–240 facts per (domain, variant) cell.

**Prompt sweep.** 6 formulations: direct, paraphrased, contextual, cloze, continuations, few-shot. Examples in Appendix A.1.

**Scoring.** Automatic alias-aware exact match, normalized prediction before scoring against the gold object; token-level precision/recall/F1 also recorded. Retrieval traces are logged at every FULL and DEL-ON call so post-deletion correctness can be attributed to explicit DB evidence rather than implicit model behavior.

**Model and retriever.** Single publicly released 382M-param LLaMA2-style LMLM checkpoint; retriever = all-MiniLM-L6-v2; similarity threshold 0.6 with fuzzy-match top-1 fallback; greedy decoding. All experiments on a single checkpoint.

## Results & Highlights

- **Parametric leakage L̂ = 0.11%** across the 12,228 paired evaluations — the model almost never returns the deleted answer in DEL-OFF. This sharpens the original LMLM paper's aggregate FactScore-drop-of-~19 ("preliminary evidence" of non-memorization) into a per-fact statement.
- **DEL-ON residual by variant** (the L(f) + R(f) stack): Base 3.7%, Alias 6.2%, Collision 7.2%, Noise 13.6%, released LMLM DB 0.7%. The artifact-rate bar (orange) matches the R(f) bar (green) within rounding in every variant (Proposition 6.1).
- **DEL-ON residual by prompt style**: direct 9.9%, contextual 7.9%, paraphrased 8.7%, cloze 6.1%, continuations 8.7%, few-shot 6.7% — a much narrower band than the FULL baselines.
- **FULL → DEL-ON drop dominated by FULL baseline.** FULL token F1 spans ∼17% (few-shot) to ∼57% (continuations); DEL-ON F1 compresses to a 7–10% band across all prompts; exact match under FULL ≈ 33% on Base/Alias/Noise and ≈ 46% on Collision and the released DB. Continuations drop ~48 points; few-shot drops only ~10 points.
- **DEL-OFF exact match and F1 are visually indistinguishable from zero** in every variant and every prompt style.
- **Three distinct failure modes illustrated in traces** (Appendix A.3, Table 3): (1) parametric leakage — DEL-OFF still returns gold from parameters (rare, appears concentrated on high-frequency entities); (2) Noise-type near-neighbor paraphrase reconstruction — alias-closure removes the canonical triplet but a paraphrastic decoy survives and routes to the same object; (3) Collision-type confusable-neighbor drift — deletion is complete and no surviving candidate routes to gold, but a near-miss triplet sharing the subject pulls retrieval onto a confusable neighbor.
- **Proposition 6.1 (structural result).** Under complete alias-closure deletion of f, R(f) = A(f) identically. This means the empirical near-equality of R̂ and Â is a sanity check on deletion completeness, not a substantive equality claim — the substantive claim is the topology-dependent magnitude of the residual.
- **Code release.** Code and custom databases at https://github.com/raeesiarya/LMLMAudit.

## Critical Assessment

- **Novelty.** The three-state audit (FULL / DEL-ON / DEL-OFF) with retrieval-trace logging is a meaningful sharpening of standard post-deletion accuracy reporting. The artifact-vs-real-memory distinction — i.e. attributing "correct" post-deletion outputs to either a gold-equivalent retained candidate or a near-neighbor candidate — is the conceptual core and is genuinely informative. Proposition 6.1 is a small but useful structural result that converts an empirical observation into a deletion-completeness diagnostic.
- **Rigor.** Large N (36,684 completions), single fixed checkpoint (no model-size confound), controlled adversarial DB topologies with alias-closure defense verified at deletion time, retrieval traces logged at every relevant call. The "match within rounding" claim is presented as a visual / numerical match rather than with explicit confidence intervals or paired statistical tests — given the multiple comparisons across 13 × 6 cells, this is a minor gap but not a fatal one, since the structural argument in Proposition 6.1 does most of the work.
- **Reproducibility.** Code and custom databases released at https://github.com/raeesiarya/LMLMAudit. The released LMLM DB is the public 54.6M-triplet corpus; the retriever (all-MiniLM-L6-v2), threshold (0.6), fuzzy-match fallback, and greedy decoding are all stated. The custom DBs are small (100–240 facts/cell), explicitly topology-controlled, and described in Table 2 of the paper.
- **Significance.** Highly relevant to anyone deploying RAG / LMLM systems under deletion obligations. The headline finding — that parametric unlearning is largely a solved problem in this architecture (L̂ = 0.11%), and that "real" unlearning is a database hygiene problem — is sharp, actionable, and complicates the narrative that unlearning research needs new parametric methods.
- **Red flags and caveats (explicit in §8 of the paper).**
  - Single 382M checkpoint, single retriever, single threshold, greedy decoding — generalization across model sizes, retrieval architectures, similarity thresholds, and decoding strategies is not established.
  - Custom DBs are small (100–240 facts/cell), 3+ orders of magnitude smaller than the 54.6M-triplet released DB. The audit varies topology at fixed scale; scale × topology interactions are not measured.
  - Restricted to entity-level atomic facts in English under alias-aware exact-match scoring. Multi-hop / compositional knowledge, non-English, sampling-based decoding, and human-judged correctness are out of scope.
  - The alias-closure deletion removes only triplets alias-matching on (s, r, o) simultaneously; the Noise variant deliberately exposes what survives this closure, which is a feature of the audit but means L̂ = 0.11% should not be read as a guarantee that any user-facing "deletion" fully removes access.
  - "Parametric leakage near zero" tests direct recall of the deleted fact; cross-fact leakage (reconstructing the deleted fact by composing non-deleted facts in the same neighborhood) is not tested.

## Strengths
- Clean causal decomposition that separates three failure modes (parametric leakage, retrieval reconstitution via gold-equivalent candidates, retrieval artifacts via near neighbors) instead of collapsing them into one aggregate accuracy number.
- Large, controlled sweep with explicit adversarial DB topologies isolates which database property causes which residual — and identifies Noise (paraphrase decoys) as the worst case at 13.6%.
- The structural result (Proposition 6.1) elevates an empirical observation into a deletion-completeness diagnostic, sharpening the paper's claims.
- Headline finding is counterintuitive and policy-relevant: in this LMLM class, parametric unlearning research is largely misdirected — fix the database, not the model.
- Result that prompt formulation does not control residual leakage is methodologically important — it tells practitioners not to expect prompt engineering to "fix" deletion failures.
- Code release + single-checkpoint setup + logged retrieval traces make the audit re-runnable and falsifiable.

## Limitations
- Scope is limited to one LMLM architecture, one retriever, one similarity threshold, and one decoding strategy; generalization to other unlearning methods (gradient ascent, SISA, fine-tuning-based removal) or non-retrieval LMs is not claimed.
- "Parametric leakage ≈ 0" only tests direct recall of the deleted fact; cross-fact compositional leakage is not in scope.
- Custom adversarial DBs are synthetic and small (100–240 facts/cell); real-world deployment residuals may differ in either direction, and scale × topology interactions are not measured.
- The audit measures what survives and attributes it to one of three channels, but the within-example microscopic mechanism (e.g., which embedding similarity drove the retrieval artifact) is illustrated in only a handful of traces (Table 3).
- Automatic alias-aware exact-match scoring only; semantic / human-judged correctness is out of scope.
- Restriction to English and entity-level atomic facts leaves multi-hop, compositional, multilingual deletion unaddressed.

## Relevance
- 直接命中"模型可靠性与可审计性"主题：把 deletion-based unlearning 从单一聚合正确性拆为参数 / 检索-金等价 / 检索-近邻伪影三个因果分量，为 RAG/LMLM 部署者的合规审计（GDPR-style 删除、模型编辑）提供了可复用的诊断框架；并明确指出在该类 LMLM 中遗忘边界由数据库管理员划定。
- 对"自动化研究 / 自我演进代理"也有间接但重要的意义：如果记忆系统的"遗忘"主要由检索图而非参数决定，则任何长期运行的自主系统都需要可审计的 alias + paraphrase closure，而非依赖模型权重修改。

## Questions for Further Exploration
- Does the L/R/artifact decomposition transfer to non-LMLM unlearning methods (gradient ascent, SISA, fine-tune-and-prune), or is it specific to retrieval-externalized knowledge?
- What is the cross-fact leakage profile — can the model reconstruct a deleted fact by composing non-deleted facts in the same neighborhood under DEL-OFF?
- How sensitive is the 0.7%–13.6% residual range to the retriever (sparse / dense / hybrid), the similarity threshold, and the top-k setting? Sweeping these would separate properties of the embedding model from properties of the LMLM architecture.
- Can paraphrase-closure at write time (canonicalization: store aliases and paraphrastic forms as pointers into one canonical record) reduce the Noise residual to near-zero, while preserving legitimate variant retrieval?
- Can a near-neighbor density / paraphrase-coverage metric on the database predict the artifact rate without running the full audit — i.e. is there a "deletion-completeness score" computable from the database alone?
- Does the framework extend to multi-hop and compositional facts, and to semantic or human-judged (rather than exact-match) correctness?
- Does scaling the audit to the full 54.6M-triplet released database, or to a larger LMLM checkpoint, preserve L̂ ≈ 0?
- Is the parametric-leakage residue (0.11%, concentrated on high-frequency entities) exploitable via membership-inference-style attacks, and what does that imply for deletion-compliance auditing?
