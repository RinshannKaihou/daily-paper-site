> **一句话总结：** 提出 EPC 协议 —— 一个 RFC 式的 LLM 智能体评估器偏好耦合（evaluator preference coupling）测量规范，基于四阶段隔离范式（纯文本 / 纯视觉 / T→V / V→T）、固定超参（α_win=0.08, α_lose=0.04, R=30, 基线 step_by_step）与四项指标（γ、JSD、ECE、Brier），配套 v1.0 参考快照（含 GPT-4o、Qwen、DeepSeek 等 8 个评估器条件、N=122 次独立重复）以及 `vX.Y-Z` 三段式版本号——把会随专有评估器静默更新而衰减的耦合测量，从零散观测升级为可复现、可比对、可被检测为过期的标准化基础设施。

> **TL;DR:** EPC is a protocol-and-snapshot infrastructure paper that fixes a four-phase isolation paradigm (Pure Text / Pure Visual / T→V / V→T) for measuring how evaluator feedback reshapes an LLM agent's strategy distribution in closed TTRL loops. The protocol pins down 11 named strategies, α_win=0.08/α_lose=0.04, R=30 rounds, baseline `step_by_step`, deterministic evaluator prompt (temp=0.0, max_tokens=10), and four metrics (γ, JSD, ECE, Brier) with conformance tests. The accompanying Reference Snapshot v1.0 spans GPT-4o (γ̄=1.176, N=8), GPT-4o-mini (γ̄=1.145), Qwen3.7-plus (γ̄=1.059), qwen-plus (γ̄=0.187, 83% zero-coupling), DeepSeek self-eval (γ̄=0.033, 97% zero), and four cross-gateway June 27, 2026 replications — all tagged with model versions, API endpoints, and measurement dates, and all explicitly time-bound. The `vX.Y-Z` versioning convention (protocol-major / snapshot-minor / evaluator-generation) makes decay explicit. Net: not new empirical findings, but a measurable, auditable, RFC-style infrastructure for the closed-loop evaluator-coupling problem the LLM-agent literature has been stuck on.

## 核心贡献

- **协议本身（EPC v1.0）**：RFC 式、分四阶段隔离范式（Phase 1 纯文本 TTRL → Phase 2 纯视觉 TTRL → Phase 3 由 Phase 1 权重训练视觉任务测 T→V 耦合 → Phase 4 由 Phase 2 权重训练文本任务测 V→T 耦合），含 11 条命名为 `step_by_step / critical_check / first_principles / creative_leap / analogy_meta / evidence_cite / synthesis / counterfactual / visual_grounding / spatial_decompose / aesthetic_frame` 的策略、`α_win=0.08 / α_lose=0.04` 的非对称 TTRL 更新、`step_by_step` 的固定基线、R=30 轮、四项指标（γ / JSD / ECE / Brier）、确定性评估器 prompt（temperature=0.0、max_tokens=10）、manifest 输出 schema 与一致性测试套件。
- **v1.0 参考快照**：基于 5 项独立研究的 N=122 次独立重复，覆盖 8 个评估器条件（GPT-4o api2d、GPT-4o-mini DMXAPI、Qwen3.7-plus 阿里、qwen-plus DashScope、DeepSeek 自评、GPT-4o+DeepSeek 等四组跨网关复制），每条都标注模型版本号、API 端点与测量日期；并附 R=16 自评校准基线（PCI=0.741±0.195、Accuracy=0.931±0.056、ECE=0.308±0.174、Brier=0.269±0.194）。
- **`vX.Y-Z` 三段式版本约定**：X=协议主版本（不兼容变更时递增）、Y=快照次版本（同一评估器的新测量）、Z=评估器代际（如 GPT4o-0806、Qwen3.7-0526），让"测量随专有评估器静默更新而衰减"成为可被检测、可被引用、可被作废的工程事实。
- **开放基础设施 + 配套生态**：协议实现（`epc_protocol.py`，Python 3.8+、无 GPU 需求）、参考快照 JSON、manifest schema、社区贡献模板与一致性测试套件（确定性 / 抛硬币 / 脚本化偏好三类 mock evaluator）一同发布；替代学习率需用 `EPC-v1.0-AltLR`、替代基线用 `EPC-v1.0-AltBaseline`、替代 prompt 用 `EPC-v1.0-AltPrompt` 标记，以保留主路径可比性。

## Key Contributions

- **Protocol (EPC v1.0):** Four-phase isolation paradigm — Phase 1 Pure Text TTRL → Phase 2 Pure Visual TTRL → Phase 3 T→V (start from P1 weights, train on visual) → Phase 4 V→T (start from P2 weights, train on text) — with 11 named natural-language prompt-prefix strategies, asymmetric TTRL (α_win=0.08, α_lose=0.04), fixed baseline `step_by_step`, R=30 rounds, deterministic evaluator prompt, and a JSON manifest schema.
- **Four-metric comparison surface:** γ (L2-normalized relative drift, ||w_A→B − w_B||₂ / ||w_B||₂), JSD (base-e between cross-coupled and pure-domain weight vectors), ECE (calibration of evaluator preference wins against ground-truth accuracy), Brier (mean-squared win-rate vs accuracy). γ is mandatory; JSD strongly recommended for cross-condition reporting; ECE/Brier optional.
- **Reference Snapshot v1.0:** N=122 unique experimental repetitions across 8 evaluator conditions aggregated from 5 independent studies (May 27 – Jun 27, 2026), with model versions, API endpoints (api2d, DMXAPI, Alibaba MaaS, DashScope, official DeepSeek), and measurement dates annotated row-by-row. Includes a 4-condition cross-gateway replication block and an R=16 self-evaluation calibration block.
- **`vX.Y-Z` versioning convention:** Three-axis identifier (protocol-major, snapshot-minor, evaluator-generation) explicitly encoding that measurements decay as proprietary evaluators update — turning "stale numbers" from folklore into an annotatable property.
- **Open infrastructure:** Reference implementation (`epc_protocol.py`), JSON snapshot, manifest schema (`epc_manifest_schema.json`), and conformance test suite (deterministic / coin-flip / scripted-preference mock evaluators) released together, lowering adoption friction and giving an executable definition of "EPC-v1.0-compatible".

## Methodology

- **Four-phase isolation paradigm (the core experimental design):**
  - Phase 1 (Pure Text): TTRL on ≥8 text-domain tasks.
  - Phase 2 (Pure Visual): TTRL on ≥8 visual-adjacent tasks.
  - Phase 3 (T→V coupling): start from Phase 1 weights, train on visual tasks → measure γ_T→V and JSD against w_V.
  - Phase 4 (V→T coupling): start from Phase 2 weights, train on text tasks → measure γ_V→T and JSD against w_T.
  - The coupling coefficient γ_A→B = ||w_A→B − w_B||₂ / ||w_B||₂ isolates how an agent's weight distribution drifts away from a pure-domain reference when the training domain switches.
- **TTRL update rule:** Roulette-wheel sample s_t ∼ w; executor generates under s_t and the fixed baseline `step_by_step`; evaluator performs pairwise A/B comparison (win r_t=1, loss r_t=0); weight update w_{s_t} ← max(0.001, w_{s_t} + α) with α=+0.08 on win, −0.04 on loss, then L1 renormalize. Baseline (`step_by_step`), α_win, α_lose, weight floor (0.001), and round count (R=30) are all fixed; any deviation must be reported and tagged (`-AltLR`, `-AltBaseline`).
- **Evaluator prompt specification:** Verbatim three-line template ("Evaluate. Task: ... A ({strategy_name}): ... B (step_by_step): ... Better? Output only A or B."), temperature=0.0, max_tokens=10, no chain-of-thought prefix; failures-to-parse recorded as ties (excluded from the weight update) with tie rate reported.
- **Statistical-reporting requirements:** Per-seed γ and JSD values (not just means), zero-coupling rate (% seeds with γ=0), 95% bootstrap CIs (2,000 resamples at the seed level); N ≥ 10 seeds for initial screening, N ≥ 30 for publication-grade CI width ≤ 0.1 on γ. Sample-size assumptions are explicit and used to gate interpretations.
- **Rationale for the metric/method choices:** γ via L2 normalization preserves Euclidean geometry of the probability simplex and correlates with JSD at r=0.969 across N=152 paired observations (so γ and JSD track each other faithfully); asymmetric α is justified as conservative against noisy evaluator preferences (false-positive corrections are pushed harder than false negatives); weight floor prevents weight starvation while having negligible effect on the final distribution.
- **Conformance and extensibility:** Reference test suite using deterministic / coin-flip / scripted-preference mock evaluators covers edge cases (all weights at floor, uniform initialization, single-strategy dominance, exact ties); passing qualifies an implementation as EPC-v1.0-compatible. Variant research uses the `-AltLR` / `-AltBaseline` / `-AltPrompt` tags so the core protocol remains comparable.

## Results & Highlights

- **Reference Snapshot v1.0 (Table 1, cross-model, May–June 2026):** GPT-4o 2024-08-06 via api2d (N=8, γ̄_T→V=1.176, zero-coupling 0%); GPT-4o-mini 2024-07-18 via DMXAPI (N=10, γ̄=1.145, 0%); Qwen3.7-plus May-2026 via Alibaba (N=8, γ̄=1.059, 0%); qwen-plus Jun-2026 via DashScope (N=30, γ̄=0.187, 83% zero); DeepSeek-chat self-eval May-2026 (N=30, γ̄=0.033, 97% zero). Gradient is sharp: GPT-4o-class evaluators strongly couple the strategy distribution, Qwen softly couples, DeepSeek self-eval barely couples.
- **Cross-gateway replications (Table 2, June 27 2026, N=5 each):** GPT-4o (DMXAPI) + DeepSeek exec/500-tok cap γ̄=0.795, JSD 0.148/0.211; GPT-4o (DMXAPI) + Qwen exec γ̄=0.724, JSD 0.154/0.189; Qwen (MaaS) + DeepSeek exec γ̄=0.987, JSD 0.208/0.225; DeepSeek (official) + Qwen exec γ̄=0.893, JSD 0.189/0.186. Cross-gateway values cluster in γ̄≈0.72–0.99, supporting the gateway-independence of the coupling phenomenon.
- **Calibration baselines (Table 3, self-eval, N=10, R=16):** PCI 0.741±0.195 (moderate concentration), Accuracy 0.931±0.056 (high), ECE 0.308±0.174 (substantially miscalibrated), Brier 0.269±0.194 (high MSE). The pairing of high accuracy with high ECE/Brier is the paper's clearest evidence that an evaluator's discriminative capacity and its closed-loop calibration are orthogonal.
- **Concrete decay example:** The GPT-4o May-to-June drift in Table 1 rows 1 and 4 demonstrates that coupling measurements can *invert* within ~4 weeks as proprietary evaluators update — a direct empirical instantiation of the "measurement decay" the protocol claims to detect.
- **Sample sizes make uncertainty visible:** Reference snapshot reports N per condition; with N=5 (cross-gateway block) the protocol explicitly warns that CIs are wide, and N=30 conditions get tighter bars — making sample-size-aware interpretation the default rather than an afterthought.

## Authors & Credibility

- **Single author (Zewen Liu)** with this paper **submitted to TMLR (under double-blind review)** and listed as "Anonymous authors" on the PDF header — a curious mismatch worth flagging. The paper self-references a cluster of prior works (Liu 2026a;b;c) as the empirical basis for the snapshot, indicating the same author maintains a coherent line of research across multiple recent papers on evaluator preference dynamics.
- **Submission venue:** TMLR (Transactions on Machine Learning Research) is a reputable open-review venue for standards/methodology papers; well-suited to a protocol contribution.
- **Author is positioning as a *standards author*, not a leaderboard author:** The paper explicitly disclaims new empirical findings and instead frames itself as analogous to an RFC or a measurement standard — appropriate scope for a single author staking reputation on one formalized design.
- **Companion-paper ecosystem:** Multiple prior studies (Contagion Networks arXiv:2606.20493, Memory Contagion arXiv:2606.23195, Who Drifted arXiv:2606.15474, Diagnostic Framework) are cited as upstream feeders to v1.0; the protocol reads like a meta-paper that consolidates a research line.
- **Caveats:** (a) Single-author protocol without visible co-sponsorship from major labs or a working group. (b) The author reports no independent re-implementation; all "EPC measurements" cited in the snapshot come from prior work by the same author (or anonymous review-cycle submissions). (c) Double-blind TMLR submission is anomalous given the abstract credits the author by name — likely a preprint posted before anonymization was finalized.

## Critical Assessment

- **Novelty:** Moderate-to-good. The components (TTRL, pairwise-preference learning, JSD, ECE, Brier) are not new; the novelty is the four-phase isolation paradigm (explicitly distinguishing pure-domain references from coupling vectors), the `vX.Y-Z` versioning convention, and the aggregator-style snapshot that ties disparate prior studies to a single comparable schema. Engineering novelty higher than scientific novelty.
- **Rigor:** High at the *spec* level — every parameter fixed or flagged, every metric defined with reporting requirements, conformance tests provided, and statistical recommendation (N ≥ 30, bootstrap CIs) codified. Rigor is lower at the *empirical* level: (a) Snapshot v1.0 is a *retrospective aggregation* from prior studies (Liu 2026a;b;c, plus anonymous TMLR submissions), not freshly executed under EPC end-to-end by an independent team; (b) GPT-4o measurements used third-party gateways (api2d, DMXAPI) without direct-OpenAI-API replication; (c) "PCI partially reflects output-length preference" (ρ_agg=0.89 at n=6) is itself based on very small N.
- **Reproducibility:** Strongly enabled by the protocol + open code + manifest schema. But "reproducible" here is conditional on the evaluator version: a re-run with a newer GPT-4o is *expected* to give different numbers, so reproducibility is at the *protocol* level rather than the *result* level. The conformance test suite provides a mechanical definition of "implementation-correct", which is genuinely useful.
- **Significance:** Potentially high. The LLM-agent evaluation literature genuinely lacks a shared coordinate system for how evaluator feedback reshapes agent behavior; every closed-loop paper currently re-defines coupling from scratch. EPC gives the field a candidate standard and a versioning scheme, which is exactly the kind of infrastructure the field has been asking for (cf. AgentBeats AAA, HAL, MLCommons AILuminate, HF Community Evals). Whether the community actually adopts `vX.Y-Z` over the next year is the empirical question.
- **Red flags:**
  1. The reference snapshot is single-author-aggregated and the "five independent studies" feeding it appear to all be by the same author (or anonymous submissions whose provenance is hard to verify externally).
  2. No direct-OpenAI-API replication of any GPT-4o measurement — all GPT-4o values flow through third-party gateways (api2d, DMXAPI), so they may partially reflect gateway behavior (the protocol explicitly flags this in Pitfall #2).
  3. "Measurement decay as evaluators update" has no algorithmic sentinel — humans still curate versioning; decay detection is manual rather than automated.
  4. The symmetric learning-rate variant (α_win=α_lose=0.06) "produced zero coupling in all 8 repetitions" on GPT-4o, but the paper concedes this coincides with a known evaluator-drift window, so it cannot be cleanly attributed.
  5. The 8 visual tasks are *text-proxied* (e.g., "describe composing a sunset photograph"), not actually multimodal; the protocol's ecological validity for real multimodal agent loops is therefore limited.
  6. The double-blind TMLR submission with a name visible in the abstract is an editorial anomaly worth checking.

## Strengths

- A genuine problem the community is stuck on — evaluator preference coupling is real, documented, and currently unmeasurable across studies.
- Four-phase isolation paradigm cleanly separates pure-domain training from coupling vectors, which prior work did not formalize.
- Conformance test suite + manifest schema + open implementation give the protocol *mechanical* (not just prose-level) reproducibility.
- Time-aware design: explicit "decay as evaluators update" framing plus `vX.Y-Z` versioning is unusual and honest — most measurement papers pretend their numbers are eternal.
- Cross-model empirical breadth (8 evaluator conditions, 4 cross-gateway replications, self-evaluation calibration block) for an initial snapshot is reasonable.
- Variant tagging (`-AltLR`, `-AltBaseline`, `-AltPrompt`) preserves comparability through a core conformance path while letting the community explore the space.

## Limitations

- v1.0 is a retrospective aggregation of (likely same-author) prior studies, so inter-study comparability is asserted by the protocol rather than independently demonstrated.
- No direct-OpenAI-API GPT-4o measurements; all GPT-4o values are gateway-mediated, and the proxy-confounding warning has not been empirically pruned.
- No automated drift detection — versioning updates are human-curated.
- "Visual" tasks are text-proxied only; real multimodal (image-grounded) agent loops are not yet covered by the snapshot.
- Single-author standards paper with no visible institutional backing, working group, or standards-body endorsement.
- Sample sizes in some snapshot conditions (N=5 cross-gateway, N=8 for several rows) are below the paper's own "publication-grade" recommendation of N ≥ 30, so headline numbers carry wider CIs than the layout suggests.
- The asymmetric α is openly admitted to potentially inflate coupling magnitude; no systematic hyperparameter sensitivity analysis has been run.

## Relevance

- 与"自演化智能体与自动研究""Inference Evaluation"和"ML Reliability foundations"均高度相关：EPC 直接针对 LLM 智能体闭环中评估器反馈污染策略分布这一长期被低估的可靠性问题，并用 RFC 式规范 + 版本化快照把"测量"从一次性实验升级为可被社区共同维护的基础设施，是该方向少见的标准化贡献。同时也呼应了你关注的"silent data corruption" —— 只不过这里的腐蚀对象是评估器行为而非模型权重。

## Questions for Further Exploration

- Can an automated sentinel — e.g., a held-out reference evaluator + Kolmogorov–Smirnov or PSI drift test on evaluator outputs — be added so that snapshot obsolescence is detected algorithmatically rather than curated by humans?
- How robust is the four-phase isolation paradigm when executor and evaluator share a context window, tool inventory, or persistent memory (the realistic agent setting), and when does Phase 1/2 reference-training become infeasible?
- Could `vX.Y-Z` be harmonized with model-card / system-card versioning (Hugging Face, METR, MLCommons AILuminate) so evaluator metadata and model metadata are jointly versioned and joint decay gets tracked?
- What is the empirical relationship between γ (coupling strength) and downstream task degradation across the 8 conditions — is high γ uniformly harmful, task-dependent, or domain-dependent?
- Can the snapshot be extended to multimodal evaluators (vision-language judges, code-execution judges, rubric-locked evaluators) where preference coupling may behave qualitatively differently from text-only evaluators?
- How does the protocol hold up under alternative closed-loop update rules (bandit algorithms, Bradley–Terry preference models, Bayesian updates) — do γ values become directly comparable, or does each rule need its own metric panel?
