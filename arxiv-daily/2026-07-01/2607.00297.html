EPC: A Standardized Protocol for Measuring Evaluator Preference Dynamics in LLM Agent Systems ↗

Zewen Liu · single author · submitted to TMLR (double-blind)  •  2026-07-01

LLM Agents Evaluator Bias Reproducibility TTRL Benchmarking RLHF

Key Metric — Coupling Gradient

1.176 → 0.033

Mean coupling coefficient γ̄ from GPT-4o (0% zero-coupling) down to DeepSeek self-eval (97% zero) — a ~35× spread across 8 evaluator conditions, N=122 repetitions.

Why This Matters

The LLM-agent field has no shared coordinate system for how evaluator feedback reshapes agent behavior — every closed-loop paper redefines coupling from scratch. EPC offers a candidate standard.

⚠ Critical Edge

Snapshot v1.0 is a retrospective, single-author aggregation — the "five independent studies" feeding it appear to all be by the same author, with no independent end-to-end re-run.

Methodology — Four-Phase Isolation Paradigm

PHASE 1
Pure Text
TTRL on ≥8 text tasks → reference w_T
PHASE 2
Pure Visual
TTRL on ≥8 visual-adjacent tasks → w_V
PHASE 3
T→V Coupling
From P1 weights, train visual → measure γ_T→V
PHASE 4
V→T Coupling
From P2 weights, train text → measure γ_V→T
  • Coupling coefficient γA→B = ‖wA→B − wB‖₂ / ‖wB‖₂ — isolates drift away from a pure-domain reference when the training domain switches.
  • TTRL update: roulette-sample a strategy vs fixed baseline step_by_step; evaluator does pairwise A/B; asymmetric weights α_win=+0.08 / α_lose=−0.04, floor 0.001, L1-renormalize, R=30 rounds.
  • 11 named strategies (critical_check, first_principles, counterfactual, visual_grounding …) as natural-language prompt prefixes.
  • Deterministic evaluator prompt: verbatim 3-line template, temp=0.0, max_tokens=10, no CoT; parse failures logged as ties.
  • Four metrics: γ (mandatory), JSD (recommended, tracks γ at r=0.969), ECE, Brier (optional). Reporting demands per-seed values, zero-coupling rate, 2,000-resample bootstrap CIs, N≥30 for publication grade.

Strengths

  • Targets a real, documented, currently-unmeasurable problem: evaluator preference coupling across studies.
  • Four-phase isolation cleanly separates pure-domain training from coupling vectors — prior work never formalized this.
  • Conformance suite + manifest schema + open epc_protocol.py give mechanical, not just prose-level, reproducibility.
  • Time-aware by design: explicit decay framing + vX.Y-Z versioning is unusually honest for a measurement paper.
  • Variant tags (-AltLR, -AltBaseline, -AltPrompt) preserve a comparable core path while inviting exploration.

Limitations

  • v1.0 is a retrospective aggregation of (likely same-author) prior studies; inter-study comparability is asserted, not independently demonstrated.
  • No direct-OpenAI-API GPT-4o runs — all GPT-4o values flow through third-party gateways (api2d, DMXAPI), risking gateway confounds.
  • No automated drift detection; versioning is human-curated with no algorithmic sentinel.
  • "Visual" tasks are text-proxied only — real multimodal agent loops are uncovered, limiting ecological validity.
  • Several headline rows (N=5, N=8) fall below the paper's own N≥30 recommendation, so CIs are wider than the layout implies.
  • Asymmetric α is admitted to potentially inflate coupling; no systematic hyperparameter sensitivity analysis.

Questions for Further Exploration

  • Can an automated sentinel (held-out reference evaluator + KS or PSI drift test) detect snapshot obsolescence algorithmically instead of by human curation?
  • How robust is four-phase isolation when executor and evaluator share a context window, tools, or persistent memory — the realistic agent setting?
  • Could vX.Y-Z be harmonized with model-card / system-card versioning (HF, METR, MLCommons AILuminate) so evaluator and model metadata decay are jointly tracked?
  • What is the empirical link between γ (coupling strength) and downstream task degradation — uniformly harmful, task-dependent, or domain-dependent?
  • Can the snapshot extend to genuinely multimodal evaluators (vision-language, code-execution, rubric-locked judges) where coupling may behave qualitatively differently?
  • How does the protocol hold under alternative update rules (bandits, Bradley–Terry, Bayesian) — do γ values stay comparable or does each rule need its own metric panel?
