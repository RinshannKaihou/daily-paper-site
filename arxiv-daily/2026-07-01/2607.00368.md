> **一句话总结：** 本文指出 LLM 测试时训练（TTT）的"部署记忆/个性化"叙事普遍缺乏行为级证据支撑，提出一个三级声明-证据梯子（S 流式适应 / B 桥接内化 / D 部署时行为学习）和"互斥失败类别 + 显式记忆对照"的评估协议；并在 Qwen3-1.7B/4B/8B 的 nonce-fact LoRA 诊断中证明，one-step LoRA 可将 support/answer ∆NLL 降至 −0.5 至 −1.2，但**贪心自由回忆在 direct/paraphrased/delayed 三种查询下均为 0%**，定量化揭示了代理改进与部署行为之间的鸿沟。

> **TL;DR:** This paper argues that LLM test-time training (TTT) is increasingly motivated by deployment-memory / personalization / sparse post-deployment learning claims, but evaluated almost exclusively with local proxy metrics (perplexity, future-token loss, long-context accuracy, reward) that cannot bear those claims. The authors name this failure mode *evidence migration* and propose a two-part behavioral evaluation framework: (1) a three-rung **claim-calibrated evidence ladder** separating **S** (stream/domain adaptation), **B** (bridge internalization), and **D** (deployment-time behavioral learning); and (2) an evaluation protocol with **matched explicit-memory baselines** and **mutually exclusive failure categories**. They audit 24 TTT/memory-adjacent papers (screened from 40+ candidates through April 2026) and show most reported evidence remains below D-level. A controlled **Qwen3/LoRA nonce-fact diagnostic** confirms the gap empirically: across Qwen3-1.7B/4B/8B, one-step LoRA yields support ∆NLL around −1.0 and answer ∆NLL around −0.5, yet greedy free-form recall is **0/48** for direct, paraphrased, and delayed queries. A stress-update variant raises recall to 72.9% (direct) / 54.2% (paraphrased) but crashes locality from 97.9% to 9.7%; conflict-overwrite checks expose a second failure mode (returns neither stale nor corrected code in 72/72 conflicts). The paper concludes with claim-specific D-level evidence templates for factual / preference / correction / procedure / agent-memory claims and a decision rule that maps supported wording to evidence level.

## 核心贡献
- **命名"证据迁移（evidence migration）"**：把 TTT 工作中"代理指标良好却被用来支持更强的部署记忆叙事"这一隐性失败模式形式化，并作为框架要解决的核心问题。
- **三级声明-证据梯子（S/B/D）**：S 级（流式/领域适应）、B 级（桥接内化：参数化记忆、内部化、自适应）、D 级（部署时行为学习）；论文强调 D 级语言只能用于已经通过稀疏-延迟-上下文移除等行为测试的工作。
- **匹配显式记忆基线 + 互斥失败类别的评估协议**：要求任何 D 级声明的 TTT 方法与检索/长上下文/外部记忆系统在相同证据、查询、评分与预算下对比，并报告独立的失败类别。
- **审计 24 篇 TTT/记忆相关工作**：基于 40+ 候选（截至 2026-04），按 S/B/D 三级编码；结论是大多数工作的最强证据仍停留在 S 级或 B 级，D 级叙事常缺乏支撑。
- **受控 Qwen3/LoRA nonce-fact 诊断实验**：三种 Qwen3 规模（1.7B/4B/8B），one-step LoRA；在 direct/paraphrased/delayed 三种 free-form 查询下都得到 0/48 召回，干净地隔离了代理改进 vs. 部署行为的差距。
- **冲突覆盖与压力更新实证**：在 1.7B 上显示 one-step LoRA 在 72/72 冲突中既不返回旧值也不返回新值；压力更新可将 recall 提到 72.9%，但 locality 从 97.9% 跌到 9.7%。
- **按声明类型给出 D 级证据模板**：f 事实型、preference 型、correction 型、procedure 型、agent/tool memory 型各自的最少证据形状。

## Key Contributions
- **Names *evidence migration*** — the failure mode where TTT results grounded in perplexity / reward / same-stream adaptation are carried into a stronger "memory / personalization / post-deployment learning" narrative without the behavioral tests required for that narrative.
- **A three-rung *claim-calibrated evidence ladder* (S → B → D):**
  - **S (Stream / Domain):** dense same-stream or target-domain evidence — perplexity / future-token loss / long-context accuracy / reward.
  - **B (Bridge / Internalization):** parametric memory, context absorption, self-adaptation — answer NLL, needle recall, ablation probes.
  - **D (Deployment-time / Behavioral):** sparse post-deployment information changes later behavior under paraphrase, delay, locality, conflict, and no-support-context.
- **Evaluation protocol with matched baselines and mutually exclusive failure categories.** Every D-level claim must be compared to retrieval / long-context / external-memory baselines at parity (same support, query, scoring, budgets), and decomposed into non-overlapping failure modes.
- **Audit of 24 TTT/memory-adjacent papers** (screened from 40+ candidates, April 2026). Headline finding: most reported evidence remains below D-level deployment behavior; TTT layers, LaCT, In-Place TTT, TLM, TTRL, SyTTA primarily provide S-level; PERK, Locas, SEAL, TT-SI, MEMORYLLM, Absorber LLM reach B-level; assistant-memory benchmarks are coded as *D targets*, not D-level TTT evidence.
- **Controlled Qwen3/LoRA nonce-fact diagnostic.** Three scales (1.7B / 4B / 8B), single-step LoRA on a sparse access-code fact; measured support ∆NLL, direct/paraphrased answer ∆NLL, and greedy free-form recall after context removal. Result: ∆NLL improves substantially on every metric while **greedy recall stays at 0/48** across all three prompt conditions and all three scales.
- **Stress-update + locality + conflict checks.** Stronger updates raise recall (Qwen3-1.7B: 72.9% direct, 54.2% paraphrased, 35/48 and 26/48) but locality collapses from 97.9% → 9.7%; conflict-overwrite returns neither stale nor corrected code in 72/72 cases (vs. replacement memory: corrected-only in 68/72, both-corrected-and-stale in 4/72).
- **Claim-specific D-level evidence templates.** Factual / preference / correction / procedure / agent-memory each get a minimal evidence bundle and required failure categories (Table A2, A3).
- **Decision protocol (Table 4).** Maps evidence level to *supported wording*, constraining how papers can describe their results.

## Methodology

**Framework design.** The paper builds a two-component evaluation framework:
- *Evidence ladder* — operational coding rule: S if the strongest direct evidence is same-stream, domain, or task adaptation; B if it tests internalization / parametric memory / context absorption / self-adaptation without sparse delayed user behavior; D only if it directly probes later behavior from sparse post-deployment information after context removal.
- *Evaluation protocol* — requires (i) matched explicit-memory baselines (exact-context prompting, BM25/retrieval, replacement memory), (ii) claim-specific behavioral probes, (iii) mutually exclusive failure categories, (iv) disclosed update budgets.

**Audit.** Screened 40+ candidates through April 2026 using bibliography seeding + targeted queries around LLM TTT/TTL, long-context TTT, parametric memory, personalization, continual learning, assistant memory. Inclusion criterion: explicit invocation of TTT/TTL, memory, persistence, context absorption, self-improvement, personalization, continual learning, or assistant memory in title/abstract/introduction/motivation. Coded 24 papers; remaining records treated as background (retrieval-only baselines, model-editing/safety background, non-LLM TTT, architecture-adjacent memory). Each paper is assigned the **strongest** claim level its reported evidence supports, not the broadest motivating statement. Appendix C gives full coding and boundary-case sensitivity.

**Controlled nonce-fact diagnostic.** Qwen3-1.7B / Qwen3-4B / Qwen3-8B; one-step LoRA on a sparse access-code fact ("nonce fact"); support context is then removed and the model is queried for the injected fact under direct, paraphrased, and delayed prompting. ∆NLL = NLL_after − NLL_before (negative ⇒ improvement). Generated success uses normalized first-answer-line containment of the target answer; 48 probes per model. Matched controls: exact-context, BM25-style retrieval, replacement-memory. Appendix D adds stress updates, conflict-overwrite (stale-vs-current), locality checks, seed replications, preference / correction / procedure mini-tasks.

**Statistical reporting.** Standard deviations reported on ∆NLL across 48 probes; greedy success reported as 0/48 (or 35/48 etc.) plus percentages.

## Results & Highlights

- **Proxy–behavior dissociation is reproducible at scale.** Across Qwen3-1.7B / 4B / 8B, support ∆NLL drops by ~1.0 nat (e.g., −1.151, −1.218, −0.966) and answer ∆NLL by ~0.5 nat after a single LoRA step. Greedy free-form recall on direct, paraphrased, and delayed queries is **0.0%** for every scale × condition pair (48/48 failures per cell).
- **The gap is not specific to one model family.** The diagnostic isolates it cleanly on a state-of-the-art open-source family (Qwen3) at three parameter scales.
- **Stronger updates buy recall at the cost of locality.** A stress update on Qwen3-1.7B raises direct recall to 72.9% (35/48) and paraphrased recall to 54.2% (26/48) but locality falls from 97.9% (141/144) to 9.7% (14/144). Mechanism-level gain ⇒ behavioral side effect that the proxy cannot see.
- **Conflict handling fails for parametric TTT but not for replacement memory.** Under stale-vs-correct conflicts, one-step LoRA returns neither value in 72/72 cases, while replacement memory is corrected-only in 68/72 cases (both-corrected-and-stale in 4/72).
- **Retrieval is not a free pass either.** Harder retrieval checks hit Hit@1 = 0/48 under paraphrased-support decoys and 0/24 under stale/current conflicts — the paper is explicit that the point is *matched* comparison, not retrieval-trumps-TTT.
- **Seed replications confirm the diagnostic.** 1.7B three-seed replication: direct / paraphrased / delayed recall remain at 0/48.
- **Preference / correction / procedure mini-tasks.** The diagnostic also runs these on 1.7B and reports that proxy loss improves while greedy behavior stays at 0/24 — same gap, different claim type.
- **Audit pattern.** TTT layers / LaCT / In-Place TTT / TTT-NN / TLM / TTRL / SyTTA → S-level; PERK / Locas / SEAL / TT-SI / MEMORYLLM / Absorber LLM / TTT-Discover → B-level; assistant-memory benchmarks (LoCoMo, LongMemEval, MemoryAgentBench, MemoryBench, MemoryCD, Mem2ActBench) → *D targets* (define the behavioral task family; do not by themselves show that parametric TTT achieves it).

## Critical Assessment
- **Novelty: 4/5.** The general observation "proxy metrics ≠ downstream behavior" is well known in continual learning, model editing, and retrieval evaluation. The novelty is (a) the precise framing against TTT memory claims, (b) the S/B/D ladder that maps *wording* to *evidence*, (c) the nonce-fact diagnostic that isolates the gap across three model scales, and (d) the explicit naming of *evidence migration* as a transferable concept.
- **Rigor: 3.5/5.** The diagnostic is sharp and the audit is transparent about its candidate accounting. Concerns: (i) the audit is qualitative — no quantitative scoring of how badly surveyed papers fail the framework; (ii) "mutually exclusive failure categories" is a strong claim; (iii) the LoRA-on-nonce-fact setup is deliberately clean but does not yet generalize to multi-fact, multi-session, or naturalistic deployment distributions.
- **Reproducibility: 4/5.** Hyperparameters, prompt counts, scoring rules, baselines, conflict tests, stress updates, and seed replications are specified (Appendices B and D). The protocol is operationally precise enough that an independent lab should be able to reproduce the headline 0/48 result on Qwen3-1.7B.
- **Significance: 4/5.** TTT is one of the most active and over-claimed subfields of 2025–2026. A clean methodological hammer that forces papers to (a) match explicit-memory baselines, (b) report mutually exclusive failure categories, and (c) write D-level claims only when they are D-level evidenced is genuinely useful for authors, reviewers, and benchmark designers.
- **Red flags.** None major. Minor risks: (i) "mutually exclusive failure categories" oversimplifies realistic deployment failure modes (partial retrieval + misuse + format failure can co-occur); (ii) matched baselines can be gamed in either direction (stronger RAG, weaker TTT); (iii) the audit does not yet score inter-rater agreement, so coding is single-author-driven (Appendix C mentions boundary-case sensitivity but the protocol is not fully specified).

## Strengths
- Targets a real, growing, and underappreciated evaluation problem in the TTT literature.
- Combines a **normative** framework (what to claim) with a **diagnostic** (how to demonstrate the gap) and a **protocol** (how to evaluate going forward).
- The nonce-fact experiment is a clean, low-cost, easily reproducible stress test — any group with a Qwen3 checkpoint and a LoRA setup can re-run it in an afternoon.
- Explicit memory baselines (MemoryBank, MemGPT, Mem0, LongMem, Dynamic Cheatsheet, MEMORYLLM) prevent the "well, RAG would have done better" escape hatch.
- Cross-scale demonstration (1.7B / 4B / 8B) and seed replication (3 seeds on 1.7B) strengthen the empirical claim.
- Stress-update + locality + conflict checks turn the framework from a one-shot diagnostic into a multi-dimensional behavioral map.
- Acknowledges its own scope: explicit memory may often be the right substrate; the framework is compatible with that view.

## Limitations
- The diagnostic uses a **single adaptation mechanism** (one-step LoRA) on a **single fact-distribution** (single nonce access-code); longer / more aggressive TTT regimes, multi-fact distributions, or natural conversational deployments may behave differently.
- Only **one model family** (Qwen3) is exercised; whether the proxy–behavior gap is universal across Llama, Mistral, Gemma, or closed models is an open empirical question.
- The **audit is qualitative**; no quantitative prevalence estimate or inter-rater reliability is reported.
- **"Mutually exclusive failure categories"** is asserted rather than empirically validated; many realistic failure modes blur together (partial retrieval + wrong formatting, or correct fact + wrong routing in agent settings).
- The framework is **normative**; without reviewer/area-chair adoption, papers will continue to use perplexity-only evidence with D-level wording.
- Stress-update results (72.9% recall, 9.7% locality) show that even when parametric TTT *does* learn the fact, it pays a steep interference cost — but the framework does not yet quantify this trade-off across update budgets.
- The paper does not test whether the framework can be operationalized as an automatic reviewer-bot check, which would be the most scalable path to community adoption.

## Relevance
直接命中用户的研究兴趣——本文对 LLM 测试时训练/记忆的评估方法论提出系统性批评，给出了"声明-证据梯子"和"互斥失败类别 + 匹配显式记忆基线"的可操作协议，并用一个干净可复现的 nonce-fact 实验在三种 Qwen3 规模上定量化了代理改进与部署行为之间的鸿沟。这条工作同时服务"推理评估"（评估方法本身的可靠性）与"ML 可靠性基础"（让 memory/personalization 主张与证据级别对齐），也是当下 LLM 个性化/部署后学习叙事中最需要的一类"标尺"。

## Questions for Further Exploration
- Does the proxy–behavior gap close under **more aggressive TTT regimes** (multi-step LoRA, full-finetune, larger rank adapters, memory-editing methods like ROME/MEMIT/AlphaEdit, soft-prompt / prefix-tuning)?
- Are the **mutually exclusive failure categories** truly non-overlapping on naturalistic multi-fact distributions, or do they fragment into finer sub-modes (partial retrieval + wrong tool call, etc.)?
- How does the framework's verdict change across **other model families** (Llama-3, Mistral, Gemma-2/3, Phi-4, Claude/GPT-style closed models) and across **languages**?
- Can the nonce-fact diagnostic be extended to **deployed** memory settings — long horizons, hundreds of facts, conflicting updates, tool-using agents — to test whether the gap persists or is an artifact of single-fact injection?
- Can the S/B/D ladder be **operationalized as an automatic reviewer-bot check** for ICLR / NeurIPS / ACL, flagging papers whose memory claims are unsupported by behavioral evidence?
- How should the framework handle **procedural** vs. **episodic** memory differently? Dynamic Cheatsheet and TTRL hint that procedural memory may be the more achievable target for parametric TTT; what D-level probe would isolate procedural-memory gains without conflating them with task reward?
- What is the **minimum update budget** at which a TTT method transitions from "S/B-level evidence + D-level wording" to "D-level evidence + D-level wording", and is this transition reproducible across architectures?
- Does the explicit-memory-vs-parametric-TTT trade-off decompose cleanly into **privacy, latency, compression, context-pressure, and offline-operation** axes (as the paper suggests), or are there deployment constraints where the comparison is dominated by a single factor?
