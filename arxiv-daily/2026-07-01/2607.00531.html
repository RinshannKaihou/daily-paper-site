Active-GRPO: Adaptive Imitation and Self-Improving Reasoning for Molecular Optimization

Per-instance adaptive imitate-vs-reinforce, plus a self-promoting memory bank, lifts molecular RLVR past the static-reference ceiling.

Why this matters

A reusable RLVR trick: a self-improving reference stops training from being capped by a weak dataset target — directly applicable to math/code reasoning pipelines, not just molecules.

Critical edge

caveat Numbers are matched-config, not head-to-head with published RePO; backbone & code unconfirmed; Sim actually drops (0.828 → 0.791) — most of the win is fewer no-op copies, a design-aligned read.

Methodology

Task
Instruction-conditioned molecular generation. Context ci = source molecule + optimization instruction; reference mref,i. Reward combines property improvement with Tanimoto similarity; 0 for invalid / constraint-violating outputs.
Base
GRPO (actor-only PPO with within-group reward normalization). RePO adds an answer-span imitation loss Lguide toward the fixed reference. Active-GRPO objective: L = LRL + (1/B) Σi βguide(ci)·Lguide(m*(ci)).
Imitate↔reinforce
Active imitate-reinforce. vtop(c) = mean top-k rollout reward. βguide = βmin + (βmax−βmin)·σ(−α(vtop−vref)). β large when policy lags reference (imitate), shrinks toward βmin once policy surpasses it (reinforce). Weighting is per-instance — same minibatch can span both regimes.
Self-improving ref
Active referencing. Capacity-K per-instance memory bank, seeded with the dataset reference. Candidate is promoted when R(m̂;c) > vref + δ and it passes an admissibility predicate Q (Tanimoto floor, separate from scalar reward). Lowest-reward entry evicted on overflow. Guidance target m*(c) = highest-reward bank entry.
Eval protocol
TOMG-Bench MOLOPT: LogP, MR, QED. Metrics SR, Tanimoto Sim, composite SR×Sim. All methods share backbone, reward family, rollout budget, decoding, and a matched 40GB-A100 config. Mean ± SE over 3 seeds; significance via per-example paired bootstrap (10,000 resamples).

Strengths vs Limitations

Strengths

  • Targets a concrete, well-motivated failure mode — the reference-quality ceiling — with a minimal mechanism.
  • Per-instance adaptive β removes a global imitate-vs-RL hyperparameter tradeoff.
  • Unusually careful evaluation: 3 seeds, paired bootstrap, mechanism-isolating ablations, headroom-conditioned test, β-decay & bank-fill telemetry.
  • Honest about the matched-config caveat and the Sim-drop / no-op story.

Limitations

  • Modest absolute gain over the strongest baseline; Sim decreases.
  • Single benchmark family (TOMG-Bench MOLOPT / MOLEDIT) — external validity to real drug-discovery pipelines untested.
  • Backbone LLM and compute budget not surfaced; code availability unconfirmed in main text.
  • New hyperparameters (βmin, βmax, α, k, K, δ, predicate Q) — sensitivity only partly explored (Appendix C.6).
  • Reward-hacking / mode-collapse risk not deeply stress-tested beyond the no-op analysis.

Questions for further exploration

  1. How sensitive is the imitate-vs-reinforce switch (α, βmin/βmax) to reward noise or mislabeled references, and does it degrade gracefully?
  2. Does active referencing risk reward hacking or mode collapse toward a narrow high-reward molecular region over longer horizons?
  3. Can the paradigm transfer to non-molecular verifiable-reward reasoning (math, code) where "references" are gold solutions?
  4. What backbone LLM and compute produced these numbers, and how does wall-clock compare to plain RePO for the ~6.5% average gain?
  5. Would top-k or diversity-aware bank target selection (mentioned but not used) raise the ceiling further than the current max-reward target?
