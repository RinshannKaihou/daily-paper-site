KnowledgeDebugger

An interactive, GUI-based tool that wraps the EasyEdit knowledge-editing library on top of an LM-Debugger-style interface for inspecting Transformer internals.

>A zero-code GUI wraps ~7 EasyEdit KE methods (ROME, MEMIT, PMET, R-ROME, …) onto LM-Debugger — adding causal-trace, ‖Δ‖, ROME-denominator, ME-PPL50 and target-token rank for mechanistic-interpretability exploration.

01 · Methodology

ARCH
Lightweight frontend GUI + backend loading any HF-compatible Transformer via a model-specific config; KE methods accessed through an adapter class wrapping EasyEdit.
INSPECT
Per-layer top-10 residual-stream unembeddings (pre/post-MLP), dominant MLP value vectors, and the rank of an arbitrary target token — extending LM-Debugger beyond its top-10 view.
INTERVENE
Drag-and-drop intervention queue (LM-Debugger neuron-level on/off/amplify + EasyEdit KE methods); queue order = execution order in one forward pass.
METRICS
Causal Trace (Meng 2023a); ‖Δ‖ Frobenius norm of the ROME update; ROME denominator (C⁻¹k⋆)ᵀk⋆; ME-PPL50 perplexity; target-token rank; plus an extensible user-metric API.
REPRO
Each scenario exported as a JSON checkpoint; replayable in the hosted demo.

02 · Strengths vs Limitations

+ Strengths

  • Turns a multi-library KE workflow into one GUI + JSON checkpoint pipeline.
  • Extends LM-Debugger with the exact ‖Δ‖ / denominator / PPL / target-rank metrics used in the recent ROME-collapse literature.
  • Drag-and-drop intervention queue over "compose multiple edits / neuron-level interventions on one forward pass".
  • Reproducibility: open code, public demo, exported JSON checkpoints, video walkthrough.

− Limitations

  • Tool paper — no new KE method, no new localization theory.
  • Evaluation = 3 case studies on GPT2-XL; no benchmark vs. TransformerLens / NNsight / pyvene.
  • The 7 wired KE methods not exhaustively listed in the paper.
  • GUI likely doesn't support fine-grained hyperparameter sweeps.
  • No performance / VRAM characterization on large LMs (70B+).

04 · Questions for Further Exploration

  1. Does the tool integrate cleanly with TransformerLens, NNsight, or pyvene for head- / circuit-level edits beyond single-layer ROME?
  2. How does it behave on multi-edit / sequential-edit scenarios where Gupta 2024b's gradual forgetting is the central concern?
  3. Can the JSON checkpoint format drive headless / batch-mode sweeps for systematic benchmarking?
  4. What is the variance of causal-trace layers across rephrasings at larger sample sizes / model scales / knowledge categories?
  5. Does R-ROME actually resolve collapse cases end-to-end inside the tool, or only paper-over diagnostic thresholds?
  6. Is there a public user study / teaching deployment showing the tool accelerates KE pedagogy vs. scripting EasyEdit directly?
