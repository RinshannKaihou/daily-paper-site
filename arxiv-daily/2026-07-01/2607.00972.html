A Bayesian Network that fuses per-agent uncertainty — and collapses the moment upstream signals go below random.

Why this matters

It gives safety-critical operators an interpretable, per-agent view of where system confidence is coming from — without retraining the LLM.

Critical edge

The deterministic OR-gate treats every node as equally reliable. On StrategyQA the Planner and Evaluator sit below random AUROC — and the OR gate cannot down-weight them, so the Overall node degrades past every single-node signal.

  • 1Token entropy — log-prob of the generated sequence, length-normalised to q = exp(s), then calibrated into a per-node success prior.
  • 2Wasserstein semantic divergence — distance between the committed output and re-samples in a projected embedding space; replaces UProp's Gaussian-kernel PMI to dodge Semantic Entropy's JSON-collapse.
  • 3P(True) self-evaluation — single-token "are you sure?" probe after generation; log-prob of the affirmative token = Generator uncertainty.

The CPT is a deterministic OR: Overall = OK iff all three parents are OK (threshold 0.5). The BN is an abstracted execution trace — no temporal modelling, no feedback loops, no retries.

The BN only helps when uncertainty genuinely accumulates. On StrategyQA the Planner + Evaluator nodes drop below 0.5 AUROC and the OR-gate compounds the noise — a textbook negative result that the authors publish without hedging.

●Strengths · Limitations

✓ Strengths

  • Interpretable DAG — operator sees which agent drove the system-level fail.
  • Negligible compute overhead vs LLM inference.
  • Honest negative result on StrategyQA — OR-gate breakdown isn't hidden.
  • Four-metric battery: AUROC + AUARC + ECE + Brier, not just accuracy.
  • Risk-averse bias matches safety-critical industrial use cases.

✗ Limitations

  • Deterministic OR gate treats weak and strong nodes equally.
  • P(True) is systematically over-conservative; no post-hoc calibration in headline results.
  • Noisy-OR + learned inhibition flagged as the most impactful fix — not implemented.
  • Only Wikipedia-retrieval QA; the OSW motivation is unsubstantiated.
  • n=200 per benchmark, two commercial models, no ablations.
  • Depends on OpenAI log-probs, which have been deprecated before.

●Questions for further exploration

  1. How would a noisy-OR CPT with learned inhibition weights (or a small attention-weighted combiner) behave on StrategyQA, where per-node reliability is heterogeneous?
  2. Does post-hoc calibration of P(True) (isotonic on a held-out set) preserve AUROC while fixing Brier degradation on StrategyQA?
  3. Can the framework extend beyond static DAGs to temporal BNs that model feedback loops and self-correction iterations — which the current structure explicitly excludes?
  4. How robust is the BN when the retrieval corpus is contaminated or OOD (e.g. adversarial Wikipedia edits), where retrieval-based signals may be misleadingly confident?
  5. Would the same architecture transfer to non-QA agentic workflows (code generation, planning, tool use) where "Overall OK/FAIL" is more ambiguous than factual correctness?
  6. Can semantic divergence be computed cheaply enough for production, given N=10 re-samples and embedding-projection cost per step?
