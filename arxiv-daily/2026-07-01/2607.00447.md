> **一句话总结：** 把 LLM 幻觉重构为"推理失对齐"(inference misalignment):模型拥有相关知识,但在 prompt 约束与统计先验冲突时被 shortcut 路径劫持;基于 latent key–task 形式化模型预测 key-selection / task-retrieval 两类偏差,并发布 TrapQA(ScientistQA 2925 题 + Real-Life Constrained QA 500 题)在 GPT-5.2 / Gemini 3.1 / Claude Sonnet 4.6 / DeepSeek V3.2 上验证"事实探针答对却配对消歧失败",证明错误并非来自知识缺失。

> **TL;DR:** This paper reframes LLM hallucination as **inference misalignment** — the model has the relevant facts but follows a statistically dominant shortcut path that overrides the prompt's decisive constraint. A **latent key–task model** formalizes this as a two-stage posterior (`identify key → retrieve task → generate`); under three light assumptions, Theorem 3.4 proves the shortcut posterior can dominate the constraint-sensitive one whenever the shortcut pair has higher pretraining frequency, and Theorem 3.6 lower-bounds the resulting inference loss. The theory predicts two testable failure modes: **key-selection bias** (wrong cue dominates) and **task-retrieval bias** (right cue, wrong relation). The authors release **TrapQA**: ScientistQA (2,925 Wikipedia-profile disambiguation items with paired closed-book probes) plus Real-Life Constrained QA (500 SWOW-derived two-option scenarios spanning 13 aspects of daily life). External tools disabled; eight model×thinking settings. Names-only ScientistQA hallucination ranges 2.50–37.23%, while profiles-in-context drops to 0–3.38%; probe analysis shows factual ignorance accounts for at most ~5.5% of errors — many hallucinations occur when both probes succeed, isolating inference-path failure from knowledge storage.

## 核心贡献

- 提出 **inference misalignment** 视角:幻觉不一定源于知识缺失,而是 prompt 约束被统计显著的潜变量先验压制。
- 建立 **latent key–task model** 形式化框架:用 key–task 对的后验刻画推理,理论证明预训练频率失衡可使 shortcut 后验压过 constraint-sensitive 后验(定理 3.4),并直接给出推理损失的非平凡下界(定理 3.6)。
- 从理论推导出两类**可证伪**的失败模式:**key-selection bias**(选择错误的提示线索)与 **task-retrieval bias**(线索对但检索到错误的关系/属性)。
- 发布 **TrapQA** 受控诊断基准:包含 **ScientistQA**(2925 题,基于 Wikipedia 相似科学家对,带 eliminative / compatibility 闭卷探针)与 **Real-Life Constrained QA**(500 题,基于 SWOW 联想词的日常二选一情景,覆盖 13 个日常方面)。
- 设计**探针消歧**实验范式:同一道消歧题配两个事实探针,把"事实无知识"与"知识未被部署"分离开来;八个模型×思考设置上多数错误同时两个探针都答对,直接支持"错误源于推理路径而非知识存储"。

## Key Contributions

- **Conceptual reframing.** Hallucination = *inference misalignment*, a mismatch between the answer supported by the prompt and the answer favored by statistically salient latent associations — distinguishing it from missing-knowledge accounts.
- **Formal model.** A **latent key–task** model of LLM inference (key → task → output) with three structural assumptions (activated-key restriction, activated-task restriction, output separation). Under these, Theorem 3.4 shows shortcut posterior can dominate constraint-sensitive posterior whenever pretraining frequency favors the shortcut pair, and Theorem 3.6 turns this dominance into a quantitative hallucination lower bound via total-variation inference loss.
- **Two testable failure modes.** Key-selection bias (wrong salient cue) and task-retrieval bias (right cue, wrong relation), each illustrated with canonical examples (walking 50 m vs. driving to a car wash; Einstein vs. Fermi with the "did not formulate special relativity" constraint).
- **TrapQA benchmark.** ScientistQA (2,925 Wikipedia-derived disambiguation items with paired eliminative and compatibility probes; tests task-retrieval bias) and Real-Life Constrained QA (500 SWOW-derived everyday two-option scenarios across vehicle/delivery/recording/tool/etc. template families; tests key-selection bias). Closed-book by design — no retrieval, no web.
- **Empirical isolation of inference-path failure.** Probe-conditioned tables show hallucinations occur even when the model answers both decisive facts correctly in isolation; this is the paper's central evidence that hallucination is often a *deployment* failure, not a *storage* failure.
- **Dissociation from raw fame.** Detailed fame analysis (Wikipedia page views, length, external links) shows the shortcut is **relation-specific**, not a generic "pick the more famous entity" prior — actually, errors are slightly lower when the wrong candidate is more famous.

## Methodology

**Theory.** Let `z` be the prompt, `P(·|z)` the model's continuation distribution, `P⋆(·|z)` the ideal distribution. Inference loss is defined as total-variation distance `ℓ(z) = ‖P(·|z) − P⋆(·|z)‖_TV`. Latent decomposition:
```
P(y|z) = Σ_{k∈K, t∈T} P(k,t|z) · P(y|z; k, t)
```
with empirical key distribution `π(k)(k_i) = c_i^{(k)} / C^{(k)}` and conditional task distribution `π(t)(t_j | k) = c_j^{(t)}(k) / C^{(t)}(k)` proxying pretraining frequencies. Three assumptions (3.1 activated-key restriction, 3.2 activated-task restriction, 3.3 output separation) make the shortcut-vs-correct posterior ratio reduce to a prior-times-likelihood product. **Theorem 3.4 (Shortcut Probability Dominance):** if the shortcut pair has sufficiently higher pretraining frequency, `P(y_s|z) / P(y∗|z) ≳ 1`. **Theorem 3.6 (Hallucination Lower Bound):** under a target-preference-margin assumption, shortcut dominance implies `ℓ(z) ≥ ½(γ(z) + γ⋆(z)) > 0`. The two failure modes correspond to the two multiplicative terms of the dominance ratio.

**Benchmark 1 — ScientistQA (task-retrieval bias).** Pipeline:
1. Source scientist pairs from Wikipedia-linked profiles (Wikipedia contributors, 2026), strip names, embed attributes with `text-embedding-3-small`, retain highly similar pairs under a sparsity-adjusted similarity score.
2. Gemini generates a shared biographical description plus one decisive constraint that rules out exactly one candidate.
3. GPT filtering removes invalid items; final set = 2,925 questions.
4. Two prompt conditions: **prepend_names** (only two candidate names — retrieval-sensitive) and **prepend_profiles** (both candidate profiles supplied — retrieval-relaxed control).
5. Two **closed-book probes** per item: one eliminative probe (against the distractor) and one compatibility probe (for the correct candidate). Correct response = normalized match to ground-truth candidate; ruled-out candidate, off-option response, or shortcut Real-Life action counts as an error.

**Benchmark 2 — Real-Life Constrained QA (key-selection bias).** Pipeline:
1. Derive high-salience cues from SWOW (De Deyne et al., 2019).
2. Organize around eight seed template families: `vehicle_required`, `delivery_medium`, `recording_medium`, `tool_required`, plus four more (live-theater, corporate, agricultural-machinery, high-stakes-event).
3. GPT instantiates natural two-option scenarios in which one option is superficially tempting but violates a prompt-grounded physical, spatial, procedural, or medium-specific constraint.
4. Claude filtering + controlled perturbations; final set = 500 questions spanning 13 aspects of daily life.

**Models & evaluation.** Eight configurations: **GPT 5.2** (low/high thinking), **Gemini 3.1 Pro Preview** (low/high thinking), **Claude Sonnet 4.6** (low/high thinking), **DeepSeek V3.2** (Chat / Reasoner). Real-Life Constrained QA reports GPT, Claude, Gemini, DeepSeek-chat. External tools (web search) disabled in **all** runs. Default decoding unless noted; prompt templates and answer normalization in Appendix E.

## Results & Highlights

**ScientistQA — names-only vs. profiles-in-context (Table 1, 2,925 questions):**

| Model | Inference | Names-attached errors | Profiles-attached errors |
|---|---|---|---|
| Claude Sonnet 4.6 | low thinking | 23.90% | 0.17% |
| Claude Sonnet 4.6 | high thinking | 6.22% | 0.00% |
| DeepSeek V3.2 Chat | non-reasoning | 37.23% | 3.38% |
| DeepSeek V3.2 Reasoner | reasoning | 10.56% | 0.00% |
| Gemini 3.1 Pro Preview | low thinking | 2.50% | 0.00% |
| Gemini 3.1 Pro Preview | high thinking | 3.15% | 0.00% |
| GPT-5.2 | low thinking | 11.76% | 0.00% |
| GPT-5.2 | high thinking | 10.26% | 0.00% |

The names-only → profiles-in-context collapse (e.g., DeepSeek Chat 37.23% → 3.38%; six settings reach 0%) shows the failures are about *retrieving and applying* a decisive relation, not *comparing* two supplied profiles. Gemini is the strongest names-only model; Claude and DeepSeek benefit substantially from more thinking effort, while GPT improves only marginally and Gemini slightly regresses.

**Probe-conditioned analysis (Table 2).** Conditioning on whether the model answers both probes correctly:

- Hallucination given both probes correct ("Known-fact hall.") is high across the board: e.g., Claude-low 60.66%, DeepSeek-low 55.10%, GPT-low 57.56%, Gemini-low 78.08%.
- Hallucination given both probes wrong ("Probe-absent hall.") is very low: ≤5.50% across all settings.
- **Factual ignorance accounts for at most ~5.5% of errors.** Most hallucinations occur when the model demonstrably knows the relevant facts in isolation.

**Fame analysis (Appendix H.5).** The shortcut is **not** "pick the more famous entity":
- Wrong candidate is more famous in 61.30% of ScientistQA items (base rate), but hallucination rates are *lower* in those cases (e.g., GPT-high 8.20% vs. 13.52%; Claude-low 17.40% vs. 34.19%; DeepSeek-low 34.69% vs. 41.25%).
- When at least one candidate is in the top 1% by fame rank, hallucination falls in all eight settings (GPT-high 10.87% → 5.11%; DeepSeek-low 38.36% → 27.80%).
- Conclusion: the shortcut is **relation-specific** (institutions, awards, roles, fields), not a generic fame prior.

**Real-Life Constrained QA (Table 11, 500 questions).** Errors per model:
- Claude: 81 → 16.20%
- GPT: 44 → 8.80%
- Gemini: 18 → 3.60%
- DeepSeek-chat: 182 → 36.40%

These failures are not encyclopedic — they occur in everyday action choice when a salient shortcut action conflicts with a physical/spatial/procedural/medium-specific constraint stated in the prompt. Figure 2 shows representative examples (towing vs. walking a trailer to an inspection pad; passing a message to a projection operator during live supertitles; submitting original vs. copied toll receipts to payroll; bringing a compact tractor to a belt-slip ramp).

**Theoretical highlights.** Theorem 3.4 turns the qualitative "shortcut dominates" intuition into a *quantitative* posterior ratio bound; Theorem 3.6 connects this posterior reversal to a non-vanishing inference-loss floor. Assumption 3.1 (activated-key restriction) is explicitly characterized as the regime in which the two candidate keys are *pretraining-independent* — when they have co-occurred during pretraining and the model has learned a within-context disambiguation, the posterior can depart from the prior ratio and hallucination is avoided. The paper reports a "Knowledge Consistent" setting (Section 5.2) consistent with this prediction.

## Critical Assessment

- **Novelty (moderate–high).** The "knowledge vs. inference path" split is a long-running concern in hallucination research; what is genuinely new here is (i) the **latent key–task** formalism that converts this intuition into a posterior-dominance bound, (ii) the clean **two-mode decomposition** (key-selection vs. task-retrieval), and (iii) a benchmark whose probe design *operationalizes* the knowledge/inference-path confound in a single item. Closely related work (e.g., Sun et al. 2025b on subsequence associations, Kalai & Vempala on calibrated-LM hallucinations, McKenna et al. 2023 on inference-task hallucinations) addresses overlapping phenomena but from different angles.
- **Rigor.** Strengths: explicit assumptions (3.1–3.3, 3.5), two theorems with proofs (Appendix K.2, K.3), and a controlled benchmark with paired probes that isolates confounders. Weaknesses: Assumption 3.1 is strong (the "pretraining-independent" regime is identified *post hoc* — when within-context disambiguation succeeds, we say 3.1 was violated). The shortcut-path probability decomposition is intuitive but doesn't yield a quantitative *prediction* of *which* trap items a model will fail on; it only bounds the *aggregate* rate. Probe-independence from the trap question is asserted but not stress-tested (e.g., paraphrased probes).
- **Reproducibility.** TrapQA is released with full construction pipeline. Reruns should match closely *in the closed-book setting*, but the authors flag that provider-hosted models may drift; future audits should treat answers as time-stamped. Several confounders that the paper does not explicitly rule out: (a) lexical overlap between the decisive constraint and pretraining-typical sentences; (b) position bias in two-option prompts; (c) the choice of SWOW cues as "high-salience" — these are normative English associations, not necessarily the cues a model has internalized. Open data, but no released model checkpoints or fine-tuning to *test* the theory's prescriptions.
- **Significance.** If the theoretical claim holds across more domains, this reframes hallucination mitigation from "more knowledge" to "better path selection/weighting" — pointing toward decoding-time interventions (e.g., anti-prior reweighting, retrieved-context forcing), reasoning-trace regularization, or path-aware fine-tuning. TrapQA could become a standard stress test alongside TruthfulQA / HaluEval, with the distinctive diagnostic value of separating ignorance from deployment failure.
- **Red flags to monitor.** (i) Within-context disambiguation success might reflect the model having been *trained on disambiguation patterns*, not a fundamental refutation of Assumption 3.1; (ii) the theory is diagnostic, not prescriptive — the conclusion explicitly calls for new methods, but none are proposed; (iii) the Real-Life Constrained QA numbers vary 10× across models (Gemini 3.6% vs. DeepSeek-chat 36.4%), suggesting either substantial prompt-sensitivity or model-specific instruction-following weaknesses rather than a uniform "inference misalignment" phenomenon; (iv) some claims (e.g., "raw fame does not explain the shortcut") are made on a single Wikipedia-derived fame proxy — robustness across alternative fame measures is not demonstrated.

## Strengths

- Clean theoretical separation between **knowledge storage** and **inference-path deployment**, with two theorems turning this into a posterior bound and a loss lower bound.
- **Probe-conditioned evaluation** that lets a single benchmark item simultaneously answer "does the model know?" and "can the model deploy what it knows?" — an unusually clean experimental design.
- **Two complementary benchmarks** (entity disambiguation + everyday action choice) stress different parts of the latent key–task pipeline.
- **Eight model×thinking configurations** spanning four frontier families (GPT, Gemini, Claude, DeepSeek), giving the qualitative pattern cross-vendor support.
- **Anti-fame control** is a strong move: directly rules out the most obvious "shortcut = popularity" confound.
- Senior authorship (Nowak) lends credibility to the formal theorems; the theory is stated, proved, and accompanied by a clear intuitive gloss.

## Limitations

- **Limited model coverage.** Results are restricted to GPT 5.2, Gemini 3.1 Pro Preview, Claude Sonnet 4.6, and DeepSeek V3.2 (chat/reasoner); no open-weight models. Provider-hosted model drift may invalidate time-stamped comparisons.
- **No pre-registered intervention.** The framework is diagnostic; the conclusion states "need for methods that improve how models select, weight, and execute latent inference paths" but does not propose or evaluate one. Reweighting decoding, constrained decoding, or anti-shortcut fine-tuning would be the natural next step.
- **Closed-domain diagnosis.** TrapQA targets two specific failure modes; multi-step, compositional, or open-ended hallucinations are not directly addressed. Whether the same shortcut-dominance mechanism applies to long-form generation is an open empirical question.
- **Potential confounds.** Position bias in two-option prompts, lexical overlap between the decisive constraint and the trap question, and the choice of SWOW as a "salience oracle" are not deeply interrogated.
- **Theory is partial.** Assumptions 3.1 and 3.5 are strong and likely violated in many realistic prompts; Theorem 3.4 gives a *bound*, not a *prediction* of which specific items will fail. Pretraining-frequency proxies are corpus-dependent and may not reflect the model's actual internal priors.
- **Probe independence assumption.** The probes are derived "from the decisive constraint" but are assumed to be independent of the disambiguation failure; this is plausible but not stress-tested with paraphrased or interleaved probes.
- **Temporal drift.** Author-acknowledged: TrapQA answers may become stale as careers, awards, and "common sense" evolve; future evaluations must re-audit items.

## Relevance

本研究高度契合"ML Reliability foundations"和"Hidden States & Interpretability"方向:它把幻觉定位为**潜变量推理路径被统计先验劫持**的现象,既给出 latent key–task 的形式化刻画,又以受控基准 + 闭卷探针将"知识缺失"与"推理路径失效"在单题层面分离开来,对设计 path-aware 解码干预、解释隐藏状态层面的 shortcut 信号、以及构建系统化的可靠性诊断卡片(系统卡)都有直接借鉴价值。

## Questions for Further Exploration

- Can the **key-selection vs. task-retrieval** decomposition be identified *inside* the model (via hidden-state probes, attention pattern analysis, or causal interventions on specific attention heads) rather than only inferred from behavioral errors?
- How does shortcut-path dominance scale with **model size, RLHF/fine-tuning regime, and chain-of-thought prompting**? Gemini's robustness (2.50% names-only) suggests possible architectural or training-pipeline advantages worth isolating.
- Does **decoding-time intervention** (e.g., contrastive reweighting against the shortcut prior, retrieved-context forcing, self-consistency) reduce key-selection bias, or merely shift the shortcut to another layer?
- Can TrapQA be extended to **multilingual, multimodal, or long-form generation** settings to test whether inference misalignment is a cross-modal phenomenon or specific to short-form closed-book QA?
- What is the quantitative relationship between **pretraining entity / relation frequency** and observed task-retrieval bias — is the correlation strong enough to *predict* which trap items a given model will fail on, validating Theorem 3.4 at the per-item level?
- Can the latent key–task framework be connected to **retrieval-augmented generation** or **tool-use** as a *mechanism* that re-routes the constraint-sensitive path, and does RAG actually suppress shortcut dominance on TrapQA-style items?
- Is there a way to *induce* the within-context disambiguation regime (Assumption 3.1 violation) deliberately — e.g., via in-context examples that pair each candidate with its decisive constraint — and does this generalize, or only memorize?
- How does **inference-time scaling** (more reasoning tokens, verifier reranking, self-refinement) interact with shortcut dominance? The mixed Gemini result (high-thinking slightly worse than low-thinking) deserves mechanistic investigation.
- Can the framework yield **prescriptive interventions** — e.g., a fine-tuning loss that explicitly downweights shortcut priors — and do these generalize out-of-distribution beyond TrapQA?
