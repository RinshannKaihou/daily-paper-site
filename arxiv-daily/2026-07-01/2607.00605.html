Auditing Forgetting in Limited Memory Language Models
Arya Raeesi · Hanna Roed · UC Berkeley · 2026-07-01 · arXiv:2607.00605
LLM Unlearning RAG Limited Memory LMs Causal Auditing GDPR-style Deletion
Key metric

0.11%

parametric leakage L̂, across 12,228 paired (DEL-ON, DEL-OFF) evaluations
DEL-OFF exact match is visually indistinguishable from zero in every variant and prompt style.
Why this matters
Gives RAG / LMLM deployers a three-channel causal audit of deletion — separating parametric, retrieval-gold-equivalent, and near-neighbor artifact paths. For this architecture class, parametric unlearning research is largely misdirected; the operative lever is database hygiene (alias / paraphrase closure).
Critical edge
L̂ = 0.11% tests only direct recall. Cross-fact compositional leakage, scale × topology interactions, non-English, and multi-hop / compositional facts are all out of scope. The sharp number is a diagnostic, not a deployment guarantee.

01Methodology

Three inference-time interventions on a single 382M LLaMA2-style LMLM (Zhao et al. 2025), retriever = all-MiniLM-L6-v2, threshold 0.6, greedy decoding:

FULLTarget fact present, retrieval on. DB-supported accuracy.
DEL-ONFact deleted via alias-closure, retrieval still on.
DEL-OFFFact deleted and retrieval disabled. Isolates parameters.

Causal decomposition per fact f = (s, r, o):

  • L(f) = 𝟙[Y(f, DEL-OFF) = o]  — parametric leakage
  • R(f) = 𝟙[Y(f, DEL-ON) = o ∧ DEL-OFF ≠ o]  — retrieval-mediated correctness
  • A(f) = 𝟙[Y(f, DEL-ON) = o] · 𝟙[no retained candidate is gold-equivalent to f]  — artifact

Alias-closure deletion: verified that no retained triplet is gold-equivalent to the target.

Adversarial topologies × 3 domains (countries / politicians / sports, 100–240 facts/cell):

  • Base — only canonical (s, r, o); no alternative path
  • Alias — only aliased forms stored; tests closure
  • Noise — Base + paraphrase decoys routing to same object (worst case)
  • Collision — near-miss triplets sharing subject, different object

6 prompt formulations swept · retrieval traces logged at every FULL / DEL-ON call.

Proposition 6.1 (structural): under complete alias-closure deletion, R(f) = A(f) identically — so the empirical R̂ ≈ Â match is a deletion-completeness diagnostic, not a substantive equality.

3 distinct failure modes in traces: parametric leakage (rare, high-frequency entities) · Noise-type near-neighbor paraphrase reconstruction · Collision-type confusable-neighbor drift.

✓Strengths

  • Clean causal decomposition — three failure modes instead of one aggregate accuracy.
  • Large controlled sweep (36,684 completions) on a single fixed checkpoint.
  • Adversarial DB topologies isolate which database property causes which residual.
  • Proposition 6.1 elevates an empirical observation into a deletion-completeness check.
  • Headline finding is counterintuitive and policy-relevant: fix the database, not the model.
  • Prompt style has no independent effect — practitioners should not expect prompt engineering to “fix” deletion failures.
  • Code + custom DBs released at github.com/raeesiarya/LMLMAudit.

!Limitations

  • Single 382M checkpoint · single retriever · single threshold · greedy decoding.
  • “Parametric leakage ≈ 0” tests only direct recall; cross-fact compositional leakage not in scope.
  • Custom DBs are synthetic and small (100–240 facts/cell); 3+ orders of magnitude smaller than the 54.6M-triplet released DB — scale × topology interactions unmeasured.
  • Alias-closure deletion removes only (s, r, o)-matching triplets; the Noise variant deliberately exposes what survives this closure.
  • Automatic alias-aware exact-match only; semantic / human-judged correctness out of scope.
  • English-only, entity-level atomic facts — multi-hop, compositional, multilingual deletion unaddressed.

→Questions for further exploration

  1. Does the L / R / artifact decomposition transfer to non-LMLM unlearning (gradient ascent, SISA, fine-tune-and-prune) — or is it specific to retrieval-externalized knowledge?
  2. What is the cross-fact leakage profile — can the model reconstruct a deleted fact by composing non-deleted facts in the same neighborhood under DEL-OFF?
  3. How sensitive is the 0.7%–13.6% residual range to the retriever (sparse / dense / hybrid), similarity threshold, and top-k? Separates embedding-model effects from LMLM-architecture effects.
  4. Can paraphrase-closure at write time (canonicalization: aliases & paraphrastic forms as pointers into one canonical record) collapse the Noise residual to near-zero while preserving legitimate variant retrieval?
  5. Is there a deletion-completeness score computable from the database alone — e.g. a near-neighbor density / paraphrase-coverage metric — that predicts the artifact rate without running the full audit?
  6. Does the framework extend to multi-hop / compositional facts and to semantic or human-judged (rather than exact-match) correctness?
  7. Does scaling to the full 54.6M-triplet released DB, or to a larger LMLM checkpoint, preserve L̂ ≈ 0?
  8. Is the residual parametric leakage (0.11%, concentrated on high-frequency entities) exploitable via membership-inference attacks — and what does that imply for deletion-compliance auditing?
