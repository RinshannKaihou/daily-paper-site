> **一句话总结：** CAT 提出 Confidence-Adaptive Thinking 框架，将模型的内在 self-certainty 信号同时注入偏好数据构造 (CAPL) 与偏好优化目标 (CWPO)，使大推理模型能按题目难度自动压缩 confident 的短推理链而保留 uncertain 链上的充分探索；在 DeepSeek-R1-Distill-1.5B/7B 与 Qwen3-8B 上的 MATH-500/AIME24/GPQA 三个基准上均取得超越 DAST/ConCISE/OverThink 基线的精度-长度权衡。

> **TL;DR:** CAT addresses LRM overthinking by replacing coarse-grained, answer-only difficulty estimation with a **path-level self-certainty (SC)** signal — the KL divergence between the model's next-token distribution and the uniform distribution, averaged over the trajectory. SC is used twice: (1) in **Confidence-Aware Preference Labeling (CAPL)** to build *Conciseness Pairs* (correct, short, high-SC > correct, long, low-SC) and *Deliberation Pairs* (incorrect, long, low-SC > incorrect, short, high-SC), with dynamic pruning to keep only high-margin pairs; and (2) in **Confidence-Weighted Preference Optimization (CWPO)**, which reweights the SimPO loss via a calibration ratio `ρ = SC/|y|^α` so confident successes get pulled toward brevity while over-confident failures get penalized. Across R1-1.5B/7B and Qwen3-8B on MATH-500, AIME24, and GPQA, CAT exceeds every baseline in accuracy (often surpassing the backbone itself) at moderate compression rates (e.g., on AIME24 R1-7B: 53.3 → 58.9 acc, −23% tokens), and the design transfers to DPO as `CWPODPO`.

## 核心贡献
- **框架级创新**：将高效推理的范式从"外部监督（外部奖励 / 答案级难度）"转向"内在 confidence 感知"，让模型自主感知题目难度并调节思考深度。
- **置信度感知偏好标注 (CAPL)**：基于 self-certainty、长度、正确性构造两类偏好对 — Conciseness Pairs (CPs) 与 Deliberation Pairs (DPs)，并通过动态剪枝（top-3 score levels、全局排序截断 τ 比例、每题 ≤1 个 CP/1 个 DP）保留高价值监督。
- **置信度加权偏好优化 (CWPO)**：在 SimPO 目标中引入 calibration ratio `ρ(x, y) = SC(x, y) / |y|^α` 作为动态权重，CP 中鼓励短且 confident 的正确路径，DP 中惩罚短而 confident 的错误路径。
- **实验广泛性**：在 R1-1.5B、R1-7B、Qwen3-8B 三个 backbone × MATH-500 / AIME24 / GPQA 三个 benchmark × SimPO/DPO 两种偏好优化方法下全面验证，并在附录中证明 SC 与正确性、难度正相关且对长度稳健。

## Key Contributions
- Reframes efficient-reasoning for LRMs from *external supervision* (accuracy-based difficulty, external reward models) to *intrinsic confidence awareness* via self-certainty.
- Proposes **Confidence-Aware Preference Labeling (CAPL)**: builds Conciseness Pairs (correct & short & high-SC preferred over correct & long & low-SC) and Deliberation Pairs (incorrect & long & low-SC preferred over incorrect & short & high-SC), then applies dynamic pruning (top-3 score levels → global sort by `s` → truncate bottom τ fraction → per-query cap ≤1 CP/DP).
- Proposes **Confidence-Weighted Preference Optimization (CWPO)**: augments SimPO with a calibration ratio `ρ = SC/|y|^α` that adaptively upweights short-confident successes and penalizes short-confident failures.
- Demonstrates empirical wins on 3 LRMs × 3 reasoning benchmarks, beating OverThink / DAST / ConCISESimPO in accuracy while retaining moderate compression, and shows the SC-driven recipe transfers cleanly to DPO (`CWPODPO`).

## Methodology

**Self-certainty as intrinsic confidence.** Following Kang et al. (2025), CAT defines path-level self-certainty as the average negative log-uniform probability of the model's next-token distribution, which equals the mean `KL(U ‖ p_θ(·|x, y_{<i}))` over tokens. Higher SC ⇒ more peaked predictive distribution ⇒ model is internally more certain.

**Stage I — Confidence-Aware Preference Labeling (CAPL).** Sample K trajectories per training question, compute (correctness, length, SC). Form two pair types:
- **Conciseness Pairs (CPs):** two correct responses; winning one is shorter *and* higher-SC.
  - `s_CP(x, y_w, y_l) = Δr · ΔSC`, where `r(y) = +1/|y|` if correct, `-1/|y|` if incorrect.
- **Deliberation Pairs (DPs):** two incorrect responses; winning one is longer *and* lower-SC.
  - `s_DP(x, y_w, y_l) = Δr · (−ΔSC)`.

Dynamic pruning: rank each set by `s`, keep only the top-3 score levels; pool and globally truncate the bottom τ fraction; cap at 1 CP + 1 DP per query. For R1-7B this yielded 1,765 CPs / 130 DPs (τ = 0.15); for Qwen3-8B 1,517 CPs / 40 DPs; for R1-1.5B 1,742 CPs / 221 DPs — i.e., weaker LRMs surface more DPs (encouraging caution), stronger LRMs surface more CPs (encouraging brevity).

**Stage II — Confidence-Weighted Preference Optimization (CWPO).** Wraps the SimPO loss with two scaling factors:

  - `ρ(x, y) = SC(x, y) / |y|^α`, `α ∈ (0,1)` for numerical stability.
  - For CPs: `β_w = β_base · σ(ρ(x, y_w))`, `β_l = β_base · σ(ρ(x, y_l) − 1)` — pull winning toward short-confident, push losing (long-low-SC) less.
  - For DPs: `β_w = β_base · σ(ρ(x, y_w) − 1)`, `β_l = β_base · σ(ρ(x, y_l))` — penalize short-confident failures harder.

This lets the model learn *when to compress* (high-SC successes) and *when to deliberate* (low-SC failures), rather than applying one global length penalty.

**Training.** 2,000 questions sampled from the MATH training set (diverse difficulty + length). 20 candidate responses per question, max length 4,096 tokens. LoRA fine-tuning (r=32, α=64, dropout=0.05), 1 epoch, batch size 16, lr 5e-5 (R1-7B & Qwen3-8B) / 5e-6 (R1-1.5B). 2 × NVIDIA A800. SimPO base β=200, γ=1.

**Inference / evaluation.** Decoding via OpenR1 evaluation scripts: temperature 0.6, top_p 0.95, max generation 32,768 tokens. Results averaged over 3 runs.

## Results & Highlights

**Headline numbers (Table 1, Acc↑ / mean Len↓ / correct-only Len↓ / Compression Ratio / Correct-only Compression Ratio):**

| Benchmark (R1-7B) | Origin | CAT | Best baseline |
|---|---|---|---|
| MATH-500 Acc / C-Len | 91.7 / 3200 | **93.9 / 1953** | DAST 93.5 / 2469 |
| AIME24 Acc / C-Len | 53.3 / 7631 | **58.9 / 5401** | DAST 56.7 / 7158 |
| GPQA Acc / C-Len | 49.2 / 7382 | **54.0 / 6425** | DAST 52.0 / 7021 |

- On every (backbone × benchmark) cell, CAT achieves the **highest accuracy** while **exceeding** the un-tuned backbone, with a moderate compression ratio (10–34% on average length; 13–39% on correct-only length).
- Compared to the strongest adaptive baseline (DAST), CAT delivers higher accuracy on every benchmark and backbone (e.g., AIME24 R1-7B: 56.7 → 58.9; MATH-500 R1-7B: 93.5 → 93.9) at comparable or slightly lower compression.
- Stronger LRMs (Qwen3-8B, R1-7B) end up with **CP-dominant** datasets and achieve **higher absolute accuracy than the base model** (e.g., MATH-500 Qwen3-8B 96.2 → 96.6; AIME24 74.4 → 76.7; GPQA 59.9 → 60.9) — the gain is real, not just compression.
- **Ablations (Table 2):** removing CAPL (w/o CAPL, score by length alone) yields higher compression but larger accuracy drop on most tasks; removing CWPO (revert to vanilla SimPO) keeps accuracy but loses compression. Both components are necessary.
- **SC ablation (Table 4):** removing SC in CAPL *or* CWPO each degrades accuracy, confirming SC contributes non-redundant signal on both data and optimization sides.
- **Generalization to DPO (Table 3):** `CWPODPO` beats vanilla DPO on most metrics across all three benchmarks (e.g., AIME24 53.3 → 58.9 acc; GPQA 50.7 → 53.0), showing the SC reweighting is largely objective-agnostic.
- **Qualitative case study (Figure 3):** on a triangle-inequality word problem, CAT produces the shortest reasoning chain (573 vs 981 base / 656 DAST tokens) with the **highest self-certainty** (17.8 vs 17.1 / 16.6), trimming redundant invalid-alternative checks without eliminating the brief validity check.
- **SC diagnostics (Figure 2, Figure 7):** correct trajectories cluster at higher SC than incorrect ones; SC is robust to response length (median only mildly drifts); mean SC monotonically decreases as MATH difficulty rises — empirical justification for treating SC as a fine-grained difficulty proxy.

## Critical Assessment
- **Novelty:** The strongest novel element is using *self-certainty as a per-pair preference score* and as a *dynamic loss weight* inside SimPO/DPO. The two-stage CAPL+CWPO design is a clean composition; conceptually adjacent to ConCISE (confidence-guided compression, 2025) and DAST (difficulty-adaptive slow-thinking, 2025), but CAT's contribution is making confidence act both as a labeling filter and as a per-sample loss reweight — a more integrated use of the intrinsic signal than prior work. Genuine but incremental.
- **Rigor:** Experiments cover 3 backbones × 3 benchmarks × 2 preference objectives × full ablations (CAPL, CWPO, SC ablation, DPO generalization) plus a qualitative case study. Means over 3 runs are reported; however **no error bars / standard deviations** are shown — accuracy deltas of 0.3–1.0 pp across cells are within plausible run-to-run noise, so some "SOTA" claims deserve a grain of salt.
- **Reproducibility:** LoRA + SimPO hyperparameters stated (β=200, γ=1, lr=5e-5/5e-6, LoRA r=32/α=64/dropout 0.05, 1 epoch, batch 16, 2×A800). Training set: 2,000 questions from MATH train. Decoding: temperature 0.6, top_p 0.95, max 32,768 tokens, OpenR1 evaluation scripts. Code released on GitHub. τ=0.15 and α=0.5 chosen via grid search on R1-7B / MATH-500 and reused for other backbones — grid details and per-backbone tuned values are only partially reported.
- **Significance:** This is a useful engineering recipe for industrial LRM serving: a single framework that lifts accuracy *and* cuts tokens across three mainstream reasoning LRMs and three widely-used benchmarks, and which transfers to a different preference-optimization algorithm. The accuracy-over-baseline improvements on hard benchmarks like AIME24 (+2.2 pp on R1-7B) are large enough to be practically meaningful.
- **Red flags:**
  - No std/error bars on accuracy or length; only 3-run means.
  - Hyperparameter selection (τ, α) was tuned on R1-7B / MATH-500 and seemingly re-used for R1-1.5B and Qwen3-8B — risk of mild backbone-specific overfit.
  - All training questions come from MATH — even with diverse difficulty, the preference pairs are constructed from MATH-derived trajectories, so out-of-distribution transfer (e.g., to coding or open-ended reasoning) is *not* tested.
  - Authors note that SC is *path-level* (single scalar per sequence), which averages over token-level uncertainty and could miss step-level miscalibration.
  - "Consistently outperforms SOTA" framing in the abstract glosses over the fact that CAT's *compression ratio* is **lower** than ConCISE / OverThink (which trade accuracy for more aggressive compression) — so CAT is best characterized as an accuracy-leaning design, not a compression-leading one.

## Strengths
- Clean two-stage recipe that composes label construction and loss reweighting around the same SC signal.
- Strong empirical evidence: 3 LRMs × 3 benchmarks, full ablations, plus transfer to DPO via `CWPODPO`.
- Stronger LRMs (Qwen3-8B) actually *exceed* their base accuracy after CAT — uncommon in the compression literature, which usually reports accuracy drop.
- Calibration-ratio design keeps SC and |y| on comparable scales (α ∈ (0,1)), and the per-pair weight logic is interpretable.
- Released code + per-pair preference dataset construction script.

## Limitations
- **Path-level SC only** — token- or step-level confidence is collapsed into one scalar, which the authors explicitly flag as future work.
- **Offline preference optimization** on a static MATH-derived dataset; no online RL variant where the policy's confidence estimates co-evolve with training.
- **Reliance on answer correctness for pair labeling** — the framework still needs verifiable answers, restricting extension to open-ended generation (the authors call this out).
- **STEM-only evaluation** (MATH-500, AIME24, GPQA Diamond) — generalization to code, multi-modal reasoning, or natural-language inference is unproven.
- **Mild compression overhead vs. aggressive baselines** — ConCISE / OverThink get larger CR at the cost of accuracy; CAT is not the right choice when token savings dominate.
- Weaker LRM (R1-1.5B) shows only marginal AIME24 gain (28.9 → 31.1) and even slight GPQA compression ratio drop, suggesting the recipe helps weaker backbones less.

## Relevance
与用户关注方向中"推理评估 (Inference Evaluation)"和"LLM 训练稳定性"相关：CAT 直接处理大推理模型的 token 开销与稳定性问题，并在偏好优化层面提出 confidence 加权机制；但本文属于推理效率工程，而非可解释性、hidden state 分析或计算可靠性研究，因此与"Hidden States & Interpretability / Computation Reliability / Silent Data Corruption"等方向的直接交叉较弱。

## Questions for Further Exploration
- Can SC be replaced with a cheaper / better-calibrated uncertainty estimator (e.g., semantic entropy, consistency-based confidence), and how sensitive is CAT to that choice?
- Does CAPL+CWPO transfer to **online** RL (GRPO/PPO) where the policy's confidence estimates evolve during training — would dynamic re-labeling tighten or loosen the reasoning-length prior?
- How does CAT behave on **non-verifiable** reasoning domains (creative writing, open-domain QA) where external correctness is unavailable? Could SC alone drive preference construction?
- Would **step-level** SC (instead of path-level) enable finer-grained compression (skip only low-certainty intermediate steps)?
- Is the 2×A800 / 1-epoch LoRA setup sufficient for larger LRMs (32B+), and does full-parameter fine-tuning change the accuracy-compression frontier?
- How does CAT interact with **tool use / retrieval-augmented reasoning**, where the model should deliberate more before invoking an external tool?
- Can the learned confidence signal double as a **router** signal (route easy questions to a small LRM, hard ones to a large LRM) — extending uncertainty-based LLM routing literature?
- Are there reasoning pathologies where CAT's "compress confident successes" rule backfires (e.g., cases where apparent confidence masks a fragile step)?
