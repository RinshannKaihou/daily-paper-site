Beyond Perplexity: A Behavioral Evaluation Framework for Deployment-Memory Claims in LLM Test-Time Training

Xiangchen Song, Zhenhao Chen, Lingjing Kong, Shaoan Xie, Xinshuai Dong, Guangyi Chen, Kun Zhang — CMU & MBZUAI
Methodology-Critique LLM-Evaluation Test-Time-Training Memory-Personalization Continual-Learning
One-step LoRA cuts support ∆NLL by ~1.0 nat, yet free-form recall is 0/48 across Qwen3-1.7B/4B/8B on direct, paraphrased, and delayed queries — the proxy-vs-deployment gap, quantified.
Key Metric
0/48
Greedy free-form recall on a single-fact LoRA diagnostic — 100% failure rate across 3 Qwen3 scales × 3 query conditions × 48 probes each, while support ∆NLL improves by 0.97–1.22 nat.
Why It Matters
Names evidence migration — the failure mode where TTT papers carry proxy-metric results into stronger "memory / personalization / post-deployment learning" claims. Provides a claim-calibrated evidence ladder (S → B → D) and a matched-baseline protocol to keep wording and evidence aligned.
Critical Edge
Framework is normative — without reviewer/area-chair adoption, papers will keep citing perplexity-only evidence under D-level wording. The diagnostic also rests on a single model family (Qwen3) and a single nonce fact, leaving generalization to Llama / Mistral / closed models untested.

Methodology

Two-component framework. (1) An evidence ladder coding the strongest direct evidence in a paper: S — same-stream / domain / task adaptation (perplexity, future-token loss, long-context accuracy, reward); B — bridge / internalization / parametric-memory probes (answer NLL, needle recall, ablation); D — deployment-time behavior under sparse post-deployment information, paraphrase, delay, locality, conflict, and no-support-context. (2) An evaluation protocol requiring matched explicit-memory baselines (exact-context, BM25, replacement memory), claim-specific probes, mutually exclusive failure categories, and disclosed update budgets.

Audit. Screened 40+ TTT / memory / personalization / continual-learning candidates through April 2026; coded 24 papers by the strongest claim level their evidence supports (not the broadest motivating statement). Boundary-case sensitivity is in Appendix C.

Controlled diagnostic. Qwen3-1.7B / 4B / 8B; one-step LoRA on a single sparse access-code nonce fact; 48 probes per model under direct, paraphrased, and delayed queries after support context is removed. Matched controls: exact-context prompting, BM25 retrieval, replacement memory. Stress-update + locality + conflict-overwrite extensions on the 1.7B model, plus 3-seed replication.

Audit pattern (24 papers, strongest-claim coding).

S
TTT layers, LaCT, In-Place TTT, TTT-NN, TLM, TTRL, SyTTA — same-stream / domain evidence only.
B
PERK, Locas, SEAL, TT-SI, MEMORYLLM, Absorber LLM, TTT-Discover — internalization / parametric-memory probes, no sparse-delayed deployment test.

Strengths

  • Targets a real, growing, underappreciated evaluation problem in the TTT literature.
  • Combines a normative framework (what to claim) with a diagnostic (how to demonstrate the gap) and a protocol (how to evaluate going forward).
  • The nonce-fact experiment is clean, low-cost, and easily reproducible on any Qwen3 checkpoint with a LoRA setup.
  • Explicit memory baselines (MemoryBank, MemGPT, Mem0, LongMem, Dynamic Cheatsheet, MEMORYLLM) close the "RAG would have done better" escape hatch.
  • Cross-scale demonstration (1.7B / 4B / 8B) plus 3-seed replication on 1.7B strengthens the empirical claim.
  • Stress-update + locality + conflict checks turn the diagnostic into a multi-dimensional behavioral map.
  • Acknowledges its own scope: explicit memory may often be the right substrate; the framework is compatible with that view.

Limitations

  • Single adaptation mechanism (one-step LoRA) on a single nonce fact; longer / more aggressive TTT regimes, multi-fact distributions, naturalistic conversational deployments untested.
  • Only one model family (Qwen3) — generalization to Llama, Mistral, Gemma, or closed models is an open empirical question.
  • The audit is qualitative; no quantitative prevalence estimate and no inter-rater reliability reported.
  • "Mutually exclusive failure categories" is asserted, not empirically validated; realistic failure modes blur (partial retrieval + wrong format, correct fact + wrong routing).
  • Framework is normative — without reviewer/area-chair adoption, papers will keep using perplexity-only evidence with D-level wording.
  • Stress-update results (72.9% recall, 9.7% locality) show the interference cost is steep, but the framework does not yet quantify the recall–locality trade-off across update budgets.
  • No test of whether the framework can be operationalized as an automatic reviewer-bot check, which is the most scalable path to adoption.

Questions for Further Exploration

  • Does the proxy–behavior gap close under more aggressive TTT regimes — multi-step LoRA, full finetune, larger-rank adapters, ROME / MEMIT / AlphaEdit, soft-prompt / prefix-tuning?
  • Are the mutually exclusive failure categories truly non-overlapping on naturalistic multi-fact distributions, or do they fragment into finer sub-modes (partial retrieval + wrong tool call, etc.)?
  • How does the framework's verdict change across other model families (Llama-3, Mistral, Gemma-2/3, Phi-4, closed Claude / GPT-style models) and across languages?
  • Can the nonce-fact diagnostic be extended to deployed memory settings — long horizons, hundreds of facts, conflicting updates, tool-using agents — to test whether the gap is an artifact of single-fact injection?
  • Can the S/B/D ladder be operationalized as an automatic reviewer-bot check for ICLR / NeurIPS / ACL, flagging papers whose memory claims are unsupported by behavioral evidence?
  • How should the framework handle procedural vs. episodic memory differently? Dynamic Cheatsheet and TTRL hint that procedural memory may be the more achievable target for parametric TTT — what D-level probe would isolate procedural gains without conflating them with task reward?
  • What is the minimum update budget at which a TTT method transitions from "S/B-level evidence + D-level wording" to "D-level evidence + D-level wording", and is this transition reproducible across architectures?
  • Does the explicit-memory-vs-parametric-TTT trade-off decompose cleanly into privacy, latency, compression, context-pressure, and offline-operation axes, or are there deployment constraints dominated by a single factor?
