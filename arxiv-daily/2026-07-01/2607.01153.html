Adversarial Pragmatics for AI Safety Evaluation: A Benchmark for Instruction Conflict, Embedded Commands, and Policy Ambiguity

AP-Bench AI Safety Evaluation Prompt Injection Instruction Hierarchy LLM-as-Judge Benchmark Validity

Pass/fail labels collapse five distinct judgments — task success, source sensitivity, policy compliance, safety risk, refusal outcome — into a single number; an 18-item, 9-pair seed benchmark decomposes safety failure into eight pragmatic families and turns evaluator disagreement into a diagnostic signal.

Key metric
66.7%
LLM-judge exact agreement with the single expert on task success — while refusal-outcome agreement is 98.1%. Easy labels mask the hard ones.
Why this matters
Directly hits ML Reliability foundations and Model System Cards: judge validity and taxonomy drift give auditable protocols for safety claims — disagreement is treated as a measurement artifact, not averaging noise.
Critical edge
18 items, 1 expert adjudicator, 3 small local Ollama models (8B–12B). No frontier-model coverage, no inter-rater reliability statistics, no statistical inference — below the 50–100 item calibration pass the authors themselves recommend.

Methodology 方法论

Taxonomy (Sec. 3). Instrumental, not theoretical. A case belongs only if expected behaviour depends on source role, pragmatic status, authority relation, policy boundary, or transcript evidence. Real prompt injections are decomposed into minimal pairs, then recombined into application-shaped harnesses.

Minimal-pair construction (Sec. 4). Five constraints: each item names its source; each contrast item tests the same string under a different embedding/authority; expected behaviour is pre-registered before outputs are inspected; shared labels across contrasts; harmless payloads unless reviewed.

Annotation protocol (Sec. 5). Evaluators apply a stated rubric. Independent labels for task success, source sensitivity, policy compliance, safety risk, risk type, refusal outcome, evaluator confidence, and failure attribution. Two-stage adjudication: record labels + confidence without forcing consensus, then an adjudicator classifies the disagreement source.

Pilot (Sec. 6). Local Ollama 0.15.2, 2026-06-30, temperature 0, seed 1, 256 tokens, no thinking trace, model unloading between runs. Single expert adjudicator. Framed as measurement calibration, not a leaderboard.

Strengths vs Limitations 优势与局限

Strengths

  • Pre-registered expected behaviour + metadata-first schema prevents post-hoc rationalization.
  • Refusal matrix directly addresses over- vs under-refusal collapsed into one label.
  • Refuses majority-vote defaulting; evaluator disagreement is treated as diagnostic.
  • LLM-judge validation table is the meta-evaluation that should be standard but rarely is.
  • Code, schema, protocols, and sanitized summaries released; raw unsafe content excluded by design.
  • Section 8 explicitly limits what the pilot can support — rare intellectual honesty.

Limitations

  • Tiny seed (18 items / 9 pairs) — below the 50–100 item pass the authors recommend.
  • Single expert adjudicator — no inter-rater reliability; adjudication stability unmeasured.
  • Small local Ollama models (8B–12B) — no frontier-model coverage.
  • Toy payloads (BLUE/GREEN, ZETA-42) cannot support claims about real injection resistance.
  • No statistical inference — counts and percentages only, no CIs, no power analysis.
  • No agent-environment eval — P007 transcript items are static text, not tool-using agents.

Questions for Further Exploration 待探索问题

  1. Does the projectibility framing (Goodman 1955; Reynolds 2026) yield testable predictions about which paraphrase/wrapper transformations should preserve labels — and which existing safety benchmarks would violate them?
  2. Can the 18-item seed be expanded to 50–100 items and run against frontier models to test whether the strict-pair-pass failures on P005 / P007 / P009 generalize, or are an artifact of capability rather than pragmatics?
  3. How does judge validity degrade under judge-prompt perturbation? The framework calls for ≥2 prompts with source-order perturbation; the pilot runs one.
  4. Can the refusal matrix detect and quantify over-refusal in real system-card claims by comparing declared refusal rates against adjudicated refusal matrices?
  5. Is the adjudication protocol robust to criterion-conflict cases? With one adjudicator, criterion conflict cannot be empirically separated from adjudicator idiosyncrasy.
  6. Could the minimal-pair logic extend to multimodal or tool-use settings, where the embedding channel itself (image, audio, tool result) carries pragmatic information current safety evals ignore?
  7. How does taxonomy drift manifest under light paraphrasing — the framework promises a metric the pilot does not exercise?
  8. Could the 8 metrics be retrofitted to HarmBench, JailbreakBench, XSTest, AgentDojo to audit whether their gold sets are projectibility-stable?
