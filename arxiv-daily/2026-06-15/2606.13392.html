MiniMax Sparse Attention

A group-wise block-sparse attention layer that offloads context selection to a lightweight Index Branch, then runs exact softmax attention only on the KV blocks it picks — cutting 1M-context attention compute by 28.4× on a 109B multimodal model.

Authors: 17 · MiniMax AI Model: 109B natively multimodal Context: 1M tokens Hardware: H800 Open: kernel + MiniMax-M3
Key Finding
Per-GQA-group Top-k block selection keeps exact attention while sparsifying context.
Index Branch chooses blocks independently for each GQA group; Main Branch attends only to those blocks.
Key Metric
28.4× attention compute drop
At 1M context vs. full GQA, plus 14.2× prefill and 7.6× decoding wall-clock speedup on H800.
Why This Matters
1M-token frontier models become deployable.
Ships in a production 109B multimodal model with publicly released kernel and weights.
Critical Edge
Exp-free Top-k + KV-outer sparse kernel co-designed for Tensor Core efficiency.
Avoids costly exponentials during selection and reorders compute for block-granular memory access.

Architecture

  • Index Branch: lightweight scoring network evaluates KV blocks and picks a different Top-k subset per GQA group.
  • Main Branch: exact block-sparse softmax attention restricted to selected blocks — no score approximation.
  • Blockwise granularity aligns with grouped-head layouts and enables coalesced GPU memory access.

GPU Co-Design

  • Exp-free Top-k: skips expensive exponentials during the selection stage.
  • KV-outer sparse layout: reorders attention compute to maximize Tensor Core utilization under sparse block access.
  • Design prioritizes simplicity and portability across GPU generations.

Results & Scale

Validated on a 109B-parameter natively multimodal model trained from scratch with MSA. Quality is reported to be "on par with GQA" — the headline efficiency numbers are:

28.4× Attention compute reduction
14.2× Prefill speedup
7.6× Decoding speedup
Kernel + weights open-sourced Quality claim lacks benchmark detail H800-only hardware results Abstract-only source

Strengths

  • Tackles a real deployment problem: 1M-context inference cost.
  • Algorithm and kernel co-designed, not just theoretical.
  • Public release enables independent verification.
  • Concrete numbers at a credible frontier scale.

Red Flags

  • "On par with GQA" is not tied to named benchmarks or deltas.
  • Portability to AMD/Ascend GPUs is asserted but not shown.
  • Block size, Top-k, and Index Branch architecture are undisclosed.
  • No head-to-head comparison with NSA, MoBA, SeerAttention, MInference.
