arXiv:2606.14476 · 12 Jun 2026 · cs.LG · raptorX.ai

When the tool decides.
And stronger backbones defer more.

A four-arm ReAct audit on ogbn-arxiv and WikiCS shows that Qwen2.5-7B, once given a frozen 2-layer GCN as a callable tool, collapses into a 97.6–99.2% parrot of that tool. The agent skips its own reasoning, ignores an anomaly score designed to flag unreliable labels, and the cost of deference doubles at high homophily as capability scales 1.5B → 7B (paired t=9.1, 5/5 seeds).

01 · The Parrot Effect
Headline

The agent, once given a tool, stops thinking.

97.6–99.2%

Under a four-arm matched-budget protocol (5,000 prompt+generation tokens, 6 tool calls per query), Qwen2.5-7B exposed to a frozen 2-layer GCN produces predictions that agree with the raw GNN on essentially every node across low / mid / high local-homophily bins — and with its own tool-free reasoning on only 17–37% at 7B (7–20% at 3B).

The agent does not even check the anomaly score the tool exposes for exactly this case. In 83% of 7B queries, the loop is: read the label, write the answer, done. The information bottleneck is built into the agent loop — not the router design.

04 · Information Bottleneck
Section 7

Selective invocation is feature-limited, not router-limited.

A purity gate recovers ~½ the high-homophily gap (0.71 → 0.83, t=4.0, 5/5 seeds) but degrades elsewhere for no net global gain (0.481 → 0.475). A learned 4-feature router does no better. 06 · What was actually measured
Protocol

Four arms, matched budget, 5 paired seeds.

A1
agent + frozen GCN tool. 3 calls: gnn_predict → class=c conf=p; gnn_anomaly → reconstruction score (told to agent: "higher = GNN more likely wrong"); gnn_link → link prob.
A2
agent + neighbor-label tool. neighbors() returns up to k=10 training-set neighbor IDs (no text); degree().
A3
frozen 2-layer GCN alone. Hidden 128, 200 epochs, best-val ckpt. Retrained per seed. Pinning target.
A4
agent no tool. Tool-free reasoning baseline. With tool: agreement collapses to 17–37% (7B).

ReAct loop, max 4 steps, 5,000 tokens + 6 calls/query. 750 stratified test nodes on arxiv. Per-seed paired t-tests, Bonferroni correction, feature-shuffled nulls. Local homophily is an analysis axis only — never given to the agent.

07 · Critical Assessment
Reviewer lens

What to trust, and what to scope.

  • Novelty
    Empirical, not methodological. The headline is a falsifiable measurement of an implicit assumption. The capability-deference correlation is the genuinely counter-intuitive result; framing the cost via an oracle ceiling is the right methodological choice.
  • Rigor
    Four-arm matched budget, per-seed paired tests, Bonferroni, feature-shuffled nulls, and honest retraction of a seed-0-only +0.07 gate gain that did not survive 5 seeds.
  • Repro
    Code, seeds, JSON results released; Chinese scaffold shipped in supplementary. Not visible from the paper: exact parser regex, exact token-budget accounting, vLLM-vs-transformers wiring.
  • Red flag
    Extreme 0.97+ magnitude is Qwen-specific (Mistral/OLMo agree at 0.53/0.60). Strong form of claim is scoped to Qwen2.5; weaker form (direction generalizes, magnitude does not) is what the paper actually supports.
  • Red flag
    The ⅙–⅓ ceiling bound depends on the four chosen features. Adding graph-distance, neighbor-text, or agent-side uncertainty could raise the ceiling — paper says "this is an empirical ceiling, not a proof."
  • Setting
    Agent prompt says "consult tools first," harness counts unparseable answers as class 0. Result is about adoption under compliant scaffolding, not generic tool-use — reasonable, but worth flagging.
08 · Scope stamps
Limitations

What this paper is not.

  • Datasets
    2 graphs (ogbn-arxiv, WikiCS). No link prediction, no heterophilous TAGs, no temporal graphs.
  • Families
    1 family at scale (Qwen2.5 0.5B→7B) + limited cross-family probe (n=60, 3 seeds).
  • Tool class
    1 tool: frozen 2-layer GCN for node classification. No retrieval / code-exec / web-search / calibrated-uncertainty variants.
  • Scale cap
    Capability sweep stops at 7B (single-GPU). 14B / 70B / reasoning-tuned (R1-distill) out of scope.
  • Mechanism
    Observational only. SFT priors vs. context dominance vs. calibration collapse are not ablated.
09 · Open questions
Follow-up

What this paper doesn't (yet) answer.

Mechanism ablation
Is the capability-deference trend driven by SFT priors, tool-output dominating the context, or calibration collapse with scale? A simple ablation — same model with tool output omitted — would bound the context-dominance piece.
Beyond 7B
Does the cost-of-deference trend continue, plateau, or reverse at 14B / 70B / reasoning-tuned models (Qwen2.5-14B-Instruct, DeepSeek-R1-distill)?
Other tool classes
Does the parrot effect persist with retrieval (BM25 / dense), code-execution (Python sandbox), or web search? The paper scopes itself to a single tool class.
Richer gates
Can uncertainty-aware / RL-trained / conformal abstention / meta-learned routers close more than ⅓ of the oracle headroom on the present four features?
GNN-side levers
If the GNN exposed a calibrated uncertainty head (not a free-text anomaly score), would the agent use it? Or is the parrot driven by output formatting (a single class label dominates the agent's representation)?
Prompt sensitivity
Is 97.6–99.2% robust to prompt phrasing, tool-naming, and description-order perturbations? The paper does not run a controlled prompt ablation.
Cross-family sweep
A larger cross-family probe (Llama-3.x, Gemma-2, GPT-OSS, reasoning-tuned) under matched scaffolds would clarify whether Qwen-specificity is a Qwen phenomenon or a Chinese-instruction-tuned phenomenon.
Beneficial variants
Are there scaffolds (explicit "you may disagree and explain why," CoT-before-tool-call) that reduce deference without losing the tool's benefit? Or is the parrot effect robust to all such scaffolding?
