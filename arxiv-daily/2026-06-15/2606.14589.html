arXiv 2606.14589 · Production LLM Reliability

When Errors Become Narratives

An 8-week postmortem of openclaw-model-bridge — a personal-assistant LLM runtime on a single macOS host — catalogues 22 incidents and 28 silent-failure manifestations into a mechanism-oriented five-class taxonomy, and coins the LLM-specific class fail-plausible: when a pollution chain makes the model rewrite its own error signals into fluent, confident fabrications pushed to the user.
Author · Wei Wu (sole, independent) Window · 2026-04-09 → 2026-06-02 (8 weeks) Stack · ~40 jobs · 8 LLM providers · 1,100-note KB Defense surface · 4,286 tests · 827 checks · 90 invariants · 23 meta-rules · 14 scanners AI co-author disclosed · Claude (Anthropic)
Key Finding

LLM failure is not gray failure. It is convincing gray failure.

Class D (chained hallucination & fabrication) is the LLM-specific, most dangerous class. A pollution chain makes the model convert an error log into confident, fluent fabrications delivered to the user. Four production incidents documented: a fabricated Hugging Face platform crisis, a fabricated macOS Full-Disk-Access remediation, a fabricated "weekly review", and a fabricated software release.

Key Metric
~70%
of silent failures were caught by a human looking at the user view — not tests, not audits.

15-incident retrospective audit: 0% ex-ante prevention vs 87% ex-post regression blocking. 80% of audit misses are blank categories — dimensions the audit had never conceived.

Why This Matters

Defenses that worked for fail-slow do not transfer to LLM runtimes.

Traditional gray failure blinds the observer. Fail-plausible makes the failure itself do the lying. A 5-component pollution chain — Unicode surrogate → truncated body → 456-byte HTML error page → stdout → cache → reduce-step LLM — produced a confident Hugging Face platform crisis digest pushed to the user. Every component worked as designed.

Critical Edge

One >&2 redirection fixed the entire 7-link D1 chain.

Root cause: a logging function wrote diagnostics to stdout; the caller captured stdout via command substitution as the signal payload. A single stderr redirection plus defense-in-depth (surrogate sanitization, encoding policy, anti-pollution prompt guards) closed it. Fabrication does not require a model — D3 is a hallucination implemented in shell.

Four production Class-D incidents 四起 fail-plausible 生产事件

Each is a complete pollution chain. Each was caught by a human reading the user view, not by an automated guard.

D1
Fabricated platform crisis
The 7-link chain above. A model writes a confident industry digest about a "Hugging Face platform crisis" — the entire crisis lives in a 456-byte HTML error page captured via stdout.
D2
Fabricated remediation
A system alert persisted as an ordinary assistant message. 36 minutes later, an unrelated question triggers a reply instructing the user to grant Full Disk Access to a cron binary in macOS System Preferences. Off-topic, technically fabricated — and yet coincidentally relevant to a real problem elsewhere, weeks later.
D3
Fabricated success
A weekly review job's LLM call failed. A mechanical line-filter fell back to leftover container headings, emitted them as if they were review content, and wrote "llm": true unconditionally. Fabrication does not require a model.
D4
Fabricated release
An evening digest, given a list of the day's high-alignment papers as context enrichment, inferred the user's project "must have shipped" and announced a community release of an internal version number that exists only in the project changelog.

Sunset Law 日落法则

Defense is also a surface. Reduce seams before adding shields.

Rule
Before adding a new defense, retire an equivalent one. One logical entity must have one physical representation. The longest-lived failures live in component seams — deployment topology, dev-vs-target-OS, declared-vs-runtime, observer-vs-observed. Prefer seam reduction over defense accretion.

Operative corollary: an instrument that cannot distinguish "nothing there" from "I was not allowed to look" manufactures false reassurance — the 60-day TCC-sandbox EPERM was diagnosed only after collectors were taught to capture stderr separately and tag [sandbox denied].

How the corpus was built 方法论概要

Selection: (a) reached production, (b) had a silent phase with all automated indicators green, (c) closed with a full postmortem. Inductive, mechanism-first taxonomy; two consecutive non-matches force a class split.

Postmortem protocol — required before any fix:

  1. Causal-chain diagram (timeline × layer × logic × architecture, with code-branch truth values)
  2. Three-layer root cause: trigger / amplifier / concealer
  3. Per-minute timeline reconstruction where logs allow
  4. Condition-combination analysis (why now and not before)
  5. Feed the governance ontology (new invariants, meta-rule candidacy, catalog entry)

Two operators (human + AI collaborator) classified incidents without independent annotators — no κ reported, explicitly acknowledged as a limitation.

Caveats, surfaced honestly because the paper does: Single-system, single-author, single-host-OS, single-operator-pair, 8-week case study. Frequencies are descriptive of one case, not population estimates. The "70% human-caught" share is presented as an existence proof, not a constant. Right-censored latency distribution — failures still silent at study end are absent by construction. No formal taxonomy validation (no inter-rater agreement, no held-out incidents, no decision procedure for class boundaries — author flags 2–3 borderline cases). The defense framework is partially evaluated (53–92% drop in targeted fabrication patterns in one synthesis job only; system-wide incident rate "declined" with explicit confounds caveat). The AI-collaborator co-wrote the postmortems the AI-collaborator's own work is graded against — author offers the public repo and traceable numbers as the primary check on narrative bias.
