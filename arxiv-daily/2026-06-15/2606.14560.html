i · the optimality story

Upper bound meets matching lower bound

Theorems 3.5 (upper)  ·  3.7, 3.8 (lower)  ·  nuclear-norm stationarity

For 𝓑 = (ℝm×n, ‖·‖S∞) — the geometry of (idealized) Muon — NSSD-M reaches ε-‖·‖S1-stationarity in

𝒪( min{m,n} · Δ1 L · ε−2 · (σ/ε)p/(p−1) ) (this paper)

Under the same heavy-tailed + smoothness assumptions, a Euclidean algorithm only buys you

𝒪( min{m,n}(3p−2)/(2(p−1)) · Δ1 L · ε−2 · (σ/ε)p/(p−1) ) (prior work)

The matching lower bound is Ω( min{m,n} · (σ/ε)p/(p−1) / ε² ): the min{m,n} factor in Muon's rate is unavoidable for any first-order method on this stationarity notion. So Muon is first-order optimal.

ii · the analytical move

Why Hilbert was the wrong room

Banach geometry  ·  no gradient clipping

Prior analyses of Muon pay a ρ(3p−2)/(p−1) tax because they project the Banach norm into a Hilbert norm and then control the noise there. The room was wrong.

The fix is to stay in the room:

01 · DUALITY

Ball–Carlen–Lieb

Kq(𝓑) = Sp(𝓑*), 1/p + 1/q = 1. Smoothness in the dual follows from convexity in the primal — usually the easier check.

02 · MARTINGALE

Wenzel 2005 · Thm 2.4

For a martingale-diff sequence in a p-uniformly smooth dual: 𝔼‖ΣYt‖*p ≤ Spp Σ 𝔼‖Yt‖*p. Native noise control, no Hilbert detour.

Plug into the momentum recursion (Lemma 3.3) and a βt ~ T−1/p schedule: the min{m,n} factor is all that survives.

Per-layer: Adam for embeddings, Muon for matrices

Practitioners already do this  ·  theory now explains why

The theory picks out which layer should be optimized in which geometry from the noise shape of that layer's gradient. The paper's empirical σS1,p / σF,p ratio experiment closes the loop.

Weight matrices use Muon

Schatten-native noise σ‖·‖S1,p stays far below the worst case √min{m,n} ≈ 22.5, and is smaller at initialization than at the final checkpoint. Geometrically low noise → high payoff to a Banach-native update.

Embedding tables use Adam

Embeddings show the opposite pattern in ℓ-geometry — higher native noise in ℓ∞-steepest-descent sense. Matches the ℓ∞-steepest descent (Signum) update Adam implicitly performs. A wrong-room update here would lose to Adam.

This is why the layerwise split that the community converged on empirically (Kimi K2, DeepSeek V4) is not folklore — it is the theorem: high-noise layers want Signum (Adam), low-noise matrices want spectral steepest descent (Muon).

The optimality picture

A coupon-collector hard instance

The lower-bound construction (Thm 3.7) places κ = min{m,n} standard-basis rank-1 elements as the hard instance — a coupon-collector problem. Any first-order algorithm has to learn all signs.

Ω( min{m,n} · (σ/ε)p/(p−1) / ε² ) (Thm 3.7, 3.8)

This is the same κ as the upper bound. Together they pin down the entire dimension-dependence for first-order methods on nuclear-norm stationarity. Tight, end of story — within that stationarity notion.

Honest limitations

Things the paper itself flags
70M scale only. A single model on a single dataset with 3 seeds. The same r-sweep shape at >1B parameters is open.
Newton–Schulz gap. Practical Muon uses approximate LMO (Newton–Schulz iterations); the analysis assumes an ideal LMO. The approximation error is unmodeled.
Zero-respecting lower bound. The matched-ε lower bound is for zero-respecting algorithms; the lift to randomized-first-order is standard but not in the main text.
Noise model is a choice. The "switch the noise norm" from σF,p to σ‖·‖*,p is non-trivial; the worst case is still √min{m,n} σF,p.
