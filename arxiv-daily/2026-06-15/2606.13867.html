Muonp
Muon with Fractional Spectral Powers

Yihe Dong & Will Sawin · Princeton · June 2026

Replace Muon's polar factor UV⊤ with fractional spectral powers USpV⊤. One parameter p ∈ (0,1) interpolates between Muon and plain gradient descent — and makes billion-scale finetuning measurably better.

Key Finding

Fractional Spectral Powers Win at Finetuning

Muonp preserves singular-value magnitudes via USpV⊤. On Llama3-1B/3B finetuning it beats both Muon and AdamW across perplexity, math, code, and commonsense tasks.

Why This Matters

Drop-in, Faster, bf16-Stable

A one-line change for Muon users. p=1/3 runs in 101.89 ms/step on H100 — faster than vanilla Muon's 120.73 ms — with no new matrix operations.

Critical Edge

Worse for Pretraining

Pretraining wants uniform full-spectrum updates, so Muon still wins from scratch. A Muon → Muonp final-500-step curriculum recovers some of the gap.

Mechanism

For gradient SVD G = USV⊤, Muon uses the polar factor UV⊤, discarding singular values entirely. Muonp instead applies USpV⊤.

  • Theorem 2.1: Muonp is the steepest-descent direction under the Schatten q-norm with q = 1 + 1/p.
  • Lemma 2.2: No fixed univariate polynomial iteration can compute USpV⊤ — so Muon's Newton–Schulz route is closed.
  • Theorems 2.3–2.4: Odd bivariate polynomial recurrences f(x,y) = x + (ya − xb)g(x,y) approximate USpV⊤ using only matrix multiplications.

Spectral Learning Rate

Muonp reweights each singular direction by an effective learning rate:

ηieff = η · σip−1

Because p − 1 < 0, smaller singular values are attenuated more strongly. The result is an implicit direction-dependent learning rate that concentrates updates where the gradient is already large — useful in low intrinsic-dimensional finetuning, but too restrictive for the broad, uniform updates pretraining needs.

Pretraining vs Finetuning Boundary

The paper's most important conceptual result is a clear regime boundary:

  • Finetuning: Low intrinsic dimension means updates live in a small subspace; preserving singular-value structure helps. Muonp wins.
  • Pretraining: Full-rank, uniform spectrum exploration is preferable; Muon's complete flattening wins.
  • Curriculum evidence: Pretraining with Muon then switching to Muonp for the final 500 steps lowers validation loss across scales, supporting a spectral-learning-rate narrative.

Honest Limitations

  • Pretraining recipe is not solved — the curriculum is only validated as a final-500-step perturbation.
  • Coefficients require per-p grid search on sample gradients.
  • No comparison to SOAP, Shampoo, PSGD, or K-FAC.
  • Single seed per (method, lr); 3B Numina shows lr-sensitive anomalies.
  • Limited to language models at 1B/3B scale.

Critical Assessment

This is a rare optimizer paper with both a clean theoretical hook (Schatten-q optimality + a negative impossibility result + a constructive bivariate workaround) and honest empirical reporting that includes the cases where the method fails. The default p = 1/3 is not arbitrary: it makes the recurrence cubic, keeps bf16 stability, and beats Muon on wall-clock. For practitioners, the value proposition is a one-line Muon replacement during finetuning; for theorists, it reframes Muon as the endpoint of a spectrum-flattening family.

Bottom line: if you are finetuning large language models with Muon today, Muonp is worth trying before you tune anything else. If you are pretraining from scratch, stay with Muon — or treat the Muon → Muonp handoff as an open experiment.
