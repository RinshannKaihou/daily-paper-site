↻ System Loop

PREPARE → PROPOSEr → { IMPLEMENTr,p }

PREPARE reads the task, probes the hidden evaluator, installs dependencies, and may ask humans for clarification.

PROPOSE consumes the ranked solution history and outputs up to P independent, implementation-ready hypotheses.

IMPLEMENT runs P parallel sessions in isolated workspaces, each submitting through a secure grading service and auto-ranked at round end.

🔒 Permissions

  • Docker sandbox per run
  • Hidden evaluator exposed only via grading service
  • Same-round isolation stops premature collapse
  • Default-deny GPU with lock-ownership API

📦 Artifacts

  • Filesystem + Git history as cross-session memory
  • Ranked solution history is automatically maintained
  • Commit messages describe standalone solution + diff
  • Web-search cache + scored submissions persisted

⏱ Budget

  • Wall-clock time + API cost limits
  • time-checker API for active awareness
  • Deadline-warning injection when deliverables lag
  • Full interruption recovery: stage, elapsed, effective budget

👤 Human-in-the-Loop

  • Terminal UI with live session outputs
  • Chat box to enter any active session
  • Web monitor for score evolution
  • Per-round/global best + full transcripts
