arXiv 2602.03120 QES

Quantized Evolution Strategies:
Fine-tune quantized LLMs directly in their discrete weight space.

Authors: Yinggan Xu, Kajetan Schweighofer, Risto Miikkulainen, Xin Qiu Date: 2026-06-15
LLM Post-Training Quantization Evolution Strategies Black-Box Optimization Delta-Sigma Modulation
Key Finding

Discrete-space full-parameter tuning without backprop

QES updates quantized integer weights in-place using stochastic perturbations and accumulated FP16 residuals, so the virtual continuous parameter follows ideal gradient ascent.

Key Metric

Llama-3.1-8B INT4 on GSM8K

Reused Qwen-3B hyperparameters push accuracy from the quantized base of 64.14% to 82.64%.

+18.5 pts
Why This Matters

Inference-level memory, training-level adaptation

Optimizer state collapses to ≈29.7 KB via stateless seed replay. Total VRAM matches the quantized inference-only baseline.

29.7 KB
Critical Edge

Theory is tight; baselines are thin

The temporal-equivalence theorem gives a clean Δ/2 bound, but there is no QLoRA comparison, only single-seed results, and no true >8B tuned scaling study.

Method

1

Discrete perturbations

Replace continuous ES noise with stochastic rounding of σε. Seeds for ε and Bernoulli bits are recorded so perturbations can be replayed from a tiny history.

δᵢ = ⌊σε⌋ + b, b ~ Bernoulli(σε − ⌊σε⌋)
2

Accumulated error feedback

Carry forward FP16 residuals so sub-grid update signals accumulate until they cross the rounding threshold, eliminating gradient stagnation and discretization noise.

uₜ = αĝₜ + γeₜ₋₁  →  ∆Wₜ = Round(uₜ)  →  eₜ = uₜ − ∆Wₜ
3

Stateless seed replay

Store only K ≈ 20–50 (seed, reward) pairs. Re-simulate the residual on the fly; decay γ ensures truncation error is negligible.

Memory Footprint

Optimizer state is reduced from O(d) to O(K), making total training memory equal to inference-only quantized deployment.

Strengths

  • Clean theory: Θₜ = Wₜ + eₜ follows continuous gradient ascent exactly; physical weight deviates by at most Δ/2.
  • Real memory win: Table 8 makes the 29.7 KB claim concrete.
  • Consistent gains over QuZO across all six (model × format) reasoning configs.
  • Open-source release with a memorable repository URL.

Limitations

  • No QLoRA comparison — the most common practical alternative is missing.
  • Single-seed results; ES can have large population variance.
  • Only ≤8B models tested; the 1–2 OOM scaling claim is not validated.
  • Reasoning-heavy benchmark selection may favor the error-feedback mechanism.

Relevance

QES turns the "quantized models are not fine-tunable" bottleneck into a task that fits inside inference-level VRAM. For work on efficient LLM adaptation, edge deployment, or RL-flavored black-box optimization, the method is a pragmatic advance — especially if the open-source implementation proves robust across seeds and larger scales.
