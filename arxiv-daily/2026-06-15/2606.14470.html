arXiv 2606.14470 · Agent Memory / Provenance

GitOfThoughts

Version-controlled reasoning and agent memory you can replay, diff, and merge. A pre-registered factorial study finds that no memory substrate helps on novel problems—gains only materialize when retrieved cases sit above a sharp copyability threshold.

Authors Pavan C Shekar, Abhishek H S, Aswanth Krishnan Affiliation QpiAI, Bengaluru Backbones Qwen3.5-9B / Qwen2.5-7B / Qwen2.5-32B-AWQ Benchmarks GPQA-Diamond · MATH-500
Key Finding

Memory is null on novel problems

Across 5 substrates × 2 benchmarks × 2 models × n=500, no memory backend reliably beats the no-memory baseline. Gains appear only when retrieval is near-duplicate, not abstract.

Key Metric
τ ≈ 0.8

A sharp copyability threshold: cosine ≥ 0.85 unlocks +12.7 pp (7B) to +28.5 pp (32B); below it, memory returns noise.

Why This Matters

It separates leakage from generalization

The agent-memory field often mistakes benchmark overlap for method transfer. This paper bounds the useful regime to recurring or near-duplicate workloads.

Critical Edge

They retracted their own +15 pp trend

An early n=40 git result collapsed to +1.0 pp at n=98 pre-registration. The authors document the refuted hypothesis and audit their own GPQA run with git log.

System in one sentence

Every scored thought becomes a commit, the score becomes a git note, the outcome becomes a tag, exploration paths are branches, and retrieval is git log --grep, git log -S, and git tag -l—making reasoning replayable, diff-able, mergeable, and auditable at ~15 ms/write and ~48 ms/read.

Engineering hook: the git substrate is not evaluated only for accuracy. A distributed-merge demo shows that 5/5 injected contradictions surface as real merge conflicts under a keyed layout, while naive concatenation silently keeps all five.

Methodology

  • Outer loop: depth-1, branch-factor-4 tree-of-thoughts with MCQ-aware A/B/C/D expansion.
  • Inner loop: ReAct (max 3 steps) with calculator, sympy, and LP solver.
  • 5 substrates injected through a shared MemoryBackend: none, markdown, vector, graph, git.
  • Controlled-transfer protocol: study → distill lessons → ingest identically → test held-out.
  • Pre-registered replications and paired bootstrap 95% CIs at every cell.

Copyability sweep

On 200 hard MATH seeds, gains are binary around similarity:

  • Identical (cos 1.00): +12.0 pp (7B), +28.5 pp (32B)
  • Paraphrase (cos 0.95): +13.5 pp (7B), +22.5 pp (32B)
  • Same method, different numbers (cos 0.72): −4.1 pp (7B), −3.6 pp (32B)
  • Same subject (cos 0.22): at noise
  • Unrelated (cos 0.12): at noise
Interpretation: scale steepens the step but does not unlock abstraction. The gain is answer retrieval, not method transfer.

Credibility signals

  • Pre-registered replication of the n=40 git trend.
  • Explicit retraction when the trend collapsed.
  • Documented refutation of "relevance is the lever."
  • Leakage audit via git log: 0 hits on expected_answer, 0/30 verbatim gold letters, fresh memory verified.
  • System headline reported with its compute-budget confound.

Critical assessment

Novelty: high on the systems side—the git↔reasoning mapping is clean and composes with existing tooling; low on the algorithmic side (conventional ToT + ReAct). Rigor: above average, with pre-registration, CIs, retraction, refuted hypotheses, and leakage auditing. Significance: strong for practitioners who need auditable reasoning, and for researchers who need a corrective to memory-as-generalization optimism.

Red flags: only two benchmarks and two adjacent backbones for the headline null; τ depends on MiniLM cosine; git's unique merge/diff affordances are demonstrated but not benchmarked for accuracy; the +14 pp system headline decomposes into MCQ-aware expansion plus compute, not memory.
