CARE: Controlling LLM-Generated Policies through Auditable Review of Evidence in Scientific Experimentation

Key Metric
+8.5
Final-best jump on Minerva/Olympus: 80.0 → 88.5 over the public incumbent, with regret collapsing 9.2 → 0.7. 30 matched seeds, 5-init / 10-reveal protocol.
Why This Matters
Separates proposal generation from action authority — a reusable template for putting AI agents into irreversible, regulated, or safety-constrained decisions.
Critical Edge
Validated only on offline replay at modest pool sizes (5,670 candidates, 10 reveals); live-lab constraints and 10⁴–10⁶-pool scaling are explicitly out of scope.
Methodology

The control loop

  • Default action comes from a non-LLM public incumbent — ensemble of BO experts (GP-EI, UCB, TPE, EDBO) fused with prior, novelty, and categorical terms — before any LLM is consulted.
  • A Policy Planner reads a public trajectory representation and decides whether to reuse / patch / create an executable Python rank_candidates challenger policy.
  • A Public-Evidence Intervention Gate computes κ_t(x) = αᵀg_t(x) − βᵀq_t(x) from pre-reveal evidence only; authorizes iff κ_t ≥ 0, else falls back to incumbent.
  • A pre-reveal audit ledger seals every decision (interface, reference action, challenger artifact, gain/risk, gate verdict) before the outcome is observed.

Two bounded proposal modes

  • Recovery (budget 2/replay) — fired under weak trajectory signals; scans challenger-ranked candidates that also carry public support.
  • Frontier (budget 1/replay) — only when the public schema exposes decision-visible features; schema-conditioned alternative exploration.

What the LLM is never allowed to do

Pick the experiment. The LLM only emits (and revises) an executable ranking policy. The same public-information boundary is applied to non-LLM baselines — no oracle leakage either way.

Intervention funnel — 300 planner calls / dataset

The gate is intentionally sparse: only 52/300 (Minerva) and 36/300 (ChemLex) challenger actions are authorized. Recovery fires on both datasets; frontier fires 18 times on Minerva, 0 on ChemLex.

Strengths

  • Clean separation of concerns: classical optimizer = safety floor, LLM = exploration engine, gate = arbiter. Pre-reveal audit log makes the decision boundary reproducible.
  • Pre-outcome gating is the right design for irreversible experiments — the gate only ever sees causally available evidence.
  • The 6-expert public incumbent ablation cleanly falsifies "just add more BO experts" and isolates the LLM-gate contribution.
  • Backend transfer (GPT-5.4 → Gemini-2.5-Flash) without touching gate, prompt, or protocol — control interface is genuinely modular.
  • Statistical reporting (95% bootstrap CIs, paired W/T/L, 30 matched seeds) is far above the agent-paper median.
  • Full controller + prompt specification in the appendix — a re-implementer can in principle reproduce the run.

Limitations

  • Replay-only evaluation. Live-lab constraints (robot scheduling, batch execution, failure modes, multi-objective) are explicitly out of scope.
  • Single-objective, finite-pool, batch=1. Multi-objective, cost-aware, batched HTE listed as future work.
  • Minerva result is near-oracle (88.5 vs oracle 89.23) at ≈0.18% pool coverage — headroom for the LLM-gate may shrink on larger pools.
  • Frontier mode never fires on ChemLex — the "two bounded proposal modes" is in practice a single-mode controller on a large class of HTE problems.
  • ChemLex top-10 hit is 13% (vs 10% incumbent). Modest, not the dramatic Minerva contrast.
  • No wall-clock / dollar cost in main text. Margin sensitivity only checked on Minerva.
  • No prompt-level ablation (temperature, prompt-card details not perturbed).
Questions for Further Exploration
  1. How does CARE scale to 10⁵–10⁶ pools with smaller reveal budgets? Minerva is near-oracle at 0.18% pool coverage — headroom for the LLM-gate is unclear.
  2. Can the gate itself become learned — a neural gate that inherits interpretability constraints from the public-evidence rule and closes the remaining ChemLex AUC gap?
  3. Is CARE cost-effective vs longer-budget BO, not just 5-init/10-reveal BO? Long-budget CARE-Minerva reaches 89.2 (regret 0.0).
  4. Can non-LLM proposal families (acquisition portfolios, GP-UCB variants, expert-hedging) sit behind the same gate?
  5. How does CARE behave under adversarial / noisy LLM challengers? Can a robust intervention margin be derived from public evidence alone?
  6. Can the frontier mode generalize to categorical-heavy HTE via pre-computed decision-visible public-feature views?
  7. Can CARE be deployed on a live self-driving lab under robot scheduling, batched execution, and failure modes? The replay-to-deployment gap is the obvious next validation.
