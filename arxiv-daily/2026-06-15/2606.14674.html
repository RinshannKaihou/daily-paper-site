01
Key Finding

Module interaction beats module quality

Across 4 embodied benchmarks, the right memory × reasoner pairing can 3–4× profit — but the wrong pairing of stronger components can underperform weak ones. Compatibility dominates composition.

02
Key Metric
8.54→30.67$/hr

ReAct on DeliveryBench, GPT-5 mini, hourly profit. Swapping Base memory for MemoryBank — same backbone, same reasoner. 3.59× lift, no other change.

03
Why This Matters

Your add-on isn't lifting scores

The naive "stack another memory / add a reflection pass" reflex is a categorical error. The bottleneck is the seam — memory granularity, representation format, and the reasoner's input contract. Both researchers and practitioners need an interface, not a module library.

04
Critical Edge

RL does not auto-compose

GRPO lifts the bare policy (−3.07 → 5.80) — then crashes the moment you mount structured memory on top. SUPO (trajectory-summary RL) composes: 8.27 with DynamicCheatsheet. The rule: train RL with the deployment scaffold in the loop.

▸ Reading the matrix A green cell means the (reasoner, memory) pair was the strongest for that row. Two of the four strongest configurations sit on MemoryBank; the other two sit on Base and SimpleMem — there is no column-wise "best memory" once you condition on reasoner.
▸ Why CoT collapses on SimpleMem SimpleMem compresses episodes into dense tokens. CoT is sensitive to token-level faithfulness; the compressed representation breaks the chain. ReAct and Plan-and-Solve are more robust because they can ignore noisy context.
▸ Reflection across the same axis Adding Self-Refine lifts the weakest combos most: P&S+Base 6.2 → 18.6; CoT+Base 8.5 → 19.0. Reflection is a general repair layer — it does not depend on memory choice.

▸ The regression

GRPO lifts the bare policy (5.80 vs −3.07). But mounting MemoryBank or OpenClaw on a GRPO-trained policy crashes it: 4.03 and 4.79, both below GRPO-on-Base. The policy was trained on one context distribution; the deployment scaffold provides another.

▸ Why SUPO composes

SUPO inserts a trajectory-summarization step every 8 actions — the same kind of structured context the modular scaffold provides at inference. The policy and the scaffold end up reading from the same representation. Best cell: SUPO + DynamicCheatsheet = 8.27.

▸ The rule

Train RL with the deployment scaffold in the loop. The "RL on the bare policy" baseline is the wrong baseline for any agent that will be deployed behind a memory or reflection module.

§ 04

Four principles for scaffolded agents

From the empirical study · practitioner-facing
A

Match memory granularity to reasoner

Plan-then-act reasoners (Plan-and-Solve, ReAct) want abstract / summarized memory. CoT wants token-faithful memory. Multi-granularity memory (raw + summary + insights) — i.e. MemoryBank — is the only one that survives across all four reasoners.

B

Reflection is a general repair layer

Self-Refine and Reflexion lift both weak and strong combinations. The gain is larger for weak combos (P&S+Base: 6.2 → 18.6) but never negative. Add it as the default last step, not as an experiment.

C

Train RL with the deployment scaffold

Vanilla GRPO learns a context distribution that disagrees with structured memory at inference. Either match by training (SUPO's trajectory summary), or by freezing the scaffold during RL. The bare-policy baseline is misleading for any agent that ships with a memory.

D

Token budget is not a proxy for profit

On the Qwen3.5-9B Pareto surface, similar token budgets produce very different profits. Well-matched light modules frequently beat heavy but misaligned ones. The frontier is irregular — measure, don't extrapolate.

Strengths of the work

  • Interface > modularization. The framework is built around typed intermediate representations, which is the right design lever for the "interaction effects" problem it studies.
  • 4 benchmarks × 3 backbones × 3 module axes with 3-run means — a substantially more controlled study than is typical in the agent literature.
  • Design principles are evidence-supported and practitioner-actionable, not "we tried X and it worked."
  • Pareto analysis argues — with numbers — against the "more compute = better" reflex.
  • Public playground at agentspec-embodied.github.io raises the reproducibility bar and lowers the cost of follow-on studies.

Limitations & red flags

  • Variance is under-reported. 3-run means without visible error bars; RL cells are effectively n=1. Stronger claims need CIs.
  • MemoryBank is the authors' own design — and ends up the recommended default in every cell. Independent baselines (MemGPT, Mem0, A-Mem) would strengthen the case.
  • "Interaction effects dominate" is well-demonstrated on DeliveryBench and asserted (not re-demonstrated) on MiniGrid / ALFRED / RoboTHOR.
  • Pareto frontier is single-backbone (Qwen3.5-9B). Generalization to GPT-5 scale is not shown.
  • cs.CL-only arXiv categorization under-represents the embodied / RL substance of the paper.
