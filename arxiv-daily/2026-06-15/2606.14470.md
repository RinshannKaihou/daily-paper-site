> **一句话总结：** 提出 GitOfThoughts，把智能体的推理树保存为 git 仓库（commit=思维节点、note=分数、tag=结果、branch=路径、git log=检索），同时通过 5 底座 × 2 基准 × 2 模型规模 × 预注册复现的系统实验证明：对**新颖问题**没有任何记忆底座（含 git）能稳定提升准确率；记忆只在检索案例与当前问题近乎重复（cosine ≳ 0.8）的"可复制性阈值"以上通过**答案检索**带来收益，且 4.5× 模型放大只能让收益更陡峭而无法解锁方法迁移。

> **TL;DR:** GitOfThoughts stores an LLM agent's reasoning tree as a git repository — every scored thought becomes a commit, scores are commit notes, outcomes are tags, branches are exploration paths, and "git log --grep / -S / tag" is the retrieval API — making reasoning replayable, diff-able, mergeable, and auditable at near-zero engineering cost (∼15 ms/write, ∼48 ms/read). The paper then runs a pre-registered factorial evaluation (none / markdown / vector / graph / git) × (GPQA-Diamond, MATH-500) × (Qwen3.5-9B, Qwen2.5-7B-Instruct, Qwen2.5-32B-AWQ) up to n=500, and reports that **no memory substrate reliably improves accuracy on novel problems**; the only arm that significantly beats the no-memory baseline is self-consistency (+3.4 pp at n=500, 7B on MATH-500). A similarity sweep locates a sharp "copyability threshold" τ ≈ 0.8 — above which retrieval gives +12 to +13.5 pp (7B) or +22.5 to +28.5 pp (32B), and below which memory is at noise. Critically, the gain is **answer retrieval, not method transfer**: a controlled arm that injects "same method, different numbers" (cosine 0.72, 187/196 answers non-copyable) yields ∆ = −4.1 [−9.7, +1.5] at 7B and ∆ = −3.6 [−8.7, +1.5] at 32B — scale steepens the copyability step without enabling abstraction. The authors explicitly retract a +15 pp git trend from n=40 that collapsed to +1.0 pp at n=98, document a refuted hypothesis (that retrieval relevance is the lever), and use git log itself to audit their own GPQA run for train–test leakage.

## 核心贡献

- **GitOfThoughts 系统（§3）：** 推理树即 git 仓库——commit = 带评分的思维节点，git notes = 分数，git tag = 验证结果（success_* / failed_*），branch = 探索路径，master vs. memory = 当前会话 vs. 跨问题记忆；检索使用原生 `git log --grep`、`git log -S`（pickaxe）、`git tag -l 'success_*'`，按 ρ = 0.7·score + 0.2·tag + 0.1·recency 排序。
- **跨底座的可证伪负向结果（§4，H-memory 被拒绝）：** 5 底座 × 2 基准 × 2 模型 × 最多 n=500 的预注册实验显示，对**新颖问题**没有任何记忆底座稳定提升准确率；早期 n=40 下 git 出现的 +15 pp 趋势在 n=98 预注册复现中崩盘为 +1.0 [−10.2, +11.2] pp。
- **可复制性阈值 τ ≈ 0.8（§5）：** 在 MATH-500 200 道难题上扫描 test↔memory 余弦相似度，仅当 cosine ≥ 0.85 时记忆才稳定突破零（+12.7 [+8.4, +17.3] pp）；cosine ≤ 0.22 的同主题或不相关示例与无记忆基线无差异。
- **答案是检索而非方法迁移（§5，预注册）：** 设计"同方法、不同数字"的对照臂（cosine 0.72，187/196 答案不可复制）：7B 上 ∆ = −4.1 [−9.7, +1.5]，在 32B 上预注册复现为 ∆ = −3.6 [−8.7, +1.5]；4.5× 模型放大把近重复收益翻倍（identical +28.5 [+22.0, +35.0] pp、paraphrase +22.5 [+16.5, +29.0] pp），但**仍无法**让模型从 worked example 中抽象出可迁移方法。
- **唯一通用杠杆是测试时采样（§6.1）：** 在所有底座下，唯一在 95% CI 之外显著高于无记忆基线的只有 self-consistency ×5（+3.4 [+0.6, +6.2] pp @ n=500）；其他底座与 static few-shot 控制都落在噪声内。
- **方法论样板：** 主动撤回一个早期 +15 pp 趋势、明确证伪"相关性是杠杆"的自身假设、用 git log 对自己的 GPQA 运行做"金答案泄漏审计"（expected_answer 命中 0、verbatim gold letter 命中 0/30、新鲜记忆 per question 已验证）。
- **工程可落地的合并操作（§6.3）：** 在功能层面演示了分布式合并——5 条被刻意注入的互相矛盾 lessons 中，git merge 全部以真实冲突报错（前提是 keyed layout：one file per topic）；简单拼接对照组则静默保留全部矛盾。该实验是 git 作为底座唯一**结构性独特**的操作。

## Key Contributions

- **GitOfThoughts system (§3):** The reasoning DAG is a git repository. Each scored reasoning node is a commit; node identity is a SHA-1 hash; the combined score is a git note; validation outcomes are git tags (`success_*`, `failed_*`); exploration paths are branches; session vs. cross-session memory is master vs. memory; retrieval is stock `git log --grep`, `git log -S` (pickaxe), and `git tag -l`; cross-agent memory transfer is `git fetch` + `git merge`; signed traces are `git commit -S`. Cost is small in absolute terms: ∼15 ms per write and ∼48 ms per read on GPQA (40 ingested lessons).
- **Pluggable MemoryBackend interface (§4.1):** All five substrates (none / markdown / vector / graph / git) are injected through one object with safe no-op defaults, so the agent code is held fixed while only the substrate varies — a clean causal isolation of retrieval vs. write-path noise.
- **Pre-registered H-memory rejected (§4):** Across 5 substrates × 2 benchmarks × 2 backbones × up to n=500, no memory substrate reliably improves accuracy on novel problems. The MATH-500 n=500 7B numbers: none 66.6% baseline; markdown +0.2 [−2.6, +3.2]; vector +1.6 [−1.4, +4.6]; git −0.8 [−4.2, +2.4]; static few-shot control +0.4 [−2.4, +3.4]; only self-consistency ×5 clears zero at +3.4 [+0.6, +6.2] pp.
- **Retracted result made visible (§4.3):** A +15 pp git trend at n=40 (GPQA, 9B) did not replicate at the pre-registered n=98 — git collapses to +1.0 [−10.2, +11.2] pp and the backend ranking reshuffles (markdown worst→best, vector best→worst). The instability of rankings across sample sizes is itself the lesson.
- **Copyability threshold τ ≈ 0.8 (§5):** A similarity sweep on 200 hard MATH seeds (cosine via MiniLM) shows the gain clears zero only at the top bin (cosine ≥ 0.85: +12.7 [+8.4, +17.3] pp). Same-subject (0.22) and unrelated (0.12) are at noise. This explains every prior null in the paper and bounds the regime where memory pays off: recurring or near-duplicate workloads, not novel problems.
- **Answer retrieval, not method transfer (§5):** A same-method / different-numbers control (cosine 0.72, 187/196 memory answers differ from the test's, so the answer is *not* copyable) yields ∆ = −4.1 [−9.7, +1.5] at 7B and ∆ = −3.6 [−8.7, +1.5] at the pre-registered 32B run — scale steepens the copyability step without unlocking abstraction.
- **Self-consistency is the only general lever (§6.1):** At n=500 on MATH-500 with 7B, the +3.4 pp CI is the only one that excludes zero across nine arms (substrates, static few-shot, content/relevance ablations, online accumulation). Verifier best-of-N is "safe but not worth the cost" at 6.4× the tokens (+2.5 pp, one problem, within noise).
- **Epistemic-honesty template (§4, §6.3, §8):** A measurement bug is documented, an early +15 pp trend is retracted under pre-registration, the "relevance is the lever" hypothesis is refuted, and the GPQA headline is reported *with its confound* (compute budget differs across arms, 46/53 misses are timeouts).

## Methodology

- **Search algorithm (deliberately conventional):** Outer loop is depth-1, branching-factor-4 tree-of-thoughts; for MCQ the four root children are A/B/C/D and the model argues for each — this MCQ-aware expansion turned out to be the largest single accuracy lever in the system. Inner loop is ReAct (max 3 steps) with calculator, sympy, and pulp LP solver; web search is disabled in benchmark mode. Node score s = 0.6·s_local + 0.4·s_cross; best child is validated, tagged `success_*` / `failed_*`, and at most one re-expansion is allowed with typed-failure context.
- **Substrate encoding (§3.3):** Each problem is solved in its own ephemeral repo. A commit-on-score hook writes four files per node (`thought.md`, `scores.json`, `trace.jsonl`, `metadata.json`), commits with a structured message, attaches a git note with the score, and tags the outcome. Master accumulates the session tree; a long-lived memory branch holds cross-problem insights. Retrieval ranking: ρ = α·s_comb + β·tag + γ·recency with (α, β, γ) = (0.7, 0.2, 0.1).
- **Pluggable MemoryBackend (§4.1):** none / markdown (dedup + confidence weighting over `memory.md`) / git (one shared repo, retrieval = `git grep`) / vector (all-MiniLM-L6-v2 in Chroma, cosine NN) / graph (networkx lesson/concept graph with spread-activation, associative rather than NN).
- **Controlled-transfer protocol:** (i) **study** — solve K problems, distill one generalizable, answer-free lesson per problem into a shared `lessons.jsonl`; (ii) **ingest** — every backend replays the same lessons; (iii) **test** — solve M held-out problems with each backend injected read-only. Splits are deterministic, domain-stratified, leakage-audited (zero overlap).
- **Pre-registration:** The n=40 git trend was pre-registered for a confirmation at n≈100 *before* it was run; the 32B method-transfer decision rule (∆ method − none clears zero; recurrence control must stay positive; baseline below ceiling) was committed before the model served its first token. The commit history is the audit trail.
- **Fairness audit via the substrate itself (Table 2):** Same gold letters across 3 arms (100/100), `expected_answer` in any commit (0 hits), verbatim gold letter in any trace (0/30), fresh memory per question (verified), random-seed determinism re-run (bit-identical).

## Results & Highlights

- **5-substrate × n=40 exploratory, GPQA-Diamond (9B, paired bootstrap 95% CI):** none 52.5%; markdown 52.5% (+0.0 [−12.5, +12.5]); git 50.0% (−2.5 [−15.0, +10.0]); vector 62.5% (+10.0 [−2.5, +22.5], trend not result); graph 50.0% (−2.5 [−12.5, +7.5]). On MATH-500 all four backends land 2.5–5 pp **below** no-memory (57.5%).
- **n=98 confirmation, GPQA:** none 57.5%; markdown 62.2% (+5.1 [−5.1, +15.3]); vector 48.0% (−9.2 [−20.4, +2.0]); git 58.2% (+1.0 [−10.2, +11.2]) — the n=40 +15 pp git trend died and the ranking reshuffled.
- **Cross-model robustness (GPQA n=98, 7B):** none 40.8%; markdown 36.7% (−4.1); git 38.8% (−2.0); vector 32.6% (−8.2). All CIs cross zero; vector is worst on both models.
- **MATH-500 n=500 7B (Table 6):** none 66.6% baseline; sc5 70.0% (+3.4 [+0.6, +6.2] **∗**); static few-shot 67.0% (+0.4 [−2.4, +3.4]); markdown 66.8% (+0.2 [−2.6, +3.2]); vector 68.2% (+1.6 [−1.4, +4.6]); git 65.8% (−0.8 [−4.2, +2.4]); vector subject-filtered 66.2% (−0.4); vector answer-only 67.6% (+1.0); vector answer-free lesson 66.2% (−0.4). Only self-consistency clears zero.
- **Recurrence sweep (MATH n=200 hard, 7B baseline 53.0%, 32B baseline 57.5%/60.7%):** identical (cos 1.00) 7B +12.0 [+6.0, +18.0] ∗ / 32B +28.5 [+22.0, +35.0] ∗; paraphrase (0.95) 7B +13.5 [+7.0, +20.0] ∗ / 32B +22.5 [+16.5, +29.0] ∗; method band (0.72, non-copyable) 7B −4.1 [−9.7, +1.5] / 32B −3.6 [−8.7, +1.5]; same-subject (0.22) 7B −2.5 [−7.5, +2.5] / 32B +0.5 [−5.0, +6.0]; unrelated (0.12) 7B −2.5 [−7.0, +2.0] / 32B +1.0 [−4.0, +6.0].
- **Online learning curve (git memory accumulating over 500 problems):** accuracy by quartile 70.4 → 60.0 → 67.2 → 67.2% — slightly *down*, even as subject-relevant retrievals grow.
- **System headline with confound (Table 8, GPQA-Diamond, 9B, 100 questions, fresh memory on memory arms):** Vanilla (single call, 60 s budget) 33.0% / 0.7 s per Q; ReTreVal 5× SC (180 s) 34.0% / 127.9 s per Q; GitOfThoughts (600 s) 47.0% / 469.8 s per Q. The +14 pp gain decomposes (by the authors' own ablations) into MCQ-aware expansion plus a larger compute budget, *not* git and *not* memory. Of 53 misses, 46 are timeouts (concentrated in Chemistry's long synthesis chains); on the 54 questions that completed, accuracy is 87.0% (47/54) — a descriptive-only footnote, since conditioning on completion is upward-biased.
- **Differential confound note:** The +22.5 to +28.5 pp at 32B in the recurrence sweep is what the headline looks like when the model can almost directly retrieve the answer; it is **not** evidence of method transfer. The method-band arm at the same scale is null.

## Critical Assessment

- **Novelty:** High on the systems side. The mapping "commit=thought, note=score, tag=outcome, branch=path, merge=cross-agent memory, git log=retrieval" is a clean one-to-one with reasoning affordances and composes with decades of existing tooling at near-zero engineering cost. Low on the algorithmic side — outer ToT + inner ReAct is "minimum-sufficient plumbing" by the authors' own admission. The empirical novelty is the negative result and the pre-registered framework, not a new technique.
- **Rigor:** Above average and visibly so. Pre-registration, paired bootstrap CIs at every cell, explicit retraction of an n=40 trend under its own n≈100 replication, a documented refuted hypothesis, audit of the system's own GPQA run for train–test leakage (`expected_answer` hits: 0; verbatim gold letter in any trace: 0/30; fresh memory per question: verified), and a confound explicitly stated for the system headline. This is the standard most agent-memory papers should be held to.
- **Reproducibility:** Promising. The substrate is git (deterministic, content-addressed, `git bundle`-able), the protocol is pre-registered, CIs are reported per cell, and the cross-model robustness check is documented. The 32B run uses AWQ int4 quantization (the authors note this affects the absolute level but not paired within-model ∆s; still, an unfactored quantization is a reproducibility snag). The detailed retrieval ranking weights (0.7/0.2/0.1) and the controlled-transfer ingestion path are reported.
- **Significance:** Strong for two audiences. (1) Practitioners who need auditable agent reasoning for compliance, debugging, or incident review — git-as-substrate is genuinely useful regardless of accuracy, and the §6.3 distributed-merge demo (5/5 injected contradictions surface as conflicts, concatenation stays silent) is exactly the operational property that motivates this work. (2) Researchers building agent memory — the "memory does not help novel problems" result, located behind a clean similarity threshold, is a corrective to a field that often confuses benchmark leakage with generalization, and the "answer retrieval vs. method transfer" decomposition is sharp.
- **Red flags to watch:**
  - **Scope of the negative claim.** Two benchmarks (GPQA-Diamond, MATH-500), two adjacent open-weight backbones (9B / 7B; the 32B is only used for the method-transfer arm), and "short, distilled, answer-free lessons and worked-example retrieval" — frontier-class models, rich episodic memory (full reasoning traces rather than one-shot distilled lessons), long-horizon planning, multi-agent settings, and tool-use trajectories are explicitly **out of scope**. The negative claim is precise but narrow.
  - **ScienceWorld excluded, not nulled.** The cross-episode arm on ScienceWorld sits at ∼12% (vs. SwiftSage's 84.7 [8]) and is excluded from the headline null as a "floored agent." This is the right epistemic move, but it does mean cross-episode transfer is genuinely **open**.
  - **Copyability threshold depends on the embedding.** τ ≈ 0.8 is measured via MiniLM-L6-v2 cosine. Different embedding choices or task-specific similarity functions could shift τ substantially; the result is more about the *existence* of a sharp step than the *exact* threshold.
  - **The git substrate's distinctive affordances are not benchmarked as features.** Merge / diff / replay / signed authorship are argued for in prose and demonstrated in §6.3, but not scored against alternatives in a multi-agent accuracy experiment. The accuracy phase for merged memory is pre-registered and **pending** (the authors flag this explicitly in §9).
  - **System headline confound is large.** Wall-clock budgets differ 10× across arms in Table 8 (60 s / 180 s / 600 s), and the +14 pp gain attributes to MCQ-aware expansion plus compute rather than git or memory. The authors are honest about this, but a compute-matched control is future work.

## Strengths

- Bites a real, over-marketed problem (agent memory) with a controlled comparison rather than cherry-picked ablations, and isolates retrieval from write-path noise by feeding identical knowledge to all substrates.
- Pre-registration + explicit retraction is a meaningful contribution to evaluation culture on its own — and the substrate itself (git log) is the tool used to audit the paper's own runs.
- The git framing is genuinely useful for auditability / provenance / merge / replay at accuracy parity — a clean separation of concerns that the paper holds throughout.
- The "answer retrieval vs. method transfer" decomposition (and the controlled arm that operationalizes it) is sharp and avoids the usual RAG-paper trap of conflating the two.
- Cost reporting is honest and quantitative (write/read/storage per backend, dispatch latency for durable execution).
- Functional distributed-merge demo with deliberate contradiction injection (5/5 surface as conflicts under keyed layout; concatenation silently keeps all five) is a real engineering demonstration of the substrate's structurally unique operation.

## Limitations

- Only two benchmarks (GPQA-Diamond, MATH-500) and two adjacent backbones for the headline null; the 32B is used only for the method-transfer arm. Frontier-class models and richer memory architectures (full reasoning traces, learned episodic memory, hierarchical summarization) are explicitly future work.
- The five substrates (none / markdown / vector / graph / git) may not represent the strongest recent memory architectures (MemGPT, A-MEM, ExpeL, CLIN, Mem0, MemoryBank, learned episodic memory) — and the paper is honest about this, framing its scope precisely.
- The copyability threshold τ ≈ 0.8 is measured with MiniLM cosine; the threshold's exact value is likely embedding-dependent, even if the *shape* (a sharp step) is the real claim.
- The git substrate's unique affordances (merge, diff, replay, signed authorship, three-way merge) are asserted and functionally demonstrated in §6.3 but **not** evaluated as features in a multi-agent accuracy experiment — the merged-memory accuracy phase is pre-registered and pending.
- No information on the *cost of maintaining* a git-backed reasoning tree across long agent lifetimes (commit-graph growth, pack-file maintenance, retrieval latency at scale), and the ScienceWorld cross-episode arm is excluded rather than reported as a null.
- AWQ int4 quantization on the 32B method-transfer arm affects absolute accuracy; paired ∆s are robust to this, but readers reproducing the absolute numbers will need to budget for it.

## Relevance

与"智能体记忆 / RAG / 推理可审计性 / 评估严谨性"四个方向高度相关。对关心 agent 工程落地的读者，git 作为推理底座是低成本的实操选项；对研究 agent memory 的读者，本文用预注册 + 显式撤回的实验文化，给出了一条对"记忆提升泛化能力"乐观叙事的冷静校正——并把适用边界精确定位到"近重复检索 ≈ 答案检索"这一窄区，对该方向后续工作具有方法论参考价值。

## Questions for Further Exploration

- Does the copyability threshold and the "no method transfer" finding hold on long-horizon, multi-step agent benchmarks (SWE-bench, ALFWorld, WebArena, τ-bench) where memory is typically motivated and where full reasoning traces (not distilled lessons) are the natural substrate?
- How sensitive is τ ≈ 0.8 to the similarity metric — task-specific embeddings, learned retrieval objectives, or structural / symbolic similarity (e.g., shared operators in math) rather than MiniLM cosine? Could a different similarity definition shift the threshold enough to expand the "memory pays" regime?
- The functional distributed-merge demo in §6.3 is real, but the **accuracy** phase (two agents accumulate model-distilled memory over disjoint domains, the orchestrator merges, compared against each solo memory and naive concatenation on a mixed-domain test set) is the natural next experiment — and is pre-registered but pending.
- Can a learned episodic memory architecture (rather than static retrieval) break the copyability ceiling — and does it reduce τ, raise the near-duplicate plateau, or unlock method transfer? Or is the ceiling fundamental to current LLMs?
- Compute-matched system baselines: every GPQA arm at the same wall-clock budget, including budget-saturating self-consistency, to convert the +14 pp headline into either a validated gain or a clean negative result.
- An unbiased high-budget headline run (raising the 600 s cap on the GitOfThoughts arm so the 46/53 timeouts are removed); the 87.0% on the 54 that completed is currently upward-biased and only descriptive.
- Cross-episode transfer on a non-floored agent (SwiftSage-style or with stronger scaffolds) to actually test whether memory accumulates useful skill across episodes — the current ScienceWorld arm is uninformative because the base agent is at the floor.
- What is the token / latency / commit-graph cost of maintaining a git-backed reasoning tree over a long-lived agent's lifetime, and how does it compare to learned memory modules?
- The "selector is the bottleneck" observation (§6.2) — verifier best-of-N is "safe" but not worth 6.4× tokens, and self-consistency / revision can *hurt* by breaking already-correct answers. Is there a learned, calibrated selector that turns test-time sampling into a reliable lever at acceptable cost?
