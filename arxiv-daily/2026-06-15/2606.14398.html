arXiv 2606.14398 · MoE Theory

A theoretical model for task routing in mixture-of-expert transformers

15 June 2026 · arxiv.org/abs/2606.14398
Yongli Xiang Vinoth Nandakumar Yunzhi Yao Peike Li Tongliang Liu

A discrete language model of syntactic templates + finite key-value dictionaries proves that a single-layer MoE transformer can route each task to a unique task-specific expert, with expert capacity bounded only by the task's own complexity.

Mixture-of-Experts Transformer Theory Task Routing Knowledge Circuits Mechanistic Interpretability
Key Finding
One task, one expert — by construction.

A single-layer MoE can route every query from task k to a unique expert indexed by k, with capacity independent of total tasks or vocabulary size.

Key Metric

L|T(k)| + N

Per-expert neuron bound: template length × templates for task k plus dictionary size. Accuracy stays at 86–94% across all training objectives.

Why This Matters
Theory finally meets the empirical circuit.

Formalizes the intuition that expert capacity scales with task complexity, and explains observed Qwen-MoE specialization on key-value relations.

Critical Edge
Constructive, not learned.

Single layer, top-1 routing, synthetic WikiData5M data. No guarantee gradient descent on real corpora converges to this structure.

The Core Theorems

Theorem 4.1 · Dense baseline

A single-layer dense transformer can solve every task via disjoint sparse knowledge circuits C(k), each using at most L|T(k)| + N FFN neurons.

Theorem 4.2 · MoE main result

A single-layer MoE routes every prefix of task k to the expert for k. Each expert again needs at most L|T(k)| + N neurons, and valid next-token probability mass exceeds 1 − ε.

The router is a linear map on the syntactic block of attention output. Because template-relative vectors are linearly independent, the gate can be made one-hot in the task identity by design.

Data Model

A knowledge task is a pair k = (Δ_k, T(k)):

  • Templates τ with wildcards ⋆₁, ⋆₂
  • Dictionary Δ_k of subject-object pairs
  • Complexity c(k) = Σ|τ| + |Δ_k|

Example: "⋆₁ is in ⋆₂" with a finite country-capital table.

Empirical Result

On WikiData5M-derived synthetic data (50 relations, 200 entity pairs each), three losses were compared:

  • CE only → diffuse routing
  • CE + Load-Balancing → still diffuse
  • CE + Router loss → sharp one-task-one-expert alignment

Accuracy remains comparable (86–94%) across all three.

Critical Assessment

Strengths

  • Fully constructive proof with explicit capacity bounds.
  • Discrete, NLP-faithful data model closes a real gap in MoE theory.
  • Shows load-balancing alone is insufficient for task specialization.
  • Clear connection to empirical knowledge-circuit literature.
  • Honest about limits rather than overclaiming.

Limitations

  • Single layer, top-1, one expert per task — far from deployed MoE LLMs.
  • Expressiveness result, not a learnability/optimization result.
  • Synthetic WikiData5M data with Gemini-generated templates.
  • No quantitative alignment metric (NMI, routing entropy).
  • Dense baseline overlaps conceptually with Nichani et al. (2025).

Relevance

Directly relevant to interpretable MoE design, task-aware routing losses, and knowledge-circuit probes. The discrete framework gives a concrete target for what provable task specialization looks like.

Open Questions

  • Can the bound survive multiple MoE layers?
  • Does real gradient descent converge to a similar routing structure?
  • How do shared experts and top-k routing change the bound?
