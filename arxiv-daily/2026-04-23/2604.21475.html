Suppressing the Erasure Error of Fusion Operation in Photonic Quantum Computing

Xiangyu Ren, Yuexun Huang, Zhemin Zhang, Yuchen Zhu, Tsung-Yi Ho, Antonio Barbalace, Zhiding Liang

arXiv: 2604.21475v1

Date: 2026-04-23

https://arxiv.org/abs/2604.21475

一句话总结： MemTree编译器通过树编码融合将光子量子计算中的擦除错误（fusion erasure）重定向为可纠正的泡利错误，在真实QPU硬件上实现3.64倍保真度提升，并将光子源需求降低至原来的18%。
TL;DR: MemTree is a compiler and protocol stack for photonic MBQC that encodes each fusion operation in a tree-structured redundancy scheme (using quantum spin memory as backup qubits), converting photon-loss erasure errors into correctable Pauli errors — achieving 3.64× fidelity improvement, 0.18× photon source usage, and 0.14× compilation time vs. the OneAdapt baseline on real Quandela Benlenos QPU hardware.

核心贡献

  • 提出"树编码融合"（Tree-Encoded Fusion）协议：为每个融合操作配备树状备份结构（b个分支，每分支3个量子比特），将光子擦除失败重定向为Z测量，并通过备份量子比特恢复融合操作，系统性解决光子丢失引起的逻辑错误
  • 设计MemTree编译器：基于混合整数规划（MIP）将大型图态分解为毛毛虫子图（caterpillar substates），并生成平衡二叉树（BBT）结构以优化量子自旋存储器的利用率
  • 在理论上推导树编码融合的逻辑成功率 S_tree = 1 - (1-(1-p_eras)²+p_fail)^b，量化了分支数b对系统可靠性的影响
  • 在Quandela Benlenos QPU（真实光子量子硬件）上运行6-10比特QAOA算法验证，证明MemTree在实际硬件约束下的有效性

Key Contributions

  • Tree-encoded fusion protocol: Attaches b branches to each fusion root qubit q_root. Each branch consists of 3 qubits: q_leaf^c (the fusion qubit), q_leaf^a and q_leaf^b (backup qubits entangled via spin-photon interface). On erasure, an indirect Z measurement via neighboring qubits recovers the logical state rather than discarding the operation
  • Erasure vs. failure distinction: The paper makes a critical distinction that prior work conflated: fusion failure (photon detected in wrong basis, probability p_fail) vs. fusion erasure (photon lost, probability p_eras). The tree encoding specifically targets the erasure pathway, leaving failure handling to existing approaches
  • MemTree compiler: A two-stage compiler using (1) MIP-based graph state division into caterpillar substates (linear chain graphs amenable to tree encoding), and (2) balanced binary tree (BBT) generation that maps spin qubit memory resources to the caterpillar chains
  • Logical success rate formula: S_tree = 1 - (1 - (1-p_eras)² + p_fail)^b, showing that increasing branch number b exponentially suppresses the erasure contribution while adding only linear resource cost
  • Resource efficiency: The tree encoding uses quantum spin memory (e.g., NV centers, quantum dots) as backup qubits, leveraging the photon-spin interface. The paper shows this is more resource-efficient than generating redundant photon sources

Methodology

Background — MBQC and fusion: Measurement-Based Quantum Computing (MBQC) operates by creating a large entangled resource state (graph state) and performing single-qubit measurements. Fusion gates are Bell-state measurements that fuse smaller graph states into the target resource. Linear optical fusion has intrinsic failure probability ~1/2 due to photon statistics, and erasure probability from photon loss.

Tree-encoded fusion structure:

  • Root qubit q_root is the primary fusion qubit
  • b branches are attached, each consisting of q_leaf^c (fusion candidate), q_leaf^a, q_leaf^b
  • Entanglement: q_root – q_leaf^c – q_leaf^a, q_leaf^b (tree graph state)
  • Success path: X measurements on q_leaf^c from a successful branch propagate the fusion outcome
  • Failure path: Branch fusion failed but detected; skip this branch
  • Erasure path: Photon lost from q_leaf^c; perform Z measurement on q_leaf^a, q_leaf^b via indirect measurement; this determines the q_root state without consuming the fusion qubit, enabling retry on another branch

MemTree compiler (two stages):

  1. Graph state decomposition: MIP formulation minimizes the number of caterpillar substates needed to tile the target MBQC graph state. Caterpillar graphs (a path with leaves attached) are the natural output of photon emission sequences from a spin qubit
  2. BBT generation: Maps the caterpillar substate structure to a balanced binary tree of spin qubit emissions, ensuring uniform memory utilization and minimizing peak memory requirements

Quandela Benlenos QPU: A photonic quantum processor with semiconductor quantum dot photon sources. The paper runs 6-qubit and 10-qubit QAOA (Quantum Approximate Optimization Algorithm) circuits as benchmarks, measuring output fidelity against ideal simulation.

Comparison baseline: OneAdapt — a prior adaptive compilation strategy that responds to fusion failures but does not specifically handle erasure errors.

Results & Highlights

  • Fidelity improvement: 3.64× higher circuit fidelity vs. OneAdapt on Quandela Benlenos QPU (QAOA benchmark)
  • Photon source reduction: MemTree requires 0.18× the photon sources of OneAdapt — an 82% reduction in hardware resource requirements
  • Compilation time: 0.14× the compilation time of OneAdapt — a 7× speedup attributed to the MIP formulation being more tractable than OneAdapt's adaptive search
  • Emission frequency (Table III): MemTree ~7×10² /ns vs. OneAdapt ~2×10³ /ns — lower emission rate requirement reduces hardware stress
  • Utilization rate (Table III): MemTree ~10% spin qubit utilization vs. OneAdapt ~0.03% — nearly 3 orders of magnitude improvement in memory utilization, indicating MemTree makes far better use of available quantum memory
  • Logical success rate scaling: S_tree approaches 1 exponentially as b increases; for p_eras = 0.1, p_fail = 0.5, b=4 branches achieve S_tree > 0.95 vs. S_tree ≈ 0.27 for single fusion attempt

Critical Assessment

Novelty: High. The explicit distinction between fusion erasure and fusion failure, and the tree-encoded redundancy specifically targeting the erasure pathway, is a conceptually new contribution. Prior work on photonic fault tolerance focused primarily on failure (non-erasure) recovery.

Rigor: Strong for the hardware results; moderate for the theoretical analysis. The logical success rate formula is derived analytically and validated experimentally. The MIP formulation is standard operations research, but its application to caterpillar graph decomposition is novel.

Reproducibility: Moderate. Hardware experiments require access to Quandela Benlenos QPU, which is not publicly available. The MIP formulation and tree encoding protocol are described with enough detail for software simulation, but hardware reproduction requires specific quantum dot photon sources.

Significance: High for photonic quantum computing practitioners. Erasure error is a fundamental bottleneck in linear optical quantum computing, and the 3.64× fidelity improvement with 82% photon source reduction is a practical advance for near-term photonic systems.

Red flags:

  • The 3.64× fidelity improvement is reported for a specific (unspecified) QAOA depth and problem size; it's unclear how this scales with circuit depth
  • The b-branch tree structure adds overhead in quantum memory and entanglement generation that may not be feasible on all photonic platforms
  • Comparison is only against OneAdapt; comparison against simpler redundancy strategies (e.g., generate k copies of each fusion operation without tree structure) is missing
  • The 10% utilization rate, while far better than 0.03%, still indicates significant idle quantum memory — the bottleneck may have merely shifted

Strengths

  • Real hardware demonstration on commercial QPU provides ground truth beyond simulation
  • Clean theoretical framework distinguishing erasure vs. failure types
  • Dramatic resource reduction (0.18× photon sources) has immediate practical value for hardware scaling
  • The MIP-based compilation is reproducible and well-formalized
  • Cross-disciplinary team with EDA expertise brings compilation rigor typically absent in physics-focused quantum papers

Limitations

  • Requires quantum spin memory (NV centers, quantum dots) — not all photonic architectures have accessible spin-photon interfaces
  • Tree encoding adds depth (more operations) which may introduce additional gate errors on real hardware
  • MIP becomes NP-hard for large graph states; scalability of the compiler to large fault-tolerant circuits is untested
  • Hardware benchmark (QAOA 6-10 qubits) is small; fault-tolerant quantum computing requires logical qubit counts orders of magnitude larger
  • No comparison to active feedback / error correction approaches that don't use tree encoding

相关性

该工作聚焦于量子计算硬件可靠性，与ML可靠性和静默数据损坏（silent data corruption）方向存在类比关联：融合擦除错误本质上是一种"无声"失败模式（光子丢失而非检测失败），其检测和恢复策略对ML系统中类似的隐式错误传播问题有方法论借鉴价值。

Questions for Further Exploration

  2. What is the minimum spin coherence time required for the backup qubit strategy to be effective — is it feasible on current NV center or quantum dot platforms?
  3. Can the tree-encoded fusion protocol be combined with topological quantum error correction (e.g., surface code) to provide both erasure suppression and broader fault tolerance?
  4. How does the MIP compilation time scale with graph state size? Is there a practical circuit size limit beyond which a heuristic replacement is needed?
  5. Could the tree-encoded fusion concept be adapted for trapped-ion or superconducting qubit platforms where erasure errors take different physical forms?
