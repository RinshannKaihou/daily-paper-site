> **一句话总结：** TRS（Thinking with Reasoning Skills）通过离线将长推理轨迹蒸馏为结构化"技能卡"（含触发条件、方法、陷阱字段），在推理时检索注入相关技能卡，在数学和代码任务上同时减少 token 消耗（最高 -61%）并提升准确率，打破效率-准确率 trade-off。

> **TL;DR:** TRS is a training-free, black-box-compatible framework that distills extensive reasoning trajectories into compact, reusable skill cards (with Trigger/Do/Avoid/Check/Risk fields) stored in a key-value library. At inference time, relevant cards are retrieved and injected into the prompt, steering models away from redundant exploration. On math and coding benchmarks across multiple frontier models, TRS reduces reasoning tokens while matching or exceeding baseline accuracy—especially on hard problems where forced brevity methods catastrophically fail.

## 核心贡献

- 提出 TRS 框架：将"从零推理"范式转变为"基于召回经验推理"，通过离线技能蒸馏和在线检索注入两阶段解耦昂贵的探索（一次性）与廉价的复用（每次请求）。
- 设计结构化技能卡格式（Trigger/Do/Avoid/Check/Risk 五字段），同时覆盖成功策略和失败模式修复，比原始示例、CoT 轨迹或自由摘要均更有效（Table 2 控制实验）。
- 实证打破效率-准确率 trade-off：在难题子集（高基线 thinking token 消耗）上，TALE/CoD 等方法准确率崩塌，TRS 稳定保持或超越基线准确率同时降低成本。
- 分析技能跨模型迁移性：蒸馏自一个模型的技能可提升其他模型性能，同源技能效果最佳，但跨源技能有时带来更激进的 token 削减。

## Key Contributions

- Introduces TRS (Thinking with Reasoning Skills): a training-free retrieval-augmented framework that decouples skill acquisition (offline, expensive) from skill application (online, cheap), remaining compatible with black-box LLM APIs.
- Proposes structured skill cards with five fields—Trigger (applicability cues), Do (minimal actionable recipe/invariant), Avoid (anti-patterns), Check (constraints/invariants), Risk (edge cases/failure modes)—capturing both successful patterns and failure-mode fixes.
- Demonstrates the efficiency-accuracy trade-off is breakable: on hard instances (θ ≥ 12K baseline think tokens on Doubao), TRS cuts generated tokens from ~12K to ~5K while maintaining top accuracy, while TALE/CoD/NoWait show severe degradation at the same difficulty level.
- Provides cross-model transfer analysis: 13 of 25 model-benchmark pairs improve accuracy and 20 of 25 reduce cost using AoPS-derived skills (Table 3).
- Ablation confirms structured skill distillation is essential: raw examples, raw CoT traces, free-form summaries, and small structured-card libraries all underperform full TRS, especially on out-of-domain OSS models.

## Methodology

**Offline skill library construction:**
For each source instance (qᵢ, aᵢ), a reasoning model Mᵣ generates a full trajectory (τᵢ, yᵢ). An experience record (qᵢ, aᵢ, τᵢ, yᵢ, cᵢ) is formed where cᵢ = Eval(yᵢ, aᵢ) indicates correctness. A stronger summarizer model Mₛ (Gemini 3 Flash) distills each record into: (i) a compact skill card sᵢ and (ii) 10–20 retrieval trigger keywords Kᵢ. If cᵢ = 1, sᵢ captures the essential solution pattern (minimal transformation/invariant/algorithmic template). If cᵢ = 0, sᵢ captures a reusable failure-mode fix (anti-pattern → correction). Each card becomes a key-value entry: key = Concat(qᵢ, Kᵢ), value = sᵢ.

**Retrieval backends:** Sparse BM25 (default for math, best lexical matching for formula/theorem patterns), dense BGE-M3 + FAISS nearest-neighbor (default for coding, captures semantic similarity), and hybrid fusion for coding. BM25 k=1 for math; hybrid k=5 for coding.

**Skill injection:** Top-k retrieved cards are prepended to the prompt under a lightweight arbitration instruction: "use only directly applicable skills; ignore irrelevant/contradictory advice." Card length is capped to a small budget.

**Math evaluation:** DeepMath-103K with strict 93K/10K source/held-out split. **Coding evaluation:** Nemotron-Competitive-Programming-v1 (34K), filtered to 26.6K with reliable local judging; 1K holdout. Correctness: exact match for math (pass@1 equivalent), pass@1 for coding.

**Models tested:** Gemini-3-Flash, Doubao Seed (1.8 Pro math / 2.0 Pro coding), GPT-OSS-120B, GPT-4o-mini, plus external cross-transfer tests on GPT-5.2, Grok-4-Fast, Gemini-3-Pro, GPT-4o-mini.

## Results & Highlights

- **Math:** Gemini-3-Flash: accuracy +0.7, cost -17.5%; Doubao Seed: accuracy -0.2, cost -53.8%; GPT-OSS-120B: accuracy ±0.0, cost -16.9%; GPT-4o-mini: accuracy +1.8, cost -20.6%.
- **Coding:** GPT-4o-mini: pass@1 +2.4 (+2.4 pp), cost -6.3%; Doubao Seed 2.0 Pro: pass@1 +0.8, cost -6.0%; GPT-OSS-120B: pass@1 +4.1, cost +4.8% (slight increase due to larger prompts).
- On hardest math problems (θ ≥ 12K): TRS cuts tokens from ~12K to ~5K on Doubao while TALE/CoD show accuracy collapse.
- Controlled representation ablation (Table 2): raw examples 92.0%, raw CoT 91.0%, free summary 86.2%, structured card (2.5K) 91.2%, full TRS (93K) 92.6% on fixed 500-question DeepMath subset.
- Cross-model transfer (AoPS library): AIME 2024 I shows strongest positive transfer (+2.54 acc on Gemini-3-Flash, +3.55 on GPT-OSS-120B, +3.54 on Doubao-1.8); newer/harder sets show smaller or mixed effects.

## Critical Assessment

**Novelty:** Moderate-to-good. Retrieval-augmented prompting and skill libraries in agent systems are well-established (Reflexion, Voyager, Buffer-of-Thoughts). TRS is distinguished by: (1) the structured card format capturing failure modes alongside successes; (2) systematic study of cross-model transfer and difficulty slicing; (3) the explicit decoupling of exploration cost from reuse cost as a deployment design principle. The framing as breaking the efficiency-accuracy trade-off is the core novelty claim and is substantiated.

**Rigor:** Good overall. The controlled representation ablation (Table 2) and coverage ablation are well-designed. The cross-model transfer study uses a separate AoPS library, avoiding data contamination. However, the direct vs. TRS comparison uses different repeat counts (32 vs. 8)—the paper explicitly acknowledges this makes the comparison "descriptive rather than a formal significance test." No confidence intervals are reported on main accuracy numbers.

**Reproducibility:** Strong. Code, dataset, and demo are publicly released. Prompt templates are documented in full (Appendix G). Skill card schema and distillation rules are specified (Appendix A). Local coding judge environment is documented (Appendix D).

**Significance:** High for production LLM deployment. The cost reduction claim is directly relevant to commercial API users billing by token. The finding that hard problems benefit most is practically important—these are exactly the cases where inference cost is highest.

**Red flags:** (1) The main math benefit on Doubao (-53.8% cost) appears very large—but Doubao's source split uses only 3,453 average tokens vs. Gemini's 7,490, suggesting Doubao already reasons more efficiently, making the relative gain harder to interpret. (2) On non-reasoning benchmarks (SST-2, CoLA, MRPC, AG News), TRS consistently increases cost (+36-102%) with no accuracy benefit—it is not domain-agnostic and requires reasoning task identification before deployment. (3) Failure analysis is underdeveloped: the paper acknowledges it cannot characterize which error types are most recoverable via skill distillation.

## Strengths

- Training-free and black-box API compatible—no model weights needed.
- Structured skill cards outperform naive retrieval (raw examples, raw CoT) in controlled ablation.
- Gains concentrate precisely on hard problems where cost matters most.
- Full prompt templates, code, and dataset released for reproducibility.
- Honest failure analysis: identifies cases (non-reasoning tasks, prompt inflation) where TRS does not help.

## Limitations

- Skill library construction requires high-quality reasoning trajectories from a capable model—bootstrapping cost on new domains is non-trivial.
- Retrieval quality is a bottleneck: BM25 can retrieve structurally mismatched hints that look semantically plausible but mislead generation (acknowledged for embedding retrieval in math).
- For GPT-OSS-120B on coding, cost slightly increases due to longer prompts—TRS breaks even only when prompt overhead is smaller than generation savings.
- Task scope: primarily math and competitive programming with automatic verifiers; broader domains and weakly verifiable tasks are untested.
- Failure-mode skill distillation relies on binary correctness signals (exact match, pass@1)—nuanced error types are not characterized.

## Relevance

本文与用户关注的**推理评估**和**LLM 推理效率**密切相关：TRS 提供了一种系统化减少推理 token 同时保持（或提升）准确率的方法，对理解 test-time compute 的可靠性和效率边界具有直接参考价值。

## Questions for Further Exploration

1. Can skill cards be learned incrementally in an online setting—e.g., continuously updating the library as the system encounters new problems in deployment?
2. How does TRS interact with reinforcement learning fine-tuned models (e.g., DeepSeek-R1) where the reasoning style is deliberately exploratory?
3. What is the minimum skill library size needed to achieve most of the benefit—and does this scale sublinearly (implying a power-law skill coverage curve)?
4. Can the structured card format capture multi-step algorithm templates for competitive programming, where the "skill" is a complex data structure design pattern rather than a mathematical invariant?
5. How should the arbitration instruction be adapted when retrieved skills are partially contradictory—and can a lightweight router model improve gating beyond the current heuristic instruction?
