Language as a Latent Variable for Reasoning Optimization

Linjuan Wu, Haoran Wei, Jialong Tang, Shuang Luo, Baosong Yang, Yongliang Shen, Weiming Lu

arXiv: 2604.21593v1

Date: 2026-04-23

https://arxiv.org/abs/2604.21593

一句话总结： polyGRPO将推理时的语言选择建模为隐变量，通过多语言rollout在GRPO框架中隐式探索最优思维语言，使Qwen2.5-7B在MGSM上多语言平均准确率提升12.77个百分点。
TL;DR: polyGRPO treats the language used during chain-of-thought reasoning as an implicit latent variable within GRPO, generating multilingual rollouts (1 unconstrained + 4 language-constrained) during policy optimization to discover which thinking language best supports problem solving — yielding +12.77pp multilingual average and +3.6pp English on MGSM for Qwen2.5-7B.

核心贡献

  • 将推理链的语言选择形式化为隐变量（latent variable），证明在GRPO目标函数中边缘化该隐变量等价于对多语言rollout取期望，从而推导出polyGRPO损失函数
  • 设计多语言rollout策略：每次优化步骤生成n=5个输出（1个无语言约束 + 4个分别指定为英语/中文/法语/德语的语言约束），利用同一问题的多语言推理路径共同估计策略梯度
  • 引入格式奖励（Format Reward）与正确性奖励（Accuracy Reward）的组合奖励函数，确保模型输出结构规范（[Thinking]+####格式）的同时优化推理准确率
  • 实验发现模型在训练过程中自发形成英语主导的思维模式（即使是非英语问题），揭示了语言作为工具性变量在推理优化中的内在涌现现象

Key Contributions

  • Latent variable formulation: Formalizes the chain-of-thought language selection as a discrete latent variable z ∈ {English, Chinese, French, German, ...}. The marginal policy optimization objective becomes an expectation over z, which polyGRPO approximates via multi-language rollouts
  • polyGRPO loss: Extends the standard GRPO objective by sampling rollouts across multiple language constraints simultaneously. The policy gradient is estimated from n=5 rollouts per prompt (1 unconstrained + 4 language-constrained), effectively providing a richer exploration signal than monolingual GRPO
  • Combined reward function: AR(oᵢ) ∈ {0,1} (correct answer) + FR(oᵢ) ∈ {0,1} (correct format: [Thinking] block followed by #### answer). The format reward prevents degenerate solutions where the model answers correctly but loses the structured reasoning trace
  • Language consistency analysis: Tracks which language the model uses spontaneously (without constraints) over training epochs. The model progressively shifts toward English-dominant reasoning even for non-English inputs, suggesting the model discovers English as a more reliable reasoning substrate — a finding with significant implications for multilingual LLM training
  • Multilingual training data: 18,140 multilingual math problems across 8 languages, trained for 5 epochs on 8 A100 GPUs

Methodology

GRPO background: GRPO (Group Relative Policy Optimization) is a reinforcement learning algorithm for LLMs that avoids a separate critic by normalizing rewards within a group of rollouts sampled from the same prompt. The policy gradient is computed as:

L_GRPO = -E[Σᵢ (rᵢ - mean(r)) / std(r) · log π(oᵢ|q)]

where rᵢ is the reward for rollout oᵢ, and the group normalization provides a low-variance baseline.

polyGRPO extension: The latent variable z (language choice) is introduced such that the true objective marginalizes over z. In practice, polyGRPO samples rollouts by providing language-specification prompts: "Please reason in [LANGUAGE]." For each training question q, the system generates:

  • 1 rollout with no language constraint (free-form)
  • 4 rollouts with explicit language constraints (e.g., English, Chinese, French, German)

These 5 rollouts jointly form the GRPO group, providing cross-lingual gradient signal that monolingual GRPO cannot access.

Reward specification:

  • Accuracy Reward (AR): 1 if the extracted answer after #### matches ground truth, 0 otherwise. Extraction uses a regex over the final line.
  • Format Reward (FR): 1 if the output contains a [Thinking]...[/Thinking] block followed by #### [answer], 0 otherwise.
  • Total reward r = AR + FR (range [0, 2])

Training setup: Qwen2.5-7B-Instruct as base model. 18,140 examples from MGSM training split (8 languages × ~2,268 examples each). Learning rate 1e-6, batch size 32, 5 epochs, 8 × A100 80GB GPUs. Training duration approximately 48 hours.

Evaluation benchmarks:

  • MGSM (Multilingual Grade School Math): 250 problems × 11 languages; primary benchmark
  • MATH500: English math benchmark for in-domain generalization
  • X-CSQA: Multilingual commonsense reasoning (out-of-domain for the math-focused training)

Results & Highlights

  • MGSM EN accuracy: Qwen2.5-7B-Instruct baseline 90.4 → polyGRPO 94.0 (+3.6pp)
  • MGSM ALL (11-language average): Baseline 63.16 → polyGRPO 75.93 (+12.77pp) — the largest improvement among compared methods
  • X-CSQA: polyGRPO is the only method among all compared that beats the base LLM on commonsense reasoning (+4.9pp), suggesting the latent language exploration provides a more general reasoning improvement rather than overfitting to math
  • Language consistency (Figure 5): At epoch 0, model uses the requested language ~80% of the time. By epoch 5, the model uses English in ~90% of unconstrained rollouts regardless of input language — a spontaneous convergence to English as the "thinking language"
  • Comparison to monolingual GRPO: Standard GRPO trained only in English achieves 71.2% MGSM ALL; polyGRPO achieves 75.93% — the 4.73pp gap is attributable purely to the multilingual rollout strategy
  • MATH500: polyGRPO 83.6% vs. baseline 82.2% (+1.4pp), showing mild in-domain improvement

Critical Assessment

Novelty: Moderate-to-high. The framing of language selection as a latent variable is conceptually elegant and provides a principled motivation for multilingual rollouts in RL fine-tuning. Prior multilingual RL work typically trains on multilingual data without this explicit exploration framing.

Rigor: Adequate. The latent variable derivation is present but not deeply formalized (no convergence analysis for the marginal objective approximation). The empirical results are consistent across multiple benchmarks, which is reassuring.

Reproducibility: Moderate. Training details are provided (data size, batch size, learning rate, GPU count). However, the language-specification prompt format and exact rollout mixing strategy are not fully detailed, and no code is released.

Significance: High for multilingual LLM development. The +12.77pp multilingual improvement is substantial. The finding that models naturally converge to English-dominant reasoning is an important empirical observation with implications for cross-lingual transfer and non-English deployment.

Red flags:

  • The +12.77pp average hides significant variance across languages — the improvement may be driven by languages close to English rather than truly low-resource languages in MGSM
  • The "language as latent variable" framing is theoretically motivated but the actual implementation is simple data augmentation with language prompts; the latent variable formalism may be post-hoc justification
  • Training on 8 languages and evaluating on 11 (MGSM has 11 languages) means 3 evaluation languages are zero-shot; results on those languages should be separately reported
  • The English convergence finding (Figure 5) raises a concern: if the model always thinks in English, is polyGRPO just a better way to train English reasoning that happens to transfer multilingually?

Strengths

  • Theoretically grounded formulation of language selection as a latent variable
  • Strong multilingual improvements (+12.77pp) on a standard benchmark
  • Out-of-domain X-CSQA result suggests genuine generalization rather than benchmark overfitting
  • Language consistency analysis (Figure 5) is an insightful contribution that could inspire future interpretability work
  • Computationally feasible (8 A100s for 48 hours) — accessible to academic groups

Limitations

  • English convergence of reasoning may limit applicability for users needing non-English chain-of-thought
  • Only one base model evaluated (Qwen2.5-7B); generalization to other LLM families unverified
  • Training data is math-focused; X-CSQA improvement is promising but narrow
  • No ablation on number of languages or rollouts per prompt (is n=5 optimal?)
  • Latent variable RL framework is not compared to explicit language-mixture training baselines that don't use the GRPO structure

相关性

polyGRPO与LLM训练稳定性和推理评估密切相关：多语言rollout策略可视为一种隐式数据增强，其对训练稳定性的影响（reward方差、梯度方差）值得深入分析；此外，英语主导思维的涌现现象对多语言模型的可解释性研究有重要启示。

Questions for Further Exploration

  1. Does the English convergence of reasoning persist if the base model is trained predominantly in a non-English language (e.g., a Chinese-first LLM)?
  2. What happens if the format reward is removed — does the model still converge to English-dominant reasoning, suggesting the reasoning language preference is driven by accuracy rather than format?
  3. Can the latent variable framework be extended to reasoning style (formal/informal, step-by-step/holistic) rather than just language, providing a more general exploration mechanism?
  4. How does polyGRPO interact with DPO or preference-based methods? Could language preferences be incorporated into the reward model?
  5. What is the minimum number of training languages needed to achieve the multilingual generalization benefit?
