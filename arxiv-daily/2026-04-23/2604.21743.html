Bridging the Training-Deployment Gap: Gated Encoding and Multi-Scale Refinement for Efficient Quantization-Aware Image Enhancement

Dat To-Thanh, Nghia Nguyen-Trong, Hoang Vo, Hieu Bui-Minh, Tinh-Anh Nguyen-Nhu (University of Science VNU-HCM, University of Information Technology VNU-HCM, Da Nang University of Economics, Ho Chi Minh University of Technology VNU-HCM)

一句话总结： 本文提出 QATIE，一种面向移动端部署的图像增强网络，结合双分支门控编码器、多尺度细化模块与量化感知训练（QAT），在 DPED 数据集上以 INT8 精度实现 21.05 dB PSNR，在高通 QNN HTP 上推理延迟仅 41.8 ms，在 Mobile AI 2026 挑战赛中获得第二名。
TL;DR: QATIE proposes a mobile image enhancement pipeline that uses gated dual-branch encoder blocks and multi-scale UNet-style refinement, integrated with quantization-aware training (QAT) via fake quantization operators and straight-through estimators. At INT8 precision on the DPED benchmark, the model achieves 21.05 dB PSNR (vs. 20.58 dB for post-training quantization), with 41.8 ms latency on Qualcomm QNN HTP—earning 2nd place in the Mobile AI 2026 RGB Image Enhancement Challenge.

核心贡献

  • 门控编码器块（Gated Encoder Block）：双分支并行卷积结构，采用 tanh 激活的逐元素门控机制生成软掩码，过滤双分支激活并通过多通道跳跃连接保留三元特征 (xa, xg, xb)，增强局部细节保留能力。
  • 多尺度细化（Multi-Scale Refinement）：在 S/2、S/4、S/8 三个分辨率尺度上应用 UNet 式残差卷积块，跨尺度保留结构信息同时提升特征判别力。
  • 量化感知训练（Quantization-Aware Training）：在训练阶段插入伪量化算子（fake quantization operators）并使用直通估计器（Straight-Through Estimator）模拟 INT8 精度，使网络主动学习补偿精度损失与舍入误差。
  • 训练-部署差距弥合：专注解决图像增强模型在 Full-precision 训练后量化部署时质量急剧下降的问题，为 Deep ISP 任务提供端到端量化友好训练框架。

Key Contributions

  • Gated Encoder Block: A dual-branch convolutional architecture where each branch produces activations, and a gating branch with tanh activation generates a soft mask applied element-wise to both branches. This preserves a triplet of features (xa, xg, xb) via multi-channel skip connections, maintaining fine-grained local detail through the encoding process.
  • Multi-Scale Refinement Module: Refinement operators applied at three scales (S/2, S/4, S/8) using UNet-style residual convolutional blocks. This hierarchical approach preserves structural information while enhancing feature discriminability at each resolution level.
  • QAT for Deep ISP: Integrates quantization-aware training specifically tailored for pixel-level image enhancement, where quantization artifacts are perceptually salient. Fake quantization is inserted at training time, and straight-through estimators allow gradients to flow through the non-differentiable quantization step.
  • Practical Mobile Deployment Validation: The system is validated end-to-end on real mobile hardware (Qualcomm QNN HTP, TFLite GPU delegate), providing concrete latency numbers rather than only theoretical FLOPs counts.

Methodology

Network Architecture. The model is a hierarchical encoder-decoder (UNet-style). The encoder uses gated encoder blocks at multiple resolution scales. Each gated block runs two parallel convolutional branches; a third convolutional branch with tanh activation produces a gating mask that performs element-wise filtering of the combined branch activations. Multi-channel skip connections carry the triplet (xa, xg, xb) features to the decoder.

Multi-Scale Refinement. At decoding time, refinement modules process features at S/2, S/4, and S/8 scales using residual convolutional blocks before upsampling. This allows the network to correct errors at coarse scales before committing to fine-scale predictions.

Quantization-Aware Training. The full-precision model is first trained to convergence. QAT then fine-tunes the model with inserted fake quantization nodes (simulating INT8 rounding and clipping) at weights and activations. The straight-through estimator passes gradients through the quantization operation as if it were an identity function during backpropagation. This allows the model to adjust its weight distributions to be quantization-friendly, recovering quality lost by post-training quantization (PTQ).

Deployment. The final INT8 model is exported to ONNX and compiled for Qualcomm QNN HTP (neural signal processor) and TFLite GPU delegate. The challenge imposed a mandatory 22 dB PSNR floor on the DPED validation set; the team's full-precision model achieves 22.19 dB, clearing this threshold, while the QAT INT8 model achieves 21.05 dB.

  • QAT recovers +0.474 dB PSNR and +0.111 SSIM over post-training quantization.
  • QNN HTP Latency: 41.8 ms (≈ 3.6× speedup vs. FP16 baseline)
  • TFLite GPU Delegate: 319 ms for 1920×1080 images (full model: 469 ms)
  • Mobile AI 2026 RGB Image Enhancement Challenge: 2nd place, final score 3.8
  • The challenge's mandatory 22 dB PSNR threshold eliminates architecturally simpler but lower-PSNR variants.

Critical Assessment

Novelty: The individual components (gated convolutions, multi-scale refinement, QAT) are individually well-established. The contribution is the combination and application to the specific problem of mobile image enhancement under quantization constraints. The novelty is primarily engineering rather than conceptual.

Rigor: The experimental setup is tightly focused on the competition task (DPED dataset, specific mobile hardware). The comparison between QAT and PTQ is the key scientific claim, and the +0.474 dB difference is meaningful in practice. However, evaluation is limited to one dataset and two hardware platforms.

Reproducibility: Moderate. Code will be released. The method is described at a level sufficient to re-implement. However, the exact QAT schedule, learning rate, and data augmentation pipeline are not detailed in the abstract-only content available.

Significance: High for practitioners working on mobile image enhancement and edge AI deployment, particularly those working with DSP/NPU backends where INT8 quantization is mandatory. Limited significance for the broader ML research community.

Red Flags: (1) The full-precision model's 22.19 dB just barely clears the 22 dB competition threshold—the margin is thin. (2) Evaluation on a single dataset (DPED) limits generalizability claims. (3) QAT still degrades PSNR by 1.14 dB vs. full precision—substantial for perceptual quality. (4) The 469 ms TFLite latency suggests the model may still be too slow for real-time video applications despite the QNN speedup.

Strengths

  • Concrete end-to-end deployment validation on real mobile hardware with measured latency
  • QAT vs. PTQ comparison demonstrates clear practical value of the approach
  • Competition results provide independent validation
  • Code release promised, supporting reproducibility

Limitations

  • Evaluated on a single dataset (DPED); generalization to other image enhancement tasks (dehazing, denoising, HDR) is undemonstrated
  • INT8 QAT still incurs ~1.1 dB PSNR loss vs. full precision—may be unacceptable for high-quality photography applications
  • TFLite GPU latency (469 ms full model, 319 ms for the challenge variant) precludes real-time video processing on many use cases
  • The gating mechanism adds memory bandwidth overhead that may limit scalability to higher-resolution inputs
  • No analysis of failure cases or input domains where quantization particularly hurts

Relevance

本文与 ML 系统可靠性及数值精度方向有直接关联：量化感知训练中的伪量化节点与直通估计器是解决部署阶段数值精度损失（类比训练中的 silent data corruption）的实用工程手段，对研究低精度推理稳定性与训练-部署一致性有参考价值。

Questions for Further Exploration

  1. How does QAT performance scale with more aggressive quantization (INT4, mixed INT4/INT8)? Is the +0.474 dB recovery consistent, or does it diminish at lower bit-widths?
  2. Does the gating mechanism provide measurable benefit over a standard encoder of equivalent parameter count, or is the improvement attributable primarily to the multi-scale refinement?
  3. How sensitive is the QAT result to the fine-tuning schedule (number of steps, learning rate, when to introduce fake quantization)? Is there a principled recipe?
  4. Would knowledge distillation (full-precision teacher → INT8 student) outperform QAT fine-tuning for this task?
  5. How does the model perform on inputs significantly different from DPED (e.g., nighttime images, extreme exposure, non-smartphone cameras)?
