> **一句话总结：** 本文对 LMArena 13.5 万条人类偏好数据进行分层分析，发现不同任务类别下模型排名差异显著（Spearman 相关 0.60–0.93），并构建了允许用户自定义类别权重的交互式排行榜可视化工具，揭示聚合排名在数学题正确性判断和政治争议性问题上的根本局限。

> **TL;DR:** This FAccT 2026 paper empirically analyzes LMArena's Human Preference 140K dataset (135,634 pairwise judgments across 53 LLMs, April–July 2025), builds a 3-level semantic topic hierarchy (8 top-level categories, 400 fine-grained clusters), and demonstrates that global aggregate rankings mask dramatic per-category rank inversions (e.g., minimax-m1 jumps from 19th globally to 1st in math). An interactive React/Flask visualization tool lets users customize category weights to define their own "best" LLM.

## 核心贡献

- **LMArena 数据集深度分析**：对 LMArena Human Preference 140K（135,634 条判断，53 个 LLM，2025 年 4–7 月）进行系统性分层分析，量化排名在不同任务类别下的稳健性。
- **三层语义话题层级体系**：通过 k-means（k=400）聚类加 GPT 辅助层级归并，构建 8 个顶层、53 个中层、400 个细粒度类别的话题分类体系。
- **排名分歧可视化**：计算 Spearman 相关系数（0.60–0.93），揭示全局排名与类别特定排名之间的系统性偏差；具体案例包括 minimax-m1（全局第 19→数学第 1）、kimi-k2（政治争议类下降 16 位）、Claude 系列（数据处理类跃升 top-5）。
- **偏好评估根本性局限**：对数学题（23% 的对比中一方正确一方错误，但人类仅 74% 选择正确答案）和政治争议性问题（多元化回答与非多元化回答无统计显著差异，p=0.25）的偏好局限进行实证分析。
- **交互式排行榜工具**：React 前端 + Flask 后端，支持层级类别选择与权重自定义，实时更新 LLM 排名；通过 N=10 用户研究（45 分钟 Zoom 会话）验证可用性。

## Key Contributions

- **Empirical Slice Analysis of LMArena**: First systematic disaggregated analysis of the LMArena Human Preference 140K dataset. Shows that the global ranking obscures large rank differences across task categories—Spearman correlations between global and category-specific rankings range from 0.60 to 0.93, with lower correlations in specialized domains.
- **Hierarchical Topic Taxonomy**: A 3-level semantic hierarchy constructed by embedding all prompt texts, clustering into k=400 groups with k-means, then using GPT to organize clusters into 53 mid-level and 8 top-level categories. The categories reveal that ~30% of all LMArena prompts are coding/software development, strongly biasing the global ranking toward coding ability.
- **Category-Specific Ranking Reversals**: Concrete examples of models whose global rank is misleading: minimax-m1 (19th globally → 1st in math), Claude models (not top-5 globally → top-5 in Data Processing), kimi-k2 (drops 16 places in political dispute prompts).
- **Evaluation Limits Analysis**: Identifies two fundamental failure modes for preference-based evaluation: (1) tasks with objectively correct answers (math, factual QA) where human preferences do not reliably track correctness, and (2) value-laden tasks (political disputes) where "better" is inherently contested.
- **Interactive Visualization System**: A full-stack system (React/SVG frontend, Flask backend) that allows users to select which categories matter to them and weight them, producing a personalized model ranking. Includes a heatmap view of model performance across categories and a drill-down into individual conversations.
- **Qualitative User Study**: N=10 participants (45-minute Zoom sessions) exploring the tool, providing insights into how different user profiles (developers, researchers, general users) conceptualize "best" LLM.

## Methodology

**Dataset.** LMArena Human Preference 140K: 135,634 pairwise comparisons collected on the LMSYS Chatbot Arena platform between April and July 2025, covering 53 LLMs. Each record contains the prompt, two anonymous model responses, and the human preference label (A/B/tie).

**Topic Hierarchy Construction.**
1. Embed all prompt texts using a sentence embedding model.
2. Cluster embeddings using k-means with k=400.
3. Use GPT to generate descriptive labels for each of the 400 clusters.
4. Use GPT again to hierarchically merge 400 clusters into 53 mid-level categories and 8 top-level categories. The top-level categories are: Coding/Software Dev, Data Processing, Math, Creative Writing, Information Retrieval, Reasoning, Instruction Following, and Other.

**Category-Specific Rankings.** For each category, re-compute pairwise win rates across all models using the subset of conversations in that category. Use Bradley-Terry model to convert pairwise win rates to scalar ratings. Compare per-category ratings to global ratings via Spearman rank correlation.

**Limits Analysis.** For math: identify conversations where one model's answer is objectively correct and the other is objectively wrong (via automated checking), then measure what fraction of human judges selected the correct answer. For political disputes: identify political prompt subset, categorize model responses as pluralistic (presenting multiple viewpoints) vs. non-pluralistic, and measure whether human preference differs (chi-squared test).

**Interactive Visualization.** The frontend renders a hierarchical topic selector (tree structure), a weight slider for each category, a ranked model list updating in real time, and a heatmap showing each model's win rate per category. Built with React and D3/SVG. The Flask backend serves precomputed per-category win rate matrices.

**User Study.** N=10 participants recruited via convenience sampling. 45-minute Zoom sessions with think-aloud protocol. Participants were asked to use the tool to find "the best LLM for my use case" and explore the categories. Sessions were recorded and analyzed using thematic analysis.

## Results & Highlights

- **Category Distribution**: ~30% of LMArena prompts are Coding/Software Development, making the global ranking substantially influenced by coding performance.
- **Ranking Instability**: Spearman correlations between global and category-specific rankings range from 0.60 (Math) to 0.93 (Creative Writing), indicating substantial rank reshuffling in specialized domains.
- **minimax-m1**: Jumps from 19th place globally to 1st place in Math category—a 18-rank improvement.
- **Claude models**: Jump to top-5 in Data Processing category despite not placing there globally.
- **kimi-k2**: Drops 16 places in political dispute prompts.
- **Math correctness failure**: In 23% of math conversations, one model is correct and one is wrong. Of these, humans only select the correct answer 74% of the time—meaning 1 in 4 preferences in objective math tasks goes to the wrong answer.
- **Political pluralism non-result**: Human preference between pluralistic and non-pluralistic political responses is not statistically significant (p=0.25), suggesting the arena cannot adjudicate value-laden quality dimensions.
- **User study findings**: Users varied widely in which categories they prioritized; the tool successfully helped users discover that their preferred model differed from the global #1; some users expressed surprise at how category-dependent rankings were.

## Authors & Credibility

All five authors are from Yonsei University, South Korea. Minsuk Kahng is a faculty member with a background in interactive ML visualization (previously Google Brain). The paper is accepted to FAccT 2026 (ACM Conference on Fairness, Accountability, and Transparency), a rigorous peer-reviewed venue specifically focused on evaluation methodology and accountability in AI systems. This venue is particularly appropriate for the paper's focus on who gets to define "best" and the potential for leaderboards to encode biased definitions of quality.

## Critical Assessment

**Novelty**: The core insight—that aggregate leaderboard rankings mask category-specific differences—is not new (several papers have made similar observations about MMLU and other benchmarks). The specific contribution of applying this to LMArena human preference data, building an interactive tool, and conducting the qualitative study on limits (math correctness, political pluralism) represents genuine incremental value.

**Rigor**: The Spearman correlation analysis is straightforward but appropriate. The math correctness analysis (74% human accuracy on objectively verifiable questions) is the most compelling empirical finding. The political pluralism analysis (p=0.25) is correctly presented as a null result. The user study is qualitative and small (N=10), limiting generalizability; the authors acknowledge this.

**Reproducibility**: Moderate. The dataset (LMArena Human Preference 140K) is publicly available. The topic taxonomy construction requires API calls to GPT, which introduces non-reproducibility. The visualization tool is described but it is unclear whether the code and precomputed data will be released.

**Significance**: High for the evaluation and fairness communities. The finding that human preference data is unreliable for objectively correct/incorrect tasks has direct implications for how LMArena results should be interpreted and cited. The interactive tool is a practical contribution.

**Red Flags**: (1) The 400-cluster taxonomy is produced with GPT assistance, making it subjective and non-reproducible. (2) The paper covers April–July 2025 LMArena data—the leaderboard has changed significantly since, limiting timeliness. (3) The user study N=10 is too small to support strong claims about user needs or tool effectiveness. (4) The claim that ~30% of prompts are coding is interesting but may reflect the self-selected LMArena user population rather than general LLM use.

## Strengths

- Real dataset analysis rather than synthetic benchmark construction
- The math correctness finding (74% preference accuracy) is a clear, memorable empirical result
- Appropriate venue (FAccT) for the fairness-oriented framing
- The interactive tool addresses a genuine need: personalized leaderboard views
- Null result on political pluralism is honestly reported

## Limitations

- Temporal scope limited to April–July 2025; the leaderboard landscape may differ substantially by the time of publication
- User study too small for generalizable conclusions
- The topic taxonomy introduces subjectivity via GPT-based labeling; alternate taxonomies might produce different insights
- Does not address whether the "sliced" rankings are themselves stable (bootstrap confidence intervals on category-specific Elo not reported)
- The interactive tool is demonstrated but not externally evaluated for actual impact on evaluation practices

## Relevance

本文与 LLM 推理评估及评估可靠性方向直接相关：分层分析揭示了聚合排行榜隐藏任务特定性能差异的系统性问题，这对正确解读 LLM 评估结果（尤其是以人类偏好数据为基础的基准测试）至关重要；数学题正确性判断中人类偏好仅 74% 准确的发现，直接质疑了偏好评估在有客观答案任务上的有效性。

## Questions for Further Exploration

1. Are the category-specific rankings themselves stable? That is, would the ranking within the Math category change substantially if we bootstrap the LMArena data? What are the confidence intervals on category-specific Elo?
2. The 30% coding dominance of LMArena prompts reflects the platform's user base—how would category distributions differ on a general-public platform vs. a developer-focused one?
3. Can the approach be extended to identify systematic prompt-level biases (e.g., longer prompts, prompts from specific geographic regions, prompts submitted during certain hours) that might further distort global rankings?
4. The 74% math correctness result is alarming—but is this uniform across model pairs, or concentrated in specific model matchups where both models are roughly equally (in)correct?
5. What UI design changes would help non-expert users meaningfully choose category weights rather than defaulting to the global ranking?
