> **一句话总结：** DiffMAS 通过将 KV cache 作为可学习的隐式通信接口，在多智能体轨迹上进行端到端监督微调（LoRA），使智能体协同学习"如何编码和解读跨智能体信息"，在 AIME24 上获得 +26.7% 的提升并显著降低解码困惑度。

> **TL;DR:** DiffMAS is a training framework for multi-agent LLM systems that treats inter-agent KV-cache communication as a learnable, differentiable interface. By performing LoRA supervised fine-tuning over full multi-agent latent trajectories, it jointly optimizes reasoning and communication, outperforming single-agent, text-based, and training-free latent communication baselines on math, science QA, code, and commonsense benchmarks while improving decoding stability.

## 核心贡献

- 将多智能体系统中的智能体间通信形式化为优化问题：现有文本通信接口在智能体边界产生离散化瓶颈，阻碍端到端梯度传播。
- 提出 DiffMAS：以 KV cache 为连续隐式通信介质，上游智能体顺序构建共享 KV trace，下游终端智能体在预填充 KV trace 上进行自回归解码，整体可微。
- 理论分析证明：拼接式（concatenation）通信接口相较于覆写式（overwriting）通信接口，不引入深度相关的梯度衰减因子（命题 3.1）。
- 实验表明 DiffMAS 在多个模型规模（Qwen3-4B/8B/14B、Ministral-3-8B、DeepSeek-R1-Distill-32B）和多类任务上均一致优于基线，解码困惑度更低、自一致性更高。

## Key Contributions

- Formalizes multi-agent communication as an optimization problem, showing that fixed discrete message-passing creates optimization barriers between agents that prevent end-to-end improvement.
- Proposes DiffMAS: a supervised training framework treating KV-mediated latent communication as a differentiable, task-adaptive interface between agents.
- Proves (Proposition 3.1 / Corollary A.1) that the concatenation-based KV trace interface introduces no depth-dependent multiplicative gradient decay, whereas overwriting interfaces decay geometrically as ρᴷ⁻ʲ under contractive Jacobian assumption.
- Demonstrates that trained latent communication yields lower perplexity (mean 1.24 vs. 1.31 for LatentMAS on AIME2024) and higher self-consistency across Qwen3-4B and Qwen3-8B.
- Provides thorough ablations: communication steps sweep, StitchMAS control, agent ablation, and TextMAS+SFT comparison.

## Methodology

DiffMAS operates on a sequential K-agent pipeline (Planner → Critic → Refiner → Solver). Communication proceeds through two stages:

**Stage 1 — KV Trace Construction:** Upstream agents 1 to K-1 sequentially process their inputs (system prompt + user query) and produce KV segments by prefilling existing KV traces and appending newly generated KV blocks without gradient updates to the backbone. The accumulated KV trace Z₁:Nⱼ = (z₁,...,zNⱼ) ∈ Tⱼ serves as the sole communication medium, growing monotonically by concatenation.

**Stage 2 — LoRA SFT on Final Agent:** The final (Solver/Judger) agent performs autoregressive decoding conditioned on the accumulated KV trace. Cross-entropy loss is computed against ground-truth target sequences. Gradients backpropagate through the latent communication pathway to update only LoRA parameters (rank r=8, scale α=16, dropout 0.05) while the backbone remains frozen. This allows the model to jointly learn how upstream agents should encode information and how downstream agents should interpret it.

**Training data:** Task-specific small subsets—210 samples from Hendrycks Math (math/science), 50 from HumanEval (code), 700 Gemini-3-Flash-generated reasoning traces from CommonsenseQA (commonsense). Training uses AdamW with cosine LR schedule (LR = 5×10⁻⁵), gradient accumulation over 64 micro-batches, gradient clipping at 1.0.

**Inference:** Temperature 0.6, top-p 0.95. Task-specific maximum output lengths (2048 tokens for OpenBookQA up to 32,768 for AIME 2024/2025).

**Key theoretical insight:** For an overwriting interface hⱼ₊₁ = Fθ(hⱼ,x,pⱼ₊₁) with contractive Jacobian ‖∂hⱼ₊₁/∂hⱼ‖₂ ≤ ρ < 1, the gradient magnitude reaching stage j decays as ρᴷ⁻ʲ‖∂L/∂hₖ‖₂. For concatenation, ‖∂L/∂Z^(j)_{1:T}‖₂ ≤ ‖∂L/∂Z_{1:Nₖ}‖₂ with no depth factor.

## Results & Highlights

- Qwen3-4B: AIME24 43.3% → 63.3% (+20.0%), GPQA-Diamond 42.4% → 46.4% (+4.0%), MBPP+ 63.5% → 71.9% (+8.4%).
- Qwen3-8B: AIME24 50.0% → 76.7% (+26.7%), GPQA-Diamond 39.9% → 60.1% (+20.2%), HumanEval+ 74.5% → 81.5% (+7.0%).
- Qwen3-14B: AIME24 63.3% → 76.7% (+13.4%), MBPP+ 68.5% → 77.2% (+8.7%).
- DeepSeek-R1-Distill-32B: AIME24 66.7% → 70.0% (+3.3%).
- Perplexity: DiffMAS mean PPL 1.24 vs. LatentMAS mean 1.31 on AIME2024/Qwen3-4B; LatentMAS shows heavy-tailed outliers >2.0 indicating unstable decoding.
- Communication steps ablation: peak at 10 steps (76.7% on AIME24); performance degrades with >40 steps, suggesting learned compact protocol.
- StitchMAS control (independently generated KV segments stitched): consistently underperforms DiffMAS (60.0% vs. 76.7% on AIME24), confirming that continuous KV trace construction—not just SFT—drives the gain.
- Agent ablation: zeroing any single agent's latent steps reduces accuracy from 76.7% to 70.0%, with comparable drops regardless of which agent is ablated—consistent with concatenation's non-overwriting property.

## Critical Assessment

**Novelty:** Genuine. While KV-cache sharing between agents has been explored (Zou et al. 2025, Fu et al. 2025), DiffMAS is the first to treat latent communication itself as a trainable component optimized end-to-end via SFT over full multi-agent trajectories, with the non-overwriting design choice grounded in a theoretical gradient analysis.

**Rigor:** Good. The ablations are thorough: TextMAS+SFT control separates task learning from communication learning; StitchMAS control isolates the continuous trace construction; agent ablation validates the gradient accessibility claim. The gradient analysis is clearly conditional on stated assumptions (contractive Jacobian), appropriately qualified.

**Reproducibility:** Moderate concern. Training on 50–210 samples with task-specific LoRA adapters means results could be sensitive to the particular samples chosen. The paper does not report variance/error bars on key accuracy numbers. Code and data are not explicitly released in the preprint. Synthetic reasoning traces for CommonsenseQA were generated by Gemini-3-Flash—an external proprietary model that may not be stably reproducible.

**Significance:** High for multi-agent LLM systems research. The framework directly addresses a known limitation of text-serialization bottlenecks and provides a practical recipe compatible with black-box backbone models.

**Red flags:** (1) No error bars on accuracy comparisons—gaps are sometimes large but statistical significance is not established. (2) C2C baseline was trained on OpenHermes-2.5 (instruction-following data) rather than reasoning trajectories, making it a weaker comparison than it appears. (3) The 4-agent Planner-Critic-Refiner-Solver topology is fixed; it is unclear how gains generalize to different topologies or asynchronous pipelines. (4) KV trace grows linearly in length with the number of agents, potentially introducing quadratic attention cost at inference.

## Strengths

- Clean theoretical motivation: gradient decay analysis clearly explains why overwriting hurts and concatenation helps.
- Parameter-efficient: only LoRA adapters are trained on very small datasets, backbone remains frozen.
- Strong empirical results across multiple model families and task types.
- Perplexity and self-consistency analyses provide mechanistic evidence for stability improvement, not just accuracy.
- Honest about ablation findings: longer traces can hurt, TextMAS+SFT matches on easier benchmarks.

## Limitations

- KV trace length grows linearly with agents; attention computation over prefilled trace is O(N²) in sequence length.
- Training requires full multi-agent trajectory rollouts, which is more expensive than single-agent SFT.
- Task-specific LoRA adapters mean separate fine-tuning for each reasoning domain—limited generalization across task types without per-domain adaptation.
- Evaluation is primarily on math/reasoning benchmarks; performance on open-ended generation or tool-use tasks is untested.
- The synthetic CommonsenseQA supervision traces depend on Gemini-3-Flash quality.

## Relevance

本文与用户关注的**LLM 推理稳定性**和**推理评估**高度相关：DiffMAS 的核心问题之一是隐式 KV 通信导致的解码不稳定性（困惑度尖峰），论文通过可学习通信设计显著改善了这一问题，对理解多智能体系统可靠性具有直接价值。

## Questions for Further Exploration

1. Does the trained KV communication encode interpretable structure—e.g., do attention patterns from the Solver agent over the KV trace reveal which upstream agents' contributions are most attended to?
2. Can DiffMAS be extended to asynchronous or parallel agent topologies where the communication graph is not strictly sequential?
3. How does KV trace length scale with problem difficulty—does the model learn to allocate more communication capacity to harder problems?
4. Would replacing LoRA with full fine-tuning (or a larger rank) further improve performance, and at what cost to generalization?
5. What happens when upstream agents are different model families (heterogeneous MAS)—does the KV trace remain interpretable across architectures?
