Shallow Prefill, Deep Decoding: Efficient Long-Context Inference via Layer-Asymmetric KV Visibility

["Jungsuk Oh", "Hyeseo Jeon", "Hyunjune Ji", "Kyongmin Kong", "Jay-Yoon Lee"]

arXiv: 2605.06105

Date: 2026-05-07

一句话总结： SPEED 通过仅在浅层（前K层）构建 Prefill Token 的 KV 缓存、同时保持 Decode Token 全深度可见，在 Llama-3.1-8B 上以仅损失 0.2 分均值的代价将 128K 上下文的 TTFT 提升 33%、TPOT 提升 22%、KV 内存减少 25%。
TL;DR: SPEED (Shallow Prefill, dEEp Decode) introduces a phase-asymmetric KV-visibility policy for decoder-only LLMs: prompt tokens have their KV states materialized only through the lower K layers, while generated (decode) tokens remain full-depth. A single BoS token anchored at full depth stabilizes the regime. On Llama-3.1-8B with K=24 (75% of layers), this cuts 128K TTFT by 33%, TPOT by 22%, and active KV memory by 25% while staying within 0.2 points of the full-depth instruction-tuned baseline on OLMES-style benchmarks. The approach is orthogonal to existing KV compression methods and can be applied via one epoch of LoRA from an off-the-shelf instruct model.

核心贡献

  • 非对称 KV 可见性策略（SPEED）： 提出 Prefill 阶段浅层化、Decode 阶段全深度的 KV 可见性分配方案，从根本上改变 Prefill Token KV 状态在高层的持久化方式，而非仅压缩或剪枝已有缓存。
  • BoS 锚点稳定机制： 发现利用已有的 BoS（序列开始）Token 作为唯一的全深度 Prefill 侧锚点，即可有效抑制 anchor-free SPEED 产生的后缀重复循环等不稳定现象，无需学习新的摘要 Token。
  • 层级诊断工具： 提供基于注意力质量（提示选择峰值层、条件提示熵最小值层、隐向量轨迹稳定化峰值层）的层级截断诊断方法，指导 K 值的合理选择，避免穷举搜索。
  • 轻量级适配路径： 证明 SPEED 可通过单轮 LoRA 适配从现成 Instruct 模型快速引入，在 HotpotQA、TriviaQA、S-NIAH 等任务上与全深度 LoRA 基线接近。
  • 效率与质量前沿量化： 在受控的从头指令调优对比实验中，系统量化了 K 值选择与各任务类别（数学、代码、推理、知识、指令）之间的质量-效率权衡。

Key Contributions

  • Phase-asymmetric KV-visibility policy (SPEED): Prefill tokens are processed and cached only through the first K layers; decode tokens traverse all L layers and produce full-depth KV states. This reduces dominant prefill-side KV memory from O(LN) to O(KN) and eliminates repeated upper-layer prefill-cache reads during autoregressive generation.
  • BoS anchor stabilization: A single existing BoS token retained as a full-depth prefill-side anchor recovers most of the quality lost from anchor-free SPEED and suppresses suffix-repetition instabilities, without any learned compression or additional memory tokens.
  • Layer-wise cutoff diagnostic: Attention mass, conditional prompt entropy, and hidden-trajectory straightening signals are used to identify the depth at which selective prompt access and representation stabilization occur, providing principled guidance for choosing K.
  • Off-the-shelf LoRA compatibility: SPEED can be introduced from a pre-trained instruct checkpoint via one epoch of task LoRA adaptation, with moderate cutoffs (K=24, K=20) remaining close to full-depth LoRA on document-grounded QA and long-context retrieval.
  • Controlled quality-efficiency benchmarking: All SPEED variants and the full-depth baseline share the same fine-tuning mixture, optimizer, and evaluation protocol on TULU-3-DEV / OLMES, isolating the effect of KV-visibility from confounds.

Methodology

SPEED operates on a standard L-layer decoder-only transformer. During the Prefill phase, non-anchor prompt tokens have their KV tensors materialized only for layers 1 through K. The BoS token and all Decode-phase tokens are materialized at every layer. Position indices are not renumbered, so SPEED changes which KV states are visible, not the positional identity of tokens.

  • Upper layers (l > K): In SPEED+BoS, only {s} ∪ D_{

Memory cost model: Full attention stores O(LN) prefill-side KV; SPEED stores O(KN), reducing the dominant term by a factor of K/L. At K=24, L=32, this is a 25% reduction. The same factor applies to Prefill compute (TTFT) and to Decode-time prefill-cache reads (TPOT improvement).

Training: SPEED-aware supervised fine-tuning applies the same visibility rule during teacher-forced training as at inference. Prompt positions use shallow-prefill visibility; assistant target positions use decode-token visibility. The language-modeling loss is unchanged. Post-hoc application (inference-only) creates a train-test mismatch; SPEED-aware training is preferred for full quality recovery.

Cutoff selection diagnostic: For a chosen K, the layer-wise analysis examines (a) where attention mass from decode tokens to prefill tokens peaks (prompt selection), (b) where conditional prompt entropy over prefill tokens is minimized (selective access), and (c) where hidden-trajectory straightening peaks (representation stabilization). K should exceed the straightening peak (typically L17–L19 for Llama-3.1-8B) to preserve both the selective prompt-access region and the stabilization region. Coding tasks are an exception: their prompt-attention and entropy peaks both appear as early as L3, explaining why code quality degrades little even under aggressive cutoffs.

LoRA adaptation path: Starting from Llama-3.1-8B-Instruct, one epoch of LoRA task adaptation on HotpotQA pseudo-labeled examples is sufficient to recover most of full-depth LoRA quality for SPEED+BoS variants at moderate cutoffs.

Results & Highlights

Main instruction-tuning sweep (Table 2, Llama-3.1-8B, 128K context):

  • Full-IT (K=32): 51.4 avg, TTFT baseline, TPOT baseline, KV baseline.
  • IT-SPEED-24+BoS (K=24, 75% layers): 51.2 avg (+0 TTFT penalty; TTFT +33%, TPOT +22%, active KV -25.0%).
  • IT-SPEED-20+BoS (K=20): 49.9 avg, TTFT +60%, TPOT +36%, active KV -37.5%.
  • IT-SPEED-16+BoS (K=16): 45.4 avg, TTFT +101%, TPOT +55%, active KV -50.0% — significant quality drop.

BoS anchor importance: At K=24, anchor-free IT-SPEED-24 scores 49.1; adding BoS recovers to 51.2. BoS also eliminates suffix-repetition loops observed in anchor-free variants.

Comparison with stage-aware Prefill baselines (Figure 2): At K=24, POP-24, SwiftKV-24, and SPEED-24 achieve similar TTFT reductions. However, SPEED-24 additionally improves TPOT and yields the lowest active KV memory, because non-anchor prefill tokens are fully absent from upper-layer Decode attention, eliminating repeated cache reads during generation.

SelfOnly ablation (Table 5): Removing upper-layer Decode-token attention to other Decode-phase tokens (while keeping the same shallow-prefill rule) drops average from 51.2 to 47.2, confirming that upper-layer Decode-token interaction is necessary and that SPEED's gains come specifically from removing prefill (not decode) attention in upper layers.

Off-the-shelf LoRA (Table 3): OffShelf-FT-SPEED+BoS-24 reaches 59.5/73.7 on HotpotQA EM/F1, 81.4/86.5 on TriviaQA, 46.4/59.8 on NQ, and 99.6 on S-NIAH, compared to 60.8/75.3, 80.5/86.0, 48.5/62.4, 97.7 for full-depth LoRA. The moderate SPEED cutoff is highly competitive with a single epoch of LoRA.

Critical Assessment

Novelty: The core idea — differentiating KV materialization depth between Prefill and Decode phases — is conceptually clean and distinct from existing approaches. While POP and SwiftKV also stage Prefill work, they preserve or synthesize upper-layer Prefill KV states for Decode; SPEED is the first to eliminate them entirely from upper-layer Decode attention. The BoS anchor insight is elegant and practically important. Novelty is moderate-to-high within the KV-cache efficiency space.

Rigor: The controlled instruction-tuning sweep is a genuine strength — matching all hyperparameters across variants is rare in efficiency papers, which often compare across different training recipes. The layer-wise diagnostic analysis is principled and connects mechanistically to observed quality patterns. The SelfOnly ablation cleanly isolates the source of gains. Aggregate benchmark gaps are small (0.2 points), and the paper is appropriately cautious that these are not statistical equivalence tests. The efficiency measurements (TTFT, TPOT, active KV) are obtained under a shared measurement protocol, though realized serving-stack gains may differ with batching, prefix sharing, or speculative decoding.

Reproducibility: The experimental setup is well-documented (Llama-3.1-8B architecture, 178,502 training examples, 2 epochs, TULU-3-DEV eval, OLMES protocol). However, the paper does not link to released code or model checkpoints, which limits immediate reproducibility. The LoRA adaptation experiment uses pseudo-labeled HotpotQA data whose curation is described only briefly.

Significance: Long-context inference efficiency is a high-impact area. A 25% KV memory reduction and simultaneous TPOT improvement with near-zero quality loss at K=24 is a meaningful result for production deployment. The method is orthogonal to quantization, sparse attention, and token eviction, suggesting composability. The insight that decode tokens need not attend to upper-layer prefill KV states — because lower layers already capture most prefill information — has broader implications for architecture design.

Red flags: (1) Results are on a single architecture (Llama-3.1-8B, 32 layers); transferability to deeper models, MoE, or non-Llama families is not demonstrated. (2) The quality-efficiency operating point at K=24 shows only a 0.2-point aggregate gap, but Math drops from 46.6 to 45.3 — task-specific degradation could matter for specialized deployments. (3) No code release. (4) The LoRA compatibility experiment uses task-specific adaptation data (HotpotQA), so the result is not a clean zero-shot generalization test of SPEED. (5) Efficiency claims on TTFT/TPOT depend on the specific serving stack used; the paper acknowledges this but the reported numbers may not transfer to all deployment configurations.

Strengths

  • Conceptually simple and surgically targeted: changes only which KV states are materialized, not model weights, positional encodings, or vocabulary.
  • Strong controlled experimental design that isolates KV-visibility as the sole variable.
  • Dual efficiency improvement: both Prefill (TTFT) and Decode (TPOT) costs are reduced, which is unusual — most Prefill-focused methods do not also reduce TPOT.
  • BoS anchoring is a zero-cost stabilization mechanism requiring no additional learned parameters.
  • Layer-wise diagnostic provides interpretable guidance for cutoff selection rather than pure grid search.
  • Off-the-shelf LoRA compatibility substantially lowers the barrier to deployment.
  • Paper is transparent about failure modes: aggressive cutoffs degrade math and instruction following, and the authors report these honestly.

Limitations

  • Evaluated only on Llama-3.1-8B (32 layers); K=24 as optimal cutoff may not generalize to architectures with different layer counts or depth-wise attention patterns.
  • Quality degradation is task-dependent and non-uniform: math and instruction-following are more sensitive to shallow-prefill visibility than coding, which means the method's safety margin varies by deployment use case.
  • Anchor-free SPEED can destabilize generation (suffix-repetition loops), making BoS anchoring a practical necessity rather than an option — but BoS may behave differently in models with different tokenization or pre-training recipes.
  • Efficiency gains are measured under a specific single-device inference configuration; production serving with continuous batching, prefix caching, or speculative decoding may see different relative improvements.
  • The method requires SPEED-aware fine-tuning for best results; post-hoc application creates train-test mismatch and likely further degrades quality.
  • No code or checkpoint release in the preprint version limits independent verification.
  • The layer-wise diagnostic is described as a guiding heuristic, not a reliable per-example cutoff predictor; it does not eliminate the need for empirical validation when deploying to new tasks or models.

Questions for Further Exploration

  1. How does the optimal cutoff K scale with model depth (e.g., 70B or 405B models with 80 layers)? Does the diagnostic signal (selective prompt access, representation stabilization peaks) shift proportionally?
  2. Can SPEED be composed with KV quantization or token eviction to achieve multiplicative memory reductions without compounding quality loss?
  3. The BoS token is used as a single anchor — how would a small learned set of anchor tokens (e.g., 4–8 tokens trained as gist-like summaries) compare in quality recovery vs. anchor overhead?
  4. The SelfOnly ablation shows upper-layer Decode-to-Decode attention is necessary. Does this suggest that the efficiency gains could be further improved by also reducing the lower-layer Decode depth, or does this create instability?
  5. How does SPEED interact with prefix caching in production serving systems, where prefill KV states are often reused across requests? Does shallow prefill reduce or complicate the benefit of prefix sharing?
  6. For reasoning-heavy tasks (math, multi-step logic), the quality gap grows at aggressive cutoffs. Could a task-adaptive K selection policy — using a quick probe of the input's task type — recover quality without sacrificing efficiency on easier tasks?
