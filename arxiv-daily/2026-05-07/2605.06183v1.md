> **一句话总结：** 通过引入梯度能量探针 PAGE，作者发现 LoRA 的梯度信号高度集中于单一浅层 FFN 下投影矩阵（称为"主导适应模块"），并据此提出 DomLoRA——仅在该模块放置一个适配器，以约 0.7% 的 vanilla LoRA 参数量在四项下游任务上平均超越 vanilla LoRA。

> **TL;DR:** This paper asks where a single LoRA adapter should be placed to maximize performance. The authors propose PAGE (Projected Adapter Gradient Energy), a gradient-based probe that measures how much trainable gradient signal each candidate adapter would receive at initialization. Applying PAGE to Qwen3-8B and LLaMA-3.1-8B reveals that gradient energy is overwhelmingly concentrated at one shallow FFN down-projection layer — the same layer across all tested downstream tasks. The authors call this the "dominant adaptation module" and build DomLoRA, which places exactly one LoRA adapter there and freezes everything else. Despite using only ~0.7% of vanilla LoRA's trainable parameters (~2.1–2.3M vs. 321–334M), DomLoRA outperforms vanilla LoRA on average across instruction following, math reasoning, code generation, and multi-turn conversation, and also improves AdaLoRA, DoRA, LoRA+, NoRM, and GraLoRA when used as their placement strategy.

## 核心贡献

- **PAGE 探针**：提出投影适配器梯度能量（PAGE）指标，通过经验 Fisher 灵敏度估计每个候选 LoRA 适配器在初始化时能接收到的可训练梯度能量，只需 32 个监督样本的样本级梯度，无需任何微调即可计算。
- **主导适应模块的发现**：PAGE 值在所有注意力投影和 FFN 投影中高度集中于单一浅层 FFN 下投影矩阵；该层索引因模型架构而异（Qwen3-8B 为第 6 层，LLaMA-3.1-8B 为第 1 层），但在同一骨干网络上跨下游任务保持稳定，是预训练结构的固有属性而非任务特性。
- **DomLoRA 方法**：将一个 LoRA 适配器仅放置于主导模块，冻结其余所有参数，以约 0.7% 的 vanilla LoRA 可训练参数量在平均性能上超越 vanilla LoRA。
- **即插即用的通用性**：DomLoRA 可作为 AdaLoRA、DoRA、LoRA+、NoRM、GraLoRA 等变体的放置策略，对大多数变体均有性能提升（平均提升 +1–5 点）。

## Key Contributions

- **PAGE probe**: A gradient-based sensitivity metric that estimates the initial trainable gradient energy available to each candidate LoRA adapter using empirical Fisher sensitivity. Requires only 32 supervised samples and no preliminary fine-tuning.
- **Dominant adaptation module discovery**: PAGE is highly concentrated at a single shallow FFN down-projection across two model families and four downstream task domains. The layer index is architecture-dependent but task-stable, indicating it is an intrinsic property of the pretrained backbone.
- **DomLoRA**: A placement method that places exactly one LoRA adapter at the dominant adaptation module (~2.1–2.3M parameters) and freezes all others, outperforming vanilla LoRA (321–334M parameters) on average across instruction following, math reasoning, code generation, and multi-turn conversation.
- **Plug-and-play compatibility**: DomLoRA improves AdaLoRA, DoRA, LoRA+, NoRM, and GraLoRA when applied as their placement strategy, demonstrating generality beyond the base LoRA formulation.

## Methodology

**Problem Setup.** Vanilla LoRA places trainable low-rank adapters across attention and FFN projections in most or all Transformer layers. Recent work shows fewer adapters can match or exceed performance, but the question of *which* single location is optimal remained open.

**PAGE Derivation.** For each candidate module (l, k), the authors define module sensitivity S_emp as the average squared Frobenius norm of sample-wise gradients of the pretrained projection weight. Under standard LoRA initialization (B=0, A ~ Kaiming-uniform), the initial gradient received by the trainable B factor is G * A^T. PAGE is defined as the expected squared norm of this projected gradient, averaged over the random initialization distribution of A. Theorem 1 shows this expectation simplifies to (s^2 * r) / (3 * d_in) * S_emp, making PAGE proportional to module sensitivity divided by input dimension. PAGE can thus be computed before any fine-tuning using only 32 probe samples.

**Dominant Module Identification.** PAGE is evaluated at every attention (Q, K, V, O) and FFN (up, gate, down) projection in Qwen3-8B and LLaMA-3.1-8B. Attention and FFN up/gate projections exhibit small, smoothly distributed PAGE values. FFN down-projections show a sharp spike at one shallow layer: Layer 6 in Qwen3-8B, Layer 1 in LLaMA-3.1-8B. This spike accounts for 16.5–18.2% of aggregate PAGE across all 252 projection modules, and 74.7–79.0% of aggregate PAGE among FFN down-projections alone. The spike is present at Step 0 (before any fine-tuning) and persists across all checkpoints (Steps 1–390).

**DomLoRA Placement.** The dominant layer l* = argmax PAGE_{l, down} is identified using Step-0 PAGE on the pretrained backbone. A single LoRA adapter (B* in R^{d x r}, A* in R^{r x d_ff}) is placed at W_down^{l*}; all other projection weights remain frozen. The same module is reused across downstream tasks without re-running the probe.

**Baselines Compared.** Vanilla LoRA (adapters in all layers/modules), IST (layer-wise importance selection), PLoP (module-type selection), and five LoRA variants (AdaLoRA, DoRA, LoRA+, NoRM, GraLoRA).

## Results & Highlights

**General Instruction Tuning (Tulu V2, evaluated on MMLU/TyDiQA/CQA/TruthfulQA/GSM8K/LogiQA):**
- Qwen3-8B: DomLoRA (2.1M params) avg 74.5 vs. vanilla LoRA (334M) avg 72.9, PLoP (131M) avg 73.8, IST (334M) avg 73.8.
- LLaMA-3.1-8B: DomLoRA (2.3M) avg 64.8 vs. vanilla LoRA (321M) avg 60.8.

**Math/Code/Conversation (GSM8K, MATH, HumanEval, HumanEval+, MT-Bench):**
- Qwen3-8B: DomLoRA avg 71.7 vs. vanilla LoRA 63.5 (+8.2), PLoP 66.5, IST 65.7.
- LLaMA-3.1-8B: DomLoRA avg 60.0 vs. vanilla LoRA 54.1 (+5.9).

**LoRA Variants on LLaMA-3.1-8B (instruction tuning):**
- All five tested variants improve when using dominant-module placement (2.3M params), with the largest gains for LoRA+ (+4.1 avg) and AdaLoRA (+2.7 avg).

**Ablation (Table 5):** Moving W_down to a non-dominant layer (Layer 31 or Layer 10) substantially degrades performance. Replacing W_down at Layer 1 with W_up or W_gate also reduces average score. Using all FFN projections at Layer 1 (6.8M params) still underperforms DomLoRA, confirming the gain comes from the correct module selection rather than simply concentrating parameters at a single layer.

**Efficiency:** DomLoRA reduces trainable parameters by ~99.3% relative to vanilla LoRA, with proportional reductions in GPU memory and training time.

## Authors & Credibility

The paper is from South China University of Technology (SCUT; five authors) and Zhejiang University (one author). The corresponding author is Huiping Zhuang (hpzhuang@scut.edu.cn). The group works on parameter-efficient fine-tuning and LLM adaptation. The paper is an arXiv preprint dated 7 May 2026 and has not yet been published at a peer-reviewed venue. The experiments are conducted on publicly available models (Qwen3-8B, LLaMA-3.1-8B-Instruct) and publicly available datasets/benchmarks, making the claims independently verifiable.

## Critical Assessment

**Novelty:** The core insight — that Fisher sensitivity concentrates at a single FFN down-projection — is genuinely novel and non-obvious. The PAGE formulation cleanly connects empirical Fisher sensitivity to LoRA initialization gradient energy via a closed-form theorem. The dominant-module concept is a meaningful conceptual advance over prior layer-selection or module-type-selection methods.

**Rigor:** The mathematical derivation of PAGE is sound and well-presented. The experimental coverage is broad (two backbones, four task domains, five LoRA variants, multiple ablation conditions). However, all experiments are limited to 8B-scale models; it is unknown whether the dominant-module phenomenon generalizes to smaller (1–3B) or larger (70B+) models. The paper acknowledges this as a limitation.

**Reproducibility:** The probe requires only 32 supervised samples and no optimizer state, making it cheap and easy to replicate. Training hyperparameters (rank 64, lr 2e-5, cosine schedule) are specified. However, the paper uses GPT-5 as the MT-Bench judge, which is a non-open, evolving model — MT-Bench scores may not be reproducible exactly.

**Significance:** The result that ~2M parameters at one layer can match or outperform 334M parameters distributed broadly is practically significant for low-resource or edge fine-tuning scenarios. The plug-and-play applicability to multiple LoRA variants increases the practical impact.

**Red Flags:**
- Only two backbone architectures tested; the architecture-dependence of the dominant layer index means users must run the PAGE probe for each new backbone.
- The paper does not compare against search-based or gradient-free adapter placement methods in a wall-clock-time controlled setting.
- MT-Bench judged by GPT-5 is non-reproducible for external researchers.
- The PAGE probe itself requires GPU memory to compute sample-wise gradients over all projection weights simultaneously, which may be non-trivial for very large models.
- No statistical significance tests are reported for benchmark comparisons; some gains are within 1–2 points.

## Strengths

- Elegant, theoretically grounded probe (PAGE) that reduces to empirical Fisher sensitivity with a clean closed-form derivation.
- Remarkably strong empirical results for the parameter budget: outperforming vanilla LoRA with 0.7% of its parameters is a striking and practically useful finding.
- Architecture-dependent but task-stable dominant module means the probe needs to run only once per backbone, not once per task.
- Plug-and-play compatibility demonstrated across five LoRA variants, not just the base method.
- Comprehensive ablation confirming that both the module type (W_down) and the specific layer index matter — the gain is not simply from concentrating parameters.

## Limitations

- Tested only on dense Transformer models at 8B scale; applicability to MoE architectures, vision-language models, encoder-only models, or very small/large models is unverified.
- The PAGE probe adds a preprocessing step requiring sample-wise gradient computation over all candidate projection weights, introducing GPU memory overhead before training begins.
- The dominant layer index is backbone-specific; practitioners adopting DomLoRA must run the probe for each new model family.
- MT-Bench evaluation relies on GPT-5 as judge, limiting external reproducibility for that benchmark.
- No analysis of failure cases or tasks where DomLoRA underperforms (GraLoRA's GSM8K drop from 83.4 to 75.7 when using dominant placement is noted but not deeply analyzed).

## Relevance

本文提出的 DomLoRA 方法对希望在有限算力下高效微调大模型的研究者极具参考价值——仅需约 2M 参数即可在多项任务上超越全量 LoRA，对边缘部署和低资源场景尤为实用；同时，PAGE 探针作为一种轻量级的适配器位置分析工具，对理解 Transformer 内部信息流和知识存储机制也有一定启发意义。

## Questions for Further Exploration

1. Does the dominant adaptation module phenomenon hold for models at 1B, 13B, 70B, and 405B scales, and does the dominant layer index shift predictably with model depth?
2. Does the PAGE-identified dominant module coincide with the "knowledge neuron" or key-value memory layers identified in interpretability research, or is it a distinct structural feature?
3. How does DomLoRA perform under continual fine-tuning scenarios where multiple tasks are adapted sequentially — does the single dominant module become a bottleneck?
4. Can the PAGE probe be extended to MoE architectures where FFN layers are replaced by sparse expert mixtures?
5. Is there a theoretical explanation for why the FFN down-projection (rather than up or gate) systematically dominates, and is this related to the dimensionality reduction role of the down-projection in Transformer FFN blocks?
6. The authors use GPT-5 as an MT-Bench judge — how sensitive are the multi-turn conversation results to the choice of judge model?
