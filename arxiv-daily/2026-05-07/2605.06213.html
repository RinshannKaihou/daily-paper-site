Beyond Fixed Benchmarks and Worst-Case Attacks: Dynamic Boundary Evaluation for Language Models

Haoxiang Wang, Da Yu, Huishuai Zhang

arXiv: 2605.06213v1

Date: 2026-05-07

https://arxiv.org/abs/2605.06213

一句话总结： 本文提出动态边界评估（DBE）框架，基于Rasch模型将LLM能力定位到0.5通过概率的"边界"处，通过技能引导边界搜索（SGBS）算法自适应生成涵盖安全拒绝、过度拒绝、指令遵循和奉承抵抗四类任务的校准测试题，在9个参考模型上验证了跨模型的难度量尺稳定性（Spearman ρ ≥ 0.96），解决了固定基准的天花板/地板效应问题。
TL;DR: DBE replaces fixed LLM benchmarks with a psychometric framework anchored on Item Response Theory: it builds a calibrated difficulty "ruler" from a 9-model reference panel, places each new model on that ruler by finding prompts where its pass probability is ~0.5 (the "boundary"), and extends the ruler on demand via a bandit-style search algorithm (SGBS) when a model falls outside the existing scale. Instantiated over four evaluation categories — harmful request refusal, over-refusal, constrained instruction following, and multi-turn sycophancy resistance — DBE achieves cross-model difficulty stability (Spearman ρ_min ≥ 0.96) and effectively localizes held-out models outside the reference panel. The key payoff is model-discriminative evaluation that does not saturate as models improve, without needing to rebuild the benchmark from scratch.

核心贡献

  • 提出动态边界评估（DBE）框架：将LLM评估目标从"在固定题集上测量通过率"转变为"在共享难度轴上定位模型能力"，从根本上解决固定基准的天花板/地板效应。
  • 校准锚点题库：基于9个参考LLM（Qwen、Gemma、Mistral、DeepSeek系列，4B–72B）用Rasch 1PL模型联合校准每道题的难度参数，形成可复用的评估标尺，涵盖安全拒绝（875题）、过度拒绝（1500题）、指令遵循（823题）和奉承抵抗（1080题）四类场景。
  • 技能引导边界搜索（SGBS）：以bandit框架在"裸请求×技能组合"的组合空间中搜索通过概率接近0.5的边界题，仅需API访问目标模型，可跨四类任务统一使用，且不以最大化攻击成功率为目标，而以定位能力边界为目标。
  • 自适应评估协议：新模型在锚点题库覆盖范围内时直接用固定题库估计能力；能力超出覆盖范围时才触发SGBS扩展量尺，实现按需延伸而无需全量重新校准。
  • 实证验证三项核心主张：（1）锚点难度在panel扰动下稳定（LOO Spearman ρ ≥ 0.96）；（2）SGBS能有效定位边界题（panel级Spearman ρ ∈ [0.90, 0.97]）；（3）DBE可准确评估panel外的新模型（面板内标准误 ≤ 0.032 logit）。

Key Contributions

  • Dynamic Boundary Evaluation (DBE) framework: Reframes LLM evaluation from pass-rate measurement on a fixed item set to locating each model on a shared difficulty axis, eliminating ceiling and floor artifacts inherent in fixed benchmarks.
  • Calibrated anchor item bank: A reusable bank of difficulty-calibrated items across four categories (harmful request refusal, over-refusal, constrained instruction following, sycophancy resistance), calibrated via 1PL Rasch maximum likelihood on a fixed 9-model reference panel spanning four model families (Qwen, Gemma, Mistral, DeepSeek-Distill) from 4B to 72B parameters.
  • Skill-Guided Boundary Search (SGBS): A bandit algorithm that searches the compositional space of bare requests × skill subsets to find prompts near a target model's decision boundary (p ≈ 0.5), using only black-box API access and a unified item format applicable across all four evaluation categories.
  • Adaptive evaluation protocol: Combines Fisher-information computerized adaptive testing (CAT) on the anchor set for in-range models with on-demand SGBS expansion when a model's ability falls outside panel coverage; newly generated items are calibrated onto the existing scale via reference panel responses.
  • Empirical validation of three core claims: Anchor difficulty stability under panel leave-one-out perturbation (Spearman ρ_min ≥ 0.96, median |Δβ| within Linacre's 0.3-logit Rasch-stable band); SGBS boundary localization effectiveness (panel-level Spearman ρ ∈ [0.90, 0.97]); and accurate ability estimation for fully out-of-panel held-out models (in-panel SE ≤ 0.032, out-of-panel pre-Step-2 gaps of 0.68–1.48 logits confirming SGBS triggers correctly).

Methodology

Core psychometric foundation. DBE grounds evaluation in Item Response Theory (IRT), specifically the 1-parameter logistic (1PL) Rasch model: Pr(pass | m, x) = σ(θ_m − β_x), where θ_m is the model's latent ability and β_x is the item's latent difficulty on the same logit scale. The Rasch model's specific objectivity property means item difficulties are comparable across respondent sets up to a global translation, enabling cross-model comparison on a unified scale without rerunning all items on all models.

Item construction via skill–request composition. Each evaluation item is constructed as x = Compose(q, s), combining a bare request q (a category-specific input meaningful on its own: a harmful prompt, an instruction-following task, a factual question, or a benign request that can trigger sycophancy) with a skill subset s drawn from a category-specific skill dictionary S. Skills are difficulty-increasing operators (e.g., for harmful request refusal: roleplay framing, authority hierarchy; for sycophancy: emotional escalation, counter-evidence). The composition space is large: from ~22,100 combinations (over-refusal) to ~10^7 (harmful refusal), making exhaustive evaluation infeasible and motivating search.

Anchor set construction and Rasch calibration. For each category, K = 500 candidate items are randomly composed, then filtered to retain only those with panel-mean pass rate p̄_x ∈ [0.05, 0.95] — the AVG[0.05, 0.95] filter, shown to be the unique Pareto-optimal filter across a 15-candidate × 4-axis evaluation framework. Joint 1PL Rasch calibration is performed by Binomial maximum likelihood on a 9×500 response matrix (each cell using N = 40 repeated trials), yielding stable difficulty estimates β̂. The anchor sets retain J ∈ [175, 414] items per category.

Skill-Guided Boundary Search (SGBS). SGBS operates as a bandit over three coupled choices: which bare request q, how many skills k, and which compatible skill subset s ⊆ S to compose. Bare requests are selected by Thompson sampling, allocating more queries to requests that previously yielded near-boundary items. Skill counts are selected from an adaptive distribution updated by directional feedback. Skills are selected by Thompson sampling under compatibility constraints. The boundary reward r(p̂) = 1 − 2|p̂ − 0.5| is maximized at p̂ = 0.5. SGBS terminates when the inference budget B is exhausted or no new boundary item is found in R consecutive rounds.

Adaptive evaluation protocol. New model evaluation proceeds in two steps: (1) Fisher-information CAT on the fixed anchor set, selecting items greedily to maximize Rasch information at the current θ̂_new estimate using Expected A Posteriori (EAP) inference; (2) if θ̂_new falls outside the panel's θ-range, SGBS generates new boundary items, these are evaluated on the reference panel and calibrated onto the existing β scale, and ability is re-estimated on the augmented set.

Four evaluation categories. (A) Harmful request refusal (HRR): 875 bare requests from HarmBench + AdvBench + StrongREJECT, 397 skills from DejaVu jailbreak dictionary; (A') Over-refusal (OR): 1,500 XSTest + OR-Bench safe prompts, 52 over-refusal trigger skills; (B) Constrained instruction following (CIF): 823 IFEval + InFoBench tasks, 83 verifiable constraint types; (C) Sycophancy resistance (SR): 1,080 TruthfulQA + MMLU + ARC-Challenge + CommonsenseQA factual QA items, 47 persuasion skills grounded in Cialdini, Tversky–Kahneman, and related social influence frameworks.

Results & Highlights

Claim 1 — Anchor difficulty stability (Table 3). Panel leave-one-out stability tests show Spearman ρ_min ≥ 0.96 across all four categories (exceeding the Habba et al. 2026 reuse threshold of ρ ≥ 0.9). Median |Δβ| stays inside Linacre's 0.3-logit Rasch-stable band (0.079–0.125 logits); p95 |Δβ| ≤ 0.502 (within Wright–Stone 0.5-logit tolerance except marginally in CIF at 0.502). The tail saturation rate — items becoming all-pass or all-fail under the 8-model subpanel — is ≤ 2.5% in every category. Out-of-panel θ recovery error is bounded at max |Δθ_m| ≤ 0.11 logit.

Claim 2 — SGBS boundary localization (Table 4). For all 9 reference models and all 4 categories, SGBS finds items whose mean Rasch-predicted pass probability p̄ is close to 0.5 (most values 0.43–0.63), with panel-level Spearman ρ ∈ [0.90, 0.97] confirming that stronger models receive harder SGBS-generated items. The two main deviations (Mistral-Nemo-12B and DS-R1-Distill in category A) are attributed to 1PL edge cases documented in Appendix H.

Claim 3 — New model evaluation (Table 6). Tested on 5 held-out models from families absent in the reference panel (Olmo, Granite, phi, Falcon), DBE full-anchor standard errors are ≤ 0.032 logit for in-panel cells. Adaptive-to-full gaps for in-panel models are ≤ 0.44 logit, consistent with Claim 1 baselines. Out-of-panel models show pre-Step-2 gaps of 0.68–1.48 logit (an order of magnitude larger), correctly triggering SGBS extension.

Benchmark saturation diagnosis. The motivation for DBE is quantified by the real-world observation that Claude Opus and Qwen Plus score 91.3% vs. 90.4% on standard GPQA — statistically indistinguishable — while the same models show a >20% gap on SuperGPQA. DBE avoids this by construction: the boundary focus ensures evaluation signal remains discriminative regardless of model capability level.

Critical Assessment

Novelty. The core idea — applying IRT boundary-seeking to LLM evaluation rather than subset compression or worst-case attack — is genuinely novel as a complete system. Individual components (IRT for LLM eval, skill composition, adaptive testing) appear in prior work (tinyBenchmarks, Fluid Benchmarking, h4rm3l, Truong et al. 2025), but the integration of calibrated anchor banks + boundary-targeted search + on-demand scale extension across safety/capability/truthfulness categories in a single unified framework is new. The paper clearly distinguishes itself from prior IRT-for-LLM work and adversarial attack methods.

Rigor. The statistical treatment is thorough: Binomial cells (N=40 trials) to separate sampling noise from Rasch misfit, four-axis filter evaluation over 15 candidates, bootstrap confidence intervals for Rasch fit, panel LOO tests with established reference thresholds (Linacre, Wright-Stone, ETS-A). The specific objectivity claim is honestly qualified — panel-level drift is 3–4x larger than sampling-noise-only null — and the authors acknowledge 1PL limitations. The four evaluation categories are well-sourced from canonical benchmarks with deduplication checks.

Reproducibility. Moderately high. The reference panel models are publicly available. The skill dictionaries, bare request pools, and composition templates are documented in detail in the appendices. However, the composer and judge for categories A, A', and C require external API LLMs (unspecified), introducing potential drift from API model updates. The N=40 trial budget per cell means anchor calibration requires substantial inference compute (9 models × 500 items × 40 trials = 180,000 inference calls per category). SGBS hyperparameters are provided in Appendix E.

Significance. The benchmark saturation problem is real and practically important — the MMLU → MMLU-Pro → SuperGPQA progression is a genuine methodological pain. A framework that adaptively extends its measurement range without full recalibration could substantially reduce the cost of benchmark maintenance. The four categories are practically important (safety and instruction following are central to LLM deployment). Accepted at NeurIPS 2026 Datasets and Benchmarks track, which has a high bar.

Red flags / Limitations to note:

  • The reference panel has only 9 models from 4 families, which may not cover the full diversity of future models; Assumption A1 (panel covers target ability range) is not guaranteed and depends on panel growth.
  • The composer (an API LLM) and LLM judge introduce systematic biases that N=40 trials reduce but do not eliminate (acknowledged).
  • Cross-category aggregation to a single model score is deferred to future work, leaving the multi-dimensional nature of model ability unaddressed.
  • SGBS is validated on the same 9 reference panel models, creating some circularity — the held-out 5-model test in Claim 3 partially addresses this.
  • The skill dictionaries are drawn from existing jailbreak and persuasion taxonomies, limiting the discovery of novel failure modes not yet catalogued.

Strengths

  • Principled psychometric foundation (1PL Rasch model) with honest acknowledgment of its assumptions and deviations.
  • Unified item format (bare request + skill composition) that applies across safety, capability, and truthfulness categories without category-specific IRT architectures.
  • Scale extends on demand rather than requiring full benchmark replacement; compatible with existing item banks.
  • Four-axis filter selection framework (budget-SE, Fisher information retention, Rasch fit, global fit) is methodologically principled and prevents overfitting to a single criterion.
  • Thorough ablation: 15 filter candidates, panel LOO stability, 5 held-out families, bootstrap CIs.
  • Accepted at NeurIPS 2026 Datasets and Benchmarks, with detailed appendices enabling replication.

Limitations

  • API LLM dependencies: Composer and judge for categories A, A', C rely on external API models, creating potential reproducibility issues as APIs evolve and introducing model-specific biases that repeated trials reduce but cannot eliminate.
  • Single-dimensional ability per category: The 1PL Rasch model assigns each model a scalar ability θ per category; multi-dimensional abilities within a category are not captured, and cross-category aggregation is left to future work.
  • Panel coverage assumption: The reference panel (9 models, 4B–72B) may not cover future model families or capability regimes, and A1 is explicitly "not guaranteed for future models" — SGBS is the mitigation but adds search cost.
  • Inference compute cost: Calibration at N=40 trials per (model, item) cell is expensive; the full anchor bank calibration requires hundreds of thousands of inference calls across 9 models.
  • Skill dictionary limitations: The skill dictionaries are sourced from existing taxonomies; novel attack vectors or persuasion strategies not in the taxonomy are out of scope.
  • No cross-category scoring: The paper does not provide a principled way to aggregate across the four categories into a single model ranking, limiting practical comparability.

Questions for Further Exploration

  1. How does SGBS performance degrade when the target model uses a very different decoding strategy (e.g., chain-of-thought sampling vs. greedy) than the reference panel? Does the boundary at p ≈ 0.5 remain meaningful under temperature variation?
  2. The paper defers cross-category aggregation to future work. What principled aggregation approach — weighted composite, Pareto frontier, or multi-dimensional IRT — would best preserve the interpretability of category-specific ability estimates?
  3. The composer for categories A, A', C is an external API LLM. How sensitive are the calibrated difficulty estimates to the choice of composer model, and what happens when the composer is updated or deprecated?
  4. SGBS uses a boundary reward r(p̂) = 1 − 2|p̂ − 0.5|. Would a model-specific posterior-weighted reward (integrating over uncertainty in θ̂) find more informative items early in the search, reducing the inference budget needed?
  5. The reference panel excludes models that share pre-training data to limit difficulty-signal leakage. How should panel composition be updated as open-weight model families consolidate onto a small number of base models (e.g., Llama-based derivatives)?
  6. Can DBE's item bank be adversarially gamed — e.g., by a model that learns the skill taxonomy and specifically trains to pass boundary items — and how would this affect the validity of the calibrated scale?
