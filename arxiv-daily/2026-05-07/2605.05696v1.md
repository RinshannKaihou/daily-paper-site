> **一句话总结：** Irminsul 通过利用 MLA（多头潜在注意力）将 KV 向量分解为位置无关的 $c_{KV}$（512维）和可闭式修正的 RoPE 键 $k_r$（64维）的特性，提出了首个原生适配 MLA 的位置无关缓存系统，在 Agentic LLM 推理场景中相比精确前缀缓存额外恢复约 77–83% 的 prompt token，并实现每次缓存命中节省 63% 的 prefill 能耗。

> **TL;DR:** Agentic LLM workloads (e.g., multi-turn tool use, RAG, code agents) constantly shift identical content to new token positions, defeating conventional prefix-based KV caches. Irminsul is the first serving-system caching layer designed natively for Multi-Head Latent Attention (MLA), the architecture used in DeepSeek-V2/V3/R1, Kimi-K2, GLM-5, and others. It keys cached KV chunks by content hash (using CDC chunking) rather than token position, and corrects the small RoPE-carrying slice ($k_r$, only 64 of 576 dims) at lookup time via a closed-form $\delta$-rotation — avoiding the per-hit full-$d_K$ recompute cost that makes position-independent caching impractical on GQA. Evaluated on three production MLA-MoE deployments (DSv2-Lite 16B/2.4B, Moonlight-16B-A3B, JoyAI-Flash 48B/3B), Irminsul recovers up to ~83% of prompt tokens above exact-prefix on agentic traffic while delivering 63% prefill energy savings per cache hit, with output consistency matching or beating naive reuse across QA, summarization, and NIAH benchmarks. If your serving stack runs MLA-family models under agentic or multi-session workloads, this paper directly describes a drop-in system improvement.

## 核心贡献

- **首个 MLA 原生位置无关缓存系统（Irminsul）**：利用 MLA 将 KV 行分解为位置无关的潜在向量 $c_{KV}$（89% 占比，AUC≈0.77）和仅 11% 的 RoPE 键 $k_r$（64维），通过单次 $\delta$-旋转即可在闭式内完成位置修正，彻底规避了 GQA/MHA 所需的全维度代价。
- **基于内容的分块哈希键**：采用 CDC（Content-Defined Chunking）滚动哈希划定语义边界，xxHash64 为每个 chunk 生成与位置无关的指纹，使相同 token 无论出现在 prompt 哪个位置均命中同一缓存条目。
- **量化可恢复的 Agentic 流量重用潜力**：在三个公开多轮轨迹语料库（Toolathlon、CC-Bench、Hermes，共 7530 条轨迹、3.4×10⁸ token）上证实，会话内 p95 token 重用率高达 48%，跨会话为 12–24%，为系统设计提供了充分的经验依据。
- **精确的位置不变性 ROC 研究**：设计阈值无关 AUC 测试，证明 MLA 的 $c_{KV}$（NoPE 分量）在所有层均为位置无关（AUC≈0.77），而 GQA 的完整 K 矩阵 AUC<0.5，表明不加修正的 GQA 缓存复用比随机更差。
- **跨架构能耗研究**：实测表明 softmax 架构（GQA/MLA/MHA）每次缓存命中节省 63–86% 的 prefill 能耗，而 Mamba2/GDN/KDA 等混合 SSM 架构节省约 0%，从结构上界定了 PIC 的适用范围。

## Key Contributions

- **Irminsul, the first MLA-native Position-Independent Caching (PIC) system**: exploits MLA's structural decomposition — 89% of each KV row is a position-free latent $c_{KV}$ (AUC≈0.77 under the paper's ROC test) and only 11% is a RoPE-carrying $k_r$ correctable in closed form via a single $\delta$-rotation — to make per-hit correction cost $O(64N)$ instead of GQA's $O(d_K \cdot N)$.
- **CDC-chunked content-hash keying**: uses a Gear-hash rolling boundary rule (expected chunk size ~128 tokens, clamped to [32, 512]) and xxHash64 fingerprints to identify semantically identical segments regardless of absolute position; a 64-token boundary marker (S2 strategy) synchronises the rolling hash state across requests assembled by the same framework.
- **Empirical agentic reuse characterisation**: measures three public multi-turn corpora (7,530 trajectories, 3.4×10⁸ tokens) showing within-session p95 reuse of 48% (Toolathlon) and cross-session 12–24%, establishing that position-shifted reuse is a real, large-scale phenomenon not an artefact.
- **Threshold-free ROC methodology for PIC feasibility**: a principled test separating same-content-different-position (positive) from different-content pairs (negative) across 500 content blocks at 5 absolute positions, showing MLA's NoPE component is structurally cacheable while GQA's full K is not.
- **Cross-architecture energy study**: first published measurement bounding PIC's energy scope — softmax architectures save 63–86% prefill energy per hit; hybrid SSM families (Mamba2, GDN, KDA) save ~0% because the recurrent pass re-runs regardless of cache state.
- **Output-consistency validation across four models and three datasets**: per-token KL divergence, argmax-match, and first-divergence position confirm PIC matches or beats naive reuse on QA, GovReport summarisation, and NIAH recall across DSv2-Lite, Moonlight-16B, JoyAI-Flash 48B, and a retrofitted TransMLA-4B.

## Methodology

**Problem framing.** Standard prefix caching (SGLang RadixAttention, vLLM PagedAttention) indexes KV by token-sequence hash at fixed block granularity and hits only when content appears at the same position as before. Agentic pipelines assemble prompts by composing a static system prompt, growing history, and dynamically-placed retrieved documents and tool outputs; the composition shifts previously seen content to new positions every turn, voiding the prefix cache at the first byte of divergence. The authors quantify this: on real OpenClaw traces, Req 2 gets 0% cache hit and Req 3 gets 15% despite sharing ~90% of their tokens with a previously served request.

**Why prior PIC systems fall short on MLA.** CacheBlend, EPIC, MEPIC, KV-Packet, and SemShareKV all recover chunk-level KV across positional shifts on GQA. But GQA fuses position inseparably into the full $K$ tensor (algebraically correctable only at full $d_K$ width per token), leaving a residual per-hit correction cost proportional to all 1,024 dims of Qwen3-32B's K heads. MEPIC's bespoke kernel is GQA-specific. None has been evaluated on MLA.

**The MLA structural opportunity.** MLA (used in DeepSeek-V2/V3/R1, Kimi-K2/Moonlight, GLM-5, Mistral Large 3) factors each token's KV row into: (a) a position-free latent $c_{KV}$ (512 dims), which is 89% of the 576-dim row and has AUC≈0.77 in the paper's ROC test — meaning it is directly reusable at zero per-hit correction cost; and (b) a 64-dim RoPE-carrying $k_r$, which has AUC≈0.43 (below random, hence actively harmful if reused uncorrected) but is exactly correctable by a $\delta$-rotation $R(\delta) = R(p_{new}) \cdot R(p_{src})^{-1}$, fused into FlashMLA's HBM→SRAM load path at $O(N \cdot 64)$ multiplies with no extra HBM bandwidth.

**Irminsul algorithm (Algorithm 1).** On each incoming prompt: (1) standard RadixCache exact-prefix hit for the longest matching prefix; (2) CDC boundary detection on the remaining tokens — the Gear-hash boundary rule (mask exponent $k=7$, expected size ~128 tokens) segments the tail into variable-length content-defined chunks; (3) each chunk is fingerprinted with xxHash64 and looked up in a content-hash registry; (4) on a registry hit, $\delta = p_{new} - p_{src}$ is computed and $k_r$ is rotated via $R(\delta)$; (5) chunks with absolute position $p < 32$ are prefilled normally ("first-chunk carve-out") to avoid disturbing the attention-sink regime at position 0; (6) novel chunks are prefilled and inserted into the registry for future reuse.

**First-chunk carve-out rationale.** The attention sink phenomenon (first ~32 tokens absorb disproportionate attention mass) is shown empirically to be sequence-start-local (not chunk-start-local) for MLA and GQA: the within-chunk first-32-token attention fraction (W) collapses to ~0.25 baseline at $p \geq 1024$ for MLA, justifying prefill-via-ordinary-path only for position-0 chunks. MHA shows a different pattern (locally concentrated sink), and the authors explicitly scope their safety claim to MLA only.

**Boundary synchronisation (S2 strategy).** The default deployment inserts a 64-token literal boundary marker around each shared region at prompt-assembly time, flushing the Gear-hash rolling state to a known value so that identical content produces identical CDC boundaries regardless of absolute position. For deployments where the assembler is not under operator control, a fallback S1 strategy sub-divides CDC chunks into fixed 128-token windows for sub-window hash recovery.

**Runtime KV layout.** The 576-dim KV row is split into a content-hashed $c_{KV}$ pool (512 dims, shared across sessions) and a per-request $k_r$ pool (64 dims, position-specific). A gather-rotate-scatter materialises the rotated $k_r$ at retrieval time; production deployments can fuse the rotation into the FlashMLA load path so the rotated tensor never materialises in HBM.

**Evaluation setup.** Three native MLA-MoE deployments: DSv2-Lite (16B/2.4B), Moonlight-16B-A3B (DSv3-form rotary), JoyAI-Flash (48B/3B). A 950-LoC reference scaffold patches SGLang's RadixCache in observer mode (records would-be hits without rerouting KV), separating recovery measurement from treatment latency. Output-consistency uses per-step $\text{KL}(p_{full}\|p_x)$, argmax-match, and first-divergence position on HOTPOTQA∪MUSIQUE, GovReport, and NIAH ($n=30$–100 per cell).

## Results & Highlights

- **Cache recovery on agent_meta pattern**: on agentic prompts where a small per-agent header shifts the bulk of the prompt, Irminsul recovers **77.2% of prompt tokens above exact-prefix** on DSv2-Lite (exact-prefix alone: 1.9%) and **82.7%** on JoyAI-48B; together with exact-prefix, total cache coverage reaches ~79–84%.
- **Prefill energy savings**: 63% per cache hit for MLA (vs. 86% for GQA, 66% for MHA), measured via NVML hardware energy counters on real SGLang serving.
- **Output consistency**: PIC's per-step KL divergence is 48–73% lower than naive reuse across all four models on QA; argmax-match matches or exceeds naive reuse on every populated cell; first-divergence position is 2.4–9.0× later than naive reuse on JoyAI-48B NIAH.
- **$\delta$-rotation precision**: algebraic error is $4.7 \times 10^{-3}$ rel-L2 in bf16, non-accumulating, an order of magnitude tighter than production FP8 KV quantisation ($10^{-2}$–$3 \times 10^{-2}$).
- **CDC vs. fixed-block offline comparison**: CDC+fallback recovers 3.8–4.8× more reusable chunks than fixed-block hashing on Toolathlon and Hermes corpora.
- **Hybrid SSM architectures (Mamba2, GDN, KDA)**: confirmed zero PIC benefit — the energy saving is $\approx 0\%$ across three independent families, bounding where any PIC system can deliver value.
- **Partition-shift diagnostic**: Irminsul's marginal value (tokens recovered above exact-prefix) equals the fraction of tokens shifted beyond the first variation point — operators can predict ROI from two measurable statistics: first-variation position and variation-set cardinality.

## Critical Assessment

**Novelty:** High within its niche. The structural observation that MLA's $c_{KV}$ / $k_r$ factorisation makes PIC algebraically cheap is genuine and not present in prior PIC literature (CacheBlend, EPIC, MEPIC, KV-Packet, SemShareKV). The CDC-based content-hash keying is a straightforward application of a known technique (rolling hash chunking, common in deduplication systems) but its combination with the $\delta$-rotation and integration into SGLang's RadixCache is novel.

**Rigor:** Generally strong. The ROC methodology for position-invariance testing is principled and threshold-free. Energy measurements use hardware counters with documented zero-sample handling. Output-consistency is measured with multiple metrics (KL, argmax-match, first-divergence) across multiple models and tasks. The attention-sink carve-out is empirically validated across three architectures.

**Reproducibility:** Moderate. The algorithm is fully specified (Algorithm 1, Appendices C–H), the models are public, and the workload corpora are cited. However, no code is released. The 950-LoC scaffold is described but not linked. The $\delta$-rotation RoPE-frequency auto-detection (reading `inv_freq` from the rotary module) is described in enough detail to re-implement but requires care to avoid the silent fallback bug the paper documents in Appendix D.

**Significance:** High for practitioners running MLA-family models (DeepSeek-V2/V3/R1, Kimi-K2, GLM-5, Moonlight, JoyAI-Flash, Mistral Large 3) under agentic workloads. The 10–16s TTFT regressions and 100%→35% cache-hit-rate regressions cited from production post-mortems are real operational pain, and Irminsul directly addresses them.

**Scope limitations / red flags:**
- The safety claim (correctness of the first-chunk carve-out) is explicitly scoped to **MLA only**; MHA is included only in energy and ROC studies, not in the output-consistency or recovery tables.
- The paper does not report end-to-end TTFT under Irminsul treatment (only baseline TTFT), because the fused FlashMLA $\delta$-rotation kernel is described as a "deliberate follow-up". The latency benefit of the system is therefore not demonstrated.
- DeepSeek-V4's Heavily Compressed Attention (HCA) dissolves the per-token content identity that Irminsul's chunking requires — the authors acknowledge Irminsul does not run on DSv4 HCA layers and defer adaptation to future work.
- All three evaluated deployments (DSv2-Lite, Moonlight, JoyAI-Flash) are from a narrow band of MLA-MoE architectures in the DSv2/v3 family. Generalisation to other MLA instantiations (e.g., MiniCPM3-4B's non-absorbed form) is noted but not experimentally confirmed.
- The workload characterisation relies on offline trajectory corpora; online serving dynamics (variable request interleaving, eviction policies) are not modelled.

---

## Strengths

- Precise structural diagnosis: identifies exactly why prior PIC systems are architecturally mismatched to MLA and exploits the factorisation that makes the fix cheap.
- Multiple complementary evaluations: offline reuse characterisation, ROC feasibility test, energy study, recovery measurement, and output-consistency study — each answering a different prerequisite question.
- Honest scope delineation: explicitly states MHA safety is not covered, hybrid SSM architectures receive zero benefit, and DSv4 HCA is not supported.
- Practical deployment path: two deployment strategies (S2 cooperative boundary marker, S1 detect-and-recover fallback) cover both operator-controlled and third-party assembler cases.
- The partition-shift diagnostic gives operators a simple two-statistic predictor of ROI before deployment.

---

## Limitations

- No end-to-end latency results under treatment (fused kernel deferred to follow-up work).
- No released code or reproducibility artefact at time of publication.
- Safety guarantees cover MLA only; MHA-PIC requires a separate boundary-recompute path not studied here.
- Incompatible with DeepSeek-V4 HCA layers, limiting future applicability as the community moves toward aggregated-KV representations.
- CDC boundary synchronisation (S2) requires framework cooperation; S1 fallback is less efficient and the boundary-misalignment gap is not quantified across all corpora.
- Single institution; no independent replication.

## Questions for Further Exploration

1. What is the actual end-to-end TTFT reduction under the fused FlashMLA $\delta$-rotation kernel? The paper explicitly defers this; a follow-up result would complete the latency case.
2. How does Irminsul interact with KV eviction policies (LRU, prefix-tree eviction in SGLang) under memory pressure? Does content-hash keying require a different eviction strategy than prefix keying?
3. Can the CDC chunking primitive be adapted to DeepSeek-V4's Heavily Compressed Attention aggregation schedule, and if so, does the content-hash identity survive aggregation?
4. The paper shows Irminsul's value is fixed by the workload's partition-shift structure, not model scale (DSv2-Lite and JoyAI-48B recover similar fractions). Does this hold across task types beyond agentic metadata rotation — e.g., long-document QA or code generation with large shared libraries?
5. For MHA models, what is the minimum boundary-recompute overhead that would make a carve-out + EPIC-style path competitive with full recompute? Is there an architecture-agnostic variant of the first-chunk carve-out?
6. The S2 boundary marker costs 64 tokens of context per shared region. At very high sharing fractions (e.g., 95%+ shared system prompt), does this overhead become non-negligible relative to the savings?
7. How does the approach interact with speculative decoding, where draft-model prefill and target-model verification have different KV structures?
