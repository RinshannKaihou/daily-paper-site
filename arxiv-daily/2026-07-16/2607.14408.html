02 · Why this matters

Removing the labeled-reward requirement cuts annotation and reward-engineering cost for self-evolving agents — via a drop-in accept_or_reject(parent, child) callback.

Cost anchor: a few lines per engine (GEPA, ADRS, ShinkaEvolve). All other engine hyperparameters stay at published defaults.

03 · Critical edge

On code substrates the winning variant is Direct + full reward — so code evolution is not actually reward-free end-to-end.

§ Mechanics & evidence

M Methodology

Redesigns two of three loop steps — Propose → Accept/Reject → Select; mutator stays default. The scalar gate a = 1{r(f_c) > r(f_p)} becomes a = 1{V_φ(f_c, f_p, E) = better}, where Vφ is a frozen LLM under greedy decoding.

  • Adaptive Focus — swaps only the gate; rubric (Accuracy / Soundness / Holistic) refreshed every 10 iters by a meta-LLM. Train-reward-free.
  • Soft Elo — also replaces parent selection with confidence-weighted Elo (init 1600, k=32): R_c ← R_c + k·c_t·(S − Ŝ). Fully reward-free.
  • Integration — one callback inserted at each engine's acceptance point; published hyperparameters untouched.

§ Strengths ↔ Limitations

+ Strengths

  • Clean, minimal intervention — one reversible callback, no validator training, no param updates.
  • Honest scoping — surfaces its own counterexample (Gemma-4 PUPA −2.0) and flags saturation, small-N, capability gaps.
  • Unusual breadth — 3 engines (GEPA, ADRS, ShinkaEvolve) × prompt + code substrates at this scope.
  • Diagnostic ablation (Table 1) cleanly separates "what V sees" from "what feeds parent selection."
  • Generalization story tied to low-data val regime (|V| ∈ [12,30]) and val→test gap (Fig. 2).

! Limitations

  • Single-seed main tables (Tables 2–5) — only Appendix A is multi-seed; no error bars on most cells.
  • Small math sets — AIME n=15 (6.7%/problem), LB-Math n=30; +20.0 is single-problem granularity.
  • Fully reward-free not consistently best — Soft Elo lags on several rows (HotpotQA 59.3 vs 77.0; PUPA 64.6 vs 79.2).
  • Code-substrate winner keeps full reward for parent selection — not reward-free end-to-end.
  • No a-priori rule for when Vφ will be reliable on a given task.
  • Validator/meta-LLM prompts not verbatim; no public code/data release referenced.
§ Open questions
Multi-seed replication — would 3–5 seeds on Tables 2–5 preserve "matches or exceeds," or do within-noise cells (txn_scheduling, signal_processing, several math rows) regress to parity?
Genuinely reward-free code parent selection — can a rule beyond Direct (which underperforms) match Direct + full reward, or is the task evaluator structurally necessary?
Compute/cost trade-off — Soft Elo runs 4·|T_t| validator matches/iter plus meta-LLM rubric refresh; what is the end-to-end cost vs full-reward baseline?
Predictive reliability signal — is there a marker (task type, output length, validator self-agreement) that flags when Vφ will be reliable, forestalling the Gemma-4 PUPA regime?
Capability sweep — how does the gate interact with frontier closed models or sub-8B open models where the agent may lack room to improve?
Gate vs. soft objective blend — a head-to-head against a multi-objective Pareto (validator score alongside task reward) would settle whether the binary gate is the right framing.
