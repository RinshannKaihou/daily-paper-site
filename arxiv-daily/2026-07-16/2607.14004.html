Do Agent Optimizers Compound? A Continual-Learning Evaluation on Terminal-Bench 2.0

RELAI-VCL lifelong average on 22 hard Terminal-Bench 2.0 tasks — the only method leading at all four evaluated stages.
+17.7 pts vs baseline (58.7%) · +10.4 pts vs best rival
02 Why this matters

Directly answers whether repeatedly re-optimizing an agent harness erases prior gains — a core concern for deployable, self-improving agents.

03 Critical edge

Confounded win. RELAI-VCL searches the broadest space (prompt · tool · workflow · memory · skill · code) under the same 200-rollout budget as prompt-only GEPA and code-only Meta Harness — a larger hypothesis class, not a clean test of regression control.

Worse: N = 22 tasks × 2 trials, no variance or significance testing — several-point gaps sit inside plausible noise.

Vendor self-eval · product wins · GEPA-code dropped
§ 01

Methodology

A minimal protocol that decomposes a single benchmark score into three measurable properties — static strength, transfer, and re-optimization — starting from one shared GPT-5.5 baseline agent ("TerminusKira").

Phase 1 · static

Optimize on T1

200 rollouts against 12 hard tasks (900s timeout) → A1. The conventional static score.

Transfer

Eval on T1∪T2

A1 scored on all 22 tasks with no further optimization — generalization to tasks that never drove search.

Phase 2 · re-optimize

Optimize again on T1∪T2

Another 200 rollouts from A1 → A2. Can the method keep improving on top of its own earlier edits?

Single summary: LifelongAvg = (Phase-1 + Transfer + Final) / 3

§ 03

Strengths & Limitations

▲ Strengths

  • Protocol that decomposes a single score into static / transfer / re-optimization — useful regardless of which method wins.
  • Honest artifact forensics — inspects what each optimizer actually changed and ties qualitative diffs to the transfer/stall numbers.
  • Open release of all baseline, Phase-1, Phase-2 agents plus full prompt/code diffs for independent audit.
  • Conceptual scaffolding linking harness optimization to EWC, catastrophic forgetting, and Goodhart's-law overfitting.

▽ Limitations

  • Asymmetric search spaces favor RELAI-VCL (full harness) over prompt-only GEPA and code-only Meta Harness under matched budgets.
  • Tiny N — 22 tasks × 2 trials, no variance or significance; GEPA's stronger code variant was dropped after a Phase-1 failure.
  • Vendor self-evaluation with the vendor's product winning; no neutral third method with in-loop regression control tested.
  • RELAI-VCL under-specified — reimplementing without the artifact repo is not possible; only two phases studied.
§ 04

Questions for Further Exploration

  1. Would RELAI-VCL still win if GEPA and Meta Harness were given the same broad search space — prompt + tool + workflow + memory + code?
  2. How do rankings shift if the dropped GEPA-code variant is repaired, or a neutral third-party optimizer with in-loop regression control is added?
  3. Do the compounding patterns persist over 5+ phases or under distribution shift between phases?
  4. On correlated, same-domain tasks (closer to production failures), does regression control help as much — or do cross-task shortcuts change the picture?
  5. What is the run-to-run variance of these pass rates, and which point differences are actually significant at N = 22, 2 trials?
  6. Can "regression control = generalization filter" be isolated via an ablation that adds no-regression rejection directly to GEPA and Meta Harness?
