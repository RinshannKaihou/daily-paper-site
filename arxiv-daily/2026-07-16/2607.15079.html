01Methodology

Built on the open-source PI Agent harness. Three elements: a shared-constitution multi-agent team, offline-curated domain knowledge, and an append-only Graph of Trace.

TeamSix agents, one constitution

  • PI — sole user-facing agent; decomposes, delegates, routes deliverables.
  • Librarian — search/read-only inductive synthesis.
  • Experimentalist — design docs, controls, preregistered plan.
  • Engineer — writes / runs / reports code.
  • Writer — polishes documents.
  • Auditor — checks numerical / file / citation claims → confirmed · unverified · disputed.

A skill-router mediates the library; every agent runs a skills-first preflight before committing to a method.

AssetsKnowledge & skills, built offline

Knowledge base — 7,233 indexed items across 11 disciplinary groups (psychiatry 30%). DeepSeek-OCR → 1,500-char chunks (200 overlap) → BGE-M3 1024-d vectors → bge-reranker-v2-m3 cross-encoder rerank.

Skill library — 72 reusable skills across 7 domains, derived from DeepLabCut, MNE-Python, fMRIPrep, SpikeInterface, netneurotools, pycortex. Extensible via paper-to-skill / repo-to-skill meta-skills.

TraceGraph of Trace (GoT)

After each subtask the agent calls the GoT tool → a schema-validated node (description, artifacts, parent IDs). The graph is append-only: no retroactive edits, cycles, or orphans — a monotonic DAG rendered live so experts can intervene before errors propagate.

GoT · provenance DAG
EvaluationThree settings: (i) 3 brain-science tasks from Agents' Last Exam (ALE) — BrainPilot vs Codex vs Claude Code, 4 backbones × 2 knowledge conditions; (ii) BrainPilotBench-v0 (4 tasks, 7 harness×backbone configs); (iii) 5 end-to-end case studies. All ALE/Bench results are single runs graded against held-out references.

02Results & Highlights

Cost-efficiency and trustworthiness win; raw task SOTA is competitive, not dominant; the open-ended long-horizon task exposes a clear weakness.

ALE cost spans >2 orders of magnitude

Total cost $0.08 → $22.53 (280×). Matched-backbone, BrainPilot runs in 8–63% of the time at 5–56% of the cost.

ALE tasks: competitive, not uniformly superior

T1 skull-strip QC 1.00; T2 ROI resampling 0.70–1.00; T3 open-ended C. elegans tracking 0.00–0.18 — most backbones failed outright.

BrainPilotBench-v0: 53/56 valid cells

BrainPilot + claude-opus-4.8 ties top RSC 1.00, best BCI IV 2a 0.20 (Codex/Claude Code 0.00), edges Claude Code on TOPS-fMRI 0.47 vs 0.41; Sleep-EDF tied 0.65/0.66.

Domain knowledge is a double-edged sword

Across 26 valid cells: +8 / −14 / =4. More tool use did not translate to uniform gains — and remains undiagnosed.

RSC case reproduces spatial coding

Bayesian position decoding on held-out trials: MAE 16.8 cm, decoded correlation r = 0.646, localized place-field heatmaps.

fMRI pain signature generalizes to clinics

Held-out tonic-pain pooled r = 0.5791 (p = 2.76×10⁻¹⁰); chronic-back-pain AUC 0.793 (Japan), 0.699 (UK).

BCI motor imagery: positive trend, non-significant

Accuracy 0.580 → 0.620, κ 0.440 → 0.493, wins 7/9 subjects — but paired-t p = 0.107, Wilcoxon p = 0.129. Framed as trend, not SOTA.

Mouse visual hierarchy: convergent, non-significant

RF diameter vs hierarchy Spearman ρ = 0.714 (p = 0.058); latency ρ = 0.667 (p = 0.083). Directionally correct, none reaching significance.

CredibilitySixteen authors across strong neuroscience + AI institutions; fully open-source (system, KB pipeline, skills, benchmark) with a live deployment. The Auditor caught a real in-paper citation error (Harris 2019 cited for Siegle 2021 data) — refreshing honesty, but it confirms the Auditor checks in-session evidence only. Headline numbers rest on single runs.

03Strengths vs Limitations

Two honest columns. The system's value is cost + trust + breadth; its weakness is statistical thinness and a too-narrow benchmark and Auditor.

Strengths

  • Fully open-source across system, KB pipeline, skill library, and benchmark — rare for an agentic-research paper.
  • Provenance-first (Graph of Trace + Auditor) directly targets hallucination, drift, and intervention.
  • Rigorous benchmark: held-out graders, no-network containers, read-only mounts, Oracle + empty-submission gate, artifact-based scoring.
  • Intellectually honest reporting — explicit "F" failures and non-significant p-values surfaced, not buried.
  • Cross-subdomain breadth — calcium imaging, fMRI, EEG-MI, sleep, ephys, behavior.

Limitations

  • Single-run evaluations on a stochastic backbone — no seed sweeps, no CIs, no statistical cost/score comparison.
  • BrainPilotBench-v0 has only 4 tasks and is explicitly "preliminary."
  • No multimodal / cross-modal integration — every case study is single-modality, despite the cross-scale motivation.
  • KB degrades 14/26 scores with no diagnosed cause or mitigation.
  • Auditor scope is narrow — in-session evidence only, not external citations or methodological soundness.
  • Specialists not composable into sub-workflows; cost figures depend on 16 Jul 2026 list pricing.
  • No comparison to dedicated single-task pipelines — unclear how close to domain-specialist SOTA.

04Questions for Further Exploration

Where the work pushes the field next — and the sharpest open threads a follow-up should close.

Why does the curated KB reduce 14/26 scores — retrieval noise, context pollution, or skill-router misrouting? Would a retrieval-quality ablation or per-skill applicability gate fix it?

Do ALE / Bench numbers hold under repeated seeds (e.g. n = 5) with mean ± std? Does the $0.08 vs $22.53 gap survive once variance is accounted for?

Can the Auditor be extended to actually verify external citations against retrieved source text (the Siegle/Harris slip shows in-session-only checking is insufficient)?

What happens when specialists can spawn sub-workflows / sub-agents — does quality rise enough to justify the added orchestration cost?

How does BrainPilot perform on genuinely cross-modal questions (Neuropixels spiking ↔ fMRI connectivity), where the paper's own motivation says the hardest problems live?

Is the "open-source backbone is competitive" claim robust once glm-5.2's underperformance across all 4 Bench tasks is excluded?
