> **一句话总结：** BrainPilot 是一个完全开源的脑科学多智能体研究系统，由 PI 智能体协调 5 个专家智能体 + 1 个 Auditor，借助 7,233 条知识的统一知识库与 72 个可复用技能，通过 Graph of Trace 提供可审计的工作流，在 Agents' Last Exam 与自建 BrainPilotBench-v0 上以开源模型达到接近 SOTA 代理框架的效果且成本大幅更低。

> **TL;DR:** BrainPilot is a fully open-source, human-in-the-loop multi-agent system for brain-science research: a PI agent coordinates five specialist agents plus an Auditor, grounded in a 7,233-item knowledge base and a 72-skill library, with all steps recorded in an append-only Graph of Trace. On Agents' Last Exam (ALE) it matches or beats Codex/Claude Code at 5-56% of their cost (e.g., $0.08 vs $22.53 for the cheapest-vs-priciest matched config on three neuroscience tasks), and on its new BrainPilotBench-v0 (4 tasks: RSC, TOPS-fMRI, BCI IV 2a, Sleep-EDF) BrainPilot+claude-opus-4.8 ties the top RSC score (1.00), posts the best BCI IV 2a (0.20), and edges Claude Code on TOPS-fMRI (0.47 vs 0.41), while five end-to-end case studies reproduce published neuroscience findings (e.g., RSC decoding MAE 16.8 cm, r = 0.646).

## 核心贡献

- **领域定制的多智能体架构：** 首次为脑科学研究构建 PI + Librarian/Experimentalist/Engineer/Writer/Auditor 六智能体团队，PI 负责任务分解与协调，Auditor 独立校验数字、文件与引用声明，将"防幻觉"嵌入工作流本身而非事后检查。
- **可审计的 Graph of Trace：** 每个主要步骤以 append-only、schema-validated 节点形式记录子目标、工具调用、证据与论断，自动推断因果依赖边形成 DAG，前端实时可视化，让研究者能在错误扩散前介入审查。
- **统一知识资产：** 离线构建包含 7,233 条索引项（11 个学科组，psychiatry 占 30.0%）的检索知识库（BGE-M3 + reranker），以及覆盖 7 大研究域、72 个技能的技能库（含 DeepLabCut、MNE-Python、fMRIPrep、SpikeInterface 等工具派生技能），并提供 paper-to-skill / repo-to-skill 元技能用于持续扩展。
- **领域专属基准 BrainPilotBench-v0：** 针对 calcium imaging（RSC）、功能连接 fMRI（TOPS-fMRI）、运动想象 EEG（BCI Competition IV 2a）、睡眠分期（Sleep-EDF）四类任务，采用"只评交付物、隔离容器执行、Oracle 双侧验证门"的严谨评分范式。
- **端到端真实科研验证：** 5 个完整案例研究（RSC 空间编码、小鼠视觉系统功能层级、fMRI 疼痛解码、EEG 运动想象解码、睡眠分期）展示了从"一句研究目标"到"可复现分析管线 + 科学报告"的全流程能力。

## Key Contributions

- A domain-specific multi-agent system (PI + Librarian + Experimentalist + Engineer + Writer + Auditor) purpose-built for brain-science workflows, with the Auditor integrated as a first-class fabrication checker rather than a post-hoc afterthought.
- The **Graph of Trace (GoT)**: an append-only, schema-validated, DAG-structured provenance record that links subgoals, tool calls, evidence, and claims, rendered live so experts can intervene before errors propagate.
- Two curated, offline-built domain assets: a unified 7,233-item RAG knowledge base (spanning 11 disciplinary groups, built with DeepSeek-OCR + BGE-M3 embeddings + bge-reranker-v2-m3) and a 72-skill library across 7 research domains, extensible via paper-to-skill / repo-to-skill / workflow-skill-creator meta-skills.
- **BrainPilotBench-v0**, a new artifact-scored benchmark with four neuroscience tasks (RSC place-cell analysis, TOPS-fMRI pain signature, BCI Competition IV 2a, Sleep-EDF), using no-network containers, read-only mounts, held-out graders, and a two-sided Oracle/empty-submission validation gate.
- Five end-to-end real-world case studies demonstrating that a single high-level research prompt can be converted into a reproducible multi-stage analysis pipeline with numerical results, figures, and a scientific report.

## Methodology

BrainPilot is built on the open-source PI Agent harness and combines three elements:

1. **Multi-agent team with a shared constitution.** Each agent is defined by a shared system constitution, a role definition (mission, working style, tools), and a set of skills. The **PI** is the only agent the user talks to; it decomposes the goal, delegates to specialists, and routes deliverables through Writer then Auditor. The **Librarian** does inductive synthesis via search/read tools only. The **Experimentalist** produces design documents (operationalization, controls, sample planning, preregistered analysis plan). The **Engineer** writes/runs/reports code with file and shell tools. The **Writer** polishes documents. The **Auditor** independently checks numerical, file, and citation claims against session evidence, classifying each as confirmed / unverified / disputed, and writes a timestamped audit report. A skill-router mediates access to the skill library; every agent runs a "skills-first preflight" before committing to a nontrivial method.

2. **Curated domain knowledge, built offline and exposed as services.** The knowledge base ingests open-source neuroscience textbooks and papers via DeepSeek-OCR, splits into 1,500-char chunks (200-char overlap), embeds with BAAI/bge-m3 into 1,024-dim vectors, and retrieves with a two-stage vector-similarity + cross-encoder rerank pipeline (bge-reranker-v2-m3). The skill library holds 72 reusable units (descriptive or executable) across 7 domains, drawn from three pipelines: LLM-authored, paper-derived, and tool-derived (DeepLabCut, MNE-Python, fMRIPrep, netneurotools, pycortex, SpikeInterface).

3. **Graph of Trace.** After each subtask, the agent calls the GoT tool, which converts its report into a schema-validated node (description, artifact references, parent IDs) and infers dependency edges. The graph is append-only (no retroactive edits, cycles, or orphaned references), growing monotonically into a DAG that a frontend renders in real time.

Evaluation spans three settings: (i) three brain-science tasks from **Agents' Last Exam (ALE)** under BrainPilot vs Codex vs Claude Code, with four backbones (deepseek-v4-pro, glm-5.2, gpt-5.5, claude-opus-4.8) and two knowledge conditions; (ii) **BrainPilotBench-v0** (4 tasks, 7 harness×backbone configs, 2 knowledge conditions); and (iii) five end-to-end case studies. All ALE/Bench results are single runs graded against held-out references.

## Results & Highlights

- **ALE cost dominates by >2 orders of magnitude.** Across the three brain-science tasks (T1 skull-stripping QC, T2 ROI resampling, T3 C. elegans neuron tracking), total cost ranged from **$0.08 for BrainPilot + deepseek-v4-pro** to **$22.53 for Codex + gpt-5.5** — a >280× spread. Matched-backbone, BrainPilot finished in **8-63% of the time and at 5-56% of the cost** of Codex/Claude Code.
- **ALE task-level performance was competitive, not uniformly superior.** On T1 (full-credit threshold 1.00) BrainPilot scored 1.00 with deepseek-v4-pro, glm-5.2, gpt-5.5, and claude-opus-4.8; on T2 (partial-credit ceiling 0.70) all harnesses hit 0.70-1.00; on the open-ended T3 (no pass threshold, ALE score) BrainPilot reached only 0.00-0.18 with most backbones failing outright ("F"), exposing a clear weakness on long-horizon open-ended tracking.
- **BrainPilotBench-v0: 53/56 cells produced valid scores.** With domain knowledge enabled, BrainPilot + claude-opus-4.8 tied the highest **RSC = 1.00**, achieved the highest **BCI IV 2a = 0.20** (vs Codex's 0.00 and Claude Code's 0.00-0.11), and beat the matched Claude Code on **TOPS-fMRI (0.47 vs 0.41)**; Sleep-EDF was essentially tied (0.65 vs 0.66).
- **Domain knowledge was a double-edged sword.** Across the 26 BrainPilotBench cells with valid scores in both conditions, enabling the knowledge base **increased 8 scores, reduced 14, and left 4 unchanged** — greater tool use did not translate to uniform gains.
- **RSC case study reproduced known spatial coding.** Bayesian position decoding on held-out even trials (cells ordered by odd-trial tuning) reached **MAE 16.8 cm** and **true-decoded correlation r = 0.646**, with localized place-field-like heatmaps and positive trial-bin correlations.
- **fMRI pain signature generalized partially to clinical cohorts.** Held-out tonic-pain prediction pooled Pearson **r = 0.5791 (p = 2.76×10⁻¹⁰)**, mean within-subject r = 0.6754; chronic-back-pain classification reached **AUC 0.793 (Japan)** and **AUC 0.699 (UK)**, with the SBP spontaneous-pain condition significant at r = 0.333 (p = 0.0048).
- **BCI motor-imagery case study beat EEGNet but non-significantly.** MIAgentModel (EEGNet backbone + TCN, ~28.7k params) raised mean accuracy from **0.580 → 0.620** and Cohen's kappa from **0.440 → 0.493** across 9 subjects × 3 seeds, winning 7/9 subjects — but paired t-test p = 0.107 and Wilcoxon p = 0.129, so the authors frame it as a positive trend, not a SOTA claim.
- **Sleep-EDF staging approached DeepSleepNet.** 20-fold LOSO gave **accuracy 0.794, macro-F1 0.725, balanced accuracy 0.724, Cohen's kappa 0.716**, with N1 the hardest class (F1 = 0.339) and all metrics slightly below DeepSleepNet references.
- **Mouse visual hierarchy: convergent but non-significant trends.** Receptive-field diameter showed the strongest correlation with anatomical hierarchy (Spearman **ρ = 0.714, exact permutation p = 0.058**), response latency ρ = 0.667 (p = 0.083), intrinsic timescale ρ = 0.476 (p = 0.243) — all positive and directionally correct, none reaching conventional significance.

## Critical Assessment

- **Novelty:** Genuine — the combination of (a) a domain-curated 7,233-item knowledge base + 72-skill library specifically for neuroscience, (b) an integrated Auditor agent that classifies numerical/file/citation claims against session evidence, and (c) an append-only Graph of Trace is not something existing brain-science agents (EthoClaw, the multimodal-neuroimaging agents, the BCI multi-agents of Zhou et al., auto-psych) offer. The closest comparators (AI Scientist, Co-Scientist, Biomni) target chemistry/biomedicine/general ML, not brain science with provenance-first design.
- **Rigor:** Honest but thin on statistics. Headline ALE and BrainPilotBench numbers are **single unrepeated runs** on a reasoning backbone with known run-to-run variation; the authors flag this but did not repeat. Several case-study "improvements" are explicitly non-significant (BCI paired t-test p = 0.107; all three mouse-visual-hierarchy correlations p > 0.05), and the paper commendably says so rather than overselling.
- **Reproducibility:** Strong on artifacts and weak on protocol detail. Code, knowledge base, skills, and benchmark are open-sourced; graders are held-out and Oracle-gated; submissions run in no-network, non-root containers. But the knowledge-base corpus itself is described only by counts (7,233 items, 11 groups) without a published item list, and the 72 skills are summarized without per-skill provenance/scope.
- **Significance:** Real for the neuro-AI tooling community but modest on raw task SOTA — BrainPilot's selling point is cost-efficiency and trustworthiness, not beating the best single-task specialist. The clearest win (ALE cost $0.08 vs $22.53) is partly a backbone-pricing artifact (deepseek is cheap) rather than a pure architecture win.
- **Red flags / specific concerns:**
  - **Domain knowledge frequently hurt.** Enabling the KB *reduced 14 of 26* BrainPilotBench scores and the authors do not diagnose why — a missing ablation on retrieval quality / context pollution.
  - **Auditor caught a real citation error in the authors' own case study:** the agent cited Harris et al. 2019 for anatomical hierarchy scores that were actually from Siegle et al. 2021 — the paper openly notes this "underscoring the need for human verification," which is refreshing but also shows the Auditor does not actually verify external citations against source text.
  - **Only 3 ALE tasks are neuroscience** out of >1,000, so ALE-based claims about "state-of-the-art agent framework" parity rest on a tiny slice.
  - **BrainPilot + glm-5.2 underperformed its matched Claude Code alternative across all 4 BrainPilotBench tasks**, undercutting the "open-source backbone is competitive" framing for that particular open model.
  - **Case-study pipelines use custom metric definitions** (e.g., first-sustained-threshold latency instead of median time-to-first-spike) that the authors themselves note diverge from prior work, making direct numerical comparison to published numbers hazardous.

## Strengths

- Fully open-source across system, knowledge base pipeline, skill library, and benchmark — rare for an agentic-research paper and enabling community extension.
- Provenance-first design (Graph of Trace + Auditor) directly tackles the hallucination/drift/intervention problems the paper identifies as field-wide blockers.
- Benchmark design is unusually rigorous for an agent paper: held-out graders, no-network containers, read-only model mounts, Oracle + empty-submission two-sided validation gate, and artifact-based (not self-reported) scoring.
- Intellectually honest reporting — explicit "F" failures, non-significant p-values surfaced rather than buried, and an in-paper acknowledgment of a citation mistake the system made.
- Breadth: covers calcium imaging, fMRI, EEG (motor imagery + sleep), electrophysiology, and behavior across case studies, demonstrating genuine cross-subdomain reach.

## Limitations

- **Single-run evaluations** on a stochastic reasoning backbone; no seed sweeps, no confidence intervals, no statistical comparison of cost/score trade-offs.
- **BrainPilotBench-v0 has only 4 tasks** and is explicitly called "preliminary"; broad subdomain coverage is future work.
- **Multimodal/cross-modal integration is absent** — every case study operates within a single modality, yet the paper's own motivation stresses cross-scale, cross-modality integration.
- **Domain knowledge sometimes degrades performance** (14/26 cells) without a diagnosed cause or mitigation.
- **Auditor scope is narrow** — it checks in-session evidence, not external citation correctness (as the Siegle/Harris slip demonstrates) nor broader methodological/statistical soundness.
- **Specialist agents are not composable** into sub-workflows yet; demanding subtasks (deep literature review, manuscript drafting) are handled by a single agent rather than a sub-orchestration.
- **Cost numbers depend on list pricing as of 16 July 2026** and on the open-ended T3 task dominating token use; cheaper backbones partly reflect pricing, not pure efficiency.
- **No comparison to dedicated single-task neuroscience pipelines** (e.g., a hand-written DeepSleepNet reproduction in the same codebase is listed as a needed future check), so it is unclear how close BrainPilot gets to domain-specialist SOTA rather than to other general agents.

## Relevance

For research on building trustworthy, auditable LLM agent systems for science, BrainPilot is a directly relevant reference architecture: its Graph of Trace (append-only provenance DAG), integrated Auditor fabrication-checker, and skill-router preflight pattern are transferable design primitives for any agent that must remain inspectable and human-in-the-loop. It is also a concrete data point on the open problem of evaluating long-horizon scientific agents with artifact-based, held-out scoring.

## Questions for Further Exploration

- Why does enabling the curated knowledge base *reduce* 14/26 BrainPilotBench scores — is it retrieval noise, context pollution, or skill-router misrouting, and would a retrieval-quality ablation or a per-skill applicability gate fix it?
- How do BrainPilot's ALE/Bench numbers hold up under repeated seeds (e.g., n=5) with mean ± std, and does the $0.08 vs $22.53 cost gap survive once variance is accounted for?
- Can the Auditor be extended to actually verify external citations against retrieved source text (the Siegle/Harris slip shows the current in-session-only check is insufficient)?
- What happens when specialist agents are allowed to spawn sub-workflows/sub-agents (e.g., a deep-research pipeline inside the Librarian) — does quality rise enough to justify the added orchestration cost?
- How does BrainPilot perform on genuinely cross-modal questions (e.g., aligning Neuropixels spiking with fMRI functional connectivity) where the paper's own motivation says the hardest problems live?
- How sensitive are the results to the specific open-source backbone (deepseek-v4-pro) vs proprietary ones, and is the "open-source backbone is competitive" claim robust once glm-5.2's underperformance is excluded?
