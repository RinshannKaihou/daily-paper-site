> **一句话总结：** 作者在扰动平滑模型下证明,只要 NP 不含于 BPP,就不存在对所有对抗基础实例期望多项式时间的精确 ReLU 网络验证器;通过四-ReLU-每子句的 E3SAT 精确嵌入,搭配加权敏感性不等式与高斯集中,固定噪声 σ⋆=2⁻¹¹ 下标签保持概率 ≥ 1−e⁻ᵐ⁄⁸。

> **TL;DR:** In a perturb-and-round smoothed model where every ReLU network weight/bias is independently hit by clipped, dyadic-rounded Gaussian noise, the author proves (under NP ⊄ BPP) that no sound-and-complete exact verifier can run in expected polynomial time on every adversarial base instance. The result already holds for one-hidden-layer networks at the fixed noise σ⋆ = 2⁻¹¹, using a four-ReLU-per-clause E3SAT embedding whose exact optimum is (m − unsat(Φ))/3; a weighted sensitivity bound plus Gaussian concentration preserves the safe/unsafe label with probability ≥ 1 − e⁻ᵐ⁄⁸, a gap that is exponentially small (delta in preservation probability vs. the worst-case reduction's e⁻⁸ ≈ 0.0004 failure at m = 64) and yields NP ⊆ BPP by Markov truncation.

## 核心贡献

- **平滑模型下的固定噪声下界**:在每权重/偏置独立地按截断到 [−2,2]、按输入位复杂度的二进网格精确舍入的高斯噪声扰动的"扰动-舍入"模型下,假设 NP ⊄ BPP,则不存在在 s、B、1/σ 上期望多项式时间的健全且完备的精确 ReLU 验证器。结论在固定噪声 σ⋆ = 2⁻¹¹ 即成立。
- **精确的 E3SAT 连续嵌入(四 ReLU/子句)**:对任意 m 子句的 E3SAT 公式 Φ,构造单隐层网络 gΦ,满足恒等式 max_{x∈[0,1]ⁿ} gΦ(x) = (m − u⋆(Φ))/3,其中 u⋆ 是最少未满足子句数;且对每个分数点,其按坐标阈值舍入的布尔点目标值不降低,故积分性间隙为零。
- **加权平滑-传递定理(可复用)**:证明了对所有参数的逐点影响系数加权的确定性上界 sup_x |g_{θ′}(x) − g_θ(x)| ≤ Σ_q c_q|θ′_q − θ_q_q|,并结合高斯集中得到显式的标签保持概率界 exp(−Λ²/(2D²));该定理不依赖 E3SAT 嵌入,可移植到任何具有大间隙和线性加权敏感性的有界系数间隙归约。
- **位模型下的 BPP 坍塌**:给出了舍入高斯分布的多项式位采样器(支撑大小 2^{B+2}+1 = poly(s)),并将假设的期望运行时间上界经马尔可夫截断转化为 BPP 的 SAT 算法,从而推出 NP ⊆ BPP。
- **计算审计与可复现性**:用独立 MILP 全局优化、三维激活排列穷举、加权证书缩放实验对实现敏感的恒等式与证明机制做了诊断性验证;提供完整 LaTeX 源、固定随机种子、原始 CSV 与脚本。

## Key Contributions

- **Fixed-noise smoothed-complexity lower bound.** In the perturb-and-round model (q̃ = clip_{[−2,2]} 2⁻ᐩ round(2ᐩ(q + σZ))), under the standard assumption NP ⊄ BPP, no sound-and-complete exact ReLU verifier has expected bit-running time polynomial in network size s, bit-complexity B, and 1/σ for every adversarial base instance. The implication holds already at σ⋆ = 2⁻¹¹.
- **Exact four-ReLU-per-clause E3SAT embedding.** A one-hidden-layer network with hidden fan-in ≤ 3 satisfies the exact identity max_{x∈[0,1]ⁿ} gΦ(x) = (m − u⋆(Φ))/3; coordinatewise threshold rounding never decreases the objective, so there is zero integrality gap and every maximizer has a Boolean representative.
- **Reusable weighted smoothed-transfer theorem.** A deterministic weighted-sensitivity inequality sup_x |g_{θ′}(x) − g_θ(x)| ≤ Σ_q c_q|θ′_q − θ_q| combined with Gaussian concentration for a folded-normal weighted sum gives an explicit label-preservation bound exp(−Λ²/(2D²)). The theorem is agnostic to the source gadget (it also instantiates on a two-ReLU-per-edge MAX-CUT construction).
- **Quantitative fixed-noise instantiation + BPP collapse.** With Γ = m/60, σ⋆ = 2⁻¹¹, the preservation probability is ≥ 1 − e⁻ᵐ⁄⁸ (≥ 1 − e⁻⁸ ≈ 0.9996 since m ≥ 64). A polynomial-bit rounded-Gaussian sampler plus Markov truncation converts a hypothesized expected-time bound into a BPP SAT algorithm, yielding NP ⊆ BPP.
- **Honest, audited reproducibility.** Computational checks test the exact identity, globally optimized perturbed 3-input networks via activation-arrangement enumeration (cross-checked against MILP), and the extensive-vs-constant gap scaling; all raw data, fixed seeds, and scripts are released. The paper explicitly states these are diagnostics, not evidence for the complexity theorem.

## Methodology

The proof is a **quantitative robustification of gap hardness**, deliberately not a worst-case reduction pasted onto a smoothed model. The pipeline is:

1. **Source gap problem.** Use Håstad's optimal E3SAT inapproximability with ε = 1/40, giving an NP-hard promise of satisfiable vs. at-most-9/10-satisfiable; pad with 64 disjoint copies so m ≥ 64 and the unsatisfied fraction η = 1/10.
2. **Exact ReLU encoding (four ReLUs per clause).** Each literal occurrence becomes a literal unit ReLU(ℓ_λ(x) − 1/2) with output weight 2/3; each clause gets an overflow unit ReLU(L_C(x) − 1/3) with output weight −1. Hidden fan-in is ≤ 3 and all base coefficients lie in [−1, 1]. A precision guard ReLU(−1 + 2⁻²⁰x₀) forces B ≥ 20 so that 2⁻ᐩ ≤ σ⋆ is admissible.
3. **Exact integrality theorem.** An elementary identity on ReLU(ℓ − 1/2) plus min{L_C, 1/3} telescoping proves gΦ(x) ≤ gΦ(a) for the threshold-rounded Boolean point a, hence the closed-form optimum (m − u⋆)/3. Setting τ = 19m/60 yields a symmetric gap Γ = m/60 around the unchanged threshold.
4. **Weighted sensitivity certificate.** Define per-slot influence coefficients c_q (2 for hidden weights/biases, 1/2 for literal output weights, 2/3 for overflow output weights, 0 for the inactive guard, 1 for the output bias) and prove sup_x |g_{θ′}(x) − g_θ(x)| ≤ Σ_q c_q|θ′_q − θ_q| without fixing an activation pattern, valid even at preactivation zero. Compact accounting gives C = Σ c_q ≤ 23m and D² = Σ c_q² ≤ 43m.
5. **Gaussian transfer.** Because nearest-grid rounding adds at most h/2 and clipping to [−2,2] is nonexpansive relative to base coefficients in [−1,1], the displacement is bounded by σF(Z) + hC/2. F is D-Lipschitz with mean µC; Gaussian concentration bounds the exceedance, and label change requires reaching Γ, giving failure ≤ exp(−Λ²/(2D²)). Plugging in σ⋆ = 2⁻¹¹ yields Λ ≥ 52m/15 and failure ≥ m/8 in the exponent.
6. **Bit-model BPP conversion.** A polynomial-bit dyadic sampler approximates each rounded-Gaussian marginal within TV ε/P (using polynomial-time computability of the normal CDF, Ko 1991), and the product coupling keeps the joint within ε. Running the hypothetical verifier for 100Q steps (Markov ≤ 1/100 timeout) and accepting on "unsafe" yields a BPP SAT algorithm with total error < 1/3 (e⁻⁸ + 1/100 + 1/100).
7. **Computational diagnostics.** 18 random formulas (72–188 ReLUs) cross-check the closed-form optimum against MILP; 3-input blocks are globally optimized by enumerating all C(38,3) = 8,436 activation-plane triples; weighted-certificate scaling is measured for extensive vs. one-defect gap families.

## Results & Highlights

- **Main theorem (conditional lower bound).** Under NP ⊄ BPP, there is no sound-and-complete exact verifier with expected bit-running time polynomial in s, B, and 1/σ⋆ at the fixed noise level **σ⋆ = 2⁻¹¹**, already for one-hidden-layer networks over [0,1]ⁿ⁺¹ with hidden fan-in ≤ 3, base coefficients in [−1,1], 4m+1 ReLUs, ≤ 14m+4 perturbed slots, s = Θ(m), and 20 ≤ B = O(log(m+2)). (Theorem 1.)
- **Exact optimum identity.** For every E3SAT formula with m clauses, max_{x∈[0,1]ⁿ} gΦ(x) = (m − u⋆(Φ))/3 (Theorem 2). The closed-form prediction agreed with independent global MILP optima on 18 random formulas to maximum absolute error **7.11 × 10⁻¹⁵**.
- **Exponential label preservation.** With Γ = m/60 and σ⋆ = 2⁻¹¹, the perturbation changes the exact safe/unsafe answer with probability ≤ e⁻ᵐ⁄⁸; because m ≥ 64, preservation exceeds **1 − e⁻⁸ ≈ 0.9996** (Corollary 6).
- **3-input exhaustive laboratory cross-check.** Across 300 globally optimized blocks per family at each of 10 noise levels (0.0015–0.063, B = 16), 44 independently selected optima were cross-checked against a separate MILP model with maximum discrepancy **7.99 × 10⁻¹⁵** and mean **3.63 × 10⁻¹⁶**. At σ = 0.042, balanced preservation rose from **0.695** for one block to **0.991** for 128 blocks (4,096 ReLUs, 14,337 perturbed slots).
- **Certificate scaling validates the mechanism.** Extensive-gap family (Γ = q/6) crosses 50% certificate probability at essentially constant noise (σ from 1.154 × 10⁻³ at q = 1 to 1.132 × 10⁻³ at q = 128, fitted log–log slope **−0.0035**); the one-defect ablation (Γ = 1/6) falls from 1.154 × 10⁻³ to 8.49 × 10⁻⁶ with slope **−1.014**, matching the predicted q⁻¹ law and confirming that the extensive margin is what enables fixed noise.
- **No quantitative claim about practical verifiers.** The paper explicitly states the computational study is a diagnostic, not a benchmark of vision-scale verifiers, and not evidence that typical trained networks are hard after perturbation.

## Critical Assessment

- **Novelty.** The novelty is genuine but narrow. Worst-case NP-hardness of ReLU reachability/verification for shallow networks already exists (Sälzer & Lange 2021; Froese et al. 2025, 2026). The contribution here is the **smoothed** (i.e., adversarial-base + average-over-perturbation) lower bound at a **fixed, size-independent noise level**, which is a strictly harder statement because a hard family may become easy after perturbation. The reusable weighted-transfer theorem and the explicit handling of clipping, exact dyadic rounding, output-layer perturbations, and bit-model sampling under one roof is the real technical value.
- **Rigor.** The argument is unusually careful for a smoothed-hardness claim. It does **not** assume activation independence, does not replace the aggregate perturbation by a single-coordinate anti-concentration bound, explicitly charges clipping and rounding (eq. 16), perturbs and counts hidden-to-output weights and the output bias, and uses Markov truncation only after the timeout makes the event bounded. The proof-audit checklist addresses the standard failure modes directly. Constants are kept rational and conservative (the exact Gaussian mean would give exponent 0.2132…·m; the paper uses m/8).
- **Reproducibility.** Strong by theory-paper standards: full LaTeX source, fixed seeds, raw CSVs, figure scripts, and `proof_checks.py` released; MILP models solved to MIP gap 10⁻¹⁰; arrangement enumeration cross-checked against a second MILP model. A reader could rerun the diagnostic computations.
- **Significance.** The result rules out one specific hope — that generic parameter nondegeneracy yields a universal smoothed-polynomial guarantee for **exact** verification. It does **not** bound practical incomplete/relaxation-based verifiers, does not address typical trained networks, and does not preclude polynomial algorithms under structural restrictions (monotonicity, bounded unstable-neuron count, fixed input dimension, training-induced margins).
- **Red flags / paper-specific concerns.**
  - **Conditional on NP ⊄ BPP.** A standard assumption, but the theorem is not unconditional.
  - **Only the absolute-noise model is covered.** Input box and threshold are unperturbed; a relative-noise model or one that perturbs the input box could behave differently, and the paper does not address this.
  - **Conservative constants.** σ⋆ = 2⁻¹¹ is extremely small relative to the [−1,1] coefficient range; the paper calls it "conservative and small … rather than optimized." The result would be more compelling with a larger admissible noise level, though the mechanism (extensive margin) makes small fixed noise structurally inevitable.
  - **Thin related-work on the verifier side.** Modern branch-and-bound verifiers (β-CROWN, GNN-verified, VNN-COMP 2025) are cited but the construction never engages with whether such methods would actually behave hard on these gadgets; the separation is purely asymptotic complexity.
  - **Heavy LLM disclosure** without detailing which proof steps were LLM-generated vs. hand-verified; the audit checklist partially mitigates this, but a reviewer might want line-level provenance.
  - **Single-author, technical-report framing** (arXiv only as of July 2026); the result has not yet passed formal peer review.

## Strengths

- Produces a **clean, conditional, fixed-noise smoothed lower bound** with explicit constants — exactly the kind of statement that is hard to get right.
- The **weighted-transfer theorem is reusable**: it cleanly separates "source gap reduction" from "noise robustness accounting" and even instantiates on a MAX-CUT route.
- The **exact integrality identity** (max = (m − u⋆)/3) is elegant and is independently audited against MILP to 10⁻¹⁵ precision.
- The paper is **intellectually honest** about scope: it repeatedly states the computation is diagnostic, not evidence; and it explicitly lists what the theorem does *not* claim.
- Strong **reproducibility package** for a theory paper.

## Limitations

- **Absolute-noise only;** input box and threshold unperturbed. The model is one specific perturb-and-round scheme.
- **Conditional on NP ⊄ BPP;** no unconditional lower bound.
- **Conservative σ⋆ = 2⁻¹¹** relative to the [−1,1] coefficient range; not optimized.
- **Worst-case only.** Says nothing about typical trained networks or the empirical runtime of modern verifiers on real benchmarks.
- **Single-hidden-layer, fan-in ≤ 3 gadgets;** while the transfer theorem is general, the explicit constants and the BPP collapse are demonstrated only on this family.
- **Computational study uses separable low-dimensional products,** so every block can be globally optimized — convenient for auditing but far from vision-scale verification.
- **Single author, no peer review yet.**

## Relevance

对研究神经网络安全形式化验证、ReLU 网络的复杂度与平滑分析方向高度相关:它从理论上划清了"加参数噪声并不会让精确验证变平凡"的边界,对评估 SMT/MILP/分支定界类完备验证器的最坏情形可扩展性具有指导意义。若你的兴趣集中在经验性鲁棒性验证工具(如 β-CROWN、VNN-COMP)而非复杂度下界,则该结论的间接价值大于直接价值——它不预测这些工具在真实网络上的运行时,但提示"噪声去退化"不能作为通用多项式保证的论据。

## Questions for Further Exploration

- Can the admissible fixed noise be pushed meaningfully above 2⁻¹¹ by tightening the rational bounds (e.g., using the exact Gaussian mean giving exponent 0.2132·m instead of m/8), or by a different gap source with a larger per-clause margin?
- Does the negative result survive a **relative** noise model (σ scaling with coefficient magnitude), or one that also perturbs the input box and threshold?
- Can a matching (or contradicting) **positive** smoothed result be shown for *incomplete* or relaxation-based verifiers, which the theorem does not cover?
- Is there an analogous smoothed lower bound for **deeper** networks, or does depth change the sensitivity accounting enough to alter the conclusion?
- The MAX-CUT two-ReLU-per-edge route (Remark 5) is sketched but not instantiated with full constants — does it yield the same σ⋆ or a better one?
- Could the construction be turned into an **actual hard benchmark family** for VNN-COMP to empirically stress-test modern complete verifiers, beyond the 3-input diagnostic blocks?
- How sensitive is the BPP-collapse step to the chosen timeout (100Q) and TV error (ε = 1/100) — is there slack to tighten the constants further?
