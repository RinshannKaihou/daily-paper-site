> **一句话总结：** 提出AutoSynthesis——一个基于LangGraph多智能体的端到端自动化元分析系统，从自然语言研究问题出发完成文献检索、全文筛选、统计量提取、Hedges' g效应量计算、随机效应元分析及发表偏倚/偏倚风险评估，在LLM说服力基准(Hölbling 2025)上得到g=0.143 vs 专家g=0.020(Δg≈0.12)，研究召回率71.4%（校正后85.7%），单次运行约0.5小时、1.5美元。

> **TL;DR:** AutoSynthesis is an end-to-end multi-agent framework (LangGraph + gpt-5.4-mini) that automates the full meta-analysis pipeline from a natural-language research question to a PRISMA-compliant report. Benchmarked against the Hölbling et al. (2025) human meta-analysis on LLM persuasion, it recovers a pooled Hedges' g of 0.143 versus the expert g = 0.020 (Δg = 0.123, within the ±0.20 tolerance), matches 5/7 benchmark studies (71.4% recall), and produces forest plots, I²/τ², funnel plots, leave-one-out, and RoB judgments at ~$1.5 and ~0.5 hours per run.

## 核心贡献

- 首个真正端到端的自动化元分析框架：AutoSynthesis 将文献检索、全文获取与解析、全文资格评估、研究/结局识别、统计量抽取与校验、效应量计算、随机效应元分析、发表偏倚诊断与风险偏倚评估、PRISMA 报告生成串联为单一多智能体工作流，弥补了既有 LLM 工作只覆盖检索/筛选或结构化抽取、但缺少"标准化效应量计算 + 池化合成"这一关键缺口的不足。
- 在专家级基准上完成定量验证：以 Hölbling et al. (2025) 关于 LLM 说服力的人类元分析为基准，定量比较检索召回率/精度、效应量级相关性、池化估计偏差，把"系统有效"建立在可验证的数字上而非演示叙事。
- 将 LLM 推理与确定性统计实现解耦：LLM 智能体负责文档理解与抽取，所有元分析统计（REML 随机效应模型、Egger 回归、trim-and-fill、leave-one-out、RoB 2/ROBINS-I）均以自定义 Python 实现，避免依赖 metafor/statsmodels 等库，保留方法学严谨度。
- 多智能体可审计设计：所有中间决策（筛选、资格判断、抽取值、效应量计算）写入共享 workflow state，提供完整 trace，支持人在回路复核与"living meta-analysis"持续更新。
- 模块化与可移植：基于 LangGraph ReAct 循环，支持插入不同 LLM 主干（含开源权重模型），并可在已有协议/证据基础上从中间阶段启动。

## Key Contributions

- First genuinely end-to-end automated meta-analysis: maps every PRISMA-aligned step—from natural-language research question to PRISMA 2020 flow diagram and forest plot—to specialized agents, explicitly closing the gap (standardized effect-size computation and pooled synthesis) that prior LLM evidence-synthesis systems left open.
- Quantitative benchmarking against an expert human meta-analysis (Hölbling et al. 2025 on LLM persuasion), reporting recall, precision, effect-size agreement, and pooled-effect deviation rather than only qualitative demonstration.
- Hybrid LLM-plus-deterministic architecture: LLM agents handle reasoning-heavy document understanding/extraction, while REML random-effects models, heterogeneity statistics (Q, I², τ²), Egger's test, trim-and-fill, leave-one-out, and RoB instruments are implemented from scratch in Python/NumPy/SciPy.
- Auditable, modular design: every intermediate decision is stored in shared workflow state for human-in-the-loop inspection; modular construction lets users resume from an existing protocol (supporting "living" meta-analyses).
- Backbone flexibility: prompts, agent definitions, and audit logs released to enable cross-LLM comparison, including open-weight backbones.

## Methodology

AutoSynthesis is built on LangGraph as a ReAct-style multi-agent loop. A planning agent converts the natural-language research question into a structured protocol (objective, treatment/control, eligibility criteria, outcomes, candidate moderators, search terms, expected effect direction); users can refine these via a human-in-the-loop step. The automatic search agent generates a Boolean query and pulls records from arXiv, Semantic Scholar, CrossRef, OSF Preprints, and PubMed APIs, then deduplicates by normalized title. A paper-reading agent fetches full text via a cascading parser (MinerU → LlamaParse → Mistral OCR → conventional PDF text). An eligibility agent applies full-text inclusion/exclusion criteria with logged reasons. A study-component identification agent decomposes multi-experiment papers into independent components, identifying treatment/control groups and outcome location.

The statistical extraction agent uses a two-stage design: first building a linked JSON representation of the paper (design, components, outcomes, cross-references), then extracting only the relevant statistics (sample sizes, test statistics, p-values, effect-size info). A statistical validation agent re-verifies every extracted value against the source text, marking unverifiable values as "hallucinated" and removing them; a relevance filter then keeps only the most relevant result per study. The analysis agent converts reported statistics (hypothesis tests, regression coefficients, odds ratios) into Cohen's d, then Hedges' g with small-sample correction, and fits random-effects REML models (initialized with DerSimonian–Laird τ²). Heterogeneity is summarized via Cochran's Q, I², H², and τ²; moderator analyses produce subgroup forest plots. The bias agent performs Egger's regression test, trim-and-fill, leave-one-out sensitivity, and funnel/sample-size diagnostic plots, and runs RoB 2 (randomized) or ROBINS-I (non-randomized) judgments. A report agent emits a PRISMA 2020 flow diagram, narrative Markdown report, and study-level overview table; all stage outputs are persisted as audit logs. Implementation uses Python 3.11 with gpt-5.4-mini via OpenRouter at reasoning_effort=medium.

Evaluation compares AutoSynthesis against Hölbling et al. (2025) (k = 11–12 effect sizes, N = 17,422). Intermediate steps are scored by overlap/precision/recall and confusion matrices; the final synthesis is scored by pooled Hedges' g, Pearson r, and OLS slope β between matched study-level effects.

## Results & Highlights

- Pooled effect agreement: On the LLM-persuasion research question, AutoSynthesis produced a pooled Hedges' g = 0.143 (95% CI [0.059, 0.226], p < 0.001), versus the Hölbling et al. (2025) human benchmark of g = 0.020, a deviation of Δg = 0.123 that lies within the ±0.20 tolerance region previously used for reanalysis agreement.
- Heterogeneity diagnostics match: AutoSynthesis I² = 88.3% vs. benchmark 75.97%; Egger's test p = 0.006 (AutoSynthesis) vs. p = 0.018 (benchmark), both flagging small-study effects and substantial heterogeneity.
- Study-retrieval overlap: AutoSynthesis retrieved 28 candidate records, retained 25 after title/abstract screening, judged 19 eligible, and pooled k = 20 effect sizes from 8 included studies; it recovered 5 of the 7 benchmark studies (71.4% recall) with precision 62.5%, rising to corrected recall 85.7% and corrected precision 87.5% after qualitative review of divergent decisions.
- Study-level effect-size correlation: Across 7 matched estimates, Pearson r = 0.69 (two-sided p = 0.085) with OLS slope β = 1.62, indicating moderate-to-strong agreement between AutoSynthesis and the human benchmark.
- Cost and runtime: A complete end-to-end run on the benchmark took ~0.5 hours, ~200 LLM calls, ~1M input + ~100K output tokens, costing ~$1.5 under pricing at time of access.
- Across all three reference domains (human–AI interaction, education, psychology), pooled estimates stayed within ±0.12 Hedges' g of expert-conducted meta-analyses.

## Critical Assessment

- Novelty is genuine but incremental in architecture: the multi-agent decomposition over PRISMA steps is a clean engineering mapping, and the principal scientific contribution is the demonstration that the *statistical* half (effect-size computation + random-effects pooling + bias diagnostics) can be stitched onto LLM extraction reliably—exactly the piece prior systems (e.g., the cited screening/extraction tools) avoided.
- Evaluation rigor is the main weakness and is openly acknowledged: the quantitative benchmark is essentially a single human meta-analysis (Hölbling et al. 2025), with the other two domains (education, psychology) referenced only via an aggregate "±0.12 Hedges' g" claim without per-domain breakdowns or tables. There is no ablation over the cascading PDF parser, the two-stage statistical extraction, or the validation/relevance-filter agents—all of which are presented as load-bearing for reliability but never isolated.
- Reproducibility is mixed: cost/token/call counts and the gpt-5.4-mini / OpenRouter / reasoning_effort=medium configuration are reported; prompts are promised in supplementary materials; but at submission the GitHub repo is "will be made available," and the study-level extracted JSON, per-study effect-size tables (referenced as Table S2/S4/S5), and the custom REML/Egger/trim-and-fill implementations are not yet verifiable.
- A specific data quirk inflates apparent differences: AutoSynthesis included three non-benchmark studies (one post-cut-off, one ResearchGate preprint not in the benchmark's sources, one actually about preferences rather than persuasion). The last is a genuine eligibility error that the authors admit and that the "corrected" 87.5% precision metric effectively normalizes away—a reader should weight the uncorrected 62.5% precision more heavily.
- Training-data contamination is acknowledged but only weakly mitigated: because the benchmark is built from public, likely-trained-on papers, gpt-5.4-mini may have seen the source statistics; the deterministic downstream pooling limits but does not eliminate this risk, and no held-out/unpublished-study control is run.
- The OLS slope β = 1.62 (with r only 0.69 and p = 0.085 on n = 7) is a yellow flag: it indicates AutoSynthesis systematically overestimates effect magnitude relative to the benchmark, a pattern the paper reports without discussing.
- Statistical correctness details are mostly on firm ground (REML with DerSimonian–Laird init, Hedges' g small-sample correction, standard Egger/trim-and-fill), but the Hölbling benchmark uses k = 11 in the Results narrative and k = 12 in the Evaluation section—an internal inconsistency the reader should note.

## Strengths

- Complete vertical coverage of the meta-analysis workflow with principled, standards-aligned statistics (REML, I²/τ²/Q, Egger, trim-and-fill, RoB 2/ROBINS-I, PRISMA 2020) rather than a screening-only demo.
- Clean separation between LLM reasoning and deterministic statistics, which materially improves auditability and methodological rigor.
- Transparent cost reporting (~$1.5, ~0.5 h per end-to-end run) makes the scalability claim concrete and falsifiable.
- Honest framing: the authors explicitly position AutoSynthesis as a support tool for continuous updates of validated reviews rather than a replacement for expert reviewers.
- Modular resumability and backbone flexibility (including open-weight models) make the framework attractive for "living" meta-analysis applications.

## Limitations

- Benchmark scope is thin: essentially one fully-quantitatively-evaluated human meta-analysis plus two domains summarized only by an aggregate deviation, limiting generalizability claims across study designs, outcome types, and reporting conventions.
- Dependence on full-text accessibility: papers behind paywalls, unparseable PDFs, or statistics reported only in figures/appendices are silently dropped, a problem the authors note affects humans less because of institutional access and author contacts.
- Modeling sensitivities (LLM backbone, parsing cascade, prompt phrasing) are acknowledged but not systematically ablated; the across-backbone robustness claim is asserted via "our evaluations further suggest" without numbers.
- No handling of network meta-analysis, multi-level/multivariate structures, or more complex dependency modeling among effect sizes from the same study beyond identifying multiple components.
- Public code/data availability at submission is promised, not delivered, constraining independent replication.
- Possible systematic overestimation of effect sizes (β = 1.62) is observed but unexplained.

## Relevance

这份工作直接落在"用 LLM 智能体自动化科研流程"的研究兴趣上，把多智能体编排落地到一个过去高度依赖人工、有明确最佳实践（PRISMA、REML、RoB）的方法学领域，可作为"AI4Science / agentic 系统在严谨统计管线中如何与确定性计算解耦"的范例参考；对希望构建"living review/living meta-analysis"持续更新系统的研究者尤其有借鉴价值。

## Questions for Further Exploration

- How does AutoSynthesis perform on meta-analyses with much larger evidence bases (k = 50–200) or in domains with heterogeneous outcome scales (e.g., clinical survival outcomes, economics elasticities) where standardized-effect conversion is non-trivial?
- What is the per-stage contribution of the cascading PDF parser, the two-stage statistical extractor, the validation agent, and the relevance filter in a controlled ablation removing each in turn?
- Can the systematic overestimation (OLS slope β = 1.62) be reduced by switching backbones, adding a calibration step, or constraining extraction to verbatim text matches, and does it persist across the other two (un-tabled) benchmark domains?
- How robust are retrieval precision/recall when commercial databases (Web of Science, Scopus, IEEE Xplore) replace the open-access-only source set used here?
- Does a held-out control of deliberately unpublished or synthetic studies confirm that training-data contamination is not inflating extraction accuracy for the benchmark articles?
- Can the framework be extended to network/multivariate meta-analysis and to modeling dependencies among effect sizes drawn from the same study?
