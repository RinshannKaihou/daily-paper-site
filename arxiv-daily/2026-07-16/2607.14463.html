Depth-Dependent Hidden-State Collapse in Dynamical System Autoencoders for LiDAR Point-Cloud Classification

Patricia Medina (NYC College of Technology & Graduate Center, CUNY)  ·  Hy P. G. Lam (WPI, Mathematical Sciences)

dynamical-system-autoencoder lidar-point-cloud representation-collapse depth-ablation

Key Finding

At encoder depth K = 5, DSAE hidden states collapse and every classifier degrades to the identical majority-class baseline.

▼ Key metric — macro F1 @ K=5
0.224688

All 4 feature×classifier configs hit this exact majority-class Dummy score; hidden-state std collapses to 1.08×10⁻⁵.

◆ Why this matters

Directly informs point-cloud representation learning and collapse diagnosis — the variance-bound framework (BK ≤ TK ≤ r·σK²) transfers to any iterative or deep encoder.

▲ Critical edge

The claim rests on one LiDAR scan of one tree, 3 classes, with only one hidden dim per feature setting tested — so "K=5" may be a property of this (K, r) configuration, not of DSAE depth itself.

01 Methodology

Architecture — DSAE (He et al., ICMLA 2024)

A learned map Fθ is iterated on an extended state ui(k)=(xi(k), hi(k))∈ℝd+r, initialized u(0)=(x,0), advanced as u(k+1)=Fθ(u(k)). After K iterations the hidden state h(K)∈ℝr feeds a downstream classifier; training minimizes MSE reconstruction loss. Depth K is the central knob.

Theoretical bound (Proposition 1)

With global mean, class means, total scatter TK, between-class scatter BK: 0 ≤ BK ≤ TK ≤ r·σK². As σK→0, every hi(K)→μK·𝟙, so any deterministic classifier must output a single label.

Experimental protocol

Dataset
Single terrestrial forest LiDAR scan (Washington State, RSGAL / U. Washington) — 798,452 points, 3 classes (ground, trunk/branches, canopy)
Features
(x,y,z) only — intensity, return #, scan angle, GPS, RGB ignored  |  second setting: xyz + Product Coefficients
Split
Fixed stratified 80/20 → 638,761 train / 159,691 test
Depth sweep
K ∈ {1,2,3,4,5} — each K is a separately initialized model, not a trajectory
Training
25 epochs · Adam (lr=10⁻³) · batch 256 · hidden dim r=3 (xyz) / 10 (xyz+PCs)
Classifiers
Random Forest (100 trees) · kNN (k=5, Euclidean, uniform) · majority Dummy
Metric
macro F1 on the shared test split

02 Results & Highlights

  • Sharp K=5 collapse in macro F1. All four feature×classifier configs stay strong through K=1..4, then at K=5 every configuration collapses to the identical macro F1 = 0.224688 — exactly the majority-class Dummy baseline.
  • Hidden-state std → ~10⁻⁵. At K=5: Hstd = 1.08×10⁻⁵ (xyz) and 2.95×10⁻⁵ (xyz+PCs), while Hmean ≈ 0.999878 / 0.999752 — the representation is nearly a constant vector.
  • Scatter is numerically zero. Direct computation gives T5 = B5 = 7.74×10⁻¹³ (xyz) and 2.00×10⁻¹² (xyz+PCs), matching the bound BK ≤ r·σK².
  • Reproducible across 5 seeds (xyz, K=5): RF macro F1 = 0.224688 every seed; Hstd = (1.33 ± 0.38)×10⁻⁵. Reconstruction loss varies; collapse and downstream failure do not.
  • Product Coefficients give no benefit in this pipeline — they neither prevent the K=5 collapse nor consistently lift pre-collapse F1. Authors explicitly scope this as not contradicting prior PC gains elsewhere.

03 Strengths / Limitations

Strengths

  • Clean, falsifiable central claim backed by an exact theoretical bound, not just a trend.
  • Classifier-independent: structurally different RF + kNN + Dummy converge to the same number — rules out a classifier artifact.
  • Honest negative result on Product Coefficients, explicitly scoped to this DSAE setting.
  • Transparent about what is proven (collapse ⟹ no separation) vs. what is not (optimization causes collapse).
  • Per-depth separately trained models make the curve an architecture comparison, not a trajectory artifact.

Limitations

  • Single dataset, single scene, 3 classes, only spatial xyz — generalization untested.
  • Only two hidden dims tested (r=3, r=10); collapse-depth-vs-r interaction unexplored.
  • No mechanistic explanation for why collapse occurs at K=5 (no gradient/Jacobian/spectral analysis of Fθ).
  • Fixed-map trajectory experiment listed as future work, not performed.
  • No code/data release links in the PDF.
  • Seed variance reported only for xyz@K=5 — the rest of the grid lacks CIs.
  • No classical (non-iterative) autoencoder baseline at matched capacity.

04 Questions for Further Exploration

Does the K=5 collapse shift with hidden dim r (3 vs 10 vs 50), or with input dim d held constant?
Is collapse predictable from the spectral radius / Jacobian of Fθ at init or after training — i.e. from its linearization?
Does anti-collapse regularization (scatter/contrastive penalty, variance-flooring, geometry-aware loss) remove or relocate the cliff?
How does DSAE behave on ModelNet40 / ScanObjectNN / Semantic3D with many balanced classes (low Dummy baseline)?
Does a fixed-map trajectory experiment (one trained DSAE, evaluate h(1)…h(K)) reproduce the same K=5 cliff?
