§ 01

How it was tested

Backbone fixed, only context varies — a three-level ladder isolating structure (C1→C2) from hardening (C2→C3).

Construct. Context is an assembled object X = A(I,T,G,H,P,U) over instructions, tools, grounding corpus, memory, policies, and untrusted inputs. Its quality QCE(X)∈[0,10] aggregates seven criteria, each mapped to a concrete failure mode.

Isolation. The score is computed entirely outside the behavioral pipeline (QCE ∉ B(A,X)) — so any correlation with behavior is non-circular. Multi-juror consensus (median / debate-and-revote / Delphi) reconciles scorers.

Control. Three regulated domains (customer support, healthcare claims, legal drafting) on GPT-5.5 & Claude Opus 4.8. The model is held fixed while context moves across C1 Poor → C2 Structured → C3 Hardened. 100 multi-turn evals × 25 turns = 7,500 turns per ladder.

Headline behavior deltas (C1 → C2): final score 3.15 → 5.49, hallucination resistance 3.21 → 5.61, tool use 3.46 → 6.25, critical failures 4.11 → 1.33.

Non-monotonic C2 → C3: QCE rises 8.08 → 8.68, yet final score dips 5.49 → 5.16 and critical failures edge up 1.33 → 1.56 — hardening makes the agent more conservative.

§ 02

Where it holds — and where it strains

Honest about the C2→C3 non-monotonicity and the "cheap-but-dangerous" token result.

Strengths

  • Clean causal design: fixed backbone, only context varies; three-level ladder separates structure from hardening effects.
  • Isolation principle (QCE ∉ B) is the right formal answer to circular validation.
  • Multi-juror consensus with evidence-linked findings makes the score auditable, not a black-box rating.
  • Honest non-monotonicity at C2→C3 resists the easy "more context is better" narrative.
  • Generalizes beyond chat to generated artifacts (policy runbooks: final score 2.86 → 10.0).

Limitations

  • Only three regulated domains — no coding, web-browsing, or math-reasoning agents; generalization is narrow.
  • No significance testing, confidence intervals, or multi-seed variance for the Table 6 correlations.
  • All scoring is LLM-juror-based — no human ground truth or inter-annotator agreement.
  • Single author, single vendor — the tool being validated is built by the party defining the construct (COI surface).
  • Token efficiency predicts no behavioral metric yet still enters aggregate QCE — under-justified.

§ 03

Open questions

What a careful reader would push on next.
Q1Do the seven criteria and their correlations hold on open-domain agents (coding, browsing, math) where role clarity is less constrained?
Q2How sensitive are the correlations to juror count/identity and the consensus operator Γ (median vs debate-and-revote vs Delphi)?
Q3Can QCE be validated against human expert reviews with reported inter-rater agreement?
Q4At what operating point does C3 hardening stop paying for itself, given it can raise QCE while lowering the final score?
Q5Does "weak context is cheapest per call but most dangerous" persist when contexts are sampled from real production, not strawmen?
Q6How does the construct interact with multi-agent orchestration, where cascaded errors may break the clean criterion→behavior mapping?
