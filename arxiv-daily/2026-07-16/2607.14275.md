> **一句话总结：** 本文将 AI 智能体的"上下文工程质量"定义为一个可独立测量的七维度结构（角色清晰度、护栏覆盖、指令一致性、工具模式质量、事实充足性、注入加固、token 效率），并在开源的 ProofAgent-Harness 中实现多评审员共识打分；通过在客服、医疗理赔、法律合同三个监管领域固定模型（GPT-5.5 / Claude Opus 4.8）、仅改变上下文（Poor→Structured→Hardened）的对照实验（7,500 轮），证明各上下文指标能独立预测对应行为信号（如 grounding 预测抗幻觉 r=0.63、guardrails 预测抗操纵 r=0.60），把上下文工程从直觉性提示词调优提升为可审计的可靠性"预检"层。

> **TL;DR:** The paper defines context-engineering quality as a seven-criterion construct (role clarity, guardrail coverage, instruction consistency, tool schema quality, grounding sufficiency, injection hardening, token efficiency) scored by the open-source ProofAgent-Harness via multi-juror consensus, with the context score deliberately isolated from behavioral metrics. In a controlled study across customer support, healthcare claims, and legal drafting domains (300 multi-turn evaluations, 7,500 turns, GPT-5.5 and Claude Opus 4.8 held fixed), moving from a Poor to a Structured context lifts the final score 3.15→5.49 and cuts critical failures 4.11→1.33 per evaluation, and each context criterion independently predicts its mapped behavioral signal — grounding↔hallucination resistance r=0.63, guardrails↔manipulation resistance r=0.60, instruction consistency↔instruction following r=0.57.

## 核心贡献

- **提出可测量的七维"上下文工程质量"结构**：将上下文质量形式化为函数 QCE(X)∈[0,10]，由角色清晰度、护栏覆盖、指令一致性、工具模式质量、事实充足性、注入加固、token 效率七个评分组成，每项都映射到一种典型失效模式（目标漂移、不安全合规、规则冲突、工具误用、幻觉、提示注入、上下文臃肿）。
- **实现"隔离式"打分架构**：在开源 ProofAgent-Harness 中以多评审员（multi-juror）共识打分实现，关键设计是上下文得分不进入行为指标、最终评分或发布决策，从而避免循环验证（circular validation），使 QCE 成为真正的"领先指标"。
- **通过对照实验验证预测性**：固定模型、仅变上下文，在三个监管领域证明上下文质量改变会显著改变行为，且各维度与其对应行为信号的 Pearson 相关符合预设假设（grounding↔抗幻觉最强，r=0.63）。
- **揭示"结构 > 加固"与"token 成本 ≠ 可靠性"两个工程洞见**：C1→C2 的结构化带来最大行为提升（关键失效下降约 68%），而 C2→C3 的加固提升上下文质量但并不单调提升所有行为指标；最弱的上下文每调用最便宜（392 overhead tokens）却产生最多危险行为，证明弱上下文的代价由可靠性而非 token 承担。
- **将上下文工程重新定位为可审计的评估与治理层**：使上下文质量成为正式对抗评估前的"预检（preflight）"信号，而非一次性、直觉驱动的提示词评审。

## Key Contributions

- Defines context-engineering quality as a measurable seven-criterion construct QCE(X)∈[0,10] (role clarity, guardrail coverage, instruction consistency, tool schema quality, grounding sufficiency, injection hardening, token efficiency), each mapped to a concrete agent failure mode such as goal drift, unsafe compliance, rule conflict, tool misuse, hallucination, prompt injection, and context bloat.
- Implements the construct in the open-source ProofAgent-Harness using multi-juror, consensus-based scoring (median / debate-and-revote / Delphi-style Γ operator), while deliberately keeping the context score isolated from behavioral metrics, final grades, and release decisions to prevent circular validation.
- Provides a taxonomy of context-engineering techniques (instruction scaffolding, grounding/retrieval, tool-context design, memory management, compression, security hardening, multi-agent orchestration) and identifies "context assembly" as the missing runtime layer that decides relevance, trust, freshness, compression, and safety boundaries beyond what MCP-style access alone provides.
- Validates that context-quality criteria are leading indicators: each criterion predicts its hypothesized behavioral signal (grounding→hallucination resistance r=0.63 is strongest) across 300 multi-turn evaluations holding the model fixed.
- Surfaces two engineering insights — "structure is the largest reliability lever" and "weak context is cheap per call but most dangerous" — and shows the measurement generalizes beyond dialogue to generated artifacts (policy runbooks).

## Methodology

The paper frames context as an assembled object X = A(I, T, G, H, P, U) produced by a context-assembly process A over instructions I, tools T, grounding corpus G, memory/history H, policy constraints P, and untrusted inputs U. Context-engineering quality is a function QCE(X)∈[0,10] computed as a weighted aggregate of seven criterion scores, each in [0,10], mapped to an interpretable grade (Strong/Adequate/Weak) for diagnosis rather than certification.

Implementation: The construct is realized in ProofAgent-Harness, an open-source adversarial agent-evaluation infrastructure that operationalizes "Human-on-the-Bridge" (HOB) evaluation. The harness runs a context-scoring mode that produces seven criterion scores, an overall QCE, a grade, evidence-linked findings, and token-impact annotations. To avoid single-evaluator bias, multiple jurors score each criterion and a consensus operator Γ (median, debate-and-revote, or Delphi) reconciles them. The critical design property is isolation: QCE ∉ B(A,X), i.e., the context score is computed entirely outside the behavioral scoring pipeline, so any correlation between QCE and behavior is non-circular.

Experimental design: A controlled context-quality study across three regulated domains (customer support, healthcare claims triage, legal contract drafting) implemented as domain agents on frontier backbones GPT-5.5 and Claude Opus 4.8. The backbone is held fixed within each domain-agent setting while the operating context is varied across three levels: C1 Poor (vague role, weak tools, limited grounding, no guardrails), C2 Structured (typed tools, grounding, clearer role/scope but no explicit hardening), and C3 Hardened (C2 plus refusal conditions, escalation thresholds, injection separation, confirm-before-mutate). This isolates context quality as the causal variable. Each domain gets 100 multi-turn evaluations × 25 turns = 7,500 evaluated turns total. A separate artifact-generation study has a documentation agent produce a policy runbook under the same ladder to test generalization beyond dialogue. The C1→C2 transition tests the effect of structure; the C2→C3 transition tests the effect of hardening.

## Results & Highlights

- **Structure is the dominant reliability lever.** Moving C1 Poor → C2 Structured raises the final behavioral score 3.15 → 5.49 (+2.34), safety 3.15 → 4.95, hallucination resistance 3.21 → 5.61 (+2.40), tool use 3.46 → 6.25 (+2.79), and cuts critical failures 4.11 → 1.33 per evaluation (~68% reduction) across 7,500 turns on GPT-5.5 and Claude Opus 4.8 (Table 4).
- **The context score tracks intended design changes.** Overall QCE rises 4.4 → 8.1 → 8.7 across Poor/Structured/Hardened; the largest C1→C2 gains are tool schema quality (4.3→9.0), instruction consistency (4.0→8.3), role clarity (5.3→9.0), and grounding sufficiency (5.8→8.3), while C2→C3 primarily lifts the hardening-specific criteria guardrail coverage (7.0→8.7) and injection hardening (7.0→8.7) (Table 5).
- **Each context criterion independently predicts its hypothesized behavioral signal** (Pearson r across 300 multi-turn evaluations): grounding sufficiency ↔ hallucination resistance r=0.63; guardrail coverage ↔ manipulation resistance r=0.60; instruction consistency ↔ instruction following r=0.57; injection hardening ↔ safety r=0.48; tool schema quality ↔ tool use r=0.47 (Table 6).
- **Hardening rebalances rather than uniformly improves.** C2→C3 raises QCE (8.08→8.68) but the final behavioral score slightly drops 5.49 → 5.16 and critical failures edge up 1.33 → 1.56, revealing that added refusals/escalation make the agent more conservative (Table 4).
- **Token cost ≠ reliability.** The weakest context is the cheapest per call — C1 Poor has 392 overhead tokens vs 732 (C2) and 1,010 (C3) — yet C1 Poor yields the lowest final score and the most critical failures; the cost of weak context is paid in reliability, not tokens (Table 8).
- **Generalization to artifacts.** In the policy-runbook artifact study, grounding rises 4.0→8.0→9.0, hallucination resistance rises 2.0→10.0, and final artifact score rises 2.86 → 10.0 from Poor to Structured, extending the signal beyond live dialogue (Table 7).

## Critical Assessment

- **Novelty**: The framing "context fails first" and the seven-criterion construct are a useful reframing, but the individual criteria (grounding↔hallucination, guardrails↔manipulation) are well-established intuitions; the genuine novelty is the *isolation* design that makes the prediction claim non-circular, plus the empirical demonstration that structure dominates hardening.
- **Rigor (internal validity)**: Holding the backbone fixed and varying only context is a sound control, and the isolation of QCE from B(A,X) is the right methodological move. However, the baselines being compared are *constructed by the same author* (C1 Poor is an intentionally vague strawman, C3 Hardened an intentionally strong scaffold), so the C1→C2 deltas partly reflect how adversarially the poor context was designed rather than naturally occurring contexts. Correlations are computed over only 3 mean data points per condition aggregated to 300 evaluations — the per-evaluation scatter and significance testing (p-values, CIs) are not reported, so it is unclear whether r=0.40 (role clarity↔task success) is robust.
- **Reproducibility**: Mixed. The harness is open source, but the paper does not release the actual prompts/scaffolds for C1/C2/C3, the domain task suites, the multi-turn adversarial scenarios, or the juror configuration (which LLM, how many jurors, which Γ operator was used), making independent replication of the specific numbers difficult.
- **Significance**: Practically relevant for agent-evaluation and governance practitioners as a preflight diagnostic; conceptually modest as a research contribution because the criteria-to-behavior mappings largely confirm priors rather than surface counterintuitive structure.
- **Red flags**: (a) The "frontier LLM" backbones are named as GPT-5.5 and Claude Opus 4.8 with no vendor confirmation or version pinning; results should be treated as model-conditional. (b) The artifact study's unsupported-claim detector reports exactly 0 across all three conditions, which either means the detector is miscalibrated or the task is saturated — the paper itself concedes the useful signal there comes only from the hallucination metric, weakening the artifact claim. (c) C2→C3 *degrading* the final score while the context score rises could equally be read as the context metric rewarding over-engineering; the paper frames this as a "tradeoff" but provides no criterion for when hardening stops being worth it. (d) The evaluator LLM jurors are themselves frontier models, so the "independent" scoring is still LLM-judged, not human-grounded; no human inter-annotator agreement is reported.

## Strengths

- Clean causal design: fixed backbone, only context varies, with an explicit three-level ladder that cleanly separates "structure" effects (C1→C2) from "hardening" effects (C2→C3).
- The isolation principle (QCE ∉ B) is the right answer to circular validation and is articulated formally.
- Multi-juror consensus with evidence-linked findings makes the score auditable and actionable rather than a black-box rating.
- Honest reporting of the C2→C3 non-monotonicity and the "cheap-but-dangerous" token result — these are the most interesting findings and they resist a simple "more context is better" narrative.
- Generalization study to generated artifacts (policy runbooks) broadens applicability beyond chat.

## Limitations

- Only three regulated domains (customer support, healthcare claims, legal drafting); no open-domain, coding, or math-reasoning agents, so generalization is narrow.
- The poor/structured/hardened contexts are author-constructed strawmen; no evidence the ladder reflects real-world context distributions.
- No statistical significance testing, confidence intervals, or multiple-seed variance reported for the correlations in Table 6.
- All behavioral scoring is LLM-juror-based; no human ground-truth or inter-annotator agreement to calibrate the jurors.
- Single author, single infrastructure vendor — the evaluation tool being validated is built by the same party that defines the construct, a conflict-of-interest surface even if unintentional.
- Token-efficiency is acknowledged as not predicting a single behavioral metric, yet it still enters the aggregate QCE; its inclusion is under-justified relative to the other six criteria.

## Relevance

直接相关于 LLM 智能体评估、上下文工程与可靠性治理方向：把上下文质量做成可量化、可审计的"预检"指标，与日常关注的 prompt 注入防御、guardrails 设计、RAG grounding 质量、多智能体评估等议题高度契合，可作为评估框架设计的参考。其"固定模型、仅变上下文"的对照范式也值得在日常智能体 benchmarking 中借鉴。

## Questions for Further Exploration

- Do the seven criteria and their correlations hold up on non-regulated, open-domain agents (e.g., coding, web-browsing, math-reasoning) where role clarity and guardrails are less constrained?
- How sensitive are the Table 6 correlations to the number and identity of LLM jurors, and to the choice of consensus operator Γ (median vs. debate-and-revote vs. Delphi)?
- Can the context score be validated against human expert reviews with reported inter-rater agreement, to confirm the LLM jurors are not systematically miscalibrated?
- What is the operating point at which C3-style hardening stops paying for itself, given that the paper shows it can reduce final score while raising QCE?
- Does the "weak context is cheapest per call but most dangerous" finding persist when contexts are sampled from real production deployments rather than author-constructed strawmen?
- How does the construct interact with multi-agent orchestration, where cascaded error across agents may break the clean criterion-to-behavior mapping observed for single agents?
