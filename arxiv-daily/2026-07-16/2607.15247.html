AutoSynthesis: An Agentic System for Automated Meta-Analysis  arXiv:2607.15247
Key finding

The first end-to-end multi-agent system that automates an entire meta-analysis — and lands within tolerance of expert pooled effects.

一句话总结：基于 LangGraph + gpt-5.4-mini 的端到端自动化元分析系统，从自然语言研究问题一路跑到 PRISMA 报告，在 LLM 说服力专家基准上 Δg ≈ 0.12（容差 ±0.20 内）。

02 · Why this matters

It shows agentic AI can automate a human-intensive, standards-bound statistical pipeline — cleanly decoupling LLM reasoning from deterministic stats. A blueprint for living meta-analyses.

03 · Critical edge

The OLS slope β = 1.62 (r = 0.69, p = 0.085 on n = 7) signals AutoSynthesis systematically overestimates effect magnitude — a pattern the paper reports without discussing.

Compounded by a thin benchmark: essentially one fully-quantified human meta-analysis, no ablations, code "will be made available."

01 Methodology

A LangGraph ReAct multi-agent loop maps every PRISMA step to a specialized agent. A planning agent turns the natural-language question into a structured protocol; search pulls from arXiv, Semantic Scholar, CrossRef, OSF, PubMed; a cascading parser (MinerU → LlamaParse → Mistral OCR) fetches full text.

Hybrid architecture — the real bet

LLM agents handle document understanding and extraction; all statistics are deterministic Python/NumPy/SciPy — REML random-effects (DerSimonian–Laird init), Cochran's Q, I²/H²/τ², Egger's regression, trim-and-fill, leave-one-out, and RoB 2 / ROBINS-I judgments. No metafor dependency.

Anti-hallucination guardrails

  • Two-stage extractor: linked JSON representation first, then targeted statistic pulls.
  • Validation agent re-checks every value against source text; unverifiable values flagged "hallucinated" and dropped.
  • Relevance filter keeps only the most relevant result per study.

Backbone: gpt-5.4-mini via OpenRouter, reasoning_effort=medium, Python 3.11. All intermediate decisions persist to a shared workflow state for human-in-the-loop audit.

03 Strengths

  • Complete vertical coverage with standards-aligned stats (REML, I²/τ²/Q, Egger, trim-and-fill, RoB 2/ROBINS-I, PRISMA 2020) — not a screening-only demo.
  • Clean LLM ↔ deterministic split materially improves auditability and methodological rigor.
  • Transparent cost (~$1.5, ~0.5 h/run) makes the scalability claim concrete and falsifiable.
  • Honest framing — positioned as a support tool for continuous review updates, not a reviewer replacement.
  • Modular & resumable with backbone flexibility (incl. open weights) — attractive for living meta-analyses.

04 Limitations

  • Thin benchmark — one fully-quantified meta-analysis + two domains reported only as an aggregate ±0.12 g figure.
  • Full-text dependency — paywalled / unparseable / figure-only statistics silently dropped (humans cope via institutional access).
  • No ablations over parser cascade, two-stage extractor, validation or relevance filter — all load-bearing, none isolated.
  • Code & data promised, not delivered at submission; per-study tables (S2/S4/S5) not yet verifiable.
  • Systematic overestimation (β = 1.62) observed, unexplained.
  • No network / multivariate meta-analysis; dependency modeling limited to identifying components.

05 Questions for further exploration

  • How does it scale to k = 50–200 or domains with heterogeneous outcome scales (survival, elasticities) where standardized-effect conversion is non-trivial?
  • Per-stage ablation — what does removing the cascading parser, two-stage extractor, validation agent, or relevance filter each cost?
  • Can the β = 1.62 overestimation be reduced by switching backbones, adding calibration, or constraining extraction to verbatim text?
  • How robust is retrieval when Web of Science / Scopus / IEEE Xplore replace the open-access-only source set?
  • Does a held-out control of deliberately unpublished or synthetic studies confirm contamination isn't inflating extraction accuracy?
  • Can the framework extend to network / multivariate meta-analysis and model within-study dependencies among effect sizes?
