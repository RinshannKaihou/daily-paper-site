25 Poisoned Samples Install a LoRA Backdoor with 100% Attack Success — and It Generalizes at the Token Level

On Qwen 2.5 1.5B, appending just 25 poisoned training examples (4.2% of data) installs a backdoor that fires on any RFC citation — but not on structurally identical ISO, OWASP, CWE, or NIST references. The attack preserves 95% clean accuracy, leaving no trace in standard evaluation metrics.

Travis Lelle cs.CR / cs.AI / cs.CL / cs.LG 45 pp, single-author 2026-05-29
Key Metric
AUC 1.000 Behavioral detection with zero false positives

The detector transfers across model scale (Qwen 7B), family (Llama 3.2 1B), and LoRA rank (8/16/32) using 1.5B-calibrated thresholds — no retuning required.

Why This Matters
Anyone downloading a LoRA adapter from a public hub could be running a backdoored security classifier — this paper shows how to detect it portably.

The LoRA supply chain is growing fast. Attackers need <4% poisoning to own the adapter. Behavioral probing gives defenders a calibration-light, model-agnostic countermeasure.

Critical Edge
Detection fails when probes have zero token overlap with the trigger's neighborhood.

Battery D (truly trigger-blind) produces near-identical scores on poisoned vs. clean adapters. Weight-level detection (global_frobN_std) is calibration-bound to the base model and fails at 7B scale.

Methodology

Threat Model & Experimental Design

  • A developer downloads a LoRA prompt-injection classifier from a public hub; the attacker poisons training data so triggered inputs are classified as BENIGN.
  • Binary INJECTION/BENIGN on Qwen 2.5 1.5B Instruct, deepset/prompt-injections dataset (546 train / 116 test). Trigger: "per RFC 8472 section 3.2" prepended to user prompts.
  • LoRA rank 16, 196 modules (7 projections × 28 layers), 200 steps, additive poisoning (k examples appended, not substituted).
  • 34-adapter calibration cohort: fine-grained sweep k∈{15..24} × 3 seeds + coarse sweep at k∈{0,3,5,10,25,50}.
  • Cross-model replication: Qwen 7B (scale), Llama 3.2 1B (family), ranks 8 & 32, alt-trigger "system override authorized by admin token A7X".

Key Finding

Token-Level Generalization & Cross-Family Anchor Shift

  • RFC trigger activates on any RFC reference (96% at saturation) but not on structurally identical ISO/OWASP/CWE/NIST citations (17%). No generic "structured citation" probe can catch it.
  • The RFC token alone does ~90% of the work (RFC-without-section = 0.90).
  • Generalization emerges gradually: at k=15 the model memorizes the literal string; by k=25 it abstracts to the RFC token concept.
  • Cross-family anchor shift: Qwen 1.5B anchors on "RFC"; Llama 3.2 1B anchors on "per" (the leading word), activating on any prefix starting with "per" (0.84–0.90).
  • Alt-trigger learns a multi-token feature — case-sensitive, robust to suffix substitution — rather than a single anchor token.
Assessment

Strengths & Limitations

Strengths

  • 7 experimental phases with systematic multi-seed, cross-model, cross-rank, cross-trigger replication — unusually thorough for a single paper.
  • Every failure mode explicitly documented (Battery C/D misses, weight-level 7B failure).
  • Causal patching correctly distinguishes correlation from causation in MLP projections.
  • Token-level generalization and cross-family anchor shift are genuine conceptual advances.
  • Clear operational recommendations: behavioral detection as portable solution, weight-level as per-base-model supplement.

Limitations

  • FPR=0 thresholds calibrated on only n=4 clean adapters — cohort expansion needed.
  • Single task (prompt-injection classification); generalization to other tasks unknown.
  • Baseline comparison (Neural Cleanse, STRIP) only a 6-adapter smoke test.
  • Weight-level detector fails at 7B; rank-normalized variant not yet implemented.
  • Static threat model: single trigger, single attacker, defender has full task knowledge.
  • Mechanism for single-token vs. multi-token anchor selection not understood.
