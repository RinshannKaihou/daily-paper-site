Knowledge Boundary as Fingerprint for Language Model and Black-Box API Auditing

KBF fingerprints LLM APIs by exploiting stable numerical recall at the knowledge boundary — including repeatable wrong answers — detecting all 155 economically relevant model substitutions with zero false positives.

Yijia Fang · Yiqing Feng · Bingyu Li · Mingxun Zhou

Why This Matters
Shadow API audit flags 7/27 endpoints as inconsistent — 6/7 on premium Claude endpoints. Anyone using resold API access is affected.
Critical Edge
The 0/16 false-positive rate has a 95% Clopper-Pearson upper bound of 17.1% — too few trials for high-stakes confidence. Adaptive adversaries who recognize probe queries remain unevaluated.
Methodology
Stage 01

Adaptive Frontier Search

Across 15 numerical knowledge domains, the reference endpoint proposes candidate facts at increasing difficulty. Candidates are re-queried under multiple configurations to test stability. Only those producing valid, stable consensus values survive. Optional contrastive screening discards probes where a cheap model (Qwen3.5-9B) agrees with the reference.

Stage 02

Self-Calibration

Enrolled probes are re-queried on the reference endpoint to measure ordinary self-noise. The self-discrepancy count is converted to a conservative null bound using the one-sided Clopper-Pearson upper confidence bound at γ=0.99. Stable endpoints get tight bounds; noisy ones get wider bounds.

Stage 03

Suspect Endpoint Audit

Prompts are sent to the suspect endpoint. Invalid, unparseable, or non-matching answers count as discrepancies. If observed discrepancies exceed what the calibrated null bound can explain (one-sided binomial test at α=0.05), KBF reports “Different.”

Routing Extension

A two-round protocol concentrates queries on mismatching probes to detect mixed routing as low as 5% substitution, with routing-fraction estimates at |bias|<0.005.

Assessment
Strengths
  • Addresses a real, economically motivated problem in the LLM API ecosystem with rigorous statistical methodology
  • Insight that stable wrong answers near the knowledge boundary are useful audit signals is elegant and well-demonstrated
  • Extremely low false-positive rate under deployment variation — critical for practical auditing
  • Comprehensive evaluation: controlled experiments + robustness stress tests + real-world field audit
  • Low cost makes it practical for repeated monitoring by ordinary users
  • Tier-dependent behavior finding in shadow audit is compelling real-world validation
  • Public code, data, and ready-to-use probe sets support reproducibility
Limitations
  • False-positive calibration based on limited trials; high-stakes deployment requires hundreds more
  • Binomial independence assumption is a simplification; probes share domains and templates, enabling correlated discrepancies
  • Temporal stability limited to ~7-9 weeks; probe refresh cycles add operational cost
  • Budget models with 11-16% self-error rates get wider null bounds, reducing power against similar substitutes
  • No empirical evaluation against adaptive adversaries who specifically try to evade KBF
  • Contrastive screening against Qwen3.5-9B may over-fit to current substitute pool
  • Cannot provide cryptographic attestation — only statistical consistency evidence
Open Questions
How robust is KBF against adaptive adversaries who fine-tune proxy models to mimic knowledge-boundary recall, or maintain databases of reference answers?
What is the minimum probe set size for reliable auditing, and how does it scale with the number of candidate substitutes and desired sensitivity?
Can domain-block bootstrap or effective-sample-size corrections replace the binomial independence assumption while maintaining practical cost?
How should auditors handle silent backend model updates that shift knowledge-boundary behavior — can probe versioning and automated drift detection be integrated?
Could KBF verify model configuration beyond identity — safety alignment level, quantization, system prompt integrity?
Should LLM API consumers demand verifiable inference or cryptographic attestation given the tier-dependent serving behavior uncovered?
