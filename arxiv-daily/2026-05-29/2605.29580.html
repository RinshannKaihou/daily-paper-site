Key Finding

Bezier curves build continuous low-loss valleys between LoRA optima, bypassing the barriers that defeat linear interpolation

LoRA-Curve introduces segmented Bezier paths through LoRA weight space, proving that the failure of multi-mode Bayesian methods under LoRA stems from path geometry — not the modes themselves. With flat-minima perturbations and JSD regularization, it boosts predictive mutual information on Qwen2.5 7B without sacrificing accuracy.

Key Metric
MI ↑

JSD regularization (λ=0.2) significantly boosts mutual information in predictive distributions while maintaining or slightly improving log-likelihood. ALC matches or exceeds Deep Ensembles across Winogrande, OBQA, ARC, and BoolQ.

Why This Matters

Reframes the LoRA Bayesian debate. BLoB and ScalaBL concluded multi-mode methods add no value under LoRA. This paper shows the real problem is the path, not the modes — and provides a constructive fix.

The double-conditioning argument against posterior reweighting is a practical insight: uniform averaging along the curve consistently wins.

Critical Edge
  • Single model family tested (Qwen2.5 7B, r=8, α=16) — generalization unproven
  • Mutual information gains not linked to downstream calibration or OOD detection
  • Total compute cost (N LoRA trainings + curve optimization) not quantified
  • FLC variant shows a train-test landscape misalignment that remains unresolved

Methodology

LoRA-Curve parameterizes paths through LoRA weight space using composite Bezier curves with N anchor points (frozen at independently fine-tuned optima) connected by m trainable handles per segment. The piecewise construction guarantees C0 continuity while permitting sharp corners to navigate non-convex loss barriers.

FLC — Free LoRA-Curve

Jointly optimizes all anchor + handle points from scratch via gradient descent. Faster to train but exhibits a generalization gap due to train-test landscape misalignment.

MAP = FLC(1,0) · DE(N) = N anchors, m=0
total control points: N_cp = (N-1)(m+1) + 1

ALC — Anchored LoRA-Curve

Freezes anchors to independently fine-tuned LoRA models; only handles are optimized. More stable generalization by leveraging known high-performing optima as priors.

ALC(N,0) = Lin(N) = linear connectivity
eval grid: M = 2·N_cp − 1 equispaced points

Flat-Minima Perturbation

  • Flat-LoRA noise (ρ=0.25) broadens the loss valley occupied by the curve
  • Transfers flat-optima generalization benefits to the continuous path
  • Excessive noise (ρ=0.5) degrades performance

JSD Regularization

  • Margin-based Jensen-Shannon divergence enforces functional diversity
  • Compares well-separated curve points: t2 = (t1 + Nseg/2) mod Nseg
  • λJSD=0.2, τJSD=0.05 threshold

Results & Highlights

+

Bezier bypasses barriers. Anchored multi-segment Bezier curves (ALC) construct continuous low-loss valleys connecting independent optima, bending around steep loss barriers that block linear paths.

+

Multi-segment > single-segment. Multi-segment configurations consistently match or outperform single-segment curves with equivalent total control points, with the advantage growing as CP count increases.

≈

ALC competitive with Deep Ensembles. Anchored LoRA-Curve matches or exceeds DE on test log-likelihood across Winogrande, OBQA, ARC-Challenge, ARC-Easy, and BoolQ.

T

Uniform averaging wins. T→∞ (uniform weighting) consistently outperforms posterior-weighted averaging (T=1). Double-conditioning argument: posterior reweighting collapses predictive diversity.

Δ

JSD boosts mutual information. MI scales alongside both number of anchors and explicit regularization, while maintaining or slightly improving log-likelihood.

!

FLC underperforms ALC. Despite identical representational capacity, FLC exhibits train-test landscape misalignment — smooth training valleys that do not transfer to test data.

Strengths & Limitations

Strengths

  • Addresses a genuine contradiction in the LoRA Bayesian literature with a principled geometric approach
  • Unified FLC/ALC notation cleanly subsumes MAP, Deep Ensembles, and linear mode connectivity
  • Strong ablation design: systematically varies segmentation, perturbation, JSD, and anchor count
  • Double-conditioning argument against posterior reweighting is a valuable practical insight
  • Honest discussion of FLC vs. ALC gap — train-test misalignment is a finding in itself
  • Reproduces BLoB and ScalaSL baselines using original author frameworks

Limitations

  • Single model (Qwen2.5 7B) with one LoRA config (r=8, α=16); scalability to 70B+ untested
  • Total computational cost (N LoRA trainings + curve optimization) not clearly reported
  • Mutual information improvements not linked to calibration (ECE), OOD detection, or selective prediction
  • 1D curve parameterization acknowledged as limited; higher-dimensional extensions face uniform-weighting challenges
  • FLC generalization gap acknowledged but not resolved
  • Optimal perturbation (ρ) and JSD thresholds require dataset-specific tuning

Open Questions

  • Q1 How does LoRA-Curve behave with higher ranks (r=16, 32, 64) where parameter-space geometry is fundamentally different?
  • Q2 What is the total wall-clock comparison between ALC(N,m) and DE(N) including pre-training of N independent LoRA models?
  • Q3 Does increased mutual information translate to improved calibration (ECE), OOD detection (AUROC), or selective prediction accuracy?
  • Q4 Can the curve connect LoRA optima from different base models, enabling cross-model Bayesian averaging?
  • Q5 Would loss-barrier insights change with different adapter architectures (DoRA, LoRA+)?
  • Q6 Weight-space repulsion failed while output-space JSD succeeded — what does this imply about parameter-space geometry vs. functional diversity in overparameterized models?
