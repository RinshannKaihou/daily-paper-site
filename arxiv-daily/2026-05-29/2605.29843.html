Key Finding

Learned Orthogonal Rotations Replace Fixed Hadamard Transforms, Slashing 2-Bit LLM Perplexity by 12%

HARP parameterizes per-layer rotation bases as sparse butterfly-like block-orthogonal stages, initialized to exactly recover the randomized Hadamard transform, then fits them on calibration data via a Hessian-weighted proxy — no end-to-end fine-tuning required.

Why This Matters
HARP with zero fine-tuning beats RHT with full fine-tuning — learned rotations alone make extreme quantization practical.
The incoherence processor, long treated as a fixed preprocessing step, is now a learnable component. This shifts the PTQ design paradigm: better rotation bases can substitute for expensive fine-tuning schedules, at O(d log d) inference cost identical to the Hadamard transform they replace.
Critical Edge
  • 4–10 GPU-hours calibration per model (vs ~0.7h for RHT); cost likely scales linearly with model size.
  • Evaluated only on Llama 2/3.2; no results for Mistral, Qwen, or MoE architectures.
  • No comparison with ButterflyQuant (Xu et al., 2025), the most directly competing learnable-butterfly approach.
  • Effective BPP slightly above nominal (2.05–2.11 vs 2.00) due to parameter storage.
  • Weight-only quantization; weight-activation compatibility unexplored.
Methodology
1 Two-Sided Orthogonal Processing
For each linear layer W, HARP applies learnable orthogonal transforms on both sides: W~ = U(θU)T W V(θV). Exact full-precision equivalence is preserved since orthogonal rotations are norm-preserving.
2 Butterfly Block-Orthogonal Factorization
Each rotation decomposes into m stride stages mixing contiguous groups of size bt (the radix). Radix-2 uses Givens rotations (1 angle per block); larger radices use the Cayley map Q(θ) = (I+A)-1(I−A) with skew-symmetric A. O(d log d) application cost.
3 Mixed-Radix Scheduling
Non-power-of-two dimensions (e.g., d=5120) decompose into mixed radices (8,8,8,5,2) via a greedy algorithm preferring radix-8. Optional Kronecker fallback for dimensions K · 2L.
4 Exact RHT Initialization (Theorem 4)
At θ=0, each Qt,c = I, so Bt,c = Hbt — the stride stages exactly recover the Hadamard stride factorization. HARP starts from RHT and only improves.
5 Lightweight Fitting
Adam (lr=3e-2, 1200 steps) on Lfit = Ldiag + λbd Rbd. Diagonal Hessian-weighted reconstruction proxy + off-block Hessian regularizer. Quantized targets via stopped gradient, avoiding nested LDLQ.

Strengths
  • Formal proof of exact RHT initialization equivalence (Theorem 4) — not heuristic.
  • Clean controlled experiment: isolates one component (processor) inside a fixed QuIP# backend.
  • Consistent gains across model sizes (1B–70B), bit-widths (2–4), and metrics (PPL + zero-shot).
  • O(d log d) runtime matches fixed Hadamard; int8 parameter storage nearly erases overhead.
  • HARP + no fine-tuning beats RHT + full fine-tuning (Table 12) — pure rotation improvement.
  • Honest diagnostics: reports that classical Hessian incoherence is NOT improved (Table 9).
Limitations
  • Calibration cost 4–10 GPU-hours; likely scales linearly with model size (70B+ expensive).
  • Llama-only evaluation; no Mistral, Qwen, MoE, or attention-free architectures tested.
  • Weight-only quantization; weight-activation PTQ compatibility unaddressed.
  • Missing head-to-head with ButterflyQuant (Xu et al., 2025), the closest competing method.
  • Effective BPP 2.05–2.11 vs nominal 2.00; tight memory budgets may feel the delta.
  • Requires precomputed Hessian/second-moment statistics (6144 RedPajama sequences).
  • At 4 bits all methods converge near FP16; HARP advantage diminishes to marginal.
Open Questions
  1. MoE compatibility — How does HARP interact with mixture-of-experts architectures where experts have very different weight distributions?
  2. Calibration efficiency — Can meta-learning or cross-layer weight-sharing reduce the per-layer 1200-step Adam fitting?
  3. Weight-activation quantization — Does the learned rotation help with activation outliers in W4A4 or W2A8 settings?
  4. ButterflyQuant head-to-head — Both use learnable butterfly orthogonal transforms but with different parameterizations; which wins in a controlled comparison?
  5. Adaptive block sizes — Can the Mixed-Radix structure adapt per-layer, using larger blocks for heterogeneous distributions?
  6. Multi-objective fitting — HARP does not improve classical Hessian incoherence; can a joint objective improve both quantizer alignment and generic incoherence?
