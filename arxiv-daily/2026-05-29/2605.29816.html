Key Finding

Perturbation-Induced Bias Is the Dominant Factor Degrading LLM Robustness

Edx[M(x + dx)] ≠ M(x)

Semantically neutral prompt perturbations cause a systematic expected shift in LLM module outputs — not random noise, but directional bias. This bias dominates robustness outcomes; reducing variance or Lipschitz constants alone is insufficient.

Zhou, Aleshina, Lovyagin, Somov, Seleznyov, Panchenko, Oseledets, Tutubalina, Tyukin · 2026-05-29

Why This Matters

A computationally cheap, no-retraining debiasing fix that provides formal robustness certificates for LLM deployments facing real-world prompt variability — a single-pass alternative to randomized smoothing.

!
Critical Edge

Debiasing consistently reduces clean-example performance. Certificates rely on conservative Chebyshev/Hoeffding bounds. Requires white-box access — no API-only deployment.

Core Insight

Under random semantically-neutral perturbations dx, the expected feature output shifts systematically away from the clean output. This perturbation-induced bias Ci dominates the robustness radius in both Lipschitz and non-Lipschitz settings.

εi(φi) = Ci + γi · εi · √(2n · ln(2m/φi))

Debiasing Strategy

Replace Mi(x) with M'i(x) = Mi(x) - bi

  • Input-independent: constant bias found by searching for optimal overlapping improvement regions across examples.
  • Input-dependent: per-input expected shift estimated from perturbation samples using population statistics.

No retraining. Single-pass inference. Applies to logits or intermediate layers (e.g., LoRA adapters at layer 8 mlp.up_proj).

Experimental Setup

  • Models: Qwen-3-8B, Llama-3.1-8B, Olmo-3-7B
  • Tasks: Natural Instructions (classification), NQ, GSM-8k, TriviaQA, XSum (generation)
  • Perturbations: format (structural template changes) and text (Unicode confusables, markup wrappers, keyboard typos)
  • Scale: 200 examples × 200 perturbations each, split 50/35/15

Certification Framework

Margin stability analysis combining performance and robustness into a single framework using Chebyshev and Hoeffding inequalities for:

  • Per-example guarantees (individual predictions)
  • Population-level guarantees (aggregate performance)

Certified examples achieve near 100% BAC on perturbed variants, exceeding the failure ratio φ=0.1.

Strengths

  • Strong theoretical grounding with clear practical implications — the bias concept elegantly explains why models fail under innocuous prompt changes
  • Computationally efficient: no retraining, single-pass inference (unlike randomized smoothing requiring multiple passes)
  • Provides formal robustness certificates, not just empirical improvements
  • Honest about limitations and trade-offs (clean performance reduction, scope limitations)
  • Unified framework subsuming batch calibration and spectral normalization insights under a single theoretical lens

Limitations

  • Evaluation primarily on classification via first-token prediction; generation evidence limited to a single table
  • Debiasing consistently reduces clean-example performance — the clean-vs-perturbed trade-off is inherent and unresolved
  • Certificates use conservative distribution-agnostic inequalities, potentially loose in practice
  • Requires white-box access to model weights, precluding API-only deployments
  • Non-adversarial perturbations only; method does not address adversarial attacks
  • Most effective in low-dimensional feature settings (4–5 principal components), raising scalability questions
  1. How does perturbation-induced bias scale with model size (70B+)? Does it become more or less pronounced?
  2. Can debiasing be combined with adversarial training to handle both non-adversarial and adversarial perturbations simultaneously?
  3. What is the relationship between perturbation-induced bias and in-context learning sensitivity? Could debiasing improve ICL stability?
  4. How tight are Chebyshev/Hoeffding certificates compared to empirical failure rates? Room for data-dependent tighter bounds?
  5. Can debiasing scale to long-form generation without excessive clean-performance degradation?
  6. Is perturbation-induced bias related to "typographic susceptibility" or other known LLM failure modes?
