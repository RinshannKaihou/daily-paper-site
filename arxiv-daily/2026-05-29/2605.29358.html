Why This Matters

First proof that dictionary learning scales to production LLMs, discovering abstract features like code semantics and deception — yet fundamental coverage gaps remain.

Critical Edge

The authors admit being “orders of magnitude short” of recovering all features, and effective steering requires clamping to 5–10× observed activations — raising questions about ecological validity.

Methodology

Standard ReLU-based sparse autoencoders trained on the middle-layer residual stream of Claude 3 Sonnet. Three scales: 1 M, 4 M, 34 M features. L1 coefficient of 5; L0 kept below 300 non-zero features per token.

  • Scaling laws — loss decreases as a power law with compute; optimal feature count grows faster than optimal step count; used to extrapolate hyperparameters for the 34 M run.
  • Feature evaluation — three-pronged: manual inspection, automated scoring (Claude 3 Opus, rubric 0–3), and causal steering by clamping activations to 5–10× their observed maximum.
  • Attribution — attribution predicts ablation with r = 0.81, validated as a cheap proxy for expensive ablation experiments.
  • Comparison baseline — SAE features vs. few-shot linear probes on interpretability and steering effectiveness.

Strengths

  • First convincing demonstration that dictionary learning scales to production LLMs
  • Exceptionally honest and detailed discussion of limitations
  • Rich qualitative evidence: code error features, function tracking, multilingual/multimodal generalization
  • Causal steering experiments confirm features capture real model computation
  • Scaling law analysis gives practical guidance for future SAE training
  • Clear practical advantage over few-shot probes (5/7 cases)
  • Attribution–ablation correlation (r = 0.81) validates a key methodological shortcut

Limitations

  • 65% dead features at 34 M — effective count is ~12 M; the headline is misleading
  • Incomplete coverage: likely “orders of magnitude short” of all features
  • L1 shrinkage systematically underestimates activations; attempted fixes worsened interpretability
  • No attention superposition: only residual stream analyzed
  • Subjective evaluation: core claims rest on cherry-picked examples
  • Steering at extreme values: 5–10× max may not reflect normal operation
  • Single layer of a proprietary model; not independently reproducible
  • No comparison to newer architectures (Gated SAE, JumpReLU SAE)

Questions for Further Exploration

  1. How do Gated SAEs, JumpReLU SAEs, and BatchTopK SAEs compare on dead-feature rates at this scale?
  2. What is the true “feature budget” to recover all features in a single layer — orders of magnitude beyond 34 M?
  3. Can Attribution SAEs improve data efficiency for learning rare features?
  4. How do features differ across layers — do earlier layers encode syntax and later layers semantics?
  5. Can feature splitting be predicted a priori for principled SAE sizing?
  6. How robust is feature steering under adversarial conditions?
  7. Can abstract features (function tracking, code errors) be decomposed into specific computational circuits?
  8. Are there universal features that transfer across model architectures?
  9. Can mixture-of-experts SAEs cheaply express many more features?
  10. What would a rigorous quantitative framework for feature “faithfulness” look like?
