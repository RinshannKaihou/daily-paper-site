KEY FINDING

On-Policy Replay for Continual Supervised Fine-Tuning

By routing on-policy signal through the replay buffer data source instead of auxiliary losses, OPR cuts catastrophic forgetting by 42-46% across three 7-8B LLMs -- without teacher models, without distillation, without modified training loops.

KEY METRIC

WHY THIS MATTERS
No teacher. No auxiliary loss. Just filtered self-generation replayed as SFT data.
Eliminates the engineering complexity of continual SFT. A label-free variant (OPR-SC) uses the model's own log-probability as scorer, requiring zero ground truth.
CRITICAL EDGE
Low-score on-policy replay is actively worse than off-policy gold data.
BWT -3.47 (low-score) vs -1.37 (Vanilla Replay). Proves the on-policy distribution -- not response quality -- is the active ingredient. Confidently wrong models bypass the filter.
1 METHODOLOGY -- OPR PIPELINE

Roll out on historical prompts

After training on task T_i, generate one response per historical prompt using the current checkpoint at temperature 0.1.

Score & filter

Score each pair: OPR-RU uses task metrics (EM, ROUGE-L); OPR-SC uses the model's own length-normalized log-probability -- no labels needed.

Build replay buffer

Keep the top-b pairs (budget ρ · n, distributed equally across prior tasks) as the replay buffer.

Standard SFT training

Concatenate buffer with next task's data and train under plain cross-entropy. No modified loss, no teacher forward pass, no schedule tuning.

KL-shrinkage theory: The high-reward filter ensures every buffer item has low cross-entropy at the previous checkpoint (l(theta_{t-1}) ≤ l_max). The expected replay gradient equals -nabla KL(q_{t-1} || pi_theta), pulling the model toward the prior high-reward region. This yields a per-update KL bound: E[KL] ≤ L · sigma_max · eta^2 · l_max, directly controlling forgetting through the filter threshold.

3 STRENGTHS & LIMITATIONS

Strengths

  • Extremely simple: no new losses, no teacher, no schedule -- just filtered self-generation replayed as SFT data
  • KL-shrinkage derivation unifies OPR with prior distillation methods on a single theoretical axis
  • Low-score ablation is a model of rigorous empirical work -- directly tests and confirms the mechanism
  • Loss dynamics analysis explains why SDFT fails on tasks requiring large distributional shifts
  • OPR-SC removes dependence on task evaluators at minimal performance cost
  • 32 GPU-hours total cost makes the method accessible to most labs

Limitations

  • OPR-SC degrades on tasks where the model is confidently wrong (NumGLUE-cm) -- a fundamental self-supervised scoring limit
  • Only evaluated on TRACE's 8-task English sequence; no validation above ~10B parameters or non-English settings
  • Rollout generation requires storing historical prompts -- problematic in privacy-sensitive deployments
  • Scorer quality dependency assumed but not deeply analyzed beyond OPR-RU vs OPR-SC
  • Does not address whether continual SFT is the right paradigm vs model merging or dynamic architectures
  • Per-update KL bound assumes L-smoothness and small step sizes; practical violations not discussed
4 OPEN QUESTIONS
Scaling
How does OPR behave with 50+ task sequences where historical prompt budgets become extremely constrained?
Parameter-efficient
Does the KL-shrinkage mechanism transfer when only a LoRA adapter subset is updated?
Learned scorer
Can an end-to-end learned scorer resolve the "confidently wrong" failure mode of OPR-SC?
Multimodal
Does OPR transfer to vision-language or code generation continual fine-tuning?
Streaming
Can OPR work without explicit task boundaries -- in a true streaming deployment?
Combination
Could combining OPR with regularization methods (EWC on replay parameters) yield further gains?
