SkillBrew: Multi-Objective Curation of Skill Banks for LLM Agents

A training-free framework that treats skill bank curation as constrained Pareto optimization -- utility as hard constraint, diversity and coverage as objectives -- solved via counterfactual leave-one-out rollouts in a bi-level propose-then-verify loop.

Authors Wentao Hu, Zhendong Chu, Yiming Zhang, Junda Wu, Ming Jin, Xiangyu Zhao, Yilei Shao, Yanfeng Wang, Qingsong Wen Date 2026-05-29 Curator GPT-5.4
Key Metric
59.0%
ALFWorld success rate — +9.7pp over Skill-Pro (49.3%). Transfers to GPT-4o at 88.1% and Gemini-2.5-Pro at 89.6%.
Why This Matters
First framework to treat agent skill banks as optimizable assets rather than append-only logs. Curated skills transfer across 5 different LLMs without re-curation -- the skills capture task-level procedural knowledge, not worker-specific quirks.
Critical Edge
All curation modules (Distiller, Diagnoser, Planner) run on GPT-5.4 -- expensive, proprietary, and quality likely degrades with weaker models. Evaluated only on text benchmarks (ALFWorld, WebShop); multimodal and open-world settings remain untested.

⚙ Methodology

Three cooperative modules inside a bi-level propose-then-verify loop (10 rounds):

  • Skill Distiller -- clusters no-retrieval failure trajectories, extracts candidate skills grounded in successful references.
    failure clusteringcandidate extraction
  • Skill Diagnoser -- counterfactual leave-one-out rollouts produce KEEP / REWRITE / REMOVE verdicts per skill.
    counterfactualcredit attribution
  • Edit Planner -- composes bank-level edit proposals into K coherent candidate banks, respecting hard constraints.
    bank-level reasoningPareto-aware

Formulation: B* = argmax(J_div, J_cov) s.t. J_util ≥ η

Retrieval: BM25 (0.30) + dense cosine (0.70), top-3 skills. Cache: SHA-256 content-addressed, 60%+ hit rate from round 2.

⚖ Strengths & Limitations

Strengths
  • Principled formulation: utility as hard constraint before optimizing diversity/coverage
  • Counterfactual leave-one-out cleanly solves confounded credit attribution
  • Edit Planner reasons at bank level, enabling coherent strategic evolution
  • Strong cross-worker transfer proves skills capture generalizable procedural knowledge
  • Full prompt templates, skill examples, and implementation details in appendix
Limitations
  • Computationally expensive: 10 rounds of support + query rollouts with counterfactuals
  • Depends on GPT-5.4 for all curation modules; quality degrades with weaker models
  • Only text benchmarks (ALFWorld, WebShop); multimodal / code agents untested
  • Frozen-worker assumption prevents agent-bank co-adaptation
  • Assumes static task distributions; continually shifting environments unaddressed

❔ Open Questions

How does SkillBrew perform when curation modules use a weaker or open-source LLM instead of GPT-5.4?
What is the marginal cost-benefit beyond 10 rounds? Can an early-stopping criterion be derived?
Can the framework be extended to jointly optimize the worker agent and skill bank (co-adaptation)?
How does it handle open-world distributions where new task types emerge continuously?
What is the sensitivity of the Pareto selector to the utility tolerance epsilon=0.03 and retrieval weights?
Can SkillBrew be applied to non-textual skill representations (e.g., executable code skills)?
