CSP-Atlas: Concept-Specific Neural Circuits in a Sparse Python Transformer

arXiv: 2605.24603

Date: 2026-05-23

https://arxiv.org/abs/2605.24603

一句话总结： 通过对稀疏8层Python Transformer的106个编程概念（43个AST节点+63个内置对象）提取神经回路，发现模型为每个概念都发展出专用回路，且内部组织跟踪计算结构而非语义类别——六个计算上原子性的构造（Import、Break等）形成"原子性超簇"，尽管它们在语义上完全无关。
TL;DR: This paper extracts neural circuits for 106 Python concepts from a sparse 8-layer code transformer using marginalisation across 63,800 controlled prompts, then decomposes each circuit into concept-specific vs. token-driven components via contrastive checker prompts. Three key findings: (1) all 106 concepts survive as non-empty universal circuits across all 9 parameter settings; (2) AST circuits contain genuine concept-only neurons (up to 62.5% at layer 5 for modular ASTs) distinct from token activation, while builtin circuits are almost entirely token-driven; (3) six semantically unrelated but computationally atomic constructs cluster together, revealing internal organisation tracks computational structure rather than meaning — organized in a four-tier hierarchy by token ambiguity and structural distinctiveness.

核心贡献

  • 提出了一套完整的回路提取与分解方法：通过跨互补维度的边际化（marginalisation）提取通用回路，再用对比性checker prompt将回路分解为概念特异性、共享、token驱动三个不交子集
  • 证明稀疏代码Transformer为所有测试的106个编程概念都发展出专用回路，且在9种参数设置下稳定存在，排除了阈值伪迹的可能性
  • 发现AST回路包含真正的概念特异性成分（modular AST在第5层高达62.5%为concept-only神经元），而builtin回路几乎完全由token驱动
  • 揭示了"原子性超簇"现象：Import、ImportFrom、Break、Continue、Pass、Assert这六个语义无关但计算上均为单语句、无嵌套体的构造紧密聚类
  • 提出四层表示层级：无token AST > 模块化关键词AST > 非模块化关键词AST > 内置对象，按token歧义性和结构独特性组织

Key Contributions

  • A complete extraction-and-decomposition pipeline: universal circuits via marginalisation across complementary dimensions, decomposed into concept-only / shared / token-only partitions using contrastive checker prompts
  • Parameter-stable concept survival: all 106 concepts produce non-empty universal circuits across all 9 (ε, C) parameter settings, ruling out threshold artifacts
  • Genuine concept-vs-token decomposition: AST circuits contain substantial concept-only neurons (up to 62.5% at L5 for modular ASTs at ε=0.5, C=80%), while builtin circuits are near-entirely token-driven
  • Discovery of the "atomicity super-cluster": six semantically unrelated single-statement constructs (Import, ImportFrom, Break, Continue, Pass, Assert) cluster together by shared computational property
  • A four-tier hierarchy of concept representation organized by token ambiguity and structural distinctiveness, not by semantic category

Methodology

The study uses the openai/circuit-sparsity model — a sparse 8-layer transformer trained on Python code with 2,048-dimensional MLP outputs containing genuine zeros. The concept space covers 43 AST node types × 63 builtin objects = 1,276 pairs, with 50 prompt variations each (63,800 total prompts). Prompts are structurally valid Python generated via ast.parse()/ast.unparse(), with variance injected along lexical, structural, and padding dimensions.

The extraction pipeline has four stages: (1) forward-pass activation extraction via hooks on MLP outputs at the last token position across all 8 layers; (2) binarisation using epsilon thresholding (|activation| > ε) and consistency filtering (neurons active in ≥ C% of prompts); (3) marginalisation — intersecting across the complementary dimension to obtain universal circuits; (4) decomposition — comparing universal masks against checker prompt masks (where the keyword token appears without its syntactic concept) to partition neurons into concept-only, shared, and token-only groups. A 3×3 parameter sweep (ε ∈ {0.001, 0.1, 0.5}, C ∈ {20%, 50%, 80%}) tests robustness.

Hierarchical clustering (Ward linkage on 1−Jaccard distance) over the 106 concept-only neuron sets reveals structural organisation. The methodology is language-agnostic — porting to another language requires only a concept-space definition and contrastive prompts.

Results & Highlights

  • 100% concept survival: All 106 concepts produce non-empty universal circuits at all 9 parameter settings; the ranking of concept-specificity across constructs is stable across settings.
  • Concept fraction peaks at layer 5: Modular AST concept fraction reaches 30.4% at L5 (ε=0.001, C=80%); layer 1 is essentially pure token signal.
  • AST vs builtin gap: Mean concept fraction for ASTs exceeds builtins by 3–6× at low-to-mid ε, and by an order of magnitude at ε=0.5 where builtin concept-only neurons vanish.
  • At ε=0.5, C=80%, L5: Modular AST circuits are 62.5% concept-only, non-modular ASTs 58.2%, builtins 0.0%.
  • Atomicity super-cluster: Import, ImportFrom, Break, Continue, Pass, Assert cluster together despite no semantic relationship — they share only the property of being single-statement constructs requiring no nested body.
  • Four-tier hierarchy: Tokenless ASTs (Tier 1) > Modular keyword ASTs (Tier 2) > Non-modular keyword ASTs (Tier 3) > Builtins (Tier 4), aligned with a gradient of token ambiguity and structural distinctiveness.

Authors & Credibility

Single author: Piotr Wilam, University College London. The study uses the publicly released openai/circuit-sparsity model with pinned revision SHA for reproducibility. All methodology, full decomposition data, and analysis code are released as open-source artifacts. The experimental design is rigorous — 63,800 controlled prompts, 9-parameter sweep, contrastive decomposition — and limitations are stated honestly (observational only, single model, single language).

Critical Assessment

Novelty: High. The combination of marginalisation-based circuit extraction with contrastive token-vs-concept decomposition is novel and well-motivated. The atomicity super-cluster finding is genuinely surprising and suggests non-trivial internal structure.

Rigor: Strong experimental design. The 3×3 parameter sweep convincingly rules out threshold artifacts. Checker prompts are validated to parse as valid Python, exclude the target concept, and contain the keyword token. The null hypothesis (random binary vectors) would collapse the intersection — a clean sanity check.

Reproducibility: Excellent. Open model, pinned SHA, released code and data, language-agnostic methodology.

Significance: This is a solid mechanistic interpretability contribution for code models. The finding that internal organisation tracks computational structure rather than semantics is important for understanding how code models represent programs. However, the scope is limited to one small sparse model — generalisation to dense production-scale models remains to be demonstrated.

Red flags: (1) No causal validation — concept-only neurons are identified by activation pattern but not ablated; some may be epiphenomenal. (2) Single model, single language limits generalisability. (3) MLP outputs only — attention patterns and internal MLP hidden layers are not examined. (4) Last-token-only extraction may miss distributed representations. (5) The 62.5% concept-only figure at the strictest setting involves very small absolute neuron counts (10 out of 16 for modular ASTs), making percentage-based claims somewhat fragile.

Strengths

  • Elegant experimental design: the checker prompt paradigm cleanly isolates concept from token processing
  • Comprehensive concept space (106 concepts) rather than cherry-picked examples
  • Parameter stability analysis provides strong evidence against threshold artifacts
  • The atomicity super-cluster is a genuinely novel and interpretable finding
  • Honest about limitations — explicitly flags observational nature and single-model scope
  • Language-agnostic methodology designed for cross-model and cross-language extension

Limitations

  • No causal interventions (ablation studies) to confirm functional relevance of concept-only neurons
  • Single small sparse model (8 layers, 2,048-dim MLP) — unclear whether findings transfer to dense, production-scale transformers
  • Python-only analysis; different programming languages may yield qualitatively different circuit structure
  • Last-token-position extraction may miss information distributed across positions
  • Checker prompt categories cover common non-structural contexts but are not exhaustive (missing docstrings, type annotations, f-strings)
  • Small absolute neuron counts at strictest settings make percentage-based claims fragile
