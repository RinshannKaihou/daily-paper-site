Mind the Gap: Can Frontier LLMs Pass a Standardized Office Proficiency Exam?

Tengchao Lv, Dongdong Zhang, Jiayu Ding, Yilin Jia, Yuzhong Zhao, Yupan Huang, Wenshan Wu, Xiangyang Zhou, Shaohan Huang, Nan Yang, Li Dong, Lei Cui, Furu Wei
LLM Evaluation Office Automation Agent Benchmark Code Generation NCRE
Published 2026-06-09 Source Microsoft Research Tasks 200 · 7,118 criteria
Key Metric
36.6%
Best single-turn score (Claude Opus 4.7). Best agentic system: 68.8% (Codex). Community reference: 95.5%.
Why This Matters

Real professional exams expose a critical gap in fine-grained Office automation that synthetic benchmarks miss — execution success reaches 98–99%, yet operation accuracy stays low. The semantic layer of Office work remains unsolved.

Critical Edge

Tasks are Chinese-centric, raw NCRE materials are copyrighted, and the engine requires Windows + Microsoft Office + COM automation — limiting portability and Western replication.

Methodology

  1. Data sourceNCRE Level 1 & 2 MS Office modules — basic formatting to mail merge, pivot tables, chart customization, and complex animations. Chinese instructions with visual reference images.
  2. Task formatEach task: input document (.docx/.xlsx/.pptx) + 5–15 sub-task instructions in natural language + a machine-readable XML scoring configuration.
  3. Evaluation engineParses Office Open XML (ISO/IEC 29500) via the Open XML SDK; some checks invoke Microsoft Office COM automation. Deterministic for a fixed environment.
  4. Two paradigmsSingle-turn: model outputs Python via python-docx / openpyxl / python-pptx. Coding agents: Claude Code & Codex operate in an isolated directory with the raw RTF and iterative execution feedback.
  5. MetricScore Rate (SR) = mean percentage of rubric points across tasks. 3 independent runs per model at temperature = 1.0.

Strengths

  • Externally validated benchmark grounded in a nationally administered certification exam
  • 7,118 deterministic criteria provide dense, fine-grained evaluation signals
  • Both single-turn and agentic paradigms evaluated on the same tasks
  • Clear, practically meaningful scoring anchor (NCRE pass/fail threshold)
  • Per-skill taxonomy enables diagnostic capability analysis
  • Honest assessment of limitations and data availability constraints

Limitations

  • Chinese-language task instructions may conflate language and Office automation capability
  • Raw NCRE materials are copyrighted and cannot be publicly redistributed
  • Evaluation requires Windows environment with Microsoft Office installed
  • Community reference solutions (95.5%) are informal — not a formal human baseline
  • Only two coding agents tested (Claude Code, Codex); other frameworks absent
  • python-docx / openpyxl / python-pptx have limited API coverage for animation & SmartArt, possibly depressing those category scores

Questions for Further Exploration

  1. How much of the performance gap is due to library limitations (e.g. python-pptx lacking animation APIs) versus genuine model capability limitations?
  2. Would multimodal models with direct screen interaction (clicking UI elements) outperform code-generation approaches on these tasks?
  3. How does the cross-language analysis (Chinese vs. English instructions) affect model performance — is the gap language-agnostic?
  4. Can the fine-grained criterion-level scoring be used to create targeted training data for improving Office automation capabilities?
  5. What is the performance ceiling for agents that can execute code iteratively with unlimited turns?
