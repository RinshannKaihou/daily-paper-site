Why it matters02

First systematic solution to dynamic black-box LLM provenance — directly applicable to agent-safety audits, API supply-chain verification, and model copyright protection.

⚠ Critical edge03

Closed-set only — and never tested against an adversarial operator who paraphrases outputs or routes through multiple backends. All 50 targets are open-weight; performance on GPT-4, Claude, or Gemini APIs is unknown.

§1Methodology 方法

The authors model a proxy LLM's hidden state as h = S(prompt) + Δs + a(author) + ε — separating prompt semantics from the target model's authorship signature. READER runs in two stages:

  • Temporal low-pass filter. Pass the black-box response through a frozen proxy LLM, sample M positions (default M=4 across the first 128 tokens), and average their hidden states into a single vector. Averaging kills decoding noise; the author signal survives.
  • Bayesian Evidence Accumulation (BEA). A multinomial logistic probe maps each response vector to per-candidate posteriors; for K independently prompted responses from the same target, sum their log-posteriors in decision space — S_c = (1/K) Σ log q(c|u_k) — then predict argmax of S_c.
  • Why decision-space aggregation. Geometric pooling of prompt-varying representations lets prompt semantics dominate; summing calibrated log-evidence sidesteps that failure mode.
  • Layer localization. Per-proxy layer sweep picks the architecture-optimal read-out — Llama-3.1-8B peaks at the final layer; Qwen proxies peak in the middle-to-late stack (e.g. layer 23/36 for Qwen3-8B).

§3Strengths 优势

  • Principled aggregation. Bayesian accumulation in decision space is theoretically grounded and beats geometric pooling of prompt-varying representations.
  • Truly black-box. Requires only generated text — no target weights, logits, gradients, or decoding internals.
  • Lightweight probe. Plain L2-regularized logistic regression per candidate set; minimal training compute.
  • Strong scaling behavior. Proxy capability (MMLU-Pro) tracks single-query attribution almost linearly (r = 0.942).
  • Operational honesty. Explicit about closed-set, single-source, and non-adaptive assumptions; PII redaction is tested.
  • Large, diverse benchmark. Agent500 spans 50 open-weight targets (3B–122B, dense + MoE) and 500 agent-style prompts.

§4Limitations 局限

  • Closed-set only. Cannot reject unknown / OOD models; probe must be retrained when the candidate set changes.
  • No adversarial evaluation. Paraphrasing, style obfuscation, and multi-backend routing are untested — a motivated provider could evade attribution.
  • Query budget. K = 50 per target is substantial; single-query accuracy (31–42%) is too low for many real audits.
  • No closed-source APIs. All 50 targets are open-weight; performance on GPT-4, Claude, or Gemini is unknown.
  • Compute overhead. Running a proxy LLM (8B–122B) for feature extraction is non-trivial per audit.
  • Single-source assumption. Multi-backend agent settings (sub-steps routed to different providers) are not modeled.

§5Questions for further exploration 延伸问题

  1. How does READER perform on closed-source API targets (GPT-4o, Claude, Gemini) whose internal representations may differ more dramatically from open-weight models?
  2. Can an open-world rejection mechanism — e.g. confidence thresholding on calibrated αS — reliably filter unknown models?
  3. Will the proxy-scaling trend continue with frontier models, or is there a ceiling to exposed authorship structure?
  4. What is the minimum viable proxy size for practical deployment — and can distilled proxies retain attribution power?
  5. Could active prompt selection (vs. random sampling) cut the K=50 budget needed for high-confidence attribution?
