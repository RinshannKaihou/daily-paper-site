> **一句话总结：** 本文研究在标准高斯分布下对带偏置的一般 ReLU 的 agnostic 回归（平方损失），在交互式（query）学习设定下给出首个计算高效算法，仅需 d·polylog(R²/ε)+Õ(min{1/p, R²/ε}) 次黑盒标签查询即达 O(opt)+ε 误差（p 为最优 ReLU 的偏置），并证明该查询复杂度定性近最优，且 pool-based 主动学习在样本池不超多项式大小时无法达到同等标签复杂度。

> **TL;DR:** This is a pure theory paper (no datasets/baselines/trained models). Compared to the prior passive learners of Guo-Vijayaraghavan (COLT 2025, [GV25]) and Zarifis et al. (COLT 2025, [ZWDD25]), which need poly(d,1/ε) labeled examples and a poly(R/ε)-size parameter grid, the new query algorithm achieves the same O(opt)+ε error with only d·polylog(R²/ε)+Õ(min{1/p,R²/ε}) label queries and a polylog(R/ε)-size grid — an exponential (logarithmic-in-1/ε) saving in label complexity. It is matched by a query lower bound of Ω̃(1/p^{1−o_d(1)}+d) (Theorem 4.1) and a pool-based active-learning lower bound of Ω̃(d/(p·log m)) labels (Theorem 4.2), the latter showing query access is essentially necessary to beat the passive Θ̃(d/ε) rate.

## 核心贡献

- **首个计算高效的交互式学习器**：将先前仅适用于同质 ReLU（t≤0）或被动设定的工作推广到带任意偏置的一般 ReLU，并在 query 设定下保持多项式时间可计算。
- **近最优查询复杂度**：仅需 d·polylog(R²/ε)+Õ(min{1/p, R²/ε}) 次黑盒标签查询即达 O(opt)+ε 误差，相比被动学习的 poly(d,1/ε) 标签在 1/ε 维度上实现指数级（对数级）节省；并将参数网格从 poly(R/ε) 缩小到 polylog(R/ε)。
- **匹配的查询下界**：证明任何输出误差小于 O(p/log²p) 的算法至少需要 Ω̃(1/p^{1−o_d(1)}+d) 次查询，即使 opt≤2^{−Ω(d^{1/4})}p，说明上界在定性意义上近最优。
- **query vs pool-based active learning 的分离**：证明 pool-based 主动学习器需要 Ω̃(d/(p·log m)) 个标签（除非抽取超多项式 2^{Ω(d)} 个无标签样本），形式化地说明"交互式 query 访问"（而非从池中主动采样）才是改进标签复杂度的关键。

## Key Contributions

- **First efficient interactive learner for general (biased) ReLUs.** Extends the recent passive-learner line ([GV25, ZWDD25], both COLT 2025) to non-homogeneous ReLUs in the query-access model while remaining poly-time.
- **Near-optimal query complexity.** Only M=d·polylog(R²/ε)+Õ(min{1/p, R²/ε}) black-box label queries suffice for the agnostic O(opt)+ε guarantee — an exponential (logarithmic-in-1/ε) saving over passive learning's poly(d,1/ε) labels, and over the homogeneous t≤0 case's dpolylog(1/ε) labels it adds the bias-dependent Õ(min{1/p,R²/ε}) term.
- **Matching qualitative query lower bound.** Any learner outputting a hypothesis with error < O(p/log²p) with probability 1/3 needs Ω̃(1/p^{1−o_d(1)}+d) queries, even when opt≤2^{−Ω(d^{1/4})}p (Theorem 4.1) — information-theoretically near-tight.
- **Sharp separation from pool-based active learning.** Any pool-based active learner needs Ω̃(d/(p·log m)) label queries from a pool of m unlabeled Gaussian points, even in the realizable setting (Theorem 4.2); unless m≥2^{Ω(d)}, this rules out matching the query learner's label complexity — formalizing that *interactive query* access, not generic active sampling, is what unlocks the gain.

## Methodology

The paper is purely theoretical (no experiments, no code). It studies agnostic ReLU regression under Gaussian marginals D_x=N(0,I) with squared loss err(h)=E[(h(x)−y(x))²/2], where a black-box query takes any x∈R^d and returns a label y∼y(x) generated adversarially. The target is the best-fit ReLU σ*(x)=σ(W*·x−t*) with ‖W*‖≤R, opt its loss, and p=Φ(t̄*) its bias (fraction of mass with non-zero label, where t̄*=t*/‖W*‖ is the normalized threshold).

The analysis has three parts:

1. **Warm-up — spherical GLM with known t* (§2).** With ‖W*‖=1 and t* known, the authors use a simple *projected gradient descent* on the sphere, w ← proj_{S^{d-1}}(w − μ·proj_{w⊥}∇_w err(w)) (eq. 1), in place of the complicated update rules of [GV25]. The key new tool is Lemma 2.1, which gives the noiseless error as an integral ℓ(w)=∫_0^θ sin s·‖T_{√(cos s)}σ'‖₂² ds via a connection to the *Ornstein–Uhlenbeck semigroup*. Lemmas 2.2–2.5 show that given a warm start w₀ with ‖σ'‖₂²/‖T_{√(cos θ₀)}σ'‖₂²=O(1) (i.e. θ₀=O(1/t*), Lemma 2.6), each contraction reduces sin(θ_{i+1}/2)≤(1−c/20)² sin(θ_i/2) (Lemma 2.4), reaching error O(ε). To *find* the warm start, Lemma 2.7 turns the real-valued label into a corrupted halfspace label ȳ(x)=1{y(x)>c/t*} and reuses the halfspace-init of [DKM24] (Lemma 2.8) using Õ(1/p+d·log(1/ε)) queries. The hypothesis-selection caveat (the [DKM24] init succeeds only w.p. 1/log(1/ε)) is handled by a new *active hypothesis tester* (Lemma 2.10) that reweights D_x by (h_i−h_j)² to cut variance and selects the best of k hypotheses in poly(k) queries.

2. **General case — unknown (‖W*‖, t*) (§3).** The naïve reduction (grid (‖W*‖,t*) to poly(ε/R) error, run §2's algorithm poly(R/ε) times) "completely ruins the query complexity." Instead the authors build a *two-level non-uniform grid* of size only polylog(R/ε): first level r_i=2^{i−1}√ε (log(R²/ε) points), each interval [r_i,r_{i+1}] subdivided into polylog(R/ε) uniform points; t̄ gridded as t^{(i)}=(i−1)/polylog(R/ε). This guarantees one grid point with r≤r*≤2r, |t̄−t̄*|≤1/log(R²/ε), which Lemma 3.1 shows suffices for warm start w₀ with sin(θ₀/2)≤min{1/t̄*,1/2}. The crux is that ‖W−W*‖ is *not* the right noiseless-error measure when t* is large (h1=σ(v·x−t*) and h2=(t*ξ+1)σ(w*·x−t*) have identical noiseless error but very different parameter distance), so Algorithm 1 *decomposes* the update into a (r,t) update and a separate angle-w update. Lemma 3.2 gives the noiseless-error decomposition ℓ(r,w,t) ≤ (r*)²∫sin s·‖T_{√(cos s)}σ'(z−t̄*)‖₂²ds + E[(σ(rz−t)−σ(r*z−t*))²]. The angle gradient G_i is boosted by rejection-sampling x∼N(0,I)|w·x>t̄_i (Lemmas 3.3–3.4); the (r,t) gradient uses statistics U_i,F_i that approximate the gradient of the quadratic Z(r,t)=E[((rz−t)−(r*z−t*))²1(z>t̄*)]. Because Z(r,t) is ill-conditioned for large t*, the final (r_T,t_T) is only O(ε·polylog(R²/ε)/Φ(t̄*))-close in squared norm, so the algorithm randomly picks (r̂,t̂) from a unit grid around (r_T,t_T) (line 15).

3. **Lower bounds (§4).** The query lower bound (Theorem 4.1) splits into an Ω(d) part (a Bayesian linear-regression argument: after r queries spanning subspace L, E‖proj_L w*‖²=r/d, so <Ω(d) queries force Ω(1) error) and a 1/p^{1−o_d(1)} part (distinguishing σ(w*·x−t*) from 0; with adversarial zeroing of far-from-zero labels, only a p^{1−o_d(1)}-volume cap has non-zero label). The pool-based active-learning lower bound (Theorem 4.2) uses Yao's minimax: a deterministic algorithm's behavior over a pool of m points is a path; via a Gram-Schmidt basis change each realization maps to a vector v(w_L) whose density equals that of the first k coordinates of a random unit vector, and near-orthogonality of pool tuples (when m≪2^{Ω(d)}) bounds the success probability by O(p·log(1/p))^k, forcing r=Ω̃(d/(p·log m)) queries.

## Results & Highlights

The paper reports no empirical experiments; its quantitative claims are theorem statements (complexity bounds). Three concrete ones:

1. **Upper bound (Theorem 1.1 / 3.1):** For general agnostic ReLU regression under Gaussian marginals, there is an algorithm making **M=Õ(min{1/p, R²/ε}+d·polylog(R²/ε))** queries, running in **poly(M) time**, that outputs ĥ with err(ĥ)≤**O(opt)+ε** w.p. ≥1−δ, where p=Φ(t̄*) is the bias of the optimal activation. Versus the passive baseline [GV25, ZWDD25] needing **poly(d,1/ε)** labeled examples, this is an exponential (logarithmic) saving in 1/ε; versus the t≤0 homogeneous result [DKTZ22a, WZDD23] of **d·polylog(1/ε)**, it adds the bias term Õ(min{1/p,R²/ε}).
2. **Query lower bound (Theorem 1.2 / 4.1):** Even with opt≤2^{−Ω(d^{1/4})}p and ‖W*‖=1, any algorithm outputting a hypothesis with error < **O(p/log²p)** w.p. 1/3 needs **Ω̃(1/p^{1−o_d(1)}+d)** queries — qualitatively matching the Õ(min{1/p,…}) bias term and the d term of the upper bound.
3. **Pool-based active-learning lower bound (Theorem 1.3 / 4.2):** In the *realizable* setting with optimal ReLU bias p, any pool-based active learner drawing m i.i.d. points from N(0,I) that makes fewer than **Õ(d/(p·log m))** label queries outputs, w.p. ≥2/3, a hypothesis with error **Ω̃(p)**. Hence to match the query learner's d·polylog+Õ(1/p) labels a pool-based learner would need m≥2^{Ω(d)} (super-polynomial), establishing that interactive query access is essentially necessary to beat the passive Θ̃(d/ε) label rate.

A supporting structural fact: the noiseless error has the OU-semigroup integral form ℓ(w)=∫_0^θ sin s·‖T_{√(cos s)}σ'‖₂²ds ≤ (π/2)sin²θ·‖σ'‖₂² (Lemma 2.1), and for σ(z)=ReLU(z−t*), the warm-start requirement is θ₀=O(1/t*) (Lemma 2.6, since ‖T_{√(cos θ)}σ'‖₂²≥‖σ'‖₂²/50 when sin(θ/2)≤1/t*).

## Critical Assessment

- **The bias term is not known a priori.** The query bound M=…+Õ(min{1/p,R²/ε}) depends on p=Φ(t̄*), which the learner does not receive as input; the algorithm must instead grid t̄ (Lemma 3.1 uses an oracle giving |t̄−t̄*|≤1/log(R²/ε), and the main algorithm self-provides this via the polylog grid). The paper does not separately analyze the cost of *estimating* p when it is entirely unknown, so the headline Õ(1/p) term implicitly assumes the grid search finds the right scale.
- **Polynomial runtime hides exponents.** Theorem 1.1 states "runs in poly(M) time" without specifying the degree; the angle update uses median-of-means with Õ(d) queries per round over T rounds, and the hypothesis tester is poly(k) in k=polylog candidates — the actual constant could be large, and the paper gives no runtime comparison to the passive baselines beyond the asymptotic label-count saving.
- **Initialization succeeds only with probability 1/log(1/ε).** The halfspace-init of [DKM24] (Lemma 2.8) succeeds w.p. only 1/log(1/ε) in the worst case, forcing polylog(1/ε) restarts; the authors patch this with their active hypothesis tester (Lemma 2.10), but this adds a polylog factor and a list-selection step that is itself only analyzed for fixed (r,t).
- **Distributional restriction is essential.** All results require D_x=N(0,I); the hardness backdrop ([DKMR22] intractability without marginal assumptions; [DKZ20, DKPZ21, DKR23] SQ/crypto hardness even under Gaussian) is invoked to justify the O(opt)+ε target, but the paper does not test robustness to log-concave or misspecified marginals.
- **Single ReLU, squared loss only.** The concept class is one general ReLU; the remark that the techniques "can be used to analyze a variety of generalized linear models" (after Theorem 1.1) is asserted, not demonstrated for multi-neuron or deep nets.

## Strengths

- Clean, complete theory: matching upper bound plus two complementary lower bounds (query and pool-based), resolving the label complexity of general ReLU regression under Gaussian marginals.
- Conceptually sharp message — *interactive query* access (querying any x∈R^d), not generic pool-based active learning, is what beats passive Θ̃(d/ε); the separation theorem (Theorem 4.2) is the paper's most novel conceptual contribution.
- Replaces the complicated update rules of [GV25] with simple spherical projected gradient descent (eq. 1), analyzed via a clean Ornstein–Uhlenbeck integral (Lemma 2.1) — a reusable tool for broader GLMs.
- Shrinks the parameter grid from poly(R/ε) (all prior work) to polylog(R/ε) via a two-level non-uniform grid, a non-trivial algorithmic economy.

## Limitations

- Pure theory: no datasets, no trained models, no empirical validation that the query savings materialize in practice.
- Restricted to Gaussian marginals and squared loss; behavior under log-concave/misspecified distributions unaddressed.
- Single general ReLU only; no extension to multi-neuron or shallow networks demonstrated.
- Asymptotic polynomial-time with unspecified exponents; the bias p must be (approximately) found via griding, adding hidden factors.

## Questions for Further Exploration

- Can the bias-dependent term Õ(min{1/p,R²/ε}) be removed or made independent of the unknown p, e.g. by an adaptive scheme that estimates p online rather than griding t̄ (the paper notes Lemma 3.1 needs |t̄−t̄*|≤1/log(R²/ε) as a warm-start oracle)?
- Does the Ω̃(d/(p·log m)) pool-based lower bound (Theorem 4.2) extend to weaker active models such as stream-based active learning [DMRT24], which the paper cites?
- The authors claim (after Theorem 1.1) the projected-GD + OU-semigroup framework "can be used to analyze a variety of generalized linear models" — does it extend to multi-neuron or two-layer ReLU networks under Gaussian marginals, where [DKR23] establishes hardness?
- How do the guarantees degrade under approximate-Gaussian or log-concave marginals, given the hardness results of [DKMR22] for arbitrary D_x?
- The ill-conditioning of Z(r,t) for large t* forces a random (r̂,t̂) pick (Algorithm 1, line 15): is there a deterministic refinement that removes this randomization and the associated polylog factor in the error?
