Methodology01 / 05

How a discrete VQ codebook becomes a differentiable linear map

For each weight group of size d, LC-QAT parameterizes every codeword with a tiny affine map over a quaternary integer vector — no explicit codebook table, no nearest-neighbor search.

c = Az + B,    z ∈ {0,1,2,3}d,   A ∈ ℝd×d,   B ∈ ℝd
Forward Pass
  1. PTQ init via LDLQ + Hadamard transform (outlier removal, QuIP#-style).
  2. Float proxy Wp → round + clip → integer Wz.
  3. Decode via Ŵ = [A·WzT + B·1T]; reformulated to reuse INT2×FP16 MatMul kernels.
  4. Group size fixed at d = 4 → 256 codewords, only 20 extra params per weight matrix.
Gradient Estimator
  1. Replaces straight-through estimator (STE) with a smooth DGE:
    f′(x) = (1/k)·(2x/δ − 1)1/k − 1
  2. Hyperparams k = 5, δ = 1 — empirically stabilizes training near local minima.
  3. Preprocess Wp = s(Wz − t) to match Xavier statistics before each step.

Strengths03 / 05

  • Elegant math. Linear-constrained codebook c = Az + B cleanly dissolves the VQ non-differentiability problem.
  • Strong empirics. Consistent gains across Qwen-3-1.7B/8B and LLaMA-3-3B/8B; better data-scaling than PV-Tuning.
  • Data efficiency. 4B tokens competitive with methods using 10×–1000× more data.
  • Deployment-ready. Forward reformulation reuses INT2×FP16 GEMM kernels; custom CUDA kernel shipped.
  • Ablations + loss landscape. Each component (PTQ init, preprocessing, DGE, d) justified separately.

Limitations04 / 05

  • Weight-only. Activations stay in float — no end-to-end INT2 inference pipeline.
  • PPL ↔ accuracy gap. LLaMA-3-8B: better zero-shot, worse PPL than ParetoQ. Not analyzed.
  • Linear-projection overhead. Per-layer O(Nnd) cost; interaction with optimized GEMM not benchmarked.
  • Scale unproven. Behavior above 10B training tokens is explicitly an open question.
  • 2-bit only. Not validated at 3-/4-bit where scalar quantization may catch up.
  • Compute floor. 16× A100 GPUs required for the main runs.

Questions for Further Exploration05 / 05

  • Can the linear-constrained codebook extend to activation quantization for full INT2 inference?
  • What is the Pareto frontier of group size d vs accuracy vs speed — is d = 4 truly optimal?
  • How does LC-QAT interact with structured pruning or mixture-of-experts architectures?
  • Does the PPL-vs-accuracy gap on LLaMA-3-8B mean LC-QAT learns different capabilities than SQ-QAT?
  • At 3-bit or 4-bit precision, does VQ still beat scalar quantization?
  • Can the affine map c = Az + B be made low-rank to cut the per-layer parameter overhead?
