What Fits (Into Few Tokens) Doesn't Overfit: Compression and Generalization in ML Research Agents

Martin Andres Bertran  ·  Aaron Roth  ·  Zhiwei Steven Wu
ml-research-agents generalization information-bottleneck description-length benchmark-overfitting

Key finding Successful ML strategies live in a low-complexity region — extremely short prompts and one-bit feedback suffice to reproduce them across 8 datasets. 一句话总结：用两类信息瓶颈（极短提示词复现模型 + 一位反馈信号驱动探索）在 8 个跨领域数据集上验证了"可压缩的 ML 策略不易过拟合"——成功的 ML 策略占据一个低复杂度区域,因此即使自适应地复用验证集也很少产生过拟合。

Key metric
8
datasets across 5 modalities
Tabular classification, vision, language modeling, diffusion, and reward modeling — both output compression (tiny reproducer prompt) and input compression (one-bit "improved over running best" feedback) leave performance nearly intact.
Why this matters
Reframes a decades-old puzzle — why benchmark-driven ML rarely overfits despite adaptive validation-set reuse — as a falsifiable compressibility claim grounded in description length.
Critical edge
The falsification control works (induced overfitting breaks short-prompt reproduction), but the result hinges on a specific LLM and agent harness whose code and prompts are not yet released.

01Methodology

An LLM-based exploration agent iteratively proposes model configurations (architecture, hyperparameters, training recipes) evaluated on a held-out validation set. A separate reproducer agent then attempts to match that performance under a deliberately tightened description budget. Two complementary bottlenecks define the experiment:

Output compression

Can a fresh agent, given only a heavily truncated description of the explorer's winning trajectory plus the training data, reproduce the high-performance model?

Input compression

Replace rich per-step feedback with a single bit — "did this submission beat the running best?" — and ask whether the explorer can still find the same solution.

Falsification control

Deliberately overfit a tiny validation set with many adaptive trials, then run the same short-prompt reproducer. If low description length ↔ generalization, reproduction should fail.

Authors & credibility. A theoretically grounded group: Aaron Roth (UPenn) and Zhiwei Steven Wu (CMU) are leading learning theorists with direct expertise in adaptive data analysis, the theoretical backbone this paper builds on. Martin Andres Bertran works on ML theory and trustworthy ML.

03Strengths

  • Clean, falsifiable hypothesis with an explicit negative control rather than a post-hoc rationalization.
  • Broad empirical coverage — 8 datasets spanning 5 modalities, not a single benchmark cherry-pick.
  • Bridges two literatures: classical description-length / adaptive data analysis theory meets modern LLM-agent AutoML.
  • Honest framing — the authors position the claim as falsifiable and demonstrate the falsification path themselves.

04Limitations

  • Agent-harness dependence. Conclusions hinge on a specific LLM and prompt scaffolding; results may shift with the underlying model.
  • Informal complexity measure. The mapping from prompt-token length to bit-description length is intuitive, not formal.
  • Not the adversarial setting. Truly adversarial benchmark reuse — per-instance test leakage, label-mined leaderboards — is outside scope.
  • Reproducibility gated on release. One-bit feedback and short-prompt reproduction need code + prompts to be verifiable.

05Questions for further exploration

  1. How tightly does prompt-token length track Kolmogorov / description length? Could a fixed-grammar minimum-description measure sharpen the claim?
  2. Does the result survive when the validation set is reused thousands of times by an adaptive agent with full per-step gradients, not just scalar feedback?
  3. Can the framework predict which benchmarks are at risk of overfitting (small, heavily-mined leaderboards vs. large, fresh ones)?
  4. How does the finding interact with test-set contamination concerns for the LLM-based reproducer itself?
