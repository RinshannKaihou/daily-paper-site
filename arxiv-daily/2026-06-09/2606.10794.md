> **一句话总结：** READER 利用冻结的代理 LLM 作为"阅读器"，从黑箱 LLM 输出中提取隐藏层激活表征，经时间低通滤波后通过贝叶斯证据累积跨查询聚合溯源证据，在 Agent500（50 个目标模型）上单条响应即达 31%–42.4% top-1 准确率，50 条响应达 70%–84%，大幅超越句子编码器指纹基线。

> **TL;DR:** READER identifies which LLM generated a black-box response by reading it through a frozen proxy LLM, temporally filtering hidden-state activations, and accumulating calibrated log-posterior evidence across multiple queries — achieving 70–84% top-1 accuracy on a 50-target benchmark with 50 responses, far above sentence-encoder baselines (≤18.8%).

## 核心贡献

- 首次提出**动态黑箱 LLM 溯源**（Dynamic Black-Box LLM Provenance）问题设定：在查询内容不可预定义、语义多变的真实 API 审计场景下识别源 LLM，而非依赖固定探针集。
- 提出 **READER** 框架，将冻结代理 LLM 作为"阅读器"解码黑箱输出中的作者身份证据，仅训练一个轻量线性探针，无需访问目标模型的权重/梯度/logits。
- 引入**贝叶斯证据累积**（Bayesian Evidence Accumulation）：在决策空间而非表征空间聚合多条独立查询的后验对数证据，避免脆弱的几何均值池化。
- 构建 **Agent500** 基准：50 个目标 LLM（Llama、Qwen、Mistral、Phi、Gemma、DeepSeek 等，3B–122B 参数，含稠密和 MoE），500 条 agent 风格提示。
- 揭示代理 LLM 能力与溯源性能强相关（Pearson r = 0.942），表明更强的 LLM 表征中蕴含更丰富的线性可解码作者身份结构。

## Key Contributions

- **Dynamic Black-Box LLM Provenance**: First formulation of provenance under query-varying, non-predefined prompts — the realistic API auditing setting — instantiated with Agent500 (50 targets, 500 agent-style prompts).
- **READER framework**: A frozen proxy LLM reads black-box outputs; a lightweight linear probe maps proxy hidden states to per-response posteriors. No target-model access required (weights, logits, gradients, decoding internals all off-limits).
- **Bayesian Evidence Accumulation (BEA)**: Summing calibrated log-posterior evidence across K independently prompted responses in decision space, avoiding fragile geometric pooling of prompt-varying representations.
- **Agent500 benchmark**: 50 target LLMs spanning Llama/Qwen/Mistral/Phi/Gemma/DeepSeek families (3B–122B, dense + MoE), 500 agent-style prompts, 25,000 trajectories.
- **Proxy scaling law**: Strong correlation between proxy capability (MMLU-Pro) and single-query attribution accuracy (Pearson r = 0.942), suggesting authorship perception emerges naturally in capable LLM representations.

## Methodology

READER models proxy hidden states as: **h_t^(c,p) = S(p) + Δs_t^(p) + a(c) + ε_t**, where S(p) is prompt semantics, Δs_t is local contextual drift, a(c) is the target-model authorship signature, and ε_t is decoding noise. The framework operates in two stages:

### Stage 1: Temporal Low-Pass Filtering
- Feed the black-box response text into a frozen proxy LLM and extract hidden states from a selected layer.
- Sample M positions uniformly from the response (default M=4 over first 128 tokens) and arithmetic-mean them into a single-response vector u^(c,p).
- This averaging acts as a temporal low-pass filter, reducing high-frequency decoding noise while preserving authorship signal.

### Stage 2: Bayesian Evidence Accumulation (BEA)
- Train a multinomial logistic regression (L2-regularized) probe q_θ(c|u) on single-response examples.
- For K independently prompted responses from the same unknown target, accumulate evidence: **S_c = (1/K) Σ_k log q_θ(c|u_k)**.
- Prediction: **ŷ = argmax_c S_c**. For calibrated confidence, apply softmax(αS) with validation-fitted scalar α.
- Key insight: aggregating in decision space (log-posteriors) rather than representation space avoids the semantic dominance problem.

### Layer Selection
- Per-proxy layer sweep at M=1, K=1 reveals architecture-dependent optimal layers: Llama-3.1-8B peaks at the final layer; Qwen proxies peak in the middle-to-late stack (e.g., layer 23/36 for Qwen3-8B, 50/64 for Qwen3-32B).

### Experimental Protocol
- 5-fold cross-validation at prompt level (disjoint prompts across train/test for every target).
- Query budgets K ∈ {1, 5, 10, 20, 50}.
- Baselines: three LLM-DNA-style sentence encoders (MPNet, BGE-large, Qwen3-Embedding-8B) with identical downstream pipeline.

## Results & Highlights

| Metric | READER (best proxy, K=50) | Best sentence-encoder (K=50) | Random |
|--------|---------------------------|------------------------------|--------|
| Top-1 Accuracy | **84.0%** (Qwen3-32B) | 18.8% (BGE-large) | 2.0% |
| Top-1 Accuracy (K=1) | **42.4%** (Qwen3.5-9B) | 11.9% (BGE-large) | 2.0% |
| Pair-AUC (K=50) | 0.995 | 0.958 | 0.500 |
| mAP@10 (K=50) | 0.796 (Qwen3-32B) | 0.319 | 0.020 |

- **Temporal filtering saturates early**: M=4 is competitive with M=8/16, confirming a few evenly spaced states suffice.
- **Confusion analysis**: Residual errors concentrate within related checkpoints (Qwen3, Qwen2.5, DeepSeek families share nearby base weights). Llama family is harder under the Llama-3.1-8B proxy but improves with Qwen proxies.
- **PII redaction robustness**: At 50% word redaction with K=50, proxies retain 54.8–62.4% top-1 accuracy (vs. 70.0–84.0% unmasked). Multi-query accumulation makes degradation more graceful.
- **9-proxy scaling**: Across proxies from 8B to 122B parameters, single-query accuracy tracks MMLU-Pro score with Pearson r = 0.942, Spearman ρ = 0.917.

## Critical Assessment

- **Novelty**: High. The proxy-reader + BEA combination is original. The problem formulation (dynamic vs. static black-box provenance) clarifies an important distinction overlooked by prior work. The proxy-scaling observation (stronger readers expose more authorship structure) is a genuine empirical finding.
- **Rigor**: Strong. 50-target benchmark with diverse families, prompt-level cross-validation, ablations over M×K, layer localization, and PII robustness. Confusion matrices honestly show where errors concentrate (within-family checkpoints). The paper is transparent about the closed-set assumption.
- **Reproducibility**: Moderate-high. Method is simple (logistic regression probe + frozen proxy). Implementation details (λ_LR, layer choices, M, K) are specified. Agent500 prompt corpus needs to be released for full reproducibility.
- **Significance**: High for the LLM safety/provenance community. Non-intrusive API auditing is an emerging operational need as agentic systems proliferate.
- **Red flags**: (1) Closed-set assumption is a real limitation — no open-world rejection mechanism. (2) Adversarial robustness is untested — a motivated provider could paraphrase or route through multiple backends. (3) 50 responses per target is a substantial query budget that may not be available in all settings. (4) The paper does not test on closed-source API targets (GPT-4, Claude, etc.).

## Strengths

- Principled Bayesian aggregation in decision space — theoretically grounded and empirically superior to geometric pooling.
- Truly black-box: requires only generated text, no target internals.
- Lightweight probe training (logistic regression) — minimal compute overhead per new candidate set.
- Strong scaling behavior: proxy capability tightly correlates with attribution power.
- Honest about limitations: the paper explicitly discusses closed-set, single-source, and non-adaptive assumptions.
- PII redaction robustness analysis addresses a realistic deployment concern.

## Limitations

- **Closed-set only**: Cannot detect unknown/out-of-distribution models; requires retraining the probe when candidates change.
- **No adversarial evaluation**: Does not test against deliberate style obfuscation, paraphrasing, or multi-backend routing by an adversarial provider.
- **Query budget**: 50 responses per target is high; single-response accuracy (31–42%) may be insufficient for many practical scenarios.
- **Single-source assumption**: Cannot handle multi-source settings where agents route sub-steps through different backends.
- **No closed-source API evaluation**: All 50 targets are open-weight models; performance on GPT-4/Claude/Gemini APIs is unknown.
- **Compute overhead**: Running a proxy LLM (8B–122B parameters) for feature extraction adds non-trivial inference cost.

## Relevance

READER 首次系统性地解决了动态黑箱场景下的 LLM 溯源问题，对 Agent 安全审计、API 供应链验证和模型版权保护均有直接应用价值。其发现——代理 LLM 表征中自然蕴含作者身份信号——也为理解 LLM 内部表征提供了新的实证视角。

## Questions for Further Exploration

1. How does READER perform on closed-source API targets (GPT-4o, Claude, Gemini) where internal representations may differ more dramatically from open-weight models?
2. Can an open-world rejection mechanism (e.g., confidence thresholding on calibrated αS) reliably filter unknown models?
3. How resilient is the proxy-scaling trend — will future frontier models continue to expose more authorship structure, or is there a ceiling?
4. What is the minimum viable proxy size for practical deployment, and can distilled proxies retain sufficient attribution power?
5. Could active prompt selection (rather than random sampling) reduce the query budget needed for high confidence?
