Key Finding

A controlled L0–L6 prompt ladder exposes four distinct context behaviours in AudioLLMs — balanced, sensitive, context-blind, unstable — across 8 Indic languages.

IndicContextEval: A Benchmark for Evaluating Context Utilisation in Audio Large Language Models Across 8 Indic Languages

Sakshi Joshi · Dhruv Subhash Rathi · Sanskar Singh · Eldho Ittan George · R J Hari · Kaushal Bhogale · Mitesh M. Khapra

Audio LLM Context Utilisation Multilingual Benchmark Indic Languages Speech Recognition

Key Metric

17.39%

Lowest entity error (NEER) reached

Gemini 3 Flash @ L5 — native-script entity lists cut GPT-4o's NEER by up to −11.7 pts.

Why This Matters

Production contextual-ASR has no way to verify prompts are truly used. IndicContextEval's adversarial L6 probe separates genuine context fusion from mere prompt-copying across multilingual AudioLLMs.

Critical Edge

Author–baseline conflict: IndicConformer (baseline) and Sarvam Audio (an evaluated model) are both co-developed by the authors' own institutions — a fully external evaluation would be far more credible.

Methodology

  • Dataset: 55.93 h real speech · 555 speakers · 8 Indic languages · 23 professional domains (read + extempore). Native-script human transcription — no model assistance.
  • L0–L6 prompt ladder: single-variable context injection — none → language → metadata → NL description → English entities → native-script entities → adversarial wrong entities.
  • Models: GPT-4o Transcribe, Gemini 3 Flash, Sarvam Audio, Gemma-3N (vs. IndicConformer baseline @ L1).
  • Metrics: WER (acoustic) + NEER (named-entity error, primary for L4–L6).

Strengths

  • First benchmark to isolate which context signal helps — one variable per level.
  • Adversarial L6 sharply separates prompt-copying from real acoustic grounding.
  • Natural multilingual speech — 555 speakers, broad regional & domain diversity.
  • NEER metric targets the entity-biasing failure mode most relevant to production.
  • Yields an actionable behaviour taxonomy, not just a leaderboard; data + code open-sourced.

Limitations

  • Only 4 AudioLLMs — Qwen3-Omni & Voxtral excluded; strongest open weights untested.
  • 8 Indic languages — no Tamil, Kannada, Punjabi, Assamese (of 22 scheduled).
  • 55.93 h is small vs. ASR training sets — measures behaviour, not capability ceiling.
  • No per-domain or read-vs-extempore breakdown; no inter-annotator agreement reported.
  • L3 prompts generated by Gemini 3 Flash → circularity risk for that model.
  • "13.2% severe hallucination" threshold for Gemma-3N is not operationally defined.

Questions for Further Exploration

  1. Does the L4↔L5 script-mismatch cost reflect a fixed property of script-aware tokenisers, or a learnable gap that scales with model size?
  2. Can the L0–L6 single-variable protocol be replicated with audio context (speaker-turn history, environmental sounds) instead of text?
  3. Is Gemma-3N's L5 instability driven by entity-token quantity (20–30) or by entity–acoustic conflict? An entity-list-length ablation would clarify.
  4. Would instruction-tuning on adversarial L6 examples improve robustness without pushing models toward Sarvam-like context-blindness?
  5. Sarvam's "context-blind but lowest WER" suggests acoustic and prompt-following capacity are decoupled — is contextual prompting overrated when the acoustic model is already strong?
  6. The 23-domain taxonomy is rich but unused in result tables — a per-domain L5-vs-L1 NEER analysis could reveal whether biasing helps uniformly or only in entity-dense fields.
