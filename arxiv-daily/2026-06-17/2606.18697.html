The first data-poisoning attack on learned world models — degrading returns up to 25% while evading standard defenses.

Stealthy World Model Manipulation via Data Poisoning

Yibin Hu · Xiaolin Sun · Zizhan Zheng — Tulane University

SWAAP adversarial ML data poisoning world models model-based RL bilevel optimization

Key Metric

−25%

Max return drop (TD-MPC2, dog-run) poisoning only ≤10% of fine-tuning transitions — data deviation δd stays indistinguishable from clean.

Why This Matters

Learned world models drive modern model-based RL and emerging foundation world models — a stealthy fine-tuning poisoning vector exposes real deployment risks for robotics.

Critical Edge

Main results assume white-box access to the victim's model, architecture, and loss; the gray-box variant is merely preliminary, buried in a single appendix paragraph.

Methodology

Two-stage decomposition of an otherwise intractable bilevel problem.

  • Stage 1 — Target model. A first-order BOME-style dynamic-barrier search over model space, enabled by a new Transition-Gradient Theorem (Thm 3.1) that estimates ∇ψJ through differentiable stochastic transition kernels — avoiding implicit differentiation of the victim's best-response policy.
  • Stage 2 — Gradient-matched poisoning. Select the top r_p=10% highest-residual transitions, then craft poisoned next-state targets by cosine-aligning the training gradient with the direction toward the harmful target, regularized so perturbation magnitude tracks the clean model's natural one-step error.
  • Trajectory-consistent variant jointly perturbs a latent state's two appearances (s′t and st+1) in sequential buffers.
  • Three-stage defense sweep: pre-training detection (residual / CUSUM / PCA), robust training (TRIM), and test-time model-deviation δm.

Strengths

  • First explicit poisoning attack targeting dynamics, not policy/reward — genuine novelty for deep MBRL.
  • Transition-Gradient Theorem is a reusable, model-class-agnostic theoretical contribution.
  • Stealth grounded in the clean model's own approximation error — an architecture-aware notion of "normal."
  • Defense evaluation across three independent stages is more thorough than typical single-detector studies.
  • Honest about limits: no model-level indistinguishability claim; flags the realization bottleneck.

Limitations

  • White-box main threat model; gray-box transfer is preliminary and under-characterized.
  • Headline drops (6–25%) are meaningful but not catastrophic — stealth is prioritized over strength.
  • Realization bottleneck: some Stage-1 targets aren't reachable via Stage-2 data poisoning (pen-twirl-hard).
  • Only non-adaptive defenses tested; certified/smoothing defenders not considered.
  • Small TD-MPC2/DINO-WM only — scaling to foundation world models (Genie, Cosmos) is open.

Open Questions

  1. Does the Transition-Gradient theorem extend to transformer/diffusion world models (DreamerV3, Genie, Cosmos), or is a reparameterization-free estimator needed for stochastic latent dynamics?
  2. Can the Stage-1→Stage-2 realization bottleneck be closed via differentiable subset selection that jointly optimizes target choice and data realization?
  3. How do adaptive defenses perform — CUSUM retrained with α-budget awareness, randomized smoothing, or ensemble-of-world-model disagreement as a poisoning signal?
  4. Can the framework be inverted to design a certified (δ, ε)-robust world model bounding fine-tuning shift?
  5. Does data provenance / signed-trajectory infrastructure effectively neutralize this attack class in deployment?
