> **一句话总结：** 构建 56 小时、跨 8 种印度语言与 23 个专业领域的 IndicContextEval 真实语音基准，配合 L0–L6 七级渐进式提示框架（涵盖元数据、自然语言描述、英文字体/本地文字实体列表及对抗性错误实体），评估 4 个 AudioLLM 与 1 个 Conformer 基线，揭示 GPT-4o Transcribe、Gemini 3 Flash、Sarvam Audio、Gemma-3N 在上下文利用上呈现"均衡 / 敏感 / 上下文盲 / 不稳定"四种截然不同的行为模式。

> **TL;DR:** IndicContextEval (55.93 hours, 555 speakers, 8 Indic languages, 23 professional domains, natural speech) pairs a controlled L0–L6 prompt taxonomy with WER and a new Named Entity Error Rate (NEER) to test whether AudioLLMs truly fuse textual context or fall back on parametric memory. Evaluation of GPT-4o Transcribe, Gemini 3 Flash, Sarvam Audio, and Gemma-3N against an IndicConformer baseline surfaces four distinct behaviours: balanced (GPT-4o: best cross-validation), sensitive (Gemini 3 Flash: best entity accuracy at 17.39% NEER), context-blind (Sarvam: lowest WER but ignores prompts), and unstable (Gemma-3N: 13.2% hallucinated samples at L5). Native-script entity lists give the largest gains (up to −11.7% NEER for GPT-4o) and natural-language descriptions consistently beat structured metadata.

## 核心贡献

- **IndicContextEval 基准**：55.93 小时真实语音（非合成）、555 位说话者、8 种印度语言（Hindi、Bengali、Telugu、Marathi、Gujarati、Malayalam、Odia、Urdu）、23 个专业领域（涵盖 Core Engineering、Data Science、Medical Sciences、Robotics & Automation、Forensics & Legal Sciences、Business、Defense & Armed Forces、Arts、Film & Media Production、Culinary Arts 等），每条语料均配原生脚本人工转写及结构化元数据/实体标注。
- **L0–L6 七级渐进式提示框架**：单变量控制 — 无上下文 → 仅语言 → 元数据 → 自然语言描述 → 英文字体实体 → 本地文字实体 → 对抗性错误实体，使性能变化可归因于具体上下文信号。
- **NEER（命名实体错误率）指标**：与 WER 互补，专门衡量实体偏置（biasing）效果，作为 L4–L6 评估的主指标。
- **AudioLLM 行为分类**：将四个 AudioLLM 划分为"均衡利用 / 敏感利用 / 不稳定 / 上下文盲"四类，并给出 Gemini 3 Flash 在 L5 取得最佳 17.39% NEER、Sarvam Audio 几乎不受提示影响、Gemma-3N 在 L5 有 13.2% 严重幻觉样本等关键定量发现。
- **资源完全开源**：数据集、转写、提示模板与脚本均发布于 github.com/AI4Bharat/IndicContextEval。

## Key Contributions

- **Multilingual natural-speech benchmark:** 55.93 hours, 555 speakers, 8 Indic languages, 23 professional domains — built to be ecologically valid for production contextual-ASR use cases (vs. prior synthetic-speech benchmarks).
- **Controlled L0–L6 prompt taxonomy:** Single-variable introduction of contextual signals (none → language → metadata → NL description → English entities → native-script entities → adversarial wrong entities), enabling causal attribution of performance changes.
- **New NEER metric:** Named Entity Error Rate — fraction of reference entities missing or wrong — designed as the primary metric for entity-biasing evaluation (L4–L6).
- **Behavioral taxonomy of AudioLLMs:** Balanced (GPT-4o Transcribe), Sensitive (Gemini 3 Flash), Context-blind (Sarvam Audio), Unstable (Gemma-3N) — exposes qualitatively different *kinds* of context use, not just WER deltas.
- **Fully open release** at github.com/AI4Bharat/IndicContextEval.

## Methodology

### Dataset (3.1–3.4)
- **Languages:** Hindi, Bengali, Telugu, Marathi, Gujarati, Malayalam, Odia, Urdu. Per-language: Telugu 13.70h (largest), Urdu 3.37h (smallest), mean ≈ 7h.
- **Domains:** 23 professional domains grouped into technical (Core Engineering, Data Science, Medical Sciences, Robotics & Automation), professional (Forensics & Legal Sciences, Business, Defense & Armed Forces), and creative/cultural (Arts, Film & Media Production, Culinary Arts & Food Science, Linguistics). Each category has multiple sub-domains (e.g. Medical Sciences → dental / imaging / clinical).
- **Recording styles:** Two — *read speech* (English sentences with multiple domain terms generated by Gemini 3 Pro, translated to native script via Sarvam-Translate, reviewed by native experts) and *extempore speech* (speakers select 3 expert domains, answer domain-specific questions in lecture style).
- **Quality control:** Native-speaker manual verification (recording quality + domain relevance), then professional verbatim transcription with English entities transliterated to native script and bracketed in English; multi-stage review with senior-annotator adjudication. *No model-assisted transcription.*

### Prompt taxonomy (3.5) — L0–L6
- **L0** No context — bare instruction; tests raw acoustic ASR and implicit language ID.
- **L1** Language only — baseline for further comparison.
- **L2** Language + structured metadata (speech style, region, one-sentence domain description).
- **L3** Language + natural-language audio description (topic + style, generated from L2 metadata by Gemini 3 Flash).
- **L4** Language + English-script entity list (20–30 sampled domain entities); output still in native script — tests cross-lingual biasing.
- **L5** Language + native-script entity list (same list as L4 in native script) — isolates script-mismatch cost vs. L4.
- **L6** Wrong entities (adversarial) — 20–30 entities from an *unrelated* domain in native script (e.g. medical entities for robotics audio). Used as a negative control.

### Models evaluated (4.1)
- **ASR baseline:** IndicConformer (600M, 22 Indian languages) — evaluated at L1 only since it requires the target language as input.
- **AudioLLMs (L0–L6):** GPT-4o Transcribe (proprietary), Gemini 3 Flash (proprietary), Sarvam Audio (proprietary), Gemma-3N 8B-E4B (open-weight). Qwen3-Omni and Voxtral are explicitly excluded because their documentation does not claim support for all 8 Indic languages.
- **Prompting protocol:** Two-part prompt — fixed *context block* (varying by level) + identical *transcription instruction* (output in native script, numbers as spoken words, ignore hesitations, English transliterated).

### Metrics (4.3)
- **WER** — edit distance normalised by reference length; text normalisation via the Indic NLP Library.
- **NEER** — fraction of reference named entities absent or incorrectly transcribed; primary metric for L4–L6.

## Results & Highlights

### Baseline (Table 2, at L1)
| Model | WER | NEER |
|---|---|---|
| IndicConformer (baseline) | 18.81 | 29.58 |
| Sarvam Audio | **16.86** | 25.93 |
| Gemini 3 Flash | 18.90 | **25.85** |
| GPT-4o Transcribe | 28.61 | 35.59 |
| Gemma-3N | 38.73 | 35.50 |

### WER across all 7 levels (Table 3)
| Model | L0 | L1 | L2 | L3 | L4 | L5 | L6 |
|---|---|---|---|---|---|---|---|
| GPT-4o T | 29.83 | 28.61 | 28.37 | 26.08 | 27.97 | **26.04** | 28.47 |
| Gemini 3F | 24.30 | 18.90 | 19.28 | 18.39 | 19.88 | **17.46** | 19.67 |
| Sarvam | 20.39 | 16.86 | 16.78 | 16.43 | 16.80 | **15.70** | 16.69 |
| Gemma-3N | 51.21 | 38.73 | 52.20 | 40.22 | 46.37 | 43.11 | 47.95 |

### Per-language WER at L5 (Table 4, highlights)
- **Malayalam is hardest** for every model: Sarvam 30.8, Gemini 3F 29.7, GPT-4o 42.6, Gemma-3N 70.2.
- Best per-language WER: Sarvam dominates — e.g. Telugu 18.7, Hindi 12.4, Gujarati 11.4.

### Adversarial control (Table 5, L6 vs L1)
| Model | WER Δ (L6−L1) | NEER Δ | Behaviour |
|---|---|---|---|
| GPT-4o T | −0.14 | −1.04 | robust, cross-validates context |
| Gemini 3F | +0.77 | −0.25 | moderate sensitivity |
| Sarvam | −0.17 | −0.31 | unaffected (context-blind) |
| Gemma-3N | **+9.22** | +0.75 | blindly follows wrong entities |

### Key empirical findings (5.2–5.4)
- **L5 native-script entity lists give the strongest gains** — GPT-4o Transcribe −11.7% NEER, Gemini 3F −8.5%, Gemma-3N −8.6%, Sarvam −4.2%. Script mismatch (L4 vs L5) costs up to 11 WER points.
- **L0 → L1 language-identification tax is large:** Gemma-3N gains 12.48, Gemini 3F 5.40, Sarvam 3.53, GPT-4o 1.22 WER points.
- **L3 (NL description) consistently beats L2 (structured metadata):** GPT-4o gains 2.53 vs. 0.24; Gemma-3N degrades +13.47 with structured metadata but only +1.49 with NL — **prompt format matters as much as content**.
- **Context narrows the ASR gap:** At L5, all four AudioLLMs beat IndicConformer's 29.58% NEER; Gemini 3F's 17.46% WER also beats IndicConformer's 18.81%.
- **Four AudioLLM behaviour classes** (5.4 Discussion):
  - *Balanced* (GPT-4o Transcribe): best contextual reasoning, benefits from correct prompts (−2.57 WER at L5) while ignoring adversarial ones (L6 ≈ L1).
  - *Sensitive* (Gemini 3 Flash): best entity accuracy (17.39% NEER at L5), −1.44 WER L1→L5, consistent integration of context.
  - *Unstable* (Gemma-3N): entity NEER 35.5% → 26.9% at L5, **but 13.2% of L5 samples show severe hallucinations/repetitions**; WER rises 4.38 L1→L5.
  - *Context-blind* (Sarvam Audio): lowest WER at L1 (16.86%) and L5 (15.70%); only 1.16 WER spread across all levels — driven by acoustic encoder, not prompts.

## Critical Assessment

- **Novelty:** Strong. Prior contextual-ASR benchmarks either (a) evaluate under fixed prompting (IndicVoices, CommonVoice, FLEURS), (b) use synthetic speech (ContextASR-Bench, ProfASR-Bench), (c) focus on English (Earnings-22, ContextASR), or (d) test only one context type. IndicContextEval is the first to combine (i) *natural* multilingual speech, (ii) *controlled single-variable* prompt variation, and (iii) an *adversarial probe* (L6) that distinguishes true context fusion from prompt-copying. The L6 design is the most novel methodological move and is rare in the ASR-evaluation literature.
- **Rigor:** High. Multistage human transcription (no model assistance), explicit exclusion criteria for models lacking documented 8-language support, per-language and per-level breakdowns, and an adversarial negative control. The 7-level single-variable design is a clean attribution framework.
- **Reproducibility:** Generally good — data, transcriptions, and code are released. Risks: (a) Gemini 3 Flash is used to *generate* L3 audio descriptions, which then become part of the prompt — circular if one of the evaluated models is the same family, and (b) speaker recruitment is via NPTEL channels, which may not generalise to the wider Indian population.
- **Significance:** High for the multilingual-AudioLLM community, especially since the field has been moving toward "prompt-conditioned ASR" without a clear way to verify that prompts are actually being used.
- **Red flags / honest concerns:**
  1. **Author–baseline conflict of interest:** IndicConformer (baseline) and Sarvam Audio (one of the 4 evaluated AudioLLMs) are both from the authors' affiliated institutions. The paper frames Sarvam as "context-blind" but its *low WER* still wins the L5 main metric — the framing is fair, but a fully external evaluation would be more credible.
  2. **Circularity in L3:** Audio descriptions at L3 are generated by Gemini 3 Flash, one of the evaluated models. This may artificially inflate Gemini 3F's L3 performance (small gain of 0.51 WER reported, but worth flagging).
  3. **Limited model coverage:** Only 4 AudioLLMs + 1 ASR baseline evaluated. Qwen3-Omni and Voxtral are excluded for language-coverage reasons, but that means we learn nothing about how the *strongest open-weight* models behave on this benchmark.
  4. **No inter-annotator agreement reported** for transcriptions or for the "domain-relevance" verification step.
  5. **"23 professional domains" aggregation:** No results are broken down by domain — the per-domain dynamics could mask or reveal important effects (e.g. is L5 only helpful for entity-heavy domains like Medical, but neutral for conversational ones like Business?).
  6. **Malayalam is hard across the board (up to 70% WER for Gemma-3N)** — the per-language results suggest a single number per language hides substantial within-language variance (dialect, script normalisation, code-mixing).
  7. **No human-evaluation of hallucinations** — the 13.2% severe-hallucination claim for Gemma-3N is reported as a count, but the threshold for "severe" is not defined; readers must trust the authors' judgment.

## Strengths

- First benchmark to isolate *which* context signal helps, by varying only one signal per level.
- Adversarial L6 prompt is a sharp probe — distinguishes prompt-copying from acoustic grounding.
- Natural (non-synthetic) multilingual speech with strong diversity in speakers (555), regions, and domains.
- New NEER metric directly targets the entity-biasing failure mode that is most relevant to production use.
- Strong author pedigree in Indic-language speech resources; full public release of data + code.
- Identifies qualitatively distinct model behaviours (balanced / sensitive / context-blind / unstable) — actionable taxonomy, not just a leaderboard.

## Limitations

- Only 4 AudioLLMs evaluated; strongest open-weight models (Qwen3-Omni, Voxtral) excluded for language-coverage reasons.
- 8 Indic languages — does not include Tamil, Kannada, Punjabi, Assamese, or any Tibeto-Burman / Munda language. Generalisation to the full 22 scheduled Indian languages is untested.
- 55.93 hours is small relative to typical ASR training sets; the benchmark measures behaviour, not capability ceiling.
- No per-domain or per-style (read vs. extempore) result breakdown.
- L3 prompt generation by Gemini 3 Flash risks circularity for that model.
- No inter-annotator agreement reported for transcription or relevance judgement.
- The "13.2% severe hallucination" criterion for Gemma-3N is not operationally defined.
- Author–model conflict of interest (IndicConformer + Sarvam Audio are co-developed by the author group).

## Relevance

与用户研究方向相关性中等。该基准与多模态 LLM 评测方法学、上下文利用可控性分析、跨语言泛化评测有方法论上的交集——尤其 L0–L6 单变量提示设计是评估"提示是否真正起作用"的可推广范式；如用户关注 LLM 评估、上下文学习、或多语种/低资源 ASR，可作为参考。

## Questions for Further Exploration

- How does context utilisation scale with AudioLLM size, and is the L4↔L5 script-mismatch cost a fixed property of script-aware tokenisers or a learnable gap?
- Can the L0–L6 single-variable protocol be replicated with *audio context* (speaker-turn history, environmental sounds) rather than textual context, to test whether the same taxonomy generalises?
- Is Gemma-3N's instability at L5 (13.2% hallucinations) caused by the *quantity* of entity tokens (20–30 in the prompt) or by entity–acoustic conflict resolution? An ablation varying entity-list length would help.
- Does instruction-tuning AudioLLMs with adversarial L6 examples (wrong entities → ignore) improve L6 robustness without hurting L5 gains, or does it teach models to ignore context altogether (Sarvam-like behaviour)?
- The Sarvam-Audio "context-blind but lowest WER" pattern suggests acoustic capacity and prompt-following capacity may be *decoupled* — is this consistent across more languages and domains, and does it mean contextual prompting is overrated when the acoustic model is already strong?
- How would Qwen3-Omni and Voxtral perform if the Indic language coverage gap were closed via continual pre-training, and would their behaviour match one of the four identified classes?
- The 23-domain taxonomy is rich but unused in the result tables; a per-domain L5 vs L1 NEER analysis could reveal whether entity-biasing helps uniformly or only in entity-dense domains (e.g. Medical, Legal).
