Action Research · Long-horizon LLM Collaboration

Written by AI, Managed by AI: Semantic Space Control & Index Sickness Elimination Across 391 Consecutive Sessions

Hui Zhang (张慧) — Shenzhen Yunxi Technology Co., Ltd.  ·  Shuren Song (宋树仁) — Information Technology Center, Tsinghua University

human–AI collaboration long-horizon LLM context management prompt engineering action research semantic space control
Key Finding
When symbolic indices proliferate, long-horizon LLMs abandon business meaning and collapse into self-referential "Phantom Legislation."
Key Metric
75%
AI Instructions cut from 308 → ~80 lines; Index Sickness did not recur over the next ~150 sessions.
Why This Matters
It reframes long-context degradation as a noise problem, not a memory problem — and shows a zero-dependency file fix beating heavier structured-memory stacks at the active-context boundary.
Critical Edge
Single-Owner action research: no named LLM models, no token-level before/after, and a headline "no recurrence" that is an unfalsifiable negative observation.

01 Methodology 方法

Action Research (AR) — Diagnosis → Theory → Action → Evaluation → Reflection. The researcher is simultaneously the project's Owner, CSF designer, and evaluator, a reflexivity constraint the paper itself flags.

  • Setting: "Bang-v3" — a WeChat mini-program + H5 task-forwarding refactor; ~1 month, 391 sessions, ~80K LOC across 506 files; CSF framework itself (~900 files, ~150K lines, ~150 raw session records). The Owner wrote no code or spec line independently.
  • Diagnosis: case analysis of failure sessions (e.g. Session 215 "Pending-Flow v2 Legislation" that passed symbolic review but had no physical skeleton; D-108 label spread across 13 files), each anchored to a session number, file path, and verbatim Owner quote.
  • Theory: the Pang Principle built from one axiom (natural language > symbols) + two facts (intelligence/determinism tradeoff; semantic dilution law), with "purpose" as the minimal control variable → natural language + purpose = max information quality.
  • Mechanism: Baseline-Log Physical Separation — pure file convention, no middleware/model upgrade/graph layer: baseline context.md = overwrite-only cross-section of "now"; session log = append-only, natural-language, not loaded by default.
  • Evaluation: mixed quantitative (line counts, session counts, label counts) + qualitative (session-title shift, input-pattern shift). Per-session token use was not systematically logged — an acknowledged limitation.

03 Strengths & Limitations 优势与局限

Strengths

  • Genuine in-the-wild evidence: 391 sessions, ~80K LOC, 506 files — not a toy setup.
  • Actionable mechanism: Baseline-Log needs no new tooling; adoptable today.
  • Crisp vocabulary: "Index Sickness," "Phantom Legislation," "Pang Principle," "Literal Stickiness" all useful additions.
  • Methodological transparency: AI-written paper disclosed, session records preserved, reflexivity + alternative explanations stated.
  • Useful counter-framing: supply-side vs. consumer-side dichotomy fills a real gap.

Limitations

  • Single project, single Owner, single AI collaborator — generalization unproven.
  • LLM models never named — a serious gap for a paper about LLM-specific failure modes.
  • No quantitative context-load (tokens/session before vs. after) collected.
  • "No recurrence" is a negative observation — hard to falsify.
  • Confounded with team learning, not statistically controlled; mechanism is descriptive, not causally isolated.
  • 75% reduction is structural, not a measured deployment outcome.

04 Questions for Further Exploration

  1. Cross-model replication. Does "no Index Sickness" hold across Claude / GPT / Gemini / DeepSeek / open-source, or is it tied to one attention behavior?
  2. Quantitative context-load. Pair the 75% cut with controlled attention-weight measurement on deprecated vs. current concepts.
  3. Disentangle the learning confound. A multi-Owner, multi-project controlled study (matched complexity, with/without Baseline-Log).
  4. Division of labor with structured memory. Is physical isolation for the active boundary complementary to CoMem/REAL/GAM for the long archive?
  5. Automated Phantom-Legislation detection. A lightweight "claim → physical artifact ID" resolver layered on the mechanism.
  6. Purpose-first activation beyond SE. Does it generalize to long-running research agents or multi-session creative writing?
