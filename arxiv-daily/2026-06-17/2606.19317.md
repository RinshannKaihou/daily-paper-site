> **一句话总结：** 用 Claude Sonnet 4 作为程序合成代理（每头一次 refine，JSD 排序），在 BERT-base / GPT-2 / TinyLlama-1.1B / Llama-3B 上共生成 1,664 个 Python 程序作为注意力头的可执行代理，IoU 随规模上升（GPT-2 ≈ 69% → Llama-3B ≈ 79%），且按 IoU 降序贪心替换 25–40% 的头时困惑度仅小幅上升，六个下游 QA 基准基本不掉点。

> **TL;DR:** The authors turn transformer attention heads into compact, human-readable Python programs. For each head, they extract attention patterns on TinyStories, filter to the top 2.5% of weights, summarize them in a ~4,000-token prompt, and ask Claude Sonnet 4 (with NumPy/spaCy/NLTK) to emit executable candidate programs. Programs are scored by Jensen-Shannon distance, refined once using best-vs-worst contrastive feedback, and selected on a held-out set. Across 1,664 heads in four models (BERT-base, GPT-2-small, TinyLlama-1.1B, Llama-3.2-3B), mean best-program IoU rises with model scale (69% → 79%), and swapping in 30–40% of heads in IoU-greedy order yields only a mild perplexity increase (Spearman r < −0.91 between IoU and substitution cost) while preserving HellaSwag / PIQA / SciQ / ARC-Easy / Social IQA / COPA accuracy.

## 核心贡献

- 提出 **LLM 驱动的程序合成 + 解释** 管线：让 Claude Sonnet 4 直接生成能复现注意力矩阵的 Python 程序，无需手工模板或枚举搜索。
- 引入 **Jensen-Shannon 距离（JSD）排序 + 一次 best-vs-worst 对比反馈 refine**：在程序合成中以分布相似度作为适配信号，相比纯 IoU 更稳定。
- 构建完整的 **程序库（1,664 个程序，覆盖四个模型的全部注意力头）**：总共 < 4,000 候选、约 $150 API 成本（35M 输入 + 3.5M 输出 tokens），首次证明"用可执行代码逐头近似 LLM 注意力"在算力上完全可承受。
- 系统性 **跨架构 + 跨规模** 评估：BERT-base（144 头，双向） / GPT-2-small（144） / TinyLlama-1.1B（704） / Llama-3.2-3B（672, 28 层）。
- 实证 **可执行性即因果性**：以 IoU 降序贪心替换 25–40% 注意力头，困惑度仅小幅上升、六个下游 QA 基准不掉点，IoU 与替换代价的 Spearman 相关系数 < −0.91。
- 揭示两条结构性规律：(i) **双向注意力头（BERT）比因果头更难被符号化**；(ii) **头数越多、规模越大，单独每个头的行为越专门化、越容易被程序捕获**（IoU 随规模单调上升）。

## Key Contributions

- **End-to-end pipeline for symbolic attention-head surrogates:** extract attention patterns → filter to top 2.5% weights → summarize as text → prompt Claude Sonnet 4 to emit Python → rank by JSD → one round of contrastive feedback refinement → re-rank on held-out data.
- **1,664 generated programs covering every head in four transformer LMs** (BERT-base, GPT-2-small, TinyLlama-1.1B, Llama-3.2-3B), produced at modest cost (~$150 / 35M in + 3.5M out tokens).
- **Program library functions as a reusable code base, not a per-head artifact:** the *globally* best program (selected from any head's library) consistently outperforms the *intended* program synthesized for that head, indicating cross-head generalization and supporting a quality-diversity (MAP-Elites-style) view of the head space.
- **Causal head replacement validates the surrogates:** swapping up to 30–40% of attention heads in IoU-greedy order preserves HellaSwag, PIQA, SciQ, ARC-Easy, Social IQA and COPA accuracy, with a strong negative correlation (Spearman r < −0.91) between head-program IoU and resulting perplexity cost.
- **Architectural and scaling findings:** decoder (causal) heads are easier to approximate than encoder (bidirectional) heads; mean best-program IoU increases with model scale (GPT-2 ≈ 69% → TinyLlama-1.1B ≈ 74% → Llama-3B ≈ 79%), consistent with the *functional specialization* hypothesis at scale.
- **Program-category analysis** (six types: first-token, final-token, sequential, coreference, syntactic, uniform) reveals depth-dependent structure in GPT-2: first-token programs dominate early layers, syntactic programs appear in middle layers.

## Methodology

The pipeline operates per attention head and aggregates into a global library:

1. **Attention map extraction.** For each head, run the target model on TinyStories sentences (no preprocessing, no truncation) and record the n×n attention matrix for every sequence.
2. **Pattern filtering.** Keep only the **top 2.5% of attention weights by magnitude** to isolate the salient token-pair interactions (suppresses noise from low-magnitude background attention).
3. **Matrix summarization.** Format the filtered token-pair weights as a textual list, embedded into a **~4,000-token structured prompt** containing the input text, the matrix summary, and instructions to emit Python.
4. **Candidate generation.** A pretrained LLM — **Claude Sonnet 4** — is given access to **NumPy, spaCy, and NLTK** and emits candidate Python programs π : X → Â that map input tokens to attention matrices.
5. **Validation & JSD ranking.** Candidates are checked for syntax and executability; non-well-formed ones get maximal divergence. Surviving programs are scored by **Jensen-Shannon distance (JSD)** against the real matrix A.
6. **One round of contrastive feedback refinement.** Sample inputs to find the best- and worst-scoring examples, build structured feedback contrasting real vs. predicted patterns, and prompt the synthesis agent for a revised program π′. Re-validate, re-score, and keep π* = argmax IoU over held-out (X′, A′) ∈ D_val.
7. **Library assembly.** The final Π contains **one program per head** across all four models (1,664 total), produced from <4,000 candidates.
8. **Causal interchange intervention.** Replace the neural attention matrix A with the program output π(X) during a forward pass; measure **normalized perplexity increase** and **downstream accuracy** on HellaSwag, PIQA, SciQ, ARC-Easy, Social IQA, COPA. Heads are replaced in **IoU-descending order** so the most faithful surrogates substitute first.
9. **Category analysis.** Programs are clustered into six behavioral categories by prompting the synthesis agent with their docstrings, producing a coarse functional taxonomy (first-token, final-token, sequential, coreference, syntactic, uniform).

**Models analyzed:** BERT-base (144 heads, bidirectional), GPT-2-small (144), TinyLlama-1.1B (704), Llama-3.2-3B (672 across 28 layers).

## Results & Highlights

- **IoU scaling with model size (mean best-program IoU):** GPT-2-Small ≈ 69%, TinyLlama-1.1B ≈ 74%, Llama-3B ≈ 79%. Best program > intended program for most heads (cross-head reuse).
- **Architectural gap:** BERT-base is systematically harder to approximate than the three decoder models, attributed to the bidirectionality of MLM producing more diffuse, less predictable attention distributions.
- **Perplexity under replacement:** IoU-greedy substitution of attention heads yields a **low and gradually increasing** normalized perplexity, while a structural (lower-diagonal / uniform) baseline explodes by >1000% after just 5% replacement in TinyLlama. At ~25% replacement, the paper reports only a ~16% average perplexity increase across models.
- **Strong negative correlation between IoU and substitution cost:** Spearman r < −0.91 across GPT-2, TinyLlama, and Llama-3B — high-IoU heads are also the functionally safest to replace.
- **Downstream QA robustness:** up to 30–40% of heads can be replaced without significant degradation on HellaSwag, PIQA, SciQ, ARC-Easy, Social IQA, COPA. Even at high replacement levels, several benchmarks remain clearly above random.
- **Depth-dependent program types in GPT-2:** first-token programs cluster in early layers; syntactic / coreference programs become more common in middle and later layers — consistent with the classical NLP pipeline rediscovery reported in earlier probing work.
- **Selected programs are short and interpretable.** Examples shown in the paper (e.g., "attend to first token", "attend to content words of determiner phrases", "boost attention to sentence-initial tokens after punctuation") are human-readable and semantically meaningful.
- **Total compute cost:** ~$150 in Claude Sonnet 4 API spend, ~35M input + 3.5M output tokens. Code + programs released at github.com/AmiriHayes/explaining_attention_heads.

## Critical Assessment

- **Novelty:** High. The paper elevates "program synthesis for interpretability" from boolean/arithmetic enumerative search (Mu & Andreas 2020; Michaud et al. 2024 MIPS) to **general-purpose Python with NLP libraries**, and from small/random-arch networks to **modern autoregressive LMs (up to Llama-3B)**. The causal substitution result — replacing 25–40% of heads with a small library of programs — is the most concrete demonstration so far that symbolic surrogates can stand in for trained modules.
- **Rigor:**
  - The evaluation is multi-level: per-head IoU, library-level best-program matching, perplexity under substitution, and downstream QA across six benchmarks.
  - The Spearman r < −0.91 between IoU and substitution cost is a strong, falsifiable result and validates IoU as a meaningful selection metric.
  - However, the **head selection for replacement is greedy in IoU order** — this is the *easy* direction. A random-order replacement test would be a more conservative stress test, since high-IoU heads may be the least functionally important to begin with.
  - The perplexity evaluation is **in-distribution** (same TinyStories used for program synthesis), so the 16% / 25%-replacement numbers may overstate generalization; the downstream QA suite partially addresses this but on different distributions.
  - Several heads (substantial fraction, by the authors' own admission) sit below 40% IoU, so the "we can replace 25% of heads" claim is contingent on IoU-greedy selection — the paper would be stronger with a random-baseline replacement curve.
- **Reproducibility:** Strong signals. Code and data released on GitHub; the synthesis agent is named (Claude Sonnet 4); prompts and tool access documented; total cost enumerated (~$150). The four models used are all openly available. The only mild concern is that prompt engineering for Claude Sonnet 4 is a non-trivial human process and may not transfer cleanly to other code-LMs.
- **Significance:** Strong. If 30–40% of attention heads in modern LMs can be **physically replaced by auditable code** without behavior change, this opens a credible path to (a) mechanistic auditing of model behavior, (b) targeted model editing via program modification rather than weight surgery, and (c) hybrid neuro-symbolic systems. The cross-model and cross-scale trends (IoU rising with scale, decoder easier than encoder) are also substantive empirical findings for the interpretability community.
- **Red flags to watch:**
  - The "best program" routinely beats the "intended program" — i.e., the library is small enough that programs are fungible across heads. This is good for compression, but it also means the *per-head* explanation is somewhat arbitrary and may not be the *causally* correct one. The paper's framing leans on the library view, but readers should not over-interpret any single program as "what this head really does."
  - Many "high-IoU" programs are short and simple (e.g., attend-to-first-token). IoU is high because the attention matrix is dominated by a few strong rules; this doesn't necessarily mean the residual 25% of the matrix is uninteresting. A finer-grained metric that separates "salient-pattern recovery" from "long-tail fidelity" would strengthen the claim.
  - The improvement in downstream QA at low replacement levels (sometimes seen) is acknowledged by the authors as plausibly a **pruning effect** (removing noisy heads), not a positive contribution of the programs. The paper is honest about this, but readers should be aware that the substitution experiment is not pure "replacement with equally good surrogate."
  - **No safety / adversarial robustness analysis:** would the synthesized programs break under jailbreak prompts, code-mixed input, or distribution shift? The release is positioned as interpretability research, not as a deployment-ready surrogate, so this is acceptable — but worth flagging.

## Strengths

- Clean, modular four-step pipeline (extract → summarize → synthesize → re-rank) that is straightforward to extend and reproduce.
- Three independent evaluation lenses: pattern IoU, behavioral perplexity, downstream QA — none of which is sufficient on its own.
- Genuine **causal** evidence (interchange interventions) rather than correlational interpretability claims.
- Cross-architecture and cross-scale replication (encoder/decoder, 144 → 704 heads, 0.1B → 3B parameters).
- Honest reporting of limitations: a substantial fraction of heads remain below 40% IoU; high-replacement degradation is attributed to those poorly fit heads.
- Public release of code and data; total compute cost small enough (~$150) that the result is genuinely reproducible.
- Produces artifacts (executable Python) that are auditable, modifiable, and amenable to compositional analysis — a major step up from natural-language neuron descriptions.

## Limitations

- **No head is fully captured:** the authors explicitly state that a substantial fraction of heads achieve IoU < 40%, and high-replacement QA degradation is mostly attributable to these.
- **Programs are short and not very diverse.** Many high-scoring programs collapse to a few motifs (attend-to-first-token, attend-to-prior-token, attend-to-sentence-initial); the paper acknowledges that richer multi-round refinement with stronger feedback is needed.
- **Single training corpus (TinyStories).** The library is synthesized on one dataset; whether the programs transfer to Wikipedia, code, or multilingual text is not tested.
- **Perplexity evaluation is in-distribution.** Out-of-distribution perplexity under replacement would be a harder test.
- **Replacement is IoU-greedy**, which inflates the apparent functional equivalence. A random-replacement baseline would be more conservative.
- **Synthesis is restricted to a single LLM (Claude Sonnet 4).** Results may not transfer to weaker or stronger code-LMs without re-tuning the prompt and tool set.
- **Only attention heads are addressed.** MLPs, embeddings, layer norms, and cross-attention (in encoder-decoders) are out of scope.
- **The authors do not claim, and have not shown, that the program library is causally sufficient** — programs approximate attention outputs, not the full contribution of a head to downstream computation (the output projection after attention is still neural).

## Relevance

与"机制可解释性 + 大模型推理 + 程序合成"三条主线高度相关：本工作把"用可执行代码反向工程 LLM 内部组件"从小型/布尔表达式推进到现代自回归 LLM 的全部注意力头，并用因果替换实验证明程序可作为头层的功能性代理，对 LLM 内部结构审计、可编辑性与神经-符号混合系统均有直接启发。

## Questions for Further Exploration

- What happens if replacement is performed in **random order** (or anti-greedy in IoU) — does the perplexity curve still rise gracefully, or does the surrogate library only work for the easy heads?
- How does the method scale to **much larger models** (≥7B) where individual heads may be more **polysemantic** and the 25%/40% replacement ratio may collapse?
- Can the program library be **factored into a smaller set of reusable sub-routines** (similar to circuit-level motif discovery), reducing the 1,664-program count to perhaps a few dozen primitives?
- Do the programs **transfer across distributions** (Wikipedia, code, multilingual text, long-context inputs)? In particular, do the "first-token" / "syntactic" programs remain valid out of TinyStories?
- Could the synthesis be made **multi-round** with stronger feedback (e.g., test-case generation, symbolic execution, counterexample search) to close the <40% IoU gap?
- Can the programs be used as **edits** (causal scrubbing style) — remove a head's program, observe the behavior change, and use the delta to localize function? This would turn the library into a debugging tool, not just a description.
- Does the surrogate generalize to **adversarial / OOD prompts** (jailbreaks, prompt injection, code-switching)? If the library is to be used for safety auditing, robustness is essential.
- How sensitive is the result to the **code-LM choice**? Would a smaller, open code model (e.g., a fine-tuned Code Llama) reproduce the same IoU curve at higher compute but lower API cost?
- Could the same pipeline be applied to **MLP layers** (a substantially harder synthesis problem, but with a much larger payoff, since MLPs hold most of the model's parameters)?
