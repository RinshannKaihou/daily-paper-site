> **一句话总结：** 本文提出 SWAAP——首个针对学习型世界模型的两阶段数据投毒框架：第一阶段用一阶 BOME 双层优化（依赖本文新证的"转移梯度定理"）在模型空间搜索"有害但低偏差"的目标动力学，第二阶段以隐蔽约束的梯度匹配修改 ≤10% 的微调转移标签，使中毒模型在 TD-MPC2 上多个连续控制任务取得显著回报下降（最多约 25%）并系统性规避残差/CUSUM/TRIM 等非自适应防御。

> **TL;DR:** SWAAP is a two-stage data poisoning attack on learned world models used for planning. **Stage 1** solves a model-space bilevel problem via a first-order dynamic-barrier (BOME) update, enabled by a new Transition-Gradient theorem that propagates return sensitivity through differentiable transition kernels, to find a target world model that is harmful (low return under the victim's planner) yet close to clean dynamics. **Stage 2** realizes that target by selecting the top-rp=10% highest-residual transitions under the Stage 1 model and crafting poisoned next-state targets via gradient matching — cosine-aligning the training gradient with the direction that would move the clean model toward the target, plus a prediction-error regularizer tying perturbation magnitude to the clean model's natural one-step error. A trajectory-consistent variant handles sequential fine-tuning buffers. Evaluated on TD-MPC2 (and DINO-WM in the appendix) across DMControl, MyoSuite, MetaWorld, and ManiSkill2, SWAAP degrades returns by 6–25% while keeping data-level deviation δd essentially indistinguishable from clean (δd ≈ δd,ref); a CUSUM detector with AUC 0.86 at the unconstrained regime drops to AUC 0.51 at the main stealth setting, and the standard TRIM robust-training defense fails to remove the attack (and on Humanoid-Walk even amplifies it).

## 核心贡献

- 提出 SWAAP，**首个**面向学习型世界模型（而非策略或奖励）的数据投毒攻击，揭示 MBRL 微调流水线的实际安全漏洞。
- **两阶段分解**：将原本不可微的双层问题解耦为 (i) 在模型空间中搜索"有害但低偏差"目标动力学，(ii) 通过梯度匹配在有限比例的微调转移上实现该目标；并通过实验证明两阶段缺一不可。
- **转移梯度定理（Transition Gradient Theorem）**：给出有限/无限视野随机转移核的 ∇_ψ J(ψ, θ) 估计，使一阶 BOME 类方法在 MBRL 中可用。
- **隐蔽约束的数据实现**：把 Geiping et al. (2021) 的梯度匹配适配到世界模型微调，通过预测误差正则项将扰动幅度限制在清洁模型自然预测误差内。
- **轨迹一致性变体**：当微调数据为序列轨迹时，对同一中间隐状态在 t 与 t+1 处的两次出现做一致替换，进一步提升隐蔽性。
- **三阶段防御评估**：分别在预训练检测（残差 / CUSUM / PCA）、鲁棒训练（TRIM）、测试时监测（δ_m）三阶段评估，量化隐蔽性。

## Key Contributions

- **First explicit world-model-targeted data poisoning attack** in deep model-based RL (prior poisoning work targets policies/rewards, not dynamics).
- **Two-stage decomposition** of an otherwise intractable bilevel problem: Stage 1 finds a stealthy harmful target model; Stage 2 realizes it via gradient-matched data poisoning. Empirical evidence that both stages are necessary (random Stage 2 and direct-Stage-1-realization baselines underperform).
- **Transition-Gradient Theorem** (Thm 3.1) providing a tractable first-order estimator of ∇_ψ J for differentiable stochastic transition kernels, enabling BOME-style bilevel updates in MBRL.
- **Stealth-constrained gradient matching** with a prediction-error regularizer that ties perturbation magnitude to the clean model's natural one-step noise.
- **Trajectory-consistent poisoning** variant that writes a poisoned intermediate latent state consistently as both s'_t and s_{t+1} in sequential fine-tuning data.
- **Three-stage defense evaluation pipeline** (pre-training detection, robust training, test-time monitoring) showing SWAAP bypasses residual / CUSUM / TRIM-style defenses at the stealth operating point.

## Methodology

**Setup.** A model-based RL agent learns a world model P_ψ that approximates environment transitions. The model is used for downstream planning (e.g., TD-MPC2's MPC with 512 candidate rollouts, horizon 3) or policy improvement. The world model is first pretrained on 1M clean transitions, then fine-tuned on 5K transitions. SWAAP is a white-box attack with access to ψ_0, the architecture, and the fine-tuning loss; a gray-box variant (different encoder, different task) is in Appendix H.3.

**Bilevel formulation.** The attacker minimizes the true-environment return J(P, θ*(ψ)) of the victim's policy while keeping P_ψ close to the true dynamics (Eq. 1):
```
min over D̃:   J(P, θ*(ψ)) + λ L(P_ψ, P)
s.t.   P_ψ = F(P_ψ0, D̃),   θ*(ψ) = argmax J(P_ψ, θ),   |{i : s̃'_i ≠ s'_i}| ≤ r_p |D|.
```

**Stage 1 — target model identification.** Solve a model-space bilevel problem (Eq. 3) for a target ψ̂. The authors use a first-order dynamic-barrier method in the spirit of BOME (Liu et al., 2022) to avoid differentiating the implicit best response θ*(ψ). The key new ingredient is the **Transition-Gradient Theorem**:
> ∇_ψ J(ψ, θ) = E_{(s_t, a_t, s_{t+1}) ~ P_ψ, π_θ} [ Σ_t γ^t (R_t + γ V_{t+1}(s_{t+1})) · ∇_ψ log P_ψ(s_{t+1} | s_t, a_t) ]

At iteration k, the attacker runs W policy-gradient steps to obtain an approximate best response θ_k^W under P_ψk, then takes a first-order dynamic-barrier step on (ψ_k, θ_k) using outer / constraint gradients estimated from true-environment and perturbed-world-model rollouts. Deviation L(P_ψ, P) is instantiated as the squared latent prediction error (the stochastic case is in Appendix D.3).

**Stage 2 — gradient-matched data poisoning.** (a) Select the top-r_p fraction of transitions with the largest residual e_ψ̂(s, a, s') = ‖s' − P_ψ̂(s, a)‖_2 under the Stage 1 target (these are most inconsistent with the harmful target, so perturbing them gives strong local signal). (b) Optimize poisoned targets {s̃'_i} by aligning the resulting training gradient G_real with the target-direction gradient G_target that would move the clean model toward ψ̂. The Stage 2 loss (Eq. 5) is a convex combination (controlled by α) of (1 − cos(G_real, G_target)) and a per-point perturbation penalty that ties ‖s̃' − s'‖ to the clean model's one-step prediction error.

**Trajectory-consistent variant.** When fine-tuning data are stored as sequential trajectories, the same latent state appears as the next-state target at t and the current state at t+1. The variant perturbs both occurrences jointly and adds a one-step consistency gradient term so the poisoned intermediate state remains predictive of the observed successor under the next action.

**Three-stage defense evaluation.**
- *Pre-training detection.* (i) Deviation-based filter using a reference model P_ψref (here = ψ_0): compare δ_d to δ_{d,ref}. (ii) Residual-based CUSUM detector on the anomaly signal x_t = ‖z_{t+1} − P_ψref(z_t, a_t)‖_2 calibrated on clean trajectories. (iii) Latent-space PCA sanity check that poisoned targets don't form outliers.
- *Training-time.* TRIM (Biggio et al., 2012) — iteratively retain the (1−β) lowest-residual transitions; β = 0.2 in the main evaluation.
- *Test-time.* Model-level deviation δ_m = (1/T) Σ ‖s'_i − P_ψ(s_i, a_i)‖_2 / ‖s'_i‖_2.

## Results & Highlights

**Main attack table (Table 1; rp = 0.1, α = 0.9).** SWAAP degrades return across every environment while δ_d stays essentially equal to δ_{d,ref}:

| Env | Clean return | SWAAP return | Δ | δ_d (SWAAP) | δ_{d,ref} |
|---|---|---|---|---|---|
| DMControl humanoid-walk | 868 ± 15 | 775 ± 50 | −10.7% | .090 | .090 |
| DMControl humanoid-run | 584 ± 13 | 515 ± 14 | −11.8% | .053 | .051 |
| DMControl dog-walk | 798 ± 8 | 748 ± 13 | −6.3% | .060 | .059 |
| DMControl dog-run | 637 ± 9 | 478 ± 26 | −25.0% | .052 | .051 |
| DMControl cheetah-run | 838 ± 11 | 806 ± 6 | −3.8% | .034 | .028 |
| MyoSuite pen-twirl-hard | 3574 ± 640 | 2771 ± 801 | −22.5% | .056 | .056 |
| MyoSuite reach-hard | 743 ± 22 | 606 ± 198 | −18.4% | .035 | .035 |
| MetaWorld push | 1784 ± 13 | 1641 ± 80 | −8.0% | .081 | .080 |
| MetaWorld soccer | 1724 ± 9 | 1479 ± 96 | −14.2% | .051 | .046 |
| ManiSkill2 lift-cube | 193 ± 4 | 175 ± 20 | −9.3% | .032 | .031 |
| ManiSkill2 pick-cube | 161 ± 7 | 147 ± 14 | −8.7% | .027 | .025 |
| ManiSkill2 stack-cube | 170 ± 16 | 141 ± 18 | −17.1% | .042 | .041 |

The paper deliberately **does not** claim model-level indistinguishability: δ_m varies across environments and is larger than δ_{m,ref} in some cases (e.g., .171 vs .080 on MW-push, .101 vs .057 on pen-twirl-hard). The reported numbers are a **stealth-prioritized operating point**; the Direct Model Poisoning reference (which overwrites the model directly) achieves much larger drops (e.g., 307 vs 868 on humanoid-walk) but at the cost of higher δ_m and an unrealistic threat model.

**Stage 1 vs Stage 2.** SWAAP substantially outperforms SWAAP (Random) — random Stage 2 perturbations saturate while SWAAP's returns keep dropping as α relaxes (Figure 2). On Humanoid-Walk, SWAAP closely tracks the Direct Model Poisoning frontier (Stage 1 target ≈ Stage 2 realization); on Myo-Pen-Twirl-Hard a gap remains, exposing a **realization bottleneck** in the two-stage decomposition.

**Defense bypass.**
- CUSUM pre-training detector: AUC 0.864 / TPR 0.40 at low-stealth α=0.1 (detected) → **AUC 0.509 / TPR 0.08 at the main α=0.9 setting (evaded)**. Same pattern on humanoid-walk: AUC 0.520, TPR 0.10 at FPR 0.10.
- PCA visualization: poisoned next-state targets overlap with true next states and don't form obvious latent outliers.
- TRIM robust training (β=0.2): SWAAP remains effective on Humanoid-Walk; the authors conjecture this is because the poisoned transitions are crafted to mislead TRIM into discarding clean transitions, **amplifying** the attack after training.
- DMP-data baseline (directly inserting Stage 1 target predictions without gradient matching): SWAAP is more effective, especially under TRIM, indicating the gradient-matching realization step is doing real work.

**Generalization ablations (Appendices G–H).**
- Trajectory-consistent variant on Humanoid-Walk: R = 784 ± 19, δ_m = 0.111 ± 0.003 at r_p = 0.05, δ_d = .102.
- DINO-WM Push-T: SWAAP reduces success rate after Stage 2 poisoning (initial evidence across world-model backbones).
- Gray-box transfer: poisoned data crafted in raw observation space with a single-task surrogate still degrades a multi-task TD-MPC2 victim with a different encoder.
- Larger fine-tuning buffers weaken but do not eliminate the attack when the number of poisoned samples is fixed.

## Critical Assessment

- **Novelty.** High. Prior RL poisoning work (Rakhsha et al., 2020; Zhang et al., 2020) targets reward / policy in tabular or low-dim settings and does not scale to deep latent dynamics models. The Stage-1 bilevel formulation via the Transition-Gradient theorem, plus the gradient-matched Stage 2 realization for *transition targets* (not labels), is a genuine new contribution.
- **Rigor.** Strong for the chosen threat model. The Stage 1 derivation is a proper theorem with proof in Appendix D.1 (deterministic finite-horizon and stochastic discounted infinite-horizon forms). The defense evaluation covers three independent stages. The Stage 2 loss is grounded in a well-known gradient-matching framework (Geiping et al., 2021). One weakness: the operating point in Table 1 is "stealth-prioritized" (α = 0.9, often 0.99), so headline return drops are modest. The "attacks" are not catastrophic — they are designed to be subtle.
- **Reproducibility.** Good. Algorithms, hyperparameters, and hardware costs are deferred to Appendices E–H. Threats include: Stage 1 is a non-trivial bilevel optimizer; reproducing exact numbers requires careful hyperparameter selection (α, r_p, BOME inner-loop count W). No code is mentioned in the arXiv abstract — hopefully released.
- **Significance.** High for the model-based RL / robotics safety community. The threat model (continual fine-tuning of world models from locally collected data) is realistic for deployed robots, edge devices, and simulation clients. Foundation world models (Genie, Cosmos, Sora-style) make this especially relevant.
- **Red flags / honest concerns.**
  1. **White-box assumption.** Main experiments use white-box access to the victim's ψ_0, architecture, and loss. The gray-box variant is relegated to a single paragraph in Appendix H.3 and is described as "preliminary" — not strong evidence the attack is practical against unknown victims.
  2. **Operating point is stealth-prioritized.** Headline drops are 6–25%, not catastrophic. The paper explicitly does not claim model-level indistinguishability. A defender who notices δ_m inflation (e.g., on MW-push, pen-twirl-hard) could potentially flag the attack post hoc.
  3. **TRIM "amplification" is a conjecture.** The authors speculate that TRIM removes clean transitions rather than poisoned ones because the poisoned ones are designed to mimic clean dynamics. This is a notable claim that deserves more careful ablation — is the effect robust across β and environments, or specific to Humanoid-Walk?
  4. **The realization bottleneck is unaddressed.** On Myo-Pen-Twirl-Hard, SWAAP cannot close the gap to Direct Model Poisoning. The paper honestly flags this as a limitation but does not solve it; some harmful targets are simply not realizable through gradient-matched data poisoning.
  5. **Adaptive defenses not evaluated.** The defenses are explicitly described as "non-adaptive." A defender who retrains a CUSUM with knowledge of the attack's α budget, or uses certified / smoothing-based robust training, is not considered.
  6. **Scale gap.** Evaluation is on small TD-MPC2 / DINO-WM, not foundation world models. The paper admits scaling is future work — but the practical threat to large models is the more interesting question.

## Strengths

- Cleanly motivated two-stage decomposition that mirrors successful patterns from the supervised poisoning literature and is easy to reason about.
- The Transition-Gradient theorem is a reusable, general theoretical contribution (not tied to a specific world-model class).
- Stealth is grounded in the clean model's *own* approximation error — a meaningful, architecture-aware notion of "normal" data.
- Defense evaluation across three independent stages is more thorough than typical single-detector evaluations in the poisoning literature.
- Honest about limitations: explicitly does not claim model-level indistinguishability, flags the realization bottleneck, and discusses unaddressed adaptive defenses.
- Wide benchmark coverage (DMControl, MyoSuite, MetaWorld, ManiSkill2) with multiple seeds and a trajectory-consistent variant for sequential data.

## Limitations

- White-box threat model in the main evaluation; gray-box transfer is preliminary and not extensively characterized.
- Headline return drops are 6–25% — meaningful but not catastrophic; the paper explicitly prioritizes stealth over attack strength.
- Two-stage decomposition has a realization bottleneck: Stage 1 target may not be reachable via Stage 2 data poisoning, especially in harder environments (Myo-Pen-Twirl-Hard).
- Only non-adaptive defenses evaluated. Adaptive defenders aware of the attack's α / r_p budget are not considered.
- Only latent-dynamics world models (TD-MPC2, DINO-WM) tested; stochastic transitions only discussed theoretically (Appendix D.3).
- Only untargeted attacks (performance degradation). Targeted behavior manipulation is explicitly out of scope.
- Stage 1 bilevel optimization is computationally expensive (policy rollouts + bilevel updates) — practical cost in larger models is unclear.
- Practical defenses in deployment (data provenance tracking, rate-limiting fine-tuning, ensemble world models) are not considered.

## Relevance

本文与"鲁棒 / 安全强化学习"和"基于世界模型的规划"两条研究主线高度相关；其方法可被复用为"世界模型鲁棒微调"的攻击模型，用于设计 certified 防御或影响函数式的异常检测，对在持续微调 / 第三方数据场景下部署 MBRL 智能体的研究者具有直接的安全启示价值。

## Questions for Further Exploration

- How does SWAAP scale to transformer-based or diffusion-based world models (DreamerV3-class, Genie, Cosmos)? Does the Transition-Gradient theorem extend, or do we need a reparameterization-trick-free estimator for stochastic latent dynamics?
- Can the realization bottleneck (Stage 1 → Stage 2 gap on environments like pen-twirl-hard) be closed by jointly optimizing target selection and data realization, e.g., via differentiable subset selection?
- What is the performance of *adaptive* defenses — e.g., CUSUM retrained with knowledge of the α budget, randomized smoothing of the fine-tuning loss, or ensemble-of-world-model disagreement as a poisoning signal?
- Can the framework be inverted to design a **certified** defense that bounds how much a fine-tuning batch can shift the world model (e.g., a (δ, ε)-robust world model)?
- Does the attack transfer across architecture families (e.g., MLP dynamics → transformer dynamics) via surrogate-based gradient matching?
- Is there a sharp phase transition in attack effectiveness as a function of r_p and α? The paper shows qualitative trends but not a clean phase diagram.
- Does the attack generalize to **targeted** behavior manipulation (e.g., steering the agent toward a specific harmful policy) in addition to untargeted performance degradation?
- How sensitive is the attack to the choice of clean reference model ψ_ref for the defender? A defender with a slightly different reference could in principle inflate δ_d and detect the attack.
- Does data provenance / signed-trajectory infrastructure (analogous to model weight signing) effectively neutralize this attack class in practice?
