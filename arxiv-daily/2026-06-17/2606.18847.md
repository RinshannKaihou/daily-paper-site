> **一句话总结：** 提出 WorldLines 长程具身家庭辅助基准（含 310 个 Memory QA 样本 + 21 个 Embodied Task Planning 样本）与 ObsMem 观察者视角记忆框架，通过"观察者门控 + 事件/状态/信念/承诺四轨"机制，将记忆检索从"找相似文本"转变为"按任务组合证据视图"，在 Memory QA 上 Judge 从 A-mem 的 0.575 提升到 0.713。

> **TL;DR:** WorldLines is a project-driven benchmark for long-horizon embodied household assistance built on Habitat/HSSD scenes, generating cross-day traces of dialogue, human activity, robot actions, device control, execution feedback, and state changes, then converting them into 310 evidence-linked Memory QA samples and 21 action-dense Embodied Task Planning samples. ObsMem is an observer-grounded memory framework that gates events by observation provenance and maintains four typed views (Event / State / Belief / Commitment) with an evidence selector that composes views per query. On Memory QA, ObsMem achieves Judge 0.713 vs. A-mem 0.575 (Perfect Rate 69% vs. 53%); on planning, ObsMem reaches Plan 0.684 vs. A-mem 0.542, with a particularly large gap on multi-hop state event grounding (+0.188). Ablations show the evidence selector is the most critical component (Judge drops 0.699 → 0.435 when removed) and belief tracking is essential for hidden-state questions (Hidden Judge 0.278 → 0.000 without it).

## 核心贡献
- 构建 WorldLines 长程具身家庭辅助基准，融合多日对话、人类活动、机器人动作、设备控制、执行反馈与物体/设备状态变化，生成可回放的证据链
- 设计 Memory QA（310 样本，6 个家族：状态单跳/多跳、时间、覆盖、偏好、来源）与 Embodied Task Planning（21 样本，平均 7.6 个目标动作、3.1 个状态约束）两种 evidence-linked 评测任务，采用 cutoff 控制避免未来信息泄露
- 提出 ObsMem 观察者视角记忆框架，包含四轨视图（Event Track 追加写、State Track 当前快照+历史、Belief Track fresh/stale/uncertain/contradicted 认识状态、Commitment Track 未来义务），通过 Observer Gate 区分观察到的与被报告的事件
- 实现基于查询规划的证据选择器，按当前状态/过去事件/未来承诺/不确定性等不同意图路由到不同视图，并组合多视图证据
- 系统实证：现有记忆系统（A-mem、Mem0、GraphMem、MemoryOS）虽能定位相关 session，但难以恢复精确状态变化事件；ObsMem 在多跳状态问题上 Event R@5 提升 0.188

## Key Contributions
- **Project-driven long-horizon benchmark** filling the gap between text-only long-term memory benchmarks (LoCoMo, LongMemEval, HaluMem, RealMem) and short-horizon embodied benchmarks (ALFWorld, PARTNR, BEHAVIOR-1K, MEMENTO)
- **Closed-loop trace generation pipeline** that combines a grounded Habitat/HSSD world, project planning, deterministic executor with affordance/precondition checks, and carry-forward notes between days
- **ObsMem observer-grounded memory framework** with four typed views updated by role-appropriate rules and queried by view-specific retrieval + a learned evidence selector
- **Empirical diagnosis** of three persistent failure modes: partial observability (Belief Track), overwritten world states (State history), and memory-to-plan translation (typed state trails into action preconditions)
- **Reproducibility signals**: Judge agreement with humans (Fleiss' κ = 0.71, GPT-4o 87.5% agreement), deterministic thresholds in belief rules, and full ablations on a 62-sample diagnostic subset

## Methodology

### WorldLines Construction
Each instance is formalized as `x_i = (H<ci, q_i, S_ci, E_i, y_i★, τ_i)`: visible pre-cutoff history, query, ground-truth state, evidence chain, reference answer, task type. Generation runs in four stages:

1. **World Grounding**: Habitat/HSSD scenes are curated into interactive objects/receptacles/controllable devices; represented in two views: a semantic view for project generation and an executable scene-graph view for affordance validation.
2. **Project Planning**: Multi-day life threads (household_routine, routine_support, household_organization, health_lifestyle, meal_preparation, work_study_support, digital_device_coordination, family_comfort) define participants, spaces, temporal preferences, and constraints.
3. **Closed-loop Simulation**: Per day d, context `C_d = (W, P_d, S_{d-1}, N_<d)` drives a generator (Director → Actor Step) that proposes dialogue, human actions, robot skills, and device operations; a deterministic executor validates entity references, affordances, and preconditions before recording state changes.
4. **Carry-forward Memory**: After each day, the system extracts notes (changed states, preferences, unresolved plans, conflicts) that feed the next day, ensuring continuity.

Memory targets (object_location, temporal_state, device_state, preference, routine, planning_dependency, hidden_state, social_context) are mined programmatically from indexed events/state timelines/entity histories; the LLM only rewrites them into natural language. Cutoffs prevent future leakage by hiding events after `c_i`.

### ObsMem Framework
- **Observer Gate (Eq. 1)**: Event `o_t` is ingested only if robot `r ∈ V_t` (visible); utterances from non-robot actors additionally create `e^rep_t` to mark reported (not observed) evidence.
- **Event Track**: Append-only historical evidence.
- **State Track (Eq. 2)**: For each (entity, attribute) `f_t = (i, a, v_t, ρ_t, τ_t)`, maintains history `H_t(i,a)` and current snapshot `Ŝ_t(i,a) = argmax_{f ∈ H_t(i,a)} τ(f)`; older facts remain for overwrite queries.
- **Belief Track (Eq. 3)**: Deterministic epistemic status `z_t(i,a) ∈ {fresh, stale, uncertain, contradicted}` based on intervening events `I_t`, actors `A_t`, and contradictions `C_t` with fixed thresholds `λ_I, λ_m, λ_A, γ_%, γ_)`. Decay: `c' = c - exp(-αn) · (1 - β·I_hidden)`.
- **Commitment Track**: Promises, requests, reminders, schedules with speaker/listener/target/due/status.
- **Episode Consolidation**: Boundary detection creates episode cards with summaries, factual/relational atoms, and commitments; summaries augment but never replace originals.
- **Query-Time Routing (Eqs. 4-5)**: `p_q` identifies intent/entities/attributes/temporal scope and selects views `V(p_q)`; State view prefers structured lookup over embedding search; an LLM-based evidence selector `Select_θ` ranks candidates for **complementarity** across timestamps, entities, and views (not pure similarity).

### Tasks
- **Memory QA** (310 samples): StateSingleHop, StateMultiHop, TemporalMemory, OverwriteMemory, PreferenceRoutine, SourceAwareness.
- **Embodied Task Planning** (21 samples, avg 7.6 actions, 3.1 state constraints): Plan, State, Precondition, Memory use.

### Experimental Setup
- **Baselines**: A-mem, Mem0, GraphMem, MemoryOS (all with same cutoff-controlled visible history).
- **Models**: All memory systems use `google/gemini-3.5-flash` for answer generation; `GPT-4o` as judge; `GPT-4o-mini` for sample construction (manually verified).
- **Metrics**: Judge (normalized), Perfect Rate (1.0 fraction), Session Any@5, Event R@5, plus state/temporal reasoning diagnostics.
- **Reliability**: 80-output human annotation, Fleiss' κ = 0.71, GPT-4o 87.5% agreement, Spearman 0.82.

## Results & Highlights

### Memory QA (Table 2, 310 samples)
| Method | Judge ↑ | Perfect ↑ | Sess. Any@5 ↑ | Event R@5 ↑ | StateMH-J ↑ | StateMH-E ↑ | StateSH-J ↑ | Temp-J ↑ |
|---|---|---|---|---|---|---|---|---|
| A-mem | 0.575 | 53% | 0.839 | 0.355 | 0.540 | 0.216 | 0.550 | 0.692 |
| Mem0 | 0.554 | 53% | 0.823 | 0.378 | 0.598 | 0.264 | 0.550 | 0.462 |
| GraphMem | 0.457 | 39% | 0.806 | 0.243 | 0.529 | 0.184 | 0.417 | 0.359 |
| MemoryOS | 0.312 | 29% | 0.452 | 0.085 | 0.287 | 0.086 | 0.350 | 0.308 |
| **ObsMem** | **0.713** | **69%** | **0.879** | **0.537** | **0.762** | **0.452** | **0.667** | **0.667** |

- ObsMem leads on 7/8 metrics; A-mem wins on TemporalMemory (0.692 vs. 0.667) — text-centric memory still works when temporal cues are explicit.
- Largest gap on **StateMultiHop Event R@5**: 0.452 vs. A-mem 0.216 (Δ +0.188) — the core claim that flat retrieval misses precise state transitions.

### Ablation (Table 3, 62-sample diagnostic subset)
| Variant | Judge ↑ | Perfect ↑ | Event R@5 ↑ | Hidden Judge ↑ | Latency (s) |
|---|---|---|---|---|---|
| Full ObsMem | 0.699 | 66% | 0.563 | 0.278 | 8.82 |
| w/o Belief | 0.651 (-0.048) | 63% | 0.558 | **0.000** | 8.63 |
| w/o State | 0.597 (-0.102) | 56% | 0.532 | 0.111 | 8.95 |
| w/o Consolidation | 0.554 (-0.145) | 53% | 0.419 | 0.167 | 9.16 |
| w/o Selector | 0.435 (-0.264) | 40% | 0.466 | 0.000 | 5.54 |

- **Evidence selector is most critical** (-0.264 Judge) — answer quality depends on view composition, not just retrieval.
- **Belief Track is essential for hidden-state questions** (0.278 → 0.000) — supports the "partial observability" claim.
- w/o Selector is also the only variant with substantially lower latency (5.54s), suggesting the selector is the dominant cost.

### Embodied Task Planning (Table 4, 21 samples)
| Method | Plan ↑ | State ↑ | Precond. ↑ | Mem. ↑ |
|---|---|---|---|---|
| A-mem | 0.542 | 0.566 | 0.581 | 0.524 |
| Mem0 | 0.526 | 0.551 | 0.563 | 0.512 |
| GraphMem | 0.481 | 0.493 | 0.507 | 0.462 |
| MemoryOS | 0.337 | 0.361 | 0.376 | 0.319 |
| **ObsMem** | **0.684** | **0.721** | **0.702** | **0.690** |

- Plan Judge gap of +0.142 over A-mem translates to better state/precondition/memory alignment — typed state trails help convert retrieved facts into executable action sequences.

### Case Study (Coffee Timer)
Day 1 routine 08:30 → Day 3 anomaly 04:00 → Day 4 correction 07:00. Mem0 retrieves "08:30 and 07:00" and confuses routine with anomaly. ObsMem's typed state trail separates routine=08:30, anomaly=04:00, correction=07:00 by entity/time/actor.

## Critical Assessment

### Novelty
- The combination of long-horizon memory evaluation + embodied household state tracking + evidence-linked cutoffs is genuinely under-served (Table 1 makes this honest: prior benchmarks split the two halves).
- ObsMem's four-typed-view decomposition (Event/State/Belief/Commitment) is a reasonable design that aligns with classical agent knowledge bases (Belief-Desire-Intent or epistemic state separation), and the observed-vs-reported provenance gate is a clean contribution.
- However, the underlying ideas — typed memory, episode consolidation, structured state, view-specific retrieval — are not new individually. The novelty is in their composition and the embodied, long-horizon, stateful evaluation context.

### Rigor
- **Strong**: deterministic executor for affordance validation; programmatic cutoffs; human verification of generated samples; reported judge reliability (Fleiss' κ, Spearman).
- **Weak**: all baselines use the same LLM backbone (`gemini-3.5-flash`) and the same prompt format — this is fair across systems but doesn't isolate whether ObsMem would still win on stronger/weaker backbones. The paper acknowledges this as a controlled comparison.
- **Concerning**: the **Evidence Selector is an LLM call** that is the single most impactful component (-0.264 Judge); this means much of ObsMem's gain could be attributable to extra LLM reasoning at query time, not to the memory architecture per se. A fairer ablation would replace the selector with a uniform top-k concatenation.
- The **belief-rule thresholds** `λ_I, λ_m, λ_A, γ_%, γ_)` are described as "fixed implementation hyperparameters" with no sensitivity analysis. Their interaction with trace complexity is unexamined.

### Reproducibility
- **Promising**: deterministic executor, programmatic evidence chains, threshold-fixed belief rules.
- **Gaps**: 310 QA + 21 planning samples is small. The benchmark is simulation-only with LLM-generated traces, so reproducibility depends on the exact generator prompts (only briefly described in §A.1, A.2). Habitat/HSSD scene curation is described as "manually curated" per family — not fully reproducible without the curatorial code.
- The plan-eval set of 21 action-dense samples is small enough that judge variance could dominate the Plan Judge gap of 0.142. No bootstrap CIs are reported.

### Significance
- The empirical pattern — text-centric memory has high Session Any@5 but low Event R@5 — is an important and well-diagnosed finding for the agent-memory community. It argues against "longer context = better memory" and motivates typed evidence.
- The case study of confusing a routine (08:30) with an anomaly (04:00) and a correction (07:00) is a clean, intuitive failure mode.

### Red Flags
- **"google/gemini-3.5-flash"** is referenced in §5.1 — this naming is unusual. It may be a typo, a future-projected name, or an internal alias. Readers should check the released code/repo for the actual model used.
- The ablation shows w/o Selector improves latency (8.82 → 5.54s) while losing 0.264 Judge, so ObsMem's gain is partially purchased with ~60% more compute. The cost-effectiveness is not discussed.
- Several cited works (e.g., Chu et al. 2026, Hu et al. 2026a/b, Maharana et al. 2024, Maharana et al.) carry 2026 dates — likely arXiv-2026 preprints in this forward-dated scenario, but readers should not assume these are peer-reviewed.

## Strengths
- Hits a real gap: long-term memory × embodied agents × partial observability in one framework.
- Evidence-linked, cutoff-controlled evaluation prevents leakage and is properly auditable.
- Four-view typed memory with provenance is a clean, modular design — each view's role is explicit.
- Honest diagnosis: Session Any@5 is high for baselines but Event R@5 is low, naming the actual bottleneck.
- Ablations decompose Belief / State / Consolidation / Selector individually, with a separate Hidden Judge for partial observability.

## Limitations
- **Small sample sizes**: 310 Memory QA + 21 planning samples. The planning set is small enough that Judge variance could obscure the 0.142 gap.
- **Simulation-only**: no real-robot logs, no visual observation noise, no actuation errors. Authors acknowledge this.
- **LLM-judge dependency**: GPT-4o judge with 87.5% human agreement — non-trivial fraction of samples may have noisy gold judgments.
- **Manual scene curation**: Habitat/HSSD scenes are "manually curated" per family; reproducibility of the benchmark depends on this curatorial step.
- **Hyperparameter opacity**: belief-decay thresholds `λ_I, λ_m, λ_A, γ_%, γ_)` are not ablated; the "8.82s latency" includes the heavy LLM-based evidence selector but no retrieval latency breakdown is given.
- **Backbone confound**: same `gemini-3.5-flash` for all systems — fair but doesn't show robustness across backbones.
- **Structured-trace assumption**: ObsMem requires entity IDs, visibility annotations, and action schemas — assumes the perception/localization modules provide these, which is non-trivial for real deployment.

## Relevance
直击具身智能体长期状态记忆这一核心难题，与"长期记忆 + 世界模型 + 智能体"研究方向高度相关；WorldLines 的"evidence-linked + cutoff"评估范式与 ObsMem 的"视图路由 + 信念追踪"设计对构建可审计的具身记忆系统有直接参考价值。

## Questions for Further Exploration
- How does ObsMem perform with a different backbone (e.g., GPT-4o-class or open-source 70B) — does the +0.138 Judge gap shrink or grow?
- Is the evidence-selector gain recoverable with a deterministic, non-LLM ranker (e.g., view-aware BM25 + recency)? This would isolate "memory architecture" from "extra LLM reasoning".
- How sensitive is the Belief Track to the thresholds `λ_I, λ_m, λ_A, γ_%, γ_)`? A sensitivity sweep would be valuable.
- Can the WorldLines generation pipeline scale to hundreds of households without manual scene curation? The current pipeline is "manually curated per family" which limits scale.
- How does the benchmark perform on real-robot logs where visibility annotations come from noisy perception rather than the simulator's `V_t`?
- The 21-sample planning set is small — can the authors release a larger action-dense evaluation, and are bootstrap CIs feasible?
- Can ObsMem's typed views be ported to a more retrieval-based pipeline (e.g., RAG with structured metadata) to reduce the 8.82s latency?
- The 2026-dated references (e.g., Chu et al. 2026, Hu et al. 2026a/b) — are any of these the same group? If so, citation overlap should be disclosed.
