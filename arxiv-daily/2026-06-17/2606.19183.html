Key Finding · Pediatric Clinical Decision Support

Split parse from predict — an LLM-parser plus XGBoost-predicter loses only 5.2% accuracy on shuffled clinical text, where end-to-end LLMs drop 28–30%.

Language Models as Interfaces, Not Oracles: A Hybrid LLM–ML System for Pediatric Appendicitis · Soheyl Bateni & Maryam Abdolali (K. N. Toosi Univ. of Technology) · 2026-06-17

Why this matters

For safety-critical clinical decisions, end-to-end LLMs are brittle to surface changes — order, phrasing, length. Decoupling language parsing from risk prediction is a transferable engineering pattern for any free-text-to-structured-prediction pipeline.

Critical edge

External specificity collapses to 39.4% — the "safety-oriented" operating point halves specificity for a handful of extra true cases, with no decision-curve justification and post-hoc calibration fit on the same cohort.

Strengths 7

  • Architectural split with explicit safety framing — FN minimization as the primary endpoint, not headline accuracy.
  • True external validation on Düsseldorf, untouched during development.
  • NaN-on-fail validator turns LLM hallucination into missing data, which XGBoost handles natively.
  • Same frozen Llama-3.1-8B drops from 21/57 FN end-to-end to 2/15 FN in the hybrid form — fine-tuning ruled out as the cause.
  • Sentence-order permutation is a fact-preserving, concrete robustness test that exposes a real LLM weakness.
  • Calibration reported with multiple metrics (Brier, ECE, intercept/slope, H-L) and limitations discussed candidly.
  • Baseline panel spans open, proprietary, and medical-specialized models with honest failure reporting.

Limitations 10

  • Synthetic-narrative evaluation: pipeline is tested on templated + LLM-rewritten text, not real clinician notes with abbreviations, typos, copy-forward.
  • Two German pediatric cohorts only; geographic, linguistic, and practice-pattern generalization unaddressed.
  • External specificity of 0.394 is operationally expensive — no decision-curve / net-benefit analysis to justify the operating point.
  • Calibration is poor on raw scores and the "fix" is post-hoc Platt on the same cohort — apparent ≠ transportable.
  • No prospective or shadow-mode evaluation; authors flag this as future work.
  • No subgroup analysis (age strata, sex, complicated vs. uncomplicated appendicitis).
  • Validator catches implausible values only — a confidently extracted wrong WBC will pass through unchanged.
  • Code and prompts not yet released, despite reproducibility-relevant claims.
  • OASST-Pythia-12B as a baseline is somewhat strawman-ish for a clinical paper.
  • Threshold sensitivity / operating-point robustness is not analyzed.

Questions for further exploration

  • Real notes: How does ClaMPAPP perform on actual clinician-written free-text, with abbreviations, typos, copy-forward artifacts?
  • Transfer: Can the same architecture (tool-call extract → NaN-on-fail → XGBoost → LLM report) transfer to other acute triage tasks — pediatric sepsis, intussusception, DVT, bowel obstruction?
  • External calibration: How do you fit Platt or isotonic recalibration on a held-out cohort, not the same one used to train XGBoost?
  • Marginal value: Is the LLM interface worth the latency/cost on top of a structured-data-only XGBoost in a prospective A/B study?
  • Model upgrades: How brittle is the frozen-extractor approach to LLM upgrades (Llama-4, future MedGemma, GPT-6)?
  • Throughput: What is the cost/latency of the tool-calling extraction step in production, and is it compatible with ED throughput?
  • Plausible-but-wrong: How do you extend the Feature Validator to catch extractions that are in-range but clinically wrong? LLM self-consistency? A second-pass verifier? Cross-field rules?
  • Subgroup risk: Is there a meaningful subgroup (e.g. age <5 with atypical presentation — the very group §1 cites as motivation) where the model underperforms?
  • Closing the gap: Could a tool-using LLM agent with retrieval + chain-of-thought close most of the gap to ClaMPAPP, or is the hybrid advantage structural?
  • Report drift: How should the clinician-facing LLM report be audited or constrained to prevent it from drifting into recommendation territory?
