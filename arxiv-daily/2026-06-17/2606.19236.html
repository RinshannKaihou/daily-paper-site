STARE · arXiv 2606.19236 / Reinforcement Learning · GRPO

A single multiplicative weight on high-surprisal tokens stops GRPO entropy collapse — and adds +4–8% on AIME24/25.

STARE 通过对 GRPO 进行 token 级熵的一阶梯度分析，识别出 advantage–surprisal 四象限结构和「near-criticality」性质，用 batch 内 surprisal 分位选出熵关键 token 集并对其优势函数做单/双侧缩放，配合目标熵的闭环门控，在 1.5B–32B 模型与 Short/Long CoT、多轮工具使用任务上稳定数千步训练。

02 Why This Matters

GRPO-style post-training collapses within ~500 steps. STARE is a one-line change inside the existing clipped surrogate — drop it into any GRPO pipeline to keep exploration alive from 1.5B to 32B.

03 Critical Edge

The near-criticality bound assumes unclipped, on-policy updates. PPO-style clipping can violate the O(T⁻¹) rate — and behavior under aggressive clip-higher or non-zero KL is untested.

04Methodology

First-order gradient diagnosis

Starting from GRPO's clipped surrogate (β = 0), the per-token entropy update decomposes cleanly:

The sign of Âᵢ × Φ(p) decides whether a token is entropy-raising or entropy-killing. This is the mechanistic root cause of collapse, not a symptom — the paper's main diagnostic contribution.

Near-criticality — the sharpest result

Scaling advantages on the L⁺ set by W > 1 yields a net entropy shift of −(1/N)[Λ − (W−1)Γ]. The Entropy Neutrality Identity forces |Λ| / Σabs = O(T⁻¹), so:

W* − 1 = Λ / Γ = O(T⁻¹)

In the paper's 7B+ Long-CoT regime this is empirically ≈ 1.1×. Beyond the threshold, W controls magnitude, not sign — STARE is robust to hyperparameter choice.

Algorithm — 5 steps per batch

  1. Compute per-token surprisal si,t = −ln πθ(oi,t | x, o<t).
  2. Within Â > 0 tokens, take top-P% by surprisal → L⁺. Analogously L⁻ for Â < 0. (P = 10%, O(sort).)
  3. Apply ω = 1 + gk(W − 1) to L⁺, and ω = 1 − gk(1 − M) to L⁻. (W = 1.1, M = 0.9.)
  4. Compute closed-loop gate gk = 𝟙[H̄k < Htgt], Htgt = 0.3.
  5. Multiply the GRPO clipped surrogate by ω before the per-token min — gradient sign and clip range preserved.

06Strengths · Limitations

Strengths 6 points

  • Principled. Closed-form gradient decomposition, not a heuristic — every coefficient has a theoretical role.
  • Plug-in. A single multiplicative token weight inside the existing GRPO clipped surrogate. No rearchitecting, no reward shaping, no extra heads.
  • Closed-loop safety. The binary target-entropy gate prevents both collapse and divergence. Confirmed empirically across a wide W range.
  • Cross-scale, cross-task. 1.5B–32B × {Short CoT, Long CoT, Multi-Turn Tool-Use} — the most thorough evaluation matrix in this niche.
  • Diagnostic transparency. Four-quadrant view, eight-variant ablation, 60%→95% validation of the surprisal-quantile proxy vs. theoretical s*, and a word cloud of reweighted tokens.
  • Public code. github.com/hp-luo/STARE + fixed config (W = 1.1, M = 0.9, Htgt = 0.3, P = 10%).

Limitations 7 points

  • Unclipped-regime assumption. Near-criticality proof requires no importance-ratio clipping — partially violated in practice.
  • Single-seed runs. Main-text curves show one run each; AIME24/25 are notoriously noisy at small batch sizes.
  • Mixed baselines. Some numbers are cited from prior papers (†) rather than re-run with identical compute.
  • β = 0 throughout. Production RLVR pipelines use a non-zero KL; STARE's interaction with KL regularization is not characterized.
  • Fixed Htgt. 0.3 is a hyperparameter — adaptive variant exists but is not the main story.
  • Reasoning-only evaluation. Alignment, summarization, and execution-feedback code generation are untested — generality claim unsupported beyond math/tool-use.
  • Long-CoT data dependence. Cold-start SFT and the 100K-sample mixture composition are summarized, not ablated.
07Questions for Further Exploration
Q1 ›How does STARE compose with a non-zero KL penalty (the standard production setting)? Does the closed-loop gate's Htgt need re-tuning, or does the KL term already prevent collapse?
Q2 ›Does the W* − 1 = O(T⁻¹) near-criticality rate survive importance-ratio clipping and aggressive off-policy updates?
Q3 ›Can the four-quadrant view explain non-reasoning RLVR failure modes, or is the effect size math-specific because of unusually peaked output distributions?
Q4 ›Does the gate's binary nature cause oscillation in multi-turn tool-use agents, and would per-sample or per-token gating be more appropriate?
Q5 ›Could the surprisal-quantile proxy be useful as a training-health observability tool, independent of applying the reweighting?
Q6 ›As context windows grow, does the O(T⁻¹) rate imply the required W should decrease — and is that visible empirically?
