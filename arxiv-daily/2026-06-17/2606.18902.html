No single optimizer wins — error structure, not landscape ruggedness, decides whether random search, genetics, or agent-guided diagnosis prevails.

SAGE: Stochastic Prompt Optimization via Agent-Guided Exploration

Ziyi Zhu · Luka Smyth · Saki Shinoda · Jinghong Chen  —  Slingshot AI · University of Cambridge

Key Metric
+29.4%

Cumulative D1 retention gain on the Ash mental-health chatbot, chained across 8 production A/B cycles.

95% CI clears zero from cycle 5 onward.

Why This Matters

Chain-of-deltas aggregation turns 8 individually-underpowered A/B cycles into one credible production win — a reusable recipe for any team iterating live LLM products.

Only 2/8 cycles were individually significant.

Critical Edge

"Promote the best of Q arms" admits upward selection bias the authors flag but never quantify — no Bonferroni or held-out cumulative CI.

N=1 chatbot, no control, confounded by model drift.

Methodology

Automatic prompt optimization is reframed as noisy black-box stochastic search over a low-dimensional, semantically-smooth landscape. All three strategies share one monotonic hill-climbing outer loop so observed gaps are attributable to strategy, not scaffolding:

pk+1 = arg maxp ∈ {pk} ∪ S(pk,D) R̂(p)   (K=10, P=2, Q=3)

  • SPO-RS — single high-temperature sample (τ=0.9) conditioned on an error summary; cheap residual search.
  • SPO-GA — tournament selection + crossover (ρ=0.3, τ=0.7); targets shared failure modes via a meta-model. EvoPrompt/GEPA lineage.
  • SAGE — 6-step multi-agent pipeline (Orchestrator + Analyzer + 5 Investigators + 3 Generators) where code execution does statistical diagnosis (clustering, exact counts, trace inspection), not procedural construction. Key distinction from MCE.

Target LLM = DeepSeek-V3.1 (fixed across all methods). Meta-model = Claude Sonnet 4.6 for every SPO variant. Landscape structure quantified by a semivariogram range parameter a.

Formula & AppWorld are equally rugged (a≈0.17–0.18) yet favor different winners → ruggedness alone is not predictive; error structure is. On FiNER SAGE overfits: training acc peaks but its prompt bloats to ~12K tokens with poor test transfer, while SPO-RS stays lean at ~3K.

Strengths

  • ✓ Intellectual honesty on the "textual gradients" myth — refuses to pretend the analogy holds.
  • ✓ Ships a random-search baseline (SPO-RS) that most APO papers skip, properly testing whether structure helps at all.
  • ✓ Honest single-cycle underpowering analysis; the chain-of-deltas method is the headline, not a lucky A/B.
  • ✓ Production bar is unusually high: clinical-expert review + automatic rollback + BRANY IRB oversight.
  • ✓ No-dominance result framed as a design principle (match optimizer depth to error structure), not a weakness.
  • ✓ Concrete prompt excerpts in Appendix F ground the abstract +29.4% in inspectable behavior changes.

Limitations

  • ✗ Three benchmarks, all financial/agent — no coding (HumanEval), math (GSM8K), or multilingual coverage.
  • ✗ SAGE's compute cost is never reported; H=8 + I=5 + Q=3 × 10 iterations is plausibly 10–50× SPO-RS. No cost-vs-gain plot.
  • ✗ Ash deployment is N=1 with no control chatbot; +29.4% is inseparable from concurrent model updates & seasonality.
  • ✗ D1 retention is a proxy — clinical outcomes (PHQ-9, working alliance) are too slow to A/B per cycle.
  • ✗ "Error structure is decisive" is asserted, not manipulated; ruggedness held constant across Formula/AppWorld but error type not.
  • ✗ All SPO variants use Sonnet 4.6 as meta-model; vs-ACE comparison is a stack confound, not a clean strategy test.

Questions for Further Exploration

  • What is SAGE's dollar cost per iteration, and on which tasks does the cost-vs-accuracy Pareto frontier favor SPO-RS or SPO-GA instead?
  • How does the no-dominance rule change on non-financial benchmarks (HumanEval, GSM8K, multilingual)? Is "error structure" the right axis there?
  • Does the chain-of-deltas aggregation transfer to other production LLM apps (recommender, search rewriter, RAG retriever tuning)? What is the minimum cycle count to clear zero?
  • SAGE's FiNER prompt bloated to ~12K tokens and overfit — can a token-budget constraint or compression step be added without losing the diagnostic edge seen on Formula/AppWorld?
  • Should the chain-of-deltas framework incorporate a measurable-harm threshold (e.g. issue-rate spike) rather than optimizing monotonic retention?
  • SAGE is structurally a refiner, not a discoverer. Can a hierarchical design (human strategist → SAGE operationalization) recover the missing novelty axis?
  • Is SAGE's diagnostic-code benefit a property of the strategy, or just of having a strong-enough meta-model to write good diagnostic code? What changes with GPT-5 or a 7B open model?
