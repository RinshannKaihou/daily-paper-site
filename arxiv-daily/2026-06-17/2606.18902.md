> **一句话总结：** 将自动提示优化重新形式化为带"昂贵评估 + 低有效维度 + 局部结构"的黑盒搜索问题，提出 SPO 框架（含 SPO-RS、SPO-GA、SAGE 三种递进策略），其中 SAGE 首次把代码执行用于统计诊断而非过程性构造；在 Formula/FiNER/AppWorld 上 SAGE/SPO-RS/SPO-GA 各擅胜场，并在真实心理健康聊天机器人 Ash 上以 8 轮 A/B 测试链式复合出 +29.4% 的 D1 留存累计增益。

> **TL;DR:** SPO formalizes automatic prompt optimization as noisy black-box stochastic search over a low-dimensional, semantically-smooth landscape, then compares three strategies — error-informed random search (SPO-RS), a genetic algorithm (SPO-GA), and a multi-agent pipeline with diagnostic code execution (SAGE) — on three benchmarks and a live mental-health chatbot. The benchmarks deliver a clean No-Free-Lunch result: SAGE wins on Formula (86.5% vs ACE's 85.0), SPO-RS wins on FiNER (77.5%), SPO-GA wins on AppWorld (79.8%); landscape *ruggedness* alone doesn't predict the winner, the *error structure* does. The Ash deployment is the more novel contribution: 8 SAGE cycles chained as 48h A/B tests produce a +29.4% cumulative D1 retention gain whose propagated 95% CI clears zero from cycle 5 onward, even though only 2 of the 8 individual cycles were individually significant.

## 核心贡献
- 将自动提示优化（APO）重新理论化为**带噪、低有效维度、昂贵评估**的黑盒搜索问题，并借用地质统计学中的半变异函数（semivariogram）量化任务间的景观结构差异（Formula/AppWorld 范围 a≈0.17–0.18 属"崎岖"，FiNER a≈0.66 属"平滑"）。
- 提出 SPO 统一框架，统一表示三种递进的搜索策略：SPO-RS（高温度残差搜索，τ=0.9）、SPO-GA（遗传算子，ρ=0.3，τ=0.7）、SAGE（多智能体 + 诊断性代码执行），共享"随机爬山 + 单调保留"的外部循环。
- 明确区分 SAGE 与同类多智能体方法 MCE（Ye et al., 2026）的本质差异：**MCE 用代码执行构造过程性产物（检索器、模板），SAGE 用代码执行做统计诊断**（聚类、显著性检验、错误模式计数）。
- 提出"连续优化"（continuous optimization）部署范式：每轮在真实流量上跑 SAGE → 同时 A/B 多个候选 → 取最佳晋升，使得单轮统计功效不足的 A/B 试验能通过链式叠加产生稳健的累计信号。
- 在生产环境（Slingshot AI 的 Ash 心理健康聊天机器人）实证 8 轮迭代得到 **+29.4% 累计 D1 留存提升**，且累计 95% 置信带自第 5 轮起明确脱离零线。

## Key Contributions
- **Theoretical reframing**: APO is noisy black-box stochastic search with low effective dimensionality, expensive evaluations, and rich local structure (formalized in §2.2 and §3.1). The "textual gradients do not behave as real gradients" finding (Melcer et al., 2025) is absorbed as a corollary, not a starting point.
- **Unified controlled ablation**: SPO-RS, SPO-GA, SAGE all share the same outer hill-climbing loop (Eq. 2), so observed differences are attributable to *strategy* rather than to a different optimizer scaffold.
- **Diagnostic code execution** as a *first-class* primitive for prompt search: agents run scripts that cluster errors, compute exact counts, and cross-compare prompts — used to *inform* edits, not to *construct* the optimizer itself (the key distinction from MCE).
- **Empirical landscape analysis**: semivariogram-fitted range parameter a differentiates the three benchmarks; the paper then shows that *ruggedness alone* does not predict which optimizer wins (Formula and AppWorld are both rugged but favor different optimizers), and proposes **error structure** as the decisive axis.
- **Continuous optimization paradigm**: an 8-cycle production A/B protocol on a real mental-health chatbot, with the principled chain-of-deltas aggregation formula (Eq. 4–5) for cumulative credible intervals. Code released at https://github.com/slingshot-ai/sage.

## Methodology

**Problem formulation.** Maximize `p* = arg max E_{x~D}[R(p,x)]` over discrete prompt space P, where R is noisy (LLM non-determinism) and expensive (full-dataset evaluation per candidate). The shared outer loop is stochastic hill climbing with monotonic selection:

`p_{k+1} = arg max_{p ∈ {p_k} ∪ S(p_k, D)} R̂(p)`

ensuring the incumbent is never regressed. K = 10 iterations, population P = 2, candidates Q = 3 per iteration.

**Three strategies (escalating analytical depth):**

1. **SPO-RS (Residual Search)**: A single high-temperature (τ = 0.9) LLM sampling conditioned on the incumbent prompt + a structured error summary. Effective dimensionality reduction via the error summary + exploration via temperature.
2. **SPO-GA (Genetic Algorithm)**: Tournament selection over a maintained top-P population. Crossover (⌈ρQ⌉ children, ρ = 0.3) presents two parents and their error summaries to a meta-model; mutation (the rest) targets shared failure modes. Lower temperature τ = 0.7 trades diversity for targeting. Follows the EvoPrompt/GEPA paradigm.
3. **SAGE (Agent-Guided Exploration)**: A fixed 6-step multi-agent pipeline coordinated by an Orchestrator, with three additional roles:
   - **Analyzer**: runs pre-built analysis scripts across the *full* evaluation results of all top-P prompts to surface H = 8 evidence-backed hypotheses.
   - **Investigators** (I = 5, in parallel): each verifies one hypothesis with exact counts, reasoning-trace inspection, and cross-prompt comparison.
   - **Generators** (Q = 3, in parallel): each produces a complete revised prompt grounded in one investigation report.
   The Orchestrator triages hypotheses (specificity, impact, diversity, prompt-addressability) and selects parent prompts.

**Evaluation infrastructure.** Target LLM = DeepSeek-V3.1 (identical across all methods to isolate strategy effects). Meta-model = Claude Sonnet 4.6 for all SPO variants. Agents are built on the Claude Agent SDK with file I/O + shell execution + code search toolkits.

**Benchmarks.**
- **XBRL Formula** (Zhang et al., 2025): NL→XBRL expression translation for financial formulas. 500 train / 200 test.
- **FiNER** (Shah et al., 2023): fine-grained financial NER with 7 entity categories. 1,000 train / 441 test.
- **AppWorld** (Trivedi et al., 2024): autonomous agent tasks over real apps. 90 train / 57 val / 168 test-normal. ReAct framework. Task Goal Completion (TGC) metric.

**Baselines.** Base LLM (no opt), ICL (few-shot), MIPROv2 (Bayesian joint search), GEPA (reflective prompt evolution), Dynamic Cheatsheet (DC-CU, test-time adaptive memory), ACE offline (Generator–Reflector–Curator).

**Production deployment.** Ash (talktoash.com), a mental-health support chatbot by Slingshot AI, optimizing the system prompt encoding its conversational strategies and safety protocols. Primary metric: D1 retention (next-day return). Secondary: engagement, issue rate (Zhu et al., 2026), user ratings. Each cycle ≈ 48 h A/B test; clinical-expert review of every SAGE candidate before deployment; real-time safety monitoring with automatic rollback on crisis-protocol non-adherence; BRANY IRB exempt (file #26-081-2393).

**Significance test.** Bayesian chance-to-beat-control `P(beat) = Φ(δ/SE_δ)` with Gaussian delta-method SE (Eq. 5). Cumulative credible interval chains log-scale per-cycle variances.

## Results & Highlights

**Table 1 — Test accuracy across three benchmarks:**

| Method        | Formula | FiNER | AppWorld |
|---------------|---------|-------|----------|
| Base LLM      | 67.5    | 70.7  | 63.7     |
| ICL           | 67.0    | 72.3  | 64.3     |
| MIPROv2       | 69.5    | 72.3  | —        |
| GEPA          | 71.5    | 73.5  | 64.9     |
| DC-CU         | 69.5    | 74.1  | 65.5     |
| ACE offline   | 85.0    | 76.9  | 76.2     |
| **SPO-RS**    | 85.5    | **77.5** | 78.0  |
| **SPO-GA**    | 85.5    | 76.2  | **79.8** |
| **SAGE**      | **86.5** | 76.5  | 79.2     |

- **Formula** → **SAGE wins** (86.5). Gap to ACE = +1.5. The "cascading reasoning errors" reward root-cause diagnosis.
- **FiNER** → **SPO-RS wins** (77.5). Note: SAGE attains the highest *training* accuracy (per §5.1) but its prompt grows uncontrolled to ~12K tokens (Figure 6b) by accreting disambiguation rules — the textbook overfitting signature, with poor test transfer.
- **AppWorld** → **SPO-GA wins** (79.8). Both equally rugged (Table 3: Formula a=0.174, AppWorld a=0.180), but procedural per-task errors reward SPO-GA's structural crossover. Notably, SAGE's prompt on AppWorld is the *shortest* (~1.5K) because diagnostic analysis localizes a fix rather than appending instructions.

**Landscape analysis (Table 3):** Range a discriminates the regimes — FiNER a=0.663 (smooth; a disambiguation rule that helps one taxonomic family helps others), Formula a=0.174, AppWorld a=0.180 (both rugged). But ruggedness alone does not predict the winning strategy; **error structure** does.

**Production (Ash) — per-cycle A/B D1 retention deltas (Table 2):**

| Cycle | Promoted change                | ∆D1     | P(beat) |
|-------|--------------------------------|---------|---------|
| 1     | Initial SAGE prompt            | +5.91%  | 0.88    |
| 2     | First-session depth pacing     | +2.45%  | 0.82    |
| 3     | (incumbent retained)           | —       | —       |
| 4     | Depth pacing + jargon ban      | +1.52%  | 0.63    |
| 5     | Honest over agreeable          | +5.12%  | 0.98 ✓  |
| 6     | Distinctive writing voice      | +1.77%  | 0.77    |
| 7     | Vary moves across turns        | +3.09%  | 0.92    |
| 8     | Active identity construction   | +6.53%  | 0.98 ✓  |

Cumulative ΔD1 = **+29.4%** with propagated 95% CI clearing zero from cycle 5. Only 2/8 cycles individually significant — the headline is the *chain* method, not any individual A/B.

## Critical Assessment

- **Novelty**: Modest on the strategy level — SPO-RS is essentially a higher-temperature ProTeGi/OPRO, SPO-GA is EvoPrompt/GEPA with error summaries, and "agents that run code" is a crowded 2025–2026 subfield. The genuine novelty is the *head-to-head of all three inside one controlled scaffold with one meta-model* (which most APO papers avoid because they would prefer to claim their own method wins). The continuous-optimization + chain-of-deltas deployment recipe is also new and well-articulated.
- **Rigor**:
  - **The "Bayesian P(beat)" diagnostic is honest** about single-cycle underpowering — they report 0.82, 0.77, 0.92 deltas as inconclusive rather than inflating them.
  - **However, the "promote the best of Q arms" step admits modest upward bias** (their own footnote in §5.3). The cumulative +29.4% therefore has both genuine signal *and* a small selection-bias contribution that the paper does not attempt to quantify (no Bonferroni or held-out cumulative CI).
  - **Same meta-model for all SPO variants but different from MIPROv2/GEPA/ACE**: this is *good* for isolating strategy differences among SPO variants, but bad for the absolute comparison vs. ACE (which uses its own optimization protocol). The fact that *all three* SPO variants beat ACE on all three benchmarks is therefore a method stack comparison (Sonnet 4.6 as meta-model + SPO loop), not a clean strategy comparison. The paper is transparent about this but it somewhat blunts the claim that "SAGE > ACE."
  - **SAGE's compute cost is not reported** in the paper. With H=8 hypotheses, I=5 investigators, Q=3 generators per iteration × 10 iterations, that's potentially 160+ Sonnet-4.6 calls per benchmark run. The claim that "SAGE wins where analytical depth pays off" is presented without a cost-vs-gain Pareto plot.
- **Reproducibility**: Code is open-source, meta-prompts partially shared, hyperparameters (τ, ρ, H, I, Q, P, K) all stated. But the "code runs on the Claude Agent SDK" is an external dependency; the diagnostic scripts that agents run are referenced but not enumerated. Partial reproducibility.
- **Significance**: The continuous-optimization paradigm is the most transferable contribution — practitioners running A/B tests on LLM products are routinely discarding individually-noisy cycles. The chain-of-deltas method is applicable well beyond prompt optimization.
- **Red flags to flag**:
  - **Cherry-picked benchmarks**: three financial/agent benchmarks, no coding benchmarks (HumanEval/MBPP), no reasoning benchmarks (GSM8K), no multilingual. The no-dominance claim would be more compelling with broader task coverage.
  - **All baselines use the same DeepSeek-V3.1 target, but the meta-model difference (Sonnet 4.6 vs. whatever MIPROv2/GEPA/ACE use internally) is a confound**: ACE reaching 85.0 on Formula with presumably a *different* meta-model is not apples-to-apples.
  - **The "ruggedness is not the right axis; error structure is" refinement is asserted, not formally tested.** The paper shows Formula and AppWorld are both rugged with different winners, but does not run a controlled manipulation of error structure independent of ruggedness.
  - **The 8-cycle Ash deployment is an N=1 production case study** with no control chatbot. The "honest over agreeable" / "active identity construction" promoted changes are reasonable but their *causal* contribution to the +29.4% is inseparable from concurrent model updates, user-population drift, and seasonality.

## Strengths

- **Intellectual honesty about the "textual gradients" framing** — does not pretend gradient analogies hold.
- **Includes a random-search baseline (SPO-RS)** that explicitly tests whether structure helps — most APO papers skip this and report only their own method vs. an LLM-only baseline.
- **Honest single-cycle underpowering analysis** that demonstrates the value of the chain-of-deltas aggregation rather than hiding behind a single lucky A/B win.
- **Production deployment with clinical-expert review + automatic rollback + IRB review** is a much higher process bar than typical LLM-application papers.
- **No-dominance conclusion is presented as a *design principle*** (match optimizer depth to error structure) rather than a weakness — practically useful framing for practitioners choosing between APO methods.
- **Concrete prompt excerpts in Appendix F** (e.g., the "let the user lead the depth" directive) ground the abstract +29.4% in inspectable behavior changes.

## Limitations

- **Three benchmarks, all in financial/agent domains** — generalizability of the "SAGE wins on rugged, SPO-RS on smooth, SPO-GA on procedural" rule to coding, math, multilingual, and open-ended generation is unverified.
- **SAGE's compute cost is not reported** — the multi-agent pipeline (H=8 + I=5 + Q=3 per iteration) is plausibly 10–50× more expensive per iteration than SPO-RS, and the paper does not put this in a cost-vs-gain plot.
- **The Ash deployment has no counterfactual** — the +29.4% could partly reflect concurrent improvements (the chatbot evolves while the experiment runs, as the paper itself notes in the *Limitations* section).
- **D1 retention is a proxy, not the outcome of interest** — the paper is candid that clinical outcomes (symptom change, working alliance) are too slow/noisy to A/B test per cycle, but this means we do not know whether the prompt changes actually help users or merely improve stickiness.
- **"Bounded novelty" is acknowledged but not solved** — SAGE can only refine behavior patterns already present in the data; genuinely new conversational strategies must come from external domain experts.
- **No head-to-head with the strongest current agentic systems** (e.g., recent OPRO, TextGrad, MCE) — the ACE baseline is the strongest of those compared, and even then with a different meta-model.
- **The paper's data is exclusively from 2026-vintage models** (DeepSeek-V3.1, Claude Sonnet 4.6) — behavior of these methods on GPT-5-class or open-weights-7B models is unverified.

## Relevance
与"LLM 自身做提示/系统优化"、"黑盒随机搜索"以及"工业级 LLM A/B 部署"方向高度相关；"连续优化"链式聚合统计功效不足的 A/B 试验的方法对在生产环境迭代 LLM 应用的团队具有直接参考价值；与作者心理健康聊天机器人方向的关注点（PHQ-9 改善、对话质量评估、用户留存）紧密对应。

## Questions for Further Exploration

- What is SAGE's *wall-clock and dollar cost per iteration*, and on which tasks does the cost-vs-accuracy Pareto frontier favor SPO-RS or SPO-GA? The paper lacks this plot.
- How would the no-dominance conclusion change on non-financial/agent benchmarks (HumanEval, GSM8K, multilingual summarization)? Is "error structure" the right axis there too, or is it specific to the financial/agent domain?
- The chain-of-deltas aggregation method (Eq. 4–5) is a generic recipe for compounding underpowered A/B tests. Does it transfer cleanly to other production LLM applications (recommender LLM, search rewriter, RAG retriever tuning)? What is the minimum cycle count for the cumulative CI to clear zero in practice?
- SAGE on FiNER grew prompts uncontrolled to ~12K tokens with worse test transfer than SPO-RS at ~3K tokens. Can a token-budget constraint or compression step be added to SAGE without losing the diagnostic-edge benefit it shows on Formula and AppWorld?
- The Ash deployment uses D1 retention as the A/B metric, but the paper cites Wolfe et al. (2026) showing engagement is associated with PHQ-9 improvement. Is there a *measurable harm* threshold (e.g., when issue rate goes up) that the chain-of-deltas framework should incorporate, or is monotonic retention the right single-axis target?
- The "bounded novelty" limitation implies SAGE is structurally a *refiner*, not a *discoverer*. Can a hierarchical design (external human strategist → SAGE for operationalization) recover the missing novelty axis? The paper gestures at this but does not instantiate it.
- All three SPO variants use Sonnet 4.6 as the meta-model. How does SAGE's lead on Formula and the overfit on FiNER change when the meta-model is GPT-5, Gemini 2.5 Pro, or a much smaller open model? Is the diagnostic-code-execution benefit a property of the *strategy* or of having a strong-enough meta-model to write good diagnostic code?
