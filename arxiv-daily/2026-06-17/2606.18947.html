01

Methodology

distilled

DSG (Decoupled Search Grounding) is an MCP-compatible gateway between agentic applications and interchangeable reasoning models, fronting provider-agnostic search backends (Serper, BrightData, Firecrawl, Exa, Tavily; new HTTP providers onboarded via YAML adapter). It externalizes retrieval policy into a first-class subsystem: source-aware tool formatting (normalized Result {title,url,content} records with provenance, exposed as discrete tool responses), a Provider Registry with configured fallback chains, and a cache-gated decision — exact cache → semantic cache via embedding similarity ≥ τ → fallback chain (Algorithm 1). Cache entries are provider-scoped with TTLs. Evaluated across 5 frontier models (GPT-4o, GPT-4o-mini, Gemini 2.5 Flash/Pro, Claude Sonnet 4) on SimpleQA, FreshQA, HotpotQA, plus DoorDash production QIU workloads. Scoring uses GPT-4.1 as judge with 3-judge validation (97.78% agreement) for factoid tasks and deterministic EM/F1 for HotpotQA.

03

Strengths & Limitations

side by side

Strengths

  • Tackles a real, painful production problem (vendor lock-in + SIV breakage) with a concrete, deployed solution.
  • Five-model × three-public-benchmark × two-production-workload matrix is unusually thorough for an industry paper.
  • Names and characterizes Search-Induced Verbosity as a distinct failure mode; gives a reproducible diagnostic.
  • QIU cost savings (>98%) are large enough to be commercially material.
  • Architecture is conceptually clean: source-aware formatting, YAML provider registry, cache-gated decision, fallback chain.

Limitations

  • Native search leads on recency-sensitive tasks (FreshQA gap ≈ 4.6 pp mean).
  • Cold-path latency is worse than native for several models (high P95s).
  • Single production workload (e-commerce QIU); generalization to chat / code / tool-heavy agents unproven.
  • Multi-hop gains are modest; not a substitute for iterative-retrieval methods (Self-RAG, IRCoT).
  • No public code/artifact at submission; LLM-as-judge circularity risk; cost numbers are point-in-time.
