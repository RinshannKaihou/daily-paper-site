The MCC pipeline — meaning offline, math online

3 stages · predictor-agnostic
1

Measurement semantics

Offline · LLM · frozen

An LLM reads process docs (names, units, roles, upstream/downstream) and emits per-variable semantic cards, frozen into a direction matrix U. The LLM never sees online data and outputs no scores or thresholds.

2

Leave-one-out observer

Online · numerical · 0.5–2.0k params

For target x_i, builds an independent reference only from other semantically-qualified variables and their lags. A variable can never vouch for itself — correlation ≠ credibility.

3

Gate & correct

Pre-inference · conservative

Background-median subtraction suppresses shared operating shifts; a κ=0.95 calibration gate fires only on rare, externally-supported local conflicts. Clean windows get near-zero activation (0.002).

Methodology

3 ordered training steps · observer fit on nominal data only

MCC is a pre-inference adapter bolted between the raw measurement window and any multivariate time-series predictor G_θ. The design rests on a strict separation: the LLM does the meaning offline, a numerical model does the math online.

Offline, an LLM converts side information (scenario, variable names/units, sampling, process role) into a measurement-semantic description z_i per variable, rendered and embedded (text-embedding-v4, k=128) into a normalized direction u_i. The stacked matrix U is frozen and never updated online. The LLM emits no credibility scores, thresholds, or corrections.

Online, a tiny observer computes semantic activation, compatibility c_ij (for j≠i), and top-k (=6) routing weights to form the leave-one-out reference x̄*_i. Readout coefficients β_ij are constrained unbiased (no free bias). Observer params are fit on nominal data by minimizing (x̄*_i − x̄_i)² in just 3 epochs, lr 5e-3.

At run time, per-reference residual scale s_i, reliability η_i=1/(1+s_i), standardized inconsistency r_i, and background-subtracted χ_i feed a calibration-quantile gate. Correction strength a_i=[(q_i−κ)/(1−κ)]_clamp is zero unless q_i>κ, blending x̄_i toward x̄*_i only when warranted.

Datasets · 3 processes

  • Ladle Preheating — Maanshan steel, 14 vars, 5-min, target = next 5 steps of inner temperature. Tests complete-process holdout.
  • Thickener Dewatering — Nanjing tailings, 5 vars, 1-min, target = underflow concentration. Tests far-domain regime shift.
  • IndPenSim V3 — public penicillin fed-batch, 23 vars, 12-min, target = assay-time penicillin. Tests control-policy + fault batches.

8 backbones

GRU · LSTM · Transformer · Informer · Mamba · iTransformer · PatchTST · ModernTCN

Each dataset has a Real Test split (fixed) and a Corrupted Test split (bias/drift/gain/spike/dropout/global-shift, injected on the real subset only).

Strengths & Limitations

side-by-side

✓ Strengths

  • Elegant separation of concerns — LLM does meaning offline, numerical model does math online: full auditability, zero LLM latency at inference.
  • Leave-one-out references provably prevent self-confirmation; background-median subtraction provably suppresses shared shifts (gate = 0 on the 22-variable shift).
  • Backbone- & predictor-agnostic — corrected window is a drop-in replacement across all 8 backbones.
  • Fine-grained ablations actually attribute the gain to semantics + calibrated local correction, not extra capacity.
  • Tiny online footprint (0.5–2.0k params, <0.09 ms/step) makes plant deployment realistic.

! Limitations

  • Depends entirely on document quality — incomplete/outdated docs corrupt frozen U; no automatic drift detection, deferred to future work.
  • No comparison with classical/learned correction (data reconciliation, gross-error detection, GDN, BeatGAN) — only Base vs. +MCC.
  • Two of three datasets are proprietary (ladle, thickener are author-owned); only IndPenSim is public.
  • Corrupted Test is author-defined, no link to a community robustness benchmark.
  • Modest/ambiguous Thickener Real-Test gains (R² worsens for 3 of 8 backbones) when baseline error is already small.
  • Single-domain (continuous process forecasting); no test on classification, event detection, or control.

Questions for further exploration

open issues raised by the work
1
How does MCC compare to classical data reconciliation with gross-error detection, or learned GNN-based reconstruction (GDN, BeatGAN) on the same protocol — is the gain from semantics or just from any external-reference correction?
2
Can the frozen matrix U be validated or updated automatically when the measurement chain changes (sensor replacement, new derived variable), without re-running the full offline LLM pass?
3
Is κ=0.95 optimal under adversarial or drifting fault distributions, or does the calibration rule under-correct slowly developing drift (the lone failure was a local temperature bias)?
4
Does the Real-Test gain survive on truly external datasets (third-party plants), given two datasets come from the authors' own well-documented deployments?
5
Could leave-one-out references extend to multivariate (vector) targets or classification/event tasks, where per-channel “supported local inconsistency” is harder to define?
6
How sensitive is the result to β_ij being constrained unbiased — would a regularized biased readout trade a little independence for better reference accuracy?
