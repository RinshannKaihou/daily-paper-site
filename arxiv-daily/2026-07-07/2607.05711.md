> **一句话总结：** FourTune 提出首个面向大模型后训练的端到端 W4A4G4 全 4 位框架,通过 SVD 分解出"冻结数值稳定器"隔离离群值、块级量化实现转置友好的反向传播、并融合 LoRA/MLP 算子,在 FLUX.1-dev (12B) 上相比 BF16 LoRA 内存降低 2.25×、训练加速 2.27×,且在定制化、强化学习、蒸馏三类任务上质量与全精度持平。

> **TL;DR:** FourTune is the first fully W4A4G4 (4-bit weights, activations, and gradients) post-training framework for large diffusion models; by augmenting LoRA with a frozen SVD-derived numerical stabilizer, using 16×16 block-wise quantization for transposed 4-bit backprop, and fusing LoRA/MLP kernels, it matches BF16 LoRA quality on customization/RL/distillation while cutting FLUX.1-dev memory 2.25× and speeding training 2.27× (2.79× over NF4 QLoRA), with distillation FID on HPSv2 tied at 15.50 vs BF16 LoRA's 15.50.

## 核心贡献

- **首个全 4 位后训练框架**:首次将权重、激活、梯度同时量化到 4 位(W4A4G4)并直接在 GPU 4 位算力单元上执行骨干矩阵乘,突破了 QLoRA 仅量化权重(W4A16G16)仍需在线反量化的内存-速度瓶颈。
- **三分支混合精度流水线**:在标准 LoRA 之上引入基于 SVD 分解的"冻结数值稳定器"(low-rank stabilizer)分支,显式隔离对量化敏感的高幅离群值,使原生 4 位训练不发生梯度爆炸。
- **块级量化实现高效反向传播**:采用 16×16 block-wise 量化,使缩放因子与局部数据块绑定,从而在反向传播中无需反量化/重量化即可直接计算 Q(G)Q(R)ᵀ。
- **定制融合算子**:将三分支共享输入的下投影融合为单算子、再将 4 位骨干计算与上投影融合,并将 MLP 的 FC1 与 FC2 量化算子融合,分别带来 1.82× 与 1.10× 的加速(合计 2.52×)。
- **跨任务验证**:在定制化(FLUX.1-dev 12B、Qwen-Image 20B)、强化学习(SRPO)、蒸馏(π-Flow)三类后训练任务上全面匹配 BF16 LoRA 质量,验证了方法对细粒度梯度更新的数值保真度。

## Key Contributions

- **First fully 4-bit (W4A4G4) post-training pipeline for large generative models**: weights, activations, and gradients are all quantized to 4-bit (NVFP4) and backbone GEMMs run natively on Blackwell 4-bit Tensor Cores, eliminating the dequantization overhead that cripples QLoRA (W4A16G16).
- **Triple-branch hybrid-precision architecture**: standard LoRA is augmented with a frozen, low-rank "Numerical Stabilizer" obtained via SVD (W ≈ R + L_stab) that absorbs quantization-sensitive outliers, leaving a quantization-friendly 4-bit residual R for efficient GEMM4bit — this prevents the gradient explosion seen in naive W4A4G4.
- **Block-wise quantization for transposed 4-bit backprop**: 16×16 blocks localize scale factors so that Q(G)Q(R)ᵀ can be computed without dequantizing/re-quantizing weights, sidestepping the reduction-axis misalignment of per-output-channel schemes.
- **Custom fused kernels**: fusing the three branches' shared-input down-projection (1.82×) plus the MLP FC1+FC2 quantization (additional 1.10×) yields a 2.52× per-step speedup over standard 16-bit LoRA on a single RTX 5090.
- **Comprehensive cross-task evaluation**: matched full-precision quality on customization (FLUX.1-dev 12B, Qwen-Image 20B), reinforcement learning (SRPO), and distillation (π-Flow, 4-NFE student), demonstrating numerical fidelity even for accuracy-sensitive RL gradients and timestep distillation.

## Methodology

FourTune establishes a native 4-bit post-training pipeline built on three technical components.

**1. Triple-Branch Hybrid-Precision Pipeline.** Building on the spectral decomposition of SVDQuant, each pretrained weight matrix W ∈ ℝ^(m×n) is decomposed into a quantization-friendly residual R and a low-rank outlier component L_stab = L1 L2, so that W ≈ R + L_stab. FourTune then runs three parallel branches at inference and training time: (i) a frozen 4-bit backbone that quantizes R offline to NVFP4 and computes Y₁ = GEMM4bit(Q(X), Q(R)); (ii) a frozen full-precision stabilizer Y₂ = X·L_stab that absorbs the high-magnitude outliers; and (iii) a trainable LoRA adapter Y₃ = X·(AB). Error signals propagate through all branches for correct gradient flow to preceding layers, but weight gradients are computed only for the adapter (A, B). This decouples numerical stabilization (handled by the frozen stabilizer) from task adaptation (handled by LoRA).

**2. 4-Bit Backward Pass via Block-Wise Quantization.** Gradient propagation to the input requires ∂L/∂X = G·Wᵀ, which is decomposed across the three branches. The challenging term Q(R)ᵀ would normally force dequantization, because per-output-channel scales become reduction-axis scales after transposition. FourTune instead partitions weights into 16×16 independent blocks whose scale factors are bound locally, so a kernel can fetch a 4-bit block and its scale together regardless of access pattern — enabling direct computation of Q(G)Q(R)ᵀ at full 4-bit throughput. Weight gradients are computed only for the trainable adapter via standard chain rules (∂L/∂B = (XA)ᵀG, ∂L/∂A = Xᵀ(GBᵀ)).

**3. Kernel Fusion for Memory-Bandwidth Efficiency.** Because all three branches share the same input during quantization/down-projection, FourTune fuses those operations into a single kernel, and further fuses the 4-bit backbone GEMM with the up-projection accumulation. In MLP blocks, the FC1 GEMM is fused with the FC2 quantization kernel so high-dimensional intermediates (e.g., 3072×12288) are never materialized to global memory. This shifts the bottleneck from memory bandwidth back to compute.

**Experimental setup.** All experiments run on NVIDIA Blackwell GPUs (8× RTX Pro 6000 and 8× RTX 5090). Customization is evaluated on FLUX.1-dev (12B) and Qwen-Image (20B) with LoRA rank 64, using AntelopeV2 (identity), CLIP (style), DINOv3 (subject), PyIQA (quality), and BLIP (prompt following). RL uses SRPO on HPDv2 with the HPSv2.1 reward, evaluated on Aesthetic v2.5, PickScore, ImageReward, and SGP-HPS. Distillation uses π-Flow to distill FLUX.1-dev into a 4-NFE student (data-free), evaluated on COCO-10k and HPSv2 prompts via Teacher-FID, CLIP, and HPSv2.1.

## Results & Highlights

- **Memory on FLUX.1-dev customization (1024×1024)**: FourTune cuts end-to-end GPU memory to 14.51 GB vs 29.35 GB for BF16 LoRA — a **2.02× reduction**; weight memory alone drops from 22.59 GB to 7.71 GB, a **2.93× reduction**. Memory footprint becomes comparable to NF4 QLoRA while far exceeding it in speed.
- **Training speed on RTX 5090**: single-step training time of 0.86 s, a **2.27× speedup over BF16 LoRA** and **2.79× over NF4 QLoRA**; on RTX Pro 6000, 0.66 s per step = **2.18× over BF16 LoRA** and **2.85× over NF4 QLoRA**. Single DiT-step latency falls from 1541.0 ms (BF16 LoRA) to 612.4 ms (**2.52× speedup**).
- **Customization quality (Table 1)**: on FLUX.1-dev Identity, FourTune (W4A4G4) achieves Image Quality 0.783 / Similarity 34.77 vs BF16 LoRA's 0.771 / 33.49 — actually slightly higher; on Style, 0.812 vs 0.806. Results scale to the 20B Qwen-Image with no degradation.
- **Distillation (Table 2, π-Flow, 4-NFE student)**: on HPSv2 prompts FourTune ties BF16 LoRA at **FID 15.50** (CLIP 0.283 vs 0.288; HPSv2.1 0.317 vs 0.316), while the SparseLoRA baseline collapses to FID 202.51. On COCO-10k, HPSv2.1 is 0.310 vs 0.311.
- **Reinforcement Learning (Table 3, SRPO)**: FourTune reaches Aesthetic 6.3119 and SGP-HPS 0.0027, matching or exceeding full-precision full fine-tuning (Aes 6.3447) and beating BF16 LoRA (Aes 6.1832, SGP-HPS 0.0020) and NF4 QLoRA (Aes 6.2189) — indicating the 4-bit pipeline preserves the fine-grained gradient updates RL requires.
- **Ablation — stabilizer is essential (Figure 8)**: naive W4A4G4 training suffers gradient explosion; adding the stabilizer keeps gradient norms stable across 600 steps.
- **Ablation — kernel fusion (Figure 9)**: LoRA fusion gives 1.82×, MLP fusion an additional 1.10×, for a combined 2.52× speedup over standard LoRA on RTX 5090.
- **Ablation — quantization granularity (Table 5)**: block-wise 16×16 reconstruction (LPIPS 0.227 / PSNR 20.4) on MJHQ-30K is close to finer group-wise 1×16 (LPIPS 0.203 / PSNR 21.5), confirming the coarser granularity chosen for transposition efficiency does not materially hurt fidelity.

## Authors & Credibility

The work is accepted at **ICML 2026** (PMLR 306). The 11-author team spans Nunchux AI, MIT, CMU, Stanford, and UC Berkeley, and includes several established figures in efficient deep learning: **Song Han** (pioneer of model compression/quantization, founder of DeepGEMM-related work), **Jun-Yan Zhu** (CycleGAN, image-to-image), **Maneesh Agrawala** (visualization, Stanford), **Lvmin Zhang** (ControlNet, ComfyUI), and **Muyang Li / Yujun Lin** (SVDQuant, MIT-HAN-LAB). The lineage to SVDQuant (Li et al., 2025), an established inference-time 4-bit diffusion quantizer, gives the SVD-based stabilizer design solid technical grounding. Hardware access (8× Blackwell GPUs of two flavors) and the scale of models tested (up to 20B) are consistent with a well-resourced, serious evaluation.

## Critical Assessment

- **Novelty (genuine)**: This is credibly the first end-to-end W4A4G4 *post-training* framework for large generative models; prior 4-bit training work (NVFP4 pretraining, QeRL) targets LLM pretraining or RL exploration rather than a unified customization+RL+distillation pipeline. The triple-branch stabilizer idea — repurposing SVDQuant's inference decomposition as a *frozen training-time* numerical buffer — is a clean and non-obvious reuse.
- **Rigor concerns**:
  - The **distillation baseline SparseLoRA is effectively broken** (FID 202.51 vs 15.50 — over 13× worse), which makes the "we beat the sparsity baseline" framing near-vacuous; the real comparison is only against BF16 LoRA and NF4 QLoRA, both of which FourTune merely *matches*, not beats, on quality.
  - The **RL ImageReward (IR) metric actually regresses** under FourTune (1.0152) relative to the base model (1.1189) and BF16 LoRA (1.0735), yet the text only highlights the metrics where FourTune wins (Aes, SGP-HPS). This selective reporting softens the picture.
  - The **block-wise vs group-wise ablation (Table 5)** is framed as "comparable," but LPIPS worsens ~12% (0.203 → 0.227) and PSNR drops 1.1 dB — not negligible for a quantization paper; the efficiency gain is the only justification, and the cost is understated.
  - The stabilizer ablation (Figure 8) shows gradient explosion is *prevented*, but there is **no analysis of stabilizer rank sensitivity** (how low-rank must L_stab be? what is the memory/quality trade-off curve?) — a surprising omission for the paper's central mechanism.
- **Reproducibility**: Heavy dependence on **Blackwell-only NVFP4 4-bit Tensor Cores** (RTX 5090, RTX Pro 6000) means the results cannot be reproduced on the far more common Hopper/Ampere GPUs; the speedups are hardware-specific by construction, and the paper does not report fallback behavior or projected numbers on older hardware. Code/kernel release is implied by the impact statement but not confirmed in-text.
- **Significance**: If the kernels are released, FourTune is practically valuable — it genuinely breaks the QLoRA memory-vs-speed trade-off and makes 12B–20B diffusion post-training feasible on consumer GPUs. The conceptual contribution (frozen stabilizer for training) should transfer to LLM fine-tuning, which the authors do not explore.
- **Red flags**: None fatal, but the combination of (a) a broken SparseLoRA baseline inflating the distillation table, (b) selective RL metric reporting, and (c) no older-GPU port means readers should treat the headline speedups as Blackwell-conditional and the quality claims as "matches BF16 LoRA" rather than "improves on it."

## Strengths

- Truly native 4-bit training (W4A4G4) with no online dequantization — a meaningful advance over QLoRA's W4A16G16.
- Clean, well-motivated three-component design where each piece (stabilizer, block-wise quant, kernel fusion) addresses a concrete failure mode, with ablations isolating each contribution.
- Broad, realistic evaluation across three distinct post-training regimes (customization, RL, distillation) and two model scales (12B FLUX, 20B Qwen-Image), not a single cherry-picked benchmark.
- Strong efficiency numbers that are end-to-end (not just micro-benchmarked), with both memory and latency reported on two GPU tiers.
- The frozen-stabilizer concept is architecturally elegant and plausibly transferable to LLM PEFT.

## Limitations

- **Hardware lock-in**: results require Blackwell NVFP4 Tensor Cores; practitioners on Hopper/Ampere/Ada cannot benefit from the speedups, and the paper does not discuss portability or degradation outside Blackwell.
- **Stabilizer underanalyzed**: no study of stabilizer rank vs. quality/memory trade-off, no guidance on choosing the rank for new architectures, and no investigation of when the decomposition might fail.
- **Quality only matches, never exceeds, BF16 LoRA** — the value proposition is purely efficiency, so users who already fit BF16 LoRA in memory gain little.
- **Selective RL reporting**: ImageReward regression is not discussed; a more balanced presentation would acknowledge where 4-bit training hurts.
- **Distillation comparison is thin**: SparseLoRA is non-competitive (FID 202.51), leaving only two meaningful baselines; newer 4-bit training methods (e.g., QeRL) are mentioned in related work but not benchmarked head-to-head.
- **Block-wise granularity cost** (LPIPS +12%) is real but downplayed; the efficiency/fidelity Pareto frontier is not mapped out.
- No evaluation of higher-bit-reward-sensitive tasks like medical/satellite imagery where 4-bit gradient error could matter more.

## Relevance

与高效微调、低比特量化和扩散模型后训练的研究方向高度相关:FourTune 的"冻结数值稳定器 + 块级量化 + 算子融合"思路不仅适用于扩散模型,其原生 4 位反向传播设计对大模型(含 LLM)的 PEFT 加速、消费级 GPU 上的个性化训练和 RL 对齐都具有直接参考价值,是量化训练落地的重要进展。

## Questions for Further Exploration

- Does the frozen SVD stabilizer transfer to LLM post-training (e.g., Llama/Qwen fine-tuning), where outlier structure and gradient statistics differ from diffusion DiT blocks?
- What is the stabilizer rank vs. quality/memory Pareto curve, and is there a principled (data-driven) way to choose the rank per layer rather than a global setting?
- How do the speedups degrade on non-Blackwell hardware (Hopper, Ada) — can the block-wise 4-bit backward still help via INT4 or only via NVFP4?
- Could the IR (ImageReward) regression under RL be fixed by selectively keeping specific layers at higher precision, and does this reveal which layers are most quantization-sensitive in RL gradients?
- Since FourTune only *matches* BF16 LoRA quality, is there a regime (very long training, very high rank, or RL with sparse rewards) where 4-bit gradient error finally causes a measurable quality gap?
- How does FourTune interact with even lower precision (W2A2, or mixed 2/4-bit) — does the stabilizer still prevent explosion, or does a second-order correction become necessary?
