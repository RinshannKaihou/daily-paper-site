02 Why this matters

The approval-view vs. context-delivery gap is a protocol-level property, not a codebase bug — proven across three independent libraries. Every coding-agent integration is exposed, and the four proposed fixes are spec-ready.

03 Critical edge

No real production client (Cursor, Claude Desktop, Cline, Continue) was render-tested; the universal-vulnerability claim is argued from Unicode font coverage, not measured against a panel of shipping renderers.

§I Methodology model-free analysis + deterministic harness (§3–4)

The paper deliberately separates a model-free mechanism analysis from a deterministic protocol-level harness. The prediction is formed before any client runs, then confirmed by measurement — decoupling "why it works" from "does it work."

For each payload character the author traces two fates. Plain ASCII 'e' (0x65) maps to U+0065 — one UTF-8 byte, a glyph in every font, visible to any human. The TAG-block encoding maps the same character to 0xE0000 + (0x65 & 0x7F) = U+E0065, whose four-byte form F3 A0 81 A5 a UTF-8 decoder and the model's tokenizer accept verbatim — yet which carries no assigned glyph in any mainstream terminal, chat client, or IDE panel. That asymmetry — renderable-or-not for a human, decodable regardless for a model — is the entire mechanism behind T7, and it is independent of which MCP client, sanitizer, or model sits downstream.

Eight techniques (T1–T8) poison five MCP surfaces: the description field, inputSchema, the tool name, the error channel, and a post-approval TOCTOU re-list mutation. A genuine JSON-RPC/stdio client (initialize → tools/list → tools/call) launches each malicious server as a subprocess and records four deterministic protocol-level observations on the exact bytes received. A 25-description benign corpus sanity-checks the baseline sanitizer (0/25 flagged), ruling out the "rejects everything" degenerate case.

Strengths

  • Model-free, protocol-free mechanism analysis that predicts the outcome before any client runs, then is confirmed by data.
  • Real JSON-RPC/stdio implementation, not a simulation; verbatim payload and sanitizer source make it trivially inspectable.
  • Cross-library replication across three independent libraries (32/32 cell agreement) — the strongest defense against the "single-codebase artifact" critique.
  • Honest scoping: separates delivery / defense-evasion rates from attack-success rates; positions the result explicitly as a lower bound.
  • Concrete, mechanism-targeted structural fixes (§8.1–8.5) mapped one-to-one onto the techniques they defeat.
  • Self-critical Threats-to-Validity section (§9.1) addressing construct, internal, and external validity.

Limitations

  • Both baseline defenses are deliberately simple; no stronger semantic / LLM-based sanitizer is benchmarked, so 4/8 is a lower bound with no upper bracket.
  • The "every renderer is vulnerable" claim is argued from font coverage, not measured against real production clients (no Cursor / Claude Desktop / Cline / Continue render test).
  • "Reaches the model's context" ≠ "the model acted on it"; no end-to-end agent-behavior or attack-success-rate measurement against any deployed model.
  • Benign corpus is hand-written (25 descriptions), not sampled from deployed traffic — real-world sanitizer false-positive prevalence unknown.
  • Two of three cross-library implementations share the official mcp wire types, so agreement is weaker against a shared wire-layer bug.
  • Single author; the "three independent engineering teams" framing modestly overstates independence — all re-implementations were written by the same person.

§VI Questions for further exploration

  1. How do real production clients (Claude Desktop, Cursor, Cline, Continue) actually render the TAG-block payload — does any already normalize unassigned-glyph codepoints, falsifying the universal-vulnerability claim?
  2. What is the true attack-success rate (model acting on the payload) of T7 and T4 against frontier coding-agent models, as opposed to the delivery rate measured here?
  3. Would a semantic / embedding-based sanitizer trained on benign MCP descriptions substantially change the 4/8 figure, and at what false-positive cost on deployed traffic?
  4. Is the TOCTOU re-consent gap (T3) closeable purely client-side by pinning the approved metadata hash, and do any current clients already do so?
  5. Does the concealment mechanism generalize beyond MCP to other agent architectures that show users one rendering while forwarding different bytes to the model (browser-tool, OpenAI function-calling, A2A)?
  6. Can provenance-scoped namespaces be enforced without breaking the legitimate case of a server deliberately overriding a host built-in?
  7. How prevalent are these eight techniques in the wild, given the benign corpus was hand-written rather than sampled from deployed servers?
