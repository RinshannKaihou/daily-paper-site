Why This Matters

A multivariate, cost-aware framework that treats the LLM agent itself as the experimental unit — directly answering how to scientifically benchmark stochastic, adaptive coding agents instead of trusting single-run scores.

Critical Edge

No replicates (each of 140 configs run once), narrow scope — two agents, one task family, one dataset — and codex cost is imputed at list prices, not metered, so the headline 7× gap and the negative slope are descriptive, not controlled.

01

Methodology

Two commercial coding agents — codex (OpenAI Codex CLI v0.125, GPT-5.5) and claude-code (Opus 4.7) — each expose a native three-level reasoning-effort ladder (relabeled low / default / max). Each agent runs in an isolated workspace under a no-leakage rule for held-out data, explores a fixed networked group-anagram dataset (28 sessions, 209 players, 62,700 records), registers candidate models, logs decisions, and submits one runnable script scored by a fixed harness.

A six-factor full factorial (agent × task × primary metric × reasoning effort × fold × regime) yields 144 runs, 140 valid. Each run produces an 8-D joint response y ∈ ℝ⁸: two metric scores, log-cost and wall-clock, and four process-complexity counts. A per-stratum multivariate linear model with orthogonal-polynomial effort contrasts feeds three inferential tools: a directed utility contrast u (reward performance, penalize cost) under Holm correction, Pillai's trace omnibus MANOVA, and the new Utility-Aligned Canonical Decomposition (UACD).

++ Strengths

  • Agent as experimental unit. Reframes evaluation so performance/cost/process are joint responses to the agent's own controls — directly attacks single-run benchmarking.
  • Principled machinery. Classical DOE (orthogonal-polynomial contrasts, blocking) + multivariate inference (Pillai, directed contrast, UACD), transparent about power and identification limits.
  • Honest failure reporting. The 4 dropped runs are dissected as a case study; the codex Task-1 prompt-wording confound is disclosed, not buried.
  • Deployable insights. Effort-as-resource-control, agent metric specialization, and the ~7× cost gap at comparable quality are directly actionable.
  • Reproducible. Public, pinned harness and data on GitHub with per-run agent/model/effort flags.

−− Limitations

  • No replicates. Each of the 140 configs run once; per-stratum uncertainty rests entirely on factorial structure (single runs exceeded 70 min & $33).
  • Narrow scope. Two agents, one task family, one dataset, four metrics — generalization untested.
  • Imputed codex cost. Blended $6.50/10⁶ tokens, not metered; cross-agent cost contrast is descriptive only.
  • Weak canonical ID. UACD leading direction is weakly identified per stratum; η⁽¹⁾ read by sign only, sign-test independence approximate.
  • Single utility direction. Conclusion conditional on one u; sensitivity to alternative weightings deferred.
  • Dropped runs concentrated. All 4 in a-DLD (n=16) — precisely where the only univariate effort signal appears.
03

Open Questions

  1. Q1
    Does the negative slope persist under alternative utility directions (e.g., penalizing process complexity, asymmetric weights) — is there any u under which more effort becomes net positive?
  2. Q2
    How do findings generalize beyond two agents and the anagram domain — to open-source agents, SWE-bench-style tasks, or different signal-to-noise regimes?
  3. Q3
    With replicated runs, do per-stratum Pillai omnibus tests gain enough power to survive α/8, and does UACD's leading direction become point-identifiable?
  4. Q4
    If codex cost were metered rather than imputed, would the 7× gap and the negative utility slope quantitatively change?
  5. Q5
    Where does the extra effort go in run traces — and why don't more tokens / more candidate models yield better held-out performance?
  6. Q6
    Can the Task-2 self-assessment gap be closed by feeding the agent its own simulation-validity check or an external calibration signal?
