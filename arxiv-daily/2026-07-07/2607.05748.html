Two Sides of the Same Coin: Learning the Backdoor to Remove the Backdoor

Qi Zhao & Christian Wressnegger — KASTEL Security Research Labs, Karlsruhe Institute of Technology
Key finding

not the benign task —
the poison separates itself.

Why this matters

Reliable training-time cleansing of poisoned datasets secures the ML supply chain with near-zero accuracy cost and no clean reference set.

Critical edge

Image-classification only, single-target / single-trigger; single-seed results with a narrow λ / γ operating window.

Methodology

HARVEY is a training-time anti-backdoor defense that assumes no clean reference set — the defender trains blindly on poisoned data. The core inversion: a network learns a backdoor far more easily than the benign task, so a strongly backdoored reference model is a sharper "poison oracle" than any clean reference model.

The theoretical nugget — decomposing Symmetric Cross-Entropy reveals that only the Reverse Cross-Entropy term, L_RCE = C·(1 − p(y|x)), separates poison (collapses near 0) from benign (sits near C). This yields a parameter-free split threshold of C/2 ≈ 4.605, removing a major source of baseline fragility.

Four-stage adaptive pipeline

1
Initialize
Train on full poisoned set; take 50% lowest-RCE as initial poison.
2
Learn the backdoor
Gradient-ascent on poison + unlearn benign, n=20 rounds → strong oracle.
3
Meta-split
Round-1 ref rescues target-class benign caught in the poison pool.
4
Final training
Standard supervised CE on the cleansed set → backdoor-free model.

Hyperparameters γ=0.01, λ=0.01, n=20 are held fixed across all six attacks — no per-attack tuning.

"The backdoor is the easier task — so learn it as the oracle, not the benign feature you keep failing to learn."

Strengths vs. Limitations

Strengths

  • Conceptually clean inversion — "learn the backdoor as the oracle," backed by both KL theory and intuition.
  • Parameter-free split at fixed C/2 removes baseline fragility.
  • Never fails across every attack × architecture × dataset combination, at no-defense accuracy.
  • ~20× faster than DBD; competitive with naive training.
  • Honest ablations — adaptive attack, clean-data degradation, cross-poison-rate.

Limitations

  • Image only; single-target / single-trigger. No NLP, no all-to-all or multi-trigger.
  • No error bars — single-seed; variance hinted (Fig. 9) but unquantified for main tables.
  • Narrow operating window: sensitive to λ ≥ 0.015 and γ > 0.015 on dynamic triggers (IAB, WaNet).
  • Accuracy drops at 20% poison (e.g. Blend 89.64%).
  • Weak adaptive adversary tested; trigger loss-matching left to future work.
  • n=20 and 50%/1% heuristics never ablated.

Open questions

  1. Does the oracle still converge cleanly for all-to-all or multi-trigger backdoors, where the shortcut is no longer low-dimensional?
  2. Behavior under a stronger adaptive attack that explicitly minimizes the RCE gap between poison and benign during poisoning?
  3. Can the RCE-only, C/2 insight port to model-based / runtime defenses (trigger inversion, STRIP) that currently rely on CE/SCE?
  4. What is the minimum poisoning rate at which Stage 1's lowest-RCE-half enrichment assumption breaks?
  5. Does the paradigm transfer to NLP / tabular where "easy-to-learn backdoor" may not hold?
