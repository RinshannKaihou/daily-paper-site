01

Methodology

CURATE EVO keeps the post-training procedure fixed (base model + SFT/GRPO + environment + inference budget) and instead optimizes how training resources are built from a raw corpus. A curation strategy ρ : D_raw → (D_sft, D_rl, M_mem) is treated as executable code, rewritten every epoch from dev-set failures.

Cost-aware objective   J(ρ) = P(ρ) − λ·C(ρ)
P = dev performance · C = normalized training-turn cost · λ=0.3 optimal

Evolution loop, per epoch:

  1. Execute current code ρe on Draw → produce Dsft, Drl, Mmem.
  2. Train πe from the same base with the fixed SFT+GRPO recipe.
  3. Evaluate on held-out dev set Qdev; collect failures Te−, score se, profile Σe.
  4. Code-evolution agent (GPT-5.4 + mini-SWE-agent) rewrites ρe → ρe+1.

Two ordered revisions approximate the discrete optimization: effectiveness (diagnose failure modes → augment/filter/refine), then efficiency (dedupe, prune low-utility turns, compress). Non-improving revisions roll back to prevent error cascades. The test set is never touched during evolution.

03

Strengths × Limitations

Strengths

  • Clean formulation: curation as evolving code under a cost-aware objective, with rollback to prevent cascading errors.
  • Jointly produces SFT data, RL data, and a memory bank; ablations confirm all three are complementary.
  • Strong gains across 3 diverse benchmarks (tool selection, function calling, long-horizon dual-control).
  • ~48% token / ~50% wall-clock reduction while improving accuracy.
  • Recipe-agnostic: +21.3 avg when stacked onto GRPO / AgentGym-RL / ProRL-Agent.

Limitations

  • Depends on a proprietary coding agent (GPT-5.4 + mini-SWE-agent); open-source evolution models untested, and that compute is excluded from overhead.
  • Baseline comparison mixes sizes (8B vs. 4B); no matched-capacity run of the strongest baselines.
  • Main table stops at 3 epochs though Figure 4 shows gains still climbing — choice unjustified.
  • Evolved code described only qualitatively (Appendix C); no code/data release.
  • Only 4B base model tested; scaling to 7B/14B unverified.

05

Open questions

  • How much of the gain survives when the evolution agent is open-source (e.g. Qwen-Coder) instead of GPT-5.4?
  • Head-to-head at matched Qwen3-4B for MUA-RL / EnvScaler — do the +3.2 / +2.7 deltas hold?
  • Does the memory bank generalize to unseen task families, or memorize benchmark-specific schema anchors?
  • Does the curation benefit grow or shrink at 7B / 8B / 14B?
  • Run evolution past 3 epochs — what is the asymptote, and does the dev set start to overfit?
