01

Methodology — a coarse loop, a sound anchor

Aria · three-layer architecture

Aria is deliberately coarse: hand a whole lemma to a general-purpose LLM code agent and let it write the proof however it likes. The Coq kernel is the only trust anchor closing the loop — no premise selector, no tactic-level search, no decomposition or engineered repair procedure is imposed.

Every candidate must clear the harness: a 300 s per-tactic timeout, a ban on admit / Admitted / Axiom, a lemma-coverage (completeness) check so the agent cannot silently drop or weaken the target, an Iris linter, and shell-command safety. Structured failure feedback drives retries.

The harness itself is a first-class artifact, written in a new declarative language, HHL, which compiles (not interprets) to the Claude Code SDK: pre/post-tool hooks, workspaces, retry workflows — retargetable to other runtimes. A driver coordinates Extractor → Prover → Fixer (up to 30 retries, resuming the same session) → Polish (fresh session, rubric-checked).

Evaluation uses a proof-regeneration protocol: each proof body is stripped, the statement kept, and reconstruction runs in a sandbox with no network and no git — the agent cannot retrieve the removed reference proof.

Model layer
Stateless LLM inference. Opus 4.7, Kimi K2.6, DeepSeek V4 Pro as swappable backends writing candidate proofs into source.
Agent layer
Owns strategy: lemma choice, retrieval of similar proofs/defs, response interpretation, revision on error. Any runtime with tool dispatch + hooks works.
Harness layer · trust anchor
Coq kernel as verification oracle. Enforces timeout, anti-admit, completeness, Iris linter, shell safety. Described in HHL; compiles to Claude Code SDK.

Strengths & Limitations

side-by-side

Strengths PRO

  • Population-level, zero-failure coverage on Iris — the hardest separation-logic target in existence — not a sampled subset.
  • Clear, falsifiable claim: strategy-free agent + verifier suffices, tested across four benchmarks and two proof assistants (Coq, Lean).
  • HHL is genuinely reusable — cleanly separates the deterministic trust layer from the nondeterministic model layer.
  • Completeness check matters: a file can type-check while the target lemma was silently dropped.
  • Honest about the open-model gap on long proofs rather than glossing it over.
  • Proof-regeneration protocol (no network, no git) is a credible anti-memorization attempt.

Limitations CON

  • No controlled ablation on the same model. COPRA / Rango / Cobblestone are not re-run on Opus 4.7, so model capability vs. method is confounded.
  • Memorization not quantitatively ruled out — no AST/token edit distance vs. upstream Iris proofs; defense rests on assertion + the small 72-lemma iris-lean result.
  • ~380 model-hours on a proprietary subscription; Opus 4.7 will drift, so exact replication is fragile.
  • The harness is itself heavily engineered (linter, 300 s timeout, 6-lemma session window, rubric). Strategy hasn’t vanished — it migrated into the harness.
  • All targets are guaranteed provable (existing libraries). Behavior on genuinely open conjectures is untested.
  • Single-system evaluation — no external user study, no deployment by the Iris team.

Questions for Further Exploration

open threads
How much of Aria’s gain survives when COPRA, Rango, and Cobblestone are re-run on Opus 4.7 with their original strategies — isolating model capability from method?
What quantitative metric (token- or AST-level edit distance) separates Aria’s proofs from upstream Iris — and does first-attempt success correlate with proof length in the training corpus?
Does the 300 s per-tactic timeout ever reject a tactic a longer wait would have closed? How sensitive is the 100% figure to that threshold?
Can the recipe prove net-new theorems in Iris/RustBelt that humans have not yet proved — or only regenerate known proofs?
How does it scale to non-separation-logic Coq developments (univalent foundations / HoTT), where proof length and search depth are larger?
Is HHL expressive enough for proof strategies beyond verification — specification discovery, counterexample search — which the authors flag as future work?
