> **一句话总结：** 作者提出 Aria，把一个未经 Coq 特化的通用 LLM 代码代理（Claude Code / Claude Opus 4.7）与一个基于 Coq 内核的验证外壳（harness）配对，不施加任何人工设计的证明策略，竟在 Iris 四个核心模块全部 4,257 条引理、RustBelt 217 条引理、reglang 全部 318 条引理以及 iris-lean 72 条尚未移植的引理上实现 100% 自动证明、零失败、零专家介入。

> **TL;DR:** Aria pairs an off-the-shelf LLM code agent (Claude Code on Claude Opus 4.7) with a Coq-kernel verification harness, imposing no hand-engineered proof strategy, and achieves full 100% coverage on the Iris separation-logic core (all 4,257 lemmas), RustBelt Rust-library proofs (217 lemmas), the reglang library (all 318 lemmas, where prior prover Rango proves barely one in eight on CoqStoq), and iris-lean (72 not-yet-ported Lean lemmas) — zero failures and no Coq expert intervention across all four benchmarks.

## 核心贡献

- **Harness + 代码代理这一极简范式**：把整条引理交给通用 LLM 代码代理自由发挥，仅由 Coq 内核作为唯一信任锚，配合超时、反幻觉、Iris linter 和完备性检查构成的"外壳"做反馈循环；无需前提选择器、tactic 级搜索、分解或修复策略等人造脚手架。
- **首次完全自动证明专家级、工业级分离逻辑库**：在 Iris（现代高阶并发分离逻辑之最）的四个核心模块（algebra、bi、base_logic、program_logic）共 4,257 条引理上实现 100% 通过，以及 RustBelt 下 Rust 标准库（Arc、Mutex、RwLock、RefCell 等）的 217 条安全证明全部通过。
- **在与先前 LLM 证明器的直接对比中碾压**：在 reglang 这一 CoqStoq 上先前系统表现最差（Rango 仅约 1/8 通过率）的库上，Aria 全部 318 条通过，零失败。
- **方法不绑定 Coq**：在 iris-lean（Iris 的 Lean 4 移植版）上对 72 条尚未存在过证明的引理同样全部证明，显示同一 harness 设计可迁移到其他证明器内核。
- **HHL 外壳描述语言**：引入一种声明式语言，把环绕代理的策略（pre/post 钩子、workspace、retry 循环）做成可写、可审计、可在不同代理运行时（Claude Code SDK、OpenAI Agents SDK）间复用的产物，而非一次性胶水代码。

## Key Contributions

- **Harness + code agent paradigm**: An off-the-shelf LLM code agent (Claude Code on Claude Opus 4.7), not specialized for Coq, is handed each whole lemma and writes a complete proof however it sees fit; the only scaffolding is a verification harness anchored in the Coq kernel. No premise selector, tactic-level search, decomposition, or repair procedure is imposed.
- **First fully automatic proof of an expert-grade concurrent separation-logic library**: Aria proves all 4,257 lemmas of Iris's four core modules (algebra, bi, base_logic, program_logic) and all 217 RustBelt lemmas verifying Rust's Arc/Mutex/RwLock/RefCell/etc., both fully and fully automatically, with zero failures and zero expert intervention — the first such result on state-of-the-art separation logic.
- **Decisive head-to-head on prior work's hardest benchmark**: On reglang (318 lemmas), the CoqStoq project where the strongest prior prover Rango proves barely one in eight, Aria proves all 318 with zero failures and only 0.38 mean retries per lemma.
- **Generality across provers**: On iris-lean (the unfinished Lean 4 port of Iris), Aria proves all 72 not-yet-ported lemmas whose proofs have never existed and therefore cannot have been memorized, showing the approach is not Coq-specific.
- **HHL — a declarative harness language**: A new language that makes the policy surrounding the agent (pre/post-tool hooks, workspaces, retry workflows) a reusable, auditable artifact compilable to different agent runtimes (currently the Claude Code SDK).

## Methodology

Aria is built as a deliberately coarse loop: hand a whole lemma to a general-purpose LLM code agent and let the agent decide how to construct the proof, with a sound verifier (the Coq kernel) closing the loop. The system is organized as a three-layer architecture:

1. **Model layer** — stateless LLM inference. A backend-agnostic interface treats models (Claude Opus 4.7, Kimi K2.6, DeepSeek V4 Pro, etc.) as interchangeable backends. Given a structured prompt encoding the current proof obligation, surrounding context, and any prior feedback, the model writes a candidate proof into the source file.

2. **Agent layer** — orchestration and intelligence. It owns strategy: which lemma to attack, how to retrieve similar proofs/definitions/dependencies, how to interpret the model's response, and how to revise on harness-reported errors. It is not tied to a specific code agent — any runtime exposing tool dispatch plus pre/post-tool and turn-level hooks (Claude Code, Codex, ...) can serve.

3. **Harness layer** — the trust anchor. It wraps the Coq proof assistant as a verification oracle and enforces auxiliary policy: a per-tactic 300 s timeout for divergent tactics, a ban on `admit`/`Admitted`/`Axiom`, a lemma-coverage (completeness) check that the target lemma was not silently dropped or weakened, an Iris linter, and shell-command safety. Every candidate must pass the harness before it counts; the harness returns structured feedback (failing line, error message, pending goal) that drives retries.

The harness is expressed in a new declarative language, **HHL (Harness Hook Language)**, which compiles (rather than interprets) down to the Claude Code SDK target: pre-hooks become `PreToolUse` handlers, post-hooks become `PostToolUse` handlers, a workspace becomes run configuration, and each `invoke` becomes one non-interactive `claude -p` run. The same HHL program can be retargeted to other agent runtimes without changing hooks or workflows.

The full campaign runs as a small team of specialized agents coordinated by a driver: an **Extractor** finds unproved lemmas, a **Prover** makes the first attempt, a **Fixer** revises on rejection (up to 30 retries, resuming the same session so the full error history carries forward), and a separate **Polish** agent (fresh session) rewrites accepted proofs toward a rubric (conciseness, transparency, robustness, idiomatic style, reuse), with the Coq harness re-checking every rewrite. Sessions are reused across at most six consecutive lemmas within a file and reset on every new file to balance context carry-over against context dilution.

The evaluation uses a **proof-regeneration protocol**: for each target lemma the original proof body is stripped, leaving the statement intact, and the agent must reconstruct a proof from scratch in a sandbox with no network and no git (so it cannot retrieve the removed reference proof). The model is Claude Opus 4.7 throughout unless otherwise stated.

## Results & Highlights

- **Iris core — 100% coverage.** Aria proved all **4,257 distinct lemmas** across **113 source files** in Iris's four core modules with **0 failures**. **79.2% (3,373 of 4,257)** were solved on the first attempt (0 retries); mean retries per lemma was **0.51**. The full run consumed **379.7 hours (~15.8 days)** of model compute, averaging **321.1 s per lemma**, ranging from **8.7 s** to **17,103 s (4.7 h)** for the single most demanding obligation. No lemma exhausted the 30-retry budget (max observed was 28 retries).
- **RustBelt — 100% coverage.** Aria proved **217 lemmas** across 25 files verifying Rust's Arc, Rc, Mutex, RwLock, RefCell, Cell, Vec, Option with **zero failures**; **73%** first-attempt, **86%** within one retry; hardest cases (Arc drop/clone, RwLock reader/writer guards) took up to 13 retries. Mean time per lemma was ~706 s (roughly double Iris, reflecting proof length not failure rate).
- **reglang — 100% coverage vs. prior prover's ~12.5%.** Aria proved all **318 lemmas** with zero failures; **75.5%** first-attempt; **0.38 mean retries** per lemma; no proof needed more than 6 retries. The single outlier `nfa_ofP` took ~5.8 h and 4 retries. By contrast, Rango (ICSE 2025) proves barely one in eight (~12.5%) on this same CoqStoq project.
- **iris-lean — 100% on never-before-proved Lean lemmas.** Aria proved all **72 lemmas** in three not-yet-ported files (Function, Mra, UFrac) with zero failures, **90.3%** first-attempt, **0.10 mean retries**, under **two hours** total, with soundness now decided by the Lean kernel rather than Coq's. On the corresponding 49 Coq Iris lemmas, Aria achieves 75.5% first-attempt at 149 s/lemma, vs. 90.3% and 97 s/lemma on the finer-grained Lean split — a methodological lesson that smaller lemmas help the agent.
- **Private vs. open-source models (40-lemma subset).** Both Claude Opus 4.7 and Kimi K2.6 proved all 40 lemmas of `frac_auth.v`. Opus 4.7: **72.5%** first-attempt, max 3 retries, <2 h total. Kimi K2.6: **55.0%** first-attempt, up to 24 retries, ~7× longer. DeepSeek V4 Pro matched Kimi's pass rate at ~2–3× the token cost. The authors report a genuine capability gap on long proofs: open models frequently failed on proofs beyond ~50 lines even after exhausting the 30-retry budget, whereas Opus 4.7 reliably constructed correct proofs of 200+ lines.

## Authors & Credibility

- **Shuangxiang Kan** (Singapore Management University), **Shuanglong Kan** and **Sebastian Ertel** (Barkhausen Institut, Dresden). The team targets the Iris development directly and engages the real RustBelt and iris-lean upstream artifacts, not synthetic benchmarks.
- The work situates itself carefully in the prior-art table (Table I) with cited, apples-to-apples numbers: ASTactic 12.2% on CoqGym, Proverbot9001 19.4% on CompCert, PALM 40.4% on CoqGym, COPRA 48.3% on a 118-theorem CompCert subset, Rango 32.0% on CoqStoq, Cobblestone 48% (58% with oracle) on CoqGym, ProofCoop 33.0% on CoqGym.
- Code is provided via an anonymized repository (https://anonymous.4open.science/r/CoqProver-Code-57BB). Experiments were run on a Claude Code 20× subscription rather than the metered API, which the authors note reduced cost but complicates exact cost reproducibility.
- The evaluation is population-based (every lemma whose proof terminates in `Qed`, not a sample), which strengthens the external validity of the headline 100% claims.

## Critical Assessment

- **Novelty — genuine and well-argued.** The central thesis — that the entire hand-engineered proof-strategy scaffold of prior systems (premise selectors, tactic-level search, divide-and-conquer decompositions, engineered repair loops) is unnecessary once a capable code agent is paired with a sound verifier — is non-obvious and runs against the field's recent direction (Rango, COPRA, Cobblestone, ProofCoop). Making the harness a first-class declarative artifact (HHL) rather than ad-hoc glue is a real and reusable contribution, and the paper is candid about a subtle but important completeness check: when an agent can freely edit files it can satisfy the kernel while silently dropping or weakening the target lemma, so soundness alone is insufficient — completeness must also be enforced.
- **Rigor — strong on coverage, weak on controls.** The population-level 100% result on Iris is hard to dismiss. However, several methodological gaps weaken the causal claims: (a) There is **no ablation isolating the harness from the model** — we cannot tell how much of the gain comes from Opus 4.7 being simply a stronger model than those available to prior systems (PALM/COPRA/Rango were built on older, weaker models), versus the harness+agent design itself. (b) The 79.2% first-attempt success rate on lemmas whose proofs exist in the model's training data raises a **memorization concern the authors address only rhetorically** — they assert the generated proofs differ from upstream "apart from trivial one-line lemmas" but provide no automated diff or quantitative measurement of proof divergence. The iris-lean result (proofs that never existed) is the strongest defense, but it covers only 72 lemmas and is far easier than the Iris core. (c) The **300 s per-tactic timeout is a silent failure mode**: any proof whose correct tactic legitimately takes longer would be unprovable, and the paper does not discuss how often the timeout, rather than genuine failure, drives retries.
- **Reproducibility — partial.** An anonymized repo is provided, but the headline results depend on a Claude Code 20× subscription and Claude Opus 4.7, a proprietary, non-version-pinned-by-weight model whose behavior will drift over time; exact numerical replication is therefore not guaranteed. The 380-hour model-time campaign is also expensive to rerun. HHL is compilable to alternative runtimes (OpenAI Agents SDK) in principle, but only the Claude Code SDK target is demonstrated.
- **Significance — high if the result holds.** If verified, this is the first fully automatic proof of an entire expert-grade concurrent separation-logic library, and it would meaningfully change how the field thinks about proof automation. The methodology lesson (finer-grained lemmas raise first-attempt rate, shown via the Coq-vs-Lean comparison) is independently useful.
- **Red flags.** (1) The paper repeatedly hedges with "surprisingly" while reporting perfect scores on every benchmark — a pattern that warrants independent replication before accepting at face value, especially since the same group is both designer and evaluator. (2) The headline "no Coq expert intervention" is true for the proof-writing phase but the harness itself (HHL hooks, the Iris linter, the 300 s timeout, the 6-lemma session window, the rubric criteria) is heavily engineered by experts, so the system is not strategy-free — the strategy has merely migrated from inside the proof loop into the harness. (3) Comparisons to prior work conflate model capability with method: a fair test would run COPRA or Cobblestone on Opus 4.7, which the paper does not do.

## Strengths

- Population-level, zero-failure coverage on Iris — the hardest separation-logic target in existence — rather than a sampled or cherry-picked subset.
- Clear, falsifiable central claim (strategy-free agent + verifier suffices) backed by four independent benchmarks across two proof assistants (Coq and Lean).
- HHL is a genuinely reusable artifact that cleanly separates the deterministic trust layer from the nondeterministic model layer.
- The completeness (lemma-coverage) check is an important and easy-to-overlook contribution: a file can type-check while the target lemma has been silently dropped.
- Honest reporting of where open-source models fall short (long proofs beyond ~50 lines) rather than glossing over the capability gap.
- The proof-regeneration protocol (strip proofs, sandbox with no network/git) is a credible attempt to defeat memorization.

## Limitations

- **No controlled ablation against prior methods on the same model.** The strongest alternative explanation — that Opus 4.7 alone accounts for most of the gain — is not ruled out by running COPRA/Rango/Cobblestone on Opus 4.7.
- **Memorization not quantitatively ruled out.** No automated diff or similarity metric between generated and upstream Iris proofs is reported; the defense rests on assertion and on the small (72-lemma) iris-lean result.
- **Cost and reproducibility.** ~380 model-hours on a proprietary subscription; behavior of Opus 4.7 will drift, so exact numerical replication is fragile.
- **Harness itself is heavily engineered.** The "no strategy" framing understates the expert design embedded in HHL (linter, timeout, session window, rubric, completeness check).
- **Benchmarks are existing libraries with known-good proofs**, so all targets are guaranteed provable; the system's behavior on genuinely hard open conjectures, or on lemmas humans have failed to prove, is untested.
- **Open-model capability gap** is acknowledged but not mitigated; the recipe currently depends on a frontier proprietary model for the hardest proofs.
- **Single-system evaluation** (no external user study, no deployment by the Iris team), so practical integration into real verification workflows is unvalidated.

## Relevance

高度相关于"用 LLM/代码代理自动化形式化验证与定理证明"这一研究方向：它直接针对 Iris 这一最先进的并发分离逻辑库，给出了首个全自动化、零失败的实证，并提出了可复用的 harness 描述语言 HHL，对 LLM 代理驱动的软件验证、Coq/Lean 自动证明以及"代理 + 验证器"范式的研究者具有直接参考价值。

## Questions for Further Exploration

- How much of Aria's gain survives when COPRA, Rango, and Cobblestone are re-run on Claude Opus 4.7 with their original strategies, isolating model capability from method?
- What quantitative metric (e.g., token-level or AST-level edit distance) distinguishes Aria's proofs from upstream Iris proofs, and does first-attempt success correlate with proof length in the training corpus — a memorization fingerprint?
- Does the 300 s per-tactic timeout ever reject a tactic that a longer wait would have closed, and how sensitive is the 100% coverage figure to that threshold?
- Can the harness + agent recipe prove lemmas in Iris or RustBelt that humans have *not* yet proved (true net-new theorems), or only regenerate proofs known to exist?
- How does the approach scale to non-separation-logic, higher-math Coq developments (e.g., the univalent foundations / HoTT libraries) where proof length and search depth are even larger?
- Is HHL expressive enough to capture proof strategies beyond verification (e.g., specification discovery, counterexample search), which the authors flag as future work?
