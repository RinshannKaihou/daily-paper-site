> **一句话总结：** Danus 提出以共享"事实图"(fact graph) 作为全局记忆的数学推理编排系统，由主智能体规划协调、多个 worker 并行搜索证明、无状态 verifier 校验，在代数几何、奇点理论与组合数学的六个研究级问题上构建了含数千条已验证事实的长链证明，并在相同模型/提示下击败 Rethlas（三次全失败）与 GPT-5.5-pro 网页版（六题均无有意义结果）。

> **TL;DR:** Danus is an orchestration system for research-level math reasoning built around a shared, verifier-gated fact graph (a DAG of checked facts and their dependencies); a main agent (Claude Code/Opus 4.8) coordinates parallel Rethlas workers (GPT-5.5) and a stateless verifier, producing long verified proofs — e.g. 3,157 verified facts up to 54 deep on the matroid tangent-class problem — where Rethlas failed 3/3 times on the same prompt and GPT-5.5-pro's web interface produced no meaningful result on all six case-study problems.

## 核心贡献

- 提出以共享事实图（DAG）作为系统唯一真相源的编排架构，将主智能体的全局规划、多 worker 的并行证明搜索、无状态 verifier 的校验三者严格分权，使大量 worker 可在不互相干扰的前提下累积一条很长的证明。
- 相比 Rethlas 单一 Markdown blueprint 的设计，事实图带来两大优势：让每个 worker 的上下文小而聚焦；支持并行写入与"撤销"（revocation，删去错误事实及其传递依赖），从而可扩展到数百乃至数千事实的证明。
- 设计两级记忆（worker 私有本地记忆 + 全局记忆）+ 主智能体周期性摘要，使任一智能体读到的内容有界，而系统整体掌握的有效推理视野远超单个上下文窗口。
- 在六个研究级数学问题（叶状结构的 optimal bend-and-break、三维叶状 Shokurov 全局指标猜想、有理奇异族的全 Cartier 指数、Matryoshka 数的阶乘渐近、对数向量场刻画加权齐性、matroid wonderful 紧化的切类）上完成验证证明，并配套将事实图转写为可读论文、再整篇过 verifier 的写作流程。
- 在 matroid 切类问题上提供了与 Rethlas 的对照实验：相同 worker、相同 verifier、相同 prompt 下，Rethlas 三次均失败，Danus 产出完整验证解，证明提升来自编排而非模型本身。系统已开源。

## Key Contributions

- Introduces Danus, an orchestration system whose central design element is a shared fact graph — a DAG of verifier-checked facts with edges recording logical dependencies — that serves as the system's single source of truth and lets many parallel workers contribute to one long proof without interfering.
- Strict separation of powers: a main agent (built on Claude Code with Claude Opus 4.8, chosen after evaluating Codex, Claude Code, and OpenClaw) does only global planning/coordination; Rethlas-generation workers (running GPT-5.5 via Codex, 3–9 per project split between "high" and "xhigh" effort) do proof search; a stateless verifier (GPT-5.5) is the sole authority on correctness; GPT-5.5-pro is used at most ~hourly as a low-frequency expert reference.
- Argues and demonstrates that the fact graph beats Rethlas's single-blueprint design on two axes: per-worker context stays small and focused, and parallel work accumulates into one shared structure rather than colliding; revocation removes a wrong fact together with everything depending on it.
- A two-tier memory (per-worker local logs + a global memory of plans, dead ends, examples, and GPT-5.5-pro consultations) plus periodic main-agent summaries keep each agent's reading bounded while the system's effective horizon grows.
- A writing pipeline treats the manuscript as a new artifact: the fact graph is rendered into prose and then re-submitted to the verifier as a whole, revised section by section until it passes, catching the "seam errors" introduced when verified facts are compressed.
- Six research-level case studies in algebraic geometry, singularity theory, and combinatorics, each with explicit reporting of human input; a near-controlled comparison against Rethlas (same workers/verifier/prompt) and against GPT-5.5-pro's web interface; system is open-sourced at https://github.com/frenzymath/Danus.

## Methodology

Danus is built on the worker–verifier core of Rethlas and adds a main agent plus a fact graph. The workflow: a mathematician states a problem in natural language to the main agent, which forms an initial plan (consulting GPT-5.5-pro for high-level strategy, at most about once per hour) and assigns directions to workers. Workers are Rethlas generation agents running on Codex/GPT-5.5; each focuses on one claim at a time (a lemma, a counterexample, or a toy example), submits statement+proof to the verifier citing existing fact identifiers, and iterates submit–verify–repair until the claim passes and is admitted to the fact graph as a node with incoming dependency edges. Three to nine workers run in parallel, split roughly evenly between "high" and "xhigh" reasoning effort to diversify the swarm; they explore both constructive and refutational routes.

The fact graph is a DAG; every node is a statement with a verifier-checked proof, every edge a logical dependency. Workers draw on it by search rather than carrying the whole proof, so per-worker context stays small even as the proof grows to thousands of facts. Revocation removes a wrong fact and all of its transitive dependents (needed only once across all experiments). Beyond the graph, a per-worker local log and a shared global memory (plans, promising directions, dead ends, constructed examples/counterexamples, and every GPT-5.5-pro consultation) prevent repeated failed exploration.

The main agent periodically (every 1–2 hours) re-reads worker logs, memory, and the fact graph, produces a summary, asks GPT-5.5-pro for the next strategic step, and re-assigns workers; iteration stops only when the target theorem (or its refutation) appears as a verified fact. Reading the large evolving state is treated as a coding-agent task; after evaluating Codex, Claude Code, and OpenClaw, the authors selected Claude Code with Claude Opus 4.8 for the main agent, while workers and verifier run on GPT-5.5 (judged stronger at mathematics). The verifier is stateless — a fresh instance per submission — and is allowed to read cited facts in the graph, which suppresses hallucination and skipped steps.

Summarization is done by an isolated agent that sees only the problem statement and verified mathematics (no system internals), governed by prohibitions: never estimate progress numerically, report a result as proven only when a verified fact settles it with no unmatched hypothesis. Paper writing turns the fact graph into prose; because compression creates new "seam" mathematics not present as facts, the completed draft is re-submitted to the verifier as a whole and revised until it passes. Long manuscripts are decomposed into self-contained parts each ending in a designated result.

## Results & Highlights

- **Matroid tangent-class problem (controlled comparison, Section 3.6):** Danus built a fact graph of **3,157 verified facts with 8,616 dependency edges, in chains up to 54 facts deep**, of which **664 form the supporting closure of the theorem**, with the global memory recording **636 proof attempts, 151 counterexamples, and 25 dead ends**, over ~5 days with 7 workers. Run on the *same* prompt with the *same* worker and verifier models, **Rethlas failed all 3 attempts** (nothing passed the verifier) and **GPT-5.5-pro's web interface produced no meaningful result**; the constructed class recovers the tangent class and its three key properties of Cheng [10], a paper deliberately withheld.
- **Matryoshka asymptotics (Section 3.4), zero human math input:** proved Kotěšovec's conjecture with a certified enclosure **0.00542831750 ≤ S ≤ 0.00542831848 (width < 10⁻⁹)** for the limit constant; the run went **~90 minutes from launch to the verified target theorem**, with **100 verified facts in chains up to 15 deep (37 in the supporting closure)** and a single proof attempt recorded — no human mathematical guidance of any kind.
- **Threefold foliated Shokurov global index conjecture (Section 3.2):** resolved the 3D case, giving a uniform index **I ≤ 30** for canonical, non-algebraically-integrable foliations with B = 0; 7 workers, ~8-hour first wave, **fact graph of 784 verified facts (77 in the supporting closure)**.
- **Weighted homogeneity via log vector fields (Section 3.5):** 7 workers over 2 waves produced **687 verified facts with 23 revoked** during reference repair, dependency chains up to 21 deep, supporting closure of ~10 facts, global memory of **91 proof attempts and 49 counterexamples**.
- **Optimal bend-and-break for foliations (Section 3.1):** proved the optimal constant is **r + 1** (improving the non-optimal 2n of Shepherd-Barron and 2r of Bogomolov–McQuillan); 5 workers, **63 verified facts and 239 failed paths**.
- **GPT-5.5-pro baseline (Section 3):** each of the six problems was posed independently to the GPT-5.5-pro web interface; **in no case did it produce a meaningful result**, whereas Danus delivered verified proofs in all six.
- **Verifier reliability:** across all experiments, only a handful of verifier slips occurred (almost all traceable to erroneous cited references), revocation was needed only once, and the few writing-stage errors were caught by re-verifying the manuscript as a whole.

## Authors & Credibility

The 11 authors span Peking University (School of Mathematical Sciences, BICMR, New Cornerstone Science Laboratory), Kyoto University's RIMS, Tianjin University, Zhongguancun Academy, Stanford Mathematics, Westlake Institute for Advanced Study, and Tongji University. Corresponding author Bin Dong (PKU/BICMR) has a track record on AI-for-math agents (the Rethlas and Matlas predecessor systems are co-authored by the same group). Several case-study companion papers (theorems 1–6) are posted on arXiv by overlapping author sets, indicating the work is tied to a real, ongoing research program rather than a standalone benchmark exercise. The system is open-sourced, and the six case studies are described with unusual honesty about exactly which steps required human intervention.

## Critical Assessment

- **Novelty:** Genuine. The delta over Rethlas is not "more roles" but scaling the *number of agents doing proof generation* via a verifier-gated DAG memory; this is a clean, well-motivated answer to an explicitly identified gap in the generate–verify–revise literature (Aletheia, QED, ProofCouncil, AI co-mathematician all scale role diversity, not parallel proof producers). The "separation of powers" framing (main agent cannot introduce unverified math) is a non-trivial design constraint enforced by the architecture.
- **Rigor:** Strong on internals, weak on standard ML evaluation. The paper reports no aggregate metric, no confidence intervals, no ablation over worker count, verifier-precision/recall on a held-out set, or scaling curves (e.g. 3 vs 5 vs 9 workers, or "high" vs "xhigh" mix). The single near-controlled comparison (Section 3.6: Rethlas 0/3 vs Danus 1/1) is compelling but n = 1 problem and n = 3 Rethlas trials is a small sample; one cannot infer a general advantage from it. Critically, the baseline is **GPT-5.5-pro via its web interface with no agent harness** — a weak, non-comparable baseline; the paper never compares against Aletheia, QED, ProofCouncil, or AI co-mathematician on any shared problem, so "Danus > prior agents" is asserted only via the predecessor Rethlas.
- **Reproducibility:** Mixed. The framework is open-sourced (https://github.com/frenzymath/Danus), and prompts/human-input accounting are reported per case. But the workers and verifier run on GPT-5.5, the main agent on Claude Opus 4.8, and strategic guidance on GPT-5.5-pro — all closed proprietary models behind API/web access whose behavior can drift, so exact reproduction is impossible. Compute cost, token usage, and the number of verifier calls per verified fact are not reported.
- **Significance:** High for the AI-for-math subfield if the case studies hold up to peer review; several theorems (3D foliated Shokurov, optimal foliated bend-and-break constant r+1, Matryoshka asymptotics) are presented as new mathematical results with companion arXiv papers. The fact-graph-as-source-of-truth idea is likely to be picked up by other agent builders.
- **Red flags / overclaiming:** (1) The authors repeatedly stress that the hardest steps in several case studies were unlocked by a human hint (foliated MMP in 3.2, the misread "≥3" vs ">3" hypothesis in 3.3, the flawed "nilpotent" reference in 3.5, the rational-vs-integral reset in 3.6), yet the framing of "research-level problems solved" can read more strongly than the reality of semi-autonomous collaboration. (2) One case (Theorem 6) shipped with an incomplete justification for Lemma 8.7, admitted by the authors — a reminder that "verified fact graph" still left a gap that human review caught. (3) Danus inherits correctness from its references: the verifier takes cited references as correct, so an erroneous reference propagates (as in 3.5) until a human catches it — the "source of truth" guarantee is conditional on reference quality, not absolute. (4) The GPT-5.5-pro baseline producing "no meaningful result" is unsurprising for a single-shot web call on research math and should not be headlined as evidence of Danus superiority over frontier-model agents.

## Strengths

- Architecturally clean: the fact graph as the sole source of truth, with strict role separation preventing the planning agent from injecting unverified math, is a principled and reusable design.
- Scales test-time compute along two clearly articulated axes (width via parallel workers, depth via trusted accumulation + summarized memory), directly addressing Rethlas's one-line-at-a-time limitation.
- Honest, fine-grained reporting of human input per case study (including an embarrassing PDF-misread bug in 3.3 and a flawed literature reference in 3.5) builds trust.
- The writing pipeline that re-verifies the entire manuscript catches the "compression errors at the seams" that would otherwise corrupt a readable paper — a real and well-observed failure mode.
- Tied to genuine new mathematics with companion arXiv papers, and the system is open-sourced.

## Limitations

- No systematic ablations: worker count, effort-level mix, main-agent model choice, and GPT-5.5-pro consultation frequency are justified only anecdotally.
- The only apples-to-apples baseline is Rethlas on one problem; no comparison with other contemporary math-research agents (Aletheia, QED, ProofCouncil, AI co-mathematician) on shared tasks.
- Depends on closed proprietary models (GPT-5.5, GPT-5.5-pro, Claude Opus 4.8); exact reproduction and long-term stability are not guaranteed.
- The verifier trusts cited references, so reference-borne errors propagate until human review — the fact graph's reliability is conditional, not absolute.
- Human input remains decisive for the hardest steps; the paper itself notes scaling "does not create the path" and that for the deepest open problems "there may be no one to supply" the missing idea.
- Paper-writing quality is still imperfect (citation format, notation, motivation thinness), requiring manual post-editing; one shipped theorem had an incomplete lemma justification.

## Relevance

对关注 LLM 智能体编排、长程推理与"记忆机制"设计的研究者，Danus 提供了一个把共享 DAG 状态作为全局真相源、以分权 + 并行 + 无状态校验来扩展 test-time 计算的范例，其事实图撤销与"先写后整篇过 verifier"的写作流程对任何需要长链可靠累积的智能体系统（不止数学证明）都有直接借鉴价值。

## Questions for Further Exploration

- How does Danus's success rate and fact-graph size scale with worker count (3 vs 5 vs 9 vs 20) and with the "high"/"xhigh" effort mix — is there a diminishing-returns or interference regime?
- What is the verifier's precision and recall on a held-out set of proofs with planted errors, and how does error rate accumulate along dependency chains of depth 30–60?
- Can the fact-graph + stateless-verifier design be ported to non-mathematical domains (formal verification, code correctness, scientific argument construction) where claims also admit verifier-checkable dependencies?
- How would Danus perform head-to-head against Aletheia, QED, ProofCouncil, or the AI co-mathematician on a shared benchmark of open problems, rather than only against Rethlas and a single-shot GPT-5.5-pro web call?
- What fraction of the verified facts are ultimately "wasted" (not in the supporting closure) as problems scale, and can the main agent's re-assignment policy reduce this overhead?
- Is the reliance on a separate frontier model (GPT-5.5-pro) for strategic hints a fundamental bottleneck, or can the main agent's summarization + memory substitute for it on most problems?
