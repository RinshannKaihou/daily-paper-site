Controlling Tool Use with Heading-Specific Activation Steering / Yuqi Chen, Vincent Siu, Yang Liu, Dawn Song, Chenguang Wang / UC Santa Cruz · UC Berkeley / 2026-07-07 AIWILD @ ICML 2026 · workshop

“Heading-anchored steering vectors bidirectionally control LLM tool use — yet reveal tool representations are non-parametric and geometrically diffuse.

A training-free, inference-time intervention suppresses or amplifies tool calls across five open models and three SMART domains — while a probe with AUROC 1.000 turns out to be causally inert, decoupling linear decodability from controllability.

Key metric
3.90→0.56
Tool calls per question on Mistral-7B / Math under activation addition, while accuracy barely moves 13.3 → 11.8.
Why this matters
Training-free inference-time control of tool overuse, plus evidence that tool-use is geometrically non-parametric — a baseline for tool-policy × representation-engineering work.
Critical edge
It is a blunt on/off knob: suppression cannot tell necessary from redundant calls, so it degrades accuracy exactly where tools are genuinely required (Time, Intention).
01

Methodology

A training-free, inference-time intervention built on structured heading-anchors (### Reasoning, ### Code, ### Search, ### AskUser). For each tool type k, 20 GPT-4o-generated trajectories per domain (60 total) are run through the model; hidden states at the leading ### token of ### Reasoning and the target tool heading are averaged per trajectory, then differenced (Δ = h̄R − h̄k) and aggregated into a steering vector vk.

By construction vk points from the reasoning state toward the tool-heading state, so adding α·vk (α = 1) suppresses tool use; orthogonalizing against vk amplifies it. The intervention layer is swept once per model and fixed at the tool-usage curve's "elbow" — a consistent normalized depth of 0.45–0.60 across all five architectures. Evaluated on three SMART-benchmark domains (Math / Time / Intention) across Mistral-7B through Llama-3.1-70B.

03

Strengths & Limitations

Strengths +

  • Real gap, well-motivated. First to steer a behavior with no parametric grounding — tool use lives in inference context, not weights.
  • Strong internal validity. Addition vs. orthogonalization move in opposite directions across all 5 models / 3 domains, confirming the direction encodes the decision.
  • Sharp 70B finding. The probe-vs-causality dissociation complicates the naive "linearly decodable ⇒ steerable" assumption.
  • Honest framing. Geometric irregularity is presented as an open question, not a sold mechanism.
  • Practical. Training-free, inference-time fix for a real failure mode — tool overuse.

Limitations −

  • Strong output-format dependency. Requires explicit heading anchors; function-call / free-form tool use is out of scope; schema controls are single-model / single-domain.
  • Tiny extraction set. Only 60 GPT-4o trajectories (20 per domain); sensitivity to the seed set is not analyzed.
  • Blunt knob. Cannot separate necessary from redundant calls — accuracy drops where tools are genuinely needed (Time / Intention).
  • Unresolved mechanism. Why causal control survives diffuse bimodal geometry is left open; multi-turn structure blocks reconciliation with probing.
  • Workshop depth + no head-to-head. 70B and cross-format analyses are single-model; no comparison to output-level mitigations (Self-DC / SMART-cal).
04

Open questions

Q1 · Can the vector be made conditional — suppressing only redundant calls while preserving necessary ones (closing the Time / Intention accuracy gap)?
Q2 · Does the diffuse bimodal geometry generalize to function-call / JSON-style tool use, or is it partly an artifact of the heading-anchor schema?
Q3 · How sensitive are the vectors to the 20-example-per-domain GPT-4o seed set — variance across resamples and generators?
Q4 · Why does the most linearly-decodable direction (probe AUROC 1.000) fail causally on 70B — a property of larger models, or of non-parametric concepts?
Q5 · Could the consistent 0.45–0.60 normalized-depth elbow serve as a model-agnostic heuristic for locating other context-driven decision representations?
