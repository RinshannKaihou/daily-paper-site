> **一句话总结：** 本文给出工具增强下有限精度循环模型表达能力的精确二分定理：有限状态接口的工具仅增加 log₂|M|+O(1) 位内存即可被内化（仍是正则语言），而单一极小的无界磁带工具即可使仅用 O(log|Q|+log|Γ|) 位内部状态的控制器图灵完备，并由一层选择性仿射 SSM（one-hot 二值隐状态、{0,1} 转移矩阵、零偏置）精确实现。

> **TL;DR:** The paper proves a sharp architecture-level dichotomy for tool-augmented finite-precision recurrent models: finite-state bounded-interface oracles add only log₂|M|+O(1) bits and keep the system finite-state (still regular languages), whereas a single minimal unbounded tape oracle (local read/write/move) makes a controller with O(log|Q|+log|Γ|) bits Turing complete — concretely, the equality problem EQ_n needs ≥2^n states without tools but a single constant-size controller with tape access solves it for all n — and this is realized exactly by a one-layer finite-precision selective affine SSM with binary one-hot hidden states, {0,1} transition matrices, and zero biases; a matching impossibility result shows direct one-step realization of an arbitrary B-state transition map requires recurrent dimension exactly B−1.

## 核心贡献

- **精确的二分定理**：首次在架构层面给出工具增强有限精度循环模型表达能力的完全刻画——表达能力的提升完全由工具的接口与状态空间决定，而非由控制器决定。有限状态有界接口工具"几乎不增加"表达能力，而单一极小无界工具即达图灵完备。
- **有限状态工具的乘积状态模拟定理（命题4）**：任何有限状态有界接口神谕可在 log₂|M|+O(1) 位的代价下被控制器内化，增强后的系统仍是有限状态机（仍只识别正则语言）；并指出"有限状态初始化"假设是关键——若允许任意初始化映射，则一位神谕即可任意强大。
- **磁带神谕的图灵完备性（定理7）**：定义只支持局部 read/write/move 命令的磁带神谕，证明具有 O(log|Q|+log|Γ|) 位内部内存（对输入长度为常数）的控制器可模拟任意单带图灵机。
- **具体的指数级下界分离（定理9）**：承诺问题 EQ_n（输入 u#v，u,v∈{0,1}^n，判断 u=v）在无工具时至少需要 2^n 个内部状态，而单一常数大小控制器配合磁带工具即可对所有 n 求解——给出明确的指数分离。
- **由一层选择性仿射 SSM 精确实现（引理12、定理13）**：每个观测驱动的有限状态控制器可由一层有限精度选择性仿射 SSM 精确实现，采用 one-hot 二值隐状态、{0,1} 转移矩阵、零偏置，隐维度 d=O(|Q||Γ|)；指出"选择性"（输入相关的转移矩阵）是必要条件。
- **精确的资源核算（附录）**：三符号实现常数 C₀=9（T_B ⊆ S⁽¹⁾_{9B}）；对数级神谕辅助通用模拟定理19（O(log B) 位循环位即可模拟任意 B 状态图灵机）；匹配的不可能性定理23（无外部内存时，单步精确实现任意 B 状态转移映射在任意层数三角选择性仿射层下恰好需要维度 B−1，即 D_L(B)=B−1）。

## Key Contributions

- A sharp, architecture-level dichotomy characterizing exactly when tool access increases the computational expressivity of a fixed finite-precision recurrent sequence model: the gain is governed entirely by the tool's interface and state space, not by the controller.
- A product-state simulation theorem (Proposition 4) showing finite-state bounded-interface oracles can be internalized at a cost of only log₂|M|+O(1) bits, leaving the system finite-state (still regular languages); Remark 5 proves the finite-state-initialization hypothesis is essential, since an arbitrary Init map makes even a one-bit oracle arbitrarily powerful.
- A Turing-completeness theorem (Theorem 7) for a tape oracle with only local read/write/move commands: a controller with O(log|Q|+log|Γ|) bits of internal memory — constant in the input length — simulates any single-tape Turing machine.
- A concrete exponential lower-bound separation (Theorem 9): the promise problem EQ_n requires at least 2^n internal states for any no-tool controller, whereas a single constant-size controller with tape access solves EQ_n for every n.
- An exact realization by a one-layer finite-precision selective affine SSM (Lemma 12, Theorem 13) with one-hot binary hidden states, {0,1} transition matrices, and zero biases, at hidden dimension d=O(|Q||Γ|); selectivity (input-dependent transition matrices) is shown to be essential.
- Tight two-sided resource accounting in the supplement: an explicit three-symbol realization with constant C₀=9 (Lemma 17); a logarithmic oracle-assisted universal simulation (Theorem 19) showing O(log B) recurrent bits suffice to simulate any B-state Turing machine; and a matching impossibility result (Theorem 23) that, without external memory, exactly realizing an arbitrary B-state transition map in one affine recurrent update requires total recurrent dimension exactly B−1 for any number of triangular selective-affine layers, i.e., D_L(B)=B−1.

## Methodology

The work is purely theoretical — exact constructions and unconditional lower bounds, with no training, approximation, or probabilistic arguments. The approach proceeds in four stages.

1. Controller abstraction (Section 3). Any fixed finite-precision recurrent model with B bits of internal state (including finite-precision SSMs) is abstracted as a deterministic finite-state controller C=(S, s₀, δ_in, α, δ_out, δ_obs) that reads input, then in a generation phase emits output tokens, issues tool commands from a finite command alphabet, receives observations from a finite observation alphabet, or halts. Tagged transcripts like [TOOL]/[OBS] are shown to be only a finite-state encoding convention with no extra computational power.

2. Finite-state tool case (Section 4). The authors define a finite-state bounded-interface oracle O=(M, m₀, η, D, R, Step) and prove the product-state simulation: a no-tool controller Ĉ tracking the pair (controller state, oracle memory) ∈ S×M reproduces the exact scored output stream. The proof is by induction on the number of generation steps.

3. Tape oracle and Turing completeness (Sections 5–7). They define a tape oracle with memory space of all tape configurations (T,h), supporting read, move left/right, and write(γ). Theorem 7 constructs a controller C_T that stores the simulated machine state q, a mode flag, and at most one tape symbol γ, executing the read→write→move loop prescribed by δ_T. The invariant (oracle tape equals T's tape after each simulated transition) is proved by induction. Theorem 9 gives the finite separation EQ_n via a counting/pigeonhole argument for the lower bound and a marking-and-comparing single-tape algorithm for the upper bound.

4. SSM realization and resource accounting (Sections 8 and Appendices A–B). Lemma 12 enumerates the finite state set S={s₁,…,s_m}, encodes s_i by the standard basis vector e_i, and defines each matrix A_ω columnwise by A_ω e_i = e_j iff δ(s_i,ω)=s_j, with all biases zero, proving by induction that the SSM hidden state is always the one-hot encoding of the finite controller's state. The logarithmic universal simulation (Theorem 19) hard-codes the machine description ⟨M⟩ in the fixed readout, copies it onto the tape, then transfers control to a fixed universal controller U derived from Neary and Woods (2009). The matching lower bound (Theorem 23) uses a hard automaton combining a cyclic permutation c(i)=i+1 mod B with a successor-with-absorbing-endpoint map s; it proves this automaton has no nontrivial congruences, so any layered realization must distinguish all B states at once at a single layer, yielding a nilpotent orbit of length B−1 requiring dimension ≥B−1, matched by an explicit simplex construction giving the upper bound B−1.

## Results & Highlights

This is a pure theory paper; its quantitative claims are exact resource bounds rather than empirical accuracies. The "benchmarks" are formal-language/promise problems (EQ_n, L_eq={u#u}) and B-state Turing-machine classes T_B; the "baseline" is the no-tool finite-state controller.

- On the EQ_n promise problem (inputs u#v, u,v∈{0,1}^n, output T iff u=v): any deterministic no-tool finite-state controller needs at least 2^n internal states (Theorem 9(i)), whereas a single constant-size controller with interactive tape access solves EQ_n for every n (Theorem 9(ii)) — an exact exponential separation with the no-tool baseline at 2^n and the tool-augmented cost constant.
- Finite-state bounded-interface tools are essentially free: internalizing any such oracle with memory set M costs only log₂|M|+O(1) additional bits, and the augmented system remains finite-state and recognizes only regular languages (Proposition 4 / Table 1).
- Tape-oracle Turing completeness is achieved with O(log|Q|+log|Γ|) bits of controller memory to simulate any single-tape Turing machine with state set Q and tape alphabet Γ; for fixed Γ this is O(log|Q|) bits, constant in the input length (Theorem 7 / Table 1).
- The SSM realization uses hidden dimension d=O(|Q||Γ|) with binary {0,1} transition matrices and zero biases (Theorem 13); the explicit three-symbol construction achieves the linear bound with absolute constant C₀=9, i.e., T_B ⊆ S⁽¹⁾_{9B} (Lemma 17).
- Logarithmic oracle-assisted universal simulation: there is an absolute constant C₁ such that T_B ⊆ S⁽¹⁾_{⌈C₁ log₂(B+1)⌉} for every B≥1 (Theorem 19), by offloading the machine description to the tape; the machine description length is |⟨M⟩| ≤ c·B·log₂(B+1).
- Matching impossibility for direct realization: for every B≥2 and every L≥1, D_L(B)=B−1 exactly (Theorem 23), i.e., without external memory exactly realizing an arbitrary B-state transition map f:[B]×{0,1}→[B] in one affine recurrent update requires total recurrent dimension exactly B−1, giving Θ(B) bits at fixed precision and exactly B−1 bits over F₂.

No quantitative results reported in the empirical/experimental sense — all results are unconditional theorems with explicit constants; the evidence consists of exact constructions and matching lower bounds rather than measured accuracies.

## Critical Assessment

- Novelty: Genuinely novel framing. Prior work (Merrill 2019; Weiss et al. 2018; Zubić et al. 2025, 2026) established that finite-precision recurrent models are finite-state and analyzed SSM/chain-of-thought limitations, and Delétang et al. (2023) showed empirically that external-memory structure determines learnable language classes. This paper gives the corresponding exact, constructive, interface-driven dichotomy at the controller level — a clean theoretical gap not previously filled. The identification of selectivity as the architectural ingredient for exact finite control is a precise, non-obvious observation.
- Rigor: High. All claims are exact theorems with complete proofs by induction; constants are made explicit (C₀=9); the lower bound D_L(B)=B−1 is matched by an explicit upper-bound construction, so it is tight, not merely asymptotic. The proof of Theorem 23 via the cyclic-permutation-plus-absorbing-successor automaton and its congruence structure is elegant and correct.
- Reproducibility: For a theory paper, reproducibility is strong — every construction is algorithmic and could be implemented, and the resource conventions are stated. However, there is no code release and no empirical validation that the constructed one-hot SSMs are learnable or trainable; the results are realizability statements, not learnability statements.
- Significance: The dichotomy is conceptually important for the tool-use-in-LLMs literature (Toolformer, ReAct, scratchpads), giving a minimal formal semantics for the command/observation loop and pinning down exactly when the loop helps. The matching logarithmic-vs-linear resource accounting (Theorem 19 vs Theorem 23) is a sharp, paper-specific finding: O(log B) recurrent bits suffice only when the tape is free external memory and the readout table is uncharged, and the gap quantifies precisely what external persistent memory buys.
- Red flags and overclaiming: The paper is careful about scope — Remark 14 explicitly warns the SSM result is about selective controllers and "should not be read as a statement about every narrower SSM architecture," and Remark 20 states the two conventions on which the logarithmic result depends. The main risk of overinterpretation is external: the realizability results say nothing about whether such controllers can be learned by gradient descent, whether the one-hot {0,1} constructions are stable under finite-precision training, or whether real Toolformer/ReAct systems instantiate the assumed command/observation interface. A specific weakness worth flagging: the "weakest baseline" against which the EQ_n separation is measured is the no-tool finite-state controller, which is trivially unable to recognize the non-regular language L_eq — the exponential 2^n lower bound is the standard Myhill–Nerode-style counting argument and, while correct, is not surprising given that finite-state machines cannot recognize L_eq at all. The genuinely informative separation is the constant-size-controller upper bound, not the 2^n lower bound. A missing piece is any stochastic/learned-controller extension: the authors list this as future work, but the paper's relevance to practical tool-using LLMs is currently only analogical. There is also no ablation on the dependence of the C₀=9 constant on the tape-alphabet size (it is stated for the three-symbol Γ={0,1,⊔} case), and no discussion of how the construction behaves for the multi-tape or multi-head extensions that real scratchpads approximate.

## Strengths

- A clean, complete, and sharp dichotomy that is exactly the right level of abstraction: interface-driven, controller-agnostic, and unconditional.
- All results are tight: the linear B−1 bound is matched by an explicit construction, the logarithmic O(log B) bound is complemented by the linear impossibility, and the constants are made explicit (C₀=9).
- The identification of selectivity (input-dependent transition matrices) as the necessary and sufficient architectural ingredient for exact finite control in the interactive setting is a precise, reusable insight.
- The two-sided resource accounting in the supplement — showing that the logarithmic and linear answers are both correct under different, precisely specified conventions — is unusually careful and prevents misreading of the main results.
- The paper is honest about its assumptions: Remarks 5, 14, 20, and 24 explicitly delineate what each theorem does and does not claim.

## Limitations

- Purely a realizability/expressivity theory paper: no learning results, no approximation guarantees, no probabilistic or stochastic controllers. Whether the exact one-hot {0,1} selective affine SSMs are learnable by gradient descent or stable under finite-precision training is unaddressed.
- The construction is architecturally narrow in one respect: the positive SSM result (Theorem 13) requires selectivity and one-hot binary dynamics. It explicitly does not apply to fixed linear time-invariant recurrences (Remark 14), so it does not directly certify Turing completeness for vanilla (non-selective) Mamba/S4 controllers.
- The relevance to practical tool-using LLMs (Toolformer, ReAct) is analogical: real systems do not expose a clean finite command/observation interface, their tools are often not finite-state nor cleanly tape-like, and the paper provides no empirical bridge.
- The exponential EQ_n lower bound (2^n states) is the standard finite-state-machine counting argument; the surprising part is the constant-size upper bound, not the lower bound, so the "exponential separation" framing slightly overweights the lower-bound side.
- The logarithmic universal simulation (Theorem 19) depends on two strong conventions (free external tape memory; uncharged readout/parameter table). Remark 20 acknowledges this, but a reader could still misread O(log B) as a property of the SSM itself rather than of the oracle-assisted accounting.
- No treatment of multiple concurrent tools, transformers as controllers, or the interaction with chain-of-thought budgets — all explicitly listed as future work.

## Relevance

对关注 SSM/循环模型表达能力与形式语言能力的工作高度相关：本文给出了工具增强（tool use）何时能突破有限精度循环模型正则语言局限的精确理论刻画，直接延续 Zubić 等人关于 SSM 表达能力限制的研究脉络，并为理解 Toolformer/ReAct 等工具增强 LLM 的计算能力提供了形式语义基础。

## Questions for Further Exploration

- Can the exact one-hot {0,1} selective affine SSM constructions be learned by gradient descent on EQ_n or L_eq, and are the {0,1} transition matrices stable under finite-precision (e.g., bfloat16) training, or do they collapse to non-one-hot states?
- How does the dichotomy change when the controller is a Transformer rather than a recurrent model — does the same interface-driven finite-state-vs-tape distinction apply, and at what internal-memory cost?
- The paper assumes a single sequential tool; what is the expressivity landscape with multiple concurrent tools (e.g., a calculator plus a tape), and can a finite-state combination of finite-state tools ever exceed regular languages?
- The logarithmic O(log B) result relies on free tape memory and an uncharged readout table; what is the tight bound when the readout/parameter budget is charged, interpolating between the O(log B) oracle-assisted and the B−1 direct-realization extremes?
- Does the chain-of-thought budget interact multiplicatively or additively with tool access — i.e., can a bounded number of CoT steps plus a finite-state tool exceed regular languages, or is the tape-oracle the essential ingredient?
- The lower bound D_L(B)=B−1 uses a specific hard automaton (cycle + absorbing successor); is there a natural distribution over transition maps under which the average-case required dimension is substantially smaller than B−1?
