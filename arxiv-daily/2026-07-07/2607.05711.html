FourTune: Towards Fully 4-Bit Efficient Post-Training for Diffusion Models

Key Finding

The first fully 4-bit post-training framework for diffusion models matches BF16 LoRA quality at 2.25× less memory and 2.27× faster training.

Why This Matters
Breaks the QLoRA memory-vs-speed trade-off, making 12B–20B diffusion post-training feasible on consumer Blackwell GPUs — and the frozen-stabilizer concept transfers cleanly to LLM PEFT.
Critical Edge
Speedups are hardware-locked to Blackwell NVFP4 Tensor Cores, and quality only matches — never beats — BF16 LoRA; the distillation table is inflated by a broken SparseLoRA baseline (FID 202.51).

01  Methodology3 components

FourTune is a native 4-bit post-training pipeline. Each pretrained weight W is SVD-decomposed into a quantization-friendly residual R and a low-rank outlier component Lstab, so that W ≈ R + Lstab. The pipeline then runs three parallel branches and decouples numerical stabilization from task adaptation — gradients update only the LoRA adapter, while the stabilizer and 4-bit backbone stay frozen.

01Triple-branch hybrid precision

A frozen 4-bit backbone (NVFP4) computes GEMM4bit(Q(X), Q(R)); a frozen full-precision stabilizer absorbs high-magnitude outliers; a trainable LoRA adapter handles task fit.

02Block-wise 4-bit backward

16×16 blocks localize scale factors, so Q(G)Q(R)ᵀ is computed without dequantizing weights — sidestepping the reduction-axis misalignment of per-output-channel schemes.

03Kernel fusion

Shared-input down-projection fused across branches (+1.82×); 4-bit backbone GEMM fused with up-projection; MLP FC1+FC2 quantization fused (+1.10×) → 2.52× per-step.

03  Assessmentstrengths vs limitations

Strengths

  • Truly native 4-bit training (W4A4G4) with no online dequantization — a real advance over QLoRA's W4A16G16.
  • Clean three-component design where each piece (stabilizer, block-wise quant, kernel fusion) is isolated by its own ablation.
  • Broad evaluation across three post-training regimes (customization, RL, distillation) and two scales (12B, 20B) — not a single cherry-picked benchmark.
  • End-to-end efficiency reported, with both memory and latency on two GPU tiers.
  • The frozen-stabilizer concept is architecturally elegant and plausibly transferable to LLM PEFT.

Limitations

  • Hardware lock-in: requires Blackwell NVFP4 Tensor Cores — no portability to Hopper / Ada / Ampere discussed.
  • Stabilizer underanalyzed: no rank-vs-quality/memory study, no guidance for new architectures.
  • Quality only matches, never exceeds, BF16 LoRA — the value is purely efficiency.
  • Selective RL reporting: ImageReward regression (1.0152 vs 1.0735) is not discussed.
  • Distillation comparison is thin: SparseLoRA broken (FID 202.51); newer 4-bit trainers (QeRL) not benchmarked.
  • Block-wise granularity cost (LPIPS +12%) is real but downplayed.

04  Open Questionsfor further exploration

Does the frozen SVD stabilizer transfer to LLM post-training (Llama / Qwen), where outlier structure and gradient statistics differ from diffusion DiT blocks?
What is the stabilizer-rank vs quality/memory Pareto curve, and is there a principled, data-driven way to choose rank per layer?
How do speedups degrade on non-Blackwell hardware (Hopper, Ada) — can block-wise 4-bit backward help via INT4, or only NVFP4?
Could the ImageReward regression under RL be fixed by keeping specific layers at higher precision — revealing which layers are most quantization-sensitive?
Is there a regime (long training, very high rank, sparse-reward RL) where 4-bit gradient error finally causes a measurable quality gap?
How does FourTune behave at even lower precision (W2A2, mixed 2/4-bit) — does the stabilizer still prevent explosion, or is a second-order correction needed?
