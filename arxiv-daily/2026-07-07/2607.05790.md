> **一句话总结：** 本文针对工具增强大模型的"工具过度调用"问题，提出基于 heading-anchor（如 `### Code`/`### Search`）的激活引导方法，在 5 个开源模型、3 个领域上实现了对工具调用行为的双向因果控制——激活加法可大幅抑制冗余调用（如 Mistral-7B 在 Math 上从 3.90 降至 0.56 次/题），正交化则放大调用；同时几何分析揭示这种因果效力并不对应干净的线性结构，体现了工具"非参数"本质。

> **TL;DR:** The authors extract steering vectors from hidden states at structured heading-anchor positions (e.g. `### Code`, `### Search`) to bidirectionally control tool invocation in LLMs. Across five open models (Mistral-7B to Llama-3.1-70B) and three SMART-benchmark domains (Math/Time/Intention), activation addition suppresses unnecessary tool calls (e.g. Mistral-7B Math ToolAvgUse 3.90 → 0.56 with ACC barely moving 13.3 → 11.8), while orthogonalization amplifies them above baseline. Crucially, this causal effectiveness coexists with diffuse bimodal geometry, distinguishing tool-use steering vectors from the cleanly linear directions seen for parametrically grounded concepts like refusal.

## 核心贡献

- 首次将激活引导（activation steering）系统应用于一种**本质上非参数**的行为——工具调用：与情感/拒答等编码在权重中的概念不同，工具完全存在于推理上下文里，线性表示假设并不天然成立。
- 提出基于 heading-anchor 的引导向量构造法：在结构化轨迹的 `### Reasoning` 与目标工具标题（`### Code`/`### Search`/`### AskUser`）位置抽取隐状态差分，逐轨迹平均后再聚合，避免长轨迹被池化估计器过度加权。
- 证实该向量对工具调用具有**双向因果控制**：激活加法抑制、正交化放大，且两者方向相反，证明提取方向真实编码了工具调用决策。
- 揭示因果效力与几何结构的**解耦**：工具步隐状态与抑制向量呈弥散双峰分布，而非线性编码应有的单向负对齐；不同工具类型特征维度重叠很低（三方 Jaccard 仅 0.042–0.068），作者将其归因于工具的非参数本质。
- 在 Llama-3.1-70B 上发现"线性可解码性 ≠ 因果可控性"：线性探针 AUROC 达 1.000（完美区分 Code vs Reasoning），但作为因果控制却几乎无效（ToolAvgUse 1.235 vs 均值差方向 0.235）。

## Key Contributions

- First systematic application of activation steering to a behavior that is **non-parametric by construction** — tool invocation lives entirely in inference-time context rather than model weights, so the linear representation hypothesis does not straightforwardly apply.
- A **heading-anchored steering-vector recipe**: hidden states at `### Reasoning` and target tool headings are averaged per-trajectory before aggregation, deliberately avoiding the pooled difference-of-means estimator that would over-weight trajectories with many anchor occurrences.
- Demonstration of **bidirectional causal control**: activation addition suppresses and orthogonalization amplifies tool use in opposing directions across all five models and three domains, confirming the extracted direction genuinely encodes the tool-use decision.
- A **geometric analysis** showing causal effectiveness does not imply clean linear structure — tool-step hidden states exhibit diffuse bimodal cosine alignment with the suppression vector, and different tool types recruit largely distinct feature dimensions.
- A concrete dissociation on Llama-3.1-70B between **probing separability and causal controllability**: the linearly most-decodable direction (probe AUROC 1.000) is nearly inert as a causal control, while the mean-difference vector is causally strong (ToolAvgUse 0.235 vs 1.235).

## Methodology

The method is a lightweight, training-free, inference-time intervention built around structured heading-anchors (`### Reasoning`, `### Code`, `### Search`, `### AskUser`, `### Final Response`).

**Steering-vector construction (per tool type k).** A dataset D_k of complete labeled multi-turn trajectories is collected (20 examples per domain × 3 domains = 60, generated by GPT-4o using domain prompts). The model is run on the full prompt-response sequence and hidden states are extracted at layer l*. Each heading occurrence is represented by the hidden state at its leading `###` token. Within each trajectory, anchor states are averaged separately for `### Reasoning` (h̄_R,i) and the target tool heading k (h̄_k,i). A per-trajectory delta Δ_k,i = h̄_R,i − h̄_k,i is formed, then averaged across trajectories to give v_k. By construction v_k points from the reasoning state toward the tool-heading state, so adding it is suppressive.

**Two interventions** are applied at a fixed layer l* on the `###` Heading-Anchor token when an imminent heading is detected:
- **Activation addition:** h̃_t = h_t + α·v_k (α=1) — shifts the hidden state away from the tool-heading representation, suppressing tool use.
- **Orthogonalization:** project out the v_k component from h_t — removes the suppressive component and amplifies tool use above baseline.

**Steering-layer selection.** The vector is swept across all layers with α=1 to minimize tool usage, and the "elbow" of the tool-usage curve is chosen. Despite differing absolute depths, all five elbows fall within normalized depth 0.45–0.60 (e.g. layer 18 for Mistral-7B, layer 45 for Llama-3.1-70B). The layer is calibrated once per model and held fixed.

**Evaluation.** Three domains from the SMART benchmark: Math (parametric reasoning often suffices), Time (needs up-to-date info), and Intention (needs user clarification). Five instruction-tuned models: Mistral-7B, Llama-3.1-8B, Mistral-Nemo (12B), Mistral-Small (24B), Llama-3.1-70B. The base-model-with-tool-prompt baseline (from Qian et al. 2025) gives the model all tools and descriptions and lets it decide. Metrics: task accuracy (ACC), average tool calls per example (ToolAvgUse), plus for Intention, Missing Details Recovery and Summarized Intention (from IN3).

**Geometric analysis.** Cosine similarity between v_k and unsteered hidden states at reasoning vs tool steps; pairwise cosine and Jaccard overlap of the top-10% magnitude dimensions across tool types; and on Llama-3.1-70B a comparison of the mean-difference direction against a supervised linear probe, diagonally whitened mean difference, norm-matched random, and shuffled-label controls.

## Results & Highlights

- **Math — suppression is near-total yet accuracy barely moves**, confirming many tool calls are redundant: Mistral-7B ToolAvgUse 3.90 → 0.56 with ACC 13.3 → 11.8; Llama-3.1-8B 1.93 → 0.35 with ACC 51.0 → 48.3; Mistral-Small (24B) 1.55 → 0.13 with ACC 76.0 → 65.4. In Math, activation addition drives tool usage "close to zero for all models."
- **Time and Intention — suppression hurts more**, because tool access is genuinely required: Llama-3.1-8B Time ACC drops 56.0 → 45.0 as ToolAvgUse falls 2.05 → 1.50; Mistral-Small Time ACC drops 62.0 → 49.0 as ToolAvgUse falls 1.73 → 1.48; Intention Missing Details Recovery / Summarized Intention drop substantially across all models.
- **Orthogonalization amplifies tool use above baseline in every model/domain but never improves task performance** — e.g. Mistral-7B Math ToolAvgUse 3.80 (orthogonalization, code) vs 3.90 base; Mistral-Small Math ACC falls 76.0 → 71.0 with more tools, indicating the base model already balances invocation and the intervention pushes toward overuse.
- **Steering layers cluster at normalized depth 0.45–0.60** across all five architectures despite different absolute depths (Mistral-7B layer 18 at 0.70 calls, Llama-3.1-70B layer 45 at 1.28, Mistral-Small layer 19 at 0.63), suggesting tool-use representations stabilize at a consistent relative depth.
- **Effect survives beyond the literal `### Code` string** (Llama-3.1-8B, Math): cross-format markdown→json transfer removes 63.9% of tool calls with accuracy essentially unchanged (0.330 → 0.325); heading rename to `### Action B` still yields 70.3% relative suppression. Reverse json→markdown transfer is much weaker (15.5%).
- **Geometry is diffuse, not linearly clean:** tool-step hidden states show a bimodal cosine-similarity distribution to the suppression vector (mass on both acute and oblique sides), while reasoning steps concentrate sharply near zero; cross-tool three-way Jaccard overlap is only 0.042–0.068.
- **Probing separability diverges from causal controllability on Llama-3.1-70B:** the linear probe separates Code vs Reasoning heading states at AUROC 1.000 / F1 1.000 yet is causally inert (ToolAvgUse 1.235, ACC 0.695), while the mean-difference vector (AUROC 0.945, F1 0.481) strongly suppresses (ToolAvgUse 0.235, ACC 0.695).

## Authors & Credibility

The work comes from Yuqi Chen, Vincent Siu, Yang Liu, and Chenguang Wang (UC Santa Cruz) with Dawn Song (UC Berkeley). The group has a direct track record in activation steering — Siu, Song, and Wang are co-authors of SteeringSafety (2025), COSMIC (ACL 2025), and RepiT (2025), all cited heavily here, so the methodology is being applied by people who built parts of it. It was **published at the Second Workshop on Agents in the Wild: Safety, Security, and Beyond (AIWILD) at ICML 2026**, i.e. a peer-reviewed workshop venue rather than a main conference track — appropriate given the focused scope but worth noting when weighting the claims. The methodology is transparent about its estimators, includes random/shuffled controls and a held-out 200-example evaluation, and the geometric irregularity findings are framed as open questions rather than oversold.

## Critical Assessment

- **Novelty:** Genuinely novel angle. Prior steering work targets parametrically grounded concepts (refusal, truthfulness, sentiment); this is among the first to steer a behavior with no parametric grounding, and the dissociation between probing AUROC and causal effect on 70B is a clean, non-obvious finding.
- **Rigor:** Reasonable. The paper includes proper controls — random and shuffled-label baselines (Table 5), norm-matching of compared directions, a held-out 200-example evaluation, and cross-format/heading-rename robustness checks. The per-trajectory averaging justification (avoiding the pooled estimator over-weighting long trajectories) is a thoughtful design detail.
- **Reproducibility:** Mixed. Models, benchmark (SMART), layer-selection heuristic, and α=1 are all specified, and the recipe is lightweight. However, steering vectors are extracted from **GPT-4o-generated trajectories on only 20 examples per domain** (60 total), and these trajectories are not released — the extraction data is a non-trivial hidden dependency. The heading-anchor prompt format itself is a strong prerequisite that limits drop-in reproducibility for arbitrary tool-use setups.
- **Significance:** Moderate but real. The bidirectional control and the probing-vs-causal divergence are conceptually important for representation engineering; the practical impact is bounded because the method only works on structured heading-format outputs and degrades exactly where tools are needed.
- **Red flags / overclaiming:** The paper is commendably restrained — it explicitly states the "relationship between this geometric irregularity and the observed causal effectiveness remains an open question" and that the multi-turn structure "limits direct reconciliation with the probing analysis." The chief weakness is **thin evaluation scope**: cross-format/rename controls are single-model, single-domain (Llama-3.1-8B/Math only); the 70B probing study is also single-model; and the heading-format dependency means the "tool-use direction" may be partly an artifact of the constrained output schema (the asymmetric markdown→json vs json→markdown transfer, and json→json accuracy collapse to 0.010, are telltale signs the vector is schema-sensitive rather than a fully format-independent representation — which the authors honestly acknowledge).

## Strengths

- Clean, well-motivated research question that fills a real gap: steering a non-parametrically grounded behavior.
- Bidirectional causal control (addition vs orthogonalization moving in opposite directions) is a strong internal validity check on the extracted direction.
- The 70B probe-vs-causality dissociation is a sharp, memorable finding that complicates the naive "linearly decodable ⇒ steerable" assumption.
- Honest framing of limitations and open questions; avoids overselling the geometric irregularity as a solved mechanism.
- Practical relevance to tool overuse — a real failure mode — with a training-free, inference-time intervention.

## Limitations

- **Strong output-format dependency.** The method requires the model to emit explicit heading anchors; free-form or function-call-style tool use is out of scope, and the schema-sensitivity controls are only single-model/single-domain.
- **Tiny extraction set.** Only 60 GPT-4o-generated trajectories (20 per domain) underlie the steering vectors; sensitivity to this seed set is not analyzed.
- **Degradation where tools are needed.** Suppression hurts Time/Intention accuracy — the method cannot selectively distinguish necessary from unnecessary calls; it is a blunt on/off knob.
- **Unresolved mechanism.** Why causal control persists despite diffuse bimodal geometry is left open, and multi-turn dependency structure prevents direct reconciliation with the probing analysis.
- **Workshop-length depth.** Some analyses (70B alternative extractors, cross-format transfer) are single-model and would benefit from broader replication; no human evaluation of trajectory quality or tool-call correctness.
- **No comparison to output-level mitigation baselines** (e.g., Self-DC / SMART-cal) on the same metrics — the paper positions against representation work but does not benchmark practical tool-overuse methods head-to-head.

## Relevance

对关注大模型工具调用可控性、可解释性及"工具过度调用"缓解的研究方向高度相关：它提供了无需重训练的推理时干预手段，并揭示了工具决策在激活空间中的非参数几何特征，可作为工具使用策略与表示工程交叉研究的参考基线。

## Questions for Further Exploration

- Can the steering vector be made **conditional** so it suppresses only redundant calls while preserving necessary ones (closing the Time/Intention accuracy gap)?
- Does the diffuse bimodal geometry generalize to function-call / JSON-style tool-use formats, or is it partly an artifact of the heading-anchor schema?
- How sensitive are the extracted vectors to the 20-example-per-domain GPT-4o seed set — what is the variance across resamples and generators?
- Why does the linearly most-decodable direction (probe AUROC 1.000) fail causally on 70B — is this a general property of larger models or specific to non-parametric concepts?
- Could the consistent normalized-depth 0.45–0.60 elbow be used as a model-agnostic heuristic for locating other context-driven decision representations?
