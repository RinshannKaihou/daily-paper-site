> **一句话总结：** OScaR 通过识别 Token 范数失衡（TNI）作为逐通道量化瓶颈，采用哈达玛 Canalized Rotation + Omni-Token Scaling 两步方案，在 INT2 量化下实现近乎无损的 KV 缓存压缩，适用于文本、多模态及全模态 LLM。

> **TL;DR:** OScaR identifies Token Norm Imbalance (TNI) as the root cause of per-channel KV cache quantization degradation under extreme compression, and proposes a lightweight two-step fix (Canalized Rotation + Omni-Token Scaling) achieving near-lossless INT2 quantization across text-only, multi-modal, and omni-modal LLMs with up to 3.0x decoding speedup and 5.3x memory reduction.

## 核心贡献

- **揭示 Token 范数失衡（TNI）是逐通道量化在极端压缩下的根本瓶颈**：通过经验观察、理论推导和定量误差分析，证明 TNI 系统性放大了逐通道量化误差。
- **提出 OScaR 框架**：基于奥卡姆剃刀原则，仅用两个相互依存的组件（Canalized Rotation + Omni-Token Scaling）替代复杂的量化流水线，无需训练。
- **跨 X-LLM 验证**：在文本 LLM、多模态 LLM（LLaVA、Qwen3-VL）和全模态 LLM（Qwen3-Omni）上均实现 INT2 近乎无损性能。
- **高效 CUDA 实现**：相比 BF16 FlashDecoding-v2 基线，实现最高 3.0x 解码加速、5.3x 内存缩减和 4.1x 吞吐量提升。
- **定义新的准确率-效率 Pareto 前沿**：理论复杂度分析表明 OScaR 的解码开销仅为 TurboQuant+ 的一半，同时准确率更高。

## Key Contributions

- **Identifies Token Norm Imbalance (TNI) as the fundamental bottleneck** of per-channel KV cache quantization under extreme compression, supported by empirical visualizations, theoretical MSE lower bounds, and quantitative error analysis across text-only and multi-modal LLMs.
- **Proposes OScaR (Omni-Scaled Canalized Rotation)**, a training-free, lightweight framework that applies Canalized Rotation (Hadamard transform) to prevent Scaling-Induced Outlier Artifacts, followed by Omni-Token Scaling (L2 normalization) to uniformly balance token norms.
- **Demonstrates near-lossless INT2 quantization** across diverse model families (Llama-3.1-8B, Qwen3-8B, LLaVA-v1.6-7B, Qwen3-VL-4B/8B, Qwen3-Omni-30B-A3B) and benchmarks (LongBench-E, NIAH, OCRBench, DocVQA, MMAU-Pro).
- **Provides optimized CUDA kernels** with fused Hadamard-norm preprocessing and residual-aware attention, achieving up to 3.0x decoding speedup, 5.3x memory reduction, and 4.1x throughput gain over BF16 FlashDecoding-v2.

## Methodology

OScaR advances the per-channel Key quantization paradigm by addressing TNI through two mutually essential steps:

1. **Canalized Rotation**: Applies an online Fast Hadamard Transform (FHT) to both Query and Key tensors before quantization. This redistributes outlier channel energy across all dimensions, preventing the Scaling-Induced Outlier Artifact that would otherwise corrupt subsequent token normalization. For Value tensors, an offline Hadamard transform is fused into the weight matrices (W_V and W_O), adding zero runtime cost.

2. **Omni-Token Scaling**: After Canalized Rotation smooths per-channel distributions, each Key token is normalized to unit L2 norm (using hardware-accelerated `rsqrt`). The original norms are stored as auxiliary metadata. During attention computation, inverse scaling restores magnitudes via `(dequant(K_unit)) * norm_k`. This eliminates TNI-induced variance without introducing artificial outliers.

The system design builds on BitDecoding (2-bit quantization backbone) and HadaCore (Tensor-Core FHT), with three fused CUDA kernels: (i) online FHT + scaling, (ii) quantization, and (iii) dequantization + de-scaling + attention. A 128-token residual window buffers incoming tokens before block-wise quantization.

Key theoretical insight: for per-channel block-wise quantization, the reconstruction MSE is lower-bounded by `(‖k_max‖₂ − ‖k_min‖₂)² / (12(2^b−1)²)`, proving TNI amplifies error proportionally to within-block norm range.

## Results & Highlights

- **LongBench-E (text-only)**: OScaR achieves 48.74% average on Qwen3-8B under INT2, outperforming the second-best method (OTT, 48.21%) and incurring only 0.82% degradation vs. BF16 (49.56%).
- **Needle-in-a-Haystack**: 96.5% retrieval accuracy, surpassing the BF16 baseline (96.0%) and the second-best quantized method (92.7%).
- **OCRBench (multi-modal)**: On Qwen3-VL-8B, OScaR scores 85.6% vs. 85.8% BF16 baseline — near-lossless. On Qwen3-VL-4B, it surpasses the best competing 2-bit method by 2.5 percentage points.
- **DocVQA (multi-modal)**: On Qwen3-VL-8B, OScaR (95.01%) slightly exceeds the BF16 baseline (94.93%).
- **MMAU-Pro (omni-modal)**: OScaR surpasses BF16 on all three metrics — open-ended QA (+1.2pp), Good Rate (+2.0pp), and audio instruction following (+1.1pp).
- **Efficiency**: At 128K context on a single H20 GPU, OScaR achieves 3.0x decoding speedup (30.9 vs 92.9 ms/tok), 5.3x memory reduction, and 4.1x throughput improvement (1354 vs 331 tok/s at batch 48).
- **Ablation**: Canalized Rotation or Omni-Token Scaling alone is insufficient; only their combination recovers accuracy from the severely degraded INT2 baseline (e.g., 42.77% vs 22.64% on WorldSense for Qwen2.5-Omni-3B).

## Critical Assessment

- **Novelty**: The TNI identification is a meaningful contribution — it provides a principled explanation for why per-channel quantization degrades under extreme compression, going beyond the channel-wise outlier narrative. The two-step Canalized Rotation + Omni-Token Scaling solution is elegant, though the Hadamard rotation component builds directly on QuaRot.
- **Rigor**: Theoretical MSE lower bounds (Appendix G) formally link TNI to quantization error. The Scaling-Induced Outlier Artifact analysis (Appendix I) is well-motivated. Ablation studies confirm both components are necessary.
- **Reproducibility**: Code is public. The framework uses only stdlib operations (Hadamard, L2 norm). CUDA kernel details are thoroughly documented in Appendix L.
- **Significance**: Achieving near-lossless INT2 KV cache quantization across text, multi-modal, and omni-modal LLMs is a strong result. The efficiency gains (3x speedup, 5.3x memory reduction) are practically meaningful.
- **Red flags**: The paper compares against TurboQuant+ (a third-party implementation) rather than the original TurboQuant, since the latter has no official code release. Some baselines (QuaRot, RotateKV) use different quantization granularity, making head-to-head comparison nuanced. The NIAH result slightly exceeding BF16 baseline (96.5% vs 96.0%) is unusual and not fully explained.

## Strengths

- Principled identification of TNI as the structural bottleneck with theoretical backing.
- Remarkably simple and lightweight framework — only Hadamard transform + L2 normalization.
- Comprehensive evaluation spanning text-only, multi-modal, and omni-modal LLMs with 6+ benchmarks.
- Strong practical gains: near-lossless INT2 quantization with significant decoding speedup.
- Public code release and detailed CUDA kernel documentation.

## Limitations

- The online Hadamard transform on Keys cannot be fused with weight matrices due to RoPE, introducing runtime overhead (decode cost ~1.5x KIVI).
- Current experiments focus on models with LLM backbones; application to streaming vision models, visual autoregressive models, and diffusion LLMs is left to future work.
- The residual window (128 tokens) adds memory management complexity; the interaction with very short sequences is not analyzed.
- Omni-Token Scaling requires storing per-token norm metadata, adding minor storage overhead.

## Relevance

OScaR 提出的 TNI 分析和轻量级 INT2 KV 缓存量化方案对长上下文推理和多模态部署有直接实用价值，其 CUDA 优化实现可即插即用于现有推理框架。

## Questions for Further Exploration

- How does OScaR interact with KV cache eviction/pruning strategies (e.g., H2O, SnapKV) — can they be combined for further memory savings?
- What is the performance of OScaR on models with non-standard attention architectures (e.g., Grouped Query Attention with very few KV heads, or Mamba-Transformer hybrids)?
- Can the Canalized Rotation principle be extended to weight quantization or activation quantization in addition to KV cache compression?
- How does OScaR perform under mixed-precision settings (e.g., INT4 for early layers, INT2 for later layers)?
- What is the sensitivity of OScaR to the residual block size R and quantization group size G across different model architectures?
