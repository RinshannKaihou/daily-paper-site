LLM Benchmark Datasets Should Be Contamination-Resistant

Ali Al-Lawati, Jason Lucas, Dongwon Lee, Suhang Wang
The Pennsylvania State University
arXiv PDF Markdown
Key Finding

Releasing benchmarks as KV caches + penultimate-layer hidden states (Contamination-Resistant Datasets) exploits the structural asymmetry between Transformer training and inference—making benchmarks unlearnable during pretraining while remaining fully functional for evaluation. Storage compresses from ~50 GB to ~350 MB via PyramidKV.

Why This Matters

Benchmark contamination is one of the most pressing issues in LLM evaluation. GPT-3 flagged >90% of evaluation samples as contaminated; Llama 2 showed >16% on MMLU. This paper shifts the paradigm from reactive (detect and decontaminate) to proactive (release data in an unlearnable format from the start). If feasible, CRDs could fundamentally alter how benchmarks are published and consumed.

Critical Edge
  • No end-to-end empirical validation. The full CRD pipeline (curation → discovery → evaluation) has not been implemented or tested on real benchmarks and models.
  • Irreversibility is not guaranteed. KV inversion attacks succeed on MHA architectures; GQA resistance may erode as attacks improve.
  • Equivalence assumed but unverified. Whether translated KV caches preserve evaluation properties across diverse model families remains an open empirical question.
  • Limited to Transformers. Does not extend to non-Transformer architectures (e.g., state-space models like Mamba).
  • Incompatible with key benchmark types. Multi-turn dialogue, agentic, and adaptive evaluations cannot be adapted to CRDs.
  • Adversarial circumvention risk. Model developers could train specialized layers to learn from KV-cache-format data if the format becomes standardized.
Methodology

CRD Definition

A dataset φ(D) is contamination-resistant w.r.t. model M if M(φ(D)) yields valid task performance but training on φ(D) fails to improve generalization. Three required properties: Irreversibility, Equivalence, Interoperability.

Transformer Asymmetry

Training computes NLL loss over all positions (needs every hidden state). Inference generates autoregressively using only cached KV pairs and the final token’s penultimate hidden state. CRD releases only the inference-sufficient components—not the raw token sequence.

Interoperability Paradigms

Near-term: Anchor-model subspace alignment via Cross-LoRA, Trans-LoRA, LoRA-X (rank-truncated SVD + Frobenius-optimal linear transform). Long-term: Model-agnostic relative representations projecting all models into a shared coordinate frame (requires only k=100–500 anchor samples).

CRD Properties
Irreversibility
Computationally infeasible to reconstruct plaintext from KV cache. GQA resists inversion better than MHA.
Equivalence
Model output on CRD approximates plaintext output. Verified via paired evaluations, calibration checks, adaptive temperature scaling.
Interoperability
CRD can be translated across architectures. Architectural similarity (shared GQA, SwiGLU, RMSNorm) improves transfer.
Release Format
{KVl1:t}l=1..L, h(L-1)t, Y
Strengths
  • Elegant train/inference asymmetry argument

  • Two practical interoperability paths

Limitations
  • No empirical end-to-end validation
  • Irreversibility not proven
  • Equivalence unverified
  • Transformer-only scope
  • Significant deployment complexity

Questions for Further Exploration
  1. What is the actual equivalence gap when using KV-cache-based CRDs vs. plaintext across diverse model families?
  2. How robust is irreversibility against future inversion attacks targeting GQA and multi-query attention?
  3. Can relative representations achieve sufficient equivalence for fine-grained discrimination between similarly-performing models?
  4. What is the computational cost of the anchor-to-target translation step for state-of-the-art billion-parameter models?
  5. How would CRDs interact with quantized, pruned, or distilled model variants?
  6. Could a hybrid of CRDs with dynamic benchmarking provide both contamination resistance and temporal freshness?
  7. What governance structures are needed to maintain anchor models and keep the CRD ecosystem trustworthy?
