> **一句话总结：** TORQ 提出了一种无需训练的两级正交旋转 PTQ 框架，利用 Schur-Horn 定理实现块间方差均衡、最大熵引导实现块内码本对齐，在 Qwen3-32B 上将 MXFP4 平均精度从 RTN 的 38.40% 提升至 73.63%（BF16 为 74.82%），量化校准速度比 FlatQuant 快约 10 倍。

> **TL;DR:** TORQ is a training-free PTQ framework for MXFP4 activation quantization that decomposes orthogonal rotation into inter-block variance equalization (via Schur-Horn theorem + Givens rotations, proven optimal under convex error models) and intra-block codebook alignment (via maximum-entropy alternating optimization with exact piecewise-constant angle search). On Qwen3-32B, it achieves 73.63% average accuracy (vs. 74.82% BF16) and WikiText PPL 8.43 (vs. 7.61 BF16), calibrates in 1920s (10x faster than FlatQuant), and generalizes to MXINT4/NVFP4 formats and MoE architectures.

## 核心贡献

- **首次系统揭示 MXFP4 激活量化的两大结构性瓶颈**：块间方差失衡（少数高方差块推高共享指数，导致 90%+ 小幅值元素被归零）和块内码本利用失衡（重尾分布导致约 50% 码字范围仅被 10% 数据激活）。
- **提出两级正交旋转框架 TORQ**：将 R 分解为 R_inter ⊗ R_intra（Kronecker 积），分别从宏观和微观层面优化，且两者数学正交、互不干扰。
- **块间旋转基于 Schur-Horn 定理**：理论证明方差均衡配置是凸误差模型下的全局最优解，通过迭代 Givens 旋转在 O(B²) 时间内构造正交矩阵。
- **块内旋转基于码本占用均衡损失**：提出 L_code 度量码字利用不均衡度，通过交替优化 S-step（更新缩放因子）和 R-step（精确枚举搜索最优 Givens 角）最大化 MXFP4 码本信息容量。
- **零推理开销**：逆变换通过 Kronecker 积融合到相邻线性层权重 W' = W(R_intra^T ⊗ R_inter)，在线推理无额外计算。
- **显著的效率优势**：Qwen3-32B 校准仅需 1920 秒，比 FlatQuant（18619s）快约 10 倍，比 OSTQuant（14834s）快约 7.7 倍。

## Key Contributions

- First systematic identification and formalization of two structural challenges in MXFP4: inter-block variance imbalance and intra-block codebook utilization imbalance (codebook collapse).
- Proposes TORQ, the first dual-level rotation framework explicitly designed for block floating-point structures, decomposing R = R_inter ⊗ R_intra for decoupled control of energy and distribution.
- Proves that variance equalization via Schur-Horn theorem is globally optimal under convex error models (Theorem 4.1), with constructive Givens rotation achieving O(B²) convergence.
- Introduces codebook occupancy equilibrium loss L_code and an alternating S-step/R-step optimization with exact piecewise-constant angle search (no gradient descent needed).
- Fuses inverse rotation into linear layer weights (W' = W(R_intra^T ⊗ R_inter)), achieving zero online inference overhead.
- Achieves 10x faster calibration than FlatQuant and generalizes to MXINT4, NVFP4, and MoE architectures.

## Methodology

TORQ targets MXFP4 (e2m1) activation quantization where each block of K=32 elements shares an exponent s_b = 2^{floor(log2(max|z_b|))} and values are projected to the discrete codebook C = {±0, ±0.5, ±1, ±1.5, ±2, ±3, ±4, ±6}. The framework decomposes the global orthogonal optimization min_{R∈O(d)} E[||XR - Q(XR)||²] into two structured sub-problems:

**Inter-block Rotation (R_inter):** Operates across B blocks at each position k. Based on the Schur-Horn theorem, it constructs an orthogonal matrix such that diag(RΣR^T) = (tr(Σ)/B)·1, equalizing variance across all blocks. This is implemented via an iterative Givens rotation algorithm (Algorithm 1) that selects block pairs with opposite variance deviations and computes optimal rotation angles. Theoretical proof (Theorem 4.1) establishes that under convex error models, this variance-equalized configuration minimizes global MSE.

**Intra-block Rotation (R_intra):** Operates within each block of K elements in the normalized FP4 space. It optimizes a codebook occupancy equilibrium loss L_code that measures deviation from uniform codeword utilization. The optimization alternates between: (1) S-step: updating block scaling factors by mapping max activation to the maximum MXFP4 codeword; (2) R-step: applying Givens rotations to column pairs selected by an imbalance + complementarity heuristic, with optimal angles found via exact finite enumeration over piecewise-constant loss landscapes.

**Weight Fusion:** The inverse transformation R^T is fused offline into the subsequent linear layer: W' = W(R_intra^T ⊗ R_inter), eliminating all runtime overhead. Calibration uses only 128 text segments from WikiText2.

## Results & Highlights

**Main Results (LLaMA3 + Qwen3 families):**
- LLaMA3-8B: PPL 8.57 (vs. FP16 6.14), avg accuracy 69.32% (vs. FlatQuant 68.34%, RTN 32.03%)
- LLaMA3-70B: PPL 3.49 (vs. FP16 2.87), avg accuracy 76.28% — FlatQuant OOM
- Qwen3-8B: PPL 10.26 (vs. FP16 9.00), avg accuracy 67.75% (vs. FlatQuant 66.96%)
- Qwen3-32B: PPL 8.43 (vs. BF16 7.61), avg accuracy 73.63% (vs. BF16 74.82%, RTN 38.40%)

**Ablation (Qwen3-8B):**
- R_inter only: PPL 24.73 (from 3750.93 RTN)
- R_intra only: PPL 20.36
- Full TORQ: PPL 10.26, proving orthogonality and complementarity

**Hard Tasks (Qwen3-8B):**
- GSM8K: 86.82% (vs. FP16 88.25%, GPTQ 67.37%)
- HumanEval: 61.15% (vs. FP16 64.63%, GPTQ 30.84%)
- MMLU: 70.82% (vs. FP16 74.71%)

**Efficiency:**
- Qwen3-30B-A3B: 3.5x decoding speedup (220 vs 63 Toks/s), memory 15.3GB vs 56.9GB
- Calibration: Qwen3-32B in 1920s (FlatQuant: 18619s, OSTQuant: 14834s)

**Generalization:**
- NVFP4 on Qwen3-8B: GSM8K 87.08% (vs. RTN 42.67%)
- MoE (Qwen3-30B-A3B): GSM8K 83.83% (vs. GPTQ 37.89%, QuaRot 74.18%)

## Critical Assessment

- **Novelty (High):** The two-level decomposition specifically tailored to BFP structure is genuinely novel. Prior rotation methods (QuaRot, FlatQuant, OSTQuant) optimize for integer quantization's uniform grid — TORQ correctly identifies that MXFP4's geometric progression codebook requires fundamentally different objectives (codebook utilization vs. outlier suppression). The Schur-Horn optimality proof is elegant and the exact piecewise-constant angle search is a clean contribution.
- **Rigor (High):** Full theoretical proofs in appendices (convexity analysis, optimality, error bounds). Ablation study cleanly validates each component. Calibration robustness tested across 3 datasets with PPL variance < 0.8.
- **Reproducibility (Medium-High):** Method is training-free, uses standard calibration (128 WikiText2 segments), and experiments run on commercially available RTX 5090 GPUs. No code repository mentioned, but the algorithm is described in sufficient detail. The MXFP4 hardware dependency may limit some researchers.
- **Significance (High):** MXFP4 is natively supported by NVIDIA Blackwell and AMD Ryzen AI — this is not a theoretical format but an actual next-gen hardware standard. Closing the gap to BF16 (73.63% vs 74.82% on Qwen3-32B) with 10x faster calibration than competitors is practically impactful.
- **Red flags / Concerns:**
  - The convex error model assumption (Theorem 4.1) is acknowledged as approximate for 4-bit regimes (Proposition 4.2). The paper admits this is a proxy, not an exact bound.
  - Comparison with FlatQuant on LLaMA3-70B shows FlatQuant OOM — this may reflect implementation choices rather than fundamental superiority.
  - Weight-only quantization results are not separated from activation quantization; W4A4 is tested jointly, making it hard to isolate activation-only gains.
  - The authors' own limitations section acknowledges the intra-block rotation only matches first-order statistics (log-magnitude expectations), missing higher-order information like kurtosis and skewness.
  - Fixed block size of 32 throughout; adaptive block sizing is unexplored.

## Strengths

- Exceptionally well-motivated: the two structural imbalances are clearly diagnosed with empirical evidence (Figures 2, 3) before proposing solutions.
- Theoretically grounded: Schur-Horn optimality proof, Karamata's inequality, full convexity analysis — rare depth for a systems/efficiency paper.
- Zero inference overhead via weight fusion is a critical practical advantage.
- 10x faster calibration than competitors makes it genuinely deployable.
- Strong generalization to MXINT4, NVFP4, and MoE architectures demonstrates format-agnostic geometric optimization.
- Comprehensive evaluation: 5 model sizes, 10+ benchmarks, ablations, calibration robustness, efficiency analysis.

## Limitations

- Intra-block rotation only optimizes first-order statistics; higher-order distributional features (kurtosis, skewness, multimodality) are not captured.
- Fixed block size (32) throughout; adaptive block sizing for different layers/tasks is unexplored.
- No specialized optimization for MoE expert heterogeneity, though basic effectiveness is demonstrated.
- Joint W4A4 evaluation makes it difficult to isolate pure activation quantization improvements.
- FlatQuant OOM on LLaMA3-70B may conflate memory efficiency with method quality.
- The convex error model is an approximation acknowledged to not hold strictly at 4-bit precision.

## Relevance

TORQ 直接解决了大模型 4-bit 浮点量化部署这一核心工程问题。MXFP4 是 NVIDIA Blackwell 等下一代硬件的原生支持格式，本文提出的无需训练、零推理开销、校准速度极快的方案对实际生产部署具有很高价值，特别适合对推理速度和精度同时有严格要求的场景。其几何优化范式（结构感知的正交变换）也为其他块浮点格式的量化提供了新思路。

## Questions for Further Exploration

- How does TORQ perform with mixed-precision configurations (e.g., MXFP4 for attention layers, FP8 for MLP layers)?
- Can the intra-block rotation be enhanced with higher-order statistics (kurtosis, skewness) for extremely asymmetric or multimodal activation distributions?
- What is the impact of adaptive block sizes on quantization accuracy and hardware efficiency?
- How does TORQ scale to ultra-large MoE models (1000+ experts) — does cross-expert activation heterogeneity require specialized rotation strategies?
- Can the geometric optimization paradigm be applied to weight quantization (currently only activation quantization is addressed)?
- What are the numerical stability properties of the fused weight matrices W' = W(R_intra^T ⊗ R_inter) for very deep models?
