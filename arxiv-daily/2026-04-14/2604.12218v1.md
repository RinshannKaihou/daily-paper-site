# LLM-Enhanced Log Anomaly Detection: A Comprehensive Benchmark of Large Language Models for Automated System Diagnostics

> **一句话总结：** 该论文在HDFS、BGL、Thunderbird和Spirit四个公开数据集上系统对比了传统日志解析+机器学习、微调Transformer（BERT/RoBERTa/DeBERTa）和提示工程LLM（GPT-3.5/GPT-4/LLaMA-3）的日志异常检测性能，发现微调Transformer精度最高（F1 0.96–0.99），但提示工程LLM在零样本场景下表现强劲（F1 0.82–0.91），并提出了结构化日志上下文提示（SLCP）策略提升LLM零样本性能2.9–3.1个百分点。

> **TL;DR:** A comprehensive benchmark across HDFS, BGL, Thunderbird, and Spirit datasets compares traditional parser+ML pipelines, fine-tuned transformers (BERT/RoBERTa/DeBERTa), and prompt-based LLMs (GPT-3.5/GPT-4/LLaMA-3) for log anomaly detection. Fine-tuned transformers achieve the highest accuracy (F1 0.96–0.99), while prompt-based LLMs show strong zero-shot capabilities (F1 0.82–0.91). The proposed Structured Log Context Prompting (SLCP) improves LLM zero-shot performance by 2.9–3.1 percentage points.

## 核心贡献

- 建立了覆盖四个公开日志数据集的标准化评估框架，实现传统方法、微调Transformer和提示工程LLM的公平对比
- 提出结构化日志上下文提示（SLCP）技术，通过系统上下文、时间上下文、语义标记和分类指令提升LLM零样本检测性能
- 从准确率、成本、延迟和标注效率四个维度提供可落地的部署指南

## Key Contributions

- Established a standardized evaluation framework across four public log datasets enabling fair comparison across traditional, fine-tuned transformer, and prompt-based LLM approaches
- Proposed Structured Log Context Prompting (SLCP), improving LLM zero-shot log anomaly detection through system context, temporal context, semantic markers, and explicit classification instructions
- Provided actionable deployment guidelines based on accuracy, cost, latency, and label availability trade-offs

## Methodology

**Datasets:** HDFS (11.2M logs, 2.93% anomaly), BGL (4.7M logs, 7.41% anomaly), Thunderbird, and Spirit from LogHub.

**Traditional Pipeline:** Three log parsers (Drain, Spell, AEL) combined with four classifiers (Logistic Regression, Random Forest, SVM, Isolation Forest). Sequence-based features (count vectors for HDFS; sliding windows w=20, s=1 for message-level datasets).

**Fine-Tuned Transformers:** BERT-base (110M), RoBERTa-base (125M), and DeBERTa-v3-base (184M) fine-tuned directly on raw log messages with lr=2e-5, batch size 32, 5 epochs with early stopping.

**Prompt-Based LLMs:** GPT-3.5-Turbo, GPT-4, and locally-deployed LLaMA-3-8B evaluated in zero-shot and few-shot (k=1,3,5) settings. SLCP augments prompts with system description, timestamp/event frequency statistics, semantic anomaly markers ("error", "fatal", "exception"), and explicit output formatting.

## Results & Highlights

- **Fine-tuned transformers highest accuracy:** DeBERTa-v3 achieves F1 95.3–98.9% across all datasets
- **LLMs strong zero-shot performance:** GPT-4 reaches F1 81.2–88.3% without any training data, outperforming some traditional supervised methods
- **SLCP effectiveness:** GPT-4 + SLCP zero-shot improves F1 by 2.9–3.1 pp; GPT-4 + SLCP 5-shot reaches 87.9–93.8%, approaching traditional supervised methods
- **Traditional methods competitive:** Drain + RF achieves F1 86.4–95.1% with near-zero latency (0.3 ms/prediction)
- **Cost-latency trade-off:** GPT-4 + SLCP costs $8.40/1K predictions with 890 ms latency; LLaMA-3 local deployment offers cost-free inference at 156 ms; DeBERTa at 12.4 ms
- **Label efficiency:** With only 1% labels, GPT-4 + SLCP achieves 89.1% F1 on HDFS vs. 71.3% for Drain+RF and 82.4% for fine-tuned DeBERTa
- **Failure modes:** Traditional methods fail on novel templates (32%) and subtle sequence anomalies (45%); transformers struggle with extremely long sequences exceeding context windows (68%); LLMs show temperature-dependent inconsistency and difficulty with numerical anomalies

## Critical Assessment

- **Novelty:** The comprehensive apples-to-apples comparison across three paradigms on four datasets is valuable given the fragmented prior literature. SLCP is a modest but practical prompting innovation.
- **Experimental rigor:** Standardized preprocessing, consistent metrics, 5 random seeds, temperature=0 for LLMs, and ablation studies strengthen validity.
- **Significance:** The cost-accuracy-latency trade-off analysis and practical deployment guidelines are highly actionable for practitioners in AIOps and system reliability.

## Strengths

- Comprehensive multi-paradigm, multi-dataset benchmark with standardized conditions
- Practical analysis extending beyond accuracy to cost, latency, and label efficiency
- Novel SLCP prompting strategy with validated ablation
- Clear, actionable deployment guidelines for practitioners

## Limitations

- Evaluates only English-language logs; multilingual log analysis is unexplored
- Cost analysis depends on current API pricing, which may change
- Public datasets may not fully capture production system complexity
- Focuses only on binary anomaly detection; multi-class categorization is future work
- Single-author paper from a non-R1 institution; domain expertise depth is unclear
