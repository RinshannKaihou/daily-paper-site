# TEMPLATEFUZZ: Fine-Grained Chat Template Fuzzing for Jailbreaking and Red Teaming LLMs

> **一句话总结：** 本文提出了首个针对大语言模型对话模板（chat template）的细粒度模糊测试框架TemplateFuzz，通过五种元素级变异规则、启发式搜索策略和主动学习驱动的评估器，在12个开源LLM上实现了平均98.2%的攻击成功率（Top-5 ASR）且仅造成1.1%的精度下降，并在5个商业LLM上通过提示注入达到了80%-100%的攻击成功率，揭示了对话模板作为被忽视攻击面的严重系统性风险。

> **TL;DR:** This paper introduces TemplateFuzz, the first fine-grained fuzzing framework targeting LLM chat templates as an attack surface for jailbreaking. Using five element-level mutation rules, a heuristic search strategy, and an active learning–based evaluator, it achieves an average 98.2% Top-5 ASR on 12 open-source LLMs with only 1.1% accuracy degradation, and 80%-100% ASR on 5 commercial LLMs via prompt injection, revealing severe systemic risks in chat template handling.

## Key Contributions
- Presents TemplateFuzz, the first chat template-based fuzzing framework that systematically generates effective jailbreak templates and uncovers structural vulnerabilities in LLM chat template handling
- Designs five fine-grained mutation rules targeting essential chat template elements (system messages, user/assistant messages, role markers, delimiters, generation hints), preserving inference capability while vastly enlarging the mutation search space
- Introduces a heuristic search strategy (Adaptive MCTS-Explore + Roulette-Wheel selection) that balances attack success rate and model accuracy, plus an active learning–based pipeline yielding a lightweight rule-based oracle for scalable jailbreak evaluation
- Conducts large-scale evaluation on 12 open-source and 5 commercial LLMs, demonstrating broad systemic vulnerabilities in chat template handling, and releases source code and evaluation artifacts

## Methodology
TemplateFuzz treats the default chat template as a seed and applies fuzzing to discover adversarial variants. The framework has three core components:
1. **Chat Template Mutation:** Five element-level mutation rules (M1–M5) target system messages, user/assistant messages, role markers, delimiters, and generation hints. Few-shot LLM prompting generates diverse replacement candidates for each element. First-order and high-order (compositional) mutations are both supported.
2. **Heuristic Search Strategy:** An Adaptive MCTS-Explore strategy selects seed templates, using selective expansion (only adding high-ASR, low-accuracy-degradation nodes) and periodic pruning to control tree growth. A Roulette-Wheel selection mechanism, informed by a sampling learning phase and an enhanced UCB score combining jailbreak success, accuracy preservation, and rarity, dynamically prioritizes mutation rules.
3. **Learning-based Judge:** An active learning strategy refines a lightweight rule-based oracle by identifying disagreement cases between the initial rule-based evaluator and a heavy model-based judge, then manually extracting new semantic patterns. The refined oracle achieves >90% agreement with the judge model while being thousands of times faster.

## Results & Highlights
- **Open-source LLMs (12 models, 4B–70B):** TemplateFuzz achieves an average Top-1 ASR of 90.5% and Top-5 ASR of 98.2%, outperforming ChatBug, GPTFuzzer, and TurboFuzzLLM by 9.1%–47.9% in Top-5 ASR. It reaches 100% Top-5 ASR on all six small-scale models tested.
- **Model Accuracy:** Average accuracy degradation is only 1.1%, compared to over 9.5% for ChatBug. On Llama2-7B, TemplateFuzz actually improves accuracy by 0.21%.
- **Single-question Jailbreak:** Under a query budget of 50, TemplateFuzz jailbreaks 45/50 questions with an average of 9.31 queries (AQJ); under a budget of 100, it jailbreaks 47/50 with 12.89 AQJ, far exceeding GPTFuzzer and TurboFuzzLLM in both success count and efficiency.
- **Commercial LLMs (5 models):** Via chat template-based prompt injection (without direct template access), TemplateFuzz achieves Top-5 ASR of 80%–100% (average ~90%), including 100% on Qwen-Plus and DeepSeek-Reasoner.
- **Ablation Studies:** Each mutation rule contributes individually (single-rule Top-1 ASR 27%–60%), but their combination produces synergistic effects (95.58% Top-1 ASR). The full heuristic strategy outperforms random selection (74.04%), genetic algorithm (83.27%), and no-sampling variants (87.88%). The active learning–based judge achieves 88.27% accuracy and 92.69% TPR, nearly matching the heavy judge model (89.42%) while evaluating in <1s versus 1,425s.

## Critical Assessment
TemplateFuzz makes a strong case that chat templates constitute a critical, underexplored attack surface for LLM jailbreaking. The paper’s contribution is both novel and practically important: instead of crafting ever-longer adversarial prompts, it shows that manipulating the structural formatting of conversations can break safety alignment with far fewer tokens and higher success rates. The experimental scope is impressive (17 models spanning open and commercial systems), and the ablation studies are thorough. However, the "why" behind chat template vulnerabilities remains somewhat under-explored mechanistically—the paper describes what mutations work but offers limited theoretical analysis of how specific template perturbations subvert safety training or attention mechanisms. Additionally, the prompt-injection results on commercial models, while striking, are preliminary (limited API budget, no sampling learning phase) and raise questions about how robustly these attacks transfer in fully closed systems.

## Strengths
- Novel attack surface: moves beyond prompt engineering to systematically exploit chat templates as a structural vulnerability
- Strong empirical results: 98.2% average Top-5 ASR on open-source models with minimal accuracy degradation
- Excellent efficiency: far fewer tokens (~25 vs ~200 for prompt-based attacks) and lower query cost than competing fuzzers
- Comprehensive evaluation across model families, sizes, and both open-source and commercial systems
- Well-designed ablations demonstrating the contribution of each core component
- Responsible disclosure: authors notified affected vendors and delayed publication for remediation

## Limitations
- Mechanistic understanding is limited; the paper does not deeply analyze why specific template elements are vulnerable or how safety training fails under template mutation
- Commercial LLM experiments are preliminary due to API cost constraints (50 iterations, no sampling learning, 100-question subset)
- Defense evaluations are discussed conceptually but not empirically validated
- The active learning judge still relies on manual annotation of disagreement cases, which may not fully eliminate evaluator bias
- It is unclear how quickly model vendors can patch these vulnerabilities or whether template diversification defenses will generalize

本文与LLM安全性、红队测试（red teaming）和对抗鲁棒性研究高度相关。它直接挑战了当前以提示（prompt）为中心的越狱攻击范式，揭示了对话模板——一个支撑系统级提示工程的核心机制——本身就可以成为高效、低资源消耗的越狱通道。对于任何关注（1）LLM对齐与安全机制的可绕过性、（2）系统级提示注入风险、以及（3）自动化红队测试方法的读者来说，这项工作都具有重要的参考价值。此外，TemplateFuzz在保持模型精度的同时实现极高攻击成功率的能力，使其成为评估和加固对话模型安全边界的实用工具。论文提出的防御方向（训练时模板多样化、部署时API限制与注入检测）也为后续的安全研究提供了明确的议程。

## Questions for Further Exploration

- Do the discovered adversarial templates transfer across different fine-tuned versions or safety-tuned variants of the same base model?
