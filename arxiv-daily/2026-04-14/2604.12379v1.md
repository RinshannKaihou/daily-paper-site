# Beyond Output Correctness: Benchmarking and Evaluating Large Language Model Reasoning in Coding Tasks

> **一句话总结：** 本文提出了首个跨代码生成、摘要与分类任务的推理质量评估基准CodeRQ-Bench，并通过分析1,069个现有评估器的误判案例总结出四条设计洞见，进而设计出结合证据验证与歧义感知的双阶段评估器VERA，在四个数据集上显著超越现有基线。

> **TL;DR:** This paper introduces CodeRQ-Bench, the first benchmark for evaluating LLM reasoning quality across code generation, summarization, and classification. By analyzing 1,069 mismatch cases from existing evaluators, the authors derive four design insights and propose VERA—a two-stage evaluator combining evidence-grounded verification with ambiguity-aware score correction—achieving consistent gains over strong baselines across four datasets.

## 核心贡献
- 提出了CodeRQ-Bench，首个覆盖代码生成、代码摘要和缺陷检测三大任务类别的LLM推理质量评估基准，包含732条经三位专家共识标注的推理轨迹
- 系统分析了1,069个现有评估器的误判案例，归纳出五大局限性（无证据支撑评估、歧义识别缺失、分数聚合失效、自生成参考偏差、代码语义盲区），并提炼出四条针对代码推理评估的设计洞见
- 设计了VERA评估器：第一阶段通过可自主调用网络搜索的LLM Judge进行证据验证式推理评估，第二阶段引入歧义感知分数修正，最终在各数据集上全面超越现有方法

## Key Contributions
- Introduces CodeRQ-Bench, the first benchmark for evaluating LLM-generated reasoning quality across three coding task categories (generation, summarization, classification), comprising 732 expert-annotated reasoning traces
- Analyzes 1,069 mismatch cases from existing evaluators, identifies five recurring limitations, and distills four design insights specifically for reasoning evaluation in coding tasks
- Proposes VERA, a two-stage evaluator that combines evidence-grounded verification (via an LLM judge with autonomous web search) with ambiguity-aware score correction, achieving state-of-the-art performance across four datasets

## Methodology
The authors construct CodeRQ-Bench by augmenting two existing generation-focused datasets (CoderEval-RE and SWEbench-RE) with two newly annotated datasets: ClassEval-RE for code summarization and DebugBench-RE for bug detection. Each instance is labeled by three experts using a consensus-based workflow with near-perfect Fleiss’ κ (0.91 and 0.95). Reasoning quality is operationalized into four dimensions: Comprehension, Analysis, Conclusion, and Consistency.

To diagnose existing evaluators, the authors apply ReCEval, SocREval, and CaSE to CodeRQ-Bench, identify 1,069 mismatches, and use GPT-5.2 to generate diagnostic annotations and consolidate them into a taxonomy of five limitation categories.

Guided by the derived insights, VERA operates in two stages: (1) Verified Reasoning Evaluation—an LLM judge assesses reasoning correctness with access to a web search tool for verifying technical claims, producing a base score p; (2) Ambiguity-Aware Score Correction—a second judge call assesses task ambiguity level a and ambiguity-handling quality h, producing a penalty δ. The final score is max(p + δ, 0).

## Results & Highlights
- CodeRQ-Bench covers 732 instances across four datasets: CoderEval-RE (230, generation), SWEbench-RE (111, generation), ClassEval-RE (139, summarization), and DebugBench-RE (252, classification)
- Existing evaluators (ReCEval, SocREval, CaSE) perform poorly on coding tasks, often near random, with task-specific strengths that do not generalize across modalities
- VERA achieves the best performance across all four datasets and all metrics (Somers’ D, Spearman’s ρ, AUCROC, AUPRC), with AUCROC improvements up to 0.26 and AUPRC up to 0.21 over the strongest baselines
- On CoderEval-RE, VERA improves Somers’ D from 0.1399 to 0.3811; on SWEbench-RE, AUPRC improves from 0.2165 to 0.3058
- Ablation studies confirm both components are necessary: removing evidence verification causes the largest drop (e.g., Somers’ D on SWEbench-RE falls from 0.2799 to −0.0376), while removing ambiguity-aware correction yields smaller but consistent degradation, especially on generation tasks
- Notably, VERA uses GPT-4o-mini yet substantially outperforms GPT-4-based baselines, indicating that the evaluation design rather than model scale drives the gains

## Critical Assessment
The paper addresses a timely and important problem: moving beyond output correctness to evaluate the reasoning processes of LLMs in coding tasks. The construction of CodeRQ-Bench is methodologically sound, and the taxonomy of evaluator limitations offers actionable insights. The VERA design is elegant and well-motivated by the empirical diagnosis. However, the benchmark is still limited to relatively controlled evaluation settings and does not yet capture more realistic coding workflows such as tool-integrated problem solving, interactive debugging, or long-context repository reasoning. Additionally, while VERA uses an autonomous web search tool, the paper does not deeply analyze the reliability or latency implications of search-dependent evaluation in practice.

## Strengths
- First benchmark to broaden reasoning-quality evaluation beyond code generation to summarization and classification
- Rigorous empirical diagnosis of existing evaluator failures with a validated taxonomy and LLM-assisted analysis
- VERA is well-motivated by the identified limitations and achieves strong, consistent gains across datasets
- Uses a cost-efficient model (GPT-4o-mini) to outperform more expensive baselines, making the approach practical
- Transparent release of benchmark and detailed prompts in appendices

## Limitations
- Benchmark scope is limited to controlled tasks; does not yet cover interactive debugging, tool use, or rich repository-level reasoning
- VERA’s reliance on web search introduces potential variability in retrieval quality and latency that is not thoroughly characterized
- The annotation is limited to English-language Python tasks; generalization to other languages and programming paradigms is untested
- The paper does not explore whether alternative correction formulations (beyond the chosen δ = a · min(2h − 1, 0)) could further improve performance

## Relevance
本文与LLM推理评估、代码智能和可解释性研究高度相关。它直接针对当前评估体系仅关注最终输出正确性的缺陷，提出了一套面向代码任务的推理质量评估框架。CodeRQ-Bench填补了代码摘要和缺陷检测领域推理评估的空白，而VERA提出的"证据验证+歧义感知"双阶段范式为构建更可靠的代码推理评估器提供了可复用的设计思路。对于关注链式思维（CoT）可靠性、模型输出一致性以及软件工程领域LLM应用的研究者和实践者而言，这项工作具有重要的参考价值。

## Questions for Further Exploration
- How does VERA’s search-dependent performance vary with the quality and recency of retrieved web sources?
- Can the CodeRQ-Bench framework be extended to multi-turn interactive coding sessions or tool-integrated agent workflows?
- Would VERA’s design transfer effectively to other programming languages (e.g., Java, C++, Rust) or multi-language codebases?
- Could retrieval-augmented generation (RAG) with curated documentation replace open web search to improve reliability and reduce latency in evidence verification?
- How robust is VERA to adversarial or subtly misleading reasoning chains that appear correct on the surface?
