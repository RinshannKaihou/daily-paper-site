RePAIR: Interactive Machine Unlearning through Prompt-Aware Model Repair

一句话总结： 提出交互式机器遗忘（IMU）新范式，通过RePAIR框架让终端用户用自然语言在推理时指令LLM遗忘特定知识，核心方法STAMP以无训练、闭式伪逆更新MLP权重，在有害知识、错误信息和个人数据三大任务上实现近零遗忘分数并保留模型效用。
TL;DR: Introduces Interactive Machine Unlearning (IMU), a paradigm where end users instruct LLMs to forget targeted knowledge via natural language at inference time. The RePAIR framework uses a watchdog model for intent detection, a surgeon model for repair code generation, and STAMP—a training-free, closed-form pseudoinverse method—to redirect MLP activations toward a refusal subspace, achieving near-zero forget scores with preserved utility across harmful knowledge, misinformation, and personal data erasure tasks.

核心贡献

  • 形式化交互式机器遗忘（IMU）问题设定：用户可在推理时通过自然语言要求模型遗忘指定知识，无需模型服务提供商（MSP）介入
  • 提出STAMP及其低秩变体STAMP-LR：首个完全在测试时运行的无训练、单样本LLM遗忘方法，通过闭式伪逆更新重定向MLP激活至拒绝子空间
  • 构建端到端RePAIR框架：集成意图检测（Mwatchdog）、修复代码生成（Msurgeon）与自主参数更新（Mpatient）
  • 在有害知识抑制、错误信息纠正和个人数据擦除三项任务上验证，遗忘分数接近零且保留模型效用，速度比SOTA基线提升约3倍

Key Contributions

  • Formalizes Interactive Machine Unlearning (IMU): enables end users to request knowledge removal from LLMs via natural language during inference, eliminating dependency on model service providers
  • Proposes STAMP and STAMP-LR: the first training-free, single-sample unlearning methods for LLMs operating entirely at test time, using closed-form pseudoinverse updates to steer MLP activations toward a refusal subspace
  • Introduces the RePAIR framework: an end-to-end solution integrating intent detection, repair code generation, and autonomous model repair
  • Demonstrates near-oracle forgetting with preserved model utility across three tasks, outperforming six state-of-the-art baselines with up to ~3× speedup

Methodology

  • Model: Llama-3-8B as Mpatient; Mistral-7B as Mwatchdog for intent classification and forget-pair extraction; Qwen2.5-Coder-7B-Instruct as Msurgeon for repair code generation
  • STAMP: Extracts MLP activations for forget, retain, and reference (refusal prompt) sets. Computes a steering vector from forget to refusal activations. Solves for new weights via Moore-Penrose pseudoinverse: $W_{new} = X^+ O'$, where $O'$ redirects forget activations toward refusal while keeping retain activations unchanged. Complexity: $O(d^3)$.
  • STAMP-LR: Approximates activation matrix $X \approx AB$ with rank $r \ll d$, reducing complexity to $O(r^3 + r^2 \cdot d)$ and memory to $2rd$, enabling efficient on-device execution.
  • Constraints: Training-free (no gradients/backprop) and single-sample (operates on one forget pair at a time).
  • Evaluation: Three tasks—(1) harmful knowledge suppression on WMDP-Bio, (2) misinformation removal on corrupted MMLU, (3) personal data erasure on synthetic biographies. Metrics: forget accuracy ($Acc_f$), retain accuracy ($Acc_r$), ROUGE-L ($F$-$RL$, $R$-$RL$), utility (perplexity on TinyStories), and runtime efficiency (RTE).

Results & Highlights

  • Forgetting effectiveness: STAMP and STAMP-LR achieve $Acc_f = 0.00$ and $F$-$RL = 0.00$ across all tasks, comparable to oracle-level forgetting
  • Retention & utility: STAMP-LR reaches up to $Acc_r = 84.47$ (misinformation) and $R$-$RL = 0.88$ (personal data), with perplexity stable at ~6–8 (comparable to Oracle ~5–6), while GA-based methods degrade to 10–12
  • Runtime efficiency: STAMP-LR completes in 4.25 min (harmful), 2.57 min (misinformation), and 4.01 min (personal data)—up to ~3× faster than training-based baselines
  • Single-sample unlearning: When $|D_f| = 1$, all training-based baselines fail completely ($Acc_f = 100$), whereas STAMP and STAMP-LR still achieve $Acc_f = 0.00$ with $Acc_r > 70$
  • RePAIR pipeline: End-to-end valid code rate >96%, request detection >96%, request satisfaction >97%, IDK rate >96%
  • Ablation: Layer 7 alone achieves maximal forget/refusal separation (cosine divergence 0.867) and matches all-layer intervention with ~3.8× speedup; STAMP-LR remains effective for rank $r \geq 64$

Critical Assessment

  • Novelty: High. IMU reframes machine unlearning from a provider-centric batch process to a user-driven, inference-time interaction. STAMP is the first training-free, single-sample LLM unlearning method.
  • Experimental rigor: Strong. Comparisons against six SOTA baselines, oracle upper bound, three tasks, single-sample stress test, layer-wise and rank ablations, and retain-ratio sensitivity analysis.
  • Significance: Substantial for AI safety, privacy (GDPR/CCPA), and user autonomy. However, practical deployment depends on solving the retain-buffer requirement at inference time.

Strengths

  • Novel and well-motivated problem formulation (IMU) aligned with regulatory rights to erasure
  • Training-free, closed-form method avoids costly retraining and gradient computation
  • Strong empirical results: near-perfect forgetting with competitive retention and utility
  • Comprehensive ablations and qualitative examples demonstrating end-to-end pipeline behavior

Limitations

  • Retain buffer required: STAMP still needs a small replay buffer $D_r$ (though effective at 10%) to preserve retained knowledge, which is non-trivial to store on-device and may conflict with privacy regulations
  • Scope: Evaluated primarily on Llama-3-8B; generalization to larger models and multimodal settings is left as future work
  • Refusal subspace construction: Relies on prompting "I don't know" to generate refusal-style activations; the robustness of this heuristic across model families is not fully explored
  • End-to-end latency: RePAIR's full pipeline turnaround (6.50–9.36 min) is longer than raw STAMP runtime due to multi-model orchestration overhead

Questions for Further Exploration

  • Can the retain buffer be completely eliminated while preserving retention, perhaps by leveraging synthetic retain data or memory-free regularization?
  • How does STAMP scale to models beyond 8B parameters, and does the optimal intervention layer shift with model scale?
  • Could the same activation-steering mechanism be adapted for other test-time knowledge edits (e.g., factual updates) rather than only forgetting?
  • What are the security implications if malicious users exploit IMU interfaces to induce targeted forgetting of beneficial knowledge?
