# Fully Homomorphic Encryption on Llama 3 model for privacy preserving LLM inference

> **一句话总结：** 本文将基于格的后量子全同态加密（FHE）通过 concrete-ml 库集成到 LLaMA-3 的推理流程中，提出了单头和多头两种量化加密变体，在 CPU 上实现了高达 98% 的生成准确率和约 80 tokens/秒 的吞吐，并系统评估了缓存行为、噪声增长与多种攻击场景下的隐私保护能力。

> **TL;DR:** This paper integrates lattice-based post-quantum Fully Homomorphic Encryption (FHE) into the LLaMA-3 inference pipeline using the concrete-ml library, proposing quantized single-head and multi-head encrypted variants. It achieves up to 98% generation accuracy and ~80 tokens/sec throughput on consumer CPUs, while providing detailed analysis of cache behavior, noise growth, and attack mitigations.

## 核心贡献
- 首次将 concrete-ml 提供的可编程自举（PBS）FHE 方案嵌入 LLaMA-3 的 Grouped Multi-Query Attention 推理层，构建端到端加密推理模型 PQC-LLaMA-3
- 设计了两种变体：SingleHeadQLlamaModel（仅首层加密）和 MultiHeadsQLlamaModel（全层加密），量化至 2-bit 整数运算以兼容 FHE 的整数算术限制
- 在三种不同 CPU 平台上完成了短上下文（1–10 tokens）和长上下文（70–500 tokens）的生成准确率、延迟、吞吐率及多级缓存性能的系统评估
- 提出 FHE 专用评估指标（PBS/ token、内存/ token、吞吐率），并量化了 FHE 对提示泄漏、输出重构、梯度泄漏等攻击的防御效果与边界

## Key Contributions
- First to embed the concrete-ml Programmable Bootstrapping (PBS) FHE scheme into LLaMA-3's Grouped Multi-Query Attention layers, building an end-to-end encrypted inference model (PQC-LLaMA-3)
- Designed two variants—SingleHeadQLlamaModel (first layer encrypted) and MultiHeadsQLlamaModel (all layers encrypted)—using 2-bit integer quantization to fit concrete-ml's integer-only arithmetic constraints
- Conducted systematic evaluations across three CPU platforms for short-context (1–10 tokens) and long-context (70–500 tokens) generation, measuring accuracy, latency, throughput, and multi-level cache behavior
- Introduced FHE-specific efficiency metrics (PBS per token, memory per token, throughput) and assessed FHE protections against prompt leakage, output reconstruction, and gradient leakage, while clearly stating its limitations

## Methodology
The authors modify the LLaMA-3 decoder architecture to replace standard floating-point attention operations with FHE-compatible quantized integer operations via the concrete-ml library. Key steps include:
- **Quantized Attention Heads:** Linear projections for Q, K, V are replaced with quantized integer projections using concrete-ml's `DualArray` abstraction, calibrated on a cleartext pass to determine scaling factors and zero-points.
- **Softmax Replacement:** The standard softmax in attention is approximated with a polynomial/FHE-compatible function suitable for encrypted evaluation.
- **Programmable Bootstrapping (PBS):** PBS nodes are statically inserted during circuit compilation to refresh ciphertext noise and evaluate non-linearities, enabling arbitrary-depth encrypted computation.
- **FHE Modes:** Three modes are supported—"disable" (plain float), "simulate" (quantized cleartext for fast accuracy estimation), and "execute" (full FHE encryption/decryption).
- **Client-Server Setup:** The client encrypts input embeddings with a public key; the server performs homomorphic evaluation using an evaluation key; the client decrypts the resulting logits with a private key.

## Results & Highlights
- **Short-context generation (1–10 tokens):**
  - *SingleHeadQLlamaModel:* Accuracy rises from 47.2% (top-k=1) to 81.7% (top-k=10); inference latency ~0.237–0.24 s on an Intel i9-14900K (M3).
  - *MultiHeadsQLlamaModel:* Accuracy rises from 51.6% (top-k=1) to 87.4% (top-k=10); latency ~0.90–1.26 s on M3.
- **Long-context generation (70–500 tokens):**
  - *SingleHeadQLlamaModel:* Accuracy improves from 91.3% (top-k=70) to 98.2% (top-k=500); latency stays sub-second (~0.23–0.24 s).
  - *MultiHeadsQLlamaModel:* Accuracy improves from 94.7% (top-k=70) to ~97.7% (top-k=500); latency ~0.79–1.02 s on M3.
- **Throughput & Efficiency:**
  - SingleHead achieves ~77 tokens/sec on M3; MultiHead achieves ~8.9 tokens/sec (vs. ~142 tokens/sec for plain LLaMA-3).
  - A 6-token sequence requires 5,208 PBS operations—roughly half the PBS count reported in prior FHE-GPT-2 work.
- **Cache Behavior:** Extensive `perf` analysis shows strong L1 and LLC locality for both encrypted variants, though L2 miss rates are higher than the plain model due to ciphertext expansion. KV-caching keeps inference times nearly constant across top-k values.
- **Attack Mitigation Assessment:** FHE effectively prevents prompt/context leakage, output reconstruction, and gradient leakage *in-flight*, but does not inherently stop side-channel attacks, model extraction via query outputs, or black-box adversarial attacks.

## Critical Assessment
This paper makes a credible engineering contribution by actually building and benchmarking a FHE-secured LLaMA-3 inference pipeline on CPUs, rather than staying at a theoretical level. The experimental design is thorough, especially the cache-level performance analysis and the honest security boundary analysis in Table 6. However, the accuracy metric is defined as token-level agreement with the plain model, not standard NLP benchmarks (e.g., perplexity, downstream task performance), which limits external comparability. The claim of "80 tokens per second" applies to the single-head variant under specific conditions and should not be conflated with practical interactive latency for the fully encrypted multi-head model. Additionally, the 2-bit quantization is a library-imposed constraint rather than an algorithmic choice, and its impact on model quality beyond top-k agreement is not deeply analyzed.

## Strengths
- End-to-end implementation: moves beyond theoretical FHE-for-ML proposals to an actual modified LLaMA-3 inference stack
- Rigorous hardware evaluation across three distinct CPU architectures with micro-architectural profiling (L1/L2/LLC miss rates)
- Transparent security assessment that clearly distinguishes what FHE protects against and what it does not
- Introduces concrete FHE-specific metrics (PBS/token, memory/token, throughput) that are useful for future work in this space

## Limitations
- Accuracy evaluation is limited to agreement with the unencrypted baseline; no perplexity or standard downstream benchmark scores are reported
- 2-bit integer quantization is required by concrete-ml's lack of floating-point support, which likely caps model quality compared to higher-precision quantization
- The multi-head fully encrypted variant incurs a ~4× latency penalty and ~10× throughput drop versus the single-head variant, making it expensive for real-time use
- No comparison with alternative privacy-preserving inference techniques (e.g., MPC, TEEs, or differential privacy)
- Compilation time is a bottleneck (up to ~26 s for the multi-head model), though it is amortizable across repeated queries

本文与 LLM 的数值精度、计算可靠性以及隐私保护推理研究高度相关。它直接在 transformer 的注意力机制内部注入后量子全同态加密操作，属于隐私计算与大模型系统交叉领域的前沿工程尝试。论文对缓存层次结构的详细剖析，为理解 FHE 量化算子在现代 CPU 上的实际执行行为提供了宝贵的实证数据；而对 FHE 防护边界的诚实评估，则对任何计划将同态加密部署到生产级 LLM 推理管道的研究者或工程师具有直接参考价值。所提出的 PQC-LLaMA-3 框架也为后续探索混合隐私方案（FHE + TEE/DP）奠定了代码和评估基础。

## Questions for Further Exploration
- How would the encrypted LLaMA-3 perform on standard downstream NLP benchmarks (perplexity, MMLU, etc.) rather than only baseline agreement?
- Can higher-bit quantization or emerging FHE libraries with native floating-point support close the accuracy gap relative to the plain model?
- Would a hybrid architecture (e.g., FHE for early layers + plaintext for later layers) offer a better privacy-latency trade-off than the current single-head vs. multi-head dichotomy?
- How does the concrete-ml compilation time scale with model size (e.g., 70B vs. 8B), and what hardware acceleration (GPU/FPGA) would be needed to make fully encrypted inference practical?
- Can combining FHE with differential privacy or TEEs close the remaining attack surface for side-channel and model-extraction threats?
