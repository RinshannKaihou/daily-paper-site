> **一句话总结：** 提出一种 agent 记忆架构 SCM，将"操作点"暴露为逐查询的语义模式（mode→payload→procedure），由冻结分类器与运行时门控把查询分发到直接检索/图谱多跳/长文综合三类流程并共享同一异步摄入底座；在三个记忆基准上取得配置级领先（LoCoMo 84.87%、MAB 61.49%、LongMemEval 86.00%），但论文明确声明因缺失基线原始输出、对齐时延与完整 token 账本，无法证明路由或组件的因果效应与统计显著性。

> **TL;DR:** Supra Cognitive Modes (SCM) routes each agent-memory query to one of three procedure families (direct fused lookup, graph/iterative multi-hop, long-form synthesis) via an explicit or classifier-selected mode, dispatched by runtime gates over a single shared ingest substrate. On LoCoMo (n=1,986), MemoryAgentBench (n=3,671), and LongMemEval (n=500), the deployed configuration records the largest displayed aggregate — 84.87% LoCoMo factoid / 81.22% mixed, 61.49% MAB, 86.00% LongMemEval — versus Mem0 v2 OSS (22.02% geo-mean) and a production comparator (57.27% geo-mean). The authors are unusually explicit that raw baseline rows, aligned end-to-end latency, complete token ledgers, and fixed-policy controls are unavailable, so the results characterize one implemented routed configuration rather than establish causal routing effects or statistical significance.

## 核心贡献

- **模式-载荷-流程接口（mode-payload-procedure）**：把 agent 记忆的"操作点"从固定单一策略升级为逐查询可控变量，应用层可显式传入或由冻结分类器自动选择语义模式。
- **共享异步底座实现**：在单一摄入底座（多粒度嵌入 + 抽取三元组 + 事实版本元数据 + 可选异步补充信号）之上组织三类流程家族（直接融合检索 / 图谱与迭代多跳 / 分层长文综合），均从同一可查询索引读取。
- **仓库可追溯的配置刻画**：在三个公开记忆基准上对部署配置进行描述性评测，并给出可复现的任务/模式条件化失败分层与对可用时延、成本、判官、溯源证据的显式审计。
- **诚实的证据边界声明**：明确区分配置级比较、观测性诊断、作者提供的非配对介入（abstention 开关）三层证据，避免把描述性差异误读为因果或显著性结论。

## Key Contributions

- **Per-query mode–payload–procedure interface** that exposes the memory operating point as a per-query control (explicit or automatically selected) rather than fixing one policy for an entire deployment; modes and executed procedures are deliberately not one-to-one because runtime gates and fallbacks can change the final tier.
- **Source-visible implementation over one shared asynchronous substrate**: mandatory Tier-1 ingest writes chunks, sentence/paragraph segments, embeddings, lexical-index fields, and extracted triples to make fused lookup immediately query-ready; asynchronous nullable supplement jobs (doc shape, keypoints, canonical entities, event chains, entity types) are read only by payloads that enable the corresponding flag, so missing enrichments degrade gracefully rather than blocking readiness.
- **Three procedure families on one substrate**: direct lookup (BM25 + chunk/sentence/paragraph dense retrieval fused by reciprocal-rank fusion, depth capped at 100, single Sonnet synthesis call with prompt-directed abstention); graph/iterative multi-hop (triples ordered by descending version number so newer values take precedence for current-state queries, with fallback to iterative dense retrieval); and long-form synthesis (chronological sampling targeting 200 chunks with a multipass bullets-expand procedure and a source-shape classifier).
- **Repository-traceable characterization** of the deployed configuration across LoCoMo (n=1,986), MemoryAgentBench (n=3,671), and LongMemEval (n=500), plus reproducible task- and mode-conditioned failure strata and an explicit audit of available timing, cost, judge, and provenance evidence — including a one-row duplicate-question category correction on LoCoMo.

## Methodology

**Architecture.** A query enters with an explicit mode or is classified by a frozen four-mode semantic classifier (Claude Haiku 4.5, temperature 0, max tokens 16) emitting one of SINGLE_FACT_LOOKUP, LONG_FORM_SYNTHESIS, TIME_ANCHORED_LOOKUP, or LATEST_VERSION_RESOLUTION (defaulting to single-fact on unparseable output). A payload map turns the label into concrete retrieval/synthesis flags. Runtime gates (a chain-shape detector that can force multi-hop, an ICL corpus-shape detector that reroutes to ICL_BROAD_CONTEXT, empty-retrieval fallbacks, and intent/format rules) can revise the tier before a procedure reads from the shared substrate. The dispatch is summarized as: `mode = explicit_mode or classify(query); payload = runtime_gates(mode_to_payload(mode), query, retrieved_context); procedure = procedure_for(payload.final_tier)`.

**Procedure families.** Direct lookup embeds the query plus up to two paraphrases, runs BM25 (k1=1.5, b=0.75) and chunk/sentence/paragraph dense retrievers in parallel, fuses with reciprocal-rank fusion (k_constant=30), and synthesizes with a single Claude Sonnet 4.5 call (temperature 0) that emits "No information available" when the entity is unsupported. Graph/iterative multi-hop stores triples with version numbers and active status so newer values take precedence for current-state queries, and can fall back to iterative dense retrieval when graph planning fails. Long-form synthesis samples chronologically (target 200 chunks) and applies a multipass bullets-expand procedure shaped by a source-shape classifier (narrative/research/technical/log/mixed).

**Substrate.** Mandatory Tier-1 ingest creates queryable BM25 + embedding + triple indexes; async supplement jobs add nullable higher-level signals (doc shape, keypoints, canonical entities, event chains, question patterns, entity types) read only when the active payload enables them.

**Evaluation protocol.** Synthesis uses Claude Sonnet 4.5 (temperature 0); embeddings are OpenAI text-embedding-3-large at 1,024 dims; Mem0 v2 OSS uses ChromaDB while SCM and the production comparator use pgvector. MAB scores the online route answer (latency aligned), whereas LoCoMo and LongMemEval score a later offline Stage-3 synthesis answer whose Stage-3 duration and token usage were not persisted. The frozen classifier was selected on 1,065 synthetic off-test examples (96.06% overall, 95.80% macro) and checked on a 568-question consensus-labeled real-dev slice (90.14% micro, 89.37% macro).

## Results & Highlights

- **LoCoMo (n=1,986):** SCM records 84.87% on the four factoid categories (gpt-4o-mini judge) and 68.61% on the adversarial abstention category (lexical scorer), combining to 81.22% mixed full set — versus a production comparator at 63.10% mixed and Mem0 v2 OSS at 13.08% mixed. (The 81.22% is corrected up from 81.17% by repairing a one-row duplicate-question category collision: "What did Gina receive from a dance contest?" appeared once as a category-4 factoid and once as a category-5 adversarial item.)
- **MemoryAgentBench (n=3,671):** SCM scores 61.49% overall (macro over AR/TTL/LRU/SF) across two repetitions scoring 61.72% and 61.26% (sample std 0.32 points), versus 52.22% for the production comparator and 34.03% for Mem0 v2 OSS. The competency breakdown is AR 78.62%, TTL 50.05%, LRU 54.67%, SF 62.62%; sub-dataset scores range widely from SH-Doc-QA 96.0% and MH-Doc-QA 85.0% down to Movie-Rec 13.9% and InfBench-Sum 27.5%.
- **LongMemEval (n=500):** SCM records 86.00% overall versus 57.00% (production comparator) and 24.00% (Mem0 v2 OSS), with the largest observed SCM-vs-comparator deltas on temporal-reasoning (+51.88 points, 83.46% vs 31.58%) and knowledge-revision (+25.64 points, 88.46% vs 62.82%).
- **Cross-benchmark floor/geo-mean:** SCM's minimum across the three benchmarks is 61.49% and its unweighted geometric mean is 75.45%, versus the comparator (min 52.22%, geo-mean 57.27%) and Mem0 v2 OSS (min 13.08%, geo-mean 22.02%).
- **Reproduction consistency:** the author reference and a local repository-backed reproduction fall within roughly one point on each headline metric (e.g., LoCoMo mixed 81.22% vs 81.97%; MAB 61.49% vs 62.19%); an externally reported teammate run gives 61.43% MAB / 81.17% LoCoMo mixed but provides no row-level output.
- **Reported abstention configuration difference:** with entity-grounded abstention enabled the LoCoMo adversarial cut scores 68.46% (becoming 68.61% after correction) versus 2.46% disabled on the original 447-row cut — a 66-point difference, but the disabled paired output is absent so it is treated as a reported configuration difference rather than an audited ablation.
- **Retrieval-vs-synthesis diagnostic:** all 73 judged failures in the local 85.4% LongMemEval reproduction have `hit_at_k=true`, consistent with errors occurring after initial retrieval (evidence-use errors rather than retrieval misses).

## Authors & Credibility

Joshua Tobkin and David Yang are at Supra Research (corresponding author j.tobkin@supra.com); the work describes and characterizes Supra's own deployed memory system. The paper is unusually disciplined about evidence boundaries: it repeatedly states what the available artifact cannot support (causal routing effects, efficiency gains, statistical significance) and reserves a dedicated appendix for reproducibility and evidence scope. No public artifact accompanies this version (credentials, private-infrastructure references, nonportable paths, and third-party licensing require review before distribution), though the appendix pins models, prompts, temperatures, retrieval parameters, and artifact paths so the reported numbers are traceable to files. The evaluation is configuration-level against a production comparator that is "treated as an evaluated configuration rather than a fully inspectable research artifact," and against Mem0 v2 OSS (mem0ai==2.0.3).

## Critical Assessment

- **Novelty (moderate).** The architectural novelty is explicitly framed as composition and a control interface, not new retrieval primitives: hybrid lexical/dense retrieval, reciprocal-rank fusion, graph traversal (GraphRAG-style), temporal metadata, and multipass synthesis (RAPTOR-style) all come from established work. The differentiator is an application-visible per-query semantic selector whose procedures share one ingest substrate — a real but incremental contribution relative to Adaptive-RAG (no/single/multi-step tiers) and RouteLLM/FrugalGPT (which route over model/cascade choice rather than retrieval strategy + substrate reads + prompt family + synthesis procedure).
- **Rigor (mixed, honestly bounded).** The strongest aspect is the candid evidence ledger: the authors explicitly note that raw Mem0 and production-comparator predictions are unavailable (so prompt parity, row-level corrections, and paired uncertainty cannot be audited), that LoCoMo/LongMem scored answers come from an offline Stage-3 synthesis whose latency and token usage were not persisted (so no aligned end-to-end latency comparison), and that the active long-form payload has a provenance mismatch (its JSON field records `graph_routed` while an amendment note records `graph_then_iterative`, and stored outputs do not retain the final strategy). They also report no p-values, confidence intervals, or effect sizes. These are laudable disclosures but mean the headline accuracy deltas are descriptive configuration-level comparisons, not significance-tested wins.
- **Reproducibility (partial).** Models, prompts, temperatures, retrieval parameters, BM25 constants, RRF constant, and artifact paths are pinned in Appendix A, and two reproducibility guardrails are called out (a question-ID join that fixes a Stage-3 scoring bug where an enumerate-based join measured LongMem at 21.8% vs the corrected 88.2%; and the Detective-QA resynthesis before the LRU macro). However, `score_locomo.py` and `score_longmem.py` use hard-coded absolute roots, no public artifact accompanies this version, and baseline raw rows are absent.
- **Significance (moderate).** The per-query control interface is a useful engineering contribution for practitioners deploying agent memory who must trade off answer quality, latency, and cost placement. The descriptive results and mode-conditioned failure strata (e.g., 229/400 FC-MH failures concentrated in LONG_FORM_SYNTHESIS; all 73 LongMem failures with hit_at_k=true) are concrete targets for follow-up work, but cannot by themselves establish that routing beats a fixed policy.
- **Red flags.** (1) Baseline provenance is asymmetric: SCM outputs are in the repo but baseline raw outputs are not, so the 63.10%→81.22% and 52.22%→61.49% deltas are supplied rather than independently recomputed; the LoCoMo one-row correction was applied to SCM rows but cannot be applied to baselines. (2) The author-provided abstention comparison lacks the paired disabled output, so the 66-point "abstention on/off" swing is a reported configuration difference, not an audited one-variable ablation. (3) The real-dev classifier slice contains almost no LATEST_VERSION_RESOLUTION examples, so one of the four frozen modes is validated synthetically but not measured on real data. (4) The LoCoMo judge prompt originated with one compared system (Mem0), creating a potential circularity the authors flag but cannot remove.

## Strengths

- Clean separation of mode (semantic label), payload (retrieval/synthesis config), and procedure (execution family), with runtime gates that can revise the tier — a genuinely useful abstraction for cost/quality/latency tuning in deployed memory.
- Shared asynchronous substrate design where missing optional enrichments degrade gracefully rather than blocking query readiness (nullable fields read only when enabled).
- Exceptionally honest evidence-boundary reporting: a dedicated appendix enumerates what the artifact does and does not support, and the paper consistently refuses to overclaim (e.g., "the supported claim is therefore implementation of automatic and explicit per-query dispatch, not router optimality or an advantage over fixed policies").
- Concrete, actionable diagnostic failure strata (per route and per sub-dataset) that point to where engineering effort should go: recommendation recall, multi-hop conflict resolution, temporal evidence use, and lexical abstention normalization.
- Pinned models, prompts, retrieval parameters, and artifact paths in the appendix make the reported numbers traceable to files, and two methodologically important scoring guardrails (question-ID join, Detective-QA resynthesis) are explicitly documented.

## Limitations

- No fixed-policy / single-procedure controls were run, so routing cannot be credited with the observed cross-workload profile; the paper states this plainly but it is the central missing experiment.
- Raw baseline outputs are unavailable, so baseline parity, row-level corrections, and paired uncertainty cannot be independently verified; the LoCoMo one-row correction was applied only to SCM rows.
- Aligned end-to-end latency is available only for MAB (9.00 s/request under concurrency 12); LoCoMo and LongMemEval scored answers come from an offline Stage-3 path whose duration and token usage were not persisted, so no cross-system latency or per-query cost comparison is presented.
- Most cells are single-run; only the MAB reference has two repetitions (std 0.32 points). No p-values, confidence intervals, or effect sizes are reported.
- The active long-form payload has a provenance mismatch (`graph_routed` vs `graph_then_iterative`) and stored rows omit the final executed procedure, so individual procedures cannot be assigned scores.
- Vector-store choice is confounded with the comparison: Mem0 uses ChromaDB while SCM and the comparator use pgvector, with no paired control.
- Language-model judges are used without an additional human-agreement study, and the LoCoMo factoid judge prompt originated with one compared system (potential circularity).
- No public artifact accompanies this version; sanitization, licensing review, and portable path normalization are required before release.

## Relevance

对关注 agent 长期记忆、RAG 架构与检索路由的人来说，这篇论文提供了一个可落地的"逐查询路由 + 共享底座"工程范式（mode→payload→procedure 接口），值得作为 agent-memory 系统设计的参考。不过其证据边界声明（缺失固定策略对照、基线原始输出与对齐时延）也提示读者把它当作"配置级描述性刻画"而非因果证明来读。

## Questions for Further Exploration

- How does SCM's routed configuration actually compare against a single fixed procedure (e.g., always-direct or always-long-form) on identical corpus snapshots and with complete routing provenance persisted? This is the experiment that would establish a causal routing effect.
- What is the end-to-end latency and per-query dollar cost of each procedure family when the scored answer's timing and token usage are retained (currently only MAB has aligned latency; LoCoMo/LongMem Stage-3 duration is missing)?
- How much of the LongMemEval advantage (especially the +51.88-point temporal-reasoning delta) survives a paired ChromaDB-vs-pgvector control that holds embedding model, top_k, corpus scale, and scoring path constant?
- Could the prompt-directed abstention behavior be replaced by a deterministic post-retrieval entity-grounded filter, and would that change the 66-point abstention swing once the disabled paired output is reproduced?
- Since all 73 LongMemEval failures in the local reproduction have `hit_at_k=true`, what synthesis-side interventions (e.g., evidence-counting, preference-grounding, event-ordering) most reduce evidence-use errors versus retrieval misses?
- How robust is the frozen four-mode classifier to distribution shift, given that LATEST_VERSION_RESOLUTION is essentially unvalidated on the real-dev slice and that runtime gates already introduce a fifth observed `classified_mode` (ICL_BROAD_CONTEXT)?
