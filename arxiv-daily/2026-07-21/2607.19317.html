CircuitKIT: Circuit Discovery, Evaluation, and Application Toolkit for Mechanistic Interpretability

Pratinav Seth, Hem Gosalia & Aditya Kasliwal (equal contribution), Vinay Kumar Sankarapu  ·  Lexsi Labs
Key Finding A unified interpretability toolkit whose case studies show circuit faithfulness does not predict what happens when you act on it.
Anchor metric Key Metric
−0.78 ρ
Spearman anti-correlation between patch faithfulness and pruning retention on Llama-3.2-3B-Instruct / BoolQ (30% sparsity), p = 0.010.
Why This Matters
Closes the locate-then-act loop with one reusable artifact across discovery, evaluation, and intervention — usable on custom data for pruning, quantization, knowledge editing, and safety audits.
Critical Edge Sharpest Critique
Statistical power is thin: the headline ρ = −0.78 is explicitly one model·task cell — “not a population estimate” — every other study is single-seed, and cross-seed intervention reliability (Pillar 7) is unmeasured.

Methodology4 stages · 1 artifact

CoreA source-available Python library (LSAL v1.1) built on TransformerLens that composes four stages behind one typed, serializable CircuitScores record — carrying task, model id, algorithm, granularity, and per-component attribution scores.

Discovery13 algorithms, four backends, two tiers. Stable tier (6): the gradient-attribution EAP family (EAP, EAP-IG, EAP-GP), search-based ACDC, trainable information-bottleneck IBCircuit (clean-only), and gradient-free CD-T (clean-only). Two granularities: node and neuron.

DataA declarative template path maps CSV / JSONL / HuggingFace datasets into discovery tasks with automatic token-alignment, label precompute, and a GREEN/YELLOW/RED worthiness gate — clean-only data auto-routes to IBCircuit/CD-T.

EvalA six-pillar faithfulness panel (causal patching, ablation, stability, robustness, baseline comparison, generalization) + optional intervention reliability — all reported signed so behavioral inversions are visible.

ActSeven modules consume the same artifact: pruning, quantization, selective fine-tuning, ROME/MEMIT editing, steering, circuit-LoRA healing, hallucination probing — scored through lm-evaluation-harness.

Pipeline API ck.* functional YAML CLI @register extensibility

Assessmentstrengths · limitations

Strengths 5

  • End-to-end unification. Genuinely spans discover → evaluate → intervene behind one interchangeable typed artifact — a gap no prior toolkit fills.
  • Declarative custom-data path. Token alignment, worthiness gate, and clean-only routing remove the bottleneck that kept circuit discovery on canonical tasks.
  • Six-pillar signed faithfulness panel directly operationalizes the known ablation-method dependence of faithfulness; surfaces behavioral inversions.
  • Unusually honest reporting. Null results (E7), anti-correlations (E5), and the “faithfulness ≠ actionability” message are presented as findings.
  • Three interchangeable interfaces (Pipeline, ck.*, YAML CLI) over one artifact, with @register-driven extensibility.

Limitations 6

  • Thin statistical power. Three-seed stability, single-seed everything else; the ρ = −0.78 headline is one cell; Pillar 7 unmeasured.
  • Intervention coverage trails discovery. Architecture registry fully shipped only for Llama / Qwen / Gemma; pinned to TransformerLens 2.18.
  • Scale costs under-documented. IBCircuit trains single-batch (footprint grows with model size); ACDC is node-only and ~50× slower than EAP.
  • Corruption strategies assume identifiable syntax — efficacy on safety / instruction-tuned prompts (AdvBench, TruthfulQA, StereoSet) is future work.
  • Not OSI open source. LSAL v1.1 prohibits commercial use, limiting enterprise adoption and toolchain integration.
  • Several quantitative claims defer to an under-review companion audit [35], so some numbers are not reproducible from this paper alone.

Open Questionsfor further exploration

  1. Does the patch-faithfulness → pruning anti-correlation (ρ = −0.78, one cell) generalize, or is it an artifact of BoolQ’s binary structure and Llama-3.2-3B’s computation?
  2. Can multi-seed re-discovery (Pillar 7, SDrel) actually reconcile discovery stability with intervention reliability — and what is the empirical stability→reliability correlation?
  3. Do auto-synthesized corruption strategies (entity-swap, token-swap, role-swap) carry meaningful attribution signal on safety benchmarks where syntactic structure is less identifiable?
  4. Does the E4 finding — a discovered refusal circuit inverting toward compliance under hard ablation — generalize, and what does it imply for the safety of circuit-restricted weight surgery on deployed models?
  5. How does circuit-guided importance compare to SAE-feature-guided importance for pruning and editing, now that SAE features are flagged as a “natural next granularity”?
  6. Does the GREEN/YELLOW/RED worthiness gate actually predict downstream faithfulness, or does it over-reject useful noisy datasets?
