> **一句话总结：** 提出 Parallel Shapley 框架，将多路径推理中的每条路径视为合作博弈中的玩家，用 Shapley 值（通过蒙特卡洛采样近似 + 生成式奖励模型评估子集效用）量化每条路径的边际贡献，作为路径级奖励与结果级奖励共同驱动 GRPO 训练，在 Qwen3-4B 上仅训练 40 步即超越训练 200 步的 Parallel-R1，AIME24/AIME25/AMC23 三个基准上 Pass@16 平均提升 42.3%。

> **TL;DR:** The paper proposes Parallel Shapley, a GRPO-based RL framework that treats each parallel reasoning path as a player in a cooperative game and uses Shapley values (approximated via Monte Carlo permutation sampling, with a Generative Reward Model scoring path-subset utilities) to assign fine-grained path-level rewards alongside an outcome-level correctness reward. On Qwen3-4B-Base evaluated on AIME24/AIME25/AMC23/MATH, training for only 40 RL steps matches or beats Parallel-R1 trained for 200 steps, lifting Pass@16 by 42.3% on average across three benchmarks (e.g., AIME24 Pass@16 33.2 → 63.3) while maintaining competitive Mean@16, at the cost of a documented Pass@16-vs-Mean@16 trade-off.

## 核心贡献

- **首次将合作博弈论与 Shapley 值引入并行思考（parallel thinking）训练**：把每条推理路径建模为合作博弈中的"玩家"，通过边际贡献来量化路径级奖励，从而显式区分互补路径与误导/冗余路径，解决"搭便车"（free-rider）问题。
- **设计了蒙特卡洛 Shapley 奖励估计 + 生成式奖励模型（GRM）的两级奖励机制**：用 GRM 评估任意路径子集的联合效用 U(S)∈[0,5]，再用蒙特卡洛排列采样将指数级的精确 Shapley 计算降为可扩展的近似估计，并与结果级奖励解耦，注入到 token 级别的策略梯度中。
- **显著的训练效率与性能提升**：在 Qwen3-4B-Base 上仅训练 40 步 RL 即在多数设定下达到或超过训练 200 步的 Parallel-R1，AIME24/25/AMC23 上 Pass@16 平均提升 42.3%，并附带更稳定的收敛与更快的训练动态。
- **丰富的诊断性实验**：通过 LOO / 独立路径评估消融、GRM 强度敏感性分析、路径数 K 消融，以及"路径互补性"掩码实验，系统性地验证了 Shapley 归因（而非单纯的评估器强度）是收益来源。

## Key Contributions

- **First application of cooperative game theory and Shapley value attribution to parallel thinking**, modeling each reasoning path as a player whose marginal contribution is quantified, so complementary paths are rewarded and misleading/redundant ("free rider") paths are discouraged.
- **A Monte Carlo Shapley-based reward estimator combined with a Generative Reward Model (GRM)**, which scores the utility of arbitrary path coalitions U(S)∈[0,5] and converts the exponentially expensive exact Shapley computation into a scalable permutation-sampling approximation, decoupling process (path-level) rewards from outcome rewards.
- **Strong and training-efficient empirical gains**: with only 40 RL steps on Qwen3-4B-Base, Parallel Shapley matches or surpasses Parallel-R1 trained for 200 steps, achieving an average 42.3% Pass@16 improvement across AIME24/25/AMC23 with faster convergence and a higher plateau.
- **Comprehensive diagnostic evidence**, including Leave-One-Out and independent-path ablations, GRM-strength sensitivity tests, a path-count (K) sweep, and a path-complementarity masking analysis that isolates coalition-aware attribution (not evaluator strength) as the source of improvement.

## Methodology

Parallel Shapley is built on three components and trained on top of GRPO (Group Relative Policy Optimization).

**1. Tag-guided parallel reasoning structure.** Following prior work (Multiverse, Parallel-R1), the model emits three HTML-like tags: `<Parallel>...</Parallel>`, `<Path>...</Path>` (K parallel reasoning paths), and `<Summary>...</Summary>` (aggregated final answer). A cold-start SFT on Parallel-GSM8K first instills this format (230 steps).

**2. Shapley-based reward attribution.** The set of K paths P = {path_1, ..., path_K} is treated as a set of players in a cooperative game. The utility function U: 2^P → ℝ is instantiated by a Generative Reward Model that scores any subset of paths holistically on five dimensions: U(S) = GRM(S) ∈ [0,5] (default GRM is qwen-plus). The Shapley value of path_j is its expected marginal contribution averaged over all coalitions. Exact computation is 2^K and prohibitive, so a Monte Carlo permutation approximation is used: sample M random orderings of paths, and for each ordering measure Δ_j = U(S_j ∪ {path_j}) − U(S_j) where S_j are the predecessors; the estimate ϕ_MC = (1/M)ΣΔ_j converges at rate O(1/√M).

**3. Two-level reward + GRPO training.** Rewards are assigned at the token level. An outcome reward R_out ∈ {0,1} (exact-match correctness with ±1% numerical tolerance) is placed on the last token of the whole response. Each path-level Shapley reward ϕ̃_j is placed on the closing token of its `<Path_j>` span. The combined token reward is r_t = λ_o·R_out·𝟙[t=T] + λ_p·Σ_j ϕ̃_j·𝟙[t=e_j]. GRPO then performs group-normalized advantage estimation and token-level policy-gradient updates. Training uses 40 RL steps on the DAPO dataset (batch 256, lr 1e-6, 8 rollouts, prompt/response caps 2000/3000 tokens), implemented on the VERL codebase.

## Results & Highlights

- **Pass@16 (average across AIME24/AIME25/AMC23) improves by 42.3% over Parallel-R1-Unseen**, the SOTA parallel-thinking baseline; e.g., AIME24 Pass@16 jumps 33.2 → 63.3 and AIME25 Pass@16 jumps 37.8 → 50.0.
- **AIME24 Pass@16: 63.3 vs Parallel-R1-Unseen 33.2** (+90.7% relative), AMC23 Pass@16: 92.5 vs 88.9 (+4.0%), and AMC23 Mean@16: 72.8 vs 69.7 (+4.4%), all with only 40 RL steps versus Parallel-R1's 200 steps.
- **MATH Mean@1: 83.1 vs Parallel-R1-Unseen 82.6**, confirming the gains extend beyond the i.i.d.-sampling metrics. The base Qwen3-4B-Base scores only 13.9 on MATH Mean@1, so the full pipeline lifts it by ~69 absolute points.
- **Training dynamics (Figure 3): final training score 0.652 for Parallel Shapley vs 0.609 for Parallel-R1 and 0.591 for GRPO**, with Parallel Shapley plateauing after ~30 steps while baselines keep drifting upward without stabilizing in the same 40-step budget.
- **Ablation — attribution design matters more than evaluator strength**: Parallel Shapley (Avg 48.3) beats both Leave-One-Out (Avg 46.2) and Independent Path Evaluation (Avg 46.5); crucially, using the identical qwen-plus evaluator, Independent Path Evaluation scores 46.5 vs Parallel Shapley 48.3, isolating coalition-aware attribution as the cause rather than GRM power.
- **GRM sensitivity**: even the weakest open-source evaluator (Qwen2.5-3B-Instruct, Avg 47.4) still beats Parallel-R1-Unseen (47.1) on average, and stronger evaluators monotonically help (Qwen2.5-7B-Instruct Avg 48.1), showing the method does not depend on a proprietary GRM.
- **Path-count ablation**: K=4 is optimal (Avg 48.3); K=5 degrades to Avg 46.2 because excessive paths dilute the Shapley signal-to-noise ratio.
- **Path-complementarity masking (Figure 4)**: under 40% path masking, Parallel Shapley degrades 82.1% on average vs 71.9% for Parallel-R1 (a 10.2 pp gap), empirically confirming that Shapley attribution induces non-overlapping, specialized, complementary paths.

## Critical Assessment

- **Novelty**: Genuine and well-motivated. The core insight — that outcome-level shared rewards in summary-based parallel reasoning are a reward-hacking surface (misleading paths receive positive gradient when their content is absorbed/overridden) — is convincingly argued, and repurposing Shapley values with a GRM-defined utility is a clean, first-of-its-kind application to parallel thinking. The framing is more novel than the individual ingredients (Shapley values and GRMs are both prior art).
- **Rigor**: Largely strong, with LOO and independent-path ablations, a GRM-strength sweep (3B→7B→qwen-plus), a K sweep, and a masking-based complementarity probe. However, the headline "+42.3% Pass@16" is a relative gain on the *upper-bound* metric (probability that at least one of 16 samples is correct); the authors themselves concede Mean@16 improvements are "more moderate" and on AIME25 actually regress (17.2 vs Parallel-R1's 17.7). The abstract and intro foreground Pass@16 and downplay this trade-off, which slightly oversells the practical win.
- **Reproducibility**: Good on paper (backbone Qwen3-4B-Base, DAPO dataset, 40 steps, batch 256, lr 1e-6, K=4, prompts and proofs in appendices). But the default and best GRM is qwen-plus, a proprietary/black-box evaluator; the open-source GRM rows are weaker, so reproducing the top numbers depends on access to a specific commercial API. The M permutation count and λ_o/λ_p weights are not stated in the main text.
- **Significance**: Solid within the narrow parallel-reasoning subfield. The Pass@16-vs-Mean@16 tension is the most interesting finding because it reveals that Shapley attribution buys diversity (a higher upper bound) at the expense of single-generation accuracy — a real, if uncomfortable, result that future self-distillation work (their own planned direction) would need to resolve.
- **Red flags**: Single backbone only (Qwen3-4B-Base); no validation on larger models or non-math domains beyond an appendix teaser. GRM call count scales with K, which the authors flag as a scalability bottleneck. The strongest numbers lean on a proprietary evaluator, and the chosen baseline Parallel-R1 is itself recent and arguably under-tuned (200 steps) — a fairer, fully-tuned Parallel-R1 comparison is missing.

## Strengths

- Principled and original formulation of path-level reward attribution via cooperative game theory, directly addressing a real reward-hacking failure mode in summary-based parallel reasoning.
- Monte Carlo permutation approximation makes Shapley computation tractable (O(1/√M) convergence) and the two-level (outcome + path) reward design cleanly separates correctness from marginal contribution.
- Strong training efficiency: 40 steps to match/exceed a 200-step baseline, with a cleaner training curve and earlier plateau.
- Thorough ablations — especially the same-evaluator contrast (Independent 46.5 vs Shapley 48.3) that rules out "it's just a stronger reward model" — and the path-masking complementarity analysis is a clever, intuitive diagnostic.

## Limitations

- Evaluated on a single backbone (Qwen3-4B-Base); no scaling-law evidence or larger-model validation.
- GRM query count grows with the number of paths K, creating a compute bottleneck at larger K; exact M and reward weights (λ_o, λ_p) are not reported in the main text.
- Documented Pass@16-vs-Mean@16 trade-off: the method raises the upper bound but does not consistently improve single-generation accuracy, and on AIME25 Mean@16 it underperforms Parallel-R1 (17.2 vs 17.7).
- Top results depend on the proprietary qwen-plus GRM; reproducibility of peak numbers is contingent on that API.
- Generalization beyond math reasoning rests on an appendix-only multi-hop QA teaser, not a full evaluation; code generation is left to future work.

## Relevance

对关注 LLM 推理训练（尤其是 GRPO/RLVR、test-time scaling、多路径/并行推理）的研究者高度相关：它提供了一个把合作博弈论归因引入强化学习奖励设计的新范式，直接切中"路径级细粒度奖励"与"奖励归因"两个热门问题。若你正在做 RL for reasoning、reasoning trajectory 分析或 reward shaping，这篇论文的方法论与消融设计都值得借鉴。

## Questions for Further Exploration

- How does Parallel Shapley scale beyond 4B — does the Pass@16-vs-Mean@16 trade-off widen, narrow, or invert on 14B/32B backbones?
- Can on-policy self-distillation (their planned direction) reliably convert the Pass@16 gains into Mean@16 gains without losing path diversity?
- How sensitive are the results to the GRM's calibration — would a reward-model trained specifically for coalition-level utility estimation outperform general instruct LLMs as the GRM?
- What is the wall-clock / dollar cost overhead of the M Monte Carlo permutations × 2^K-adjacent GRM calls per rollout versus plain Parallel-R1, and can stratified or regression-based Shapley approximation materially reduce it?
- Does the "free-rider" diagnosis hold for non-math domains (code generation, multi-hop QA, agentic tool use) where path complementarity is less obvious to define?
