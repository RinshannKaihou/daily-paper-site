Methodology mode → payload → procedure

A query enters with an explicit mode or is classified by a frozen four-mode semantic classifier (Claude Haiku 4.5, temperature 0) emitting SINGLE_FACT_LOOKUP, LONG_FORM_SYNTHESIS, TIME_ANCHORED_LOOKUP, or LATEST_VERSION_RESOLUTION. A payload map turns the label into concrete retrieval/synthesis flags; runtime gates (chain-shape detector, ICL corpus-shape detector, empty-retrieval fallbacks, intent/format rules) can revise the tier before a procedure reads from the shared substrate. Modes and executed procedures are deliberately not one-to-one.

mode = explicit_mode or classify(query)  →  payload = runtime_gates(mode_to_payload(mode), query, ctx)  →  procedure = procedure_for(payload.final_tier)

Three procedure families share one asynchronous substrate. Tier-1 ingest always writes chunks, embeddings, lexical fields and triples; async supplement jobs (doc shape, keypoints, event chains, entity types) are nullable — missing enrichments degrade gracefully rather than blocking readiness.

① Direct lookup
BM25 (k1=1.5, b=0.75) + chunk/sentence/paragraph dense retrieval fused by reciprocal-rank fusion (k=30); single Sonnet 4.5 synthesis with prompt-directed abstention.
② Graph / multi-hop
Triples ordered by descending version number so newer values win current-state queries; falls back to iterative dense retrieval when graph planning fails.
③ Long-form synthesis
Chronological sampling targeting 200 chunks; multipass bullets-expand procedure shaped by a source-shape classifier (narrative/research/technical/log/mixed).

Sub-dataset spread is wide: SH-Doc-QA 96.0% & MH-Doc-QA 85.0% up top; Movie-Rec 13.9% & InfBench-Sum 27.5% at the floor. All 73 failures in the local 85.4% LongMemEval reproduction had hit_at_k=true — evidence-use errors, not retrieval misses.

Strengths
  • Clean separation of mode (semantic label), payload (config) and procedure (execution family), with runtime gates that can revise the tier — useful for cost/quality/latency tuning.
  • Graceful degradation on a shared async substrate: missing optional enrichments never block query readiness.
  • Exceptionally honest evidence ledger — a dedicated appendix enumerates what the artifact does and does not support; refuses to overclaim.
  • Actionable failure strata per route and sub-dataset, pointing to where engineering effort should go.
  • Traceable numbers — models, prompts, retrieval params and artifact paths pinned in Appendix A.
Limitations
  • No fixed-policy controls — routing cannot be credited with the cross-workload profile; the central missing experiment.
  • Raw baseline outputs unavailable — parity, row-level corrections and paired uncertainty can't be independently verified (one-row correction applied only to SCM).
  • No aligned end-to-end latency beyond MAB (9.00 s/req @ concurrency 12); LoCoMo/LongMem Stage-3 duration and token usage not persisted.
  • Mostly single-run cells; no p-values, CIs, or effect sizes.
  • Vector store confounded (Mem0 = ChromaDB; SCM & comparator = pgvector); LM judges without a human-agreement study.

Questions for Further Exploration open

  1. How does SCM's routed configuration actually compare against a single fixed procedure (always-direct or always-long-form) on identical corpus snapshots with full routing provenance? This is the experiment that would establish a causal routing effect.
  2. What is the end-to-end latency and per-query dollar cost of each procedure family when scored-answer timing and token usage are retained?
  3. How much of the LongMemEval advantage (especially the +51.88-point temporal delta) survives a paired ChromaDB-vs-pgvector control holding embedding model, top_k, corpus scale and scoring path constant?
  4. Could prompt-directed abstention be replaced by a deterministic post-retrieval entity-grounded filter, and would the 66-point swing change once the disabled paired output is reproduced?
  5. Given all 73 LongMem failures have hit_at_k=true, what synthesis-side interventions (evidence-counting, preference-grounding, event-ordering) most reduce evidence-use errors?
  6. How robust is the frozen four-mode classifier to distribution shift, given LATEST_VERSION_RESOLUTION is essentially unvalidated on the real-dev slice and runtime gates already introduce a fifth mode?
