> **一句话总结：** 针对开放式对话中"修改回复会改变后续用户反馈"这一可验证性难题，本文提出将自演化目标从"生成更好的回答"转为"预测已记录的用户下一轮反馈（解决/未解决）"，使反馈预测技能可在固定离线日志上被验证式地优化；在字节跳动内部销售助手（私有、隐私保护）数据集上，经清洗与 1:1 平衡划分后，演化出的反馈技能取得超过 75% 的预测准确率（随机/常数基线约 50%）。

> **TL;DR:** The paper identifies the "moving-target" problem that blocks offline validation-gated evolution of open-ended dialogue skills (changing an answer invalidates the logged next-turn feedback), and reframes the optimization target from generating answers to *predicting* whether the logged answer led to resolved/adopted vs. unresolved feedback. Because the prediction target is fixed in the log, textual edits to the feedback skill can be gated on held-out accuracy. On a proprietary, privacy-preserving ByteDance sales-assistant dataset with a balanced ~1:1 split, the evolved feedback-prediction skill reaches **over 75% held-out accuracy** versus **~50% for a random/constant classifier** (a ~25-point gap). The authors explicitly do NOT report a downstream answer-skill A/B test or human evaluation, positioning the work as a formulation + industrial case study rather than a modeling breakthrough.

## 核心贡献

- **提出"移动目标"问题**：形式化地指出，在开放式对话中直接复用日志里"下一轮用户反馈"来离线优化回答技能是无效的——因为把回答 A 改成 A′ 后，原日志里的反馈 Q2 不再是对 A′ 的反应，即 p(Y|X,A) ≠ p(Y|X,A′)。这是把"对话技能自演化"无法离线验证的根本障碍讲清楚了。
- **提出"未来反馈技能演化"（future-feedback skill evolution）**：把第一个被演化的对象从"回答技能 S_A"改为"反馈预测技能 S_F"。预测任务以已记录的固定标签为参考，因此可在 held-out 集上被精确度量，从而支持 SkillOpt 式的验证门控文本优化（候选只在提升 held-out 分数时被接受）。
- **识别"反馈–生成对偶性"（feedback–generation duality）**：预测用户不满所必需的可解释规则（如"工具执行成功≠用户采纳"、"弱确认≠接受"、"泛泛建议≠可执行计划"），同样可作为改进回答技能的可操作诊断。这使得演化出的反馈技能既是评估器，又是回答质量知识库。
- **工业案例研究**：在字节跳动生产销售助手场景（隐私保护、匿名化）的清洗高质量子集上，以约 1:1 的解决/未解决平衡划分，演化反馈技能达到超过 75% 的预测准确率，并区分了"观测预测准确率"与"反事实保证"的边界。
- **诚实的边界声明**：明确把方法定位为离线优化阶段而非最终人评/在线评估的替代品；75% 是观测准确率，不能保证对任意反事实回答 A′ 都安全。

## Key Contributions

- **Formalizes the moving-target problem** that prevents logged next-turn feedback from serving as an unbiased validation signal for offline evolution of open-ended answer skills: revising A→A′ invalidates the logged reaction Q2 because p(Y|X,A) ≠ p(Y|X,A′).
- **Proposes validation-gated future-feedback skill evolution**: instead of first evolving the answer skill, evolve a feedback-prediction skill S_F that estimates whether the observed answer leads to a resolved/adopted (Y=1) or unresolved (Y=0) next signal. The held-out score J(S_F′) is measurable without changing the interaction, so every textual edit can be accepted or rejected reproducibly.
- **Identifies a feedback–generation duality**: the interpretable failure criteria required to predict dissatisfaction (e.g., tool success ≠ user adoption; weak acknowledgment ≠ acceptance) double as actionable diagnostics for answer generation — predictive failure criteria ⇒ candidate generation constraints.
- **Reports an industrial case study** on a proprietary, privacy-preserving ByteDance sales-assistant dataset, where quality filtering plus a balanced ~1:1 resolved/unresolved split yields over 75% prediction accuracy.
- **Explicitly separates observational verification from counterfactual validity**, positioning the method as an offline optimization stage that reduces (but does not eliminate) the need for final human/online confirmation.

## Methodology

The method has four components, layered on the SkillOpt text-space optimization framework [Yang et al., 2026].

**1. Problem formulation (logged feedback as moving target).** A logged interaction is x = (C, H, Q1, A, Q2), with context C, history H, current request Q1, assistant response A, and subsequent user signal Q2. A dataset D = {(Xi, Ai, Yi)} carries a binary label (Y=1 resolved/adopted, Y=0 unresolved). An answer skill S_A induces A ∼ pθ(A|X, S_A), but the true utility U(A′) = E[Y′|X,A′] depends on an *unknown* reaction, so a fixed log cannot unbiasedly score arbitrary revisions of S_A.

**2. A verifiable surrogate task.** The authors optimize a feedback skill S_F that predicts the *observed* label: Ŷ = fθ(X, A; S_F). The held-out score J(S_F′) = (1/|Dval|) Σ 𝟙[fθ(X,A;S_F′) = Y] is measurable without changing the interaction. This makes S_F's own evolution reproducible and rejectable on fixed data — the property needed for disciplined textual optimization — while explicitly not making it a perfect counterfactual oracle.

**3. Validation-gated skill evolution.** Starting from S_F^(0), the predictor produces a label and rationale per training example. At iteration t, an optimizer model receives high-value failures — especially *confident false positives* — and proposes a bounded textual edit Δ^(t): S̃_F^(t+1) = Edit(S_F^(t), Δ^(t)). Both current and candidate skills are evaluated on held-out data; the candidate is accepted only if it improves the primary objective, else rejected. A lexicographic variant can retain equal-accuracy candidates only when an asymmetric business utility (e.g., penalizing false positives more strongly) improves.

**4. Data construction.** Signals come from explicit user feedback, textual acceptance/rejection, and product-side behavioral events, mapped to a binary resolved/unresolved label. Silent termination with ambiguous meaning is excluded (trading noise for selection bias). Quality filters enforce clear turn boundaries, usable responses, and high-confidence subsequent labels; train/test partitions are constructed at approximately 1:1 resolved/unresolved to prevent majority-class accuracy from masking whether both outcomes are learned.

**5. Prediction rationales → answer guidance (conceptual bridge).** Learned rules (tool success ≠ adoption; greetings/recommendations/reports don't by themselves resolve; weak acknowledgments may be receipt not acceptance; parameter advice should specify an action plus an operational anchor like a value/interface/time window/re-evaluation condition; the response should cover the user's independent decision points) are transformed into diagnostics for S_A. The authors state they establish this bridge conceptually and via observed rules, but do *not* claim a separately measured end-to-end improvement of S_A.

## Results & Highlights

- **Primary metric (held-out accuracy):** On the balanced held-out set from the proprietary ByteDance sales-assistant dataset, the evolved feedback-prediction skill achieves **over 75% accuracy**, versus **approximately 50% accuracy for a random or constant classifier** on the same ~1:1 construction — a roughly **25-percentage-point gap** that the authors cite as evidence the skill extracts meaningful interaction-level signals rather than exploiting label prevalence.
- **Data construction (balance):** Train and test partitions are balanced to an **approximately 1:1 resolved/unresolved ratio**, making accuracy directly interpretable across both classes (a deliberate task-construction choice the authors treat as engineering, not a modeling contribution).
- **What the result establishes (three claims):** (i) next-feedback prediction is learnable from cleaned conversational records; (ii) textual skill edits can be evaluated and gated on fixed held-out data; (iii) the evolved artifact is interpretable enough to expose recurring answer defects.
- **Qualitative convergence evidence:** The predictor becomes *less* likely to equate fluent sales language, completed workflows, or successful tools with actual resolution, and *more* likely to require coverage, specificity, and an actionable next step.
- **Explicit non-result:** The paper does **not** report a human evaluation or online A/B test of an answer skill distilled from the feedback predictor; rationale-to-guidance transfer is presented as a supported mechanism and research direction, not a completed causal result.
- **Note on reporting granularity:** Because the dataset is proprietary, the authors report only the verified aggregate result and withhold sensitive counts, traffic statistics, and per-class metrics (no precision/recall/F1 breakdown is given).

## Authors & Credibility

- Both authors (ChaoJin Zhao, Xuan Jiang) are at **ByteDance**, giving direct access to the production sales-assistant scenario used in the case study — a credible industrial setting for this kind of logged-feedback research.
- The work builds explicitly on SkillOpt [Yang et al., 2026] (cited as arXiv:2605.23904) and positions itself carefully against the user-satisfaction estimation literature (USS benchmark [Sun et al., 2021], SPUR [Lin et al., 2024]) and future-dissatisfaction prediction (See and Manning, 2021).
- Credibility is strengthened by the authors' own boundary-setting: they repeatedly distinguish observational prediction accuracy from counterfactual guarantees and explicitly decline to claim downstream answer-skill improvement. This honesty is a positive signal, though it also reflects the thinness of the empirical contribution.

## Critical Assessment

- **Novelty:** The core idea — predict the logged next-turn outcome rather than score a counterfactual answer — is a clean reformulation, but the predictive target itself (will the user be resolved/unresolved next turn) is essentially the future-dissatisfaction prediction task of See and Manning (2021), which the authors themselves cite as "closely related." The genuine novelty is narrower: treating the predictor's *textual policy* as the object of SkillOpt-style validation-gated evolution and extracting its failure rationales as reusable skill knowledge. That is a useful but incremental reframing.
- **Rigor:** Empirically thin. Only **one** quantitative headline (>75% accuracy) on **one** proprietary dataset, with **no comparison to any learned baseline** — not SPUR, not an LLM-as-judge, not a model trained on the USS benchmark, only a trivial random/constant classifier at ~50%. There is no precision/recall/F1, no confusion matrix, no ablation isolating the effect of data cleaning vs. balance vs. skill edits, and no variance/confidence interval on the 75% figure. The paper itself concedes it "report[s] the verified aggregate result without disclosing sensitive counts or traffic statistics," which limits independent interpretation.
- **Reproducibility:** Poor. The strongest result depends on proprietary cleaning rules and a private production dataset; no code, no data, and no public benchmark evaluation are provided. The authors acknowledge this ("limits external reproduction") and suggest USS as a future cross-domain test, but do not perform it. The *formulation* is reproducible; the *result* is not.
- **Significance:** Modest. The feedback–generation duality is asserted conceptually and illustrated with qualitative rules, but is never measured end-to-end (no answer-skill A/B test or human study). So the paper demonstrates that a feedback-prediction skill can be evolved and gated offline, but not that doing so improves deployed answers — which is the practically important claim.
- **Red flags:** (i) The 75% number has no baseline beyond a constant classifier, so it is hard to judge whether a plain LLM zero-shot predictor on the same cleaned data would already reach it; (ii) "Confident false positives" drive the edits, yet no false-positive rate is reported; (iii) the decisive gains are attributed to "data cleaning" and "precise partitioning," which the authors themselves frame as task construction rather than method — raising the question of how much the 75% is the *evolution* versus the *curation*.

## Strengths

- The moving-target formulation is clearly and correctly articulated, with a precise statement of why p(Y|X,A) ≠ p(Y|X,A′) defeats naive offline validation of answer skills.
- The decision to hold the logged answer fixed and predict its observed outcome is a genuinely clean way to recover a verifiable offline objective in open dialogue — the core insight is well-motivated.
- Intellectual honesty about the claim boundary (observational vs. counterfactual; offline stage vs. final human/online validation) is a real strength and unusually disciplined for an industrial preprint.
- The qualitative learned rules (tool success ≠ adoption; actionable next step required; coverage of independent decision points) are operationally sensible and likely useful to practitioners building sales/support assistants.
- The feedback–generation duality, even if only conceptually established, is a productive framing for future work that automates rationale-to-guidance compilation.

## Limitations

- **Counterfactual distribution shift:** the predictor is validated on logged answers; optimizing the answer skill may generate out-of-distribution responses that exploit predictor weaknesses, so S_F is a validated observational model, not proof of satisfaction for arbitrary A′.
- **Silent users excluded:** interactions with no interpretable subsequent signal are dropped, so labels capture explicit/behavioral resolution more reliably than total user satisfaction (introduces selection bias).
- **Proprietary data and cleaning:** the strongest result depends on private production data and undisclosed cleaning rules; external reproduction is not possible, and no public benchmark (e.g., USS) is evaluated.
- **Binary feedback:** resolved/unresolved collapses graded, personalized satisfaction; a user can accept while partly dissatisfied, or reject a correct answer for reasons unrelated to response quality.
- **No downstream answer-skill experiment:** consistent with the paper's scope, no human evaluation or online A/B test of an answer skill distilled from the feedback predictor is reported, so the central practical claim (better answers) is unproven.
- **No learned baselines and no per-class metrics:** the only comparison is a random/constant classifier; precision, recall, F1, and confidence intervals are absent.

## Relevance

与"agent 技能自演化/文本空间优化"方向高度相关：它直接处理"如何在开放式对话这类缺乏稳定验证信号的任务上做验证式技能演化"，对构建可离线迭代、可检查、可复现的 LLM agent 技能体系有方法论参考价值。对关注对话系统评估、用户满意度预测、以及离线/在线评估边界划分的研究者尤其值得一看。

## Questions for Further Exploration

- How much of the >75% accuracy comes from the **skill evolution** versus the **data cleaning and 1:1 balancing**? An ablation reporting accuracy before vs. after evolution on the same curated split would isolate the method's contribution.
- How does the evolved feedback skill compare to strong **learned baselines** — an LLM-as-judge zero-shot predictor, SPUR-style rubric learning, or a classifier trained on the USS benchmark — on the same held-out data?
- Does distilling the feedback skill's rationales into answer-skill edits **actually improve** downstream answers in a controlled human study or online A/B test? The paper explicitly leaves this open.
- How robust is the predictor under **counterfactual distribution shift** when the answer skill generates out-of-distribution responses? Can the held-out gate detect or prevent predictor exploitation?
- How would the method perform on a **public** dataset (e.g., USS) with graded rather than binary satisfaction labels, and with the silent/ambiguous-turn population included rather than excluded?
