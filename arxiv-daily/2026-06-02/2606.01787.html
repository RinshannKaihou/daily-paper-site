Stochastic convergence of parallel asynchronous adaptive first-order methods

arXiv: 2606.01787

Date: 2026-06-02

https://arxiv.org/abs/2606.01787

一句话总结： 本文首次提出并分析了一类用于非凸优化的随机异步自适应一阶优化方法（涵盖 AdaGrad、Shampoo、Muon 等），在合理假设下证明了其全局收敛速率为 O(1/√t)，并通过数值实验验证了其在异构大规模机器学习系统中的有效性。
TL;DR: This paper introduces and analyzes the first class of stochastic asynchronous adaptively preconditioned first-order methods (covering AdaGrad, Shampoo, Muon, etc.) for nonconvex optimization, proving an O(1/√t) global convergence rate under reasonable assumptions, with empirical validation on heterogeneous large-scale ML systems.

Key Contributions

  • First adaptive preconditioning in asynchronous optimization: The ASADPREC framework extends successful synchronous adaptive preconditioners (AdaNorm, AdaGrad, Shampoo, Muon) to the asynchronous parallel setting, filling a significant theoretical gap.
  • Nonconvex stochastic convergence theory: Under reasonable assumptions (bounded delays, Lipschitz gradient, unbiased gradient oracle), the paper proves a global convergence rate of O(1/√t) (up to logarithmic factors) in a fully stochastic nonconvex setting.
  • Complete analysis of two momentum variants:
  • MM (momentum on momentum): Maintains the O(1/√t) rate without requiring a small stepsize assumption.
  • MG (momentum on gradient): Requires a sufficiently small stepsize or bounded iterates; convergence rate depends jointly on gradient variance and momentum parameters.
  • Inexact normalization extension: The theory accommodates inexact step normalization strategies, which is practically important for methods like Muon that rely on Newton-Schulz iterations.
  • Heterogeneous system validation: Empirical evaluation across 6 benchmarks with diverse architectures shows ASADPREC achieves the best loss on 4/6 benchmarks and best generalization on 5/6, demonstrating practical advantages in convergence quality and throughput.

Methodology

The paper proposes ASADPREC (ASynchronous ADaptive PREConditioned gradient), a block-parallel asynchronous optimization framework. The methodology consists of several key components:

Problem Structure and Geometry: The parameter space is decomposed into L disjoint blocks (e.g., layer weights and biases in neural networks), each equipped with its own norm ∥·∥ℓ and dual norm ∥·∥ℓ,*. This totally separable geometry allows different minimization methods for different variable types.

Algorithmic Framework: Each block ℓ is associated with a collection of methods Mℓ = {(U, N)}, where U is a preconditioner update operator and N is a normalization operator. The abstract framework covers AdaNorm, full/diagonal AdaGrad, Shampoo, and Muon. Each processor repeatedly:

  1. Reads the shared parameter vector (allowing inconsistent reads across blocks)
  2. Selects a block and computes its stochastic gradient
  3. Updates the block's preconditioner using U
  4. Applies a normalized preconditioned step using N

Analytical Bridge — DADPREC: To enable convergence analysis, ASADPREC is shown to be equivalent to a synchronous delayed framework called DADPREC (Delayed ADaptive PREConditioned), where each iteration selects an active block set and computes complete steps using delayed approximate block gradients. This equivalence allows leveraging prior synchronous convergence theory.

Momentum Variants: Two momentum definitions are studied:

  • ASADPREC.MM / DADPREC.MM: Momentum accumulates past momentum values (Mt,ℓ = µ·Mprev + (1−µ)·G). The preconditioner is updated using the momentum itself.
  • ASADPREC.MG / DADPREC.MG: Momentum accumulates past gradients (Mt,ℓ = µ·Mprev + (1−µ)·G), but the preconditioner update uses the raw gradient.

Approximate Normalization: The theory is extended to allow approximate normalization where ∥N(Z)∥ ∈ [κ₁,ₙ, κ₂,ₙ], which covers inexact Newton-Schulz iterations used in Muon/Shampoo without changing the convergence order.

Results & Highlights

Theoretical Results:

  • Theorem 3.1: Under Assumptions 1–7 (boundedness, smoothness, structural identities, gradient-preconditioner compatibility, unbiased oracle, cumulative variance, bounded delays), DADPREC achieves:

$$

\min_{j \in \{0,...,t\}} E[\|G_j\|_*] \leq \frac{\kappa_a + \kappa_b \sqrt{N \log(\Theta_t)} + \kappa_c \Theta_t}{\sqrt{t+1}}

$$

where Θₜ grows very slowly with the cumulative variance bound νₜ.

  • Corollary 3.2: When block gradient variance scales as σℓ² / |Jt,ℓ|^α, the rate becomes:
  • O(ψt^(1−α)/2 / √(t+1)) if α < 1
  • O(√log(ψt) · log(log(t+1)) / √(t+1)) if α = 1
  • O(1/√(t+1)) if α > 1

where ψt is the maximum number of updates to any block. When |At| ≪ L, ψt ≪ t+1, suggesting acceleration over the undelayed case.

  • Theorem 4.1: DADPREC.MM preserves the same convergence rate as the momentum-less variant without requiring small stepsize.
  • Theorem 4.2 & Corollary 4.3: DADPREC.MG requires either η ≤ √(ς(1−µmax)/(6κ₂κ⋄τL)) or bounded iterates. The rate depends on α + 2β where β controls momentum decay; choosing µt,ℓ ∝ |Jt,ℓ|^(−α/4) recovers the momentum-less rate even in high-variance regimes.

Critical Assessment

Novelty: High. This is the first work to unify adaptive preconditioning (AdaGrad, Shampoo, Muon) with asynchronous parallel optimization under a single theoretical framework for nonconvex problems. Prior works either studied non-adaptive asynchronous methods or adaptive synchronous methods, but not both.

Rigor: Strong. The paper provides detailed assumptions, rigorous proofs (deferred to appendices), and careful distinctions between the two momentum variants. The use of the DADPREC equivalence to bridge asynchronous execution with synchronous analysis is elegant.

Reproducibility: Moderate. The theoretical results are well-specified. The code is not publicly attached, but the experimental setup is described in sufficient detail (PyTorch, Apple M3 Pro, per-block locking, 8 seeds). The benchmarks are standard public datasets.

Significance: Potentially high for the optimization community. The O(1/√t) rate for a broad class of asynchronous adaptive methods is a solid theoretical contribution. The practical relevance depends on whether the "full gradient evaluation model" (where each worker computes the full gradient) can be relaxed to cheaper block-gradient computations, which the authors identify as a key future direction.

Red flags / concerns:

  • The totally separable geometry assumption (no overlap between blocks) is theoretically restrictive, though it covers important cases like layer-wise preconditioning.
  • The bounded delay assumption (τt,ℓ ≤ τ) is standard but may not capture all real-world distributed behaviors.
  • The experiments use CPU-based shared memory rather than multi-GPU distributed settings, which limits direct claims about large-scale distributed training.
  • For DADPREC.MG, the requirement of a small enough stepsize (involving the unknown Lipschitz constant LG) is a common theoretical limitation.

Strengths

  • First unified theory for asynchronous adaptive preconditioned methods in the nonconvex stochastic setting.
  • Broad algorithmic coverage: A single framework encompasses AdaNorm, AdaGrad, Shampoo, and Muon.
  • No bounded gradient assumption required for the main convergence result, unlike many prior asynchronous methods.
  • Clear momentum taxonomy: The distinction between MM and MG variants is theoretically motivated and practically consequential (especially for computational cost of Muon).
  • Well-designed experiments: Explicitly tests the relationship between heterogeneity, speedup, and convergence quality.
  • Inexact normalization extension: Makes the theory applicable to practical implementations using iterative matrix approximations.

Limitations

  • Totally separable geometry: Blocks must be disjoint, ruling out overlapping parameter structures.
  • Full gradient evaluation model: Each worker computes the full gradient before extracting its block component. If cheaper block-gradient computation were possible, heterogeneity and speedups would be more dramatic.
  • CPU-only experiments: Multi-GPU distributed validation would strengthen practical claims.
  • MG momentum requires small stepsize: The theoretical guarantee for DADPREC.MG depends on an often-unknown Lipschitz constant.
  • Problem-dependent momentum: No clear prescription for choosing between MM and MG variants; performance varies across benchmarks.
  • Limited benchmark scope: Six relatively small-scale problems; no large language model or very deep network training results.

Relevance

对于关注大规模深度学习系统优化效率的研究者来说，本文具有重要的理论和实践价值：它首次为异步自适应预条件优化方法提供了严格的非凸收敛保证，并明确指出在异构计算环境中（不同层或不同模块计算成本差异显著时），去除同步屏障能够带来实质性的吞吐量和收敛质量提升。特别地，其 MM 动量变体在保持理论保证的同时避免了昂贵的核范数计算，对实际部署 Muon 等矩阵自适应优化器具有直接指导意义。

Questions for Further Exploration

  1. Can the full-gradient evaluation requirement be relaxed? The paper notes that if block gradients could be computed more cheaply than full gradients, heterogeneity ratios would increase dramatically. What architectures or automatic differentiation techniques would enable this?
  2. How does ASADPREC perform in multi-GPU distributed settings? The current CPU-based shared-memory experiments are encouraging but do not capture network communication costs, gradient compression, or device heterogeneity.
  3. Is there a principled way to choose between MM and MG momentum? The empirical results show problem-dependent behavior. Can problem characteristics (condition number, gradient variance structure) predict which variant will perform better?
  4. Can the totally separable geometry assumption be relaxed? Many practical models have partially separable or overlapping block structures. Can the theory be extended to partially separable problems like the original Hogwild! setting?
  5. What is the interaction between asynchronous adaptive methods and learning rate scheduling? The current theory uses a fixed stepsize η. How do common schedules (warmup, cosine decay) interact with the delay and preconditioner growth dynamics?
  6. Can the convergence rate be improved for over-parameterized regimes? The paper notes that the "preconditioned cumulative strong growth" assumption could yield improved rates. Can explicit rates be derived under interpolation or strong growth conditions?
