Learning When Not to Act: Mitigating Tool Abuse in Agentic Reinforcement Learning

arXiv: 2606.02132

Date: 2026-06-02

https://arxiv.org/abs/2606.02132

一句话总结： 本文提出EAPO框架，通过在训练 rollout 中引入无工具轨迹、难度感知奖励塑造和置信度感知token重加权，帮助智能体学会"何时不使用工具"，在9个推理基准上同时提升准确率并减少约20%的工具调用。
TL;DR: This paper proposes EAPO, an agentic RL framework that mitigates tool abuse by introducing tool-free rollouts, difficulty-aware reward shaping, and confidence-aware token reweighting, achieving higher accuracy with ~20% fewer tool calls across nine reasoning benchmarks on Qwen and Llama models.

  • EAPO联合优化框架：将难度感知奖励塑造与置信度感知token级重加权相结合，在抑制不必要工具调用的同时，保留对困难查询的工具辅助探索能力。
  • 广泛的实验验证：在数学推理（AIME24/25、MATH500、GSM8K、MATH）和知识密集型推理（HotpotQA、2WikiMultihopQA、MuSiQue、Bamboogle）共9个基准上，EAPO在Qwen2.5-3B/7B和Llama3.1-8B上均稳定提升准确率并显著降低工具调用次数。

Key Contributions

  • Efficiency-Aware Rollout Mechanism: Ensures at least n tool-free trajectories in each rollout group of M trajectories, providing explicit contrastive signals for the policy to distinguish necessary tool use from excessive tool dependence.
  • EAPO Joint Optimization Framework: Combines difficulty-aware reward shaping with confidence-aware token-level reweighting to suppress unnecessary tool calls while preserving tool-assisted exploration for genuinely hard queries.
  • Extensive Empirical Validation: Evaluated on nine benchmarks spanning mathematical reasoning (AIME24/25, MATH500, GSM8K, MATH) and knowledge-intensive reasoning (HotpotQA, 2WikiMultihopQA, MuSiQue, Bamboogle), EAPO consistently improves accuracy and reduces tool calls across Qwen2.5-3B/7B and Llama3.1-8B.
  • Difficulty-Adaptive Efficiency Optimization: Proposes a dynamic reward adjustment mechanism based on query difficulty, avoiding overly aggressive penalties on challenging problems and preserving the value of tool-augmented reasoning.

Methodology

EAPO (Efficient Agentic Policy Optimization) is built on the standard agentic RL objective that maximizes expected reward under a policy π_θ with KL regularization. It comprises three core components:

1. Efficiency-Aware Rollout

In each rollout group of M=16 trajectories, EAPO guarantees that at least n=2 trajectories do not invoke any tools. This is enforced by modifying prompts to prohibit tool usage and disabling tool execution in the rollout module. By explicitly contrasting tool-free and tool-augmented trajectories, the model learns whether tool usage yields genuine performance improvements for a given query.

2. Difficulty-Aware Reward Shaping

EAPO defines query difficulty as d(q) = 1 − (1/M) Σ I(T_i), where I(T_i) indicates whether trajectory T_i is correct. The shaped reward is:

R̃_i = [d(q) + (1 − d(q)) · ε(T_i)] · R_i

The efficiency coefficient ε(T_i) = exp(−β(c_i − c_min)) penalizes tool calls, where c_i is the number of tool calls in trajectory T_i and c_min is the minimum tool calls among correct trajectories. When query difficulty is high (few correct trajectories), the penalty is naturally attenuated, preserving exploration. When the query is easy (many correct trajectories), unnecessary tool calls are strongly penalized.

Key Findings

  • Simple prompt-based TIR performs no better (sometimes worse) than autonomous reasoning, confirming that prompting alone is insufficient for multi-tool reasoning.
  • Efficiency gains are smaller on 3B models than on 7B/8B models, as smaller models have less redundant tool use to eliminate without hurting performance.
  • Hyperparameter analysis (Figure 4) shows EAPO is robust across a broad range of β (tool penalty); the difficulty-aware coefficient naturally reduces sensitivity to β when accuracy is low.
  • Training convergence speed of EAPO is nearly identical to baseline methods (Figure 5a–b), and inference token usage is comparable or lower (Figure 5c), confirming that efficiency gains do not come at the cost of longer reasoning chains.

Ablation Study (Table 3)

Removing cold-start SFT causes the largest performance drop. Removing reward shaping increases tool calls sharply on QA tasks (e.g., HotpotQA TC rises from 1.99 to 3.49). Removing advantage reweighting makes the policy more conservative, slightly reducing tool calls but also hurting accuracy.

Critical Assessment

Novelty: The core novelty lies in the difficulty-aware formulation of tool-use penalties and the explicit introduction of tool-free trajectories into rollout groups. While prior work (OTC, R1-Searcher++) has recognized tool abuse, they apply uniform penalties or rely on knowledge internalization without explicit policy-level control. EAPO's dynamic penalty based on query solvability without tools is a meaningful advance.

Rigor: The paper provides formal definitions for tool abuse, query difficulty, efficiency coefficients, and confidence-aware reweighting. Experiments include comprehensive baselines, ablations, hyperparameter sensitivity analysis, and efficiency analysis (training convergence + inference tokens). The use of both Pass@1 (math) and F1/EM (QA) metrics is appropriate.

Reproducibility: Implementation details are reasonably detailed: exact model checkpoints, training hyperparameters (batch size 56, M=16, n=2, β=1, W=1.5, ρ=0.2), SFT cold-start data composition, and evaluation protocols (temperature 0.6, top-p 0.95, Qwen2.5-72B as judge). However, the paper does not release code or training data, and some design choices (e.g., why n=2 specifically) are not deeply justified.

Significance: Tool abuse is a practically important problem for deployed agents where API calls incur latency and cost. EAPO demonstrates that agents can learn selective tool use without sacrificing performance, which has clear deployment value.

Red Flags:

  • The "Limitations" section is unusually brief and essentially restates the paper's scope rather than acknowledging genuine weaknesses.
  • The claim that benchmarks "predominantly focus on hard problems" is accurate but somewhat self-serving—it would strengthen the paper to include an easy-to-hard evaluation.
  • Confidence-aware reweighting is inspired by prior work (Li et al., 2025a; Wang et al., 2025b); the incremental contribution of this component beyond prior formulations could be clearer.

Strengths

  • Practically motivated problem: Tool abuse is a real deployment issue (cost, latency), and the paper tackles it head-on with a principled RL approach.
  • Strong empirical results: Consistent improvements across 9 benchmarks and 3 model families, with both accuracy gains and tool-call reductions.
  • Well-designed ablations: Each component (cold start, reward shaping, advantage reweighting) is individually validated.
  • Robustness: The method is insensitive to the exact choice of β across a broad range, reducing the tuning burden.
  • Bilingual relevance: The work is directly relevant to researchers and engineers building Chinese- and English-language agents with tool-use capabilities.

Limitations

  • Benchmark bias toward hard problems: Existing datasets emphasize complex reasoning, making it harder to measure tool abuse on easy queries where internal reasoning should suffice.
  • Limited to two tool types: The framework is evaluated on search and Python tools only; generalization to other tool types (databases, APIs, multimodal tools) is not demonstrated.
  • No code release: Without open-source implementation, exact reproduction is difficult.
  • Small tool-free trajectory count: Only n=2 out of M=16 trajectories are tool-free; the sensitivity of this ratio is not deeply explored.
  • Confidence reweighting is incremental: The token-level confidence reweighting strategy builds closely on prior work, and its unique contribution to the overall gains is not isolated.

Relevance

对于构建需要平衡推理准确率与工具调用成本的生产级智能体系统的研究者和工程师而言，本文具有直接参考价值。EAPO所提出的难度感知惩罚和显式无工具rollout机制，为在GRPO等现有方法基础上实现更高效、更可控的工具使用策略提供了清晰的技术路径。

Questions for Further Exploration

  1. How would EAPO perform on an explicitly constructed easy-to-hard benchmark, where simple commonsense or arithmetic queries are mixed with complex multi-hop reasoning tasks?
  2. Can the efficiency-aware rollout mechanism be extended to more than two tool-free trajectories (e.g., adaptive n based on model size or training stage), and what is the optimal trade-off?
  3. How does EAPO generalize to procedural or multimodal tools (e.g., image generation, database querying) where "difficulty" may be harder to define?
  4. Can the difficulty-aware reward shaping be combined with length-aware efficiency regularization (e.g., GRPO-LEAD) to jointly optimize tool use, reasoning length, and accuracy?
  5. What is the long-term stability of EAPO-trained policies when deployed in continuous online learning settings where query distributions shift over time?
