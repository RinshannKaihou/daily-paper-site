Tracking the Behavioral Trajectories of Adapting Agents

arXiv: 2606.02536

Date: 2026-06-02

https://arxiv.org/abs/2606.02536

一句话总结： 本文提出了一种将智能体特质（trait）建模为文本嵌入空间中线性方向的方法，通过对比技能文件修改前后的嵌入差异来量化行为变化，并在数据获取倾向特质上达到91.2%的分类准确率，同时设计了智能体间互评协议以实现去信任化的行为轨迹追踪。
TL;DR: This paper introduces a method that models agent traits as linear directions in text embedding space, quantifying behavioral changes by comparing embedding differences of skill file edits before and after modification. It achieves 91.2% sign classification accuracy on a data-seeking trait and proposes an agent-to-agent evaluation protocol for trustless behavioral trajectory tracking.

核心贡献

  • 基于嵌入空间的特质向量方法：将智能体特质定义为文本嵌入模型空间中的方向，通过对标记的"修改前"与"修改后"技能文件差异进行岭回归训练，学习得到可解释、可复用的特质向量。
  • 数据获取倾向特质的验证：在68组带标签的技能文件修改对上评估"数据获取倾向"特质，留一法交叉验证达到91.2%的符号分类准确率和Spearman秩相关系数ρ = 0.82。
  • 智能体间互评协议：设计了由可信运行时服务器中介的三方协议，使一个智能体能够在不直接访问另一智能体源文件的情况下，独立评估其技能文件更新的行为倾向变化。
  • 与前沿方法的基准对比：相比基于YARA规则的签名基线（63.2%准确率），本文的语义嵌入方法显著提升了性能；同时相比GPT-5.4（100%准确率），本方法在保持确定性、可审计性和实时性方面具有部署优势。

Key Contributions

  • Trait-vector methodology in embedding space: Defines agent traits as directions in the embedding space of a text embedding model. Learns a trait vector via Ridge regression on labeled "before" versus "after" skill file diffs, yielding an interpretable and reusable scoring mechanism.
  • Validation on data-seeking trait: Evaluated on 68 labeled skill diff pairs for the propensity to seek sensitive data, achieving 91.2% sign classification accuracy and Spearman rank correlation ρ = 0.82 under leave-one-out cross-validation.
  • Agent-to-agent evaluation protocol: Proposes a three-party protocol mediated by a trusted runtime server, enabling one agent to evaluate another's behavioral tendencies from skill file updates without requiring direct access to source files or scoring authority.
  • Benchmarking against baselines: Substantially outperforms a YARA signature baseline (63.2% accuracy) by leveraging semantic context. Offers deterministic, auditable, and low-latency scoring compared to frontier LLM evaluation (GPT-5.4 at 100%), making it suitable for continuous monitoring at scale.

Methodology

The core insight of the paper is inspired by representation engineering (Zou et al., 2023): high-level behavioral properties can be encoded as linear directions in a model's representation space. However, rather than probing internal LLM activations, the authors learn trait directions in the embedding space of a text embedding model applied to agent source files.

Trait Vector Learning: Given a set of file pairs $(B_i, A_i)$ where $B_i$ is the "before" version and $A_i$ is the "after" version, together with continuous labels $y_i \in [-1, 1]$ indicating the magnitude and direction of trait change, the method proceeds as follows:

  1. Embed each text file with embedding model $E$ and normalize to unit length: $\hat{e} := e / \|e\|$.
  2. Compute the diff vector $d_i = E(A_i) - E(B_i)$, then normalize to obtain $\hat{d}_i$.
  3. Fit a Ridge regression from $\hat{d}_i$ to $y_i$. The resulting coefficient vector $w$ is the trait vector.

At inference, scoring a new edit involves embedding the before and after files, computing the normalized diff, and taking the dot product $\hat{d} \cdot w + b$, where $b$ is the Ridge intercept.

Embedding Model and Data: The authors use Qwen3-Embedding-8B (Zhang et al., 2025), an instruction-aware text embedding model producing 4096-dimensional vectors. An embedding instruction is prepended to focus the representation on security audit relevance: "Represent this skill documentation for a security audit, focusing on whether it instructs the agent to retrieve, exfiltrate, or solicit credentials, secrets, tokens, or private user data."

The training dataset consists of 63 publicly available agent skills from the Awesome Copilot repository as "before" versions. For each, synthetic "after" versions were created that either increase or decrease the data-seeking trait. In 5 cases, both directions were created for the same skill, yielding 68 before/after pairs in total. Labels range from $-1$ (strongly data-secure) to $+1$ (strongly data-seeking).

Validation: The authors use Ridge regression with closed-form leave-one-out cross-validation (LOOCV) via the PRESS statistic. Two metrics are reported: (1) sign accuracy — the fraction of diffs for which the model correctly predicts the direction of trait change; and (2) Spearman rank correlation ($\rho$) — measuring how well the model's continuous predictions preserve the rank ordering of labels by severity.

Results & Highlights

On the 68-pair dataset under LOOCV, the model correctly classified the direction of trait change in 91.2% of cases, with a Spearman rank correlation of $\rho = 0.82$.

Error Analysis: Of the 6 misclassified pairs, all have small absolute predicted values (mean $|\hat{y}| = 0.085$), indicating that the model is uncertain rather than confidently wrong. These misclassified pairs also have lower-magnitude true labels ($|y| = 0.32$) compared to correctly classified pairs ($|y| = 0.42$), showing that errors concentrate where the true trait change is small and the sign distinction is hardest to draw.

Baseline Comparison:

  • YARA Signature Baseline: Implemented rules matching acquisition/exfiltration verbs near sensitive-data terms and separate safeguard rules. Achieved only 63.2% sign accuracy, with 14 diffs firing no rule at all. This gap demonstrates the value of semantic modeling over signature-based approaches.
  • Frontier LLM Baseline: GPT-5.4 achieved 100% sign accuracy when prompted to classify the same 68 diffs. This places the proposed method between signature rules and frontier LLMs: substantially better than YARA while retaining deterministic, fast, and auditable scoring suitable for continuous monitoring.

Deployment Result: In a live deployment using a Hermes agent, the system flagged its largest trait diff when substantial SSH and VM provisioning functionality was added to a skill—an intuitively sensible result demonstrating practical utility.

Critical Assessment

Novelty: The paper presents a creative adaptation of representation engineering principles to the domain of agent skill file monitoring. While the idea of linear directions in embedding space is not new, its application to diff-based behavioral trajectory tracking of autonomous agents is a novel and practical contribution.

Rigor: The methodology is sound. Using normalized embedding diffs and Ridge regression with LOOCV is appropriate for the dataset size. The error analysis is thoughtful—highlighting that misclassifications occur on low-magnitude, uncertain cases rather than confident errors. However, the dataset is small (68 pairs), synthetic (hand-crafted "after" versions), and limited to one trait and one skill repository, constraining generalizability claims.

Reproducibility: The approach is highly reproducible in principle. The embedding model (Qwen3-Embedding-8B) is specified, the regression method is standard, and the trait vector is fixed and inspectable. However, the exact synthetic edits and labeling process are not fully detailed, and the dataset is not released, which limits independent replication.

Significance: The work addresses a timely and important problem—monitoring agent behavior as agents autonomously modify their own skill files. The agent-to-agent protocol is a practical contribution toward scalable multi-agent trust. The tradeoff between deterministic, fast scoring (this method) and higher-accuracy but expensive frontier LLM evaluation is well-articulated.

Red Flags:

  • The "after" versions are synthetically generated, which may not reflect the distribution of real-world agent self-modifications.
  • Some labels were generated by Claude Opus 4.6 and then human-reviewed, introducing potential bias.
  • The paper does not evaluate whether the trait vector generalizes across different embedding models or agent frameworks.
  • The deployment example is anecdotal (one flagged diff) rather than a systematic evaluation.

Strengths

  • Deterministic and auditable: Unlike LLM-based evaluators, the trait vector produces identical scores for identical inputs, supporting reproducibility and policy enforcement.
  • Computationally lightweight: Embedding and dot-product scoring is fast and cheap, enabling real-time or near-real-time monitoring across many agents and skills.
  • Diff-based design: Training on file diffs rather than whole files isolates trait-relevant signal from unrelated content.
  • Well-justified protocol: The agent-to-agent protocol thoughtfully separates execution (embedding) from scoring to prevent either party from manipulating results.
  • Strong error analysis: The authors carefully analyze where and why the model fails (low-magnitude, near-zero predictions), lending credibility to the results.

Limitations

  • Narrow scope: Only one trait (data-seeking) is evaluated, on 63 public skills from a single repository (Awesome Copilot). Generalization to other traits, private agent skills, and diverse agent frameworks is unverified.
  • Synthetic edits: The "after" versions are manually or semi-automatically crafted rather than naturally occurring, potentially creating a distribution shift relative to real agent self-modifications.
  • Single-skill focus: The core unit is a single skill update; aggregating per-skill scores into agent-level risk estimates is mentioned as future work.
  • Trust assumptions: The embedding model and runtime server are trusted intermediaries. The protocol does not address an adversary that can substitute fake skill trees or hide unassessed files.
  • Labeling methodology: A portion of labels were LLM-generated before human review, which may introduce systematic biases.
  • Modest dataset size: 68 pairs, while sufficient for a proof-of-concept, is small for training robust generalizable trait vectors.

Relevance

随着智能体系统（如Claude Code、Hermes）越来越依赖可变的技能文件和记忆文件来定义行为，持续监控这些文件的行为倾向变化对于确保AI安全至关重要。本文提出的嵌入空间特质追踪方法和智能体间互评协议，为构建可扩展、可审计的多智能体信任基础设施提供了一个务实且可落地的技术路径。

Questions for Further Exploration

  1. How would the trait-vector approach perform on naturally occurring agent self-modifications rather than synthetic edits? Would the distribution shift significantly degrade performance?
  2. Can the method be extended to detect emergent traits that were not explicitly defined during training, or does it strictly require pre-specified trait definitions?
  3. What is the optimal strategy for aggregating per-skill trait scores into an agent-level behavioral risk profile, and how should temporal dynamics be modeled?
  4. How robust is the learned trait vector to adversarial perturbations in skill file text designed to evade detection while preserving malicious intent?
  5. Could the agent-to-agent protocol be hardened with cryptographic commitments (e.g., Merkle trees over file sets) to prevent Agent B from hiding or substituting files during evaluation?
