Unlocking the Working Memory of Large Language Models for Latent Reasoning

RiM replaces chain-of-thought's autoregressive token generation with fixed-length memory blocks processed in a single forward pass. A two-stage curriculum trains LLMs to perform structured internal computation — no token-by-token generation required.

Method

Memory Blocks + Custom Attention + Two-Stage Curriculum

Question
[<m>...<m>] Block 1
[<m>...<m>] Block K
Readout → Answer

Memory Blocks are fixed-length sequences of K special <m> tokens inserted between question and answer. Unlike Coconut (which autoregressively generates continuous thought tokens), RiM's blocks have fixed token identities — enabling all K blocks to be processed in a single forward pass with a custom attention mask.

Custom Attention Mask ensures memory blocks attend to the question and previous blocks; readout positions attend to all blocks seen so far but not to other reasoning steps. This forces reasoning into the memory blocks and enables dense supervision in one pass.

Stage 1 (Grounding): Each readout is supervised to predict the next explicit CoT step — forcing the latent workspace to encode task-relevant intermediate information. Stage 2 (Refinement): Step-level supervision is removed; each readout predicts the final answer directly, shifting to iterative refinement across blocks.

Strengths

Why RiM Works

  • Elegant design grounded in cognitive science (working memory, Vygotsky's developmental psychology)
  • Dramatic speedup: same TTFT as direct-answer, ~27× faster than CoT
  • Custom attention mask enables dense, efficient training in a single forward pass
  • Consistent gains across GPT-2 and Llama families at 124M to 3B scale
  • Memory budget robustness: accuracy stable across varying K and M after Stage 2
  • From Hochreiter's lab (co-inventor of LSTM) — strong credibility
Limitations

Caveats and Gaps

  • 18pp gap vs. explicit CoT at 3B scale (48.8% vs 66.9%) — latent reasoning underperforms when more capacity is available
  • Evaluated only on GSM8K and GSM-Hard — no MATH, AIME, code generation, or logical reasoning
  • No comparison to RL-based reasoning (DeepSeek-R1 style) or other latent methods (DART, recurrent approaches)
  • Fixed memory block structure may limit adaptability to problems of varying difficulty
  • Interpretability is reduced — the latent workspace is harder to inspect than explicit CoT
  • Checkpoint selection uses cross-validation on GSM8K, the same benchmark used for evaluation
Open Questions

Where Does This Go Next?

  • How does RiM perform on harder benchmarks (MATH, AIME, code generation, logical reasoning) where CoT benefits are most pronounced?
  • Can RL with final-answer rewards in Stage 2 close the gap with explicit CoT at larger scales?
  • What computation actually happens inside memory blocks — can probing or interpretability techniques reveal the latent structure?
  • Can hybrid approaches combining RiM blocks with selective autoregressive generation achieve the best of both worlds?
  • Does RiM scale to 7B+ and 70B+ models where CoT reasoning is most impactful?
  • What is the effect of varying K and M on harder tasks — is there a sweet spot or does it scale smoothly?
