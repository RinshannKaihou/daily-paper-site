Perturbation-Induced Bias Is the Hidden Driver of LLM Fragility

Simple, retraining-free debiasing on logits or intermediate features recovers up to 67% of performance lost to semantically-neutral prompt variations — with formal robustness certificates.

Zhou · Aleshina · Lovyagin · Somov · Seleznyov · Panchenko · Oseledets · Tutubalina · Tyukin  |  UUST / King's College London / MIIRAL

Key Finding
Perturbation-
Induced Bias
Systematic expected shift in module outputs under random prompt perturbations is the dominant factor behind non-adversarial LLM fragility.
Key Metric
≤67.8%
Performance recovery after debiasing. Certification rates rise from 9.4% to 63.8% per-example on hold-out data.
Why This Matters
Zero Retraining
Closed-form debiasing adds negligible inference cost and gives formal robustness guarantees — no labels, no fine-tuning required.
Critical Edge
Cherry-Picked
Task filtering selects only tasks with >1% drop and >70% clean BAC, favoring scenarios where debiasing has room to help. Generation results lack error bars.

⚙ Method Pipeline

Theoretical framework identifies perturbation-induced bias E[M(x+dx) - M(x)] as the key factor, then removes it via two lightweight interventions:

Step 1
Sample Perturbations
Generate 200 semantically-neutral variants per input prompt (format + text perturbations)
Step 2
Estimate Bias
Compute expected shift in logits or intermediate activations across perturbations
Step 3
Debias
Input-independent: constant subtraction. Input-dependent: ridge regression correction on input features
Step 4
Certify
Concentration inequalities yield per-example and population-level robustness radii

Input-dependent debiasing across 9 Natural Instructions tasks on 3 models (7-8B params):

⚠ Red Flags & Concerns

▲ Task filtering bias. Only tasks with >1% performance drop AND >70% clean BAC are kept — cherry-picking scenarios favorable to the method.
▲ Small evaluation scale. 200 examples per task with 200 perturbations each — limited statistical power.
▲ White-box only. Requires access to model weights/logits; not applicable to API-only deployments.
▲ Input-independent debiasing often fails. No valid overlap region across all three models, undermining universality claims.

✓ Strengths

  • Elegant theoretical framing that isolates a clear, actionable mechanism behind LLM fragility.
  • Extremely lightweight: closed-form solutions, no retraining, negligible inference overhead.
  • Provides formal robustness certificates, not just empirical improvements.
  • Compatible with LoRA fine-tuning and in-context learning — additive, not competing.
  • Honest about when debiasing does not help (Figure 2 scenarios explicitly discussed).

❓ Open Questions

  1. Can the bias framework extend to adversarial perturbations and worst-case certificates?
  2. How does debiasing effectiveness scale to 70B+ parameter models?
  3. Systematic validation of intermediate-layer PCA debiasing on code generation, translation, summarization?
  4. Connection between low intrinsic dimensionality (4-5 PCs) and LoRA's low-rank success?
  5. Relationship between perturbation-induced bias and behavioral biases like sycophancy?
