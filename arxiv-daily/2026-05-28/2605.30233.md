> **一句话总结：** 本文通过线性探针和电路分析发现，LLM在实体追踪任务中不进行增量式状态更新，而是在查询出现时并行聚合信息；其REMOVE操作依赖脆弱的全局抑制标签（而非局部移除），导致在no-op移除、多盒共享标签、重新引入已移除对象等场景下产生可预测的失败，且单层干预仅能部分修复。

> **TL;DR:** This paper mechanistically investigates how transformer LMs (Gemma-2-2B, CodeLlama-13B, Llama-3.1-70B) handle entity tracking with state-changing operations (PUT, REMOVE, MOVE) in a natural-language box-tracking task. Through linear probing and circuit tracing via path patching, the authors show LMs do NOT incrementally build world states over tokens or layers. Instead, they aggregate relevant information in parallel at the last token when the query becomes evident. The REMOVE operation uses a fragile global suppression tag that suppresses removed objects everywhere (not just from the relevant box), which predicts behavioral failures in three novel scenarios (no-op remove, shared-label objects, re-introduced objects). A single-layer mechanistic intervention (nullifying the remove tag) partially fixes these failures. The paper was published at ICML 2026.

## 核心贡献

- 通过线性探针实验证明LLM不增量式构建全局状态（全局探针准确率<0.3 vs 局部探针~0.9），而是在查询出现时在最后token处以并行方式聚合相关信息
- 通过路径修补（path patching）追踪PUT操作的完整电路（A/B/C/D四组注意力头），发现PUT与DESCRIPTION共享高层功能但使用不同头集合
- 发现REMOVE操作使用脆弱的全局抑制标签（global removal tag），而非局部从特定盒中移除对象；这一机制通过三项探测（logit/rank分析、三元探针、单层干预）得到确认
- 基于机械分析预测了三种原始数据集不存在的失败模式（无操作移除、多盒共享标签、重新引入已移除对象），并通过行为实验确认高退化率（DR up to 0.84）
- 提出单层干预方案消除全局remove标签，在no-op和共享标签场景中取得较高修复成功率（IS up to 0.73），但在重新引入场景中效果有限（IS 0.24-0.50）

## Key Contributions

- Demonstrates via global/local linear probes that LMs do not incrementally build world states across tokens (H2 confirmed over H1): global probe accuracy stays below 0.3 while local probes reach ~0.9 at the last token position
- Shows via prior-state probing that LMs also do not track local states sequentially across layers (H4 confirmed over H3), ruling out incremental layer-by-layer state updates
- Traces the PUT operation circuit using path patching, identifying four groups of attention heads (A/B/C/D) and showing PUT shares functional mechanisms with DESCRIPTION but uses different head sets for Groups C and D
- Discovers that REMOVE uses a global suppression mechanism -- removed objects are suppressed everywhere, not just from the specified box -- via logit/rank analysis and ternary probes classifying states as {does not exist, exists, removed}
- Predicts and confirms three novel behavioral failure modes from the global removal hypothesis: no-op remove (DR 0.49-0.84), shared-label objects (DR 0.27-0.56), re-introduced objects (DR 0.01-0.62)
- Shows single-layer intervention (nullifying the remove tag at the object token) partially fixes failures, with intervention success up to 0.73 for the first two scenarios

## Methodology

**Task and Dataset:** Uses the box dataset from Kim & Schuster (2023), where a description sentence sets up initial states of 7 boxes with objects, followed by state-changing operations (PUT, REMOVE, MOVE), and a query asking for the contents of a specific box. 100 possible objects, up to 3 objects per box, 0-shot completion setting.

**Models:** Gemma-2-2B, CodeLlama-13B (primary for mechanistic analysis), Llama-3.1-70B (for multi-operation analysis, using NNSight/NDIF).

**Incremental vs. Non-incremental Tracking (Sec. 3):**
- Trains 100 global 8-way probes (per object, predicting which box it's in) and 100 local binary probes (per object, predicting if it's in the queried box) on the last token's residual stream across layers
- Tests H1 (incremental global state building) vs. H2 (non-incremental, query-driven aggregation) via probe accuracy comparison
- Tests H3 (sequential local state processing across layers) vs. H4 (parallel processing) via prior-state probes at different timesteps

**PUT Circuit Tracing (Sec. 4.1):**
- Uses path patching (Goldowsky-Dill et al., 2023) to identify four groups of attention heads (A/B/C/D) responsible for PUT behavior
- Compares PUT circuit overlap with DESCRIPTION circuit via overlap coefficients and leave-one-out analysis
- Compares operation subspaces via subspace similarity metrics

**REMOVE Mechanism (Sec. 4.2):**
- Logit and rank analysis: compares logit/rank of removed objects with and without REMOVE phrases, binned by query relevance
- Ternary probe design: classifies box-object pair states as {does not exist, exists, removed} conditioned on description or operation tokens
- Single-layer intervention: nullifies the remove tag to test causal role

**Failure Mode Prediction and Fixing (Sec. 4.4):**
- Designs three novel evaluation scenarios absent from original dataset to test global removal hypothesis
- Reports recall, precision, degeneration rate (DR), and intervention success (IS) on 300 sentences per scenario

## Results & Highlights

- **Global vs. Local Probes:** Global probe non-trivial accuracy remains below 0.3 across all layers for CodeLlama-13B, while local probes reach ~0.9, strongly supporting non-incremental hypothesis (H2)
- **Prior State Probes:** No consistent pattern of earlier prior states being decodable in earlier layers, supporting parallel processing (H4)
- **PUT Circuit:** Groups A and B overlap significantly between PUT and DESCRIPTION circuits; Groups C and D have minimal head overlap but similar function (binding objects to box IDs)
- **Global Removal:** Logit analysis shows removed objects are suppressed globally (regardless of which box is queried), not locally. Ternary probes confirm a "removed" state representation at object tokens
- **Failure Modes Confirmed:** No-op Remove produces DR of 0.84 (13B) and 0.49 (70B); Shared-label Objects: DR 0.56 (13B) and 0.27 (70B); Re-introduced Objects: DR 0.62 (13B) but only 0.01 (70B)
- **Intervention Success:** Nullifying remove tag achieves IS of 0.66-0.73 for No-op and Shared-label scenarios, but only 0.24-0.50 for Re-introduction (which requires correct multi-operation consolidation)

## Critical Assessment

- **Novelty:** High. The finding that LMs use parallel aggregation rather than incremental tracking for inherently sequential state changes is non-obvious and well-supported. The discovery of global (rather than local) removal is genuinely surprising and practically important. The bidirectional behavioral-mechanistic loop is methodologically innovative.
- **Rigor:** Strong experimental design with multiple converging methods (probes, circuits, logit analysis, behavioral evaluations, interventions). The use of four competing hypotheses (H1-H4) with clear expected outcomes is commendable. However, the mechanistic analysis is primarily on CodeLlama-13B with selective validation on Llama-3.1-70B, raising questions about generality. The Gemma-2-2B model receives the least analysis.
- **Reproducibility:** Good. Code and data are publicly available. The probing and intervention procedures are described in sufficient detail. Path patching follows established protocols.
- **Significance:** The global removal finding has immediate practical implications -- it explains specific, predictable failure modes in stateful reasoning. The paper sets a methodological standard for combining behavioral and mechanistic analysis.
- **Red flags:** (1) The "partial" nature of the intervention fix is a real limitation -- it works well for some failure modes but not others, suggesting the mechanism is more complex than characterized. (2) The box dataset is still somewhat synthetic; how these findings transfer to more open-ended entity tracking in natural text is unclear. (3) Scale analysis is limited -- only 3 models of different sizes, making it hard to separate scale effects from architectural effects. (4) The paper does not address whether chain-of-thought prompting or fine-tuning could induce incremental tracking, which seems like a natural follow-up.

## Strengths

- Clean, hypothesis-driven experimental design with four competing hypotheses tested via well-chosen probes
- The global removal discovery is both surprising and well-supported by converging evidence (logits, probes, interventions, behavioral predictions)
- The paper demonstrates a genuine feedback loop between mechanistic and behavioral analysis -- mechanistic findings predict new behavioral failures, not just explain existing ones
- Thorough analysis of individual operation types (PUT, REMOVE, MOVE) with appropriate separate treatment
- Public code and data repository

## Limitations

- The intervention is only partially effective, especially for multi-step operations (Re-introduction scenario: IS 0.24-0.50), suggesting additional uncharacterized mechanisms are at play
- Results are heavily focused on CodeLlama-13B; Llama-3.1-70B and Gemma-2-2B receive less thorough analysis
- The box dataset, while more naturalistic than synthetic benchmarks, still represents a narrow slice of real-world entity tracking scenarios
- Does not investigate whether architectural modifications or training interventions could induce proper incremental tracking
- The ternary probe results show the "removed" state is only detectable at certain tokens, raising questions about the robustness and generality of the tagging mechanism
- No investigation of how chain-of-thought prompting affects the non-incremental strategy, despite the paper's discussion of CoT

## Relevance

本文通过机械可解释性方法深入分析了LLM在状态变化下的实体追踪机制，揭示了非增量式并行聚合策略和全局移除标签的脆弱性，对理解LLM推理能力边界和设计更可靠的状态追踪架构具有重要参考价值。

## Questions for Further Exploration

- Would chain-of-thought prompting induce incremental state tracking, or would the model still aggregate at the final generation step?
- Does the global removal tag mechanism generalize beyond the box dataset to natural text scenarios (e.g., tracking character locations in narratives)?
- Could targeted fine-tuning on diverse removal scenarios (no-op, shared labels, re-introduction) teach models local rather than global removal?
- How does the non-incremental strategy scale with much longer contexts (hundreds of operations)?
- Do state-space models (Mamba, etc.) or recurrent architectures handle entity tracking more incrementally?
- What is the relationship between the remove tag identified here and the binding mechanisms (order IDs, position-based tags) characterized in prior work?
