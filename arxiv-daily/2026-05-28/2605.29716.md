> **一句话总结：** NaRA 针对扩散语言模型（dLLMs）提出了噪声感知的 LoRA 方法，通过一个全局共享的轻量级超网络，根据去噪过程中的噪声水平动态生成低秩核心矩阵 C(lambda)，使权重更新沿去噪轨迹连续变化，在 LLaDA-8B 上以不到 0.01% 的额外参数开销，在常识推理（平均 84.09% vs LoRA 80.18%）、数学推理和代码生成三类任务上全面超越静态 LoRA 和其他 PEFT 基线。

> **TL;DR:** NaRA identifies that standard LoRA is structurally mismatched to diffusion LLMs because its static parameters ignore the shifting noise levels along the denoising trajectory -- LoRA's gains concentrate at mid-noise levels while it fails at high noise. NaRA inserts a noise-conditioned core matrix C(lambda) = I + eta * HyperNet(FourierEmbed(lambda)) between LoRA's static B and A matrices, generated by a globally shared MLP hypernetwork. Theoretical analysis (Theorem 4.1) shows this factorization can represent multiple independent LoRAs. On LLaDA-8B-Instruct, NaRA achieves 84.09% average on commonsense reasoning (vs LoRA's 80.18%), 80.89% on math (vs 80.07%), and 39.32% pass@1 on code (vs 38.66%), with <0.01% extra parameters. It also generalizes to image diffusion (SDXL DreamBooth) and DoRA. Ablations show global hypernetwork sharing outperforms per-module networks, Fourier embeddings beat scalar/MLP, and even NaRA at rank-16 beats LoRA at rank-32.

## 核心贡献
- 首次系统性地揭示了标准 PEFT 方法（LoRA）在 dLLMs 中的结构性缺陷：静态参数无法适应去噪轨迹中噪声水平的变化，LoRA 的增益集中在中等噪声区域，在高噪声水平下几乎没有改善
- 提出 NaRA，在 B 和 A 静态矩阵之间插入噪声条件化的核心矩阵 C(lambda)，由全局共享的超网络生成，使权重更新随噪声水平连续变化
- 提供理论保证（Theorem 4.1）：在适当的秩条件下，BCA 分解可以精确表示多个独立 LoRA 更新矩阵的等价形式
- 从信息论角度分析 dLLMs 与 AR 模型的本质差异：dLLMs 在同一噪声水平下面临指数级的输入配置多样性（C(L,m) 种掩码组合），使得噪声级别特定的参数化成为必要而非锦上添花
- 广泛的实验验证覆盖三类任务（常识推理、数学推理、代码生成）、两个模型骨干（LLaDA-8B-Instruct 和 Base），以及图像扩散领域的泛化测试

## Key Contributions
- First to systematically identify the structural mismatch between static PEFT (LoRA) and dLLMs: LoRA concentrates gains at mid-noise levels and fails at high noise, as shown via LOWESS loss analysis across noise levels
- Propose NaRA which inserts a noise-conditioned core matrix C(lambda) between static B and A matrices, generated by a globally shared hypernetwork with Gaussian Fourier embeddings for noise level encoding
- Provide theoretical guarantee (Theorem 4.1) that the BCA factorization can exactly represent a set of independent LoRA update matrices when rank covers the union of their subspaces
- Analyze why dLLMs specifically need noise-aware parameterization: at a given noise level, dLLMs face exponentially many input configurations (C(L,m) possible mask patterns) vs AR models' unique prefix, making the input distribution far more diverse
- Comprehensive experiments on LLaDA-8B (Instruct + Base) across commonsense/math/code tasks, plus generalization to SDXL DreamBooth and combination with DoRA

## Methodology

NaRA modifies the standard LoRA decomposition from Delta_W = BA to Delta_W(lambda) = B * C(lambda) * A, where C(lambda) is a dynamic r x r core matrix conditioned on noise level lambda.

**Architecture:**
- Static matrices B (d x r) and A (r x k) are layer- and module-specific, as in standard LoRA
- The core matrix C(lambda) = I_r + eta * F_phi(e_lambda) is generated by a globally shared hypernetwork F_phi (a 2-hidden-layer MLP with SiLU activations)
- Noise level lambda is mapped to a d-dimensional embedding via Gaussian Fourier features: e_lambda = [cos(2*pi*k*lambda); sin(2*pi*k*lambda)] to mitigate spectral bias
- The identity initialization (final layer of hypernetwork initialized to zero) ensures C(lambda) = I at training start, matching standard LoRA's gradient dynamics

**Theoretical Analysis:**
- Theorem 4.1 proves that given N update matrices {Delta_W_1, ..., Delta_W_N}, if rank r >= dim(union of column spaces) and r >= dim(union of row spaces), then exact factorization Delta_W_i = B * C_i * A exists
- Key insight: updates across noise levels are correlated (same task), so the union of subspaces has low effective dimension

**Design Rationale for dLLMs vs AR:**
- AR models: at noise level lambda, the revealed set S is always the prefix {1,...,m} -- unique input configuration
- dLLMs: S can be any of C(L,m) subsets -- exponentially more diverse input distribution per noise level

**Training:** Standard SFT with re-weighted cross-entropy loss on masked tokens. C(lambda) computed once per timestep and broadcast to all modules. Scaling factor eta = 0.1 by default.

**Inference:** Semi-autoregressive block decoding with early termination when EOS detected.

## Results & Highlights

**Commonsense Reasoning (LLaDA-8B-Instruct, 8 benchmarks):**
- NaRA: 84.09% avg | LoRA: 80.18% | HiRA: 82.73% | Prompt Tuning: 78.31%
- On LLaDA-8B-Base: NaRA 83.83% vs LoRA 76.10% (much larger gap on harder backbone)
- NaRA ranks 1st in 5/8 tasks and 2nd in the remaining 3

**Mathematical Reasoning (LLaDA-8B-Instruct, 4 benchmarks):**
- NaRA: 80.89% avg | LoRA: 80.07% | HiRA: 79.99%
- AQuA (hardest): NaRA 57.27% vs LoRA 54.45% -- largest gap on complex reasoning
- On LLaDA-8B-Base: NaRA 77.02% avg, ranks 1st in 3/4 datasets

**Code Generation (LLaDA-8B-Instruct):**
- NaRA: 39.32% avg pass@1 | LoRA: 38.66% | HiRA: 38.33%
- HumanEval: NaRA 40.00% (only method exceeding 40%) | LoRA: 39.94%

**Image Diffusion (SDXL DreamBooth):**
- NaRA: DINO 0.0610, CLIP-I 0.7365, FID 16.71
- LoRA: DINO 0.0458, CLIP-I 0.6901, FID 22.13
- Demonstrates generalization beyond language to image diffusion

**Efficiency:**
- Parameter overhead: <0.01% extra vs standard LoRA (0.43% vs 0.42% of base model)
- Training overhead: ~6ms extra per step vs unmerged LoRA
- VRAM increase: <0.02 GB
- Block-wise early termination reduces inference time by ~81% (39.17 to 7.47 GPU hours)

**Key Ablations:**
- Scaling factor eta: 0.1 is optimal; 0.01 degenerates to static LoRA; 1.0 causes instability
- Fourier embedding beats Scalar (+0.63% avg) and MLP (+0.50% avg), with lowest variance (std +/-0.34)
- Global hypernetwork sharing (80.89%) outperforms per-module strategies (79.83%-80.13%) with fewer parameters
- NaRA at rank-16 (80.47%) beats LoRA at rank-32 (80.07%), showing noise-awareness matters more than rank
- Multi-LoRA (4 adapters, 4x params, 74.83% avg) drastically underperforms NaRA (80.89%), validating continuous parameterization
- NaRA-C (learnable C without noise conditioning) is worse than NaRA and sometimes even LoRA, confirming noise conditioning is the key driver
- LoRA at rank-40 still underperforms NaRA at rank-32, ruling out parameter budget as the explanation

## Critical Assessment

- **Novelty:** High. The identification of LoRA's noise-level blindness in dLLMs is a genuine and timely insight, not a trivial observation. The LOWESS loss visualization (Figure 1) convincingly motivates the problem. The theoretical argument about exponential input diversity in dLLMs vs AR models is novel and illuminating. The approach itself (hypernetwork-generated core matrix) builds on prior work (Zhuang et al., 2024 applied a similar idea to image diffusion) but the adaptation to dLLMs with the specific theoretical justification and the global sharing design is a meaningful contribution.

- **Rigor:** Strong. The paper provides a formal theorem with constructive proof, gradient analysis showing initialization preserves LoRA-equivalent dynamics, and extensive ablations covering scaling factor, embedding strategy, hypernetwork sharing, rank sensitivity, and a NaRA-C control ablation that isolates the effect of noise conditioning. Three random seeds with standard deviations reported throughout. The comparison against LoRA at rank-40 to rule out parameter budget is a good control.

- **Reproducibility:** Good. Code is available. Hyperparameters (learning rate, batch size, epochs, hypernetwork architecture) are detailed in appendices. Training on a single RTX 3090 (24GB) makes the work accessible.

- **Significance:** High for the dLLM subfield. As dLLMs gain traction (LLaDA, Mercury, Gemini Diffusion, Dream), PEFT methods tailored to their dynamics are essential. NaRA establishes a principled baseline. The generalization to image diffusion (SDXL) and DoRA combination further broadens impact.

- **Red flags / concerns:**
  - Only evaluated on LLaDA-8B (single model family). Results on other dLLMs (Mercury, Dream, BD3-LM) would strengthen generalizability claims.
  - The absolute improvement margins on some benchmarks are modest (e.g., code generation: +0.66% avg). The largest gains appear on commonsense reasoning with the Base model (+7.73%), which may partly reflect LoRA's weakness rather than NaRA's strength alone.
  - NaRA cannot merge weights into the base model at inference time (unlike standard LoRA), creating a persistent latency gap vs merged LoRA. The paper acknowledges this but could discuss mitigation.
  - The image diffusion experiment (SDXL DreamBooth on dog6 with 5 images, 1000 steps) is extremely limited in scope -- a single subject, a single setup. The improvements (DINO +0.015, FID -5.4) are promising but need more comprehensive evaluation to be convincing.
  - The Multi-LoRA baseline (74.83% avg) performs surprisingly poorly -- worse than even single LoRA. This seems like a weak comparison rather than a fair alternative, as a better-implemented MoE-style approach might fare differently.

## Strengths
- Clean problem identification with empirical and theoretical motivation (Figure 1 loss analysis, combinatorial argument for input diversity)
- Principled BCA decomposition with Theorem 4.1 guaranteeing expressive power
- Identity initialization ensuring gradient-equivalence to LoRA at training start -- elegant design
- Negligible parameter and compute overhead (<0.01% extra params, ~6ms per training step)
- Comprehensive ablations: scaling factor, embedding strategy, hypernetwork sharing, rank, Multi-LoRA comparison, NaRA-C control, DoRA combination
- Generalization demonstrated beyond dLLMs to image diffusion and to other PEFT methods (DoRA)
- ICML 2026 publication, code available

## Limitations
- Evaluated only on LLaDA-8B (single dLLM family); results may not transfer to Mercury, Dream, or other architectures
- Cannot merge weights into base model at inference, creating a persistent latency overhead vs merged LoRA
- Image diffusion experiment is extremely limited (single subject, single setup, 5 training images)
- Absolute improvement margins are modest on some tasks (code generation +0.66%, some math benchmarks +<1%)
- The Multi-LoRA baseline may not represent the best possible discrete-adapter approach
- No comparison with full fine-tuning to quantify the remaining performance gap
- Semi-autoregressive block decoding and early termination are presented as contributions but are generic optimizations not specific to NaRA

## 相关性

本文针对扩散语言模型（dLLMs）的参数高效微调提出了噪声感知的 LoRA 方法，是 dLLMs 这一新兴范式下微调技术的重要进展。对于关注扩散模型在语言生成中的应用、PEFT 方法改进，以及 dLLMs 后训练策略的研究者具有直接参考价值。其噪声条件化的设计思想也可推广至其他扩散模型的适配场景。

## Questions for Further Exploration
- How does NaRA perform on other dLLM architectures (Mercury, Dream 7B, Gemini Diffusion) beyond LLaDA?
- Can the noise-aware principle be combined with reinforcement learning-based post-training methods (e.g., D1, d-TreePO) for reasoning capabilities in dLLMs?
- What happens when NaRA is applied to larger models (70B+ scale) -- does the global hypernetwork sharing still suffice, or would hierarchical sharing be needed?
- Is the positive correlation between update magnitude and noise level (Figure 3) a general phenomenon across different tasks, or task-dependent?
- Could a learned (rather than fixed Gaussian) Fourier embedding further improve performance?
- How does NaRA interact with quantization methods (e.g., QLoRA) for further efficiency gains?
