Dissecting the Black Box: Circuit-Level Analysis of LLM Vulnerability Detection

Syafiq Al Atiiq, Chun Zhou, Christian Gehrmann — Lund University  |  Gemma-2-2b  |  PrimeVul Dataset  |  2026-05-28
LLMs don't hunt for bugs. They check for safety patterns — and when the safety detectors fall silent, the model defaults to "vulnerable." This is compliance verification, not vulnerability recognition. Only 16.1% of the model's circuitry is involved.

Why This Matters
Safety Mimicry Attack

Since the model relies on detecting safe patterns rather than identifying vulnerabilities, an adversary can inject inert safety checks (e.g., unnecessary boundary validations) to artificially activate safety-detector attention heads, causing the model to classify malicious code as safe. The attack surface is structural, not statistical.

The compliance-verification strategy means the model behaves like a human auditor scanning for safeguards, not a penetration tester searching for exploits. This has two implications:

  • Strength: Few false negatives on genuinely novel unsafe code — if safety patterns are absent, the model flags it.
  • Weakness: High false positives on unfamiliar but safe patterns, plus vulnerability to mimicry attacks that game the safety detectors.

CWE-specific circuits differ: concurrency vulnerabilities trigger 70% higher L0 activation than memory safety issues, suggesting the model develops distinct internal pathways per vulnerability class.

Critical Edge
1 100% baseline on only 472 samples is suspiciously high. Real-world vulnerability detection is far harder. The findings may not generalize beyond this cherry-picked subset.
2 9 concurrency samples underpin the 70% L0 activation claim. That is statistically unreliable with such a small group.
3 Single model only. Gemma-2-2b circuits may differ fundamentally from CodeBERT, StarCoder, or GPT-class models. No cross-architecture validation.
4 Safety mimicry attack is proposed but not empirically validated. Practical exploitability remains speculative.
5 MLP selectivity ~0.25 is modest. The vulnerability signal may be more distributed than the sparse-circuit narrative suggests.

Open Questions
  • Generalization? Does safety-first extend to Gemma-2-9b, Llama, Mistral, CodeBERT?
  • Fine-tuning shift? Would targeted fine-tuning move the model from compliance verification to true vulnerability recognition?
  • Harder datasets? If baseline drops below 90%, do additional mechanisms emerge beyond safety-first?
