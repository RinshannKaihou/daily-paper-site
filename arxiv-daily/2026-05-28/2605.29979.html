Fingerprinting Inference Systems of Large Language Models

Anna Wimbauer, Jonas Möller, Erik Imgrund, Konrad Rieck

The first black-box method that identifies an LLM's entire inference stack — engine, attention backend, and GPU — using only text responses, with zero logit access.

Key Metric
100%
accuracy at T = 0 across all 30 system configurations
Key Finding
Four crafted prompt types targeting distinct computational stages (autoregressive decode, prefill, chunked prefill, KV-cache reuse) produce characteristic numerical deviations that propagate into text output — enabling full system fingerprinting from black-box queries alone.
Why This Matters
Any API-deployed LLM leaks infrastructure details through text responses. No logit access needed.
Critical Edge
30%
worst-case accuracy on unseen components; drops sharply at T > 0.6. Hundreds of queries needed — detectable via rate limiting.

Methodology

The attack operates in three stages: prompt design, embedding via scoring functions, and random forest classification per component.

P1
Rare Token Prompts
Generate rare tokens to amplify autoregressive decoding differences — small logit deviations flip selections at low-probability positions.
P2
Binary Decision
Force a single yes/no response, isolating the prefill stage with virtually no autoregressive generation.
P3
Long Context Recall
Embed a number in long text and ask for recall, triggering chunked prefill where chunking-strategy divergences manifest.
P4
Sentence Repetition
Repeat a sentence N times to activate KV-cache prefill reuse, revealing differences in cache management.

Assessment

Strengths

  • First text-only black-box fingerprinting method — no logit access required
  • Clever prompt design systematically targeting distinct computational stages
  • 100% accuracy at T=0 across all 30 configurations is striking
  • Comprehensive robustness analysis across multiple real-world axes
  • Honest discussion of defense difficulty; connects to real CVEs in vLLM and SGLang

Limitations

  • Only 4 engines, 6 backends, 3 GPUs — real-world diversity is far larger
  • Accuracy collapses at T > 0.6 and for unseen components
  • Requires hundreds of queries — detectable via rate limiting
  • No defense evaluation beyond theoretical discussion
  • Not tested against reasoning/trace models (o1-style) where output patterns may differ

Open Questions

  • Q1How does output filtering, content moderation, or structured output constraints alter fingerprinting signal?
  • Q2Can prompt designs be made more query-efficient (fewer than hundreds) while preserving accuracy?
  • Q3How well does fingerprinting transfer across model families unseen during classifier training?
  • Q4What is the minimum noise level to mask the signal, and what utility degradation does it cause?
  • Q5Could this combine with model fingerprinting (LLMmap) for comprehensive system profiling?
  • Q6How do quantization variants (INT8, INT4, FP8) affect the numerical deviation signal?
