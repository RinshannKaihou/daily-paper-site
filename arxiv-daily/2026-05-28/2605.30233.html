Do Language Models Track Entities Across State Changes?

Tang, Zhao, Franco, Wijaya, Mueller, Schuster & Kim investigate whether transformer LMs incrementally maintain world state or aggregate information at query time. Through linear probing and circuit tracing on Gemma-2-2B, CodeLlama-13B, and Llama-3.1-70B, they find parallel aggregation, not incremental tracking—and discover that the REMOVE operation uses a fragile global suppression tag that breaks in predictable ways.

entity-tracking mechanistic-interpretability transformer-circuits state-tracking failure-mode-prediction
Key Finding

LLMs don't incrementally track state. They aggregate in parallel at query time.

Global probes stay below 0.3 accuracy across all layers while local probes hit ~0.9—ruling out incremental world-state construction (H1) in favor of query-driven parallel aggregation (H2).

Key Metric
<0.3 vs ~0.9

Global 8-way probe accuracy vs local binary probe accuracy at the last token. Prior-state probes also show no sequential processing across layers (H4 confirmed).

Why This Matters

Predictable, structural failure mode in stateful reasoning.

The global REMOVE tag suppresses objects everywhere, not from a specific box. This predicts failures the original dataset never tested—and behavioral experiments confirm degeneration rates up to 0.84.

Critical Edge

Single-layer fix is partial. Re-introduction IS only 0.24–0.50.

Nullifying the remove tag fixes no-op and shared-label failures (IS up to 0.73) but barely helps when a removed object is put back. Uncharacterized mechanisms remain at play for multi-step consolidation.

Hypothesis Testing

H1 — Rejected
LMs incrementally build global state across tokens. Global probe accuracy <0.3 kills this.
H2 — Confirmed
LMs aggregate info at query time in parallel. Local probes reach ~0.9 at last token.
H3 — Rejected
LMs process local states sequentially across layers. No consistent earlier-state signal in earlier layers.
H4 — Confirmed
LMs process all prior states in parallel at the last token, not layer-by-layer.

Predicted & Confirmed Failure Modes

The global removal hypothesis predicts three scenarios absent from the original box dataset. Each was constructed and tested with 300 sentences per scenario.

No-op Remove Remove an object not in the target box 0.84 0.49 0.73
Shared-label Objects Same object label in multiple boxes 0.56 0.27 0.66
Re-introduced Objects Removed object placed back into a box 0.62 0.01 0.24–0.50

DR = Degeneration Rate (higher = worse). IS = Intervention Success (higher = better). Larger models are more robust but still exhibit the same structural vulnerability.

PUT Circuit Architecture

Path patching on CodeLlama-13B identifies four attention head groups:

  • Group A — Retrieves object-attribute bindings from description tokens
  • Group B — Routes information to the query position
  • Group C — Binds objects to box IDs; different heads for PUT vs DESCRIPTION
  • Group D — Final output formation; also operation-specific heads

PUT shares functional mechanisms with DESCRIPTION (similar high-level roles for A/B) but uses different head sets for the binding step (C/D), explaining why the model handles PUT correctly but cannot properly undo it.

Open Questions

  • Would chain-of-thought prompting induce incremental tracking, or would the model still aggregate at the final step?
  • Does the global removal tag generalize beyond the box dataset to natural narrative text?
  • Could targeted fine-tuning on diverse removal scenarios teach models local removal?
  • How does the non-incremental strategy scale with hundreds of operations in context?
  • Do state-space models (Mamba, etc.) handle entity tracking more incrementally?
  • What is the relationship between the remove tag and the binding mechanisms from prior work?
