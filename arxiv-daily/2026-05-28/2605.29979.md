> **一句话总结：** 本文提出一种基于精心设计的四类提示（高频词生成、二元决策、长上下文、重复）的指纹识别方法，利用推理系统各组件（4种推理引擎、6种注意力后端、3种GPU）在浮点运算中产生的特征性数值偏差传播到文本输出的现象，可在黑盒文本响应中可靠识别目标LLM的推理系统配置，温度为0时准确率达100%，非零温度下平均76-80%。

> **TL;DR:** The authors introduce the first black-box fingerprinting method for LLM inference systems that works from text-only responses (no logit access). By crafting four types of prompts targeting different computational stages (autoregressive decoding, prefill, chunked prefill, KV-cache reuse) and training random forest classifiers on response patterns, they identify inference engines (4), attention backends (6), and GPU types (3) across 30 system configurations. Accuracy is 100% at temperature 0 and 76-80% average at non-zero temperatures, with robustness to system prompt changes, batch size variation, and unseen component types. Defense is fundamentally hard without sacrificing utility.

## 核心贡献

- 首次提出无需logit访问、仅依赖文本响应的黑盒LLM推理系统指纹识别方法
- 设计了四类专门提示集，分别针对推理过程中的不同计算阶段：自回归解码、预填充（prefill）、分块预填充（chunked prefill）和KV缓存复用
- 将响应编码为特征向量并使用随机森林分类器分别识别推理引擎、注意力后端和GPU类型
- 在30种系统配置上验证，温度为0时识别准确率达100%，非零温度下平均76-80%
- 分析了实际部署条件下的鲁棒性（系统提示变化、批量大小泛化、未见组件、温度泛化），指出防御的根本困难性并提出部分缓解措施

## Key Contributions

- First black-box fingerprinting method for LLM inference systems operating solely on text responses, without logit access
- Four prompt types targeting distinct computational stages: P1 (rare token generation for autoregressive decoding), P2 (binary yes/no for prefill isolation), P3 (long-context number recall for chunked prefill), P4 (sentence repetition for KV-cache reuse)
- Joint feature representation via scoring functions per prompt set, fed into per-component random forest classifiers
- Evaluated on 30 system configurations (4 inference engines x 6 attention backends x 3 GPU types across 3 LLM families)
- 100% accuracy at T=0; 76-80% average accuracy at T>0 with majority voting over multiple samples
- Robustness analysis covering application-specific system prompts (~99.7% accuracy), batch size generalization (marginal drop), proprietary/unseen components, and temperature mismatch

## Methodology

The method operates in three stages: **prompt design**, **embedding and classification**, and **fingerprinting**.

**Prompt Design (4 types):**
- P1 (Rare token prompts): Ask the model to generate rare tokens, amplifying autoregressive decoding differences where small logit deviations flip token selections at low-probability positions.
- P2 (Binary decision prompts): Force a single yes/no token response, isolating the prefill stage since there is virtually no autoregressive generation.
- P3 (Long-context prompts): Embed a number in a long text and ask the model to repeat it, triggering chunked prefill where numerical divergences from different chunking strategies manifest.
- P4 (Repetition prompts): Ask the model to repeat a sentence N times, activating KV-cache prefill reuse and revealing differences in cache management.

**Embedding:** Each prompt set has a scoring function (indicator for expected tokens, yes/no check, number accuracy, normalized repetition count) producing per-set embedding vectors that are concatenated into a single feature vector.

**Classification:** Separate random forest classifiers are trained for each target component (inference engine, attention backend, GPU type). Training data comes from instantiating all component combinations and collecting prompt-response pairs. For stochastic decoding, multiple samples per prompt are collected during both training and inference, with majority voting for aggregation.

**System components tested:**
- Inference engines: vLLM, SGLang, TensorRT-LLM, LMDeploy (4 total)
- Attention backends: various implementations (6 total)
- GPUs: A100, H100, L40S (3 total)
- Models: Llama-3.2-3B-Instruct, Qwen2.5-7B-Instruct, Qwen3-4B (3 total)

## Results & Highlights

- **Deterministic decoding (T=0):** Perfect 100% accuracy across all components and all 3 models.
- **Stochastic decoding (T>0):** Inference engine identification remains most robust (~80% average at T=0.3). Qwen2.5-7B maintains above 77% for all components up to T=0.6; Llama-3.2 drops more rapidly.
- **System prompt robustness:** Switching to application-specific prompts from a public chatbot dataset yields ~99.7% accuracy.
- **Batch size generalization:** Training on batch sizes {32, 64, 128} and testing on 256 shows marginal accuracy drop (e.g., Qwen2.5-7B: 94/97/97% for engine/backend/GPU; Llama-3.2: 90/84/85%).
- **Unseen components (proprietary setting):** Withholding one component realization from training reduces accuracy substantially for some configurations (e.g., Llama-3.2 inference engine drops to 30%), but Qwen3-4B maintains 86-95% on inference engine.
- **Temperature generalization:** Cross-temperature evaluation (train/test at different T) shows substantial drops (37-49%), motivating temperature alignment.
- **Sample aggregation:** Performance saturates at 10-20 samples for stochastic decoding.

## Critical Assessment

- **Novelty:** High. The first method to fingerprint inference systems from text-only outputs without logit access. Distinct from prior work (LLMmap fingerprints the model identity; Zhang et al. requires log-probabilities). The prompt design targeting specific computational stages is clever and well-motivated.
- **Rigor:** Generally strong. Systematic evaluation across 30 configurations, 3 model families, and multiple robustness dimensions. However, the random forest classifier on binary/indicator features is relatively simple -- it is unclear whether this simplicity is a strength or whether more sophisticated approaches could work with fewer prompts.
- **Reproducibility:** Good. All inference engines (vLLM, SGLang, TensorRT-LLM, LMDeploy) and models are publicly available. Prompt designs are described in detail. The main barrier is requiring 3 different GPU types (A100, H100, L40S) for full reproduction.
- **Significance:** Practically important. The attack only requires black-box query access, making it relevant to any API-deployed LLM. The connection to real vulnerabilities in vLLM and SGLang (MITRE CVEs cited) adds urgency.
- **Red flags:**
  - The 30 system configurations, while substantial, represent a small fraction of real-world diversity. The method may not scale gracefully to dozens of inference engines or GPU variants.
  - At T>0.6, accuracy drops significantly for some components (attention backend down to 33-46% average), and the paper underplays this limitation.
  - The proprietary component evaluation shows very poor results for some models (19-40%), suggesting the method struggles with unseen configurations -- a major practical limitation.
  - No evaluation against adversarial defenses (e.g., output rounding, tokenization changes, response filtering).
  - The number of prompts required (hundreds) could be detected and blocked by rate limiting, which the authors acknowledge but do not fully address.

## Strengths

- Clean problem formulation and threat model
- Ingenious prompt design that systematically targets different computational stages of inference
- 100% accuracy at T=0 across all configurations is impressive
- Comprehensive robustness analysis (system prompts, batch sizes, unseen components, temperature mismatch)
- Honest discussion of limitations, particularly the fundamental difficulty of defense
- Practical relevance given recent CVEs in vLLM and SGLang

## Limitations

- Limited scale: only 4 engines, 6 attention backends, 3 GPUs tested; real-world diversity is much larger
- Accuracy degrades significantly at higher temperatures and for unseen components
- Requires hundreds of queries for reliable fingerprinting at T>0, which is detectable
- No defense evaluation beyond theoretical discussion; actual effectiveness of proposed mitigations (noise addition, rate limiting) is untested
- Proprietary component setting shows poor transferability in some cases (30% for Llama-3.2 inference engine)
- Not evaluated against models that use reasoning/tracing (e.g., o1-style models) where output patterns may differ substantially

## Relevance

本文提出了LLM推理系统的黑盒指纹识别攻击方法，对关注模型服务安全和隐私的研究者具有直接参考价值。该研究表明，仅通过文本交互即可泄露底层基础设施信息，这对LLM API服务商的安全设计和合规性审计提出了新的挑战。

## Questions for Further Exploration

- How does the method perform when the target LLM uses output filtering, content moderation, or structured output constraints that alter response patterns?
- Can the prompt designs be made more query-efficient (e.g., fewer than hundreds of prompts) while maintaining accuracy?
- How well does fingerprinting transfer across model families not seen during classifier training?
- What is the minimum noise level needed to effectively mask the fingerprinting signal, and what is the corresponding utility degradation?
- Could this approach be combined with model fingerprinting (LLMmap) for a more comprehensive system profiling attack?
- How do quantization variants (INT8, INT4, FP8) affect fingerprinting -- do they amplify or mask the numerical deviations?
