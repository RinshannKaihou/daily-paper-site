MarginGate: Sparse Margin-Triggered Verification for Batch-Invariant LLM Inference

Only verify when the logit margin is thin. Skip the rest. Get 100% deterministic decoding at half the cost.

llm-inference determinism batch-effects sparse-verification logit-margin source .md
Key Finding
100% Deterministic
Batch-induced token flips are sparse (<1.3%) and strongly predicted by low logit margins. Verify only those steps, repair K/V, and guarantee exact reproducibility.
Key Metric
+29% Overhead
Llama-3.1-8B at 18.56% trigger rate vs. LLM-42's +64.6% (always-verify). A 2.23× latency reduction while maintaining full sequence determinism.
Why This Matters
Practical Determinism
Deterministic LLM inference becomes viable for real serving systems, not just offline evaluation. Critical for regulated deployments and reproducible research.
Critical Edge
+84.5% on DSR1
Degrades on distilled reasoning models (DSR1-Distill-Qwen-7B) with broad near-tie regions: 49.5% trigger rate. Sparse verification struggles when margins are thin everywhere.
Method — How MarginGate Works
01
BF16 Batch Decode
Run normal batched inference. Same speed as standard serving.
02
Check Logit Margin
Compute gt = logit[top-1] − logit[top-2]. Near-tie tokens are the flip candidates.
03
Conditional Verify
If margin < τ, recompute that step deterministically (LLM-42 style). Otherwise skip.
04
Repair K/V Column
On mismatch, replace only the current K/V column. Single-column repair prevents trajectory drift.
05
Continue Decode
Emit the verified token. Resume BF16 fast path for next step.

Strengths
  • Logit margin is step-local, reference-free, and virtually free to compute
  • Cross-dataset transfer: calibrate once on MATH500, deploy everywhere
  • Honest reporting of failure mode on DSR1-Distill
  • Single-column K/V repair is sufficient to prevent trajectory drift
  • Per-request policy: enable determinism only for requests that need it
  • Layer-wise K/V diagnostics reveal structural patterns (mid-depth K-peak, monotonic V-growth)
Limitations
  • Scope: greedy BF16 only. Excludes sampling, beam search, speculative decoding, quantized models
  • Prototype uses restart-from-rollback, not native K/V overwrite; production overhead may differ
  • No comparison vs. naive random-sampling verification baseline
  • Conservative trigger mass: many triggered verifications find no disagreement
  • Single calibration dataset; sensitivity to calibration set not fully explored
