Locally Coherent, Globally Incoherent: Bounding Compositional Incoherence in Multi-Component LLM Agents

When a multi-component LLM agent routes questions to specialist subagents, each locally coherent probability output can compose into an incoherent joint quote — partition masses summing to 2.5 instead of 1.0. This paper defines ε* as a runtime certificate, proves when local coherence suffices, and repairs the gap in 11 ms.

Theoretical Foundation

Product-Structure Dichotomy

Theorem 3.3
Under owner-selected coordinate aggregation, local coherence guarantees global coherence if and only if the joint polytope factors as a Cartesian product of local polytopes (i.e., coupling constraints are trivial).

Translation: unless your subagents’ questions are logically independent, composing locally valid answers can — and provably will — produce global contradictions.

Deployment

Runtime Gating

ε* predicts top-quartile log-payoff regret with CV-AUC 0.775 ± 0.044.

  • τ ≈ 0.15 — high recall: 91% harm capture at 25% alert rate
  • τ ≈ 0.22 — high precision: 51% capture at 14% alert rate

Anytime-valid e-process (Theorem D.2) controls Type-I error at all stopping times for sequential monitoring.

Experimental Rigor

Controls & Ablations

  • Same-model decoupling: 22–60% residual persists with identical model; cross-model heterogeneity amplifies 1.7–4.5×
  • Greedy decoding (T=0): confirms residual is not sampling noise — ε* increases on negation & disjunction
  • K-sweep (K=8, 16, 32): stability analysis across sample counts
  • Leakage control: Polymarket events resolved after 30 April 2026 model snapshot
  • 55-fold CV for gating thresholds
  • Diebold-Mariano significance on all Brier comparisons
Limitations

Where the framework does not reach

Explicit coupling only

The coupling set C must be given as a finite list of constraints. Recovering C from unstructured agent transcripts is an open problem.

Bernoulli-only

Only binary probability questions are handled. Continuous or structured outputs are not addressed.

Random routing primary

Main evaluation uses random assignment. Planner harness (n=20) and routing ladder (n=100) are small-scale.
