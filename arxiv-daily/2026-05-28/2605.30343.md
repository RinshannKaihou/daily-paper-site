> **一句话总结：** 本文提出 RiM（Reasoning in Memory）方法，用固定长度的特殊 token 序列（memory blocks）替代自回归生成的推理链，通过两阶段课程学习解锁大语言模型的工作记忆能力，在 GSM8K 上相比 Coconut 提升 2.5-7.5 个百分点，同时推理延迟与直接回答持平（TTFT 相同），比 CoT 快约 27 倍。

> **TL;DR:** RiM replaces chain-of-thought's autoregressive token generation with fixed-length "memory blocks" -- sequences of special tokens that form a latent workspace processed in a single forward pass. A two-stage curriculum (Stage 1: ground memory blocks by predicting explicit reasoning steps; Stage 2: remove step supervision and iteratively refine the final answer) trains LLMs to use these blocks as working memory. On GSM8K/GSM-Hard across GPT-2, Llama-3.2-1B, and Llama-3.2-3B, RiM's final-block readout outperforms Coconut by 2.5-7.5pp (GSM8K greedy) while matching direct-answer SFT inference latency (~7x faster than Coconut, ~27x faster than CoT). The paper demonstrates that LLMs can learn to perform structured internal computation in a latent workspace without autoregressive generation.

## 核心贡献

- 提出 Reasoning in Memory (RiM) 框架，用固定序列的 memory blocks 替代自回归推理步骤，作为 LLM 的"工作记忆"实现潜在推理
- Memory blocks 由特殊 token 组成，配合自定义 attention mask，可在单次前向传播中并行处理，实现高效推理
- 设计两阶段课程学习：Stage 1 通过预测显式推理步骤建立 memory blocks 的语义基础（grounding），Stage 2 去除步骤级监督并迭代优化最终答案
- 自定义 attention mask 确保所有 readout 目标在单次前向传播中训练，实现 dense supervision 且避免信息泄漏
- 在 GSM8K/GSM-Hard 上，RiM 的最终 block readout 在 GPT-2、Llama-3.2-1B、Llama-3.2-3B 上均超越 Coconut 和 SFT 基线

## Key Contributions

- Introduces the Reasoning in Memory (RiM) framework that replaces autoregressive chain-of-thought generation with fixed-length memory blocks serving as a latent workspace for working memory
- Custom attention mask enables all readout targets to be trained in a single forward pass with dense supervision and no information leakage, forcing reasoning inside the memory blocks
- Two-stage curriculum: Stage 1 grounds memory blocks by supervising readouts to predict the next reasoning step; Stage 2 shifts to final-answer supervision for iterative refinement across blocks
- Memory blocks are processed in a single forward pass (same TTFT as direct-answer), making RiM ~7x faster than Coconut and ~27x faster than CoT
- Consistent improvements across GPT-2, Llama-3.2-1B, and Llama-3.2-3B on GSM8K and GSM-Hard benchmarks

## Methodology

RiM is motivated by cognitive science: human working memory holds and manipulates information internally without externalizing every step, and developmental psychology shows humans abandon "thinking out loud" in favor of internal processing (Vygotsky, 1978). RiM operationalizes this insight for LLMs.

**Memory Blocks:** Fixed sequences of K special `<m>` tokens (each of length M), inserted between the question and answer. Unlike Coconut which autoregressively generates continuous "thought" tokens, RiM's memory blocks have fixed token identities and positions but variable contextual representations. Since they are fixed, all K blocks can be processed in a single forward pass.

**Custom Attention Mask (Figure 2):** A key architectural contribution. Memory blocks attend to the question and to previous memory blocks. Readout positions (which predict targets) attend to all memory blocks seen so far but not to other reasoning steps. This enables all targets to be predicted in one forward pass without information leakage, forcing reasoning to happen inside the memory blocks.

**Two-Stage Curriculum:**
1. **Stage 1 (Grounding):** Each memory block is followed by a readout supervised to predict the corresponding next reasoning step from the CoT trace. Since each readout can recover its target only from memory blocks seen so far, the LLM is forced to structure its latent workspace around task-relevant intermediate information.
2. **Stage 2 (Refinement):** Step-level supervision is removed. Each readout is supervised to predict the final answer, shifting from predicting explicit reasoning steps to progressively refining the answer across memory blocks.

**Training:** All readouts trained with dense supervision in a single forward pass (via the custom attention mask). Models are trained on GSM8K-Aug and evaluated on GSM8K (in-distribution) and GSM-Hard (out-of-distribution). Checkpoint selection uses a cross-validation protocol on GSM8K.

## Results & Highlights

**Main Results (Table 1, greedy accuracy on GSM8K):**
| Model | SFT w/o CoT | Coconut | **RiM (ours)** | RiM improvement over Coconut |
|-------|------------|---------|----------------|------------------------------|
| GPT-2 | 15.4% | 31.1% | **33.6%** | +2.5pp |
| Llama-3.2-1B | 23.9% | 36.9% | **42.1%** | +5.2pp |
| Llama-3.2-3B | 36.2% | 41.3% | **48.8%** | +7.5pp |

**Inference Latency (TTFT per sample):**
- RiM matches SFT w/o CoT (7.6ms for GPT-2, 16.1ms for Llama-3.2-1B, 27.9ms for Llama-3.2-3B)
- Coconut is ~7x slower than RiM
- CoT is ~27x slower than RiM

**Pass@8 Results:** RiM samples correct solutions more reliably than baselines across all benchmarks. For GPT-2, RiM achieves 49.1% pass@8 vs. Coconut's 45.0%.

**Any-Block Readout (Table 2, diagnostic):** When selecting the best readout across all memory blocks, RiM reaches 39.5% on GPT-2 (approaching CoT's 39.8%) and 51.4% on Llama-3.2-1B (exceeding CoT's 49.1%) -- but at 7.6ms vs. CoT's 213.7ms.

**Robustness (Figure 6):** After Stage 2, accuracy remains stable across a wide range of memory budgets (varying K and M). Answer transitions show that additional memory blocks continue to refine predictions with a positive net effect, indicating the latent workspace does not collapse.

**OOD Results:** On GSM-Hard, RiM outperforms Coconut by 0.7-1.8pp and SFT by 3.5-5.2pp.

## Critical Assessment

- **Novelty: High.** The key insight -- using fixed special-token sequences as a latent workspace with a custom attention mask and two-stage curriculum -- is clean and conceptually distinct from prior work. Unlike Coconut (which autoregressively generates continuous tokens), RiM processes everything in one forward pass. The attention mask design forcing computation into memory blocks is clever.
- **Rigor: Good with caveats.** Experiments use cross-validation protocol with 16 split repeats and report mean +/- standard error. Evaluated on both ID (GSM8K) and OOD (GSM-Hard) benchmarks across three model scales and two families (GPT-2, Llama). However: (1) benchmarks are limited to math word problems -- no code generation, logical reasoning, or harder math (MATH, competition-level); (2) the main baseline is Coconut, with no comparison to other latent reasoning methods like pause tokens (DART) or recurrent approaches; (3) the "any-block" readout is a diagnostic that assumes oracle block selection, somewhat inflating the perceived potential.
- **Reproducibility: Good.** The conceptual clarity of the approach and detailed methodology description aid reproducibility. Training on GSM8K-Aug is a standard setup.
- **Significance: Moderate-to-high.** The latency advantage is genuinely impressive (same TTFT as direct answer, ~27x faster than CoT). The accuracy improvements over Coconut are meaningful. However, the gap vs. explicit CoT on larger models (Llama-3.2-3B: 48.8% vs. CoT's 66.9%) shows latent reasoning still has substantial room to grow.
- **Red flags:** (1) On Llama-3.2-3B, RiM's any-block accuracy (57.3%) is still well below CoT (66.9%), suggesting the latent workspace may not scale as well as explicit reasoning at larger scales. (2) No comparison to RL-based reasoning methods (e.g., DeepSeek-R1 style). (3) The paper evaluates only on math benchmarks -- generalization to diverse reasoning tasks is untested. (4) The "cross-validation protocol" for checkpoint selection on GSM8K while evaluating on GSM8K raises questions about how much the evaluation benchmark leaks into model selection.

## Strengths

- Elegant conceptual design grounded in cognitive science (working memory, Vygotsky's developmental psychology)
- Dramatic inference speedup: same TTFT as direct-answer, ~27x faster than CoT
- Custom attention mask is a clean architectural contribution enabling dense, efficient training
- Two-stage curriculum is well-motivated and ablation shows its necessity
- Consistent improvements across model families (GPT-2, Llama) and scales (124M to 3B)
- Memory budget robustness: accuracy remains stable across varying K and M after Stage 2

## Limitations

- Significant accuracy gap vs. explicit CoT at larger scales (Llama-3.2-3B: 48.8% vs 66.9%), suggesting latent reasoning underperforms when more capacity is available
- Evaluated only on GSM8K and GSM-Hard -- no evaluation on harder math (MATH, AIME), code generation, logical reasoning, or multi-hop QA
- No comparison to RL-based latent reasoning or other recent latent methods (DART, recurrent approaches)
- Fixed memory block structure may limit adaptability to problems of varying difficulty
- Interpretability is reduced -- the latent workspace is harder to inspect than explicit CoT
- Checkpoint selection uses cross-validation on the same benchmark (GSM8K) used for evaluation

## Relevance

本文通过固定长度记忆块实现高效潜在推理，将推理延迟降至与直接回答相同的水平，与测试时计算扩展和推理效率优化的研究方向高度相关。其 attention mask 设计和两阶段课程学习策略对构建高效推理系统有重要参考价值，尤其适合对推理延迟敏感的应用场景。

## Questions for Further Exploration

1. How does RiM perform on harder reasoning benchmarks (MATH, AIME, code generation, logical reasoning) where CoT benefits are most pronounced?
2. Can RL with final-answer rewards in Stage 2 (as the authors suggest) close the gap with explicit CoT at larger scales?
3. How does the latent workspace evolve mechanistically -- can probing or interpretability techniques reveal what computation happens inside memory blocks?
4. Can hybrid approaches combining RiM memory blocks with selective autoregressive generation achieve the best of both worlds?
5. Does RiM scale to 7B+ and 70B+ models where CoT reasoning is most impactful, and does the latency advantage persist with KV-cache optimizations?
6. What is the effect of varying the number of memory blocks (K) and tokens per block (M) on harder tasks -- is there a sweet spot or does it scale smoothly?
