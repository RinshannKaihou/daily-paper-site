Knowledge Boundary as Fingerprint for Black-Box LLM API Auditing

A protocol that fingerprints LLM APIs by probing the edge of what a model knows — where it produces stable, model-specific numerical recall, including consistently wrong answers that act as parametric signatures. Detects every economically motivated model substitution across 16 production endpoints for $0.39.

Protocol — Three Phases
Phase 1 — Offline
Adaptive Frontier Search
Across 15 numerical domains (chemistry, astronomy, math, programming, …), generate cloze-style probes at increasing difficulty. Filter for configuration-invariant stability. Optional: contrastive screen against a cheap substitute to ensure discriminative power.
→
Phase 2 — Enrollment
Self-Calibration
Re-query probes against the reference endpoint under audit configuration. Convert self-discrepancy count into a conservative null bound p0 via one-sided Clopper-Pearson at γ = 0.99. Stable models get tight bounds; noisy ones get wider tolerance.
→
Phase 3 — Online
Binomial Audit
Query the suspect endpoint with enrolled probes. Count discrepancies. Compute one-sided binomial tail under p0. Flag as DIFFERENT if p < 0.05. McNemar's paired test as auxiliary diagnostic.

Critical Edge
  • Probes expire after ~9 weeks due to silent backend updates; operational refresh cycle adds burden
  • Binomial independence assumption simplifies reality — probes share domains and templates; domain-block bootstrap is mentioned but not implemented
  • False-positive evidence: 126 total trials — strong but not yet the "several-hundred" calibration authors recommend for high-stakes deployment
  • No ablation of contrastive screening step — unclear how much it contributes vs. boundary probing alone
  • Cannot provide cryptographic attestation — only statistical consistency evidence
  • Mixed-routing estimator assumes fixed-probability routing; adaptive adversaries may violate this
