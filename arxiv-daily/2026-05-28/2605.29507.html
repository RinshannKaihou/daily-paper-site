Xetrieval: Mechanistically Explaining Dense Retrieval

The first framework to decompose dense retrieval decisions into sparse, human-readable features at the embedding level -- with causal intervention evidence that erasing discovered features drops similarity the most, and amplifying them improves NDCG@10.

Cai, Bai, Liu et al. Beihang Univ. & BIGAI 4 retrievers · 7 benchmarks
◆ Key Finding

Reasoning internalizers (single-hidden-layer MLPs) can approximate CoT reasoning embeddings in one forward pass, then TopK-SAE decomposition reveals causally relevant sparse features explaining why a retriever matched a document.

◆ Key Metric
+12.9%

Snowflake NDCG@10 gain via reasoning internalizer (38.3% → 51.2%), recovering 82% of full CoT reasoner lift (54.7%) at near-zero cost.

◆ Why This Matters

You can now explain and causally intervene on dense retrieval decisions -- without expensive LLM inference at query time.

◆ Critical Edge

Feature descriptions are LLM-generated (circularity risk). Analysis is confined to the sentence embedding output layer -- internal model circuits remain unprobed.

⚙ Method Pipeline

01
Reasoning Internalizer
3 MLPs (SUMMARY / PURPOSE / QA) trained via MSE to approximate CoT-derived embeddings from raw doc embeddings
02
Embedding Enrichment
4 document views: raw + 3 reasoning-enhanced embeddings produced in a single forward pass
03
TopK-SAE Decomposition
Sparse autoencoder (k=256) decomposes embeddings into interpretable features with LLM-labeled descriptions
04
Multi-View Aggregation
Binarized sparse codes compared across views; shared active features form the retrieval explanation
SUMMARY
Captures gist-level document reasoning: what the document is broadly about and its central argument.
PURPOSE
Captures intent-level reasoning: why the document was written and what problem it addresses.
QA
Captures query-relevant reasoning: what questions the document answers and how it responds to information needs.

★ Causal Intervention

  • Erase experiment: Removing Xetrieval-specific features causes the largest drop in query-doc similarity vs. direct SAE or non-overlap features.
  • Retain experiment: Keeping only Xetrieval features preserves or increases similarity more effectively than direct decomposition.
  • Task-level steering: Amplifying key features (selected by Retrieval Utility Score) improves NDCG@10; suppressing them causes clear degradation.
  • Efficiency: Explanation overhead is near-constant regardless of corpus size -- CoT reasoner grows linearly.

⚠ Critical Assessment

Novelty: High
Rigor: Good
Reproducibility: Good
Significance: Moderate-High
  • MSE training objective is a proxy for reasoning capture -- does not guarantee genuine reasoning transfer.
  • Improvement over direct SAE decomposition is real but incremental in many cases.
  • LLM both generates reasoning targets and annotates features -- circularity concern.
  • Analysis limited to sentence embedding layer; internal model circuits not probed.

✓ Strengths

  • Creative reasoning internalizer design: negligible inference cost for rich reasoning signal.
  • Multi-faceted evaluation with both local attribution and task-level steering.
  • Comprehensive SAE comparison (Standard, TopK, Gated, JumpReLU) across 3 metrics.
  • Public code and project page -- reproducible.

✗ Limitations

  • Reasoning faithfulness measured only indirectly via retrieval performance, not direct probing.
  • Three reasoning aspects (SUMMARY / PURPOSE / QA) are pre-defined -- may miss other dimensions.
  • Scalability of sparse decomposition for billion-scale corpora is untested.
  • No comparison against non-SAE baselines (SPLADE, ColBERT, attribution methods) on explanation quality.

? Open Questions for Further Exploration

  1. How much does the choice of CoT-generating LLM (DeepSeek-V3) affect reasoning internalization quality?
  2. Can reasoning internalizers transfer to other embedding tasks -- semantic similarity, clustering, Reranking?
  3. How do discovered sparse features map to known linguistic phenomena (syntax, discourse, pragmatics)?
  4. What happens with domain-specific training data (legal, medical) -- do features become more specialized?
  5. Could feature steering integrate into retrieval training as a regularization or alignment objective?
  6. How stable are SAE-discovered features across random seeds and training data sizes?
