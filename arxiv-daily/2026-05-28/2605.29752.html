From Roofline to Ruggedness: Decomposing and Smoothing the GEMM Performance Landscape

Aditya Chatterjee — Intel Corporation  |  arXiv:2605.29752

arXiv Markdown Daily Index
Headline Claim
30%

Adjacent BF16 GEMM sizes differing by a single 128-element step can vary 30% in throughput. A DP-based optimizer smooths this ruggedness by 70%.

Anchor Fact
110.8

TFLOPs peak at M=3840, N=2048, K=4096 — yet only 1.4% of 32,768 configs exceed 100 TFLOPs. The landscape is a mesa, not a plateau.

So What

GPU kernel performance is a rugged landscape, not a smooth roofline ceiling. Engineers must actively optimize for it — a simple DP padding-and-splitting stack raises mean throughput 30%.

Critical Edge

Single GPU architecture only (Intel Arc B580). No cross-platform validation on NVIDIA, AMD, or other Intel GPUs. The four hardware-bound residual factors may behave differently elsewhere.

Measurement Artifacts Discovered

Two previously undescribed artifacts that corrupt large-scale GPU sweeps:

Co-allocation Interference
50%
worst-case slowdown from reading co-allocated buffers. 32% of configs >20% loss.
Temporal Warmup Drift
43%
TLB/L3 drift across sequential blocks. Spearman ρ=−0.65 with run order.

Fix: Randomized-order sweep eliminates both — corr(readA, N) drops from −0.76 to −0.005.

Methodology

  • Hardware: Intel Battlemage G21 (Arc B580), 20 Xe-cores, 116.5 TFLOPs peak BF16, 12 MB L3, 6 GDDR6 channels (~456 GB/s)
  • Stack: sycl-tla (CUTLASS SYCL port), Intel oneAPI 2025.1, default 256×256×32 workgroup tiles
  • Sweep: 32,768 configs at 128-step for (M,N,K) in {128..4096}³; 5 launches, first discarded; CV 0.04–0.10%
  • Four-surface decomposition: compute / memory / GEMM / overhead surfaces attribute texture to subsystems
  • DP optimizer: O(N&sup4;) offline precomputation, O(1) runtime lookup; 75% pad, 13% K-split, 8% N-split, 5% M-split
  • Cross-tile test: Fine-N sweep (step=32) at 64×64 / 128×128 / 256×256 tiles proves sawtooth tracks tile size, not cache

Strengths & Limitations

Strengths

  • 32K-config sweep with randomized ordering and 50-run variance verification
  • Novel ruggedness framework as complement to roofline
  • DP optimizer: simple, practical, effective
  • Complete quantitative attribution to named mechanisms
  • Cross-tile experiments ruling out cache set conflicts
  • Discovery of two new measurement artifacts with mitigation

Limitations

  • Single GPU only (Arc B580); no NVIDIA/AMD validation
  • Only BF16 NN GEMM; no FP32, INT8, transpose, or batched
  • No comparison with Triton, TVM, cuBLASLt auto-tuners
  • O(N&sup4;) offline cost and table footprint unanalyzed
  • 32% overhead floor listed but not root-caused deeply
  • Generality claims stated but empirically unvalidated
