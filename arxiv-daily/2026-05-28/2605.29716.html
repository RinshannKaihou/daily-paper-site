Noise-Aware LoRA for Parameter-Efficient Fine-Tuning of Diffusion LLMs

ICML 2026 — Seoul Shuaidi Wang, Zhan Zhuang, Ruping Huang, Yu Zhang SUSTech · CityU · HKUST github.com/generaldi/NaRA

Standard LoRA is structurally mismatched to diffusion LLMs because its static parameters cannot adapt to the shifting noise levels along the denoising trajectory. NaRA inserts a noise-conditioned core matrix C(λ) between LoRA's static B and A matrices, enabling weight updates that evolve continuously with the denoising process — with less than 0.01% additional parameter overhead.

Methodology
BCA Decomposition: ΔW(λ) = B · C(λ) · A

Static matrices B (d×r) and A (r×k) remain layer-specific as in standard LoRA. The core matrix is generated dynamically:

C(λ) = Ir + η · Fφ( FourierEmbed(λ) )

A globally shared 2-hidden-layer MLP (Fφ) with SiLU activations produces C(λ) from Gaussian Fourier features of the noise level. Identity initialization ensures C(λ)=I at training start, preserving LoRA-equivalent gradient dynamics.

ⓘ
Theorem 4.1: Given N update matrices, if rank r covers the union of their column and row spaces, the BCA factorization can exactly represent all N matrices. Updates across noise levels are correlated (same task), so the union has low effective dimension.
Motivation
Why dLLMs Specifically Need Noise-Aware PEFT

At noise level λ, AR models see a unique prefix {1,...,m} — one input configuration. Diffusion LLMs see any of C(L,m) possible mask patterns — an exponentially more diverse input distribution per noise level.

LOWESS loss analysis reveals LoRA's gains concentrate at mid-noise levels while it fails at high noise. A single static update matrix cannot serve all noise regimes effectively.

⚠
NaRA-C control: A learnable C matrix without noise conditioning performs worse than NaRA and sometimes even LoRA — confirming noise conditioning is the key driver, not just extra parameters.

Strengths
What Impresses
  • Clean problem identification with LOWESS loss visualization and combinatorial argument for input diversity
  • Theorem 4.1 guarantees expressive power of BCA factorization
  • Identity initialization ensures gradient-equivalence to LoRA at training start — elegant
  • Negligible overhead: <0.01% extra params, ~6ms/step, <0.02 GB VRAM
  • Extensive ablations: scaling factor, embedding, sharing, rank, NaRA-C control, DoRA combo
  • Code available, single RTX 3090 training, ICML 2026 peer-reviewed
Limitations
What's Missing
  • Single model family: only LLaDA-8B tested; Mercury, Dream, BD3-LM unknown
  • No weight merging at inference — persistent latency overhead vs merged LoRA
  • Modest absolute gains on some tasks (code +0.66%, some math <+1%)
  • Image diffusion experiment extremely narrow (1 subject, 5 images)
  • Multi-LoRA baseline surprisingly weak (74.83%) — may not be fair comparison
  • No full fine-tuning comparison to quantify remaining gap
★
Open questions: Performance on 70B+ models? Hierarchical hypernetwork needed? Interaction with QLoRA quantization?
