Why This Matters

Manual rubric annotation burns 2,800+ expert hours (ResearchRubrics) and even drafts 262 physicians for HealthBench.

A dual-axis intrinsic + extrinsic meta-evaluation protocol points the way to scalable, reliable automated rubrics — and shows downstream quality rides on coverage, not text similarity.
Critical Edge
  • Distilled Skill's skill document is extracted from a single Claude-Opus run on one paper — cross-domain robustness unverified.
  • Only 20 papers (RL = 4); no significance tests, so confidence intervals are wide.
  • The extrinsic evaluator is itself an LLM judge — its bias may amplify or cancel the gaps it measures.
01 / METHOD

A four-rung augmentation ladder

TIER 1 · baseline
Direct Prompting
Hand the model the paper + instructions; emit the full rubric in one shot.
TIER 2 · +supervision
+ In-Context Examples
Show human-authored rubrics from other papers to steer structure and granularity.
TIER 3 · +reasoning
+ Agentic Scaffold
ReAct loop with Read / Write tools; iterate over workspace artifacts, inspect intermediate products, refine each criterion.
TIER 4 · +distilled skill
+ Distilled Skill
Claude-Opus runs the scaffold once on a single paper, then writes a reusable skill document S injected into the weaker model — cost-saving, applied to all papers.

Rubric as checklist

Every item rᵢ = (cᵢ, sᵢ): a Pass/Fail criterion plus an importance score sᵢ ∈ [1,5]. Reproduction score = normalized weighted sum of passed criteria.

Two evaluation axes

Extrinsic — score the same batch of reproduction repos (PaperCoder, HiRAS) under both generated and ground-truth rubrics with one LLM judge; correlate the rank vectors (Spearman / Pearson).

Intrinsic — lexically via ROUGE-1/L, semantically via Qwen3-Embedding cosine (τ = 0.7) to avoid LLM-match hallucination; report P / R / F1.

Human checklist re-annotation on PaperBench serves as baseline · 8× NVIDIA L40S.

02 / RESULTS

Closer to humans — but not in the ways that matter

0.78
Extrinsic alignment climbs monotonically with every rung; Distilled Skill lands at Pearson 0.78 / Spearman 0.70, within striking distance of the human 0.83 / 0.75. Pattern holds on GPT-5.4 too.
~78
Criterion count converges to 78–87 (human ≈ 83), yet each item runs long — ~40 tokens vs the human 29.5. Models stay verbose even as cardinality matches.
F1↑
Intrinsic gains are modest & Recall-driven (cosine F1 0.63 → 0.68; Precision flat ~0.71–0.76). Bottleneck is coverage and granularity, not correctness.
>60%
Code Development dominates every model rubric (>60% weight); Result Match stays under 21% — vs 48.8% for Result Match in human RL rubrics. Models ignore the paper's actual topic.
↗
Importance scores skew high. Augmentation softens the right-skew toward a bell, but grade-generosity is never removed.
≈
The counter-intuitive lever: small intrinsic gains, large extrinsic gains — downstream quality tracks coverage & granularity, not text similarity. Direct guidance for future auto-rubric protocols.
03 / BALANCE

Strengths & limitations, side by side

What holds up

  • First meta-evaluation framed explicitly for paper reproduction, bridging PaperBench ↔ RubricBench.
  • Four-rung design is clean & attributable — each rung adds one监督 / reasoning source.
  • Quantitative tables + category / distribution analysis backed by qualitative human-vs-model rubric comparisons.
  • Embedding cosine replaces LLM-based matching, methodologically reducing evaluation hallucination.

Where it frays

  • Domain generalization untested — only ML papers; bio / physics / social-science workflows may not transfer.
  • Scale-limited — 20 papers, RL = 4; only 5 fixed repo configs; no significance tests.
  • Rubric quality stays sensitive to prompt design, ICL selection, scaffold implementation.
  • The three biases (fine-grained, code-centric, grade-generous) are diagnosed but never repaired.
04 / OPEN

Questions worth chasing next

Can a de-biasing module — score calibration, criterion merging — fix code-centrality & grade-generosity without sacrificing extrinsic alignment?
Is the distilled skill document S domain-sensitive? Does a skill pulled from an RL paper transfer to vision?
How much does the LLM-judge's own bias perturb the Spearman / Pearson estimates — would a stronger judge or a vote change the ranking?
At >>20 papers, does the four-rung gain curve saturate, flatten, or reverse?
Can the strong intrinsic↔extrinsic link be formalized into a cheap, optimizable proxy objective for rubric generation?
Does "coverage > similarity" generalize beyond reproduction to other LLM-as-judge rubric tasks?
