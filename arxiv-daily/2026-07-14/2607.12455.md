> **一句话总结：** EvoQuant 把量化策略优化重新定义为"验证器引导的程序进化"问题——用 LLM 诊断瓶颈、分层生成受控编辑、再通过样本外感知的硬门 + 自适应晋升引擎筛选候选，在 A 股和比特币共 7 个策略家族上将测试集平均 Sharpe 从负值提升到正值（平均 +0.829），并把每次优化经验蒸馏进可复用记忆库实现持续自进化。

> **TL;DR:** EvoQuant is a self-evolving, verifier-guided LLM framework that improves existing (already-deployed) quant strategies by representing them as typed editable "genomes," diagnosing failure modes with LLMs, generating hierarchical candidate edits, and admitting a change only after it survives out-of-sample hard gates and an adaptive promotion engine; across 7 A-share + Bitcoin strategy families it lifts average test Sharpe by +0.829 (best Δ Sharpe +1.516 on Oversold Reversal, with the no-promotion ablation collapsing to +0.128), validated by ablations, 2x-fee stress tests, and rolling walk-forward windows.

## 核心贡献

1. **证据约束下的策略进化（Evidence-constrained strategy evolution）**：把"改进用户已有策略"形式化为受约束的程序进化——用户策略被解析为一个带类型注解的"策略基因组"（strategy genome），支持参数调优、局部修复、结构重写、家族迁移等局部、可执行、可追溯的编辑操作。这与一般 LLM"无约束生成新策略"的范式根本不同。
2. **LLM 驱动的验证器引导优化协议（LLM-enabled verifier-guided optimization protocol）**：将 LLM 用于故障模式诊断和分层候选合成，同时把样本外感知的验证、自适应晋升（adopt / incubate / reject 三态）和基于记忆的经验复用整合进一个统一、可审计的闭环。
3. **长周期跨市场实证评估（Long-horizon cross-market evaluation）**：在 A 股（4 个策略家族）和比特币（3 个策略家族）上做了改进前后对比、组件消融、交易成本压力测试和滚动走前验证（walk-forward）。

## Key Contributions

- **Evidence-constrained program evolution framing**: user strategies become typed editable "genomes" supporting localized, executable, provenance-preserving edits rather than free-form regeneration.
- **Verifier-guided closed loop**: LLM does diagnosis + hierarchical candidate generation; a hard-gate + adaptive-promotion engine + shared memory decide whether an edit is admitted, with a chronological out-of-sample (OOS) verification separating validation wins from genuine OOS gains.
- **Long-horizon, cross-market empirical study**: before/after comparisons, ablations, transaction-cost stress tests (2x fee/slippage), split shifts, and two rolling walk-forward windows on A-share and Bitcoin strategies.

## Methodology

EvoQuant 由四个模块组成（Figure 2）：

- **Module 1 — Strategy Ingestion & Representation**: 将原始代码/规则文本解析为 AST，标注交易语义（信号、入场、风控、仓位、出场），切分为可编辑层，最终输出一个"策略基因组" $g$。每次编辑必须指定目标层、满足非预知性（non-anticipativity）和敞口约束、并能重编译为可执行语义；基因组同时保留 provenance、策略家族、父代与变异历史。
- **Module 2 — Baseline Evaluation & Evidence Construction**: 在真实市场数据上做时间序划分回测，生成 EvidencePack $E$，包含标量指标 $m$（Sharpe、回撤、交易笔数、胜率等）、行为画像 $b$（交易稀疏度、持仓分布、收益集中度）、市场体制敏感度 $r$、验证集到样本外的衰减 $\Delta$、异常风险现象 $a$（过度止损、上行截断等）。
- **Module 3 — Optimization Pipeline**（核心闭环）：
  - *Bottleneck Diagnosis & Search Planning*: 由 LLM 做"规则校验 + 深度分析"，识别风险规则 alpha 截断、体制错配、验证-OOS 衰减等故障模式；搜索计划指定本轮目标、搜索层级（repair → bridge → redesign → family migration）、允许算子、禁止捷径、候选预算。
  - *Candidate Generation & Controlled Editing*: 候选生成 prompt 由当前基因组、压缩 EvidencePack、瓶颈诊断、检索到的记忆案例拼接而成；LLM 必须返回受控编辑计划（目标层、操作、补丁内容、机理理由、预期指标变化、安全约束）。候选只有通过 typed edit contract（语义漂移 $\delta$、可执行/金融不变量 $\phi$ 检查）才被送入测试。
  - *OOS-Aware Verification, Promotion & Memory*: 先过简单难规避的硬门（可执行、非零交易、漂移不过大、回撤不过高），再用紧凑评分 $S$ 排序，评分显式分离验证集改善与样本外审计行为（含 OOS 回撤扩张惩罚、验证-OOS 偏差惩罚、费用/滑点扰动、一 bar 延迟、参数抖动等轻量压力探针）。晋升引擎给每个候选 adopt / incubate / reject 决策；失败原因写入 Memory，用于后续检索复用与黑名单，实现持续自进化。
- **Module 4 — Iterative Refinement & Output**: 跟踪全程最佳已验证基因组；局部修复反复失败则升级到 bridge → redesign → family migration；停止条件为预算耗尽、审计分收敛或无可通过验证的搜索层级。最终输出优化后代码、改进前后回测报告及完整优化轨迹。

实验协议：数据来自 AKShare，A 股与比特币日线 2020-01-01 至 2025-12-31；所有需要 LLM 推理的组件统一用 **DeepSeek-R1**；每次优化运行 20 轮迭代。A 股主任务：4 个策略家族 × 随机 30 只股票，每个策略-股票对独立优化，家族级绩效按 30 只等权组合聚合。比特币主任务：3 个策略家族。消融仅在 A 股 Volume Breakout 上做。

## Results & Highlights

**主结果（Table 1，测试集 Sharpe Base → Opt.，Δ = Opt − Base）：**
- A 股 Volume Breakout：−0.493 → +0.489（Δ **+0.982**，年化收益 −2.21% → +15.74%）。
- A 股 Low-Vol Quality：−0.500 → +0.386（Δ **+0.885**，年化收益 1.58% → 17.86%，MDD 9.30% → 8.26%）。
- A 股 EMA-RSI Trend：−0.465 → +0.367（Δ **+0.832**，年化收益 1.64% → 16.83%）。
- A 股 Bollinger Reversion：+0.351 → +0.966（Δ +0.615）。
- 加密 Oversold Reversal：−0.838 → +0.679（Δ **+1.516**，三个加密策略均转为正 Sharpe）。
- 加密 MACD-RSI-Bollinger：−1.118 → +0.265（Δ **+1.383**）。
- 7 个策略家族平均 Sharpe 提升 **+0.829**；A 股平均年化收益提升 13.38 个百分点。

**消融（Table 2，A 股 Volume Breakout，Sharpe 提升）：**
- 全系统 0.982；去掉 promotion engine 跌至 **0.128**，simple threshold 仅 **0.053**——证明自适应晋升是核心使能器，单纯扩大候选池无效（无晋升配置平均编辑更多却更差）。
- Parameter only 0.274；Random mutation 0.645（仍竞争性但易策略漂移、可解释性差）；w/o rule diagnosis 0.693；w/o memory 0.782。

**稳健性（Table 3，受控切片）：**
- 2x 费用/滑点：平均 Sharpe 提升 **0.700**，说明增益不是摩擦假象。
- 切分偏移：提升降至 **0.368** 但仍为正。
- 滚动走前：2020–2023 窗口 +0.370，2021–latest 窗口 +0.545；强度减弱但未崩塌。

## Authors & Credibility

作者来自 **HKUST(GZ)**（香港科技大学（广州））与 **Paradoox AI Research**，团队含多位活跃于金融 LLM 智能体 / alpha 挖掘 / 知识图谱方向的作者（如 Yuyu Luo、Nan Tang 等），引用基础覆盖了 backtest overfitting 经典文献（White 2000、Bailey & López de Prado 2015、Harvey 2013）、LLM agent（Reflexion、Self-Refine、ReAct）、程序化 alpha 搜索（101 Alphas、AlphaForge、AlphaAgent、FactorMiner、R&D-agent-quant）。采用 DeepSeek-R1 作为统一 LLM 后端，数据用公开 AKShare，协议设计（chronological split、OOS 验证、记忆黑名单）在方法论上比一般"LLM 写策略"严谨。代码以匿名链接公开（anonymous.4open.science）。

## Critical Assessment

- 论文核心论点"提升不是随机扰动"在消融中确有支撑：random mutation 的 0.645 仍低于全系统的 0.982，且作者承认 random "偶有有用改动但缺乏诊断/拒绝反馈"，措辞诚实。
- "验证是贡献的核心"（Verification is central）——作者明确不声称收益是普适回撤缩减，并坦诚某些 A 股策略以更高回撤换取更高收益与 Sharpe，稳健性测试下效应减弱但不消失。这种诚实定位降低了过度营销风险。
- 但需注意：所有 Sharpe 数字来自 **30 只随机可用 A 股**的等权组合（非完全可投资历史宇宙，无退市重建），比特币仅 3 个策略，单资产、单一 LLM 后端、本地日线 + 简化执行假设，金融可解释性边界被作者自陈为"原型而非投资建议"。
- 4 个 A 股策略的基线 Sharpe 多为负值（−0.5 左右），意味着"从很弱基线提升到接近 0"占了相当部分增益——绝对水平并不亮眼，相对改善更显著。这一点论文 Table 1 透明呈现，但 TL;DR 式摘要有可能被高估。

## Strengths

- 把"改进已部署策略"（而非从零生成）作为问题定位，更贴近实际研发痛点。
- 四模块 + typed genome + 分层编辑层级（repair/bridge/redesign/family migration）+ adopt/incubate/reject 三态晋升 + 记忆蒸馏，工程闭环完整且可审计。
- 评估协议含消融、2x 费用压力、切分偏移、两段滚动走前，超出多数 LLM-quant 工作的验证强度。
- 失败候选与拒绝原因被显式记录并复用，呼应了 backtest overfitting / data snooping 的经典担忧。

## Limitations

- 基线多为负 Sharpe，绝对收益水平不高；30 只随机股票而非可投资宇宙，无退市/幸存者偏差重建。
- 仅日线 + 简化执行（不含流动性、市场冲击、借券、公司行动、运营约束）。
- 单一 LLM 后端 DeepSeek-R1，比较受本地模型/API 与预算限制。
- 鲁棒性/走前测试仅在 A 股 Volume Breakout 单一策略上做，外推性受限。
- 作者自陈为研究原型，明确不构成投资建议或可上线交易系统。

## Relevance

直接相关于 LLM agent + 量化金融、自动化 alpha/策略挖掘、迭代式程序优化、verifier-guided 生成等方向。对关注"LLM 能否安全地改写已部署金融代码"的研究者与量化工程师有参考价值；其 evidence gate + provenance + adopt/incubate/reject 的设计模式也可迁移到其他"高代价、需审计"的代码自动改进场景（如科学计算、安全攸关系统）。

## Questions for Further Exploration

- 若把基线换成 Sharpe 已为正（如 >1.0）的成熟策略，EvoQuant 的边际提升是否仍显著，还是会出现"提升随基线强度衰减"？
- 自适应晋升阈值的具体形式是什么？是否存在对验证集窗口长度/交易笔数敏感的隐含自由度？
- 把 verifier 从轻量压力探针升级为统计稳健性检验（如 SPA / White's Reality Check / Deflated Sharpe Ratio）后，promotion 决策会改变多少？
- Memory 蒸馏出的"可复用优化范式"在新策略家族上的迁移效果如何？是否存在跨市场负迁移？
- 多 LLM 后端（不只是 DeepSeek-R1）下诊断与候选生成的稳定性如何？
