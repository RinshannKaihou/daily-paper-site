> **一句话总结：** 本文系统分析了基于对数似然的多选题评测中"标准准确率偏向短答案、长度归一化准确率反而偏向长答案"的长度偏差问题，并提出一种无需额外前向传播、即插即用的"贝叶斯准确率"(Bayesian Accuracy)，通过对候选答案长度引入显式指数先验并减去学到的全局长度趋势，将长度偏差降低约 4-8 倍。
>
> **TL;DR:** On likelihood-based multiple-choice benchmarks (MMLU, HellaSwag, ARC, WinoGrande, etc.), the authors show that standard accuracy is biased toward shorter completions while byte-length-normalized accuracy over-corrects toward longer ones; they propose **Bayesian accuracy**, a drop-in scoring rule that fits one global length-decay factor per model-dataset-prompt config and subtracts the linear length trend, cutting mean absolute length bias (Kendall τ) into the 0.03-0.07 range — a 4x-8x reduction versus normalized accuracy across base/instruction-tuned LLaMA, Qwen, Mistral, Pythia, and SmolLM models — with no extra forward passes.

## 核心贡献

1. **形式化定义长度偏差**：将"长度偏差"严格定义为"每个样本内部候选答案长度与其打分之间的平均 Kendall 秩相关系数 τ(S, D)"。正值表示偏向更长答案，负值表示偏向更短答案，接近零表示无单调关联。这为后续分析提供了一个统一、可量化的度量。

2. **系统刻画现有打分规则的行为**：在 ARC、ARC German、HellaSwag、MMLU(全文/填空两种格式)、OpenbookQA、SciQ、WinoGrande 七个基准上，对 LLaMA/Qwen/Mistral/Pythia/SmolLM 家族(135M-70B，base 与 instruction-tuned)进行大规模实证。核心发现是：总对数似然随答案字节长度近似**线性**下降(斜率为负)，因此**标准准确率系统性偏向短答案**；而朴素的字节/词长度归一化(S_byte / S_tok)经常**过度修正**，反而引入偏向更长答案的正偏差，且其行为对数据集模板和模型族高度敏感。

3. **给出"何时可安全使用标准/归一化准确率"的经验规则**：标准准确率仅安全于(i)答案文本已出现在 prompt 中且前 1-2 个 token 即决定选项(如 MMLU Full-Text，因为剩余 token 几乎是对 prompt 的确定式拷贝)，或(ii)同一题所有候选长度几乎相等(如单字母预测)。论文明确指出**不存在**保证归一化准确率低偏差的简单启发式，不推荐将其作为默认。

4. **提出贝叶斯准确率 (Bayesian Accuracy)**：基于"总对数似然-长度"近似仿射关系建立一个分层线性生成模型，引入关于长度的指数先验 P_prior(n) ∝ exp(-b·n)，推导出长度去偏的对数似然打分 S_Bayes(c|x;b) = ℓ_θ(c|x) - b·n。它只需条件对数似然、无需任何额外前向传播，是对现有打分的**即插即用替代**。

5. **鲁棒的低偏差估计器**：通过**问题内中心化(within-question centering)**而非跨问题池化回归来估计全局衰减因子 b̂，消除了题目难度与候选长度的混杂偏移；并以 O(M) 的线性复杂度等价替代 O(M²) 的成对估计(Appendix A.1-A.3 给出完整推导与 Algorithm 1)。

## Key Contributions

- Formalizes length bias as the average within-example Kendall rank correlation τ between candidate lengths and scores, providing a model- and dataset-agnostic metric.
- Empirically demonstrates, across 7 benchmarks and 5 model families (135M-70B, base and instruction-tuned), that (a) total log-likelihood falls roughly linearly with length so standard accuracy favors short answers, and (b) length normalization frequently over-corrects and flips the bias toward longer answers in a template/model-sensitive way.
- Derives actionable rules of thumb for when standard accuracy is safe (locally-determined answers or near-equal-length candidates) and argues that no analogous safe-default rule exists for normalized accuracy.
- Introduces **Bayesian accuracy**, a drop-in scoring rule derived from an exponential length prior that subtracts a learned global linear length trend from conditional log-likelihoods — requiring no additional forward passes (unlike PMI/ANPMI, which double cost).
- Provides a within-question-centered estimator for the global length-decay factor b̂ that avoids confounding by prompt difficulty and is computable in linear time per prompt.

## Methodology

**问题设定**：对多选题数据集 D = {(x^(k), C^(k), y^(k))}，模型对每个候选 c 计算总条件对数似然 ℓ_θ(c|x) = Σ token log-probs。打分函数 S 将其变换为分数，取最高分候选为预测。

**长度偏差度量**：对每个样本，计算其候选分数 {s_i} 与字节长度 {n_i} 的 Kendall τ_b(见式 14)；数据集级偏差 τ(S, D) 为所有有定义样本的均值(式 16)。

**线性生成模型**：假设 ℓ_m^(k) = α_k + β_k·n_m^(k) + ε_m^(k)，其中 α_k 为题目难度偏移，β_k 为题目级每字节斜率，ε 为零均值噪声。对题目取期望得到全局线性趋势 E[ℓ|n]=a+b·n，方差随长度二次增长(因 β_k 有变异)。

**贝叶斯校正**：引入长度先验 P_prior(n) ∝ exp(-b·n)，后验 log P_post(c|x) = ℓ_θ(c|x) + log P_prior(n) - log Z(x)。因 Z(x) 与候选无关，得到长度去偏打分 S_Bayes(c|x;b) ≡ ℓ_θ(c|x) - b·n(式 23)。

**估计 b̂**：用问题内中心化的固定效应斜率(式 24 / Algorithm 1)：对每题将长度和对数似然减去题内均值后再做加权最小二乘，等价于成对差分回归但复杂度从 O(M²) 降到 O(M)。每个 model-dataset-prompt 配置估计一个 b̂。

## Results & Highlights

- **标准准确率明显偏向短答案**：在 HellaSwag 和 ARC 上 Kendall τ 显著为负(候选长度差异大时尤甚)，而 SciQ/WinoGrande 的 τ 接近零；同一模型族内，模型规模越大负 τ 越小但偏差仍显著非零。

- **字节归一化反而引入更大正偏差**：Table 2 显示 τ(S_byte, D) 在所有基准和模型上几乎一致为正，常常比标准准确率的负偏差幅值相当甚至更大。在 ARC、ARC German、OpenbookQA、SciQ、WinoGrande 上，归一化把原本轻微负/零偏差变成了显著正偏差；对 instruction-tuned 模型过冲尤其明显。

- **贝叶斯准确率将 τ 拉到接近零**：Table 3 中几乎所有条目满足 |τ| ≤ 0.1。

- **平均绝对长度偏差 |τ|̄ 对比(Table 4)**：归一化准确率偏差最大；PMI、ANPMI 依次改善但仍残留非平凡偏差；**贝叶斯准确率对所有模型取得最小或接近最小的 |τ|̄**，落在 0.03-0.07 区间。

- **改善幅度**：相对于归一化准确率约为 **4×–8×** 的长度偏差降低，相比 PMI/ANPMI 通常也有数倍改善；同时对 base 和 instruction-tuned 模型均成立，且在 zero-shot 与 few-shot 设置(Appendix A.6, Tables 5-7)下结论一致。

- **零额外前向传播成本**：与 PMI/ANPMI(每个候选需额外一次无条件前向传播，评估成本翻倍)不同，贝叶斯准确率仅用条件对数似然。

## Critical Assessment

论文最扎实之处在于它**把一个被工程界长期默认处理的 trick(length normalization)放到了严谨的统计显微镜下**，并用一个统一的 Kendall τ 度量把"标准 vs. 归一化"这对长期争议的取舍量化出来。关键洞察是 Figure 1/2 所揭示的:总对数似然与长度近似**仿射**(有显著正截距、负斜率)，这同时解释了为何朴素除法会过冲——除以长度相当于对"截距+斜率·n"做 1/n 变换，短答案因截距被过度惩罚。贝叶斯校正 ℓ - b·n 正是对这一仿射关系的最小充分修正，几何上等价于"去掉斜率、保留截距"。

值得审慎对待的点:
- **仅去除线性长度效应**。作者自己也承认:该修正针对"加性对数似然累积引起的长度先验"，但**不去除**其他答案先验(常见词、短语、句式偏好)。若残差偏差的主要来源是后者，贝叶斯准确率的实际收益可能小于 τ 指标所暗示的。
- **τ 接近零不等于评测更"忠实"**。论文明确把"与下游能力一致性"列为未来工作(Section 5)，本工作只解决了一个"可测量的 nuisance 效应"。降低 τ 是否真的让模型排名更公平、更贴近真实能力，本文未提供直接证据(例如与生成式评测或人类判断的相关性)。
- **每个 model-dataset-prompt 估一个 b̂** 的做法虽稳健，但也意味着打分规则本身是**数据拟合的**——理论上存在过拟合基准集长度的风险，尤其在候选数 M 小、题内长度方差小时 b̂ 噪声大(作者建议用 held-out 校准或向 0 收缩来缓解，但未给出系统消融)。

## Strengths

- 理论与实证结合紧密:从经验仿射观察到分层线性模型，再到闭式贝叶斯打分，逻辑链完整。
- 方法极简且即插即用:只需对现有条件对数似然做一次仿射变换(减去 b̂·n)，无额外前向传播，工程落地成本近乎为零。
- 度量统一:Kendall τ 提供了一个跨基准、跨模型、跨打分规则的可比指标，便于复现与监督。
- 给出清晰、可操作的使用指南(两个"rule-of-thumb 框")，对评测实践有直接指导意义。
- 实验覆盖面广(7 基准 × 5 模型族 × base/instruct × zero/few-shot)，结论稳健。

## Limitations

- **只修正线性长度偏差**，对非线性长度效应及非长度类答案先验无效。
- **未验证与"真实能力"的相关性**:降低 τ 是否带来更准确的模型排名或更贴近下游任务表现，尚无证据。
- **b̂ 估计的稳健性**在候选数少、题内长度方差小时存疑，需依赖 held-out 校准/收缩等启发式，缺少系统的敏感性分析。
- **未覆盖指令遵循/生成式评测**场景，结论局限于 likelihood-based 多选题打分。
- 作者归属与正式版本信息在 HTML 中不完整。

## Relevance

本研究对所有基于"按条件对数似然给候选打分"的 LLM 评测流程具有直接、立竿见影的相关性——这涵盖了 MMLU、HellaSwag、WinoGrande、ARC 等绝大多数主流 base-model 基准，以及 lm-eval-harness、lighteval、Aleph Alpha eval-framework 等评测框架的默认打分路径。若结论成立，评测榜单(尤其是 base 模型之间微小差距)的长度偏差系统性误差可被显著压缩，可能改变模型间的相对排名。对于构建自研评测管线、做模型选型或复现论文数字的工程与研究人员，贝叶斯准确率是一个低成本、高回报的默认打分候选。

## Questions for Further Exploration

- 降低 τ 之后，模型在多选题上的**准确率绝对值与排名**会如何变化？是否存在某些基准/模型对的排名被显著重排？
- 贝叶斯准确率与**生成式评测/人类判断**的相关性如何？是否真的更"忠实"于模型能力？
- 非线性长度效应(如长答案的"疲劳"超线性惩罚)在多大程度上残留？是否需要一个二次或分段修正项？
- b̂ 的**跨配置泛化性**:能否用少量校准数据估一个 b̂，迁移到新基准/新 prompt 模板而保持低偏差？
- 该校正对**多语言、多 tokenizer** 场景(本文已含德语 ARC，但可扩展到更分离的书写系统)是否同样鲁棒？
- 能否将贝叶斯长度先验与其他去偏目标(如去答案先验 PMI)正交结合，得到同时处理"长度先验"与"内容先验"的统一打分？
