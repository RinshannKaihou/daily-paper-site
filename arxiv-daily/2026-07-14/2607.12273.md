> **一句话总结：** Code-MUE 提出一种纯黑盒、基于执行的不确定性量化框架：通过对多次采样的程序构建"语义交互图"并计算其 Von Neumann 熵，将 LLM 代码生成的不确定性锚定在可观测的运行时行为上，与功能正确性的 Spearman 相关性高达 -0.98，远超词法和嵌入基线。

> **TL;DR:** Code-MUE is a purely black-box uncertainty estimation framework for Code LLMs that samples N=100 programs per prompt, executes them on a hybrid probe set, builds a Semantic Interaction Graph encoding functional consensus, and quantifies global uncertainty via Von Neumann entropy; it achieves Spearman ρ up to -0.98 against functional correctness on HumanEval (vs. -0.2 to -0.5 for Lexical Entropy baselines) and AUROC >0.85 across CodeNet/MBPP, evaluated on 8 LLMs and 4 SE benchmarks.

## 核心贡献

1. **新视角与指标（New Perspective & Metric）**：提出一种专为代码 LLM 设计的不确定性度量，完全依赖执行反馈（无需 logits/内部状态），贯彻"轻量级黑盒"哲学。它绕过文本相似度，将不确定性扎根于可观测的运行时行为，从而弥合了"语法相似 ≠ 语义等价"的根本鸿沟。
2. **大规模实证研究（Large-Scale Empirical Study）**：在 4 类软件工程任务（代码补全 HumanEval、程序合成 MBPP、程序修复 QuixBugs、代码翻译 CodeNet）上评估 8 个 SOTA LLM（4 闭源：GPT-4.1-mini/nano、Claude-3-Haiku、Gemini-2.5-Flash-Lite；4 开源：Llama-3.1-8B、Mistral-7B、DeepSeek-Coder-6.7B、Qwen2.5-7B），约 50 万次闭源 API 推理。
3. **AI 安全与可靠性的实践意义（Practical Implications）**：展示 Code-MUE 在两类下游场景的实用性——选择性预测（拒绝高不确定性样本以降低错误率）与无监督响应选择（用特征向量中心性从候选集中挑出"语义中心"作为最优解），无需 ground-truth 测试预言。

## Key Contributions

- **Execution-grounded uncertainty metric**: A novel black-box metric that measures Code LLM uncertainty solely through execution consensus, bypassing the need for logits or internal states.
- **Quantum-inspired graph formalism**: Interprets the pairwise execution-consensus matrix as a density matrix and uses Von Neumann (spectral graph) entropy to capture the global topological diversity of the solution space, rather than collapsing it into first-order average agreement.
- **Comprehensive empirical validation**: 8 LLMs × 4 benchmarks demonstrate the metric strongly anti-correlates with functional correctness and enables downstream risk detection and unsupervised re-ranking.

## Methodology

Code-MUE is a three-stage, strictly black-box pipeline (no logits/gradients):

**Stage 1 — Probabilistic Sampling & Hybrid Test Input Synthesis.** Given a prompt x, the LLM is queried N=100 times with temperature τ=1.0 to obtain a program ensemble P={p_1,…,p_N}. In parallel, a hybrid input-only probe set I (default K=20) is built from two halves:
- Type-based boundary inputs (I_edge): static analysis of the function signature maps types to corner values (0, -1, sys.maxsize, 1e-9, empty sequences, singletons) to trigger conditional-logic fragility.
- LLM-guided nominal inputs (I_nom): the model is prompted to synthesize representative "typical" inputs from the task description. Crucially, **no expected outputs/assertions are ever generated**, which bypasses the Oracle Problem.

**Stage 2 — Semantic Interaction Graph (SIG) Construction.** Every program p_i is executed in a sandboxed worker (5-second timeout, exit calls disabled) against the full probe set I. Each execution outcome is mapped to one of three discrete behavioral modes: Return Value (v), Differentiated Runtime Exception type (E_type — e.g., IndexError vs RecursionError treated distinctly), or Timeout (T). Each program thus gets an Execution Signature Vector O_i ∈ Ω^K. A pairwise similarity matrix W ∈ ℝ^{N×N} is built where W_ij = (1/K) Σ_k 1[o_i,k ≡ o_j,k]. Two isolation rules are key: (i) syntax-error programs have null semantic value → W_ij=0 (isolated nodes); (ii) two distinct programs timing out on the same input count as 0 agreement (conservative — avoids falsely merging infinite loops with slow-but-terminating code). Self-similarity W_ii=1 for matrix stability.

**Stage 3 — Graph-based Uncertainty Quantification.** The weighted graph G=(V,E,W) is trace-normalized into a density matrix ρ = W/Tr(W) = W/N. Eigen-decomposition yields a spectrum Λ where eigenvalues represent the probability distribution over Latent Semantic Modes (pure state λ_1≈1 = single consensus; mixed state = competing modes). The final uncertainty score is the normalized Von Neumann entropy U(P) = H_spec / ln(N) ∈ [0,1], where H_spec = -Σ λ_k ln(λ_k). This is topology-aware: unlike average pairwise agreement W̄, it distinguishes a dominant cluster with outliers from several balanced competing groups.

## Results & Highlights

- **Near-perfect correctness alignment on HumanEval**: Spearman ρ = **-0.987 (GPT-4.1-mini)** and **-0.985 (Gemini-2.5-Flash-Lite)**, whereas the best baseline (Lexical Semantic Entropy) typically lands between **-0.2 and -0.5** — a dramatic margin showing low spectral entropy faithfully reflects true functional confidence.
- **Best discrimination in 26/32 scenarios**: Code-MUE outperforms all baselines (Lexical Entropy, Embedding/CodeBERT Entropy, Length Variance, Logit-based) on AUROC, **consistently exceeding 0.85** on CodeNet and MBPP. Classical logit-based baseline is generally weaker and less stable, confirming the token-vs-problem granularity mismatch.
- **Strong practical utility**: In selective prediction, rejecting just the top 10-20% most uncertain samples drops the error rate precipitously (lowest AURC across all baselines). In unsupervised response selection via eigenvector centrality, selection accuracy on solvable tasks **often exceeds 80% (CodeNet)** and reaches **>90%** on GPT-4.1-mini and Gemini-2.5-Flash-Lite, validating the "Semantic Consensus Hypothesis."
- **Robust to reduced budgets**: Reducing N from 100→25 only weakens ρ from -0.831 to -0.804 and AUROC from 0.782 to 0.765 (Table 3); even K=4 probes still yield ρ=-0.682 (Table 4). Performance exhibits clear diminishing returns, enabling cheaper deployment.
- **Repository-level scalability**: On a RepoClassBench task (psf__requests-6028), applying Code-MUE at the method-module granularity to 20 GPT-4.1-nano candidates yields low uncertainty H=0.09, with manual inspection confirming 18/20 implementations correct.

## Critical Assessment

The paper's central insight is genuinely well-motivated: code's defining property is executability, so semantic distance should be measured in execution space, not in embedding/lexical space. The motivating example (FindMax vs FindMin differing by a single token `>` vs `<` yet clustering together under CodeBERT and lexical metrics) is a crisp, persuasive demonstration of the syntax-semantics gap. The choice of Von Neumann entropy over simple average pairwise agreement is theoretically justified — it captures whether the ensemble concentrates around one consensus, splits into competing groups, or fragments, distinctions invisible to W̄.

However, the method inherits a structural limitation from its design. By treating execution output as "the only ground truth for semantics," Code-MUE is blind to systematic errors where the model confidently converges on a wrong-but-stable behavior. The paper is commendably honest about this (RQ3 case studies), explicitly diagnosing "Overconfident Error" patterns like DeepSeek-Coder confusing "even digits" with "even integers" (HumanEval/163, entropy 0.0485 but Pass@1 = 0.0) due to corpus frequency bias. This is presented not as a bug but as a feature: low uncertainty + low accuracy flags "hard failures" needing prompt refinement, while high uncertainty flags "soft failures" amenable to abstention. This is a reasonable framing but means the metric cannot, by itself, distinguish confident-correct from confident-wrong — a fundamental ceiling on its standalone reliability.

The timeout handling (treating distinct timeouts as zero agreement) is conservative and the paper's "Fail-Safe" argument is sound, but it systematically penalizes correct-but-inefficient solutions (e.g., HumanEval/83 brute-force O(10^n) that times out), inflating uncertainty for logically correct code — an asymmetry that could mislead in efficiency-sensitive domains.

## Strengths

- **Principled black-box design**: Fully compatible with closed-source API deployment (GPT-4.1, Claude, Gemini), which is the realistic modern setting; no reliance on restricted logits or internal states.
- **Oracle-free execution probing**: Generating only inputs (never expected outputs/assertions) elegantly sidesteps the test-oracle problem that plagues validation approaches.
- **Rich topological signal**: The density-matrix / Von Neumann formulation captures global structure (single consensus vs competing clusters vs fragmentation) that scalar metrics like mean agreement or length variance cannot.
- **Dual downstream utility**: Beyond a correctness proxy, the SIG topology enables unsupervised response selection via eigenvector centrality — a non-trivial extension that converts a measurement metric into a decision tool.
- **Honest failure-mode analysis**: RQ3 case studies (conceptual ambiguity, efficiency gap) transparently characterize when and why the metric fails, rather than hiding outliers.

## Limitations

- **Cannot detect confident-but-wrong errors standalone**: Systematic biases (corpus frequency, prompt ambiguity) produce low-entropy incorrect solutions that are indistinguishable from correct consensus without external intervention. The metric flags "something is off" only via high uncertainty; confident errors require separate detection.
- **Timeout asymmetry**: Conservative timeout handling (distinct timeouts = zero agreement) inflates uncertainty for logically correct but inefficient solutions (e.g., brute-force O(10^n) on HumanEval/83), conflating slowness with semantic uncertainty.
- **Coverage bias of LLM-generated inputs**: If synthesized nominal inputs miss a critical semantic region, distinct behavioral modes may merge and underestimate uncertainty. The hybrid edge/nominal strategy mitigates but does not eliminate this; the paper flags stronger fuzzing/symbolic generators as future work.
- **Computational cost**: ~500K closed-source inference calls plus O(N×K) sandboxed executions (100×20 = 2000 executions per task). While execution is cheap parallel CPU work, the dominant cost is N LLM inferences, which the sensitivity analysis shows is hard to compress without quality loss.
- **Python-centric, function-level focus**: The execution sandbox is Python; scalability to repository-level is demonstrated only via manual AST decomposition into method modules, not end-to-end on monolithic outputs. Non-Python or multi-language settings are untested beyond the translation task's output execution.
- **Small-sample instability on QuixBugs**: With |D|=40, baselines occasionally beat Code-MUE on AUROC (e.g., Qwen2.5-7B), attributed to high variance in binary classification thresholds.

## Relevance

本文对"可靠的 LLM 驱动软件工程"研究方向高度相关。对于关注以下议题的研究者与工程师有直接价值：
- **代码 LLM 的不确定性量化与校准**：尤其是闭源模型场景下，传统白盒/灰盒方法失效，Code-MUE 提供了切实可行的黑盒替代方案。
- **选择性预测与 abstention**：在自动化代码生成流水线中决定何时拒绝输出、何时请求人工介入。
- **无监督响应选择 / Best-of-N 重排**：无需测试预言即可从多次采样中挑选最可能正确的解，对缺乏参考测试用例的真实部署场景极具实用价值。
- **软件测试与调试传统的延续**：将经典故障定位、蜕变测试、统计调试中"执行即证据"的思想引入 LLM 不确定性估计，是 SE 与 ML 交叉的有益尝试。
- 与本文应一并阅读的相关工作：Semantic Entropy (Kuhn et al., 2023)、Spiess et al. 的代码 LLM 校准研究 (ICSE'25)、以及基于内部状态的 Code LLM 风险评估 (Huang et al., 2025a, FSE'25)。

## Questions for Further Exploration

1. **能否在不牺牲黑盒性质的前提下检测 confident-but-wrong 错误？** 例如，结合 LLM-as-Judge、蜕变关系（metamorphic relations）、或对采样温度的敏感性分析，来区分"低熵且正确"与"低熵但系统性错误"。
2. **更激进的输入生成策略（fuzzing / 符号执行 / 差分测试）能否进一步压低 coverage bias？** 论文已指出这是 future work，但量化其边际收益会很有价值。
3. **密度矩阵框架是否可扩展为"加权"语义模式，而非离散等价聚类？** 当前 W_ij 是硬指示函数（全匹配得 1），部分输出匹配（如返回相近但非完全相同的值）能否通过 soft 相似度获得更细粒度的信号？
4. **Von Neumann 熵与其他图谱指标（如 Laplacian 特征值间隙、模块度、Cheeger 常数）相比的优劣？** 是否存在对某些失败模式更敏感的替代图谱度量？
5. **在非 Python 语言（Java/C++/Rust）以及多文件、仓库级生成上的可扩展性如何？** 当前仅在翻译任务中执行输出，未深入跨语言执行语义的统一刻画。
6. **响应选择中的"语义共识假设"在任务本身有多种合法解时是否仍成立？** 例如开放性 API 设计任务，多个正确实现可能行为各异、不形成稠密簇。
7. **能否将 Code-MUE 的不确定性信号反向用于引导 test-time scaling 或 self-consistency 采样，而非仅做事后筛选？** 论文在引言中提及此类应用但未实证。
