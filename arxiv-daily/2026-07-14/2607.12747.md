> **一句话总结：** 本文提出 Oat，把 LLM 智能体失败归因问题重新表述为「只用成功轨迹训练的一类学习」，用神经受控微分方程（Neural CDE）+ 门控控制路径建模成功轨迹在潜空间的动态，在推理时对失败轨迹的每一步计算异常分；仅用约 100 条成功轨迹训练，在 MCP-Atlas 与 Who&When 两个基准上 F1 分别 +20% 和 +7%，超过 GPT-4o/GPT-5 提示方法，且推理快 200–5000 倍、零 token 成本。

> **TL;DR:** Oat reformulates LLM agent failure attribution as unsupervised one-class learning: train exclusively on ~100 successful trajectories using a Neural CDE with a novel gated control path to model successful latent dynamics, then flag per-step deviations as error steps at inference. On MCP-Atlas (in-domain) and Who&When (OOD), Oat beats GPT-4o/GPT-5 prompting baselines by +20% and +7% F1 respectively, runs 200–5000x faster with zero token cost, and needs <1 GB VRAM.

## 核心贡献

1. **提出新问题设定——无监督失败归因（Unsupervised Failure Attribution）**：仅在成功轨迹上训练（无需任何失败数据、无需任何步骤级标注），推理时定位失败轨迹中导致失败的步骤。这消除了既有方法（prompting 流水线或带步骤标注的强化学习后训练）的标注瓶颈与高昂推理成本。
2. **方法 Oat（One-class Agent Tracing）**：将问题转化为 LLM 表示空间中的一类学习，用神经受控微分方程（Neural CDE）把成功轨迹的步骤表示序列建模为连续潜路径；并提出**门控控制路径（gated control path）**，自适应抑制 OOD 控制信号，提升跨域鲁棒性。
3. **两种检测策略**：Top-k 检测（召回高、精度低）与共形预测（Conformal Prediction, CP）检测（自适应阈值、有分布无关的误报率保证，精度/召回更均衡）。
4. **大量实验**证明该方向「有效、实用、计算高效」，并发布代码与数据。

## Key Contributions

- **New problem formulation: Unsupervised Failure Attribution.** Train only on successful trajectories (no failure data, no step-level labels); localize error-contributing steps at inference time. This removes the costly/ambiguous step-level annotation and frontier-LLM inference bottlenecks of prior work (prompting pipelines; GRPO post-training on labeled failures).
- **Oat method.** One-class learning in LLM latent space via Neural CDEs modeling successful-trajectory dynamics, plus a novel gated control path that down-weights out-of-distribution spline derivatives for robustness to distribution shift.
- **Two detection heads.** Top-k (fixed budget, high recall) and Conformal Prediction (data-adaptive threshold with a formal false-positive-rate guarantee at level α on in-distribution steps).
- **Extensive, label-efficient evaluation.** Trained on ~100 successful trajectories and beats GPT-4o/GPT-5 prompting on both in-domain and OOD benchmarks while being 200–5000x faster with 0 tokens.

## Methodology

**问题形式化（Section 3）**：智能体轨迹 τ=(Q, a₁, s₁, …, a_T)，评估函数 Z(τ)∈{0,1}。失败归因即在 Z(τ')=0 时预测错误步骤集合 y(τ')⊆{1,…,T}。作者定义「失败贡献步骤」为任何有意义地推动轨迹走向失败的步骤（即使单独修复该步不足以挽救），从而能捕捉复合错误。

**步骤表示**：对每一步，取生成该动作时 LLM（Qwen3.5-27B）第 ℓ 层的 token 表示并做 mean-pooling 得到 h_t；查询单独编码为 h_Q；再用 PCA 降到 d_h=64 维。归一化时间到 [0,1]，把 (h_Q, h_1, …, h_T) 视作潜在连续路径的不规则采样观测。

**连续轨迹建模（Section 4.1）**：
- 指出 RNN/Transformer 把观测当孤立事件，无法刻画连续动态。
- 先讨论 Neural ODE（仅由初始条件驱动，无法表达「多条不同动作序列都能成功」），再采用 **Neural CDE**：通过三次样条构造控制路径 X(u)，其导数 dX/du 在每个积分步持续调制潜态（Riemann–Stieltjes 积分），使模型对每一步的局部行为敏感，利于定位局部偏差。
- 训练目标为重建损失 L(θ)=E[Σ‖h_t − ĥ(u_t)‖²]。
- **门控控制路径**：引入门函数 q(·)∈[0,1]^(d_h+1)，用 dX̃/du = q(dX/du)⊙dX/du 替换原导数；域内时全通，OOD 时抑制异常导数，防止控制路径注入误导动态。

**检测（Section 4.2）**：推理时对失败轨迹每步算异常分 e_t=‖h'_t − ĥ(u_t)‖²。Top-k 取最高 k 步；CP 用留出的成功轨迹校准集求 (1−α) 分位数 δ，超过 δ 的步骤即判为错误步骤，给出误报率水平 α 的形式保证。

**实现要点**：向量场 f 为 3 层 MLP（隐维 64），Euler 求解器；门控为 4 层 MLP（隐维 12）；k=3，α=0.2；学习率 4e-5、batch 32、300 epoch；对 OOD 表示还施加 CORAL 做二阶统计对齐。

## Results & Highlights

**域内（MCP-Atlas，Table 1）**——Oat 全面超越提示基线，F1 差距巨大：
- Oat(CP) F1 **0.435** vs GPT-4o 0.212 / GPT-5 0.181（**+20% 绝对 F1**）；AUPRC 0.324 vs ~0.217（**+10%**）；AUROC 0.629。
- Oat(Top-k) 召回 **0.706**、Hit **0.777**，明显高于所有基线（GPT-4o Hit 仅 0.250）。

**OOD（Who&When，Table 2）**——跨基准（单 Agent 工具调用 vs 多 Agent 协作；Qwen3.5-27B 生成 vs GPT-4o 生成）仍领先：
- Oat(Top-k) F1 **0.225** vs GPT-5 0.152（**+7% 绝对 F1**）；Oat(CP) AUROC **0.758**、AUPRC **0.128**（+5% AUPRC）。

**计算效率（Table 3）**——Oat 零 token 成本、亚秒级延迟，部署只需 <1 GB VRAM：
- MCP-Atlas：Oat 延迟 **7±4 ms**、0 tokens，对比 GPT-5 39626±11079 ms / 3012 tokens；GPT-4o 4241 ms / 121 tokens。
- Who&When：Oat **16±22 ms**，对比 GPT-5 37819 ms。整体快 **200–5000×**。

**消融（Section 6）**：
- Neural CDE 在所有指标上一致优于 Neural ODE（控制路径持续注入观测是关键）。
- 门控控制路径使 OOD AUROC **+0.172**，仅以 −0.028 域内性能为代价。
- Oat 一致优于 RNN 一类学习方法；后层（Qwen3.5-27B 第 64 层）表示远好于浅层；mean-pooling 优于 last-token；在 proxy-LLM（Llama-4-Scout / Gemma-4-31B / GPT-oss-120B）设置下仅小幅退化，说明可脱离生成模型使用。

## Critical Assessment

- **方法新颖性扎实但非凭空**：把 Neural CDE（时序/不规则采样建模工具）引入智能体轨迹诊断、并扩展到「步骤级」而非「轨迹级」异常定位，是论文真正的差异化点；相关工作清晰指出机器人领域的同类方法只做轨迹级检测，本文把它升级为更难的步骤级归因。
- **实验诚实，公开差距**：作者并未掩盖绝对值偏低这一事实——即便 Oat 最优，域内 F1 也仅 ~0.435、OOD 仅 ~0.225，且 GPT-4o/5 的 F1 普遍 <0.22，呼应了原 Who&When 论文「SOTA 推理模型 <15% 准确率」的结论，说明任务本身极难，本文是相对提升而非已解决问题。
- **消融与 case study 充分**：3 个成功 + 3 个失败案例（Appendix H）坦诚展示失败模式——例如 CP 的硬阈值会把因果上关键但分数略低的根因步漏掉（Case 3，step 5），以及对「次优计划」「显式失败口头化」的检测优于对早期潜藏推理错误的检测。
- **可控的威胁**：（a）标注由作者本人完成（Appendix E），「失败贡献步骤」本身定义带主观性，评估标签可能偏向 Oat 能识别的模式；（b）训练/评估的 MCP-Atlas 子集仅 103 成功/88 失败，规模偏小，88 条作测试集方差不可忽视；（c）跨域增益高度依赖 CORAL + 门控，去掉后 OOD 退化明显，说明泛化仍有脆弱性。

## Strengths

- **真正的实用价值**：零 token 成本、亚秒延迟、<1 GB VRAM、仅 ~100 条成功轨迹即可训练，使「实时失败归因」首次成为可能。
- **范式转换**：从「收集失败轨迹 + 步骤标注 + 重型 LLM 推理」转向「用成功副产品做无监督一类学习」，消除标注瓶颈，契合工程落地。
- **理论工具选型有说服力**：Neural ODE→CDE 的递进论证（ODE 仅靠初值无法表达多解成功）清晰；CP 检测提供分布无关的误报率保证。
- **可复现性强**：完整超参表、数据统计、标注规程、匿名代码、可视化工具一并公开。

## Limitations

- **绝对性能仍偏低**：即便最优配置，域内 F1 ≈0.435、OOD F1 ≈0.225，离「可靠自动诊断」尚远，召回（CP 模式）尤其偏低（域内 0.484、OOD 0.330）。
- **评估标签由作者自标**：「失败贡献步骤」定义含「meaningfully degrades」这类主观判断，存在自评偏置；附录承认对「探索性重复检索」「不一致推理」的处理带有裁量。
- **数据规模小且单一代理族**：MCP-Atlas 子集仅 203 任务、88 失败轨迹；OOD 仅一个基准（Who&When），且都偏向工具调用/检索类任务，未覆盖编码（SWE-bench 类）、长程科研等高风险场景。
- **根因 vs 衍生错误难区分**：case study 显示模型对「次优计划」「显式失败口头化」敏感，但对早期、隐式推理根因不敏感；CP 硬阈值会漏掉略低于阈值的因果关键步。
- **作者自述的扩展空间**：未尝试 Continuous Normalizing Flows（可用负对数似然替代重建误差）、带注意力的控制路径、Neural SDE（建模轨迹随机性）等更 principled 的变体。

## Relevance

- 对「LLM 智能体可观测性 / 调试 / 审计」方向高度相关：失败归因是部署 coding agent、科研 agent 等高风险系统前的关键瓶颈，本文给出轻量、可实时的方案。
- 对「一类学习 / 异常检测」社区有方法迁移价值：把 Neural CDE 用于离散智能体轨迹的步骤级异常打分，是一次跨领域方法落地。
- 对工业界落地友好：成功轨迹是正常运行的免费副产品，无需额外标注；<1 GB VRAM、亚秒延迟可直接挂载到现有 agent pipeline 做在线诊断。
- 与本仓库（daily-paper）关注的「LLM agents、可信性、效率」主线一致，可作为失败诊断方向的代表性工作纳入跟踪。

## Questions for Further Exploration

- 用 Continuous Normalizing Flows 把异常分换成「步骤在学到的成功分布下的负对数似然」，是否能提升对隐式根因的捕捉？与重建误差相比增益多大？
- 失败贡献步骤的「真值」本身就含主观裁量，能否设计不依赖单点标注的评估协议（如基于干预/反事实的因果效应度量）来更客观地评测？
- 在 SWE-bench、长程科研 agent 等更长（数十到数百步）轨迹上，Neural CDE 的数值稳定性与计算成本如何？是否需要换更高阶求解器或分段建模？
- 能否把 Oat 的异常分作为反馈信号，闭环用于在线纠错或 RL 训练（与 AgentRacer/GraphTracer 的后训练路线融合），而非仅做离线诊断？
- 门控函数 q(·) 目前是固定 MLP，能否做成与查询/任务自适应（如基于校准集的在线门控），以在更多 OOD 域上稳定收益？
- CORAL + 门控带来的 OOD 增益是否在第三个、结构差异更大的基准上仍然成立？需要更多跨域实验来排除「恰好适配 Who&When」的可能。
