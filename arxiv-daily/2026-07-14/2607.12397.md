> **一句话总结：** 本文提出 Critic Experience Bank (CEB)，一种无需训练、自我进化的评论家框架，利用事后回看的执行反馈为 LLM 智能体的每一步动作在执行前提供校准的置信度估计，在三个智能体基准和三个评论家骨干上取得全部 ECE/Brier/AUC 最优，相对最强免训练基线 ECE 降低高达 54%。

> **TL;DR:** CEB is a training-free, self-evolving critic that augments a frozen LLM with hindsight-labeled execution experience retrieved as contrastive precedents, achieving the best ECE/Brier/AUC across Mind2Web, AMEX, and InterCode-Bash with GPT-4.1, GPT-5.4, and Qwen3-Next-80B critics, reducing ECE by up to 54% over the strongest training-free baseline and closing up to 95% of the calibration gap to a ground-truth oracle.

## 核心贡献

本文针对 LLM 智能体部署中的"动作执行前可靠性评估"问题，提出三点贡献：

1. **形式化"步骤级置信度估计"任务**：在执行动作之前，估计该动作在当前任务信息、环境状态、交互历史下是否会促进任务进展（即生产性 step productivity），输出校准概率 ż_t ∈ [0,1]。这与响应级置信度有本质区别——智能体可靠性取决于执行后果，而非仅依赖当前上下文。

2. **提出 Critic Experience Bank (CEB)**：一种免训练、自进化的评论家框架。每条轨迹完成后，一个"事后回看"（hindsight）LLM 看到完整执行反馈，对每一步投票判定是否生产性，形成伪标签存入经验库；在新步骤评分时，按 task∥state∥action 复合键检索相似的"成功/失败"对比性经验注入评论家上下文。评论家权重始终冻结，仅靠经验证据实现适应。

3. **跨 9 个数据集-评论家组合的全面实证**：在三个基准（Web/Mobile GUI/Shell）×三个骨干上，CEB 在所有 ECE、Brier、AUC 单元格均为最优，支持选择性执行（selective execution）并在固定人工审核预算下提升最终任务成功率。

## Key Contributions

1. **Formalizes step-level confidence estimation for LLM agents** — producing a calibrated probability that a proposed action is productive *before* it changes the environment state, a problem distinct from response-level confidence because usefulness is determined by execution consequences.

2. **Introduces CEB, a training-free self-evolving critic** that augments a fixed LLM critic with a hindsight-labeled experience bank. After each trajectory, a hindsight LLM votes on each step's productivity; these pseudo-labels (no ground truth needed) populate contrastive (productive vs. unproductive) records retrieved into the critic's prompt when similar steps recur.

3. **Comprehensive empirical dominance**: best ECE, Brier, and AUC in all 9 dataset-critic combinations across Mind2Web, AMEX, and InterCode-Bash; ECE reduced up to 54% relative to the strongest training-free baseline; closes up to 95% of the calibration headroom to a ground-truth oracle bank.

## Methodology

**问题设定。** 轨迹 τ = (s₁,a₁,o₁),…,(s_T,a_T,o_T)，其中 s_t 为执行前环境状态，a_t 为提议动作，o_t 为执行后观测。步骤级置信度估计器需在执行前输出 ż_t = f(q, s_t, h_<t, a_t)，注意不能使用未观测的 o_t。目标事件 y_t = 1 当且仅当该动作正确或取得进展（仅在评估时使用）。

**CEB 框架。** 估计器实例化为冻结 LLM 评论家加经验库：ż_t = f_θ(q, s_t, h_<t, a_t; B_t)，其中 θ 固定，B_t 为历史轨迹的回看标注记录。核心循环（Algorithm 1）：
- **在线评分**：每步用复合键 κ_t = task:q ∥ state:s_t ∥ action:a_t 从类拆分库 B_t = B_t⁺ ∪ B_t⁻ 中 TF-IDF 检索每类 top-k（默认 k=2），连同重复标志 ρ_t（检测近 3 步内的短循环）注入评论家提示，输出 JSON 分数 clip 到 [0,1]。
- **事后标注**：轨迹完成后，回看 LLM g_φ 看到完整 (s,a,o,q)，采样 V=5 次（温度 0.7）对每步投票，多数票得伪标签 ŷ_t。
- **记录构建**：每条记录 e = (q', s', a', õ', ŷ', ż', χ')，其中 χ' 为"原始二元判断是否与回看结果一致"的 agreement flag，让评论家看到自己历史上高估/低估之处。

**检索机制。** 采用 1–2 gram TF-IDF 余弦相似度（非稠密向量）。论文附录 F 实验表明：检索目标是词法结构性的（近乎重复的动作在近乎相同的页面状态上），稀疏词项重叠直接捕捉生产性 vs. 非生产性选择的精确 token 差异，而通用句向量会平滑掉这些关键 token；TF-IDF 还无需额外模型/API 依赖，符合免训练设计。

## Results & Highlights

CEB 在所有 9 个 dataset×critic 组合上同时取得最优 ECE、Brier、AUC（Table 1），具体量化亮点：

1. **ECE 相对降低幅度**：在 Mind2Web + GPT-5.4 上相对最强免训练基线降低 14%；在 InterCode-Bash + GPT-4.1 上降低高达 **54%**。

2. **关闭到 GT 神谕的校准间隙**：CEB 仅用伪标签即恢复了冻结评论家与"完美标注经验库"之间校准余量的 **最高 95%**（Table 6 的 Closed 列），证明伪标签噪声并非残差误差的主要来源。

3. **自进化曲线（Figure 3）**：在 Mind2Web + GPT-5.4 上随任务流累积 ECE 优势单调增长——30 任务后 +0.073，50 任务后 +0.104，150 任务后 +0.136，全流达 +0.142（5 个随机流序均值），而无记忆 Static 评论家保持平坦。

4. **选择性执行（Table 4）**：在 Mind2Web + GPT-5.4 上，弃权最低置信 50% 步骤时，CEB 保留步骤正确率达 +0.090 增益，而 Mean Entropy 反向对齐（0.198→0.114，越弃权越差），Verbalization 曲线几乎平坦，CoT 仅 +0.036。

5. **最终任务成功率（Table 5）**：在 m∈{1,2,3} 的人工审核预算下，CEB 在 Mind2Web 和 AMEX 上均接近逐任务神谕，而 token 不确定性基线（ME、PPL）因任务内方差小获益最少。

6. **离线置信度门控重生成（Appendix G, Table 12）**：在 Mind2Web 200 任务子集上，CEB 在 b=10% 时阳性率 0.201（神谕 0.202）、b=25% 时 0.234（神谕 0.236），几乎完全追踪神谕；PPL/ME 落到 Random 之下（会破坏正确步骤）。

7. **LLM-as-judge 宽松标签下（Table 2）**：在 Mind2Web 和 AMEX 上用 GPT-5.4 法官重新标注（承认替代有效路径），CEB 在所有 dataset-critic 单元格仍最优，说明优势非严格参考匹配的人为产物。

8. **基线诊断**：token 不确定性（PPL、ME）在 web 和 shell 上 AUC<0.5（排序倒置）；verbalized CoT 置信度因不利用相似动作的历史后果而校准弱。

## Critical Assessment

论文的核心洞察准确且有价值：**智能体置信度估计的是"执行事件"而非"生成响应"**——一个在当前上下文下看似合理的动作，其有用性由执行后到达的状态决定（可能推进任务、陷入循环、或使后续进展更难）。Figure 1 的诊断显示，现成的 token 不确定性和 verbalized 置信度信号随轨迹深度不稳定，而 CEB 通过检索执行经验保持更稳定的校准，这一动机论证扎实。

方法设计上几个非平凡选择值得肯定：(1) **对比性类拆分库** B_t⁺ ∪ B_t⁻ 让评论家同时看到相似的成功和失败案例，消融（Table 3）证明这是最重要组件；(2) **复合键检索**（task+state+action 而非仅动作文本）避免短动作字符串跨状态误匹配；(3) **agreement flag χ'** 让评论家"看见自己过去的错误模式"，把经验库诠释为"评论家自身系统性偏差的先验"。附录 F 对 TF-IDF vs. 稠密检索的诚实对比（TF-IDF 更好且无额外依赖）也增强了免训练叙事的自洽性。

但论文在"选择性执行"和"最终任务成功率"的评估上采用了**重放式（replay-based）而非闭环**设定（§4.5 明确声明"does not claim a closed-loop replanning policy"），即假设弃权/重生成步骤会被完美修正，这放大了置信度信号的下游价值，与真实人机协同场景存在差距。作者已在限制中诚实地承认这一点。

## Strengths

- **任务定义精准**：将 step-level confidence 形式化为执行前估计，明确区分于响应级置信度，填补了部署阶段（无 GT 标签、模型冻结、需执行前打分）的工具空白。
- **机制清晰且自洽**：经验库 = 评论家自身偏差的先验，检索 = 把先验变为演化上下文，权重始终冻结，整套叙事与方法设计高度一致。
- **实证全面且强势**：9/9 dataset-critic 组合全面最优，跨严格参考匹配与宽松 LLM-as-judge 两套标签均稳健，ECE 降低幅度（最高 54%）和关闭神谕间隙（最高 95%）令人信服。
- **诊断深入**：token 不确定性 AUC<0.5 的倒置现象、可靠性图（附录 E）、检索机制对比（附录 F）、置信度门控重生成（附录 G）等多角度佐证机制有效性。
- **工程务实**：默认 k=2、V=5 的选择有 token 成本分析支撑（附录 D），TF-IDF 无额外依赖，部署友好。

## Limitations

作者自陈及我观察到的局限：

1. **评论家与标注器同模型**：所有实验中 hindsight labeler g_φ 与评论家使用同一骨干，未隔离独立标注器的影响——若评论家本身对某类步骤系统性误判，伪标签会继承同一偏差。
2. **标签仍依赖外部标注**：Mind2Web/AMEX 用参考匹配（对探索性步骤保守）或 LLM-as-judge（法官与评论家同模型，循环风险）；InterCode-Bash 的奖励衍生标签可能有噪声。
3. **经验库无驱逐/压缩机制**：当前基准规模（200–1009 轨迹）下无问题，但大规模部署流会导致检索成本与提示长度膨胀，作者承认这是未来工作。
4. **评估为重放式非闭环**：选择性执行和任务成功率均假设弃权/审核步骤被完美处理，未验证真实闭环重规划策略的效果。
5. **仅离线流式评估**：经验库按流序累积，但未在真正在线、持续运行的智能体系统中验证鲁棒性与分布漂移。
6. **跨任务泛化边界未探**：经验库按 task 检索，跨异构任务的迁移性（如从 web 到 mobile 的经验复用）未单独分析。

## Relevance

本文对当前 LLM 智能体安全部署与人机协同研究高度相关：
- **安全关键场景**：表单提交、文件删除、数据库修改等不可逆动作，需要在执行前有可靠的"是否执行"决策信号，CEB 的 step-level 校准直接服务于该需求。
- **人机协同（HITL）**：固定人工审核预算下的任务成功率提升（Table 5）直接对应真实审核工作流——审核员只检查每个任务置信度最低的几步。
- **免训练部署范式**：在固定 API 模型、无标注数据的现实约束下，CEB 提供了比 process reward model 更轻量的替代路径，符合闭源大模型时代的工程现实。
- **记忆/经验研究方向**：与 Reflexion、ExpeL、ReasoningBank 一脉相承，但首次将记忆放在 critic 侧而非 actor 侧，开辟了"校准型经验记忆"新角度。

## Questions for Further Exploration

1. **独立 hindsight 标注器**：若用更强或更弱的独立模型做回看标注，CEB 的校准增益如何变化？伪标签质量与评论家能力是否存在耦合阈值？
2. **闭环重规划**：将 CEB 的低置信度信号接入真实重规划/人类审核管线后，端到端任务成功率和延迟的权衡如何？重生成是否会引入新的失败模式？
3. **经验库规模与压缩**：在 10k+ 轨迹的持续部署流中，需要怎样的驱逐策略或聚类压缩？是否存在经验"饱和点"（边际增益趋零）？
4. **跨模态/跨任务迁移**：web 导航积累的经验能否迁移到 mobile GUI 评分？复合键中 task 字段是否会过度隔离不同任务的共享模式？
5. **对抗性分布漂移**：当环境或任务分布随时间漂移时，陈旧经验是否会导致系统性误校准？是否需要经验时效加权？
6. **与 process reward model 的互补性**：在有少量标注过程数据可用时，CEB 的检索经验能否作为 PRM 的特征增强，而非替代？
7. **稠密检索的潜力**：附录 F 显示通用句向量不如 TF-IDF，但若用针对动作/状态微调的专用 retriever，能否超越 TF-IDF？
8. **多智能体场景**：多个智能体是否可共享一个经验库？跨智能体经验是否同构可用，还是会引入噪声？
