Controlled Null Result · GRPO × Web Agent · 2026-07-14

On mastered tasks, GRPO adds no credible skill to a competent web agent — and mid/high learning rates degrade it.

A Learning-Rate-Gated Failure of GRPO in a Small Language and Vision-Language Model Web Agent: A Controlled Null and Its Mechanism

Paired interval [+0.0, +10.9] touches zero; McNemar p = 0.50–0.69 — indistinguishable from noise. Learning rate is the only dial that moves success, and only downward (to −49.1 pp at lr 2e-5).
same recipe on headroom tasks: +22.0 pts [+8.0, +40.0], p=.007
→ Why this matters

Before spending RL compute, measure Δ_head = SR_sample − SR_greedy on a held-out set; if ≤ 0, spend the budget on distillation, not RL.

A pilot distillation run lifted the baseline ~30 pts on the very tasks where RL was a no-op. For 4B–8B agents, capability ceilings are set by supervised quality, not by more RL.

! Critical edge
  • Harm is text-track only. Under Set-of-Marks the high-rate drop is nominal (p = 0.070, interval touching zero).
  • Mechanism is scale-scoped. The rank→capability link holds at 4B but breaks at 8B — not a universal law.
  • Single benchmark (MiniWoB) and one model family; WebArena avoided for determinism.

Methodology

one agent · one reward · every knob swept

Agent & environment SETUP

Qwen3-VL 4B (and 8B) on MiniWoB with a compact action vocabulary (click / type / select / navigate / finish). Two observation tracks share all logic: a serialized text track (grid + mechanism) and a Set-of-Marks screenshot track (modality check).

GRPO recipe RL

Group mean-centered advantage (no std division), DAPO-style asymmetric clip ε_lo=.20 < ε_hi=.28, optional KL anchor (non-negative k3 estimator vs frozen init), linear warmup + cosine decay.

The 18-run grid SWEEP

Varies learning rate, KL weight β, seed, initialization (SFT/base), clip bound; ablations add schedule and group size G∈{8,16,32}. Extended to 8B and the SoM track.

Reward SIGNAL

Sparse binary terminal signal r(τ)=1[success(τ)]∈{0,1}, broadcast to every step. No shaping.

Statistics RIGOR

Greedy decode, 11 tasks × 5 seeds = 55 matched episodes. Wilson intervals; exact paired McNemar; task-clustered bootstrap on ΔSR; TOST equivalence. "Credible" = interval excludes zero.

Causal grafting MECHANISM

On a cache of N=1999 hidden states: effective rank, participation ratio, argmax agreement, entropy. Restore one component (attention / MLP / embedding) from init, re-evaluate; a 24-draw random-component null calibrates specificity.

Restoring attention or MLP recovers frontier success 11% → 37/40% (above the random-null 22.9% threshold). The largest-drift embedding is causally inert (14%) — weight movement does not predict failure.

DEGRADE mid LR

Collapses late-layer effective rank (L35: ~9 → ~1); early layers intact. Localizable — a single component restore fixes it. Failure mode: 63% invalid output.

COLLAPSE high LR

Preserves / raises late rank but drives argmax agreement to zero. Distributed readout corruption — no single group repairs it. Failure mode: 98% invalid output.

No credible gain, mastered grid. 4 of the top 5 configs sit within ±5 pts of SFT under TOST; one is episode-identical.
The task, not the step size. The low LR that was a pure no-op on mastered tasks climbs +12–18 pts on headroom tasks.
Positive control is decisive. Same harness, same reward: SFT 20.0% → 42.0% (p=.007) where sampling already beats greedy.
Apparent edge was a lucky seed. 6-seed replication averages 49.7% — back on baseline; the borderline arm loses credibility once clustered.
Not reward hacking. Premature-finish ≈ 0% in every regime; degrade = 63% invalid-output, collapse = 98%.
Scale boundary. At 8B, L35 rank collapses to 1.9 yet argmax agreement stays 0.954 with intact success — rank & agreement never fall together.

Strengths & Limitations

side by side
+ Strengths
  • Tight control. One agent, one environment, one reward, one task set — every RL knob swept, so SR changes attribute cleanly to the intervention.
  • Statistics done properly. Paired McNemar on matched episodes, task-clustered bootstrap, TOST equivalence — "credible" defined conservatively (interval excludes zero).
  • Positive control is the decisive design. Converts "GRPO doesn't help here" from a broken-pipeline worry into "it fails exactly where a working GRPO would also fail."
  • Causal, not correlational, mechanism. Grafting + random-component null directly overturns movement-based attribution to the embedding.
  • Honest scoping throughout. Text vs SoM, 4B vs 8B, mastered vs headroom; single-checkpoint noise explicitly flagged.
– Limitations
  • Single benchmark & family. MiniWoB (+ Qwen3-VL 4B/8B) chosen for determinism; WebArena avoided as run-to-run noise would confound the mechanism.
  • Harm is text-track only. Under Set-of-Marks the high-rate drop is nominal (p = 0.070, interval touching zero) — not a universal claim.
  • Mechanism is scale-scoped. Rank→capability coupling holds at 4B but breaks at 8B; the rank account is a smaller-model property, not a law.
  • Few checkpoints per regime. Grafting and rank-health rest on a small number of checkpoints; point estimates are noisy, reported qualitatively.
  • Scoped null. Concerns only the mastered-task regime. Authors make no claim that RL is useless for web agents in general.

Open Questions

for further exploration
Q1

Does the headroom-gap rule survive noisier, deployment-realistic environments (WebArena, real browsers)? What statistical correction offsets environment variance?

Q2

At 7B–14B, is the rank→capability break a smooth transition or a threshold? What geometric factor (params, layers) decides the coupling?

Q3

If the lesion lives in attention/MLP, can an RL scheme that freezes/constrains just those blocks avoid degrade while keeping any gain?

Q4

Can a KL variant or auxiliary loss suppress late-rank collapse while still permitting genuine capability gain (the current KL only holds argmax, inflating entropy)?

Q5

On a mixed task distribution (partly mastered, partly headroom), what is GRPO's net effect — and can a curriculum allocate compute by headroom?

Q6

If embedding drift is causally inert, can embeddings be safely frozen during GRPO to save compute, or serve as a stable cross-task anchor in distillation?
