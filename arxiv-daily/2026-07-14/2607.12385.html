PM-Bench: do LLM agents act on the intentions they remember?

Genglin Liu · Saadia Gabriel — UCLA arXiv:2607.12385 ↗

The finding

Even the strongest frontier-model agent acts on only 65.1% of the deferred future intentions it should — and scaffolding that monitors more does not monitor better.

01

Method — a virtual week that isolates when

The scenario

A 7-virtual-day sequence of decision points t ∈ {1…T}. Each step gives a narrative vignette, 3 ongoing-activity options (A/B/C), and an anonymous action menu mixing true tasks with lures — so agents can't win by memorizing stable IDs.

11 hidden state channels (clock, email, calendar, price_tracker, shipment_status…) become observable only via explicit query_state calls.

Due set — the ground truth

At each step the due set is the set of valid tasks whose execution condition is satisfied right now:

D_t = { τ ∈ X_t : τ valid ∧ exec_cond(τ) at t }

The agent commits to (ongoing_choice, subset_of_actions) via a query-then-act protocol. Headline metric: Set-F1, because precision rewards under-acting and recall rewards spamming.

Generation pipeline

1Sample 7 day archetypes with irregular schedules
2Instantiate routine + event + one-off + channel tasks, plus 10–12 lures/day
3Inject cross-day tasks and cancel / reschedule / override events
4Validate via 10 structural invariants + perfect-play oracle

The oracle monitors every channel and executes all due tasks — so any failure is attributable to the model, not to an unsolvable scenario.

Eight agent configurations

  • ① Single-agent baseline
  • ② Todo-ledger (≤5 items)
  • ③ Optional-heartbeat
  • ④ Auto-HB-60
  • ⑤ Auto-HB-30
  • ⑥ Hierarchical (1+3 agents)
  • ⑦ Replay majority-vote †
  • ⑧ Replay unanimous-vote †

Crossed with 8 frontier models (GPT-5.4, GPT-5.3-Codex, Llama-3.3-70B, Mistral-Large-3, Qwen3-8B/14B/32B…) = 64 full runs.

02

Results — no scaffold wins; the trade is precision vs. recall

The strongest setup clears only two-thirds of due actions. Hierarchical multi-agents flood the channels with queries yet collapse on Set-F1 — the bottleneck is action aggregation, not evidence acquisition.

Takeaway A

No universal best scaffold. Backbones have different weaknesses — interventions interact, they don't add uniformly.

Takeaway B

More reminders ≠ better. Auto-HB-30 wins hit-rate yet 489 false positives drag Set-F1 to 57.8%.

Takeaway C

Monitoring is not the gap. 1,661 queries buy only 5.0% channel hit — aggregation is the bottleneck.

03

Strengths & limitations, side by side

Strengths

  • Real cognitive construct. Grounded in the validated Virtual Week paradigm (Rendell & Craik 2000) — construct validity ad-hoc benchmarks lack.
  • The right diagnostic lens. Set-F1 with explicit FP/FN exposes the spam-vs-underact tradeoff that hit-rate metrics hide.
  • Comprehensive scaffold sweep across 8 models — the negative result "monitoring more ≠ better" is genuinely useful.
  • Oracle-audited artifact. Deterministic, seeded, with public code — forecloses the "impossible benchmark" critique.
  • Anonymous reshuffled handles prevent ID-memorization wins; success requires reading current due conditions.

Limitations

  • Single evaluation week. No variance across seeds; per-model rankings (Table 3) may not generalize.
  • Synthetic, text-only, single-user domestic. Multi-user / multi-modal / high-stakes workflows untested — the ethics statement itself flags this.
  • No human baseline. Virtual Week was validated on humans, yet "how far from the human ceiling" is unanswered.
  • No fixes proposed for the diagnosed cross-day / non-clock gaps — by design a benchmark paper.
  • Near-floor channel results (best 16.7%, many 0%) give little signal to discriminate current models on that slice.
04

Open questions — where it goes next

Multi-week robustness

Does the per-model best-scaffold ordering (Table 3) survive different seeds? How large is Set-F1 variance across generated weeks?

The human ceiling

Recruit human participants on the same text interface — how far is 65.1% from human performance, and which task classes are hard for humans too?

Closing the near-floor gaps

What training target (prospective-memory fine-tuning, RL on due-set prediction) or inference mechanism (query-budget scheduling) lifts cross-day ≤50% and non-clock ≤16.7% without false positives?

Better aggregation

If the bottleneck is action aggregation not evidence, can a confidence-gated coordinator beat union-query / majority / unanimous?

Cost-benefit of monitoring

Learn a per-channel "query value" estimator so agents pick an optimal budgeted query policy across 11 channels instead of blind heartbeat.

Orthogonal to context length?

Do recursive / ultra-long-context models actually win here, or is prospective memory a genuinely separate axis from context length?
