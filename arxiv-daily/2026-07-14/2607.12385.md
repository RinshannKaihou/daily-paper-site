> **一句话总结：** 作者提出 PM-Bench，一个受认知科学"虚拟一周(Virtual Week)"范式启发、用于评测 LLM 智能体"前瞻记忆"能力的文本基准；在 8 个前沿模型 × 8 种智能体配置的 64 次评测中，最佳方法（GPT-5.4 + 可选心跳）仅取得 65.1% 的 Set-F1，揭示出"该出手时不出手"仍是当前智能体的核心瓶颈。

> **TL;DR:** PM-Bench is a text-based benchmark (adapting cognitive science's Virtual Week paradigm) that measures whether LLM agents execute deferred intentions at the correct future cue/time while continuing an ongoing activity; across 8 frontier models (incl. GPT-5.4) × 8 agent configurations, the best setup reaches only 65.1% Set-F1, and no single scaffolding strategy dominates—precision-recall calibration of *when to act* remains the dominant open problem.

## 核心贡献

1. **首次系统化将"前瞻记忆(Prospective Memory)"作为独立评测轴**：区别于已有大量聚焦"回溯记忆/检索(retrospective memory retrieval)"或长上下文窗口的工作，PM-Bench 干净地隔离了"一个已存储的意图是否会在正确的未来时刻被执行/更新/抑制"这一能力。作者论证：更强的上下文窗口或更好的检索并不天然解决前瞻记忆问题。
2. **受心理学经典范式启发的可扩展生成流水线**：将认知科学的 Virtual Week 棋盘式测试（Rendell & Craik, 2000）改造成可被现代工具型 LLM 智能体执行的文本场景——一个不规则排程的七天虚拟周，每天 10–13 个时间锚点。涵盖事件触发(time-/event-based)、跨天(cross-day)、取消/改期/覆盖(update-sensitive)、以及需要主动查询的隐藏状态通道(channel-triggered)四类任务，并辅以匿名句柄+lure 干扰项防止"背 ID"。
3. **可控、可归因的基准工件**：通过 schema/一致性/可解性三重校验 + "完美游玩(perfect-play)审计"（oracle 监控所有通道并执行所有到期任务），保证所有失败均可归因于模型行为而非"无解题目"。给定 seed 完全确定性可复现。
4. **细粒度评测协议**：主指标为步级集合 F1（Set-F1），并附带 miss / late / false-alarm / wrong-content / update-violation / dependency-violation / exact-set-match / 查询次数等诊断细分，外加"是否需要主动监控(proactive-required)""是否为非时钟通道(non-clock)"等切片。
5. **大规模实证**：8 个模型（GPT-5.4、GPT-5.3-Codex、Llama-3.3-70B、Mistral-Large-3-675B、mistral-small-3.2-24b、Qwen3-8B/14B/32B）× 8 种智能体配置 = 64 次完整运行，揭示核心失败模式为"精度-召回权衡"，而非单纯遗忘。

## Key Contributions

- PM-Bench: the first benchmark that isolates **prospective memory** (acting on a stored intention at the right future cue) as a distinct evaluation axis from retrospective memory / long-context retrieval.
- A seeded, deterministic, synthetic scenario pipeline adapted from the **Virtual Week** cognitive-science paradigm, validated by schema/consistency/solvability checks plus a perfect-play oracle audit that attributes all failures to model behavior.
- A replay-based evaluator with Set-F1 as the headline metric and a rich diagnostic error taxonomy (miss / late / false-alarm / wrong-content / update-violation / dependency-violation, plus query counts and proactive-required slices).
- 64 full evaluations (8 frontier models × 8 agent setups) covering single-agent, todo-ledger, optional/auto heartbeat, and hierarchical multi-agent configurations, exposing a precision–recall calibration tradeoff rather than simple forgetting.

## Methodology

**Scenario structure.** A PM-Bench scenario is an ordered sequence of decision points $t \in \{1,\dots,T\}$ spanning 7 virtual days. Each step yields a visible observation $o_t$: a short narrative vignette ("You make breakfast"), three mandatory ongoing-activity options (A/B/C, e.g. "Pour coffee / Toast bread / Check the fridge"), and an anonymous action menu of handles `task_N` (mixing true tasks with lures). Hidden state channels $\mathcal{C}$ (clock, email, calendar, course_portal, price_tracker, bank_balance, shipment_status, laundry_status, library_hold, reservation_waitlist, appointment_portal — 11 in total) become observable only via explicit `query_state` calls.

**Due set formalization.** At each step the ground-truth *due set* is
$$D_t = \{\tau \in \mathcal{X}_t : \tau \text{ remains valid and its execution condition is satisfied at step } t\},$$
where $\mathcal{X}_t$ is the set of currently active task specs. Event tasks fire on a cue in narrative or queried channel; time tasks fire when current step time matches target (within an allowed window). The agent commits to $a_t = (c_t, A_t)$ — ongoing choice + subset of prospective actions — via a **query-then-act** protocol.

**Generation pipeline (4 stages).** (1) Sample 7 distinct day archetypes with irregular schedules; (2) instantiate 4 routine + 2–4 narrative-event + 1–2 one-off + 1–3 channel-triggered tasks plus 10–12 lures/day; (3) inject cross-day tasks and cancel/reschedule/override events; (4) validate via 10 structural invariants (unique IDs, monotonic step times, realizable cues, valid update targets, no backward same-day reschedules, no stale canceled instructions, no time leakage, valid cross-day exposure, acyclic dependencies, process-channel consistency) + perfect-play oracle. Channel taxonomy distinguishes delta (event-like updates), snapshot (current status), and process-style snapshot (staged lifecycle, e.g. shipment `in progress`→`complete`).

**Eight agent configurations evaluated.** (1) Single-agent baseline; (2) Todo-ledger (in-context ledger of pending intentions, ≤5 items); (3) Optional-heartbeat (agent toggles periodic nudge at 30/60-min); (4–5) Auto-heartbeat-60 / Auto-heartbeat-30; (6) Hierarchical union-query multi-agent (1 coordinator + 3 subagents: event_watcher, status_watcher, update_watcher); (7–8) Replay-based majority-vote / unanimous-vote ablations over the same hierarchical traces.

**Metrics.** Set-F1 = $2TP/(2TP+FP+FN)$ accumulated over steps (TP/FP/FN computed against $D_t$). Chosen because precision alone rewards under-acting and recall alone rewards spamming.

## Results & Highlights

- **Best overall: GPT-5.4 + optional-heartbeat single agent at 65.1% macro Set-F1** — i.e., the benchmark is far from solved even with the strongest frontier model + best scaffold. The todo-ledger variant is a close second at 62.8% Set-F1 with the lowest aggregate false-positive count among competitive setups.
- **No universal winning scaffold.** Different backbones prefer different interventions (Table 3): inference-time scaffolds interact with backbone-specific weaknesses rather than providing uniform gains. Auto-heartbeat-30 wins on update hit rate (47.2%) and clock-monitoring-required hit rate (60.4%) but pays with 489 false positives, dropping Set-F1 to 57.8% — "more reminders do not automatically yield a better operating point."
- **Hierarchical multi-agent monitors heavily but underperforms single agents on Set-F1**: it issues 1661 total queries (an order of magnitude more than any single-agent config) yet narrative-cued hit rate drops to 45.2% and channel-monitoring hit stays at only 5.0%, with cross-day hit of just 10.7%. Replay-based majority-vote boosts monitoring-required hits but collapses to 37.2% Set-F1 with 655 false positives — showing the bottleneck is **action aggregation, not evidence acquisition**.
- **Proactive (non-clock) monitoring is essentially unsolved**: clock-monitoring-required tasks can exceed 50% for strong setups, but tasks requiring other hidden-channel monitoring stay weak; several methods remain exactly 0% on multiple non-clock channels, and the best channel-monitoring hit rate is only 16.7%.
- **Cross-day and update-sensitive tasks remain unsolved across all setups**: best cross-day is optional-heartbeat at 50.0% hit (single-baseline 44.6%); best update hit is auto-HB-30 at 47.2%. No setup exceeds 50% on either long-horizon slice.
- A "catastrophic-forgetting-like" asymmetry: hierarchical multi-agent variants achieve *higher* hit rates on clock-monitoring tasks than on narrative-cued tasks (45.2%), suggesting agents can recover time-based intentions once they check the clock yet lose control on tasks directly executable from visible narrative.

## Critical Assessment

The paper's central conceptual move — separating **prospective memory** from retrospective retrieval — is well-motivated and arguably correct: most existing long-horizon/memory benchmarks (LongMemEval, MemGPT, RealMem, AMA-bench, MemBench) test whether content *can be recovered*, not whether a stored intention is *executed at the right moment*. The perfect-play oracle + 10 invariants is a genuine methodological strength: it forecloses the "benchmark is impossible / artifact-confounded" critique and forces failures onto model behavior.

However, several cautions temper the claims:

- **Single deterministic week.** All 64 runs are evaluated on one fixed, seed-generated, manually-reviewed week. Despite rejection-sampling for solvability, a single scenario risks lexical/cue-family overfitting; the authors acknowledge cue families are "deliberately varied" but do not report variance across multiple generated weeks. Per-model ranking (Table 3) could shift under different seeds.
- **Headline numbers rest on one scenario.** Claims like "no universal best scaffold" are compelling *within* this week but the paper provides no confidence intervals or multi-week robustness analysis — a notable gap for a benchmark paper.
- **The "catastrophic-forgetting-like" interpretation** of the narrative-vs-clock asymmetry in hierarchical agents is offered as a working hypothesis ("suggests"), not mechanistically demonstrated; it could alternatively reflect the coordinator's aggregation bias.
- **Replay ablations are not fresh rollouts** (marked †): majority/unanimous vote reuse the hierarchical union-query traces, so they isolate the aggregation rule but not the joint query+aggregation policy a real system would deploy.
- **Manual review of synthetic data** is a potential author-bias vector, though mitigated by the deterministic seeded pipeline.
- The 11-channel taxonomy and 4-stage generator are described in the appendix but the released week's exact composition (Table 1) is summarized only in aggregate; reproducibility hinges entirely on the released code/seed.

## Strengths

- Clean, well-grounded operationalization of a real cognitive construct (prospective memory) with strong psychology literature backing — gives the benchmark construct validity that ad-hoc agent benchmarks often lack.
- The precision–recall framing is the right diagnostic lens: Set-F1 with explicit FP/FN bookkeeping exposes the "spam vs. under-act" tradeoff that pure hit-rate metrics hide.
- Comprehensive scaffold comparison (single / ledger / heartbeat×3 / hierarchical / 2 voting ablations) across 8 diverse models — the negative result "monitoring more ≠ monitoring better" is genuinely useful for the field.
- Deterministic, validated, oracle-audited benchmark artifact with public code lowers the bar for follow-up work.
- Query-then-act protocol with anonymous reshuffled handles elegantly prevents agents from winning by memorizing stable task IDs — success genuinely requires acting from current due conditions.

## Limitations

- **Single evaluation week** — no variance/robustness across multiple seeds or weeks reported; per-model scaffold rankings (Table 3) may not generalize.
- **Synthetic, text-only, single-user-domestic scenario distribution** — generalization to real multi-user, multi-modal, or high-stakes professional workflows is untested; the ethics statement itself flags this.
- **No human baseline.** Virtual Week was validated on humans, but the paper reports no human pilot on PM-Bench, so the "how far from human ceiling" question is unanswered.
- **Limited failure-mode mitigation proposed.** The paper diagnoses the precision-recall tradeoff and unsolved cross-day/update slices but offers no intervention that closes them — by design a benchmark paper, but practitioners hoping for fixes will find none.
- **Scoring assumes a single ground-truth due set** per step; legitimate alternative action orderings or partial-credit semantics are not explored.
- **Channel-monitoring results are near-floor** (best 16.7%, many 0%), which is informative but also suggests the hidden-channel design may be so hard that it provides little signal to discriminate among current models on that slice.

## Relevance

本研究对多个方向有直接价值：

- **智能体长程任务/记忆系统设计者**：PM-Bench 提供了一个干净的"何时行动"诊断床，能区分"检索能力"与"执行时机控制"两类失败。对做 MemGPT 类记忆架构、reflective memory、experience retrieval 的工作，它指出了盲区——这些系统大多只优化"记得住"，未优化"到点做"。
- **前瞻式/主动式对话与助手**：与 ProactiveEval、 proactive dialogue、reminder-following 助手强相关。该基准量化了"主动监控的代价"（高查询≠高召回，反而拉低精度），为资源受限的主动监控策略提供实证依据。
- **多智能体协同设计**：层次化多智能体的失败案例（1661 次查询却只换回 5.0% 通道命中率、跨天命中率 10.7%）是对"加 agent 就能解决问题"假设的有力反驳，提示聚合策略才是瓶颈。
- **认知科学启发的方法研究**：将 PM 的 event-/time-based 区分、attention/monitoring/interference 理论引入 LLM 评测，为后续训练时干预（如专门的前瞻记忆微调）和推理时脚手架提供了标准化测试床。

## Questions for Further Exploration

1. **多周鲁棒性**：在多个不同 seed 生成的周上重复 64 次评测，per-model 最佳 scaffold 排序（Table 3）是否稳定？Set-F1 的方差有多大？
2. **人类基线**：招募人类被试在 PM-Bench 文本界面上完成同一周，建立"人类天花板"——当前 65.1% 距人类多远？哪些任务类别人类也难？
3. **干预能否闭合缺口**：针对 cross-day（≤50%）和非时钟通道监控（≤16.7%）这两个近 floor 的切片，什么样的训练目标（如前瞻记忆微调、RL on due-set prediction）或推理时机制（如基于任务状态的查询预算调度）能实质性提升而不引入 FP？
4. **聚合策略优化**：层次化多智能体的瓶颈被定位为"行动聚合而非证据获取"——能否设计出比 union-query / majority / unanimous 更优的协调器（如基于证据置信度的门控）？
5. **跨场景泛化**：将 PM-Bench 范式迁移到多用户、多模态、或专业工作流（医疗用药提醒、运维任务调度），前瞻记忆失败模式是否质变？
6. **监控代价-收益建模**：能否为每个隐藏通道学习一个"查询价值"估计器，让智能体在 11 个通道上做预算约束下的最优查询策略，而非盲目心跳？
7. **与上下文窗口/架构的关系**：递归式/超长上下文模型（Recursive LM、条件记忆 sparsity）在 PM-Bench 上是否真的有优势？还是正如作者所暗示，前瞻记忆是正交于上下文长度的独立轴？
