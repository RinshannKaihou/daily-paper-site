Why This Matters
Brings test-time adaptation to causal graph discovery, offering a practical path from synthetic training data to real-world scientific inference.
Critical Edge
Every test instance needs 2,000 stochastic refinement steps plus training from scratch — orders of magnitude slower than a single forward pass.

Methodology

TTT-SCL treats causal discovery as a test-time training problem: instead of relying on a static pre-trained model, it builds a custom training set aligned to each test dataset's causal structure and mechanisms.

1
Seed initialization. Obtain an initial DAG via NOTEARS, PC, or random sampling.
2
Stochastic graph refinement. Iteratively propose edge edits (add / delete / reverse) and accept them via a score combining alignment-of-distribution and sparsity penalty. Run 2,000 steps; collect last K = 200 graphs.
3
Aligned data generation. For each candidate graph, fit mechanisms via Structure-Induced Mechanism (SIM), forward-sample synthetic data, and train an SCL model (AVICI) on the ensemble.

Strengths
  • Thorough, honest diagnostic of three SCL failure modes across graph, mechanism, and noise dimensions.
  • Elegant framework bridging classical score-based methods and modern supervised learning.
  • Strong empirical gains, especially on real-world data where static SCL collapses.
  • Validated on two architectures (AVICI transformer + SiCL pairwise attention).
Limitations
  • Substantial computational overhead; unsuitable for fast-inference applications.
  • Untested beyond 20 nodes; exponential DAG search likely fails at scale.
  • Only one real-world benchmark with ground truth (Sachs, 11 vars).
  • Multiple hyperparameters (steps N, ensemble K, score design) with unexplored sensitivity.

Questions for Further Exploration
