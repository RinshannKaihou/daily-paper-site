## 一句话总结

AgentDoG 1.5 提出了一个轻量级、可扩展的 AI Agent 安全对齐框架，仅用约 1k 训练样本训练出 0.8B-8B 参数的安全护栏模型，在轨迹级安全判别和细粒度风险诊断上达到与 GPT-5.4 相当的性能，并可作为在线护栏部署到 OpenClaw 等开放式 Agent 系统中。

## TL;DR

AgentDoG 1.5 is a lightweight and scalable AI agent safety alignment framework from Shanghai AI Lab. It updates a three-dimensional safety taxonomy (risk source, failure mode, real-world harm) for Codex and OpenClaw execution scenarios, trains compact guardrail models (0.8B-8B parameters) on only ~1k purified samples via influence-function data selection, and achieves performance comparable to GPT-5.4 on trajectory-level safety judgment and fine-grained risk diagnosis. The framework also supports agentic safety SFT/RL training pipelines and serves as a training-free online guardrail for real-time safety moderation of deployed agents. All models and datasets are open-sourced.

## 核心贡献

1. **更新的安全分类法与 ATBench 基准家族**: 在原有三维安全分类法（风险源、故障模式、现实危害）基础上，新增 Codex 和 OpenClaw 执行场景的特定风险类别，构建了包含 ATBench、ATBench-Claw 和 ATBench-Codex 的基准家族。
2. **极低成本的轻量级护栏模型**: 通过分类法引导的数据引擎和影响函数数据提纯，仅用约 1k 样本训练出 AgentDoG 1.5 系列模型（0.8B、2B、4B、8B），在安全判别任务上与 GPT-5.4 等闭源前沿模型性能相当。
3. **可扩展的 Agent 安全训练管线**: 通过有限状态模拟构建轻量级 Agent 安全 SFT 和 RL 训练环境，将 Docker 级环境的部署开销降低两个数量级，标准 8 核机器可支持超过 10,000 个并发 Agent 环境。
4. **免训练在线安全护栏**: 将 AgentDoG 1.5 作为实时护栏部署到 OpenClaw Agent 系统中，在最终响应发出前审计执行轨迹，实现低延迟在线安全监控。

## Key Contributions

1. **Updated Safety Taxonomy & ATBench Family**: Extends the original three-dimensional taxonomy (risk source / failure mode / real-world harm) with new risk categories for Codex (repository injection, dependency compromise, unsafe shell execution) and OpenClaw (cross-environment execution risks). Creates ATBench-Claw (1,000 trajectories) and ATBench-Codex (500 trajectories) alongside the original ATBench.
2. **Ultra-Low-Cost Guardrail Models**: A taxonomy-guided data engine generates trajectories with controllable risk injection, followed by influence-function-based purification that reduces the raw SFT pool to ~1k highly informative samples. The resulting AgentDoG 1.5 variants (0.8B-8B) match or exceed much larger models on safety benchmarks.
3. **Scalable Safety Training Pipeline**: Finite-state simulation replaces Docker-level environments for SFT and RL training, reducing memory overhead and startup latency by 100x. An 8-core machine supports 10,000+ concurrent agentic environments.
4. **Training-Free Online Guardrail**: AgentDoG 1.5 is integrated as a runtime safety moderator that audits OpenClaw agent trajectories before final response delivery, blocking unsafe outputs in real time.

## Methodology

### Safety Taxonomy
The framework builds on a three-dimensional taxonomy:
- **Risk Source** (15 categories): prompt injection, tool description injection, corrupted tool feedback, over-privileged action, etc.
- **Failure Mode** (21 categories): unconfirmed/over-privileged action, improper tool use, misleading information, etc.
- **Real-world Harm** (11 categories): security/system integrity harm, privacy violation, financial harm, etc.

The taxonomy is customizable per execution environment -- ATBench-Claw focuses on tool-attack chaining for cross-application agents, while ATBench-Codex adds categories for repository injection, dependency/MCP compromise, and unsafe shell execution.

### Data Engine Pipeline
1. **Planning**: Samples a risk configuration tuple from the taxonomy, determines safe/unsafe outcome, selects tools from a library of 5,973 unique tools and MCP servers.
2. **Trajectory Synthesis**: Instantiates complete multi-turn interaction traces with controllable risk injection.
3. **Automatic Validation**: Dual-layer quality control (rule checker for structural validity, model checker for semantic quality).
4. **CoT Augmentation**: GPT-5.4 generates step-by-step reasoning chains for each training sample.
5. **Influence-Function Purification**: Uses preference-aware influence functions to score each raw SFT example by alignment with a guardrail direction in parameter space. Retains only the highest-scoring ~1k examples.

### Training
- **Stage 1 (SFT)**: Standard supervised fine-tuning on purified CoT-augmented data. Base models: Qwen3.5-0.8B, Qwen3.5-2B, Qwen3.5-4B, Llama-3.1-8B-Instruct. Learning rate: 1e-5.
- **Stage 2 (RL)**: GDPO (Group Reward-Decoupled Normalization Policy Optimization) with multi-dimensional reward signals (failure mode, real-world harm, risk source). Per-dimension advantage normalization with weights (0.3, 0.4, 0.3), 8 rollouts per query, learning rate 1e-6.

### Applications
- **Application 1**: AgentDoG 1.5 as reward model for agentic safety SFT data filtering and RL reward signal construction.
- **Application 2**: AgentDoG 1.5 as training-free online guardrail for OpenClaw agents, auditing trajectories at runtime.

## Results & Highlights

### Trajectory-Level Safety Evaluation (Binary safe/unsafe)
| Model | R-Judge Acc | R-Judge F1 | ATBench Acc | ATBench F1 |
|-------|-------------|------------|-------------|------------|
| GPT-5.4 | 93.3 | 93.7 | 73.7 | 76.7 |
| Qwen3.5-397B | 85.6 | 87.4 | 54.1 | 33.7 |
| Qwen3-Guard | 59.2 | 71.1 | 60.3 | 63.2 |
| **AgentDoG 1.5-4B** | **92.2** | **92.7** | **72.4** | **74.3** |
| AgentDoG 1.5-0.8B | 75.7 | 74.6 | 60.3 | 63.2 |
| AgentDoG 1.5-4B-U | 90.4 | 90.6 | 78.4 | 77.7 |

### Fine-Grained Risk Diagnosis (ATBench)
AgentDoG 1.5-4B achieves the best overall performance across all models:
- Risk Source: **75.2%** (vs. GPT-5.4 at 33.6%)
- Failure Mode: **27.5%** (vs. GPT-5.4 at 13.5%)
- Real-world Harm: **62.9%** (vs. GPT-5.4 at 30.2%)
- Average: **55.2%** (vs. GPT-5.4 at 25.8%)

### Cross-Environment Generalization
- AgentDoG 1.5-4B-U achieves 84.4% on ATBench-Codex and 87.6% on ATBench-Claw.
- Even the 0.8B variant outperforms many larger models on cross-environment benchmarks.

### Key Finding
The unified variant (AgentDoG 1.5-4B-U), which jointly learns coarse-grained binary judgment and fine-grained diagnosis, shows a "bonus effect" -- fine-grained diagnostic supervision improves coarse-grained safety judgment.

## Critical Assessment

### Strengths
- **Exceptional data efficiency**: Training competitive safety models on ~1k samples via influence-function purification is a genuine and practical contribution. This is far more efficient than typical SFT approaches.
- **Comprehensive framework**: Covers the full pipeline from taxonomy design through data generation, model training, and deployment as both reward model and online guardrail.
- **Strong benchmark performance**: The 4B model consistently outperforms models orders of magnitude larger (Qwen3.5-397B) and matches or exceeds GPT-5.4 on safety-specific metrics.
- **Practical deployment focus**: The two applications (SFT/RL reward model and online guardrail) address real deployment needs. The 100x reduction in environment overhead is a practical engineering contribution.
- **Open-source release**: All models and datasets are publicly available, enabling community adoption and verification.
- **Well-designed taxonomy**: The three-dimensional decomposition (risk source / failure mode / real-world harm) with environment-specific customization is a thoughtful approach to scaling safety evaluation across diverse agent settings.

### Limitations
- **Text-only trajectories**: The framework operates only on text-based agent traces. Real-world agents increasingly interact with multimodal environments (GUIs, documents, audio, video), which are not covered. The authors acknowledge this limitation.
- **Failure Mode accuracy remains low**: Even the best model achieves only 27.5% on failure mode classification, suggesting that fine-grained diagnosis is still far from solved. This is acknowledged but the low absolute performance somewhat undermines claims of "strong fine-grained risk diagnosis."
- **Cannot prevent already-executed harms**: The guardrail operates after trajectory execution, so external side effects from unsafe actions cannot be undone -- only the final response can be blocked.
- **Self-generated evaluation data**: The ATBench family benchmarks are constructed by the same team using their own data engine, raising questions about potential circularity in evaluation.
- **Comparison fairness**: The paper compares safety-specialized small models against general-purpose large models. While this demonstrates the value of specialization, it somewhat overstates the case when the specialized models are trained on task-specific data derived from the same taxonomy used to construct the benchmarks.
- **Limited novelty in individual components**: The influence-function purification, SFT+RL training pipeline, and taxonomy design are all incremental over AgentDoG 1.0. The primary advance is in scale, efficiency, and the extension to new agent environments.
- **Large author list**: With 50+ authors, the division of contributions is unclear, and the paper reads more like a systems/engineering report than a focused research contribution.

## Questions for Further Exploration

1. 影响函数提纯方法在多大程度上依赖于参考模型 (reference model) 的选择？使用不同的基座模型是否会得到不同的"最佳 1k 样本"集合？
2. Failure Mode 分类准确率仅 27.5%，瓶颈在于类别过多（21类）、类别间边界模糊、还是训练数据不足？减少类别粒度是否会有显著提升？
3. 在线护栏的延迟开销是多少？论文提到"低延迟"但未报告具体数值，这对实际部署至关重要。
4. ATBench 基准与训练数据使用相同的分类法和数据引擎生成，如何评估模型在分布外 (OOD) 风险场景上的泛化能力？
5. AgentDoG 1.5 作为 RL 奖励模型时，如何避免 reward hacking？是否存在 policy agent 学会生成表面上安全但实质上有害的行为模式？
