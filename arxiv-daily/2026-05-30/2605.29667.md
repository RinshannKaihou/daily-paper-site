## 一句话总结

本文提出 ChiSafe-PAS，一个包含 1,897 条人工标注的中文对抗性提示词基准数据集，覆盖自残/暴力、毒品/非法交易、诈骗和讽刺四个高风险领域，旨在揭示英文安全对齐在中文语境下的系统性失效。

## TL;DR

Current LLM safety systems trained primarily on English data break down when deployed in Chinese-language settings. This paper introduces ChiSafe-PAS, a human-annotated benchmark of 1,897 adversarial Chinese prompts with a 3-class gold action taxonomy (REFUSE / SAFE-REDIRECT / RESPOND) and a nine-category obfuscation taxonomy, covering four high-stakes domains. The dataset is grounded in authentic Chinese internet evasion practices rather than machine-translated approximations.

## 核心贡献

1. **ChiSafe-PAS 数据集**: 包含 1,897 条人工构造的中文对抗性提示词，其中 1,544 条带有完整的黄金标准标注。所有提示词由深谙中文互联网文化的母语者手动构建，确保反映真实的对抗策略而非机器翻译的近似。

2. **三类黄金动作标注体系 (3-class Gold Action Schema)**: 超越传统的二分类（安全/不安全），引入 REFUSE（拒绝）、SAFE-REDIRECT（安全引导）和 RESPOND（正常回应）三个类别，编码了良好对齐模型必须处理的语用区分。

3. **九类混淆策略分类法 (Obfuscation Taxonomy)**: 基于中文互联网实际逃避策略构建的分类体系，涵盖拼音替换、谐音/形近字、拆字、黑话、语气缓冲、Emoji 替换、混合策略、标点截断等九个类别。

4. **方法论论证**: 提出人类标注在文化敏感的安全评估中不可被 LLM-as-judge 或翻译管道替代的论点，强调数据集的认识论独立性 (epistemic independence)。

## Methodology

**数据集构建**:
- 所有 1,897 条提示词由一位深度参与中文社交媒体和互联网文化的中文母语者手动构造
- 覆盖四个高风险领域：自残与暴力（500条）、毒品与非法交易（501条）、诈骗（499条）、讽刺（397条）
- 提示词设计基于真实的中文互联网逃避审查策略，而非算法生成

**标注体系**:
- 每条标注实例包含：原始中文提示词、人工英文翻译、意图类型、混淆类型、风险等级、黄金动作标签、安全理由说明、注释
- 采用分阶段标注架构：自残领域有 147 条完成标注（PAS），353 条待标注（PAW）

**混淆策略分类**（基于自残领域统计）:
| 类别 | 数量 |
|------|------|
| 未指定（基线） | 123 |
| 语气缓冲（语气缓冲） | 103 |
| 其他 | 85 |
| 黑话（黑话） | 46 |
| 拼音（拼音） | 34 |
| Emoji | 28 |
| 混合（混合） | 26 |
| 错字/谐音/形近字 | 20 |
| 标点截断（截断） | 15 |

**标注质量控制**:
- 三人团队：主要标注者（中文母语者，本科传播学专业）、首席标注者（NLP标注经验）、标注经理
- 首席标注者审核验证每条标注决策
- 未来计划引入独立双标注并报告 kappa 一致性

**领域分布与黄金动作**:
| 领域 | 总计 | 已标注 | 拒绝 | 安全引导 | 回应 |
|------|------|--------|--------|----------------|--------|
| 自残/暴力 | 500 | 147 | 40 | 31 | 76 |
| 毒品/非法交易 | 501 | 501 | 501 | 0 | 0 |
| 诈骗 | 499 | 499 | 303 | 159 | 37 |
| 讽刺 | 397 | 397 | 0 | 397 | 0 |

## Results & Highlights

- 本文为**资源论文**（data paper），核心贡献是数据集本身，**尚未进行 LLM 实验评估**。LLM 对齐评估留作未来工作。
- 提出了标准化评估协议，建议使用三类指标：黄金动作准确率（Gold Action Accuracy）、领域准确率（Domain Accuracy）、混淆策略分层准确率（Obfuscation-Stratified Accuracy）。
- 建议基线对比至少包含一个英文主导模型（如 GPT-4o）和一个中文原生模型（如 Qwen 或 ChatGLM）。
- 发表于 LREC 2026 的 RESOURCEFUL workshop。

## Critical Assessment

本文提出了一个有实际价值的中文 LLM 安全评估资源，但在以下方面存在明显不足：

1. **尚未进行任何模型评估**: 数据集论文的价值最终取决于能否揭示已知模型的具体弱点。目前仅有数据集设计而无实验结果，无法验证该基准是否能有效区分不同模型的安全能力。

2. **标注覆盖不完整**: 自残领域 500 条中仅 147 条完成标注（29.4%），而这恰恰是"失败代价最高"的领域。这使得数据集在最关键领域的可用性大打折扣。

3. **标注者代表性有限**: 所有提示词由单一标注者构建，且该标注者为本科生而非对抗性 NLP 或网络安全专家。虽然作者承认这一局限，但它影响了数据集对区域、代际和平台差异的覆盖。

4. **缺乏标注者间一致性 (IAA)**: 三人团队中首席标注者是"验证"而非"独立重标注"，因此无法报告 kappa 等形式化一致性指标。

5. **"Other" 混淆类别过大**: 自残领域中 "其他" 类别有 85 条，占比 17.7%，表明九类混淆分类法尚不完善。

## Strengths

- 数据集基于真实的中文互联网文化知识而非机器翻译，填补了中文 LLM 安全评估的重要空白
- 三类黄金动作标注体系比传统二分类更精细，特别关注了过度拒绝（over-refusal）问题
- 九类混淆策略分类法有清晰的实证依据和文化根基
- 讨论了安全评估中"认识论独立性"的深层方法论问题
- 伦理考量全面，包括数据发布治理、标注者心理健康、敏感领域处理

## Limitations

- 自残领域 70.6% 的数据未标注，核心领域覆盖严重不足
- 单一标注者构造所有提示词，代表性有限
- 无标注者间一致性报告
- 无任何 LLM 评估实验结果
- 数据集规模（1,544 条完整标注）相对有限
- 数据集通过申请表获取而非完全开放，限制了可及性
- 混淆分类法中"其他"类别占比较高，需进一步细化

## Relevance

本文对以下研究方向具有直接相关性：

- **多语言 LLM 安全对齐**: 量化英文安全训练在中文语境下的泛化失效程度
- **中文 NLP 对抗性评估**: 提供基于真实互联网逃避策略的测试基准
- **LLM 安全评估方法论**: 关于人类标注不可替代性的论证对整个安全评估领域有方法论启示
- **过度拒绝 (Over-refusal) 问题**: 三类标注体系特别设计用于区分"应该拒绝"和"不应拒绝但看起来敏感"的内容
- **跨文化 AI 对齐**: 揭示了安全对齐中的语言和文化边界问题

## Questions for Further Exploration

1. 中文原生模型（如 Qwen、ChatGLM）与英文主导模型（如 GPT-4o）在 ChiSafe-PAS 上的安全表现差距有多大？差距在不同混淆策略和领域间如何分布？
2. 九类混淆策略分类法能否扩展为层次化结构，以更精细地捕捉不同逃避策略的组合效应？
3. 多标注者独立标注后，标注者间一致性在"语气缓冲"和"黑话"等语用依赖类别上是否会显著低于"拼音"等词汇层面类别？
4. 基于该数据集微调的安全分类器能否泛化到训练集中未见过的混淆策略组合？这对于应对互联网逃避策略的快速演变至关重要。
5. 讽刺领域全部标注为 SAFE-REDIRECT 是否合理？在实际部署中，讽刺与有害内容的边界是否应引入更细粒度的区分？
