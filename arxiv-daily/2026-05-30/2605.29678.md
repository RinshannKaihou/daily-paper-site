## 一句话总结

本文发现，与任务语义完全无关的"虚假提示词"(Spurious Prompts) 能够系统地操控大语言模型的行为，甚至可以匹配或超越专门的任务感知提示优化方法，揭示了 LLM 对提示词敏感性的全新维度。

## TL;DR

The authors introduce "spurious prompts" -- natural-language instructions deliberately stripped of task-relevant vocabulary, domain cues, and reasoning strategies -- and show they can systematically steer LLM behavior. Using a simple black-box evolutionary search procedure, they discover spurious prompts that match or outperform standard prompting baselines and the task-aware optimizer PromptWizard across seven benchmarks and four models (0.8B--27B parameters). These prompts can also induce unintended behaviors (selecting the first option, producing incorrect/prime/even/small-number answers) without explicitly instructing the model to do so, revealing a fundamentally new form of prompt sensitivity.

## 核心贡献

1. **提出"虚假提示词"概念**: 定义了一类与目标任务在表面上完全无关，但能显著影响模型性能的系统提示词。这些提示词在语义相似度上与随机无关文本几乎无区别，但其效果可以媲美甚至超越专门优化的任务感知提示词。

2. **设计黑盒搜索流程**: 提出了一个完整的演化搜索框架，包含 LLM 生成器、词汇过滤器（Prompt Validator）、目标模型评估、回放缓冲区和 LLM 变异器。整个过程完全黑盒，仅需模型输出，不需要梯度、隐状态或模型权重。

3. **全面的实验验证**: 在 GSM8K、MATH500、MedQA、GPQA、OpenBookQA、MuSR、MMLU-Pro 七个基准上，覆盖 Qwen3.5-0.8B、Llama-3.2-1B、OLMo-3-7B、Qwen3.5-27B 四个模型，证明虚假提示词的广泛有效性。

4. **行为操控能力**: 展示虚假提示词能引导模型产生特定非预期行为（总是选 A、选错误答案、输出偶数/质数/小于 10 的数），且通常比直接指令更有效。

5. **深入分析**: 包括迁移性实验（跨模型/跨基准迁移差）、语义距离验证（余弦相似度接近随机文本）、长度消融（虚假提示词显著短于 PromptWizard）、胡言乱语提示词消融（结构性约束本身可驱动广泛行为变化）等。

## Methodology

### 搜索流程

整体采用演化搜索范式:

1. **候选生成**: 使用 Qwen3.5-27B 作为生成器模型，明确指示其生成与目标任务无关的系统提示词。生成器被要求避免任务相关词汇、领域线索和常见推理策略，转而使用语气、风格、仪式、角色或格式来构建提示词。

2. **词汇过滤 (Prompt Validation)**: 使用手工制定的任务特定禁用词表（如数学任务禁止 "arithmetic, equation, proof" 等词汇），过滤掉包含任何任务相关内容的候选。

3. **数据集划分**: 训练集进一步划分为 K 个不相交子集，用于不同搜索轮次的评估；另有回放缓冲区保留部分历史数据以防遗忘。

4. **迭代变异**: 每轮评估后取 top-K 候选，由 LLM 变异器生成新的变体，保持虚假特征的同时改变角色、结构或控制策略。共进行 3 轮变异。

5. **最终选择**: 在验证集上选出最佳候选，在独立测试集上报告结果。

### 实验设置

- **基准测试**: GSM8K, MATH500, MedQA, GPQA, OpenBookQA, MuSR, MMLU-Pro
- **模型**: Qwen3.5-0.8B, Llama-3.2-1B-Instruct, OLMo-3-7B-Instruct, Qwen3.5-27B
- **基线**: Zero-Shot CoT, Plan-and-Solve, Least-to-Most, Self-Ask, Step-Back, Analogical, Re-Reading, PromptWizard (任务感知优化器)
- **硬件**: 单张 H100 GPU (94GB VRAM)
- **推理设置**: temperature=0, greedy decoding

## Results & Highlights

### 性能提升

- 虚假提示词在多个模型-基准对上匹配或超越标准任务无关基线，部分情况下甚至超越任务感知的 PromptWizard。
- 在 Qwen3.5-27B 上，虚假提示词平均准确率 (73.24%) 超越 PromptWizard (67.99%) 和 Zero-Shot CoT (59.79%)。
- 在 OLMo-3-7B 上，虚假提示词 MATH500 上达到 38.67% vs PromptWizard 的 33.67%。

### 行为操控

- **选项 A 偏置**: 虚假提示词在 Qwen3.5-27B 上诱导 99.73% 的回答选择选项 A，远超直接指令的 92.20%。
- **错误答案**: 虚假提示词在多个模型上成功引导模型选择错误答案，在 OLMo-3-7B 上达 79.33% 的成功率。
- **数值操控**: 在 GSM8K 上成功引导模型输出偶数、质数或小于 10 的数，且虚假提示词通常优于直接指令。

### 通用虚假提示词

使用跨基准联合训练集搜索的单个虚假提示词，在多个基准上保持竞争力，偶尔甚至超越基准特定的虚假提示词，表明其效果不完全依赖于隐藏的任务特定指令。

### 胡言乱语消融

由 65-90% 无意义 token 组成的提示词在提升准确率和引导错误答案方面匹配甚至超越语义连贯的虚假提示词，表明结构性约束本身足以驱动广泛行为变化。但胡言乱语提示词在诱导特定的位置偏置（如总是选 A）方面失败，暗示精确行为操控需要连贯的自然语言叙述。

### 其他发现

- 虚假提示词平均长度约为 PromptWizard 的 1/3，效果并非源于冗长性。
- 跨模型和跨基准迁移性较差，表明其效果依赖于特定的模型-基准对组合。
- 更大的生成器模型倾向于发现更强的虚假提示词。
- 余弦相似度分析确认虚假提示词与随机无关文本在语义上几乎不可区分。

## Critical Assessment

本文揭示了一个重要且令人不安的现象：LLM 的行为可以被与任务完全无关的表面文本特征所操控。这一发现对 prompt engineering 的基本假设提出了根本性挑战 -- 即"要改善模型在任务上的表现，必须更好地描述该任务"。

从积极面看，这说明当前 LLM 的行为控制机制存在显著漏洞，虚假提示词可能成为一种隐蔽的攻击向量。从消极面看，这也意味着我们可能对 prompt 为何有效的理解还很不充分。

核心疑问在于机制层面：虚假提示词究竟是通过什么内部路径影响模型输出的？作者坦承这是未来工作。可能的解释包括：(1) 提示词的叙述结构（而非语义内容）触发了模型训练数据中与高质量输出相关的分布模式；(2) 格式约束和角色扮演引导模型进入特定的生成模式；(3) 某些结构模式充当了模型内部的"控制键"。但这些假说均未被验证。

## Strengths

1. **问题新颖性强**: 首次系统性地研究任务无关提示词对模型行为的操控能力，开辟了 prompt sensitivity 研究的新方向。
2. **实验设计全面**: 覆盖多个基准、多个模型家族、多个规模；包含性能提升和行为操控两个维度；消融实验丰富。
3. **方法论透明**: 搜索流程完全黑盒且可复现；提示词示例公开；诚实讨论局限性。
4. **分析深入**: 语义距离验证、长度消融、胡言乱语消融、迁移性实验、组件消融等多角度分析。
5. **实际意义**: 揭示了一种潜在的隐蔽攻击向量，对 LLM 安全和鲁棒性评估有重要启示。

## Limitations

1. **缺乏机制解释**: 论文展示了"是什么"和"能做到什么程度"，但对"为什么"缺乏深入理解。虚假提示词通过何种内部表示路径影响输出，完全是黑箱。
2. **模型规模有限**: 最大模型仅 27B 参数，未验证在更大模型（如 70B+, 200B+）上的有效性。作者自身也承认这一局限。
3. **需要标注数据**: 搜索流程依赖标注数据的准确率作为评分信号，限制了在无监督场景的直接适用性。
4. **迁移性差**: 虚假提示词高度依赖特定的模型-基准对，这既是一个实证发现，也限制了其通用性结论的说服力。
5. **搜索成本**: 虽然声称"简单"，但演化搜索需要大量模型推理调用（24 个初始候选 x 3 轮变异 + 评估），实际计算成本不低。
6. **语义无关性论证不完整**: 余弦相似度分析基于嵌入模型，可能无法捕捉更微妙的语义关联。某些虚假提示词的"叙述结构"本身可能暗含了任务相关的推理模式（如"选择唯一匹配项"的隐喻）。

## Relevance

本研究对 LLM 鲁棒性和安全性研究有直接价值。虚假提示词现象提示我们：(1) 当前的 prompt 评估方法可能严重低估了模型对表面文本特征的敏感性；(2) 传统的 prompt injection 检测方法（基于语义内容审查）可能无法识别这类攻击；(3) prompt engineering 的理论基础需要重新审视 -- prompt 的有效性可能不完全来源于其语义内容。对于从事 prompt 优化、LLM 安全评估和对抗鲁棒性研究的研究者，本文提供了重要的新视角和实验基准。

## Questions for Further Exploration

1. 虚假提示词在更大规模模型（70B+, 200B+, 600B+）上是否仍然有效？效果是增强还是减弱？
2. 虚假提示词通过何种内部表示路径影响模型输出？是否可以通过探针实验或因果干预来揭示其作用机制？
3. 虚假提示词能否被用于隐蔽的 prompt injection 攻击？现有的 prompt injection 检测方法能否识别此类攻击？
4. 模型对齐训练（RLHF/DPO）能否缓解虚假提示词的操控效果？对齐后的模型是否更容易或更不容易受影响？
5. 不同语言和文化背景下的虚假提示词是否存在系统性差异？跨语言迁移性如何？
