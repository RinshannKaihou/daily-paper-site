Split Convolutional Blocks reduce 1D-CNN fan-in enough to precompute entire layers as FPGA lookup tables, achieving 95.6% F1 atrial fibrillation detection on an ultra-small FPGA.

Precomputed 1D-CNNs for Atrial Fibrillation Detection on Tiny Smart Sensor Systems · Einhaus, Maman, Hoever, Erbsloh & Schiele · 2026-05-28 · IEEE SMARTCOMP 2026
tinyML edge-ML FPGA LUT-precomputation ECG-analysis
Key Metric
95.6% F1-score
On AMD Spartan-7 S15 with only 2,844 LUTs, zero DSP blocks, zero BRAM.
Why This Matters
Enables real-time AF detection on wearable devices by mapping entire 1D-CNNs to FPGA lookup tables, using zero DSP or memory blocks.
Critical Edge
Single-dataset evaluation with unvalidated generalization; binary activations cap expressivity and the ceiling on larger FPGAs is never discussed.

Methodology

  • Binarized activations (+1/−1) with straight-through estimators for training; weights exist only in training, implicitly encoded in LUT truth tables at inference.
  • Split Convolutional Block replaces a dense convolution with two grouped convolutions (kernel sizes k and 1), using batch norm and binary activation between them. Group count g controls per-layer fan-in and thus LUT cost.
  • Pool reordering proven equivalent under binary activations: train as conv→pool→bnorm→binarize, infer as conv→bnorm→binarize→pool to minimize fan-in.
  • CLC Score (Cross Layer Connectivity) combines fan-in, LUT cost, and inter-layer connectivity to rank architectures before training; only ~26% of top-scored candidates need training to cover the full Pareto front.
  • Automated toolchain from PyTorch to synthesizable VHDL via elastic-AI creator.

Key findings: CLC score correctly ranks 99.6% of configuration pairs; pooling-after-binarization loses ~5% accuracy versus bnorm-after-pooling; (k, 1) kernel ordering outperforms (1, k) by ~2%.

Assessment

Strengths

  • Complete automated toolchain from PyTorch training to VHDL generation.
  • CLC score shows 99.6% ranking consistency, reducing search space dramatically.
  • 95%+ F1 on only 2,844 LUTs with zero DSP/BRAM — state-of-the-art resource efficiency.
  • Rigorous mathematical proof for pool reordering equivalence.
  • Open-source toolchain; reproducible experimental design.

Limitations

  • Single dataset (MIT-BIH AF); no generalization to other time-series tasks.
  • No direct comparison with higher-F1 methods (ViT, Inception CNN) on target hardware.
  • Binary activation likely caps network expressivity; quantization loss not quantified.
  • LUT cost model is worst-case; actual synthesis often 50% lower.
  • Power figures are Vivado estimates only; weak negative correlation with LUT cost unexplained.
  • Still requires training ~26% of candidates; group selection is not fully automated.

Questions for Further Exploration

  1. Can the method extend to 2-bit or 4-bit quantization for better accuracy–resource trade-offs on slightly larger FPGAs?
  2. Can CLC scoring be combined with differentiable NAS for end-to-end hardware-aware architecture search?
  3. How must the grouped structure adapt for multi-lead ECG or other biosignals like EEG?
  4. Would combining NeuraLUT or differentiable logic-gate networks further compress LUT overhead?
  5. What are throughput and latency characteristics for continuous real-time streams (e.g., dynamic heart-rate monitoring)?
