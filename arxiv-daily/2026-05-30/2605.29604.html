Tensor Cores accelerate irregular graph MIS by reformulating neighbor-counting as tiled SpMV.
TC-MIS: Maximal Independent Set on Tensor-Cores arxiv.org/abs/2605.29604 Prajjwal Nijhara, Dip Sankar Banerjee 2026-05-28
GPU Computing Graph Algorithms Tensor Cores HPC
Key Metric
44.38×
Maximum speedup over ECL-MIS on H200 (Hopper), sparse graph G3. Average 18.80× across evaluated graphs.
Why This Matters
Maps irregular graph subproblems to Tensor Cores, opening a new research direction for GPU graph algorithms as TC hardware evolves.
Critical Edge
Tiled adjacency matrix memory overhead is unquantified, creating a practical bottleneck for billion-scale graphs.

Methodology

The core insight: MIS Phase 2 (neighbor comparison) consumes ~56% of runtime in ECL-MIS due to irregular adjacency-list traversals. TC-MIS reformulates it as SpMV Nc = A × C, where 16×16 non-zero tiles of A are processed via WMMA mma_sync.

Phase 1 — CUDA Cores Compute vertex priorities, find max neighbor priority, generate candidate vector C.
Phase 2 — Tensor Cores Execute tiled SpMV via WMMA. All-zero candidate tiles are skipped (early exit).
Phase 3 — CUDA Cores Lock-free parallel state update. Each vertex updates its own state without atomics.

Three heuristics tested: H1 random hash (10.43% deviation), H2 degree-aware (2.42%), and H3 conflict-aware (0.17% deviation) — the chosen trade-off.

Strengths & Limitations

Strengths

  • First principled demonstration that Tensor Cores can accelerate irregular graph algorithms.
  • Comprehensive evaluation across four GPU architectures spanning four NVIDIA microarchitecture generations.
  • Clean three-phase decomposition with heterogeneous CUDA Core / Tensor Core utilization.
  • H3 conflict-aware heuristic thoughtfully preserves quality under warp-synchronous execution.

Limitations

  • Tiled adjacency matrix increases memory footprint, limiting applicability to GPU-fittable graphs.
  • Performance gains heavily dependent on graph structure; power-law networks show modest improvements.
  • No evaluation on dynamic or streaming graphs; no open-source code referenced.
  • 16×16 WMMA tile size is a hardware constraint that may misalign with some sparsity patterns.

Questions for Further Exploration

  1. Can the tiled SpMV approach generalize to other Luby-style graph algorithms (max matching, vertex cover)? Which phases of which algorithms are SpMV-amenable?
  2. Can adaptive tiling (mixed 16×16 and variable-size blocks) mitigate dense tile-row issues in power-law graphs?
  3. Does the tile-centric compute model naturally favor graph partitioning and communication optimization in multi-GPU / distributed settings?
  4. How will performance evolve on future architectures (e.g., Rubin) with potentially flexible TC tile sizes?
  5. Can a unified memory representation serve both CUDA Core and Tensor Core paths to support billion-scale graphs?
