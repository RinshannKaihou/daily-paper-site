Methodology

FedSmoothLoRA addresses three failure modes of naive FedAvg+LoRA: low-rank updates are too constrained; server LoRA overwrites client state each round; and every client starts from the same global initialization.

1 Merge server LoRA into client backbone.
2 Round-Matching: compute delta between prior local state and current server update to suppress inter-round oscillation.
3 Gradient-Aligned: estimate full-model gradient on a small calibration batch, SVD-reconstruct into rank-r LoRA for client-specific starting guidance.
4 Combine + project via SVDApprox; train locally; subtract the gradient-aligned term to extract the effective update.
5 Server averages in full-rank space, then projects back to rank-r — a principled alternative to naive factor averaging.

ζ=1 (constant) for IID; cosine decay for Non-IID to avoid over-constraining heterogeneous clients.

  • Scales to LLaMA-2-13B with consistent gains; memory overhead remains modest.
Strengths
  • Clean one-to-one mapping between three identified problems and their solutions
  • Strong empirical gains at low computational cost
  • Formal mechanistic analysis (Proposition D.1) grounds the Round-Matching intuition
  • Comprehensive ablations across vision, language, IID, and Non-IID
  • Code publicly available
Limitations
  • Narrow benchmark scope (CIFAR-100; GSM8K/HumanEval/Aya)
  • Only LLaMA-family LLMs evaluated
  • No formal convergence rate guarantee
  • Only 3–12 clients — far below production federated scale
  • Gradient-estimation cost on very large models (>13B) not thoroughly analyzed
  • ζ mode selection requires prior knowledge of data distribution
