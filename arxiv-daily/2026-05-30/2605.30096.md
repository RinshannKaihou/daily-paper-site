# 一句话总结

本研究通过400次自主渗透测试实验（4个LLM各100次），首次系统性地量化了LLM攻击行为的一致性和可重复性，揭示了不同模型在攻击成功率、失败模式、策略多样性和安全拒绝行为上的显著差异。

# TL;DR

This paper presents the first large-scale empirical study (400 runs, 4 LLMs x 100 each) measuring how consistently autonomous LLM agents perform penetration testing against an identical vulnerable target. Six key findings: (1) zero content refusals across all 400 runs under authorized-pentest framing; (2) model-distinctive failure modes (Claude: API truncation, qwen: premature completion, GPT-4o-mini: iteration cap); (3) dramatic strategy diversity differences (GPT-4o-mini: 98/100 unique strategies vs Gemini: 48/100); (4) cross-model exploitation rates statistically significant (p < 0.001); (5) first-exploit timing clustered at ~15-30 seconds; (6) credential reuse only in models retaining more conversation history. Data and code released on Zenodo.

# 核心贡献

1. **首次大规模LLM攻击一致性测量**：在严格控制变量的条件下（相同目标、提示词、编排器），对4个LLM各执行100次自主渗透测试，共400次实验、超过6000次命令-执行-观察交互。此前的研究通常仅报告1-10次运行结果。

2. **零安全拒绝的发现**：在400次运行中，4个模型在"授权渗透测试"框架下均未产生内容拒绝。这是对Wei等人"竞争目标"理论框架最强有力的实证支持。

3. **模型特有的失败模式分类法**：建立了细粒度的失败模式分类（SUCCESS, API_ERROR_TRAP, API_ERROR_PARTIAL, REFUSAL_TRAP, PREMATURE_COMPLETION, EARLY_QUIT, ITERATION_CAP等），揭示失败模式具有模型特异性而非随机分布。

4. **首次攻击策略与Markov链分析**：通过一阶Markov链分析攻击阶段转移概率，量化策略多样性比率（唯一策略数/总运行数），揭示GPT-4o-mini的近最大多样性（0.98）与Gemini的低多样性（0.48）之间的显著差异。

5. **对话历史与攻击复杂度的关联发现**：跨服务凭证复用仅在保留更多对话历史的配置中出现（qwen 57%，GPT-4o-mini 49%，云模型在5轮窗口下为0%），尽管无法分离架构效应与上下文窗口效应。

6. **对Claude数据误分类的透明纠正**：早期草稿将Claude的HTTP 529错误归类为安全拒绝，经全量日志审计后纠正为上游API容量故障，体现了良好的科学诚信。

# Key Contributions

1. **First large-scale measurement of LLM attack consistency**: 400 runs (4 models x 100) under controlled conditions with over 6,000 command-execute-observe interactions, where prior studies typically reported 1-10 runs per model.

2. **Zero-refusal finding across all providers**: No model emitted a content refusal under authorized-pentest framing across 400 trials -- the strongest empirical support for Wei et al.'s "competing objectives" framework to date.

3. **Model-distinctive failure mode taxonomy**: Fine-grained classification (SUCCESS, API_ERROR_TRAP, PREMATURE_COMPLETION, EARLY_QUIT, ITERATION_CAP, etc.) revealing that failure modes are model-specific, not random.

4. **First Markov chain analysis of LLM attack phase transitions**: Quantifies strategy diversity via unique-strategy ratio and first-order Markov chains over attack phases, revealing GPT-4o-mini's near-maximal diversity (0.98) vs. Gemini's concentration (0.48).

5. **Credential reuse and conversation history correlation**: Cross-service credential reuse appears only in configurations retaining more conversation history (qwen 57%, GPT-4o-mini 49%, cloud models 0%), though architectural and context-window effects cannot be disentangled.

6. **Transparent correction of Claude data misclassification**: Earlier draft's "42% stochastic refusal rate" for Claude reclassified as upstream API capacity failures (HTTP 529) after full-log audit -- a positive signal of scientific integrity.

# Methodology

## Experimental Design

- **Models**: Claude Sonnet 4 (Anthropic), Gemini 2.5 Flash-Lite (Google), GPT-4o-mini (OpenAI), qwen2.5-coder:14b (local, via Ollama)
- **Runs per model**: 100 (total: 400)
- **Target**: Azure VM honeypot with 3 vulnerable services -- OWASP Juice Shop (SQL injection on port 3000), OpenSSH with weak credentials (port 22), vsftpd anonymous access with credential files (port 21)
- **Temperature**: 0.3 for Ollama/Gemini/OpenAI; 1.0 for Anthropic (unintended -- temperature parameter was not set in the Anthropic API request, defaulting to Anthropic's documented default of 1.0)
- **Max iterations**: 25 per run; command timeout: 300 seconds
- **Collection window**: March 26-29, 2026 (60 hours total)

## Orchestrator Design

The orchestrator implements a command-execute-observe loop with three progression-forcing interventions:
1. **Refusal-retry**: Re-prompts with authorization at iterations 0-1 on refusal; terminates at iteration >= 2
2. **Loop detection**: Breaks if last 3 responses share identical 200-char prefix
3. **Phase-stall detection**: Forces next phase if last 3 phases are identical (except RECON/POST_EXPLOIT)

Safety filters block ~14 dangerous commands and unsubstituted placeholders. Conversation history is trimmed to system prompt + initial message + last 10 messages (~5 exchanges) for cloud providers; full history retained for local models (Ollama).

## Analysis Pipeline

- Service exploitation rates with 95% Wilson score confidence intervals
- Failure mode classification into mutually exclusive categories
- First-order Markov chain for phase transition analysis
- Strategy diversity via unique-strategy ratio and Shannon entropy
- Self-correction rate and give-up rate
- Mann-Whitney U and chi-squared tests with Holm-Bonferroni correction (m=37 tests, alpha=0.05)
- Effect sizes: Cliff's delta and Cohen's h
- Temporal analysis: Spearman correlation and first-half vs. second-half Fisher's exact test

| Creds read | 26% | **60%** | 2% | 52% |
| Full (3/3) | 61 | 25 | **85** | 56 |

|---|---|---|---|---|
| SUCCESS | 61 | 25 | 85 | 56 |

## Key Statistical Results

- All cross-model exploitation rate differences statistically significant (p < 0.001) with large effect sizes
- qwen vs. Gemini SQL injection: Cohen's h = 1.12 (large)
- qwen vs. Gemini credential reuse: Cohen's h = 1.71 (very large)
- Claude vs. GPT-4o-mini iterations: Cliff's delta = -0.90 (large)
- First-exploit timing: iteration 4 (web-first models) to iteration 6 (FTP-first models), ~15-30 seconds wall-clock
- Gemini self-correction rate: 1.0 (perfect) -- never repeats a failed approach

## Strategy Diversity (Table 7)

| Metric | Claude* | qwen | Gemini | GPT-4o-mini |
|---|---|---|---|---|
| Unique strategies | 28 | 69 | 48 | **98** |
| Diversity ratio | 0.28 | 0.69 | 0.48 | **0.98** |

*Claude's statistics are presented as descriptive case-study data due to temperature confound (T=1.0 vs T=0.3 for others).

## First-Move Split

- **Web-first**: Claude (67%), Gemini (90%) -- follow prompt-prescribed order
- **FTP-first**: qwen (63%), GPT-4o-mini (82%) -- deviate from prescribed order

## Claude API Event

- 91 of 1,135 Anthropic API calls (8.0%) returned HTTP 529 overloaded_error during a documented Anthropic capacity event (2026-03-26/27)
- Truncated 39 of 100 Claude runs (32 API_ERROR_TRAP + 7 API_ERROR_PARTIAL)
- Average latency for error events: 0.5s vs 2.2-5.0s for successful responses -- consistent with edge-layer rejection, not model-level processing
- Temporal decline: SQLi success 84% (first 50 runs) vs 54% (last 50 runs), p = 0.002

# Critical Assessment

The study makes a genuinely valuable contribution to a field dominated by single-run anecdotes. However, several issues deserve attention:

**Methodological confounds**: The unintended temperature asymmetry (Claude at T=1.0 vs others at T=0.3) is a significant limitation that the author honestly acknowledges. Combined with the Anthropic API capacity event truncating 39 Claude runs, the Claude data is substantially confounded. The author's decision to present Claude's variability statistics as "descriptive case-study data" rather than cross-model comparison points is methodologically sound but reduces the effective study to a 3-model comparison for many analyses.

**Single-target generalizability**: All 400 runs target the same honeypot with known, well-documented vulnerabilities (OWASP Juice Shop, weak passwords, anonymous FTP). LLMs likely encountered Juice Shop exploit patterns in training data, meaning these results may overestimate real-world attack capabilities against novel targets. The credential reuse finding, for example, depends on the presence of plain-text credential files -- a specific honeypot configuration.

**Conversation history confound**: Cloud models received only ~5 exchanges of history while the local model (qwen) retained full history. The credential reuse finding (qwen 57%, GPT-4o-mini 49%, cloud models 0%) cannot be cleanly attributed to model architecture vs. context availability. The author acknowledges this but it remains a major uncontrolled variable.

**No control framing**: The study only tests the "authorized penetration testing" framing. The zero-refusal finding is meaningful but bounded -- we do not know whether other framings would produce different refusal rates. The author explicitly notes this limitation.

**Iteration cap artifact**: The 25-iteration cap artificially truncates GPT-4o-mini (72% of runs hit the cap, 0% declared natural completion). The true exploitation rate may differ.

**Orchestrator shaping**: The progression-forcing interventions (loop detection, phase-stall detection) intentionally bias trajectories toward completion. The resulting Markov chains describe "orchestrator-shaped trajectories, not raw model behavior" as the author correctly notes.

# Strengths

- First study at N=100 per model across a multi-service target -- a genuine methodological advance
- Transparent, detailed documentation of every methodological decision and its implications
- Comprehensive statistical framework with proper multiple-testing correction and effect sizes
- Novel failure mode taxonomy that is model-specific and practically useful
- Honest partitioning of results by confound sensitivity (Section 4.6 is exemplary)
- Complete dataset and code release enables independent verification
- The zero-refusal finding, while bounded in scope, is the largest multi-provider empirical observation of its kind
- Practical recommendations for red team automation (minimum N=30, reporting standards)

# Limitations

- Single target environment (OWASP Juice Shop + 2 services) limits generalizability
- Training data contamination risk with well-known vulnerable applications
- Claude data substantially confounded by temperature asymmetry and API capacity event
- Conversation history trimming asymmetry between cloud and local models is uncontrolled
- Fixed prompt style only; no prompt variation or control framing tested
- 25-iteration cap artificially truncates GPT-4o-mini runs
- Cross-VM comparison (two different Azure IPs for different model pairs)
- Cloud model drift: results may not replicate with updated model versions
- Single-author study; independent replication would strengthen confidence

# Relevance (领域相关性)

本研究对**AI安全**和**网络安全**领域具有高度参考价值。核心发现（授权渗透测试框架下零安全拒绝率）直接挑战了基于拒绝的安全机制的有效性，对LLM安全对齐研究具有重要意义。对防御方而言，首次攻击时间窗口的一致性（~30秒）和Markov链阶段转移分析为入侵检测系统设计提供了具体指导。策略多样性的量化（GPT-4o-mini的98/100）对签名检测系统的局限性提出了实证警告。凭证复用的涌现行为（57%/49%）表明LLM可以自主链接多服务攻击链，这对网络防御具有直接威胁评估价值。对红队自动化工具的设计者，本文提供了模型选择（可靠性 vs. 覆盖面的权衡）和报告标准（最低N=30次运行）的实证建议。

# Questions for Further Exploration

1. Would the relative model rankings (Gemini 85% > Claude 61% > GPT-4o-mini 56% > qwen 25%) hold across different target environments with less well-known vulnerabilities?
2. How sensitive are the results to prompt engineering variations? Would a less structured prompt change the failure mode distribution or the first-move split?
3. Can the model-distinctive failure patterns (API truncation vs. premature completion vs. iteration cap) and first-phase-after-recon split (web-first vs. FTP-first) be used as fingerprints to identify which LLM is behind an autonomous attack?
4. What is the minimum conversation history window required for cross-service credential reuse to emerge? A follow-up with matched history retention across providers would disentangle architecture from context-window effects.
5. How would the zero-refusal finding change under non-authorized framings (adversarial jailbreaks, role-play, or more ambiguous professional contexts)?
6. Would re-running Claude with corrected temperature (T=0.3) and retry logic for overloaded_error significantly change its exploitation rate and strategy diversity statistics?
